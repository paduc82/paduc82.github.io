Family Safety Screen Time Windows 10 failure details and observations - Microsoft Community

chiefgeek157 say:

Family Safety Screen Time Windows 10 failure details and observations

Like many others, my family has been using Windows 10 Family Safety, and in particular the Screen Time time-limiting feature. We have multiple Windows 10 PCs and laptops (five altogether), so a solution that limits time across all devices is very useful. Like many others, we have discovered to our dismay that Screen Time, and several other Family Safety features, simply do not work.

In order for Screen Time to work, I created a Microsoft account for each member of my family (six of us) and set up logins and family membership, and Screen Time settings, and all that other stuff through the Microsoft Family web site. This post is NOT about setting up Family Safety--got that figured out. I have two adults and four children, using the Family Safety meaning of those terms.

Being the geek that I am, I tried to trace what was going on when Screen Time fails. Here is what I discovered in Windows 10. These observations probably do not apply to earlier versions of Windows, and I do not have access to earlier versions.

In the folder "C:\ProgramData\Microsoft\Windows\Parental Contols", there are several interesting folders and files. The structure appears to be as follows:

* One folder called "settings"

* One folder for each login, named for the Security ID (SID) for each user, in the form S-1-5-21-xxxxxxxxx-xxxxxxxxxx-xxxxxxxxxx-1xxx.

** Note that these folders will not appear until that user logs in for the first time.

In the folder settings, there is a file called "settings.bin". It turns out that this is a JSON-formatted text file that can be opened in Notepad. I highly recommend making a copy to the desktop first as this file gets overwritten every minute or so. Inside this file are the various options configured on the Microsoft website. In the  SID folders is a file called TimeCache.bin, which is really a binary file with an unknown format (presumably a binary date).

To the best of my observations, this file is synced FROM Microsoft TO each device, and not the other way around.

Open the file, and you will see structured data describing the daily time limits, web restrictions, app restrictions, and any "time extensions".

OBSERVATIONS

Once a child has logged in, there should be a folder named for the SID. You can figure out which child belongs to each SID using a simple command (See http://pcsupport.about.com/od/registry/ht/find-user-security-identifier.htm):

C:\> wmic useraccount get name,sid

The local username is the first five characters of the Microsoft account email (presumably with an algorithm to correct conflicts). The actual SID is specific to each machine, but the last four digits should be 1xxx. Again, on each machine the specific "user number" will be different depending on the order created.

In settings.bin, there will be an entry for each that has logged in. Again, until the child user logs in, there will not be a folder or an entry in settings.bin.

The entire Parental Controls folder can be deleted. Reboot the PC and the folder will be recreated and all settings re-synced from Microsoft. Again, each child must log in for their settings to be retrieved.

Having multiple PCs, I was able to confirm that as each child logs in, the settings are downloaded from Microsoft to each PC (with caveat below).

PROBLEMS

I logged into one of my desktop PCs using each child's log in. I found that two out of four logins received a SID folder and an entry in settings.bin, but the other two did not. The fact that the two other children's settings did not sync seems to me to indicate that their restrictions will not be enforced. WHY the settings did not get synced is beyond my reckoning at the moment.
