  1. Open a web browser.

  5. Select "Basic Settings" from the sub menu.

wunders say: You probably want to use a command like the following:
     for  /f  %i  in (list.txt) do copy /B %i  destDir
Where "list.txt" is a file that contains the list of files that you want to copy and "destDir" is the directory you want to copy them to.  If you put this line into a ".bat" file, you may need to replace each "%" character with "%%".

Or, if it is a short list of files, you could do something like:
     for  %i  in (file1, file2, file3) do copy /B %i destDir

aaron_sum say:

Hi,

Thank you for writing to Microsoft Community Forums.

The ‚ÄòSystem Power Report‚Äô generates a report of what is actually draining the system battery. Screen brightness, duration of time the screen remains active, the color contrast (mainly if you have an OLED panel), currently running processes and background processes are few things that contribute to system‚Äôs battery drain.

That being said, in a given session the process being named ‚ÄòUnknown‚Äô shows to drain a lot of power from your system. This is because, each individual device has a specific module applied across each of the components of your system. The power estimation is made by running those modules against the estimation engine and then it is finally checked with the battery driver to see how much energy is consumed. Depending on each session, the process consuming the battery and the components used in the system, this accuracy will vary from 80% to 30%, this is the reason you may sometimes see sessions with a high number of ‚ÄòUnknown‚Äô. However, this is still displayed with the relative accuracy. This just means that the system was not able to accurately measure some of the attributes draining the battery, but still collectively provided a relative report of these attributes under one name.

To get accurate results, it is a requirement to use systems which have integrated metering chips.

Hope it helps.
