If you are able to install successfully in the new user account that you created, the old user account files might be corrupt. In this case, I would suggest you to try these steps:

 

Note: You need at least three user accounts to perform this step. If you don’t have another account, you can create a new user account apart from the new user account you already created.

 

a) Log on as a user other than the new user you just created or the user that you want to copy files from.

b) Open File Explorer, click on the drive where Windows is installed, click on users, open you user account and then open “My documents folder”.

c) Click the View tab, and then click Show hidden files, folders, and drives.

d) Clear the Hide protected operating system files check box, click Yes to confirm, and then click OK.

e) Locate the C:\Users\Old_Username folder, where C is the drive that Windows is installed on, and Old_Username is the name of the profile you want to copy files from.

f) Select all of the files and folders in this folder, except the following files:

Ntuser.dat

Ntuser.dat.log

Ntuser.ini

g) Click the Edit menu, and then click Copy. Locate the C:\Users\New_Username folder, where C is the drive that Windows is installed on, and New_Username is the name of the new user profile you created.

h) Click the Edit menu, and then click Paste.

i) Log off, and then log back on as the new user.

davelkp say: Well I finally solved my problem but don't know which action achieved it or whether it was a combination of actions.  If anyone is interested here's what I did.

On the Vista machine I edited the sharing of the Public Folder, all sub-folders and files to be fully shared .

Checked the security settings for all Public Folders, sub-folders and files and set all users to Full Control.

Added "Everybody" as a user and allocated Full Control to this user.

Rebooted after each step but still couldn't access the shared folders on the Vista machine.

In desperation I edited the registry; went to HKEY_LOCAL_MACHINE / SYSTEM / CurrentControlSet / Control / Lsa
Change the value of a key called "restrictanonymous" to 0 instead of 1

Quick reboot and Hey Presto my networking is working properly

My network is secure and so I didn't mind reducing some apparent security settings.  Don't know if this is suitable for everyone but it might give a few clues.
