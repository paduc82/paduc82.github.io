Activation depends on what type of license you have of Windows 7/8/10.

If Windows 7/8/10 came preinstalled with the computer, that is called an OEM license. For an OEM license, the license dies as soon as Motherboard dies. So essentially it is non-transferable. If that is the case you need to get a new license.

However, this is not same for retail licenses i.e. you purchased the license through Microsoft store. That does carry transfer rights. If that is the case, please see the below steps:

Run the activation troubleshooter on the new device using the below steps:
Go to Settings>Update and security>Activation and run the troubleshooter.
Select I recently changed hardware on this device.

- You have done all the tricks, so I don't know why you would be having this problem, but regarding you comment....

The AMD folder in the root of C:\ would be the one the drivers get unpacked to during the install process.

This folder can be deleted manually once the driver install is completed, if DDU has not done so

(I wait at least until I've rebooted after installing the driver).

If the AMD folder has never been deleted it could waste a fair bit of drive space if it has several earlier drivers in it (Nvidia works the same way).

The only benefit keeping this folder is if you were to want to repair the driver install it would be a few seconds quicker than unpacking

the files from the installer package again (I usually keep the installer packages for drivers a couple of versions back on my backup HDD) .

- Any folders/files still in any of the three C:\ Program folders after uninstalling a driver shouldn't normally be a problem.

- Similarly there would be basic generic AMD driver files in the C:\Windows folder which shouldn't normally cause problems.

If your motherboard has an AMD chipset some of these would also be related to that.

- Or, if you want to make a clean sweep, after first uninstalling your AMD graphics drivers with  DDU,  you could try moving any

AMD graphics sub-folders you find in the three Programs folders somewhere as backup.

When you install your driver again they should get replaced (your personal game settings would have to be moved/copied back later).

If all goes well you can delete the backup folders/files later.

- If Windows blocks this action I would leave the folders rather than mess with permissions in an attempt to override Windows.

- I wouldn't advise it these days, but there was a time when I would also delete AMD or Nvidia files from Windows\System32 when I changed

graphics processor brands (as per articles advising doing this), but haven't done so since XP.
