antoneth ber say:

Hello Miha,

There will be times that your computer will run slow despite of no activities running. There are a couple of factors that you can consider when experiencing slow PC performance like start up services. I suggest that you perform clean boot to check if there are start up programs or services that could be causing the issue. To do so, please follow these steps:

  1. From Start, search for msconfig.
  2. Select System Configuration from the search results.
  3. On the Services tab of the System Configuration dialog box, click to select the Hide all Microsoft services check box, and then click Disable all.
  4. On the Startup tab of the System Configuration dialog box, click Open Task Manager.
  5. On the Startup tab in Task Manager, for each startup item, select the item and then click Disable.
  6. Close Task Manager.
  7. On the Startup tab of the System Configuration dialog box,  click OK, and then restart the computer.

Note: After the troubleshooting, boot your computer in normal mode by checking this  link and look for How to reset the computer to start normally after clean boot troubleshooting.


You may also refer to this thread for additional troubleshooting that you may want to give a try.

Do get back to us on how the clean boot goes.

Windows 7 — Understanding and fixing issues with newer Bluetooth - Microsoft Community

andy_e_d say:

Windows 7 — Understanding and fixing issues with newer Bluetooth audio devices like the Bose QuietComfort 35 and others.

I wanted to share some information that I have put together regarding issues with newer Bluetooth audio devices like headphones, speakers, and headsets that have been cropping up in Windows 7.

My name is Andy, and as of posting this discussion I work at Plugable Technologies in Redmond, WA. Plugable currently produces the #1 selling Bluetooth adapter on Amazon, the USB-BT4LE.

Problem Devices And Understanding Why They Have Issues In Windows 7

Over the past few months I have spent a considerable amount of time tracking down issues with Bluetooth in Windows 7 and finding workable solutions to resolve them. Given that Windows 7 accounts for about half of Windows-based computers in active use, this is certainly important.

Some of the devices that have been identified as having issues with pairing include:

Bang & Olufsen BeoPlay S3 Speaker
Bose SoundLink around-ear wireless headphones II (SoundLink AE2) Headphones
Bose SoundLink Color II Speaker
Bose SoundSport Wireless Earphones
Bose QuietComfort 35 Headphones
Ultimate Ears Boom 2 Speaker
Vizio S4251w Soundbar
Sennheiser PXC 550 Headphones


The primary issue with these devices is that they appear to incorporate some form of Bluetooth Low Energy (BLE) to make the pairing process easier on mobile devices, where Bluetooth Low Energy technology is fairly ubiquitous. Unfortunately, BLE support was not added in a functional way to Windows until Windows 8.

Bluetooth audio devices that add BLE typically present themselves for pairing as two separate devices. One is a Bluetooth Low Energy device, and the other is a Bluetooth Classic device. If Windows inadvertently tries to with the Bluetooth Low Energy connection (which is what happens in most cases), it may report that the pairing was successful, but in reality the pairing never completes, and the device will not allow you to select it as an audio output device.

This is why in some rare cases, the device may pair and work once, or will pair and work occasionally, then fail other times. It all depends on whether Windows tried to pair with the BLE or Bluetooth Classic signal it is detecting from the device.

Another common issue is not having the device appear for pairing at all. However, this is usually because the audio device is paired to another device already, or in cases where there are multiple Bluetooth adapters (Windows only supports one Bluetooth adapter), or when there is a Bluetooth driver conflict.

How To Get These Devices Working

To Start With â€” Learn More About What Bluetooth Solution You Have

Unfortunately, there is no silver bullet that will get the these devices working for all users in all cases. In Windows 7, Vista, and Windows XP, Bluetooth software provided with devices essentially bypassed the vast majority of built-in Bluetooth functionality of Windows in order to provide their own software known as a "Bluetooth Stack".

The most common non-Microsoft stacks for USB Bluetooth adapters are:

â€¢ WIDCOMM (BTW) by Broadcom

â€¢ Harmony by CSR (now Qualcomm)

â€¢ BlueSoleil by IVT Corporation

The most common non-Microsoft stacks for built-in Bluetooth adapters are:

â€¢ WIDCOMM (BTW) by Broadcom

â€¢ Toshiba Bluetooth Stack by Toshiba

â€¢ Intel PROSet by Intel

â€¢ Unknown by Qualcomm/Atheros

The stack that you use is largely dependent on what chip is used in your Bluetooth adapter, and what Bluetooth stack a vendor chose to include with their adapter.

Many of these stacks have not been updated with the advent of audio devices incorporating BLE. This is in large part because with later versions of Windows, Microsoft is encouraging vendors to create more basic drivers that work with the built-in Windows Bluetooth stack, rather than creating a separate software stack. Another reason is that corporate mergers and acquisitions that have resulted in these stacks being abandoned.

Solution For Broadcom Adapters Using WIDCOMM (BTW) Bluetooth Stack

I have in-depth information for this stack because it is what I regularly provide support for at Plugable.

In Windows 7, there were a few major revisions of the WIDCOMM software stack (xxxx represents minor revision values):

â€¢ 6.4.0.xxxx

â€¢ 6.5.0.xxxx

â€¢ 6.5.1.xxxx

In my testing of various revisions in Windows 7, version 6.5.0.2000 has the best behavior with these newer devices. The 6.5.1 variants erroneously pair with the BLE signal about 98% of the time, and I have had no success with 6.4.0 or 6.3.0. For Beats headphones in particular, which have a different pairing issue, the newest driver package (6.5.1.6930) can help and we have this driver package available on the product page for the USB-BT4LE.

The latest information about this will be maintained at â€” http://plugable.com/2017/02/07/bluetooth-audio-problems-cant-get-your-bose-quietcomfort-35-connected-check-here-first/

Note: If you are using an employer-provided computer, or a computer that is joined to a domain, the drivers may have issues installing. It is recommended to get the assistance of your IT staff to install the drivers. Additionally, many accounts with administrator-level privileges in these environments will still often have some restrictions that prevent the installation of these drivers.

Solutions And Information For Other Bluetooth Stacks And Devices

Intel â€” In my observation, the latest Bluetooth drivers provided by Intel for their built-in Bluetooth adapters works well with these devices.

Toshiba Bluetooth Stack â€” Providing that this stack is provided for your adapter, or can be made to work with your adapter, this stack behaves well with these newer devices despite not having updates since 2015. CSR Harmony, and Qualcomm Atheros â€” I don't have data for this personally. Please share any information you have in the discussion below.

Conclusion

Many of these issues are because of the aging nature of Windows 7, and the advent of newer devices that have been designed with modern technology in mind. These audio devices tend to work better in newer versions of Windows, and because of that I would generally recommend making a consideration for upgrading to a newer version of Windows if you are able to do so.

Here at Plugable, we will continue to provide guides and information for various issues relating to Bluetooth, be sure to check out our site for updates and information.

amaranthe say:

Windows 7 - Remote Desktop Connection - Protocol Error when a attempting to remote control a session from a Windows 2003 server in tsadmin

A number of people have been reporting this issue when remote controlling sessions from the RDP to a Windows 2003 server on the web. 

Error Details:
Program Title Window (on your Windows 7 Computer): Remote Desktop Connection
Contents: Because of a protocol error, this session will be disconnected.  Please try connecting to the remote computer again. 
Options: OK/Cancel

If you reconnect to the Windows 2003 server it brings you back right at your disconnected session.  Of course, you are not remote controlling the user you tried to when it kicked you off.

So far I know of two possible solutions.  Virtual PC with an XP VM, or running Windows XP mode.  I'll include the XP Mode instructions in a reply.
amaranthe say:
  1. Install Windows XP Mode
  2. Startup the Windows XP Mode
  3. Right-Click "Start" -> "Explore all users"
  4. Open "Programs"
  5. Right-click then select "New" -> "Text Document"
  6. Name the document "RDP - XP"
  7. Right-click the document then select "Edit"
  8. Type in the notepad window: start mstsc.exe
  9. File -> Save
  10. Close 'Notepad'
  11. On the "Programs" window go to "Tools" -> "Folder Options"
  12. Select the "View" tab
  13. Uncheck the "Hide extensions for known file types"
  14. Click "OK"
  15. Rename the "RDP - XP.txt" to "RDP - XP.cmd"
  16. Close your XP Mode (it will show you hibernating dialog)
  17. On your Windows 7 go to the Windows icon -> Programs -> Windows Virtual PC -> Windows XP Mode Applications -> RDP - XP
  18. After it starts up the Virtual application you will be able to log into your Windows 2003 Servers and Remote Control the user sessions in the tsadmin without the crazy protocol error.
