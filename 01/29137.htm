How to restore a .mig file - Microsoft Community

How to restore a .mig file

matt h - say: HI AHAWKEYE,

Here are some basic instructions on how to use easy transfer.
If you need more instructions, please follow this link.
http://support.microsoft.com/kb/928634

  1. Locate the removable media or the network drive that contains the Windows Easy Transfer wizard files, and then double-click Migwiz.exe to start Windows Easy Transfer.
  2. If you are running any other programs, you will be prompted to close these programs. You can save your work in each program, and then close the programs individually. You can also click Close All in Windows Easy Transfer to close these programs immediately. Click Next .
  3. Click Burn a CD or DVD .
  4. In Choose your media box, type the path of the CD or DVD. Click Next .
  5. Click Everything - all user accounts, files, and program settings (recommended) to migrate all the files and the settings. If you want to determine exactly which files and setting you want to migrate, click Only my user account, files, and program settings , or Custom .
  6. Review the list that contains the files and the settings that you want to migrate, and then click Start . To add or remove files or settings from the list, click Customize .
  7. Click Next after you complete burning the CD or DVD.
  8. Click Close after Windows Easy Transfer has completely moved the files and the settings.
  9. Put the CD or DVD in the destination computer.
  10. Start Windows Easy Transfer, and then click Next .
  11. Click Continue a transfer in progress .
  12. In Where did you copy your files , click Read CD or DVD .
  13. In Choose your media box, select the CD or DVD drive, locate the files and the settings, and then click Next
  14. In Choose the drives for files on your new computer , select the user names on the destination computer that match the user names on the source computer. If there is not any user account that is created on the destination computer, you may have to create new accounts. To create an account on the local computer, directly type a user name. To create a profile or a domain user, type a user name in the following format: domain\user .
  15. In Choose the drives for files box on the destination computer, select the destination drive for each source drive.
  16. Review the list that contains the files and the settings that you want to migrate, and then click Start . To add or remove files or settings from the list, click Customize .
  17. Click Close after Windows Easy Transfer has completed migrating the files and the settings.

Can anybody suggest anything else?

Possibly unrelated -- the 3 desktops can see the tablet in Network, but the tablet cannot see the desktops.

gt3455 say:

I think I have a kludgy fix, posting here in case anybody else comes across the same problem.

I was able to create a new homegroup on the new tablet computer that had never been part of the home network, and was able to join that new homegroup from 2 of the 3 desktops (not the main desktop that held the previous homegroup that I could not leave).  For good measure, I also changed the workgroup name on all 4 machines (don't know whether that had any effect).

With those changes, I was able to correctly see the other 3 computers from the new tablet and from the main desktop (even though it was now in a different homegroup).  The other two desktops were a bit quirky  -- from one I could see the other 3 computers -- twice (duplicates showing up in network).  From the last computer, I could see only the tablet.

Then I made a new user account on the main desktop and created another new homegroup there, and placed links in the libraries under that username to the places (drives) I really wanted remote access to.

Then, from the desktop that would not connect properly, I went through the new homegroup to the drives that it didn't know it could see, and mapped those (map network drives).  So now, even though the remote computers do not show up as being in the network, I can access the drives that are sitting on them.

If anyone has any suggestions on how to do this properly, I'd appreciate hearing them.  Otherwise, I believe I have everything connected at an operational level.

These errors may be caused by any of the following issues:

  • Programs or processes that interfere with Internet communications
  • Resource issues on the computer
  • High Internet activity
  • Recoverable database errors



Important In most cases, these errors are caused by small interruptions in communications between the computer to be updated and the update to be downloaded. These issues usually resolve themselves. Before you try the troubleshooting methods in this article, try one or both of the following.

  • Exit Windows Update, wait 10 to 15 minutes, and then restart Windows Update to check for updates.
  • Turn on Windows Automatic Update so that updates are installed automatically every 24 hours.
