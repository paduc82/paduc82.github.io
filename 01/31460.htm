lemp say:
Did you use the /safeboot option in msconfig?  If so, there is some malware that will prevent you from restarting in normal mode.

From frequent contributor Jose Ibarra (now posting in these forums under a different name):

There is some malicious software that if you enable the /SAFEBOOT option in msconfig, your system will never boot again in any mode (I will never suggest that idea again!) - until you get rid of the option or temporarily rename your c:\boot.ini file so you don't have one, and then you can fix it using the XP Recovery Console. 

This particular malicious software recognizes that you are using the /SAFEBOOT option to try to find it and get rid of it, so it won't let your system boot hoping that you will give up and reinstall your XP from scratch.

In a single partition configuration, XP will complain a missing c:\boot.ini file but still boot just fine (if there are no other problems).  Non believers - try it!

Here is what I would do:

Create a bootable XP Recovery Console CD and boot on that (instructions below).

Enter these commands:

cd  \                                          (back up/return to the root directory on the C drive)
attrib  -shr  boot.ini                (remove system, hidden and read only attributes)
ren boot.ini  boot.ini.bak      (rename the afflicted boot.ini file)

Remove the CD and reboot on your HDD and XP will complain about the missing boot.ini, but should still boot.

In Explorer, navigate to your C:\ folder, and rename the boot.ini.bak file to boot.ini (this is so msconfig will be able to find it).

Run msconfig again and remove the afflictions like /SAFEBOOT and save the changes, etc.  (you don't have to reboot if you don't want to).

If you want, you can just edit the boot.ini by hand, but not everybody is comfortable doing that.
