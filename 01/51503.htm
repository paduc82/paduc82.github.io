Clipboard content gets deleted after the UWP app from which it was - Microsoft Community

a. user say:

Clipboard content gets deleted after the UWP app from which it was copied is closed.

Whenever you copy some text from any UWP app, it can be pasted anywhere until the app is closed. So, essentially, after the app is closed, the copied content is deleted from the clipboard.

Steps to reproduce the problem:
   1. Copy some text from any UWP app, such as OneNote.
   2. Close the app.
   3. Try pasting the text into Notepad or anywhere else.

 

You'd observe that you can't paste anything, because the clipboard content is persistent only until the UWP app stays open. The above restriction, however, doesn't apply to classic Windows apps. 

I have also posted this issue on Feedback Hub. So, in case anyone else is also experiencing it, can upvote the post.

Additional details:
OS experienced on: Windows 10 April 2018 update.
