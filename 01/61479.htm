    dalenot say:

    Finally figured it out.

     

    When windows is installed hibernate is active... NO MATTER what you see in control panel/Hardware and Sound/Power Options/Edit Plan Settings/Change Advanced Power Settings.  Hibernate settings and sleep settings can show as disabled in these advanced settings but you will STILL have a file called "Hiberfil" in the root of C drive and it will be or was on my system ...24GB in size.

     

    You will have to un-hide all files/folders to see this file.

     

    24GB windows has set aside as used space for hibernation/sleep modes.  24GB of Wasted space!!

     

    Also I have 32GB of system memory installed.  When you install windows... by default... it sets the page file/Virtual Memory size to the same amount of memory you have installed.  In my case a little over 32GB was my page file size.

     

    SO you have windows which takes up about 12GB upon 1st install without any updates.

    Plus 24GB Hibernation and 32GB page file.

    So when you look at "My Computer" after a clean install and no updates yet.... Windows is taking up 12 + 24 + 32GB = 69GB... 69GB just for a windows install!!!  And the hard drive is only seen as 119GB.  Once you do all the "windows updates" the installation will be several GB more.   

     

    So my 119GB SSD drive will be 75% or more full just having windows installed.

     

    SOLUTION: To get rid of or disable Hibernation and reclaim 24GB of drive space....go to Start>Programs>Accessories and right click on Command Prompt and select "Run as Administrator".

     

    THis runs the command prompt function in "Elevated Command Prompt" mode. 

     

    Your Command Prompt window should read as  C:\Windows\System32 in "Elevated" mode.

    If your Command Prompt window reads similar to C:\Users\Tony...... then you are not in elevated mode.

     

    Type the following in the command prompt window to disable Hibernate: powercfg -h off   and press enter.

    there is a space after the 'g' in powercfg and also a space between the "-h" and the word "off"

     

    The "Hiberfil" file in your root of C will now be gone and wah-la.... you have 24GB more free space on your drive now.

     

    SOLUTION #2: Right click on "My Computer" and select properties".  On window that pops up on left side column... click on "Advanced System Settings".  The window that pops up will be the "Advanced" tab.  Click on settings in the performance section of that tab. On the window that pops up this time click on "Advanced" tab again.  At bottom under "Virtual Memory" click on the "Change" button to change your virtual memory settings.

     

    In my case I stopped windows from managing the virtual memory and changed the size of virtual memory from window's setting of 32GB because I have 32GB system memory installed.... I downsized my virtual memory to 8GB which is still a huge amount! 

     

    But by doing so I recovered another 24GB of drive space having my virtual memory size set to 8GB.

     

    So now once you do both of those actions, you can go to "My Computer" and see that now you only have about (in my case) 20GB of used space instead of the original 68 to 69GB of used space.

     

    So now I have right at 98GB free space on a 119GB drive... with the windows installation only taking up about 20 to 25 GB of space as you would normally see.

    Before... I had only about 45GB of free space (after updates).

     

    TURN OFF hibernate and make sure you don't have an ungodly amount of virtual memory set aside for windows usage!!
