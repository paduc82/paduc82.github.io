    Okay found the problem and the solution.

    I found that all msi packages would fail.  I tried to install another package as a .exe and received the error:  "the folder path 'program files (x86)' contains an invalid character".  

    This lead me to wonder about long file name support.  After searching I found the registry key "Win31FileSystem" was set to 1.  This system was originally setup to be able to save files to a Windows 3.1 embedded computer (don't ask). It turns out that having this turned on disables or is incompatible with the naming of "Program Files (x86)"...  Setting this registry key back to 0 and restarting resolved the msi problem.
