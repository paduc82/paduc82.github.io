Many updates require restarting the device. Save your work and close all open applications.

Next, select the Start button, go to Start / Shutdown, and select Update and restart or Update and shutdown.

This could solve the audio problems automatically.

Now, to run the problem solver, select the Start button, type the audio problem solver, select Find and solve audio playback problems in the results list, and then choose Next.

You should also check the Device Manager to verify if the audio driver is disabled, or not available for this update. Try these steps:

• Select the Start button, type Device Manager and then select Device Manager in the results list. In Sound, video and game controllers, select the sound card, open it, select the Driver tab and then Update driver. If Windows does not find a new driver, look for one on the device manufacturer's website and follow its instructions.

• If that does not work, try the following:

• Uninstall the audio driver: open Device Manager, right-click (or press and hold) on the audio driver and then select Uninstall. Restart the device and Windows will try to reinstall the driver.

And, if that does not work, try using the generic Windows audio driver. This is the way to do it:

• Open Device Manager, right-click (or press and hold) on your audio driver> Update driver> Search for driver software on your computer> Let me choose from a list of device drivers on my computer, select Device from High definition audio, then Next and follow the instructions to install it.

0x8007054F stands for ERROR_INTERNAL_ERROR which isn't much help except saying the error is unexpected by the installer.

The component in the log that throws the error out is identified as {844EFBA7-1C24-93B2-A01F-C8B3B9A1E18E}, which is Microsoft.VC80.CRT. Good news is this component is well-known, you can get more information by downloading Visual C++ 2005 redistributable and run it in verbose logging mode. 

See http://blogs.msdn.com/b/astebner/archive/2007/02/07/update-regarding-silent-install-of-the-vc-8-0-runtime-vcredist-packages.aspx for instructions, but append the command line with /l*v %temp%\vcredist_x86.log" to enable verbose logging. After collecting the logs you can share it on the internet and post the link back.

If you get the error with other component too (not limited to the one identified by the thread starter) then the issue is most likely to the Windows Installer engine, like crashed Installer or power loss caused registry keys under HKEY_LOCAL_MACHINE\COMPONENTS to be in inconsistent state.
