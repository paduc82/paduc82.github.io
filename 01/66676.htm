a. user say:

The actual executable file for the Inbox Repair Tool is scanpst.exe which is loscated in C:\Windows\Program Files\Microsoft Office\Office12.  Login as an administrator.  If when going there you receive an access is denied message along the way, (perhaps even at the Programs Files folder), right click on the folder and click on Properties and then select the Security Tab.  Add yourself as an authorized user and give yourself Full Rights permissions.  Then it should let you pass on to the next folder.  I don't think you'll have any further access is denied problems because permissions for the sub-folders are probably set to be inherited from the parent folder (at least that's how it worked when I set permissions to access the folders on my own system).  Once there, scroll through the long list of files to scanpst.exe and double-click on it.  It should open a box asking you to enter the inbox you want to scan and provides a browse button so you can locate your Outlook.pst file and select it and then run the program.  Hopefully with giving yourself the permissions to access the file, you won't receive an access is denied message when attempting  it this way.  I'm not certain if this will resolve the problem, but it is a tool to help fix problems with PST files so it very well might work.

I think the reaon you couldn't open Outlook.pst directly is because it is set to be only opened by the Outlook program  itself - but double-clicking the PST file should have resulted in opening Outlook instead of giving you an access is denied message.  Were you logged in as an Administrator when you tried?  You can try the same procedure as above with the folder in which the Outlook.pst file is located (starting with the beginning of the file path as before) until you receive an access is denied message and then changing permissions as described above.  The it may open Outlook when you open it as it should - though that shouldn't be any different from opening Outlook directly unless the path to the correct Outlook.PST file has somehow been changed in Outlook's settings and so it can no longer find it.

One solution to the file size problem is to empty the deleted items folder and Archive some of your older posts (maybe those older than a few months or whatever timeframe you feel comfortable doing.  You can then also open the Archive.pst file in your available folders so you'll have quick and easy access without needing to open the Archive.pst file each time.  Here's the procedure: http://email.about.com/od/outlooktips/qt/et_archive_mail.htm (it may vary a bit based on your version of Outlook, but it should be enough to guide you through the process no matter what version you are using).  This is also a good idea because it makes searches for emails much faster as the file is much smaller.  I personally archive anything older than 4 months old and I find I rarely need to go to that file to find anything as most of what I deal with is only current stuff. You can even select which folders to archive in case you want to keep the entire history of a particular folder but don't mind archiving the others.

You can also change the maximum size limit of the PST and OST files as described in the following article: http://support.microsoft.com/kb/832925.  While this may solve the problem for a time (until you hit the new limit again), I think your best solution is to use the archiving process as described above.  When files become very large, they are more prone to be affected by some minor corruption - keeping the PST files smaller helps prevent this from happening.

If the above doesn't help, then please follow Michael Murphy's advice and repost your question in the Outlook Forum  at http://answers.microsoft.com/en-us/office/default.aspx?tab=3#tab=3.

I hope this helps and that you don't need to go to the other Forum, but certainly feel free to do so if it doesn't or if you ever have any other questions or problems concerning Outlook.

Good luck!

FAT16 was the only variety used for hard drives, so just calling it
FAT was sufficient.

It was used almost exclusively for floppy disks.

FAT16 was used on hard drives (or partitions) up to 4GB, and had a
cluster size that varied from 2kb to 64kb, depending on the size of

j w stuart say:


In round numbers XP, Vista and Windows 7 32-Bit Versions of the Operating System can only address 4GB. Does not matter how much memory you have installed 4GB is the max. From that max your video card has XXXMB of ram on the card so that limits the operating system to 4GB minus the cards XXXMB of memory.

 

In addition other hardware also takes away additional amounts of memory (usually a small when compared to the video card)
and the remaining memory (address space) is available for Windows XP, Vista or Windows 7 OS and your applications.

 

So that 4th GB thru 16 GB of ram you installed or will install goes unused on a 32-Bit OS. Does not matter what the BIOS reports or what the OS shows as installed memory it still limited to that 4GB address barrier.
