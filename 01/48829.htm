Does Windows perform CRC when copying files between drives? - Microsoft Community

paasc say:

Does Windows perform CRC when copying files between drives?

Hi.

I have a habit of computing and comparing CRC checksums when copying large or important files between drives. I use one of 7-Zip's context menu functions, mostly CRC-32, and compare the checksums of the file at the source and destination locations.

In some (very) rare cases, the numbers don't match and I copy the file again until they match. I'm suprized that this can happen. Doesn't Windows perform it's own CRC tests when copying files? And if it does, how does it deal with errors?

Why are there sometimes still CRC mismatches when Windows claims copying had been successful?

robertjwin say:

Not sure, but I guess Windows simply reports to the user whatever the drives tell it, i.e. whether there was any error while copying from the source or to the destination but not whether any bits got accidentally flipped during the process. Verifying (using hashes) each copy operation would surely result in a performance hit, which could be significant depending on the hardware.

Third-party software such as TeraCopy (which can register itself as the default copy handler) and xxcopy are able to perform the verification you're looking for.

a. user say:

Windows does not perform error detecting when copying files. That would introduce extra overhead that would slow a computer's performance. Besides, hardware and software have advanced to the point where error detecting routines are unneeded for file copies and moves.

Error detecting routines are still very much in use when downloading from and uploading to the internet, due to the inherent uncertainties of transferring data over a packet-switched network.

That said, when I transfer huge (for me) and critical files from place to place, like my wedding video, I calculate and compare SHA-1. I probably don't need to do it, just being superstitious.

And, you know, CRC is so passe. Everyone uses MD5 and SHA-1 now, and even MD5 is falling out of favor.
