mike hall... always say:
Low space error for which drive? C? D?
 
If the space was truly freed up, the warning would go away
 
The system will report that D, most likely a recovery partiton,  is low on space if you have scheduled backups in Windows Backup.
 
The problem is that Windows Backup does as it should by selecting a partition or drive other than C, but does not take into account that the only other partition or drive may be a recovery partition which has been sized large enough that Windows doesn't think that it is over full, but small enough that it is not wasting space. 
 
The above is the most common cause of the low disk space warning.
 
If you have upgraded an aging XP system, it could be your C drive which is squawking. Windows 7 takes up more space than XP. Yes, the advertizing tells you that it is not full of unnecessary stuff, but that is in relation to Vista, not XP.
 
Windows 7 plus whatever you install by way of applications and other stuff exists well on a partition size of 80gb or more. It needs to be this size to allow for the OS, whatrever you put on it, updates and patches that will be released, space for a decent pagefile size PLUS free space such that the system has room to move, switch around, defrag etc.
 
For best disk performance, 25% free space is a very good thing, and is not a waste of drive space at all. The OS needs free space.
 
If your partition/drive is less than 80gb, consider increasing it if thehard drive will allow, or cloning it all onto a larger hard drive.
 
To see what is on your partitions/drives, use this utility
 
Treesize..   http://www.jam-software.com/treesize_free/

Why does File Sharing wizard "Choose people to share with" force Everyone to be included?

This network has mixed Windows 7 Pro and Windows 8 Pro PCs.

When I right-click on a folder in a Windows 8 Pro PC and select Share with, the Choose people to share with dialog box lists Administrators and that PC's username.

When I do the same thing on all the Windows 7 PCs -- except for one! -- the list consists of that PC's username and sometimes Administrators.

But there's this one Windows 7 Pro PC that lists the PC's username, Administrators and Everyone (Read/Write). This PC has one administrator user (where I'm doing all the folder sharing) and 3 standard users (who exist only so they can access specific shares on this PC from their own PCs). These 3 users want to keep their shares private from each other, so when using the sharing wizard I delete Everyone from the list of "people" and add the appropriate user.

The problem? Once I remove Everyone from the list the share becomes inaccessible to the specified user. If I leave Everyone present then all users can access this share.

Why is Everyone a requirement for sharing a folder on this PC?  Incidentally, I created a new Admin user on this system to see if the original admin profile is corrupted, but the new user profile exhibits the same behavior.


vinnietestosterone say:

Here's the solution:

Note I had been trying to share folders on the "D:" drive.

I opened the Properties of the D: drive and looked at the Security tab. Under "Group or user names" there was only one entry: Everyone

Normally, four Groups are listed.

I added those Groups, with the appropriate Permissions, and now I can share folders on the D: drive using the Sharing Wizard, remove Everyone from the default list, add the desired user, and it works as one would expect.

Not knowing why the NTFS Security settings had been set that way I'm leaving it still in the list because I don't want to break anything.
