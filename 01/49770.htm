This type of startup error message is something you get when an antivirus program deletes a malware file but fails to remove that file’s “startup entry” in the registry, or in a scheduled task. This particular error message has actually been a huge issue for about 4 months now – the “main” thread for the issue has over 400,000 views, and still counting. We learned 2 important lessons from the main thread:

http://answers.microsoft.com/en-us/protect/forum/protect_scanner-protect_scanning/run-dll-error-messagebackground-containerdll/49612202-667e-4a71-8e9a-d02161d8bc19   

1.) We learned how to remove the BackgroundContainer.dll startup error message with Autoruns – directly, and safely, by using the “File > Find” option.

2.) We subsequently learned that the “startup entry” that Autoruns was finding for this file was in fact a scheduled task named “BackgroundContainer startup” that was left behind when the rest of the Conduit Malware was removed. That information allows us to completely bypass the use of Autoruns and simply delete the orphaned task that’s still attempting to run the missing BackgroundContainer.dll file at startup. So we moved very quickly from a complicated, “multiple-download” remediation procedure to a simple fix that doesn’t ask you to download anything at all, and that only takes about 1 minute to complete:

How to get to the Xbox Startup Troubleshooter

  1. Unplug the network cable if you're using a wired network connection.
  2. Power off your console, and then unplug the power cord to ensure that the console is completely powered off.
  3. Wait 30 seconds, and then plug the power cord back in.
  4. Press and hold the BIND button (located on the left side of the console) and the EJECT button (located on the front of the console), and then press the Xbox button on the console.
  5. Continue holding the BIND and EJECT buttons for 10-15 seconds.
  6. Listen for two “power-up” tones a couple of seconds apart. You can release the BIND and EJECT buttons after the second power-up tone.
