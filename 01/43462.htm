Hi Adam,

This issue could occur due to some incorrect settings or due to an issue with the Audio driver. If the file plays in the Music app, then the issue must be with the Windows Media player. In this case, I would suggest you to run the Windows Media player settings troubleshooter and check. Refer to these steps:

  1. Press Windows key +F to bring up the search Charms bar.
  2. Select settings in the drop down menu below search and type Troubleshooting.
  3. Click on Troubleshooting and select the view all option on the left pane.
  4. Click on Windows Media player settings troubleshooter and follow the prompts.

If the file doesn’t play in the built-in Music player as well, then the issue might be with the Audi device or its driver. In this case, I would suggest you to run the Audio troubleshooter and check. Refer to these steps:

  1. Press Windows key +F to bring up the search Charms bar.
  2. Select settings in the drop down menu below search and type Troubleshooting.
  3. Click on Troubleshooting and select the view all option on the left pane.
  4. Click on Playing Audio troubleshooter and follow the prompts.

If the Audio troubleshooter didn’t detect any incorrect settings, then it is possible that the Audio driver is either corrupt or incompatible. I would suggest you to install the latest Audio driver from the computer manufacturer’s website and check. Try these steps and let us know the results.

    1) Click the Start Button on the Desktop.
    2) Right click on 'Computer'.
    3) Click on the 'Properties' option.
    4) At the bottom of the window click on the Change product key link and enter your Product Key.

    Hi Tony,

    Please do the following if you have a desktop computer. If you have a laptop, skip to section 2.

    Open the Windows Search tool, copy/paste the following and then click Search: qmgr.dll

    Search should show that file in the following locations:

    C:\WINDOWS\system32
    C:\WINDOWS\ServicePackFiles\i386
    C:\WINDOWS\system32\bits

    It may only be present in C:\WINDOWS\system32, but provided you can see it in at least one of the locations, do the following to register the service.

    Go to Start, Run and copy/paste: regsvr32 qmgr.dll and click OK
    (hit Windows key + R if you can't see Run on the Start menu)

    Do the same with this file: regsvr32 qmgrprxy.dll

    Reboot the PC and then copy/paste this command into the Run field:
    %windir%\System32\rundll32.exe setupapi,InstallHinfSection DefaultInstall 132 %windir%\inf\qmgr.inf

    If you're prompted to insert the Windows installation CD, ignore that and in the dialog box at the bottom, change the path to C:\WINDOWS\ServicePackFiles\i386

    If you could only see qmgr.dll in C:\WINDOWS\system32  change the path to that instead.

    Restart the PC again and then go to Services and you should be able to see BITS in the list.

    Double click it and then on the General tab, click the Start button to try and start the service manually. This is just a test to ensure that it's working correctly.

    If you get Error 2: cannot find the file specified, go to Start, then Run again and copy/paste the following command and then click OK:
    Reg add HKLM\SYSTEM\CurrentControlSet\Services\BITS\Parameters /v ServiceDll /t REG_EXPAND_SZ /d %windir%\System32\qmgr.dll

    Reboot again and then try once more to start BITS. It should work properly now, but in the unlikely event that it fails again, run the following command and then hit Enter:
    Reg add HKLM\SYSTEM\CurrentControlSet\Control\BackupRestore\FilesNotToBackup

    Reboot once more and then do the following.

    Go to this link and install the  IE8 cumulative December update:
    http://www.microsoft.com/en-gb/download/details.aspx?id=41404

    If you use IE7, use this link instead:
    http://www.microsoft.com/en-gb/download/details.aspx?id=41442

    Finally, go to the Windows Update site and you should be able to download normally.

    tryx3 say:

    You seem to have 4 choices:-


    1  Make a Multi-Edition installation disk based on the Win7 Pro N ISO [and accept the limitation that the disk can only be used for installation and not for the full range of repair tasks that a normal installation disk can do].

    If you use the Windows 7 Professional N x64 English X17-59337.iso [from Digital River so it is a valid download - I found it listed in the index provided by MyDigitalLife] then you make the installation USB using  MS download tool as normal.  Then you go to the USB's Sources folder & delete the file ei.cfg.  When you run the installation, it will stop and ask you which Edition you want to install, Ultimate N will be one of the options in its list.

    Later on, you'll also need to download the Media Feature Pack to get WMP etc but that is something you will have done before, I imagine.


    2  Make a system image & a System repair disk [as a means of recovery] then just try an installation disk of the standard Ultimate Edition x64 Windows 7 Ultimate (x64) [with the risk of having wasted your time]

    You will need the System repair disk so you can be certain of being able to use it as a tool to boot from to restore your previously saved system image whatever condition your system ends up in as a result of having tried this option.


    3  Use the activation helpline.  Enter slui 4 in the Start menu search box then follow it through to get the activation helpline for your region.  When you call , press no buttons at all - stay on the line and a human operator will eventually answer.  Explain your dilemma and ask if you can use the Windows 7 Ultimate N Product Key to activate Windows 7 Ultimate.


    4  Transfer your existing Ultimate N installation disk to USB.  Using somebody else's PC, you can make an image of that disk using several utilities including Paragon Backup & Recovery Free Edition.  You can then use the same utility to restore that image to a USB and the USB created will be bootable & full functional.
