josé oliver-didier say:

When you say within the application I am assuming you are refer to metadata which only is stored in the Windows Photo Gallery local database and is not written back to the file.

First, you should understand how Windows Photo Gallery works. WPG stores the file information it reads to its local database for quick access in a process generally referred to as indexing. The information which comes from the image file gets precedence when creating this database. Microsoft used to use the term "The truth is in the file" to explain this. 

Now, most information which you edit (Captions, Descriptive Tags, GeoTags) in WPG will be stored in the database first and then an attempt will be made to write it back to the file metadata. I say an "attempt", because a file may be set to read only or may not support metadata (such as png). If the information cannot be written to the file, WPG will keep the information in the local database only.

Now, generally speaking, WPG is "good" in supporting industry metadata standard metadata standards such as EXIF, IPTC and XMP in a basic sense. Reading up on these standards will help you better understand how this works. A good article can be found here.

In Windows Photo Gallery: Captions, Descriptive Tags, People Tags, Geo Tags, Star Ratings, Dates, Author and most information which is editable in the Windows File Properties dialog will get written to the file and is accessible if another app reads the location in which this information is kept. To my knowledge, only the WPG "Flags" are not written back to the file.

Even though there are industry standards, every app I have stumble upon may write and read file metadata a bit differently. I often use a command line tool call exiftool to read and examine the file metadata (There is also Exiftool GUI which is an easier graphical version). 

My opinions here came from using Windows Photo Gallery over the years, Hope this helps.
