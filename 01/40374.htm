Windows anniversary update installation problem unnamed incompatible ap

kafkaschimp say:

Thanks to various German websites, I solved the problem of trying to find out what the app was, and since quite a lot of people seem to have had this problem, here are the steps needed:

After the update has stooped, and you have cancelled it:

1. Put in the File Explorer location bar (or use start button +R): C:\$WINDOWS.~BT\Sources\Panther

2. This will get you into the folder you need.

3. Find the most recent CompatData......xlm file, and view it in the preview or open it in Notepad

4. The very last entry will look something like this:

</DriverPackages><Progams><Program

Id="000058ebb4dc35c5076f0b8d1d8e34d797f80000ffff"

IcinId="000058ebb4dc35c5076f0b8d1d8e34d797f80000ffff"

<CombatibilityInfo BlockingType="Hard" StatusDetail="Upgradeblock"/><Action Name="ManualUnistall"

DisplayStyle="Text" ResolveState="NotRun"/></Program></Programs></CompatReport>

This tells the computer that the app is not compatible, tells it to block the upgrade (Upgradeblock), includes the info to manually uninstall, and says show the box that I illustrated in my original post.

However, it also tells us the id of the program (000058ebb4dc35c5076f0b8d1d8e34d797f80000ffff).

5. Copy it.

6. Now go back to the Panther folder, and open .........(a series of letters and numbers)_APPRAISER_HumanReadable.xlm  (I used Adobe InDesign, but any web creation program should work) so you can read it easily.

7. In the program you have opened, do a search for the id number you have found in step 4.

8. When it finds the id number, you will see nearby the actual name of the associated program.

Here's an example for Expression Design 4:

  <Property Name="Name" Value="Microsoft Expression Design 4" />
        <Property Name="OSVersionAtInstallTime" Value="10.0.0.10586" />
        <Property Name="ProgramId" Value="00005fd8e6e994a5c55a3ce29d01fe5294b600000904" />
        <Property Name="ProgramInstanceId" Value="0000bd2cc5064cd9011bad8f87eb8d8d55366b50ecbb" />
        <Property Name="Publisher" Value="Microsoft Corporation" />

9. Now you have the name of the program, you can go and uninstall it manually, restart the computer, and then do the upgrade again.

In my case, the update had been cancelled on id="000058ebb4dc35c5076f0b8d1d8e34d797f80000ffff", and this turned out to be Windows Mobile 6.1. I uninstalled it, and had no problems then installing the upgrade.

Others have found that Windows Mobile prevents the upgrade, so if you have it installed, simply uninstall it without going through all these steps! However, it has been reported that other programs have similarly prevented the upgrade, and this method should work in exactly the same way for those programs.

Hope this helps others with the same  experience.

Mark

(One oddity. When I did that search for the id number, I actually go two hits for the same id number: one for Microsoft Office 2016, and one for Windows Mobile. Since I assumed that Microsoft would not be so contrary that Office would be preventing the upgrade, I assumed it was Windows Mobile, and so uninstalled that).
