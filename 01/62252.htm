UEFI, SSD and Prefetch - Microsoft Community

drl15son say:

UEFI, SSD and Prefetch

I installed Windows 7 64bit Pro in UEFI mode on an Intel 520 SSD.  I have a hard drive for storage.  The SSD Tool Box says that Prefetch should not be enable.  Is this correct?  Should it have a paging file?

patrick barker say: Hi,

Prefetching is not necessary on an SSD due to the low access times. It'll increase your SSD's overall performance to disable it. By default, Windows 7 will disable Superfetch (different than Prefetch), ReadyBoost, as well as boot and application launch prefetching on SSDs with good random read, random write and flush performance. These technologies were all designed to improve performance on traditional HDDs, where random read performance could easily be a major bottleneck. 

 

Regards,



Patrick

bossdweeb say:
  Should it have a paging file?
- A lot of people suggest the Page File is redundant when a machine has a lot of RAM (8GB or more),
but the Page File is still viable. My understanding is the PF is used by System Restore ( System Restore points are kept on the C:/ drive, however), and it is still used by some software and games.
- If a software uses the Page File and you do not have one you will get an error to the effect
that there is not enough RAM.
You can setup the Page File on your HDD instead of the SSD to reduce writes to the SSD.

- Others may also argue that doing regular backups negates the need for System Restore.
Personally - System Restore is your friend. Even with System Restore you may sometimes have
to reinstall something, or redo some settings after the restore, but System Restore is more
imediate than doing regular back-ups and can save time vs having to do a full backup restore.

I've been using a 2GB min and max for the Page File for a while now and it seems to work well.

The search index only includes your selected locations. These locations can be filtered for what file types, file properties, and file contents you want yo be indexed. Here are the steps that you can follow to rebuild search index in Windows 10:

1. Open the Control Panel (icons view), click/tap on the Indexing Options icon, and close the Control Panel.
2. Click/tap on the Advanced button.
3. In the Index Settings tab, click/tap on the Rebuild button under Troubleshooting.
4. Click/tap on OK to confirm.
5. Indexing will now be in progress.
6. When indexing has completed, click/tap on Close.
