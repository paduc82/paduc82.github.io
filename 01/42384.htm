carey frisch say:

I had similar issues with an HP Pavilion a1748x.  In the course of troubleshooting, I replaced all internal components (RAM modules, hard drive, video card, CPU... even the PSU) except the motherboard.  The intermittent blue screen issues continued.  After removing and replacing the RAM models, it boiled down to this:  the second bank of RAM slots (Nos. 3 & 4) were defective.  I could use RAM installed in banks #1 and #2 and never experience a blue screen, but whenever banks #3 & #4 were used with additional RAM modules...BAM... random blue screens!  Since there is no way to fix this (outside of replacing the proprietary HP motherboard), I run with RAM installed (total 2GB) in the #1 & #2 RAM slots only.

Yes, motherboards do go bad and the exact cause is oftentimes very difficult and time-consuming to troubleshoot if the issue is intermittent.

Hi,

Either the drive is failing or the file system is corrupt.

If the drive is failing, then replace the drive. It is malfunctioning and will only get worse, you cannot fix this. You can verify if this is the case by downloading and running a drive diagnostic from the drive or system manufacturer. These are usually free and run from a bootable disk or thumb drive.

If the file system is corrupt, save what data you can from it and then delete the system volume and rebuild it from disk manager (diskmgmt.msc), do not just reformat. Only by rebuilding it can you correct the problems with the file system.
