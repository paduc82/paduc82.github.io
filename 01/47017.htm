In windows XP it was very easy it was part of the Folder Options dialogue.

fppilot say:

This appears to be a problem with the drivers for USB connected devices from Saitek/Madcatz company. The devices are game controllers such as flight yokes used in flight simulations.  The issue occurs when the Saitek USB are unplugged from the system when the system is running.  The only solution found so far is to open Control Panel, then Device Manager, expand the HID section, locate the Saitek devices, and one by one (for as many as are connected) right-clicking and choosing to disable them.  Then they can be safely unplugged without the issue that affects the mouse.  NOTE: it is necessary then before using these devices again to of course plug them back in, and go back into Device Manager and enable them.

This disable/enable routine is not what I would consider to be a permanent solution to this issue.

The only other solution is to use the devices, and when the simulation or game is over, shut down the system before unplugging them.
