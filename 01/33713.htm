You didn't say what games your have getting low frame rates with, but Intel HD4400 Integrated graphics is not

a dedicated graphics card, and is not capable of high frame rates with system intensive games.

A cpu running @2.29GHz is also too low for high spec games, or to use high settings on games it can run successfully.

One reason Integrated graphics does not perform as well as an dedicated graphics card is it has no VRAM.

Integrated graphics uses some of the system RAM, which is not as fast as VRAM.

Check the benchmarks for the games in the reiview below -

https://www.notebookcheck.net/Intel-HD-Graphics-4400.91979.0.html

games you have, or wish to play. This will give you some idea of if the game will run well on your HD4400 graphics processor.

Unfortunately most games don't list integrated graphics with their system requirements, so it can be difficult at times to

know how well game X will run on your HD4400.

Your Ping is also too high for good online performance. Even if the game were to run smoothly high ping will create lag

when online, which will make things difficult for you. You would do better to try and find servers with a ping of between 30-50ms,

but may do alright with some games up to 100ms or so, if they don't require quick reactions and responses such as with fps games.

a. user say:

Hi Shirley,

I will be glad to help you with the issue you have with the folders.

The main problem is that the Application Data 'folder' is not a folder, it is called a Junction.

Some of the system folder names were changed in Windows to assure compatibility with older programs that are hard coded to look for these former names, Windows uses Junctions with the older name to redirect these older programs to the new location.

The C:\Users\Den\AppData\Local\Application Data junction redirects to: C:\Users\Den\AppData\Local.

These Junctions have the permissions set to Everyone = Access Denied. They cannot be used as a folder to store any files.

This problem occurs when a user changes these permissions in an effort to force access to that junction. Changing the permissions allows any action to 'recursively walk through' the junction and this creates a recursion (Also called an infinite loop) the result of which is what you are describing.

Hope this information is helpful.

Thank you.
