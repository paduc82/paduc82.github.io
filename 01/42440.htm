microcompsunltd say:

First let me address the confusion you have on the restarting procedure delays.  By default Windows 8 and 8.1 have on what is called Fast Startup.  This only occurs when you shutdown the computer, and does not occur upon doing a restart.  A restart is exactly that, a full restart.  However, when Windows 8.1 has it's defualt set, shutting down the computer actually does what is called a hybrid shutdown.  It does this by performing a partial hibernation which results in the kernal and device driver files being saved to the hard drive, similar to what a hibernation would do, so the computer can start back up faster when you turn it back on.  Due to this, your startup is actually faster when you do it after the computer has been shutdown, over doing a restart which saves nothing and starts everything fresh.

Now to determine what programs are starting up, right click on your status bar, and left click on Task Manager.

Click the startup tab

Here you have listed all the programs that start up during boot, along with a column that shows you what impact they have on the startup portion.  You can also disable easily any of these by clicking on one and then clicking Disable.  In Windows 8 and 8.1 this is the preferred method to turning on or off startup programs, versus the old msconfig.

I always suggest that you first turn off all high impact programs that are not driver related.

The difference is just the type of processor. At a basic level, it's how many paths are available in the processor for simultaneous computing.

64-bit computers are more modern, allow you to take advantage of more memory, and are faster.

Programs built for 32-bit computers will work fine on a 64-bit computer because Windows has the ability to mimic a 32-bit computer (they call this WoW, or "Windows on Windows").

Today it's nearly impossible to buy a computer that isn't 64-bit, and I would not recommend it even if you had the choice.

hairyfool say:

The primary difference between 32 bit and 64 bit Windows is the address space available so how much memory can be used

A 32 bit address bus can only access a maximum of 4GB memory. If you were to install more the excess would be ignored. Due to other technical issues even with 4GB installed if you were to look at the system properties you would find that is only has around 3.2 GB available (there are slight variations).

64 bit can access up to 16 terabytes of memory although different versions of Windows have lower limits. This means that with more memory applications can remain in active memory instead of having to be swapped out in virtual memory.

There are some true 64 bit applications although the majority in active use are 32 bit natively, the program specification should make it clear if there is a particular need.

However the minimum specification for 64 bit Windows is 2GB vs 1 for 32 bit. This has to be remembered because if you have only 4GB of memory although it would all be used with 64 bit the extra memory demand would wipe out most if not all the extra. azeez n say: Hi andyandlily,

Thank you for using Microsoft Forum

Windows Mail stores your email, folders for newsgroups, and account settings in its own storage folder. By default, this storage folder is placed on the C drive, under your username in the AppData\Local\Microsoft\ Windows Mail directory.

In vista and windows 7 the path is C:\Users\username\AppData\Local\Microsoft\Windows Mail storage folder.

I suggest you to try to below steps to backup messages from windows mail and export them to windows live mail.

Steps :

Note: This method uses the traditional 'Drag and Drop' approach. Some may find this approach more familiar or easier.

Drag Messages from Windows Mail to an Explorer Folder

1.      Create a new folder using Windows Explorer on the desktop(e.g. WMDragEmail)

2.      Create sub-folders in the new folder with identical names as the folders in your Windows Mail mail account
- Inbox, Outbox, Sent, etc

3.      Open Windows Mail

4.      Resize each window to fit on the desktop(one for the new folder and one for Windows Mail)

5.      Switch to Windows Mail and select messages from a Windows Mail folder e.g Inbox
- Control A can be used to Select All

6.      Drag those messages to the identical named sub-folder in the earlier created Windows Explorer folder.

7.      Repeat for each Windows Mail folder(Outbox, Sent,etc)

8.      Close Windows Mail

Drag Messages from an Explorer Folder to Windows Live Mail

1.      Open Windows Live Mail

2.      Resize again to show two windows--the Explorer folder and Windows Live Mail

3.      Ensure an email account is created

4.      Drag the messages from the Windows Explorer folder to the like named folder in Windows Live Mail
