4. If Required enable USB Boot

6. Save BIOS changes, restart your System and it should boot from the Installation Media

Windows 10 1803 update caused system image creation to fail - Microsoft Community

grickm say:

Windows 10 1803 update caused system image creation to fail

I recently updated an older system to Windows 10 1803. The update caused the 450 MB Recovery Partition to end up with 431 MB used. Since the volume shadow copy function requires at least 50 MB free space on this partition, the system image creation tool to failed with error 0x80780119, insufficient storage to create a volume shadow copy. Since disk management GUI offers no access to the recovery partition, I had to use diskpart to assign a drive letter to verify that the partition was too full. I then used disk management to shrink my C: partition by 100 MB, then a 3rd party tool, because disk management only expands to its right, to expand the recovery partition into the 100 MB that was freed up. The disk layout is [system][c: (Windows 10)][recovery partition][d:]

With the resized recovery partition system image creation proceeded as expected.
