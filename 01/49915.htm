Windows xp sp3 system root error Hal.dll corrupt or not found after - Microsoft Community

robertmcneil say:

Windows xp sp3 system root error Hal.dll corrupt or not found after auto system update windows won't load.

On 7-12-2011 my system automatically downloaded a system update and rebooted when it restarted it would not load windows and gave this error message. What do I do? I have tried removing the hard drive and putting it into another computer so I could look at the problem but I am not sure what to do to correct it. My backup information is on another hard drive in the original computer. I use Norton 360 for my backups.

a. user say:

It is extremely unlikely there is anything wrong with your hal.dll file in spite of what the error message says (do not try to replace it). 

It is extremely likely that XP just can't find the hal.dll fine where it is being told to look for it.

XP might not be able to find the file is the file system is corrupted (Norton does that sometimes), or if your c:\boot.ini has somehow become corrupted and is telling XP to look for files in never never land.

You should first run a chkdsk /r on your afflicted HDD and if that doesn't do it, then you should rebuild your boot.ini file.

If you have the drive in another machine, you can open the c:\boot.ini file with a text editor and copy/paste the contents back here for analysis.

You can also rename your c:\boot.ini file to something like c:\boot.ini.bak (so now you don't have one).  In a single partition installation, XP does not even need a c:\boot.ini file to boot.  It will complain briefly, but still boot (non believers - try it).

t may take a long time for chkdsk to complete or it may appear to be 'stuck'.  Be patient.  If the HDD light is still flashing, chkdsk is doing something.  Keep an eye on the percentage amount to be sure it is still making progress.  It may even appear to go backwards sometimes.

You should run chkdsk /r again until it finds no errors to correct.

I would keep an eyeball on your Norton since when your system loads up again, Norton may corrupt your file system again and you will have to repeat the chkdsk /r to fix it (then you can fix Norton).

I would put the drive back in the original system and fix it there using the XP Recovery Console.  After all, that is where it needs to be working, right?

If you have no genuine bootable XP installation CD from which you can boot the XP Recovery Console CD, you can make a bootable XP Recovery Console CD (no XP media required).
