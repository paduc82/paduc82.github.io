jeeped say:
.DLL or Dynamic Link Library files are collections of routines or function that are accessed and used by one or more programs when required. The Ordinal Point is an address or entry point where a program can expect to find a specific routine or function. Different versions of a .DLL can have differing collections of routines held within and as such the addresses can vary.

It would appear that the 'transformation pack' had a different (e.g. updated...?) version of the urlmon.dll file and 'deleting' the transformation pack has rendered the version remaniing on your XP system with problems.

You might try re-registering the existing urlmon.dll. Tap the Windows key+R or click Start, Run and paste or type in the following into the Run dialog's Open text box,

regsvr32 urlmon.dll

... and click OK. If this does not resolve your problem, you may also want to try a full System File Check. Tap the Windows key+R or click Start, Run and paste or type in the following into the Run dialog's Open text box,

cmd

... and click OK. When the command window has opened, type in the following,

sfc /scannow

... and press <enter>. The process will take a few minutes depending upon the speed of your computer. This process"Scans integrity of all protected system files and repairs files with problems when possible".


Please note that you have mentioned a urlmonn.dll. I am not familiar with that specific .DLL and have provided directions for re-registering the urlmon.dll instead. If the errant file is indeed urlmonn.dll please adjust the directions accordingly.

Try repairing boot records, using these steps:

1. Start your Windows 10 machine, and quickly shut it down before it reaches or if it is at boot screen. Do this two times repeatedly.

BOOTREC /FIXMBR

BOOTREC /FIXBOOT

BOOTREC /REBUILDBCD

6. Close Command Prompt.
