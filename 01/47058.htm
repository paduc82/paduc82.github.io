3.5mm Jack part of "Speakers" (Solved, Read Below) - Microsoft Community

lilshrimp01 say:

3.5mm Jack part of "Speakers" (Solved, Read Below)

Solved

It turns out that Realtek HD Audio was the source of the problem, and the solution should've been easy, but was somewhat complicated. In order to fix the problem temporarily, uninstall Realtek HD Audio Driver via the control panel. After you do so, it will ask if you want to restart your computer --- abstain from doing so, or else it will reinstall Realtek HD Audio after you restart. Instead, after your sound has stopped working, begin the audio troubleshooter. After a few minutes, it will install the "High Definition Audio Device," the driver that should be the default audio device from Microsoft. In doing so it might ask you if you want to install Realtek, in which case skip it and say no. From there your speakers and headset will come up as two different playback devices, however, if you want to fully fix the problem continue below.

After all of that happens, there is a possibility that your computer will automatically reinstall Realtek HD Audio, and there is nothing you can do about it because it is a part of Windows 10 and I assume all the other windows versions (from what I've seen on other forums). When it does reinstall the driver, go to Device Manager and disable the Realtek HD Audio Driver. From here, navigate to your Programs folder in the File Explorer, then Realtek, then right click on Realtek and left click Properties. If you go to the Security tab, you'll notice that this specific file and all the files within it are able to access your entire computer (aha, source of the problem!). What you're gonna want to do is alter the Security properties of the file so that all access for it is Denied! From people on other forums, all you have to deny is SYSTEM and ALL APPLICATION PACKAGES, however I denied access for everything on that folder except myself, so I can access it later if I really need to. That's all you had to do, this entire time! From there, Realtek, although installed on your computer, won't be able to take control of your audio system. I hope this helps, and good luck!

Problem

A few days ago my system overheated and shut itself off, to protect itself. When the system rebooted, Realtek HD Audio had been "uninstalled," so I had to troubleshoot and reinstall the software. Since then I've had problems with my headset, where when I plug it in both the Speakers and Headset send sound. Upon further investigation, it turns out that a Device under the Playback tab in Sounds called "Speakers" has its jack information as "Analog Jack" and "3.5mm Jack." Upon more investigation, I find out the 3.5mm Jack is where my headset plugs itself in. I've surfed the internet and found no solution. I've reinstalled the driver, made sure that "Show Disabled Devices" and "Show Disconnected Devices" were checked, I've consulted the Audio Troubleshooter, and nothing has helped thus far. This is my last resort, and I hope I get an answer, please!

I'd also like to add that, while reinstalling Realtek HD multiple times while the headset was plugged in, the headset did in fact come up as a device, and for a few minutes I was actually able to use my headset problem free. However, once the driver was fully installed, the headset was taken away as a device (like it literally disappeared) and it was replaced with this "Speakers" device.
