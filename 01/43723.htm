Can't Restore System Image From A Network Drive - Microsoft Community

travis_lloyd say:

Can't Restore System Image From A Network Drive

This is actually an answer to a question I had which I have not found posted per se anywhere else and I hope it will spare somebody the many hours I've spent searching for (what turned out to be) a fairly simple solution, yet one which I never saw described in plain language.

I had created a System Image on a 2TB Western Digital NAS drive as well as a Windows Emergency Restore Disc (DVD) at the same time using a new Dell Inspiron 620 desktop with 32-bit Windows 7 Professional on it with no problem whatsoever.  After that I decided to test it out before I actually needed to use it, having learned about Murphy's Law the hard way (and more than once) - this time I wanted to make sure, as the Who once said, that I "Won't Get Fooled Again".  Well good thing, because all went well until it came time to search for the System Image file on the network drive so that I could restore it.

At that point, after I put in the path information and everything else it asked for, all I got was the totally nonsensical message, "An internal error occurred. The following information might help you resolve the error:  The network location cannot be reached. For information about network troubleshooting, see Windows Help. (0x800704CF)" which told me nothing, although I painstakingly tried over and over again, even going so far as creating a new System Image just in case I had done something wrong somewhere along the way.  I was just getting ready to post my question here when I was looking at the procedure again one last time and after seeing the option to load some drivers it hit me:  For some dumb reason when the Emergency Restore Disc DVD was created it failed to include the NIC drivers as part of the creation process, something you would just assume it would do, especially since the drivers did not natively come as part of Windows 7.

This computer has Realtek PCIe Gigabyte integrated NIC hardware on it and I had to download the drivers from the Dell Website (they're also available from the Realtek Website) in order for it to have network connectivity.

Anyway, when I finally figured all this out all I had to do was copy the NIC driver files to a USB thumb drive, plug the thing into the computer and direct the "Install a driver" option of the restore program to the thumb drive before searching for the System Image on the NAS drive. Once the NIC drivers loaded-up (it took about a minute or so) everything went smooth as silk, it found the Image nearly immediately, and the Image Restore process went off without a hitch.

Why this simple but vital step has not been mentioned before (that I have seen, anyway, and certainly not in the Windows Help Section) is a mystery to me, since I'm sure there are a ton of NIC hardware configurations out there that Windows does not have built-in drivers for.

Please Microsoft, if you are listening, fix this obvious bug in the next Windows 7 Service Pack; just rewrite the code so that any drivers (NIC or otherwise) not native to Windows are copied to the Emergency Restore Disc during the creation process; I can guarantee that this would absolutely stump about 90% (or more) of the non-geeks out there and torpedo their efforts to recover their crashed hard drives, just when they need them the most.

I hope this has helped somebody out there and that they will pass this along in some other forums; I think it can be a true "missing link" in recovering a System Image off of a Network Drive when your computer uses non-native Network Interface Card drivers.
