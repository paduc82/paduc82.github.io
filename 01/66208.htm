Windows 10 (and Windows 7 Pro) supports multiple monitors natively out of the box, you did not need any other version of Windows. Only Windows 7 Starter edition would not support dual display

In most cases the limitation is with the hardware. Not all graphics cards will support multiple displays and some motherboards will completely disable the on board graphics if an add on graphics card is installed. a. user say:

Previous Versions are created when a new Restore Point or backup operation is completed.

When you right click a folder there is an option to Restore Previous Version that applies to just that folder:

Then you will see the previous versions offered by date:

I have not changed anything in that folder since October 13.

The first previous version (11/6/2015) was automatically created when this system make a new Restore Point at 1:09 AM today:

If I open that folder and delete a file, then restore previous version to this morning, the file I just deleted is restored - and nothing else.

Look at the dates of the Previous Versions offered and the dates of the Restore Points and you can see the other previous versions are the same as the System Restore points that might be created on your system.  When your system make a Restore Point for whatever reason (or backup), that will create a Previous Version.

RPV is different that a regular system restore since RPV contains previous versions from Restore Points and backups and applies to a single folder where a System Restore applies to your entire system.

If you restore a previous version of a folder, it will only apply to that folder and not the entire system.

This is a good "trick" I learned from Frederik Long that works on a folder level and will indeed let you restore old versions of non Windows files in personal folders, etc. that are not included in System Restore.

Generally, yes - if you are looking in Task Manager, it is normal to see several/many svchost Processes running.  You might see 5-10 svchost Processes running in Task Manager, maybe less or maybe more.  But what you can't see with Task Manager is what is running behind them.

If your system is not afflicted with malicious software, important XP Services run behind those svchost Processes - sometimes just one Service, sometimes several.

The User Name you see might for the svchost Processes might be LOCAL SERVICE or NETWORK SERVICE and that is how the XP Services attach themselves to login to your system using these special accounts, for example:

LOCAL SERVICE  would be Local Services - things like the Webclient or Application Layer Gateway Services.

NETWORK SERVICE would be Network Services - things like the DNS Client or Remote Procedure Call RPC Services.

Those XP Services are running behind/under those svchost Processes using those special account names.

Some malicious software knows that using Task Manager, you can't see what is running behind the svchost Processes so the malicious software will run under one of them hiding so you can't see it.  Allegedly, some malicious software may disguise itself as a svchost Process to fool you even more.

When things like that happen, you will usually see a svchost Process that has run amok consuming lots of CPU or memory when they normally don't, so if you are seeing that, you should run some reputable malware scanners first.

a. user say:

Hello PeteWYS,

Thank you for posting here.

I understand that you are encountering an error message " JET_ErrFileNotFound" when fixing the

esentutl.exe -d "%userprofile%\AppData\Local\Microsoft\Windows Live Mail\Mail.MSMessageStore"

It's possible that the issue was caused by a corrupted program file in Windows Live Mail client installed in your desktop. Kindly repair Windows Live Mail by following the steps below:

Control Panel.

2. Under Programs, click Uninstall a Program.

3. Locate Windows Live Essential 2011 then click Uninstall/Change.

4. When a window appears, select Repair all Windows Live programs.

5. Restart your computer after the repair.

If the issue persists, please follow the steps below:

1. Restart the computer (this is to ensure that Live Mail is fully shut down, and that none of it's components are in use when the computer starts)

2. Click Start, type CMD but do not press enter

3. You will see cmd.exe in the search results, right click on it and select Run as Administrator - type your admin password if asked, and if UAC pops up asking premission, allow it

4. You should now see the command prompt, with the following displayed:
Microsoft Windows [Version 6.1.7600] Copyright (c) 2009 Microsoft Corporation. All rights reserved. C:\Windows\system32>

5. Copy the following (Ctrl+C) esentutl.exe -d "%userprofile%\AppData\Local\Microsoft\Windows Live Mail\Mail.MSMessageStore" Note: the "" are necessary

6. Right click in the command prompt Window, and select Paste, so you now see the following:
Microsoft Windows [Version 6.1.7600] Copyright (c) 2009 Microsoft Corporation. All rights reserved.C:\Windows\system32>esentutl.exe -d "%userprofile%\AppData\Local\Microsoft\Windows Live Mail\Mail.MSMessageStore"

7. Press Enter. Essential will now defragment the Live Mail database file

8. Once done, you can close the command prompt, and restart the PC again

Let us know if the steps above resolved the issue.

Regards.

¡firedog say:

Since the file wasn't found, either

  1. %userprofile% was pointing at the wrong user profile - it has to be the one whose Windows Live Mail isn't working. Did you perhaps log on to Windows with an Administrator account to run esentutl?

    or
  2. Your 'performance enhancing' program fouled things up by removing vital files. The message database consists of an index file (Mail.MSMessageStore) and a number of auxiliary files. Some of these have extensions like .chk, .tmp and .log, which are commonly misidentified and removed by clueless third-party utilities. If any of them is missing, esentutl will throw the error you're seeing.

You have no need for programs like PC Helper 360. While they may occasionally do some good, the risk of them doing something bad is just too great.
