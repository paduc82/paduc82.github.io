ami07ms say:

Hi everyone,

Thought I'll share my findings and solution in case someone else encounters the same issue.

The Surface dock Ethernet driver crashing was not related directly to sleep mode, it happened occasionally even when the PC was not in sleep mode. The root source is a double fault - one of Microsoft, and one of Apple.

The trigger for the driver crash was Apple's Bonjour software, which is installed by default with iTunes.

The Bonjour service is Apple's 'zero configuration networking'. https://support.apple.com/bonjour

Bonjour has a known issue - it raises an error when it is connected to the same network via both Wi-Fi and Ethernet, as it detects it as a duplicate network name.

The Application log shows the following cryptic error: "The description for Event ID 100 from source Bonjour Service cannot be found. ...."

My network uses the same router for both Wi-Fi and Ethernet, and of course they share the same name.

Microsoft's fault here, is that this error causes the Ethernet driver to completely crash, and the device to become unavailable.

This should never be allowed to happen...

The only way to bring the device back to life, is to restart, or to disable and enable the device from Device manager as I've shown.

Solutions:

--------------

1. Disable WiFi when connected to Ethernet

2. Use a different Wi-Fi network name, most modern Wi-Fi routers also offer an additional 5 GHz network, typically suffixed with -5G.

3. Uninstall Bonjour - iTunes works great without it too. iTunes will complain about it once only.

    Tip:While you are at it, you can also uninstall Apple's resident software updater, you will still be prompted to update when iTunes launches and an update is available.

It is easily reproducible.
