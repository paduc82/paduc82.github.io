How to Partition already allocated 100% Space to C: - Microsoft Community

ssreddy555 say:

How to Partition already allocated 100% Space to C:

I have alotted all the space on my disk to a single drive C on which I have installed the OS. Can I now partition this into two or more drives (Primary or Extended)?

burrwalnut say:

Can I now partition this into two or more drives (Primary or Extended)?


Yes you can.

1. Right-click Computer > Manage > Storage > Disk Management. Right-click the graphic of the C drive, choose Shrink Volume and the amount of space to shrink it by then click Shrink.

2. Right-click on the new unallocated space, click on New Simple Volume, give it a meaningful Volume Label and run a Quick format. 

3. You may want to further divide the space in which case dont allocate all the space in (2) above.

Notes:
If you have any issues with shrinking, running a defragmentation may move some of the unmovable files. Also, if youre running System Restore, you could try temporarily turning it off, do the Shrink and turn it back on.

You only need to use an Extended partition if you intend to create more than four partitions. You would make the fourth one Extended and create Logical drives within it.

Good luck.

considerate_guy say:

Quick question: When you were saving frequently, were you saving with different file names each time? If so, just go back to the prior version before this one and open that project.

Because Movie Maker can crash, as you said, it is important to save your project regularly. But it is also important to save the project with different names so that if the project file becomes corrupted (which is what has apparently happened to you), you can go back to the saved version before this one.

I've seen this reported many times and essentially what has happened is that Movie Maker crashed and wrote wrong information into the project file. If you open the project file (.wlmp) in a text editor (like Notepad), you'll see the details. Each VideoClip is supposed to have an inTime and an outTime. But when the project gets corrupted, Movie Maker writes the wrong details into outTime. That makes it so the outTime is earlier than the inTime or it is zero. The result is a clip that logically has a negative duration.

The next time Movie Maker tries to open the project, it sees this negative duration and won't fully load the storyboard area. It is also prone to more crashing after that.

If you don't have a differently named version of the project file, and you don't have file versions turned on (where you can rollback to a prior version), about your only viable option is to start over with a blank project.

You could try making a copy of the project file and trying to manually fix the corruption, but without knowing exactly what you are doing and the format of the project file, you aren't likely to be able to get it back to a working state. This is apparently a frequently occurring issue, but not one that Microsoft cares to acknowledge.

So going forward, make backups of your project as you go along and use different names with File > Save project as, rather than just File > Save project

jj_russell say:

It means the system restore function has become disabled, possibly via a group policy.

To re-enable it:


1. Click Start, Run and type regedit.exe and press Enter

2. Navigate to the following key:

HKEY_LOCAL_MACHINE \ Software \ Policies \ Microsoft \ Windows NT \ SystemRestore

In the right-pane:

1. Delete the value DisableConfig
2. Delete the value DisableSR
3. Exit the Registry Editor.
4. Reboot.

In Windows XP Professional, you can accomplish the above using Group Policy Editor.

1. Click Start, Run and type GPEDIT.MSC
2. Navigate to this path:

Computer Configuration-> Administrative Templates-> System-> System Restore

3. Set Turn off System Restore to Not Configured
4. Set Turn off Configuration to Not Configured
5. Exit the Group Policy Editor
6. Reboot.
