john john __ say: Are you sure that the virus was properly removed?  Run a TDSSKiller and a Malwarebytes scan on the machine.

This greyed out "Hidden" check box usually means that the folder also has the "System" attribute set on it, this can only be resolved by using the Attrib command at an elevated (Run as administrator) command prompt.

At the elevated command prompt use the CD command to navigate to the root folder:

cd\

At the root folder you can get a listing of your folder attributes with the following command:

attrib /s /d

The list will be long, it's better to pipe the output to a file:

attrib /s /d >c:\attrib.txt

to open the file from the command prompt simply enter the path:

c:\attrib.txt  (and press <Enter>)

A peculiar quirk of the Attrib command is that if both the System and Hidden attributes are set you have to remove them both at the same time, you cannot remove only one while the other exists.  To remove the system and hidden attribute:

attrib -h -s  /s /d

When issued from the root folder this command will remove the hidden and system attribute on all the folders on the drive... problem is the attrib command automatically processes all files so it will also remove the system attribute on all the files, including all the files in the Windows directory and sub-directories.

These are the kind of mess that you sometimes have to deal with when dealing with viruses or other such pests.  Another even worse problem that you sometimes have to deal with as a result of viruses is that of modified permissions, it is sometimes difficult to know what exactly a virus has done and how the system is compromised as a result of an the infection, for these reasons IT administrators in business or enterprise environment very often consider servers or mission critical workstations to be compromised beyond trust after an infection and the only acceptable solution in those environments is to flatten the box and restore an image or rebuild the machine.
