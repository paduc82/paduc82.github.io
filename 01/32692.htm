Hello SP-925,

 

Flashing the BIOS can be fraught with problems, especially on older computers. Unless you know what you are doing I always recommend that you leave well alone. Of course, flashing the BIOS in more modern PC's is a little easier, but not much.

Error 0x80070015 Solution - Trying to download Windows 10 updates. - Microsoft Community

spencer simon say:

Error 0x80070015 Solution - Trying to download Windows 10 updates.

Whenever I tried to download updates for Windows 10, ALL of the updates would end with Error 0x80070015. Since I was unable to use my microphone online in PC games, a "DirectSoundCapture" problem, I thought a fresh installation of Windows 10 would fix both problems. The microphone problem was fixed, but I still could not download Windows updates. I have had the same problem for weeks, causing me to go insane enough to go into my registry and tinker with anything "Windows Update-related."

(NOT RECOMMENDED UNLESS YOU UNDERSTAND WHAT YOU ARE DOING)!

I google'd for hours and realized how many people had the same Error code. 

Link to the post that helped me: 

"Windows Diagnostic Policy Fix" I apologize, as I do not know the rules pertaining linking to other sites.

To cut it short, the reason that Windows Update was bringing up the Error code was due to the fact that "...Got a message that Network Diagnostics cannot run because the Diagnostics Policy Service is not running. Opened Service Control Manager to try and start that service, and got a message that Windows could not start the service on Local Computer, followed by Error 5: Access is denied."  

I traced the problem to "Diagnostic Policy Service" not "Running" (open "run" using WIN+R, type in "services.msc", and check if the "Diagnostic Policy Service" is "Running".) Right click DIAGNOSTIC POLICY SERVICE, and in the properties, make sure that its start-up type is Automatic, Automatic (Delayed), or Manual. ANYTHING EXCEPT "Disabled".

To fix this problem:

  1. Right click the Windows Icon in the lower left-hand corner, select "Command Prompt (Admin)." 
  2. At the prompt, type: "net localgroup Administrators /add networkservice" then press enter. It should follow saying that the command had been successful, or something being a success. 
  3. Then type:"net localgroup Administrators /add localservice" Press enter, then type: "exit", and press enter. The same line about the command being a success. 
  4. Restart your computer (even though idk why I chose not to).
  5. Open the Services task (using the WIN+R or "run" command, and enter "services.msc".
  6. Make sure that the task states that it is "Running", and if it does not say so, go into the properties of the Diagnostic Policy Service and click on "Start". If you do not receive any error message(s), and the Service says "Running" now, then you are one step closer.
  7. Go to Windows Update and click "Retry". Hopefully, you will no longer receive Error 0x80070015, but instead it will start downloading all of those updates, and soon enough, they will all go from Status: Pending install, to Installing:, to prompting you to restart your computer.

I was able to download every update that was listed, and after installing my computer, my system is now up-to-date. Hopefully this will help out everyone with the "Error 0x80070015" problem when trying to download updates for Windows 10. If it did not help, I will try my best to back track the previous steps not listed, in the case that they did anything useful. Good luck, and let me know what happens!

-Looking for devices for possible partition or formatting issues in Disk Manager. They do not show at all

-Running the hardware troubleshooter.
-Updating drivers for Intel(R) USB 3.0 eXtensible Host Controller - 1.0 (Microsoft)
-Updating drivers for USB Root Hub (USB 3.0)
-Uninstalling both of the above and rebooting for automatic install
-Running Microsoft Surface Diagnostic Toolkit

barb bowman say:
Sorry you are having this issue. It looks like you've done a lot of trouble shooting already. There is one more thing you can try. Go to https://www.microsoft.com/en-us/download/detail... and download SurfacePro4_Win10_16299_1709001_0.msi to your Surface. Now disconnect from the Internet. Go into device manager and uninstall and this time DELETE the drivers for the two USB items you previously uninstalled. Now run the MSI. When installation is completed, restart your Surface. Try one of the USB sticks that was not being recognized. (Connect to the Internet after trying a USB stick).

If the above fails, if you are within 10 days of installing 1709, you should be able to go back to your previous version of Windows. If past 10 days, sadly the only option is to wipe and reimage from a downloadable recovery image https://support.microsoft.com/en-us/surfacereco... (and I know how painful that is).

1709 seems to be causing issues for folks, although I've only seen one other report in this forum of this USB issue.
