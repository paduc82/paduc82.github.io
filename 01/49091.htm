a. user say:

Hi Michelle,

Thank you for your interest in Windows 10.

Group policy is usually set by companies on employee machines.  Has your machine ever been a part of a company network before?  Even as a bring-your-own-device sometimes companies have policy pushed down to these machines.

If this is a machine actively managed by a company you'll need to speak to your IT department, because even if you fix this issue it will likely reoccur every time you connect to the company’s network.

Assuming this is your personal machine, you'll need to use an admin account to fix your settings:

In your start box or a run command type: gpedit.msc and press enter.  This should launch Local Group Policy Editor.

In Local Group Policy Editor go to Computer Configuration > Windows Settings > Administrative Templates > Windows Components > Windows Defender.

Here you should see an option that says "Turn off Windows Defender" that will likely say "Enabled" as State.  Double click and reset this setting to "Disabled" in the upper left.  Save your changes and then after a reboot you should no longer have this issue (again unless you're part of a network that pushes these changes).

You may also try to launch Windows Defender from the below location: C:\Program Files\Windows Defender\MSASCui.exe

Hope this information is helpful. Please do let us know if you need further assistance, we’ll be glad to assist you.

Hi Alan,

 

It would be great if you can answer these questions for a better understanding:

 

1)       What is the exact model number of the computer?

2)       Did you install all the pending updates for your Operating System via Windows store before the attempt to install the Windows 8.1 update?

3)       Are the drivers updated for all the devices on the computer?

4)       Do you have any external devices (except the mouse and the keyboard) connected to the computer?

 

The error “0xC1900101-0x30018” occurs during the “sysprep” in the first boot while installing the Windows 8.1 Update. So, if you have any unnecessary external devices (except the mouse and the keyboard) connected to the computer, then please disconnect them and then try installing the Windows 8.1 update and check if that helps. I would also suggest you to update the drivers for all the devices connected to the computer.

 

Method 1: Update the Device Drivers on the Computer.

 

Step 1: Check the Windows updates for any latest driver updates.

 

Check in Windows updates if any driver updates are available. If so, install the same and check if it helps.

 

a)       Press “Windows Logo” + “W” keys from the keyboard.

b)       Type “Updates” in the search bar and click on “Install optional updates” from the left pane.

c)       From the “Windows Update” window, click on “Check for Updates” on the top left corner, to receive a fresh list of updates and then install all the updates listed there.

 

For any issues while installing Windows Updates, you may also refer to this link:

 

Troubleshoot problems with installing updates

 

If no driver updates are available via Windows Update, then you may install them from the Computer Manufacturer’s website.

 

Step 2: Update Driver Software from the Manufacturer’s website:

 

Go to the computer manufacturer’s website and search for the latest Windows 8.1 compatible drivers available for all the devices on the computer and then install them by following the instructions given in the website.

 

If the issue still persists, then let’s also try clearing the Store Cache and check if that helps.

 

Method 2: Clear the Store cache

 

You may try these steps to clear the Store cache and check if that helps:

 

a)       Press the “Windows Logo” + “R” keys on the keyboard.

b)       Type “wsreset.exe” and press “Enter”.

c)       Reboot the computer and then try installing the Windows 8.1 update again and check the status.

 

If you have any third party security program, then you may go ahead with the next method.

 

Method 3: Temporarily disable the third party security program. (If you have installed any)

 

I request you to contact the technical support for the security program installed on the computer to temporarily disable it and then check the issue.

 

Important: Antivirus software can help protect your computer against viruses and other security threats. In most cases, you shouldn't disable your antivirus software. If you have to temporarily disable it to install other software, you should re-enable it as soon as you're done. If you're connected to the Internet or a network while your antivirus software is disabled, your computer is vulnerable to attacks.

 

If the issue still persists, then we need to collect some Setup & Rollback error logs from the computer for further troubleshooting on the issue. We have sent you a Private Message requesting for the logs and with the instructions on how to collect it. Please check the Private Message sent to you on this thread and reply to it with requested information.

 

Hope this is helpful. Please get back to us with the results.
