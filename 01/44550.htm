what you see is the abort function from the Visual C++ runtime being called from the windows explorer process.

However, Windows does not come with the Visual C++ runtime so the code calling the abort function can't be from Windows. The offending is loaded by or injected into Windows Explorer. Windows Explorer supports loading plugins (context menu, icon, column provider etc) and certain programs (e.g. global hooks made by virus, antivirus, input methods etc) can inject code into any process.

From the pattern of crashing on right click, the most likely cause is a faulty context menu extension. You can use nirsoft's ShellExView utility to find context menu extensions and disable them one by one to see which extension is at fault, then report this to the extension's author.
