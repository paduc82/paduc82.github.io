monica cam say:

Hi John,

You may access Safe mode in Windows Recovery Environment by making your computer think that the startup is failing. Windows Recovery Environment (WinRE) is a recovery environment that can repair common causes of unbootable operating systems. To do so, follow the steps below:

  1. Turn on your computer.
  2. When you see the loading dots, immediately turn off your computer.
  3. Repeat steps one and two. Three times will be enough for the computer to think that you are having issues with start up.
  4. Once the computer boots up for the third time, you will see the loading dots on the screen and the message Preparing automatic repair.
  5. Afterwards, your PC may turn blank then it will show Diagnosing your PC.
  6. It will ask you to restart your device or proceed to the advanced options.
  7. Choose Advance Options.

Once you're in the Windows Recovery Environment, you may go to Troubleshoot > Advanced Options > Startup Settings to access Safe Mode.

Should you need further assistance, feel free to get back to us.

Have your kid try the following:

With the headset connected to the controller, and the controller connected with a Micro USB cable (be sure it's a USB data cable, not a smartphone charging cable which often lacks the data lines to save money), launch the Xbox Accessories app, and check that the controller has an up-to-date firmware. 

If the controller is up-to-date, have the kid compose a message to a friend (they don't have to send the message, so it doesn't matter to whom they send it to). 

Instead of typing the message, have the kid press the Microphone icon, and record the voice message. 

After recording the message, you can listen to your own voice. 

Does the Microphone work with the voice message?

If not, it might be that the controller is defective. 

If the voice message worked, but Party chat doesn't, it might be a network problem, or your kid accidentally pressed X on the party screen and muted themselves in the party. 

For network issues, check the NAT status in the network settings of the console. Make sure the NAT is shown as "open". 

In order to open a NAT, you need to configure your internet router, as NAT is not a setting in the console, but a method to connect to specific devices "behind" a router, that bundles several devices (like your PC, smartphones, tablets and consoles) through one public internet connection.

This is my generic how to for proper driver updates : (update your major drivers and BIOS)

This utility makes it easy to see which versions are loaded :

DriverView - Free - utility displays the list of all device drivers currently loaded on your system.
For each driver in the list, additional useful information is displayed: load address of the driver,
description, version, product name, company that created the driver, and more.
http://www.nirsoft.net/utils/driverview.html

For Drivers check System Maker as fallbacks and Device Maker's which are the most current.
Control Panel - Device Manager - Display Adapter - write down the make and complete model
of your video adapter - double click - Driver's tab - write down the version info. Now click UPdate
Driver (this may not do anything as MS is far behind certifying drivers) - then Right Click -
Uninstall - REBOOT this will refresh the driver stack.

Repeat that for Network - Network Card (NIC), Wifi, Sound, Mouse and Keyboard if 3rd party
with their own software and drivers and any other major device drivers you have.

Now go to System Maker's site (Dell, HP, Toshiba as examples) (as rollback) and then Device
Maker's site (Realtek, Intel, Nvidia, ATI as examples) and get their latest versions. (Look for
BIOS, Chipset and software updates at System Maker's site while there.)

Download - SAVE - go to where you put them - Right Click - RUN AD ADMIN - REBOOT after
each installation.

Always check in Device Manager - Drivers tab to be sure the version you are installing actually
shows up. This is because some drivers rollback before the latest is installed (sound drivers
particularly do this) so install a driver - reboot - check to be sure it is installed and repeat as
needed.

Repeat at Device Makers - BTW at Device Makers DO NOT RUN THEIR SCANNER - check
manually by model.

Manually look at manufacturer's sites for drivers - and Device Maker's sites.
http://pcsupport.about.com/od/driverssupport/ht/driverdlmfgr.htm

go look for new ones manually if you wish.

wearenotamused say:

Something short of re-installing all of Windows that I would try is creating a new Windows user profile. There's a part of the registry that's user-specific and created from scratch for each new user. (You'd have to move any documents, pictures, etc over to the new profile, but until you deleted the old user, you could easily go back to get anything you overlooked.) As the desktop is user-specific, I wouldn't be surprised if the cause of that error were in the user-specific portion of the registry.

First step: try creating a new Windows user profile, log in as it, and see if it suffers the same problem. If it doesn't, then consider just moving your files over and being done with it. If it does, then we've learned a little bit more about where the problem must lie. Let us know.
