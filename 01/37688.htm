Hi,

Thank you for posting your query in Microsoft Community.

As you have mentioned that you want to know the ways to fix runtime broker.

Runtime Broker is a Windows process in Task Manager that helps manage permissions on your PC for apps from Windows Store. It should only use a few megabytes of memory, but in some cases, a faulty app might cause Runtime Broker to use up to a gigabyte of RAM or more.
If your RAM use is high and your PC is running slowly, an app may be the cause of the problem. Press Ctrl+Shift+Esc to open Task Manager and then, on the Processes tab, check to see how much memory Runtime Broker is using. If it’s using more than 15% of your memory, you probably have an issue with an app on your PC. To stop Runtime Broker from using so much memory, select Runtime Broker in the list, select End task to close Runtime Broker, and then restart your computer.

If still the issue persist suggest OP to run in-built system maintainace troubleshooter and check-- give appro steps;

1) go to control panel.

How to create a folder with the same permissions as - Microsoft Community

a d r i a n say:

How to create a folder with the same permissions as "C:\Program Files" folder.

I am making a 2nd Program Folder directory on another drive, but I want it to be secure like the original.  What permissions do I need to set and how do I set them?

I've already set "ALL APPLICATION PACKAGES" group on it, but "CREATOR OWNER" and "SYSTEM" I'm unsure of.  When I check the permissions on "C:\Program Files" I get those with "Special permissions", but what permissions are they?  Modifying my directory, I cannot click on that checkbox, and I cannot change SYSTEM's permissions to match up with the original.

Maybe there's an easier way of doing this?

A

stevehd say:

The best way I've found to do this is to copy the permissions from one folder to another rather than trying to duplicate them using windows explorer.  One way to do this is use an administrative command prompt (right click on start menu and select "Command Prompt (Admin)") to enter the following 5 lines (you can copy and paste each line using your mouse or touchpad):

icacls "C:\Program Files" /save \progacl.txt

mkdir "D:\Program Files"

icacls D:\ /restore \progacl.txt

icacls "D:\Program Files" /setowner "NT SERVICE\TrustedInstaller"

del \progacl.txt

What this does is:

1.  Create temporary text file progacl.txt containing a copy of the folder permissions.  Note if you want to change the name of your program folder on the 2nd drive you can edit this file (type notepad \progacl.txt) and change the folder name (top line) to whatever you want (such as "ProgFiles" or "2nd Program Files").

2.  Create your new program folder.  Make sure you create it on the drive you want (such as D:, W:, etc.) and name it the same as what is in your progacl.txt file.  If you have already created the folder skip this command.

3.  Restore folder permissions for the matching folder on the D drive.  Again, make sure you use the drive letter you want.

4.  Set the owner to TrustedInstaller.

5.  Delete the progacl.txt file.  This step isn't actually required but if you're like me you want to clean up and delete anything you don't need.

BTW, another option is to open Notepad and copy/paste the 5 lines above in (making sure to press "enter" at the end of the last line) and save this as "Create Program Files on D.bat" somewhere (perhaps create a folder called "Setup" in your OneDrive so you can use it with other PCs you've added drives to).  Now right click on the batch file and select "Run as administrator" to create D:\Program Files with the correct permissions.

BTW2, I'm sure there is a powershell command that can do this in one step by piping get-acl and set-acl together (it might require another second command to set the owner).  Perhaps a powershell expert could post how to do this and provide a breakdown of the command.
