When you add extentions (from Indexing Options > Advanced button > File Types tab), make sure you click the option at the bottom for "index properties and file contents".

bab9e9 say:

Are you sure you want to delete it?  "EA DATA. SF" holds "extended attributes" that belong to other files on the file system.  "EAs" are part of NTFS and the old HPFS file systems.  The "EA DATA. SF" is a kludge to allow "EAs" to be stored to FAT file systems and later recovered.  

To get rid of it: 

  1. format the low level device. Lose everything.
  2. Use wild-card ? tol match with the blank (0x20) embedded in the name: del "EA?DATA.?SF"  

Caveat:  I haven't tried this with U3. There may be other problems.

BTW, PortableApps does pretty much what U3 does, but isn't as possessive about your drive.
