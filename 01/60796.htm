a. user say:

You are confusing two separate ideas. I'll explain below but first let's get to your problem:

In your case, it seems that you either removed the Administrators account, or interfered with the Administrators account in some other way. Now Windows 10 can't find an Administrators account, which is why you can't use the elevated permissions available to an Administrators account.

Unfortunately, the fix is not simple. Follow the steps here:

Windows 10: If the Admin prompt has a greyed out or missing Yes button

There are two types of Windows user account. The first type of user account depends on where the account was created. A local account is created on your own computer, and is only accessible from your own computer. A cloud account is created on the internet and is accessible from any computer that is connected to the internet. A Microsoft account is simply Microsoft's version of a cloud account. (Just like a Google account is Google's version of a cloud account, and iCloud is Apple's version of a ... you get the idea.)

The second type of user account depends on what kind of permissions (aka privileges) the account has. An Administrators account has the ability to make changes that affect the entire computer, like installing software. A Standard account only has the ability to make changes to itself.

These two types of account can be combined, so you can have a local Administrators account or a local Standard account or a Microsoft Administrators account or a Microsoft Standard account.

So if you have a local Administrators account, that's perfectly fine. In fact, it's safer than a Microsoft Administrators account.
