Page file in multiple partitions - Microsoft Community

belarmino vicenzo say:

Page file in multiple partitions

Will I have a few increase on my PC's performance if I put a page file for each partition?
bossdweeb say:

Ah! The good old infinite Page File debate. You may want to just play with the options and find what you think

works best for your system.

My observations and conclusions are that while it should not be disabled, as many people

( especially gamers) have been doing since they started running  machines with 8GB or more RAM, the Page File's

significance to overall performance has decreased dramatically since XP and earlier days.

Some argue having a PF can reduce performance if you have adequate RAM, but if it does it would only be in

synthetic benchmarks. It would not be noticeable in 'real world' running your programs or games.

While the Page File was important in the day of XP and 512MB or less RAM,  it is not much benefit since

Win7 and large quantities of RAM moved in.

Other than Windows needing the Page File for creating memory dumps, some software still seeks the Page File and will

have issues if it cannot find one. And while that software may use the Page File, it will only use the amount it

was developed to use. No more. So there is no need for a great amount of HDD space to be allocated to the PF.

System Managed is still the recommended (industry standard ? ) setting, but I've been getting by with 2048MB

min and max Custom setting for at least five years.

- Having a PF on every partition/drive is not going to improve performance on a modern system, if it ever did.
