vanessa yar say:

Hi MarkFilipak.Windows,

It's really a curious case which you've raised but let me help you shine some light over your query. I may not be able to answer all the queries you have here in detail but let's try to focus more on the command "explorer.exe shell:AppsFolder".

Before we look at the whole picture, we'll have to dissect the said command and cut it to three parts: explorer.exe, Shell, and AppsFolder.

What is explorer.exe?

The program explorer.exe which you can find in C:\Windows\System32 is what we commonly call as File Explorer. This is the file manager application of Windows which allows you to browse, manage, edit, and view files and folders in your computer's storage (e.g. HDD, SSD, etc.) Check this article if you need some Help in File Explorer. 

What is Windows Shell?

Windows Shell is the graphical user interface of Windows which includes, but is not limited to, Desktop, Start Menu, taskbar, and other components. It implements a "shell namespace" which allows programs to access the computer's resources through some shell objects. If you are a developer or interested in developing programs for Windows, reading the articles from this link will help you understand the Windows Shell.

What is AppsFolder?

With our basic understanding of Windows Shell, we now know that it implements a "namespace" which is simply giving "names" to a given set of unique identifiers. AppsFolder is a name for one of the default or "Known Folders" of Windows Shell. This is a virtual folder, unlike the other folders which you can easily find by typing the location on the address bar. Virtual folders allows you to view and group files from different folder locations, with a given criteria, into a single folder or view just like the Quick access folder. In this case, AppsFolder is a virtual folder which shows files, with a given criteria, from different locations in a single folder. To know more about other Known Folder IDs which are part of the Windows Shell namespace, feel free to visit this link.

To sum it up, running the command "explorer.exe shell:Appsfolder" calls File Explorer to open a "virtual folder" which would group together applications from different file locations into a single window. From what I've noticed, it only shows apps which has shortcuts that can be found in C:\ProgramData\Microsoft\Windows\Start Menu\ and its sub-directories and it also shows Microsoft Store apps just like the 3D Builder and Alarams & Clock apps.

To answer your other queries - it's an application (which is explorer.exe), it doesn't have an actual name but it's display name is "Applications," and you can "edit" it, in a sense, but it will affect the programs which shows on your Start Menu since the location which I stated above is where Start Menu fetches the app shortcuts. To "edit" it, you can go to C:\ProgramData\Microsoft\Windows\Start Menu\ and it's sub-directories, delete, rename, or add some shortcuts to other applications in its folders. The "About Java" program which you saw on the list may be a shortcut for a non-existent program (possibly previously installed) so you can just delete the shortcut file and it will be deleted from the AppsFolder view as well.

Hope this answers your queries.
