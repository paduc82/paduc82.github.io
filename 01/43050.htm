Local vs admin accounts - Microsoft Community

richardg06 say:

Local vs admin accounts

I have been told that a local login account is safer than an administrator account with respect to someone gaining access to my computer and accessing my information and changing or adding programs or viruses, and I have been told that there's not much difference. Your thoughts?

stephanie sal say:

Hi Richard,

To answer your query with regard to local account and administrator account, a local account is the same as any account you're using to login to a Windows operating system. It allows you to access the system resources in your own user space. You cannot make system changes, but you can install software on your computer.

An administrator account is the one who can make changes on a computer that will affect other users of the computer. Administrators can change security settings, install software and hardware, access all the files on the computer, and make changes to other user accounts.

Feel free to post back if you have further queries.

Regards.

Local account is safer because of limits on what can be done, and this is recommended.

However these limits can be frustrating to those who what to do more than basic tasks.

It's pretty difficult... impossible, to maintain a PC without having an admin account available.

- Using an admin account with UAC enabled improves on the security of an admin account somewhat.

I keep an eye out for irregularities in system use ( cpu, RAM and disk ) as well.

AKA - Diligence and due care.

And I have a special account for my Pay Pal transactions that never has more than around $500 in it.
