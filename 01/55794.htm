Ideally, any executable file can reach your system via internet connection, physical installation media, FTP (File Transfer Protocol, that again needs network connection) or peer to peer 'network'. If a system is set isolated or in a secured environment, there has to be some medium that transfers the file to that secure location. It could be via wireless connection, cable or transferred physically. Even if the driver is 'passable', it needs some connection or medium to get passed or transferred. We can make the internet connections secure as much as we can but we cannot isolate the system and expect any incoming driver from the outer world. Also, when we say "we need a way to download the drivers to storage from MS's site and move them to the secure destination computer", we need to determine a connection or medium on which the file will move to reach the secure destination computer.

You have mentioned a Brothers link in your last post, which shows download from Windows Updates for your printer model. Windows Updates need internet connection to get patches and packages from the Microsoft server for Windows Updates to show, download and sometimes install on your system. Unless the system is connected to any network, you cannot install the drivers using Windows Updates as well.

However, there is a solution on how to install XP drivers in later versions of Windows. Again, if you get access to raw executable driver files (via internet or physical installation medium from the manufacturer), you may refer to the following steps to install Windows XP or Vista drivers on Windows 7 system in compatibility mode:

  1. Right click on the downloaded driver file.
  2. Click on Properties and then select Compatibility tab.
  3. Tick mark the option Run this program in Compatibility mode for: and then select Windows Vista or Windows XP from the drop down menu.
  4. Click OK and install the driver.
