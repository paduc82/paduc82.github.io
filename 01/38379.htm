cmdrkeene say:
Very true.  And even more nuanced than that, it matters how frequently you're using the program.  Even if it's a big giant program hogging a bunch of memory, if the program doesn't do anything for a while, the operating system will save the memory to the hard drive temporarily in the swap/paging file, so that the memory can be used for something you are using.  If the big hog program needs it again later, the operating system will handle juggling and swapping the data back into active RAM to use it.  Too much of this swapping between RAM and storage is what can slow down a computer significantly, because the hard drive is hundreds of times slower than RAM, and you need data to be in RAM to actually use it.
