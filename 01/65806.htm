Unable to delete a wav or mov file in Windows 10? Try disabling - Microsoft Community

billjacobs1 say:

Unable to delete a wav or mov file in Windows 10? Try disabling the handler

I looked for hours for a way to delete a wav file that has been crashing my File Explorer. I tried all different ways to delete it. Nothing worked. Found all kinds of scary but wrong solutions online (like mess with your registry). Then I found this here on MS, and want to share it with you because it is so easy. Also works on .mov files. It has something to do with Windows searching files for metadata. You need to disable the "handler" that is doing that. Disable the wav "handler" in seconds with this: ShellExView - Shell Extensions Manager. Install it, open it, search for your file type (in my case wav) and right click and select disable. Reboot. Now you can delete the wav file. It's that easy. I was up all night trying to fix my computer before I found this. I wanted to spare you. Thank me.

billjacobs1 say: One more reply. It is the MF WAV Property Handler that needs to be disabled. You can use the ShellExView for this. After I deleted the wav file that was causing the crashes, I went back and enabled it. I'm not sure what it does, but I thought I'd leave it the way it was originally.

anamon say:

That handler tries to extract metadata from WAV files when you select them in Windows Explorer. Due to this bug, I consider it to do more harm than good, so I personally left it disabled. I have better software to check and edit metadata, anyway. And I assume it would only be a question of time until I'd run into another WAV file that causes Explorer to freeze.

The background on this is that certain tools exporting WAV files put additional metadata in them, which the Microsoft Media Foundation handler gets confused by. It keeps reading the same parts of the file in an infinite loop (you can see this by running FileMon while the problem is happening).

I have found more information on this issue on this blog:

http://itsclipping.org/2014/08/04/explorer-using-high-cpu-wav-files-markers/

It also links to a Microsoft knowledge base article, where they claim that it's the fault of "corrupt" WAV files. Of course, neither are those files corrupt, nor would that be an acceptable excuse for a metadata extractor to freeze or crash when encountering them. It's a robustness bug in Media Foundation that would be Microsoft's responsibility to fix.

billjacobs1 say: Agreed. The wav files are not corrupt. It's definitely a MS problem with reading metadata. Some applications, such as digital audio workstations (DAWs) and other audio apps, add metadata to wav files which Windows sometimes can't handle. That's not the same as being "corrupt." And the fix is so easy, it's surprising that MS hasn't offered the fix themselves.
