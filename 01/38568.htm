a. user say:

A clean install gives you a fresh start, that's for sure. However, there's no reason to buy a new hard drive unless you suspect that a physical problem with the existing hard drive is the causing your issues, and that's really unlikely. Any hard drive manufactured over the last 10 years has a built-in utility called S.M.A.R.T. that tells you when a hard drive is about to fail. A messed up operating system doesn't cause damage to a hard drive.

Although a clean install gives you a fresh start, that comes with several caveats:

If your problem(s) are caused by hardware, reinstalling software won't make a difference. For example, if your hardware is not supported on Windows 10, reinstalling Windows 10 isn't going to make a difference.

Likewise, if your problem(s) are caused by applications, reinstalling the same applications won't change anything. For example, if you are running applications that aren't supported on Windows 10, reinstalling the same applications is not going to help.

If your problems are caused by malware, a clean install of Windows and all your software will usually help, but not always. Sophisticated malware can survive even a clean install. In that case you would have to do a low level format before the clean install. In a low-level format you fill your hard drive with zeroes.

Finally, and I need to be blunt, if you have problems because you made mistakes, a clean install will erase those mistakes, but if you don't learn from your mistakes they will only come back to haunt you again.

A clean install properly done is no walk in the park. If you want to do it right the first time you will need to invest many hours and probably days in preparation. You'll want to read about the procedure in detail - there's more than one way to go about it - so that you know what to expect. There should be no surprises. And you want to have all the software you'll need ready and waiting on a USB flash drive before you start. This includes hardware drivers and your antivirus application.

There's nothing here that you can't do. All it takes is research and preparation. Those are my secrets and they work.

If your trying to connect to a remote PC using Remote Desktop [RDC] from your PC that is behind an AT&T DSL modem then there is nothing to configure. You call using its public IP, ie. the IP assigned by the ISP or a fully qualified domain name.

If the remote PC is behind a DSL modem that is also a router then you or someone may need to open TCP Port 3389 through the router to the local LAN IP of the RDC host PC depending on the model of the DSL modem router device.

So which case is it and what is the exact model of modem and/or router are you talking about? On the RDC host end you can test port forwarding by going to the http://www.canyouseeme.org site and running the test on TCP Port 3389.

Please describe each end of the RDC connection link to help us help you.
