MHTML, short for MIME HTML, is a web page archive format used to combine in a single document the HTML code and its companion resources that are otherwise represented by external links (such as images, Flash animations, Java applets, and audio files). The content of an MHTML file is encoded as if it were an HTML e-mail message, using the MIME type multipart/related.

The first part of the file is an e-mail header. The second part is normally encoded HTML. Subsequent parts are additional resources identified by their original URLs and encoded in base64. This format is sometimes referred to as MHT, after the suffix .mht given to such files by default when created by Microsoft Word, Internet Explorer, or Opera. MHTML is a proposed open standard, circulated in a revised edition in 1999 as RFC 2557.

The .mhtml (Web archive) and .eml (e-mail file) file extensions are interchangeable (the files can be renamed). The first can be sent by e-mail (and displayed by the email client if the html code is basic enough) and an e-mail message can be saved to an OS file and renamed to a Web archive extension.

a. user say:

Hi,

Welcome to Microsoft Community.

I understand that the USB devices fails to get detected on Windows 10. I will certainly help you.

This issue might occur if the device drivers are outdated, corrupted, incompatible or due to hardware problems.

I suggest you to follow the troubleshooting methods listed below and check if it resolves the issue.

Method 1: Run the Hardware and Devices troubleshooter

Windows 10 has a built-in troubleshooter to check and fix issues with devices and hardware. Here's how:

1. Open Control Panel by right click on Start Button.

2. In the search box, type troubleshooter, and then click Troubleshooting.

3. Click on the view all option on the left panel.

4. Run the Hardware and Devices troubleshooter.

Method 2: Uninstall and Reinstall the USB controller drivers

Use Device Manager to uninstall and reinstall all the USB controllers. This lets the controllers recover the USB port from its unresponsive condition. The USB controllers represent the USB ports in Device Manager.
To disable and re-enable the USB controllers, follow these steps:

1. Right click on Start and select Device Manager.

2. Expand Universal Serial Bus controllers.

Note: You might have to scroll down the list to find this item.

3. Right-click the first USB controller under Universal Serial Bus controllers, and then click Uninstall to remove it.

4. Repeat step 3 for each USB controller that is listed under Universal Serial Bus controllers.

5. Restart the computer. After the computer starts, Windows will automatically scan for hardware changes and

    reinstall all the USB controllers that you uninstalled.

6. Check the USB device to see whether it is working.

Hope this helps. If the issue remains unresolved, please get back to us and we would be happy to help.

Thank you.

What was the problem the led to you needing to reinstall the printer?

Let's try the simplest method first:

1.  If the printer is connected to the computer by USB, disconnect it.

2.  Go to Control Panel > Programs and Features and uninstall all Brother software

3.  Go to Devices and Printers, right-click on any icons for your Brother printer and select "Remove device.  If you're unable to remove the printer, the likely cause is that there are documents left in the print queue.  Right-click and select "See what's printing" and in the Print Queue window click "Printer > Cancel all documents."

4.  Still in Devices and Printers, select (click once) any item in the Printers and Faxes category, then click on "Print server properties" in toolbar at the top of the window (to the right of "Add a printer").

5.  Click the "Drivers" tab and "Remove" any Brother drivers.  Remove both the driver and "driver package."

6.  Close everything, reboot the computer, and install the printer using a freshly-download driver package.  Don't connect the printer until prompted by the software or after the driver installation has completed.

lemp say:
It sounds as if you have one computer ("this laptop") and two identical printers.  There should be no problem in installing two instances of the printer on the laptop and directing them to the two physical printers.

How are the two printers accessed?  That is, are they both network printers accessed via wi-fi or is one of them connected by USB?

Your problem is compounded by the fact that your printer is a multifunction device.  It's actually fairly easy to install multiple instances of wi-fi connected printers; I don't have any multifunction devices so I can't experiment with how you should connect the scan function.

The steps that I gave you should have cleaned out your earlier installations.  I guess it's time for a more thorough cleaning.  In the instructions below, I assume that BOTH printers will connect to the laptop via wi-fi.

Download the free trial of Revo Uninstaller Pro and get rid of any remnants of Brother software:  https://www.revouninstaller.com/revo_uninstaller_free_download.html

Then clean out your printing subsystem:

  1. Disconnect the printer from the computer if connected by USB
  2. Go to https://github.com/jdickson289/Print-Reset-Tool/ and download print-reset.ps1 and make a note of where you save the file
  3. Click Start, type PowerShell into the Search box, right-click on the link to Windows PowerShell, select "Run as administrator" and respond as needed to the UAC prompt 
  4. Type the following command into the PowerShell window and press Enter
    Set-ExecutionPolicy Unrestricted
  5. In response to the "security risks" question, type Y and press Enter
  6. Type the following command into the PowerShell window and press Enter.  In typing the command, use the full path of the location where you saved the print-reset.ps1 file.  In the example shown below, print-reset.ps1 was saved in C:\Temp
      C:\Temp\print-reset.ps1 -full
  7. Type R to run the script
  8. After the script completes, restart the computer
  9. Download a fresh copy of the driver package from Brother:  http://support.brother.com/g/b/downloadend.aspx?c=us&lang=en&prod=mfcl2740dw_us_eu_as&os=93&dlid=dlf100997_000&flang=8&type3=11
  10. Run the installation package and follow the on-screen connections to connect the first printer
  11. Go to Devices and Printers and confirm (a) that the printer is installed and set as the Default Printer and (b) right-click on the icon, select Printer properties, and click "Print Test Page"

This is what the PowerShell window should look like as you proceed through steps 3 through 7 (I didn't enter R because I didn't want to reset my own printing system).  Don't forget to use the actual location where you saved the print-reset.ps1 file rather than C:\Temp as shown below.

Note that in step 8 if you omit the -full switch and run the script with no switches, you'll get a complete description of what the script does.  See the screenshot below.

If the first printer successfully installs, you now need to setup the second printer.  I suggest doing it manually.  The relevant instructions are in the "online" manual for your printer.  If you don't have this on the CD that came with the printer or it wasn't installed on your laptop when you installed the first printer, you can access it here:  http://support.brother.com/g/s/id/htmldoc/mfc/cv_dcpl2520dw/use/manual/index.html#GUID-6724461D-8AB1-4461-B32B-CA39AE21FD9A_358

Go to this section: Home > Network > Additional Methods of Configuring Your Brother Machine for a Wireless Network > Configure Your Machine for a Wireless Network Using the Machine's Control Panel Setup Wizard

When you complete the steps described there, the printer should produce a wireless status report.  Make a note of the IP address assigned to the printer.

On your laptop, open Devices and Printers, click "Add a printer" and select "Add a network, wireless, or Bluetooth printer."  When the search completes (it may take several seconds), you should see the IP addresses of both printers.  Select the IP address for the second printer and step through the wizard.  When asked, choose to use the driver that is already installed.  When you get to the step of naming the printer, give it a name that will let you easily identify which printer it is.
