Update: Workaround

I've discovered a workaround that solves my set of circumstances, but doesn't actually cancel the repair installation. Those that still need to find how to cancel it won't be in luck, though the solution I did come up with may help others.

Firstly I removed the hard drive from this computer and ran it as a slave on a working PC. I then usedMagical Jelly Bean Keyfinder to attempt to find the product key. My default that program finds the product key of the windows currently running (ie not the one on the slave hard drive), so I used the 'Load Hive' option and navigate to the Windows directory of that other hard drive (for me this was g:/Windows). This then gave me the product key for that hard drive.

However this key wouldn't work for the repair installation either!

I reasoned that their XP Pro product key was for a different XP Pro CD, so looked through my old laptops until I found an XP Pro product key that was accepted. The installation continued and completed normally.

Once Windows had loaded, I downloaded and ran the Windows Product Update Tool from the Genuine MS products website. I used this tool to switch to using their product key, which was accepted - see I told you it was genuine!

The computer restarted and the world was good. The original problem that led to me running the repair install is fixed, and their comp is back to using their XP Pro installation and their XP Pro product key. I hope that helps you too!

The pagefile is memory on disk to be used when the requirements for memory exceed the amount of RAM available. Reading and writing to the pagefile is much slower than to RAM. This means that if the system starts to rely on the pagefile, the user will see slower system performance. On a system with 3gb RAM I would expect to see a 3gb pagefile and certainly no more than 5gb. You do not need a 61gb pagefile.

When memory becomes a factor limiting system performance you investigate whether you can increase the amount of installed RAM. The solution is not to increase the size of the pagefile. In your case you have a 32 bit version of Windows so you have a system set maximum of 4gb RAM. You also have a maximum of 4 gb set by your motherboard. This means that you can only have a maximum on your computer of 4gb RAM.
