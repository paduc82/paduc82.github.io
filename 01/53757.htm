hounddog999 say:

I have resolved this problem.

In Windows' Device Manager (devmgmt.msc), the SXUPTP driver was showing as disabled. Windows gave a warning that the device was known to have stability issues. I found out that SXUPTP is a driver from a company called Silex. Silex makes network USB hubs, so I assume that the Belkin hub is a rebranded Silex hub.

In order to update the Silex driver follow these steps:
1. Close the Belkin network hub's tray application.
2. Download the latest version of SX Virtual Link (sx-virtual_link_win_3120.zip or the latest version).
3. Extract all of the files.
4. In Windows Explorer, navigate to sx-virtual_link_win_3120\s-33a3120\Files\Bin\Sxcsapi\WinVistaX64 if you are running a 64 bit OS, or sx-virtual_link_win_3120\s-33a3120\Files\Bin\Sxcsapi\WinVista if you are running a 32 bit OS.
5. Right-click on "Sxuptp.inf" and select "Install" from the menu.

The driver will be installed and the warning icon in the device manager should disappear. You may need to reboot for the changes to take effect.

Hope this helps.

Excel files can not be opened in Task Scheduler in Windows 8 - Microsoft Community

rickneilson say:

Excel files can not be opened in Task Scheduler in Windows 8

Inserting the name of an Excel file into the Action tab of a Task Scheduler under Windows 8 will not, correctly, open the file - as it would under Windows 7 or even XP.  The same applies to Word files - I do not know about anything else.  The symptoms of the failure vary, based on the length of the Excel file name - this is because the actual bug relates to the name of the File to be Opened being truncated back to about 30 characters as it is passed from Task Scheduler - hence the opening fails, in varying ways.  Microsoft Technical Services have confirmed that this is in fact a bug in Windows 8 - they say it has been reported and "will be fixed in due course".

I have found that the following workaround allows me to do all that I was able to do using all previous operating systems.

Create the Scheduled Task as normal.  Set "General"; "Triggers"; "Conditions"; & "Settings" as required.
Under the "Actions" tab DO NOT set the "Program/script" to the file you wish to run, as would be normal - rather set that to the Excel application - ie. the full path, in quotes, to the EXCEL.EXE program.  Then, in the "Add arguments(optional)" box put the full name, in quotes" of the file you actually want opened.  And it all works as one would wish.

As with many computer-related tasks, you need both hardware and software to burn DVDs.


No computer can burn DVDs without a drive that is capable of doing that.  A CD-RW drive will only burn CDs.


Even with the required hardware, Windows XP can not burn DVDs on its own.  You need to install DVD burning software.  Some such software is free, such as ImgBurn or the program suggested by Leon Kenny; some you have to buy, such as Nero Burning ROM.

If you're not sure what your optical drive is, open Device Manager (Start > Run > devmgmt.msc OK), expand the section for DVD/CD-ROM drives by clicking the +, and report the complete name of the device(s) listed there.
