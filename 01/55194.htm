ralf_g say:

I have both versions of Win10, on different computers- quad core, dual core and single core CPUs. The quad core computer is less affected by background tasks competing with the browsers for CPU time but other than that browsing speed is essentially the same on all of them.  None of them are anything like as slow as you've described yours.

Tasks running in the background, including antivirus programs which scan content in the web pages before they allow the browser to display them, are the usual suspects for slowing the computer's response times. Especially processes that also use up network bandwidth. Various browser Add-ons that pre-scan web pages before they load can also slow down browsing response.

You can use Task Manager (in its expanded window) to monitor if any processes are running with high CPU usage while you browse (or otherwise for that matter) and if any process is consuming your network bandwidth excessively.

One process that consumes network bandwidth, which is built in and enabled by default, is the Windows Updates  "Update from more than one place" setting. This one actually enables the  "sending" of update data from your computer to other random computers on the internet. I have seen it consume my entire upload bandwidth at times. This can be turned off in Settings - Update and Security- Update Settings- Advanced Options- Choose how Windows Updates are delivered.

Other Windows settings can also use internet bandwidth on their own, like cloud based protection in Defender and automatic synching in OneDrive (or other cloud storage) and to a lesser extent Feedback and Diagnostics.

One browser setting which could also slow down browsing response, at least initially, is predictive pre-caching of web pages... ie. it allows the browser to load and cache web pages that you aren't currently looking at based on the links that are present in the pages that you are looking at.  This can also interact with your antivirus software to slow things down even more as the AV scans all of those web pages that the browser is caching.
