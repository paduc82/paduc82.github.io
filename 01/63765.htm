considerate_guy say:

Windows DVD Maker was phased out after Windows 7. There is no separately download for any other version. It came with Windows Vista and Windows 7 but not later versions of Windows (8, 8.1, 10).

It's much like the way that Apple got rid of iDVD after the 2011 version of their iLife Suite. They no longer supply this program for creating legacy video DVDs.

My guess, apart from the cost of licensing the MPEG-2 codec for DVD creation, both Apple and Microsoft appear to consider the DVD format an older format (e.g. can't handle HD video, came from the era of analog TVs, etc.).

Note: Both the .wmv and .mp4 files that are created by Movie Maker can be read directly by a Blu-ray player or a modern Smart TV. You can either put the files on a USB or onto a disc (even an old DVD disc) and they will be playable; and in HD rather than SD.

Windows Script Host supports scripts written in VBScript or JScript. When you start a script from your desktop or from the command prompt, the script host reads and passes the specified script file contents to the registered script engine. The script engine uses file extensions (that is, .vbs for VBScript and .js for JScript) to identify the script. As a result, you do not need to know the exact programmatic identifier (that is, the ProgID) of the script engine. sarahkong say:
This maybe an issue with a system file being corrupt.
You can try a couple things.

Right click your start button and select Command prompt (Admin) to bring up elevated Command prompt.
In Command prompt type sfc /scannow and press enter.
Restart the computer.

If that doesn't work then try this,

Right click your start button and select Command prompt (Admin) to bring up elevated Command prompt.

In the Administrator: Command Prompt window, type the following commands.
Press Enter key after each command:

DISM.exe /Online /Cleanup-image /Scanhealth

DISM.exe /Online /Cleanup-image /Restorehealth

Important: When you run this command, DISM uses Windows Update to provide the files that are required to fix corruptions.
To close the Administrator: Command prompt window, type Exit, and then press Enter.
Restart your computer.

Hi there? If you still suffer from this error code and after a lot of research I found the problem and the solution... for me at least. Let me deal with two different situations.

1. If you run Windows 7 or earlier then you probably also run the Intel 'booster' package. This makes Win7, Vista and others start faster.... but it also BUILDS A RAM DRIVE!! If you open Device Manager at look at your hard drive details you should see something called IDM-0 AS WELL AS your real drive. You might just be able to delete this and then run the Windows updates but a reboot (needed later) will bring it back! So I suggest you UNinstall the Intel Booster app package AND delete the RAM drive!! Restart the PC and away you go!! If you put it back I suspect you will get the problem again!

2. If you are running Win10 then you will also see the IDM-0 RAM drive so do the same as above. But you need not have installed it and certainly do not repeat your error. Win10 does NOT need the Intel 'go-faster strip'!!

If you are unsure of what you have then check for the IDM-0 drive in Device Manager. If it is there then dump it... if not - and you still have the error - then look for something similar!!

It definitely seems to be caused by an unwanted or inadvertent RAM drive!!
