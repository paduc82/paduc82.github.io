helpforsure say:

Hi,

The error message you are getting is basically pointing towards Port 3389 not working on the XP Client machine.

Please check in Device Manager->Show Hidden Devices, where you've all the RDP Services started, if not then please start them.

You should also check if the port number 3389 is specified correctly in registry editor or not.

HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Control\Terminal Server\WinStations\RDP-Tcp

HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Control\Terminal Server\Wds\rdpwd\Tds\tcp

Also, you can try to merge these 3 keys from a healthy WinXP Sp3 machine:

HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Control\Terminal Server

HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\services\TermDD

HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\services\TermService

If this doesn't help, please get Termdd.sys from a healthy machine and replace it on this machine in safe mode[WFP Disabled].

davem121 say:
Running a System Restore generally puts the Registry back into working order:

Start PC, just as the manufacturers logo disappears and Windows attempts to load, press and hold Power Button for 5 - 10 seconds to perform a Hard Shut Down

Do this twice

On the third start Windows will boot into the Recovery Environment and from there you can access System Repair, Safe Mode, Command Prompt . . . etc.

Go to Troubleshoot - Advanced Option - System Restore
Check to see if you have a restore point you can go back to
