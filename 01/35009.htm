lemp say:

If you click on the "Dependencies" for the Security Center service, you'll find that it "depends on" the following services:

  • Remote Procedure Call (RPC)
  • DCOM Server Process Launcher
  • RPC Endpoint Mapper
  • Windows Management Instrumentation

What this means is that the Security Center service won't start unless all of those other services have started first.

Open the Services applet and take a look to see which, if any, of those services is not running.

Because a ton of services depend on the Remote Procedure Call (RPC), DCOM Server Process Launcher, and RPC Endpoint Mapper, my guess is that if any of those 3 services wasn't running, you'd see a lot more problems.  So look first at the Windows Management Instrumentation service.

The Windows Management Instrumentation service should have "Local System account" selected on the "Logon" tab, and its "Startup type" should be set to "Automatic"

Just in case:

Remote Procedure Call (RPC) -- Startup type: Automatic;  Logon:  This account > Network Service

DCOM Server Process Launcher -- Startup type: Automatic; Logon:  Local System account

RPC Endpoint Mapper -- Startup type: Automatic;  Logon:  This account > Network Service

jeeped say:
... All my programs have the windows media center icon ...

I've seen this happen when upon opening a program through a shortcut (e.g. .LNK file), you are asked what program to open it with. After selecting Windows Media Center (or some other program) via the Select a program from a list of installed programs, and checking the Always use the selected program to open this type of file (circled in red below), the document or program referenced by the original shortcut opens in Windows Media Center as it should.

                    

The trouble is that now all shortcuts (e.g. .LNK files) on the desktop or throughout Explorer will want to be opened by Windows Media Center and use the Windows Media Center icon to display the shortcut regardless of what the shortcut was originally intentioned to open. This is known as a file association and default icon. The same condition could occur if you performed those 'Open With' actions on a .JPG or even an .EXE, but the .LNK explanation is far more likely for items on the desktop.

The good news is that a small free utility has been written to undo the .LNK file association that was created. There is also a registry hack to accomplish this, but I recommend the unAssoc utility for its ease of use. I used to send people to another site to download this utility, but that site has become confusing with all the ads and false download links. I've put a copy on by own SkyDrive here.

                                    

Just download, then use the utility to unassociate the .LNK files for either the User or globally Delete the file type and letting Windows rebuild a new one to defaults.
