jessen p say:

Hello,

 

WER stands for Windows Error Reporting (WER). Windows Error Reporting collects and offers to send post-error debug information (a memory dump), using the Internet, to the developer of an application that crashes or stops responding on a user's desktop. No data is sent without the user's consent. When a dump (or other error signature information) reaches the Microsoft server, it is analyzed and a solution is sent back to the user when one is available. Solutions are served using Windows Error Reporting Responses. Windows Error Reporting runs as a Windows service and can optionally be entirely disabled. If Windows Error Reporting itself crashes, then an error reports that the original crashed process cannot be sent.

 

You can delete the WER files from

 C:\Documents and Settings\<user name>\Application Data\Local\Microsoft\Windows\WER

 

Check the link mentioned below as a reference.

http://technet.microsoft.com/en-us/library/bb457179.aspx

rami-- say: Elemsi,
The steps that Martin suggested are not accessed through cmd. The only reason he recommended opening cmd as administrator is to enable the built in administrator account through the "net user administrator /active" command. So I will retype these steps with a bit more emphasis:

Click Start -> In the start search field type: cmd, when the results come up, right click on cmd and click Run as administrator.

Now when you see the "C:\Users\Username" prompt type the following command next to it:
Net user administrator /active:yes
press enter.
Reboot your computer and log in under the newly activated Administrator account.
In this account take the steps outlined in the below linked document to create a new user profile:
http://windowshelp.microsoft.com/Windows/en-US/Help/769495bf-035c-4764-a538-c9b05c22001e1033.mspx

Hope this helps
