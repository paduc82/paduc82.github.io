considerate_guy say:

When Movie Maker crashes in this fashion, it corrupts the project file. There are ways for you to try to manually "uncorrupt" the file but it's a time-consuming process. I've done it previously for others but I'm not feeling like doing this manually for a project containing 138 clips.

Since it is your project, you might be willing to do this, so let me give you the steps I would perform:

  1. Make a *copy* of the original project and only work in the copy. Leave the original corrupted file as a backup and don't touch it in case you need to start over. If that happens, again work with a *copy*.
  2. Open up the project with Notepad or a similar text editor like Notepad++.
  3. Search for any instances of the string 'outTime='. If the number that follows is 0 or otherwise smaller than the 'inTime=' value on the same line, that's going be considered an invalid clip with a negative duration. You need to edit the outTime value to make it bigger than the inTime value.
  4. Once you have found all the problem clips, save the text file (still with the same extension of .wlmp) and try opening the project again in Movie Maker.
  5. If it opens, you'll still have to manually review all the clips that were edited and probably have to trim their endpoints to the actual point you wanted.

Here are some pointers for avoiding this in the future.

  • Work with smaller projects. If you have too many clips in a single project, Movie Maker is more likely to crash. You can always work with several *sub-projects*, save them separately as a movie and then join the movies together at the end in a final master project.
  • Save frequently and with a *different* name each time. For example, if you were saving as Project001, Project002, Project003, etc. and it fails in Project047, you could just go back to Project046 to hopefully lose only a few minutes of editing rather than everything.
