andre da costa say:

The computer’s seemingly spontaneous rebooting could be caused by hardware factors or software issues. On the hardware side, if the fan is ailing or the air vents are blocked, the machine may be overheating. If this seems to be the case, replacing the fan, cleaning the vents and clearing space around the computer may help. (A utility program that displays the processor’s temperature may be helpful; Core Temp and SpeedFan are two such programs and shareware archives have others.)

A bad memory chip can also bring a machine down. A free memory test program like HCI Design’s MemTest or the Windows Memory Diagnostic Tool can help confirm or rule out faulty RAM as the culprit. Problems with the computer’s motherboard, power supply or hard drive could also be the cause but may require a computer technician to properly diagnose and repair.

On the software side, malicious programs like viruses and spyware can cause erratic behavior and random reboots, so make sure you have an up-to-date security program installed on the computer. Basic versions of Avast and AVG Anti-Virus Free are free. Microsoft offers its own suite of free security programs as well.

If you have installed new hardware devices recently, the machine may be suffering from a driver conflict. Drivers are small files that let Windows communicate with the hardware that’s attached to it, but sometimes these driver files  clash with each other. Microsoft has tips for using the Device Manager utility to troubleshoot driver problems.

Printer ip confusion - Microsoft Community

petermat say:

Printer ip confusion

I have a recurring problem with printing to printers on my network. I have three (sometimes four) Win 7 pcs and two HP printers on my local network with a D-Link router type CIR 655 – with up to date firmware. Both the Printers are set for DHCP ip allocation. The problem is in four parts – in rough order of importance:

  1. At (apparently) random the ips the router thinks is allocated to the printers becomes different from what the printers report their ip being. (Recently, for example,  the ip’s got crossed between the printers resulting in a command to print to one printer producing reams of paper from the other.) Rebooting the pc, router and or printers has no effect.
  2. The only recourse seems to be to remove the printers and then add them back in. Adding in by name never works so they are always added in by the ip the printer wizard finds.
  3. Deleting the ports the printers with the wrong ips seems to be a useful part of the operation – but is often declined by windows.
  4. Similarly ‘removing’ a printer seems to always work – until the ‘windows and devices’ page is refreshed when usually the ‘removed’ printers return. They are ‘greyed out’ but appear in the list of printers offered when a ‘print’ command issued – causing confusion.

 

As one observation, sometimes it is possible to tinker with the port allocations on one pc to make them match what the printers define as their ip. If this is done, then printing from that pc then works, but does not (of course) from the other pc’s.

Another observation – changing the name of the HP 2600N printer via browsing the printer resulted in the router seeing the name change correctly. Changing the OfficeJet 6700P name the same way also changed the name seen by the router – but not to the same value as the printer reported.

 

Any suggestions short of throwing the printers / router / pcs out of the window welcomed.

wunders say:

The printers should not be configured for DHCP IP allocation because there is no guarantee that their IP addresses will remain constant.  If you must use DHCP, then you should configure your router for "DHCP Reservation" for the printers which will guarantee that the printers will always receive the same IP address from the router.  I could not find your "CIR 655" model at the D-Link website, but did find a "DIR-655" model which explains how to set DHCP reservation on page 37 of the downloadable manual from D-Link.

I personally prefer to fix the IP address for my printers to use an IP address that shares the same first 3 IP address numbers with the rest of the subnet and the last number chosen outside the DHCP range of the router's DHCP server.  The method of doing this differs from printer to printer.  Check your manual.

Either way, "nailing down" the IP address of your printers should solve your problem.

HTH,

  JW

wunders say:

I have found that often when you install a printer using the manufacturer's CD or install program, it also installs a program or service that continually runs in the background and attempts to discover printer additions and deletions and attempts to adjust the appropriate configuration (i.e. attempts to make it "idiot-proof").  I would blame these programs for resurrecting your printers once deleted.  You might use Task Manager to see if you have any such programs running.

My preference has always been to install just the minimal driver for a printer, fix the printer's IP address, and configure the printer myself.  Sometimes the printer manufacturer doesn't give you that choice and  I have to install the whole package.  In that case, fixing the IP address seems to make it easier on those extraneous programs and I rarely have problems -- but then again, I only have one printer in my setup.

HTH,

  JW

Sometimes your computers firmware, which deals with things like power management, which is related to sleep needs to be updated. Remember, the issue you are experiencing here is outside of Windows, where its not resuming from sleep. Windows is in a saved state on the hard disk and can't manage the hardware resources, thats left up to Firmware (your BIOS) to handle. So, its obvious this more a hardware issue than software, but updated driver software can fix it.

It could suggest you need to update your chipset drivers and BIOS.
I recommend you update the most critical ones: BIOS, Chipset, Display Driver, Network adapters.
If that does not work, I recommend you try two things
First: Boot to the Windows 10 desktop
Press Windows key + X
Click Device Manager
Expand Batteries
Right-click Microsoft ACPI-Compliant System.
Choose Update Driver Software.
Make sure you are connected to the Internet, the check for updates.

When complete, restart then check if you can sleep and wake.

Second option
uninstall your display driver then restart
Boot to the Windows 10 desktop
Press Windows key + X
Click Device Manager
Expand Display adapters
Right click the current display adapter
Click Uninstall
Exit Device Manager
Restart
