The error you are getting is STATUS_DLL_INIT_FAILED. 

Presumably upc.exe is not able to initialise a DLL it depends on.  If it can't find a DLL, I'd expect the error to have been  0xc0000135 STATUS_DLL_NOT_FOUND so it might require a little more debugging work than just running Process Monitor looking for file not found/access denied (https://docs.microsoft.com/en-us/sysinternals/downloads/procmon) although that would be a good start to get a quick overview of what might be happening to the upc.exe process.

I would do as follows:

1. Download the Software Development Kit (SDK) of the platform you are on.  For example:
Windows 10:
https://developer.microsoft.com/windows/downloads/windows-10-sdk

Windows 7:
https://www.microsoft.com/download/details.aspx?id=8279
Note: You can Google SDK and the OS version and it should be the top result.

2. Install the "Debugging Tools for Windows" feature,. 

Note the install location when installing as you will need it later.



Once complete you will have a directory such as:

C:\Program Files (x86)\Windows Kits\10\Debuggers\x64\

The applications in the directory we will be using are: 

  • gflags.exe
  • cdb.exe

1. Launch an administrative command prompt.

2. Change directory to where the tools above are installed to. e.g. 

CD "C:\Program Files (x86)\Windows Kits\10\Debuggers\x64\"

3. Run the command:

gflags.exe /i upc.exe +sls

Note: This will turn on loaded snaps for the upc.exe process. Essentially creating the registry keys:

GlobalFlag = 2
HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\Windows NT\CurrentVersion\Image File Execution Options\upc.exe
GlobalFlag = 2

4. In the same command prompt then run (one line):

Note: Change the path to upc.exe if required.

I assume you will then see a stream of text go by and you''ll end up with the same error in your original post.  You can dismiss the error.  Then in the command prompt at the "0:000>" prompt you can type the following commands hitting enter after each:

QUIT 

Under %temp% you will have the log file logloader.txt.  If you can post/attach the contents that would be helpful.

To remove the loader snaps for this process, run:

gflags.exe /i upc.exe -sls

The ProgramData specifies the path to the program-data folder (normally C:\ProgramData). Unlike the Program Files folder, this folder can be used by applications to store data for standard users, because it does not require elevated permissions.

The path can be on a volume other than the system drive and should not cause performance loss nor cause confusion to Windows , as long as it meets the following requirements:
