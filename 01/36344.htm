robert aldwinckle on forums say:
[gle=0x80004005]

What else can I try to finally get this update installed?

Try running ProcMon to help "read between the lines" of these diagnostics.  Filter with Operation Is WriteFile to help find and correlate them all.  You can infer a match based on the Timestamp, the filename in the Path field and the Length= given in the Details field.  Once you are sure you have one match the others can follow readily by counting from it.  However, the real benefit of using ProcMon is that you can supplement the meaning of any message by looking at the preceding File and Registry events and assuming that they could have had a role in the production of the message you are interested in.  FWIW I suspect that the above Error message contains no real diagnostic value without such supplementary clues.  Note that its Length= will be considerably bigger than any of the other surrounding messages and thus should make matching it easier.

Good luck
