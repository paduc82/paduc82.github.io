Hi Franclin

My name is Andre Da Costa; an Independent Consultant, Windows Insider MVP and Windows & Devices for IT MVP. I'm here to help you with your problem.

The SVCHOST process is a normal process in Windows 10.

Beginning with the Windows 10 Creators Update (v1703), previously grouped services will be separated such that each will run in its own SvcHost process. This change is automatic for systems with greater than 3.5 GB of RAM running the Client Desktop SKU. On systems with 3.5 GB or less RAM, services will continue to be grouped into a shared SvcHost process.

Benefits of this design change include:

Increased Reliability by insulating critical network services from the failure of another non-network service in the host, and adding the ability to restore networking connectivity seamlessly when networking components crash.
Reduced Support Costs by eliminating the troubleshooting overhead associated with isolating misbehaving services in the shared host.
Increased Security by providing additional inter-service isolation
Increased Scalability by allowing per-service settings and privileges
Improved Resource Management through per-service CPU, I/O and memory management and increase clear telemetry (report CPU, I/O and network usage per service).

A question about Windows 7 networking:-

However, I've noticed that whatever 'make' of router I use seems to get allocated what appears to be the vanilla default name ROUTER, meaning that storage devices such as USB flashdrives attached to my Belkin 'versatile USB port' (in order to create a simple NAS for example) get allocated an UNC name starting \\ROUTER - eg \\ROUTER\mystick\myfolderthereon\myMSWordtherein.doc .

So far so good, normal and it all works perfectly. However... i would like to change the first part of the UNC so that rather than starting ROUTER ... , it starts with something of my own choice eg just a plain R as in \\R\mystick\myfolderthereon\myMSWordtherein.doc .

ywsah say:

Cracked it myself!

Ok so as above I found I can't change my router's internal devicename ' \\ROUTER ' cos, as above, Belkin don't seem to supply an 'official' interface to my Play N600 to

But Microsoft - via its Windows\System32\drivers\etc\hosts file - does kindly provide me with an interface to create an alias for an ip address!

So, since my router's base address is Belkin's default 192.168.2.1 (yes, ..2.1 unlike say Netgear's equivalent in case anyone was wondering), I simply added the following line at the end of the text in my existing 'hosts' file:-

192.168.2.1 R

and 'bingo' now I can just use my desired UNC  \\R\mystick\myfolderthereon\myMSWordtherein.doc etc etc to address files and folders on a memorystick in my router's USB port.

Anyone else wanting to try this 'fudge that works', please note :  one would need to modify the Windows\System32\drivers\etc\hosts on any Windows machine on one's LAN in which one wants to use such an 'alias' of course; it doesn't matter if one is inconsistent either, though if you're like me you'll probably prefer to be consistent!

[ To clarify: you could use 192.168.2.1 R on one machine but, on another:-

192.168.2.1 Rtr

and then use appropriate UNCs starting \\Rtr or \\RTR (etc etc) on that machine, though I can't really see the point!

Me, I'll be using plain 'R' from now on! ]

Hooray!
