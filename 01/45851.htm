Then when you click OK, you see a message like this:

RUNDLL
Error loading wzcdlg.dll
%1 is not a valid Win32 application.

The problem is that the c:\windows\system32\dot3api.dll file is somehow corrupted (it is not missing) and needs to be replaced.

The other file - c:\windows\system32\wzcdlg.dll file might be okay, but keep in mind that it too might need to be replaced.  Start by just replacing the dot3api.dll file.

Since the file is in the list of 3498 files that Windows File Protection cares about there should be "good" copies on your system in a backup folder. 

You need to replace the file(s) in c:\windows\system32 with a good copy that you can find elsewhere on your system.

Search your system and find backup copies of the files.

Depending on your system and how XP was installed, you may find copies of the file here:

c:\windows\system32\dllcache
c:\windows\ServicePackFiles\i386

If your system has an installation folder that contains an i386 folder (perhaps on a D drive), there may be copies there or if  you have a genuine bootable XP installation CD, you could expand the files from the CD.

Your goal is to replace the suspicious file(s) in c:\windows\system32 with good copies.

When you find the copies, you should boot in Safe Mode so the files are not in use, navigate to the windows\system32 folder and  rename the suspicious file to something you can remember (like dot3api.old, etc.) so in case something goes wrong you can put the original files back).

Then copy in the "good" replacement file from the other location over the top of the old file or replace the files if they are totally missing (unlikely).

Respond in the affirmative if you are asked to overwrite or replace the existing files.

On Windows 8 x64  I also want to access a help file from an ancient application that I use daily. I found the same problems and the same solutions noted above worked for me -

Problem 1 - Windows8-RT-KB917607-x64.msu refuses to install with message "This update is not applicable to your computer".

Solution 1 - obtain the old winhlp32.exe from http://www.eightforums.com/software-apps/7540-winhlp32-exe-how-can-i-enable-legacy-windows-help-system-windows-8-a.html. Then deal with...

Alternatively - use Control Panel/Add language to install & enable English (US) - took ages, required reboot, showed error message (failure configuring Windows) but finally worked. (NB. Before rebooting, ensure you know how to enter any special characters in your login password on a US English keyboard.)

Then with US English enabled,  Windows8-RT-KB917607-x64.msu installs and you get a more up to date version of winhlp32.exe.
