cmdrkeene say:

As I said, the task scheduler can only start a program (a task) for you.  It cannot, by itself, perform actions without that program for you.

If the program you want it to start is a script (a PowerShell, batch file, shell script, whatever), then the script can take actions for you (but then it is not Task Scheduler doing those things.  In other words, it just started the process.  It dropped the first domino and watched them fall.  In essence, that's all the Task Scheduler can ever do.

You mentioned stopping a task, but that is also not the task scheduler doing so: that's you doing it by your own actions.  You could have done the same thing from Task Manager instead of Task Scheduler.

To answer your specific questions:

  1. Rename a task: not from the Task Scheduler UI no.
  2. You can export a task, then import it as another name (effectively duplicating it) and then editing one of those copies.  Right-click a task to export it.
