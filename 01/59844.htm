¡firedog say:

another error message showing a server error code 0x800CCC90 - "problem retrieving message" ...               

It's always a good idea to quote the error message verbatim, because it usually contains clues to what's wrong. I'm going to guess that the error message you're seeing contains a line reading Server Response: -ERR [SYS/TEMP] problem retrieving message. If this is the case, then you can appreciate that this is a temporary system error at the server, so there's nothing you can do about it in Windows Live Mail. This is not unusual with AT&T Yahoo accounts, which are notoriously sensitive to minor corruptions in messages in the mailbox. To clear the fault, then, 

  1. Log in to the account in webmail and start by emptying the spam and trash folders.
     
  2. Move all of the messages in the inbox to a user-created folder.
     
  3. Once the inbox is empty, do send/receive for the account in Windows Live Mail. You should see a No new email messages status message at the right-hand end of the status bar.
     
  4. Move the oldest half of the messages in the temporary folder that haven't yet been downloaded back into the webmail inbox and again send/receive in Windows Live Mail.
     
  5. Rinse and repeat, halving the number of messages moved each time until you can identify any message that is interfering with the POP3 download. Deal with it there.

PS  Because this is a problem at the server end, removing the account then adding it back again will only have the effect of permanently deleting all the account folders and all the messages in them, meaning that the entire contents of the account inbox will have to be re-downloaded.
