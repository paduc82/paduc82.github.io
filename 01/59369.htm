            You have to understand FAT data structures and how the FAT file system stores files.  Basically all the files on the disk are stored inside directories in a hierarchical structure, the top most directory in the hierarchy is the root directory.  The root
 directory is a special directory but otherwise a directory is really nothing more than a file containing a directory table made up of 32 byte structures which in turn contain information about files stored in the directory.  This table contains information
 such as the file or sub-directory name, file attributes, creation date, size, and so on.  The table also contains the all important information about the file's starting cluster.  The file itself is stored in the FAT.  The FAT is chained, that is clusters
 that make up the file are chained together, each cluster holds a pointer to the next cluster in the file and the last cluster in the file has an EOC (end of cluster-chain) marker, so the operating systems knows where the file starts and ends and the cluster
 map in between.



As you already know files are not exactly stored in an orderly fashion on a disk, in reality they can be stored all over the place!  Lets say you have a small file occupying 5 clusters.  For simplicity's sake let's say that the file is stored in clusters 5,
 12, 32, 78 & 102.  Cluster 5 will contain information telling the OS that the next file segment is stored in cluster 12, cluster 12 will contain information telling the OS that the next file segment in stored in cluster 32 and so on until the OS reaches cluster
 102 containing the EOC marker telling it that the file ends. When you want to work with files the OS gets the file name and other information including the starting cluster from the directory table then it finds the file in the FAT from there.



A directory is just another file with an entry in the directory table but the OS know that it is a directory so it knows how to  read it and find information about the files stored in the directory.  So, how does the OS know that a file is a directory?  Simple,
 the file is marked as a directory with the "directory attribute" bit.



A directory is just another file so Notepad can read some of the contents of the directory file but otherwise it doesn't know what to do with this information and the OS doesn't know that the file is a directory, it may be as simple as the directory bit not
 being set or there may be corruption and part of the directory table may be damaged.   You could use a disk editor to examine the directory structure and try to correct the problem but you really need to have deep knowledge and understanding of the FAT data
 structures to attempt this, in reality if you don't know what you are doing you will just cause further damage or completely screw up the disk.



All your files are most likely all there, it's just that the OS doesn't have any information about them and it doesn't know where to find the all important starting clusters, good data recovery software or competent data recovery firms usually have good success
 rates with these kinds of logical file system problems.
