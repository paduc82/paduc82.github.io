When you format a hard disk, you can chose the type of file system.  In the good old days, one would chose FAT32 (File Allocation Table 32 bit version) but then as operating systems and hardware evolved (faster and bigger), there were many limitations/issues with a FAT32 file system, so they invented NTFS (New Technology File System) and NTFS is what most modern systems are running, so NTFS is what XP should be reporting (that is a good thing).

Click the underlined links above to read more (much more) about each.  Maybe just the first couple paragraphs is enough knowledge unless you are really interested in them.

When you reboot your system, XP automatically checks all volumes (all your HDDs) to see if any of them are "dirty" and if it finds one that is dirty, it will run automatically run a chkdsk with error correction to try to clear the dirty bit.  A volume might be marked dirty from a power interruption, aborted restart, ungraceful shutdown, etc.

The only thing that can clear the dirty bit on a volume is a successful run of chkdsk with error correction.  Most of the time, that will clear it up but sometimes they are stubborn and you might have better luck using something like the XP Recovery Console or Hiren's to run the chkdsk.  I like to do that sometimes since you can "see" what it is doing and watch it.  Since your system has no CD/DVD, you have to boot from a USB thumb drive to do that.  And, I don't think there is a way to run something like a chkdsk with error correction from Linux (which also uses NTFS).

Once the dirty bit has been cleared, XP will still automatically and silently check the volumes on every reboot, but if there are not dirty volumes, the automatic chkdsk should not run.

You can query the dirty bit on a volume to see if it is dirty or not dirty, but no matter how hard you try or what other people might tell you, you can't manually clear the dirty bit - only a successful run of chkdsk with error correction can do that and if your HDD is really afflicted, sometimes even that won't clear the dirty bit  If the volume is dirty, you will see an annoying problem where chkdsk runs on every reboot until you fix it.

Here's how to query the dirty bit on a volume (go ahead and give it a whirl):

Click Start, Run and in the box enter:

cmd

Click OK to open the command prompt window.

To query the dirty bit on drive C, type:

fsutil  dirty  query  C:

Sample output:
Volume C: is dirty
Volume C: is not dirty

Type 'exit' to close the command prompt window.

There are lots of other good tools with Hiren's and you can do much more with that (and it is easier) than using the XP Recovery Console.
