laurent alquier say:

Access permissions keep getting reset on home network since latest Windows 10 Creators update

Since the removal of Homegroup from the Windows 10 Creators update, I have been running into a strange issue on my home network.

BACKGROUND

Home network - single WiFi access point and router.

I am trying to keep some key folders in sync between a Surface Book 2 and a desktop PC. 

Both machines are up to date with the latest windows updates. 

I am using Defender out of the box for firewall and security and GoodSync as the synchronization software.

I created a local account on both machines, with the same credentials, to use for local file sharing. Both accounts are set as Administrator.

Before the Creators update dropped Homegroup, both machines could easily sync files once they were set up as part of the same Homegroup.

After the Creators update, file sharing was suddenly impossible. The Surface Book could not see file shares on the Desktop PC anymore.

I got that part working again by reinstalling SMBV1 as described here (https://appuals.com/fix-cannot-access-network-shares-after-update-1709/).

PROBLEM

After adding the local account to the permissions of the remote folders with Give Access To, I can see and browse my remote shared folders from the Surface Book.... for a time. 

A day or so after I add the account to the permissions of the folders, I suddenly get Access Denied when trying to write back to the remote folders. 

WORKAROUND

The only solution I found so far is to remove the local account from the permissions using Give Access To and re-add the same account again.

That allows me to synchronize files and write to the remote folders for a time. Until it gets locked in read only mode again. 

Obviously, this workaround is getting old fast since I have to remove and re-add permissions for a dozen folders each time I want to synchronize them.

So in short, I can share access to remote folders for about a day or so before suddenly losing write access again.

I haven't found any comments about this particular issue so I am wondering if it can ring a bell for anyone here.
