¡firedog say:
There have been many reports of this error affecting charter.net IMAP accounts, with so far no resolution. It will take some determined troubleshooting involving generating logs of the client/server traffic to discover why the server is returning a BAD_RESPONSE. In the meantime, you might review all the IMAP settings to ensure that there's no problem with them. Here are some suggestions - you may find on experimentation that different settings work better:
 
  1. Reset the list of IMAP folders
    Right-click on the account name in the folder pane and select Show/hide IMAP folders dialogue. Click Reset List to refresh the folder list, then use the buttons to show only the folders you want to see in the folder pane. Note the exact names of the 'special' folders.
     
  2. Map the local special folders to the IMAP folders on the server
    Right-click on the account name in the folder pane and select Properties.
    On the IMAP tab, select Store special folders on IMAP server,  then enter the server's names for the special folders in the boxes. If the folder for junk email at the server is called Spam, then enter Spam (not spam or SPAM) into the box alongside Junk email path and so on. You may have to provide a Root folder path to make this work properly.
     
  3. Make Deleted items folders work the same way for all accounts
    When viewing your message list, press Ctrl-Shift-O for Options.
    On the Advanced tab, select Use the 'Deleted Items' folder for IMAP accounts.
     
  4. Check for extra folders at the server
    Sign in to the account in webmail and check that there's only one each of folders for sent items, deleted items and junk. If there is more than one (e.g. both Sent items and Sent), move messages out of the non-native folder into the native one, then remove the empty non-native one, which will have been synchronized from Windows Live Mail earlier.
    Back in Windows Live Mail, repeat (1).

One user found that a similar error cleared when he moved many old messages out of the server inbox and deleted a lot of messages he didn't need to keep, then emptied the server's deleted items folder. This was some time ago, before any recent change that Charter may have made to their server configuration, so it won't necessarily help you. It won't hurt, though, and it's free to try.
