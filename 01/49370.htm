After doing research for a couple of hours, this simple solution eventually worked for me (Fedora 24 on a Dell XPS 13 9350):

Grub was still on my system, but was bypassed after the Windows update had changed the information in the master boot sector (or what that is called).

sudo grub2-install /dev/sda

sudo grub2-mkconfig -o /boot/grub2/grub.cfg

That fixed it.

This works with UBUNTU 16.04 / 14.04 and Windows 10 on a x64 architecture laptop. I've had to do this a couple of times so I'm confident it can work, however, I'm not sure entirely WHY it works (I got it of one of the Ubuntu sites) and someone more expert than me in Ubuntu can probably do better and cut out redundant stuff.

First, you need a dvd or a memory stick with a bootable version of ubuntu on board - you can download it within Windows and blow it to the media of choice - I use dvd.

Ensure your pc can boot from the media you gave selected, and ensure that that media is first choice for booting. This requires a BIOS approach. Also, ensure that secure boot is DISABLED in BIOS, or your system will never boot into the grub loader to allow a LINUX selection. You may want to check this first before going into my recovery routine.

OK here goes:

1) boot into your media linux, and select the "Try Linux without Installing" option

2) You need to get the Grub-Repair program from the apps store:

   a) pull up a terminal

   b) type "sudo add-apt-repository ppa:yannubuntu/boot-repair"  <CR> .... you will be asked for a password

   c) type "sudo apt-get update" <CR>

   d) type "sudo apt-get install -y boot-repair && boot-repair" <CR>

3) The boot-repair program will start up.

4)select the "Advanced" tab, and select "grub installation and repair" ... follow the instructions given (may be differences depending on the version of"grub-repair" downloaded

5) remove Ubuntu media, reboot to Windows

6) open an administrator command terminal (not Powershell)

7) type "bcdedit /set {bootmgr} path \EFI\ubuntu\shimx64.efi" <CR>  //you should get a confirmation that the change has been successful

8) exit Windows and restart holding whichever function key forces a boot into your BIOS resident OS boot list - on my HP Envy its F9 [note this is not a BIOS entry]. This may be a recovery menu of some description!

9) from the OS list, select Ubuntu. If this doesnt work, you may be able to manually search from within the recovery menu for a boot file - if you can look for grubx64.efi and see if you can boot from that. In my recent Windows disaster, I had to do this, but in less recent ones it hasn't been that hard as the OS selection came up and worked.

10) In Ubuntu, pull up a terminal.

11) Run the following

   a) type "sudo cp /boot/efi/EFI/Microsoft/Boot/bootmgfw.efi bootmgfw2.efi" <CR>

   b) type "sudo cp /boot/efi/EFI/ubuntu/grubx64.efi /boot/efi/EFI/Microsoft/Boot/bootmgfw.efi

   c) type "sudo gedit /etc/grub.d/25_custom  EDIT the line containing "bootmgfw.efi" to "bootmgfw2.efi"

   d) look at /etc/default/grub for a file called "25_custom"... if it exists open it and make the same edit

    e) type "sudo update-grub" <CR>

you should now be able to boot into a grub menu allowing you to select operating systems!

Firstly, you must set the 'Windows DVD Player' as the 'default' player for the .VOB file extension, being the file type for DVD recordings.  To do this you can open the 'System' configuration tools in 'Settings' and on the left click on 'Default apps'.  The DVD Player option may not show, so click on 'Choose default application by file type'.  In the long list presented scroll down to find the .VOB file type (using the wheel in your mouse is best for this).  If the .VOB file type is associated with anything other than the Windows DVD Player it needs to be changed.   • When was the last time it was working?
  • Do you remember making any changes to the computer recently?

First I would suggest that you disconnect any external drive connected to the computer except the mouse and keyboard.

This error message might occur if there are some corrupt system files on the computer. Would suggest that you run a SFC scan which would scan for corrupt system files on the computer and replace them.

 

Steps to perform SFC scan:

1. Click the Start button

2. From the Start Menu, Click All programs followed by Accessories

3. In the Accessories menu, Right Click on the Command Prompt option

4. From the drop down menu that appears, Click on the 'Run as administrator' option

5. If you have the User Account Control (UAC) enabled you will be asked for authorization prior to the command prompt opening. You may simply need to press the Continue button if you are the administrator or insert the administrator password etc.

6. In the Command Prompt window, type: sfc /scannow and then press Enter

7. A message will appear stating that 'the system scan will begin'

8. Be patient because the scan may take some time

9. If any files require replacing SFC will replace them. You may be asked to insert your Vista DVD for this process to continue

10. If everything is okay you should, after the scan, see the following message "Windows resource protection did not find any integrity violations"

11. After the scan has completed, Close the command prompt window restart the computer and check.

The scan may take some time, so be patient. Windows will repair any corrupted or missing files that it finds. If information from the installation CD is needed to repair the problem, you may be prompted to insert your Windows Vista.

How to repair the operating system and how to restore the operating system configuration to an earlier point in time in Windows Vista

http://support.microsoft.com/kb/936212
