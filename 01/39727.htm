To fix an unspecified error occurred during System Restored 0x80070005 in Windows 10:

1. Type Services.msc in Start Menu Search Box, hit Enter.

http://technet.microsoft.com/en-us/sysinternals/bb896653.aspx


Look at what the command line used to launch the process was, what the actual path of the running process is ( to verify it's not malware masquerading as a legit process ), and what threads the process currently has running. All of these are available through Process Explorer


Beginning with Chipset, update drivers


Explanation

If you’ve been around Windows for any amount of time, you’ve seen the zillions of *.dll (Dynamic Link Library) files in every application folder, which are used to store common pieces of application logic that can be accessed from multiple applications.

Since there’s no way to directly launch a DLL file, the rundll32.exe application is simply used to launch functionality stored in shared .dll files. This executable is a valid part of Windows, and normally shouldn’t be a threat.

Note: the valid process is normally located at \Windows\System32\rundll32.exe, but sometimes spyware uses the same filename and runs from a different directory in order to disguise itself. If you think you have a problem, you should always run a scan to be sure, but we can verify exactly what is going on… so keep reading.

There are various ways to run Chkdsk~


Start > Search bar > Type cmd (right click run as admin to execute Elevated CMD)

Elevated CMD should now be opened, type the following:

chkdsk x: /r

x implies your drive letter, so if your hard drive in question is letter c, it would be:

chkdsk c: /r

Restart system and let chkdsk run.



    Open the "Computer" window
    Right-click on the drive in question
    Select the "Tools" tab
    In the Error-checking area, click <Check Now>.


Press Windows Key + R and type powershell.exe in the run box

get-winevent -FilterHashTable @{logname="Application"; id="1001"}| ?{$_.providername –match "wininit"} | fl timecreated, message | out-file Desktop\CHKDSKResults.txt

This will output a .txt file on your Desktop containing the results of the chkdsk.

patrick barker say: Hi,

The attached DMP files are of the UNEXPECTED_KERNEL_MODE_TRAP (7f) bug check.

This bug check indicates that the Intel CPU generated a trap and the kernel failed to catch this trap.

BugCheck 7F, {8, fffff801c6a1ee70, ffffd00020fb1ff0, fffff800041c12d4}

The 1st parameter of the bug check is 0x00000008, or Double Fault, indicates that an exception occurs during a call to the handler for a prior exception. Typically, the two exceptions are handled serially. However, there are several exceptions that cannot be handled serially, and in this situation the processor signals a double fault. There are two common causes of a double fault:
  • A kernel stack overflow. This overflow occurs when a guard page is hit, and the kernel tries to push a trap frame. Because there is no stack left, a stack overflow results, causing the double fault.
  • A hardware problem.

DNS Srever might be unavailable - Microsoft Community

rick dc say:

DNS Srever might be unavailable

I keep getting connectivity issues, with the message stating that my DNS server might be unavailable. The Microsoft software that checks connectivity says there is one there (yellow triangle), but fails to solve the problem. How do I determine if the problem is with the ISP, or on the computer. My suspicion is that it is the ISP because the problem appears largely an intermittent issue.
balaji kundalam say:
Go to start->type cmd->right click on it->click Run as administrator.

Type ipconfig /flushdns and press enter.
Now, check the issue persists.

If it does, try the following steps sequentially:

1. Type ipconfig /all and press enter.
Note the default gateway. Also, make sure the IP address does not start with 169.254.xxx.xxx.
If the IP address starts with 169.254.xxx.xxx, DHCP is disabled or there is an issue with the router/modem regarding DHCP service. Internet connectivity is affected if DHCP is disabled. Resetting the modem should solve the issue.

2. Type ping <the default gateway address> [for eg, ping 192.168.1.1] and press enter.
Make sure you see 0% Loss.
If this step is true, there is no issue with the router/modem. But if DHCP is disabled, contact the router manufacturer support for further assistance.

3. Type ping <a website address> and press enter. For eg, ping www.google.com.
Here also, make sure you see 0% Loss.

4. Use tracert/pathping to check any website.
For eg, pathping www.microsoft.com or tracert www.microsoft.com
For more information on tracert and pathping have look at here:
http://technet.microsoft.com/en-us/library/cc940123.aspx

If steps 1 and 2 fail, contact the router/modem manufacturer support for assistance.
If steps 3 and 4 fail, contact your ISP. The issue is on the ISP side.

Note: If you use ping with Microsoft website, you won't get a clear reply. That's why Google is suggested here.

This will remove the update and you can try redownloading it. 

2: Make sure you have enough storage space
 

The latest console software update requires at least 190 megabytes (MB) of storage space (or 256 MB if you live in Japan). If you don’t have enough free space, you need to move or delete some items. See Copy, move, or delete content.
 

3: Clear the system cache. Removing the temporary files on your console might solve the problem. See Clear your Xbox 360 system cache.
 

After you clear the cache, try the update again.
 

4: Reinsert the storage device
 

  • If you’re using an Xbox 360 Memory Unit or a USB flash drive instead of a hard drive, remove and reinsert the Memory Unit or USB flash drive.
  • If you have a hard drive, try removing and adding the hard drive again.


After you replace the storage device, try the update again.
 

5: Copy the update to removable media and install it from there. Here’s how:
 

    • Copy to a USB flash drive
       

Note The USB flash drive must use the FAT32 file system.
 

      1. Plug your USB flash drive into a USB port on your computer.
      2. Download the update file.
      3. Click Save to save the console update .zip file to your computer.
      4. Unzip the file.
      5. Copy the contents of the .zip file to the root directory of your USB flash drive.
      6. Unplug the USB flash drive from your computer.
      7. Plug the USB flash drive into a USB port on your Xbox 360 console.

    3. Unzip the file.
    4. Insert a blank, writeable CD or DVD into your computer.
    5. Copy the contents of the .zip file to the root directory on your CD or DVD, and write (burn) the files to the disc.
    6. Remove the disc from your computer and insert it into the disc drive on your Xbox 360 console.
    7. Restart the console.
    8. When the console restarts, the installation program starts automatically. Select Yes when you’re prompted to update the new console software.
