patrick barker say: Thanks very much!

The attached dumps are almost entirely made up of the following bugcheck

WDF_VIOLATION (10d).

The Kernel-Mode Driver Framework was notified that Windows detected an error
in a framework-based driver

In all of your dumps, it's pointing to the culprit MijXfilt.sys on the stack, etc, which is the MotioninJoy simulate Xbox 360 controller (virtual Xinput device) driver: http://www.motioninjoy.com/blog/moti...-xinput-device

In the *10d dumps, the process at the time of the crash is also very consistent, which is DS3_Tool.exe.


To my knowledge, using this specific driver rather than the official drivers is not recommended... I've seen MijXfilt.sys tons in the past in cases of analysis. Remove this driver and install the official driver:

http://www.microsoft.com/hardware/en...-00004#support

As far as I know, removing MotioninJoy via add / remover programs uninstall list doesn't work and the driver still remains, so you must uninstall it via the uninstall list, AND THEN navigate to c:\windows\system32\drivers and delete MijXfilt.sys.

alicesloan say:

SanDisk not recognized on the computer.

my pc. with win8.1 does not acknowledge my SanDisk, nor does it acknowledge my Polariod iF045 when connected by cable.  What has happened?  How do I get photos transferred to computer now?

a. user say:

Hi Alice,

If the PC is not recognizing USB devices, then there could be an issue with the USB port or the USB drivers. I would suggest you to try these steps and check:

First, I would suggest you to try using a different USB port to check if the issue is with the USB port.

Method 1:

Run the hardware and devices troubleshooter which can automatically detect and fix most of the incorrect settings. Refer to these steps:

  1. Press Windows key +F to bring up the search Charms bar.
  2. Select settings in the drop down menu below search and type Troubleshooting.
  3. Click on Troubleshooting and select the view all option on the left pane.
  4. Click on Hardware and devices troubleshooter and follow the prompts.

Method 2:

If the troubleshooter didnt detect any incorrect settings, then I would suggest you to manually check the Device manager for any errors. Refer to these steps:

  1. Press Windows key + X, select Device manager.
  2. Expand the Universal serial bus controllers section.
  3. Check if there is a yellow exclamation mark or a red cross mark any USB controllers.
  4. Right click on device and select properties.
  5. In the General tab, check the device status.

If the device status reports any driver issues, reinstalling the driver could resolve the issue. Visit the computer manufacturers website to download the latest USB driver and BIOS, chipset drivers. Save the drivers in a temporary location. Now uninstall the current USB drivers by following these steps:

  1. Press Windows key + X, select Device manager.
  2. Expand the Universal serial bus controllers section and locate the USB controller.
  3. Right click the device and select Properties.
  4. Click on the driver tab to check the current driver installed for that device.
  5. Click on the uninstall button to uninstall that driver.
  6. Now restart the computer. Windows will boot the computer with the generic drivers. Check if the issue persists with the generic drivers.
  7. If yes, install the latest drivers that you downloaded and check. Also install the latest BIOS and chipset updates.

Try these steps and let us know the results.
