Activate the DNS Client Service, start it manually and switch it to run automatically in future and it might be that also on your computer all the apps will work fine in future!

bill block say:

I ran into the same problem and I discovered that the DNS Client Service was not running.

 

To verify that the DNS Client Service is running:

 

  • From the start screen search for "Command Prompt"
  • Run the command prompt
  • type in "services.msc"
  • find DNS Client in the list
  • right click on it and check its properties
  • Startup Type should be Automatic
  • Press the Start button under Service Status to start the service

Good luck!

johnny sinister say:

Microsoft uses the achievements as the criteria for completion, not the gamerscore. If you look at your games played on xbox.com, there will be a completion bar for gamerscore, a completion bar for achievements (i.e 45/50) and a completion percentage. The completion percentage is tied directly to the achievement completion bar.

For example, on Superman the Video game, I have 1000/1000, but I did not unlock the achievement for using a cheat code (it is a 0GS achievement). So, it says I have 17/18 achievements and only 94% completion.

Xbox.com uses the number of achievements to calculate the completion percentage. But if you calculated it gamerscore-wise I'd still consider that a good representation of your completion percentage. john ado say:

Hi,

This occurs when the program you are trying to run requires the MSVCP140.dll file to be present on your computer system.  MSVCP140.dll is a Microsoft C Runtime Library file of size around 626 KB, located in the System32 folder, and is installed by Microsoft Visual Studio. If any DLL file goes missing or gets corrupted, you can see such error messages on your screen.

With that in mind, we'd like to know:

  • Does it only affect a specific program?
  • Were you able to launch this program before?
  • Have you tried reinstalling the program?

If you have reinstalled the app the issue still persists, download and install the Microsoft Visual C++ Redistributable for Visual Studio here.

lemp say:

Contrary to what you may read elsewhere, you do NOT need to regularly update your drivers, particularly printer drivers. 

Moreover, if you do want to update a printer driver (because (a) you know that the new version fixes a specific problem you have or (b) you know that the new version adds a feature that you want or need), you should ONLY get the updated driver from the printer manufacturer. 

The only circumstance for which you should get a printer driver from Windows Update is when no driver is available from the printer manufacturer for your edition of Windows (which is, unfortunately, becoming more common).

If your subject line is accurate and the update in question is coming from Windows Update, right-click it (in the list of Windows Updates) and select "Hide Update."

For curiosity only, what is the specific model of your HP printer and what is the KB number of the update?
