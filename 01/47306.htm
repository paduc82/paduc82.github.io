xbf diego m say:

Hey JumboGiraffe,

I'd be happy to provide some input.

To answer your question, it's not normal for an Xbox to occasionally not read a disc. The software or operating system may have gotten glitched due to the disc being ejected midway through the installation. It seems once you canceled the download, the Xbox read the disc. That process may have corrupted the installation and therefore was affecting the software.

Usually when a disc is being read for the download, you must either pause or cancel the download before you take out the disc.

Anytime unusual things like this happen, we recommend to fully reboot the console by holding the power button on the Xbox for about 10 seconds until it automatically shuts off. Then power cycle the Xbox by unplugging the power cord from the Xbox and the wall outlet for 1-2 minutes.

If the Xbox is not able to read multiple discs and happens frequently not occasionally, then the disc drive may be going bad. However, I think the issue was related to a corrupt installation and there are other troubleshooting options you could try before pinpointing the issue with the disc drive itself.

I hope that information helps!

Diego,

If you're ready to begin, let's start with the basic system checks

Please run a full CHKDSK and SFC scan....

Click on Start > All Programs > Accessories

Right-click on the Command Prompt entry

Select Run as Administrator and accept the UAC prompt - the Elevated Command Prompt window should pop up.

At the Command prompt, type

CHKDSK C: /R

and hit the Enter key.

You will be told that the drive is locked,

and the CHKDSK will run at he next boot - hit the Y key, and then reboot.

The chkdsk will take a few hours depending on the size of the drive, so be patient!

After the CHKDSK has run, Windows should boot normally (possibly after a second auto-reboot) - then run the SFC.

SFC -System File Checker - Instructions

Click on Start > All Programs > Accessories

Right-click on the Command Prompt entry

Select Run as Administrator and accept the UAC prompt - the Elevated Command Prompt window should pop up.

At the Command prompt, type

SFC /SCANNOW

and hit the Enter key

Wait for the scan to finish - make a note of any error messages - and then reboot.

Copy the CBS.log file created to your desktop (you can't manipulate it directly) and then compress the copy and upload it to your SkyDrive (http://skydrive.live.com) and post a link to it so that I can take a look.

Post a new MGADiag report with details of any error messages encountered.
