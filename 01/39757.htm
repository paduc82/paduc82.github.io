michelle vil say:

Hello Ernesto,

If you're going to clean install your HDD drive, the files on your SSD drive will not be deleted as long as you remove it (SSD driver) prior to clean installing. However, if you'll going to perform it the other way around meaning clean install the SSD drive, it will definitely delete all the files that are saved there.

About your other concern, the files that you can transfer are those personal files such as, music, documents etc.

To know more about performing a clean install, we highly suggest visiting these helpful articles posted by Greg Carmack and Andre Da Costa: Clean Install Windows 10 & How to: Perform a Clean Install or Reinstall of Windows 10.

If you have any other questions or concerns, feel free to share them with us here. Your input is very much welcome in our Community.

michelle vil say:

You're very much welcome. To answer your question, yes, you can do that. Aside from disconnecting your SSD, you can also remove all the external cables from the computer except mouse and keyboard before performing the install.

¡firedog say:

No promises, but this is the procedure:

Close down all running programs.

  1. In Windows Explorer (Windows key + E), navigate to
    C:\Users\{user}\AppData\Local\Microsoft\Windows Live Mail
    where {user} is your Windows user name. This is your store folder where all your messages and account details are stored. Copy as path.
  2. Type cmd into the start search box and press Ctrl-Shift-Enter (all three keys simultaneously). You'll have to give UAC permission, and then this should open an administrator's command window with a prompt like C:\Windows\system32>
  3. Type cd and a space at the prompt, then right-click and select Paste. Press Enter. This should change the prompt to
    C:\Users\{user}\AppData\Local\Microsoft\Windows Live Mail>
  4. Back to Explorer. Copy the whole folder to somewhere safe - e.g. your Documents folder -  so we can just put it back if everything goes pearshaped. This might take some time if you have a lot of messages.
  5. Back to the command window. Type very carefully into the window at the prompt
    esentutl /r edb
    Note the spaces on either side of /r.
    This should recover your database, i.e. restore it to the state it would have been in had the last close-down process not been interrupted. With any luck, you'll see a message about success, something like Operation completed successfully in 10.112 seconds.
  6. Now let's try to repair the database.
    In the command window, type at the prompt
    esentutl /p mail.msmessagestore
    Again, there are spaces each side of the switch /p.
    You may have to dismiss a warning about potential loss of data. This might happen; you may lose the last few messages and transactions (moves, deletes, replies etc.) before the database went south. It's presumably a small price to pay for getting the thing working again.
    If you see a message saying Operation completed ..., you're probably in luck. You will probably also see a message about backing up the database. Let's leave that to Windows Live Mail.
  7. In the command window, type exit to close it.
  8. Reboot the computer, and then try launching Windows Live Mail.

If you meet an insurmountable hurdle at any stage, stop and post back here saying what happened. I'll be watching from about 08:00 on Thursday morning, so pull over to the hard shoulder and wait for rescue. Good luck!
