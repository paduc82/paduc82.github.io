prashant kamani say:

Hi Nirmal/Slance310. Thanks for your response. The problem is not about how to set permissions for all folders. I am a CS engineer myself and know how to do that :) No, I can't just go up to the lowest common parent and change the permissions there for all folders, since I would need to enable inheritance for all subfolders to inherit from this folder's permissions, and that is not what I want to do (there would be other subfolders that are not part of any libraries, and their permissions would be modified too).

 

The problem is specifically about Homegroup messing up permissions of shared folders. Instead of just ADDING RX permissions for the security group HomeUsers, it REPLACES existing permissions with RX permissions for HomeUsers. I have renamed the topic back to reflect this issue. It is very easy to repro:

1. From start screen, type HomeGroup, and choose HomeGroup from search results in Settings.

2. If Documents is shared (by default, it isn't), unshare it.

3. Check permissions of your Documents folder. Note that it inherits from the parent as expected and has Full Control for Admins, System and yourself.

4. Go to HomeGroup again and share Documents. Wait a minute for permissions to propagate.

5. Check permissions of your Documents folder. Note that ALL existing permissions are gone, and the only permission left is RX permissions for the group HomeUsers.  You can see pictures of Before/After/Expected permissions here: http://sdrv.ms/TrtYKu

 

Basically the issue is that when a folder is shared with the HomeGroup, the system needs to grant the security group HomeUsers RX permissions to the folder. If the folder is inheriting permissions from the parent, the only way these new permissions can be granted is by disabling inheritance. However, when the system disables inheritance from the parent, instead of COPYING existing permissions from the parent, it REMOVES existing permissions. This causes the permissions to be messed up, resulting in all sorts of issues. To name two - I can't modify anything in Documents folder, and built-in Music and Video apps not seeing any media in your library.
