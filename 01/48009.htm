nithyananda j say:

Hi DioMac,
  
Method 1:

To Customizing the Icon for a File Type without using any third party programs.

a. Click Start, type Regedit.exe and press ENTER

b. Navigate to the following branch:

HKEY_CURRENT_USER \ Software \ Microsoft \ Windows \ CurrentVersion \ Explorer \ FileExts \ .abc \ UserChoice

c. In the right-pane, note down the value data for Program id. This is the user chosen Programmatic Identifier for the file extension.


d. If the User Choice branch does not exist, navigate to the following branch:

HKEY_CLASSES_ROOT \ .abc
e. In the right-pane, note down the (default) value data (e.g. abcfile). This is the Prog ID for the file extension.

f. Navigate to the following key (where abcfile is the Prog ID noted in Step 2 or Step 4 above):

HKEY_CLASSES_ROOT \ abcfile \ DefaultIcon
Note: If the Default Icon key does not exist by default, you need to create it manually.

g. In the right-pane, double-click (default) and mention the path to your custom icon (.ico file or reference to an icon resource from an EXE/DLL file).

For example, to set the 260th icon in Shell32.dll (contains a text pad icon), type the following path:

C:\Windows\System32\Shell32.dll,260

h. Exit the Registry Editor.

Restart Windows for the icon change to take effect. In case it doesn’t, you can force an icon refresh manually, or try rebuilding the icon cache using Disk Cleanup.

Hi David,

 

This bug check indicates that the Intel CPU generated a trap and the kernel failed to catch this trap. This trap could be a bound trap (a trap the kernel is not permitted to catch) or a double fault (a fault that occurred while processing an earlier fault, which always results in a system failure).

 

To assist you further with this issue, I would like to know some information regarding the same.

1. Are you able to boot to desktop?

2. Have you made any changes to the computer prior to this issue?
