a. user say:

First of all, fltmgr.sys is not in the system32 folder - it is in this folder:

%Systemroot%\system32\drivers

Second of all, if you were to rename the fltmgr.sys (or any of the 3498 XP Protected Files) it would be quickly and silently replaced by Windows File Protection (assuming WFP is not also somehow afflicted).  That would replace the file from the backup copy in the dllcache folder.

When you rename a Protected File (causing it to become "missing"), you will see a message in the Event Viewer System log like this when the file is replaced:

Event Type:    Information
Event Source:    Windows File Protection
Event Category:    None
Event ID:    64002
Description:
File replacement was attempted on the protected system file c:\windows\system32\drivers\fltmgr.sys. This file was restored to the original version to maintain system stability. The file version of the system file is 5.1.2600.5512.

The file replacement will probably happen before you had a chance to complete your next breath, so there would be no time for a next step.

Finally, the System File Checker (SFC) only replaces missing or corrupt files in this folder:

%Systemroot%\system32\dllcache

Running sfc /scannow will not replace any files in any other folder especially not the system32 or system folders.

If the fltmgr.sys file needs replacing, it is not hard to replace, but not using those methods.
