I suggest you to follow the below troubleshooting methods and check if it resolves the issue.

  • Reinstall the Photo app and check if the issue persists.

If issue persists, I suggest you to uninstall and reinstall the Photo app. Here's how:

1. Press Start, and type Powershell.

2. From results, right click on Windows PowerShell and select Run as administrator option.

3. Type Get-AppxPackage and press Enter.

(This command will output a long list of all installed apps containing lots of information about each app).

4. Type Get-AppxPackage | Select Name, PackageFullName and press Enter.

(This command will show only the name of the app and its PackageFullName information).

5. Select the PackageFullName information of the desired app and copy it.

6. Type Get-AppxPackage PackageFullName | Remove-AppxPackage and press Enter.

(This command is to remove the app from the system, replace PackageFullName with the copied information with Step 5).

For example, if you want to remove Photo app, the command to remove it will be as following:

Get-AppxPackage *photos* | Remove-AppxPackage

7. Wait till completion of the command, and check if the app is uninstalled, if not restart the system and

check.

Note:

If you want to uninstall the app from all user accounts, use the following command format:

Get-AppxPackage -allusers PackageFullName | Remove-AppxPackage

If you want to remove the app from any particular user account, use following command:

Get-AppxPackage -user username PackageFullName | Remove-AppxPackage

(Replace username in the above command with the correct user account name).

8. Launch the Windows Store and reinstall the Photo app.

Hope this helps. If the issue remains unresolved, please get back to us and we would be happy to help.

I use system images.  I had a problem ages ago with a failed Restore point attempt and have relied on system images ever since.  I used to think that system images were just things that IT departments used but they have become my first response to any problem that I cannot immediately solve or that has symptoms that defy diagnosis.

There are a few things worth pointing out.

1  Images are made in a folder called WindowsImageBackup that comes straight off the root of the drive where it has been made.  So you'll see, for example, F:\WindowsImageBackup.  System images are best made onto an external drive so you can recover from a failed hard drive.

2  If you make a new image then it will be saved on top of the old one. Both will be in F:\WindowsImageBackup.  Windows will make its own decisions about which bits of the old one to keep and which to overwrite.  So you can never be sure just what you are restoring.  Windows is probably making excellent decisions but ....

3  The way to avoid any confusion is, when preparing to make a new image, to move the existing image somewhere else such as F:\ImageLibrary\20150517\WindowsImageBackup.  You'll get a UAC prompt ["Elevation UI" prompt] so you'll need to give Admin permission for the move.  It's worth using a folder name with the date in it to avoid confusion later on.  That frees up the drive to be able to accept a new image at F:\WindowsImageBackup that is completely independent of the old one.  In moving the old image, keep the whole of WindowsImageBackup together - you'll probably have a look and see how it is structured inside but Windows will not accept anything that has been altered or extracted if you later try to restore an image.  You can keep as many images in your image library as you wish as long as you have a separate folder for each one and as long as you don't tamper with anything inside WindowsImageBackup.  I make images onto two external hard drives alternately and I keep two or three images on each inside an image library - if I get a problem but realise it's been building up for two weeks then I can restore from an image made before any symptoms started or before a glitch occurred that I now guess might be the cause even though I don't understand what happened or why.

4  If you decide to restore an image, decide if you want the latest one or one that's in the library.  If it's the latest one that you need then you are ready to restore.  If not then you need to move the latest WindowsImageBackup folder into your library and move the chosen WindowsImageBackup folder from the library into position at F:\WindowsImageBackup or G:\WindowsImageBackup or H:\WindowsImageBackup.  Windows goes not care which drive you are trying to restore the image from as long as WindowsImageBackup is off the root of the drive.  If your computer is so badly corrupted that you cannot use it to move images around then either decide to use the latest image [as you will have left it in the correct folder] or use another computer to move things around.

5  You can restore an image from within Windows or by booting from a Windows 7 System repair disk or installation disk.  I prefer the booting method as it has never let me down.  The procedures are given at

Recovery method 11 - Windows Backup System Image restoration using System repair disk          [PDF version]

Recovery method 12 - Windows Backup System Image restoration using Windows 7 Installation Disk          [PDF version]

6  Once you have restored and checked your system, decide which is the image you are most likely to want to restore from in the future.  Keep that one in F:\WindowsImageBackup and the others in the image library.

7  If you have separated your system drive from your data drive [either by having two hard drives or by partitioning a single disk] then a system image is likely to be less than 40GB and take only 25 mins or make [and 25 mins to restore from].  Do you know what I am talking about here?

8  A system image is a replica of the system drive at the moment that it was made.  Any changes made to that drive since the image was made are lost.  Other drives are pretty safe - I always back up my data drive before restoring a system drive image but it has never gone wrong and messed up the data drive.

9  Windows system images are not tolerant of hardware changes - if you ever have a complete hard drive failure then you ought to be able to recover the last system image onto a new hard drive if that hard drive is of the same type as the old one.  To make the process easiest, the new one should need the same driver as the old one so buying a replacement of the same make, model & size is best.  You would probably decide to get a larger one though and that can make the restoration more difficult.  If you are forced to replace the hard drive and you decide to get one of a different type [larger or just a different model] then you could have one try at restoring before then buying Acronis TI 2015 which can cope with restoring Windows system images despite hardware changes a lot better than Windows own utilities can.  When I had to replace my hard drive & bought one of a different make-model, I restored my old image but I could not use Windows update or update drivers from Device manager.  I gave up and installed everything from scratch.  Since then, I have paid attention to relevant discussions in this forum and have seen many of the competent contributors to this forum recommend Acronis TI.  The latest version [2015] actually boasts of its ability to cope with hardware changes.  Personally, if I have to change hard drives, I will buy Acronis TI 2015 without even waiting to have a single restoration attempt without it.  There are lots of alternatives but it's Acronis that I keep seeing recommended.
