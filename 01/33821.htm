john john __ say:

I have just one question more. Why it's important to have the ports 137-139 are open on the router if this is about a local traffic (Layer 2)?

That is a good question.  Everything was working properly before the router was changed, you are not double natting and all the machines are obtaining DHCP address from the router (no static addresses on any of the workstations) so we can pretty well eliminate subnetting errors or browsing accross subnets as possible causes.  The only thing that was changed is the router, nothing was changed on the workstations, so I'm left to believe that the router is blocking local NetBIOS, I don't know where else the problem would lie.  If you still have the old router plug the machines back in it and see if things work properly.  But you are right, you wouldn't want to open the NetBIOS ports to the outside network, I just think something in the router is blocking NetBIOS.

As for Master Browser, general guidelines:

  •  Make sure that all the machines are on the same subnet and same (correct) subnet mask.  NetBIOS cannot browse across subnets without additional hardware or without a WINS server.  If all the machines are on the same subnet but if the subnet mask is not properly configured the Master Browser may not work properly.   These steps here are not usually a problem when all the machines are configured to obtain  their IP addresses automatically and you have a router handling DHCP, if you assign static addresses do not neglect this important advice.
  • Select which machine will act as the Preferred Master Browser and which ones may act as Potential Browsers, make sure that any machine not selected to act as one of the above is not participating in "browser elections".  Assign the Preferred Master Browser duties to a machine that is always or usually always turned on, assign "Potential Browser" duties to a machine that is also usually turned on, a machine that can take over if the preferred computer is not turned on.  DISABLE the master browser on ALL other machines, a machine infrequently turned on is a poor candidate for master browser duties and may just cause "Browser Wars". 

Disabling the Computer Browser service will in effect prevent a machine from participating in browser elections and becoming a master browser but the proper way to disable the master browser is by setting the IsDomainMaster value to "False" and the MaintainServerList value to "No" at the HKLM\SYSTEM\CurrentControlSet\services\Browser\Parameters key.

On the Preferred Master Browser set the IsDomainMaster value to "True" and the MaintainServerList value to "Yes".  On the potential browser set the IsDomainMaster value to "False" and the MaintainServerList value to "Auto".  Unless the computer is configured as the preferred master browser, the value of the IsDomainMaster entry is always set to False or No .

But then all was working properly before the router change, it only stopped working after the new router was introduced into the network so I don't see how the master browser settings would have changed, it all seems to be pointing at the new router.  Did you look in the event log for relevant errors?

I think ISP routers are fine for limited household use but otherwise I don't really like them handling small office/work networks, I always prefer having ones own router and placing the ISP router in bridge mode, that way there usually is the least disruption to the work network when dealing with ISP issues.

vibroplex say:

Most likely there's a port blocked... ports can be blocked in your router, at your ISP.  Some NAT problems are subtle and don't appear in the Test Xbox Live Connection test.

Since the Xbox 360 and Games For Windows Live uses different ports than other PC & Mac apps use, you can't see the effect of a port blockage except on the Xbox or in GFWL.

 

A sequential power cycle is the first step in clearing a port blockage:

1) turn off the Xbox (you do not need to turn off the PC for this) then

2) unplug power from the router (Some folks have a combination modem-router AKA 'gateway'. If so, skip step 3).

3) unplug power from the modem.

4) Wait at least 30 secs then replug the modem (or gateway) power.

5) Wait 'til the light on the front of the modem (or gateway) which says ONLINE, READY, CABLE or DSL is on solid (not flashing), then plug in the router. If you have a gateway instead of a modem and router, then when that light says it's ready, skip step 6.

6) Wait until the WiFi light on the router is on; you may not use WiFi, but it's still a good indicator your router is finished booting up.

7) Turn on the Xbox.  For more background on Ports, see whatismyipaddress.com/port
