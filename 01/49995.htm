According the Dell manual, it looks like the 4700 has a SATA hard drive:

Externally accessible:    one 3.5-inch drive bay
                                  two 5.25-inch drive bays
Available devices         SATA hard drives, floppy drive, USB memory devices,

That means that even if you use a genuine bootable XP installation CD to get into the XP Recovery Console, you will not be able to sufficiently access the HDD to run a chkdsk.  You can usually get to a Command Prompt but things won't go well after that.

That is because the XP installation CD does not come with any SATA drivers on it.  You could take your genuine bootable XP installation CD and through a process called slipstreaming, create a new CD that does have the SATA drivers slipstreamed (integrated) into it, and that is a fun project if you have a lot of time and want to learn a bunch of new stuff but it is also a fair amount of work if you are the geeky type - but why do all that when there are easier ways.

On some systems you can sometimes fiddle with the BIOS to temporarily change the HDD interface to something that the XP installation CD does understand, run your chkdsk  /r  and then change the BIOS back.  Some folks don't understand or feel comfortable messing with the BIOS.

The Hiren's Boot CD doesn't have this problem.  It understands SATA drives just fine and also does not care about any XP Administrator password that sometimes thwarts folks trying to get into the Recovery Console.

That is why I suggest it instead of the Recovery Console - fewer issues and fewer things that might not work, especially if you are not sure if the system has SATA drives or not.  It always works.

There could be something else wrong besides NTFS corruption though, but it never hurts to start with that and then you can at least say you did it. 

Using Hiren's, you can also copy off all your personal files (documents, pictures, music, etc.) to an external USB drive in case you end up having to reinstall - can't do that with the XP Recovery Console since that is not what it is designed to let you do.

Now let's say your friend sends you some XP installation CD, well, if you are going to do a Repair Install, that CD better have the same Service Pack as your installed and unspecified SP or it is not going to work.  Most folks don't have a genuine bootable XP installation CD that is the same SP as their installed SP unless they made it themselves and it still isn't going to help if your system has SATA drives.

Step 2:

How to configure Windows XP to start in a "clean boot" state

http://support.microsoft.com/kb/310353

Note : After you used the clean boot to resolve your problem, you can follow these steps to configure Windows XP to start normally.

a.Click Start, and then click Run.

b.Type msconfig, and then click OK.
The System Configuration Utility dialog box is displayed.

c.Click the General tab, click Normal Startup - load all device drivers and services, and then click OK.

d.When you are prompted, click Restart to restart the computer.
