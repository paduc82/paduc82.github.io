ronnie vernon say:

Hi normwilk

There is a safety net in Windows 7 for this situation.

 

If all Administrator accounts have been removed or changed to Standard, you can boot into Safe Mode. When the log in screen appears in safe mode you will see an account named Administrator. This is the built-in administrator account.

 

There is no password associated with this account, by default. You should be able to simply select that account and log on.

 

Once the safe mode desktop appears, you can go into Control Panel/User Accounts and change your existing account back to administrator or create a new Administrator account.

 

Start your computer in safe mode:

http://windows.microsoft.com/en-IN/windows7/Start-your-computer-in-safe-mode

 

1. Open Control Panel / User Accounts.

 

2. Select Manage another account.

 

3. Select your account or select Create a New Account.

 

4. Select Change account type.

REGISTRY FIX FOR VIDEO_TIMEOUT_DETECTION_AND_RECOVERY FOR WINDOWS 10 - Microsoft Community

raju.msc.mathematics say:

REGISTRY FIX FOR VIDEO_TIMEOUT_DETECTION_AND_RECOVERY FOR WINDOWS 10 64 BIT

In some users experiencing their nvidagpus are crashing while playing games or watching videos, there is a fix available for the above problems.

we need to modify certain registry keys under [HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Control\GraphicsDrivers]

in some windows, we have to create new keys or modify existing keys ,  

before modifying the registry keys take backup and save as backup.reg  

for instructions, how to create backup registry settings visit this Microsoft site here

for the fix , copy and paste all the lines in notepad and save as  TDRFIX.reg 

Windows Registry Editor Version 5.00

[HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Control\GraphicsDrivers]
"TdrLevel"=dword:00000003
"TdrDelay"=dword:0000003c
"TdrDdiDelay"=dword:0000003c
"TdrTestMode"=dword:00000000
"TdrDebugMode"=dword:00000002
"TdrLimitTime"=dword:0000003c
"TdrLimitCount"=dword:00000005

[HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Control\GraphicsDrivers\DCI]
"Timeout"=dword:0000003c

Sources  from Microsoft site here
