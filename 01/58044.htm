a. user say:

Hi Darius,

The error code you got, 0x8007001f - 0x3000d, is due to a problem with user profiles. It can occur due to corrupt registry under HKLM\SOFTWARE\Microsoft\WindowsNT\CurrentVersion\ProfileList or invalid files in the \Users directory.

If a previous upgrade did not complete, invalid profiles might exist in the Windows.old\Users directory. To repair this error, ensure that deleted accounts are not still present in the Windows registry and that files under the \Users directory are valid. Delete the invalid files or user profiles that are causing this error.

Based on the log file, a user-mode driver failed to be added. A driver might be incompatible with Windows 10 version 1709. Can you tell us if you have checked with the manufacturer's website if the computer is compatible with Windows 10 version 1709? Or, you can check this article created by Andre Da Costa to determine if your computer is compatible with Windows 10 version 1709.

A further look in to the log, there are two (2) logs that points to a user profile not existing. However, a log is pointing out that a profile exists but the path could not be found. These three (3) logs could be related, and the profile they are pointing to could be corrupted, causing the upgrade to fail. Kindly check for user profiles in the system that should not be there.

If you're not seeing that kind of pattern in your setupact.log then I doubt you have a similar issue and you may as well skip reading the rest.

As I said I googled and tried setting the ownership of the profile folder at c:\users\mike to the DOMAIN\mike account but as I said that did not work, but i still got the same error sequence in the setupact.log.

I then changed ownership of the c:\users\mike folder to LOCAL\Administrators Group. I did this using Windows Explorer but you need to be an Administrator on the machine to do this. 

In windows explorer open c:\Users

Right click on 'mike' folder (i.e. the specific user folder mentioned in the error) and select 'Properties'

Select the Security Tab and click the 'Advanced' button at the bottom

At the top where it says 'Owner' click the 'Change' link

In 'Locations' Select the local machine instead of the domain

Type 'Administrators' in the find box and click 'Check Names' which should resolve the entry to the Local Administrators group on the machine. click OK

Below the Change link tick the box that says 'Replace owner on subcontainers and objects'

Click on Apply and once its applied, 'OK' out of the dialogs.

Then run the update again. This worked for me but its possible that you might just clear this error and hit 'the next' so check back to the setupact.log to see if its changed and move on with the next issue.

There is one other thing that I think I did at the same time as doing the above that having read another forum post subsequently made me wonder whether it was relevant.
When I was reviewing the registry key mentioned in one of the previous posts HKLM\SOFTWARE\Microsoft\WindowsNT\CurrentVersion\ProfileList I noticed that my profile entry (against my sid and therefore the profile that was causing the error) was the only one with a sub key (I think it was called 'preferences'). I took a chance and deleted that subkey because it stood out as different form the other entries in the list that weren't causing a problem. I cannot now find the link, but I subsequently read a post that included removal of that entry by way of resolving an upgrade issue (might not have been 1709). They were a little more careful than me, they exported the sub key, deleted it, ran the upgrade (successfully) then restored the exported sub key.

Worth checking in your registry too. 

I thought I'd take the time to explain in detail what worked for me as I know how much frustration this upgrade can cause, particularly if, like me, you can't just do a fresh install. As I said the errors in your error log shared similarities with my own, hence why I posted.

I hope it helps, but if it doesn't I hope you find your way around the issue some other way. Happy to answer any specific questions you have on the process I followed.

good luck
