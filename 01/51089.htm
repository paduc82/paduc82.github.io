a. user say:

I think Windows only loads the data in .pf (prefetch) files when you launch the associated application and does not load them at boot (maybe unless the program is on startup). Prefetch works as a sort of cache, that is when you open an application, Windows checks the Prefetch folder, looks at the applications .pf file (if it exists), and uses that as a guide to start preloading data that the application will use.

Three minutes is awfully long boot time though, have you tried performing a clean boot? Use msconfig to disable non-Microsoft services and startup programs. A startup program or service can be one of the offenders (for example, I think Chrome also prefetches data).

Try also if you can reproduce the issue when starting in Safe mode. But yeah, the usual suspects for this issue are the Windows Search, Prefetch, Storage Spaces and BITS services.
