drew polstra say:

Ah, solved. If you run into this error, my problem was that I had plugged in something new that didn't work (an incompatible DVI adapter) the previous day. It didn't do anything at all yesterday, but this morning that was what was keeping the computer from booting. 

So if you find yourself in my situation, ask yourself if you installed anything new since you last restarted your computer.

dax1792 say:

None of this looks unusual.

MsMpEng.exe is Microsoft Security Essentials.

Svchost.exe (netsvcs) hosts Windows Update. Memory use will go up to around 1MB if available during update check or installing MSE definition updates.

An out of memory message when there is available RAM is caused by the paging file being too small. Windows guarantees that it is able to supply virtual memory that programs have requested even if they have not used it. If the committed memory would exceed the sum of RAM and paging file space, Windows begins to reject memory allocation requests.

jasonvey say:

Problem solved!

It's that stupid Fast Access Facial recognition software that Dell puts on the computer that causes it. The fix is fairly basic but a bit tricky.

First, you need to get into safe mode, which is hard to do in 10 if you can't log into your computer. For some inexplicable reason Microsoft decided only to let you reboot into safe mode after you log into your computer normally...which kind of defeats the purpose of safe mode.

There are two ways. First, you can use the Windows 10 clean install tool offered by Microsoft and linked by Wasemulla S above, to create a bootable media. This is probably the easiest way. You will be able to get into repair functions (including rebooting into safe mode) that way.

http://www.microsoft.com/en-us/software-download/windows10

Alternatey you can trick your system into booting into legacy mode, which will allow you to hit F12 right at boot to get into setup. I believe if you interrupt the boot cycle (by turning off the computer partway through--hold down the power button till it shuts down) it will go into repair mode eventually, allowing you to get in here. But the repair ISO is probably a safer bet.

Once you're into Safe Mode you'll be able to get in. The trick now is uninstalling Fast Access. Windows doesn't activate its uninstaller in Safe Mode. You can, however, fix this. Here's how:

https://www.raymond.cc/blog/uninstall-programs-packaged-with-windows-installer-in-safe-mode/ 

As an addendum to the above, your Start menu won't function in safe mode, either. If you right click on the Start button, however, you'll bring up a popup menu with all the options you need. 

Once you have the uninstaller running again, simply go into Control Panel and uninstall Fast Access. Reboot normally. Problem solved!!!

Windows 8.1 Update Fix - Error 0xC1900101 ID 0x30018 and 0x3000D SOLVED!

If you cannot install Windows 8.1 update, try this:

hairyfool say:

If I was to make any change I would use a USB to SATA adapter to connect the (ex) RAID disk to the new build AFTER the system has booted. That way you know that the O/S is not going to make any changes to the drive during the boot when you have no control or visibility.

 

I know many sneer at the software RAID functions within Windows and in business circumstances probably correctly so but it has one advantage in that it is completely non dependant on any hardware. With RAID 1 being just a Mirror it has little performance degradation in comparison with a card based solution.

 

I have also found that RAID based on the Intel chipset migrates very well to another Intel motherboard. Even to the extent that when I plugged one into a small PC with only one SATA interface the RAID config screen showed when the BIOS itself did not support it (hard to do with only one SATA port:))
