Losing time is not entirely a battery problem; it's can be a crystal oscillator problem, too.  Depending on the crystal, your computer may lose or gain time over the couse of the day.  This may even vary as the temperature in you case varies.

If your RTC (Real Time Clock) battery is dead, your system will not "remember" the time nor will it keep time when the system is off.  As suggested, replace the battery if the system can't "remember" the time.  BTW, I've seen these batteries in grocery stores (Kroger)!

However, if your system is connected to the Internet, you can set your system to get the time via SNTP (Simple Network Time Protocol).

This functionality is provided through the "Date and Time" Control Panel.  This can be opened in the normal manner or by right-clicking on the time-of-day display in the task bar and selecting "Adjust. Date/Time".  Once it has opened, select the "Internet Time" tab.  In the upper left corner, there is an "Automatically synchronize with an Internet time server" check box.  Select it.  By default, the timeserver selected is "time.windows.com" and there is an alternative site, time.nist.gov.  Time.Windows.Com is in Redmond, WA (big brother is watching?) and Time.NIST.Gov is at NCAR (National Center for Atomsphiric Research) in Bolder, CO.  It as also *VERY* busy.  There is yet another alternative, use the one of the NTP Pool Project Servers.  In the US, the choices are {0,1,2,3}.us.pool.ntp.org.  These four sites have a list of ~75 servers per site presented three at a time in round-robin fashion (for load balancing).  Windows will use the first one that answers.  So, type in 0.us.pool.ntp.org (1.us..., 2.us..., or 3.us...) and click "Update".  If all is well with your Internet connection, your computer will synchronize with a time server.  BTW, the country code doesn't have to be US.  Try your two-digit county code (CA,UK, MX, ...) or a region (europe, africa, oceania,asia).

The system will set it's clock via the Internet every time it boots and, if you leave your computer on, WindowsXP will, by default, next re-synchronize in 604800 seconds (1 week!).  This can be adjust by modifiying "SpecialPollInterval" in the HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Services\W32Time\TimeProviders\NtpClient key.  I've set mine to re-synchronize three time a day (i,e, 8 hours, 0x7080 or 28800 seconds).  Also. there is some code in the operating system which, given a reasonable polling interval (not a week!), will "skew" the time so the your computer RTC will keep better time.

Windows Registry Editor Version 5.00

[HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Services\W32Time\TimeProviders\NtpClient]

"SpecialPollInterval"=dword:00007080
