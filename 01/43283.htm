a. Press “Windows key +W” key.

b. Type “users” without quotes in the search box.

c. Then select “Users” from the left panel.

d. Select “Switch to Microsoft account” right below your user account name.

e. Enter the required details (Primary Email ID).

f. Restart the computer and then try to login using new Primary account.

kube say:

Ah, I'd done that - didn't work. The error message is actually a bit of a red herring - turns out it can pop up for all sorts of reasons.

 

Turned out that my gut was right about it being a registry issue... the MS Visual C++ 2005 Redistributable entries had broken somehow, preventing the windows installer from installing any more instances of the same thing. Of course, most installers start with a MSVC Redist install, so that's why they were all failing. Thankfully I didn't have to edit the registry manually.

I found a program on the Microsoft Download Centre to help out. Can't remember what it was actually called but it's primarily to fix broken MS Office installations (when they go half way so there are entries in the registry but the install hasn't actually completed... so windows won't let you install or uninstall). It just uninstalls things properly and takes their keys out of the registry. I zapped the MSVC 2005 entry that was there and then all my installers started working again!

 

Fantastic.

 

So yeah, sorry, that's probably all really confusing... was actually broken registry in the end. Fixed by googling for a couple days. :)
