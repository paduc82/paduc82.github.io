It appears that the author of the page I mentioned got things back to front. His title says that he can disable long file names. What he actually means is that he can enable or disable short file names. The technique is well known.

I suspect there is no way to disable long file names in Windows Explorer. For testing purposes you could use the batch file below. When executed inside some folder it will copy all existing files to the subfolder "83Folder", using their 8.3 names. Note that the script is primitive. It could trip over certain "poison" characters in the file names.

The script expects a colon to be used in the files' time stamps (e.g. 07:15).

@echo off
if exist 83Folder rd /s /q 83Folder
if not exist 83Folder md 83Folder
dir /x | find ":" | find /v "<" | find /v /i "Directory of" > "%temp%\Dir.txt"
for /F "delims=" %%a in ('type "%temp%\Dir.txt"') do call :Sub %%a
goto :eof

:Sub
copy %4 83Folder\%4
