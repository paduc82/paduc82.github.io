Silly question perhaps, but did you install the chipset drivers for the motherboard?

Also, there's something I discovered with graphic cards installation less than an hour ago. I've been trying to fix a problem which keeps cropping up in Event Viewer concerning an Nvidia file called nvinitx.dll on my brand new laptop. But after uninstalling the previous drivers and rebooting, it would appear that the "Found New Hardware Wizard" runs silently in the backgound and installs whatever drivers it finds. When I say "silently", I mean it's invisible to you and therefore you can't stop it. To prevent that happening just in case it's relevant, what you have to do is to go to Control Panel, then Windows Update and set it to "Never check for updates".

But before performing both those steps, go to Program and Features  and check to see if the Wizard has installed anything you weren't expecting.

xircal say:

Just to add an alternative method of preventing the "Found New Hardware Wizard" from running in the background on Startup.

  1. Hit Windows key + Pause/Break to take you to "System".
  2. In the left hand menu, click the link to "Advanced System Settings".
  3. Next, click the "Device installation settings" button.
  4. In the next menu, checkmark; "No, let me choose what to do" and "Never install driver software from Windows Update".
  5. Remove the checkmark from: Automatically get the device app...etc.
  6. Click Save Changes to exit.

In my experience, installing drivers via Windows Update is best avoided anyway because it doesn't uninstall the current driver version first. This can lead to system instability.
