shenan stanley say:
Again - mirroring (the way you are presenting) is *not* "backing up".  It has similarities - but it has flaws.
 
- If something fries the two hard drives (possible - more so actually than just one of the drives dying a natural death) - you've lost everything if this is the only way you were protecting your data.
- If something happens to the system as a whole (stolen, fire, etc) - *poof* goes the data.
- If you are infested/infected with malware/virus - it affects *both* drives at the same time.
- If something starts erasing/moving/replicating to one drive - it does it to both.
 
You can mirror if you like; it will decrease your performance in some ways.  It will not *protect* the data except in some very specific circumstances.  I would still *highly* suggest some point-in-time backup solution on *external* media even if you decide to continue with the mirror concept.  If you really want to mirror - do hardware mirroring - fastest/least resource intensive/best results.
