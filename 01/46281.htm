More and more users use laptops and mobile devices and connect to the internet via WiFi. As a consequence, a modern computer rarely connects to one single network. Instead, "alternate configurations" and other semi-manual workarounds are devised, to address a problem that is caused because of plain counter-intuitive engineering.

jason of oz say:

That's just the way networking works? IP addresses have always been bound an interface or an adaptor, otherwise routing would fall apart because the machine wouldn't know which interface to send traffic out of for a particular hop (or routing tables would need to be amended to include interface details, not just next hop addresses and metrics).

They have devised a way to automatically change IP addressed based on network requirements, they called it DHCP.

I can't think of too many reasons why a portable device, jumping between networks would need a static IP on each of the networks it joins but if you've got such a reason, why not just setup a DHCP reservation so a particular client gets the same IP each time it connects?

If DHCP just isn't an option, and you're sick of manually updating your static addresses, just prepare yourself a couple of scripts to do it for you. Here's a link to a PowerShell resource on the topic http://www.howtogeek.com/112660/how-to-change-your-ip-address-using-powershell/ that page also has a link another page telling you how to achieve the same thing using Command Prompt.
