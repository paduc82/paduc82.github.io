francoisecoscarella say:

how to use the 2nd hard drive

my (big) laptop has 2 500 Gig hd - sata 5400. So far, only the first one is used. What can I do to enable me to switch from one to the other? I'd like to use the 2d one either as a download center, or as a back up center. Anyway, it is stupid to waste 500 Gig!

ken blake - microsoft mvp say:
my (big) laptop has 2 500 Gig hd - sata 5400. So far, only the first one is used. What can I do to enable me to switch from one to the other? I'd like to use the 2d one either as a download center, or as a back up center. Anyway, it is stupid to waste 500 Gig!
Thanks for the instructions.


1. Whever you save a file, regardless of what kind of file it is, and regardless of what program you are using to save it, you are given a choice of where to put it. Usually you are also given a default (almost certainly on C:), and if you don't change it, that's where the file goes. So when you get a dialog box that tells you where the file will be saved, use that dialog box  (at least some of the time) to change to your second drive.

 

2. Almost certainly you do not have two drives, but rather two *partitions* on a single drive. Putting downloads on a second partition is fine, but putting backups there is not. It's better than no backup at all, but just barely. If the drive fails (or rather, *when* the drive fails, you will simultaneously lose the original and the backup. Even if it's a second physical drive,

I don't recommend backup to a second non-removable hard drive because it leaves you susceptible to simultaneous loss of the original and backup to many of the most common dangers: severe power glitches, nearby lightning strikes, virus attacks, even theft of the computer.

 

In my view, secure backup needs to be on removable media, and not kept in the computer. For really secure backup (needed, for example, if the life of your business depends on your data) you should have multiple generations of backup, and at least one of those generations should be stored off-site.

 

You might like to read this article on backup that I've wrotten: "How to Back Up Your Computer Regularly and Reliably" at http://www.computorcompanion.com/LPMArticle.asp?ID=314

Hi,

Sometimes the Automatic Repair fails, but the Startup Repair that was mentioned will work... Sometimes it will also fail...

There is a 3rd option... using the Command Prompt with some boot repair commands...

There is a trick to this involving your type of Bios in use... Especially when booting from installation media and doing a Repair..

Usually no problem with the older Legacy Bios, but sometimes...

The main problem is when using UEFI Bios... The repair can fail because the machine is not booted with the installation media  specifically for a UEFI Bios machine... This would apply to the Startup Repair as well...

UEFI Boot Repair...

1. Put the Windows installation disc\flash drive, and then start the computer.

2. Press F9, F11, F12, ESC, or whatever your machine uses to access Boot Options at the Manufcturer's Logo Screen..

Select the CD\DVD Drive or Flash Drive that starts with "UEFI"... Press any key when "Press any key to boot from CD or DVD ". appears. You may see this for a Flash Drive as well...

If using Legacy Bios, ignore the drives starting with "UEFI"...

3. Select a language, a time, a currency, and a keyboard or another input method, and then click Next.

4. Click "Repair your computer" on the lower left of the "Install" screen....

5. Click the "Troubleshoot" or "Advanced" depending upon which appears...

6. Select "Command Prompt". (Capital letters not necessary)

7. Type BOOTREC /FIXMBR, and then press ENTER.

8. Type BOOTREC /FIXBOOT, and then press ENTER.

9. Type BOOTREC /REBUILDBCD, and then press ENTER.
