    Thanks very much!

    The attached DMP file is of the DRIVER_POWER_STATE_FAILURE (9f) bug check.

    This bug check indicates that the driver is in an inconsistent or invalid power state.

    By default the driver being blamed for blocking an IRP for too long is pci.sys, however it is not the true cause. If we run an !irp on the 4th parameter of the bug check:

    >[ 16, 2]   0 e1 fffffa8008fd6050 00000000 fffff8000352a210-fffffa800aff3580 Success Error Cancel pending
              *** ERROR: Module load completed but symbols could not be loaded for e1e6232e.sys
     \Driver\e1express    nt!PopSystemIrpCompletion

    The true cause is e1e6232e.sys which is the Intel(R) PRO/1000 PCI Express Network Connection driver.

    ------------------------------------

    The reason this driver is the fault behind the crash is due to the fact that you are using a Conexant Modem. The device drivers for this modem are nearly 8 years old, therefore it's by no means compatible with the OS, and it cannot transition from sleep > wake states properly.

    The only (unfortunate) workaround/solution is to replace this with an up to date method of connecting to the internet.
