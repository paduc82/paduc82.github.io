When you upgraded from a previous version of Windows or receive a new computer preinstalled with Windows 10, what happened is the hardware (your PC) will get a digital entitlement, where a unique signature of the computer will be stored on Microsoft Activation Servers. The Windows 7 or Windows 8 genuine license you were previously running will be exchanged for a diagnostics key.

Anytime you need to reinstall Windows 10 on that machine, just proceed to reinstall Windows 10. It will automatically reactivate. You will prompted to enter a product key a couple times through the installation, click I don't have a key and Do this later.

There is no need to know or get a new key, but if you have issues with Digital Licensing or the Activation Troubleshooter, you can utilize an existing Windows 7 or Windows 8/8.1 product key you already own to reactivate if necessary. or use the reset function in Windows 10.

    Our users do not have rights to install programs, or edit registry items, we do apply DoD STIGs (Security Technical Implementation Guide)... again, ActiveSync installs fine and runs when using an account with "Domain Admin" permissions.  When the user tries to launch activesync it fails.... however if I put the user in the domain admin group, run it once so that it installs, then remove the user from the domain admin group...     If anyone knows how to fix this PLEASE PLEASE PLEASE help me out!!

    hypnoticautopsy say:

    Answer for all whom get this same error no matter the software:

    Microsoft was absolutely "NO F'ing" help with this one.  Here is the answer to this issue;  If you are in an Enterprise environment, meaning permissions are controlled and or restricted such as being in a DoD, WHS, DHS, ect... The user will need permissions to two registry keys.

    1) [HKEY_CURRENT_USER\Software\Microsoft\Windows CE Services]

    2)[HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\Windows CE Services]

    Follow these steps, as this is what fixed the issue for me, believe me I had been working this issue for 3 months, after it was passed on to me from one of our IA guys that worked it for 3 himself.

    1) Give the user "Domain Admin" rights (be sure the user is not there, and make sure you run it past your IA folks first)

    2) Log in as the user, install activesync * this is a MUST!!, otherwise the registry key for HKCU will not get created, this is why you get the error message stating the name is not a valid short file name*

    3) Open regedit, then navigate to the two registry keys mentioned above; and give the "user" or "domain users" full control of those registry keys. *If you do not, then the very next time the user logs in, ActiveSync will attempt to re-install, you will get the same error message and be back at square 1*

    4) Log off as the user, remove the user from the "Domain Admin" group.

    5) Test, log in as the user again, run activesync, you will not get that error message again.

    The situation:  I have reinstalled Windows twice onto SSD's for a laptop and a PC. I have had the same issue on both machines until I figured it out since Microsoft won't explain it.

    What I think Windows is doing:  Updater is trying to download ALL updates regardless of how they must be installed.  Many are dependent on a successful install of a previous update, i.e. a reboot after an update.  For example, in one case Windows Updater tried to download and install five consecutive updates, but NOTHING indicated that my laptop had to be rebooted after each update in order for the next one to be installed.

    What I did:  After each round up updates, I manually rebooted my laptop, and sure enough, on the shutdown and start screens, it provided a percentage status of update installations.  I then ran UPDATE again, it failed, I rebooted and ran the update screens, the failed ones redownloaded, install failure, reboot, updated while shutting down and restarting, ran UPDATE again, and so on.

    This can be time-consuming but it is the ONLY way I have ever found to get all updates eventually installed. No running the joke of a troubleshooter, no resetting updates, no deletion of update files, nodoing any of the other myriad of other solutions that have been suggested.
