marilyn o say:

Hello TimAZ987,

You can share files but you can't move installed programs. That is the whole reason for Windows XP mode to install programs that don't work in Windows 7.

One of the issues is that shared drives do not get drive letters assigned to them inside the virtual machine with Windows Virtual PC. It is more difficult to access shares from command line but easy to use from GUI. 

1. Go to the Settings for your XP Mode Virtual Machine.
2. Change the network adapter binding from NAT to the physical network adapter of your Windows 7 machine.
3. Create a share for the My Documents folder on the XP Virtual Machine.
4. On the Windows 7 machine, map a drive to the share on the XP Virtual Machine.
 

Map a shared drive to a drive letter inside the virtual machine by simply running a command like this:

  net use j: \\test\c

Where “c” is the drive letter that you are trying to access on the physical computer and “j” is the drive letter that you want to use inside the virtual machine.  Once you have done this you can change to the drive letter you just used and run command line operations like you would for any other drive.

If you have specific questions on Windows XP mode and Virtual PC, you can post in the Microsoft Virtualization Forum. Click here for a link.

 

Hope this helps.

bossdweeb say:

Which games are you looking at ?


- If a game specifies 2GB RAM, it is probably happy to find that in the 8GB of system RAM on your machine, regardless of the 1GB

allocated to your graphics processor [EDIT: I just verified this - results at bottom of post ].

- Your machines graphics processor is allotted 1GB of the 8GB system RAM to process graphics. This amount is set by the BIOS.

This is only a minimum that is set aside exclusively for graphics. If a game needs more it can still access the rest of the

system RAM still available.

Apparently a lot of current laptops don't have an option to boost the quantity of 'dedicated' graphics RAM, but if the Surface Go does

your user manual should have instruction how.

- If the games minimum requirements specify 2GB VRAM (Video RAM) it is not likely to run even if you do have 2GB RAM allocated to graphics.

VRAM is found only on dedicated graphics 'cards', so in this instance your machine would not likely meet the games requirements

no matter how much system RAM were allocated to graphics.


- The Intel HD 615 integrated graphics and dual core cpu on you machine are not meant for system intensive gaming graphics,

and I doubt you would get good performance with most games that actually use 2GB RAM.

Any game that mentions Intel HD (or AMD APU) graphics in the requirements will run on your machine. eg - Asphalt9: Legends .

However, some games, even a few old games, do not run on integrated graphics at all .

Beyond that there is a lot of guess work which games run. Older games would be safe, and some recent games have a big gap

between low settings and max, while other have  relatively high low end graphics settings so you often need to ask around when

running a low end system to know if it will run games you may be interested in.

If you haven't seen game benchmarks for you Intel HD615 yet, check the review below.

https://www.notebookcheck.net/Intel-HD-Graphics-615.174029.0.html

[EDIT:  Looking at the benchmarks, I see Fortnite is playable in spite of the minimum spec calling for 2GB RAM, so this partly

- Back again. My machines motherboard only allows a max of 512MB to be 'dedicated' to the Intel HD4600 Integrated graphics,

- Skyrim (min 1GB RAM) with all low settings using 1024x768 resolution @60fps.

- Batman Arkham City, (min 2GB RAM), but not as smooth as Skyrim. ]
