markfilipak.windows say:

[SOLVED] How do I install .NET Framework 3.5 from dotnetfx35.exe?

=====

In a nutshell, the answer is: You can't install .NET Framework 3.5 from dotnetfx35.exe. However, you can install .NET Framework 3.5 by another method. I've added the method as a preamble to this thread so you don't have to read the thread.

It turned out to be pretty simple. The procedure is outlined in this article:

https://docs.microsoft.com/en-us/windows-hardware/manufacture/desktop/deploy-net-framework-35-by-using-deployment-image-servicing-and-management--dism

but I've added the steps below so you don't have to read the article.

Here's what I did.

1 - I mounted Win10_1709_English_x64.iso on drive G:. ...In my system, drive G: is a virtual DVD drive. You might need to actually burn a DVD and mount it on a real DVD drive, but believe me, that's a small price to pay for what this procedure does. ...By the way, you can download Win10_1709_English_x64.iso from Microsoft for free.

2 - I opened a command line (aka DOS box) as administrator and submitted this command:

DISM /Online /Enable-Feature /FeatureName:NetFx3 /All /LimitAccess /Source:G:\sources\sxs

...Substitute "G:" with the drive letter of your actual DVD drive (or your actual virtual DVD drive).

That's it! DISM installed .NET Framework 3.5 in less than a minute.
