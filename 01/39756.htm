rajesh govind say:

 

Hi Rohan,

 

It is absolutely not possible to manually create an UEFI System Partition with Windows 8 boot files. If you are trying to convert an MBR disk to a GPT disk, then you will lose all the data in that disk, as this process involves deleting all the partitions or volumes in the disk. So, you may either need to take a backup of all the data in your computer or use the Easy Transfer Utility that you have in Windows 7.

 

Additionally, I would also like to inform you that, though converting MBR disk to GPT disk is considerably necessary to make full use of the disk space, but, please note that the program prevents you from converting system disk, since GPT system disk can boot from UEFI motherboard, but MBR system disk cannot boot from this configuration.

burt harris (msft retired) say:

Warning:  If you are working on anything but a (virtual) test-lab machine, it's probably a good idea to create a system restore point before experimenting with this.   From the command line or start menu, run `systempropertiesprotection` so you can recover if there's a problem.

On modern builds of Windows (e.g. Windows 10), the alternative to renaming I'd suggest is creating NTFS junction points.   This has a better chance of maintaining a stable and maintainable OS install...   You can setup your system to do this with the following commands from an admin-enabled command line:

C:\>cd c:\

Junction created for ProgramFiles <<===>> c:\Program Files

Junction created for ProgramFilesX86 <<===>> c:\Program Files (x86)

...

After this, "Program Files" has an alternative name: "ProgramFiles"!   Of course if you are on a 32-bit platform, you won't need "ProgramFilesX86", but running 64-bit.    You still will want to patch-up the environment variables like PATH, but with this approach it's not as critical.

I've tried with less success setting environment "ProgramFiles" and "ProgramFiles(x86)" too, without as much success.   These I think are special, set based on the "Known Folder" locations the OS maintains.  

Finally, there might be a way of doing a full rename with shell API, but I think it will take something more than a command script.   See https://stackoverflow.com/questions/25709398/set-location-of-special-folders-with-powershell for a hint how that MIGHT work.   I'm afraid it will take a lot of work to make this approach work.
