If the input method keeps on getting installed, you can use an elevated Windows PowerShell command to remove it on your system.

  1. Right-click the Start button and choose Windows PowerShell (Admin).
  2. If prompted, select Yes or enter the administrator password.
  3. Type or copy/paste the command below and then press Enter.
    • Get-WinUserLanguageList
  4. Take note of the LanguageTag of the input method you want to remove.
  5. Type or copy/paste the command below and then hit Enter.
    • $LangList = Get-WinUserLanguageList
    • $MarkedLang = $LangList | where LanguageTag -eq "<languagecode>"
      • Replace <languagecode> with the actual LanguageTag of the input method you want to remove.
    • $LangList.Remove($MarkedLang)
    • Set-WinUserLanguageList $LangList -Force
  6. Close Windows PowerShell and then restart your PC.

gerry c j cornell say:

Erwin

When you have your C partition on a conventional fixed hard drive ( drive other than a solid state drive) and you use a Windows managed page file, it creates a situation where fragmentation occurs more quickly. The pagefile gets scattered all over the disk. If file fragments between scattered parts of the pagefile are removed the next file to be written, if it is of any size, is immediately in a fragmented state. This is because the available free disk space is fragmented. Using a third party defragmenter like Defraggler you can defragment free disk space.
