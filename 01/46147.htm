Is it possible to log-off and put the computer to sleep at the same - Microsoft Community

kekoa1969 say:

Is it possible to log-off and put the computer to sleep at the same time?

Original title: log off AND sleep

Is there a way to make windows 7 log off & go to sleep in one step? I have 3 users on our computer & when I wake it up after someone else has put it to sleep, it would be nice if it displayed the user selection screen upon waking up instead of the previous user.
ron howard_ac say:

I know this is old, but I thought I’d drop an answer for anyone who may be researching this.  It is actually possible to log-off and sleep essentially in one step.  You can create a custom batch file that does this with basically two lines.

 

Just open notepad and enter the following lines of code:

 

%windir%\System32\shutdown.exe /l /f

%windir%\System32\rundll32.exe powrprof.dll,SetSuspendState

 

Now, go to ‘save as,’ select where you want to save it, give it a name that ends with .bat and that’s it.

 

You can even create a shortcut to the batch file on your desktop or taskbar and change the icon on the shortcut to an icon that looks like a power button.  When you click on this file or its shortcut, it will immediately begin the logoff procedure and sleep the computer.  The logoff will finalize once the computer resumes, but it happens so quickly on a modern computer that you hardly notice it.  The only caveat is that each user would have to remember to click on it when they are done. 

 

I think you can probably automate this batch file using task scheduler and having it only execute the batch file if the computer is idle for ‘X’ amount of time.  You could have it ‘execute’ this task at ‘log-on’ but specify under the conditions that it only executes if the machine is idle for a certain period of time.  This would create an auto-sleep/logoff routine that happens if someone walks away from the computer and forgets to log-out.  I haven’t tested the task scheduler setup as I have not had a reason to automate this function, but I suspect it should work.

 

The only problem I see is if someone is working on something and forgets to save it.  They will probably lose their work.  The /f switch forces all programs to close.  It’s something to consider.
