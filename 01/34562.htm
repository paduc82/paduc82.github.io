    Windows XP has a built-in account named Administrator.  When you first set up a Windows computer, the "best practice" is to create one or more user accounts, which can be either "computer administrator" accounts or "limited user" accounts.  As soon as you create an account (and I don't remember if this happens with any account or if you have to create a "computer administrator" account), the built-in account named Administrator is hidden from view.  The reason for this is to keep an account with administrator privileges for emergency use -- for example, if you forget the password to your "daily use" account(s) or your "daily use" account(s) becomes corrupted.

    Because you can't remember or don't know if the account you are trying to use is the built-in account, try the following:

    • Restart the computer and repeatedly tap the F8 key, starting immediately after the Power On Self Test (POST) is completed.  If you see the Windows logo, you waited too long to start tapping and you'll have to restart and try again.
    • When you get to the Advanced Boot Options menu (white text on a black screen), use the up/down arrows to select "Safe Mode" and press Enter (the mouse will not work on this screen)

    If the built-in account is still named Administrator, it will show on the Welcome screen when the computer restarts into Safe Mode.  The default password for the built-in account is blank (no password). 

    If you don't see "Administrator" on the Safe Mode Welcome screen, you must have changed its name and have been using it as a daily user account.  If you do see "Administrator" on the Safe Mode Welcome screen but a blank password doesn't work, you must have change the password.  In either case, that's as far as we can go here.  Google is your friend.
