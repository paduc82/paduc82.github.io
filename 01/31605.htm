Windows live mail keeps changing my pdf attachments to ._pdf file

Windows live mail keeps changing my pdf attachments to ._pdf file which makes it unreadable for the person i send it too

¡firedog say:

Are you sure that WLMail is changing the file name?

As far as I can tell, WLMail will only change the filename in this way if there are two periods before the file extension. This can happen if you allow the application that is producing the PDF to create the file name. For example, a Word document entitled Annual Report. could well become a file named Annual Report..pdf or Annual_Report..pdf when saved as PDF.

There is a default option in Windows (XP and later) that Hides extensions for known file types. With this option selected, it's not easy to detect faults like the one you're seeing. Those example PDFs would appear in Windows Explorer simply as Annual Report. and Annual_Report.

While it is legal in some circumstances to use a period ('.') in a file name, many systems will choke on such a file name because the period is used to separate the file name part from the extension. If there is more than one period, a program may not be able to work out where the file name stops and the extension begins. That is why WLMail changes one of them to an underscore character.

To avoid the problem, ensure that the Hide extensions... option is not selected at Control Panel > Folder Options > View, and don't use periods in file names.

For the recipient to be able to use the file you sent, he only has to remove the underscore character from the file extension. In many mail programs - including WLMail - you can right-click on an attachment and select Save as. This gives you the opportunity to change the file name.
