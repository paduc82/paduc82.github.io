Windows 10 Media Creation Tool - Bugs & Invalid Hash

I just wanted to let the Microsoft Team and the community know about this:

The way to fix most of the bugs in the Windows 10 upgrade is with a clean install or an operating system refresh after the upgrade. Most users are not aware of this procedure, but it allows you to reinstall the operating system's core files without data loss. The only way to do it is with a burned ISO file.

Whenever I use the Media Creation tool to download the ISO, it doesn't download completely. I compare it with the file hash taken from entries on the MSDN Subscriber Downloads website: https://msdn.microsoft.com/en-us/subscriptions/downloads/hh442898.aspx

The hash of the file downloaded by the Media Creation tool doesn't match the actual hash. Here are the right ISO file names on the MSDN to compare hashes with:

Windows 10 Media Creation Tool 1607 for both Home & Pro (regular download):

  • Windows 10 (Multiple Editions), Version 1607 (Updated Jul 2016) (x86)
  • Windows 10 (Multiple Editions), Version 1607 (Updated Jul 2016) (x64)

Just search for these iso files on the MSDN Subscriber Downloads site and copy the corresponding SHA1 hash into a program like HashTab to compare.

To legitimately direct download the official ISO file from Microsoft, follow this tutorial:

http://techdows.com/2015/08/use-this-trick-to-get-windows-10-iso-direct-download-links.html

This workaround may not be available if Microsoft decides to change their download page. However, the file hash now matches.

Without a formal way to compare file hashes, many users can receive partial ISO files. With those or with a non-refreshed Windows upgrade, users can be subjected to install errors (in which they blame Windows 10 for, thus hurting its reputation unnecessarily). Microsoft's indirect way of displaying file hashes for their ISO files should be improved. Anyone agree?

nielo tm say:

Enabling fixed V-Sync is not always ideal. For an example if you play a game synced to 60Hz and the frame rate drops below 60fps, the game is automatically  re-synced to 30fps. If it drops again (below 30fps), it is re-synced to 15fps and if the same happens again, frames are dropped (meaning it won't be send to the screen). So to provide somewhat stable rate of motion , developers disable V-Sync, which means the majority of frames rendered by the console are send to the screen. But that leads to two different types of problems. 1. Unstable motion and 2, tearing. This is because it's not possible to fit number other than 30 and 15 into 60. So some developers program the game to drop frames while maintaining somewhat stable motion (or repeat the frame 2-3 times). End result: Clean frames but the motion isn't consistent, which is ok for certain games but not for all games. The other option is to merge two frames into one, this leads to tearing (as you're clearly witnessing atm). This provide consistent motion at the cost of visual quality. Certain developers isolate tearing near the top (HALO 3, ODST and Reach all programmed to tear at the top) but not developers have managed to achieve this. If you're playing it on the PC, you can modify the files or force the GPU to enable V-Sync, but on console, there's no such option.
