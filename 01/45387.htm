Does my hard drive need to be replaced? - Microsoft Community

a. user say:

Does my hard drive need to be replaced?

i've learned not to trust System Mechanic to try to repair errors it says it has detected on my hard drive. i've just done used CHKDSK instruction instead. When i last ran CHKDSK, my laptop tried to run the check again on the next reboot. i've read in certain forums that indicates my hard drive should be replaced. Please advise what information is proper to decide whether a hard drive needs to be replaced, thanks.

wunders say: When your system boots, the default behavior is for it to check the "Dirty" bit on your hard drive.  If the disk shows "dirty", then it will attempt to run chkdsk as part of the boot.  Under normal conditions, the dirty bit will only be set if your system last powered down non-gracefully such as a power failure or tripping over the cord or sudden shutdown due to overheating or power supply fault.  Once the complete chkdsk operation completes successfully, it should clear the dirty bit and it should not be run again at startup unless the dirty bit is set again.

I would not use this all by itself as an indication that the disk is going bad.  If you suspect that your disk is going bad, identify the manufacturer of the disk (the model number usually shows in the Device Manager --    Start -> Run -> "devmgmt.msc") and go to that manufacturer's website.  They have diagnostic programs that will test for and indicate the health of your hard drive.  There are failure mechanisms that fool these diagnostics (like bad magnetic retentiveness that won't show up until the next day), but these diagnostics should be your first step.  Nobody knows your disk drive like its manufacturer.

The default behavior for checking your disk at power-up can also be changed using the "CHKNTFS" command from a command prompt window (Start -> Run -> "cmd").  This program will show the current state of your "dirty" bit as well as include or exclude drives from dirty-bit checking at startup.
