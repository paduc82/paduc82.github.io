a. user say:

Hi Aaron,

There are a lot of possibilities on why you are experiencing such. As what you have stated, one of the possibilities is due to a virus or a malware (malicious software) which changed the permissions and security settings on some components of your system files. Other reasons can be with a problem with your user profile, a third-party program, or a system file corruption. Below are various methods which you can do to find out what is causing the issue and address those.

Method 1: Run the Microsoft Safety Scanner

The Microsoft Safety Scanner is a free downloadable security tool that would help you scan for and remove viruses, spyware, and other malicious software. It does not require to be installed and would work with your existing antivirus software. You can download it for free through this link. (Note: Any data files that are infected may only be cleaned by deleting the file entirely, which means there is a potential for data loss.)

Method 2: Test with Another User Profile

The easiest way to find out if the issue is caused by your User Profile is to create another one. Below are the steps on how to create a new User Profile with administrative access which you can test with:

  1. Press Windows key + I
  2. Click Accounts
  3. Click Family & other people
  4. Click Add someone else to this PC
  5. Enter a user name, password, password hint, and then select Next.
  6. Click on the account which you have created, click Change Account Type, and then select Administrator.

If you are also unable to create a new User Profile, try doing it while in Safe Mode.

Method 3: Run Windows in Safe Mode

If the issue is caused by a third-party program, you can find that out by running Windows in Safe mode. If your access works fine in Safe mode, chances are there is a program on your computer which causes the restriction. You can check the latest programs which you have installed and uninstall those (make sure you have the installer before doing so, just in case it isn't the program that's causing the issue). To run Windows 10 in Safe mode, follow the steps below:

  1. Click the Start button
  2. Click the power icon, and hold down the Shift key on your keyboard. While holding down the Shift key, click Restart. This would bring you to the Windows Recovery Environment
  3. Click Troubleshoot
  4. Click Advanced options
  5. Click Startup Settings. It would prompt you to restart your computer, click it.
  6. Look for Safe Mode among the list and press the number on your keyboard which corresponds with it.

Method 4: Check for Possible System File Corruption

When none of the above-given methods work, it would be best to look into the system files for a possible corruption. To check the integrity of your system files, we can use the System File Checker (SFC) and Deployment Image Servicing and Management  (DISM) tools. Here are the steps to run SFC and DISM:

  1. Press Windows key + X
  2. On the menu which should pop-up on the lower-left side of your screen, click Command Prompt (admin)
  3. Type the following on Command Prompt (wait for the SFC scan to finish before running DISM):

    sfc /scannow
    dism /online /restorehealth /cleanup-image

Check the result of the scans, and if it states that it was able to fix some corrupted files, simply restart your computer to apply the fix and check if the issue persists.

lemp say:

Although you certainly can put a pagefile on a volume other than the system volume, even if you do this, Windows still requires a small pagefile on the system volume to hold crash dumps. 

If you don't create any pagefile on the system volume and you haven't disabled crash dumps, Windows will create a temporary pagefile on the system volume equal to the amount of RAM you have.

See the comments from "Camille" and "D-Wa" (both near the bottom of the page) here:  http://superuser.com/questions/237813/how-can-i-move-the-page-file-to-another-physical-disk-location

gerry c j cornell say:

When I add a second internal HDD, will I then be able to create a paging file on that drive?

Yes. Create a fixed pagefile before you put anything on the partition. You can create a  partition at the beginning of the drive and dedicate it to the pagefile. That minimises fragmentation on other partitions of the drive if data is constantly being written to and removed from the pagefile.

To set a page file locate the drive in the box in the Virtual Memory window. Place the cursor on the drive where you wish to create a page file. Check the box before Custom size, System managed size or No paging file. If you have checked Custom size you should enter an Initial size and a Maximum size. Most users who adopt a custom size set the Minimum and the Maximum at the same figure. You then click Set, OK and the Apply button in the next window. You have to restart the computer to apply the new setting.

robtoe say:

This bugged me too, and I stumbled across a fix by accident.   Old topic, but thought I'd share nonetheless:

"Most Used" list is is populated at the top of the Start Menu as you use items.   However, if you remove an item from this list (right-click, More -> Don't Show in this List) it is removed forever.   It does not come back even if used frequently.

I randomly found that this list is generated on a per language (or at least input language) basis.

Note - This will remove all items in your Most Used list, but essentially resets it, so you can re-add the items you like through usage, just like when you first started using Windows 10.

Fix:

Add a new input language, set it as default, reboot.   Set original language back to default, delete temporary language.   Done.

Detailed steps:

1.  Click on Start - Settings - Time & Language - Region & Language (left margin) or, just type in "Language" in the Search box.

2.  Click "+" to Add a language, any language, but may be best to select a variant of your existing, i.e. if using English (United States), try English (United Kingdom).    

3.  Search for the base language, i.e. "English", and click on it to reveal variants.   Click on your choice.

4.  Back on the previous page, you'll see the new language underneath your default.    Click on it, and "set to default".   It should now appear at the top of the list.

5.  Restart Windows.   (This is required once.)

6.  After restart, you'll see no Most Used list.   This is the new one for the language you added.   You may also notice an input language (keyboard) on the taskbar near the clock.

7.  Head back to Region & Language (Step 1).

8.  Click on your original language (2nd or lower on the list) and "set to default".   Click on that temporary language, and remove it.

No need to restart again.   You'll notice that as you click on applications you use frequently, they'll get added to the Most Used list again.   It only took 2 launches for my browser of choice to appear there.

Enjoy - Mark as a solution if this works for you.   I'll post this in a few more places where it has been asked.
