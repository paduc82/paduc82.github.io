Hybrid shutdown feature of Windows 8 decreases the total startup time by hibernating the kernel session instead of closing it which reduces the overall Windows loading time because resuming from hibernated system session is very less work compared to a full system initialization.

a. user say:

I just had this precise problem and I think I figured it out, and it's fairly funny.

The symptom was that at start-up, Windows 7 would ask me "What program do you want to use to open this file", and it was a file named 'Hugh' - my first name.  Which is kind of creepy, because NO software on my computer ever uses my first name.

I found the file 'Hugh' in C:\Users and deleted it, but still wondered why Windows was trying to open it at start-up.

Here's the answer: A program that I had installed, in my case Amazon CloudDrive, had added itself to the list of programs to run at start-up, only with a little mistake.
(You find the list by running the program 'msconfig' and looking at the Startup tab.)

CloudDrive had inserted this as a startup command:
C:\Users\Hugh Lastname\AppData\Local\Amazon\Cloud Drive\AmazonCloudDrive.exe

When Windows executes that command, it tries to open the file "C:\Users\Hugh" - because that first space ends the filename.  So the problem will only show up when your user name has a space in it!

I found that startup command in the registry and added quotes ("...") around it, and the problem stopped.
If you aren't 100% comfortable editing the registry, I think your choices are:
1. Uninstall the problem program.
2. Uncheck the entry in the msconfig Startup list so that command isn't run at start-up.
3. Inform the program's publisher and wait for them to fix it in an update.
