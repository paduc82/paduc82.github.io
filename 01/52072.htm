nickdrouin say:

Probably a content issue.  I suspect that your MKV does not have a CUE section (the table-of-contents-like section of the MKV which allows a file-reader to know where to jump to in the file).  

You can re-write your MKV file with a few different programs that will author the CUE section for you.

With FFmpeg, the command line is:

ffmpeg -i inputfile.mkv -c copy -map 0 outputfile.mkv

Where -c copy just copies the media in the tracks without re-encoding (which takes forever), and -map 0 (zero) ensures that all tracks are copied.

There is also MKV Tool Nix which has MKV Merge GUI, which is easy to use: drag, drop, click 'start muxing'.

The Computer Browser service relies on the SMBv1 protocol to populate the Windows Explorer Network node (also known as "Network Neighborhood"). This legacy protocol is long deprecated, doesn't route, and has limited security. Because the service cannot function without SMBv1, it is removed at the same time.
