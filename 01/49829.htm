You see, when the data are downloaded from servers, files get pulled off those encrypted appx packages. If the EAppx package (GoW4 for instance) gets changed by an update, it means the package integrity is no longer valid, it logically fails to download the game. The SHA-1 does not fit the current (dunno what type of verification the MS Store uses - if any) download and the system does not know what to download. The CRC check of the package is different and it stucks. Then the process of downloading gets restarted from scratch as that is the only solution to solve integrity violation of package CRC checksum. I don't know how the system works, but since single files are pulled off the Appx packages and stored in "WindowsApps" folder, then the MS Store must somehow ask about file location in the Appx packages. And since the Appx package got changed due game update...

If those updates are on regular time base and your connection is slow, I mean really SLOW, you are pretty screwed... before you even manage to download the game, it gets updated and you are busted.
