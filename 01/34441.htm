selvynperez say:

I found a temporary fix for this problem.  It is definitely a problem with the Intel driver that is installed with Windows Update.  The problem with Windows Update is that it will always automatically download and install updates (at least for the home version.  I heard for Win10 Pro that you can disable auto-updates).  I read the following solutions and fixes for this problem, but either they didn't work, or it wasn't a good option:

Attempted Fix 1 - Disable Windows Update from Services - This isn't a good option because as soon as you turn it back on to install other updates, the faulty Intel driver will still install.

Attempted Fix 2 - Download Troubleshoot Tool from Microsoft - I went through the steps using this tool to hide the Intel update, however it continued to be downloaded and installed from Windows Update.

Attempted Fix 3 - Registry Hack - This seemed to work for earlier builds of Windows 10.  It would allow you to enable the Windows Update option from the legacy Control Panel.  However, this did not work for me.  It wouldn't allow me to make the changes needed in the registry.

So after so much trial-and-error and thousands of restarts, I figured out what is the best solution right now:

Working Fix - Everyone is trying to uninstall the Intel driver that is causing the BSOD but the problem is that if you uninstall it, Windows Update will detect that and will reinstall it.  Therefore the best option is to NOT uninstall it.  Keep it installed but just use a different working version of it.  In my case, I am using an ASUS G550JK and my working driver (Intel HD Graphics 4600) is version 10.18.10.3574 dated 4/23/2014.  This is what you do:

1. Make sure your working driver was previously installed on your laptop.  In my case to get this to happen I had to delete the driver from Device Manager and then select "Scan for Hardware Changes".  It then proceeded to install the 4/23/2014 Intel driver.  NOTE:  Make sure that you do not Roll Back the driver.

2. Let Windows Update install the updated driver.

3. Go back to Device Manager and right click on the Intel Driver.

4. Click "Update Driver Software"

5. Choose "Browse my computer for driver software"

6. Choose "Let me pick from a list...."

7.  You should see a list of compatible hardware.  In my case, I see both versions of the Intel Driver.  I then chose the working driver from 4/23/2014.

8.  Success!!  No more BSOD!!!!

9.  Windows Update will no longer be a problem because you did NOT uninstall the faulty Intel driver.  You merely aren't using it.

10.  Wait until Intel (or possibly Asus in my case) comes up with an updated driver that fixes the problem!

Hope this helps!
