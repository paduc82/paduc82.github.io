    Arguably the single most important option reset is the incredibly dangerous and remarkably stupid hiberboot.  IIRC, it's labeled fast startup or something.  While safe enough on a single OS system or a system where no other systems must ever share a partition with the Windows installation, this MUST be turned off to safely allow partitions to be shared between different operating systems (The EFI boot partition is particularly notable here, but this applies to any shared partition).  Be VERY careful after the update to disable this again before doing anything that would update that partition (or any other partition you share between Windows and your other operating systems), since Windows will always be hibernated rather than off, and may corrupt the heck out of the partition if it has changed in this state.  You've been warned.

    * Conclusion

    Basically, there are some severe caveats, and I took a few steps to make the process easier for the installer, since no information about a fix seems forthcoming.  Basically, if you can create enough free (unpartitioned) space for it in a location where that space could be used to form an additional partition (500MB would have been enough in my case, but I don't know if the required size varies by system), I think you should be able to manage the installation.  After installation, you should check any options you consider important, as they may well have been changed without your consent or knowledge, and you should definitely disable (or re-disable) hiberboot if you're using multiple operating systems and any share partitions with the Windows system.  Cheers, and good luck when you do decide to go for it.
