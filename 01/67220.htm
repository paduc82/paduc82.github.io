Note: When you want to delete a user account from your computer:

  • You must be signed in as an administrator to be able to delete a user account.

  • You will not be able to delete a user account that is currently logged in.

  • Have at least one enabled administrator account on the PC to avoid not being able to perform actions that require administrator rights.

Method 1: Steps to remove a user account in the Settings.

  1. Press Windows key, click on Settings.

  2. Click on Account, click on Family and other users.

  3. Select the user you want to delete under Other users and click on Remove.

  4. Accept the UAC (User Account Control) prompt.

  5. Select Delete account and data if you wish to delete account and the data and follow onscreen instructions.

Method 2: Steps to remove a user account using the Command Prompt.

  1. Press Windows + X keys from the keyboard, select Command Prompt (Admin).

  2. Type net user and hit Enter to list the user accounts on your device.

  3. Type net user "User account " /delete and hit Enter to delete the user account.

FINAL UPDATE, SOLVED (although partially)

I managed to update, but I had to to the following:

  • I decided that I will start from scratch with a fresh install of 8.1 and then upgrade to 10
  • backed up all my personal files with the File History tool in Windows,
  • backed up the contents of C:\$WINDOWS.~WS to a different drive,
  • ran Belarc Advisor (it is a system auditing tool - but it returns all the serial numbers for installed software on the pc together with a list of currently installed programs so it's good for future reference)
  • I did the refresh/reset procedure from under Windows  (I chose the remove everything and do a clean install). To my surprise the tool didn't work! When the PC rebooted I got the blue screen with the same error I was getting from the Windows 10 installer. That convinced me that there was something very wrong under the hood of my current installation. The old version of 8.1 wasn't changed and still worked. Luckily for me I had a USB drive with the Windows 8/8.1 installer (I don't know which version it was now).
  • So anyway - I booted from the USB, installed a fresh copy of 8.1, let it do all the updates, then did the reserve your copy of Windows 10 (which is apparently needed to legitimize the future version of WIndows 10)
  • After doing that I copied the contents of my previously backed up $WINDOWS.~WS folder back to C:\ and ran setup from inside the Windows folder in there (as Admin)
  • The installation went smoothly and 10 is now installed... So there must have been a software mess on the old installation or something wrong in the MBR, I don't know.

Some things to note - It's good to have the original serial number from 8.1 handy as thats needed for the initial windows 8.1 activation and then via add additional functions one can add the Media Center Edition serial number to get the DVD player software in Windows 10.

So I have a fresh copy of Windows 10 running now, and all the installation and getting stuff back from File History ahead of me.

I hope that my few days of frustration can help somebody out maybe ;)

You can configure the Windows Media Player so that it automatically downloads and installs updates and codec enhancements from the Internet when available. This makes more sense in most cases than manually in the Internet codec packs to search for. Some of the circulating on the Internet, free codec packs are not mature or with malicious software. In the worst cases, these programs break the functionality of Windows Media Player.

* Open Windows Media Player. Click Organize on the menu bar and select Options.

* On the Player tab, select either Once a day or Once a week under Automatic updates.

* Click OK.

* Click the Network tab. select all the protocol check boxes, except Use Ports. Click OK.

  • When connecting to a server running Windows Media Services, Windows Media Player attempts to use all the selected protocols.

* Now reboot your PC and re-launch the Windows Media Player to see if it works fine.

Do check this issue by playing those files using some other Media Player. This will confirm whether those files are corrupted or working fine.

Keep us posted if you require further assistance.
