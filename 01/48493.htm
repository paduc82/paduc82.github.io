No, the automatic configuration of mail accounts in Windows Live Mail is not the same as that in Outlook. The program can configure accounts from many common mail service providers using market-specific data contained in a configuration file, but unfortunately the data are often obsolete. The program will use standard settings (i.e. ports 110 and 25, no SSL and no authentication) for any types of account that it doesn't have data for.

¡firedog say:

OK, I see what you're trying to do. I don't think it will be easy or quick.

The configuration file is oeconfig.xml, and it seems to be refreshed from the MS servers on every program start and sign-in. The latest file is copied to the root of the store folder. So far as I am aware, you'd have to persuade your local MS company to include the data for your domain in the market-specific file, e.g.

<ISP dm="domain.co.in" itype="POP3" isvr="pop.domain.co.in" iport="995" issl="1" osvr="smtp.domain.co.in" oport="587" ossl="1" auth="1" />

A few years ago, I noticed that there was an error in the data for the biggest ISP in the UK. They had confused SMTP authentication with security for passing the SMTP credentials: the config file had ossl="1" auth="0" instead of ossl="0" auth="1". It took about six months to get it corrected, and thousands of users were meanwhile unable to send messages from WLMail.

However, your mileage may vary. Contact your local MS headquarters and find out who's responsible for this particular file and explain what you want them to do. Best of luck with that!

Alternatively, if your service uses standard settings, you don't have to do anything other than tell your users the POP3 and SMTP server names and what to use as username.
