Windows Live Mail Error ID: 0x8CCC0003

The photo email feature depends on your being signed in in Windows Live Mail with a Microsoft Account. This is because the feature uploads the pictures to OneDrive and only includes small thumbnails in the actual mail message together with a link to the online pictures. This makes it much easier to share pictures without clogging up mailboxes with large messages. You have to be signed in for the program to know which OneDrive account to upload the pictures to.

This error often arises if you use the Send to > Mail recipient or similar function to start a new message with a picture while you're not signed in.

wunders say:

Safe Mode is Windows starting up with a bare minimal of startup programs and many programs and processes that are started with a Normal startup simply aren't running.  Since you can start in Safe Mode, the implication is that one of the programs that normally starts but doesn't start in Safe Mode is causing your problems.  One way you find out which program this is is to use Microsoft's Configuration Editor (msconfig) to turn off startup programs until you find the one(s) that are causing your problem.

 

Begin by starting in Safe Mode, then bring up the configuration editor:

   Start -> Run -> "msconfig"

When it starts, begin by disabling all "Startup" items, then reboot your computer and see if you still have the problem.  If you don't, then go back to configuration editor and turn some of them back on again, reboot, and continue the process until you narrow down on the problem.  It's a long, iterative process...  (If "Startup Items" doesn't fix it, next try "Services").

 

"How to Troubleshoot Configuration errors by using the System Configuration Utility in Windows XP"

   < http://support.microsoft.com/kb/310560/en-us >

Since you can boot in Safe Mode, go ahead and configure your system to create a crash dump file so that the next time XP crashes (if it gets far enough along) there will be a crash dump file with some clues.

Here's how:

XP is set up to automatically reboot on some system failures, so you need to disable that feature so the Blue Screen of Death (BSOD) information will stay on the screen for you to see it.  You should also configure XP to create a small memory dump file for each BSOD so you will have a crash dump file to look at later if more debugging is needed.

If your system crashes later, the information you need will still be on the screen and then you can use the information in the memory dump files to figure out what the problem is and fix it.

Disable Automatic restart on system error

XP automatically restarts on a system failure, so configure your system so it doesn't do that, then you can see the error message.

Configure your system to not automatically restart on system failure and to create a crash dump file when it does crash.

Right click My Computer, Properties, Advanced, Startup and Recovery Settings.

In the System failure section:

Put a check mark in the "Write an event to the system log" box
Put a check mark in the "Send an administrative alert" box
Uncheck the "Automatically restart" box

In the Write debugging information section, choose:

Small memory dump (64 KB)

Set the Small dump directory to:

%SystemRoot%\Minidump  (if there is no Minidump folder, it will be created when needed)

Click OK, OK to save the settings.

Restart your computer and then wait for the next restart/crash.

After the next BSOD, boot back into Sfae Mode and look here for a crash dump file with the current date/time and ends in .dmp:

c:\windows\minidump
