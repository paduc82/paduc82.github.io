Windows is still setting up the class configuration for this device (code 56)" (tech)

1. Uninstall VPN client and restart immediately. See if it restores Network Adapters.

2. Do you have access to a USB WIFI Adapter? Plug it in and see if the Adapter appears and is Active

3. Do you have third-party network adapters installed? Uninstall them and Reboot. Install an updated Realtek Network Adapter when system reloads.

Disk Management could not start virtual diskservice (vds) on computer

1. Check that the service "Virtual Disk" is installed and started. If it is not set to start automatically then set its start up type to automatic (or "Automatic (Delayed Start)") and then start it. (You can do this using Computer Management, which should be launched from Server Manager.)

2. This requires the firewall on both the local and remote computers to be configured. What you need to do on both the local computer and the server is to configure Windows Firewall to enable "Remote Volume Management". You can do this by running the following on both:

netsh advfirewall firewall set rule group="Remote Volume Management" new enable=yes

be sure to configure the firewall on the client as well as the server.
You will need to restart "Computer Management" before trying to connect to Disk Management, otherwise it will remember that it was unsuccessful last time and not try to reconnect.
For versions of Windows other than Hyper-V-Server (or Server-Core) another work around is to use Remote Desktop Connection (RDC) to connect to the server and then run Disk Management on the server directly.

j w stuart say:

I personally only manually download the critical/important security updates, have been doing this since Windows XP. Optional or driver updates need to be looked at on a case by case basis. Not all optional updates are problems but some are and drivers are the most likely or number one on the list to cause problems.

Therefore for drivers I go to the PC manufacture's web site first or since I build computers also the motherboard manuafacture's support web site for drivers. If there are no drivers from the previously mentioned sources and Windows 7 does not have a built-in driver then if there is an optional update from Microsoft I'll try the Microsoft driver as a last resort and sometimes things work just fine after the driver update and sometimes they don't, so I always create an Image Backup of Windows before I install any updates, applications, Etc. as its my ticket back to sanity should something go wrong.

A "Custom" install as long as you delete the existing partition, create a new partition and format it is as clean as you can get.

In defense of Microsoft when you take into account all the PC manufactures and their various models over a period of years, Intel and AMD processors, NVIDIA and ATI Graphics, RAID controllers, Sound cards or chipssets, Network/LAN chips plus an infinate number of motherboard manufacture's for people like myself who build their own computers, if Microsoft developed all the device drivers for every last combination of computers out there they would never have time to do anything else.
