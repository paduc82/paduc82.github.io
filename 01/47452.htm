According to this NEC web page, "Drivers for Microsoft Windows versions newer than Windows XP are not available for any NEC printer model."  Even worse, however, all of the links on that page no longer work.

What happens when you try to run the installer on the driver disc you have?

Windows 7 x64 is able to run 32-bit programs.  If the installer on your disc doesn't work, it may be because it is actually a 16-bit program.

Note that 64-bit Windows does not support running 16-bit Windows-based applications. The primary reason is that handles have 32 significant bits on 64-bit Windows. Therefore, handles cannot be truncated and passed to 16-bit applications without loss of data. Attempts to launch 16-bit applications fail with the following error: ERROR_BAD_EXE_FORMAT.

Given that you have Windows 7 Pro, you can download (free) Microsoft's XP Mode, which will give you a fully-licensed version of Windows XP (32-bit) running on a virtual machine hosted by your Win 7 system.  You should then be able to install  your printer on the virtual XP machine.

It's a bit of a kludge, but it should enable you to print until your other ink comes (assuming, of course, that the ink in your old NEC printer hasn't dried out and clogged the print heads -- I suggest trying to do a test page on that old printer using its front panel controls before you do anything else).

Alternatively, if you search using your particular model number, you may find that a driver for another printer will work.  For example, this site claims that the HP LaserJet III driver for Windows 7 x64 works for an NEC SuperScript 870:  http://www.tomshardware.com/forum/56041-3-windows-driver-superscript
