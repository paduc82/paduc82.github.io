I think I finally found the culprit of the problem and a solution. (Finger crossed. But at least the last 60min seemed without that issue.)

I'm typing this here so follow-readers with the same or similar problem might benefit.

Background: When I migrated from my "old" Dell Precision M3800 laptop to my "new" Dell Precision 7530 laptop, I was so taken by the simply plug&play setup of win10 that I just assumed my docking station (or rather USB 3.0 dock) would work as well. And why not?

If i just put in the USB 3 port, all seemed to work: I could run multiple monitors, have my LAN go through the port as well as my mouse and keyboard - exactly as with my old M3800.

I WAS WRONG.

It turns out, that DELL even has a little page on the issue, which I luckily discovered today: http://dell.to/2l3vK5S

In short: I had to make sure that the following steps are followed:

- the Dell USB3.0 dock D3100 is not connected to the laptop

- all according drivers and software are uninstalled ( - I hadn't any installed - )

- all other DELL drivers are up-to-date (which they were for me), in particular:

     - system BIOS

     - USB 3 drivers

     - chipset drivers

     - video drivers

- Windows itself was full up-to-date (which it was for me)

Then (still not connected) I downloaded and installed the drivers for the  Dell USB3.0 dock D3100.

And only afterwards I connected the D3100.

And wonder over wonder (or just tech?) - everything worked fine. I have my mouse/keyboard connected via the DisplayLink dock (Dell USB3.0 dock D3100), and now nothing stutters or freezes.

Important lesson learnt:

I might have found that issue earlier, if it wouldn't have been for the misleading fact that - as long as the DisplayLink was connected - it didn't make a difference whether or not the keyboard&mouse where connected directly to the laptop (on a free USB 3) or via the dock.

In other words: Having that dock connected always produced the stutter also on other USB 3 connections.

In retrospect, I should have tested the mouse/keyboard lag with nothing else connected at all - but one is always wiser afterwards.

EDIT: Unfortunately, it's not only the drivers.

While it certainly didn't hurt to get these things in order, the issues reoccurred a little bit later. Doing a bit more experiments with the docking station, it seems that the problem is always occurring when the Ethernet cable is also connected to the link. Might it just be a bandwidth issue?

Strangely, I never noticed this under win7 on my old laptop. Maybe win10 is just way more often calling over the network than win7 did?

In any case, luckily, the Dell Precision 7530 has a separate LAN connector slot, so I don't need to plug-in via the DisplayLink. (I had to, for the M3800) If I connect the Ethernet via that plug, and the keyboard/mouse/monitors(2) via the DisplayLink, I seem to be fine and without lags.

I can live with that, although it would have been nice to run all via the DisplayLink (and then have a USB 3 switch between various machines.)
