a. user say:

Hello Canon,

Thank you for posting your query in Microsoft Community Forum.

Files and folders you encrypt using EFS can only be decrypted using the Windows login that encrypted the file. Other users on the system will be able to see the files but will not be able to open them, even if they are running as administrator. That means that you also need to be careful you do not forget your login, or you will be locked out of your own files.

Based on the above comprehension, I would like to suggest you that if the encrypted files and folders are backed up (irrespective of an external hard drive, DVD or any other portable media), they still retain the encryption (green colour of the EFS feature) after restoration.

 

Now, if you restore the files in a different user account or PC, you may not be able to access the files without having the backup of the corresponding EFS certificates. On the other hand, if you restore the files in the same user account, the files will be accessible but you will still notice that the filenames will be green in colour. When the files are opened in the same user account, the EFS certificates (that serves the decryption key for the files) are already present. So, when you have recovered your Windows you might not have the previous user account until recovered from a system image backed up earlier. Therefore, as the certificates are lost, the EFS files are expected to be corrupted or unreadable.

I hope this information serves your primary question. However, if you still have the query or notice any misinterpretation of your question, please feel free to reply with the correct explanation. We'll be more than happy to assist you again.

Thank you.
