yy should be from 0 to 255

zz should be from 0 to 255 (all zz the same number.

don cuthbert say:

Dell uses F12 to bring up a boot menu where you can select what device to boot from.  I would first enter BIOS (F2 at power-up) and select to reset all to Factory default.  That should remove any question as to any changes that you have selected there.

The laptop should now be in UEFI BIOS mode.  You should use Windows 10 64-bit, not 32-bit.  For UEFI, the USB stick should be formatted as FAT32, rather than NTFS (although my Dell/Alienware can boot with NTFS USB, even in UEFI mode).

With the USB inserted, when you tap F12, during power-up, it should offer you the option to select the USB to boot from.  If the USB format needs to be different (e.g FAT32 vs NTFS) you may not see it listed.  Also, if you have Windows 10 32-bit install media, there is a chance that you may not see it listed.  However, it is also possible to create a USB containing both 32-bit and 64-bit Windows by using the Media Creation Tool.

From your specific comments, it sounds like you are trying to select an upgrade instead of a clean install.  It is possible that you are selecting a different Edition of Windows than the laptop came with.  From the Dell website, I see that model may have come with Windows 7, 8.1 or 10.  The Edition refers to Windows Pro, Home, etc.  It is most likely that it came with Windows Home Edition, whether it was originally Windows 10, 8.1 or 7.  The Product Key for the original OS would be embedded in the BIOS/firmware. When you boot with the install media, it will detect the Product Key and corresponding Windows Edition.  If it came with Home and you upgraded to Pro at some point, you can go ahead and install Home Edition and then it is easy to upgrade it to Pro, using a generic Pro Product Key.

Sometimes your computers firmware, which deals with things like power management, which is related to sleep needs to be updated. Remember, the issue you are experiencing here is outside of Windows, where its not resuming from sleep. Windows is in a saved state on the hard disk and can't manage the hardware resources, thats left up to Firmware (your BIOS) to handle. So, its obvious this more a hardware issue than software, but updated driver software can fix it.

It could suggest you need to update your chipset drivers and BIOS.
I recommend you update the most critical ones: BIOS, Chipset, Display Driver, Network adapters.

If that does not work, I recommend you try two things
