Windows 10 File Explorer crash bug fix!!!

EUREKA!  After a week of trials and errors, I found the fix!!!

Reset Windows!

OK, so in Windows, under your Admin account, you'll want to go into Settings > Update & Security > Advanced Options > View Your Update History > Uninstall latest Preview Build (Since if you click on Uninstall Updates, it launches File Explorer which instantly crashes, you don't have much choice) > Reset This PC (Click on Get Started).

A box will pop up and ask if you want to "Keep my files" (Recommended) or "Remove Everything" (Basically reformatting your HDD and reinstalling Windows).  Your system will reboot and you can just sit back and relax.  Now, it will tell you that your programs will be uninstalled. 

The Good news:  When it's all done, an HTML file will show up on your desktop labeled "Uninstalled Apps."  This file is a list of the apps uninstalled and most (if not all) will be links to the websites to re-download it.  This is a good opportunity to ask yourself, "Did I really need this?"  Because chances are, you didn't/don't anymore.  Meanwhile your updates will all have been automatically uninstalled.  Oh, and File Explorer will launch and not crash.

Hope this helps.  Cheers!

ronnie vernon say:

Hi

The Libraries were added as a feature in Windows 7 to solve the problem of modern computing where a user may have many gigabytes of documents, pictures, videos, and Music files. The problem is that all of these files can quickly fill up a system drive (C:).

All of these folders can be moved to a separate hard drive or partition and then added to the library on the system (C:) drive for easy access from one central location. A Library is a Virtual Folder that does not use any space on the system drive. They provide a 'mirror' of the actual, physical files files and folders.

Using the Admin\My Documents as an example.

The My Documents folder can be added to the Documents Library (or any other library that you create). When you open the Documents Library and open a file there, this opens the actual  physical file from the Admin\My Documents folder.

Some things you need to be aware of:

1. If you Delete a folder or file from within a Library, this deletes the physical file.

2. You can use the Remove option in the Library to remove folders that have been added to the Library, without harming the physical folder or the files it contains.

Please see the following links for a complete description about using the Libraries.

Libraries: frequently asked questions
