rohn007 say:
Yes, they both provide backup, and should be used together. But they work in fundamentally different ways.

The Win7 backup is an image copy type backup. It takes "snapshots" of the drive and stores them in a backup program specific format as a single large file. The snapshots can be "full image copy" and "incremental".

Full image copies take a snapshot of the whole drive, or of specified folders. In case of a "disaster" you can restore the image to have a working copy of your installation as of the date of the backup. Restoring using the image copy wipes out your current drive.

Incremental copies are a supplement to the full image. Part of the backup process (full and incremental) is to turn off the "Archive" flag on files. After that, if the file is edited, the archive flag is turned on again. When you do an incremental backup, it copies only files with the archive flag turned on. The output of incremental backups is substantially smaller and therefore faster than a full image copy. So you can run them more often than a full image, ie a strategy may be full image monthly, incremental weekly (or even daily). In a disaster, you restore the most recent full image, then all of the subsequent incremental backups to provide a more complete, more up to date restore.

File History works at the file level. Every time you save a changed copy of the file, File History creates a backup copy, so you lose much less data than when you use an image copy or even encremental backup. File History allows you to easily and quickly restore individual files.

By contrast, the OneDrive "Autosave" feature now does almost real time saves so in theory you don't lose anything. There is also an online version feature that is similar to File History.

Here is a Wiki with more detailed description of the backup options

Intro
Win 8 has refined your backup and restore options. Before Win 8 you could create an Image Copy backup of the full drive (or selected directories), but it was hard to separate backups of the OS, Applications and personal Data unless you created a separate data partition. I still think it is a good option.
Out of the box, MS has been advertising the ‚ÄúRefresh‚Äù and ‚ÄúReset‚Äù options. Personally, I think they missed a valuable selling point by not mentioning RecImg.EXE in conjunction with Refresh. It removes my main concern/objection to Refresh.
Reset is equivalent to using the ‚ÄúRecovery Partition‚Äù on previous versions of Windows. The factory would create a ‚ÄúRecovery Partition‚Äù that you could use to ‚Äúreset the computer to factory fresh‚Äù. It would typically include all of the factory ‚Äúcrapware‚Äù and it would wipe out all of your data files and installed applications. A less than desirable outcome but an appropriate reaction to a bad malware infection. The ‚Äúrecovery partition‚Äù type of restore typically had to be accessed at boot time while Reset can be accessed from within Windows 8.
Refresh is roughly equivalent to an image copy backup. A Reset will restore a backup of the OS and Metro applications. It will wipe out ‚Äúlegacy‚Äù / x86 desktop type applications BUT it will not affect your personal data files, which is much better than a Reset. But there is still that issue of wiping out your installed desktop applications. I do not have documented confirmation, but I‚Äôm guessing that Metro apps are treated like personal data and those directories/folders are simply not reset back to the old condition.
RecImg.EXE is a backup tool that ships with Windows 8. It creates the backup ‚Äúimage‚Äù that is used by the Refresh tool. This image that was created at the factory is why using Refresh will wipe out installed desktop programs in the ‚ÄúC:\Program files‚Äù and ‚ÄúC:\Program Files (x86)‚Äù folders because they haven‚Äôt been backed up. However, using RecImg.EXE you can create new images that will include recently installed desktop applications.
RecImg Manager is a third party tool the provides a GUI to access the RecImg.EXE command-line tool and backup files.
File History is a new Win 8 feature. It is an extension to the ‚ÄúPrevious Versions‚Äù and ‚ÄúVolume Shadow Copy‚Äù features. File History continuously monitors files stored in Libraries, Desktop, Contacts, and Favorites folders. It runs on a predetermined schedule, and when it detects changes in any file, it makes a backup of that file to an External drive. It isn‚Äôt enabled by default.
