greginmich say:

Any actual solutions?

The Windows forum has more wrong answers for this question than you can shake a stick at – and that’s probably because everyone here seems to have assumed that this is a Windows Update issue. But it’s actually caused by a failure to update the DisableAntiSpyware and Disable Antivirus DWORDS in the Windows Defender registry key when a third-party AV app is installed.

This puts Windows Defender into a state of limbo (an indeterminate state) between being turned off in the Security Center, and turned on in its own registry key – and one of the side effects of this error state is failed definition updates. So the issue can be resolved by any action that completely disables Defender. Properly uninstalling, running the cleanup utility, and reinstalling should theoretically resolve the issue; but this has been reported to fail, especially for Norton products. Disabling Windows Defender in the Policies registry key also solves the definition update problem – but only by overriding the faulty entries in the Windows Defender key.

But when I reproduced this issue on a test machine running Avast Free, I found that the issue could be fully resolved in about 1 minute by following these simple steps:

1.)    Disable Avast completely (“permanently”) with the shields control.

2.)    Start Windows Defender.

3.)    Enable Avast with the shields control.

This apparently triggers the same mechanism that normally gets triggered with a clean install of the Avast app. Since a similar solution for Norton apps was reported in this discussion thread, I’m thinking that the above steps might also be applicable for other third-party AV apps:

http://answers.microsoft.com/en-us/windows/forum/windows_10-update/definition-update-for-windows-defender-kb2267602/d4edd314-a438-4ac2-b7a9-dc1c45a1ef62

Since the initial troubleshooting that was provided seems doesn't fix the issue, we also suggest following the steps below and check if it makes any difference:

Solution 1 – Run netsh winsock reset command

1. Press Windows Key + X and choose Command Prompt from the menu.
2. When Command Prompt starts, enter the following and press Enter to run it:

netsh winsock reset

3. Close Command Prompt and restart your computer.

Solution 2 – Remove problematic software

• Certain software can cause a random freeze on Windows 10. Users have reported that software such as Speccy, Acronis True Image, Privatefirewall, McAfee and Office Hub App can cause problems with Windows 10. If you have any of these programs installed on your computer, we advise that you remove them by following these steps:

1. Open Settings App and go to System.
2. Go to Apps & features section and delete the aforementioned apps.
3. After you’ve uninstalled these apps, restart your computer.

Solution 3 – Disable F.lux

It seems that F.lux software has some compatibility issues with certain display drivers. Due to these issues, you might experience a random freeze on Windows 10. In order to fix this problem, it’s advised that you disable F.lux software on your computer. If you want to keep using this software, you can also try updating your display drivers to the newest version.

Solution 4 – Disconnect all USB devices

• USB devices such as external hard drive, printers, etc. can sometimes cause Windows 10 to freeze randomly. If you have any USB devices connected to your computer, you should disconnect them before you turn on your computer.

Solution 5 – Download latest chipset drivers

• To update your chipset drivers, you need to visit your motherboard manufacturer’s website and download latest chipset drivers for your motherboard. 

Solution 6 – Update network drivers

• You might want to visit your network adapter manufacturer’s website to download the latest drivers.

Solution 7 – Update your BIOS

• To perform a BIOS update, you have to visit your motherboard manufacturer’s website and download the latest version of BIOS. 

Get back to us if the issue still persists.
