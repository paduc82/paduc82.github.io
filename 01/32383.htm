By default, the “Administrative” tab is enabled and is visible. However you can disable the same using group policy settings.

 

Follow these steps to enable/show the “Administrative” tab under “Region and Language” settings:

 

a.    Click “Start”.

b.    Type “gpedit.msc” in the Start search box and hit “Enter”.

c.    Click “User Configuration”.

e.    Double-click “Hide regional and language options administrative options”

f.     Click “Disabled”.

g.    Click “Apply” and “OK”.

 

This should enable the “Administrative” tab.

 

Note:The above steps need to be carried out using an account with administrative privileges.

File system errors are occasionally the result of corruption on the disk that can have at its root a real hardware problem. These errors can therefore result from any system problem that can cause disk corruption, such as resource conflicts, bad drivers, etc. Far more often, however, file system problems occur as a result of a software problem.

 

I suggest you to Run Chkdsk /f /r to detect and resolve any file system structural corruption. And check if it helps.

Chkdsk: The Chkdsk (Check Disc) scan will try to identify and repair bad sectors found on the hard drive. Any data present on these sectors might be lost in the process. The computer can also go into a no boot state. Proceed at your own risk.

Follow the below steps:

1.      Press Windows + X keys on the keyboard, click Command Prompt (Admin). 

2.      In the Command Prompt, type the following command, and then press ENTER:  
        chkdsk /f  /r

3.      You may be asked whether to schedule this volume to be checked next time when you restarts (Y/N).

4.      Press Y key on the keyboard and press Enter key.

5.      Restart the computer and check.

Hope this helps. If the issue remains unresolved, please get back to us and we would be happy to help.

pxjn say:

WINDOWS 10 CANNOT BOOT BECAUSE OF FAT_FILE_SYSTEM ERROR - SOLVED

I was able to get past the FAT_FILE_SYSTEM BSOD error on reboot, and successfully boot into Windows 10. I am writing this in case it helps someone out there. (Microsoft Support's response above showed clueless-ness... I had said in the original title and description that I couldn't reboot, yet their first response was to have me press the Windows+X key and run Chkdsk!!!)

I had a second internal hard disk (E:) in my desktop, and while trying various things, I happened to disconnect it and reboot. And it worked! Once I was able to reboot, I ran chkdsk on the main drive (C:) -- no errors. I suppose the problem was with the additional hard drive. That's believable, since I had issues with this drive in the past, but it is surprising that it was preventing Windows 10 from booting. (This additional drive was not the boot drive.)

This is similar to what someone else had reported on the internet - that they were able to reboot once they removed a plugged-in drive.

So - if FAT_FILE_SYSTEM is preventing you from booting, try removing all additional drives - hard disks, USB drives, memory cards, etc.

Hope this helps someone.
