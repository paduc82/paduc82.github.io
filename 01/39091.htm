If you have problems booting from a boot disc, you may have UEFI BIOS:

rickgonz say:

I had the same problem "couldn't update system reserved partition".  It appears that when Windows 7 was installed, Windows allocated 100MB for the system reserved partition.  It is almost full, and has no space for what Windows 10 wants to put into it.  BTW, this error also showed up when people tried to upgrade from Windows 7 to Windows 8 a couple of years ago.

Windows 10 will likely allocate 450 MB if installed clean.

There are some options that I have read about around the interwebs, but I have not done any of them:

1. Backup all your data, install Windows 10 clean from the self-booting disk. I do not know that anyone has done this successfully.

2. Buy one of the data partition management software on the market and do the following:

       prep:  backup all your data and have a system recovery disk to reinstall Windows 7 if necessary

       step 1: shrink disk D:

       step 2: move D partiton rightwards where there is spare unallocated space

       step 3: move C partition  rightwards

       step 4: there will now be unallocated space to the right of the system reserved partition. Theoretically, you can now extend the size to 450 MB.

3. Turn C: active, so that you can combine the MBR and C: into one partition, and reboot clean three times.  This procedure is unclear to me, and I need to read more.

If anyone has any other methods, please reply. If anyone has successfully done any of the above, please reply in detail.

Thanks,

rick
