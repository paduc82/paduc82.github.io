Ok, it sounds like you want to disable tapping, if it's some other feature that's ok.  I had this same problem with deployment of Windows 7, this feature is on by default and whatever I tried in unattend.xml, setupComplete.cmd, the registry adjustment wasn't taking effect.  This is better than a registry tweak.

Copy the Alps device driver install files to some folder.  I had the Dell version of this install and it let's me unzip the contents to a folder.  Look for an .INF file like Apfilter.inf somewhere.  This text file contains all the settings that are applied during the driver install.  So in my case, I was looking for 'Tapping', they all had 1's for a value.  For each Tapping value I found, changed them all to zero, save file.  Now, in your case, I would go into Device Manager and uninstall all versions of this driver, reboot and then run this install with that adjusted .INF file.  Now, by default, Tapping is disabled.  If you go to some other user's account, it will be disabled for them too.  If you want to change some other feature, it will be in that same .INF file.

For the deployment guys out there, do the same thing in your build and 'Tapping' will be disabled by default for every user without any other kind of tweak required.
