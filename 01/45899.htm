Prevent auto switching between windows

How do I prevent auto switching between windows, e.g. - if I'm working with a small window sitting on top of a large window and accidentally touch any part of the large window Windows auto-switches to the large window hiding the smaller window from view.  I would like Windows to require me to click on the window that I prefer to be the dominant and visible window.
cmdrkeene say:
Windows can't do that on its own, but many mouse makers provide that option (typically on laptops).  If you're using a laptop, you can also adjust the sensitivity so that a light brush of the pad doesn't get translated as an intentional touch and switch to another app.  To change this, open your Mouse settings in Control Panel, then use the hardware tab.

Most laptops use a brand of touchpad called Synaptics, and they also have a PalmCheck feature that detects if any part of your hand other than a fingertip touches the pad.  This is a pretty handy feature as well.

Windows Vista cannot obtain an IP address from certain routers or from certain non-Microsoft DHCP servers
To resolve this issue, disable the DHCP BROADCAST flag in Windows Vista. To do this, follow these steps:
Click Start, type regedit in the Start Search box, and then click regedit in the Programs list.
If you are prompted for an administrator password or for confirmation, type your password, or click Continue.
Locate and then click the following registry subkey:
HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Services\Tcpip\Parameters\Interfaces\{GUID}
In this registry path, click the (GUID) subkey that corresponds to the network adapter that is connected to the network.
On the Edit menu, point to New, and then click DWORD (32-bit) Value.
In the New Value #1 box, type DhcpConnEnableBcastFlagToggle, and then press ENTER.
Right-click DhcpConnEnableBcastFlagToggle, and then click Modify.
In the Value data box, type 1, and then click OK.
Close Registry Editor.
By setting this registry key to 1, Windows Vista will first try to obtain an IP address by using the BROADCAST flag in DHCP Discover packets. If that fails, it will try to obtain an IP address without using the BROADCAST flag in DHCP Discover packets.

Zogzog60,
There are several things we can try to resolve this issue.  Thank you for providing such thorough information.  From what I can tell, your wireless card is not receiving IP addressing information from the DHCP server in your wireless router.  Why it is not doing so is always a mystery.  But the fact that your wireless LAN connection has an IP address that starts with 169. tells you it is self assigned by the device because it did not receive an address from the router in the alloted time.  Also, there is no Gateway address and the DNS server information is wrong, so even when or if it received an IP address it would be able to connect to the wireless router but not to the Internet.  I also see several extraneous connections called Tunnel Adapters which are not actual physical adapters.  Let's start by disabling all other connections to try and force the wireless connection to request the information it needs from the router.  So go to Start > Control Panel > Network and Sharing  Center > Manage Network Connections.  Click on each connection that is listed EXCEPT the wireless LAN connection and then click Disable this network device.  Then see if  your wireless LAN will connect.  You will most likely have to run the Diagnose and Repair Utility from the Network and Sharing Center menu to force the wireless card to ask for IP information from the router.

If all that fails, go back into the Network and Sharing Center, Manage network connections and right click on the wireless LAN connection and select Properties.  Then click on Internet Protocol version 4 and select Properties.  Click in the box that says "Use the following IP address" and type in the information that was shown in the IPCONFIG report for your wired Local Area Connection above.  Do the same for "Use the following DNS servers".  Be sure you type the IP address, subnet mask, default gateway, and DNS addresses exactly the same as what the Local Area Connection had when it got the information from the router.

If none of these things work let us know and we will continue to assist you.
