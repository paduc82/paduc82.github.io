cmdrkeene say:

Probably an issue with DWM (even though there's no transparency effects, the desktop in Windows 8.1 is drawn using the hardware graphics card in the way that Windows Vista/7 would with Aero).  I remember a similar issue using an older version of Adobe Fireworks CS3 I think it was.  Since the mouse cursor is itself hardware accelerated normally, it's not usually an issue. But if the program is drawing it's own cursor, such as crosshairs, it falls back to software rendering and introduces flicker.

I believe the solution would be one of:

  1. Updated graphics drivers from manufacturer (instead of from Microsoft)
  2. Compatibility modes on the program
  3. A newer version of the program
  4. Mouse options > Pointer Options > Enhance Pointer Precision (try on and off)
