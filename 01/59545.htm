How OneDrive Syncs between Multiple Clients and the OneDrive Server - Microsoft Community

stanimal21 say:

How OneDrive Syncs between Multiple Clients and the OneDrive Server

Technical Level : Intermediate

Summary
Describes how OneDrive communicates and syncs changes between your PC, phone, Xbox, and tablet and describes the basics of real-time co-authoring based on the Office Web Apps.
Details

OneDrive is Microsoft's consumer cloud storage and collaboration tool and is designed to sync content between all of your devices connected your Microsoft account. Technically, cloud storage, or cloud computing is defined as:

"Cloud computing is a phrase used to describe a variety of computing concepts that involve a large number of computers connected through a real-time communication network such as the Internet.[1] In science, cloud computing is a synonym for distributed computing over a network, and means the ability to run a program or application on many connected computers at the same time." -- http://en.wikipedia.org/wiki/Cloud_computing

OneDrive is just one of these "distributed computing" environments that also communicates with your client devices, i.e. your phone, computer, laptop, and tablet. The term "the cloud" has evolved to include software as well, allowing users to interact with and securely share their content, for example, Office Web Apps or Google Docs editor.

Client-Server Relationships

OneDrive is designed around the basic client-server computing model. Basically, the client (your computer or device) initiates conversation with the server (OneDrive) and requests information. The server fulfills the request by sending requested data back to the client. The OneDrive applications request directory structures, file names, and image thumbnails from the server to provide a view of your files and folders. (graphic from Information Technology Services, University of Hong Kong http://www.itservices.hku.hk/news/ccnews125/p2p.htm)

Changes and updates are submitted to the server, always initiated by the client device, and new material is regularly downloaded. All communication is initiated and all information is requested by the client device. This basic structure helps facilitate OneDrive and create a collaborative environment for consumers and applies to every application that communicates with a server, including Google Drive, OneDrive, Microsoft Office Outlook, etc.

OneDrive Syncing Process

OneDrive syncing follows a very simple, established process when updating information from the server. When you modify or add information, the client (your device) sends an update request to the server along with the updated information. The clients also regularly check in with the server comparing file versions and updating as necessary. Your client device will either download or send new or updated material.

For example, you update a Microsoft Word file on your computer and save it to the OneDrive folder on your computer. The OneDrive program immediately scans and records the new file and any other metadata to send to the OneDrive servers. It initiates a connection with OneDrive servers and requests to upload information on the server, finally uploading the content. OneDrive adds the new file to it's database and any device that syncs with OneDrive will download that new file.

Real-Time Co-authoring

Syncing with OneDrive becomes more complex when using live editing through Office Web Apps and other browser-based live editing tools (Google Docs does this too). Essentially, the client-server model described earlier is changed slightly and the OneDrive server will "push" updates to the clients, reducing communication and internet traffic. According to the Office Web App team, when something changes, the server notifies all browsers currently logged in and editing the file there is a change and to update when they get a chance. For more detail, read this Office Web Apps article from the Office Blogs describing how Microsoft implemented real-time co-authoring:

http://blogs.office.com/2013/11/19/real-time-co-authoring-in-the-excel-web-app-why-and-how-we-did-it/

The article is directed towards the Excel Web App, however you can apply the same concept to the Word, PowerPoint, and OneNote Web Apps as well. Since updates appear almost immediately, there is increased communication between the client and server devices, but less data is transferred to the browser; it'll only download what you can see on your screen. Real-time co-authoring requires some tweaks to the client-server model with some peer-to-peer characteristics.

What activity can users expect from OneDrive?

Users should understand that your devices cannot directly communicate with each other via their own OneDrive apps; information and documents don't sync from your PC directly to your phone. All information must pass through OneDrive servers first, and then sync down to your other devices. Keep this concept in mind when troubleshooting your devices with OneDrive: if your other devices don't see content after updating files on one device, check OneDrive through your browser to see if the server has the most recent changes.
