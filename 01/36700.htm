no.compromise say:

For the time being, it won't affect anything else, if AppInit_Dlls is empty.  LoadAppInit_Dlls is a flag that tells Windows to "respect" (load the DLLs in AppInint_DLLs) the AppInit_DLLs value.  Older, pre-Vista apps won't know to set LoadAppInit_DLLs to 1, so the DLLs the would add to this location would not be loaded.  This can be viewed as a good thing, as this was a quick way to get code (good and bad) loaded into the address space of every process that loaded user32.dll (pretty much all of them), which processes AppInit_DLLs.  Newer apps are savvy to the LoadAppInit_DLLs setting so they set it to 1 and also add their components to AppInit_DLLs.  In your case it seems that some program added DLLS to AppInit_DLLs, set LoadAppInit_DLLs to 1, and then the DLLs were removed (manually? cleanup program? uninstall program?) from AppInit_DLLs, but LoadAppInit_DLLs was not restored to the default value of 0 (or not being present at all).

So, our exploration indicates that Windows bases the decision to log the event about "Custom Dynamic Link Libraries being loaded for every application" on the LoadAppInit_DLLs setting, regardless of the contents of the AppInit_DLLs value.

marilyn o say: Hello lost Bill,

Thank you for posting to the Microsoft Answers forum.

Sometimes a user profile will become corrupt mainly from programs that don't release handles when the program is closed. Over time this will cause the user profile to become corrupt and the best way to handle it is to create a new user account and move old files over.

Copy Files to the New User Profile

1. Log on as a user (Adminstrative account) other than the user whose profile you are copying files to or from.
2. In Windows Explorer, click Tools, click Folder Options, click the View tab, click Show hidden files and folders, click to clear the Hide protected operating system files check box, and then click OK.
3. Locate the C:\Documents and Settings\Old_Username folder, where C is the drive on which Windows XP is installed, and Old_Username is the name of the profile you want to copy user data from.
4. Press and hold down the CTRL key while you click each file and subfolder in this folder, except the following files:  Ntuser.dat, Ntuser.dat.log, Ntuser.ini
5. On the Edit menu, click Copy.
6.  Locate the C:\Documents and Settings\New_Username folder, where C is the drive on which Windows XP is installed, and New_Username is the name of the user profile that you created.
7. On the Edit menu, click Paste.
8. Log off the computer, and then log on as the new user.

One way to keep profiles from getting corrupt is a utility called User Profile cleanup - UPHClean.
Download from the location here. It will force handles to be released and the registry closed when the program is closed.

Hope this helps.

Camera App - Microsoft Community

howardtarabour say:

Camera App

Camera App not working in newly upgraded Windows 10?
jeff d - windows camera team say:

That error means the camera app was not able to initialize the camera driver. This is most often because there is a problem with the camera driver.

The first thing you should do is make sure you have all the latest Windows Updates installed--many Windows and Camera bugs have been fixed in the last few weeks, and this has helped many people.

To get the latest updates, type "Windows Update" into the Cortana Search box on your taskbar, then click the "Check for updates" link. Install any updates that are found.

If your camera is still not working, there might be an (easy to fix!) problem with your camera driver.

A quick way to check the camera driver is by trying to use the camera in the Skype app. If neither Windows Camera nor Skype is able to access your camera, then your camera driver most likely needs to be repaired.

The easiest solution is to reinstall the camera driver. You can do this by uninstalling and deleting the camera driver in Device Manager, then having Windows reinstall the driver. This can be a 1 minute fix.

Steps:

  • Hold down the Windows key and press R (Win+R)
  • Type devmgmt.msc, then click OK
  • Find "Imaging Devices" and expand it by clicking the '>' to the left.
  • Right click the camera that isn't working
  • Click Uninstall
  • When the confirmation window shows, check the "Delete the driver software..." box and press OK. (This step is important because it's the only way to assure that Windows completely reinstalls the camera driver.)
  • After the driver is uninstalled, select the "Scan for Hardware Changes" button in the Action menu. (There is also a "search" button on the tool bar that does the same thing.

If all goes well, Windows will automatically reinstall the driver for your camera and everything should be working.

If Windows cannot find a driver for your camera, you should go to the manufacturer's web site and download it.

If it still doesn't work, you either have a hardware problem or the camera probably wasn't working before you updated.

Good luck!
