asanchez_5915 say:

We found that these exact symptoms may be cause in Windows 10 by the Quick Access feature being too full.  It's a bug.  Shouldn't happen like that.  Try this:

Open File Explorer

Right-click Quick Access

Select "Options"

Under Privacy click the Clear button

You may have to close and open File Explorer once or twice before it stops crashing (blank screen, taskbar rebuilds, etc.). Seems like it needs to actually clear the Quick Access before it starts working again.

You may want to consider unchecking one or both of the 'Show recently used xxx in Quick Access' to prevent it from filling up again.

asanchez_5915 say:

An update theory on why this happens.  Probably better theory than the 'too full' idea.

I believe it goes like this: there’s something about certain files being “corrupted” as perceived by file explorer, which is what causes it to crash when we sort by date.  Then, because that folder was recently accessed, it gets added to quick access, so when file explorer is opened again into “quick access,” it perceives the corrupted file, and immediately crashes again.  But if we go into “This PC,” it avoids quick access entirely, so file explorer opens normally.

And going to 'this PC' as described next may allow you to get to the Quick Access and change the settings before it crashes on you:

Open File Explorer through Cortana (the search bar/circle thing) and typing “Computer,” then click on “This PC”
