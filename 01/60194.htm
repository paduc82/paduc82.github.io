bossdweeb say:

Far away, but in the same building, I assume...

You need a long 'Crossover' LAN cable if connecting directly between machines.

More commonly would be to connect through the Modem, so you could put the modem half way if

you can't find a cable that that is long enough to reach the second machine. But you would need a phone

plug near the modem.

- There are also special LAN adapters that plug into power outlets which use the  buildings electrical wiring as a LAN cable. This option saves having to route a LAN cable to another area in the building.

- Or use a wireless modem, but each machine has to have wi-fi  as well.

- You also need to setup permissions between the machines if you want to share stuff.

I don't do this often, and manage to stumble through the setup when it's necessary, but there would be

tutorials that can be found with a www search.

don cuthbert say:

I would start over. First check for any BIOS updates from the motherboard or OEM maker and install that. Set BIOS to Factory or Optimal defaults. A PATA (IDE) drive should be no problem. In a Generation 1 (Legacy BIOS) Hyper-V Virtual Machine, an IDE drive is required to be the boot device, even if you also have a SCSI interface installed, so I assume this is true on physical devices also.

Boot from Windows 10 install media on USB or DVD. Choose Custom install. First delete all partitions from the disk until it only has unpartitioned space. Then install Windows to that space and it will be partitioned according to Windows 10 defaults. On an MBR disk, that is normally a 500MB System/Recovery NTFS partition followed by one large NTFS partition for the rest of the disk. But Windows setup will do that for you.

It's possible that the drive is failing in some way and is developing bad sectors. The new install may have tried to write to these, while they were relatively dormant in the previous install, with some large data file residing there. If the system has any diagnostics you can try to run those, or run chkdsk with a surface scan to mark any bad sectors so that they will be isolated from any write attempts.
