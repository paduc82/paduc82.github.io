If Control Panel opens in Safe Mode but not in when you boot Windows normally, it could be that a service might be blocking Control Panel from launching. We suggest that you perform a clean boot in Windows. A clean boot starts Windows by using a minimal set of drivers and startup programs. This way, you may eliminate software conflicts that arise in Windows.

Notes:

  • You must log on to the computer as an administrator to be able to perform a clean boot.
  • Your computer might temporarily lose some functionality when performing a clean boot. However, if you boot the computer normally, the functionality returns.

Here's how to perform a clean boot in Windows:

  1. Press the Start button, then type msconfig, and select System Configuration.
  2. On the Services tab of the System Configuration dialog box, click the Hide all Microsoft services check box, then click Disable all.
  3. On the Startup tab of the System Configuration dialog box, click Open Task Manager.
  4. On the Startup tab in Task Manager, for each startup item, select the item, then click Disable.
  5. Close Task Manager.
  6. On the Startup tab of the System Configuration dialog box, click OK, then restart the computer.

After the computer is restarted, open Control Panel. If it opens, then the issue is caused by an application or service interference. If that's the case, then follow the steps below on how to determine what is causing the problem:

  1. Log on to the computer by using an account with administrator rights.
  2. Press the Start button, then search for msconfig, and select System Configuration.
  3. Click the Services tab, then click the Hide all Microsoft services check box.
  4. Select the upper half of the check boxes in the Services list.
  5. Click OK, then select Restart.
  6. After the computer finishes restarting, determine whether the problem still occurs.
    • If the problem still occurs, repeat steps 1 to 5, but clear the lower half of the check boxes in the Services list that you originally selected.
    • If the problem does not occur, repeat steps 1 to 5, and select only the upper half of the remaining check boxes that are cleared in the Service list. Repeat the steps until you have selected all the check boxes.
    • If you still experience the problem when only one service is selected in the Service list, this means that the selected service causes the problem, and you should go to step 10. If no service causes the problem, go to step 7.
  7. Repeat steps 1 to 3 in this section.
  8. Click the Startup tab, then select the upper half of the check boxes in the Startup Item list.
  9. Click OK, then click Restart.
    • If the problem still occurs, repeat steps 7 and 8, but clear the lower half of the checked boxes in the Startup Item list that you originally selected.
    • If the problem does not occur, repeat steps 7 and 8, and select only the upper half of the remaining check boxes that are cleared in the Startup Item list. Repeat these steps until you have selected all the check boxes.
    • If you still experience the problem after only one Startup Item is selected in the Startup Item list, this means that the selected Startup Item causes the problem, and you should go to step 11. If no Startup Item causes this problem, a Microsoft service probably causes the problem. To determine which Microsoft service may be causing the problem, repeat steps 1 through 6 without selecting the Hide all Microsoft services check box in each step.
  10. After you determine the startup item or the service that causes the problem, contact the program manufacturer to determine whether the problem can be resolved. Or, run the System Configuration utility, and then click to clear the check box for the problem item.

I got mine middle click using the program AutoHotKeys.

It lets you write little scripts to remap mouse clicks and keyboard keys.

 

I remapped the Win+RClick to be middle click. The syntax is below. The Window key is "#". So install the software, and right click on your desktop. Create a new script and paste the following into it.

 

#LButton::MButton

 

 

If you have a wedge keyboard and want Alt+F4 to work with out using the function key add the following line too. It will make Alt+F4 work as normal, and as a bonus, if you just push the F4 key it will still make the volume increase.

 

!Volume_Up::!F4

sayed ibn masud say: I had same problem on my dell n5110 i5 laptop. I fixed it by updating BIOS. If you're using default system bios, then most probably your bios version is A09. Dell has published an BIOS update(version A11) to fix windows 8 known issues. This update also solve this upgrade problem. 

So, what you can do is visit dell support website to download bios update.

Dell support: http://support.dell.com

And since you are using n5110-i5 then you can directly visit this link to download the BIOS update.

Dell n5110 bios update link: http://www.dell.com/support/drivers/us/en/19/driverdetails?driverid=CXRD8

barb bowman say:

Try a two button restart after you shut down completely and remove from the dock:

Step 1: Swipe in from the right and tap or click Settings, tap or click Power, and then tap or click Shut down to shut down Surface.
If you can’t shut down Surface normally, press and hold the power button on your Surface for 30 seconds.
Step 2:

After Surface turns off, press and hold volume-up button and the power button at the same time for at least 15 seconds, and then release both.

The screen may flash the Surface logo, but continue holding the buttons down for at least 15 seconds.
Step 3: After you release the buttons, wait 10 seconds.
Step 4: Press and release the power button to turn your Surface back on.
