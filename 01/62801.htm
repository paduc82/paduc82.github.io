jonathan gin say:

Sorry. I guess I should have explained your options.

1. See Above.

2. Don't do that. It means you throw everything into that port on your xbox. I don't think it should let you do that anyway.

3. Decides on the fly where it should go, which is probably 3074.

Some people also just do a DMZ and point everything at their local IP on the xbox with the assumption that everything can go there and then figure out if it shouldn't go there. I think there are a few additional ports that the xbox 360 uses, so you may want to add those too. This goes under the assumption that there is no network exploit to hack an xbox.   • Port 88 (UDP) 
  • Port 3074 (UDP and TCP) 
  • Port 53 (UDP and TCP) 
  • Port 80 (TCP)

I stumbled upon this same issue on a Windows Server 2012 box and wanted to share my fix, just in case it helps anyone else.  You'll need to grant Administrator access to edit these, but to do so you'll need to change the ownership of the registry entry first: 

1.  IMPORTANT: Run Regedit as 'Administrator' and find your class id in question.

2.  Right click on the entry and edit Permissions > Advanced Properties and change OWNER to 'Administrator' and then Apply.

3. Back on the permissions tab Add 'Administrator' with full control permissions.

Now open the components services and find your entry and you should now have the options available.  If you have component services already open be sure to refresh the folder first. 

In my case I need to do this to both the Immersive Shell and Immersive Shell Broker.  All works like a charm now.

Good Luck! 

Daniel
