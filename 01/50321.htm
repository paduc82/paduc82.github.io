lemp say:

To start at the beginning, when a computer's network adapter (either Ethernet or wireless) shows an IP address in the range 169.254.0.1 through 169.254.255.254, it means that the network adapter is configured to use Dynamic Host Configuration Protocol (DHCP) to obtain its IP address but the adapter has been unable to obtain an IP address from a DHCP server.  IP addresses in this range are assigned by a process called "Automatic Private IP Addressing" (APIPA). 

In the case of a wireless network connection, this situation generally is caused by a failure to authenticate to the router -- in other words, not having the correct password for the wireless network.

Because there is no password involved in an Ethernet connection, it's more difficult to determine the cause.

The WINS tab is not relevant to this issue.  NetBIOS over TCP/IP is a method of identifying network resources by "NetBIOS Name" rather than by Internet Protocol address (IP address).  Changing the setting from "Default" to "Enable NetBIOS over TCP/IP" generally has no effect, and in any event, won't assign an IP address -- which is the problem you have.  Leave the setting at "Enable NetBIOS over TCP/IP."

Similarly, your comment that "when the Network Connection lease expires, the 169 IP is once again assigned" is not likely to be the case.  You can check in your router's configuration utility, but I expect that the default length of the IP address "lease" is one day, rather than a few hours.  In addition, in normal operation the DHCP client (your network adapter) will attempt to contact the DHCP server (your router) and renew the lease well before the lease expires.  For the TL;DR version, see http://www.tcpipguide.com/free/t_DHCPLeaseLifeCycleOverviewAllocationReallocationRe.htm

Sometimes, resetting the network components using the netsh commands you mention will fix the problem.  It's not clear from your question whether the problem is fixed after running those commands or if the APIPA address has returned.

I don't know what your "orange light" means.  Is your understanding that it means that you have a slow connection based on the User Guide, or is that just a guess?  What specific model ARRIS router do you have?

If after using the netsh commands your IP address problems recur, I would start to suspect a hardware problem:  either the Ethernet cable, the Ethernet network adapter in your computer, or your router.  It will be difficult to track down these problems. 

  • Does the problem still happen if you use a different Ethernet cable?
  • If your router has more than one LAN port, does the problem still happen if you connect to a different LAN port?
  • If you have another computer, does it suffer from the same problem when connected to the router by Ethernet?

tryx3 say:

It is possible that your computer is trying to send a resolution & frequency that the TV cannot handle.  I suspect this because the picture appears during startup but then disappears when Windows takes over control - until Windows takes over, the resolution sent will be a lower one.

You need to check the TV manual for its specs.  Mine, just for example, has a straightforward table of the resolutions & vertical frequency combinations that it can handle.  Then look at Control panel, Display, Change display settings to see what your computer can produce and choose one that suits both machines.  If the best resolution display setting does not work then go to the next best resolution [or work up from 640x480 to find out which works best].

Once set up, Windows will output the chosen setting to the TV screen every time so you will not need to keep setting it up for each session.

Every time you connect the TV you can just use the Windows key + P key combination to switch displays.

I don't know if you want the TV to be an Extended desktop or a sole display.  You might find, as I did, that the TV can use a higher resolution output if it is the sole display.  I suspect that this is a result of the computer's display adapter capabilities as it has less work to do if there is only one display required at a time.

Once you have it set up, you might find that  Control panel, Display, Change display settings, Advanced has useful additional settings but you would probably need to have a good Help guide for your display adapter to make any sense of the options available.

If this does not help then I suggest you try to find an online forum for your TV set in the hope that somebody there has done what you are trying to do.

Best of luck,
