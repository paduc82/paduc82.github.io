App icons mixed up - Simple solution if only a few icons are impacted - Microsoft Community

sf321 say:

App icons mixed up - Simple solution if only a few icons are impacted

The icon for one of my apps recently switched to that of a different app.

As a result, whenever I had both the app with the wrong icon and the app from which it "stole" the icon open, the taskbar showed 2 instances with the same icon.  Also, files that open with the respective app showed the same wrong icon.  That's a hassle if you are used to select files or switch between apps based on the icon.

 

Unless the affected app link is on your desktop or a user folder, type the name of the affected app in the Start / Search box, and choose "Right-click → More → Open File Location" to find the start menu link.

Once you opened that file location, right-click on the link there, once again using "Right-click → More → Open File Location" to find the original *.exe file.

{Going 2 levels deep should suffice to find the original app file.}

 

Turned out that only the link under

 %ALLUSERSPROFILE%\Microsoft\Windows\Start Menu\Programs

 %ALLUSERSPROFILE%\Microsoft\Windows\Start Menu\Programs

(for all users, typically C:\ProgramData\Microsoft\Windows\Start Menu\Programs)

or for user-specific start menu items:

 %APPDATA%\Microsoft\Windows\Start Menu\Programs

 %APPDATA%\Microsoft\Windows\Start Menu\Programs

(usually c:\Users\%USERNAME%\AppData\Roaming\Microsoft\Windows\Start Menu\Programs)

had the wrong icon.  The original *.exe file that the affected link pointed to still showed the correct icon.

 

{Note: In case the icon for the original file is mixed up, rebuilding the icon cache might be the only option - see below.}

 

In order to correct the icon for the link, I had to re-create it in the 

 %ALLUSERSPROFILE%\Microsoft\Windows\Start Menu\Programs

 %ALLUSERSPROFILE%\Microsoft\Windows\Start Menu\Programs

location, which is write-protected.  You will need to switch to an Admin account to delete or better rename the link to sth. like "*_wrong", then go to the location of the original *.exe file, and e.g. Ctrl-Shift-drag it with your mouse to the

 %ALLUSERSPROFILE%\Microsoft\Windows\Start Menu\Programs

 %ALLUSERSPROFILE%\Microsoft\Windows\Start Menu\Programs

folder to create a new link, which then should show the correct icon.

 

Note that the name of the link in the Start Menu is often more descriptive than the name of the original *.exe file, and when you create the new link it will typically show just the name of the *.exe file, sometimes including the .exe extension. 

In order to fully restore the original link name you will need to rename it, also as an Admin. Hence, it is helpful if you just renamed the mixed up link so that you can easily copy/paste the portion of the original link over the new link.  Don’t forget to delete the "*_wrong" link.

 

If your link with the wrong icon was in a user-specific folder such as the

 %APPDATA%\Microsoft\Windows\Start Menu\Programs

 %APPDATA%\Microsoft\Windows\Start Menu\Programs

or another folder (incl. Desktop) under

 %USERPROFILE%

 %USERPROFILE%

you will not need to switch to an Admin account to make the above changes to the link folder.

 

If you have more than a handful of icons mixed up, see e.g. the "Icons mixed up" thread indicating how to rebuild your icon cache: http://www.vistax64.com/tutorials/117229-icon-cache-rebuild.html, or the description in the "Mixed Icons in Windows Explorer" thread.  That should fix the issue in most cases (I personally have not tested it, though).

 

Hope this lengthy description will help shed light on some of the situations with messed up icons, and will help you fix it.

Once you figure out how to diagnose and fix the above, following the steps is actually less daunting than the length of the detailed description above makes it appear.

greg carmack - windows mvp 2010-2020 say:
Hi Mohammed. I'm Greg, an installation specialist and 9 year Windows MVP here to help you.

Update the Touchpad driver from the PC maker's Support Downloads web page, using the full model number, Serial Number or Dell Service Tag on sticker. Compare the latest drivers available for download with the ones presently installed in Device Manager reached by right clicking the Start Menu. Make sure you also have the latest BIOS or UEFI firmware, Chipset, Display (Video), Sound, USB3, Bluetooth, Network and all other drivers, or else download and install the latest now.

If that doesn't help then enter Device Manager reached by right clicking the Start button, go to Mice > Touchpad and then Driver tab. Choose Roll Back or if not present choose Uninstall, restart PC using keyboard to Ctrl Alt Del, on blue screen tab to Power Icon in bottom right, press Enter, arrow down to Restart, press Enter to restart to reinstall the driver.

See the troubleshooting steps here: https://support.microsoft.com/en-in/help/402786...

Go to Settings > Devices > Touchpad to go over the Settings. Then choose Additional Settings which opens the Mouse properties box. The Touchpad software installs a tab onto it - usually the last one - that should have more settings to check, including gesture features of that touchpad and a checkbox to Disable Touchpad when USB pointing device is atached. This prevents palm drag from causing the cursor to jump


I hope this helps. Feel free to ask back any questions and let us know how it goes. I will keep working with you until it's resolved.
