keith a. miller say:

This happens due to corrupted saved folder views in the registry.

It can be fixed by deleting the keys that store folder views.  If you don't mind losing all saved folder views, the fix is quick.

  1. Open Registry Editor
  2. Close all open File Explorer windows.
  3. In Regedit, navigate to HKCU\SOFTWARE\Classes\Local Settings\Software\Microsoft\Windows\Shell
  4. Delete the subkeys named BagMRU and Bags.
  5. Close Regedit.
  6. Right-click the Taskbar & launch Task Manager.
  7. Scroll down to locate the Windows Explorer background process.
  8. Right-click & select Restart.

 That's it.  Launching File Explorer should now show Quick Access in it's proper view.

a. user say:

Hi Brian,

Thank you for posting in Microsoft Community.

1) What is the exact error it displays while removing/deleting files?

2) Are you the administrator of the computer?

Sometimes you cant delete a certain file or folder because you dont have appropriate permissions for it. In order to gain full permission of the file or folder, you need to change the ownership of the file and gain full control of it. Heres what you should do to get the full control of a certain folder or file:

  1. Right click on the file you want to delete and choose Properties
  2. Under Security tab, click Advanced button
  3. The Advanced Security Settings window will appear, and youll spot Owner
  4. In some cases SYSTEM is listed as owner, and in some it is TrustedInstaller, click on Change option, next to the name of Owner
    • Note: Trustedinstaller is a Windows built-in account which removes and modifies updates and other Windows components. But you should be warned that if TrustedInstaller is listed as an owner of a certain file, you shouldnt change it. Because if you rename or delete the file your system may become unstable even more, so change the ownership from TrustedInstaller only if you know what youre doing.
  5. Enter the username you want to become the owner file, press Check Names to make sure if its okay, and press OK
  6. Youll get back to the Advanced Security Settings window, but youll notice that name of the Owner has changed, and youll also notice a checkbox called Replace owner on subcontainers and  objects, check that checkbox and click Apply
  7. Close Windows Security Properties (Windows will require you to restart it)
  8. Open the Properties again, by right clicking on the file, Security and then Advanced
  9. Under Permission tab, check Replace all child object permission entries with inheritable permission entries from this object
  10. After that, click Edit
  11. In Permission Entry window check Full Control and click OK
  12. Try to delete file/folder again

Hope this helps. Please reply to us with the status of the issue.

Hi

 

The error message that you are seeing, the TDR (Timeout Detection and Recovery).
This behavior can occur for one or more of the following reasons:

1. You may need to install the latest updates for your display driver.
2. Visual effects, or too many programs running in the background may be slowing your PC down.
3. The Graphics device is taking more time than permitted to display graphics to your monitor.

 

When the installed graphics device takes too long to respond, Windows will attempt to restart the device. If this happens too often or if it fails, then the TDR error will occur.

 


In your case, the driver (idgkmd32.sys) for the installed Intel Graphics Media Accelerator 3600 Series is causing the problem.
