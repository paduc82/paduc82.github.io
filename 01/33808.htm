davefmsft say:

This is a real bug in 1803 or some update in that timeframe.

The problem is caused by very long folder names and/or paths. Folder properties file enumeration *breaks* when it encounters one of these.

The potential for loss of data seems very real. One scenario: a folder containing a set of nested folders forming a long path that breaks folder properties, with no files in any of the folders. This top-level subfolder is followed by other subfolders with thousands of files. Folder properties would show 0 files thanks to this bug. User shift-deletes the root folder thinking it's safe and watches in horror as their data is purged.

Browsers and modern dev tools like npm create these long paths all the time!

Recipe for serious problems. This bug needs attention immediately. No Windows user should be expected to use a third party utility to work around one of the most basic services of an operating system.

midnightsun1 say:

Unable to defragement system drive

Original Title: System Defragmentation Help

 

I'm trying to degfragment my SYSTEM disk where it says 16% fragmented. I heard that if it's above 10%, it needs to be defraged. I'm trying to "defragment disk", but when I do it doesn't do anything, it still says 16%. I successfully defraged the other drives, but this one doesn't want to do so. What can I do? Thank You.

yaqub k say:

Hi,

 

There are several technical approaches to defragging, but most require that you have enough free space on your hard disk for a copy of the largest file that needs defragging. Typically defragmentation just require some percentage of free space, like 10% or 15%. If there's not enough room for a second, temporary, copy of the file then the file cannot be defragged. The most common cause for files not getting defragged is that there's not enough free space on the hard disk to do so.

The second most common cause is that the file is in use by some program. That's why most defragging utilities suggest you close down all running programs prior to attempting to defrag. One of the things you can do when you run into this situation is to look at the list of files that were not defragged and see if they are in use.

 

I would suggest you to follow the method mentioned below and check if it helps.

Method 1:

Try to run Disk defragmentation in Command Prompt. Advanced users can use the command line tool Defrag.exe to generate detailed reports and perform other advanced tasks. To run Disk Defragmenter from a Command Prompt, follow these steps:

a.       Click on Start.

b.      Type cmd in search bar.

c.       Right click on cmd and select Run as administrator.

Now if you are trying to run defragmenter of C drive, then this should be the following command.

defrag  C: /F

(Where C is volume or drive)

d.      Hit enter.

This forces defragmentation of the volume when free space is low.

 

 

Method 2:

If the above method doesnt help, I would suggest you to run check disk and check if it helps.

Follow the steps:

a. Click on Start.

b. Type cmd in search bar.

c.  Right click on cmd and select Run as administrator.

d. Chkdsk /f/r

e. Hit Enter.

Important: While performing chkdsk on the hard drive if any bad sectors are found on the hard drive when chkdsk tries to repair that sector if any data available on that might be lost.
Note: Make sure that you back up the data before proceeding with Check disk.
For more information about how to back up and restore the registry, follow the steps from the following link:
Back up the registry
http://windows.microsoft.com/en-US/windows7/Back-up-the-registry

 

 

More information : http://technet.microsoft.com/en-us/magazine/ff458356.aspx

 

 

Please let us know if the issue persists. We will be happy to assist you further.
