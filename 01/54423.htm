If you don't know where DMP files are located, here's how to get to them:

 1.    Navigate to the %systemroot%\Minidump folder.

2. Windows key + Pause key. This should bring up System. Click Advanced System Settings on the left > Advanced > Performance > Settings > Advanced > Ensure there's a check-mark for 'Automatically manage paging file size for all drives'.

3. Windows key + Pause key. This should bring up System. Click Advanced System Settings on the left > Advanced > Startup and Recovery > Settings > System Failure > ensure there is a check mark next to 'Write an event to the system log'.

Ensure Small Memory Dump is selected and ensure the path is %systemroot%\Minidump.

4. Double check that the WERS is ENABLED:

Start > Search > type services.msc > Under the name tab, find Windows Error Reporting Service > If the status of the service is not Started then right click it and select Start. Also ensure that under Startup Type it is set to Automatic rather than Manual. You can do this by right clicking it, selecting properties, and under General selecting startup type to 'Automatic', and then click Apply.

a. user say:

This is a commonly misunderstood topic. Here's the source for the misconception: Just because an updated driver is available, that doesn't mean you should install it.

It's contra-intuitive, but a newer driver is not necessarily a better driver. Updated drivers are issued to fix problems with the previous driver. If you're not experiencing one of the problems that the newer driver fixes, there's no reason to install the newer driver.

Moreover, installing a driver that you don't need to install can be a bad idea: Every time you add new code to a computer, you risk breaking something. That's how software is. Crazy, right? So if your hardware is not experiencing a problem specifically addressed by a new driver, don't install the new driver. I have some drivers so old they were delivered to me on cuneiform tablets.

Your antivirus program may be aware that a new driver has been issued. Does that mean that you should install it? Almost certainly not, but here's how to know for sure:

Go to the manufacturer's website and find the web page from which you can obtain new drivers. Every new driver includes an explanation of exactly what the new driver is fixing. Sometimes the information is inside what's known as a 'change log' or a 'readme' file. Take a look: Does the driver solve a specific problem you are experiencing .... no? Then don't install it.

In general, antivirus software should stick to chasing viruses, and not try its hand at areas outside its expertise.
