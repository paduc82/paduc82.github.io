Hey everyone,

Just had this problem today on a bootcamp installation of Windows 8.1 and was able to figure a fix. It seems to be a glitch in the Apple Wireless Keyboard Bluetooth driver built in to the bootcamp installation suite. As you're probably aware, Apple doesn't seem to update these all that often.

My issue began when I rebooted into windows: my mouse worked fine, but my keyboard was entirely unresponsive. Holding down the power button to turn the keyboard off and then turning the keyboard back on again didn't do anything to help. This seems to be the same issue that a lot of you are having.

I first used the Accessibility options from the login screen to bring up the virtual keyboard, and type in my password so that I could log in. Next, I clicked the little bluetooth settings icon in the taskbar area (next to the clock) to see if my keyboard was connected: it was listed in the menu as disconnected, but would not connect. I also noticed that there was an unidentified bluetooth device ready for pairing. This is interesting because the keyboard is the only bluetooth device anywhere near my computer.

I was able to solve the problem in this menu by removing the malfunctioning keyboard device in the bluetooth menu, by clicking the "Remove Device" option. When I did this, the unidentified bluetooth device disappeared from the menu screen, as did the malfunctioning keyboard entry, which was labelled simply "Keyboard." An entry for "Apple Wireless Keyboard" did then also appear, and was ready to pair.

I was then able to go through the normal pairing process (type in a randomly generated string of number and hit enter on the keyboard) to get the keyboard working again. It took me two tries, but I'm not sure if that was a driver issue or just a Bluetooth glitch. It easily could be either one.

On the attempt that failed, the progress bar under the Apple Wireless Keyboard entry seemed to freeze in place, so I clicked the Remove Device option. This immediately completed the progress bar, and then subsequently removed the device, allowing me to restart the pairing process. The second time it worked with no issues.

Hopefully this helps!
