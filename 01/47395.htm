wunders say:

My initial take on your problem is that you are confusing a NTP Time "Client" with a Time "Server".

If you would like to synchronize your time with an external machine, that makes you the Client and you must specify a Time Server as your external machine to synchronize with.  The KB article you cite gives instructions on setting up a Server, not a Client.

It is unclear how you entered the server IP address or DNS name that you are trying to time sync to, but in the past, I've had some success with the following command:

    net  time  /setsntp:xxx.xxx.xxx.xxx

followed by

    w32tm /config /update

followed by

    w32tm /resync /rediscover

Your Time server may not be working properly or perhaps it isn't a server.  Machines will return a "ping" even if they aren't a server.  Try a different time server. To get the IP address of several public NTP servers, visit the site:

    http://www.pool.ntp.org/en/use.html

The above site gives instructions on how to set a NTP server address on a Windows machine.

It is unclear to me on your "w32tm /stripchart ..." command what machine xxx.xxx.xxx.xxx is.  Is that a server machine or another client machine on your network?

HTH,

  JW
