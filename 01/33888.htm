You haven't mentioned what version of Windows you are using, but I'll assume that the one you are having trouble with is Windows 7.
Pinning Windows Explorer onto the taskbar with an intended target is a bit more involved than previous version of Windows. You will need to create a full shortcut using the explore.exe application and a special environment variable on the desktop and use that to Pin to Taskbar.
  1. Right click a blank area of the desktop and select New, Shortcut.
  2. Use the following as the entry for Type the location of the item:
              %windir%\explorer.exe "::{20D04FE0-3AEA-1069-A2D8-08002B30309D}"
      ...and click Next.
  3. Supply a name for your new shortcut and click Finish.
  4. Right click the new desktop shortcut and select Pin to Taskbar.
Anything that is to be pinned to the taskbar or start menu needs a reference to the underlying application that will launch the process and not just the target document, URL or folder.

There are different methods on how we can have this resolved. Following the step by step methods would be vital on our troubleshooting steps.

Disable "Receive segment coalescing" (Rsc) on your wireless adapter

  1. Open an Admin Command Prompt window:
    Start > All Apps > Windows System > Right-click Command Prompt > More > Run as administrator.
    • Note: If you want this available on your Start's Right-click menu, you can go to Settings > Personalization > Taskbar > Replace Command Prompt > Set to Off.
  2. Get your network adapter name:
    powershell Get-NetAdapter
  3. Note your adapter's name.
  4. Get your network adapter Receive segment coalescing (Rsc) settings:
    powershell Get-NetAdapterRsc
  5. Disable Rsc for your network adapter:
    • powershell Disable-NetAdapterRsc -Name ADAPTER_NAME_FROM_BEFORE
    • Note: Be sure to replace ADAPTER_NAME_FROM_BEFORE with whatever your network adapter name is.
  6. Verify Rsc is disabled for ipv4 and ipv6, on your network adapter:
    powershell Get-NetAdapterRsc
    • Should now show IPv4Enabled False, and IPv6Enabled False, for your network adapter.
  7. Check  your network connection.

If that did not change anything, change MTU to 1400 (from Default, which may be 1500).

To view your MTU:
Right Click Start > Command Prompt (Admin) > Type or right-click-paste the following command:
netsh interface ipv4 show subinterfaces

make a note of your wireless one, it might be called Wi-Fi or Wireless Network Connection.

netsh interface ipv4 set subinterface "YOUR_WIRELESS_CONNECTION_NAME" mtu=1400 store=persistent
