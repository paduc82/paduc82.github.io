jallanite say:

Ever since at least Windows 3.1, holding down the Alt key and typing numeric-keypad 0 followed by the decimal value of the current 256-character character set has produced the corresponding character in that character set. This has never had anything to do with Unicode and is unchanged in Windows Vista. If you type a number greater in value than 255, the character value has always wrapped. Vista works in exactly the same fashion.

In Microsoft Word and in WordPad a fix was added to enable Unicode characters if you typed a decimal number greater than 255, with or without a leading 0, on the numeric keypad while holding down the Alt key. Then the number typed would cause the corresponding Unicode character to appear. But this only works in Word or WordPad. You must be misremembering if you believe this worked generally.

If you want this to work in the entire Windows system rather than just in Microsoft Word and WordPad, I suggest you download and install the free Quick Unicode Input Tool from http://www.cardbox.com/quick.htm .

Search on EnableHexNumpad in Google to see a method which Microsoft set up to allow Unicode entry, but does not enable, probably because the keystrokes required often cause a menu to activate.

shilohbob say:

Simple answer: No

You have dual graphic cards installed

1 -Intel HD Graphics 4600, and

2 -Nvidia GeForce GTX 750ti

They require their own driver/s

When you are using low graphic demand (general everyday use) the PC/Laptop will use the lower powered graphics card- Intel HD Graphics 4600.

When you have demand for a more powerful graphic, such as a full HD movie or gaming the PC/Laptop will automatically utilise the power of the Nvidia GeForce GTX 750i graphics card.

This done to conserve power especially if running on battery power.

Your monitor/s also require their own driver/s, basically to connect to the PC/laptop.

The graphics cards provides information to be shown on your monitor through your PC/Laptop.

This is a basic description of the process.

keith a. miller say:

As DaveM121 has said, you will be guided through the OOBE. The two most common issues users have with a new computer afterwards is that they don't like the name of their User profile folder and having their user folders (Documents, Pictures, Desktop) being synced/redirected to OneDrive.

If you accept the recommended setup and provide an email address associated with your Microsoft account, your User profile folder name with be the first five characters of the email address. If you want control over the folder name, you need to create a local account and then link it with your Microsoft account after creation.

The OneDrive issue is easier to deal with -- just easy to miss. One of the screens will offer to 'protect' your files; the 'Continue' (not sure of exact nomenclature) button is huge, the option to save files only locally is smaller and on the lower left-hand side.

Your first step should be to determine if this is a hardware of a software problem. Here is how you can do it:

  1. Boot the machine into Repair Mode.
  2. Open a Command Prompt.
  3. Check if the problem persists.
