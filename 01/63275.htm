I'm having the exact same issue on Surface 3.

N-trig, a touchscreen solution which Surface is using, has a long standing issue that's been around for years related to weakness in dealing with electrostatic charges in and outside of the device. It usually shows up as touch acting up strangely, like in Surfaces' case, so I suspect the issue with Surface devices to be also caused by the same reason. Older N-trig drivers for other devices used to have a utility which users can run manually to make the device calibrate the touchscreen's electrostatic profile. Microsoft didn't include this utility on Surface 3 and Surface Pro 3.

You can solve the problem for a short time by wiping the screen with your palm to clean up any faulty touch registrations and irregular charged spots on the screen(but the problem returns in a very short time, like in a matter of seconds), and I think this supports my speculation of the cause of the trouble.

I am hoping Microsoft is aware of this issue and is already working on a driver or firmware update...

bossdweeb say:

Some more information such as the name of the game and the brand and model graphics processor could be helpful

as some games will show this sort of error if the machine does not meet a games minimum system requirements.

- The DirectX versions your graphics processor is capable of using is shown in dxdiag > Display tab > Feature Levels.

- The DirectX Version shown under the System tab is useless. This only shows the highest level of DX Win10 can use.

(copy/paste from a post I made on the topic  yesterday).

Run the DirectX Web Installer to verify if all DirectX files are installed on your Win10

Download DirectX End-User Runtime Web Installer from Official Microsoft Download Center

- Win10 has DX9,10,11 and 12. Very few games use DX12 at this point. Windows does not determine what version,

or which DirectX file gets used, the game does.

- Old games are limited to using DX9 (even older games use only DX7 or DX8).

- More modern games (since Vista & Win7)  use both DX9 and 10 or 11. This allows the games to run on lower spec,

and older systems that don't have the power or capability (the graphics processor has to be capable of using

the version of DX in question) to use DX10/11.

- A number of recent games are restricted to DX10 or 11 and have no option for DX9. I'm not certain if any of the

few DX12 titles have a DX9 option.

- Some games have a DX10/11 option in the games setup, but others use the DX10/11 files automatically when the user

selects Very High or Ultra settings. If the graphics processor is not capable of using a particular version of DX

the relevant graphics settings is normally greyed out in the game graphics setup.

Method 1.
In Windows 7 this option is moved to a different location. Here are the steps to locate it.
a. Launch Windows Media Player.
b. Go to Tools and then Advanced.
c. Now click on Restore deleted library items.

Method 2.
If the above steps do not help you may need to rebuild the Windows Media player database.
Here is how:
To resolve this problem, clear the Windows Media Player database. To do this, follow these steps:
   a. Exit Windows Media Player.
   b. Click Start, click Run, type %LOCALAPPDATA%\Microsoft\Media Player, and then click OK.
   c. Select all the files in the folder, and then click Delete on the File menu.
       Note: You do not have to delete the folders that are in this folder.
   d. Restart Windows Media Player.
 
Note: Windows Media Player automatically rebuilds the database.
