Do you have COMODO or AVG Antii-virus installed on your system?  There are known problems with these two programs and MCbuilder.exe.  I don't have Microsoft Resource Cache Builder as a service in Services (so I can't enable or disable it there) - what version of Vista are you using (version, bit-number, SP-number)?  I have 32-bit Vista Business SP2. I can see no way to disable it except to shut it down with Task Manager when it appears - but that could become tiresome if it happens frequently and with multiple instances.  I could find nothing useful in the Microsoft Knowledge Base about it hogging resources though I saw a few threads using Bing (which is where I came up with the COMODO and AVG Anti-virus conflicts).

You may be infected with malware (which may be causing the program to activate and run for "no reason"). While in programs: http://www.malwarebytes.org/mbam.php and http://www.safer-networking.org/en/index.html.  You may also want to try a free trial of Pandasoft (www.pandasoft.com).  Before downloading pandasoft, uninstall whatever anti-virus program you are currently using (because installing and running two at the same time can cause conflicts, freezes, and all sorts of problems).  Once you've uninstalled your current AV program, download, install, update, and scan using Pandasoft.  I think you might be amazed at how much it finds that the other software missed.  Once complete, uninstall pandasoft and re-install your current AV program (unless you decide to switch which I did when I first tried it about 4 years ago).  You may also want to try the new, free Microsoft Security Essentials http://www.microsoft.com/security_essentials/default.aspx (with the same caveat that only ONE AV program can be installed and running on your system at any one time). Reboot after completing all the scans and see if the problem has been resolved.

Try a System Restore to a point in time BEFORE the problem began (which may be a problem if it began all the way back in June - you should have posted here then when this process would probably have worked).  Here's the procedure: http://www.howtogeek.com/howto/windows-vista/using-windows-vista-system-restore/.  Be sure to check the box to show more than 5 days of restore points (but I doubt you'll have one far enough back so System Restore may not be a viable option).  If the first attempt fails, then try an earlier point or two.  NOTE: You will have to re-install any software and updates you installed between now and the restore point, but you can use Windows Update for the updates.

If the System Restore doesn't work, do a Startup Repair by booting to the genuine Windows Vista Installation Disk (or one you can borrow from ANYONE) or from a Recovery Disk.  Here's the procedure: http://www.bleepingcomputer.com/tutorials/tutorial148.html.  To boot to the CD you may need to change the BIOS to make the CD-drive first in the boot sequence.  To do that, wait for the screen that tells you the F key to push to access the boot menu or boot setup.  Push it quickly. Make the changes, save your work, and exit.  Put the CD in the drive and reboot.  When prompted, push any key to boot from the CD.

If you don't have either disk, you can make a bootable Recovery Disk using http://neosmart.net/blog/2008/windows-vista-recovery-disc-download/ along with burning software like: http://www.snapfiles.com/get/active-isoburner.html and, of course, a blank CD.

If that doesn't work, try to boot into safe mode (repeatedly click the F8 key while booting and go to safe mode with networking).  Then let's check some of your system files:

Go to Start / All Programs / Accessories / Command prompt and right click on command prompt and click run as Administrator.

Type sfc /scannow and enter and let it run.  It will scan and try to fix some of your system files.  Hopefully it will complete with no corruption it could not repair (if there is such corrouption post back here or try to analyze it to find the problem file(s) using http://support.microsoft.com/kb/928228.  I'll bet there's something there involving this program or related to this program.

If that doesn't work, we'll need to do a system repair/upgrade using the genuine Windows Vista Installation Disk (one you own or one you can borrow from ANYONE).  Here's the procedure: http://www.vistax64.com/tutorials/88236-repair-install-vista.html along with the upgrade from an earlier version of Windows section of the following: http://support.microsoft.com/kb/918884.  Although this will not affect your data, settings, or programs, you should still backup your data before starting just to be on the safe side.

Hopefully one of these procedures will resolve your problem (probably the SFC Command or the anti-virus checks).  If not, post back and we'll try something else.  Post back in any case so we'll know how it turned out.

BTW, for update problems like you're having with SP2, post to the Windows Update Forum at: http://social.answers.microsoft.com/Forums/en-US/vistawu/threads where people who specialize in update issues will be happy to assist you with your concerns and get SP2 running on your system (perhaps despite this particular problem).  As you say, it may even fix the problem - who knows.

Good luck!

christo_ say:

I found the solution - for my particular notebook (Lenovo B575 model 1450, from 2011) - based on AMD E450 APU processor, and SB850 southbridge SATA/USB/PCI/PCIe chipset.

I went into Windows Device Manager and foundthe AMD SATA driver. This is the hardware specific AHCI driver that talks to the SB850 southbridge chipset, and to Microsoft's storport.sys generic storage interface system driver.

I rolled back the AMD AHCI IDE driver, from the April 2014 version from AMD Catalyst Drivers 14.04, to the April 7, 2010 version from AMD Catalyst Drivers 10.04, the high CPU and lag went away, the "Interrupts" process went from consuming 80-90% of one core, down to 5-8% of one core!
