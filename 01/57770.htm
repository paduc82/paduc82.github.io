Windows abstracts the zip file into a folder. So all Explorer operations preformed are file/folder operations. Zip files don't support Window's shell objects operations (designed for random access media). So it needs to be translated on an individual operation basis to work on a sequential file.

 

Programs like dedicated unzippers perform sequential operations.

 

Take a 2 file zip. 7Zip will extract two files in one sequential operation. Explorer will query the zip as a folder (forcing it to be read fresh), extract the first file using windows shell functions (causing the zip to be read fresh), then the second file using windows shell functions (causing the zip to be read fresh).
