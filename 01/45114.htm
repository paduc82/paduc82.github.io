a. user say:

Ok, here's a possible solution to your problem. As I do not run Vista anymore, I'm not 100% sure if it works on Vista, but I think this new method of servicing got introduced with Vista.

But, first, let's make clear, this is not a problem with this update. You are or were using a program that corrupted your registry (actually, it may have been the uninstall of that program that corrupted it, but it could also be a security or any other program still running/or on the machine). So, there will be more arbitrary updates and programs that you won't be able to uninstall. This is a separate problem and you should troubleshoot it.

As for removing this update, open a command prompt and issue the following command (do not type, use copy and paste, so you cannot mistype):

FORFILES /P %WINDIR%\servicing\Packages /M *3013455*.mum /c "cmd /c echo package @fname"

If this command lists some packages (should be 4) then the following command will uninstall the package. You may need to run that command in an elevated command prompt.

FORFILES /P %WINDIR%\servicing\Packages /M *3013455*.mum /c "cmd /c echo Uninstalling package @fname && start /w pkgmgr /up:@fname /norestart"

You have to restart the system to finish this!
