2. The integration with Exchange scenarios:

    Knowledge:

    The following logic is used by SfB / Lync clients and Devices to locate Exchange Web Services within an Exchange environment from an internal or external network.

    SfB / Lync client will attempt to read any existing Autodiscover data with a valid TTL which may have been previously retrieved (by Outlook

    The SfB / Lync client will query for the users PrimarySMTP.

    A users PrimarySMTP is determined by querying the Active Directory Email address field which is populated by Exchange when a user is mail enabled.

    The smtpdomain will be pulled for Exchange CAS Autodiscover lookup in the following order

    * https://<smtpdomain>/autodiscover/autodiscover.xml

    * https://autodiscover.<smtpdomain>/autodiscover/autodiscover.xml

    * http://autodiscover.<smtpdomain>/autodiscover/autodiscover.xml

    In the event of Autodiscover failure, retries will occur as follows:

    * If Autodiscover fails, retry on a three minute interval over a period of two hours

    * If Autodiscover fails after three successive attempts, use MAPI but continue to retry

    * If Autodiscovery succeeds, refresh cached values on an hourly basis

    2.      Type the following command, and then press ENTER:

    sfc /scannow 

    A message will appear stating that 'The system scan will begin'. Be patient because the scan may take some time

    3.      If any files require a replace SFC will replace them. You may be asked to insert your Win7 DVD for this process to continue

    4.      If everything is okay you should, after the scan, see the following message "Windows resource protection did not find any integrity violations" 

    5.      After the scan has completed, close the command prompt window, restart the computer and check.
