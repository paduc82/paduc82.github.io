A library, also known as a dynamically linked library, is a collection of commonly used functions that a program can refer to over and over, which saves the developers the need to code these functions into their product over and over.

The 'procedure entry point' is the place inside the library file where the code for a function begins.

A file in your game was trying to use one of the functions in a library but couldn't find where the code begins.

This is not uncommon in software. It often means that the library file got damaged, which also happens. That's life. All you have to do is reinstall the program, and you'll get all new files. That almost always solves the problem.

¡firedog say:

Hotmail doesn't support IMAP, so you're leaving them on the server for some other purpose. If you used HTTP instead of POP3, messages would still be left on the server until you delete them either there or in the client.

I have found the messages display in the unread folder but not in the hotmail inbox under accounts so I have to save them in storage folders rather than leave them in the inbox.

If you can see messages in Quick views, then they are in an account folder. That's all Quick views is - a view of a messages in folders selected according to the criteria specified.  WLM 2011 which uses an http server for a hotmail account downloads attachments such as pdf as _pdf so cant be opened by reader and have to be exported and renamed.

If you see that an attachment has an extension like _pdf, this is probably because the filename specified by the sender ended with a full stop. This would give rise to a full file name with two dots before the extension: Annual report..pdf. Because some programs will choke on file names like that, WLMail converts one of the dots to an underline character. It's just a pity that it converts the wrong one.

It's generally not a good idea to open attachments from within a mail message. This can circumvent some of the security measures in place to guard against malware. It's always best to use Save as to save the attachment first. This will convert the attachment to a file, which will be properly scanned by your security software. You also get the opportunity to specify your own preferred file name. My bank sends monthly statements which always arrive as statement.pdf. I right-click on the attachment and select Save as to rename it Statement-2013-03.pdf and save it in the appropriate folder.

¡firedog say:
why is Word 12 allowing illegal characters in the filename,
It's not illegal to create filenames including punctuation, in that Windows will normally allow it. It's just that some (especially older) programs will choke on them. WLMail is just trying to save that hassle.
 
the user who is used to opening the pdf directly has trouble with saving and renaming the file
Right-click Save as shouldn't be too hard to learn. WLMail will normally remember the last folder saved to, so that could be My documents. As for renaming, all he has to do is type a new name and end it with .pdf Someday, someone will send him an attachment with no filename (it's not required), so all he will see is something like ATT000345.txt. If he knows it's a Word doc, then he'll have had practice in renaming files before saving them.
 
most AV vendors have an email scanner built in and also scan on opening
Yes, but you're not really opening a file in the normal understanding of that process. It's my understanding that if you double-click on an attachment, WLMail will decode the attachment and ask Windows to pass the decoded text stream to whichever application is called for in the attachment's Content-Type header. If this is, say, application/msword, then Windows will launch Word to deal with the stream. This is all happening in memory; the AV program might not get its hands on it until Word has created its temporary file and written it to disk. If there's a devious hacker at work, this might be too late. If you save the attachment first, the stream is converted to a file and written to disk. That's when the AV will intercept it and scan it, potentially saving you from harm.

One major shortcoming with XP and later versions of Windows is that extensions for known file types are hidden by default. This means that a criminal could distribute mail attachments called Brad Pitt with no clothes on.jpg.exe containing something very nasty. On a new Windows installation, this will show only as Brad Pitt with no clothes on.jpg; given a gullible recipient, the temptation to view might be too great and the nasty executable runs. This is one good reason why WLMail shows attached image files as thumbnails in the message header bar rather than at full size the bottom of of a message as Outlook Express did. Fake image files won't show in this way.

Your user is well protected with MSE. However, it's easy for hackers to impersonate 'trusted senders' and so take people unawares. Learning the simple process of saving attachments before opening them might pay for itself in saved grief sooner rather than later.
