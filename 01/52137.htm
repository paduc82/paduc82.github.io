Follow the steps below to try to fix Wireless connectivity issue.

Method 1:

Verify if DHCP is enabled on your computer.

1. Open Network and Sharing Center through the Control Panel or by right-clicking Network and selecting Properties.

2. Click View Status

3. Click Properties

4. Click Continue when Windows asks for your permission to continue.

5. Highlight Internet Protocol Version 4 (TCP/IPv4) and click Properties.

6. Ensure That Obtain an IP Address Automatically and Obtain DNS server address automatically are selected.

7. Click OK, and exit back to the Desktop.

Once you follow the above steps, right click on command prompt and click on ‘Run as Administrator’ and try to release and renew IP address.

Method 2:

If enabling DHCP does not help, you can try to disable  IP Helper Service:

Disable the IP Helper service:

1. Hold the Windows key and type R, enter "services.msc" (without the quotes) and press Enter

2. Scroll down to the IP Helper service, right click on it and select Properties

3. In the dropdown box that says "Automatic" or "Manual", set it to Disabled and then click on "Apply"

4. Then click on "Stop" to stop the service from running in the current session

5. Click OK to exit the dialog

I only run into issues like this if I oversaturate my receiver (or even the server) with a bunch of random Cast-To requests.  I find that it pays to be patient, and not queue up 600 video files that are being streamed via UPNP server.  I've watched the media server become "saturated" with requests for reencoding that it must "process" out, despite having cleanly exited the "Cast To" play list.  I'm referring to Windows Explorer & Windows Media Player "Cast To", as Movie & TV App is only going to allow one file at a time to be casted.

mb3wontfixit say:

Since my last post I have installed K Lite codec pack.  I can now cast wmv files and SOME .mpg files to my TV.  Any .mp4 files continue to show a silent movie.  It would appear that codecs have something to do with the problem which makes it all the more annoying since all my video files used to work well in windows 7

When it comes to video files, there's actually two codecs involved, the other being the audio codec.  While accessing the properties of a media file may not always show you complete codec information in Windows Explorer...

Or windows media player...

It is possible in Windows Media Player to view them in the library by switching to detail view and adding column fields "Audio Codec" and "Video Codec" (I also throw in type for good measure, because sometimes folks rename file extensions)....

Doing this can help shed a light on your lack of sound/video streaming to a media receiver.  Because despite what you believe to be a correct codec configuration, there may be an issue with the media receiver supporting the audio/video codec, or you're indeed missing a codec that is not standard.  A lack of support just means you need to check codec splitters to ensure they are properly reencoding the file as a supported type that the receiver requests. 

When streaming, there's a bit of handshaking or conversation that goes a little something like this:

Server: I want to give you a file to play

Receiver: well ok, great.  What is the original file type, and what file types can you reencode it that best meets my needs?

Server: This is all I have to offer bud...

Receiver: That won't work, go away.

As you can tell by this file type in the screenshots, it is an mp4 that was created from a Windows Phone (version 7).  This .mp4 can "Cast To" my Xbox One without any problems from Windows Explorer, Movie & TV App, and Windows Media Player.

Lastly.  K-lite used to be my goto codec pack, but with the latest and greatest version of windows, there isn't really a need for it anymore, as long as the multimedia files are appropriately encoded.  Any files created back in the day of MANY codecs, will still need those dusty old codecs, or converting to a better format.  Having said all that, I'd recommend Shark007 if you are getting into anything as decadent as my setup: Multimedia Archive from NAS running on a stand alone virtual PC running Windows 8 for the sole purpose of running Windows Media Player for media sharing services, allowing UPNP access to this media server from any device to "Cast To" another device.  Essentially, I browse my archive on my phone or tablet using a UPNP client and Cast To my Xbox One or TV.

The most important thing is understanding your media receiver.  My TV (coincidently a Samsmug as well) only supports AVI and a few other moldy oldies, bummer, meh that's what the Xbox One is for.  Go pro or go home.

a. user say:

Hi amcoops,

It sure looks like a hardware problem to me - either with the monitor or with the video card (it's hard to tell which without swapping out a good one for one of the potentially bad ones to see if the problem moves).

Go to Device Manager by going to start / search box and type device manager and enter and then double-click on the program icon that appears.  Check each device for a red x, yellow ! or white ?.  These identify devices with problems (probably drivers, but also conflicts or something else).  Click on each for further details and troubleshooting tips.  Get updated device drivers for both the monitor and the video card.  If you need to get drivers, do so from the computer vendor or device manufacturer (NOT from Microsoft Updates).  In fact, you should turn off automatic driver updates from Windows Update as follows: http://www.addictivetips.com/windows-tips/how-to-disable-automatic-driver-installation-in-windows-vista/. Proceed as follows to get the drivers: http://pcsupport.about.com/od/driverssupport/ht/driverdlmfgr.htm.  Once you have the drivers, you can install them through Device Manager as follows: http://www.vistax64.com/tutorials/193584-device-manager-install-driver.html.  Use these procedures if you're having trouble - there might be something you're not doing correctly.

If you update both drivers but the problem persists, then it's almost certainly a hardware problem with one or both of those devices.  You'll need to either send it back to the computer manufacturer for testing and replacement or find a competent, reputable computer repair technician (NOT Geek Squad or any of those big store centers).  If you have a spare of either device that you know works, swap it with the one on your system to see if the problem goes away.  If it does, then that's the bad device.  If it doesn't, then it's the other device that's bad.

I hope this helps.

Good luck!
