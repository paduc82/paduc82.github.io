lemp say:
For most people, the way that a virus gets into the System Volume Information folder is because the virus has infected a file that is backed up by System Restore. That's not really a problem, because the virus is effectively quarantined unless you actually restore using the infected restore point.  And after a period of time, the restore point will get deleted as System Restore does its own housecleaning.

It's possible, I suppose, for a malware author to deliberately store malware in the SVI folder, but I haven't ever seen anyone mention that.

In fact, the folder you reference is a System Restore Point (the _restore in front of the long string and the sub-folder name "RP45" should have been clues).

You may think that you have turned off System Restore, but it wouldn't kill you to right-click My Computer, select Properties, and click the System Restore tab to have a look.  (I just did that to confirm the method before I put it in this Reply, and to my surprise, SR had been turned back on for all of my non-system partitions, including my external drive.  I know I had it turned off before.)  And SR had gone back to its default of using 12% of my hard drive.

Because System Restore is an important safety feature, Windows makes it particularly difficult to get at the System Volume Information folder.  It's not intended to "save trojans" but to make it hard for people with a quick Delete key finger and too little knowledge to seriously damage their systems.  You don't need to resort to Linux, however, to get at this folder:  "How to gain access to the System Volume Information folder."  But be careful.  If you start deleting things manually from the _restore folder, you'll screw up System Restore (but if you want to turn it off anyway, that doesn't much matter).

System Restore doesn't always work exactly the way you might think it works, but if you check the box to "Turn off System Restore on all drives" and click OK (and agree to the warnings), the _restore folder and all of the RP subfolders in it will be deleted.
