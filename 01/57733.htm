Windows Store App broken - Win10 Pro V 1803 - After removing Store - Microsoft Community

kapten_spellczeque say:

Windows Store App broken - Win10 Pro V 1803 - After removing Store Apps with Powershell for sysprep

In a recent RDSH VDI deployment I needed to sysprep a fresh build of Win 10 version 1803.  I configured and installed all my software (which is a significant amount as we are a software development company) and installed all the available Windows Updates and then went to sysprep.  I received the classic error pointing to some random Microsoft Store App (in my case the windows.calculator) that was not installed as "All Users." 

To fix the issue I ran the Get-AppxPackage -AllUsers | Remove-AppxProvisionedPackage with the -Online switch.  This removed the problem package and allowed me to sysprep and roll out my VDI pool.  Shortly after getting users setup on the VDIs however I started to run into issues.  Calc.exe wouldn't run even if launched from the system32 directory stating "you will need a new app to open this ms-windows-store" ... then I realized I can't even open the MS Windows Store App at all anymore ... getting the same dialogue with an "ok" button that I can't click.  This only seems to be affecting half of my VDIs ... which makes it even more strange since they are all built off the same master image.

After a whole lot of searching and attempted fixes I had gotten to the point where nothing was working and I thought my only option was going to be to re-install windows and re-install all my software and settings and re-deploy all my VDIs ... UGH!!!  I tried every solution out there.  Ran Powershell scripts, removed SIDs from Registry, Tried fixing in the Apps and Features GUI, creating a new local user etc.  Nothing worked.  Every forum out there I found said to run ...

Get-AppxPackage  -AllUsers | Foreach {Add-AppxPackage -DisableDevelopmentMode -Register "$($_.InstallLocation)\AppxManifest.xml"}

I was receiving this error however ... "cannot find path 'c:/windows/winstore/appxmanifest.xml' because it does not exist."  and when I ran "Get-AppxPackage -AllUsers" the windows store app wasn't listed, nor was it listed in the "Apps - Apps & Features" in the GUI.  So I had given up.  I started building my new VDI and installed the same release of  Windows 10 (Version 1803), applied updates.  After a fresh install I logged in and made sure I was a local admin and I decided to try something off the wall, and I'll be damned ... it worked!  Could be a fluke, but just in case if it can help someone else out there, these are the steps I took.


  1. First I had to change the permissions on the C:\Program Files\WindowsApps folder … make myself the owner and add me with full control and add “All Application Packages” full control from the Broken PC not the domain (a solution I saw on some random forum but can't recall ... I will post the link later if I can find it)
  2. Look for Microsoft.WindowsStore_* (in my case it should have been at C:\Program Files\WindowsApps\Microsoft.WindowsStore_11808.1001.9.0_x64__8wekyb3d8bbwe) … in the list of apps folders in the WindowsApps folder ... if it is not there (which it was not in my case) then do the same for the permissions in the C:\Program Files\WindowsApps folder on my newly built VDI and copy the folder(s) for the Microsoft.WindowsStore* and all of the contents of said folder(s) from this location on my fresh new VDI to the same location on my Broken production VDI
  3. Then open an elevated powershell and run the following command, but make sure the folder name matches exactly what you copied and pasted in the previous step.  In my case it was …

    Add-AppxPackage -register “C:\Program Files\WindowsApps\Microsoft.WindowsStore_11808.1001.9.0_x64__8wekyb3d8bbwe\AppxManifest.xml” –DisableDevelopmentMode

This fixed my problem where the above "$($_.InstallLocation)\AppxManifest.xml" did not work.  This saved me from a re-install/re-build/re-fresh or in-place upgrade and several hours of work installing software, configuring settings and setting up a new VDI Pool, transferring users profiles etc. which would have really sucked.  So, hopefully this helps you too.

1. Run Windows 10 Battery Diagnostics

If your battery isn't fully charging, the first thing you can try is the Battery troubleshooter in Windows 10.

Open Start > Settings > Update & security > Troubleshoot
Scroll down then click Power
Click Run the troubleshooter

Complete the wizard then restart your device to see if the problem is resolved.

Most laptop computers include their own factory diagnostics utilities, too. It is recommended you also use those to test your battery. Accessing the diagnostics will vary by manufacturer, so, consult the documentation that came with your computer. Before you run a test, you might need to first fully charge the device to 99 or 100 percent, disconnect the AC adapter, boot into the diagnostic utility then run the test.

2. Check if Your AC Power Supply is Properly Connected

Sometimes the power cable connected to your AC power supply brick might be loose. Check to make sure it is fitted and secured properly.

3. Try a Different Wall Outlet and Check for Low Voltage and Electrical Issues

The other day, my house had some serious electrical issues going on. You would be amazed how much voltage issues can affect the performance of your system. Plugging in my laptop into an outlet within my room dimmed the light, while an outlet in the living room caused it to wink.

Another room it remained constant but charging was horribly slow. This, of course, meant I had low voltage issues and with so many appliances competing for the little bit of faulty energy, it made it even worse. Ultimately, I ended hiring an electrician to sort out the issues. After fixing it up, my laptop started charging normally.

4. Test with Another Charger

I mentioned the AC power adapter earlier. It's possible your charger might be the culprit. If you have a working spare laying around the home, give it a try to see if it charges. You can also borrow one from your work colleague if you donâ€:tm:t have any. If you don't have access to another charger, try yours on another laptop to see if it charges the device.

If you are mixing and matching laptop power adapters, make sure you choose the correct power supply (https://www.amazon.com/s/ref=nb_sb_noss_2?url=s...). Read the brick and make sure it has the same polarity and voltage as the adapter for your laptop.

5. Remove All External Devices

If you keep external devices such as a smartphone, a USB external drive, mouse, printer and anything else attached to your system while charging, it will take longer for your battery to reach 100%. Make sure you disconnect all of those devices so your system wonâ€:tm:t be competing with all the other electronics.

6. Check Your Connectors for Dirt or Damage

If your laptop has a modular battery, remove it then check for any damage or to see if it needs cleaning around the contacts and inside the battery compartment. Also, check if there are any dust particles inside the charging port that might be preventing the battery from charging.

7. Check for BIOS and Chipset Driver Updates

Sometimes your system might be overdue for an update for components such as your BIOS and Chipset. Updates can help resolve bugs that might be preventing the battery from charging efficiently.

8. Press and Release Power Button Reset

Sometimes unknown glitches can prevent the battery from charging. An easy way to fix it is to power down your computer, hold down the power button for 15 to 30 seconds, plug in the AC adapter, then start the computer.

9. Disable Apps and Check Battery Usage in Windows 10

Going back to Windows 10 a bit, recently, I noticed my Surface Pro was taking an unusually long time to charge and the fans started to rev up. Little did I notice the culprit was right there on my Taskbar all along. Slack's Windows client was waiting in an attempt to connect to the Internet. So, checking your apps to determine whats running in the background can help you not only conserve battery life but limit what prevents it from charging faster.

There are a couple ways to do that. Open Start > Settings > Privacy > Background apps. Scroll down then toggle off the apps that might be preventing your device from reaching a full charge.

Still in Settings, open System > Battery > Battery usage by app. This will tell you which apps are using the most battery and you can make a decision what to do, whether to let Windows decide if it should run in the background while not sacrificing battery life.

10. Replace Charger or Battery Module

Ultimately, if you don't see any improvements after trying all the above, then itâ€:tm:s maybe best that you replace either the charger or battery itself. This will require contacting your laptop manufacturer and placing an order. Batteries and the chargers are not necessarily cheap, especially if its an older system. But, if you have no choice, its something you will just have to do to get your system back in working order.

With DISKPART, is it possible to take Disk0 offline, make Disk1  the active system partition, and reassign drive C: to the 465GB NTFS partition now labeled as disk "I:"  ?   Perhaps not in that order?  If so, could you please give the commands in their proper order?

frederik long say:

With DISKPART, is it possible to take Disk0 offline, make Disk1  the active system partition, and reassign drive C: to the 465GB NTFS partition now labeled as disk "I:"  ?   Perhaps not in that order?  If so, could you please give the commands in their proper order?

With DISKPART, is it possible to take Disk0 offline

-> You need to tell the BIOS which disk is your primary boot disk. This might fix your incorrect drive letters.

make Disk1 the active system partition

-> You already have an active partition on Disk 1!

reassign drive C: to the 465GB NTFS partition now labeled as disk "I:" ?

-> You can do this through the registry editor. Navigate to 

HKLM\SYSTEM\MountedDevices

then rename the various DosDevices on the right-hand side to suit your preferences. Best to use snippingtool.exe beforehand to create a snapshot of the status quo. If a name is already taken, use a temporary name, e.g. like so:

rename DosDevices\C: to DosDevices GG (G: is the desired name)

rename DosDevices\G: to DosDevices DD (D: is the desired name)

rename DosDevices\GG to DosDevices\G: 

After rebooting and setting the primary boot disk, everything should be fine.
