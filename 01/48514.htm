1. If you have reserved through the Win10 App. Open the menu (top left). Click View Confirmation then cancel it.

2. Go to Windows\SoftwareDistribution\Download. Delete all files and folders in there (Do not delete the Downlaod folder itself).

4. Go to Explorer and open the drive with your current OS on it. You will hopefully see transparent folder named $Windows~BT and possibly one like $Windows~MS. Delete them.

5. Go to Windows update and select View Update History. Then select 'Troubleshoot problems installing updates'. This will download a troubleshooter, run it and it will check and fix Windows Update and clear out any residual problems(Hopefully all green ticks).

6. Get a USB Drive and download Win10 ISO from https://www.microsoft.com/en-gb/software-download/windows10

Select the 'Install on another computer' This will create the required media that you can then use anytime you want to try the upgrade. Simply put the USB drive in when in Windows and run setup.exe.

Hope that helps.

al h_100 say:

I was hoping it wouldn't do that to you! Ok. The system is trying to use the folder and so it is stuck with the permission until it finishes the infinite loop it is in. Run Step 5. and restart your PC and see if that clears the permission and lets you delete.

It isn't essential to delete the folder but as you can imagine a clean a drive as possible is best and the fresh install will need that folder.

If it doesn't delete after trying the above you will need to force a permission change to delete it. It happened to me on one of the many failed install attempts I had, and it took some playing around, but that is in Windows 8.1.

Try this. Right click on $Windows~BT. Click properties. Go to the security tab.

First off there should be an 'Edit' button. Click that and a new window pops up that should have an 'Add' button. Click it and another window pops up and you should see a blank box at the bottom with 'Enter the object name...' just above it.

In that blank box type Users and then click check names. It should auto-fill to the correct naming convention (Desktop/Users) or similar. Click ok. This takes you back to the previous window which lets you set the extent of the permission. Highlight Users and set it to full control.

Click ok. then ok again to close all the windows. Then again right click, go to properties ----> security. This time go to the 'Advanced' button at the bottom. This opens up another window and you should see near the top it say Owner: System and a clickable 'change' next to it. Click change then click cancel. At the bottom you will see a tick box saying about 'replace all child objects. Tick the box. Then click change again. Put in Users then check names as you did before. Again click through the OK's to close all the windows.

Again right click. Then properties. Click Advanced and remove any in the list that are not Desktop/Users. Click ok. Then click edit. Again remove any in the list that are not Desktop/Users. Click through the Ok's to close the windows.

The order in which these things are done may need changing and swapping. But the goal is to end up with one permission only, for Users on the first window and one permission only, for Users in the Advanced. With Users also being the Owner of the Folder and it's children (i.e. subfolders and files)

Now try delete the folder. Might need a restart but try first.
