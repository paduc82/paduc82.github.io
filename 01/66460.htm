Windows XP Recovery Commands

a. user say:

Dear Hwy,

Please find below the list of commands in order to boot from the recovery drive:

  1. Boot from a standard Windows XP Install CD (not a Network Install CD).
  2. At the first setup screen, hit R to repair Windows using the Recovery Console.
  3. Select the Windows installation that you would like to repair by entering its number, followed by Enter.
  4. At the Recovery Console, type each following commands followed by Enter:
    • CD..
    • ATTRIB -H C:\\boot.ini
    • ATTRIB -S C:\\boot.ini
    • ATTRIB -R C:\\boot.ini
    • del boot.ini
    • BOOTCFG /Rebuild
      • Set the Load Identifier to: Microsoft Windows XP Professional
      • Set the OS Load Options to: /noexecute=optin /fastdetect
    • CHKDSK /R
    • FIXBOOT
  5. Type exit to reboot the computer.

This will rebuild the OS back for you.

hyncharas say:

The reason why the Troubleshooting Guide for XB1 controllers doesn't work most times is because a very important step is missing.

The controllers themselves are wireless, and in order to reassign or update them this requires you to have a USB cable - HOWEVER, this MUST be done whilst no batteries/battery pack are attached.  The reason for this is, when on, the controllers are constatly trying to connect wirelessly, which disables connecting them with the cable... as a result, you're shooting yourself in the foot trying to fix them.

Microsoft really should update their guides to reflect this.

hairyfool say:

Instead of using Rufus use the Microsoft tool to build the USB drive:

https://www.microsoft.com/en-us/download/windows-usb-dvd-download-tool

Check the documentation or the BIOS boot screen for a boot meu option. It varies by manufacturer  but one of the F keys (F2, 8, 10, 12 have all been used) which lists all bootable devices found and allows you to select one. This is irrespective of what the boot order is in the BIOS and is sometimes referred to as the "one time boot menu" as the next time the computer boots it starts normally. It is NOT the Windows 10 dialogue but a BIOS generated menu.

Check the BIOS for Secure Boot and turn it off. Windows 8/10 certified computers have a mechanism which prevents changes to the boot sectors. In order to change the O/S it must be off (and remain off)
