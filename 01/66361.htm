evil capt kirk say:

I had the same issue after the 1709 update and I see other reports of this problem as well. This fix is pretty straightforward and it worked for me.

1. Log into your computer as an administrative user - this must be a different user account than the one affected. I was able to create a new administrator account via the windows GUI - if you don't currently have permissions to do that, you can enable the hidden built-in administrator by following the instructions here. 

2. Once logged in as administrator, you run the registry editor, locate the profile that contains the modified path (i.e. user.computername) and edit it to match the old desired path. Instructions start at step #6 here. The earlier steps with the command prompt stuff are not really necessary -  when running regedit you can just click on each item under ProfileList in the left pane and observe the value of ProfileImagePath in the right pane until you've located the item that needs to be changed.

Once you are done with #2, log out of the admin account and log back in to the affected account - you should see all of your icons, desktop, bookmarks etc restored.
