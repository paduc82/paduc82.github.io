Hi Ryan,

Based from the information that you've provided, your concern can be about a fatal error that occurred during the installation which may be caused by the following reasons:

  • Windows Installer is attempting to install an application package that is already installed on your PC.
  • The folder that you're trying to install the Windows Installer package is encrypted.
  • The drive that contains the folder that you're trying to install the Windows Installer package to is accessed as a substitute drive.
  • The SYSTEM account does not have Full Control permissions on the folder that you are trying to install the Windows Installer package to. You notice the error message because the Windows Installer service uses the SYSTEM account to install software.

In order for us to resolve your concern, we suggest that you perform the following methods:

Method 1: Check if the installation package is already installed on the PC. If so, uninstall and reinstall the app.

Check from your Programs and Features if the installation package is already installed on your computer. If yes, you may need to uninstall it first by following the steps below:

  1. Click Start Menu, type Control Panel.
  2. In Control Panel, click Uninstall a program.
  3. In Programs and Features, find the application package of the said driver. Select the package, right-click, and select Uninstall.
  4. Restart your computer to complete the process.

Method 2: Check if the downloaded file is not encrypted and doesn't have a blocked access reason.

  1. Right-click on the file and select Properties.
  2. Under the General tab, ensure that the attribute of the file is not on Read-only.
  3. Under the Attributes section that says Security, tick the Unblock checkbox.
  4. Click OK to apply the changes.

Method 3: Ensure that your installation package is saved on the Desktop.

Move the installation package from the download folder to your Desktop.

Method 4: Grant Full Control permissions to the SYSTEM account.

To grant Full Control permissions to the SYSTEM account, follow these steps:
  1. Open File Explorer (or Windows Explorer), right-click the drive that you want to install the Windows Installer package to (this time, select the driver letter where your operating system is installed, i.e. Drive C:), and then click  Properties.
  2. Click the Security tab. Verify that the Group or user names box contains the SYSTEM user account. If the SYSTEM user account doesn't appear in the box, follow these steps to add the SYSTEM account:
    1. Click Edit. If prompted, approve the User Account Control.
    2. Click Add. The Select Users or Groups dialog box appears.
    3. In the Enter the object names to select field, type SYSTEM, and then click Check names.
    4. Click OK.
  3. To change permissions, click Edit. If prompted, approve the User Account Control.
  4. Select the SYSTEM user account, and verify in the Permissions section that Full Control is set to Allow. If not, select the Allow check box.
  5. Close the Permissions dialog and return to the Properties dialog. Click Advanced.
  6. Select Change permissions. If prompted, approve the User Account Control.
  7. In the Permissions tab, select the SYSTEM entry and click Edit.
  8. Click the Applies to drop-down and select This folder, sub-folder and files. Click OK.
  9. Wait for the operating system to apply the permissions that you have selected to all child folders.
  10. Restart your computer and run the Windows Installer package.
