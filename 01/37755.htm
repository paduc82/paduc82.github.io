tryx3 say:

Skald,

I recommend getting all current Dell updates, including BIOS/firmware updates, as suggested above. 

You could then make a Ver 1703 installation disk [on another computer since you have no current internet connection on the target computer] and run that as a Repair install. 

  • Make the installation disk [USB/DVD] using http://windows.microsoft.com/en-us/windows-10/media-creation-tool-install to create an installation disk [DVD/USB] - it will include the Creator's update.
  • Then start your computer normally [do not boot from the installation USB].
  • Connect the installation USB.
  • Open File explorer and browse to the USB.
  • Run its setup.exe file then just respond to its prompts [there are some at the start & some at the end].
  • You will be asked for your permission to search for updates during the Repair install but it will keep things simpler to say no.
  • It will work its way through overwriting Windows files so whatever is causing any existing problems will probably be fixed.  The installation USB is a repair tool as well as an installation-update tool. 
  • You should backup your own files first but they ought not to be affected.  I have never had this procedure fail on me.
  • Your installed applications are not affected either.
  • You will need to run Windows update afterwards because there will be further updates that have been released since the Creator's update was released.

This procedure is called a Repair install or an InPlace upgrade.  It is a common way of updating to a new version and it is very reliable.  There are instructions in

A Repair install can fix [by overwriting] many Windows problems but not, in general, user account problems because it does not directly affect user accounts.

Speaking personally, I always make sure that I have an additional [spare, unused] password-protected is ever going to go wrong with your user accounts then it will go wrong for the accounts in routine use; unused accounts tend not to be corrupted. 

Denis

spiritx say:

Hi,

I recommend the Hidden Admin Account be disabled and only used if needed as last resort
since it is. That is the reason it is hidden. 

Make another ADMIN account with your same Password to test and fix. Always keep a spare
ADMIN account that is not used on a regular basis.

Disable the Hidden Account unless needed.

Microsoft recommends using a User Account and saving the ADMIN you make only when
needed for security reasons. However using a User Account on a regular basis is to
restricting for most of us. Hence have two ADMIN accounts one to use and one to test
and fix. Either way leaving the Hidden Account for emergencies.
