Options for freeing up space:
Windows over time will accumulate data outside of the normal types of data we store. This can include things like copies of Windows Updates, memory dump files used to diagnose and troubleshoot error messages, temporary internet files for web pages we often visit and Windows Upgrade files, commonly used to assist with installing updates for Windows. Sometimes we don't need this type of information and removing it can free up precious disk space inaddition to improving overall system performace.
Having enough disk space is especially important if you are performing task such as upgrading to the new Windows 10. It is one of the the first things Windows setup checks before it starts copying files. Large programs too will often require that you have enough disk space available.
Lets start.
Launch Disk Cleanup, the quickest way is this:
Press Windows key + R then type: cleanmgr.exe
Then hit Enter
Windows 10 > Start > All Apps > Windows Accessories > System Tools > Disk Cleanup
Once Disk cleanup opens, select the drive where Windows is installed if its not already.
Windows will scan your hard disk for files it can clean up.
Note some of these files too can include older installations of Windows. For example, if you upgraded from a previous version of Windows 10, a copy will be kept at the root of the hard disk in a folder called Windows.old. Please becareful though, this folder can actually be useful if you need to go back to your previous version of Windows for whatever reason. So be mindful before deleting it. Removing it though can significantly free up disk space.
After Disk Cleanup is finished scanning, you will see the Disk Cleanup window. This window provides a list of areas of the system you can clean up safely. Before we start cleaning up the system, you will notice appears on the window called Clean up system files. Clean it and the Disk Cleanup will run a thorough check of the system again to include every file that can be removed.

w_r_h say:
I ran into the problem last night where I couldn't install, reinstall, or update any apps through the Windows Store.  I am still running Windows 8 and not 8.1 for known reasons but that's for another place.

This problem took me ALLL day to research and solve, and I mean about 8 solid hours.  The error codes I was getting from the windows store kept pointing to Windows Update problems or other system corruptions, but after finding all of those solutions and running the associated checks several times nothing was wrong with my Windows Update.  I should have guessed considering how I could still download updates from MS just fine.

What solved MY problem (and I'll bet most of yours as well), was in fact a folder permissions problem.  There are TWO, count them 2 folders that you need to be sure your user has Full Permissions for.  I saw fixes out there that talk about one or the other, but you need BOTH.  These are the file path's to those folders: (keep in mind "C" is my root drive, <user name> is the name of each user on the system, and if you customized your windows installation, or user folder location, you will need to alter these pathway's to match)

C:\Program Files\WindowsApps

C:\Users\<user name>\AppData\Local\Packages

Before Following this procedure, go to the Windows Store and cancel any pending (or failed) installations.

To set the permissions:  (if at any point the system asks for Administrator permission you must grant it)
1. Make sure you are logged-in with an Administrator account.
2. Open File Explorer and navigate to C:\Program FIles.
3. Locate the WindowsApps folder (all the way at the bottom)
     a. NOTE: this is a Hidden Folder so be sure you go to the top of the window and click the "View" tab, then click the check-box next to "Hidden items" in the Show/hide section.
4. Right-click on the WindowsApps folder and choose "Properties".
5. Click on the "Security" tab.
6. Click the "Advanced" button.
7.1 If your user is listed in the list under Principal double-click on it (otherwise skip to 7.2)
  7.1a- Ensure the Type: is "Allow", and the Applies to: is "This Folder, subfolders and files"
  7.1b- Click the check-box for Full control
  7.1.c- Click OK
7.2 If your user is not listed, click the "Add" button beneath the list (you might need to click "Change" for the "Add" button to appear)
  7.2a- Click "Select a Principal" link at the top.
  7.2b- Click "Advanced", then "Find Now"
  7.2c- Find your username in the list and double-click on it
  7.2d- Click OK
  7.2e- Ensure the Type: is "Allow", and the Applies to: is "This Folder, subfolders and files"
  7.2f- Click the check-box for Full control
  7.2.g- Click OK
8.  You should now see your user name in the list of "Advanced Security Settings for WindowsApps" with "Full control" under the Access column (If you don't, go back to 7.1 or 7.2 and carefully follow the steps again).  Click OK
9. Navigate to C:\Users\<user name>\AppData\Local
  a. NOTE: AppData is also a hidden folder so be sure you have the "Hidden items" check-box checked
10. Right-click on the WindowsApps folder and choose "Properties".
11. Click on the "Security" tab.
12. Click the "Advanced" button.
13. Follow the same procedure in steps 7.1 or 7.2 to add your user to this folder's permissions list with Full control.
14. Restart your computer.

That's it, once your computer restarts you should be able to once-again use the Windows Store to install apps and updates.

Once you confirm this works for you I recommend creating a system image of your main Windows volume.  All of this happened to me while I was trying to get everything set up initially, but once you do, using the image as a good quick restart from a Known Good Configuration is a MUCH faster way to solve many problems with Windows... EVEN A WAY TO GO BACK TO Windows 8 from Windows 8.1 SHOULD YOU FIND THE NEED; I KNOW I DID!

Hope that helps everyone!
