Power plan is ignored once a month - Microsoft Community

technonath say:

Power plan is ignored once a month

I wanted to highlight an issue that I have finally solved with win 8.1 seemingly randomly ignoring the power plan.

I have come up with a workaround but I thought I would document it here to help others.

This is my explanation of why machines with "power off the screen" set to never in their power plan, (such as information display screens) once a month are going in to screen power off mode.

An examination of  the event log is showed the following two events:-


These two events are showing that the power scheme is being changed from the current plan where the screen is set to never power off, to the "High performance" power plan, which by default "Turn off the display" set to 15 minutes of inactivity.

And as the machine has not had any keyboard or mouse activity for 15 minutes, as soon as this power plan changes - it powers the screen off.

So which task is doing this?

A bit of detective work reveals that it is the "Working set swap assessment maintenance task", although it doesn't show as being scheduled monthly - but the time matched up with the event log and if I force this task to run and the machine has been idle for longer than 15 minutes - the screen will go immediately blank.


So how do we fix it ?

I don't know how to stop the Working set swap assessment maintenance task as it appears it is not actually scheduled to run again!

So the answer is to modify the "High performance" power plan so that it too has "Turn off the display" set to never.
