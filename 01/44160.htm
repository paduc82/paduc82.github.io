eddieb- say: Hi MarcDB_978,

the problems generally lie with third party software, or faulty hardware. You can use the following general guidelines when dealing with a system freeze; it is largely a matter of trial-and-error.

Consider running chkdsk on all partitions. Let chkdsk complete on each partition and see if that helps.

Also consider running SFC /SCANNOW.

Eliminate as much unnecessary hardware and software from the system and see if you can stabilize the system. Slowly add things back to the system noting when what has changed when/if the behavior returns.

If you boot into safe mode, do you still experience freezes? What about if you do a clean boot, or device clean boot?

It can be helpful to use Driver Verifier's deadlock detection. To enable Driver Verifier... start->verifier.exe->OK->Create standard settings- >Next->select driver names from a list->Next->sort by Provider->select all non-Microsoft drivers->Finish, and OK your way out of the dialog.

Then, reboot and use the system as you normally would, and wait for a problem. In the event that the system does not boot completely after enabling driver verifier, boot into Safe Mode and run driver verifier, and tell it to delete the changes.

Wait for a bugcheck to occur after enabling verifier as described (verifier will bugcheck the system if it detects a deadlock condition in this case), and then upload it to your SkyDrive, and provide a link.

Consider testing memory with Windows Memory Diagnostic or memtest86. Note that memory that passes tests is not necessarily good memory - it just hasn't failed a test. Consider systematically eliminating RAM from the system - run with a couple of modules for a while, and see how things go. Then try the other modules.

Other common suggestions include ensuring drivers are up to date (including video drivers), as well as ensuring that you're using the latest BIOS.

You can also consider trying to create a manual crash dump and analyze that.

Hello, I just did this again...

Some questions:

-is this mach running now?

-which OS is running now?

For me, I installed 2ea 240gb SSDs in RAID0 for my primary drive (it gave me a 447gb volume to work with), then I installed my Win 7 Pro x64 bit OS, got everything running, all updates, etc...

I also installed 2ea 3tb mechanical drives in RAID0, but then I split them into two GPT volumes D: E: for storage... Note: any volume over 2tb must be GPT, but 2tb and under can be MBR, it's up to you what you wish to do, each volume will have a drive letter assigned...

After everything was running correctly and all updates installed, I used Disc Manager (right click my computer>select manage, then click Disk Management (let it load your disk configuration). If you only have one disk, there will only be one entry showing your C: drive as your Boot drive partition, there should also be a reserved 100mb partition in front of the C: partition and a 450mb restore partition at the end of the drives total volume (don't do anything with these two partitions).

Click on the C: partition to highlight it, then right click and select "shrink volume"... At this point you will need to do some calculations/decide what you want to do... I split my 447gb in half, so that each OS has the same size volume, with the other smaller partitions already in place, the calculations were not equal, but close... Right click and format this new volume...

If you're doing this from scratch, you can use the "Custom install" option in Win 7 to partition the drive and format it (I named the partitions Win 7 and Win 10, so I could tell them apart later).
