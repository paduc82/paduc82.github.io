Windows Media Center crashes under Remote Desktop Connection after - Microsoft Community

sanslos say:

Windows Media Center crashes under Remote Desktop Connection after installing KB2592687

When viewing another computer (Win7 Pro to Win7 Pro) via Remote Desktop Connection, this update (KB2592687) causes Windows Media Center to either not start or immediately exit, in either case briefly showing a popup which reads:

  Windows Media Center has stopped working

Both computers have KB259687 installed.  WMC works correctly when logging directly into either machine.  If KB259687 is uninstalled on the machine being connected remotely via Remote Desktop Connection, you can once again run WMC in the remote window.  Note the video is not available, but you are able to check existing recordings, change schedule, view the TV grid, etc. (just like before).  Recordings scheduled on the PCs work correctly, I did have one system crash (Blue Screen) apparently due to this.

  Note that there is no problem viewing WMC when using Remote Desktop Connection to a Win8 Pro machine when logged into a Win7 Pro machine.  WMC is installed on the Win8 Pro machine, and is controllable from inside the Remote Desktop Connection window on the Win7 Pro machine.
georgef_msft say:

I had the same problem, but uninstalling the Update did not fix it for me.   I investigated and found that the reg key permissions for HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Control\Terminal Server\WinStations\Rdp-Tcp key did not include any permissions for the Users group, as the other subkeys did.  Perhaps the RDP Update noted above modified these permissions?  Anyway, when I added read permissions for this key for Users, WMC shell once again would start inside a RDP session.  I also found that running the ehshell "as Administrator" would work under RDP, but that was not necessary in the past.
