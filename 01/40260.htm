zacabeb say:

Well,

1. The output scaling in the Xbox 360 is done on-the-fly, and as far as I know it does not cause any substantial lag.

2. Most games render natively for an output buffer with a resolution of 1280x720. There are variations however, with many games having been rendered at various custom resolutions for memory and performance reasons. They may not map directly to a standardized output resolution. Examples of odd resolutions include early games such as Ridge Racer 6 (1440x810) and Project Gotham Racing 3 (1024x600), later games such as Halo 3 (1152x640), and some Xbox Live Arcade conversions of classic games such as Sonic Adventure and Crazy Taxi (both at 888x666).

In some cases the rendering resolution is scaled in software to 1280x720 for overlay of HUD and frontend graphics, in some cases the original resolution is kept as-is. The result is then scaled to whatever the output is set to. Lighting, shading, and textures are unaffected.

Games can however handle multiple output resolutions seperately and render in a resolution which matches the output. This was used in early games for rendering standard definition output at 640x480 or 853x480. The Xbox 360 Guide is also rendered at 853x480 regardless of output resolution.

3. There has only been a handful of retail titles for the Xbox 360 that render natively to 1920x1080. Some Xbox Live Arcade games do, especially conversions of classic 2D games where the original low resolution such as 320x224 is mapped into a 1920x1080 buffer to retain edge sharpness.

Some conversions of classic 3D games do run in 1920x1080 though, such as Daytona USA, OutRun Online Arcade (also supports 640x480 and 1024x768 natively), Banjo-Kazooie, and Banjo-Tooie. A few original Xbox Live Arcade games also render to 1920x1080, such as Sonic the Hedgehog 4: Episode 1 (but not Episode 2).

If you are currently receiving the error "A problem occurred trying to open this message. A problem has occurred. Please try again ", it possible that the MsMessageStore file of Windows Live Mail is corrupted. As primary troubleshooting, please follow repair it by following these steps:

 

1. Restart the computer (this is to ensure that Live Mail is fully shut down, and that none of it's components are in use when the computer starts).

 

2. Click Start, type CMD but do not press enter.

 

3. You will see cmd.exe in the search results, right click on it and select Run as Administrator - type your admin password if asked, and if UAC pops up asking premission, allow it.

 

4. You should now see the command prompt, with the following displayed:

Microsoft Windows [Version 6.1.7600]

Copyright (c) 2009 Microsoft Corporation. All rights reserved.

C:\Windows\system32>

 

5. Copy the following (Ctrl+C)

esentutl.exe -d "%userprofile%\AppData\Local\Microsoft\Windows Live Mail\Mail.MSMessageStore"

Note: the "" are necessary

 

6. Right click in the command prompt Window, and select Paste, so you now see the following:

Microsoft Windows [Version 6.1.7600]

Copyright (c) 2009 Microsoft Corporation. All rights reserved.

C:\Windows\system32>esentutl.exe -d "%userprofile%\AppData\Local\Microsoft\Windows Live Mail\Mail.MSMessageStore"

 

7. Press enter. esentutl will now defragment the Live Mail database file.

 

8. Once done, you can close the command prompt, and restart the PC again.

 

If the issue still persists, please let us know.

 

Thank you.
