If you are trying to enter the BIOS and get asked for a password, somebody before you put a BIOS password on the machine (probably from a computer repair shop).  A BIOS password is optional on many system BIOS.

The (some) repair shops like to put a BIOS password on your system even if your machine comes into the shop without a BIOS password.  This way, you are almost forced to bring the system back to them if you need help later.  When you can't figure it out - you take it back.   They like that!  Of course they put the password on only toprotect you.  If you call and ask what it is, they are either out of business or claim to have no idea what you are talking about and please bring it in... or you can just fix it yourself.

If the BIOS supports a password, you can usually reset/clear it using a jumper on the motherboard and sometimes by removing the motherboard battery for a few minutes.  This will clean the password and sometimes set the BIOS back to the defaults, so you will have to go through the BIOS configuration again (maybe).  There may be a jumper just to clear the password only.

The complexity or ease of the process depends of your system make and model, so tell us what it is.  Chances are there is a manual you can download to read about such things.

If you are not sure, pay attention to and report back any screens with text on them and what they say when you boot up that may reveal the system make and model, the motherboard type, the BIOS etc.

a. user say:

When you boot in Safe Mode and the last thing you see on the screen is mup.sys there are usually only a couple possibilities that I know of.

First of all, despite what people and "experts" tell you about about replacing the mup.sys file. there is probably nothing wrong with the mup.sys file for it has already loaded.  The problem is what comes after mup.sys.

Sometimes the problem can be an afflicted USB device like a mouse, keyboard, external HDD, CD/DVD drive, so if you have any USB devices connected unplug them all temporarily to see how things look.  If your keyboard is USB, see if you can use a wired PS2 keyboard instead.  The only thing you need to boot your system is a keyboard so get yourself as simple a hardware configuration as you can then if things start to work, add the other devices back one at a time until you isolate the culprit.

The next thing that happens after mup.sys loads (and checks your USB devices) is XP is going to check the dirty bit on the HDD volumes to see if XP needs to run a chkdsk with error correction on any of them or if there is a chkdsk with error correction that has been scheduled to run on the next reboot.

The dirty bit on a HDD can get set if there has been a power interruption, aborted restart, ungraceful shutdown, using the power switch/button, plug pulling, battery removal, etc.

Has any of that occurred prior to this incident?

When XP boots it automatically checks the dirty bit on all volumes and if it finds a dirty volume, it will want to run a chkdsk with error correction to try to fix it.  When you boot in Safe Mode, that check occurs shortly after mup.sys loads.

If XP finds the dirty bit set, it is going to run a chkdsk with error correction and the last thing you will see on the screen is mup.sys then it will appear that your system is hung on mup.sys - but that has already loaded mup.sys and is way past that.

If the chkdsk with error correction is running either because the drive is dirty or you scheduled a chkdsk with error correction to run on the next reboot, you may be able to tell if your HDD access light is blinking or you can hear the HDD "working".

A chkdsk with error correction can take a long time (perhaps many hours) depending on the speed of your system, size of the volume(s), the amount of data on them and what the chkdsk finds to do and in Safe Mode you can't see what it is doing so you just have to wait it out.  If you restart before it finishes, you are just going to get stuck again.

An XP repair will probably not help you since you likely have a hardware issue but you can sure try if you have a genuine bootable XP installation CD that is the same Service Pack as your unspecified installed Service Pack (most people don't).
