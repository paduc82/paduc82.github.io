a. user say:

XP is supposed to create a new Restore Point (System Checkpoint) every 86,400 seconds (24 hours).

Software Distribution 3.0 RPs are most likely because you have Microsoft Security Essentials installed.

If MSE updates every day, before the MSE updates are installed a new RP will be created called Software Distribution Service 3.0. so that explains why you might see one of those every day - or more often if MSE updates multiple times a day.  Those are normal.

SDS 3.0 RPs will also be created prior to installing any downloaded MS updates.

In spite of what the alleged "experts" have to say, there is no such thing as a "Software Distribution Service" to uninstall or disable that will keep SDS 3.0 RPs from being created (especially if MSE is installed).

If XP sees that a RP was already created in the last 24 hours (probably from a MSE update) it will not automatically create a new System Checkpoint since a RP has already been created in the last 24 hours.  So if MSE updates every day, you will probably only see SDS 3.0 RPs in the RP calendar and that does not mean there is a problem - it makes perfect sense.

If there is a need to create a System Checkpoint they will only be created when the system is "idle" and idle means means no mouse, no keyboard, no surfing, no scanning, no instant messenger, no Skype, no movies, no Magicjack, no XP Indexing Service, etc.  Anything that uses 10% of the CPU even for a split second can prevent RPs from being created "naturally".  It could be some third party background process or service you have installed that prevents idleness.

There is not enough information about your system to provide anything other than guesses about why your System Restore might not behave.

What is your system make and model?

What is your XP Version and Service Pack?

When System Restore seems to be misbehaving it is usually because of some third party software installed on your system like antivirus/antimalware applications that are "protecting" your system from modification.

It is easy to reinstall System Restore and here's how:

Sometimes System Restore is so afflicted and none of your Restore Points will work or maybe you don't trust any of them (if your AV software reports they contain infections) it is sometimes best to just start all over and reinstall System Restore.  That will get you a fresh start.

When you reinstall System Restore, all of your current Restore Points will be removed and you must restart your computer when the installation is complete.  After the restart, one new Restore Point will automatically be created as a new System Checkpoint in your SR calendar.

If your system had a malicious software infection, it could be that some of the files in your old Restore Points are also infected so wiping out the old ones and starting fresh is often a good idea so you don't wonder if the old Restore Points are infected or not.

As you go through the reinstallation of SR, the reinstallation will ask you for some files as you go along and ask for your XP installation CD.  If you have a genuine bootable XP installation CD that is the same Service Pack as your installed SP, you can use it but on the planet called Earth, most people don't.

If you do not have a genuine bootable XP installation CD that is the same Service Pack as your installed Service Pack (most people don't) , you can change the path of where the installer looks for files one at a time as they are needed.  More on that later.

Assuming your XP is installed in the Windows folder on the C drive of your machine:

Browse to c:\windows\inf and right click the following file:

sr.inf

Choose Install

If you cannot see c:\windows\inf folder or the sr.inf file therein, configure your system to show hidden files and to show file extensions.

Here's how:

In Windows Explorer, click Tools, Folder Options, View and enable "Show hidden files and folders" and uncheck "Hide file extensions for known file types" then you should be able to see the c:\windows\inf\sr.inf file.  You can undo these adjustments later if it is your desire.

The installation may ask for files from the c:\windows\inf\386 folder.  That would be your genuine bootable XP installation CD if you have one and the Service Pack installed on your system needs to be the same as the CD you are going to use.  If you don't have that (most people don't), not a problem.

You do not absolutely need a genuine bootable XP installation CD to reinstall SR since the files must already be on your system some place if SR was ever working before.

If you are asked for a file and you can't find it where I suggest in my example below (it works on my system), you will have to search your hard disk and find where the files are on your system.

If asked for a file and you don't know where it is, leave the installation waiting and click Start, Search and search for the requested file and find it on your system.  If SR ever worked before, the file is already on your system somewhere.

When Search finds the requested file, switch back to the installation and enter the path to where you found the requested file and click OK.  The installation may ask for several files along the way, but you can find them all.

If you still can't find the files, let us know what the file is you need and some nice person can upload it to their OneDrive and you can download it.

what is notepad? - Microsoft Community

the shelly say:

what is notepad?

what is notepad? when i saw some files that looked out of place i tried to open them and i got notepad. which you know you cant read what is it and why is there so many files in it

a. user say:

Notepad is just a simple .txt file reader that also reads some other file types as well.  Sometimes it is set as the default option for file types that the system doesn't know to which program to assign a default and those files are frequently gibberish because Notepad really isn't the right program (and the user needs to find, download, and install a program that can read and open the file type of the file they are working with).  Some system file types can't be read but the system typically defaults them to Notepad when it has no program installed to read or open those file types.  When you come across a file type you can't read and it opens in Notepad and is gibberish, enter the file type into Bing in your internet browser and check out the articles - there will be several which will offer to explain the extension and they usually provide a list of programs that can create or open or work with such file types.

To change the default program that opens with a particular file type go to Start / Deffault Programs / Associate a File or Protocol with a program and scroll to that file type and click on change to see if it presents any alternatives.  If not, you can always click on browse and search your Program Files for the executable file of program that you want to use as the new default program - if you have one installed that can read or work with that file type.

I hope this helps.

Good luck!
