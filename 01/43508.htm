I wish I could tell you why you're having no luck using the Alt codes for the old ANSI Code Page 1252, which is where those keyboard Alt-commands derive from, a relic of Windows from the 80s and 90s. Rather than fight the issue though, perhaps you might care to simply make the switch to Unicode and its keyboard commands which have actually been the de facto standard for keyboard inputs for 15 years at least? It's really not as bad as you think, trust me.

For the following instructions, boldface type indicates an entry or selection to make, not that the entry or selection itself needs to be bold when you perform it. I prefer this to the use of quotation marks to indicate inputs, which makes my pupils turn into spirals like a hypnotized cartoon character from the inherent ambiguity)

We need to add one simple registry value to Windows, so press the Windows Key + R and type regedit then hit Enter, agreeing to the UAC prompt for Administrator privileges. In the key tree that is shown in the Registry Editor's left panel, expand the following hierarchical keys (they look like folders): HKEY_CURRENT_USER, Control Panel and finally Input Method. Now right-click in the right panel and click on New -> String Value. Give it the name EnableHexNumpad and click OK or press Enter on the keyboard. All that's left to do is turn it on, which is accomplished by giving it the value of 1, so double-click on it or just press Enter on the keyboard since it should already be highlighted, enter 1 and say OK. You're done, so close the Registry Editor and either restart the computer or just log out of your account then log back in and the change will be in effect.

Now you are able to enter the modern Unicode equivalents to the old ANSI CP-1252 Alt-codes. In the case of your need for the lowercase Greek pi, the value for it is 3C0. The only wrinkle is that once you've depressed the left Alt key on your keyboard, all the Unicode values start with the plus key. In short, for the œÄ character, the command is to hold the left Alt key, then in sequence press, +, 3, C and 0. The plus and the numerals are always entered from the NumPad. Congratulations, you can now say that you can use hexadecimal notation to input Unicode.

All the Unicode values are shown in the status bar of the Windows Character Map application in the format U+hexcode. The U indicates holding the left Alt key, and the rest is tapped in sequence just as you did above, with leading zeros optional, which is why I indicated 3C0 and not 03C0.

The reason I asked is that *.mp3 IS a "partial file name".

If the file in question is an mp3 (music) file, I doubt whether you would ever find it by searching for "content in the file," so the relevant failures are why it wasn't found using the full file name or another part of the name.

If the file isn't hidden, the only other thing I can think of is that there is some special character in the file name that isn't obvious when you look at it.

For example, there was a pretty big "Philly soul" hit in 1974 called "When Will I See You Again," by the Three Degrees. Suppose you decided to name an mp3 file of this 3¬∞.mp3. That would be a legal file name, but a search for 3¬∫.mp3 won't find it. That's because in the name of the file I used the "degree symbol" (Unicode 00B0) and the search string uses the "masculine ordinal symbol" (Unicode 00BA). These symbols look almost identical to the nekkid  eye(can't use the real word here), but not to a computer search engine.

Replacing the entire part of the file name except the extension by using the * wildcard would make the search work, but if the "partial" name you used includes whatever this character is, that might cause the problem.
