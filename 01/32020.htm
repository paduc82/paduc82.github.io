If a message received in Live Mail on a Windows 7 Home Premium 64 bit machine contains a PDF file attachment, that attachment cannot be opened from Live Mail. Word file attachments and others can be opened.

Thanks

Ron Carruthers

a. user say:

I had this same problem, tried Bruce Hagen's idea and it didn't work. So I tried to find out how to reset a file association by using Google and somebody, in reply to a whole different issue, pointed at the Windows registry, more specifically the HKEY_CLASSES_ROOT section. 

So I took the .pdf subkey in HKEY_CLASSES_ROOT, made a backup (export) and simply deleted it. Then I went on to associate a random PDF file to Adobe Reader again and guess what: 

Now it again works in Windows Live Mail!!! Yoohoo!

Looking at the key afterwards I also saw the contents of it had changed, for instance now it says a Content Type and in 'standard' there is "AcroExch.Document", just as came up with "assoc .pdf" in the command window.

This makes me guess that Windows 7 is perhaps a little 'sloppy' when reassociating a file type, because this happened to me after I removed an old version of Adobe Reader (Adobe Acrobat Reader 6) and installed the latest version. 

I really hope this will resolve the issues others have here! It really saves me a lot of frustration here! ;)

Create a file association.

  1.     Download and save the attachment on the desktop.
  2.     Right click on the saved file.
  3.     Select: Open With from the pop-up menu.
  4.     Click: Choose [default] Program.
  5.     Select: Adobe Reader (or the PDF viewer you use)
  6.     Check: Always use the selected program to open this kind of file.
  7.     Click OK.

This should enable you to open these attachments directly from the e-mail now.

Windows Live Mail stores two files in Temporary Internet Files for every message that's viewed in the program. They are not removed when the program has finished with them, so you can end up with hundreds of thousands of them, occupying gigabytes of disk space. When you get up to the half-million mark, WLMail starts having problems displaying messages. Ron's advice is intended to remove this possible cause of your problem. Since you say it didn't help, I've unmarked Ron's post as the answer.

The problem only occurs with some messages, so there is something about the affected messages themselves that's causing the problem.

The problem only occurs on one machine, so there is something about that machine that's causing the problem. 

Finding that combination won't necessarily be easy, The only analogous case I can remember is where affected messages included a font attribute specifying Helvetica (or variant). If a Helvetica Type 1 font was installed on the machine, affected messages wouldn't display. This happened, I think, only after installing IE9.

  • Which OS is installed on the machine that has the problem?
  • Which OS is installed on the machines that don't have the problem?
  • Which version of IE do you have installed on the problem machine? And on the others?
  • What are the KB numbers of the updates that you think might have triggered this behaviour?
    If KB2817183 is among them, does uninstalling it in Programs and Features > Installed updates make the problem go away?

keith a. miller say:

Sorry for the delay in responding. Had a health scare with that handsome dog in my profile picture.

I'm assuming your User folder is still displaying in Explorer as 3D Objects.

Open an elevated Command Prompt. Type the following commands:

> cd \users

> dir

You should see the real name of your user folder, which I'll refer to as <Username>

> attrib -r <Username> /d

> cd <Username>

> attrib -h -s desktop.ini

Now switch to Explorer and move the desktop.ini file from C:\Users\<Username> to D:\Andrew\3D Objects.

Now back to your Command Prompt:

> D:

> cd Andrew

> attrib +r "3D Objects" /d

> cd "3D Objects"

> attrib +h +s desktop.ini

This should both restore the display of your User folder name as well as give your new 3D objects folder its custom icon.
