a. user say:

Hi Paul,

The VSSVC.exe is the file name for the Volume Shadow Copy service which is a set of COM interfaces that implements a framework to allow volume backups to be performed while applications on a system continue to write to the volumes. This service is used for creating backup of files.

You have mentioned that you see this process in task manager every time the system is about to get locked up. What exactly do you mean when you say the computer locks up for 15 to 20 minutes? Does the computer freezes? To troubleshoot this issue, I would suggest you to try the following steps and check:

First, I would suggest you to boot the computer in a clean boot state and check if the issue persists. If the issue doesn’t persists in a clean boot state, then it means that one of the third party software that was disabled during the clean boot procedure was causing the conflict. Refer to this article for more information on clean boot procedure:

http://support.microsoft.com/kb/929135

 

Note: If the issue didn’t occur in clean boot, refer to the ‘More information section’ in the article find information about determining which application is causing the issue. After troubleshooting, reset the computer to start as usual. Refer to the step How to set the computer to start as usual after clean boot troubleshooting under more information.

If the issue occurred in clean boot, then I would suggest you to check the Event viewer for more information on what is causing the issue. The Event viewer holds a record of all system events. Checking these logs will help us determine the root cause of the issue. Refer to these articles:

http://windows.microsoft.com/en-us/windows/open-event-viewer#1TC=windows-7

http://windows.microsoft.com/en-us/windows/what-information-event-logs-event-viewer#1TC=windows-7

You can also narrow down the logs by looking for logs filed around the time the issue occurred last. When you find the relevant logs, copy the contents and post them in your reply.

  3. On the run window, type in %appdata%\Skype and hit enter. Look for a file named shared or shared.xml and delete it.

  4. Open the folder that has your Skype name.

  5. Locate and delete the config or config.xml

  6. Open Control Panel and look for Internet Options.

  7. Go to the advanced tab and select Reset.

  8. Put a check mark under "Delete Personal Settings", then hit okay and enter.

  9. Close it when prompted.

  10. Restart Skype.

Once you are done with all those steps, try to sign out again from your Skype account and ask some of your Skype contacts if there is now changed to your Skype status.
