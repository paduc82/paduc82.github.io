ken blake say:

It sounds like you are running 32-bit Windows.

All 32-bit client versions of Windows (not just XP/Vista/7/8/10) have a  4GB address space (64-bit versions can use much more). That's the theoretical upper limit beyond which you can not go.

But you can't use the entire  address space. Even though you have a  4GB address space, most people can only use *around* 3.1GB of RAM. That's because some of that space is used by hardware and is not available to the operating system and applications. The amount you can
use varies, depending on what hardware you have installed, but can range from as little as 2GB to as much as 3.5GB. It's usually around 3.1GB.


Note that the hardware is using the address *space*, not the actual RAM itself. If you have a greater amount of RAM, the rest of the RAM goes unused because there is no address space to map it to.

  • The PC boots into Windows RE (Recovery Environment).

Method 2: There are situations where you need to reset the Recycle Bin store folder to fix problems such as Recycle Bin not showing the files that have been deleted, or you are unable to completely empty the Recycle Bin. Each drive has a hidden and protected folder named $Recycle.bin, which is where the Recycle Bin stores files and folders that you (and other users in the PC) delete. IMPORTANT: The following procedure permanently deletes all files and folders that are already in the Recycle Bin (from all user accounts in the system) and cannot be recovered.

(This clears out the $Recycle.bin folder from the C:\ partition. You may have to do this for each hard disk partition in your system.)

c. Type exit to close the Command Prompt window.
