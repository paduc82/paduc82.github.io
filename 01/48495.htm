lemp say:

You didn't answer the most important question:  How is the printer connected (USB, wi-fi, something else (please describe)?

There are several assumptions in the comments below.  If any of them are incorrect, please let me know what the actual facts are.  Also please don't forget to supply the information requested in Step 5.

From your description, my guess is that the printer is connected via wi-fi.  If this is the case, the fix is exceedingly simple.  The following assumes that the printer is connected via wi-fi.

My old Windows XP system has died, so this is from memory ...

1.  Open Printers and Faxes.  Are there two icons that refer to an Officejet Pro 6830 or only one?  The following assumes that there is only one HP 6830 icon.

2.  Right-click on the icon for the HP 6830 and select "Properties"

3.  Select the "Ports" tab.  There should be one line that is checked and highlighted.  What is shown in the "Port" column?  You may have to click and drag on the faint vertical line between "Port" and "Description" in order to see the full width of the Port column (this image is from Windows 7, but your XP dialog should look very similar)

4.  Leave that window open and go to the printer.  On the printer's touch panel display, touch the wireless icon.  This is supposed to display the "Network Status Screen."  I don't know what is included here, but if it shows the printer's IP Address, make a note of that.  Otherwise, select "Print Details."  This should print a network configuration page on which you should find the printer's IP address.

5.  Please report in your Reply here both the printer's IP address from Step 4 and the Port IP address from Step 3.  If these two IP addresses are not identical, the computer will not be able to send print jobs to the printer.

6.  These two IP address will almost certainly look either like 192.168.1.xxx or 192.168.0.xxx (where xxx can be anything between 2 and 254).  Even if the IP addresses are not identical, they should be the same except for the "xxx" parts.  If they are not, you have a more serious problem.

7.  Assuming that step 5 revealed that you had non-identical IP addresses in the two locations, the quick and semi-permanent fix is to make the Port IP address on the computer the same as the IP address assigned to the printer.  Go back to the printer's properties dialog and click on the "Add Port" button.  On the next screen, select "Standard TCP/IP Port" and click "New Port."

8.  The "Add Standard TCP/IP Port Wizard" should start.  Click Next.  Enter the printer's IP address from Step 4 and complete the Wizard.

9.  Back at the "Ports" tab, you should now see another line with the new port that you just created.  UNcheck the line that was for the old port and check the line for the new port so that the only line that is checked has the IP address that is the same as the printer's IP address from Step 4.

10.  OK your way out.  You probably don't have to restart the computer, but if you can't print (try opening Notepad and printing a test page), restart the computer and try again.

vibroplex say:

I would instead suggest a house-wide, all Xbox sequenced power cycle.  Some NAT problems are subtle and don't appear in the Test Xbox Live Connection test. Most likely there's a port blocked or misdirected away from your Xbox, in your router, modem, or at your ISP.

Since the Xbox 360 uses different ports than PCs & Macs ordinarily use, you can't see the effect of a port blockage except on the Xbox, for the NAT problem is really the blockage or misdirection of ports. A sequential power cycle is the first step in clearing connection problems, and the order and timing of the steps is important to success:

 

1) turn off all the Xboxen then

2) unplug power from the router

3) unplug power from the modem.

4) Wait at least 5 minutes then replug the modem power.

5) Wait 'til the light on the front of the modem which says BROADBAND, ONLINE, READY, CABLE or DSL is on solid (not flashing), then plug in the router.

6) Wait until the WiFi light on the router is on; you may not use WiFi, but it's still a good indicator your router is finished booting up.

7) Turn on the problematic Xbox & run the Test Xbox LIVE Connection.  

8) Power up the next Xbox and test it.

9) Repeat with remaining Xboxen.

10) Go back to the problematic Xbox and re-run the Test Xbox Live Connection test. Is it OK? Please let us know.

 

For more background on Ports, see whatismyipaddress.com/port
