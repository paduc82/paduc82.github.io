cmdrkeene say:

Right-click causing issues is usually caused by bad extensions (the extra options you see on right-click menus like "scan this file now" or "open with 7-Zip").  These extra menu options are all integrated with other programs, and if your rightclick causing a problem, you know one of these extensions has the issue.  It could be onedrive, it could be something else.  It could even be something that's only a problem if you happen to have onedrive PLUS that other thing that only cause problems when they are together.  There's no way for me or anyone here to know for sure, but that's what the typical case is.

You can use a program called AutoRuns (from Microsoft) or ShellExView (some 3rd party tool) that lets you see which extra options have been integrated into your shell right-click menus and then you can turn them on and off and figure out which one(s) are a problem.  If you search this forum (or the internet at large) for right-click crash or right-click hang there's quite a few great guides to it as well

The problem happened because the "1394 Windows Debug Driver(Kernel Mode)" was installed and Debug Mode was turned on.

bill smithers say:

Hi,

No, you can't transfer an OEM License.

If the Laptop/Computer came with the Windows Operating System preinstalled (Dell, HP, Acer, etc) , that Product Key that came with the Laptop/Computer is for the preinstalled OEM License and is non-transferable.

The OEM License/Product Key (preinstalled Operating Systems on Dell, HP, etc machines) is tied to the first machine it is installed on and dies when that machine dies.

The above rule also applies to OEM System Builders Licenses.

Retail Operating System License, that you buy and install yourself, are allowed to be transferred to another machine.

But Retail Licenses can also only be installed on one machine at any particular time.
