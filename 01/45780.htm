Had this same question a few years ago and after a lot of discussion with file system folks, it basically came down to "doesn't really matter, but we recommend NTFS for the improved resiliency."

The reason it doesn't matter is because the readyboost cache is really just a single file, so the cluster and block size and methods of the underlying file system really don't matter.  But NTFS is capable of recovering from failures on the fly, and thus suggested for use.
