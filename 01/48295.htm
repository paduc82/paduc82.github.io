Windows 10 task bar icons disappear and start tiles are missing

A friend of mine had been using Windows 10 for many months now and everything has been OK. However a couple of days ago she suddenly found that all her task bar icons had gone away and no matter how many times she clicked on the start button there was no response - no tiles showed up at all - right mouse clicks on the start button did work as normal though. No messages had appeared indicating any problem and there was nothing obvious in the Event Log. This left my friend with no way to start any program which did not have a shortcut on the desktop - a pretty annoying situation. Also strangely enough she was not able to view pdf files which had been sent to her as email attachments - she could get as far as the window saying "open the file" but nothing would happen when she clicked on open (it should have opened the Edge browser and displayed the document).

Having scoured the web it appeared there were many people experiencing the same or similar problems, however there seemed to be little or no mention of the problem in the Microsoft Community forums - I couldn't quite figure out why this should be since I have encountered this problem twice now on two totally different people's systems. There were many You Tube videos claiming to have solutions to the problem, however as is often the case many of the web blogs and You Tube "solutions" were nothing of the sort.

On my friend's system we followed some of the potentially more likely solutions including entering commands in Powershell but again these seemed to have no effect.

The one thing that did rather strangely seem to cure the problem was when we followed the advice to create a new user - the new user's desktop came up with a task bar containing the standard icons and also the program (app) tiles when the start button was left mouse clicked. We were thinking about the implications of this and concluded that at least it proved the system was capable of operating correctly and that there was probably something corrupt in the settings for her userid. We were contemplating moving all her data to the new userid and whilst thinking about this we logged off the new userid and logged on to her old one. To our amazement the old userid now came back with all her task bar icons (some of which she had added, so they were not just the standard one) and the start button operated normally, and Edge was able to display pdf files sent via emails!! However that was not all. We also discovered that the system had reinstated some startup tasks that had previously been disabled plus (and this may be a coincidence) it brought up an AMD update related to the Catalyst program.

So the good news was that creating a second userid seemed to cause the system to reset itself globally which in tunr resolved the problem of the missing task bar icons, the left mouse click functionality on the start button and the ability of the Edge browser to display pdf files plus reset the "start at startup" status of startup tasks that had previously been disabled.

I offer this not as a universal solution to the problem described in the heading but merely as my personal experience of how we luckily resolved the problem in one particular instance.

Hi Les

I've been running W10 on my XT3 for a while now, and all is going smoothly.  But when I first got my XT3 I had the same ghost-click issues/windows opening etc which you report. The solution for me, and many others initially I believe, was to change the screen-refresh rate in the Intel-Graphics control panel, from 40p Hz to 60p Hz. (40p Hz sends the screen and touch crazy). I also disabled the rubber track button in the middle of the keyboard (which was reported to help initially, but not sure if it really does or not - but I don't use it anyway). 

Tracking/lag with the n-trig pen are all very good  with the new W10 API's.

Screen-edge calibration ( once I ran a proper calibration  . . . http://forum.xda-developers.com/showthread.php?t=2171198   ) is very good. 

And the latest Wintab driver (1.0.0.20  . . .I think . . . .available through one of the MS Surface Pro downloads) works well. (Although I don't use Photoshop, so not sure on that one). 

It was a bane to me when I first updated that I couldn't get the N-trig Duosense drivers, as they came with Control Panel user settings, which I usually  set at "Auto" so you could disable the 'touch' screen with a double-tap and still use 'pen' without worrying about your palm making marks all over the place. But in the end I solved this in W10 by running a little task-bar application, "Tray Touch Toggle", which allows you to turn the touch screen off and on. 

Hope this helps a little; let me know if you find any other tweaks or solutions to the mystery of where the **** n-trig went to;
