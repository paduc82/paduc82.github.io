l_tyson say: I know this is old but just in case someone else has the same issue this may help them.    I believe the issue has to do with the way Linux keeps time versus Windows.  Linux allows you to keep your Bios time set to Greenwich Mean Time (GMT) or Universal Time (UT).  Linux will then adjust it's clock upon log in accordingly based on what time zone you set it for when you installed.  So for instance  I live in Central Time in the United States  so that is GMT - 6.  So if my Bios says it is 7:00pm ,  then Linux will adjust the time to be 1:00pm in my local time.   The problem is Windows can not do this neat trick, it relies on you to have the same time in your Bios as you do in Windows.  If the Bios time is set to 7:00pm then Windows will be 7:00pm.  Windows is incapable of adjusting for GMT.  People that dual boot Linux and Windows will sometimes have this issue.  The only thing I know to do is to disable the feature in Linux.  See this link for more information:  http://www.linuxsa.org.au/tips/time.html

Hope this helps.  I would love to see this feature in Windows 8?  :-)
-Lloyd
