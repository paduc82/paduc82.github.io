    Hi,

     

    I would suggest you to follow the steps and check if it helps.

     

    Step 1: First check the free space on this volume by following the following steps:

    a. Start -> Computer -> Right click -> Manage
    b. Select Storage -> Disk management.
    c. Verify that you have a System Reserved partition of 100 MB size.
    d. If yes, right click on this volume and select Properties.
    e. View used and free space.

    Because the default System reserved partition cannot be extended, creating a new system volume is the workaround.

    Step 2: These are the steps to move the system volume to any other volume:

    a. Choose where you want to have your system volume. Few things you need to keep in mind:
    b. System volume can only be created on a primary partition of MBR disk.
    c. If system volume & boot volume are together, then BitLocker feature cannot be used to encrypt volumes on your machine.

    d. Suggest creating a new volume (say F:) on the same disk that contains the boot partition of size of about 490 MB (be careful to keep it less than 500 MB).

    e. Assuming Windows 7 is installed on C: on your machine. From an elevated command-prompt run: bcdboot.exe /s C:\Windows /s F:.

    f. From elevated command-prompt run: DISKPART

    g. From the disk part command-prompt:
    DISKPART> select volume F
    DISKPART> active

    Finally now you can reboot your machine and F: will become the system volume. You can undo this by repeating the same steps using the original system volume (you have to assign drive-letter to it) to revert to your previous configuration.

    babu v say:

    Hi Leventis,


    Prefetch copies frequently accessed files together into a contiguous area on the disk so they can be located and loaded faster. Which is very much required in old HDDs for faster data retrieving.


    SuperFetch predicts which applications you will run next and preloads all of the necessary data into memory. It also does that with boot files. And for every three days it sends a Defragmentation command to the OS drive which will cause issues in the SSD drives.


    In traditional hard disks (HDDs) the data is scattered across the disk, in order for the operating system to access them, it will take more time. So it is suggested to perform Disk Defragmentation. But the storage technology is different in SSDs, where in, it dynamically saves data and retrieves it faster. That will low down the burden on the RAM and this RAM can be utilized for other purposes like for opening High End Graphic Games, etc.
