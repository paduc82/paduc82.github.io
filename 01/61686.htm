jegesq say:

Real answer to how to disable remote gateway as the default setting in IPv4 and IPv6 settings for "Properties" of VPN networking:

Open PowerShell as administrator and type in the following commands:

1.     "Get-VpnConnection" (without the quotes).   Hit return.  The name that returns should be inserted in the line below where it reads "myVPN".  

Then type the following (again, inserting the name of your VPN connection in place of "myVPN" but leaving that in quotes: :

2.      "Set-VpnConnection -Name "myVPN" -SplitTunneling $True"  (again without quotes)

Once you hit return,  your settings should change so that you will have disabled the "remote default gateway" thus allowing your local computer to use its own ISP internet connection while at the same time connecting remotely via VPN to whatever remote server you're connecting with.

The other methods above do NOT work.   The one posted by storcy immediately abovein particular, while allowing you to edit the file mentioned (rasphone.pbk) will not give you the necessary info to edit the setting which controls whether the VPN connection uses the remote gateway by default.  It will allow you to edit and input lots of other info, but not the one you want to change.

jegesq say:

@BrianBaxterEnvision:

You will not find DNS values in the command line within Power Shell, at least not from using the commands line entries I posted.

You just load PowerShell as administrator.  Then you enter the command lines.   You should not need to do anything with DNS settings in your VPN connection other than to "untick" the default setting which causes "Use Remote Gateway" to be set as the default.   With that setting as default, you can't use your local internet connection, and thus will be limited only to connecting to the remote VPN server, and (if you can) using that server's DNS gateway and not your own local DNS.  

This is why you want to change the settings (allowing Power Shell to change the gateway so it's "split tunneling"), i.e., it uses both the remote server for access to remote shares on that server, and also allows you to maintain your live connection to the internet, so you can continue to browse the web and get e-mail from other servers even while connected to your VPN.

Try running the commands again, exactly as set out above and that should work.

When the first command line is executed it will return the name of your particular VPN connection(s).   Select the one name of the VPN adapter that you are trying to change settings in, and that is name you should insert into the second command line in place of "myVPN".

This has worked for me and for many others.  Give it another try.

If you want to change other settings in DNS, well, yeah, you're on your own there and maybe someone else can help you out, or you can try the suggestion made by storcy, which will give you access to other settings (but you'd better know what you're doing when you make edits in the file).
