danny dickenson say:

Cumulutive update KB3163018 fails to install

KB3163018 fails to install. Windows 10 downloads it, the system reboots and attempts to apply the update, fails and then undoes the changes. This seems to be a problem others are having as well (e.g.: see here, here, and here).

Here's the short error message from the event viewer:

Installation Failure: Windows failed to install the following update with error 0x800F0922: Cumulative Update for Windows 10 Version 1511 for x64-based Systems (KB3163018).

danny dickenson say:

Updating the thread after I managed to install the update. Perhaps it'll help someone, if they get error code 0x800F0922.

(At least for me) the problem was some sort of a contention between the update and my system using the UEFI secure boot option. If that's the case, you need to (temporarily) disable UEFI secure boot and install the update.

More info and credits to the guy who posted this thread:

http://www.bleepingcomputer.com/forums/t/526163/uefi-secure-boot-may-cause-windows-updates-to-fail-with-error-code-0x800f0922/


Hope this helps someone.

---

p.s.: the procedure described in the thread above requires disabling/enabling UEFI secure boot. Here's how to do it:

- You need to reboot your Windows into troubleshoot mode by holding the SHIFT key while clicking on "Restart".

- After a few seconds, you should get to a menu where you need to select:

    Troubleshoot -> Advanced Options -> UEFI Firmware Settings -> Restart

- After the reboot, you'll get the Boot Manager. This stage is different for each Boot Manager, but you need to search for an option called something like UEFI options/settings and disable UEFI secure boot.
