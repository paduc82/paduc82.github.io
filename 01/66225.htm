johnny1089 say:

I Found a fix:

Go to the manufacturer's website and download the driver. Chances are you will have to extract them with something like WinRAR. Also know that many times there will be seperate downloads depending on whether your using a 32 or 64 bit Windows OS. (I usually download both, and add the other under "Printer Properties>Sharing, so others on my network won't have to repeat the process)(Also note, sometimes there is a different driver, depending if you're using the printer via USB, ethernet, or WLAN, etc)

In "Classic" Control Panel, click "Devices and Printers"

In ribbon panel, click "Add a printer

Click "The printer that I want isn't listed"

Select the "Add a printer using a TCP/IP address or hostname"

Enter the IP address of the wireless printer, and click next (you can try leaving the "Query the printer" checkbox selected)

Windows will spend a minute or two "Detecting the TCP/IP...."

When it gets to the "Additional port information required" dialog box, ;eave the "Generic Network Card" radio button selected (You may have to play around with the options). Click "Next"

Now, click the "Have Disck..." button and a "Install from disk" dialog box will pop up. Click "Browse"

Navigate to the folder you extracted the manufacturer's drivers to. Note that many driver packages will have multiple .INF's, so you might need to play around with them one by one until one works.

voila! Try printing a test page.


NOTE: After all is through, you can go to the device manager, and right click your wireless printer (Usually under "Print queues"), and on the popup context menu, click "Update Driver Software". If you're using Windows 7+ (or Vista+, I think), you can click "Browse my computer for driver software", navigate to the root of your downloaded extracted driver folder, and check the "Include subfolders" checkbox. It will find the best .INF to use (Some driver packages include dozens/hundreds of drivers in a bundle, and just because Windows allowed you to use the one you picked previously when trying them one-by-one, doesn't mean it's the optimal/fully functioning driver. Also, Once your printer is installed, Windows Update might recognize your printer and have a better match as well!

Information:

Many Toshiba laptops come with the factory software recovery image stored in a special (hidden) area on the hard disk.  From this special hidden system recovery image, you can restore your computer to its factory-fresh software condition by following these steps:

  1. Make sure the computer is turned off ("Shut-down" from the Windows Start menu)
  2. Remove any peripherals such an external mouse, keyboard, monitor, USB flash drive etc.
  3. Make sure the AC Adapter is plugged in and working.
  4. Press and hold down the 0 (zero) key on the keyboard while powering on the computer.  Release it when the recovery warning screen appears.
  5. If the recovery process offers a choice of Operating Systems, select the appropriate one for you.
  6. A warning screen appears, stating that when the recovery is executed all data will be deleted and rewritten.
  7. Click Yes to continue.
  8. When the Toshiba Recovery Wizard opens select "Recovery of Factory Software" or the appropriate option for your needs.
  9. Continue to follow the directions restarting as necessary.

peter_london say: Thanks David, both for your reply and your original post with some very useful info on it.

It turns out that the problem I had was setting up the RAID array - the P8H67-M PRO has an old version of Intel Rapid Storage Technology (i.e. the raid controller) that doesn't support >2.2Tb drives.

Apparently, version 10.5 or above of the RST does support 2.2Tb, however, this is not available in the current asus bios (version 1003).

The mistake I also made was to not switch the bios back into AHCI mode after failing with the raid - windows got confused and report only 280Gb. In ACHI mode windows is quite happy to install.

My workaround is to use windows 7 software raid (only available in professional or above) until asus sort out a BIOS update!

Many thanks, Pete.
