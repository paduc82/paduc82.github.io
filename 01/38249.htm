azam k say: Hi,
Welcome to Microsoft answers.
 
After you duplicate an image with the Sysprep.exe tool, two COM ports may be enumerated in Device Manager as COM2.
 If you remove the COM ports in Device Manager and then redetect the hardware in the computer, the ports are detected as COM2 and COM3.

This issue can occur if the device ID information for the COM ports in the BIOS in the master image is different from the device ID information for the COM ports in the BIOS on the target computer.
For example, this issue could occur in the following scenario:
 
You create the master image on a computer in which the BIOS is programmed to assign COM1 and COM2 to devices with _UID 0 and 1.

You then duplicate the image on a computer in which the BIOS is programmed to assign COM1 and COM2 to devices with _UID 1 and 2.
 
For more information, log on to:
http://support.microsoft.com/kb/304979
http://support.microsoft.com/kb/311272
http://support.microsoft.com/kb/311907

 I suggest you  to install the latest drivers for the modem or the network adapter.
Also install the latest drivers for all the devices from the manufacturers website and install the latest windows updates.

Also you can check in safe mode with networking if the same issue re-occurs, here are the steps:

1. Restart your computer if it is powered on.
2. Tap the F8 key after your computer initially powers on.
3. Once you see the Advanced Boot Options menu you can stop tapping.
4. Use the up/down arrow keys to highlight your selection.
5. Select Safe Mode with Networking and press Enter.
6. You should see drivers loading, and then Please wait.
7. You should then be at the Welcome Screen.
8. Logon to your computer using an account with Administrator privileges
