lemp say:

1.  Generally, anything in a "temp" directory may be safely deleted.  Rather than go through the whole permissions thing, while booted into Windows 7, click the Start Orb, type "cleanmgr" (without quotes) into the Search box, and press Enter.  This will run the Windows "Disk Cleanup" tool, which should clean out that temp folder. 

There are likely to be several folders named "temp" and Disk Cleanup doesn't know about all of them.  If the one you're looking at isn't cleaned by Disk Cleanup, what sort of files are in it and what is the path to it?

There is a third-party tool that does a good job of cleaning Windows -->  http://www.piriform.com/docs/ccleaner

Just do NOT use the "Registry Cleaner" feature.

2.  Pagefile.sys is the file that the Windows operating system uses to manage memory.  Don't mess with it.  For much more detail, see https://blogs.technet.microsoft.com/askperf/2007/12/14/what-is-the-page-file-for-anyway/

3.  Hiberfile.sys is the place where Windows stores a "snapshot" of the contents of your RAM when the system "hibernates" -- goes to low power but doesn't perform a system shut down.  If you don't need or use hibernation (which enables a quicker restart), you can turn it off and hiberfile.sys will be deleted.  See https://support.microsoft.com/en-us/kb/920730

4.  The System32 folder is a central part of the Windows operating system.  Don't mess with it.
