wunders say:

XPFixR,

It sounds like you were doing just fine until you did the Repair Install of Windows. Replacing registry files from the Repair folder puts back an ancient copy of your registry.  The hope is that this will allow you to boot long enough to do a System Recovery operation either from Safe Mode or Safe Mode with command prompt where more recent registries are put in place.  Newer backup registries are actually stored in the "System Volume Information" folder but Windows protects that folder well.  At that point in time, you could have resumed KB307545 at Part Two Step 6 by booting from a free Live Linux CD such asKnoppix .  "Tonsil Crusha" might yet be able to do this.

The "Repair Install" was all but broken by Microsoft with the advent of SP2 & 3 and IE 8.  The following article addresses that:
  "How to perform a repair installation of Windows XP if a later version of Internet Explorer is installed"
    < http://support.microsoft.com/kb/917964 >

If I were where you are at this point, I'd boot from a Live Linux CD, use it to backup important files to an external USB Drive or Network drive, then reinstall Windows.  It will probably be faster in the long run and you will have confidence in a good install of Windows.
