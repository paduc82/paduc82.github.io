graye say:

Well, let's start with a history lesson...

Back in the FAT16/FAT32 days, the directory's mod date was never changed based upon the element's changes.  

This feature was added to the NTFS.  But as file systems grew in size, it became obvious that this behavior was too "expensive" (meaning it was slowing things down).

So, somewhere around Win2000, the NTFS file system added an "lazy update" feature... so that it would eventually update the directory's mod date... but it wouldn't attempt to do so at the same time as the element.

Some filesystems (like SAMBA) don't bother at all...

https://support.microsoft.com/en-us/kb/299648
