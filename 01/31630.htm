i = 28

Cur = 0

Method 2:

I would suggest you to reset the icon cache database and then if that does not help then rebuild icon cache.

To Reset Icon cache follow these steps:

a. Right click on empty area of the desktop and click on Personalize.
b. Click on the Display Properties link.
c. Under Colors, select Medium (16-bit) and click on Apply. (See screenshot below)
NOTE: If you already have Medium (16-bit) selected, then select Highest (32-bit) instead. Sometimes you will need to do a restart first, then proceed to step d. Try both ways.

d. Now, select Highest (32-bit) and click on OK.
NOTE: If you already have Highest (32-bit) selected, then select Medium (16-bit) instead.
            i) Click on Yes to confirm the new settings.

To Rebuild icon cache follow the below mentioned steps -
a. Open Folder Options in Windows Explorer and under View tab, check Show hidden files and folders.
b. Open a Windows Explorer window.
c. Go to C:\Users\(User Name)\AppData\Local.
d. Right click on IconCache.db and click on Delete.
e. Click on Yes to confirm the deletion.
NOTE: This deletes the file to the Recycle Bin. It is safe to empty the Recycle Bin when finished.
f. Close the window.
g. Empty the Recycle Bin.
h. Restart the computer.
i. When you go back, you will notice the Size of the IconCache.db file is smaller, and the Date Modified is now the current date.

NOTE: If for some reason IconCache.db is not there or the size has not changed much, then restart the computer again. You may need to restart a couple of times in some cases.

Standard User AND Administrator accounts in Windows 8 - Microsoft Community

gregconnected say:

Standard User AND Administrator accounts in Windows 8

With the advent of Windows Vista and Windows 7, Microsoft recommended that users log in with a Standard User account, for security reasons.

Eg: http://windows.microsoft.com/en-us/windows/why-standard-user-account#1TC=windows-7

Now with Windows 8, Microsoft accounts and App Store product ownership, how can this work?

Ideally you would need both Standard User and Admin accounts both tied to the same Microsoft account to allow the admin account permission to purchase and manage apps and the Windows system, while the Standard User account uses the apps and data for every day use.

What exactly does Microsoft recommend now? One, Admin user with Microsoft account?

cmdrkeene say:

Microsoft recommendation remains the same: use a standard account for your daily use, and only use an admin account when required.  You don't even have to login to the admin account, because when needed you can just self-elevate with the UAC dialog from the standard account.

cmdrkeene say:

Don't mistake local and Microsoft Accounts as separate things: they are both still a normal local account.  The only difference of a Microsoft Account is that it's local+connected (linked online). It's still just a single account, entirely local (can be admin or not, your choice, just like always).

This Microsoft Account stuff isn't even new, it's just less known that Windows Vista and 7 both had this before too:

This Online ID let you sync some settings and use your Online Account on your local PC, just like in Windows 8.

Self-elevation from a standard account does still require admin, no change there.  So the recommendation is still the same to have a local admin-level account (I recommend not attaching it to any online account, but that's your choice), and then a standard account you use daily (which being your own daily account, would likely be the one you connect online and use for purchases and stuff.  You could make this local account admin (allowing it to self-elevate) of course, just like Windows 7, but for security I recommend against it still.
