bossdweeb say:

"I have installed the latest video drivers, didn't fix the problem."

- The latest driver ain't always the greatest, particularly with 'older' graphics cards.
Driver versions 355.60 and 359.06  seem more universally stable than the more recent 36x.xx  series,
and are a good option with Win10 while troubleshooting (or when not... I'm still using 359.06 with
a GTX770 and GTX980 ). Apparently Nvidia withdrew one of the 360+ series drivers due to instability.

- When installing the Nvidia drivers use the Custom option and install only the Driver and PhysX, at least

while troubleshooting.

- In the event of a corrupt driver uninstall the drivers using Driver DDU, as this will delete some files and

registry keys that are left when uninstalling via Programs and Features.

Display Driver Uninstaller Download version 14.1.0.0

- The beeps may be Boot Post Code beeps that result when BIOS does not detect the graphics card.
- Have you monitored the graphics cards temps ?
- If the gpu temps are high, you may only need to clean the dust from the cooler fins, and case fan filters.
But the if the cooler is clean and temps go over 90C under full load the card may be going down.
100 C is the most common thermal cut off temp for gpu's, and the time from the crash to the time of reboot
would see the gpu temp drop significantly, allowing the machine to boot into Windows again.
However if the card is under full load it could also crash a few degrees before reaching the

thermal protection cut off point.


- GPU-Z  is a good graphics monitoring tool and has an option to create a log, so you can see later
what the temp was when the machine crashes.

GPU-Z Video card GPU Information Utility

Probably should check cpu temps as well :          Real Temp   


- If the graphics or cpu temps are high cleaning and re-applying thermal paste to the gpu/cpu may

help in some cases.
It can be a gamble worth taking (the gamble being the cost of good TIM paste ) at times.
The 500 series is starting to age, and bit by bit they are beginning to fail. My GTX560ti died without warning

a few months ago. Should have done a bit better than 4.5 years, but it did have lots of work and hours on it.

- If your power supply is low end brand/model, or running close to it's amperage limits, this could also

cause issues such as yours. Gaming can put a fair bit of prolonged stress on psu's, and many cheap

psu's are not up to the task.

- HDD and RAM can also cause this sort of crash, so you may need to test both of them as well.

a. user say:

Settings>>Update and security

 just scan for updates and restart the PC's when they are ready to install. Seems like a very simple solution for only 8 PC's

Make sure the do not download over metered connection is NOT ON.

Set the active hours again and set to automatically restart pc.

Then you can select these below:

Add office and Microsoft updates in windows update:

Settings>>Update and security>>advanced options

Install Microsoft updates as well as windows updates

Delivery Optimization>>allow download from other pc's >>ON

select one of the two options.

Delivery Optimization>>advanced options

check or set upload / download options

Make sure the paused updates is not on. For Pro and enterprise.

when all set up on each pc, restart them all.

set the time for the fastest PC for update times. Then set the rest of the PC's one hours ahead of the main PC .

Example:

Main PC set windows update time from 1am to 6am

set the rest for 2am to 7am

Issues with windows update ???

Settings>>Update and security>>troubleshoot

windows update

this will try and fix update issues.

You can run these 2 commands as admin in command prompt.

right click on start menu button>>Command Prompt (ADMIN)

Press enter after each one and restart PC when finished. (Copy and Paste)

sfc /scannow

DISM /Online /Cleanup-Image /RestoreHealth

Restart PC

always restart all pc's when making changes.

I would also after all updates are installed, I would reboot their modem and restart all pc's again.
