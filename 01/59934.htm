a. user say:

Backup and Restore (Windows 7) was intended to make it possible to restore backups that you made on Windows 7 to Windows 10. It was not intended to be used for making backups on Windows 10, although people try to do that anyway with varying degrees of success or, in many cases, failure.

Windows 10 has two backup utilities: File History and System Image. That's what you should be using if you want to use Windows 10's own utilities for backup.

I personally use third-party programs for backing up because, in my opinion, they are easier to work with, more convenient and have many more features than the utilities in Windows 10. That's not a criticism of Windows. Many of Windows' own utilities are there to provide a basic amount of functionality for people who don't know how to install third-party programs.

If you're okay with third-party software, you will find much better choices for backing up.

P.S. On the one hand, backups are typically compressed, so you'll need much less than 500 GB of space to store a 500 GB backup. On the other hand, you'll want to maintain a rotating history of backups - say, the last five backups or the last week of backups - so it was a good idea to purchase a 1 TB external drive.

initially it does everything in the selected folders, then only the files that have changed since last run.

you do not want hiberfil, pagefile or swapfile to be included, this will just fill up your file history drive because they are constantly changing.

hiberfil.sys is where it saves the system state when hibernating

pagefile and swapfile are used for virtual memory. 

if deleted they will be recreated by the os when needed.

file history should be used only for your data files not the system.

create a system image (or two) as suggested by bulldogXX and exclude the route of c: and any temp folders from file history.

Well, What is your hard drives specification (The RPM for each?)? (It all depends on this) 

Do you have any files on them (large files like media files and photos)? If you have too many files then the only way to improve the performance is to perform disc defragment and clean up. If your drive is very old then checking your disk for error could also help.

 

I think that you have done most of above already. Therefore I would suggest that if you are not using these drives all the time i.e. if you are not going to install programs or Operating System on them then it is better to use them as external hard drives and connect them only when you need anything from them.

 

I would suggest to have one hard drive with good spec (lot of space say 800 GB) for your Windows Operating System then create a separate partition (say 100GB) for saving your files (That you use more often ) and install your third party programs there if you can using custom installation. then leave your drive C: or OS drive alone and try not install too many programs or save files there.

 

Another thing to check if you want to have all drives connected to your pc, is to check your boot order. make sure that the priority is set to drive that has your OS or windows 7 on it. This might helps a bit on boot performance.

 

In general then more hard drives or external devices are connected to your PC , the slower your boot time will be. Therefore they have to be high spec and to have up to date drivers (Compatible with windows 7 ) to work properly with highest possible performance.

 

Therefore if it is not necessary for them to be connected, just use them for back up or keeping your files. This way your files will be safe even if something goes wrong with your PC.
