tryx3 say:

Michael,

There are explanations of the options available to you in Recovery options in Windows 10 - Windows Help

Refresh was a Windows 8 facility. 

  • There is information about it in How to refresh, reset, or restore your PC - Windows help. 
  • To find out if it applies [in practice] to Windows 10, you could search this forum & TenForums but your search will be hampered by the fact that many people write Refresh when they mean Reset [Reset does apply to Windows 10]. 
  • In your search, I would urge you not to trust advice from "Microsoft" posters, such as "Microsoft" forum moderators, because they are notorious in this forum for posting rubbish.

One useful repair method that the Recovery options page does not mention is a Repair install [also known as an In-Place upgrade]. 

  • No applications are lost by running a Repair install. 
  • Repair installs can cure a host of different Windows problems.
  • A Repair install involves use of a Windows 10 installation disk but it is not a re-installation.
  • During a Repair install, your computer will overwrite many Windows system files with those from the installation disk. 
  • A Repair install does not reset the Registry which is why you do not have to reinstall applications afterwards.
  • A Repair install does not generally have any effect on user account problems. 
  • Some Windows settings might be reset to default values by a Repair install [in the same way that Windows update can change them] so you do need to check them afterwards.
  • Some customised Registry entries can be reset by a Repair install so if you have used RegEdit to customise your computer you will need to check those entries afterwards.
  • You should always run Windows update after a Repair install in case it has overwritten updates that you had previously installed.
  • In addition to its use as a repair method, you can also use a Repair install to update Windows to a new version.  So, for example, rather than allow Windows update to update my computers to Ver 1703, I created a Ver 1703 installation disk using the Media creation tool and then used that to run Repair installs.

The desktop shortcut you created may have some from a browser's File, Send, Shortcut to Desktop command or something similar. This creates the equivalent of half-a-shortcut in that it records the target URL but not the program that lauches the target URL. Instead, it relies upon the file association and default browser settings to launch the web page from the URL. Since you have no executable program in the shortcut, you are not given the choice of opening in a maximized, minimized or floating application window.

The following should be the correct procedure to creating an Internet Explorer shortcut to <your url> that will allow setting the display state of the application window.
  1. Right-click a blank area of the desktop and select New, Shortcut.
  2. For Type the location of the item: type or paste in,
          "%PROGRAMFILES%\Internet Explorer\iexplore.exe" <your url here>
    When you have this in the text box, click Next. If the location of the browser application is incorrect, you will be asked to repair it.
  3. Supply a name for the new shortcut and click Finish in the lower right.
  4. Right-click your new desktop shortcut and select Maximized from the Run options on the Sharing tab.
Your new shortcut should now open in a maximized window. Another option is to open Internet Explorer in full kiosk mode (e.g. full screen) by adding -k to the command line like,

"%PROGRAMFILES%\Internet Explorer\iexplore.exe" -k <your url here>

canadian tech say:

I doubt very much you are using a product named Norton Antivirus.  That product has not been sold for nearly 2 years.  You are running one of their all-in-one packages, which I would suspect could be the villan.  Try disabling it.

Before you start Windows Update (WU), if your computer has not been up and running for a few days or more, start it and let it run without using it for some hours.  Turn off the sleep functions if they are set.  If yours is a laptop, make sure it is powered with the power cord.  Start Task Manager (right-click on the task bar) and look at the CPU % in use at the bottom of that window.  If it is more than 10%, you have to wait until what ever the computer is doing finishes before starting Windows Update.

Functions like Antivirus update and  scans, scheduled defragmentation, updates to Java, Flash Player, Adobe Reader and others must be completed BEFORE you start Windows Update.

I just did this on a laptop that had not been used for about 70 days.  It took the best part of a day to finally settle down and be ready.

Windows Update is dramatically slower than it was in the Spring of 2015, before Windows 10 began downloading.  There may be nothing wrong with your computer!  The problem is a hopelessly inadequate Windows Update service.  Likely it is overwhelmed sending millions of unrequested 5 gigabyte Win10 files to Win7 computers, using the same equipment and network that normally services you and I.

From Microsoft’s perspective, the updates you want are an extremely low priority.  They do not want you updating Windows 7.  They want you replacing it with Windows 10.

There is no reason to do this upgrade unless you are a real adventurer.  Windows 7 will be supported through January 2020.

Excellent read   http://www.infoworld.com/article/2953655/microsoft-windows/windows-10-review-hold-off-if-you-use-windows-7.html#tk.ifw-infsb

I have seen cases where the green bar in Windows Update continued overnight and finally found the updates, then spent many more hours downloading those updates.

Turn off sleep and hibernate functions.  Leave WU running at least overnight, maybe longer

Check to make certain the 4 needed services are starting properly.  If any one of the services, were not already started, type services in the text box above the start globe and find each of those services in the list (list below). If they are not set to automatic, double-click the service and set.  You can do this while Windows Update is running.

Background Intelligent Transfer Services, Cryptographic services, Windows Installer, Windows Update. They should be shown as Automatic or Automatic (delayed). If they are not, double-click on each as necessary, and change setting to Automatic.

Set Windows Update settings to NEVER before you start.  The never setting will prevent that phenomena where WU is running and then you check and see an yellow exclamation point on the shut down button. That situation results in a kind of tug of war between competing WU rivals.

Check Task manager BEFORE you start WU.  If your system is very busy, it may produce that 80070490 error and that “try again” button.  Wait and watch that % at the bottom of the task manager window for a while untill it settles down to below 10%.  Then try again.

Next, we need to prepare your system to properly handle Windows Updates.  It will take a long time (like a day or more), but in almost all cases this gets Windows Update back on track.

  1. Download and run the correct version for your system of the System Update Readiness tool KB947821.  need X64.
  2. Download KB3145739 from the Microsoft web site and install it.32 bit version:  

    Now re-start Windows Update.  Do NOT restart Windows.   Be very, very patient, it will likely take a long time.  Recently, the whole process has required 24 hours or more elapsed time in my work on my client computers.

    Now, your challenge is to be PATIENT.  This may take many hours.

    VPN, won't disconnect, freezes - Microsoft Community

    kimirox say:

    VPN, won't disconnect, freezes

    Okay, so I have a brand new Dell Latitude E4300, Vista.  Everything works as it should until I get to messing with the VPN.  I set up it all normally and everything, no problem.  However, it is when I turn it on the first time that everything goes haywire.

    I can connect just fine.  When I go to disconnect, it says that it does, but doesn't.  In the task bar wireless connection icon freezes as does the network sharing center. 

    On the slim-to-none chance that it chooses not to freeze, then other oddities surface.  For example, although I am offline, it shows that I am connected to both the VPN and the wireless connection.  Or, on the rare occasion that it does disconnect, it refuses to re-connect.  Either way, it takes FOREVER for the system to reboot.

    Even though it is a new laptop, I rebuilt the system from scratch
    I added local service to admins
    I ran virus software
    I turned off both Windows firewall and Symantec
    Turned off IPv6 on all connections
    A reboot will solve the issue until the VPN is turned back on.

    Any ideas?
    kimirox say: JUST AN FYI.  I finally figured it out.  It was Symantec blocking the application.  I made sure all Windows updates were installed, all Symantec, and then turned off intrusion prevention.
