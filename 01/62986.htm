lemp say:
If you have a Gigabyte of RAM -- and you do -- you should be OK for most normal uses on a Windows XP machine ... but it does depend on what is running on the computer.

If you have 18.5 Mbps download capacity, that should be enough to stream videos without problems ... but one of the things about cable Internet services is that the overall capacity of the cable connection is shared among many subscribers.  This makes your cable connection susceptible to exactly the same thing that Charter claimed was affecting Amazon:  it slows down at peak times, when everyone's home from work using their computers.  If you live in a market where Verizon FiOS is competing with a cable internet service, you'll have heard this argument in Verizon's commercials.  Thus, it's important to do that speed test at close to the same time that you experience video streaming problems (but not simultaneously with watching the video).

What's more suggestive of your problems is not the RAM usage but the CPU usage.  If the CPU usage is spending any significant time at 100%, you have a problem, but I don't think the amount of physical RAM is the cause.  When the CPU usage hits 100%, what process is taking the most? (Look on Task Manager's Processes tab and click the "CPU" at the top of the column twice to sort the entries).

What are the specs for the CPU of your computer?  Right-click My Computer and select Properties. The information is at the lower right.  See http://www.staysafeontheinternet.com/images/antivirus/xpsysprop.jpg  While you're there, what version of Windows are you using, including service pack (that's the upper circle in the image)?

Go back to the Task Manager Performance tab and look at the section on the left under "Commit Charge."  Do this after you have been watching a video.  Here is the relevant part of the page I suggested you read:


  • If the Total Commit Charge exceeds the Total Physical Memory, XP has to use the virtual memory page file that is substantially slower than physical memory, suffering performance degradation as a result. One excellent reason to load the system with RAM.

  • Compare the Commit Charge Peak to the Physical Memory Total. If the Commit Charge peak is higher than the Physical Memory Total, not only is the page file being used, but the System Cache is not being used. Keeping the installed RAM well above the average Commit Charge peak allows XP to use the extra RAM for System Cache.

  • Keeping the Commit Charge Peak running well below the Total Physical Memory allows you to decrease the size of the page file, reclaiming hard drive real estate that can be used for storage rather than slow virtual memory.

What is your Commit Charge Peak, and how does it compare to your 1 Gigabyte Total Physical Memory?  If you are using the virtual memory page file a lot, that would explain both the 100% CPU spikes and the poor video quality.

You also may have other things running on your computer that are causing the slowdown.  This can be from overly-intrusive "security" software (what are you running as an antivirus application?) to too many "add-on" applications running in the background to malware.  I suggest closing all other applications when you are watching videos.  And on general principles, download, install, update, and run scans with each of the following tools.  The scans will take a while, so start one up and go do a non-computer chore.  I suggest scanning with these about once a week or so -- after updating.

MalwareBytes AntiMalware

SUPERAntiSpyware

Finally, you can "tune" your Internet connection.  This is not the same as using programs that claim to speed up your computer.  I strongly suggest that you don't use any of them, and certainly do not ever run an application that wants to "clean" or "optimize" your registry.  

Go to http://www.dslreports.com/tools and scroll down the page to "Tweak test" and click the Go button.

You may get a java security warning "The publisher cannot be verified by a trusted source." Click OK anyway.  Click the Start button on the Java Tweak Tool and let it run.

Read the recommendations, read the "Tweak FAQ" and if you have questions, ask in the Tweaks Forum (see FAQ #8).

Hi,

Check if the Bluetooth is detected in Device Manager.

 Refer:

Open Device Manager

http://windows.microsoft.com/en-us/windows7/Open-Device-Manager

 

If its detected in device manager, you may check if your computer is discoverable to Bluetooth devices, or set Windows to alert you when a new device tries to connect to your computer. To do this, you need to control how your computer pairs with (connects to) Bluetooth devices.

  To control how your computer pairs with Bluetooth devices

1.    Open Control Panel by clicking the Start button, and then clicking Control Panel.

2.     In the Control Panel search box, type Bluetooth, and then click Change Bluetooth settings.

3.     In the Bluetooth Settings dialog box, click the Options tab and do any of the following:

         To make your computer discoverable to Bluetooth devices, select the check box for Allow Bluetooth devices to find this computer.

When a Bluetooth enabled computer is in discovery mode, it broadcasts a wireless signal that allows it to be detected by other Bluetooth enabled computers or devices. This is sometimes called pairing mode.

         To allow Bluetooth devices to connect to your computer, select the check box for Allow Bluetooth devices to connect to this computer.

         To be notified when a Bluetooth device is trying to connect to your computer, select the check box for Alert me when a new Bluetooth device wants to connect.

 

Access the link below to know more about adding Bluetooth on your computer.

 

http://windows.microsoft.com/en-US/windows7/Add-a-Bluetooth-or-other-wireless-or-network-device-frequently-asked-questions

http://windows.microsoft.com/en-US/windows7/Add-a-Bluetooth-enabled-device-to-your-computer

 

If the Bluetooth is not detected in device manager, you may have to contact Dell support to check with the issue.
