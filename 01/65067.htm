frederik long say: My recipe in such cases is to hit the problem with hammers of increasing size (which take increasing amounts of effort to manufacture). Here are a few of them:
  • Boot the machine into Safe Mode, then log on as Administrator.
  • Boot the machine into Windows Repair Mode, then open a Command Console and rename the file. Note - drive C: might become drive D: or E: in this mode but it won't be drive X:,
  • Create a Scheduled Task to invoke the batch file c:\Tools\Hammer.bat. Make sure it runs under the System account. Here is the code for this batch file:
    @echo off
    if not exist c:\Tools md c:\Tools
    ren  "c:\..\..\pending.xml.01cc84d0b0f2bf34"  "bad.xml"  1>c:\Tools\Hammer.txt  2>&1
    (make sure to specify the correct path, then test the batch file by invoking it manually from the Command Console. Have a look at c:\Tools\Hammer.txt when finished testing.)
  • Use a sledge hammer by booting the machine with a Ubuntu boot CD (http://www.howtogeek.com/howto/windows-vista/use-ubuntu-live-cd-to-backup-files-from-your-dead-windows-computer/) and renaming the file. I have never seen a file that could resist the Ubuntu approach.

pauli o say:

Microsoft has removed Homegroup. Here is some things you can do.

To get you PC's to visible under File Explorer network section then most important part is to start some required services.

Push Win Key and type "Services", locate those services, start them and set Start type to Automatic (Delayed Start). 

Services are:

   - Function Discovery Provider Host

   - Function Discovery Resource Publication

   - SSDP Discovery

   - UPnP Device Host

Look more details from example how I have setup my network to work with Win 10 1803.

https://answers.microsoft.com/en-us/windows/forum/windows_10-files/homegroup-removed-how-to-get-network-sharing-work/01277332-2916-4a68-853a-116696b20743

pauli o say:

"never used Homegroup so its absence did not affect me"

Unfortunately removing Homegroup has effected also another parts of networking. For example referring PC's with names in the network may not work correctly on some cases.

Setting start mode to Automatic make it start during boot up with all other services and Automatic (Delayed) start those services when all "critical" services are started in boot up.

With fast PC's and SSD disks that doesn't make big difference, but with slow PC's that may effect how fast system get running.

Even on my old slow Core 2 duo laptop with SSD I have those in Automatic.

I have made all those settings on my example to all PC's. (W7, W10 and W10 Insider Previews)
