This is a bit complicated...  Normally, in a domain environment, you'd be able to sync your setting to a domain controller.  That way, the settings would always stay within the domain... and never leave the premises.

However, the current version of Windows Server doesn't yet support syncing with Edge.  So, some folks have been using Azure Active Directory instead (since it does support "Enterprise State Roaming").

But for a lot of folks, moving from an on-premises domain to an Azure domain, is just not practical (or cost effective).

BTW: No, linking your domain account to your Microsoft Account doesn't solve the problem completely.  Since this would require that every employee also have a Microsoft Account.  Also many secure environments will not allow connections to off-premises data storage services.

However, there is a free 3rd-party application that can help solve this problem without having to migrate your domain to Azure AD.   This solution is not as feature rich as "Enterprise State Roaming", but it does work.

Take a look, if you like at:  http://www.emmet-gray.com/Articles/EdgeSync.html
