m_tika say: I too have suffered from the woes of the Microsoft Windows 8 Pro upgrade process but have discovered a solution (maybe more of a work around). Dell sent me a XPS 12 loaded with Windows 8 standard rather than the requested Windows 8 Pro. They were nice enough to send me a commercial upgrade key to Windows 8 Pro w/Media Center. Like others I used the add windows features wizard, entered the install key and upgraded to Windows 8 Pro(or so I thought). After a couple of failed attempts to use the connect computer wizard to join the Server 2012 Essentials domain, I was able to add the machine to the domain  manually, but the server apps ran very slowly and final refused to run at all.  I looked at the disk management of the machine.  It was a mess with two different recovery partitions and several other duplicate partitions.  I called Dell and he sent me the steps to completely reinstall the OS.  I did that, upgraded again and still could not make the wizard work.  Reinstalled one more time, upgraded to Pro again and the stopped to take a look.  It turns out that the Microsoft upgrade process does not install 8 default groups that are installed if you get a machine with Windows 8 Pro installed from the manufacturer.  Among those groups is the "Remote Desktop Users" group which is necessary for the successful completion of the Server 2012 Essentials connect computer wizard.  I saw a suggestion from this blog, supposedly from Microsoft, to run a Windows 8 "Refresh".  With the upgrade in place and nothing to lose I did that.  It asked for the original disk, which I inserted in attached DVD drive (You may have to download a demo of Windows 8 Pro to get this if you do not have a reinstall disk).  When the refresh completed, it had reinstalled 6 of the 8 missing groups, including the remote desktop users group.  I was still missing the Cryptographic Operators group and the Hyper-V Administrators group.  Since I was not in a FIPS environment nor concerned with Hyper-V, I decided to take a chance.  The Server 2012 Essentials connect computer wizard ran flawlessly.  I was able to add the machine to the domain, the desk top launch pad installed but the Lync connect app did not install. The Server Apps (Eclipse and ACT) are running well.  The Windows 8 Pro upgrade process is definitely broken and hopefully will be fixed soon. As I said, this is probably classed as a workaround and may be old info, but it worked for me.  Good Luck!

Yes, absolutely.


In case there's confusion, the Bluetooth part in the controller is only for connecting it to a Windows PC. Bluetooth is not used on the Xbox One S, as the console does not actually have a Bluetooth chip.

It even works vice versa! You can also connect a regular Xbox One controller to an Xbox One S.
