I apologize, I know this has been posted many times before. But there seem to be so many conflicting answers, that I just had to get this clarified. Generally I have got very good, precise, and to the point answers and solutions on these forums, so it was surprising to see this.

    That is correct.

    That is correct, two different algorithms.

 

Now what I want to know is:

 

  • Does a Retail Win7 Disc get activated with an OEM Win7 key when it is installed on the SAME computer and motherboard that this OEM Key was originally activated with?

    There are two different types of OEM keys, you have the OEM SLP key that is hard coded to the BIOS, these are distributed with new systems that are preloaded with Windows 7 from name brand OEMs such as DELL, HP, Lenovo etc.


    You have another OEM key, these are distributed to persons who are called System Builders who build custom computers that are resold to customers. Usually Mom and Pop PC builders.

    More info about SLP:
  • Computers, which are built by large manufactures that come with Windows Pre-Installed, come with two (2) Product Keys:

    A)   OEM SLP: This key comes pre-installed in Windows, when it comes from the Factory. This key is geared to work with the OEM Bios Flag found only on that Manufacturer's computer hardware. So when Windows was installed using the OEM SLP key (at the factory) Windows looks at the motherboard and sees the proper OEM Bios Flag (for that Manufacturer and that version of Windows) and Self-Activates. (that's why you did not need to Activate your computer after you brought it home)

     B)    COA SLP: This is the Product key that you see on the sticker on the side (or bottom) of your computer. It is a valid product key, but should only be used in limited situations (sush as if the OEM SLP key stops self-activating for whatever reason). The key must be activated by Phone. (Note: All manufacturers that use the OEM SLP system are required by contract to include a Certificate of Authenticity (COA) sticker, that has a COA SLP key, on the computer)

    Your Windows is using an OEM SLP key, but (for whatever reason) cannot see the special instructions in the computer’s motherboard and is unable to Self-Activate.To fix the issue, you will need to change out the OEM SLP key with the COA SLP key.  The normal way to do this is to click the ‘Start’ button, right-click ‘Computer’, select ‘Properties’ and then click ‘Change Product Key’ (located in the lower right-hand side of the window). michaelmooremljm say:

gpt drive reverts to mbr

New system build, with 3 terabyte Seagate HDD and Asus M5 A97 R2.0 Mainboard. All mainboard setting are at default.

 

I converted the drive to GPT ( drive space is unallocated). this allows win 8 to install, however, once installed the drive reverts to MBR and only 2 T is recognized.

  

How is that even possible???

 

TIA,

 

 Mike

a. user say:

Hi Michael,

 

Master boot record (MBR) disks use the standard BIOS partition table. GUID partition table (GPT) disks use unified extensible firmware interface (UEFI). In order to use GPT partitions, the computer needs to be running in UEFI mode and not the usual BIOS mode. I would suggest you to refer to Motherboard user manual to set it to UEFI mode. Once you have changed it to UEFI mode, now you can set the partition as GPT and check if the issue persists.

 

MBR disks do not support more than four partitions on each disk and can support only hard drives up to 2.2 TB in size. That’s why only 2 TB of the hard drive is recognized.

 

Hope this helps. Feel free to get back to us if the issue persists or if you need any further assistance.
