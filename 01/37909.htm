dma44 say: Steve,

Thanks so much for the link to that article.
It was most helpful.

I also found this forum post helpful:
http://www.techsupportforum.com/networking-forum/networking-support/114373-cant-map-new-network-drive.html

In particular, it said to check the event log for the computer you are trying to connect to (the remote computer) to look for an event ID 2011 which would have been entered at the time you last tried to connect and failed.
I did this by alt-clicking on MY COMPUTER icon (of the remote computer) and choosing MANAGE, then clicking on EVENT VIEWER folder (in left pane) , then SYSTEM folder under that.
On the right pane, I found a listing for event ID 2011 at the time I had last tried to connect.

To test this, I went to the VISTA laptop and tried to connect to the XP desktop again, giving another error message on the VISTA laptop.
When I went back to the XP desktop that I was trying to connect to and looked at the event log again, sure enough ANOTHER event ID 2011 was listed with the new time of my added attempt to connect.

This confirmed that it was likely that the IRPStackSize was the problem, as your article suggested.

I then printed out the article HOW TO BACKUP AND RESTORE REGISTRY at:  http://support.microsoft.com/kb/322756/
then backed up the registry (both by running a system restore/save and by exporting from the top level of regedit).

I then went into regedit on the XP Desktop  to the
HKEY_LOCAL_MACHINE\SYSTEM\CURRENTCONTROLSET\SERVICES\LANMANSERVER\Parameters folder described in your article.
There was NOT an entry for IRPStackSize in the right pane, so I followed the instructions to add one, and set it to decimal 15.
I rebooted and tried to connect but still got the same error.
The article had said that the default value was 15, so I assumed that adding the IRPStackSize entry for 15 did not change anything.

So I went back into regedit and changed the value of IRPStackSize from 15 to 18, rebooted and it solved the problem.

I found the following articles also useful in solving this problem (aside from the one you gave):

http://support.microsoft.com/kb/285089


http://www.smartcomputing.com/editorial/article.asp?article=articles/archive/r0803/10r03/10r03.asp

I typed this all up so that it might help others who may encounter this same
"Not enough server storage is available to process this command" error when trying to
map a network drive (or just trying to connect)  to an XP computer from another computer.

Thanks so much for the lead.

Hi Alori,

A user profile is a collection of settings that make the computer look and work the way you want it to for a user account. It is stored in the user's C:\Users\<user name> profile folder, and contains the account's settings for desktop backgrounds, screen savers, pointer preferences, sound settings, and other features. User profiles ensure that your personal preferences are used whenever you sign in to Windows. However, if you want to delete the user profile, we recommend to follow the steps below:

  1. You must be signed in as an administrator to be able to delete a user profile. You'll need to make sure that the user account you want to delete the profile of is signed out before continuing.
  2. Press the Win+R keys to open Run, type SystemPropertiesAdvanced.exe, and click/tap on OK to open Advanced System Properties.
  3. Click/tap on the Settings button under User Profiles.
  4. Select the profile of the user account, and click/tap on Delete.
  5. Click/tap on Yes to confirm.
  6. The profile of the user account (ex: "Example") will now be deleted. A new profile for the user's account will automatically be created the next time that user signs in.
  7. You can now close the User Profiles and System Properties windows if you like.
