ralf_g say:

If you switch the monitor connections do you still get the same behaviour with either monitor? VGA->DVI, DVI->VGA


It could be a hardware detection issue.

In Device Manager, has the computer detected both monitors by their model# or does it simply enter in  a generic monitor type? If the latter you can try sourcing monitor drivers (a simple .INF file) from the monitor manufacturer. TVs don't necessarily have such though and also might not report their display capabilities to Windows via firmware either.


eg. This:



NOT this:



 If Windows cannot detect that 1920x1080 is a possible resolution on that monitor when it initializes the VGA port it would default to any lower resolution that it can detect or default to whatever options are available for a generic VGA monitor type. Assuming that the original Nvidia drivers or an updated set that you've downloaded from either Nvidia or your computer OEM are installed, look for a configuration option to 'force' detection of the second monitor or TV and see if setting that allows Windows to retain the 1920x1080 setting after a reboot. 


Your computer likely originally came with dedicated Nvidia drivers and their associated graphics configuration utility. From what you wrote here I suspect you might have allowed MS updates to overwrite the original video driver at some time and that is often problematic. Unless you uninstall the factory drivers and related utilities first there are liable to be incompatibilities arise because MS drivers tend to be quite basic and may not support hardware features that the original Nvidia, ASUS, or custom OEM driver suite enabled.
