Update: I posted much of the following in another member's post and include it here in case someone has the same problem, Googles and ends up here.

Microsoft Answer Desk has advised me that there is a flaw in the Windows 10 upgrade process that randomly (random because they don't know the conditions that cause it) effects some people, and some installs, and that will result in some or all of the programs installed no longer being installed. As I posted above, the symptoms are, the application is no longer installed in the "Program Files" folder, but is still in the "App & Features" list, "Program Data" folders, and "Start Menu", all icons on the desktop pointing to the missing programs are gone. Microsoft says until their developers can resolve the issue, their only advice is,  perform the Windows Update Recovery process and restore your previous operating system. Since the problem appears to be random, they advise to then rerun the Windows 10 upgrade. Many people have had success recovering most if not all their applications on the second install.

In this particular case, the solution to the missing app problem isn't simply to reinstall the missing programs because you will still be left with a Windows registry, Program Data folder, and Startup Menu that is not in sync with your installed apps, and that will eventually bite you in the ****.

IF what you've described is accurate, it appears that your files and folders were somehow not moved across in the final process (interrupted or just plain buggy)

Anyways, if you're up to trying and assuming that you did not tinker further under the hood (hacked the registry settings), here's one possible solution to your issue. It is a slightly tedious process if you want to do it right, and without messing things up.

goto

C:\Windows.old\

COPY the following folders and all their sub-folders to where they should rightfully be after the upgrade:

  • C:\Windows.old\Program Files\ --> C:\Program Files\
  • C:\Windows.old\Program Files (x86)  -->  C:\Program Files (x86)\
  • C:\Windows.old\ProgramData -->  C:\ProgramData\
  • C:\Windows.old\Users\  -->  C:\Users\

*** NOTES ***

  • IF asked "The folder already exist, do you want to merge", click YES
  • IF asked "The destination already has a file with the same name...", click on SKIP THIS FILE and check on the the checkbox at the bottom left that says something along the lines of "DO THE SAME FOR THE NEXT ### FILES FOUND"

*** WARNING ***

Doing so in bulk will also move over obsolete files and/or older versions of Windows programs that Windows no longer support in Windows 10, so RULE OF THUMB : NEVER OVERWRITE / REPLACE ANY FILES THAT ALREADY EXIST IN THE DESTINATION FOLDERS.

A safer, but more time-consuming method is to go into the 4 folders mentioned above, and selectively copy over the folders which I did not mark "DO NOT COPY"

Attached screencaps below are some of the folders which you SHOULD NOT copy over (simply because there will be new/updated versions already in their rightful places)

  • BLUE rectangles are directory structures you should keep note of (there's more than one "COMMON FILES" folder that needs to be copied over, make sure you copy to the correct sub-folders)
  • GREEN marks the folders you want to COPY
  • RED are folders that you DO NOT want to copy because if your Win10 is working, those left behind in Windows.old are the Win8.1 versions. *NOTE* Your computer might have different setup / hardware, what I'm showing are files left behind in my windows.old folder after a successful upgrade, ie, old files / files that are no longer needed in Win10.

In summary, here is a list of some of things I recommend based on what I learned:

1. Use the compatibility check only as a guideline. Check with the manufacturer of your PC and any hardware that was added, to confirm compatibility. The more obscure the manufacturer, the more unlikely that upgraded drivers are not available. The old drivers may or may not work. Also remember, operating system upgrades generate new revenue if they force you to upgrade hardware, so there is more incentive to steer you in that direction instead of pouring money into updating drivers (read "built-in obsolescence"). 

2. Identify your most important programs and check with the developers to confirm compatibility and licensing restrictions. If the update goes as advertised and all programs are still installed, the program may not function until you go through some kind of reactivation process. Be familiar with that process.

3. Avoid installing data partitions and system and windows partitions on the same hard drive. Assume that sometime in your lifetime, windows will fail and the last resort to recover, is to restore the hard drive. The backup and restore of the whole drive will be much simpler, smaller and faster, if it is a complete drive backup of only the Windows portion. Buy the fastest hard drive you can afford at a capacity to hold Windows and installed apps.

4. Backup! Backup regularly! Turn on Windows backup facilities and schedule them to either perform automatically or to remind you if they have to be performed manually. The frequency depends on how much work you want to do to get back to where you were. I have restored Windows from 6 months old backups and been up and running in a few days.

5. Perform a complete backup of the whole Windows drive before starting the Windows 10 upgrade. If you have data partitions on the same drive and you are backing them up separately, make sure you know what you are backing up. I performed a synch backup thinking I was backing up the data partition on the Windows drive, when in fact I was backing up a different drive altogether. Having to delete the data partition to install Windows, I am now faced with recovering the partition from a one year backup. Some of the data will be unrecoverable. I accept full responsibility. Hence recommendation no 4.

6. Is this really necessary? Is this really necessary right now? If you are currently running a Windows 7 or Windows 8 pc, support for these OSs will continue for some years. Are the Windows 10 features so desirable or necessary to you to perform the upgrade now? The less computer knowledgeable you are the more I recommend staying away from the bleeding edge. Wait a few months before making that leap. Try to involve someone in your social circle who is computer knowledgeable to help you. Check forums such as this one to get a feel for how things are rolling out. Keep in mind that the majority of people reporting on forums are experiencing problems. People with successful uneventful installs would not constitute the majority of posts. So put the number of problems reported into a realistic perspective.

So it couldn't get any worse! Oh yeah!

Providing recoverability from a failed Win 7 to Win 10 upgrade in the recommendations above, was focused on the Win 7 test desktop computer that Win 10 was being installed on. It turns out that all other Win 8 computers that you are already using your Microsoft account id on, are at risk because of setting synchronization.

Some background: Window 8 introduced the ability to synchronize the settings of all Windows 8 computers that you use your Microsoft account id as a login. The settings are stored on your OneDrive. Synchronization of settings is turned on by default on both Win 8 and Win 10 installs.

So, say you have a couple of desktops and a laptop, they don't even have to be connected to the same home LAN. You install Windows 8 on one of them. You create an account on the computer using your Microsoft account id. Window synchronization is turned on by default. You customize your Windows 8. You install a third party program to, say, bring back the start menu. You install a third party anti virus program, Norton Internet Security.

Some time later you  upgrade one of the other PCs from Win 7 to Win 10 using a clean install (some of the consequences I describe may or may not apply if you are doing a Windows update instead of clean install). Windows synchronization is turned on by default. The moment you login to Win 10 with your Microsoft account id, Win 10 and Win 8 start synchronizing settings. You don't really want this to take place at this time because you don't want the settings of this virgin Win 10 account to be propagated to Win 8 yet. As documented above, the Win 8 desktop background immediately gets changed from whatever it was, to the Win 10 background. If you reboot your Win 8 computer, the next login to your account on that computer hangs when the third party start menu fails to load properly because the Win 8 settings have been changed back to reflect the original Win 8 start menu. Not to worry. You just reinstall the third party start menu program. And since there is no third party antivirus program installed on the Win 10 computer yet, the settings on the Win 8 computer are changed to reflect no antivirus program is installed and Win 8 offers to install Windows Defender. Norton Internet Security is still installed but fails to startup properly. You uninstall NIS and reinstall it. It runs and seems to be working properly, but Win 8 notification warns that no antivirus program is running. Things seem to be very out of synch.

That's the current status of my Win 8 computer.


You can't trust the Win 10 upgrade risk to be restricted to the computer the install is taking place on, so before you start the upgrade:

0.1 Temporarily turn off synchronization on all Windows 8 pcs that you are using your Microsoft account as a login. This is documented here:

http://windows.microsoft.com/en-CA/windows-8/sync-settings-pcs

0.2 Create a Win 8 system restore point, just in case.

If you decide to reactivate the synchronization feature after the Win 10 upgrade, activate it on one computer. conservatively select the settings to sync (that is .. pick those settings, like Windows themes, that are less likely to interfere with program startups). and then do the same on the other computer. Make sure everything is working as expected on both computers, before your proceed to turn on more settings to sync.

The settings synchronization feature issue above is more serious than I thought.

Once the install of Win 10 on one pc has poisoned the settings of Win 8 on the other, attempts to recover Win 8 using restore points are futile (this may also apply to any hard drive backup restores you have of your system), even restore points created before the Win 10 upgrade. The reason is that if the restore point was created while synchronization is turned on, the first thing that will happen when recovery is completed and you login to your account, Win 8 will download the poisoned settings from OneDrive.

Even if you turn off synchronization on both pcs, the poisoned settings are still on OneDrive. You can apparently remove all backed up settings using the procedure documented here:
