When you initially activate Windows 10, a unique identifier is generated from your hardware, that is then stored in the Microsoft Activation Servers as a digtial entitlement for your PC.

Then when you reinstall Windows 10 at any time, Windows will automatically contact the Activation servers looking for this digital entitlement and activate automatically, with no user intervention

Since you previously had Windows 10 installed and activated on that PC during the installation process skip the steps which ask for a product key and select the option 'I am Reinstalling Windows 10 on this PC', and activation will not be an issue, your PC will have a digital entitlement stored on the Microsoft Activation Servers

hairyfool say:

I start out by admitting I have not done this my self but...

 

The Disk Manager cannot create the mirrors in part because it cannot convert the GPT disks to Dynamic. I recollect however that the command line tool Diskpart can do this. Diskpart has more functionality than Disk Manager although it will do some "unsafe" tasks that Disk Manager protects you from, it does still protect the O/S drive from catastrophic changes. If you have an internal card reader it would avoid confusion to disconnect it to prevent the drives appearing in any lists. Also disconnecting the existing mirror would protect it from any changes.

 

Open a Command prompt windows at Administrator level and issue the diskpart command.

 

In the new command window List Disk to identify the disks you want to work with. You need to be 100% certain which ones you want to change. The list will be numbers but I will use letters

 

  • DISKPART> Select disk A
  • DISKPART> Convert dynamic
  • DISKPART> Select disk B
  • DISKPART> Convert dynamic
  • DISKPART> Exit

     

    You will need to reboot the PC (possibly twice) to recognise the "new" disks

     

    Enter Diskpart again, List Disk to verify everything is as it should be.

     

    Then Create Volume Mirror Disk=A,B should create a mirror using all of the disks A and B. You can add the parameter Size=X(MB).  I have no idea how long this task will take but I think synchronisation is on the fly still so should be reasonably quick.

     

    The focus (Select) then automatically shifts to the new volume so the command Assign Letter=<driveletter>  will give the new volume its letter.
