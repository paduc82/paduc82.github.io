My Office shortcuts pinned to the taskbar are now showing up as blank icons in Windows RT. How do I fix this? My Office shortcuts pinned to the taskbar are now showing up as blank icons in Windows RT. How do I fix this?
marilyn o say: Hello Joe-C,

If your Office Home and Student 2013 RT shortcuts pinned to the taskbar show up as blank icons you can un-pin and then re-pin them to make them display correctly.

 

First, un-pin the blank shortcuts:

 

With touch, press and hold on the blank shortcut for a few seconds and then release. Tap Unpin this program from taskbar. Do this for each blank icon.

 

With a mouse, right-click each blank icon, and then click Unpin this program from taskbar.

 

Now, re-pin shortcuts for each Office program:

 

With touch, find the Office application tile on the Start screen and swipe down the tile to select it. Tap Pin to taskbar.

 

With mouse, right-click the Office application tile, and then click Pin to taskbar.

 

Use the Fix it in the below article to automatically fix the problem:

Office taskbar shortcuts may appear as blank icons in Windows RT

 

For more information, read the Known issues section of this knowledge base article: Update for Office Home and Student 2013 RT Preview.

 

To view information about finding Office applications in Windows RT, read Find and start Office applications in Windows 8 or Windows RT.

 

Thanks,


Marilyn

brian tillman [outlook mvp 2007-2019] say:

Because your post was off-topic in the Feedback forum, I moved it to the Windows 8 "Performance and Maintenance" forum, where it's more on-topic.

 

While viewing the Start page, type "default printer".  A list will appear on the right side of the screen.  Tap "settings" and "Change Default Printer" will appear on the left.  Tap it.  Your Devices and Printers folder will open on the Desktop.  You'll see that "Send to OneNote" will have a green check mark next to it.  Right-click the printer, if you already installed it, and select "Set as Default".  Close the folder.  From then on, when you print, your printer will be the default device.

 

If you have not installed a printer yet, make sure you download the latest Windows 8-compatible driver for it from the manufacturer's web site and install the driver following the manufacturer's instructions.  Then set the device as the default.

    sheng jiang 蒋 say:
    Visual C++ redistributables are components used by programs (games, Microsoft software etc) developed using Visual C++. Most programs bind to the most recent update of the same major version, but some programs bind to one particular version Visual C++ redistributable and don't accept any other version. Personally I released one program like this because I found a bug in the latest update and have no choice but to bind to the Visual C++ redistributable that I developed against. 

    A Microsoft Visual C++ 2005 redistributable cannot be replaced by a Microsoft Visual C++ 2008 one. The binding would fail and any program that depends on the Microsoft Visual C++ 2005 redistributable will report "This application has failed to start because the application configuration is incorrect. Reinstalling the application may fix this problem." on startup. The only way to change a binding from Visual C++ 2005 to Visual C++ 2008 in a program is to have the program's author to upgrade the source code to a new Visual C++ version, test, and release an updated version of the program, then you upgrade to that version of the program.

    Unless you have a few hours to go through all of your programs to see which program depends on which version of Visual C++ redistributable, I suggest to keep all of them.
