Single Colour Based Text Rendering Used in Recent Microsoft Softwares - Microsoft Community

jordan mele say:

Single Colour Based Text Rendering Used in Recent Microsoft Softwares

The following text rendering changes affect some users of IE10 for Windows 7, all IE10/11 for Windows 8/8.1 + app environment and Microsoft Office 2013.


In previous software, Microsoft used ClearType to improve readability but having lighter colours to highlight text characters. This worked extremely well but was ditched by Microsoft, technically with the release of Windows Phone 7 (however, on such a small screen it isn't an issue).


It is speculated by many, and highly probable that it was not used some parts of Windows 8 due to the operating system being designed primarily touch (which is mainly on tablets where the screen can be rotated easily, ClearType which uses the individual red, green and blue sub pixels does not cope well with this.) Additionally, ClearType does not work well with animations.


Instead of ClearType, Microsoft has used greyscale (it actually uses shades of the colour text it is smoothing, not shades of grey) text rendering to enhance readability.

However, this greyscale method of rendering is not correctly tunned, and cannot be tunned manually. As a result of the lack of tunning, text is inconsistent in width and as a result harder to read.


Some users will notice it, others will might have a harder time reading and not notice while for some it will go unnoticed. For me, I've been getting headaches when I'm reading articles. Regardless, the issue needs to be resolved.



If your unsure whether this issue affects you, do a comparison. Open a webpage in Internet Explorer and open that same webpage in a different browser (FireFox/Chrome). Snap the 2 side by side and compare. While very subtle, you will likely find it easier to read on the non-IE browser since most make use of ClearType like most desktop programs.

If you find that the issue does in fact effect you, click on 'Me Too' at the top of the article so that we can have Microsoft fix this.
