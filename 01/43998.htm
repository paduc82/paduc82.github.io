Hello Scorpiogirl,

The reason you see the Apply to Folders option grayed out is because you are inside of a Library, either the Document, Video, Music or Pictures library.

 

A library is very similar to a folder with the exception that no file is actually stored inside of the library. Instead the library displays the files of the locations it is set to pull from. For example, I set the User Profile’s Documents folder and a folder on my desktop named Important Documents as locations for my Documents library. When I open the library I can view all of the files in both the User Profile’s Documents and the desktop folder Important Documents.

 

You can edit the way your folder is display and save the layout in the actual folder that the documents are stored, but the library will always be set as it is by default.

tryx3 say:

When you select Run as Admin, you are confirming to the computer that Admin permission is going to be required** - you are issuing a warning not granting that permission.  The dialog you then see is asking for Admin permission; this dialog cannot be bypassed - this has been a fundamental protection component since Windows 7 was introduced.  This behaviour is the same whether you are logged in to an Admin account or a Standard user account [only the content of the dialog that appears is any different].

[** Fully compliant applications will initiate the dialog seeking Admin permission in any case if they need it.  Selecting Run as Admin first is just a precaution to allow you to work correctly with applications that are not completely compliant such as old applications.]

If SysInternals DiskMon needs Admin permission every time it runs because of the way it works then that is a limitation that might be annoying but results from the design of the utility.  Given the age of the current DiskMon, I would not expect any updated version to be developed.

I do not know why the shortcut in the Startup folder is not doing the same as the shortcut you have put on your Taskbar.  It ought to.  The two startup folders you can use are

C:\ProgramData\Microsoft\Windows\Start Menu\Programs\StartUp [to run the shortcut for all users]

C:\Users\YourUserName\AppData\Roaming\Microsoft\Windows\Start Menu\Programs\Startup [for your own user account only]

- - - - - - - - - -

Check the SysInternals download page in case it has any advice about Admin permissions [but I doubt it, it's too old]. 

Search for SysInternals DiskMon & SysInternals Disk Monitor in this forum, TenForums and possibly TechNet as well.  It is possible that somebody else has developed a similar utility that only requires Admin permission for its installation rather than its normal running.  If these searches do not yield any useful references to similar utilities then your next best bet might be to Google something like SysInternals DiskMon alternative. 

I don't know of any alternative and I cannot imagine one that would not need Admin permission every time it ran - Admin permission is needed because revealing disk activity to you is potentially revealing the activity of other users i.e. it is exceeding the normal bounds of what is allowed without permission.
