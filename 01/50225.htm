geoffc1 say: @ Eugene R. - I think that you've got your facts almost entirely wrong.

WLPG will save Descriptive Tags, Captions, Geotags, People Tags (if you've identified faces in the image) and Ratings as metadata into JPEG files, as well as holding this information in its local database.

However, WLPG does not save Flags as metadata in the image files, but only in its local database.

So, going back to the original question of MoiFou, it is certainly possible to import family photos from data CDs onto a PC's hard drive, and then use WLPG to add metadata to the photo files (as long as they are in a supported format, such as JPEG).

Once the metadata has been added to the files, then it is possible to burn them onto CDs/DVDs (formatted as data discs). Since the CDs/DVDs now contain copies of the files (complete with their embedded metadata), these can be used on other PCs of other family members. If those PCs are running WLPG (or another application that understands image metadata), all the embedded metadata will be revealed by WLPG.

There are a couple of things to be aware of:
1) What WLPG calls a "geotag" is not the usual meaning of the term. WLPG writes a textual string, like a street address, into the image metadata. If MoiFou's father means geotagging to be adding coordinate data (Lattude and Longitude) to a photo, then WLPG doesn't do this. He could use GeoSetter to do this.
2) WLPG will write the "Caption" into the metadata field of an image file that is usually used for the "Title" of a file. These are not the same thing. While Windows itself knows the difference between "Title" and "Caption" metadata, WLPG confuses the two.

More on these issues can be found here: http://gcoupe.wordpress.com/2011/12/10/picasa-versus-windows-live-photo-gallery/
