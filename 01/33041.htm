Hi Everyone

I have a client machine which has Windows Firewall enabled, which prevents me from pinging the machine or logging in remotely. The user logged in on that machine does not have permissions to disable the firewall. How would I disable the Firewall on that machine from my side?

The machine runs Windows XP and is on the same domain as my pc.

Thanks

mike groenewald say:

I seem to have found the solution:

First you need to get the correct name of the service, apparently the Display name of the service is not the actual service name.

sc getkeyname "Windows Firewall"

This gave me the correct name for the service,"mpssvc".

I then ran the command again:

sc \\computername stop mpssvc

This stopped the service.

¡firedog say:

... on the Windows 7 setup I had the storage folders (contacts and mail)  pointing to a different drive (D) which has all data files.

I wonder how you did that. It was possible in earlier versions of the program, when contacts were kept in the store folder. This proved problematical for users who relocated the store folder, because Windows Live Contacts are managed by a different program (wlcomm.exe) so that they are available to all Windows Live Programs whether you have any others installed or not. In Windows Live Mail 2011 and 2012, contacts are stored in a separate folder in the user profile, and that's where they have to stay so other programs can find them.

There are two ways that I can think of of relocating Windows Live Contacts to a different drive:

  1. Relocate the entire C:\Users folder. This will not usually cause problems except for programs hard-coded to look for the user profile on C:, rather than at the location given by %userprofile%. Instructions here: How to Move Users Profile Folder to another Location in Windows 10
     
  2. Use a symbolic link to fool Windows Live into thinking that contacts are in C:\Users\{user}\AppData\Local\Microsoft\Windows Live\Contacts when in fact they're in, say, D:\Contacts. This used to work, so long as D: was guaranteed to be always available. I haven't tested this with Windows Essentials 2012 on Windows 10, so I can't say whether it would work or not, or what problems it might bring with it. Post back if you can't find instructions for how to do this.
