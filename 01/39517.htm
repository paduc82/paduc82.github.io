Upon further checking under

the dependencies tab, the following was still listed:LexBce Server. I'm sure that refers to Lexmark How do I remove it? And then what next?

a. user say:

Get to a command prompt as follows:

Click Start, Run and in the box enter:

cmd

Click OK to open a command prompt window.

Run this command to see the current Spooler Service Dependencies:

sc  qc  spooler

You can't really "remove" the Lexmark dependency, so you just tell the Spooler Service to only depend on the Remote Procedure Call (RPC) Service (that is the default).

To do that, enter this command:

sc config spooler depend=  rpcss

Be sure to include the space after the equals sign or the command will not work.

Then query the Spooler Service again to be sure it worked:

sc  qc  spooler

This bug check indicates that the driver is in an inconsistent or invalid power state.

By default, the dumps are faulting pci.sys or ntkrnlmp which is not the true cause. If we run an !irp on the 4th parameter of the bug check (blocked IRP address), we get the following:
