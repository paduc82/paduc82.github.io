andre da costa say:
If you add a secondary language to Windows 10 version 1803 that shares the same base language as the Windows display language, neither language can be removed.

For example: If English (United Kingdom) is the current Windows display language, and you then add English (India) to the Preferred languages list, afterwards neither English (United Kingdom) nor English (India) can be removed from the Preferred languages list.

Note: The most notable effect of having multiple related languages in the Preferred languages list is additional entries in the text input selector. You can remove unwanted language list entries using these steps:

1. From the Start menu, search for powershell.

2. Right-click Windows Powershell in the search results and choose Run as administrator.

3. Type or copy/paste the following command at the command prompt, and then press Enter.

Get-WinUserLanguageList

After typing this command, make note of the “LanguageTag” listed for the language you wish to remove, as you will need it in the next step.

4. Type or copy/paste each of the following commands at the command prompt, and then press Enter. In the following commands, instead of typing <languagecode>, replace it with the language tag you wish to remove, such as en-in for English (India).

$LangList = Get-WinUserLanguageList
$MarkedLang = $LangList | where LanguageTag -eq "<languagecode>"
$LangList.Remove($MarkedLang)

Set-WinUserLanguageList $LangList -Force

Trouble-shooting is often a process of splitting the task into small steps until you know which one fails. In your case I would now do this:
  1. Create a System Restore point on a healthy machine.
  2. Boot this machine with your Windows Repair CD.
  3. Select System Restore.
  4. Check if you can see the restore point from Step 1.
If this does not work then there is something wrong with your repair CD.
If it works then the repair CD is unable to cope with your problem PC. My next step would now be to replace its hard disk with a spare hard disk, then try to get into repair mode by booting the machine with the repair disk.

If this works then the repair disk is unable to cope with your problem disk.
If it does not work then the repair disk is unable to cope with the problem PC. In this case I would get the machine serviced.

davnel019 say:

OK, let's get back to basics. I assume, at this point, that there's no real reason to keep the contents of the disk. Windows is messed up, and Ubuntu seems to also have problems. I would back up any critical data, boot up on something like Parted Magic or Acronis Disk Director, and strip the disk. Split the disk in half, and set up two equal partitions on the disk. When both are set up, delete the first, leaving the beginning of the disk unallocated. Install Windows in that first partition, set it up, and activate it (although that's not strictly necessary at this point).

 

When Win 7 is set up and running, start the Ubuntu installation, use it to delete the second partition and install itself there, using it's defaults unless you are a heck of a lot smarter than I am. Ubuntu will set itself up, then install a GRUB boot loader which will take care of both OSs. That setup should run fine.

 

I suspect, although you didn't say so, that you installed both systems in the same partition (which is allowed but not advisable) and they were conflicting. Windows, in it's typical arrogant manner, assumes it has total control of the disk partition it's installed in, and doesn't like intruders. The control file for Grub will be in the Linux partition.

 

You might notice some minor hardware interference when going back to Windows from Linux. I've found occasionally that the system clock was messed up, or that the NIC ports were changed, but usually not much of a problem.

 

I'm probably a mile off, but that sounds right.

 

 It occurs to me that you can use the Windows installation to clear the drive and create the two partitions. When that's done, delete the first one and proceed as directed. Windows will make two partitions in that space, one for the System Reserved portion and one for Windows itself.
