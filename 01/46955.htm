Uninstalling the default windows apps is not generally recommended.  They do not take up much space, and can be hidden if not used.  There are Powershell workarounds, and some other advanced tool ways of removing them, but it is likely that the next windows update would re-install them anyway.

Found this on the tech adviser site. Hopefully it helps.

Open the Windows 10 Search Bar, and type in PowerShell.

Right-click the PowerShell app and click “Run as administrator”.
Type the following command and press the Enter key:
Get-AppxPackage Microsoft.XboxApp | Remove-AppxPackage
Wait until the process is finished. It should only take a few seconds to run.
Type exit and press the Enter key to exit PowerShell.

rohn007 say:

Articles about Win 10 (8 and 7) Privacy issues and how to fix them

Technical Level : Intermediate

Summary

MS added many new "telemetry" features in Windows 10. 

Telemetry is MS's way of saying that Windows is "phoning home", sending various information back to MS.  There is been a lot of comment about this in the media. MS says this information is used to "improve" Windows and is "anonymous".  Some people are concerned that what may be sent out from their computers is leaking sensitive information about them or the files on their computer.

This Wiki is a collection of articles on the subject. Many by various tech reporters, some from MS.  Some of them include instructions on how to disable the various telemetry setting both during install and later as customization.

Why should you care about "telemetry"?  This article is about what google could do a couple of years ago, but MS through Bing, Cortana and Office Telemetry is doing much the same thing now:

https://web.archive.org/web/20160123090622/http://windowssecrets.com/top-story/why-the-u-s-needs-better-privacy-laws-now/

Why should we care, MS says the data is anonymous.  Then again, maybe we can't rely on that. Here are a just a couple of examples of the way data can be easily "de-annonymized":

Data-mining technique outs authors of anonymous email

http://www.theregister.co.uk/2011/03/15/anonymous_email_algorithm/

fixit felix say:

I know what the problem you're having is. Your Store app is unable to write data in C:\ProgramData\Microsoft\Windows\AppRepository\Packages, so you can not download any apps from the store

Here is how to fix it:

+)First, you need to take ownership the folder AppRepository on C:\ProgramData\Microsoft\Windows\AppRepository. (If you don't have the folder ProgramData, go to Folder Options, check the Show hidden files and folders option). You download the Take Ownership registry  here http://www.howtogeek.com/howto/windows-vista/add-take-ownership-to-explorer-right-click-menu-in-vista/  and follow the instruction on that site to take ownership of that folder.

+)Second, you need to give the SYSTEM account and the ALL APPLICATION PACKAGES account full control on the folder C:\ProgramData\Microsoft\Windows\AppRepository\Packages. Here's how you do it

- Right-click on the Packages folder and select Properties.

- Go to the Security tab and click Advanced.

- In the Advanced Security Settings, click the Add button

- Click on the "Select a principal" link located at the top of the window

- Type "ALL APPLICATION PACKAGES" (without quotes) in the textbox of the new window and click OK.

- Check the following permission Full control, Then click OK  

- And do the same with SYSTEM account

It works perfectly for me. I also tried every solotions on the internet but none of them worked. It turned out to be very simple, the Store app is unable to write data on the installation folder. Let me know if it works for you
