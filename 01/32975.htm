darkest shimi say:

Cheers all! We managed to resolve this through a combination of advice here and the help of Xbox support chat.

This is what ended up working, for anyone who's having the same issue:

  1. Put the Xbox into offline mode (Settings > All Settings > Network > Network settings > Offline)
  2. Launch Fallout
  3. Delete the backlog of saved games, other than the last few
  4. Play Fallout for a few hours in offline mode. 
  5. Exit Fallout
  6. Go back online
  7. Launch Fallout

We still had some syncing, but it was relatively quick and we were able to get into the game afterwards. We've had no problems since.

Xbox Support had suggested we'd have to do this step after number 7 above, but it wasn't an option that came up for us:

  • Once launchd, you will be asked what data you are going to sync, cloud data or offline data. Please select Cloud Storage Data.

skeeter quill say:

The reason Fallout 4 takes so long to sync is that the save files are huge, especially 9 days of play. You're looking at somewhere between 1.5-2GB of save files.

I've found that the best solution is to disconnect from Xbox Live and load up a game offline. then delete as many saves as possible, keeping only the very last and if you need to, some at major branching points in the game if you think you'll want to play from them again. Then connect back to Xbox Live and exit the game. Then hopefully, when you load up the game again, it will remove most of the saves from the cloud and syncing should only take a few minutes.

When you do this, be sure to do it for all characters if you have different playthroughs with alternate characters.

It can only share with "app store apps" (also known as Universal Windows Apps).  The old "Office" version of the apps don't have the programming to receive incoming shares (they were created before such sharing existed).
