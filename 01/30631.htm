That's unlikely to help. 

There are a number of network activities that take place when your computer is booted up.  If your network adapter is configured with the Windows default to have its IP address allocated automatically, one of the activities is a short series of messages between your computer and a DHCP server (generally in your home router).  Unless your computer has been turned off for more than a day or so, the "lease" of the computer's IP address will probably have not expired, and the computer will request that the DHCP server "reallocate" the same address.  This is a shorter process than the IP address "allocation" process that happens when a computer is first connected to a network (or is connected after having been powered off for a time long enough for its lease to have expired).

However, whether your computer is asking to have an IP address allocated for the first time, asking to have a previously-allocated IP address reallocated, or using a manually-configured (static) IP address, one of the final steps taken before "getting down to business" is checking to see if there is another computer on the network that is using the same IP address.  It does this by broadcasting a message to ALL of the devices on the network using a technology called ARP (address resolution protocol).

The check for potential duplicate IP address goes something like this:
  1. When ready to check (e.g., after being tentatively allocated an IP address by a DHCP server or sooner if all of the parameters normally supplied by the DHCP server have been manually configured), the system waits for up to 1 second before making its broadcast.
  2. The system then sends 3 "probe" packets, evenly spaced 1 to 2 seconds apart. 
  3. If no other device on the network responds by 2 seconds after the last probe request is sent, the system assumes that the IP address that it wants to use is OK.

There thus can be a delay of up to 7 seconds before the system decides that its IP address -- even a manually-assigned static IP address -- is safe to use.


If NIS attempts to use the Internet during that period and gets "confused" resulting in a longer than normal boot up period, that's a failure on the part of the Norton design team and yet another reason to get rid of that product.

    Disclaimer: What follows relates to comparison between Windows 7 and 10 parental controls (or family management, as it is called in the latter), I never used 8.1 parental controls.

    Gist of the message: parental controls on Windows 10 are change for the worse

    For the two main reasons:

    1. You can't change them locally - internet access is required. Now, imagine situation of traveling with the kid on the back seat dying of boredom and asking to be given a laptop.     2. Screen time can be set only as a whole bulk - so, "On time", "Off time", "Overall usage time" in between the first two. Now, in Windows 7 I used to handpick hours kid could have used laptop in - the point being, I don't only want to be able to limit the overall screen-time, but also span down sessions. The way it has been handled in Windows 10, and knowing my kid (9 years old gaming addict :( ), he will use all his hours up in one go - damaging his sight in the process, I fear.

    Or else, I will have to keep going to the control page every day, every odd hour, tweaking it all the time so he wouldn't be spending more than an hour at the laptop at a time. Thank you very much, Microsoft, I could not have dreamed of anything better to do with my time then doing the drill from now on daily.

    Now, apply the need to my point one - lack of net access. What if I'm in a meeting? What if my phone battery dies? What if I'm in a place with bad coverage? What if I fail to connect to the net, fail to tweak the time? What is I simply forget, one day? The kid is left without an hour of gaming they expected to have in their usual time (drill applied in Windows 7 was - 1 hour in the morning, 1 hour after school, 1 hour before bedtime), that hour they believe to be rightfully/habitually theirs?

    We have hurt feelings and quarreling. Thank you very much Microsoft, once again...

    Solution:

    1. We need an option to be able to tweak parental controls locally, on the machine itself, not in the cloud only
    2. We need an option to be able to pick out "allowed hours", not only "As early as" till "No later than" with "limit per day" in between.

    Come on! Even android apps can do that, I have one on the kid's phone, blocking everything but calling and messaging according to a schedule. Having seen Windows type of parental controls, I very much doubt I'll be switching to Windows phone for the kid any time soon. 

    It is nice to have an ability to change settings for all windows devices in one go, I admit.  And time limits with enough flexibility to let kids themselves choose when to use said limits up might be a great option for older kids/teenagers, but for 9 year olds and younger, who can't yet control themselves, who get carried away and can't keep track of time while gaming, it's not the best of solutions, believe me.
