Feedback: Miscellaneous problems with Windows 8.1 Pro - Microsoft Community

paul coddington say:

Feedback: Miscellaneous problems with Windows 8.1 Pro (x64)

Here is a list of miscellaneous problems encountered with Windows 8.1 Pro (x64) GA that I have collected so far.

 

Some minor, some not.

 

I have not had time for detailed confirmation and tests (sorry!).

 

Internet Explorer 11

  1. Google web sites are broken when Microsoft compatibility lists are enabled. [FIXED 2013.11.07]
  2. User-defined compatibility lists do not persist. They are erroneously deleted when browser cache is purged.

Modern Interface

  1. Many Start Screen items are wrongly flagged as NEW, and the NEW flag frequently reappears for old programs "at random". For workaround, see: http://bit.ly/HvNW4h. 
  2. It is mysteriously impossible to have two Start Screen shortcuts with the same command line (all but one will be hidden). Work around this by appending empty string(s) ("") to command line.
  3. Running Disk Cleanup utility causes various Modern applications to lose their privacy settings for Location and Devices (especially built-in applications such as Skype, Finance, Weather, etc). It's as if these settings are being stored in the %TEMP% folder. An incoming call is not the time to discover that Skype no longer has access to microphone and camera!
  4. File storage settings revert to SkyDrive without warning at seemingly "random" intervals, even though they were explicitly set to save to Desktop.
  5. Frequent unhandled script exceptions in numerous apps, especially built-in apps such as the Windows Store (less frequent after GA updates, but still present - cannot count on background apps to keep running). This has improved dramatically over time.
  6. Background/lock screen apps often do not seem to launch automatically (and therefore cannot be relied upon for notifications).
  7. Live tiles are frequently incorrect. Example: Weather tile displays different temperature and precipitation information on the Live Tile compared to when the application is opened full screen. It might say "Fine 20¬∞" on the live tile and "Rain 16¬∞" when opened (the latter is always the one that is correct). If any favorites are set, the live tile displays the wrong city (home location is set to Tokoroa, but it displays weather in Wellington, although using the location sensor by mistake would report Hamilton).
  8. Live tiles are frequently out of date Example: Mail and Facebook seem to be perpetually unaware that other clients, such as Office Outlook (using IMAP) and Internet Explorer have collected/deleted mail or read notifications. Facebook keeps flashing "recent" notifications as if they are "new" even when they are old and read).

Desktop Interface

  1. File Explorer frequently temporarily loses the ability to select multiple items with the Ctrl and/or Shift key in combination with mouse click. This is triggered by some action to do with navigating between folders but is very difficult to pin down. Similar to another problem where a selection rectangle "randomly" appears in Explorer (the latter carries over from Vista/7/8).
  2. File delete/copy/move/empty dialogs do not display graphics correctly (jagged edges, black border gap on left).
  3. 'Open with' context menu in File Explorer does not display icons for Modern applications.
  4. Files copied to Desktop are moved to top right corner, rather than sit where placed by the click that activated the Paste command context menu (and thereby cause some confusion by hiding behind an open window  - "did I just copy that to the Desktop or didn't I?").
  5. File History restore window cannot be minimized, then opens a second File Explorer window on the target folder unnecessarily (carried over from Windows 8).
  6. Items deleted from Windows Desktop continue to be displayed for some time or until the Desktop is manually refreshed. If they are re-deleted, a "file not found" error occurs. This does not happen in a File Explorer pane, only the Desktop proper.
  7. Internet shortcut Comment and Rating properties cannot be edited (carried over from Windows 7/8, worked fine in Windows Vista). Error message is different to that of Windows 7, now implying a permissions problem is the cause. Workaround by using text editor to edit shortcut, unless dealing with Unicode (Japanese), in which case it is easier to bring up a Vista virtual machine to do the job (yes, this is absurd).
  8. Recent Places icon spuriously appears on Desktop when Recent Places is cleared, until Desktop is refreshed (carried over from Windows 7/8).
  9. Taskbar buttons stop bringing up existing instances of applications and start launching new instances from time to time. Sometimes this is accompanied by the bottom half of the Taskbar buttons not being drawn.
  10. Progress dialog for File History purge operation displays file size as 0 no matter how many versions of files are being deleted (carried over from 8).
  11. Frequent IMECustomerEvent IPX Assertion errors are reported in Problem and Reports (about 30 errors per sign in). Japanese IME is installed but not in use when this happens.
  12. Windows Media Player crashes when clicking the close (X) button while MP4 video is playing.
  13. Windows Media Player cannot be launched to play a media file within several seconds of having been launched to play another file, it reports a server error (carried over from Windows 7/8). Applicable only when opening media files individually in File Explorer.
  14. Taskbar buttons display the wrong icon under some circumstances. For example, launching a batch file with HSTART when the path is incorrect results in a Windows Media Player icon being displayed in the Taskbar for the file not found error message. Also, Taskbar icon does not match specified WSCRIPT (VBS) dialog icon (displays Question (?) icon when dialog is a Warning (!) icon).
  15. When deleting multiple subfolders, File Explorer spontaneously navigates to the parent folder rather than stay in the current folder as expected.
  16. When a folder is opened with File Explorer and the first item in the list is double-clicked, all items in the folder become selected and are launched (carried over from Windows 7/8, perhaps earlier). Intermittent but common, exact trigger still uncertain.

I think this one is already known and being worked on:

 

Resuming from sleep is often greeted by a black screen (no video signal). So far, this has always been resolved by hitting the power button to hibernate the machine and then re-awakening it (fortunately I had set the power button to hibernate before this happened or it would have been a real problem). There is a knowledge base article that describes this happening when TPM is active, but BitLocker on my machine is running off a USB key (no TPM).

 

GENERAL COMMENTS FOR THOSE CONSIDERING UPGRADING:

 

The underlying system (critical aspects) of Windows 8.1 seem to be high performance and rock solid, but there are minor errors in the user interface surface layers which give a bad impression. Do not let the above list put you off - there are many great features that make life easier to enjoy. It is really important to make sure you have modern hardware that is on the compatibility list, otherwise you will run into trouble quickly.

 

BitLocker improvements alone are worth the upgrade. They save me a lot of time compared to Windows 7. I am greatly relieved that the serious CHKDSK bug in Windows 7 that crushed the system by eating all available memory is fixed in 8.x. And for those who prefer XP - I was using an XP machine the other day and was quite surprised at how difficult, feature lacking, and limiting it was compared to using 7/8.x. Consider that! (;P).

 

Also, when I use the word "random" above, this is a euphemism for "turns up unexpectedly without the triggering action being obvious". When I say "collected" I mean "personally encountered" not "read somewhere".
