    balaji kundalam say:
    To remove the Linux boot menu, you need to insert Windows installation disk and perform repair operation.

    Insert the Windows installation disk and boot from the disk.

    Once Windows installation screen appears, click Repair Computer.

    Then, the repair will search for any Windows installations.

    Select the installation and click Next.

    Once done, System Recovery Options dialog will show up.

    Click Command Prompt.
    Once opened, type the following:

    C:
    <press enter>
    bootrec /fixboot
    <press enter>

    Once done, restart the PC in normal mode and check. If the issue persists, try bootrec /fixmbr in the Command Prompt window as did with bootrec /fixboot.

    http://support.microsoft.com/kb/927392

    FWIW I have an ASUS monitor too and have had to switch to its VGA port with my Surface 2 Pro since W10 FCU;  otherwise I had your symptom.  However, if you want there are software workarounds available.  Press Win+Ctrl+Shift+B (more than once may be needed with psychic pauses between retries).  Another way to refresh the video system is to use the Project dialog to make a change in the display configuration; e.g. press Win+P, CursorUp, Enter.  If you switch to Second Screen Only you need to be prepared to respond to a Keep this? prompt; press Tab, Enter for that.  It helps to have Narrator running (Win+Ctrl+Enter) so you can at least hear what you are doing even if you can't see it.  Unfortunately the default audio can be associated with the HDMI port and then you can't hear Narrator and I have not yet figured out a workaround for ensuring that Narrator can use the default communications device (e.g. my headset).
