4.      Ignore the files and folders that give an error message.

5.      Click Start, type cleanmgr and then hit enter.

I don't think there'll be a 'fix' from Microsoft. As you'll see below, the type descriptions come from the application you've chosen to open the file, so the burden is most likely on VLC, you may want to contact them.

In the HKEY_CLASSES_ROOT hive of the registry, file extensions are associated with FileTypes and FileTypes with programs. The easiest way to determine the FileType for a particular extension is via a Command Prompt using the assoc command:

C:\>assoc .txt
.txt=txtfile

C:\>assoc .mp3
.mp3=iTunes.mp3

Then open RegEdit and navigate to:

HKEY_CLASSES_ROOT\<FileType>

It's the FriendlyTypeName value that Explorer queries for the Type description. Although it will be a resource reference (@C:\...), a literal string is valid as well. Give it any description you like:

nikita rousseau say:

Loss of brightness control / Intel HD Graphics 4000 / DELL XPS 15 L521X

Hello,

This thread will explain in a few steps how to fix the loss of brightness control with Windows 10 updates (Redstone / Creator Update etc.).

Version Control : thread updated on 31/01/2018 (dd-mm-yyyy)

This problem was first encountered when I updated my system from Windows 8.1 to Windows 10.

This post is an updated version of the following one : https://answers.microsoft.com/en-us/windows/forum/windows_10-hardware/loss-of-brightness-control-intel-hd-graphics-4000/64c255eb-517f-44d7-8af3-b5cfd7c4b79d?page=2

Symptom :

  1. "I have a XPS 15 (L521X, Mid 2012), originally shipped with Windows 8. I upgraded to Windows 10 and I lost the ability to adjust the brightness of my screen"
  2. "This problem is recurrent and occurs on each yearly update of Windows 10"

Impacted hardware, drivers and operating system :

  • Windows 10 Pro Genuine (Microsoft Windows [version 10.0.16299.192])
  • Dell XPS 15 L521X
  • Intel(r) HD Graphics 4000 w/ driver "10.18.10.4885"
  • NVIDIA GT 640M

Fix:

  1. Update your Intel drivers : Intel® Graphics Driver for Windows* [15.33] ( Version: 15.33.46.4885 (Latest) Date: 1/3/2018 ) at https://downloadcenter.intel.com/download/27417/Intel-Graphics-Driver-for-Windows-15-33-?v=t
  2. Download / Install / Reboot
  3. With "regedit.exe" update the key (create if it doesn't exists) [see below for a ready-to-use file] :
    [HKEY_LOCAL_MACHINE\SYSTEM\ControlSet001\Control\Class\{4d36e968-e325-11ce-bfc1-08002be10318}\0000]
    "FeatureTestControl" with dword32 , hex value : "f008"
  4. Same operation, add / update the following key :
    [HKEY_LOCAL_MACHINE\SYSTEM\ControlSet001\Control\Class\{4d36e968-e325-11ce-bfc1-08002be10318}\0001]
    "FeatureTestControl" with dword32 , hex value : "f008"
  5. Update registry keys / Reboot
  6. Update your GT640M Drivers with nVidia support site (I'm using version 23.21.23.9077)
  7. Reboot / Enjoy !

Registry file : https://pastebin.com/8VkZvngB

Source : this method is inspired by the one described by John and Robin Reynolds
