By default, Windows is in a Workgroup called Workgroup. A Domain is controlled by a Domain Controller (a server) and operates much differently than Workgroups. If the network you're on is a Workgroup with a different name than other workstations, then other workstations/resources may not see you on the network. 

When a system restore fails, then no changes should've occurred to the PC. It simply reverts it back to its current broken state that it was in when you started.

I wouldn't copy the whole user folder because you will copy data that messed you up in the first place. I've had a few clients that had corrupt user profiles, and luckily a Check-Disk routine fixed them right up, so perhaps try that if you haven't. If the username of the old profile and the new profile is the exact same, then put a .old on the old one, so your PC knows what profile to log into, as well as you knowing what profile you're making changes to. Personally, I would rather copy my data to a fresh profile and reinstall programs. I would never suggest taking a whole User folder, and slapping it into a new one.
