1     System images - a possible future strategy for you to consider

OK, I now appreciate that you have not made any System images.  I cannot help with your current problems.  When those problems are sorted out or you have given up and installed from scratch again then I suggest you make regular system images as your primary repair mechanism - they can avoid a lot of stress. 

You can learn about system images in Windows Help.  They are snapshots of the Windows drive that can be restored to the drive if corruption has occurred.  In brief, you can boot the system using your installation disk, connect the external drive that contains the System image that you made then restore that image to the system drive.  The system drive will then be in the exact state that it was at the moment that you made the image - so any subsequent corruption is bypassed because everything on that drive will have been overwritten.  Application settings and anything else on the system drive will also be restored to their values when the image was made.

A System image is roughly the same size as the amount of space used up on your system drive; it takes me about 25 minutes to make and 25 minutes to restore.  You write them onto external drives by starting Windows Backup and then using the link you'll find on the left hand side of its opening dialog - Create a System image.  It is the only part of Windows Backup that I use but you can, if you wish, use the whole of Windows Backup and get it to make a new system image at the same time.

For dual-boot computers, the System image procedure can include both OS drives.  The first OS can be imaged on its own.  The second OS will probably insist on making an image of both drives.  So, in my case:-

I image my Windows 7 drive on its own. 

When I image Windows 10, the procedure forces me to image both my Windows 7 and my Windows 10 drives [if I force it to image windows 10 on its own then the image is useless and cannot be restored from].

Like you, I have my data on a separate Data drive.  I relocated my C:\Users\MyUserName folders onto my Data drive.  I have also relocated my MS Office Outlook pst file onto my Data drive.  I don't have any other awkward applications to interfere with this system-data split [but I see from your comments that you do so you cannot have a perfect split].

2     A cautious approach to a Repair installation

If you are nervous about running a Repair install then make system images before you start.  If you have any problems with the Repair install then you can restore the images to get back to where you started [it will be as if you have never run the Repair install].  If the Repair install succeeds then all well and good.

I have only run a Repair install twice.  I did it purely to check the procedure and it worked without any apparent problems.  Before I started, I made a System image.  As soon as I had finished, I restored the System image just on case there were any non-apparent problems.

3     A Repair install procedure [a copy of my own procedure notes]

For Windows-related problems such as yours [rather than application problems], you can consider a thorough system repair by using a Windows 7 installation disk to run a Repair Install [also called an In-Place Upgrade].  A Repair install is not a procedure to be taken lightly and you might prefer to delay such a drastic step until you have exhausted all other options.  If, however, SFC continues to report unfixable system files then you really have no choice but to complete the repairs to your system by running a Repair Install. 

3.1  A Repair install should not affect any user files - it should only affect the operation of the system.  A repair install can disrupt some drivers, some applications & some Windows settings so they might need to be re-installed or reset afterwards.  You will need to use Windows Update for some quite substantial updates afterwards if you choose not to update the installation at the start of the procedure [see para 3.4 below].

3.2  Note that, if you have updated to SP1 since installation, you ought to use an SP1 installation disk to do a Repair Install.  If you have not updated to SP1 than either an original installation disk or an SP1 installation disk can be used to run a Repair install but using an SP1 installation disk will complete the SP1 update for you.  If you have updated to SP1 but the installation disk is pre-SP1 then the Repair install might refuse to run and you are left with no choice but full re-installation followed by endless updates.

3.3  To prepare for the Repair install -

3.3.1  As a precaution, backup all user files & downloaded installation files as well as activation keys or other registration information.

3.3.2  close all applications except your real-time anti-malware utility [if you have problems running the procedure with your malware protection operational then you might have to temporarily disable it but this does risk infection so keep it running if you can],

3.3.3  log in to an Admin account,

3.3.4  disconnect from any unnecessary networks,

3.3.5  connect to the internet,

3.3.6  insert your installation disk.

3.4  To run the Repair install, use the illustrated procedure* at Repair Install - Windows 7 Forums and also have a look at the MS summary at How to Perform an In-Place Upgrade. 

[*  But when the Repair install has all but finished & prompts you for your Product Key do not enter it, instead clear the box for Automatically activate Windows when Im online then click on Next.  This is particularly important for preinstalled Windows 7 such as ours.]

3.5  After completing the Repair install, there is some tidying up to do.

3.5.1  Check your activation status at Control panel, View by - Small icons, System.  It is shown as the second-to-last entry near the bottom of the window.  You will probably find that its status has not been affected by the Repair install.

3.5.2  Check drivers, applications & Windows settings as they might need to be re-installed or reset. 

3.5.3  Check & adjust settings for your networks.

3.5.4  Re-enable your malware protection if you had to disable it to run Repair install.

3.5.5  Make a new system image.
