wunders say:

The MBR is fixed-located at Sector 0, Offset 0 of the drive.  With the problems that you are having, I would guess that the SSD memory at this location is no longer functional.  Without being able to read or write at this location, your SSD is probably useless.  Unless the SSD manufacturer has a utility to re-map good memory to this location, your drive is probably not usable.  Check the drive manufacturer's website for diagnostic tools that you might be able to perform on this drive.

john john __ say:

Fixmbr & Fixboot will not mess up the MFT, the MFT is not in the MBR or Boot Sector.  You can use the NT6 (Vista/Windows 7) tools to do this with the caveat that the NT6 Fixboot command will dislodge the Windows XP boot loader (ntldr) from the boot sector and replace it with the Vista/Window 7 boot loader  (BootMgr), this would prevent a Windows XP installation from being able to boot, on a data disk it would not make a hill of beans of difference.  The NT6 Fixmbr command will not affect the MBR IPL on an an already partitioned  Windows disk, the MBR IPL is the same for all Windows operating system so a Windows XP installation would still be able to boot after running the NT6 Fixmbr command.  However, running the Fixmbr from any NT version would dislodge the Linux GRUB/LILO boot loader from the MBR on a Linux OS disk and the Linux installation would no be able to boot.

John
