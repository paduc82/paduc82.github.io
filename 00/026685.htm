joeu2004 say:

The problem is not with VLOOKUP.  Instead, the problem is:  the value in F5 is indeed infinitesimally larger than the value in C4.

Note that =MATCH(F5,C4,0) returns #N/A.  That indicates they do not match exactly.

Also note that =SUM(F5,-C4) formatted as Scientific returns about 1.49E-08, indicating that F5 is larger by that amount.

FYI, =F5-C4 returns exactly zero because of a trick that Excel plays on us.  When the true arithmetic result is "close" to zero, sometimes Excel replaces the true result with exactly zero arbitrarily.  The operative word is "sometimes", as demonstrated by the difference we see with the SUM function.

[EDIT] For similar reasons (but using different criteria), =(F5=C4) returns TRUE, even though the MATCH expression above demonstrates they are not equal.

Moreover, =MATCH(INT(F5),F5,0) returns #N/A, indicating that it is not the exact integer that Excel displays.

In contrast, =MATCH(INT(C4),C4,0) returns 1, indicating that it is indeed an exact integer.

Part of the confusion arises from the fact that Excel formats only up to 15 significant digits, an arbitrary limitation.  In fact, the value in F5 is exactly 53939634.0000000,1490116119384765625.

(I use period (".") as the decimal point and comma (",") to demarcate the first 15 significant digits.)

You might ask:  how did this happen?

I presume that either you copy-and-pasted the value into F5, or you imported the value from an XML file.

[EDIT] Or you used a VBA macro to store a calculated VBA value into F5.  There are probably some other ways to do it that I cannot think of at the moment.  The point is....

In any case, you inherited the infinitesimal difference from the original source.

bob jones aka: cybertaz say:
I'm afraid your description really doesn't narrow down the possibilities very much :-) And I have no idea what "The text names are stair-stepped one below another (Indented as in an 'outline')" means, but here are some things to check...

The standard behavior for text is that it flow along the row as long as there is no content in the adjacent cells. Keep in mind that just because the content isn't visible doesn't mean it isn't there... I've seen similar situations because the user thought the 'correct' way to delete a cell's content was to select the cell & press the space bar. All that does is replace the visible content with a space which cannot be seen.

What happens if you select the cell you believe the text should flow across then look at the Formula Bar? -- Formulas can be written in a way that causes the cell to display nothing, but the formula still resides there. It's also possible that a font color such as White renders the cell content invisible in the sheet but it will display in the Formula Bar.

Once you're certain that the cell doesn't have any content you aren't willing to lose, go to Edit> Clear> Contents.

Another possibility is that the cell containing the text has the Wrap Text property applied to it. If the height of that cell's row has been manually adjusted it may not be tall enough to display the additional lines but the row height will not adjust automatically.

You also have the option of widening columns.

When I only reference one cell, it appears. Can you reference multiple cells from another worksheet in the same workbook in one cell?
hansv mvp say:

Would you like to concatenate the values of multiple cells?

If you just want the values behind each other, without anything in between, you can use a formula such as

=CONCAT(Sheet2!A2:A6)

If you want the values separated by commas, for example:

=TEXTJOIN(", ",TRUE,Sheet2!A2:A6)

or each value on its own line:

=TEXTJOIN(CHAR(10),TRUE,Sheet2!A2:A6)

(make sure that Wrap Text is on for the cell with the latter formula)
