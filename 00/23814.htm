ken sheridan say: You'll find an example of a query restricted  by multiple selections in a list box in DatabaseBasics.zip in my public databases folder at:

https://onedrive.live.com/?cid=44CC60D7FEA42912&id=44CC60D7FEA42912!169

Note that if you are using an earlier version of Access you might find that the colour of some form objects such as buttons shows incorrectly and you will need to  amend the form design accordingly.  

If you have difficulty opening the link, copy the link (NB, not the link location) and paste it into your browser's address bar.

In this little demo file the section on 'retrieving data from the database' incudes a form in which one or more contacts can be selected in a multi-select list box.  In the demo the query's results are output as a report with the following code:

    Dim varItem As Variant
    Dim strContactIDList As String
    Dim ctrl As Control
    
    Set ctrl = Me.lstContacts
    
    If ctrl.ItemsSelected.Count > 0 Then
        ' if at least one item has ben selected in the list box
        ' loop through the control's ItemsSelected collection
        ' and build a value list of ContactID values
        For Each varItem In ctrl.ItemsSelected
            strContactIDList = strContactIDList & "," & ctrl.ItemData(varItem)
        Next varItem
        
        ' remove leading comma
        strContactIDList = Mid(strContactIDList, 2)
        ' assign value list to hidden text box control
        Me.txtContactIDList = strContactIDList
        
        DoCmd.OpenReport "rptContacts_1", View:=acViewPreview
    Else
        MsgBox "No contacts selected", vbInformation, "Warning"
    End If

This code writes a list of the selected ContactID values to a hidden text  box, txtContactIDList, which is then referenced  as a parameter by the query as follows:

SELECT [FirstName] && " " && [LastName] AS FullName, Address, City, Region,
Country, Employer, LastName, FirstName, Contacts.ContactID
FROM (Countries INNER JOIN Regions ON Countries.CountryID = Regions.CountryID)
INNER JOIN (Employers INNER JOIN ((Cities INNER JOIN Contacts
ON Cities.CityID = Contacts.CityID) INNER JOIN ContactEmployers
ON Contacts.ContactID = ContactEmployers.ContactID)
ON Employers.EmployerID = ContactEmployers.EmployerID)
ON Regions.RegionID = Cities.RegionID
WHERE InParam(Contacts.[ContactID],[Forms]![frmReportDialogue]![txtContactIDList])=TRUE;

To reference the text box as a parameter the query uses the InParam and GetToken functions published by Microsoft, the latter called by the former.  These functions can be found in the basInParam module in the demo.

In your case you would need to modify the code so that, instead of opening the report, it exports the query's result table to Excel by calling the TransferSpreadsheet method of the DoCmd object.

hansv mvp say:

Take any select query in your database. It doesn't matter what it selects and from which table(s).

Copy and paste it.

Name the copy qryExport.

On the Create tab of the ribbon, click Form Design.

Place a list box on the form.

Set the following properties:

Column Count: 2

Column Widths: 0

Row Source: the name of the table that lists the states.

Name: lstStates

Multi Select: Simple (or Extended, if you prefer to select items the way you select files in File Explorer).

Place a command button on the form.

Set the following properties:

Caption: Export to Excel.

Name: cmdExport.

Activate the Event tab of the ribbon.

Click in the On Click event.

Select [Event Procedure] from the dropdown menu in this event.

Click the builder dots ... to the right of the dropdown arrow.

This will activate the Visual Basic Editor. You'll see code like this:

Private Sub cmdExport_Click()
 
End Sub

Make it look like this (you can copy/paste the code):

Private Sub cmdExport_Click()
    Dim strSQL As String
    Dim strIn As String
    Dim itm As Variant
    If Me.lstStates.ItemsSelected.Count = 0 Then
        MsgBox "Please select at least one state!", vbExclamation
        Exit Sub
    End If
    For Each itm In Me.lstStates.ItemsSelected
        strIn = strIn & "," & Me.lstStates.ItemData(itm)
    Next itm
    strSQL = "SELECT * FROM Contact WHERE ST_ID In (" & Mid(strIn, 2) & ")"
    CurrentDb.QueryDefs("qryExport").SQL = strSQL
    DoCmd.TransferSpreadsheet _
        TransferType:=acExport, _
        SpreadsheetType:=acSpreadsheetTypeExcel12Xml, _
        TableName:="qryExport", _
        FileName:="Export.xlsx", _
        HasFieldNames:=True
End Sub

Note: you can specify a different filename of course, and additionally, specify the path.

Save the form.

When you want to export, open the form, select one or more states from the list box, then click the command button.

What is smooth scrolling? - Microsoft Community

kennethbarry say:

What is smooth scrolling?

What is smooth scrolling setting and its effects?

***Subject edited for clarity.***

patricia yu say:

Hi Kenneth,

Smooth scrolling is a feature available in Internet Explorer that allows you to scroll through a web pages smoothly, as the name of the feature suggests. If this is turned off, when scrolling through a web page, you'll notice that the scroll bar slides down as if it's slightly delayed, but if this is turned on, the scroll bar slides down smoothly, without delay.

This is useful when scanning through long web pages or articles.

To turn on/off smooth scrolling, see the following steps:

  1. Open Internet Explorer, click the gear icon, and then click Internet Options.
  2. Click on the Advanced tab.
  3. Locate Use smooth scrolling, and tick the check box if you want it turned on/off.

Hi Kamil,

OneDrive sync client have three update rings. Insiders, Production, and Enterprise. Insiders is the first public ring. It takes about three days to roll our to this ring. You can join to Office Insider program if you want to try out the latest features as soon as possible.

The next is the Production ring. The updates are rolled out to some users in the ring first, and slowly roll the updates out to the rest of the users. It typically takes one to two weeks on this ring.

hansv mvp say:
PS are the words MACRO and SUBROUTINE interchangeable?

A subroutine and a procedure are the same thing:

Sub something(argument1, argument2, ...)
    ...
End Sub

A macro is a subroutine/procedure that has no arguments:

Sub something()
    ...
End Sub
