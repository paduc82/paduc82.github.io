why do my deleted emails stay in my inbox and don't go to trash?

brian tillman [outlook mvp 2007-2019] say:

What you describe is the proper behavior of IMAP accounts.  When you "delete" a message in an IMAP mailbox, the server just marks it as queued for deletion but it doesn't actually get deleted until you tell the server to purge the mailbox.  You can configure your account to purge the mailbox automatically when you exit Outlook.

It is also proper behavior for your IMAP account to have its own set of folders distinct from the default folder set.  IMAP operaties differently from POP.  All messages remain on the server by default and the folder set you see reflects what is contained on the server.  Outlook (and other IMAP clients) create folder sets that match the contents of the server.  Since IMAP cannot contain non-mail data, like Contacts, Calendar, Tasks, etc., those folders must be kept in a separate data file, which will be your default data file.

In summary, there is nothing to fix.  What you see is correct behavior for IMAP.

If you need to present these presentations, not only print them, you could use one overview slide with hyperlinks:

1. Create a new presentation that has only a title slide an one Title-only slide.

2. On the Title-only slide insert 15 rectangles (or other shapes)

3. Move the different PowerPoint files to the same folder as the overview presentation.

4. Rename the presentations to standard names, e.g. PresA.pptx, PresB.pptx, etc.

5. Insert hyperlinks on the overview slides, linking the shapes to the presentations.

6. When presenting, start the overview slide and use the hyperlinks, to move from file to file.

7. Next week, move the presentations to an archive folder. Copy the new weekly presentations to the folder of the overview slide.

8. Rename the presentations with the linked standard names.

9. If you receive less presentations than last week, make the surplus shapes invisible with the visibility pane.

steve rindsberg say:
Let's say you have a presentation called MAIN and fifteen others called A, B ... N, O.
All should be in the same folder.

Open MAIN.  Put any "permanent" slides you like into it, then add fifteen blank slides.
Open A, put it in Slide Sorter view, right-click the one slide in it and choose COPY.
Switch back to MAIN, go to the first blank slide, choose Paste Special and put a check next to LINK.  Click OK.
Resize the pasted linked slide to fill the blank slide you're pasting it onto.
Close A.

Repeat for the rest of the fifteen subsidiary slides and save MAIN.

Now when you open MAIN, the linked slides will update (after issuing a message from MS Legal about the impending end of civilization as we know it should you open a presentation with GASP! links in it).

Once it's been updated, and before sending it out, I'd go to File | Info and click Edit Links To Files on the right.  In the resulting dialog box, select all the links then click Break Links.

That will "lock" the presentation at its current state; no more updates from linked files, and no more silly warnings to alarm recipients of the file.

steve rindsberg say:
You should be able to extend the same idea ...

If Presentation A has 5 slides instead of just 1, you'd copy/paste/link each of the slides from A rather than just one.

This should work, so long as the information on the individual slides is edited but the number/order of slides doesn't change.

But if the owner of Presentation A (B, C, etc)  might delete some slides, add others or rearrange the slides in the presentation, then linking won't work.

Or a completely different approach ... with some VBA, you might be able to run an "Update" macro that starts with an open presentation, goes out to a specific directory, locates each presentation in it, and inserts all the slides from that presentation.

In other words, no links to update, but a simple way of re-creating a new presentation each time it's needed.
