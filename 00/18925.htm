Solving OneNote Sync issues - Microsoft Community

onno say:

Solving OneNote Sync issues

Recently, I started getting synchronization problems with MS OneNote 2010 (on Windows 7, latest updates installed d.d. 18 May 2015). I managed to resolve them, but since I haven't found these exact steps for resolving the issues elsewhere, I'm posting them here. Perhaps someone else will be helped by this!

Situation

Multiple OneNote notebooks, stored in a Dropbox folder.

On the problematic PC, OneNote is running inside a Virtual Machine (guest OS Windows 7), with the Dropbox folder being a shared folder with the host OS. Dropbox is actually running inside the host OS. (Not sure this is relevant, but just in case!)

Symptoms

On the problematic PC:

  • A notebook's synchronization status indicates "incomplete sync, error code "0xE000002E". On inspecting the details, one or more sections show an "error code 0x57".

On other PCs:

  • On some computers, some of the pages in the notebook appear empty, with a message "this page is not available yet" (something along the lines of the source computer not having synced properly yet).
    (Note that on the problematic PC, all pages appear fine).

Resolution

On the problematic PC:

  1. I split the notebook with the sync errors up into multiple smaller notebooks. I read somewhere that having very large notebooks may cause issues.
  2. I then exported the notebooks, one by one, to OneNote Single File Packages. Right-click each notebook, click "Save as...", choose "Current Notebook" in the "Page Range" list, and store the resulting *.onepkg file somewhere.
  3. Once all notebooks have been safely exported, close OneNote. Make sure you export all your notebooks, or things might disappear in the next few steps!
  4. Optionally, make a backup of OneNote's cache folder, before we're going to delete it. In the Start menu, choose "Run...", and enter "%localappdata%\Microsoft\OneNote\14.0". Backup the files and folders in here, they're going to be removed in the next step.
  5. Start OneNote in safe boot mode, by going to the Start menu, choose "Run...". Enter "onenote /safeboot" (as also described elsewhere on the internet), a under step 2, one by one.

After you've done step 6 for all exported files, you should keep an eye on the sync status of all notebooks. For me, these quickly returned to normal again. Problem solved, without data loss!

What did not work

What did not work for me:

  • Copying the problematic sections into a new notebook on the problematic PC. For me, these new notebooks immediately started to produce the same sync errors as the original notebook.
  • Clearing the cache: if you haven't exported the notebooks to OneNote Single File Packages before you do this, you might actually loose pages! These pages are stuck in the corrupted cache, so clearing the cache will make you loose them.

I hope this helps someone out there!

Best,

Onno
