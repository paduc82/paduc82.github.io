Hello again Jules,

I previously included the method of adding current date and time as an example but seems you do not want that so I have modified the code.

For validating the dates and times you have 3 options as follows:

1. Use Graham Mayor's suggestion to add the Date and Time picker so that no validation is really required. The down side of this is that the control must be added to every computer that the application is required to run on and there are reports of people still having problems loading and running it.

2. Use separate combo boxes to enter Year, Month and Day so that the Day combo list is populated after Year and Month is entered so that correct number of days is present for the selected Year and Month. Can accomodate Feb during leap years. The data is then concatenated to create the date.

3. Validate the TextBox entry as per the code below. It is not 100% foolproof in that if a user enters the date in d/m/y format and the day is equal or less than 12 then there is no way of validating this issue.

The code example actually alows the user to use 1 or 2 digit numbers for the month and day and either 2 or 4 digits for the year and validates the date and re-writes the date into the textbox as mm/dd/yy format. Similarly the user can use 1 or 2 digits for the Hour and Minutes and it is corrected by the code.

I have provided a complete copy of all the code so you can just delete the old code and paste in the new code.

Note the comments against the 2 Dim statements at the top of the code. These must be at the top of the module before any subs. Reason for this is to make them available throughout all subs in the module and are then used to write the date and time values to the worksheet.

You may be interested in the reasons for creating the 4 digit years as I have done in the code. Some more info at the following link. (See the section "Change the way two-digit years are interpreted")

http://office.microsoft.com/en-us/excel-help/change-the-date-system-format-or-two-digit-year-interpretation-HP010054141.aspx#BMchange_the_way_two-digit_years_are_in

 

Hope it all helps.

 

Dim dteDate As Date     'This line at top of module before any Subs
Dim dteTime As Date     'This line at top of module before any Subs

Private Sub txtDate_BeforeUpdate(ByVal Cancel As MSForms.ReturnBoolean)
Dim strMsge As String
Dim arrSplit
Dim lngDay As Long

dteDate = 0     'Initialize so does not hold date from previous entry
If Trim(Len(txtDate)) = 0 Then Exit Sub  'Required to allow deletion of date

'Following 'Creates zero based array using "/" as separators
arrSplit = Split(txtDate, "/")  'Creates zero based array using "/" as separators

'arrSplit should have 3 elements addressed as 0, 1 and 2 if 2 slashes present
'Therefore if top element is not 2 then does not have 2 slashes
If UBound(arrSplit) <> 2 Then  'Does not have 2 slashes
    strMsge = "Incorrect date format. Use slashes as separators."
    GoTo msgLabel
End If

'Test number of digits entered for year (Allows 2 or 4 digits)
If Len(arrSplit(2)) <> 2 And Len(arrSplit(2)) <> 4 Then
    strMsge = "Year must be entered as 2 digits or 4 digits."
    GoTo msgLabel
End If

'Convert 2 digit years to 4 digits (Can't validate without doing this)
If arrSplit(2) >= 0 And arrSplit(2) <= 29 Then
    arrSplit(2) = 20 & arrSplit(2)  '0 to 29 changed to 2000's
End If

If arrSplit(2) >= 30 And arrSplit(2) <= 99 Then
    arrSplit(2) = 19 & arrSplit(2)  '30 to 99 changed to 1900's
End If

If arrSplit(0) < 1 Or arrSplit(0) > 12 Then
    strMsge = "Invalid date. Month must be between 1 and 12."
    GoTo msgLabel
End If

'Find the last day of the month for year and month entered
lngDay = Day(WorksheetFunction.EoMonth(DateSerial(arrSplit(2), arrSplit(0), 1), 0))

If arrSplit(1) < 1 Or arrSplit(1) > lngDay Then
    strMsge = "Day must be between 1 and " & lngDay & " for year and month entered."
    GoTo msgLabel
End If
   
dteDate = DateSerial(arrSplit(2), arrSplit(0), arrSplit(1))

txtDate.Value = Format(dteDate, "mm/dd/yy") 'Re-write date in correct format.
'txtDate.Value = Format(dteDate, "mm/dd/yyyy") 'Alternative to Re-write as 4 digit years

Exit Sub    'If gets to here then date OK so exit here

msgLabel:
Cancel = True   'Cancels the update and cursor remains in the TextBox control
MsgBox strMsge & vbCrLf & "Re-enter date correctly in mm/dd/yy format."

End Sub

Private Sub txtTime_BeforeUpdate(ByVal Cancel As MSForms.ReturnBoolean)
Dim strMsge As String
Dim arrSplit

dteDate = 0
If Trim(Len(txtTime)) = 0 Then Exit Sub  'Required to allow deletion of time

arrSplit = Split(txtTime.Value, ":")

If UBound(arrSplit) <> 1 Then  'Does not have colon separator
    strMsge = "Incorrect time format. Use colon as separator."
    GoTo msgLabel
End If

If arrSplit(0) < 0 Or arrSplit(0) > 23 Then
    strMsge = "Hours must be between 0 and 23."
    GoTo msgLabel
End If

If arrSplit(1) < 0 Or arrSplit(1) > 59 Then
    strMsge = "Minutes must be between 0 and 59."
    GoTo msgLabel
End If

dteTime = TimeSerial(arrSplit(0), arrSplit(1), 0)

txtTime.Value = Format(dteTime, "hh:mm")    'Re-write time in correct format.
'txtTime.Value = Format(dteTime, "hh:mm am/pm") 'Alternative to Re-write as am/pm format

Exit Sub    'If gets to here then time OK so exit here
msgLabel:
Cancel = True   'Cancels the update and cursor remains in the TextBox control
MsgBox strMsge & vbCrLf & "Re-enter time correctly in hh:mm format."

End Sub

Function nextRow(ws As Worksheet) As Long
     'Finds the next blank row on the worksheet
     Dim rnglastRow
     With ws
       Set rnglastRow = .Cells.Find(What:="*", _
           After:=.Cells(1, 1), _
           LookIn:=xlFormulas, _
           LookAt:=xlPart, _
           SearchOrder:=xlByRows, _
           SearchDirection:=xlPrevious, _
           MatchCase:=False)
     End With
    
    If Not rnglastRow Is Nothing Then
         nextRow = rnglastRow.Row + 1
     Else
         nextRow = 1 'If sheet is blank then nothing is found
     End If
 
End Function

Do you have another monitor you could attach?  It sounds like your primary display has switched to it and it is at the Lock screen.  This happens inexplicably occasionally.  Having a second monitor to check on the possibility is very convenient and assists with any workarounds you may have to try.   For example, when this happens the display you are looking at would never change until you actually signed in again.  One thing you could try without another monitor would be starting Narrator (Win+Ctrl+Enter) so you could at least hear what is going on.  However, note that sound might not happen if the default audio device was associated with a non-functioning display device, e.g. on the same HDMI port.  Also, starting Narrator is a way to get past the Lock Screen and on to the Login screen where more things would be possible, including changing the Display configuration with a Win+P command.  For example, Win+P, End, Enter would choose Second Screen Only.  But then if that would cause a switch you would be asked if it was OK.  There is a ridiculously short time given to try to position a mouse pointer on the Yes for that even when you can see it so pressing Tab, Enter is what I do, even if I can't see it because sometimes I can't even switch to the correct screen before the timeout occurs.  Another command you may want to try is Win+Ctrl+Shift+B but in my experience success with this is variable.  Try pressing it more than once with psychic pauses if you hear it seeming to do anything.  One good thing about it is that it may (sometimes) switch the Display configuration back to PC Screen Only.

BTW note that a black screen instead of just a blank one would be equally possible from the above hypothesis.  E.g. all that would be required is to have the Display configuration set to either PC Screen Only or Second Screen Only and if necessary both video devices reversed.   For example, if you were successful with the above procedure and the PC screen turned black you would know that you were on the right track and that the system at least still knew which monitor was the "PC Screen".
