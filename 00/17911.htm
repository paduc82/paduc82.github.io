jay freedman say:

This macro cycles through all three markup types, in the order Simple -> All -> None -> back to Simple.

Sub CycleMarkup()

    With ActiveWindow.View.RevisionsFilter

        Select Case .Markup

            Case wdRevisionsMarkupSimple

                .Markup = wdRevisionsMarkupAll

            Case wdRevisionsMarkupAll

                .Markup = wdRevisionsMarkupNone

            Case wdRevisionsMarkupNone

                .Markup = wdRevisionsMarkupSimple

        End Select

        .View = wdRevisionsViewFinal

    End With

End Sub


jay freedman say:

Insert the line

ActiveWindow.ScrollIntoView Selection.Range

between the "End With" and the "End Sub".

If the location of the insertion point is off the screen, that statement will scroll the document so that place is visible -- usually at or near the top of the window. If the insertion point is already (still) on screen, the statement won't do anything.

suzanne s. barnhill say:

What you want is called AutoText. The article at http://word.mvps.org/FAQs/Customization/AutoText.htm, though old, explains the general principles. Here's how to do it in Word 2010:

  1. Type the text you want to save as an AutoText entry.
  2. Select the text.
  3. Press Alt+F3, which opens the Create New Building Block dialog.
  4. In the Name box, type the text you want to use as a trigger for the entry. You could name it "normal," but you might want to use a string that is not an ordinary word.
  5. Leave all the other settings as is unless you want the text inserted in its own paragraph, in which case, select that instead of "Insert content only."
  6. When you type the entry name in a document, you will get a popup (AutoComplete tip) telling you to press Enter or Tab to insert the entry.

For shorter strings, you can use AutoCorrect instead; see http://wordfaqs.mvps.org/AutoCorrect.htm. AutoCorrect is especially helpful in Word 2007, in which AutoComplete for AutoText is not enabled.

jennshauger say:

Powerpoint 2011 font issues!

Hello,

Any idea how to make sure when i send over a presentation, the fonts will always be installed?

Help! thanks!

john korchok say:

This is an example of why it's a really good idea to design in the same version your client is using, by means or Bootcamp or Parallels. As Steve has mentioned, font families work differently under OS X and Windows. OS X will happily gather a group of fonts and make one big family from it, but in Windows, the family must be constructed by the foundry and always has a maximum of 4 members.

The best idea is to visit your client and find out which Neutra weights are in their most used font family. This will probably be the Roman, Italic, Bold and Bold Italic, but you should check. Then on your Mac uninstall all the other weights. Set the base font in the font theme (here's my article that tells you how: XML Hacking: Font Themes). When you're creating the presentation, set the font dropdown to the theme font and leave it. The font dropdown will read Neutra (Headings) or Neutra (Body) when it's set to the theme font. Then access the other weights by using only the Bold and Italic buttons in PowerPoint. This will work 95% of the time, but you still need to test before final delivery.

BTW, your version of PowerPoint should be up to 14.6.3. Please use Help>Check for Updates to get the most recent version.
