andreas killer say:
The compression routine of the pictures is changed to provide a smaller file size. Therefore your code needs longer to execute with the default settings. But you can affect the compression, have a look at "File\Options\Advanced\Image Size and Quality"

But before you change these settings for your file, you should revise your code basically because IMHO it is slow and we can improve it a lot.

a.) I can not see any declaration, that means all your variables have the type VARIANT. Counting with such variables is slow, if you declare them as LONG your code works faster.
  Dim i As Long, n As Long

I recommend to place the line
  Option Explicit
at the top of your module, so you will not miss any variable.

b.) At the beginning of sub Burls you use a loop to extract the directory path, you can do the same with one line
  dirname = Left$(FileToOpen, InStrRev(FileToOpen, "\"))

c.) We can do the most of your code in that sub with a FOR loop and shorten it a lot.
d.) The use of the array ffilename is superfluous, because you never look back to a filename that you have stored in that array.
e.) Remove any code that affects the screen, ActiveWindow.SmallScroll in this part. It's superfluous and slow.

Here is an example for the first import part. Note: PlaceImage is changed, see next section.

  Dim RowPos, RowDir, ColPos, ColDir, FileMask
  Dim FName As String
  Dim r As Long, c As Long, rd As Integer, cd As Integer, Count as Long
  Const Up = -1
  Const Down = 1
 
  RowPos = Array(2, 53)
  RowDir = Array(Down, Down)
  ColPos = Array(6, 6)
  ColDir = Array(Down, Up)
  FileMask = Array("SL-lft*.jpg", "SL-Rgt*.jpg")
 
  For i = LBound(FileMask) To UBound(FileMask)
    FName = Dir(dirname & FileMask(i))
    r = RowPos(i)
    c = ColPos(i)
    rd = RowDir(i)
    cd = ColDir(i)
    Count = 1
    Do While Len(FName) > 0
      PlaceImage dirname & FName, Cells(r, c)
      c = c + cd
      Count = Count + 1
      If Count Mod columns_s = 0 Then
        Count = 1
        c = ColPos(i)
        r = r + rd
      End If
      FName = Dir
    Loop
  Next

f.) Your code will no longer work in Office 2010 as expected, because the Insert method has been changed. It no longer imports the image, but creates a link to the file. Use the ActiveSheet.Shapes.AddPicture method instead.
g.) Never use SELECT, SELECTION in your code, it's slow and error prone. Always access the objects directly.
h.) If you want to pass arguments to a sub, use BYVAL ever, except you want to get a result back.
i.) If you want to place a picture in a specific place, use the range object. So you can insert pictures without switch to that sheet.

Private Sub Example_PlaceImage()
  'Insert a picture in the active sheet
  PlaceImage "C:\test.jpg", Cells(2, 4)
  'Insert a picture in the second (not active) sheet
  PlaceImage "C:\test.jpg", Sheets(2).Range("D4")
End Sub

Sub PlaceImage(ByVal xfilename As String, ByVal R As Range)
  Dim S As Shape
  With R
    Set S = R.Parent.Shapes.AddPicture(xfilename, _
      False, True, .Left, .Top, 36, 48)
  End With
  R.Parent.Hyperlinks.Add Anchor:=S, Address:=xfilename
End Sub

j.) Again, never use SELECT, SELECTION in your code, this one is much more faster.

Lookup partial matches and return (multiple) row numbers - Microsoft Community

jw aldridge say:

Lookup partial matches and return (multiple) row numbers

I need to lookup H7 in column D, and return matching row numbers. Column D may have an added suffix. I am really trying to find partial matches. 

bernie deitrick say:

Array-enter (enter using Ctrl-Shift-Enter) in I7:

=IFERROR(SMALL(IF(LEFT($D$6:$D$1000,LEN($H$7))&" " = $H$7&" ",ROW($D$6:$D$1000)),ROW(A1)),"")

and copy I7 down until the formula returns blanks.

If you need the comma-delimited string then you can use a formula like this, extended to cover as many returns as you expect to have

=I7 & IF(I8<>"","," & I8,"") & IF(I9<>"","," & I9,"")
