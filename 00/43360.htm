Modifying the partitions cannot corrupt the BIOS in any way. This would require write enabling the BIOS which in most cases can only be done by the BIOS flash utility. Prove this by disconnecting the drive and booting, it should load into the BIOS OK. The BIOS knows nothing about partitions, it is only aware of entire drives.

 

Once the BIOS check is complete it then hands off boot to the 1st sector of the boot device to load the drive boot block. This is most likely where the problem lies.

 

Esc key is a little unusual to get into the BIOS, what make/model of PC is it? As far as I know only the DTK BIOS uses the ESC key, most others it is an F key or the Del key.

 

Did you reboot after formatting the D drive? I suspect the Windows BCD will still be looking for an O/S on the D drive as this will still be listed. You may need to connect the drive into another system to repair the MBR or use the repair function by booting from the Win 8 DVD
