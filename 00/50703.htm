The problem is that when you copy the bootloader, it is looking for 3 partitions on disk 0, and wants to boot from partition 3. Your new disk only has 1 partition.

    1. Click Start menu.
    2. Select the file or program that you want to always run in administrator mode and right click.
    3. Select Properties. (A new page will pop-up)

    Hi Colin

    The prompt that asks you to elevate a process with your administrator privileges is not designed to be irritating, it is designed to protect you. It's called the Least Privileged User component.

    If you have been using computers for a few years, then you will remember the Blaster Worm, the I Love You virus and many other major viral infections that infected tens of millions of computers worldwide and caused those millions of computers to lose their valuable data. The main reason this happened was because everyone ran Windows XP with an unprotected administrator account.

    These malicious files were received in emails, or downloaded to unsuspecting users computers by something as simple as clicking an internet link. Even the strongest antivirus software programs were not able to stop these infections, initially. The first thing these infections would do would be to to transparently take over the computer owners administrator account to gain access to the entire system.

    In Windows 7, all administrator accounts run as standard users unless the user (or an unknown component) initiates a process that requires administrator privileges and this is when the system will ask the logged on user for those privileges. This gives the control back to the user and allows you to approve any administrator elevation.

    Many of these same virus writers are still out there infecting older versions of windows and they are also using their Social Engineering skills to convince unwary users that the Vista/Windows 7, one-click solution is just there to irritate people, and too high of a price to pay.

    It is  your  computer, but only until you intentionally defeat the built-in security components. Then it will belong to them.

    Edit: If you must bypass the UAC prompt for a particular program that you run often, the information in the link that Andre.Ziegler posted is a good workaround, since setting this up for a program requires administrator priveleges. It still runs the program in a secure environment.

    Hope this helps.
