  Dim C As Range

Private Sub Worksheet_Change(ByVal Target As Range)

Dim sEmployee As String, sSheetName As String


  On Error Resume Next


  Const sTemplateSheet As String = "TIME SHEET TEMPLATE"

  Const sTemplateDescr As String = " Time Sheet"

ken sheridan say: The values, being alphabetic, are of text data type, so 'A10' sorts before 'A2'.  The best solution would be to store the data in two columns of text and integer number data types respectively.  You can then sort correctly.  With the data as it is, assuming the letter prefix is always a single character you can parse the values and format the numbers with leading zeros to enable a correct sort order.  You can see the difference in the debug window with:

? "A10" > "A2"
False
? Left("A10",1) & Format(Mid("A10",2),"0000") > Left("A2",1) & Format(Mid("A2",2),"0000")
True

john w. vinson say:

Ken's suggestion of having two fields is certainly the most correct way to go. An alternative is to parse out the number. Again, assuming one leading letter followed by an arbitrary number, you could use an Order By expression like

 

ORDER BY [fieldname], Val(Mid([fieldname], 2))

 

The Mid() function will return the substring from the second byte to the end (e.g. A11 will return "11"), and Val() will convert this text string to a number which will sort correctly.
