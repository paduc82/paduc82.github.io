ken sheridan say: The Format property or function returns a value which is the representation of a number as a percentage, so for this to be a percentage between 1 percent and 99 percent the number must be a fractional value of 0.01 to 0.99.  It appears that the value of Test%Comp is, in your example, already a fractional value of 0.2, so you do not need to divide it by 100.  You can see how in works in the debug window with:

? Format(0.2,"Percent")
20.00%
? Format(0.2/100,"Percent")
0.20%

A number as a percentage is merely the number multiplied by 100.  Again in the debug window:

? 0.2*100 & "%"
20%

To return a number as a percentage of another number the first number is divided by the second number and multiplied by 100, e.g.

? 22.5/90*100 & "%"
25%

 The Format function or property removes the need to divide by 100:

? Format(22.5/90,"Percent")
25.00%

But there is an important difference here in that the arithmetical expression 22.5/90*100 returns a value of 25, whereas formatting the value with Format(22.5/90,"Percent") merely presents the result of the expression 22.5/90, i.e. 0.25 as a percentage; the underlying value is still 0.25.

ken sheridan say:

You don't need to change the number at all, you need to format it, which is simply a matter of setting the Format property of a control bound to the original number column in a report or form to Percent.  

Do not format a number in an expression in the query by calling the Format function; that does change the value by returning a string expression.  You should only format a numeric value as the very last stage of presentation, whether that is in a report or form.  The underlying value should be left as the original number which in this case is a value between 0 and 1.  You can then do any necessary arithmetic on the values.

A simple example of this would be the net price of a product to which a tax, e.g. VAT, rated at 20% needs to be added to return the gross price.  The tax rate should be stored as 0.2.  If you need to display it as a percentage you'd simply set the Format property of the bound control in a form or report as Percent.  While it would show as 20.00%, the value of the control would be 0.2.  The gross price is then easily computed with the expression [Price]*(1+[TaxRate]).
