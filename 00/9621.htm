It's easy to close Skype accounts, however there are a few tips to remember ...

... all Skype accounts were transitioned to Microsoft accounts in September 2016

... closing the Skype account also closes its associated Microsoft account

... if the Skype account is linked or merged with a Microsoft account and that has services used or subscriptions such as One Drive, personal Office 365 subscriptions or XBox services, then please remember that closing the Microsoft account will close all of the other services currently in use

ken sheridan say: Combo boxes have never had a MultiSelect property.  List boxes have had this property for as long as I can remember, and still do.  What makes you think otherwise?

A combo box can be a control for a multi-valued field.  I, like many Access developers, feel that this means of representing  many-to-many relationship type by storing multiple values in a single column in a table should not be used, however, other than in the very specific situation where it is necessary to interface with a SharePoint list, but that the relationship type should be modelled in the conventional way by a table which resolves it into two one-to-many relationship types.

For an example of using a multi-select list box as means of representing a many-to-many relationship type, similar to the use of a multi-valued field, while still using a conventional model, see StudentCourses.zip in my public databases folder at:

https://onedrive.live.com/?cid=44CC60D7FEA42912&id=44CC60D7FEA42912!169

Note that if you are using an earlier version of Access you might find that the colour of some form objects such as buttons shows incorrectly and you will need to  amend the form design accordingly.  

If you have difficulty opening the link copy its text (NB, not the link location) and paste it into your browser's address bar.

This little demo file illustrates a number of interfaces for a simple many-to-many relationship type, including the use of a multi-select list box, but I would regard this as little more than an exercise in ingenuity when the conventional use of a form/subform is not only far simpler, but also allows the relationship type to have non-key attributes, as with the Status attribute in my example.
