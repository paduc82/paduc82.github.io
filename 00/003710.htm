First, I just want mention that when you force a reboot, it does interrupt the install, and it does, always roll back to your old build/OS, so there's no mystery there.

Next, when the install falters at the stage you have described, it's almost always because it's run into previously installed software that it will not tolerate ... and that's usually a 3rd party security program of some kind. Sometimes, it's a "registry cleaner", or that sort of thing that it finds to be objectionable.

We ask that all 3rd party security programs be cleanly uninstalled, which, in some cases, requires a special uninstaller provided by the manufacturer. (And, disabling it is not the same as cleanly uninstalling it.)

One person said, "no, I'm not going to uninstall it ... I've used it for years, and I fully trust it". I explained that it's not a question of trusting it, it's whether or not Win 10 sees it as compatible, so he reluctantly uninstalled it ... then, after the installation completed smoothly, he didn't want to talk about it.

mel77084 say:

Some of them can be reinstalled after installing the Windows upgrade, and will appear to be working, but upon reboot, cause failure. Others will reinstall and not cause complete failure, but may cause various malfunctions that usually get blamed on the OS, perhaps generating feedback that is misleading. 

If one that has caused install failure, and has been removed, it is very likely that, if reinstalled, it will also cause trouble with installing the next build, and need to be removed again.

Obviously, some are compatible now, so won't cause a problem at all; however, when a build install fails in late stages, it's quite likely that cleanly uninstalling the 3rd party security program is what's needed.

Note that each section can have up to three different headers (and three different footers), which is the case if you have selected "Different odd and even" and "Different first page" (also on the Header & Footer Tools Design tab). When you have multiple headers (footers), each type must be separately unlinked from its "neighbor" in the preceding section.
