When using IPv6 on Windows 7. If we do the following to get the IPv6 EUI-64 address:

netsh interface ipv6 set global randomizeidentifiers=disabled netsh interface ipv6 set global randomizeidentifiers=disabled netsh interface ipv6 set global store=persistent netsh interface ipv6 set global store=persistent It takes, but if you reboot, then the Radomize Identifier goes back to enabled.

Same thing happens when disabling the Privacy Setting for IPv6.

netsh interface ipv6 set privacy state=disabled netsh interface ipv6 set privacy state=disabled netsh interface ipv6 set privacy store=persistent
netsh interface ipv6 set privacy store=persistent

When it reboots the Privacy state goes back to being enabled.

The store=persistent command does not keep the information persistent
upon reboots. Keep in mind that this all works fine with Vista.
