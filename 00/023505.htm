marcochap say:

Hi

I had a similar issue.

By enabling the log in Lync

Menu Lync->Preferences->General

and when trying a new connection, I figured out the problem I had.

On my side, it was due to bad SSL Certificate authentication

To check if it's the same issue, launch a Terminal and do :

$ cat ~/Library/Logs/Microsoft-Lync.log

check if you see errors like :

"OnNotified, SSL Certificate Verification Failed - reason: 28"

If it's the case, first remove the OC_KeyContainer_XXX-db file in 

~/Library/Keychains/*

$ cd ~/Library/Keychains

$ mv OC_KeyContainer_XXX-db OC_KeyContainer_XXX-db.save

Then reinject the specific server certificates used to connect to your lync server(s).

(to be provided by IT or a friend that have it on his machine, could be linux/windows/.., there are the sames)

Then reinject these certificates manually inside Keychain access application, 

just click on the *.crt/*.pem files

Then for each certificate in Keychain access application, click on each certificate and in Trust section

"When using this certificate: "  Select Always trust

Then save It

and restart Lync and it should be ok..

Cheers

Marco

Hi,

AFAIK this cannot be done with CF. A CF formula applies formats  but does not change a value. You have to include it in the formula, e.g.

=IFERROR(ROUND(<your formula>,5),0)
