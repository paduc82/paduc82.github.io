ken sheridan say: My guess is that your report's RecordSource property is a query which references the cmbReportEmployee control as a parameter.  Because the combo box has no value at each iteration of the loop through its list the parameter is evaluating to Null, which is why the report is empty.

If I'm correct, I would suggest that you do not restrict the report's output by a parameter in the query but, when returning an individual employee's data via your first button do so by means of the WhereCondition argument of the OpenReport method, e.g.

DoCmd.OpenReport "Letter Report", _
    View:=acViewPreview, _
    WhereCondition:="EmployeeID = " & Forms!ReportOpener!cmbReportEmployee

assuming that EmployeeID is the combo box's BoundColumn.

When opening multiple re[orts with your second button do as Scott suggests and iterate through a recordset of all rows from the Employees table.  You can then reference the recordset's EmployeeID column to filter the report to the current employee, e.g.

DoCmd.OpenReport "Letter Report", _
    View:=acViewPreview, _
    WhereCondition:="EmployeeID = " & rst.Fields("EmployeeID")

where rst is the name of the object variable which returns a reference to the recordset.

Normally, when you open a spreadsheet in Excel, each row is numbered and each column has a header on top, such as "A", "B", "C", etc. If these are not displayed on your spreadsheet and you wish to turn them on or off, do the following:

1. Click "Tools" - "Options".
2. When the "Options" multi-tabbed dialog box appears, click the "View" tab.
3. Under "Window Options", check or uncheck "Row & column headers" as desired.
4. Click "OK" to close the dialog box.
