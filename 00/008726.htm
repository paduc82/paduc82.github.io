rob koch say:

That explains the why the process is being ran by system. One of them is being ran as a network service which has no service running which seams to be odd to me. 

When reading any technical explanation related to computers always concentrate on key statements rather then trying to find the specific answer to your question, since these will rarely precisely match.

From the article I linked the important points are as follows:

"It’s a completely legitimate executable—as long as it’s running from the system32 folder, and is signed by Microsoft."

"Checking it out in Process Explorer under Windows 7 shows that the conhost.exe process is running underneath the csrss.exe process."

"The conhost.exe process sitting in the middle between CSRSS and cmd.exe allows Windows 7 to fix both of the problems in previous versions of Windows—not only do the scrollbars draw correctly, but you can actually drag and drop a file from Explorer straight into the command prompt:"

So these indicate that any instance of conhost.exe will always have an instance of csrss.exe with which it is associated and also that the conhost.exe is legitimate as long as it's a Microsoft signed executable contained in the Windows\System32 folder.

Now this information is enough to remove any concern I would have, but that may be because I understand what the Network Service account is, so here's a relatively readable explanation of that account and why it exists as well.

http://windowsitpro.com/systems-management/understanding-local-service-and-network-service-accounts

Note that the most important section here is the following, since it indicates that the Network Service account has less authority than SYSTEM, so this actually reduces the potential for something using them to take control of the system to do something malicious.

"In earlier versions of Windows, most system services run under the powerful SYSTEM (aka Local System) account. Because most services don't need SYSTEM-level authority, Microsoft created the two new principals with less authority than SYSTEM and reconfigured many of the services on Windows 2003 and XP [or newer] to run as either Local Service or Network Service."

Based on the information you provided, my guess would be that you have something shared like a printer or file/folder on that PC and the second conhost instance being accessed via the Network Service is probably another computer or other device accessing that shared resource.

Note that none of this latter information changes anything about the fact that as long as both of the conhost.exe instances are referencing the Microsoft signed file found in the system32 folder then the file isn't malware, which is all that really matters.

robert19 say:
  • I am not sure this is what you are after, but it is working in Word 2007.

    1. Create a centered Heading 1 style with a multilevel numbering format. In the multilevel numbering setup dialog, choose to follow number with “nothing”.
    2. When this Heading 1 style has been assigned to all the appropriate paragraphs, do the following Find/Replace with Wild Cards ON across the whole document:

    Find What: *{1,}^13
    With Format: Heading 1 style

    Replace With: ^l^&

    This will replace any paragraph associated with the Heading 1 style with itself, while adding a soft return in front of it.

    Note that you might need to use *{1;}^13 instead as a Find What expression, depending on which list separator is current on your system (Windows Regional Settings).

    HTH.

IMAP sync bug Outlook 2016 - Microsoft Community

l3kris say:

IMAP sync bug Outlook 2016

Yesterday, seemingly out of the blue, my Outlook 2016 stopped IMAP syncing with the mailserver. Spending a lot of time searching online how to fix it (and not finding an answer per se although I got the general idea how to proceed), I hereby want to relay my solution in case others can use it and probably save some trouble.

My problem was that the inbox stopped syncing, while trash and sent folders synced alright. I knew that there was new mail because they showed up on my phone but not in my inbox.

Right-click on the inbox and select IMAP folders.

Press Query and let the list of folders populate.

Select Inbox and press Stop Subscription.

Press "Use" (or whatever the button to the right is named in English).

Select Inbox again and press Subscribe.

Press OK.

This fixed it for me. I hope it also fixes your problem.
