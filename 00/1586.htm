On the multiple selection issue: A document variable is capable of holding a single string value, nothing more. jay freedman say:

Answering the question in the comment line: It should be just oVars. Because the Dim oVars statement is inside the LBProduct.Click procedure rather than outside any procedure, the oVars variable is local to that procedure. The Me qualifier is needed only for variables or objects that are properties of the userform as a whole, not for local variables.

For unpacking the results of the userform, you can do something like this:

Sub x()
  Dim result() As String
  Dim productname As Variant
  UserForm1.Show
  result = Split(ActiveDocument.Variables("Product").Value, "|")
  For Each productname In result
      MsgBox productname
  Next productname
End Sub

The parentheses in Dim result() declare it as an array of unspecified size.

The Split function separates the pieces of the concatenated string into an array, specifying the "|" character as the separator (which is discarded).

The variable "productname" is declared as a Variant only because the For Each structure requires it; if you were going to walk the array by using an index variable, then productname could be declared as a String.

In place of the MsgBox, you would use the members of the array to construct your filename.

Happy hunting!
