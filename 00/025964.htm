Hi Dr._Rod_McB,

From my understanding, it seems you were viewing message headers of the received emails. As far as I know, the message header doesn't include content written by composers. People can only compose in the message body. For your reference, see the instructions of message format in Wikipedia.

Internet email messages consist of two major sections, the message header and the message body, collectively known as content. The header is structured into fields such as From, To, CC, Subject, Date, and other information about the email. In the process of transporting email messages between systems, SMTP communicates delivery parameters and information using message header fields. The body contains the message, as unstructured text, sometimes containing a signature block at the end. The header is separated from the body by a blank line.

Also, you can find the definition of email in RFC 5322. So you can see the message header is apart from message body, and it works like an envelope of the mail, we cannot modify information in it except From, To, CC, Subject, Date. Meanwhile, it's not human-readable cause it's designed as machine-readable. 

However, if you'd like to find the original sender of this email, you can paste the message header to Microsoft Message Header Analyzer website to help you do the analysis. It can help you have a clear view.

Besides, Office 365 provides message trace feature which can find all inbound and outbound emails records within Exchange server. For your reference, you can learn more details of message trace via the following link. Kindly note that it requires Office 365 admin account in your organization to perform the operation.

https://docs.microsoft.com/en-us/office365/securitycompliance/message-trace-scc
