Entering Dashes - Microsoft Community

jett1013 say:

Entering Dashes

Does anyone know how to enter a dash in between each letter of a word?

Ex:  Home               H-o-m-e

ronrosenfeld say:
On Wed, 8 Dec 2010 15:15:20 +0000, jett1013 wrote:
 
>
>
>Does anyone know how to enter a dash in between each letter of a word?
>
>Ex:  Home               H-o-m-e
 
Kind of depends on exactly what you mean by a "word"
 
And what you want to have happen if there are multiple words in a
cell.
 
Here's a macro that will place dashes between the letters of each
word.  A word, in this instance, is defined as a sequence of
characters that includes letters, digits or underscores.
 
If there are multiple words in a cell, it will not put a dash after
the last character of a word.
 
To enter this Macro (Sub), <alt-F11> opens the Visual Basic Editor.
Ensure your project is highlighted in the Project Explorer window.
Then, from the top menu, select Insert/Module and
paste the code below into the window that opens.
 
To use this Macro (Sub), first select the range of cells you wish to
process.  Then<alt-F8> opens the macro dialog box. Select the macro by
name, and <RUN>.
 
======================================
Option Explicit
Sub InsertDashes()
    Dim rg As Range, c As Range
    Dim re As Object
Set rg = Selection
Set re = CreateObject("vbscript.regexp")
    re.MultiLine = True
    re.Global = True
    re.Pattern = "(\w)(?!\s*\b)"
For Each c In rg
    c.Value = re.Replace(c.Text, "$1-")
Next c
End Sub

pbike908 say:

I think I figured out the problem.  For some reason, at least on the Droid Turbo 2, Android contact managers will display duplicate data within some contact records if the same outlook.com account is connected in multiple apps.


Android contact apps provide an option as to which contacts one wants to display.  So, if I have a corporate outlook account (built in mail app), Outlook app (Accompli), and Nine connected and I tell the built in (or many 3rd party) contact managers to only display Nine contacts (or corporate only or outlook only), duplicate information will appear within some records.  If I delete the other outlook.com apps meaning I only have the Outlook.com connected in a single app then everything appears to work fine.

When I am referring to multiple accounts, I mean the same Outlook.com account connected in multiple email/contact manager apps.

Talk about strange.  I guess Android contact managers just can't filter out the data individually if one has the same outlook.com connected to multiple apps on the same device -- at least the Droid Turbo 2.

The only reason why I had it connected multiple times is I was testing out different email clients.
