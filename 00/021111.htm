...

A general reason, external drives can be easily lost, losing the application.

Note, there are 2 general types of application. Ones that require an "install" process to put files in the right location(s) and to create registry entries. This type of application does not react well when the application drive cannot be found.

The other type is called "portable". They are "self contained". They typically come in a compressed ZIP (or equivalent) format file. All you have to do is "unzip" them and they are ready to run. They do not store anything in the registry, they use things like .ini files to hold their "initialization" or configuration options and settings. These applications are totally suited to be run from external drives. Many small utilities are written this way. Back in 2010, even Office Starter had an "Office-to-Go" option that installed on external drives. MS quickly killed that. The same with Windows-to-Go, it is gone too.

******************

When (DOS) computers moved from running from floppy disks or tape drives to HD's, the disks had very low storage in a very large package, measured in KB. And there typically wasn't room to install a second HD.

Extreme example:
https://www.techspot.com/trivia/98-first-5mb-ha...
The original IBM Mainframe disk drive stored approx 5MB (yes MB) in a package that weighed over 1TON! and used 50 x 24inch platters for that storage. Compare that to the $5, 8GB thumb drive you have in your pocket. The one that stores over a million times as much data!

Granted, by the time computers evolved to desktop PCS, the drives were much smaller with similar or SMALLER capacity, that is 8MB NOT 8GB!.

This state continued as HD capacity increased (while physical size stayed about the same, approx the size of a thick paperback) and MS moved from DOS to Windows. The ability, the hardware, the space to install multiple drives were still reserved for expensive business "server" class computers. It's only been the last 20 years or so that consumer class devices could support multiple physical drives. So everything in Windows has been hard coded, aka "hard wired", to only look to the C: drive for the OS and Applications. And most "installed" applications are designed with that bias. Even MS Office is written that way now. Yes, prior to the "Click to (not) Run" environment and Office 365 there was a "custom install" option to move some, NOT ALL, of the programs to a drive other than C:\.

So with all of that built in programming bias, it is not a good idea to "install" applications that are tied to the OS on removable drives. The OS and applications tend to NOT handle it gracefully when the removable drive cannot be found / is not connected..

It has only been the last few years, 2-3, that all of the hardware, including drives, has shrunk enough to be truly hand-held. That started back around the time of Windows 7 and 8. They were significantly smaller than Windows 10, so MS and HD industry settled on 20-32 GB as being "adequate" for these small portable devices. But by the time Win 10 came along, although technically 32GB was adequate by MS's "minimum specifications", in practice that is no where near big enough to be practical in real world use.
