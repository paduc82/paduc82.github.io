You could change the location of the startup folder, though that is not likely to go down well with the genius who came up with that restriction; or you could put the code in the user's normal template, which should be accessible, even if it is on the C: drive or Word cannot function correctly.


As for restricting the code to particular documents, you need to determine what is common about those documents e.g. all created from the same template, or list them in the macro as below and restrict the function to those documents e.g.



Option Explicit
Public WithEvents oApp As Word.Application

Private Sub oApp_DocumentBeforeSave(ByVal Doc As Document, SaveAsUI As Boolean, Cancel As Boolean)
Dim theAns As VbMsgBoxResult
Dim toc As TableOfContents
    Select Case Doc.name
        Case "DocumentName1.docx", "DocumentName2.docx", _
             "DocumentName3.docx", "DocumentName4.docx"
            '
            theAns = MsgBox(Prompt:="Do you want to run the spell checker before saving", _
                            Buttons:=vbYesNo + vbQuestion, _
                            Title:="Spell Check?")
            '
            ' Run the spell checker?
            If (theAns = vbYes) Then
                Doc.CheckSpelling
            End If
            '
            ' Always try to regenerate the Tables of Contents
            '
            For Each toc In Doc.TablesOfContents
                toc.Update
                ' toc.UpdatePageNumbers ' (alternate - update page numbers only)
            Next toc
        Case Else
            Exit Sub
    End Select
End Sub

Why am I not receiving the latest Windows 10 Insider Preview - Microsoft Community

anannya podder say:

Why am I not receiving the latest Windows 10 Insider Preview build?

Why am I not receiving the latest Windows 10 Insider Preview build?
vidyaranjan a v say:

Updated (7/9/15)

There are a number of reason why you might not receive the latest build of Windows 10 Insider Preview:

  1. You must be running a connected account with the Microsoft Account (MSA) you used to register for the Windows Insider Program.  You can find more information about this new requirement

  2. Make sure you’ve installed all the latest Windows Updates.  If you are running a Core Edition of the Insider Preview, you must install KB3070677 before Windows will detect new preview builds. 

  3. If the setting for how preview builds are installed is set to Slow, you will not receive the more frequent Fast builds as they are released.  (This option can be found under Settings > Update & Recovery > Advanced options.)

  4. If you switch that setting from Slow to Fast or back again, it can take up to 30 minutes before that setting results in a new build being offered.

  5. If you have just installed Insider Preview from media, then it may take up to 24 hours to receive the next build.

  6. If you are signed into a user account that joined your device to an Azure Active Directory (AAD), you won’t be able to receive new builds.  If this applies to you, you’ll need to create an additional administrator user account for the device through Settings > Accounts > Other Users.  The new account should be the Microsoft Account (MSA) you used to register for the Insider Program, and must not be an AAD account already added to the device.  You will then be able to configure Insider Preview to receive new builds through this additional user account.

Additionally, please verify you have done the following:

  1. Go to Settings -> Update & Security -> Windows Update

  2. Click “check for updates”

  3. No error appears; UI just says “Your device is up to date”.

If none of the above conditions apply to you and you have manually checked for updates, then please post your current build configuration , including the OS architecture (x86 vs. x64) and which OS language version you installed, as well as your installation history in this thread so we can dig into any potential problems.

(Example:  :  I upgraded from Windows 8.1 Pro to the x64 version of Build 10130 from media two weeks ago, using the English(United Kingdom) ISO file download.)

jasles say:
Hi HeartlessAU,

I'm an independent advisor here to help.

Try resetting the OneDrive sync app:

1. Press these two keys on your keyboard at the same time: Windows key and the R key. This will open the Run dialog box on your screen
2. Copy and paste this text into the Run window and press Enter:

%localappdata%\Microsoft\OneDrive\onedrive.exe /reset

That will reset OneDrive
After a couple of minutes, the OneDrive cloud icon should show in the system tray.

If it doesn’t start, Click on Windows Start, then find OneDrive and start it
