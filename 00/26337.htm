Hello,

 

I have fixed it.

 

Thank you for your quick reply, expected to take longer - good work guys :).

 

I managed to validate my domain after trail and errors.

 

For everybody's record, my problem was that even though I made the DNS modifications the domain ownership was still not validated.


The issue was that I just added a preferred MX to the domain as indicated by Microsoft (*removed for privacy*). It was preferred MX, with Priority 10 as recommended by Microsoft and updated properly.

 

Trick is that I didn't deleted my old MX record, who was now secondary MX with priority 15 just as a backup.

 

Didn't see this coming, but Microsoft's automated domain validation script queries for the MX records and even though the first one  is *.pamx1.hotmail.com and is ok, it returns an error when it sees the second MX.

 

When I deleted second MX and left only the one indicated by Microsoft, domain got validated without a problem.

 

I would consider this a bug or unexpected behavior, as there is no indication in the validation page or the help pages (nor I have found it in these forums) regarding the fact that the Microsoft MX must be the *only* MX, not the one with highest priority.

 

Of course after the validation, I could configure back my domain with a backup MX as well, since there is a reason after all for those proprieties :).

 

So my fix to validate my domain was:

Delete all MX records (even if they have low priority)

Insert Microsoft MX with priority 10 and check with some 3rd party tools that it's the only MX associated for the domain.

Ownership validation now works.

I found the culprit. OneDrive which I do not use was evidently tacking a lot of resources.

I disabled OneDrive as  follows:

  1. Press Win + R keyboard accelerator to open Run dialog box.
  2. Type GPedit.msc and hit Enter or OK to open Local Group Policy Editor.
  3. Navigate to Local Computer Policy -> Computer Configuration -> Administrative Templates -> Windows Components -> OneDrive.
  4. In the right pane, double click on policy named Prevent the usage of OneDrive for file storage.
  5. Select the Enabled radio button.

All is good now.
