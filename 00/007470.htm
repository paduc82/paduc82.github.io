barb way say:

Depending on how you created your custom shapes, it is possible that one of the ShapeSheet cells in each of  the Geometry sections is disabling the fill property.  I just created a couple of shapes by using the "Combine" operation, and sure enough - no fill.  If I create the shape using the "Join" operation, though, it fills just fine. 
Combine simply puts all of the geometry sections into one shape, while Join actually creates a single Geometry section in a single shape which has all of the lines/objects.  So you should try using Join to create your shapes, if possible.

It is possible to make the Combined shapes work, however.  These steps are not for the casual user, but I was able to fix my own shapes this way:

To fix the shapes, I opened the ShapeSheet for each one, and went to each Geometry section.

I looked at the first cell in each Geometry section (Geometryx.NoFill), and found they were set to "True".

I edited the cell values to replace with "=False" (without the quotes) and saved.

Then I could add Fill color to the shapes :-)

Tip: To get to the ShapeSheet for a shape, enable Developer Mode (File tab / Options / Advanced / Run in Developer Mode).  Then you can right click on a shape and select "Show ShapeSheet" and see all of the detailed properties for each shape.  Do no change cells randomly, as you might cause your shapes to behave badly in various ways!  I would not normally suggest you use the ShapeSheet, and only offer this in case it will be an easier way to update your existing shapes.

Hope that helps!

x58456 say:

Actually, after playing with it for a while, here is what I have observed seems to be the cause.

The method by which the shape is originally constructed in the shape editor is critical.  It appears Visio will not reorder the vertices to create a region as expressed by a geometry page under the ShapeSheet.  If the order jumps around a bit then the region cannot be filled.  For example, to draw an oval requires six elements: four arcs and two line segments.  If they are numbered counterclockwise, then constructing the oval by drawing the elements in the order 1,5,2,4,6,3 will not yield a filled shape.

Therefore, planning the shape drawing is critical.

1.  Decompose the target shape into primitive filled elements and complex, non-primitive filled elements which can be used to create the final shape.  Primitive filled elements are those which Visio already provides, such as ellipse and rectangle.

2.  Begin with the most complex, non-primitive filled element and construct it using the non-primitive line elements (line, arc, and so forth) in a clockwise direction.  Once the final stroke is layed, the irregular shape should fill itself.  Delete the element and repeat this step until it is filled.

3.  Proceed to the less complex non-primitive filled elements and then finish the shape using the primitive filled elements.

4.  A final "Join" operation will ensure the shape is filled cohesively.

john visio mvp say:

The problem is that Visio deals in Geometry Sections rather than line segments. What may appear to a user  as four lines that make a box may be several Geometry sections. A Geometry section that contains two parallel lines, could be filled by setting the NoFill property of the Geometry section to false. A Geometry section that contains a closed loop can be prevented from being filled by setting the NoFill property to true. 

This will require going into the Shapesheet and playing with the Geometry section.

So the behaviours observed are correct, but you need to look at the shapesheet to understand when the behaviour is contrary.
