master_thovy say:

Pinging a network drive is one thing and mapping it is another. Pinging uses TCP/IP protocol, while Mapping requires TCP protocol and NFS Protocols. 

  • Verify if NFS protocols are enabled on your network drive (by default it is already enabled in Windows 10).
  • What network drive are you using (NAS, external hard drive, etc)

jackin say: Accessing file server located geographically in different state, but both computers are on same domain/connected to same network.        

master_thovy say:

Might be an SMB issue.

Try the following step:

  1. Open Registry Editor (Windows+R > type regedit).
  2. Locate HKEY_LOCAL_MACHINE/SOFTWARE/Microsoft/Windows/CurrentVersion/Policies/System. 
  3. Create a new DWORD entry with the name EnableLinkedConnections and value 1. (Edit > New > DWORD))
  4. Open Command Prompt in Admin mode and perform the following command:

sc.exe config lanmanworkstation depend= bowser/mrxsmb10/nsi

sc.exe config mrxsmb20 start= disabled

5. Restart you computer.

Check now if you can map your server address.

Windows 8 and Windows 7 share the same driver model. Windows 8 comes with a standard set of class drivers for common hardware devices.
