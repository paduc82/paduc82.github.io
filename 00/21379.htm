split unique ID codes, from col L, in separate new sheets

[update Mar. 01, 2017]

 Sub SplitData_AutoF()
'Feb 25, 2017
Const N As Integer = 11 '<< headers in row 11
Const sCol$ = "L" '<<< search in column L
Const shN$ = "Report" '<<< Source Sheet Name
Dim ws As Worksheet, ws1 As Worksheet
Set ws = Sheets(shN)
Dim r As Long, x As Long, i As Long
Application.ScreenUpdating = False
ws.AutoFilterMode = False
r = ws.Cells(Rows.Count, 1).End(xlUp).Row
'
'step1 \ delete old sheets ###
Application.DisplayAlerts = False
For Each sh In Sheets
If sh.Name <> shN Then sh.Delete
Next
Application.DisplayAlerts = True
'
'step2 \ export unique ID codes, from col L, by ascending order ###
Dim c As New Collection, cItem
Dim v As Variant, vv As Variant
On Error Resume Next
For Each cItem In ws.Range(sCol & N + 1 & ":" & sCol & r).SpecialCells(xlCellTypeConstants)
c.Add cItem, cItem
Next
ReDim v(1 To c.Count)
For i = 1 To c.Count
v(i) = c(i)
Next
For i = 1 To c.Count
For x = i + 1 To c.Count
If v(x) < v(i) Then
vv = v(i): v(i) = v(x): v(x) = vv
End If
Next x
Next i
'
'step3 \ split data  in new sheets ###
For Each vv In v
ws.Range(ws.Cells(N, sCol), ws.Cells(r, sCol)).AutoFilter Field:=1, Criteria1:=vv
Set ws1 = Worksheets.Add(after:=Worksheets(Worksheets.Count))
ws1.Name = vv
ws.Rows(N & ":" & r).SpecialCells(xlCellTypeVisible).Copy
With Range("A1")
.PasteSpecial Paste:=xlPasteFormats
.PasteSpecial Paste:=xlPasteValues
End With
Application.CutCopyMode = False
ActiveSheet.UsedRange.EntireColumn.AutoFit
Next

'

Sheets(1).Select
ws.AutoFilterMode = False
Application.ScreenUpdating = True
End Sub

 Just make sure the audio file is:

  • In MP3 or WMA format.
  • Less than 40 seconds.
  • Less than 1 MB.
  • Not protected with digital rights management (DRM).

To make any file that fits these criteria into a ringtone, all you have to do is assign it the genre of ringtone. If you have a PC, you'll use the Zune software. If you don't have the Zune software on your computer, see Install the Zune software. On a Mac, use iTunes to assign the genre, and then sync the file using the Windows Phone Connector for Mac. To learn more, see Sync and get updates with my Mac.


To create a ringtone using the Zune software:
  1. Open the Zune software and find the file you want to turn into a ringtone.
  2. Right-click the file, and then click Edit.
  3. In the GENRE list, click or type ringtone, then click OK
  4. Connect your phone to your computer using a cable

    mbizup say:

    To do this, you really only need to properly set up your query and the sorting/grouping in your report's design.

    1.  Assuming that your date field is a time stamp, add a field to your query to catch the date-only portion of that field, without the time:
           FormattedDate: Format([YourDateField], 'yyyy-mm-dd')

    2.  Add your new FormattedDate field to the report's Sorting/Grouping, and include the options for both a group header and a group footer.

    3.  In the group Header section, add a textbox for the "Day Start" mileage, with the control source as follows (include the = sign):

       = Min([Mileage])

    4. In the group Footer section, add a textbox for the "Day Start" mileage, with the control source as follows:

       = Max([Mileage]) 

    4.  Also in the group Footer section, add a textbox for the "Total for the day" mileage, with the control source as follows:

       = Max([Mileage]) -  Min([Mileage])

    5.  For a grand total, add a textbox to the Report footer section with a control source as follows:

        = Max([Mileage]) -  Min([Mileage])

    (The calculation in the GROUP footer is 'per day'; the calculation in the REPORT footer looks identical but spans all of the days included in the report)

    scottgem say:

    I would do it a little differently. 

    In your query, I would add a column like so:

    CalcMiles: IIF([PurposeofVisit] = x OR [PurposeofVisit] = y,[Mileage],0)

    x and y would be your codes for Day Start and Day End. Then use this column to display the mileage. 

    ken sheridan say: The SQL for a query to compute these sort of consumption values is pretty standard:

    SELECT V1.Location, DATEVALUE(V1.[Date]) As VisitDate, V1.Mileage,
    V1.PurposeOfVisit, V1.Mileage -
         (SELECT MAX(Mileage)
          FROM Visits AS V2
          WHERE DATEVALUE(V2.[Date]) = DATEVALUE(V1.[Date])
          AND V2.[Date] < V1.[Date]) AS VisitMileage
    FROM Visits AS V1
    ORDER BY V1.[Date];

    This will return the mileage between the current location and the previous one in each row apart from the first per day.  Note that these sort of queries do rely on distinct date/time values, so your 'Date' column must store the full date/time value in each row, not merely the date.

    Building a report based on such a query is a simple task.  Group the report by the computed VisitDate, and then order it by [Date].  Hide the Mileage control in the detail section's Format event procedure with:

    Me.Mileage.Visible = (Me.PurposeOfVisit = "DayStart" Or Me.PurposeOfVisit = "Day End")

    In the group footer Sum VisitMileage to give the mileage per day, and do the same in the report footer to give the overall mileage.

    But don't use Date as a column name in a table.  It's the name of a built in function, so as a 'reserved word' should not be used as an object name.  If you do use it be sure to wrap any reference to it in SQL or code in square brackets.  However, I'd strongly advise that you change it to a more specifically descriptive term such as VisitDateTime.

    BTW if using a query as a report's RecordSource you can ditch the ORDER BY clause. A report uses its internal sorting and grouping mechanisms and will generally ignore a query's ORDER BY clause, which will only slow things down.

    Something like DNEXT?

    In other words if I have a table like:

    Any query/VBA UDF to return the following or similar recordset?

    0001 | ABC. | 1000. | 2000
    0002 | ABC. | 2000. | 1500
    0003 | ABC. | 1500. | 200
    0005 | ABC. | 200. | 7000
    0009 | ABC. | 7000. | 6000
    0010 | ABC. | 6000. | <Null>
    0004 | DEF. | 700. | 1000
    0006 | DEF. | 1000. | 800
    0007 | DEF. | 800. | 100
    0008 | DEF. | 100. | <Null>
    RecID may be used as to be in an ascending order but might have gaps due to deletion of records
    Thanx in advance 
    hansv mvp say:

    Let's say your table is named tblData. The following query will return all fields from this table plus a "next amount" field:

     

    SELECT tblData.*, (SELECT T.RecAmt FROM tblData AS T WHERE T.RecID = (SELECT Min(S.RecID) FROM tblData AS S WHERE S.Customer=tblData.Customer AND S.RecID>tblData.RecID)) AS NextAmt
    FROM tblData;

     

    hansv mvp say:

    An alternative SQL using DLookup and DMin instead of subqueries:

     

    SELECT tblData.*, DLookUp("RecAmt","tblData","RecID=" & Nz(DMin("RecID","tblData","Customer='" & [Customer] & "' AND RecID>" & [RecID]),0)) AS NextAmt
    FROM tblData;

     

    With a UDF:

     

    Function DNext(ReturnField As String, TableName As String, KeyField As String, _
            Optional WhereCondition As String) As Variant
        Dim NextKey
        NextKey = DMin(KeyField, TableName, WhereCondition)
        If Not IsNull(NextKey) Then
            DNext = DLookup(ReturnField, TableName, KeyField & "=" & NextKey)
        End If
    End Function

     

    The SQL then becomes

     

    SELECT tblData.*, DNext("RecAmt","tblData","RecID","Customer='" & [Customer] & "' AND RecID>" & [RecID]) AS NextAmt
    FROM tblData;

     

    These two methods will still be slow if you work with a large number of records, but they should be much faster than using subqueries.
