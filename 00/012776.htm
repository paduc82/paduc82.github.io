combo box in access report - Microsoft Community

hira iftikhar say:

combo box in access report

I am making a simple school project in which i need to make a report from data

Can you tell me is there any way i can make a combo box in place of account name and by selecting that name report will only display credit debit and balance for that account. 

runningmanhd say:

Yes you can. 

On the form that would open your report, do the following:

  1. Create a combo box to display the account names.
  2. Open the Properties dialog for the new combo box.  With the combo box selected, use the keyboard and press Alt+Enter. 
  3. Select the Data tab.
  4. In the property Row Source, enter a select query that selects your account id and account name.  This can either be a saved query object or SQL.  An example of the SQL would be:

    SELECT AccountID, AccountName FROM Account ORDER BY AccountName;
    Note:     You may need to substitute your actual table and field names in the SQL.

  5. In the property Row Source Type, make sure that the value is set to Table/Query (default).
  6. In the property Bound Column, ensure the value is 1 (default).
  7. In the property Limit To List, ensure the value is set to Yes.
  8. Select the Format tab.
  9. In the property Column Count, type 2.
  10. In the property Column Widths, type 0.
  11. Select the Other tab.
  12. In the property Name, type cbxAccounts.
  13. Close the Properties dialog.  Using the keyboard, press Alt+Enter.
  14. Create a command button next to your new combo box: cbxAccounts.
  15. Open the Properties dialog again.  With the command button selected, use the keyboard and press Alt+Enter. 
  16. Select the Format tab.
  17. In the property Caption, type Open Report.
  18. Select the Other tab.
  19. In the property Name, type cmdOpenReport.
  20. Select the Event tab.
  21. In the property On Click, type [.  "[Event Procedure]" will automatically display in the field.
  22. Open the VBA editor.  Using the keyboard, press Ctrl+F2.  The VBA editor will be displayed with a new procedure for your command button created, Private Sub cmdOpenReport_Click(). 
  23. Add the following code for your new procedure:

Private Sub cmdOpenReport_Click()
   
    On Error GoTo Err_Process
   
    If (Not IsNull(Me.cbxAccounts)) Then
        DoCmd.OpenReport "your report name", acViewPreview, , "AccountID=" & Me.cbxAccounts
    Else
        MsgBox "You must select an account." & vbCrLf & vbCrLf & "Please try again.", vbExclamation, "No Account Selected"
    End If

Exit_Process:
    Exit Sub
Err_Process:
    MsgBox "Procedure: cmdOpenReport" & vbCrLf & vbCrLf & Err.Number & " " & Err.Description, vbExclamation, "Error"
    Resume Exit_Process
   
End Sub

Note: You may need to edit the procedure to substitute the statement "AccountID=" with your actual field name in your Account table.  You will also need to edit the statement "your report name" to add your actual report name.

  1. On the menu bar, select Debug>Compile ....
  2. On the menu bar, select File>Save ....
