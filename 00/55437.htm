programatix say:

I finally resolved the issue. The Microsoft.Windows.ManagementUI.CombinedControls.NetworkListManager.GetNetworkList gives the cue to where the problem lies. I actually tried debugging it and when the FileNotFound exception is thrown the filename is actually null.

To resolve the problem,

  1. Run regedit
  2. Go to HKLM\Software\Microsoft\Windows NT\CurrentVersion\NetworkList
  3. Delete everything under the following path,
    • Nla->Cache
    • Nla->IntranetEnabled
    • Nla->Wireless
    • Profiles
    • Signatures->Managed
    • Signatures->Unmanaged
  4. Reboot

Task Scheduler should be running fine now. Please note that all network location awareness profiles has been reset and you'll need to reconfigure it (public and private network thing).

stephen stanley say:

I eventually found a solution for this - it worked for me but may not work for you:

Symptoms:

Start Task Scheduler - it would open up and show the top of the task tree hierarchy with an egg timer - then crash with the error: "MMC has detected an error in a snapin and will unload it"

 

If you run

$ schtasks /query /FO CSV > outputfile.csv

This will generate error messages such as

The system cannot find the file specified

...but will also generate a csv file listing all the tasks that task scheduler finds in the registry

You will find more details of the error in the csv file including the GUID {a long series of numbers between curly brackets}  of the tasks that is causing the problem

To fix them, run regedit and search for the GUID.  You should find two entries that match in the task scheduler hierarchy. Export the root of each matching key to a safe place and repeat for each of the files not found errors

If you want to try to recreate the tasks that were failing then before you delete them you should

a) additionally, run the query above with the /V modifier to get verbose output to a second csv file

b) copy the folder hierarchy /windows/system32/tasks to a safe place

You can then try to identify which tasks have been deleted by comparing the three sets of data to see what maps to the entries that have missing files

In my case, the issue was caused by me removing a couple of user accounts who had scheduled tasks associated with their user ids
