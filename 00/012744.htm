john w. vinson say:

One way would be to use a Union Query as the row source including a NULL value in the bound column, and some appropriate choice in the first visible column (such as 

"   No Value Selected", with a leading blank so it sorts first).

ken sheridan say: In a combo box you can simply delete the value in the control.  In a list box you can only assign a Null to the control's value in code.  List boxes are not really designed to be used as bound controls, so one alternative would be to include an item such as N/A at the top of the list by making its RowSource something like this example from one of my own files:

SELECT DrugName, 1 As SortColumn  
FROM Drugs  
UNION
SELECT "N/A", 0  
FROM Drugs
ORDER BY SortColumn, DrugName;

The code processing the selection would then ignore the value of 'N/A' and only take some action if a real drug name is selected.

If the above list box were a bound control, then the Drugs table would need to have a row with a value of 'N/A' at the DrugName column position of course, and its RowSource would be:

SELECT DrugName, 1 As SortColumn  
FROM Drugs
WHERE DrugName <> "N/A"  
UNION ALL
SELECT DrugName, 0  
FROM Drugs
WHERE DrugName = "N/A"  
ORDER BY SortColumn, DrugName;

In the latter there is no need to suppress duplicates in the UNION operation so the more efficient UNION ALL operator can be used.

chrabis001 say:

I was having the same problem with the certificates and found the answer in another blog.  Sorry, I don't have the link to the other blog anymore but this is how it was resolved for me.

I had just set up an *** Email address is removed for privacy *** email account in Outlook 2007.

The account worked fine (both for incoming and outgoing mail), except that I kept getting the certificate pop-ups for both i/c and o/g mail.

The solution to the problem was in the certificate dialog itself.

I just exchanged my mail server IDs (in email account setup in Outlook) with the mail server IDs mentioned in the certificates.

It may not be the same for you, but my original setup (from Yahoo help site) directed me to use the following servers which both produced the problem certificate:

i/c server:  mail.y7mail.com

o/g server: smtp.y7mail.com

The required servers indicated in the 'View Certificate" option of the problem certificate dialog box were:

i/c server: legacy.pop.mail.yahoo.com

o/g server: smtp.mail.yahoo.com

Yahoo, no more certificates and mail works a bit faster.
