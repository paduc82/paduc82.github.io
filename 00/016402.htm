tom van stiphout say:

Input masks are not appropriate for most datetime validations, and this is no exception. It's too inflexible and too much of a straightjacket.

MUCH better is to write code in the [control].BeforeUpdate event. Here are some ideas.

The textbox would have a Format property of General Date, so it shows a full date/time.

You want to force people to enter a time component. Why? You did not tell us what this field is about so I am free to speculate. If one doesn't enter a time component, by default Access will choose midnight. Why is that wrong? Why force the user to enter "05/09/2017 12:00:00 am" if they can achieve the same using "5/9"?

Once we agree on that, let's do some validation. Maybe a time of 12am is really not acceptable because this is an AppointmentTime and we are only open for appointments from 8am to 5pm:

Private Sub txtAppointmentDateTime_BeforeUpdate(Cancel As Integer)
    Dim dt As Date
    dt = TimeValue(Me.txtAppointmentDateTime)
    If dt < #8:00:00 AM# Or dt > #5:00:00 PM# Then
        MsgBox "Yo! Only from 8-5!", vbExclamation
        Cancel = True
    End If
End Sub

Of course appointments in the past are useless, so we can add more validation for the date:

if DateValue(Me.txtAppointmentDateTime) <= Date() then

        MsgBox "Yo! Only future dates!", vbExclamation
        Cancel = True

end if

Interestingly this will also fire if user only enters "8a", because Access default date is 12/30/1899.

I think you get my drift. None of this can be done using a date mask.

That's very much closer to the PC version.  OK, it's not a released product yet, expect some crashes.  And stuff that doesn't work!

In this one, you SHOULD convert the file to .docx.  And you should go one step further, and on the File menu run the "Convert document" command to convert it to the latest version of .docx.

Try it on a copy, obviously.

The issue is that Word saves a picture in layers.  One of those layers is the 72-dpi screen image.  The older versions of Word then get confused and cannot find the high-resolution version of the image, which is a different layer.  The new version is running the PC-Word code for the most part, so it should dig them out correctly for you.

Some of those pictures you may HAVE to dig out and re-insert into the new version.  To do that:

  1. Make a copy of the file in .docx format (must be .docx, you can't extract pictures form a .doc, they are embedded in the binary).
  2. Change the extension on the file to Zip
  3. Right-click the zip and choose Open With: Archive utility
  4. Open the resulting folder: it's a little website.
  5. In the Media folder, you will find the native versions of every picture in the document, in full resolution.

No, it is not possible to put the document back together again after you have done this on a Mac (unless you know how to zip a folder with the command line in terminal...).

Hope this helps
