Hi,

 

As you wish, let's see a demo of how to use Windows libraries (PostMessage & FindWindow) to handle this issue.

 

'API declarations:

Private Declare PtrSafe Function PostMessage Lib "user32" Alias "PostMessageA" (ByVal hwnd As Long, _
    ByVal wMsg As Long, _
    ByVal wParam As Long, _
    ByVal lParam As Long) As Long

Private Declare PtrSafe Function FindWindow Lib "user32" Alias "FindWindowA" (ByVal lpClassName As String, _
    ByVal lpWindowName As String) As Long

'Sent as a signal that a window or an application should terminate.
Private Const WM_CLOSE = &H10

'The class name of Msgbox window.
Private Const MSGBOX_CLSNAME = "#32770"

'Change the caption name of a pop-up MsgBox if necessary.
Private Const MSGBOX_CAPNAME = "Microsoft Excel"

'To save the handle of a pop-up MsgBox.
Public lHwnd As Long
Public dteRunWhen As Date

 

'#######################################
'Callback subroutine for OnTime method.
'#######################################
Sub MsgBoxProc()
    lHwnd = FindWindow(MSGBOX_CLSNAME, MSGBOX_CAPNAME)
    If lHwnd <> 0 Then PostMessage lHwnd, WM_CLOSE, ByVal 0&, ByVal 0&
    DisableMsgBox
End Sub

 

'################################
'Start auto-close a pop-up MsgBox.
'################################
Sub DisableMsgBox()
    dteRunWhen = Now + TimeValue("00:00:01")
    Application.OnTime dteRunWhen, "MsgBoxProc"
End Sub

 

'####################################
'Stop auto-closing a pop-up MsgBox.
'####################################
Sub EnableMsgBox()
    Application.OnTime dteRunWhen, "MsgBoxProc", , False
End Sub

 

Please paste the code above in a Module (Alt+F11 -> Insert -> Module).

For auto-closing, run macro DisableMsgBox, otherwise run macro EnableMsgBox.

 

Note:

[1]. The default window title name of MsgBox is 'Microsoft Excel' in Excel application, you can change MSGBOX_CAPNAME to what you had seen in your screen.

[2]. DisableMsgBox will close all the Msgbox windows with the same title name.

[3]. The new PtrSafe attribute enable you to use this Declare statement on either 32-bit or 64-bit systems. The PtrSafe attribute indicates to the VBA compiler that the Declare statement is targeted for the 64-bit version of Office 2010. Without this attribute, using the Declare statement in a 64-bit system will result in a compile-time error. Note that the PtrSafe attribute is optional on the 32-bit version of Office 2010. This enables existing Declare statements to work as they always have.

stefan blom say:

Note that cross-references make use of hidden bookmarks that Word adds to cross-referenced paragraphs (that is, to captions, numbered items, headings and so on).

The problem is that if you add content to the beginning of a bookmarked paragraph, the bookmark will expand to include what you just added. In particular, if you press Enter at the beginning of a bookmarked paragraph, the new paragraph will be inside the bookmark. This obiously means that your cross-reference will have additional, undesired content.

Recreating the cross-references will fix the problem. To prevent it from occurring in the first place, always add new content at the end of the preceding paragraph, not at the beginning of a bookmarked paragraph.

graham mayor say:

You can have more than one PST file open in Outlook and you can use vba to extract messages from an added PST file e.g. the following will open a named file - here "D:\My Documents\Test\Temp\Personal Folders.pst" it will then address each message in the Inbox folder of that file. You can then do what you want with those messages. Finally it closes the added PST file.

Sub GetMessagesFromAddedPSTFile()
    Dim olNS As Outlook.NameSpace
    Dim olItems As Outlook.Items
    Dim olItem As Outlook.MailItem
    Dim olFolder As Outlook.MAPIFolder
    Dim vFold As Variant
    Set olNS = Application.GetNamespace("MAPI")
    With olNS
        'Open the PST file
        .AddStore "D:\My Documents\Test\Temp\Personal Folders.pst"
        Set olFolder = .Folders.GetLast
        'Rename the folder
        olFolder.Name = "TempImport"
        Set vFold = olFolder
        'get the Inbox folder of the added file
        Set olItems = olFolder.Folders("Inbox").Items
            For Each olItem In olItems
                'do what you want with the messages here e.g.
                MsgBox olItem.Subject
            Next olItem
        'close the added folder
        .RemoveStore vFold
    End With
    'clear variables
    Set olNS = Nothing
    Set olItems = Nothing
    Set olItem = Nothing
    Set olFolder = Nothing
    Set vFold = Nothing
End Sub
