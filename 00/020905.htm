megadodo---01 say: The "charger" is just an energy source! Its the phone that controls the current delivery to the battery. If the battery can take the current the phone will allow what ever the charger can deliver. Anything lower and the phone will "limit" the supply to the battery anyway. If the charger cant deliver the battery requirements (e.g. USB charging if the battery is very flat) then the worst that can happen is that it takes longer to charge! I hope that helps? This is only valid for Lithium batteries.
Because the current delivery is variable depending on the battery's state then you cant just say a 500mAh charger will take 2 hrs to charge a 1000mAh battery. It WILL take longer.

ken sheridan say: As, from your earlier image, the ItemCode is clearly a text data type, the code in the ItemCode control's AfterUpdate event procedure should be:

    Me.WholeSalePr = DLookUp("WholeSalePrice","Inventory","ItemCode=""" & Me.ItemCode & """")

1.  The literal quotes characters which are required around the ItemCode text value are each represented by a contiguous pair of quotes characters.

2.  The Me keyword returns a reference to the instance of the class in which the code is running; think of it as referencing the current form.

3.  There is no need to qualify the reference to the ItemCode column with the table name.

4.  Object names which, correctly, do not include spaces or other special characters do not need to be wrapped in square brackets.

If you are unfamiliar with entering code into a form's, report's, report section's  or control's event procedures, this is how it's done in form or report design view:

1.  Select the form, report, section or control as appropriate and open its properties sheet if it's not already open.

2.  Select the relevant event property and select the 'build' button (the one on the right with 3 dots).

3.  Select Code Builder in the dialogue and click OK.  This step won't be necessary if you've set up Access to use event procedures by default.

4.  The VBA editor window will open at the event procedure with the first and last lines already in place.  Enter or paste in the code as new line(s) between these.
