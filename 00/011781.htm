Your description is a tad "involved" to say the least for what in reality is a fairly simple task so just going to cover some of the basic items you raised

#1 - POP3 email accounts can either (1) have a separate PST file for each account or (2) use a single one (as you already know). In reference to using multiple PST's for safety purposes - I have all of my POP3 account delivered to a single <Inbox> and then have multiple additional PSTs for historical purposes. I don't use the "archive" process - just move items from the <Inbox> to the appropriate PST file/folder when taken out of the but everyone deals with historical data differently

#2 - A pst file can be saved to anywhere you want (I keep mine on a separate local hard drive) as opposed to Outlook "default" locations - easier to backup and don't have to worry about things on the odd occasions that I have to restore an image of that drive.

In short, just a matter of saving your old PST files to where you want and then just open each PST file in Outlook
File --> Open & Export --> Open Outlook Data File

#3 - Merging is not complicated - just need to decide on which of the PST files you want to be the "destination" for the merged data OR create a new PST file and use that as the "merged" data store. 

If you use a new PST file - just a matter of moving/copying folders to where you want them (I strongly suggest using the <copy process> leaving your original data intact. Alternatively, you can move/copy items to an existing folder in the destination PST file.
