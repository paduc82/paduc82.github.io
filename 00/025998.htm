ossiemac say:

I think that anyone who is smart enough to crack the password (and there are plenty who are) then they will also be able to find and delete any code you include to delete the worksheet.

Passwords, like locks, only keep out the honest people but they do serve a purpose to alleviate accidental deletion of formulas etc by users.

If the data is so sensitive then I would be looking at other methods like linking to another workbook that is not distributed. The end user will just get a message about updating links that they can't do but that should not be a problem because it just uses the data from the last update.

Having said that you could include the following code in ThisWorkbook module so that if the specific sheet is activated then it is deleted. Note that you cannot put the code in the sheets code module because the module gets deleted with the sheet and you will get an error.

I think the code is a bit dangerous in that you need to make sure that you always have a backup of your workbook before installing code otherwise you will lose your worksheet if you unhide and activate the sheet.

Private Sub Workbook_SheetActivate(ByVal Sh As Object)
  Application.DisplayAlerts = False
  If Sh.Name = "Sheet1" Then
    Sh.Delete
  End If
  Application.DisplayAlerts = True
End Sub
