Try using a more accurate expression to calculate age.  This returns the age in whole number of years and is more accurate then the expression you are using. 

CurrentAge =DateDiff("yyyy",DOB,ReviewDate) + Int(Format(DOB,"mmdd") > Format(ReviewDate,"mmdd"))

ken sheridan say: For greater flexibility add the following function to a standard module in the database:

Public Function GetAge(dtmDoB As Date, Optional varAgeAt As Variant) As Integer

    If IsMissing(varAgeAt) Then varAgeAt = VBA.Date
   
    GetAge = DateDiff("yyyy", dtmDoB, varAgeAt) - _
        IIf(Format(varAgeAt, "mmdd") < Format(dtmDoB, "mmdd"), 1, 0)

End Function

The second argument is optional, so if it's called with just the date of birth passed into in, GetAge([Dob]) then it returns the age at the current date, but if a second argument is passed into the function, it returns the age at that date, so in your case you'd put the following in the query design grid:

Age at Review: GetAge([DoB], [Review Date])
