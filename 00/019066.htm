Is there a way to make the outline on a checkbox the same size as the - Microsoft Community

karena61 say:

Is there a way to make the outline on a checkbox the same size as the shaded area?

When I keep form shading enabled the check box outline is not the same size as the shading. Is there a way to correct that?

bob jones aka: cybertaz say:

It isn't something that can be "corrected" because it isn't incorrect :-) The checkbox is an element (actually a character) contained within the Form Control. The shading applies to the control, not its content.

You can select the checkbox & change the Font Color which will apply to the border of the box but you cannot apply a Fill Color to the box. The only possible exception, if there is one, would involve using VBA.

karena61 say:

Thank you.

I am not as concerned about the fill color relative to the border color as much as the fill color fitting inside the border.

john korchok say:
There are no formatting controls to keep the shading inside the box. though you could reduce the amount of grey below the checkbox but not having the font raised (Format>Font>Advanced>Position>Normal). If the shading is a problem, you can turn it off with the Developer>Form Controls>Form Shading button.

karl timmermans [outlook mvp 2012-2018] say:

The definitive answer to your question is "Depends" on how the two DL's were merged into a 3rd list.

 

Option #1 - (where original DLs can NOT be deleted)

 

A 3rd DL is created which only has two entries (for the sake of this explanation) - one for each of the other DL's. Deleting the original DLs in this case will negate the use of the newly created 3rd DL

 

Option #2 - (where original DLs CAN be deleted)

 

The contents of the 3rd DL contains one entry for every entry found in the other two DLs (i.e DL#1 - has 10 members, DL#2 has 30 members - DL#3 should have 40 members in this instance ignoring duplicates that may exist between DL#1 & 2)

 

A general comment - always be sure that you have a current backup of your critical data regardless of what application that is in reference to so that if something is changed/deleted which should not have been, will be a simple issue to revert back. karl timmermans [outlook mvp 2012-2018] say:
To perform Option #2, what is the correct Office Outlook 2007 Procedure to achieve this.


Well the most labor intensive "brute-force" way would be to recreate a new DL manually adding each of the members one at a time

 

Another way would be to

 

1) Open DL1

2) From menu --> File --> SaveAs --> Select .txt as the output file and save the file (DL1Mbrs.txt as an example)

(that creates a tab-delimited file containing one line for each member - Name & Email Address

3) Close DL1 - Open DL2 and repeat step #2 - close DL2

4) Open the file "DL1Mbrs.txt" in any text editor and remove the first 4 lines (the title lines)

5) repeat the same process for the 2nd DL text file

6) Copy all the members from the DL2 text file to the DL1 list so you have one consolidated list of members (you will have to deal with any overlap (duplication) of members and delete those accordingly)

 

In Outlook,

7) create a new Contact Group - give it a name

8) Add new members using <Add Members> <From Outlook Contacts>

9) highlight all the members from the file in step #6, right-click & "copy"

10)  back in Outlook, after doing step #8 - <Paste> the data from step #9 into the <Members> field and click OK

 

You now have a new DL which should have all the members from DL1 & 2.
