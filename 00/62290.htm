Hi ListenToPANDA, most likely you are in a Temporary Profile, caused by a Windows Update

Before you do anything or move any files around
Restart (not shut down) your PC 4 times in a row, this will fix this issue a lot of the time . ..

Open that folder and check inside Pictures, Documents . . . etc. and see if your files are there

If your files are there:

Create a new user profile with Admin privileges
Log into the new user profile
Copy all your data from the old profile to the new profile (Documents Library, Pictures Library . . . etc.)

If you still have not found your files, look for a Windows.old folder on your C Drive, expand that to open the users folder contained inside - check there for your files, icons . . .etc.

wunch say:

Okay, I just figured out my problem.  Previously, I had set the "Windows Media Player Network Sharing Service" to disabled to reduce overhead because I wasn't using it.  All I had to do was re-enable it, and voila, Internet streaming can now be enabled.

 

So:

  • Start -> type "services.msc" -> enter
  • Scroll down and find "Windows Media Player Network Sharing Service"
  • Right-click on it -> Properties
  • Change the startup type to "Automatic (Delayed)"
  • Click on the button to "Start" the service
  • Click Okay

After doing this, I now get the confirmation dialog when trying to enable Internet Streaming in WMP.

It seems to me like this should be considered a lack-of-proper-feedback bug.  WMP should check to see if the service is running when a user tries to enable Internet Sharing and give an error message if it is not (or even better, give the user the option to enable it without having to use services.msc).
