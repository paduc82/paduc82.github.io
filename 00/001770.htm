scottgem say:
First, I would not recommend using what is referred to as a Multi-Value Field (MVF). While MVFs make for a nice interface they cause problems in retrieving the data. A MVF simulates a many to many relationship with a hidden junction table. I would recommend using an actual junction table instead.

However, an MVF needs to be set at the table level not the form level. You need to change the datatype to match the data type of the ID field. Then use the Lookup tab and change to a Combobox. One of the options will be to allow multiple values. Make sure that is checked. Set the Rowsource as you have it and the bound column too 1 and the Column count to 2 and Column widths to 0";2". then delete the control on your form and drag the fields from the Additional Fields list onto your form.

jay freedman say:

On the Home tab, click the little arrow in the bottom right corner of the Styles group (or press Ctrl+Alt+Shift+S) to open the Styles pane, and click the Manage Styles button at the bottom of the pane.

In the Manage Styles dialog, go to the Restrict tab.

In the list of styles, select all the table styles you don't want to see in the ribbon -- most of the ones whose names start with "Grid Table", "List Table", "Plain Table", and "Table Grid". You can select a whole group by clicking the first one, then holding Shift while clicking the last one. For each selected group, click the Restrict button.

When you've restricted the ones you want to get rid of, check the box for "Limit formatting to permitted styles" and click OK.

I'm not sure why you couldn't delete the custom styles. Try using the Delete button in the Edit tab of the Manage Styles dialog.
