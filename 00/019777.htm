john w. vinson say:

In that case, I'm sorry to say that your table structure is WRONG: it's appropriate for a spreadsheet, not for a relational table. The proper structure for a table would have two tables in a one to many relationship: a Parts table (with no forecast data), related one to many to a Forecast table with fields for the PartNumber, the week number, and the forecast for that week. If your forecasts could (as they surely must) span more than one year, you should probably actually use a forecast date field, rather than a week number. Given this proper tall-thin table, a very simple totals query will get your sum (or the sum for last year, or next year, or any other desired date range).

With your current wide-flat design you'll need either your existing query or some VBA to open a recordset and loop through the fieldnames.

ken sheridan say: The correct way to model this in a relational database would be by means of tables like this:

Parts
....PartNumber (PK)
....PartDescription

Forecasts
....PartNumber (FK)
....ForecastWeek
....ForecastValue

The primary key of the Forecasts table is a composite one made up of the two columns PartNumber  and ForecastWeek.  The values in the ForecastWeek column can be sequential numbers from 1 to 52, but 'week beginning' dates would be better.

You can then return the cumulative values in a query like this:

SELECT F1.PartNumber, PartDescription, F1.ForecastWeek, F1.ForecastValue,
SUM(F2.ForecastValue) AS CumulativeValue
FROM (Forecasts AS F1 INNER JOIN Forecasts As F2
ON F1.PartNumber = F2.PartNumber
AND F1.ForecastWeek >= F2.ForecastWeek)
INNER JOIN Parts
ON Parts.PartNumber = F1.PartNumber
GROUP BY  F1.PartNumber, PartDescription, F1.ForecastWeek, F1.ForecastValue;

If you want the query to be updatable you'd need to call the DSum function to compute the cumulative values rather than joining two instances of the Forecasts table.  For examples see Balances.zip in my public databases folder at:

https://onedrive.live.com/?cid=44CC60D7FEA42912&id=44CC60D7FEA42912!169

If you have difficulty opening the link copy its text (NB, not the link location) and paste it into your browser's address bar.

To output the data in the sort of horizontal layout you currently have you could use a crosstab query, or, better in my view, a multi-column report with 'across-then-down' column flow.

dennissma say:

Access 2010 No License error

I just purchased and downloaded Access 2010. The install went fine. When I launch the application I get a message saying 'Microsoft Access can't start because there is no license for it on this machine'.

I put my product key in during the installation. The installation went fine. I have never had a trial version of office or Access on my machine. Any ideas?

I'm running Vista Home Premium service pack 2
nathan o. - microsoft support say:

Hi Dennis,

 

This error tends to happen in two scenarios.  The first scenario usually occurs when you have a trial version of Access on your machine (which you said you do not).  Do you have any other Office 2010 products on the machine, if so, which suite / products do you have installed?  The only reason I ask, is because Office Home and Business 2010 and Office Home and Student 2010 usually include a free trial of Office 2010 Professional with them.  Unfortunately, it wont list Office 2010 Pro in the add / remove program list, so you may not realize it is on the machine.

 

If you dont have any other versions of Office or you already removed the trial, then the issue most likely is related to a registry permission issue.  If you open the registry and navigate to the key below, do you receive any error messages?  If so, this may be related to your problem.

 

HKCR\Licenses\73A4C9C1-D68D-11d0-98BF-00A0C90DC8D9\14.0\Retail

 

If you are uncomfortable with registry and / or havent used the registry, then it may be a good idea to have you open a case with Microsoft so that we can look into the issue with you.

 

Online Assisted Support Options (create cases online) - http://support.microsoft.com/select/default.aspx?target=assistance

 

Best Regards,

Nathan O.

Microsoft Online Community Support
