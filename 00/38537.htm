Most YouTube videos are in a modern SWF (flash) format called flv/swf. This is capable of anything from slow ads to very high definition video and uses codec H264. When you connect to such a video on the server of the site it is on, you connect to a swf file which is a container file which contains a custom player and possible text, links, logos, etc. Also flash must be installed on your computer. The swf file also starts downloading of a flv video file to a temporary file on the computer. While the video is still downloading, the video starts to play after an initial delay. If playback catches up with the download, there must be a pause to allow download to progress more before playback is resumed. Thus the reason for your problem is likely that there is some slowdown in download speed anywhere from the server to your computer. If you allow the video to completely download and then replay the video without moving away from the site, the video should then play from the temporary file on your computer without stopping. However if you move on to another site, the video likely will be lost from the temporary download file.

Welcome to Microsoft Answers Forum!

LimeWire relies on the Gnutella network to search for and download different types of files

 

The Gnutella network can, however, be blocked by a firewall program, preventing LimeWire from connecting to find content. Firewall programs are security programs that protect a computer from network intrusions, but can also block programs relying on an Internet connection. Unblocking LimeWire takes a few minute and will allow you to regain connection to the Gnutella network.

Note that Limewire is known to be a high security risk software if user still want to install.

1. Check the LimeWire connection to see if a firewall is blocking access to the Gnutella network. To do so, launch LimeWire and click the "File" button on the top menu.

 

2. Scroll down and click "Advanced Tools." Select the "Connections" tab. You should see the connection status. If you see "Connecting" on the ports under "Host," it means there is a firewall blocking the connections. Move to the next step to unblock LimeWire connections.

3. Click the Windows "Start" button and select "Control Panel." Double-click "Firewall" to open the firewall settings window.

 

4. Click the "Exceptions" tab and place a check on "LimeWire." If you do not see the LimeWire program, click "Add Program" and upload LimeWire from your computer. Place a check on it to add it to the exceptions list.

 

5. Click "OK" to save the changes. Close the Control Panel.

 

6. Add LimeWire to the exceptions list on any third-party security applications you have installed on your computer. This process varies with different security tools; therefore, refer to your program's documentation for the proper steps.

 

7. Launch LimeWire again. Check the connection status on the bottom. LimeWire connections should now be unblocked.

Adding an Exception for the Gnutella Port:

 

1. Within LimeWire, go to: Tools > Options > Advanced > Firewall Config, and make a note of the listening port located in the box next to "Listen on Port".

 

2. Open the settings of your security program (usually by right-clicking the system tray icon).

 

3. Locate a section for adding exceptions for specific ports.

 

4. Add the port specified in LimeWire's settings as an exception for both UDP and TCP, incoming and outgoing (also known as opening the port).
