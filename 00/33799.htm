How to change user profile name

I recently sent my laptop back to HP for repairs and the hard disk had to be re-imaged.  The technician created a user account "TEST" as the primary user.  As a result all programs, documents etc are saved under this user.  I want to change the "TEST" account to my name instead.  How do I change it? Does it have to be changed in the registry? I am using Windows 7, 64 bit.
b69ca say: Hi,

Assuming you are logged onto the machine using the TEST account, if you go into Control Panel, User Accounts and then select the Test account you will see an option to "Change your account name".  That will rename the TEST account to whatever you want it to be.

Now, the problem with this however is that it just changes the account's full name and not the account username itself.  Therefore, your account will display as, say "Bob" but the actually machine account would still be TEST.

If you want to actually rename the account you would want to use the Local Users and Groups option by right-clicking on the Computer icon and choosing "Manage".  You should be aware that this option / tool is not available in all Windows 7 distributions and you didn't mention your version.

In there, you can find your account, right-click it and choose rename.  The only thing this doesn't do is rename the profile below C:\Users but you can probably live with that.

Honestly, the easiest thing to do, create another account using your name and copy all of the stuff from your TEST account to the new account and then delete the TEST account.  This will save you a boat load of hassle.

If you still have problems please do let us know.

patrick holden say:

I had the same problem I couldn't play anything.

This was a new pc build I did and installed Windows 10.

I did a restore from a backup of my Music that had been on a previous Windows 7 OS

Here is what I found.

I went to view the permissions of the folder that I restored the Music to and it showed I needed read Permissions.

But my account is in the Administrators group.

For some reason the folder did not have the permissions

I selected Advanced to continue and I saw there was no owner to the folder.

I changed the owner to my ID selected Replace owner on subcontainers and objects

I also checked the box Replace all child object permission entries.... selected ok and I was on my way.

You might get prompted to apply your admin privelages as you hit other folders, but just select Ok and you should be good.

Now your situation might be different, but check your user/id privileges on the Music folders or files and see if you have access.  If you see a SID, which will begin with S-1-5-21 then that can be a problem.   Reapply permissions to the folder or file with the ID you are using and it might correct the problem for you.
