A simple troubleshooting step to perform is to do a Safe Boot. It will take a long time for your Mac to start up in Safe Mode. When you do a Safe Boot in El Capitan Apple's Disk Utility runs and fixes any troubles with your Mac's directory and file system. Permissions are repaired. The font cache is emptied. Only a limited number of fonts are enabled. Applications that run at startup don't run automatically. 

 

Once you reach the desktop in Safe Mode try running PowerPoint. Is the crash still there? If not, then maybe simply performing a Safe Boot fixed the ailment. Or perhaps one of the fonts that wasn't loaded is bad. Or perhaps some other software utility is interfering. After testing in Safe Mode, do regular restart. Try PowerPoint again. Is the crash still there?

 

If the crash was gone in Safe Mode but returns during a regular boot then there are more things to try. The first would be to use a font utility to validate ALL your fonts. Mac OS comes with a free font utility called Font Book. Get rid of any font that doesn't pass as clean and green, restart your Mac and then empty the trash. If you've gotten this far and still have a crash, then there are more things to explore. But if you get this far you've probably gotten rid of the crash. Either way, please reply with your results.

dan at it associates say:

There is no "Digital River" version.

 

There are no Office version Upgrade licenses.  ALL Office 2010 retail consumer licenses are FULL licenses, not upgrade licenses.  Under some circumstances, Office 2010 Setup will offer you an option to conduct an upgrade installation....this is an installation method and has nothing to do with a version upgrade license.

 

Microsoft specifically recommends that most people use the 32-bit version of Office 2010, which installs just fine on both 32-bit and 64-bit operating systems.  This recommendation is based on the fact that the 64-bit version of Office 2010 is incompatible with the vast majority of Office Add-In programs that have been developed up to now, and based on the fact that the only significant capability that 64-bit Office has that is superior to 32-bit Office is the ability to handle unusually large spreadsheet files.

johnny_55 say:

You basically have 3 options here. I personally use VMware-workstation-full-14.1.1-7528167 which is a paid option. I have about 8 different versions of windows I can run at any given moment.

You could also use Oracle VM VirtualBox which is a free program. This one is probably the simplest to setup.

Finally you could use Hyper-V which is included with Windows 10. To get to this one open up Control Panel / Programs / Programs and Features / Turn Windows features on or off. Scroll down the popup for Hyper-V and check the box and reboot.

After you make your choice of which way you want to go, you will have to install Windows inside one of the virtual machines. This would be a separate windows and would require it's own activation as it is not tied into your main machine. You could run without activation but you will lose the ability to personalize your settings.

Just a note if you happen to run BlueStacks by any chance, it will not work with Hyper-V turned on. I am really only familiar with VMware so others might point out the advantage or disadvantage of the other 2.
