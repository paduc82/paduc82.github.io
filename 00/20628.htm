jeeped say:

You will not be able to change the program that launches a Hyperlink by modifying the File Association for any particular file extension (e.g. .JPG). Hyperlinks are based upon the Hypertext Markup Language (e.g. HTML). You would have to change your default browser in order to change the program that launches the hyperlink and that is an undesirable solution.

You can, however, modify the method by exchanging Assign Macro for Hyperlink. I'll provide a generic example that opens a picture from the public\pictures folder with Paint.

 

First, insert your picture into a worksheet and position, size and format it as you prefer. Right click it.

              

Select Assign Macro from the right click menu.

         

In the Assign Macro dialog you may optionally rename the suggested macro name but this is unnecessary. Click New. When the Visual Basic Editor opens, paste this code into the empty macro Picture1_Click function,

Sub Picture1_Click()
    sCmdLine = "MSPAINT.EXE " & "C:\Users\Public\Pictures\Profile_Avatar.jpg"
    iTaskId = Shell(sCmdLine, 1)
End Sub

Note that the sCmdLine variable constructs the command line <program> <path\picture file>. This is a very common method of launching a file with a program but specific programs should be researched as to their specific command line parameters.

If you leave out the program (e.g. MSPAINT.EXE) and simply use the path and filename of the picture, it will launch according to the default for that file extension (e.g. what you were unable to accomplish earlier with the hyperlinks).

When you are satisfied with your modifications to the command line, press ALT+Q to close the Visual Basic Editor and return to Excel. The picture can now be clicked on to launch the picture in Paint.

 

Note that this workbook will have to be saved as a Macro Enabled Workbook (e.g. XLSM).

Using Expression Builder in a table to build an "IF" - Microsoft Community

simone_53 say:

Using Expression Builder in a table to build an "IF" statement...

Hello MS Community!

I am very 'green' at designing in Access and have a question on how to put together an expression with an "If" statement.

1) I have a field in a table which is actually a question: "Employee's absence totals at least 30 consecutive work days?"

2) I need an expression that says, "If the answer is 'No', then display the following message "Employee may not be eligible for any program benefits at this time."

Do I need a new field which is the 'If' expression alone or can #2 above be added as a Validation Rule?

Many thanks,
Simone

 

ken sheridan say: You can call the IIF function in a query based on the table.  If we assume that the 'question' field is called Abs30 and it is a Boolean (Yes/No) data type, then in the 'field' row of a blank column in query design view put:

Eligibility: IIF(Not [Abs30],"Employee may not be eligible for any program benefits at this time.","")

The query will return a column named eligibility with the text where the value of the Abs30 column is False.  The query will be updatable, so can be used as the RecordSource for a form for data entry.  Once the value of the Abs30 column is changed, the value of the computed Eligibility column will change to reflect the new value.

If the value of the Abs30 column determines whether a value can be entered in another column in the table then you can prevent a value being entered in that column via the form if Abs30 is False by putting code in the BeforeUpdate event procedure of the control bound to the other column, e.g.

Const MESSAGETEXT = "Employee is ineligible for any program benefits at this time."

If Not [Abs30] Then
    MsgBox MESSAGETEXT, vbExclamation, "Invalid operation"
    Cancel = True
End If
