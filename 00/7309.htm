ashish mathur say:

Hi,

 

Try this

 

1. Suppose weights are in range G6:G10

2. Suppose prices are in range H6:H10

3. Enter weight in cell G14

4. In cell H14, enter this formula

 

=IF(ISNUMBER(MATCH(VLOOKUP(G14,$G$6:$H$10,2,1),$G$6:$G$10,0)),VLOOKUP(G14,$G$6:$H$10,2,1),((INDEX($H$6:$H$10,MATCH(G14,$G$6:$G$10,1)+1,1)-INDEX($H$6:$H$10,MATCH(G14,$G$6:$G$10,1),1))/(INDEX($G$6:$G$10,MATCH(G14,$G$6:$G$10,1)+1,1)-INDEX($G$6:$G$10,MATCH(G14,$G$6:$G$10,1),1))*(G14-INDEX($G$6:$G$10,MATCH(G14,$G$6:$G$10,1),1)))+INDEX($H$6:$H$10,MATCH(G14,$G$6:$G$10,1),1))

bill manville say:

I assume the price is linear between the adjacent points?


Suppose that you have a set of weights in a column range named Weights and the prices for those weights in an adjacent column range named Prices. 

Like this:

0 Â£0
5 Â£2
10 Â£3
20 Â£4
50 Â£5

And you have a weight for which you want a price in cell D3.


It is easiest to use an intermediate cell to hold the row number within Weights for which has the highest weight which is not greater than the value in D3.  Say this is in E3 which contains the formula
=MATCH(D3,Weights)

 

Then the price will be:
=INDEX(Prices,E3)+(D3-INDEX(Weights,E3))/(INDEX(Weights,E3+1)-INDEX(Weights,E3))*(INDEX(Prices,E3+1)-INDEX(Prices,E3))

hansv mvp say:

That is 'normal' behavior. When you start creating a record by entering something, the new AutoNumber value is assigned, and it will not be reused even if you cancel out of that record.

This should not be a problem since an AutoNumber is intended to be used as a meaningless unique identifier.

If you compact and repair the database (from the Database Tools tab of the ribbon), the "next" AutoNumber value will be reset to (largest currently used value)+1. Gaps in the sequence will not be filled in, however.

john w. vinson say:

A Form with an updateable recordset will always have a blank new record displayed after the last real record. This isn't actually stored on disk as a blank record, it's just a "placeholder" open for new data. Are you sure this isn't what you're seeing? Or are you getting an actual new record and incrementing the autonumber value? If so, either you're unintentionally dirtying the new record by typing something into it, or you may have some code or a macro doing so.

 

ken sheridan say:
Record is half finished and exited out of without saving

What do you mean by 'without saving'.  A bound form automatically saves an edited current record if the form is closed or moved to another record, including an empty new one.   You can exit a new record if no data has been manually inserted by the user, or no values have been inserted by code behind the scenes, but if you've entered data and then deleted it from the form's controls then you must undo the record by pressing the Esc key or by other means, or Access will attempt to save the record.

 

If you wish the saving of a record only to be specifically undertaken by the user then you must set up the form so that it does not automatically do so.  You'll find an example as SaveDemo.zip in my public databases folder at:

 

https://skydrive.live.com/?cid=44CC60D7FEA42912&id=44CC60D7FEA42912!169

 

If you have difficulty opening the link copy its text (NB, not the link location) and paste it into your browser's address bar.

 

Another point is that, if a record is being saved with only the autonumber value, then there is something seriously wrong with the table design as it means that all columns bar the autonumber column are allowing Nulls.  No table should do this; at least one and usually most, columns should disallow Nulls by having their required property set to True (Yes) in the table design.
