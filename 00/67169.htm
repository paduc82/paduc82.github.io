graye say:

When using Internet Explorer, it was (and still is) easy to move a favourite item from one folder to another.  Eg drag & drop, cut & paste or the ‚Äúorganise favourites‚Äù from the pulldown menu.

To be clear, I am not asking about moving a folder up or down the list.  Nor am I asking about moving an element within a folder.  In fact, Edge offers a neat way of doing this with the mouse.

I am however asking how to move an item (element?) from one folder to another.

Thanks

The technique is kinda strange.... because the favorites pane uses a "breadcrumb list" at the top to show which folder you're currently in.

So, lets say you're deep in a folder structure and want to move an item up several levels (and then perhaps over a few levels).  In this case, you have to move the item in steps...   first you'd move it to the "parent" folder by doing a drag-n-drop on to the name of the parent folder in the "breadcrumb list".   Yes, you actually drop it onto the word "favorites" (or whatever is the name of the parent folder)... not an icon.

Next, you'd have to navigate up to the parent folder, and do the move over again... (perhaps "up"... or perhaps "over").  Repeat until you're done.

Yes, the whole thing is tedious, clumsy, and a very poor design.

Luckily, the new version of Edge that should be available on the 2nd of August has a full "tree view" instead of "breadcrumbs".  That makes moving items between folders a whole lot easier, as you can see all items in the tree at once.

Meanwhile, if you don't want to wait until August... there is a free 3rd-party application called EdgeManage that can do this now.

shashwat vikram say:

Hi Shreyash,

Even after deleting the background data for Spotlight, if it's not functioning; I would recommend you to reinstall the Spotlight. However Spotlight is not an application, but a Windows Feature, part of Content Delivery Manager. So we need to re-install the Content Delivery Manger.

(Don't worry, this won't mess with other parts of your system)

To re-install this app, we need a specific parameter which is the full path of it's XML file. All modern apps contain this XML file in their program files folder and this XML file is required to restore the app. It's located in the folder below : 

C:\Program Files\WindowsApps

This folder is hidden as well as restricted. So you won't have the direct access to retrieve the parameter's value. Follow the steps below to obtain it :

  1. Open PowerShell as Administrator
  2. Enter the Command : Get-AppxPackage -allusers | Select Name, PackageFullName
  3. Hit ENTER

Select the PackageFullName for Microsoft.Windows.ContentDeliveryManager and copy it to Notepad.

Now execute following command in PowerShell (as Administrator) by replacing the PackageFullName part in the command below, with the value copied in Notepad. Replace only the PackageFullName:

Add-AppxPackage -register "C:\Program Files\WindowsApps\PackageFullName\appxmanifest.xml" -DisableDevelopmentMode

-> Hit ENTER

That's it. It'll reinstall the app. You may need to restart the computer.

Wait for some time with active Internet Connection ensuring that your current Network is not set as Metered Connection and Spotlight Feature is selected for Lock Screen.

Then switch to Lock Screen, and see if it works !!!
