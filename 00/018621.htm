doug robbins - mvp office apps & services (word) say:

If by  "can the mail merge dialog when opening the document be ignored"?, you are referring to the message "Opening this document will run the following SQL command ...", you can prevent that form occurring by modifying the Windows Registry as detailed in the Knowledge Base Article at:

https://support.microsoft.com/en-au/help/825765/you-receive-the-opening-this-will-run-the-following-sql-command-message-when-you-open-a-word-mail-merge-main-document-that-is-linked-to-a-data-source


I guess is you cannot access the Startup folder, modifying the registry is probably also going to be ruled out.

The following code in a Class Module in your Master Document can be used to remove the data source when a document is closed

Option Explicit
Public WithEvents oApp As Word.Application

Private Sub oApp_DocumentBeforeClose(ByVal Doc As Document, Cancel As Boolean)
With ActiveDocument
    If .MailMerge.MainDocumentType <> wdNotAMergeDocument then
            .MailMerge.MainDocumentType = wdNotAMergeDocument
            .Save
            .Close
    End If
End With
End Sub

For it to work, you would need the following code in an ordinary module in your Master Document

Option Explicit
Dim oAppClass As New ThisApplication
Public Sub AutoOpen()
Set oAppClass.oApp = Word.Application
End Sub

Of course, if you use this, when the document is re-opened, it will be necessary for the user to change the document back to a mail merge main document and to re-attach the data source.  That is all done by my add-in without the user having to do anything.

If you show us the code in your Master Document, so that we can see exactly what it is doing, it might be possible to make other suggestions.
