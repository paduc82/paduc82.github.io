kerwinyang_msft7476 say:

Hi Anna,

We cannot use PowerShell to move a top level site collection to another top level site collection for SharePoint Online.

As a workaround, we can use Save as a template feature. Here I’d like to share with you the following steps:
1. Navigate to the top-level site of your site collection which you’d like to move, then go to Settings > Site Settings > Under Site Actions > Click Save site as template> Specify a template name> Select the Include Content box.

2. Go to this top-level site> Settings > Site Settings> In the Web Designer Galleries section> Solutions, select the template and activate it if necessary, then click its name to save it to the local.

3. Navigate to another top-level site, click Settings> Site Settings> In the Web Designer Galleries section> Solutions, upload the specific template in the local, then activate it.

4. After that, you will be able to create a new sub-site under this site collection with this template in the Custom tab during the creation process.

For your reference:

Using vlookup to find values using the prefix in a data set with - Microsoft Community

jordan66 say:

Using vlookup to find values using the prefix in a data set with populated and unpopulated cells in the lookup value column.

Hello,

 

I am currently using this formula: =IF(ISNA(VLOOKUP(LEFT(Y2,3)&"*",THIRD_PARTY,1,FALSE)),0,L2) in an attempt to return either a dollar value or a 0 for values in column Y based on whether or not their prefixes match the data in "THIRD_PARTY." This formula works great for populated cells in column Y, but not every cell has a value. I would enter them in but I am creating a template for others to pull data from the web and paste directly into so it needs to account for blank cells. I know that &"*" is where I am running into issues as blank cells then match everything. Is there another way to ensure that my prefixes are matched exactly while excluding blank cells?

 

Thank you!

jeeped say:
Okay, I think I have a better grasp of the situation. You only want to determine if the first three letter prefix in Y2 is found within THIRD_PARTY so MATCH() might be more efficient but COUNTIF() would definately improve things.

Since COUNTIF() will return a 0 if not found, then you can do away with the IF() function as well since 0 × L2 will equal 0. It sounds link you will only return a 1 or a 0 with COUNTIF() but for future expansion, it may be best to wrap it in SIGN() which will return a 0 if the COUNTIF() ifs 0 and a 1 if the COUNTIF() is anything else.

So the possibility of a blank Y2 still remains. SIGN(LEN(Y2)) will accomplish the same as SIGN(COUNTIF()). If Y2 is blank, then L2 would be multiplied by 0 resulting in 0.

=SIGN(COUNTIF(THIRD_PARTY,LEFT(Y2,3)&"*"))*SIGN(LEN(Y2))*L2

Not found non-blank value in Y2:
0 × 1 × L2 = 0
Found non-blank value in Y2:
1 × 1 × L2 = L2
Found blank value in Y2:
1 × 0 × L2 = 0

Addendum: it may be of worth to note that while SIGN() can return a -1, nether COUNTIF() or LEN() will ever return a negative number so the point become moot.
