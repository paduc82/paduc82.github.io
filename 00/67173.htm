The syntax

        fsutil dirty query e:

is correct (unless, of course, you were trying to use it from the Recovery Console, where the fsutil command is not available).  Are you logged on as a user with administrative rights when using fsutil?

Similarly, the /f and /x switches are not available for chkdsk in the Recover Console.

See Recovery Console commands.

Click My Computer then right click on the icon for drive E:  and select Properties.  What is reported for:

  • File System
  • Used Space
  • Free Space
  • Capacity
Now double click on the icon for E:  Are any files shown?  If not, click Tools > Folder Options > View and select the radio button to "Show hidden files and folders" and UNcheck the box to "Hide protected operating system files (Recommended)".  Now do you see any files?

Whatever you read about "low level formatting" is nonsense.  No modern PC drives can be low level formatted by end users.  You might want to read this, but there really is no necessity even to "zero fill" your E drive: http://seagate.custkb.com/seagate/crm/selfservice/search.jsp?DocId=203931 (I haven't checked to see if the WD LifeGuard tools has similar features to SeaTools for DOS).

If you become satisfied that there is nothing of value on E (such as a backup or restore facility), you can either format it by right clicking on the drive's icon in My Computer and selecting "format" or you can start up Disk Management as described earlier and use that either to format E or delete the partition entirely.  If you delete the partition, the space becomes "unallocated."

Since I assume that C is your system drive (where Windows is), you may be able to use the Windows diskpart tool to expand the F partition to use the unallocated space left if you decide to delete E although it is far easier (and safer) to do with third-party disk management tools such as free Easeus Partition Manager: http://www.partition-tool.com/personal.htm
