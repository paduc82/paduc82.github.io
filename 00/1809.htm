On 1st boot of Outlook after upgrading to 2016, Outlook needs to wait for AutoDiscover to run to get the Exchange mailbox connectivity settings. tssky say:

Excel VBA - Find Cell Address of Index Match with Multiple Criteria

Hello,

I have a huge table that looks like this (only 1 row is shown):

I want to find the calculated area (6th column) from this table using 2 CRITERIA: device and wafer size. I want the CELL ADDRESS of this return value to replace it with something else.

For example, if device = "Pressure Sensors" and waferSize = "300", then I want to find the cell address of 2.00 (return value) and replace it with 6.00 for example. 

I tried this code, but I keep getting Type Mismatch error.

' Get die size cell address of the device & wafer size user selected
' Example format: =INDEX(DieSizeApprox[Calculated Area (mm2)], MATCH("Pressure Sensors"&"300", DieSizeApprox[Device]&DieSizeApprox[Wafer Size (mm)], 0))
' DieSizeApproxTbl.ListColumns(6) is Calculated Area column 6
' DieSizeApproxTbl.ListColumns(1) is Device column 1
' DieSizeApproxTbl.ListColumns(3) is Wafer Size (mm) column 3

dieSizeLookup = Application.WorksheetFunction.Index(DieSizeApproxTbl.ListColumns(6).DataBodyRange, Application.WorksheetFunction.Match(userDeviceName & userWaferSize, DieSizeApproxTbl.ListColumns(1).DataBodyRange & DieSizeApproxTbl.ListColumns(3).DataBodyRange, 0)).Address

Any help would be great!

a. user say:

First, declare these additional variables...

    Dim vMatchRow As Variant
    Dim sCritRng1 As String
    Dim sCritRng2 As String

Then try...

    With DieSizeApproxTbl
        sCritRng1 = .ListColumns(1).DataBodyRange.Address(, , , True)
        sCritRng2 = .ListColumns(3).DataBodyRange.Address(, , , True)
    End With
    
    vMatchRow = Evaluate("MATCH(""" & userDeviceName & """&" & userWaferSize & "," & sCritRng1 & "&" & sCritRng2 & ",0)")

    If Not IsError(vMatchRow) Then
        DieSizeApproxTbl.ListColumns(6).DataBodyRange.Cells(vMatchRow).Value = 6
    Else
        MsgBox "No match found.", vbInformation
    End If

Note that the Evaluate method has a 255 character limit.  So you'll get an error if the number of characters in the string exceeds it.  If this is the case, a workaround method will be needed.

Hope this helps!
