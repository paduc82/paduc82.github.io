scottgem say:
The combobox wizard that creates a search combo creates an UNBOUND control. This has to be unbound. Therefore when you use the navigation buttons to cycle from record to record, the unbound control doesn't change. If you want the search combo to reflect what is in the current record, put this code in the ON Current event fo your form:

Me.combobox = Me.PrimaryKey

Where combobox is name of the search combo and PrimaryKey is the name of the control bound to your primary key .

ken sheridan say: I'm not sure I follow.  Are you saying you want to carry the values entered in one record forward into one or more new records?  If so, set the DefaultValue property of each of the controls to the value entered, which you can do in the form's AfterInsert event procedure with code along these lines;

Me.[Incentive Type].DefaultValue = """" & Me.[Incentive Type] & """"
Me.[Dollar Amount].DefaultValue = """" & Me.[Dollar Amount] & """"
Me.[Eligible Claim Date].DefaultValue = """" & Me.[Eligible Claim Date] & """"
' and so on for other controls

Note that the objects names are the names of the bound controls, which might not be the same as the columns to which they are bound, and the values should be wrapped in literal quotes characters as above, regardless of their data type.  Note also that setting the DefaultValue property does not initiate a new record; that only happens when the user enters additional data or changes one of the values.

A combo box shows only one value when it's not dropped down; you can use its ColumnCount and ColumnWidths properties to display more when it is dropped down.

But what do you mean by "collect"? What will you be doing with the X and S once you have selected them? If you're trying to store them redundantly in two additional fields in some table other than your CODES table, that's almost surely a mistake!

If you just want to display the SL1 and SL2 values on the form, you can put two unbound textboxes on the form; set their Control Source to

=[nameofyourcombobox].Column(n)

where n is the zero based position of the field you want to see - that is, if SL2 is in the 4th column of the combo box use Column(3).
