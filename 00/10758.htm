Fonts with PostScript outlines sometimes display incorrectly after being edited in Word's Print Layout.

 

Several workarounds for this issue in order of most convenient to least convenient are:

1. Add a Right-to-Left language, such as Hindi, to your list of editing languages. [(In Word) File -> Options -> Language -> [Add aditional editing languages] -> Hindi -> Add]

2. Each time a document is opened with the incorrect font being displayed change to a different printer (and back). [(In Word) File -> Print]

3. Uninstall all printers from Windows. [(In Windows) Start -> Devices and Printers]

Nevermind, I fixed the equation and it is working now; reference below for anyone with similar issues:

SELECT Customers.[Last Name], Customers.[First Name], Allowances.Category, Allowances.Allowance, Sum([Bid Log].Bid) AS Bids, IIf([Allowances]![Allowance]-Sum([Bid Log]![Bid])>=0,"True","False") AS Status
FROM (Customers INNER JOIN Allowances ON Customers.[Customer ID] = Allowances.Customer) INNER JOIN [Bid Log] ON Allowances.ID = [Bid Log].Allowance
GROUP BY Customers.[Last Name], Customers.[First Name], Allowances.Category, Allowances.Allowance, Allowances.ID;
