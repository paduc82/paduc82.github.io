Technically, the problem is this: In the kernel32.dll file that comes with Windows XP, there is a function named GetDateFormatA and one named GetDateFormatW. Windows 7 and later added a third variation named GetDateFormatEx. The original release of Office 2010 was programmed to use one (or maybe both) of the A and W varieties. Some Office update (KB4461522 according to what I found on the Web) changed Word's program file to call the Ex version, probably because some programmer didn't realize that it doesn't exist in XP.

I'm surprised that uninstalling updates all the way back to July didn't restore the original function call, but there may be some other factor involved. A repair may or may not restore it, but a reinstall really should (at least until another rogue update comes along!).
