Any number of things could be the source of the increased size of PDFs.

PDFs from Mac Office apps go through the MacOS pdf printer driver, which has had a history of producing bloated PDFs.  The Windows Office apps use a completely different process, one that I hope gets ported to Mac Office ASAP.  Meanwhile, if you have a Windows PC with Office available, you might try making the PDF there.

Next, are you using effects like transparency, glows, shadows and the like?  All of these can contribute to monster PDFs. 

One possible workaround that occurs to me, if you don't need to preserve action settings/links/etc, is to export all of the slides in the original presentation as pictures, then create a new presentation by inserting the pictures into a blank presentation.  I'm not where I can check this on my Mac, but it may be built in ... see if there's an option to save as a "Picture Presentation". This would do the job for you if it's available.  Be sure to work on a COPY of your original, just in case.

daniel g. grau say:

You should have added "for me" because that solution did work for at least 18 other users.

So let's try this, which I'm surprised we did not suggest before. It's called a safe boot. It's a simple procedure that will attempt to load in your Mac only software, add-ins, fonts, etc. etc. that are official versions of Apple software and other approved vendors. If Word works, it probably means that some software is interfering with it (i.e. software that was not allowed to execute under safe boot conditions).

  • Shut down you Mac
  • Turn it on
  • As soon as you hear the initial "boing" sound, press Shift (that's Apple's instructions; I press is as soon as I see the screen turn on)
  • Hold it down until you see the Finder and it seems to stop loading

Once the Finder loads, test Word. If it works, the problem is caused by non-Apple software added to your user account or your Mac.

There is a similar procedure involving the creation of a brand new user. This ensures the user does not have any additional stuff in their account. You could try that and then test word from that new user.

How large is your document? I can't think of any allowable document size that would use anything like the capacity of RAM available on your PC. The speed of opening a document is largely determined by how fast Word can load it from disc. If you add to that the overhead of (say) network access then it can take a long time to open a document, and this has nothing to do with the availability of RAM. If you were to use a local SSD drive then you would find that most things happen very much faster. However, if it takes an hour (or more than a very few minutes) merely to open a large document, then there is something else wrong with your system (or document) that has nothing to do with Word's use of memory.

Based on past experience, my first guess would be that the file contains many links to external data sources. One thing you can do to try and localize the problem is start splitting the file.

Insert a new blank para at the very bottom of the document.  Open a new blank document, then select and copy the upper half of the document, paste it into a new file and save it.  Open another blank file, select the bottom half of the original file, carefully excluding the very last (blank) para, and save it too.

Try opening the 2 files to see if the response time is more reasonable.  Split the upper half, again and again until you get reasonable response time.  Then you can look for specific elements that may be causing the slowdown.
