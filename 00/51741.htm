¡firedog say:

I wonder why you need a larger mailbox when you're using a mail client anyway. Windows Live Mail stores each message you receive and send as an individual file. Keeping copies of messages received also on the server (i.e. in the mailbox) is a sort of backup, in that you have two copies of each message, one local and one in the cloud.

Changing the type of mailbox should not have any effect at all on Windows Live Mail, so long as you continue to use the same method of connecting to it - POP3, or IMAP.

Removing an account from Windows Live Mail deletes all the accounts folders and permanently removes their contents. It isn't easy to do this accidentally, but just in case, you may want to consider copying or moving important messages from account folders to archive folders under Storage folders. These won't be affected by changes to the mail account. It's then easy to maintain a backup of important messages simply by keeping a copy of Storage folders somewhere safe - on an external disk drive, say, and/or at OneDrive, where you already have 5GB of free storage.

One last suggestion: it's often not a good idea to keep old messages in the inbox at the server. The more messages you have there, the greater the chances of something going wrong. This can be avoided simply by doing some regular housekeeping, say once a month or so. Create archive folders in webmail and move messages out of the inbox into them, leaving only the most recent ones in the inbox.

a. user say:

In reply to deleted message

This problem occurs because there are incorrect entries in the Shell folder of the user who is logged on to Microsoft Windows. When Windows Installer accesses the Shell folder assignment of the user who is logged on to the computer, Windows Installer cannot locate the correct entry. The solution is inhttp://support.microsoft.com/default.aspx/kb/886549.

The steps to correct this problem involve editing the Windows Registry.  Modifying REGISTRY settings incorrectly can cause serious problems that may prevent your computer from booting properly. Microsoft cannot guarantee that any problems resulting from the configuring of REGISTRY settings can be solved. Modifications of these settings are at your own risk. I suggest you first backup your registry as follows: http://windows.microsoft.com/en-US/windows-vista/Back-up-the-registry.

Here are the steps to fix it.

1.     Click Start, click Run, typeregedit in the Open box, and then click OK.

2.     Locate the following registry key:

HKEY_CURRENT_USER\Software\Microsoft\Windows\CurrentVersion\Explorer\User Shell Folders\

3.     Delete the entries that show the path that is listed in the error message.

From KB 886549http://support.microsoft.com/default.aspx/kb/886549: Open Regedit again. In the left pane, right-click User Shell Folders, point toNew, click Expandable String Value, type the Name value that you want to reinstate (AppData) and then press ENTER. Right-click that value, clickModify, type the value in the Value data box for the Value name, and then clickOK. The value data should be: %USERPROFILE%\AppData\Roaming.

Step 8 of KB 886549 suggests that you also check the values in HKEY_LOCAL_MACHINE\Software\Microsoft\Windows\CurrentVersion\Explorer\User Shell Folders. However, in fixing my problem, I did not have to modify any of the values in that registry key. If this does not work, follow ALL the steps in the referenced article.

Good luck.

USB devices are “Plug and Play” devices, and so, Windows usually detects these devices when they're plugged in and if necessary, installs the drivers. If Windows is unable to detect the device, then the reason behind that would be unavailability of an appropriate driver for the device, faulty device or defective USB port or if the installed drivers become corrupted. So I would suggest you to check the driver for the device in the manufacturer website.

b. Search for the USB device that you are having issues with, right click on it and select “Properties”.

c. Click on the “General” tab, and then look in the “Device status” box to see if it mentions any problems with the device or its driver.

a. user say:

And here is the correct answer, which I obtained with extensive internet searching:

If an otherwise perfectly functioning UFD is recognized on other computers but not on your computer, the problem may be that your computer previously assigned this UFD a drive letter that is currently used or reserved on your computer.

To solve this, attach the UFD to another computer where it is recognized, and use the other computer's Disk Management tool to assign a drive letter to the UFD which is not currently in use or reserved by your computer. In my case, I assigned drive letter X, which I have never used on my computer.

The next time I inserted the UFD to my computer, Windows 8 recognized it instantly and issued it an available drive letter. Problem solved.
