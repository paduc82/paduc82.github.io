m,d. kokosnuss say:

It appears that, instead of actually shutting down, when the shutdown button is clicked Windows enters a hibernation state. By disabling hibernation I was able to change this behavior, so that it now actually shuts down my machine.

For anyone wondering how to do this (credit to windows10update.com) :

  1. Press Windows + X keys to open Power menu and select Command Prompt (Admin)
  2. To enable the Hibernate, type command powercfg/h on.
  3. To disable the Hibernate, type command powercfg/h off.

You didn't include the version of Works you have, the version of Windows you have, or the specifications of your computer (particularly the amount of RAM you have), all of which might have some bearing on your problem.

However, you probably have the problem described here:  https://www.small-business-technology.net/other-microsoft/out-of-memory-error-message-when-printing-in-works-7-2003-and-2004/

The link to the Microsoft Knowledge Base article in the article linked above is no longer functional; fortunately, the Internet Archive Wayback Machine comes to the rescue:  https://web.archive.org/web/20150401070755/http://support.microsoft.com:80/en-us/kb/841220

"Method 1" in that old Microsoft KB article is the place to start.  Go to the website of your printer's manufacturer and download the driver appropriate for your version of Windows.  If you have difficulty with this, please identify the make and specific model of your printer.  Also identify your edition of Windows and include whether you have a 32-bit or 64-bit system.  If you're not sure, press the "Windows Logo" key and the "Pause/Break" key simultaneously to see this information.

If you already have the latest printer driver AND you used to be able to print Works spreadsheets and databases without any problem, I suggest that you remove the printer driver that's now installed by using the following directions and then install a freshly-downloaded copy of the driver.

  • Uninstall printer-related software from Control Panel > Programs and Features
  • Open an Elevated Command Prompt (click Start, type Command Prompt in the Search box, right-click the link to Command Prompt, and select "Run as administrator")
  • Type the following in the black Command Prompt window and press Enter
        
            printui /s /t2
            exit

    Be sure to remove the driver AND the package.

If installing the latest driver doesn't help, and you already have configured your printer as the default printer, I'd suggest going with the work-around suggested in the small business technology article rather than attempting to figure out what you might be able to use as an "emulation printer driver."  I'm not familiar with pdf995, but I can recommend CutePDF Writer, which is also free and doesn't have a nag screen as PDF995 apparently does:  http://cutepdf.com/Products/CutePDF/writer.asp
