jordan [msft] say:

Hi all, thank you for the questions. We've made a change in the latest builds to not show Defender definition updates in update history anymore. Since definition updates are released so frequently, they tend to flood the update history and make it difficult to parse and read, especially after you've used Windows 10 for a while.

Separately, you've identified that the update history is currently erased when you install a new build. This is partially by design since once you install a new build, updates released for previous builds no longer apply - they are no longer offered to your PC and you can no longer uninstall them.

We are working on improving update history in future builds, but I cannot give specifics or a timeline yet. If you have feedback though please let us know. One of the first improvements we'll make is to ensure that you can see in your update history when a new OS build has been installed.

Neil - we definitely want to make sure that definition updates are installed successfully. However, we know that if Defender definition updates are failing to install, there is usually nothing you can do to make it work. It instead would be because of a content issue that the Defender team needs to fix. When this happens, the Defender team is able to quickly identify it in telemetry and revise the content to fix the issue. Showing those failures in Update History would unfortunately not provide you with the information you'd need to fix it.

Thanks,

Jordan
