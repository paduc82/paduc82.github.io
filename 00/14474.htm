vesko_m say:

OK,

I will try to be as much as helpful as I can :)

Firstly - the necessary config out of a Cisco 2801 that is acting as VPN server for Cisco VPN Clients and Nokia VPN clients:

 

! 

crypto isakmp policy 16
 encr 3des
 authentication pre-share
 group 2

!

crypto isakmp client configuration group YOURGROUP
 key YOURPASSWORD
 dns x.x.x.x
 domain abc.def
 pool SDM_POOL_1
 acl 100
 save-password
 include-local-lan
 max-users 5
 netmask 255.255.255.0

!

crypto ipsec transform-set ESP-3DES-SHA esp-3des esp-sha-hmac

!

crypto dynamic-map SDM_DYNMAP_1 1
 set security-association idle-time 900
 set transform-set ESP-3DES-SHA
 reverse-route
!        
crypto map SDM_CMAP_1 client authentication list sdm_vpn_xauth_ml_1
crypto map SDM_CMAP_1 isakmp authorization list sdm_vpn_group_ml_1
crypto map SDM_CMAP_1 client configuration address respond
crypto map SDM_CMAP_1 65535 ipsec-isakmp dynamic SDM_DYNMAP_1

! 

interface FastEthernet0/1

 crypto map SDM_CMAP_1

!

ip local pool SDM_POOL_1 x.x.x.240 x.x.x.250

!

access-list 100 permit ip y.y.y.0 0.0.0.255 any

 

As you can see there is nothing special about this configuration, I even used SDM to enable VPN clients connecting to this router.

After you succesfully connect with a PC with Cisco VPN Client to this router, you have to configure the Nokia VPN Policy tool as follow:

 

Go to File -> Load

in Program Files\Nokia\Nokia Mobile VPN Client Policy Tool\GatewayProfiles\Cisco\ASA\Cisco_ASA_pskxauth.pol

in VPN Gateway address: type the outside ip address of your router.

in Identity Value: YOURGROUP from the router config above

in PRESHARED KEY - Format:STRING_FORMAT

in Key: YOURPASSWORD

 

after that go to Advanced View:

there in IPsec-SAs-CISCO_ASA_PSK

in Encryption algorithm: 3DES

in Hash algorithm: SHA1

there in IKE-Proposals-3DES-CBC

in Encryption algorithm: 3DES-CBC

in Hash algorithm: SHA1

in OAKLEY group: MODP_1024

 

Then generate the VPN Policy.

 

I think I was detailed as possible :smileywink:

These above are from working config of a router and working VPN client on a Nokia 5800, E65 and E66

 

As far as I understand your problem - I think there is maybe a problem with user authentication after your Nokia connects to the router. Probably you have configured the Nokia VPN Policy correctly with the vpn group and password. After that it takes IP address from the Pool, and I think that after that you have to enter the user and password for authentication on the router, if I am not mistaken.

 

Good luck,

hope I was of little help.

hansv mvp say:

There is a macro action RunSavedImportExport, but it's not listed by default.

In order to use it, your database must be stored in a Trusted Location. See Create, remove, or change a trusted location for your files for detailed information.

Next, open a macro in design view.

On the Design tab of the ribbon, in the Show/Hide group, click to highlight Show All Actions.

The RunSavedImportExport action should now be available.

You have to specify the name of the saved export.

You can add as many RunSavedImportExport actions to the macro as you want.

Good luck!
