I can tell from scanning through the code whether there's anything there that relates to the message you're getting.  My guess would be that it isn't, if as you say all you did was move the database from one computer to another.

However, often when you move a database to another system, references get broken.  Open the database on your laptop, open the VB Editor, and click Toolss -> References... to display the References dialog.  Look among the checked references for any references marked MISSING.  If you find one, note down what it is, uncheck the reference, and close the dialog.  Then, open the dialog again and scan down the list of unchecked references for the reference that you removed.  If you find it, check it again and close the dialog.  If you don't find it, see if your database really needs it:  close the References dialog and click Debug -> Compile.  If it compiles without errors, you didn't need the reference.

If you don't find any MISSING references in the References dialog, then this is probably not your problem and we'll have to look deeper.
