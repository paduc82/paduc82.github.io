hairyfool say:

You cannot use the /x parameter on the boot drive.

When you dismount the drive all open handles are closed which the running O/S will not allow as it is its host disk.

Use CHKDSK /F.

It will then advise that it will not run but will schedule it for the next boot. /scan will do an online scan but will not fix

If you really think the /R parameter is required scrap the drive immediately. The drive electronics will automatically remap bad sectors using spares at the end of the drive, you should not be able to detect that it has. The only time you should get any benefit from /R means it has run out of spares and you are in a bad way.
