OUtlook desktop folders and server synching - Microsoft Community

epbutler say:

OUtlook desktop folders and server synching

I am using Outlook 2016 desktop app and have an outlook.com email address.  The subfolders I have are stored on the server but I want them to be in the local desktop app.  How can I accomplish this?

karl timmermans [outlook mvp 2012-2018] say:
Not entirely clear about the question. Is the Outlook.com account currently configured in Outlook Desktop?

An Outlook.com account would by default be configured using the Exchange protocol which means that all data would be sync'd with the Outlook desktop client assuming the account is configured and was not manually configured as a POP3 account.

If manually configured as an IMAP account, all email folders would be sync'd. Contact/calendar folders would be "local only".

epbutler say:

I have 2 accounts included within Outlook .  See pic for how they are connected.

When I Look at the subfolders I have it shows the line at the bottom stating there are more on the server.  I don't want them on the server.  How can I retain them locally in the outlook folder structure?

karl timmermans [outlook mvp 2012-2018] say:
First things first................how IMAP and Exchange protocols work

With both of the above two protocols, the OST data file associated with each account is a "mirror" of what is contained on the server. In terms of Exchange, this would include all folder types whereas IMAP only involves email folders.

In both cases, if the underlying OST file is in accessible to Outlook for any reason, Outlook will recreate that file and re-sync all folders. In the case of IMAP (for OL'2013/'2016) if it was the first email account configured in the Outlook profile, it would also contain folder names with the added description of "This Computer Only" (specifically, contact/calendar folders) which means its :"local only" to that computer since IMAP does not sync that data with the server - nothing to do with Outlook, it's the way that IMAP works.

Some background articles in case they're of interest

Understanding Email Protocols (POP, Imap, MAPI, EAS, Exchange)
https://www.contactgenie.info/understanding-ema...

(I realize that this is not part of your question - just providing it as a FYI for future reference)
How to Protect Outlook '2013/'2016 Imap Contact/Calendar Data
https://www.contactgenie.info/how-to-protect-ou...

Now to the second part

Your second scrnshot shows "More items", just so that there's no mis-understanding, that simply means that there are more items on the server what haven't been downloaded to the local store. That's something that's controlled in the account set-up.

From OL menu go to File --> Account Settings --> Email tab --> highlight the email account of interest and click on <Change>

Set the amount of "email to keep offline"

Now if you permanently want to retain email data locally and "off the srever", you can create a PST file
From OL Ribbon --> Home --> New Items --> More Items --> Outlook Data File (browse to where you want to create this file and give it a name).
This will add the PST file to the Outlook Nav Pane, at which point you can create the folders you want and then "move" items from the server sync'd folders in the email account folder trees. Or alternatively, you can completely "move" the applicable folders altogether and remove them from the server depending on what it is that you want to accomplish.
