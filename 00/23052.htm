Oh, but it can be done quite easily. Paste the following three-line macro in Normal:

Sub AutoOpen()
Application.ActiveWindow.Caption = ActiveDocument.FullName
End Sub

How to paste a macro
  • In your main menu bar, select Tools > Macro > Visual Basic Editor
  • You should see a Project box
  • In there, select Normal (Normal.dotm)
  • In the main menu bar of the Editor, select Insert > Module
  • A new window will open (there will be a default name, but you can type your own)
  • Paste the macro code
  • Save your Normal.dotm template (File > Save Normal.dotm) . Otherwise, Word will prompt you when quitting.
  • In the main menu of the Editor, select Close and return to Microsoft Word (or type Command Q)

Note, however, that when paths are long you will need a really wide window.

To disable the macro, just delete it.

Paths in Finder windows
To obtain a similar display in Finder windows:
  • Open Applications > Utilities > Terminal
  • Paste the following bold code after the $ prompt:
  • defaults write com.apple.finder _FXShowPosixPathInTitle -bool YES
  • Quit Terminal and restart your Mac

To cancel the effect, use this command:
defaults write com.apple.finder _FXShowPosixPathInTitle -bool  NO

steve rindsberg say:

You probably already know this, but I'll mention it in case it helps:

The language packs convert the Office user interface ... the ribbons, menus, messages and so on ... to different languages.

This may be what you need (for example, so that a user who doesn't read Japanese can use Office).  I'm guessing that's what you need, and if so, skip the next paragraph. ;-)

On the other hand, if you simply need to switch between typing Japanese and English, you can install the different languages in Windows, then use the language bar to switch from one language to the other on the fly.  

Since safe mode is a workaround, I'd have a look to see what, if any, COM add-ins are installed and disable them all, test by starting PPT normally (ie, not in safe mode), then re-enable them one at a time, testing after each one is enabled.  I doubt it'd be a VBA add-in at fault; safe mode doesn't disable those, no matter what anyone else may tell you, so safe mode wouldn't cure any problems they introduce.

I'd also temporarily set the Windows default printer driver to Microsoft's XPS driver if that's available to you.  PPT can be cranky about printer drivers; the English language pack might cause it to go toes up when it encounters a printer driver in Japanese.  The XPS driver is more likely to be language-agnostic, or so my theory goes.
