sync of the library using ngsc is not supported when minor versions are enabled and draft item security is set to other than "any user who can read items".

solution

5. under general settings > versioning settings.

6. check the document version history and draft item security, and,

 a. for document version history, choose no versioning or create major versions

 b. if create major and minor (draft) versions is enabled, select any user who can read items for draft item security

grahmd say:

Macro to copy/paste based on value in column

     I'm a VBA novice, and I'm trying to find a way to copy the text in one column based on the value in another column.  For example, I need to copy the text in Column B, and use the value in Column A to determine in which column to paste it.  

 

          A          B          C          D          E

           0        ABC     [ABC]

           1        DEF                 [DEF]

           2        GHI                              [GHI]

 

     I inherited the workbook with the following VBA code to solve this, but I'm sure there's a more elegant solution that doesn't take so long to run.

 

ActiveSheet.Range("A2").Select
    ActiveWindow.FreezePanes = True
    NU = ActiveCell
    First = NU
    For T = 1 To 65500
        NU = ActiveCell
        On Error Resume Next 
        If NU <> Int(ActiveCell) Then GoTo Here1
        If ActiveCell <> First Then
            ActiveCell.Offset(0, 1).Activate
            Selection.Copy
            ActiveCell.Offset(0, NU + 1).Activate
            ActiveSheet.Paste
            ActiveCell.Offset(0, -NU - 2).Activate
        Else
            ActiveCell.Offset(0, 1).Activate
            Selection.Copy
            ActiveCell.Offset(0, NU + 1).Activate
            ActiveSheet.Paste
            ActiveCell.Offset(0, -NU - 2).Activate
        End If
Here1: ' added 7-24-09
        ActiveCell.Offset(1, 0).Activate
        If ActiveCell = "" Then Exit For
    Next T

 

     I look forward to any advice.

graham mayor say:
You can only calculate numeric values, if the macro is completing and then throwing an error, it would appear that the next value in Column A after the last value to be processed is not a numeric value.

-4162 is the numeric equivalent of the directional indicator xlUp and iRow is the last row with data in Column A. In the following I have used xlUp instead. The result is the same.

You could test for the numeric value in the cell in column A and use

Dim i As Long
Dim iRow As Long, iCol As Long
Dim xlSheet As Worksheet
Set xlSheet = ActiveWorkbook.Sheets(1)
iRow = xlSheet.Range("A" & xlSheet.Rows.Count).End(xlUp).Row
For i = 1 To iRow
    If IsNumeric(xlSheet.Range("A" & i).Value) Then
        iCol = 3 + xlSheet.Range("A" & i).Value
        xlSheet.Cells(i, iCol) = "[" & xlSheet.Range("B" & i).Value & "]"
    End If
Next i

instead. This would skip any row with text in column A and also ignore the text in the header row and avoid the error, so you could start from 1 again as shown here.
