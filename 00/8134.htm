mike middleton say:

You wrote > By saying it is discontinued, does MS have other functions to replace CEILING? <

I did not write that Microsoft has discontinued the CEILING function.

A mathematically "smooth" function y=f(x) has small changes in output y for small changes in input x. Search functions like Excel's goal seek will usually work fine for smooth functions.

A mathematically "discontinuous" function y=f(x) has large changes in output y for small changes in input x. Search functions like Excel's goal seek will often fail when used with a discontinuous ("step") function like CEILING.

You wrote > If you have ever encountered this or have any solution to bypass the CEILING and willing to share I will be very grateful. <

I think you should consider the solution I proposed. Remove the CEILING function from cell E2 so that E2 contains =F48+...+G127. Use goal seek with cell E2. In some other cell, enter =CEILING(E2,8).

You should be able to open the .mdb file with each of those versions, but if you are opening the same file with multiple Access versions at the same time, you could easily corrupt the file.  If you are having an issue with your database becoming corrupt and you are using multiple versions of Access to open the file, then you should split the database.  Once the database is split, the backend data file should be placed on a file server and each user should be given their own separate front end database file to work in.
