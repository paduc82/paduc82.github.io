if you receive the eror message "sharing failed: syntaxerror: unexpected end of input" when sharing sites with default group, it is an expected behavior.

to clarify, there are three groups we can use in sharepoint.
1. default group, which is created along with the creation of site collection. e.g. team site member/team site visitor.
default group is used to assign sharepoint permission. when we use the group to invite people, we will get the error ‚Äúsharing failed: syntaxerror: ***‚Äù. the error may differ if we use different browsers.

2. sharepoint group, which can be created in site settings > site permissions > create group
sharepoint group is also used to assign sharepoint permission. when we use the group to invite people, we will successfully share the files/libraries. however, the user in group won‚Äôt receive the email notifications. because there is no email address assigned to the sharepoint group, and the group can‚Äôt deliver emails to every user in the group.
reference: https://support.office.com/en-ca/article/create-and-manage-sharepoint-groups-b1e3cd23-1a78-4264-9284-87fed7282048 

3. security group.
there are two kinds of security group. one is normal security group, which is created in office 365 admin center > groups. it has the same function and behavior as sharepoint group does.
another one is mail-enabled sharepoint group, which is created in exchange admin center. the difference is that mail-enabled sharepoint group has an email address assigned. so when we use mail-enabled sharepoint group to invite people, all the users in the group will receive the notifications email in their mailbox. for details, please refer to https://technet.microsoft.com/en-us/library/bb123521.aspx
