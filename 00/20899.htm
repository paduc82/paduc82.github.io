rich007a say:

Having thought over night, this version might work better, it will also copy shapes on each SlideMaster, as well as each Custom Layout (in each SlideMaster).

 

Sub CopyFromCustomLayoutsToSlides()
    Dim pres As Presentation
    Dim des As Design
    Dim cl As CustomLayout, clBlank As CustomLayout
    Dim sld As Slide
    Dim i As Long
   
    Set pres = ActivePresentation
   
     'Get a Custom Layout obect to use for all new slides:
    Set des = pres.Designs(1)
    For i = 1 To des.SlideMaster.CustomLayouts.Count
        Set clBlank = des.SlideMaster.CustomLayouts(i)
         'EDIT NAME OF PREFERRED CUSTOM LAYOUT HERE:
        If clBlank.Name = "Blank" Then Exit For
        Set clBlank = Nothing
    Next i
   
     'Check we found the chosen Custom Layout:
    If clBlank Is Nothing Then
        MsgBox "There isn't a custom layout called 'Blank' in the first SlideMaster!"
        Exit Sub
    End If
   
    For Each des In pres.Designs
         'Get shapes on the SlideMaster:
        If des.SlideMaster.Shapes.Count > 0 Then
            CommandBars.ExecuteMso ("ViewSlideMasterView")          'Goes to active slide's layout
            DoEvents
            des.SlideMaster.CustomLayouts(1).Select                          '1st Custom Layout
            CommandBars.ExecuteMso ("ViewSlideMasterView")          'Goes to SlideMaster
            DoEvents
            des.SlideMaster.Shapes.SelectAll
            ActiveWindow.Selection.Copy
            ActiveWindow.ViewType = ppViewSlide
            Set sld = pres.Slides.AddSlide(pres.Slides.Count + 1, clBlank)
            sld.Select
            ActiveWindow.View.Paste
        End If
       
         'Now get shapes on each of the Custom Layouts in this SlideMaster:
        For i = 1 To des.SlideMaster.CustomLayouts.Count
            ActiveWindow.ViewType = ppViewSlideMaster
            Set cl = des.SlideMaster.CustomLayouts(i)
            If cl.Shapes.Count > 0 Then
                cl.Select
                cl.Shapes.SelectAll
                ActiveWindow.Selection.Copy
                ActiveWindow.ViewType = ppViewSlide
                Set sld = pres.Slides.AddSlide(pres.Slides.Count + 1, clBlank)
                sld.Select
                ActiveWindow.View.Paste
            End If
        Next i
    Next des
    ActiveWindow.ViewType = ppViewNormal
End Sub

karl timmermans (outlook mvp 2012-2018) say:
Hello Richard

Some important things about how data is shared across devices which applies to any email client (of which Outlook is only one)

Comes down to how an email account is configured where how = POP3, Imap or Exchange

For Exchange accounts, nothing needs to be copied/shared since everything is stored on the server and each device would sync with the server.

For IMAP, only email data is stored on the server and sync'd with all devices connected to it. Contact/Calendar data is local to the device where the contact/calendar data is created. In Outlook, the data is stored in an .OST file which can only be opened/read by the Outlook profile/email account used to create the OST file. In short, you cannot not share an OST file among different computers.

For POP3, only downloads the <Inbox> data in a one-way one-time direction. Nothing is "sync'd" with the server. POP3 accounts store the data for the account in a PST file.

To get back to some of your questions about "sharing" Outlook files and summarize things

#1 - For Exchange account - nothing needs to be shared
#2 - For IMAP accounts - can't share an .OST file

#3a - For PST files whether or not associated eith e POP3 account, these files cannot be stored in the cloud and shared among different computers. Similarly, PST files on a shared network drive is also not supported - can easily result in data corruption.

#3b - storing a PST file on a thumb drive and moving it from computer to computer - is possible but can be prone to data corruption depending on how careful you are. Can be done but personally not my first choice as an approach.

So - what's the best approach

All depends on how (POP3, Imap, Exchange) your email accounts are configured. There are 3rd party tools like <Sync2> and <CompanionLink> that can sync data between devices. Whether one of those would be the best solution for you is something only yo can decide.
