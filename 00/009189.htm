Hiding Labels in a Access Report

This is survey data but for various reasons, I want to be able to shrink away the fields that a respondent skipped. john w. vinson say:

The simplest way I've found to do this is to do so in the Query, not in the report itself. Concatenate the string with the desired label to the value of the field, using the + concatenation operator rather than the & operator - doing so will return NULL if the value is NULL. For example, you could use

Q15: "Annual Income " + [AnnualIncome]

to display the annual income with a label, or nothing at all if the user declined to fill in that choice.

You can then display this concatenated result in a textbox without any label at all, and set the textbox's Can Shrink property to Yes to hide it.

That this is a problem indicates that your database is not designed properly. A survey database requires a very specific structure. The basic structure would look like this:

tblQuestion: QuestionID (PK Autonumber), Question

tblRespondent: RespondentID (PK Autonumber), Firstname, Lastname, etc.

tblResponse: ResponseID (PK autonumber), RespondentID (FK), ResponseDate

tblAnswer: AnswerID (PK Autonumber), QuestionID (FK), ResponseID (FK), Answer

The main point here is I suspect that you have a field for each question's answer. And that is definitely the wrong structure. The correct structure would have each Answer being a record in a table. With that structure you simply filter out any questions with no answer from the report. No need to jump through hoops to hide fields with no answer.
