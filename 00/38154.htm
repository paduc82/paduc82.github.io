The above issue might have occurred due to incompatible or non-updated device drivers or third party applications conflicting with the action performed on the computer.

 

CRITICAL_PROCESS_DIED error indicates a critical system process died when executing a program or application on the computer.

 

The IRQL_NOT_LESS_OR_EQUAL message indicates that a kernel-mode process or driver attempted to access a memory address to which it did not have permission to access. The most common cause of this error is an incorrect or corrupted pointer that references an incorrect location in memory. A pointer is a variable used by a program to refer to a block of memory.

Safe mode is a diagnostic mode that starts your computer with limited drivers necessary to run Windows.
