Really appreciate the assistance on this OssieMac.  It works great, just a few things that I am trying to modify for the end result.  How do we get it to append only the newly added rows to the 'ProductionApplications' tab?  Right now when it runs it will add all 'Yes' responses to the end, which includes those that have already been added, causing duplicates.  Looking for it to automatically copy just that new row once Yes is selected.  Sorry if I am missing something basic here, new to VB.  :)

Sub CopyYesData()

    Dim wsSource As Worksheet
    Dim wsDestin As Worksheet
    Dim rngSource As Range
    Dim rngDestin As Range
    Dim rngCel As Range
    
    Set wsSource = Sheets("ProductionTracking")
    Set wsDestin = Sheets("ProductionApplications")
    
    With wsSource
        'Following line assumes column headers in Source worksheet so starts at row 2
        Set rngSource = .Range(.Cells(2, "E"), .Cells(.Rows.Count, "E").End(xlUp))
    End With
    
    For Each rngCel In rngSource
       'Uppercase used for compare and disregards whatever case used in worksheet
        If UCase(rngCel.Value) = "YES" Then
           'Next line offsets 3 cols to left and then resizes to include 3 columns
           Set rngSource = rngCel.Offset(0, -3).Resize(1, 3)
            With wsDestin
                'Following line assumes column headers in Destination worksheet
                Set rngDestin = .Cells(.Rows.Count, "A").End(xlUp).Offset(1, 0)
            End With
            rngSource.Copy Destination:=rngDestin
        End If
    Next rngCel

End Sub

Also forgot to mention that I am looking to have the Macro run automatically when the value in E is set to Yes, if possible, so that those editing this document will not have to run the Macro manually.  

To install the code below in the Worksheet's module. (Must be in Worksheet's module or will not work)

  1. Right click the worksheet tab name
  2. Select "View Code" (this will open the VBA editor at the Worksheets's module.
  3. Copy the code below and paste into the editor
  4. Do not change the sub name or it will not work.
  5. Now every time a value in column E is changed to Yes then the code will run
  6. You still require the UDF in the standard module even if you decide to delete the previous code that copies all Yes values. Testing if the record already exists is essential because if a user were to select a Yes cell and then click in the formula bar and then press enter without changing the Yes, Excel still thinks that the value has bee changed.

Private Sub Worksheet_Change(ByVal Target As Range)
    Dim wsDestin As Worksheet
    Dim rngSource As Range
    Dim bFound As Boolean
    Dim rngDestin As Range
   
    'Next line: Not nothing then is something so is in col E
    'Also test that only one cell is changed
    'Me is generic object for worksheet to which this module belongs
    If Not Intersect(Target, Me.Columns("E")) Is Nothing And _
                    Target.Cells.Count = 1 Then
                   
        If UCase(Target.Value) = "YES" Then
       
            Set wsDestin = Sheets("ProductionApplications")
           
            Set rngSource = Target.Offset(0, -3).Resize(1, 3)
           
            bFound = FindExistData(wsDestin, rngSource)
           
            If bFound = False Then
                 With wsDestin
                     'Following line assumes column headers in Destination worksheet
                     Set rngDestin = .Cells(.Rows.Count, "A").End(xlUp).Offset(1, 0)
                 End With
                
                 'Following 3 lines is in lieu of Copy -> PasteSpecial -> Values
                 'because it does not leave the destination range selected
                 With rngSource
                    rngDestin.Resize(.Rows.Count, .Columns.Count).Value = .Value
                 End With
            End If
        End If
    End If
End Sub

mel77084 say:

It's really pretty straight forward ... it downloads the build (currently 14342) posted in the Microsoft site https://www.microsoft.com/en-us/software-download/windowsinsiderpreviewadvanced, and performs a clean install (allowing you to choose to keep a couple of things such as mail). Therefore, one should plan on needing to reinstall all desired software programs.

As long as you have your data in a different partition, it won't be bothered. In the case of multibooting, one might need to update the boot menu afterward.

A better option is to capture the install.esd files from the current build installation, and make an ISO file, which can then be used for either a clean or repair install.

And, of course, if one has made a disc image backup (saved on a different hard drive) of the OS partition as soon as a build is cleaned up after installation, it only takes a few minutes to restore the build (including all installed programs) without needing to rely on a (perhaps malfunctioning) OS to do it.

That said, though, doing a clean install of the latest build occasionally is a very good way to go.

For one who is knowledgeable enough to do the things described above, the new Refresh Tool holds little value.

a. user say:

Hello Pedro,

 

Thank you for giving us opportunity to assist you.

 

  • If you are verified admin of the yammer network than I would request you kindly download the export users file by navigating to the yammer home network as a verified admin > Settings > Network Admin > Export Users > Export all users > and click on export.

 

  • You will find a file as users.csv in the downloaded export folder.

 

  • We can differentiate in users.csv by Filter with column “state” and select soft_delete. Uncheck the box of “active”. Applying this filter would give us only soft_delete users.
    • Filter “joined_at” column which contains only blank value.
    • Filter “deleted_at” column which contains only blank value.
    • Filter “suspended_at” column which contains only blank value.
    • By doing above steps we can differentiate the users are currently in deleted state if they have “deleted_at” time stamp.
    • If the user with “suspended_at”  than that user is suspended or in block state.
    • If the user is having none of the states “deleted_at” and “suspended_at” then the user is in pending state

 

You can refer below link for your reference purpose:-

https://technet.microsoft.com/en-us/library/dn783348.aspx

 

  • If you are using yammer free basic network and admin group than unfortunately you can download such users.csv file to see the pending user list.

 

  • However you can navigate to the group settings and click on add or remove user and can see list of the active user and admin of the group with the star mark below profile.

 

  • If you know the email id of the users whom you invited than kindly search the name of the user and you can click on the user name and there it will show as pending user.

 

If you find my reply useful, would appreciate if you can click it as helpful.

 

Please let me know if you have any further query or concerns. Please to assist you.
