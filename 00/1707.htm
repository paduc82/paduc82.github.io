jllatham say:

MadaKnarf - if I may, without offending, make a comment about your code which does work.  A potential problem I see with it is that it marks the UsedRange on the sheet all the way out to column XFD.  This creates a larger than needed file when it is saved.  Basically it is saving all of the blank columns between the last one "really" used and XFD for all the rows used on the sheet.

So with dates down to row 33 and an entry in C13, after running your code if I save the workbook, it is 15KB in size on disk, but it would only be 9KB in size if we had not put the formula out in XFD1.  While this doesn't seem like much in these days of huge disks and systems with many gigabytes of memory, it does affect file save and load time and if the sheet had thousands of rows of dates in column B, then it makes a really big difference.

It definitely works, and shows another way of getting the task done.

Now, what you could do is 'steal' cell C13 to put your formula into!  I modified your code a little and this does work just as well:

Sub dynamic_dates()
  Dim mycount As String
      
  With Range("C13")
    .NumberFormat = "####0"
    .FormulaR1C1 = "=COUNT(R[17]C[-1]:R[9999]C[-1])"
  End With
  mycount = Range("C13").Value
  Range("B" & mycount + 17).Copy
  Range("C13").PasteSpecial xlPasteAll
  Range("A1").Select
End Sub
