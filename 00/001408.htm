Is there only going to be one station where people can sign off?

If there can be multiple signers, then you need a child table to hold each signoff. Have an unbound control on the form where the signer enters their password. In the After Update event of that control, you check the password against the table. Something like:

Dim varPW As Variant

varPW =DLookup(("UserID","tblUsers","Password = '" & Me.Password & "'")

     MsgBox "Password not matched!"

     Me.Signoff = varPW

End If

Have you examined the indexes of the tblRequirements table to see if there are any unique indexes?  This will not necessarily be a column of a number data type, or even a single column; an index frequently does contain more than one column, and in many cases such an index will be unique as the table is modelling a many-to-many relationship type between two or more tables..

jerry83560 say:

MailMerge data filename as merge field

I am using Mail Merge with Word 2010, and I want to include the Excel data filename (pathname/filename) in the output documents, i.e. as a custom merge field.

I have searched for a solution but without success.

Any tips or pointers would be appreciated.

graham mayor say:

The short answer is no, but you can do it with a macro.

Put the following code in the ThisDocument module of the merge document and save the merge document as a macro enabled document. Close then re-open the document (or run the Document_Open macro) to setup the events code. Insert in the merge document a docvariable field (use CTRL+F9 for the brackets) at the place where you want the field name to appear.

{ DOCVARIABLE varDataSource }

When you run the merge, the macro writes the datasource path/name to the variable and updates it to display the value in the document. I have left in a message box so that you can see something happening. But you can delete that if you find it a distraction.

Option Explicit
Dim WithEvents wdapp As Application

Private Sub Document_Open()
    'Get a reference to the Word application to handle mail merge events.
    Set wdapp = Application
lbl_Exit:
    Exit Sub
End Sub

Private Sub Document_Close()
    Set wdapp = Nothing
lbl_Exit:
    Exit Sub
End Sub

Private Sub wdapp_MailMergeBeforeMerge(ByVal Doc As Document, _
                                       ByVal StartRecord As Long, _
                                       ByVal EndRecord As Long, _
                                       Cancel As Boolean)
Dim sFName As String
    sFName = ThisDocument.MailMerge.DataSource.Name
    Doc.Variables("varDataSource").Value = sFName
    Doc.Fields.Update
    MsgBox sFName
lbl_Exit:
    Exit Sub
End Sub
