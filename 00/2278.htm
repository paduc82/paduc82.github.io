andreas killer say:
Therefore I want the least amount of user intervention and having them press an "Ok" button is out of the question. That's why I need to trap user input at the TextBox level and move them to the next phase without any delays.
Additionally, you can use the TextBox_Exit or TextBox_Change event to trap the user input, but I would not use it.

If you have a command button (Ok button) that finishes the process, set the Default property of that button to True.

If the user has made the input in the text box, he can press the ENTER key, which calls the Click event of the Ok button. The code in the Click event stores the values in the sheet and sets the focus to the text box at the end, so the user can enter the next value directly. That is a usual behavior of form windows in many programs.

And if he press ESC, the cancel button is called automatically if you set the Cancel property of the Cancel button to True.
