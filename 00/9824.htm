jay freedman say:
Quick answer: The .Name property of the shapes does not have to be unique in a document. The duplication you see isn't the cause of the corruption, and probably not even a symptom of it.

As proof, modify the following macro with filenames that exist on your computer and run it in a new blank document. There's no error when the same name is assigned to both shapes, and the Debug.Print to the Immediate windows shows that they are indeed named the same.

Sub x()
    Dim shp As Shape
   
    Set shp = ActiveDocument.Shapes.AddPicture( _
        FileName:="c:\pictures\nohumor.gif", _
        Left:=0, Top:=0, _
        Width:=InchesToPoints(2.5), _
        Anchor:=ActiveDocument.Paragraphs(1).Range)
    shp.Name = "picture1"
   
    Set shp = ActiveDocument.Shapes.AddPicture( _
        FileName:="c:\pictures\congarats.gif", _
        Left:=InchesToPoints(3), Top:=0, _
        Width:=InchesToPoints(2.5), _
        Anchor:=ActiveDocument.Paragraphs(1).Range)
    shp.Name = "picture1"
   
    Debug.Print ActiveDocument.Shapes(1).Name & vbCr & ActiveDocument.Shapes(2).Name
End Sub

It might be useful for maintenance to rename all the shapes in sequential order, but I don't think that will solve the corruption problem.

talking moose say:
If you're trying to access Hotmail's additional features such as calendars and contacts in Outlook for Mac then the answer is "no".

Outlook for Windows has a plug-in architecture that allows for software like the Hotmail Connector to access this data and ActiveSync for mobile devices can do this as well.

Outlook for Mac does not have a plug-in architecture and does not support more than just POP when connecting to Hotmail.

Please be sure to let Microsoft know you'd like to see this feature in future versions by using the Help --> Send Feedback mechanism in any Office application.
