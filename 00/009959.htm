When an account is upgraded to Outlook Mail, the only way to view the number of emails in a folder is to select all emails and then to mark all as unread.

Clicking on the check box to the left of the folder name at the top of the message pane will select all items in view.

Then you have the option to Select everything.

This changes to show that everything in the folder is selected instead of showing how many emails are selected.

Now that everything is selected, right click on a message and select Mark as unread.

The unread count will be the number of emails in the folder.

Right click on a message, Mark as read.

Click on Clear all check boxes.

What is needed is the actual count of messages selected.

Please describe exactly HOW the document looks terrible. Investigate to see if you can find specific features that cause the problem.

 

A possibility is that you are using 2010 features that are not supported in 2007. The most obvious one is Home tab / Font group / Text Effects command. In theory when you read the file in 2007 the effect should simply not be displayed.

It is a sad reality that moving documents between versions of Office is not a sure thing. Depending on the specific Word features you use, sometimes problems like those you describe happen. Another problem that can arise if you both edit the document is eventual file corruption. Neither of you will be able to use the file.

Just thought of another option. Create an empty file in Word 2007 and save it. Use the blank doc as the base to create new documents in 2010. In explorer, just copy the blank file, then open the copy in 2010 to create the doc. 2010 will open the 2007 file in "Compatibility Mode". That is OK, it just means that only 2007 features are supported.

Another option you can try is saving the files in RTF format. It is a slightly more portable format, but even that is not a guarantee, many years ago I tried it as a workaround and it only worked for a short while.

I think it is a minor failing of Office 2010 that the apps cannot create 2007 DOCX formatted files (as it can create 2003 DOC files), files that will run in compatibility mode.

Save as PDF IS the best answer for sharing finished files, especially resume's.

To expand on Doug's suggestion. If you both don't happen to have the same printer, decide on a common printer and make sure that you both haveexactly the same printer driver installed. If the printer driver is the problem, even if one of you don't can't use the printer to print, just selecting it may be enough to allow viewing.

 

Another thought, when you are sharing files, if you only need to read them, not edit or mark them up MS has 2007 and 2010 viewers that you can download to read the files. That may be enough to avoid the formatting problems you see.
