Hi Cullian,

This seems to be more of a settings issue on the Mac PC, for which I will recommend you to contact the Mac forums support or Mac technical support.

Once you have the settings done on Mac PC, you may follow the given steps for settings on Vista PC:

1.      Open Windows Vista's Network console (by clicking Start | Network).

2.      Networked systems should appear, including Apple PCs. Communication between both platforms isn't perfect, however. If you don't see an icon or entry for the Apple computer or computers, enter the Apple's IP address within Windows Vista's address field (using this format: \\10.0.0.12, substituting the IP address of the Apple system in question in place of 10.0.0.12) and press Enter. 3.      The Apple system should appear.

4.      Depending upon a few factors, including whether you've logged on to the Mac before and permission configuration, the Mac may prompt for a username and password of an account having permission to access the resource.

5.      The resources should then appear within Windows Vista; drag and drop files to move them between systems. You can also double-click a file to begin using it (just remember Windows must possess an application associated with the file capable of processing it).

 

In some cases, Windows Vista may refuse to process the username/password properly between the Windows and Mac. To enable access to shared Macintosh resources within Vista:

1.      Click Start, in Search box type “secpol.msc” without quotes and press Enter.

Note: Windows Vista will display a warning message; click Continue.

2.      Highlight Local Policies and Double-click Security Options.

3.      Scroll down to the Network Security: LAN Manager Authentication Level policy entry and double-click it.

4.      Change the value from the default setting of Send MTLMv2 Response Only to Send LM & NTLM -- Use NTLMv2 Session Security If Negotiated, then click OK.

5.      Close the Local Security Policy console.
