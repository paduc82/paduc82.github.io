ken j. metcalfe say:

I had the same problem.

One of the recent builds changed the device definition of the battery from the generic "Microsoft ACPI-Compliant Control Method Battery" to a MS Surface specific device, which has it's own driver - surfaceacpibattery.inf, which was only created this year and obviously needs a few bugs squashing.

Managed to fix it by uninstalling the Surface Battery in Device Manager  ( You must tick the "Delete Drivers" box on the uninstall confirmation screen, otherwise the same device will get  reloaded ).

Still in Device Manager, run "Scan for Hardware Changes".

The battery description will change to "Microsoft ACPI-Compliant Control Method Battery". which uses the generic battery driver - which works.
