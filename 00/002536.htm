jay freedman say:
The folks who design and maintain VBA aren't always consistent. In this case you might think that trying to access the value of a nonexistent custom property might raise runtime error 5825, "Object has been deleted", which is what you get if you try the same thing with a document variable. But no, you get Error 5.

The way around this is to iterate through the list of existing custom properties until either you find the one you want or you run out of properties to test. Try it this way in a new blank document:

Function CustomPropExist(WhichDoc As Document, NameOfProp As String) As Boolean
    Dim prop As DocumentProperty
    Dim ret As Boolean
    ret = False
    For Each prop In WhichDoc.CustomDocumentProperties
        If LCase(prop.Name) = LCase(NameOfProp) Then
            ret = True
            Exit For
        End If
    Next
    CustomPropExist = ret
End Function

Sub test()
    Dim PropName As String
    PropName = "Foo"
    If CustomPropExist(ActiveDocument, PropName) Then
        MsgBox "Value of " & PropName & " is '" & _
             ActiveDocument.CustomDocumentProperties(PropName).Value & "'"
    Else
        MsgBox "Custom property " & PropName & " does not exist... adding now."
        ActiveDocument.CustomDocumentProperties.Add Name:="foo", Value:="bar", _
             LinkToContent:=False, Type:=msoPropertyTypeString
        If CustomPropExist(ActiveDocument, PropName) Then
            MsgBox "Value of " & PropName & " is '" & _
                ActiveDocument.CustomDocumentProperties(PropName).Value & "'"
        Else
            MsgBox "Adding " & PropName & " failed." & vbCr & Err.Description
        End If
    End If
End Sub

Windows Media Player on WM5.0 and earlier windows mobile OS devices do not provide a way to create playlists on the fly, but if you have a more recent windows mobile device that is running WM6 or WM6.1, there is an option that lets you save playlists on the fly.  On a WM6 or WM6.1 device, queue up the songs you want to include on the playlist, go to the "Now Playing" window, check to make sure the songs are in the order you want, then tap Menu > Save Playlist.

ericbachtell say:

I have found a way to get them to not show.

If you would like to remove the ModernUI shortcuts from the All Apps section of your Start Menu you can

STEP 1:  Remove any unwanted shortcuts by removing folders located in this folder:

"C:\Users\%username%\AppData\Local\Microsoft\Windows\Application Shortcuts\"

Note:  The folder names are cryptic but easy to figure out.  Rather than remove them you can build a backup folder and move them there so if you want them in the future you have them. See Caution below.

STEP 2: Followed by removing shortcuts in:

"C:\Programdata\Microsoft\Windows\Start Menu\Programs"

STEP 3: And finally there's some remaining in:

"C:\Users\%username%\AppData\Roaming\Microsoft\Windows\Start Menu\Programs\System Tools"

Once all ModernUI shortcuts have been removed there will be no trace of ModernUI.

CAUTION: These shortcuts should be backed up, especially ones like MS Store in the event you need them. There is no restore option for this.

IF YOU ONLY WANT TO REMOVE ModernUI SHORTCUTS THAT HAVE (install) IN THE APPS LIST, YOU ONLY NEED TO REMOVE THE FOLDER IN STEP 1.
