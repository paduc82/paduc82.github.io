ken sheridan say: It's easily done in a report by setting the CanGrow property of the relevant controls, and the section in which the controls are located, to True (Yes).

ken sheridan say: The lstQuestions control is not a subform, but a list box control.  The lines in a list box can't be wrapped, so if your questions are too long to fit within the control, the only realistic solution would be to use a smaller font size and/or redesign the form so that you can make the list box wider.  At present the font size is 9 points, so you don't have a lot of room for manoeuvre there; 8 points is probably as small as you could go without making it too small to read comfortably.  Widening the control shouldn't be too difficult, though.  On my screen the form occupies around two thirds of the screen width, so its width could be increased significantly, giving you space to increase the width of the control, and the rectangle which surrounds it.

Theoretically you could use a subform in place of the list box, simulating the behaviour of a list box, but with greater control over the formatting.  My OneDrive folder does include an example, DummyList.zip, but it would require quite a lot of work, more than I have time available for I'm afraid, to incorporate a subform like this into my Questionnaire demo.

julie sheets say:
Hi Ivan,

When you create resources on the Resource sheet,you set the resource's max units.  100% max units means that for every hour of working time in the resource's calendar, he/she can generate 1 hour of effort.

Let's say you have a resource who is available to your project for 80 hours per month - or 20 hours per week.  I would set the resource's max. units to 50%.  For every 40 hours, the resource can generate 20 hours of effort.

As you begin assigning the resource to tasks, Project will assign the resource at 50% - or his/her max. units.  Lets say you assign the resource to a 2 week duration task at 50%.  For those two weeks, the resource is utilized at the max units - he/she has no other availability.  But, for the other two weeks, he/she isn't assigned at all.  You can view a resource's remaining availability in the Resource Usage view by adding the Remaining Availability field to the time-scaled portion. (right side).

I hope this helps.  Post back with more questions as needed.

Julie

rod gill say: What Julie said. If you have a training task, simply assign the resource at 100% for that task. Look at hours per week needed by week, not day for a better indication of whether a resource is over or under allocated.

If you set a resource to 2h/week availability then assign them to a task for 2h on a Monday, then that's the work for that resource for the week and no more work can be done until the following week.

Fixed duration tasks are high maintenance tasks, because of your resources don't have enough hours in the task's timeframe to do the work, you will get scheduling error messages. hansv mvp say:

You could use this function, originally from Steve Aprahamian:

Function VLookupAll(vValue, rngAll As Range, iCol As Integer, Optional sSep As String = ", ")
    Dim rCell As Range
    Dim rng As Range
    On Error GoTo ErrHandler
    Set rng = Intersect(rngAll, rngAll.Columns(1))
    For Each rCell In rng
        If rCell.Value = vValue Then _
            VLookupAll = VLookupAll & sSep & _
                rCell.Offset(0, iCol).Value
NextCell:
    Next rCell
    If VLookupAll = "" Then
        VLookupAll = CVErr(xlErrNA)
    Else
        VLookupAll = Mid(VLookupAll, Len(sSep) + 1)
    End If
    Exit Function
ErrHandler:
    Select Case Err
        Case 13
            ' Type mismatch - resume
            Resume NextCell
        Case Else
            VLookupAll = CVErr(xlErrValue)
    End Select
End Function

The formula becomes

=VLookupAll(A2, Sheet2!A1:A100, 1)

Adjust the range A1:A100 as needed, but do not use an entire column in this formula!
