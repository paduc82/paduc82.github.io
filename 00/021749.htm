Just means that the .ost data file associated with the IMAP account has been set as the "default data file" for the profile.

The "default data file" contains the default contact/calendar folders for the profile. Go to OL menu --" File --> Account Settings --> Data Files tab --> highlight the PST file that has the contact/calendar folders you want to use and set it as the default"

To copy all the contact/calendar items from within the folders labeled with "This Computer Only"

- open the applicable folder (in the case of Calendar items - set the folder to a "List View"

- press CTRL+A to highlight all the items

- right-click the highlighted items --> Move --> Copy to folder -- select the destination folder

rinse and repeat for each folder involved

Once you have confirmed that everything has been copied over completely and properly, then the easiest way to remove the "This Computer Only" folders from the IMAP folder tree is to remove the account the profile and re-add the account back at which point all the email folders will sync again and the "This Computer Only" folders will not exist in the folder tree.

wo things you need to remember when defining a Where Clause for a Navigation Control

1) The Parent and Child Controls CANNOT be of the same name.

2) The Child Control MUST be an actual Field in the Recordset and be a bound Control.

If both Controls have the same name, then add a new control bound to the same field but with a different name. For instance, if [ProductID] is on both the parent and child, add a new control on the parent form and name it [txtProductID]. Then add [txtProductID]=[ProductID] to your WHERE Clause.

One last bit of advise, do NOT precede the Clause with an '=' equal sign, enter it simply as this:

[txtProductID]=[ProductID]
