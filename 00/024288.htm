macropod say:

Hi tantree,

You could add the following two macros to Word's 'Normal' template. The AutoOpen macro goes into an ordinary code module in the template; the Document_New macro goes into it's 'This Document' module. The AutoOpen macro has some extra lines for ensuring rulers are displayed and for showing the document path in the title bar. You can delete those lines if you don't want them.

Sub AutoOpen()
With ActiveWindow
  'Reduce flickering while changing settings
  .Visible = False
  'Switch to Print Layout view
  If .View.SplitSpecial = wdPaneNone Then
    .ActivePane.View.Type = wdPrintView
  Else
    .View.Type = wdPrintView
  End If
  With .ActivePane.View.Zoom
    'Set for a 1-page view
    .PageColumns = 1
    'Initialize for best fit
    .PageFit = wdPageFitBestFit
    'Test zoom % and reduce to 100% max
    If .Percentage > 100 Then .Percentage = 100
  End With
  'Display the Rulers
  .ActivePane.DisplayRulers = True
  'Displays the Document Path in the Title Bar
  .Caption = ActiveDocument.FullName
  'Restore the window now that we're finished
  .Visible = True
End With
End Sub

Private Sub Document_New()
Call AutoOpen
End Sub

rob koch say:

Since as Steve already stated, these submissions via MAPS are typically quite rare unless you are constantly encountering malware, this really shouldn't be a big deal.

However, the thing to realize is that in most cases, even if something you encounter is submitted to MAPS, no human will ever actually see it.  This is because the MAPS system is huge and collects large numbers of submissions from many different PC systems around the world every day.  Only a tiny number of these would ever be looked at individually, since often hundreds or even thousands of the same malware might be collected within hours.

So even if MAPS does request that you send a submission including a file, which it will prompt you for if that might contain personal data, the only real reason to be concerned is if that file actually contains something you'd rather no one else have access to.  For example, truly personal data like identity (SS#, address, phone, name, etc.) or private business data.

In most cases, the data requested is actually a file received via something like an email attachment or web download that wasn't truly personal, but might be infected and even a fake document or other attempt to trick you into infecting yourself.  When web pages containing either infected files or links are encountered, these typically generate only a tiny upload containing the URL to the offending content and some additional data such as the IP address of the website to better identify the source.  As the Privacy Statement you read indicated, these rarely contain any private data beyond possibly a personal portion of a link, such as if you were accessing a personal account when the malware was encountered.

Since most such web based malware is actually found on public pages, it's not common for these to contain such private data within the link, so it's not really something I've ever been concerned about.  In fact, I've so rarely been prompted to make a submission that I truly can't remember the last time it occurred, even when I did purposefully attempt to access malware to test the operation of MSE/Defender, since they'll only typically request a file submission if the [potential] malware is something either very new or unknown that's acting like malware.

If MAPS does prompt, the name[s] of any files they are requesting will be identified clearly in the request so you can decide whether to send them.  Other details aren't typically identified, but don't really matter since they'd obviously be related to the files identified in some way, such as the full path to the folder or similar related information.  Typically the file itself would contain the only potentially private information of value.

Rob

The “MP Scheduled Scan”entry that already exists in Task Scheduler is for the scan scheduled within Security Essentials itself, which runs with a blank “Start in” field, and as “NT Authority\System”. If you want to change the configuration of the internally scheduled scan, you would use the Settings tab in Security Essentials, as dreamsburned suggested. If you want to set up an independent scan in Task Scheduler you would run the Scheduled Task Wizard by double clicking on “Add Scheduled Task”.

The syntax that’s used in the MP Scheduled Scan entry (which you presented) should work, but it will run a default type scan, which might not be what you want. Since I have a Quick Scan set up in Security Essentials and wanted to run my Full Scans in Task Scheduler in order to avoid CPU throttling and cut down the scan time (on my desktop), I set up my scan with the Wizard (double click on “Add Scheduled Task”), selecting “Microsoft Security Essentials” and then replacing “msseces.exe” with the documented command line for a Full Scan:http://social.answers.microsoft.com/Forums/en-US/msescan/thread/809b0b84-9396-49b2-95c4-68e7ec539fc7

So the paths look like this:

Run: C:\PROGRA~1\MICROS~2\MpCmdRun.exe -scan -scantype 2

Start in: C:\PROGRA~1\MICROS~2

Aiscer is correct; the workaround that I previously specified for a blank password only works with Windows XP Pro. The recommended solution for the Home version would be to use a password: http://technet.microsoft.com/en-us/kb/kb00310715.aspx

With a computer that has a low level of resources, scheduling your scans in Security Essentials itself will conserve resources and improve the responsiveness of the system because of the CPU throttling applied, but the scans will run twice as long. If your computer has ample resources, then scheduling the full scans in Task Scheduler will speed things up without too much impact on system performance. Obviously a switch for the CPU throttling feature would make things a little easier for people with more powerful computers.

As a footnote I should add that the internally scheduled scan entry was not appearing in Task Scheduler on my desktop, and I just discovered that this was because I had unchecked “Start the scheduled scan only when my computer is on but not in use”. That’s a little confusing.

Update and summary for Windows 7/8/8.1 and 10 

How to create a scheduled task that will wake up the PC to run a scan; keep the PC awake while the scan is running; shut the PC down when the scan is done; and run the scan later if the PC isn’t powered on at the scheduled time:

1.) Open Task Scheduler:

In Windows 7, click on the Start button, type “task scheduler”, and then hit the Enter key.

In Windows 8/8.1/10, press the Win +R keys, type "taskschd.msc" in the Run box, and hit the Enter key.

2.) Click on the Action menu and select the Create Basic Task option.

3.) Name the task something like “MSE (or Windows Defender) Full Scan”. Click Next.

4.) Select Weekly (for example) for the task trigger and then click Next. Pick a time and day, and then click Next.

5.) We’re going to use the default action – Start a Program, so just click Next. Enter (copy and paste) the appropriate file path (with quotation marks) in the Program/script box:

       "C:\Program Files\Microsoft Security Client\msseces.exe"     is the file path for MSE.

       "C:\Program Files\Windows Defender\MSASCui.exe"     is the file path for Windows Defender.

And then enter (copy and paste)      -FullScan      in the Add arguments text box. Click Next.

6.) In the Summary page, check the Open the Properties dialog… check box, and click Finish.

7.) In the General tab of the Properties dialog, click on the Change User or Group button, type “system”, and click OK.

8.) Optionally, we can add a shutdown action for the task: Click on the Actions tab, and then on New. Then enter (copy and paste)     Shutdown     in the Program/script text box and     -s -t 300     in the Add arguments text box. 

9.) Click on the Conditions tab and check the Wake the computer to run this task option.

10.) Optionally, we can enable a “catch-up” scan if you want missed scans to run at the next opportunity: Click on the Settings tab, and check the Run this task as soon as possible after a scheduled start is missed option and click OK, and you’re done with the setup.

Note: Both of the above options are enabled for the built-in Windows Defender daily Quick Scan (in Windows 8/8.1), and they’re essentially what’s responsible for the greatly increased reliability of this fully-automated scan, relative to the built-in MSE scheduled scan. 

Parameter options for Microsoft scheduled scans:

Documented arguments for a command line scan: MpCmdRun.exe

       -Scan -ScanType 1     …for a Quick Scan

       -Scan -ScanType 2     …for a Full Scan

       -Scan -ScanType 3 -File <file path>     …for a Custom Scan of a specific file, folder, or drive

Undocumented arguments for a command line scan with a prior signature update: MpCmdRun.exe    

       -Scan -ScheduleJob     …for a Quick Scan w/update

       -Scan -ScheduleJob -ScanType 2     …for a Full Scan w/update

Undocumented arguments for a user interface scan: msseces.exe for MSE or MSASCui.exe for Windows Defender 

       -FullScan     …for a Full Scan

       -QuickScan     …for a Quick Scan

       -UpdateandQuickScan     …for a Quick Scan w/update

For an example of what John's mentioned:

Start a new presentation at 4x3 size.

Enter some text into the title placeholder.

Start another new presentation, same size.

Enter the same text into the title placeholder.

Now change the slide design to make the slide size 40 x 30.

The text in both slides will appear to be the same size, but select the text in the 40x30 version of the presentation and take a look at its size on the Home tab. WAY different, because PPT has kept the proportions of the text vs slide constant.

If you then make the text size the same as in the first presentation, it'll be tiny.  Though the point size is now the same, it's on a much larger "field" so seems smaller. 

john korchok say:

Fonts don't change point sizes. 12 points is 12 points is 12 points.

But people (especially graphic designers) build PowerPoint slides in all kinds of wacky sizes, which makes a given point size appear larger or smaller by comparison. Using Design>Slide Size>Custom Slide Size, you can find the dimension of the ones you're working with.

Standard slide sizes are 13.333" x 7.5" for a widescreen and 10" x 7.5" for a 4:3 presentation.

karl timmermans [outlook mvp 2012-2018] say:

Short answer - you can't.

AB is a view that cannot be modified in terms of the fields presented.

karl timmermans [outlook mvp 2012-2018] say:
Getting included in the AB is based on having a valid email / fax address. Your only choice is to set the AB display names to the FileAs field making sure that the FileAs field in turn is set to <Last, First (Company)> or <Company (Last, First)> which is the only way that every contact has an identifier (equally presuming that your contacts have at least a value for either Name or Company. Also note, the new default format will only take effect the next time Outlook is opened so close and restart Outlook for it to be applied immediately.

If the current default FileAs setting is not set for one of the above, you will have to change it but keep in mind that this setting will only apply to newly created contacts - the FileAs value for existing contacts will not change - you will have to do that either via code or use a 3rd party utility

To change the FA value manually via code
Change Contact's File As format
http://www.slipstick.com/contacts/fileas.asp 

Alternative that may be of interest is the FileAs standardization function in 

ContactGenie Toolkit
http://www.contactgenie.com/toolkit_4_outlook.htm


To change your AddressBook to display as per the FileAs value instead of First/Last Name

From O'2010 menu bar 
File --> Info --> Account Settings --> click on AddressBooks tab 
     --> highlight Outlook Address Book --> Change 
          --> select FileAs under <Show Names by>
