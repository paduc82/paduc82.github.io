yang zheng msft support say:

Hi NeilPotter,

Here are the answers to your questions:
1. Will the migration work if users account log ons are based on a .local AD setup, or is there a fix to get around it?
Yes, the migration will work. You dont need to implement Azure AD connect or ADFS before perform cutover migration.

2. Once we have ran the first migration and everything has synced over, should we keep the migration running until we have
a) setup all users against O365

b) DNS has replicated.
The answer is yes. As Office 365 mailboxes were synchronized at least once after mail began being sent directly to them, you should make sure that the value in the Last Synced Time box for the migration batch is more recent than when mail started being routed directly to Office 365 mailboxes. Then stop and delete the migration batch so that all users are using Office 365 mailboxes. After the batch is deleted, mail sent to mailboxes on the on-premises Exchange Server isn't copied to the corresponding Office 365 mailboxes.

3. What happens if a user still connected to the in house exchange sends an internal email.  Is that delivered/synced to O365 immediately, or does the migration / sync run every so many minutes / hours?
The synchronization process will keep running until you delete the migration batch. After the first synchronization or called initial synchronization is complete, the mailboxes in the on-premises Exchange organization and the corresponding Exchange Online mailboxes are synchronized every 24 hours. This is called incremental synchronization. Before you have pointed the MX record to Office 365, the emails will firstly go to the on-premises server and then sync to Office 365 during the incremental synchronization period.

For more details about cutover migration, you can refer to the following articles:
https://technet.microsoft.com/en-us/library/jj898490%28v=exchg.150%29.aspx?f=255&MSPPError=-2147217396
https://support.office.com/en-us/article/Perform-a-cutover-migration-of-email-to-Office-365-9496e93c-1e59-41a8-9bb3-6e8df0cd81b4?ui=en-US&rs=en-US&ad=US
