Are text boxes any good at providing line numbers for alternating - Microsoft Community

galahad72 say:

Are text boxes any good at providing line numbers for alternating margins?

Word provides line numbers but only in the left margin. We're used to printing them in alternating opposing margins. On a left-hand page, they're in the left margin. On a right-hand page, they're in the right margin. How about using text boxes instead? Once they're sized the way you want want them, they can be stored as building blocks and invoked manually for successive pages. Also saved, of course, are the size and font of your line numbers. If a short page is faced with too many line numbers in the text box, just delete them; and their position can be adjusted by hard returns for blank lines.

I did see a criticism of this approach but can't rediscover it here. I tried frames, but in manual application they're more work than text boxes. Frames might be more suitable for indicating the original page numbers in the other margin, as is sometimes found in scholarly editions.

What could go wrong in applying pre-line-numbered text boxes? Perhaps positioning them uniformly would be too much of a chore if the text is changed? The text boxes, unlike Word's own line numbers, shift down with the insertion of blank lines, which is OK when new text is inserted.
charles kenyon say:
Earlier versions of Word did line numbering using textboxes in headers and footers. You could put one set in an odd-page header/footer and the second in an even-page header/footer.

What could go wrong? You could have different spacing in your body than the page numbers in your text box.

Your text boxes, of course, would have no borders.

Since graphics filters are shared files, a simple reinstall
of a program will not update the filter.

First...you must rename the filter in question and then
reinstall the software...this way a fresh copy of the filter
will be installed.

The .WMF filter is  wmfimp32.flt  you could do a search
for  wmfimp32.flt  or you could navigate to the following
location:
 
C:\Program Files\Common Files\Microsoft Shared\grphflt\wmfimp32.flt

Rename the filter from wmfimp32.flt to wmfimp32.OLD.
Now reinstall MS HomePublishing 2000.

FWIW...if you can spare the hard drive space...there's a
way to install the complete Home Publishing program to
your hard drive and eliminate disk swapping when you
need graphics/artwork.

#15. Installing onto a network or installing everything to

jean-guy marcil say: Using Graham's code as a base, here's an option.
It is kind of convoluted because it uses the InputBox function, it would be cleaner with a form, but that would require more code...

Sub Resize()

Dim strHeight As String
Dim strWidth As String
Dim sngHeight As Single
Dim sngWidth As Single
Dim boolSingle As Boolean

Const strQuestion As String = "What size do you want for the "

boolSingle = False
Do
    strHeight = InputBox(strQuestion & " height?", "Resize - Inches")
    If strHeight = "" Then Exit Sub
    boolSingle = IsNumeric(strHeight)
    If Not boolSingle Then
        MsgBox "You must enter a numerical value.", vbExclamation, "Not a number"
    End If
Loop Until boolSingle = True

boolSingle = False
Do
    strWidth = InputBox(strQuestion & " width?", "Resize - Inches")
    If strWidth = "" Then Exit Sub
    boolSingle = IsNumeric(strWidth)
    If Not boolSingle Then
        MsgBox "You must enter a numerical value.", vbExclamation, "Not a number"
    End If
Loop Until boolSingle = True

sngHeight = CSng(strHeight)
sngWidth = CSng(strWidth)

With Selection
    If .InlineShapes.Count > 0 Then
        With .InlineShapes(1)
            .LockAspectRatio = msoFalse
            .Height = InchesToPoints(sngHeight)
            .Width = InchesToPoints(sngWidth)
        End With
        Exit Sub
    Else
        With .ShapeRange
            .LockAspectRatio = msoFalse
            .Height = InchesToPoints(sngHeight)
            .Width = InchesToPoints(sngWidth)
        End With
    End If
End With

End Sub

graham mayor say:
If you need to be able to input the sizes then you will need to introduce some error correction and using an input box to collect the data is not ideal, but

Sub Resize()
Dim strHtWid As String
Dim vHtWid As Variant
strHtWid = InputBox("Enter the height and width (in inches) in the format " & _
                    "'0 x 0'. i.e. two numbers separated by 'x' with or without spaces.")
If strHtWid = "" Or InStr(LCase(strHtWid), "x") = 0 Then GoTo ErrorHandler:
vHtWid = Split(LCase(strHtWid), "x")
If IsNumeric(Trim(vHtWid(0))) = False Then GoTo ErrorHandler:
If IsNumeric(Trim(vHtWid(1))) = False Then GoTo ErrorHandler:
With Selection
    If .InlineShapes.Count > 0 Then
        With .InlineShapes(1)
            .LockAspectRatio = msoFalse
            .Height = InchesToPoints(Trim(vHtWid(0)))
            .Width = InchesToPoints(Trim(vHtWid(1)))
        End With
        Exit Sub
    Else
        With .ShapeRange
            .LockAspectRatio = msoFalse
            .Height = Trim(vHtWid(0))
            .Width = Trim(vHtWid(1))
        End With
    End If
End With
Exit Sub
ErrorHandler:
MsgBox "Incorrect or missing data"
End Sub

should work. I don't suppose that you want to enter the values in points, so the example uses inches. Enter the values in inches with decimals as required the two numbers separated by x or X, with or without spaces.

If you prefer to work in centimeters then replace the three instances of Inches with Centimeters (use the replace function - CTRL+H - in the vba editor).
