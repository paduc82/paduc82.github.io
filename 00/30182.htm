SOLVED!

After 3 hours of troubleshooting, I have found the solution to this problem!

It is caused by a corrupt or damaged boot option in the MBR.

It is especially common on hard drives that contain multiple partitions (ie. almost all OEM installs).

Sometimes the MBR just becomes damaged, other times a boot virus is the cause.

 

In any case, heres what fixes it:

Boot into recovery manager via a Win7 DVD or via the built in system repair option.

If you use the Win7DVD, click next and then choose the option to Repair your computer.

If you use the built in system repair option, cancel any automatic repairs that may start.

Choose the advanced recovery options, and then choose the command prompt.

Enter the following command: bcdedit /enum

 

This is the tricky part. Your boot manager may be different depending on the OEM or installer.

Identify all the installation references. There should be more than one. Mine had 2.

One of these is a valid record, the other is corrupt/invalid. We need to delete the invalid record and then rewrite it.

My 2 records were {bootmgr} and {default}

I'm not sure about this, but I believe it will always be the {default} record that is corrupt. (It was for me)

So, let's assume I'm correct and the {bootmgr} record is always generic and the {default} value tends to be corrupted.

 

Enter the following command: bcdedit /delete {default}

The command should complete successfully.

 

Now that we've deleted the corrupted record, we need to allow windows to identify and rebuild the correct one.

Enter the following command: bootrec.exe /scanos

Windows should scan for any Windows installations on all drives, and it should indentify at least 1.

Choose y to add it to the boot manager.

The command should complete successfully.

 

Now lets clean up and make sure Windows has some defualt boot settings.

Enter the following command: bootrec.exe /fixboot

The command should complete successfully.

Enter the following command: bootrec.exe /fixmbr

The command should complete successfully.

 

Restart. Done! Let me know if it worked for you! "

considerate_guy say:

This is exactly the expected behavior of Movie Maker.

When you add a video clip to your storyboard, Movie Maker takes the first frame and uses it as the "thumbnail" for that video. That thumbnail will get repeated multiple times if necessary to fill up the rectangle, but that doesn't mean your video is made up of a single frame.

If you play that video, you'll see the playback marker move along the storyboard and Movie Maker shows the actual movie to the left in the preview area. Use the preview area to figure out what is in the video clip and to make edits.

Now, if you were to split that clip at some point, Movie Maker would create a new thumbnail for the split portion with the first frame in that newly split video. 

Another thing -- if you find this expanded view of your video confusing, you can always use the zoom slider in the lower right corner. Drag it all the way to the left and you'll only be presented with a single small thumbnail for each individual clip.

In any case, there is nothing wrong. This is exactly the way Movie Maker is designed to behave.

Hopefully some of the Microsoft moderators like Syed are reading this and won't give a misleading reply next time. They could probably spend a little more time using Movie Maker so they are familiar with its features and not just assuming that there is an issue with the format of the file. I honestly think they just have some generic response templates and they just use whichever one seems closest.

Update:

One further note. It seems like you may have previously created this as a project where you had each of the individual elements showing on the timeline. But then you saved it as a movie. If you reopen that movie in Movie Maker, it creates a new project and shows a single video representing the movie.

If what you really want to do is to make further edits, you must be reopening your *project* file, not the finished movie. Go to the File menu and look to the right. You'll see the list of recently opened projects. Find the project file (.wlmp) that represents this project and reopen it. It should then show you all the individual elements.

Summary:

Project file (.wlmp) -- a text file containing the editing state of your project. It has a list of the media files used, where you trimmed and split files, what effects/transitions/titles you have added, etc. It is the project file that you want to use when you need to make further edits. The project file can't be played, shared, etc. because it only contains text, not any actual media. It's like the recipe for making your movie.

Movie file (.wmv, .mp4) -- a video/audio file containing the actual edited version of your movie. This is the final output of your editing where it follows your instructions and creates a new movie. It has the video/audio/etc. all merged together into a single file that you can then play in a media player, shared with others, etc. But it can't really be edited down to the individual elements anymore. It's like the final baked cake you make from following the recipe.
