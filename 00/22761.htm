thezar say:

Thank you for this pointer. Seems that this does, indeed, fix the problem (for me, at least).

For my setup (Windows 8.1, Surface Pro 3, Office 365 - Office 15) the specific instructions are few and simple:

1) Go to C:\Program Files\Microsoft Office 15\root\office15

2) Right-click on powerpnt.exe.manifest and change the permission so you can modify the file

3) Right-click on powerpnt.exe.manifest and open with Wordpad

4) Change the line

    <dpiAware>True/PM</dpiAware>
    to

    <dpiAware>False/PM</dpiAware>
5) Save file

(at least with Office 365, you don't need to do anything in the registry as it's already looking at the external manifest file.)

Yes, line 4 seems counter-intuitive, but it's correct. Once you change this to False, Powerpoint will scale moving from high to lower DPI monitors. I've not encountered any other issues, but I've not used it for very long, yet.

Thanks to the previous pointers/authors for this tip.

    suzanne s. barnhill say:

    You can include the word "Lesson" as part of the numbering. For an example, see the multilevel list that has "Chapter 1" as the first level. You can type a limited (but fairly generous) amount of text before or after an auto number in the "Enter formatting for number" box.

    In this particular instance, however, it will definitely be tricky because you've already got "Lesson" as the text of the first line and a number (either auto or manual) as the content of the second. (BTW, I wouldn't advise wasting built-in headings on all of these paragraphs unless you've just got plenty to spare.)

    Perhaps someone else can suggest another approach, but the only options that occur to me are:

    1. If you don't have lower-level numbering based on the lesson number, then type this number manually instead of using auto numbering. The "1" will then be plain text, and you can add "Lesson" and a space as "numbering" without any actual number ("Number style" set to "(none)").
    2. You could use a TC field to supply the content for that TOC entry. Since you could include the word "Lesson" in the TC field, you wouldn't even have to bother adding "numbering" to the TOC style. One approach would be to type the number in the TC field manually (continuing to use an auto number for the actual paragraph). Another would be to use a REF or StyleRef field to pick up the autonumber in the TC field. Note that if you choose a StyleRef field, you have to put the TC field in the following paragraph (assumed to be in a different style); otherwise you'll get an error message.

    The second option would require a lot of monkeying with the TOC field to fine-tune it. At the very least you'd need to ensure that the TOC includes TC fields (represented by checking the "Table entry fields" box in the Table of Contents Options dialog) and otherwise omits the entry for that particular style (not a problem if it's not a built-in heading style and has an outline level of "Body text").

    I'm sorry if that all sounds very confusing. It is very confusing, and there will doubtless be a good bit of trial and error involved, but if you conquer this challenge, I assure you you will feel a terrific sense of accomplishment!
