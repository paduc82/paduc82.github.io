Cutover Exchange Migration and Single Sign-On  (if you don’t have SSO, please ignore the step 3.)

erickchu_msft say:

Hi TylerD,

If you use Exchange 2010, there are no official documents about converting mailboxes to mail users. Officially, we recommend Exchange 2010 users to set up hybrid. Your understanding is highly appreciated.

About “To be sure, for a soft match : the mailuser only need the cloud ProxyEmailAddresses, PrimarySMTP?”, the answer is yes. You need to use SMTP addresses to match them. You can click the link below to follow the detailed steps:

How to use SMTP matching to match on-premises user accounts to Office 365 user accounts for directory synchronization


Question 1: There is benefits to do a GUID match against a PrimarySMTP soft match?

Generally, Soft Match is easier and more widely-used. For Hard Match, it’s a little bit complicated. It requires the sync tool being disabled to change the Office 365 users ImmutableID. For example, after you finish some users' Hard Match and enable the sync tool, if you want to hard match other users, the DirSync needs to be disabled again. Moreover, converting the objectGUID to ImmutableID is complicated too.
However, Hard Match can be used in some specific scenarios. For example, the primary SMTP address cannot be set to the Office 365 users with special licenses, such as EOP standalone.


Question 2: If i add the msExchMailboxGuid attribut from the Cloud to the Onpremise Mailuser, AADConnect will do a GUID matching?

For the Hard match, users should use ObjectGUID. Therefore, your question’s answer is no.


Question 3: What append with Non Routable UPN defined Onpremise, when we do a AADConnect Sync? (AADconnect will change the cloud UPN, or keep the UPN (equal to PrimarySMTP) migrated with Cutover?)

Do you mean any local domain email addresses? If so, these domains will not be synced.


Question 4: Can we force ADDConnect to do a GUID or Soft match?

Hard match and Soft match are different. When you choose any one method and finish it, AAD connect will sync it automatically.

Best Regards,
Erick
