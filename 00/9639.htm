graham r seach say:
If you want to add a bunch of names in batch, you need two things. Firstly, you need a button to 'select all', and secondly, you need code to add the records.
 
But before we implement this, I recommend altering the way you do things. Right now, the instant you select something from the listbox, your code automatically adds a new record. That's problemmatic because it's possible, nay easy, to add an unwanted record by accidentally selecting the wrong listbox item. As far as selecting multiple items is concerned, what if you want to select all but one? In that case, the easiest way is to select all the listbox items, then deselect one of them. What I'm saying is that you need the flexibility to freely select items without fear of inadvertently creating records. Only when you're happy with your selection do you want to create records.
 
So...Add two buttons to the form (cmdSelectAll and cmdDeselectAll). Add the following code to the respective buttons.
 
Private Sub cmdSelectAll_Click()
    Dim intIndex As Integer
   
    If Me.lstMyList.ListCount > 0 Then
        For intIndex = 1 To Me.lstMyList.ListCount
         ;    Me.lstMyList.Selected(intIndex) = True
        Next intIndex
    End If
End Sub
 
Private Sub cmdDeselectAll_Click()
    Dim varItem As Variant
   
    For Each varItem In Me.lstMyList.ItemsSelected
        Me.lstMyList.Selected(varItem) = False
    Next varItem
End Sub
 
Then add another button (cmdAddRecord), and the following code. WARNING: Untested.
 
Private Sub cmdAddRecord_Click()
    Dim varItem As Variant
    Dim strSQL As String
 
    For Each varItem In Me.lstMyList.ItemsSelected
        strSQL = "INSERT INTO myTable ([Name], MR, Unit, Area, Admitted, Alert) " & _
                        "VALUES (""" & Me.lstMyList.Column(0, varItem) & """, """ " & _
                                "Me.lstMyList.Column(1, varItem) & """, """ " & _
                                "Me.lstMyList.Column(2, varItem) & """, """ " & _
                                "Me.lstMyList.Column(3, varItem) & """, """ " & _
                                "Me.lstMyList.Column(4, varItem) & """, """ " & _
                                "Me.lstMyList.Column(5, varItem) & """)"
 
        DBEngine(0)(0).Execute strSQL, dbFailOnError
    Next varItem
End Sub
 
Then get rid of your existing cboNames_Click() procedure.
 
I can see that your form is bound to the same table to which your code is adding records. Unless there are other controls on this form (aside from the textboxes listed in your code), you can get rid of all of them, an it'd probably be OK to unbind the form.

wildlyirrational say: I am not sure how to propose a solution but this is what I did to get the flickering on IE8 resolved.

It happens on both of my laptops and on web-sites rendering Adobe Flash, also javascript mouse hover-over flickers such as pages running Dynamics CRM and Silverlight (i.e. Netflix). The problem is with the video drivers. Even updating the drivers from manufacturer website does not resolve the issue. What you need to do is get the driver from the graphic card site, such as ATI Technologies. Then do a clean uninstall of all display drivers first, reboot and re-install.

OS: Win7 Ultimate 64-Bit & Win7 Enterprise 64-bit
Video: ATI Mobility Radeon HD 3400 Series (Lenovo laptop) & ATI Mobility Radeon HD 3670 (Dell Studio XPS 16)

1) Go to your graphic card vendor download and save the driver. DO NOT INSTALL YET.
2) Open Device Mgr and uninstall your display driver.
3) Open Add/Remove programs and uninstall any display driver software, i.e. ATI Catalyst Control Center.
4) At this point your screen is in low VGA mode.
5) Reboot
6) When Windows load it will try to install a VGA driver from WDM. Let it go.
7) Install the driver you download from step 1.
8) Reboot

Good luck!

I just found a solution for the problem:

First start a new email or message and do the delay delivery option to whichever future date or time, let called this message or email Test

Once you hit send with the delay delivery option, the message will go to your Outbox folder, and if you go to your Outbox folder, in the sent column it should read the "None." DO NOT ERASE "Test " email or message - - the next techniques are the solution

Now, without erasing "Test" go back and start another message or email, this time called it Test-2, and for this email, do the delay delivery option to a future time, say 2 minutes pass your current time, which should be enough time to hit send.  Here is the tricky part, when you hit the send button for Test-2, go back to the outbox, and do not touch, preview, select or point to open Test-2 message.  Because if you do, Test-2 will go to None in the sent column.  Wait for the two minutes, and the delay delivery will work, also you could see the Test-2 sent column with the date and time when you hit the send button.  Other points, if you should need to revise or edit Test-2 email, since it will go back to None in the sent Column or you will lose the delay delivery option, you will need to start over again.  You can try with real emails or message and it should work, now keep in mind you will need always the Test email in the outbox.
