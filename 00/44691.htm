darkequus say:

"...USER account MUST be created in order to share files between computers in a household WORKGROUP."

You are largely correct as far as implementing a user-level account and password file access scheme.  The one fact you may have missed is that you do NOT have to log into that shared account in order to share files...you just have to log into the folder you shared using that account credentials and saving that in the accounts you prefer to use.

This is the style of sharing I set up on my home net also, after running into an intractable problem of getting Homegroup to "sync" across a network switch.  An aside. I suspect that was the real reason Microsoft yanked Homegroup out of the Win10 Fall 2018 version, but that is only my speculation.

Back at the ranch, I set up sharing my music collection this way...I have one OLD computer (running Windows 7, and can't even be upgraded to 8.0 or 8.1, as it is 17 y.o. and counting) that I use as my file and print server.  So, first, I logged in as Administrator on this Ancient One, and created the account I wanted to use for sharing. pairing the  account name with a password.  Then, I shared the music folder on this computer to EVERYONE, but I require those users to log onto the server  with that "shared" account information.  I also shared the attached printer to use this same account.  My third step was to then to go to each of my other computers and used File Explorer (all of my "client" computers are running Windows 10) to find my "server" computer on the network, then drilled down to the shared items.  When each computer first connects to a resource shared on the "server" in this way, it should prompt for the account and password that was set up for sharing.  Enter those, and click "remember my credentials" before you hit enter.  That way, even after rebooting. you shouldn't be required to enter them again.  and since the printer was shared using the same account, simply clicking to connect, and maybe installing the printer driver does not generate a second prompt for the log in info.  All done!

A couple of things bear watching.  First, the account I created has never been logged into on any of my "client" computers.  I DID create the same account with the same password on each, but I'm not entirely sure it was necessary.  So I've three computers with basically empty profiles for that account.  Secondly, I did make sure the same WORKGROUP name was used on every computer, Without that, I don't think I'd have been able to "discover" the "server" computer, even though it is physically sitting on the same network wiring.

Using this method gives you at least a couple of options, as far as "ownership" of the music files.  You can limit the shared logon account to just reading the shared folder, which makes you have to add music only on the computer from where it is shared, or you can give that account read and write access, meaning any computer logged in can add music to the shared folder.   I would reserve full access privileges to only the "server" computer, as a precaution.  Just a heads up.
