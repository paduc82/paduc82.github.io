jeffconrad say:

Hi,

Sure, we can help you get started on this. Im currently in the process of creating some Help articles on making data macros in Access web apps so unfortunately I cant just easily point you to those articles Im working on.

Lets take a simple example which you can study and then adapt to your specific needs.

Assume I have two tables called Table1 and Table2. They each have identical fields of ID, Field1, and Field2. (They dont have to identical structures but for the purposes of this example, it will make it easier.) And in this example we want to copy all the records from Table2 into Table1.

To start creating a named data macro, open your app in Access and then click Advanced > Data Macro in the ribbon.

Access opens a new data macro design window. In the Add New Action box on the design surface, select the action called ForEachRecord. Youll see Access add this action to the design surface and any arguments it has. Next, In the For Each Record In argument, select Table2 as that is the table that has all the data we want copied. Leave the Where argument blank in this case. This action will essentially then loop through every record in Table2 and then do something. Well do the do something part next.

Within this ForEachRecord block youll see another Add New Action box. In this box, select the action called Create Record. Youll see Access add this action to the design surface inside the ForEachRecord block and any arguments it has.

Next, In the Create A Record In argument, select Table1 as that is where we want to copy the records into. Within this CreateRecord block youll see another Add New Action box. In this box, select the action called SetField. Access then adds this action into the Create Record block with two arguments called Name and Value.

For the Name argument, enter [Table1].[Field1] and for the Value argument enter [Table2].[Field1] which essentially means that you want to set (or fill if you want to think of it that way) the Field1 field in Table1 with the value found in Field1 of Table2. As you start typing the word table in those arguments, notice how Access helps you with IntelliSense to help guide you along.

Now right below the SetField action add another SetField action from the Add New Action box so you have a second one. For this second SetField action well copy the values of Field2 over so enter [Table1].[Field2] and for the Value argument enter [Table2].[Field2] for this case.

We dont want to copy or attempt to write the ID value over since Access and SQL Server will take care of that. The ID field is just used as record identifier.

Your completed simple data macro should look like this:

Save your macro changes and lets just call it Macro1 for simplicity. Close the macro too.

Now in order to run this named data macro, you need to attach it to some user event such as clicking a button on a view. Open up any list view currently in your web app in Edit mode. Click the Button control in the ribbon to add a new button control onto your view design surface.

To make this button run your named data macro, select it with your mouse and youll see Access display three small property buttons next to it. Click the Actions property button (it has lightning bolt icon on it) and youll see Access display an On Click action available for the button. Click the area where it says On Click now to open the user interface macro design window where we can create some macro logic attached to the buttons On Click event.

When the macro window opens, select the action called RunDataMacro from the Add New Action box. For the Macro Name argument, use the provided drop-down list and select Macro1 which you created previously. Your completed user interface macro to run the data macro should look like this:

Now save and close the macro window and then save and close your view changes.

Now navigate to that view in your web browser and click the button. You wont see anything happen or receive any complete/finished type of indication but if you were to check the records in Table1 youll see that all of the data and records from Table2 have been copied over to Table1.

This is a simplified example of course, but this is the general idea of creating a data macro to copy some data from one place to another. Data macros are very powerful once you learn how to use them.
