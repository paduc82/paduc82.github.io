ken sheridan say: To highlight the currently selected row you can use conditional formatting.  To facilitate this you first need to set the value of a hidden unbound text box control in the detail section to the value of the primary key.  You'll find an example in FindRecord.zip in my public databases folder at:

https://onedrive.live.com/?cid=44CC60D7FEA42912&id=44CC60D7FEA42912!169

Note that if you are using an earlier version of Access you might find that the colour of some form objects such as buttons shows incorrectly and you will need to  amend the form design accordingly.  

If you have difficulty opening the link, copy the link (NB, not the link location) and paste it into your browser's address bar.

In this little demo file the option to open simulated 'split form' opens a form in which the currently selected row in a subform is highlighted.  The fact that it's a subform is immaterial.  The code in the subform's Current event procedure is:

    ' set value of hidden control to current AddressID value
    ' to facilitate conditional formatting
    Me.txtAddressIDHidden = Me.AddressID

Each visible control in detail section is conditionally formatted on the basis of the following expression:

    [AddressID]=[txtAddressIDHidden]

As to how it might be done when the mouse pointer hovers over the row I can't help you.

Created a batch file to launch outlook 2013 and attach the scanned file to a new email and then converted the batch file to an executable.

Batch file as follows (update path to outlook.exe if different)

@echo off
"C:\Program Files\Microsoft Office 15\root\office15\outlook" /a %1

I then used bat2exe converter (on Sourceforge) to convert the batch file to an executable.

Add the executable bat2exe generates as the command line to a new entry you add for scan to email in control center's configuration.
