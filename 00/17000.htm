ossiemac say:

The problem with "Private Sub Movie_DropButtonClick()" is that it runs on a change of the DropDown list; not as a result of clicking the DropDown button.

Therefore when the list expands as a result of clicking the DropDown arrow then the code runs and then it runs again when the list collapses as a result of selecting a value from the list. You can establish that it runs twice with the following simple code example.

Private Sub Movie_DropButtonClick()
    MsgBox "Sub Movie_DropButtonClick called"
End Sub

You can clear the value in the DropDown with the following code but it will only run when the Combo gets the focus. By that I mean that if the cursor is currently in a cell on the worksheet and you then click on the combo. If you select a value in the combo and then decide it is wrong and select another while the combo still has focus then the GotFocus code will not run.

Private Sub Movie_GotFocus()
    Me.Movie.ListIndex = -1     'Clears the current value in the Combo
End Sub

As another suggestion you might like to change the method of allocating your list to the ComboBox and it saves a lot of code running in a loop to populate the combobox list.

Assuming that you have a column header on the column with the list then you can create a dynamic named range and use it in the ComboBox ListFillRange property.

  • Select the column header
  • Select Formulas ribbon
  • Select "Define Name"
  • Enter a name of choice for the defined name (It might default to the column header name with spaces replaced with underscores so can either use default or name of choice.)
  • Copy the following formula and paste into the "Refers to:" field. (Delete the default info from the field first)


=OFFSET(Sheet1!$A$1,1,0,COUNTA(Sheet1!$A:$A)-1,1)


The formula assumes the list is in Column A so you will need to edit the formula if this is not correct. If you need to edit the "Refers to:" field then click in it and press F2 to enter "Edit mode" otherwise when you use the arrow keys Excel thinks you are selecting ranges on the worksheet and adds them to the formula.


Click OK to Finish defining name.

  • Next Select Developer ribbon and turn on Design mode.
  • Right click the combo box and select Properties
  • In the field for ListFillRange enter the defined name; DON'T use quotes. (If you mis-spell the defined name then when you move off the field, it will disappear.)
  • Close the properties dialog (cross top right)
  • While still in design mode right click the combobox and select "View Code"
  • It will default to ComboBox_Change (or maybe cursor in other sub if you already have code for the combo box)
  • At top of VBA editor select ComboBox name in left dropdown (probably should have defaulted to that)
  • In the right dropdown select "GotFocus" and it will generate the sub and end sub for GotFocus.
  • Enter the following code.

Private Sub Movie_GotFocus()
    Me.Movie.ListIndex = -1     'Clears the current value in the Combo
    Me.Movie.ListFillRange = "Movie"        'Edit "Movie" to defined name
End Sub

The above forces an update to the list if any changes made to the list on the worksheet since last use of the Combo.

Feel free to get back to me if you still haveproblems.
