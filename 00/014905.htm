robert aldwinckle on forums say:
I still have the same problem with Hotmail. I go to msn.ca, click on Hotmail, and nothing happens.


Again, DNS and redirects will be involved and such destinations could be the problem cause rather than the originating host name.

However, in the case of Hotmail you may be able to get more informed help with better access to diagnostics by opening an incident on the Hotmail help site.  That apparently has now become part of Answers, e.g. handled in its Hotmail forum.

FWIW I once was involved with an incident which could be similar to yours which lasted for several months even though it was clearly being caused by a DNS error.   A workaround was available in that case, since the same redirected host name kept appearing and it was found that another host's IP address could be used instead of the problem lookup; e.g., it was possible to devise a temporary overrride to that lookup in a user's HOSTS file.

A way to proceed on this tack yourself would be to use IE9's Developer Tools' Network Capture tool to trace what is going on just before your error message.   Tip: since redirects are expected you should uncheck any option which would "Clear entries on navigate".   Unfortunately, the Network Capture tool does not itself expose IP addresses, just host names but you can then use those with both nslookup and ping to derive the necessary details to construct an override.

Another workaround was also available then which you might find easier to use and equally effective.   Simply connect to your Hotmail account using  Windows Live Mail (a component of Windows Live Essentials) instead of trying to use your browser.   There were still redirects in that case too but they apparently took a different path which avoided the problem lookup.

 

Good luck
