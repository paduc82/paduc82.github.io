Feedback on OneDrive for Business: Syncing malfunction and - Microsoft Community

thiago lopes carneiro say:

Feedback on OneDrive for Business: Syncing malfunction and freezing


Hi!

I would like to report some issues I have been dealing with OneDrive for Business and make some suggestions.

A) When I move folders, OneDrive goes crazy. 

How I discovered the problem:

I am using OneDrive for Business along with SharePoint to share files with my team. I was in charge of organizing files and folders after my team put it all together for the first time. I waited SharePoint to Sync files with my computer, and then I used the my local SharePoint folder to organize the files. I expected that, once my work was done, Sharepoint would sync and our team folder would be OK. 

However, OneDrive app (the little blue clouds on taskbar) worked in loop, demanding all the CPU capacity. It was so serious that I had to call our IT support - they erased my user profile from windows (yes, you got it right, the hole user profile was compromised) and I had to start all over with a brand new profile. 

After a few days, I had the same problem again after I moved a few folders on the SharePoint team folder. My Windows user profile had to be reset. 


Suggestion:
Improve how OneDrive monitors the local folders changes. I believe it tries to delete files from the first folder and copy them to the destination folder. The bug happens somewhere inbetween. 

The OneDrive app could instead simply change the file path, each time the user drags and drops files and folders. 





B) The OneDrive app (the little blue clouds on taskbar) does not offer good feedback to the user nor controls to avoid sync problems.

There is no clear progress bar to help the user to know if the syncing is stuck or actually progressing. 

The green bar bellow the clouds is not enough. For several times, they ran endlessly. I suspected something went wrong. But what happened? I checked the Load Center (the white arrow in an orange circle), but it gave me no clear info. 

Eventualy, I found a small Word file (~120kb) which was not syncing. But what could I do? I tried a lot of ramdom things, and the only thing that worked was to move the file out of the OneDrive folder, change its name and put it back. It was not the case of a long name, I kept the same number of characters. And I did not have a clear control to solve the problem - like a button for "try to load again". 

I suggest you study the app GoodSync. It is very well-done. Commands are clear and advanced user have total control over uncommon situations. For example, the application offers controls to decide about syncing conflits, to set how often each task will be run, to protect folders against accidental deletions and other essential controls. 






C) Long file names

Yes, I have experienced some long file names issues. 

Once I adopted OneDrive as an option, I had to move some files from older folders to OneDrive folders, which added some characteres to the file path.

Suggestions: 

Create a character counter for the file paths. It could be shown on Windows Explorer. Actually, it would be useful to all the folders and files, not only for OneDrive. But please, this is urgent. Long names are an anoying problem. It is hard to solve, as most of the time Windows Explorer "freezes" when we try to rename a file in a long path. 

Whenever a long file path is detected, Windows Explorer should prompt the user to rename the file, offering a user-friendly window to do it. The "long path rename window" should count the exceeding characters (like Twitter does). 

Also, the rename window should exibit all the name at a glance. Instead of one line, the file name should be shown on at least 4 lines. 




D) Editing files from a local SharePoint/OneDrive folder can be ****.

When I open a file on MS Office, right from a local SharePoint/OneDrive folder, the OneDrive app (yes, the little blue clouds on taskbar) keeps trying to sync the file all the time. This is especially anoying when we save the file. Sometimes it takes unbearable 10 seconds to save a simple file. Office slows down a lot if I try to keep editing before the syncing is done.

To avoid this, I move the file out of the SharePoint/OneDrive folder, edit it, save it and put it back to the SharePoint/OneDrive folder.

Suggestion:
Come on, Microsoft, you can make this better! You should either choose to set OneDrive to sync files AFTER the saving is done, or you choose to improve the on-the-fly syncing until it is imperceptible. 

This issue has a strong negative impact on the user experience. It is impractical to edit files from SharePoint/OneDrive folders. Please do not take long to fix it. 




Best regards,
Thiago
