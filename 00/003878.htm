hansv mvp say:

Try the following:

  • On the Insert tab of the ribbon, click Table > Insert Table...
  • Specify 2 columns and 15 (fifteen) rows, then click OK.
  • Select the entire table.
  • Activate the Layout tab of the ribbon (under Table Tools).
  • Click Properties.
  • Activate the Row group.
  • Tick the Specify Height check box.
  • Enter a height in the box, e.g. 0.3".
  • Select Exactly in the "Row height is" dropdown.
  • Click OK.
  • Select the first group of 5 cells in the left hand column, then click Merge Cells.
  • Repeat for the second group of 5 cells and the third group of 5 cells in the left hand column.
  • Select the first group of 3 cells in the right hand column, then click Merge Cells.
  • Repeat for the second, third etc. group of three cells in the right hand column.

don cuthbert say:

Of course, your Windows 8 OEM product key would only activate a clean install of Windows 8.1 (not Pro) and upgrade to Windows 10 Home.  Are you sure that your 8.1 Pro is not a purchased upgrade of your Windows 8 license?

Standalone software does have Transfer rights, if that is what you bought.  That would have left your OEM Windows 8 license unused since whenever you purchased Pro, as opposed to that license having been upgraded to Pro.

The name or e-mail address displayed by winver is populated by a single registry key that can be changed.  If this was an upgrade from Windows 7, it would instead display the local user account name that was used to install Windows 7 and usually Microsoft (or the OEM manufacturer) as the RegisteredOrganization.

HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\Windows NT\CurrentVersion\RegisteredOwner

Also if Windows 10 is clean installed using a local account and then switched to an MSA login, the name displayed by winver will still only reflect the local account name that was used.  So winver displays the account name first used to install Windows (whether an MSA or a local account name) and can be easily edited to whatever you choose.  It is not a valid indicator of license ownership.

Also an MSA is not required to activate a clean install of Windows 10, as the license is registered to that device, even if it was an upgrade from standalone software.  The user owns the license of course, but it is the device that has been registered as a Windows 10 licensed device.  This allows future clean installs using no product key, since there is no unique product key for the free upgrades.

If it was an upgrade from standalone software, we have Transfer rights for the Windows 10 license.  But since there is no unique product key for these free upgrades, it poses a technical problem when trying to transfer the stand-alone Windows 10 license.  As far as I know, the only technical method to transfer a stand-alone Windows 10 free upgrade license is to transfer the qualifying stand-alone Windows 7 or 8 OS (such as with slui 4 automated phone activation on the new device) and then free-upgrade to Windows 10.  Windows 10, 8 or 7 would no longer be licensed for use on the original device and would have to be removed.
