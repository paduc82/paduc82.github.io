gaylemn say:

Blue text on cross-references changes to black when I update document

When I insert cross-references in my documents (eg, to a table or figure), I change the text to blue so the reader can tell it's a link.  However, when I update the entire document (ctrl+F9), the text change back to black.  I'm using Word 2016.  Why is this happening, and how can I stop it?

stefan blom say:

By default, cross-references have the same formatting as the surrounding text. What you can do is show field codes by pressing Alt+F9 and add the \* CHARFORMAT switch. If you see \* MERGEFORMAT, delete it and replace it with \* CHARFORMAT. Once you have done that, font formatting that you apply to the field will be preserved.

Use F9 to update the field display (or go to Print Preview by clicking File | Print and then go back to Print Layout view by clicking ESC). Use Alt+F9 to hide field codes again.

Specifically, the \* CHARFORMAT switch displays the formatting applied to the first character in the field code, that is, to "R" as in "REF."

macropod say:
If you insert hyperlinks instead of cross-references, they'll be blue by default and won't revert to black when you update the document. The potential downside is that, if you insert them with the destination text displayed, then change the destination text, the displayed text for the hyperlink doesn't update, so you'd have to update those separately. The upside, of course, is that you can make the hyperlink display text different from whatever you're hyperlinking to.

Got on the phone with Tim at MS and he saw the issue was in the config.xml for the language packs.  When you configure the .xml, you need to also include the existing core install language.

For example.

Core English .xml:

<Configuration>
<Add OfficeClientEdition="32">
<Product ID="O365ProPlusRetail">
<Language ID="en-us"/>
</Product>
</Add>
<Updates Enabled="TRUE"/>
<Display Level="None" AcceptEULA="TRUE"/>
<Property Name="PinIconsToTaskbar" Value="TRUE"/>
</Configuration>

Original Language Pack .xml:


<Configuration>
<Add OfficeClientEdition="32">
<Product ID="O365ProPlusRetail">
<Language ID="es-es"/>
</Product>
</Add>
<Updates Enabled="TRUE"/>
<Display Level="None" AcceptEULA="TRUE"/>
<Property Name="PinIconsToTaskbar" Value="TRUE"/>
</Configuration>

Corrected Language Pack .xml:

<Configuration>
<Add OfficeClientEdition="32">
<Product ID="O365ProPlusRetail">
<Language ID="en-us"/>
<Language ID="es-es"/>
</Product>
</Add>
<Updates Enabled="TRUE"/>
<Display Level="None" AcceptEULA="TRUE"/>
<Property Name="PinIconsToTaskbar" Value="TRUE"/>
</Configuration>

Once I made the edits above and deployed, English labels were intact.
