Is it normal for Onedrive to start up so slowly since there is no changes to the onedrive folder?  why would it take so long even though there is nothing to sync? Thanks.
cmdrkeene say:

OneDrive doesn't know if there had been no changes if it was shut down, so upon installation it goes through every file and compares it to the one on the server to see which one has been changed.  Then it has to compare the meta data to find out which one "wins the version battle" to become the synced version.  So yes, it's normal for disk and network traffic to be highly utilized as it starts and initializes again.

PS. I would recommend using "sleep" instead of full shutdowns.

cmdrkeene say:

Well, I have some bad news (or maybe not depending on it).

Office Picture Manager was discontinued back in 2013.  Since it's a discontinued "dead" product, it won't ever get any updates to make it work with new picture formats.

When you take a picture on a phone, the phone records what direction is "down" (based on the sensor in the phone, the same thing that makes the screen rotate when you turn the phone around).  But the Office Picture Manager is too old to know about that kind of technology, so it can't read it and show the picture the right way up.

Really the only thing you can do is use a more modern program that is built to understand modern picture formats.
