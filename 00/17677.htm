jay freedman say:
Because floating shapes are not in the same "layer" as the body of the document, Replace All doesn't even see the text in the shapes. You need a macro like this one:

Sub ReplaceInShapes()
    Dim sh As Shape
    Dim rg As Range
    Dim strFind As String, strRepl As String
   
    strFind = InputBox("Text to find:")
    strRepl = InputBox("Replacement text:")
   
    For Each sh In ActiveDocument.Shapes
        Set rg = sh.TextFrame.TextRange
        rg.Find.Execute findtext:=strFind, replacewith:=strRepl, Replace:=wdReplaceAll
    Next
End Sub

Even this may not work for various reasons. For example, I can't be sure that what you're calling "shapes" are really what Word VBA considers to be Shape objects -- there are other kinds of objects you can insert in a document that look like shapes to you but are really other things (e.g., SmartArt). Also, if the shapes are in (or anchored in) a header or footer, this macro would need some alteration to find them.

If you need to know how to use the macro, see http://www.gmayor.com/installing_macro.htm.

Occasionally, I will enter a formula into a cell and Excel will not accept it but rather will enter the computed value in the cell. Do I have some new option set or is there an unregistered error that I am encountering?

jllatham say:

That kind of thing can, indeed, be difficult to track down.  A couple of things that may help you along:

When you are exercising the code within the UserForms, try setting calculations to manual and turning off responses to other events (like Worksheet_Change()) while writing to any worksheets in the workbook:

Application.Calculation = xlCalculateManual

Application.EnableEvents = False

You will need to turn them back on in the code before exiting the routine(s) in the UserForm:

Application.Calculation = xlCalculateAutomatic

Application.EnableEvents = True

  Also, if you should encounter a run-time error after turning them off then you'll have to reset them in the Immediate window.

Sometimes there are errors that take place in the code for a UserForm that only get reported in a rather round-about way, like the announcement of Error 1004: Automation Error - and to find those you pretty much have to put a Stop statement somewhere early in the UserForm's processes and single step through it all until suddenly, without warning or message, it just kind of jumps out of the code completely - that's where such a problem lies.  It usually takes me 2 or 3 tries to find something like that because you have to remember what line of code was to be executed when it just quietly blows up in your face.

Good luck with it.

rory archibald say:
If you enter a formula that evaluates to 0 in a cell that is linked to a checkbox on a form as its ControlSource, you will see the formula replaced by FALSE. (similarly if the formula evaluates to a non zero number, you would see TRUE. Simple answer is don't link controls on forms directly to cells. :)
