albert d. kallal say:

As a general rule, I not aware this has anything to do with you using a 365 version. They are the same as the desktop version.

However, the “slow” switch to design mode usually occurs when you are have a split database. (You leave out this detail – it rather significant).

In most cases you can “eliminate” the slow switch by simply opening any linked table (even ones that have zero to do with the form or report you are attempting to switch into design mode). So open any “linked table” and minimize that table. Now try the opening any form in design mode – the delay you note should go away.

If you not dealing with any linked tables and your application is not split, then next up is to check your default printer. If your default printer is a network printer or non-existing printer, then again you often see “long” delays. (So I setup a PDF printer as my default local printer – and it comes in handy for testing reports etc. and not wasting paper).

So I would check + test the above two ideas.

When you save as picture, you get PPT's idea of how the picture should be saved.

Instead, I'd go for the original AND in the process, save all the tedium of directing the pictures to where you want them.

PPT files are now actually ZIP archives.  If you open one with an unzipping tool, you can browse to the media folder inside the archive and there you'll find all of the *original* images that have been inserted into the presentation.  Pull them out, drop them where you want and edit away.
