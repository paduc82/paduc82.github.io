Enable GPS Location Service - Microsoft Community

pete ame say:

Enable GPS Location Service

I have reviewed all posts regarding using GPS (GNSS) functionality with the Windows Location Service in 8.1 and everyone has been frustrated by the default behavior of using the WiFi or IP address for location instead of integral GPS(es).  My Dell Venue 11 Pro has a built-in GPS along with the Sierra Wireless GPS in the WWAN card.  It was frustrating for me to see the location hundreds of miles from me when using LTE.  After reading through the posts, one thing struck me: "Windows Location Provider uses WiFi or IP but not GPS".  So I tried what turned out to be a solution: disable the Windows Location Provider.

I verified in the Device Manager (Control Panel) that my GPS "sensors" were recognized under Sensors.  I also saw the Windows Location Provider "device".  I opened the Windows Location Provider device and Disabled it.  I then tested all of the apps I normally use with location sensing and they -all- worked using GPS (Maps actually showed the area of accuracy get smaller as the GPS got more satellites locked in).

I did notice one time when looking at the Device Manager that Windows Location Provider wasn't listed.  I simply opened Maps, went back to the Device Manager and there it was so I could disable it.  It now shows continuously in Device Manager as disabled.

"Simple" solution to the ever-present assumption by Microsoft that people don't really want to get the most out of their software and hardware.  I don't know which GPS is being used but I know for a fact that at least one is.

Summary:

Make sure Windows Location -Service- is enabled and app sharing is enabled under Privacy.

Make sure your GPS is recognized under Sensors (may be just listed as "Sensor Collection") in Device Manager.

Disable the Windows Location Provider under Sensors in the Device Manager. If the Windows Location Provider isn't displayed, start an app that uses location services and check again.

Have fun!

Pete
