Are you using VSTO?

I was not sure whether you are saying

If it is (a), then the problem is that there is no equivalent of the Excel XML mapping process in Word. There are several things you could do - probably the simplest would be along the following lines:

 b. Use VBA (or whatever) to get the data you need from the external XML and copy it into the Custom XML Part

brian tillman [outlook mvp 2007-2019] say:

Archiving works on the Modified date of the items in the data store, not the Received date.  Add the Modified field to the header row and you'll probably see that the dates are all more recent than the archive criteria you've set.  That's why they don't archive.

You can change this behavior in Outlook 2007 or 2010 with a registry setting.  See http://www.howto-outlook.com/faq/archivenotworking.htm#Modified_Date.  See also http://support.microsoft.com/kb/2553550/

danny seager say:

Have you tried Manually spliting the database?

Take 2 copies of your database, copy 1 of them onto the server. Open that database up and delete all the forms, queries, reports etc - just leave the tables.

Then open the other copy of the database and delete the tables, then go to EXTERNAL DATA > ACCESS (under the import section). This is pretty much all the splitting wizard does for you - I personally never use it.

I am running Windows 8 Pro Media Centre on an older Dell XPS 420.

doug robbins - mvp office apps & services (word) say:

The error message "Word could not create the work file. Check the temp environment variable". Can be caused by incorrect registry values in the key:

HKEY_CURRENT_USER\Software\Microsoft\Windows\Current Version\Explorer\User Shell Folders.

Click Start and in the start search field, type

Regedit

and press enter.

In the registry editor, browse to:

HKEY_CURRENT_USER\Software\Microsoft\Windows\Current Version\Explorer\User Shell Folders

On the right hand pane, check the value data for Cache.

It has to be %USERPROFILE%\AppData\Local\Microsoft\Windows\Temporary Internet Files.

If it is different, double click on Cache and type:

%USERPROFILE%\AppData\Local\Microsoft\Windows\Temporary Internet Files

under value data and click ok. Close the registry and  restart the computer and check if it works.
