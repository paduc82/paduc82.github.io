Hi,

Excel has no native way of doing this so we must create a UDF to do it. ALT+F11 to open vb editor, right click 'ThisWorkbook' and insert module and paste the code below in. Close VB editor.

Back on the worksheet call with

=Countcolour($A$1:$J$1,K1)

Where:-

A1:J1 is the range you want to count
K1 is a cell with the fill colour you want to count.

Function Countcolour(rng As Range, colour As Range) As Long
Dim c as Range
Application.Volatile
For Each c In rng
    If c.Interior.ColorIndex = colour.Interior.ColorIndex Then
        Countcolour = Countcolour + 1
    End If
Next
End Function

joeu2004 say: Ronald wrote:
I have made a payroll calculator whereas it looks like the following:

A1: Hours               A2: Rate                A3: Gross       A4: Deductions    A5: Tax    A6: Net
B1: INPUT DATA   B2: INPUT DATA   B3: = B1xB2  B4: =B3x0.0183    B5: ???  B6:  =B3-B4-B5

Let's say for example that I input B1 as 10 and B2 as 10 then B6 would read 98.17 following the formulas above.  I am trying to solve for B5 essentially.  I would like B5 to equal any number that would round B6 to the nearest multiple of 5 by decreasing only.  [....] In the example above, it should be B5 = 3.17 and B6 = 95.00.  The formula I am requesting is for B5 since I need it to round B6 down to the nearest 5th multiple.  If B6 reads 101.53 (before subtracting the tax in B5), then B5 needs to equal 1.53 and B6 to equal 100.00.  If B6 reads 104.99 (before subtracting the tax in B5, then B5 needs to equal 4.99 and B6 to equal 100.00.

The formula in B5 should be:

=B3-B4-FLOOR(B3-B4,5)

But it is prudent to always round dollar-and-cents calculations, especially in row 4.  For example, in last example, B3-B4 is 104.99, B5 is 4.99 and B6 is 100.00 only if B3 is 106.95.  But in that case, without rounding, B4 would be about 1.9572.  By coincidence, with the formula above, B5 would appear to be 4.99 when formatted to 2 decimal places.  But actually, it would be about 4.9928.  These calculation errors (yours) might throw off other dependent formulas, e.g. the sum row 5.

So the formulas should be:
B3:  =ROUND(B1*B2,2)
B4:  =ROUND(B3*1.83%,2)
B5:  =ROUND(B3-B4-FLOOR(B3-B4,5),2)
B6:  =ROUND(B3-B4-B5,2)

Note:  Some of the uses of ROUND might seem superfluous.  They are used to correct infinitesimal arithmetic "errors" that creep into Excel calculations for technical reasons.

Private Sub Calendar1_Click()
    Me.Calendar1.Visible = False

I think it would be easier to create two Recurring Tasks:  one for the weekly meeting and one for the Monthly meeting.  That would be simpler provided that the weekly meeting is held on the same day each week, and the monthly meeting is held on the same day of the month.  You can insert a Recurring Task in your project by completing the following steps:

Robert_ITman, I have found that the root certificate issuers generally have a place to download the individual root certs so you can manually install them. Once you've downloaded one, you can install it just like you'd install a new private cert:

1. E-mail to yourself

2. Open e-mail on phone

3. Open the attached cert, which triggers the phone to ask if you want to install it

4. Go ahead and install it

That should work without too much problem. The worst thing with this is that you have to update each individual root cert one-at-a-time. That's fine if it's just for one or two web sites. But it could become a real pain if you need 20+ root cert updates.

Windows Phone or IE on Windows Phone should support an automated root cert update that updates all root certs, say, once per year.
