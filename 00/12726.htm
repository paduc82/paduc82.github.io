Typically the problems that occur due to 3rd-party security applications are related to either complexity of use or installation and removal.  The tendency for consumers to hop from one product to the next for various, usually unimportant reasons only makes these issues worse.

Everyone I've installed either MSE for or explained Defender to is still using them to this day, which should tell you something as well.  There may be some slightly better paid products out there, but for most consumers the important issues are low cost/free and easy to use and understand.  This is exactly why the original OneCare, later MSE and Defender clients were created by Microsoft and that's still true.

I have been involved with these Microsoft security products since they were in their first beta versions in 2005-6.  I've also had the opportunity to sit across the table from the developers and product managers responsible for the original OneCare and later MSE clients.  This provides me with a greater understanding of the reasoning behind the design and operation of these security applications than most others here.

I also have years of experience supporting PC and other early computing systems in small to medium businesses, government, and education including engineering.  The one consistent thing I've seen is that simple works better than highly complex, regardless of whether the computer users involved are highly trained in other fields.  This is why I've kept to the KISS principle (Keep it Simple Stupid) in my own support of these systems.

The Microsoft security applications have generally kept to this principal since their early versions, which over time have been integrated more deeply into the Windows operating system itself.  Though Windows and these associated security clients are highly complex underneath, they are still quite simple to use and understand, which is why I'll continue to use and support them.

What should I explain more? :-)

Take my sample file and write values > 0 in column A in sheet as you like. Go into sheet main and you'll see the cell is colored. (Means we have no zero inside the data).

Go into sheet Data and change any value you like to zero (or add a zero anywhere below the data in column A). Go into sheet main and you'll see the cell is not colored.

=NOT(ISERROR(MATCH(0,Data!A:A,0)))
