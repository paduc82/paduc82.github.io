First, it does not matter that Android doesn't support Office 365. if you are creating a WEB app it runs in the browser, so it will run inside an Android based browser (i.e. Chrome).

Second, your Office 365 subscription can create user IDs for a number of users to log in with. The # depends on the version of 365 you subscribe to. 

Third, the data will be hosted on a SQL Azure database. You can link to that database from your desktop app, but you will not be able to make structural modifications.

Fourth, you can use a local SQL Server installation instead of Azure, but it will still need to be accessible to the cloud. For a small business, a hosted SQL Server database might be a better choice.

Fifth, yes Sharepoint is necessary, but its included in your 365 subscription.

Finally, an Office 365 subscription, probably a small Business type.
