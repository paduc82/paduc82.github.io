suzanne s. barnhill say:

With roman numerals, whose width varies so greatly, right alignment (as you show in your first example) is generally preferred. This permits you to keep the space between the number (and following period) and the text constant. If you prefer to left-align the numbers, then you will have to set the text indent far enough to the right to accommodate the longest numeral in your list (for short lists, VIII or XIII). Whichever plan you choose, all the indent settings ("Aligned at" and "Text indent at") must be set in the Define New Multilevel List dialog; these settings override settings you make in the Paragraph dialog when defining a style.

For further instructions on setting up multilevel lists, see http://shaunakelly.com/word/numbering/numbering20072010.html

marshall barton say:

You can't do that with Field Vaue option.  You would use the Expression Is option with this kind of expression:
   [thetextbox] Is Null

OTOH. If do not want the text box and its label to show.  I think it's a lot easier to forget Conditional Formatting, leave the label alone and just make the text box (and its label) invisible using a line of VBA code in the section's Format event procedure:
   Me.thetextbox.Visible = Not IsNull(Me.thetextbox)

With the text box (and its label) invisible, CanShrink could be used if desired.

allen browne say:

In report design, right-click the label and Change To | Text Box.

Now set the Control Source property of the new text box like this:
    =IIf([City] Is Null, Null, "City:")
replacing City with the name of your original text box.

File Compression

After you do all other file cleanup, and before Defragging, consider turning on XP System File Compression.  File compression will add a little delay when you read and write file, but is a worthwhile trade when you are short of space. You can identify a lot of data files you don't use frequently.  The same goes with Applications. While you don't want to compress the whole "Programs" folder, you could compress application subfolders for applications and application sub elements you don't use frequently. Things like "graphics" or "artwork" or "samples", or even "Help" are very rarely used, so taking a little longer to open them is not going to be a significant impact on your performance.

 

Defragment Utility

After you finish all of the file deletion, run a file defragmentation utility. This will consolidate files and free space.  it may coincidentally free up a little space. The main thing is by defragging your HD it will read and write files faster.
