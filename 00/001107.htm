The subform of this form is [Part Deviation Log], which is built from the [PartDeviations] table and also has the [PartNumber] control.

The parameter query is called [IssuesByPartNumber]. It prompts you for the part number, then pulls up all the records from the [Part Deviation Log] for that part. There is a report based off of this query as well (called [Detail Report By Part Number]), where you enter the part number in the prompt and only get those records on the report.

My question is, can I put a button on either the form or subform so that it will print that report automatically, based on the current record without having to type in the part number? Right now, if I put a button on the form to print, it still gives you the box prompting you for the part number before printing the report. I want it to recognize that I only want the report for the current PartNumber that's open.

If I take the [Select PartNumber] out of the query, it pulls up a report with ALL the part number records. I am having trouble with the Where condition. Am I going about this the wrong way?
Thank you guys!
hansv mvp say:

Option 1:

 

Change the parameter [Select PartNumber] in the query to [Forms]![Part Deviation - Main Form]![PartNumber]

The query will now return records with the same PartNumber as that in the current record on the main form.

 

Option 2:

 

Remove the parameter from the query.

Place a button cmdReport on the main form with the following On Click code:

 

Private Sub cmdReport_Click()
    DoCmd.OpenReport ReportName:="Detail Report By Part Number", View:=acViewPreview, _
        WhereCondition:="[PartNumber]=" & Me.[PartNumber]
End Sub

 

This assumes that PartNumber is a number field. If it is a text field, use

 

       WhereCondition:="[PartNumber]=" & Chr(34) & Me.[PartNumber] & Chr(34)

 

Chr(34) is the double quote character ".

 

Thanks to Ken Sheridan for pointing out an error in the last line of code (now corrected)
