tom van stiphout say:

Thus proving Marshall wrong.

Properties generally DO accept expressions, and BackColor is no exception. Per the help file:

The BackColor property contains a numeric expression that corresponds to the color used to fill a control's or section's interior.  (emphasis added TvS)

The reason the OP's code failed, among others, is because the expression
IIF(NewTicket.Value <>"", "Yellow", "White")
returns a string, which is not a "numeric expression" as required by the property.

The RGB function DOES return a numeric value and is thus a better choice. So would be any custom VBA function that returns a long int, or an IIf expression like:

IIF(NewTicket.Value <>"", #FFF200, #FFFFFF)
