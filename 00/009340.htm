Actually, never mind that question... I found a work-around where I do not need to know it in advance. Give this macro a try (change the StartRow constant assignment in the Const statement to the actual first row number for your first piece of data... the code assumes Column A contains your text)...

 

Sub SplitOutDataOriginallyFromPDF()
  Dim X As Long, LastRow As Long, ArrIn As Variant, ArrOut As Variant, DollarSplit() As String
  Const StartRow As Long = 2
  LastRow = Cells(Rows.Count, "A").End(xlUp).Row
  ArrIn = Cells(StartRow, "A").Resize(LastRow - StartRow + 1)
  ReDim ArrOut(1 To UBound(ArrIn), 1 To 9)
  For X = 1 To UBound(ArrIn)
    ArrOut(X, 1) = Left(ArrIn(X, 1), InStr(ArrIn(X, 1), ",") - 1)
    ArrOut(X, 2) = Split(Replace(ArrIn(X, 1), "-", ","), ",")(1)
    ArrOut(X, 2) = Trim(Left(ArrOut(X, 2), Len(ArrOut(X, 2)) - 3))
    DollarSplit = Split(ArrIn(X, 1), "$")
    ArrOut(X, 3) = Mid(Trim(DollarSplit(0)), InStrRev(Trim(DollarSplit(0)), " ") + 1)
    ArrOut(X, 4) = "$" & Split(DollarSplit(1))(0)
    ArrOut(X, 5) = Split(DollarSplit(1), " ", 2)(1)
    ArrOut(X, 6) = "$" & Split(DollarSplit(2))(0)
    ArrOut(X, 7) = Split(DollarSplit(2), " ", 2)(1)
    ArrOut(X, 8) = "$" & Split(DollarSplit(3))(0)
    ArrOut(X, 9) = Split(DollarSplit(3), " ", 2)(1)
  Next
  Cells(StartRow, "B").Resize(LastRow - StartRow + 1, 9) = ArrOut
End Sub

dianeoforegon say: There is an unofficial way to setup accounts and rules, but you probably won't save much times.

Under File > Export select the default action to export all items as .olm file.

  1. Create a new Identity.
  2. Quit Outlook
  3. Option -drag to copy over these folders from your current Identity to the new Identity. (if you don't copy and this fails, you could loose data)

Data Records
  Mail Accounts
  Recent Addresses
  Rules
  Schedules <--not sure about this one
  Signature Attachments
  Signatures

  • Open Database Utility and select to rebuild the new Identity.
  • You can safely delete the copy with the [Backed up ....]
  • Open this Identity and verify all the data is there that was moved.
  • Import the .olm file you exported from your other Identity.
The imported .olm file will show up under a folder in the folder list. You'll need to select the messages in the default folders (Inbox, Drafts, Sent, Deleted Items) and move to the local folders. You can drag any custom folder to it's correct location. You can delete the imported folder structure after you move all the messages.

You'll also need to move the contacts, events, tasks, notes to their local folders.

Rules will need to be reset once you get all your data imported.
