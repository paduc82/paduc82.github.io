dhookom say:

If you don't need to edit the data in the combined query and since the 10 different tables have the exact same fields, you can use a union query. You will need to edit the query in SQL view. Your query might look something like:

SELECT FieldA, FieldB, FieldC, FieldD

If you want to add a column that identifies the source table, you can use:

SELECT "Table1" as SourceTable, FieldA, FieldB, FieldC, FieldD

The message is "The changes you requested to the table were not successful because they would ceate duplicate values in the index, primary key, or relationship. hansv mvp say:

The Form_Error event procedure has a Response argument. By default, it has the value acDataErrDisplay, meaning that Access will display the built-in error message. To tell Access that you have displayed your own message (or handled the error otherwise), you can include a line

 

    Response = acDataErrContinue

 

in your event procedure. This suppresses the built-in error message.

 

Added: I think I misinterpreted your question. See Jack Stockton's reply below.

jack stockton say:

I then to use a simple coding style. If I want to display my error message for a specific err, in this sample 99, I add an IF branch.

 

Public Function FunctionName()
    On Error GoTo proc_err

 

    '...your code here

 


proc_exit:
    Exit Function
proc_err:
    If Err.Number = 99 Then
        MsgBox "Your error message", vbOKOnly + vbCritical, "Error"
        Resume proc_exit
        Resume
    Else
        MsgBox Err.Number & " " & Err.Description, vbOKOnly + vbCritical, "Error"
        Resume proc_exit
        Resume
    End If
End Function
