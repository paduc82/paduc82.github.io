tryx3 say:

You can have multiple shortcuts with different properties.

The shortcuts are independent so if you change one the others are not affected.  Shortcuts have no inheritance ability. 

Windows shortcuts can often find a target that has been moved & amend themsleves accordingly but that is normally only applicable to datafile shortcuts rather than the program shortcuts you refer to.  Program shortcuts would attempt to find a new target location in the same fashion but the results would be unpredictable.

gts-nj say: Compatibility mode settings are saved in the registry and are directly associated with the exe. 
(You raised an interesting problem.  I was also not aware that differing shortcuts couldn't be used for this until your question led me to dig into it.)

Compatibility mode settings are stored in -

HKEY_CURRENT_USER\Software\Microsoft\Windows NT\CurrentVersion\AppCompatFlags\Layers
    Look for a entry with the path to your executable.

If not shown there check HKEY_LOCAL_MACHINE\Software\Microsoft\Windows NT\CurrentVersion\AppCompatFlags\Layers

See the MSDN aritcle below, particularly, the section "Managing Settings Made by PCA" for more information.
Application Compatibility: Program Compatibility Assistant (PCA)
http://msdn.microsoft.com/en-us/library/bb756937.aspx

To switch between the two modes you could use a script to select the mode and change the registry key before launching the exe, or save two .reg files and update the registry entry from two batch files for the desired mode.

(That article doesn't specify the key values, but you could identify them easily by changing the mode and seeing what it stores in each case.)
