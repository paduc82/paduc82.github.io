E72 and BirdStep SmartConnect - Microsoft Community

grschinon say:

E72 and BirdStep SmartConnect

Technical Level : Intermediate

Summary

Details

BirdStep SmartConnect (see http://www.birdstep.com/Products/Birdstep/SmartConnect/) is an application that's designed to provide network-aware applications with a feature similar to Symbian's "destinations" when those network-aware applications only support connectivity via a specific network access point.

 

It creates a new access point that you can place in any of your destinations, but when invoked, it walks through a prioritised list of access points, establishing the connection that's the highest possible on your list.

 

One example of such an application is the Nokia Messaging application that ships with the E72. The E72 is a S60v3FP2 device that supports destinations, but it is only possible to specify a particular access point to use in order to synchronize your e-mail, for example.

 

BirdStep SmartConnect works on the E72 with firmware versions up to and including V023. V031 includes SmartConnect in the ROM image, but the version supplied doesn't actually work (d'oh!).

 

BirdStep have released a patch to install on V031, which gets the whole thing working again. See at the bottom of this page:

 

http://www.birdstep.com/Products/Birdstep/SmartConnect/SmartConnect-FAQ/

Co-authoring on a document, is one of the coolest new features in Microsoft Word 2010.  There was even a nice blog post explaining how it worked.  Documents can be co-authored on either SharePoint 2010 or SkyDrive, so long as both authors have edit permissions on the document.  Co-authoring is also supported on Word 2011 for Mac.

Recently, co-authoring was added to the Word Web App on SkyDrive.  While this does work cross platform and cross browser, it is subject to any limitations of the Word Web App.  Since the Word Web App cannot edit documents with tracked changes (or comments), you would need to resolve all tracked content (either accept or reject the change) and remove all comments in order to simultaneously edit the document your co-author in Arizona.  After comments and tracked content is removed, the Word Web App will happily open (and co-author) the document.

Hope that helps,
Rob Rolnick

rob rolnick say:

Hi Bandituk,

This issue occurs when instead of downloading the actual Word document,  Microsoft Word pulls down a .srf file from SkyDrive.  A .srf or "server response file" is not a supported Microsoft Word document.  In this case, it is actually a website response saying something along the lines of "Javascript Required to Sign-In."

When trying to open the file in Microsoft Word, were you prompted to provide your Windows Live ID credentials?

Regardless, several different threads have suggested responses in the past.  Since different threads have different posts marked as answers, I'll share numerous possibilities.  Ocassionally this comes up with Microsoft Excel too, so where necessary replaces Microsoft Excel with Microsoft Word in these suggestions.

-------

One Thread:  SkyDrive cannot open doc in Word2007 - javascript required to sign in??

Had the following suggestion

I had the same issue and obviously the given answers here were no help.  I was finally able to resolve this by getting the error in Excel...

1) Going to File->"Open from Office Live" -> "Sign Out"

2) Going to IE and Signing Out

3) Going back to Excel and repeating Step 1

4)Closing Word

5)Opening Word and going to File -> "Open from Office Live" -> "Sign In"

6)Leave it open, go back to IE and Sign In to SkyDrive

7) Open in Word from SkyDrive and the file opened in Word properly finally

Hope this helps, it seemed like it was a matter of the sign-in process 

-----

Another thread:  Skydrive and Office application integration problems

Suggested the following:

Internet Explorer settings/cookies/etc. is not the problem.  SkyDrive requires Live-ID (Passport) authentication to access documents, but when you signed in to the web app you only gave Internet Explorer your password.  Word also needs that information which is why it's showing the .NET Passport Wizard.  It is possible to disable this dialog as noted in this MSDN article.  Save the bolded text below as a ".reg" file and apply it to the machines:

Windows Registry Editor Version 5.00

[HKEY_CURRENT_USER\Software\Microsoft\Windows\CurrentVersion\Internet Settings\Passport]
"NumRegistrationRuns"=dword:00000006

That said, this only disables the wizard.  When you click "Open in Word", Word is still going to ask you for your Live-ID/Passport username and password - without it SkyDrive will not let Word access the document.  So you'll need to enter your credentials again.  Although the Sign-In Assistant is supposed to bridge this gap, I'm not sure it works with Office 2007.

Regardless, I don't think a public terminal with a generic Windows account is very reliable or secure.  As soon as somebody (say, Bob) clicks the "remember me" checkbox when signing in to SkyDrive, that account is stuck there and it's tricky to get off.  If somebody else tries to use that machine (say, Alice) she'll have access to Bob's files and difficulty getting to her own.  A domain or workgroup with individual Windows accounts will be more reliable and secure: once Bob logs off the machine that will lock Alice out of his SkyDrive.

-----

Some other threads such as:  JavaScript required to sign in  and JavaScript required to sign in  (yes they had the same name) both seem to be confusing Java and Javascript.

-----

I've also seen suggestions that clearing your browser cache, stored passwords, and cookies can work.  Most browsers allow you to get to this functionality by hitting ctrl+shift+delete.  Keep in mind that clearing your cookies will sign you out of any website that maintains your credentials between visits.

Finally, I've heard that going: "start --> run --> control keymgr.dll", and clearing any credentials that reference passport, live, or hotmail... and then rebooting your computer, can work.

-----

As you can imagine, with so many different suggestions (and so many different people saying that one or the other worked for them), it is hard to give a definitive answer to this problem.

That said, the fact that the Word Web App almost certainly means that Javascript is running on your computer.
