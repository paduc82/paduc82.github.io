jeeped say:
The query that the form is based upon is malformed. Whenever Access cannot determine a field parameter in a query, it assumes that it must be a parameter so you are getting the prompts. The non-determination could be as simple as a misspelled field or it could be improperly aliased. It is hard to offer more without seeing the SQL behind the query, but if you open up the query to SQL view you might be able to go through it and see where it has gone wrong.

Get the query running first and then build the form on it. If you post the SQL of what you've got back here, someone might be able to spot the error.

ken sheridan say: The unrecognized references which are causing the parameter prompts are not necessarily within the form's RecordSource query.  They could be in properties of objects within the form, possible candidates being the RowSource properties of combo or list boxes.  So if the query itself opens successfully you will need to check the form's objects to find the references.  To judge by the names they refer to dates for the fitting and replacement of tyres, so that should give you a clue where to start looking.
