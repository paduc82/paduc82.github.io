todd_s_88 say:

Sorry but just to double check based on your description you've already done the typical profile rebuild steps?

-Go to HKLM\Software\Microsoft\Windows NT\CurrentVersion\ProfileList. Find which SID has the ProfileImagePath of C:\Users\<theirusername>, backup the reg key if desired and rename .old or delete it

-Go to C:\Users and rename their user profile to .old

-Have them try to log in again

Again it sounds like you did this already but since that is the first step just wanted to double check. The other thing I've had happen before that caused this is due to Windows getting an Access Denied when trying to copy the Default profile. Usually with that it will cause new profile issues across the board, but if you haven't tried this yet what you can do is:

-Go to C:\Users and right click the Default profile->Properties

-Go to the Security Tab and click on the Advanced button

-Click on "Change Permissions"

-Typically I'll have 4 different permission ACEs on the list in this screen: SYSTEM, Administrators, Users, Everyone. For each of those security principals select it and tick the "Replace all child object permissions with inheritable permissions from this object" and click apply.

-Try to log in again

The last thing I've seen with temp profiles is again an access denied issue, and was caused by the presence of some folders under C:\Users\Default\Application Data\<Random Folders>. In that instance the random folders were causing the access issue, so after deleting the random folders under application data (but NOT the application data folder itself) I was able to get the user in. This may have been specific to my old company but I have seen that once as the cause so at least wanted to throw that out there.

Also one helpful blog I've found on profile issues is https://helgeklein.com/blog/2011/05/helges-profile-toolkit/ , and specifically https://helgeklein.com/blog/2011/11/temporary-user-profiles/ https://helgeklein.com/blog/2013/09/error-message-explained-user-profile-service-failed-logon/
