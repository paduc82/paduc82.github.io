ken sheridan say:

It sounds like you are looking specifically for repairs which include two specified parts, possibly along with other parts.  If so perhaps we can illustrate this by using some data from Northwind, and equating your Repairs (the table about which which Scott commented 'Not sure what the value of that table is') with Orders, Your Parts with Products and your Repair Events with Order Details.  What you can do is join the three tables in a query, group by the Orders columns (your Repairs), restrict the result set to two products (parts in your case), and return only those distinct rows where the COUNT of all rows returned per group is greater than 1:

SELECT Orders.[Order ID], [Order Date]
FROM Products INNER JOIN (Orders INNER JOIN [Order Details]
ON Orders.[Order ID] = [Order Details].[Order ID])
ON Products.ID = [Order Details].[Product ID]
WHERE [Product Name] = "Northwind Traders Chai"
OR [Product Name] = "Northwind Traders Coffee"
GROUP BY Orders.[Order ID], [Order Date]
HAVING COUNT(*) > 1;

With my Access 2007 version of Northwind this returns two orders.

We can simulate your reference to controls in a form as parameters by including two simple parameters in the query in place of the literal product names:

SELECT Orders.[Order ID], [Order Date],
[Enter product 1:] & " and " & [Enter product 2:] AS [Products Ordered]
FROM Products INNER JOIN (Orders INNER JOIN [Order Details]
ON Orders.[Order ID] = [Order Details].[Order ID])
ON Products.ID = [Order Details].[Product ID]
WHERE [Product Name] = [Enter product 1:]
OR [Product Name] = [Enter product 2:]
GROUP BY Orders.[Order ID], [Order Date]
HAVING COUNT(*) > 1;

Note how this now enables the two product (part) names to be returned in the result table by concatenating the values entered as the two parameters (in the controls in the form in your case) into a single string expression as the computed Products Ordered column.  Note also that this computed column does not have to be included in the GROUP BY clause.

You can only do this where the number of parts sought is fixed, however.  To seek repairs where the number of parts sought varies you'd have to use one of the methods from my MultiSelect demo.
