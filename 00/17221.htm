Paste is a worksheet level command - it pastes the clipboard contents into the worksheet's current selection.

Instead, change

.Cells(erow, 1).Paste

to

.Cells(erow, 1).PasteSpecial xlPasteAll

But you still have issues with the activesheet since you are not fully specifying your range parent. You can fix that and avoid the case select code replication by changing your code to

    With LogWS

        LastRow = .Range("A" & .Rows.Count).End(xlUp).Row

        StartRow = .Cells(1, 9).Value


        For i = StartRow To LastRow

            .Range(.Cells(i, 1), .Cells(i, 6)).Copy

             With Worksheets(.Cells(i, 4).Value) 'This uses the correct worksheet

                 eRow = .Cells(.Rows.Count, 1).End(xlUp).Offset(1, 0).Row

                 .Cells(eRow, 1).PasteSpecial xlPasteAll

             End With

        Next i

        Application.CutCopyMode = False


    End With

Referencing a custom property in the Document Title? - Microsoft Community

dwpscl say:

Referencing a custom property in the Document Title?

Hello all,

I would like to add custom properties to a Word Document, then set the Document Title to be a combination of these properties.

For example, if I've added:
New Property: PropA
New Property: PropB

Then I want to set the title to be: {PropA} Document-Version {PropB}

where when the Title is updated it pulls the values from PropA and PropB.

So if PropA is set to "My" and PropB is set to 1, the Title becomes "My Document-Version 1".

Does anyone know how to set this up? 

Thanks,

DW

a. user say:

To set the title, you either have to do it

 a. manually or

 b. programmatically, e.g. in VBA or

 c. by updating a suitable TITLE field in the document.

IN recent versions of Word, Word typically uses a content control to insert the title (e.g. when you use Insert->Text->Quick Parts->Document Property->Title, but to do this you have to replace that by an ordinary TITLE field with some nested fields so you have this:

{ TITLE "{ DOCPROPERTY PropA } Document-Version { DOCPROPERTY PropB }" }

where all the {  } are the special field code brace pairs that you can enter using ctrl-F9 (you cannot just type them on the keyboard).

Unlike the modern content control version, this one does not autoupdate when you alter its contents - if you modify PropA or PropB you will need to select the field and re-execute it. But modifying the title elsewhere does not modify the resul tof this field - if someone does that, their modified title will be replaced when you next update the TITLE field.

1. run elevated command prompt (cmd with administrator rights): type

Dism.exe /online /Cleanup-Image /StartComponentCleanup

Dism.exe /online /Cleanup-Image /StartComponentCleanup /ResetBase

freed about 100-200Mb

freed many Gbs!

It should help - run at your own risk
