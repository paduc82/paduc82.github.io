When a radio button is selected the code in the after update event of the option group assigns the desired select statement to the row source of the combo box but the old selected value (bound column) is still displayed in the combo box.  I would like to add the following functionality:

     - When a new radio button is selected the new bound column (always column 0) is displayed in the combo box and the drop down           is now at the same record position as the previous radio button selection.

Example:  

     - The ID radio button is selected and the user has selected record (ID) 29 from the combo box.  29 is now showing in the combo box

     - The user now selects the EquipID radio button

     - I would like the EquipID for record 29 to now be displayed in the combo box and the drop down to be at record 29

ken sheridan say: From your description it's not working exactly in the way I envisaged.  I suspect this is because the option group's value is not set by default to 1 as I advised.  If you are happy with the way it works, then don't change this, but the way I'd envisaged it working is this:

1.  The form's Current event procedure is executed when the form opens and when it moves to another record.  This sets the value of the combo box to the current ID, EquipID or CableID in the form's record.  Which of these it assigns to the combo box is determined by the current value of the option group, which in turn determines which column is the first in the RowSource property and hence the BoundColumn of the combo box if this property is 1.  This keeps the combo box in sync with the form if the user navigates to another record by other means than the combo box.

2.  In the option group's AfterUpdate event procedure the combo box's RowSource, ColumnWidths and Value properties are changed depending on which option has been selected.  The SQL statement set as the RowSource property does not return a single row as you surmised.  The Value property is set to the current value in the form of whichever of the three columns is the first column in the RowSource property.  It is this which selects the row in the combo box.

3. The combo box's AfterUpdate event procedure causes the form to move to the record with the ID, EquipID or CableID selected in the combo box.  Whichever of these is the combo box's value is determined by whatever is currently selected in the option group.  The form is moved to the selected record by first finding the row with the selected value in the form's RecordsetClone, which, as its name suggests, is an exact copy of the form's current recordset.  The form's Bookmark is then synchronized with the RecordsetClone's Bookmark, which has the effect of moving the form to the relevant record.

I've just posted an add-in that will help with this task. After you install it, each time you save any document the location and size of that document's window is saved in document variables in the file. The next time that document opens, the add-in will retrieve the values of the variables and put the document's window back in the same place. Because Windows uses a continuous range of Left and Top values across multiple monitors, the window will appear on the same monitor as before. (I've tested this on my two-monitor system, but you'll have to tell me whether it works the same on three monitors.)

Go to https://jay-freedman.info and download the SaveLocation.zip file, and follow the instructions for installing the add-in.

To open a large batch of documents with one double-click, create a VBScript file (a plain-text file made in Notepad and renamed with a .vbs extension) containing code like this:

Set objWord = CreateObject("Word.Application")
objWord.Visible = True
objWord.Documents.Open "d:\Documents\doc1.DOCX"
objWord.Documents.Open "d:\Documents\doc2.DOCX"

with as many Open statements as needed.
