jgkzdl say:

boldasdoug,

I guess you are using DoCmd.OutputTo.

Using DoCmd.TransferSpreadsheet instead will give the result you wish I think, although the same as the manually export.

.

Something like this:

Sub ExportToExcel()
    Dim varA As Variant
    Dim ItemA As Variant
    varA = Array("Query_01", "Query_02", "Query_03")
    For Each ItemA In varA
        DoCmd.TransferSpreadsheet acExport, acSpreadsheetTypeExcel12Xml, ItemA, "D:\Data\MyExcelFile.xlsx"
    Next
End Sub

ahg862 say:

Assuming varA is the name of the tabs, try:

with column headers:

DoCmd.TransferSpreadsheet acExport, acSpreadsheetTypeExcel12Xml, ItemA, "D:\Data\MyExcelFile.xlsx", True, varA

Without column headers

DoCmd.TransferSpreadsheet acExport, acSpreadsheetTypeExcel12Xml, ItemA, "D:\Data\MyExcelFile.xlsx",False , varA

True/False will/will not include the column headers, and varA will be the the tab name.

MS documentation will tell you that this doesn't work, that you can only specify range on an import.

Here is the help page

"A string expression that's a valid range of cells or the name of a range in the spreadsheet. This argument applies only to importing. Leave this argument blank to import the entire spreadsheet. When you export to a spreadsheet, you must leave this argument blank. If you enter a range, the export will fail"

The documentation is incorrect. This does work. I use it all the time
