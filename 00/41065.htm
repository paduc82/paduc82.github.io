That problem comes (almost certainly) from the fact that in your hard drive, there is a messed up Windows 10 (or 8) that was previously installed (in UEFI mode, I bet). Suggested solution:

With C: the supposed letter of your Windows partition (you can use "e:" command to go to parition e, and "dir" command on each one to see its content, just to make sure you're on the good partition. (Of course, make sure you have no personal / important data on that partition.)

ian ser say:

Hi DJSlice659,

This can be usually done if the other user account is only a Standard user. What I advise for you to do is to make the app Run as Administrator. This way, before running the app, the Admin credentials are needed. Follow these steps:

  1. Press Windows key on your keyboard, find the app, right-click and select Open file location.
  2. Right-click the program and select Properties.
  3. On the Shortcut tab, click Advanced... button.
  4. Tick the Run as administrator check box and click OK.
  5. Press Windows key again on your keyboard, and type UAC.
  6. On the User Account Control Settings window, set the bar to Always notify. This way, every time the app is opened, Admin credentials are needed. NOTE: This will also prompt you whenever an app or software will attempt to make changes to your computer. In a way, it's also a good security feature so that you'd always be notified of changes on your PC.


Let me know if there's anything else you need.
