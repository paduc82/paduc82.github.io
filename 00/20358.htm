jay freedman say:

To make the tables' sizes constant, repeat this for each table:

  • Right-click in the table and click Table Properties.
  • In the Properties dialog, go to the Rows tab.
  • Set the height of each row to the desired value, and select "Exactly" in the dropdown. Use the Next Row button to work your way through the rows.
  • Click OK.

If a user tries to enter more lines in a cell (such as the Comments) than will fit in the allowed height, the extra text will disappear below the bottom of the cell. There's no reasonable way to prevent that.

To prevent editing of the text outside the intended cells:

  • On the Review tab of the ribbon, click the Restrict Editing button.
  • In the Restrict Editing pane, check the box under "Editing restrictions".
  • Leave the dropdown set to "No changes (read only)".
  • Select one or more of the cells that should be editable, and check the box in the pane labeled "Everyone". Repeat until all the editable cells have been marked this way.
  • Click the "Yes, Start enforcing protection" button, and click OK in the dialog box that pops up. (You can set a password there to limit who can turn off the protection, but you can leave it blank to have no password.)

split summary task - Microsoft Community

irenerodsa say:

split summary task

Hello everybody, 

I would like to know if is it possible to split a summary task in project.

Thanks in advance

john - project say:

Irene,

Simple answer is "no", at least if you are referring to the split task function in Project.

Please note that a summary task is not really a task at all despite the name, rather it is a summary of the tasks under it. The start of a summary line is defined by the start of the earliest subtask and the finish of a summary line is defined by the finish of the latest subtask.

You can "split" a summary by breaking the subtasks under it into two separate summary groups. That would effectively "split" a summary task.

Hope this helps.

Database Back End Password - Microsoft Community

confused slugs say:

Database Back End Password

I have a split Access database. The front end has no password protection. Is it possible to password protect the backend. I have tried to set one but nothing happens. If not, what options are available to prevent easy access to the raw data in the backend tables?
Thanks, I'm Confused
daniel pineault say:

To further the discussion, here are a few more ideas.

1- In a corporate environment, one of the best and easiest things to do is to restrict access to the directory that contains the database by setting up the appropriate permissions on the folder.  Setup a group comprised of the people that should have access and then give that group permission on the directory.  Be sure to use a group and not individual user profiles as this quickly gets out of hand to maintain when compared to a group...

2- Also if you are at all concerned with people copying the back-end,... messing around with data on their computers... I often embed the back-end with code that checks where it is located, if it is not on the corporate server, it send out an e-mail if it can with some nice information for me (IP, computer name, user name, email, ...) and then simply shuts down.

3- Also, only give your users access to the front-end.  They don't even need to know where the back-end is located.  As long as the tables are linked properly, it really is none of there business and avoid the issue altogether!

4- Depending on your environment, setup the users with the runtime version of Access rather than the full blown version of Access.  Then they can't even see the table little alone mess around with them!

5- Lastly, you could always upsize your back-end db to SQLServer... to give you another layer of security, but that is another can of worms...

Hope this gives you a few ideas to help you out.

christopher miller say:
If the message was sent as an inline image, you'll see a large square empty image as a placeholder. Just tap on the image placeholder once. It should then download the picture (give it a minute) and then insert a file link with the filename of the picture into the message.  You can tap on that link to view the image.  If the image was sent as a file attachment, it should show up as a filename link.  Click on that link to download the image, then after it has downloaded click on the link to view the file.
