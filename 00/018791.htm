sschultz0956 say:

Making a StDev into an IF statement...

I have a huge list of stacked data.  There are about 200 product names over years of data (each product has multiple lines) with quantities in the next column.  I want to calculate the StDev of each product without having to dump my data into minitab and add a number of extra steps. 

I tried something like: =IF(A:A = C4, STDEV(b:b))  Where A and B are the list of duplicate products with quantities and C as the list of unique products.  This didn't work.  Any thoughts on how I can make this work?

Thanks in advance.

joeu2004 say: sschultz wrote:
I tried something like: =IF(A:A = C4, STDEV(b:b))  Where A and B are the list of duplicate products with quantities and C as the list of unique products.  This didn't work.  Any thoughts on how I can make this work?

I'm guessing that you want the following array formula [*]:

=stdev(if(A:A=C4,B:B))

But I don't know if XL2007 supports references like A:A in that context.  XL2003 does not.  So you might need to write the following array formula:

=stdev(if(A1:A200=C4,B1:B200))

Adjust the ranges accordingly.

[*] Enter an array formula by pressing ctrl+shift+Enter instead of just Enter.  Excel will display the formula surrounded by curly braces in the Formula Bar, viz. {=formula}.  You cannot type the curly braces yourself.  If you make a mistake, select the cell, press F2 and edit as needed, then press ctrl+shift+Enter.

PS....  You probably want references of the form $A:$A and $B:$B or $A$1:$A$200 and $B$1:$B$200 if you intend to drag the formula down or across.

The error message "Word could not create the work file. Check the temp environment variable". Can be caused by incorrect registry values in the key:

HKEY_CURRENT_USER\Software\Microsoft\Windows\Current Version\Explorer\User Shell Folders.

Click Start and in the start search field, type

Regedit

and press enter.

In the registry editor, browse to:

HKEY_CURRENT_USER\Software\Microsoft\Windows\Current Version\Explorer\User Shell Folders

On the right hand pane, check the value data for Cache.

It has to be %USERPROFILE%\AppData\Local\Microsoft\Windows\Temporary Internet Files.

If it is different, double click on Cache and type:

%USERPROFILE%\AppData\Local\Microsoft\Windows\Temporary Internet Files

under value data and click ok. Close the registry and  restart the computer and check if it works.

 

I think that the error can also be caused by having the File>Preview facility enabled in the File Open dialog.  So, if the above does not fix the problem, check for that.
