ivana-likualova say:

For example:

A1=30
A2=20
A3=10
A4=0
A5=60
A6=50
A7=40
A8=90

I want B1 to be the first occurrence of a number greater than A1 in the array A2:A8. It should return "60", since, though A6 is also greater than A1 it is not the first number in the array to fit that criteria. Any ideas?

joeu2004 say:
I want B1 to be the first occurrence of a number greater than A1 in the array A2:A8. It should return "60", since, though A6 is also greater than A1 it is not the first number in the array to fit that criteria. Any ideas?

Array-enter the following (press ctrl+shift+Enter instead of just Enter):

=INDEX(A2:A8,MATCH(TRUE,A2:A8>A1,0))

When correctly array-entered, you will see curly braces around the entire formula in the Formula Bar; that is, {=INDEX(...)}.  Note that you cannot enter the curly braces yourself.  That is just Excel's way of denoting an array-entered formula.

[EDIT] You don't say what you want if there is none greater.  Perhaps the following will do what you want (again, array-entered):

=IFERROR(INDEX(A2:A8,MATCH(TRUE,A2:A8>A1,0)),"")

quietman7 say:
Trojan.Ransomcrypt.D (DirtyDecrypt.exe) encrypts various types of documents files (.doc, .docx, .docm, .xls, .xlsm, .xlxs, .avi, .jpg, .jpeg, .png, .pdf, etc), locks the Desktop and asks for a payment in order to unlock the infected computer.

The newer crypto malware ransomware variants are encrypting files using a RSA encryption which utilizes a Public and Private key pair. All the local encryption is done with Windows API but the keys are encrypted with RSA. Once the encryption of the data is complete, decryption is usually not feasible...you need BOTH the Private RSA key (unique to the infection machine) and the corresponding Public RSA key (generated and contained in the registry) on the infected computer. The unique Private key is stored on the malware developer's command and control server which makes access to it, and the restoration of the affected files near impossible. Bottom line: there is no way to break RSA/AES encryption without both decryption keys and the encrypted data cannot be recovered.

The only reliable way to protect your data and limit the loss in case of this type of malware infection is education and to have an effective backup strategy. This is not only effective against ransomeware but also helps with other catastrophic scenarios like hard disk failure. Unfortunately, since many users do not have a backup strategy they are forced to pay the ransome demand to recover their data.

However, some victims of crypto malware infection have reported success recovering data using Shadow Explorer.
