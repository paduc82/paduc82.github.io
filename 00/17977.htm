msiexec /i <folder where the contents where extracted to at step 2>\edumathaddin_x64.msi fromsetup=1 word=1 onenote=1 ALREADYRUNNING=0 DOTNET4=1

rob koch say: Not sure I see the mystery here.  Petite 2.1 is an executable unpacker, so if it's included with the EXE file it must be packed and thus required for the unpacking process.  Since some form of code within the execuatble would need to invoke or reference the unpacker, any scanning of the executable would directly imply the reference and cause MSE to scan that also.

We know from previous posts that the MSE Heuristcs performs pre-execution testing, so this would also need to perform testing of both the executable and the unpacker, resulting in whatever overhead this might create.

My guess with the definitions updates would be that as with any such update, it's possible that something might have changed to detect something new, so all previous 'learned' information is reset to require the scanning process to check with the new definitions.  Whenever the learning process has performed enough passes to determine the particular items are 'safe' it again simply skips over portions of the process until either the specific executables or definitions change state (are modified in some way).

Rob
