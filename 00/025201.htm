a. user say:

I don't know precisely what the definition of a "Jason Tab" is, but AFAIK the Styles used by the TOC have one tab defined, and in that case the dotted leader should always be there no matter how short the text. If you have modified the style, either directly, or so that it is based on a style that has more tabs defined, then if the heading text does not extend beyond the first tab position, Word will only tab to that position, and if the tab does not have a dot leader defined, you won't get a dot leader (rather obviously, I suspect!). So if you click in the offending paragraph and look at its paragraph style details, you should be able to check that stuff. If there is another tab, it's then a question of working out how that tab definition got there. And the "default tabs" should not be a factor, i.e. they should be overridden by the presence of non-default tabs.

If there is a specific known bug that causes this, you probably know more than I do.

suzanne s. barnhill say:

By default, Word 2010 uses the List Paragraph style for numbered and bulleted lists. This is what you will doubtless have to modify. In the Styles task pane, with Recommended styles displayed, in "As Recommended" order, List Paragraph will be the very last one. It is based on Normal, which is based on the document defaults, so it would appear that you have not changed the default font in the document (which you may not want to do).

If you don't want to use the List Paragraph style, there is an option to "Use Normal style for bulleted or numbered lists." Find it in the Compatibility Options (Layout Options) at the very bottom of File | Options | Advanced.
