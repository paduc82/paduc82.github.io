In addition to everything else, make sure nobody has the database open when you try to compact. Navigate to the folder containing the database using explorer, and look for a file with the same name as your database but with either a .ldb or .laccdb extention. This is a locking file Access creates when the database is opened.

 

If that file exists, somebody hads the db open, or Access thinks it is open. If the DB is not properly exited (ie it crashes), then sometimes the locking file is not deleted. If you know for a fact the db is closed, try deleteing the locking file.

 

Like everyone has said, you need to split your mutiuser DBs.

The endnotes are properly cross-referenced in this document. However, he is continuously updating his version of the document, and so the text of my version, other than the endnotes, is out of date.

What I would like to do is find a way to paste in all the endnotes (nearly 600 of them) into his version of the document while maintaining the cross referencing. Is this possible? The only other alternative I can think of is to manually paste each note into the new document, which will be painstaking to say the least. Eventually, once the formatted endnotes and the updated text is synced up, we will section off the chapters with their respective endnotes, and save them in separate documents (which is easy).

Thank you in advance for any help you can provide.
deanh say:

Do the compare the other way, open your document (the one with the 600 (!) endnotes), save as a new version or copy (just to be safe), then Compare and Merge (Review tab, Compare group, click Compare) and select his new version "into" this new version/copy document.

This way you will have all the endnotes references in the right place and you will see (using the usual Track Changes notation) the changes the author has done. You can now work your way through the document, accepting the changes, you can accept all in one go if you feel brave :-)

See http://www.shaunakelly.com/word/trackchanges/HowTrackChangesWorks.html if you are unsure on how TC works.

Hope this helps
