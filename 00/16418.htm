chip pearson say:

Excel 2010 can read and update 2003 files just fine. Worksheet size is limited to the 2003 number of rows and column. In your 2003 worksheet, don't use any feature added since 2003. If you do, you'll get a message when saving the workbook indicating that some features will not be saved. You can get the 2003 Compatibility Pack that allows Excel 2003 to use 2010 workbooks. When  you do so, you can't add any 2010 features; you're limited to 2003 features.  In Excel 2010, you can create workbooks and save them in the 2003 so others who still use 2003 can use them. Again, just don't use any new features (later than 2003) in the 2003-format workbooks.

Very little, if any, code will need to be modified to make a 2003 workbook compatible with 2010. I have a hundred thousand lines of code in my developer library and I can't recall making any changes to existing code for compatibility with 2010.

You can have both Excel 2003 and Excel 2010 installed on the same machine. Just ensure that they are installed in chronological ascending order (e.g. 2003 before 2010). I have several versions of Office on my main machine and the different versions of Excel play well together.

Note that all this applies only to 32-bit Office. If you are using 64-bit, you must remove Office 2003 before you install Office 2010. You cannot have both 32-bit and 64-bit Office programs on the same machine.

cindy meister say:

Hi Pegstar

<<Working with Word 2007, I have created a new caption label that I use throughout the document. When I save the changes I made to the document and open the document at a later stage, the new caption label is not available from the 'Insert Caption' drop-down box.

If I recreate it, it continues the numbering from the last caption number (i.e. in the previous version of the document). By what I read in Word 2007 help, the new caption labels are supposed to be available once created. Is there a setting for this>>

Caption Labels are stored in the Normal.dotm template. If you quit Word and changes aren't saved, then the caption label will be lost.

By default, changes to Normal.dotm should be saved. Various reasons change would not be saved:

  • you have the option set to prompt for changes and explicitly said, "No"
  • Normal.dotm is saved in a folder where you don't have "Write" permissions
  • you have an Add-in that is telling Word to not save changes to Normal.dotm

As long as it's not the second point listed above, you can force a save of Normal.dotm by

  • press Alt+F11 to open the VBA editor
  • Make sure the selection is in "Normal" in the "Project" window (usually top-left)
  • File/Save
