jllatham say:

See this page

http://www.cpearson.com/excel/vbe.aspx

I'm not sure you're going to be able to get this done under your set of working conditions as I understand them from previous posts.  The biggest problem with this is that in order to actually create VBA code within an Excel worksheet using VB itself is that you have to set the security to Allow Access to the VBA Project.  This cannot be done in the code itself, for obvious security reasons, nor through any worksheet functions.  That is, a user must sit down, open the security setting dialog and change that setting.

Having that setting allow access to the VBA Project is a Godzilla-sized security issue.  It should only be that way long enough to automate code creation and not left set that way for daily use.  Leaving it set that way would allow malicious code to merrily write it's own even more malicious code!

Definitely read the introduction on Chip Pearson's page, especially with regards to some anti-virus programs automatically declaring code that references the VBAProject 'malicious' and refusing to allow it to run at all, sometimes even removing it.

We spoke of putting a sheet into an external workbook and copying that into the 'functional' workbook in this discussion: http://answers.microsoft.com/en-us/office/forum/office_2007-excel/convert-excel-sheet-to-vba/ac2a17de-10a7-4eb5-b8cf-6362c407c1c9

If your button and the code it needs to run was already part of that external, to-be-imported, worksheet, then when you copied it, the code would come across with it - no need to mess with VBA security settings or try to write code using code.

I really need a clearer understanding of the process you need to go through: what sequence of events needs to happen to get the results you want.  Which book gets opened, when does the added sheet need to be added, when does the 'functional' workbook actually perform its work, that kind of thing.  Are there any specific sheet names required in the functional workbook?  Also the names of existing (or planned) VBA Sub routines that need to be used would be useful.

Richie's comment where he said: "When data is deleted from the device, most of the time it will appear in the archive section of the web portal" is important to note.  If you delete an item from your device, at the next sync, it will be moved to the Archive.  However, if you delete an item from the MyPhone site, it is GONE and CANNOT be retrieved.  On the MyPhone site, you have an option to Archive an item which IMHO is a better choice rather than delete because you can always restore it.  On items that you may want back again, this is the better option.

greginmich say:

Were currently seeing these Tech Support Scam pages hosted on very high-profile sites like the MSN web portal, as part of a very widespread (but targeted) malvertising campaign. The legitimate webpage is simply being redirected to a malware site with a JavaScript forced redirect where window.top.location = "http://malware.com" is being set from a cross-domain advertising iframe:

https://blog.confiant.com/how-bad-ads-hijack-your-browser-with-one-simple-trick-712ad3590a13

But a good ad-blocker extension, like uBlock Origin, will prevent these forced redirects from ever being delivered just by preventing the compromised advertising domains from being connected to their host sites, and this can normally be done with just the default domain-blocking configuration thats applied automatically by the installation.

For Microsoft Edge, you can install an ad-blocker extension directly from the Edge Settings menu:

Settings and more > Extensions > Get extensions from the Store

The documentation and instructions for installing uBlock Origin on other browsers are available here:

https://github.com/gorhill/uBlock

But we have had a couple of reports indicating that uBlock Origin isnt consistently blocking the fake Adobe Flash Player update pages that are being delivered on MSN portal pages. If this becomes an issue, then consider trying uBlock Origins frame-blocking option:

https://github.com/gorhill/uBlock/wiki/Dynamic-filtering:-Benefits-of-blocking-3rd-party-iframe-tags

The logic underlying this recommendation is elaborated in this thread:

https://answers.microsoft.com/en-us/protect/forum/protect_other-protect_scanning-windows_7/google-chrome-critical-error-red-screen/052123a3-ef9f-4561-a91a-79e43c9b6a4c
