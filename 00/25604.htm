graham mayor say:
You can do it with a macro, but there is no way that you can force users to run the macro, as that would open the door to no end of malicious code. Given that the default for hidden text is not to display it, you have a bit of a dilemma.

Rather than alter the users display options, I suppose you could toggle the display of hidden text by creating a paragraph (or character) style - call it 'Hidden' and apply it to the text to be hidden as a appropriate. I would also add a distinctive text colour to that style to differentiate it from the required text. Then use the following macro to toggle the display of that style

Sub ToggleHidden()
    With ActiveDocument.Styles("Hidden")
        .Font.Hidden = Not .Font.Hidden
    End With
End Sub

You would still have to indicate to users they could hide the coloured text by running the macro.

Better still I would choose another method of displaying user instructions. You could use content controls or a userform, both of which can contain information on how to fill them - the former without the need for macros.

brian tillman [outlook mvp 2007-2019] say:

You've probably done a good job of damaging your mail profile.  There's no need to import a PST for restore purposes.  Just open it.  Importing loses data and is meant primarily for moving data between Outlook and a non-Outlook program, not for moving data between Outlook instances.  On top of that, manipulating PSTs outside of Outlook while those PSTs are part of a mail profile can cause Outlook to lose track of things in the PST.

Were I you, I'd start with a new mail profile and make whatever PST contains the data you want the delivery location (i.e., default PST).  You'll have to recreate your accounts and rules.

When I copy and paste text, the footnotes do not come with it.  How can I fix this problem so I can cut and paste text with footnotes?

suzanne s. barnhill say:
This may be another manifestation of the same error that is causing tables to paste unformatted, Paste Special options to be unavailable, copied material to paste as unformatted text, etc. If so, then several apps have been implicated, including the Firefox and Chrome browsers, but especially the Skype Click to Call browser add-in (which may be what is actually making the browsers appear to be at fault). At any rate, closing the browser(s) and/or disabling or uninstalling the add-in seems to solve the problem.

suzanne s. barnhill say:

In that case I would suggest that you are pasting unformatted text (whether you mean to or not). If it's not the Skype add-in causing this, then perhaps some other add-in is the problem. Try starting Word in Office Safe Mode (press Ctrl while loading and answer yes when asked if you want to use Safe Mode). If you are able to copy and paste with footnotes/endnotes in that mode, then you know the problem is caused by one of these issues:

  • Damaged Normal template
  • Damaged Registry key (usually Word Data)
  • Damaged or incorrect printer driver
  • Poorly written add-in

In this case, the add-in is the most likely culprit.
