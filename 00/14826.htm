2. If I split a bulleted list or indented paragraph between two pages, it looks right on the odd page, and wrong on the even.  And if I put it entirely on the odd page it works.  But if I move it entirely to the even page, it doesn't work.  No change in formatting involved...just a change in page.

3. Distance is maintained between the two markers, but position is ignored.  By this I mean:
- On an ODD page, if I set the bottom marker to -.5 inches, and the top marker to +.5 inches, the first-line-indent is .5 inches from the margin, like it should be.

- So it's as if, on my even pages, word is reading the bottom marker as being on the margin-edge, no matter where it is, and measuring the top marker accordingly.

Mirrored margins will automatically change the position of your text.

Formatting of bullets or numbering should NOT be done on the ruler.

Rather these should be dealt with in the dialogs that deal with them.

  • How to control bullets in Ribbon Versions of Word by Shauna Kelly

A couple of things to try manually. (I would first copy the file and try this stuff on the copy).

1) Use Excel's Find command to look for [prices.xls] in formulas. You may find some cells using this as a link.

2) Use the Ribbon Formulas -> Use in Formula -> Paste Names -> Paste List to paste a list of all your named ranges into a blank worksheet. Check that none of the names have formulas that refer to the external link.

3) Use the Ribbon Data -> Edit Links and change the link source to point to the same workbook that you are editing. This effectively removes the links to an external file. You would then need to check your workbook to see if anything is broken.

ronrosenfeld say: The macro below seems to work whether the sheet is formatted as text or General.  It will also work if the entries in a row are not consecutive.

For debugging purposes, the output goes five lines below the input table.  But that is easily changed if things work out.

Depending on the real layout of your worksheet, the part of the routine that finds the region to be right justified may need to be changed.  For this initial try, I merely used Range("A1").currentregion, assuming that will pick up all the cells involved.

I chose to work in an array within the VBA module, as multiple reads/writes calls to/from the worksheet can be time consuming.

To enter this Macro (Sub), <alt-F11> opens the Visual Basic Editor.
Ensure your project is highlighted in the Project Explorer window.
Then, from the top menu, select Insert/Module and
paste the code below into the window that opens.

To use this Macro (Sub), <alt-F8> opens the macro dialog box. Select the macro by name, and <RUN>.
======================================
Option Explicit
Sub MoveIt()
    Dim rSrc As Range, vSrc As Variant
    Dim i As Long, j As Long, k As Long, l As Long
    Dim sTemp As Variant
    Dim bNoMove As Boolean
'select the region
'may need a fancier method, depending on
'  what the sheet really looks like

Set rSrc = [a1].CurrentRegion
vSrc = rSrc
For i = 1 To UBound(vSrc)
Do
    bNoMove = True
    For j = UBound(vSrc, 2) To 2 Step -1
        If vSrc(i, j) = "" Then
            'move everything one to the right
            For k = j To 2 Step -1
                vSrc(i, k) = vSrc(i, k - 1)
                If vSrc(i, k - 1) <> "" Then bNoMove = False
                vSrc(i, k - 1) = ""
            Next k
        End If
    Next j
Loop Until bNoMove = True
Next i

rSrc.Offset(rowoffset:=rSrc.Rows.count + 5) = vSrc

End Sub
==================================
