stephane zwel say:

@E d watters,

Thanks for the file.

It appears upon inspection that the reason why this "empty" file is so big is because it maintains a cache of cells from 8 external files.

Whenever a formula makes a reference to an external file, Excel stores a cache of all the values in the range, because it thinks it may never be able to reconnect to that external file ever again. It's a good thing, unfortunately it has the very consequence that this cache is arbitrarily big.

Here is the rundown of the internal cache of values from the external files referenced in your file :

(77 MB) Casemix - SHA-18-MASTER-6K.xlsm
(6 MB) CASEMIX HOSPITAL-EXTRACTION MODEL-MASTER-1-T.xlsm
(772 KB) Test Naas1-7.xls
(124 KB) XLS/CM - 19082014/SHA HOSPITAL COMPILER-INPUT-2.xlsm
(10 KB) ASF Merge output - 20062013paste specC&amp;AG audit certFINAL.xls
(8 KB) 01_Hospital Costing file - (JMES)  2012 (Constructed on 2010) (version 2).xls
(1 KB) ComboCode Tester.xlsm
(554 B) Luked


The total size is much bigger than your file size itself, which is "only" 8.5 MB. The reason why is that Excel compresses XLSM files. That's why the underlying problem is not obvious.

At this point, there are two choices :
- severe the ties with the external files
- clear the cache

Severing the ties cuts both the references and the cache. It may or may not be wise to do so depending on what you want because obviously if you are severing ties with the external files, you may just as well start with a brand new Excel file.

If you are willing to do this, on a external file by external file basis, it is very easy. Just click on the ribbon / Name Manager. Then click on the "Refers To" column in order to sort the external references according to the files. If you scroll down you will see the links to Casemix - SHA-18-MASTER-6K.xlsm. You can multiple select those links and then click on Delete and then OK. When you save your file, it goes down to 800 KB, or just 10% of your file size. You may iterate over the remaining 7 external files.

Clearing the cache is another beast because there is no way to do this with Excel. So I just did this manually with special tooling. A benefit is that even though the file went down to as low as 35 KB, it still has the ties to all 8 external files. Which means whenever you are going to use the corresponding defined names in formulas Excel will be able to reconnect to the files and fetch the values (it will ask). The unfortunate consequence is that whenever Excel refreshes new values it will recreate the cache that was just freed.

bkwsnx say:

Using Data Validation to create a dropdown menu that displays numbers only in a range with text and numerical values

Hello,

I have a column in which A1:A100 contains a value based on information from another part of the workbook.

Each cell contains either the text value "N/A" or a unique whole number.

What I would like is to have a drop-down menu created that will list only the numerical values, in order from A1:A100, completely ignoring any cell with "N/A"

I've tried combinations using INDEX, IFERROR, ISNUMBER and OFFSET but haven't been able to get just the output I'm looking for.

Any help is greatly appreciated!
ashish mathur say:

Hi,

 

Try this

 

1. Since you want the numbers to appear in ascending order in the data validation list (minimising the use of spare columns), i have had to use an addin (Morefunc) to get the result.  For downlaod and installing the addin in Excel 2010, refer to point 4 onwards at the following link - http://www.ashishmathur.com/extract-data-from-multiple-cells-of-closed-excel-files/

 

Once you have downloaded and installed the addin, try this

 

1. I have assumed that data is in range A1:A10

2. In cell F2, enter this formula and copy down

 

=INDEX(VSORT($A$1:$A$10,$A$1:$A$10,1),SMALL(IF(ISNUMBER(VSORT($A$1:$A$10,$A$1:$A$10,1)),ROW($A$1:$A$10)),ROW(1:1)),1)

 

VSORT is the function of the Morefunc addin.

 

3. In cell G2, go to Data  > Validation > List.  in the Source box, enter this formula

 

=OFFSET($F$2,COUNT($A$1:$A$10)-1,0,-COUNT($A$1:$A$10),1)

 

Hope this helps.

Why not use a summay row?

On row 22 enter a formula like this in B22:

=IF(COUNT(B10:B21),AVERAGE(B10:B21),"")

Copy across to K22

Then your MAX formula is simple:

=MAX(B22:K22)

And for the nth largest value:

=LARGE(B22:K22,n)

Where n = the nth value you want.
