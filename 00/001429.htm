Tom,

 

I'm not sure I understand this statement.  Excel has headers and footers, and they can be customized in many different ways.  They also show up on every page of the printed worksheet.

 

In addition, there is the functionality of setting certain rows to show up on every printed page, thus providing a second type of header - the headers to the columns.

 

It is also quite possible to limit changes to the worksheet structure (such as column width changes) that the OP is wanting to do.

 

NSNR - here are some things you can try:

  • Copy all of your existing data from the Word document to an Excel worksheet.
  • Format everything as you wish - font, font size, column headers, column widths, etc
  • Go to the Page Setup dialog, on the Header/Footer tab, and enter what you wish to have for headers and footers (document title, date, page numbers, etc)
  • Also in Page Setup, on the Sheet tab, you can specify which rows at the top of the worksheet should show up on every printed page - select the rows with your headers.
  • Once you have everything formatted the way you want it, select only the Columns that you want the user to be able to change.  Right-click the mouse on the selected cells, and choose "Format Cells".  On the Protection tab, uncheck the "Locked" option - this will allow users to change the contents of those cells after the worksheet and workbook are protected (next steps).
  • Go to the Tools menu and select "Protection/Protect Workbook...".  Make sure "Structure" has a check mark next to it.  If you want, you can supply a password that must be entered later if you want to unprotect the workbook structure.  Click OK
  • Go to the Tools menu and select "Protection/Protect Sheet...".  Make sure there is a check mark next to "Select unlocked cells".  Make sure nothing else has a check mark next to it, unless you want the user to be able to take that action (for example, inserting or deleting rows).  Again, you can supply a password if you desire.  Click OK
  • You now have a worksheet in which the user can enter new data and edit existing data, but cannot change how the worksheet looks or prints out - you have set the way you want that to work, and the user cannot change it.  The user will be able to change some things about the printout - Protrait or Landscape, custom header or footer, but not the print area or rows/columns to repeat on each page.

joeu2004 say:

Nico wrote:

Y values: 0.123, 0.138, 0.145, 0.169, 0.189, 0.213, 0.245, 0.269, 0.281, 0.296

The very last part of your question confuses me because I thought that might be exactly what you want.

You do not say how you use FORECAST with that data.  The "obvious" way is FORECAST(1, $Y$1:$Y$10, $X$1:$X$10).

And I thought that "didn't work" for you because that interpolates the best-fit linear regression line, which does not fit the original data exactly.  Consequently, for example, FORECAST(X1, $Y$1:$Y$10, $X$1:$X$10) returns about 0.1157, not the value in Y1.

If indeed you would be happy with pairwise interpolation (I would), try the following formula, where "x" is in A1:

=IF(A1<$X$1, FORECAST(A1, $Y$1:$Y$2, $X$1:$X$2),

FORECAST(A1, INDEX($Y$1:$Y$10, MATCH(A1,$X$1:$X$9)):INDEX($Y$1:$Y$10, 1+MATCH(A1,$X$1:$X$9)),

INDEX($X$1:$X$10, MATCH(A1,$X$1:$X$9)):INDEX($X$1:$X$10, 1+MATCH(A1,$X$1:$X$9))))

Note the use of $X$9 in the MATCH expression, not $X$10.  That is on purpose.

For "x" less than X1, the formula extrapolates the line defined by the first two data points.  For "x" greater than X10, the formula extrapolates the line defined by the last two data points.  Otherwise, the formula interpolates the line defined by surrounding pair of adjacent data points.

Thus, for the values in X1:X10, the formula returns the values in Y1:Y10.

Obviously, it would more efficient to calculate MATCH(A1,$X$1:$X$9) one time in a cell, and replace every MATCH expression with that cell reference.  But often, that over-complicates the worksheet design.

The formula might be easier to read if we use OFFSET.  But that is a "volatile" function; so that formula and any directly or indirectly dependent cells would be recalculated every time any cell in the workbook is edited.  The INDEX:INDEX paradigm is not "volatile".

kha422 say:

Thanks for ALL the guidance ... Here is what I ended up with after reviewing your remarks in this thread ...

Also, not sure if merged cells are issue for Macro's, but by changing Columns AA:AE to Range (AA3:AE3) everything seems to work as I wished ...

 

    ActiveSheet.Unprotect
   
    Range("AA3:AE3").Select
    If Selection.EntireColumn.Hidden Then
        Selection.EntireColumn.Hidden = False
    Else
        Selection.EntireColumn.Hidden = True
    End If
   
    ActiveSheet.Protect DrawingObjects:=False, Contents:=True, Scenarios:= _
        False, AllowFormattingColumns:=False, AllowFormattingRows:=False, _
        AllowDeletingColumns:=False, AllowDeletingRows:=False, AllowSorting:=True, _
        AllowFiltering:=True

    Range("F4").Select
    Range("A4").Select
End Sub

jay freedman say:

If you use Single line spacing in the Paragraph dialog, Word will always adjust the spacing to be greater if there are any characters that extend above or below the nominal font size -- whether subscripts, superscripts, or regular characters in a larger size or from a larger font.

The only way to prevent that is to use "Exactly" instead of any of the other settings in the dropdown, and then specify the number of points to allow between baselines. That's typically a point or two larger than the font size, although that varies by font.

To keep this spacing uniform throughout the document, it's best to make this setting in the style (such as Normal or Body Text) that you apply to the text.

suzanne s. barnhill say:

In addition to using exact line height, which is really the only easy solution for Word 2013 and above, there is a Compatibility Option in previous versions, "Don't add extra space for raised/lowered characters." This option is available in Word 2013 and above only for legacy documents (in Compatibility Mode). If for some reason you find exact line spacing unworkable, you can make this option available by saving the document in Word 2010 format using these steps (courtesy of Jay Freedman):

  1. Press Alt+F11 to open the VBA editor.

  2. Press Ctrl+G to open the Immediate window at the bottom of the editor.

  3. In that window, paste this line and then press Enter:
    ActiveDocument.SetCompatibilityMode wdWord2010

  4. Close the VBA editor.

  5. Go to File | Options | Advanced and scroll to the bottom.

  6. Change the dropdown "Lay out this document as if created in" to "Microsoft Word 2010". At that point, the list of 2010 Layout Options will appear below the dropdown.

You will, of course, sacrifice the use of some features that were new in Word 2013.
