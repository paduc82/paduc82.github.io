I will add some bits not covered by Rohn007, but won't bother repeating all the good stuff already written.

5. Are some styles really worth the effort? Example, setting a style for emphasis? It seems like it takes longer to select the text & apply a style than to Ctrl+ B if I just want a simple bold for emphasis.

You might change your mind and decide you want emphasis text to be bold and undelined, or be a different colour etc. So applying a style means you can make this change later. It might also mean that in other media (eg screen reading software) it treats this differently. You can easily assign shortcuts to styles , including re-assigning common ones such as Ctrl+B to add emphasis style instead of direct bold formatting. Personally, i admit I just used Bold and italic the old-fashioned way (and almost never use underline).

8. When I modify or update a Style in a style set, does it change that style set for good? If I change a style, am I saving over Word’s default styles or can I revert back?

When you apply a style set, Word copies the styles defined in the set into the current document, but does not keep any link or association with the set. If you make modifications they do not affect the style set unless you explicitly go and save the style set and overwrite the existing one. You will not override default styles unless you tick to save the changes to "new documents" as well, and save the template when closing. You can always re-apply the standard Office (or Office 2003) stylesets unless you deliberately save over them.

9. I don’t understand the shortcut menu items in the Styles group on the Home tab. When I click “Modify…” am I overriding a preset Word style or creating my own style based on Word’s style?

As above, it all depends when you do this - if you create your own template and change the styles in that template, that't the only place they change. If this is your normal template, then yes, you overwrite the built-in defaults, but can restore them from a style set (probably).

10. How do I specify the Heading level of a style that I create from scratch (so that it shows up in the Navigation panel/outline view)?

Heading levels (more properly, outline levels) are part of paragraph formatting, so when modifying or creating a style, go to paragraph formatting and choose the outline level number you want, or body text.

How do you put your various customized styles under a Set (assign them to a Set)?

You can't "assign" them to a set, but you can load a set, make changes then save the set and overwrite it. Style sets are saved in template files, so you can manage and rename them, copy them between machines etc just like any other files.

What’s the difference between using Style sets & applying a Theme? Are themes basically just used for color?

Style sets contain only text styles (I think), not other formatting things such as colour palettes, effect options, table styles etc. Style sets are a Word-only thing. A theme will store the font scheme, colour palette, possibly effect options (although I have never met anyone who bothered customising these), and in PowerPoint will also store slide masters.You can also store these separately, eg save just a color scheme on it's own.

A single theme may be used to apply consistent formatting across Office, ie a thmx file may be applied to a Word document, Excel spreadsheet or PowerPoint slideshow to use the same fonts, colours etc. Many companies deliberately don't do that since the colours you may want in Excel charts are likely very different from the ones you want for corporate branding in a PPT show, and font choices may also differ for on-screen use (Excel, PPT) and printed matter (Word especially). I often see separate themes for different apps, especially PPT because ther thmx file can get quite large if they embed lots of standard slides as masters which include high-res pictures.

Fill example that sets a gradient fill with varying transparency:

        Set GradientShapeFill = theShape.Fill
        With GradientShapeFill
                .ForeColor.RGB = RGB(0, 128, 128)
                .OneColorGradient msoGradientVertical, 1, 1
                scaleHSL.Hue = tHSL.Hue
                scaleHSL.Saturation = myMin(tHSL.Saturation, 0.99999)
                scaleHSL.Luminance = tHSL.Luminance * 0.7
                tRGB = HSLtoRGB(scaleHSL)
                .GradientStops.insert RGB(tRGB.Red, tRGB.Green, tRGB.Blue), 0#

                scaleHSL.Hue = tHSL.Hue
                scaleHSL.Saturation = myMin(tHSL.Saturation, 0.99999)
                scaleHSL.Luminance = tHSL.Luminance * 0.85
                tRGB = HSLtoRGB(scaleHSL)
                .GradientStops.insert RGB(tRGB.Red, tRGB.Green, tRGB.Blue), 0.5

                scaleHSL.Hue = tHSL.Hue
                scaleHSL.Saturation = myMin(tHSL.Saturation, 0.99999)
