scott .. b say:

Sorry I couldn't be of more assistance.......... 

To your question ".........Why should I have to do all of that?........."

The issue is that many web developers have not upgraded their HTML code to work on the new Windows 8.1/IE 11 operating system because to do so would be quite labor-intensive.  The same is true for servers, particularly those in foreign markets.  Consequently, many web developers still use the "older" code that was developed for Windows 7/IE9 which works perfectly with older servers and computers still running   operating systems such as Vista, XP, etc..  

With the introduction of the new Win8.1/IE11 operating system, however, many sites do not operate at all unless the webmaster updates the code.  Folks cannot open links to their their banks,  work locations, etc., because of the disconnect between old and new.  The "workaround" (yes, that's what it is)  was to create a browser emulation routine (Compatibility View), which essentially converts the older code to the newer code and allows limited browsing until the sites are upgraded.

So, why is it that Chrome doesn't suffer this (your question)?  Because Chrome has developed a protocol (called (SPDY/3.1) that supplants HTTP by framing layers for multiplexing over a single TCP,  (tech stuff), meaning that applications that run on HTTP today will work using the SPDY/3 protocol with little or no change required to the website by its developer. (It bridges the gap, so to speak). 

Per your last post, you said that you didn't install any of the "Optional" updates.  Understand the Bing, but if you have this optional update not yet installed, recommend you install it.  This particular update is a roll up of fixes for a few dozen known IE problems. 

   " Update for Windows 8.1 for 64-based Systems (KB 2955164)"  5/13/2014

       Details...........  http://support.microsoft.com/kb/2955164

Me.comboboxname.Visible = Me.checkboxname

stefan blom say:
You can apply numbering to the Comment Text paragraph style as follows: With the insertion point within the text of any comment, press Ctrl+Shift+S to display the Apply Styles pane. Verify that the style name displayed is Comment Text. Click the Modify button. In the dialog box, click the Format drop down, and then click Numbering. Select the desired format and click OK. If you want to transfer the modified style to your template, click "New documents based on this template" before clicking OK.
