ganesh kumar n say:

This issue occurs because Publisher sends the publications as one print job, instead of sending separate print jobs for each publication. The printer cannot distinguish the end of one publication from the start of another publication.

Change the number of records for each batch in the registry.

Note: - Serious problems might occur if you modify the registry incorrectly by using Registry Editor or by using another method. These problems might require that you reinstall your operating system. Microsoft cannot guarantee that these problems can be solved. Modify the registry at your own risk.

1.      Quit all Windows programs.

2.      Click Start, and then click Run.

3.      In the Open box, type Regedit, and then click OK.

4.      Locate and select the following key in the Registry: HKEY_CURRENT_USER\Software\Microsoft\Office\12.0\Publisher\Printing

5.      Right-click the Printing Registry entry, point to New, and then clicks String Value.

6.      Rename the new string value registry entry to "NumberOfRecordsPerBatch".

7.      Right-click the NumberOfRecordsPerBatch registry entry and then click Modify.

8.      Type 1 in the Value Data box, and then click OK.

9.      Quit Registry Editor.

Publisher 2007 will send one record at a time to the printer. The printer can then collate and staple the document.

Function AutoFilterTopRow() As Long
  'Attention! Works only with small ranges!!!
  Dim WS As Worksheet, R As Range
  Set WS = ActiveSheet
  'Return 0 if not filter is set
  If Not WS.AutoFilterMode Then Exit Function
  'Get the whole range from the filter
  Set R = WS.AutoFilter.Range
  'Remove headings
  Set R = R.Offset(1, 0).Resize(R.Rows.Count - 1, R.Columns.Count)
  'Get the visible cells if any
  On Error GoTo ExitPoint
  Set R = R.SpecialCells(xlCellTypeVisible)
  'Return the row of first cell
  AutoFilterTopRow = R.Row
ExitPoint:
End Function

  'Attention! Works only with small ranges!!!
  Dim WS As Worksheet, R As Range, A As Range

  'Get the whole range from the filter
  Set R = WS.AutoFilter.Range

  Set R = R.Offset(1, 0).Resize(R.Rows.Count - 1, R.Columns.Count)
  'Get the visible cells if any
  On Error GoTo ExitPoint
  Set R = R.SpecialCells(xlCellTypeVisible)
  'Get the last area
  Set A = R.Areas(R.Areas.Count)
  'Return the row of last cell in this area
  AutoFilterBottomRow = A(A.Count).Row
ExitPoint:
End Function

Function GetFilteredRangeTopRow() As Long
  Dim HeaderRow As Long, LastFilterRow As Long
  On Error GoTo NoFilterOnSheet
  With ActiveSheet
    HeaderRow = .AutoFilter.Range(1).Row
    LastFilterRow = .Range(Split(.AutoFilter.Range.Address, ":")(1)).Row
    GetFilteredRangeTopRow = .Range(.Rows(HeaderRow + 1), .Rows(Rows.Count)). _
                             SpecialCells(xlCellTypeVisible)(1).Row
    If GetFilteredRangeTopRow = LastFilterRow + 1 Then GetFilteredRangeTopRow = 0
  End With
NoFilterOnSheet:
End Function

Function GetFilteredRangeBottomRow() As Long
  Dim HeaderRow As Long, LastFilterRow As Long
  On Error GoTo NoFilterOnSheet
  With ActiveSheet
    HeaderRow = .AutoFilter.Range(1).Row
    LastFilterRow = .Range(Split(.AutoFilter.Range.Offset(1).Address, ":")(1)).Row
    GetFilteredRangeBottomRow = .Range(.Rows(HeaderRow + 1), .Rows(LastFilterRow + 1)). _
                                Find(What:="*", After:=.Rows(LastFilterRow + 1).Cells(1), _
                                SearchOrder:=xlRows, SearchDirection:=xlPrevious, LookIn:=xlFormulas).Row
    If GetFilteredRangeBottomRow = LastFilterRow + 1 Then GetFilteredRangeBottomRow = 0
  End With
NoFilterOnSheet:
End Function

jeeped say:

... Could you please tell me is there ios another way to allow hyperlink take the symbol # ...


The biggest problem I see here is the use of the # symbol in the URL or Uniform Resource Locator. The # symbol is a reserved character in a URL and is used to note a location bookmarked or otherwise named within the document pointed to by the remainder of the URL.

Reserved characters are used for special purposes. There are similar restrictions to what can be used when creating a named range reference in Excel, a field name in Access or even a file name in Explorer. You problem appears to be that the restrictions on a file name and the restrictions on a URL are not consistent, but they were never designed to be 100% compatible.

As an example of the proper use of the # symbol within a URL, consider the following like that links the provided hyperlink with the Syntax section of the Uniform Resource Locator document on Wikipedia.

Uniform Resource Locator - Syntax

The only possible work-around I can think of to this reserved character restriction might be to use VBA to directly shell execute the file as opposed to hyperlinking to it. Details on the exact nature of your problem are a little scarce, but perhaps that will point you in the right direction. Failing that, you may want to consider renaming the files in a manner that satisfies the restrictions on both file names and URLs.
