Hello Dan,

It's a good idea to delete temporary Internet files periodically to free hard disk space.

  1. Open Internet Explorer by clicking the Start button . In the search box, type Internet Explorer, and then, in the list of results, click Internet Explorer.            

  2. Click the Tools button, and then click Internet Options.

  3. Click the General tab, and then, under Browsing history, click Delete. 

  4. In the Delete Browsing History dialog box, select the check boxes for the information you want to delete, click Delete, and then click OK.

Once you delete the temporary files try to access Internet Explorer and check if you receive any popups.

You can also consider blocking pop ups in Internet Explorer 11.

Even if you turn pop-up blocker on, Internet Explorer won't block pop-ups from sites that are in the Local intranet or Trusted sites security zones. If you want to block all pop-ups‚Äîincluding pop-ups on Local intranet or Trusted sites‚Äîyou'll have to change the blocking level.

  1. Open Internet Explorer by clicking the  Internet Explorer icon on the taskbar.

  2. Click the Tools button , and then click Internet options.

  3. On the Privacy tab, under Pop-up Blocker, click Settings.

  4. In the Pop-up Blocker settings dialog box, under Blocking level, set the blocking level to High: Block all pop-ups (Ctrl+Alt to override).

  5. Click Close, and then click OK.

Good.  Let's look at your example again in that context.  Here's what I would do for a start with it.

First examine the File path implied by the File: URL

file:///C:/Users/Dan/AppData/Local/Microsoft/Windows/Temporary%2520Internet%2520Files/%257BF56870BF-42ED-44A9-B8F3-6BA00AD400B0%257D/%257B0B6B648B-ED27-4AA1-888C-731671EEA43B%257D.html

That looks like it's in the TIF, so it's probably the same as your  shell:cache.   So start from there.   E.g. enter that in the File Explorer address bar and open the TIF.  Ignore the fact that it opens with the Special Folder View.  Then overtype that address with cmd.exe and press Enter.   Result should be a cmd window open at that path.  Oops.  This is IE11 so now I see it not opening there.  It's opening a sibling directory called INetCache.

Well, I got most of it right.   So, a slight detour from my planned route.   CD ..  and the dir/ad   Oh Ho.  It's a Junction!  Who knew?  So I was right.   Let's go back to  INetCache again and continue where I had planned.  CD  InetCache  and then  dir/ad

FWIW I don't see any strange directories like the one that you had.   It's posssible that there are still some permissions issues so you should probably fire up an Administrator Command Prompt  and then navigate to the TIF manually, e.g. with

cd %LocalAppData%\Microsoft\Windows\INetCache

and then repeat the  dir/ad 

FWIW I still don't see any strange directories that way.   Nor do I see any like that under Low.

The only other possibility that I can think of is that whatever it is, is hidden even from the Administrator.  Then perhaps using  icacls.exe  would give us a clue that way.

BTW notice that the directory name in your File URL has (escaped) opening and closing curly brackets encoded in hex (i.e., { = 0x7B and } = 0x7D) so you

dir/a/b/s   {*

works for me.
