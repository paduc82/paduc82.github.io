FIX: Microsoft security essentials wont start when computer starts - Microsoft Community

nappy_rat say:

FIX: Microsoft security essentials wont start when computer starts and having to manually start the service.

 

Hi everyone,

 

The short answer to the fix is below if you don’t want to read the whole thing!

 

I too have had this same problem on my custom built PC and have seen this issue on many other custom built machines and major manufacture PC's Dell, HP, Toshiba, E-machine etc. I had hoped that Microsoft would eventually come out with an update to address this problem as seeing that this is not a localized issue with one or two pre-built PC's which may or may not have had previous virus or mal-ware protection, but with a gambit of different custom/fresh installs and pre-built OEM systems from all major manufactures. But here is my story to inform you of my fix with some tech info that may not matter to you, of my system but wanted to include it to minimize any questions.

 

System info: custom build

Motherboard: Foxconn (N68S7AA-8EKRS2H)

CPU: Intel Core2 Quad CPU Q6600 @ 2.40GHz

Memory: DDR2 800 (3072MB RAM)

Page File: 772MB used, 4184MB available

Hard drives: NVIDIA RAID 0+1 149.06G (4x80GB Western Digital)

Optical drives: LITE-ON DVDRW LH-20A1P

Video:Dual NVIDIA GeForce 8800 GTS SLI

Sound: Realtek HD Audio on-board / Creative Sound Blaster Audigy

Power: Thermaltake Toughpower 750 Watt

OS: Windows XP Home Edition (5.1, Build 2600) Service Pack 3 (2600.xpsp_sp3_gdr.100427-1636)

(Clean install)

Security software: Microsoft Security Essentials Version: 1.0.1963.0

(Clean install no previous Security software)

Direct X Version: Direct X 9.0c (4.09.0000.0904)

 

OK, now that I got that out of the way, here is the symptoms I was having:

 

I would start my system and MSE (Microsoft Security Essentials) and MSC (Microsoft Security Center) would inform me that MSE didn't start. And after waiting for a few seconds I would have to manually start the service by opening MSE and clicking on the start button provided. This some times doesn’t occur and MSE starts normally, but for the most part it will never start on its own automatically. Once its started everything else in MSE works as it should, and both MCE and MSC report it is working properly.

 

I did a lot of reading on theses forums for a solution to no avail. Most where solutions of making sure previous Anti-Virus and Anti-Malware software was not still present on the system. Which in my case was not a problem since this was a fresh OS (Fully Updated) and MSE (Fully Updated) install. So I started to poke around to see what I could find.

 

I started with services: start>control panel>Performance and Maintenance>Administrative Tools>services. I looked for three settings:

 

Cryptographic Services: set to automatic

Microsoft Antimalware Service: set to automatic

Security Center: set to automatic

 

It was also mentioned in the forums to look at Windows Installer and it was recommended to set to automatic and I did that but it had no effect so I set it back to manual. Plus I really don’t believe this would have done anything to fix the problem anyway and in my opinion posses a security risk. The first three setting were already set to automatic in my system so they were not the problem.

 

I then went to the log: start>control panel>Performance and Maintenance>Administrative Tools>Event Viewer. And found this under system:

Error: Service Control Manager

I double clicked on it to view the error event and it said: “The Microsoft Antimalware Service service terminated with the following error:

The binding handle is invalid.”

 

So I copied and pasted that error tho the forums and looked for a solution. Nothing substantial.

But I did read something that peaked my interest about “software racing”. This is where multiple software are trying to start up and can be conflicting with each other. Since this system has very little in the area of software I was intrigued as to what could be causing the conflict. I really dint know how to check but I assumed the likely culprit was a conflict with MCS and MSE. I really didn’t know how to resolve this kinda issue so I started stumbling around my system and stumbled on to a gem of a fix in less that 2 minutes and was the very first thing I did which resolved my issue with out having to change ANY other setting.

 

SHORT AWSNSER:

 

Navigate to start>control panel>Performance and Maintenance>Administrative Tools>services

 

Find the Microsoft Antimalware Service.

 

Double click it.

 

Click on the “Recovery” tab.

 

You will see three drop down menus for:

First failure

Second Failure

Subsequent Failures

 

Make sure that ALL THREE are set to “Restart the Service”

 

Click OK and restart.

 

For me, the Subsequent Failures field was set to “Take No Action” which could account for me having to manually start the service and it working fine afterwords and there being a possible “software racing” issue.

I wanted to test this so I had restarted my system many times and even created a batch file to auto start some programs i.e. iexplorer and such to make worse a possible “software racing” issue. I never had the problem come back. And MSE and MSC work as intended.

 

I hope this will fix a lot of other peoples MSE not starting properly issues and I will start implementing this fix to every system that I come across with this issue. Please keep in mind that this has worked properly for me and may not solve your issue, but if your symptoms sound close to what I have described I would suggest trying the fix to see how it worked for you and then come back and leave a comment on your findings.

 

Good Luck!

 

 

 

dr. strangelove say:

None of this should need to be done for MSE to start properly.

The verifications below should be done.  If they are, MSE should start properly:

Verify that you have removed/uninstalled all other real-time antimalware products that were ever installed on your PC, including free/trial products that were installed when the PC was purchased (Norton, Symantec, McAfee, TM, AVG, Avast, Avira, Malwarebytes (paid), etc). 

Then, use the cleanup tools here: List of anti-malware program cleanup/uninstall tools If yours is not listed, go to the manufacturer and get it. 

 

Verify that your PC clock is correctly set (date, time, time zone). If not, correct it.

Verify that Windows Defender is uninstalled (if XP), or Disabled (if Vista/W7).

Verify that you are up-to-date with SERVICE PACKS and updates (including .NET and IE). Yes, it can take some time to do this.

Verify that the Windows Firewall is on, and set to defaults. 

Verify that you are using version 2.0.657.0 of MSE (as of 2010-12-16).

Remove Zone Alarm if you have it and use the Windows Firewall.

Verify that you are not using a proxy server.

Verify that you have “install updates automatically” selected in Windows Update (even though the MSE definition updates are optional).

In IE, reset all security zones to default level.

 

Then, restart your PC.

 

You may need to remove and reinstall MSE as well. If so, you can use the procedure(s) here to uninstall MSE:  Uninstalling MSE. 

Then, use IE or Firefox, and go to www.microsoft.com/security_essentials and click DOWNLOAD.  Do not navigate and download via TOOLBARS.

 

To improve performance (and MSE's startup icon showing when you restart your PC), do a disk cleanup and defrag, and then perform a full scan with MSE (this can take several hours).

 

Also, examine the necessity of the number of startup programs you have. 

To examine your startup programs, go to START, RUN (or START, SEARCH) and enter MSCONFIG. 

Then, use the START UP tab to control the start up programs. Then, restart your PC.
