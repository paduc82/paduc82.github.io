mike h.. say:

Hi,

You said "This is what I am trying to do. ‘Copies the row from A2:AH across’"

Thats what this macro I gave you does. It will find the last used column in row 2 and copy that range to the first empty row; based upon column A, in sheet Number1

Dim LastRow1 As Long, LastCol2 As Long
Sheets("Number2").Activate
LastCol2 = Sheets("Number2").Cells(2, Columns.Count).End(xlToLeft).Column
LastRow1 = Sheets("Number1").Cells(Cells.Rows.Count, "A").End(xlUp).Row + 1
Sheets("Number2").Range(Cells(2, 1), Cells(2, LastCol2)).Copy _
Destination:=Sheets("Number1").Range("A" & LastRow1)

First to get some things out of the way just for the sake of clarity.

a) <email accounts> and <data files> are two entirely separate things. The Outlook NAV Pane reflects all the data files configured in the profile. the "default" email account - has no bearing on this.

b) generally, the "default data file" would be the first data file shown in the Nav Pane. The default data file is set via <File --> Account Settings --> Data Files tab). The default data file is the one that is expanded automatically when Outlook is opened regardless of it's location in the Nav Pane

c) to change the order in the Nav Pane - click the left mouse button on the <Root> of the data file you wish to resequence - keep it pressed down while dragging and move it to where you want. Once you have highlighted the <Root> folder and started to move it - you will see a horizontal black line appear as shown below. Move it to either above or below any other data file you want to achieve the desired sequence

As for your AddressBook question and which contact will display by default when it's opened - done by

1) Open the AddressBook from the RIbbon

2) from the AB menu bar --> Tools --> Options

3) select whatever contact folder you want to open by default as shown below

A little more information for you:

  • Suzanne is correct as far as the point that creating such a system may require some sort of programming. However, it doesn't have to be macro programming, which some companies ban because of the possibility of macro viruses. Recent versions of Office can use a different means of programming, using JavaScript. An introduction begins at https://docs.microsoft.com/en-us/office/dev/add-ins/reference/javascript-api-for-office and continues through the links there. In fact, the JavaScript system is the only way to get a custom pane like the one you proposed; a UserForm looks instead like a custom dialog box.
    • An advantage of the JavaScript arrangement is that it requires a cryptographic certificate in order to run, and that should make the insertion of malicious code very difficult or impossible.

  • Any data records you might store in Excel can just as well be stored in a table in a Word document, which mail merge can recognize as a data source. In that way, you could have your system run on Word alone.
