It arises from tiny numbers which cause "underflow". These are numbers smaller than 2.2250738585072E-308.  Excel "should" (according to this link) display underflow as 0. It appears that in this version it returns #NUM!.

The below Power Query recreates the problem and shows how rounding can eliminate the problem.

let
    Source = "4,94066E-324",
    #"Imported Text" = Lines.FromText(Source),
    #"Converted to Table" = Table.FromList(#"Imported Text", Splitter.SplitByNothing(), null, null, ExtraValues.Error),
    #"Renamed Columns" = Table.RenameColumns(#"Converted to Table",{{"Column1", "Value"}}),
    #"Changed Type" = Table.TransformColumnTypes(#"Renamed Columns",{{"Value", type number}}),
    #"Duplicated Column" = Table.DuplicateColumn(#"Changed Type", "Value", "Value - Copy"),
    #"Renamed Columns1" = Table.RenameColumns(#"Duplicated Column",{{"Value - Copy", "Value rounded"}}),
    #"Rounded Off" = Table.TransformColumns(#"Renamed Columns1",{{"Value rounded", each Number.Round(_, 10000), type number}})
in
