Your formula has some errors:

(a) syntax - too many parentheses

=IF(A26=B26,"1",)(IF(A26=B27,"2",)(IF(A26=B28,3,)

should be

=IF(A26=B26,"1",IF(A26=B27,"2",IF(A26=B28,3,....)

(b) no need for quotes

=IF(A26=B26,1,IF(A26=B27,2,IF(A26=B28,3,....

(c) I think this last 3 needs to be 4

=IF(A26=B26,"1",)(IF(A26=B27,"2",)(IF(A26=B28,3,)(IF(A26=B29,3,)(IF(A26=B30,3,)

see my screenshot.

I try very hard to avoid nesting IF to more than 3 levels. The screenshot shows some alternatives. The first two need you to add a column next to the dates in B26:B55. Then we can use either

=INDEX(C26:C55,DAY(TODAY()))    This could be =INDEX(C26:C55,DAY(A26))

or

=VLOOKUP(TODAY(),B25:C55,2)  which could be =VLOOKUP(A26,B25:C55,2)

The third one takes the day of the current date and subtracts how many Sundays there have been since the first of the month and today. This makes the table B26:B55 unnecessary and will work next month

=DAY(TODAY())-SUMPRODUCT(--(WEEKDAY(ROW(INDIRECT(EOMONTH(TODAY(),-1)+1&":"&TODAY())),2)=7))

Sub PivotTable_SelectMultipleItems1()
'Nov 16, 2015
Const sht$ = "Criteria" '<< sheet name / multiple items in column A (A1-A4000)
Const sV$ = "ID_Codes" '<< field name

Const sMax As Long = 4000 '<< max number of selected items
Dim N As Long, x As Long
Dim sStr As String
ActiveSheet.PivotTables(1).PivotFields(sV).ClearAllFilters
sStr = InputBox("select number of items", "Select a number from 1 up to " & sMax, sMax)
If sStr = "" Then Exit Sub
N = Val(sStr)
If N > sMax Or N < 1 Then
MsgBox "wrong, try again"
Exit Sub
End If
Dim t1 As Double, t2 As Double
t1 = Now
Application.ScreenUpdating = False
ActiveSheet.PivotTables(1).ManualUpdate = True
For Each pi In ActiveSheet.PivotTables(1).PivotFields(sV).PivotItems
x = WorksheetFunction.CountIf(Sheets(sht).Range("A1:A" & N), pi.Value) '<< items in column A
If x = 0 Then pi.Visible = False
Next

ActiveSheet.PivotTables(1).ManualUpdate = False
Application.ScreenUpdating = True
t2 = Now

MsgBox Format(t2 - t1, "hh.mm.ss")

End Sub

gillesgravier say:

Going from a dual CPU to a single CPU doesn't mean going backwards... depending on what CPU it is in each case.

 

Take the photo industry.

 

Canon comes out with a processor : DIGIC... then does a dual DIGIC...

Then they come out with a DIGIC 2... which is faster than the dual DIGIC... then they do a dual DIGIC 2...

And so on.

 

Today they are at dual DIGIC 4 systems. Next rev will be single DIGIC 5.

 

So... depending on what processor is in each phone, not a problem to go from dual older-generation CPUs to single new generation on...

cpitchford say:

I aggree, however I'm not sure these devices are frog leaping to that extent.

 

In the case of the N95, the dual core refers to core CPU and an additional DSP and GPU

 

N95: OMAP2420 - 330 MHz ARM1136 + 220 MHz C55x DSP + PowerVR MBX GPU

 

The N97, however,is of the same architecture (ARMv6) but with a higher clock rate (434MHz) It isn't a new generation chip per se. It's just running faster.. and without the GPU.. 

 

The N96 is a bit different.. Apparently the difference between the N95 v N96:

 

CPU: N96 has dual ARM9 264MHz with no floating point instructions (N95 has dual ARM11 332Mhz with vector floating point or graphics accelerator)

 

So, the N97 has the speed of the ARM11 from the N95, but all of the same missing features as the N96.. more or less.. 

 

This makes the prospect of a dual cpu s60 5th all the more intriguing
