You can do this with a macro (which is probably simplest) or you can bookmark the Bill of Lading number with the bookmark name (say) Number and insert a REF field formatted with your bar code font e.g. *{ Ref Number \*charformat}*

For the macro, at its very simplest, the following will insert the number with the barcode on the next line.

Dim oRng As Range
Dim sText As String
Dim sFont As String
Dim iSize As Long
Set oRng = Selection.Range
sText = InputBox("Enter Bill of lading number")
With oRng
    sFont = .Font.Name
    iSize = .Font.Size
    .Text = sText
    .Collapse wdCollapseEnd
    .Text = vbCr & "*" & Trim(sText) & "*"
    .Start = .Paragraphs(2).Range.Start
    .Font.Name = "Free 3 of 9" 'barcode font name
    .Font.Size = 80
    .Collapse wdCollapseEnd
    .Text = vbCr
    .Font.Name = sFont
    .Font.Size = iSize
    .Collapse wdCollapseEnd
    .Select
End With

bob jones aka: cybertaz say:

Glad it worked out :-) As for how it happened, the most likely reason is that at one time you accidentally switched to Notebook Layout View. Or it may even be that you accidentally created the new document as a NBL, then immediately switched to a different view as soon as it appeared. That's all it takes... even if you switch views immediately the document already has been tagged as a NBL document & just saving the document while in a different view does not remove that internal identifier.

The only way to avoid it happening is to not select that view in the first place. If you do select it in an existing document there is a prompt to confirm the conversion which allows you to either confirm, cancel, or create a new document. If the new document is created in NBL, just close it without saving, then create another new document using the New Blank Document command from the New button's list on the Standard Toolbar or from the Project Gallery.

To answer your additional questions...

The Tabs I referred to are Index Tabs on the upper right edge of a NBL document, as you would have in a physical notebook. They represent Sections in the document. In a new document (or a document having only 1 Section) the document starts off with 1 tab labeled Section 1 & there is a smaller tab having a plus sign that can be used to add additional Sections.

VBA is Visual Basic for Applications, which is a programming language that can be used to automate processes & create procedures that go beyond the routine features of a program. The individual blocks of code often are simply referred to as macros.

andreas killer say:
It is obvious that the server / windows locks that file. Maybe it is a similar issue as the "access denied" when you try to copy and rename a folder.

Windows 7 creates thumbs.db files while browsing network folders. So if you copy a folder on a network drive, Windows 7 creates the thumbs.db file in the background and locks that folder until the creation is done. Sounds fast, but on networks drives it can take a while. If it tried to delete the copied folder immediately, I got always an access denied. But when I browse the folder it was empty!

In fact the folder isn't empty, there is a thumbs.db in that folder and this file is the reason.

Follow this steps to disable the thumbs.db on network folders:


1. Open your registry editor.
2. Navigate to HKEY_LOCAL_MACHINE\SOFTWARE\Policies\Microsoft\Windows\
3. If key â€œExplorerâ€ doesnâ€™t exists create a key named â€œExplorerâ€
4. In key â€œExplorerâ€ create a new DWORD 32 bit value named â€œDisableThumbsDBOnNetworkFoldersâ€ with value = 1
5. Reboot your system

If that doesn't work, try to open the file readonly:
  Workbooks.Open "Y:\InvoiceLog\Exports\apptexport.csv", ReadOnly:=True

If you need to write into that file, check if the file is closed before you open the file.
