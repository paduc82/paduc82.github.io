pete_uk say:

As you have found, you can't use the wildcard characters in all situations. The comparison operator, =, is one such situation. You can achieve what you want to do using this:

 

=IF(ISNUMBER(SEARCH("east",A1)),"east","not east")

 

If you want a solution using the wildcard character, you could use this:

 

=IF(COUNTIF(A1,"*east*")>0,"east","not east")

 

Hope this helps.

 

Pete

pete_uk say:

No, there is no CONTAINS function in Excel, nor INSTRING, although this is available as Instr in VBA. You either use SEARCH to see if the text exists within the other string, or COUNTIF using asterisks around the text.

Private Sub workbook_sheetactivate(ByVal sht As Object)
Dim sh As Worksheet

ActiveSheet.EnableCalculation = True

For Each sh In Sheets
  If sh.Name = ActiveSheet.Name Then
    ' nothing
  Else
    Worksheets(sh.Name).EnableCalculation = False
  End If
Next sh


End Sub

Dan

andreas killer say:
The reason is that you use LinkedCell and/or ListFillRange within the ComboBox.

If you change the calculation state from False to True, the sheet recalculates and all linked controls are updated.

Instead of using the LinkedCell property, you can write the value into the sheet within the ComboBox change event, resp. write the value into the ComboBox within the Worksheet_Change event.

And you can load the items into the ComboBox within Workbook_Open, instead of using ListFillRange.

manuela.a say:
Hello,

This is happening, because this is not a user mailbox, but a shared mailbox. Once you are granted with full access permissions for it, it will appear automatically in your Outlook (through automapping). You have 2 possibilities:
1. Remove your permissions to the mailbox if you no longer need it.
2. Disable automapping in Powershell and remove the mailbox from Outlook: https://support.microsoft.com/en-us/help/264650...
Both of those actions have to be done from the O365 global admin in your organization.
Hope that answers the question.

Let me know if something is not clear.
