Might be able to create a symbolic link to a local (unused) folder on a local machine and it MIGHT (*might*) be able to fool the Samsung application.

symbolic link: http://technet.microsoft.com/en-us/library/cc753194.aspx

so...

Map a network share directory you would use this command:


net use \\Server\ShareName


Then... if you want to access the network share through a location on your local C: drive, you'll want to set up a symbolic link:

mklink /d  C:\Folder\NewShareName \\Server\ShareName
 
Now when you navigate to C:\Folder\NewShareName you'll see the contents of \\Server\ShareName ...
 
So MAYBE you could point Samsung to C:\Folder\NewShareName
(*NewShareName should not exist in C:\Folder before you run the "mklink" command.)
