jay freedman say:

That message should not appear unless the macro is doing something that modifies the template (not just the documents that the macro worked on). The most likely way that could happen is if the macro refers to "ThisDocument" when it should refer to "ActiveDocument" -- the keyword ThisDocument always returns the template or document that contains the running macro code.

If all else fails, you could try putting this statement at the end of the macro:

     ThisDocument.Saved = True

which tells Word that the template is not "dirty" so no save is needed.

clmickle say:

Contacts weren't restored when PST file imported

I was about to format my HD and so I exported an Outlook PST file for a backup.  I also made a copy of the entire "my computer."  I reinstalled windows, imported the PST file, and now my contacts aren't here!!!  Please help!  
roady [mvp] say:

Exporting and importing your pst-file isn't the way to backup and restore your pst-file. You should take the original pst-file and copy it to a safe location. To restore it, copy it back and reconnect it via; File-> Open-> Open Outlook Data File...

If you do not have the original pst-file and reconnecting to your exported pst-file doesn't show you Contacts in any of the Contacts folder, then you did not include them in your export.

For more info and future reference, see; http://www.howto-outlook.com/howto/backupandrestore.htm
