Search Options: Search Box vs Search Icon and Search Disabled - Microsoft Community

mauricedurbin say:

Search Options: Search Box vs Search Icon and Search Disabled

The search icon is universally recognised and accepted; why do we need the search box option? At best it is redundant and at worst it is creating a number of issues.
I tested search from the desktop and from the start menu.

Desktop.
Search Icon option functions correctly with the task bar in any position, with large and small buttons and with task bar auto-hide enabled or not.
Search Box option only functions with the task bar at the bottom of the screen and large buttons. In other positions or with small buttons it is replaced automatically by the Search Icon, overriding user choice.
Search Disabled option. This disables search from the task bar, it does not affect Search from the Start Menu. This isn't clear from its description which suggests that search will be totally disabled; several people have asked to have Search removed from the task bar - this is how you do it!

This is a useful function for those who only want to search from the Start Menu.

Start Menu.
Search Icon option selected.The Search Icon function does not work. The Search Box function is provided regardless of search option settings. It works but there are issues:
1. With the task bar in top, right or left positions the Search Icon is visible in the task bar but does not function.
2. With the task bar at the bottom of the screen small task bar buttons are obscured by the Search Box.
3. With the task bar at the top or left the Search Box obscures ‚ÄòAll Apps‚Äô. 
4. Inconsistency, the user has chosen to use the Search Icon but user choice is overridden.
Search Box option selected.
The Search Box function works with the task bar at the bottom of the screen and large task bar buttons selected. In other cases there are issues.
1. With the task bar at the bottom of the screen small task bar buttons are obscured by the Search Box.
2. With the task bar at the top or left the Search Box also obscures ‚ÄòAll Apps‚Äô.
Search Disabled option selected.
This is the one case where the Start Menu Search Box has a useful function, but it still has the same issues with task bar buttons and ‚ÄòAll Apps‚Äô being obscured in some cases. It would be better to place a Search Icon button alongside the Power button. It would have the same function without the issues.

Recommendations.
1. Drop the Search Box and go with the Search Icon throughout. 
2. Replace the 3 Search Options with a single check box ‚ÄòEnable/Disable Search from Task Bar‚Äô.
3. Enable the task bar search icon to function on the Start Menu if Task Bar Search is Enabled.
4. Place a Search Icon on the Start Menu next to the Power Button if Task Bar Search is Disabled.

ossiemac say:

This problem has been around for as long as I can remember. The work around is to copy the array to another range without hidden rows etc and then copy that range back to the required final destination.

Additional note. When dimensioning variables on one line like the following:-

Dim r1, r2 As Range      '(r1 becomes a variant; not a range.)

If you want it to be specifically a range then dimension like the following:-

Dim r1 As Range, r2 As Range

Here is the work around code example.

Sub Test()
    Dim r1 As Range, r2 As Range
    Dim v, v2 As Variant
   
    Set r1 = Sheet2.Range("Range_1")
    Set r2 = Sheet2.Range("Range_2")    'Range_2 should be a single cell
   
    v = r1.Value
   
    'Copy array to a range without hidden cells filtering
    'Destination is single cell resized to match array size.
    Sheet3.Cells(1, 1).Resize(UBound(v, 1), UBound(v, 2)).Value = v
   
    'Copy the range back to the required range.
    'r2 is a single cell (first cell) of the destination.
    Sheet3.Cells(1, 1).Resize(UBound(v, 1), UBound(v, 2)).Copy Destination:=r2
   
    'Cleanup temp range
    Sheet3.Cells(1, 1).Resize(UBound(v, 1), UBound(v, 2)).ClearContents
    
 End Sub
