uninstall any "other" antivirus, and also run its "removal" program, that can

by entering the Registry.     Select "start" and type "regedit". Select "Run

as Administrator".  On the Registry Editor screen, navigate through the

following branch by selecting the > to the left of each item.

HKEY_Local_Machine\Software\Policies\Microsoft and select the folder

Windows Defender. The only key that should be in this folder is "Default".

Any others, are changes that have been made to Windows Defender.

your Registry from getting cluttered. When you reach Hkey_Local_Machine,

Hello,

I assume you have a clustered bar chart with horizontal bars. To show text instead of the number labels on the X axis, you need to add an additional series with the values 1 to 5. 

Add this series to the chart (copy the cells with the numbers 1 to 5, select the chart, use paste special and hit OK). Format the new series and change the chart type to XY Scatter.

Then edit the source data for that series and select the Y axis values, which should all be zero. Now the data points are aligned on the x axis. Add data labels to this series, below the data markers. Edit each data label, click the Formula bar, hit the = sign and click the cell with the desired text.

The macro named "Sort_Description" is completed within 10-15 seconds.  The other one ("Sort_Date") takes about 45-60 seconds...

Can I do anything to speed things up?

Thanks !
j.e. mcgimpsey say: XL macros often do run slower on Macs, but you can certainly speed these up.

First, there's no reason to sort the range twice in the first macro. Instead, use Key1 and Key2 to do the sort in one step.

Second, it's almost always faster to operate on the Range Object directly, rather than the selection. 

Third, you can use the With ... End With  construction to eliminate multiple evaluations of the Range Object. If you're not familiar with the construct, the "." in ".Sort" and ".Cells" refers to the object in the With statement, so here .Sort is equivalent to Range("A5:F10000").Sort, but in this case the range doesn't have to be reevaluated for .Cells(2, 1) and .Cells(2, 2) - the reference is stored in a temporary variable.

Fourth, it appears from your ranges that there is indeed a header. You'll probably save several nanoseconds by changing xlGuess to xlYes (don't blink...).

So this should be faster than your original:

    Public Sub Sort_Date_Faster()
        With Range("A5:F10000")
            .Sort Key1:=.Cells(2, 1), Order1:=xlAscending, _
                    Key2:=.Cells(2, 2), Order2:=xlAscending, _
                    Header:=xlYes, OrderCustom:=1, _
                    MatchCase:=False, Orientation:=xlTopToBottom
        End With
    End Sub

On my machine (MacBook Pro), a sort of 10000 rows of random numbers took 0.32 seconds.
