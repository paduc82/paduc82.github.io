suzanne s. barnhill say:

There are several solutions in this kind of situation. One is to edit the text so that the URL will all fit on the same line with part of the text. Unfortunately in this case the URL is so long that it doesn't even fit on one line. So it will have to be broken. In this case, you can just insert a space. It's generally suggested that you insert the space before a slash or underline so it is obvious (since the slash or underline will start a line) that the URL has been broken.

If there is a chance that further editing will cause the URL to fall in a different position, you can insert several possible breaks that will not result in a visible space (somewhat like a conditional hyphen that is displayed only when a word is broken at the end of a line). For this purpose you can use either U200C, Zero Width Non-Joiner (which Word calls "No-Width Optional Break"), or U200B, Zero Width Space. The former has the advantage that you can insert it from the Special Characters tab of Word's Symbol dialog. When one of those spaces falls at the end of a line, the URL will break.

This solution will work even if the link must remain active because the editing you are doing is only to the display text. If you look at the Edit Hyperlink dialog, you'll see that the "Text to display" contains the spaces, but the "Address" remains the same. You can see the same if you press Alt+F9 to display the Hyperlink field code, which displays the unbroken URL.

Meeting Invites Added to Delegate Calendar

I'm experiencing an issue similar to the one found in this thread.

I have a User and a Delegate. User has two mailboxes: Visible and Personal. Anyone can send messages to User.Visible, which forwards to Delegate without keeping a copy in its mailbox. Delegate forwards important emails to User.Personal, as well as managing the calendar attached to User.Visible. On the User.Visible account, delegation is set up as described in the linked thread.

So: User receives emails at User.Personal, and uses their calendar at User.Visible.

When anyone sends a calendar invite to User.Visible, Delegate receives the invite in their inbox and clicks "Accept." The requester receives an email saying that Delegate accepted the invite, and the event is added to Delegate's calendar but NOT User.Visible.

I have deleted & re-added Delegate to User.Visible in PowerShell, the web GUI, and Outlook. None of the changes have had any effect. What am I doing wrong?

a. user say:

I think I figured out what was going on!

When the "Keep a copy..." option is turned off, Outlook acts as though the user to whom emails are being forwarded was the original recipient of the calendar invite. When I turned this option on, User.Visible's calendar acted as expected and created an event.

The forwarded email still creates a duplicate event on the forwardee's calendar, but at least now the event shows up on User's calendar at all.
