The subform should not be linked to the main form, i.e. the Link Master Fields and Link Child Fields properties should be blank.

Create the following event procedure for the subform:

 

Private Sub Form_Current()
    If Me.NewRecord Then
         DoCmd.GoToRecord acDataForm, Me.Parent.Name, acNewRec
    Else
        Me.Parent.Bookmark = Me.Bookmark
    End If
End Sub

 

This will keep the main form synchronized with the subform

How to add conditional text based off drop down menu selection - Microsoft Community

b_b_ecolab say:

How to add conditional text based off drop down menu selection

I have added a drop down list content control to my word document with a tag name of LOTO. Based off the selection, I want conditional text to come up. If they answer Yes from the dropdown menu, I would like a text field that says "Procedure Number" to come up with an input text field to allow the user to enter procedure number.  Thank you in advance!

jay freedman say:

I'll assume that the dropdown list contains choices "Yes" and "No", and that the "text field" for the procedure number is a Rich Text or Plain Text content control rather than a legacy formfield.

Select the paragraph containing the text content control and its label (including the paragraph mark), and apply a bookmark. In the code below I've used the bookmark name "procname", but you can use any name that meets the syntax rules. While the whole paragraph is still selected, apply Hidden font formatting to it.

Open the macro editor and double-click the ThisDocument icon under the document's name in the Project pane on the left. Copy the following macro and paste it into the code pane on the right.

Private Sub Document_ContentControlOnExit(ByVal ContentControl As ContentControl, Cancel As Boolean)
    If ContentControl.Tag = "LOTO" Then
        If ContentControl.Range.Text = "Yes" Then
            ActiveDocument.Bookmarks("procnum").Range.Font.Hidden = False
        ElseIf ContentControl.Range.Text = "No" Then
            ActiveDocument.Bookmarks("procnum").Range.Font.Hidden = True
        End If
    End If
End Sub

Close the macro editor and save the document.

When the user first opens the document, the Procedure Number paragraph won't be visible. If they select "Yes" in the dropdown and exit the control, the paragraph will appear; if they go back and change the choice to "No" and exit the control, the paragraph will disappear. Choosing the placeholder text ("Click here...") won't change the visibility either way. Whatever the state of the controls at the time the document was last saved, the document will reopen in the same state.

Next [if running a version of OS X other than El Capitan] run Disk Utility to Repair disk permissions on your HD followed by a Restart of your Mac. If running El Capitan, just Restart.

Do a fresh launch of Excel & change the Preference. Quit the program then launch it again to see if the preference setting sticks. If not...

User settings & preferences typically are not stored in cache files. For your version of Excel the name of the file is com.microsoft.Excel.plist which is stored in the Preferences folder of the Library for your User Account. To reset the Preferences;

  1. Quit Excel
  2. Go to ~:Home:Library:Preferences
  3. Drag com.microsoft.Excel.plist to the Trash (but don't empty the Trash)
  4. Launch Excel & change the Preferences
  5. Quit & restart the program to check the status of the Preferences

If all is well you can empty the Trash. Otherwise, you can do the same or replace the new instance of the file with the old one if you have reason to do so.
