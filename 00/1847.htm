One nice feature of the UEFI disk partitioning scheme is that you can totally delete and replace the System/EFI partition. I have found that when repairing it fails, replacing the partition will work.

If you boot with install media, from the first screen use SHIFT-F10 to get a Command Prompt. Then you can run diskpart to delete and recreate the EFI partition, and format it FAT32.  You can use bootsect and bcdboot to recreate the System files on it.

bootsect /help and bcdboot /? will display the correct syntax for each. It helps to have a drive letter assigned to your new System/EFI partition.  I usually use S:.  In diskpart, with the new EFI partition selected: assign letter=S

You must know the drive letter currently in use for the Windows partition. Usually on a UEFI device it remains C:, but often shifts to D: or other on a Legacy BIOS device. Yours seems to be on D: at the moment. You will not need to use bootrec.exe.
