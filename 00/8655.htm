Is this a Win7 driver file, IE10 driver file, or a driver file for an Ad-On?

 

urlmon.dll is a fairly low-level system module.   It will not be the cause of the crash.  Regard it as a victim of its caller or of a caller's caller.   You would need to extract a Stack Back Trace from the crashing thread to see what those other modules are.

No.   One thing it does though is set Protected Mode: Off.   So you could experiment with Security Settings to turn that off and see if the symptom is still avoided.   In fact, if it is happening only while browsing a certain host that you want to continue using you could try adding it to your Trusted Sites zone, since by default that includes the same change.

If you have done a RIES you supposedly have eliminated add-ons.   Then that would suggest there is something else that is running in your Startup which is the cause.   Then you may try a safe mode boot to see if you can recreate the problem in that environment, which presumably would not be running as many things as you would be normally.

I still do not own a certified Windows to Go USB stick/flash drive, but I just created something similar on a normal USB 3 stick that I have and installed Build 14383 on it.  I am using it to post this reply.  It is a bit sluggish, because it is still updating hardware drivers, but seems to run quite well.

All I did was create an empty NTFS partition and mark it Active and then used DISM to Apply Index:4 of my Build 14383 ESD file to the USB drive.  I then ran a bcdboot command to set it up for booting Windows.  When I started my computer, I use F12 to choose a boot device.  For some reason, my UEFI computer allows me to boot an NTFS-formatted USB stick without enabling any Legacy mode settings, so it showed up as an available option in the UEFI boot menu.

Windows 10 installed and Activated without ever using an MSA (Microsoft Account) login during the install, so it clearly uses the device hardware from my computer, which has Digital Entitlement to run Windows 10 Pro, to base Activation on.  The next thing would be to test if it will work if moved to another PC, especially one without entitlement for Windows 10 Pro.  I expect that it would boot, but not show as Activated.
