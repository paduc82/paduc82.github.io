scottgem say:

Before you try to run this query, please review what Duane said. 

Second point your query states you are updating the Vehicles table, yet your SET clause refers to the Vehicle field in the Expenses table. Third point depends on what UNIT-M is. Is that the link fields that connect Vehicles to Expenses? And the final point is (as Mimmo said), What is the datatype of the Vehicle filed in Expenses. Your Update query is trying to set it to a String value and, from the error message, it doesn't appear to be a Text datatype.

Bottom line here is it appears you want apply an expense to a Vehicle. If so, the ONLY data about the Vehicle that you need in the Expense table is the VehicleID as a foreign key.

If you want to display the Year, Make & Model then you link the tables on the VehicleID and pull that data from the Vehicle table. 

So what we are saying is forget the Update query and make sure the design of your table structure is correct.

ken sheridan say: A much simpler method would be to use the FileDialog property of the Application object, as with this simple little function:

Public Function BrowseFile() As String

    Dim strFile As String
    With Application.FileDialog(1)
        .Title = "Select File"
        If .Show Then
            strFile = .SelectedItems(1)
        Else
            MsgBox "No file selected", vbInformation
            Exit Function
        End If
    End With
    
    BrowseFile = strFile
    
End Function
