john - project say:

John,

The error occurs because you have a convolution of two different Events, the Project.BeforeSave and the Application.ProjectBeforeSave. The first runs at Project level and has one parameter, while the second is at application level and has three parameters. You have specified the Project level Event but given it 3 parameters, so the compiler is confused and raises the error.

The Project level Event is relatively easy to implement but it won't do what I think you are attempting. The Application level Event will do what I think you are trying but is more complex to implement as it requires creation of a Class module in addition to the Private sub.

So, what exactly are you trying to do, and how involved do you want to get?

John

john - project say:

John,

In order to do what you want (i.e. cancel a save based on certain conditions), you will need to develop an application level event which as I said previously, is more complex to implement as it requires the creation of Class module. You can read about that in the VB help file under "Using Events with Application and Project objects".

However, if you are not quite so adventuresome, you could come close to what you want with the Project level BeforeClose event. The following code will implement your pseudo-code. It will save the file if the user elects to do so, but if the user elects to not save, he/she will still be presented with the built-in message asking if the file should be saved or not. Since a "no" answer still results in an additional entry by the user, it might be better to simply do the test and display the message with a vbInformation option and then let the built-in follow-on message determine the "save" or "not save" action.

John

Private Sub project_beforeclose(ByVal pj As MSProject.Project)
Dim t As Task
Dim FndFlg As Boolean
Dim ans As Long
For Each t In ActiveProject.Tasks
    If Not t Is Nothing Then
        If t.Summary = False Then
            If t.ActualStart <> "NA" And t.ActualStart > ActiveProject.StatusDate Then
                FndFlg = True
                'found at least one, no need to continue
                Exit For
            End If
        End If
    End If
Next t
If FndFlg = True Then
    ans = MsgBox("One or more tasks show an actual start earlier than the status date." & vbCr & vbCr & _
        "Do you still wish to save?", vbYesNo)
    If ans = vbYes Then FileSave
End If
End Sub

Hi Denny.ca,

Thanks for your reply.

The E-mail Notification option only exists in Task lists and not in the custom list.
You can post your idea to the SharePoint UserVoice to improve the product.

As a workaround, following are the steps to create a sending email Workflow in a custom list.
1. Download SharePoint Designer 2013 according to this link: SharePoint Designer 2013.
2. After you download and install SharePoint Designer 2013, open the application and click on Open Site.
Insert your site name and insert password to connect to the site: https://xxx.sharepoint.com

3. Click on the Lists and Libraries and select the custom list.

4. Click on the New to create a Workflow.

5. Insert your Workflow Name and select the Platform Type as SharePoint 2013 Workflow.
The Workflow Name will appear as a new column in your list after you finish publishing the Workflow

6. Insert “Email” in “(Start typing or use the insert group on the Ribbon.)” and press enter.
Press “Go to” in “Transition to stage” and press enter.
You will get Email these users and Go to a stage.

7. Click on the “these users” and pop up a window.
Click on book icon> Select Workflow Lookup for a User…> Add>>> Select Current Item in Data Source> Select your Assign to column name> OK. 

8. Edit the email and subject.

9. Click on the “a stage” and select End of Workflow.

10. Save the Workflow> Click on Workflow in Navigation> Select the Workflow and check all the section under Start Options. 

11. Save and Publish, Workflow creation finished.
Then you can test to assign task to one user to see if he can receive the email.
