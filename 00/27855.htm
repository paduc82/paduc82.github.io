Alright. This took me a long time, but I was finally able to fix it. 

SHORT VERSION: For some reason, when I went to outlook.com, there was a cHAT folder on the left side. I right-clicked on that folder, pressed Delete, then sync worked perfectly fine.  

 

LONG VERSION: If you really want to know how to solve this problem, or the short version didn't fix/describe your issue, here's some troubleshooting steps:

  1. Go to outlook.com.
  2. Click the settings gear icon in the top right corner.   
  3. Click on Options.   
  4. On the left side of the screen, choose the General tab, and select the Mobile devices category.   
  5. Select the non-syncing device in the list and press the Start logging icon (looks like a piece of paper with an arrow pointing to the right).   
  6. Select Yes in the resulting Device synchronization log. This lets you know that the resulting log will be emailed to your account.
  7. file should open it.
  8. If you are using multiple syncing devices (as stated previously), there may be multiple logs. In this case, for each log, try using Ctrl+F to search for the name or model of your non-syncing device. Start by using Ctrl+F with "model" or "friendlyname". Make sure you're looking at the right log for the non-syncing device.
  9. If you are unable to find the name or model of your non-syncing device in the log, use Ctrl+F and search for "X-FE-ClientIP". There should be an IP address listed on the same line (four numbers separated by a '.'). In your web browser, go to whatismyipaddress.com. Confirm that both the given IP addresses are the same. NOTE: This will only work if your non-syncing device is the only one connected to your local network (ex. if you also have a syncing desktop connected to the same network as a non-syncing laptop, it will likely have the same IP address as the laptop).
  10. In my case, while looking through the non-syncing log for indications of error, I found a folder that is supposed to sync with NULL characters in the ServerId field. It looked like this in Sublime Text 3                                                                                                                                                                                                                                                                                                                                                          and like this in the standard Microsoft Windows Notepad program.                                                                                                                                                
  11. As this seemed to reference a folder called cHAT that wasn't syncing, I simply deleted it from my Outlook account by going to outlook.com, finding the cHAT folder on the left side, right-clicking that folder, and selecting Delete. After I did this, mail was able to sync normally on my non-syncing device.                                                                  
  12. If your logs don't show a folder that is having issues, unfortunately this is where you'll have to pick up the troubleshooting. Maybe try using Ctrl+F and search for the word "error", or just look for something else in the log that seems out of place like the NULL character patterns shown above. Even though Microsoft Support has been failing us miserably with this issue, if you're able to contact their support team via chat or email and provide them the log file they might be able to figure out what the issue is. Hope this helps!
