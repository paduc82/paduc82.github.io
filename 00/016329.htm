First, no such thing as an unbound field. Fields are parts of a record. Forms don't have fields but controls that may or may not be bound to a field in a table. This is a subtle but important distinction.

I don't see an Update to VehicleOdometer. I see EmployeeID, DateAssigned, VehicleRemarks and VehicleTag.

But if you need to not run the Update query if the VehicleOdometer control is blank you can use an IF Statement before you run the query.

The easy solution will be to make the odometer reading a mandatory entry, thus eliminating the possibility of it being blank.

DoCmd.RunSQL "UPDATE tbl SET tbl.field1 = Null, tbl.field2 = Me.field2, tbl.field3 = (Iif Me.field3 Is Null, DLookup("field3","tbl","field4 = Me.field4"), Me.field3) WHERE tbl.field4 = '" & Me.field4 & "';"
