jeeped say:

Your value in A3 is not 2.48. It is 2.4768, but rounded in the display to a number format of a fixed 2 decimals. This makes the actual value in A4 29.9968 which is displayed as 30.00 due to a number format of a fixed 2 decimals. This is compounded in A6 by multiplying the A4 result by two, increasing the disparity between actual and displayed twofold. The difference is now enough that rounding the displayed value shows up as a penny difference.

Solution: Round off the calculated values that could potentially cause problems before increasing the errors through multiplication. e.g. A3 should read:

=ROUND(A2*A1,2)


Figure 1.0 - Excel's ROUND() Function

BTW, there is no reason to wrap your multiplication inside a SUM(). The formula works just fine as =A1*A2 and removing the SUM() 'wrapper' may alleviate confusion for others (e.g. What is being SUM()'ed here?)
