graham mayor say:

I was about to reply to this yesterday in the Answers forum, and we had a power cut for the rest of the afternoon, and when the power returned I couldn't find the thread, presumably because you had marked it as answered :(

You can automate this from Word using a macro e.g. as follows.

The macro assumes that the data is on Sheet1 of the workbook using columns A to C. The macro names the templates from Column A and uses the header and footer information in Columns B and C. It will produce as many templates as there are entries in column A.

Put the path of the worksheet where indicated.


I have duplicated this answer in the other forum.


Sub MakeTemplates()
Dim oDoc As Document
Dim oHeader As HeaderFooter
Dim oFooter As HeaderFooter
Dim xlApp As Object
Dim xlBook As Object
Dim LastRow As Long
Dim i As Long
Const strWorkbookname As String = "D:\My Documents\Test\HeaderFooter.xlsx"        'path of the Excel workbook

    On Error Resume Next
    Set xlApp = GetObject(, "Excel.Application")
    If Err Then
        Set xlApp = CreateObject("Excel.Application")
    End If
    On Error GoTo 0
    Set xlBook = xlApp.Workbooks.Open(Filename:=strWorkbookname)
    LastRow = xlBook.Sheets(1).Range("A" & xlBook.Sheets(1).Rows.Count).End(-4162).Row
    For i = 1 To LastRow
        Set oDoc = Documents.Add
        For Each oHeader In oDoc.Sections(1).Headers
            oHeader.Range = xlBook.Sheets(1).Range("B" & i)
        Next oHeader
        For Each oFooter In oDoc.Sections(1).Footers
            oFooter.Range = xlBook.Sheets(1).Range("B" & i)
        Next oFooter
        oDoc.SaveAs2 Options.DefaultFilePath(wdUserTemplatesPath) & "\" & xlBook.Sheets(1).Range("A" & i) & ".dotx", 14
        oDoc.Close 0
    Next i
    xlBook.Close 0
    xlApp.Quit
    Set xlApp = Nothing
    Set xlBook = Nothing
    Set oDoc = Nothing
End Sub

suzanne s. barnhill say:
Your screen shots appear to be corrupted, but the general answer is no; you can size cell width independently in various rows, but you can't change cell height in various columns. What you can do is create multiple shallow rows and then merge and split cells to create cells of different height. In some cases, it may suffice just to remove cell borders selectively; in other cases, you'll actually need to merge cells. For example, if you have 12 rows, then you can have a cell of any height from 1 to 12 rows anywhere in any column; you could have one column with three cells 4 rows high and another with four cells 3 rows high, or any other combination of cells within any column. You can also split cells within those rows, and you can merge a split portion with another cell above or below.

hansv mvp say:

A formula such as =IF(AF30="Stopped",G30,VLOOKUP($AD30,Data_D!$A$6:$G$2256,6,FALSE)) is a "standard" formula, there is no need to confirm it with Ctrl+Shift+Enter.

The formula =MAX(IF(B5:B30<0,B5:B30)) is an array formula because the first argument of IF is normally a single condition, but here there is a condition for each of the cells in B5:B30. Hence, IF(B5:B30<0,B5:B30) does not return a single value, but an array of values. Such formulas must be confirmed with Ctrl+Shift+Enter.
