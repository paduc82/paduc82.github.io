Having Preferences restored is a simple & painless procedure. There is no reason to be hesitant, especially in PowerPoint -- there are very few Preferences that might have to be reset :-)

Before you do, though, it would be best to first confirm that you have Office fully updated -- 14.3.8 is the current level. jay freedman say:

There are a number of ways to do this, but probably the simplest is to use the built-in Open dialog, which combines choosing the file and opening it into one operation. Try this instead of your call to Documents.Open:

           

    With Dialogs(wdDialogFileOpen)
        If .Show = -1 Then

             sFname = oChanges.Name

        End If
    End With

Another way is to use the Application.FileDialog object to get the name. Here's some sample code:

Sub x()
    Dim sFname As String
    With Application.FileDialog(msoFileDialogOpen)
        .AllowMultiSelect = False
        .InitialFileName = Options.DefaultFilePath(wdDocumentsPath) & "\*.*"
        .Show
        If .SelectedItems.Count Then sFname = .SelectedItems(1)
    End With
    If Len(sFname) = 0 Then
        ' user canceled the dialog
        Exit Sub
    End If

    ' continue with rest of macro
    MsgBox sFname
End Sub

You can set some additional properties of the FileDialog before calling .Show. For instance, there's a .FilterIndex property that sets the default in the file-type dropdown; if you want to limit the display to only *.docx files, set .FilterIndex = 3.

ken sheridan say: Strictly speaking there is no value of date/time data type which represents a date of 30 December 1899, i.e. a value to a precision of 24 hours.  Zero represents the date/time of 30 December 1899 00:00:00, i.e. the point of time at midnight at the start of the day.  This because the date/time data type is implemented as a 64 bit floating point number.  It's really just the measure of duration in days from 30 December 1899 00:00:00.

The Nz function can cater for the sort of situations you describe in which Null is not accepted as an argument, but is also important in arithmetical expressions in which one of the values might be Null.  In such expressions because Null is not a value  it propagates in the way you describe, so the result will always be Null.  The Nz function returns a zero by default in place of a Null, but can be used to return any value of any data type by entering the value as the second argument of the function., e.g. Nz(MyDate,Date()) would return the value of MyDate if it has one, or the current date it it's Null.
