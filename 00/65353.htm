Since the beginning of the PC industry - if not earlier - computers have been designed on the 'principle of least privileges' (you can Google that) which is to say that each user has the least amount of privileges necessary for its role. That's why you cannot give yourself the ability to delete any file you think should be deleted.

If you don't agree with this principle, that is your right. I obviously can't speak for an entire industry, but I would guess that no one really cares if you want to cause damage to your own personal computer.

The problem is that computers are interconnected through networks, the largest of which is the internet, so one computer with compromised security can cause damage to thousands of other computers.

For this reason, Microsoft makes it difficult, virtually impossible, to give a user carte blanche to do whatever they think is right in their own mind.

a. user say:

TrustedInstaller has superior permissions to Administrators, that is true. Here's why: Administrators is a user account - you have to sign in with a username and a password - while TrustedInstaller is a system account - used by Windows itself so it doesn't have a username or a password.

A user account is relatively easy to hack -- the attacker can guess or otherwise obtain the password. A system account is harder to hack. If an Administrators account gets hacked, the attacker still has to find a way to overcome TrustedInstaller to disable or uninstall software, for example: your anti-malware software.

You'll hate me for this, but I ain't telling you how to weaken TrustedInstaller. If you lack the skills to search the internet for instructions, you're in no position to fool around with Windows security.
