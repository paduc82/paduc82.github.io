hansv mvp say:

The Single data type only has a precision of about 7 significant digits, so beyond that, you're likely to see rounding errors.

If you use CDbl instead of CSng, b will be a Double, with a precision of about 15 significant digits. If you don't specify a type, you'll get a Double, hence the second example results in 0.15 exactly.

There is generally little reason to use variables of type Single (except perhaps if you want to create humongous arrays).

hansv mvp say:

Keep in mind that Excel and VBA store numbers in binary format. It is not possible to represent 0.15 exactly in binary - the nearest approximation that fits into the selected data type will be used.

With the limited number of bits available in the Single data type, that nearest approximation is noticeably off.

Note that this does *not* happen if you use a = 1.5 or a = 15. Both these values can be represented exactly in binary format. But .15 and .015 can't.

hansv mvp say:

I suspect that you haven't declared the variables a, b and c.

Using the line

b = CSng(a)

forces b to be a variable of type Single, with limited precision.

Next, Round(b, 5) is computed as a single. Since 0.15 is stored in a single as 0.150000005960474 (approximately), rounding leaves it as 0.150000005960474. A value of type Single can't do better than that.

If you declare a, b and c as variables of type Double:

Dim a As Double, b As Double, c As Double

the variable b will still be 0.150000005960474, because that's the result of CSng(a), but it will be stored as a Double.

The last calculation Round(b, 5) will then be performed with the extra precision that a Double offers, and hence the result will be 0.15.

If you single step through the code

 

Sub Test()
    Dim a As Double, b As Double, c As Double
    a = 0.15
    b = CSng(a)
    c = Round(b, 5)
    [A1] = c
End Sub

 

you will see a = 0.15, b = 0.150000005960474, and c = 0.15 (and hence cell A1 will contain 0.15).

This is one of the reasons for explicitly declaring variables.
