dale howard [mvp] say:

Diederick --

Let me see if I can restate your question.  You want to be able to create a task than runs 7 days/week so that work continues during weekends and holidays.  Is that correct?  If so, what you could do is the following:

  • Create a custom calendar called something like 7x8 Work Week.  Set up the working schedule to be 8:00 AM - 12:00 PM and 1:00 PM - 5:00 PM for every day of the week.
  • Assign this calendar to the task as a Task Calendar, which will override the task schedule so that the task work continues 7 days per week.  Assuming you are assigning resources to the task, you will also want to select the option called "Scheduling ignores resource calendars" when you set the calendar as a Task Calendar so that the resource is scheduled to work on the weekends as well.

There is another possible scenario you do not mention in your original post.  Do you want the task scheduled at 8 hours/day or at 24/hours per day?  If you want to schedule the task using a 24 hours/day schedule, you do not need to create a custom calendar and assign it as a Task Calendar.  Instead, what you could do is to enter the Duration as an "elapsed Duration" value.  The way to do this is to prefix the Duration value with the letter "e", such as "eDays" or "eWeeks", for example.  When you enter an elapsed Duration value, Microsoft Project automatically schedules the work 7 days per week and 24 hours per day.

Just a couple of additional thoughts for you.  Hope this helps.
