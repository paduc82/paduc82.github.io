alexsun_2012 say:

Hi Mitesh,

The mailbox can only exist in one location after Dirsync is enabled. After cutover migration is completed, the on-premises Exchange mailboxes need to be converted to mail enabled users. It is not feasible to convert a mail user into a user mailbox directly in Office 365.

Considering the issue only affects one user in your organization, a workaround we suggest is 1.Backup the user mailbox data into PST files. 2.Delete the user mailbox or convert the user mailbox into a mail enabled user in Exchange 2010. 3. Re-sync the user to Office 365 and assign the Exchange Online license to provision a new cloud mailbox and then import the data from the PST files.

If you run into any problems, please let us know.

Alex
