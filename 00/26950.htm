Hello Jenny,

Thank you for posting in our community. We're here to help.

If you are using a Microsoft account email, the POP access is disabled by default. Therefore, you will need to enable it from the Outlook on web access. Here are the steps:

  1. Open a computer browser and log in to your Microsoft email account.
  2. Click on the Gear icon located on the top-right corner of the page to access the Settings.
  3. Select Options and go to Mail > Accounts > POP and IMAP.
  4. Under the POP options, tick Yes under the Let devices and apps use POP.
  5. Click Save.

Once you have made the changes, add your Microsoft email account on your device again.

If you are using a different email service provider, please reach out to their support team and ask how you can enable POP access.

rob koch say:

Windows 10 on the PC and other devices using Intel based processors and code are later generations of the original Windows line of operating systems, while those operating with either Windows RT or the Windows Phone versions of the os are only visually similar and instead operate on entirely different processors with none of the Intel code or older Windows API sets upon which most malware are based.

In earlier versions of Windows Phone upon which the Windows Phone 10 version is based, the limitations placed upon apps were far more stringent than those of the earlier PC operating system.  The key element is that no applications are allowed access directly to the core operating system components, nor are they allowed to change these in any way.  This precludes the ability for malware within an app to directly attack the operating system and so infect the device at a low level.

For this reason, the Windows Phone device doesn't require an antimalware in the traditional way you are accustomed, since the only risk with apps is the level of access to other data and applications they can acquire.  And for this reason the applications themselves are limited in their access by default and any increased access must be prompted and accepted by the user.  Also, each application must be digitally signed by the developer and is so able to be validated both as to its creator and the integrity (lack of corruption) of the application.

So if an application were known to act maliciously it could easily be identified and removed and if it is damaged or otherwise modified in any way the operating system can simply refuse to run it.

For these reasons, if Microsoft where to create an ability to add an external security application it would necessarily require access to both the operating system and full access to other applications.  This would inherently open multiple security vulnerabilities to the security application and as a result make it the greatest security risk to the entire system.  For this reason the ability to add such an inherent risk is nonsensical and so clearly shouldn't be allowed.

The problems inherent in the much older and less secure design of the original Windows disk-based operating system are well known and the most significant risk to it's future.  Microsoft understood this when creating both Windows RT and Phone, so they simply designed out all of the technical issues possible in these newer versions.  Though the visual and other user related aspects of the Windows 10 family have been designed to mimic each other as closely as possible, the internal operation of each of these are often as different as night and day.  That they can trick a typical consumer into believing they are the same is a testament to the work that the Microsoft developers have done.

Rob

rob koch say:

Basically you seem to understand what I stated about adding an anti-malware program.

Norton is a product, not a company, it's been owned by Symantec for over a decade now.

A firewall blocks opens or blocks ports to allow or disallow network traffic, nothing more.  What some security vendors like to call a firewall is actually a network monitoring and malware detection tool, integrated into the firewall in order to attempt to trap malware before it can enter the device through a communications channel, typically from the Internet.

Microsoft long ago in Windows Vista removed the ability for security vendors to integrate their own firewall programs directly into the operating system and instead provided them with an interface called the Windows Filtering Platform (WFP) which provides these network monitoring abilities for them.  All current firewalls on Windows use this platform and so are simply add-ons which view and manage the activities which the WFP performs.  Microsoft itself uses the WFP to create it's own firewall and also to monitor for network attacks within Windows Defender and MSE.

So no, Windows Phone 10 does not allow a 3rd-party firewall to be installed since it's still an unnecessary component, as whatever minimal firewall functions are actually required on the phone are integrated into the operating system itself.  Unlike the desktop Windows with which you are familiar, Windows Phone is a much more limited and secured operating system with none of the historical access points or vulnerabilities that existed on that ancient platform.

As I stated above, though the Windows Phone 10 display may look like that of the desktop version, that's where the similarities end.  The processor, code, API sets and other internal functions of the device are either completely different or only share some minimal similarities.  The fact that they can run some apps that appear to be the same is a trick of design and cross-compiling via specialized development tools.  If you don't know what I just said, it means that though the apps look and act the same to a human, they are really technically different and so can't be easily turned into malware to attack the phone the way they could on a PC.

Hi,

Try this

1. Click on cell A1 and go to Home > Conditional formatting > New Rule > Use a formula to determine which cells to format

2. In the Formula bar there, enter this formula

=COUNTIF($B$1:$B$10,LEFT(A1,SEARCH("_",A1)-1)&"*")>0

3. Click on Format > Fill > Red > OK/Apply

4. Copy cell A1, right click A2:A3 > Paste Special > Formats > OK
