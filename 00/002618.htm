Hi Stacey,

Unfortunately, there's not really an option of getting the Authenticator App from your old phone to your new one, as you need to set it up once more. You can't really set it up unless you're able to login to it, so you'll have to undergo the update procedure for your account. 3. You'll be asked for a new way of getting a code, you can provide another email address or phone number to get it.

You can do so by going to your Account Settings https://account.live.com/proofs/manage/ then click on "Turn off Existing Apps".

What computer or other device has been assigned the IP address 192.168.8.100, which is the address that eset Smart Security is indicating is sending malicious traffic?  If you don't know, then look in the DHCP address list on the Huawei router device, since this is typically a dynamically assigned (DHCP) address range on many routers.

This was the device that eset was pointing at, though if that's a DHCP address, it's possible these have changed by now, since the DHCP leases are typically only for one day.

If you haven't seen a new instance of this supposed port scanning activity since that earlier event, the problem may either be gone or was simply a false detection.  If it's continuing, then you need to look into why that specific device at 192.168.8.100 might be performing a port scan of your local (internal) network address block or devices.

Port scanning can be used to look for vulnerabilities on a device to attack, but it may also be used for other non-malicious purposes.  The eset software is simply informing you of the event and has no way of knowing whether the port scanning is malicious or not, so it simply informs you and leaves that question up to you.

I figured it out, after some more research and playing around I combined two codes around the 'net into one.

Allen Browne (the man, the myth, the legend!) had a code to check if the file exsists and there was an example of using the Import VBA.

I combined these into an if statement:

Allen's pre-code code

Option Compare Database
Function FileExists(ByVal strFile As String, Optional bFindFolders As Boolean) As Boolean
    'Purpose:   Return True if the file exists, even if it is hidden.
    'Arguments: strFile: File name to look for. Current directory searched if no path included.
    '           bFindFolders. If strFile is a folder, FileExists() returns False unless this argument is True.
    'Note:      Does not look inside subdirectories for the file.
    'Author:    Allen Browne. http://allenbrowne.com June, 2006.
    Dim lngAttributes As Long

    'Include read-only files, hidden files, system files.
    lngAttributes = (vbReadOnly Or vbHidden Or vbSystem)

    If bFindFolders Then
        lngAttributes = (lngAttributes Or vbDirectory) 'Include folders as well.
    Else
        'Strip any trailing slash, so Dir does not look inside the folder.
        Do While right$(strFile, 1) = "\"
            strFile = left$(strFile, Len(strFile) - 1)
        Loop
    End If

    'If Dir() returns something, the file exists.
    On Error Resume Next
    FileExists = (Len(Dir(strFile, lngAttributes)) > 0)
End Function

Function FolderExists(strPath As String) As Boolean
    On Error Resume Next
    FolderExists = ((GetAttr(strPath) And vbDirectory) = vbDirectory)
End Function

Function TrailingSlash(varIn As Variant) As String
    If Len(varIn) > 0 Then
        If right(varIn, 1) = "\" Then
            TrailingSlash = varIn
        Else
            TrailingSlash = varIn & "\"
        End If
    End If
End Function

On Error Resume Next

    john p spencer say:

    Marshall did not take into account the different CR# in his proposed solution.

    PERHAPS the following will work for you.  This type of query can only be built in the SQL view window.

    SELECT A.[CR#], A.StartDate, A.EndDate
    FROM YourTable as A LEFT JOIN YourTable As B
    ON A.[CR#] = B.[Cr#]
    AND A.StartDate = B.EndDate+1
    WHERE B.EndDate is Null
    UNION
    SELECT A.[CR#], A.StartDate, A.EndDate
    FROM YourTable as A LEFT JOIN YourTable As B
    ON A.[CR#] = B.[Cr#]
    AND A.EndDate = B.StartDate-1
    WHERE B.StartDate is Null

    marshall barton say:

    Kind of messy, but I think this will do that for you:

    SELECT tbl.[CR#], tbl.STARTDATE, tbl.ENDDATE
    FROM (tbl
       LEFT JOIN tbl AS [Next]
           ON tbl.ENDDATE= Next.STARTDATE-1)
       LEFT JOIN tbl AS [Prev]
           ON tbl.STARTDATE= Prev.ENDDATE+1
     WHERE (Next.STARTDATEIs Null And tbl.ENDDATE< (SELECT Max(X.ENDDATE) FROM tbl As X))
              OR (Prev.ENDDATEIs Null And tbl.STARTDATE> (SELECT Min(X.STARTDATE) FROM tbl As X))

    IMPORTANT:  That kind of Join can not be represented in the query designer so you have to use SQL view exclusively.
