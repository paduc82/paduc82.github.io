vijay a. verma say:

You can customize it in following way -

1. For removing that Grey font under Sat and Sun

Home tab > Conditional Formatting > Manage Rules > Last Rule has below formula which checks for S for color coding. H$8 = "S" is something which formats it like this. This needs to be changed appropriately.

=OR(H$8="S",COUNTIF(lstHolidays, H9)>0)

For this to happen, you need to change row no. 8 to double character days like Mo, Tu, We...(As checking for T will cause both Tuesday and Thursday to be formatted)

Hence, for Thursday, Friday, you can put following formula in this 

=OR(H$8="Th", H$8="Fr", COUNTIF(lstHolidays, H9)>0)

2. For counting number of working days, in the formula, you can replace NETWORKDAYS with NETWORKDAYS.INTL and if you read the help you will understand how to control weekends in this formula.

The spawned process is running independently from Excel, it is not dependent on Excel being "alive" or not... it doesn't even know that Excel kicked it off. Since you have marked "it's unlikely it can be done" as an answer you may not be interested in the below... {grin}

Depending on how the process that calls Excel kills it, the below may or may not work.

In the Excel workbook, I shelled Notepad and made Excel its parent.  Now, if the process that called Excel terminates the entire tree, the dependent task (Notepad) will also get terminated.

In the Excel workbook (named testparent.xlsm and saved in the folder c:\temp):

In the ThisWorkbook module:

Option Explicit

Private Sub Workbook_Open()
    setParent
    End Sub

In a standard module:

Option Explicit
Private Declare Function SetWindowLong Lib "USER32.DLL" _
    Alias "SetWindowLongA" (ByVal hWnd As Long, _
    ByVal nIndex As Long, ByVal dwNewLong As Long) As Long
Private Declare Function OpenProcess Lib "kernel32" ( _
    ByVal dwDesiredAccess As Long, ByVal bInheritHandle As Long, _
    ByVal dwProcessId As Long) As Long

Public Function setParent()
    Dim ProcessID As Long, aHandle As Long
    ProcessID = Shell("c:\windows\system32\notepad.exe", vbNormalFocus)
    Const SYNCHRONIZE = &H100000
    aHandle = OpenProcess(SYNCHRONIZE, 0, ProcessID)
    Const HWNDPARENT As Long = -8
    Dim Rslt
    Rslt = SetWindowLong(aHandle, HWNDPARENT, Application.hWnd)
    End Function

In the process that starts off Excel:

Option Explicit

Sub testParent()
    Dim ProcessID As Long
    ProcessID = Shell("""c:\program files (x86)\microsoft office\office14\excel.exe"" ""c:\temp\setparent.xlsm""", vbNormalFocus)
    Application.Wait Now() + TimeSerial(0, 0, 3)
    Shell "taskkill /pid " & ProcessID & " /t"
    End Sub
