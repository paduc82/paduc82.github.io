Include commas etc within SQL INSERT...INTO statement - Microsoft Community

geoff_lewis say:

Include commas etc within SQL INSERT...INTO statement

I'm running Office 2007 Microsoft Access on Windows XP...

I have an unbound text box, on a form, into which the operator can type a message, notes etc.  There are no restrictions on punctuation used and commas, quotation marks, forward slashes etc which are necessary within the scope of the information that is likely to be entered.

I want to pick this text up and use it in an INSERT INTO SQL statement and enter it, along with other information, into a memo field within a table.  However, simply using the syntax:

"INSERT INTO tblFeedback (Notes) VALUES (" & Me.txtbox_1 & ")"

produces errors that are attributable to a variety of punctuation issues.  Is there any way that I can 'wrap' the whole text up so that the query doesn't notice the punctation that confuses it?  I could search for each likely character and insert speech marks etc round it but that seems a bit heavy duty.  It also only catches characters that I happen to have included in the search and replace function.

Any ideas?

Thanks in anticipation.

Geoff

djsteele, mvp say:

Since Notes is presumably a text field, you need to enclose what you're trying to insert in quotes:

"INSERT INTO tblFeedback (Notes) VALUES ('" & Me.txtbox_1 & "')"

In case it's not obviously, exagerated for clarity that's

" INSERT INTO tblFeedback (Notes) VALUES ( ' " & Me.txtbox_1 & " ' ) "

If there's a chance that there might be an apostrophe in what you're inserting, use the following instead:

"INSERT INTO tblFeedback (Notes) VALUES (""" & Me.txtbox_1 & """)"

(that's three double-quotes in a row)

If there's a chance that there might be double quotes in what you're inserting, use the following instead:

"INSERT INTO tblFeedback (Notes) VALUES ('" & Replace(Me.txtbox_1, "'", "''") & "')"
Again, exagerated for clarity, that's

" INSERT INTO tblFeedback (Notes) VALUES ( ' " & Replace(Me.txtbox_1, " ' ", " ' ' ") & " ' ) "

jeeped say:
I think a real example would help sort things out.

    

The formula in B3 is,

=IFERROR(VLOOKUP($A3,$AA$3:$AC$5,2,FALSE),"")

You don't need to concern yourself with the IFERROR() wrapper. That is there simply so error codes are not displayed when there is not code in column A.

The formula takes whatever is typed into A3 and tries to find a match in the first column of AA3:AC5. In this case, it finds a match in AA4. The 2 tells it to return the values from the second column in AA3:AC5. This would be AB4. The FALSE tells it to look for an exact match which is what you want in this case.

Sometimes, instead of 2, the formula will be input with COLUMN(B:B). This simply resolves to 2. The reason it is used is that when filled right COLUMN(B:B) becomes COLUMN(C:C) and COLUMN(D:D), etc which resolve to 3, 4, etc. When the VLOOKUP() uses this and is filled right, it return the second, then the third, then the fourth, column from the lookup table. This is easier than manually editing the 2 to be 3 etc.

The table of data in AA2:AC5 does not have to be on the same worksheet but I put it there in this case for display purposes. The two number codes used in AA3:AA5 and A3:A5 are actually single digit numbers formatted as 00 to they show leading zeroes. It is important that these codes are either both numbers or both text that looks like numbers. If you mix and match text with numbers, they will not lookup properly.
