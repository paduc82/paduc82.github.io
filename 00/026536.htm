ken sheridan say: 1.  Make sure you have a reference to the Micosoft ActiveX Data Objects library (Tools | References on the VBA menu bar)

2.  Paste the following function into a standard module in the database:

Public Function GetList(strTable As String, strColumn As String, strSortColumn As String, strDelim As String, Optional strFilter As String = "True") As String

    Const NOCURRENTRECORD = 3021
    Dim rst As ADODB.Recordset
    Dim strSQL As String
    Dim strList As String
    
    strSQL = "SELECT " & strColumn & " FROM " & strTable & _
        " WHERE " & strFilter & " ORDER BY " & strSortColumn
   
    Set rst = New ADODB.Recordset
    
    With rst
        Set .ActiveConnection = CurrentProject.Connection
        .Open _
            Source:=strSQL, _
            CursorType:=adOpenForwardOnly, _
            Options:=adCmdText
        
        On Error Resume Next
        strList = .GetString(adClipString, , strDelim & " ", strDelim & " ")
        .Close
        Select Case Err.Number
            Case 0
            ' no error so remove trailing space and semi-colon
            ' and return string
            GetList = Left(strList, Len(strList) - 2)
            Case NOCURRENTRECORD
            ' no rows in table so return
            ' zero length string
            Case Else
            ' unknown error
            GetList = "Error"
        End Select
    End With
    
End Function

3.  Create a simple query, qryVehicles say, which returns rows from the Vehicles table joined to the VehicleMake table

SELECT Vehicles.ID, VehicleMake.Make
FROM Vehicles INNER JOIN VehicleMake
ON Vehicles.Make = VehicleMake.VMID;

I've assumed that the make column in Vehicles is a foreign key referencing the VMID primary key of VehicleMake.

4.  As the ControlSource of the text box control in the parent from call the GetList function like so:

=GetList("qryVehicles","[Make]","[Make]",",","[ID] = " & [ID])

You don't need to bring the subform into play at all.  The function simply uses the ID value from the parent form to restrict the list to the vehicles which match the parent form's current Contacts record.
