Word Template - Microsoft Community

scacci say:

Word Template

I have a Word template that is being used between two departments.  What they are hoping for is the potential to select their department from a dropdown and the questions pertaining to that department would be fillable.  If the questions are not applicable to their department, they would be greyed out.  Is this possible?

Thank you in advance for any suggestions!

jay freedman say:

The first thing you need to do is open the dropdown's Properties dialog and assign a name in the Tag box. You can use the same name "Dropdown" as you had for the form field. The macro will use the Tag value to identify the content control. You can see in the macro that there is error-handling to make sure there is a control with that name (and it is case-sensitive; what's in the macro must match what's in the Tag box).

Sub SampleForSubmitButtonCode()
    Dim CC As ContentControl
    
    On Error GoTo errHdl
    Set CC = ActiveDocument.SelectContentControlsByTag("Dropdown")(1)
    
    If CC.ShowingPlaceholderText Then
        MsgBox "Please select a result from the dropdown list!", vbExclamation
        CC.Range.Select
        Exit Sub
    End If
    
    ' here you do the work to submit the form
    
    Exit Sub
errHdl:
    MsgBox "No content control with Tag = 'Dropdown'", vbCritical
End Sub

Content controls differ from form fields this way: Every form field must have a unique name, which is also a bookmark. There can be many content controls in a document with the same Tag value (and the same is true for the Title value), so the SelectContentControlsByTag method returns an array of content controls that have the specified Tag value, and that array may have zero, one, or more members. The "(1)" at the end of the statement says to take the first member (there should be exactly one if that's how you created the template). However, that could generate an error if there are zero members, which is why the On Error trap is needed.

The text that appears in the dropdown if the user hasn't made a selection can be changed (by first clicking the Design Mode button), but the ShowingPlaceholderText method doesn't depend on the exact text.

Finally, the dropdown content control doesn't have a bookmark to go to, but it's easy to select the content control because we already know which one it is.

The Excel team has released a change in HTML/XLS file behavior in today’s security update for Excel 2010, 2013, and 2016. Excel will warn about the mismatch between the file extension and HTML content, but will now open the workbook in Protected View as an additional layer of security. If you trust the workbook, you can then enable editing. XLA and XLAM files are not part of this change, they will continue to not open in Excel.

 

How do I get the update? It depends on which version of Officeyou have.

  • Office 365 subscription (Click-to-Run)—install the latest updates

  • Windows Installer version (MSI)—you could wait about a week for the next security update for MSI, and it will update automatically. Or, to get the fix today, use the Download Center to get the KB for your version of Office:

    • Office 2016: https://support.microsoft.com/kb/3115438

    • Office 2013: https://support.microsoft.com/kb/3115455

    • Office 2010: https://support.microsoft.com/kb/3115476

 

Issue recap:

The Excel team has made a change in the behavior of certain file types to increase security. This change came in the security updates KB3115262, KB3170008, and KB3115322. Previously, when you tried to open an HTML, XLA, or XLAM file with an .XLS file extension from an untrusted location, Excel would warn about the mismatch between the file extension and content, but would still open the workbook without Protected View security. macropod say:
Do note that you can use single-cell tables in much the same way as textboxes and you can insert various objects into them, too. However, if you want to have text wrap around such an object, you'll need to use a table cell and format the object's wrap format to 'square' before positioning over the table cell. You can do this even when the table's wrap format is set to 'around', so you can wrap text around the object within the cell and at the same time have other text wrapped around the table itself.
