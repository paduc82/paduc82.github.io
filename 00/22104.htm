brian tillman [outlook mvp 2007-2019] say:

Because you are using a PST as the delivery location, the folders in your PST are the ones that Outlook will use for its activities.  No one on the Exchange server will be able to see anything on the Calendar when it's local.  You should be using your Exchange mailbox as the delivery location.

To get the benefits of your Exchange mailbox for the calendar and the benefits of a PST for your mail, use a rule that moves only mail messages to your PST's Inbox.  All other items, like meeting invitations and assigned tasks will stay on the Exchange server where they belong.  They won't take up much space compared to your mail.

The rule will use the condition "uses the form name form".  When you add that condition, click "form name", choose "Application Forms" from the drop-down at the top of the dialogue, select "Message" (and possiblly "Multimedia Message" "Text Message", "Digitally Signed Message", and "Encrypted Message") from the left-hand pane, click Add, then Close.  For the actions, choose "move it to the specified folder" and "stop processing more rules", select your PST's Inbox for that folder, and finish the wizard.  Put this rule at the top of your rule list, but below any rules you have that deal with messages you with to delete.  The rule will be a client-side rule that will work only when Outlook is running, but that's just what you want.

Content Control Dropdown List Placeholder text

When you create a new dropdown list content control Word automatically defines the placeholder text "and" first dropdownlistentry "Choose an item"

Using the dialogs you can then add additional dropdownlist entries.

When the user enters the content control the placeholder text is displaye (grey text) and when the click the arrow to make a selection "Choose an item" is listed as one of the items that they can choose.  In reality "Choose an item" is not an item put a prompt.

a. user say:

FYI I've figured out a workaround that allows me to import multiple files.

I editted the XSD file for the data to change the non-repeating XML elements into repeating elements.  This can be done by changing maxOccurs="1" to maxOccurs="unbounded" for these elements.

Then when importing multiple files, Excel just appends all information for the mapped repeating elements in the worksheet columns.

I'd be interested hearing if there are any other ways to do this.
