ken sheridan say: You could try opening multiple instances of the form at different records.  The following is the code for a standard module I put together as a simple demo of this, to show each company from a Companies table in a separate instance of a frmCompanies form.   Firstly the following goes in the Declarations area of a module:

Dim frm As Form
Dim colPopups As New Collection

Then add the a function along these lines to the module:

Function StackCompanyForms()

   Const conSQL = "SELECT * FROM Companies Order By Company"

   Dim dbs As DAO.Database, rst As DAO.Recordset
   Dim strSQL As String
   Dim n As Integer

   Set dbs = CurrentDb
   Set rst = dbs.OpenRecordset(conSQL)

   Do While Not rst.EOF
       n = n + 1
       Set frm = New Form_frmCompanies
       colPopups.Add frm, frm.hwnd & ""
       frm.Caption = "Company " & n
       frm.Filter = "CompanyID = " & rst.Fields("CompanyID")
       frm.FilterOn = True
       frm.Visible = True
       DoCmd.MoveSize n * 360, n * 360
       rst.MoveNext
   Loop

End Function

To open the multiple instances of the form you simply call the StackCompanyForms function.  Calling the MoveSize method 'stacks' the forms form top left to bottom right, but with more than a small number of records you'd soon run out of screen space, in which case you'd have to amend the code so it restarts the 'stack' after a certain number.  This is clearly slightly unrealistic as you'd be unlikely to want every company record opened simultaneously.  In real life you'd be more likely to restrict the companies by passing a value or values into the StackCompanyForms function as argument(s) which could then be used when building the SQL statement for the recordset to restrict it to certain companies.  Alternatively you might dispense with the recordset and the loop completely and simply open a new instance of the form, in which the user can then navigate to a specific record.

To close all instances of the form simultaneously put the following function in the same module:

Function CloseCompanyForms()

   Dim n As Long

   For n = 1 To colPopups.Count
       DoCmd.Close acForm, "frmCompanies"
   Next n

End Function

pamcaswell say: When you drag a table by its handle, it changes from  in line to text wrapped, and then, as you have discovered, behaves in unexpected ways.  To get it back to in line,  go to Table > Table Properties > Table tab and set the  wrap to none.

When you want to put something above a table that is the first thing on the page, put the cursor in the top left cell and click Table > Split Table.  Word will put a return () above the table. 

The table splitting method is more reliable than putting the cursor in the top and left most table position and pressing enter, which does work most of the time.

You can add a Microsoft Forms web part in SharePoint. following are the steps:

neil dolinger say:

Centering a picture as a bullet point in PowerPoint

I am using a picture as a bullet point.  I have successfully imported it and resized it, but PPT is centering the bullet point at the bottom of the picture, so the picture is higher than the text it is highlighting.  How can I make the center of the picture the center of the bullet, so it is on the same level as the text?

steve rindsberg say:

>> How can I make the center of the picture the center of the bullet, so it is on the same level as the text?

 

One trick that works is to add extra background at the bottom of the picture you're using as a bullet image so that the live area of the picture is forced upward.

 

 

neil dolinger say: Thanks, Steve.  However, the picture is already too high -- if anything, I need to force it downward.  But since the bullet is centered on the bottom of the picture, the converse (adding extra background to the top of the picture) doesn't seem to work. 

steve rindsberg say:
What about removing b/g from the bottom of the picture then?

neil dolinger say:

That helped a little, but there wasn't much white space at the bottom to begin with -- I made sure to crop the picture very close to the edges of the image.

Am I correct to infer then that it is not possible to make PowerPoint change the centerpoint of bulets from the bottom to the center of a picture?

steve rindsberg say:

Bullets don't center on text other than by chance or trickery.

Bullet pictures sit on the text's baseline; you control the bullet height by telling it what percentage of the text height to be.

By adjusting that and the amount of dead air round the picture it *should* be possible to force it to center on the text.

But PPT won't do it automatically.
