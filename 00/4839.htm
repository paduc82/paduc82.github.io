ronrosenfeld say:

I would approach this differently.  It's somewhat complex and I, as you, find it difficult to keep track of what's going on with the various loops.

I would define an object (class) that consists of all the information that you want in a row in the final table; the properties of that object would be the contents of each column.  I find this, along with appropriate naming of the properties, helps me keep track of what I am doing, which, in turn, makes the code easier to debug/modify/maintain.

I would then go through each line on Sheet1; and each "job" in that line; to generate an object for each job, and for each transport type to be allocated to that account.  So if there are five jobs, and two transport types, there would be ten objects generated.

As much as practical, I would do this within VBA, as it will be orders of magnitude faster than repeated reads and writes from the worksheet.  Although, to get the transport types and associated pricing from Sheet2, I do use the Range.Find method as this executes pretty quickly.  But, if speed becomes an issue, it would be a place to evaluate.

You will need to enter a Class module, as well as a Regular module.

Sub TravelCosts()
    Dim cTC As cTravelCosts, colTC As Collection
    Dim vTimeAlloc As Variant
    Dim rFees As Range
    Dim vres() As Variant, rRes As Range
    Dim wsTimeAlloc As Worksheet, wsFees As Worksheet, wsRes As Worksheet
    Dim I As Long, J As Long
    Dim FirstAddress As String
    
Set wsTimeAlloc = Worksheets("sheet1")
Set wsFees = Worksheets("sheet2")

Set wsRes = Worksheets("Final Table")
    Set rRes = wsRes.Range("A1")

'Get job allocations
With wsTimeAlloc
    'assumes 5 jobs as per documentation
    ' could be made dynamic if necessary
    vTimeAlloc = .Range("a2", .Cells(Rows.Count, "A").End(xlUp)).Resize(columnsize:=14)
End With

'Collect time alloc data
Set colTC = New Collection
For I = 1 To UBound(vTimeAlloc, 1)
    For J = 5 To 9
        Set cTC = New cTravelCosts
        With cTC
            .Line = vTimeAlloc(I, 1)
            .Account = vTimeAlloc(I, 2)
            .Location = vTimeAlloc(I, 3)
            .TimePerCent = vTimeAlloc(I, 4)
            .JobPerCent = vTimeAlloc(I, J)
            .ProdNo = vTimeAlloc(I, J + 5)
        End With
            
            'Get Transport Fees
            With wsFees.Columns(1)
                Set rFees = .Find(what:=cTC.Account, after:=.Cells(1, 1), _
                    LookIn:=xlValues, lookat:=xlWhole, searchdirection:=xlNext, _
                    MatchCase:=False)
                If Not rFees Is Nothing Then
                    FirstAddress = rFees.Address
                    With cTC
                        .TravelFees = rFees.Offset(0, 1)
                        .TravelType = rFees.Offset(0, 2)
                    End With
                    colTC.Add cTC
                    
                    Do
                        Set rFees = .FindNext(rFees)
                        Set cTC = New cTravelCosts
                        With cTC
                            .Line = vTimeAlloc(I, 1)
                            .Account = vTimeAlloc(I, 2)
                            .Location = vTimeAlloc(I, 3)
                            .TimePerCent = vTimeAlloc(I, 4)
                            .JobPerCent = vTimeAlloc(I, J)
                            .ProdNo = vTimeAlloc(I, J + 5)
                        End With
                        If FirstAddress <> rFees.Address Then
                            With cTC
                                .TravelFees = rFees.Offset(0, 1)
                                .TravelType = rFees.Offset(0, 2)
                            End With
                            colTC.Add cTC
                        End If
                    Loop Until FirstAddress = rFees.Address
                End If
            End With
    Next J
Next I

ReDim vres(0 To colTC.Count, 1 To 9)

'Write column headers in first row or results array
    Dim arrHeaders As Variant
    arrHeaders = Array("Line", "AccountNo", "Location", "TimeAlloc%", "ProdNo", "Job%", "TravelType", "TravelFees", "Total$$")
    For I = 0 To UBound(arrHeaders)
        vres(0, I + 1) = arrHeaders(I)
    Next I
    
'Write data to results array
    For I = 1 To colTC.Count
    With colTC(I)
        vres(I, 1) = .Line
        vres(I, 2) = .Account
        vres(I, 3) = .Location
        vres(I, 4) = .TimePerCent
        vres(I, 5) = .ProdNo
        vres(I, 6) = .JobPerCent
        vres(I, 7) = .TravelType
        vres(I, 8) = .TravelFees
        vres(I, 9) = .TotalDollars
    End With
    Next I
   
'Write results array to worksheet and format
Set rRes = rRes.Resize(UBound(vres, 1) + 1, UBound(vres, 2))
With rRes
    .EntireColumn.Clear
    .Value = vres
    .Columns(4).NumberFormat = "0%"
    .Columns(6).NumberFormat = "0%"
    .Sort key1:=.Columns(1), order1:=xlAscending, _
            key2:=.Columns(7), order2:=xlAscending, _
            MatchCase:=False, Header:=xlYes
            
    .EntireColumn.AutoFit
End With
       
End Sub

Out of Office AutoReply - Microsoft Community

peri h say:

Out of Office AutoReply

I am trying to set up an out of office auto reply however the out of office option on the tools menu is greyed out and does not respond
talking moose say:
The Out of Office... menu item is only available for Exchange accounts. If you are using POP or IMAP mail then you cannot use this feature.

You do have two options for POP and IMAP accounts:
  1. You can go to Tools menu --> Rules... and create a rule that automatically responds to every mail message you receive. If needed, you can limit the scope of this rule to only respond to messages that Outlook does not think are junk. This option requires that you leave Outlook open and running on a machine all the time for it to work, which may not be ideal.

john p spencer say:

The problem is that GROUP BY truncates memo fields to 255 characters.  Which field is the memo field.

The posted query does not seem to have what I would consider to be a memo field. 

If you want only one memo field and it does not make any difference which of the memo fields you can use first or last to get the memo field value from one of the records in the grouped records.  Or if the first 255 characters will determine unique memos you can group by that.

SAMPLE A

SELECT FirstName, LastName, First(MemoField) as FMemo
FROM SomeTable
GROUP BY FirstName, LastName, Left(MemoField,250)

Sample B

SELECT FirstName, LastName, First(MemoField) as FMemo
FROM SomeTable
GROUP BY FirstName, LastName

Sample A should return multiple records assuming that there are multiple records for the combination of FirstName and Lastname  with different memo fields.  Sample B will return ONE record with a more or less randomly selected memo for the FirstName and LastName.
