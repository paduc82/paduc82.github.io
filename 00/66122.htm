Hi Dilynn,

QuickPlay is a technology pioneered by Hewlett-Packard that allows users to directly play multimedia without booting a computer to the main operating system. QuickPlay software versions 3.0 and newer included in notebooks shipping with Windows Vista, solely retain the Windows-only component, as the "Direct" component is no longer implemented due to unresolved compatibility issues. Instead, users must first boot Windows Vista and login into their user accounts before the Windows-only version of the QuickPlay software can be run.

 

If you want to disable QuickPlay at startup then you need to use system configuration utility to disable it. Follow the steps on how to disable startup items to disable startup items using system configuration utility. 

 

1. Click Start, type msconfig in the Start Search box, and then press ENTER. (If you are prompted for an administrator password or for a confirmation, type the password, or click Continue.)  

2. On the General tab, click Selective Startup.  

3. Under Selective Startup, click Startup Items Tab.

4. under that look for HP QuickPlay, the other called HP Quick launch buttons. 

5. Now select the one at a time and click on disable

6. Restart the computer, Now quickplay should not start automatically.

 

However, if you wish to remove QuickPlay from the computer use the program and features to uninstall the program completely.

I appreciate your timely replie to this inquiry.  But none of your posts qualify as 'answer', imho.  Here is my candidate for 'answer' status in this thread:

 

1. 'Power management' in Windows XP SP3 is strictly valid only for a stand-alone workstation   For workstations connected to a network, its local settings can be overridden by network traffic, making 'power management' meaningless in a networked context.

 

2. Windows 7 has a utility, powercfg.exe, in which the user can identity the process or service that woke up the system: '-lastawake' and '-requests'.  These command line options are not present on the Windows XP SP3 version of powercfg.exe

 

3. Diagnosing the cause of unwanted wakeup from standby can be done manually by testing it against each running service individually.  A gross identification can be done by disonnecting the LAN cable to the NIC or switch, to see if the disturbance is coming from the network or not.  This will partition the problem between network and local sources, before attempting to test each local service individually.  therwise there are no diagnostics for this problem in XP, and no direct history log (which includes event logs, which do not record these events).

 

4. There may be third party tools which compensate for this shortcoming in XP, but an hour or two of googling 'powercfg' did not locate them.
