Hello HGL920,

A lot of factors could affect the download speed of Windows Store, this could be due to multiple applications being downloaded all at the same time,  slow internet connection, other applications are being updated during the download, and the like.

To troubleshoot the issue, you may follow the methods suggested below:

Method 1
Run the Windows apps and store troubleshooter. This utility will sort any possible case of system glitch causing the slow download speed of your Windows Store.

Method 2
Check your internet connection. Make sure your connection is stable. Use Wi-Fi over LAN as some users had reported having better speed when using Wi-Fi.

Method 3
Stop other applications from updating while your download is in the process of completion. Check for other applications being downloaded or updated before proceeding to download and install a new application.

Method 4
Disable any third-party antivirus. It is possible that it could also be causing the slow download speed if not blocking the process completely.

Method 5
Re-register Windows Store.
1. Press Windows key + I to open Settings, then select System.
2. Go to Apps & features, then select Store.
3. Click Advanced options, click the Reset button.
4. Click Reset again to confirm.

ken sheridan say: I'd suggest three amendments:

1.  Concatenate the values of the controls in to the string expression rather than referencing them.

2.  Define the range as on or later than the start date and before the day following the end date.  This will ensure that any rows with values on the last day of the range which include a non-zero time of day element are included.  Unless you have taken specific steps in the table definition to disallow such values you cannot be confident that such rows don't exist.  A between....and operation will exclude these.

3.  When building the string expression format the date parameters to the ISO standard of YYYY-MM-DD to ensure that the values are correctly interpreted regardless of the system's regional settings.

So:

Dim strFilter As String
Dim strDateStart as String
Dim strDateEnd As String

strDateStart = "#" & Format(Forms![LogOn]![WeekEndingDate],"yyyy-mm-dd") & "# -7"
strDateEnd = "#" & Format(Forms![LogOn]![WeekEndingDate],"yyyy-mm-dd") & "# +1"

strFilter = "TaskDate >= " &  strDateStart & " And TaskDate < " &  strDateEnd

Me.Filter = strFilter
Me.FilterOn = True

Ken Sheridan, Stafford, England

saastubli say:

I've managed to figure out most of it.

It seems that ESENT databases can't be moved and are somehow tied to the path they were initially in.

Running 'esentutl /g' on files in new location reported corrupt databases, while running it on the same files in old location said they were fine.

Solution was to place files back in the original location and to copy them with esentutl to new folder with 'esentutl /y'

I did that on the two files mentioned in my original question and most of the problems went away.
