  4. Create a new stencil.
  5. Move rectangle A from the document to the stencil.
  6. Change the document to Variant 2, move rectangle B to the stencil.
  7. Change the document to Variant 3, move rectangle C to the stencil.
  8. Change the document to Variant 4, move rectangle D to the stencil (should be no more rectangles in the document).
  9. Place a rectangles A, B, C, and D onto the document.
  10. Change between Variants 1, 2, 3, and 4 and observe the shape styles.

Unexpected behavior:

Just for the sake of clarity and to bring this down to a manageable level - can you clarify the following:

Whatever the case, sounds very much that something has become corrupted along the way and without knowing exactly what was done from the very first step - impossible to guess as to what the cause might be. Suffice it to say that after a multitude of years dealing with import/export - have yet to encounter a scenario where conventional Outlook import/export  didn't work if all things were done correctly. There is something  missing in this problem description - you don't sync the contact folders with the AB - everything appears accordingly. However, run one of many different programs that interacts with outlook contacts and all bets are off as to what the results could end up being.

Very simple way to resolve the issue if you still have the source file available:

#1 - Create a new profile

#2 - Create/use a new default PST file

#3 - Re-import the contacts ensuring that every field has been manually mapped from the file being imported to the applicable Outlook field. Check the results to see if everything is where it's supposed to be. A secondary word of caution, DO NOT map the <FULL NAME> field AND the component parts (First, Last etc) separately - do one or the other with the latter being the preferable/most reliable approach. If it is - proceed with

#4 - Open your old default PST file and copy the contents of the various other folders (Inbox) etc to the equivalent folder in the new PST file.

#5 - when done - close the old PST file. YOu should be good to go.

There is one minor salient fact that eludes a number of people (developers in particular in this case) which is that it is possible to update contact FullName, component name and fileas fields that will have an adverse affect on other views - all depends on <HOW> the data was updated (gets down to if the Outlook Object Model (OOM) was used or a secondary MAPI library. If a secondary library was used it further comes down to HOW a particular field was updated since there is more than one way to do it - one approach produces the problem the other doesn't depending on a given sequence of events). The Outlook import/export wizard is not one of those that suffers from the problem.
