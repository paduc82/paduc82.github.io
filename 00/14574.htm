john mcghie say:

Yes, you are!  You are missing the macro code that does the work.

I am sorry: this is a really difficult concept to explain, unless you have been working with Word for years.  But let me try again (I can't believe I did not explain this properly last time, I am a Technical Writer, I am supposed to be good at this!).

In Word, there are typically three files in play for each document you have open:

  • The document
  • The Document's Attached Template
  • The Global Template

What makes it confusing is that each of the three can inherit functions that normally belong in one of the others. Fortunately, in this case that does not happen so we don't need to worry about it.

When you said "I opened a brand new calendar template..." what you did was effectively a File>New>From Template, and you chose one of the template files.  For the purposes of this explanation I am going to assume you chose the Calendar Template known as "Horizontal Monday Start".

On my system, the template is here: 

You won't be aware of the file name or location unless you go looking, because Word hides what is actually happening from you (which annoys me intensely, because that means people never learn how to do this for themselves).

When you click OK, instead of performing an "Open" on the file, Word actually triggers a different command: "New From Template".  It reads the file, detects that the file is one of the "template" types, and changes from "Open" to "New From Template".

and some of them are not full.  The various file types: .docx, .docm, .dotm, .dotx can be thought of as different "shops".  They all have some shelves, but some files do not have particular shelves because they do not stock that kind of merchandise.

"New From Template" copies only the "Document" properties from the file: things such as Page formatting, section formatting, styles, and any text or graphics that might be there.

It does NOT copy the code. The Calendar templates are actually quite complex little Word Applications behind the scenes: there's a form pops up to ask you to choose the month, and some quite complex code that runs behind the scenes to populate the cells with dates.

What happens instead is that Word sets a LINK to the original source template.  If you choose a calendar and set the month, then look in Tools>Templates and Add-Ins in the "Document Template" field, you will see the link to the template that you chose.

This link is almost never used after the document has been created: the only thing it is used for is to call in the code that does the work if you ever decide to change the Month.  You will not that whenever you create a new Calendar, a new menu appears on the Menu Bar beside the Window menu, named "Calendar".  That menu is inserted there by the code when it runs.

However, the file you have on the screen after you create a Calendar is a .docx.  You won't be able to see that, because Word simply gives it a serial number and no extension: in my case, "Document 2".  But if you try a File>Save As... at this point, Word will then show you that it proposes to save the file as a .docx type, because that is what it is. The internal containers ("shelves") that can contain the macro code are missing: purposely removed, because a .docx can't contain macros.

You could override this file type (in fact you did, during your testing) and save as a .dotm.  The missing shelves will be created, but since the file does not contain any code, they will remain empty.  Even though the file type is now .dotm, the content will be only what was allowed in the .docx.

To do what you want to do, you need to locate the template and copy that across to the other machine.  On my computer it is in 

Macintosh HD/Applications/Microsoft Office 2011/Office/Media/Templates/Print Layout View/Calendars/Horizontal Calendar - Monday Start.dotm

You need to find Horizontal Calendar - Monday Start.dotm and copy it to the other computer.

On the other computer you need to store that file in a template location that Word knows about, otherwise when you open File>New... Word won't offer it to you.

Templates can be anywhere you put them.  You "could" of course place the file in the path above.  However, I suspect that location does not exist on the other computer.  I suspect that someone has been at that other computer with one of those applications that promises to make Office run faster by "deleting unnecessary files".  Around the world, users have wasted centuries of their life attempting to fix the damage caused by those things.  If the location did exist, all this would have worked a week ago and you would not have the problem!

The easiest way to find out where to put them is to ask Word where it wants to keep its templates.  Go to Word>Preferences>File Locations and highlight "User Templates" .  Click Modify.  Don't make any changes: the folder that opens is where Word wants its templates.  You will see the file Normal.dotm there already.

Make a note of that location (you can drop down the disclosure arrows in the folder name field at the top of the dialog to see the full path).  Then quit Word (for safety: Word does not like having its private parts played with while it is running...).

Move your calendar template in there.  Word will offer it to you next time you do a File>New From Template...  It will work exactly the same as it does on the original machine.

Just for grins, have a look at this: http://www.john.mcghie.name/Calendar.xlsx

That's a copy of one of the other Word calendar templates, re-expressed in Excel.  It does not use VBA, so it does not need to maintain a link to any other files. Everything works off the "Month" field.  It's not actually a "Month" field, it's a complete date: Year, Month, Day; formatted so only the month is shown.

Initially Excel populates that date with today's date from the system clock.  To get a different month, simply type the date of the first of the month in that cell: 1-jun-2014 or whatever.  When you type, the cell should display only the name of the month.  If it displays anything else, Excel did not recognise what you typed as a full date: try again, using a date format Excel knows.

Otherwise leave it, and each time you open it, it will reset to the current month.

You can do whatever you like to the formatting, just ensure that the formulas remain in place.

SOLUTION: Unchecked "wake computer to run" under settings in the software application.  (They were crafty, the setting was buried.)  Issue resolved.

Event View system log message of issue: "Wake Source: Timer - Windows will execute 'NT TASK\Remediation\AntimalwareMigrationTask' scheduled task that requested waking the computer."
