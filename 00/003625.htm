Hey,

I had this problem too.  There are other threads which offer a solution (it worked for me):

 

You need to make sure your phone or memory card has a folder called "Dictionary".  If not,  add one yourself:

Gallery > Options > Add Folder     [the folder will be in phone memory]

or

Gallery > Memory Card > Options > Add Folder     [the folder will be on your memory card]

 

then type "Dictionary" and press <OK> to create the folder.

 

I think it's up to you whether you want all your custom words to be stored in the phone memory (less space) or on your memory card (larger, but if you switch memory cards you'll have to recreate the dictionary).

 

Hope this helps someone!

steve rindsberg say:

It's not the size of the file, but more likely the number/size of hyperlinks.  This gives you some background to the problem:

 

Lost hyperlinks, hyperlinks link to wrong place, hyperlinks stop working
http://www.pptfaq.com/FAQ00401.htm

 

Links to slides within the presentation include the entire slide title.  One way to minimize the amount of link space is to Duplicate the title text.  Drag the original title off the slide and change it to something short:  "Sld12" for slide 12, for example.  Leave the duplicate in place; it looks the same as the original title but isn't a title.

You may have to re-create links to that slide to get PPT to reset them to the new shorter title, but if too much link text is the problem, this'll fix it.

 

Have a look here too ... might save you a lot of manual clickyclicky:

Convert slide titles to shapes to solve hyperlink limit problems
http://www.pptfaq.com/FAQ00683.htm

bernard liengme say:

If you are trying to sum the hours are greater than 8 for each alternative cell in D4:N4:

=(D4-8)*(D4>8)+(F4-8)*(F4>8)+(H8-8)*(H4>8)+(J4-8)*(J4>8)+(L4-8)*(L4>8)+(N4-8)*(N4>8)

Let's look at (D4-8)*(D4>8): this first part (D4-8) find the hours over 8, the second part (D4>8) returns 1 if D4 is greater than 8 or 0 otherwise, this stops us counting negative value. So this gives 1, for F4 we get 0.6, for H4 we get 0 becuae H4 is less than 8, for J4 we get 0,6, for L4 we get 0,5 and for N4 we get 0. These total 2.7

An alternative formula is

=SUMPRODUCT(--(ISEVEN(COLUMN(D4:N4))),--(D4:N4>8),D4:N4-8)

We need columns D, F, H, J,L and N - these have even column numbers. We are interested only in value that exceed 8. We want to find each amount that exceeds 8.

Sure, no problem if you use the numbering feature.

  1. Number your list
  2. enter your support text ie "See step: "
  3. Insert tab / Links group / Cross-Reference to display Cross-Reference dialog box
  4. Select your "Reference Type:" you want "Numbered Item" and the "Insert Reference To", you want "Paragraph Number"  (these should be defaults)
    Select the numbered item you want to  refer to from the list displayed
  5. Click on Insert button

A new field will be inserted with the list number. If you adjust the list, just select the cross reference field (or whole document, <CTL A>)  and update it by pressing <F9>
