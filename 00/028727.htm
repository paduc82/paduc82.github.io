I think you've probably identified the reason. Windows 7 installs a huge number of fonts, and these undoubtedly include updated versions of some older fonts. You can copy the font files from your new laptop to the Fonts folder of the old one.

suzanne s. barnhill say:

Latin Extended Additional is a character subset of any given font. If you look at the Insert | Symbol dialog, you will see these characters in some (but not all) fonts. In order to use these characters, you must use a font that contains them. If a given font on your old laptop doesn't contain these characters, it's possible another font will; if you're using the same font on both systems, then you need to copy the newer version of the font (which does contain the characters) to the Fonts folder of the old system.

boblarson say:

.accdr is intended for use with Access 2010 Runtime,

 

ACCDR is intended to be able to run AS IF it was being run by the runtime.  Where'd the "it is intended for use with Access 2010 Runtime" come from Hans?  There is no need for the ACCDR extension if it is being run by the runtime.  It is only a substitute for having to use the previous /r switch with a shortcut to open as a runtime app for those with full versions of Access.  Of course it does not really protect anything as anyone can just rename it back.  But you can create the ACCDE file and then rename it as ACCDR and it will then run as if it was on Runtime with the full version of Access. 

 

So it will let you simplify things and not have to remove the Ribbon and such which you would normally need to do if you were giving the users the ACCDB or ACCDE file when they had full versions of Access.  By using the ACCDR format it behaves as IF it were an ACCDB or ACCDE file being run by the runtime.

Powerpoint VBA - auto-populate generic text - Microsoft Community

echo34 say:

Powerpoint VBA - auto-populate generic text

I have a presentation that has several introduction slides, these contain generic information but make references to a specific audience name in the text that needs to be changed depending on the audience.

Is there a way to make these references auto-populate the name, so that if I want to change the audience I can just type the new name once, and it will update the text on all the generic introduction slides?  

The name is in the text several times across about 10 slides, and just fits into the sentences on the slides. To change each reference manually will take quite a long time, and may result in mistakes, as the presentation will be presented by a variety of people, so I want to make it as simple as possible. 

john korchok say:

Here's one approach to solving this:

  1. Create your parent document in Word with generic text.
  2. Select a passage to be inserted and Copy it.
  3. In the child presentation, click on HOME and then on the bottom half of the Paste button.
  4. Choose Paste Special, then choose Paste link.
  5. Choose Microsoft Word Document Object as the format. Click on OK.

The parent text is pasted as a link. Save and close the child presentation.

Now edit the text in the parent document and save. Reopen the child presentation: it asks if you want to update the links. Choose Update Links and the linked text is updated. Tell your other presenters to click Cancel when they open the file.

To make this solution portable across computers, put your parent document at c:\ or c:\Source\. Links are absolute by default, so planning ahead allows you to change computers in the future and still have the links work.
