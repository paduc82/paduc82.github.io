rob koch say:

You're correct Andrew, only someone inside Microsoft's anti-malware definitions analysis group could truly answer this and they won't be monitoring these forums.

The collection of these files is completely automated by the behavioral monitoring portions of the Microsoft security products including Windows 10 Defender, so even someone who understands how this all works couldn't tell you why your particular files were requested.  That's determined by the activity that the behavioral monitoring detected as potentially suspicious and so based on the specific situation that these saw on your PC that caused it to question them.

The reality is that there are likely thousands to possibly millions of such requests a day via MAPS, many of them invisible to PC users who've selected the Advanced settings option to "Send file samples automatically when further analysis is required", which will only prompt if a file is likely to contain personal information.  Most of these are not likely to result in any further action, since once they are automatically collated with other similar files by the Microsoft cloud systems that analyze them, they are found to be non-malicious and eventually used to refine the behavioral sensors which detected them.

Since these sensors don't have the ability to fully analyze on your local machine, they can't provide any real intelligence either, so this comes later once the MAPS automated and possibly human analysis is completed.  If they are either found to be malicious or very common false positive detections they are more likely to receive a fast response via new detections which deal with them appropriately.

That's why monitoring these uploads via prompts is simply frustrating, since inherently there's no useful information provided directly to the PC user, so hiding the upload process when possible is the most effective selection.  Unfortunately, this isn't possible if the item contains possible personal information, so either removing the file(s) in question or disabling the MAPS uploads completely may be the only way to stop them.

However, disabling MAPS also removes the ability for your PC to download any urgently released definitions updates designed to deal with them, since such a dynamic signature service request has always just preceded the request to upload the files.  So this decision shouldn't be made lightly and if selected, I'd personally either choose to remove the questionable file(s) or at least perform some additional sanity checks using 3rd-party on demand security tools before making this [temporary] change.
