jllatham say:

OK, I did some looking around and found this page:

http://www.get-digital-help.com/2007/09/16/excel-remove-blank-cells/#array

I took what they offered there and have modified it for your use.

Note that each of these initial formulas must be entered as an ARRAY formula - this is done by terminating (commiting) the formula with the 3-key combination of [Ctrl]+[Shift]+[Enter] instead of just the [Enter] key.  When done properly, the formula will be shown as surrounded by {} brackets that Excel puts in (you can't just type them in).

On your PROPOSAL sheet put these formulas in the cells indicated and commit them each as an array formula as you enter them and then fill them down for the 12 rows needed.  As you can see by the bolded and underlined portion of the formulas, the formulas are all the same except for 1 set of addresses:

 

In Proposal!C26

=IFERROR(INDEX(Pricing!$S$37:$S$48, SMALL(IF(Pricing!$S$37:$S$48=0, "", ROW(Pricing!$S$37:$S$48)-MIN(ROW(Pricing!$S$37:$S$48))+1), ROW($A1))),"")

in Proposal!D26

=IFERROR(INDEX(Pricing!$C$37:$C$48, SMALL(IF(Pricing!$S$37:$S$48=0, "", ROW(Pricing!$S$37:$S$48)-MIN(ROW(Pricing!$S$37:$S$48))+1), ROW($A1))),"")

and in Proposal!L26

=IFERROR(INDEX(Pricing!$Q$37:$Q$48, SMALL(IF(Pricing!$S$37:$S$48=0, "", ROW(Pricing!$S$37:$S$48)-MIN(ROW(Pricing!$S$37:$S$48))+1), ROW($A1))),"")

luke moraga say:
This macro (which is rough, I admit) will do most of what you ask. It doesn't create the arrows, but you should be able to do that via conditional formatting.

Sub CompareList()
Dim Part1 As String, Part2 As String
Dim Qty1 As Integer, Qty2 As Variant
Dim MPN1 As String, MPN2 As String
Dim xOutput As String
Dim FindCheck As String
Dim NumberRows As Integer


For i = 2 To 1000'Set number to reasonably high amount
'Clear values
Part2 = ""
Qty2 = ""
MPN2 = ""
QtyDetla = ""
QtyCheck = ""
MPNCheck = ""
xRow = ""

Part1 = Cells(i, "A")
Qty1 = Cells(i, "B")
MPN1 = Cells(i, "C")

'If at end of list, exit loop
If Part1 = "" ThenExit For

On Error Resume Next
xRow = WorksheetFunction.Match(Part1, Range("D:D"), 0)
On Error GoTo 0
If xRow = "" Then'Value not found
FindCheck = "REMOVED"
GoTo CreateOutput
End If

Qty2 = Cells(xRow, "E")
MPN2 = Cells(xRow, "F")
If Qty2 <> Qty1 Then
QtyDelta = Qty2 - Qty1
    If Qty2 > Qty1 Then
        QtyCheck = "QTY INCR"
        Else
        QtyCheck = "QTY DECR"
    End If
End If
If MPN1 <> MPN2 Then
    MPNCheck = "MPN"
End If
If QtyCheck <> "" And MPNCheck <> ""Then
    xJoiner = "/"
    Else
    xJoiner = ""
End If
FindCheck = QtyCheck & xJoiner & MPNCheck
If FindCheck = "" Then FindCheck = "NONE"
GoTo CreateOutput


CreateOutput:
'Output
Cells(i, "G") = Part1
Cells(i, "H") = Qty1
Cells(i, "I") = MPN1
Cells(i, "J") = FindCheck
Cells(i, "K") = Qty2
Cells(i, "L") = QtyDelta
Cells(i, "N") = MPN2
Next i

'Find any new parts
Dim RecordCount AsInteger
RecordCount = NumberRows + 1
For x = 2 To NumberRows
Part2 = Cells(x, "D")

'If at end of list, exit loop
If Part2 = "" ThenExit For

xCheck = ""
On Error Resume Next
xCheck = WorksheetFunction.Match(Part1, Range("A:A"), 0)
On Error GoTo 0
If xCheck = "" Then
Cells(RecordCount, "G") = Part2
Cells(RecordCount, "J") = "ADDED"
RecordCount = RecordCount + 1
End If
Next x


End Sub

rohn007 say:

That is a tough question. A few possible issues: first is complying with license terms, second is will the installer let you do it, finally will the activation system let you do it.

 

If you have a retail license, it allows you to install on a "licensed device" and a portable device. But there is a term that allows only one copy on the licensed device

<snip 2010 mslt>

2. INSTALLATION AND USE RIGHTS. a. One Copy per Device.

You may install one copy of the software on one device. That device is the “licensed device.”

b. Licensed Device.

You may only use one copy of the software on the licensed device at a time.

c. Portable Device.

You may install another copy of the software on a portable device for use by the single primary user of the licensed device.

3d. Use with Virtualization Technologies.

Instead of using the software directly on the licensed device, you may install and use the software within only one virtual (or otherwise emulated) hardware system on the licensed device.

</snip>

So clauses 2b and 3d  seem to indicate that MS doesn't want you to do that .

 

If you are adventurous, I'd say just try doing it. You may get lucky and the installation process and the activation process are not smart enough to confirm that you are installing a second copy inside a virtual machine. Frankly, the installation and activation procesess were designed and built long before virtualization became commonly available and may not smart enough to make the distinction.
