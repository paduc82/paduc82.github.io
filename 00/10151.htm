jay freedman say:

I probably haven't tested this enough, but try it on a copy of your document and see what happens:

Sub x()
    Dim par As Paragraph
    For Each par In ActiveDocument.ListParagraphs
        If Not par.Range.Information(wdWithInTable) And _
                Not par.Style = ActiveDocument.Styles("List Bullet") Then
            par.Range.ListFormat.ApplyBulletDefault
        End If
    Next par
End Sub

Most of the paragraphs in numbered (or "lettered") lists will have the List Paragraph style. In testing, I found that paragraphs with the List Bullet style were included in the .ListParagraphs collection, but applying the .ApplyBulletDefault method to them caused the bullets to disappear. That's why that style is excluded in the If statement. You may find other styles that have to be excluded.

stefan blom say:

Following up on Jay's idea, I would create a paragraph style with the desired bullet formatting and then loop through the ListParagraphs collection and apply the bulleted paragraph style to all of those paragraphs (checking first that the paragraph in question is not inside a table cell).

Sub ApplyBulletFormatting()
'Macro created by MVP Stefan Blom, March 2017
Dim p As Paragraph
For Each p In ActiveDocument.ListParagraphs
If Not p.Range.Information(wdWithInTable) Then
p.Style = "My Custom Bullet Para Style"  'Specify the name of YOUR bulleted paragraph style
End If
Next p
End Sub

stefan blom say:

One possibility is that the macro will run better with a For ... Next loop instead of a For Each ... Next loop, which may be a bit slower.

See if the following works better:

Sub ApplyBulletFormatting()
 'Macro created by MVP Stefan Blom, March 2017
 Dim i As Long
 Dim ParaCount As Long
 Dim p As Paragraph
 ParaCount = ActiveDocument.ListParagraphs.Count
 For i = ParaCount To 1 Step -1
 Set p = ActiveDocument.ListParagraphs(i)
 If Not p.Range.Information(wdWithInTable) Then
 p.Style = wdStyleNormal  'First apply Normal style to clear the current list format
 p.Style = "My Custom Bullet Style"  'Specify the name of YOUR bulleted paragraph style
 End If
 DoEvents     'Suggested change by MVP Jay Freedman
 Next i
End Sub

jay freedman say:

Something else that should help is to insert the statement

DoEvents

just before the Next statement. That allows Word and Windows to respond to any requests that were blocked while the macro was working on the current paragraph. If too many requests pile up while the loop is running, that can cause a crash.

If you have a <Suggested Contacts> folder in your folder tree and using Office 365, it just means that this involves a PST file used in Outlook '2010 prior to upgrading to Office 365. The <Suggested Folder> is no longer used in later versions of Outlook but that doesn't remove the <Suggested Contacts> folder if it exists.

The <Suggested Contacts> folder cannot be removed from within Outlook since that folder was set as a system level Outlook folder when it was created and as such cannot be deleted (system level = same as <Inbox>, <Contacts> folders etc)

Possible Bug With MS Office Excel 2016 - Ctrl Z - Microsoft Community

fredmontgomery_nwi say:

Possible Bug With MS Office Excel 2016 - Ctrl Z

Create 2 Separate Environments for Excel and a blank workbook in each environment. 

In Excel Environment #1 - Workbook #1, enter random data into cells, delete some data, add some more data.

In Excel Environment #2 - Workbook #1, enter random data into cells, delete some data, add some more data. And other things, you may want to do.

Now check the drop down button at the top of the screen for undo.  You will see recorded actions for both workbooks, NOT just the workbook you are currently in.

This is a problem. 

In Excel 2010, the Undo collection is correct for each workbook.

In Excel 2016, the Undo collection records actions for both environments.

So, if you do a Ctrl -Z-, in one workbook, it will do actions in both workbooks, and you may not be aware of it.
