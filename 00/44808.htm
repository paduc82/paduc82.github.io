techie007l say:

    Next, you need to boot your computer from the USB drive.  Insert the USB drive and turn on your computer.  Immediately, start rapidly pressing [F12] repeatedly.  This is the BIOS boot menu shortcut key on many computers.  Some newer HPs use [Escape] instead, and will present you with a menu of options where you would need to select an option like "Boot Menu" or "Startup Menu" or "Boot Device" or "Boot Manager".

    When the boot menu appears, you should see options like "SATA0: HDD", "SATA1: CD/DVD", "USB storage".  Choose the USB option.  Your computer should boot to a Windows Setup screen.  If you can't get the boot menu to appear, or can't find the USB boot option when it does appear, or your computer just boots normally, I will need the make and model of your computer (e.g. HP ProBook 6455B).  Alternately, you can try Googling your computer's make and model followed by "boot from usb" and see if you can figure it out on your own.

    Once you have successfully booted from the USB drive:

  1. On the Windows Setup screen, select your language preferences and click [Next].
  2. On the next screen, it will give you the options: Install now and Repair your computer.  Rather than choosing any of them, press [Shift]+[F10].  Some laptops reassign the function keys only to work properly when [Fn] is pressed, so you may have to press [Shift]+[Fn]+[F10].  A black command window should open.
  3. Type the following command and press [Enter]: regedit
  4. Registry Editor should open.  Click File -> Import...
  5. In the dialog that appears, Double-click This PC.
  6. Scroll down to the Devices and drives section, and try to identify your main C: drive.  Usually it will have a letter other than C:.  You're looking for a large disk (like >100GB).  Chances are that it will be the largest disk in the list.  Ignore the small ones with labels like "System Reserved" and  "Boot".  Most likely, the one you're looking for will be named "Local Disk" and will have a letter of D: or E:.  Double-click each candidate to see what's inside.  The correct one should have folders named PerfLogs, Program Files, Program Files (x86) (only if your machine is 64-bit), Users, and Windows (there may be other folders too).  If it doesn't look right, back out and try another one.
  7. Once you have figured out the correct one, remember its letter, close the Import dialog, and close Registry Editor.
  8. Back in the command window, type that drive's letter followed by a colon, and press [Enter].  In my case, this was: D:
  9. The command prompt should change to a D:\>_ prompt (again, with the correct letter for your computer).
  10. Type the following command and press [Enter]: cd windows\system32
  11. The command prompt should change to D:\Windows\System32>_ (again, with the correct letter for your computer).
  12. Type the following command and press [Enter]: ren sethc.exe -sethc.exe
  13. Nothing should happen.  If you get an error message, chances are that you selected the wrong disk.  You can go back to step 3 and figure again, or if you have another likely drive letter in mind, go back to step 8 and try it.
  14. Type the following command and press [Enter]: copy cmd.exe sethc.exe
  15. It should inform you that one file was copied.  If you got no error messages, you're successfully completed the hardest part!  Remove your USB drive, close the command window, close the Windows Setup window, and confirm that you wish to restart your computer.
  16. Let your computer restart normally.  When the logon screen appears and is ready, press the spacebar to dismiss the lock screen, and then press the left [Shift] key five times in a row.  A command window should open.  If nothing happens, try pressing the [Shift] key five more times.
  17. In the command window, type the following command and press [Enter], replacing "MyUsername" with the logon username you wish to use (it must be different from the one that you can't login to, and be sure to keep the quote marks): net user /add "MyUsername" 1234
  18. Type the following command and press [Enter], replacing "MyUsername" with the same username you chose in the previous step: net localgroup administrators "MyUsername" /add
  19. Type the following command and press [Enter]: taskkill /f /im logonui.exe
  20. The logon screen should flash, and the new user account should appear in the lower left side of the logon screen.  Click on it and login.  Its password will be 1234, and you can easily change it later by pressing [Ctrl]+[Alt]+[Delete] and clicking Change a password.

    At this point, the computer should be usable again.  There are several more things you'll need/want to do, like restoring your files to the new user account, and the re-securing your computer by undoing the trick we did earlier to gain access.  Make sure that Windows 10 is working properly, let me know how things are going, and depending on what's needed, I'll give you the steps for the rest in another post.

Hi,

The attached DMP file is of the DRIVER_POWER_STATE_FAILURE (9f) bug check.

This bug check indicates that the driver is in an inconsistent or invalid power state.

If we run an !irp on the 4th parameter of the bug check (blocked IRP address), we get the following:

>[ 16, 2]   0  1 ffffe0002ccda8e0 00000000 00000000-00000000    pending
           \Driver\usbccgp

^^ USB Common Class Generic Parent driver (not the true cause).

Please enable Driver Verifier:

Driver Verifier:

What is Driver Verifier?

Driver Verifier is included in Windows 8/8.1, 7, Windows Server 2008 R2, Windows Vista, Windows Server 2008, Windows 2000, Windows XP, and Windows Server 2003 to promote stability and reliability; you can use this tool to troubleshoot driver issues. Windows kernel-mode components can cause system corruption or system failures as a result of an improperly written driver, such as an earlier version of a Windows Driver Model (WDM) driver.

Essentially, if there's a 3rd party driver believed to be at issue, enabling Driver Verifier will help flush out the rogue driver if it detects a violation.

Before enabling Driver Verifier, it is recommended to create a System Restore Point:

Vista - START | type rstrui - create a restore point
Windows 7 - START | type create | select "Create a Restore Point"
Windows 8/8.1 - http://www.eightforums.com/tutorials/4690-restore-point-create-windows-8-a.html

How to enable Driver Verifier:

Start > type "verifier" without the quotes > Select the following options -

1. Select - "Create custom settings (for code developers)"
2. Select - "Select individual settings from a full list"
3. Check the following boxes -
- Special Pool
- Pool Tracking
- Force IRQL Checking
- Deadlock Detection
- Security Checks (Windows 7 & 8)
- DDI compliance checking (Windows 8)
- Miscellaneous Checks
4. Select  - "Select driver names from a list"
5. Click on the "Provider" tab. This will sort all of the drivers by the provider.
6. Check EVERY box that is NOT provided by Microsoft / Microsoft Corporation.
7. Click on Finish.
8. Restart.


- If Driver Verifier finds a violation, the system will BSOD. To expand on this a bit more for the interested, specifically what Driver Verifier actually does is it looks for any driver making illegal function calls, causing memory leaks, etc. When and/if this happens, system corruption occurs if allowed to continue. When Driver Verifier is enabled, it is monitoring all 3rd party drivers (as we have it set that way) and when it catches a driver attempting to do this, it will quickly flag that driver as being a troublemaker, and bring down the system safely before any corruption can occur.

- After enabling Driver Verifier and restarting the system, depending on the culprit, if for example the driver is on start-up, you may not be able to get back into normal Windows because Driver Verifier will detect it in violation almost straight away, and as stated above, that will cause / force a BSOD.

If this happens, do not panic, do the following:

- Boot into Safe Mode by repeatedly tapping the F8 key during boot-up.

- Once in Safe Mode - Start > Search > type "cmd" without the quotes.

- To turn off Driver Verifier, type in cmd "verifier /reset" without the quotes.
・    Restart and boot into normal Windows.

If your OS became corrupt or you cannot boot into Windows after disabling verifier via Safe Mode:

- Boot into Safe Mode by repeatedly tapping the F8 key during boot-up.

- Once in Safe Mode - Start > type "system restore" without the quotes.

- Choose the restore point you created earlier.

-- Note that Safe Mode for Windows 8/8.1 is a bit different, and you may need to try different methods: 5 Ways to Boot into Safe Mode in Windows 8 & Windows 8.1

How long should I keep Driver Verifier enabled for?

I recommend keeping it enabled for at least 24 hours. If you don't BSOD by then, disable Driver Verifier. I will usually say whether or not I'd like for you to keep it enabled any longer.

My system BSOD'd with Driver Verifier enabled, where can I find the crash dumps?

They will be located in %systemroot%\Minidump
