scottgem say:

Sure it makes sense. Since it was set to Text it is sorting according to alpha. So first it sorts on the first character, then the second and third and so on. So that's why 2 comes after 102.

The only way to get it to sort numerically is by making it a number. You have a few choices there. 

1) First backup your database, then test this on a copy. Just change the datatype of the field to Number>Long integer. See if it sorts properly with no loss of data.

2) Add a new field as a Number>Long Integer and run an Update query to populate the new field:

UPDATE table SET newfield = VAL(oldfield);

3) Leave the table/field alone and use a Query for your sorting. Use a column like: Alias: Val(fieldname) and sort on that.
