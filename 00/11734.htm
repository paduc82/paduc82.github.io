doug robbins - mvp office apps & services (word) say:

jay freedman say:

That behavior indicates that the two objects you're calling a header and footer are actually anchored in the body of the document, not in the header pane and footer pane respectively, and formatted to position at the top and bottom of the page. That's why putting text or empty paragraphs into the document makes the objects move. To fix it:

  • Go to File > Options > Display and check the box for "Object anchors". This will cause an anchor icon to display in the left margin when an image or object is selected. It's important to know where the object is anchored! Also check the box for "Paragraph marks".
  • Open the template itself, using the File > Open > Browse dialog instead of double-clicking its icon.
  • Open the header pane, either by double-clicking near the top of the page or by clicking Insert > Header > Edit Header.
  • If the "Different first page" setting on the Header & Footer Tools ribbon is unchecked, check it. You should see a dashed line across the page, with a "First Page Header" tag.
  • Close the header pane.
  • Click on the logo/company name object to select it. You may or may not see the anchor icon, but it's to the left of the first paragraph mark in the document body.
  • Cut the object to the clipboard.
  • Reopen the header pane, and paste the object from the clipboard.
  • Depending on how the vendor set the positioning of the object, it may appear in the same place as before. If it's out of place, right-click the object, click Size and Position in the menu, go to the Position tab of the dialog box, and change the settings there.
  • Close the header pane.
  • Repeat with the footer object: Select and cut it, open the footer pane, paste, and (optionally) reposition.
  • Save and close the template, and start a new document based on it. Confirm that the header and footer objects now appear only on the first page, regardless of how little or how much text you add.

If you want the footer object also to appear on all the other pages, you'll need to edit the template again, temporarily add a page break (Ctrl+Enter) to create a second page, copy the footer object from the first page footer, and paste the copy into the footer on the second page. Then delete the page break and save the template.

More background on this is at http://wordfaqs.ssbarnhill.com/AnchorToHeader.htm and http://wordfaqs.ssbarnhill.com/HeaderFooter.htm.

greginmich say:

Controlled Folder Access, Network Protection, and Attack Surface Reduction all require Windows Defender’s real-time protection – so these features are most assuredly not being turned on by third-party AV apps, they’re being turned off by third-party AV apps – right along with Defender’s real-time protection. In Defender’s Limited Periodic Scanning mode, these are the only features that will be disabled:

https://docs.microsoft.com/en-us/windows/security/threat-protection/windows-defender-exploit-guard/windows-defender-exploit-guard

Now I can’t say for sure that there won’t be any genuine issues with Controlled Folder Access – but the primary issue that we’re seeing now is just that users don’t know how to allow an app through Controlled Folder Access when they see the Unauthorized changes blocked notification. The TechNet and Windows IT Pro Center documentation does show us how to do this in the Windows Defender Security Center app:

https://cloudblogs.microsoft.com/microsoftsecure/2017/10/23/stopping-ransomware-where-it-counts-protecting-your-data-with-controlled-folder-access/

https://docs.microsoft.com/en-us/windows/security/threat-protection/windows-defender-exploit-guard/customize-controlled-folders-exploit-guard

But there’s a shortcut that lets us do this without even opening the WDSC; and that shortcut is included in this Windows Support document:

If you see an App is blocked message when you try to use a familiar app, you can simply unblock the app. If this message displays:

  1. Write down or take note of the path of the blocked app.
  2. Select the message, and then select Add an allowed app.
  3. Browse for the program you want to allow access.

https://support.microsoft.com/en-us/help/4012987/windows-10-virus-threat-protection-windows-defender-security-center

When you get an Unauthorized changes blocked notification, just jot down the file path that’s provided there, and then click on the notification – and that will immediately bring up the Allow an app through Controlled folder access window (and remove the notification). If the file path is truncated to the point where you can’t locate the app’s executable file in the Open (file) dialog; then open Event Viewer; navigate to the Windows Defender Operational log; and locate the blocking event (Event ID 1123):

1. Right-click on the Start button and select Event Viewer.

2. Navigate to Applications and Services > Microsoft > Windows > Windows Defender > Operational

3. Filter for (or just look for): Event ID 1123

Controlled Folder Access prevents ransomware from encrypting your personal files, so it’s definitely worth a try. Of course if you have fail-safe backups of everything, then you won’t need to depend on this feature. You’ll find some other tips on how to maximize Defender’s level of protection in this recent discussion:

https://answers.microsoft.com/en-us/protect/forum/protect_defender-protect_scanning-windows_10/antivirus-program-in-windows-10/5858bbe7-6891-44ce-be62-dc047bcc17d2

bob jones aka: cybertaz say:

This is a slightly convoluted process :-)

By default all cells in an excel worksheet have a Locked property applied, but it has no effect unless sheet protection is used. What you first need to do is select the cells which you want to remain editable then go to Format> Cells - Protection to clear the Locked checkbox. Afterward, use the Protect Sheet tool on the Review tab [or go to Tools> Protection> Protect Sheet] to clear the check for Select locked cells as well as make any other preferred settings adjustments. (Note that if you do not use a password anyone can turn the protection off.)

Navigation will be the standard left-to-right, top-to-bottom pattern but only selecting the cells which are not locked. Likewise, the user cannot directly select any locked cells.

See the Excel Support article:
