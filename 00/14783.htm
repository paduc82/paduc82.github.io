Going from MS Word to MS Access and then back again - Microsoft Community

jeremy shaffer say:

Going from MS Word to MS Access and then back again

I use MS Word to make a simple report form that I use everyday.  I would like to be able to fill out this report and have it then stored in MS Access and then be able to use MS Access to then either produce a spreadsheet(which I can do) or for it to be able to regenerate the original report.  One of the MS Word fields can have multiple bullets in it.  (MS Office 2007) Any suggestions?
ken sheridan say: I agree with the other respondents that there is no point in using Word for this.  Access will do all that's required as far as I can see.  However, you appear to be under a number of misconceptions regarding Access and relational databases in general.


Spreadsheets have cells, a relational database has tables in which data is stored as explicit values at column positions in rows.  In fact it was Codd's Rule #1 when he introduced the database relational model in 1970, the Information Rule, that data is stored in this way and in no other way.  Moreover there can only be one value at one column position in one row in a table.  This is required for a table to be in First Normal Form (1NF), whose definition is:


Loosely speaking, in the language of the relational model, a relvar (relation variable) equates to a table, a tuple to a row (record) and an attribute to a column (field).

I'm aware that Access 2010 appears to violate these principles with its multi-valued fields, but they really are only a representation of a normalized model under the skin and are designed for interfacing with SharePoint.


Reports do not store data; they are a means of presenting data.  All data is stored in rows in tables; Codd's Information Rule to which I referred above.  So there can never be any question of data being
Regarding the bullet points, each of these would be represented by a row in a related table.  Taking a very simple example, let's say the database is modelling a questionnaire.  You might have a table Questions with columns QuestionID and Question.  You might have table Respondents with columns RespondentID, FirstName, LastName etc.  There is a many-to-many relationship type between them modelled by another table which I'll call Responses.  If we assume, somewhat unrealistically, that the responses are free text answers rather than fixed values from a multiple-choice list, this table would have columns QuestionID, RespondentID and Response, the first two being the composite primary key of the table.  Each row in this table could be presented as a bullet point in a report which is based on a query which joins the three tables.  The Report would be grouped first by RespondentID, then by QuestionID and in the detail section would be the responses, each text value preceded by bullet, which would simply be a label control.  I should point out that this is a grossly oversimplified model and is purely to illustrate the basic methodology.  The model for a real questionnaire database would be considerably more complex.

ken sheridan say: One correction to  my earlier post.  I said that QuestionID and RespondentID would be the key of the Responses table.  That would be the case if there could be only one response per question per respondent, but as in this case it is assumed that there can be multiple responses per question, the table is in fact all key, i.e. the three columns QuestionID, RespondentID and Response comprise the key.

To determine whether a performance issue or an error message is caused by corruption in the temporary Internet files or in other cached information that is used by Internet Explorer, you must clear cached data. To do this, follow these steps:

Internet Explorer 8

  3. In Delete Browsing History area, click Delete.

Method 2: Reset security settings for Internet Explorer

If you configure security settings to be too restrictive, you may prevent Internet Explorer from displaying certain Web sites. To determine whether an issue is caused by overly restrictive security settings, revert to default security settings. To do this, follow these steps:

  1. Open Internet Explorer.
  2. Click Tools, and then click Internet Options.
  3. Click the Security tab.
  4. Click Reset all zones to default level, and then click OK.

Method 3: Run Internet Explorer in "No Add-Ons" mode

Internet Explorer add-ons, such as ActiveX controls and browser toolbars, are used by some Web sites to provide an enhanced browsing experience.
