Cross-Referencing Two Tables in Access 2010 - Microsoft Community

marialabat say:

Cross-Referencing Two Tables in Access 2010

I have two tables that I am currently working with: Table A contains all of my company's product data, including freight class, which is the information that I am interested in, while Table B contains a list of products that one of our dealers carries and asks for the freight class of each product. Is it possible to cross-reference these tables so that the freight class from Table A will appear in the "Freight Class" column on Table B?

gina whipp say:
Is there a field you could *join* on?  Then you could use a query to get what you want.

scott mcdaniel say:

If you can "relate" your tables, then you can create a query that will pull data from both tables. For example, if TableA contains a column named "ProductID" which contains the Product Number for your products, and TableB contains a field that contains the same value, then you could create a query with a Join:

SELECT * FROM TableA INNER JOIN TableB ON TableA.ProductID=TableB.ProductNumber

This would show you all data from both tables where the values in those two fields are the same. You could then pare down the columns to show only the items you need. 


Note an Inner Join will NOT show any records from either table where there is no match - so if you have a record in TableB that does not match any record in TableA, that record would not be included in the query's output. If you need that sort of information, you can use a LEFT or RIGHT Join.

scottgem say:

Table B does not appear to be designed properly. Can you answer some questions for me. 

First, the products that this dealer carries, can then be carried by any other dealers? Second, do you need to record anything else about these particular products other than what's in your products table? 

One of the principles of a relational database is that data should exist in one table and one table only. So if the freight class already exists in your products table it should NOT exist in Table B.

Can you change the office 365 Group ID or copy all group contents to another group?

Is there any way to change the office 365 Group ID/Email?

If not, is there a way to copy all the contents of one office 365 group to another?

alan wu msft say:
Hi Chunt901,

Yes, please try changing your Office 365 Group’s email address via the following steps: (The following steps should be performed by an admin. You can contact your admin for help if you are not an admin)
1.    Connect to Exchange Online PowerShell with an admin credential.
2.    Run the following cmdlet
Set-UnifiedGroup -Identity group@contoso.com -PrimarySmtpAddress group1@contoso.com
Note： group@contoso.com stands for your original Office Group’s email address and group1@contoso.com stands for your Office 365 Group’s new email address

Regards,
Alan

andre for directly say:
Try uninstall then reinstall Skype:

Press Windows Key+X
Click Windows Powershell (Admin)

Enter the following command then hit Enter

Get-AppxPackage -allusers Microsoft.SkypeApp | Remove-AppxPackage

Open start menu
Type in "powershell"
Right click on "Windows PowerShell"
Run as Administrator

Then paste the following command

get-appxpackage -AllUsers Microsoft.SkypeApp | foreach { Add-AppxPackage -DisableDevelopmentMode -Register "$($_.InstallLocation)\AppXManifest.xml" }

Press Enter

After the install process finishes, you have the Windows 10 Skype app back

Still not working?
