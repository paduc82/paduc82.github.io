scottgem say:
Generally, if I want a sort to "stick", I'll put the field in the Order By property of the form. the form will generally follow the Sort order in the Recordsource, but it may not. 

ken sheridan say: I would recommend that you do not use a saved query as a form's RecordSource property, but its SQL statement.  You can still build the query in design view; just don't save it, but instead simply confirm that you want the property updated when you close the query design window.  That way the RecordSource property will be the SQL statement, not the name of a query.  This way you'll get no interference from the OrderBy property of the querydef object which might inadvertently have been set during the design process.  Why a query has such a property in the first place is difficult to understand.  There'll be no performance loss doing it this way; the SQL statement will still be compiled.
