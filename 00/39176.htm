¡firedog say:

... is [this] a legitimate message from Microsoft?

Yes. I think everyone with a Microsoft Email account set up to use HTTP in Windows Live Mail has received or will receive the same message.

... essentially telling me that the Windows Live Mail service I use (with a @hotmail.com address) will be closed unless I install an update.

That's not really what it says. I think you misunderstood the email, along with most others that received it. Windows Live Mail 2012 won't 'be closed'. The change only involves Outlook.com accounts, which are being upgraded to Outlook Mail. Outlook Mail doesn't support the DeltaSync protocol that Windows Live Mail uses to synchronize mail, contacts and calendar, so the KB3093594 update adds support for another protocol - Exchange ActiveSync, EAS - which the program will then use instead of DeltaSync for upgraded accounts.
 

If you use the contacts list and calendar while you're signed in with a Microsoft Account, you'll have to apply the update once that account is upgraded to Outlook Mail. If you don't normally sign in, you don't have to apply the update. Your POP3, IMAP and NNTP accounts won't be affected at all.

Has anyone else received this message and gone ahead with the update? If so did it work ok?

You don't need to apply the update until your account is upgraded. When it is, you'll start seeing synchronization errors in Windows Live Mail. When that happens, you can check whether the errors are because of the upgrade to Outlook Mail by signing in there with your Hotmail account. The banner at the top of the page is different for an upgraded account:

Banners compared

Did it work OK? Not really. There are bugs in the update which irritate some users more than others; calendar sync doesn't work well, and there are some oddities in contacts sync as well. We're just hoping that most of these will be ironed out sooner or later.
