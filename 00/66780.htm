While in Command Prompt, type chkdsk /f /r and enter and let it run.  It will want to schedule itself to run at the next restart.   Answer yes and then reboot to run the program.  It will scan and try to fix any corruption or bad sectors on your hard drive and mostly remove that as a potential cause. The rest either must be done from safe mode with networking or from a different user account if that works.

 

If that doesn't work, then please post any error messages from the Event Viewer concerning logon or security (Start / Control Panel / Administrative Tools / Event Viewer).  Here's how to use Event Viewer:http://www.petri.co.il/vista-event-viewer.htm.  Look in the System section.

primo_2 say:

I too found that the KB3094186 workaround regularly failed, but the reason was not because the hyperlink was too long, but because it contained special characters such as ampersand (&).

That workaround requires the creation of a Registry Key containing a command such as the following (which is to open the link in a new tab):

cmd /c start %1

The %1 parameter contains the hyperlink to be opened.

The problem with this is that the Windows command interpreter will interpret any characters within that parameter that it recognises as special.  This includes removing ampersand characters and everything that follows it.

To solve this I modified the command to include double quotes around the parameter to stop its contents being interpreted.  Here's the command that is now in my Registry Key named "HKEY_CLASSES_ROOT\htmlfile\shell\opennew\command":

cmd /c start "Hyperlink" "%1"

Note that I also had to provide a Title immediately after the start command, otherwise it would have treated "%1" as the Title and simply opened a command shell (I chose "Hyperlink", but anything within double quotes will do).
