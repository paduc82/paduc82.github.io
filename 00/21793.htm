That is fairly standard except I would do some things differently. First, I would have ONE Items table. If the only difference between labor and parts is time, I would just leave that blank for Parts.

Second, it is not necessary to have the Time and Description fields in tblRepairDetail. I'm guessing this info is static so it can be pulled from the Items table whenever needed. One of the principles of a relational database is that data should be in ONE place only.  However there is an exception to this rule. That is when the info is time sensitive and you need to capture the data as it was at a point in time. The UnitPrice falls into that category.

But it will fill in the value of bound controls. I've written a blog on Displaying Related data, that illustrates some of these techniques.

jp ronse say:

I was, years ago, involved in a thread to sum and subtract acres & guntas and came with a very long & complex formula.

Applying your idea ...

Sum: =DOLLARFR(SUMPRODUCT(DOLLARDE(LARGE(A2:A4,ROW(A2:A4)-1),40)*40)/40,40)

Subtract: =DOLLARFR(ROUND(SUMPRODUCT(DOLLARDE(LARGE(A10:A12,ROW(A2:A4)-1),40)*40)/40,5),40)

Both are standard formulas.

Note the use of ROUND in subtract.

Acres.Guntas Guntas
5.35 235
-2.35 -115
-1 -40
2.00 80

Without the ROUND, the formula returns 1.40. This caused by the floating point error.

=(SUMPRODUCT(DOLLARDE(LARGE(A10:A12,ROW(A2:A4)-1),40)*40)/40)-2-0 =>

-6.66134E-16

Sumproduct returns in fact 1.9999999.... but is rounded up to 2.00000... but DOLLARFR takes the real value and rounds it after the calculation 1.3999999... => 1.40 iso 2.

Here's an example:

Revenue $120,000

Start Jan 1, 2017

End Dec 31, 2017

The output should be $10k per month under each column.

I am open to formatting suggestions but right now I'm thinking Col 1 would be revenue, Col 2 start, Col 3 end and then every column after that would represent the output month/amount (Jan - Dec).

amit tandon say:

Refer below image:

Enter below formula in cell G2 & copy to the right:

=(G$1<=EOMONTH($E2,0))*(G$1>EOMONTH($D2,-1))*($C2/DATEDIF(EOMONTH($D2,-1),EOMONTH($E2,0),"m"))

Presumed that the number of months will be from the month of start date & month of end date (with both months included) so that:

if start date is 3/31/2017 & end date is 5/31/2017 this will be taken as 3 months,

if start date is 3/1/2017 & end date is 5/1/2017 this will be taken as 3 months,

if start date is 3/1/2017 & end date is 5/31/2017 this will be taken as 3 months, 

if start date is 2/28/2017 & end date is 5/31/2017 this will be taken as 4 months.

This seems to be your requirement & if you want otherwise the formula can be updated.
