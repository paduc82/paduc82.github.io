hendersam say:

want to write vba code using File System Object to delete any file that hasn't been modified for 2 months

A folder is used as a dumping ground for csv uploads. I want to write vba code using File System Object to delete any file that hasn't been modified for 2 months. There are currently in excess of 10,000 files in this folder. Any help will be appreciated.

 

Sam

graham mayor say:
The following macro (which should run from Word or Excel) will remove any CSV files in the indicated folder more than 56 days old. The files are deleted permanently and without opportunity to change your mind, so do test it on sample files before committing it to your main folder.

Sub DeleteOldCSVs()
Dim fs As FileSystemObject
Dim f As Object
Dim strFileName As String
Const strPath As String = "D:\Test\" 'The folder with the files
Set fs = CreateObject("Scripting.FileSystemObject")
strFileName = Dir$(strPath & "*.csv")

While Len(strFileName) <> 0
    Set f = fs.GetFile(strPath & strFileName)
    If f.DateLastModified < Now - 56 Then Kill strPath & strFileName
    strFileName = Dir$()
Wend
Set f = Nothing
Set fs = Nothing
End Sub

karl timmermans [outlook mvp 2012-2018] say:

Not sure what you mean exactly by "........for each email account imported to Outlook".


That said, IMap accounts will maintain their own infostore (which you can't change) whereas POP accounts can be directed to use a single PST file.


In short, you can't have Imap and POP accounts going to a single set of folders (infostore) and as for whether or not it's better to have a different PST file for each POP account - that's a personal choice.


Note if you configure a POP account first and set it as the default, any subsequent Imap accounts configured are going to use those Contact/Calendar folders. If you configure an IMap account first in O'2010, a separate PST file will be created for contacts/calendar.
