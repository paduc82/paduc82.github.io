hansv mvp say:

In the first place, since you have a line On Error Resume Next, errors occurring after that line won't be reported. I'd remove that line.

In the second place, xlVeryHidden is not a property of a worksheet, it is a possible value of its Visible property. A line such as

            Worksheets("Sheet6").xlVeryHidden = True

should be

            Worksheets("Sheet6").Visible = xlVeryHidden

or officially

            Worksheets("Sheet6").Visible = xlSheetVeryHidden

In the third place, your Select Case statement uses a variable Sections that it neither declared nor set, so it will be empty, and none of the cases match.

Perhaps you meant this:

Sub WorkbookOpen()
    Dim UName As String
    UName = Environ("UserName")
    Select Case UName
        Case "jtsmith"
            Worksheets("Sheet1").Visible = xlSheetVisible
            Worksheets("Sheet2").Visible = xlSheetVisible
            Worksheets("Sheet3").Visible = xlSheetVisible
            Worksheets("Sheet4").Visible = xlSheetVisible
            Worksheets("Sheet5").Visible = xlSheetVisible
            Worksheets("Sheet6").Visible = xlSheetVisible
            Worksheets("Sheet7").Visible = xlSheetVisible
            Worksheets("Sheet8").Visible = xlSheetVisible
        Case "name2", "name3", "name4", "name5", "name6"
            Worksheets("Sheet1").Visible = xlSheetVisible
            Worksheets("Sheet2").Visible = xlSheetVisible
            Worksheets("Sheet3").Visible = xlSheetVisible
            Worksheets("Sheet4").Visible = xlSheetVisible
            Worksheets("Sheet5").Visible = xlSheetVisible
            Worksheets("Sheet6").Visible = xlSheetVeryHidden
            Worksheets("Sheet7").Visible = xlSheetVeryHidden
            Worksheets("Sheet8").Visible = xlSheetVisible
        Case "name7"
            Worksheets("Sheet1").Visible = xlSheetVisible
            Worksheets("Sheet2").Visible = xlSheetVisible
            Worksheets("Sheet3").Visible = xlSheetVisible
            Worksheets("Sheet4").Visible = xlSheetVisible
            Worksheets("Sheet5").Visible = xlSheetVisible
            Worksheets("Sheet6").Visible = xlSheetVisible
            Worksheets("Sheet7").Visible = xlSheetVeryHidden
            Worksheets("Sheet8").Visible = xlSheetVisible
        Case Else
            MsgBox "You are not authorized to use this Workbook"
            ThisWorkbook.Close SaveChanges:=False
    End Select
End Sub

Please note that this macro will not be executed automatically when the workbook is opened. If you want it to be run when the workbook is opened, call it from the Workbook_Open event procedure in the ThisWorkbook module.

steve rindsberg say:
If you insert object and choose Link, or use a hyperlink, then the object is NOT embedded.  It's linked, and the file that contains it must be distributed along with the PPT that contains the link.

If you insert an object and do NOT choose Link, the object will be embedded ... meaning that the entire contents of the file will be included in the PPT/PPTX file, and recipients of the PPT/PPTX will be able to launch the embedded file by double-clicking it (assuming that they have the necessary application installed on their PC).

steve rindsberg say:

>> I believe what you are saying is, PDF links can't be embedded in PPTs unless you send the PDFs with the PPT presentation? Which undermines the purpose of PDF links?

No.

By their nature, LINKS are not embedded and EMBEDDED files are not links.  

Links are simply bits of computer "stuff" that point to the location of the linked file on your computer so PowerPoint can find and launch the file when you click the link.  Obviously, for these to work, the file must be where it's supposed to be and must accompany the PowerPoint file if you move it to another computer.

An embedded file is literally that:  EMBEDDED inside the PowerPoint file.  The PPT file doesn't need to carry any other files with it.

Each has its advantages and drawbacks.

Change style of all footnote and endnote references at once?

I'm working on a long document with hundreds of footnotes and endnotes. The formatting for the references (superscript) keeps getting changed to "Normal" for some reason.

Instead of doing every single one by hand, how can I select and change ALL my endnote references to my "endnote reference" style, and do the same with my footnotes and "footnote reference" style?  


OS High Sierra 10.13.4

Word 16.13.1

macropod say:

You could use a macro like the following:

Sub Demo()
Application.ScreenUpdating = False
Dim i As Long
With ActiveDocument
  For i = 1 To .Footnotes.Count
    With .Footnotes(i)
      .Reference.Style = "Footnote Reference"
      With .Range
        .Collapse wdCollapseStart
        .Previous.Words.First.Style = "Footnote Reference"
      End With
    End With
  Next
  For i = 1 To .Endnotes.Count
    With .Endnotes(i)
      .Reference.Style = "Endnote Reference"
      With .Range
        .Collapse wdCollapseStart
        .Previous.Words.First.Style = "Endnote Reference"
      End With
    End With
  Next
End With
Application.ScreenUpdating = True
End Sub

For PC macro installation & usage instructions, see: http://www.gmayor.com/installing_macro.htm
For Mac macro installation & usage instructions, see: https://wordmvp.com/Mac/InstallMacro.html

Yes, your thoughts are completely correct.
