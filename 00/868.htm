Most of the time (in my opinion) there's no real reason to store the input mask in the table -- why bother storing characters that cannot be changed?  However, if you want to store the input mask characters with the field data, the Input Mask property has a second, optional part that may be specified after the mask itself, separated by a semicolon.  If that second part is present and specified as 0, the literal characters of the mask will be stored in the table.

For more information about the extra options of input masks, see the help topic on "Input Mask Property".  Or probably if you use the builder for the Input Mask property of the field, it will guide you through a series of questions that include deciding whether or not to save the literal characters with the value.
