Short Answer: No

The .ost data file associated with the Outlook.com account is a mirror of what is located on the server. If the Outlook.com account is removed from Outlook, then the .ost file is also deleted.

#1 - If you don't have the Exchange account configured that's associated with Bus Essentials, then

c) "copy" (with emphasis on "Copy" as opposed to "Move") the contacts from the Outlook.com contacts folder to the one just created

bob jones aka: cybertaz say:

PMFJI, but perhaps I can help preserve some sanity :-)

The Size device on the Layout tab of the Ribbon has been unreliably buggy since Word 2011. Typically it will make the correct change to the document when a selection is made there but will not correctly reflect the actual Paper Size of the document.

In order to confirm and/or set the correct Paper Size, go to the source rather than the UI control:  Format> Document, then click the Page Setup button there. Once you've made you choice click OK to return to the previous dialog window. If the size is to be used as the default for new documents, click the Default... button there, then click OK and confirm updating the Normal template.

Don't be surprised if the Size list on the Ribbon indicates something other than what you selected but that, in fact, is immaterial. The correct size will be used for both display & output. Also, Borderless is irrelevant unless the document has content that extends outside the Margins (or possibly if the Margins set are smaller than the minimum requirement of the printer). Its primary purpose is for photo printing. Content outside the Margins usually will not Print/PDF unless Borderless is specified. If there is no such content it makes no difference whether or not Borderless is indicated.
