You can use the Data/Text to columns  wizard.
 
Most likely, the token in the cell that causes the phone number to be on the second line is a <CR> (carriage return).  That being the case:
 
1.  Select the cells that contain the data to split (I assume it is in a single column).
 
2.    Data / Text to Columns
    Delimited
    In Step 2 you select the delimiters.  Select the "Other" option and, with the cursor in the box next to it,  while holding down the <ALT> key, type the three digits  010 on the numeric keypad (do NOT use the numbers at the top of the regular keyboard).
    <Finish>
 
If that doesn't work, we will have to determine what the delimiter is between the email address and the phone number.

It's not supposed to "take any digits" as I understand that.
 
By holding down <ALT> while typing 010 (one at a time)  on the numeric keypad, and then releasing the <ALT> key, you generate the <CR> character, which is a single character that as the ASCII code of 10.  But you won't be able to see anything in that box, since the <CR> character is not usually visible.
 

peggyshepard say:

If the phone number is consistently 10 digits separated by hyphens between the 3rd and 4th/6th and 7th numbers and is at the end of the cell content, the following will remove nonprinting characters such as returns that cause the text to go to the 2nd line, and will also remove any spaces after the last character.

Assuming that the data is in column A, starting in cell A1 - enter the following in cell C1, and copy the formula down.

=IF(LEFT(RIGHT(CLEAN(TRIM(A1)),5),1)="-",RIGHT(CLEAN(TRIM(A1)),12),"")

enter the following in cell B1, and copy the formula down.

=left(clean(trim(a1)),len(clean(trim(a1))-len(c1))

Windows Defender cannot be used along with MSE. MSE includes the protection of Defender and adds virus protection. On XP, Defender must be removed. On Vista and 7, Defender is disabled.

 

The reality is that nothing can protect a computer fully from threats. XP is harder to protect. Windows security has been improved considerably in later versions. -Make sure that the Windows Firewall is enabled.

Is there any way I can detect these tables so as to be able to work with them?

oren_882 say:

I found a way to do it by entering to TopLevelTables first, this way:

                  

Sub tables_test()

Dim Otbl As Table
Dim Itbl As Table

For Each Otbl In ActiveDocument.Range.TopLevelTables
    For Each Itbl In Otbl.Tables

        Itbl.Select

    Next
Next

End Sub

andreas killer say:

if anyone has a different way to do it, please share.

Sub Example_AllTables()
  Dim T As Table
  For Each T In AllTables
    Debug.Print T.NestingLevel
  Next
End Sub

Function AllTables(Optional ByVal D As Document) As Collection
  'Return all tables of document D in one collection
  Dim Stack As New Collection
  Dim T As Table
  Set AllTables = New Collection
  If D Is Nothing Then Set D = ActiveDocument
  If D Is Nothing Then Exit Function
  Stack.Add D.Tables
  Do While Stack.Count > 0
    For Each T In Stack(1)
      AllTables.Add T
      If T.Tables.Count > 0 Then Stack.Add T.Tables
    Next
    Stack.Remove 1
  Loop
End Function
