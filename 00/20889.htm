jay freedman say:
In the Sharing Building Blocks part of http://blogs.office.com/b/microsoft-word/archive/2009/02/03/managing-and-administrating-building-blocks.aspx, you'll find a registry entry that can be used to point to a template in a publicly accessible folder on a network share. The SharedDocumentParts entry goes into the key HKEY_CURRENT_USER\Software\Microsoft\Office\12.0\Common\General (or replace the 12.0 with 14.0 for Word 2010, or with 15.0 for Word 2013).

Each user who needs access to the shared template will need to have the registry entry placed in the registry on their computer. That could be accomplished by creating a .reg file that can be emailed to each user with instruction to double-click it and follow the prompts. However, it would probably be better to ask the IT department to make it part of the network login script.

ken sheridan say: If the form is opened in normal mode you can do so from the code inn the buttons' Click event procedures:

     DoCmd.OpenForm "NameOfParentForm"
     
     With Forms("NameOfParentForm")
        .Revisado.Enabled = False
        .Revisado.Locked = True
        .[Revisado Por].Enabled = False
        .[Revisado Por].Locked = True
        .[Revisado En].Enabled = False
        .[Revisado En].Locked = True
   End With

     DoCmd.OpenForm "NameOfParentForm", OpenArgs:=Me.ListBox_ComFinMemberNames
     
     With Forms("NameOfParentForm")
        .Revisado.Enabled = False
        .Revisado.Locked = True
        .[Revisado Por].Enabled = False
        .[Revisado Por].Locked = True
        .[Revisado En].Enabled = False
        .[Revisado En].Locked = True
    End With

     DoCmd.OpenForm "NameOfParentForm", OpenArgs:=Me.ListBox_RevisarTrabajoDe
     
     With Forms("NameOfParentForm")
        .Revisado.Enabled = True
        .Revisado.Locked = False
        .[Revisado Por].Enabled = True
        .[Revisado Por].Locked = False
        .[Revisado En].Enabled = True
        .[Revisado En].Locked = False
    End With

By using both the Enabled and Locked properties the user cannot then move focus to the control when it is locked.

If this were to be done when opening a form in dialogue mode it would not be possible to do it in the buttons' procedures, but multiple arguments can be passed to the form via the OpenArgs mechanism.  You'll find a module which enables you to to do this in Args.zip in my public databases folder at:

https://onedrive.live.com/?cid=44CC60D7FEA42912&id=44CC60D7FEA42912!169

Note that if you are using an earlier version of Access you might find that the colour of some form objects such as buttons shows incorrectly and you will need to  amend the form design accordingly.  

If you have difficulty opening the link, copy the link (NB, not the link location) and paste it into your browser's address bar.

You could then pass an additional named or numbered argument to the form, e.g. "LockControls" or "UnlockControls", and conditionally set the Enabled and Locked properties of the controls on the basis of the argument's value, again in the form's Open event procedure in addition to assigning a value to the hidden text box.

ken sheridan say: I don't think you need to post a file online for this.  It should be possible to use a single query which references a hidden unbound text box control in the parent form:

SELECT [Created By], [Dia Contado], Entrada, Cantidad, Proposito,
[Hacer Cheque], [Fecha de Cheque], [Num de Cheque],
[Cheque Digirido A], Notas, Revisado, [Revisado Por],
[Cheque Hecho Por], [Fecha Cheque Creado], [Revisado En],
[Created On], [Last Updated On], [Last Updated By], Documentos.FileData,
Documentos.FileName, Documentos.FileType
FROM tbl_Ofrendas
WHERE ([Created By] = [Forms]![NameOfParentForm]![NameOfHiddenTextBox]
OR [Forms]![NameOfParentForm]![NameOfHiddenTextBox] IS NULL)
ORDER BY [Created On];

When you open the parent form then for 'All' you don't need to pass anything to the form, so the code is simply:

    DoCmd.OpenForm "NameOfParentForm"

For 'My' you'd pass the value of the list box control:

    DoCmd.OpenForm "NameOfParentForm", OpenArgs:=Me.ListBox_ComFinMemberNames

For 'Other' you'd pass the value of the RevisarTrabajoDe control:

    DoCmd.OpenForm "NameOfParentForm", OpenArgs:=Me.ListBox_RevisarTrabajoDe

In the parent form's Open event procedure assign the value to the hidden text box control and requery the subform control:

    If Not IsNull(Me.OpenArgs) Then
        Me.NameOfHiddenTextBox = Me.OpenArgs
        Me.NameOfSubFormControl.Requery
    End If

NameOfSubFormControl should be the name of the control in the parent form which 'houses' the subform.  This might or might not be the same as the name of its source form object.
