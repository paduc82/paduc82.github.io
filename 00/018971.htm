tom van stiphout say:

I don't know how to say this delicately, but your table design needs work. Without the proper design everything else becomes that much harder.

You should not have a table name and a field name with the same name (like your Rep ID). How about tblEmployees for the table name? Btw we recommend not to use spaces and other funny characters in object names.

You should not name a field "Rep ID" and then store a rep name in it. By convention, the "ID" part refers to a number, likely an Autonumber. Such IDs are often used to tie tables together in a relational database design.

The standard design for an employee table with managers is:
tblEmployees
EmployeeID autonumber primary key
FirstName text20, required
LastName text30, required
ManagerID long integer, required  (Note: the top boss has herself as manager)

Then create a self-join in the Relationships window, and enforce it.

There is no need to record the manager in tblErrorTracking: it can be derived from tblEmployees. Only exception is if managers change a lot, and you want to record the manager AT THAT TIME.

Maybe before you build your db design, let alone your app, you need to study these concepts a bit more. Here is an interesting book to get started:http://www.wiley.com/WileyCDA/WileyTitle/productCd-0470525746.html

john w. vinson say:

I fully agree with Tom's constructive criticism of your design. You've been lucky, but this will eventually get you in trouble!

That said... you can put some VBA code in the AfterUpdate (not BeforeUpdate) of cmbRepID to "push" the rep's manager into the field: just use DLookUp("[Mgr]", "[Rep ID]", "[Rep ID].[Rep ID] = """ & Me!cmbRepID & """") (assuming that Rep ID in fact contains a name).

To have the Rep ID combo box dependent on the Manager combo box, you don't need a filterbyselection at all; instead simply set the Rep ID combo's Rowsource to a Query referencing the manager combo box as a criterion:

Select [Rep ID].[Rep ID] FROM [Rep ID] WHERE [Rep ID].[Mgr] = [Forms]![YourFormName]![cmbMgr] ORDER BY [Rep ID];

Requery cmbRepID in the AfterUpdate event of cmbMgr and you'll be in.
