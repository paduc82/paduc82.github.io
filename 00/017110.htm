ankitchatorikar say: How to do that if i am having legacy mode

don cuthbert say:

The current standard Windows 10 install on an MBR disk (Legacy BIOS) has a 500MB combined System and Recovery NTFS partition, followed by a single partition for Windows and everything else. If you boot with Windows install on a USB stick or DVD, you can get a Command Prompt by hitting SHIFT-F10. Assuming that you have the 500MB partition at the beginning of the hard disk, you can run Diskpart from the Command Prompt.

In Diskpart, list volume will show you what volumes exist. See if it is setup as I said above. Check what drive letters are assigned. Sometimes the Recovery partition gets drive C: and Windows ends up on drive D: at this point. It doesn't matter, as long as you figure out which drive letter points to each one.

In Diskpart, select disk 0 and then list partition to see the partitions and then select the first one select partition 1. Then type active to ensure that it is marked Active.

You might be able to fix it just by entering a couple of commands outside of Diskpart, if the recovery files are still in the System partition. Type exit to leave Diskpart.

bootsect /nt60 ALL /force /mbr

bcdboot C:\Windows <-- use D:\Windows if Windows is on D: at the moment.

Then restart the computer and see if it works now.

If not, sometimes you might have to delete and recreate the System/Recovery partition and copy winre.wim back to it. If the only copy of winre.wim is on it now, you might have to move it to the Windows drive while you recreate the System partition.
