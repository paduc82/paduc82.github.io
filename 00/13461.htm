joeu2004 say:

Jack wrote:

I want to type a formula in a cell above that column that produces the maximum value expressed in one of the column's cells as long as that value is less than 1.0.  How do I do that?

If you mean "the maximum, but no more than 1", normally-enter the following (just press Enter as usual):

=MIN(1, MAX(A2:A100))

If you mean "the maximum, but blank if more than 1", normally-enter the following:

=IF(MAX(A2:A100) > 1, "", MAX(A2:A100))

If you mean "the maximum of only values less than 1", see Gord's formula.

But that formula returns zero if no cell meets the requirement.

If you want the formula to appear blank in that case, either format the cell as Custom [=0]"";General if an actual value of zero is acceptable, or if the actual value should be the null string, array-enter the following formula (press ctrl+shift+Enter):

=IF(COUNTIF(A2:A100, "<1"), MAX(IF(A2:A100 < 1, A2:A100)), "")

Jack wrote:

Similarly, I want to type a formula for a different column that produces the minimum value expressed in one of the column's cells that is greater than 0.0.  How do I do that?

Similarly:

if you mean minimum, but no less than zero:

=MAX(0, MIN(A2:A100))

or if you mean if minimum, but blank if less than zero

=IF(MIN(A2:A100) > 0, "", MIN(A2:A100))

or if you mean minimum of only values greater than zero:

Gord's formula, optionally formatted as Custom [=0]"";General

or

{ IF(COUNTIF(A2:A100, ">0"), MIN(IF(A2:A100 > 0, A2:A100)), "") }

Formulas with curly braces ("{...}") are array-entered (press ctrl+shift+Enter).  Do not type the curly braces yourself.

rob koch say:

CFA is not a trusted application (e.g. whielisting) security system.

Instead, it's purpose is merely to block access to certain folders that ransomware most often attacks when attempting to encrypt and corrupt user files.  This means that any application trying to access these specific folders will also be blocked, regardless of who created the application or how trusted it might be.

So the question you need to ask about those particular Excel files is where they are stored and possibly what other folders or files they might access, since this is what's actually triggering CFA and not Excel or Word itself.

You can choose to allow an application access to specific folders if you trust it, but the opening you are creating is that the application itself is now a pathway into that particular folder, so if it becomes compromised it can be used to attack the files contained in that location.
