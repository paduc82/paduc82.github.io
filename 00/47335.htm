My workaround for this is to have a separate 'audiophile'  (standard) user account and I keep the wav. format files there. I can access this account from my main account to play the files so there is no need to keep switching users. You'll just need to open the second account if you wish to rip music to it. That way the library in your main account does not get messed up with all the 'unknown' stuff.

Plan B would be to ditch wav. and use WMA lossless - it depends how good your ears are! - if there is a difference it is minimal unless you are running your pc through seriously high-end equipment. WMA lossless sounds pretty good through my Marantz/Mission set-up.

a. user say:

Hi,

I will be glad to help you with the issue you have turning on Network Discovery on private network.

To turn on network discovery
1. Open Advanced sharing settings by clicking the Start button , and then clicking Control Panel. In the search box, type network, click Network and Sharing Center, and then, in the left pane, click Change advanced sharing settings.
2. Click the chevron to expand the current network profile.
3. Click Turn on network discovery, and then click Save changes.  If you're prompted for an administrator password or confirmation, type the password or provide confirmation.

Make sure that the following services are enabled on the computer.

a) Type "services.msc" in the search and click on services. Check if these services are enabled:

- TCP/IP NetBIOS Helper service

- DNS Client

- Function Discovery Resource Publication

- SSDP Discovery

- UPnP Device Host

To start the service and set it to automatic, follow these steps:

a) Right-click each of the services listed above and click on Properties.

b) Click the General tab, and then, next to Startup type, select Automatic.

c) Click Apply, and then click Start.
