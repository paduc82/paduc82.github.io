mel77084 say:

By default, the build sets your system for fast startup, and that means that when you shut it down, it will turn off the monitor, and then spend the next 30 seconds to a minute or so getting things ready for the fast start. Once that is done, it finishes shutting down. During that time, you will see evidence (such as hard drive activity) that it is still on.

You can eliminate that behavior by turning off fast startup.

If yours stays on longer than described above, you might have a wrong setting in Settings, Power & Sleep, Advanced Power Settings.

jay freedman say:
Starting in Word 2007, the Chart feature in Word was changed substantially. The Chart object is no longer an OLE object, at least as far as VBA is concerned. Instead, it is simply one of the many types of InlineShape -- in this case, ActiveDocument.InlineShape(1).Type returns 12, or the constant wdInlineShapeChart from the WdInlineShapeType collection.

You can use code like

Sub test()
    Dim is1 As Word.InlineShape
    Dim oChart As Chart
   
    Set is1 = ActiveDocument.InlineShapes.AddChart()
    Set oChart = is1.Chart
   
    With oChart
        .SetSourceData Source:="='Sheet1'!$A$1:$D$5"
        .ChartArea.Shadow = True
    End With
End Sub

Ta Da, a solution.  Not what you would expect.  Turns out the problem is with Outlook using the MAPI over HTTP protocol.  Something in the recent updates borked MAPI/HTTP.  The MS Tech put in a registry key to disable MAPI/HTTP.  We restarted outlook, did a repair on the exchange account (File | Account Settings | Highlight exchange account and hit repair) and restarted outlook again.  BANG.  Works normal again.  So for those of you who want to try it here is the key that you need to add to the registry on the local machine:

HKCU\Software\Microsoft\Exchange

Create a reg DWORD called MapiHttpDisabled

Set it to 1

If you want a .reg file that you can just double click to install (or let users install) just create a txt file called OulookFix.reg

Put the following text into the file:

    Windows Registry Editor Version 5.00

    [HKEY_CURRENT_USER\Software\Microsoft\Exchange]

    "MapiHttpDisabled"=dword:00000001

Save.  You put this file on the local machine and double click it to import to the registry.  

Restart outlook, make sure that you repair the exchange account.  Restart outlook again.  

Please try it and see if the issue can be resolved.

If the issue persists, I want to know whether you have a hybrid deployment and if yes, what's the version of your on-premises Exchange server.
