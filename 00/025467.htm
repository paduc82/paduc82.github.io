Note that if you are using an earlier version of Access you might find that the colour of some form objects such as buttons shows incorrectly and you will need to amend the form design accordingly.  

If you have difficulty opening the link copy its text (NB, not the link location) and paste it into your browser's address bar.

This little demo file imports data from Excel into a single 'holding' table, and then decomposes it into a set of correctly normalized predefined operational tables.  This is done by a series of 'append' queries.  The demo takes you through these step by step, with a brief explanation of each.  The order of execution of the queries is crucial.  In essence, data must first be inserted into the referenced tables before inserting rows into the referencing tables.

While my demo uses quite a simple model, the same principles apply to more complex models.  It is of course very important that the target tables are each fully normalized and that their keys are correctly defined.  It is also important that all relationships are enforced.

With any append operation like this the consistency of the imported data is important.  I once found myself recorded in one old 'flat-file' database as author of technical articles in my own field of work in three separate ways, K Sheridan, K W Sheridan and K V Sheridan (the last an error).  When the data was imported into a relational database I was represented as three separate people until the Authors table and its referencing table were manually corrected (the latter first of course).
