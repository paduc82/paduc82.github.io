eric kyle say: This is more an issue with the POP3 mail protocol then the email client. If there's a change to the profile or software of significance, The POP3 server thinks it's a new setup and will automatically push any emails it has to fill the account. The only solutions are either to ride out the storm, then delete, or remove them from the server. As the draw back to POP3, is unless all the waiting mails comes down, it'll start over on the next connection.

diane poremsky mvp (slipstick.com) say:
This happens when something corrupts the message manager file - such as scanpst or antivirus scanning, you opened the pst in a different profile (which reset the message manager file), or something is messed up on the server and the messages get new ids.

If you cant delete the old mail from the server , set up a rule to delete mail that arrived before a certain date (and change the outlook option to delete mail when it’s deleted from outlook. )

joeu2004 say: Akshay wrote:
I want to change the CELL FORMAT to Number with 0 decimal places.  I do it several times in MS Excel 2007 .csv file but when reopen the file after successful Save, i need to Format Cell again to Number Type. It appears to be in general form always.  I want to permanently change the format so I need not to change it every time when I reopen the file.

By definition, a CSV file is a text file with no format information.  So yes, Excel uses the default General format for all cells when we open a CSV file.

Moreover, when we save to a CSV file, Excel writes numbers as they are displayed, not their actual value.  So if the value 123.45 is displayed as 123, the CSV file will contain 123, and we lose precision.

It is safer to save as Excel if you intend to open the file in Excel again.

(Saving as XML might be another alternative if the file also might be opened by other applications that understand XML.  It seems to do what you require.  However, I have no experience with XML.  I don't know all the pros and cons.)
