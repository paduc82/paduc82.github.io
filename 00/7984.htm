ben m. schorr -bschorr say:

Sort of depends upon how many e-mails you're talking about.  If it's less than half-a-dozen I'd probably just compose your reply once, CTRL+A to select all text in it, CTRL+C to copy it to the clipboard. Then send your first response.

Click Reply on the second message, CTRL+V to paste from the clipboard, and send.

Rinse.  Repeat.

If you're talking about a LOT of messages then you might be able to create a Rule in the Rules Wizard that will automatically respond to those messages with a template.  You'd have to find a common factor (text in the body, same sender, same subject, etc.) to specify as the criteria.  Or maybe you could create your own by assigning a custom category to those messages.

Create the rule, create the response template, Run Rules Now to run the rule against already received messages.

That's a bit more work, which is why it's not really worth it unless it's a LOT of messages you're responding to.

hansv mvp say:
Does the record source of the form contain a unique identifier (primary key)?

mrk98v say: it does

hansv mvp say:

1) Create a select query.

It doesn't matter what it does, we're going to change it later.

Save it as qryExport.

You can hide this query if you like.

2) Place a command button on the form and name it cmdExport.

Create the following On Click event procedure for the command button:

Private Sub cmdExport_Click()
    Dim strSQL As String
    ' Check that we are not on a new record
    If Me.NewRecord Then
        MsgBox "You can only export existing records!", vbExclamation
        Exit Sub
    End If
    ' Save the record if necessary
    If Me.Dirty Then Me.Dirty = False
    ' Assemble SQL string
    strSQL = "SELECT * FROM [tblData] WHERE [ID]=" & Me.[ID]
    ' Set the SQL of qryExport
    CurrentDb.QueryDefs("qryExport").Sql = strSQL
    DoCmd.TransferSpreadsheet _
        TransferType:=acExport, _
        SpreadsheetType:=acSpreadsheetTypeExcel12Xml, _
        TableName:="qryExport", _
        FileName:="ExportedRecord.xlsx", _
        HasFieldNames:=True
End Sub

Change tblData to the name of the record source of the form.

Change ID to the name of the primary key

Change ExportedRecord.xlsx to the path and filename of the target file.
