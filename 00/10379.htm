graham mayor say:
It will be a macro security issue rather than a vba problem. A vba problem would give you error messages - and it would only be a potential problem with the 64 bit version of Outlook 2010. The 32 bit versions of Office will work on the 64 bit Windows 7 and their vba should be fine provided any calls to Object Libraries that are not part of Office or Windows 7 are not made (but then you'll get error messages warning of the fact).

For a workaround, in the Office folder you will find selfcert.exe which will enable you to create a certificate for your own macros.

Having done that, set the macro security in Outlook to provide warnings for all macros. Allow the macros at the security warning when you see it.

From the Outlook VBA editor > Tools > Digital Signatures, add your newly created signature to the project.

Close Outlook saving the project at the warning.

Re-open Outlook and you will again get a security prompt for the vbaproject. From that dialog choose the option to trust items from the publisher (your certificate).

Thereafter the macros should work normally without the prompts on your PC.

The remote user would have to do the same - though with their own certificate.

Here is the process I discovered, which has worked every time:
A) - In Outlook: create subfolders for each person/phone.  Create all contacts in those subfolders or copy from the main contacts window if you've already created them there.  Now, the phone appears to only sync the numbers in the main contacts window, so copy all the contacts from whichever subfolder for whichever phone, to the main contacts window (1: press ctrl+a OR highlight the first, hold shift, then scroll to the last and click it, then release shift - to copy to clipboard; 3: switch to main contacts folder and press ctrl+v -
B) Connect Phone to Computer (assuming Activesync is already installed)

C) - In Activesync on computer: follow on-screen prompts, selecting to sync contacts, then wait for the sync to complete.  Then on the Activesync screen on your computer, select tools>options and change the computer name (in my case it was "Windows PC" which I changed to any arbitrary name).

D) Disconnect Phone from Computer

E) Uninstall Activesync through start>control panel>add/remove programs.

F) - In Outlook: delete all contacts out of main contacts folder (they should all still be in the subfolder).  Go ahead and use the same process as above in step 'A,' to copy the contacts out of the next subfolder, into the main contacts folder.

G) Close Outlook and Restart Computer

H) Reinstall Activesync (if you don't have a CD, you can download it from Microsoft.com)


Repeat Process with Each Phone, from step 'B' to 'H'

When you have finished setting up all your phones, and you've reinstalled Activesync for the last time, every time you initially connect a phone, the Activesync dialog should pop-up, at which point you can either:
1) select which item you want to continually sync and set up the phone for that;
2) follow the prompts keeping all options unchecked - if you'll never want to sync anything on the phones (without uninstalling/reinstalling Activesync again); OR what I do
3) click cancel on the first screen (it will continute to pop up every time you connect the phone and just click cancel again) and use Activesync to transfer files and browse your phone on your computer with no sync'ing.

Way to view Dynamic Distribution group members in Admin console - Microsoft Community

miteshjain_931 say:

Way to view Dynamic Distribution group members in Admin console

Hi Team,

Is there any way to view members of the dynamic distribution group in Admin portal in Office 365?

yangzheng_msft6300 say:

Hi Mitesh,

Based on my research, it is not feasible to view the members of a dynamic distribution group in the Office 365 Admin Portal. As a workaround, an admin can use PowerShell to view the members of a dynamic distribution group. Here are the steps:
1. Connect to Exchange Online using PowerShell with an admin account.
2. Run the command below
$FTE = Get-DynamicDistributionGroup "Dynamic Distribution Group"
3. Run the command below
Get-Recipient -RecipientPreviewFilter $FTE.RecipientFilter
The first command stores the dynamic distribution group object in the variable $FTE. The second command uses the Get-Recipient cmdlet to list the recipients that match the criteria defined for the dynamic distribution group.

Yang
