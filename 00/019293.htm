bob jones aka: cybertaz say:

Joe is correct as far as he goes... By default, Custom Dictionaries are stored in a specific location on the local HD, but there are at least 3 options that may address your need. Based on your description, however, I really don't believe any involvement of a Custom Dictionary is the appropriate way to go -- It really is not a spelling issue to be gin with :-). Adding a plethora of correctly spelled words just because they are in a different Language is cumbersome & can present problems further down the road, not to mention that the markup still will be present if the document is displayed on any but your 2 Macs. If you want to go that way, though, these are 2 possible approaches;

  • Copy the Custom Dictionary to the desired location on the other Mac, then in the Spelling & Grammar Preferences Add & select it as the Custom Dictionary to be used.
  • A Custom Dictionary can be stored in any location accessible to the program. You just need to be certain that the correct Custom Dictionary is specified in each installation of Office that needs to access it. There is a slight concern if the Dictionary might be accessed simultaneously by more than 1 installation at a time.

However, IMO, the best & easiest method is to use Character and/or Paragraph Styles to apply the correct Language attribute to the text in the first place. If the Language of the text is formatted correctly, Word will check it using the appropriate proofing tools if it has them. If it doesn't have them, it will know not to mark the text as misspelled -- it will simply ignore that text from a Spelling & Grammar standpoint.

Furthermore, a style not only is simple to apply, but it can be readily identified, easily modified/updated. And, Styles inherently travel with the document, so the correctly specified Language prevents the markup in any installation of the software.

bob jones aka: cybertaz say:

Adding the Language attribute to a Style is rather straightforward... In addition to choosing any other specifications when you create or modify a Style;

  1. In either the New or Modify Stye dialog, click the Format list button in its lower left corner
  2. Select Language from the list
  3. Include the Do not check... attribute if you have reason to
  4. Make sure the Style does not have the box checked to Automatically Update

IMO, it's usually best to use a Character Style for something like this rather than a Paragraph Style, although you may want to create both. Please note that you shouldn't add this attribute to an existing style unless it is applied only to text that needs to be formatted as that Language. Also, avoid the common temptation to modify the Normal paragraph style or the Default Paragraph Font character style.

You may be able to quickly apply the style by using Advanced Find & Replace. Use the Expand jelly bean button in the lower left of the dialog window if necessary. You then can click the Format button, select the Style listing & pick the Style by name.

Don't hesitate to assign a keyboard shortcut to the Style so you can more readily apply it on individual instances. You may also be able to take advantage of a macro or an AppleScript as described here: Change Language in a Single Click in Word 2008

@GEN82012, this is probably the wrong thread for enquiring on updates but...

No you can't download directly from Nokia if the update hasn't been released for your product code.

Updates are by Product Code, type *#0000# on the Call screen. For example mine is 059C7R0.

These codes are country and operator specific.

Be aware there is a difference between a "Vanilla" handset brought directly from Nokia and an "Unlocked" one.

If a carrier "Unlocks" the handset it means you can use it on other networks but the Product Code will remain the original one and can't be changed.

If the phone has been repaired at some stage it may have a non standard board in it as well and I think those can only be updated at Nokia Care.

If you take the phone to Nokia Care with proof that it is legally unlocked they may be willing to load the local "Vanilla" variant for you.
