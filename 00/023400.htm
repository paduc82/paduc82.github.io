jay freedman say:
You're getting close! Try these fix-ups to see if it gets the form working.

First, I'm assuming that the form is a "protected form" using legacy form fields, and that the field you want to fill from the combo box selection is named "text28". If that isn't true, let me know.

Going with that assumption, the Sub openmyform() macro needs to go in a regular module in the form document (or template). The rest of the code needs to go in the userform, which is also in the form document.

The Sub openmyform() macro should be set as the "on entry" macro for the text28 form field.

On the userform surface, you should have the combo box and a command button. It's a good idea to change the caption of the command button to "OK". (It's also a good idea to add a second command button, and give it the caption "Cancel", in case the user decides not to make a choice right away. But that's an issue for later.)

Replace the Sub insertdata() code with this:

Private Sub CommandButton1_Click()
ActiveDocument.FormFields("text28").Result = Me.ComboBox1.Value
Unload Me
End Sub

There are several ideas here that need explaining:

  • Your insertdata() was never getting run, because Word doesn't have any idea what it is or how to run it -- it just sits there. Instead, the name CommandButton1_Click() is special: when the user clicks the CommandButton1 button, Word automatically looks for and runs a procedure named CommandButton1_Click() if it exists.
  • If the CommandButton1_Click() procedure contained your original statement about ActiveDocument.Bookmarks("text28").Range.Text, it would have given you an error when it ran. The problem is that the range of the bookmark "text28" extends into the protected part of the form to the left and right of the form field, and you (and VBA) aren't allowed to modify its .Text. Instead, the statement needs to set ActiveDocument.FormFields("text28").Result, which is the part inside the form field.
  • Also, your code mentions Me.textbox1.Value. If your userform contains a text box named textbox1, then you haven't done anything to copy the value of the combo box into the text box; and if there is no control on the userform named textbox1, then you'd get an error when the code tries to get its value. I avoided the whole issue by using Me.ComboBox1.Value instead.
  • You need the Unload Me statement to make the userform disappear when the button is clicked.


If you make these changes and the form still doesn't work, email a copy of the file to me and I'll fix it. You can get my address from http://jay-freedman.info/contact/contact.htm.

You only need one userform for all the text boxes. The trick is to change the openmyform macro so it can tell the userform the name of the field that needs to be filled in. There are a few parts to change:

First, in the code of the userform, add this line at the very top:

   Public fieldname As String

Then change the CommandButton1_Click code to this:

   Private Sub CommandButton1_Click()
   ActiveDocument.FormFields(fieldname).Result = Me.ComboBox1.Value
   Unload Me
   End Sub

Now the openmyform macro needs to be replaced with this version:

   Sub openmyform()
    Dim UF As UserForm1
    Set UF = New UserForm1
    With UF
        .fieldname = Selection.Bookmarks(1).Name
        .Show
    End With
    Set UF = Nothing
   End Sub

The last task is to go into the Properties dialog of each form field (text box) and select openmyform as the entry macro. Also, make sure there is some unique name in the Bookmark box of each Properties dialog -- it must not be blank for any of them.
