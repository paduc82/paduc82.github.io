A form has a Record Source property that can be set to a table/query name or an SQL statement like: SELECT FieldA, FieldB, FieldC, ... FROM [YourTableName].

All other MS Word 2003 files can be opened and edited in Word 2016 just fine.

jay freedman say:

Since you're dealing with Forms protection (and not the several other kinds of protection that are available), the simplest way is this:

  • Go to File > Options and choose Quick Access Toolbar in the left-hand column.
  • Set the "Choose commands from" dropdown to "All Commands".
  • Click on the Lock command, and click the Add button in the center of the dialog.
  • Use the Lock button on the Quick Access Toolbar to toggle the forms protection on and off.

The alternative is to click the Review tab of the ribbon, and click the Restrict Editing button in the Protect group. That opens a pane in which you can stop and start protection, or choose a different type of protection (in documents that don't use form fields).

A form document must be protected for forms in order to make the content of the fields editable. It must be unprotected in order to make the fields' Properties dialogs available.

In all versions of Word from 2007 onward, there is a Developer tab that's hidden by default. If you need to add fields to the form, first go to File > Options > Customize Ribbon and put a check mark next to Developer in the right-hand column. Click OK, then go to the Developer tab. The form fields (now called "Legacy Form Fields") are in a dropdown menu from the Legacy Tools button in the Controls group (the one that looks like a black toolbox).

Can I get a textjoin() add-in for 2013 - Microsoft Community

pittsburghbuz say:

Can I get a textjoin() add-in for 2013

Can I get a textjoin() add-in  for 2013. I purchased my PC last fall and it is running excel 2013 and I really do not want to spend more money for just one application.
myall_blues say:

Hi

You could create one if you don't mind a bit of dabbling with VBA. 

(Waring: I'm not familiar with textjoin() but wrote this code based on the description. Not extensively tested.)

Steps:

1. Open a blank workbook

2. Press ALT+F11 to open the VBA editor

3. In the Project window on the left, right-click on the workbook name and choose Insert ->Module

4. Paste this code in the resultant window

Option Explicit
Function TEXTJOIN(delimiter As String, ignore_empty As String, ParamArray textn() As Variant) As String
    Dim i As Long
    For i = LBound(textn) To UBound(textn) - 1
        If Len(textn(i)) = 0 Then
            If Not ignore_empty = True Then
                TEXTJOIN = TEXTJOIN & textn(i) & delimiter
            End If
        Else
            TEXTJOIN = TEXTJOIN & textn(i) & delimiter
        End If
    Next
    TEXTJOIN = TEXTJOIN & textn(UBound(textn))
End Function

5. Save the workbook as an add-in type (.xlam)

6. Load the add-in into Excel (File ->Options ->Addins)

Usage is the same as the real textjoin function except that text1 is also optional.

i.e. TEXTJOIN(delimiter, ignore_empty, text1, text2, ...., etc)
