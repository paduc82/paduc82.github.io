joanne li msft say:

Hi Anoonan,

Based on my understanding, you are trying to achieve this:

  1. Generally, some applications like Flow for Office 365 and Power Apps for Office 365 should be turned off for most users, and applications like Skype for Business should be turned on for select users.
  2. But there are some exceptions which are different from above situation, so you want to view some report to see the exceptions to manage them.

If my understanding above is correct, we can try using PowerShell cmdlets to vie the exceptions, please follow the steps below:

1.  Connect to Office 365 PowerShell.

2.  After connecting to Office 365 PowerShell, use the following command to view summary information about your current licensing plans and the available licenses for each plan:

Get-MsolAccountSku

The result will be like this:

3.  Now that we have the AccountSkuId (here mine is ENTERPRISEPACK), we can enter the following to get the included Service Plans.

Get-MsolAccountSku | Where-Object {$_.SkuPartNumber -eq "ENTERPRISEPACK"} | ForEach-Object {$_.ServiceStatus}

Which should be like this:

4.  Now is the time where the order in which the services appear is extremely important. Windows PowerShell assigns an index number to each entry in the list. The first entry is 0, the next entry is 1, and so on. The results are explained in the following table, note that your plan may seems different from mine, just modify the number to adjust your situation:

To see what these oddly-named service plans really represent, you can refer to PowerShell Licensing SKU's in Office 365.

5.  Let’s try an example. Suppose we’d like a list of all the licensed users who have been enabled for Flow for Office 365. To do that, we can use the following command:

Get-MsolUser | Where-Object {$_.isLicensed -eq $true -and $_.Licenses[0].ServiceStatus[3].ProvisioningStatus -ne "Disabled"}

The result should be like this:

 

To get more information, you can refer to View account license and service details with Office 365 PowerShell.

Joanne

Norman is correct but there is no point in searching for any miracle solution. 

When I was investigating Office 2007 password protection I discovered that the online "helpers" offering to crack the file's passwords were often just boasting.  Once you get down into the detail, they can really only crack Office 2003 passwords.  Office 2007 uses a different password system that is much, much harder to crack [and Office 2016 is more than a single order of magnitude much harder to crack than Office 2007].
