=dlookup("Price","Products","PartNumber="& Combopartnumber)

Thank you.  

scottgem say:

I suggest you read my blog on Displaying data from related tables..

I would not use a dLookup here.

The RowSource of the Part number combo should look like this:

SELECT ProductID, PartDescription, Price

FROM Products

ORDER BY PartDescription;

In the After Update event use code like:

Me.txtPrice=Me.ComboPartNumber.Column(2)

What you were doing is not the best move for two reasons. First, aggregate functions like Dlookup can be dogs in performance. Adding the column to the combo will perform better. Second, it appears you were using the DLookup as the Controlsource of a textbox. That would mean that the price would not be stored with the transaction. While normally you don't want to copy a data value from another table, one of the exceptions is time sensitive data. Prices change over time and you want to capture the price at the time  of the transaction.

mimmo57 (d. laurenza) say:

Hi,

I fully agree with Scott. However in general to referto a control in a query by a mask you have to qualify the control with the name of the form that contains it.

Then you should write:

=Dlookup("Price","Products","PartNumber=" & Forms.yourFormName.Combopartnumber)
