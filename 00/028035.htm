Shared Audio playback issue with Powerpoint for Mac version 15.15

I created a Powerpoint presentation on my MacBook Air (using Powerpoint for Mac version 15.15) and shared it with a Windows PC, running an older version of Powerpoint.

The audio file on one of the Mac-created slides would not run on the Windows machine - giving the error message Codec not available.

I see that Powerpoint 2016 and Powerpoint 2013 have Optimize Media Compatibility buttons. I can't find that on Powerpoint for Mac version 15.15.

What should I do to make a slide with an audio file compatible with the recipient's set up?

john korchok say:

While PowerPoint on the Mac can import a variety of sound files, some of those formats are only Mac compatible. Formats that are more universal are MP3 and WAV. If the sound file is something like AIFF, it's very unlikely to play in Windows.

To convert sound files from one format to another, try downloading and installing the free Audacity sound editor. 

To convert the existing files in your presentation, please follow these steps:

  1. Make a copy of your presentation to a separate folder.
  2. Add .zip to the file ending. OS X will ask if you want to do this, choose Use .zip.
  3. Right-click on the file and open it with the Archive Utility instead of PowerPoint. You will see several folders and a file called [Content_Types].xml.
  4. Open ppt/media. All the sound files in the presentation are here.
  5. Open the ones that don't play under Windows in Audacity and convert to either WAV or MP3, keeping the file names the same except for the ending.
  6. Go back to the folder where you unzipped the presentation.
  7. Select [Content_Types].xml plus the folders, right-click and choose Open with and select Archive Utility. (If you don't see Archive Utility in the list of programs, click on Other, then find Archive Utility in Applications/Utilities.)
  8. Give the file a .pptx ending and try opening it in PowerPoint.
  9. If PowerPoint exhibits any problems opening the converted sound files, open another copy of the original presentation in PowerPoint, delete the problem sound files and replace them with the converted versions you just made.
  10. Test in Windows.

How do i write a validation rule for a date field? - Microsoft Community

sherymesiha say:

How do i write a validation rule for a date field?

How do i write a validation rule for a date field?

I am working on a database project.

I have a date field in my table  I want to enforce the data entry user to enter  the end date one day more than or equal the previous date

Im hoping there is some simple way to do this. thanks .

ken sheridan say: You'll need to do this at form level.  In the BeforeUpdate event of the control in your data entry form bound to the date field put:

    Dim strMessage As String
    Dim dtmLastdate As Date
     
    dtmLastdate = Nz(DMax("YourDateField", "YourTable"), 0)
     
    If dtmLastdate <> 0 Then
        If Not (Me.[YourDateField] = dtmLastdate Or Me.[YourDateField] = dtmLastdate + 1)  _
            Or IsNull(Me.[YourDateField]) Then
            strMessage = "Date must be " & dtmLastdate & " or " & dtmLastdate + 1
            MsgBox strMessage, vbExclamation, "Invalid Operation"
            Cancel = True
        End If
    End If
