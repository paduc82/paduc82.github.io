tom ogilvy say:

so yes, you can do it manually.  (doing the sort after the unmerge leaves out anything that keeps the city rows together - I will add those steps in my description).

after you did the unmerge, then you would click the range in column A (left column), hit F5 to show the Goto dialog, choose special and then select Blank Cells (then OK).   Now say A3 is the activecell and only the blank cells are selected.  We want to fill in the city name so in the formula bar put in =A2  (assuming you are using A1 style addressing).  Then hit Ctrl+Enter rather than just enter.  This should fill all the cells with the proper city name (note that the actual formula would be =Cell above the Active Cell relative address).     Now you would select column A; do a copy, then paste Values to replace the formulas with the values they produced. 

Then you could sort the data like any table (multiple columns) using column A as the Key field.  Excel does a stable sort, so rows with identical keys (city name) will maintain their same relative position.   

If you are happy with that, then you are done.  But if you want to restore the merged cells, then that is where manual would be Tedious since you will have to recreate each cities merged block with some individual selection.  

I can select say   A2:A5, hold down the control key and Click in A6 then drag down to A20 (A6:A20 as a block), and continue that way.  I can then do the merge manually and the individual blocks will be separately merged. 

When you select the merge, you will be warned for each block that the data in that block except the top left will be lost and  you will have to click OK.

Tom Ogilvy
