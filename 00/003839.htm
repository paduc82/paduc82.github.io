No. You can fill a slide to the edges with a linked chart and it will not expand beyond the slide edges. It could actually shrink a little.

The size change happens when you paste a worksheet selection. Charts on their own have a fixed size that doesn't get enlarged by the view change.

The enlargement happens when you update the links. If nothing has changed on the source, click on the Cancel button instead of updating. Then even a worksheet selection will keep its expected size on the slide.

A couple of things come to mind. One is the font issue that John mentioned.

Another is tabs. Tabs in 2007 and 2003 work differently. Did you use tabs in your textboxes or placeholders?

Yet another is a printer driver thing. This happens more often with 2007 than with 2003, but it could be an issue -- basically, PPT relies heavily on the installed printer driver for display, and if it doesn't like the driver that's installed or if the driver is installed on a network, not directly on the computer, then sometimes you'll get weird display issues like this.

I would probably send a PDF of the presentation. PPT 2007 has this capability (if you've installed Service Pack 1, that is -- or you can download it). If you go to Office Button | Save, there's a PDF/XPS option. Maybe give that a shot.

bob jones aka: cybertaz say:

You don't need a Table for this. The Headers in Word already are designed to provide the Alignment you want based on default Tab Stops. You do have to change some of the default settings but the L/R margins for the Header cannot differ from those of the Section to which it belongs. You need to adjust the L/R Indents of the Header instead.

  1. Go into the Header
  2. On the Header & Footer tab of the Ribbon tick the check box for Different First Page
  3. Go to Format> Paragraph - Indents and Spacing:
    1. In the Indentation fields set both Left & Right to -0.5" [Don't overlook the - sign]
    2. Click the Tabs button at the bottom
      1. select the 6.4" stop in the list, then click the ➖ to delete it
      2. Select the value in the top field & type 7" then click the ➕ to add it
  4. OK out of the Tabs dialog which dismisses the Paragraph dialog as well

PST files can be in one of 2 formats - ANSI and UNICODE. The ANSI format is the original format used in Outlook '2002 and earlier with a max file size of 2GB. Unicode format PST files can grow to the size indicated in the registry. For the sake of this post - don't think anything related to ANSI PST files is relevant or applicable.

How to determine whether a PST file is ANSI or Unicode? Right-click on the root of the folder tree in the Outlook Nav Pane --> Data File Properties -->click on <Advanced> button --> if the <Format> description includes "97-2002" - then it's an ANSI formatted file (and if that's the case, really should be converted to Unicode).

One thing to keep in mind is that "very large" PST files start to cause performance issues - personally prefer to keep my PST files to be less than 10gb regardless as to what is allowed which leads into the other question about how to best archive information.

#1 - You can manually create as many PST files as you like (within reason) to store your data. The "within reason" means that just because "you can" add 100-200+ PST files to a profile that "you should"

#2 - Everyone has their own way of organizing information so there is no "one size fits all" method that applies to everyone. Using the "archive" function within Outlook results in data being archived "by date" which may be ideal for some. However, not an approach I find useful since I prefer to maintain info by primary "subject matter" such as Sales, Support, Projects, Marketing etc etc and within that primary Folders similar to a physical <File Cabinet with drawers. However, everyone has their own way of retrieving info beyond just using the "search function". 

#3 - The other side benefits of not working with an extremely large PST file are:
a) should a PST file get corrupted beyond repair - you don't lose everything

b) whole lot faster to back up smaller files

c) don't need to repeatedly backup PST files that don't continue to change (i.e. Projects - 2000thru2010 as a for instance)

It's possible you are running into one of two issues.

The first issue is Forbidden Characters used in the name of the File(s).

Some of the Forbidden Characters are:

Numbers before names, Superscript/Subscripts.And these Characters:

+ * ” ‘ \ | / ; [[ ]] << >>  <  >  [  ]

Numbers after Names are okay but before extensions.

You cannot use these characters in the name of:

A) Hard Drive.

B) Directory (Folder) Names.

C) Any File Names.

D) Any Device Names.

E) And on servers

For Servers: no names can have the name File or Fileserver in any path names.

You can use _ (underscores).

NOTE: I haven’t included dashes/hyphens because I haven’t found any information.

The second issue is, as a result, of Apple's security system  Unless Saved in the appropriate place recommend by Apple. You have to go to the path shown in the error message select the file and Choose to accept (okay).
