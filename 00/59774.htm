ionut lazar say: Hi,

Con and others (Com1 to Com9, Lpt1 to Lpt9, Nul, Prn, Aux) are reserved names in Windows and you cannot use these names.
These are some standard devices used by Windows (Con = Console, Prn = Printer, Aux = Auxiliary device, Com = Serial ports, Lpt = Parallel ports).

I have some users who received some new HP laptops(850 G1's) running Windows 7 pro 64 bit..  They are all part of a domain.  When they boot their laptop with the Ethernet plugged in, it takes a very long time for them to get to the control alt delete screen and then to their desktop once they put in their password.  These laptops are equipped with SSD's.  It should be very fast to boot up.  Not take several minutes.  I looked at the LAN NIC in the network and sharing center and instead of it showing the name of the domain, it showed "Network 2."  If I disable the NIC and then re-enabled it, it then shows the name of the domain, and all the network drives came back up and everything was fast.  So,  if I take out the network cable and then boot the laptop it boots super fast.  So, it seems that the laptop is searching for the domain upon boot up but doesn't find it, causing it to slow way down.

I have updated the NIC driver, no change. 

Plugged the Ethernet directly into the laptop, bypassing the docking station, no change.

Restarted the Small business server domain controller, no change.

Disabled Cisco Anyconnect VPN at start up, no change.

Anyone know what is going on or have any other suggestions?
