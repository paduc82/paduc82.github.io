Ditto on what YaminiFranzini stated. Adding only your cluster address to trusted sites and not login.live.com causes it to bounce back and forth between trusted and untrusted zones forcing you to re-authenticate. Eventually this times out causing the error you have called out.


There are technically two solutions:

  1. Remove from your trusted sites (this is more preferable, the site is designed to operate normally under the Internet zone, no need to unnecessary exposure).


OR


  2. Add all of the following:

https://mail.live.com

https://dub126.mail.live.com (or whatever your cluster is)

https://login.live.com


In this case it will work too. The first option is a far more preferable state for multiple reasons, primarily because over time users are moved between clusters which would break the latter option again until the new address was added.


If the address is not listed under your trusted sites and you are still experiencing this post back and we can troubleshoot in a bit more depth.

john w. vinson say:

You could put code in the textbox's GotFocus event:

 

Private Sub textboxname_GotFocus()

Me.textboxname.SelStart = 0

End Sub

 

The downside of this is that if the user is trying to legitimately change text somewhere in the middle of the box (to correct a typo, say) it will change the cursor unexpectedly.

 

Edit: This is fixable: only move the focus if the textbox is empty.

 

Private Sub textboxname_GotFocus()

If Len(Me.textboxname) & "" = 0 Then

   Me.textboxname.SelStart = 0

End If

End Sub

 

(end edit)

hansv mvp say:

Select G1:G9. I'll assume that G1 is the active cell within the selection.

On the Home tab of the ribbon, click Conditional Formatting > New Rule...

Select 'Use a formula to determine which cells to format'.

Enter the formula   =AND(E1>=2.3,G1="")

Click Format...

Activate the fill tab.

Select a fill color.

Click OK twice.

Repeat the above steps, but with the formula   =AND(E1>=2.9,G1="")   and another fill color.
