granitestatecolin say:

I finally figured out the solution, but this may only apply to my particular Fujitsu model (T904) or that plus a few similar models.

  1. Open Fujitsu Display Manager
  2. Go to the Dock/Undock tab
  3. Flip the screen around so that it is physically in tablet mode (note this is NOT the same as "Tablet Mode" in Windows where everything runs full screen, this just means that the screen is folded around so you can't access the keyboard) 
  4. Set the screen orientation (and any other desired attributes, if there's something else you want to change when physically in tablet mode)
  5. Press the "Save current profile" button.

That's it. Now when you flip the screen so the computer is physically configured as a tablet, it will use the orientation you set, instead of the original default portrait orientation.

The key lesson here for anyone is that it is NOT a Windows issue, but rather specific to the tablet model. I don't have a Surface Pro or Surface Book to compare, but I'd curious how Microsoft handles changing the orientation when flipping the screen around on the Book. Maybe they don't.

Deleting protected folders - Microsoft Community

mattmcmillen say:

Deleting protected folders

Hi, when I replaced a dying hard drive, I saved the 'program files (x86)' folder on my desktop. I've now removed everything I needed from it and would like to delete it. I have managed to delete all the files in the folder and subfolders, but I can't delete the folders themselves. I've updated my permissions so that I have full control and I am signed in as the administrator on my computer, which is not on a network. I've been able to rename the folders, but when I try to delete them, it still says I need permission from the administrator. What can I do? Thanks!
wunders say:

The next thing you should try is to take ownership of this tree of files/folders.  By default, Administrators are granted the "TakeOwnership" privilege, so while logged in as an administrator, open an elevated command-prompt window as before.  When it comes up, attempt to take ownership of these folders using the following command:

  takeown /F C:\Users\quartet1977\Desktop\fluffy  /A /R

The above command will [should] assign ownership of the entire tree to the Administrators group.  Once you have ownership, as an administrator user, you should be able to grant yourself access to the folders:

  icacls  C:\Users\quartet1977\Desktop\fluffy /grant Administrators:F  /T /C

Then after that, you should be able to delete the folders.

If "takeown" complains that you don't have privilege, then you need to check to make sure that Administrators are granted the "TakeOwnership" privilege...   From a command prompt, type:

   gpedit.msc

When the Group Policy Editor comes up, navigate to:

  Local Computer Policy -> Computer Configuration -> Windows Settings ->Security Settings ->

      Local Policies -> User Rights Assignment

and in the right hand pane, look for "Take ownership of files or other objects" and make sure that this privilege is assigned to "Administrators".

HTH,

   JW
