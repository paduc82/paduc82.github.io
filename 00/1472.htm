jllatham say:

OK - VBA inside of a UserForm.

First - I have prepared a workbook with user form and code as described here, so that may be a help in following along.  You can get that workbook from this link:

http://www.mediafire.com/view/?5444th8l5djty79
for_Gaffnr.xlsm 25KB

The Userform has 9 controls of interest to us:
1 combobox named ComboBox1

4 labels to hold player names, named lbl_Name1, lbl_Name2, lbl_Name3 and lbl_Name4

4 text boxes to hold their scores, associated with the labels by number.  Their names are txt_Score1, txt_Score2, txt_Score3 and txt_Score4.

The setup on Sheet1 where we are going to put the scores is:  A1 is empty.  B1: D1 have the course names, and A2:A5 have the player's names.  This figures in because we are going to use the information from the worksheet entries to fill ComboBox1 and to set the labels on the four labels.  This will assure that everything matches up when we go to try to find things on the sheet.

The Userform has an _Initialize() process that is used to fill the combo boxes and the labels for the names:

 

Private Sub UserForm_Initialize()
'we fill the combo box and name labels with the list of
'courses and names that starts at B1/A2 on Sheet1
'so that they match up, and if they change
'the list changes also.
  Const scoreSheetName = "Sheet1" ' change as needed
  Dim SourceList As Range
  Dim anySourceEntry As Range
  'this assumes no breaks (empty columns) in the list of courses
  'first we set up the combo box list of courses
  Set SourceList = Worksheets(scoreSheetName).Range("B1:" & _
   Worksheets(scoreSheetName).Range("B1").End(xlToRight).Address)
  'now work through the list 1 at a time
  For Each anySourceEntry In SourceList
    Me.ComboBox1.AddItem anySourceEntry.Value
  Next
  'now a little drudge work to make sure that our
  '.Find efforts work later when we need to post scores
  'assumes list of names starts in A2 and has no blank rows.
  'can't use a loop because labels can't be an array
  'we just have to "Know" how many to fill and where data is at
  Me.lbl_Name1.Caption = Worksheets(scoreSheetName).Range("A2")
  Me.lbl_Name2.Caption = Worksheets(scoreSheetName).Range("A3")
  Me.lbl_Name3.Caption = Worksheets(scoreSheetName).Range("A4")
  Me.lbl_Name4.Caption = Worksheets(scoreSheetName).Range("A5")
End Sub

 

As you can see, because we can't have an array of labels or text boxes, we do have to do some things "long-hand" when setting it all up.

 

But after the initialization, which is done when the form is loaded into memory, we're ready to go.

 

You would make a selection of the course from the ComboBox1 control, and then simply enter scores for the players for that course and finally click a command button to take the scores and put them on the worksheet.  NOTE: as written, you'd have to enter scores for all players to make sure that you didn't overwrite previously entered scores for just a few of the players for that course.  It wouldn't take too much to deal with that better.

 

Here is the code that executes when the button is clicked to put the scores onto the worksheet.

 

Private Sub cmd_PostScores_Click()
  Const scoreSheetName = "Sheet1" ' change as needed
  Dim courseName As String
  Dim players(1 To 4) ' 1 for each player's name
  Dim scores(1 To 4) ' again 1 for each player's score
  Dim LC As Integer ' to work through the arrays.
  Dim CourseList As Range
  Dim anyCourse As Range
  Dim foundCol As Long
  Dim NamesList As Range
  Dim anyName As Range
  Dim foundRow As Long
 
  courseName = Me.ComboBox1.Text
  If courseName = "" Then
    'no sense going further, did not choose a course
    MsgBox "You have to choose a course to post the scores."
    Exit Sub
  End If
  'this is all dependent on the number of players and names you
  'gave to the label and score entry text box you set up for
  'each one.  No magic here, plain drudge-work during design.
  players(1) = Me.lbl_Name1
  scores(1) = Me.txt_Score1
  players(2) = Me.lbl_Name2
  scores(2) = Me.txt_Score2
  players(3) = Me.lbl_Name3
  scores(3) = Me.txt_Score3
  players(4) = Me.lbl_Name4
  scores(4) = Me.txt_Score4
  'now we CAN use some 'magic' to finish the job
  'we want to include cell A1 in both ranges so that
  'the found row/column associates with the sheet properly.
  'but we determine the end of each list based on the actual
  'first entry in each one.
  Set CourseList = Worksheets(scoreSheetName).Range("A1:" & _
   Worksheets(scoreSheetName).Range("B1").End(xlToRight).Address)
  Set NamesList = Worksheets(scoreSheetName).Range("A1:" & _
   Worksheets(scoreSheetName).Range("A2").End(xlDown).Address)
  'find the column for the course, only have to do this once
  For Each anyCourse In CourseList
    If anyCourse = courseName Then
      foundCol = anyCourse.Column
      Exit For ' no need to look any more.
    End If
  Next
  For LC = 1 To 4
    For Each anyName In NamesList
      foundRow = 0
      If anyName = players(LC) Then
        foundRow = anyName.Row
        Exit For
      End If
    Next
    If foundRow > 0 Then ' safety valve
      Worksheets(scoreSheetName).Cells(foundRow, foundCol) = _
       scores(LC)
    End If
  Next
End Sub

 

Essentially what it does is find which column the course entries go into by matching the entry in ComboBox1 with the entries in row 1 of the worksheet.  Then it finds each of the rows that the specific name is on based on entries in column A.  That gives us an "x-y" coordinate pair (row, column) that we can use with the .Cells() property to put a score in the right place on the worksheet.

 

Actually, I'm thinking as I look at it, that if we just change that last piece of the command button code to read like this:

    If foundRow > 0 Then ' safety valve
      If Val(scores(LC)) > 0 Then
        Worksheets(scoreSheetName).Cells(foundRow, foundCol) = _
         scores(LC)
      End If
    End If

Then if you left a score blank/empty on the UserForm, nothing would be written to the worksheet, and it would leave any previously entered score for a person unchanged.
