Hi,

The DRIVER_IRQL_NOT_LESS_OR_EQUAL (d1) errors appear to be caused by the L1C63x64.sys :


BugCheck D1, {0, 2, 0, fffff8094d386cb3}

*** WARNING: Unable to verify timestamp for L1C63x64.sys
*** ERROR: Module load completed but symbols could not be loaded for L1C63x64.sys
Probably caused by : L1C63x64.sys ( L1C63x64+6cb3 )

The L1C63x64.sys is the driver for the Atheros network adapter and appears to be at version 2.10.16.

Can you update to v2.1.0.26 available from the following link:

https://www.ath-drivers.eu/download-driver-nr-315-for-atheros-AR8151-and-Windows10.html

The driver has to be installed using the Device Manager - post back if you need any help with that.

The DRIVER_VERIFIER_DETECTED_VIOLATION (c4) error was caused by the HECIx64.sys which is related to Intel Management Engine Interface but this may only indicate an issue the HECIx64.sys has with the Driver Verifier but you could try updating to the latest Intel Management Engine Interface available from Gigabyte for the motherboard.

There was one DRIVER_POWER_STATE_FAILURE (9f) error that seemed to be caused by the vhusb3hc.sys :

*** ERROR: Module load completed but symbols could not be loaded for vhusb3hc.sys
 \Driver\vhusb3hc   ffffcf87613f2fe0  Cannot read info offset from nt!ObpInfoMaskToOffset

The vhusb3hc.sys appears to be from Virtual Here - something to consider if issues persist.

IMHO, It's not 'messed up' so much as 'impossible'. 

After all - millions upon millions of possible configurations exist and without a database of every one of the variables - there's no 100% guarantee.  (How many different video cards, audio devices, motherboard chipsets, processors, printers, scanners, network devices, etc. do you think there are and how many different combinations could they possibly be in?  Throw on top of that the software packages and the varying versions and patch levels of each of those applications as well...)
