doug robbins - mvp office apps & services (word) say:

While it's a bit illogical to have to Heading 1 style paragraphs in succession, the issue is not peculiar to that particular style, or for that matter a mixture of styles.  When two paragraphs that are formatted to have a border above them follow one another, the border does not appear on the second paragraph.

To get the border to appear, you will need to separate the paragraphs with another paragraph that does not have a border and for which you minimise the space that it occupies by setting the space before and after it to zero and the line spacing to Exactly 0.7pt

scottgem say:

So you want to copy the current record displayed on a form and add it as a new record in your table. Using Copy and Paste is not the best practice here. First, if you have an autonumber PK (as you should have), you can't copy and paste that value. Second, It would be a rare instance where you would need to make an EXACT copy of a record. If you are doing that frequently then your database is probably not normalized properly. So you probably only want to copy CERTAIN fields into the new record. The easiest way to do this is with a SQL Append statement. Something like:

Dim strSQL As String

strSQL = "INSERT INTO table (field1, field2, field3) " & _
              "VALUES(" & Me.Control1 & ", '" & Me.Control2 & "', #" & Me.Control3 &"#);"

CurrentDB.Execute strSQL, dbFailOnError

Me.Requery

DoCmd.RunCommand acCmdGotoLast

In this example field1 is a numeric field, field2 is text and field3 is a date. Text fields have to be wrapped by single quotes and dates by octothorpes, Numbers have no wrapper. You can add as many fields as you want to copy into the new record, just add the corresponding controls to the Values clause.

 

Hope this helps, Scott<> P.S. Please post a response to let us know whether our answer helped or not. No you don't want to duplicate even "MOST" of the fields, but if you are doing so, there are probably design problems with your data. Even if there weren't, you don't want to copy values, then go to the last record and delete them. That makes no sense and is harder to code. What you want to do is append a new record to your table and populate just the fields that you need duplicated. That's what the code I gave you does. The field list in the first line would be a list of the fields where you want to duplicate values. The Values clause will pull each fields value from the controls on the form. The code will then requery the form and go to the last record (the newly added one) so the user can fill in the additional info not duplicated.
