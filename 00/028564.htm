brook zhang msft say:

Hi Admin,

 

In Exchange hybrid deployment, customers should keep all the settings configured by HCW. We don’t suggest customer customize these settings. 

 

Both emails from Office 365 and non-Office 365 are routed through internet to your local Exchange servers.

The connector in Office 365 will check the IP address of dmzedgetrp.contoso.com. Emails from non-Office 365 will also check the MX of email suffix, aka dmzedgetrp.contoso.com, and further check the IP address of dmzedgetrp.contoso.com. Thus, both this two kinds of emails will be delivered to the same location.

 

The Edge server and the Linux server have different address. To achieve your goals, you can

1. either modify the connector to route emails by using the IP address of the Edge server

2. or change the MX record of your domain to another domain and add an A record for the domain to connect it to the Linux server. In this scenario, the A record of the domain dmzedgetrp.contoso.com needs to be pointed to the Edge server so that Office 365 deliver emails to the Edge server.
