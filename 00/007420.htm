TOC indentation problems

The section headers are indented much too far; they should have the least indent. And the page numbers to not line up to make a neat column. See picture.

The TOC is automatically generated. All the headings are formatted using the heading styles (H1, H2, etc). So far as I can tell, they have no custom formatting applied. And in Format -> paragraph, all of the headings are set to an indent of 0.

Thank you!

suzanne s. barnhill say:

Unless you have modified the TOC styles, they are all set to update automatically. This means that if you change the formatting of a single entry at a given level, the change will be applied to the style so that all the other entries are immediately corrected. The easiest way to do this for paragraph formatting of this nature is to drag the indent and tab markers on the horizontal ruler. With the insertion point in one of the TOC 1 paragraphs, drag the left indent back to the left margin and drag the right-aligned tab marker to the right margin. Repeat for the tab marker of TOC 3.

Note that, if you have applied a right indent to any of the TOC styles, you won't be able to drag the tab marker to the right of the indent. You can set it to the right of the indent, provided it is inside the right margin, but you have to do it through the Tabs dialog.

smitty having trouble say:

Solution:

 

Due to recent changes in Microsoft IMAP servers that handle Outlook.com/Hotmail for third-party email programs, the default settings for incoming and outgoing mail servers must be updated. Due to bottlenecks in server allocation, using the default server settings in third-party email programs can lead to server timeouts, server disconnects, and slow loading of messages. At the present time, the issue has been confirmed for Thunderbird email. Note that the fix below applies to email programs configured for IMAP, not POP.

 

To update settings in Thunderbird...

 

In Account Settings, click "Server Settings" on the left, below your Outlook.com account name.

The box beside "Server Name" on the right is the incoming mail (IMAP) server. Change it to...

outlook.office365.com

(FYI, the port should remain as-is: 993.)

 

At the bottom of the list on the left side, click "Outgoing Server". If Outlook.com is your default server, click the edit button. Change the Server Name to...

smtp.office365.com

(FYI, the port should remain as-is: 587.)
