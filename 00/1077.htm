huni shen msft say:

Hi Mike,

 

The correlation ID is a GUID (globally unique identifier) that's automatically generated for every request that the SharePoint server receives. It's unique to each request, not each error. However when an error occurs, the error message contains the correlation ID that was valid for the request at the time.

 

If you‚Äôre an end user, you probably won‚Äôt have access to the logs that provide the information where a correlation ID can be helpful. As an IT Pro or Admin, to figure out what happened, you need to find the ULS logs for the time the event occurred, and search for the Correlation ID in those logs.

 

You can use a tool such as ULSViewer to facilitate looking at this data and filter out the requests you don‚Äôt need to see. See the SharePoint logging and troubleshooting using ULSViewer blog post for more ideas and tips.

 

For your reference: https://support.office.com/en-us/article/sharepoint-correlation-id-in-error-messages-what-it-is-and-how-to-use-it-5bf2dba7-43d2-484c-8ef4-e059f76e3efa

Microsoft Office OneNote 2007 automatically creates backup copies of your notes. Only sections that have changed since the last backup are backed up again.
