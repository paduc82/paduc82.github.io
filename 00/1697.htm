What you want sounds very difficult. Layers are a property of a page, they are not document wide, i.e. each page of a document can have totally different layers.

When a shape (a master from a stencil) is dropped on a page any layers specified within that master shape's page get added to the page (if the layer doesn't exist already). Combining two PowerPoint presentations with internal hyperlinks into - Microsoft Community

mpoitras say:

Combining two PowerPoint presentations with internal hyperlinks into one presentation.

I have two PowerPoint presentations.  Each have hyperlinks referring to other slides within the same deck.  When I combine the two presentations, the hyperlinks do not automatically change to the new correct slide number.  Is there a way around this without having to re-do all of the hyperlinks?   
shyam pillai say:

Here is the macro which will run thru to the presentation and remap the internal hyperlinks to match slide titles. It uses the 1st matching title it locates as the target slide.

'=================================================================

Option Explicit

Sub ResetLinksToTitles()
Dim oPres As Presentation
Dim oSld As Slide
Dim I As Integer
Dim Titles() As String
Dim Target() As String
Dim TargetSlideIndex As Long
Dim sNewLink As String
Dim Ctr As Long

Set oPres = ActivePresentation
Ctr = 0

'Load all the titles
ReDim Titles(1 To oPres.Slides.Count)
For I = 1 To oPres.Slides.Count
    If oPres.Slides(1).Shapes.HasTitle Then
        If oPres.Slides(1).Shapes.Title.TextFrame.HasText Then
            Titles(I) = oPres.Slides(I).Shapes.Title.TextFrame.TextRange.Text
        End If
    End If
Next

For Each oSld In oPres.Slides
    For I = 1 To oSld.Hyperlinks.Count
        With oSld.Hyperlinks(I)
            If .Address = "" Then
                Erase Target
                'Get the link info and parse it.
                Target = Split(.SubAddress, ",")
                'Locate a slide for that title, if any.
                If Len(Trim(Target(2))) > 0 Then
                    TargetSlideIndex = FindSlideByTitle(Target(2), Titles)
               
                    If TargetSlideIndex > 0 Then
                        ' Check if the id of the slide found already matches the existing link
                        If Val(Target(0)) <> oPres.Slides(TargetSlideIndex).SlideID Then
                            ' If not then update the link with the new id and slide index.
                            sNewLink = CStr(oPres.Slides(TargetSlideIndex).SlideID) & _
                                                "," & CStr(TargetSlideIndex) & "," & Target(2)
                            Debug.Print "Changing " & .SubAddress & " to " & sNewLink
                            .SubAddress = sNewLink
                            Ctr = Ctr + 1
                        End If
                    End If
                End If
            End If
        End With
    Next
Next

Debug.Print CStr(Ctr) & " links changed."

End Sub

Function FindSlideByTitle(Title As String, Titles() As String) As Long
Dim I As Integer
FindSlideByTitle = -1
For I = 1 To UBound(Titles)
    If Title = Titles(I) Then
        FindSlideByTitle = I
        Exit Function
    End If
Next
End Function
'=================================================================

If you have the option selected to update links on open then you will get a prompt, so turn that option off. You can then add the line

    ActiveDocument.Fields.Update

to an autoopen macro in the document template; and if the link is in the body of the document that will update the fields in the document (including links), when it is opened, without prompting. Whether it is a good idea to do this I will leave it to you to decide.

Next stage would be to just update link fields and for that you need a different construction.

    Dim oFld as Field
    For Each oFld In ActiveDocument.Fields
