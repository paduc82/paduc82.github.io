albert d. kallal say:

The simple matter is that you cannot use a VBA function that resides in another database and have that VBA code run from that other database.

If you use ADO, or DAO, say from FoxPro, vb.net etc., or in fact any system can request that query? Well those other systems (including your copy of Access) needs full use of that database. Heck, you not even have msaccess + VBA installed and yet that query has a VBA depedancy.

The simple solution is to

Automate a copy of the other database, and then run + call code that resides inside of that database which in turn runs the query in question.

Eg:

    set accessApp = createObject("Access.Application")

In above, the the sub TimeUpDate is standard VBA code in a standard code module that resides in that target application.

Another possible would be to import the query and code function into your existing application, and run that query.

So you cannot “out of the blue” execute VBA code that resides in other access applications unless a “copy” of that Access application is in fact loaded and running at which THEN existing VBA code and modules that are required from that application are now available. So some query in a database cannot use VBA code from another application.

You don't mention "how" you using this query from the other database, but any query that has VBA depdenances will require the SAME instance of Access running at the time when such queries are run to allow use of VBA. As  noted per above, automation of a copy, and then "calling" a standard VBA sub as per above will work.

You could also consider pulling the query + VBA code function into the other database which is attempting to use + run that query with the VBA dependency. So a query that requires a VBA function requires the VBA function to exist in the current running copy of Access, not some other external application.

ed hansberry (work acct) say:

It is possible the files are XLS files with the new extension. 2 things to try:

  1. When in the file, select file|convert. That should ensure it is a 2007+ xlsx format.
  2. Save the file as an XLSX file via File|Save As then make sure you pick Excel Workbook xlsx.

If that doesn't work, try manually renaming the file as file.xls again, then try the above two steps.
