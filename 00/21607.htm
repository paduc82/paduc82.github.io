and replacing them with these lines:

    Selection.Style = ActiveDocument.Styles("Heading 2")

The Font.Reset and Paragraphs.Reset commands will make the selection's formatting exactly what the style's definition says, without any direct (non-style) formatting.

john korchok say:

Word renames graphics when importing them, but does not convert to a different file type. Your PNG remains a PNG. 

If you have created alpha channel transparency in PhotoShop, the resulting PNG can be blurry in a saved PDF. Instead use File>Save for Web or File>Export>Save for Web (depending on your version of PhotoShop) to create a PNG with a single transparent color.

If you previously compressed the picture to a lower quality, choosing a higher quality later does not restore the lost resolution. You should replace the photo with the original file to restore the image quality.

Word does not keep track of picture resolution, so the dropdown always reads Best for printing (220 ppi) regardless of the actual resolution of the picture. It's just crummy interface design and is misleading.

In my experience, Word 2011 does not compress pictures unless you tell it to. However, Office 2011 and 2016 have some issues with resolution of graphics in saved PDFs that look like image compression. If it prints well and looks sharp on-screen, the real issue is a PDF export problem.
