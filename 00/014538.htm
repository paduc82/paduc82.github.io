Safe attachments analyzes attachments that are common targets for malicious content, such as Office documents, PDFs, executable file types, Macros, and Flash files. Outlook for Windows (150MB)

marshall barton say:

To expand on Gina's comments, you should not combine two pieces of information in a single field.  That means the Orders table should have the order date in a separate date field and the sequence number in its own Integer field.  In this case and because you want to start the sequence number over each day, the code in the orders form'sBeforeUpdate event would be like:

If Me.NewRecord Then
   Me.seqnum = Nz(DMax(seqnum". "Orders", "orderdate = " & Format(Me.orderdate, "\#yyyy-m-d\#")), 0) + 1
End If

Whenever you want to display an order in a form or report, use a text box with an expression that combines the two fields:

   =Format(orderdate, "yymmdd") &"-" & Format(seqnum, "000")
