per jessen say:

Hi Dave

We can use column C to calculate last row, then we can build the formula string and insert it in A2. Then use FillDown to copy the formula down as required.

 

Sub InsertFormula()
Dim LastRow As Long
Dim MyFormula As String

LastRow = Range("C" & Rows.Count).End(xlUp).Row
MyFormula = "=Vlookup(B2,C$2:S$" & LastRow & ",F2,False)"

Range("A2").Formula = MyFormula
Range("A2:A" & LastRow).FillDown
End Sub

john mcghie say:
Yes, there are several ways.

The issue is that the NUMBERING in Word is separate from the Style (stupid design decision...) but the RESTARTS are separate from both the Numbering AND the style (even stupider design decision!).

Essentially this means that it was "Update Styles" that was your undoing: every time you do this, you wipe out your numbering :-)  So turn that off.

Now, you have four options, as explained here by Margaret Aldis:
http://word.mvps.org/faqs/numbering/listrestartmethods.htm

For your situation, I would recommend "Restart using a higher-level style".  Pick a style that will always occur between one list and the next (I usually use "Body Text") and make that the top of your set of numbering styles.  Set your next level of numbering (Level 2) to Restart after...  "Body Text".

There is no need to have numbering applied to Body Text to use it to restart the list.  This method has the advantage that it will restart the entire numbering scheme, so the restart will behave correctly all the way down the hierarchy for nine levels of numbering if you have them.

The method Jim recommended is "Restart numbering manually with the Restart Numbering command."  As you say, not the most suitable for your purpose, but it is one of the easiest.

Restarting numbering using ListNum fields is also available to you, but I wouldn't use it because there won't be any ListNum fields in InDesign.

For my money, you are going the wrong way :-)  You should be creating in Word and formatting in InDesign.  Or simply creating and formatting in Word: you will find that a lot mroe straightforward.

It would be worth your spending time in Shauna Kelly's website: that explains Word's list numbering very well.

Hope this helps

sally75 say:

Inserting Photo Box using content control developer tools

I just noticed when I put a photo in the using developer tools photo frame changes size smaller to suit the photo rather than the photo suiting the frame? is there a way to lock this?  When I tried ticking the box do not allow editing in the Control section it took away the ability to insert a photo?

jay freedman say:

In Word 2007, when I insert a Picture content control and put in a small picture (JPG, PNG, GIF and so on), it does enlarge the picture to match the width of the control. That's usually not a good idea, because the picture becomes blurry and jaggy -- that's what always happens when you enlarge a bitmap-type picture beyond 100% of its saved resolution.

I don't know why your control behaves differently, unless it has something to do with the type of picture you're inserting. Certainly there are no options or other ways of controlling it.

The "do not allow editing" option is something completely different. All of the different kinds of content controls have an option with the same wording -- it really means "don't allow the user to change what's shown in this control". If there's no picture in the Picture control when you check that box, then you won't be allowed to change it to show some picture -- it must remain empty. In practice you would never do that; you might insert a picture first and then check the box so the picture couldn't be replaced with a different one.

macropod say:
When you use a picture content control, the size of any picture inserted into it will be constrained to fit within the content control's dimensions, whilst maintaining the correct aspect ratio for the picture. If you want to change how large the picture can be, click on the  picture content control's light-blue shaded area, then use sizing tools on the 'Picture Tools' ribbon tab to adjust the content control's size.
