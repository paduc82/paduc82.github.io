chiefjoboo say:

Your speed tests with your xbox and on your computer won't be the same because the speed tests on the xbox go to an Xbox server while your computer's tests go to a different server (depending on the site you're using).

The latency on your xbox's speed test doesn't necessarily mean that all of your games' connections will be at 150ms ping as multiplayer games are either played on dedicated servers or peer-to-peer connections. Games that run on dedicated servers and games that use peer-to-peer connections will not always have the same connection strength or latency. Some servers you might have 30ms or less ping to while some peer-to-peer servers might have 100ms ping+ (or the opposite).

Now, to be sure, the 'Lag' you're talking about isn't the lag associated with LED TVs? Most new, big, LED TVs actually have a latency form when you move your controller to when it shows on your screen. That, you can not get rid of, unfortunately. The best kind of TVs to play games on are LCD TVs because almost all of them have a 5ms latency for reaction. Look your TV up and see what it says for latency in its specs.

Matthew, 

We appreciate you posting in Microsoft Community. We noticed that your post was idle for almost three days already. We're so sorry about the long delays that left your post unattended. We would like to know if you would still need assistance on this?

If so, and to address your query, the command line was designed not change the entire exe file nor its path at all. Therefore, if you would pin a shortcut with a command line and another without any command line on its path, the Start menu will read the shortcuts as one and will not be able to pin them both. 

The work around for this is to create a copy of the original Excel executable file (EXCEL.exe) on the same file location. Create a shortcut with a command line for this one and pin it on the Start menu. Start will no longer read them as one exe file, and you will now be able to have them both.
