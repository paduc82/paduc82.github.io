lcm99 say:

Word INCLUDETEXT Select Specific Element Via XPath

Hello, I am trying to use the INCLUDETEXT xml option to populate a list of references in a Word document. 

I want to specify a specific xml node based on an Xpath that selects the node based on specific child node value.

For example, for the below xml:

<?xml version="1.0" encoding="UTF-8"?>

<events> 
  <event> 
    <title>Number 1</title>  
    <status>Draft</status>  
    <description/> 
  </event>  
  <event> 
    <title>Number 1</title>  
    <status>Live</status>  
    <description/> 
  </event>  
</events>

the XPath:

events/event[status='Live']/title

should select the title "Number 1".  I believe the XPath is correct.  In my real application, I am using namespaces correctly, and I am able to get basic XPath queries to work.

Is this XPath functionality supported in Word?  I am able to select a specific nodes via the [] selector, but this is not flexible enough for my needs.

I am using Word 2013.  

Thanks in advance.

**Post moved / split by the moderator to the appropriate forum category / another thread

a. user say:

The starting point is something like

{ INCLUDETEXT "x.xml" \x "events/event[status='Live']/title" }

However, you may also need to define a namespace to reference the nodes correctly. e.g.

{ INCLUDETEXT "x.xml" \n "xmlns:n='suitableNSname'" '\x "n:events/n:event[status='Live']/title" }

or for more than one namespace,

{ INCLUDETEXT "x.xml" \n "xmlns:n1='NS1' xmlns:n2='NS2'" '\x "n1:events/n2:event[status='Live']/title" }

Something to bear in mind is that if you are going to use the \t parameter to specify a transform, the xPath must select a single node. The transform should then be able to process that node and its children. If the xpath selects multiple nodes, the transform must, in effect, process multiple documents, and (a) version 1 of XSL, which we are stuck with, does not let you do that and (b) Microsoft has not attempted to work around the shortcoming, e.g. by invoking XSL with multiple requests.

bill manville say:

The code worked for me.

Two potential issues you have highlighted.  Identifying the last cell - I should have said Ctrl+down arrow - ie to get to the last populated cell in column A, assuming that the data is contiguous from row 7 to that row.  Alternatively, if it's easier to work up from the bottom

  lLastRow = Cells(Rows.Count, 1).End(xlUp).Row

Second issue - start the filling at row 10.  Change the 2 occurrences of 8 to 10

Sub CheckBoxClicked()
  Dim iCol As Integer
  Dim lLastRow As Long
  lLastRow = Cells(7, 1).End(xlDown).Row
  With ActiveSheet.CheckBoxes(Application.Caller) ' the checkbox that was clicked
    iCol = .TopLeftCell.Column
    If .Value = xlOn Then
      Range(Cells(10, iCol), Cells(lLastRow, iCol)).Value = "f"
    Else
      Range(Cells(10, iCol), Cells(lLastRow, iCol)).ClearContents
    End If
  End With
End Sub

The name of the checkbox that is clicked is available to the macro as Application.Caller and the TopLeftCell of that Checkbox is the cell it resides in so that's how we get the column.
