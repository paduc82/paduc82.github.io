glenn_sherwood say:

AutoSave button grayed out and not working in Excel -- Work Around

I am using Excel Version 1806 (Build 10228.20080).

I've have had a long-standing problem with turning on the "AutoSave" function in "some" of my files. In these files the "AutoSave" button is grayed out and the help information that comes up when the mouse is hovered over the icon tells me to save the file to my OneDrive.  Nothing I've done through Save-As saves from within the file itself, uploads from my computer to the OneDrive from within OneDrive, drag and drop.... has been successful.

Searches of this Community and the web in general have brought up some discussions, but without resolution for me. 

One frequent observation I make is that Microsoft is aware of the problem and is working on it.  If they've got the correction in place, I'm unaware of it and assume that I should have been covered as my Office suite is automatically upgraded with each new version published.  If my assumption is wrong, you need read no further.

I think I have a work around to share.  The approach is, in brief, to copy all tabs from the misbehaving file into a "New" blank file, that you save to the OneDrive destination before you excute the tab copy command.  Steps follow, if they help.  

  1. Open the file in question.  Observe the "AutoSave" toggle icon is indeed grayed out.  
  2. Open a second blank file using Open, New commands.  
  3. Immediately save 2nd file to your OneDrive using a distinct file name that you can change later.  Observe the "AutoSave" toggle icon to see that it is active.  If not, stop here as what follows will not work.
  4. Switch to the first file and select "all" tabs in that file.  Click on the first tab, move the mouse to the last tab and while holding the shift key, click on the last tab.  All tabs should be highlighted.  
  5. Right click on a tab to bring up a drop down menu. 
  6. Click on the "Move or copy..." option.
  7. In the first field specify the 2nd file using the drop down arrow as the destination for the Move or Copy command.  When you do, the second field in this dialog box will change to list the single (empty) tab available in the 2nd file.  It will be listed as "Sheet1", unless you renamed it.  If you are trying to move or copy many tabs, then I recommend you click on the  "(move to end)" option listed below "Sheet1".  Doing so will make it easier to select the "Sheet1" tab in the 2nd file for the purpose of deleting or using it later.  
  8. I also recommend clicking on the "Create a copy" box.  This will preserve your original file for deletion after you are satisfied the work around has worked.  
  9. Click okay.

Now you should be able to save this file to the OneDrive location, simply by hitting the save icon or using File, Save commands.  And the "AutoSave" icon should be active.  

If the bug has been corrected, please advise!

ken sheridan say: Base the subform on a query which references the option groups as parameters, e.g.

SELECT *
FROM [Patient List]
WHERE ((Forms![Patient Search]!frCat = 1 AND Category = "Adult")
    OR (Forms![Patient Search]!frCat = 2 AND Category = "Child")
    OR Forms![Patient Search]!frCat IS NULL)
AND <do similarly for other option groups>;

In the case of the Boolean (yes/no) column to which the check bound in the subform is bound it's not quite clear from your post what you want.  If you want rows with a True value in the column returned if one option is selected and rows with a False value in the column returned if the other option is selected then you'd do it like this:

AND ((Forms![Patient Search]![OptionGroupName] = 1 AND [BooleanColumnName] = TRUE)
    OR (Forms![Patient Search]![OptionGroupName] = 2 AND [BooleanColumnName] = FALSE)
    OR Forms![Patient Search]![OptionGroupName] IS NULL)

On the other hand, if you want rows with either True or False values returned if the second option is selected then you'd do it like this:

AND ((Forms![Patient Search]![OptionGroupName] = 1 AND [BooleanColumnName] = TRUE)
    OR Forms![Patient Search]![OptionGroupName] = 2
    OR Forms![Patient Search]![OptionGroupName] IS NULL)

In the AfterUpdate event procedure of each option group requery the subform control with:

    Me.[SubformControlName].Requery

NB:

1:  SubformControlName is the name of the control in the parent form which houses the subform.

2.  The positioning of the parentheses in the query's WHERE clause are important to force the OR and AND operations to evaluate in the correct priority.
