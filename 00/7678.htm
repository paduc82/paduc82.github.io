    Marco - First, I realize this thread is now over 2 years old.  But I thought this information might be useful to others in the future.

    When you use the VBA commands GetSetting() and SaveSetting the information actually has its root in

    HKEY_CURRENT_USER\Software\VB and VBA Program Settings\

    So if you execute this statement:

    SaveSetting "MyUniqueKEY","JLATHAM","UName","JLatham"

    You end up with:

    HKEY_CURRENT_USER\Software\VB and VBA Program Settings\MyUniqueKEY\JLATHAM

    with a value in it called UName that has "JLatham" assigned as its value.

    The

    HKEY_CURRENT_USER\Software\VB and VBA Program Settings\

    default path is not user alterable.

     

    If you want to confirm this or find where these writes to the registry are being placed at, you can always execute a SaveSetting using a very unique entry for the first parameter and then use the registry editor to search for that entry to lead you to its location.

    ken sheridan say: It sounds like you want to do something broadly similar to what I did a while ago with phone numbers for sales staff, where one number is designated as the primary number by means of a Primary column of Boolean (Yes/No) data type in a SalespersonPhoneNumbers table.  In the BeforeUpdate event procedure of the check box bound to the primary column in the subform the following code executes, subject to user conformation, to set the value of the Primary column in all other rows for the current salesperson to false:

        On Error GoTo Err_Handler
       
        Const conMESSAGE = _
            "Another number is set as primary landline " & _
            "number for this salesperson.  Do you wish to replace " & _
            "it with this number?"

        Dim dbs As DAO.Database
        Dim strCriteria As String
        Dim strSQL As String
       
        strCriteria = "SalespersonID = " & Me.SalesPersonID & _
            " And PhoneType = ""LandLine"" And Primary = True"
           
        ' get user confirmation to change primary landline
        ' number for this salesperson
        If Me.Primary Then
            If Not IsNull(DLookup("salespersonID", "SalespersonPhoneNumbers", strCriteria)) Then
                If MsgBox(conMESSAGE, vbQuestion + vbYesNo, "Warning") = vbYes Then
                    ' set all other landline numbers for this salesperson to non-primary
                    strSQL = "UPDATE SalespersonPhoneNumbers " & _
                        "SET Primary = FALSE WHERE SalespersonID = " & Me.SalesPersonID & _
                        " And PhoneType = ""LandLine"""
                    Set dbs = CurrentDb
                    dbs.Execute strSQL, dbFailOnError
                    Me.Refresh
                Else
                    Cancel = True
                End If
            End If
        End If

    Exit_Here:
        Exit Sub
       
    Err_Handler:
        MsgBox Err.Description, vbExclamation, "Error"
        Resume Exit_Here

    In your case the principle is similar, but you'd need to assign an expression to the strCriteria variable to examine each of the four columns in the table for True values in the case of the MatchIt or MarckWac columns, or for Not Null or not a zero length string in the case of the MakeCustSecID column, or Not Null or not 0 in the case of the Val column (note Tom's caveat about the name of this last column, however).  You'd also need to extend the SQL statement to update all four columns, setting them to Null or zero as appropriate.
