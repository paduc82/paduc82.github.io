aldint.4428 say:

The problem is solved.

We didn't have any problems whatsoever with our setup so it didn't look like an issue, but apparently we need an internal cname record for autodiscover.

We had only an SRV record, but no CNAME record.

Also the connectivity analyzer gave no warnings before so we thought that everything was okey with our Autodiscover.

So, the problem was solved after we've created an internal CNAME record in our dns.

biddu say: Hi,

You can use MATCH function within VLOOKUP to utilize the relative coding of the table structured references.  Please find below Excel 2007 and 2010 examples to achieve the same result as your hardcoded formula above.  Replace "Column10" in the below formula with the Header name of the 10th column in the second table to achieve the result you are looking for.

MS Excel 2010 Formula:

=VLOOKUP(Table1[@LoanNumber], Table2[#All], MATCH(Table2[@Column10],Table2[@], 0), FALSE)

MS Excel 2007 Formula:
=VLOOKUP(Table1[[#This Row],[LoanNumber]], Table2[#All], MATCH(Table2[[#This Row],[Column10]],Table2[#This Row], 0), FALSE)

Hope this helps!

It's a paragraph mark. See http://www.word.mvps.org/FAQs/Formatting/NonPrintChars.htm.

Usually it appears and disappears (along with the dots in the spaces between words, and arrows for tab characters) when you click the Â¶ button on the Home tab. jay freedman say:
The behavior of that set of options is a bit unintuitive. It works like this:
  • Clicking the Â¶ button or pressing Ctrl+Shift+8 or checking/unchecking the "Show all formatting marks" box in the Options dialog are all equivalent. If you turn on the button so it's highlighted, that will also check the box in Options; and if you check the box in Options, that will turn on the button. The result is that all of the formatting marks in the document are shown, even when all of the other boxes in that section are unchecked.
  • If you check one or more of the boxes for specific types of formatting marks -- just paragraph marks, say, or just tab characters -- then those kinds of marks will continue to be shown regardless of the setting of "Show all" or the Â¶ button. The kinds of marks whose boxes are still unchecked will be shown or hidden by turning the "Show all" on or off.

So what you saw was exactly what is supposed to happen, although the rules aren't obvious.
