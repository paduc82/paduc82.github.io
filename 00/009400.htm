So what's your question?

Yes, different devices support different Bluetooth profiles. This is normal and true for almost every device out there. There are many to choose from. it's no-ones fault if two particular devices don't overlap. it just means that Jeep and MS and HTC didn't get around a table and discuss what they were doing, they just released 2 different products to do two different things. There is no legislation requiring them to work together.

MAP is one method of sharing text messages between devices. this means the car can compose a message (by voice) and then forward that to your phone to be sent as a text message (SMS) it's there to provide people with dumbphones an ability to text from their cars, by the car doing the message composition, or text to speech conversion, on phones that can't.

Given that WP already has such speech facilities built it, you're better off using a hands free profile, and simply feeding the audio into the phone for processing, rather than using a separate profile for voice calls, text messages, and possibly music, or other, I can't see a reason to increase the cost, or complexity, for the sake of a one trick pony profile.

If you want to have text messages read out to you, and dictate them back, then you can do this already without MAP. If what you actually want to do is send and receive text messages via MAP, then no, you can't, but I can't see what the difference is.

1. Find where SkypeSetup.exe downloads to (C:\ Windows\Temp), note the path, and delete SkypeSetup.exe

2. Right click on Win 10 Start Menu icon, click on Command Prompt (Admin) or PowerShell (admin), and paste in this command at the prompt. If Win 7 or 8 run Command Prompt as administrator.

3. At the prompt type - netsh advfirewall firewall add rule name=SkypeSetup dir=in action=block program=[Path to where SkypeSetup.exe downloads to like C:\ Windows\Temp] enable=yes

For example my command was:

netsh advfirewall firewall add rule name=SkypeSetup dir=in action=block program=D:\Temporary_System_and_User_Files enable=yes

Works great, until... I save the print area to PDF using the inbuilt functionality in Excel 2010.

 

The generated PDF shows hyperlinks that look OK but the link actually only takes you to the first part of the concatenated URL (the base URL).

 

Eg this link: https://www.support.unisys.com/common/epa/macro.aspx?path0=a-s&path1=ple&path2=web-select&SELECT=PLE-PLE-TO-IC+eq+MCP-054.1A.117&TITLE=PLE(s)+released+since+IC:+MCP-054.1A.116
