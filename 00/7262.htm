(For anyone experiencing similar problems)

I had a look at the OP's Excel file, and it seemed to have been created in a way that bypassed some of Excel's normal date-time formatting. So the OP decided to have a closer look at how the workbook was created.

I wondered whether the inconsistencies might result from some common procedures such as opening a .csv file or importing from a delimited file. But simple tests suggest that Excel "gets it right" when you do that.

I had a look at the XML representation of the spreadsheet inside the .xlsx

The result is basically that in a column with no formulas that Word (i.e. the Excel OLE DB provider) has determined is a date type column 
 a. the XML representation for each cell either contains a value or a reference to a value in a â€œShared String Tableâ€.
 b. the only cells that Word treats as Dates rather than displaying as â€œ12:00:00 AMâ€ are cells that directly contain a value rather than containing a reference to a value in the string table.
 c. The only values that Word treats as dates are Excel Date/Time numbers. The date in these is a day number starting 1/1/1900 (or 1899 or 1901 - I forget). So, a date stored as anything other than a directly stored day number  (such as 42772) is not treated as a date by Word, however much it looks like one. If the cell value  is â€œ01/01/2017â€, â€œ1/1/2017â€, â€œ01/1/2017â€ etc. etc,, Word does not treat it as a date. if it is formatted as text or has a text value (e.g. where you precede it by a "'" mark in the formula box in Excel), Word does not treat it as a date.
 d. A Date cell with value 42772 is displayed as a date and treated as a date by Word
 e. A General cell with value 42772 is displayed as a number and treated as a date by Word.

 f. Some other combinations are treated as errors when you open the .xlsx. For example, if you have a cell formatted as Date and you try to put â€œ2/5/2017â€ as the value, rather than putting â€œ2/5/2017â€ in the Shared String Table and making a reference to it, Excel will complain. Excel also complains if you put something such as â€œ42772" in the Shared String Table, saying â€œNumber stored as textâ€.


If you have a column that the OLE DB provider has determined to be *text*, then things are different. For example, a text that looks like "1/1/2017" will be treated as a date. But that isn't really the scenario I was looking at in this case.

bernie deitrick say: Chip's page does not explain how to use the function as a worksheet function.  So, I modified his code for you. Copy the function below, paste it in your module, then select the cells below B31 - say, B32 to B100 - and array enter (enter using Ctrl-Shift-Enter) the formula

=SplitExChars(B31,"{}:,[]")

When you do that correctly, Excel will show the formula with { } around it. If you don't get blanks at the bottom of your entry cells, increase the size of the range you are using.

And here is the function definition:

Function SplitExChars(ByVal InString As String, _
Delims As String) As Variant
    Dim Arr As Variant
    Dim Ndx As Long
    Dim N As Long
    Dim iSize As Integer
    
    If Len(InString) = 0 Then
        SplitExChars = ""
        Exit Function
    End If
    
    For Ndx = 1 To Len(Delims)
        InString = Replace(InString, Mid(Delims, Ndx, 1), Chr(1))
    Next Ndx
    
    N = InStr(1, InString, Chr(1) & Chr(1), vbTextCompare)
    Do Until N = 0
        InString = Replace(InString, Chr(1) & Chr(1), Chr(1))
        N = InStr(1, InString, Chr(1) & Chr(1), vbTextCompare)
    Loop
    
    If Left(InString, 1) = Chr(1) Then InString = Mid(InString, 2, Len(InString))
    If Right(InString, 1) = Chr(1) Then InString = Left(InString, Len(InString) - 1)
    
    'This removes the double quotes
    InString = Replace(InString, """", "")

    Arr = Split(InString, Chr(1))
    
    iSize = Application.Max(UBound(Arr), Application.Caller.Rows.Count, Application.Caller.Columns.Count) - 1
    If iSize > UBound(Arr) Then
        ReDim Preserve Arr(0 To iSize)
    End If
    
    If Application.Caller.Rows.Count = 1 Then
        SplitExChars = Arr
    Else
        SplitExChars = Application.Transpose(Arr)

This may be a bit simpler since you mention Excel;

  1. Remove the bullets in the list
  2. Cut the list
  3. Paste into an Excel sheet [let's say A1]

  8. Paste into the Word document
  9. Reapply the bullets
