ron coderre say:

If you create a sheet-level, Dynamic Named Range (DNR)named: Print_Area
and refer to you list, it will automatically expand and contract
to accommodate the number of rows in your list.

Example:
Your list is on Sheet1, cells B3:D27
There may be values above that range, but NO non-list values below it

• Insert.Name.Define
...Names in workbook: Sheet1!Print_Area
...Refers to: =OFFSET(Sheet1!$B$3,,,COUNTA(Sheet1!$B$3:INDEX(Sheet1!$B:$B,1048576)),3)
...Click: OK

Now, if you add more data to the bottom of the list or delete rows from the list,
The DNR will still include the list items.

NOTE: Do NOT use File.Set_Print_Area on that sheet. It will replace the DNR with a fixed reference.

Is that something you can work with?
