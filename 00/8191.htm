Changing the date on multiple worksheets at once - Microsoft Community

susan stroud - ssc say:

Changing the date on multiple worksheets at once

I have 52 worksheets in a workbook and want to enter a date on the first worksheet and each additional worksheet to add 7 to that date.  is there a formula that will do this automatically if I select all sheets or will i have to enter the formula on all 52 sheets manually.  This would be very helpful

 

Thank you!

rick rothstein, mvp - excel say:

Assuming the first 52 sheets in the workbook are the sheets you are talking about, running this macro will install the correct formulas in Worksheets 2 thru 52 (Worksheet 1 is where you manually enter the date to start things off). I've assume cell A1 on each sheet is the cell where your dates are displayed...

Sub PutFormulaOnSheets()
  Dim X As Long, WeekNum As Long
  For X = 2 To 52
    With Worksheets(X).Range("A1")
      WeekNum = 7 * (X - 1)
      .Formula = "='" & Worksheets(1).Name & "'!A1+" & WeekNum
      .NumberFormat = "mm/dd/yyyy"
    End With
  Next
End Sub

This macro can be deleted once you have run it as you won't need it again after you have saved the workbook.

Now as far as all your contact folders go, the first suggestion I would make to better follow along is to set your Outlook Navigation Pane to "Full Folder" view by pressing CTRL+6. That will show each data file in your profile and the folders contained in each of them.

From the screenshots

#1 - You have several contact folders with the designation "This Computer Only" which means that these are created in the folder tree for an IMAP account. One of those folders named (Contacts - This Computer Only), means that the first email account configured in this profile is an IMAP account. That folder can't be deleted from within Outlook (exception to this will be noted later on)

#2 - You have countless PST files for <yokum @ npquality......>. Not sure what that's all about or if you need all those files given that they seem to have the same name. #3 - You have a number of iCloud contact folders so that would mean you were or are still using iCloud which would have its own contact folder(s).

So the real question now becomes, which contact folder(s) do you actually want to keep? Answer to that will dictate what you ultimately have to do but some background info regarding contact/calendar folders is in order

a) An PST file that has a "Contact" folder in it means that it's from a prior installation where it was previously set as the "default data file" in the profile. That folder cannot be removed from within Outlook. If you need all those PST files, then you will need to keep all the contact folders. (Note: if you're quasi technically inclined, those folders can be removed using a tool called MFCMapi but will leave that for another post)

b) The first email account configured in an Outlook profile sets the associated data file for that account as the "default data file" which is where the default contact/calendar folders are stored. Every subsequent POP3/Imap account added will not have any contact/calendar folders in the folder tree for that account, instead using the default contact/calendar folders.

So how to clean this all up...................

#4 - First decide what you need and don't need in this profile by way of data files

#5 - You have one PST file set as the default already, so logic would dictate that this is where you would want to keep your contact data. If that's the case, make sure to copy all the contact you want/need from all the other folder to the contact folder in this folder tree. (you can't set the iCloud data file as the default for the profile - that's managed by the iCloud addin)

#6 - If you don't want any contact folders in your IMAP folder tree and you have everything copied over that you need, then remove the IMAP account from the profile and then re-add it. Since you already have a PST file set as a default data file, the IMAP folder tree won't have any contact folders unless you create them yourself at some point in the future.
