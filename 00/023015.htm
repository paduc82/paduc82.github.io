My Nokia BH 903 review ! - Microsoft Community

thechosenone22 say:

My Nokia BH 903 review !

Given I'm a bit of an afficienado of Bluetooth headsets, I jumped at the chance of reviewing the Nokia BH 903 Bluetooth stereo headset and found it to be a fully featured device with some interesting features.

Links for the UK version of all the devices detailed in this review can be found at the bottom of the page.

Opening the box itself you get the device itself, a couple of foam earpads, a new style (2mm) Nokia charger as well as the manual.

Although described as a "headset" the first thing to note about this device is that it uses earphones and not the "in the ear" canal type connectors found on the BH 103 model or the "traditional" over the head type headphones such as the BH 604 model.

This is particularly important if you have the type of ears where you always find your iPod earphones always fall out if you're running for the bus ! For this reason I've always tended to purchase devices with the "in the ear" canal type connectors, however I found the BH 903 to be very pleasant in use and I think this is in no small part due to the triangular shape of the earphone itself which is partly encapsulated in rubber with foam pads also being supplied with the unit. For those with the "iPod ear issue" as I like to call it, these devices should definitely be added to the "must consider" list.

The major problem I've found with most Bluetooth stereo headsets so far is actually portability. Devices such as the Nokia HS 81 or (even worse) the Motorola S9 have a hard "frame" which goes around the neck of the wearer which whilst comfortable, is definitely not something you can put in your pocket whilst the device is not in use without worrying about whether you are going to hear a cracking sound when you sit down.

Happily there's no such issue with the BH 903 - it features a fully adjustable soft "neck" band which sits around the neck with the actual device hanging off this similar to how a pendant would be worn. Each of the earphones is then threaded through each end of the neckstrap and can be adjusted depending on how close you want the main controller unit to sit near your neck or how much slack you require for each earpiece. On the lead running to the left earpiece is the "satellite" controller unit which houses the microphone and answer/end/mute keys.

The device itself is A2DP compliant and so can be paired with any music player also adhering to this standard (as well as any phone !). It is capable of remembering the last 4 paired devices and features its own subliminal display (similar to that found on the front of the N93i) which enables it to give much more feedback and functionality to the user than other lesser featured units. The styling of the BH 903 most closely follows that of the recently announced N78/N96 and features a similar touch sensitive area by which the headset volume can be controlled.

The BH 903 manages to pack a lot of features in it's remarkably small frame - the device contains a FM receiver with the earphone cables doubling as the antenna - I found reception to be crisp and it managed to automatically pick up and save local radio stations complete with station name (the BH 903 supports RDS) with absolute ease.

As the device has a display it is also able to offer additional functionality and feedback to the user which is not apparent in other devices - for example if Bluetooth connectivity is lost to the host device, it can be reinitiated from the headset itself. The main control unit also offers music player control settings as well as mirroring the call control functionality available on the microphone unit. With compatible devices, the BH 903 can also display the current time, and has automatic keyguard functionality.

In use, I found the volume for both music playback and calls absolutely fine unlike other non Nokia devices which haven't really been loud enough for noisy environments. Likewise the microphone also performed well, although I found that due to the technology and location it did pick up quite a lot of ambient sound as well as the caller's voice - so if you're at a pub or club when you should be somewhere else this is definitely NOT the headset you want to be using to call your other party ! :)

Not for the first time in evaluating Nokia devices, I am now lamenting the fact that I already have a device of the same family that I am currently reviewing, as I would quite happily have purchased this device had it been available at the time of purchase instead of the one I am currently using. Definitely a headset to put on your shortlist !

graham mayor say:

If you have control over the printers and you know the names of the printer drivers and the Tray IDs for those printers (seehttp://www.gmayor.com/fax_from_word.htm ) then it is a fairly simple matter to create a macro that will print pages 1 to 4 and 6 to one tray and page 5 to another (or any combination of trays and pages)

You haven't said which printers you actually have, so I cannot be 100% sure that the tray IDs I have quoted are correct - you can use the above link to verify. With that established, the following macro should work. Note that the three Inst statements which search for the driver names are case sensitive. If you have more than one HP or Lexmark driver available you will have to fill the required names fully, or pick from a list - see the routine for selecting a colour printer.

Dim sPrinter As String
Dim sTray1 As String
Dim sTray2 As String
Dim sDefaultTray As String
'Ascertain which is the default printer
sPrinter = Application.ActivePrinter
'And set the trays appropriately
sDefaultTray = Options.DefaultTrayID
If InStr(1, sPrinter, "HP Laser") Then
    MsgBox "Printing to HP LaserJet", vbInformation, "Printing"
    sTray1 = 261
    sTray2 = 260
End If
If InStr(1, sPrinter, "Lexmark") Then
    MsgBox "Printing To Lexmark", vbInformation, "Printing"
    sTray1 = wdPrinterUpperBin
    sTray2 = wdPrinterLowerBin
End If
If InStr(1, sPrinter, "HP Laser") + _
InStr(1, sPrinter, "Lexmark") = 0 Then
    MsgBox "Set the active printer to HP LaserJet or Lexmark" & vbCr & _
    "and re-run the macro", vbCritical, "Wrong Printer"
    Exit Sub
End If
'Print pages 1 to 4
Options.DefaultTrayID = sTray1 'print to tray 1
Application.PrintOut FileName:="", Range:=wdPrintRangeOfPages, Item:= _
        wdPrintDocumentContent, Copies:=1, Pages:="1-4", PageType:= _
        wdPrintAllPages, ManualDuplexPrint:=False, Collate:=True, Background:= _
        True, PrintToFile:=False, PrintZoomColumn:=0, PrintZoomRow:=0, _
        PrintZoomPaperWidth:=0, PrintZoomPaperHeight:=0
'Print page 5
Options.DefaultTrayID = sTray2 'print to tray 2
Application.PrintOut FileName:="", Range:=wdPrintRangeOfPages, Item:= _
        wdPrintDocumentContent, Copies:=1, Pages:="5", PageType:=wdPrintAllPages, _
         ManualDuplexPrint:=False, Collate:=True, Background:=True, PrintToFile:= _
        False, PrintZoomColumn:=0, PrintZoomRow:=0, PrintZoomPaperWidth:=0, _
        PrintZoomPaperHeight:=0
'Print Page 6
Options.DefaultTrayID = sTray1 'print to tray 1
Application.PrintOut FileName:="", Range:=wdPrintRangeOfPages, Item:= _
        wdPrintDocumentContent, Copies:=1, Pages:="6", PageType:=wdPrintAllPages, _
         ManualDuplexPrint:=False, Collate:=True, Background:=True, PrintToFile:= _
        False, PrintZoomColumn:=0, PrintZoomRow:=0, PrintZoomPaperWidth:=0, _
        PrintZoomPaperHeight:=0
'Reset default
Options.DefaultTrayID = sDefaultTray

rohn007 say:

Yes, the activation process does count installs. 

What you are talking about is known as "transferring" the license from the dead machine to the new, "third" install.  That is allowed in the license and activation system.   In order to do that you have to ensure that the "old machine" is no longer running Office.  Normally that is done by uninstalling Office, BUT ..., a "dead" computer also qualifies as an "uninstall".

The uninstall process does NOT talk to the activation system, unfortunately.

For the 3rd machine, you do a normal install and activation.  Sometimes the activation system will recognize that the "dead" machine has not been used in some undefined long period and will assume you are transferring the installation and allow internet activation.  As you are concerned about, normally it will not, triggering an error message about "too many installs".  That is still not a problem.  Instead of internet activation you have to do a phone activation.  But you have to follow a special procedure.  If you follow the normal phone activation prompts you will also get the same error message.  Instead, when you phone in you have to wait a long time without replying to the prompts (or enter a bunch of wrong replies) until the phone system recognizes that you "need help" and offers you the option of talking to a live "service" rep.  Explain that you are transferring the license from a dead computer. They should immediately allow the activation to complete.  If they do not co-operate, ask to speak to a supervisor.

As Palcouk pointed out, MS has partially changed the activation system with the current version of Office.  Office 365 licenses allow the USER to control their multiple activations by logging in to a web page.  USERS now can 'deactivate' an installation to allow them to easily use that install count on another machine without MS intervention. 

HOWEVER, the "Office 2013" licenses, (separate from "Office 365/2013") still does not give users that control.

If you have POP and IMAP accounts only, all contacts are going to be "on your computer". If you have an Exchange account, you will have a section for the Exchange server, and another one labelled "On My Computer".

You have to make sure the Me card is selected amongst the On My Computer section.
If you have an Exchange account but the group doesn't appear, check the General Preferences under Outlook and make sure the Hide On My Computer Folders option is not selected.
