IF you're using a postscript printer, and assuming your image is in the page header, you could erase the image at the printer by embedding a PRINT field in the header immediately after the image's anchor. The PRINT field would be coded as:

{PRINT \p page "erasepage"}

Presumably other printer languages have a similar "erasepage" fucntion, but whether they work the same way as the postscript one probably depends on the implementation.

The macro solution is simple. Assuming the image is the only image in the page header, then the following will do the job: Add the QuickPrint button (FilePrintDefault) to the QAT (Quick Access Toolbar) and click that to print.

Sub FilePrintDefault()
Dim oHeader As HeaderFooter
Dim oShape As Shape
Set oHeader = ActiveDocument.Sections(1).Headers(wdHeaderFooterFirstPage)
Set oShape = oHeader.Shapes(1)
oShape.PictureFormat.Brightness = 1#
MsgBox "Printing" 'Optional
'ActiveDocument.PrintOut 'Remove apostrophe from the start of the line after testing
oShape.PictureFormat.Brightness = 0.5
End Sub

ron coderre say:

Well...the first formula uses SUMIFS and the second formula uses SUMPRODUCT.
The SUMIFS function is covered fairly well in Excel Help and is easiest to describe:

This formula:
=SUMIFS('Daily Tracking'!2:2,'Daily Tracking'!1:1,">="&(TODAY()-DAY(TODAY())+1),'Daily Tracking'!1:1,"<"&TODAY())

would read this way:
Sum all cells in 'Daily Tracking'!2:2
where the cells in 'Daily Tracking'!1:1 are greater than or equal to the first day of the current month
AND...
where the cells in 'Daily Tracking'!1:1 are also less than the today.

Study the formula to see the component values. 

The SUMPRODUCT function has a few variations and is very powerful.
Bob Phillips exhaustively covers it at his website:
http://www.xldynamic.com/source/xld.SUMPRODUCT.html
