ken sheridan say: Are you sure that the 'certain' dates are compatible with the system's regional date format settings?  What happens if you format the controls as short date and enter the same dates by means of the date-picker?

One other, probably unconnected, issue is your use of RIGHT OUTER JOINS.  What is the reason for this?  As your are restricting the query on a column in a table (Sale) on the left side of a RIGHT JOIN this will suppress the preservation f rows on the right side of the join.  If you think about it, it is logically inconsistent to return rows where there is no match in a referenced table, but at the same time restrict the result set to rows in that table where there is a specific value in one of its columns.  In effect the join becomes an INNER JOIN.

If it is necessary to do this sort of thing, i.e. return all rows from one table regardless of a match in another table, but at the same time restrict the data returned from the other table to only those rows from the other table which meet a criterion, the outer join should not be between the tables, but between one table and a subquery in which the restriction is imposed.  You'll find an example in the 'building more advanced queries' section of DatabaseBasics.zip in my public databases folder at:

I think you've inadvertently enabled Draft Font. Go to File | Options | Advanced: Show document content and clear the check box for "Use draft font in Draft and Outline views" (the default draft font is Courier New).

The macros just turn off the Draft font, but to ensure that the program starts in Draft view you can easily add a line of code as follows:

Sub AutoOpen()
