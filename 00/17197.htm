joeu2004 say: IYF wrote:
I just was stuck on the IS function.
Dates are simply numbers formatted as dates.  If you temporarily format A1 as Number, you will see what I mean.  Normally, a date is represented by the number of days since 12/31/1899 (aka 1/0/1900) on MSWin.
I'm thinking the "COUNT" function is similar to "ISNUMBBER"

In this context, they serve the same purpose.  Use whichever one you feel comfortable with, IMHO.

I chose ISNUMBER as an analog to the IS functions you were using, namely ISBLANK and ISTEXT.  But it might be noted that COUNT can be used in more general circumstances.  For example, COUNT(A1:A3) can be used to easily determine if all 3, any 2 or exactly 1 is a number.

ISNUMBER looks at only one value and returns TRUE or FALSE based on whether it is a number.  COUNT, as Biff used it, counts the number of cells that contain a number, not text or empty.  So in this context, COUNT will return 1 or 0, which are interpreted as TRUE or FALSE.  (Any non-zero value is interpreted as TRUE.)

Note that if A1 contains the __text_string__ "1234", ISNUMBER(A1) will return FALSE, and COUNT(A1) will return 0; they are consistent.  But COUNT("1234") returns 1, interpreted as TRUE in your context, whereas ISNUMBER("1234") returns FALSE.  That is one subtle difference between COUNT and ISNUMBER.  It should not affect your usage.

When A1 contains a number, ISNUMBER(A1) is about 8% faster than COUNT(A1) on my computer.  (YMMV.)  That should not be surprising, since COUNT is designed for a bigger task, namely to count over a range.

However, in XL2003, when A1 contains text, ISNUMBER(A1) is significantly slower than ISNUMBER(A1) when A1 contains a number -- about 1.9 times slower on my computer.  That seems inexplicable.

In contrast, when A1 contains text, COUNT(A1) is slightly faster than COUNT(A1) when A1 contains a number -- by about 2% on my computer.  That makes sense since when A1 is text, there is nothing for COUNT to count -- less work to do.

Consequently, when A1 contains text, ISNUMBER(A1) is about 1.8 times slower than COUNT(A1) on my computer.  Similar differences, albeit slightly smaller, are apparent in comparing the performance of IF(ISNUMBER...) and IF(COUNT...).

But keep in mind that I am talking about __nanosecond__ differences, the largest being about 300 nanoseconds on my computer.  You would need to recalculate 3.3 million of these formulas in order to see a 1-second difference in performance [1].

In conclusion, the performance differences are nothing to lose sleep over, IMHO.

-----

Endnotes

[1] I am referring to suggestions for your original requirements, namely IF(ISNUMBER(A1),"C","N") and IF(COUNT(A1),"C","N").
