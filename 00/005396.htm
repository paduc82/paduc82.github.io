Excel VBA to fill range by user form - Microsoft Community

ericleungkf say:

Excel VBA to fill range by user form

Hi, I am new to excel VBA, please can anyone advise if any codes can help to achieve the following task:

spreadsheet : column A: PO No., column B: item No., column C: received date

within one PO no., there will be many item no.s, is it possible to develop code allowing user by input form calling the relevant PO no. and then input the received date at one time so that all items within this PO no. can be filled in the input date?

Thank a lot!

jllatham say:

If you really want to go with a UserForm, just say so - the process would be much the same as in the code below.

But what I'm thinking is "why not do it directly on the worksheet while the user is using it" instead of having to enter a PO number, call up a UserForm, enter the PO number again along with a date.

 

What this does is watch for a change in just 1 cell in column C, and if that change is not a [Del] action, then it picks up the PO Number from column A and then finds all of those PO numbers on the sheet in column A and sets their date to the date just entered.

 

This is Worksheet event level code.  To put it into your workbook, open the workbook; select the worksheet that is where you'd be entering the dates and RIGHT-click on it's name tab and choose [View Code] from the list that pops up.  Copy the code below and paste it into the module presented to you.  Close the VB Editor and test by entering a date or two.

 

If you like it, be sure and save the workbook as type .xlsm (macro enabled).  You'd have to do that with a UserForm in it anyhow.

 

Here's the code:

 

Private Sub Worksheet_Change(ByVal Target As Range)
  'when date is entered into column C, all rows
  'with same PO Number that appears in column A
  'are filled with the same received date
  'If there is no PO number, nothing happens
  '
  'Nothing happens if you delete a received date
  '
  'You don't have to be at the first row with the
  'PO number to do this, you could even be at
  'the bottom of the list
  '
  'only works with 1-cell changes in column C
  'change constants as needed
  Const RcvDateCol = "C"
  Const PONumberCol = "A"
  Const FirstPORow = 2 ' assumes labels in row 1
  'end user adjustable settings
  Dim POList As Range
  Dim anyPO As Range
  Dim PO2Update As Variant ' not sure what form your POs take
  Dim offset2Date As Integer
 
  If Target.Column <> Range(RcvDateCol & 1).Column _
   Or IsEmpty(Target) _
   Or Target.Cells.Count > 1 Then
    Exit Sub
  End If
  If Range(PONumberCol & Rows.Count).End(xlUp).Row < FirstPORow Then
    Exit Sub ' no PO numbers entered!
  End If
  'if no PO number for row with date entered, do nothing.
  If Not IsEmpty(Range(PONumberCol & Target.Row)) Then
    PO2Update = Range(PONumberCol & Target.Row).Value
    Set POList = Range(PONumberCol & FirstPORow & ":" _
     & Range(PONumberCol & Rows.Count).End(xlUp).Address)
    offset2Date = Range(RcvDateCol & 1).Column - Range(PONumberCol & 1).Column
  'turn off event processing to keep from calling this routine over and over
  'with each change that it makes!
    Application.EnableEvents = False
    Application.ScreenUpdating = False ' performance improvement
    For Each anyPO In POList
      If anyPO = PO2Update Then
        anyPO.Offset(0, offset2Date) = Target.Value
      End If
    Next
    Set POList = Nothing
  End If
'*MUST* turn .EnableEvents back on, not so for .ScreenUpdating
  Application.EnableEvents = True
End Sub
