Seems like I found a solution, I'd name it the "technical trick". Not sure it is universal solution.

But I believe there will be more adequate one in future.

MS Guys please add this stuff into the Graph API :)) in /organization info for example.

So the trick:

1. You know the tenant Id and your application is already registered in this tenant (using admin consent flow)

2. Using Graph API you can retrieve SharePoint server URL and admin site URL.

  2.1 Use this endpoint to get information about particular drive root dir https://graph.microsoft.com/v1.0/drives/<driveId>/root

  (To get drives list use https://graph.microsoft.com/v1.0/drives)

  (To test API you can use this helpful link https://graph.microsoft.io/en-us/graph-explorer#)

   In "webUrl" property you can find "server related URL", so I suppose it is main SharePoint server for the tenant.

   For example it is name.sharepoint.com.

  2.2 Now assume that admin site URL always (by default) is name-admin.sharepoint.com

3. Using CSOM query site collections list using Tenant.GetSiteProperties()

  3.1 Get App Only token like this using your application certificate and (serverUrl = admin site URL)

private static string GetAppOnlyToken(string serverUrl)
{
    string AuthURL = @"https://login.windows.net/<TENANT ID>";
    
    //set the authentication context
    //you can do multi-tenant app-only, but you cannot use /common for authority…must get tenant ID
    AuthenticationContext authenticationContext = new AuthenticationContext(AuthURL, false);

    //read the certificate private key from the executing location
    //NOTE: This is a hack… Azure Key Vault is best approach
    var certfile = System.IO.File.OpenRead(PrivateKeyPath);
    var certificateBytes = new byte[certfile.Length];
    certfile.Read(certificateBytes, 0, (int)certfile.Length);
    var cert = new X509Certificate2(
        certificateBytes,
        PrivateKeyPassword,
        X509KeyStorageFlags.Exportable |
        X509KeyStorageFlags.MachineKeySet |
        X509KeyStorageFlags.PersistKeySet); //switchest are important to work in webjob
    ClientAssertionCertificate cac = new ClientAssertionCertificate(AppId, cert);

    //get the access token to SharePoint using the ClientAssertionCertificate
    Console.WriteLine("Getting app - only access token to SharePoint Online");
    var authenticationResult = authenticationContext.AcquireTokenAsync(serverUrl, cac).Result;
    return authenticationResult.AccessToken;
}

  3.2 Get client context

private static ClientContext GetClientContextWithAccessToken(string serverUrl, string accessToken)
{
    ClientContext clientContext = new ClientContext(serverUrl);

    clientContext.AuthenticationMode = ClientAuthenticationMode.Anonymous;
    clientContext.FormDigestHandlingEnabled = false;
    clientContext.ExecutingWebRequest +=
        delegate (object oSender, WebRequestEventArgs webRequestEventArgs)
        {
            webRequestEventArgs.WebRequestExecutor.RequestHeaders["Authorization"] =
                "Bearer " + accessToken;
        };

    return clientContext;
}

  3.3 And NOW!.. Get site collections list

private static void QueryRootSites()
{
    var token = GetAppOnlyToken(SpServerAdminUrl);
    using (var clientContext = GetClientContextWithAccessToken(SpServerAdminUrl, token))
    {
        var tenant = new Tenant(clientContext);
        SPOSitePropertiesEnumerable spp = tenant.GetSiteProperties(0, true);

        clientContext.Load(spp);
        clientContext.ExecuteQuery();

        foreach (SiteProperties sp in spp)
        {
            Console.WriteLine(sp.Url);
        }
    }
}
