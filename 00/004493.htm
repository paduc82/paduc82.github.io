ken sheridan say: You are quite right; you can simply 'open' the Site Prospects form filtered to the current tenant.  If it's already open it will be filtered to the tenant, if not it will be opened filtered.  The code for this would be:

Const conFORM = "Site Prospects"
Dim strCriteria As String

strCriteria = "Tenant = """ & Me.Tenant & """"

DoCmd.OpenForm conFORM, WhereCondition:=strCriteria
Forms("Site Prospects").Tenant.DefaultValue = """" & Me.Tenant & """"

This assumes that the Tenant column is of text data type.  If it's a number data type, which it may well be without you realizing it if you've used the 'lookup field' wizard, the code would need amending to:

strCriteria = "Tenant = " & Me.Tenant

Do not change the line which set's the DefaultValue property, however.  This property is always a string expression regardless of the data type, so should be wrapped in literal quotes characters in any case.

Also, in the Tenant Clients form's current event put the following:

Dim strCriteria As String

strCriteria = "Tenant = """ & Me.Tenant & """"

On Error Resume Next
Forms("Site Prospects").Filter = strCriteria
Forms("Site Prospects").Tenant.DefaultValue = """" & Me.Tenant & """"

again amending the code if the data type is numeric.  This will keep the Site Prospects form in sync with the Tenant Clients form if the former is open and you navigate to a different tenant record in the former.

bob jones aka: cybertaz say:

Re the Cmd+Left Click tip: It refers only to shifting the Sheet tabs left or right to expose the first group or last group of sheet tabs in the workbook. It does nothing unless the number of sheets (in combination with their width) exceeds the space the tab display area is adjusted to.

To display the Activate list of sheet names you shouldn't have to use Cmd+Click first... Just use Control+Click [aka, right-click) as long as you're not clicking within the boundary of the tip box when it's displayed.
