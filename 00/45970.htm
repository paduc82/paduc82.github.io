saltgrass say:

It can be a little confusing when discussions about "images" are encountered.  There are several types of images, but possibly the one you are referring to is a System Image Backup which is created from the File History dialog and the System Image Backup option on the lower left of the window.

If you made that image, files are created which can recreate the current install on your system.  So if you were running 8.1, it would be of that install.  Restoring that image would just put you back to when the image was created.  Restoring a System Image Backup will also wipe the drive, so if you have data partitions, which you did not include in the image, they will disappear after the Restore operation.

System Image Backups will normally only backup four partitions on an UEFI install.  The EFI system, Recovery, MSR, and OS partitions.  It will not include other partitions unless they have a drive letter and you choose to include them.  The larger Recovery (image) partition and any non-lettered partition will not be included.

The wbadmin.exe utility Jessen mentions can include non-lettered partitions, but you have to include them manually.  On my Dell system, there is a 40 MB OEM partition which cannot be included with either method, but I am still trying to figure that out.

rmdeq say:

Kudos to kwdavids! The thread:

Camera App crashes on Surface 3 Pro after Windows 10 upgrade

gets you started on fixing this problem.

I had to both remove bad entries from the Library (Go to Windows Explorer, View Options, Show Libraries, Pictures, and then delete any bad entries, ie entries that you get an error when you click on them).

However, this did not completely solve the problem. I started getting and "Oops something went wrong" error on startup, which was better than the white flash.

So then I had to go to:

C:\Users\ME\AppData\Roaming\Microsoft\Windows\Libraries\CameraRoll.library-ms

and select Properties. Both of the entries in there were invalid, so I deleted them an added an entry for C:\Users\ME\Pictures\Camera Roll

Now, the Camera app works for that user, and the problem is resolved.
