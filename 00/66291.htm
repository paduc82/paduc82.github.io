Trouble with Windows Fax and Scan after Anniversary Update - migration - Microsoft Community

mkeppler say:

Trouble with Windows Fax and Scan after Anniversary Update - migration of old faxes, assigning fax modems on account without admin rights

Windows 10 anniversary update caused a lot of trouble on the App "Windows Fax and Scan".

PLEASE FIX THESE FOR THE NEXT UPADATE - everybody will have these Problems!!!

When updating from Win 7 to Win 10 I had to solve two problems manually - exactly the two problems listed below. I somehow got around and was happy to have it running.

A while ago I received the anniversary update to latest Win 10. Surprisingly, it came as a complete new installation and not just as an update to be integrated into the existing Win 10.

As consequence, I was back in all the hassle with "Windows Fax and Scan". These are the two problems I faced, and they need to be solved in the code of Windows 10 for the next update, if you want to have customers loving Win 10...
 
1. "Windows Fax and Scan" uses a dedicated folder to save sent and incoming faxes. For which reason ever, fax files are only shown in the app, if the file name follows a dedicated rule. Currently, my system uses file names looking like this: "S-1-5-21-2479xxxxxx-8080xxxxx-3180xxxxx-1001$201d217138111f1.tif" The App only shows files starting with "S-1-5-21-2479xxxxxx-8080xxxxx-3180xxxxx-1001$201". Unfortunately the new installation of Win 10 anniversary edition caused this unique number to change, so all my old received and sent faxes were not shown in the new app, even though being available in the correct folder. The solution was to manually rename them from the old unique number to the new unique number.

Solution to solve the problem for the next Windows update: Please either keep the unique name the same or alternatively provide a migration from the old file names to the new ones next time you provide an update.

2. I use 2 accounts on Win 10, one is the administrator account and the other is the daily use account without admin rights. I ususally work on the account without admin rights. After the update to anniversary edition, the fax modem was not assigned to the new fax account any more. When trying to assign the fax, this did not work. There was a symbol indicating admin rights being required. When starting Windows Fax and Scan with admin rights, I could assign the fax modem properly. However, when trying to send a fax later I noticed, that the fax modem had been assigned to the fax account only when running Fax and Scan from the admin account. When running from daily use account, the modem was still not assigned. My solution was to temporarily give the daily account admin rights, then I could properly assign the modem. After installation I took admin rights away from this account again.

Solution to solve the problem for the next Windows update: Please allow to assign a fax modem to the fax account without requiring admin rights. If this is not possible, please make sure that the modem is assigned to the account Windows Fax and Scan are running from.
