I will try to explain how I understand it works and what your options are

Apps are designed to default to auto sign in when you sign into your computer so that you can use a single sign-in to authenticate.  If you log out, then this will disable this convenience in the Skype (UWP) App.    If you do not do this, then it will continue to log you in automatically.  If you do not want to be signed in by default on any of your Apps, you would have to create a local account on your computer instead of using your Microsoft account.  This will in effect, start you in offline mode as far as Apps are concerned.    This is how all my Windows 10 computers are configured as I do not want any App, not just Skype, to automatically sign in by default unless I explicitly keep them signed in.  This action still will not automatically log you off.  It will simply minimize the chances that you are signed in automatically onto any App, unless you explicitly choose to.

john mcghie say:

I don't think I am going to be much help: you need to figure out what changed.

"file type that is blocked" can mean that the file contents and the file extension and the file type code and creator code do not match.  I don't think either Word or OS X are using the type and creator codes any more, but if they are present and wrong they can cause issues.

"too many files" means Word can't get a file handle from the server.  The usual cause is that the user has not restarted the computer this year :-)

Autorecover:  The Word user needs Read/Write/Remove permissions not only to the directory but to the level above it as well.  This "could" be funny characters in the location pathname: http://word.mvps.org/Mac/cantsavefilenames.html

"cannot be accessed" means exactly what it says. Word could not get data from the server (local OS X file server, in this case).  It got a file handle, and an edit lock, but did not get any data.

I am afraid I have told you nothing you did not already know.  Sadly, these are all "Contact your System Administrator" type issues, which gets really annoying when you ARE the System Administrator.

I would be interested to see what happens if you unplug the network cable from one of those machines: I have a "niggle" that's whispering "I wonder if this is an authentication issue?"
