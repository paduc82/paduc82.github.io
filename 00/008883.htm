    diane poremsky mvp (slipstick.com) say:
    >> Am I being stupid, but wouldn't that remove all emails sent & received by Account B, past, present and future?

    Removed from where? It just delivers all Account B to the same data file that account A uses.

    By default, when Outlook sets up a new account, it puts each account into a separate pst file. It sounded like you are using rules to move mail into a single pst, so you have one Inbox shared by both accounts. If so, changing the delivery folder is a better way to do it. Sent mail is put in the desired sent folder automatically.

    Outlook is (generally) smart enough to know which account downloaded the message and will reply using that account. New messages will always use the default account, unless you change the From account before sending.

    If Outlook can't find the account that downloaded the message, it will reply with the default account. This would not be the cause of replies from A being sent by B though, since you said A is the default account. Unless... A & B shared the same mailbox on the server and have the same username/password - then Outlook sees it as one account and will download all mail by whichever account checks the mail. In this case, if B downloaded the mail, then B will send the reply.

    You can add the Email Account field to the view to see which account downloaded the message. Screenshot: https://www.screencast.com/t/YlWatDMZ

    To add the Email Account, slide the divider between the message list and reading pane to the right until you drop to the line view. Right click on the row of field names, choose field chooser. In All mail fields, find the Email account field and drag it to the row of field names. You'll see the account name that downloaded the messages. That should be the account outlook uses to reply. When you are finished, slide the divider to the left to go back to the compact view.

    Try something like:

    • Alt+F11 to open the VBA editor
    • Alt+IM to insert a new code module
    • In the new module, paste the following code 
    '=========>>
    Option Explicit

    '--------->>
    Public Sub ToggleHideRows()
        Dim WB As Workbook
        Dim SH As Worksheet
        Dim Rng As Range
        Dim rCell As Range
        Dim hideRange As Range
        Dim iLastRow As Long
        Dim Res As Variant
        Dim Btn As Button

        Set SH = ActiveSheet
        With SH
            Set Btn = .Buttons(Application.Caller)
            iLastRow = LastRow(SH, .Columns("A:A"))
            Set Rng = .Range("A2:A" & iLastRow)
        End With

        On Error GoTo XIT
        Application.ScreenUpdating = False
        With Btn
            If .Caption = "Unhide Rows" Then
                Rng.EntireRow.Hidden = False
                .Caption = "Hide Rows"
                Exit Sub
            End If
        End With

        For Each rCell In Rng.Cells
            With rCell
                If .Value = 0 _
                   And .Offset(0, 1) = 0 _
                   And .Offset(0, 2) = 0 Then
                    If hideRange Is Nothing Then
                        Set hideRange = rCell
                    Else
                        Set hideRange = Union(rCell, hideRange)
                    End If
                End If
            End With
        Next rCell

        If Not hideRange Is Nothing Then
            hideRange.EntireRow.Hidden = True
            Btn.Caption = "Unhide Rows"
        End If

    XIT:
        Application.ScreenUpdating = True
    End Sub

    '--------->>
    Public Function LastRow(SH As Worksheet, _
                            Optional Rng As Range)
        If Rng Is Nothing Then
            Set Rng = SH.Cells
        End If

        On Error Resume Next
        LastRow = Rng.Find(What:="*", _
                           after:=Rng.Cells(1), _
                           Lookat:=xlPart, _
                           LookIn:=xlFormulas, _
                           SearchOrder:=xlByRows, _
                           SearchDirection:=xlPrevious, _
                           MatchCase:=False).Row
        On Error GoTo 0
    End Function
    '<<=========

    • Save the file as a macro-enabled workbook with an xlsm extension
    • Select the Developer tab on the ribbon
    • Select Insert
    • Add a button from the forms controls onto the sheet

    • Right-click the button
    • Select the option Assign Macro and select  ToggleHideRows

    The first time that you click the button, any rows with zero values in columns A,B and C will be hidden and the button caption will change to Unhide Rows  , Thereafter the captiion will change at each click from Unhide Rows to Hide Rows and the functionality of the code eill change accordingly.
