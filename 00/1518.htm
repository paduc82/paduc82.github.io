SOLVED: Outlook 2016 "Failed to register a VB Script dll. - Microsoft Community

jclamta say:

SOLVED: Outlook 2016 "Failed to register a VB Script dll. Reinstall or run Regsvr32.exe Vbscript.dll to self register."

After installing Windows 10 enterprise, 64-bit, and updating via windows updates to 1089, I proceeded to install office 2016 via office 365 download.  All programs (Excel, Word, etc) seemed to work EXCEPT for Outlook.  The first attempt at starting Outlook 2016 produced the error "Outlook cannot start" or something similar, the second and subsequent attempts produced the error "Failed to register a VB Script dll. Reinstall or run Regsvr32.exe Vbscript.dll to self register."  I tried all of the following, but to no avail:

-didn't work: as per the error suggestion, opened a Command Prompt as Administrator, and successfully register the dll

-didn't work: as per other threads, delete/move the .CFG files under "C:\Program Files\Microsoft Office\root\Office16\FORMS\1033", which did allow Outlook to start up, but then when attempting to create an email, or do any action in Outlook, the error of not loading the action, was created

 SOLUTION:

Summary: 1) uninstall Office 2016 with the original user "bob-smith", 2) Create a new local user on the machine (using ONLY regular characters, western a to z, eg "bobsmith") 3) add my new user to Administrator group, 4) login with that new user "bobsmith", 5) install fresh copy of office 2016, 6) everything is working fine

If you wish, under Accounts, assign regular Microsoft or work Account

DETAILS:

My user name under "C:\Users\" contained a hyphen, eg. " C:\Users\bob-smith", given that I setup Windows using the AzureAD account, and it included a hyphen "-" automatically, and other threads indicated that special characters may cause this error in opening Outlook 2016, as there is a local FORMS folder for each user under: "C:\Users\(user-name)\AppData\Local\Microsoft\FORMS", where a local Forms Cache file is managed “FRMDATA64.DAT”

This file didn’t seem to be created or managed properly by Outlook 2016 when using my “user-name” (bob-smith) account

When installing and using Office 2016 with my “username”, eg "bobsmith", local account, there was no issue, and the FRMDATA64.DAT file was created and managed correctly.  No need to regsvr32.exe vbscript.dll or delete .CFG files, etc.
