karl timmermans [outlook mvp 2012-2018] say:

Questions

1.  The View by Company/Category is grayed out. 

2.  Is all Outlook data (mail, contact, calendar, notes,) stored in the .ost file?  I don't see a .pst file.

3. What is the best way to back-up all Outlook data?

4.  Does (This Computer Only) mean that this data is not stored in the Cloud?

(not in order)

#1 - Don't use IMAP accounts with a single OST file for reasons mentioned below but it's the only contact folder that is exhibiting the behaviour so suspect that this is specific to the Contact Folder "This Computer Only" related to the IMAP account. Will gladly stand corrected if someone else has more accurate info in that regard

#2 - If only one account is configured in Outlook '2013 OR the IMAP account is the first or multiple accounts configured, then everything resides in the .OST file.

#4 - Correct - any folder with the description "This Computer Only" means just that - resides locally only.

As a prelude to the answer for item #3 - an OST file can only be opened in the Outlook profile by the email account that originally created it. It gets recreated any time the IMAP account is configured OR if the >OST file is found to be missing. The problem with this approach in Outlook '2013 is that if the email account gets deleted from the profile, all data in the folders marked "This Computer Only" is lost

There are two ways you can backup your data in folders marked "This Computer Only" - the following is based on there only being IMAP account configured in the current Outlook profile

Approach #1

a) - create a new PST file

b) - copy the contents of any desired folders to equivalent folders in this PST file

c) - copy the PST file to another drive which provides a backup

d) - remember to repeat steps #1 thru #3 on an on-going basis

(have over-simplified the instructions since strongly suggest you adopt Approach #2

Approach #2 (recommended)

a) create a new PST file (there are several ways to do this but for this exercise - this is the quickest)

  - File --> Account Settings --> Data File tab --> Add

  - While there - make this new data file the default for the profile

  - This will result in a series of default folders being added to the newly created PST file which
     will include Contacts/Calendar etc (in addition to another <Inbox> which won't be used)

  - close Account Settings

b) Copy all data from the desired "This Computer Only" folders to the equivalent ones in the newly created PST file. This set of folders are not the default folders in terms of contacts/calendar which is where all new contact/calendar items will be created. You can't delete the "This Computer Only" folders from within the Outlook UI.

c) All other email items for the IMAP account is stored on the server. If you rename/delete the OST file associated with this account, it will get recreated without the "This Computer Only" folders included in the IMAP folder tree.

d) From that point on - your contact/calendar info will always be accessible regardless of whether or not you ever delete the Imap email account from the profile. Just need to copy the PST file to an external drive (or wherever) to have a "backup" of your contact/calendar data.

*** Outlook should always be closed whenever you copy a PST file to another location if it's being used in the current Outlook profile to safeguard against problems (i.e. corruption) occurring.

Speaking w/ my Linux/Embedded guru (and Kernel maintainer) hat on:

As far as I know, SkypeKit is dead for many years now. Even if you still found an old version and got it running somehow, it's unlikely that it still working, as MS is changing the server side (protocols) w/ virtually every moon phase. (maybe it's even the intention behind to lock out old client)

Older skype had an desktop bus API for remote controlling (that had also been the way how we integrated it into generic chat clients like pidgin), but I doubt that's still supported by Skype8, which is a complete rewrite.

Theoretically, we could try running it in a container, against an virtual framebuffer X server and write some tool that basically emulates a human user. (think of the old funny "Windows recorder", just more complex ;-)).

As the Skype8 GUI is basically running via an built-in chrome, there's another option: try to inject an own chrome extension that handles the remote controlling (in that case we'd have direct access to DOM trees, catch and send DOM events, etc). Similar approach could be done w/ the Skype android app (running in an customized ART environment).

Of course, I can't give any guarantees that this actually works, and maintenance of that thing will be pretty complex and costly. Embedded usually means hard requirements on stability and long term (20 years) maintenance - such an solution doesn't really fit in there.

Still won't activate?

This issue might be caused as results of incorrect product keys or previous entered credentials to activate Office in the past.

If the incorrect information was ever entered in the email address field, you may need to clear out the Mac's keychain since it saved the incorrect information.

To clear the keychain:

1. Close all Microsoft Office applications (Outlook, Word, Excel, PowerPoint, etc.)
2. Open Applications folder in Finder
3. Open Utilities folder
4. Open Keychain Access
5. Select the Keychain Access Search bar in the top right corner
6. Search and delete all references to the following items:
Exchange
Office
ADAL
7. Close Keychain Access


At this point, go ahead and enter the product key information you have with you and see the results.
