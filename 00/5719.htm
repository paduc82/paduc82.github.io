joeu2004 say: Michael wrote:

First I generated a chart ( a scatter plot with a trend line that goes through), from a set of data.
Now that I have this chart, can I enter a value for x (known) and based on the graph find the value of y?

The graph is semi-log: logarithmic in y and linear in x.


Yes.  But the precise formula will depend on the trendline that you choose.  Then there are two approaches:  the quick-and-dirty, and the precise.

Since you are using a log y-axis, I presume that you expect an exponential trendline.  When you select the trendline, also select the option "display equation".  For an exponential trendline, you might see a formula of the form y = 0.1203e^0.7033x, where I am using "^" to indicate superscript.


=0.1203*exp(0.7033*A1)

where A1 represents the value of x for which you want to find the value of y.

Usually, it is better to display the trendline constants with greater precision.  Right-click on the trendline textbox, click Format Data Labels and the Number tab, and select a numeric format and number of decimal places.  Generally, Scientific with 14 decimal places is always the most precise; but most people are more comfortable with the Number format.  You can copy-and-paste the constants from the trendline formula textbox into the Excel formula.

You should be able to apply that method to any trendline formula that you see, changing the Excel formula accordingly.

this web page (link).

For example, for the exponential trendline example above, if your orginal data are in X1:X12 (x values) and Y1:Y12 (y values), you can determine the 0.1203 coefficient with the expression

EXP(INDEX(LINEST(LN(Y1:Y12),X1:X12),1,2))

and the 0.7033 coefficient with the expression

INDEX(LINEST(LN(Y1:Y12),X1:X12),1)

Although you could write the following formula:

=EXP(INDEX(LINEST(LN($Y$1:$Y$12),$X$1:$X$12),1,2))
*exp(INDEX(LINEST(LN($Y$1:$Y$12),$X$1:$X$12),1)*A1)

it would be much more efficient to compute each coefficient one time in separate cells (e.g. C1 and B1), and use the following formula:

=$C$1*exp($B$1*A1)

That's not a V its more likely the two characters written and placed together (\ and /)  in other words you might have files or folders with the \ or / character in the name, (Forward or Back Slash).

This s fairly new to Mac  Up until about system OS X.6 the Mac system ignored  such character and a Mac user could name Files, Folders, Directories anything they wanted could user any character they desired. Many would rename Directory and directories with either the forward or back slash, as an additional way of organizing files. Then all of a sudden Apple changed the system so that characters should not have been used to start with; but, were ignored are no longer ignored.

  1. Numbers before names, Superscript/Subscripts.
  2. These Characters:+  *  "   '   \  |  /  <  >  <<  >> {  }  {{ }}  [  ]  [[ ]]

    a) Hard Drive.
    b) Directory (Folder) Names.
    c) Any File Names.
    d) Any Device Names.
    E) And on servers
  5. For Servers: no names can have the name File or Fileserver  in any path names. 
  6. You can use _ (underscores).

herb tyson say:

Cells can be referred to in field formulas using column (letters) and row (numbers). For example, the upper left cell in a table is A1. To its right is B1, and below it is A2. They can be used in formulas. For example, the field {=C4*D4} might be used in row 4 to multiply whatever is in the 3rd column (such as price) by whatever is in the 4th column (such as quantity).

I believe that for non-numerical table entries, you would need to rely on bookmarks and references to them. FWIW, Word tables are intractable enough with respect to references that I tend to embed Excel tables into Word documents, and use Excel's much better facilities instead.

For working with table cell data, you might find Greg Maxie's macro to be useful:

http://gregmaxey.mvps.org/Table_Cell_Data.htm

pamcaswell say:

Hi, Herb,

508 compatibility (It's a short name for the Americans with Disabilities Act of 1998)  is something a little different.  Online documents, screens, etc. must be marked so assistive devices (like a screen reader) can present information to the person with a disability (such as blindness). A screen reader has to be able to identify the row and columns headers of a table and to identify them for a particular cell, because it is going to read the information aloud to the person in a structured way.  If a person moves the cursor about in the table the screen reader should be able to tell the person what's in the cell and what headings apply, for example.

W2007 and 2010 tables have the same basic structure as XML and HTML tables, which, properly coded, can be read by these devices.

Pam

who spent much of her final year working for the US gov, updating several hundred web pages to html4 and 508 compliance.
