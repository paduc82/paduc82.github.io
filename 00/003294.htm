The DRIVER_IRQL_NOT_LESS_OR_EQUAL indicates that a kernel-mode driver attempted to access pageable memory at a process IRQL that was too high.

This occurs when a driver tries to access an address that is pageable (or that is completely invalid) while the IRQL was too high. This bug check is usually caused by drivers that have used improper addresses or a driver that was trying to execute code when the code itself was paged out. Possible causes for the page fault include the following:

  • The function was marked as pageable and was running at an elevated IRQL (which includes obtaining a lock).

  • The function call was made to a function in another driver, and that driver was unloaded.

  • The function was called by using a function pointer that was an invalid pointer.

However, please do take note that Group Policy Editor is only available on Windows 10 Pro, Enterprise, and Education editions. Here are the steps:

  1. Open the Start menu, type gpedit and press Enter to open the Local Group Policy Editor.
  2. Navigate to Computer Configuration/Administrative Templates/Windows Components/Edge UI.
  3. On the right pane, double-click/tap on the Allow edge swipe policy to edit it.
  4. Select Enabled and press OK. Refer to the screenshot below.
  5. Restart your computer.

Alternatively, you also enable/disable the edge swipe function through the Registry Editor. Follow these steps:

  1. Click the Start menu button, type regedit and press Enter to open Registry Editor.
  2. Navigate to HKEY_LOCAL_MACHINE\SOFTWARE\Policies\Microsoft\Windows\EdgeUI.
  3. On the right side, create a new DWORD (32-bit) Value and name it AllowEdgeSwipe.
  4. Double-click on the newly created value and set the Value data to 0 (zero), then press OK.
  5. Restart your computer.

greginmich say:

I have another AV app installed (Norton, McAfee, Avast, etc.) so why are Windows Defender definition updates downloading and then failing to install?

Weve already identified the most likely cause of this issue Windows Defender quite often doesnt get properly turned off in its registry key when a third-party AV app is installed.  

But Error 0x0070643 is a generic error code, so how can I be sure that this registry issue is really the problem in my case?

The formal diagnostic procedure for this issue is to run this command at the command prompt (copy, paste, and enter this line at the command prompt):

reg query "hklm\software\microsoft\windows defender" /v DisableAntispyware

If the return value is 0x0, then Windows defender hasnt been properly disabled in its registry key.

But Ive been warned to stay away from the registry; and I have a severe case of Command Line Phobia isnt there some other way to see if this is actually my problem?

Of course there is. The informal diagnostic procedure is to simply type Defender in the Cortana search bar and then hit the enter key once the Windows Defender entry is selected. If you see this screen, then Windows Defender hasnt been turned off in its registry key. Its still turned on; but the service cant be started so Windows Defender is in a state where it cant install the definition update.

OK, it looks like thats my problem; but do I really need to fix it?

No, you dont there havent been any serious consequences resulting from this issue. Other updates seem to be processed without any trouble, and the associated Error 577 (on attempting to start the Windows Defender Service in services.msc) is a false alarm. Theres really nothing wrong with the Windows Defender Service executable (MsMpEng.exe).

This stuck update really bothers me; so how can I fix this problem?

Repeating the installation of the third-party AV app after uninstalling it and running its clean-up utility should manage to switch Windows Defender off in its registry key. However, this has been reported to fail repeatedly with Norton products and redoing the installation might be considered too time-consuming in other cases so we have a shortcut that uses Group Policy to completely turn off Windows Defender. This is done with the Group Policy Editor in the Professional edition of Windows 10, and with the Ten Forums Turn_Off_Windows_Defender.reg file in the Home edition. Everything you need is right here:

http://www.tenforums.com/tutorials/5918-windows-defender-turn-off-windows-10-a.html

We also have a prospective solution that appears to directly update the relevant registry entries in the Windows Defender key when the third-party AV app is disabled; Windows Defender is started; and the third-party app is then turned back on while Defender is still turned on. This seems to work because it mimics the state changes that occur in a reinstallation:

http://answers.microsoft.com/en-us/windows/forum/windows_10-update/definition-update-for-windows-defender-kb2267602/d4edd314-a438-4ac2-b7a9-dc1c45a1ef62

Important Notice & Disclaimer:

Forum moderator Stephen Boots recently published this Wiki article on the subject of Limited Periodic Scanning:

http://answers.microsoft.com/en-us/protect/wiki/protect_defender-protect_scanning/limited-periodic-scanning-in-windows-10-to-provide/3dcfc8dd-36b4-43f2-9a0a-732e43191cf3

So we see that Microsoft has opted for a new feature that could either add chaos to confusion for this definition update issue, or potentially resolve it but which in either case will probably break our current ideas about the proper operational state for Windows Defender when a third-party AV app is installed. But on the bright side, its good to hear from Steve.

    Dim DiagramServices As Integer

    ActiveDocument.DiagramServicesEnabled = visServiceVersion140 + visServiceVersion150

    UndoScopeID1 = Application.BeginUndoScope("Size & Position 2-D")
