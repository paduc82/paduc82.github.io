I found the answer here, and it worked great for me.

 

http://www.ehow.com/how_5473226_make-instead-windows-desktop-search.html

I did not have to uninstall Windows Desktop Search.  For the change to take effect, you might have to reboot your PC.

 

The instructions at the above URL are as follows:

 

1.      Log in to Windows as a Power User or Administrator - you'll need elevated priveleges to access Registry Editor.

 

2.      Open My Computer or Windows Explorer. Expand your C: drive. Right-click the Program Files folder and select "Search".

 

3.      Windows Desktop Search (WDS) will open. At the bottom left of the screen you'll see "This folder is not indexed". Click the "...please use Search Companion" link to open Search Companion. This step ensures that the needed Registry value is created.

 

4.      Close the Search Companion and WDS windows.

 

5.      Click the Windows Start button, then "Run" (or press the WINDOWS + R keys on your keyboard.) In the Open dialog, type REGEDIT then click OK. This will open Registry Editor.

 

6.      To change the setting for all users of the PC, browse to HKEY_LOCAL_MACHINE\Software\Microsoft\Windows Desktop Search\DS.
To change the setting for the current User account only, browse to HKEY_CURRENT_USER\Software\Microsoft\Windows Desktop Search\DS.

 

7.      In the right window pane, double-click the "ShowStartSearchBand" item. In the Value Data field, change the "1" to "0". Click OK, then close Registry Editor.

 

8.      Now, when you right-click a folder in Explorer and select "Search", the Search Companion window will open instead of WDS.

sheng jiang ËíãÊôˇ say:
Visual C++ redistributables are components used by programs (games, Microsoft software etc) developed using Visual C++. Most programs bind to the most recent update of the same major version that they use in development, but some programs bind to one particular version Visual C++ redistributable and don't accept any other version. Personally I released one program like this because I found a bug in the latest update and have no choice but to bind to the Visual C++ redistributable that I developed against. 

A Microsoft Visual C++ 2005 redistributable cannot be replaced by a Microsoft Visual C++ 2008 one. The binding would fail and any program that depends on the Microsoft Visual C++ 2005 redistributable will report "This application has failed to start because the application configuration is incorrect. Reinstalling the application may fix this problem." on startup. The only way to change a binding from Visual C++ 2005 to Visual C++ 2008 in a program is to have the program's author to upgrade the source code to a new Visual C++ version, test, and release an updated version of the program, then you upgrade to that version of the program.

Unless you have a few hours to go through all of your programs to see which program depends on which version of Visual C++ redistributable, I suggest to keep all of them.
