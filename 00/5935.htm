I can't answer your question about saving to the Desktop (which I don't recommend), but you can remove your name from the comments as follows:

First you need to understand the concept of Network, Internal Network, External Network, and External Groups.

Each company that uses Yammer is allowed ONE internal network (for internal staff interactions etc), and unlimited number of external networks (where outsiders can join).

The internal network is meant to be for employees only. For some particular discussions that may involve an outsider, an external group, within the internal network, is created.

For interaction with external communities, a company may elect to create an External Network tied to their Yammer Internal Network, for example the Microsoft 365 Partner Community is such a network that Microsoft uses to network with its partners. It is an External Network belonging to Microsoft's Yammer Network. The nature of a External Network is both Microsoft staff and Outsiders (partners) can join the network and participate in discussions inside the network's groups.

The question I have to ask you is, are you trying to create an External Group out of Microsoft's External Network ?

If that is what you are trying to do, it is not necessary as it is already an External Network that allows outsider access, so technically since the entire network allows outsiders, then every group is also an external group (allows outsiders).

However there are some other possible scenarios 

1. You are trying to create an external group from within Microsoft's Internal network (as you are given access to a External Group). This will not be possible as you are not a Microsoft Employee. You would need a sponsor to create that network for you.

2. You are trying to create an external group from your companies (Ally Inc) internal network. You then have to check with your Office365 administrator to see if you have been given rights to do so.

Hi Alan,

Thanks for your updates and I'm glad to hear that create a rule can help you resolve this issue. 

About your question to create a rule apply to Sent Items folder, in Outlook client, simply do following steps:

1. In Outlook client, click on the Sent items folder and go to Rules > Manage Rules & Alerts > New Rule > under Start from a blank rule, select the option "Apply rule on message I send" > Next.

2. In the next step, set or don't set conditions according to your requirement and click Next.

3. In the next step, select the option "Move a copy to the specific folder" and set the folder. Note: this option will not move and delete items in the Sent items folder but you can delete messages in the Sent items folder when you finish to create and run this rule for the Sent Items folder.

4. In the Finish rule setup window, don't forget to select the option "Run this rule now on messages already in Sent Items, click Finish.
