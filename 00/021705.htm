Word for Office 365 Word 2019 Word 2016 Word 2013 Word 2010

When you're using a version of Word that's newer than the version used to create your document, you typically see Compatibility Mode in the title bar. Compatibility Mode temporarily disables new or enhanced Word features so that the document can still be edited by people using the earlier version of Word. Compatibility Mode also preserves the layout of the document.

Converting the document to the newer format lets you use newer features, but conversion might create difficulties for people who are using earlier versions of Word to edit the document. steve rindsberg say:

A bit of background that may save you some time in diagnosing the problem:

It won't matter which version of PowerPoint you save a PPSX file from, and in fact there's really not much difference between a PPSX and PPTX other than the extension itself and possibly a few internal bits that indicate which type of file it is.

The extensions are registered with Windows and instruct it what to do with each type of file when you doubleclick it or what options to show when you right click the file. For example, change the extension on a PPTX file to PPSX and when you right click the first option will be Show rather than Open as it would have been before the rename.

At a guess, when you open the link to a share drive, it may be MSIE or one of its components that decides what to do with the file based on the extension.

You mentioned:

"When I save the file to a share drive and provide a link to that file, it opens as an editable PowerPoint presentation - it does not automatically start, it does not open in full screen."

Can you give us more specifics? 

A share drive where?  On the local network, in SharePoint, on a web server or some kind of cloud storage (OneDrive, etc)?

What's the link path look like when you edit the link?

How exactly did you create the link?
