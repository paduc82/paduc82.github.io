Using a macro to ensure proper hyphenation - Microsoft Community

retdlj say:

Using a macro to ensure proper hyphenation

Editor here. I've never used macros before, but I would like to be able to hit a button and automatically replace improperly hyphenated words with properly hyphenated words. I can provide a list of both, so this function would basically find and replace based on two columns (commonly used improperly and properly hyphenated words, respectively). For example, non-private turns into nonprivate.

Any help would be much appreciated, thanks!

john korchok say:

To create a macro in Word, you'll first have to make the Developer tab visible:

  1. In Word, choose Word>Preferences>Ribbon.
  2. In the Customize section, scroll down to the Developer choice and check it. OK out.
  3. Click on the Developer toolbar, then on Macros.
  4. In the Macro name field, type ReplaceFromTableList, then click on Create. The VBA editor opens with the outline of a macro.
  5. Edit the macro to look like this:
  Dim oReplacements As Document, oDoc As Document
  Dim oTable As Table
  Dim oRng As Range
  Dim rFindText As Range, rReplacement As Range
  Dim i As Long
  Dim sFname As String

  'Change the path in the line below to reflect the name and path of the table document
  sFname = "Mac HD:Users:YourUserName:Documents:WordReplacements.docx"

  'Assign the document to be spellchecked to a variable
  Set oDoc = ActiveDocument

  'Assign the document containing the table of hypenation replacements to a variable and open it
  Set oReplacements = Documents.Open(FileName:=sFname)

  'Assign the table of replacements to a variable
  Set oTable = oReplacements.Tables(1)

  'Set up a loop
  For i = 1 To oTable.Rows.Count

    'Assign the contents of the document to be spellchecked to a variable
    Set oRng = oDoc.Range

    'Assign the contents of the first cell in a row (the incorrect hyphenation) to a variable
    Set rFindText = oTable.Cell(i, 1).Range

    'Shrink the range to eliminate the cell end marker from the text
    rFindText.End = rFindText.End - 1

    'Assign the contents of the second cell in the row (the correct hyphenation) to a variable
    Set rReplacement = oTable.Cell(i, 2).Range

    'Shrink to eliminate cell end
    rReplacement.End = rReplacement.End - 1

    'Set up find and replace for this word pair
    With oRng.Find
      .ClearFormatting
      .Replacement.ClearFormatting
      Do While .Execute(findText:=rFindText, _
        MatchWholeWord:=True, _
        MatchWildcards:=False, _
        Forward:=True, _
        Wrap:=wdFindContinue) = True
        oRng.Text = rReplacement
      Loop
    End With

  'Next iteration of the loop
  Next i

  'Close the document of word replacements
  oReplacements.Close wdDoNotSaveChanges
End Sub

The document WordReplacements.docx contains one 2-column table with incorrect hyphenation in column 1 and correct hyphenation in column 2. It is placed inside the Documents folder. You can move it elsewhere, just edit the path in the code to reflect that.

Edit the path, replacing Mac HD with the name of your hard disk, and YourUserName with your user name.

Open a test document with incorrect hyphenation and test the macro by running it from the Developer>Macros dialog. When it's working correctly, you can assign a keyboard shortcut:

  1. Click on Tools>Customize keyboard.
  2. In Categories, scroll down to Macros and select it.
  3. In Commands, select your macro.
  4. Click in Press new keyboard command and press the combination you want to use. (Control + H is available).
  5. Click on the Assign button (very important, I often forget)!
  6. OK out.

Thanks to Graham Mayor for the original Windows macro that I modified for Mac.

Sub Delay(Length As Long)
  Dim ThisTime As Long, EndTime As Long
  ThisTime = Now
  EndTime = Now + Length
  Do Until ThisTime = EndTime
    DoEvents
    ThisTime = Now
  Loop

I don't think that approach will work in Access 2010 -- it doesn't work in
Access 2003, I know, but I can't test Access 2010 at the moment to see if
they have added those custom formatting options to date field.  In Access
2003, the custom formatting options for Null and negative values are not
available for date field.
 
If it *would* work for Access 2010, I would expect something like this to be
the way to specify it in the Format property of the text box:
    dd-mm-yyyy;[Red]"(required)"
 
Please try that to see if maybe it will work.
 
If that *doesn't* work, here's an alternative. Create another text box the
same size and shape of the original text box, and place it right on top of
the original text box.  Give the new text box the following properties:
    Name: txtDateRequired
   Back Style: Transparent
   Fore Color: 255
   Border Style: Transparent
   Control Source: =IIf([YourDateControl] Is Null,"(required)",Null)
   On Got Focus: [Event Procedure]
   Tab Stop: No
 
Now create a very simple event procedure for the GotFocus event of
txtDateRequired:
    Private Sub txtDateRequired_GotFocus()
        Me.YourDateControl.SetFocus
    End Sub
 In the above properties and event procedure, replace "YourDateControl" with
the name of the text box that is bound to your date field.
 
Now, when your date control is Null, txtDateRequired will show "(required)"
