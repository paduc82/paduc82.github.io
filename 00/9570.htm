scottgem say:

Actually, the form is based directly upon the tables.  With the form based upon data from multiple tables, it is my understanding that it will not be updatable if the data is based upon a query.

As others have noted your understanding is incorrect. When you try to base a form on multiple tables, Access will generate a SQL statement joining the tables as the Recordsource.

The way around this is to use subforms. What you didn't tell us is whether the relationship between the main table and the other tables is 1:1 or 1:many. A 1:1 relationship is usually rare, so if it is 1:1 it would help to explain what the data is. As John said, basing database design on a paper form is probably not going to go well. 

The solution here is to use subforms. If the relationships are 1:many then you can use continuous form subforms to enter the multiple records. If 1:1 you can still use subforms, but you can remove the borders and make the subforms look like the main form.
