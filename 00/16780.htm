Document protection and rich text controls in Word 2007 - Microsoft Community

ryanl83 say:

Document protection and rich text controls in Word 2007

I have a form that I am putting together that will feature a couple of rich text controls. We would like to use document protection to restrict editing access to other areas of the form. The problem is that when we turn on document protection, we lose the ability to format text inside of the rich text field. Is there a way to enable that behavior so that we can keep the document protection and still allow the user to apply formatting within the rich text control?
jay freedman say:

Document protection "for forms" is intended for use with legacy form fields. It doesn't play nicely with several of the newer content controls, especially the Rich Text kind. There are two alternative methods:

  • In the Restrict Formatting and Editing pane, instead of choosing "Filling in forms" in the dropdown, choose "No changes (Read only)". Then select one or more content controls (you can hold down the Ctrl key while selecting more after the first one), and check the box next to "Everyone" in the pane. Or you can select one control and check Everyone, select another and check Everyone, etc. When the controls have all been marked, start protection.
  • Alternatively, select the entire document; then on the Developer tab, click the Group button and choose the Group command. That adds an invisible "Group" content control around everything.

In both cases, everything outside the content controls will refuse attempts to edit or format. The differences, which may determine which method you prefer, are:

  • The "No changes" protection lets you set a password to prevent users from turning off protection. The drawback is that by default the Restrict Formatting pane appears next to the document and the unprotected parts are highlighted with an obnoxious mustard color. Users can turn off the highlighting and close the pane, but that's a pain.
  • The "Group" protection can be turned off simply by clicking Group > Ungroup. There is no provision for locking it down.

john korchok say:

Yes, this is possible by following these steps:

  1. With your document open in Word, select a Rich Text Content Control.
  2. Click on the Developer tab and choose Protect Document. The Restrict formatting and Editing task pane opens at the side of your screen.
  3. Under Editing restrictions, check Allow only this type of editing in the document. Leave the drop-down at No change (Read only).
  4. Under Exceptions, check Everyone.
  5. Click on Yes, Start Enforcing Protection. I do not recommend entering a password. All parts of the document are protected from editing except the field you had selected.

To add more editable fields, click on the Stop Protection button. Select another Rich Text Content Control and repeat the steps. Word remembers which Content Controls were editable before and adds the new one to the list.

Baseline Execution Index (BEI) Formula for Microsoft Project - Microsoft Community

joshr3 say:

Baseline Execution Index (BEI) Formula for Microsoft Project Pro 2016?

I am trying to find an existing formula (or complete my formula below) to calculate BEI and also eliminate Milestones and LOE activities from the result. Currently 0-day milestones and LOE are included in the formula and they shouldn't be by DCMA standards.

Below is the formula I am currently using that is inherently including those 0-day milestones and LOE activities.

Planned Finish (Field: Number 1)

IIf([Baseline Finish]<[Status Date],1,0)

Actual Finish (Field: Number 2)

IIf([Status]="0",1,0)

BEI Calculation (Field: Number 3)

[Number2]/[Number1]

What can I do to eliminate Milestones and LOE activities?

tom_bpc say:

JoshR3,

John's got it right.  Project 2016 has no designated "LOE" field as far as I can tell, so use of an LOE (or hammock) flag is about the best approach.

A couple minor technical notes:

  1. According to the last published DCMA guidance I was familiar with (~2009), any tasks that have no baseline dates must be added to the denominator since they are already implicitly included in the numerator, i.e. your first condition becomes  ([Baseline Finish]<[Status Date] Or [Baseline Finish]=ProjDateValue("NA")).
  2. Be careful about presumed equivalence between "Tasks with a baseline duration of zero" and "Milestones." Zero-baseline-duration is the primary condition being referenced, and a "Milestone" in Project need not comply with that condition. 

Based on that, this is what I've got:

 BEI Denominator:  

IIf([Baseline Duration]>0 And Not [Flag20] And ([Baseline Finish]<[Status Date] Or [Baseline Finish]=ProjDateValue("NA")),1,0)

BEI Numerator:

IIf([Baseline Duration]>0 And Not [Flag20] And [Actual Finish]<>ProjDateValue("NA"),1,0)

This presumes Flag20 is your LOE marker, that you are working in an English-language version, and that you are using sum-rollups for your summary tasks.

(Like John, I usually prefer nested IIf's, but I find these easier to read.)  Good luck, tom

karl timmermans (outlook mvp 2012-2018) say:
>>
The outlook 365 account is IMAP. Are you saying I should use the Trash folder and delete the Deleted Items folder.?

You are not going to be able to delete <either> of those folders. The Trash folder belongs to the IMAP server folder tree whereas the Deleted Items folder is the standard Outlook folder.

The <Trash> folder should be showing up under your <Inbox> and is one that is located on the server (mirrored on the desktop) whereas the <Deleted Items> folder would be local only. FWIW, have checked multiple profiles where I have IMAP accounts configured and none of the IMAP account folder trees have both a <Trash> and <Deleted Items> folder in the same folder tree.

Typically items deleted from an IMAP folder located on the server would result in deleted items being placed in the <Trash> folder so two questions
