rohn007 say:
A related but separate issue is the question of working directly off of a USB Flash drive.


Use flash drives to "sneakernet" files from one computer to another, but always copy them to the local HD to edit them. Copy them back to the flash drive for next move.

Potential Causes of DOCX Corruption - Problems saving to USB Devices

Windows does not write files directly to a USB flash drive / thumb drive. Due to technical limitations of Flash memory, files are first saved in a cache on the HD. When a full "block" worth of data is available, it is only then moved to the USB device. When you shut down the USB Device "Properly", Windows flushes any partial files from the cache to the USB.  In the following link, Tony J says that his experience is that the most important part of the DOCX files, the text content, is usually located at the end of the file structure. So it is the text content that gets left behind.

http://social.answers.microsoft.com/Forums/en-US/wordcreate/thread/73a1e1b0-e6cf-43b8-a97f-0b01b5c33da3/  This discussion is primarily about missing XML tags, but on page 2 it also discusses the effects of pulling USB devices without stopping them correctly

gailjurczyk say:

Transferring a OneNote notebook from an old PC to a new one

  • Source: The machine that you are transferring from
  • Target: The machine that you are transferring to

Here is what you need:

  • A flash drive
  • I used OneNote 2010 in Windows 7 on both machines for this example

Turn on each computer and start OneNote on Source. Insert the flash drive in Source.

Do the following in OneNote on Source:

  1. File -> Save As -> Notebook -> OneNote Package (*.onepkg) -> Save As
    The Save As dialog box appears.

  2. Before you navigate to the flash drive, take note of the default directory that OneNote uses on Source. You will need to know this to clean these OneNote files from Source after they are successfully transferred to Target. Write the OneNote Default Directory for reference:
    _____________________________________________________
  3. Navigate to the flash drive.
    Use the default filename (on my system, it is Personal.onepkg).
    Leave Current Notebook selected.
    Click [Save].
    A progress bar appears with the message “Packing...”
  4. Open Windows Explorer to confirm the file copied onto the flash drive.
  5. Eject the flash drive from Source.

Do the following on Target (make sure that you have fully exited from OneNote):

  1. Insert the flash drive.
  2. Open Windows Explorer and navigate to view the *.onepkg backup file.
  3. Doubleclick once with the right mouse button to open the file.
    OneNote opens and prompts you for the location of the name and properties of the OneNote Notebook you are transferring from Source to Target.
  4. Accept the defaults.
    On my machine, OneNote named the file Personal and used the following path, where myName is my user name on Target:
    C:\Users\myName\Documents\OneNote Notebooks
  5. Click [Create].
    I had already opened OneNote on this machine, so I received the following message:
    OneNote cannot create a new notebook at:
    C:\Users\myName\Documents\OneNote Notebooks\Personal
    The specified location is already open in another notebook.

    This happened because I had opened OneNote on Target once before, even though I had created nothing in OneNote on Target!

       5.1.   Click [OK].
        5.2.   In the Name field, give it another name. I gave it the name of my employer, let us say

                AcmeCorp.

        5.3.   Click [Create] again.

  1. OneNote opens and copies the new notebook inside. It defaulted to displaying the first page of the first section.
  2. Close OneNote by selecting File -> Exit.
  3. Eject the flash drive from Target.

Go back to Source and clean the OneNote files from the computer.

  1. Close OneNote if it is open by selecting File -> Exit.
  2. Open Windows Explorer and navigate to the OneNote Default Directory (the one you wrote down in Step 2 above).
  3. Select and delete the files within the folder. If the folder still has the default name, such as Personal, leave the empty folder there.
  4. There may be a subfolder called Onenote_RecycleBin. Open it and delete any files inside there, too. Leave the empty folders in place.
