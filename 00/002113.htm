Sub mcrEnumSheets()
    Dim i As Long, sLbl As String
    sLbl = "Worksheet Names"
    Range("A1") = sLbl
    For i = 1 To Sheets.Count
        Cells(i + 1, Application.Match(sLbl, _

As you require this in a very specific context, I'd suggest you create a very specific function rather than using a generic one.  By far the most efficient way of doing this is by means of the GetString method of the ADO recordset object, so first make sure you have reference to the Microsoft ActiveX Data Objects Library (Tools | References on the VBA menu bar).  Then create the following function:

Public Function ToList() As String

    Const NOCURRENTRECORD = 3021
    Dim rst As ADODB.Recordset
    Dim strSQL As String
    Dim strList As String

    strSQL = "SELECT eMailAddress FROM qryMyEmails"

    Set rst = New ADODB.Recordset

    With rst
        Set .ActiveConnection = CurrentProject.Connection
        .Open _
            Source:=strSQL, _
            CursorType:=adOpenForwardOnly, _
            Options:=adCmdText

        On Error Resume Next
        strList = .GetString(adClipString, , ",", ",")
        .Close
        Select Case Err.Number
            Case 0
            ' no error
            Case NOCURRENTRECORD
            ' no rows in query's result table so return
            ' zero length string
            Case Else
            ' unknown error
            ToList = "Error"
        End Select
    End With

End Function

You then simply call the function to return the list of addresses.  In the above I've assumed that the qryMyEmails query has no parameters.  If it does, then, when assigning a value to the strSQL function, build the SQL statement by concatenating the values of the parameters into the string expression rather than referencing them.

word user 66 say:

I have the same issue.

When a clip art image is in edit mode, it is divided into little pieces & parts so that you can change them individually. When you are done editing, you need a way to bring those pieces & parts together into one unified image again so that you can enlarge/rotate/etc. it. In older versions of Word there was a way to quickly ungroup and then regroup all those pieces & parts into one image again. In the 2010 version, if you right-click on your image in edit mode, the "group, ungroup, regroup" options are grayed out.

Holding down "shift," clicking on every single little piece & part, hoping you've found them all, so that you can then group them, is not an efficient way to complete this task.

So how do  you quickly & easily with a click or two unify the edited image and get out of edit mode?

To answer wannabacat's question:

I believe this might work in the 2007 version (or maybe it just works in even older versions of Word):

1) right click on the outer edge--or frame, if you will--of the box your edited image is in (don't click on the corners, only the straight lines of the box)

2) choose the "grouping" option in the pop-up box

3) choose the "ungroup" option; every piece & part of the edited image should then have the little editing dots appear around it

4) right click again on the outer edge/frame of the editing box

5) chose the "regroup" option; your image should be one unified object again

Hope this helps!
