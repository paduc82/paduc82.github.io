The wireless network icon in the system notification area (the right-hand side of the Taskbar) lists all nearby wireless networks that your computer's network adapter can detect.

Keep in mind that wireless signals are just like radio signals, and your computer's wireless network adapter has an antenna. If the antenna picks up a signal, you'll see it displayed.

So this is not a listing of saved or known networks ... it's a display of all the networks that can be detected at that particular moment. At the very least, you should see your own home network, since your wireless network adapter can surely detect that signal from your own router. But you may or may not see anything else.

kristoffer abs say:

Hi Marc,

There are two ways on how we can restore the  Network Profile on your laptop. The first one is by using System Restore which will take your PC back to an earlier point in time. The second option is by exporting the Network Profile from your desktop and importing it into your laptop, since you mentioned that your desktop still has all the saved/known Wi-Fi networks. To do this, follow the steps below:

Method 1: System Restore

  1. Select the Start  button, type control panel and then choose it from the list of results.
  2. Search Control Panel for Recovery.
  3. Select Recovery > Open System Restore > Next.
  4. Choose the restore point related to the problematic app, driver, or update, and then select Next > Finish.

Note: If you’re not seeing any restore points, it might be because system protection isn’t turned on. If there are no system restore points, you may proceed to the second method.

Method 2: Export/Import Network Profile

  1. On your Desktop PC, create a new folder under the C:\ drive and name it as wifi.
  2. Click Start, type cmd, right-click on Command Prompt and select Run as Administrator.

  3. In the command prompt window, type the following command:

    netsh wlan show profiles
  4. This will display the available profiles on the computer.


  5. To backup all the available profiles on the folder that we created, type the following command:

    netsh wlan export profile key=clear folder=c:\wifi


    Note:    (This command will backup all the saved network profile from your Desktop PC to C:\wifi)

  6. Copy the wifi folder to a USB flash drive or an external hard drive and move it to your laptop's C:\ drive.

  7. On your laptop, open the command prompt window again and Run as Administrator.

  8. To import the Network Profile to the laptop, type the following command:
    netsh wlan add profile filename="c:\wifi\Wi-Fi-Account.xml" user=current

    Note:     Replace "c:\wifi\Wi-Fi-Account.xml" with the actual path to the desired backup file. You can check the C:\wifi folder for the list of profiles.



  9. Do this for all the remaining profiles and restart your computer.

Let us know if you need additional assistance regarding the steps provided above.

Could I remove these items with my Disk Clean Up? - Microsoft Community

annthefan say:

Could I remove these items with my Disk Clean Up?

What would be in Temporary files, Web Client/Publisher Temporary files, Compress Old files, Catalog files for the content indexer?   Could I removed these items using Disk Clean Up.  I already removed Temporary Internet files, Recycle Bin , Offline Internet web sites. I have an Windows XP home edition.

a. user say:

If your Disk Cleanup is still offering you to Compress Old Files, you may consider removing that from your Disk Cleanup.

When you run Disk Cleanup, it calculates how much space you could save on your HDD if you choose to compress "old" files. Nobody seems to have a good answer for what XP thinks is an old file though... and with modern technology there is not going to be much of an improvement in your disk space and in fact, your system may run slower since it may need to uncompress these file when they are needed later.

It is just not a good idea to let XP compress your "old" files (whatever old means seems to be unspecified) since it can take a long time and your system may hang or appear to hang while XP tries to figure it out.

If you have a million files on the volume that XP thinks might need compressing and it is going to check them all, it could take a long time to figure that out and your system could appear to stop responding.

Sometimes users interpret this as a problem with their system (and power off or reboot) and that can result in file system corruption, but Windows is really just doing what you told it to do.

It may also may really be hung since MS also tells you how to stop this check for file compression in a knowledgebase article (maybe they don't like it either).

If you have no interest in compressing old files and/or it just takes too long, you can have Disk Cleanup skip that calculation entirely (a good idea).

You'll be cleaning up your system in no time when that check for files that might need compressing is removed.
