Function GetBoiler(ByVal sFile As String) As String
'Dick Kusleika
    Dim fso As Object
    Dim ts As Object
    Set fso = CreateObject("Scripting.FileSystemObject")
    Set ts = fso.GetFile(sFile).OpenAsTextStream(1, -2)
    GetBoiler = ts.readall
    ts.Close
End Function

Query with criteria and calculations - what occurs first? - Microsoft Community

kidde say:

Query with criteria and calculations - what occurs first?

If a query contains some criteria and also some calculations, does the query first filter the records based on the criteria and then perform the calculations in any fields? 

I want to make sure that the calculated fields are not done for every record and then the results are reduced based on the criteria.

 

 

john w. vinson say:

In a Totals query you have your choice: criteria applied using the WHERE clause are applied first, limiting which records go into the total; criteria applied using the HAVING clause are done last, after all the totalling and calculating is finished.

The query grid can be misleading in this context, since it defaults to putting criteria in the HAVING clause. There is a "Where" option in the dropdown for the Totals row which should be used to limit which records are chosen prior to totalling.

 

hansv mvp say:

If you have criteria on calculated fields, Access will have to perform the calculations before applying the criteria.

If you want to force the criteria being evaluated first, create two queries: a first query that applies the criteria, and a second query based on the first one to perform the calculations.
