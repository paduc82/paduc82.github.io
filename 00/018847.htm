john w. vinson say:

The first thing to check is... where do you have the code for RetrvGoal? If it's in a Module which is ALSO named RetrvGoal you'll get this error: modules and functions share the same "namespace", so if you have the same name used for both Access can't find the function. Try naming the module basMyFunctions or something else distinctive.

The second issue is where you're calling it. If you want to see the returned value in your query, don't put it in Criteria; instead put

GoalNum: RetrvGoal([Dte])

in a vacant Field cell in the query grid. Note that it's brackets not quotes - "Dte" would be passing the function a three-character text string, not a date!

dhookom say:

Did you name the module the same as the function? If so, change it. That's one of the reasons most of us mature/old developers using naming conventions.

bernd p. say:

What you call a OneNote folder is by definition called a OneNote notebook.

All OneNote notebooks get synced, even the locally stored notebooks. Sync means that changes are synced from the local OneNote cache to the notebook's master, where ever it is stored.

I assume you created the new notebook without paying attention where it should be created. The default place for new notebooks created in OneNote 2016 is OneDrive. If you create it without a working sync (your case) you create a volatile one.

If for any reason this notebook gets closed (seems your case) it's lost.

If you want to create a new local notebook (avoiding OneDrive) use File ->New -> This PC.

Bernd
