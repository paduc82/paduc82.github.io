kosh vorlon say:

Hi Hullaballo,

 

The simple answer is yes.  The procedure is similar to what you did to find the culprit program - but I think it will be faster and easier.  We need to exclude the file(s) and/or process(es) causing MSE to block the program.

 

To find it easiest, please download and install Autoruns http://technet.microsoft.com/en-us/sysinternals/bb963902 (which is in essence a more powerful version of MSCONFIG).

 

Go to the Everything tab and wait until it fully loads shown at the bottom.  There are 4 columns:  Name, Description, Publisher, and Image Path.  Go through the entire list and find any that are associated with the keyboard.  Try looking for any by that publisher or that have keyboard or anything looking like that keyboard's name or part of that name in any of the columns.  For each, make a note of the full Image Path and proceed until done.

 

Now open MSE and go to the Settings Tab,  One-by-one, add each image path you found to the excluded file and folder list and to the excluded process list and then check the keyboard.  If it starts working, you've found the group containing the culprit.  Now, one-by-one, remove each first from one exclusion list and then from the other until the keyboard stops working.  That's the problem one. Add it back and confirm the keyboard works again.  Continue with the others until only those that need to be excluded are excluded.  If it works, you're almost done. 

 

If it doesn't start working, then go through the Autoruns list and see if you can find any other possible entries.

 

If not, then download, install and run Process Explorer http://technet.microsoft.com/en-us/sysinternals/bb896653 (a more powerful version of Task Manager).  Now do essentially the same thing as you did with Autoruns but for any running processes as something at startup apparently starts something else that isn't in startup and that's the cause - but startup was the logical place to start.  This time you should be able to find the problem if it wasn't in start-up.

 

One final step.  Go to https://www.microsoft.com/security/portal/Submission/Submit.aspx and for each file you needed to exclude, submit the file and indicate it should not be detected as malware.  Eventually, the MSE definitions will hopefully change and you'll be able to remove the exclusions (checking every month or two until it happens).

 

I hope this helps.

 

Good luck!
