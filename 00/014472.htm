Deanagain,

 

My apologies - my intent was to inform you of the availability of the userinterfaceonly option.  I did put the command above your existing code to illustrate - but I didn't test the code.  Here is the tested code that works for me with a sheet already protected with the password "Whatever"

 

Sub abc()

ActiveSheet.Protect Password:="Whatever", DrawingObjects:=True, UserinterfaceOnly:=True
  Range("D3:J6").Select
     With Selection.Validation
         .Delete
         .Add Type:=xlValidateList, AlertStyle:=xlValidAlertStop, Operator:= _
         xlBetween, Formula1:="=xFILTER"
         .IgnoreBlank = True
         .InCellDropdown = True
         .InputTitle = ""
         .ErrorTitle = ""
         .InputMessage = ""
         .ErrorMessage = ""
         .ShowInput = True
         .ShowError = True
     End With
End Sub

 

I believe since the dropdowns are created, the more sophisticated version of protect introduced in xl2002 and xl2003 requires some attention and thus in this case objects must be enabled.   Back in Excel 97 and 2000, you could use the userinterfaceonly option to run code on a worksheet without even providing the password - it was very robust but you can see where it might have made excel more susceptible to virii.  So they have tightened it up a bit.  Similar to the call function which used to be a way to do very sophisticated things with a worksheet function.  It still exists, but has been somewhat limited.  Chip Pearson has a page on it if your interested/curious.

http://www.cpearson.com/excel/Call.htm

While unprotecting and protecting sheet does work, it also risks leaving your sheet unprotected if an error occurs or your code is halted (user hits esc key) after your code has unprotected the sheet and it has not yet been reprotected.  But I would suspect that has a minor chance of occuring.

It seems there's some problem with the template file (Normal.dotm) that Word uses as the base for new blank documents. The cure for this is to delete the existing file, and the next time Word starts it will create a clean new one.
