It isn't the Header [or Footer] that's the problem -- even though they can be removed the blank page would still be there... it just wouldn't have a Header or Footer :-)

What's causing the problem is that there is an empty paragraph at the top of that page, but it isn't in the Header. It's in the body of the document. It's there purposely because a Table in a Word document must be followed by one. The program automatically generates it & it cannot be deleted.   • Quite honestly, all empty ¬∂s should be removed, then create the required spacing between paragraphs by formatting the paragraphs containing text with the
  • As a last resort, select the ending ¬∂, then reduce the Font Size to as little as 1 pt.

charles kenyon say:
No, but you could have a macro that repeats the settings.

Printer settings in each Word session start with the default printer settings for the computer. Any changes to printer settings in one document apply to all documents printed thereafter in that Word session.

So, if used it should either be the last job done in the Word session or you should have another macro to return the settings to their previous status. Such a macro could be run upon opening the document or intercept the print command in the document.

dale howard [mvp] say:

NelsonTang --

When you are assigning a resource to a Fixed Duration task, you will need to specify the correct Units value that reflects the number of hours per day that the resource will work.  For example, on a task with a Duration of 10 days, if you assigned the resource at a Units value of 25% (representing that the resource will work an average of 2 hours per day over the life of the task), Microsoft Project will calculate 20 hours of Work for the resource.

And if you use a Fixed Duration task, and then you change the Duration value, Microsoft Project will always recalculate the Work and will not mess with the Units value.  Hope this helps.
