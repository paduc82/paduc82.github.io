tom_bpc say:

John A.

Notwithstanding the issues of ensuring that you have the latest updated MS Project installation, it looks like you are using a fairly unconventional technique to load cost into your schedule.  Introducing logically-connected tasks whose only purpose is to bear monthly costs - extending from beginning to end of the project - is a sure way to confuse/corrupt your critical path.  Other techniques to consider:

1. Loading costs on summary tasks.

2. Loading costs to specially-created hammock activities that are logically disconnected.

3. Loading costs on manually-scheduled monthly tasks.

4. Loading costs on auto-scheduled tasks that are logically disconnected and date-constrained.

5. Loading costs on recurring monthly tasks (effectively the same as prior suggestion, but differently implemented.)

For a simple project, I see no good reason to load the costs onto a number of monthly activities.  One will do; Project will allocate the costs per month in the Task Usage view if needed.

Good luck, tom

When I looked at this, I made all the links in Word by using cmd-C to copy a cell, range of cells, or named range of cells in Excel, then Edit->Paste Special->(Paste Link, using either the object or the image format in Word). { LINK Excel.Sheet.8 "theFullPathAndNameOfTheWorkbook" "theRange" parameters }

3. When you save that in .docx format, (which is s a set of .xml and other files within a ZIP format file), the text of the LINK field is not saved. What is saved is the XML representing a linked object. This records that it is an Excel.Sheet.8 object, some information about the format used to insert it in the document, and a reference to a "relationship" which records the details of the linked object (the "target")

The "relationship" record has to record the name of the target in a single string. It records it as

rhonda rankin say:

Adding Dates to Comments

Is there a way to set Excel comments so that when you add a comment, it not only shows who added, but the date that it was added?  I know that it can be typed in manually, but would be nice if it could be set up to be automatic.
frank arendt-theilen say:

Hi,

there is no automatic way to set the date the comment was added. You must set the whole comment by VBA:

example:

Sub SetComment()
  With ActiveCell
    .AddComment "Your name:" & Chr(10) & Format(Date, "dd/mm/yyyy")
  End With
End Sub
