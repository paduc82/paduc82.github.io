For a selected set of columns this will set the column width to the widest in the selection.

 

 

Sub col_width()
Dim c As Range, r As Double
r = 0
For Each c In Selection
   If c.ColumnWidth > r Then
        r = c.ColumnWidth
   End If
Next
Selection.ColumnWidth = r
End Sub

I recently purchased Microsoft Wireless Mouse 4000 for my Laptop DELL Latitude E6410. The laptop came with the default pointing device config application. I do not have Microsoft Intellipoint software. I have Win7 64 bit.

Here are the problems with the scroll wheel that I noticed when I bought Microsoft Wirelss Mouse 4000

o) The scroll wheel does not scroll through the combobox items when the combo is NOT dropped down but has the input focus.

e.g. Control Panel -> Mouse -> Pointers -> "Schemes" combobox.

o) The scroll wheel does not scroll the ribbon of MS Office 2007.

o) In Word 2007 the Ctrl+Scroll wheel does not zoom in out

o) The scroll wheel does not scroll the XML Notepad 2007

Solution:

This solution is not mine - it was there on internet already.

It worked for me.

I am posting this with additional steps for 'how to get there'

Go to Mouse Properties -> Hardware tab

Select the problematic mouse from the list "HID-compliant mouse"

Click "Properties"

Go to Details tab

Select the "Device Instance Path" from the combobox.

A value will be displayed e.g.: HID\VID_045E&PID_0745&MI_01&COL01\8&5538EC&0&0000

Note this value.

Open registry editor and navigate to:

HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Enum\HID\VID_045E&PID_0745&MI_01&Col01\8&5538ec&0&0000\Device Parameters

The "Device Instance Path" is the path of the reg key that corresponds to this instance of the mouse.

In the Device Parameters, add the following DWORD (32 bit) reg keys

HScrollHighResolutionDisable = 1

VScrollHighResolutionDisable = 1

Delta = 120 (decimal)

Remove the transreceiver and put it back (this kind of re-initializes the driver). You can choose to restart the PC, too - whatever.

This should work.

Initially I had not put the "Delta" key. But because of that it was scrolling as if I pressed the "Home" and "End" keys i.e. either all the way down or all the way up. After putting the "Delta" key, it worked.

rob koch say:

What you don't realize is that a scan that's been paused is basically useless, since this doesn't allow the antimalware application to perform the entire scan within a single session, which Is required in order to insure context for any active malware that might be present as it can change states during the pause.  My first discussion of this occurred with the developer of a 3rd-party security application several years ago, when he explained this same issue to several of his customers asking the same basic question.

In fact you've just indicated that you understand the custom scan is present, which though it has the same basic problems as the full scan, can be used to appropriately work around this problem by simply selecting smaller chunks to scan which can be completed within the time available.  It may take some experimentation to find the most effective groupings of files, but typically once you do these remain fairly consistent over time.

The most basic confusion that consumers have with scanning for malware though is that there's anything especially useful about a full scan, since in fact these are far more error prone and less targeted than the quick scan, which is specifically designed to target the files, registry and other items where existing malware are known to operate.

The lead developers of Windows Defender, Microsoft Security Essentials and the Microsoft antimalware engines upon which these are based had told me personally that a full scan was only considered useful when either malware had already been detected or was suspected.  This is because the quick scan would already have found anything actively operating, so all the full scan actually does is find possible inactive leftovers such as installers or 'droppers', which may have been left behind either during initial installation or after the malware had been operating for a while.

What this all means is that a full scan is often rife with false positive detections, which though they may be the remnants of actual malware, are often incapable of re-infecting since they contain items like scripts or compressed packages that can only be activated during a session of web browsing.

The chance that a full scan might discover a malware executable that was missed during download due to no definition having been available is very small and would be detected by Defender's real-time protection when the file is next encountered.  In fact, Windows 10 Defender real-time automatically scans all files not recently scanned as you browse into a new folder with Windows Explorer or any other file manager, so it should typically find any malware before you'd even have a chance to attempt to execute the file.

The reason many consumer PC users don't understand why manual scanning isn't really needed at all anymore is that they aren't aware of how the improvements in Windows itself have gained from the corresponding improvements in the processors and additional memory required.  For example, Windows 8/10 both require processors with PAE/NX/SSE2 support since along with other built-in Windows security features like DEP and ASLR, Defender is one of the applications that uses the SSE2 instruction set to improve performance.

So unlike in the past when the PC user often needed to either manually scan recently downloaded files before opening or perform other relatively obsessive compulsive activities in order to remain protected, the modern security applications typically do all of this in the background.  That wasn't always true as recently as Windows 7, since both the processing power of the systems as well as the sophistication of the security applications themselves were far less capable, though over time even some of these have added many of the improvements.

The problem is that the technology has advanced so far and fast that even most relatively knowledgeable PC users aren't aware how any of this works.  I'm only aware because I was a Microsoft MVP during the years that Windows Live OneCare and the later MSE and Defender products were being created and so had direct access to the lead developers and others responsible for these, as well as read every Microsoft article about them available.

I don't even do full scans anymore unless I've been purposefully browsing the web in a dangerous manner and so wish to confirm that nothing got past the more advanced security settings I use along with Defender.  Back in the early days of antivirus I religiously manually scanned almost daily, but now allow the automatic quick scan to take care of that along with the real-time components that do the real job continuously.
