For more information about the commands or options of DISM, in command line follow /? immediately after it, namely press Win+X, A, Alt+Y to open a Command Prompt (Adimin) window and run the following command in it:
dism /?


In here, WIM means "Windows image", a .wim file namely means a Windows image file; it is just like a mirror, while you need to know your face, you would find or watch yourself in a mirror, a Windows image also plays a similar role, namely to the data within a drive it's just as a mirror; to a hard disk's partition, "capture it" is equivalent to "backup it", so "/Capture-Image" namely means from a partition capture or backup a Windows image to a .wim file; "apply it" is equivalent to "restore it", so "/Apply-Image" namely means from a .wim file apply or restore a Windows image to a partition.


So we can use "dism /Capture-Image" command to capture or backup the images of the partitions,
also can use "dism /Apply-Image" command to apply or restore the images (.wim files) to the partitions.


For more information about the dism (Deployment Image Servicing and Management tool) command's options and their arguments, please specify an option immediately before /?.
For examples:
dism /Capture-Image /?
dism /Apply-Image /?


Someone maybe thinks that they are more technical or complicated, but in fact they are very easy and simple.


Let's see.


dism /Capture-Image /ImageFile:<path_to_image_file> /CaptureDir:<source_directory> /Name:<Name> [/Description:Description] [/ScratchDir:<path_to_directory>]


The /Capture-Image option captures an image from a specified drive to a new .wim file.
Captured directories include all subfolders and data, yet an empty directory cannot be captured, namely a directory must contain at least one file.


Necessary arguments:


/ImageFile:<path_to_image_file>
Specifies the storage location and name of the image file that want to be captured.
<path_to_image_file> consists of three parts: the drive letter in Windows Recovery Environment + the path you save to + the file name you want to
For example:
/ImageFile:G:\Software\Backups\Recoveries\Win10Ent.wim
Notice:
The drive letter must be the correct letter that is in Windows Recovery Environment, just as in the above example, in Windows Recovery Environment its drive letter is G, but in normal system it originally is E; here it follows /ImageFile: that just must be G.
About Windows Recovery Environment, please refer to the threads:
Drive letters' confusion in Command Prompt window of Windows Recovery Environment
The consequences of deleting recovery partition
Your PC/Device needs to be repaired - Error code: 0xc0000098


/CaptureDir:<source_directory>
Specifies the image caputuring's source, namely specifies the root directory of the drive you want to capture.
<source_directory> should be the dirve letter in Windows Recovery Environment plus \ (path to the root).
For example:
/CaptureDir:C:\
Notice:
This is the drive letter C of Windows Recovery Environment, not of normal system.


/Name:<Name>
Give it a name for the Windows image you want to capture.
For example:
/Name:"Win10DriveC"
or
/Name:"Windows 10 on drive C of Trybird's PC"


Optional arguments:


[/Description:Description]
Writing clear description is a good habit.
Each thing might not necessarily be remembered into your brain, perhaps after over some time, you forgot what is your Windows image doing for, then you could use "dism /Get-ImageInfo" or "dism /Get-WimInfo" to query and get the result that contains the description you once wrote - maybe you would shout: aha, I see.
If the description string includes spaces or slashes etc., it should be enclosed in quotation marks, as shown in the below example:
/Description:"Windows 10 Enterprise Insider Preview Build 14279 on drive C of Trybird's computer - March 6th, 2016"


[/ScratchDir:<path_to_directory>]
For /ScratchDir argument usage, type "dism /ScratchDir /?" and hit Enter key to execute it:
/ScratchDir:<path_to_directory>
Specifies a temporary directory to be used for extracting files for images servicing. The directory must exist.
If not set, the temporary directory %Temp% will be used.
So the scratch directory on a drive with sufficient space is specified proactively, just is for preventing the error occurs that there is not enough space on the drive which places the default temporary directory in.
You know, the %Temp%, namely the default temporary directory, in my normal system its location is C:\Users\Trybird\AppData\Local\Temp, in the Windows Recovery Environment its location is x:\windows\TEMP, neither spaces are enough,
so the /ScratchDir argument need to be specified explicitly to the path of a drive that has enough free space.

/ScratchDir:G:\Software\Backups\Recoveries\Scratch

This is the drive letter G of Windows Recovery Environment, not its original drive letter E in normal system.


The operation of /Capture-Image option is to back up, yet as reversal the operation of /Apply-Image option is to restore.


dism /Apply-Image /ImageFile:<path_to_image_file> /ApplyDir:<target_directory> {/Index:<image_index> | /Name:<image_name>} [/ScratchDir:<path_to_directory>]


The /Apply-Image option applies an image from a .wim file to a specified drive.


Necessary arguments:


/ImageFile:<path_to_image_file>
Specifies the path and file name of the image that want to be applied.
The drive letter that it is contained must be of Windows Recovery Environment.
If it includes space symbols, it should be enclosed in quotation marks.


/ApplyDir:<target_directory>
Specifies the image applying's target, namely specifies the root directory of the drive you want to apply.
The drive letter that it is contained must be of Windows Recovery Environment.
If it includes space symbols, it should be enclosed in quotation marks.


/Index:<image_index> or /Name:<image_name>
Above two arguments, you may choose any one to use;
or, although unnecessary, you also may use both;
but if you use neither, error will be reported.
I usually prefer to use /Index:<image_index>.
As I had said earlier in my thread - Set startup types of many services once and for all (by using batch commands or powershell scripts)
"Each computer user has his or her own operating culture, need to mutually respect each other's using customs."
Someone likes to repeatedly use "dism /Append-Image" to add additional images to a .wim file, such will make a multiple-image file, namely each unique image file contains multiple image copies, when an original .wim file which had only a single image copy, its default index number which is automatically generated by command always is 1, then append an image into it, new index number will be 2, then append again, new index number will be 3, ..., each additional capture will automatically add a new index number, namely, each append an image to a .wim file, new index number will automatically add 1 for the new image. So, a multiple-image file contains more than one indexes, if you want to use the /Index:<image_index> argument for an image in it, you must first use "dism /Get-ImageInfo" or "dism /Get-WimInfo" to identify which index number is just what you need. (Certainly, if you choose using /Name:<image_name> argument, you also neet first to use "dism /Get-ImageInfo" or "dism /Get-WimInfo" to identify which image name is just what you need.)
Notice:
If you prefer using multiple-image file and want to use "dism /Append-Image" to add an additional image to a .wim file, please first ensure that you have enough disk space for the /Append-Image option to run. If you run out of disk space while the image is being appended, you might corrupt the .wim file.
Someone, like me, does not like to use multiple-image file, only prefer to use single-image file, namely that each unique .wim file is only captured once and only stored a single image copy in it, and its index number of image always is 1.
So, to me, here will be always using the argument as below:
/Index:1


Optional arguments:


[/ScratchDir:<path_to_directory>]
The scratch directory on a drive with sufficient space is specified proactively, just is for preventing the error occurs that there is not enough space on the drive which places the default temporary directory in.
If it includes space symbols, it should be enclosed in quotation marks.
