jay freedman say:

I don't see any good way to load building blocks from more than one folder. As an experiment, I tried storing custom building blocks in a template in a temporary folder, and placing a shortcut to that template in the %appdata%\Microsoft\Document Building Blocks folder, but Word didn't load the template by indirection.

Even if you could put a building blocks template in a shared folder, there would be a problem: When any one user has the file open, no other user (including the administrator) would be able to "edit" it by trying to add or modify a building block. We sometimes get a similar request for a shared Normal.dotm template, which would have a similar problem.

The solution usually proposed for this situation involves this: The "master" building block template is stored on the server in a folder where everyone has read permission but only the administrator (or a delegate) has write permission. Everyone gets a network logon script that copies the master template to the user's personal Document Building Blocks folder; the script knows where the source and destination folders are, so the users don't have to think about it.

You'll need a standard procedure for users to submit items to be added to the master template's building block collection. You'd probably want that anyway, just to have someone review the additions for correctness.
