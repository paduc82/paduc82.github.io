VBA how to use a textbox to determine a file location - Microsoft Community

johnpike_1973 say:

VBA how to use a textbox to determine a file location

I have created a module form with a textbox, that I want to the user to place the complete file location.

what or how do I declare the file location.

Any help will be greatly appreciated.

ossiemac say:

If you use the method you are referring to then I can see the user making mistakes in entering the path for the file location and then the code will fail.

How about a change in the method of doing this.

Below I have provided 2 subs. The first one will display a dialog box that will permit the user to navigate to the location and select the folder and then the code will populate the text box with the folder path.

The second sub will allow the user to select the actual file required and then the path and file name are copied to the textbox.

The file example does not actually open a file. Need to use the Open method with the selected path and filename to open the document.

The code examples are run from a CommandButtons on the Userform.

Note that you should edit the InitialFileName line to reflect the folder that you want as the default location. The User can then navigate to any other folder of choice.


Private Sub CommandButton1_Click()
    Dim SelectedPath As String
   
    With Application.FileDialog(msoFileDialogFolderPicker)
        .Title = "Select required files"
        .AllowMultiSelect = False
        .InitialFileName = "C:\Users\User\Documents\Word\Test Macros"
      If .Show Then
        SelectedPath = .SelectedItems(1)
        Me.TextBox1 = SelectedPath
      Else
        MsgBox "User cancelled." & vbLf & vbLf & _
              "Processing terminated."
              Exit Sub
      End If
    End With

End Sub


Alternatively the following code will allow the user to actually select the required file

Private Sub CommandButton2_Click()
    Dim SelectedFile As String
   
    With Application.FileDialog(msoFileDialogFilePicker)
        .Title = "Select required files"
        .AllowMultiSelect = False
        .InitialFileName = "C:\Users\User\Documents\Word\Test Macros"
        .Filters.Clear
        .Filters.Add "Word Documents", "*.docx", 1
      If .Show Then
        SelectedFile = .SelectedItems(1)
        Me.TextBox1 = SelectedFile
      Else
        MsgBox "User cancelled." & vbLf & vbLf & _
              "Processing terminated."
              Exit Sub
      End If
    End With

End Sub

If running code from Word then you can use code like the following to open an Excel file and print a worksheet from the Excel file and then close the Excel file and close Excel.

The code integrates the previous code I provided to find the required file with the additional code to open Excel and print.

Note that there is no real need to copy the Excel file name to a TextBox because you can use the variable to which it is assigned but I have provided the option to do this if required. (See comments.)

 

Private Sub CommandButton3_Click()

    Dim xlApp As Object
    Dim xlWb As Object
    Dim xlWs As Object
    Dim SelectedFile As String
   
    With Application.FileDialog(msoFileDialogFilePicker)
        .Title = "Select required files"
        .AllowMultiSelect = False
        .InitialFileName = "C:\Documents and Settings\User\My Documents\Excel"
        .Filters.Clear
        .Filters.Add "All files", "*.*"
      If .Show Then
        SelectedFile = .SelectedItems(1)
        Me.TextBox1 = SelectedFile
      Else
        MsgBox "User cancelled." & vbLf & vbLf & _
              "Processing terminated."
              Exit Sub
      End If
    End With
   
    'Open Excel application and assign to an Object variable
    On Error Resume Next
    Set xlApp = GetObject(, "Excel.Application")
    On Error GoTo 0 'Resume error trapping ASAP
   
    If xlApp Is Nothing Then 'Excel is not already running
        Set xlApp = CreateObject("Excel.Application")
    End If
   
    xlApp.Visible = True    'Optional
    Set xlWb = xlApp.Workbooks.Open(SelectedFile)  'Open workbook and assign to variable
    'Alternative code to previous line using TextBox value
    'Set xlWb = xlApp.Workbooks.Open(Me.TextBox1)   'Open workbook and assign to variable
   
    Set xlWs = xlWb.Sheets(1)   'Assumes first sheet in workbook
    'Alternative code to previous line using sheet name
    'Set xlWs = xlWb.Sheets("Sheet1")
   
    xlWs.PageSetup.PrintArea = "$A$1:$B$10" 'Optional to Set print area
   
    xlWs.PrintOut Copies:=1 'Print the worksheet
   
    xlWb.Close SaveChanges:=False   'Close workbook (SaveChanges can be True or False)
   
    xlApp.DisplayAlerts = False 'Required if SaveChanges = False in previous line
    xlApp.Quit  'Close Excel
    xlApp.DisplayAlerts = True
   
    Set xlWs = Nothing
    Set xlWb = Nothing
    Set xlApp = Nothing
   

End Sub

gord dibben say:

This worksheet event code will do that.

Private Sub Worksheet_Change(ByVal Target As Range)
    Dim r As Range
    Dim rr As Range
    Dim iVal
    Set r = Range("A1")          'single cell with dv list
    On Error GoTo ws_exit
    Application.EnableEvents = False
    vals = Array("red", "green", "blue", "purple")
    fruit = Array("apple", "lime", "berry", "eggplant")
    For Each rr In r
        iVal = ""
        For i = LBound(vals) To UBound(vals)
            If rr.Value = vals(i) Then
                iVal = fruit(i)
            End If
        Next
        If iVal <> "" Then
            rr.Value = iVal
        End If
    Next
ws_exit:
    Application.EnableEvents = True
End Sub

don cuthbert say:

I have been testing Insider builds of Pro for Workstations for a few months. It was originally called "Windows 10 Pro for Advanced PCs" in older Insider builds. As you have discovered, there are Insider Preview builds of it.

The only install method that works properly to get Windows Update Cumulative Updates and Security patches (with the Public build 16299.15) is to upgrade to Workstation from Pro. If you upgrade directly to Workstation from Home Edition (by inputting a Workstation Product Key) it installs Pro for Workstation in a manner that currently will not get many updates (although new OS builds, such as Insider builds, do seem to work if installed that way). Maybe upgrading from Home via the Store does not do this.

Insider builds rarely get any patches or Cumulative Updates, so it may not matter how the upgrade was done.

To check if a Pro for Workstations install will take Cumulative Updates, in PowerShell run:

Get-WindowsEdition -Online

or

DISM /Online /Get-CurrentEdition

If the command returns Professional then it will take Cumulative Updates. If the command returns ProfessionalWorkstation the CU's will not install.
