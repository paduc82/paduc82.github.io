Try something like:

  • Alt+F11 to open the VBA editor
  • Alt+IM to insert a new code module
  • In the new module, paste the following code: 

'=========>>
Option Explicit

'--------->>
Public Sub Tester()
    Dim WB As Workbook
    Dim SH As Worksheet
    Dim Res As Variant
    Dim srcRng As Range, delRng As Range, rCell As Range
    Dim iLastRow As Long, iCtr As Long
    Dim sStr As String
    Dim CalcMode As Long
    Const mySheet As String = "Sheet1"                   '<<=== Modify

    Res = Application.InputBox( _
          Prompt:="Insert the search string", _
          Title:="SEARCH STRING", _
          Default:="1234", _
          Type:=2)

    If Res = False Then
        '\\ operation cancelled by user, do exit!
        Exit Sub
    End If

    Set WB = ThisWorkbook
    Set SH = WB.Sheets(mySheet)
    With SH
        iLastRow = LastRow(SH, .Columns("B:B"))
        Set srcRng = .Range("B1:B" & iLastRow)
    End With

    On Error GoTo XIT
    With Application
        CalcMode = .Calculation
        .Calculation = xlCalculationManual
        .ScreenUpdating = False
    End With

    For Each rCell In srcRng.Cells
        With rCell
            .Select
            If CBool(InStr(.Value, Res)) Then
                If delRng Is Nothing Then
                    Set delRng = rCell
                Else
                    Set delRng = Union(rCell, delRng)
                End If
                iCtr = iCtr + 1
            End If
        End With
    Next rCell

    If Not delRng Is Nothing Then
        With delRng
            sStr = .Address(0, 0)
            .Delete shift:=xlToLeft
        End With
    Else
        '\\ nothing found, do nothing
    End If
XIT:
    With Application
        .Calculation = CalcMode
        .ScreenUpdating = True
    End With
    Call MsgBox(Prompt:=iCtr & " cells deleted at the following locations: " _
                      & sStr, _
                Buttons:=vbInformation, _
                Title:="REPORT")
End Sub

'--------->>
Public Function LastRow(SH As Worksheet, _
                 Optional Rng As Range, _
                 Optional minRow As Long = 1)
    If Rng Is Nothing Then
        Set Rng = SH.Cells
    End If

    On Error Resume Next
    LastRow = Rng.Find(What:="*", _
                       after:=Rng.Cells(1), _
                       Lookat:=xlPart, _
                       LookIn:=xlFormulas, _
                       SearchOrder:=xlByRows, _
                       SearchDirection:=xlPrevious, _
                       MatchCase:=False).Row
    On Error GoTo 0
    If LastRow < minRow Then
        LastRow = minRow
    End If
End Function

  • Select Tester | Run

PowerPoint can (sometimes) recover from crashes *when it detects that it's crashed*.  In some cases it gets so abruptly shut down by unexpected circumstances (or just its own bugs) that it doesn't detect the crash, so it doesn't offer to recover any saved backups.

It *sometimes* saves you when PPT crashes.

dale howard [mvp] say:
jlbreyer4 -- If you delete an enterprise resource from the Enterprise Resource Pool, it has the following consequences: 1. The system removes the deleted resource from all past, current, and future projects and replaces them with a same-named local resource. 2. The system removes the resource from the Enterprise Resource Pool. 3. The system removes the user account for the resource. #1 consequnce above is the only issue that I think you should be aware of. Because of #1 above, Data Analysis information about the deleted resource's assignments would be changed from Work and Cost for an enterprise resource and would be changed to Work and Cost values for a Local resource. The data won't be gone, but it will appear in a different way in Data Analysis views. Hope this helps.
