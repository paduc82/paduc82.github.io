ken sheridan say:

The fact that the grouping is by date in my example is irrelevant.  To mirror my example you need to do four things:

1.  Declare a variable in the report's module's declarations area, of whatever is the data type of the column by which the report is grouped.

2.  In the detail section's Format event procedure assign the value of the column by which the report is grouped to the variable.

3.  In the page header's Format event procedure set the return value of the Cancel argument to True if the value of the column by which the report is grouped does not equal the current value of the variable.

4.  Set the ForceNewPage property of the group footer to 'After Section' to force each group to start on a new page.

In principle the example you've cited works in a similar way to my current example.

    2. Create a Microsoft account association with a new Microsoft email - This may require you to send a new invite to their newly created address
  4. FYI - For anyone who needs to associate their current address with a Microsoft account...the authentication is a 2 part process
    1. Part 1: They will need to fill in their name, address, phone number and date of birth etc.
    2. Part 2: They will receive an email verification from Microsoft which will require them to click a link - This proves they aren't a robot.
