That's a perfect example of one small portion of Microsoft's dilemma, which as the malware and other attacks have become more vicious and global in scope, the default security settings including those within Windows Defender have consistently tightened in order to try and aid the typical consumer user more as time has passed.

When I first started helping with the Windows Live OneCare product in 2005-6, the configurability of such items was more available and liberal by default, giving the user more control over the situation.  However, as time went on, the mistakes that consumers often made taught Microsoft that they needed to remove or restrict many of these settings, since it was clear that no matter how simple they seemed to be some number would still manage to misunderstand their purpose.

Add to that situation the use of application cracks and other technically illegal choices, many of which often contain malware as well and there was little reason left for Microsoft to hold back in their efforts to protect the consumer from himself.

That's exactly why I suggested the option of choosing a 3rd-party security application, since even with the free versions of these, the additional configuration options these provide can often allow a workaround to some of these less easily justified situations.

With the fact that Microsoft's security apps are intended for the use of family's often including children, their choices must be designed to best protect even these less aware users, so this is another reason I'd never expect to see these general trends change significantly.  Yes this makes it appear that all users are being handled like children, but as a long-time helper in these forums I can state this appears to more skilled users to be appropriate in many cases.

My point is that Microsoft isn't prohibiting anyone from making other choices, they simply aren't always available within the free security product included within Windows and instead require the use of either a free or paid 3rd-party option.  Since the abuse of what Microsoft and many other commercial software providers feel is legally appropriate use of their software is involved, this seems completely fair to me and many others in the wider community, so you aren't likely to find a lot of sympathy for the illegal or questionable use of such software regardless if that software itself is truly at fault.  From my own standpoint, I'd avoid such shoddily designed software from the start, since this is typically filled with the types of vulnerabilities which can be used by attackers to infiltrate a system.

I'm not sure what the future for such games running on Windows might be, since the obvious direction that Windows 10 is now taking is towards the leaner Windows Core, which will remove the legacy API sets that include the vulnerabilities most often abused by malware.  Once this occurs, the only remaining ability to run such code will either be within an emulator or using one of the older legacy Windows versions, either of which will likely have a lifecycle that will eventually come to an end.

Unless the developers of these games take the effort to port to the newer Windows platform, there may be an end date to their lifecycle.  I can't say how difficult that might be, but it's what I've seen coming ever since the fast development environment of Windows 10 first became available.  I just don't think some number of developers really get what's actually going on.

jay freedman say:

The simplest solution is to put a shortcut to the document on your desktop or your Start menu, and use that shortcut to start up Word.

 

The next simplest is to pin the document to the Office button's list of recent documents, and use the shortcut Alt+f and then the number assigned to the document in the list.

 

If you really must have this specific document open automatically every time Word starts (and I have trouble imagining such a requirement, since it prevents opening Word by double-clicking any other document without the automatic document also opening), record a macro while you open the document. Name the macro AutoExec and save it in Normal.dotm. You'll then have to open the macro in the VBA editor (press Alt+F8, select the AutoExec macro's name, and click the Edit button) and add the full path of the document to the file name, or otherwise Word will look for the document in the Program Files folder where Word's EXE file is.

Jay Freedman MS Word MVP
