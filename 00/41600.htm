The fact that it seems to run fine in safe mode seems to rule out any major hardware problems (or they would probably occur in safe mode as well), but many things that run in normal mode don't run in safe mode so it's most likely that one of those things is causing the problem - though finding it may take some time.

Do you know when this problem began?  Try a System Restore to a point in time BEFORE the problem began.  Here's the procedure: http://www.howtogeek.com/howto/windows-vista/using-windows-vista-system-restore/.  Be sure to check the box to show more than 5 days of restore points.  If the first attempt fails, then try an earlier point or two.  NOTE: You will have to re-install any software and updates you installed between now and the restore point, but you can use Windows Update for the updates.  Use the recovery disk if the system prompt doesnâ€™t work. If that doesn't work,  I suggest you update as many of your device drivers as possible.  In safe mode, the system sometimes uses other more basic drivers rather than the main drivers used in normal mode and this may be the cause of the problem.  Go to Device Manager by going to start / search box and type device manager and enter and then double-click on the program icon that appears.  Check each device for a red x, yellow ! or white ?.  These identify devices with problems (probably drivers, but also conflicts or something else).  Click on each for further details and troubleshooting tips.  If you need to get drivers (and you should do so for as many devices as possible since there's no way of telling which device may be causing the problem - and this will help your system in other ways beyond just potentially resolving this problem), do so from the computer vendor or device manufacturer (NOT from Microsoft Updates).  In fact, you should turn off automatic driver updates from Windows Update as follows: http://www.addictivetips.com/windows-tips/how-to-disable-automatic-driver-installation-in-windows-vista/. Proceed as follows to get the drivers: http://pcsupport.about.com/od/driverssupport/ht/driverdlmfgr.htm.  Once you have the drivers, you can install them through Device Manager as follows: http://www.vistax64.com/tutorials/193584-device-manager-install-driver.html.

If that doesn't help, try doing a clean boot http://support.microsoft.com/kb/929135 and see if the problem persists.  If the problem goes away, it's just a matter of following the procedures in the article to find the culprit causing the problem and then delete, remove, deactivate, or uninstall it.  Once done, be sure to return your system to normal status according to the procedures and reboot.  See if things now work normally.  If the problem persists in clean mode, just return the system to normal status according to the procedures and reboot - this procedure is not going to help resolve the problem.

If that doesn't work, go to Start / All Programs / Accessories / Command prompt and right click on command prompt and click run as Administrator (you can do this in safe mode if necessary).

Type sfc /scannow and enter and let it run.  It will scan and try to fix some of your system files.  Hopefully it will complete with no corruption it could not repair (if there is such corruption post back here or try to analyze it to find the problem file(s) using http://support.microsoft.com/kb/928228.  Try to post any corrupted files here so we can see if they can be repaired with good copies from the installation disk (unless there are too many).

While in Command Prompt, type chkdsk /f /r and enter and let it run. It will want to schedule to run at the next startup. Answer yest and then reboot to run the program.   It will scan and try to fix any corruption or bad sectors on your hard drive and mostly remove that as a potential cause.

You may have a lot of updates to re-install (including any service packs you had to remove).  If the version on the system came with SP1 or SP2 pre-installed and the disk is an earlier version, then you'll need to make a slipstream disk as follows: http://www.vistax64.com/tutorials/151606-vista-sp1-slipstream-installation-dvd.html.

The native Disk Manager (diskmgmt.msc) has a defragging tool. Don't expect too much from a defragging action. bossdweeb say:

"Boot and shutdown times feel slower than they should."

- The only way to determine if perception is a reality is to time bootup and shutdown, but

you still have to allow for at least two variables - If you have installed software that loads with Windows (msconfig>Startup), or have been making a lot of changes that need to be written

during shutdown.

I have no idea why 'system' is displayed on your machine in addition to the OS partition, but

I stopped using Windows 7 defragmenter within a couple of weeks and went for AusLogics Disk Defragmenter.  AusLogics has both a quick defrag and 'optimise' defrag option, and seems to

work well.

While the HDD, including the partition containing the OS (in the case of using an HDD which has

been partitioned) should be defraged regularly, it makes sense that the OS  itself cannot be defraged as it would not be possible to move OS files while the system is in operation.

(Note that the OS is limited to the Windows folder, so the rest of C:\ , including your User folder,

and the three Program folders do get defraged.)

However I would assume that the OS does contain fragmented files,  due to Windows updates being installed, if for no other reason.

A boot time defrag, as you seem to have done previously (in your other thread), would be the only option, but this is the first I've heard of a boot time defrag, and I doubt it is really necessary (with very rare exception).

- My Win7 install is still running smoothly since being installed early 2010, and that includes being imaged to an SSD on a new machine with entirely different hardware (AMD/ Intel >  Nvidia /AMD).

Of course my perception may be skewed by not being able to verify how quick my Windows is,

because I don't have a clean version  to compare with.

I have no issues running high spec games though, so what me worry... (ignorance is bliss ).

Regarding the Page File fragmentation. You can defragment the PF by disabling it, rebooting and
