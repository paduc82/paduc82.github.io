jeeped say:
Once problem with approximate matches your data might run into is that the column being looked up needs to be sorted. While your data is actually sorted, you are only showing data from rows 786 to 792 and I don't think that all of your data is sorted.

The best way to do this may be a 'helper' column that splits off a keyword that can be used as an exact match. Your Drug Lookup worksheets would look like this.

   

The formula in D786 is,

=TRIM(LEFT(B786,IFERROR(FIND(CHAR(32),B786),99)))

But you will probably want to start at at D6 and fill down.

The Phar**macy worksheet could reference that column to return the Drug Class from column C.

  

The formula in G3 is,

=INDEX('Drug Lookup'!$C$6:$C$999, MATCH(TRIM(LEFT(F3, IFERROR(FIND(CHAR(32), F3),99))),'Drug Lookup'!$D$6:$D$999,0))

Addendum: your data looks like it is in a table so the formula might be closer to this,

=INDEX(Table1[[#All],[Drug Class]], MATCH(TRIM(LEFT(F3, IFERROR(FIND(CHAR(32), F3),99))),Table1[[#All],[Helper]],0))

Splitting off a word and relying upon it as the key to a lookup is not always successful but with a little massaging of the data and keyword column (and setting some rules for input) you should be successful.

A bound form is one  whose RecordSource property is the name of a table or query, or it can be an SQL statement.  In a form bound to a table for instance, you would have controls on the form in which the ControlSource property of each control would be the name of a column (field) in the table.  You simply enter the data into the control in the case of a text box, or select an item in the case of a combo box.  The record is automatically saved when you move to another record in the form, close the form or otherwise explicitly save the record.  No code is necessary.

As you are new to relational database design you might like to take a look at DatabaseBasics.zip in my public databases folder at:

https://onedrive.live.com/?cid=44CC60D7FEA42912&id=44CC60D7FEA42912!169

Note that if you are using an earlier version of Access you might find that the colour of some form objects such as buttons shows incorrectly and you will need to  amend the form design accordingly. 

If you have difficulty opening the link, copy the link (NB, not the link location) and paste it into your browser's address bar.

This little demo file provides a simple introduction to some of the principles and methodologies used in the design of relational databases in Access.  As regards forms in particular, you might also like to look at FormsDemo.zip in the same OneDrive folder.

scottgem say:

Bound forms are directly connected to a table by using the table in the form's recordsource. Bound forms are one of the bigger advantages of access. With a bound form Access handles all the I/O between form and table. YOU don't have to (as you have done). While there are times when an unbound form is needed, I don't see it in this case. So I would recommend you rebuild the form using the Form Wizard and let it build the form for you. Then you won't have to worry about the Insert Into statement. 

And I would do this after you add the Modules tables.

gord dibben say:

You can link the checkboxes to the cell in which it lies then filter on TRUE or FALSE.

You can hide the TRUE or FALSE in the cell by formatting as  ;;;

Here is some code to add the link to each existing checkbox.

Sub LinkCheckBoxes()
    For Each cb In ActiveSheet.CheckBoxes
        cb.LinkedCell = cb.TopLeftCell.Address
    Next cb
End
