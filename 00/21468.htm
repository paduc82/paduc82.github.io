You are making the problem much more difficult than it is. Access has provided a builtin function to do what you ask: StrConv.

Try using the textbox's AfterUpdate (not LostFocus) event:

Private Sub Full_Name_AfterUpdate()

No looping, indexes, substrings, etc. needed.

jay freedman say:

Did you make the form with "legacy form fields" or with "content controls"?

If you used content controls, then grouping the whole form is the correct action, and you should not protect the document for filling in forms.

Protecting the document for filling in forms is meant for legacy form fields, and only for forms that use them.

peroivkovi say:

I make form with "legacy form fields" and I tried to protect document.

When I make form with "legacy form fields" then I can tab fillable form, is that possible with content controls?

jay freedman say:

I make form with "legacy form fields" and I tried to protect document.

When I make form with "legacy form fields" then I can tab fillable form, is that possible with content controls?

Yes, you can tab from one content control to the next. If you group the whole document, users will be able to edit only inside the content controls. If you try to edit anywhere else in the text, a message will appear in the status bar at the bottom of the Word window.

There are two differences in the behavior between legacy form fields and content controls.

  • Users cannot tab out of Rich Text content controls, because the Tab key inserts a tab character in that type of control. Instead, you can use the arrow keys or the mouse. This might be a reason to avoid Rich Text controls if possible and use Plain Text ones instead.
  • With content controls, tabbing from the last control in the document does not jump to the first control, the way it would with legacy form fields.
