john mcghie say:

Hi Pol:

Seriously: create some styles. The macro approach is not going to be stable or reliable, please trust me on this: adding and removing tab stops as direct formatting is likely to lead to document corruption.

If you have a BodyText style, create BodyTextT to be exactly the same but WITH the tab at 7 centimetres. Don't be afraid to change styles in a document, that's what Word expects, and it is built for it.

I don't believe for a moment that the code I suggested is out of your league at all: you are using correctly-typed variables, Arrays are just one step up the ladder from that.

Several of the people posting in here would write it for you if you offered to pay, I would run it up for you myself, except I am right out of time: it's that time of the year.

This is roughly how the store routine would look.

  Dim TabArray()

 

 

For Each p In Selection.Paragraphs

    ReDim TabArray(p.TabStops.Count, 1)

    For Each aTab In p.TabStops

        With aTab

            TabArray(aTab, 0) = .Position

            TabArray(aTab, 1) = .Alignment

        End With

    Next ' aTab

Next ' aParagraph

 That declares a "Dynamic" array named TabArray. We do not decide how many rows and columns yet because we don't know how many tabs we have.

 Then in ReDim TabArray(p.TabStops.Count, 1) we count the tab stops in the paragraph and set up as many rows in the array as there are tab stops, and two columns, 0 and 1.

Redimming the array has the side benefit of throwing out the previous data.

 Then it just walks along the paragraph, storing the position and alignment of each tab stop in a row of the array.

Having done that, you can use your existing routines to inspect the content of the array instead of the paragraph, and delete the tab at 7 centimetres if you find it, add it if you don't.

Then clear ALL of the tabs in the paragraph.

Then iterate through the array adding every tab you find in there to the paragraph.

Then do the next paragraph.

Hope this helps

jay freedman say:

I'll take a flying guess that what you're describing is plain text that shows lines breaking in the middle of words rather than between words. If that's what you see, it has nothing to do with any buttons or options or other things that you can control. Instead, it indicates one of two things: a corrupted document file, or the use of a font that's considered to be a "symbol" font rather than a text font. Even if you're using a standard text font, some kinds of corruption can make Word see it as symbols.

One thing you can try is to copy the entire document except the last paragraph mark (clicking the Â¶ button to show those marks will make it easier to exclude the last one) and paste it into a new blank document. If your document contains section breaks, copy/paste each section separately except the section breaks at the beginning and end of the section.

If there is corruption in the original, and if it's in the part of the file represented by the last paragraph mark and/or section breaks, this procedure will leave it behind. If that works, you'll have to reconstruct any headers and footers because those are also stored in the section breaks and the last paragraph mark.

More details at http://www.word.mvps.org/FAQs/AppErrors/CorruptDoc.htm.

Drop caps - Microsoft Community

smoke_998 say:

Drop caps

I find that sometimes the drop cap function does not work. That is, it works on some paragraphs, but other paragraphs cannot have drop caps. Why does it do this and how can I rectify this?
suzanne s. barnhill say:

A drop cap must be the first letter (or letters) in the paragraph. Make sure that you are actually at the beginning of a paragraph (previous paragraph ends with a paragraph break, not a line break), and that the character selected is actually the first (though actually, I find that even if a different letter--or no letter--is selected, Word will make a drop cap out of the first letter of the paragraph).

 

If the paragraph begins with a first-line indent, Word will drop the cap and indent the first three (or selected number of) lines, but if you have a tab character or spaces before the first letter, the Drop Cap option is not available.

If anyone is having a cycle of updates error once they download (Windows 10 Build 10130) [Solution]

  1. Start Command Prompt as an administrator.
  2. Write "net stop wuauserv" without the quotes. if it fails try it again and it'll say update service has been stopped successfully.
  3. Write "rd /s SoftwareDistribution" again without the quotes.
  4. Write "net start wuauserv" to start the service back and start updating. 

it worked for me so I thought I'd point it out.
