You can copy the formula down to allow for future data growth. You would test a cell to the left to see if it contains data and if so then execute some calculation. If the cell to the left is empty then the formula would return a blank.

Something like this...

Let's assume your query data gets dumped into the range A2:D10. You have some formula in column E starting at cell E2. Your data dump may eventually grow down to row A100:D100. So, you want to enter the formula in column E and copy it all the way down to row 100 testing the cell in column D to see if it is populated.

Something like this:

=IF(D2="","",do_something_here)

Excel Page Layout/Highlight Problems - Version 1708 - Microsoft Community

abq_dev say:

Excel Page Layout/Highlight Problems - Version 1708

The latest monthly channel release of Excel (Ver. 1708 - Build 8431.2079) seems to have a formatting problem with the page layout view. When a header or footer is present (with any content inside) in a spreadsheet, any cell highlighting performed using the latest build will not be reflected in the page layout view. In fact, when attempting to add highlighting while in page layout view to a spreadsheet that has content in the header/footer, the selected range of cells turns completely black for a second and then appears to be unaffected by the fill tool. However, switching to any other view or printing the spreadsheet will show that the cells have indeed been highlighted. Switching back to page layout view still shows a blank cell background for the affected cells, even after saving the file, closing/re-opening Excel and re-opening the file. Strangely, pre-existing cell highlights from documents that were edited and saved with previous builds of Excel still display correctly under the same conditions.

All of my users on this version of Excel are running into this issue with spreadsheets that we use and update frequently. We are all running Windows 10 (64-bit) Ver. 1703 (Build 15063.608) on various Surface Pro devices.

Steps to reproduce:

  • Open Excel and create a blank workbook
  • Highlight a range of cells in any color with the fill tool
  • Switch the view to Page Layout (View tab -> Workbook Views section)
  • Enter some text in any header or footer cell on page 1
  • Select a cell in the body of the spreadsheet to move the focus away from the header/footer cell

When you perform the last step above, any highlighting that you've done will seemingly disappear.  However, the highlighting will show again if you switch to Normal or Page Break Preview view or if you delete all content from the header and footer of the spreadsheet and repeat the last step above.

If anyone else has experienced this and found any workarounds, please comment.
