We ran into this exact issue with Outlook 2016 on OS X Sierra: one particular sender's emails would crash on REPLY or FORWARD when received.

Fortunately we control both the sender and recipient sides, so we did a number of tests to determine that the signature HTML was involved. We then experimented until we found the offending HTML, and removed it.

Specifically, an IMG tag like this, that was generated from a 3rd party product (exclaimer) had added styling to an IMG element that was responsible for crashing Outlook. This is also "quoted printable", hence the =3D in the code below:

img id=3D"IMG1" style=3D"BORDER-TOP:0px;BORDER-RIGHT: 0px;BORDER-BOTTOM:0px;BORDER-LEFT;0px"

When this STYLE was removed from the IMG element in this particular signature, Outlook 2016 on OSX Sierra stopped crashing.

Long story short: it's the signature HTML's fault, or outlook's inability to encode/re-encode/parse/whatever this code when REPLY or FORWARD is attempted.
