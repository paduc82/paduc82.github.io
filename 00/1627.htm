wayne-i-m say: The basic problem is that sub-reports do not "remember" what is shown in another part of the report (main or sub).  You can of course reference the main report form the sub, but this will not give you (I think) what you need.


There are a few methods of doing it, but they are all work-arounds.  The simplest method I can think of would be to make the sub reports "main".  You could just copy the details from the current main and create a new report with just the details which you now have in your sub.  Note in this case you would need to amend the underlying querry to add the details you had in the old report.  Then (if you do this) you could just use the code I gave above.

If you really want to use a sub then (again in the querry) you could add a calculated field with the text that want to insert (this is the first page", "continuation sheet", etc.  You need to reference the record above in the query so you have a count of the record - if it's (for example) over 10, 20, etc than have Continuation (or whatever).  Then, in the sub report, just limit the number of records per page.  etc.


As I said there is no simple mehtod of the using the sub-report's footer (as they don't really have one).  Of course you can add an other sub and count the number of rows in the first and if it's over a certain amount the use an "if" to specificy the text.


You will need to come up with a work-around if you want to keep using a sub, but again the simplest method (which will work) is to not use a sub.  You can simply print, email, etc, any number of reports so the user would not realise that you'd altered the design to put all the items which were in the old main into the new report's "report header" so it will only show on the 1st page.

e.j. gun say:

When you link a button to a particular slide, it will stay linked to that slide, even if you add slides in front of that slide, or you move the location of that slide to somewhere else in the document.  Try it!

Make sure you add the "Custom" button so that you can choose which slide you want to go to.

 

HTH,

Eric

thatguy52 say: Huh, thats odd it works. I tried adding slides before and after and it definitely was not working a few days ago. I'm at my wits end. Oh well, at least it works now.

steve rindsberg say:

You might have been linking to Next/Previous, which will ALWAYS link to the slide after or before the current one, so in that case, adding/deleting slides on either side would change the slides you linked to.

As Eric says, when you link to *specific* slides, the links hold, even though you move/delete/add slides. 

A possible exception:  if you have bazillions of links, you could exceed the amount of link storage that PPT allocates.  After that, adding further links is liable to cause existing links to break.
