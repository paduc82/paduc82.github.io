Best way to do long, multi-chapter manuals - Microsoft Community

paulaharlow say:

Best way to do long, multi-chapter manuals

I am currently working on a manual that will have at least 5 chapters and be several hundred pages long. There will be many figures and drawings in this document. Should I create each chapter separately? If so, how do I make the Chapter numbers different for each chapter so the Figure captions and page numbering is correct?

Thank you for your help.

doug robbins - mvp office apps & services (word) say:

Word can handle a document of that size as a single document so there is no real reason to create each chapter as a separate document.  For conveniences however, you could create each chapter as a separate document, basing each one on the same template and then when finally ready to publish or print the document, copy and paste each chapter  after the end of the previous chapter so that you then have it all in the one document.

You should of course make regular backups of your work in case something goes haywire, which ever way you do it.

macropod say:
Word can handle a single document of up to 512MB in size, including graphics, and containing as much as 32MB of text. That said, Word performs better with smaller documents so, depending on your requirements and system performance, you might find it better to have a separate document for each chapter. If you need to consolidate the chapters into a single document, this can be done via INCLUDETEXT fields or, if you just need a common Table of Contents, RD fields.

paulaharlow say:

Thanks. Now I'll just Google INCLUDETEXT and RD fields!

macropod say:

For some reason, most of the MS links on these fields seem to be broken. For the INCLUDTEXT field, try:

https://support.microsoft.com/en-us/kb/212041

The syntax for the RD field is essentially the same, except that the RD field has a relative path switch (\f) and doesn't support any of the other  INCLUDTEXT field switches. See also:

http://windowssecrets.com/forums/showthread.php/154379-Word-Fields-and-Relative-Paths-to-External-Files

http://wordribbon.tips.net/T008557_Using_RD_Fields_with_Chapter_Headings.html

suzanne s. barnhill say:

I would incline toward Doug's advice rather than Paul's on this one. I've created very complex documents in a single file, and I would strongly recommend that a single file be your ultimate goal. If you feel it would be easier to handle the chapters separately, you can do so, but do plan to put them together in the end. This greatly simplifies creating a TOC and index, not to mention numbering pages, chapters, figure captions, etc.

As Doug says, base all the individual files on the same template. Use styles consistently and, if your captions include a chapter number, base them on the same autonumbered heading style in each chapter. Then when you put the chapters together, the figure caption numbers will automatically sort themselves out. The same will be true for footnotes and endnotes. If you're using endnotes and want them at the end of the book but before an index or appendixes, set them to "End of section" in each separate chapter file. When you put the book together, you can suppress the endnotes for each section up to the last chapter, and they will all appear after the last chapter and before any end matter.

If you allow compression (ie, UNcheck "Do not compress images ...") but choose High Fidelity as your default resolution, PowerPoint should leave your images alone.  That means that you can use an image editing program to resize your images to whatever pixel dimensions you've determined will work best on your monitor, likely something approaching 4k.  Most such programs allow you to set the quality level when you save as JPG.  Bumping it down from 100% to, say, 95 or even 90% can give you considerably smaller images w/o materially affecting image quality.  Some programs (IrfanView is a favorite of mine, and free) let you do the resizing/JPG conversions with quality settings, etc. in batch mode. 

For photos, I'd go with JPG rather than PNG because they'll be markedly smaller.

Alternatively, you can import your photos as-is, no pre-processing, and choose one of the higher resolution settings in the Advanced pane; 150, 220 or 330 ppi, for example.  A bit of testing with the images you typically use will show you which setting is best.

steve rindsberg say:

JPG doesn't pixelate any more than any other raster format (PNG, TIF, etc) at a given resolution.  EVERY raster image will pixelate if enlarged enough (or to come at it from the other side, if it's too low a resolution for its intended size).

JPG *can* show compression artifacts; sometimes these look a lot like pixelation, sometimes like hairlines around sharp edges (the edge of text, for instance).  If you save the same image several times but use different JPG compression options each time, then any ill effects on the image will be due to compression artifacts.
