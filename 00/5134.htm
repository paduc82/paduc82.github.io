denisec[msft] say:

Hello-

We are aware of this issue.  Here is some detailed information and workaround.

SYMPTOM:

In Mac the OS may repeatedly ask for permission to access the keychain or deny Office access to the keychain all together. These issues may manifest in several ways:

  • Repeated prompt that “Microsoft <Office APP> wants to use your confidential information stored in Keychain-Item-Name”. As seen in Image_1
  • Office asking you to sign in to your account every time the application starts.
  • After signing in you get the error message “Authentication Session Expired”.

 

In any of this cases, Office is unable to correctly read all the keychain items.

 

WORKAROUND:

To give Microsoft Office access to the keychain please follow these steps:

  1. If any of these apps are open please close them: Word, PowerPoint, Excel, OneNote and Outlook
    1. Other running applications may also try to access the keychain. It is recommended to close as many running applications as possible.
  2. Open your 'Keychain app' 
  3. Select your default keychain on the left side, this is usually the 'login' keychain.
  4. Now click on the lock icon on the top left side to ensure the lock is in the locked state.
    1. Sometimes it takes several seconds or retries for the keychain to get locked again. Specially if other applications are running, this may take several seconds.
  1. Now open any Office apps(Word, Excel, PPT, OneNote or Outlook), you should see a dialog asking for permissions to access the keychain. This dialog may appear multiple times so you will have to ensure to provide the correct password and click on Always Allow when prompted.
  2. After this, you may be asked to sign in again, if this happens please sign in.
  3. After signing in, please close the application and open it again. This time you should not be asked to sign in.

 

MORE INFORMATION:

This issue is becoming more problem as macOS has updated its security requirements to prevent rogue apps to access the keychain. As a result now there are more keychain prompts, which leads to frustration and a tedious user experience when booting the app. When you see a keychain prompt for Office please make sure to select Always Allow to ensure that you don’t see this dialog again. If you by accident clicked on Deny, the app may misbehave and you will have to restart the app and allow access to the keychain items again.

Denise, OneNote Team

andreas killer say:
You have to parse the data by yourself. Select the range that contains your data and execute the macro below.

Andreas.

Sub Test()
  Dim R As Range, C As Range
  Dim Value
  'Exit if not cells are selected
  If Not TypeOf Selection Is Range Then Exit Sub
  'Visit each row
  For Each R In Selection.Rows
    'Search the number
    For Each C In R.Cells
      If IsNumeric(C) Then
        Value = R.Cells(1) & " " & C
        R.ClearContents
        R.Cells(1) = Value
        Exit For
      End If
    Next
  Next
End Sub
