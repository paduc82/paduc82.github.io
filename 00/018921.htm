jay freedman say:

If you're opening from and saving to the OneDrive folder on your hard drive, Word (and OneDrive) cannot "ignore" the fact that you're syncing to the cloud.

The solution for you is to use an add-in such as the one from http://www.gmayor.com/SaveInTwoPlacesAddIn.htm. With that add-in active, you must always open the document from your local (non-OneDrive) documents folder, and configure the add-in to save to that local folder first, and only secondarily to the OneDrive folder.

If something goes pear-shaped with OneDrive in that scenario, the only effect will be that the document may not be complete when opened on another computer from OneDrive. The file in your local-only folder will still be intact and can be re-saved to the OneDrive folder.

rob koch say:

Mickey,

Though I'm not certain what the intended behavior for these settings might be when a major Windows 10 update occurs, I suspect that this is what is causing the settings to return to default.

In earlier upgrades the Windows 10 privacy settings had sometimes returned to defaults, causing those concerned about these settings to complain, with changes to the handling for these made during later updates.

I don't know that I'd depend upon any settings being retained through a significant update process, simply because that would require specifically preserving their state in a separate database and then returning them to the previous state after the update completes.  This isn't always going to be desired when the update process is attempting to fix issues with the underlying code that may include changes to the portions which actually perform the operations the settings control.

For that reason, I will personally never assume that any settings I make will be preserved by Windows itself and will always verify these after any Windows update cycle performs a significant set of patches to the operating system.
