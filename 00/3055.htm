Make sure that the complete line here too is

Me.Filter = "[ItemNumber] = 0 AND [Argument] = 'Default' "

dirk goldgar say:

The best way to do this sort of thing, in my experience, is not to try to build a query with very complex criteria that can handle all the permutations of the fields on your criteria form, but rather to use code on the criteria form to build an SQL statement or a filter string that includes only the criteria for those form fields that are , and then assign that SQL statement or filter string where it needs to be.  For example, you might have code on frmCri along these lines:

'--------------- start of example code ----------------

Dim strFilter As String

' Assuming Field1 is a number field

If Not IsNull(Me.Field1) Then

    strFilter = strFilter & " AND Field1 = " & Me.Field1

End If

' Assuming IssueNr is a text field

If Len(Me.IssueNr & vbNullString) > 0 Then

    strFilter = strFilter & " AND IssueNr = '" & Me.IssueNr & "'"

End If

' ... various other fields ...

' Assuming Field13 is a date field, with no time

If Not IsNull(Me.Field13) Then

    strFilter = strFilter & " AND Field13 = #" & Format(Me.Field13, "mm\/dd\/yyyy") & "#"

End If

' Trim off the first " AND ", if present, from the filter string

If Len(strFilter) > 0 Then

    strFilter = Mid$(strFilter, 6)

End If

' Open a form to display the results, filtered as specified.

DoCmd.OpenForm "MyForm", WhereCondition:=strFilter

'--------------- end of example code ----------------

Yes, it's probably possible to write a query that, all by itself, will look at the criteria on frmCri and figure it out, but the query is very unlikely to be at all efficient.

Incidentally, I took the "Like" operator out of my example criteria, to simplify any confusion over whether it was being used deal with Nulls or whether you really wanted to return all records where IssueNr contained the specified string anywhere within it.  If the latter, then you need to revise the criterion to use Like, after all.
