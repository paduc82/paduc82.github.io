    murrayw say: Unfortunately, the Microsoft ID used on a Windows 8 device (computer or phone) is an "account" in itself, and it is not based on any hardware identifier (as you correctly suggested - it is not linked to phone number, IMEI, MAC Address, etc.). At this stage it is not possible to "merge" accounts, or transfer content from one account to another. So here is the deal:

    a) If you used the Outlook site to upgrade/migrate your Hotmail account, then both accounts will be actually be the same (linked via two addresses, but the same account), and you should not have any issues. Please note, however, that to test this (or change the current Microsoft ID on an existing handset), you will need a new or reset handset to start the initial configuration process using the new Microsoft ID. You cannot switch accounts after initial setup.

    b) If you have already claimed a new Outlook account, separate to this migration option (i.e. created a new account via the sign-up process), then that account will be completely separate and will not retain your purchases. There is currently no way to fix this as you cannot currently merge or transfer purchases from accounts. And, unfortunately, there is a 270 day period between account deactivation/deletion and the ability to reuse the email address chosen - so you cannot even go down that path.

    c) On the up side, if you aren't too attached to your new Outlook email address, you should still be able to upgrade your existing Hotmail account to an Outlook account and use that as your new Microsoft ID. You will, of course, have to choose a new email address to use as the ID and re-purpose or close your existing Outlook address

    A Gmail account is an IMAP account which is limited to only syncs email. Therefor, Calendar, Contacts, Tasks and other folder types only exist on your local machine and are not synched with Gmail.

    As Outlook is a classic desktop application, there is no Live Tile support for it.

    normd say:

    Not having any definitive reply to this question, we have created a workable solution that others can use as required.

    Points of interest: (a) the built in Navigation For does not offer the developer any special functionality.  (b) "Tabs" are not dynamically managed to scroll with drop list if the screen is too narrow; (c) they acan not be added dynamically; (d) if they are hidden by code, their space is not closed up, and on-and-on

    1.So, you can accomplish the same UI effect of the Navigation From/Control with more application control by creating a main form and placing push buttons in the Form Header section (or a radio group with the button style).  The same Quick Styles and Themes that are applicable to these push buttons that are available for the Navigation Control tabs.

    Next, add a sub-form control to the form's detail section with a generic name like frmSubForm. Add the following code to the Form_Resize event. 

          On Error Resume Next
          frmSubForm.Width = Me.InsideWidth - (frmSubForm.Left * 2)
          frmSubForm.Height = Me.InsideHeight - frmSubForm.Top

    If you use pushbutton controls, then for each button click action, add the following event handler to set the sub-form's object.  If you use Radio Buttons then use the frame's click event with a case statement for the value of the button that was clicked.

       Private Sub btnLoggedEvent_Click()
          frmSubForm.SourceObject = "<form name to show>"
       End Sub

    This is all that the Navigation Form/Control does for you.

    You will quickly see that using this approach as an alternative to the Navigation Form/Control that you have more control of the UI presentation.  For example, if the user resizes the Access application window so that the buttons do not fit horizontally, then with code you can resize your custom Navigation form's header section height and re-position the pushbuttons into multiple rows so that they will all still be visible to the user.  One person posed the related question of how to control when the target form is opened for Add or Edit.  With the Navigation Form/Control you do not have any real good control of this since all that you can use is the simplistic "Navigation Where Clause" condition.  With this self-created alternatative, you have full control as the developer to manage the OnClick() event rules for each pushbutons'; OnClick() event, including running pre-check queries, denying access to the default form, changing it's target or event redirecting the user to some other form.  Also keep in mind that you can decide whether to open the form in the sub-form control or as a separate form in the Tabbed set or as a modal dialog window.  None of this flexibility comes with the built in Navigation Form/Control (especially since it fails to operate in a compiled ACCDE).

    I hope that this is beneficial to other's with the same requirement for distributing compiled Access applications with the concept of the Navigation Form/Control as part of their UI design.

    Macro for duplicates and move rows - Microsoft Community

    jonathan_ja say:

    Macro for duplicates and move rows

    Hello,

    I have an excel sheet with around 20,000 rows of data with multiple columns. One column contains the account number
    and there are multiple rows of data having the same account number on this sheet.

    First, I need to find duplicate rows having same account number then, leave one row on the sheet and move rest of
    the rows into a new sheet on the workbook.

    If I have the code for the above, I can run the same macro on the new sheets recursively until I have several
    sheets with non duplicate data.


    Any help is greatly appreciated!

    Thanks in advance!

    tom ogilvy say:

    Jonathan_JA,

     

    this assumes you data starts in row 2 with headers in row 1 so that the first header is in A1 and that you have headers out to the last column in row 1.

     

    it adds a sheet on the end of the tab order for the duplicate rows.

     

    make the sheet you want to process the activesheet before you run the macro.   So you can run this recursively.

     

    Test is on a copy of your workbook to be sure it does what you want.

     

    Sub abc()
    Dim sCol As String, sh As Worksheet
    Dim r As Range, cell As Range, lastColumn As Long
    sCol = "E"   ' column with account number
    Set sh = ActiveSheet
    Set r = sh.Range(sCol & "2", sh.Cells(sh.Rows.Count, sCol).End(xlUp))
    lcol = sh.Cells(1, sh.Columns.Count).End(xlToLeft).Column
    Set r1 = r.Offset(0, lcol - r.Column + 1)
    sform = "$" & sCol & "$" & 2 & ":" & "$" & sCol & 2
    r1.Formula = "=Countif(" & sform & ",$" & sCol & 2 & ")"
    r1.Formula = r1.Value
    sh.Cells(1, r1.Column).Value = "HeaderZZZ"
    On Error Resume Next
     sh.ShowAllData
    On Error GoTo 0
    sh.Copy after:=Worksheets(Worksheets.Count)
    Set sh1 = ActiveSheet
    sh.Range("A1").CurrentRegion.AutoFilter Field:=r1.Column, Criteria1:="<>1"
    Set r2 = sh.AutoFilter.Range
    Set r2 = r2.Offset(1, 0).Resize(, 1)
    Set r4 = sh1.Range(r2.Address)
    sh1.Range("A1").CurrentRegion.AutoFilter Field:=r1.Column, Criteria1:="=1"
    On Error Resume Next
      Set r3 = r2.SpecialCells(xlVisible)
      r3.EntireRow.Delete
      Set r3 = Nothing
      Set r3 = r4.SpecialCells(xlVisible)
      Application.Goto r3
      r3.EntireRow.Delete
      sh.AutoFilterMode = False
      sh1.AutoFilterMode = False
      sh1.Columns(r1.Column).EntireColumn.Delete
      r1.EntireColumn.Delete
    On Error GoTo 0
    End Sub
