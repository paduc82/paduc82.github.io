john - project say:

Nick,

Pardon me for stepping but I feel you should be aware that dynamic masters in Project are subject to corruption. Files in the structure must never be renamed, moved, overwritten or saved off to another location. Also, ideally all files should reside on a local drive under the same directory and the structure should not be operated over a network.

If you have the discipline to adhere to the above, then the probability of corruption is minimal. If not, you might want to consider using a static master instead of a dynamic mater. As Julie noted, by default the master will by dynamic. To create a static master, un-check the "link to project" option in the lower right corner of the Insert Project window. A static master is a snapshot in time of all the inserted subproject, so it needs to be re-created each time you need a update of the whole structure, but this can be automated.

Numbered Lists in Tables

I am trying to create numbered lists in a 2 column table.

The first column uses the standard 1, 2, 3, etc. number format. The second column uses 1.1, 1.2, etc, where the first number corresponds to the step number in the first column. Each row starts a new step.

I have created styles for the two list types, however my document uses multiple tables. While I can "Restart at 1" for the first column of the new table, the second column continues the number sequence from the previous table (e.g. 4.3 instead of 1.1).

Is there a way to fix this?

Thanks!

suzanne s. barnhill say:
Instead of manually restarting, set up the list with an unnumbered style as Level 1, using Levels 2 and 3 for your numbered paragraphs. The unnumbered style can be any style that reliably intervenes between tables. With Level 2 set to restart after Level 1, you should avoid manual restarts.

luxanais say: So in other words, I would need a new style for the first column for each new table in the same document?

suzanne s. barnhill say:

No, I don't think that would work, at least not if you have text in the first column in each row (the numbering would restart in each row). Instead, assuming that there is text separating the tables (which there has to be), you would make that the Level 1 style. For example, if you have a table title or caption, you could make that style the top style, or it might be Normal, Body Text, or some other style that intervenes between tables. See http://word.mvps.org/faqs/numbering/listrestartfromstyle.htm.
