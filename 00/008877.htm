OK, here's what you want to do:

Insert your video.  I'd start over after deleting the existing one in case there are any "left-over" settings that'll interfere with what we're going to do next.

While the newly inserted video is selected (or select it again if need be), go to Animations tab.

Click the PLAY animation.

In the Timing section, choose Start: With Previous (assuming you want the video to play immediately; otherwise, choose On Click).

Go to the Transitions tab.

Choose whatever transition you'd like, then in the Timing/Advance Slide section, put a check next to After: and set it to 00:00:00

Done.  What we've done here is to tell PPT to IMMEDIATELY advance to the next slide, but since PPT will never advance until after any animations are done, it'll play the video first, then advance. Automatically.

It's cumbersome to use IIF in the way you've used it. The best way is to build a custom SQL statement; only including values that are being inserted. In the following code, I've assumed the following business rule - if there is no subscription amount, there is also no corresponding date.
 
Private Sub Archive_Subs_Click()
    Dim strSQL As String
    Dim strSQL1 As String
    Dim strSQL2 As String
   
    If Not IsNull(Me![Subs Amount]) Then
        'Only add arguments and values for Subs Amount if there is something to add
        strSQL1 "[Subs Amount],[Subs Paid],"
        strSQL2 = Me![Subs Amount] & ",#" & Me![Subs Paid] & "#,"
    End If
   
    If Not IsNull(Me![Subs Other Amount]) Then
        'Only add arguments and values for Subs Other Amount if there is something to add
        strSQL1 = strSQL1 & "[Subs Other Amount], [Subs Other Paid],"
        strSQL2 = strSQL2 & [Subs Other Amount] & ",#" & [Subs Other Paid] & "#,"
    End If
   
    If Len(strSQL1) > 0 Then
        'Get rid of the training commas
        strSQL1 = Left(strSQL1, Len(strSQL1) - 1)
        strSQL2 = Left(strSQL2, Len(strSQL2) - 1)
       
        'Build the resulting SQL statement
        strSQL = "INSERT INTO [Subscription History] (" & strSQL1 & ") VALUES (" & strSQL2 & ")"
    End If
   
    DBEngine(0)(0).Execute strSQL, dbFailOnError
End Sub
