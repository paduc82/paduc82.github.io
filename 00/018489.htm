Multiple Date Formats in a Table? - Microsoft Community

z_a_s say:

Multiple Date Formats in a Table?

Hello.

Just a quick question. I have a table that lists individuals, addresses, and their date of birth. The date of birth field is set so dates must be entered in this format "mm/dd/yyyy". However, I do not always know what the Month/Day is, and only have the year. Unfortunately, Access will not let me type in just yyyy. Is there a way I can have both?

Thanks

ken sheridan say:

Assuming the value will be either the full date, or the year only, or Null,  you could do so in a single column of text data type, not date/time.  If you then wanted to return their current age from the column, or their age at a given date, assuming that the mm/dd/yyyy format is the local short date format on the system, a standard function for returning  a person's age could be modified as follows:

Public Function GetAge(varDoB As Variant, Optional varAgeAt As Variant) As Variant

  If Not IsNull(varDoB) Then
      If IsMissing(varAgeAt) Then varAgeAt = VBA.Date

     If IsDate(varDoB) Then
            varDoB = CDate(varDoB)
     Else
            varDoB = DateSerial(Val(varDoB), 7, 1)
      End If

      GetAge = DateDiff("yyyy", varDoB, varAgeAt) - _
            IIf(Format(varAgeAt, "mmdd") < Format(varDoB, "mmdd"), 1, 0)
    End If

End Function

The amendments to the original function are shown in italics.

This would give an accurate age where the full DoB is entered, and an approximate age if only the year is entered by assuming a DoB at the midpoint of the year.  You can see how it works with my own DoB in the debug window, using in this case the UK local short date format of dd/mm/yyyy:

? GetAge("01/11/1946")
 70

to get my approximate age as of today  on the basis of the year only:

? GetAge("1946")
 70

would be correct, but:

? GetAge("1946",#2017-08-01#)
 71

to return my approximate age on 1 August would make me older than I actually would be on that date!
