1. Anywhere in the source document footer, press Ctrl-F9 twice to create a nested pair of fields, thus { { } }.

2. Edit these fields so you get {SET PageCount {NUMPAGES}}, then press F9. The fields will probably disappear, but that's normal.

3. Save the source document.

4. In the destination document footer, insert an INCLUDETEXT field pointing to the PageCount bookmark in the source document.*

5. Select the number inserted by the INCLUDETEXT field in the document and press Ctrl-F9 to enclose it in field braces (e.g { 25 }).

6. After the number inserted by the INCLUDETEXT field  (25 in the example above), press  Ctrl-F9 again to generate another pair of field braces '{ }'. You should now have { 25{ } }

7. Edit the field code so you end up with {=25+{PAGE} \# 0}

8. Press F9 to update the field.

jeffrey min say: Hi Brett,

I can confirm that the custom IMAP port bug is a known issue for servers that use implicit SSL connections (a.k.a. IMAPS).  I don't know which builds of WM6 (if any) are affected by this bug, but I imagine only the very first wave of WM6 devices are affected.

From my understanding of the bug, implicit SSL will only work on the standard port (993).  Custom ports can use unencrypted and explicit SSL (a.k.a. STARTTLS).
