qa guy commelec say:

I see several problems ---

You do not have a FROM clause outside of a subquery and your JOIN is only inside of a subquery.

Your INNER JOIN will cause all counts to be the same.

I think you need to create a UNION query to build a combined recordset of the contents of Field1 and Field2.   Then LEFT JION that query to your 2 tables and do your counts.

dirk goldgar say:

I'm not sure where the "can't repeat table name" comes from, but you can't do what you seem to be trying to do that way.  I think you could do it by summing a union query, along these lines:

SELECT Sum(Table1Count) AS SumOfT1Count, Sum(Table2Count) AS SumOfT2Count, Sum(JoinCount) AS SumOfJoinCount
FROM (SELECT COUNT(*) As Table1Count, 0 As Table2Count, 0 As JoinCount FROM
        (
        SELECT DISTINCT Field1 FROM Table1
        )
UNION ALL
    SELECT 0 As Table1Count, COUNT(*) As Table2Count, 0 As JoinCount FROM
        (
        SELECT DISTINCT Field2 FROM Table2
        )
UNION ALL
    SELECT 0 As Table1Count, 0 As Table2Count, COUNT(*) As JoinCount FROM
        (
        SELECT DISTINCT Table1.Field1 FROM 
             Table1 INNER JOIN Table2 ON Table1.Field1 = Table2.Field2
        )
)

ossiemac say:

You can use number format and select "Custom" and then insert the following as the custom number format.

yyyy,mm,dd

Alternatively you can use the following method.

You can't actually insert a formula in a cell to change the format but you can insert a formula in an adjacent temporary column like the following (where cell A2 contains the date)

=TEXT(A2,"yyyy,mm,dd") 

You can then select the range with the formulas and Copy and then Select PasteSpecial -> Values to convert them to display the result without the formulas. Then you can copy the column of dates and paste them over the previous column and then delete the temporary column.

Note that the alternative method creates text values and the values cannot be use in formulas as actual dates.

diane poremsky mvp (slipstick.com) say:

You can either use two profiles or two accounts in one profile.  Outlook 2010 and 2013, support 2 exchange accounts in a profile.

 

Two profiles: pros: Absolutely no risk of sending email with wrong account

cons: you need to close and reopen outlook to switch profiles

 

Two accounts in one profile: Pros: you don't need to close and reopen outlook to check mail. Replies will be sent from the correct account.
cons: Outlook uses the email account belonging to the folders you are looking in to send mail, but if you aren't careful, you can accidently use the wrong account.

 

I use 2 accounts in one profile method and I do occasionally send from the wrong account, but in my case, its an annoyance more than anything because the two accounts are both with the same company.
