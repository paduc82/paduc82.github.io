Matching zero or one occurrences of a string in Microsoft Word - Microsoft Community

arielrogson say:

Matching zero or one occurrences of a string in Microsoft Word

I have documents that will sometimes have lists that I need to process.  For example, one list might include the text "110, 105, 115", which I will want to put into numerically sorted order: "105, 110, 115".  So, I need to be able to locate these lists, parse and process them, and then replace the original text with the processed results.

The problem I have is that sometimes the list will include the final "and", and sometimes not.  So, one list might read "110, 105, 115", and another list might read "110, 105, and 115".  The processing is the same either way; the only difference is whether the text "and " is included in the original and final results.

Since the processing is the same either way, I would like to include only one code loop.  I can think of two solutions, but I can't get either working.

1) I could use Microsoft Word's built in Find and Replace functionality.  But I don't think Microsoft Word offers the ability to search for an optional string like "and ", so I would have to include two code loops: one including the optional string and one excluding it.  This would mean having to maintain two loops of nearly identical code, which increases the likelihood that bugs will not get fixed correctly in both loops.

2) I could use a Regular Expression object.  The Regular Expression object can perform a search and return a Collection of all strings that match the regular expression pattern.  But since the items in the collection are read-only, I cannot simply overwrite the value of one of the items with the processed list and achieve a replacement in the original document.  I suppose I could use the index and length properties to find exactly where the original text was in the document and perform the replacement that way, but that seems cumbersome and prone to error.

Does anyone have a suggestion how I could make either solution actually work?

Thanks in advance,

Ariel

arielrogson say:

OK, I came up with my own "solution".  It's not ideal, but it works well enough.

I used the Regular Expression object to find all the lists.  Then, for each list, I parsed it into the individual numbers, re-built the list from the sorted numbers, and then used Find And Replace to locate and replace all strings that matched the original list at once.  It feels like a kludge, since it ignores information about where the original list was in the document, but it works well enough.

diane poremsky mvp (slipstick.com) say:
For 32-bit Outlook:
1. Verify these registry keys exist:
HKEY_CLASSES_ROOT\Wow6432Node\CLSID\{F8E61EDD-EA25-484e-AC8A-7447F2AAE2A9}
HKEY_LOCAL_MACHINE\SOFTWARE\Wow6432Node\Classes\CLSID\{F8E61EDD-EA25-484e-AC8A-7447F2AAE2A9}

2. Delete the Mapi16 registration entries under 64-bit CLSID locations:
HKEY_CLASSES_ROOT\CLSID\{F8E61EDD-EA25-484e-AC8A-7447F2AAE2A9}
HKEY_LOCAL_MACHINE\SOFTWARE\Classes\CLSID\{F8E61EDD-EA25-484e-AC8A-7447F2AAE2A9}

3. Close Outlook.
4. Restart the Windows Search service in Task Manager. This will restart SearchIndexer.exe. This is required since SearchIndexer.exe only checks the protocol locations once when it starts up.
5. Reopen Outlook and wait for indexing to complete. Since you have a large mailbox, it may take a few hours. (Sorry. :( )

On my own installation, I have the first 2 keys but not the 2 I shouldn't have. I also don't have problems searching.

I don't know if you are aware of the new feature added to the Registry editor in Windows 10. They moved the path from the status bar to below the menu and you can paste keys there then press enter to jump to the key - beats browsing. (I hated the redesign until i discovered that. :))

I'm assuming you know how to open the registry editor, but if not, press Windows key + R to open Run, then type regedit and press Enter.
