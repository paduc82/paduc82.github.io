Access allow selecting a Multiple Items Form item only once - Microsoft Community

karol szustakowski say:

Access allow selecting a Multiple Items Form item only once

I have a multiple items form. I want it to make each item selectable only once (It will then be used as a subform).
Here is my subform:

As You can see I can select the same item twice. I want to prohibit this. How to do it?

ken sheridan say: Include the two columns of the subform's table in a unique index.  In the case of a subform these would normally be the foreign key column on which the subform is linked to the parent form, and the column which is to be restricted, e.g. OrderID and ProductID.

The simplest way to do this is to make the two columns the composite primary key of the table.  If you have a 'surrogate' autonumber primary key already, OrderDetailID say you would need to create the index independently.

This is done in table design view by selecting Indexes on the Design ribbon.  In the dialogue which opens enter a suitable name for the index in the first empty row of the Index name column, OrderProducts say.  In the Field Name column of the same row enter OrderID, and with this row still selected, in the Index Properties below, select 'Yes' as the Unique property.  Then enter ProductID in the Field Name column in the row immediately beneath OrderID.  Don't enter anything in the Index name column for this row.  Close the dialogue and save the table design.

You can reinforce this at control level if you wish by entering code in the BeforeUpdate event of the combo box.  For this you do need to give the subform's table a surrogate primary key, however, to avoid the possibility of the code finding a specious duplicate if an existing record is amended:

    Const MESSAGE_TEXT = "This product has already been selected in this order."
    Dim strCriteria As String

    strCriteria = "OrderID = " & Me.ProductID & " And ProductID = " & Nz(Me.ProductID,0) & _
        " And OrderDetailID <> " & Me.OrderDetailID

    If Not IsNull(DLookup("OrderID", "OrderDetails", strCriteria)) Then
        MsgBox MESSAGE_TEXT, vbExclamation, "Invalid Operation"
        Cancel = True
    End If

Replace Default in Advanced Find and Replace - Microsoft Community

j. kateley say:

Replace Default in Advanced Find and Replace

I am using Word 2011 Version 14.4.7 with Yosemite 10.10.3. How do I open the "Advanced Find and Replace..." dialog box with the "Replace with: " text entry box already showing? Everything I try opens this dialog box with only the "Find what: " text entry box showing.

a. user say:

FWIW if this is possible, it certainly isn't obvious how to do it, but perhaps someone else can make a suggestion.

The background is that 
 a. using, e.g. Application.Dialogs(wdDialogEditReplace).Show displays the same dialog box as "Advanced Find and Replace", but it shows the tab that you last selected when you ran the full version of that dialog box, and greys the other two out. So if you last did a "Find", you actually get the "Find" version of the dialog and no possibility to switch to "Replace" or "Goto". None of the documented builtin dialog arguments ( see http://msdn.microsoft.com/en-us/library/office/aa211930(v=office.11).aspx ) seems to affect this behaviour, e.g. setting the replacement text sets the text but nothing else.
 b. there does not appear to be a wdWordDialogs enum value (or any value up to 65000 or so) that gets you the full dialog. So I do not think that the full dialog, with all tabs enabled, can be displayed via the Dialogs().Show mechanism. Even if it could, you'd still have to be able to select the tab programmatically, but
 - you can't use Sendkeys because it doesn't work on Mac
 - if you could use the .DefaultTab property of the Dialog, it isn't obvious how because no values are listed for that particular dialog in the wdWordDialogTab enum, and it doesn't look as if they are either allocated sequentially (1,2,3...) or have any obvious relationship to the dialog number or commandbar ID (see below).
 c. You can display the full dialog using the newer CommandBars mechanism, e.g.

Application.CommandBars.FindControl(ID:=23055).Execute

but then you don't appear to have even the level of control over the dialog that you get with the Dialog object, so again, there is no obvious way to pop up the thing that you actually want.

IMO a way to work around that would be to write your own dialog box in VBA to collect the find/replace texts and other settings, then issue the relevant selection.find or range.find VBA statements. Or find an existing implementation. But I don't think a full emulation of the builtin dialog box would be trivial.
