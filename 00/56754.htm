artgs say:

"The default gateway is not available" and intermittent/no connectivity under Windows 8 and 8.1

Intermittent or no connectivity and the error message 'the default gateway is not available' obviously plague thousands of Windows 8 or 8.1 users, as a Google search will show you. Questions on how to fix this issue have already been posed several times on the MS community forum - for example here and here - but have not been answered conclusively. Some solutions fix the problem for some users but for many of us no suggested solution seems to work.

Here, I'd like to share my experience with other users who face the same persistent problem, in the hope that if many people contribute their own experiences, what they've tried, and with what results, eventually Microsoft will work out a real solution, rather than suggest workarounds.

An important note: the connection was intermittent from the start and suddenly disappeared. That is, my laptop was able to connect to the WLAN network but not to internet via one router (router 1), as part of a network, but was unable to connect to the WLAN network at all via my own router (router 2), as the main PC. The router's software notified my that the network card (although detected) could not connect to the router.

Having tried a host of suggested solutions (see list below), I finally downloaded a piece of freeware called Complete Internet Repair (version 2.0.0.1490) and tried various options one by one. The option 'Restore the default host files' miraculously not only restored my connection to the internet via both router 1 and router 2, but also maintained the connection stable for at least 48 hours, without the error message 'the default gateway ...'.

However, on the third day, the connection on router 2 disappeared again. What had intervened? Microsoft automatic updates, that's what. The night before, the PC installed a batch of automatic updates - both Windows and Office 2007 ones - and when I next switched it on, the internet connection had gone. Neither system restore, nor Complete Internet Repair, nor any of the solutions I'd already tried (see below) resolved the issue. However, Complete Internet Repair managed to restore internet access via router 1.

From my (utterly frustrating and hugely time-consuming) experience, I note the following key factors:

  1. Some of Microsoft's automatic updates, either for Windows components or for Office (2007) must seriously interfere with the files, settings, or processes that control connectivity (e.g., some crucial setting may be changed, or some crucial process may be stopped).
  2. The damage to the connection is not reversible via either System Restore or resetting the machine to the factory settings.
  3. Whether the computer is part of a network or the main PC in the network may make a difference to its ability to connect to the network and/or the internet.
  4. The router model does not seem to affect connectivity.
  5. Assigning the IP address manually does not solve the issue.
  6. Third-party software, such as Complete Internet Repair, may resolve some issues effectively but is unlikely to work for all users.
  7. Connectivity problems to do with the 'default gateway' under Windows 8 and 8.1 are likely to be caused by a variety of factors. This suggests that the Windows 8/8.1 components that determine connectivity are inherently problematic and weakly designed.

Hi,

 

Thank you for posting this question on Microsoft Community Forums.

 

I would suggest you to restart the computer and check if you still see the message of Backup in progress.

 

Windows backup continuously runs if some components cannot be copied. I suggest you to temporary disable the anti-virus and firewall and check if the issue still persists.

 

I would like to inform you that there are no automatic settings which will delete the backups, periodically or when necessary. You will have to manually do it, by following these steps:

 

a.     Click on the Start Button and click on Control Panel (All Items View).
