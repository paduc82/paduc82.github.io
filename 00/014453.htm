After export the received files may contain rows in any random order: ABCD, DBCA, ACBS etc. When imprting - the same, the rows in csw file ABCD will apper in the created table as BCDA, ABCD ets. The order of lines is crucial for me. How to make this process stable?


The order of rows in a table is arbitrary.  If a table has a primary key, then Access generally returns them in ascending order on that key; however, there is no design rule that stops Access from returning rows in any order it chooses.  The principles of relational

What to do, then?

1. Give your table a primary key, if it doesn't have one. 

2. When exporting, instead of exporting the table directly, export a query that explicitly sorts the rows in the order you want.

3. When viewing the table in Access, if it doesn't show the records in the order you want, use a query or the table's Order By property to sort the records as desired.

4. If there is no existing key that establishes the desired line order, you can either:

    a. modify the input file to add line numbers that can be used for sequencing, or

    b. import the file by using code to reading it line by line, parsing each line and writing it to the table, in the process adding your own line numbers to the table.
