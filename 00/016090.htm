When I make changes in my custom template dotm file while working on a - Microsoft Community

wideglidejrp say:

When I make changes in my custom template dotm file while working on a document, the changes do not get saved. How do I save the template changes?

When I make changes in my custom template dotm file while working on a document, the changes do not get saved. How do I save the template changes?
john mcghie say:
Each time you change a style, you MUST check "Add to template" in the Modify Style dialog.

Otherwise the change is made only in the document you are working on.

When you are working in Word, the style table for a document is copied into the document at the instant of creation.  After that, Word makes no further reference to its template unless you force it (for example, by checking "Add to template").

Unless you force Word to re-connect the template, any changes you make update only the local style properties table stored inside each document.

When you quit Word (or Save All) if the "Add to template" box has been checked, Word overwrites the definition of every style in the template with the definition of the same-named style in the document.

If the style in the document does not exist in the template, it will be added.  If the template contains styles that are not in the document, they will be left unchanged.

Checking "Add to template" ONCE in an editing session is sufficient.  From the user interface, it causes the entire style table to be written out on save.  However, it's good practice to check it out every time, because it is "unset" by the next save.

Of course, this answer presupposes that the document you were working on is attached to the Normal.dotm template, and that you are looking at the correct instance of Normal.dotm.  Neither may be true :-)

I used to get myself totally confuzzled until I learned to ENSURE that there could only be ONE active copy of any template on the system, and that I knew which one it was.

Hope this helps

kerwinyang_msft7476 say:

hi darren,

thanks for your updates.

we cannot let active directory users without exchange license receive the sharepoint email notification.

about your requirement for no payment for exchange license, may you can use the personal email account such as a outlook account or gmail account which have the ability to receive and send emails as a workaround:
1. as a sharepoint admin at least, go to sharepoint admin center > user profiles > people > manage user profiles> input the user name which you want to change in the find profiles dialog box and click find.

2. select this user profile and click the “edit my profile” option.
 
3. scroll down to find the work email option, then replace the original email address with your own personal email account such as a outlook account or gmail account which have the ability to receive and send emails.
 
4. save and close it. after that, when you share a document with a single user, he will receive the sharepoint notification email in his own personal account.

kerwin
