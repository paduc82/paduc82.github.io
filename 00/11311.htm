jwalker107 say: I've been playing with this in a couple of virtual machines for the last couple of days.  What I'm seeing is that this error, 0x8050800c, has been showing up when I am out of memory in the virtual machine.

The MSSS tool requires 768 MB of RAM on the machine.  Where my virtual machine is configured for only 512 MB, the tool initially fails to load definitions (may get this error on startup), or shows a definitions date of 2006 and fails to update from Internet or local files with this error code.

First suggestion, if you only have 512 MB of RAM, get more.  While 512 is certainly acceptable for Windows XP, the MSSS tool actually includes & boots from Windows PE, a stripped-down version of Windows 7.

Second suggestion, if you are really intent on doing this, is to read up on WinPE, esp. WinPEShl.ini and wpeutil.exe.  I've found that through the combination of replacing MSSS's default shell with a command prompt, on a low-memory system I can boot, then enable a pagefile pointed to the local hard drive, then run MSSS (sometimes needing to run it twice to get the pagefile written), then I can get the system scanned.  I've tried this down to 512 MB but haven't tried with any less RAM than that.

Be aware, that after booting the .WIM file that hosts MSSS's WinPE, the tool will scan to find the definitions file and a filelistxx.dll file from _outside_ of the WIM.  If you get into more advanced setup, like multiple WinPE images on a single thumb drive, be aware you'll need to grab these files from outside of the WIM and store those on the root of the flash drive, or MSSS will crash on startup.

In Word, press Alt+F11; the Visual Basic Editor opens. Press Ctrl+G to activate the Immediate window. In that window, type

     NormalTemplate.OpenAsDocument

and press Enter. Normal opens, but note that it may not be on top of all windows. You can close the Visual Basic Editor. Make the desired changes to the Normal file, and then save and close it. thinkcoolsa say: In reply to deleted message
Since Outlook 2011 does not have VBA support, I have no other option than posting it here.

Anyways, I got the answer for my question "Getting contents of a Incoming Message" .

tell application "Microsoft Outlook"
    set select_message to current messages
    if select_message is {} then
        return
    end if
    repeat with theMessage in select_message
        set theText to content of theMessage
   end repeat
end tell

This gives the content for an incoming email. I have tested it and, it works.
