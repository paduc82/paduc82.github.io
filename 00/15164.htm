a. user say:

You say that you want to count backwards 13 columns from the newly pasted data.  This means that there will always be 13 columns of data being displayed.  If you actually only want the last 12 columns (presumable the last 12 months of data), replace each instance of "- 12" with "- 11".

Dim ws As Worksheet
Dim rLastCell As Range

Set ws = Worksheets("Summary")

Set rLastCell = ws.Cells.Find(What:="*", After:=ws.Cells(1, 1), LookIn:=xlFormulas, _
    LookAt:=xlPart, SearchOrder:=xlByColumns, SearchDirection:=xlPrevious, MatchCase:=False)
    
If rLastCell Is Nothing Then
    MsgBox "No data found...", vbInformation
    Exit Sub
End If

If rLastCell.Column = 1 Then
    MsgBox "No data found beyond Column A...", vbInformation
    Exit Sub
End If

With rLastCell
    With .EntireColumn
        .Copy .Offset(, 1)
    End With
End With

If rLastCell.Column - 12 > 1 Then
    Range(ws.Cells(rLastCell.Row, "B"), rLastCell.Offset(, -12)).EntireColumn.Hidden = True
End If

ws.Activate

Set ws = Nothing
Set rLastCell = Nothing

Hope this helps!

ms office is not able to read my pdf files. - Microsoft Community

priyankawalia say:

ms office is not able to read my pdf files.

why my MS OFFICE is not able to read pdf files?

charleskenyon (ia) say:
Since Word 2013, Word has had the ability to directly open pdf document. However, some pdf documents (from scanners mostly) are really pictures of text rather than the text itself. Just as you can't look under the hood of a picture of a car, you can't edit a picture of a document.

In that case, you need to convert the picture to text. This is a process known as optical character recognition. This is built into Adobe Acrobat and is also in Office OneNote. Most scanner software comes with an OCR component as well. Once you have text, you can edit it with Word.

If you simply want to write on the document (but not in it) you can add a Text Box floating on top of the document layer, whether or not it has been put through the OCR process.

Word documents that have been saved as PDF will not need the OCR process, they retain their text, although not all of their Word structure and formatting.

Finally, documents converted from pdf (or really any other format) to Word can be tough to edit because the conversion process never has a one-to-one matching of how formatting is done. This means that a converted document will seldom be formatted in Word in a way that makes sense. An example is multiple section breaks to change margins, where in Word you would simply change the paragraph indent.

Actually, the issue is related to Exchange Active Sync which seems to not support adding new messages to folders from outside the e-mail account (moving messages between folders does work).  It doesn't just affect Microsoft Outlook, but other mail programs (e.g. Exchange mailboxes accessed via the iPhone).

This problem is relevant to outlook.com because it will make it harder for people to move their e-mails over from another webmail provider to outlook.com, thus acting as a disincentive to switch.  I know that people could move e-mails via an automated process, but sometimes manually moving e-mail is less laborious than cleaning up a mess caused by an automatic move where each side interprets things differently.

The workaround is to use the Hotmail protocol as provided in Windows Live Mail to move your messages from an IMAP mailbox to an outlook.com mailbox.  I think this uses deltaSync which does seem to support adding new messages.

Microsoft, you should enhance Exchange ActiveSync to allow the addition of messages, appointments, tasks, etc. from outside, and you shoudl implement this in your programs (e.g. Outlook 2013) and 3rd party programs (e.g. the iPhone). I believe this will increase the value of outlook.com.
