iif func in access FOR blank cell - Microsoft Community

prosenjitdey say:

iif func in access FOR blank cell

i have a table where two order no : one is old no and other is new no, both are text field. Some order no are converted to new and some are remain in old value, now i want to prepare one new field where if no new order not found then keep old no otherwise insert new no such as

Old no New No No
25258789 25252525 25252525
25256897 25256879

 i use iif func such that : iif(old no), is not null,(new no),(old no)),  but it not work properly, that means some value still remain in blank cell, how can i do it.

ken sheridan say:

If you are retaining the old and new number columns in the table then you should not create a new No column as this introduces redundancy and leaves the table open to the risk of update anomalies.  Instead return a computed column in a query:

SELECT [Old No], [New No],
IIF([New No] IS NULL, [Old No],[New No]) AS [No]
FROM [YourTable];

If you intend to delete the old and new numbers, and retain only the [No] values, then simply update the [New No] column with

UPDATE [YourTable]
SET [New No] = [Old No]
WHERE [New No] IS NULL;

Then delete the [Old No] column from the table and rename the [New No] column to [No].

PS: Relational database tables do not have cells, which is a spreadsheet concept; tables have values at column positions in rows in tables, columns being often loosely referred to as fields and rows as records.
