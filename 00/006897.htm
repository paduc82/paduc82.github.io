The consequences of deleting recovery partition - Microsoft Community

trybird say:

The consequences of deleting recovery partition

I had posted a thread: Drive letters' confusion in Command Prompt window of Windows Recovery Environment
and in it I once said that there were two approaches to enter the Windows Recovery Environment:


1. To enter the Windows Recovery Environment directly from the running Windows operating system's dedicated recovery partition.

2. To enter the Windows Recovery Environment from a booting Windows installation media (like Windows installation disc).

Computer users who are well aware of the importance of having dedicated recovery partition seldom delete it or at least create a USB recovery drive before deleting the dedicated recovery partition. Since deleting the recovery partition is much easier than creating one, novice users often delete recovery partition to gain some disk space, but without doing any necessary steps before deleting.


If I deleted the recovery partition, what will occur?
That is:
The above 1st approach will be failed or resultless.


Since I posted the thread just at build 10130 that before the RTM version, the Windows Recovery Environment already had been changed or improved by Microsoft, such as there was not the "Press F10 for more options" option in the "Startup Settings" screen when it was accessed previously.


I will explain below, if the recovery partition is deleted, how the Windows Recovery Environment is inaccessible.


For entering the Troubleshoot's "Startup Settings" screen, I can select one of the following ways:


Start from start button or start button's context menu:


Keyboard:
Win, Up, Up, Enter, Up, Shift+Enter
or
Win+X, U, Shift+R


Mouse:
click Start Button, click "Power", press and hold down Shift key and click "Restart"
or
rightclick Start Button, point to "Shut down or sign out", press and hold down Shift key and click "Restart"


Touch:
tap Start Button, tap "Power", press and hold down Shift key and tap "Restart"
or
press-and-hold Start Button, tap "Shut down or sign out", press and hold down Shift key and tap "Restart"


Start from sign-in screen:


If I am just in the sign-in screen of Windows, I can click the "Power" tile in the bottom right of the screen, and then a shortcut menu will pop up, then I can press and hold down Shift key and click the "Restart" menu item.


Start from "Settings" immersive control panel:


In Windows 10, press "Windows Key + I" key combination to open the "Settings" immersive control panel, then select the follows in there:
Settings > Update & security > Recovery > under Advanced Startup, click [Restart now] button.


Then the system prompts: "Please wait",
then the "Choose an option" screen appears, I click "Troubleshoot" tile;
then under "Advanced options", I click "Startup Settings" tile;
then under "Startup Settings", I click "Restart" button.


Then the computer will reboot and come to the "Startup Settings" screen.


There prompts:


Startup Settings


...


Press F10 for more options


Press Enter to return your operating system


I press F10, then there prompts:


Press a number to choose from the option below:


Use number keys or functions keys F1-F9


1) Lauch Recovery Environment


I press 1 or F1, then there prompts:


Recovery


Your PC/Device needs to be repaired


A required device isn't connected or can't be accessed.


Error code: 0xc000000f


You'll need to use recovery tools. If you don't have any installation media (like a disc or USB device), contact your PC administrator or PC/Device manufacturer.


Press Enter to try again


Press F8 for Startup Settings


So, you see, after deleted the system reserved partition and recovery partition, the "Lauch Recovery Environment" option is inaccessible.


The consequences of deleted recovery partition are not limited to only this one, under this case whether I run "Microsoft Windows Repair Disc" or run "Recovery Media Creator", all cannot succeed, because their same source just is the deleted recovery partition.



So it had better to remain the recovery partition.


Otherwise, before I want to delete the recovery partition, the following steps are necessary:


Use "Microsoft Windows Repair Disc" to create a system repair disc


Use "Recovery Media Creator" to create a USB recovery drive


Capture Image (.wim file) of USB recovery drive using dism (Deployment Image Servicing and Management tool)


Download or make a ISO file of Windows installation media (please refer to Microsoft's web page - get Windows 10 for non-enterprise edition and Andre Da Costal's thread - Create a bootable ISO file using ESD Image)


Burn the ISO file to a disc


Certainly, if my Windows 10 is not an Enterprise edition, I also can use Media Creation Tool to make an installation media (a USB flash drive or an ISO file):
on the web page, click the [Download tool now] button
or click the direct download link.


Then run the MediaCreationTool.exe, and follow the instructions on its screen.

But if my system is an Enterprise version, how do I do?


As I said previously, one of the necessary steps before deleting recovery partition is making a Windows installation disc, for example, make a Windows 10 Enterprise Build 11082 installation disc, then it will come in handy at any necessary point such as the above 2nd approach - entering the Windows Recovery Environment from installation disc.


So, you see, how useful an installation disc is!
