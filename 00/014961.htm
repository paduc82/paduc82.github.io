tom-s say:

Amy,

I can suggest one way to do this with a couple of named ranges and conditional formatting.

Put the digits 0 to 9 in say A1:J1 and name the range digits

Put the number texts zero to fifty in say A2:AY2 and name the range nums

The apply this conditional formatting formula to your text strings (assuming starting in A3):

=OR(SUM(--ISNUMBER(SEARCH(digits,A3))),SUM(--ISNUMBER(SEARCH(nums,A3))))

 

*** EDIT ***

Come to think of it, you wouldn't need the whole of zero to fifty number texts since there are many part repeats; just zero, one ... twenty, thirty, forty, fifty. So these would go A2:X2

gary's student say: If there is a text string in A1 that might contain numerals, then this array formula:

=COUNT(1*MID(A1,ROW($1:$9),1)) 

will count them. So if =COUNT(1*MID(A1,ROW($1:$9),1))=0 then there are no numerals in A1

Remember the formula must be entered with CNTRL-SHFT-ENTER, not just the ENTER key

graham mayor say:
The numbers are easy enough but we may need an Excel expert to help with a better way of checking number texts. In the meantime the following should help and is fairly fast given the amount of data that has to be checked individually by this method. The macro assumes that the texts are in column A starting at row 1 and it formats the text colour of matching cells to red.


Option Explicit
Sub MarkNumbers()
Dim xlsheet As Worksheet
Dim i As Long, j As Long
Dim vText As Variant
Set xlsheet = ActiveWorkbook.Sheets(1)
For i = 1 To xlsheet.Range("A" & xlsheet.Rows.Count).End(-4162).Row
     vText = Split(xlsheet.Cells(i, 1), Chr(32))
     For j = 0 To UBound(vText)
        If IsNumeric(vText(j)) Or IsTextANumber(vText(j)) Then
           xlsheet.Cells(i, 1).Font.Color = RGB(255, 0, 0)
        Exit For
        End If
     Next j
Next i
End Sub

For example, I wasn't sure what set the conditional define in the project properties meant; I found the conditional compilation argument line (which I've never used) but I'm not sure what goes in there, this is new territory for me.
At first I have to say that the line "On Error GoTo 0" is missing in my example, I've forgotten to enable the error handler after the connection to Outlook is established. I've changed the code above.

You can set a conditional define in a code module with the #CONST statement, but that affects only this code module.

For example if you copy the sub Test from above into a regular module in a new file and you compile the code, you get no errors, because the conditional define if not in affect, means the line
  Dim olApp As Object
is compiled.

If you add the line
  #Const Develop = 1
at the top of the module and compile the code again, you'll get an error, because the line
  Dim olApp As Outlook.Application
is used from the compiler.

There is no difference until this point between the conditional define in the code module or the conditional define in the project properties.

But now add another regular module and move the line
  #Const Develop = 1
to that module. If you try to compile the project, you'll get no error.

And now open the project properties and add the line
  Develop = 1
into the field "Conditional Compilation Arguments". Click Ok and try to compile the project.

As you see you get an error; because that define is valid for the whole project.

BTW, you should use the same GetObject/CreateObject part to access Outlook in sub EMail_Alert.
