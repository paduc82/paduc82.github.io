stuthomson say:

Access 2007 lock file on back-end, users are getting locked out

I have a split Access 2007 database. The back-end is set to "Shared", but people are getting locked out when they open the front-end (on their C drives) . I notice a lock file appears in the backend folder (on our Shared drive). Is this supposed to happen?  How do I fix this?

marshall barton say:

If the "lock" file you are referring to is an LDB file, then it is normal for every open database to have one in the same folder as the MDB/ACCDB file.  The LDB file is used to keep track of which records in which tables are being used by various users so multiple users do not interfere with each other.

Your FE db may be restricted to a single user if it is opened in Exclusive mode, if any object in the file is opened in DesignView, or the folder or the FE db file does not have the needed permissions

stuthomson say: They all have read/write permissons to the BE folder and their FEs are set for "Shared". They are only running standardized querys - no editing.  Should I set the BE file to "read only". 

marshall barton say:

All database files are normally read/write.  Even if users are not updating or adding data, Access likes to be able to write to the file in order to keep track if what's going on.

You said the FE is shared???   If more than one user is using a FE file at the same time, there is a reasonably high probability that the file will become corrupted.  Every users needs to have their own copy of the FE db, preferably on their local hard drive, but at least in their private share area on a server.

If/When Access tries to write to a FE db, that may(?) prevent other users from doing the same thing.  Give a COPY of the FE db to another user and check if that user can work at the same time as someone else is using the shared FE.

 

dirk goldgar say:

When you say the users are "locked out", what is the exact error message they get?

On the shared folder containing the back-end, do they all have read, write, create, and delete permissions?\

The lock (.ldb) file ought to appear, and in fact if it doesn't it's a sign that users will be locked out.  But you say it does appear.  If it appears no matter who is the first user into the database, and goes away when the last user exits the database, then that part seems to be okay.
