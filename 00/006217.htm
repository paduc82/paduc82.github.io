There is a prompt for confirmation, so you most likely deleted the item. But restoring an item from quarantine isnt the same thing as allowing it, so when a quarantined item is restored, it wont appear in the Allowed items list, and there wont be any indication in the History tab. Instead, a temporary path exclusion will be automatically created for the item but that type of exclusion isnt visible in the Settings tab either, so there wont be any indication of which action you took anywhere in the main interface.

 

However, when you restore an item from quarantine it will generate a 1009 event (Microsoft Antimalware has restored an item from quarantine) in the System Event Log (Event Viewer); which will be immediately followed by a 5007 event (Microsoft Antimalware Configuration has changed) that will explicitly record the creation of the temporary path exclusion entry in the HKLM\SOFTWARE\Microsoft\Microsoft Antimalware\Exclusions\TemporaryPaths key.

 

On the other hand; when you remove an item from quarantine, a 1011 event (Microsoft Antimalware has deleted an item from quarantine) will be generated in the System Event Log.

 

 

GreginMich

greginmich say:

1.) Open Event Viewer by typing eventvwr (or just event) in the search box and hitting the Enter key.

2.) Expand Windows Logs and click on the System folder.  

3.) Click on the Filter Current Log option in the Actions pane or Action menu.

4.) Select Microsoft Antimalware as the Event source; enter 1009,1011 for the Event ID; and click "OK".

 

 

Remove is an ambiguous term in this context, but its being used in the sense thats equivalent to delete.

    Slow internet and browsing experience

    I'm experiencing some internet performance problems when I'm using my HTC Windows Phone 8X for browsing the web with Internetexplorer (IE) or certain apps, who rely on a solid internet connection to update their content (like CNN, engadget or other news apps). In Switzerland, I've a really solid HSDPA+ connection provided by my carrier Swisscom, there shouldn't be the problem I now describe. I never experienced anything like this with my Samsung Galaxy S2 for more than a year of heavy usage. 

    The bug is the following: The internet connection sometimes turns really slow and different internet pages (theverge.com, microsoft.com, googe.com take whatever you want..) need ages to load in IE, even if I have full HSDPA+ connection After activating and deactivating flight mode, IE is speedy as usual and loads the conntent really quickly. The same is happening with apps sometimes, they've a really slow connection and need a lot of time to load their content. I'am always on full HSDPA+. Sometimes the apps, sometimes IE and sometimes both have a slow internet connection.  The only thing, which helps, is activating and deactivating flightmode. Afther this operation, the speed is as usual. 

    And now comes the mistery: I discussed the issue with some experts and they gave me the hint to switch my APN settings to IPv4 only. Before, I was on IPv4 and IPv6 hybrid mode, and needed to restart my whole device to resolve the slow-internet-issue. If IPv4 and IPv6 were activated and I activated and deactivated flight mode, I had no internet connection at all in IE and only the apps where able to load content at the usual speed. Only a restart of the device helped to recover the original internetspeed and connection of IE. Afther changing to IPv4, the activation and deactivation of flight mode resolves the issue fully, no restart was needed. The experts mean, that Windows Phone 8 has some serious kernel bug regarding the routing. They're thinking, Windows Phone 8 doesn't handle the DHCP lease time correctly. So I think I need to talk to my carrier for further investigations. I've already checked my APN settings together with my carrier.
