The way the cross-reference feature works is this: When you create a cross-reference, Word inserts a hidden bookmark around the text that the reference points to. The cross-reference field then points to that bookmark.

To see the field codes, press Alt+F9. (To turn off the codes, press Alt+F9 again.) In the code of the cross-reference field, the name of the hidden bookmark is the "_Ref" followed by a long number.

To see where the bookmark is (if it exists), open the Insert > Bookmark dialog, check the "Hidden bookmarks" box, click on the name that's in the cross-reference code, and click the Go To button.

If the bookmark name from the cross-reference code doesn't appear in the Bookmark dialog when hidden bookmarks are displayed, then the cross-reference mechanism isn't creating the correct bookmark. In that case, try repairing the Office installation by the procedure in https://support.office.com/en-us/article/Repair-an-Office-application-7821d4b6-7c1d-4205-aa0e-a6b40c5bb88b.

You will need to send the client both the Word file and the .pptx PowerPoint file.

This error has nothing at all to do with memory or disk space per se.  Correctly translated, the error messages should be "I could not find something in the memory that I expected to be there."  The person who wrote the error message figured that the only way that could happen (since he put it there...) is that there was not enough memory to put it in.  Sadly, what really happened is that when he attempted to put the object onto the memory, Office for Mac mangled it, so that when it tried to bring it back, it did not recognise it!

Unfortunately, this won't be a simple fix, so it is unlikely to arrive before the next version (Office 2013, which I guess is about a year away).
