jay freedman say:

The AutoCorrect2007 utility just lets you create a document containing all the entries currently in your AutoCorrect list -- both those that come with the program at installation and those you add later. It won't tell you much about where Microsoft's entries come from.

During development and beta testing, employees and volunteers type a lot of text, and testing software examines the results for common errors such as transposed letters and exchanges of nearby letters on the keyboard. The most common such errors are added to the standard AutoCorrect list. I don't know whether Microsoft maintains or consults any "database" other than that list.

I don't know what you mean by the "scope of the AutoCorrect function". Every time you type a space or punctuation character in an Office program, the code examines the "word" to its left. If that equals an entry in the AutoCorrect list, the replacement is made.

The recognition of names is not part of the AutoCorrect feature. Instead, a selection of common names are included in the main dictionary of the Spelling feature. You can add other names to your Custom dictionary.

jay freedman say:

https://jay-freedman.info/SaveViewLocMulti.zip

Hey, how can this one be used? I'm having trouble working this one out. 

The SaveViewLocMulti add-in contained in that zip file -- which is completely unrelated to the AutoCorrect2007 file -- should be stored in the folder %appdata%\Microsoft\Word\Startup. The instructions for that are on the same page as the link to the zip.

The add-in automatically stores information in the active document during a save, including the view, the zoom factor, the screen location, and the window size. When the document is reopened, that information is used to restore the document exactly as it was when saved. The "multi" part is that if you have multiple windows open to the same document, each one possibly in a different view and a different location, then each one will be restored as it was.

stefan blom say:
You have to click to deselect the Link to Previous option (on the Header & Footer Tools Design tab) before inserting the footer content.

By default, footers (and headers) in a newly created section are linked to those of the preceding section. This applies to each type of footer (header), which must be separately unlinked from its "neighbor" in the preceding section. For more, see http://wordfaqs.mvps.org/HeaderFooter.htm.

charles kenyon say:
Also take a look at using the StyleRef Field. It is possible to get different text in the footers (or headers) for different parts of the document with the same header/footer.

See also Sections / Headers and Footers in Microsoft Word 2007-2013.
