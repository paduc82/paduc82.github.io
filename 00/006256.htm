Bug Report - Sharepoint - Excel - COM Add In - Control/Object not - Microsoft Community

calebcoey say:

Bug Report - Sharepoint - Excel - COM Add In - Control/Object not loaded if COM Add In has a pop up and Excel is loaded from Sharepoint

Bug Report - Sharepoint - Excel - COM Add In - Control/Object

Control/Objects are not loaded if a COM Add In has a pop up (in my case requesting a username and password to create a database connection) and Excel is loaded from Sharepoint.

Control/Objects load and excel file functions as expected when opened from the local drive, only is an issue if loaded from SharePoint.

I tested ActiveX and standard buttons, unexpected behavior is the same for both.

The COM Add-In I have experienced this with (although any COM Add-In that requires a pop-up should do) is ExcelCompanion released by Honeywell.

The excel file Control/Object loads as expected using an older version of the COM Add-In that I have installed on another computer that does not have a pop-up that asks for a username or password. 

Leading me to the conclusion that the pop-up is the event that is interrupting the Excel loading process from SharePoint.

Workaround #1:

If I disconnect the COM add-in, then load the file from SharePoint it will load as expected. I can then connect the COM add-in and enter the username and password, refresh the workbook and all updates as expected.

Workaround #2:

The excel workbook will eventually "time out" for lack of a better term. Once it has "timed out" the border and ribbon will begin to show what you are hovering over, although the mouse cursor will still show the busy animation and the cell area of the workbook will still not visually update.

Once the "time out" has occurred click the "X" in the upper left corner to close the workbook and excel will give you the "Save your changes" pop-up. At this point the mouse cursor will still be stuck in the busy animation but the workbook cell area will begin to update when you make selections or changes.

Untested Possible Workaround #3:

You can use VBA code to forcefully disconnect and reconnect some COM Add-Ins. The ExcelCompanion COM Add-In does support this. Below is an example of my forced connection VBA code (COM Add-Ins have 2 state toggles Disabled/Enabled and Disconnected/Connected. The error handling in my code is for if the COM Add-In is Disabled which is a state unable to be changed using VBA code). I did not try this method because I am not aware of how you would ensure the disconnect and reconnect occurred at the right moments to allow the controls/objects to finish loading. There is also the concern of the Workbook_OnCreate() at least in the past not being called when excel was opened from SharePoint leading to questions of where you put the VBA code you need to run on open.

    On Error Resume Next
    Dim FailureMessage As String
    Application.COMAddIns("ExcelCompanion").Connect = True
    If Err.Number <> 0 Then
        'If the Uniformance AddIn is deactivated somehow it would have thrown an error trying to find "ExcelCompanion" to set Connect = True
        'In that case we want to notify someone and exit excel without sending a report full of errors
        FailureMessage = "Uniformance COMAddIn Disabled - " & ThisWorkbook.Name
        SendAddInFailureEmail (FailureMessage)
        ThisWorkbook.Saved = True
        Application.Quit
    End If

If you need any other information please feel free to ask.

Thanks,

~Caleb Coey

LaTeX math formatting in Word outside an Equation is not working for - Microsoft Community

namwam say:

LaTeX math formatting in Word outside an Equation is not working for super script

In WORD 2010 Professional Plus, I'm using Math AutoCorrect Rules Outside an Equation as described here. These rules seem to work when, for example I type \alpha, \pi etc they get converted to Greek symbols correctly. But when I type r^2 it does not convert it to r squared as claimed by an example in this post. What I may be doing wrong?

a. user say:

As far as I know, the expression "Math AutoCorrect Rules Outside an Equation" has to be taken very literally.

i.e. it does not mean that everything that you might type into an "equation box" will be interpreted as if you were actually typing into an equation box. It means that Word recognises the autocorrect "keywords" \alpha etc. and replaces them. In other words, it does what it does with non-math autocorrect keywords. 

You can see that if you type \quadratic, for example. Instead of the nicely laid-out equation you get inside an "equation box", you just end up with some plain text.

In other words, for proper equation layout, you can't escape the "equation box". So to have r^2 interpreted correctly, you can instead type alt+=, which creates the equation box, at least on Windows desktop Word, then r^2. 

However, if you want to use that particular expression a lot, you could consider creating your own autotext or autocorrect that would insert the entire equation box and content.
