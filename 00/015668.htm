No, Tables represent entities with attributes. What you are doing is defining data using their Object names, not by the data the objects contain. And that's not correct design.

tblProducts

ProductID (PK Autonumber)

ProductName

tblProductDetails

ProductDetailID (PK autonumber)

ProductID (FK)

DetailTypeID (FK)

Detail

ReorderLevel

tluDetailType

DetailTypeID (PK Autonumber)

DetailType (i.e. Waist, Length; Size. etc.

ProductID (FK)

The ProductID is so you can filter the DetailTypes for the products they apply to.

Quantity is NOT tracked in the main table. Quantity is tracked in a Transaction table that records all movement of stock in and out. Something like:

tblTransaction

TransactionID (PK Autonumber)

ProductDetailID (FK)

OrderID (FK)

Quantity

TransactionType (either Incoming or Outgoing)

UnitPrice

You would also need an Orders table to record sales and Purchase orders.
