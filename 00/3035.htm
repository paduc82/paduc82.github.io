bill manville say:

I doubt you will be able to achieve your goal.

The only Excel object that has a MouseMove event is a Chart, and you would need such an event to be able to trigger the display of a picture.

 

As far as I know there is no way to hold pictures within a pivot table structure, so you would need to hold them elsewhere and somehow determine which picture was to be displayed when some event occurred that your program can know about.

 

I think the best you could do would be to have the user click a cell to display a relevant picture.

This would trigger a Worksheet_SelectionChange event.  You might be able to use the following to get you started on the way to discovering what item was clicked and so what picture to display.   


Private Sub Worksheet_SelectionChange(ByVal Target As Range)
  If Not Target.PivotCell Is Nothing Then
    MsgBox Target.PivotTable.Name & ">" & Target.PivotField.Name & ">" & Target.PivotItem.Name
  End If
End Sub
