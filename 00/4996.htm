If your data in column A is formatted as date, try this formula:

=SUMPRODUCT(--(MONTH(A1:A10)=1),--(A:A<>""))

But if your data in column A is formatted as text, try this formula:

=SUMPRODUCT(--ISNUMBER(FIND("January",A:A)))

Note: If you can limit the range in column A to e.g. A1:A100 instead of A:A, the formulas will calculate more quickly.

jim gordon mvp say:

In 2014 your daughter most likely purchased Office 2011 perpetual license.

A stand-alone perpetual license is not transferable from one Mac to another. While the product was still supported, you could call a special phone number and Microsoft would make a policy exception and issue a new license key to keep you going. However, all support ended for Office 2011 in October 2017, and the special phone number no longer is available.

In 2015 Microsoft introduced Office 365 subscriptions for Office for Mac. A subscription allows you to activate and deactivate users at any time (as long as the subscription is active), which allows you to change hardware at will. Stand-alone perpetual licenses are still available, but they still have restrictions on installing on different hardware.

boyd trimmell aka hitechcoach say:

A stored query saved in a compiled state not as text. This can provide enormous speed improvements (especially under stress) as since they do not have to be  interpreted every time it is run.

SQL statements in VBA code is text and must first be interpreted and compiled first before executing. This happens every time it is run.  

Curious, why do you think SQL as text in VBA code  is faster and more efficient that stored queries?
Boyd Trimmell aka HiTechCoach HiTechCoach.com Microsoft Access MVP

apollo67 say: Stored queries are faster than building a query in code.  This is because the stored query is already compiled.  However, there is no problem calling on the stored query from VBA to be executed.  Just to add, this doesn't mean that you shouldn't create queries in code, it simply illustrates the benefit to a stored query over a VBA statement.  Often times building a query on the fly is necessary.

Sub BuildDOTWithMacros()
  Dim Doc As Document
  Dim vbComp As VBComponent
  Dim Contents As String

  Set Doc = Documents.Add

  'Create sub that compiles the project if the file is created
  Set vbComp = Doc.VBProject.VBComponents("ThisDocument")
  Contents = _
    "Private Sub Document_New()" & vbCrLf & _
    "  With VBE.CommandBars.FindControl(ID:=578)" & vbCrLf & _
    "    If .Enabled Then .Execute" & vbCrLf & _

mehdibens say:

Hello!

I understand your frustration and I will try to provide you with a solution. Some threats are persistant in nature and either use technologies to evade detection or are particularly difficult to remove. There's one good and one bad news. The good one is that Windows Defender appears to 'detect' the threat so a removal technique probably already exists in the databases. The bad news is that the threat keeps coming back for some reason. Some reasons I can cite:

- Active threats can keep injecting code after it's been removed. The active threat cannot be removed/detected.

- Windows Defender is failing to detect a malware downloader which keeps re-downloading other malware

- You may have a rootkit that evades detection and/or is particularly hard to remove, and keeps downloading malware

- You may be still accessing the source that infected you from the first place.

etc...

Possible solution:

In order to remedy the situation, you may try to run an offline scan using Windows Defender offline. These kind of offline scans allow you to scan the disk from another bootable medium, other than where the infected system is stored. The main virtue being that the malware (especially rootkits or others like APTs) do not have a chance to start and compromise the system in a way conducive to hiding themselves or avoiding removal.

You can download the tool from Microsoft: here (you'll also find easy to follow instructions).

Make sure to select the correct edition of Windows (either 32 or 64 bits). To verify which version of Windows you are using, please visit: http://go.microsoft.com/fwlink/?LinkId=184763

The Windows Defender offline tool will walk you through the creation of a bootable medium.

Please note the following before you start:

1- It is advised to download and run the tool from a clean (presumably) computer other than the compromised one.

2- You'll need a spare USB, which will be formatted and transformed by the tool to become an active bootable disk containing some Windows components necessary to empower Windows Defender offline. You can similarly use an empty CD or DVD in case you don't have a USB available for use.

3- You will need to know a bit your way around your computer, especially how to change the boot order in the BIOS (or through advanced startup if you have UEFI). Really you just need to know how to boot from your USB/CD/DVD...

Note: if your main OS partition is BitLocker protected (if you've turned encryption on), you'll need to either suspend BitLocker or have your recovery keys (48-digit keys) ready for when prompted by the Windows Defender offline system.

If you continue to have problems, or find something in this post to be unclear, please let me know.
