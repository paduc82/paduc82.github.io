Windows 10 LTSB Magnifier: Ctrl + Alt + mouse scroll wheel - Microsoft Community

xx_asteriz say:

Windows 10 LTSB Magnifier: Ctrl + Alt + mouse scroll wheel does not work

Since the original question got closed, this is attempt to reopen it and confirm that the problem is real and not the matter of "one box". All other Ctrl-Alt variants work fine, just not the scroll wheel. The event even propagates to whatever windows is under the mouse pointer (tested with Chrome, it reacts to Ctrl - wheel). 

Version 1607 (OS Build 14393.2551)

There is probably a larger root cause here.

LTSB is a promise, cast in stone, that no "upgrades" will be pushed. But, first of all, fixing this and publishing the a new .msi to install new binary(es) is not "upgrade" - it's a bug fix.

Second, serious triage needs to be applied to some other aspects like Defender related changes not being built for LTSB. In particular "Controlled Folder Access".

The problem with that one is probably that the implementation has unholy Win Store/UWP tie up and that is huge no-no for LTSB. It might even be just the matter of not having full build separation. I've heard that people trying to dev something with WinRT can't use debug dll-s/libs because they are apparently built with a tie up to a latter OS build.

The iron clad "no upgrades" promise of LTSB Is actually (supposed to be) "no fluff, no new bugs" promise and not "we won't do anything, period".  The thing that is needed in order to keep the LTSB promise is to release important updates as separate packages to WUP/SUS and make sure that people can pick what they want and what not.

For example, just about everyone would take "Controlled Folder Access" (as long as it's not fluffy => 1 small section in Defender settings, 1-2 sections in Exclusions + straight reg keys), people with hi density laptop LCDs and 4K monitors smaller than 32" would take magnifier.

On the other side no one would take an attempt to push "desktop edge margin" (not letting you place icons inside 0.5-1" margin, let alone place it half width out) under any excuse since that's "anniversary" time bug which no one wants to tackle, because it is hard :-) and it steals a lot of screen real estate unless you have something like 43" monitor.
