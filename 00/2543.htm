The second error you encountered -

This is a Windows Update issue. MSE shares the pipeline used by Windows Update.

See: http://support.microsoft.com/kb/910336

 

The original one simply tells you that the engine crashed down and couldn't restart.

 

You could have contracted a virus during the file transfer. The files did not simply move from a to b. The OS read the file on the source, loaded it into memory and then wrote the file to the destination. While that was happening, MSE real time protection was scanning the files as they were being accessed.
