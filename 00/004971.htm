If you want to break up a large amount of text across multiple slides, put the entire text into a NOTEPAD file, wordwrap OFF.  Each slide's worth of text should be a single paragraph, so press ENTER at the end of any line where you want the next bit of text to be on a new slide.

pauldcook say:

open multiple access tables in a single workbook

Hi,  I have an Access MDB file that contains sixteen tables and I need to be able to place a workbook in the same folder as the mdb file and when I open the workbook I would like to have Excel open the mdb file in the folder (the mdb file always has the same name) each table's data appear in a worksheet that has the same name as the table from which it is presenting the data.

cristin yan say:

Hi,

 

Please try MDB2Excel as follows:

 

Public Sub MDB2Excel()
    Dim adoConn         As New ADODB.Connection
    Dim rstSchema       As New ADODB.Recordset
    Dim rstData         As New ADODB.Recordset
    Dim strConn         As String
    Dim strQuery        As String
    Dim strTable        As String
    Dim strProgress(1)  As String
    Dim wsDest          As Worksheet
    Dim i               As Long
    Dim j               As Long
    Dim k               As Integer
    Dim lngFieldsCount  As Long
    Dim lngRecordCount  As Long
    Dim intTablesCount  As Integer
    Dim colTableNames   As New Collection
   
    strConn = "Provider=Microsoft.ACE.OLEDB.12.0;Data Source=" & "D:\Test.MDB;Persist Security Info=False"
    adoConn.Open strConn
    Set rstSchema = adoConn.OpenSchema(adSchemaTables)
   
    Do Until rstSchema.EOF
        If rstSchema!TABLE_TYPE <> "ACCESS TABLE" And rstSchema!TABLE_TYPE <> "SYSTEM TABLE" Then
            strTable = rstSchema!TABLE_NAME
            colTableNames.Add strTable
        End If
       
        rstSchema.MoveNext
    Loop
   
    If Not colTableNames Is Nothing Then
       
        intTablesCount = colTableNames.Count
       
        '/* Loop each table.*/
        For k = 1 To intTablesCount
            CloseRst rstData
           
            strQuery = "select * from " & colTableNames(k)
           
            With rstData
                .CursorLocation = adUseClient
                .LockType = adLockOptimistic
                .CursorType = adOpenStatic
                .Open strQuery, strConn
               
                lngFieldsCount = .Fields.Count
               
                ' Test if destination sheet exists.
                On Error Resume Next
               
                Set wsDest = Worksheets(colTableNames(k))
               
                If Err <> 0 Then
                    Err.Clear
                   
                    On Error GoTo 0
                   
                    ' Insert a new worksheet after the last worksheet in the active workbook.
                    Sheets.Add After:=Worksheets(Worksheets.Count)
                    ' Rename the added sheet's name as current table's name.
                    ActiveSheet.Name = colTableNames(k)
                Else
                   
                    ' Empty cells.
                    wsDest.Cells.Delete xlUp
                    wsDest.Activate
                End If
               
                strProgress(0) = "Processing: " & k & " of " & intTablesCount & " (Tables)"
               
                '/* Get all fields from current table. */
                For i = 1 To lngFieldsCount
                    Cells(1, i).Value = .Fields(i - 1).Name
                Next
               
                lngRecordCount = .RecordCount
               
                '/* Get all data from current table.*/
                For j = 1 To CLng(lngRecordCount)
                   
                    strProgress(1) = strProgress(0) & "; " & j & " of " & lngRecordCount & " (Records)"
                   
                    ' Change the status bar.
                    Application.StatusBar = strProgress(1)
                   
                    '/* Fill data in cells.*/
                    For i = 0 To lngFieldsCount - 1
                        Cells(j + 1, i + 1) = .Fields(i).Value
                    Next
                   
                    .MoveNext
                Next
            End With
        Next
       
        MsgBox "Task completed!", vbOKOnly, "Tips"
       
        ' Restore the status bar.
        Application.StatusBar = ""
    End If
   
    CloseRst rstSchema
    adoConn.Close
End Sub

 

Public Sub CloseRst(rst As ADODB.Recordset)
    If rst.State = adStateOpen And Not IsEmpty(adStateOpen) Then rst.Close
End Sub

 

Note:

[1]. Remind to check Tools >> References >> Microsoft ActiveX Data Objects x.x Library (I checked 6.0) in VBE environment;

[2]. You can change "D:\Test.mdb" in this code line to change your MDB file path: 

      strConn = "Provider=Microsoft.ACE.OLEDB.12.0;Data Source=" & "D:\Test.mdb;Persist Security Info=False"

[3]. Sometimes the macro is running so quickly that you might not always see the progress entirely in the status bar.

julie fka profjulie say:

When you tap the attached PDF file, the file will open up and SAVE it within the file system of the app that is used to open the file. So if you use the Windows Phone Adobe Reader app, you can find it later by opening up Adobe Reader and browsing through the files that are stored within the app.

I prefer to use OneReader as the PDF reader for my phone since it provides many more options than the Microsoft PDF reader app and the Adobe Reader app. Files that are opened and stored within the OneReader app can be:

  • Pinned to the phone's home screen - this would work well for your boarding pass scenario
  • Shared via OneDrive (the file is uploaded to your OneDrive public folder & you can email a link to the file afterwards), email (the file can be sent as an attachment), bluetooth (the file can be sent to any other bluetooth connected device)

You could also forward your email (that includes the attached PDF file) to OneNote. When you do this, a new note is created in the Unfiled Notes section of your default OneNote notebook.  In the screenshot below, I forwarded a copy of my resume and references to OneNote via email.  You can see that the full text of the files are not included in the note - the files are included there.

To view the text of the files, I would have to tap each file to open it (and save it) within the PDF reader app that is installed on my phone. I don't think this method gives you anything better than just opening (and saving) the file when you receive it (as described in the 1st paragraph above). See this article to learn how to "Email your notes into OneNote."

When I travel and am using an eBoarding Pass, I'll open the Boarding Pass up on my phone, then take a screenshot of it, then pin the screenshot to my phone's homescreen (I use the "Pin image to start" app)...that way, the eBoarding Pass is easy to get to when I'm going through security & boarding the plane. The screenshot is stored locally on the phone, so I don't have to worry about the phone's internet connection when I'm at the airport.
