    Unfortunately there is no completely effective IE Repair tool.  The so-called troubleshooters, including RIES, seem to focus on setting customization features back to defaults rather than on repairing the OS support of the browser's operation.   BTW do you have the IE Performance troubleshooter?  Its appearance seems hit or miss, depending perhaps on the version of the OS.  Perhaps Srimadhwa can show us where to get it even if it isn't available?  Again, though, I would not be very hopeful about what it would do for you for this symptom.

     

    However, before trying any new troubleshooting it would be helpful to clarify whether you did those regsvr32 commands under an elevated cmd window?   If not they will be even less effective.  Also, you can try different versions of those commands according to whether you are re-registering the 32-bit side or the 64-bit side...

     

    http://iefaq.info/index.php?action=artikel&cat=42&id=133&artlang=en

     

     

    If that doesn't help some claim that unchecking  IE as a feature, which evidently involves an extraordinary amount of machine grinding, can act as a repair.   YMMV.   FWIW I think it is a lost opportunity for doing that and just another example of minimally implementing a function rather than generalizing it usefully.   E.g. it probably does whatever is needed to allow another browser to be used and then get IE back later but nothing like rebuilding the OS browser functionality (such as would occur by reinstalling the OS, for example.)

     

     

    Then I think a less disruptive thing to try would be more of what you have already done, e.g. uninstall IE9 and go back to IE8 level support (i.e., W7 base level support).   Note though that getting that done probably would involve uninstalling the OS service pack.
