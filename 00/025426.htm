Saving last position of userform - Microsoft Community

ossiemac say:

Saving last position of userform

I was not able to find much on the subject of saving the last used position of a Userform so that the next time the Userform is shown, it will be in the same place. Therefore, I thought that it might be worth posting the solution that I came up with.

The method is to save the last position by using the "UserForm_QueryClose" event. The Left and Top of the Userform is saved to Defined Names.

The defined names can be manually created first or use the following Sub and UDF to first test for the existence of the Defined Names and if they do NOT exist, then create them and Show the Userform.

The user can then move the Userform to the desired location on the screen and provided the workbook is saved before closing the workbook, then the Userform position is saved ready for the next time it is Shown.

Feel free to comment and in particular, if you believe you have a better solution, then post it. Personally I don't like the solutions where the Registry has to be altered.

 The following code is associated with the code to Show the Userform. (Normally in a standard module):

Sub ShowUserform()
   
    '*****************************************************************************************
    'After the first time the code is run, IF THE WORKBOOK IS SAVED then
    'the code between the asterisk lines can be deleted
    'or commented out because no longer required.
   
    'If running from ActiveSheet then the following line is superfluous
    Worksheets("Sheet1").Activate
   
    'If the Defined name does not exist then create it with dummy values to initially position Userform.
    If Not NameExists("frmLeft") Then
        ActiveWorkbook.Names.Add Name:="frmLeft", RefersTo:="3"
    End If
  
    If Not NameExists("frmTop") Then
        ActiveWorkbook.Names.Add Name:="frmTop", RefersTo:="3"
    End If
    '******************************************************************************************
   
    UserForm1.Show
   
End Sub

Public Function NameExists(strDefName As String) As Boolean
    'This UDF is called from the Sub ShowUserform to determine if the Defined Names exist.
    'If the code to test for existance of Defined Names is disabled (see comment in Sub ShowUserform)
    'then this Function is no longer required.
    On Error Resume Next
    NameExists = Len(Names(strDefName).Name) <> 0
    On Error GoTo 0
End Function

The following code goes in the Userform's code module:

Private Sub UserForm_Initialize()
   
    'If StartUpPosition property set to Manual in Design mode
    'then the following line of code is superfluous.
    Me.StartUpPosition = 0
   
    'Following 2 lines removes the equals sign from the defined name value
    'and assigns the values to the Userform Left and Top properties
    Me.Left = Replace(ActiveWorkbook.Names("frmLeft"), "=", "")
    Me.Top = Replace(ActiveWorkbook.Names("frmTop"), "=", "")

   
End Sub

'See following link for more info on UserForm_QueryClose
'https://msdn.microsoft.com/en-us/library/office/gg278635.aspx?f=255&MSPPError=-2147217396

Private Sub UserForm_QueryClose(Cancel As Integer, CloseMode As Integer)
    'On Close Userform this code saves the last Userform position to Defined Names
    ActiveWorkbook.Names("frmLeft").Value = Me.Left
    ActiveWorkbook.Names("frmTop").Value = Me.Top
End Sub

How to do a file search

In Word and Excel 2003 when you were opening a document you could run a file search by entering a keyword and it would search all documents and pull up the ones containing that specific keyword. How do I do this in Word and Excel 2007?
terfar say:
File | Open (Crtrl+O) which opens a Windows Explorer window with the Documents folder expanded. In the top right corner, you will see a find box: type in your search string and press enter. You will be presented with a list of all the documents in that folder containing the string.
 
If it doesn't work, go to the Control Panel, Index applet and make sure that Indexing is set to include your documents folder.
 
You can also do this outside Word. Press START and type your string. The results will be displayed in the Start list but will include all files containing the string and not just Word files.

Murray,

If you decide to pursue this beyond the Desktop workaround that you are currently employing then I suggest these steps.  Note that if the first or second step fixes it then there is no need to proceed any further.

I appreciate that steps 1 & 2 are technical procedures that you might not normally use.  They can be done, however, even without knowing about the complexities of the permissions schemes involved.  Just read each stage and complete it before proceeding to the next.

1  Use the procedure given in the MS Knowledge Base article that Doug gave a link to above - icacls "%programfiles%\Microsoft Office 15" /grant *S-1-15-2-1:(OI)(CI)RX needs to be changed to

[for computers running 32 bit Windows 10]

icacls "%programfiles%\Microsoft Office" /grant *S-1-15-2-1:(OI)(CI)RX

or [for computers running 64 bit Windows 10]

icacls "%programfiles(x86)%\Microsoft Office" /grant *S-1-15-2-1:(OI)(CI)RX

or [for those rare cases of 64 bit MS Office 2007 in either type of Windows 10]

icacls "%programfiles%\Microsoft Office" /grant *S-1-15-2-1:(OI)(CI)RX

2  Disable OneDrive in Task manager, Startup tab and restart the computer for this change to take effect.  Then use the augmented permissions resetting procedure provided by another user, an MVP for Outlook, in  Fix read-only access to Office files. Whilst the procedure was written with Outlook in mind, it applies to write permissions for all the documents etc folders - start from its sub-heading Resetting all permissions for your user account.  One essential component of the procedure is in its para 6 - Replace all child object permissions entries with inheritable permission entries from this object because omission of that step fatally undermines the fix being attempted. 

3  If you subsequently use OneDrive [by, for example, opening an Office document from a OneDrive folder], check folder permissions again as some users have reported that the problem returned when doing so.  If this happens then you can try repeating steps 1 & 2 above or moving on to step 4 below.

4  If the fault persists, reappears after a system restart or reappears after re-enabling OneDrive use the further procedure created by the user Figment in  [BUG] Windows 10 OneDrive @ Default Settings Causing Major ...   

4.1  You will see references in that thread to using the "Default" ["Built-In"] Admin account if mistakes are made during fixing with the result that "permissions" are set so badly that further remedial action is inhibited.  That's fine but it is much safer to create two additional password-protected local Admin accounts beforehand instead and to make a "password reset disk" for each of them.  By avoiding use of the Built-In admin account as much as possible, the chances of this special & irreparable account suffering user profile corruption are minimised so that it ought to remain fully functional & ready for a day when there might be no alternatives left.  Having two additional password-protected local Admin accounts ready is not just a precaution for this read-only problem, it is a precaution that all users ought to take as soon as they have installed Windows.

4.2  The standard precaution of making an up to date system image beforehand also applies.  As does booting from your Recovery drive first just to make sure that it works and that it will therefore be available to enable restoring from that system image should the need arise.

4.3  Personally, I do not agree with using the tool Figment suggests [CCleaner] even just to clean up files.  Windows has a perfectly satisfactory & safe tool for doing that [in File explorer, right-click on a drive, select properties then Disk cleanup].  CCleaner can do this job but some say it is not safe even for this limited use;  moreover, it has a Registry cleaning mode that is definitely dangerous as it can seriously & irreversibly corrupt the Registry.
