ken sheridan say:

Aside from the other problems with you code, rather than converting the text en bloc in the control's AfterUpdate event procedure, do it as each character is entered.  Add the following function to a standard module:

Public Function ConvertToCaps(ByRef KeyAscii As Integer) As Integer
' Converts text typed into control to upper case

   Dim strCharacter As String

   ' Convert ANSI value to character string.
   strCharacter = Chr(KeyAscii)
   ' Convert character to upper case, then to ANSI value.
   KeyAscii = Asc(UCase(strCharacter))

End Function

and call it in the control's KeyPress event procedure with:

    ConvertToCaps KeyAscii

NB:  in this case you cannot call the function directly from the properties sheet as the KeyPress event property as you might expect.  It must be called from within the control's event procedure.

yang zheng msft support say: Hi Charlie,

Sorry for the delay.
In your screenshot of the transport rule, you have set the condition to ‘apply the rule if the recipient is located outside the organization’. That means the transport rule takes effect when emails sent out from your Office 365 email accounts to email addresses outside your Office 365 tenant instead of receiving from email addresses outside your Office 365 tenant.
