Merging Word docs - Microsoft Community

katwms say:

Merging Word docs

I have a two questions: 1) is it possible to combine 2 or more documents as a single letter, having them both use the same source document but merging different data for each?  2) how do I merge multiple items into one letter? (I'm familiar with 'NextRecord' and I don't believe it works in this application.)

Here is what I have: one source document listing several individuals (with multiple additional fields by individual) who teach different subjects.  I need to have the first document list all of the courses one individual teaches onto one page, then the following documents would be one sheet per class that that particular individual teaches, merging different fields from the same source document -- then move on to the next individual and all the courses they teach, and so on.  Thank you for any help! 

macropod say:

Provided both documents use the same data source, that is quite easy. Presumably, the letters would have conditions that are needed to determine which versions to insert. That can be done via simple IF field encoding, with the entire variable content within the IF field or with the variable content being pulled in from external documents via INCLUDETEXT fields. In either case, the variable content can include more mergefields, if applicable.

What you seem to be describing for your second question is a merge with multiple, grouped records per letter. For that, you can use Word's Catalogue/Directory Mailmerge facility for this (the terminology depends on the Word version). To see how to do so with any mailmerge data source supported by Word, check out my Microsoft Word Catalogue/Directory Mailmerge Tutorial at:
http://windowssecrets.com/forums/showthread.php/154370-Microsoft-Word-Catalogue-Directory-Mailmerge-Tutorial
or:
http://www.gmayor.com/Zips/Catalogue%20Mailmerge.zip
The tutorial covers everything from list creation to the insertion & calculation of values in multi-record tables in letters. Do read the tutorial before trying to use the mailmerge document included with it.
 
The field coding for what you want to do is complex. However, since the tutorial document includes working field codes for all of its examples, most of the hard work has already been done for you - you should be able to do little more than copy/paste the relevant field codes into your own mailmerge main document, substitute/insert  your own field names and adjust the formatting to get the results you desire. For some worked examples, see the attachments to the posts at:
http://www.msofficeforums.com/mail-merge/9180-mail-merge-duplicate-names-but-different-dollar.html#post23345
http://www.msofficeforums.com/mail-merge/11436-access-word-creating-list-multiple-records.html#post30327
http://windowssecrets.com/forums/showthread.php/157725-Word-2010-Merge-from-excel-into-Table-Directory?p=928391&viewfull=1#post928391
 
Another option would be to use a DATABASE field in a normal ‘letter’ mailmerge main document. A DATABASE field even allows you to use a different datasource than the mailmerge main document for the table it outputs and can also be used without recourse to a mailmerge. An example of non-merge usage can be found at: http://www.msofficeforums.com/mail-merge/21847-mail-merge-into-different-coloumns.html#post67097
 
Alternatively, you may want to try one of the Many-to-One Mail Merge add-ins, from:
Graham Mayor at http://www.gmayor.com/ManyToOne.htm ; or
Doug Robbins at http://bit.ly/1hduSCB
 
In addition to a 'Many to One' merge, the latter handles:
• Merge with Charts
• Duplex Merge
• Merge with FormFields
• Merge with Attachments
• Merge to Individual Documents
• Merge, Print and Staple

The setting for Line Spacing can be set by using the Format> Paragraph - Indents & Spacing dialog as indicated in the 2nd screen shot from Virgil Wu.
