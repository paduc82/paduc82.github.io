If you are using Avast's email scanner, I wouldn't be too quick to eliminate it. Email scanners are, in essence, a pair of liars on either side of a mail clerk. When you send an email, you're talking to the first liar who represents himself to be your ISP's server. As far as Outlook is concerned, he IS the ISP server, Outlook sends the message, the liar accepts, Outlook puts the message into Sent items, and goes on about its business. The first liar then hands the message to the mail clerk (the actual scanner), who removes it from the envelope, examines it, removes anything objectionable, puts it into another envelope, writes the recipient address on it, then hands it to the second liar who represents himself as your email client (Outlook in this case). The second liar sends the message to your ISP's server, trusting that the new envelope is addressed correctly. If these three make a mistake, the email doesn't get to its recipient, but Outlook doesn't and cannot know it because it was lied to.  You simply do NOT need an email scanner...the anti-virus application, yes, but not the email scanning component. With this in mind, uninstall Avast, then re-install it without the email scanner (the Disable setting for email scanners don;t always work).

oren rosen say: Hi,

Problem is with the DDE. apparently, Microsoft Excel is relaying on it. Every time run start Excel, Explorer sends a broadcast to ALL running processes (even those that are suspended) and waiting for reply. Maximum time out is 120 seconds so eventually the excel will open.

A workaround for this is to disable the Excel using the DDE. For this you'll need to make modification's in the registry. (make sure to make backups before making any changes in the registry).

Do the following:

1. In the registry, go to key : HKEY_CLASSES_ROOT\Excel.Sheet.12\shell\Open\Command
2. In (Default) Change the data at the end of the line from /dde /n to "%1"
3. Delete the command value.
4. Delete the key HKEY_CLASSES_ROOT\Excel.Sheet.12\shell\Open\ddeexec

that's it. no need to logoff or restart.

Again, make sure to make backups before making any changes in the registry.

Thanks,
Oren
