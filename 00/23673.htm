a. user say:

> Can part of a footer be "linked to previous" while the other part is not?


No, Word does not allow that.

Continuous page numbering does not rely on having linked footers. It only relies on the fact that each section's page numbering is set to be "Continue from previous section."

So in theory, you can achieve what you need by keeping every footer unlinked and ensuring that "continue from previous section" is checked for every section after section 1.

Assuming you are in a position to use VBA, you can ensure that is the case using code like this:

Sub continuousFooterNumbering()
Dim hf As Word.HeaderFooter
Dim i As Integer
With ActiveDocument
  For i = 2 To .Sections.Count
    .Sections(i).Footers(wdHeaderFooterPrimary).PageNumbers.RestartNumberingAtSection _

   = False
  Next
End With
End Sub

This assumes that you are only using the primary footer throughout - no special first page footers or even/odd type footers. Also, although the RestartNumberingAtSection looks as if it is applied independently to a particular header/footer type, it actually applies to all { PAGE } fields in the section.

IN practice, because Word basically does continuous numbering and linked sections by default, and because it is difficult to manage 136 separate headers/footers by hand, I think you might be better off using a single footer, linked throughout, with continuous page numbering.


How to do that?

Well, it requires that you insert your unique policy number for each section using a bit of field coding. A fairly simple way to do that would be as follows:
 a. at the end of your document (so that it is out of the way), insert a { SET } field for each section. Let's say your unique policy numbers are "policy one", "policy two", "policy three" and so on.

Then you can use

{ SET sect1 "policy one" }{ SET sect2 "policy two" }{ SET sect3 "policy three" } and so on.


In all cases, the {} are the special field code brace pairs that you can insert using ctrl-F9 on Windows Word.

Select them all and press F9 to update them.

In your section 1 footer, you use the nested field code

{ REF "sect{ SECTION }" }
or 

{ "sect{ SECTION }" }

So in section 1, you get a reference to bookmark "sect1", i.e. "policy one", and so on.
As you add sections, I think Word will probably link the footers and do continuous numbering by default anyway. But if it does not, you can use the following macro to do it

Sub continueAndLinkFooters()
Dim hf As Word.HeaderFooter
Dim i As Integer
With ActiveDocument
  For i = 2 To .Sections.Count
    With .Sections(i).Footers(wdHeaderFooterPrimary)
      .LinkToPrevious = True
      .PageNumbers.RestartNumberingAtSection = False
    End With
  Next
End With
End Sub

That means you basically only have *one* footer. If you lose it, re-insert the footer for section one and if necessary, re-run the macro.

FInally, suppose you want to insert a new policy early in the sequence, say, before section 4. That would require that you change all the SET fields from SET sect4 onwards (e.g. change 4 to 5, 5 to 6 etc., then update the SET fields again. A bit tedious. But if you do your SET fields like this instead


{ SEQ s \r0 \h }{ SET "sect{ SEQ s }" "policy one" }{ SET "sect{ SEQ s }" "policy two" }{ SET "sect{ SEQ s }" "policy three" } and so on

you can insert/delete a complete { SET } field at the appropriate point and update all the SET fields to achieve the required result without a lot of manual effort.


Finally, if you are familiar with VBA, you can replace all those in-document SET fields by creating Word Variables instead, e.g. called sect1, sect2 etc., and modify the footer field construction to 

{ DOCVARIABLE "sect{ SECTION }" }

(In this case, I would maintain a VBA Sub something like this:

Sub setPolicyNumberVariables()

Const varstem as String = "sect"

Dim i as Integer

i = 0

With Activedocument
  i = i +1

  .Variables(varstem & cstr(i)).Value ="policy one"

  i = i +1

  .Variables(varstem & cstr(i)).Value ="policy two"

  i = i +1

  .Variables(varstem & cstr(i)).Value ="policy three"

' and so on

End With

End Sub
