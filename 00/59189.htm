XP cannot recognise Win7 Homegroups so the Win7 machines need to be networked using user accounts and passwords.


All of the computers should be in the same workgroup and same subnet (ie. 192.168.1.x OR 192.168.2.x)


You didn't as much say but it seems as if you might be using 2 routers, or a router and some other device that is also using DHCP to assign a different range of IP numbers on your LAN. There should only be one DHCP server on the network. If you are using 2 routers, rewire the connections between them such that one router is the sole DHCP server and providing all of the routing services while the second router is acting simply as a switch with no routing services.


The network type for the Win7 computers should be set to Home or Work. Choosing Work avoids having to deal with getting rid of the Homegroup defaults.


Make sure network discovery and file & printer sharing are enabled for Home or Work Networks on the Win7 computers


As for accessing shared files etc. , the XP computers will need to log on to any of the Win7 computers with the username and password of a user on that Win7 computer. The commonly recommended workaround is to create a single common username with the same password on every computer in the network. (If you find a better way I'd love to know what it is myself) 


Encryption level shouldn't affect connectivity (highest is best) as long as it is the same for all machines.


The Win7 firewalls might be defaulted to block incoming Pings, or it could be your router configuration. Anything coming from a 192.168.1.x IP could be regarded as an external network connection by the 192.168.2.x router and incoming Pings might be blocked because of that.
