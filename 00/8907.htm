Is an interface built on square tiles really the most effective or - Microsoft Community

honest abe say:

Is an interface built on square tiles really the most effective or appealing interface?

All new interfaces need a chance to gain acceptance and prove themselves.  Nevertheless, after acknowledging there are some great benefits to the new tiled interface, we really question whether it's an overall improvement or not as it relates to a desktop or laptop computer.

We feel the previous Windows interface was more visually distinguishable and allowed the user to more effectively identify programs or areas of interest.  Some of these prior attributes could even be used to improve, not replace, the new tiled interface for mobile devices.  Furthermore, the old interface seemed to provide more flexibility for customizing screen elements to the liking of the user.

For example, the previous desktop and application ribbon icons had a little more color, detail, and their outlines were better defined, yet they were not too bold or overwhelming.  For us that translated into identifying and finding things quicker.  The same goes for window frames and other screen elements that have become almost invisible.  The new interface is often so visually suppressed (less color, less definition) that much of it just flows together and becomes indistinguishable.


We understand the importance of balancing beauty with efficiency, legibility, and utility.  But what happened to the options users had become accustomed to, for customizing screen elements, when we moved to the 'Tiled' interface.  Where are the options for tiles with rounded corners, tiles with 3-D bevels, tile shadows, tile color variations, tile transparency, font changes, and shape or size modifications?

In some respects, we feel the mediocre acceptance of Windows 8 was, in part, due to the rigid, somewhat unappealing, Kindergarten-like building block interface.  While the tiled look can obviously simplify the interface for mobile devices, it also robs the desktop user of other options that can help visually set apart and organize specific applications, icon groupings, folders, gadgets, utilities, taskbars positions, launch menus, docking positions and window configurations.

In general, the traditional desktop allowed users to visually organize and beautiful their own interface according to their workplace requirements.  In some ways, the tiled interface was a step back in time to the primitive, rigid interfaces of earlier computers.  In short, even the traditional desktop elements Microsoft has preserved, have occasionally been overly simplified and de-emphasized.  However, with time, we are warming up to some of the advantages of simplicity and uniformity.

Nevertheless, in our opinion, there is a definite place for both interfaces.  We do understand the need for a streamlined interface to implement on mobile devices, but, at very least, couldn't we allow the user to modify some of the basic visual elements of their tiles?  We also see the need for a traditional desktop interface because of its incredible flexibility and diversity.  And shouldn't we also keep building on some of the strengths of the traditional desktop?

cody c. say: Hey RevJSS,

I'm sorry to hear you're having a difficult time installing the 14.2.4 update. I split your post off into it's own thread so that we can better focus on your specific issue.

Without knowing which solutions you've tried, I'm afraid I might be recommending solutions you've already attempted. Sorry in advance if that happens.

The first issue (and most common) stems from re-installing on top of existing installations. In general, don’t do this. Re-installing on top of existing installations can re-introduce files that should not be present. If you need to re-install, please be sure to remove Office 2011 as described on the following links:

 

Video | Uninstall Office 2011

Step-by-Step Instructions | How to completely remove Office for Mac 2011

 

One workaround for the 14.2.4 update that has helped some users is to temporarily move or rename the /Library/Automator folder. You can do this by following the steps below:

 

  1. With Finder open, choose Go > Go to Folder...

  2. Type in the following path and then click Go:
    /Library

  3. Find the Automator folder, select it, and then click the name of the folder again to Rename the folder. Name the folder Automator_Old

    Note    :     OS X may prompt you for your password. This is normal.

  4. Attempt to install the 14.2.4 update again.

  5. If the update installs correctly, rename the Automator_Old folder back to its original name: Automator

 

If that solution does not work and you know you have re-installed Office 2011 on top of an existing Office 2011 installation, you’ll need to remove Office 2011 properly and re-install. 

 

The second issue was some users changed the size of the /tmp/ directory on their machine which was too small for the update to fully install. Re-sizing the /tmp/ directory resolved the install issue for these users.
