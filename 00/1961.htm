Remove License For Mailbox on Litigation Hold - Microsoft Community

mattmcnabb say:

Remove License For Mailbox on Litigation Hold

We currently employ Litigation Hold for all of our mailboxes for compliance and so that data is retained when users leave the organization. I know that when a user is deleted and their mailbox is on Litigation Hold, the mailbox will be converted to Inactive.

However, we have some users on Litigation Hold who will not be deleted, but their mailbox license will be removed. Should these mailboxes also become inactive? I have removed the license for them, but so far I've seen no changes, and in fact it appears the mailbox is still valid and available.

Can you please advise on how I should remove mailbox access for these users while retaining the data. Thanks!

mattmcnabb say:

Ok, based on this conversation, working with MS support in a service request, and some of my own testing, I have come up with this process to decommision mailboxes for active users.

- make sure the mailbox is set to litigation hold

- remove the Exchange License and the Exchange archiving license if necessary

- The mailbox will no longer be accessible by the user, but won't be converted to inactive

- Add the mailbox to a distributiongroup "DisabledMailboxes"

- Create a transport rule that blocks sending and receiving from members of this group and sends a bounceback message

- If in the future the user is deleted, because litigation hold is still enabled the mailbox converts to inactive

This took a lot of discussion and testing, but this is acceptable for our needs. Ultimately I'd like it if the mailbox converted to inactive when the license was removed rather than when the user is deleted, but there may be an underlying reason why this won't work.
