ossiemac say:

This is a common occurrence due to the binary notation used by computers. See the following link for more information.

https://support.microsoft.com/en-my/help/78113/floating-point-arithmetic-may-give-inaccurate-results-in-excel

joeu2004 say:

Write =ROUND(SUM(A1:A3), 8)

In general, when you expect a calculation to be accurate to n decimal places, use ROUND to explicitly round to

Do not round to an arbitrary number of decimal places, as some people suggest. That does not always work.

-----

As for "why?"....  Excel uses 64-bit binary floating-point to represent numeric values internally. Most decimal fractions cannot be represented exactly in that binary form.  Moreover, the binary approximation of a particular decimal fraction varies depending on the magnitude of the integer part of the numeric value.

That often causes infinitesimal arithmetic differences like 3.79E-17. And it causes other anomalies like:

IF(10.12-10=0.12,TRUE) returns FALSE(!) because 10.12-10 is visibly different from 0.12 if we format to display 15 significant digits.
