gary's student say:

Put the following event macro in the worksheet code area:

 

Private Sub Worksheet_Calculate()
Dim s As String
s = "sheet 1"
If Range("L3").Value <> "" Then
    s = Range("L3").Value
End If
ActiveSheet.Name = s
End Sub

 


Because it is worksheet code, it is very easy to install and automatic to use:

   1. right-click the tab name near the bottom of the Excel window
   2. select View Code - this brings up a VBE window
   3. paste the stuff in and close the VBE window

If you have any concerns, first try it on a trial worksheet.

If you save the workbook, the macro will be saved with it.


To remove the macro:

   1. bring up the VBE windows as above
   2. clear the code out
   3. close the VBE window

ken sheridan say: You will also need to Requery the list box in code after inserting a new row into the table, to include the new row in the list.  It will only be included of course if the new row satisfies the parameter values currently in the two unbound date controls and the text box.

The code should be in the AfterInsert event procedure of the form in which the new agency is entered, and would be like this:

Forms!frmMain!YourListBoxName.Requery

An alternative approach, and the one I'd use in this context, would be to open the second form in dialogue mode like this:

DoCmd.OpenForm "YourFormName",DataMode:=acFormAdd,WindowMode:=acDialog
Me.YourListBoxName.Requery

When a form is opened in dialogue mode, code execution pauses until the form is closed or hidden.  Consequently the list box will only be requeried when the other form is closed.  There is therefore no need to do so in its AfterInsert event procedure.

Partial Text to Column in Access 2010

I'd like to create a new column that replicates all text (numbers and letters) to the left of the third underscore in a preexisting column.  Any suggestions on the correct SQL or query design to accomplish this?

Existing column data format:

BAND_FJUMO1T06_03_1

PINN_PIPO_06_11

Want to create a second data column that looks like this:

BAND_FJUMO1T06_03

PINN_PIPO_06

john w. vinson say:

You can use a couple of builtin string functions to do this in a query (this derived data should NOT be stored in any table:

AllButLast: Left([existingcolumn], InStrRev([existingcolumn], "_") - 1)

InStrRev - "in string reverse" - starts at the end of the string and finds the position of the last instance of its second argument (here a literal underscore) in the first argument.
