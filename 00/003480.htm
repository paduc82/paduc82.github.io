cbarnhorst say:

Insider previews do not have their own special licenses.  With Windows 10 the device is licensed, not the software.  The procedure is to purchase a Windows 10 license and enter it just like any copy of Windows 10.  The licensing software will then connect to Microsoft's licensing servers and register the device.  Once the device is properly licensed, the device activates the software automatically. 

Instead of recording the product key in software, the unique hardware IDs of the device are registered with MSFT along with the licensing data.  Device registration is then permanent so you can reinstall Windows 10 as often as you like without need of a product key, even on a virgin HDD or SSD.

scott mcdaniel say:

This is definitely an Access error message.

Often this means you have a hanging lock file. Access creates a lock file whenever anyone opens the database, and if others cannot access that lockfile they can receive this message. To see if you have a lockfile issue, get everyone out of the database and then examine the folder hosting your backend. Do you see any files that are named the same as your Access db, but with a .ldb or .accldb extension? If so, delete that file and try again. Access should automatically delete that file when the last person exits, but sometimes fails to do that.

You could also have corruption in your database. Have you done a Compact on the backend lately? If not, make a backup first and then C&R the database.

Do you have the file location set as a Trusted Location in Office? You can do that in Access under the Trust Center settings.

Finally, do you have this system properly setup for multiuser access? The proper way to set this up is to have a single Backend database stored in a folder where all users have been give Read/Write/Create/Destroy permissions on the folder. Each user must have their own copy of the FrontEnd database (the user interface) installed directly on their machine, and each of those copies link back to the same backend. If you're trying to allow everyone to share the same FE from a network folder - that is, you've got a single FE and everyone just has a shortcut to that FE - then you're just asking for troubles, especially with newer versions of Windows.

Also, be sure that all machines are fully up to date with Windows and Office installs.

A couple of points to add.
