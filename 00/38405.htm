ajay k say:

Hi

 

Welcome to Microsoft answers!

 

This issue may occur,

a)    The file type is supported by the Player, but the file was compressed by using a codec that is not supported by the Player.

b)    The file might have been encoded by using a codec that is not supported by Windows Media Player. If this is the case, Windows Media Player cannot play the file. 

 

I would suggest you to try the following steps:

 

Step 1: Firstly, Check if Windows Media Player streaming video settings is enabled. To do this, follow the steps below:

a)    Open Windows Media Player and click on Organize.

b)    Go to options and select Network tab.

c)    Under Network tab select all the check boxes and try.

 

Step 2: To determine what codec was used with a specific file, play the file in the Player, if possible. While the file is playing, right-click the file in the library, and then click Properties, On the File tab, look at the Audio codec and Video codec sections.

 


Step 3: Download and install the Codecs:

 

Do the following in Windows Media Player:

a)    Click the arrow below the Now Playing tab, and then click More Options.

b)    On the Player tab, select the Download codecs automatically check box.

http://windows.microsoft.com/en-US/windows-vista/Codecs-frequently-asked-questions

(See the section Does the Player always try to download codecs automatically?)

Step 4:  when you open Windows Media Player beside, Organize option you will find Stream menu bar. In Stream menu bar, please apply the settings you would like to stream and play video under streaming options.

If this does not help associate the video file type which you are trying to play with windows media player in Default Programs.

You can make the program a default by using Set Associations.

a)    Open Default Programs by clicking the Start button, and then clicking Default Programs.

b)    Click Associate a file type or protocol with a program.

c)    Click the file type or protocol that you want the program to act as the default for.

d)    Click Change program.

e)    Click the program that you want to use as the default for the file type you selected, or click the arrow next to Other Programs to show additional programs. (If you don't see Other Programs, or your program is not listed, click Browse to find the program you want to use, and then click Open. If no other programs are installed that are able to open the file type or protocol, your choices will be limited.)

f)     Click OK.

 

 

Step 5: Configure protocol settings

 To change protocol settings for the Player

a)    Click the arrow below the Now Playing tab, click More Options, and then click the Network tab.

b)    Make the changes you want.

c)    If you clear the TCP check box, or if your firewall blocks TCP streams and the Player are configured to use the RTSP protocol, the Player will try to receive the streams by using the UDP protocol. If you also clear the UDP check box, or if your firewall blocks streams that use these protocols, and the Player is configured to use the RTSP protocol, the Player will try to receive the streams by using the HTTP protocol.

http://windows.microsoft.com/en-US/windows-vista/Which-protocols-does-Windows-Media-Player-use-for-streaming

 

Now check if the issue is resolved.

 

Please see the link below for your reference:

http://support.microsoft.com/kb/911710
