Sharepoint 2010 how to export item URL to Excel? - Microsoft Community

freshgrad say:

Sharepoint 2010 how to export item URL to Excel?

Hi,

 

I have a list of 10 items in SharePoint 2010 and I export them to Excel.  How can I export item's URL so that when clicking the link on Excel spreadsheet, it can bring me back to the item in SharePoint.

 

Thanks.

teylyn say:
Hello,

the export to Excel does not include a hyperlink to the item. But you could use a formula in Excel in an additional column to create a hyperlink to the original list item. 

First, make sure that the view that you export to Excel includes the column for the item ID. If you put this into position 1, then it will be rendered in column A in Excel.

Export the list to Excel. The item ID is now in column A.

In SharePoint, go to the list, right-click any list item's title link and copy the shortcut. It will look something like this:

http://SharePointSite/_layouts/listform.aspx?PageType=4&ListId={C2721BF2-29D3-40DD-B73E-B1FBDBED5045}&ID=1&ContentTypeID=0x01002290BD1E182003479BE681C394F4E5B4

Paste that into a text editor and copy the part in curly brackets after ListId. The gobbledy gook between the curly brackets is the List GUID.

Switch to Excel and add a column to the exported table with the following formula, starting in row 2 (row 1 has the column labels)

=HYPERLINK("http://SharePointSite/_layouts/listform.aspx?PageType=4&ListId={000000000-0000-0000-0000-0000000000000}&ID="&A2,"click here")

Replace "SharePointSite" with your site name and replace the 000000000-0000-0000-0000-0000000000000 with the List GUID you copied from the link. Copy the formula down.

Now you have links to each individual list item in the Excel file. 

No VBA required. 

cheers, teylyn

teylyn say:
Good approach. 

With regards to the cell showing the formula text instead of the formula result: Make sure the cell format is set to "General", not "Text". Then enter the formula. If you have formulas already written, change the format to "General", edit the formula and confirm. If a manual edit is not practicable, because of too many cells, select the cells that are now formatted as "General", do a find and replace and replace the = sign with a = sign. That will force a recalc of the cells. :-))

Hello,

there are two possible ways to handle this.

1. Create a custom list with the values in the sort order you want. To do that, go File > Options > Advanced, scroll down to almost the end and click Edit Custom Lists. scott .. b say:

Reason it "goes back to IE 10 settings" is that the f12 Developer Tools only works on one page...........the one that is open when you select f12.  When you move to a different page, it displays whatever the developer has set for THAT page (presumably IE10).  There is no way to make the IE9 emulation that you get from Developer Tools into a default setting so that it will work on EVERY page. 

 

Bottom line......there is no IE9 emulation and, as you discovered (after many hours of reloading your programs) REFRESH wasn't the answer.  Suggest you describe the problems you are having with IE10 so we can help you work through them.
