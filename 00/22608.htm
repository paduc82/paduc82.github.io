word file is protected but can still be edited - Microsoft Community

linda o'brien say:

word file is protected but can still be edited

I saved a document and made it "final" but it still can be edited.  When I email it as an attachment the receive can edit my letter.  How do I change this so that the letter cannot be edited?
jay freedman say:
Marking a document as final is really just a hint, as the information box says, "to discourage editing". If you want to prevent editing, you need to take steps to password-protect the document.

In the Save As dialog, click the Tools button at the bottom and choose General Options. Supply passwords to open and to edit the document and click OK (they can be the same or different). You'll then be prompted to enter the passwords again, to make sure you got them correct.

With this method, the password for opening is used to encrypt the file on the disk; it can't be opened without the password. If the password to open is correct, the user will then be prompted for the password to edit, or the document can be opened "read-only". This doesn't prevent making changes, but the document can't be saved to the same name.

It's also possible to use the Protect Document pane to mark a document as not editable. However, in this case the document isn't encrypted, and a user could theoretically remove the password protection.

I could write 10 more paragraphs on that issue but it wouldn't help your situation, soooo...

I can't say that it is the cause but your very first point is most likely the main contributor to the crashes:-)

Unlike many, I believe Track Changes to be a useful, reliable & stable feature -- every keystroke (errant as well as intended) , the correction of same, formatting changes, repositioning & sizing of objects, anything that may cause pagination to change frequently [like cut & paste], etc. it is inevitable that a document - more sooner than later -- gets tangled up in its own underwear. Corruption is also endemic to Tables, especially if they are lengthy, complex and/or nested. Tracking changes exponentially increases the likelihood of misbehavior frequently followed by a complete collapse of the document.

I'm not suggesting that you not use Track Changes, but I'd respectfully suggest that you re-evaluate heavily & a history of the variants needs to be maintained, consider saving different versions of the document. The Word> Preferences> Save can be quite useful for that purpose.

To address current issues, the first point is to save using the current .docx format if you aren't already doing so. It's far more efficient & stable than the former .doc format.

Next, have a look at this web page for some methods for clearing up corruption in existing documents. Make a copy, turn Track Changes off, Accept/Reject all changes, then follow Procedure #2. Work with that copy for a while & see if conditions improve.

It may also help to follow the guide here to clear up any font issues that may be contributing to the situation: Font Weeding.
