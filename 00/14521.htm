The issue is, once the ‘Add’ permission is granted for the student group, they then effectively have permission to be able to create/add content for any app on the page – I don’t want this to be the case, as otherwise students would be able to create new homework tasks and upload documents to the other feature document library.

ericyang_msft6286 say:

Hi James,

A feasible method you may consider is, breaking the inheritance of the specific libraries or lists to which you don’t want the students to have Add permissions and then set up unique permissions for them, while keeping the inheritance of other libraries and lists to inherit the Add permission from the site. For example, if you don’t want the users to edit the homework task list, you can break the permission inheritance of the list and then remove the permissions from the list. To break permission inheritance and set up unique permissions for libraries and lists, you can refer to this article: Edit permissions for a list, library, or individual item.

ruler225 say:

Hi, I'm running windows 10 preview alongside windows 8.1 in a dual-boot configuration. Except, instead of using a separate partition, or using a VM, I recommend creating a vhd file, or a Virtual Hard Disk, and booting from there. This way, you can take full advantage of your hardware, and also do it safely on a single file. A VHD is the same file that is used by VMs, so if you screw up windows 10, at least you don't screw up the whole thing (That has never happened with me, hopefully nothing will happen :))

To create a VHD, you need to go to Disk Management:

-Right click the start button and select Disk Management
-Under the Action menu at the top of the window, select Create a vhd
-Select a location to store the vhd. Make sure you're happy with location because you won't be able to change it once you install windows 10 on it.
-Select the size of your vhd (maybe 60 GB or more) 
-Please leave everything else the way it is (a dynamic type will not work)
-After the vhd is created, leave everything the way it is and boot to a usb

You will see "install now" on the setup, but you're not ready yet. Press shift+f10 and you should see a command prompt. Type the following commands:
Diskpart


Then type: Select vdisk file="C:\whereeveryourfileis.vhd"


You should see something like it successfully selected the file. Type: Attach vdisk
You should see it successfully attached, so close command prompt and click install now.
Click custom install, and select the disk with the size you specified earlier for your virtual disk. (also it should say "unallocated" and don't worry if it says windows can't be installed on this drive just click next)

BE CAREFUL NOT TO SELECT THE WRONG DISK BY MISTAKE


Enjoy your dual-boot configuration!

Depending on what hyper-visor you are using, you can also use its existing VHD (Virtual Hard Disk).

I did this successfully with Hyper-V.

-Right click the start button and select Disk Management (assuming that you are using Windows 8)

-Note the driver letter that is assigned when the drive is mounted. Mine was 'L'.-From an Admin CMD run the following with no quotes:

"bcdboot L:\windows"

You will need to substitute the drive letter L: to the drive letter you noted before when the VHD was mounted.

When you are done with your testing/preview; you can then launch "msconfig" from RUN and delete the entry from the boot tab.

Sway Privacy Settings - Microsoft Community

jam784 say:

Sway Privacy Settings

Is there a way to make an embedded Sway viewable and leave full screen mode as an option, but not downloadable, editable or have the option to "open in another tab"? I want to embed them in my course, and having them either not viewable, or editable won't work.

jason (office engineer) say:

Only authors will be able to edit your Sway.  Everyone uses the same link to open it, but people not signed in as the author will only get to view the Sway and not get the editing UI.

You can block duplication by opening "Settings for this Sway" in the ... menu and unchecking the "Viewers can duplicate this Sway" box.

There is no way to remove the open in another tab option.

Jason

jam784 say:

Hi Jason,

jason (office engineer) say:

Sways are basically web pages.  Currently, there are only a few ways to restrict who can open them:

  1. Only the owner - by selecting "Just me" in the share dropdown
  2. Only people in your organization - if you are using an Office 365 account (like from a school or business) you can restrict the Sway to only people in that same organization (this is the default permission level for new Office 365 account Sways)

Otherwise, anyone who has the link will be able to open the Sway.  There is not currently a way to remove the Share button that you are referring to, but even if it was hidden, anyone who got the URL to the Sway (by looking at the contents of the web page, for instance) would be able to publicize it.

One thing you could consider doing, if you're using an Office 365 account, is password-protecting your Sway (from the Share dropdown under "More options").  Then people would need to have both the URL and the password to view it.  With that said, if someone publicized both the URL and password you'd be back in the same place, but at least it would be a little more secure.

We are considering other types of permission features for Sways in the future, but these are the only ones that exist right now.

Jason

jason (office engineer) say:

We've just shipped a new feature that changes one of the things I send in my previous reply.  There is now a way for Office 365 users to hide the Share button from viewers.  If you open a Sway as the author and open the Share dropdown, under More options, there is now a checkbox to turn this off.  Again, users will be able to copy the URL and share this manually, but at least turning off the button will make it harder to do.

Hope this helps!

Jason
