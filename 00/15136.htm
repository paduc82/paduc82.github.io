lwarneradm say:

These are tracking files. It lets Outlook know where to resume uploading data from the .ost file to Office 365.

You often see them in the case where you import a decent size .pst file. Outlook first copies the data from your local .pst source file to Outlook's .ost file. It then starts synchronizing the .ost data to Office 365 at a background rate. This can take quite a long time, and Outlook will probably be opened and closed multiple times while the process occurs, so in order not to lose its place, Outlook creates the inferences files.

That's why you can watch them disappear if Outlook is open for a bit - it catches up, and no longer has to track anything. When you close Outlook, it writes the file so it knows where to pick up next time.

I get similar messages when my phone (or any device) is connected, but not logged into a public WiFi hotspot that redirects http traffic to a sign on page. the email client cannot interpret the sign on page, as it has a different certificate, so presumes something is wrong with your email credentials.

this can occur long after you left the area of the hotspot in question, as a single failure can sometimes block further connections for security (your phone knows it isn't talking to the mail server, so stops trying instead of talking to the wrong page) until you see the message, accept it (or cancel), and it then tries again.

Unfortunately due to the diversity of public hotspots out there with randomly different sign on systems, the mail client cannot interpret all the ways it can fail, so you get a generic 'couldn't log in' error, rather than something more informative.

Windows 10 64 bit comes in handy when you need to address at least 4 GBs or more of RAM. Windows 10 32 bit can utilize up to 3.2 GBs of RAM. Because the memory address space is much larger for 64 bit Windows, that means, you need twice as much memory than 32 bit Windows to accomplish some of the same task, but you are able to do so much more, you can have more applications open, do things like run an Antivirus scan in the background without it affecting your system performance.

Windows 10 64 bit is more secure too, malicious code cannot easily infiltrate it, drivers are more reliable since they must be signed before they can work with 64 bit Windows 10.

As for compatibility, you will need 64 bit device drivers for any hardware devices you might have. Also, there is no 16 bit subsystem in Windows 10 64 bit, which means, your applications must be 32 bit only, not 16 bit installer or uninstallers. Also, if you decide to move to Windows 10 64 bit in the future, there is no upgrade path from 32 bit Windows, clean install only.

Windows 10 32 bit can run bit applications, but the 16 bit subsystem is disabled by default.

You can enable it by doing the following:

Click Start, type: 16 bit application support

How to replace the table/figure caption labels in an existing - Microsoft Community

shirley_lu say:

How to replace the table/figure caption labels in an existing document?

Hello there, 

I'm using Microsoft 2003 Word to work on a document in which tables/figures are captioned like "Table-1" or "Figure-1"
These labels are requested to be replaced by "Table 1." or "Figure 1."
My questions are:
1) Is there any convenient way to replace the caption labels for all tables/figures in the document?
2) Is there any way to automatically add a "." after the caption numbering?

 

 
suzanne s. barnhill say:

Since Word's Caption feature doesn't allow the production of captions such as "Table-1" or "Figure-1," it would appear that these captions were created manually, not using Insert | Reference | Caption. That being the case, it would be easy enough to replace them using the Replace function.

  1. Ctrl+H to open the Replace dialog.
  2. Click More to expand the dialog.
  3. Check the box for "Use wildcards."
  4. In the "Find what" box, type
    (Table-)([0-9]{1,})
  5. In the "Replace with" box, type
    Table \2.
  6. Click Replace All.
  7. Repeat substituting Figure- and Figure<space>.

If you want to replace these manual captions with "real" captions:

  1. Use Insert | Reference | Caption to insert a Table caption (type a period following it so it says "Table 1.").
  2. Select the caption you just inserted and copy it.
  3. Ctrl+H to open the Replace dialog.
  4. Click More to expand the dialog.
  5. Check the box for "Use wildcards."
  6. In the "Find what" box, type
    (Table-)([0-9]{1,})
  7. In the "Replace with" box, type
    ^c
  8. Click Replace All.
  9. All the replacement captions will say "Table 1.", but they just need to be updated; use Ctrl+A (Select All) followed by F9 (Update Fields) to update them.
