The leading characters may or may not be spaces and even Trim and Clean may not work. Sometimes they are other non printable characters. The code processes both columns A and B and sets the data to Text with 15 characters.

To install the code.

  1. First make a backup of your workbook in case you have problems and the code does not do as you require.
  2. Ensure that the worksheet to process is the selected worksheet.
  3. Hold Alt key and press F11 to open the VBA editor.
  4. Select menu item Insert -> Module.
  5. Copy the code below and paste into the VBA editor (In the large blank space)
  6. Position the cursor anywhere within the first sub and press F5
  7. Hold Alt key and press F11 to revert back to the worksheet.
  8. Check if the data has been processed as required.
  9. If you want to keep the code then use "Save as" to save the workbook as Macro enabled. Sub SetRangeToProcess()
    'To run the code just click anywhere in this sub and press F5
    Dim rngToProcess As Range
    With ActiveSheet
        Set rngToProcess = .Range(.Cells(2, "A"), .Cells(.Rows.Count, "A").End(xlUp))
        rngToProcess.NumberFormat = "@"  'Set the NumberFormat to text ready for the conversion
    End With
   
    Call RemoveNonNumeric(rngToProcess)
   
    With ActiveSheet
        Set rngToProcess = .Range(.Cells(2, "B"), .Cells(.Rows.Count, "B").End(xlUp))
        rngToProcess.NumberFormat = "@"  'Set the NumberFormat to text ready for the conversion
    End With
   
    Call RemoveNonNumeric(rngToProcess)
   
End Sub

Sub RemoveNonNumeric(rng As Range)

Dim rngCel As Range
Dim lngLgth As Long
Dim i As Long
Dim varNewValue As Variant

For Each rngCel In rng
    varNewValue = ""
    For i = 1 To Len(rngCel.Value)
        If Mid(rngCel.Value, i, 1) Like "[0-9]" Then
            varNewValue = varNewValue & Mid(rngCel.Value, i, 1)
        End If
    Next i
    varNewValue = Format(Val(varNewValue), "000000000000000")   'Text with 15 digits
    rngCel.Value = varNewValue
Next rngCel
   
End Sub

jeeped say:

Merge code are for all intents and purposes field codes. You can revert field codes to their values by first tapping Ctrl+A (aka Select All) then tapping Ctrl+Shift+F9.

However, unless this is a single document I wouldn't recommend that. Typically, merged documents are a number of documents with each individual document representing a row of merged data. If you use Ctrl+Shift+F9 on a merged document, it will strip the merge codes and leave the document as a single document with the first (?) row of data.
