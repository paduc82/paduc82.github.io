vincentwu_msft say:

Hi Adam,

I'd like to explain the process of picture synchronization from Exchange Online to SharePoint Online. It is triggered by the user itself when rendering his/her own picture. Other users cannot initiate the sync for a user. To trigger a sync, a user who has exchange online license and has a picture in EXO already must render his/her own large thumbnail.


The sync is using the SMTP claim from the current user's claims to initiate autodiscover and execute a serverside call to OWA attempting to get the picture using OAUTH delegating the user's credentials. The SharePoint system account does not have permissions to get the image from OWA. Once the picture is successfully retrieved, SharePoint stores it in the mysite host in a picture library called User Photos in a localized folder called Profile Pictures (name can change based on the language of the web).

Given the situation, I'd like to suggest posting the SharePoint Developer Forum for expert help. Thanks for your understanding.
