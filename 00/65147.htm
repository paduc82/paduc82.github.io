a. user say:

BlueScreenView is a way to get a quick look at BSOD information but as you can read in their documentation regarding the Caused By Driver field:

Caused By Driver: The driver that probably caused this crash. BlueScreenView tries to locate the right driver or module that caused the blue screen by looking inside the crash stack. However, be aware that the driver detection mechanism is not 100% accurate...

The key words there are "probably" and "tries".  So BSV might be right about the cause but it is often not right.  Sometimes it is right.

To zero in on the real cause you need to provide some more information and upload your crash dump files where someone with the Windows debugging tools can analyze them.

Here's how to get started:

The problem (errors 116 and 117) is most likely a video driver, a thermal (heat) issue or both.

Video intensive operations can contribute to thermal related BSODs. 

Can you reproduce the problem anytime you want...  say by playing games, watching videos, etc.? 

Answer the following questions, reply with answers, the pasted information (two things to paste) and links to crash dump files:

shareenbailey say:

I just bumped into this problem when we upgraded to windows 8.1 64 bit from XP.  It took about 15 different attempts, but I was finally able to get it to work on both the printer and the scanner.  Here's how:

1)  Download the 64 bit English North America drivers from the Cannon site to your desktop, and unzip them.  Note the contents - there are three sets of drivers, one for printer (MF3200AU.inf, MF3200AU.cab, and MF3200AU.cat), one for the FAX (CNXRPUA6.inf, CNXRPUA6.cat, and FAX.cab), and a third set for the scanner (MF21SCN.inf, MF21SCN.cat, and scan.cab).  When unzipped, they are all placed together in the same directory.  And what happens is that Windows 8 assumes that the one for the printer is also the one for the fax and the scanner (which is wrong), so it blindly takes the same driver for all of them.  Once you understand what is going on that makes it not work, the way to fix it becomes apparent. 

2) I separated the three sets of drivers into three subfolders (printer, scanner, and fax) so that I could easily tell them apart.  then I uninstalled all printers, faxes or scanner installations through device manager (be sure to also delete the drivers) before beginning.  I unplugged the USB to the printer, and then rebooted.

3)  When I had a clean re-boot, I started by going into control panel (with the printer USB still unplugged), and followed the process for a manual installation of the drivers, not allowing windows to detect the drivers at all.  Select 'add a printer', then 'the printer I want isn't listed', then 'add a local printer or network printer with manual settings', select 'USB001' as the connection, then 'next' then select Canon from the list, and select 'have disk'.  Navigate to the printer folder on your desktop, and point to the MF3200AU.inf file.  when you get the list of the two canon MF3200 printers, select the top one (not the fax).  Continue with the installation until you get to the print a test page.  do not print a test page or plug in the USB.  once the installation is complete, go and verify that in the control panel by looking at installed printers.  You should see it.  Reboot the computer and repeat the entire process of installing another printer, but this time select the Canon MF3200 fax, and point to the fax folder and the CNXRPUA6.inf driver.  Complete that installation, verify that it shows up in control panel installed printers.  Reboot again.

4)  That installs the two printers, but not the scanner.  This time, when things boot up, plug in the USB, and let it detect the scanner.  It should not find a driver for it.  It should show up in device manager under "other devices" with no driver associated.  When you see that, go ahead and replace the driver for that device, and this time point at the scanner folder and the MF21SCN.inf file.  Once you are done, unplug the USB again, and reboot a third time.  Now all three correct drivers should be correctly installed.

5)  Neither the toolbox or the pagemaker software that came with the printer will work with windows 8.1 so do not install that software.  Instead, pin MS Paint to your quickstart bar.  Once you have re-booted, plug in the USB, print a test page on the printer, and send a test fax if you want.  Use MS Paint to select the scanner source, and it should connect right away.  If it hangs, unplug and re-boot, and plug the USB in again after a clean re-boot.  But it should see and scan from the printer in MS Paint. 

sorry this was so long.  but it took be about 10 hours of head banging to figure out how to make all three functions work and avoid buying a new printer.  good luck.
