john mcghie say:
Hi Jason:

If you are talking about change MARKING, that will remain in the document until you "Accept all changes", regardless of whether change tracking is on or not.

A new feature in Word is forces tracked changes to be visible every time you open the document.  This is to prevent the problems caused by people who did not know how to use tracked changes.  They used to "hide" the changes, thinking they had "turned them off" and "removed" them.  They had not, they had simply hidden them.  Word would continue to collect, and mark, changes, the users just didn't know that was happening.

Until the edited documents would get sent to the customer, with all of the deleted text still in the document, and visible to the customer.

If that's what is happening, the only way to make these changes "go away" is to "Accept All Changes in Document" (or "Reject all...).  Word will continue to force tracked changes visible each time you open any document that contains them, so you know they are there; so you know the document contains information you may not want your boss or customer to see.

Hope this helps

query to show multiple-month and -quarter sales - Microsoft Community

poliasks say:

query to show multiple-month and -quarter sales

Assume my table has three fields: Customers, Months, MonthlySales.

 

Is there a fancy way to make a query from this table, where it shows customers as rows, and shows sales for each month and quarter in separate columns? (I know if I only need to show each month in separate columns, a good shot is to use crosstab query. But I don't know how to adding per quarter data at the same time using crosstab).

 

Currently, my solution is to:

 

Step 1: make a crosstab query to show columns by month;

step 2: make a 2nd query based on the crosstab query to show columns by quarter;

step 3: make a 3rd query to pull all months' and quarters' sales data from the crosstab and the 2nd queries.

 

This approach makes me feel somewhat dummy (But I am an Access beginner anyway, if not a dummy).

 

Appreciate your advice.

 

marshall barton say:

Union queries are used to combine two similar sets of records vertically without doing anything to tables.  An Append query copies one table's records to another table.

 

If you use a union query like:

 

SELECT salesperson, Format(saledate, "mmm") As Col, Sum(saleamount) As MonthTotal
FROM salestable
WHERE Year(saledate) = 2013

GROUP BY salesperson, Format(saledate, "mmm")
UNION ALL

SELECT salesperson, Format(saledate, "\Q\t\r q"), Sum(saleamount)
FROM salestable
WHERE Year(saledate) = 2013

GROUP BY salesperson, Format(saledate, "\Q\t\r q")

 

You can then create a crosstab query based on that union query with columns:

    Apr  Aug  Dec Feb  Jan  Jul  Jun  Mar  May  Oct  Qtr 1  Qtr 2  Qtr 3  Qtr 4  Nov  Sep

with the salesperson as row heading and the total sales for each sales person in the month/quarter - person cells.

 

The reason the columns are in alpha order is because that's the default.  The order of the columns can be changed by using the ColumnHeadings property as long as you know the column names when you create the union query.

marshall barton say:

Since the columns in a crosstab are data values (month or quarter), there is no way one crosstab query can do both.  Your idea of using separate queries for the two different types of data is a good way to go.

 

The third query that combines the data from the other can then arrange the combined field list in whatever order you want.  The On clause that joins the two queries might be a little tricky and you will probably have to make the months relative to a start month parameter so the field names don't change when you want a different set of months.  How you would do that usually depends on how you are going to view the results of the third query.  A form or report makes it relatively easy, but the query's datasheet makes it near impossible.
