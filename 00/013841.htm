jllatham say:

There is almost no way around the changing of the formulas when you insert/delete rows on the referenced sheet.  Excel is always going to adjust the formulas.  I don't necessarily think that's right, but that's the way it is.  But I said 'almost no way'...

We can use INDIRECT() and a little math.  INDIRECT() takes a text string and treats it as if it were an address.  Let's say that your first formula, ='Tracking List'!$B$2  goes into a cell on row 2 of the other sheet.

Enter this formula:

=INDIRECT("'Tracking List'!A" & 1+ROW()-1)

and now drag that down the sheet.

For this discussion, let us say you are going to put these formulas into column X.  But it could be any column.  I'm using X to clearly distinguish cells on this 'other' sheet and the ones on your Tracking List sheet.

Things to note:  First, notice that Tracking List is enclosed in single-quote marks, so the formula is starting out like:  =INDIRECT(  "  '  Tracking List'    but without the spaces I've just shown.

Second:  the 1+ROW()-1  is calculating a row number.  If we put that first formula into X2 on the 'other' sheet, then it evaluates to  1+2-1 = 2.  We need the math like that in case you fill it up to row 1.

If we wanted the formula in X2 to pick up the value from 'Tracking List'!A1  we would change the math portion to become:

=INDIRECT("'Tracking List'!A" & ROW()-1)

ROW() is going to return the row that the formula is in, so it evaluates to 2-1 = 1  and gets us the information from Tracking List!A1.  You just have to play with that math to get the first formula right and then fill down.

If the formula you entered in the other sheet to get 'Tracking List'A1 was put into row 9 on this other sheet, then you would start with:
=INDIRECT("'Tracking List'!A" & ROW()-8)

since ROW() = 9, then 9-8 = 1 is the result of the evaluation.

If you change the name of sheet 'Tracking List', you have to change it in these formulas: you could either edit one and fill-again, or select them all and use Edit --> Replace (with "look in formulas") to change them all to the new sheet name.

jllatham say:

DO NOT READ THIS unless there is a very strong chance that your users may change the name of the 'Tracking List' worksheet!  It will only confuse you and I don't want to do that.  If there is little or no chance of them changing that worksheet's name, go with what I gave you above, because this all leads to a very ugly formula.

But if there is a strong chance of someone changing the name of 'Tracking List' then this would get your INDIRECT() formulas to automatically change the name of the sheet when they do that and continue working without having to edit the formulas.

The CELL() function can be used to return several different pieces of information about a cell on a worksheet.  By using the "address" parameter, we can get it to tell us what sheet a cell is on, but we have to dig into the returned information to get it.  Also, you want to reference a cell on the original sheet that won't be deleted (as in delete a row or column where the cell is that you're referencing)  So I've picked 'Tracking List'!A10 as that cell for these examples:

First, the entire ugly formula - followed by a breakdown of its parts:

=INDIRECT("'" & IFERROR(MID(CELL("address",'Tracking List'!A$10),FIND("]",CELL("address",'Tracking List'!A$10),1)+1,FIND("'",CELL("address",'Tracking List'!A$10),2)-FIND("]",CELL("address",'Tracking List'!A$10),1)-1),MID(CELL("address",'Tracking List'!A$10),FIND("]",CELL("address",'Tracking List'!A$10),1)+1,FIND("!",CELL("address",'Tracking List'!A$10),2)-FIND("]",CELL("address",'Tracking List'!A$10),1)-1)) & "'!A" & 1+ROW()-1)

We start off with INDIRECT(" ' " &  without spaces - this gives us the first ' if it is needed (Excel will ignore it if it is not needed)

Now, that whole IFERROR() section with all of those MID() and FIND() functions in it: this is simply digging out the worksheet name all by itself from what the basic CELL("address",ref) function is returning throughout the formula.  We need the IFERROR() because we have to get a good result whether or not there is a ' in the returned value, and the first try is for CELL() to return a result based on there being ' in the formula, which may not be true.  Because this is a true formula, the references to 'Tracking List' in the formula WILL automatically change to the proper sheet name if a user changes it!

Finally, after all of that we have

& "'!A" & 1+ROW()-1)

which completes our INDIRECT() function with the needed column and row reference again.  Like before the math of the 1+ROW()-1 part may need to be adjusted depending on which row you start in on the second sheet (the one with this formula in it).

As I said, all of this may (and probably is) overkill and much more than you need.  But I thought I'd offer it up just in case.  While this does give you a set-it-and-forget it formula, it is also going to have an impact on speed/performance - how much impact depends on how far down your worksheet you have to have formulas like this.

There are other alternatives such as putting the whole IFERROR() portion into a single cell somewhere and just referencing that in our INDIRECT() formulas - that way the calculation of the 'Tracking List' sheet name only has to take place one time, not 10s or 100s or thousands of times.

A little about absolute references.  Let us say that on a sheet you had a sales tax or value added tax rate in cell A1.  In column B perhaps you have amounts of sales.  In column C you want to calculate the tax on the amount of sale.  A regular looking (relative addressing) formula for the sale in B2 would look like this:

=B2 * A1

However, as you drag that formula down the sheet, B2 changes to B3 and that is ok, but the A1 also changes to A2, which is not what you want to happen.

If you change the formula to

=B2 * A$1 

then as you drag it down the sheet, only the row reference for column B will increase, the row for the rate in column A remains 'absolute'.  If you also needed to copy the formula over to some other column on the sheet, then the A would change since it was relative, but if you entered the formula as

=B2 * $A$1

then both the column and row references for the tax rate cell are absolute and don't change.  So if you copied your formula over into column E, Cell E2, it would change to become

=D2 * $A$1

I hope that helps with the understanding of relative and absolute.  Here are a couple of web pages with more detailed explanations, with pictures even!

http://web.pdx.edu/~stipakb/CellRefs.htm

http://www.cpearson.com/excel/relative.aspx  which even mentions the use of INDIRECT() to deal with issues like you have at this time, but no pictures.

bicyclops say:

Results of Query as OR Criteria for another Query

I'm trying to generate a query where the criteria can be a number list with an undetermined number of values. So the criteria might be13 OR 17; but it might also be 4 OR 155 OR 12 OR 200.

I can write a query easily enough that will produce the list of numbers. Can I use the results this Select query as criteria for another query?

john p spencer say:

If you can write a query to produce the numbers then use it as a subquery.

SELECT FieldA
FROM TableA
WHERE FieldB in (SELECT NumberField FROM TableC)

In query design view, under the number field enter the criteria as
    IN (<<Text of your query>>)

Your subquery must return only one field for this to work.

A second option is to use your query as a table and join it to the table in another query.

SELECT *
FROM TableA INNER JOIN qryWithNumbers
ON TableA.NumberField = qryWithNumbers.NumberField
