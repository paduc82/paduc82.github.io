I think what you're asking about is a paragraph mark, ¶ (sometimes called a pilcrow). That mark appears at the end of every paragraph in a Word document, where you've pressed the Enter key. dale howard [mvp] say:

Akshay --

The are two sets of steps that must be performed to transfer a project to a new project manager.  The first set of steps must be performed by either the person who created the enterprise project, or by someone with Administrator permissions in Project Online.  These steps are:

  1. Launch your web browser and navigate to Project Web App.
  2. Display the Project Center page and click the name of the project that needs to be transferred to a new PM.
  3. At the top of the Quick Launch menu on the left side of the screen, click the Project Details link, if necessary.
  4. At the top of the page, click the Project tab to expand the Project ribbon.
  5. At the left end the Project ribbon, click the Edit button.
  6. Click the pick list button to the right of the Owner field and select the name of the new project manager.
  7. At the left end the Project ribbon, click the Save button.
  8. When the system completes the save and publish operations, click the Close button on the left end of the Project ribbon.
  9. When prompted to check in the project, click the Yes button.

Completing the above set of steps will allow the new project manager to open and edit the project in question using Microsoft Project.  The new PM for the project will now need to complete the following set of steps as well:

  1. Launch Microsoft Project and connect to Project Online.
  2. Click File > Open and then open the project.
  3. At the top of the project window, click the Check Out button.
  4. Display the Gantt Chart view.
  5. Right-click on the Duration column header and then select the Insert Column item on the shortcut menu.
  6. In the list of available columns, select the Status Manager column.
  7. In the Status Manager column, the new PM should select his/her own name for EVERY TASK in the project.  Use the Fill Handle capabilities (like in Excel) to speed up this process.
  8. Save the project.
  9. Publish the project.
  10. Close and check in the project.

I want to emphasize to you that BOTH sets of steps MUST be performed by the relevant parties.  The second set of steps MUST be performed by the new project manager, and cannot be performed by anyone else on his/her behalf.  Hope this helps.
