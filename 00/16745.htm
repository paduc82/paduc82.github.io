You can;t exactly do that, but you can simulate it with an unbound main form containing two subforms in Continuous Forms view, both bound to the same table.  The top one is sized to show only one record, and its Data Entry property is set to Yes, so it always shows a blank new record to be entered.  The bottom subform is sized to show as many records as you like, and its Allow Additions property is set to No, so it doesn't show a new blank record.  In the AfterInsert event of the (top) data entry subform, requery the other subform to make it show the record you just added.  The code for that would be something like this:

'------ start of example code ------
Private Sub Form_AfterInsert()

    Me.Parent!sfExistingRecords.Form.Requery

End Sub
'------ end of example code ------

(where "sfExistingRecords" is the name of the other subform control, showing the existing records).
