To insert line breaks in an other wise single line of code when the line of code is enclosed in double quotes.

  • Insert double quote at break point.
  • After the double quote, Insert ampersand, space and underscore then press Enter to move remainder of code to next line.
  • Insert double quote at start of new line.
  • Repeat for subsequent line breaks.

Note the double quotes are not used unless the break is within text that is already enclosed in double quotes.  Advisable not to attempt to insert a break inside text that has 2 leading and ending double quotes like in ""FILTEROUT"" above. ossiemac say:

When you use Option Explicit then during development and before running subs you can select menu item Debug -> Compile and many errors can be identified before attempting to run the program. In particular it picks up errors like misspelt variables (In my case this is frequent and I am still looking for a keyboard that types what I meant to type and not what I did type). It does NOT pick up errors where the word is enclosed in double quotes like a misspelt worksheet name when referencing a worksheet.

If you select menu item Tools -> Options and check the box against "Require variable declaration" then Option Explicit is automatically inserted on every new VBA module.

However, all variables must be dimensioned. (ie. Dim statements at top of sub). This is good because without dimensioning the variable, a misspelt variable just creates a new vaiable and errors like this can be very difficult to find.
