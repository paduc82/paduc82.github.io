Sub splitter()
' splitter Macro
' Macro created by Doug Robbins to save each letter created by a mailmerge
' as a separate file, retaining the header and footer information.
Dim i As Long, Source As Document, Target As Document, Letter As Range
Set Source = ActiveDocument
For i = 1 To Source.Sections.Count
    Set Letter = Source.Sections(i).Range
    Set Target = Documents.Add
    Target.Range = Letter
    Target.Sections(2).PageSetup.SectionStart = wdSectionContinuous
    Target.SaveAs FileName:="Letter" & i
    Target.Close
Next i
End Sub

Look closely at this variation of your code. It will loop and find all criteria of your search.

    Dim done As Boolean
    Selection.Find.ClearFormatting
    With Selection.Find
        .Text = "[0-9]{1,2}[-/][0-9]{1,2}[-/][0-9]{2,4}"
        .Replacement.Text = ""
        .Forward = True
        .Wrap = wdFindStop
        .Format = False
        .MatchCase = False
        .MatchWholeWord = False
        .MatchAllWordForms = False
        .MatchSoundsLike = False
        .MatchWildcards = True
        .Execute
    End With
    Do Until done
        If Selection.Find.Found Then
            Selection.MoveRight Unit:=wdCharacter, Count:=1
            Selection.MoveDown Unit:=wdParagraph, Count:=1, Extend:=wdExtend
            Selection.Delete Unit:=wdCharacter, Count:=1

        Else
            done = True

    Loop

Tabbing through my form template not working correctly - skipping from - Microsoft Community

connie_852 say:

Tabbing through my form template not working correctly - skipping from page 1 to page 4.

I'm new at creating forms. I created a 4 page fillable form and locked it for users to complete and send back to me. When a user opens the form, they can tab through the first page fine, but when you hit tab to go to the next field on the next page it skips to the last page - skipping the first 2 fields on that page and going to the date picker. If you scroll up and place your cursor in the first field of the 2nd page it will then let you tab through the rest of the document fine. I’m using Legacy form fields. It wasn't doing this in the original form I created. After creating the form and giving it to my boss for review, he advised a few changes. It was after that when it started doing this. Things I added: 1. He wanted the Name field to automatically populate on pages 2-4 after the user fills in their name on page 1. I put a styleref in the header but had to insert a continuous page break so it would not appear on page 1. I thought that is what caused it but when I deleted the break it continues to skip. 2. He didn't want the tables or headings to be orphaned so I added "keep with next" to some of the paragraphs. 3. He wanted some of the instructions to be typed over in the field boxes as they are completing the form. I don't think the last 2 changes have anything to do with the skipping problem but those are the only changes I made to the document when it started to skip pages. Any help you could provide is greatly appreciated! Thanks!!!

suzanne s. barnhill say:

Instead of inserting a Continuous section break, enable "Different first page" and leave the First Page Header empty, putting your StyleRef field only in the Header, which will appear on pages 2ff.

Tab order for form fields is usually quite linear (with the understanding that fields in tables will go left to right before going down) unless the form fields are in text boxes or frames, in which case they will be accessed in the order of the paragraphs to which the text boxes or frames are anchored, but this should affect layout on a single page only; a field on page 4 could not be anchored to a paragraph that precedes fields on page 2.

One clue, however, might be that a date picker field is not a legacy form field.
