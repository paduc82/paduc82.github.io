massmamma say:

Is it possible to conditionally export Access table columns?

john w. vinson say:

Ah... that makes it a bit clearer: at first we thought you wanted to export only the empty columns. It makes more sense that you want to SKIP the empty columns! 

One other question: will there ever be a case where a column is only partially full - some rows null, some with a value? If so what do you want to do with it?

To do the export, I think you will need to dynamically construct the SQL of a new query, stepping through the fields looking for fields containing data; or (maybe better) use Excel Automation to do the export. The problem with doing it in a query is that you're talking about changing the actual structure of the query (which fields are included) based on the data in those very fields; the query can't (at all easily) say "Look in FieldC and if there's nothing there, then DON'T look in FieldC".

dhookom say:

You could create a small public function that builds an SQL statement of only the fields that contain data. I'm not sure how the performace will be with many fields and records

Public Function BuildSQL(strTable As String) As String
    Dim strSQL As String
    Dim db As DAO.Database
    Dim td As DAO.TableDef
    Dim fd As DAO.Field
    Set db = CurrentDb
    Set td = db.TableDefs(strTable)
    For Each fd In td.Fields
        If DCount(fd.Name, strTable) > 0 Then
            strSQL = strSQL & "[" & fd.Name & "], "
        End If
    Next
    Set td = Nothing
    Set db = Nothing
    BuildSQL = "SELECT " & Left(strSQL, Len(strSQL) - 2) & " FROM [" & strTable & "]"
End Function
