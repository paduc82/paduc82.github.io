rory archibald say:

By default passing arguments is done ByRef so any changes are returned to the calling sub. Therefore you can use something like:

Sub Parameters1(numR As Long, Gps As String, Sgl As String, Rnum As Long)
'Windows("Input Sheet.xlsm").Activate

'Storage1
    If Sheet1.Range("B6").Value = "Single" Then
        numR = 3    'finding starting row number for Single Item needed +1
        Sgl = "A4:A39"    'Range Lookup for single item
    Else
        numR = 46    'finding starting row number for Group of Items needed +1
        Gps = "A47:A370"    'Range lookup for Groups
    End If

End Sub
Sub main()
    Dim numberR               As Long
    Dim sGroups               As String
    Dim sSingle               As String
    Dim Rnumber               As Long

    Call Parameters1(numberR, sGroups, sSingle, Rnumber)

    Debug.Print numberR
    Debug.Print sGroups
    Debug.Print sSingle
    Debug.Print Rnumber

End Sub

a. user say:

I have never personally seen Word spontaneously insert breaks, but for a column break, just bear in mind that a column break in the last column acts as a page break (which may be an odd or even page break. So for example if you have a column break in column 2 of a 3-column section, and you insert text before the break, pushing the break into column three, you will end up with an extra page break as well.

If the "spontaneous inserts" happen often enough that you only have to wait a few seconds or a few minutes for a problem to occur, it may be worth starting Word in Safe Mode and seeing if the problem disappears.

macropod say:

Argue all you like, but it's NOT a Section break. It is a page layout option and, quite plainly, is not a Section break option.

I am also confident MS is aware that Section breaks sometimes change from one type to another. This is a long-standing, well-documented behaviour. stefan blom say:

As Paul/macropod pointed out, if the type of section changes (in the Page Setup dialog box) that is not the same as Word adding a new section break.

The type of section may change in conjunction with the printer driver (for example, some printers do not support printing portrait and landscape on the same sheet of paper which may result in an Odd Page section as opposed to a Next Page one). In such a situation, you can check for yourself that the number of sections in the document hasn't changed.

Note that Word will add a new section for you as you change the page layout (for example, the margins or the number of newspaper columns) for a selected text range.

Adding data to a filtered list - Microsoft Community

crtate say:

Adding data to a filtered list

Perhaps an obvious question - but is it ok to a filter a list then continue adding data to the filtered list?

Thanks.

ossiemac say:

Yes. You can add data to the blank lines below the filtered list.

However, data will not be filtered until the filter is reapplied. Do this by selecting the Data ribbon and in the Sort & Filter block towards the middle of the ribbon click on the Reapply icon.

Alternatively, click the filter drop down and then click OK.

tom ogilvy say:

Just to add to OssieMac's great answer, you can also add data by inserting rows (or cells) within the existing filter range.  Same as what OssieMac stated, you need to reapply the filter by either of the methods he described to get the updated filter display.

If you try to insert rows between two visible rows separated by some hidden rows,  the insertion will be above the selected row.  If this is not exactly where you want it, then you might need to reorder your data when you remove the filter.
