ken sheridan say:
In effect, I need master as well as detail table in datasheet view.

You cannot do that in datasheet view, but it is possible with the form and subform in continuous forms view.  This can be done in two ways:

1.  Create the parent form in continuous forms view and give it a form footer section.  Create the 'child' form also in continuous forms view, and then embed it in the form footer of the first form as a subform, linking them on HubID.

2.  A more flexible way, which allows more scope with the layout, is to embed both forms in an unbound parent form and correlate them.  To do this the second subform is based on a query which references the HubID control of the first form as a parameter.  The second subform is requeried in the Current event procedure of the first subform.

You'll find an example of the second method, using data from Northwind, as CorrelatedSubs.zip in my public databases folder at:

https://skydrive.live.com/?cid=44CC60D7FEA42912&id=44CC60D7FEA42912!169

If you have difficulty opening the link copy its text (NB, not the link location) and paste it into your browser's address bar.

jay freedman say:
Before you post a document for customer access, open it and go to the Developer tab, click the Document Template button, and attach Normal.dotm in place of the original .dotx. Make sure the "Automatically update document styles" box is NOT checked. Click OK and save the document.

When a customer downloads the document, Word will attach their local Normal.dotm template without the delay.

As long as the automatic update is turned off, the change of templates will have no effect on any styles in the document (ref. http://www.shaunakelly.com/word/templates/attachtemplate.html).
