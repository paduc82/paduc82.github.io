doug robbins - mvp office apps & services (word) say:

Your reasons for using DDE are not valid.

Using the Default OLE DB Database Files method of connecting to the data source does not impose a limit on the number of merge fields. 

The number of characters that can be merged per field can be restricted by the type (size) of data in the first 6 or 8 records.  However there will be no restriction if a the first (a dummy record if need be) contains enough data in each field to set the type of data that is expected.

The "half full" limit that you mention is most likely caused by the use of a DDE connection.

hal hostetler - mvp-outlook say:

GMail uses IMAP and the new IMAP service works differently than the same service in 2010. The new service downloads entire messages, not just the headers. If the service encounters an error on the server of some sort, it halts in its tracks, where the prior service would just keep on going. I have an old IMAP account that, somehow over the years, developed a "disconnect" between some of the messages and their headers. Outlook 2010 (and every other IMAP client I know about) downloaded the headers fine, but would produce an error when the body of one of these disconnected messages tried to open. By contrast, Outlook 2013 stops cold on the first such message it hits. The fix, for me, was to use some other IMAP client to move these messages to a different folder, then back again (which, on the server, reconnected the headers to the messages). You might give that a try.
