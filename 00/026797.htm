The "PAGE_FAULT_IN_NONPAGED_AREA" error is usually caused by either a software or hardware. Windows cannot continue and crash (Blue/green screen of death) when it cannot find the critical data from the memory. To troubleshoot, we suggest that you follow the steps below:

Step 1: Run the check disk.

  1. On the Search bar, type CMD.
  2. From the search result, right-click the Command Prompt and select Run as administrator.
  3. Type this command in the Command Prompt: chkdsk /f /r then hit Enter.
  4. Once the process is complete, you may proceed to step 2.

Step 2: Run the System File Checker.

  1. In the Command Prompt, this command: sfc /scannow and press Enter.
  2. Wait for the scan to finish.

If the issue still persists, try to uncheck the Automatically manage paging files for all drives.

  1. Open the File Explorer and scroll to This PC.
  2. Right-click This PC and click Properties.
  3. Click on Advanced system settings and go to Advanced tab.
  4. In the Performance box, click Settings.
  5. In the Virtual memory box, click Change then uncheck the Automatically manage paging files for all drives.
  6. After that, you can set the custom size for the page file.
  7. Once you are done, click OK then reboot your PC.

Feel free to respond and let us know the end result.

Charles 

This is my scenario I have create about 15 documents to be used as templates, now the specification of the space after and font have changed.  My boss read that we could use +body and +heading to automatically update the changes and wants to change our Normal.dot or insert a Theme.  Would this automatically fix the changes we need to the font and space after the paragraph?

guruprasad ra say:

Follow the steps below and check if this helps.

 

Open Outlook > Open a new mail message > Click on Minimize button > close the new mail message window.

 

Now open a new mail message window and check if open in minimized size.

 

If the issue persists, check if Outlook is running more than one instance. 

 

Try the below mentioned steps:

1.     Open Outlook.

2.     Press and hold Ctrl + Shift + Esc keys to open Task manager.

3.     Click on the process tab at the top and check if there is more than one Outlook.exe listed or not.

4.     If there is more than one Outlook.exe, select it and click end process all.

5.     Once Outlook is closed, open Run (press and hold Windows key + R key) and type Outlook->click ok.

6.     Repeat the step 2.

7.     Check if there is one Outlook.exe running in the process tab or more than one.

 

If it is more than one Outlook.exe, run Outlook in safe mode and check.

 

Click on Start > All Programs > Accessories > Run > Copy and Paste the following command in the open box and press Enter.

 

Outlook /safe

 

If Outlook safe works as expected, disable the add-ins:

 

http://office.microsoft.com/en-us/outlook-help/add-or-remove-an-add-in-HP005242782.aspx

 

Now save the changes in the new mail message and check the results.
