I am in an area where it is essential to have encrypted volumes on my computer.  As such, I installed DriveCrypt and created encrypted volumes for security purposes.  It seems there is an incompatibility between Office 2010 and DriveCrypt.  Anytime I open an encrypted file from the secure volume, once closed, I must use Brutal Dismount to close the volume.  This does not happen on any other types of files other than Office.  I am running Windows 7 Home.  After investigation, here is what I found:

When I open an Office document from within an encrypted volume (or anywhere else), a process called OfficeVirt.exe *32 starts from C:\Program Files (x86)\Common Files\Microsoft Shared\Virtualization Handler.   After closing the document, the process stays open and DriveCrypt will not close the safe (without using Brutal Dismount) until I go into Task Manager and end the OfficeVirt.exe *32 process. It appears that DriveCrypt is detecting that the file is still open from the process still running, even though the working Office document has been closed. 

Here is the response I received from DriveCrypt:

The fault here, appears to lie with the Microsoft Office software keeping a handle to the document file open even though it was saved and closed by the user.

OfficeVirt.exe*32 is clearly the culprit because terminating it with the task manager will release any open handles it has and then allow DC to dismount normally and you report that this is indeed the case.

On Windows, ending a process automatically releases any open handles and therefore closes any open files  that process still has. Once that is done, the tests Dc does on dismount, to determine if files are still open (attempting to be granted exclusive and sole access to the disk volume using FSCTL io controls ) will pass, (the lock will be
granted) and the disk can then be dismounted.

DriveCrypt will simply refuse to dismount "normally", any disk volume which still has open files on there for obvious reasons; it may result in data loss! Indeed it is windows itself which informs DriveCrypt that the disk is still in use, by refusing to grant an exclusive lock on the drive, which DriveCrypt asks for at the time of normal dismount, to ensure that the drive is now now longer in use.

Brutal dismount simply bypasses this process and closes the disk anyway but at the risk to the user of data loss and possible other much rarer problems such as possibleblue screens but on newer systems this does not happen anymore sinceWindows 2000 .

This is not a DirveCrypt problem and any well behaved encryption program which works in the same way as drivecrypt, will (or should) do exactly the same.


IN SUMMARY:
As far as I can tell, the problem is that "OfficeVirt.exe" is still keeping a handle open on the file, and will release it when the program is quit with the task manager. It is this problem which needs addressing, and therefore perhaps a complaint be submitted to Microsoft .

 

I need to be able to use Office 2010 and to keep documents sufficiently secured.  I apologize if this is in the wrong forum, there did not seem to be a better place to address this issue.  Any help getting this to Microsoft through the proper channel or suggestions to eliminate this problem would be greatly appreciated.
