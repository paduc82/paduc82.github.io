"Lock Protected Sections" option is disabled (grey) in - Microsoft Community

arnobpl_ say:

"Lock Protected Sections" option is disabled (grey) in OneNote universal app

I am new in OneNote. I want to protect some notes with password in OneNote. But the option is disabled. When I right-click a section, I see the following options (the disabled option is red-circled):

Can anyone explain why the option is disabled (grey)? I am using Windows 10 Pro by the way.

swischner say:

1. As of now the Universal App (Win 10 App) cannot encrypt and password protect any unprotected sections. You need another version (e.g. OneNote 2016 free version for Windows or Mac version) to do that. The menu entry "Lock protected section" is NOT meant to encrypt any unencrypted section (note the "protected" in the command, meaning "already protected").

2. What the Universal App can do is decrypt / open an existing password protected section. This section is then "unlocked". It remains unlocked for a certain time (I think 10 mins.). If you don't edit the note in that time it gets locked (encrypted, closed, hidden) again automatically. 

3. Now let's assume you want to leave your device unattended but the encrypted note is still unlocked because the 10 minutes are not over yet. In that case you may choose the "Lock protected section" command to manually and immediately hide the protected section (in fact ALL currently unlocked protected sections).

So if there is no protected section unlocked at the moment, the command is greyed out as there is nothing to close/hide. As soon as you unlock an existing protected section by entering its password that command gets availible. 

I hope this explanation has been more helpful.

idastalnaker say:

How do I calculate running totals for 90 consecutive days?

I need to calculate totals for each 90 consecutive days (January 1 - April 1, 2013; January 2 - April 2, 2013...).  How do I create a formula to accomplish this?  Thanking you in advance for your assistance. 

bernie deitrick say: The best formula depends on your data, but let's say that you have dates in A and values in B to be summed. In C2, enter

=SUMIFS(B:B,A:A,">=" & A2 -90,A:A,"<=" & A2)

though this is actually 91 days (Jan 1 to April 1) - change the -90 to get exactly 90 days  (-89 instead of -90) if that is what you really want

Then copy down. If you don't want formulas for before April 1, then delete those...


jeeped say:
That is going to depend a bit on whether you mean 90 days or 3 months. January 1, 2013 to April 1, 2013 is 90 days if you don't count either the start date of the end date (non-inclusive) but the same logic returns 89 days for February 1, 2013 to May 1, 2013. You can plug in 90 as Excel sees every day as 1 or you could use EDATE() to adjust the date used in the criteria.

                   

The above use a 3 month period and the dates are inclusive. You can adjust the >= and/or <= to > and/or > if yo prefer non-inclusive. The formula in C2 is,

=SUMIFS(B$2:B2,A$2:A2,">="&EDATE(A2,-3),A$2:A2,"<="&A2)

This should give you a running total.
