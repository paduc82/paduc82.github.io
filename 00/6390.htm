This reflects what you have stated in your shared workbook.

First, you need a sheet that is not in your list that will be the activesheet when the workbook is opened.   Name this sheet Main.

Put this code in the thisworkbook module. 

Private Sub Workbook_SheetActivate(ByVal Sh As Object)
Dim v As Variant, vv(1 To 5) As Variant
v = Array("Input", "Commission", "PO")
vv(1) = Array("INPUT1", "INPUT2", "INPUT3", "INPUT4")  ''vInput =
vv(2) = Array("COMMISSION1", "COMMISSION2", "COMMISSION3", "COMMISSION4", "COMMISSION5") ''vComm =
vv(3) = Array("ESTIMATE", "FINAL")   ''vPO =
vv(4) = Array("ESTIMATE1", "ESTIMATE2", "ESTIMATE3")  ''vEstimate
vv(5) = Array("FINAL1", "FINAL2", "FINAL3")  ''vFinal
Select Case UCase(Sh.Name)
   Case "MAIN"
      For i = LBound(v) To UBound(v)
       Worksheets(v(i)).Visible = xlSheetVisible
      Next i
      For i = 1 To 5
          For j = LBound(vv(i)) To UBound(vv(i))
             Worksheets(vv(i)(j)).Visible = xlSheetHidden
          Next j
      Next i
   Case "INPUT"
      For i = 1 To 5
          For j = LBound(vv(i)) To UBound(vv(i))
           If i = 1 Then
             Worksheets(vv(i)(j)).Visible = xlSheetVisible
           Else
             Worksheets(vv(i)(j)).Visible = xlSheetHidden
           End If
          Next j
      Next i
   Case "COMMISSION"
      For i = 1 To 5
          For j = LBound(vv(i)) To UBound(vv(i))
           If i = 2 Then
             Worksheets(vv(i)(j)).Visible = xlSheetVisible
           Else
             Worksheets(vv(i)(j)).Visible = xlSheetHidden
           End If
          Next j
      Next i
   Case "PO"
      For i = 1 To 5
          For j = LBound(vv(i)) To UBound(vv(i))
           If i = 3 Then
             Worksheets(vv(i)(j)).Visible = xlSheetVisible
           Else
             Worksheets(vv(i)(j)).Visible = xlSheetHidden
           End If
          Next j
      Next i
   Case "ESTIMATE"
      For i = 1 To 5
          For j = LBound(vv(i)) To UBound(vv(i))
           If i = 4 Or i = 3 Then
             Worksheets(vv(i)(j)).Visible = xlSheetVisible
           Else
             Worksheets(vv(i)(j)).Visible = xlSheetHidden
           End If
          Next j
      Next i
   Case "FINAL"
      For i = 1 To 5
          For j = LBound(vv(i)) To UBound(vv(i))
           If i = 5 Or i = 3 Then
             Worksheets(vv(i)(j)).Visible = xlSheetVisible
           Else
             Worksheets(vv(i)(j)).Visible = xlSheetHidden
           End If
          Next j
      Next i
End Select
End Sub

Now select any other sheet (even if you need to create a new sheet).  Then select main.  

Now save your workbook

if you  created an additional sheet, you can delete that sheet.  

Select Main anytime you want to reset to the 4 sheets   Main  Input Commission  PO

Now select tabs as you wish.

  7. Then click on Apply and OK.

a. user say:

Hi Rob,

 

This is related to 1)how MFA (Managed Folder Assistant) process the retention tags for specific items and 2)how Outlook handles retention tags, I will try my best to explain it to you.

 

1)MFA is one of the mailbox process assistants running on the server, which is used to process Retention policy/tags. For any item which has an explicit/ inheritable tag on it, MFA stamps a start date, an expiration date and a related retention action on it. Knowing this background is important for understanding the complicated processing mechanism in Deleted Items folder. from a folder that doesn't have an inherited or explicit retention tag. Please check following picture.

 


2)With Cached Exchange mode enabled Outlook client, it pulls folder's retention tag and calculate the date itself, which may cause a contradictory in the Delete Items folder based on the pattern how date is calculated in this folder. For example, if the scenario is conform to the ones that the start date is calculated by the date when the item is being processed. As a workaround of this behavior, try to check the expiration date in OWA for the items in Deleted Items folder.

Retention tags on Deleted items folder: https://blogs.technet.microsoft.com/vikass_blog/2013/12/13/retention-tags-on-deleted-items-folder/
