Hey Folks!

I updated another thread but forgot to update this one.   I found a solution that works for me.   It seems that the 1511 Windows update *does* remove the Media Feature Pack if you have it installed.  That's because the version that most of us had installed for Windows 10 N is no longer compatible with the 1511 release.   Unfortunately, MS didn't make that clear.  I found the updated Media Feature Pack ( KB3099229_x64.msu ) at the following link:

https://www.microsoft.com/en-us/download/details.aspx?id=49919

If you try installing the original Media Feature Pack for Windows 10 N ( Windows10-KB3010081-x64.msu ) it will appear as if it installed correctly (and will claim that it did so), but in reality, it will have installed nothing.  

Once I installed KB3099229_x64.msu, WMP reappeared, and a few other dependent applications of mine started working again too!

Hope this helps.

a. user say:

If you mean you are getting a STOP error that says something like:

SESSION3_INITIALIZATION_FAILED

*** STOP: 0x0000006F (0xc0000034, 00000000, 00000000, 00000000)

The usual culprit is some (poorly designed) antivirus software that has deleted or quarantined the Windows Session Manager file:

c:\windows\system32\smss.exe

I suppose it could have been really infected and quarantined, but that has never been my experience.  It is usually just deleted in error.

What antivirus software are you running and did you just use it prior to this incident?

You will not boot XP without the file, but you can replace it in just a few minutes by booting into the XP Recovery Console using a genuine bootable XP installation CD or make a bootable XP Recovery Console CD yourself.

After getting into Recovery Console then we can see if the file is missing and replace it if needed, or do additional troubleshooting if needed.

 

You can create a bootable XP Recovery Console CD when no XP media is available by following the directions in this link:

http://www.bleepingcomputer.com/forums/topic276527.html

When you get Recovery Console going, return and let me know and receive further instructions.

JoseIbarra

a. user say:

It would be good to know what AV software is acting up so we can straighten it out or replace it.

The usual culprit for this message is a missing or quarantined:

c:\windows\system32\smss.exe

That file should not be missing, so if it is - or if it is even even suspicious, use the Recovery Console to replace it from the c:\windows\system32\dllcache folder by executing the following commands from the Recovery Console prompt (you should be in the C:\WINDOWS) folder after booting RC, right?).

(XP has copies of most of your important system files in c:\windows\system32\dllcache...)

From the RC prompt:

 

cd system32

copy c:\windows\system32\dllcache\smss.exe

exit

 

You should get a successfully copied message.  If asked to replace the existing file, respond in the affirmative.

 

Sometimes, other files are missing, so first replace the smss.exe and see how that goes with a regular boot.  

If you still have the problem, boot back into RC and repeat the copying process for these files:

lsasrv.dll

ntdll.dll 

Once booted, I would review your antivirus strategy.
