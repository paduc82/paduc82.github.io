john mcghie say:
After going to bed last night, I had another thought:  the last macro is simpler to use in this format:

Sub DeleteBadStyles()
Dim aStyle As Style

With ActiveDocument
    For Each aStyle In .Styles
        Select Case aStyle
        Case aStyle = "Body Text", "Comment Text", _
            "Footer", "Header", "Heading 1", "Heading 2", "Heading 3", _
            "Heading 4", _
            "Heading 5", "List Bullet", "subTitle", "Table Body Text", "Table Bullet", _
            "Table Header", "Table Heading", "Table Number", "Title"
Case Else
        aStyle.Delete
    End Select
    Next aStyle
End With

End Sub

This way you feed it the list of styles you want to KEEP, not the ones you want to delete.  If it finds a wanted style, it simply does nothing.  If it finds any other style name, the "Else" case fires and deletes the style.

I should also have mentioned that although they "claimed" to have fixed the "in use" property of styles in VBA in Word 2003, it is still not reliable: you still have to look the slow and painful way to find out whether a style really is still in use.

john - project say:
PMP849,
As a follow-on to Julie's reply, your desired items 1 and 3 are doable using VBA but as far as I know, there doesn't seem to be a way to export Project data to Outlook. It's interesting that Outlook tasks CAN be imported to Project but the opposite isn't true, at least not in any obvious way when reviewing the Outlook Object Library. I believe the functionality to communicate between Project and Outlook is available with Project Server, but I don't know for sure because I don't use Project Server.

With regard to your first item, yes a macro could be used to poll each of your projects, get the desired data, export it to Excel and create the report. You can also see the combined data from all your projects by using a master file, however a linked structure does carry the possibility of corruption if not used carefully.

For your third item, again a macro can be used to export that data to Excel however the "automatically" part of your requirement is a little more complex since on the surface it would seem to require an event based macro, which is not as straight forward as a "normal" macro. Also including all the columns complicates the export. It is much easier to export a defined set of field data.

The bottom line is this is probably not something someone is going to step up and say, "sure I can do it and I'll do it for free", but who knows, you might get lucky. I have a macro that I wrote several years ago that exports the Project Notes field and a few other select fields to Excel. It could be modified to provide more functionality if you want to tackle that. Otherwise, if you want to hire someone to write the macro you described in your original post, I suggest you post your request to the Project customization and programming forum at: http://social.technet.microsoft.com/Forums/projectserver/en-US/home?forum=project2010custprog.
