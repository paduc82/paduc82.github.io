A dotm template that is saved in the Word Startup folder is a Global template, in that any macros that are in it are available to all documents.  While that is also true of macros that are created in the Normal.dotm template, that template should not be shared.  It is a template that is personal to the user.

Hi,

maybe you all try to do this from the "wrong" side.

This will not work if you have a view that is based on the n table and in the related items control (RIC) want to show data from the 1 table(s).

To work around this you can use a query instead of the tables i.e. try to

  • create a query that contains all tables (maybe using outer joins) and all fields you want to show in the view and in the RIC
  • be sure to include all relation building fields in the query
  • let the view base on the new query
  • now this query should also show up in the datasource list for the RIC tabs
  • if not then delete the old RIC, save the view and create a new RIC - I think I've sometimes seen problems in this respect.

You can also use multiple queries i.e. one query for the view containing all tables but only the necessary fields for the view and the relation building fields of the 1 tables, another query with the fields for the first tab in the RIC, another for the second tab etc.

wxpauser say:

This query solution works for me--and here's why.

A glance at Access Web templates shows that Related Items control shows tables that reference the current table (in the view). This only works one way.

The Product view can automatically have a Related Items control with Installations, because Installations is linked back to Products. Likewise a Customer view can automatically have an Installations Related Items control.

As I said, the query solution works for me. HOWEVER, it can mess with the pop-up property! ON the Related Items tab, under the Data cylinder, there is an option "Open Popup". But most queries have the wrong fields to make this work right. So tread lightly.

I prefer to use the Subview, holding a Datasheet view. When setting up the Subview, it asks for Parent field and Child field. These fields don't even have to be linked in the tables. And if you want the calculation features of the Related Items tab, build a query, activate Totals on the ribbon, and look at the new options for each field, such as Count.

There is also a related new symptom in Outlook mail: when I first type an e-mail, the font appears much larger than the size seen in the text format box in the ribbon (usually 11).  However, this is is not a major issue as far as I know now, because when I send the e-mail, text appearance magically returns to normal...!!  

I would appreciate reliable advice on this weird issue.

stevekoyama say:

moamon say: No one replied to my post with a solution, but I figured it out.  It was quite simple.  It turns out you can increase or decrease the font size in e-mails and in contact notes by turning the mouse thumb wheel while holding the Ctrl key.  The surprising part (to me at least) was that once you do that, the last setting is locked, even after you quit Outlook.  So my wife or I must have inadvertently increased the font size by touching the Ctrl key while scrolling with the thumb wheel, and then we didn't know how to bring the font size back to normal!
