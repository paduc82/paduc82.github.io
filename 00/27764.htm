Any updates done in the web app are immediately (to the best of my knowledge) reflected in the desktop version.

In desktop Access, if one person (User "A") is editing the current record, other users (Users "B","C","D") will not be able to make edits to the same record until it has been committed (no longer the current record) I would hope that this holds true for the web app as well.

The underlying reality to understand is that Excel is "document-based", intended for a single user. Access has multi-user capability with record-locking ... Excel does not.

So sorry, Microsoft had dropped that ball long ago. ;)

Create a split Access desktop app, and distribute the front-end to your users. Store the back-end as an ACCDB in a shareable folder in your Sharepoint (or maybe even OneDrive) site so that it will be available to anyone that you grant permission to. 

donleverton say:

A bit of an update / edit to my previous reply:

I was incorrect when I said that Excel does not have multi-user capabitity.

Apparently it does (but it still sucks, IMO)

https://support.office.com/en-us/article/Use-a-shared-workbook-to-collaborate-79bd9dee-1aa3-49b5-bc27-a1ad28ffcbce

When you share a workbook, multiple users can't work in the same file at the same time. Instead, you merge changes from one or more copies of that workbook back into the one that you shared.
The process works like this: You share a workbook, and as part of sharing, you save the workbook. Someone else then copies the shared workbook and adds their data to the copy. When they're done, you merge the data from the copy back into the original.
One of the best ways to share a workbook is to place it on a network location or a OneDrive folder. That allows others to easily access the workbook, copy it, and add their changes.

OK - here's a modification I don't know if you need or not.  But let's say you have what is shown in the previous 2nd graphic and you add to the list like this:

Basically this code checks the list, deleting rows with no entry in both the B/O and Qty columns and then re-expands it.  The code:

 

 

Sub RowExpand()
  'assumes first item is at D14 and is name of an item
  'assumes this is first time this record has been expanded
  'expansion is based on the greater of B/O or Shipped entry
  'for a row.
  Const topRow = 14       ' where to stop!
  Const BO_Col = "B"      ' used
  Const Shipped_Col = "C" ' not used at this time
  Const Qty_Col = "D"     ' used
  Const Desc_Col = "E"    ' used to determine last initial row
 
  Dim initialLastRow As Long
  Dim rowPointer As Long
  Dim echoCount As Integer
  Dim newRow As Integer
 
  initialLastRow = Range(Desc_Col & Rows.Count).End(xlUp).Row
  If initialLastRow < topRow Then
    MsgBox "No entries found in the Description column.", _
     vbOKOnly + vbExclamation, "No Processing Done"
    Exit Sub
  End If
  'this will make it faster if the list is long at all
  Application.ScreenUpdating = False
'this would 'clean up' a previously expanded list by
'deleting any rows that do not have entries in both
'the B/O column and Qty column
'collapse the entries!
  For rowPointer = initialLastRow To topRow Step -1
    If IsEmpty(Range(BO_Col & rowPointer)) _
     And IsEmpty(Range(Qty_Col & rowPointer)) Then
      Range(BO_Col & rowPointer).EntireRow.Delete
    End If
  Next ' end rowPointer loop
'now re-expand the list
  initialLastRow = Range(Desc_Col & Rows.Count).End(xlUp).Row
  If initialLastRow < topRow Then
    MsgBox "No entries found in the Description column.", _
     vbOKOnly + vbExclamation, "No Processing Done"
    Exit Sub
  End If
  'start from the bottom and work up!
  For rowPointer = initialLastRow To topRow Step -1
    'base number of new rows needed on larger of the
    ' B/O entry and the Quantity Entry
    echoCount = Range(BO_Col & rowPointer)
    If Range(Qty_Col & rowPointer) > echoCount Then
      echoCount = Range(BO_Col & rowPointer)
    End If
    'already have original row, need count of copies needed
    echoCount = echoCount - 1
    For newRow = 1 To echoCount
      'insert a new row below the current row
      Range(BO_Col & rowPointer + 1).EntireRow.Insert
      'copy the description to the new row
      Range(Desc_Col & rowPointer + 1) = Range(Desc_Col & rowPointer)
    Next ' end newRow loop
  Next ' end rowPointer loop
  MsgBox "Record Expansion Completed", vbOKOnly + vbInformation, "Task Finished"
End Sub
