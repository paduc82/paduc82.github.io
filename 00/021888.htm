tom van stiphout say:

It starts with creating the correct database design. You have introduced what's called a "Repeating Group" which is a violation of proper database design rules. You need to spin these off in their own table; perhaps:

tblContacts

ContactID autonumber PK

ContactFN text(20) required

ContactLN text(30) required

tblContactLevels

ContactLevelID autonumber PK

ContactLevelName text(50) required uniqueindex

(this is where "Primary Contact" and others go)

tblContactsForService

ServiceID long int required PK

ContactID long int required PK

ContactLevelID long int required

Now you can have Contact1 for Service1 in the role of Primary Contact, while same person is Secondary Contact for another service.
