dave peterson say:

So you want to replace a formula that results in #N/A with a simple string?

If you select the range and use
Edit|Replace (in xl2003 menus)
you'll see that there isn't an option for looking at values (like Edit|find
has).

But there may be alternatives.

If those are the only two errors in the range and you want to change each to the
same string, you could:

Select the range
Edit|Goto (or ctrl-g or F5)
Special
Check formulas and only keep Errors
then click ok.

Now only those cells with errors are selected.

Type your replacement string and hit ctrl-enter to fill the selection with that
string.

========
If you want to replace each error with different strings (or you have other
errors in the range that you want to keep) AND you're using xl2003 or higher...

Select the range
Edit|find
search for your error (say #N/A)
(click on the options button if Options aren't shown)
Change to look in: Values
Click Find all

Click in the bottom of the Edit|Find dialog where the individual "finds" are
shown.
Hit Ctrl-A to select all of them

Type your replacement string and hit ctrl-enter to fill the selection with that
string.

And then repeat for #DIV/0!

Dynamic Named Range as Row Source in Userform Combo Box using Excel - Microsoft Community

maneesh massey say:

Dynamic Named Range as Row Source in Userform Combo Box using Excel VBA

Hi,
How can I use a dynamic named range as a RowSource in a userform Combo Box. I have a Combo Box on a userform. I would like to have the rowsource for the Combo Box keep updating as a new ID number is added to the worksheet in column B. The name of the worksheet is 'Agent'. I am using Excel 2013.
Appreciate all help!
jeeped say:
Create a new named range using the OFFSET() function to define the Refers to: parameter.

           

Go into Formulas, Name Manager and create a new name. Give it an appropriate name like nr_User_IDs and (as per my example in the image) use a formula for the Refers to: like,

=OFFSET(Sheet1!$A$2,0,0,MAX(COUNT(Sheet1!$A:$A),1),1)

Note that there is a minimum [height] of 1.

When the named range has been defined, right click your combobox control and choose Format Control. On the Cell tab, set the Input Range to be nr_User_IDs.

As new numbers are added to column A, the nr_User_IDs named range will expand and the new entries are automagically picked up by the nr_User_IDs Refers to: definition and populated into the combobox.

OFFSET function Returns a reference offset from a given reference
MAX function Returns the maximum value in a list of arguments
COUNT function Counts how many numbers are in the list of arguments

jllatham say:

Usually an out of stack space means that one or more macros are calling one another reiteratively without ever actually returning or terminating.  The 'return to' addresses keep piling up until there's no place left to put them.

Unless you can determine what's looping forever and clean it up, you're probably just going to have to start with a regular, clean workbook (.xlsx type) and create your own macro.  Trying to clean up someone else's mess can be a frustrating nightmare.  Bad enough when you have to clean up your own.

Your 'new' workbooks from a 'Blank Template' shouldn't start out with macros in them at all.

It doesn't even take a lot of code to do this:  these two Subs() will create the same error in very short order - doesn't even matter which one you choose to start with.

Sub FirstSub()
  DoEvents
  SecondSub ' calls SecondSub, which never returns here
End Sub
Sub SecondSub()
  DoEvents
  FirstSub ' calls FirstSub, which never returns here either
End Sub
