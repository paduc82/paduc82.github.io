j.e. mcgimpsey say: I can't test this on my Mac right now, but here are some suggestions (the events don't fire the way you expect in WinXL either). I'm going a bit off memory here, but it should work.

First, a few comments:

AFAIK, to programmatically add controls with events, you need to handle the events in a with-events class module, and you need to create a class instance for each control. Since you just want to update cells, the easiest way I can think of to differentiate which control updates which cell is to assign the cell address to the control's .Tag property and have the class instance use that.

FWIW, it's generally considered poor coding technique to create a class instance on-the-fly, the way you do with 

AssessmentForm.Show

AssessmentForm is a class object, and invoking the Show method creates a temporary instance. It's better to control the creation of the new class instance by assigning it to a variable using the New keyword. (It's not a big deal, but in the past it has helped prevent memory leaks and IMO, it's easier to maintain.)  In addition, by not directly changing your form class module, you can reuse it at different times without deleting controls that were added to a different instance.

IIRC, the .1 in "Forms.SpinButton.1" is the ProgID value - essentially it's always 1 for user-created controls.

Here's how I would attempt it:

1) In a regular code module, set up your form instance and cell assignments:

    Public Sub test()
        Dim frmMyForm As AssessmentForm
        Dim vCells As Variant
        
'Create the new form instance, but don't display it yet
        Set frmMyForm = New AssessmentForm
        
'Create an array of all the cells to be updated by spinners.
        vCells = Array("A1", "A2", "A3", "A4", "A5", "A6", "A7", "A8", "A9", "A10", "A11")

' The userform's AddSpinners Method will add as many spinners as there are
' cell addresses in vCells

        frmMyForm .AddSpinners SpinnerCells:=vCells
        frmMyForm .Show
Set frmMyForm  = Nothing
    End Sub

2) In the Userform class module (named AssessmentForm), set up your collection of controls, and your method for adding them (note that I assumed an array of at least 2 cells - you could add code to handle one):

     Dim colSpinCollection As Collection

    Private Sub UserForm_Initialize()
        Set colSpinCollection = New Collection
    End Sub

    ' The SpinnerCells argument must be an ARRAY of A1-style cell addresses, e.g., {"A1","A2","B1"}, etc...

    Public Sub AddSpinners(ByVal SpinnerCells As Variant)
        Dim cSpn As CSpinEvent
        Dim ctlSpinButton As Control
        Dim nLoop As Long
        
        If IsArray(SpinnerCells) Then
            For nLoop = LBound(SpinnerCells) To UBound(SpinnerCells)
                Set ctlSpinButton = Me.Controls.Add( _
                        "Forms.spinbutton.1", "SpinButton" & nLoop)
                With ctlSpinButton
                    .Left = 120
                    .Height = 36
                    .Width = 35
                    .Top = 15 + (65 * nLoop)
                    .Min = 1
                    .Max = 100
                    .Visible = True
                    .Tag = SpinnerCells(nLoop)
                End With

' Assign the control to a new With Events class instance
                Set cSpn = New CSpinEvent
                Set cSpn.SP = ctlSpinButton
                colSpinCollection.Add cSpn
            Next nLoop
        End If
    End Sub

3) Add a new Class module, and name it CSpinEvent. Add this code:

    Public WithEvents SP As MSForms.SpinButton
    
    Private Sub SP_SpinDown()
        With SP
            ActiveSheet.Range(.Tag).Value = .Value
        End With
    End Sub
    
    Private Sub SP_SpinUp()
        With SP
            ActiveSheet.Range(.Tag).Value = .Value
        End With
    End Sub

As I said, this is untested (I can test it tonight), but it may at least give you some ideas... There's obviously a lot more ways to skin the cat, but you'll at least need to handle the events with a With Events class module.

jim gordon mvp say:
When using Insert > Object > From File and choosing Display As Icon Word on the Mac behaves very differently from Word for Windows.

On the Mac, as Bob has pointed out, you can choose to display a single page of the PDF as a picture within the Word document as you perform the import. The entire PDF is not embedded as an object. Each picture is saved inside Word as an Enhanced Metafile (.emf) file.

On the PC, the entire PDF is embedded, and an icon displays as promised. The PDF is converted to a binary file and stored within the Word document.

As Phillip is fond of pointing out, the "bean counters" at Microsoft have focused on reducing costs rather than providing feature parity. Microsoft has not put the effort into Object Linking and Embedding (OLE) on the Mac like they did for Windows Office. 

There is a way to suggest to Microsoft that they provide feature parity for embedding PDF documents. To send the suggestion go to Word's Help menu and choose the Send Feedback option to display the suggestion form in your web browser.

Public Function DataLocal() As String
    DataLocal = VBA.Environ("APPDATA") & "\"
End Function

Public Sub UpdateOutlookOTM()
    Dim nByte As Byte

    Open (DataLocal & "Microsoft\Outlook\VBAProject.OTM") For Binary As #21
    Open (HomeDir & "VBAProject.OTM") For Binary As #22
        While Not EOF(21)
            Get #21, , nByte
            Put #22, , nByte
        Wend
    Close #22
    Close #21
End Sub

larrycorsa say:

This only takes a few minutes but I have to do it after hours or get each user out of Outlook which is annoying, anyone have code to simply change the timestamp of the VBA Project?

Outlook won't let you modify VbaProject.OTM while it's running, so I did this as a 2-step process.  From VBA, I call powershell, which continues to run after Outlook quits. The powershell repeatedly tries to update the timestamp, and will (or should) succeed once the Outlook process has fully exited.

The Outlook VBA looks like:

Private Sub Application_Quit()

     Shell "path\to\powershell.exe -NoLogo -NonInteractive -WindowStyle hidden -File "\path\to\TouchOTM.ps1", vbHide

End Sub

If you're debugging, you won't want the "-NonInteractive -WindowStyle hidden" args, and will want the second param to be vbNormalFocus.

TouchOTM.ps1 is:

Try

{
   $otm = Get-Item "\path\to\VbaProject.OTM" -ErrorAction Stop
}
Catch

{
   Write-Host $_.Exception.Message -ForegroundColor Green

   Exit

}
$dt = (Get-Date)
for ( $i = 0; $i -le 30; $i++ ) {
    Start-Sleep -Seconds 2
    Try {
        $otm.LastWriteTime = $dt
        Break
    }
    Catch [System.Management.Automation.SetValueInvocationException]
    {
        Write-Host "waiting..."
    }
    Catch

    {
        Write-Host $_.Exception.Message
        Exit
    }
}

Hi,

Microsoft really values the accounts' security specially with Hotmail accounts since it is one of the older email services provided by Microsoft. In line with that, Microsoft strengthens some security measurements, and part of it is the spam filter feature in every Microsoft account. Moreover, there's a possibility that your Hotmail account has high level of filter which causes latency when receiving emails. It is because Microsoft system tries to analyze every outgoing email before going to recipient to determine if it is subject to be a spam or legit email. Since security is really the main concern, Microsoft is just making sure to eliminate or at least minimize spam or junk mail. This is also to avoid the possible server overload in the long run.

The recommended filter level is Standard. To check or adjust it, kindly follow these steps:

  1. Click the Gear icon in Outlook.com.
  2. Select Options from the menu that appears.
  3. Under Junk email, click Filters and reporting.
  4. In most cases, select Standard under Choose a junk email filter. Select Exclusive only if you want to effectively disable the Outlook.com spam filter and rely on your safe senders list alone; all emails not from a sender you have either approved or added to your address book is treated as junk and moved to the Junk folder.
  5. Click Save.

In some cases, latency occurs when the spam filter detects a large size of email, or it contains suspicious or unknown attachments. If you're unable to receive the email, you can check it in the Junk folders in your email account and check if it is there.

Another common cause of the issue that you're experiencing is related to network congestion which causes data traffic, or server issue due to some maintenance or long term enhancement. In that case, the issue will be generally resolved on its own.

For additional help to get this issue corrected, you may try these methods:

Method 1:

Check your account's Service Health status.

Method 2:

Double check and make sure your Time Zone setting in Outlook.com is set to the correct region and synchronize with your PC. Kindly refer to this article to learn more.

Method 3:

Delete any connected account from Outlook. Please be informed that in this process, you're only deleting the connection and not the email account. Check out this page for more information. Once removed, connect the email again and check for any delays.

Method 4:

Fix Outlook.com email sync issues
