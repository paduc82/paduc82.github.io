cortig say:

The problem with winmail.dat files is always on the sender's side. I've never ever sever seen it be an IMAP server issue.

It's a recurring problem that's all over the Web regardless of the email client (other than Outlook for Windows).

The cause is well known: They have Outlook for Windows configured to send with the TNEF format. The .dat file has the copy of the message with formatting and sometimes the attachment too. 90% of the time, you can safely ignore the winmail.dat file (since it's just a copy of the message), but sometimes it contains the attachments.

Outlook for Mac (and Mail, and Thunderbird, etc.) is unable to directly read the winmail.dat files to display the content as it would for regular formatted messages (and no app I know of does actually), but there are a number of apps that should be able to extract the content. Look for TNEF on a site like MacUpdate for instance and you'll find plenty. The one Jim mentioned does just that‚Ä¶

Long term though, as Jim was trying to emphasize, the only solution is to ask the sender to change the configuration fo Outlook on their machine to disable TNEF (the format that encapsulates the message in the winmail.dat attachment). I perfectly understand that it could be a touchy situation and if you are reluctant to mention it to them, then you have to use an app to extract the content of the winmail.dat file.

In a way, they probably don't even realize that they are sending messages in that format had have contacts right and left complaining about these attachments. You might find a diplomatic way to make them aware of it (and the solution), that might end up helping them on the long term.
