a. user say:

IMO it is a fairly difficult problem which may require several iterations and a lot of clarity as to what can, exactly happen in the document. I hope someone has existing code that deals with as many issues as possible, but failing that...

...as an experiment, you could try running the following VBA on one or more "tables" or "partial tables" in your document. It's been thrown together so I would be surprised if it works at all, but if your document has consistent tab settings, it may do something useful.

The idea is that you select a "table" consisting of a "header row" and the following data rows. The macro tries to compare the left or right positions of each chunk of text in each row with the equivalent positions taken from the "header row". It then writes the text chunks to the appropriate cell in a table in a second document.

If you don't have VBA skills, it will be hard to make this work, and it would be hard for anyone to progres without a copy of your document.

The VBA makes a lot of assumptions, by no means all documented, e.g. that the tab definitions are identical in every row. It has a stab at coping with the various things you have mentioned. So I certainly don't expect it to work. But the name of the game in this kind of scenario is primarily towork out whether it's actually going to be simpler to do the job manually, whether the code can successfully convert at least some of your data into tables, and so on.

The main assumption is that your document has multiple tables and that this code would only convert one at a time. So in a production version, it would be worth working out how to detect where each table begins and ends.

To run it, you'll need to install the VBA. Graham has some instructions somewhere on his site. Back up your document and work with a copy. Then select a "table header"+"data rows", run the macro and see what happens. I wouldn't put much effort in if the macro fails or does nothing useful.

Sub tblconvert1()

Dim bFound As Boolean
Dim c As Word.Cell
Dim i As Integer
Dim j As Integer
Dim lhp As Single
Dim rhp As Single
Dim p As Long
Dim s As String
Dim sa() As String
Dim sb() As String
Dim srs As Long
Dim sre As Long
Dim srp As Long
' tab alignment types
' At the moment we only attempt to deal
' with left and right
Dim tsa() As Word.WdTabAlignment
Dim td As Word.Document
' tab positions from paragraph1
Dim tsp() As Single
' tab positions from the TabStops collection
'Dim tspt() As Single
Dim tt As Word.Table

' Originally, I tried to get this from the
' tabstops collection of the first paragraph
' in the selection, but that results in
' positions that do not quite match

' So instead, we derive the positions from the
' locations of the appropriate characters in
' the first row.
'
' At the moment, I am assuming that the
' tabstops are the same in each paragraph,
' and that the first paragraph defines
' our column count, and that no tabs are
' "skipped" in the first paragraph

Set sr = Selection.Range
srs = sr.Start
sre = sr.End
srp = sr.Paragraphs.Count
With sr.Paragraphs(1)
  s = .Range.Text
  ' sa is a base 0 array with ubound(sa)+1 elements
  sa = Split(s, vbTab)
  Debug.Print "columns", UBound(sa) + 1
  ' There should always be 1 more array element
  ' than tabs used
  ' col1<tab>col2<tab>col3<tab>..<tab>coln<crlf>
  ReDim tsa(UBound(sa))
 ' ReDim tspt(UBound(sa))
  ReDim tsp(UBound(sa))
  cp = Len(sa(0)) + 2
  For i = 1 To UBound(sa)
    tsa(i) = .TabStops(i).Alignment
    'tspt(i) = .TabStops(i).Position
    If tsa(i) = wdAlignTabLeft Then
      tsp(i) = .Range.Characters(cp).Information(wdHorizontalPositionRelativeToTextBoundary)
    Else ' if right aligned
      tsp(i) = .Range.Characters(cp + Len(sa(i)) - 1).Information(wdHorizontalPositionRelativeToTextBoundary)
    End If
    Debug.Print tsa(i), tsp(i) ', tspt(i)
    cp = cp + Len(sa(i)) + 1
  Next
End With
    
' create the output table and populate the
' header row

Set td = Documents.Add()
Set tt = td.Tables.Add(td.Content, srp, UBound(sa) + 1)
tt.Cell(1, 1).Range.Text = sa(0)

' fix up alignment. We could also attempt to fix up the column widths.
For i = 1 To UBound(sa)
  If tsa(i) = wdAlignTabRight Then
    For Each c In tt.Columns(i + 1).Cells
      c.Range.ParagraphFormat.Alignment = wdAlignParagraphRight
    Next
  End If
  tt.Cell(1, i + 1).Range.Text = adjustsa(sa, i)
Next

' Process the data paragraphs

For p = 2 To srp
  Set sr = sr.Document.Range(srs, sre)
  Dim sp As Range
  Set sp = sr.Paragraphs(p).Range
  s = sp.Text
    ' sa is a base 0 array with ubound(sa)+1 elements
  sb = Split(s, vbTab)
  sa = Split(s, vbTab)
  
  ' to try to cope with the situation where
  ' the last "field" in the paragraph has wrapped
  ' we replace the final text chunk with a single character
  ' Otherwise, the start/end text locations are arbitrary
  ' We probably need to process the columns backwards,
  ' replacing text until the paragraph's text only occupies
  ' a single line, but I haven't had time for that yet.
  
  sp.SetRange Start:=sp.End - Len(sa(UBound(sa))), End:=sp.End - 1
  sa(UBound(sa)) = "x" & vbCr
  sp.Text = "x"
  Set sp = sr.Paragraphs(p).Range
  
  Debug.Print "columns", UBound(sa) + 1
  cp = Len(sa(0)) + 2
  tt.Cell(p, 1).Range.Text = adjustsa(sb, 0)
  For i = 1 To UBound(sa)
    lhp = sp.Characters(cp).Information(wdHorizontalPositionRelativeToTextBoundary)
    Debug.Print "lhp", lhp
    rhp = sp.Characters(cp + Len(sa(i)) - 1).Information(wdHorizontalPositionRelativeToTextBoundary)
    Debug.Print "rhp", rhp
    bFound = False
    For j = 1 To UBound(tsa)
      If tsa(j) = wdAlignTabLeft Then
        If lhp = tsp(j) Then
          tt.Cell(p, j + 1).Range.Text = adjustsa(sb, i)
          bFound = True
          Exit For
        End If
      ElseIf rhp = tsp(j) Then
        tt.Cell(p, j + 1).Range.Text = adjustsa(sb, i)
        bFound = True
        Exit For
      End If
    Next
    If Not bFound Then 'what?
    End If
    cp = cp + Len(sa(i)) + 1
  Next
  sp.Document.Undo
Next

finish:
Set tt = Nothing
Set td = Nothing
Set sr = Nothing
End Sub

Function adjustsa(sa() As String, i As Integer) As String
Dim s As String
s = sa(i)
If i = UBound(sa) Then
  s = Left(s, Len(s) - 1)
End If
adjustsa = s
End Function
