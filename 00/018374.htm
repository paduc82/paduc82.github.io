jay freedman say:
Following up on Stefan's post, in Office 2003 it was possible to uninstall the built-in templates through the Control Panel (Add/Remove Programs on Windows XP, or Programs & Features on Vista or Windows 7). That version has a subcategory under Word for "Wizards and Templates" that you can mark as Not Available. Office 2007 and 2010 don't have that subcategory, and the capability is gone.

To add to Suzanne's comments, there's a reason you should never assign the User Templates location in Word to a folder that's common to two or more users: Word requires the Normal.dotm template to be in that folder. Because Normal.dotm is a sort of personal scratchpad for all sorts of styles, macros, and possibly building block entries, and there can never be more than one Normal.dotm file in that folder, each user's changes will step on the toes of the other users. For a common collection of templates, use only the Workgroup Templates location.

It isn't necessary to add the Workgroup Templates folder's shortcut to the User Templates folder -- any templates in the Workgroup Templates folder will automatically appear in the File > New > My Templates list, mixed in with those in the User Templates folder.
