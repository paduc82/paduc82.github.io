Sub Resource_CF_To_Task_Usage()

Dim Reso As Resource

Dim Task_As As Assignment

Dim Reso_As As Assignment

Dim Job As Task

For Each Reso In ActiveProject.Resources

    If Not Reso Is Nothing Then

        For Each Reso_As In Reso.Assignments

            Reso.Text7 = Reso_As.Text7

            Set Job = ActiveProject.Tasks(Reso_As.TaskID)

            For Each Task_As In Job.Assignments

                If Task_As.ResourceID = Reso.ID Then

                      Reso_As.Text7 = Task_As.Text7

                End If 'TaskID

            Next Task_As

        Next Reso_As

    End If 'Nothing

Next Reso

End Sub

jllatham say:

Try the following code on a COPY of your workbook.  You'll need to add a new sheet and name it WorkingLists.  That sheet can eventually become a hidden sheet so people don't ask "why is this in here?".

 

Change any of the worksheet names and cell/column references in the code as required.  Everything is defined at the top of it before the work actually begins for easy maintenance.  Hopefully I've put in enough comments for you to grasp what's going on.

 

After it's done, you'd select all of the data and sort by the 2 added columns - the second added column will hold the original row numbers of the entries which will facilitate keeping groups together and in the same sequence when they are sorted, and distinguish between multiple groups that end up with the same ranking.  Actually, if I knew more about your sheet layout (which columns need to be sorted and whether it has column labels in row 1, AND if we could move the X and Y columns used here close in to the original table) then we could perform the sort in the code also and clean up columns X and Y when that had been done.

 

Sub RankAndSort()
  'data begins in D2 and then
  'in each 7th row (as 9, 16, etc)
 
  'build unique list of values from total scores
  'we will use another sheet that is named WorkingLists
  'to store our information
  Dim srcWS As Worksheet
  Dim destWS As Worksheet
  Dim lastSrcRow As Long
  Dim rowPtr As Long
  Const srcSheetName = "Sheet1" ' change as required
  Const totalScoreCol = "D"
  'results columns - see comments below
  Const srcRankCol = "X"
  Const srcSeqCol = "Y"
  'for quick referencing of data on WorkingLists sheet
  Const destSheetName = "WorkingLists" ' can be a hidden sheet
  Const destScoreCol = "A"
  Const destRowCol = "B"
  Const destRankCol = "C"
  Dim startingRowsList As Range
  Dim anyStartRowEntry As Range
  Dim startRow As Long
  Dim rankValue As Long
 
  Set srcWS = Worksheets(srcSheetName) ' sheet with data to sort on it
  Set destWS = Worksheets(destSheetName) ' can even be hidden
 
  'start with clean destination sheet
  destWS.Cells.Clear
  destWS.Range(destScoreCol & 1) = "Total Score"
  destWS.Range(destRowCol & 1) = "Original Row"
  destWS.Range(destRankCol & 1) = "Ranking"
 
  lastSrcRow = srcWS.Range(totalScoreCol & Rows.Count).End(xlUp).Row
  For rowPtr = 2 To lastSrcRow Step 7
    destWS.Range(destScoreCol & Rows.Count).End(xlUp).Offset(1, 0) = _
     srcWS.Range(totalScoreCol & rowPtr)
    destWS.Range(destRowCol & Rows.Count).End(xlUp).Offset(1, 0) = _
     rowPtr
    destWS.Range(destRankCol & Rows.Count).End(xlUp).Offset(1, 0).FormulaR1C1 = _
     "=RANK(RC[-2],C[-2])"
  Next
 
  'now put the ranks back onto the source data worksheet
  'along with their original row number as a secondary sort key
  'use column X for rank value, column Y for original row #
  'can change X and Y as needed
  'but they need to be 2 columns we can have full use of
  Set startingRowsList = destWS.Range("B2:" & _
   destWS.Range("B" & Rows.Count).End(xlUp).Address)
  'clear any old info in our sorting columns
  'and put labels on them for sorting convenience
  With srcWS
    .Columns(srcRankCol & ":" & srcRankCol).ClearContents
    .Range(srcRankCol & 1) = "Rankings"
    .Columns(srcSeqCol & ":" & srcSeqCol).ClearContents
    .Range(srcSeqCol & 1) = "SecondaryKey"
  End With
 
  For Each anyStartRowEntry In startingRowsList
    rankValue = anyStartRowEntry.Offset(0, 1)
    startRow = anyStartRowEntry
    For rowPtr = startRow To startRow + 7
      srcWS.Range(srcRankCol & rowPtr) = rankValue
      srcWS.Range(srcSeqCol & rowPtr) = rowPtr
    Next ' end rowPtr loop
  Next ' end anyStartRowEntry loop
  'the list should now be ready for sorting
  'using the 2 new fields X and Y
'housekeeping cleanup
  Set startingRowsList = Nothing
  Set destWS = Nothing
  Set srcWS = Nothing
End Sub

alfredamoah say:

Unable to install app to another device

My market place says I have reach the maximum number of accounts what should I do please?

***Edited title from my market place says i have reach the maximum number of accounts for,what should i do pls? to Unable to install app to another device.***

joe men say:

Hello, Alfred.

Thank you for posting your query to Microsoft Community. I understand that you're having issues installing an app to another device.

There is a limit on how many times you can install a specific app under one Microsoft Account.

You will need to remove a device if you're trying to install an app or a game on a Windows 10 device and you've reached your device limit. Please be aware that you can install apps or games on up to 10 devices only.

You can manage your account linked devices by following the steps below:

  1. Go to https://account.microsoft.com/devices/about
  2. Login (if not already)
  3. Click the little down arrow next to your name
  4. On the menu that appears, it should show your current device and any other devices registered to your account
  5. Click on the phone you want to remove
  6. On the next page that appears, click on Remove this phone
  7. A warning page will appear ... please read it carefully ... if you agree, click remove.

Hope this clarifies your Microsoft Account app installation limitation.

Feel free to post back if you need further assistance.
