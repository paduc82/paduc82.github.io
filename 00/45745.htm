Hey horsethief,

To disable this error message from appearing at startup, follow these steps.

Disable the startup item from loading.

To do so, start System Configuration Utility.

1. Click on the Windows 7 start icon in the bottom left corner of your screen.

2. Type MSCONFIG in the search box and then either press enter on your keyboard or double-click on the MSCONFIG program that appears in the search results.

3.  Windows 7 will launch Microsoft's System Configuration Utility. Click on the Startup tab.

4. This takes you to a page with a list of "startup items." Startup items are programs that are automatically loaded every time you turn on your computer.

5. Expand the column out so that you can read all of the text, and look for keywords. Proceed with un-checking the startup items one at a time in the “Startup Item” column to disable it. Alternatively you may follow the process of elimination by disabling half the startup items.

6. A message will pop up telling you that you must restart your computer before these changes will take effect. Click on "Restart" and your computer will automatically reboot.

7. Repeat this procedure till you find out which startup item is generating this error message. You can uninstall the software that comes with it or delete the startup item from registry. Note down the location of the startup item.

To delete the item from registry:

Important: This section, method, or task contains steps that tell you how to modify the registry. However, serious problems might occur if you modify the registry incorrectly. Therefore, make sure that you follow these steps carefully. For added protection, back up the registry before you modify it. Then, you can restore the registry if a problem occurs. For more information about how to back up and restore the registry, click the windows help article below.

Back up the registry
http://windows.microsoft.com/en-us/windows7/Back-up-the-registry
 
1. Go to start and type regedit.exe.
2. Navigate to the location of respective problem software.
3. Delete the item by right clicking on it and selecting delete.
4. Close registry editor.

Restart the computer. The error message should be gone by now.

The following parameters are displayed on the blue screen.


Parameter		 Description

1

Specifies source file and line number information. The high 16 bits (the first four hexadecimal digits after the "0x") identify the source file by its identifier number. The low 16 bits identify the source line in the file where the bug check occurred.

2

If NtfsExceptionFilter is on the stack, this parameter specifies the address of the exception record.

3

If NtfsExceptionFilter is on the stack, this parameter specifies the address of the context record.

4

Reserved

 

Cause


One possible cause of this bug check is disk corruption. Corruption in the NTFS file system or bad blocks (sectors) on the hard disk can induce this error. Corrupted SCSI and IDE drivers can also adversely affect the system's ability to read and write to disk, thus causing the error.


Another possible cause is depletion of nonpaged pool memory. If the nonpaged pool memory is completely depleted, this error can stop the system. However, during the indexing process, if the amount of available nonpaged pool memory is very low, another kernel-mode driver requiring nonpaged pool memory can also trigger this error.

Resolution

To debug this problem: Use the .cxr (Display Context Record) command with Parameter 3, and then use kb (Display Stack Backtrace).


To resolve a disk corruption problem: Check Event Viewer for error messages from SCSI and FASTFAT (System Log) or Autochk (Application Log) that might help pinpoint the device or driver that is causing the error. Try disabling any virus scanners, backup programs, or disk defragmenter tools that continually monitor the system. You should also run hardware diagnostics supplied by the system manufacturer. For details on these procedures, see the owner's manual for your computer. Run Chkdsk /f /r to detect and resolve any file system structural corruption. You must restart the system before the disk scan begins on a system partition.


To resolve a nonpaged pool memory depletion problem: Either add new physical memory to the computer (thus increasing the quantity of nonpaged pool memory available to the kernel), or reduce the number of files on the Services for Macintosh (SFM) volume.

douglasnigelb say:

It could be that some display driver circuit that decodes the y-axis of the display area has suddenly 'decided' to malfunction and this is the cause of any screen flicker you are currently experiencing with your computer.

This is a known problem with LCD panels, and it is very likely that the malfunctioning circuit is located on the display panel itself, rather than the graphics card.

Most display panel circuits require three or four purpose specific chips on each of the x-axis and y-axis, to decode the graphical rows and columns. When one of the chips malfunctions, then the screen output corresponding to that portion of the display starts flickering. 

Does screen flicker occur while the computer is active i) boot sequence, ii) the windows 7 start screen, iii) the desktop environment? If the answer is 'Yes' to all these then the problem is more than likely local to the display hardware.

Can you still see change of colo/u/rs --sorry color if you prefer --  underneath the flicker zone? Again this would indicate that the panels row driver is malfunctioning. It is unusual to see half the display go at once. That might mean more than one chip has malfunctioned.

What can you do to fix it?
1) If the computer supports an interface to an external monitor (HDMI, DVI or VGA), then you can use either a TV or another VDU to connect to your PC. If the flicker stops then you will know that the original fault was caused somewhere in the computer's LCD panel. Only the manufacture  could service or repair such a fault, so you would be well advised to contact their web site and find if there is a service centre near you, and check you have any warranty left.

Learn how to make a disk image and you will never need to reinstall Windows.

A disk image - also called a system image - is an exact copy of your hard drive, compressed into a single file. The image includes the drive's metadata, which makes the drive bootable.

Now let's suppose something bad happens and your computer gets all messed up - what do you do? First, you laugh because people take life too seriously. Then you restore a disk image you made yesterday and in a matter of minutes your computer has gone back in time to yesterday.

This is a fabulous way to backup your entire computer in one shot. Third-party image software makes this easy, and you can schedule the images to occur automatically, while you sleep. Backup at any interval you like: every week, every day, every 12 hours, etc. Once you start doing this you will wonder how you lived without it.

To be fair: The technical concepts behind imaging are a little complex, but it's nothing you can't learn. Do some research on the web and in time you'll learn everything you need to know. There's lots of good third-party imaging software. Some of it is free but even the paid applications have free trials. The one I use is called Macrium Reflect but there are plenty of other products.

Believe me, using a computer is much more enjoyable when you know that you can recover from disaster if you need to. Why waste a day re-installing Windows, updating drivers, re-installing your software applications and setting up everything all over again when, in just a few minutes, you can go back in time like nothing happened?

    The INACCESSIBLE_BOOT_DEVICE bug check has a value of 0x0000007B. This bug check indicates that the Microsoft Windows operating system has lost access to the system partition during startup.

    Cause

    The INACCESSIBLE_BOOT_DEVICE bug check frequently occurs because of a boot device failure. During I/O system initialization, the boot device driver might have failed to initialize the boot device (typically a hard disk). File system initialization might have failed because it did not recognize the data on the boot device. Also, repartitioning the system partition or installing a new SCSI adapter or disk controller might induce this error.
