Word built-in styles and language/locale of the Word application

Hi,

I'm working with a Software-as-a-Service provider managing our documents and creating PDF renditions. We have a problem of fidelity of the PDF when using built-in Word styles (Normal, Header...) as it substitutes Calibri where the document defines Times New Roman as the font for these styles.

Our vendor explained the following: "The problem with default styles is that they do not guarantee a specific styling, and will change depending on the language/locale of the Word application opening it."

I have doubts about this statement. Can some expert provide some advice as to whether this is actually true?

macropod say:

AFAIK, your provider's advice is only true insofar as certain locales don't use the western character set; all western languages use the same fonts for their built-in Style definitions. What does change is the local name of the Styles. See: https://www.thedoctools.com/downloads/DocTools_List_Of_Built-in_Style_English_Danish_German_French.pdf

Word will also substitute fonts if a given character isn't supported the font in use.

The real problem, though, is likely to be that users have changed the attributes of the default Styles or have overridden them with direct formatting and your provider's process doesn't allow for that. Another possibility, is that your provider's system doesn't have the fonts you're using, so its conversion process is substituting other fonts.

Frankly, I can't see why you'd bother with an external provider for PDF production when Office can do that directly - without the risk of formatting changes.

This sample uses column U (the 21st column) for the date.  Enter the following event macro in the worksheet code area:

 

Private Sub Worksheet_Change(ByVal Target As Range)
Application.EnableEvents = False
    Cells(Target.Row, "U").Value = Date
Application.EnableEvents = True
End Sub


Because it is worksheet code, it is very easy to install and automatic to use:

   1. right-click the tab name near the bottom of the Excel window
   2. select View Code - this brings up a VBE window
   3. paste the stuff in and close the VBE window

If you have any concerns, first try it on a trial worksheet.

If you save the workbook, the macro will be saved with it.


To remove the macro:

   1. bring up the VBE windows as above
   2. clear the code out
   3. close the VBE window

Thank you so much!!
hansv mvp say:

Let's say the text boxes are named TextBox1, TextBox2, ... TextBox 10:



Private Sub TextBox1_BeforeUpdate(ByVal Cancel As MSForms.ReturnBoolean)
    Cancel = TestDuplicate
End Sub
 
Private Sub TextBox2_BeforeUpdate(ByVal Cancel As MSForms.ReturnBoolean)
    Cancel = TestDuplicate
End Sub
 
...
 

Private Sub TextBox10_BeforeUpdate(ByVal Cancel As MSForms.ReturnBoolean)
    Cancel = TestDuplicate
End Sub
 
Private Function TestDuplicate() As Boolean
    Dim i As Long
    Dim col As New Collection
    Dim strVal As String
    On Error GoTo ErrHandler
    For i = 1 To 3
        strVal = Me.Controls("TextBox" & i).Value
        If strVal <> "" Then
            col.Add Item:=strVal, Key:=strVal
        End If
    Next i
    Exit Function
ErrHandler:
    MsgBox "This value is not unique!", vbCritical
    TestDuplicate = True
End Function



If your text boxes are named txtSerialNum1, txtSerialNum, ... txtSerialNum10, replace TextBox with txtSerialNum throughout the code.
