How to print out image with a macro? (PowerPoint) - Microsoft Community

a. user say:

How to print out image with a macro? (PowerPoint)

Hi,

I have an image on a PowerPoint slide with an action button next to it, saying print on it. Is there any way I can then create a macro for the action button to print this image that I have on the slide?

Moved from: (Office | PowerPoint | Windows 10 | Office 365 Home)

john sr wilson say:

There is really no code to do that directly that I know of. A possible workaround

Make a new blank slide

copy / paste the image to it

resize to fill slides as near as possible

Print the new slide

Delete the new slide

Outline of code to do this:

Sub printimage()
    Dim osld As Slide
    Dim oshp As Shape
    ' obviously change this to point to the image
    Set oshp = ActivePresentation.Slides(1).Shapes("Picture 1")
    Set osld = ActivePresentation.Slides.Add(ActivePresentation.Slides.Count + 1, ppLayoutBlank)
    oshp.Copy
    osld.Shapes.Paste
    With osld.Shapes(1)
        If .Width / .Height > 1 Then
            .Width = ActivePresentation.PageSetup.SlideWidth
            .Left = 0
            .Top = ActivePresentation.PageSetup.SlideHeight / 2 - .Height / 2
        Else
            .Height = ActivePresentation.PageSetup.SlideHeight
            .Top = 0
            .Left = ActivePresentation.PageSetup.SlideWidth / 2 - .Width / 2
        End If
    End With
    With ActivePresentation.PrintOptions
        .Ranges.ClearAll
        .OutputType = ppPrintOutputSlides
        .RangeType = ppPrintSlideRange
        .Ranges.Add ActivePresentation.Slides.Count, ActivePresentation.Slides.Count
    End With
    ActivePresentation.PrintOut
    osld.Delete
End Sub

sven johannsen say:

Can't get you all the way there, but we can get started.  As you are likely aware There is a router in between your ISP and your local network (PAN/LAN). That router has an IP address assigned by your provider. The router assigns addresses in a private IP space on your LAN, typically 192.186.1.x.  x being 1-255 and 1 normally being the router itself. The router does address translation between the outside and inside.  If you ask for something from a PC on your LAN, say from 192.168.1.150, the router forwards that to the internet, with the request coming from your ISP provided IP. When the response comes back, the router remembers who asked and delivers the response to the address that asked for it, in this case 192.168.1.150.

If a request comes in from the outside, it only knows about your ISP provided IP, and your router has no way of knowing who to forward that too. So coming in from a work PC or via your phone's connection is dropped by the router. That is actually a fairly effective firewall, as nothing from the outside is typically routed to any device on the inside, that wasn't initially requested by an inside device.

Two ways to get around that. One is port forwarding. This is set up on the router, and sends anything thing that comes in on a specified port to a particular IP on your internal network. This is often used for a network camera. If the camera IP is 192.168.1.175 on port 8040, you can set the router to send all traffic coming in to the router from outside on port 8040 to IP 192.168.1.175.  Some routers have a feature called UPnP (Universal Plug and Play) which can talk to internal devices and set up port forwarding automatically, which is why some home surveillance cameras just work. For your situation you would need to know what ports a Remote Desktop connection comes in on and set up port forwarding to the PC you want that request to go to. I don't know what that port, or ports, is, offhand, but that should get you started.

The other option is a service like DynDNS (Dynamic Domain Name Service) which associates a PC to a name usable outside your local network, to a PC inside your network. The Domain Name System is what allows you to use words rather than IP's to address things. It is why you can use Bing.com vs having to know Bing's IP address.  There are services that let your PC (inside) talk to the service and set up a name that is associated with the path backwards through your router.  So you wind up being able to use arty-PC.com, instead of some IP and port.

In either case you need to be careful that you have passwords on the remote connection set up, because anyone could use the DynDNS, or the IP and port, to try to use RDP (Remote Desktop Protocol) to talk to your PC. They should need a username and password to get farther.  That opening though, does make the internal network more vulnerable. Just like the web camera or thermostat or lights you can access/control from your cell phone, there is a hole in the router firewall that allows that communication to happen that could conceivably be exploited.

I know I didn't get you all the way there, but hopefully I gave you enough info you can do a little more digging and get the rest of the way.  You can Bing DynDNS (they aren't the only option) and UPnP to get some more insight. Hope I helped some.
