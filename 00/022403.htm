wunders say: It sounds like at least half of your problems are due to insufficient memory.  How much RAM memory do you have?  To determine, right-click on "My Computer" then "Properties" it should then display in the window.  Another method is:
   Start -> Programs -> Accessories -> System Tools -> System Information.
My Son's [old] computer only has 512 MByte of RAM and takes many minutes to boot and shutdown as you describe.  The only way it boots quickly is if I start it in Safe Mode (repeatedly tap F8 while booting).  When you boot your machine, look at your hard drive activity light.  Is it constantly on?  Do you hear your hard drive beating away in a frenzy? When you run out of Physical RAM memory, the computer starts using hard disk space as extra RAM (virtual memory) and this is much, much slower than physical RAM.  The only solution for this is to add more RAM to your computer.  IMHO, 1 GByte is a bare minimum for XP and anything over 3 GByte is wasted unless you have a 64-bit XP.  If extra RAM is needed, you might find that this extra RAM might cost about 1/4 the price of a brand new computer (something to think about).

My next step would be to check the health of your hard drive.  Use chkdsk with  the /F or /R option.
"How to perform disk error checking in Windows XP"
  < http://support.microsoft.com/kb/315265 >

With as many problems as you have, you would save a lot of time by backing up your files and then reinstalling Windows on the machine.  This would save you potentially months of troubleshooting and would pretty much guarantee that you are malware-free.  Go to your manufacturer's website and download the manual(s) for your computer.  This will instruct you how to reinstall Windows.

Regarding Internet Explorer Favorites, these favorites are saved in the folder:
   C:\Documents and Settings\<yourUsername>\Favorites
If you want to sync up favorites between machines, you will need to copy (merge) this folder from one machine onto the corresponding folder on the other machine.

HTH,
   JW

wunders say: > It reads as follow: Windows cannt open Help and Support because
> a system service is not running. To fix this problem, start the
> service named Help and Support. So I thought this mean the open
> Help and Support, so I did and got the same message.

This means to start the  service  Help and Support.  To do this:
   Start -> Run -> "services.msc"  ->  "OK".
    When the Services window opens, look for "Help and Support".  Its status should be "Started" and its Startup Type should be "Automatic".  If it is not that way, double-click on it and make it that way.

> Still researching manufacturer's website. When I get detail
> information I will let you know but in the mean while you are
> just saying wipe everything out and start over like I just
> purchase the computer - right?

Yes.  When you have a lot of problems or suspect bad malware infections, sometimes it is just quicker to start again from square one than to beat down each problem.  A last resort, but can also be a time-saver.  In some corporations, if an IT Technician can't solve a problem in less than one hour, they simply re-image the hard drive and you start from scratch.

> The Power on Self Test(POST) I didn't quite understand that.

Each time your computer starts, it performs a power-on test that, among other things, determines how much RAM is installed in your machine and configures itself to run with that much RAM.  My experience is that Windows XP, as it exists today, does not run well with less than 1 GByte of RAM.  If you have two 512MByte RAM Modules and one were to fail, system would report only 512MByte after POST and symptoms similar to yours could start to appear.  Verify that the RAM reported by Windows matches the physical RAM you have installed.  If it does not match, try removing and re-installing the RAM modules (might fix a bad connection) before purchasing new RAM.

Something else you can do is check your Event Viewer for logged errors in the System and Applications Log.
    Start -> Run -> "eventvwr.msc"
Note that it is normal for some errors to appear in the log.  Look for errors that apply to your symptoms.

brian tillman [outlook mvp 2007-2019] say:
Categories have always applied to the entire series of a recurring event.  You cannot assign a category to a single recurrence.  To do what you want, you'll need to create discrete events for each entry.  You can automate this, however, by creating another calendar folder, creating the recurring event in that folder, then exporting that folder to a CSV file.  This will convert the recurring event to discrete events.  Import the CSV file to your main calendar and you'll have all the individual events, to which you can assign the categories you want.

jay freedman say:
The screen display, and screen shots made from it, are not a reliable indication. By default Word adjusts the character spacing to reflect what will be printed on paper, but the screen resolution is much lower than the printer resolution, so letters may be a pixel off their correct location. Dealing with color pixels makes this problem worse.

You can try going to the Options dialog's Advanced page, scrolling to the Display group, and checking or unchecking the option for "Optimize character positioning for layout rather than readability". One way or the other may make the spacing better for screen viewing (but won't change anything in printouts).

ron coderre say:

Note: This is easier to do than it is to read (or write!)

Starting with this, in cells A1:B3
A 100
B 200
C 300


Insert a column before Col_A (which moves the range over by 1 column)
Fill the blank cells with the number series: 1,2...etc
Tip: 
After you put the first two values:
...select those two cells
...Click and drag down the Fill Handle (the little black box in the lower right corner of the second cell)
...That will continue the series: 1, 2, 3, 4...etc

Now, A1:C3 contains:
        Col_A     Col_B        Col_C
1            A 100
2            B 200
3            C 300

Insert a column betwenn Col_B and Col_C.
Fill those blank cells with 1.5, 2.5, etc
(You can use the click and drag technique to create that series, too)

A1:D3 now contains:
        Col_A     Col_B        Col_C         Col_D
1           A 1.5 100
2           B 2.5 200
3           C 3.5 300

Cut the C1:D3 cells and paste them below the Col_A and Col_B data:
        Col_A     Col_B
1                A
2                B
3                C
1.5 100
2.5 200
3.5 300

Now...you guessed it...Sort those 2 columns by Col_A, ascending

The end result:
        Col_A     Col_B
1                A
1.5 100
2                B
2.5 200
3                C
3.5 300

Last, delete Col_A

Is that something you can work with?
