thierryjanssens say:

SOLVED!


here we go:



Microsoft creates a Cache user for your notebooks to be available in offline mode. 


This hoses your Onenote when opening the program again out of sleep mode.  


Onenote is looking to the Cached user for credentials and there are none. 


So when it tries to Authenticate and you keep clicking on enter password it will never sign in.  


Close Onenote and Go to


Control Panel


>User Accounts and Family Safety in Category view


(or Manage Accounts in Icon View)


>Credential Manager


>Windows Credentials


>There you will see the Cached users,


they are usually labeled as MicrosoftOffice15_Data:Live:cid=some set of numbers. 


Delete those users as for every cached notebook there will be a cached user. 


Once you have deleted those user open Onenote and enter the password for your live account. 


It will connect and you will be able to sync your notebooks again.

christopher miller say:

Hi Luke,

With Honda's HandsFreeLink, it imports the first 10000 phonebook entries from the phone and stores it.  When you go back to the Phonebook, it's pulling the numbers from the car's memory not the phone.  If you are getting an error just going back to the phonebook, then it sounds like a problem with the Honda.

One thing that you try is in the HandsFreeLink Phone settings, turn Automatic Phone Sync to off.  That will set HandsFreeLink to not update the call history or phonebook from the phone, only use what is currently stored in HandsFreeLink.  If you turn it off and no longer get the error message, then the problem is when HandsFreeLink requests updates from the phone.

Which phone is it, the Lumia 925 or the Lumia 930?  You have mentioned them both.  I have a Lumia Icon paired to my 2014 Honda Odyssey.  The Icon is basically the Verizon only version of the 930 and has the same Bluetooth radio.  I can't imagine that HandsFreeLink changed dramatically between 2014 and 2015, but it may have.
