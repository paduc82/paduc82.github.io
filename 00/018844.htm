jay freedman say:

It sounds like Vlasp01 expects to just plug in the old drive as an external drive and start the already-installed Office programs that are on that drive. I don't believe that will work.

A large part of the installation of Office is the creation of many (certainly hundreds, maybe a few thousand) registry entries that tell Windows where the programs are and what to do when they're invoked via shortcuts or by "starting" existing documents. Since Office has not been installed since Windows was reinstalled on the new computer, none of those entries will be in its registry. You'll just get "not found" errors when the programs try to run.

Assuming the original disk and product key are still available, you can certainly do a reinstallation onto the external drive if you wish. That will create the necessary registry entries. You might be better served to install to the new C: drive, though, because its access time and transfer rate will probably be far better than what you'll see with a USB-connected external drive (an eSATA-connected drive might be OK).

matthewearly say:

I had this issue as well with Outlook 2016 and Windows 10.  I had an exchange account and a gmail account linked up to Outlook.  I did end up fixing it by removing the app data files and letting it redownload/recreate them. 

Steps:

1. Close Outlook.  Go to your user profile folder.  'C:/Users/[profilename]'

2. Show hidden files. Go to the 'View' tab and check 'Hidden Items'

3. Navigate to AppData/Local/Microsoft/.  Full path of 'C:\Users\[profilename]\AppData\Local\Microsoft'

4. Delete the folder 'Outlook'

5. Reopen Outlook and let it recreate all these files. Outlook should load up once complete.

This worked for me, can't promise it will work for everyone but thought I'd share.

Based on my research, the below columns are Read Only and are used by SharePoint Online internally for security and compliance purpose. And these columns cannot be modified by end users. Also, these are not site columns and are present only at the document library level.

ID of the User who has the Set the Protection Tag

Protection Action

Protection Tag

Protection Tag Modified

Record

_ComplianceFlags --> was removed

_ComplianceTag was renamed to ProtectionTag

_ComplianceTagModified was renamed to ProtectionTagModified
