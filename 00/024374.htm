dave peterson say:

You may get lucky.

When xl2003 and older saves the file, it saves it as a temporary file with a
funny name (8 characters--no extension) in the same folder. 

If the save is successful, xl will delete the original (or rename it to its
backup name (like "backup of book1.xlk)) and if that's successful, xl will
rename the funny named file to the original's name.

I think that this is the same way xl2007+ works, but I haven't used that enough
to notice/check.

If you're lucky, you may be able to use a recovery program that can retrieve
that old file.

Lots of these programs have been developed to recover picture files from SD
cards (I think).

But maybe they'll work for you.

You may want to look at:
recuva (http://www.piriform.com/recuva)
or
PhotoRec (http://www.cgsecurity.org/wiki/PhotoRec)

Heck, you may find lots more from a search of google.

But before you do too much, you should stop writing stuff to this harddrive.
You don't control what sector of the harddrive gets overwritten when you save a
file (or download files -- even cached files for your browser!) and any of those
could overwrite the physical location of that original excel file.

ps.  I use Mozy (www.mozy.com) for online backups.  (And I have USB harddrives
for local backups.)

The free version of Mozy allows 2GB (+ more if you can get others to sign up
using you as a reference).

The 2GB won't backup all my pictures and music, but it's big enough for my
important data files (excel/word/ppt...).

1. Run the Remove Office tool (/Applications/Microsoft Office 2008/Additional Tools/Remove Office) and remove ALL versions of office that show up.
2. Go to Hard Drive/Library/Application Support. Delete the Microsoft folder if it exists.
3. Go to Hard Drive/Library/Fonts. Delete the Microsoft folder if it exists.
4. Go to Hard Drive/Library/Preferences. If anything in there has Microsoft in it, remove it.
5. Go to Hard Drive/Library/Receipts. Remove all items that begin with "Office2008_"
6. Go to Hard Drive/Applications. If the Microsoft Office folder still exists, delete it.
7. Go to the /Users/ folder, where is the name of the user that installed Office. I'll abbreviate it from here on as "~/"
8. Go to ~/Library/Application Support and remove the Microsoft folder
9. Go to ~/Library/Preferences. Remove anything that starts with com.microsoft and the Microsoft folder.
10. Finally if you are on Snow Leopard, go to the Finder. From the Go menu, select "Go to Folder...". Type in: /private/var/db/receipts and press return. Delete any files with Microsoft in the name.
11. Empty the trash.
12. Reinstall Office.

What are the dangers of using Precision As Displayed? - Microsoft Community

webadr31 say:

What are the dangers of using Precision As Displayed?

I am working with % increases to currency calculations and would like Excel to ignore all decimals beyond 2 (cents) rather than round values past that point.  I have searched a number of answers here on rounding and I see many people mention Precision as Displayed but then they simply state they don't recommend it.  However, I do not see any detailed explanation of why it is not recommended?
joeu2004 say: Webadr31 wrote:
I am working with % increases to currency calculations and would like Excel to ignore all decimals beyond 2 (cents) rather than round values past that point.  I have searched a number of answers here on rounding and I see many people mention Precision as Displayed but then they simply state they don't recommend it.  However, I do not see any detailed explanation of why it is not recommended?

The primary problem with PAD is that it is error-prone.

* It irreversibly changes constants.  And PAD applies to the entire workbook.  So when you first set PAD, you run the risk of losing precision inadvertently because you neglect to review and correct all cell formats before setting PAD.  This can have a dramatic and cascading effect on dependent formulas.

   Note:  Changes to the precision of formulas are reversible; simply change the format.

* Similarly, after PAD is set, you run the risk of irreversibly losing precision inadvertently because you neglect to set the cell format properly before entering
* There is no way to enter or calculate values with one precision and display them with less precision.  In financial applications, for example, it is not uncommon to display interest percentage rates to only 2 basis points, but you want the actual value to be the full precision.

* PAD applies only to the final value in a cell.  Consequently, PAD does not solve all problems that arise due to the way that Excel represents numbers and performs arithmetic (binary floating-point).  For example, IF(10.01-10=0.01,TRUE) still returns FALSE(!).

   Arguably, that simply means we must still use explicit rounding selectively.  But many people lack the skills to know when explicit rounding is needed v. when they can rely on PAD to do the job for them.

* PAD does not have a noticable effect on cells formatted as General.  So in financial applications, for example, if you neglect to format some cells with 2 decimal places, you might continue to get "mysterious" results even though PAD is set.  This is easily rectified.  But many people lack the skills to debug such problems, and it always takes time.

   Of course, the same can be said for relying on explicit rounding.  The difference for me is:  once I am in the habit of using explicit rounding, failure to use it is easy to see; just select the cell and look at the Formula Bar.  Formatting errors are harder to spot.

* But in fact, PAD actually does have an effect on cells formatted as General:  values are rounded to 15 significant digits.  That applies to other numeric formats as well.  This might seem unimportant, especially for decimal fractions.  But for example, integers between 1,000,000,000,000,001 and 9,007,199,254,740,992 (2^53) can be represented exactly in Excel, even though Excel displays only the first 15 significant digits rounded.  However, PAD changes the actual value to the first 15 significant digits rounded, at least.

monique tang msft say:

Hi Craig,

If you want to set Current Mailbox as the default search scope for the main mailbox, you can click File> Options> Search> choose Current mailbox> OK to change it.

It’s not feasible to set Current Mailbox as the default search scope for the shared mailbox. Only Current Folder, All Mailboxes, All Outlook Items are for your choice. All Mailboxes is set as the default scope and you need to manually change it.

If you choose Current folder under Options, Current Folder can be set as default for both the main mailbox and the shared mailbox.
