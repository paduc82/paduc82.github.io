suzanne s. barnhill say:

From your description, the check box that has been used is the one from the Legacy Forms rather than the check box content control. A form that uses legacy form fields has to be protected for forms. You have two ways forward:

  1. If the form contains other content controls, substitute a check box content control. These are found on the Developer tab.
  2. If the form contains other legacy form fields, protect the document as follows:
  • On the Developer tab, in the Protect group, click Restrict editing. This will open the Restrict Editing task pane.
  • In the task pane, under "Editing restrictions," check the box and choose "Filling in forms."
  • Click Yes, Start Enforcing Protection. You don't have to add a password, just click OK to close the Start Enforcing Protection dialog.

Create a user defined function in a standard module>  It could look something like this untested air code:

 

Public Function DelNonNum(S As Variant) As Variant

Dim K as Integer
If IsNull(S) Then Exit Function

 

For K = 1 To Len(S)

   If Mid(S, K, 1) Like "#" Then DelNonNum = DelNonNum & Mid(S, K, 1)

Next K

 

End Function

Hi Netivity ITD,

For the first question, I would like to confirm if you want to know whether there is such kind of reporting tool in the Office 365 portal and Exchange admin center.

If so, it is not available. However, we can use message trace and eDiscovery to find emails and the result will show where an email is from and to. What’s more, we can use eDiscovery to check how many emails are sent. One more setting is named journaling. A journaling rule can be created to record inbound and outbound email communications.
