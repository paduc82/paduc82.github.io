lemp say:

Autoruns shows a lot of information, and as tempting as it may be, deleting items other than in the "Logon" tab can be potentially quite dangerous.

The colors in Autoruns:

Blue: shows the location where the following group of entries is found - usually a Registry key

Pink/Red: no Description or Company Name in the file version resources or no version resource (always true for script files). If the Verify Signatures option is on, file is unsigned.
Yellow: file not found in the expected location
Green: shows changed entries when using Compare from the File menu

The "Verify Signatures option" is under Options > Scan Options.  If you select that, most of the lines highlighted in red will become "Verified" and the red will be removed.  The ones that are left are either scripts (like your "gathernetworkinfo.vbs") or they have no digital signature or a signature that is invalid for one reason or another.

Generally, an item highlighted in yellow refers to a registry entry that points to a file that is no longer there.  These are usually the sort of "orphaned" registry entries that registry cleaning programs will delete.  My suggestion is to leave them alone unless they result in one of those pop-up warnings that complain that a file can't be started.  This generally only happens with entries in the "Logon" tab.

The entry for igfxcui (which is also on my Win 7 Pro system) is related to your Intel graphics.  On my system, this entry points to a file named igfxpph.dll although according to some Google results, on some systems it points to igfxdev.dll.  In either case, the file is missing from the location where the registry entry is pointing.  I don't know why or what exactly these files do, but my suggestion is to ignore this entry in Autoruns unless you are experiencing some difficulty in accessing your Intel Graphics control panel through the right-click context menu (i.e., when you right-click on a blank spot on your Desktop).

The GatherNetworkinfo.vbs script is part of Windows 7 and is, in and of itself, safe.  The comments you quoted (similar to the post from the same person at http://jeffwouters.nl/index.php/2015/12/you-can-trust-gathernetworkinfo-vbs/ display, IMO, a fairly high level of paranoia.  However, if it would make you feel more secure, go to C:\Windows\system32 and rename the file from GatherNetworkinfo.vbs to GatherNetworkinfo.xxx (but make a note somewhere about what you did so that you'll be able to rename it should you ever feel the need to run this script).

You can read more about the script here https://answers.microsoft.com/en-us/windows/forum/windows_7-security/does-anyone-know-what-gathernetworkinfovbs-is-its/63a302a6-cf69-4b9a-a3ef-4b2aff1b2514 particularly the long December 11, 2013 post by Paul-De and here https://www.verboon.info/2011/06/the-gathernetworkinfo-vbs-script/
