maor bracha say:

When adding permissions using the ECP portal, the added mailbox would be auto-mapped, causing it to appear in Outlook automatically.

However, you may override this configuration with a PowerShell command.

To add permissions with automapping disabled:

Add-MailboxPermission -Identity <Mailbox to be opened> -User <Mailbox to get permissions> -AccessRights FullAccess -AutoMapping:$False

In case you already added permissions with automapping, you would have to revoke them and reassign using the command provided.

Following is a modification of Gord's VBA code. However, I am inclined to think that it might have been better to use a lookup table on a worksheet in conjunction with VBA code so that the user could easily modify. If you want a lookup table then let me know.

'Original code from Gord Dibbon

'Original code from Gord Dibbon
'Contents of fruit array modified by OssieMac to insert a formula in the cell in lieu of a value
Private Sub Worksheet_Change(ByVal Target As Range)
    Dim r As Range
    Dim rr As Range
    Dim iVal As Variant
    Dim vals As Variant
    Dim fruit As Variant
    Dim i As Long
   
    Dim test As String
    Set r = Range("A1")          'single cell with dv list
    On Error GoTo ws_exit
    Application.EnableEvents = False
    vals = Array("red", "green", "blue", "purple")
    fruit = Array("=(C1-5)/2 & " & """ Apples""", "=(C1-5)/2 & " & """ Lime""", "=(C1-5)/2 & " & """ Berry""", "=(C1-5)/2 & " & """ Eggplant""")
   
    For Each rr In r
        iVal = ""
        For i = LBound(vals) To UBound(vals)
            If rr.Value = vals(i) Then
                iVal = fruit(i)
            End If
        Next
        If iVal <> "" Then
            rr.Formula = iVal
        End If
    Next
ws_exit:
    Application.EnableEvents = True
End Sub
