Get Started with Bash on Ubuntu on Windows - Basics for the new user. - Microsoft Community

andre da costa say:

Get Started with Bash on Ubuntu on Windows - Basics for the new user.

Technical Level : Expert

Summary

The Windows 10 Anniversary Update promises to be a developer friendly revision of Microsoft's desktop OS. We previously showed you how to setup and configure the Windows Subsystem for Linux on Windows 10. Bash on Ubuntu on Windows 10 gives users access to Linux command line tools. These command line tools provide deep access to a repository of applications developers might need; right from within the comfort of the Windows environment. If you are new to Linux or the Command Line, this can come across intimidating. I must admit, my knowledge of this part of Windows is basic, so I am learning with you too. In this article, we try out some of the basics, such as launching the command line and trying out a few commands.


Details

First, you need to setup the Windows Subsystem for Linux and install Bash on Ubuntu on Windows 10. Check out the following Wiki article for instructions:

Unix shell and command language written by Brian Fox for the GNU Project as a free software replacement for the Bourne shell.[4][5] Released in 1989,[6] it has been distributed widely as the shell for the GNU operating system and as a default shell on Linux and OS X. It was announced during the 2016 Build Conference that Windows 10 has added a Linux subsystem which fully supports Bash and other Ubuntu binaries running natively in Windows.[7]

Source: Wikipedia 

With the availability of Bash on Ubuntu on Windows 10, there are now three command line families available in Windows 10.

  • Command Prompt
  • Windows Powershell
  • Bash

How to launch Bash on Windows 10?

There are a couple ways you can launch Bash on Windows 10.

Using the command prompt, press Windows key + X then click Command prompt. At the command prompt, type: bash then hit Enter.

Click Start > All Apps > under the letter B click Bash on Ubuntu for Windows

If you want to be able to access the local file system, press Windows key + X > Command Prompt (Admin) then type bash at the prompt.

You can also access Bash from the Windows Powershell command line too. I notice Bash works best under Windows Powershell or Bash itself, some of the commands just don't work in Command Prompt or require multiple attempts.

Lets try out some basic commands.

If you want to display the current date on screen type: date at the command prompt then hit Enter

To find out the currently logged in user, type whoami then hit Enter.

To clear your screen, press Control key + L

If you want to see your current calendar, type: cal at the command prompt then hit enter.

To list all files in the current directory, type ls at the command prompt then hit enter. Here you will see the contents of your personal folder.

You can access the manual for a particular application, type: man then application name, ie. man cal

To change your password, type: passwd then hit Enter. Enter the new password then hit Enter again.

I notice some programs are not installed. To install popular apps remember this command: apt-get install applicationname

Here is an example: apt-get install xterm

Change a directory - if you want to go to a different part of the file system, type cd which stands for change directory, then the path /mnt/d, ie. cd /mnt/d will mount the local drive name D:\

Bash includes a Text Editor called VI (short for visual), which you can use to create and modify files. To start VI, at the command prompt, type VI then hit Enter. When you start VI all you will see is a command line interface, known as command mode. To start entering text, press the Enter key or Backspace key.

Unlike WordPad or Microsoft Word, using VI requires memorizing certain commands. When you want to make edits, you need to enter Command Mode. This will allow you do things like insert text to the left of the cursor location, insert text in a new line, overwrite text or open a new line above the new one. You should also know that, these commands are case sensitive.

  • i - insert text to the left of the current cursor location
  • I (capital i) - Insert text to the left of the current line
  • a - append text to the right of the cursor
  • A - Append text to the end of the current line.
  • R - Replace (type over) existing text
  • o - open a new line below the current line and move the cursor there
  • O - Open a new line above the current line and move the cursor there

Anytime you need to apply any of these editing options, you need to enter command mode. To do that, press Esc then apply the command.  Please remember, the commands are case sensitive.

To save your work, enter command mode (Esc)

Press the colon key (:) then type the letter w then hit Enter

To quit the editor, press the colon key (:) then type the letter q then hit Enter

Please note, its quite buggy at the moment, so you might not be able to write your file to disk.

So that's a quick look at using Bash on Windows for the first time. This is course just the tip of the iceberg, but if you feel there is something a beginner might want to know, be sure to leave a comment.

graham mayor say:
ActiveX controls are a throwback to when Word was used (hopelessly) to create web pages. They are best avoided. You can make check box form fields behave like radio buttons by grouping them and using a macro - http://word.mvps.org/faqs/tblsfldsfms/ExclusiveFmFldChbxs.htm. - then protect the form which will stop them from being deleted.
If however you want to permanently fix the values so they are no longer buttons, you are going to need code to replace the fields with checked/unchecked box characters, however ultimately it is not possible to prevent a determined staff member from changing anything in a Word document you allow him/her access to. If the person reviewing/approving the results of the form does not have to extract the data for use in another document or application, then the simplest solution is to send the report as a PDF format document.
