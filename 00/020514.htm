/** moderator note: original title "vba for bulk data to segrigate" **/
jeeped say:
I've written two routines for you. The first calls the second one or more times. I've set it up to be called for both cities that were in your sample. The second does all of hte work of constructing a new worksheet based upon the name provided and segregates the data according to the city name given. Since this is operating with wildcard filters, the city names must be unique; one name cannot be contained within another name.

Sub mcr_Split_Data()
    Call mcr_Single_Data("Bangalore")
    Call mcr_Single_Data("Gangavati")
End Sub

Sub mcr_Single_Data(sWS As String)
    Dim w As Long, app As Application
    Set app = Application
    app.ScreenUpdating = False
    app.EnableEvents = False
    app.DisplayAlerts = False
    For w = 1 To Sheets.Count
        If LCase(Sheets(w).Name) = LCase(sWS) Then
            Sheets(w).Delete
            Exit For
        End If
    Next w
    Sheets("Data").Copy After:=Sheets(Sheets.Count)
    Sheets(Sheets.Count).Name = sWS
    With Sheets(sWS)
        With .Cells(1, 1).CurrentRegion.Cells
            .AutoFilter Field:=4, Criteria1:="<>*" & sWS & Chr(42)
            If CBool(Len(.Cells(2, 1))) Then _
                .Cells.Offset(1, 0).EntireRow.Delete
            .AutoFilter
        End With
    End With
    Sheets("Data").Activate
    app.DisplayAlerts = True
    app.EnableEvents = True
    app.ScreenUpdating = True
    Set app = Nothing
End Sub

You will not see mcr_Single_Data in the Macrtos dialog as it requires a parameter. Just make duplicates of the Call mcr_Single_Data("Bangalore") code line in mcr_Split_Data and change the name of the city to what you want to parse out.

It is probably of worth to point out that if a worksheet already exists, it is deleted and a new one with the city name is constructed with the city data.

joel ivory johnson say:

VZW had a history of locking down the GPS hardware in their devices (they also used to lock down the bluetooth and other functionality in the device, but that's a different discussion). In times past only the VZW navigator was the only application allowed to use the GPS. Any and all other applications that tried to use it would fail. About a year ago VZW lightened up and started releasing firmware updates that allowed users to open up the GPS hardware to other applications.  The updates would initially default to keeping the GPS hardware locked until the user went into a special program on the phone to unlock it.

You will need to ensure that you are running the latest firmware version available for your phone and if you are you will need to find the application on the phone to unlock the GPS hardware. When you do this you'll get some warning about potential loss of privacy by allowing third party applications to use GPS. After you agree to that your othe rGPS based applications will work.
