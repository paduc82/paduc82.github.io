john mcghie say:
I explained this to Peter:  You need to use a "Reset Style", commonly known as "The Margaret Aldis Method".

I don't recommend it, particularly for Peter, who was intending to use a Master Document and multiple sub-documents with EndNote citations.  He also has the issue that while Heading 1 should not "display" a number, it should also not have a number in the TOC.

Mac word simply doesn't have the stability or ruggedness for this kind of complexity: it will blow up.

To do what he wants, you have to use Daniel's method: set the numbering "white" or "hidden" on the top level.  But the numbering must EXIST, otherwise it cannot appear in Heading 2 where he wants it.

Then you have to fudge the TOC!  To do that, you add only Headings 2, 3, and 4 to the Table of Contents.  PLUS switch on "Table Entry Fields" in the Table of Contents dialog.  Insert>Index and Tables>Table of Contents>Options...  In the Options dialog you will see "Build table of contents from" and at the bottom of the Dialog, "Table entry fields".  Switch that on.

Now:  There is a truly ancient field ion Word called the "TC" (Table of Contents) field.  It pre-dates the "automatic" TOC that we now use almost always.  It enables you to determine exactly the string that will appear in the TOC.  You "tag" the text that you want to appear in the TOC for each Heading 1 with a TC field (like tagging an index).  Then set those tags to be compiled at Level 1.

The text you decide to show in the TOC can be completely different to the text of the heading.  The TOC entry will show what you type, regardless of what the heading contains.

This hides the number from the TOC, and from the printed page, while allowing it to continue to exist!

See here: http://office.microsoft.com/en-us/word-help/field-codes-tc-table-of-contents-entry-field-HP005186197.aspx

And here: http://office.microsoft.com/en-us/word-help/table-of-contents-iii-use-fields-to-create-a-toc-and-create-multiple-tocs-RZ010266698.aspx?section=5

One of my favourite things about Word is that generally, if you can think of it, Word can do it.  But some methods require a lot of work and knowledge of Word at "the bit level".

Those of us who started on mainframes and WordPerfect have this level of knowledge and can use these very arcane and deeply buried features.  But most users these days are far too busy to acquire this level of knowledge.  Which is nearly impossible anyway, because Microsoft has taken it all out of the Help :-)

    Unable to boot to windows, Windows boot partition is accidentally - Microsoft Community

    rocky_78 say:

    Unable to boot to windows, Windows boot partition is accidentally overwritten

    Hi There, I've accidentally overwritten my windows boot partition and hence not able to boot to Windows,

    Can you please help with steps to recover windows boot partition? Right now it is not able to find /Bootloader/bootmgfw.efi file

    ***Post moved by the moderator to the appropriate forum topic.***

    don cuthbert say:

    Assuming that you mean the System partition, it can be rebuilt quite easily. The Windows/Boot partition is actually where Windows and all of your User data normally resides. If you have deleted all that, only recovery software could recover anything. Also, anything recovered needs to be written to a different disk or storage media than it was recovered from.

    How to rebuild the System partition (on a UEFI computer):

    Create any version of Windows 10 install media on a bootable USB stick and boot to Windows setup from USB. Once Windows setup loads, you can use SHIFT-F10 to get a Command Prompt. From there you can run a series of commands to rebuild the System partition (diskpart; bootsect; bcdboot).

    https://www.microsoft.com/en-us/software-download/windows10?wa=wsignin1.0

    diskpart

    list vol

    sel vol 3 <-- this should be a FAT32 partition, but if you formatted it, it could be something else. your data on it.

    del par override

    create par efi

    format quick fs=fat32 label=System

    assign letter=S

    list vol

    exit

    bootsect /nt60 S: /force

    bcdboot C:\Windows /s S: /f UEFI

    bcdedit

    Now exit the Command Prompt and exit the Windows setup to restart the computer. It should boot into Windows 10.
