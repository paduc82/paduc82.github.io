kantx say:
If Word is correctly installed, there are two things that can go wrong:

  • Your Normal.dotm template
  • Your personal preferences and settings

There is no way to bypass one or the other with Shift, so you need to do this manually. It involves some detective work. You can try the following steps in sequence, or do them separately or in reverse. The idea is to identify which is corrupt: your Normal template, your preferences or both.

1) Bypass the Normal template
  • Quit Word
  • Go to the Finder
  • Select Go > Go to Folder
  • In the field, type this: ~/library (don't forget the twiddle ~)
  • Locate this folder: Application Support/Microsoft/Office/User Templates
  • Drag file Normal.dotm to the Desktop (in case something goes awfully wrong, you can place it back)
  • Start Word and test
If Word works OK, you are done. If not, the fault should be your preferences, although there is a chance that both parties are involved. So, even though at this point you could restore your Normal template to its natural folder, better leave it on the Desktop for now.

2) Bypass your Word preferences
  • Repeat the first four bulleted steps from before
  • Locate this folder: Preferences
  • Drag file com.microsoft.Word.plist to the Desktop
  • Start Word and test
If Word works OK, you are done. If not, try dragging other preference files, such as com.microsoft.office.plist. You can drag anything starting with com.microsoft… out of that folder, but the less things you drag out, the less your current preferences will be changed.

Once you get Word running (and you will), you can try restoring Normal.dotm to its former glory in its original folder and check if Word still runs.

Finally, dispose of the Desktop files that caused the problem.

Your VM can probably be permanently activated, as long as you use the same VM.  Even using the VHD hard disk from that VM on another VM may not activate, as Windows is licensed per device, whether virtual or physical.

boblarson say:

We use Citrix and both Access 2007 and 2013 work fine in Citrix.  You need to set the Trusted Locations in Citrix for each user and so we have a script that runs a reg file each time the user starts an Access app.  It sets it for them.  See the contents of the .reg file below that we use and the batch file that calls it.  We also put a copy of the front end into a temp folder for the user so that they can each have their own copy of the frontend.  The batch file copies the file each time the database is started (we publish the batch file and have the icon look like the Access app).

 

 

The REG File:

 

Windows Registry Editor Version 5.00

[HKEY_CURRENT_USER\Software\Microsoft\Office\12.0\Access\Security\Trusted Locations\Location0]
"Path"="E:\\bky_temp\\"
"AllowSubfolders"=dword:00000001
"Description"=""
"Date"="9/25/2012 1:00 PM"

 

The BATCH file:

 

@echo Please wait while the version of the database you requested is updated.

@echo off
reg import e:\Access2007TrustedLocation.reg
MD E:\BKY_temp\%username%

copy R:\FE\Current\Consolidated_Reports.mdb E:\BKY_temp\%username% /y

Start /max "E:\Program Files\Microsoft Office\Office12\MSACCESS.EXE" "E:\BKY_temp\%username%\Consolidated_Reports.mdb"

exit

macropod say:
If you insert a single-column table into the document, giving it a fixed column width, most content inserted into that table will be constrained to fit the column width; what doesn't fit (e.g. text with tab-stops outside the print width or -ve left/right indents) will be hidden from view.
