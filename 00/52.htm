If I'm understanding you, you have created text boxes for each desk area on the floor plan. So you want the text box to display not just the Location number but also the occupant's name.

It is returning all the records because you put a criteria from the UPTO table. That negates the LEFT JOIN making it an INNER JOIN.

john w. vinson say:

The fact that you have a criterion on NOM_DATE is causing the join to be effectively an INNER JOIN - if there is no record in UPTO, then the NOM_DATE field will be NULL, which is not "between" anything. Try changing the criterion to

(UPTO.NOM_DATE Between #10/1/2015# And #9/30/2016# OR UPTO.NOM_DATE IS NULL)

If you want the query to work next year, you may want to consider replacing the literal dates; e.g.

(UPTO.NOM_DATE >= DateSerial(Year(Date()) - 1, 10, 1) And UPTO.NOM_DATE < DateSerial(Year(Date()), 10, 1) OR UPTO.NOM_DATE IS NULL)

How can I create a column-line chart which does not apply a date - Microsoft Community

sikov say:

How can I create a column-line chart which does not apply a date axis?

I want to plot rainfall as a column chart with river level as XY scatter, with both sets of data correctly aligned.  This is a fairly standard plot in my line of work - I have done this in the past with previous versions of excel without any problem, but now all control of the axis type appears to have been taken away from the user when using a column chart (I'm using the Clustered Column).

The rainfall data has been provided at hourly intervals, but the gauged river levels are at different time intervals so there is a different number of items in each data set.  The common denominator is the date/time.

How can I regain control of the x-axis on the column chart so that it plots the column data as per the date/time value associated with it?
- If axis type "Date axis" is used, the lowest unit I can apply is days which groups my hourly data into daily.
- If axis type "Text axis" is used, all control of the start and end date/time goes out the window

The result is that I cannot get the two datasets to line up based on date/time.
e.j. gun say:

sikov,

 

There is a way!  It's just a bit complicated, but I've done the technique described below, and you can get (roughly) what you're trying to get.  I'll just describe the steps to go through, and you can experiment and see if it works for you.

The main "trick" to use here is to use Error Bars to simulate a column chart for the rainfall data.

 

Steps:

1.  Plot both sets of data on an XY-scatter chart.

2.  Move one of the sets of data to the secondary y-axis (to get better scales for each set of data)

3.  Select the rainfall data series and do the following sub-steps:

      a.  On the Layout Tab of Chart Tools in the Ribbon, select "Error Bars"

      b.  Click on "More Error Bar Options".  Horizontal and Vertical error bars show up on your graph.

      c.  Select the horizontal error bars and delete them.

      d.  Select the vertical error bars, and in the "Format Error Bars" dialog box, select "Minus".

      e.  Click on the "Custom" radio button for the "Error Amount"

       f.  Click on the "Specify Value" button

      g.  Set the "Positive Error Value" to zero (0)

      h.  Click in the Negative Error Value text box and delete what's in there.

      i.   "Drag" across your Rainfall data - the actual rainfall amounts only, not the dates.

      h.  Click Okay.  You now have very thin columns!  You can adjust their thickness by the Line Style Width option.

4.  Select the rainfall data series again.  Turn off the lines and markers.  Now you're left with only the error bars - which simulate columns.

4.  Set your x-axis scale to whatever you want it to be.

5.  You now have a simulated combo column/scatter chart.

 

HTH,

Eric
