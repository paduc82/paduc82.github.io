a. user say:

I used to have a NAS attached but it was in storage for a while.

I just attached it back onto the network and noticed it was no longer accessible.

I have a solution and it is a simple feature that has been disabled however please understand that the reason the feature was disabled in the first place by the team at Microsoft is because enabling this setting this allowed the Petya Ransomware attack to spread.

That malware used an exploit in the SMB protocol, or more importantly, the SMB v1 protocol. This is what gets used to communicate with older NAS devices and is commonly used when communicating across OS' etc.

If your computer is on a publicly accessible network (connected to the internet) and you do not have a proper firewall and have not blocked the ports from external traffic, you are taking a risk that your machine can be infected.

With that in mind, you do this at your own risk and as annoying as it is to have something stop working, this was done because many businesses had this setting enabled and also leave their server open to the web so you can remote into work or connect to the mail server for work emails. This is why so many businesses where impacted so in an update, it was most likely disabled for security purposes.

To enable this, you only need 1 setting switched back on.

In Win10, click on the Windows icon and search for "features"

You will get a result that say "Turn Windows features on and off"

Scroll down until you find the SMB 1.0 option.

I have been able to connect back to my NAS but I only enabled the Client. This allows my machine to connect back to a device using the SMB protocol but I have NOT enabled the Server as that would allow someone to connect to my machine using the SMB protocol.

After you click OK, you will have to restart you machine. The second I booted back in, I went to Explorer, Network and my NAS appeared. It did take a few seconds to load up but after that it was fine.

If you want more info on why this was disabled, do a search online for Petya SMB and should find all the information you need.

Sadly it is a flaw in the way the SMB protocol was designed and the very cross-platform connectivity that made it great was also the reason it was so easy to exploit and a few ransomware attacks have used this service.

The best advice is to log into your router and drop all inbound traffic from the web to ports 137, 138, 139 and 445 as they are the ports the SMB protocol uses but you will need those for internal comms. You still need it for internal tho so blocking it at the Windows firewall level may cause issues when it tries to speak to the NAS. bootrec /scanos

bootrec /fixBoot

Check if the issue persists.
