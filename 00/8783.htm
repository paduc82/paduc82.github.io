stefan blom say:

The code you are using creates a new list format each time you run it. You need to make use of a variable that points to a specific ListTemplate object stored in the document. The following code looks for a ListTemplate object called "mylist," sets a reference to it if it already exists, or creates it if it doesn't exist, and then it applies the list to the selection:

Sub TestCreateListTemplateObject()
Dim lt As ListTemplate
Dim s As ListTemplate
For Each lt In ActiveDocument.ListTemplates
If lt.Name = "mylist" Then
Set s = lt
Exit For
End If
Next lt

If s Is Nothing Then
Set s = ActiveDocument.ListTemplates.Add(Name:="mylist")
End If
With s.ListLevels(1)
        .NumberFormat = "AI-%1"
        .TrailingCharacter = wdTrailingTab
        .NumberStyle = wdListNumberStyleArabicLZ
        .NumberPosition = 0
        .Alignment = wdListLevelAlignLeft
        .TextPosition = InchesToPoints(0.75)
        .TabPosition = InchesToPoints(0.75)
        .ResetOnHigher = 0
        .StartAt = 1
        .LinkedStyle = ""
End With
    Selection.Range.ListFormat.ApplyListTemplateWithLevel ListTemplate:= _
        s, ContinuePreviousList:= _
        True, ApplyTo:=wdListApplyToWholeList
End Sub

Note that the best approach to auto numbering is linking it to a paragraph style (one for each numbering level). This is usually done better in the user interface, as discussed at http://www.shaunakelly.com/word/numbering/numbering20072010.html.

How Does One Print a "30-labels-per-page" "Labels - Microsoft Community

vwp1 say:

How Does One Print a "30-labels-per-page" "Labels report" from a single record?

How does one print a "30-labels-per-page" "Labels report" from a single record, using the wizard?

I wish to print my own "Return Address" labels.

ken sheridan say: First create a table Counters, with one column Counter of number data type, and fill it with values from 1 to at 100 say.  The important thing is that there must be at least the number of rows for the maximum number of labels to be printed at any one time.

Then create a query in which the 'Addresses' table returns only your own address, and which includes the Counters table, but without specifically joining this to the 'Addresses' table.  Restrict the Counters table to rows with a value of 30 or less, e.g

SELECT Addresses.*
FROM Addresses, Counters
WHERE AddressID = 42
AND Counter <=30;

This assumes that your address in the table has an AddressID value of 42 (amend the query as necessary).

By not explicitly joining the tables the query returns what is known as the Cartesian product of the two tables (a Cartesian coordinate is each of a set of coordinates describing the position of a point in relation to a set of intersecting straight axes).  This in effect joins each row of one table to each row of the other, so as the Addresses table is restricted to returning one row and the Counters table to those rows with values from 1 to 30, the query returns 30 identical rows of your address.

You then simply use the labels wizard to create a labels report based on this query.

    john korchok say:

    Have you tried saving template B as a theme, then opening presentation A and applying the theme to it? When you do that, it doesn't rename the duplicate layouts and add them, it adds the slide master and all layouts under it as a separate master, then applies all layouts that have the same name to relevant slides..

    Re: your comment about wanting to name text boxes. PowerPoint already has a tracking system built in. Each placeholder has an idx number that is applied when you create placeholders on a layout. If you create new placeholders by copying and pasting, chaos ensues because the pasted placeholders have the same idx number as the original. There are 2 solutions:

    1. Create all placeholders from the menu commands in the order in which they should be filled (usually from top to bottom and left to right), avoiding any copying and pasting, or
    2. After creating the layouts using copying and pasting, edit the XML to give each placeholder a unique idx number for that slide. I'm writing an article about this, but it not finished yet. You can read my inroduction to XML hacking here: XML Hacking: An Introduction

    Just a note on your use of a subhead text box. Since you show the subhead being reused in the main text box, it would be better to dispense with the subhead box and create the subhead as a new text level on the slide master. That way a subhead can appear anywhere in the text by using Increase/Decrease List Level to apply the subhead style. This approach would also eliminate the problem with PowerPoint adding some text to the wrong placeholder, since you wouldn't have an extra subhead placeholder any more.
