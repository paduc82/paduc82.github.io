ken sheridan say: The classic model for a bill of materials is by means of an adjacency list table, which models a binary relationship type between two instances of a single parts table which contains all atomic parts, assemblies and sub-assemblies up to and including the complete product (kit in your case).  The image below is of such a model:



In this Parts and Parts_1 are not separate base tables, but instances of the same table.

The above model is from BoM.zip which you can download from my public databases folder at:

https://onedrive.live.com/?cid=44CC60D7FEA42912&id=44CC60D7FEA42912!169

Note that if you are using an earlier version of Access you might find that the colour of some form objects such as buttons shows incorrectly and you will need to  amend the form design accordingly.  

If you have difficulty opening the link, copy the link (NB, not the link location) and paste it into your browser's address bar.

The generation of a BoM to an arbitrary number of levels involves recursive querying.  There are a number of standard algorithms for this, but, unlike client server products like SQL Server and Oracle, Access does not support recursive querying.  It can be simulated however, by progressively writing rows to a temporary table, and the BoM.accdb file in my demo illustrates this.

As you only want to generate a BoM to a fixed three levels, however, this can be done more simply by means of a query on multiple instances of the two tables.  The PartsTree.accdb file in my demo illustrates this, generating a BoM to a maximum of nine levels.  The query for this is as below in design view:




The PartsTree.accdb file also illustrates how to output the BoM as an indented report.

There is a much more efficient method using Joe Celko's Nested Set Model.  This models a tree, however, in which there is only one possible path between two nodes.  A BoM is not a tree in the mathematical sense, however, but a convergent graph, in which multiple paths between two nodes are possible.  Joe has in the past suggested that aliases can be used to differentiate multiple instances of the same part, but in all but the simplest models I suspect this could soon get out of hand.

barry xie msft say:

Hi Ssbtech,

 

A shared mailbox is a mailbox that multiple users can use to read and send email messages. Shared mailboxes can also be used to provide a common calendar, allowing multiple users to schedule and view vacation time or work shifts.

 

The reason why we use a shared mailbox is that with it, people in a group can communicate conveniently. Other advantages include:

1. It makes it easy for a specific group of people to monitor and send email from a public email alias like info@contoso.com or contact@contoso.com.

2. It allows a person in the group to reply to a message sent to the shared mailbox, and that reply will appear to be from the shared mailbox, not the mailbox of the individual user.

3. It allows people in the groupbased on their individual permissionsto access shared contacts and a shared calendar associated with the shared mailbox.

Open and use a shared mailbox in Outlook 2016 and Outlook 2013.

Shared mailboxes in Exchange Online.

This is not available in Word 2010. So you will have to look elsewhere (like e.g. Acrobat X).
