copying a site collection - Microsoft Community

jboylan63 say:

copying a site collection

Is it possible to copy an existing Sharepoint Online site collection to a new Sharepoint Online subscription.    

felix tao msft say:

Hi Jboylan63,

We couldn't copy the SharePoint Online site collection to another Office 365 subscription (including SharePoint Online subscription).

As a workaround, we can save the site as a template and then upload the template into another Office 365 tenant. And then use the template to create a new site collection. To do that:

1.Turn the Custom Script on by referring to Turn scripting capabilities on or off. It takes about 24 hours for the change to take effect.

2.Save the site as template:

a. If we haven't enabled publishing features on the site collection, we can go to Site settings>Site Actions>Save site as template.>Move to step b.

If we have enabled publishing features on the site collection, we can't see the Save site as template option. We can follow the steps below to save the site as template:

Open the SharePoint Designer 2013>Sites>Open Site>Input the site collection URL and open it>Click Site Option>Under Parameters, check if Value of the SaveSiteAsTemplateEnabled is True. If so, please click the  Save as Template button and move to step 3. If it is false, click Modify to change the value to True, and then click Save as Template

 

b.Input the required information and check the Include Content >Click OK to save it.>Click solution gallery to find the template>Click the template to download it to the PC.


2.Create a new site collection:

In the new Office 365 tenant, go to the SharePoint admin center>site collections>Click New to create a new site collection>In the Template Selection section, choose Select template later under Custom.

3.Upload and select the template:

a.When the site collection is created successfully. Open the site collection>In the Solution Gallery section, click Solution Gallery.>Under the SOLUTIONS tab, click Upload Solution>Choose the saved template and upload it>Click Activate to activate the tempate

 

b.Open the site collection again>In the Template Selection section, select the uploaded template under Custom>Click OK to use it.

Note: the limitation of site template size is 50M, and some features are not supported. For the details, please refer to What gets saved in a site template.

If the libraries or lists of the site exceed the limitation, we may not save the site template when checking the Include Content. In that scenario, we need to move the files of the library manually. We can use Open with Explorer feature to move the files from the previous library to the new one. For the lists, we can export the items to Excel and then copy the data to the new SharePoint Online lists.

If we have added some apps from SharePoint Store, we couldn't save the site as a template. We may need to recreate all the contents of the site in the new SharePoint Online and then manually move the data.

For other workarounds, any community members who have related experience are welcome to share your insights here.

[Updated by Felix Tao MSFT, 11:57 AM, September 4, 2016 (UTC)]

Thanks,
Felix

Here it is:

Ok so my IT guy came down and in less than 1/2 an hour I was back up and running.  I do not know exactly the steps he took but here is my evaluation of what he did to get things back up and running.  First he downloaded a utility called Process Explorer found at: http://www.microsoft.com/technet/sysinternals/utilities/processexplorer.mspx.  When run it will show you all of the processes that are running on your machine and what program they are associated with.  When we ran this on my machine it had a program called Open Case that had downloaded with an NBC viewer that had attached itself to my ActiveSync 4.5 program.  He then downloaded a program called Unlocker found here: http://ccollomb.free.fr/unlocker/.  This program allowed him to unlock the programs from the processes and then we uninstalled the OpenCase and NBC files.  There were one or two programs that would not completely uninstall from my machine and so he then downloaded a free trial of Absolute Uninstaller from: http://www.softpedia.com/get/Tweak/Uninstallers/Absolute-Uninstaller.shtml and we were able to uninstall and remove the remaining programs from the add and remove programs area in control panel.  After doing these things we reinstalled ActiveSync 4.5 and it is now working like a charm.  I would guess that this might work for most of you with problems once you have used the Explore Processed to find which programs had attached themselves to ActiveSync.  I hope it helps.

julie fka profjulie say:

PDF files are saved to a phone via the app that is used when the files are created or opened for the first time. Since you are viewing the files via Adobe Reader, they are saved in Adobe Reader's sandbox (or Adobe Reader's exclusive file storage). Access to an app's file storage is only available via the app's options. Of all the PDF reader app that are available for windows phone, the Adobe Reader app is the most limited in regards to allowing user access to the files that are stored in its sandbox.

Adobe Reader doesn't provide a way for you to transfer files from the device to any other device (like a computer). You might want to check out the One Reader app.
