joeu2004 say:

Jeff wrote: "wrong value for =1001^5? It returns a value of 1,005,010,010,005,000  Correct Value is 1,005,010,010,005,001"

Excel displays the wrong value. But it

You can see this with the following formula, assuming your formula is in A1:

=A1-TEXT(A1,"0.00000000000000E+0")-0

That is 14 zeros after the decimal point. The redundant -0 is not necessary in this case; but sometimes it is, as demonstrated below.

Excel formats only the first 15 significant digits (rounded).

But the precision of the internal binary representation allows for representing all integers from 0 to 2^53 exactly. 2^53 is 9007199254740992, which is much larger than your example.  Some larger integers can also be represented exactly.

FYI, if A2 has the formula =2^53, the formula =A2-TEXT(A2,"0.00000000000000E+0")-0 is an example of when the redundant -0 is necessary.

[EDIT] PS....  TEXT(...,"0.00000000000000E+0") is a method for rounding a value of any magnitude and number decimal places to 15 significant digits. So I am in the habit of writing that expression. Of course, for these example, we could simply use 1005010010005000 and 9007199254740990.
