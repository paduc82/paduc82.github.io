You can use something like this:

Sub GetNumberStringForLastItem()

'Macro created by Stefan Blom, MVP, April 2016
On Error GoTo errhandler
Dim LastRef As Long
Dim ListNumValue As String

LastRef = ActiveDocument.Content.ListParagraphs.Count

ListNumValue = ActiveDocument.Content.ListParagraphs(LastRef) _
   .Range.ListFormat.ListString
MsgBox "The last paragraph is numbered " & ListNumValue
errhandler:
MsgBox "No numbered item found"
Exit Sub

End Sub

webbrewers3 say:

First you have to figure out if it's a team site or a public site. A public site's default sub-domain will be:

https://<tenantname>-public.sharepoint.com. The tenant name is the name you chose when you created the O365 account and is usually also found in user id's as in user@<tenantname>.onmicrosoft.com. You can look in the SP Admin center or just enter the -public url in a browser. 

If it is a public site, your options are very limited. Msft offers no migration assistance whatsoever, despite their somewhat deceptive claims about this. As you found out, a Sharepoint site cannot be migrated to another platform and will only work on a server with Sharepoint installed. So I would download all your content asap so you dont lose it while you figure out where to build your next site. There's a free tool called httrack that does a good job of that at http://httrack.com. To build the new site, try out weebly and wix. Msft claims to have "partnered" with wix but this is bogus - their "partnership" offers no advantage over creating your own free account,

If it turns out the site is a team site, you can continue to use it if all your users have O365 licenses that include SP. If they don't, you can create free external licenses for them by sharing content with them (they will just need to convert any email address into a Microsoft account). Another option is for you to create a user access Msft account yourself, share the site with that id and then give the id and password to all your members. It's a little less maintenance that way.

Another workaround if the site is a public one I suppose would be to create a team site and move your public content to it  ahead of the deadline. This may be tricky though and you may need a SP expert to help.

No matter what you decide, I'd consider moving off Sharepoint at some point. As you found out Msft has no loyalty or respect for their customers and you have no need of them for the type of site you're running.

scottgem say:

I am trying to write a query to sum the check totals for that record and put it in a new table. When I run it, I am being prompted to "Enter Parameter Value" and when I do, it pulls the correct information

This is the wrong approach. As a general rule we do not store calculated values, You can use a query to display the Check total for any ReconcileID very easily using a Group By query:

SELECT ReconcileID, SUM(CheckAmount) AS CheckTotal

FROM Checks

GROUP BY ReconcileID;

Lets call this query qryCheckTotals. No if you want to see the total for a specific ReconcileID, you can use:

SELECT ReconcileID, CheckTotal

FROM qrCheckTotal

WHERE ReconcileID = [Enter ReconcileID]:

This will prompt you to enter a ReconcileID when you run the query.

But even better is to reference the ReconcileID from a form:

SELECT ReconcileID, CheckTotal

FROM qrCheckTotal

WHERE ReconcileID = Forms!formname!controlname:
