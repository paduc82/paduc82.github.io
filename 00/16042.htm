ronrosenfeld say:

You just change the LineFeed character (which I initially used to separate multiple products) to a comma.

Option Explicit
'Set reference to Microsoft VBScript Regular Expressions 5.5
Function myProduct(S As String) As String
    Dim RE As RegExp, MC As MatchCollection, M As Match
    Dim sTemp As String
    Const sProduct As String = "product\s*:\s*(.*)" 'regex to extract product names
    
Set RE = New RegExp
With RE
    .Global = True
    .IgnoreCase = True
    .Pattern = sProduct
    If .Test(S) = True Then
        Set MC = .Execute(S)
        For Each M In MC
            sTemp = sTemp & "," & M.SubMatches(0)
        Next M
        myProduct = Mid(sTemp, 2)
    End If
End With
End Function

Outlook 2013 Ignores CSS Margins in Stationery Themes - Microsoft Community

bartdurbin say:

Outlook 2013 Ignores CSS Margins in Stationery Themes

Example:  When using a stationery theme, such as NOTEBOOK, your email message background has a spiral notebook image.  In the NOTEBOOK.HTM file, the CSS styling contains the following: 

body
{
font-family: "MS Sans Serif", "sans serif";
font-weight: normal;
font-size: 12pt;
color: black;
margin-left: 50px;
}

The "margin-left" attribute is what pushes the text over to the right so the text of the email message is not overlapping the spiral edge of the "notebook."  In Outlook 2010, this works.  However, in Office 2013, the CSS margin attributes (and padding, too) are completely ignored!!!

Outlook 2013 comes with the same stationery themes as Outlook 2010, yet Outlook 2013 cannot display them properly.

What is the official fix for this?  While stationery themes aren't critical for sending emails (I advised the user to have no theme set at all for now), I can imagine this issue being very far reaching across any industry that relies on normal HTML and CSS formatting in their emails.  Why would margins and padding be ignored?  They are two of the most basic fundamentals of CSS styling.

If there is currently no fix available, when can we expect one?
