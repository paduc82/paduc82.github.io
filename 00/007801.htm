dim db as dao.database

dim td as dao.tabledef

dim fld as dao.field

set db=currentdb

on error resume next  'I think next loop may fail if a field has no description

You can use a select error handler to resolve this issue:

 

On Error GoTo err_handler  'Loop will fail if no description. This will send to handler which runs for no description

 

err_handler:
Select Case Err.Number
    Case 3270 '// No description. Print only field name then resume sub
        Debug.Print fld.Name
        Resume Next
    Case Else    '// Other error. Whatever you usually use for unhandled error
        strMsg = "An error occurred." & vbCrLf & "Error # " & Err.Number & " - " & Err.Description
        MsgBox strMsg, vbExclamation
End Select
