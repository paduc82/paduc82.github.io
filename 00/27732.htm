jeeped say:
Assuming that the first column in membership is the one holding the values to lookup B5 then in H5,

=VLOOKUP(B5, membership, 3, FALSE)

If there is a chance that no exact match will occur and you want an empty string (e.g. "") in the cell for no match then add an IFERROR() wrapper,

=IFERROR(VLOOKUP(B5, membership, 3, FALSE), "")

If the first column in membership is not the column to lookup B5, then an INDEX(MATCH()) pair should be used. This is a bit more complicated with named ranges. If the column to lookup B5 was the 5th column in membership and you still wanted to return the value in the 3rd column, then in H5,

=INDEX(membership,MATCH(B5, INDEX(membership, , 5), 0), 3)
... and,
=IFERROR(INDEX(membership,MATCH(B5, INDEX(membership, , 5), 0), 3),"")

how to use vlookup

A vlookup looks for a value in the leftmost column of a table and then returns a value from the row that the searched for value is found (how far along that row is determined by you).

The 4 parts to a vlookup formula:
  1. SearchFor this is the value that is being searched for. In our dictionary example it would be the word ‚Äúelucidate‚Äù.
  2. WhereToSearch this is the range in which to search and in which the answer lives. In our example, it would be columns A to C.
  3. WhichColumn this is the column in the WhereToSearch range which has the answer in it. In our dictionary example, it would be the third column.
  4. NearOrExact you decide whether the vlookup should search for a close value or an exact value (in our dictionary example we would set this to false as we do want it to search for the exact word ‚Äúelucidate‚Äù).
Structure of the vlookup formula:

=vlookup ( SearchFor , WhereToSearch , WhichColumn , NearOrExact )

brian tillman [outlook mvp 2007-2019] say:

When you're connected to an Exchange server, it is Exchange that informs Outlook when new messages arrive.  Outlook doesn't poll Exchange like it might poll an IMAP or POP server.  As far as I can tell, Outlook can't control when messages show up in the Inbox.  Even if your send/receive group explicitly excludes the Exchange account, Exchange will still tell Outlook when messages arrive on its own time table.

I seem to recall that there is a registry setting that can make Outlook poll Exchange, but I think that's reserved for RPC over HTTPS instances (i.e., Outlook Anywhere).  See the reference to HKLM\System\CurrentControlSet\Services\MSExchangeIS\ParametersSystem\Maximum Polling Frequency here: http://www.informit.com/articles/article.aspx?p=170892&seqNum=3

Thanks in advance for the slap upside the head to remember this.

jay freedman say:

What you've already done (clicking OK in the password window while the boxes are empty)should have removed the password protection.  I can't tell you why it didn't, but one of these methods will probably do it.

  • Unprotect the form, using the password. Press Alt+F11 to open the VBA editor and then press Ctrl+G to open the Immediate window at the bottom right. Paste this into the Immediate window and press Enter at the end of the line:

       ActiveDocument.Protect Type:=wdAllowOnlyFormFields, NoReset:=True, Password:=""

  • Open a new blank document. Use the Insert > Object > Text From File command and choose your form document. The body of the form will the inserted, unprotected and without a password. Protect it and OK the password window with the boxes empty.
