rohn007 say:

These "nuisance" messages are Informational warnings or Error messages. Each one will be controlled in a different place, probably somewhere in: Office button / Options. Some will not be "controllable".

To give you answers we need to know the exact text of each message and which application it applies to. Some information about what you are doing to trigger the message would help too.

For example, in Word, one place you can control a "there have been changes made" type message is Office button / Word Options button / Advanced option / Save section / "Prompt before Saving Normal Template" option. This will be triggered if you make changes to your default word configuration. Word wants to know if you want the changes to apply to future sessions.

If you are getting "there have been changes made" type messages for documents that you have simply opened and not actively edited, that  implies that you have fields or links in the document that have been automatically updated when you opened the doc, creating a change that optionally has to be saved.

One place this type of change may happen is if you do a Print Preview. Again in Word Options / Display option / Printing Options / there is an option to "Update fields before printing" and "Update linked data before printing".

FYI: Sometimes the "there have been changes made" messages are triggered by events you are not even aware of. A while back, on a corporate network I started getting the "there have been changes made, do you want to save Normal.DOT" type message when I simply opened word and closed it. Turned out that the company had installed a new PDF printing tool that was automatically downloaded to my computer. But they didn't tell us about it. I kept saying no because I knew I hadn't made any changes, and that type of change can be a symptom of a malware attack. It took a while for the LAN admins / help desk to figure out the cause and effect relationship between the 2 events. Once it was clarified, I said yes to saving the change and no longer got the message.

briantassell say:

Hi

Are we allowed to do file history backups to USB Memory sticks. When I try I get the error below. I have tried to format the USB with both NTFS and FAT, with the same result. I have no problem copying files to this USB. Thanks in advance for any tips. (Build 10576)

You can use a USB stick to do File History backups, but your screen shot says that you are doing a System Image (not File History), for a System Image you need to do it to a removable drive i.e. a USB external drive, a different internal drive or a network drive.

A System Image is a snapshot of your system, also gives the option at the end to create a recovery disc. The File History file can then be used to take a copy of file changes after the System Image was taken, giving a total recovery for the system back to the latest point.
