bob jones aka: cybertaz say:

As I replied to Joe, the screen shot will do you no good, but the 1st conversation you refer to does answer the question... at least as much as possible without more descriptive details about the documents involved.

Office 2008 does not have a Ribbon. In that version of the program, Header & Footer tools are a contextual component of the Formatting Palette as explained in my reply to the OP of that question. Display it by going to View> Toolbox> Formatting Palette, or by clicking the Toolbox button on the Standard Toolbar. When the Formatting Palette is displayed the Header & Footer Tools will appear once you either go to View> Header & Footer or double-click in a Header or a Footer area.

Alternatively, you can go to Format> Document - Layout as Phillip suggested in that same conversation.

As for the reference to Sections... All Word documents consist of at least 1 Section. It may have more depending on whether Section Breaks have been used in order to vary Layout specifications, page numbering, etc. (Section Number is displayed in the Status Bar @ the left of the Page Number.) Headers & Footers are properties of the Section in which they appear, so if there is more than 1 Section there may be more than 1 set of Headers & Footers.

Hello

This is an old thread that I also reported.  I've found a fix to this:

Create VBA code that runs on lost focus event e.g.

Sub listboxname_LostFocus()

Dim varItm as variant

For each varItm in listboxname.ItemsSelected

    listboxname.Selected(CLng(varItm)) = False
    listboxname.Selected(CLng(varItm)) = True

next varItm

End Sub

You can set the control and/or use the With clause as required

This code needs to be added for all the extended multiselect controls in your application

Excel 2003 Saving Macros for use in multiple workbooks - Microsoft Community

jan sawicki say:

Excel 2003 Saving Macros for use in multiple workbooks

I have created a macro and saved it as This Workbook.  I have a process which requires me to repeat the process each month.  I would like to have this macro available next month without having to open previous month's workbook, add the new information and save it as a current month just to use the macro.  Is there a way to save macros for use on other workbooks?  Also shared amongst others if on a network?
zack barresse say:

Hi there,

 

Save it in your Personal.xls file.  If you do not have one, create one.  This saves into the XLSTART folder and will open (hidden) with Excel, thus will be available to all workbooks when you open your application.  If you run a macro from the macros dialog (Alt + F8), you'll see your macro name (i.e. MyMacro ) preceeded with the name "PERSONAL.XLS!", to look like "PERSONAL.XLS!MyMacro".

If you don't have this file, probably the easiest way to create it is to record a macro.  When asked where to store the file, from the drop down menu choose Personal Workbook.  Stop the macro recorder immediately, as it will have already created the workbook at this point.  Open your VBE (Alt + F11) and you should now see it in the Project Explorer (which if you don't see that, press Ctrl + R).  Expand it to see all the modules.  The macro should be in the last module in the named series ModuleX (where X is a number of modules with that name, i.e. Module1).

HTH

Regards, Zack Barresse
