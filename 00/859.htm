IIF with a Date Range in a calculated field? - Microsoft Community

garland w milhoan say:

IIF with a Date Range in a calculated field?

I have a table with a column of dates. I'd like to create a calculated field which assigns a semester name to a date based on the range of dates for each semester. If a date is 1/15/17, I'd like this field to say Spring 2017. If it says 8/29/2017, I'd like it to say Fall, 2017 and so forth.

I've found a lot of explanations for individual parts of what I want to do but nothing that ties it all together.

Any thoughts?

Bill Milhoan

ken sheridan say: I would strongly recommend that you store the semester start and end dates in a Semesters table with columns Semester, DateFrom and DateTo.  If you compute the semester in an expression data is being hard-coded in the expression.  A fundamental principle of the database relational model is the Information Principle (Codd's Rule #1). This requires that all data be stored as values at column positions in rows in tables, and in no other way.

With a Semesters table the semester name for any given date can then be returned in by a JOIN in a query, or by a computed control in a form or report which calls the DLookup function.  I'd be inclined to wrap the code in a little function, which can then be called whenever necessary for any date, or the current date if no date is passed into the function:

Public Function GetSemester(Optional varDate As Variant)

    Dim strCriteria As String
    
    ' set default date to current date
    If IsMissing(varDate) Then varDate = VBA.Date

    strCriteria = "#" & Format(varDate, "yyyy-mm-dd") & _
        "# Between DateFrom And DateTo"
        
    GetSemester = DLookup("Semester", "Semesters", strCriteria)
    
End Function
