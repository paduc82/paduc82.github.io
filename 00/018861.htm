karl timmermans [outlook mvp 2012-2018] say:

What your actual options are really depends on the answer to item #2 in terms of whether or not the UDFs reside in a "default" contact folder since it makes a difference.

Essentially comes down to 2 options

Option #1 (this is the free manual approach)

Based on

a) contact folder involved is NOT a default contact folder on "computer b"
b) assumption is that "computer b" has no additional UDFs defined at the folder level

Approach

#1 - Copy contact folder on "compute A" to a new PST file with a new contact folder 
#2 - Delete the contacts in the folder that you just copied leaving an empty folder with the UDFs still defined at the folder level
#3 - Copy contact folder from step #2 to computer "B"
#4 - Copy the contacts from the original computer "B" folder to the one you just copied with the UDFs. Delete the original contact folder on computer B

*** this method unfortunately doesn't work if the contact folder involved on computer B is a default folder since it cannot be deleted

 

Option #2 - not free if target contact folder has more than 10 contacts

#1 - On computer "A" - save a contact that has a value assigned to every UDFas a .msg file
#2 - Take the contact from step #1 - save it to the target contact folder on computer B (that will result with the contact having what are otherwise known as "orphan" user-defined fields
#3 - Use the UDF functions in ContactGenie Toolkit which will find orphan UDF's in a contact folder and optionally let you rebuild the "User-defined fields in folder" list

http://www.contactgenie.com/toolkit_4_outlook.htm

 

Beyond that - not aware of any quick, simple, easy approach to moving UDF definitions at the folder level but certainly gives me some "food for thought" for a new capability in the the upcomig new CG versions
