stefan blom say:

Language format is a property of text in Word. In the affected document, select all the content and then (re-)apply the correct language via the Language dialog box (Review | Language | Set Proofing Language).

Also, click to deselect "Do not check spelling or grammar" for the selected text. (For a large selection, you may see a filled box, and then you have to click more than once for the check box to be completely cleared.)

Note that you should click OK in the Language dialog box, that is, you should NOT click Set As Default, since you are only trying to change the language for the selection in a particular document. (In addition, you can modify the language for the Normal style, but for some documents that won't be sufficient.)

Since (unformatted) AutoCorrect entries are language-specific, they will probably work properly as soon as you have the language sorted out in your document.

ken278 say: There is a program in the Utilities folder called Activity Monitor. Start it up. It will produce a list of all of the processes ongoing in the system. 

Click the %CPU column header (descending). This will sort the processes according to the percentage of the CPUs that each process is using.

If you find a process using a lot of the CPU consistently (for example, using 50 to 100% of the CPU over several minutes or longer), then that will be the source of the problem. There are two processes that have given me the same kind of problem you described in Excel and Word. One related to syncing of data between MobileMe and/or iCal and/or Entourage (possibly called Conflict Resolver or similar).  The second had something to do with Windows XP running under Parallels. 

Two other possible causes that I know of:  first, when you set up Dropbox with a lot of files being uploaded, this will really create sluggishness until Dropbox is finished, which could be hours later. Uploading via most ISP's is usually much slower than downloading, plus Dropbox hogs the CPUs.

Second, some backup programs (Carbon Copy Cloner for instance, not Time Machine as far as I know) can also hog the CPU or other such things and slow everything down. 

So with Dropbox and CCC, I have learned to turn them on only when my computer will be idle for several hours (at night usually).

You may find something else. But what you described sounds a lot like what I've experienced above.  Good luck.
