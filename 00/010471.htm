macropod say:

If you've activated 'Track Changes', you'll now how copies of whatever has been deleted, plus whatever's been added. Accepting all changes (via the Ribbon's Review tab) and switching off Track Changes should fix that.

Reinserting the b/w line drawing map could increase the file size if the new one is higher resolution or hasn't been compressed in Word the same as the original.

Saving in the .doc format is also liable to result in an increased file size. if you've done that, save it back to the docx format.

Importing content from other sources, with different Styles or Style definitions, or overriding paragraph Styles with hard formatting can also increase the file size.

Updating all references (via Ctrl-A, F9) then saving can also reduce file size.

As you can see, numerous factors can account for significant changes in file size.

suzanne s. barnhill say:
"Save Thumbnails" (and, in previous versions, "Save preview picture") will definitely result in file bloat. I've never seen background saves indicted as a cause. The traditional "causes of file bloat" were:

  1. Fast Saves: Disable this on the Save tab of Tools | Options. This one was eliminated many versions ago.
  2. Preview Picture: Clear the check box on the Summary tab of File | Properties. This is the equivalent of the current Thumbnails.
  3. Versions (File | Versions): Make sure "Automatically save version on close" is not turned on. Versioning was also eliminated several versions back.
  4. Revisions (Tools | Track Changes): Highlight Changes: Make sure "Highlight changes on screen" is turned on (or that "Final Showing Markup" is displayed). Accept/Reject Changes: If "Accept All" or "Reject All" is available then revisions are present; accept or reject all changes, then turn Track Changes off. This one is still a problem (though the path to eliminate markup is different). 
  5. Keep track of formatting (Tools | Options | Edit): This is reportedly a major cause of file bloat in Word 2002 and above. I have no evidence of whether it matters in recent versions.
  6. Embedded True Type fonts (Tools | Options | Save): Embedding fonts should be avoided wherever possible. This is still true.
  7. Embedded linguistic data (Tools | Options | Save). Possibly still an issue.
  8. Embedded graphics: When feasible, it is preferable to link the graphics. That is, when you insert the graphic, click the arrow beside Insert in the Picture dialog and choose Link to File rather than Insert or Insert and Link. This is still true, but documents with linked graphics are less portable.
  9. Embedded objects: These are even worse than ordinary graphics saved with the document. If you see an { EMBED } code when you press Alt+F9, the graphic is an OLE object. Unless you need to be able to edit the object in place, unlink it using Ctrl+Shift+F9. Still valid.
  10. File format: Make sure you are saving as a Word document; in some cases .rtf (Rich Text Format) files are significantly larger than .doc files. As noted, .docx files will be smaller than the corresponding .doc format.

mdc56 say:

Filter Multiple Keywords in a Column

How do I do more than one, in any combination?

Thanks!

bob jones aka: cybertaz say:

Unfortunately, no. The AutoFilter allows only 2 criteria. If you need to use more than that you'd need to use the Advanced Filter. With it you can use as many criteria as you wish. This article uses a Windows version as the model but there really is no difference in the Mac version - except that you'll find the feature in the Data menu or by clicking the Advanced button on the Data tab.

Learn advanced filtering in Excel by tracking...

At a glance, I do have a few quibbles with the article... primarily because it doesn't fully cover the power & versatility of the feature... It;s a very simple introduction to the [rather obscure] feature. That gives the impression that it has to be done as the article says. It's a great write-up as far as it goes, but IMO it just doesn't go far enough. :-)

  • I prefer to have more than 4 rows available in the Criteria Range.
  • If the Criteria Range is directly above the Data Range there must be at least 1 empty row separating the 2 ranges.
  • The Criteria Range need not be directly above the Data Range. It can be anywhere on the sheet or even on a different sheet in the workbook. I prefer to have the Criteria Range above but directly off to the side of the Data Range, but there must be at least 1 empty column separating the 2 ranges. Note that you don't want the Criteria range to occupy the same row as any records, otherwise you may lose sight of that criteria if the record on the same row isn't included in the found set.
  • (If there are many columns in the Data Range this is a good time to take advantage of Freeze Panes or Split.)
  • The Criteria Range need not include all field names from the Data Range. It can be as few as 1 single field name, depending on how may fields you want to be able to filter on.
  • It doesn't mention that if you have your selector in the Data Range the extent of the range will be entered for you automatically... There's no need to select the entire Data Range. You need to make a selection only if you want to query a portion of the Data Range.
  • You can use the same criteria in any given column more than once in conjunction with different criteria in other columns. Using their model, for example, you can have Sockeye Salmon in both C2 & C3. Then in the Average Price per Pound column you could have <11 in F2 & >15 in F3. The result would omit any records for Sockeye Salmon having an Avg. Price/Pound of 11 thru 15. (BTW, the article doesn't clarify that when using comparison operators the results are inclusive of the specified values... IOW, <11 actually will include 11.)
  • It isn't mentioned that the Advance Filter not only supports comparison operators, but wildcards as well. So to find 'all fish that begin with the letter c' you could use c* in the Seafood Type column.
  • It's fairly obvious, but the article doesn't tell you how to display all records again after having used Filter in Place... Just click the Clear button on the Data tab or select Clear Filters in the Data menu.

There are unlimited possibilities... The fundamental factor to keep in mind is that the rows in the Criteria Range are connected by 'Or' operators, the columns are connected by 'And' operators. Have fun :-)
