The messaging app on windows phone doesn't have many features.  What you've discovered is pretty much all there is to know about it and there aren't any 3rd party apps in the marketplace that will give you more control or options to manage your text messages, but I'll take each question you asked and answer it with my best knowledge:

Is there a capacity limit of stored messages?  ANSWER: The capacity limit would be the amount of memory that is available on the phone

Do older messages eventually get deleted automatically to make room for new messages?  ANSWER: No messages will be deleted automatically.  You must delete them manually.

Will these messages eventually consume the storage on my Lumia?  ANSWER:  Yes....if you don't delete any text messages that are received to the phone, eventually the storage on the phone will be taken up by text messages.

Other than deleting message threads, there isn't a way to delete groups of messages (unless you hard reset the phone - then you'll delete everything on the phone with one fell swoop, but that's pretty extreme and something you won't want to do).

joeyspinosa say:

You are correct sofspra. Windows file explorer will mount an .iso file on a virtual drive when double clicked.

Many file apps like PowerISO, BurnAware, etc. will grab the .iso extension in the registry, and own it. Hence, if you have PowerISO installed, double clicking on an .iso file in file explorer will bring up PowerISO.

Those programs are handy for many chores, but not needed for the above mounting of the .iso.

What this user has done (besides finding an ISO image for build 16226) is called an "in-place upgrade".

This is why we love ISO files. If you have it, you can double click it from file explorer and it will mount. Then you can run set-up and it will default to an "in-place upgrade" which will keep all your applications and data files. You can even do this with an ISO of your current version of windows. For example, now that this fellow has upgraded to 16226, he could repeat the same procedure with his 16226 ISO and it would do another "in-place upgrade" of 16226 to 16226... Why would you want to "upgrade" to the same version you are currently running? It's an easy way to do a "windows refresh" without losing any of your stuff!

Having an ISO image is great. The best part is, instead of just double clicking and mounting it, you can use a 3rd party utility like Rufus, PowerISO, Burnaware, etc. to burn a copy of the image to a DVD or USB stick. You would NEED to do this if you wanted, or have to, do a clean install. Windows will NOT let you clean install from within windows... You have to burn the ISO to a bootable media, and then boot to that device (this usually means you have to do a system restart and get into your BIOS and tell it not to boot from your normal drive, but wherever you burned the image - DVD, USB-stick, etc.) Booting the ISO image is the only way to do a clean install.

net start msiserver
