MEMORY_MANAGEMENT Bugcheck I see.

This error sure has been coming up a lot lately.

BCCode: 1a   0x0000001a  

The MEMORY_MANAGEMENT bug check has a value of 0x0000001A. This indicates 
that a severe memory management error occurred.

BCCode: 1a   0x0000001a <- read this 
http://www.faultwire.com/solutions-fatal_error/MEMORY-MANAGEMENT-0x0000001A-*1041.html?order=date

Unfortunately memory tests do not catch everything such as mismatched memory, when 
slower memory is placed in front of faster memory, and other issues.

You can try Powering Off all power and removing and reseating the memory. One or two 
stick(s) at a time if possible to see if the system bluescreens and if so on which sticks.

Other possibilities are BIOS and antivirus/antispyware/security programs and even 
motherboard problems.

Drivers are possible however that would be unusual - maybe chipset drivers.

Look in the Event Viewer to see if anything is reported about those.
http://www.computerperformance.co.uk/vista/vista_event_viewer.htm

MyEventViewer - Free - a simple alternative to the standard event viewer of

crashes, and displays the information about all crashes in one table - Free
http://www.nirsoft.net/utils/blue_screen_view.html

lemp say:

One of the benefits of trying to help people on this forum is that there is always something new to learn.

From the "Description" of your critical hotfix, it is clear that it relates to Java.  When I clicked on the link, I learned that -- for Oracle, Java's current owner -- "CPU" in this context means "Critical Patch Update."  So what Belarc Advisor is trying to tell you is that there is a critical patch update, issued in January 2016, for a particular version of Java.

You have an outdated version of Java and you should either remove Java entirely or at the very least update it.  This situation is a bit surprising, because by default, Java 8 should be set to automatically check for updates.

Another somewhat unusual aspect is that the update is for the 64-bit version of Java.  Although many people today (perhaps a majority) have 64-bit operating systems, most web browsers are 32-bit versions, even on 64-bit systems, and should use the 32-bit version of Java.  See this page --> http://java.com/en/download/faq/java_win64bit.xml

There are a couple of different ways to approach things.  Although normally I would suggest just visiting this web site (http://java.com/en/download/installed8.jsp)and clicking the big red "Verify Java version" button, you might want to do the following instead:

  • Click the Start Orb and type
          Configure Java
    into the Search box and press Enter
  • When the Java Control Panel opens, click the "About" button on the "General" tab.  This will show the version of Java you have installed.  The current version is Version 8 Update 73 (build 1.8.0_73-B02) -- at least for the 32-bit version of Java
  • Close the "About Java" window, click the "Update" tab, make sure that there is a check in the box to "Check for Updates Automatically," and then click the "Update Now" button.  (Before clicking Update Now, you can, if you want, click the "Advanced..." button and select the times and frequencies that the check for updates will occur; you can also use the drop-down box to change the notification from "before downloading" to "before installing").

After Java has been updated, reboot your computer and then run Belarc Advisor again.  Confirm that the warning is now gone.

If the warning is NOT gone, go to Control Panel > Programs and Features and let me know what entries there are that start with "Java."

greg carmack - mvp 2010-2020 say:

It depends on if you bought the PC preinstalled with WIndows 10 in which case it's Factory OEM, which lives and dies with the first motherboard installed upon.

If it's Upgrade then it is retail and not OEM  although if your original OS came pre-installed it inherits the restrictions for OEM including no change of motherboard. 

Hope this answers your questions.  If not ask back any others until its clear.

greg carmack - mvp 2010-2020 say:

If  Windows 7 Home Premium came preinstalled by the manufacturer then it is Factory OEM which cannot change motherboards, so both the WIndows 8 and 10 inherit that restriction. 

If you want to make a motherboard change you can try following the procedure in How to Use Your Free Windows 10 License After Changing Your PC’s Hardware however note that you cannot change entire PC's to move that Upgrade.  Sometimes when activation fails if you do the phone activation they will make allowances on OEM, but this cannot be counted upon.

There is no Product Key for Windows 10 free Upgrade.  It is a retail copy of WIndows permanently tied to that PC to as often as you want Clean Reinstall Windows 10 Upgrade.

You do have until next July 29th to do another free Upgrade on any other PC or changed hardware you want.  After that I would rely on a saved System Image Backup to use in place of reinstall in 20 minutes, as this retails activation.
