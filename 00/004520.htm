2. Within this key ("folder") add a new string value named "SharedDocumentParts" (without quotation marks).

3. Click "Modify" and change the value to "DriveLetter:\\Directory\\Subdirectory" (again without quotation marks). So essentially provide Office with the path to the directory where the portable dotx file is saved.

graham mayor say:

How is that going to help you transport the building blocks to another computer, where you may not have access to the registry? Without that registry setting the other computer will not know about your file.Furthermore when you use a USB drive on another computer, you will have no guarantee that it will adopt the same drive letter on that computer.

It is bad practice to save documents etc to removable media like flash drives for the simple reason that documents create temporary files in the working folder, the size of which you have no control over. If Word cannot create its temporary files, the documents will be corrupted without warning. It is less of a problem with USB flash drives than floppy discs, by virtue of the extra space available, but it is a real problem nevertheless. You should always COPY to and from removable media when working with Word.

jeeped say:

Custom TOCs are created using a combination of Bookmarks, PageRef and Ref. This is a quick run through:

  • Go to each chapter title and highlight the title text you want displayed in your custom TOC. Create a unique Bookmark for each one. Something like chap01, chap02, chap03, etc.
  • At the beginning of your document, first insert a PageRef field referencing the bookmark chap01. Tap TAB and insert a Ref field referencing the bookmark chap01.
  • Start a new line and repeat for chap02, chap03, etc.

With the field code view set as View Field Codes, it should look something like this,


Figure 1.0 - Custom TOC in Word (Field Codes)

graham mayor say:

Surya's link may help, but I suspect that the issue has more to do with the Windows file association for the DOC format. Right click a DOC file in Windows Explorer. Select 'Open With ...'. Browse down the list to Works WordProcessor (or if not listed use the browse button to locate C:\Program Files\Microsoft Works\WksWp.exe) Select the application and check the box to always open with this application.

Note that Works has limited compatibility with Word documents and if uninstalling the Trial version of Office has also uninstalled the text converters it will have no compatibility with DOCX format. You can check that by opening the Works word processor and check if File > Open > Files of Type lists Word 2007 document.

The native format for Works documents is WPS. Note that many people you send this document to - particularly potential employers - may not be able to open this format and those employers will not be taking the trouble to install filters to do so. Use SaveAs to save in DOC/RTF format which they will be able to open.
