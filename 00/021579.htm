One thing you didn't mention was your regional setting - or more specifically - what is the "list character" defined in your regional settings options - makes a difference on how you can go about things. What you're options are also depends on exactly how your "csv" file has been originally constructed

Some potential options without knowing the answers

#1 - If you are certain that a semi-colon is only used to delimit the fields in each row - then easiest solution would be to use a text editor and do a find/replace on semi-colon replacing it with a comma. Would have made a slightly different suggestion if every field included a "field qualifier" but your sample indicates that this isn't the case (i.e. field  qualifier = a double quote before and afer each field like ....."FirstName","LastName" etc

#2 - If you are in North America or in any country where the list separator character is defined to be a 'comma" (Control Panel --> Regional & Language Settings --> Regional Options --> Customize --> List Separator Character defined here - then

2a) rename your file with a "txt" extension instead of ".csv" - open up in Excel - follow the screens to define the delimitor as a semi-colon and when the file has completed opening - resave it as a .csv file

#3 - If you are in a country where the list separator character is defined to be a semi-colon - the .csv file will open correctly in Excel 'as is" since .csv files created by Excel in these cases - generates a CSV file using the list separator character used in the regional settings. In this case - change the regional settings to a comma, follow steps of #2 & 2a, and then change your regional settings back to semi-colon (or more specifically - back to whatever is the norm for your region)

or..............

#4 - open the csv file in Excel - if your regional settings have the semi-colon defined as the list character - will open normally,

- create a "named range" for the data you want to import covering the first row which should contain the header row and all subsequent data rows
- save the worksheet as an .xls file
- import the Excel file into Outlook

or.............

#5 - if you don't want to bother with any of the above - ContactGenie QuickPort may be of interest - supports "custom delimited" text files - just need to specify the "semi-colon" as the delimitor

suzanne s. barnhill say:
To add to what Neha has said, note that the "Last Printed" property (and the PrintDate field) will not be updated unless you save the document after printing.

neha singh msft say:

Hi KylieMR,

Here are the answers:

  • Total Editing Time - Total time taken to edit the document.
  • Last Modified- Date on which the document was last edited/modified.
  • Created- Date on which the document was created.
  • Last Printed- Date on which the document was printed.
  • Author- The name of the user who originally created the document.
  • Last Modified By- Name of the user who last modified the document.

The above properties are triggered when you perform the action.
