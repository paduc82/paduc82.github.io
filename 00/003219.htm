jay freedman say:

The first question is, which type of protection is applied to the document?

If it's "filling in forms," that's not usable in this situation -- it prevents all changes anywhere in the document except inside legacy form fields (and some content controls).

If it's "no changes (Read only)," then you have to mark the places where changes (including inserting quick parts/building blocks) will be allowed. To do that:

  • Open the template and open the Restrict Editing pane.
  • Click the Stop Protection button.
  • Select a place in the document (anything from one character up to a batch of paragraphs) that can be changed.
  • In the Restrict Editing pane, check the "Everyone" box under Exceptions. Repeat for other places that need to be changed.
  • Click the Start Enforcing Protection button and save the template.

charles kenyon say:

Supplementing Jay's excellent advice:

If it is "filling in forms" you can work around this using vba (macros). However, this is an order of magnitude more difficult than simply using "restrict editing" control. Also, it is not suitable for any environment in which macros are not allowed.

IF you do not already have such a solution developed, I would urge you to instead use content controls. This would involve a macro that unprotects the document, inserts your AutoText (building block) and then reprotects the document. It probably would be best handled with a UserForm (custom dialog box). Again, this can be done, but it is not easy if you haven't done it before.
