conditional SQL

Assuming that both students and teachers can use the database, I would like to set the query up so that if a student is using it, the sub-report is populated with only information on the students in the table. If a teacher is using it, the report is populated with all the records from the table.

 

Right now, the SQL looks something like this:

 

SELECT *

FROM SCHOOL;

 

When the form containing the sub-report opens, the database knows both the user type (Student or Teacher) as well as the user's ID number. So I would like to do something like create a variable when the form loads such as:

 

If UserType = "Student" Then

strWhere = "where UserType = Student"

Else strWhere = ""

End If

 

And then somehow modify the query so that the code looks something like:

 

SELECT *

FROM SCHOOL

strWhere;

 

But so far I'm not having any luck. Is something like this even possible? Is it possible to pass a variable containing the WHERE condition to a query so that I can use a single query and execute code conditionally?

 

Thank you in advance for the help!

 

-LG

tom van stiphout say:

You are on the right track.

 

In the Form_Open event write:

If UserType = "Student" Then

  Me.mySubformControl.Form.RecordSource = "select * from school where UserType='Student'"

Else

  Me.mySubformControl.Form.RecordSource = "select * from school"

End If

 

Note that the above is much better than applying a filter:
if ...
  Me.mySubformControl.Form.Filter = "UserType='Student'"
  Me....FilterOn = True

because filters can be cancelled by the user.

 

Also note my use of single-quotes to wrap the UserType value. That is required for text fields like UserType apparently is.

I therefore recommend cards no bigger than 1GB due to Nokia's poor mass storage mechanism, poor playlist in music player, and lack of FAT32 support.

tom ogilvy say:

My assumption was that you are placing the value in the cell where you want the behavior.  So you would type the source data or paste it. 

Private Sub Worksheet_Change(ByVal Target As Excel.Range)
   Dim r As Range, v As Variant
   On Error GoTo ErrHandler
   Set r = Me.Range("G4")    '<==  changed to cell G4
   If Target.Count > 1 Then Exit Sub
   Application.EnableEvents = False
   If Not Intersect(Target, r) Is Nothing Then
       v = Target.Value     ' put the value of the changed cell in variable v
       Application.Undo    ' undo the edit so the previous value is in the previously changed cell
       Target.Offset(-1, 0).Value = Target.Value  ' write old value in cell above
        Target.Value = v    ' put the new value back where it belongs
  End If
ErrHandler:
Application.EnableEvents = True
End Sub

if you want to just run a macro that does what you describe  (move the existing value from G4 to G3 and copy the new value from Daily Water Report cell F365) you could do run this macro whenever you wanted to do that.  

Sub abc()

Dim sh1 as worksheet, sh2 as worksheet

sh1.Range("G3").Value  = sh1.Range("G4").Value

sh1.Range("G4").Value = Sh2.Range("F365").Value

End Sub
