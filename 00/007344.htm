When I upgrade a preinstalled (OEM) or retail version of Windows 7 or Windows 8/8.1 license to Windows 10, does that license remain OEM or become a retail license?

If you upgrade from a OEM or retail version of Windows 7 or Windows 8/8.1 to the free Windows 10 upgrade this summer, the license is consumed into it. Because the free upgrade is derived from the base qualifying license, Windows 10 will carry that licensing too.

If you upgrade from a retail version, it carries the rights of a retail version.

If you upgrade from a OEM version, it carries the rights of a OEM version.

Full version (Retail):

- Includes transfer rights to another computer.

- Doesn't require a previous qualifying version of Windows.

- Expensive

Upgrade version (Retail):

- Includes transfer rights to another computer.

- require a previous qualifying version of Windows.

- Expensive, but cheaper than full version

OEM :

OEM versions of Windows are identical to Full License Retail versions except for the following:

- OEM versions do not offer any free Microsoft direct support from Microsoft support personnel

- OEM licenses are tied to the very first computer you install and activate it on

- OEM versions allow all hardware upgrades except for an upgrade to a different model motherboard

- OEM versions cannot be used to directly upgrade from an older Windows operating system

What happens if I change my motherboard?

As it pertains to the OEM licenses this will invalidate the Windows 10 upgrade license because it will no longer have a previous base qualifying license which is required for the free upgrade. You will then have to purchase a full retail Windows 10 license. If the base qualifying license (Windows 7 or Windows 8.1) was a full retail version, then yes, you can transfer it.

From the end user license agreement:

15. UPGRADES. To use upgrade software, you must first be licensed for the software that is eligible for the upgrade. Upon upgrade, this agreement takes the place of the agreement for the software you upgraded from. After you upgrade, you may no longer use the software you upgraded from.

17. TRANSFER TO ANOTHER COMPUTER. a. Software Other than Windows Anytime Upgrade. You may transfer the software and install it on another computer for your use. That computer becomes the licensed computer. You may not do so to share this license between computers.

joeu2004 say: Ed wrote:

Well, you might consider something simple like the following VBA UDF:

Function findHighlight(r As Range)
Dim c As Range
' Application.Volatile
For Each c In r
   If c.Interior.ColorIndex <> xlNone Then
      findHighlight = c
      Exit Function
   End If
Next
findHighlight = ""
End Function

Then in B7, you might enter the formula =findHighlight($B2:$D2), which you can copy into B8 and B9.

(Note that I write $B2, not $B$2.  So it changes to $B3 and $B4 automagically when you copy the formula.)

However, I would think that is inadequate because the formulas in B7:B9 are not recalculated if you only change the cell "highlight" (Format Cells > Font > Pattern), even if we uncomment the Application.Volatile statement.

And I do not like to use Application.Volatile because those formulas would be recalculated every time you "edit" any cell in any worksheet in the workbook -- where "edit" means change the cell contents.

Also, I did not find any Worksheet event macro that is called after changing only the cell "highlight".

jeeped say:
I created a UDF (e.g. User Defined Function) that pulls the item and color from a cell and returns the appropriate value from an array.

                       

Press Alt+F11 to open the VBE and immediately use the pull-down menus to Inert, Module. In the new pane entitled something like Book1 - Module1 (Code) paste in the following,

Public Function fcnFindHighLights(cHiLite As Range, rLookup As Range, iCols As Integer) As Double
    iHiLite = cHiLite.Interior.Color
    vHiLite = cHiLite.Value
    vReturnValue = 0
    For Each cell In rLookup
        If cell.Value = vHiLite Then
            For Each c In Range(cell.Address, cell.Offset(0, iCols))
                If c.Interior.Color = iHiLite And iHiLite <> 16777215 Then
                    vReturnValue = c.Value
                    Exit For
                End If
            Next c
            Exit For
        End If
    Next cell
    fcnFindHighLights = vReturnValue
End Function

Press Alt+Q to return to the Excel worksheet when you are done. To use the custom function put it in a cell like this,

fcnFindHighLights(<cell with item and color to lookup>, <range of cells for item>, <number of columns to the right to look for matching color>)

In the sample pictured above, B7 holds the formula.

=fcnFindHighLights(A7,A2:A4,99)

... which look in A2:A4 for Bats (value in A7) and returns the value in B2:B100 with a matching color.

Please note that the colors were manually added with the commands available with the Home tab's Font section. Colors applied through Conditional Formatting require a very different approach.

Remember, I did say you should check the exact names of the navigation form and its subform control.

I've checked, and by default Access creates these as "Navigation Form" and "NavigationSubform", so if you didn't change them from the default names, this might work:

     [Forms]![Navigation Form]![NavigationSubform]![Group]

It's just possible that, Access 2010 being extra-picky about these things, you might need the fully expanded syntax:

    [Forms]![Navigation Form].[Form]![NavigationSubform].[Form]![Group]

But I don't think all those extra ".Form" qualifiers should be necessary.

If that doesn't work, first check the name of your navigation form itself.  If, for example, it is "Master Navigation Form", then you would use this:

    [Forms]![Master Navigation Form]![NavigationSubform]![Group]

If none of the above suggestions works, check the name of the subform control (on the navigation form) that will be used to display your former main form.  To do that, open the navigation form in design view, click once on the subform to select it, open the property sheet, and check the Name property (on the Other tab of the property sheet).  What does it say?

Whatever it says there, that's what you need to use in the expressions above where I've put "NavigationSubform".
