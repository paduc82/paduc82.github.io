Excel 2013 - How to add 1 to a cell reference rather than adding 1 to - Microsoft Community

ianweatherall say:

Excel 2013 - How to add 1 to a cell reference rather than adding 1 to cell contents.

Hello,
I have a row of sales values on a worksheet called Sales, it simply a row of sales for each day from the previous year.
On a worksheet called 2014 i have a calendar, each day references the sales sheet in order to give this years daily goals.

Can I write a formula that will add 1 to the cell that is reference rather than adding 1 to the value.

ex.

f3: =SUM(Sales!D2)
f4: =SUM(Sales!D2+1)

The above contents will simply add 1 to the value in Sales!D2

What i want to do is make f4: =sum(sales!E2)

I hope I have explained correctly.

Thanks in advance.
Ian

jllatham say:

Copy this formula and put it into F2 and copy it down as far as you need:

=OFFSET(Sales!D$2,0,ROW()-ROW(F$3))

 

The $ symbols are important, in this case they keep the row reference from changing in the formula as it is filled down the sheet.

OFFSET() says take a basic cell address [D$2] and look some number of rows above/below its row (in this case we use row offset of 0, so we're leaving alone and always looking at Row 2 on the Sales sheet) and also how many columns left (negative) or right(positive) of that basic cell to look at.

 

For the column offset we use  ROW()-ROW(F$3)

and that says take the current row number that the formula is in and subtract the row number of cell F$3 from it to find out the number of columns we need to offset.  In the first formula that evaluates to 3-3=0  but when you copy it into F4  it becomes 4-3=1  which says look 1 column to the right of Sales!D2, in other words look in cell Sales!E2.  When you copy it into F5 it will evaluate to say "look in cell Sales!F2".

 

====

A note, and I know that several of Microsoft's examples make it look like it is required, but when you are just going to get the value of a single cell you don't need the SUM() part of the formula:

=SUM(Sales!D2)

is the same as

=Sales!D2

 

SUM() is actually a math operator that adds the values referenced in it, like this:

=SUM(Sales!D2:G2)
would total up all the values over on the Sales sheet that are in D2, E2, F2 and G2

It can also be written like this if you just like to type:

=SUM(Sales!D2, Sales!E2, Sales!F2, Sales!G2)

and you can use it to total up cells that are scattered around, like

=SUM(Sales!D2:G2, Payments!R3:R6, F5)

Which would get the sum of Sales!D2:G2, add it to Payments!R3:R6 and finally, add in the value in F5 on the same sheet the formula is on.

' basShellExecute Module
Option Compare Database
Option Explicit

Public Const SW_HIDE = 0
Public Const SW_SHOWNORMAL = 1
Public Const SW_SHOWMINIMIZED = 2
Public Const SW_SHOWMAXIMIZED = 3
Public Const OP_OPEN = "Open"

Declare Function ShellExecute& Lib "shell32.dll" Alias "ShellExecuteA" (ByVal _
hwnd As Long, ByVal lpOperation As String, ByVal lpFile As String, ByVal _
lpParameters As String, ByVal lpDirectory As String, ByVal nshowcm As Long)

Sub ShellToFile(strPath As String, _
            Optional strOperation As String = OP_OPEN, _
            Optional lngShow As Long = SW_SHOWNORMAL)

    Dim lngRetVal As Long
    Dim lngHwnd As Long
    
    lngHwnd = Application.hWndAccessApp
    
    lngRetVal = ShellExecute(lngHwnd, strOperation, strPath, _
        vbNullString, CurDir, lngShow)
        
    If lngRetVal <= 32 Then
        MsgBox "Unable to open file " & strPath, vbInformation, "Warning"
    End If
    
End Sub
