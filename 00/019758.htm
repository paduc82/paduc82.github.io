john w. vinson say:

This can be a little bit tricky: if the user typed a search term such as "Thomas the Tank Engine" you certainly would not want all movies containing the word "the"!

You'll actually need some VBA code to "parse out" the single text string "Thomas Engine" into its components, and construct a Query containing the criteria. Here's some *untested air code* that might do so:

Private Sub cmdSearchKeyword_Click()
Dim strKeywd() As Variant ' define an array to hold terms
Dim strSQL As String ' define a string variable for query
Dim iPos as Integer ' define a number as an array index
strKeywd() = Split(Me!txtSearchTitle, " ") ' split string on blanks
strSQL = ""
For iPos = 0 to UBound(strKeywd) ' loop through array
   strSQL = strSQL & "[Title] LIKE '*" & strKeywd(i) & "*' AND "
Next iPos
strSQL = Left(strSQL, Len(strSQL) - 4) ' trim off final AND
DoCmd.OpenReport "yourreportname", WhereCondition:=strSQL  ' open report
End Sub

This will build a query criterion such as

[Title] LIKE '*Thomas*' AND [Title] LIKE '*Engine*'

and then open the report with this as a search criterion.

Note that this code has no error trapping, provision for noise words such as "and" or "a", tolerance for spelling errors, etc. but I hope it will get you started.

 

John W. Vinson/MVP

Enter below formula in cell E2 and copy down:

=IF(A2="","",A2+MAX(INDEX(($G$2:$G$5)*(NOT(ISNA(MATCH($F$2:$F$5,B2:D2,0)))),,)))

You may add further values at any time in column F & G for matches by just updating the range $G$2:$G$5 (if you want to add 2 more values in column G, just update this range to $G$2:$G$7) and the formula will remain as short as at present.
