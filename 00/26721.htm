To copy files to the new user profile

After you create the profile, you can copy the files from the existing profile. You must have at least two user accounts on the computer to complete these steps, including the new account you just created.

  1. Log on as a user other than the new user you just created or the user that you want to copy files from.

  2. Press Windows key + E to open file explorer, click on view on top panel and then tap or click on options.       

  3. Click the View tab, and then click Show hidden files, folders, and drives.

  4. Clear the Hide protected operating system files check box, click Yes to confirm, and then click OK.

  5. Locate the C:\Users\Old_Username folder, where C is the drive that Windows is installed on, and Old_Username is the name of the profile you want to copy files from.

  6. Select all of the files and folders in this folder, except the following files:

    • Ntuser.dat

    • Ntuser.dat.log

    • Ntuser.ini

  7. Copy all the files and folders except the above mentioned files.

  8. Locate the C:\Users\New_Username folder, where C is the drive that Windows is installed on, and New_Username is the name of the new user profile you created.

  9. Paste the files in the new user folder.

  10. Log off, and then log back on as the new user.

If you have e‑mail messages in an e‑mail program, you must import your e‑mail messages and addresses to the new user profile before you delete the old profile. If everything is working properly, you can delete the old profile.

geoffrey chaucer say:

Olivia,

David is absolutely right. Forcing visitors to listen to music is regarded as bad manners. I urge you to consider making the music optional.

This is what you could/should do:

1- Create two equal size images of buttons  in your graphic/photo editor: one that says something like "Music on" and the other "Music off" and save them temporarily in any convenient folder on your local disk.

2- On the work page in which you want the music, insert any one of the buttons and drag it to a convenient place that will not interfere with the rest of the content of the page.

3- Make sure the "Snap to Objects" option in Tools is ticked, then click on the HTML fragment icon and draw an html fragment box exactly around the button. When the HTML fragment dialog box opens, type in <iframe name="music" width="the width of the buttons image in pixels " height="the height of the buttons image in pixels " frameborder=0 src="musicoff.htm" scrolling=no></iframe>

4- While the fragment is still selected click on the "Send to Back" icon on the tool bar, then click on the button image and delete it, leaving only the HTML fragment on the work page.

5- Create a blank separate Publisher webpage and save it As web Page in a separatePublish folder. Then rename the index.htm file musicoff.htm and manually copy it to your main website folder in the same folder as the web page where the music is to play. Next, in that same folder create a sub-folder and name it "musicbuttons" or whatever, and fetch the two buttons images you created earlier from their temporary abode and place them in that sub-folder.

Complicated hey? Oh, but wait ! there's more . . .

6- Back to the separate work page. In the very top left-hand corner of the work page, draw an HTML fragment box (as you did in step 3 above and of exactly the same dimensions) and type in:

<a href="musicon.htm"><img src="name of the folder where you saved the buttons/ full file name of the Music on button .gif or .jpg as the case may be " border=0></a> then click OK.

7- Save as Web page again and as before rename the index.htm to read musicoff.htm and in your main website folder replace the first musicoff.htm you created earlier by this edited one.

8- Go to the Publish folder where you renamed index.htm to musicoff.htm and rename itmusicon.htm , then copy it in your main website folder and in the same folder where musicoff.htm is located.

9- Go to your main website folder and double-click on the musicon.htm document to launch it in your local browser.

10- Click >view>Source . In the Notepad document, insert the following code snippet above the </head> tag: <embed src="write your music name file.mid(or .wav as the case may be) autostart="true"> then scroll the Notepad to locate the code sequence:

<a href="musicon.htm"><img src="name of the folder where you saved the buttons/ full file name of the Music on button .gif or .jpg as the case may be " border=0></a>

Then edit the sequence to read:

<a href="musicoff.htm"><img src="name of the folder where you saved the buttons/ full file name of the Music off button .gif or .jpg as the case may be " border=0></a>

11- >Save the Notepad file and upload all edited and new documents and folder to your host server.

Epilogue:

As you can see, it's a lot of trouble for little result. Visitors to your site who use Firefox will still need to have a plug-in installed to hear the music. But in internet explorer they'll hear the music without seeing the media player which will be hidden behing the page, and they'll love you for giving them the choice to listen or not.
