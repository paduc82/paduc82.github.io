bob jones aka: cybertaz say:

Quite often it's a matter of what type [file format] of image it is. For example, JPEG (or JPG, jpeg, jpg) is a compressed format to begin with, so trying to compress it further produces very little - if any - reduction.

The best approach is to use a decent photo editing program to process the images you intend to include in a document. In that way you can specify all parameters, including print dimensions, pixel dimensions (resolution), etc. in order to optimize a copy of each image based on its intended use. In that way you need only insert the image into your document & position it as necessary without trying to rely on a word processing program's convenience tools for anything other than minor adjustment.

If the document is to be printed (commercial or high quality), another approach is to Link to the images rather than embedding them. That keeps the document file size at an absolute minimum, but the image files need to accompany the document in order to render optimum quality when printed.

Thanks for your post and repro steps.  I was able to reproduce the issue on my end and this issue has previously been reported so the proper people are aware of it.  Unfortunately, I can‚Äôt guarantee if or when the issue will be fixed in Access 2007, but the issue has been fixed in Access 2010.

 

From what I can see the problem happens when you type these characters into a table or query, it changes the TabularCharSet property of the table to a value other than 0.  Then if you create a query off of this table, it will then pick up this same value for the TabularCharSet property.  In addition to this if you create a report off of the query or table; it will then set the TextFontCharSet property for each field on the report to a value other than 0.

 

At this point, the only workarounds I have are:

1.       Make use of Access 2010.

2.      I was unable to reproduce the problem if I used the ‚ÄúOutputTo‚Äù command within a macro.  So you could try creating macros that use the ‚ÄúOutputTo‚Äù command to do the export process.

 

If you want to make use of code, then you could try using these other workarounds:

3.      You can set the TabularCharSet property of your tables back to 0 by running code similar to the following on each table:

 

CurrentDb.TableDefs("<tablenamehere>").Properties.Append CurrentDb.TableDefs("<tablenamehere>").CreateProperty("TabularCharSet", dbByte, 0)

 

4.       You can set the TabularCharSet property of your queries back to 0 by running code similar to that above.  Just replace TableDefs with QueryDefs.

5.      If you have reports already created, then you would need to change the TextFontCharSet property for each control on each report.  Unfortunately, I don‚Äôt have code available that can easily do this.  Your code would have to go through each report and then look at each field and set this property for each field.  Once you change the properties of each table, you may want to try exporting the table to ensure it exports correctly.  Once it does, then try recreating the report.
