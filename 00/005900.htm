VBA program/function to search text string for consecutive Same - Microsoft Community

karma180 say:

VBA program/function to search text string for consecutive Same Letter with numbers in between, then sum the numbers in b/w, and eliminate those duplicated letters.

Hello, everyone.

 

I am looking for the best solution for optimizing text strings (many thousands of them) stored in an Excel column.

 

Example of one type of text string:

 

Z9R15U78WD23L1D18R7D31D1D2D20R27U31VR4U3U12VL4R3R1R14L8U8L3L15L7

 

 

What I would like this example to be transformed to:

 

Z9R15U78WD23L1D18R7D54R27U31VR4U15VL4R18L8U8L25

 

 

The "consecutive same letter" could be "D", "L", "R", or "U".  They are only considered "consecutive" if it's the same letter repeated, separated by numbers -- the numbers can be 1-3 digits (1-999). 

 

Once any of these type of occurrences are found, those numbers would then need to be summed, and all except for one of the replicated letters in that sequence eliminated, followed by the previous referenced sum.

 

So, in the example above, the original string had "D31D1D2D20" and was transformed to "D54" (31+1+2+20 = 54 --> D54).

Then "U3U12" was transformed to "U15".

Then "R3R1R14" was transformed to "R18".

And, "L3L15L7" to "L25".

 

 

 

I hope someone is up to the challenge!  Thank you in advance.

ronrosenfeld say: Here's a bit shorter UDF that makes use of Regular Expressions.

To enter this User Defined Function (UDF), <alt-F11> opens the Visual Basic Editor.
Ensure your project is highlighted in the Project Explorer window.
Then, from the top menu, select Insert/Module and
paste the code below into the window that opens.

To use this User Defined Function (UDF), enter a formula like
  =Optimize(A1)
 in some cell.

It looks through the string for the pattern you describe; replaces the letters with "+", and uses the VBA Evaluate method to Sum the values.  It then replaces that into the original string, starting from the right so as to maintain the locations properly.

If characters other than DLR or U are possibilities, they can be added to the character list in the brackets in the Pattern below.  Or, if any alphabetic character might be in the "repeat", you can substitute [A-Z] for the [DLRU] below.

=================================
Option Explicit
Function Optimize(s As String) As String
    Dim re As Object, mc As Object
    Dim i As Long
    Dim D As Double
    Dim sTemp As String
Set re = CreateObject("vbscript.regexp")
With re
    .Global = True
    .Pattern = "([DLRU])(?:\d{1,3})(?:\1\d{1,3})+"
End With

If re.test(s) = True Then
    Set mc = re.Execute(s)
    For i = mc.Count - 1 To 0 Step -1
        D = Evaluate(Replace(mc(i), mc(i).submatches(0), "+"))
        s = WorksheetFunction.Replace(s, mc(i).firstindex + 2, mc(i).Length - 1, D)
    Next i
End If

Optimize = s
End Function

jay freedman say:
I have a feeling I'm stepping into quicksand here, but I'll take a few guesses:

  1. Move the line DocToOpen.AttachedTemplate.Saved = True down to just before the line DocToOpen.Close SaveChanges:=wdSaveChanges. You're setting the template's Saved property too early; if anything happens to "dirty" the template between then and the closing of the document, then its Saved property will be changed to False by Word. The same might be true for the NormalTemplate, but nothing should be touching that.
  2. Your If...Then...Else condition is backwards. When DocToOpen.Saved is True, then there is nothing to save; when it's False, then a save is needed.
  3. I don't think you really need to call DocToOpen.UpdateStyles. The official description of that method is "Copies all styles from the attached template into the document, overwriting any existing styles in the document that have the same name." Unless you think the document's styles are different from those in the attached template and that's an undesirable condition, the UpdateStyles is at best a waste of time, and at worst will wipe out changes in the document's styles that you want to keep.

Sub Demo()

  .Saved = True
  'Let Word do any required housekeeping

    .Close SaveChanges:=False
  Else
    MsgBox ("About to save [" & FileName & "] that has been updated")
    .Close SaveChanges:=True
    MsgBox ("Finished Save")

End With
Set DocToOpen = Nothing
