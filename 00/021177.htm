Redefining Theme Packs- It's About Windows UI

Windows is used by billions today and while being happy about this Microsoft must also accept that it's completely impossible to satisfy those billions with a single skin. As a power user, I find it absolutely impractical for the Windows team to decide the UI that pleases everyone. Let's take a T-Shirt for example. Does everyone in this world wear the same T-Shirt? Your team might say you've given customisation options by giving users the choice to change colour, but what if I reframe my question this way: Does everyone in the world wear the same bubble-pattern T-Shirt? I think you'll get my point now. While the bubbles on this hypothetical T-Shirt might be offered in various colours what remains same is the texture, which in this example is "bubbles". Everyone in this world obviously doesn't wear a "bubbles" T-Shirt. Some might like to wear polka dot T-Shirts, while others might prefer a "leopard skin" T-Shirt. What I'm trying to emphasise is that till this date, Windows doesn't offer truly customisable UI. Why don't you people offer something like a UI Style Designer that can truly customise every element of Windows UI, whether it's the Start Menu, Taskbar, Title Bar or even the Minimise, Maximise and Close buttons? I know this isn't a new idea. Power users like me have always found tweaks that can do the same stuff I'm requesting for today. What I want is that such tweaks should be made publicly available. This feature should be a part of core Windows features. Now that I've briefly explained my idea, I'll go into the details as to how this feature could be implemented in a practical manner.

  1. We've got the Theme Packs already. We just need to redefine it. Theme Packs should not be just about background images and event sounds. They should have the ability to define the complete UI of the OS.
  2. Theme Packs should have some attributes related to every element of Windows UI, whether it's the Start Menu background, Taskbar, Title Bar, Window Frame, Buttons, Minimise button, Maximise button, Close button, Menu Bar, Menu Item and others.
  3. Attributes should define how a specific element in the Windows UI looks like. The following could be some of the attributes of a UI element:
    • Colour
    • Opacity
    • Translucency
    • Image or texture
    • Custom animations based on cursor movement or finger movement on screen
  4. As I mentioned earlier, there should be an official software provided by Microsoft so that people can design UI based on some strict guidelines.
  5. These custom Theme Packs should be made available in the Windows Store. There should be an option to sell Theme Packs as well. This will encourage more people to design UI for Windows and monetise on this.
  6. Only Theme Packs available in Windows Store should be installable and Microsoft should maintain some guidelines on how the Windows UI can be designed. This will prevent people from installing absurd Theme Packs. Quality should be maintained.

I know allowing such customisation could have an impact on performance and might require better hardware configuration so there should also be an option to disable third party Theme Packs that also change the UI apart from wallpapers.

If you really want to make Windows truly personal just like Windows Phone, I think this is the best bet.

Hope you get my idea. :)
