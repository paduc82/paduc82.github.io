Uploading files to a SharePoint Library - Microsoft Community

nrgins say:

Uploading files to a SharePoint Library

I create PDFs in my Access 2010 app, which I then e-mail automatically each night. I'd like to upload them to SharePoint for the client to be able to retrieve from there. Is that possible? I know there's a thing called Sharepoint Workspaces which allows syncing with a local folder. But that's only available in the Office 365 E3 account. I have the P1 account, so it's not available. Is there some other way, such as FTP or other? Thanks!

albert d. kallal say:

Actually, you can execute a standard windows file copy command (and this includes the native one in Access).

So you can run a report, save the pdf to your hard drive. Then simply do a file copy from your computer to the target sharepoint site.

Just keep in mind you have to use dos style "back slashes" ("\") and not typical web forward slasheds ("/").

So a file copy from your hard drive will look something like this:

Copy c:\mypdf\SalesAugust2012.pdf

\\yourSubDoman.sharepoint.com\TeamSite\Documents\Aug2012.pdf

 

(the above copy would be on one line of the dos prompt in windows.

Note in the above you replace "yoursubdomaon" with your sub domain. And note how we do NOT include http, https, or even www. here. You can always get the correct path name by opening the document library in your web browser, clicking on the doc library ribbon tab, and in the connect and Export, your see a "open with exploer". This launches the sharepoint folder as a standard windows desktop folder in which you can then drap + drop files to that folder just like it was a local folder.

And you as a general rule don't have to do the above to use the filecopy command either.

So you could code do something like this:

 

   DoCmd.OpenReport strRpt, acViewPreview, , strWhere
   DoCmd.OutputTo acOutputReport, strRpt, acformtpdf, strOutputDoc, False
   DoCmd.Close acReport
  

   strSharePoint = "\\xxxx.sharepoint.com\TeamSite\Documents\sales.pdf"
  
   FileCopy strOutputDoc, strSharePoint

albert d. kallal say:

I am quite sure if you not logged in, then a logon prompt will appear (and for unattended, that going to be an issue).

As an alternative you could force a logon with the net use command, and then the unc path name copy would work.  (so my answer is, am I not really sure how to provide a logon â€“ I have not tested, but adding password to the unc path name might work here - this would have to be tested).

 

> I've always just used OutputTo without doing an OpenReport first. Just wondering what the reason is for doing the OpenReport first. Are there some Access "gotchas" that that step eliminates?

Yes the reason for coding that way is to allow the use of a filter, or what is more correctly called a "where clause". If I want to restrict the report to say this month (and assume I have added a few extra columns to the query based on InvoiceDate (using month() and year() functions), such as InvoiceMonth, InvoiceYear, then in code one could go:

Dim    strWhere   as string

StrWhere = "InvoiceMonth = " & month(date()) & _
                     " and InvoiceYear = " & year(date())

Then the rest of the code makes sense:
Eg;
DoCmd.OpenReport strRpt, acViewPreview, , strWhere DoCmd.OutputTo acOutputReport, strRpt, acformtpdf, strOutputDoc, False DoCmd.Close acReport

The problem or limitation of Output to is you cannot provide a "where" clause to restrict the records to some type of criteria that you might want. So for a monthly reports etc, you can continue to use a "where clause". So there is no other reason than the above "common" issue of needing to provide a filter on the report.

At the end of the day, at least now one knows how to use a "where clause" with output to. About the only other issue when doing this is to remember to close the report after like above, since when outputto does not open the report, it will not close it for you either.

Provided that you want the conditional format to use the same conditions and simply want to expand the range then the easiest way is to copy the cell with the conditional format and then select the range to which it is to be applied (You must include the original cell that you are copying) and then PastSpecial -> Formats.

  3. Select the required rule
  4. At the right hand side of the "Applies to" field, click the little icon (another window displays) and then select the new range to which it is to be applied. (Ensure you include the original cell with the Conditional formatting).
  5. You can hold the Ctrl key after selecting the first range and then select additional ranges if your ranges are not adjacent so you can select various non contiguous ranges.
  6. When finished selecting the ranges, click the icon at the right hand end of the field that is populated with the selection/s.
