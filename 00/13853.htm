ken sheridan say:

Call the Execute method of the database object and trap any error:

    Const  MESSAGE_CONFIRM = "Update Sterling data file?  Click YES to continue or NO to exit."
    Const MESSAGE_SUCCEED = "The table was updated"
    Const MESSAGE_FAIL = "The table could not be updated"

    If MsgBox(MESSAGE_CONFIRM, vbYesNo, "Sterling Data") = vbYes Then
        On Error Resume Next
        CurrentDb.Execute "Sterling_Append", dbFailOnError
        If Err.Number = 0 Then
            MsgBox MESSAGE_SUCCEED, vbInformation, "Success!"
            DoCmd.GoToControl ("START")
        Else
            MsgBox MESSAGE_FAIL, vbInformation, "Doh!"
        End If
    End If

BTW, when setting Warnings off (False), as you were doing it is important that they be set back to True before exiting the procedure.

Here's a troubleshooting technique that may help to eliminate some possibilities. Hold down the Ctrl key while you start Word, and say yes to the prompt, to get it running in safe mode. That causes Word to run without loading Normal.dotm, any add-ins, or any non-default registry entries.
