donleverton say:

Mapping a specific ADDRESS is fairly simple.

Here is some VBA code that I found to do that.

1.) Create a code module (I called mine "modMapping")

2.) Paste this code into it:

======================

    

Function OpenMap(Address, City, State, Zip, Country)

    Dim strAddress As String
    strAddress = Nz(Address)
    strAddress = strAddress & IIf(strAddress = "", "", ", ") & Nz(City)
    strAddress = strAddress & IIf(strAddress = "", "", ", ") & Nz(State)
    strAddress = strAddress & IIf(strAddress = "", "", ", ") & Nz(Zip)
    strAddress = strAddress & IIf(strAddress = "", "", ", ") & Nz(Country)

    If strAddress = "" Then
        MsgBox "There is no address to map."
    Else
        Application.FollowHyperlink "http://maps.live.com/default.aspx?where1=" & strAddress
    End If
End Function

===================

I call this function from a command button which is on a form that has the address fields for a selected record (Parts Supplier in my case)

===================

Private Sub cmdOpenMap_Click()
OpenMap [Address], [City], [Prov], [PostalCode], "Canada"
End Sub

===================

Note that all of this code starts up a separate instance of my default web browser which displays the address in Bing Maps.

Routing, however, would probably require considerably more time and coding ... which would require an InternetExplorer.Application object and filling in web-form textbox elements with values supplied by your form.

(Trust me ... this is not something for the faint-of-heart)

JUST in case you are interested in this technique, here is some code that opens up an on-line VIN (Vehicle Identification Number) decoder web-form and fills in the VIN supplied by a textbox on my form.

(yay, no typing!)

Private Sub cmdDecodeThis_Click()
'Decode This VIN Decoder
Dim Msg As String
Dim CR As String
CR = vbCrLf
Dim MyVIN As String
Dim VinLength

MyVIN = Trim(Me![VIN])
VinLength = Len(MyVIN)

If VinLength = 17 Then ' A VIN must be 17 digits in order to be decoded

    Dim objIE As Object
    Dim Mywebsite As String
    Mywebsite = ""
    Mywebsite = "http://www.decodethis.com/VIN-Decoded/vin/" & MyVIN

    Set objIE = GetIE 'Calls a function that creates an IE application
    objIE.Navigate Mywebsite
    objIE.Visible = 1
    apiShowWindow objIE.hWnd, SW_MAXIMIZE

Else

    Msg = ""
    Msg = Msg & "The VIN must be 17 characters long in order to be decoded." & CR & CR
    Msg = Msg & "The VIN, as entered, is "
    Msg = Msg & VinLength
    Msg = Msg & " characters."
    MsgBox (Msg)

End If

End Sub

========================

The code (above) calls this function, which creates an IE application object

=====================

Function GetIE() As Object
  On Error Resume Next
  Set GetIE = CreateObject("InternetExplorer.Application")
End Function

Social Network Analysis (SNA) - designing the basics in Access - Microsoft Community

iongeorgiou say:

Social Network Analysis (SNA) - designing the basics in Access

I am trying to design the fundamental structure of a database for social network analysis. The basic components consist of the following (I am not sure how many should be tables, queries, etc, so I use the term "component"):

  1. One component should have the names of all the people in the network. I assume this should be a table.
  2. Another component should record the relations between them. I am not sure whether this should be a table or a query.


I illustrate my dilemma below with some abstract "names":

Names of people in social network:
A
B
C
D
E
F
G

Relations between the people:
A likes B
B likes C
D like G
G likes F
G likes A

Note that the "likes" are mutual, so that if "A likes B", there is no need to record that "B likes A" too.

In the format of an Access table, the relations listed above should be listed as records in two columns, as follows:
A B
B C
D G
G F
G A

From the sample records of relations above, you will note that A, B and G occur in both columns. I need this flexibility when inputting the relations records.
I will use the database to find all the likes of a particular name. For example, if I ask for all of A's likes, I want to get a list as follows:
B
G
But, as you can see from the relations, above, B and G occur in different columns when liked by A. So Access needs to scan both columns in order to find all of A's likes, and then list them as one column.

So, how do I program my names, relations, and the search for any one name's likes.

Any assistance would be most welcomed.

PS Incidentally, there are plenty of SNA softwares available, but nothing on programming Access for SNA. I do not want to be the prisoner of some prefabricated black box. I want the whole thing done in Access where the programming may be transparent and easily evaluated.
john w. vinson say:

This kind of relationship is called a "many to many self-join": each Person can Like zero, one or many other Persons. The typical structure of this involves two (or three) tables:

People

   PersonID (Primary Key, autonumber for example)

   LastName

   FirstName

   <other biographical information, contact information, etc.>

RelationshipTypes 

  TypeID (Primary Key)

  RelationshipType (e.g. "Like", "Dislike", "Spouse", whatever is needed)

Relationships

  Person1ID (link to People, first person in the relationship)

  Person2ID (ditto, second person)

  TypeID (link to RelationshipTypes, what kind of relationship is this)

You can implement this using a Form based on People with a Subform based on Relationships; the Subform would use PersonID as the Master Link Key and Person1ID as the Child Link Field, and contain a combo box to select the TypeID and another combo bound to Person2ID. You may want to add some code to ensure that the two PersonID fields aren't identical (unless narcissists are included :-{) ); you can create a unique Index on the combination of Person1ID and Person2ID to ensure that each relationship is entered only once.

Note that the relationships are directional - it's perfectly possible for person A to "Like" person B, while Person B does not "Like" A, or even "Dislikes" him.
