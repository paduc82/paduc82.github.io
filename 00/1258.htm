What do you mean by "data" in this context?  If you want to count all the cells in column A containing constants or a formula, or both, you can use something like:

Set myRange = Range("A:A")
With myRange
    lCellCount = .SpecialCells(xlCellTypeConstants).Count + _
                    .SpecialCells(xlCellTypeFormulas).Count
End With

If you only want to count cells with constants + those with formulas that show a non-null result, you could try something like:

lCellCount = 0
With myRange
    Set myData = Union(.SpecialCells(xlCellTypeConstants), .SpecialCells(xlCellTypeFormulas))
    For Each myCell In myData
        If Len(Trim(myCell.Text)) > 0 Then lCellCount = lCellCount + 1
    Next myCell
End With

If you want something else, define what you mean by Data.

Ron wrote:

With myRange
    lCellCount = .SpecialCells(xlCellTypeConstants).Count + _
                    .SpecialCells(xlCellTypeFormulas).Count

Caveat lector:  I think it would be prudent to have an On Error statement before that code.  If there are no cells the meet the criteria, range.SpecialCells raises an error instead of simply returning zero.

More importantly, range.SpecialCells has some limits.  If the limts are exceeded, range.SpecialCells silently returns the entire range instead of raising an error.

I will not attempt to articulate the limit.  But the following experiment demonstrates one extreme.

Ensure that column A is empty.  Enter 1 into A1.  Copy A1:A2 and paste into A:A, creating a column of alternating values and empty cells.

Range("a1:a16384").SpecialCells(xlCellTypeConstants).Count correctly returns 8192.

But Range("a1:a16385").SpecialCells(xlCellTypeConstants).Count returns 16385, and Range("a1:a16385").SpecialCells(xlCellTypeConstants) includes non-constant cells.

Similarly for any number greater than 16385.

I am using Excel 2007.  YMMV.
