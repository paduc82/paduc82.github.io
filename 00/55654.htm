"Internet Protocol (TCP/IP)" vs "Internet Protocol Version 4 (TCP/IPv4)"

anannya podder say:

 Hi Justin,

There are currently two version of Internet Protocol (IP): IPv4 and a new version called IPv6. IPv4 (Internet Protocol Version 4) is the fourth revision of the Internet Protocol (IP) used to identify devices on a network through an addressing system. It works the same as the earlier version with enhanced features. To specify the versions of TCP/IP it is has the number 4 and 6 after TCP/IP. Your computer should work the same as it worked with a previous version of Windows.

Let us know in case you need any help on this issue or any issues related to Windows.

justin heinecke say:

I am well aware that "Internet Protocol (TCP/IP)" is in fact "Internet Protocol Version 4 (TCP/IPv4)", that is, the old style assumed version 4.  The problem is that in Windows 8, it's not called "Internet Protocol (TCP/IP)", it's called "Internet Protocol Version 4 (TCP/IPv4)" which is not the same string.  Some software written in XP days explicitly searches for the string to determine if you have tcp/ip installed.  Not finding the correct string, the software refuses to continue.

So for compatibility, there needs to be a way to install, well, a virtual protocol called "Internet Protocol (TCP/IP)" which is just an alias for, or a link to, "Internet Protocol Version 4 (TCP/IPv4)" so that the software is satisfied that you have installed the protocol it wants.  Thereafter, I'm sure the software doesn't care, and just makes API calls which default to the version 4 routines.

By changing the name of the protocol in Windows 8, it is not backwards compatible with software whose authors had no reason to suspect they had to differentiate v4 from v6.  What if all the software looking for USB support suddenly had to specify 1.0 or 1.1 or 2.0 or 3.0 and if not specified, wouldn't work?  There has to be a default, a fallback, or else an alias or link, a "fall forward" if you will.

So, any idea how to do this?
