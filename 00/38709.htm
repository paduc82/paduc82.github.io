Your question is far too vague/broad.

Webpages are written using a "language" called HTML (HyperText Markup Language).  See, e.g., http://www.yourhtmlsource.com/starthere/whatishtml.html

You can use Notepad to manually create HTML, but that's not very easy for anything but fairly simple webpages.  See, e.g., https://www.w3schools.com/html/html_editors.asp That's why programs called HTML editors were created.  See, e.g., https://en.wikipedia.org/wiki/HTML_editor

Various word processor programs are capable of opening, editing and saving HTML documents.  Microsoft Word is one of these, but it's not very good (it inserts a bunch of extraneous HTML code).

You could, of course, open your Notepad document in one of these word processors like Word and then "save as" an HTML document, but that might or might not result in something that you actually could use on a website.

This is the method I use for Windows 10 Home Edition (No group editor). It involves moving system files:

Disabling Windows 10 updates by renaming + moving system files:

Windows 10 basically ignores metered settings, services.msc, ignores task scheduler, ignores the registry keys. As of now, all those ways do not work. The way I'm using that in Windows 7. Warning: this method may damage your system. Here are the 5 files you need to move, located under /Windows/System32:

  • wuapi.dll
  • wuapihost.exe
  • wuauclt.exe
  • wuaueng.dll
  • wuautoappupdate.dll

Also, rename them and add a new extension just for good measure. Ideally, you also want to delete the /Windows/SoftwareDistribution, as well as any UpdateAssistant folders within the Windows folder. As the other answers said, it is also good to disable the windows update service afterwards (although doing so alone will not prevent windows from updating). After doing so, it shouldn't update again. If it does (probably due to residual files), start it in safe mode, then delete folders within Software Distribution, as well as Update assistant folders.

I've tried this method on two computers. Update works again when I move the files back and change them back to their original names.

Changing permissions:

The permissions are tricky. To get possession of the files, go to Properties > Security. Click advanced. Change change at the top of the Advanced Security Settings dialog to change ownership. Type in your username into the blank box that appears and hit enter. Now click OK until you exit. Now go back into the Advanced Security Settings dialog. Now the options to edit won't be grayed out. Click on administrators, and click edit. Make sure it is in allow mode, which it is in by default, and check the box that says full control. Now click OK all the way until you exit. You can now move and rename this file.

Alternate method (new)

StackExchange's SuperUser site has some really high quality answers on this topic too. One answer recommended an external software which sets up a fake WSUS server, which essentially make Windows try and download the updates from a fake update server. This is way less intrusive than the method above.

Windows 7 "Libraries" are just a mechanism for looking at related folders.  They don't actually duplicate files.

Open your "Pictures" library and click on the link next to "Includes"

In the window that opens, you can add or remove any "location" (i.e., folder) from the library.  If you right-click on a location, you can move it up or down in the list (which will affect how the library display will work, or set one of the locations as the default save location.  There's also a link to "Learn more about libraries," which you probably should click to clarify things.
