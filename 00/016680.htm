formula in excel to compare a number to a range of ranges of numbers, - Microsoft Community

john_hst say:

formula in excel to compare a number to a range of ranges of numbers, find the range that includes the number and then allocate a score.

Example

Starting with a cell containing the number 450, compare it to a column of ranges e.g. 0-10, 30-70,300-500 etc . Find the range that includes 450 and allocate a score representative of that range e.g. 300-500 could have an associated score of 8. Starting with 450 I want to be able to enter 8 in another cell. Any ideas would be most gratefully accepted!

tom ogilvy say:

create a table with your ranges and scores. 

But the table should be

0 1
11 2
31 3
70 4
90 5
100 6
300 7
500 8
700 9

and so forth, using the lower bound of your range.   The vlookup with fourth argument of True will find the largets number that does not exceed the number being looked up and requires the leftmost column to be sorted (as shown). 

so since the second number is 11, 0-10 will have a score of 1.   11-30 will have a score of 2. 

assume these start in D1 and E1

   Then you can use   =Vlookup(450,D:E,2,True)

or using a cell to hold the number to be scored, assume A1 holds 450

=Vlookup(A1,D:E,2,True)

scottgem say:
First, as the others have mentioned, adding a parameter to the parameters list  doesn't make use of the parameter. You have to assigned the parameter in a WHERE clause.

But that doesn't really answer your question. Every time a query is referenced, any parameters will be prompted for. So if a query is a sub-query that is called by other queries, each time its called the parameter will be prompted for.

This is one of the reasons I almost never use parameter prompt queries. Instead I prefer to enter my parameters using a form and then. referencing them as the criteria. For example you could have a form with an Option group to select which Quarter. In your query you would set the criteria for the Quarter column to:
=Forms!formname!optQuarter

where formname is the name of your form and optQuarter the name of the Option group. Then each time the query is called it will look to the form and not prompt you.
