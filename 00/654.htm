As Doug suggests, from your description the likely cause is that you have a hanging indent (which by default has an associated tab stop) that is farther to the right than some of your very short TOC entries. If the entries that are exhibiting the problem are only those that are quite short, then this is almost certainly the issue. If you have the horizontal ruler displayed, then, when you click in one of the problem entries, you will probably see that the hanging indent (or stray tab stop) is the cause of the problem.

If you are not using this hanging indent or could get by with a smaller indent, you can drag the indent marker to the left, possibly all the way to the left margin. If you do need the indent at that location, then there are other solutions, so post back for further information.

You need to change the Const declaration to point to the location you want to store the file at.  The file does not have to exist yet - it will be created if it doesn't already exist, but the path to it must already exist.

'change this to a path accessible to all users of the file
'best to use the full network reference than to use
'a mapped drive letter in case the location is not mapped when the file is used.
Const recordFile = "\\OurSERVER\Folder1\Folder2\useLog.txt"

Private Sub Workbook_Open()
  Dim fileBuffer As Integer
  fileBuffer = FreeFile()
  On Error Resume Next ' in case network resource not available
  Open recordFile For Append As #fileBuffer
  Print #fileBuffer, "File: " & ThisWorkbook.Name & " Opened by " & Application.UserName & " : " & Now()
  Close #fileBuffer
  If Err <> 0 Then
    Err.Clear
  End If
  On Error GoTo 0 ' clear error trapping
End Sub

Private Sub Workbook_BeforeClose(Cancel As Boolean)
  Dim fileBuffer As Integer
  fileBuffer = FreeFile()
  On Error Resume Next ' in case network resource not available
  Open recordFile For Append As #fileBuffer
  Print #fileBuffer, "File: " & ThisWorkbook.Name & " Closed by " & Application.UserName & " : " & Now()
  Close #fileBuffer
  If Err <> 0 Then
    Err.Clear
  End If
  On Error GoTo 0 ' clear error trapping
End Sub

Private Sub Workbook_SheetChange(ByVal Sh As Object, ByVal Target As Range)
  Dim fileBuffer As Integer
  fileBuffer = FreeFile()
  On Error Resume Next ' in case network resource not available
  Open recordFile For Append As #fileBuffer
  Print #fileBuffer, Sh.Name & "!" & Target.Address & " altered by " & Application.UserName
  Close #fileBuffer
  If Err <> 0 Then
    Err.Clear
  End If
  On Error GoTo 0 ' clear error trapping
End Sub

If the workbook does not already have any macros in it, you're not going to be able to hide the change in operation unless you set your security up so that the workbook can be opened without prompting the user to enable content.  In Excel 2007 you can do that by declaring the folder that the workbook resides in as a Trusted Location.  I believe you would have to do that on all individual systems that are going to use the workbook.

Another way to do much the same thing would be to add a worksheet to the workbook and go into the VB Editor and set that sheet's Visible property to xlSheetVeryHidden which will keep it from appearing in the list of sheets that can be unhidden in the user interface.  Code would write entries to that sheet much as the code above writes entries to the text file.

Drawbacks would be that the file size is going to increase as the number of entries on that very hidden sheet grows.  And while the number of rows available in Excel 2007 is over 1 million, there still comes a point where the sheet may fill with entries and either the code has to be able to deal with that situation or you have to go in (throught the VB Editor) and make the sheet visible and delete rows of entries to make room for more.
