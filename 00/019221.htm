jay freedman say:

That's probably just Word being "helpful" and trying to protect you from macro viruses. The more recent versions of Word have lots of little gotchas for that.

The first step is to store the AutoCorrect.dotm in a folder that Word considers to be a "trusted location". To see the list of trusted locations, go to File > Options > Trust Center > Trust Center Settings > Trusted Locations. Probably one of the default locations is the User Templates folder. Click on that line, and the full path will be shown below the list box. By default it will be C:\Users\<your name>\AppData\Roaming\Microsoft\Templates.

Because the AppData folder and everything under it is marked as hidden, you may have trouble reaching it in File Explorer. If you don't see it listed in the <your name> folder, go to the View tab of File Explorer and check the box for Hidden Items.

When you store the AutoCorrect.dotm file in the Templates folder, right-click it and click Properties. If there is an Unblock button in the General tab of the dialog box, click it.

The second step is to tell Word you don't want it to block all macros. Go to File > Options > Trust Center > Trust Center Settings > Macro Settings and select the option button for "Disable all macros except digitally signed macros".

Now, when you open the template or base a new document on it, the button box should appear.

Hi Dragon,

 

That's because in the History Tab there are three sections, Detected Items, Quarantined Items, and Allowed Items.  You are only looking in the Detected items section which is simple a list and you can do nothing there but delete the list or view it - you can affect no files or do nothing with them.  By the way, it remains for only 30 days and then the items are automatically removed (and there's no way to control that or change the timing).

 

Just under the Tabs are three radio buttons.  Most likely the one marked for you is Alll detected Items.  Click instead on the radio button marked Quarantined items only.  That will bring up a view of all items in quarantine.  If your item was quarantined within the last 30 days, it should still be showing there.  Click on it and the greyed out buttons at the bottom will become more visible and will include remove item, remove all items, and restore item.  If you want to restore it, then while it is still highlighted, click on restore.  That's it.  If it's not there, then it was probably removed.  See the P.S. below on methods to possibly still recover removed items (though it is sometimes a simple matter of luck).

 

Now, to prevent it from happening again, go to the Settings Tab and click on excluded files and folders and add that file to the list.  Now future scans will ignore it since it is a false positive and it won't be quarantined again either.

 

I also recommend the following settings in general because they can come in handy later and are the best way to setup MSE so you have the most options and flexibility:

 

Make sure that ALL threat levels are sent to Quarantine where you can then decide whether or not to allow them or leave them there, or remove them. It is the safest solution for having the most options for dealing with the situation. Quarantine is as safe as removed, but in some cases you may want to remove them to be more certain - and with this one I would remove it (when the time comes). To do this, open MSE, click on the Settings Tab, and for each of the threat levels, set the action to be Quarantine and save.

 

Go to the History tab and click on allowed and unless you specifically placed something in allowed, remove everything else from the allowed category. Only put things there that you ABSOLUTELY know are not problems (are "false positives") and triple-check before you do it as doing this could allow a real threat to bypass security and infect you and it won't be detected or rather won't be reported or acted upon by MSE.

 

Go to the Settings Tab and click on Real-time protection and check all the boxes and save.

 

Go to the Settings Tab and click on SpyNet and click on Advanced and save. This is not at all dangerous to you and no information sent will be used by Microsoft to identify you even if some minor identification information is sent in terms of defining the infection, but it can sometimes help with new threats by initiating an immediate update to deal with the problem if it is in the SpyNet database (and it helps Microsoft update definitions to provide you and everyone else better protection).

 

Go to the Settings Tab and click on Advanced and check the box for save Quarantine items and change it from one month to three months so you have more time to decide if you want to restore something.

 

I hope this helps.

Hi,

IMHO your error catching is not correct.

If Not Intersect(Target, Range("B4")) Is Nothing Then
      Application.EnableEvents = False
      Run "AUTO_TEXT_SEARCH"
      Run "RESET_CF"
      Run "Formulate_CF"
      Run "RELABEL_1"
      'Run "SORT_CF"

        
      On Error Resume Next
      Application.EnableEvents = True
   End If

Errors occurring in the below routines will no be catched by "On Error Resume Next"

Run "AUTO_TEXT_SEARCH"
      Run "RESET_CF"
      Run "Formulate_CF"
      Run "RELABEL_1"
      'Run "SORT_CF"

I think it is better to add an error routine:

Option Explicit

Private Sub Worksheet_Change(ByVal Target As Range)
   On Error GoTo Err_Handler
   Application.EnableEvents = False
   
   If Not Intersect(Target, Range("B4")) Is Nothing Then
      Run "AUTO_TEXT_SEARCH"
      Run "RESET_CF"
      Run "Formulate_CF"
      Run "RELABEL_1"
      'Run "SORT_CF"
   End If
    
   If Not Intersect(Target, Range("J4")) Is Nothing Then
      Run "DO_J"
      Run "RESET_KN"
      Run "Formulate_KN"
      Run "RELABEL_2"
      'Run "SORT_KN"
   End If
Exit_Sub:
   Application.EnableEvents = True
   Exit Sub
Err_Handler:
   Resume Exit_Sub
End Sub
