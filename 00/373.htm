hansv mvp say:

Closing the workbook probably interferes with the clipboard. Try this version. It doesn't require that the name of the master workbook begins with "z".

Sub LoopThroughDirectory()
    Dim MyFile As String
    Dim erow 'erow stands for empty row
    Dim wbk As Workbook
    Dim wsh As Worksheet
    Application.Cursor = xlWait
    Application.ScreenUpdating = False
    Set wsh = Sheet1
    erow = wsh.Cells(Rows.Count, 1).End(xlUp).Row 'finding the last used row
    MyFile = Dir("C:\Users\XXX\Documents\YYY\*.xls*")
    Do While Len(MyFile) > 0 'we will check the files if they have a certain length'
        If LCase(MyFile) <> LCase(ThisWorkbook.Name) Then 'I dont want that our master file is opened.
            Set wbk = Workbooks.Open("C:\Users\XXX\Documents\YYY\" & MyFile)  'the file is opened'
            erow = erow + 1 'increase the row number
            wbk.Worksheets(1).Range("A1:D1").Copy Destination:=wsh.Range("A" & erow) 'copy the range
            wbk.Close SaveChanges:=False ' we close the workbook
        End If
        MyFile = Dir 'looking for the next file
    Loop
    Application.ScreenUpdating = True
    Application.Cursor = xlDefault
End Sub

hansv mvp say:

Select cell A1.

Activate the Formulas tab of the ribbon.

Click Name Manager.

Click New..

Enter GetFormula in the Name box.

Enter the formula  =GET.CELL(6,!A1)   (yes, exactly like that) in the Refers to box.

Click OK, then Close.

 

Select the cells that you want to format conditionally.

Activate the Home tab of the ribbon.

Select Conditional Formatting > Manage Rules...

 

Click New Rule...

Select 'Use a formula to determine which cells to format'.

Enter the formula   =ISNUMBER(SEARCH("Sheet1",GetFormula))

Click Format...

Activate the Fill tab and select a suitable highlight color, e.g. pale blue.

Click OK twice.

 

Repeat the above series of steps from 'Click New Rule...', but with "Sheet3" and pale red, etc.

 

When you have defined all the rules you want, click OK to close the Conditional Formatting Rules Manager.

ken sheridan say:

Never enter data directly in a table's datasheet.  Create a form bound to the Orders table and for the CustomerID use a combo box set up as follows:

 

ControlSource:    CustomerID

 

RowSource:     SELECT CustomerID, [Customer Name] FROM Customer ORDER BY [Customer Name];

 

BoundColumn:   1

ColumnCount:    2

ColumnWidths:  0cm

 

If your units of measurement are imperial rather than metric Access will automatically convert the unit of the last one to inches.  The important thing is that the dimension is zero to hide the first column.

kathy s - microsoft support say:
Hi O.C. Jayne,

The mouse scroll wheel behavior was intentionally changed in Access 2007 to reduce user confusion. In previous versions, there were instances where the scroll wheel would scroll the scroll bar and others where it would navigate records. The scroll wheel behavior is now consistent and only scrolls the scroll bar. There is no property to change this behavior.

In addition, there were  many, many questions about how to disable the mouse wheel behavior for forms that led to this decision. So many that Stephen Lebans even posted a DLL that users can include in their Access apps to turn it off.

You could use code similar to the following to allow a form to scroll between records. However, since there are no events for tables and queries, this would not work for those objects.

Add this code to the form’s On Mouse Wheel event procedure:

Private Sub Form_MouseWheel(ByVal Page As Boolean, ByVal Count As Long)
  If Not Me.Dirty Then
      If (Count < 0) And (Me.CurrentRecord > 1) Then
          DoCmd.GoToRecord , , acPrevious
      ElseIf (Count > 0) And (Me.CurrentRecord <= Me.Recordset.RecordCount) Then
          DoCmd.GoToRecord , , acNext
      End If
  Else
      MsgBox "The record has changed. Save the current record before moving to another record."
  End If
End Sub

You will have to adjust the code above if your form does not allow additions or is read only. It assumes you can scroll to a New record. You would just need to adjust the condition in the ElseIf statement.

If the record has been updated, you will likely want to force the user to save the record before scrolling. That was one of the main reasons for this change – users would scroll the mouse wheel and automatically and unexpectedly save their changes to their record.

Hope this helps!
Kathy
