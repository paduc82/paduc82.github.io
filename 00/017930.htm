ossiemac say:

I am assuming that when you say Table that it is a table created with Insert -> Table.  Is this assumption correct?


I am also assuming that the data gets dropped into the table on the row below the headers. Is this assumption correct?


 If my assumptions are correct then the following clears any existing data from the DataBodyRange of the table and then Resizes the table to the column headers plus one row and then pastes the new data in the table starting at the first cell under the first table header. The table is automatically resized to accommodate the data.

 

Sub ResizeTable()
    Dim lstObj As ListObject
   
    Set lstObj = Sheets("Sheet1").ListObjects("Table2")
   
    With lstObj
        .DataBodyRange.ClearContents
       
        'Following line resizes the table to headers Plus 1 row
        .Resize lstObj.Range.Resize(2, .Range.Columns.Count)
       
        'Following line for test purposes.
        'The destination is the row 2 column 1 of lstObj (The table)
        Range("M2:T20").Copy Destination:=.Range.Cells(2, 1)
    End With

End Sub

matt neerincx [msft] say:

How do I install Visio with Office 365?

I'm trying to install Visio.   I installed Office 365 and it just runs and does not give me any options to select anything and then finishes and Visio is not there.   How the heck do I install Visio?
frostgiants say:

Maybe I am oversimplifying the question, but if you are logged into the Office 365 Portal, click on the GEAR icon in the upper right hand side of the screen.  This will take you to the account settings page.  Click on Subscriptions and make sure that Visio is shown there (this is the O365 Subscription that is assigned and NOT a standalone).  Next click on Settings.  You will see an option for INSTALL SOFTWARE.  Click that and then choose Visio (for me it is the third item down on the left hand side of the screen.  You should then be able to install the software with no problems.
