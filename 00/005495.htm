See a more complete explanation at http://answers.microsoft.com/message/db9436d5-3329-4afd-8b28-f21ffc89867e?threadId=0ba6fe3a-4bc6-4d98-a7f5-4183d6b5f1f0.

In a nutshell....  When you open the CSV file, the cells are formatted as General.  That format displays up to 11 characters -- in this case, 10 digits.  When you save to a CSV file, Excel writes the numbers as they are displayed.

The remedy:  after you open the CSV file, change the cell formats accordingly.  In this case, you might want Number with 2 decimal places.

Better still:  do not use CSV files to save Excel data if you intend to open them in Excel later.  By definition, CSV files lose all formatting information.  Instead, you should save to Excel to retain formatting information.

(Saving to XML might also work if you need to "transfer" Excel data to another application.  Of course, the other application must support XML input.  But I have not experience with XML files.)

joeu2004 say: bgensch wrote:
I should add something to my initial post to help make more sense.  I discovered this digit change upon seeing the results of the upload from CSV file to the accounting software.  Debits and Credits were not in balance because the value that uploaded into the software was 112978078.3 instead of 112978078.33.  Through experimenting, I found the pinpoint cause of the full value not being uploaded was from just saving the CSV file after copying data into the CSV file from Excel.

Correct.  If you look at the initial CSV file using Notepad, you will see that 112978078.33 is actually in the file.  And that is actually what Excel inputs the first time; you can confirm by looking at the value in the Formula Bar.

The value only appears as 112978078.3 because of Excel's display rules for the General format; namely, Excel will display only up to 11 characters, which might be further restricted due to column width.

When you save to a CSV file, Excel writes what is displayed, not the actual value.  So if you look the resulting CSV file using Notepad, you will see that 112978078.3 is truly in the file after re-saving.

As noted previously, the only work-arounds are:  (1) change the cell (column) format after opening or importing a CSV file; or (2) save to an Excel file, not a CSV file, after opening the CSV file the first time.

hosch250 say:

Create a new notebook at the new SkyDrive account.  Right-click each section of the old notebook and select the Move or Copy (CTRL-ALT-M) link.  Move each section, you needn't do each page - they move with the sections, to the new notebook.  It will create a new section, and you will just need to wait for them to upload to the server.  You can then delete your old notebooks, or just not use them and close them in your computer, from your old account.  I had to do this fairly recently myself.
