Error 0x80246019                     

 

0x8024600B WU_E_DM_CONTENTCHANGED A download must be restarted because the update content changed in a new revision.
0x80246FFF WU_E_DM_UNEXPECTED There was a download manager error not covered by another WU_E_DM_* error code.

The only clue you have otherwise is that (most likely) it has something to do with downloading by Windows Update.  So a reasonable guess would be that this could be a sign of an interferer and that then you would need to do something about removing that obstruction.

  Sleep 100

#If VBA7 Then
  Private Declare PtrSafe Sub Sleep Lib "kernel32" (ByVal dwMilliseconds As Long)
#Else
  Private Declare Sub Sleep Lib "kernel32" (ByVal dwMilliseconds As Long)
#End If

Try replacing Beep with an API call to play a sound file.  This worked for me and the sound played every time.  The drawback is that VBA does not regain control until the sound is finished playing, whereas with Beep it seems to be able to continue processing which is probably why you don't get distinct beeps in the middle of a loop that's going fast!

Option Explicit

'API Class to play a sound file

Public Declare Function sndPlaySound32 Lib "winmm.dll" _

Alias "sndPlaySoundA" (ByVal lpszSoundName as String, _

ByVal uFlags as Long) as Long

Sub VBASound()

    Dim i as Long

    For i = 1 to 10

        Call sndPlaySound32("C:\Windows\Media\Windows Notify.wav", 0)

        DoEvents

    Next i

End Sub
