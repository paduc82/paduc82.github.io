technogramps say: After several trial and errors I now have the network printer working as it should.  I had to assign a static IP address for the network printer through the Netgear modem,. You can do a Google search on setting static IPs to see how this is done.   I then had to delete the printers  (and related TCP/IP ports) on the computer. Note: You will have to delete any print documents that are in the queue before you can delete the ports.  Finally I had to re-install the printer on the Win 8 desktop.  What was happening was that the modem was assigning different IP addresses based on the order in which my various computers were turned on.  As a result, sometimes the IP address for the network printer in the computer did not match the actual IP address.  This produced the message on my Win 8 computer - "Device is Offline", which was misleading.

andre da costa say:
BIOS is separate from Windows Recovery environment.

The BIOS which stands for Basic Input Output System is a low level part of your computer which manages how Windows loads from your hard disk into main memory. It stores basic information about your peripherals such as mouse, keyboard and keeps the date and time.

See how to load it depending on your make and model:
https://www.groovypost.com/howto/bios-uefi-setu...

The Windows Recovery environment can be accessed several ways.

From within Windows 10
Open Start > Settings > Update & security > Recovery > Advanced startup
Click Restart now

or

Power on and off your computer three times
On the third time, your computer will boot into the Advanced Recovery environment
Click Advanced Options
Click Troubleshoot
Click Advanced Options
