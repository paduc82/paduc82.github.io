diana d say:

Hi,

If you have a FAT32 partition or volume on your computer, any user who has access to your computer can read any file on it. FAT32 also has size limitations. You cannot create a FAT32 partition greater than 32GB in this version of Windows, and you cannot store a file larger than 4GB on a FAT32 partition.

See: http://windows.microsoft.com/en-US/windows-vista/Comparing-NTFS-and-FAT-file-systems

Step 1: You may have to convert it to NTFS file format to backup up large files.

“How to convert a FAT16 volume or a FAT32 volume to an NTFS file system in Windows XP”: http://support.microsoft.com/kb/307881

Also see: “Limitations of the FAT32 File System in Windows XP”: http://support.microsoft.com/kb/314463/en-us

rpdm say:

Hi,

I've just solved this problem on my Dell Optiplex 990, running Windows 10.

It was running in RAID mode (as set in the BIOS) using iaStorAV.sys, and after an update from a hard drive to an SSD it was using often crashing with 100% of disk usage, but with no real activity (0 MB/s).

Having tried so many of the proposed solutions on the 'net (Virtual memory, disabling an interrupt, Windows Defender disable, Skype/Chrome mis-match), I finally went for changing the BIOS mode from RAID to AHCI.  This forced a change of the driver to iaStor.sys instead of iaStorAV.sys and all the problems have gone.

So it does seem as though iaStorAV.sys has a problem, at least in Windows 10 with an SSD.

Note that in order to change from RAID to AHCI in the BIOS requires doing it whilst 'in' Safe Mode - as detailed here:

http://www.tenforums.com/drivers-hardware/15006-attn-ssd-owners-enabling-ahci-mode-after-windows-10-installation.html

I hope this solves someone else's problem.

Richard
