graham mayor say:
You are correct. The rule option to move the message is not included - only a copy of it.

There is in fact no way I can think of to do this as part of a rule and you cannot run a script from a rule that applies to outgoing messages - only incoming messages - so it would have to be a manually triggered process..

Frankly I cannot even think of anything that would make it a good idea to put your sent messages in your inbox; however if you insist you can do it with a macro.

The following will move all the messages from the sent folder to the inbox -

Sub MoveSent()
Dim olItems As Outlook.Items
Dim olitem As Outlook.MailItem
Dim i As Long
Set olItems = Session.GetDefaultFolder(olFolderSentMail).Items
olItems.Sort "[Received]", True
For i = olItems.Count To 1 Step -1
    Set olitem = olItems(i)
    olitem.Move Session.GetDefaultFolder(olFolderInbox)
Next i
cleanup:
Set olItems = Nothing
Set olitem = Nothing
End Sub

but surely there are better ways to handle your messages than this?

robertpladek say:

After months of experimentation and frustration, I have found a solution to fix Microsoft Outlook 2010 "server rejects login" . This really works but it will take you a bit of time and careful planning to repair.

Only try this solution if you experience the condition where your email server (which often comes from the cable company) periodically complains that your email/password was rejected at login.

Then when you simply try to send/receive email again, it works just fine.

 

This solution will assume that Microsoft Outlook will periodically login in OK, and then at other times ask you for your password.

You check the "remember login" box and hit OK and then outlook correctly logs into the server and life works great for a time.

WHAT IS THE ACTUAL PROBLEM?

Outlook.pst got corrupted. And what is needed is to build a new "outlook.pst" from scratch.

This is not impossible to do, but it does take time to do. (Took me over an hour to do on my computer.)

First, I exported all the various sub-folders that contained old emails. Use the file tab, Advanced, Export Windows Tab for each sub-folder you have in your email file system.

Suppose your file system looked like:

outlook

inbox

vacation items

photos

For example saving the folder contents for "vacation items" was placed at c:\vacation_items.CSV

Do the same for the photos sub-folder.

YOU do not need to export the inbox because the server will replenish the inbox when we're done with the fix.

After all the sub-folders have been exported to your C:\ drive, you then need to send yourself an email with all your address contacts. To do this open a new email, address it to your own login and "attach item" "outlook item" and then you'll be able to address the address book list by highlighting the addresses you want to attach to your email to be sent to yourself.

If you don't understand completely how to use this email feature, there are ample examples on Google and YouTube to help you deal with this step.

Then I took my address contacts, and attached them into an email that I sent to myself.

(This action allowed me to rebuild my contact list after I fix the problem.)

I closed the outlook mail application.

Then I went to c:\users\admin\documents\outlook\outlook.pst (or where ever your outlook.pst file lives) and changed the name to old_broken_outlook.pst

Now that the "outlook.pst" file is gone (because it was renamed for backup purposes) when I open Microsoft Outlook, the application complains that no outlook.pst can be found, and it makes a new "CLEAN FILE" from scratch.

Next, I close Microsoft Outlook and go to the directly where the new outlook.pst file is located and make a copy called new_clean_copy_of_outlook.pst Notice that this is a copy of the brand new outlook.pst

Now I open Microsoft Outlook again and create all the file folders that I had in the old corrupted environment. I then use the "file tab - Options - Advanced tab to import each email file folder that will contains all the files I had stored one-by-one in the old corrupted environment.

Next, I clear out all login accounts in Outlook. And I start by adding a NEW Login account with my primary email address. Once this is installed, OUTLOOK may create a separate email file system that is in addition to the "main outlook" email file system. I can go into the file tab and "CHANGE where new email comes into the system back to the file path where outlook.pst lives.

I continue to add my additional email accounts to OUTLOOK. and then change all their .pst files to all be the same outlook.pst path. (A person could also leave all these separate email file systems alone if you wanted to keep various email accounts separate. Personally I use five email addresses and everyone comes back to my "main outlook.pst" file system. )

Final step here. When your primary email account logs back in with the "NEW CLEAN outlook.pst file" you will get hundreds of emails that were kept on the server. Among all that email is the email that you sent yourself with all your address contacts and group contacts.

Open the email and look for the attachment where your contacts/and/groups are located, highlight the entire group and drag the entire set of emails to your contacts file which is hopefully on the left side of your OUTLOOK screen.

Done. Your emails are all in place. You address lists are back in your contacts, and your system now has a clean non-corrupted outlook.pst file to work from.

Close Outlook and make a backup copy of the outlook.pst (For example make a copy that says " new-working_copy_of_outlook.pst"

and just keep that in your file system should you ever need it in the future.

That's it. Long story but it works and will fix the impossible condition of periodic lockouts for outlook.pst

Try this...

Select the *entire* range C3:C102 starting from cell C3. Cell C3 will be the
active cell. The active cell is the one cell in the selected range that is
not shaded. The formula will be relative to the active cell.

• Goto the Home tab>Styles>Conditional Formatting>Manage rules>New rule>
• Use a formula to determine which cells to format
• Enter this formula for the current month in the box below:
• =AND(C3>TODAY()-DAY(NOW()),C3<=EOMONTH(NOW(),0))
• Click the Format button
• Select the desired style(s)
• OK
• OK
• New Rule
• Use a formula to determine which cells to format
• Enter this formula for the next month in the box below:
• =AND(C3>EOMONTH(NOW(),0),C3<=EOMONTH(NOW(),1))
• Click the Format button
• Select the desired style(s)
• OK out
