ken sheridan say:

1.  But, is there a way to attach files to certain student profiles? I would like to scan in their applications.

2.  Also, Is there a way to add more fields to the student file so I can keep track of the forms they have and have not turned in?


1.  You could add an attachment field to the table, but I would suggest adding a text field in which the path to the file can be stored.  You'll find an example of how you can open a dialogue to browse to a file and gets its path, and how to open or print the file, as Browse.zip in my public databases folder at:

https://onedrive.live.com/?cid=44CC60D7FEA42912&id=44CC60D7FEA42912!169

Note that if you are using an earlier version of Access you might find that the colour of some form objects such as buttons shows incorrectly and you will need to  amend the form design accordingly.  

If you have difficulty opening the link copy its text (NB, not the link location) and paste it into your browser's address bar.

This little demo file uses an unbound form, but can equally well be applied to a bound form in which the path can be inserted into a bound control in the form.

2.  If you are thinking of having separate fields for each form then stop right there.  That would be very bad relational database design and you'd soon encounter problems.  The form names or whatever should be stored in a separate table.  You then have to model the many-to-many relationship type between students and forms which you'd do with another table which resolves the relationship type into two one-to-many relationship types by having foreign keys StudentID and FormID, each referencing the primary key of the Students and Forms tables.  Other non-key columns can be added to this table to represent attributes such as the date a student received the form, the date it was turned in etc.

On the Mac, Word can be configured more precisely than on the PC.

First, two caveats.

1.What I am about to explain will not work in the Home and Student edition of Office, but will work in all other editions.

2. Usually the Library folder is hidden. If your User:Library is hidden (which means you don't see a Library folder when you click on User in Finder), then follow these directions in Lion or later editions of Mac OS X to unhide the Library folder before you proceed.

So here's what to do:

1. Open the Automator Application (comes with Mac OS and is in the Applications folder)

2. Choose Workflow at the first screen

3. Drag the workflow called Take Screenshot from the left side to the big white empty space on the right side. Note that there are options for using the entire screen, a selected window, or drag a screen selection. Choose whichever option you want to use.

4. Drag the workflow called Paste Clipboard Contents into Word Documents. Notice there are options as to where the clipboard contents will be placed in the document.

5. In Automator, go to the File menu and choose Save. For the File Format, choose Application. You must save your file in this directory:

User:Library:Application Support:Microsoft:Office:Word Script Menu Items

6. Type an appropriate file name that describes the action, for example ScreenGrab Window to Top of Document and then click the Save button.

7. Start Word (Quit Word and then re-open Word if it was running when you saved the Automator Action).

When you want to use the action, look for it on Word's Script menu.

You can save additional scripts using various combinations of the options you choose.
