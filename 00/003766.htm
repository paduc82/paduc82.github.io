    After cleaning out malware, when updating the Windows Operating System (e. g. Windows updates), files are routinely removed, replaced, overwritten, etc.  Sometimes these files can become corrupted, damaged, misplaced, etc.  Many of the files are protected system files located in the Windows Component store in the WIN SxS folder. The only way to access them to do a clean-up (repair corrupted files/replace missing files, etc) is via the Deployment Image Servicing and Management Tool (DISM), a command line tool.

    Using DISM to check for and repair corruption in the Windows Component Store.

    Close all windows, programs, and apps.
    Press the Windows Key and the X key.
    Select "Command Prompt (Admin)"
    enter      Dism /Online /Cleanup-Image /ScanHealth  and press the Enter key on your keyboard. (There is one space before each (/)

    When DISM finishes running, if you get a message "No component store corruption detected. The operation completed successfully"  this means that the Windows Component Store is intact, and no further action is required. Close the window.

    If instead you get a message "The Component store is repairable. The operation completed successfully",
    enter  Dism /Online /Cleanup-Image /RestoreHealth  and press the Enter key. (space before all (/)
