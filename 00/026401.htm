copy specific columns in a range using criteria - Microsoft Community

utherpendragon say:

copy specific columns in a range using criteria

I have a range of data in Excel with 11 columns.  I want to copy only 7 of the columns to another location using a criteria (i.e. type = "D").  Is there a simple way to do this?  I looked at using MS Query, but that seems only to apply to external data.
shane devenshire say:

The advance filter approach is as follows:

Using Advanced Filter to copy data to a different sheet:

Suppose you have data on Sheet1 with the following column titles starting in A1

Col1 Col2 Col3 Col4 Col5 Col6 Col7 Col8 Col9 Col10 Col11

In Col1 one you have letter codes.  You want to copy all the rows of data which contain a code of D in column A but only for columns 2, 4, 6, 7 8, 9, and 11

1. Copy the column headings for the columns you want to the destination range, save A1 on Sheet2:
  A     B     C     D    E     F     G
Col2 Col4 Col6 Col7 Col8 Col9 Col11

2. In cell J1:J2 enter the title from column A on Sheet1 and the value you are interested in "D":
Col1
 D

3. Put an entry in cell A2 - what you enter doesn't matter.
4. Place your cursor in A1 of Sheet2 and choose Data, Advanced
5. Choose Copy to another location
6. Clear the entry from the List range box and then highlight all the data and titles in the source range on Sheet1
7. Click in the Criteria range box and highlight J1:J2 on Sheet2
8. Click in the Copy to box and highlight A1:G1 on Sheet2
9. Click OK.

Note: Excel remembers some of the information you enter into the Advanced Filter box making it easier to repeat the filter later.

Warning: the range you indicate in the Copy to area may be either the titles or the titles and many rows below them.  If you only pick the titles, as in the above example, Excel clears all the rows below those titles (1,048,576 rows) even though there may be nothing in that area.  And it doesn't warn you.  If you indicate the number of rows for the output Excel will only clear those and will warn you if there is more data to output.

If you are doing this on the same sheet as the source data then you basically do the process the same way but replace step 4 with

4.  Place your cursor in any single cell of the data range (not the Copy to range) and choose Data, Advanced 

 

If this answer solves your problem, please check Mark as Answered. If this answer helps, please click the Vote as Helpful button. Cheers, Shane Devenshire

vitoleja say:

How to subtract dates so I would get the number of months and dates?

I need to subtract dates and find out the difference in months and days. for ex. 10.01.2012 - 11.06.2012 = 6 months 1 day. countdif function doesn't work: tried all possible variants but it gives me an error.

I have tried something like "year (a3) - year (a2)*12+ month (a3)- month (a2)". It counts months only. How to add days there?

also tried separatel to subtract days only, but it gives a negative number if the day in a3 is bigger than in a2.

 

Please help!

ashish mathur say:

Hi,

 

The answer should be 5 months not 6 months.  Try this

 

1. To get months, use =DATEDIF(A2,A3,"ym").  A2 is the beginning date and A3 is the ending date.  Also, both entries should be genuine dates.  Since dot is not a recognised date seperator, you will have to find for dot and replace with / or -

 

2. To get months , use =A3-EDATE(A2,(DATEDIF(A2,A3,"y")*12)+DATEDIF(A2,A3,"ym")).  Answer will be 1

 

For months, in your specific example, you may also use =DATEDIF(a2,A3,"md").  Answer here will also be 1.  However, the "md" form of usage in DATEDIF() is buggy.  To test, it, if your beginnign date in cell A12 is 31/7/2008 and ending date in cell B13 is 02/03/2009, =DATEDIF(A13,B13,"md") will yield the answer as -1 instead of 2.

 

Hope this helps.
