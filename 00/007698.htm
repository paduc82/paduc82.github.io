To make life a bit easier in C1 enter

=20-COUNTIF($B$1:$B$19,"<"&B1)-COUNTIF(B1:$B$19,B1)

and copy down to C19, this will give you a rank of scores High to Low. Where there is a duplicate score the first in the list is ranked higher than the second, but all will be included.

=INDIRECT("A"&MATCH(ROW()-20,$C$1:$C$19,0))

=INDIRECT("B"&MATCH(ROW()-20,$C$1:$C$19,0))

Which will match the scores to the names.

These formulas are specific to the data you gave as an example so if there are more than 19 students it will have to be adapted. But that is just a matter of changing a few numbers. And you can always hide column "C" if you don't want to see it on your sheet.

Just as a note in the first formula change "<" to ">" to sort the list low to high

Steven_1's formula is ROW SPECIFIC. That means if you enter the formula on a row that the formula was not written to be on then you have to make adjustments. For this reason I never make formulas row specific.

Steve D's (and Shane's) formula, while not row specific, is vulnerable to row insertions before the referenced range. This could cause incorrect results. If you will never insert new rows then this isn't a problem. However, the method to prevent or greatly reduce the chance of row insertions causing problems is fairly simple and doesn't add any undue complexity or length to the formula.

Ok, let's figure out what you want to do. You mentioned that some scores are "nil". I assume nil means numeric 0?

Try these...

Enter this array formula** A21:

=INDEX(A$1:A$19,MATCH(LARGE(B$1:B$19-ROW(B$1:B$19)/10^10,ROWS(A$21:A21)),B$1:B$19-ROW(B$1:B$19)/10^10,0))

** array formulas need to be entered using the key combination of CTRL,SHIFT,ENTER (not just ENTER). Hold down both the CTRL key and the SHIFT key then hit ENTER.

Enter this formula in B21:

=LARGE(B$1:B$19,ROWS(B$21:B21))

Select both A21 and B21 then drag copy down to A39:B39.

As you fill the formula down it then looks for number 2 in the list in column C1:C19 etc...

beth melton say:

Suzanne may have an idea for you but I'm fairly certain there isn't an alternate method for what you want to accomplish. How text is justified depends on text 'container', not the text itself. If you try to pad it with spaces or something there's noguarantee the same 'right edge' you see in Word (or alternate program) will be right edge elsewhere.

Here's an example of what I'm referring to:

  1. In the Notepad copy/paste some text from Word or add a bunch of text
  2. From the Format menu, make sure Word Wrap is turned on (it will have a check to the left of the command)
  3. If necessary, resize the Notepad window so the text wraps to the window
  4. Add some spaces to 'justify' a few lines of text
  5. Resize the Notepad window and note how the text wraps to the new size. More than likely you'll see holes in the text from the spaces you added and a new 'right edge'. Plus I suspect the holes make it look worse than the jagged right edge. ;-) 

You really can't control where text will begin and end on a line. Even in Word your text is essentially plain text that obtains its appearance (formatting) from the container the text is placed in. For example if you change your margins in a Word document, add additional words, even changing to another printer can cause the text to re-wrap. The Justify function in Word is what 'pads' the text to remove the jagged edge and it's always making modifications for you.

Now, rich text means text can be saved with character formatting codes "wrapped" around the text, such as bold/italic/underline. If you transfer the text elsewhere and if the containing application understands those codes it will translated and the text formatted accordingly.

So...how the text appears is up to the containing application. I realize that's not the answer you want, but I'm 99% certain that's the only answer.

cindy meister say:

Hi Rich

I'm not sure I'm going to be able to get to the bottom of this in one go-through, but...

The cross-reference feature bases on Word REF fields. You can toggle the field codes on by pressing Alt+F9.

The checkbox to which you refer is equivalent to the \* MergeFormat switch you may see in some of these fields. It retains formatting when the field is updated. This switch is a two-edged sword, and I usually use it only for tables linked in from Excel (where the number of rows or columns won't change), linked databases (same comment) and possibly linked pictures.

For anything to do with text I definitely remove it. And I often replace it with the switch \* CharFormat. CharFormat forces the field to display its content with the character formatting applied to the first character of the field, itself. I think this may solve the problem for you.

So: alt+F9 to see the field codes. Replace MergeFormat with CharFormat. Alt+F9 again to see the field results, then Ctrl+A (Select All) followed by F9 to update the field results.

Note: you may want to SAVE the document first in case you don't like the result :-)

talking moose say:

Consider all this "file management". File management gets you a few things:

  • Deleting unwanted messages helps curb the growth of your database and actually helps it reach a plateau at some point where it will pretty much stop growing with consistent delete behavior.
  • Filing messages into folders away from the Inbox improves synchronization with the server. This folder is updated more than the rest and therefore synchronizes more often.
  • While I prefer to search for old messages rather than file them away in a complex hierarchy of message folders, a few large folders of messages is easier to browse and synchronize than a single humongous folder.

The best advice I can give is to treat each email as a document.

  • Manage those documents as you would manage a lot of files on a server or on your physical desktop.
  • Purge what you don't need.
  • Move very old messages to an infrequently accessed storage location or archive.
  • Treat your Inbox above all other folders like a revolving door or even the inbox on your desk. Don't let things pile up. Process an item and then move it somewhere else when done.

Hope this helps!
