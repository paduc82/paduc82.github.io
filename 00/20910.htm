a. user say:

Switch from E3 to E1 Office 365 licenses

 What is possible risks when switching from E3 to E1 license? What will happen to all data? What is restrictions of switching from E3 to E1 and possible risks? For example if user has more than 50 GB mailbox usage or more than 1TB uploaded data, what will happen after switching from E3 to E1 license?

chris peng msft say:

Hi s1452,

 

For Office 365 E1 licenses, the storage of the mailbox (including the primary mailbox and the archive mailbox) is limited to 50GB. The storage of OneDrive for Business is limited to 1TB. If the user has more than 50 GB mailbox usage or more than 1TB uploaded data, switching from E3 to E1 license would cause data loss or other errors. In this situation, we suggest you back up the data and limit the usage within capacity of E1 license.

 

However, to avoid the risk of data loss, the best practice is to back up all the data before you switch from E3 to E1 license even the usage is not beyond the capacity of E1 license.

You'll have to use SEQ fields. To repeat the current heading number (for a specified level), you can use a STYLEREF field. For easier insertion, you can save the various numbering formats as Quick Parts (Building Blocks). A few examples:

[{ STYLEREF 1 }.0.0.{ SEQ myseq }]      <----- below level Heading 1 paragraphs

[{ STYLEREF 2 }.0.{ SEQ myseq }]    <------- below Heading 2 paragraphs (if numbering shouldn't continue, use a different SEQ field "name")

Excel 2007: Faster way of inserting "." between cells - Microsoft Community

seng kuang say:

Excel 2007: Faster way of inserting "." between cells

Hi,

 

When I insert "."between cells to create a code of 201.PU3.654.KLP,

I use formula =B2&"."&C2&"."&D2&"."&E2

May I know if there is a simpler and faster way to do this?

We use Text to Column Toll to split codes seperated by a dot into columns, is there a way of doing the reverse?

 

Thanks

Low

 

 

jeeped say:

... I use formula =B2&"."&C2&"."&D2&"."&E2 ... May I know if there is a simpler and faster way to do this? 


You could consider adding a User Defined Function or UDF.

Tap Alt+F11 and when the VBE opens, immediately use the pull-down menus to Insert, Module. Paste the following into the new pane titled something like Book1 - Module1 (Code),

Public Function fcnStitchTogether(rBits As Range, sBit As String, Optional bBlanks As Boolean = False) As Variant
    On Error Resume Next
    Dim rCell As Range, sStitched As String
    sStitched = ""
    For Each rCell In rBits
        If (Len(rCell.Value) And Not bBlanks) Or bBlanks Then _
            sStitched = sStitched & sBit & rCell.Value
    Next rCell
    fcnStitchTogether = Right(sStitched, Application.Max(Len(sStitched), 1) - 1)
End Function

Tap Alt+Q to return to your worksheet. Use like any other function, e.g.:
fcnStitchTogether(<range of cell to concatenate>, <delimiting character>, [optional]<add blanks>)
... or as per your sample,
=fcnStitchTogether(B2:E2,".")
... if there are blanks in B2:E2 that you do not want tot skip over then,
=fcnStitchTogether(B2:E2,".",TRUE)

                      

You will have to save your workbook as macro-enabled, e.g. XLSM.
