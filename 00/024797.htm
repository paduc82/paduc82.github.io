remy_heron say: I experienced recently the same problem on my labtop with a Belkin bluetooth key.
Finally, I found the solution on the web at this forum http://forum.eeeuser.com/viewtopic.php?id=38533

There is a conflict between the widcomm tools coming with the widcomm driver and PC Suite.
The solution is to use the Microsoft Driver instead of the Widcomm driver.

To do so, you need add a line into the C:\Windows\inf\bth.inf.
First, call Device Manager" (execute Devmgmt.msc).
Edit properties of your bluetooth key.
Find the manufacturer and find the identifier like USB\Vid_xxxx&Pid_xxxx

Then, edit with notepad C:\Windows\inf\bth.inf

In the bluetooth section of your manufacturer, add this line :
[MyManufacturer.5.1]
MyBlueTooth Key= BthUsb,USB\Vid_xxxx&Pid_xxxx

Save your file with same name.

Remove your bluetooth key.
Uninstall Widcomm driver.

Then, plug your key.
Go to your device manager and ask to change the driver.
Choose the option to select manually your driver.
Choose the one you declare into bth.inf.

It will be ok.

Re: #1 - You can't configure the same email account using a different protocol in Outlook '2016

Re: #2 - Removing the email account from Outlook does just - it removes it from Outlook. However, if you're asking if it otherwise impacts the actual email account being accessible etc - answer is: No. Also, removing a POP account from the Outlook profile doesn't remove the associated PST file from the profile - that remains unless/until you explicitly you remove it which you don't want to do for reasons mentioned later

Re: #3 - To change email account settings - do the following
a) Press CTRL+Alt+S to get the <Send/Receive> window
b) Assuming you only have one S/R group - click on <Edit>
c) Select the email account you want to change --> click on <Account Properties>
(*** not something you're going to need for the purposes of this exercise)

Re: #4 - Can't speak about anyone else having already created new articles but something that's on my to-do list

Now for what you need to do

#1 - With Outlook closed, open the Control Panel <Mail> app --> click on <Show Profiles> --> click on <Copy> --> Give it a name. ***this is done purely as a precaution so that you have a copy of the original profile
#2 - Select the original profile --> click on <Properties> --> <Email Accounts>
#3 - Remove the POP account
#4 - Add the account by entering your email address/password which defaults to configuring the account using IMAP
#5 - Switch to the Data Files tab. You'll see that the PST file is set as the default data file for the profile which is where your default contact/calendar folders are stored. You want to maintain this instead of set the IMAP OST file as the default data file to prevent potential loss of data in folders (contact/calendar etc) that will have the added words "This Computer Only" in the folder description.

If Outlook cannot find the OST file for the account for any reason, it will recreate the OST file and re-sync the email data with the server. The "This Computer Only" folders will be empty and therefore that data will be lost. Also, if you ever remove the IMAP account, the ost file also gets automatically deleted. An OST file can only be opened/used by the original Outlook profile/email account used to create it.

Hello everyone,

Thank you for bringing your concern to our attention and I apologize that it took us more time than usual to reply to your question.

I understand that you're having synching issue with your device as deleted emails on web doesn't appear on your mobile device. I'm glad to assist you on how to remedy this issue. To set your expectation, configuring your account in mobile devices will give you EAS (Exchange Active Sync) connection by default. Meaning to say, it's a two-way that will synchronize your emails, contacts and calendar. Every changes you've made on your account via Outlook.com web should also reflect on your device or vice-versa.

There's instances that we are not aware and we choose a different configuration settings when we are trying to setup our account on our devices. In case you have selected POP connection, you need to ensure that your account is set to Do what the device or app says-if it says to delete messages, then delete them. Refer to the steps below to check your settings.

1. Sign in to Outlook.com.

2. Click the gear icon on the upper right corner beside your name.

3. Select Options.

4. Under Managing your account, click Connect devices and apps with POP.

5. Select Enable.

6. Check Do what my other program says-if it set to delete messages, then delete them.

7. Click Save.

To know more on how to properly configure your account on your preferred mobile device, please refer to this Microsoft article: Access Outlook.com on your Windows Phone, iPhone, iPad, iPod Touch, or Android

Should you have other questions or any other concerns, please reply to this thread and we will be happy to further assist you.
