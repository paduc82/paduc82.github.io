rich michaels - mvp (greatcirclelearning.com) say:

I hope you read this before uninstalling ... I would only do that as a last resort.

Is this happening on all documents or just this one? If it's just happening on one document then it's probably a document corruption or the document name is stuck in memory. I recommend the following:

  1. Shutdown your computer completely by turning the power off.
  2. Wait a minute or two before starting it up again.
  3. When you startup, hold down the shift key as you press the power on button and keep holding it down until the desktop fully appears. (this is a safe start and it clears memory)
  4. Open Finder and make a copy of the problem file under a new name.
  5. Now restart the computer, you don't need to power down or hold the shift key down when starting. Just a regular restart.
  6. Open Word and then open the copied file and test if it works correctly.
  7. if the problem persists, copy everything from the file except the last paragraph mark.
  8. Paste the copied text into a new blank document and save it under a new name. You will have to recreate the headers and footers of the last section of the document.
  9. Test this new document and hopefully it now works correctly.

As a recap, the first 5 steps I listed will clear corrupted memory if it existed concerning this document. The last 4 steps will usually clear up a corrupted document layer, if it was causing a problem.

mike h.. say:

Hi,

To keep the code simple, initially you will have to manually delete to 50 rows or less then use this modified code

Sub Max_Rows()
Open Application.DefaultFilePath & "\logwhenmodified.csv" For Append As #1
Print #1, "Workbook Activity     " & Now
Close #1
Workbooks.Open Application.DefaultFilePath & "\logwhenmodified.csv"
lastrow = Cells(Cells.Rows.Count, "A").End(xlUp).Row
If lastrow > 50 Then
Rows(1).EntireRow.Delete
End If
ActiveWorkbook.Close True
End Sub
