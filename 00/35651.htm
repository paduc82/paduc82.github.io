Hi, 

Thanks for posting your query on Microsoft Community.

This issue might related to permission to share printer.

I suggest you to remove the network printer and try adding it again and check if it helps

Follow the below steps:

Step 1: Make sure that, the printer is shared on the network in the primary computer.

Step 2: Remove the network printer.

            a) Type Devices and printer in the search area on the task bar and select Device and Printer.

            b) Select the printer and click on Remove Device button.

Step 3: Restart your system.

Step 4: Add the network printer again.

            a) Type Devices and printer in the search area on the task bar and select Device and Printer.

            b) Click on Add Printer.

            c) Let Windows scan for the printer and if detected select the printer and click Next. If the     printer is not listed click "The printer that I want isn't listed" and skip to add printer by name or TCP/IP address.

Hope it helps, reply to us with the status of your issue. We will be happy to assist you.

Using HP as an example:

The manufacturer pre-installation has coding included which looks for HP's customized BIOS. If the replacement motherboard has an HP BIOS, the installation will work and will activate and stay activated.

If the replacement motherboard does NOT have an HP BIOS, it will still run and activate but, when challenged by Microsoft's Genuine test, will report that the OS is NOT running on an HP computer, and you will get a 'not genuine' message.

Bear in mind that a pre-installed OEM OS costs way less than a full retail edition, which is partly why OEM has the above limitation..

"So purchasing a used motherboard does not entitle me to a software rights, but upgrading a motherboard removes your rights to use the software?"

Note that you could buy an OEM System builder version of Windows which will install on anything initially, but being OEM, it must stay with the original machine just like the HP OEM has to stay with an HP machine.
