brian tillman [outlook mvp 2007-2019] say:

If you already imported into Outlook the CSV file generated by exporting your WLM contacts and they don't look right, it's probably because you did not map the import properly.  During the import process, you will see, just before the actual import occurs, a button that says "Map custom fields".  Click that.  ON the left will be the column headers for WLM.  In the right will be the fields Outlook expects.  Make sure the proper fields get mapped.  The import should produce correctly formed contacts in Outlook.


Then, make sure the Contacts folder is configured to appear in the Address Book interface.  Right-click the Contacts folder in the Navigation Pane and choose Properties.  Select the Outlook Address Book tab and make sure the box labeled "Show this folder as an e-mail Address Book" is checked.  This should get the contacts into the Address Book view.

karl timmermans [outlook mvp 2012-2018] say:
In reply to deleted message

In addition to Brian's reply - just one other point to add

 

If you imported your contacts to the default contact folder the profile (the one that is in the same tree as your <Inbox> if using a POP email account) then all the contacts should have already been reflected in the AddressBook for whatever contacts were imported (which may have been <none> if you did not go through the Map Custom FIelds process as mentioned by Brian). alex sun msft say: Hi Rich,

After all the mailboxes are migrated to Office 365, it is feasible to uninstall the on-premises Exchange 2003. Since you have directory sync enabled, the source of authority is your on-premises. For mailbox management, you should keep one on-premises Exchange server for AD schema extended and supported management tools like Exchange Management Shell or EAC/EMC. However, since Exchange 2003 is no longer supported. You are expected to uninstall it or upgrade to Exchange 2013 or Exchange 2016. If you don't want an Exchange server, you will have to use other tools for management, which could lead you to unsupported situation.

About your query, uninstalling the last Exchange server will remove the existing Exchange schema from AD. To my knowledge, it is OK to uninstall your Exchange before or after you upgrade your DC(s).
