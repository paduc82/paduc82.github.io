If you use an array formula in columns A of Sheet1 and Sheet2 to gather a unique list of the products from Sheet3, you can use that to lookup the other detail columns by cross-referencing the column header labels providing they match.

macropod say:

You can do this via the use of an IF field. Such a field can conditionally display content that is already in the document, without recourse to an external file, or can be combined with an INCLUDETEXT field to reference an external file. The basic field coding would look like:

{IF{MERGEFIELD ConditionField}= 1 "Text, tables & pictures for condition 1"}

{IF{MERGEFIELD ConditionField}= 2 "Text, tables & pictures for condition 2"}

etc. - all on the same line - where 'ConditionField' is the name of a field in your data source that tells Word which condition, if any, applies to a given letter. To incorporate content from an external document, you might use:

{IF{MERGEFIELD ConditionField}<> "" {INCLUDETEXT "{FILENAME \p}\\..\\ConditionsDocument.docx" "Condition{MERGEFIELD ConditionField}"}}

where the conditions are held in a document named 'ConditionsDocument.docx' in the same folder as the mailmerge main document and each condition has a bookmark named 'Condition1', 'Condition2', etc.

Note: The field brace pairs (i.e. '{ }') for the above examples are all created in the document itself, via Ctrl-F9 (Cmd-F9 on a Mac); you can't simply type them or copy & paste them from this message. Nor is it practical to add them via any of the standard Word dialogues.

jay freedman say:

I don't have a definitive explanation, but I think I know why you see that behavior.

According to the MS-EMF documentation, the EMF specification has gone through 14 major versions between the first one in 2007 and the current version in 2018. Three of those versions involved the addition of new record types. (In a curious omission, the document doesn't say which Office versions were the first to use the second and third variants.)

It's the standard behavior of copy/paste in Windows that the copying process places on the clipboard all the available variants of the copied material, and it's the responsibility of the pasting process to select the variant best supported by its program.

The implication is that Word 2007 handles only the earliest of the Microsoft-supported EMF formats, while Word 2016 can handle any of the three versions. I think this is what's happening:

  • In copying from Word 2007 to either another Word 2007 document or a Word 2016 document, only the earliest format is available on the clipboard.
  • In copying from Word 2016 to another Word 2016 document, only the latest format is used, although the other two are on the clipboard as well.
  • In copying from Word 2016 to Word 2007, the clipboard offers all the versions, but Word 2007 selects only the earliest version to paste.
