Macro for 1PM Fri -> Time and Date Code - Microsoft Community

stevearens say:

Macro for 1PM Fri -> Time and Date Code

Every week I format a table. Right now I go through and change everything manually however I'm working on making a macro to do it for me automatically.

I'm having a little trouble with 1 column.

The data comes in like this:

1PM Fri

7PM Sat

9PM Fri

Noon Fri

Normally I adjust the time and I change the "Fri" to Friday's date. From there it's recognized as a time and date so I can format it back to show 1 PM Fri but I can sort it as a date.

I would like a universal macro so I'm trying to use a formula like "=NOW()-WEEKDAY(NOW(),3) +4" to replace fri but I'm stuck on how I can keep the time and date together while still replacing the Fri with the actual date that Fri would be. Any advice?

bernie deitrick say:

This code assumes that Friday is the earliest day - Wed would be next Wednesday, not last Wednesday.  Further, I have assumed that all your strings are in column A, and all are valid time / day combinations.

            

Sub TestMacro()
    Dim rngC As Range
    Set rngC = Range(Range("A1"), Cells(Rows.Count, "A").End(xlUp))
    Dim c As Range
    Dim i As Integer
    
    Dim strD As String
    Dim dteD As Date
    
    dteD = Date - Application.Weekday(Date, 3) + 4
    
    For Each c In rngC
        c.Replace "PM", ":00PM", xlPart
        c.Replace "AM", ":00AM", xlPart
        c.Replace "Noon", "12:00PM", xlPart
        For i = 0 To 6
            strD = Format(dteD + i, "ddd")
            c.Replace strD, dteD + i, xlPart
        Next i
        c.NumberFormat = "hAM/PM ddd"
        c.Value = DateValue(Split(c.Value, " ")(1)) & " " & TimeValue(Split(c.Value, " ")(0))
    Next c
    
End Sub

How to NOT print hidden rows and then have them stay hidden - Microsoft Community

chubbco say:

How to NOT print hidden rows and then have them stay hidden

I can hide rows (Excel for Mac v16.22), and even print the doc with the rows hidden, but then the rows automatically unhide themselves.

Any way to avoid that? 

hillyman17 say:

I've used this trick in Excel (Windows, but should work with Excel for Mac).

Create an empty column A, and for each row that you don't want printed, type an "n" (for "no" but any unique character will do). Then before printing, do a Filter in column A to show rows that NOT "n". The rows with "n" are hidden by the filter and then you print. Right after you print, clear the filter and the rows reappear again.

Some nice tricks:

1. instead of typing in "n", use a formula that checks if that row has values above or below 0. You might want to test for something greater than 0.001 or less than -0.001 since sometimes Excel hasa funny rounding errors.

Say you are in row 10 and you are working in columns C to K. In column A10, write:

=IF(OR(MAX(C10:K10)>0.001,MIN(C10:K10)<-0.001,"","n")

For those that you definitely always want to hide, just type in "n".  For those rows you always want to print, such as spacer lines, leave blank.

2. You can put a macro to hide and unhide the rows. Use say Ctrl+shift+H as a toggle to hide/unhide the rows.

You may use a formula like this:

=IF(TODAY()>=DATE(2018,3,1),Sheet10!B10,"")
