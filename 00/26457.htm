Hi DBDiva,

 

This type of problem with OLE fields has been an issue for quite some time, which is why some people will recommend not using them. OLE objects, such as .jpg depend upon certain software being on the machine in order for them to render properly. In addition to this, even if you have Photo Editor installed, installing another photo editing program on a machine can easily change the OLE Server for a specific file extension like .jpg, and you will once again see the same behavior.

 

Going forward, I would recommend either using the new Attachment data type field that was introduced in Access 2007 with the .accdb file format or I would recommend not storing your pictures in the database and using VBA code to load the picture from a folder on your machine into an image control.
