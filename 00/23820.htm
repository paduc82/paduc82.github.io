mrwinniethepooh say:

Annoying behaviour with multiple monitors with different scaling (using Windows 10 build 10074)

I am using dual monitors (1920x1200 with scaling set to 100%, and 3840x2160 set to 200%).  The monitors have the same physical height. I don't think that one can expect a totally perfect user experience with such disparate monitors, but Windows 10 actually does pretty well (provided that you do not attempt to have windows crossing the monitor boundary).

However the one behaviour that I have found really annoying is the movement of the mouse across the monitor boundary, with an empty desktop. What is clear is that Windows maintains the pixel offset of the mouse pointer when crossing the monitor boundary. This means that moving the mouse laterally from the bottom of the 1920x1200 monitor takes you to half way up the 3840x2160 monitor. In the other direction, moving laterally from the bottom of the 3820x2160 monitor to the 1920x1200 monitor you run into a 'brick wall', which extends half way up the screen. To move the mouse from the 3820x2160 monitor you have to 'climb over' this brick wall. I find this really annoying - what would be nice on an empty desktop is to be able to move the mouse smoothly between monitors - without any jumps.

I realise that this would imply suddenly changing the mouse pixel co-ordinates at the monitor boundary but (at least on an empty desktop) it would provide a more satisfactory user experience. I also realise that there is a dependence on the physical size of the monitor - if my 4k monitor was twice as high, physically, as my HD monitor then the current behaviour would be fine. I am only experiencing the problem because the monitors are, in fact, of the same physical height.

I guess that this is all rather a big technological challenge, and that the Windows 10 developers have already considered these issues.

.....

(Later update) Having said that Windows 10 handles the dual monitors (1920x1200 and 3840x2160) quite well, I have found that, following a reboot, the UI elements have now become a bewildering mixture of 'large' and 'small' on both monitors.  Everything was fine before the reboot, but somehow the reboot caused problems for the scaling algorithms - and these problems persisted even after a second reboot.  The only way I found of getting back to sensibly sized UI elements was to replace the 4k monitor by a 1920x1200 monitor.     Clearly the support of mixed DPI monitors is still a work in progress.

Hi folks. 

If you're phone won't display or send pictures in text messages, I've found the solution.  First let me say I am no IT expert, far from it, but I've been playing with this for a while now and I finally figured it out.  Ensure that your carrier has MMS set-up for you before you proceed.

1.)  Data HAS to be turned on.  MMS only sends over a data network, not over wifi so if you have Data turned off to save on your usage you'll need to turn it on if you want to receive/send text pictures. 

2.)  If you're a "text hoarder", delete all your threads to free up some space.  I'm not sure if this really helps but it couldn't hurt.

3.)  Search the web for "Configure Nokia Lumia 1020 APN settings for (name of your carrier) in (country)".  Keep this page open, you'll be referring to it as you go.

4.)  On your phone go to Settings>mobile+SIM>SIM settings

5.)  Enter the information you found on the web page you found in step 3 and click save. 

Now it should work.  Apparently with the Windows Phones (any make or model) you have to configure these settings manually in order for the MMS to work, especially if your phone is unlocked.  Not all carriers will upload this information to your phone automatically.  

I should mention, my phone is unlocked and I use Bell in Canada so my search was "Configure Nokia Lumia 1020 for Bell Mobility in Canada".
