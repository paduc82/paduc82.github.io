You should not be storing LastName, FirstName, DOB or anything else regarding specifics about the Patient in the "ccc update" table. You should be storing ONLY the PatientID, and nothing else.

You also should not be storing those "Issues" column in that manner. Instead they should be broken out to a separate table, and related back to the "ccc update" table.

You can still show the Patient information on your form. To do that, include all the columns you want to show in the Combobox, and then write that data out to the form in unbound textboxs when the user chooses a patient, or in the Current event of the form. For example:

Combo Rowsource: SELECT PatientID, LastName, FirstName, DOB FROM Patient

Combo ColumnWidths: 0;1;1;0

txPatientName

txDOB

Me.txPatientName = Me.YourCombo.Columns(1) & ", " & Me.YourCombo.Columns(2)

Me.txDOB = Me.YourCombo.Columns(3)

Do the same in your form's Current event.

But get the table structure correct before doing anything else.

scottgem say:

Sorry to have to tell you but your database is NOT designed well. You have a few issues. 

First, as Scott McD said, you should not be replicating any of the patient info in ccc update. One of the principles of relational databases is that data should exist in ONE place only. Since the other info (besides the foreign key) exist in the patient table they should not exist in the related table. 

I suspect you are using one of the techniques listed in my blog on Displaying Data from Related Tables to populate the controls on your form with the patient data. But if you are, then that would explain why the data is not populating in the table. Again, that is as it should be and you should remove all those fields from ccc updates.

Second, whenever you have numbered fields like Issue1, Issue2, that is called a repeating group and violates normalization rules. You should have another table:

tblContacts

ContactID (PK autonumber)

UpdateID (FK)

Issue

ContactedPerson

TimeSpent

So you would add a record in the above table for each issue, as many or as few as needed.

Your ccc update table becomes:

tblCCCUpdate

UpdateID (PK Autonumber)

PatientID (FK)

CallDate

CallerInitials

Note that I've removed spaces from your object names and shortened them. You can use the Caption property to expand what you want to appear in a form label or column heading. Using spaces and special characters is not recommended. Notice, also, that I don't accept the default ID for your autonumber PKs, but rather name them using the table name. 

Finally, you refer to a query for your form. That is not necessary and I do not recommend using multi-table queries as the Recordsource of forms, especially ones that are being used to enter data. You should have a form bound to tblCCCUpdate. On that form will be a combobox to select the Patient. The relevant properties for that combo should be as follows:

RowSource: SELECT PatientID, Lastname & ", " & Firstname AS Patient, DOB, PatientAffiliation

FROM tblPatient

ORDER BY Lastname, Firstname;

Bound Column: 1

ControlSource: PatientID

Column Count: 4

Column widths: 0";2";0";0"

You would then have 2 textboxes to display DOB and Affiliation with Controlsources of:

=[cboPatient].Column(2)

=[cboPatient].Column(3)

respectively. Note that I have removed the spaces and changes the names of your objects. cboPatient is the name of the Patient combo.

You would then embed a subform bound to tblContacts and linked on UpdateID to enter and display the updare records.
