Try it like this:

 

1. Create a new, blank database.

 

2. Open the new database in Access.

 

3. If you are using Access 2007 or later, click the External Data tab on the ribbon, then in the Import &Link ribbon group, click Access.  (If you're using Access 2003 or earlier, click menu items File -> External Data -> Import.

 

4. In the Get External data dialog, browse to and select your original database.  You want to "Import tables queries, forms, reports, macros, and modules into the current database."

 

5. Click OK.

 

6. You'll see a tabbed dialog of all the objects in the database.  On each page except the Tables page, click the "Select All" button.  If there are tables you want to import, select them on the Tables page -- but not the ones that are giving you problems!

 

7. If you have menus, import/export specs, or nav pane groups you want to import, click the Options button and check the boxes next to those items.

 

8. When you've selected everything you want to import, click the OK button.

 

The import shouild then start, and will let you know when it's done, or if it encountered any errors.

 

Assuming the import goes okay, use this database going forward (possibly renaming the old one and naming the new one to the old name).  Any current-database options that you set for the old database, you'll have to set again for this one.

 

Don't open any queries in design view until you have your tables set up again, as Access will be confused and probably mangle the SQL if it can't find a table the query refers to.

  1. Go to your inbox.
  2. Click the gear icon and select More mail settings.
  3. Under Customizing Outlook, click Rules for sorting new messages.
  4. Check all the rules.

Check your Junk Mail Settings

  1. Go to your inbox

H11 is actually 88.125. By the way, the SUM function is superfluous here, you can simply use =F11*D11*1.5.

If you want this value to be rounded to 88.13, use =ROUND(F11*D11*1.5,2).

If you don't round H11, I11 is actually 1768.375, and hence, J11 is actually 70.735. There is no reason to use matrix multiplication here, the formula =I11*0.04 works just as well.

If you want this value to be rounded to 70.74, use =ROUND(I11*0.04,2).

1768.375 + 70.735 = 1839.11.

If you use ROUND in both H11 and J11,  K11 will result in 1839.12.

About MSDN or TechNet Product Keys

They are genuine Microsoft product keys for retail licenses, but not for Retail sale. They are intended for a particular product channel either the Microsoft Software Developer Network (MSDN) or TechNet for IT Professionals who pay a subscription fee. The main purpose is for evaluation purposes. It is for one person only to use and no one else. Unfortunately, regardless of the licensing terms, persons still abuse the program, either giveaway product keys or resell it on auction sites. 

What probably happened is, you bought an MSDN licensed key, which carries up to 10 activations unlike full packaged retail licenses which only carry 1 activation. The person who sold it to you probably sold it to 10 other persons. Somewhere along the way, one of those persons might have installed it on a second system, activated it, because it went past 10 activation threshold, Microsoft detected it that it was being abused and blocked the key from further use.
