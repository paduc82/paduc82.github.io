bob jones aka: cybertaz say:

My guess is that you are typing the heading, selecting the line (including the ¶) to directly apply bold, then going to the end of the line & pressing return. That causes the bold formatting to be carried down to the next paragraph. This is just one of many challenges one faces when relying on direct formatting. The use of Styles to apply formatting is far more efficient & effective.

When formatting directly you can obviate this by doing either of the following:

  • Press return before formatting the heading, or
  • Turn bold off before pressing return, or
  • After pressing return, turn bold off, or
  • When applying the formatting be careful to select just the text but not the ¶ at the end of the paragraph.

Styles can include a specification for which other Style is applied to the following paragraph when you press return. For example, when using many of Word's built-in Styles [Heading 1-9 as well as others] the Style for following paragraph property is defined as the Normal paragraph style. This Tutorial by MVP Suzanne Barnhill provides a great deal of excellent advice on the use of Styles in Word. Although they use the Windows version for illustration & reference they equally applies to Mac Word as well.

I'd suggest reviewing this first even if you've been using the program for quite some time but have had no format instruction: “Getting started in Microsoft Word”—tips for the absolute beginner

Followed by:  “Display, Use, and Manage Styles in Word”

FWIW, my approach would be slightly different... To do as John suggests would mean having to put a space between each pair of characters, then convert them to Tabs as he suggested. If you're going to go that way you might as well just press Tab instead of a space in the first place.

- Deleting the local databases of the Exchange 2010 (around 300 gigabytes).
- Disabling all the Exchange services on SBS2011.

a. user say:

Hi Pieter,

It is recommended that you don't decommission the Exchange server 2010 for mailbox management. The reasons are as below:
1. You have deployed ADFS so the user accounts with the federated domain cannot be created from the Office 365 (only created from the on-premises AD).
2. For the synced users, all the mail attributes can only be managed from the on-premises Exchange server (although you can manage them using ADSI edit, it is not supported by Microsoft).
3. Currently, there is no Microsoft document about decommission Exchange server 2010 (SBS 2011) in a Hybrid deployment.

Given this situation, here are some suggestions for your reference:
1. Remove the SCP record from AD.
2. Deleting the local database of Exchange server 2010
3. Leave the Exchange server 2010.

Thanks,
Edward
