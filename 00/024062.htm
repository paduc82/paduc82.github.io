Place this event code in Sheet1 worksheet module. . . . . . .

 

Private Sub Worksheet_Change(ByVal Target As Range)
Dim rng1 As Range
Dim rng2 As Range
Set rng1 = Worksheets("Sheet1").Cells(Rows.Count, 1).End(xlUp)
Set rng2 = Worksheets("Sheet2").Cells(Rows.Count, 1).End(xlUp) _
            .Offset(1, 0)
rng1.Copy Destination:=rng2
Application.ScreenUpdating = False
  Call Alphabet_Sort
Application.ScreenUpdating = True
End Sub

 

Place this macro in a General module. . . . . .

 

Sub Alphabet_Sort()
    Sheets("Sheet2").Select
    Range("A:A").Select
    Selection.Sort key1:=Range("A1"), order1:=xlAscending, Header:=xlGuess, _
            OrderCustom:=1, MatchCase:=False, Orientation:=xlTopToBottom, _
            DataOption1:=xlSortNormal
            ActiveSheet.Range("A1").Select
Sheets("Sheet1").Select
End Sub

daniel pineault say:

A database can be either open in Exclusive or Shared mode.  What this message it telling you is that the db has been opened by someone else in Exclusive mode, which mean no one else can use it while they are using it.

This is indicative of a database, which has not been properly deployed; ie: not split.  If the db was split and each user given their own copy of the Front-End, this would not happen.  Also, using a db in your current configuration is dangerous and can lead to database corruption.  Make backup copies!

To learn more about database splitting, see: http://www.hitechcoach.com/index.php?option=com_content&view=article&id=35:split-your-access-database-into-application-anddata&catid=65:split-database
