barb reinhardt say:

SUMPRODUCT WITH ISNUMBER

joeu2004 say: Barb wrote:

I'm sure I've done this before, but I just can't figure it out now.  I have this formula (because I12 has #N/A in it.  )
=SUMPRODUCT(--($B$10:$B$15=$B68),--(ISNUMBER($I$10:$I$15)),($I$10:$I$15),(K$10:K$15))
And it's just not working.  WHAT am I missing?


You can use one of the following array formulas [*]:

=SUMPRODUCT(--($B$10:$B$15=$B68),--IF(ISNUMBER($I$10:$I$15),$I$10:$I$15),K$10:K$15)

or

=SUMPRODUCT(($B$10:$B$15=$B68)*IF(ISNUMBER($I$10:$I$15),$I$10:$I$15),K$10:K$15)

or

=SUM(IF(ISNUMBER($I$10:$I$15),($B$10:$B$15=$B68)*$I$10:$I$15*K$10:K$15))

In the latter case, you might want to add parentheses if you think it makes more readable.  It's a matter of personal preference.

[*] Enter an array formula by pressing ctrl+shift+Enter instead of Enter.  Excel will display an array formula surrounded by curly braces in the Formula Bar, i.e. {=formula}.  You cannot type the curly braces yourself.  If you make a mistake, select the cell, press F2 and edit, then press ctrl+shift+Enter.

Barb wrote:

WHAT am I missing?


SUMPRODUCT does not ignore cells with Excel errors.  The ISNUMBER term creates an array of 1s and 0s, which is multplied by the I10:I15 array.  But the latter contains Excel errors, which SUMPRODUCT sees.

The IF(ISNUMBER) construct ensures that the resulting array has FALSE where Excel errors might have been.  But we must tell Excel that this is an array formula by pressing ctrl+shift+Enter.  Otherwise, Excel does not treat ISNUMBER(I10:I15) as an array when used in the IF() expression.

tammy.m say:

Hello MarvinWapnitsky,

Unfortunately there is not a time zone field written in each appointment but instead when appointments are created in Outlook the appointment time is set and managed based on the global time zone settings in Outlook which is found by clicking on the File menu, then Options, and then click Calendar, scroll down to the time zone settings.

In the time zone settings you will see that the time zone will be set to UTC, which is Coordinated Universal Time (UTC).

The appointment is managed by subtracting the UTC setting from Greenwich Mean Time (GMT). This allows you, as the user, to move through different time zones and as long as you update your system to reflect the time zone you are actively in your appointments will be adjusted.

This equation also applies in the case of Daylight Savings Times (DST). When you have DST enabled then when the USA calendar changes to DST all appointments will be moved to the appropriate time slots and adjusted to accommodate DST. However, if you live in Arizona where they do not observe DST then you would not select the DST option to not adjust the appointments.

The only suggestion I have would be to try and use the appointment location sort on.
