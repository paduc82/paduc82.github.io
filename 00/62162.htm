Add a searchable shortcut in "All apps"

In past versions of windows, a user has been able to manually add shortcuts (NOT TILES) to the start menu by placing them in the All Programs folder either in %appdata% or in %programdata%. On windows 10, with default drive lettering, the paths to these two folders are C:\Users\USER\AppData\Roaming\Microsoft\Windows\Start Menu\Programs or C:\ProgramData\Microsoft\Windows\Start Menu\Programs.

When a shortcut is added to one of these locations, it appears in the "All apps" section of the start menu. On Windows 8/8.1, a manually-added shortcut could be found by typing in the start menu search. On Windows 10, however, manually-added shortcuts can only be found by manually going through All apps -- that is, if one creates a shortcut called ABC.lnk, it will appear at the top of the All apps section but one will not be able to find it by typing ABC in the start menu.

Searching for an installed program (e.g. Firefox) will come up with a the following result: "Firefox [desktop app]", which (by right-clicking and selecting "Open file location") is located in the ProgramData start menu folder.

How can a shortcut be manually added to All apps such that it is searchable?
