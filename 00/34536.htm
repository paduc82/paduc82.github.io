neomoa say:

SOME SOLUTIONS FOR SKYDRIVE SYNCING PROBLEMS IN Windows 8.1

1- Syncing from SkyDrive.com to your SkyDrive folder:

Verify syncing from Skydrive.com to your PC by creating or uploading a little file throug the web interface of Skydrive.com.

This new created file must appear almost instantly in your PCs SkyDrive folder.


If not, it could be your Skydrive Sync Engine Host service not running for some reason explained in this thread (http://answers.microsoft.com/en-us/windows/forum/windows8_1-files/skydrive-does-not-sync-and-clutters-file-system-on/d83306c8-7383-44d5-8d9d-81b0ec4dd849) and you can try this fix by Scott Atk:


a.      Log in as an administrator

b.      Open a command prompt as admin (right click on the Windows icon and select command prompt (admin))

c.       Copy this command  reg add HKEY_CURRENT_USER\Software\Microsoft\Windows\CurrentVersion\Explorer\CLSID\{8E74D236-7F35-4720-B138-1FED0B85EA75}\ShellFolder /v Attributes /t reg_dword /d 0x00100000 /f

d.      Right click and paste the command at the cursor in the command window.

e.      Press Enter

f.       If the command is successful, the message The operation was completed successfully will be displayed




2- Syncing from your SkyDrive folder to SkyDrive.com:

Create or move a new file into your SkyDrive folder and verify that it appears in your SkyDrive.com.


If not, verify that your Windows Search service is running and if not, start it.

Also verify that your SkyDrive folder is included into indexed locations (Control Panel | Indexing Options).

If you have an SSD, this service can be disabled in order to avoid constant writes to your drive for indexing updates and reduce your SSD lifespan.


I suppose that as this service is constantly monitoring writes to your drive in order to maintain the search index, it makes sense that it also delivers this changes to other processes that need this same information like SkyDrive.

Having had to fix the 100% disk usage issue in the past, and having found previous discussions Locked from further posts by users, I thought I'd add something here on the topic.

Basically, about 1+ year ago I had the horrible 100% disk usage issue, resulting after about 5 minutes in a frozen computer.  This was after an upgrade to an SSD.  It turned out that the windows driver that was installed really had problems with the SSD and was causing the issue.  So I fixed it back then by forcing it to change driver (which I did using some RAID = off setting in the BIOS, which happened to force the use of an older driver, though I'm not sure why). 

Anyway, back to now, that same computer had a problem yesterday - a windows update appeared to happen and then all sorts of strange stuff.  It all looked quite similar to last year's problem - with the PC grinding to a halt and freezing after several minutes (with just mouse movement possible).  Oddly, it never showed more than 99% disk usage - so perhaps MS have limited the value that's shown?

So, I checked the SSD driver only to find that MS had kindly updated it (to a 2009 one, but I didn't record which one - sorry).  A quick rollback to a 2006 version (see PNG file) fixed the disk usage / crashing problem.

The Events in the PNG file show how the driver was automatically updated (after appearing to see a new device) yesterday at 8am.  Everything went downhill from there onwards.  Rolling back (to mshdc.inf) was the fix.  This was for a Crucial MX200 500GB drive.

I thought I'd post this info in case anyone else suffers this problem and wonders what to try next - but obviously be careful playing with drivers, as things could go from bad to worse.

Richard

elstaci say:

Seems like a bug in windows 10 concerning DISM. I have the same issue. took me hours and days to figure a round about way to repair corrupt files. 

First: go to windows folder and subfolder LOGS. Open DISM folder and delete the dism log.

second: Run dism /online /cleanup-image /scanhealth. 

Third: go back to windows folder and subfolder LOGS and open the DISM log. Go to the bottom. There it will show you what files are corrupted. copy all the files names if more than one.

Fourth: Unhide hidden system files on C: drive. Go to ESD folder /sources and locate install.wim or boot.wim.  Use 7zip to open the archives. I couldn't open Install.wim but I could open Boot.wim. Inside boot.wim I found the file that was corrupted on my main drive. Copied the file to documents or desktop.

fifth: Went back to my Windows folder. Located the corrupted file. Changed the Permissions on it so that I could replace the corrupted file with the new uncorrupted file from Documents or Desktop. Replaced it.

sixth: Ran SFC /scannow now - came back all good Ran DISM Restorehealth, came back all good. 

It isn't easy but I was able to replace the corrupted file which was .ttf Font file in Seguoi (WinSxS Folder).

Now I get good SFC and DISM repairs. Until Microsoft fixes the DISM problem this is the way I will manually repair my computer in the future when hopefully DISM will be working a
