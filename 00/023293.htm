Reasons why a phone cannot update:

1. It is a country variant - country variants often get the updates later, or have updates skipped altogether. A Turkish country variant product code for example, skipped v40 and updated from v31 straight to v50.

The way tables are aligned was changed between Word 2010 and Word 2013 (the latter introduced a new layout engine), so, as Andrew says, the file format is the crucial factor. scottgem say:
Hi Lisa,

There is no need to move the employee's record. What you want to do is filter out the terminated employees.

The way my employee database works, is I have an unbound option group with choices of All, Active, Terminated. In the After Update event of that Option group. I use the following code:

Select Case Me.OptView
Case 1
Me.Filter = ""
Case 2
Me.Filter = "Status = 'Active'"
Case 3
Me.Filter = "Status = 'Terminated'"
End Select
DoCmd.RunCommand acCmdApplyFilterSort

This way you can easily change which sets of employees the user can see, You don't need another form or another query.

dhookom say: Many of us old-timers simply add a field to the employee table for TerminationDate. When you want to query active employees you can simply filter where TerminationDate is Null. You could also use a status field to track active, on leave, terminated, or other.

This suggests you have one less table to maintain and possibly less forms, code, queries, etc.
