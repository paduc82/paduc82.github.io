a. user say:

I posted something a while ago, but deleted it since there was no response.

Anyway, Solver can't do Global optimization.  

Evolutionary can work harder, but at a cost of much longer run times.

If you wanted to find the maximum of    x*Sin(x)

Solver will return a local maximum depending on the start value.

It will not return a Global maximum. 

Many models that don't seem to work can be made to work if the model is re-written properly.

Also, Optimization software have the concept of Integers.  A cell on a worksheet is best described as "double precision", so that introduces a little more complexity.

In general, Solver is very useful.... as a free add-inn.  Especially since it can be run via VBA.

One can purchase higher performance Solvers if really needed.

> ... I'm surprised Solver can't generate a graph like this itself (unless I've missed something.)

Your graph is a simple 1-input, 1-output graph.

Solver can handle 200 changing cells, and a graph of more than 1 input would be impractical.

That's where VBA comes in handy when making these graphs.

The drop-off in profit just above 16k makes me wonder if the model was written correctly.

Using Solver correctly is more of an art than a science.
