Rick showed how to ignore the error, but I would drop that err.number check.

on error resume next
Worksheets("Sheet1").Shapes("Flag1").delete
on error goto 0

There are sometimes when you really want to know if there's a shape with that
name -- not just for deleting.

dim myShp as shape
set myshp = nothing
on error resume next
set myshp = worksheets("Sheet1").shapes("Flag1")
on error goto 0

if myshp is nothing then
  'it's not there
else
  'it is there
end if

=======

worksheets, workbooks, names, ...

Group by day of the week - Microsoft Community

davidkinne say:

Group by day of the week

I am building an Access 2007 database to keep track of volunteer training at our organization.
I am trying to group by day of the week in a report so it sorts Monday - Sunday rather than sorting the days alphabetically. Unfortunately, my day field [Shift Day] includes multiple values because some folks volunteer on multiple days of the week. Any suggestions? Thanks!

john w. vinson say:

Please post the structure of your table (fieldnames, datatypes and sample data). If you have multiple days jammed into one field you'll have difficulty! A basic principle of relational design is one field:one fact; storing multiple chunks of data into a single field is simply incorrect (for the very reason you're seeing). A better design might be two tables in a one to many relationship - a table of Volunteers, with VolunteerID as its primary key and fields for lastname, firstname, and contact information; and a second table ShiftDays with fields ShiftDayID (autonumber primary key), VolunteerID (link to Volunteers, who's volunteering for this shift) and ShiftDate (date/time, containing the date that they're volunteering). If a person comes in three days in a week she'd have three records for that week in this second table, and your report will be much easier.

ken sheridan say: You really do need to change the design so that you are storing the dates of each shift per volunteer as a single value of date/time data type in a single column in a related table as John describes.  At present your table is not normalized to First Normal form, whose definition is as follows:

First Normal Form: A relvar is in 1NF if and only if, in every legal value of that relvar, every tuple contains exactly one value for each attribute.

Loosely speaking, in the language of the relational model, a relvar (relation variable) equates to a table, a tuple to a row (record) and an attribute to a column (field).

Having normalized the tables, you can then use the following function to return the Monday start date for each date:

Public Function WeekStart(intStartDay As Integer, Optional varDate As Variant)

    ' Returns 'week starting' date for any date
    
    ' Arguments:
    ' 1. intStartDay - weekday on which week starts, 1-7 (Sun - Sat)
    ' 2. vardate - optional date value for which week starting
    '   date to be returned.  Defaults to current date
    
    If IsMissing(varDate) Then varDate = VBA.Date
    
    If Not IsNull(varDate) Then
        WeekStart = DateValue(varDate) - Weekday(varDate, intStartDay) + 1
    End If
    
End Function

So you can first order by WeekStart(2,ShiftDate) and then simply by ShiftDate to order the rows by the day of the week within each week.
