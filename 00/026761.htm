Function ExportToText(strQuery As String, _
                    strExportTo As String, _
                    strDelim As String, _
                    blnQuoteText As Boolean, _
                    Optional blnHasFieldNames As Boolean = True)
                    
'Accepts:
' strQuery - Text - name of query to be exported
' strExportTo - Text- path to file to export to
' strDelim - Text - delimiter character(s) to separate fields
' blnQuoteText - Boolean - True to enclose text or memo data in quotes
' blnHasFieldNames - Boolean - (optional) - True (default) to export field names as first line

   Dim dbs As DAO.Database
   Dim rst As DAO.Recordset
   Dim qdf As DAO.QueryDef
   Dim prm As DAO.Parameter
   Dim fld As DAO.Field
   Dim n As Integer
   Dim strPrintList As String
   Dim strQuote As String
   
   Set dbs = CurrentDb
   Set qdf = dbs.QueryDefs(strQuery)
   
   ' evaluate query's parameters, if any
   For Each prm In qdf.Parameters
       prm = Eval(prm.Name)
   Next prm
    
   Set rst = qdf.OpenRecordset
   
   If Dir(strExportTo) <> "" Then
       If MsgBox("Overwrite " & strExportTo & "?", vbQuestion + vbYesNo, "Export Query") = vbYes Then
           Kill strExportTo
       ElseIf MsgBox("Append rows to " & strExportTo & "?", vbQuestion + vbYesNo, "Export Query") = vbNo Then
           Exit Function
       End If
   End If
           
   With rst
       If Not (.BOF And .EOF) Then
           Open strExportTo For Append As #1
           
           If blnHasFieldNames Then
                ' include column headings in text file
                For n = 0 To qdf.Fields.Count - 1
                     strPrintList = strPrintList & strDelim & qdf.Fields(n).Name
                Next n
                ' remove leading delimiter
                strPrintList = Mid$(strPrintList, Len(strDelim) + 1)
                Print #1, strPrintList
                strPrintList = ""
           End If
           
           Do While Not .EOF
               For n = 0 To qdf.Fields.Count - 1
                   Set fld = .Fields(n)
                   strQuote = IIf(blnQuoteText And (fld.Type = dbText Or fld.Type = dbMemo), """", "")
                   strPrintList = strPrintList & strDelim & strQuote & _
                   .Fields(n) & strQuote
               Next n
               ' remove leading delimiter
               strPrintList = Mid$(strPrintList, Len(strDelim) + 1)
               Print #1, strPrintList
               strPrintList = ""
               .MoveNext
           Loop
           Close #1
       End If
   End With

Exit_Here:
   rst.Close
           
End Function

As you see this allows you to choose whether to wrap text values in quote characters or not, and evaluates the query's parameter(s) when establishing the recordset.  To include the value of the parameter in the file name, you'd just need to reference it and concatenate its value into the string expression passed into the function as the strExportTo argument.

If you merely put a .dotm in a Sharepoint library, the behaviour that you see is expected. AFAIK it doesn't have anything to do with the file associations on the user's computer, although if they are also wrong, you may face additional difficulties. But here, if I do what you do, Word opens the .dotm and runs AutoOpen if there is one.

What you are supposed to do in Sharepoint is to 

 a. create a new "content type" 

 b. associate the template with that content type

 c. associate the content type with the document library

 at which point, the template becomes available as a "new" option in the document library (how you access that depends on the version of Sharepoint you are using.

When you use that New option/button to create a document, a new document should be created and Word should run any AutoNew macros. It works here, anyway.

However, you will need to have certain permissions to create a content type. I don't know whether you have them or not, but the chances are that you need to contact your Site Administrator and get them to do this, and/or perhaps show you through the  steps. Frankly, if you haven't done administrative stuff on a SharePoint server, the process can be a bit confusing.  

I found /a/ link that describes what you are supposed to do at

http://www.documentmanagementworkflowinfo.com/sharepoint-document-management/add-document-template-sharepoint-document-library-site-content-type.htm

but you may be able to find a better one.
