suzanne s. barnhill say:

Use a paragraph style with exact line spacing. To set this, right-click on the style in the Quick Styles Gallery or Styles pane and choose Modify... In the Modify Style dialog, click Format, then Paragraph... In the Paragraph dialog, under "Line spacing," choose Exactly from the dropdown. You can use the spinner to change the line spacing in increments of 1 point, or you can type in a measurement in points and tenths (such as 9.4). Some trial and error may be required to get a distance that works well for both English and Chinese.

If you need this same line spacing for all styles, the best alternative is to change it in the Document Defaults. In the Styles pane, in the bottom left corner, click Manage Styles. In the Manage Styles dialog, select the Set Defaults tab and set the line spacing there.

bob jones aka: cybertaz say:

I'm afraid you can't have it both ways :-)

Dates in Excel are stored as sequential numbers, but the program has been designed to automatically recognize & format them based on the manner in which they're entered. See the introductory info in this Excel Help article for more detail: Date calculations in Excel

However, Excel only recognizes & formats Dates, Percentages, Currency, etc. when the cell is formatted as General. IOW, by pre-formatting all cells as Number you're implicitly telling the program to not format Date entries as anything other than their sequential numerical value.

Rather than altering the formatting, it may serve you better to use Excel> Preferences> Edit to set the preference for Display this number of decimal places. Be sure to read the Description @ the bottom of the preferences pane when you point to the preference, itself, in order to better understand how doing so affects data entry.

Another option may be to set up a workbook laid out as you typically need with the various ranges of cells formatted for the type of data intended to be entered. The workbook can be saved as a template for generating new workbook files or can be set as a default workbook & default worksheet sheet. There isn't enough info about your needs & workflow to tell which/whether these options may be useful.

Term store group access denied. Failed to create term set: Access - Microsoft Community

lisajo at work say:

Term store group access denied. Failed to create term set: Access is denied. (Exception from HRESULT: 0x80070005 (E_ACCESSDENIED))

I have set up a site collection with metadata navigation which created a collection term set. I added users to the 'group' access which I understand should mean that those users can add/remove/edit the collection term set and its terms. BUT, when I test it, as one of those users, I receive this error message when I open navigation and attempt to create a new term set in the collection (via the button provided) and, when I click on the Open the Term Management Tool link, it opens it to the term store level, does not even show a reference to the site collection, and keeps 'spinning' as if its trying to open/add the collection term set but cannot.

These users do not have Full Control. This is my goal: to give a set of users the ability to manage site content and the term set for the site collection without giving them full administration over all the design, permissioning, admin functions that FC give. I've given them Edit rights along with Group Management rights to the term set for the site collection.

Why don't theses users have the ability to access and alter the site collection's term set?

lisajo at work say:

I found the issue.

The group managers must have some form of access to the top site in the site collection. I my case, I was needing to deploy a subsite before the top site design had been completed. Therefore, I was giving access to users on the subset level and not for the top site. To remedy this, I created a perm group for the group managers with very minimal access to the top site. This seemed to do the trick.
