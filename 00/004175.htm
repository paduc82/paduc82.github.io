zorvek say:

Debugging UDFs can be a little trickier than working with regular code. Here are some tips I use.

- Create a test harness to unit/regression test your UDF. A UDF is a function and can be called from VBA just like it can be called from an Excel formula. When working in this mode, the VBE will not abort the function if an error occurs, it will show the appropriate debug dialog and you can inspect the code in it's running state.

- If you are unsure whether the code you are working on is actually being called, put a breakpoint in the code such as the first executable line. If the UDF is being called, the VBE will stop execution at the breakpoint.

- Put Debug.Print lines in your code. When these lines are executed, the output it written to the Immediate window.

As Suzanne points out, only Word can create a fully functional normal template. Saving another template with that name within Windows will cause problems. Yes, you can edit a normal template created by Word but I prefer to store most of my customizations in other templates.

The thing is:

  • all sorts of things periodically seem to go wrong with the normal template, whether from Microsoft updates or other problems. You risk losing all of your work, and custom ribbons are not easy, at least for me.
  • You should not be sharing or transferring your normal template to others. Putting your customizations in your normal template means you can't easily share them with others.
  • When a new version of Word is released, it will normally have its own normal template. If you use your old one, you will be running all of your new documents in Compatibility Mode and giving up any new features. My customizations almost all work in Word 2007-2019. That would not be true without major penalty if they were stored in my normal template.

If you store my continuous page numbering template into your Startup Folder before you start Word, its customizations will be available and your normal template is left alone. That was what I recommended in my earlier post.

I, too, have many macros in my Add-Ins. For the ones I edit frequently, I have them pinned to my recent documents list. For others, I have the Startup folder pinned. I can easily open them when I want to do any editing.

Here is a glimpse of my Project Explorer. I have lots of stuff in my normal template, but the code I use on a daily basis is almost all in other templates loaded as Add-Ins. Virtually all of the macros in my normal template are scoped as private and are there either as samples or something I can run from the vbe. I do keep regular backups of my normal template.

To revert to an earlier released version of Office 2013 or Office 2016 as an Office 365 installation, you must use a command line and specify the build number you would like to revert to.

  1. As the Office 365 installations are automatically updated in the background, you must disable the automatic updates first. This can be done in any Office 365 application via:
    File-> Office Account-> Update Options-> Disable Updates
  2. Open an elevated command prompt:
    1. Start-> type: cmd
    2. Right click on: Command Prompt
    3. Choose: Run as administrator
    4. Provide your administrator credentials or confirm the User Account Control dialog when prompted.
  3. Type the command below that applies to your version of Office and/or Windows
    • Office 2016
      cd %programfiles%\Common Files\Microsoft Shared\ClickToRun\
    • Office 2013 on Windows 32-bit
      cd %programfiles%\Microsoft Office 15\ClientX86\
    • Office 2013 on Windows 64-bit
      cd %programfiles%\Microsoft Office 15\ClientX64\
  4. You can specify the built number to return to in the following way:
    officec2rclient.exe /update user updatetoversion=<build number>
    • Replace <build number> with the build number that you want to return to. There is an overview of build numbers you can return to for Office 2013 and Office 2016.
    • Example to revert to the January release of Office 2016:
      OfficeC2RClient.exe /update user updatetoversion=16.0.6366.2062
    • Example to revert to the January release of Office 2013:
    • OfficeC2RClient.exe /update user updatetoversion=15.0.4787.1002

    jari hiltunen say:

    I think I found solution (at least worked for 10 test runs):

    1. Save files to Ubuntu fileshare, disable indexing

    2. From Windows Defender add Exclusions and disable:

    - folder you are saving

    - file type xlsm

    - process excel.exe
