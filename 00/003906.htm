a. user say:

The following VBA code is the best I can do right now. I suggest you
 a. make a copy of your best test document (probably the one with the largest variety of EQ field sizes and so on)
 b. open the copy
 c. run the macro
 d. save as HTML
 e. open in your browser.

See if the result is anything like what you need.

The copied pictures are slightly different from the originals, and the alignment is slightly out. It's not obvious how to fix that, because there may be several factors involved (such as line height/paragraph spacing.

If you need instructions for installing macros, please see Graham Mayor's page at

http://www.gmayor.com/installing_macro.htm

Sub EQs2Pics()

Dim bShowFieldCodes As Boolean
Dim fld As Word.Field
Dim fldWidth As Double
Dim picWidth As Double
Dim picHeight As Double
Dim rngStart As Word.Range
Dim rngEnd As Word.Range
Dim shpHeight As Double
bShowFieldCodes = ActiveWindow.View.ShowFieldCodes
ActiveWindow.View.ShowFieldCodes = False
'Set fld = ActiveDocument.Fields(1)
For Each fld In ActiveDocument.Fields
  If fld.Type = wdFieldFormula Then
    Set rngEnd = fld.Result
    Set rngStart = fld.Result
    rngStart.Move wdCharacter, -1
    fldWidth = _
      rngEnd.Information(wdHorizontalPositionRelativeToPage) - _
      rngStart.Information(wdHorizontalPositionRelativeToPage)
    fld.Copy
    rngStart.PasteSpecial Link:=False, DataType:=wdPasteEnhancedMetafile, _
        Placement:=wdInLine, DisplayAsIcon:=False
    rngStart.SetRange rngStart.Start - 1, rngStart.Start
    With rngStart.InlineShapes(1)
      With .PictureFormat.Crop
        .PictureHeight = 1.01 * .PictureHeight
        shpHeight = .ShapeHeight
        .ShapeWidth = fldWidth
      End With
    End With
    ' The vertical position is slightly out, but
    ' not clear how to improve the algorithm.
    rngStart.Font.Position = -(shpHeight / 2)
    Set rngStart = Nothing
    Set rngEnd = Nothing
  End If
Next
'Set fld = Nothing
ActiveWindow.View.ShowFieldCodes = bShowFieldCodes
End Sub

bernie deitrick say:

SUMIF does not allow 3D references, but you can create one in VBA. Copy the code below into a standard codemodule in your workbook, and use it like this to replicate the formula you want:

=SUMIF3D("0002","0156","G5","=0","G4")

 

'Put this into a standard codemodule
Function SUMIF3D(strS1 As String, strS2 As String, strR1 As String, strC As String, strR2 As String) As Double
    Dim i As Integer
    Dim j As Integer
    Dim k As Integer

    i = Worksheets(strS1).Index
    j = Worksheets(strS2).Index

    For k = Application.Min(i, j) To Application.Max(i, j)
        If Application.Evaluate(Worksheets(k).Range(strR1).Value & strC) Then
            SUMIF3D = SUMIF3D + Worksheets(k).Range(strR2).Value
        End If
    Next k
End Function

jrafael83- [o365] say:

Hi there, I have good news, and I want to share it with you. I finally found the error that was preventing the Ribbon to load, and here is the reason:

My team developed a beautiful masterpage, but the Customer wanted also a FavIcon to show in all the browsers, we included the approach to add the following line:

<!--MS:<SharePoint:SPShortcutIcon runat="server" IconUrl="/SiteAssets/Intelec/favicon.ico"/>-->

Inside the PlaceHolderAdditionalHead content placeholder...

We also had to add the following <link href="/SiteAssets/Intelec/favicon.ico" rel="shortcut icon" type="image/x-icon" /> inside the main <head> tag in order to be compatible with Chrome navigator... It was perfect, the FavIcon was shown on Mozilla, IE and Chrome, but, we did not saw the error until the page needed a change in the content.

We had to rebuilt the masterpage to realize that the lines that caused the trouble was the lines where we add the FavIcon... the problema was, the Customer did not want his page without the FavIcon... the solution:

Don't use neither the <!--MS:<SharePoint:SPShortcutIcon runat="server" IconUrl="/SiteAssets/Intelec/favicon.ico"/>--> or  <link href="/SiteAssets/Intelec/favicon.ico" rel="shortcut icon" type="image/x-icon" /> lines on your code in a publishing site... use the following line inside your HTML <head> tag:

<link href="/SiteAssets/Intelec/favicon.ico" rel="icon" type="image/x-icon" />


and it will work with your custom masterpage, and will not prevent the SharePoint Ribbon to load... I think it would be great to have a list of what you should and what should not do when you customize a SharePoint Online masterpage.

=DATE(MID(A1,8,4),IF(LEFT(A1,3)="Jan",1,IF(LEFT(A1,3)="Feb",2,IF(LEFT(A1,3)="Oct",10))),MID(A1,5,1))
