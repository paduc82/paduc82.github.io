jeeped say:
The SUMPRODUCT() function can be used to SUMIF() or COUNTIF() depending upon the parameters fed into it. You can multiply each parameter as I did in my example or you can separate them by commas (e.g. semi-colons in some non-english languages, see the Control Panel's Region and Language settings for your own).

If you prefer to separate by commas (aka semi-colons) then any TRUE/FALSE parameter needs to be mathematically operated (for lack of a better word) in order to definitively become a 1 or 0 (e.g. one or zero) for internal multiplication and addition. The double hyphen (e.g. double minus) effectively does this. It is like saying (0-(0-<parameter>)).

If you use the parameter multiplication method as I did in my own sample formula, this is not necessary as the multiplication performs the TRUE/FALSE equals 1/0 on-the-fly.

Using the comma (aka semi-colon) based syntax, my own formula would have been,
=SUMPRODUCT(--($B$2:$B$9999>=$D2),--($B$2:$B$9999<$D3),($A$2:$A$9999))
... instead of,
=SUMPRODUCT(($B$2:$B$9999>=$D2)*($B$2:$B$9999<$D3)*($A$2:$A$9999))
Note that only the TRUE/FALSE parameters (e.g. criteria) get the double minus.

Is this making any sense?

You can use a sumproduct formula where dates are in C2:C10 and values are in B2:B10 and D1 is the month number to sum over:

 

=SUMPRODUCT(--(MONTH(C2:C10)=D1);B2:B10)

Option Explicit

Sub DeleteInMergedCell()
  Const Title = "DeleteInMergedCell"
  Dim Count As Variant
  'Delete columns or rows in the selected merged cell

  'Be sure a merged cell is selected
  If Not ActiveCell.MergeCells Then
    MsgBox "Select a merged cell an try again", vbInformation, Title
    Exit Sub
  End If

  'Okay, we have one, delete rows or columns?
  Select Case MsgBox("Delete columns (Yes) or rows (No) or cancel?", _
        vbYesNoCancel + vbDefaultButton3 + vbQuestion, Title)
    Case vbYes
      With ActiveCell.MergeArea
        Count = Application.InputBox("How many columns (1.." & _
          .Columns.Count & ") ?", Title, 1, Type:=1)
        If Count <= 0 Or Count > .Columns.Count Then Exit Sub
        If Count = .Columns.Count Then
          'Delete the merged cell
          .Delete xlShiftToLeft
        Else
          'Delete some empty columns
          .Resize(, Count).Offset(, 1).Delete xlShiftToLeft
        End If
      End With
    Case vbNo
      With ActiveCell.MergeArea
        Count = Application.InputBox("How many rows (1.." & _
          .Rows.Count & ") ?", Title, 1, Type:=1)
        If Count <= 0 Or Count > .Rows.Count Then Exit Sub
        If Count = .Rows.Count Then
          'Delete the merged cell
          .Delete xlShiftUp
        Else
          'Delete some empty rows
          .Resize(Count).Offset(1).Delete xlShiftUp
        End If
      End With
  End Select
End Sub

rena yu msft say:

Hi Charlotte,

Refer to Users don't receive SharePoint Online alert notifications, the users need at least “Read” permissions to do that.

I assume you may only assign them the “View Only” permission.

Therefore, if you assign users (including external users) at least "read" permission, they will be able to set alert to themselves.

Please notice that people with "read" permission will be able to download files while they can't with only "view only" permission.

As the alert feature is in Classic experience, the steps below are all for Classic experience.

  • To check the permission, you can select the folder > click the “…” collapse button > share > shared with > advanced > check permissions > insert the related user account information > check whether the permission is at least “Read” but not “view only”.

  • To change the permission, you can select the related user account > edit user permissions > change to at least “read” permission then save the changes.

  • If you want to share this folder with other people with “read” permission, you need to go to this advanced permission management page and click “grant permissions” > Invite people > click “show options” button and drop down permission levels under “Select a permission level” option window.

Because those permission levels are now showing if you share with people not in the advanced permission management page, see below, if you share with people directly in document library view, you will only see “can edit” or “can view” permissions options. They are “edit” and “View only” permission level.
