john - project say:

Madison,

Let me add a couple more thoughts about your process. I think you are trying to import too much data and that is causing conflicts. Remember that Project is a scheduling application and a basic task needs only a few data points. It needs a start date, unless driven by a predecessor. It needs an estimated duration. It needs a resource name, if a performance task (i.e. non-summary line). If the task has started and/or finished, it also needs an actual start and actual finish.

From that limited set of data Project automatically calculates a finish date and percent complete. If you attempt to also import those two data points there will very likely be a conflict and you won't get the results you expect, which seems to be the issue you are facing.

You also do not want to import summary line data. Again Project calculates summary line data based on the performance (subtasks) below the summary.

Hope this helps.

John
