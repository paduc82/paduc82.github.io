The Mixers table should include a column for the form name relevant to the mixer.  You can then set up the combo box as follows:

Name:    cboMixer

RowSource:    SELECT MixerID, FormName, Mixer FROM Mixers ORDER BY Mixer;

BoundColumn:    1
ColumnCount:     3
ColumnWidths:    0cm;0cm;8cm

The code in its AfterUpdate event procedure would be:

Dim strForm As String

strForm = Me.cboMixer.Column(1)
DoCmd.OpenForm strForm, WindowMode:=acDialog

Opening the form in dialogue mode means it has to be closed (or at least hidden) before the user can return to the calling form.  Code execution is paused when a form is opened in dialogue form, so any subsequent code in the calling procedure can, if necessary, process data entered by the user into the dialogue form.

As regards the design of the mixer forms, if you follow Scott's model, because the table for each mixer models a sub-type of the mixers (super) type, which has columns for the attributes common to all mixers,  you can base each form on a query which joins the two tables, rather than using a subform.  Entering data into a type hierarchy is one of the few instances where, because the relationship type is one-to-one, data can be entered into a single form rather than a form/subform.  In your case the forms would then include a subform for the phase data.  You'll find examples in TypeHierarchy.zip in my public databases folder at:

https://onedrive.live.com/?cid=44CC60D7FEA42912&id=44CC60D7FEA42912!169

Note that if you are using an earlier version of Access you might find that the colour of some form objects such as buttons shows incorrectly and you will need to  amend the form design accordingly.

This little demo file uses a simple model of personnel in an academic institution to illustrate how a type hierarchy is modelled.  In this you'll note how the students form includes a subform for the courses taken by the student, analogous to your phase data.

hubert hdf say:

Hi Yulianto,

Please try the following steps and check if this resolves the issue:

  1. Run the License Removal Tool
  2. In Spotlight Search (the magnifying glass) on your Mac, search for and open Keychain Access.
  3. In Keychain Access, delete the following files:
    • Microsoft Office Identities Cache 3
    • Microsoft Office Identities Settings 3
    • Microsoft Office Ticket Cache
  4. Restart your Mac and open any Microsoft Office app to reactivate.

Note: You will need local admin privileges to make changes to your Keychain Access.

Thanks,

Neha

a. user say:

You can disable it:

If you have Windows 10 Home do this:

Right-click start, run, type regedit and press enter

Navigate to HKEY_LOCAL_MACHINE\SOFTWARE\Policies\Microsoft\Windows\System

RightClick > new > DWORD (32bit) value

rename New Value #1 to DisableAcrylicBackgroundOnLogon

double-click DisableAcrylicBackgroundOnLogon, change value from 0 to 1

Restart your computer

For Windows 10 Pro do this:

Disable acrylic on the sign-in screen. This can be found in Group Policy Editor under Administrative Templates > System > Logon > Enable “Show clear logon background”.
