mike h.. say:

Hi,

View|Toolbars|Control toolbar.

Put s checkbox on the worksheet called 'Bookings". Right click it view code and paste this code in

 

Private Sub CheckBox1_Click()
Swap_Sheets
End Sub

 

Now right click 'ThisWorkbook' view code and insert module and paste this in

Sub Swap_Sheets()
Dim dTime As Date
dTime = Now + TimeValue("00:00:10")
Application.OnTime dTime, "Swap_Sheets"
If ActiveSheet.Name = "Sales" Then
    Sheets("Bookings").Activate
Else
    Sheets("Sales").Activate
End If
If Sheets("Bookings").CheckBox1.Value = False Then
Application.OnTime dTime, "Swap_Sheets", , False
End If
End Sub

Close VB editor and exit design mode on the checkbox toolbar. Now if you check the checbox the code start, clear the checkbox and it stops.

Hi Agtia,

Did you verify your send\receive connectors?

technet.microsoft.com/.../dn751020(v=exchg.150).aspx

When you have issues with your send\receive connectors it's most likely due to TLS configuration. If you are not very familiar with it, try temporarily disabling TLS to get the most basic understanding. If it works with TLS disabled, you need to troubleshoot TLS (certificates, fqdn, etc.)

In case you found your connectors valid, open your Exchange management and look for your users "Remote Mailbox" objects. Are there any? If so, Do they have a "Remote Routing Address" assigned with the @onmicrosft.com domain? does this address also show in the secondary email addresses?

It is important to make sure that these contditions apply since Exchange uses them to route mail traffic.
