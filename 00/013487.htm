julie sheets say:
I'm not sure I understand your requirement.  Are you trying to calculate the number of days from the start date of the project to the current date?  If so, enter the following formula in a custom duration field:

ProjDateDiff([Project Start],[Current Date],"standard")

If you want to calculate the difference between a task start date and the current date use:

ProjDateDiff([Current Date],[Start],"standard")

If you want to calculate the difference between the project start date and a task's start date use:

ProjDateDiff([Project Start],[Start],"standard")

I am assuming you are using the Standard calendar as the project calendar.  If you are using a different project calendar, replace "standard" with "your calendar name".

rod gill say:

Another way is to zoom the Gantt Chart until you have days in the lower tier then double-click the timescale and change the label to Day1, Day2 (from start).

Zoom in or out afterwards to restore the usual timescale.

a. user say:

Okay then, to put a finer point on my earlier reply:

Installing software outside the system partition has never been supported, even though people try it with varying degrees of success. It's a matter of how deeply the software is tied to the system partition.

Simple programs that don't interact with the system partition at all can be installed in any partition. For other programs that do interact with the system partition, their connection to the system partition can be interrupted when the system partition changes, such as when Windows is upgraded.

If you're installing in another partition because you're running out of space on your system partition, and you've already tried all the tricks to free up space on the system partition, it's time for a larger hard disk.
