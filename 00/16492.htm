Let's do some math:

BP-4L supplies 1500 mAH at 3.7V...

.15W = 3.7V x Amps
Amps = .15W / 3.7V = .0405 Amperes.

1.5 AH / .0405 Amperes = 37.03 Hours

If you are indeed pulling .15W, the battery life should be about 1.5 days.


OK let's look at the heavy use side:

.7W = 3.7V x Amps ... So current pulled is .1892 Amperes.

1.5 AH / .1892 Amperes = 7.9 Hours ...

Hence your "it lasts about 6 hours" complaint.

Conclusion: you're really heavily using your E90. Try turning off the WLAN when you're not using it. Or exiting web-based apps (like Mail--which constantly uses the "radios" to poll for mail); even when the application is not in the foreground. Also, some websites (especially those that use javascript) also use RF or CPU when they are in the background.

.7W sounds high. I consider myself a heavy E90 user--I check mail all the time, surf the internet all the time, and read books anytime I have to stand in line for longer than 5 minutes... and I average about .15W in *heavy* use... Typically, I average less than .10W and get about 72 hours of battery life.

So if indeed you are using .7W, it probably indicates that you need to learn some "energy conscious" habits. But if you use an E90 this hard and it lasts 6 hours, I hazard to guess that Windows Mobile cellphones under the same habits will last only 2-3 hours.

in terms of an "executive summary" regarding "accuracy" - the answer is unfortunately: Yes - based on description as provided.

Is everything totally lost? All depends on what files you stored in your cloud backup <but> the not so good news is that if you have the required file by any chance, it's going to require a paid 3rd party OST2PST conversion utility.

With all that said, some explanatory background information which even if it doesn't help your existing situation, will help mitigate it from occurring again in the future.

How things work with IMAP accounts.................(for simplicity, this assumes that all your accounts were configured as IMAP both before and after the incident).

The first IMAP account configured in Outlook will create an OST data file with contact/calendar folders that have the words "This Computer Only" in the folder description. Those words mean exactly that, the contact/calendar data is stored locally only. The data file associated with the first email account configured will be set as the "default data file" which is where the default contact/calendar folders are stored.

Each subsequent IMAP account will also have an .ost file associated with it but those accounts will not have their own contact/calendar folders, instead using the "default" folders created by the first account configured.

When an IMAP account gets removed from an Outlook profile, the OST file associated with that account is also summarily deleted from the system meaning that any data in the "This Computer Only" folders will be lost. <IF>, by chance, you have a copy of the OST file that originally contained your contact/calendar folders, it can only be opened/read by the "original" Outlook profile/email account that created the ost file. Hence, this is where the OST2PST conversion utility comes into play.

The way to mitigate this from happening in the future would be to do the following:

#1 - Add a new PST file to the Outlook profile
(from Home ribbon --> <New items> --> <More items> --> <Outlook Data File)
#2 - go to File --> Account Settings --> Data Files tab --> select the newly created/added PST file and set it as the default for the profile
(the pst file will now store the "default" contact/calendar folders and this file doesn't get deleted when email accounts are removed from the profile
#3 - Create and maintain your contacts in the contact folder in the PST folder tree

If you want/need to share your contact/calendar data with other devices (mobile/tablet etc), then an alternative to the above is to
#4 - create a free Outlook.com account and add it to the Outlook profile
#5 - set the OST file for the Outlook.com account as the default data file (contact/calendar folders are sync'd with the server for Outlook.com/Exchange accounts)
