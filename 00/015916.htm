craigskinner say:

Breaking down a cell value into different cells

Hi,

Basically, what i'm trying to do is to have a certain number of cells that show the the value of another cell divided by the amount of cells that are active.

For example, lets say the main figure is 80, and I have four cells below that I want to show 80/4, thus 20, but what i also want to be able to do is that if i put a zero into one of the four cells, for the other three cells to represent 80/3 rather 80/4, or if two of the four cells had a 0 in for the other two to show 80/2 and so forth

Any ideas?

Kind regards

jllatham say:

Unfortunately in situations like this, the fact that a cell can only hold either a value or a formula, you could only do this once because as soon as you type in a 0 into one of the cells, its formula is deleted.  But if that's an acceptable setup, then this would work.  Assuming A1 holds your value of 80, and the other cells in question are A2:A5:

formula for A2: =$A$1/(4-COUNTIF(A3:A$5,0))

formula for A3: =$A$1/(4-COUNTIF(A4:A$5,0))

formula for A4: =$A$1/(4-COUNTIF(A5:A$5,0))

formula for A5: =$A$1/4

 

Then if you start overwriting the formulas with zeros from A5 UP, the counts above will change.  But again, you're deleting the formulas as you enter the zeros.

 

ONE non-destructive method:

Use one more cell to indicate the number of results to return (1 through 4).  Let's say you put that into cell B1, then use these formulas

Formula for A2:  =$A$1/$B$1

Formula for A3:  =IF($B$1>COUNT($A$2:$A2),$A$1/$B$1,0)

the formula in A3 can be filled down from there, and will become

Formula for A4:  =IF($B$1>COUNT($A$2:$A3),$A$1/$B$1,0)

Formula for A5:  =IF($B$1>COUNT($A$2:$A4),$A$1/$B$1,0)

 

A SECOND non-destructive method would be to use an extra cell for each of the calculation cells, say B2:B5 and enter a 0 or anything into the cells in column B to indicate the divide by value should be reduced.  Then the formulas for A2:A5 all become:

 

Formula for A2:  =IF(B2="",$A$1/(4-COUNTA($B$2:$B$5)),0)

and that formula can be filled down and will become:

Formula for A3: =IF(B3="",$A$1/(4-COUNTA($B$2:$B$5)),0)

Formula for A4: =IF(B4="",$A$1/(4-COUNTA($B$2:$B$5)),0)

Formula for A5: =IF(B5="",$A$1/(4-COUNTA($B$2:$B$5)),0)

 

I hope these options help you get the job done.

'Version 1.41

'Attribute VB_Name = "modArray2D"

Option Explicit
Option Private Module

'The routines of this module are primary designed for the processing of arrays that are read from _
  a sheet in Excel.

'If you are processing large amounts of data there is no faster way than read all the data into _
  an array and process this.
'This method is up to 200 times (!) faster than working directly with cells in Excel (depending _
  on version).
'One problem with the processing of arrays, is to insert, remove and move columns or rows.
'The memory architecture of VB allows to process a column of an array in one piece, because all _
  the variables are located directly behind each other in memory.
'Therefore, the processing of arrays can be enormously speed up when dealing with the memory _
  routines from the windows kernel.
'Because of this, the exact size of the array elements must be known, so we can work only with _
  typed arrays.
'In Excel, must have an array of type Variant if we want to read or write data from/into a sheet. _
  Therefore all routines of this module will only process variant arrays.

Private Declare Sub CopyMemory Lib "kernel32" Alias "RtlMoveMemory" ( _
    lpDest As Any, lpSource As Any, ByVal nCount As Long)
Private Declare Sub ZeroMemory Lib "kernel32" Alias "RtlZeroMemory" ( _
    Destination As Any, ByVal Length As Long)

Sub MoveCol(Arr() As Variant, ByVal Position As Long, ByVal Steps As Long, _
    Optional ByVal Count As Long = 1)
  'Move count columns in 2D array from position steps to left or right
  Dim i As Long, j As Long, Size As Long, Temp

  'Exit if bad input
  If Position < LBound(Arr, 2) Then Exit Sub
  If Position > UBound(Arr, 2) Then Exit Sub
  If Count < 1 Then Exit Sub
  Select Case Sgn(Steps)
    Case 0
      Exit Sub
    Case 1
      If Position + Steps + Count - 1 > UBound(Arr, 2) Then
        Steps = UBound(Arr, 2) - Position - Count + 1
        If Steps < 1 Then Exit Sub
      End If
    Case -1
      If Position + Count - 1 > UBound(Arr, 2) Then
        'Count = UBound(Arr, 2) - Position + 1
        Exit Sub
      End If
      If Position + Steps < LBound(Arr, 2) Then
        Steps = Steps - (Steps + Position) + LBound(Arr, 2)
        If Steps > -1 Then Exit Sub
      End If
  End Select

  'Save elements that moves
  ReDim Temp(LBound(Arr) To UBound(Arr), Position To Position + Count - 1)
  'For I = LBound(Arr) To UBound(Arr)
  Size = (UBound(Arr) - LBound(Arr) + 1) * 16
  i = LBound(Arr)
  For j = Position To Position + Count - 1
    'Temp(I, J) = Arr(I, J)
    CopyMemory Temp(i, j), Arr(i, j), Size
  Next
  'Next

  'Move the elements
  If Steps > 0 Then
    'For I = LBound(Arr) To UBound(Arr)
    For j = Position To Position + Steps - 1
      'Arr(I, J) = Arr(I, J + Count)
      CopyMemory Arr(i, j), Arr(i, j + Count), Size
    Next
    'Next
  Else
    'For I = LBound(Arr) To UBound(Arr)
    For j = Position To Position + Steps + 1 Step -1
      'Arr(I, J + Count - 1) = Arr(I, J - 1)
      CopyMemory Arr(i, j + Count - 1), Arr(i, j - 1), Size
    Next
    'Next
  End If

  'Fill in saved elements
  'For I = LBound(Arr) To UBound(Arr)
  For j = Position To Position + Count - 1
    'Arr(I, J + Steps) = Temp(I, J)
    CopyMemory Arr(i, j + Steps), Temp(i, j), Size
  Next
  'Next
  ZeroMemory Temp(i, Position), Size
End Sub
