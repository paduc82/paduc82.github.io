ossiemac say:

Try the following where invalid password message becomes the new InputBox prompt instead of a separate MsgBox. Easier for the user if they inadvertently enter the wrong password.

Sub RectangleRoundedCorners2_Click()

    Dim Res As String
    Dim strPrompt As String
    Dim strTitle As String
   
    Const PWORD As String = "Flower"
   
    strPrompt = "Enter password (or Cancel to exit)"
    strTitle = "Password entry"
   
    Do
        Res = InputBox(Prompt:=strPrompt, Title:=strTitle)
       
        If Res = vbNullString Then
            MsgBox "User cancelled. Procesing terminated."
            Exit Sub
        End If
       
        If Res = PWORD Then
            frmSearch.Show
            Exit Sub
        Else
            strPrompt = "The password not recognised!" & vbCrLf & _
                    "Re-enter or Cancel to exit."
        End If
    Loop

End Sub

tony jollans say:
'Ambiguous name' means you have a duplicate name somewhere in your VBA code, which means that VBA cannot run the macro it is trying to run; you may even have multiple duplicates - perhaps a block of code has been duplicated. You need to go through each macro making sure you have unique names. You do not need to delete macros, if you don't want to, just rename whatever VBA tells you is 'ambiguous' - with something you expect to be unique.
