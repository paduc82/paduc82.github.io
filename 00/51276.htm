Can I convert a WindowsXP movie maker project to a Windows8 Windows - Microsoft Community

pcolayank say:

Can I convert a WindowsXP movie maker project to a Windows8 Windows Live Movie Maker project? I have tried to open a project file to no avail.

Used Windows XP Windows Movie Maker to make movie with full music transitions, video, stills, etc.  Saved project on old machine.  Bought a new machine with Windows 8 and Windows Live Movie Maker.  It will not open my project when copied all the folders and files over.  It will not even recognize the file format.  Is there a trick or conversion available?  Exhausted!

considerate_guy say:

First, a project file isn't actually a movie.  It is like the blueprint or the recipe for making a movie.  Internally the project lists the files names of the source files (video, photos, music) and the order they belong, the names of transitions applied, etc.  But it does not actually contain any video, photos or music itself.

 

So in order to open a project file on another PC, in addition to the project file, you would need all the same source files too.

 

The next wrinkle is that a project file is intended to open in the same version in which it was created.  Windows XP came with Windows Movie Maker 2.1.  But the latest version of Movie Maker is Movie Maker 2012.  It cannot open the .mswmm project file from WMM 2.1 (XP) because it is too many versions earlier.

 

There is a possible workaround; you could install Windows Movie Maker 6.0.  This is the Vista version of the classic Windows Movie Maker.  It can open up a project file from WMM 2.x.  I would give that a try.  Be sure you have the original source files too (video, music, photos) and it should then open in WMM 6.0.  If it doesn't fully (you see Red Xs instead of video, photos or music) it means it isn't finding the source files in the same location.  This can happen if the drive letters or folder names are different.  If this is true, then double-click on a Red X and navigate to where the source file resides on the new PC.  It should fix all source files in that folder.

 

Once you have the project opened and working, I would do two things:

  1. I would save a new copy of the project file, but with a different name.  Use the File > Save project as option to do this.  This will be a .mswmm in WMM 6.0 format.  If you later want to open this in the newer Movie Maker 2012, you can.
  2. Save/Publish as a movie.  This is the step that could have helped on the XP machine.  As I stated before, a project file has no actual content within it that is viewable.  Like a blueprint, it isn't a house.  And like a recipe, it isn't edible.  It's not until you build using the blueprint, or bake using the recipe that you get something tangible.  To do that, you need to publish the movie.  In WMM 6.0, you choose File > Publish Movie.  The resulting .wmv file should be playable on another PC, even one without Movie Maker.

 

Update: One additional caveat that I should mention.  Each version handles certain transitions and effects differently.  Even in going from WMM 2.x to WMM 6.0, you may notice differences in the way certain things are previewed.  Certain title animations may behave slightly differently, for example.  If you have custom transitions/effects/titles that you used in WMM 2.1 (XP), they won't be there in WMM 6.0.  (There's a whole different set of custom add-ons for WMM 6.0.)  The differences between WMM 6.0 and the later Movie Maker versions are even greater. If you are still able to open the projects on your older XP machine, you may just want to finish any edits there and save the movie (File > Save Movie) and only bring the movie forward to the Windows 8 machine.  The same applies to WMM 6.0, that if you are able to edit there, publish the movie and bring that forward, rather than the project.

The usual advice is something along the lines of "something has been disabled in Group Policy...", and that is probably true that something has been disabled, but you need to know what that something is and what to do about it.

If you are using XP Home, you will not get far with advice to use the Group Policy Editor since there is no Group Policy Editor in XP Home and that really doesn't tell you where to look in the Group Policies even if you could run Group Policy Editor.

You need a comprehensive solution that will work for all versions of XP and do not require hunting around in the Group Policy Editor or the registry to find where things are that might have been disabled.

Unless you disabled these things on purpose, the chances are good that your system has a malicious software infection.  The malware knows what tools you are going to use to try and find and remove it, so the malware disables the things you are most likely to use and keeps them from running so you can't find the malicious software and remove it.

If your system has this kind of affliction, whatever malicious software tools you are currently using or have been using have failed to protect your system so you may want to expand your malicious software prevention and detection horizons to prevent these kinds of afflictions in the first place.

The malicious software will be happiest to fool you into thinking you need to so something drastic to fix your system - like a Repair Install, System Restore point, or a total reinstallation of XP.  That is what it would like you to do, but such measures are not required.

You need to fix the immediate problem of the tools not working, then scan your system for malicious software when you are done.

No matter what kind of malicious software scanning tools you have already used, they are unlikely to fix this problem because they cannot tell if the changes to your system were on purpose (you or an Administrator made them) or some malicious software changed them, so the scanning tools will leave these things alone (this is usually a good thing).

If your system is afflicted in this manner, there are probably other things that also don't work - like Registry Editor, System Restore and Command Prompt, so fix them all at once even if you haven't found out they are broken yet.

Try re-registering Photos app. Perform these steps:

Get-AppxPackage -allusers Microsoft.Windows.Photos | Foreach {Add-AppxPackage -DisableDevelopmentMode -Register "$($_.InstallLocation)\AppXManifest.xml"}
