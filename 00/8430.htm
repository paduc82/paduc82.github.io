ken sheridan say: As far as I can see, what you have is a many-to-many relationship type between Assessments and Risk Phrases.  This should be modelled by a third table, so, in broad outline, your tables would be:

Assessments
....AssessmentID  (autonumber PK)
....AssessmentNumber (indexed uniquely)

RiskPhrases
....RiskPhraseID  (autonumber PK)
....Risk Phrase (indexed uniquely)

and to model the relationship type:

AssessmentRiskPhrases
....AssessmentID  (FK)
....RiskPhraseID  (FK)

The primary key of the last table is a composite one comprising both columns.  Diagrammatically the model would be:

Assessments----<AssessmentRiskPhrases>----RiskPhrases

i.e. the AssessmentRiskPhrases table resolves the many-to-many relationship type into two one-to-many relationship types.  For an example of this sort of basic many-to-many relationship type and how to represent it in a form/subform, with a combo box in the subform to select an item, see StudentCourses.zip in my public databases folder at:

https://onedrive.live.com/?cid=44CC60D7FEA42912&id=44CC60D7FEA42912!169

If you have difficulty opening the link copy its text (NB, not the link location) and paste it into your browser's address bar.

BTW, note how in the subform in my demo you can insert a new row into the referenced Courses table by typing a new course name directly into the combo box.  Code in its NotInList event procedure then, after user confirmation via amessage box, inserts a row into the Courses table.

A few supplementary points:

1.  Do not use the generic ID as a column name for keys.  Always use specifically descriptive names, as I have above with AssessmentID  and RiskPhraseID.

2.  Do not use spaces or other special characters in table or column names.  Either use CamelCase or represent a space by an underscore character Like_This.

I'm well aware that Northwind includes examples which do both of these, but I'm afraid that Northwind contains more than a few examples of bad database design practices.

rohn007 say:

Office licensing is a wonderful tangle.

Assuming you have a "Retail" license, that is the boxed CD for Office 2007 Home and Student. If you take a look at the Product Key sticker inside of the box, on the bottom right corner of the sticker it should say FPP.

This type of license allows you to install and activate on up to 3 computers "in your home" (who knows exactly what that means, since it apparently included students away at post secondary school ... ) at the same time.  It allows you to re-install and re-activate on the same computer "unlimited" number of times.  It allows you to transfer the installation to a new computer (every 90 days, sooner only if hardware dies) as required.  To transfer installation you are required to ensure the current installation is no longer usable.  That means either it has been uninstalled from the "old" computer or the old computer is dead (no longer useable), or the old computer has been returned to the builder (which assumes it will either be disposed of or reformatted as part of reconditioning before resale).  In all 3 options, the old installation is no longer usable.  Then you can install and activate on the new computer.  In that case, if the activation system tells you that you have "too many activations" (a it did you) you have to do a phone activation and talk to a "service" rep. When they understand the situation they are supposed to "wave a magic wand" and the activation will complete.

FYI: Doing an "uninstall" does NOT actually contact the activation system.

The "activation" process takes a snapshot of your system identifying several pieces of hardware and software. It then generates a "hash" code and stores that in the activation system.  When you re-activate on the same computer, it checks the stored hash against the new one.  The activation system allows you to do "limited" upgrades before it considers the computer as a "new" computer and the install is a transfer rather than a re-install.  Limited means changing one, or two at most, elements ie RAM, CPU, HD, OS at the same time.  Changing the mother board is considered a new computer.

Periodically (undefined period) the activation system will check your computer's activation status.  I'm guessing that if the system does not see your computer for a while it is considered inactive, allowing the license to be activated on a new machine without human intervention.

Having activated it 9 times is irrelevant. But this is the second time this point has been raised by MS with someone trying to activate a legitimate license.

Installing "from a different link" is just voodoo for a "too many activation s" type error message. The source of the installation is irrelevant for that type of error.

Yes, this sort of error shouldn't be difficult to fix. The problem is that you have to know to contact the right support. In this case it would the phone activation support. Regular support normally doesn't do any good. You got very lucky. 

Yes, actually the "latest and greatest", Office 365, does handle licensing better (one of the few things I agree it does better).  For Office 365 they have created a "MyAccount" web site.  When you activate your 365 license you have to provide an email address.  After the initial activation you don't need your product key any more. You just log in to the MyAccount page with your email address.  From there you can install, activate and deactivate your 365 licenses on what ever machine you want. You have total and "instant" control.  This is especially handy for the 5-concurrent machine licenses.

Remove time stamp on tracked changes without removing author - Microsoft Community

c shep say:

Remove time stamp on tracked changes without removing author

I want to remove all the timestamps for tracked changes and comments, but leave the author of the change.  We have multiple people editing, so it is important to keep the author info, but I don't want to share the times changes were made.  I know I can remove ALL personal information, but that removes the author too, which I need to keep.  I want to remove ONLY the timestamps.

john korchok say:

I'm not aware of any way to remove only the date and time without removing the author except by directly editing the XML. Use a copy of the original document:

  1. Download and install Notepad++ from here (It's free): Notepad++
  2. Add .zip to the file ending. Word will warn you about this, click on Yes.
  3. Expand the Zip archive.
  4. Open the word folder.
  5. Open document.xml in NotePad++.
  6. Choose Search>Replace.
  7. In the Find what: field, paste: w:date="[\d\W]*\w[\d\W]*\w"
  8. In the Replace with: field, leave it blank.
  9. In Search Mode, check Regular expression.
  10. Click on Replace All.
  11. Save the file and close Notepad++
  12. Update the Zip archive with the modified file, or simply rezip all 3 folders and the [Content_Types].xml file, setting the file ending back to .docx and open in Word.
