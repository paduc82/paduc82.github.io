robert aldwinckle on forums say:

Each time I open an new page it opens in minimised form.


Check your assumption.   Minimized means that a task is reduced to a Taskbar icon.   In W7 and W8 with Aero enabled you could hover over it and see a thumbnail.   If you then hover over the thumbnail you would see a preview of what the opened window would look like.   Are you claiming that you don't see a window at all when you do that?   If so, it probably means that you have a normal mode window which is off screen or hiding somehow.

 

You can also use the thumbnail to give you more clues about the state of its window.   Right-click on it and examine the window's menu.   If the window is truly in its Minimized state Minimized will be grey and unusable.   Hover over the menu items to be sure about their state.

 

Often when this question is asked the poster really means that the window is in its normal state and would like it to be opened bigger.   Then the Size menu could be used to make the window maximally sized if desired.   If that window is the last IE window closed its size would persist for the next time an IE window is opened.

 

A better alternative to maximally sizing a normal mode window is to toggle into Fullscreen mode, e.g. with F11 and toggle off that mode's Auto-hide option, e.g. with Alt-Space, H.   That option would persist too.

No you don't understand me correctly, let me explain in more detail. Relational data tables should be tall and thin, not wide. So you will need more tables, but not one for each question. You don't identify data by encoding it with field or object names. You identify using the data in records.

tblQuestion

QuestionID (PK Autonumber)

Question

tblDriverAnswer

DriverAnswerID (PK Autonumber)

DriverID (Foreign Key)

QuestionID (FK)

Answer (Yes/No)

So you have the one table for the answers with a FK to link back to the driver and another FK to link back the Question. Then, yes you would use a query joining the three tables to get a listing of the answers for each driver. With this structure if you need to add or remove a question, you just modify tblQuestion. 

Does that help you understand?

As to the expression, Try this:

IIF(DateDiff("yyyy",[HireDate],Date())>=1,DateDiff("yyyy",[HireDate],Date()),DateDiff("q",[HireDate],Date())*.25)

This should return the completed years if 1 or more or a share of the year for the # of completed quarters.

brianyoung_4444 say:

Word 2010 -- Page numbering across sections.

In Brief:  I need two sets of folios on every page: "Page x of y" in the "Header area" for each "Section", then "Sequential Document Page x" in the "Footer area".  I'm using the "Quick Parts" field item "page" to lay down the "x" value.

The Problem:  Since the count value of "page" gets reset at section boundaries I can't accomplish the "Sequential Document Page x".

The Question: Is there another value; or a way of calculating the value of the sequential page folio, that can be used in the "Header/Footer" areas?

doug robbins - mvp office apps & services (word) say:

In reply to your first question, nothing needs to be done to "migrate the number of pages into the bookmark" as an attribute of the range of a bookmark is the number of the page on which it appears.

An alternative method is to leave the page numbering continuous in each Section, then for the sequential numbering of all pages, you can just use the { PAGE } field and for the page number of the page in each Section starting with Section 2, use

{ = { PAGE } - { PAGEREF Section1 } }

Then for Section 3, use

{ = { PAGE } - { PAGEREF Section2 } }

etc. where you have a bookmark Section1 on the last page of Section 1, Section2 on the last page of Section 2, etc.

One thing that you could do to perhaps simplify your process of creating the the documents would be to set up each attachment with the "page numbers" and "number of pages" manually typed into the body (not the header) of the document at the top of the page, and with

{ PAGE }

in the footer.

doug robbins - mvp office apps & services (word) say:

You will need to have the page numbering restart at 1 for each Section (via the Format Page Number facility) and use { PAGE } of { SECTIONPAGES } in the header, assuming that you want the y to reflect the number of pages in the Section.

To get the Sequential numbering, you will need to insert a bookmark at the end of each Section, say Section1, Section2, Section3 and then for the sequential numbering, for the first Section, you can just use a { PAGE } field, then in subsequent Sections, you will need to use the following field constructions

Section 2     { = { PAGEREF Section1 } + { PAGE } }

Section 3     { = { PAGEREF Section1 } + { PAGEREF Section2 } + { PAGE } }

etc.

Note, you must use CTRL+F9 to insert each pair of field delimiters { } and you use ALT+F9 to toggle off their display.

hansv mvp say:
Why not instruct the user to use Ctrl+' (Ctrl+apostrophe) to copy the value from the previous record?

ken sheridan say: Another approach would be to set the value entered as the DefaultValue property of the control and carry it forward to subsequent records, rather than bringing it forward each time.  This avoids the user's having to elect to bring it forward in each subsequent row as the value will already be present in the otherwise empty new row until cancelled.

You'll find an example as Defaults.zip in my public databases folder at:

https://onedrive.live.com/?cid=44CC60D7FEA42912&id=44CC60D7FEA42912!169

If you have difficulty opening the link copy its text (NB, not the link location) and paste it into your browser's address bar.

This little demo file illustrates how to do this for either the current session of the form only, or for the current and subsequent sessions.  If doing the latter in a multi-user environment, make the Defaults table a local one in the user's personal front end.  Separate users will then be able to set their own defaults.
