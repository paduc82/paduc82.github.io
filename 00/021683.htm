Couldn't find Outlook 2013 old sharepoint list

I am a contract consultant and recently moved to a new contract. I had removed the Sharepoint List from my Outlook 2013; at the user interface level... so no lists appeared in the Folders view. However, I continued to receive prompts for Sharepoint credentials for my old customers Sharepoint site.

The solution: If you go to File -> Accounts and click on the Sharepoint Lists tab, nothing appeared. However, I found that deleting the SharepointLists.PST file was required from C:\Users\[ME]\AppData\Microsoft\Blah\Blah\Blah resolved the issue. An easy way to do this is to search for *.pst files in the C:\Users\[me] folder (where [me] equals your windows username)

For the Microsoft Moderators: Please report this as a bug. For some reason, Outlook 2013 was still mounting that SharepointLists.PST file and attempting to build connections to the Sharepoint servers, even though the lists had been deleted and there were no lists in the "Accounts Dialague Box ->SharePoint Lists" tab. This will really bake someone noodle, if they aren't a power user. It negatively impacts the user experience of quality and is probably an easy fix.
