accesscoder say:

Hi Richie

It sounds like there might be some confusion here between the subform and the subform control.  The subform control is similar to any other control on the form in that each control must have a unique name.  However you can place the same subform in 3 different subform controls.  When referencing something on one of the subforms, you need to reference the control that the subform sits in as in "Forms!Mainform!SubformControl" and then the actual subform that resides in that control, as in "Forms!MainForm!SubformControl.Form".

I hope this helps

Okay, let me tell you another way because I see the link I provided isn't doing it for you...

Open the form you want to place the control on on design view and then open the form you want to get the value from.  If these forms are connected (form/subform) then open the one form (with subforms).

3.  In the windows that opens look for Forms Loaded and click the [+] sign.  Keep going till you find the form you want to get your value from.

4.  When you find it double-click it and it should show in the upper box.

5.  Click [OK] and that value should show in your text box.

That is the path.  Now you can use that and it should point to the value you desire.

Use it to Repair Permissions on the Macintosh HD drive and reboot after running the utility.

Now for the font.

Go to the Applications folder and run the FontBook app (located in Applications/Utilities folder) to resolve any duplicates. FontBook Help can guide you through the process. If you do find any duplicates you must restart your system after resolving them. Now check whether the fonts are displaying correctly.

If font-related problems persist, you may have a damaged font file – FontBook won’t tell you about that. The first thing to do is to click on a problem character in the Word document so you can see what font it's supposed to be. If Word can tell you that much then, assuming the font is installed on your system, delete and reinstall that font; otherwise either install the font on your system or substitute a different one.

If font-related problems still persist, there may be another damaged font causing conflicts. In that case, you'll need to identify the damaged font. To do that:

1. Move about half of the fonts there by families (and it's better to restart);
2. If the problem persists, it lies in one of the active fonts; if not, swap the fonts between the active folder and those in the isolation folder;
3. Colour the fonts in the isolation folder with green; and
4. Repeat from step 1
In this way, you'll be testing about 1/2 of the fonts, then 1/4, 1/8, 1/16, 1/32, etc., thus working through all the fonts in a few iterations.

If you're still getting font errors after this, you either have more than one corrupt font, or the problem lies elsewhere.

Bullet Styles and Templates - Microsoft Community

pwat say:

Bullet Styles and Templates

I have a Master Template in Word which contains my full set of styles. Sometimes I apply another template over the top of my Master Template that only contains styles that have different formatting to the style of the same name in the Master Template. If I have a document that has text styled with my custom 'Bullet' style from my Normal Template, when I apply my secondary template over the top, the formatting of my bullet changes, even though the bullet style doesn't exist in the secondary template. The formatting of the bullets should remain the same as the style is inherited from the Master Template, but instead it is being updated by the secondary template. Do Word templates have some built in bullet formatting that could be overriding my bullet style? How can I prevent this from happening?
john mcghie say:
No problem.  Let me put my thinking cap on.

To begin with, a "Style" is like an Array: if you think of an Excel spreadsheet, you get the idea.  Each row is a Style name, and a paragraph style has about 1,200 columns.

Not one of which contains any "List" or "List Format" information.

A List Template is, as its name implies, the instructions to create the formatting for a list.  It is "like" a style, by has two important differences: the first is that it does NOT have a name.  The second is that it is a multilevel structure: a top level with eight dependent child levels.

Each level contains the list formatting for one level of a list.

A list is made up of paragraphs.  Each paragraph is a list item.  Study HTML Lists and List Items: Word uses the same mechanism.

The "level" of a list item is controlled either by direct formatting, or by applying a style that has a level property.  For example you will find that List Bullet through to List Bullet 9 are pre-defined, waiting to be used to set the levels in a bulleted list.

So there you are: a Paragraph style controls the font and leading and spacing, the List Template controls the bullets and numbering.  If you go to Format>Style>Modify>Format>Numbering>Outline Numbering, you can set an association between each level in the list and its corresponding paragraph style.  Note the path I took to get in there.  There are several ways to get to a dialog that looks like this one, but unless you begin Format>STYLE>Modify... you won't get where you need to be and it won't work. Ignore the tabs labelled "Bulleted" and "Numbered": they are anachronistic compatibility options that really should not be there.

When you get in to that dialog, you should see the field "Link Level to Style".  If you don't, there is a small, completely unlabelled disclosure triangle at the bottom left corner of the dialog: click it to reveal the style controls.

Now:  So far you have created nine styles (you don't actually need to create more than you intend to use, but I always create at least one extra so I can fiddle with the levels while editing).  You have created a similar number of "List Levels" in a "List Template".

Whenever you apply one of your paragraph styles to a paragraph in the document, it will now make that paragraph a member of the LIST and apply the List Template to format it.

Remind me that I skipped two very important (albeit confusing...) points there. I am trying not to frighten the horses just yet.

In the Customise dialog you can see the seven most recent List Templates created in the document.  However, a document can theoretically have a very high number.  Word automatically trims the number to around 200 because there were problems with crashing as a result of a document containing too many.

A List Template sets the formatting for a List. It calls in Paragraph Styles to format the paragraphs. 

There can be only ONE instance of each List in a document, controlled by its OWN List Template.  There can be several hundred list templates, each one controlling a different list, but you will try very hard not to let that happen, because it can lead to a very unstable document.

For bullets it doesn't matter: just apply the List Bullet styles and the bullets will flow from start to finish throughout the document and all look the same.

For List Number, you can do the same thing: but normally you would set the list to "Restart" at each new text list.  It's still the same list from start to finish, you just rest it to start at 1 for each topic.

So far, we have a List format and nine paragraph styles that together form a list.  Everything works well until you copy the text somewhere...  Then you discover that the List Template does not go along with the copy.  The bullets or numbers all change formatting when they get to a new document.

Why?  Because the List Template is part of a document's fundamental structure: it cannot move.  Remember I said it does not have a name?  It has a number.  If you copy some paragraphs formatted with List Template 23 to another document: they will get the formatting of List Template 23 in the destination document.  List templates are numbered in the order they were created: two documents are unlikely to be the same.

Most people who have had a bit to do with computers over the years would recognise this as a shriekingly funny design bug.  The sort of thing Dilbert would expect the Marketing Department to create.  Oh wait...  The Marketing Department DID create it.

After about ten years of us splitting our sides, they finally sent some hapless coder in there to try to fix the mess. Sadly when they lifted the lid they discovered a foul and cancerous can of worms that would break ten years' worth of documents if they touched it.  So in true commercial coding fashion, they stuck a giant band-aide over the top.

The List STYLE.  The List Style does not CONTAIN a list Template (remember, they can't move, they're part of the document).  It contains the instructions to make one.  And which styles to link it to.  When you create a List Style, it not only enables a list template to have a Name, it also contains pointers that link the styles and list template and list style together.

So now when you apply List Bullet 3 to a paragraph, it formats that paragraph with List Bullet 3 and List Template 17.  It also makes that paragraph a member of the "My Standard Bullets" list.  When you copy one or more members (paragraph list items) from one document to the other, the List Style enables Word to collect all nine Paragraph Styles (whether you have used them or not), all nine list template levels (whether used or not) and their List Style.

When this hits the destination document, Word first looks for the list style.  If it is there, it simply joins the incoming paragraphs to it.  If not, it adds the List Style, uses that to create a List Template, copies the paragraph styles in, links them to the List Template.

If you get it right, the list will copy happily from document to document or template to document.

You will have some questions {grin}.  Remember, my purpose in being here is to answer those questions, so let me have them.  And the other guys will do the same.

Never address a question to a particular person in here: ask "The Forum".  That way your answer will come from the first person around the clock who is awake and knows the answer.

Hope this helps
