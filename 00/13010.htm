First, Windows "using" a large chunk of memory should actually be a good thing! Memory (RAM) is the fastest and best place for Windows to store what it needs for really fast access, and unless something is drastically wrong, Windows will use and organize and swap stuff in and out of your memory space effectively and efficiently and advantageously.

The exception is if you have a memory leak, and the leak uses up the bulk of memory and reserves it, so Win can't use it. That's when things get ugly and slow. Any program or process can cause a leak, and so you need to find out what's using it. But, if it's just typical Windows "stuff," swapping in and out in good order, that's exactly why you have RAM and why every article you read says that more RAM = More Speed. You WANT it to be heavily used and have a lot of stuff in it!

You can use a Windows tool called "poolmon" to peek at what's using the memory and if you have a leak. You may have to download it from SysInternals, the Win development group's toy box. They have some really great stuff. If a tool you need won't run in Win 10, try installing it in Compatibility Mode. Most tools will install and work that way.

What, if anything, is slowing down your machine is the high disk access. That's not a good thing, and can ruin the performance.

"System" is typically the Windows Kernel, and that can be running a ton of services. You may need another fine toothed diagnostic program like "ProcMon" Windows Process Monitor, from the development team to gather data and then analyze the specific service that's overloading the memory.

Some common culprits are PreFetch, Indexing, and SuperFetch. They can be toned down to not be as aggressive and hog CPU, Disk, and Memory.

The problem with ProcMon is that it frequently results in TMI and analyzing the huge pile of data that it can accumulate in mere seconds can be daunting. Learn to use the filters and watch for trends rather than trying to analyze it line by line.

BTW, when you run it, turn off things like Norton (put it into quiet mode) and be sure your computer has been sitting idle for about 15-20 minutes after a reboot so that all the housekeeping and homework processes get a chance to run and finish up.
