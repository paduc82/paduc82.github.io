jim gordon mvp say:
There are two different passwords in play here.

1. When you open a workbook and are asked for a password, that is the password which locks the file from being opened without the file password.

2. When you password protect a worksheet for forms, that is an independent password that locks cells based on the locked property of the cells on the worksheet.

So even though you entered the password to open the workbook, you still need to enter the (probably different) password to unlock the worksheet.

Unreadable content and pivot table stripped after adding a calculated field with T.INV

Hi,

If I choose "Yes" the pivot table is flattened / stripped, i.e. shows up as just values. This is very annoying since the formula is necessary for all other calculations.

I've narrowed down the issue to the T.INV function which seems to corrupt the whole pivot table - additional details below. Does anyone know why this happens and how to add the field without corrupting the table?

- The calculated field formula is =T.INV(0.05, 1000). It is the only calculated field. The issue persists regardless of how the calculated field is named.

- There is only one field in the table (the calculated one) and there is only one pivot table.

- The field is added successfully and displays correct values before closing.

- Using Excel 2010, version 14.0.6120.5000 (32-bit). Issue is present on other computers with the same version.

tmkst say: I've managed to resolve the issue myself. For those who might encounter similar problem in the future - using a function with a "dot" in the name (e.g. T.INV, NORM.DIST, FLOOR.PRECISE) in a calculated field will corrupt the pivot table. After saving and reopening the file the pivot table will be flattened/stripped. You can get around this issue by using older variants of the functions where available (e.g. TINV instead of T.INV).

The reason you are getting this error is probably because the service was disabled. To restart it, do the following steps:

1. Press WIN + R

2. Type services.msc

3. Press enter and wait for it to load

4. Look for Microsoft Office ClickToRun Service

5. You will find that it is stopped. Right click it and click 'Properties'.

6. If the startup type is set to disabled, set it to manual or automatic.

7. If you set it to Manual, you will need to start the service. To do this, right-click on the service in the list and click 'Start'.

8. Try opening Office again. It should start correctly this time.

Aside from auto-mapping issue that was mentioned above, another reason could be with the large shared mailboxes when Outlook is in cached mode. What happens is Outlook generates too many connections to the server while attempting to cache the mailbox. We suggest that you disable caching for the shared mailbox.

To do this:

  1. Open the mail profile settings in Outlook.
  2. Click on More Settings.
  3. On the Advanced tab, uncheck Download shared folders.

Also, follow the steps below on how to update folder:

  1. Right-click the folder, and click on Properties.
  2. Click on Clear Offline Items, and click OK.
  3. On the Outlook ribbon, click the Send/Receive tab.
  4. Click on Update Folder.
