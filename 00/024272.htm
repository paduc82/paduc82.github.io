scottgem say:

Why go through multiple steps when you only have to do one?

There is no such thing as a Time datatype. There is one datatype for date and time. It is actuall a Double precision type where the Integer portion is the number of days since 12/30/1899 and the time portion a fraction of a day (i.e 6AM= .25). That's why if periods span midnight you need both together.

So run ONE update query

UPDATE table SET fieldname = datefield+TimeValue(...);

You probably should add the date to all the other time fields.

ken sheridan say:

But I would still have to update the original value to insert the colon before running query you suggest wouldn't I?

Access won't see 0950 as the correct time without the colon inserted. 

I showed you how to do that earlier:

    CDate(Format([StartTime],"00:00"))

More importantly, however, if you currently have one column with the date and other columns with the times, then simply adding the date to each of the times will give you incorrect values.  If you have a date of 2015-08-14 and a start time of 18:00 and an end time of 02:00, i.e. 2:00 AM the following day, then adding the single date to each would give you a start value of 2015-08-14 18:00:00 and end value of 2015-08-14 02:00:00, i.e. the shift ended 16 hours before it started.  The elapsed duration would consequently be a negative value.  To add the date and times like this it would be necessary to take account of both times, so that 1 would be added the end value to place it in the following day.

My demo, to which I referred you earlier does exactly as you are doing, having a single date for the day on which a shift starts and separate start and end times, which, under the skin, are implemented as 64 bit numbers with a zero integer element.  The functions in my demo cater for this and return the correct duration, e.g.

? TimeDurationAsDate(#18:00#,#02:00#)
08:00:00

whereas a simple subtraction would return a negative value:

? #02:00# - #18:00#
-0.666666666666667

In the latter case the sign would be removed if the return value is formatted as time, but the result would be incorrect:

? Format(#2015-08-14 02:00# - #2015-08-14 18:00#,"hh:nn")
16:00

Separate sections with different Headers/Footers - Microsoft Community

rebeccasmoot say:

Separate sections with different Headers/Footers

Is there a way to 'Unlink' from the previous section?
suzanne s. barnhill say:

Yes. Place the insertion point in the header or footer you want to unlink from the one in the previous section. The contextual Header & Footer Tools | Design tab of the Ribbon will be displayed. In the Navigation group, click Link to Previous to turn it off.

Some things to keep in mind:

  1. Each section can have up to three different headers/footers (if you have enabled Different First Page and/or Different Odd and Even). Each of these--primary or Odd Header/Footer, First Page Header/Footer, and Even Header/Footer--must/may be unlinked from its counterpart in a previous section (you can link or unlink any one of them independently of all the rest).
  2. It may not be necessary to unlink the header/footer if the different content you want reflects content in the document, such as a chapter title: you can use a StyleRef field to pick up the current content of a particular style (such as Heading 1) in the header/footer. If you want to leave a given header/footer blank, however, you do have to unlink it.

Sorry, but you are mistaken.  The OEM license terms are different than the Retail license terms which you are referring to.   The Retail license allows install on up to 2 computers at the same time. phillip bagley say:

Hi,

I did resolve this issue for anyone else who may run into this problem.

Previously, I had been opening old ".doc" files and saving them as ".docx".  I had assumed that this would "convert" the file to the latest format before saving as a pdf.  This does not happen and the document remains unconverted and therefore continues to experience the pdf search issue described above.

The simple resolution is to open the ".doc" file,  "convert"  the file (File>Info) and then save as .pdf.  This will give you a correctly formatted and accessible pdf file that can be searched.

Hope this helps.
