Create A Class Module - Microsoft Community

jdynomite say:

Create A Class Module

I'm trying to learn to work with class modules.  I would like to create a class module that wraps the module object and adds additional properties and functions for convenience.

I would like to have a parent class called Modules that will serve as parent to the module class and work to collect all the modules in a project.

I would appreciate any advice on accomplishing this task.

Thanks,

-J

tom van stiphout say:

As you probably already know, VBA does not support all features commonly required for an object oriented language. Inheritance is one such feature.

However you can create a class myModule that implements all features of the Module class with a simple pass-through. For example I created a new class "myModule":

Option Compare Database
Option Explicit

'Private variable represents the Module object we're wrapping.
Private theModule As Access.Module

Public Sub Init(mdl As Access.Module)
    Set theModule = mdl
End Sub

Public Property Get Application() As Access.Application
    'Put this line in every public procedure.
    If theModule Is Nothing Then MsgBox "Aarrcchhhh, I told you to first call Init!", vbCritical
   
    Set Application = theModule.Application
End Property

Public Property Get CountOfDeclarationLines() As Long
    'Put this line in every public procedure.
    If theModule Is Nothing Then MsgBox "Aarrcchhhh, I told you to first call Init!", vbCritical
   
    CountOfDeclarationLines = theModule.CountOfDeclarationLines
End Property

Public Sub myMethod()
    'Do your own thing.
End Sub

Now I can use this class from the rest of my code:

Sub test()
    Dim obj As New myModule
    obj.Init Application.Modules(0)
   
    Dim app As Access.Application
    Set app = obj.Application
    Debug.Print app.Name
   
    Debug.Print obj.CountOfDeclarationLines
   
    Debug.Print obj.myMethod()
   
    Set app = Nothing
    Set obj = Nothing
End Sub

As you can see I solved the problem of "no inheritance" by using an Init method that the user is required to call before using the rest of the object.
