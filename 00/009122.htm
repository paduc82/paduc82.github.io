Purging Deleted Mail Messages in Windows 10 Outlook.com - Microsoft Community

c j 15 say:

Purging Deleted Mail Messages in Windows 10 Outlook.com

Is there a way to mass purge my deleted messages from the "Recover Deleted Messages" screen.  I want the older messages permanently gone, not taking up space anymore.  Only way I can find so far is check-marking the messages one by one by one...sooo time-consuming.

ZigZag answered my question by asking which system I'm using -- I replied I am using Outlook.com, but didn't hear any further instructions on how to proceed.  Thanks for any help.

toby ornott say:

Once you enter the Recover deleted items's "dumpster module" you can attempt to search By date; thus, that may be your best first approach at reducing some of the volume. 

I mention this only because I have no knowledge of how large your dumpster module is; it could contain months of messages, perhaps hundreds to thousands, or it could be much smaller.  Nevertheless the following will work on only smaller volumes and eliminate the process of one-at-a-time:

  1. Go to the first message in the list and mark the box to the left of the message
  2. Carefully scroll through approximately 21 messages (*See Note) and hold down the SHIFT KEY and simultaneously mark the 21st message's box.  Ascertain that all 21 messages from 1 to 21 have been then "marked" by the search engine
  3. If all appears satisfactory, then click Purge

(*Note: Unfortunately this is a tedious process if you do not purge your Deleted items on a regular basis.  I have been successful in purging only a maximum of 31 messages "en mass" (often with annoying failures).  So the above is why I suggest only 21 messages at any one time until you can get more experienced with the process.  When Outlook Mail.com first appeared, my Deleted items messages grew exponentially until I learned the above process.  Eventually I managed to deleted 10s of 100s of Junk spam garbage over a period of half-a-year.  Now my "Recover deleted items" software engine returns only "small, daily amounts", and that garbage is obliterated via the above process in less than a minute.)

So i guess you would call that merge and link? If that is even possible in powerpoint like it is in Excel.

steve rindsberg say:

There's no built-in feature for this.

In theory, you could "merge" the second one into the first one like so:

Open One and Two.

One should be in Normal view, Two should be in Slide Sorter view.

For each slide in Two, add a new blank slide in One.

Select a slide in Two, press Ctrl+C.

Go to the new blank slide in One, choose Paste Special, put a check next to Link.

Resize the newly pasted, linked slide image to fill the slide.

Repeat for each slide in Two.

The linked slides in One will now update in response to changes in Two.  

Whether it's worth the effort ... that's up to you.  ;-)

The other alternative is simply to hyperlink from One to Two; that's the approach I'd suggest if this is for presentations that'll be displayed on a screen of some sort.  It won't help if you need to print the presentations.

Duration Format - minutes.seconds,1/100th - Microsoft Community

georgelawrie say:

Duration Format - minutes.seconds,1/100th

Is it possible to format cell to accept race durations in minutes, seconds.1/100th?
joeu2004 say: George wrote:
Is it possible to format cell to accept race durations in minutes, seconds.1/100th?

It is possible to display time accurate to 1/100th second -- even 1/1000th second.  For 1/100th second, use the Custom format [m]:ss.00.

And if you always enter time in the form m:ss.ff -- that is, with explicit fractional seconds, even if zero -- Excel will correctly interpret that as min, sec and fractional seconds, although you still need to explicitly set the Custom format above.

The gotcha is:  if you enter time in the form m:ss -- no fractional seconds -- Excel will interpret that as hr and min, regardless of the cell format.

So it is more reliable to always enter time in the form h:mm[:ss[.ff]], with a leading 0: for hr.  (The square brackets are a metasyntax indicating optional input; you do not actually type them.)  Apparently, even in that form, the min part can be greater than 59 (!).

Execute a Public Function from a control on a form - Microsoft Community

lonely in ivory tower_y say:

Execute a Public Function from a control on a form

Hi, I am beginning to delve a little more into VBA in my Access database. I made an event procedure on the onClick of a form's control.  I realized I would be needing this same code on many forms throughout my database, so I moved it from a Private Sub to a Public Function in a module. 

What is the correct way to execute this public function from the onClick command of the form's control? To make it work, I created an event procedure ( a private sub) for the form's control that says Call nameOfPublicFunction.  That seems like a roundabout way to get there. I tried putting nameOfPublicFunction() directly in the onclick property of the control, but that didn't work.
john w. vinson say:

You can execute the function by putting it directly in the OnClick property - preceded by an = sign:

 

=NameOfPublicFunction()

 

If the function can be called from multiple forms, I'll often pass the form reference:

 

Public Function NameOfPublicFunction(frm As Form) As Integer

...

 

and set the event procedure to

 

=NameOfPublicFunction([Form])

 

You can then refer to other controls or properties of the form within the procedure:

 

If frm.Name = "SpecialForm" Then...

 

 

The = sign apparently tells the VBA engine that you're expecting a value to be returned; that's why it must be a Public Function (which returns a value), not a Public Sub (which doesn't). So far as I've been able to determine, you can't do anything with the returned value, so even though you could assign NameOfPublicFunction a value in the code, there doesn't seem to be anything you can do with it!
