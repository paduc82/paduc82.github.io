Ooops  :) sorry - It looks like I somehow mixed your post up with one using a Phenom 945.
Your cpu can run RAM with a max of 1866Mhz, but your mobo still has a max of 1333Mhz
if the system is not OC'd.
The 3.6Ghz is the cpu's 'turbo' speed (self overclocking). If the cpu speed is always at 3.6Ghz it
may be due to the RAM being set at 1600Mhz. If you set the RAM to default in BIOS it will not
stop the cpu from using turbo mode when it needs extra grunt.
It is also possible (I have not used a cpu with turbo, so only speculating) that the cpu OC's the RAM to 1600Mhz  when in turbo mode.
This could be verified if the RAM speed is set to default (which would be 1066 or 1333Mhz)
 in BIOS.
The mini dumps don't have anything conclusive. Only that there is a memory management issue. They suggest a possible driver conflict but can't identify the driver.
One possible such driver comes with Daemon tools. If you have Daemon tools installed then
the link below shows how to get rid of the driver.

BSOD at high memory usage, fltmgr.sys - Windows 7 Forums

A msinfo32 report is generally suggested to be included with the mini dump, and may
reveal something more.
