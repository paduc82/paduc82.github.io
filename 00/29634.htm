•Ntuser.dat
•Ntuser.dat.log
•Ntuser.ini

What is the version of Internet explorer you are using ?

Method 1: Try printing a webpage running Internet Explorer as an Administrator.

 

To start Internet Explorer as an Administrator, follow these steps:

 

a)    Close all Internet Explorer windows.

b)    Click Start, click All Programs, right-click Internet Explorer and select Run As Administrator.

c)    Browse to a website and try test printing a page while running as the Administrator.

 

If the issue persists, try next method.

 

Method 2: Try printing after disabling Protected Mode.

 

Try printing a webpage after disabling Protected Mode, to do this, follow these steps:

 

a)    Click the Tools icon in Internet Explorer

b)    Click the Security tab and uncheck the checkbox beside Enable Protected Mode (requires restarting Internet Explorer)

c)    Click Apply, and then click OK.

d)    Close all open Internet Explorer windows, and then restart Internet Explorer.

e)    Browse to a website and try test printing a page while running as the Administrator.

 

If the issue persists, try next method.

 

Method 3:


Set IE as default browser.

 

Follow the steps for the same:

 

a. From the Start screen, type Default Programs, and select the Apps filter.
b. Click Default Programs in the search results.
c. Click Set your Default Programs.
d. Choose Internet Explorer from the list.
e. Select Set this program as default, and then click OK.

mal_long say:

I did get a Win7 print driver to load in Win10.  Unfortunately a similar approach with a graphics driver that loaded in Win7 didn't get it to load.  It still failed with driver incompatible with OS level.

Below is what I documented back to Konica Minolta:

I thought it worthwhile to update you how I got the above working so you could maybe update your website and techies for any other people trying to get a BizHub 131, or indeed other devices, active under Win10.  Apologies up front for any techy-speak that is n/a as far as you are concerned!

 

I used the win7 GDI download: KM190fGDIWinx64_189mu, unzipped it all, and ran the setup.exe, both as is and then having set the compatibility to run as Win7 (running a delete option to clear things from previous run(s) each time).  Although it ‘ran’, and the BizHub 131f was there under devices and printers and Device Manager said, ‘This device is working properly’, a test print failed and it appeared there wasn’t an actual driver to allow processing.  Checking via a  Command Prompt (right click & select run as Administrator) input of ‘pnputil  –e’ which lists all drivers as OEM files, there was nothing for KM there.

 

So, I thought I would try and force a driver update direct from the folder containing the unzipped info above. I went via

Device Manager:

1.       Print queues

2.       right clicked the BizHub entry under Print Queues

3.       selected ‘Update Driver Software’

4.       ‘Browse my computer for driver software’

5.       and selected the k131fn.ini file in the KM131f folder within the English folder of the above unzipped folder

6.       Then hit Oks until it started to run it.

 

Technically I did extra steps after 4 above when I selected:

·         Let me pick from a list of device drivers on my computer

·         Select the device driver you want to install for this hardware

·         Then selected Have Disk…. button (effectively ignoring whatever was listed anyhow) before going via Browse (again) to select the INF file.  I assume it wouldn’t actually make any difference.

 

Anyhow, I got a ‘Windows encountered a problem installing the driver software for your device’ window pop up which also said, ‘The third party INF does not contain digital signature information’.  Whoopee, I thought, I know how I can force things round that as it was likely occurring before during the running of the Setup.exe but was hidden from being reported back.

 

So,  clicked on bottom left Windows icon, selected ‘Settings’, then ‘Update & Security’, then ‘Recovery’, then ‘Restart now’ button under ‘Advanced start-up’, which restarts & allows you to select ‘Troubleshooting’, ‘Advanced options’, ‘Startup settings’.  Within ‘Startup settings’ you hit the equivalent F key for the numbered options.  Hit F7 to select ‘Disable driver signature enforcement’.

 

Then, once loaded & back signed on, repeated the above to re-run the driver update and, heh presto, it all installed and I could do a Test Print as well as a test print from a Word doc’t.  Solved!

 

Now, maybe the compatibility setting can be ignored, but having got the driver installed I wasn’t going to remove/uninstall and try again to find out!  Next time I’m in I’ll also check to see whether the OEM file is there in a ‘pnputil  –e’ output as I was pushed for time & had to leave.  There is a new Win10 laptop that will need to access the printer (i.e. I’ll be setting it as shared) & hence will need the 64bit driver too, so maybe that will give me an opportunity to troubleshoot some more.  It will be interesting to see when I share the printer whether it copies the driver from the desktop or I have to re-run the above.
