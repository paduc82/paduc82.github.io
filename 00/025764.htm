sky pei msft say:

 

Hi Dave,

 

Not Any header. Please see the screenshot of my third post, I insert the third header which has a [Document Title] in it. It matches the title column in SharePoint Online. So if you give a value to the third header of the document and upload it, you can also find the value in the title column in SharePoint Online. The search result of the title will return the value of the title column prior to the name column. It is by design.

 

Obviously, if you use the first and second header in Word, the value of the header will not render in the title column in SharePoint Online. So search result of the document will return the name of the Word file, as there is no value in the title column.

 

Text blocks are not mapped to SharePoint Online. In this case, it is the Word Properties [Document title] mapped to the title column in SharePoint Online. This article introduces some Word Properties which can map to SharePoint Online column. You can have a look.

 

If you set the value of the title column as none or as same as the value in name column. The search result will return filename as title.

 

In this case, I would suggest you do a test on your side using different headers to see the behavior. (e.g. the first header and the third header)
