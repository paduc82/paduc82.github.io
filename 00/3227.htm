bernie deitrick say: To preserve your formulas, change this

 .Range(.Range("K2"), .Cells(.Rows.Count, "K").End(xlUp)).EntireRow.SpecialCells(xlCellTypeVisible).Copy _
                Worksheets("Sheet2").Cells(Rows.Count, "A").End(xlUp)(2).EntireRow

to

 Intersect(.Range(.Range("K2"), .Cells(.Rows.Count, "K").End(xlUp)).EntireRow,.Range("A:K")).SpecialCells(xlCellTypeVisible).Copy _
                Worksheets("Sheet2").Cells(Rows.Count, "A").End(xlUp)(2)

To remove duplicates on Sheet2:

Worksheets("Sheet2").Range("A1").CurrentRegion.RemoveDuplicates Columns:=Array(4, 5, 6, 7), _
        Header:=xlYes

This assumes that your data starts in A1 and is contiguous, and your removal is based on D,E,F, and G.

How to set part-time resources

I need to set a part-time time resource (let's call him Jim, working from 8 AM to 12 AM, 50%) in Project 2007. I was wondering which of the following mehods is best:

1) to set a new calendar (8-12 AM from Monday to Friday) and to assign it to Joe;
2) to set Joe availability to 50%

I thought the former is the best choice, but when I assigned Joe a 16-hour work activity, duration was set 2 days! (I expected 4 days)! Using the second method, duration was automatically set to 4 days !

john - project say:
veki_lo,
Personally I think the second method is better. If the resource is a true part-time employee, it makes sense to specify a part-time calendar but most often a resource is full-time and simply works on various things. In this case I might assume the Joe works on this task 50% of the time and another task (or tasks) 50% of the time. He therefore has a normal calendar, with perhaps some exceptions for vacation, and his assignment level is spread across multiple tasks.

Project is neither intuitive nor straightforward. That's probably why we field a lot of questions like yours. Project has a very specific definition of duration and it is partly defined by the hours-per-day setting in the options. True, there is no specific field that shows elapsed duration but the user can customize one or more fields to do so. For example, if you want to show the "true" duration in working time, you can use the following formula in a text field (e.g. Text1):
format(ProjDateDiff([Start],[Finish])/480,"#0.0#") & " d"

If you want to show "true" elapsed time, which is based on a 24 hour day, use the following formula:
format(DateDiff("n",[Start],[Finish])/1440,"#0.0#") & " d"

Pressing Ctrl+m should not open the VBA window unless you have a break set (there is a purple dot in the left margin of the VBA code window. 

If you have accidently set break mode you can unset it by 1. clicking the purple dots in the margin (its just a toggle).  2. Or I suppose you could close and reopen the file.

Ctrl+m was just my suggestion for a shortcut key.  Here is how you assign a shortcut key to a macro

1. Activate the spreadsheet, not the VBA window

2. Choose View tab, Macros, View Macros.

3. Select, but don't run, your macro, click the Option button and enter a letter in the box beside Ctrl+

4. Click Cancel (its not going to be lost)

Regarding assigning m, you can really assign any lower or upper case letter.  However, all but 3 lower case letters are used by Microsoft (and some of the upper case letters also).  If you assign something like Ctrl+c your macro will run not Microsoft's copy command.  e, m and q are unassigned lower case letter at least if you are in 2010.  (unless Microsoft has sneeked one in I don't know about.)  Now the uppercase letter are a different matter, almost no one knows or uses the assigned ones so choose them is fine.  But it does mean that to run them you will need to press Shift+Ctrl+ the letter.
