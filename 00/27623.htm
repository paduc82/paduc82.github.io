ron coderre say:

NOTE: I scrapped the awful code I originally posted and replaced it with this much cleaner code, which actually DOES loop through subfolders:

This VBA code will construct a list in a worksheet of the IE Favorites for the designated user.

• Select the workbook that will contain this code
• Right-click the workbook name in the VBA-Project window
...Insert.Module
• Tools.References
...Check: Microsoft Scripting Runtime
...Click: OK
• Copy the all of the below code and paste it into that module

Dim iCtr As Integer

Sub BuildFavoritesList()

    'Note: Replace "YOUR_USER" with your actual user name, adjust the path, if necessary
    ListFolderFavorites _
        strPath:="C:\Documents and Settings\YOUR_USER\Favorites\", _
        IncludeSubFolders:=True, _
        StartNew:=True
End Sub

Sub ListFolderFavorites(strPath As String, IncludeSubFolders As Boolean, StartNew As Boolean)
Dim fso As FileSystemObject
Dim fStartFolder As Folder
Dim fFldr As Folder
Dim fFile As File
Dim strText As String

Dim fOpenFile As Variant
Dim iURL_Start As Integer
Dim iURL_End As Integer
Dim iURL_Len As Integer
Dim strBullpen As String

    On Error Resume Next
    If StartNew = True Then
        iCtr = 0
    End If

    Set fso = New Scripting.FileSystemObject
    Set fStartFolder = fso.GetFolder(strPath)
   
    iCtr = iCtr + 1
    For Each fFile In fStartFolder.Files
       
        strText = fFile.Name
        Set fOpenFile = fso.OpenTextFile( _
            Filename:=fFile.Path, _
            IOMode:=ForReading, Create:=False, Format:=TristateFalse)
           
        strBullpen = fOpenFile.ReadAll
        iURL_Start = InStr(1, strBullpen, "BASEURL=") + 8
        iURL_End = InStr(iURL_Start, strBullpen, Chr(10)) - 1
        iURL_Len = iURL_End - iURL_Start
       
        If iURL_Len > 0 Then
            With ActiveSheet
                .Cells(iCtr, 1) = fStartFolder.Path
               
                strText = Mid(strBullpen, iURL_Start, iURL_Len)
                If Left(strText, 4) = "http" Then
                    ActiveSheet.Cells(iCtr, 2) = Mid(strBullpen, iURL_Start, iURL_Len)
                    iCtr = iCtr + 1
                End If
            End With
        End If
    Next
    If IncludeSubFolders Then
        For Each fFldr In fStartFolder.SubFolders
            Call ListFolderFavorites( _
                strPath:=fFldr.Path, _
                IncludeSubFolders:=True, _
                StartNew:=False)
        Next
    End If
End Sub


To run that code:
• Select a worksheet cell with plenty of blank space beneath it
• ALT+F8...to see the list of available macros
...Select: BuildFavoritesList
...Click: Run

john w. vinson say:

Think vertically not horizontally.

The StDev function in Access - like Sum and other aggregate functions - works DOWN a list of records. If you have a Query or a Table with a Usage field, and the query criteria limit the query to the twelve months of interest, you can simply use a Totals query with

StDev([Usage])

to calculate the result.

Hi Christopher,

If we just want to sync OneDrive for Business library with the new OneDrive sync client, we don't need to run TeamSiteSyncPreview.reg which is used to enable SharePoint Online sync for Windows.

If we want to use the new OneDrive sync client to sync SharePoint Online libraries, we need to enable SharePoint Online sync for your Windows or Macs manually. This is still a new feature, you can try it on your PC and then deploy it to other users in your organization. On Windows, you need to run TeamSiteSyncPreview.reg to enable SharePoint Online sync.

For this article :Get started syncing SharePoint sites with the new OneDrive sync client, it says, in the Before you begin section, "Ask your admin to test out syncing SharePoint sites and enable it for you. "

Note: The registry keys don't work for previewing the feature using Internet Explorer on Windows 7, or for SharePoint sites that use the classic experience.

 
If you wouldn't like to run TeamSiteSyncPreview.reg, you may keep using the Groove(the legacy OneDrive for Business sync client) to sync SharePoint Online libraries.
