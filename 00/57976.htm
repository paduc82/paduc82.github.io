Instructions (Found online modified by me)

Create a .bat file named "addfwrs.bat" without quotations using the text in italics then follow the further instructions :

@echo off
REM      BATCH FILE CREATED BY CHARLES DE HAVILLAND 20/02/2012
cls
If "%1"=="" GOTO :norulename
SET RULENAME=%1
ECHO Create in/out firewall rules for all *.exe files with the rulename of "%RULENAME%" ?
ECHO.
ECHO.

pause
Echo.
FOR /r %%G in ("*.exe") Do (@echo %%G
NETSH advfirewall firewall add rule name="%RULENAME%-%%~nxG" dir=in program="%%G" action="block" enable="yes")
FOR /r %%G in ("*.exe") Do (@echo %%G
NETSH advfirewall firewall add rule name="%RULENAME%-%%~nxG" dir=out program="%%G" action="block" enable="yes")
Echo.
Echo done.
Echo.
GOTO :Finish
:norulename
Echo Error! - You did not specify a Rulename type - Addfwrs "Rulename"
Echo.
:Finish
Echo Batch ended...

Further Instructions:

Windows 7 firewall actually works very well, but it's not easy to configure with multiple programs (very time consuming).
 
I looked for ages trying to find a way of blocking multiple .exe files through Windows 7 in one go; it seemed the only possible method was to create my own batch file (windows script). 
I have added it below for anyone to use / modify for their own purposes.
 
 
Running the batch file below (addfwrs.bat), will create inbound AND outbound firewall rules that block all .exe files within your chosen folder and subfolders from accessing the internet. 
Very useful.
 
To use:

(1) Place addfwrs.bat in the folder in which you want to block the .exe programs (It will block all exe programs in any sub-folders so you can place it in a folder without any exes so it blocks
exe programs in multiple sub-folders at once)

(2) Now run CMD as administrator.

(3) Copy the  new directory (eg. c:\Programs Files (x86)\YourProg\) of where you put the addfwrs.bat file

(4) Paste the directory into CMD with "cd" and a space before the directory (example: cd c:\Programs Files (x86)\YourProg\) and press enter.

(5) Now run the batch file by typing: addfwrs.bat "rulename"   example: addfwrs "BLOCK_PROGRAM_NAME_HERE"
(Make sure there are no spaces in the rulename of the firewall rule, so use underscores as spaces.

N.B: It's important you add a firewall rulename, so that you can identify the rules when added to the firewall at a later date.

(6) Follow the on screen information, it will advise you which rules it is creating.

(7) Check Windows Firewall for the new rules listed and you can then remove the addfwrs.bat file from where you put it (It is no longer needed although you can keep it somewhere for later).

All done.  Hope you find it useful, I certainly have!
Enjoy!

N.B: View the new rules contained within your firewall (In/Out rules), change as and when you need to.

Changing the motherboard will invalidate the license, requiring that you purchase an entirely new Windows 10 license.

You can only reactivate if the original Windows 8 license is a full retail license, if not, then you need to purchase a new license.

When I upgrade a preinstalled (OEM) or retail version of Windows 7 or Windows 8/8.1 license to Windows 10, does that license remain OEM or become a retail license?

If you upgrade from a OEM or retail version of Windows 7 or Windows 8/8.1 to the free Windows 10 upgrade this summer, the license is consumed into it. Because the free upgrade is derived from the base qualifying license, Windows 10 will carry that licensing too.

If you upgrade from a retail version, it carries the rights of a retail version.

If you upgrade from a OEM version, it carries the rights of a OEM version.

Full version (Retail):

- Includes transfer rights to another computer.

- Doesn't require a previous qualifying version of Windows.

- Expensive

Upgrade version (Retail):

- Includes transfer rights to another computer.

- require a previous qualifying version of Windows.

- Expensive, but cheaper than full version

OEM :

OEM versions of Windows are identical to Full License Retail versions except for the following:

- OEM versions do not offer any free Microsoft direct support from Microsoft support personnel

- OEM licenses are tied to the very first computer you install and activate it on

- OEM versions allow all hardware upgrades except for an upgrade to a different model motherboard

- OEM versions cannot be used to directly upgrade from an older Windows operating system

What happens if I change my motherboard?

As it pertains to the OEM license this will invalidate the Windows 10 upgrade license because it will no longer have a previous base qualifying license which is required for the free upgrade. You will then have to purchase a full retail Windows 10 license. If the base qualifying license (Windows 7 or Windows 8.1) was a full retail version, then yes, you can transfer it.

b. Stand-alone software. reed1e say:

Hi Rebecca,

Yes, Microsoft seems to have abandoned WLM support to the users.

The problem is a fault in the WLM coding which can't handle long names. If WLM is shut down before it's had chance to rebuild its database index (eg power failures or forced shut downs due to windows freezing etc) it creates recovery folders with very deep folder trees. These deeply nested folder structures are part of the naming used by the database indexing which it can't handle. This causes the problem ~ or at least it caused the problem I had.

The way of fixing it is to shorten the folder names all the way down the trees (and there are lots of them). Just as one example the tree was: Storage Folders / Recovered items / 01-03-14 823/Storage Fol d5/Imported Folder / Storage Folders/Imported Folders/Local Folders/Imported Folders /filed/...then at this level I found some of my folders, which had their own trees that I'd created.

My brother wrote a script to shorten the file names automatically, but he was able to test that with my specific configuration in a safe way.  It would probably be too risky to try it with your account.

I suggest you just plough through these folder structures and abbreviate all the folder names until you get to ones you recognise. Mine now read SF instead of Storage Folders, IF for imported folders etc.

I recommend you make a back up copy of your Windows Live Mail folder first. In case you don't know you can find where the folder is:

* click the icon just above 'Email message' and below the very top bar (when you hover it says Windows Live Mail. Click here to save, print and see everything else you can do)

* select Options / Mail / Advanced / Maintenance / Store Folder. That will bring up a window showing the current location.  I moved mine to the Root (ie C:\Windows Live Mail) to make it easier to get at.

It's also a good idea to force the index to rebuild before and after your changes go Options / Mail / Advanced / Maintenance / Compact the database on shutdown every <change this to 1> runs.  Shutdown WLM, it will rebuild the index. Then restart and change the number to say 10 until you're ready to have another go at name shortening, then repeat.

Assuming your problem is the same as mine when you've done all this you will find your emails. You may need then to spend a bit of time sorting them out but at lease you'll have them.

Good Luck
