john mcghie say:
Hi Matt:

Go back to Format>Style>Normal and uncheck "Automatically update" and check "Add to template" so you store the change for future documents.

If you apply any formatting to a paragraph formatted with a style that has "Automatically update" applied, the change flows through to everywhere the style is used.  In the case of Normal, that's the whole document.

And no, "re-installing" would not work, because this setting is not on the DVD to be installed.  But creating a new document would :-)

hi philipthomas,

based on your description, i did a test from my side and i reproduced the same result.
according to  this article, we don’t need to specify a value with this switch, which means we cannot use powershell to set the parameter “autosubscribenewmembers” to false.


as a workaround, we can let office 365 group owner disable the parameter “autosubscribenewmembers” in owa  (outlook web app).  here are the steps:
1. login to owa as the group owner.
2. select the office 365 group >>edit the group>> uncheck “subscribe new members so they receive group conversations in their inbox”.

3. save.
4. run the cmdlet below via powershell:
get-unifiedgroup –identity “office-xxx” | select autosubscribenewmembers

the parameter will be showed as false.

please note we need to connect to exchange online using remote powershell powershell before running the above cmdlets.

Have you tried Microsoft's  Windows 10 Store Apps Troubleshooter  ??    It may be that after using these (one after another) the Power Shell commands were the final process necessary to complete the task.

PowerShell -ExecutionPolicy Unrestricted -Command "& {$manifest = (Get-AppxPackage Microsoft.WindowsStore).InstallLocation + '\AppxManifest.xml' ; Add-AppxPackage -DisableDevelopmentMode -Register $manifest}"

Get-AppXPackage | Foreach {Add-AppxPackage -DisableDevelopmentMode -Register "$($_.InstallLocation)\AppXManifest.xml"}

Get-AppXPackage *WindowsStore* -AllUsers | Foreach {Add-AppxPackage -DisableDevelopmentMode -Register "$($_.InstallLocation)\AppXManifest.xml"}
