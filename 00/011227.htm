password managers? - Microsoft Community

jon straub say:

password managers?

symantec and mcafee offer password managers[you use one pass word,and the paid service creates unique logins for every site you use] does mse offer this service? in light of the password breach at linkedIn I am very concerned
kosh vorlon say:

Hi Jon,

 

The short answer to the follow-up question is, IMHO, you don't need it.

 

I do not personally use any password manager program (and never have), and I do not recommend them to my clients.

 

First, the one's you were originally considering (Norton,...) come bundled with suites that include real-time AV protection products and you could not install them and continue to use MSE - it would be a one-or-the-other choice - and you already have the best AV program out there and it's free. 

 

Second, while Mike does point to some good free alternatives, and for some people they are helpful, they do not in and of themselves offer more protection of they type you are talking about.  They simply automate how it is done so that you can typically use stronger and more complex passwords which are all different without forgetting them all (as well as the other benefits he mentioned). 

 

Third, they represent a single point of failure for all your passwords.  Not that this is likely, but it is possible that the password database could become corrupted and then every password being managed could potentially become a problem to restore as you probably will not know the real passwords that were used or be able to remember them.

 

Fourth, if your system somehow gets hacked, if someone can also hack into the password manager program, they'll have access to all your passwords everywhere.  Yes, they are saved in a way that makes doing that extrememly difficult, but not as difficult as not having them stored on your computer in the first place.  There are ways to use your browser to save online passwords (IE calls it AutoComplete) but I don't generally recommend that either for the same reason - the information is there on your computer - and there that extra layer of protecting the information also doesn't exist (and in this case, they are potentially accessible through the website depending on how you've configured browser security).

 

In short, it's a tool to be able to use different and complex passwords everywhere (which is the real benefit).  It does not really "protect" the passwords other than making them different and making each harder to crack by using versions most people would otherwise never use on their own.  It may be helpful if you can't find a way to do that and remember them on your own without needing to store them on the computer at all - which is safer no matter how well protected it is from being hacked in the event the computer is stolen or otherwise.

 

If you ARE using the browser to do this (or, heaven forbid, a normal document - online or sitting on your desk - with a list of all of them), then such a manager is a better alternative.  If you are otherwise using strong passwords and can remember them without such a tool, that is probably the best.  Whether this will increase or decrease your security really depends on what you do now in terms of passwords.  For some it will help.  For others it will not help.   The choice is yours.

 

I hope this helps.

 

Good luck!

 

Kosh

dale howard [mvp] say:

Michael G --

When you initially assign a resource to a task, assuming that the resource has a Standard Rate value set, Microsoft Project will calculate the cost for the task for the resource, and for the resource assignment on the task.  When you capture your baseline for the project, Microsoft Project will capture the current Cost value for every task, resource, and assignment in the project, along with a number of other important values as well.

After you baseline your project, suppose that at a later date you end up changing a resource on a task.  And suppose the new resource has a higher Standard Rate than the original resource.  Microsoft Project will do the following in response:

  • Calculate a new Cost value for the task and show the additional cost in the Cost Variance column.  You can see this if you display the Task Usage view and then apply the Cost table.
  • Calculate the Cost value for the new resource assignment and show the additional cost for the new resource in the Cost Variance column.  Again you can see this if you display the Task Usage view and then apply the Cost table.
  • Calculate a new Cost value for both the original resource and the new resource and show the cost differences for each resource in the Cost Variance column.  You can see this if you display the Resource Usage view and then apply the Cost table.

In the second bullet point above, you should note that the resource assignment information for the original resource assigned to the task is lost when you change resources.  This means that when you are going to change resources on a task, and the Standard Rates are not the same, you might want to add a task note to explain the Cost Variance information you will see in task views.  This is simply a limitation, and a sad limitation I might add, in how Microsoft Project handles baseline information for resources and resource assignments.

joeu2004 say:

Array-enter (press ctrl+shift+Enter instead of just Enter) the following formula:

=IFERROR(INDEX(A1:Z1, MATCH(TRUE, IF(ISNUMBER(A1:Z1), A1:Z1<>0), 0)), "")

It should appear with surrounds curly braces ("{=...}") in the Formula Bar.  But we cannot type the curly braces manually.

Note that zero is a number.  But you made it clear that you want to ignore zeros.

IFERROR returns the null string ("") if there is no non-zero number in the range.

Use a limited range like A1:Z1, as demonstrated.  Do not use a whole-row range like 1:1.
