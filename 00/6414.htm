vijay a. verma say:
It works like this -

When you start Excel, R1C1 or A1 reference style is set by first file opened. If first file opened has R1C1 set, then all other subsequent files will be opened in R1C1 mode only.

1. First file opened is always PERSONAL.xlsb if it exists. If it exists, then every subsequent file opened will have same style which PERSONAL.xlsb has. In your case, this is ruled out as
1a - You have manually looked and couldn't find it.
1b - Had it been existing, all subsequent files would have had the same style. But, it occurs only few times.

2. Hence, culprit is that your friend is opening a file first time which has R1C1 set up when he starts the Excel. If he opens a file first which doesn't have R1C1 set up, subsequent files will not be opened in R1C1. Hence, he will have to close and restart the Excel and open a file which is set in A1 reference style.

Ask your friend to experiment keeping above facts in mind and you should be able to reach to a conclusion.

jeeped say:
I had another look at your problem and I think I might understand it better now. You want to find the sequential and simultaneous growth for 65 and 100 where their percentage now equals 70 instead of the original 65%.

Here is one way to do it.

        

A2 has 65 typed in. B2 has 100 typed int. C2 uses the formula =A2/B2 formatted as a Percentage. C3 has 0.70 typed in and formatted as a Percentage. A3 has the formula,

=MIN(INDEX((((A2+ROW(1:99))/(B2+ROW(1:99))>=C3)*ROW(1:99))+((A2+ROW(1:99))/(B2+ROW(1:99))<C3)*1E+99,,))+A2

B3 has the formula,

=MIN(INDEX((((A2+ROW(1:99))/(B2+ROW(1:99))>=C3)*ROW(1:99))+((A2+ROW(1:99))/(B2+ROW(1:99))<C3)*1E+99,,))+B2

These last two are standard formulas, not array formulas.

jeeped say:
While the formula I supplied is considered a standard formula due to the fact that it does not require Ctrl+Shift+Enter, it does in fact produce array or cyclic processing within the INDEX() function. I had minimized the amount of processing required by paring down the numbers tested. ROW(1:99) is used to represent 1, 2, 3, 4, ... 97, 98, 99 in the cyclic processing. Raising the upper limit of this to ROW(1:999) is sufficient to rectify the formula's shortcomings for larger numbers.

=MIN(INDEX((((A2+ROW(1:999))/(B2+ROW(1:999))>=C3)*ROW(1:999))+((A2+ROW(1:999))/(B2+ROW(1:999))<C3)*1E+99,,))+A2
  ... and,
=MIN(INDEX((((A2+ROW(1:999))/(B2+ROW(1:999))>=C3)*ROW(1:999))+((A2+ROW(1:999))/(B2+ROW(1:999))<C3)*1E+99,,))+B2

That supplies 584 and 834 for 70% with starting points of 250, 500 and 50%. You could raise it to ROW(1:9999) or higher if the numbers got larger still. Remember that all numbers are always processing; finding the correct answer does not halt processing. ROW(1:99) produces at least 100 calculation cycles, ROW(1:999) produces 1000, and so on. Keeping this to a minimum is advised.
