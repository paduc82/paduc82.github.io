I don't work for nor speak for Microsoft however a couple of points.

1. Word is not a  page layout application..f you need a page layout application, then Word is not it.
2. This behaviour is normal and by design and has been present since at least Word 97.
3. Documents are assembled from a number of 'layers' - imagine a pile of transparencies each containing some element of the document, and contributing to the whole. Floating images are not in the text layer of the document - unlike in-line images.
4. Print layout view is not WYSIWYG. It is merely an approximation of the layout. The only true WYSIWYG view is Print Preview and it is there that you should make your final layout checks.
5. If speed of editing is an issue then the usual workaround is to work in draft view to get the text on the page, and then finish off in Print Layout view.
6. I cannot imagine what a macro would do to hide a floating image that would not equally impact on speed as having the image displayed. Macros of this type usually set the brightness to high, essentially making the image all white. The image is still nevertheless present and complete.

Thank you.  That is how I have been doing it for years.  Today I was wondering if it can be done with an ActiveCell statement. Maybe it can't be done with ActiveCell?  


Hi,

 

No you can't do that. A user defined function has exactly the same limitations as a worksheet function in that it cannot directly alter any cell other than the cell the function is in. For example and the case of your UDF, say it was in called from B1 and the active cell was A1, it cannot alter that cell.

 

If the call to your function was in B1 and the active cell was also B1 you would get a circular reference.

 

So in short a UDF can only return a value to the cell it was called from and cannot alter the format or content of any cell.

brian tillman [outlook mvp 2007-2019] say:

> t goes to the Sent Items folder without any errors, but the email is never delivered.

In most cases where this happens, the antivirus on the PC is interfering by scanning outgoing mail. When you install an AV and include the mail integration features, the AV program takes the place of the mail servers. Outlook talks to the scanner, thinking it's talking to the SMTP server and sends the message to it. The "server" acknowledges the receipt of the message so Outlook moves it from Outbox to Sent Items because it received verification that the message was accepted by the outbound server. That "server", though, fails to pass it to the real server and it never gets delivered.

It could also be caused by some other add-in

Start Outlook in safe mode, as Suvarna suggests - you can also do it by simply holding the Ctrl key when you click Outlook's icon to start it. If it works correctly, then you know it's an add-in. You can disable them all, stop and restart Outlook, then start enabling them one at a time until the error happens again. The most recently enabled add-in will be the culprit.
