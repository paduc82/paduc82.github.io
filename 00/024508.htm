Hi,

Thanks

daniel pineault say:

This is normal.  

Access is backwards compatible, therefore 2019 can run older versions (2016, 2013, 2010, ...), but isn't forwards compatible.  No software can be forward compatible as it doesn't know what is coming, what new feature will be added...  This is why the golden rule of development (and this apply to all technologies typically, not just Access) is to always do all your development in the oldest version of the software that will be used to run the file/application/database.

So say you have users running Access 2010, 2016 & 2019 you need to do all the development in Access 2010.

What you can do at this stage is create a new blank database in the oldest version of Access you have and try importing the various database objects.

scottgem say:
The 2019 file probably uses features of 2019 that are new to 2019. You cannot expect to be able to open files created in newer versions with older versions.

You will need to go back to whoever created the 2019 file and ask them to save it as an older format. That's the only way you may be able to open it.

As a rule of thumb people should develop in the lowest version their users will be using.

jay freedman say:
You'll need a macro that takes a list of the names of the 20 form fields, and sums the counts of characters in those fields' results.

It would be very difficult or impossible to make the macro display a running count as the user is entering data, because the kinds of events that are available to macros are quite limited. It could be set up to run whenever the document is saved, and pop up a message showing the total and an "exceeding the maximum" warning when appropriate. However, be aware that some users (or their companies) don't allow macros to run, and you can't force them to allow it.

The basis of the macro will be code like the following:

Sub Example()
    Dim fldList As String
    Dim fldListAry() As String
    Dim fld As String
    Dim idx As Long
    Dim totLen As Long
    Dim flds As FormFields
   
    ' Put the list of fields to count
    ' between the quotes, separated by commas
    fldList = "Text1,Text3"
   
    fldListAry = Split(fldList, ",")
    Set flds = ActiveDocument.FormFields
   
    For idx = 0 To UBound(fldListAry)
        fld = fldListAry(idx)
        totLen = totLen + Len(flds(fld).Result)
    Next
   
    MsgBox "Total length = " & totLen
End Sub

Assign your list of field names to the fldList variable as described in the comment.

Getting the macro to run whenever the document is saved -- by any of several user actions -- will take additional code.

This is not something that can be recorded by the macro recorder; it will have to be written from scratch. If you don't have the knowledge of VBA to do it, reply with the list of field names and someone here (if not me) will write it for you.
