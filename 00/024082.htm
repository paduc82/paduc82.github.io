don cuthbert say:

Since there are still some claims that you can still upgrade from 7 to 10 and activate:
One method that might work is to copy \Sources\gatherosstate.exe from the most recent public release of Windows 10 install media to a thumb drive.  From your activated Windows 7 install, run gatherosstate.exe from your thumb drive and it will create a small file named GenuineTicket.xml at the same location as the gatherosstate.exe is run from.

If an upgrade from Windows 7 to 10 would still activate Windows 10, then a clean install of Windows 10 may also activate using this file. After a clean install of Windows 10, copy the GenuineTicket.xml file to C:\ProgramData\Microsoft\Windows\ClipSVC\GenuineTicket\GenuineTicket.xml.  It should soon activate, or perhaps after a restart. If the XML file instead vanishes, without activation, give it another try and recopy the XML file to that location.

This was an old method that worked to activate clean installs of Windows 10, prior to the availability of Windows 10 version 1511 in Nov. 2015.  In the past, I occasionally had to use it to reactivate a previously activated Hyper-V VM, after a clean install of a new Windows 10 version.  More recently, I have not had similar VM activation issues.

  • The XML file is only of use on the activated Windows PC that it was created on.
  • I believe that this was the underlying method that the original Windows 7/8 to 10 upgrades used to preserve activation during an upgrade. However, it was done automatically during the upgrade.
  • Copying the XML file, instead of doing an upgrade, allowed for a much faster clean install to be done and still activate.

You could create an expandable VHDX file on your Windows 7 C: drive and clean-install Windows 10 to that (look up Native Boot for details). That would allow you to test (dual-boot) whether Windows 10 will activate, without wiping out the existing Windows 7 install.  If it doesn't activate, you can just delete the VHDX file and edit the BCD to eliminate the dual-boot menu and all will be back to normal Windows 7.

If an upgrade will no longer activate, then you are left with either running Windows 7, or purchasing Windows 10 Home or Pro, as was already said.

Search Results from the App Store need more differentiation in search results

Hello.

I enjoy using the start menu search functionality to find programs and files on my computer. What I don't like is how in Windows 10, Windows store apps that are not installed on my machine show up at the top in the same location as local files and programs. While there is a small (tiny) label underneath that says 'install app' that signifies this is not local content, it should not be in the same area as local content. It should be moved to the online results area or separated into its own area. The results above the 'line' in the start menu should only be objects that are local to the machine. Mixing these, even with (tiny) labels, leads to too much confusion and accidental launching. I have some programs that I run desktop versions of (such as Kindle), and when I search for them, I also get the Kindle app, which makes me stop and carefully consider which is the one I want as the icon is almost identical. This is bad design as it adds unnecessary confusion for the user. Please remove these entries from this 'local results' area and place them in their own designated area.
