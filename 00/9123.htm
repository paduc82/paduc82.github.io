As Olof mentioned default language of Sharepoint site cannot be changed, unless you practically rebuild your tenant. 

So I created a another test user in our local AD and added an attribute "preferredLanguage" for user with value "en-US".

When user is replicated to Azure/O365, this will add "English (United States)" as user default display language (you can find the setting in Delve -> Update Profile -> How can I change language and regional settings? -> Click here -> three ellipses -> Language and Region):

After this when you enable Known Folder Move to OneDrive, folders stay in English language.

I tested this also with old user (already existing in our AD with OneDrive enabled w/o KFM). Added attribute and waited for sync to Azure. Default language was changed in O365 and Known Folder Move was enabled and working with English language. Only thing which is not tested is that IF user already have KFM enabled and has other than English language folder names and you add the attribute, will it change folder names to English language.

By default, gridlines don't print, nor do the column and row headers.

Bring the Skype "Classic" Features to Windows 10 - Microsoft Community

ishaan awasthi say:

Bring the Skype "Classic" Features to Windows 10

Looks like the "classic" Skype users will be forced soon to switch completely to the Win 10 version. Not sure that it is decent replacement – I've been using Skype for more than 10 years and feeling lack of important features in this new version I’m used to use. Seems MS is trying to conquer the teens with this «restyling» – bad attempt because they have lots of more popular alternatives; but the «old» users’ loyalty is being lost rapidly.

Nevertheless there’s still a chance to keep the loyal users by bringing the most important and useful „classic’s“ Skype functions to the Win 10:

  1. No access to the old history! It’s possible to open the chat’s history from the beginning in the “classic” Skype while the “new” one allows scrolling up to the last several months’ messaging. And the navigation via the history is much more convenient – the same should be available in the “new” Skype.



  2. No ability to export the chats history. Starting from the Skype v.12 for the Windows 10 the chats history is stored in the “skype.db” file whereas the other versions keep it in the “main.db”. As a result, now it’s impossible no import the history just by replacing of the “main.db” file. And it would be very cool to be able to import the full chats history to all other devices where Skype is running e.g. via Dropbox, OneDrive, Skype for web etc.
  3. No contacts lists. I have hundreds of contacts and all of them are distributed by various lists together with the group chats related to the contacts in the list. It would be a disaster to get them as a single list.
  4. No in-chat search (like in the “classic” Skype). Irreplaceable feature when you know what and where to search. The “general” search returns too much rubbish so very difficult to find what you need.



  5. The renamed contacts are not searchable. I often rename the contacts and the “general” search doesn’t return them after that when typing either initial or changed contact’s name. Looks like a bug.
  6. Bring back the contact’s time. Was very useful feature when the contact’s time was displaying at the chat’s toolbar. As many of my “contacts” live and work in different time zones it was easy to find the best time to call or message them.

So Skype has several months to improve the situation. Otherwise it will share the destiny of the MSN messenger...
