joemillson say:

Project Online - Charge out rate AND pay rates per task

Hi all,

We are a professional services business. We pay our employees the 'pay rate', and we charge them out at Pay+Overhead+Profit, or the 'charge rate'

For our projects, we need to be able to analyse per task, and rolled up, the charge rates and the pay rates at the same time, and then be able to deduct those figures from a budget.

I've dabbled with using 2 standard rates over 2 cost rate tables, however I'm now stuck.

We would want this to be the set up for all our projects across the enterprise, so knowing how Project Online/Server will handle this will be useful.

dale howard [mvp] say:

joemillson --

I have been down this road before, and there is no simple way to address this need.  What I have done is to create three extra columns.  Name one of them Charge Rate, name the other Pay Rate, and name the third one Profit.  Create a formula in the Profit column that subtracts the Pay Rate column value from the Charge Rate column value.  Next, set your charge rate on Cost Rate Table A and set the employee pay rate on Cost Table B.

Next, insert the Cost Rate Table column in the Task Usage view and select Cost Rate Table A as the rate for every task.  Copy and paste the values in the Cost column to the Charge Rate column.  Change the values to Cost Rate Table B in the Cost Rate Table column.  Copy and paste the values in the Cost column to the Pay Rate column.  You should now be able to see the profit per task in the Profit column.

At the end of every reporting period, or bi-weekly, or monthly, you will want to repeat the process of setting the Cost Rate Table value for every task, then copying and pasting to the Charge Rate and Pay Rate columns to see the current state of the project.  I realize this is a bit of a laborious process, but it WILL work.  I have used this with clients previously, so I know from experience.  Hope this helps.

sky pei msft say:

Hi Martin,

I have gone through all the posts in this thread. I'd like to explain that it is not feasible to work around list view threshold limit by creating the view filtered by lookup index column. To make other community members who come across this thread find the information easily, I'd like to make a summary like below.

Symptom:

We will get the error "This view cannot be displayed because it exceeds the list view threshold (5000items) enforced by the administrator" when we filter a view using lookup column in a list which has more than 5000 items.

Cause:

Although we can index a lookup column to improve performance, using an indexed lookup column to prevent exceeding the List View Threshold does not work.

Resolution:

Use another type of column as the primary or secondary index.

suzanne s. barnhill say:

When you insert a Continuous section break, the columns are "balanced" on the page, as shown. You have a couple of options:

  1. If you enter a Next Page (or Odd Page/Even Page) break instead of Continuous, the columns won't be balanced.
  2. There is also a Layout Option (at the bottom of File | Options | Advanced), "Don't balance columns at the start of Continuous sections." (This is a per-document setting.)

Assuming you want to start a new page, (1) is what you want. I'm not sure when (2) would be appropriate for you, but if the columns aren't balanced, you won't be able to add anything more to the current page unless it's in the second column; anything below the section break would be on the next page anyway.

What remains true is that columns are a section property, so you have to insert some kind of section break when you need to change the number of columns.
