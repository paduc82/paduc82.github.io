wunders say:

What you can or cannot do with a file or folder is strictly spelled out by the Security Permissions that exist on this file or folder...  and yes, it is possible for you as administrator to remove all access to a file or folder such that even an administrator cannot access it.  Think of it as protecting you from yourself.  What the administrator can do, though, is change those permissions.  You do this by using the "Security" tab under Properties.

 

Another possibility is that this is an Encrypted File System (EFS) encrypted file.  If you do not have the certificate to access this file/folder, it may behave like this.  Bring up a command prompt window (Start -> "cmd" (in search block) then "Enter" in the search window) and enter the command:

     cipher /c  c:\temp\ProblemFolder

and see if it indicates that it is encrypted.

You may also be able to right-click the directory -> Properties -> Advanced and see if "Encrypt" is checked.

 

You may get some help or insight from the following articles:

  "Windows cannot access the specified device, path, or file ... "

   < http://support.microsoft.com/kb/2669244/en-us >

  ""Access Denied" or other errors when accessing or working with files or folders in Windows"

   < http://support.microsoft.com/kb/2623670/en-us >
