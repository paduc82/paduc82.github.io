dirk goldgar say:
So the report I'm working on is called "combined report".  When I have it in design view, and am adding the subreports in the detail section (all the subreports are full pages - 8 1/2 x 11),

Don't make your subreport controls big enough to show the whole length of each subreport in design view.  Instead, size each subreport as short as necessary (or possible) to fit them all within the page length of the main report's Detail section.  But set the CanGrow property of all the subreport controls to Yes, and likewise set the CanGrow property of the main report's Detail section to Yes.  Unless I am visualizing the situation wrong, that ought to take care of the problem.

 

You can even set each subreport control's Height property to 0, and they will still work if the CanGrow property is set to Yes.

stefan blom say:

Sorry, I typed the wrong character ($ instead of ) in my previous reply. I have now corrected it.

Anyway, I think we are close to a solution. There are two types of AutoCorrect entries in Word: formatted and unformatted ones. If users have created formatted AutoCorrect entries, then most likely the Calibri font is introduced by these entries.

Unformatted entries always assume the formatting of the surrounding text, but formatted entries may not. Just delete the existing entries and recreate them in the AutoCorrect dialog box (File tab | Options | Proofing | AutoCorrect Options).
