Excel 1900 Leap Year, DATE function, WEEKDAY function - Microsoft Community

dwendl say:

Excel 1900 Leap Year, DATE function, WEEKDAY function

It appears that Excel 2013 and Excel 2003 (the only two versions I've checked) incorrectly treat 1900 as a leap year.

Realize that "If day is greater than the number of days in the month specified, day adds that number of days to the first day in the month." (Excel help)

Using month/day/year:

=DATE(1900,2,28) returns 2/28/1900. Correct.
=DATE(1900,2,29) returns 2/29/1900. Incorrect. It should return 3/1/1900.
=DATE(1900,3,1) returns 3/1/1900. Correct

Contrariwise, although 2100 is also not a leap year, Excel does handle it correctly:

=DATE(2100,2,28) returns 2/28/2100. Correct.
=DATE(2100,2,29) returns 3/1/2100. Correct.
=DATE(2100,3,1) returns 3/1/1900. Correct

Perhaps related to this problem, the WEEKDAY function for dates between 1/1/1900 and 2/28/1900 returns incorrect values.  E.g.,
=WEEKDAY(DATE(1900,1,1),1) returns 1, when it should return 2 (Monday).

mjcorkery (mct regional lead) say:

Hi Tony,

 

There are two factors that affect the height of a row. By default, rows automatically resize to fit the tallest value in a row. So, even when you set the value of the height property this may not be reflected in the user interface. To ensure that the changes are reflected in the UI you will want to set the value of the HeightRule property also. This property specifies the behavior for the row. I have included a code sample below that forces the height of each row in a table to 99 points, regardless of the content in the cells.  Please let me know if this helps.

 

Mike

 

Sub SetRowHeight()
  Dim tr As Row
  For Each tr In ActiveDocument.Tables(1).Rows
    'HeightRule is an enumeration with three values. By specifying wdRowHeightExactly
    'for the value, we are forcing an exact height for table rows.
    'You can also specify wdRowHeightAuto (the default, row height is adjusted to
    'accomodate the tallest value in the row) or wdRowHeightAtLeast (specify a minimum
    'value. This option is selected if you manually adjust the height of a row.)
    tr.HeightRule = wdRowHeightExactly
    'Note that row height is in points, not inches (the Table Tools Layout tab in the UI
    'uses inches. There are 72 points per inch.)
    tr.Height = 99
  Next tr
End Sub

In Word, the "default" table row height, or rather the "auto" row height, is determined by the cell margins and the current font size. You could use the following code:

Sub SetAutoRowHeight()
If Selection.Information(wdWithInTable) Then

End Sub

Assign the macro to a toolbar button and/or a keyboard shortcut. For assistance, see http://www.gmayor.com/installing_macro.htm.

Sub GetOpen_AddPictures() '<< First Line
'Dec 09 2014//
On Error Resume Next
Dim vPic As Variant, vCell As Variant, v1 As Variant
Dim cel As Range
Dim x As Long
Dim p As Picture

vPic = Application.GetOpenFilename(FileFilter:="ALL files (*.*), *.*", _
Title:="open a folder, press Ctrl and select the pictures", MultiSelect:=True)
If Not IsArray(vPic) Then
MsgBox "cancel"
Exit Sub
Else
vCell = Array("A1", "B2", "A3", "B4", "A5")
Application.ScreenUpdating = False
For Each v1 In vCell
For Each p In ActiveSheet.Pictures
If Not Intersect(p.TopLeftCell, Range(v1)) Is Nothing Then p.Delete
Next
Next
t = 0

For Each v1 In vPic
Set p = ActiveSheet.Pictures.Insert(v1)
Set cel = ActiveSheet.Range(vCell(t))
cel.RowHeight = 90.75
cel.ColumnWidth = 20.75
With p
.ShapeRange.LockAspectRatio = False
.Height = cel.Height
.Width = cel.Width
.Top = cel.Top
.Left = cel.Left
.Placement = xlMoveAndSize
End With
t = t + 1
Next
End If
On Error GoTo 0
Application.ScreenUpdating = True
End Sub '<< Last Line
