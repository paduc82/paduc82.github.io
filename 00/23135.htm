jim gordon mvp say:

It isn't that XL2007 is incompatible with the Mac. On either platform it is possible to programmatically include OS specific features into your code. Your Mac has the ability to run AppleScripts and Automator actions via VBA macros, but these will never work in Windows. 


Likewise, Windows users can include Window-only components in their code, which seems to be the case here. I would be willing to bet the Windows author used Active-X controls, which are open source controls that don't run on a Mac because no one in the open source community has ported Active-X to the Mac, even after more than 10 years of Active-X being open source.


The solution for cross-platform compatibility is to use the controls provided within Microsoft Excel. Windows users often call them "legacy" controls. If you stick with Excel controls and avoid Active-X controls, your code will be cross-platform with regard to userform controls.

suzanne s. barnhill say:

No. Marketing of the Office proofing tools has evolved. In earlier versions, the only way to get those that weren't included with the product was to pay $100 or so to get the entire set. Office 2010 introduced an "improvement" for people who wanted proofing tools for only one or a few additional languages by selling each language separately (users who had a need for more languages objected to a change that resulted in having to pay more than they had with previous versions). Office 2013 is the first version to offer all languages free.           

 

This progression probably reflects increasing Microsoft ownership of the tools. In the early days, "lexicons" (for spelling and hyphenation) and rudimentary grammar tools were licensed from a third-party company, but Microsoft created the Natural Language Group and brought production of the tools in-house, adding contextual spelling, translation, and other refinements. This has allowed MS to provide the tools at reduced or no cost.

If you are connected to a domain with group policies then you need to speak with your IT

administrator via the helpdesk to change the group policy for your computer.

That is move your computer name to a different container in active directory that does

not have Allow Telemetry set to basic, it needs to be set to maximum (full).

If you are not on a domain but the group policy settings is wrong you can either reset the

Allow telemetry setting with gpedit.msc if running Pro version or manually change the settings.

Press windows + R Keys (run) or right-click start run

Navigate to HKEY_LOCAL_MACHINE\SOFTWARE\Policies\Microsoft\Windows\DataCollection

Change or add dword AllowTelemetry value to 3

Reboot Computer to take affect
