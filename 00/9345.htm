Hi All!

 

I had faced the same problem with the same skullcandy headset in the link above on E73 but managed to solve the problem immediately. Here what I did in case you haven't solved yours yet.

bill manville say:

I don't think there is any direct way to change the state of a checkbox on a worksheet by using the space bar.  In a Userform you can use the space bar because the focus is in the control; on a worksheet the focus is in the active cell.

The only way I can think of is to have code that intercepts the use of a space key and uses it to change the state of a checkbox whose top left corner is in the active cell:

Sub SpaceOn()
  Application.OnKey " ", "SpacePress"
End Sub

Sub SpacePress()
  Dim CB As CheckBox
  For Each CB In ActiveSheet.CheckBoxes
    If CB.TopLeftCell.Address = ActiveCell.Address Then If CB.Value = xlOn Then CB.Value = xlOff Else CB.Value = xlOn
  Next
End Sub


Sub SpaceOff()
  Application.OnKey " "
End Sub

You start the unusual behaviour by running SpaceOn and stop it by running SpaceOff.  The tricky bit is that you will want to use the spacebar normally in other contexts so you probably want to SpaceOn when the active cell moves into a particular range of cells and off when it moves out, or a different worksheet or workbook is activated or the workbook is closed.  Quite hard to get all the conditions right.

If you want help down that route please come back.

I should have said - the above code assumes the checkboxes are from the Forms toolbar - different code would be needed for the ActiveX type of checkbox from the Control Toolbox.

Just to add some <very important> trivia to be aware of about contact/calendar info stored in an IMAP .ost file (for folder names that include <This Computer Only> in the description - applies to '2013 and '2016

1) If the .ost file gets disconnected from the Outlook profile & email account that originally created it - you will not be able to open the .ost file on its own losing any data stored in folders marked as "This Computer Only"

2) To safeguard against loss you will need to either

a) export the data in those folders periodically to a PST file

<OR> (recommended)

b) create a new PST file in the profile and set it as the <default data file> for the profile (assuming that another one other than the .ost file already is configured as such). When done, move the data from the "This Computer Only" folder to the newly created folders that will appear after you set the new PST file as the default.

Doing step (b) will result in you always having access to your default Contact/Calendar folders whether or not the IMAP account gets deleted (or any other reason for the .ost file getting separated).

=SUM('2200:16000'!$Q100) gives me the number for all the worksheets in Q100 (which gives me the total for the ingredient number on Row 100 for all the worksheet pages).

I need to create a table on RM Usage that gives me the number for each individual worksheet.

=SUMIF('2200:16000'!,J$1,INDIRECT(I4)) to get the total for the ingredient in row 4 that is found in the page number showing in J1. But this formula returns all zeros.

ccbeebiz say:

Turns out it was much simpler than I thought. Since I already had the title of each worksheet in row 4 this formula works.

=INDIRECT(CONCATENATE("'"&J$4&"'!",$I5))
