Look - an unmountable boot volume is computer speak for "your hard drive is either dead, missing or damaged".

Let's hope that the cause of this message is that you have left an update disk, CD or USB stick in the computer during the bootup process.
Please make sure you have not.

Sometimes, it is possible that the values contained in the ROM BIOS for the boot sequence of devices gets out of sync with reality.
In that case, verify in your ROM BIOS that your primary hard drive is still listed correctly. Is the HD a SATA, ATA, SSD? Know those things before hand.

Absent an issue like the one I described above, when Windows does not boot, a few things could be happening:
1. After a system update, a driver is causing issues and stopping the boot.
2. An incompatible device is inserted in the computer

The steps I would take are as a follows.
Is this a laptop or a desktop?
On a laptop, please flip it over and remove the internal battery pack.
Keep it out for now. Unplug the power cord as well. Wait half a minute.
Connect the power cord back to the laptop.
Keep the battery out.

If this is a desktop, simply unplug the power cord and keep the PC disconnected for half a minute.
Reconnect the power cord.

Make sure you have nothing connected to the computer.
Keep your keyboard and mice connectors, but everything else goes!
No USB drives. Make sure the CD bay is empty.

Turn the computer back on. Most of the time, this jiggling around seems to prod Windows back alive.

I hope this gives you some ideas.
Please keep us updated on your progress.

malkeleah say: In general, Windows won't see the files on a Linux-formatted hard drive. There is a utility which allows exploring EXT2/3 files systems from Windows but AFAIK it doesn't work for Ext4. If you have data in Linux which you wish to share with Windows, you can put it on a separate partition formatted FAT32 which both operating systems can read. Or do the backup from the Linux side and not from within Windows. You can create a chron job to handle this or use one of the many Linux backup programs. I'm very old school Linux and am hesitant to recommend regular writing to an NTFS partition from Linux because of the risk of corrupting the NTFS partition, but the newer distros like Ubuntu *can* write to NTFS natively. Depending on your situation, a better solution might be to use a network drive (Google for "NAS") which both systems can use as a backup destination.

Hello,

There are several factors that affect regarding the maximum amount of RAM in your computer. The first and most obvious is the architecture of the system (32 or 64 bit) since 32 Bit systems theoretical only recognizes a maximum of 4GB.

As some others have pointed out, you dont see all of this memory in windows sometimes but that does not mean it is going to waste! You dont "lose" the RAM its simply being used by the system. (For things like PCI/PCI-X, graphics e.t.c.)

You really do not need to upgrade to a different OS and there will be no significant difference (in terms of seeing RAM) if you upgrade to 32 bit Win 7.

Windows can show as “Non-Genuine” for several reasons, such as incorrect Date and Time or an incorrect Product Key.

 

To verify Date and Time are correct:

 

·        Click the Start Button on the Desktop

·        In the Search field type Date and Time and press enter

·        Verify the date and time is set correctly

 

Retype the Product Key:

 

·        Click the Start Button on the Desktop

·        Select Computer

·        Select System Properties on the Taskbar

·        At the bottom click Change product key and retype your Product Key

 

Reactivate by Phone:

 

·        Click the Start Button on the Desktop

·        In the Search field type SLUI 4.exe and press enter (Note, there is a space between “I” and “4”)

·        Select your country

·        Call the phone number listed and follow the automated prompts.

 

If Windows is still showing Non-Genuine please download and run theMGA Diagnostic Tool, post the results along with any  previous troubleshooting steps performed in theMicrosoft Genuine Advantage forums.

 

Note: If your Operating System is Windows Vista SP1 or earlier you will notice you do not have access to the Start Button as Windows is in Reduced Functionality Mode.

 

If the Operating System is in Reduced Functionality Mode below are the steps needed to download and run the MGA Diagnostic Tool:

 

·        When you login to Vista, a window with several options will be available, select the option to open your  Internet browser.

·        Type:http://go.microsoft.com/fwlink/?linkid=52012 into the browser address bar.

·        A window will come up prompting to “Run” or “Save”, select “Run”

·        When the MGA Diagnostic Tool starts running click the “Continue” button, when the diagnostics is complete click the “Copy” button.

·        Paste the report into a thread in the appropriateWindows Genuine Advantage forum
