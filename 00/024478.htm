daniel g. grau say:
A hyperlink within a file can refer to:

  1. An absolute path to a file: { HYPERLINK "file://localhost/Users/USERNAME/folder/folder/... /filename" }
  2. A relative path to a file in the same folder as the source file: { HYPERLINK "filename" }
  3. A relative path to a file in a subfolder next to the source file: { HYPERLINK "subfolder/filename" }
  4. A webpage's URL: { HYPERLINK "http://www.microsoft.com" }
  5. A file on a mounted server: { HYPERLINK "file://localhost/Volumes/SERVERNAME/folder.../filename" 

When the above types of hyperlinks are contained within a given source file, and this file is transferred over to another computer:

  • Case 1 will never work if you simply transfer the source file, unless the two computers have identical folder structures and names. This is unrealistic.
  • Cases 2 and 3 will work as long as the source file and accompanying files and subfolders are also present in the local computer (and have the same names as in the original computer)
  • Case 4 will always work as long as you have an Internet connection
  • Case 5 will always work if both computers can mount the same server

When inserting a hyperlink to a local document, Word will create Case 1. To convert it into Cases 2 & 3, you need to enable the following option:

  • Word > Preferences > General > Web Options, select the Files tab and place a checkmark on Update links on save. However, you can also edit the hyperlink fields manually (use SHIFT F9 to view the hyperlink codes)

Furthermore, you'll need to transfer over the source file and all accessory files and subfolders. The best solution is to create a folder containing all of that and then transfer over the complete folder.

    Dim oFile   As Object

    Set oFolder = Nothing
    Set oFSO = Nothing

vijay a. verma say:

You have to enclose your range reference in an INDIRECT Block, hence your formula would be -

=LARGE(INDIRECT("'Monthly Gallons'!$B$18:$Z$18"),1)

vijay a. verma say:

Your formula would be -

=INDEX(INDIRECT("'Monthly Gallons'!B$17:ZZ$17"),MATCH($B5,INDIRECT("'Monthly Gallons'!B$18:ZZ$18"),0))

Inserting new columns will change B5 to C5, if you want to change B5 to B6 etc., then you will have to use following formula -

=INDEX(INDIRECT("'Monthly Gallons'!B$17:ZZ$17"),MATCH(INDIRECT("$B"&COLUMNS($A:E)),INDIRECT("'Monthly Gallons'!B$18:ZZ$18"),0))

john w. vinson say:

One way this can happen is if you have a large (over 2 billion odd) "number" in the input file, which is really an identifier of some sort: a phone number (area code 201 will work, area code 212 won't!), Social Security number, etc. Such fields must be loaded into Text datatype fields, not Number. If you're using the import wizard, Access will look at the first few rows of data and guess at the appropriate datatype, and may well guess wrong!

Normally in this kind of situation you're better off creating a local table with appropriate datatypes and field sizes, linking to the .csv file, and running an Append query rather than a MakeTable. This gives you better control over datatypes, indexes, etc.

 

John W. Vinson/MVP
