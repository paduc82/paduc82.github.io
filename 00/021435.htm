suzanne s. barnhill say:

You'll need to just remove the "Page Break Before" formatting from specific headings via the Paragraph dialog (without changing the style). I don't know of any way to make this automatic. It's possible that a macro could be written to accomplish this, but it would not occur automatically when you create the second heading.

 

FWIW, I would use Heading 1 (modified as appropriate) for the unit heading instead of creating a custom style. Word's built-in heading styles have many "magic" properties that user-defined styles can never have; see http://www.shaunakelly.com/word/numbering/UseBuiltInHeadingStyles.html. It's also a lot easier to have auto numbering if you start with the built-in styles (using a list template that has the levels already linked to the heading styles). Heading 1 doesn't have to be numbered, and Heading 2 doesn't have to restart after Heading 1, but it will simplify creating a TOC.

jay freedman say:
If you're saving the building blocks into the Normal.dotm template, which is the default, then I'd recommend making a backup copy of the Normal.dotm file whenever you add or change a building block (or a style or anything else you've stored in that template). If something corrupts your template, just copy the backup back to overwrite the corrupted template.

The more urgent question, I think, is why you're getting this problem in the first place. If it is updates -- and that's probably at least one cause -- then set Windows Update to download but not automatically install. You'll get a popup notice when updates are ready, but you can delay the installation until you've safely shut down Word and anything else that has data files open.

Word should never shut down without saving any unsaved changes, in open documents or in Normal.dotm. If it's doing that, then something is wrong with your Office installation. It may help to go to the Advanced tab of the Options dialog, scroll down to the Save section, and check "Prompt before saving Normal template".

david cornett say:

Convert a UserForm TextBox from Text to Long or Integer

Hello,

I have created a UserForm in VB for users to enter, among other data, numeric values. I have added a textbox that I want to automatically calculate based on the values of two other textboxes. I understand that values added to a textbox are strings and need to be converted to long or integer but I'm having trouble accomplishing that task. 

With the Code:

TB3= TB1+ TB2

If I enter a 5 in TB1 and a 2 in TB2, TB3 becomes 52 (not 7).

So, I tried

TB3 = CStr(CLng(TB1) + CLng(TB2))

and I get an error message "Run-Time error '13'" Type mismatch"

Any suggestions? 

Thanks. 

bernie deitrick say:

This worked fine for me:

Private Sub CommandButton1_Click()
Me.TextBox3.Value = CLng(Me.TextBox1.Value) + CLng(Me.TextBox2.Value)
End Sub
