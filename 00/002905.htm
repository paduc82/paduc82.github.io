zamzama35 say:

MS Access-2007-How to Sum up in Query Row

I want to sum up Time elapsed Calculated Field.Please help me how should I Do?

ken sheridan say: You need to repeat the original expression.  So if the elapsed time per row in the table is obtained by subtracting the start time of an event from its end time (both being a date/time data type) with:

[EndTime]-[StartTime]

you can put this in a grouped query to sum the elapsed times:

SUM([EndTime]-[StartTime])

However, if the sum of the elapsed times is 24 hours or more this will give you a result as a rather strange date, probably around the end of the 19th century.  You can convert this to a string in the format days(optional):hours:minutes:seconds with the following function:

Public Function TimeElapsed(dtmTime As Date, _
            Optional blnShowdays As Boolean = False) As String

    ' Returns a date/time value as a duration
    ' in format hh:nn:ss, or d:hh:nn:ss if optional
    ' blnShowDays argument is True.
    
    Dim lngDays As Long
    Dim strDays As String
    Dim strHours As String
    
    ' get whole days
    lngDays = Int(dtmTime)
    strDays = CStr(lngDays)
    ' get hours
    strHours = Format(dtmTime, "hh")
    
    If blnShowdays Then
        TimeElapsed = lngDays & ":" & strHours & Format(dtmTime, ":nn:ss")
    Else
        TimeElapsed = Format((Val(strDays) * 24) + Val(strHours), "00") & _
            Format(dtmTime, ":nn:ss")
    End If

End Function

With this function in a standard module in your database, you can then return the total time in the format hh:nn:ss with:

TimeElapsed(SUM([EndTime]-[StartTime]))

or in the format d:hh:nn:ss with:

TimeElapsed(SUM([EndTime]-[StartTime]),True)

ken sheridan say: You can use the LIKE operator rather than the equality operator:

    rs.FindFirst "[OrganisationName] LIKE ""*" & txtGoTo & "*"""

Though as a substring search like this is likely to return multiple rows a better approach might be to filter the form with:

    Dim strFilter As String

    strFilter = "[OrganisationName] LIKE ""*" & txtGoTo & "*"""

    Me.Filter = strFilter
    Me.FilterOn = True

Another possible approach is to use a combo box in which the user can select a name from a list.  That way the list will only allow values which exist in the column in question to be selected.  By virtue of the AutoExpand mechanism as you type a value into the combo box it will progressively go the first match in its list.
