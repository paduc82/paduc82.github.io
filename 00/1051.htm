Microsoft Outlook for Mac - Scheduling Send and Receive with Exchange - Microsoft Community

jasongodsey say:

Microsoft Outlook for Mac - Scheduling Send and Receive with Exchange Server

I found other questions asking the same, but they are all locked so I can't comment. I found a work around to long delays with exchange server sending the push notification to my Outlook Client.

Create a new Automator application, I named it /Users/Jason/OutlookSendAndReceive

Under Actions, add Run Applescript which contains the following:


try

if outlook_running() then

delay 0.899408

set timeoutSeconds to 2.0

set uiScript to "click UI Element \"Send & Receive\" of group 8 of scroll area 1 of tab group 1 of window 1 of application process \"Outlook\""

my doWithTimeout(uiScript, timeoutSeconds)

end if

end try


on outlook_running()

tell application "System Events" to "Microsoft Outlook" contains "Outlook"

end outlook_running


on doWithTimeout(uiScript, timeoutSeconds)

set endDate to (current date) + timeoutSeconds

repeat

try

run script "tell application \"System Events\"

" & uiScript & "

end tell"

exit repeat

on error errorMessage

if ((current date) > endDate) then

error "Can not " & uiScript

end if

end try

end repeat

end doWithTimeout

I then schedule this application to run every minute via cron (Sorry, I'm not well versed on Launchd)

In a Terminal window:

This will edit crontab

Run and add the following: (Note: you will be prompted by the system after saving the crontab entry, and again when it runs the first time)

$ crontab -e 

* * * * * /Users/Jason/OutlookSendAndReceive.app/Contents/MacOS/Application\ Stub

Now, while Outlook is running, it will perform Send & Receive at the top of the minute, every minute.

Sub Test()
  Dim Dest As Range, R As Range
 
  'Screen off, runs faster
  Application.ScreenUpdating = False
 
  Set R = Range("A1")
  Set Dest = Range("E2")
 
  Do
    'Go to next "mem"
    Set R = R.End(xlDown)
    If R.Row = Rows.Count Then Exit Do
    
    'E2=A2+B2
    Dest = R & R.Offset(, 1)
    'F2=a3+b3
    Set R = R.Offset(1)
    Dest.Offset(, 1) = R & R.Offset(, 1)
    
    'Go to "Number"
    Set R = R.End(xlDown)
    If R.Row = Rows.Count Then Exit Do
    Do
      'g2=a5
      Dest.Offset(, 2) = R
      'h2=b5+b6
      Dest.Offset(, 3) = R.Offset(, 1) & R.Offset(1, 1)
      'I2=a6
      Dest.Offset(, 4) = R.Offset(1)
      'j2=c5
      Dest.Offset(, 5) = R.Offset(, 2)
      
      'Next row
      Set Dest = Dest.Offset(1)
      'Next "number"
      Set R = R.Offset(2)
    Loop Until IsEmpty(R)
  Loop
 
  Application.ScreenUpdating = True
End Sub

jllatham say:

See postings on this page:

http://dailydoseofexcel.com/archives/2007/02/21/find-position-of-first-capital-letter-in-a-string/

Now, don't ask me anything about that formula - such as "why 77.5" or <13, I don't know, but I tested one of the formulas and it appears to work:

=MATCH(TRUE,ABS(CODE(MID(RIGHT($A2,LEN(A2)-1),ROW($1:$255),1))-77.5)<13,0)

Now that is an array formula and so you have to enter it with [Ctrl]+[Shift]+[Enter], not just the enter key.  That will give you the position of the 1st uppercase character in the string (B in BobSmith).  So we need to modify it a bit and pull it all together for you:

For a formula to work on a name in cell A2

=LEFT(A2,MATCH(TRUE,ABS(CODE(MID(RIGHT($A2,LEN(A2)-1),ROW($1:$255),1))-77.5)<13,0)) & " " & RIGHT(A2,LEN(A2)-(MATCH(TRUE,ABS(CODE(MID(RIGHT($A2,LEN(A2)-1),ROW($1:$255),1))-77.5)<13,0)))

Again, commit the formula with [Ctrl]+[Shift]+[Enter]  if you do it right, Excel will add {} at the beginning and end of the formula and BobSmith will turn into  Bob Smith.

You can then select and copy all of those cells with formulas, then select the first original cell with a name in it and use Edit --> Paste Special with the Values option selected to replace the original list with the new one, and then delete the column with the formulas in it.

That page has several other options that repute to achieve the goal, I didn't try any of the others.

Actually, upon reflection I'm thinking the <13 is checking that the entire string (entire name) is less than 13 characters long, so you may need to change that to the longest name you have.  You can check lengths of current entries like this:

=Len(A2) and fill down.  Then, assuming those formulas are in column B, put this into another column:

=MAX(B:B) to find out the longest whole name.  Remember that and delete all these formulas and continue on with the one above substituting the max value you got for the 13.

bill manville say:

Left margins can be created using a left alignment with indent
Right margins can be created using a number format with padding character on the right.
But for top and bottom you are out of luck through the user interface, I think.

A simple macro could do it for you, though.
This will do it for the rows containing cells you have selected

Sub AutoFitAndAddSevenPoints()
  Dim R As Range
  Selection.EntireRow.AutoFit
  For Each R In Selection.EntireRow.Rows
    R.RowHeight = R.RowHeight + 7
  Next
End Sub
 
(I assume you already have consistent vertical alignment)
