I have two questions about word.  1.  How do I shut off the automatic outline?

File tab > Options > Advanced > Proofing > AutoCorrect Options > AutoFormat as you Type


2.  How do I reset the space between paragraphs so it will stay at 0?

More complicated. First, it won't stay at 0 if you change it or if you import text that has a different setting and just start copying or typing.

What style are you using? If you don't know it is probably normal. You should learn about styles as soon as possible; it will save you a lot of work.

Under the Home tab in a new blank document click in the bottom right of the Styles Group. This will open the styles pane. The style you are using will have a blue box around it. Right-click on this and select modify.

Toward the bottom of the modify style dialog box is a place for you to check "New documents based on this template." Check that.

Click on the Format button (bottom left) and select paragraph. Set what you want in the paragraph settings. OK your way out of the dialog boxes. Close Word. If asked if you want to save changes to the template, say yes.

Reopen Word and see if the change stuck.

Note, this will not change existing documents nor documents you receive from elsewhere.

AFAIK you can only check the Application.CalculationState to test if calculation is needed. But that means all sheets in all open files.

The macro stops automatically until the sheet is calculated. Execute the macro below in a new file and look into the immediate window.

Have a look at this side to get more informations about measuring calculation times of formulas.
http://msdn.microsoft.com/en-us/library/aa730921.aspx

Andreas.


Sub Test()
  Dim T As Single
  Debug.Print "Start"
  Application.Calculation = xlCalculationManual
  Range("A1") = 1
  Range("A2") = "=A1+SIN(A1)"
  Range("A1:Z2").FillRight
  Range("A2:Z50000").FillDown
  T = Timer
  Debug.Print "Before calculation"
  ActiveSheet.Calculate
  T = Timer - T
  Debug.Print "After calculation " & Format(T, "0.00") & " seconds"
End Sub

joeu2004 say:

What you see is the side-effects of 64-bit binary floating-point arithmetic anomalies.

It appears that the times in B5:B9 and G5:G9 were calculated -- perhaps created by dragging or by using the Fill operation.  Instead, enter the following formulas and copy down the column:

B5: =--TEXT(B4+TIME(1,0,0),"[h]:m:s")

G5: =--TEXT(G4+TIME(1,0,0),"[h]:m:s")

Afterwards, you can copy and paste-special-value, if you want to eliminate the formulas.

The TEXT function effectively rounds to the second.  The double-negate converts the text result to numeric Excel time.

Excel time is stored as a fraction of a day (1 hour = 1/24).

In general, when you do arithmetic with decimal fractions or that might result in decimal fractions (e.g. division of integers), it is prudent to explicitly round to the precision that you expect to be accurate.  Use ROUND for "normal" values.  But use TEXT and a time format for Excel time values.

It appears that the times in B4, G4, D4:D6 and I4:I7 were entered as constants.  So no explicit rounding is needed.

-----

In the original example (before using the formulas above), you can demonstrate that this is a binary arithmetic anomaly by "re-entering" the times in B5:B9 and G5:G9.

For example, select B5, click on the time in the Formula Bar, and press enter.  Then, E5 correctly shows 95 instead of #N/A.

Sometimes, you can also demonstrate this effect by entering formulas of the form =D5=B5.  That returns TRUE, even though MATCH returns #N/A.

The difference is:  the "=" operator compares left and right expressions rounded to 15 significant digits, whereas MATCH compares the internal binary representation.  However, rounding to 15 significant digits does not always correct the effect of binary arithmetic.
