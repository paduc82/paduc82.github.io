ken sheridan say: I think you will need to write some VBA code for this.

Let's assume each row  in the subform occupies 1/4 of an inch height.  This is 360 twips, which is the unit used in this context.  A twip is a typographical unit of 1/20 of a point, so 1440 twips = 1 inch.  To expand or contract the subform control downwards to allow for all rows of data and one empty new row, put the following in the parent form's Current event procedure:

    Dim intRows As Integer
    
    With Me.[NameOfSubformControlGoesHere]
        intRows = .Form.Recordset.RecordCount
        .Height = (intRows + 1) * 360
    End With

If you are unfamiliar with entering code into a form's, report's, report section's  or control's event procedures, this is how it's done in form or report design view:

1.  Select the form, report, section or control as appropriate and open its properties sheet if it's not already open.

2.  Select the relevant event property and select the 'build' button (the one on the right with 3 dots).

3.  Select Code Builder in the dialogue and click OK.  This step won't be necessary if you've set up Access to use event procedures by default.

4.  The VBA editor window will open at the event procedure with the first and last lines already in place.  Enter or paste in the code as new line(s) between these.

Note that NameOfSubformControlGoesHere should be the name of the control in the parent form's Controls collection which houses the subform.  This might or might not be the same as its source form object.
