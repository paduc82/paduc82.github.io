Please note, the SMS text backup is not a backup, it's a sync, along with the contacts, email and calendars on the primary Microsoft account. It maintains a mirror = if texts are deleted on the phone, they are also deleted on the server during the next sync.
This (ActiveSync) runs in batches. Others reported that restoring thousands of texts on a new phone or after a hard reset can take days to complete.

The system backup that can be restored during first setup does not contain any text messages and there are no pictures, documents or other personal files in this backup.

The contacts+messages app may also need more time to complete with thousands of texts. There is sadly a risk the incomplete syncs and switching SMS 'backup' on and off have already wiped or corrupted some of the texts on both the phone and the server.
I suggest keep the old Lumia offline and try give the app more time to complete.

Others have reported similar issues. I'm not aware of any solution.

roady [mvp] say:

When it comes to certificates, there is nothing that you need to install client side unless you are using self-signed certificates. In that case, you'll need to install the root certificate.

However, this doesn't apply in your case since it states that the principal name on the certificate is incorrect. This means that the server is responding with a different name than that is on the certificate. This can happen when you've added multiple CNAMES for the server (like the old one) which aren't on the certificate.

To solve this, make sure that your Outlook clients are using server names which are actually on the certificate and that the mail server also responds with this name.

rohn007 say:

Pretty much any current business class computer will be adequate to your need. 

Specs:

  • From what I've seen during my recent search for a new computer, all business class computers have 64 bit CPU.  Simplest way to tell is if it comes with more than 4GB of RAM. 
  • You will probably want to get at least 8 GB of RAM.  
  • CPU, you would be looking at any i5, or i7 if you want to push the price up some more.  
  • If you can, go for a computer with a smaller SSD (128-256GB) rather than a larger (500GB-2TB) normal HD.  SSD will have largest perceivable impact on performance/response time.  For a business computer, you do probably intend to keep most of your business files on the corporate network, only files you need for travel purposes would be kept temporarily on the laptop. 
  • 15-17" monitor. 
  • Windows Pro, NOT HOME. This is going to be the hardest one to get from manufacturer (I know, this is something I've specifically been looking for, and not finding very often, even as an optional upgrade).  Or, it may be a non-issue if your company has a site or volume license that allows them to upgrade the license to Windows Pro or Enterprise on all corporate computers.

Laptops with those specs are running around $800-$1500 retail.

3-600,000 rows of data do not necessarily prove a need for 64 bit.  Unless the lines are really long, or you have many or complex formulas in the workbook.

The criteria that MS states for switch to 64 bit are file size (on hd) approaching 2GB or RAM use (when file is open in Excel) also approaching 2GB.  If either of those criteria are met, then you should switch to 64 bit.

mike h.. say:

Hi,


It's not possible to create sheets with a formula so you have choices and as I see it they are:-


1. Manually create the sheets then lots of copy and paste.

2. Manually create the sheets and then create lots of formula to reference the data sheet.

3. Use VB code.



Should option 3 be of interest the code below will do that. It will read down a column and copy to a sheet with the same name as the customer and if that sheet doesn't exist it will create it.


See the comments in the code for some small changes you may need to make.



Sub Copy_Data()
 Dim r As Range, LastRow As Long, ws As Worksheet
 Dim LastRow1 As Long, MyColumn As String
 Dim src As Worksheet
 'Change this column Letter to the one with the Co ID in
 MyColumn = "B"
 'Change the worksheet name to the one with the data on
 Set src = Sheets("Sheet1")
 LastRow = src.Cells(Cells.Rows.Count, MyColumn).End(xlUp).Row
 For Each r In src.Range(MyColumn & "2:" & MyColumn & LastRow)
         On Error Resume Next
         Set ws = Sheets(CStr(r.Value))
         On Error GoTo 0
         If ws Is Nothing Then
             Worksheets.Add(After:=Worksheets(Worksheets.Count)).Name = CStr(r.Value)
             'This row adds a header from the soyrce sheet
             'remove the ' if you want to do that
             'src.Rows(1).Copy ActiveSheet.Range("A1")
             LastRow1 = Sheets(CStr(r.Value)).Cells(Cells.Rows.Count, MyColumn).End(xlUp).Row
             src.Rows(r.Row).Copy Sheets(CStr(r.Value)).Cells(LastRow1 + 1, 1)
             Set ws = Nothing
         Else
             LastRow1 = Sheets(CStr(r.Value)).Cells(Cells.Rows.Count, MyColumn).End(xlUp).Row
             src.Rows(r.Row).Copy Sheets(CStr(r.Value)).Cells(LastRow1 + 1, 1)
             Set ws = Nothing
         End If
    
 Next r
 End Sub
