ally jamison say:

Opening a Word file using VBA in Office 2011 for Mac

I am new to Office 2011 for the MAC and trying to get some VBA code that was written and works for Windows to work on my Mac. The function simply opens a Microsoft Word document and the call to "Set moWordApplication = CreateObject("Word.Application")" is failing for  error 429 "ActiveX component can't create object. I have played with the references and at this point all are checked, but it still fails. Does anyone know where I can get a good code sample that works for me to test with? I have done numerous searches and I am not finding a good example to work from. Any help would be appreciated.
j.e. mcgimpsey say: Unlike Windows machines, Macs (conventionally not "MAC"s: all-caps MAC usually refers to a Media Access Control address) only allow one instance of an application, so if you're running your code from Word (or if you're running the code from somewhere else and Word is running on that machine), you'll get that error.

If you're only going to be running the code from within Word, then you can dispense with CreateObject, and use early binding:

Dim moWordApplication As Word.Application
    
Set moWordApplication = Word.Application


If you're running the code from another application, one way, using late binding is to test for a running app, and if it's not found, start it:

    Dim moWordApplication As Object

On Error Resume Next 
Set moWordApplication = GetObject(, "Word.Application")
On Error GoTo 0 ' or error handling routine 
If moWordApplication Is Nothing Then _ 
 Set moWordApplication = CreateObject("Word.Application")

Or, if you know your user(s) will have an installed copy of the Microsoft Word 14.0 Object Library, set a reference to it from within the project (Tools/References...), and use the more efficient early binding:

Dim moWordApplication As Word.Application
    
On Error Resume Next
Set moWordApplication = Word.Application
On Error GoTo 0     'or error handling routine
If moWordApplication Is Nothing Then _
Set moWordApplication = New Word.Application

It's not great practice, since it adds inefficient code under the hood, but if you reference the Word Library, you can instead use the New keyword in the Declaration to bypass the check for a running app (the inefficiency comes from the compiler's including the code to do the check for a running app each time moWordApplication is used in your procedure):

Dim moWordApplication As New Word.Application
    
Set moWordApplication = Word.Application

doug robbins - mvp office apps & services (word) say:

Use the Different First Page Header option and initially, when setting up your template, add a second page so that you can then access and setup the required information in the primary header\footer (the ones that will appear on the second and subsequent pages).  When you have that set up as you want it, delete the second page from the template.

aledj---01 say:

I had the same problem for a while on my N97. There are various ways to tackle this, but I eventually chose to solve it by creating a 'new' destination group.In Settings/Connectivity/Destinations/Options/Manage and select New destination. Give this any name you want, and then put into this new group your selected destinations in priority order (for me wifi first then network internet and finally network wap). Under Detinations/Access point/Options/Default connection make the new group just created the default connection.I personally, under Web/Options/Settings/General, have set the Access point to be the new group just created. All this seemed to solve the problem. While in range of my wifi this is used by default; but as soon as the connection is lost I am asked which alternative connection to use and this then works fine! I am with vodafone, but this should be network independent?All apps which need internet/data access, such as Google Maps, Amaze GPS, Facebook etc. work properly.

(Just a small point is that if you have dynamic content on the main screen, such as the BBC iplayer, which is updated by wifi, as soon as the signal is lost the phone keeps beeping with a 'Wifi connection lost' message. I simply select Options/Content to offline mode until I get back into wifi range as I don't want, or need, to update the home screen content over the network!)

I do like the N97 and (fingers crossed) have not had any of the major problems that others have reported. It is a SIM free handset.

Hope this may be of some help.
