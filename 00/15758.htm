Can I just be a bit careful with my terms here so nothing critical gets wrecked?

There is a special type of Admin account that is built into Windows.  Whilst it is normally referred to as the Built-In Admin or Default Admin, its actual account name is Administrator and if you log into this account the folder C:\Users\Administrator is created.  This account is normally disabled and it ought to remain so for normal computer operations.  We  can discuss this further if you want but I am hoping that your use of the full name Administrator above was just how you chose to write and not indicative of the name of an account you have been working with.

If this newly created Admin account is displaying everything that you would want then you can start using it as your day-to-day account and ignore the accounts that have problems.  In addition, you can link that to your MS Account - doing this will alter the name that appears for that account to the one you have used in your online MS Account Settings.  Linking existing Local accounts to MS accounts is provided for in Settings, User accounts.  Renaming accounts is a more serious step though.  There are some threads on the subject but if you read them please notice that even if you achieve renaming the account, the C:\Users folder for that account will not be renamed. 

If you do start using the newly created Admin account as your day-to-day account then just use that Admin account to create the two additional Admin accounts that I recommended.

john mcghie say:
Peter is correct:  That error is absolutely diagnostic of a macro that is running trigged by the Document Close or Document Save event.

Such macros are often distributed with things such as Citation Manager software or Document Management software.  Their purpose is to intercept the Word Save event and re-direct it.  Viruses do this too :-)

The Macro is probably IN the user's Normal.dotm.  But it could also be in a Global Add-in that is loaded via the Word Startup folder.

Open a document on the user's login, then go to Tools>Templates and Add-ins.  See if there is anything listed in the lower box: that could be it.

If you right-click that entry, you should be able to find out where it is.  Anything in the Word Startup folder will load automatically at Word launch.

Whatever it is, it's failing to compile: either because it's PC code that won't run on the Mac, or because some other parts of the solution are missing.

Hope this helps
