ken sheridan say:
How do I set the index for both QID and AuditID at the same time to not allow duplicates?


Select Indexes on the Design ribbon.  In the dialogue which opens enter a suitable name for the index in the first empty row of the Index name column, AuditQuestionIndex say.  In the Field Name column of the same row enter QID, and with this row still selected, in the Index Properties below, select 'Yes' as the Unique property.  Then enter AuditID in the Field Name column in the row immediately beneath ProjectID.  Don't enter anything in the Index name column for this row.  Close the dialogue and save the table design.

You'll get an error message if you execute the 'append' query directly.  If you execute it in code you can trap the error by means of the dbFailOnError option of the Execute method.  The following is a simple example which attempts to insert a row with a ContactID value of 4, which already exists in my Contacts table, resulting in a key violation error:

Public Sub Test()

    Dim strSQL As String
    
On Error GoTo Err_Handler
    
    strSQL = "INSERT INTO Contacts(ContactID,FirstName,LastName) VALUES(4,""John"",""Smith"")"
    CurrentDb.Execute strSQL, dbFailOnError
    
Exit_Here:
    Exit Sub
    
Err_Handler:
    MsgBox Err.Description
    Resume Exit_Here

End Sub

With the above simple example the error could be avoided in the first place, however, by amending the SQL statement:
    
    strSQL = "INSERT INTO Contacts(ContactID,FirstName,LastName) " & _
        "VALUES(4,""John"",""Smith"") " & _
        "WHERE NOT EXISTS " & _
        "(SELECT * " & _
        "FROM Contacts " & _
        "WHERE ContactID = 4)"

ken sheridan say: You can apply parameters to a crosstab query , but they must be declared:

PARAMETERS [Enter exam date:] DATETIME;
TRANSFORM Avg(qryFullNames.MARKS) AS AvgOfMARKS
SELECT qryFullNames.ADM_NO, qryFullNames.[STUDENT NAME]

GROUP BY qryFullNames.ADM_NO, qryFullNames.[STUDENT NAME], qryFullNames.EXAM_DATE
