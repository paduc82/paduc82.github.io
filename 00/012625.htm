One thing you could try. Have the client send you a blank document (based on his Normal template) created in Word 2010. On your system, the document will open in Compatibility Mode. You can use this as the basis for the documents you create for the client (I'm not sure whether it will work to save it as a template or not). Just make sure that every time you save a document, you check the box for "Maintain compatibility with previous versions of Word." If the issue is that you need to use themes or building blocks from Word 2010, then your task will be more complicated.

Strictly speaking, most of the new features in Word 2013 and 2016 were operational features (such as collaborative tools), not content, so you could try creating a document in native Word 2016 format and see whether the client has any trouble opening/using it.

Another trick that might help is to run a simple macro to set the compatibility of a given document to Word 2010. You then have the full panoply of Word 2010 Layout Options (instead of the restricted selection offered by Word 2013/2016), and presumably you will be warned about incompatible features. Unfortunately, I cannot at the moment lay my hands on this one-line macro, but probably someone else will provide it.

jay freedman say:

The macro code (which can be entered in the Immediate window of the macro editor without making a full-fledged macro) is

ActiveDocument.SetCompatibilityMode 14

(because Word 2010 is version 14.0). After typing or pasting that line into the Immediate window, press Enter. Then you need to go to the Options > Advanced dialog, scroll to the bottom, and set the "Lay out this document as if created in" box to Word 2010, and the layout options will appear.

huni shen msft say:

Hi AndyTech21,

As mentioned in the article below, Microsoft Office 365 ProPlus (2016 version) uses Azure Active Directory Authentication Library (ADAL) framework-based authentication and Office uses Web Account Manager (WAM) for sign-in workflows on Windows builds.

https://support.microsoft.com/en-my/help/4025962/cant-sign-in-after-update-to-office-2016-build-16-0-7967-on-windows-10

You can apply the following registry key to disable WAM:

[HKCU\SOFTWARE\Microsoft\Office\16.0\Common\Identity\DisableADALatopWAMOverride] - REG_DWORD "1"

The regkey disables WAM use in Office, which can degrade the auth experience and still maintains ADAL. WAM is a replacement for credential manager in Windows 10. WAM is not re-using refresh tokens correctly. The ultimate fix to WAM will come from a Windows update. If you applied this regkey, we recommend deleting the regkey once the fix is out.
