cassio pereira say:

Ok, so I actually did some digging and found out you can create macros, but you have to use VBA.

After some more digging I came up with a script that does what I want. For anyone else interested, go to View > Macros, insert a name and hit create. The code is as follows:

Sub AddSlideWithLastTitle()
    n = ActivePresentation.Slides.Count
    t = ActivePresentation.Slides(n).Shapes.Title.TextFrame.TextRange.Text
    With ActivePresentation.Slides
        Set slideObject = .Add(n + 1, ppLayoutText)
    End With
    slideObject.Shapes.Title.TextFrame.TextRange.Text = t
End Sub

I could not for the life of me find a way to assign a keyboard shortcut to the macro. The best I came up with is to click on customize (really tiny down arrow on the top left), click "more commands", select "macros" and add its icon to the quick access bar.

Hope this helps someone else out there.

john korchok say:

Glad you got it figured out, well done.

PowerPoint has little support built-in for assigning keyboard shortcuts. Here's an add-in you can purchase and download: Shortcut Manager for PowerPoint

A simpler alternative is to assign the macro to a Quick Access Toolbar icon. Items in the QAT have numeric shortcuts automatically assigned to them:

  1. In PowerPoint, click on FILE, then on Options.
  2. Select the Quick Access Toolbar panel.
  3. Change the Choose commands from dropdown to Macros.
  4. Select AddSlideWithLastTitle and click on the Add button. The macro is copied to the right-hand list. OK out.

Back in PowerPoint, you'll notice a new icon on the right end of the QAT. Press and hold the Alt key: the shortcut key displays over the icon. In the future, press Alt + the displayed number to run your macro.

Things to remember with PowerPoint macros: the presentation containing the macro must be open in PowerPoint for the macro to run. In the long run, especially if you create more macros, you may find it more convenient to place the macros in an add-in that is always active whenever PowerPoint is open. Here is Steve Rindsberg's page on this subject: Create an Add-In. This is Shyam Pillai's FAQ about Add-Ins: Add-In FAQ
