Change default OneDrive attachment authorizations from Modify to - Microsoft Community

cédricmi say:

Change default OneDrive attachment authorizations from Modify to read-only

By default, using the new feature of Outlook 2016 to link to a OneDrive file instead of attaching the file in an email gives a modify authorization to the recipients.

While this default may be useful in some organizations, it is unacceptable in others where by default, attachments should be view/read-only.

It is possible to change the authorization, but on a case by case, and it's way more complicated than it should be (e.g. it should be a one-click action.)

As it is, this default makes the new feature a pain to use.

And I'm quite sure we'll soon hear of horror stories from people who assumed they granted a read-only access to a document and discover when it is too late that the document was changed, perhaps erroneously, by a recipient.

What we need (yesterday) is a new option in Outlook 2016, to choose whether the default authorization for a linked attachment is View only or modify.

The AutoFit to Contents command clears any column and table widths that have been specified in the Table Properties dialog box, and it also selects "Automatically resize to fit contents" in the Table Options dialog box.
