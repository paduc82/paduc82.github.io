¡firedog say:

POP3/SMTP is the original way of conducting email, dating back thirty years. You have a mailbox on a server somewhere (in this case at outlook.com), and every so often your email program asks that server if there are any new messages. If there are, they are downloaded via POP3. That's it.

DeltaSync was introduced seven or eight years ago as a better way of conducting email. For a start, you can have many different folders in your account, and their contents will be kept in synchronization. This way, you can access the same account from different places and devices and always see the same messages. DeltaSync also allows for the synchronization of contacts lists and calendars between the mail program and the servers, if you sign in in Windows Live Mail.

solution_provider say: Just wanted to add to this post though already marked as answered. I was having identical issue with Win7 -> Win2008 TS via Remote Desktop. The cert on the TS server is issued from our internal CA (Certificate Authority) and with previous versions of Windows we would add our CA's cert to the clients Trusted Root Authorities. All similar to the original poster.

Unfortunately just dbl-clicking the CA cert would add it to the "users" list of Trusted Root Authorities and the "...revocation check..." error continued.

I found we needed to add the CA cert to the clients "local computer" list of Trusted Root Authorities by going through mmc.exe, then File | Add Snap-in | Certificates | Local Computer and importing the CA certificate to the Trusted Root Authorities area.

doopadoop say: Turns out the Taskbar Thumbnails Tuner link somebody else posted works by changing the following 2 registry keys:

HKEY_CURRENT_USER\Software\Microsoft\Windows\CurrentVersion\Explorer\Advanced
Create new DWORD: "ExtendedUIHoverTime" and set it to a really high number (it's in milliseconds) to delay the preview popup virtually forever (over 7 weeks, anyway). Apparently it doesn't like it if you set it to exactly 0xffffffff, I think it treats it as 0. But 0xfffffffe works just fine. Kind of weird.

and

HKEY_CURRENT_USER\Software\Microsoft\Windows\CurrentVersion\Explorer\Taskband
Create new DWORD: "NumThumbnails" and leave it at 0 to switch the previews to a list when you actually click on a stacked icon.

You'll need to restart Windows Explorer for it to take effect, easiest way is from the Task Manager.
