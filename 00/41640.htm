a_user_here say:

Just for your reference.  When the Windows 10 install/Media Creation Tool, asks for a product key it wants a Windows 10 product key, not your Windows 7 key, which is different.  No more than you could enter a Windows 7 product key into a Windows 8.1 install.

What is different about Windows 10, is the registering of the product key, connected to a device/Microsoft account.

When you did the update from Windows 7 to Windows 10 you had to do it from Windows 7.  That proved that your Windows 7 was properly licensed, so it did the upgrade.  When you logged in with your Microsoft account it registered the product key to that device.  So if you want to reinstall on that same device you simply skip entering the product key when asked, and when you get to the point where you log into your Microsoft account it will look up the device/product key and activate it.  Note if the device "changes" too much like say a motherboard is changed, this automatic activation isn't going to work, and you would have to fall back to the telephone activation or connecting with support.

Why two machines would have the same product key.  I don't have an idea.

Computers that are not updated frequently are not protected against the latest malware.  Microsoft releases major security updates on the second Tuesday of every month, so that is as often as most Windows users need to check for updates, but updates are often released daily for Microsoft's own anti-virus program, Windows Defender, so users of this anti-virus should check for updates daily.  This article will explain how to check for and install updates in Windows 8.1.


In my article about installing updates for Windows 8, I discussed installing updates through the Control Panel; however, I did not discuss using the PC Settings app because of its limitations.  Now that the PC Settings app has been updated to be more useful and accessible, I will show you how to use it to install updates.  I will not, however, discuss how to install updates through the Control Panel because that has not changed.  Please see my previous article for details on how to install updates through the Control Panel.

Thank you for being part of Windows 10.

The EFI System partition (ESP) is a partition on a data storage device (usually a hard disk drive or solid-state drive) that is used by computers adhering to the Unified Extensible Firmware Interface (UEFI). When a computer is powered up and booted, UEFI firmware loads files stored on the ESP to start installed operating systems and various utilities. An ESP needs to be formatted with a file system whose specification is based on the FAT file system and maintained as part of the UEFI specification; therefore, the file system specification is independent from the original FAT specification.

ESP contains the boot loader programs for all installed operating systems (which are contained in other partitions on the same or other storage device), device driver files for devices present in a computer that are used by the firmware at boot time, system utility programs that are intended to be run before an operating system is booted, and data files such as error logs.

So I would request you to not delete those partitions.
