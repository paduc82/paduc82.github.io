You can place two unrelated subforms on a main form. This is a nice method because it allows you to use the 2007 feature of Anchoring to adjust the size of the subforms based on the screen size. You can't reuse a calculated expression in another calculated expression, for one thing; for another (worse) you have a circular-logic situation: you're trying to decide which field to retrieve in each individual record based on a sum of the records retrieved! I think you'll need to do this as a two query situation: first create a Totals query summing the item counts, and then create a second query joining this query to your price table using the Switch() function to choose the correct price.
