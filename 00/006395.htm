rjm101 say:

SOLUTION THAT WORKED FOR ME

I had similar issues with Outlook 2013, Windows 8.1:  Initially the reminders popped up as usual, but I was unable to postpone, change or delete any of them. (The error message I received was: Cannot postpone/delete/etc. the reminder. The message you specified cannot be found.)  I followed the online advice to run the /cleanreminders command. After that, I got no pop-up reminders at all - even for newly created appointments.  In consideration of re-installing Outlook, I did the following, which magically fixed the problem:

1) In Calendar, I clicked on the FOLDER tab.

2) Under "Actions", I clicked on "Copy Calendar".

3) I pasted the copy into another user folder and labeled it Calendar Copy. (I didn't want to paste into any existing CALENDAR folders in case it would corrupt that data.)

As soon as the paste action completed, all of my reminders popped up and I was able to change, delete, postpone, etc. as usual. Hope it will work for you, too!

dl-ind say: [...   OneNote 2013 was changed so you don't have a vertical scroll bar at all until you've typed at least one page of material. If I need to insert space I can't do it until I've typed at least one page!   ...]


I use OneNote 2010. As far as I know, the display of the scroll-bar (vertical/harizontal) is based on the contents of the paper *and/or* the paper-size (auto or one of those paper-sizes).

 

OneNote displays the scroll bar, even for a blank page, if the paper-size is set and the screen is not enough to display the whole paper. If you zoom-out the page and the whole page can be displayed on the screen, the scroll-bars disappear. If it still appears, there must be something (text, image, etc) which position is out of the range of the current paper-size.

 

For an auto-size page, it depends on the content only. If the whole contents is already on the display, scroll-bar does not appear. If on this condition the vertical scroll-bar appeared, a user would think there was something down there s/he should see. That user would keep on scrolling-down the page and would see nothing and would be confused. So, for the blank page, since there is no content to be displayed, scroll-bar does not appear. A different behaviour of the scroll-bars, I think, should be provided via OneNote option.

 

This is similar to MS-Word behaviour concerning with a page in print-layout and web-layout. Many applications' scroll-bar behave that way too.

 


For the auto-size page, you can use the "Insert Space" (and put its button on QuickAccessToolbar). You can also make a template of a blank-page with, e.g., a dot far down the page.

 


I like reading your message since I can see some differences between the two versions from the real user.

joeu2004 say:

If A1 is 0.084403 and B1 is 218, then:

=ROUND(A1,2)*B1

or

=ROUND(ROUND(A1,2)*B1,2)

Use the latter to ensure the result is rounded to 2 decimal place, as well.

I then realized that the only records I needed were those with MemNum above 2198, so the simplest filter was the one that did the trick ... mostly.

This filter:
MemNum Greater than or equal to 2198
returned the records I expected, with one exception ...

a. user say:

1. Do you have any idea what changed?

2. As far as I know, if you connect to the Access database using DDE, this particular problem does not occur (although other query-related problems may arise). To connect using DDE you 
 - click Word Office Button->Word Options->Advanced
 - locate the General section (near the bottom)
 - check "Confirm file format conversion on open"
 - go through the connection process again. After you have selected the Access file name, you should see a "Confirm" dialog box, which may list a DDE option - if not, click "Show All".

It is possible that for some reason you used to connect via DDE and no longer do.

The other way to ensure that you do not get these extra criteria is to use VBA directly to set up the query. However, if your criteria change every time, If you are familiar enough with Access (Jet/ACE) SQL to be able to type the whole query each time, then you could consider having a piece of VBA that prompted for the SQL, then re-opened the data source. I'd rather not post sample code unless you think that approach might work for you.
