This is "aircode" off the top of my head and may not be exactly syntactically correct.

First I would create a form to send the e-mails from. On that form you would have a hidden textbox named OwnerID (or whatever your Owner primary key is named). Next you have a query that returns the info you want to include on your report. In that query you set the criteria for the OwnerID column to:
=Forms!formname!OwnerID
Then you have a report based on that query.

On the form you also have a button to trigger sending the e-mails, The code in the On Click event would look like this: 

Dim db As Database, rs as Recordset
Dim strSQL as String
Set db = CurrentDB()
'Create listing of email addresses
strSQL = "SELECT OwnerID, Email FROM tablename;"
Set rs = db.OpenRecordset(strSQL)
'Loop
With rs
     .MoveFirst
     Do While NOT .EOF
         'assign ownerID for current record
          Me.OwnerID = .Fields("OwnerID")
          DoCmd.SendObject acReport, "Reportname", acFormatPDF, .Fields("EMail"), , , "Renewal Notice", "Body of e-mail", False
          .MoveNext
     Loop
End With


This creates the recordset and loops through it. Each pass will set the textbox to the OwnerID of the current record. When the report is generated in the SendObject it will only include the data for that Owner. A PDF will be generated for that owner. The PDF will be attached to the e-mail and sent out. You can enter whatever text you want in the e-mail as Body of e-mail. Note: for testing you can change False to True at the end of the SendObject so you can see what the e-mail looks like Also remember to use your own object names.

Balloon Size in Track Changes - Microsoft Community

matu13 say:

Balloon Size in Track Changes

The balloon size for my track changes are microscopic!  How do I change the balloon size as well as the font inside the balloon?
suzanne s. barnhill say:

The size of the balloon is actually not the issue. It's the font size that's determining the balloon size. There's a bug in Word 2007/2010 that changes the definition of the Balloon Text (and sometimes Comment Text) paragraph styles, setting the font size to 1 point. You'll need to find and modify the Balloon Text style to correct this.

  1. Use Alt+Ctrl+Shift+S to open the Styles pane. If Balloon Text is listed, skip to step 7.
  2. In the bottom right corner, click Options...
  3. In the Style Pane Options dialog, note which setting is selected for "Select styles to show" (so that you can restore it, if desired); then change it to "All styles."
  4. Under "Select how the list is sorted," again note the current setting, then change it (if necessary) to Alphabetical.
  5. Click OK to close.
  6. Now search the styles list for Balloon Text.
  7. Right-click and choose Modify...
  8. Change the font size to 8 points (the default) or larger if desired.
  9. Select the radio button for "New documents based on this template."
  10. Click OK.

That should fix the issue. If there's still a problem with comments, repeat the exercise for the Comment Text style.
