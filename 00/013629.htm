IF I deposit $5,000 in an account that pays 8% interest per annum. How long will it take to double your money?

hansv mvp say:

The NPER function is used to calculate the number of periods. Its syntax is

=NPER(rate,pmt,pv,[fv],[type])

where:

rate = interest rate. In your example: 8%.

pmt = periodic payment. Here 0, since you don't deposit money periodically, only at the start.

pv = present value: the amount that you deposit. Since it's a negative cash flow (for you), we use -5000.

fv (optional) = future value. You want to receive twice the deposited amount, i.e. $10,000. Since this is a positive cash flow for you, we use 10000.

type (optional): we can ignore this here.

So the formula is

=NPER(8%,0,-5000,10000)

which returns 9.00468342, or almost exactly 9 (years).

joeu2004 say:
PS....  I wrote:
Looks like your trainer did this:
=12*RATE(30*12,-250,0,500000,0)
Note that the last parameter is 0, not 1 as I said.
Zero means that payments (deposits or payouts in this case) are at the end of the month.
It's a matter of interpretation.

Here is another way to look at it.  Construct the following accumulation tables:



A2:A361, B2:B361 and E2:E361 contain constants.
B1 is the formula =12*RATE(30*12,-250,0,500000,1).
E1 is the formula =12*RATE(30*12,-250,0,500000,0).

Notice that in both cases, the accumulation is indeed $500,000 in the 360th month (30 years).  The difference is how interest is calculated each month.

C2 is the formula =(C1+B2)*(1+$B$1/12), which is copied into C3:C361.  In other words, the deposit is added to the previous balance (at the beginning of the month), then interest is calculated on the total (at the end of the month).

F2 is the formula =F1*(1+$E$1/12)+E2, which is copied into F3:F361.  Thus, interest is calculated on the previous balance (at the end of the month), then the deposit is added (at the end of the month).

adam vero say:

Use conditional formatting - in 2007 and later you can specify a condition and for the cells that meet the condition you can tell it to use a different number format (unlike 2003 and earlier which was just font, fill, border formatting).

So you could get a number format that works for <503000... and >503999... and set that for all the cells in the appropriate range, then a conditional format using a formula to find cells between the values and set your alternative format instead.

This gets round the 3-condition liimtation built into the format which Ron correctly describes.

If you want some step by step instructions let us know but reading between the lines it sounds like you would be able to do this just fine.

adam vero say:
In Excel 2007, select the cells where you want this formatting to be used.
Set your original number format for all of these cells (as far as I can see, 000-00-00-000 should do the job).
Make sure you notice which of those is the active cell, highlighted in white rather than in colour (I'll assume you select these top to bottom so let's say you select B2:B100 and B2 is the active cell)
Now go to the Home Ribbon > conditional formatting > new rule.

Choose the last option "use a formula to determine which cells to format".
In the formula box, type this condition:
=AND(B2>=503000000, B2<=503999999)
Click on the "Format" button and set the format you want for cells which match this condition. In your case that means go to the "Number" tab, go to custom and put in the custom number format you need (###-##-##-## as far as I can see from reading  your question).
OK, OK to get out again.

As long as the cell reference you use is the cell in the selected range which was active, this will work. Basically cell B2 uses it's own value for the test, B3 uses B3 etc, very similar to how formulas update when you drag them down to fill, except there's no real way to "see" this in the condition, you just have to know and trust that it is working.
