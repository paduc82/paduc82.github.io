brian tillman [outlook mvp 2007-2019] say:

The fact that you see <someaddress> is not what makes the address invalid. Having a string enclosed in <> in a sender address is perfectly legitimate. Moreover, trying to block an address on a message already being delivered to the Junk Email folder is pointless because the message is already being "blocked" as far as Outlook.com is concerned. Blocking doesn't mean the message won't be delivered. It means it will be delivered to the Junk Email folder. Since that's already happening, there's nothing more you need to do.

Some examples of valid email addresses are:

"very.(),:;<>[]\".VERY.\"very@\\ \"very\".unusual"@strange.example.com

#!$%&'*+-/=?^_`{}|~@example.org
"()<>[]:,;@\\\"!#$%&'-/=?^_`{}| ~.a"@example.org

Note also that what you see in the From header isn't related in any way to the address that actually sent the mail, although legitimate mailers are kind enough to include the actual sender's address in the From. The mail systems spammers use are generally not kind.

stefan blom say:

By default, Word will redisplay tracked changes, in particular when you re-open the document after it has been closed.

You can control the behavior to a certain extent by clearing the option "Make hidden markup visible when opening or saving" in Trust Center. After you have done that, Word will save some of the view options for tracked changes with the document. Note, however, that the "Make hidden markup visible..." option does not travel with the document. In other words, if you send the document to someone else, they will see the markup.

To display Trust Center, first display the Word Options dialog box, and then click Trust Center | Trust Center Settings. Go to the Privacy Options category for the "Make hidden markup visible..." option.

Again, the only safe way to get rid of markup is to accept changes and turn off the tracking.
