As an example,  assume 2 rows in the database, with 2 columns:  item_name  and quantity.  If item 1 has a quantity of 4, I need to print 4 labels for it.  If item 2 has a quantity of 5, I need to print 5 labels for it. 

john w. vinson say:

Gina's code will certainly work, but there is a query approach too. It does require a small auxiliary table; I find this table useful in many contexts and routinely include it in my databases. I name the table Num; it has one long-integer field N as the primary key, and I fill it with values from 0 through 10000 or so (you won't need that many but it's still a tiny table).

You could base your Label report on your existing query; add the Num table to the query with NO join line at all. Put a criterion on N of

< [Quantity]

This will give you 4 records (with N equal to 0, 1, 2, and 3) if Quantity is 4. You can use this to advantage by including a textbox on the form with a control source

= [N] + 1

if you want to print out "3 of 4" on the label.

shinmila h say:

Hi imadden2000,

Welcome to Microsoft Answers.

Configure the following settings in Internet Explorer 8 (IE8) to make sure that user id and passwords are saved.

  1. Open IE 8.
  2. Go to Tools > Internet Options.
  3. Navigate to Content Tab.
  4. Click on settings button under AutoComplete.
  5. Put a checkmark on User names and passwords on forms.
  6. Also place a checkmark on Prompt me to save passwords.
  7. Click OK.
  8. On the General Tab make sure that the Delete browsing history on exit is unchecked.
  9. Click OK.
  10. Go to Privacy tab and drag the slider to the lowest so that it accepts all cookies from any website.

This should do the trick.

Most banking and other secure sites will not allow passwords to be saved. As such you may check on websites which have lesser security and see if user id and password get saved.
