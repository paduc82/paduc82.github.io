dhookom say: If you want to filter the subform to all combo boxes, you can use all combo boxes in the Link Master property and enter the matching fields in the Link Child property.

ken sheridan say:
And if it is a filter issue to the end, why not just define the criteria in the query underlying the subform itself to begin with

Why not indeed.  In fact this is the very method used by the form for 'drilling down' through a hierarchy in the demo file of the use of correlated combo boxes, ComboDemo.zip in my public databases folder at:

https://skydrive.live.com/?cid=44CC60D7FEA42912&id=44CC60D7FEA42912!169

If examining the RecordSource property of this form be sure to do so by zooming the property rather than opening the query in design view.  If you do the latter Access will move things around and the logic of the WHERE clause will be obscured.

Using a query with parameters like this generally gives much faster performance than referencing controls as the LinkMasterFields property of a subform.  In fact no subform is needed as the unbound combo box controls are simply placed in the header of the bound form.
