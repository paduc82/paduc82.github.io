ken sheridan say:

Yes, I will be doing this with single and continuous forms. Is there an example of what you describe somewhere, since my head exploded when I got to superimposed... :-)

I don't have an example of the use of hybrid controls in this particular context, though you'll find them used in another context in ComboDemo.zip in my public databases folder at:

https://onedrive.live.com/?cid=44CC60D7FEA42912&id=44CC60D7FEA42912!169

Note that if you are using an earlier version of Access you might find that the colour of some form objects such as buttons shows incorrectly and you will need to  amend the form design accordingly.  

If you have difficulty opening the link copy its text (NB, not the link location) and paste it into your browser's address bar.

In your case, if we assume the form is for inserting data into a table EmployeeProjects say, which includes an EmployeeID foreign key column you'd base the form on a query like this:

SELECT EmployeeProjects.*,
LastName & ", " & FirstName AS Employee
FROM EmployeeProjects INNER JOIN Employees
ON EmployeeProjects.EmployeeID = Employees.EmployeeID;

In the tables EmployeeID would be the primary key of Employees and a foreign key in EmployeeProjects, indexed in the latter non-uniquely (duplicates allowed). A relationship must exist between the tables, with referential integrity enforced.

The combo box's RowSource should be as I described, and remember to requery the combo box in the form's Current event procedure.

Bind the combo box to the EmployeeID column, and bind the text box to the computed Employee column.  Set the text box's Enabled property to False (No), and its Locked property to True (Yes).  Size the text box so it fits exactly over the combo box, leaving only the arrow of the latter visible to the right of the text box.  Position it carefully so it appears to be a single control.

When the user clicks the arrow to drop down the list they'll see the employees listed in the usual way and be able to select one.  The selected name will show in the read-only text box.  Unlike with the use of a single combo box control, however, the name of inactive employees in other rows will still be visible when an active employee is selected in the current row.
