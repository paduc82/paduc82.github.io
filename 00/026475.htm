davidf2 say:

If you want all pages combined to one file you will have to combine all four pages to one before you convert the HTML.

If you File > Export > Publish HTML the first option will give you an index.htm or the home page plus an index_files folder that contain the other three pages plus all the supporting graphics. Obviously you would have to link the pages before you convert to HTML and upload the multiple files.

You also have the option of File > Export > Publish HTML and the second option is to convert to a MHTML file which is a single file with the images embedded. But again you will have four pages or 4 MHTML files unless you combine all four pages before you convert them.

Web functionality was pretty much stripped from Publisher as of Pub 2010 but you can also do a file > save as and choose 'web page filtered' (the first option described above) or 'single file web page' or the MHTML option.

Bottom line is I would suggest you just upload your PDF file and link to it. Most browsers will automatically open the file or give you the option to download it. Even if you manage to create a web based file(s) to upload chances are you will have problems with formatting and layout and the PDF file will do the best job of maintaining those things.

Renu.

One of the reasons MSFT started deprecating the web functionality of Publisher was the difficulty in taking a print formatted document and converting it for the web. There are print formats and layouts that simply have no HTML equivalent. That is basically what you are experiencing with your jumbled web version. Those of us that used Publisher to build webs knew the limitations and what we could and could not do and were able to tweak the layout and formatting to produce cross browser compatible webs, but I think it is unrealistic for you to attempt this. Especially when you have a easier workaround.

I would suggest that you first make a copy of your Pub file to experiment with...if you don't like the results you can go back to your original. Then select the first picture and compress it and all the other pictures to 96 dpi which is equivalent to what you would get if you converted the Pub file to web format anyway. Then create your PDF and again choose the 96 dpi resolution/minimum size for the PDF. (When you get to the file save as a pdf screen look for the Optimize For under File Type and select Minimum Size which again sets the dpi at 96 dpi.)  Now check the file size. I am confident that you will find it to be a fraction of the size it was.

If you expect people to print this document and not just read it on-screen then test print the 96 dpi PDF version. If it is not adequate you can go back to your original Pub file, make a new copy and this time try opting for the print quality options at Desktop Printing quality. The file size will not be as small as the first that is optimized for the web but it is likely to be smaller than the original.

The bottom line is taking steps to minimize your PDF file size is going to reward you with better results than if you try to convert to HTML. Uploading a minimized PDF to your web host and providing a single link to it will be the easiest approach IMHO.

DavidF
