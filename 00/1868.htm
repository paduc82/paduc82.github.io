a. user say:

AFAIK the support for "long form" as well as "short form" citations was only ever in Mac Word 2008 and is in no other version.

Now that it appears possible to use custom "Referencing Styles" again in Mac Word 2016, at least in the Office 365 version of it, there is a possible workaround, but it's a bit messy to say the least.

How it works is that you install an extra Referencing Style (let's assume you're using Chicago and we call the new style "ChicagoX"). When you insert a citation using ChicagoX, it will insert the "long form." So the workflow looks like this:

In the Citations pane you normally have Chicago selected.

When you double-click on a Citation in that pane, word normally inserts an "SDT" (Structured Document Tag) that has some dropdown menu items and contains a field that inserts the short form citation.

However, when the insertion point is in a footnote and you double-click on a Citation in the Citation pane, Word inserts the resulting text, not the SDT+field.

So, when you want to insert a long-form citation in a footnote, you

 a. change the Referencing Style to ChicagoX

 b. with the insertion point in the footnote, double-click on the Citation in the Citation Pane

 c. word inserts the long-form Citation

 d. you change the Referencing Style back to Chicago.

Slightly disconcerting is that when you do (a), *all* existing citation SDTs in your document will display the long-form Citation as produced by ChicagoX (and any Bibliography may change, too). However, in step (d), they should all change back again.

I have created versions of the stylesheets for Chicago and Turabian based on the stylesheets from Word 2008, modified so that they should work in Word 2016. You can get them from https://goo.gl/bJXesR along with some instructions and further info., which you really should read before trying this. Here they are, with a couple of minor edits:

Instructions etc.

============

Introduction
============

The .zip file containing this readme.txt contains two Bibliographic Style sheets (or "Referencing Styles" sheets) intended to solve a problem with Mac Word 2016 described at https://answers.microsoft.com/en-us/msoffice/forum/msoffice_word-mso_mac/problem-with-citations-not-formatting-footnotes/55e2f8ec-e0b3-4e34-8d14-10a0720b211b 

The problem arises from the fact that Mac Word 2008 allowed two different Bibliographic citation formats for the same Referencing Style (e.g. "Chicago"). One was a short format for citations in the body text, and the other was a more complete version for citations inserted in Footnotes.

AFAIK only Mac Word 2008 had this feature - in all more recent versions on both Windows and Mac, only the short form was available. 

Mac Word 365 versus Mac Word 2016
=================================

As far as I know, there are actually two versions of Mac Word 2016 (this is certainly true of the Windows version). 

One is the one you get as a consequence of having an Office 365 subscription. This version is regularly updated with new features.

The other one is the one you can buy as part of a non-Office 365 subscription. The feature set for this one is (AFAIK) fixed (although there are doubtless periodic updates to fix security and other problems.

Early versions of the 365 version cannot use this workaround, because it requires you to copy a new Referencing Style into the application itself, and that used to cause Word to crash. However, that problem appears to have been fixed (I don't know when, but assuming you keep your copy of Office 365 up-to-date, you should be OK.

However, I do not have the non-365 version, s at the moment I cannot be sure that this workaround will work with it.

You will also need to use a MacOS login that gives you permissions to modify an application file (and indeed, anything in the Applications Folder). You will also need the username/password for that login.

The workaround
==============

One workaround is to create an additional Referencing Style (let's call one of the "ChicagoX") that inserts the long form citation. This is certainly not ideal, but in practice it seems to be workable. How it works, once the user has installed the extra styles, is:
 a. normally they work with the standard Referencing Style ("Chicago")
 b. when they want to insert a Footnote citation, they
 c. create the footnote
 d. change the Referencing style to "ChicagoX"
 e. double-click on the citation in the Citations Pane
 f. change the Referencing Style back to "Chicago"

There is one disconcerting side-effect, which is that in step (d), existing Citations in the body text (and possibly any existing Bibliography, will update to show the full format citation. However, when the user switches back in step (f), they change back again.

Luckily for this process, on Mac Word 2016, when you insert a Citation in a footnote, Word actually inserts the text. It doesn't insert the CITATION field code and the surrounding SDT. Otherwise, you would need a step before (f) where you right-click on the SDT options dropdown and select the option that converts the SDT to its resulting text. (The "SDT" is the boxy thing with a menu that surrounds the citation text. SDT stands for "Structured Document Tag", and SDTs are used to surround for other things in Word, such as Content Controls and Tables of Contents). I believe this extra step might be necessary on Windows Word 2016 as I think it inserts the SDT even in a footnote.

The two files in the .zip with this readme.txt are "X" versions of the Chicago and Turabian styles. They have been taken from Word 2008, modified to insert the long form instead the short form citation and in other ways needed to make them Work in Mac Word 2016

Probable Drawbacks
==================

I do not personally like having to modify *any* application file, but at present it seems to be the only workable approach.

Because the Style files (.xsl files) need to be inside the Microsoft Word program file on Mac, every time you install updates to Office you will probably need to re-install these custom .xsl files.

Going back to the 2008 Styles is not ideal. The .xsl Stylesheets may not work as well as the current ones, and the rules for what you are supposed to insert may have changed in Chicago/Turabian since the last updates in 2008. In particular, in later versions a lot more attention appears to have been paid to localisation issues (i.e. taking account of the installation/primary editing language of Word itself and the language of the citation.


Other info
==========

So far, I have only modified the .xsl field for Chicago and Turabian, because they were the ones mentioned in the original question. Word 2008 also had Styles for APA and MLA.

Considering that at least some of these Referencing Styles have changes since the 2008 version, it would undoubtedly be better either to start by modifying the 2016 .xsl files or producing new stylesheets from scratch (perhaps based on the work by Yves Dhondt)

However, reworking the 2016 versions of the Stylesheets would definitely require quite a bit of time and effort to understand exactly what needs to be changed, whereas modifying the 2008 .xsl files and verifying they work with 2016 took a few minutes.

Installation
============

1. Ensure you have two Finder Windows open (you can use Finder's File->New Finder Window menu option to open new ones.

2. Using one Finder Window, unzip the files in the .zip you downloaded, if that has not already been done. The files are:
  readme.txt
  CHICAGOx.XSL
  TURABIANx.XSL

3. I the Second Finder Window, make a backup of the Word Application file. It's quite a big file (2Gb) so if your disk is a bit full up, you may need to make some space.
 a. In Mac Finder, go to the Applications Folder.
 b. Right-click on the program - it will almost certainly be called "Microsoft Word" or "Microsoft Word.app", depending on your Finder preferences.
 c. Click "Duplicate"

The Finder should make a copy. It may take several minutes. 

4. Right-click on the program file "Microsoft Word", click "Show Package Contents". This reveals the file/folder structure within the Application's file, starting with the top -level folder "Contents".

5. Double-click "Resources, then locate the "Style" folder and double-click it

6. Select the CHICAGOx.XSL file in the First Finder Window, and copy it - e.g. command-C, or right-click and click "Copy "CHICAGOx.XSL""

7. Right-click in the Second Finder Window, and paste the item, e.g. via command-V, or right-click "Paste Item". If you can't see that option, you should be able to use the Finder's Edit->Paste menu option

8. At this point, if you are doing this for a second time, you will see a dialog box that says "An item names "CHICAGOx.XSL" already Exists in this location". Click "Replace" 

9. You should now see Finder's security dialog ("Finder wants to make changes"), where you will need to enter the password and/or username+password needed to make changes to the application folder/file. Enter that info. and Click OK.

10. Repeat steps 6 to 9 for TURABIANx.XSL.

11. You can now close the second finder window.

12. Start Word. If it crashes, this isn't going to work and you will need to restore the old version. You may be able to "back out" by repeating steps 4 and 5 above, then selecting the new .xsl files, right-clicking and clicking "Move to Trash". But if Word crashes after that, you should rename Microsoft Word (and eventually move it to trash), then rename the copy you created in step 3 to "Microsoft Word"

Now make a backup copy of the Word document you want to edit, and try out steps (a) to (f) in "The Workaround"

What other approaches might be possible?
========================================

On Windows Word, it is possible to access the master bibliography data )held in a separate file called sources.xml) and the bibliography data held in each document. So in that case, it might be possible to write VBA to construct the footnote entry you need. It's also fairly easy for VBA to get the data directly from sources.xml and the document's bibliographic data store (it's held in a thing called a "Custom XML Part"). Those VBA features are not yet in Mac Word 2016, but as/when they are added, it may be possible to create workable solutions. 

At present, the only options I can see on Mac Word might be to use 
 - VBA or Applescript to read the sources.xml file, allow selection of an entry in there and insert some formatted text. The "only" problem with that is that the sources.xml does not necessarily contain the bibliographic data that you have in the document (e.g. it won't if someone sent you a document), and it also might have an awful lot more than you need, making it harder to find the entries you need.
 - the JavaScript APIs to access that Custom XML Part. In my view, that's a "whole other ball game", but I mention it for anyone who wants to have a go.

john w. vinson say:

Again:

A Criterion simply controls which existing records you see, or don't see, when viewing an existing Table or Query.

A Criterion has absolutely nothing to do with preventing duplicate records from being created. You're just looking in the wrong place!

If you don't want records with duplicate values of AccountNumber and DateOfService to be created in the first place, create a new Index on the table using the Indexes tool. Give the index some distinctive name by typing the name in the left column of the little grid that the Indexes tool will pop up: NoDupServices say. Put the fieldname AccountNumber next to it in the right column, and DateOfService in the right column in the next row, right under AccountNumber. Check the "Unique" checkbox and save the index.

It won't save the index if there are already existing record which are duplicates. You'll need to run the "Duplicate Records" query wizard to find these, and manually delete them.

If you already have duplicates, what exactly do you want to happen? Do you just want them to be kept in the table, but suppressed from view? That seems wrongheaded - sweeping problems under the carpet! - but you can indeed do so as suggested above, using a totals query.
