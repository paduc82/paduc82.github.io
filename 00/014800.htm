hansv mvp say:

One option is to add a rule Format only cells that contain > Cell Value, equal to, =""

(not =" " with a space between the quotes, but ="" with nothing between the quotes).

Set the format to the standard formatting, and make sure that this rule is listed first.

 

Another option is to edit the rule with Cell Value, less than, =TODAY()-120.

Select "Use a formula to determine which cells to format".

Enter the formula =AND(B2<>"", B2<TODAY()-120) where B2 is the active cell within the selection (i.e. change B2 to the address of the active cell).

Specify yellow formatting as before.

stephen boots say:

I can't tell you why that single update failed in Windows Update history, but the fact that subsequent updates for MSE have been applied mean that all is well. Updates offered in WU are *optional*  for MSE. MSE checks for updates once per day. Signatures are released multiple times per day and a check for updates by WU (or via Automatic Updates at the scheduled time) may present multiple updates available since the last update internally performed by MSE. When MSE updates internally, it updates cumulatively, so that error listing should be ignored as MSE is still updating fine.

 

The manual update at the bolded link from the portal is the *FULL* definitions/signature database as well as the scanning engine. Updates performed by MSE or via WU/AU are smaller "delta" signature files.

 

Update FAQ - http://answers.microsoft.com/en-us/protect/forum/protect_updating/mse-definitionssignatures-update-faq/74e507b8-f6da-4eca-8ce7-d1aca7d3f1ba
