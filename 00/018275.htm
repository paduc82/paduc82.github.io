don cuthbert say:

For the drive is locked error, you just need to rebuild the System partition. Here is a link to a video about it. The video is a bit long and uses more manual steps than is needed, as he describes what does not work, etc. but the point is that it can be repaired.

https://youtu.be/kyBrjDpsXGc

 

Assuming your drive is initialized as GPT (it would have a FAT32 System partition) the steps to fix it are to delete and recreate the System partition, format it as FAT32 and then apply the System files and BCD (Boot Configuration Database) to it. The easiest way to get a Command Prompt is to boot with Windows 10 install media on USB. From the first setup screen, use SHIFT-F10 to get a Command Prompt. Then follow the steps below:

run diskpart

select disk 0

list partition

select partition 2 <-- select the FAT32 partition number

delete partition override

create partition EFI

format quick fs=FAT32 label=SYSTEM

assign letter=S

list volume

exit

Then run bootsect and bcdboot:

bootsect /nt60 S: /force <-- updates the master boot code for hard disk partitions - restores the boot sector on your computer

bcdboot C:\Windows /s S: /f UEFI <-- copies critical boot files to the system partition and creates a new system BCD store

If your disk is an MBR disk, you could do similar, but you normally have to move the Windows RE file from the System/Recovery partition before deleting it.

My tuppence worth on Windows Mobile Build 10166, accessibility, etc - Microsoft Community

brianlinux say:

My tuppence worth on Windows Mobile Build 10166, accessibility, etc

It is much better than previous builds. I tested them from the outset on my Lumia 735.

There are a couple of anomalies and design weaknesses in Build 10166, which should be remedied.

1. Calendar

It has a daily and agenda view, but really needs a week and monthly too. These are essential in business

2. Settings

Searching for font size or similar keywords in the settings search bar produced no results. Whereas they should point to the precise place where you can make font adjustments.

3. Accessibility

This is a more serious problem. In short, if you adjust the font size to very large it produces inconsistency in applications.

Objectively, this problem can be reproduced via:

1. Install Windows Mobile build 10166 on a Lumia 735
2. Ensure the font size is set to default
3. Bring up the calendar app, enter a few mock appointments.
4. Go into the agenda view
5. The text will appear sized correctly.
6. Return to the settings menu
7. Ease of Access ->More options->Text scaling
8. Adjust to largest size
9. Bring up the calendar app again
10. Go into the agenda view
11. Text on subsequent date labels will be malformed, displaying only the top half of such fonts.

Accessibility is very important in mobile devices. The ability to change the font size and have it be reflected across applications is essential.
