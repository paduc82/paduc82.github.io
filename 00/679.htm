Hello,

Excel is primarily a spreadsheet application, not a drawing application, but it still offers the possibility to create drawings. Therefore, don't expect all aspects of a drawing app.

If you insert several objects, group them, then want to move the group, it is entirely possible, but you simply need to pay attention to what you have selected. With the group selected, there will be only one bounding box and the whole group will move when dragged. 

Excel offers the option to change aspects of objects inside a group without ungrouping first. This additional functionality can only be achieved by allowing the selection of objects within a group. 

So, basically, the answer is: no, you cannot prevent that. It's a learning thing. Just make sure that you select only what  you want to move.

Sounds like there is an issue with the .ost file that would be associated with the AOL account. Uninstall/reinstalling or repairing are very highly unlikely to have any impact on the problem.

Suggestion would be the following

#1 - Open the Control Panel <Mail> app
#2 - Click on <Show Profiles> --> <Add> to create a new profile (no need to delete the existing Outlook profile at this point)
#3 - Configure your AOL email account - if you use more then one email account, add those after confirming that the issue doesn't involve the AOL account just to make things easier to isolate if required
#4 - When done with #3 - be sure to set the new Outlook profile as the "default" for the next time you open Outlook

If you're still having the issue, other possibilities will need to be explored. However, there is one caveat that you need to be aware of which involves contact/calendar data.

With an IMAP account, contact/calendar data is stored in folders with the words "This Computer Only" in the folder description. Translated, that means the data contained in those folders is "local only" and not sync'd with the server. Going through the above exercise means that a new .ost file will be created, the email folders will be re-sync'd BUT the contact/calendar data contained in the "This Computer Only" folders will be lost so you will need to recreate that data either from a file you can import or manually adding the info. There is a way to mitigate this from occurring in the future which I'll provide you with once you respond with the results of the suggested steps.

karl timmermans (outlook mvp 2012-2018) say:
>>
How to get data from the original profile is now the challenge. Can you help me?
<<

Unfortunately, under normal scenarios, the only way to recover data from an OST file without Outlook is to use a 3rd party OST2PST conversion utility. This only applies to contact/calendar data since the email folders will have been sync'd with the server.

One thing that you may want to try is running ScanPST on the .ost file involved with the problem profile. To find which file that is exactly, open the Control Panel <Mail> app --> open the old (problem) profile --> switch to the Data Files tab --> take note of the folder path/file name of the .ost file. When you run ScanPST, be sure to select that file. You may need to run ScanPST multiple times to eliminate all potential errors.

When you're done with the above, go back to the Control Panel <Mail> app and select the old Outlook profile as the default, then open Outlook to see if the problem still occurs. if it does, the conversion utility root is the only option.

Now - how to prevent the "loss of data" issue from occurring in the future

#1 - Open Outlook with whatever profile works
#2 - Add a new PST file to the profile by going to the <Home> ribbon --> <New Items> --> <More Items> --> Outlook data file
#3 - Then go to File --> Account Settings --> Data File tab --> Select the newly added PST file --> set it as the <default data file> for the profile. This is where your default contact/calendar folders will be stored
#4 - If the issue with the OST file was fixed via ScanPST and you opened Outlook using the original profile, then the next steps would be to copy the data from your contact/calendar "this computer only" folders to the corresponding folders in the PST file

Copying the contact data
#5 - Select the contact folder
#6 - press CTRL+A to select all items
#7 - Right-click on the selected items --> select <Move> --> select <Copy to Folder> --> select the destination folder

Copying the calendar data
#8 - Select the calendar folder
#9 - Go to the <View> ribbon tab --> <change view> --> select <List>
#10 - perform steps #6 & 7

From this point, your default contact/calendar data will be stored in the PST file which never gets deleted by Outlook should you remove any email account from the profile. The "default data file" has no impact on the "default email account" which are two separate and distinct things.
