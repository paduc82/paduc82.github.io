I had the same issue. This process is fairly complex so I'll be specific. I encountered this issue after upgrading from 8.1 to 10, by the way, so I have the Windows.old file hanging around. This will probably still work, but I'm just putting it out there our scenarios aren't necessarily identical.

1. Go to "File Explorer"

   a. If you do not have hidden files revealed, do the following steps, otherwise continue.

   b. Go to "View"

   c. Check "Hidden Items"

2. Go to C:\ProgramData\Microsoft\Crypto\RSA\

3. Rename the "MachineKeys" folder to "MachineKeys-old"

4. Create new folder "MachineKeys"

5. Right click "MachineKeys", select "Properties", then select the "Security" tab.

6. Select "Everyone" and give them the permission "Full Control"

7. Go to Control Panel, Administrative Tools, then Services

8. Start any of the following if they aren't active: "Peer Networking Identity Manager", "Peer Name Resolution Protocol", "Peer Networking Grouping", and "PNRP Machine Name Publication Service"

Plan B:

If any of those now give you a different error- 1079, I believe- here is the process to fix it.

1. Select Properties for the Service in question

2. Hit the LogOn tab, where you will edit its permissions

3.Select the "This account" button.

4. Enter "Local Service" as the account name.

5. Erase the password fields.

6. Apply, you will get a dialogue box saying an account has been given permission.

7. Start the service.

Method 1: Please follow the article mentioned below:
Open the Printer troubleshooter:


http://windows.microsoft.com/en-US/windows7/Open-the-Printer-troubleshooter

 

Method 2:

 

In order to get the printing services back up and running, follow these steps:

a. Go to Start, Control Panel and Administrative Tools. Double click on Services icon.

b. Scroll down to the Print Spooler service and right click on it and select Stop. In order to do this, you need to be logged in as Administrator. At this point, no one will be able to print anything on any of the printers that are being hosted.

c. Next you need to go to the following directory: C:\WINDOWS\System32\spool\PRINTERS. Delete all the files in this folder. d. Now you can go back to the Services console and right-click and choose Start for the Print Spooler service.
