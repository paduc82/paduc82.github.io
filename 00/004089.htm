Do NOT show a records in query if... - Microsoft Community

corptkm say:

Do NOT show a records in query if...

I am trying to find a way to show all records in a query except for the records if they they meet these requirements.. In other words...don't show these records in the query results if the condition is true.

The method trying to use iin expression is..

If field "Milestone group" = "AD *" and "Asset qualification status "= "Exception in effect" then don't show the record in the query.

Is this the best way to handle this?

tom van stiphout say:

Is the group name literally "AD *"? More likely you meant "*" as a wildcard, which means that you are looking for all names starting with "AD " (note the space). If so, you have to use "LIKE", and since you have reverse logic it should be "NOT LIKE".

"Asset qualification status " is a very bad field name. Trailing spaces will give you lots of grief. Drop them. Also multi-word field names give notational difficulties. Change to "AssetQualificationStatus".

Since you want reverse logic you can use:
AssetQualificationStatus<>"Exception in effect"

corptkm say:

Tom,

Thank you for advice. I agree about table name it is bad choice and trying to get them changed in SQL as they are linked to my Access DB.  I was lucky enough to pick this up from another developer.

I tried using the method above in my criteria and it eliminates more than half my records that are suppose to be in the results. The issues is. There are records with "AD *" in the "milestone Group" that should remain as well as the  "Asset qualification status" field. I'm trying to eliminate (not show) the records that have "Exception in effect" in the "AssetQualificationStatus" field AND that have/match  "AD *"  in the "Milestone group" field. It should remove between 3-5 records not 500+.

Hopefully this clears it up a little? Thanks again for your advice.

ken sheridan say: The query's WHERE clause would be along these lines:

WHERE NOT([Milestone Group] LIKE "AD *"
AND [AssetQualificationStatus] = "Exception in effect")

Note how the AND operation is parenthesised so that the NOT operator operates on the result of the parenthesised expression.  I'm assuming in the above that the asterisk is a wildcard character for pattern matching rather than a literal character.  If it is in fact a literal character, then the WHERE clause would be:

WHERE NOT([Milestone Group] =  "AD *"
AND [AssetQualificationStatus] = "Exception in effect")
