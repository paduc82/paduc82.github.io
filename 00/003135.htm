don cuthbert say:

Windows has to be installed completely before you will get a Safe Mode option.

You can switch your drive letters after booting from a Windows install disk or USB by using Diskpart at the Command prompt.

You may also get the option to rollback to Build 10122, where you can create a Build 10130 ISO and try an install from within Windows.

In Diskpart you can list volume to see what the current drive letters are, remove any or all of the drive letter assignments and then reassign them so that C: is on your Windows volume, etc.  As long as you keep track of where Windows is, this is usually not necessary to fix things, but it can save on confusion.

Another command to rebuild your BCD is:

C:\Windows\System32\bcdboot C:\Windows

You can also type bcdedit to see if the BCD looks right to find C:\Windows in the partition it is on.

Another boot repair command sometimes needed:

bootsect /nt60 ALL /force /mbr

Validation Rule & Text for List boxes Access 2010 - Microsoft Community

kelmelphil say:

Validation Rule & Text for List boxes Access 2010

I've created a form for users to complete records with.

One of the fields asks the user if the equipment they are using is withing it's calibration date.

I've used a list box and the options are Yes or No.

If the user selects No I would like a message to appear explaining something like 'contact supervisor immediately etc' preventing the user from continuing any further.

I'm self taught in Access and still learning. Can anybody help me out?

 

djsteele, mvp say:
Associate code with the AfterUpdate event of the list box. For instance, if your list box was named lstWithinCalibration, you could use something like the following VBA:

Private Sub lstWithinCalibration_AfterUpdate()
 
  If Me!lstWithinCalibration = "No" Then
    MsgBox "Contact supervisor immediately."
    DoCmd.Close acForm, Me.Name
  End If
 
End Sub

kelmelphil say:

Okay, so now I feel a bit silly because I've discovered I'm actually not talking about a list box but I'm using the Lookup Wizard in a table with options yes or no.

Can I still create a custom message to appear when no is selected? I assumed that's what the validation rules and text would help with but I can't find basic answers anywhere.

 

djsteele, mvp say:

I would advise extremely strongly against using the Lookup Wizard. (See The Evil of Lookup Fields in Tables at "The Access Web" for some of the many reasons why.)

 

Short answer to your question is no, you can't do any kind of editing if you're working directly against tables, and you can't use lookup fields in tables on forms.

ken sheridan say: There doesn't seem to be a lot of point using a referenced table for the two values Yes and No when Access includes a Boolean (Yes/No) data type.  Just make the column in the referencing table this data type, and you only have the two choices.  Internally Access stores the values as -1 or 0, which correspond to the Boolean constants TRUE or FALSE, but can be formatted in a variety of ways, including Yes or No.

Usually the control used for a Boolean column is a check box, but to make the choice more obvious to the user you could use an option group with two options labelled Yes and No, and with underlying values of -1 and 0 (the control wizard can build this for you).  This does assume data entry is via a form, but that's a given; data should only ever be entered via forms, never in a table's raw datasheet view.

If we assume the option group control is named optWithinCalibration you can then put code similar to Doug's in its AfterUpdate event procedure:
 
  If Not Me.optWithinCalibration Then
    MsgBox "Contact supervisor immediately.", vbExclamation, "Warning"
    Me.Undo
  End If

This will pop up the message and undo any edits to the form if the user selects No.
