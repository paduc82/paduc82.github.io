This isn't too hard to fix, but one method requires a regedit. There are some automated regfix files that are supposed to restore the file extention defaults but the only feedback I've gotten was one person that stated it didn't work. I have a followup question on how they tried the fix, but so far no response so we won't even go there for now.

First, since the default action is caused by a user choice (inadvertant as it was), you may be able to restore simply by creating a new user account. If you use this method, don't forget to copy the hidden appdata folder. Just unhide everything before you start the copy.

The other method that requires a regedit is also fairly simple, but recommend you set a manual restore point before starting. A regedit always has the potential to mess up your system. procedures are:

  1. Click Start Menu, and then type regedit in the Start Search and press ENTER.
  2. Navigate to the following registry branch:

HKEY_CURRENT_USER\Software\Microsoft\Windows\Curre ntVersion\Explorer\FileExts\[ext]

Replace the [ext] with the actual file extension that you want to restore its file type association to revert back to original Windows 7 default (probably .lnk). If you unsure, simply browse through all the sub-key under FileExts.

  1. Delete the sub-key named UserChoice.
  2. Exit from Registry Editor.

    Theoretically, either method will work.

That's the thing.  A "bad sector" is either just a bad sector, treated like you say, or it indicates controller problems which, while it could be overheating or a loose wire or wonky power supply or something else easily fixable, could also be indicative of a hard-drive failure.

If you have install media on hand you could try the sfc using that as image (ie: where to get the file from) which will put the resolution of the problem off a bit. 

It might also be useful to run chkdsk from an off-disk boot (remembering that the target disk might not be "c:").

Meanwhile another standard sfc wouldn't be out of place : sometimes it takes a few.

hairyfool say:

What you should be aware of is that the hard drive controller has been doing exactly that. The drive is manufactured with unused sectors on the platters and if any used sector become difficult to read a spare sector can be mapped into its place.

There are a few reasons why they don't all get used but if the operating system actually "sees" a bad block it signifies that all possible spares have been used. In other words the disk may be in a worse condition than just the reported bad block.

Mapping out bad sectors manually was possible some years ago,  disks now do it automatically.

Alice, are you having any specific issues related to having two drives/partitions? For various reasons it is actually recommended to have a separate partition for your data, instead of having just a single one for both your data and Windows.

If you really want to combine them regardless you will need to obtain the help of someone knowledgable, otherwise you run the risk of losing all your existing data too.

As for your lost desktop icons/files/folders, it's difficult to suggest anything without knowing what exactly the technician did. They may simply have been moved somewhere temporarily, and perhaps the technician forgot to put them back. Or they may have been deleted, in which case you'll have to use data recovery software but there's no guarantee you'll recover everything. But first, any way you can ask the technician to list all the changes made to your system?
