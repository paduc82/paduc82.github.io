tom van stiphout say:
With drop down menu, do you mean combobox? Maybe you need to remove the ControlSource property? Hard to know without more specifics.

scottgem say:
You really need to define this in more detail. A combobox (sometimes referred to as a drop down) a control that can interface with data. If the ControlSource of the combo is set to a field in the form's Recordsource, then any change in the value of the combo will be reflected in the table. So if you open a new form to the same record, that data will be shown. 

bethanywatts say: Yeah I mean combobox sorry, please can you tell me how to remove the ControlSource property?

scottgem say:
just delete it from the Properties dialog

tom van stiphout say:

Design the form. Right-click the combobox. Select Properties. In the Properties window find the ControlSource property and wipe out any value that was there.

ken sheridan say: It sounds more like the opposite to me, and your control is already unbound, i.e. has an empty ControlSource property.  If you select a value in an unbound combo box and then move to another record in a bound form the combo box will retain the selected value.  If the combo box is bound, i.e. has the name of a column (field) from the form's underlying table or query as its ControlSource property, then the value will change to whatever the value of the column is in the record you move to, or will be empty if it's a new record.

So is this a bound control for inserting a value into a column in the table, or is it an unbound control for some task or other?  If the latter then you'll need to set its value to Null in the form's Current event procedure to clear it.

niton_ say:

Option Explicit

Private Sub AttachmentNameAsSubject

Dim AttachmentName As String
Dim currItem As object

Set currItem = ActiveInspector.CurrentItem

With currItem

    If .Attachments.Count > 0 Then

        AttachmentName = .Attachments.Item(1).DisplayName

        .Subject = AttachmentName

    End If

End With

End Sub

hi jim,

from your post, i'd like to confirm if the situation is when you click the new button on the prompted sites page in sharepoint online, you receive the following error message: you are not allowed to create team sites at yourdomain.sharepoint.com/.

if so, generally, the issue is related to permissions, an administrator needs to assign the create subsites permissions to the users who need permission to use the "new site" link. here are the steps below: 

1. create a new permission level:
click the gear icon for the tools menu, and then click site settings.
click site permissions.
on the ribbon, click permission levels, and then click add a permission level.
enter a name for the permission level, and then in the site permissions section, select the create subsites check box.
click create.

2. create a group that uses the new permission level:
click the gear icon for the tools menu, and then click site settings.
click site permissions, and then click create group.
provide a name for the group, and then in the give group permission to this site section, select the check box next to the name of the new permission level that you created.
note there are also other settings available in this section. select any additional settings as appropriate for your particular scenario.
click create.

3. add and remove users from the new group in order to grant or deny the permissions that you've configured.

if the issue persists, please provide the screenshot of the information below for our investigation:
1). check if he/she has been granted the correct permission. here are the steps to check a userâ€™s permission:
   a. click sites button on the upper side > click the gear icon > click site settings.
   b. click site permissions under users and permissions section.
   c. click check permissions > type in the user name in the box > click check now.

2). a screenshot of the start a site setting in sharepoint admin center.
     

to upload screenshots, click use rich formatting in this thread and select insert/edit media.

Hi max

The key to the whole mess is to delete the entry before it rolls u back to the previous build.

Once u delete the roll back entry in the boot menu the system will then boot fine from then on until the next

build comes out.Then u have to do the Same thing all over until  the issue gets fixed.

You can alter the boot menu using Msconfig U DON'T NEED A THIRD  PARTY UTILITY. I use Easybcd because I like it & it's free.When u load the build check boot menu each time u boot then remove (delete) the roll back entry.

If u don't delete the roll back entry u will be rolled back to the previous build & end up having to make an ISO file then burning to a disk & doing a clean install.

After u load the build check the boot menu after each cold boot. I have found on my machine it takes two cold boots before entry appears in boot menu.

Hope this helps, I had to do this for the last 5 builds until it gets fixed.....drove me crazy trying to figure out what was rolling me back after build installed fine. The key again is to delete the roll back entry before it has a chance to roll u back. It also changes the time out from 30secs to O (zero) so change it back to 30secs when u edit the dual boot menu so you can see the dual boot menu when u reboot.

Good luck...
