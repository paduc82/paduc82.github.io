    Dim vLinks As Variant
    Dim lLink As Long

    ' Define variable as an Excel link type.
    vLinks = ActiveWorkbook.LinkSources(Type:=xlLinkTypeExcelLinks)
    If vLinks = vbNullString Then Exit Sub

    For lLink = LBound(vLinks) To UBound(vLinks)

                Name:=vLinks(lLink), _

    Next lLink

daniel g. grau say:
A Save As to perform a back-and-forth conversion is only required once, on the chance that the document will be fixed by Word when performing a change in file format. The first Save As would do that. The second, from that re-saved file, would be done to put the document back into its original format. In fact, you may find that going back to the original format causes the original problems to resurface, as they might be related to the document's format.

Word builds the TOC usually based on style names (by default these are Heading 1, Heading 2, etc.). The problem is that it only searches for those styles in the main document flow, not in floating shapes. Therefore, even though a floating caption has the proper style (say Heading 3), Word won't see it when preparing the TOC (or a Table of Figures). That's why it is advised that whatever you want to appear in the TOC not be placed inside a floating shape, but instead "inline," i.e., as part of the main document flow.

As an alternative, you can duplicate the caption's text within the main flow, anywhere close to the original figure, then format it as Hidden text (not sure about this option) or in white, so it won't visually show but nevertheless be inducted into the TOC.

AutoRecover will look for specially-named files in a reserved folder. If it finds them, it will offer to recover the file. You may have some leftovers from older activations of the function.
