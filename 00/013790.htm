=IFERROR(INDEX($B$2:$B$50&"?"&$E$2:$E$50, MATCH(0,COUNTIF($G$1:$G1,$B$2:$B$50&"?"&$E$2:$E$50)+IF(COUNTIFS($B$2:$B$50,$B$2:$B$50,$E$2:$E$50,$E$2:$E$50)>1, 0, 1), 0)), "")

=IF(G2="","",LEFT(G2,FIND("?",G2)-1))

Enter below formula in cell I2 & copy down:

=IF(G2="","",RIGHT(G2,LEN(G2)-FIND("?",G2)))

weighted average of filtered data and excluding blanks - Microsoft Community

orfishome say:

weighted average of filtered data and excluding blanks

I have a large spreadsheet with three columns of pertinent data.  Column A is a loan type (text) , column B is a loan balance ($), and column C is a borrower debt service coverage ratio (#.##).  Data begins in row 3 and continues through row 748.  Columns A and B are fully populated; however, only some of the rows in column C have an entry.  I need to be able to filter column A to show only a certain loan type and then calculate a weighted average debt service coverage ratio using columns B and C.  The weighted average should only consider those rows with a value in column C.  I've tried using sumproduct and subtotal(109), but that doesn't seem to exclude the rows in column C that are blank.  Treating those rows as zero values skews the average.  Can anyone help?  Thanks so much.

 

ORFisHome

mike h.. say:

Scott,

 

I think we have different understanding of what we're trying to calculate here.

 

This is the way I understand you want to do it

 

Column A is simply text and is a flag for the rows we want to include.

Column B are numbers we want the average of

Column C is the weighting we apply to get that average.

 

If that's the correct way around then the formula I gave you is correct. For the data set below the weighted average of the 2 relevant rows in Col B is 15. If we add a 4 to row 2 then the weighted average becomes 17.

 

=SUMPRODUCT((A1:A4="aaa")*(B1:B4)*(C1:C4))/SUMPRODUCT((A1:A4="aaa")*(C1:C4))

 

 

However, You seem to be saying we are averaging column C and column B is the weighting and if that's correct then we need a different formula. We need this.

 

 

=SUMPRODUCT((A1:A4=E1)*(B1:B4)*(C1:C4))/SUMPRODUCT((A1:A4=E1)*(B1:B4)*(C1:C4<>""))

 

 

For the data below this produces a weighted average of 5. because both numbers in Col C are the same so the weighting is not relevant. If you add a 4 to row 2 the weighted average then becomes 4.576923. You can demonstrate the weighting is working by putting the 4 in row 2 and getting 4.576923 and then putting a zero against that in Col B. Because the weighting of 4 is now zero the weighted average goes back to 5.

 

If this doesn't work then please tell us what your averaging and which column is the weighting.

How to set tabs precisely in Word 2010? - Microsoft Community

ellis neuman say:

How to set tabs precisely in Word 2010?

In Word 2002, you could just click Format | Tabs to precisely (by numerical value) set tabs, rather than do it imprecisely by clicking on an

approximate desired spot on the ruler.  Can anyone tell me how to do this in Word 2010?  Thanks.

jay freedman say:
Right-click the paragraph where you want to set the tabs, and click Paragraph (or click the little arrow in the bottom right corner of the Paragraph group on the Home tab). In the bottom left corner of the Paragraph dialog, click the Tabs button to open the same dialog as in previous versions.

You can also get to the Tabs dialog by double-clicking on the ruler, but that also sets a tab at the place where you double-clicked, which you might not want.
