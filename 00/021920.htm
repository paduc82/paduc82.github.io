Additional information about the Archive folder are enumerated below:

 

 - Archived emails are counted in mailbox storage

 - Archived folder expires (This means that if your account goes into inactivity, you will still lose the emails inside the archived folder)

 - Archived emails are viewable

 - We can delete emails directly from the Archive folder

 - Archived emails can also be seen if account is configured to an email client using a protocol that supports folders

 - You can set up a scheduled sweep when archiving emails

 - Emails on any folder can be archived

 - Messages are filtered as they come into the mailbox. So emails are already filtered before you archive your emails

 - You can create another folder on the Archived folder

    Exit Sub

  If All.Rows.Count = 1 Then
    MsgBox "No visible data, aborted", vbExclamation
    Exit Sub
  End If
  Set All = All.Resize(All.Rows.Count - 1).Offset(1)
  'We want the visible cells only in case of an existing filter
  Set All = SpecialCells(All, xlCellTypeVisible)
  If All Is Nothing Then
    MsgBox "No visible data, aborted", vbExclamation
    Exit Sub
  End If
  'Get the unique items
  Items = UniqueItems(All, vbTextCompare)
 
  'Save the current filter settings
  SaveView
  'Screen off
  Application.ScreenUpdating = False
  Application.DisplayAlerts = False
 
  For Each Item In Items
    'Filter each item
    A.Range.AutoFilter Key.Column - A.Range.Column + 1, Item

Private Function UniqueItems(ByVal r As Range, _
    Optional ByVal Compare As VbCompareMethod = vbBinaryCompare, _
    Optional ByRef Count) As Variant
  'Return an array with all unique values in R
  '  and the number of occurrences in Count
  Dim Area As Range, Data
  Dim i As Long, j As Long
  Dim Dict As Object 'Scripting.Dictionary
  Set r = Intersect(r.Parent.UsedRange, r)
  If r Is Nothing Then
    UniqueItems = Array()
    Exit Function
  End If
  Set Dict = CreateObject("Scripting.Dictionary")
  Dict.CompareMode = Compare
  For Each Area In r.Areas
    Data = Area
    If IsArray(Data) Then
      For i = 1 To UBound(Data)
        For j = 1 To UBound(Data, 2)
          If Not Dict.Exists(Data(i, j)) Then
            Dict.Add Data(i, j), 1
          Else
            Dict(Data(i, j)) = Dict(Data(i, j)) + 1
          End If
        Next
      Next
    Else
      If Not Dict.Exists(Data) Then
        Dict.Add Data, 1
      Else
        Dict(Data) = Dict(Data) + 1
      End If
    End If
  Next
  UniqueItems = Dict.Keys
  Count = Dict.Items
End Function

Private Function SpecialCells(ByVal r As Range, ByVal Typ As XlCellType, _
    Optional ByVal Value As XlSpecialCellsValue = &H17) As Range
  'Avoid the SpecialCells-BUG to return all cells from the current region
  On Error Resume Next
  Select Case Typ
    Case xlCellTypeConstants, xlCellTypeFormulas
      Set SpecialCells = Intersect(r, r.SpecialCells(Typ, Value))
    Case xlCellTypeConstants Or xlCellTypeFormulas
      'Special feature: Return all used cells
      Set SpecialCells = Intersect(r, r.SpecialCells(xlCellTypeConstants, Value))
      If SpecialCells Is Nothing Then
        Set SpecialCells = Intersect(r, r.SpecialCells(xlCellTypeFormulas, Value))
      Else
        Set SpecialCells = Union(SpecialCells, Intersect(r, r.SpecialCells(xlCellTypeFormulas, Value)))
      End If
    Case Else
      Set SpecialCells = Intersect(r, r.SpecialCells(Typ))
  End Select
End Function

Private Sub SaveView(Optional ByVal Wb As Workbook, Optional ByVal ViewName As String = "Temp", _
    Optional ByVal PrintSettings, Optional ByVal RowColSettings = True)
  'Save the current view of the active sheet
  If Wb Is Nothing Then Set Wb = ActiveWorkbook
  Wb.CustomViews.Add ViewName, PrintSettings, RowColSettings
End Sub

Private Sub RestoreView(Optional ByVal Wb As Workbook, Optional ByVal ViewName As String = "Temp")
  'Restore a view (this can change the active sheet)
  If Wb Is Nothing Then Set Wb = ActiveWorkbook
  On Error Resume Next
  Wb.CustomViews(ViewName).Show
End Sub

Private Function ValidSheetName(ByVal SheetName As String) As String
  'Removes invalid chars from Sheetname
  Const InvalidChars = ":\/?*[]"
  Dim i As Integer
  For i = 1 To Len(InvalidChars)
    SheetName = Replace(SheetName, Mid(InvalidChars, i, 1), "")
  Next
  ValidSheetName = Mid(SheetName, 1, 31)
End Function

Private Function SheetExists(ByVal SheetNameOrIndex As Variant, _

  'True if sheet SheetNameOrIndex exists
  On Error Resume Next
  If Wb Is Nothing Then Set Wb = ActiveWorkbook
  SheetExists = Not Wb.Sheets(SheetNameOrIndex) Is Nothing
End Function

albert d. kallal say:

While you can disable the recent tab with relative ease

 

        <tab idMso ="TabRecent" visible="false"/>

 

Unfortunately, the last "most recent file" list will remain in view (so tab recent is # MRU list)

 

The only way to hide this is to set the MRU list length = 0 in place of the default 4.

 

Unfortunately, this means modifying the registry.

 

Sample code to do this is outlined here:

http://www.accessribbon.de/en/index.php?FAQ:FAQ_Access_2010:2

 

There should be an easier way, but there is not.

You can of course hide all of the ribbon and all of backstage etc. with one line of VBA code in your startup with :

 

   DoCmd.ShowToolbar "Ribbon", acToolbarNo

 

However, if you using custom ribbons, then the above is about your only option here. In theory that MRU list should be re-set back to whatever it was when you shut down your app, but in a lot of my Access environments, they are running my application without even knowing they are using access and they never see any of the built in menus and features anyway.

In general I have to agree with Jeeped's reply. It sounds like you are describing the "context sensitive" ribbon tabs.  These are a variation of the floating menus that sometimes displayed in a similar manner in 2003 and earlier.


Starting with Office 2007 MS introduced this "hide-and-seek" feature.  The basic tabs always display. Plus there are 20 some-odd "context sensitive" tabs. They are only displayed when you click on a relevant feature, such as images or WordArt.  Exactly which tabs display depends on what feature your are clicking on.  AS WELL, these tabs are also SOMETIMES "version specific".  So the tabs displayed when you click on a 2003 vs 2007 vs 2010 WordArt are different.


If you want to see all of the possible tabs go to File menu > Options command > Customize Ribbon command. On the far right of the big white "Backstage Pane" is the "Customize Ribbon" column.  In the first drop down you can select from 3 options: "All tab", "Main tabs" and "Tool Tabs".  The Tool tabs are the context sensitive ones. They are grouped by context related feature.  Note, even though there are "Format" and "Drawing" tabs repeated under many features they are NOT exactly the same.
