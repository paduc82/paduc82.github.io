Hello DataDaly

The issue has to do with your contact/calendar folders being contained in the OST data file associated with the IMAP account. The change from earlier versions of Office ('2010 and before) is that the contact/calendar folders were not maintained in the same data file as the email folders.

In any case, since contact/calendar folders associated with an IMAP account are not sync'd with the server, you have one of two options that will provide the functionality without losing IMAP sync capability for you email folders

Option #1

1 - Create a new PST file for the profile. From the <Home> Ribbon --> click on <New Items> --> <More Items> --> Outlook Data file
2 - Once done, go to File --> Account Settings --> Data Files tab --> select the newly created PST file --> <Set as Default> - this will be where the default contact/calendar folders will be stored for the profile

To make the next of instructions easier, set the Outlook Nav PAne to <Full Foder> view (press CTRL+6)

3 - Copy all you contact/calendar items from the IMAP "THis Computer Only" folders to the contact/calendar folders in the PST folder tree
4 - For contacts - select the contact folder --> press CTRL+A to select all items --> right-click the items --> select <Move> --> <Other Folder> --> select the folder in the PST folder tree
5 - For the calendar items, set the view for the calendar folder to a <List View: and do the same as described in item #5

Option #2

If you want to be able to share your contact and calendar items across multiple devices while retaining your main IMAP account, you can

6- create a new Outlook.com email account,
7- Add it to your Outlook profile
8- once added, configure the data file for the Outlook.com account (an .ost file) to be the "default data file" (no need to change your :default email account" - those are two separate things)
9) perform steps #4 & 5

INDIRECT only works with open workbooks. If the INDIRECT formula refers to a different workbook, that workbook must be open, or the formula will return a #REF! error. There could be an add-in or a UDF to make it possible to work with a closed workbook but have not tried this.

Using Indirect with another workbook (Excel_1.xlsx) you may enter below in cell A1:

[Excel_1.xlsx]Sheet2!$A$3

& then use:

=INDIRECT(A1)

Booting into Safe Mode takes a little longer than a normal boot. Once you are either at your login screen or Desktop just reboot normally.
