andreas killer say:

you mean creating a ribbon in Visual Studio?

No, you can create AddIns with Excel as usual.

https://msdn.microsoft.com/EN-US/library/gg597509%28v=office.14%29.aspx

Then you need the Custom UI Editor to add the CustomUI.xml file.

http://openxmldeveloper.org/blog/b/openxmldeveloper/archive/2006/05/26/customuieditor.aspx

Also your AddIn must support the subs that are called from the ribbon buttons (callback routines). Here are some samples.

http://www.rondebruin.nl/win/s2/win003.htm

Theoretically it might be possible to customize the exportedUI, but I've never tried that and it makes no sense, because you have to write the same code (and do the same work) as you need for an AddIn...

You can share your AddIn with other people, so when the AddIn is loaded, you can add something to the ribbon.

But you can not share only a part of your ribbon customizations, so other people would never load your exportedUI file.

BTW, I recommend to use GUID names for the ribbon callback routines, otherwise you get conflicts when 2 AddIns have a sub named "OnAction". Here is a sample:

https://dl.dropboxusercontent.com/u/35239054/SendALink.zip
