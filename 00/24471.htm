How can I set PowerPoint 2013 to use 4:3 slide size as the default - Microsoft Community

potentilla fruticosa say:

Today I got caught off guard in a really nasty sort of way: I presented my first native 2013-created PowerPoint on an LCD projector with Office 2010 and ended up with awfully tiny fonts because it was presenting in 16:9 mode and not 4:3 mode (EVERY single LCD projector I've EVER seen is 4:3). There's an awkward workaround suggested by Microsoft themselves whereby you save a custom template and set it as the default but that's not a solution (http://office.microsoft.com/en-ca/powerpoint-help/change-the-slide-size-to-standard-or-widescreen-HA102893534.aspx).


potentilla fruticosa say:

Thank you Rohn007 and Steve. It looks like the add-in (http://www.pptfaq.com/FAQ00972_AutoWideScreen-_an_example_add-in_with_event_handling.htm) is a viable solution, with caveats.


If you're using 2010/2013 you may need to modify the macro depending on your security settings. It's designed for old software (Office 2003) so it's saved in a format that's deprecated. If you use none of the Trust Center security features in PowerPoint 2010/2013 you should be OK.


Modify autowidescreen.cfg. Open the file in WordPad or NotePad and change the first line to 7.5 x 10. That's the size, in US/UK inches for 4:3 aspect ratio.


Approach 1. Using the PPA add-in without modifying it:

1. If you are going to use the PPA (old format) macro/add-in, you have to import it through File tab -> Options -> Add-Ins -> Manage -> PowerPoint Add-Ins -> Go (yes, that's a mouthful).


2. You'll also want to have a predictable location for the add-in folder since PowerPoint doesn't actually move the PPA file. PowerPoint has a directory at %USERPROFILE%\AppData\Roaming\Microsoft\AddIns but you can save it anywhere. Do note that using the %USERPROFILE% directory means that the add-in is only available to one user. If you want to make it available to all users you'll have to save it elsewhere.


Approach 2. Updating the PPA (very old file format) to PPAM (2013 file format):

Do you use security? If not, skip to step 2.

If, like me, you use the Trust Center to restrict what files can open you'll need to re-save the macro as a PPAM fiile (I do not want .PPT files being saved as .PPT so I block .PPT from automatically Opening and Saving).


Step 1. TEMPORARIILY disable Trust Center security. To re-save the PPT as a PPAM temporarily disable the block on Opening and Saving PowerPoint 97-2003 files in the Trust Center. You can search this forum for my question on how to do that). Be sure to remember to re-enable your security settings when you're done step 2.


Step 2. Saving the PPT as a PPAM add-in:

Modify Macro. Open the AutoWidescreen.ppt file and resave it as a PPAM format file. PowerPoint 2013 defaults to saving the file in %USERPROFILE%\AppData\Roaming\Microsoft\AddIns. It will also automatically load it for you. You'll have to copy the modified autowidescreen.cfg file to that location or else it won't work.


Alternately, you can save the PPAM elsewhere but you'll still need to make sure that the .cfg file is in the same location as the PPAM and that you manually load the PPAM as an add-in.

Index Field Codes display if the non-printing characters () are toggled on. If you just want to not have them shown toggle non-printing characters off. [Command+8 toggles the display of non-printing characters.] Either way they do not display on paper or in a PDF.
