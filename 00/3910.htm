Hello Chas,

Thank you for posting your question on Microsoft community forum.

I would like to know some information about the issue so that we could help you better.

Is this issue specific to a particular website?

While trying to upload files on a website, when you browse files on your computer through website upload functionality, you see ghost files (old files which are renamed). Since, Internet Explorer 10 and 11 do not contain index.dat file, you are unable to remove these files.

The index.dat file was used by earlier version of Internet explorer to record information about websites that have been visited and details of cookies being stored on your computer.

Internet Explorer 10 and 11 use a completely new file structure. Instead of the old index.dat file, Internet explorer 10 stores all cache/history information inside a single Jet Blue database (Also known as ESE database or .edb file). This database file is WebCacheV24.dat, and it's located under

C:\Users\[UserProfile]\AppData\Local\Microsoft\Windows\WebCache folder.

I would suggest you to try following methods and check if it helps.

Method 1:
If you see any such files when you visit the files location directly (and not through a website upload functionality), possibly these are the hidden files which are set to show and you can unhide it again.

For more information, please refer the following articles:
What is hidden file?
http://windows.microsoft.com/en-us/windows/what-is-hidden-file#1TC=windows-7

Show hidden files
http://windows.microsoft.com/en-us/windows/what-is-hidden-file#1TC=windows-7


Method 2:
If you do not see those ghost files when you visit the file location directly, you may remove all Internet Explorer browsing history, cache and copies of texts, images and other data.
Refer:
Manage or delete your browsing history
http://windows.microsoft.com/en-us/internet-explorer/manage-delete-browsing-history-internet-explorer#ie=ie-11

If you wish to delete all the unnecessary temporary and ghost files which are not in use, you can use Windows disk cleanup utility.
Please refer this article:
Delete files using Disk Cleanup
http://windows.microsoft.com/en-us/windows/delete-files-using-disk-cleanup#delete-files-using-disk-cleanup=windows-7


I hope this information helps.

Please do let us know if you need any further assistance.

When you print a Microsoft Word document, the printer settings are ignored. These printer settings include the paper size, the page orientation, and the page margins.

Consider the following scenario. You change the properties of the printer driver to specify paper size, page orientation, or page margins. However, when you print a Word document, its paper size, page orientation, or page margins differ from what you specified in the printer driver properties.

Try a different Printer driver

http://answers.microsoft.com/en-us/office/forum/office_2010-excel/office-2010-excel-crashing-and-restarting/a070769c-690e-4372-a164-0a80fe2e346f

This link descibes how the user solved an Excel crashing problem by switching to an alternate printer driver.

I find it surprising, but it shows how closely Excel (and Word) are tied to the printer and printer driver.

I had faced this issue as the printer driver caused the problem.

In my case, 

 Fault Module Name: hpz3r5k2.dll caused by the printer Officejet K8600 driver.

It is not compatible with excell 2010 during print preview.

After work out almost 2 weeks, I figured out the solution, found that printer driver Officejet K7100 solved the problem. 

At the printer Officejet K8600, I change the driver to Officejet K7100 series.

Voila, no more crash and restarting when using print preview and printing working great.

(You need to download the driver if you don't have it)
