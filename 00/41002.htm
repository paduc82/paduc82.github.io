dusty harper [msft] say:

Windows Firewall uses the best options for out of box setup by default.  You may need to poke holes as needed to allow for special network communications.

As an example, take File and Printer Sharing.  By default this rule set is disabled out of box.  You can enable the rule either directly by using the firewall UI, or indirectly by just creating a file share.  Again by default, Windows Firewall opts for Security, so the default is to only enable the rule for Private (or Domain if domain joined).  This makes the rule most restrictive.  It would likely make sense for you to allow this for private (i.e. your home network) as well if you are domain joined.  This rule is unlikely needed for Public (i.e. Starbucks).  You'll need to make the same determinations for other incoming traffic.  If the traffic is initiated by the host, Windows Firewall by default does not block this, and maintains state so you don't need to explicitly open holes for responding traffic.

The essential services for the OS already have rules which allow their communication (DNS, DHCP, etc.).  As an experiment, I'd be interested to know if you move from the Work network, and then to your home network, if you turned off the firewall when you connect to the home network if the problem you describe is eliminated (do this at your own risk of course, and don't forget to turn the firewall back on).  If the issue is still present, then I'd be led to believe the issue is not firewall related, but some of the network services.

You can also dump out what your interface is classified as using Powershell:

PowerShell.exe Get-NetConnectionProfile; Pause;

Name             : ntdev.corp.microsoft.com
InterfaceAlias   : Ethernet
InterfaceIndex   : 3
NetworkCategory  : DomainAuthenticated
IPv4Connectivity : Internet
IPv6Connectivity : Internet

BSOD and black screen issues are usually not related - but since we don't have anything else to go by, we'll have to start with presuming that they are related.

If you get to see the Toshiba BIOS screen before the screen goes black, then please run the bootable hardware diagnostics that I mentioned in my previous post.

If unable to see the Toshiba BIOS screen, then try using an external monitor to see if you can see anything on it (most Toshiba's have a VGA output on the side of the case).

Just thinking out loud here - the system has to load a certain amount of Windows stuff in order for the BSOD to show up.  And, if the BSOD shows up, then the video must be working.

Try tapping F2 repeatedly after you turn the computer on - this may get you into the BIOS.  If able, set the boot options to load the CD/DVD/ODD first and the hard drive second.


If you can't see anything on an external monitor, you'll have to try booting from a CD/DVD.  Try the MemTest86+ test from the link in my previous post.

Then, turn the computer on and repeatedly tap F12.
Wait a minute or two and press Enter - does the test start?
Is the CD/DVD drive being accessed?
If not, then shut down the system.

Then, turn the computer on and repeatedly tap F12
Wait a minute or two and press the down arrow key ONCE
Then press Enter - does the test start?
Is the CD/DVD drive being accessed?
If not, then shut down the system.

Keep doing this, but change the number of times that you press the down arrow key (increase by one each time).  After trying this 8 times you will either be in MemTest86+ or the testing didn't work.

FYI - the down arrow thing is just letting you move through the different boot options one-at-a-time.

Hi

 

That 15 Documents minimum is there because some systems do not have the resources to handle opening that many files at one time. Also some printers do not provide a large enough queue to handle that many documents.

 

If you have a system that can has enough resources, you can change this setting.

 

1. Press the WinKey + R to display the Run dialog. Type  regedit.exe and click OK to start the registry editor.

 

2. Navigate to the following Key.

HKEY_CURRENT_USER\Software\Microsoft\Windows\CurrentVersion\Explorer

 

3. In the right side pane, Right Click and select  New/ DWORD (32-bit) Value.

 

4. Name the Value  MultipleInvokePromptMinimum  (Case sensitive).

 

5. Right click the new Value and select Modify.

 

6. In the Base section, select Decimal.

 

7. In the Value Data box, type the number 

 

Press the F5 key to refresh and exit the registry editor.

 

Test the results and let us know if this helps.
