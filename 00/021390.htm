Image Files and Compression in PowerPoint - Microsoft Community

roberta dombrowski say:

Image Files and Compression in PowerPoint

When you insert an image into PowerPoint does it retain it's native file format or is it converted to a different version. For example if you put in a .TIF image will it be converted to JPEG when you insert it into the presentation?

Also, I know that PowerPoint applies compression to images unless you choose for it not to. What exactly does the compression entail? Does this mean that it compresses your image EACH time it is saved or is it only being compressed when its initially put into the document?

steve rindsberg say:

>> When you insert an image into PowerPoint does it retain it's native file format or is it converted to a different version. For example if you put in a .TIF image will it be converted to JPEG when you insert it into the presentation?

It depends on the format of the original image.  PPT stores images internally as PNG, JPG or GIF.  If y you feed it one of those formats, it'll stay in that format; otherwise, it gets converted.  I've never really tried to scope out the logic that governs, say, whether a TIF gets converted to PNG or JPG.

If you're interested, though, any PPTX/PPTM/PPSX file is actually a ZIP file. You can rename it to give it a .ZIP extension and then open it up, look for the Media folder and there you'll find the images, as PPT stores them internally.

>> Also, I know that PowerPoint applies compression to images unless you choose for it not to. What exactly does the compression entail? Does this mean that it compresses your image EACH time it is saved or is it only being compressed when its initially put into the document?

When you choose compression, you set the dpi you want to compress to.

Then when PPT saves, it looks at the size of the image and calculates how many pixels to downsample to.  For example, if you've selected 96dpi compression and you've reduced your image to 1 inch on the slide, it'll be downsampled to 96 dots.  

And as near as I can tell, it makes this calculation each time you save, so if you saved once with the image at full screen then save again with it reduced to a tiny size, the image gets compressed again.

But I suspect it doesn't get re-compressed on each save, so long as the size stays the same.
