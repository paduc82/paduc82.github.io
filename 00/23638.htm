ID Vendor Name Version
0 Microsoft Excel 2014
0 Microsoft Word 2014
0 Microsoft Access 2014

john w. vinson say:

Leave the ID out of the Append query: rather than INSERT *, name the fields explicitly:

INSERT INTO applications SELECT [Vendor], [Name], [Version] FROM tempapp;

 

Note that both Name and Version are reserved words and may not be good choices of fieldname!

 

An Append query is the only way I know to insert data into an Autonumber field without triggering the increment; in this case you DON'T want that to happen, so just omit the field.

to print them with rest of sheet follow JasperP's advice

to get them from within excel, if you have links to the sheets in some Cells then use the :

B14 =CELL("address",'A vary long sheet name'!A3)

to get the cell's address and something like:

C14 =SUBSTITUTE(TRIM(LEFT(SUBSTITUTE(MID(B14,SEARCH("]",B14)+1,1000),"!",REPT(" ",100)),99)),"'","")

Sub AllSheetNames()
Dim aSht As Variant, iRow As Long, iColumn As Long

iColumn = 3: iRow = 2 'To start from cell(2,3) Aka C2

For Each aSht In ThisWorkbook.Sheets
    Debug.Print aSht.Name
    Cells(iRow, iColumn) = aSht.Name
    iRow = iRow + 1
Next

End Sub


To use the above paste it in a Module (Alt+F11 - Insert - Module)

Fix iColumn and iRow to where you want the results

Hello Robert

Sure, you can do it this way

See, "OneDrive for business" is a private drive for just the user that it belongs to.

So, if you want to move the files to someone else's docs, you will have to just open each of them in Windows Explorer and drag/drop them from one account to the other.

The easiest way to do this if by going into the SharePoint Admin Center > Settings > Manage user profiles, search for the two users and use the dropdown to choose "Site Collection Admins" and add yourself as a Site Collection Admin to that other users ODFB.

Now you want to go into you own ODFB, and change the URL to the other users account. Just change the first part after /Personal/

You want to go into the first persons, and once you go into the users ODFB, you want to open that library up in Windows Explorer by using the "Open with Explorer" option.

You want to repeat that process for the other ODFB library

Now that you have both libraries open in windows explorer, drag/drop from one window to the other.

Hope this helps

ossiemac say:

First need to confirm what rules you want to apply.

I am understanding that if any cell (or cells) in the range $D$8:$O$600 contains an error then apply the formatting to those cells with the errors. If my understanding is correct then the following code should do that.

Sub Macro1()
    Cells.FormatConditions.Delete
    With Range("$D$8:$O$600")
        .FormatConditions.Add Type:=xlExpression, Formula1:="=ISERROR(D8)"
        .FormatConditions(Selection.FormatConditions.Count).SetFirstPriority
        With .FormatConditions(1).Font
            .ThemeColor = xlThemeColorDark1
            .TintAndShade = 0
        End With
    End With
    Selection.FormatConditions(1).StopIfTrue = False
    Application.WindowState = xlNormal
End Sub

With conditional formatting, normally the formula is applied as if it applies to only the first cell of the range and Excel automatically applies the condition to the remaining cells in the "Applies to" range.

If you want to extend the "Applies to" range then select any cell in the already Conditionally formatted area and select Conditional formatting -> Manage rules and edit the "Applies to" range.

Note that to edit the "Applies to" range you should position the cursor anywhere within the field and press F2 to invoke edit mode otherwise when pressing arrows, Excel thinks you are selecting ranges. Alternatively you can simply delete the existing range and use the mouse to select a new range and it will automatically update the field with the new selection.

If my understanding of the problem is not correct then feel free to get back to me.
