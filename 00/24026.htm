I’ve had a problem upgrading every build of Windows 10.  It would always crash or encounter a problem during the install and rollback to the previous version.  In one upgrade it would get to 88% then encounter an error and rollback.  In the upgrade from 9926 to 10041 it would get to “PC Settings” phase (30%, 0%), and then bluescreen followed by a rollback.

 

In both my cases the root problem was caused by the fact that the strange 350Mb RAW partition at the start of the disk had somehow been assigned a drive letter.  Once I removed the drive letter using Disk Management, the upgrade finished successfully.

 

In the latest instance, I installed 9926 to an unpartitioned SSD (240Gb, Corsair Force 3 SSD) where the installer created 3 partitions of 350Mb, 222.79Gb and 450Mb.  Immediately trying to update to 10041 failed with a crash in ks.sys, rollback and error 0xc1900101-0x30018 displayed.  Removing the bogus drive letter and  retrying the update led to a successful update to 10041.

 

I haven’t seen this solution anywhere so I’m throwing it out there.  Hopefully it will be of use to someone.

a. user say:

Either connect to the data source using DDE (Check Word Office Button->Word Options->Advanced->General->Confirm File Format Conversion on open, go through the data source selection process again, then when "Confirm Data Source" shows, check "Show All" if you only see "OLE DB Database Files". Then find and select the DDE option.

But DDE is not recommended, requires Excel on your system, and can only reach the first worksheet in the book.

In this case, the fact that you are seeing an "Excel day number" rather than a date in Word suggests that you may have mixed data types in your date column in Excel (that is at least one possibility - see http:\\tips.pjmsn.me.uk/t0003.htm for further info) in which case you should try to ensure that all the rows have dates in them. { MERGEFIELD mydate }

or

{ MERGEFIELD mydate \*Mergeformat }

change it to 

{ MERGEFIELD mydate \@"DDDD, MMMM D, YYYY" }

or

{ MERGEFIELD mydate \@"DDDD, MMMM DD, YYYY" }

then Alt-F9 again and try previewing the data.
