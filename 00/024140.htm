Users on a Mac are a different things.

a. user say:

1. One or two things can cause Word to get the data source location wrong. These problems vary from one Word version to another. For example, if you have specified fliter options, Word may "lose" the data source when it next tries to open the Mail Merge Main Document (although I thought that problem had gone in Word 2003).

2. Also, you may have problems if at any time you have done this with the Mail Merge Main Document:

 a. Connect to a data source

 b. Save and close the Mail Merge Main Document document

 c. Move the data source

 d. Re-open the Mail Merge Main Document

 e. when prompted to find the data source, you provide the new location/name/sheet name

 f. you save and close the Mail Merge Main Document.

After (f), you would probably imagine that Word has remembered the new location and will use it next time. However, that does not always appear to be the case. In that scenario, you can sometimes fix the problem by removing the data source altogether, then re-connecting. You can do this as follows:

 g. re-open the Mail Merge Main Document

 h. go through the process of re-attaching the mail merge data source (this is probably the most straightforward thing way to go at this point)

 i. Enable the MailMerge toolbar (if necessary), e.g. using View->Toolbars

 j. Click the first icon and select Normal Word Document as the document type. This disconnects the data source

 k. Save and close

 l. Re-open, attach to the data source you really want, then save and close.
