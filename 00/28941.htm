shyamal debnath - msft say:
Hi Josh,

If you have provisioned custom domains in addition to the initial domain in Office 365, you must publish two CNAME records for each additional domain. So, if you have two domains, you must publish two additional CNAME records.

However, Office 365 automatically sets up DKIM for initial domains. The initial domain is the domain that Office 365 created for you when you signed up with the service, for example, contoso.onmicrosoft.com. You don't need to do anything to set up DKIM for your initial domain.

For example, DMARC won't work if you point the MX record to your on-premises mail server and then route email to EOP by using a connector. In this scenario, the receiving domain is one of your Accepted-Domains but EOP is not the primary MX.

For your reference:
Use DKIM to validate outbound email sent from your custom domain in Office 365
Use DMARC to validate email in Office 365

jay freedman say:

The message that "the selection is locked" usually occurs because the document is protected in one of several ways.

Go to the Review tab of the ribbon and click the Restrict Editing button, which opens a pane on the right. If you see a "Stop Protection" button on the pane, click it.

If there is no "Stop Protection" button (but instead there's a "Yes, Start Enforcing Protection" button), there may be a different kind of protection. To look for this, first go to File > Options > Customize Ribbon and click to put a check mark in the right-hand list next to "Developer", then click OK. That adds the Developer tab to the ribbon. On that tab, if the Group button is enabled, click it to open its menu; if the Ungroup item in the menu is enabled, click it.

There are a few unusual circumstances that could also cause that message, but they would probably be accompanied by other messages.

If the same message occurs when you start a new blank document, that means the protection is present in your Normal.dotm template. To fix that, you'll have to open the Normal.dotm template itself (as described in http://www.addbalance.com/word/normaltemplate.htm) and remove the protection there.

suzanne s. barnhill say:

Following up on Stefan's line of thought, if the files originated elsewhere (for example, if they were transferred from another computer via CD or email or downloaded from an online storage space), they may well open in Read Mode and/or Protected View by default. In the latter case, you'll have to turn off protection; in the former, the formatting will not be WYSIWYG.

You can avoid Protected View by storing the documents in a trusted location and/or by changing settings in the Trust Center (File | Options | Trust Center | Trust Center Settings | Protected View or File | Options | Trust Center | Trust Center Settings | File Block Settings). By default, files created in versions earlier than Word 97 will open in Protected View; if some of the documents you think were created in Word 97 are actually older, legacy documents, this could be the issue.

Email attachments will by default open in Read Mode. You can change this by clearing the check box for "Open e-mail attachments and other uneditable files in reading view" at File | Options | General.

stefan blom say:

"Compatibility mode" only indicates that the files were created in older versions of Word.

The so-called layout engine has changed over the past Word versions, most notably in Word 2013, so it is not surprising that some older documents may not look the way you expect them to. However, this shouldn't happen as long as you keep the files in compatibility mode, that is, as long as you don't convert them to Word 2013 format.

Do you see the Protected view notification on some of the documents? In that case, what happens if you click Enable Editing?

suzanne s. barnhill say:

Stefan's reply assumes that you are using a first-line indent in your paragraphs. If you have actually been using a tab stop for the indents, you can try changing the default tab stop interval in the Tabs dialog.

But it is far better to use a first-line indent and, as Stefan says, to make this part of a specific style. If you have used the Normal style for the entire document, applying direct formatting for the title page and so on, I recommend selecting the text in each chapter and applying the Body Text First Indent style, which you can then modify to your requirements. By default, this style has a 0.15" indent and 6 points Spacing After, but you can change both of those.

For paragraphs that you don't want indented, use Body Text. If you will be using Body Text, modify it first, before applying Body Text First Indent or modifying it, and the changes you made to Body Text will be reflected in Body Text First Indent, which is based on it. For more, see http://word.mvps.org/FAQs/Formatting/NormalVsBodyStyles.htm
