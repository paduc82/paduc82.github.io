How do I setup word document for note taking - Microsoft Community

mmv_729 say:

How do I setup word document for note taking

Hello-I have a meeting with a group of people to review a 90 page document.    How can I setup this document to print with a handout area.  I know that, that feature is availabe in Powerpoint.   I tried copying the doc in powerpoint and it only takes the first paragraph.  Please advise.

 

I'm using Word 2007.

suzanne s. barnhill say:

One simple way to get a margin would be to add a comment ("Notes" or "Use this space for notes") to the end of the first line of the document. This will cause the markup area to be displayed, and when you print the document, the default settings will be "Document showing markup." This will cause the document to print reduced into the top left corner of the page, leaving space on the right and at the bottom for notes.

 

The advantage to this approach is that the layout of the document is not changed, and removing the comment (or printing without markup) will give you the original version.

 

rohn007 say:
There is nothing explicitly designed to do that, but there are several features you can use to emulate the feature.


Change paragraph line spacing from single to double, triple or wider so people can write their notes between the lines, immediately next to the relevant body text.


Change the margins to provide a Wide left, or right or bottom margin


Create a wide footer starting with a line to separate it from body text and word "NOTES:" to indicate purpose.  Optionally You can also add lines for people to write on.


Those are the obvious options that pop up. I'm other people will be able to suggest more.

My personal favorite is to provide people with an e-copy of the document file. Let them make notes directly in the file in the way that suits them best: hidden text embedded directly in the body text, using the comment feature, using the track changes feature, plus the options I suggested.

Find first date of month in list - Microsoft Community

shnickins say:

Find first date of month in list

How do I find the first date in a month in a list... for example, if I have 6/17/2011 6/10/2011 6/3/2011 5/27/2011 5/20/2011 ... I would like the formula to return June 3rd.
amit zaveri say: Assuming...

Above dates are in Range A1-A5
Month Start date is in B1
Month End date is in C1

You can use this formula

=MIN(IF((A1:A5<=$C$1)*(A1:A5>=$B$1),A1:A5,""))

After you type this formula, take cursor to the end of formula and press CTRL+SHIFT+ENTER (this will put curly braces around formula)

This will give the minimum date in the range

-----------

This can be further simplified by writing a macro if needed to be done on regular basis as well as for multiple months.

How to Write/Create Query Using Start and End Date Data from One - Microsoft Community

nvrgiveup say:

How to Write/Create Query Using Start and End Date Data from One Table as Criteria for Query of Data Table

How do I Write/Create Query to use the Start and End Date Data from Table 1 as the Criteria to query the Data of Table 2?  Example:

Table 1                                                 Table2

Well    StartDate   EndDate                     Well    SaleDate      Data

11       01/01/09    12/31/09                    11       02/28/09      I'm producing

12      01/01/08     10/31/08                    11       03/31/01      More Data

                                                             12        01/31/08     Old Data

 

I would like  to grab all the data from table 2 for wells  between their start and end dates from Table 1.  Thanks!

allen browne say: Option 1: Efficient, but read-only result:
  1. Create a query using both tables.
  2. In the upper pane of query design, drag Well from Table2 and drop onto Well in Table1. Access shows a line joining the 2 tables on this field. (It may have done this for you.)
  3. Drag SaleDate from Table2, and drop onto StartDate on Table1 Again, it draws the line.
  4. Switch the query to SQL View so you can edit the SQL statement.
  5. Change the FROM clause so it looks like this:
    FROM Table2 INNER JOIN Table1 ON Table2.Well = Table1.Well
    AND Table2.SaleDate Between Table1.StartDate And Table1.EndDate

After the final step, you will not be able to switch back to design view again, so make sure you have entered any criteria before you make this change.

Option 2: Inefficient, but editable result:

Create a query using only Table2.

Use a DLookup() expresssion to read the StartDate and EndDate from Table1.

For help with DLookup() see:
    Getting a value from a table: DLookup()
