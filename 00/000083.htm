jim thomlinson say:

That line of code will refreash all tables including query tables and pivot tables. Depending on your setup and the number of tables involved that can take a long time. While that is running often XL will show as not responding. That does not necessarily mean that it is not working.

 

other code that will refresh just the pivot tables is...

 

    Dim pvt As PivotTable
    Dim wks As Worksheet
   
    For Each wks In ThisWorkbook.Worksheets
        For Each pvt In wks.PivotTables
            pvt.PivotCache.Refresh
        Next pvt
    Next wks

 

So your code would be

Sub Macro1()

    Dim pvt As PivotTable
    Dim wks As Worksheet
   
    For Each wks In ThisWorkbook.Worksheets
        For Each pvt In wks.PivotTables
            pvt.PivotCache.Refresh
        Next pvt
    Next wks

    Application.Run "HideRows"

    Application.Run "Transfer"

shuntfield---01 say: The Nokia N and E Series devices do not contain the PBAP (Phone Book Access Profile) in their Bluetooth Stack.

This means that the MMI is unable to call the data from the handset.

So therefore in this case, the problem lies squarly with Nokia why the handset does not carry PBAP is unknown, perhaps there is no space for it or its not a function that they deem is regularly used, whatever the reason calling either Nokia or Audi is likely to get them to blame each other.

The PBAP - has been discussed here at length with various users saying that its "coming soon" where they have got this info from is unknown, up until now I have not seen it on any N or E Series device, certainly my N82 doesn't have it with the very latest firmware.

There is a workaround for this, which I use in my N82 (and previously in the N95)

This application:
http://nds1.nokia.com/phones/files/software/Nokia_810_Contacts_Download_S60_3rd.sis

Should be installed in the handset. Then, delete the pairing between the phone and the car, in both the car and the phone.

Phone installation guide here:
http://europe.nokia.com/nokia/0,6771,42796,00.html

Next, run the 810 contact downloader application on the handset and re-pair the phone to the car. The contact downloader application should then start up and copy the contacts into the car.

I have tried this now with a new (and old) A4, an A6 and my S3 works fine with the downloader and the MMI/DVD Nav system.

I leave the downloader off most of the time, and occationally I switch it on to resync my contacts. After they have been sync'd the car generally recognises the SIM/Phone combination and doesnt need to download contacts and they should stay in the memory of the MMI.

There have been reports with some users here that it doesn't work, however every Audi MMI I have used >2007 has worked fine, if you have the latest audi software in your car and you have followed the proceedure above then I cannot see any problems. The last car I tried it with was a 2008 A4, and my S3 is 2007.
