doug robbins - mvp office apps & services (word) say:

No, that would not mean that they are linked by the use of an INCLUDEPICTURE field.   You can confirm that by using ALT+F9 to toggle on the display of the field codes in the document.  If they are linked by the use of an INCLUDEPICTURE field the picture will be replaced by the code of that field.

You could then use VBA to automate the Find utility to find a graphic (using ^g) in the header of the document and then replace that with an INCLUDEPICTURE field that caused the replacement graphic to appear in the header (and then perhaps unlink that field in case the document is sent to someone who does not have access to the location of the picture.)

graham mayor say:
other issues, so it is perhaps best simply to identify the graphic to be replaced and replace it with another. Assuming that the image is an inline graphic then the following will replace the first such graphic in each header with the named graphic, using http://www.gmayor.com/document_batch_processes.htm to provide the file handling. If the graphic is not in-line in the header range, then you will have to modify the code to locate and replace the required graphic


Function ChangeImage(oDoc As Document) As Boolean
Dim oHeader As HeaderFooter
Dim oShape As InlineShape
Dim oRng As Range
Dim oSection As Section
    On Error GoTo err_handler
    For Each oSection In oDoc.Sections
        For Each oHeader In oSection.Headers
            If oHeader.Exists Then
                Set oShape = oHeader.Range.InlineShapes(1)
                Set oRng = oShape.Range
                oShape.Delete
                oRng.InlineShapes.AddPicture "C:\Path\Filename.jpg"
            End If
        Next oHeader
    Next oSection
    ChangeImage = True
lbl_Exit:
    Exit Function
err_handler:
    ChangeImage = False
    Resume lbl_Exit
End Function

If you are unfamiliar with entering code into a form's or control's event procedures, this is how it's done in form design view:

1.  Select the form or control as appropriate and open its properties sheet if it's not already open.

2.  Select the relevant event property and select the 'build' button (the one on the right with 3 dots).

3.  Select Code Builder in the dialogue and click OK.  This step won't be necessary if you've set up Access to use event procedures by default.

4.  The VBA editor window will open at the event procedure with the first and last lines already in place.  Enter or paste in the code as new lines between these.
