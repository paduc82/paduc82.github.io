Superscript (Letters/numbers typed above the line of text):

After typing CO (or whatever text you want), immediately press the Ctrl key, the Shift key, and the equals (=) key at the same time. Release the keys then type your number 2. It will become a superscript and look something like this: CO¬≤. To get back to the main text line, press the Ctrl key, the Shift key, and the = key at the same time, then release.

Subscript (Letters/numbers typed below the line of text):

After typing CO (or whatever text you want), immediately press the Ctrl key and the equals (=) key at the same time, release the keys then type your number 2. It will become subscript with the number two lower than the preceeding letters. To get back to the main text line, press the Ctrl key plus the = key again.

then go to the Format drop-down menu and select Font. Under the Font tab, look at Effects and check the box that says superscript (or subscript, depending on what you want).

Trying to calculate age, with birthdate as of specific date - Microsoft Community

eugeniaww say:

Trying to calculate age, with birthdate as of specific date

How do I calculate age, if I have the the birthdate and want to use a specifice date. For instance if I want the age as of 7/31/10?
joeu2004 say: Eugenia wrote:
How do I calculate age, if I have the the birthdate and want to use a specifice date. For instance if I want the age as of 7/31/10?

Calculate the age in what units:  years?

Ostensibly, if the birthdate is in A1 and the valuation date (7/31/10) is in B1, you could write:

=DATEDIF(A1,B1,"y")

However, that does not return the "right" answer when the birthdate is Feb 29 and the valuation date is Feb 28 in a non-leap year.  For example, if the birthdate is 29 Feb 1980 and the valuation date is 28 Feb 2010, usually we want to consider that to be 30 years.  Note that EDATE(A1,30*12) is 28 Feb 2010 if A1 is 29 Feb 1980.

To adjust the DATEDIF result, you could write:

=DATEIF(A1,B1,"y") + IF(DAY(A1)=29,EDATE(A1,12*DATEDIF(A1,B1,"y")+12)=B1,0)

The test DAY(A1)=29 is purposely incompletely.  But it improves performance significantly by limiting the EDATE calculation to only when the birthdate is on the 29th of any month.

diane poremsky mvp (slipstick.com) say:
>> I am looking at the AOL mail on my other machine and those are still there.
When you use pop, mail already downloaded to other computers is not affected when you delete it from a second computer (and the deletion syncs to the server). POP is one way, down, and changes made on one computer do not sync to the other computers.

If you opened AOL in a new outlook profile, the messages you deleted from the server would not be in the new data file.

If you want changes to sync to the other computers, you need to use IMAP.

vincentchoy say:

I think you might be a bit out of your depth. In any case let me try to explain.

Office365 is a family of products, comprising for example Exchange Online, Lync Online, Sharepoint Online, Yammer, One Drive for Business, MS-Office...etc. You can subscribe to an individual product like Lync (now called One Drive for Business), or a package of products like E1, E3, Business Essentials...etc

Office365 products depend on Azure AD for authentication.

When a user logs in to Office365, the service basically goes to Azure AD for authentication. Exchange Server does not handle any authentication.

When you say "single sign on" do you mean -

1. Azure AD passes the authentication to your on-premise AD server? This is the most complex and requires 4 server roles running under HA mode. If the ADFS server fails none of your users will be able to authenticate as you will not reach your AD server. You can research Active Directory Federated Services (ADFS) to read up on this.

2. Azure AD does the authentication, but AD information is synchronised with your on-premise AD server. The Sync happens at regular intervals. This requires one server and AAD Connect/ AAD Sync. This is much simpler and the AADConnect server is not required to be HA.

Both method maintains the same user credentials between your AD and office365

Hope this clarifies.

hansv mvp say:

In B1:

="C001198302"&CHAR(64+SUMPRODUCT(1/COUNTIF($A$1:$A1,$A$1:$A1)))

Fill down as far as needed. Keep in mind that after Z, the formula will continue with [, \ , ] etc.
