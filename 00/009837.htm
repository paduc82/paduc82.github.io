The macro recorder doesn't work properly. It is looking for the cover page building block in the template attached to the current document, when it should make use of the Building Blocks.dotx template. You will have to explicitly load that template into memory and then insert the cover page. Try something like this:
 
Sub tryout()
Const TemplateFileName$ = "c:\users\sb\appdata\roaming\microsoft\document building blocks\1033\building blocks.dotx"
 
AddIns.Add TemplateFileName
 
Templates(TemplateFileName).BuildingBlockEntries("Tiles").Insert _
where:=Selection.Range, RichText:=True
 
AddIns(TemplateFileName).Delete
 
End Sub
 
Of course, you will have to adjust the path to one that is suitable for your system. And, similarly, you will have to change the name of the cover page (building block entry).

jay freedman say:

Here's another method that doesn't require knowing the path to the Building Blocks.dotx file.

The statement Templates.LoadBuildingBlocks forces Word to load all the building blocks it knows about (normally this doesn't happen until the user clicks the Quick Parts button, which is why there's a delay the first time you click that button). This adds Building Blocks.dotx (and possibly other templates) to the Templates collection, and you can find it there.

Sub demo()
Dim t As Template, BB As Template
' make sure all building blocks are loaded
Templates.LoadBuildingBlocks
' get Building Blocks.dotx as a Template variable
For Each t In Templates
If LCase(t.Name) = "building blocks.dotx" Then
Set BB = t
End If
Next
If Not BB Is Nothing Then
' insert the entry
BB.BuildingBlockEntries("Tiles").Insert _
Where:=Selection.Range, RichText:=True
End If
End Sub
