j.e. mcgimpsey say: Without knowing what formulas you're using, it's difficult to tell what error you might be getting.

However, the best way to reference values in another worksheet is to actually use a reference. For instance to get the result from Sheet1, cell B3 to show up in Sheet2, cell C6, put this formula in Sheet 2, cell C6:

    =Sheet1!B3

You can then copy that cell to other cells in the column and XL will adjust the reference accordingly.

If your sheet name has spaces in it, be sure to enclose it in single quotes, i.e.:

     ='Sheet 1'!B3

Utility to sort OneNote 2010 sections - Microsoft Community

jan roelof say:

Utility to sort OneNote 2010 sections

As a heavy user of OneNote, I am generally happy with the 2010 version. One of the drawbacks is that most add-ins, written for the 2007 version, no longer work for notebooks in the 2010 format, or are unreliable. I am beginning to get familiar with programming for OneNote 2010 and trying my hand at some utilities myself.

The OneNote utility I am sharing here is SortSections. I found that the existing one by Daniel Escapa does not work for my 2010-formatted noebooks.
This utility sorts sections within notebooks and section groups. The sorting is done by name, ascending or descending, and can be done recursively (for sections in section groups in the currently selected notebook or section group, etc) or not. For some reason, the section groups themselves apparently cannot be sorted.

This is a first attempt and I am not particularly happy with the user interface, but it appears to work, also for notebooks stored on SkyDrive, as mine are.
I hope some gallant souls are willing to try this out and give me some feedback.

To download the utility, please go to http://www.donationcoder.com/Forums/bb/index.php?topic=23629.msg214268#msg214268.
The zip-file contains an executable and a dll and the two should be kept together. Just run the executable (SortSections.exe) and you get a treeview of your notebooks. Browse to the notebook or section group whose sections you want to sort and press the button.

Cheers,
Jan Roelof

Hi Steve,

Thanks for trying this out. I don't think the problem is the fact that the notebook was SkyDrive-hosted. All my notebooks are on SkyDrive and that is no problem. The problem is that in the notebook that caused the problem you have a level-3 subpage immediately following a level-1 page. OneNote 2010 allows this, but if subpages had a truly hierarchical organization, level-3 pages could only follow level-2 pages. For sorting, I need to be certain which pages are subpages of which pages: if page Y is a subpage of page X, this must remain true also after sorting. Anyway, in this first version, I rudely abort the program when I find this "inconsistency", but the next version will have a more elegant solution.

What notebook you have currently selected plays no role. When the utility starts, it begins by analyzing the entire notebook-section group-section-page-subpage structure and just aborts when it finds this inconsistency. By closing the notebook, you eliminated it from the analysis. If you make the offending level-3 page a level-2 page, the error should go away.

Thanks for trying out and giving me feedback!

Jan Roelof
