jay freedman say:

If you're using only content controls (and no legacy form fields), don't use forms protection. Instead, select the entire document (or at least all the parts you want to protect, plus the content controls), click the Group button to the right of the controls in the Developer ribbon, and choose Group from the dropdown. That protects everything outside the content controls, but it doesn't interfere with spell checking or graphics the way forms protection does.

If you're using some legacy form fields, then don't mix in content controls -- use only legacy form fields, and apply forms protection. To be able to spell-check, you then need macro support as shown inhttp://www.word.mvps.org/FAQs/MacrosVBA/SpellcheckProtectDoc.htm .
