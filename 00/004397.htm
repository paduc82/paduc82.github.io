Firstly a couple of tips to make using VBA easier.

Ensure that all modules have the following command at the very top of the module.

Option Explicit

For all new projects you can have the system automatically insert the command with the following method.

  • In the VBA editor select menu item Tools -> Options
  • Select the Editor tab in the dialog box (should be the default)
  • Set the options as per the following screen shot.

  • Leaving "Auto Syntax Check" option unchecked stops the popup message that you get to click OK on every time there is a syntax error in the code. Without this option checked, the line of code will now just turn red which is all the notification required.
  • With "Require variable declaration" checked, the Option Explicit is automatically applied to all new modules (not to existing modules created before the option was set). Before running any macro while under development you can select menu item Debug -> Compile VBA Project and it will find errors like variables not dimensioned (No Dim statement), mis-spelt variables, mis-spelt object names (control names). I use it all the time before running any macros and while it does not pick up all errors, it finds many of them.
  • Note that if "Compile VBA Project" is disabled (greyed out) then that is because no code changes have been made since the last compile.

Changes to your project.

  • Named ranges have been created on "Lists" worksheet. Dummy names have been used to create the lists. The defined names correspond to the available lists for the ListBox.
  • The defined names have also been used for the column headers.
  • Whatever names are selected in the ListBox provides the named ranges to be assigned to the ComboBox5 DropDown list.
  • The named ranges "Refers to" field contain a formula so that the length of the range is generic to the number of entries that the user wants to insert.
  • The ComboBox 5 "Enter" event code loops through the selected names in the ListBox and then Adds the names from the Defined name range to ComboBox5. Note that I have used the Enter Event in lieu of the "DropButtonClick" event. Reason is that when using the "DropButtonClick" event, the system waould not insert the selected name in the field. I don't know the reason for this but because the "Enter" event works then we use it in lieu of "DropButtonClick"

  • I have removed the Frames from the Userform. These were creating problems with the setting of the list for ComboBox5. It would work the first time but if the user had a change of mind and went back and altered the ListBox selections and then back to ComboBox 5 then the DropDown list was not updating. It could be forced to update by selecting another ComboBox in the same frame and then re-selecting ComboBox5 but I don't think a user should have to do that. Removing the frames fixed the problem.
  • I have toggled the Caption on the CheckBox. It now toggles between Select all and Clear all. Hope you like it because I think it is more explanatory of what it does.
  • OptionButton1 and OptionButton2 are both set to False in the Userform Activate event. The code then identifies whether they are True or False and sets the Single or Multi select of the ListBox Accordingly.
  • Userform "Shifts" has an Activate Event to set the 3 option buttons to False. This is required because if the Userform is Shown again while the project is open then it retains the last option set and clicking it again will not activate the code to show the correct userform.
  • I am curious about the Q1, Q2, Q3 and Q4 and the associated adjacent fields, "Insertion". What is required in these fields? Are they associated with the Output workbook "Book101.xlsx" and where the output is to go?
  • "Book101.xlsx". In your code you are finding the first unused cell in column A of "Book101.xlsx" but that will always be the first unused cell under the table. Need more information on what is required with the Output workbook and how to determine where the output should go on the worksheet.
  • Suggestion. Use "Book101.xlsx" as a template only that is always stored in the same place and create code to use "Save As" to a new filename as soon as it is opened. That way the template is never written to and the new filename can be an intelligent descriptive name that maybe includes sufficient to describe it and also if required a date in "yyyy mm dd" format (This format supports sorting of file names).
