        Content Type  application/vnd.visio

.vss - (default) Visio.Stencil.11

        Content Type  application/vnd.visio

.vst - (default) Visio.Template.11

        Content Type  application/vnd.visio

.vsu - (default) Visio.CustomUI.11

        Content Type  application/vnd.visio

.vsw - (default) Visio.Workspace.11

        Content Type  application/vnd.visio

.vsx - (default) Visio.Stencil.11

        Content Type  application/vnd.visio

.vtx - (default) Visio.Template.11

        Content Type  application/vnd.visio

e.j. gun say:

What I provided was just an example to illustrate how to make an variable public and persistent.  You don't actually need "MySub".  The point is, the first time you set the value of "strPath", it will stay set and ANY sub in that code module will be able to use the value.  Also, you can change the value of "strPath" in any sub of that module.

 

Alternatively, you could change the declaration from:

 

Public strPath as String

 

To:

 

Public Const strPath as String = \\This\Is\My\Path\

 

(setting it to whatever your path is)

 

If you don't change paths very often, this will work.  Then, when you move to a different folder, you can change the value of the constant, and all the subs using that constant will be pointed to the new folder.

 

HTH,

 

Eric

 

P.S. - to open a specific file in a particular sub, you would not want to make the filename part of the path.  Instead, you would do something like this (from your original code):

 

Set SourceWbk = Workbooks.Open(strPath & "EXPENDITURE SUMMARY ITSD FY1213.xls")

 

or

 

strFile = "EXPENDITURE SUMMARY ITSD FY1213.xls"

Set SourceWbk = Workbooks.Open(strPath & strFile)
