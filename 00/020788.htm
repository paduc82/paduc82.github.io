john - project say:

JCHP,

A file has two "names", the one given by the operating system (Title) and the one given by the user (File name). When files are inserted into a master project, they are inserted using the Title and that's what you see when you look at the summary line insertion points for each subproject in the master.

The user can change either or both names. There are a couple ways to change the file Title. Perhaps the simplest is to show the Project Summary Task. The description in the Name field is the Title. Or, you can go to File > Info  and click on Project Information in the middle on the right side of the screen (this may show a little differently depending on which version of Project you are using). Select the Advanced Properties, Summary tab.

Once you have changed the file's Title, it will then appear that way in the master.

Hope this helps.

PS I do like the feature of short hyphens becoming long ones between words, so I rather have a solution to the specific problem above, rather than a generic one that turns off other 'features'.

Thanks. 

jay freedman say:
This occurs when the proofing language (or perhaps the keyboard language) is set to French and the AutoFormat As You Type option to replace straight quotes with "smart quotes" is turned on (http://www.wordbanter.com/showthread.php?t=38564). Removing either of those conditions will stop the nonbreaking spaces.

To change the proofing language, select the text and click Language on the Review tab. In addition to choosing another language, it's a good idea to uncheck the option to automatically detect the language.

To change the quotes replacement, click File > Options > Proofing > AutoCorrect Options, choose the AutoFormat As You Type tab of the dialog (not just AutoFormat), and uncheck the first option.

jay freedman say:
The English version of Office always includes French and Spanish proofing tools, "just because". There are, after all, a lot of people in North America who speak and write in French and Spanish at least part of the time.

You can remove the languages if you want to:

  • In the Windows Control Panel, start Programs & Features.
  • Click on the Office 2010 item and click Change.
  • When the wizard starts, select Add or Remove Features and click Continue.
  • In the list of parts, expand the Office Shared Features icon, then expand Proofing Tools.
  • Click French Proofing Tools and choose Not Available. Repeat this for Spanish Proofing Tools.
  • Click Continue and let the wizard finish.


I'm not absolutely sure this would stop Word from "accidentally" switching to French and putting in the extra spaces, but it might. The culprit is probably that "Automatically detect" option -- I think it sometimes mistakes an English word for French. Natural language is still a stumper for computers.
