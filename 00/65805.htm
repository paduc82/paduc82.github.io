don cuthbert say:

I have noted that the Windows 10 build upgrades come in a series with the same name. The first download prepares the C:\$WINDOWS.~BT folder for the new build, followed by a subsequent Windows Update download (by the same name). If you check the Reliability history report, that first small download/install often shows as an install failure, even though it worked exactly as planned. Now with UUP there are even more downloads in a sequence that show up in Windows Update if you are watching. After downloading many CAB files and then doing preparation on those, there is a separate download process for the main ESD file (which is just a small ESD under UUP) followed by more processing.

So the list of failed updates may not be an accurate representation of true update failures, at least in some cases.
