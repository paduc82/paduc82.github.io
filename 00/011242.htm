steve rindsberg say:
>> I'm curious as to what method others use to translate large amounts of text for presentations

Humans.  ;-)

Now, the humans might start with a machine-translated version AND the original.  The machine translation would at least do quick dictionary lookups of words, though it wouldn't necessarily provide the correct translation even then.  It wouldn't always know how to deal with homonyms or understand which one of several meanings of a word to hand back.

It can be hard to find a translator with native- or near-native fluency in both languages.  Heck, most of us USians couldn't even translate our own language into native-sounding UK English.

Another approach is to have a native speaker of the "source" language who also speaks the "target" language fairly well translate the "first-round" translation, then have a native speaker of the target language edit it into final form.  If the second person has some understanding of the source language, so much the better.  (As it happens, I do a little of this kind of work).

john p spencer say:

Perhaps the following query will work for you.


INSERT INTO [Inventory Transactions] ([Transaction Type], [Product Id])
SELECT DISTINCT "Usage", [Product Id]
FROM [Inventory Transactions]

Your other option is to extract the list from the combobox.

Create a VBA procedure like the following.

Public Sub sAddRecords()

Dim rst As DAO.Recordset

Dim vList as Variant

Dim i As Integer

Set rst = CurrentDb.OpenRecordset("SELECT * FROM [Inventory Transactions] WHERE 1=2")

vList = Split(Forms!FormName!ComboboxName.RowSource,";")

For i = Lbound(vList) to UBound(vList)

With rst
   .AddNew
   .Fields("Transaction Type") = "Usage"  'Or whatever the value is for transaction type
   .Fields("Product ID") = vList(i)
   .update
End With

Next i

End Sub

You should be able to call the procedure from the form
