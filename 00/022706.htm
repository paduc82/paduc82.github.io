joeu2004 say: KeWalk wrote:

I would suggest that you use a Custom format to display "bps".  A big plus:  the cell remains a numeric type, so it can still be referenced in other formulas.

Right-click on the cell(s), click on Format Cells, the Number tab, then Custom.  You might see General.  Change it to General" bps".  If you see 0.00 originally, change it 0.00" bps".

FYI, the value in the cell did not change.  It was already 17.9999999999995.  It simply displayed as 18 due to the format, which might be General or Number with perhaps 2 decimal places.

The anomalous value is the result of computational anomalies that commonly arise in Excel (native computer) arithmetic; or it might be rounding errors to be expected from the computation.

In either case, you might want to explicitly round the computation to zero or more decimal places in order to avoid propagating these rounding anomalies into other computations, where they might make a difference.  For example:

=IF(condition,"",ROUND(expression,0))

Or if the IF function can only return numeric results (both if TRUE and if FALSE):

=ROUND(IF(condition,valueIfTrue,valueIfFalse),0)

ken sheridan say: Firstly you should not hard-code data in an expression.  A fundamental principle of the database relational model is the Information Principle (Codd's Rule #1). This requires that all data be stored as values at column positions in rows in tables, and in no other way.  The correlation between each Media_Type value and another value should therefore be modelled by a Media_Types table with a Media_Type column and another column for the other values.  The Media_Type column should be the primary key of the table.

You can then join the referencing table to Media_Types on Media_Type, returning the non-key value from Media_Types in the query or "Error" if the referenced non-key column in Media_Types is Null.  You can do this by calling the NZ function:  NZ([NameOfTheNon-KeyColumnGoesHere], "Error").

Where the date difference comes in is another matter.  Bear in mind that the DateDiff function, if "m" is passed into as the interval argument, will simply return the difference between the month value of one date and the month value of other, e.g.  DateDiff("m",#2015-12-31#,#2016-02-01#)  will return a difference of 2 months even though the dates are only 32 days apart.  To return the difference in whole months, day-on-day, the following function can be used:

Public Function MonthDiff(varDate As Variant, Optional varAt As Variant) As Variant

    Dim intMonths As Integer
    
    If Not IsNull(varDate) Then
        ' use current date as 'at' date if not specified
        If IsMissing(varAt) Then varAt = VBA.Date
        
        ' get difference in months
        intMonths = DateDiff("m", varDate, varAt)
        
        ' adjust value depending on relative
        ' positions of dates with month
        If intMonths > 0 Then
            If Day(varDate) > Day(varAt) Then
                intMonths = intMonths - 1
            End If
        ElseIf intMonths < 0 Then
             If Day(varDate) < Day(varAt) Then
                intMonths = intMonths + 1
            End If
        End If
        
        MonthDiff = intMonths
    End If
    
End Function

e.g. MonthDiff(#2015-12-31#, #2016-02-01#) will return 1.

To create a No Proofing style in Word 2013:

  1. Format the text as you wish it to appear, then click on it.
  2. Press and hold Ctrl + Alt + Shift + S to open the Styles list.
  3. At the bottom of that dialog, click on the Manage Styles button.
  4. On the Edit tab, check New documents based on this template. Also check Add to the Styles Gallery. Then click on the New Style button.
  5. Give it a name like No Spellcheck.
  6. At the bottom of the dialog, click on the Format button, then on Language.
  7. Check Do not check spelling or grammar. OK out.

Now you can select words and apply the No Spellcheck style from the Style Gallery.

Private Sub PA_RR_DoTask_ListUpdate()
    CurrentRowList = "PA_RR_VRR_R" & PA_RR_RowNo & "_List"

    If Me.Controls(CurrentRowList).Value = "Damages" Then Call PA_RR_VRR_DoTask_Damages(CurrentRowList)

    'Do more stuff (this bit works fine)
End Sub

Private Sub PA_RR_VRR_DoTask_Damages(VRR_List_ID As Control)
    'Do more stuff (this bit works fine)
End Sub

The above runs as follows:

  1. PA_RR_VRR_R1_List_Change runs, marks PA_RR_RowNo as 1 and then calls the PA_RR_DoTaskListUpdate macro.
    The "list" name is assigned (notice how "CurrentRowList" builds the same as "PA_RR_VRR_R1_List". This is a control name)
  2. The "CurrentRowList" control is checked and if the value is "Damages" then we need to call PA_RR_VRR_DoTask_Damages macro which has a ByRef.
  3. PA_RR_VRR_DoTask_Damages is called and the CurrentRowList value (which is PA_RR_VRR_R1_List) is set as a Control and named "VRR_List_ID".
  4. PA_RR_VRR_DoTask_Damages macro runs, then goes back to PA_RR_DoTask_ListUpdate and then back to the Change event.

All in all, simple logic. This allows me to replicate the PA_RR_VRR_R1_List_Change code and update the PA_RR_RowNo value depending on the R value (R1, R2, R3, etc). The reason why I've done it this way is that I've got 50 R1/R2/R3.../R50 events. hansv mvp say:

The argument of PA_RR_VRR_DoTask_Damages is a Control but you pass the String CurrentRowList to it.

Solution: change

Call PA_RR_VRR_DoTask_Damages(CurrentRowList)

to

Call PA_RR_VRR_DoTask_Damages(Me.Controls(CurrentRowList))
