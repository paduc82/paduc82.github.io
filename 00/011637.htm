gord dibben say:

Copy this macro to a General module in your workbook then run it.

Suggest you do this on a copy of your workbook.

Sub Sort_All_Sheets()
    Dim wksht As Worksheet
    For Each wksht In ThisWorkbook.Worksheets
        wksht.Select
        Range("A1:E100").Sort Key1:=Range("E1"), Order1:=xlDescending, Header:= _
                xlYes, OrderCustom:=2, MatchCase:=False, Orientation:=xlTopToBottom, _
                DataOption1:=xlSortNormal
    Next
End Sub

serge knysh say:

Link to file stopped working in Word 2016

Hi guys

I have a following problem with the new Word. Links to files do not work if parent file is saved to a different location.

Word 2016 (16.0.4266.1003), Windows 7(x64)

1) Create 2 docx files on your desktop. (parent.docx and child.docx)

2) Open parent.docx and Insert>Object>Create from File and browse for child.docx on your desktop 

3) Tick "Link to file" and "Display as icon" checkboxes and click OK

4) Now if you check the source path for selected link - it is  C:\Users\<user>\Desktop\child.docx  ( correct )

5) 'Save as' parent.docx into C:\Temp\parent.docx

Windows 10 nags to get you to make your Windows ID the user name and password on you system.  I was reluctant at first but every time I wanted to access dropbox or one drive it was a chore to enter my id and complete the other security option so I caved and allowed 10 to switch my User account to my Microsoft account.   I am all for security and appreciate the effort to keep my info secure but honestly I was happy with the two step verification I am already using with windows 7.
