Sub Demo()
    Dim fso As Object 'FileSystemObject
    Dim fldStart As Object 'Folder
    Dim fld As Object 'Folder
    Dim fl As Object 'File
    Dim Mask As String
   
Application.ScreenUpdating = False
Dim newWS As Worksheet

    Set fso = CreateObject("scripting.FileSystemObject") ' late binding
    'Set fso = New FileSystemObject 'or use early binding (also replace Object types)

    'Debug.Print fldStart.Path & "\"
    ListFiles fldStart, Mask
    For Each fld In fldStart.SubFolders
        ListFiles fld, Mask

    Next
   

Dim myWB As Workbook, WB As Workbook
Set myWB = ThisWorkbook
Dim L As Long, t As Long, i As Long
L = myWB.Sheets(1).Cells(Rows.Count, "A").End(xlUp).Row
t = 1
For i = 1 To L
Workbooks.OpenText Filename:=myWB.Sheets(1).Cells(i, 1).Value, DataType:=xlDelimited, Tab:=True
Set WB = ActiveWorkbook
WB.Sheets(1).UsedRange.Copy newWS.Cells(t, 2)
t = myWB.Sheets(1).Cells(Rows.Count, "B").End(xlUp).Row + 1
WB.Close False
Next
myWB.Sheets(1).Columns(1).Delete
Application.ScreenUpdating = True
End Sub

 


Sub ListFolders(fldStart As Object, Mask As String)
    Dim fld As Object 'Folder
    For Each fld In fldStart.SubFolders
        'Debug.Print fld.Path & "\"
        ListFiles fld, Mask
        ListFolders fld, Mask
    Next
End Sub

 


Sub ListFiles(fld As Object, Mask As String)
Dim t As Long
    Dim fl As Object 'File
    For Each fl In fld.Files
        If fl.Name Like Mask Then
        t = Sheets(1).Cells(Rows.Count, "A").End(xlUp).Row + 1
            'Debug.Print fld.Path & "\" & fl.Name
            If Sheets(1).Cells(1, 1) = "" Then
            Sheets(1).Cells(1, 1) = fld.Path & "\" & fl.Name
            Else
            Sheets(1).Cells(t, 1) = fld.Path & "\" & fl.Name
            End If
        End If
    Next
End Sub

Officially, we all heard that the Zune Marketplace, and the grandfathered Zune Pass, was going to be shut down many months ago. Of course, those of us who were fans of the service weren't happy. Outlook 2013 permission error when trying to add appointment to shared - Microsoft Community

michael pratt_ccu say:

Outlook 2013 permission error when trying to add appointment to shared calendar

We have migrated all users to Exchange online (several months ago), but we have an Exchange 2016 server on-prem and we use dirsync (one way, domain to the cloud). All machines run Window 7 Pro /64bit and Office 2013 (click to run version 15.0.4815.1001).

I have a user that can no longer make appointments in shared calendars. This started this morning (Monday, Jan 9th), it was working for her on Friday and nothing changed in our environment (unless changes were made in the Exchange online cloud). She can see all the shared calendars and the entries, but trying to add one gives her an error "You don't have permission to create an entry...".  When she logs onto the web Outlook, she can make entries with no error. 
No other users have reported this issue and they are all members of a group that has Editor permission to the shared calendars. She was prompted to change her password this morning when she first logged on. So she changed it, then restarted her computer. I checked to make sure the password sync completed, then had her close and re-open outlook. This did not fix the problem. I have also turned cached mode off, then back on. I have completely removed Office 2013 and then re-installed it. Nothing makes a difference.

I just read this posting which sounds like the same problem, but I don't want to mess with my DNS entries until I have some guidance. Hope someone can help me with this?

michael pratt_ccu say:

Here is how I resolved it.

I created a text file with a list of all 85 users (email addresses) that the group needed access to. Then I ran a Powershell script to remove the group editor access for each user,  then another script to add them back as editor.

Remove:

Get-Content c:\Temp\users.txt | Foreach-Object {Remove-MailboxFolderPermission -Identity ${_}:\calendar -user 'CallCenter145255-5325564' -Confirm:$False}

Add:

Get-Content c:\Temp\users.txt | Foreach-Object {Add-MailboxFolderPermission -Identity ${_}:\calendar -user 'CallCenter145255-5325564' -AccessRights Editor}


This gave her access to most of the shared calendars again. There were a few that still gave a permission error, but when she deleted the calendar from Outlook and added it back, they worked.
So far so good!
