cmdrkeene say:

Can you press Ctrl+Shift+Escape together and in the list, see which process is using the most CPU and Disk activity?  You can click the column heading up top until the highest ones are listed first to make it easy to identify.

The processes that are using the most of these could be causing the bottleneck and slow performance, especially if there isn't a lot of Memory available, because then it has to "swap" out to the swap file on the drive, leading to even more bottlenecks.

If the programs using the most resources aren't items you're using, it's likely one of the following:

  • Virus scanner running the "first time scan" after installing your new upgrade
  • Search indexing system building up a new index after the upgrade
  • OneDrive/GoogleDrive/OtherOnlineCloudStorageThings re-scanning for any possible changes after the upgrade
  • Windows "deferred tasks" as Gabel Aul calls them (Microsoft General Manager). These are post-install cleanup tasks and other optimizations (you may be familiar with that LONG wait after upgrading android where it re-optimizes all the apps).

If any of these are busy doing their work, it'll really impact performance. And if more than one happens to be going at once, it really basically drags you down to a crawl.  I've honestly advised some users to just set their power options to not go to sleep mode, and leave it on overnight to finish all the things like that.

But that's of course not always the case.  If you don't think these above are what's impacting you, or you see other processes causing resource utilization (using CPU/Disk/Memory) that aren't these things, let me know and we'll see if we can identify them and what's causing the freeze.

PS. I'd definitely advise checking for updates from Windows Update and your PC manufacturer's website for the best drivers to use for Windows 10.

We can't connect a new Win 7 (64-bit) computer to an established ethernet printer in our local "work" network. We've reached the limit of our troubleshooting skills and need some expert help.

Our Local Network
------------------------
The printer is a Dell 5100cn workgroup laser that is connected via ethernet cable to a Cisco/Linksys 5-port 10/100 workgroup switch. The switch is connected to an integrated Motorola cable modem / 4-port gigabit router / WiFi access point. There is no server between the printer and the network.

The other computers on the network are also running Win 7 and have no trouble seeing and using the printer. Some computers are connected via gigabit ethernet cable to the Motorola router. Other computers connect wirelessly via WiFi (again, via the same Motorola device). All computers, including the new one, are assigned to the same workgroup.

The printer has a static IP address. It also has a built-in webpage that can be accessed within the network via a browser to view the status of the printer. All of the previous computers can view the printer's status webpage by entering its IP address into a browser.

The New Win 7 Computer
--------------------------------
From the Windows command prompt, we've successfully pinged the IP address of the printer and received a return signal. However, we cannot connect to the printer's IP address (and status webpage) with a browser. And we cannot find the printer when we install its printer driver. We tried disabling the Windows Firewall and it didn't help.

When we install the driver, we install it as a "local" printer and create a new "Standard TCP/IP Port" for it. We enter the printer's static IP address and name the port. We set the protocol to "LPR", enter the Queue Name ("lp") and enable SNMP status with community name "public" and index "1". These settings have worked fine with our previous computers.

When it comes time to "choose the printer" we select "Have Disk..." and use the Dell 64-bit driver for this printer. But it doesn't work and a test page will not print. We've even tried installing the printer driver as a "network" printer but this doesn't work, either. Besides, the "local" method always seemed to work best in the past.

The new computer has no trouble seeing the other computers and devices on the network. The new computer can also access the internet. It just can't seem to use the printer's IP address. We even tried changing the printer's IP address but that didn't help, either.

Despite the fact that pinging from the command prompt works---the Windows network troubleshooter says that the printer fails to reply. What are we missing? Please help.

david-eh say:

Greetings,

Hopefully, this will be our final post on this subject. We have discovered the source of the problem and can offer two "fixes" in case someone else runs into the same situation.

Using MSConfig and a process of elimination, we disabled the non-Microsoft services that were running on the computer until we discovered that "Qualcomm Atheros Killer Service V2" was the source of the problem.

Explanation

----------------

Our new Win 7 computer contains a Qualcomm Atheros e2200 ethernet/WiFi/Bluetooth communication module. The manufacturer provides a "Killer Network Manager" for this system. Its purpose is to monitor and control the data flow through a network connection in order to optimize it. For example, the user can choose to give Program-A a high priority and Program-B a low priority. This insures that Program-A has less lag in its network traffic.

The primary market for this feature, up till now, has been online gaming (hence the "Killer" name). It allows gamers to funnel most of the available bandwidth to their game and away from other programs and services running at the same time. A quicker online response gives the gamer the edge he/she needs to get the "kill" before someone else.

This bandwidth control system interfered with the returning data packets from our network printer, mucking up communication between our new computer and our printer.

Fix 1

--------------

The easiest solution for us was to disable "Qualcomm Atheros Bandwidth Control" for the ethernet NIC. This was accomplished by going to: Windows Control Panel > Network and Internet > Network and Sharing Center > Change adapter settings (in the sidebar). Then we right-clicked on our Local Area Connection (our ethernet NIC) and selected the "Properties" command. Finally, under the Networking tab, we

All that was needed after that was a quick reboot and we were able to access our printer even with the Killer Network Manager and its Qualcomm Atheros Killer Service V2 running.

Incidentally, we left the bandwidth control feature enabled for the Wireless Network Connection (our WiFi) since we're not planning on using this network printer when the computer is untethered from ethernet.

Fix 2

--------------

If you don't want the Killer Network Manager running on your computer, you may be tempted to uninstall it. Don't! Uninstalling it will also uninstall the drivers required for your ethernet, WiFi and possibly your Bluetooth as well. We've heard that some users have contacted Qualcomm and been able to coax from them the necessary comm drivers without the "Killer" baggage. But you don't need to do that, either.

All you need to do is: (1) Remove the "Killer Network Manager" from the "'Startup" folder of the Windows Start menu (under "All Programs") and (2) using MSConfig, uncheck "Qualcomm Atheros Killer Service V2" under the "Services" tab. Then reboot and you're good to go. The Killer Network Manager and its service will not be running.

By the way, the reason we didn't choose this method, ourselves, is because we can see merit in having the Killer Network Manager available so we can block network traffic to select programs. Solution 1 makes this an easy matter because bandwidth control for ethernet can be quickly enabled again if it is ever desired. And we're leaving it enabled for our WiFi.

Kind regards, David-EH

Windows 10 Digital License

When you upgraded from a previous version of Windows or receive a new computer preinstalled with Windows 10, what happened is the hardware (your PC) will get a digital entitlement, where a unique signature of the computer will be stored on Microsoft Activation Servers. The Windows 7 or Windows 8 genuine license you were previously running will be exchanged for a diagnostics key.

Anytime you need to reinstall Windows 10 on that machine, just proceed to reinstall Windows 10. It will automatically reactivate. You will prompted to enter a product key a couple times through the installation, click I don't have a key and Do this later.

There is no need to know or get a new key, but if you have issues with Digital Licensing or the Activation Troubleshooter, you can utilize an existing Windows 7 or Windows 8/8.1 product key you already own to reactivate if necessary. or use the reset function in Windows 10.

a_user_here say:

General notes about Windows 8.1 to Windows 10 differences.

In Windows 10 there is both the Start menu and the Start Screen.  By default, the Start menu is used for desktop/laptop computers, and the Start Screen for tablet/mobile devices (Start Screen can be force on for desktop in Start -> Settings -> Personalization -> Start, and can be toggled by selecting the "notifications" icon by the clock, and selecting a button for tablet mode).  Also for hybrid devices like the Surface, the defaults are Start Menu if the keyboard is connected, and Start Screen if it is disconnected.

In Windows 8.1, if you are not in the desktop you are most likely on the Start screen.  When you start typing on the Start Screen it just takes that as a search, and that is what you were doing.

If you are in tablet mode with (or just forced Start Screen) Windows 10 will behave the same.

When you are using the Start Menu you can click on the Start Button and then start typing your search.  This actually the same as clicking on Cortana's button, and start typing.  Cortana is basically search now.  If you are looking for an application it will find it right away and be at the top of the list, for other things Cortana might answer, or if it doesn't know it will just do a search in your web browser.  Personally I think that last one should be configurable, since I can do web searches on my own.  Most people really are just intending to search their machine.

On split screen.  When in tablet mode it works just like on Windows 8.1 as far as splitting the screen.

In desktop mode since everything is in a window you use snap instead to do basically the same thing.

See this on how to use snap:

https://www.yahoo.com/tech/how-to-split-your-screen-in-windows-10-twice-in-125541466359.html
