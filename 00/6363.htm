So it sounds like your problem is that you don't know how to add or edit page numbers in any document? Are you accessing the header by double-clicking? If the page numbers were added using the Insert | Page Numbers... dialog in Word for Windows 95, they'll be in frames. Before you do anything else, you need to get rid of these page numbers and the frames (see http://word.mvps.org/FAQs/Formatting/UnaccountablyIndented.htm for what can happen if the empty frame gets left behind). You can insert a simple page number anywhere in the document (including the header or footer) by pressing Alt+Shift+P. You can also insert "fancy" page numbers using the Insert | Header & Footer | Page Number feature, but if there is other header/footer content, be sure you use Current Position rather than any of the other choices, which will insert a building block that will delete the existing header/footer.

suzanne s. barnhill say:

There are several factors here:

  1. If you copied the file to your HD from a CD, it will be Read Only. You will need to right-click on the file in Windows Explorer, choose Properties, and remove the Read Only flag. If you are able to open the files but not edit them, this is probably the only problem.
  2. If the file was created in Word for Windows 95 or earlier (anything before Word 97), it will be blocked, and you may not be able to open it at all (you may get an error message).
  • If you have Word 2010 or 2013, you can go to File | Options | Trust Center | Trust Center Settings | File Block Settings and check/clear the boxes needed to allow Word to open files in this older format.
  • If you have Word 2007, see http://support.microsoft.com/kb/922849/en-us (this article actually has instructions for Word 2003, 2007, 2010, and 2013).
  • If you have Word 2003, see http://support.microsoft.com/default.aspx?kbid=938810.

I was able to fix a similar search problem (on Windows 10, for OneNote 2013) by deleting the OneNote cache file. This solution, I think, is more specific and therefore probably much faster than rebuilding the entire index in windows, which also works, as suggested elsewhere, but is slower.   

Here are the steps you have to do:

  1. Close OneNote
  2. In Windows Explorer, go to folder "C:\Users\[username]\AppData\Local\Microsoft\OneNote\15.0"
  3. Find the file "OneNoteOfflineCache.onecache" and then either delete it, move it, or rename it. 
  4. When you open OneNote again, the cache file gets automatically recreated (this may take a while). You may also have to re-sync your notebooks if they are located on OneDrive
