If you're contacts are showing as offline and they cannot receive your messages or vice versa, here are the steps that you may try to do and then check if the issue persist:

Windows 10 (UWP)

  1. Log out and Quit Skype.
  2. Press Windows key or click Cortana and type in Skype.
  3. Right-click Skype and choose uninstall.
  4. Press Windows key and R on your keyboard at the same time and type in %appdata% on the Run dialog box and click OK.
  5. Open the Skype folder and delete shared.xml folder.
  6. On the same folder, look for your Skype name and open the folder, delete config or config.xml folder.
  7. Restart your computer.
  8. Download and install Skype.

Windows desktop

  1. Log out and Quit Skype.
  2. Press Windows key and R on your keyboard at the same time. Type appwiz.cpl on the Run dialog box and click OK.
  3. Right-click the Skype icon and uninstall it.
  4. Press Windows key and R again and type %appdata% and click OK.
  5. Open the Skype folder and delete shared.xml folder.
  6. On the same folder, look for your Skype name, open it and delete config/config.xml folder.
  7. Restart your device.
  8. Download and install Skype.

Mac

  1. Log out and Quit Skype.
  2. Click anywhere on your screen and click GO on the top menu bar.
  3. Choose Go to folder and copy and paste or type ~/Library/Preferences and click GO.
  4. Drag com.skype.skype.plist to the trash and empty it.
  5. Restart your device.
  6. Re-launch Skype and sign in to your account.

iOS

  1. Log out from Skype.
  2. Tap and hold Skype and tap the red X button.
  3. Restart your phone and re-install Skype.

Android

  1. Log out from Skype.
  2. Go to your phone Settings.
  3. Tap App/Application manager.
  4. Swipe to the Downloads tab.
  5. Tap Skype then tap uninstall.
  6. Restart your phone and reinstall Skype.

Alternatively, tap and hold the Skype icon and drag it to the trash icon.

Proper table structure for Equipment and specs

I've been given a db with what I consider a questionable table structure and before I go any further was wondering if I should be reworking it.  The db is used to track our equipment, and their assemblies, etc....

As it stands, the original designer setup a table per type of equipment: pumps, motors, ...

My issue with this is that how can I set it up so when the user specifies say a pump, it automatically display the specifications that they should be filling in (and not those of other types of equipment).  Is this all to be done using VBA?

QuestionBoy
ken sheridan say:
But my question remains.  In the entry form, when the user selects that the piece of equipment is a pump, how do I show the '10' specification fields relating to a pump?  Same for a motor, show the '17' specification fields relating to it, and so on?

Within the form based on Scott's tblEquipment table you'd have a subform based on the tblSpecification table.  In the AfterInsert event procedure of the parent form you'd execute an SQL statement which inserts rows into tblSpecification, one for each row in tjxSpecCategory with the CategoryID of the category of equipment entered into the parent form.  So for a pump  10 rows would be inserted, for a motor 17 rows would be inserted.  Units is an attribute of tluSpec, for which a column can be added to that table, so the only thing to be entered by the user would be the specification value.

After executing the SQL statement to insert the rows into tblSpecification  the subform would be requeried to show the new rows, ready for the user to insert the values.

You'll find an example of this sort of operation, albeit in a different context, in the file StudentLog.zip in my public databases folder at:

https://skydrive.live.com/?cid=44CC60D7FEA42912&id=44CC60D7FEA42912!169

In the attendances subform in the courses form is a button for entering all students registered for a particular course as attending a session of the course.  The code is:

    Dim strSQL As String
   
    strSQL = "INSERT INTO CourseAttendances" & _
        "(StudentID,CourseID,CourseDate) " & _
        "SELECT StudentID, " & Parent.[CourseID] & ", #" & _
        Format(Parent.[sfcCourseSessions].[Form].[CourseDate], "yyyy-mm-dd hh:nn:ss") & _
        "# FROM CourseRegistrations WHERE CourseID = " & Parent.[CourseID]
   
    CurrentDb.Execute strSQL
   
    Me.Requery

If you think of courses as analogous to your categories, sessions of a course as analogous to items of equipment of a category and students as analogous to specifications then you'd do something similar, but it would be executed automatically when a new row is inserted into tblEquipment rather than electively on the click of a button.


scottgem say:
Actually an even better structure uses five tables:

tblEquipment
EquipmentID (PK autonumber)
Description
CategoryID (FK)
PurchaseDate
SerialNumber
other info common to all equipment

tblSpecification
SpecificationID (PK autonumber)
EquipmentID (FK)
SpecID (FK)
SpecificationValue

tluSpec
SpecID (PK Autonumber)
Spec

tluCategory
CategoryID (PK Autonumber)
Category

tjxSpecCategory
SpecID (FK)
CategoryID (FK)

tblSpecification has the specs for each piece of equipment but in a vertical structure. so each record identifies the piece of equipment, the specification being records and the value of specification. I.E. Water Pump, Flow, 10 Qt per hour. tluSpec has the list of specifications that will be needed for all types of equipment. tluCategory, the list of the different types of equipment and tjxSpecCategory, defines which categories each spec is for so you can filter your comboboxes to only specs for the type of equipment data is being entered for. 

This gives you maximum flexibility.
