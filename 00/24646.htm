    scottgem say:
    Not only is is possible, but Access has a wizard for that. If you use the combobox wizard, the third option is for a search combo. Generally I put that in a form header to differentiate it itself from the rest of the form. When a user select a record from that combo, the record pointer is changed to the selected record. 

    As John noted, when you create a form that has a table as its recordsource, the form is connected to that table and any changes made through the form are reflected in the table. This makes editing easy, but it can also be dangerous. All too often I've had data changed by a user who just started typing in a form thinking they were adding a new record. So, as the developer, you need to protect against that.

    One way I do that is to have menu options that open a form in EITHER Add or Edit mode. One of the parameters of the OpenForm method is the Datamode which can be either Add, Edit or ReadOnly. In add mode, users can only add new records. In Edit mode they can see and edit previous records.

    So you may want to give your data entry people the option to only open forms in Add mode, while others can also use Edit mode.
