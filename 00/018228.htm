IncludeText in IncludePicture? - Microsoft Community

a. user say:

IncludeText in IncludePicture?

Hi!

I got an online image generator that will be embeded in a word document. It takes a parameter, for example a name, and returns a customised image. For example:

http://site.com/generator.php?currentWord=apple

I can embed this using {INCLUDEPICTURE "http://site.com/generator.php?currentWord=apple" \d}

But... the currentWord parameter is also a dynamic string that is fetched over http. For example:

http://site.com/currentWord.txt (returns a random word, like: 'apple', 'car', 'house')

I can also include this word in my document using

{INCLUDETEXT "http://site.com/currentWord.txt"}

What I need is the INCLUDETEXT inside of the INCLUDEPICTURE, but this doesn't work:

 {INCLUDEPICTURE "http://site.com/generator.php?currentWord={INCLUDETEXT "http://site.com/currentWord.txt"}" \d}

(In the example above, currentWord will be an empty string, whilst I it should be the random word fetched from the currentWord.txt file).

Any ideas?

Thanks in advance!

a. user say:

1. Did you resolve the problem you described in your earlier post? If not, the problem you are facing is probably none of the ones I tried to address in my response, so it would be useful to know that.

2. Here, with the same construct, when I select the fields and update them (e.g. with F9) (with all the {} being the special field code brace pairs, as usual), Word does retrieve the content of the .txt file even inside the nested field, not just outside the nested field.

3. However, I do not think the approach you are attempting will work as it stands even if the INCLUDETEXT field does update correctly, because 

 a. in the case of a .txt field, INCLUDETEXT will always insert a paragraph mark at the end of the included file, even if the file only contains the exact text "apple" (i.e. has 5 bytes and no 0A, 0A or other control code bytes).

 b. so the INCLUDEPICTURE will try to include a file that has a paragraph mark in its name, which is unlikely to work whatever you do. 

 c. I don't think you can work around that by using "tricks" such as nesting the INCLUDETEXT field within a QUOTE field or trying to insert it using something like  { =1 \#"'{ INCLUDEPICTURE ....... }'" }

Probably the simplest solution to that would be to encode the word ("apple" or whatever) in an HTML file instead of a text file, with the word wrapped inside an anchor, e.g. in a file called currentWord.html with content like this:

<html><body><a name=theword>apple</a></body></html>

Then use 

{ INCLUDETEXT "http://site.com/currentWord.html" theword }

or if you are being precise, 

{ INCLUDETEXT "http://site.com/currentWord.html" theword \c html } (I think)

to include the word - that should eliminate the trailing paragraph mark. You could encode the word in an rtf file instead, but I think the HTML is simpler. I think Word will let you get away with just this:

<html><a name=theword>apple</a></html>

Again, if it's part of a mailmerge, you may need to consider using the longer encoding for INCLUDEPICTURE that I described in my previous response.
