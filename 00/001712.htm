joeu2004 say:

The issue applies to all versions of Excel.  That problem arises because most decimal fractions (e.g. x.12) cannot be represented exactly in 64-bit binary floating-point, which is how Excel represents numbers and performs arithmetic internally.  And the binary approximation of a particular decimal fraction depends on the magnitude of the integer part (211.x).  Consequently, for example, IF(10.01 - 10 = 0.01, TRUE) returns FALSE(!).

The work-around is:  whenever you intend for a calculation to be accurate to some number of decimal places (2, in your example), explicitly round to that number of decimal places.  (And not to an arbitrary number of decimal places, like 10, as some people suggest.)  For example, IF(ROUND(10.01 - 10, 2) = 0.01, TRUE) returns TRUE, as expected.

john mcghie say:

Hi Moller:

Footnotes become footnotes when they begin with a generated number.

The number generates a tag, and it's the tag the cross-reference employs.

In that document, it sounds like the numbering is corrupted.  To fix that, you will need a "Maggie".  But before you do the Maggie you will need to remove that section break (because it is probably the location of the problem.  You can replace it after you do the Maggie:

The Maggie:

1. Create a new blank document in .docx format

2. Carefully select all of the text in the bad document EXCEPT the last

paragraph mark

3. Copy it.

4. Paste in the new document.

5. Save under a new file name and close all, then re-open.

Now: be careful about pasting disparate footnotes from one document to another: this has a high possibility of returning the problem.

The Footnote Reference character is actually a storage container that contains all the details of a footnote (including which numbering scheme it belongs to).  If you paste between documents, you are likely to suffer a return of the problem where the documents have different numbering schemes for their footnotes.

Sometimes, it's better to past as Plain Text.  That will convert the footnote to ordinary text.  You can then re-create it as a footnote in the destination document: where it will be automatically added to the correct collection.

It makes no difference whether the document was created on Windows or Mac, they both use the exact same format (it's not just "compatible", it's "exactly the same").  However, trying it between .doc and .docx is a recipe for trouble.

Hope this helps a little.
