Apply new template to existing files in batch? - Microsoft Community

osu9400 say:

Apply new template to existing files in batch?

I have about 30 documents that all created with normal.dot. The files have all kinds of crazy font formatting.

For all documents, how do I:

1. Clear the formatting

2. Apply a new template

greg maxey say:

With VBA.  Something like this:  (Note this code clears all formatting both paragraph and font in all storyranges and applies the normal style.) 

Sub BatchProcess()
Dim strFilename As String
Dim strPath As String
Dim oDoc As Document
Dim fDialog As FileDialog
Set fDialog = Application.FileDialog(msoFileDialogFolderPicker)
With fDialog
  .Title = "Select folder and click OK"
  .AllowMultiSelect = False
  .InitialView = msoFileDialogViewList
  If .Show <> -1 Then
    MsgBox "Cancelled By User", , _
     "List Folder Contents"
    Exit Sub
  End If
  strPath = fDialog.SelectedItems.Item(1)
  If Right(strPath, 1) <> "\" _
  Then strPath = strPath + "\"
End With
If Documents.Count > 0 Then
  Documents.Close SaveChanges:=wdPromptToSaveChanges
End If
If Left(strPath, 1) = Chr(34) Then
  strPath = Mid(strPath, 2, Len(strPath) - 2)
End If
strFilename = Dir$(strPath & "*.doc")
While Len(strFilename) <> 0
  Set oDoc = Documents.Open(strPath & strFilename)
  ClearAllFormattingAndAttachTemplate oDoc
  oDoc.Close SaveChanges:=wdSaveChanges
  strFilename = Dir$()
Wend
End Sub
Sub ClearAllFormattingAndAttachTemplate(ByRef Doc As Word.Document)
Dim lngJunk As Long
Dim rngStory As Word.Range
Dim oShp As Word.Shape
lngJunk = Doc.Sections(1).Headers(1).Range.StoryType
'Iterate through all story types in the current document
For Each rngStory In Doc.StoryRanges
  'Iterate through all linked stories
  Do
    ClearFormatting rngStory
    On Error Resume Next
    Select Case rngStory.StoryType
      Case 6, 7, 8, 9, 10, 11
        If rngStory.ShapeRange.Count > 0 Then
          For Each oShp In rngStory.ShapeRange
            If oShp.TextFrame.HasText Then
              ClearFormatting oShp.TextFrame.TextRange
            End If
          Next
        End If
      Case Else
        'Do Nothing
    End Select
    On Error GoTo 0
    'Get next linked story (if any)
    Set rngStory = rngStory.NextStoryRange
  Loop Until rngStory Is Nothing
Next
Doc.AttachedTemplate = "C:\Letter.dot"
Doc.UpdateStyles
End Sub
Sub ClearFormatting(ByRef oRng As Word.Range)
With oRng
  .ParagraphFormat.Reset
  .Font.Reset
  .Style = ActiveDocument.Styles(wdStyleNormal)
End With
End Sub

yves dhondt say:

First thing to do is create a copy of the file APA.XSL to something else e.g. MyAPA.XSL.

Next ensure Word is closed.

Next open your new file, and look for a piece of code looking like this:

      <xsl:when test="b:OfficeStyleKey">
        <xsl:text>APA</xsl:text>
      </xsl:when>

Replace it by:

      <xsl:when test="b:StyleName">
        <xsl:text>My APA</xsl:text>
      </xsl:when>

Save the file.

Start Word. Now if you go to the dropdown of bibliography styles on the references tab, a new entry should be added labelled "My APA" (of course you can pick your own name).

To sort entries in a bibliography, the XSL generates an extra element for all sources called b:SortingString. Based on the value of the content of that element, sorting is done. So what you want to do is change that value to fit your needs.

Look in the code for "<b:SortingString>". It should only appear once. The code directly after it decides on how to sort entries. It starts with formatting the authors by generating a "author0" and "author" variable. You can ignore that part. The interesting part comes right after that:

            <xsl:if test="string-length($author) > 0 and b:SourceType != 'Case'">
              <xsl:text>&#32;</xsl:text>
              <xsl:value-of select="$author"/>
            </xsl:if>

            <xsl:choose>
              <xsl:when test="b:SourceType='Patent' and string-length(b:PatentNumber) > 0">
                <xsl:text>&#32;</xsl:text>
                <xsl:value-of select="b:PatentNumber"/>
              </xsl:when>
              <xsl:when test="string-length(b:Title) > 0">
                <xsl:text>&#32;</xsl:text>
                <xsl:value-of select="b:Title"/>
              </xsl:when>
            </xsl:choose>

            <xsl:if test="string-length(b:Year) > 0">
              <xsl:text>&#32;</xsl:text>
              <xsl:value-of select="b:Year"/>
            </xsl:if>

What it does is pretty straightforward. The first "xsl:if" displays the author if there is one and you are not citing a legal case. Then the "xsl:choose" is used to show the title (if your not citing a patent). Finally, the second "xsl:if" displays the year if there is one. So basically, the current sorting is

author title year

while what you probably want is

author year title

So all you have to do is switch the xsl:choose and the second xsl:if. So you would get:

            <xsl:if test="string-length($author) > 0 and b:SourceType != 'Case'">
              <xsl:text>&#32;</xsl:text>
              <xsl:value-of select="$author"/>
            </xsl:if>

            <xsl:if test="string-length(b:Year) > 0">
              <xsl:text>&#32;</xsl:text>
              <xsl:value-of select="b:Year"/>
            </xsl:if>

            <xsl:choose>
              <xsl:when test="b:SourceType='Patent' and string-length(b:PatentNumber) > 0">
                <xsl:text>&#32;</xsl:text>
                <xsl:value-of select="b:PatentNumber"/>
              </xsl:when>
              <xsl:when test="string-length(b:Title) > 0">
                <xsl:text>&#32;</xsl:text>
                <xsl:value-of select="b:Title"/>
              </xsl:when>
            </xsl:choose>

Let me know if that works out.
