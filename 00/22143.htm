robert aldwinckle on forums say:

Make sure the web address http://10.187.1.53:15871 is correct.

That syntax implies that you have a host which is listening on port 15871 instead of 80 for HTTP protocol.  Is there really such a server and is your firewall allowing such a request to pass through it?

Tip:  you can emulate a browser making a request to a specific port regardless of the protocol by using  telnet  in a cmd window.   So, for your example the syntax would be

telnet.exe  10.187.1.53  15871

If the screen cleared that would be a sign that you had reached a server using that address.  Otherwise you would have confirmation, independent of IE that there was a problem with that address.

graham mayor say:
You cannot open a PDF file in Word. Word is not PDF file compatible. You either need to use a PDF editor, such as Adobe Acrobat (there are cheaper products available); or if the converted file is too different from the original, you would be better editing the original Word document and convert it to PDF again.

stefan blom say:

Although Word can open PDFs, the result will vary depending on how the PDF was first created. For example, some files (especially files created by a scanner) may be treated by Word as pictures of pages rather than editable text. No matter if Word can (successfully) edit the PDF or not, there will always be a conversion when Word opens the file (because PDF is not a native file format in Word).

As Graham wrote, editing the original Word document, in Word, will be the best way to deal with this.
