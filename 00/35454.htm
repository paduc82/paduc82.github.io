Supported answer:  you can't.  There is no user-accessible way to clear those entries.

 

Better answer:  you can clear them in the registry.  Here's how:

  1. Press -R to open the Run box, and run the command regedit

  2. In Registry Editor, navigate to this key (folder):

    Hkey_Current_user\Software\Microsoft\Windows\CurrentVersion\Applets\WordPad\Recent File List

  3. Select the file you want to remove in the area to the right, and delete it.

Spooler Log File Cleanup - Microsoft Community

donpwalker say:

Spooler Log File Cleanup

My Spoolerlogs file (spooler.xml) has grown to 69,246.1 MB.  How can I reduce it and keep it from growing so large in the future?
lemp say:
Most of the information about spooler.xml seems to be related to various versions of Windows Server that the typical home user doesn't have.  On the other hand, you're not unique in having this issue in Windows XP.

The spoolererrorlog (i.e., spooler.xml) appears to be related to the Windows Error Reporting Service.  See, for example, KB 960919, which relates to Windows Server 2008.

This Technet Article discusses how to disable logging printer events for Windows Server 2003, Windows Server 2003 R2, Windows Server 2003 with SP1, Windows Server 2003 with SP2, but the options that are described are also available in XP.  Click Start > Printers and Faxes > File > Server Properties > Advanced.

KB 310414 describes how to disable Windows Error Reporting or to configure it to be used only for certain types of errors.  In Windows XP, Error Reporting is the mechanism that, when an error occurs, displays a dialog box that prompts you to report the problem to Microsoft.  I was not aware that it also might create a log on the computer's hard drive.

Personally, my approach to Windows Error Reporting has been simply to use Start > Run > services.msc > OK to set the "Startup type" of the "Error Reporting Service" to "Disabled."  See http://www.blackviper.com/wiki/Error_Reporting_Service#Windows_XP

As to your spooler.xml file, there doesn't seem any good reason not to simply delete it.  If it won't delete because it's "in use," stop the Error Reporting service and then delete it.
