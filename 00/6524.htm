After installing this build, you may see a boot selection menu each time you reboot your PC. This is because a second boot option has been added for uninstalling the Windows 10 Technical Preview.

When you get to this screen, if you do nothing, it will automatically boot to the Windows 10 Technical Preview within 30 seconds by default (which you can modify) or you can choose it on your own. In future builds, youâ€™ll be able to uninstall through the Windows Recovery Environment (WinRE).

If you would like to remove the rollback option from the boot up experience:

  1. From the Start menu or the Search icon, search for cmd

  2. Press-and-hold or right-click Command Prompt from the search results and select Run as administrator

  3. Type bcdedit /timeout 0 at the command prompt and hit ENTER

You can get the uninstall option back in the boot menu by following these steps again, but change the timeout value to something between 1 and 30, instead of zero.

A few words of warning about the new Uninstall feature:

  1. If you run Uninstall, youâ€™ll need to know your password from your previous OS.  Account information is not saved from Windows 10 Technical Preview.  Instead, the accounts from your previous OS will be put back in its entirety.
  2. Any built-in, modern apps from Windows 8.1 that were updated or replaced for Technical Preview will not work after you Uninstall back to Windows 8.1.  You can repair these apps by uninstalling the app manually and then re-downloading them from the Windows Store.
  3. There are several ways you can inadvertently break or disable the Uninstall feature:
    • Running Reset or Refresh on Tech Preview will remove the Uninstall option
    • Running Disk Cleanup and removing the Windows.old folder will remove the Uninstall option
    • Adding a new user account in Tech Preview will remove the Uninstall option
    • Removing individual files from the Windows.old folder, without deleting the whole folder, may still allow the Uninstall option to be run.  However if you do, Uninstall may not only fail, it may cause your PC to fail to boot into Windows.

If this does occur, you will need to either do a clean install of your previous OS using installation or recovery media for your previous operating system; or you can do a clean install of build 9926 of Technical Preview by downloading the ISO from another computer, creating installation media, and then booting to the installation media and running the setup program, selecting to keep your personal files when prompted.

scottgem say:

This is not a fully normalized table, but its not as bad as I thought it might be.

Who says you only need to select from a combobox? You can still type in the value and press Enter. If the date doesn't exist, the user will get a message that it doesn't exist, if it does then that record will be retrieved.

Note, you do not want them to "automatically fill the data", what you want is to retrieve the matching record. There is a difference.

Now a better structure would be like this:

tblShiftReport

ShiftReportID (PK Autonumber)

ShiftDate

PartofDay (AM or PM)

EmployeeID

Comments

tblShiftIssues

ShiftIssueID (PK Autonumber)

ShiftReportID (FK)

IssueTypeID (FK)  (i.e. Gate, Ramp, etc.)

IssueDescription

You enter this data in a mainform/subform. You can then have a search box to select date and part of day to retrieve that record.
