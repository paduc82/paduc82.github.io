brian tillman [outlook mvp 2007-2019] say:

First, keep in mind that in order to see anything on your screen, it must be on your hard drive.  There are no exceptions to this.

That said, when you allow Outlook 2013 to connect to your Outlook.com mailbox using its automatic account creation dialogs, it will set up an Exchange ActiveSync connection to the mailbox.  This is a two-way connection.  Messages stay on the server, but copies of those messages are also downloaded to your hard drive, in a file called an OST file.  Anything added on the server side is echoed in the local cache and vice versa.  If you delete a message locally, it will be deleted on the server.  This applies not only to the mail folders you have, but to contacts and calendar as well.

There are also other ways to use Outlook 2013 to access your Outlook.com mailbox.  IMAP is also a synchronizing protocol that makes a cache of your mailbox on your hard drive and it syncs in both directions.  Messages stay on the server unless you explicitly delete them.  IMAP, however, is mail-only and the calendar and contacts on the server side are ignored because IMAP can't handle them.

The third protocol is called POP3 and this is the one you're recalling.  POP3 does not synchronize.  Instead, it downloads the content of the mailbox's Inbox and puts it in the Inbox Outlook 2013 creates in a file called the PST.  Ordinarily, POP3 accounts delete messages from the server once they're downloaded, but I believe the default setting for Outlook 2013 is to leave copies on the server.  Moreover, Outlook.com has a setting that disobey your mail client if it tells the server to delete messages from the Inbox, moving them to a folder named POP on the server.  The POP3 protocol knows nothing about folders on the server side and so it can see only the Inbox.  If you put messages in folders in Outlook 2013 for a POP account, they're local to your machine only.  Again, since POP3 is mail-only, calendar and contacts on the server can't be synched.

EAS (the first protocol I mentioned) is a good way to access an Outlook.com mailbox.  All devices, such as phones and tablets, capable of using the EAS protocol will see the exact same content as Outlook 2013 sees if they connect to that same Outlook.com mailbox.  Any changes made on any device (or by using a web browser to connect with the mailbox directly) will be reflected on all devices that access that same mailbox.  Contacts and calendar will carry across them all as well.
