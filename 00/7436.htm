@Scott....  Since you neglect to show us how you (mis)use FORECAST to get a result of 3.34 (rounded?), it is difficult for us to explain your misunderstanding and misuse.

I tried to imagine some possible misuses, but I cannot coerce FORECAST to produce a value that is rounded to 3.34.

So please show us the concrete example of using FORECAST that results in 3.34 (rounded, I presume).  If you have cell references, shows us their values displayed to 15 significant digits.

-----

If you want to linearly interpolate the proper "band" (band I, in this case), the correct use of FORECAST is:

=FORECAST(299875, {0, 3.33}, {0, 299999.99})

In Excel 2010, the result is 3.32861261095375, which is less than 3.33 and which rounds to 3.33.

By the way, we can generalize the formula so that you do not have to hardcode each pair of band limits.  Let us know if you need help with that.

-----

Two possible misuses....

=FORECAST(299875, {0, 3.33, 3.34, 6.66, 6.67, 10},
{0, 299999.99, 300000, 600000, 600000.01, 900000})

=FORECAST(299875, {0, 10}, {0, 900000})

They result in 3.33224773735826 and 3.33194444444444 respectively.  Greater than 3.33, yes.  But neither rounds to 3.34.

They are not exactly the same as the first formula above because of the forced rounding of the band-wise limits.  Instead, the latter two formulas interpolate a line that "best fits" the specified data points.

Caveat:  Your band-wise limits make sense only if you round the data point (e.g. 299875) to 2 decimal places.  Otherwise, you have discontinuities between the bands.

joeu2004 say:

Scott wrote:

joeu2004 showed via FORECAST if  use all defined upper & lower boundaires; however I'd like to see a solution for just the 4 boundary values in the range row (1st row) in mye xample. joeu2004 showed 0,333, 3.334, 6.66, 6.67, 10. My range values are 0, 3.34, 6.67, and 10.

It is not likely that using FORECAST with more than 2 data points will get the exact results that you expect.

As I tried to explain previously, when there are more than 2 data points, FORECAST interpolates along a "best fit" line.

In general, that line might not go through any of the specified data points.  Likewise, interpolating data between the specified data points might not stay within the limits that you intended.

The only way to accomplish what you expect is to do pairwise interpolations.

Suppose X1:X5 contains 0, 3.34, 6.67, 10 and 11, and Y1:Y5 contains 0, 300000, 600000.01, 900000 and 900001.  Then:

=FORECAST(ROUND(A1,2), OFFSET($Y$1, MATCH(ROUND(A1,2), $X$1:$X$4, 1) - 1, 0, 2, 1),

OFFSET($X$1, MATCH(ROUND(A1,2), $X$1:$X$4, 1) - 1, 0, 2, 1))

provides the pairwise interpolation.

The values in X5:Y5 are arbitrary.  They work (i.e. do not alter the result for A1=10) because you said the max (rounded) value in A1 is 10 (Y4).

But that does not return 299999.99 for 3.33 and 660000 for 6.66, which I thought was your requirement and original concern.

For that, you would need the pairwise data points that I specified in an earlier response in this thread, to wit:  X1:X7 is 0, 3.33, 3.34, 6.66, 6.67, 10 and 11, and Y1:Y7 is 0, 299999.99, 300000, 600000, 600000.01, 900000 and 900001.  Change the ranges appropriately in the formula above.

[EDIT] Note the correction to the X and Y ranges.

PS....  Normally, I avoid using OFFSET because it is a "volatile" function; so that formula and any cells that are directly or indirectly dependent are recalculated every time any cell in the workbook is edited.  That can great adversely impact recalculation performance if there are very many instances of the formula.  Alternatively, calculate MATCH(...)-1 in a "helper" cell (e.g. B1) and replace OFFSET with INDEX($Y$1:$Y$5, B1, 1):INDEX($Y$5, B1+1, 1), for example.
