Some printers have a larger unprintable area than others, and Word will generally insist on margins that are within the printable area and warn you if they are outside them. It's possible that your colleague's printer has a larger printable area. As a general rule your printer will print what you see in Print Preview, or, to say it another way, Word's Print Preview (which is based on information provided by the printer driver) should reliably indicate what will print. If you want to print the document from your computer, on your printer, then you will need to reformat it to accommodate the limitations of that printer.

ronrosenfeld say:

It seems that what you want to do is to first arrange the names in date order, then sort by this custom name order, and then by date.

Excel allows you to perform a sort by a custom order, but you need to develop the order.  This can be done using a VBA macro.

You can see in the macro below where you set the Source and Results ranges and worksheets.  Feel free to edit as required.

I assumed your source data was in Sheet1 columns A:C  and that the results would go into columns F:H, but that's easily changed

To enter this Macro (Sub), <alt-F11> opens the Visual Basic Editor.
Ensure your project is highlighted in the Project Explorer window.
Then, from the top menu, select Insert/Module and
paste the code below into the window that opens.

To use this Macro (Sub), <alt-F8> opens the macro dialog box. Select the macro by name, and <RUN>.

========================================

Sub SortTVShows()
    Dim colNames As Collection
    Dim vSrc As Variant, vRes As Variant
    Dim I As Long, J As Long, K As Long
    Dim wsSrc As Worksheet, wsRes As Worksheet
    Dim rRes As Range, rSrc As Range
    Dim sSortArray() As String
Set wsSrc = Worksheets("Sheet1")
Set wsRes = Worksheets("Sheet1") '<-- could put results on a different sheet
Set rRes = wsRes.Range("F1")

Set rSrc = wsSrc.Range("a1").CurrentRegion '<--could use a different selection method
                                        '   as appropriate
vSrc = rSrc

'copy original to results area
'could overwrite if needed

Application.ScreenUpdating = False

Set rRes = rRes.Resize(UBound(vSrc, 1), UBound(vSrc, 2))
rRes.EntireColumn.Clear
rRes = vSrc

'Sort by date
With rRes
    .Sort key1:=.Columns(3), order1:=xlAscending, Header:=xlYes
End With

'Get names in date order for custom sort of Name column
vRes = rRes
Set colNames = New Collection
On Error Resume Next
For I = 2 To UBound(vRes, 1)
    colNames.Add Item:=vRes(I, 1), Key:=vRes(I, 1)
Next I
On Error GoTo 0

ReDim sSortArray(1 To colNames.Count)
For I = 1 To colNames.Count
    sSortArray(I) = colNames(I)
Next I

With Application
J = .GetCustomListNum(sSortArray)
If J = 0 Then
    .AddCustomList sSortArray
    J = .CustomListCount
End If
End With
    

'sort by Custom Name order, then by date
Set rRes = rRes.Offset(1, 0).Resize(rRes.Rows.Count - 1)
With wsRes.Sort
    With .SortFields
        .Clear
        .Add Key:=rRes.Columns(1), SortOn:=xlSortOnValues, Order:=xlAscending, CustomOrder:=J, DataOption:=xlSortNormal
        .Add Key:=rRes.Columns(3), SortOn:=xlSortOnValues, Order:=xlAscending, DataOption:=xlSortNormal
    End With
    .SetRange rRes
    .Header = xlYes
    .MatchCase = False
    .Orientation = xlTopToBottom
    .Apply
End With
    
Application.DeleteCustomList (J)
rRes.EntireColumn.AutoFit

Application.ScreenUpdating = True

End Sub
