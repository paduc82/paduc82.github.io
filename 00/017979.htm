You've used both SEARCH() and FIND(). SEARCH() is not case sensitive while FIND() is. Since you are looking for a hyphen (there is no upper case hyphen), it doesn't matter which but it might make more sense to use one or the other. Some error control for blank cells might help.

jason (office engineer) say:

Lauren,

The problem appears to be the type of the file you are trying to upload.  It is being reported by your computer as "application/jpeg" instead of "image/jpeg".  There are two reasons this could be happening:

  1. The file is not truly a jpg even though the file extension says .jpg.  This happens often with images and there's really no way to tell without looking at the guts of the file.  I don't think this is your issue since you said it was a screenshot that you created - it would probably only be this case if you used a special program to create the screenshot instead of just Alt+Print Screen.  In order to fix this case, you would open the image in a program like MSPaint.exe and resave as a jpg and then try to put it into Sway again.
  2. What I think is the more likely case is jpg files are registered on your machine improperly.  This often happens when some over-zealous software decides it wants to be the one program that can handle images and keep others from using it and then it, basically, changes the definition of what it means to be a jpg.  Wen you double-click on a jpg on your machine, does it open in a non-standard program?  If so, this is likely the case.  To fix, you'll want to update how jpgs are registered on your machine.  There are likely articles on how to do this if you do a web search, but one way to do this would be to right-click on a jpg on your machine, choose Open With, and then choose a more standard program like paint or photo viewer and set that to be the default program to open it with from now on.

Please reply back and let me know if this helps.  If it's still not working, I could do some more research online to point you at a more definitive fix for #2.

Jason
