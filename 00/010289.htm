doug robbins - mvp office apps & services (word) say:

Use a macro containing the following code:

Dim rng As Range
Selection.HomeKey wdStory
With Selection.Find
    Do While .Execute(FindText:="T0A", Forward:=True, _
    MatchWildcards:=False, Wrap:=wdFindStop, MatchCase:=True, MatchWholeWord:=False) = True
        Set rng = Selection.Range.Words(1).Range
        Selection.Collapse wdCollapseEnd
        rng.Font.ColorIndex = wdGray50
    Loop
End With

barkoosh say:

Since Word is going to automatically replace every dot between two digits with this decimal separator, we need to insert something next to the dot to make Word think that this is not a decimal number. The problem is that in Persian, as in Arabic, numbers are written from left to right while text is written from right to left.

Let's take for example 123.456:
- Start with typing (the Persian) 456, then type the dot.
- Go to INSERT > Symbol > More Symbols.
- Click the Special Characters tab, and from it choose "Right-to-Left Mark" (I think that, in this case, the "Left-to-right Mark" works too), click Insert, then Close. (You may assign a shortcut key to it, if you're going to use it more often.) The mark would then be inserted after the dot, but it's not visible.
- After that type (the Persian) 123.

steve rindsberg say:

Totally annoying, isn't it?

One workaround: 

Use the tab key to cycle through the shapes on your slide.  When you tab to a shape, PPT will move the window so the shape appears, even if it was offscreen beforehand.

To speed this up so you don't need to tab through endless shapes, move the off-slide shapes to back (the backmost shape will be selected immediately if you first select nothing then press tab).  Or move them to the front, and use Shift+Tab to cycle from front to back.

Once you've selected the shape, you can zoom in and the zoom will keep the shape on screen.

 

Not perfect, but it takes out some of the maddening.  ;-)
