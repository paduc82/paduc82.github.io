Hello Kay,

I saw your question as I was researching the same issue for one of my users. The problem you are experiencing occurs when you have the Document ID service enabled on a site collection. 

Here is the workaround which will help you find the "Absolute URL":

Please note - the following only appears to work with the "classic" library. I have not been able to do this in the modern library.

1. Do not use the url that has the doc id included - instead use the "absolute" URL

2. Select the file to which you want the hyperlink to point

3. Click on the "Files" tab in the ribbon

4. In the "Files" ribbon, select "View Properties"

5. Notice that the "Name" field is actually a hyperlink when you hover over it. Right click on the name and select "copy shortcut" This is the absolute URL

6. Paste the absolute URL into your word doc and users will have the normal open/edit experience

Please let us know if this helps!

Re read your post, this is what you really want

Public cn                               As New ADODB.Connection
Public rs                               As New ADODB.Recordset

Sub CopyTableFromAccess()
Dim a As Integer
With cn
    .ConnectionString = "Data Source=" & ThisWorkbook.Path & "\database.accdb"
    .Provider = "Microsoft.ACE.OLEDB.12.0"
    .Open
End With

rs.Open "Table1", cn
a = 2
With rs
    Do While Not .EOF
        Cells(a, 1) = .Fields("Clemson")
        Cells(a, 2) = .Fields("Tigers")
        a = a + 1
        .MoveNext
    Loop
End With
rs.Close
cn.Close
End Sub
