lemp say:
By default, Windows XP allocates 12% of the partition (drive) that is being monitored to use for System Restore.  For new large drives, that's far too much.  Experts recommend that you configure SR to use about 1 GB.  Yes, the restore points won't go back as far in time, but using SR to go back more than about a week or so is asking for trouble anyway.

Excellent source of System Restore information --> http://bertk.mvps.org/

Adjusting disk space used --> http://bertk.mvps.org/html/diskspace.html

Although System Restore is probably the primary user of the System Volume Information folder, it's not the only thing that puts data there.  The disk indexing system and encrypting file system, for example, put files in SVI.  And
ZoneAlarm version 6.5 had a bug that caused System Restore to grow very large.  See Why is the System Volume Information folder so large?

What "protection" software do you have installed?

Which directory did TreeSize show as growing:

C:\System Volume Information
or
C:\System Volume Information\_restore{long hex number}  



lemp say:
Even after years, I only have about 20G on my hard drive, so it was unusual when I started seeing the data volume climbing towards 40G.  This just started about 3 or 4 weeks ago....and I had not installed Any new programs.  

Now that I have set the Restore Settings to 25G, I have not seen any repeat of this "growth" in the past 2 or 3 days....so time will tell if this fixed my "quirk".

As I said, you should set SR to 1 GB ... not 25 GB.

I assume that the sentence I bolded above means that you have a separate partition for your data.  You need not have System Restore monitor that partition (drive) at all.  To extract a few bits from Bert K's site linked in my earlier post (emphasis added):

  • System Restore monitors system changes and saves the registry values will persist) and some dynamic system files.
  • The files that are monitored or excluded from monitoring are specified in the file %windir%\system32\restore\Filelist.xml.
  • To restore a system, System Restore undoes file changes made to monitored files, recapturing the file state at the time of the selected restore point. It then replaces the current registry with the one saved for the selected restore point.

If indeed you were monitoring a "data drive" and it was System Restore on that drive that was increasing, it sounds as if one of the applications that stores data on that drive does not abide by the following recommendations:

  • To ensure that your application has the desired behavior after a restore, do the following:
    • Ensure that key application binaries to be protected use extensions consistent with those used in Filelist.xml. For more information, see Monitored File Extensions.
    • Do not use monitored extension types for user-editable files. For example, if you name a user's personal data file using the extension .ini, the user may lose work as a result of a system restore.

janeane cru say:

Hi Johny,

We understand the frustration you experienced. However there are a few factors that we need to consider why your computer might have crashed like a possible 3rd party software that might have been corrupted, thus affecting the performance of your computer or it could also be that there might be a corrupted system file that needs repairing, and at worst cases a faulty driver that could also lead to hardware problems. We can have this narrowed down to what troubleshooting can we apply to have your concern with your computer addressed.

For starters you can perform a clean boot on your computer. This is to ensure that there are no 3rd party programs that could cause the issue. By enabling services one by one after each restarts, you'd be able to isolate the issue to see if there are programs running on the background that might cause the issue. If one of the services triggers the issue, best to have it reinstalled if you're familiar with that program and if you don't use it anymore, you can have it left uninstalled.

If after performing a clean boot and the issue is still the same, then the next step would be booting your computer to safe mode for testing. This is to check your hardware if it can run in lowered settings. If your Windows run on lower settings then this would mean that it could be one of your drivers needs reinstalling. At most cases, it is best to have them downloaded from the manufacturer's website so the drivers installed on your computer aren't the generic ones. In the event that the crash still happens when you boot to Safe Mode, this would mean that it could be possible that one of your hardware might be at risk which needs further diagnosing.

If you can boot the computer to safe mode and after reinstalling the drivers needed, then at most cases a Custom Install or a Repair Install is advised. But prior to performing this step, it is best advised to always have all your personal files backed up at all times. This goes from your documents down to the program installers that you've saved on your computer.

Lastly if all troubleshooting are exhausted, a clean install would be advised as a last resort. This is not only to start your computer fresh but also for your computer to be tested with it's integrity in holding it's system files. Basically, all these suggested troubleshooting would be on a case to case basis to ensure that all your files are kept and salvaged as much as possible.

And to answer your question in filing in a feedback, you can refer to the post made by Bill Smithers last August 7, 2016 on this thread as future reference.
