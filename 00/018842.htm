@chrispjones -

Trusted Installer is the Windows Service process used to install updates and components on the machine.  The fact that it is running, or using CPU cycles is because the system is updating. 

Sure, the Trusted Installer Service is using resources because it’s active – but lots of Windows users have a problem with chronic and excessive resource utilization by Trusted Installer, as should be apparent from the thread that I cited. In fact, the only Windows service with a worse reputation for pathological resource consumption is the Microsoft Antimalware Service:

http://answers.microsoft.com/en-us/windows/forum/windows_8-performance/can-one-prevent-antimalware-executable-to-start/14d43f26-c682-4b3d-80b6-59ae754b8649

950 people had this question for the Microsoft Antimalware Service, as compared to only 637 for the Trusted Installer Service – and look at the amazing answer they got from a Microsoft Support Engineer. Yes, there's no disputing that the Antimalware Service is using resources because it’s active (scanning for malware), but that doesn’t mean that all is well.

Try these basic troubleshooting steps first and check after each step and please update me. (Please note your data is stored on the cloud, hence you won't be losing data)

1. Reset the Skype
Windows Key+I > Apps > Locate Skype > Advanced Options > Reset

2. Uninstall and reinstall Skype app (Note - You will not lose any Skype data because of this)
Uninstall
Windows Key+X > Windows Powershell (Admin) > Copy the below, paste into Powershell > Enter

Get-AppxPackage -allusers Microsoft.SkypeApp | Remove-AppxPackage

Reinstall
Copy and paste below and Enter

Get-AppxPackage -allusers Microsoft.SkypeApp | Foreach {Add-AppxPackage -DisableDevelopmentMode -Register “$($_.InstallLocation)\AppXManifest.xml”}

Download and install Classic Skype - https://go.skype.com/classic.skype
