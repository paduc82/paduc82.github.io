Thanks for the crash dump.  Regrettably, I can't see a problem in it: it all looks good.  That doesn;t mean there is no problem, it just means that I do not know enough about the code in Word 2011 to be able to see the problem.

Don't worry about re-installing: you in fact have an UNLIMITED number of "Installs" on your licence key.  It does not care how may times you "install" the product, it counts the number of MACHINES you attempt to RUN it on.  If you blow it up 1,000 times on that machine, you can re-install 1,000 times: it still counts as "1 installation".  If you try to run on a different machine, then it will become unhappy.

I am going to take a wild guess that the issue is that the old File Type/Creator Code stamp in the old documents is causing your system to call bits of Word 2004 that are either not there, or can't run.

There's a couple of things I would try: the first is to attempt to open one of those documents with "File>Open".  Within Word 2011, drop down the File menu and choose Open, navigate to the document and try that.  This short-circuits the Launch Services Database and ensures that the file is actually passed by the Operating System directly to Word 2011.  If that opens, that's diagnostic: it's going to the wrong application.

Next, I would check to ensure that those old files have the correct extension on the file name.  I assume they are documents, so it should be .doc.  Windows no longer needs or cares about file extensions, but Mac OS X is Unix, and it does.  If the file extension is not there, Mac OS X will fall back to the old File Type and Creator Code embedded in the file.  Since the software that published those is no longer on your machine, you will get the above problem.

If that's the issue, Select one of those files in Finder, right-click, choose Get Info, and under Opens With navigate to and choose Microsoft Word 2011.  Then choose "Change All".  That should fix the Launch Services Database.  You may have to re-start the computer to make the change stick.

Basically what I have given you there is three descriptions of the same problem: the Launch services Database has an incorrect entry for the obsolete .doc format.

If it's not that, I am afraid you are going to have to assume that the issue is an incompatible software component, and do a scorched-earth remove-replace of Office.

Try to remember to re-save each file in the new format as you work on it.  The obsolete format is falling increasingly far behind in being able to store the things Word can create these days: you will lose progressively more and more formatting and data if you do not upgrade.  Each time you save into the obsolete .doc format, Word simply discards the information types the old format cannot store.  This does not make for reliable documents going forward.
