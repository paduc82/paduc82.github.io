bob jones aka: cybertaz say:
Since you haven't provided a detailed explanation of the issue perhaps some general info will be helpful...

The matter of Language is easier to deal with if you keep in mind that it is a formatting attribute of the text similar to Bold, Italic, Font Size, etc. It simply is not as apparent because it doesn't alter the appearance of the text... Language formatting is essentially invisible unless you specifically look for it.

When you go to Tools> Language to select your Default Language, that Language becomes the Language format used in any new documents you create from then on. Similarly, if you create a new document & immediately select a different Language (without clicking Default) that Language is what will apply to what you type from that point on.

However, two points that often cause confusion & perplex users are:
  • What is set as the Default Language in your installation has absolutely no influence on existing documents -- regardless of whether they came from somewhere else or you created them yourself, &
  • When working on a document there are influences which can cause a different Language to be used in certain parts of the document. E.g.;
  1. Since Language is a formatting property it is included in Styles, so applying a Style can cause the Language formatting of the target text to change accordingly
  2. When you cut/copy from another source the Language formatting of the content is carried along with the text when you do a typical paste. If you continue to append text from that point on, the Language formatting carries along as you type.

For additional information you might want to consult this web page: Spelling, Language, Dictionary. Don't hesitate to post any specific questions here.

barb way say:

Hi Scoz -

The behavior you are seeing is likely caused by the original Shape Data not having been applied using Shape Data Sets.  For instance, in the Organization Chart template, the shapes have some common Shape Data fields (Department, Telephone, Name, Title, Email).  However, these are coded into the shapes rather than applied through Shape Data Sets.  This means that when you apply a Shape Data Set and select the 'Remove existing data sets' box, the properties are not removed.

If you create two different Shape Data sets and apply one, then apply the second with the option you mention selected, it will in fact replace set one with set two, in my testing.

roady [mvp] say:

The ost-file is required when you configure your Outlook.com as an EAS account (which replaces the Outlook Hotmail Connector that was needed in Outlook 2007). The ost-file is the local cache and works basically the same way as how it was in Outlook 2007; it mimics the server folders.


The only way to get rid of the ost-file is to configure your account as a POP3 account but that will break the server-sync as a POP3 account is download of the Inbox folder only.


You can do without the pst-file though when you set the data store of the Outlook.com account as the default via:
File-> Account Settings-> Account Settings...-> tab Data Files


To migrate any data from a pst-file to the ost-file of Outlook.com see: Migrating Outlook pst-data to Outlook.com.
