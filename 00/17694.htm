diane poremsky mvp say:
To remove send on behalf permissions, you can use powershell:
This removes permissions for a specific user from a specific mailbox:
Set-Mailbox "mailbox name" -GrantSendOnBehalfTo @{remove="user"}

This removes specific users from all mailboxes:
Get-Mailbox | Set-Mailbox -GrantSendOnBehalfTo @{remove="user1","user2"}
As you can see from the examples, if you want to remove multiple users, use a comma separated list, otherwise, use a single user.

You can hide the from field in outlook using group policy - i don't recommend doing this if the user ha more than one mailbox in their profile.
I have instructions here: https://www.slipstick.com/how-to-outlook/group-...

bernd p. say: You have to create a "Batch" file, file type .bat

For instance MyBat.bat

You create it with notepad.

A .bat file contains commands, at least one.

The command for starting programs is (surprise !): start

Parameters are <program> and <file to open>, for instance:

start <program> <file to open>

In your case:

start notepad.exe may.txt

This variant only works with notepad and if may.txt is in the same folder as the .bat file.

For other programs and files you need a full path, similar to

C:/Users/<user>/Documents/may.txt

Tip: you can get the paths easily (in W7) by navigating to the path with explorer, right click the path at the top and select copy (to the clipboard).

For testing start the MyBat.bat by double click in explorer.

daniel g. grau say:
Try adding line numbers to the original document before enabling track changes. Otherwise, Word will number the deleted lines but, since they don't show in Balloon view, there will be a skip in numeration.

Why is Word numbering the deleted lines, if they don't show? Because it tries to be consistent by showing the same line numbers for all visible lines for a given view (in this case, Final Showing Markup), whether the deleted lines are visible or not. In this way, all line numbers will be the same for that view, whether you use Balloons or not.
