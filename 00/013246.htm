ossiemac say:

I am assuming that you are using ActiveX ComboBox. If not, I suggest you change to an ActiveX.

Set up the range for the combo box with the names in one column and the numbers in the adjacent column.

Not sure of your level of expertise so will provide full instructions in case required.

When creating an ActiveX control the Design Mode turns on by default when you create it. (Design Mode is controlled by the icon with the Set Square, Ruler and Pencil. It has an Orange background when on and blue background when off.)

While in Design mode, right click the control and select Properties and then set the following properties:-

BoundColumn   2

ColumnCount    2

ColumnWidths 20 pt;0 pt   (2nd column zero width so numbers do not display in ComboBox)

ListFillRange  Sheet1!A1:B4   (Include sheet name. Edit range to both columns of your list range.)

LinkedCell   Sheet1!F3     (Include sheet name. Edit F3 to required cell to display number)

Close Design Mode  (Click Design Mode icon and it should turn blue.)

By having the names in the first column this is what is displayed in the ComboBox. Setting the 2nd column to zero width hides the numbers in the second column so not displayed in the ComboBox.

By setting the bound column to 2, the value actually returned by the ComboBox is the 2nd column and therefore the linked cell displays the value from the 2nd column.

 

You can re open Design Mode and adjust properties at any time by clicking the Design Mode icon. (Clicking the icon again turns off Design Mode.)

Following added with Edit after initial post:-

The columns of the ComboBox are numbered 1 to number of columns in the Properties but if you want to address the columns to return the values of the individual columns using VBA then the columns number from zero. Therefore the following code will return the value of each column of a 2 column ComboBox.

Private Sub ComboBox1_Change()
    MsgBox ComboBox1.Column(0)  'First column
    MsgBox ComboBox1.Column(1)  'Second column
End Sub

In some instances, the native PDF tools actually work better than the Acrobat add-in.

scottgem say:

Depends on how you identify the record to print. Generally this is done by specifying a WHERE clause in the OpenReport method. Form example:

DoCmd.OpenReport "reportname",,,"[keyfield] = " & Me.[keycontrol]

where keyfield is the unique identifier for the record and keycontrol the control on the current form that holds the keyfield value for the record.
