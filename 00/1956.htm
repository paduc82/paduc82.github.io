a. user say:

I understand what you are trying to end up with, but it's not completely obvious exactly what the data source is, because you said
"the merge fields are bein pulled from a third party app, which stores the data on MS SQLServer 9.0"

So what I wanted to know was whether you were getting the data directly from SQL Server, or whether the third party app. is getting it from SQL Server and putting it somewhere else, e.g. a plain text file.

Your main options are along the following lines, none of them totally simple:
 a. merge to a new document and use Word VBA to look for the code and split it.
 b. Do one merge to get the data and put it in a place where it's easy to split the code apart (e.g. Excel), then do another merge
 c. Use VBA and Word's mailmerge events to split the code.
 d. If your mail merge is getting the data directly from a SQL Server table or view, then you may be able to use a small bit of VBA to execute a bit of SQL Server SQL (Transact-SQL) to get the data and split the field. e.g. if the field is called mycode and codes from a table called mytable, then the appropriate SQL would be something like

SELECT SUBSTRING(mycode,PATINDEX(mycode,'/')+1,LEN(mycode)) AS `mydigits`, * FROM mytable

(I can't easily pin down exactly what the SQL should be until you can confirm that it's aactually a SQL Server data source)

You might be able to do that by running the following VBA:

Sub modifyQuery()
ActiveDocument.MailMerge.DataSource.QueryString = "SELECT SUBSTRING(mycode,PATINDEX(mycode,'/')+1,LEN(mycode)) AS `mydigits`, * FROM mytable"
End Sub

However, it is possible that that will not be enough.

doug robbins - mvp office apps & services (word) say:

Go to the View tab and select Toolbars and then check the Mail Merge item.  Then, when you  have the mail merge main document set up, execute the merge to a new document using the first button in the last group on the Mail Merge toolbar.  Then with that document as the active document, click on Edit and the Replace and on the Replace tab of the Find and Replace dialog, type

 

MAIN/

 

into the Find what: control and leave the Replace with blank.  Then clcik on Replace All and the MAIN/ will be deleted from wherever it appears in the document leaving just the numeric string.

        ' Input values: the paths and file names of
        ' the source and destination files.

    On Error GoTo error_handler

    ' Compact and repair the database. Use the return value of
    ' the CompactRepair method to determine if the file was
    ' successfully compacted.
    RepairDatabase = _
        Application.CompactRepair( _
        LogFile:=True, _
        sourceFile:=strSource, _
        destinationFile:=strDestination)

    ' Reset the error trap and exit the function.
    On Error GoTo 0

You should also realize that you can only C&R a database when no one is in it.  So if you are in the FE with a connection to the BE, this code is no good.  It would need to be run before any connections are made to the BE, and only when no other users are in the system.

Access 2007 and 2010 now supports this type of control re-sizing as you change the size of the form.
