dirk goldgar say:

For this  you need the DISTINCTROW keyword -- otherwise Access doesn't trust that more than one record from COMPANY_NAME won't be created by the join, and it might not know which one to delete.  Try this:

    DELETE DISTINCTROW COMPANY_NAME.*
    FROM COMPANY_NAME INNER JOIN COMPANY_DUPS

    ON COMPANY_NAME.COMPANY_NUM = COMPANY_DUPS.COMPANY_NUM;

You don't need the WHERE clause you had, because that filtering is aalready accomplished by the JOIN.

hansv mvp say:

Try this:

 

Sub RemoveReadOnly()
    Dim wbk As Workbook
    Dim strFile As String
    strFile = Application.GetOpenFilename
    If strFile = "False" Then Exit Sub
    Set wbk = Workbooks.Open(Filename:=strFile, ReadOnly:=False, _
        IgnoreReadOnlyRecommended:=True)
    wbk.SaveAs Filename:=strFile, FileFormat:=xlExcel8, _
        ReadOnlyRecommended:=False
    wbk.Close
End Sub
