XML Schema - Microsoft Community

fishy jeff say:

XML Schema

I have several large Word documents that I wish to have symanticaly marked up correctly. Headings should be headings, not text that has been made larger.  I'd like to add an XML schema that enforces such rules. What would such a rule look like?

The end result will be a document that will be converted to a website. The H1 headings will be section index pages. The H2 headings will be subsection pages within the site. To do that, I need everything to real headings, instead of text marked up to look like headings. Preferably, I would like to lock out any font "formatting".

The sample XML schemas I got from MS's site will not even load in Word. Something about being reserved...

(Fishy) Jeff --- My Goldfish Forum name!

a. user say:

1. Word 2003 has a facility to let you define a "custom XML schema," associate that schema with a Word document, and optionally enforce the schema.

2. Unfortunately, because of a court decision in the USA in 2009 or 2010, many of the facilities associated with XML schemas in Word 2003 have been removed in versions of Word 2007 provided in the USA after a certain date. They have also been removed in Word 2010. So even if the XML Schema technology does what you want in Word 2003, be advised that it may not survive an update to Word. 

3. As far as Word 2003 is concerned, what "enforcement" really means is that Word lets you insert  XML elements into a document and type text into them. There are also facilities for specifying XML attributes. But what Word is really enforcing is the structure of the /data/ (and that's what you might expect, because XML is supposed to be about the structure of the data, not the presentation of the data).

4. What Word does not let you do in any simple way is associate a particular Word Style with a particular XML Element. So it's not completely straightforward to say something like "I want my Heading 2 paragraph style to be associated with Element <subtitle> in my schema". I'm not saying it cannot be done. But (a) if it can be done I never worked it out, (b) it's not going to last beyond Word 2003, and (c) it certainly wasn't how I imagined what XML support would consist of in Word.
