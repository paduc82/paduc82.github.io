One point to add to the other replies, it depends on your report layout. If you use a Columnar layout and have labels next to the controls, it will prevent the Can Shrink property from totally shrinking the control For example:

Now if there is no address, it will still show one line because of the label next to it.

Let us try if we could create a new instance of Word without any COMAddIns loaded. If so, your macro can use this instance to create the documents.

Andreas.


Sub Test()
  Dim fso As Object 'FileSystemObject
  Dim FName As String
  Dim D As Document
  Dim i As Long
  Dim App As Application
 
  'Create a temporary file name
  Set fso = CreateObject("Scripting.FileSystemObject")
  With fso
    FName = .BuildPath(.GetSpecialFolder(2), .GetBaseName(.GetTempName)) & ".docx"
  End With
  'Create the file in this instance
  Set D = Documents.Add
  D.SaveAs2 FName
  D.Close
 
  'Open Word again, don't load any AddIns and open our temporary file
  i = Shell("winword.exe /a " & FName, vbMaximizedFocus)
 
  'Refer to that instance
  Set App = GetObject(FName).Application
  'Debug.Print App.ActiveDocument.Name
 
  'Close and delete the temporary file
  App.ActiveDocument.Close
  Kill FName
 
  'Create a new document in that instance
  With App
    Set D = .Documents.Add
    .Selection.TypeText Text:="Just a test"
    D.Save
  End With
End Sub
