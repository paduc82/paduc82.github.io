scottgem say:

Two points. You refer to a "beautiful macro" then post VBA code. In Access they are not the same. The macro language in Access is totally different and separate from VBA coding.

The second point, is that you are searching for potentially non unique values. But your request is to "search for an employee record by either first name or last name and then immediately jump to and display the results". Since you are3 searching for potentially non-unique values your code doesn't work. You use "Me.RecordsetClone.FindFirst " So if the user types in John, it will find the first record in current sort order that matches John. But you will not know if there are 3 or 5 or 100 Johns. Similarly if you search for Smith it will return the first Smith only. 

So using FindFirst is not a good solution. I would be more inclined to use a Filter. I would base my form on a query of the Employee table. In that query I would have a column like: Employee: EmpFirst & " " & EmpLast. I would then use code like this:

Dim strCriteria as String

strCriteria = "Employee Like "*" & Me.txtSearch & "*"

lngCount as Long

Me.Filter = strCriteria

DoCmd.Runcommand acCmdApplyFilterSort

Me.RecordsetClone.MoveLast

lngLast = Me.RecordsetClone.RecordsetCount

MsgBox lngLast & " records match your critera."

So the user sees the first record that matches the criteria and is told how many records were returned.
