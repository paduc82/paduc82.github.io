lemp say:

If the installation of Windows from which you extracted the product key was pre-installed by the computer manufacturer, that key is not valid.  On such installations, the product key in the registry is the key used by the manufacturer when Windows was first installed.  When you re-install Windows on these machines, you need to use the unique product key on the COA sticker on the computer.

I suspect that this was, in fact, the case, because the Microsoft Windows 7 software download page explicitly does not work with such OEM product keys.

Do not post your product key here, but do a Google search yourself for your product key, or at least for the last 15 characters of the key.  If you get no hits, your key is probably a good key, but my guess is that you will find that you either have a generic key from one of the big OEM companies or a well-known pirated key.

If you "accept your installation as not genuine," you'll suffer the following:

Unlike Windows XP and Vista, failure to activate Windows 7 leaves you with an annoying, but somewhat usable system. According to a Microsoft developer blog post on "Microsoft Developer Network," if you don't choose to activate Windows 7 during installation, you will see an "Activate Windows Online Now" message in the system tray. If you don't activate then, you will see an "Activate Now" message every day from day four through day 27. Until day 30, you will get the "Activate Now" message every four hours until day 30. After day 30, you will get the "Activate Now" message every hour, along with a notice that your Windows version is not genuine whenever you launch the Control Panel. In addition, Windows 7 does not perform any system updates after the grace period. Finally, Windows will automatically turn your screen background image to black every hour even after you change it back to your preference. This behavior continues until you successfully activate Windows 7.

in the sticks say:

In my experience stop 7b is most often caused by improper setting for disk controller access mode in BIOS setup. If using AHCI/RAID switch to IDE/ATA. XP does not support AHCI or RAID without supplying drivers at install time via floppy (watch for hit F6 message). Chances are that your PC doesn't have a floppy drive if it originally had 8.1, so you will either have to slipsteam drivers for AHCI with a product such as nlite or switch the disk controller mode in BIOS setup.

Well you can safely ignore what Andrew E had to say. It is obvious he has replied without even trying to understand what your problem is and what you are doing that causes it.

If you can boot to recovery console with your XP installation CD and then type CHKDSK C: /F you will actually get an error message that reads

"The parameter is not valid. Try /? for help."

If you then type CHKDSK /? you will see that the only valid switches listed are /P and /R.

What you really need to type is CHKDSK /R

If your command prompt starts with C:\ (probably C:\Windows) then you do not need to specify the drive letter.

You can verify this for yourself but I can assure you that before posting this reply I tested it with my XP Home installation CD.

Also, if he (erroneously) meant that you should issue CHKDSK C: /F from a command prompt within Windows, it is obvious that he hasn't realised that your Windows XP is not even installed yet. Even if you could do this within Windows (where the /F switch is valid), once again, provided that your command prompt starts with C:\ (probably C:\Documents and Settings\<username>) then you do not need to specify the drive letter. Let us assume that he meant to use the /F switch from a booted Windows (with or without C:), he negelected to mention that you will get a message that reads

"Chkdsk cannot run because the volume is in use by another process. Would you like to schedule this volume to be checked the next time the system restarts? (Y/N)."
