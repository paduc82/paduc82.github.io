   Forms![name of form]![name of control]

scottgem say:

Your question indicates a denormalized structure where normalization would solve your problem. If you have a separate table with Customer information (which is what I think you mean by an unbound dbx), that table should have a primary key value that identifies the customer record. That value should be stored as a foreign key in your main table. 

Your form can then display the customer info in a variety of ways (see my blog on Displaying Data from Related tables). You should NOT be storing customer name or any other info about a customer in your data table other than the ID as a foreign key. That is a basic principle of relational databases. So, if any info about the customer changes in the customer table, that info is automatically reflected on the form because it gets that data from the customer table. 

If I've misunderstood your question please try and clarify.

Name of formatting symbol for shift+enter - Microsoft Community

arzi(2) say:

Name of formatting symbol for shift+enter

What is the name for this curved arrow formatting symbol that displays when Shift + Enter is used? Is it correct to refer to Shift + Enter as a soft return?
jay freedman say:
The symbol has the Unicode name "Downwards Arrow With Corner Leftwards"  <whew!> and the character code 21B5 (hex). As far as I know, nobody refers to it that way.

The function in Word represented by that symbol is the "manual line break". I think the only place in the program where you'll see that name is on the Special popup in the Find and Replace dialog.

Word has never used the "soft" and "hard" terminology, probably because WordPerfect did use it. (MS used to consider WP its mortal enemy.)
