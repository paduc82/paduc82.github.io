fizzyvassal say: First the solution(s):

I am assuming you want to keep your files in the same format and use the native Windows 8 Video  app, which was why I suggested the codec download. However, I have no idea what codecs you would need - your movie files could be anything (.mov doesn't tell us much).

Another option is to download QuickTime, which will naturally include the codecs needed to play quicktime files. Then, if you wanted, you could use that to play these files. This would most likely allow you to play quicktime files in the native Windows 8 Video app (I'm new to Windows 8, I don't know how it works. It would at least work in Media Player).

The last option would be to convert your files to something else, preferably something standardised like mpeg4 (mp4 wrapper). Beware if these are work-related files, I have no idea what the software you're required to use at work is or can do.

Then my addition to the thread:

Moving Picture Experts Group (have a close look at the acronym of that) are the the ISO/IEC group tasked with video standards.

Some software will run some quicktime videos out of the box, because it is rather a simple matter to add support for the wrapper. Along those lines, I have a clean install of Windows 8 and I grabbed Apple's sample quicktime files to see which of them it can play - it natively plays quicktime files (.mov) that are a wrap of the mpeg-4 standard. It doesn't play quicktime files (.mov) that wrap the .flv format (flash video). You see, not all movie files are equal: you can't really tell what will play and what won't simply by looking at a file suffix. In that regard, the only way you are guaranteed to play all quicktime files is to use QuickTime.

Regarding most used format: I believe it is MPEG-4 (many .mov files are mpeg-4). However, as a programmer who works across platforms and industries, I'd strongly advise to use the .mp4 wrapper instead of the .mov one. I can't remember the last time I worked with a .mov file, in all honesty. Saying that, my girlfriend works for a big media corporation, so I'm not blind to the fact that some industries believe they are completely invested in .mov 'files'. It's simply not true.

doug robbins - mvp office apps & services (word) say:

Use the Outlook .SendUsingAccount procedure.

Here is what the Visual Basic Help provides on its use

Returns or sets an Account object that represents the account under which the MailItem is to be sent. Read/write.
Version Information
Version Added:  Outlook 2007

Syntax

expression.SendUsingAccount

expression   An expression that returns a MailItem object.

Remarks


The SendUsingAccount property can be used to specify the account that should be used to send the MailItem when the Send method is called. This property returns Null (Nothing in Visual Basic) if the account specified for the MailItem no longer exists.


Example


The following code sample in Microsoft Visual Basic for Applications enumerates the Accounts collection to find a Pop3 account. If the account is found, then a message is created programmatically and the SendUsingAccount property is assigned to the Pop3 account. Note that you must assign the SendUsingAccount property before you call the Send method.

Visual Basic for Applications
Sub SendUsingAccount()
    Dim oAccount As Outlook.account
    For Each oAccount In Application.Session.Accounts
        If oAccount.AccountType = olPop3 Then
            Dim oMail As Outlook.MailItem
            Set oMail = Application.CreateItem(olMailItem)
            oMail.Subject = "Sent using POP3 Account"
            oMail.Recipients.Add ("*** Email address is removed for privacy ***")
            oMail.Recipients.ResolveAll
            oMail.SendUsingAccount = oAccount
            oMail.Send
        End If
    Next
End Sub

Based on that information, I believe that you would need to make the following modification to the code inside the With oItem - End With construction by using the following in place of the .Send command

        Dim oaccount As Account
        For Each oaccount In .Session.Accounts
            If oaccount.AccountType = olPop3 Then
                .SendUsingAccount oaccount
                .Send
            End If
        Next

rohn007 say:

In Windows the printer settings are controlled by Windows, not Word. So after you change the print option to 2 sided it stays that way because Word is not controlling it.

You have a couple of options.

If you have a HP printer, or one that uses HP PCL, HP Printer Control Language (rather than PostScript) you can embed a PCL command to force single or duplex printing

Duplexing Documents, by Default (Embedded PCL Code)

http://word.tips.net/T006904_Duplexing_Documents_by_Default.html

Summary: If you have a printer that will print on both sides of a piece of paper, you may want to use that ability within Word. Unfortunately it's not as straightforward as you might hope. (This tip works with Word 97 | 2000 | 2002 | 2003 | 2007.)

PCL Escape Codes to Use with the PRINT Field

http://support.microsoft.com/?kbid=135569

This article was previously published under Q135569

This article lists some of the escape codes that can be used in the PRINT field for printing documents to Hewlett-Packard (HP) or HP-compatible printers. For example, the construction {PRINT 27"&l2S"} at the top of a page causes a duplexing printer to begin printing on both sides of the page.

You probably can do the same with PostScript printers, I simply have not found the appropriate command.

The other option is to create 2 separate "Printers" in the Windows Control Panel > Devices and Printers dialog. You simply install the same printer driver a second time, with a different name.  Then you can set one copy of the driver for single sided and the second one as duplex. Use the "printer" names to identify the duplex version.  Then you just have to make sure you have the right printer selected.  If you make the single sided one "default" it will appear every time you restart Windows.  If during a Windows session you select the duplex version it will remain selected until you select the single sided printer.
