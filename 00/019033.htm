karl timmermans [outlook mvp 2012-2018] say:

- Is it possible to change the comma setting in Outlook to a semi-colon for example?

- If not possible, is it possible to "mark" the original commas in the .csv file so that they are imported into Outlook effectively as commas and not separators?

Let's start with a few basic items about .csv files

#1 - The field delimiter expected by Outlook is a 'comma" regardless of region

#2 - Any field containing a field delim character or a line break chars will be surrounded by the <Field Qualifier> of double-quotes and anything inside that field will be processed as one field treating the comma correctly

#3 - From your description - any CSV file created by Excel using your regional settings will use the semi-colon as the field delim character (which will not be recognized by Outlook)

#4 - any <Field Qual> characters that also exist inside of a single field must be "escaped" meaning that every double-quote must be preceeded by a double-quote 

#5 - IMHO - highly ill-advised to do "character" replacements of any kind in a text file to change <Field Delim> or <Field Qual> characters - too many ways for things to get messed up unless this involves the most simplest of text files

So the simplest and quickest solution for you is 

- in the Control Panel regional settings - change your region to United States (or any country where the comma is used as the <Field Delim> character (French uses semi-colon)

- save your Excel file as a CSV file which can now be imported in Outlook

- change your regional settings back to their original state

fiftysevenchevy say:

how do you clear IE pass words from sys w/o hurting anything

i tried to get help but the security essentials said no. how do you clear pass words in auto complete for win exp? cause are wrong some are good. just want to clear and start over a bit
kosh vorlon say:

Hi 57Chevy,

 

MSE has no feature to assist in clearing this type of information.  It is to detect, scan, help block and/or remove malware, viruses and spyware.  It has no feature for password removal (or really for doing much of anything else either).

 

What you want must be done in IE.  Open IE and go to Tools / Internet Options / Content / Auto-Complete / Settings.  Click on Delete Auto-Complete History to open the box.  You almost certainly want to leave the top box checked to save Favorites.  The rest are fair game.  To specifically clear Passwords, check the box next to passwords.  Bear in mind you can't pick and choose - it's an all-or-nothing thing.  Click Delete and then click OK as you back out to the main menu and that should be that.  I do not believe this is required, but if you want to be certain it is cleared from that moment forward, close all open IE windows and tabs and re-open IE to make sure the change "took."  That's it - you're done (and I hope you meant it when you did it, because I'm unaware of any way to recover them once deleted short of restoring an image backup - which as you know will not only help only back to the last backup, but you'll lose everything else since that backup as well - because all the methods I'm aware of are for file recovery and not this type of thing - and I'm not even sure if a data recovery specialist can do if but if possible, it will be very expensive and will typically cost the same whether all are recovered or none are recovered as they generally charge for the service and don't guarantee results).

 

Now it gets a little bit tricky.  It begins to need to know exactly what you mean by "Passwords."  While I believe the above should do the trick for what I think you want, some information can be saved on the Forms themselves (though usually not Passwords which are generally separately taken care of by the Passwords box above).  Also, cookies can save some site visit information (but again, usually not if ever Passwords).  I'd start by just clearing Passwords and see if that does the trick.  If not, proceed to the other two until what you want removed has been removed.

 

If you want to start over, then that should do it.  If you no longer want to save passwords, then in the Settings section, be sure to uncheck the Passwords box to stop it from continuing to store that information.  If to clear everything you wanted above you needed to go beyond Passwords, then uncheck saving those items as well if you want to stop saving them also.  And that's all there is to that.

 

I hope this helps. If I misunderstood what you meant by passwords (though use of the term auto-complete did pretty much clarify it for me as long as you are clear on the meaning of that term as well), then feel free to post back and we'll address either what you meant or what else you expected to be included that wasn't.

 

Good luck!

jay freedman say:

The list of "recent" documents is stored in the Windows registry in the key named HKEY_CURRENT_USER\Software\Microsoft\Office\16.0\Word\User MRU\LiveId_###\File MRU (where ### represents a long hexadecimal number that identifies the user profile).

It's possible that some other program or an erroneous set of permissions started in November to interfere with Word's ability to write entries in that key. I haven't seen any other complaints of this kind, but maybe it's happened to people who haven't noticed it.

Things you can try include making sure that Office is fully updated, and running a quick repair of Office. If those don't help, use the troubleshooting steps in http://support.microsoft.com/kb/921541 -- but leave the registry changes (steps 3 and 4) until after you've tried all the other steps without success.
