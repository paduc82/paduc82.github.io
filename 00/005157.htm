Hi,

 

The problem is that your using TRUE as the argument in VLOOKUP which means if it can't find an exact match it will return the nearest match.

 

To see this in action, consider this. Strip away your if statement and your formula becomes

 

=VLOOKUP(B16,Surnames1, 1,TRUE)

 

Now say there is onlt 1 name in the lookup range (Mike) and B16 cantains the name CFOB. The formula returns MIKE because that's the closest match to CFOB. You should be using FALSE instead which forces an exact match.

 

Having said that I'd do it like this

 

=IF(ISNUMBER(MATCH(B16,Surnames1,0)),TRUE,FALSE)

The "minor bug" seems to be in your usage of VLOOKUP.  If the first column of the range defined by Surnames1 is unsorted, you should use FALSE, not TRUE, in VLOOKUP's 4th parameter.

When you use TRUE with an unsorted range, VLOOKUP's behavior is unpredictable.  It might work in some cases; it might return a #N/A error in some cases; and in other cases, it might return the wrong result.

That is not a defect in VLOOKUP.  It is a consequence of VLOOKUP's algorithm when TRUE is used (i.e. binary search), based on the assumption/requirement that the first column of the lookup range is sorted.
