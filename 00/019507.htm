jay freedman say:

The Bibliography field for IEEE format is programmed to create a table, with each citation in a separate row. The left column of the table should be wide enough to hold a two-digit number and its brackets. The only time I see that the numbers don't fit is if the font size of the Bibliography style is increased.

You can use the controls on the Table Tools ribbon to make the column fit the numbers: Click on one of the broken numbers. The Table Tools ribbon should appear. Click the Layout sub-tab of the ribbon. In the Cell Size group, increase the measurement in the cell width box:

until the numbers fit on one line.

If you have to update the bibliography later, the column width will return to its original size, and you'll have to increase it again.

rohn007 say:
Yes you can tell word to look for whole words
<snip>
<> - Whole Word

You can use both the < and > characters together in a single search specification, which allows you to search for entire words.  If you wanted all the four-character words beginning with t and ending with d, you could search for <[Tt]??[Dd]>.  If you wanted any dates between 1800 and 1999, you could use <1[89][0-9]{2}>.

If you actually want to search for < or > in your document, you should precede the character with the backslash.  Thus, to search for < you would use \< in your search specification.

Remember that to take advantage of the pattern matching capabilities of Word, you just need to make sure the Use Wildcards check box is selected on either the Find or Replace tabs of the Find and Replace dialog box.

jay freedman say:
The native Find in Office programs, with or without wildcards, doesn't support disjunction of the kind you're asking about. To put it succinctly, Find is not regex.

However, real regex -- at least, Microsoft's implementation of it -- is possible in a macro by including a reference to the "Microsoft VBScript Regular Expressions" library. Some version of it (I think 5.5 is current) is part of each Windows version.

Here are a couple of articles about it (the use of regex in Excel is the same as its use in Word, but the surrounding code would be different):

http://dotnetslackers.com/Community/blogs/dsmyth/archive/2006/09/09/Regular-expressions-with-Word.aspx

http://www.macrostash.com/2011/10/08/simple-regular-expression-tutorial-for-excel-vba/

Building on those examples, you could write a macro that lets you enter a regex expression into an InputBox, assigns that expression to regEx.Pattern, and selects or highlights each match.
