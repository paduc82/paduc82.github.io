It appears you have a pretty good start on your design, it just needs a little refinement.

Starting at the bottom, no you do not need or WANT a separate DB for each audit.  You MAY want separate Tables for the audits if they have different structures.

A table contains (non repeating) information about a specific 'thing': a customer, an address, an order.

Any time you have repeating information, like your attach1 / attachment(s) (especially a variable number of repeats) you should consider adding a separate, "child", table.  You will need to create a Key, a unique "ID, for the table. The unique "child" ID will be saved in the "parent" table (called a "foreign key) so that you can access only the data in the child table row(s) that is/are associated with the parent row.

Each audit row in the audit type table would identify the specific attachment(s) required.

rich michaels - mvp (greatcirclelearning.com) say:
Hi Thomas,

It's probably because the edge of the drawing canvas is off of the page. Are you positioning the text box very close to the edge of the page?

In VBA if you do not specify adding a drawing canvas, word automatically inserts one when your VBA adds a text box. 

In addition, if you do not specify the 5th parameter, which is the anchor paragraph then your inserted text box, and its wrapping canvas, is positioned in points relative to the top, left corner of the page. In this scenario, you could end up with a canvas area that overflows the page and thus you get a negative value.

Hope this helps

rex zhang say:

Hi,

 

 

Based on my understanding, you can set the default value in the table or in the subform. writing a sql statement and execute it. For example, assuming you have two table named table1 (StockID, StockConsultPrice fields) and table2 (ID, StockID, RRP), if you want to set the default value of field [StockConsultPrice] to [RRP], you need to join these two tables using the same commom [StockID] field.

 

For example:

 

UPDATE Table1, Table2 SET [Table1].[StockConsultPrice] = [Table2].[RRP]

WHERE (([Table1].[StockID]=[Table2].[StockID]));

 

For more about update one table based on another table, you can refer to the following article.

http://office.microsoft.com/en-ca/access-help/update-one-table-based-on-another-table-mdb-HP005188710.aspx

How to create a new rule to Outlook 2016 users to defer send email by - Microsoft Community

kelvinaw say:

How to create a new rule to Outlook 2016 users to defer send email by 5 min?

I need to implement a rule that will apply to all emails users to defer all send email by 5 min. Without going thru one by one, is there any better way to do it on Office365 on Outlook 2016 on Windows 7 x64 machine.

I had tried to search many websites about GPO, registry, files etc. No luck finding any way to do it with GPO, registry or simply replace files. These server-sides rules seems written embedded to exchange user profiles.

I come across this sites.

https://blogs.technet.microsoft.com/heyscriptingguy/2009/12/16/hey-scripting-guy-can-i-use-windows-powershell-to-create-a-new-outlook-inbox-rule/ 

https://msdn.microsoft.com/en-us/library/office/ff868996.aspx

I modify to my needs, but it's only add a new rules with all emails send without rule action. Please help if anyone had successfully done it, maybe other way.

#Requires -version 2.0
Add-Type -AssemblyName microsoft.office.interop.outlook
$olFolders = “Microsoft.Office.Interop.Outlook.OlDefaultFolders” -as [type]
$olRuleType = “Microsoft.Office.Interop.Outlook.OlRuleType” -as [type]
$outlook = New-Object -ComObject outlook.application
$namespace  = $Outlook.GetNameSpace(“mapi”)
$inbox = $namespace.getDefaultFolder($olFolders::olFolderInbox)
$rules = $outlook.session.DefaultStore.GetRules()
$rule = $rules.Create(“TestRule”,$olRuleType::OlRuleSend)
$MoveRuleAction = $rule.actions.Defer
$MoveRuleAction.Enabled = $true
$rules.Save()

I tested again with other action like NotifyDelivery, it's work for me.
