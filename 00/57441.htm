a. user say:

The Volume Shadow Copy Service should not be set to Automatic start.

According to the Microsoft TechNet site VSS was introduced with Windows XP SP1 and since that time it has always been installed as Manual start since there is no need for it to be starting automatically and running all day and night with nothing to do.  Some places say different times when VSS was introduced but be sure it was a long time ago.

A popular site for seeing how Windows Services should/could be set up is here:

http://www.blackviper.com/service-configurations/

If you read about VSS on Microsoft TechNet it says:

https://technet.microsoft.com/en-us/library/hh125927(v=ws.10).aspx

So do you think that the MS TechNet site is wrong and that every installation of Windows since the days of XP has has the VSS Service installed incorrectly?  If that were the case every Windows system in the world would be having problems.

On an unafflicted system if you open the Service applet and look at the VSS Service it should look like this:

Then if you start a backup operation or even create a Restore Point, come back to the Service applet and press F5 to refresh you will see that the VSS Service has changed Status to Started:

When the operation is finished the VSS Service will stop by itself (it could take a while).

That is how it is supposed to work.

One should certainly be able to set the VSS Service to Automatic and you should be able to start it but you shouldn't need to.  Folks that say that "solved their problem" forget to mention the days/weeks of trying things that lead up to getting their system working.  There is no way to tell which of the dozens of things they might have tried really fixed the issue,

You say that System Restore worked in Safe Mode (but not in a normal boot) and if that is the case that usually means something is loading in a normal boot that is causing a problem and that something is usually an antivirus/antimalware application that is "protecting" your system from changes.  When you boot in Safe Mode those things are usually not loaded so that is why you sometimes have to do your System Restores in Safe Mode.

Known contributors to such issues are Norton, McAfee, Avast, Avira, Kaspersky and probably others.

So what antivirus/antimalware programs are you running and have you checked their support pages for known issues with System Restore/Backup Restore? 

Norton even has a page just about those problems:

https://support.norton.com/sp/en/us/home/current/solutions/v51118464_EndUserProfile_en_us

McAfee even says to completely uninstall their product before doing a System Restore:

"Intel Security recommends that you remove McAfee security products before proceeding with an operating system upgrade or using Microsoft System Restore, and then re-install your McAfee product when the upgrade/restore is complete."

So I would suspect that whatever your AV program happens to be could be part of the issue here - but we don't know what that is.

We know error 5 is a permissions/access denied kind of error but you shouldn't have to make adjustments to get VSS to start so something else is wrong.  If adjustments like what you tried were needed then every Windows system in the world would also need to make those adjustments and that just isn't happening.

Of course now your system could be further afflicted since you have been changing permissions on things trying to get VSS to start when there is probably not a problem with VSS in the first place so who knows what condition things are in now.

You might start by uninstalling whatever your AV software is (most manufacturers have a special program to do that) and eliminating third party applications that are known to interfere.  You can always reinstall them again later, right?

Restore Factory Defaults - Microsoft Community

xb1r hydros say:

Restore Factory Defaults

Login Information

  • Gateway: 192.168.0.1
  • Username: admin
  • Password: (located on the bottom of the device)

 

Restore Factory Defaults

From Hardware:

  1. Locate the back of the device, where the Ethernet port are located.
  2. Hold the reset button on the back for 30 seconds. You will need a paperclip, toothpick or pen to hold down the reset button.
  3. Let go of the reset button and wait about 1 minute for the device to reboot.

 

From Firmware:

  1. Connect a computer to the gateway with an Ethernet cable.
  2. Open a web browser on a computer.
  3. Type the gateway address: 192.168.0.1 into the web address bar, select apply.
  4. This will take you to a login page, type admin for the username and password (located on the bottom of the device).
  5. Select Utilities on the main page.
  6. On the left hand side select Restore Defaults.
  7. Next to “Restore Modem to Factory Default State” select the Restore button.
  8. Wait for the device to reboot, this may take a few minutes.
