john - project say:

Susanne,

The correct command is a little hidden because it is a holdover from early versions of Project wherein a master file was called a consolidated project file. The command you want is:

ConsolidateProjects Method

I assume you are aware of the potential for file corruption with a dynamic master file. If not, here are the best practice rules. Never rename, move, overwrite, or save off any of the files in the structure. All files should ideally reside in a single folder on a local drive and not be operated over a network.

A much safer approach is to create a static master (i.e. AttachToSource option = False). That will create a single file that is a snapshot in time of all the inserted subprojects. It needs to be re-created each time you want to update, but it is not prone to corruption like a dynamic master.

Hope this helps.

If you do not have   Internet Explorer  listed in  Windows Features  that suggests that your account has been denied permission to see that checkbox.   So use a different account which does have permission to see it and then check it.

You do have a level of browsing functionality in your OS which is equivalent to having either IE8 or IE9 enabled as your default browser.   To see this you could use (assuming you at least have permission to do this)  msinfo32.exe  and get into  Loaded Modules.   Sort that list by Version descending and you will see some IE related modules near the top.   The version given there will be the version of  IE  that you could have if you could check the box that Don told you about.
