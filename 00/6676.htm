Setting Default App(lication)s is a Pain

The way Windows 10 handles the setting/changing of default apps and applications is a bit irritating.  If I'm in an application and it tells me it's not the default and asks if I want it to be, Windows should be smart enough to do just like it always did in previous versions and set the default.  Instead, it directs me to go to Settings | System | Default Apps and set it for myself.  OK.  Can do for simple programs like my browser.

Unfortunately, Build 10240 has expanded the reach of that "Default Apps" re-direction.  I just fired up OneNote (desktop version from Office 365) for the first time and got the standard "not-the-default" bit.  Now, I've complained before about Windows dumping its app on me as the default when it knows darn well I've got Office 365 loaded, used to have that OneNote version as the default, and should probably default to the desktop version over the app anyway.  But, that's neither here nor there.  It used to be that if I told it make it the default, it worked.  Now, it doesn't.  So, off I go to "Default Apps."  Unfortunately, the list of default apps that first shows up doesn't include OneNote.  So, I have to choose between three options at the bottom of the page:

- Choose default apps by file type

- Choose default apps by protocol

- Set defaults by app.

Does Microsoft honestly believe Grandma Q. Public or Joe Bag O'Donuts is going to understand that?  No, they won't.  But, regardless of granny and Joe, choosing by file type and protocol gives a huge list that can't be sorted or filtered by the programs in use.  Instead, we have to scroll down the list looking for whatever we think might be what we want (in this case, variations of OneNote).  Setting defaults by app works a bit better since it's a smaller list.  But, for OneNote, I'm presented with the two options (app or desktop) but have no idea if I'm supposed to choose "all" or "some" of the things the program can handle.  So, I choose all.

But, this is too cumbersome and needs to change.  If a program offers to be the default, Microsoft should honor the choice right there and set the setting to match.  I suppose it's possible that this could be a legacy issue (old programs don't know how to set the default in the new OS).  But, until this build, OneNote's default could be set right by the program.  Microsoft removed functionality to handle the default setting for this application in Build 10240.

vanderghast say:

That is called a dense rank. I you use MS SQL server, see that keyword. For Jet, have a query which remove the duplicated marks:

 

SELECT  1 As swk, school, class, sec, marks FROM school_marks
UNION
SELECT 2, school, class, null, marks FROM school_marks
UNION
SELECT 3, school, null, null, marks FROM school_marks

 

(saved as, say, q1)  then use

 

SELECT School_Marks.ID, 
      School_Marks.NAME, 
      School_Marks.SCHOOL,
      School_Marks.CLASS,
       School_Marks.SEC,
       School_Marks.MARKS,
      (SELECT Count(*) FROM q1 
       WHERE swk=1
            AND q1.SCHOOL = School_Marks.SCHOOL
            AND q1.CLASS = School_Marks.CLASS
            AND q1.SEC = School_Marks.SEC
            AND q1].MARKS >= School_Marks.MARKS) AS Rank1,

      (SELECT Count(*) FROM q1
        WHERE swk= 2
            AND q1.SCHOOL = School_Marks.SCHOOL
            AND q1.CLASS = School_Marks.CLASS
            AND q1.MARKS >= School_Marks.MARKS) AS Rank2,

       (SELECT Count(*) FROM q1
       WHERE  swk=3
            AND  q1.CLASS = School_Marks.CLASS
            AND  q1.MARKS >= School_Marks.MARKS) AS Rank3


FROM School_Marks;

 

 

That is probably much less efficient than using a join, but a join would produce a normalized data representation, while here, you have a kind of crosstab  already applied (with swk the implicit pivot).
