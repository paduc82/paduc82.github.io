How do I use different watermarks in the same document (adding - Microsoft Community

meganollivier say:

How do I use different watermarks in the same document (adding section breaks does not work)

I am trying to use one watermark on the first page of a document, and a different one on the rest.  The Help instructions do not work.

 

Alternatively, I suppose I could use a cover page, but I cannot figure out how to insert a blank cover page I can then adjust.

Thanks!

jay freedman say:
The nasty bit (shall we call it a bug?) is that no matter where in a document you insert a second watermark, it deletes the first watermark. You have to know how to outsmart it.

The first thing you need to know is that when you insert a text watermark from the Watermark button, all it's doing is creating a WordArt object with a gray color and 50% transparency, and inserting it as a "behind text" picture with its anchor in the page header pane. It isn't specifically a "watermark" object, just a picture/WordArt object. If you wanted to spend the time, you could create one without using the Watermark button at all.

So...
  1. Remove any section breaks -- you don't need them -- but open the header and check the box on the Header & Footer ribbon for "Different first page".
  2. In the first-page header, insert the watermark you want for the first page; change its color and position if necessary.
  3. Click on the watermark and copy it to the clipboard -- it's going to disappear, and you need to be able to paste it back in later.
  4. Go to the second-page header pane (easiest is to click the Next Section button on the Header & Footer ribbon) and insert the watermark you want on the second and following pages; change its color and position if necessary.
  5. Put the cursor back in the first-page header pane (click the Previous Section button) and paste the first watermark from the clipboard. (It'll go into its original position in the middle of the page, wherever it was when you copied it.)
Of course, none of this should be necessary, but there you go.

hansv mvp say:

Here is an example that will loop through all cells that contain a constant value (i.e. not a formula):

Sub Test()
    Dim cel As Range
    For Each cel In Range(Cells(2, 5), Cells(150, 600)).SpecialCells(xlCellTypeConstants)
        ' Do something with cel here, for example
        Debug.Print cel.Address
    Next cel
End Sub

Note that the code does not select cells. Instead of referring to ActiveCell, refer to the variable cel.

Here is a version that takes a different approach:

Sub Test()
    Dim cel As Range
    Dim strAddress As String
    With Range(Cells(2, 4), Cells(150, 600))
        Set cel = .Find(What:="*", After:=Cells(150, 600), SearchOrder:=xlByRows, SearchDirection:=xlNext)
        If Not cel Is Nothing Then
            strAddress = cel.Address
            Do
                ' Do something with cel, e.g.
                Debug.Print cel.Address
                Set cel = .FindNext(After:=cel)
                If cel Is Nothing Then Exit Do
            Loop Until cel.Address = strAddress
        End If
    End With
End Sub

On Error Resume Next
Set FormulaCells = Rows(2).SpecialCells(xlFormulas, xlNumbers + xlTextValues)
Set ConstantCells = Rows(2).SpecialCells(xlConstants, xlNumbers + xlTextValues)
If Not FormulaCells Is Nothing Then Set Rng = FormulaCells
    If Not ConstantCells Is Nothing Then
         If Not Rng Is Nothing Then
             Set Rng = Union(Rng, ConstantCells)

        Set Rng = ConstantCells
    End If
End If
On Error GoTo 0

Now, in your code, you can process Rng.

scottgem say:

Just one point to add. You referred to a macro. In Access there are two programming languages that can be used to automate tasks. One is the macro language, the other is VBA. They are not the same. The macro language is used through the macro editor. It uses a form like interface that allows you to select Actions and the parameters for them.  Macros are much more limited than VBA. VBA is more like traditional programing where you write code instructions. But it is an extremely powerful language.

The suggestions Tom and Scott M made involve using VBA code. SendObject is a VBA method. Office Automation involves VBA code that is used to control external VBA compliant applications. 

There is a macro Action equivalent to SendObject. but it has a different name. You need to set the macro editor to show all Actions to find it.

But I suggest you get yourself familiar with VBA. As I mentioned it is much more powerful.
