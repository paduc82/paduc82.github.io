' This function checks to see if a file is open or not. If the file is
' already open, it returns True. If the file is not open, it returns
' False. Otherwise, a run-time error occurs because there is
' some other problem accessing the file.

Function IsFileOpen(filename As String)
    Dim filenum As Integer, errnum As Integer

    On Error Resume Next   ' Turn error checking off.
    filenum = FreeFile()   ' Get a free file number.
    ' Attempt to open the file and lock it.
    Open filename For Input Lock Read As #filenum
    Close filenum          ' Close the file.
    errnum = Err           ' Save the error number that occurred.
    On Error GoTo 0        ' Turn error checking back on.

    ' Check to see which error occurred.
    Select Case errnum

        ' No error occurred.
        ' File is NOT already open by another user.
        Case 0
         IsFileOpen = False

        ' Error number for "Permission Denied."
        ' File is already opened by another user.
        Case 70
            IsFileOpen = True

        ' Another error occurred.
        Case Else
            Error errnum
    End Select

End Function

brian tillman [outlook mvp 2007-2019] say:
If you were to locate a PST in the OneDrive folder on your hard drive - the folder that syncs to the cloud OneDrive, I'm of the opinion the PST would work just fine, since it is still on your hard drive, but I don't believe it would ever sync because Outlook would have it open all the time. Also, if you had two Outlook instances, each referencing a PST in their own OneDrive folder, both of which are connected to the same OneDrive cloud, if they ever did sync (because you closed one Outlook), you run a very real possibility of a race condition. The PC where Outlook was closed would sync the PST to the cloud while the other PC still had the PST open in Outlook. Then, when the second PST closed Outlook, the PST changed it had made could overwrite the changes in the PST that the first PC had made. I'm not sure OneDrive is robust enough to handle this situation and I value my email, calendar, and contacts information way too much to risk this type of thing.

As to backing up a PST to OneDrive, I really believe that's robust enough not to corrupt the PST as it travels up or down, but I'd have my PST in its usual folder, then copy the PST to the OneDrive folder only while Outlook was closed. So, the backup would be on OneDrive, but the active PST would be exclusive to Outlook. You can automate such a copy-as-backup process easily with Task Scheduler.

karl timmermans [outlook mvp 2012-2018] say:
Just to add to Brian's response - will summarize the use of PST's and Onedrive as follows:

#1 - PST files on OD should never be "connected" within an Outlook profile. Data corruption may occur

#2 - Simply copying a PST file to OneDrive should be no issue - any more then it would by copying to any other similar kind of cloud service. The caveat being that Outlook should be closed before the copy process starts

#3 - Initiating an "export to PST" on OD should equally not be a problem since the exported PST file does not automatically become part of the Outlook profile.

#4 - If this involves a POP3 account, there's nothing to export. Just close Outlook and <copy> the desired PST files

mike h.. say:

Hi,

See the snip below. The ARRAY formula in A3 is this. see below for how to enter an ARRAY formula.

=SUM(--LEFT(A1:D1,FIND(" ",A1:D1)))

This is an array formula which must be entered by pressing CTRL+Shift+Enter
and not just Enter. If you do it correctly then Excel will put curly brackets
around the formula {}. You can't type these yourself. If you edit the formula
you must enter it again with CTRL+Shift+Enter.

amit tandon say:

Enter below formula as an array formula  (CTRL+SHIFT+ENTER) in cell B2:

=SUM(--TRIM(IFERROR(LEFT(A2:A10,FIND("d",A2:A10)-1),0)))

Presumed that the alphanumerics in each cell of column A start with a number and are followed by the text "day"/"days" with or without a preceding space (ie. there could be a space or not between the number and text). Note: the formula is to be entered as an array formula by pressing the 3 keys simultaneously (Ctrl, Shift & Enter). Please update the range as required.
