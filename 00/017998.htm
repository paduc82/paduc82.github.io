a_user_here say:

Here is my opinion of these things.

1) After RAM the most important upgrade you can do is put in an SSD for at least your system files.  A SSD in an old laptop can make it feel like a new one.  It is especially true of laptops since they have such slow hard drives.

2) If you have an SSD readyboost is not only pointless, it might even slow down your computer.  If the readyboost is on an USB drive the SSD will be at least as fast as it.  If you try putting it on the SSD (not even sure you can), you are basically not changing anything since clearly reading from an SSD is equal to reading from an SSD, unlike what readyboost was for USB drive being faster than a hard drive.

3) RAM usage.  If your intent is performance, then I would say 90% of the time the operating system will make better use of the RAM then you will.  If your intent is security by clearing the temp files each time you shutdown, that is a different story.  The only time I can think that the user might possibly do better than the operating system would something like a heavily used database, where the database can fit completely in a RAM disk.  And even in this case I would be have to measure the performance to be sure, and I would much more likely push for a SSD for it instead.

Here is my reasons that I think the operating system will do better than you with a RAM disk.

1) In comparison to Unix/Linux, Windows paging system stinks, and everyone knows it.  The result is that everyone is using so much RAM, that Windows seldom actually has to page.  And as far as I'm concerned, your primary use for RAM should be avoiding paging.  You do that by simply having enough memory for all the tasks you need to run at a time.

2) If you dedicate RAM to a RAM disk, that is all it can do.  If you leave it up to Windows it will use what it needs for programs, and what is left over it will use for disk caching.  It does this disk caching based on what you have accessed, and as such you can switch from one task that needs to speed up certain disk access to another without having to say move things in and out of a RAM disk.

3) Microsoft is constantly changing the way Windows uses the memory it has and you might just be "pushing against" this.  You can see these efforts in things like prefetch, and recently the memory allocation change, that when pressed for memory it will start to compress the memory and store it in the System process instead of paging.
