ken sheridan say: The method described by Karl is the most efficient, and you should have no problem implementing it with only five controls as parameters, provide that the query is saved in SQL view, never in design view, which will move things around, and at best obscure the logic, at worst make the query too complex to open.

The following is from one of my demo files, and references three combo boxes in which a county, district and parish can be selected.  The form's RecordSource is the following query, which references the controls as parameters:

SELECT Counties.County, Districts.District, Parishes.Parish,
Locations.Location, Parishes.ParishID  
FROM ((Counties INNER JOIN Districts ON Counties.CountyID = Districts.CountyID)  
INNER JOIN Parishes ON Districts.DistrictID = Parishes.DistrictID)  
INNER JOIN Locations ON Parishes.ParishID = Locations.ParishID  
WHERE (Counties.CountyID=[Forms]![frmDrillDown_Simple]![cboGotoCounty]
  OR [Forms]![frmDrillDown_Simple]![cboGotoCounty] IS NULL)
AND (Districts.DistrictID=[Forms]![frmDrillDown_Simple]![cboGotoDistrict]
  OR [Forms]![frmDrillDown_Simple]![cboGotoDistrict] IS NULL)
AND (Parishes.ParishID=[Forms]![frmDrillDown_Simple]![cboGotoParish]
  OR [Forms]![frmDrillDown_Simple]![cboGotoParish] IS NULL)
ORDER BY County,District,Parish;

In each combo box's AfterUpdate event procedure the combo boxes below it in the hierarchy are requeried  The way it works is that each parameter is referenced as follows:

(SomeColumn = SomeParameter
  OR SomeParameter IS NULL)

If a value is selected in the parameter combo box rows will be returned only where the value at the SomeColumn column position equals the value selected in the combo box.  If the combo box is left empty, NULL, on the other hand all rows will be returned because the expression SomeParameter IS NULL will be TRUE regardless of the value at the column position.

Expressions like the above are then tacked together in AND operations:

(SomeColumn = SomeParameter
  OR SomeParameter IS NULL)
AND (SomeOtherColumn = SomeOtherParameter
  OR SomeOtherParameter IS NULL)
AND (YetAnotherColumn = YetAnotherParameter
  OR YetAnotherParameter IS NULL)

This can be continued for more parameter controls in the form of course.  Each parenthesized OR operation will, by virtue of its inclusion in parentheses, evaluate to TRUE or FALSE independently.  By virtue of the AND operations rows will be returned only where all of the parenthesized OR operations evaluates to TRUE.

You'll find the demo from which the above query is taken as ComboDemo.zip in my public databases folder at:

https://onedrive.live.com/?cid=44CC60D7FEA42912&id=44CC60D7FEA42912!169

Note that if you are using an earlier version of Access you might find that the colour of some form objects such as buttons shows incorrectly and you will need to  amend the form design accordingly.  

If you have difficulty opening the link, copy the link (NB, not the link location) and paste it into your browser's address bar.

In this little demo file the option to "Open simplified form for 'drilling down' through hierarchies to return records" is the relevant one to look at.  To see how the combo boxes are, in this example, correlated so that a County is selected first, then a district, then a parish, study the RecordSource properties of the combo boxes in my demo, and the code in their AfterUpdate event procedures.  The same query would work equally well if the combo boxes were not correlated, however.

The same demo also contains a more complex example where the hierarchy can be entered at any point, rather than having to start at the top and work down.

In your case a complicating factor is the use of LEFT OUTER JOINs. If a query using such a join is restricted on a table on the right side of the join,  this conflicts with the basis of an outer join of course, so, in effect, the join becomes an INNER JOIN.  You might find it better to leave the query unrestricted therefore, and filter the report by building a string expression as the WhereCondition argument of the OpenReport method.  The following code is an example, using four unbound controls in a form, of code in the Click event procedure of the form

    Dim strWhere As String
        
    If Not IsNull(Me.cboDepartment) Then
        strWhere = strWhere & " And Department = """ & Me.cboDepartment & """"
    End If
    
    If Not IsNull(Me.cboCity) Then
        strWhere = strWhere & " And City = """ & Me.cboCity & """"
    End If
        
    If Not IsNull(Me.txtDateFrom) Then
        strWhere = strWhere & " And DateAppointed >= #" & _
            Format(Me.txtDateFrom, "yyyy-mm-dd") & "#"
    End If
   
    If Not IsNull(Me.txDateTo) Then
        strWhere = strWhere & " And DateAppointed < #" & _
            Format(Me.txDateTo, "yyyy-mm-dd") & "#+1"
    End If
