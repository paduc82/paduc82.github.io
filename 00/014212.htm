willy denoyette say:

Some clarifications, long post ahead, sorry.

There is no such thing as WDDM chipsets. WDDM is a Microsoft driver model defining a feature set and a set of libraries (WDDM API set). Chipset/GPU's are not Microsoft specific, they handle other OS'ses which do not use WDDM at all, all they know about is DirectX and OpenGL feature levels.

The libraries used by graphics device driver devs. contain the bulk of the generic driver code. The chipset/GPU specific code is up to the ISV/OEM to implement, they are extremely complex and protected as trade secrets . The specifics can vary from one chipset/GPU to another, they can even vary from one HW revision to another. This model makes it much easier for ISV's to develop graphics drivers, the bulk of the code is already written, they only have to take care of the specifics.

The feature set defines the features that must be presented through code and/or FW (Chipset/GPU).

For instance WDDM 1.1 defines HW accelerated rendering, that means that the GPU must handle it, else it must be emulated in driver specific code.

WDDM 1.3 requires casting feature (say Miracast).

WDDM 1.1 was required for Windows 7, 1.2 was Windows 8 and 1.3 was required to qualify Windows 8.1 devices.

That means that OEM devices (PC's) shipped with Windows 7, must have been WDDM 1.1 (minimum)qualified.

The requirements for new devices shipped with Windows 10 (on sale since summer 2016) is WDDM 1.3. That's why MSBDA is WDD 1.3 qualified, it's build using the WDDM 1.3 libraries (just like on Windows 8.1) and consists of two drivers, a Display driver and a Render driver.

The full feature set for Windows 10 , is defined by WDDM 2.0, most (if not all) OEM ship new devices with WDDM2.0 or 2.1.

Does that mean that WDDM 1.1 cannot be used with Windows 10? No, but don't expect things like video rendering or  Miracast to function, don't expect real HW acceleration and don't expect power management to function correctly.

Does this mean that WDDM 1.0 does not work on Windows 10, probably not, all depends on the implementation, some WDDM 1.0 drivers may implement WDDM 1.1 features but are not qualified so they may fail or have issues.

Would this mean that new devices can run with WDDM 1.1, NO they can't, they won't even install (why would you?).

Another feature is overall device integration, for instance UEFI and power management. A graphics driver running on an UEFI enabled device must handle UEFI events, for instance switching power states (Sleep/standby).

A driver which doesn't handle UEFI state transition may fail to enter or return to/from standby, resulting in glitches, incorrect sync settings, reverting to MSBDA,  black screens (video blanked) or hangs at worst. You see, even when your device seems to work correctly with WDDM 1.1 drivers, you may have issues. Sure, you won't have such issues  running them on old hardware that doesn't implement UEFI and power state managent (think laptops), but may have issues on more recent desktops equipped with Old graphics cards.
