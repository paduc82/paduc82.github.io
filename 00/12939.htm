wandavittek say:

Excel sheet using VBA to change number display

Sub ChangeNumberDisplay()

    Dim rngC As Range

    Dim i As Integer

   

    For Each rngC In Selection

        If IsNumeric(rngC.Value) Then

            For i = 1 To 3

                If Round(rngC.Value, i) = rngC.Value Then

                    rngC.NumberFormat = "0." & String(i, "0")

                    GoTo Formatted

                End If

            Next i

        End If

Formatted:

    Next rngC

   

End Sub

hansv mvp say:

Since your formula depends on the value of K8, the formula cannot be in K8 itself.

hansv mvp say:

Will the values be entered by the user? If so, you could do the following:

Right-click the sheet tab.

Select 'View Code' from the context menu.

Copy the following code into the worksheet module. I have used columns B and E as example, but you can change that to suit your needs.

Private Sub Worksheet_Change(ByVal Target As Range)
    Dim rngC As Range
    Dim i As Integer
    If Not Intersect(Range("B:B,E:E"), Target) Is Nothing Then
        For Each rngC In Intersect(Range("B:B,E:E"), Target)
            If IsNumeric(rngC.Value) Then
                For i = 1 To 3
                    If Round(rngC.Value, i) = rngC.Value Then
                        rngC.NumberFormat = "0." & String(i, "0")
                        Exit For
                    End If
                Next i
            End If
        Next rngC
    End If
End Sub

Please note the following:

Whole numbers will be formatted with 1 decimal place.

The code doesn't reset the number format when a cell is cleared, nor when a value with with more than 3 decimal places is entered.

If (some of) the values are the result of formulas, the above won't work. You'd have to call a modified version of your macro in the Worksheet_Calculate event, but that would be a huge overhead, having a negative impact on performance.

hansv mvp say:

The line

 

Set rst_projects = dbs.OpenRecordset(...)

 

uses DAO. DAO operates at the database level, not at the Access interface level; it doesn't "know" about forms and controls on forms, so it doesn't recognize [Forms]![frm_selectDesigner]![txtDesignerId].

 

You could do the following:

 

    Dim qdf As DAO.QueryDef
    Set qdf = dbs.QueryDefs("qryDesignerProjectPrioritySet")
    qdf.Parameters("[Forms]![frm_selectDesigner]![txtDesignerId]").Value = _
        [Forms]![frm_selectDesigner]![txtDesignerId]
    Set rst = qdf.OpenRecordset(dbOpenDynaset)
