Heading 3 [word count: 548]

-content (has 537 words)

Heading 3 [word count: 502]

a. user say:

Not "real time" exactly but I would consider using a timer to run a macro every so often (every few seconds?).

There are various ways but a starting point would be Daniel Grau's answer here : https://answers.microsoft.com/en-us/msoffice/forum/msoffice_word-mso_mac-mso_mac2011/how-to-create-a-timer-vba/72427a69-622f-477c-a081-b4ba49407f0d

Your macro would then need to do its own Word count for each Heading. You can either let Word count "words" (e.g. by identifying a range, then using its .Words.Count), or use your own word count algorithm. You could insert the result in various ways - one way might be to use a { DOCVARIABLE } field for each count, write the counts to suitable Document Variables in the Document.Variables collection, then update each { DOCVARIABLE } field. You might be able to find a way to relate the naming of the variables to the numbering of the headings, but at its simplest you could use something like

{ DOCVARIABLE "count{ SEQ count }" } in each heading.

If you really need real time I think you would need to be able to work at a much lower level (probably a C/C++ WIN32 project), hook the Windows message loop and write something that forced an update for every key stroke. However, unless your code could discover very rapidly what exactly had changed, you would have to take the "sledgehammer" approach and recount everything. That seems likely to have a large impact on Word performance.
