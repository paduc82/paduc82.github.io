ossiemac say:

OK you are using ActiveX controls. To answer your question "I do not understand what you mean by top and bottom half".

The following screen capture after selecting the Insert button on the Developer ribbon shows the DropDown with the Forms controls in the top half of the DropDown and the ActiveX controls in the bottom half. The coding for the Forms controls goes in a Standard module and the code for the ActiveX controls goes in the worksheets module and the coding syntax is quite different.

I am thinking that maybe you are having some problems getting started with VBA programming with the ActiveX controls so here is a little help.

When you use ActiveX controls, setting the properties of the control and getting to the correct code module is all done in Design Mode.

To use Design Mode, Click the Design Mode icon (Set square, ruler and pencil icon). Design mode actually defaults to "On" when you create an ActiveX control. Design Mode is "On" when it has an orange background and "Off" when it has no background. (In the picture above Design Mode if "Off" with no orange background.)

To commence coding for the ActiveX control:

  • Turn on Design Mode.
  • Right click the control.
  • If you get a DropDown with only 3 options (Move here, Copy here, Cancel)  then try Right Click again. The DropDown you are looking for has about 9 options. (I have been told that the Move etc DropDown is related to exactly where on the icon you right click or the cursor was moved while clicking but I have never been convinced that is always the reason for the incorrect dropdown.)
  • Select View Code from the DropDown and it will create a default sub and end sub for the control in the worksheets module.
  • You also need to know about the ActiveX control properties that can also be selected from the Right Click DropDown.
  • One of the properties of the Option Buttons is the Group. The Group name defaults to the Sheet name. If you want different Groups of Option Buttons then you can change the Group name for each Group where you want only one of the buttons to be "On" and the rest off.
  • The Click event is the most common of the Code subs and Right Clicking the control and selecting View code will automatically create a click event sub. This sub name must not be changed because the naming of events is critical. In the VBA editor you will see two DropDowns at the top of the Editor. The left one can be used to select the required control and the right one can then be used to select the type of code event that is required. It is essential to create the Sub and End Sub by using these DropDowns because the sub names are critical and will not work unless correctly named.

ken sheridan say:

This function should do it:

 

Public Function KilosToStonesPounds(dblKg As Double) As String

 

    Const CONVERSIONFACTOR = 2.204622622

    Dim dblPounds As Double

    Dim lngStones As Long

   

    ' convert kg value to pounds

    dblPounds = dblKg * CONVERSIONFACTOR

    ' get whole stones by means of integer division

    lngStones = dblPounds \ 14

    ' get remaining pounds

    dblPounds = dblPounds - (lngStones * 14)

   

    KilosToStonesPounds = lngStones & " Stones, " & dblPounds & " Pounds"

   

End Function

 

Now that you know the methodology involved you shouldn't have much difficulty converting other units.  The format for the returned value can easily be changed of course.  Tip: when converting values made up of multiple units first convert the value to the smallest unit, e.g. with feet and inches first convert the value to inches: (feet*12)+inches.  Then use the resulting value to convert to the target units.

tom van stiphout say:

The way I do this is by writing conversion functions in a Standard Module, for example:

'The stone or stone weight (abbreviation: st.)[1] is an English and imperial unit of mass now equal to 14 pounds (6.35029318 kg).
Public Function KilosToStones(ByVal dblKilos As Double) As Double
    KilosToStones = dblKilos / 6.35029318
End Function

Then in my query I write:

select myKilosField, KilosToStones(myKilosField) as myStones from myTable;

I'm sure you can write the other conversion functions with a bit of help from Wikipedia.
