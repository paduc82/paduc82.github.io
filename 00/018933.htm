richie weldon - msft say:

Hi Mike -

There is no tech support for My Phone as it is a free product. 

Here are the steps you need to follow to ensure your data on your 6.1 phone is properly transferred to your 6.5 phone after the update.  Do keep in mind this only applied to supported data types that you have set to synchronize with the My Phone service.

1. Synchronize the data on your 6.1 phone.  Any items that were previously synchronized with your phone that are now deleted will get moved from the portal to the portal's archive.  This will happen for any of the supported data types.

2. Apply the upgrade on your phone.

3. On your 6.5 phone, open the My Phone app (or install it if needed) and ensure your have selected the exact same sync options for data types, location, and SD card that you were previously using. Make sure that if you were previously synchronizing contents on your SD card that you enabled synchronization with your SD card before doing the first sync!   Otherwise you will run into issues where all your data is placed on your device's main memory (which you could run critically low on) and may experience duplicate data in the future if you decide to synchronize your SD card again.

4. Log-in to My Phone service on your 6.5 phone and perform the first sync.  You will likely be prompted to download the newest version of My Phone upon your first sync attempt [since the update was released on March 25, 2010].  You will also want to do the first sync with the phone plugged into its charger as the first sync may take a long time and could drain your phone's battery. 

5. That's it.  :)  Your data should now be on your phone.

 

The reason steps 1 - 4 work is that the upgrade to your phone will appear to be a new device to the My Phone software.  New devices will take all the data the Web portal contains and have it copied to the phone upon the first sync.

If you're super paranoid this might not work the way you expect it to, my advice is to use WMDC to back up all your sensitive data (documents, pictures, music, movies, notes), especially your email.  (Remember, My Phone does not synchronize email contents.)  If you have any MMS contents that you want to preserve during the upgrade, you will need to save those contents to your device then synchronize with My Phone (if the contents are a supported data type) or use Windows Explorer with the device tethered to your PC to transfer that data to your PC.

Seeing as I'm a tester on the My Phone product, I've done this type of upgrade numerous times. Usually I do this after a hard reset of my phone or after installing a test version of the phone's operating system.  It's pretty much a non-event, for me at least!  :)

Good luck and enjoy WM 6.5!

 

Richie Weldon

1) No.  But if you have data in your archive that you want brought down to the newly updated 6.5 phone, go into the archive, select that data, and opt to Restore it.

    a. user say:

    As far as I can tell, the only way you can do that in modern versions of Word is to insert the "You can" in the field, so it looks like this:

    { PRIVATE You can }

    You can do that by copying/pasting the text into the field.

    But then it will be visible to anyone who can display hidden text. At one time (a long time ago) it was possible to associate data with PRIVATE, but I have been unable to do so in recent versions of Word. I can make the field an ADDIN field, then set the .Data property of the field, then change the field type to PRIVATE, but in that case, when you save the document as .docx I think the data is removed.

    You could use an ADDIN field instead, hide the field (they are not hidden by default, like PRIVATE fields). It is worth bearing in mind that ADDIN fields are not actually supported by the Office Open XML standard (although they are still available in Word 2013). The field's data is encoded (base64 or some such) when the document is saved as .docx, which is some protection, but not a lot.

    So you might be better off using an ADDIN field. You could insert one like this and store the selected text:

    Sub selectionToHiddenAddinField()
    Dim faddin As Word.Field
    Dim raddin As Word.Range
    Dim s As String
    s = Selection.Text
    Set raddin = Selection.Range
    ' we have to put the field somewhere. Here, it is at the end
    ' of the selection. Not a good place, as the presence of the
    ' ADDIN field will tell the user which option is the correct
    ' answer. Perhaps always store at the end of the answers.
    raddin.Collapse direction:=WdCollapseDirection.wdCollapseEnd
    raddin.Font.Hidden = True
    Set faddin = raddin.Fields.Add(raddin, WdFieldType.wdFieldAddin, "", False)
    faddin.Data = s
    Set faddin = Nothing
    Set raddin = Nothing
    End Sub

    and display the first addin found in the selection like this:

    Sub showSelectedAddinFieldData()
    Dim bAddinFound
    Dim f As Word.Field
    Dim s As String
    bAddinFound = False
    For Each f In Selection.Fields
      Debug.Print f.Type
      If f.Type = WdFieldType.wdFieldAddin Then
        bAddinFound = True
        s = f.Data
        If s = "" Then
          MsgBox "No data"
        Else
          MsgBox s
        End If
        Exit For
      End If
    Next
    If Not bAddinFound Then
      MsgBox "The Selection did not contain any ADDIN fields."
    End If
    End Sub
