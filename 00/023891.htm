Apparently the response that you marked "answer" in the thread "SUMPRODUCT formula is adding values" was not clear to you.

First, you do not need to enter this an array formula.  Just press Enter instead of ctrl+shift+Enter.

Second, the root cause of the problem is:  the parameter (Data!$E$2:$E$7417<50.6)+(Data!$G$2:$G$7417=Ref!$A$6)+(Data!$G$2:$G$7417=Ref!$A$13)+(Data!$G$2:$G$7417=Ref!$A$10)+(Data!$G$2:$G$7417=Ref!$A$11) can return a value >1, not just TRUE (1) or FALSE (0).

To avoid that, you need to change that expression to:


The ">0" returns TRUE(1) when 1 or more of the conditions is TRUE.  The double-negative is needed to convert TRUE or FALSE (the result of ">0") to 1 or 0, as SUMPRODUCT requires.  Any arithmetic operation would do the same.

So you can rewrite your formula in either of the following forms (remember:  just press Enter, not ctrl+shift+Enter):

=SUMPRODUCT(--(Data!$B$2:$B$7417=$A235),
--(Data!$J$2:$J$7417=Ref!$A$4), Data!$L$2:$L$7417)

or

=SUMPRODUCT((Data!$B$2:$B$7417=$A235)
* ( (Data!$E$2:$E$7417<50.6)+(Data!$G$2:$G$7417=Ref!$A$6)+(Data!$G$2:$G$7417=Ref!$A$13)+(Data!$G$2:$G$7417=Ref!$A$10)+(Data!$G$2:$G$7417=Ref!$A$11) * (Data!$J$2:$J$7417=Ref!$A$4), Data!$L$2:$L$7417)

SELECT DISTINCTROW MatchResults_2018.PlayerNames, Avg(MatchResults_2018.Score) AS [Avg Of Scores]

GROUP BY MatchResults_2018.PlayerNames
HAVING (((Avg(MatchResults_2018.Score)) Is Not Null));

dhookom say: I think the issue was PlayDate field is actually [Play Date]. Also there are some players with records on the same date.

This query should work:
SELECT MatchResults_2018.PlayerNames, Avg(MatchResults_2018.Score) AS [Avg Of Scores], Count(*) as NumOf
FROM MatchResults_2018
WHERE (((MatchResults_2018.[Entry ID]) In (Select Top 10 MR.[Entry ID] FROM MatchResults_2018 MR WHERE MR.PlayerNames = MatchResults_2018.PlayerNames AND MR.Score is not null
ORDER BY MR.[Play Date] DESC)))
GROUP BY MatchResults_2018.PlayerNames
ORDER BY MatchResults_2018.PlayerNames;
