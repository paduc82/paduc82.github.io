hairyfool say:

One thing to watch for with printers and particularly with wireless is its address changing. Router DHCP servers do have a mechanism to limit this changing but it does happen. In some drivers there is a mechanism to try and detect this and still find the printer but not always. When this happens the printer will no longer connect to print.

 

There are normally two main solutions to this. One definitely involves being able to modify the router setup the other could be achieved without it at a pinch.

 

Check that the printer driver is correctly configured and can talk to the printer then log into the router. In its menus there is normally an option to make the DHCP server always to issue the same address to the printer on the basis of its MAC address.

 

Alternatively you can set the printer not to use DHCP but use manual addressing. That way it cannot change so the connection will not drop because of a change. Ideally you would modify the DHCP range on the router to exclude a section of addresses and then set the printer in this excluded range.

 

If you do not have access to the router what you can see is the addresses being issued by the DHCP. It is not absolute but most start in the low range such as 2 upwards. This means that the likelihood of the top end ever being reached is remote. so if you use a high value such as 250 it is extremely rare for that to be used in a domestic situation so not cause a conflict.
