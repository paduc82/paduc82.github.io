jason (office engineer) say:

Hi Anne,

Sway does not have any extra support special for remotes.  However, many presentation remotes work by sending signals that mimic keyboard signals and as long as you've given focus to the Sway, it would work.  The specific keyboard commands that we listen for are:

  • Next - Spacebar, Down/Right Arrow, Page Down

  • Jump to start/end - Home/End

You might want to look at the documentation/software for your remote to see what type of signal it is sending.

jeeped say:

... I have a second spreadsheet listing all the skuâ€™s and on here I would like to see all the Document Numbers pertaining for each sku number ... 


Here are a pair of UDF functions that should gather the information that you require.

        

To implement their functionality, tap Alt+F11 and when the VBE opens, immediately use the pull-down menus to Insert, Module. Paste both coded functions into the new pane titled something like Book1 - Module1 (Code). 

Public Function udfConcatenateDocNo(rSKU As Range, rSKUS As Range) As String
    Dim sTmp As String, rSkuFind As Range
    sTmp = vbNullString
    For Each rSkuFind In rSKUS
        If rSkuFind.Value = rSKU.Value Then _
            sTmp = sTmp & rSkuFind.Offset(0, 1).Value & ", "
    Next rSkuFind
    If sTmp <> vbNullString Then sTmp = Left(sTmp, Len(sTmp) - 2)
    udfConcatenateDocNo = sTmp
End Function

Public Function udfOffsetDocNo(rSKU As Range, rSKUS As Range, lPos As Long) As String
    Dim rSkuFind As Range, l As Long, lCnt As Long
    l = 0
    If lPos > Application.CountIf(rSKUS, rSKU) Then Exit Function
    For Each rSkuFind In rSKUS
        If rSkuFind.Value = rSKU.Value Then
            lCnt = lCnt + 1
            If lCnt = lPos Then
                udfOffsetDocNo = rSkuFind.Offset(0, 1).Value
                Exit For
            End If
        End If
    Next rSkuFind
End Function

To use the first custom function to return a concatenated string of Document Numbers use a syntax like,

=udfConcatenateDocNo(A2,Sheet1!$A$2:$A$9)

... and fill down as necessary to catch all of the SKUs. To gather a set of Ducument Numbers in separate cells, use the second UDF function in the form of ,

=udfOffsetDocNo($A2,Sheet1!$A$2:$A$9,COLUMNS($A:A))

... and fill both right and down. to catch all Document Numbers for all SKUs. The COLUMNS($A:A) portion simply increments 1, 2, 3, etc as you fill right. Remember to fill right sufficiently far enough to catch all possible multiples.

nsilmike say: Couple more tips...
1) you want to look for the root domain name.  That means in the address you posted:
https://support.microsoft.com/contactus/emailcontact.aspx?scid=sw;en;1213&ws=WindowsVista
you would look to the left to see the root domain just to the right of the http or the https notation.  The root domain is named just to the left of the 'dot com' or dot edu, etc.  So microsoft is the root domain, and support is a sub-domain on microsoft.com .
2) you want to watch out for redirects...so hover your mouse over the link (don't click) and look to the bottom (typically) of your browser to see the real domain address.  also watch out for 'redirect' commands in the address itself.  Note that you can do this hovering in many email applications as well... and it is extremely important to do this there. It is easy for the bad guys to make an email look like it came from someone trustworthy, so always check links.
3) watch the http vs. https notation, especially when you are a) logging into a site, and b) on a sensitive site (like your online bank account.)  http means hypertext transfer protocol... this is just how data is transferred back and forth between your PC and the server.  http is transmitted in the clear (aka not encrypted) so is easy to listen to over wifi, such as in a wifi hotspot.  https means it is a secure connection, as in encrypted.  Still easy to listen to in a public place, but the data (your login credentials, sensitive info, etc.) is encrypted and not readable by anyone.  you should always see https (even at home) if you are logging into something, or on a site you need to be secure (like online banking, as noted above.)

That's hard to say, since we don't know your schema, but often you use LEFT or RIGHT Joins for this. For example, this query would show me ONLY those Stores that are also in the Sales table:

SELECT * FROM Stores INNER JOIN Sales ON Stores.StoreID=Sales.StoreID

If I want to see ALL Stores, regardless of whether they have a value in the Sales table, I could use a LEFT JOIN:

SELECT * FROM Stores LEFT JOIN Sales ON Stores.StoreID=Sales.StoreID

This shows me all Stores, and also shows me any records in Sales for Stores that have those Records.

If I want to see ONLY those Stores that do NOT have Sales:

SELECT * FROM Stores LEFT JOIN Sales ON Stores.StoreID=Sales.StoreID WHERE Sales.StoreID IS NULL
