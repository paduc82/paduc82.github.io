jllatham say:

There's a 'catch' with setting Calculation to Manual that you should be aware of:

Any other workbooks you open AFTER opening a workbook that has it set to manual will also get their Calculation set to Manual.  If you save them with changes, it "sticks" with them, so the next time you open those files they'll be in Manual mode and the process cascades - workbooks opened during that Excel session will also get set to Manual.

For your testing to see if this helps, only work with the one file: open it up, set calculation to Manual, see if it makes any difference.  If it does, fine and good, but then consider taking it one step further by adding a little macro code to the book that will help relieve the situation (but probably not totally prevent the accidental setting of other workbooks opened during the session to Manual).  If you do this and are saving in Excel 2010 format, you'll have to save it as an .xlsm file:


Private Sub Workbook_Activate()
  Application.Calculation = xlCalculationManual
End Sub

Private Sub Workbook_Deactivate()
  Application.Calculation = xlCalculationAutomatic
End Sub

This is workbook event code and must go into the ThisWorkbook object's code area.  See this page for good explanation of how to do it, with graphics even:

http://www.contextures.com/xlvba01.html

Look for the section on "Copy Excel VBA Code to a Workbook Module".

ronnie vernon say:

Hi

Thanks for the feedback.

You can move individual running apps between Desktops.

1. Click the Task View button.

2. Hover the mouse cursor over one of the Desktops. That desktop will appear on the screen.

3. Move the mouse cursor onto the screen and Right Click the app you want to move.

4. A Move To menu will appear. This will show all of the other Desktops in order. Click the Desktop where you want to move the app.

There has been a lot of feedback about this capability, hopefully we will see a Drag and Drop method and other enhancements for this component.
