For FormField Checkboxes to work, the document must be protected for Filling in  forms.  If the document is not so protected, if you press ALT+F9 to toggle on the display of the field codes in the document, you will see the following in place of the FormField CheckBoxes and the ActiveX Checkboxes:

To be compatible with all versions of Word, back to say Word 97, you would need to use either FormField Checkboxes or ActiveX Checkboxes.

Which you should use depends a little bit on their purpose.  If you need groups of checkboxes to act in an Exclusive Manner, that is most easily accomplished by using ActiveX checkboxes which have a "GroupName" property that causes all checkboxes to which the same GroupName is applied to act in an exclusive manner - that is only one of them can be checked at a time.

To achieve the same behaviour with the other types of checkboxes, it is necessary to use some VBA code.

jay freedman say:

When you're dealing with hundreds of objects, the only reasonable way is with a macro.

Expanding a little on Doug's criteria for which kind of checkbox to convert to:

  • Form field checkboxes require you to apply "filling in forms" protection (in the Restrict Editing pane) to make them work. That causes everything in the document to be locked and not editable except the contents of the form fields. You can, by separating the document with section breaks, mark some sections to be locked and others unlocked, but there are some things that don't work even in the unlocked sections.
  • ActiveX controls have a number of drawbacks, the most important one being a hefty increase in the size and slowness of documents that contain large numbers of them. (For more, see the "Appropriateness for the Task" section in this MSDN article.)

If you decide to use form fields, install this macro in your Normal.dotm template, open the form document, and run the macro.

  

Sub ConvertCCcheckToFormsCheck()
  Dim CC As ContentControl
  Dim FF As FormField
  Dim FFchknum As Long
  Dim FFtxtnum As Long
  Dim rg As Range

  For Each CC In ActiveDocument.ContentControls
    Set rg = CC.Range
    Select Case CC.Type
      Case wdContentControlCheckBox
        CC.Delete
        Set FF = rg.FormFields.Add(Range:=rg, Type:=wdFieldFormCheckBox)
        FFchknum = FFchknum + 1
        NameFF FF, "Check" & FFchknum
      Case wdContentControlText
        CC.Delete
        Set FF = rg.FormFields.Add(Range:=rg, Type:=wdFieldFormTextInput)
        FFtxtnum = FFtxtnum + 1
        NameFF FF, "Text" & FFtxtnum
      Case Else
    End Select
    DoEvents
  Next
End Sub

Sub NameFF(FF As FormField, newname As String)
  FF.Select
  With Dialogs(wdDialogFormFieldOptions)
    .Name = newname
    .Execute
  End With
End Sub

If you want to use ActiveX checkboxes, first insert an ActiveX checkbox in any document based on Normal.dotm, select the checkbox (in Design mode), press Alt+F3, and create an AutoText entry named ActiveX Checkbox that you save in the Normal.dotm template. Then install and use this macro instead:

Sub ConvertCCcheckToActiveXCheck()
  Dim CC As ContentControl
  Dim AX As Object
  Dim AXchknum As Long
  Dim rg As Range
  
  For Each CC In ActiveDocument.ContentControls
    Set rg = CC.Range
    Select Case CC.Type
      Case wdContentControlCheckBox
        CC.Delete
        NormalTemplate.BuildingBlockEntries("ActiveX Checkbox").Insert where:=rg, RichText:=True
        Set AX = rg.Fields(1).OLEFormat.Object
        AXchknum = AXchknum + 1
        AX.Name = "Check" & AXchknum
        AX.Caption = ""
      Case Else
    End Select
    DoEvents
  Next
End Sub

Sub ScratchMacro()
'A quick macro scratch pad created by Greg Maxey
Dim oPar As Paragraph
Dim oRng As Word.Range

r2p2 ut say:

Access: Trim function-currency field formatting

In Microsoft Access: if using the trim function and you include a currency field also with text the currency field loses it's formatting.  Instead of $1,450 the result is 1450.  How can I keep the formatting.

scottgem say:
The Trim function is used on TEXT fields to eliminate leading and trailing spaces. A field formatted as Currency will be a number datatype. (unless you used a format function). In either case, the formatting should remove any leading or trailing zeros. So you should not be using Trim() with a number/currency.
