ASUS Q550LF, NVIDIA GT745M, Windows 10

My laptop was OEM Windows 8. After upgrading to Windows 10 Creators Update, system behaviour slowly became erratic. Initially system would not shutdown properly. Subsequently hard drive LED would show continuously lit but screen would remain frozen. Finally one day when i casually was checking drivers, Display Adapter was showing that nVidia graphics card was showing error code 43. Also nVidia Geforce console was missing.One and half months of experimenting all possible solutions brought nil result.

Finally I read about BIOS flashing. My system was version 212 and current version was 215. Though ASUS site recognised my system as Windows 10 64 bit, the correct version of bios was available under Windows 8 64 bit tab.

I uninstalled Intel graphics drivers, then uninstalled nvidia graphics drivers and then flashed bios ver 215. Thereafter I installed the Intel Graphic drivers and thereafter nVidia graphics drivers in the same order.. My system became absolutey stable and is now as good as it could be.

prospectpc say: Chkdsk does not have a memory leak, it just has an enormous appetite.  It is using all of that RAM to try to work faster.  If there was a memory leak, the memory wouldn't be given back when the process finished, which it does (in all cases I have observed).

The only way I have found to work around this is to start a process that uses a quantity of RAM that you want available while chkdsk runs (a RAM drive or virtual machine works well for me).  Once chkdsk hits the limit of free RAM, it stops; at that point, you can release the other program and chkdsk will work with what it has while the RAM you just freed is available for general computing.
Another piece of advice is to make sure your computer has the extra processor resources available (extra cores more than speed), as running a chkdsk can be somewhat CPU intensive, especially if it's running on disk with problems or a USB connected drive.
I would agree that it would be nice to have an option somewhere (command line or registry) that would artificially limit the use of RAM.  If it's going to take over an hour to run, it doesn't matter if it takes an extra thirty minutes; some of us scan lots of other people's disks but don't have a whole computer to spare for the purpose.

On each PC, create a new folder. right click that folder and choose properties

Click on the Sharing Tab of the resulting dialog and click 'Share'

Then give that folder a Unique Name on your Network

Then restart both PC's
