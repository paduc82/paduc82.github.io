ken sheridan say: Here's a function to do it:

Public Function MultiReplace(varInput, ParamArray varReplacements())

    ' call like this:
    ' MultiReplace("abcdefghijk","c","V","e","X","j","Y","m","Z")
    ' to return this:
    ' abVdXfghiYk

    Const MESSAGETEXT = "Uneven number of replacements parameters."
    Dim n As Integer
    Dim varOutput As Variant
    Dim intParamsCount As Integer
    
    If Not IsNull(varInput) Then
        intParamsCount = UBound(varReplacements) + 1
        If intParamsCount Mod 2 = 0 Then
            varOutput = varInput
            For n = 0 To UBound(varReplacements) Step 2
                varOutput = Replace(varOutput, varReplacements(n), varReplacements(n + 1))
            Next n
        Else
            MsgBox MESSAGETEXT, vbExclamation, "Invalid Operation"
        End If
    End If
            
    MultiReplace = varOutput
    
End Function

norman david jones say:

Hi Training.Goddess ,

How to I find the last match in an unsorted array? Match and index only provide the first as afar as I can see?

Assume that the unsorted data array is in A2:B100, the search value is in C2, try the array formula:

=INDEX($B$2:$B$100,MAX(($C$2=$A$2:$A$100)*MATCH(ROW($A$2:$A$100),ROW($A$2:$A$100))))

Note that as this is an array formula it should be confirmed with Ctrl+Shift+Enter

Norman

amit tandon say:

You may try this non-array formula (per Norman's data range):

=LOOKUP(2,1/($A$2:$A$100=C2),$B$2:$B$100)

This will match the last occurrence of cell C2 value in column A and return the corresponding value from column B.

scottgem say:

No, the CONTROLSOURCE property makes a control bound by binding it to a field in the Form's Recordsource. I suspect you use the combobox wizard to make a SEARCH combo which is an unbound control.

Next, you should NOT use Telephone number and especially not names as keys. Names are not unique and both can change, making them impractical as a primary key. You should use an autonumber as your primary key. If you want to make Telephone Number unique you can apply a unique index to it.

So add an autonumber field to your table and make it the primary key. You will then need to add a Long Integer field to your child tables and run update queries to assign the new PK to these new Foreign keys.

Finally, link the subform on the new PK and you should be OK.

gina whipp say:

Greg,

Sounds like your combo box is UNBOUND which would then explain why it is blank.  This is the correct behaviour for an unbound combo box.  It is also correct that it would retain the value previously selected when going to a new record.  If you would like for your combo box to *follow the record* then in the On_Current event of the form you can place...

Me.YourComboBox = Me.HiddenField

...with HIDDEN FIELD being that value of you combo box in a hidden field on your form.  So you would need a hidden text box on you form that has a Control Source of [Unit No].

To answer your last question BOUND/UNBOUND refers to the Control Source of the combo box not the Row Source.

Now that I've seen your table layout... the easiest solution would be to go back and change how the report is created, so that column B contains a month for each row (Oct, Oct, Oct) in more of a database layout formula- then the formulas are easy.

 

If you can't change the source data, then what you are really asking is "starting in C1448, how can I tell where the /previous/ blank cell is located?" which is something I've never figured out an easy way to do. It is far easier just to use a user-defined function (UDF), which is really just a macro, to calculate this for you.

 

If you aren't sure how to incorporate macros into your workbook, please do a search on that topic, it is a FAQ on this forum and others.

 

Here is a sample UDF that I think will do what you want. You would put this in your workbook in a code module, then save your file as an .xlsm file. Then you would use it thusly:

 

C1448 =CountCustomNames(C1447)

 

Test it to make sure it is returning the correct number, before putting it into production :)

 

 'Note: edited to eliminate linewrap created by MSAnswers website

Function CountCustomNames(rng As Range)
If rng.Columns.Count > 1 Or rng.Rows.Count > 1 Or rng.Value = "" Then
    CountCustomNames = "Input range must be a single cell, referring " & _

                 "to the cell containing the last name of the list (cannot be blank)"
    Exit Function
End If
CountCustomNames = 1
For i = 1 To rng.Row
    If rng.Offset(-i, 0) = "" Then
        Exit Function
    Else
        CountCustomNames = CountCustomNames + 1
    End If
Next i
End Function

Can't find nornal.dot - Microsoft Community

craighamiltonjr say:

Can't find nornal.dot

I'm using Office 2007 under Windows 10. I have a Word file I've been using forever that generates an "insufficient memory" error. It works fin on my laptop. I can't find the normal.dot file to rename it, reboot and reload Word to generate a new normal.dot. I get a bunch of templates when I search, but none named normal.dot. Can you please help?

jay freedman say:

Unless you've specified a different location for the User Templates folder, Normal.dotm will be in a hidden folder at C:\Users\<your login name>\AppData\Roaming\Microsoft\Templates. The AppData folder and everything under it are marked as hidden, which means they aren't usually visible in File Explorer or Windows Search.

To locate and rename the file, open File Explorer, click the View tab, and check the box for "Hidden items"; then use the tree diagram on the left side to work down to the Templates folder. Alternatively, click in the address bar and copy/paste this:

%appdata%\Microsoft\Templates

and press Enter.

Notice that the extension of the file you want is .dotm, not .dot.
