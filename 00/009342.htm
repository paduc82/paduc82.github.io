    Update all fields in a word document at once even if they are - Microsoft Community

    dale newton say:

    Update all fields in a word document at once even if they are locked/greyed-out.

    This is not a question, rather an answer, having come across various threads in this forum and others about this question which have been are locked for no good reason or falsely marked as answered (https://forums.windowssecrets.com/sh...ut-(Word-2010)) and so cannot be replied to with the actual answer, as often one finds on internet forums.  So, here is the answer:

    To update all fields in a word document regardless whether they are locked/greyed-out or not, this is the procedure which I discovered after having the same problem:

    1) Select all the content of the document using Ctrl + A (or select the text of interest).

    2) Put all that selected text in a temporary content control box by going to the 'Developer, Content' tab on the ribbon/in the menu, then clicking on "Insert Rich Text Content Control'. This will temporarily override locks on all fields within that all-encompassing content control.

    3) Right click somewhere inside the selected text (it should still be selected), then click on Update Field.

    4) That's it.

    This method has the advantage that you can update the fields without resetting them to an unlocked state, so when you remove the super big content control around everything in the body of the document, they will still be locked or unlocked, just as you had them configured beforehand

    Note, use with caution. This doesn't seem to update the fields in text boxes within the selected text, or in footnotes. I haven't found a way to do that and Microsfot's Engineers don't seem to be capable of fixing the problem, which has existed for years now.

    Your link is to a thread on Windows Secrets, which is a non-Microsoft forum with a good reputation. It was correctly marked as solved by the original poster. That person wanted to know why the update command was not available.

    Your method will not deal with all locked fields.

    I would not want it to. Fields are locked for a reason.

    Here is a temporary link to a sample document on which your method does not work.

    It has three locked REF fields in the body of the document. The text to which they refer has been changed.

    To update all fields which are not locked in the body of the document:

    1. Ctrl+A
    2. F9

    To update all fields, including locked fields in the body of the document, see Stefan's note below. Remember, though, they were locked for a reason and they will remain unlocked after this.

    1. Ctrl+A
    2. Ctrl+Shift+F11
    3. F9

    To update all unlocked fields in a document may require a macro because not everything is necessarily in the body of the document.

    Here is such a macro:

    Public Sub FieldsUpdateAllStoryMaxey()    ' Greg Maxey
        ' https://gregmaxey.com/word_tip_pages/word_fields.html
        Dim rngStory As Word.range
        Dim lngValidate As Long ' do not know purpose of this - CK
        Dim oShp As Shape
        Dim oTOC As TableOfContents, oToa As TableOfAuthorities, oTof As TableOfFigures
        lngValidate = ActiveDocument.Sections(1).Headers(1).range.StoryType ' do not know purpose of this
        For Each rngStory In ActiveDocument.StoryRanges
          'Iterate through all linked stories
          Do
            On Error Resume Next
            rngStory.Fields.Update
            Select Case rngStory.StoryType
              Case 6, 7, 8, 9, 10, 11
                If rngStory.ShapeRange.Count > 0 Then
                  For Each oShp In rngStory.ShapeRange
                    If oShp.TextFrame.HasText Then
                       oShp.TextFrame.TextRange.Fields.Update
                    End If
                  Next
                End If
              Case Else
                'Do Nothing
            End Select
            On Error GoTo 0
            'Get next linked story (if any)
            Set rngStory = rngStory.NextStoryRange
          Loop Until rngStory Is Nothing
          'Special Cases
          For Each oTOC In ActiveDocument.TablesOfContents
            oTOC.Update
          Next oTOC
          For Each oToa In ActiveDocument.TablesOfAuthorities
            oToa.Update
          Next
          For Each oTof In ActiveDocument.TablesOfFigures
            oTof.Update
          Next
        Next
    End Sub

    The macro above could easily be modified to unlock all fields as well, but I'm not sure I would want to do so. Again, they are locked for a reason.
