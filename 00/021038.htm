Doesn't sound like its split which is probably why its unreliable. EVERY multi-user app should be split between back end with just the tables. And a Front end with everything else stored on the local user's PC. 

For your remote users, the best choice would be a Citrix or Terminal Server setup, when the users log into the Server and run the app from a folder assigned to them. Still accessing the back end also on that server.

A less expensive solution would be to allow the users to remote into a PC at the office and run the app from that PC.

scott mcdaniel say:

As others have said, using an Access database across any sort of WAN (including one that is accessed through a VPN) is just a disaster waiting to happen.

I'm a big fan of Citrix/RDS, but if you can't go that route you could always upsize to SQL Server and see if that helps. You'd still need the split configuration, and a LOCAL copy of the FE would need to be installed on each user's machine. And while upsizing to SQL Server can certainly help it's no "magic bullet". Often you'll need to do a lot of work to take full advantage of SQL Server, and if you're not very knowledgeable about Access (as you've mentioned) then you will most likely have to hire out the project, which of course adds $$ to the process.

How to solve this error code 801812DC? - Microsoft Community

taa_9091 say:

How to solve this error code 801812DC?

It says that they are having trouble backing up the phone... and it is not allowing to update my software.

a. user say:

Hi taa_9091,

 

Thank you for your contribution to the forums. The 801812DC error when updating your phone typically indicates an issue with the OS or firmware on your phone, but before we follow the steps to resolve that type of issue we will rule out any other issues first.

 

The first and best thing to try would be connecting to another PC, preferably on another network. If the update works there, great! If not, that pretty definitively rules out the issue being anywhere other than your phone. If you're not able to access another PC with the Zune software installed, we do have a few other things to try.

  • First, you'll want to check for any updates to your PC at http://update.microsoft.com/microsoftupdate. Make sure to click "Check online for updates from Microsoft Update."
  • You'll also want to try uninstalling and reinstalling the Zune software, using this guide: http://support.xbox.com/en-US/zune/software/remove-and-reinstall-zune. You will want to reinstall using this package: http://www.microsoft.com/download/en/details.aspx?id=27163.
  • Finally you'll want to try creating a new user profile within Windows. This page can show you how: http://windows.microsoft.com/en-US/windows7/fix-a-corrupted-user-profile. Once you create the profile, try updating your phone while signed into the new profile.

If none of those steps resolve the issue, or it didn't work on another PC either, we're going to need to reset the phone to factory settings. This will erase all data on the phone, however you can back up pictures, music, and video by following the steps on this page: http://support.xbox.com/en-US/zune/software/syncing-media. Your contacts and calendar data are backed up to your Microsoft Account as well. To reset the phone you'll want to go to Settings, About, and then tap Reset my phone. Once this has been done and your phone has reinitialized, you should be able to update.

 

Regards,

Sam P.
