moses125 say:

List 1 field value as comma separated string in query

So, I have a situation that I'm not sure how to work out. I need to add to my existing query a field that will list the TimeFrame field as a comma delimited string.

I have tables for Movies and TimeFrame and a join table (MovieTimeT) as 1:M. A query(MovieTimeQ) that lists that information, right join (so all TimeFrames are shown and those MovieIDs with a join). Then, I have a MovieQ (that my form is based on) that I need to add the delimited string field to.

If I add the MovieTimeQ to the MovieQ and just add the TimeFrame field then any movie with more than one TimeFrame is listed multiple times. Instead I would like the TimeFrame field to be concatenated to a single field, but I cant figure out how to do that in the query. 

I found a few ways to do it if the Form was based on the table directly but since mine is based on a query it doesnt help much. Any suggestions or points in the right direction? 

ken sheridan say: If the recordset is empty, i.e. the movie has no time frames, then the MoveFirst method will raise an error, so if this is a possibility, you need to handle it.  In my function it can be done like this:

Public Function GetAuthors(lngPublicationID As Long)

    Const NO_AUTHORS = 3021
    Dim rst As DAO.Recordset
    Dim strSQL As String
    Dim strAuthorList As String
    
    strSQL = "SELECT Author FROM Authorship INNER JOIN Authors " & _
        "ON Authorship.AuthorID = Authors.AuthorID " & _
        "WHERE PublicationID = " & lngPublicationID & " ORDER BY Author"
        
    Set rst = CurrentDb.OpenRecordset(strSQL)
    
    With rst
        On Error Resume Next
        .MoveFirst
        Select Case Err.Number
            Case 0
            ' no error
            Case NO_AUTHORS
            Exit Function
            Case Else
            MsgBox Err.Description, vbExclamation, "Error"
            Exit Function
        End Select
        On Error Goto 0

        Do While Not .EOF
            strAuthorList = strAuthorList & "; " & .Fields("Author")
            .MoveNext
        Loop
    End With
    
    GetAuthors = Mid(strAuthorList, 3)
    
End Function

strAuthorList is just a variable of string data type to hold the comma separated list.  Changing it as you have done is fine.

In the query you'd simply call the function as a computed column to return the list of time frames, if any, for the current movie, e.g.

GetTimeFrameList(MovieID) AS TimeFrames

or in design view:

TimeFrames:GetTimeFrameList([MovieID])

The MovieTimeID surrogate key in your MovieTimeT table is not necessary, and merely introduces an extra, albeit small, overhead.  You can make MovieID and TimeFrameID the composite primary key.  If you do keep the surrogate key, however, then MovieID and TimeFrameID are still a candidate key and must be defined as such by their inclusion in a single unique index.

bernd p. say:

Some facts:

Android 2.3.4 is too old to be supported anymore bei Onenote. Forget that device.

Onedrive is a cloud solution from MS. Necessary for sharing Onenote notebooks with mobile clients on Android, iOS, Windows Phone.

No other special server involved.

On Android you can open existing notebooks from Ondrive or create notebooks on Onedrive.

You can open and create notebooks on Onedrive with Onenote Online, the Web App.

On Windows you can open notebooks on Onedrive with Onenote 2013.

On Windows you can create notebooks with Onenote 2013 locally OR directly on Ondrive.

Where, you decide on File, New.

Notebooks created locally can later be moved to Onedrive by using File, Share.

All notebooks have a master (or original) and a locally cached copy as part of the local Onenote cache.

All clients on all devices have this cache, for performance and for working offline. This copy in the cache gets deleted if you close a notebook.

The master can be on Windows (normally under Documents\Onenote Notebooks). This notebook only can be shared on a LAN.

Or the master can be on Onedrive. Then it can be shared over the Internet with all devices which have a Onenote client. Notebooks newly created must be opened explicitely once on all other devices who want to share them. No automatic appearance.

So I hope all your questions are implicitely answered.

Bernd

Column A is employee name, Column B is employee job code, Column C is employee Hire Date, Column D is employee Term Date.

 

Thank you for your help,

bernie deitrick, excel mvp 2000-2010 say:

Since codes are repeated, you cannot do it automatically by formulas. For each replacement, use a formula like

=Hiredate - TermDate

where Hiredate is the cell with the new employee's hire date, and TermDate is the cell with the previous employee's termination date. You might also add or subtract one from the result depending on your definition of days required to fill the position.

You may also want to pass those dates to the NETWORKDAYS function - see help for more - if you want to exclude weekends and holidays from your count.

 

Bernie
