Sub ashish()
'   Before running this, make sure you have a summary table with column headers.
'   The output table will have three columns.
    Dim SummaryTable As Range, OutputRange As Range
    Dim OutRow As Long
    Dim r As Long, c As Long

    On Error Resume Next
    Set SummaryTable = ActiveCell.CurrentRegion
    If SummaryTable.Count = 1 Or SummaryTable.Rows.Count < 3 Then
        MsgBox "Select a cell within the summary table.", vbCritical
        Exit Sub
    End If
    SummaryTable.Select
    Set OutputRange = Application.InputBox(prompt:="Select a cell for the 3-column output", Type:=8)
'   Convert the range
    OutRow = 2
    Application.ScreenUpdating = False
    OutputRange.Range("A1:C3") = Array("Column1", "Column2", "Column3")
    For r = 2 To SummaryTable.Rows.Count
        For c = 2 To SummaryTable.Columns.Count
            OutputRange.Cells(OutRow, 1) = SummaryTable.Cells(r, 1)
            OutputRange.Cells(OutRow, 2) = SummaryTable.Cells(1, c)
            OutputRange.Cells(OutRow, 3) = SummaryTable.Cells(r, c)
            OutputRange.Cells(OutRow, 3).NumberFormat = SummaryTable.Cells(r, c).NumberFormat
            OutRow = OutRow + 1
        Next c
    Next r
End Sub

cannongeorgelenovo say:

We found the problem and a work-around:

The remote office location does not have a true conference phone.  The party at the remote office location is using a MS Surface Pro for setting up conference calls, video presentation and for audio.

 

For whatever reason, when using Skype for Business the speaker audio output on the Surface Pro is too low to be usable as a conference room speaker. (Interestingly, we don't have low volume issues when running other applications).  So he was mirroring both the video and audio to a wall-mounted flat-screen TY display; not only for the video presentation but for audio.  

It appears latency with the wall-screen TV was responsible for generating the echo for any remote callers.  Although the TV is a late model device, the echo persisted no matter if we use Bluetooth, WiFi, or cables to connect. 

The work-around is to:

1) Mute the sound on the TV

2) Mute your own microphone on the MS Surface Pro.

3) Conference in your own cell phone (select add caller), toggle "speaker mode" on the cell phone and use the cell phone as the audio speaker for those in the conference room.

Admittedly this is a bit clumsy - the audio on the cell phone is not near as loud as that available on the mirror TV display device, but much louder than what we can get on the MS Surface pro when using Skype.  But no more echo!
