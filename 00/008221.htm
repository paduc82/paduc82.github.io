how to remove a message that says 'Error! AutoText entry not - Microsoft Community

mickympl say:

how to remove a message that says 'Error! AutoText entry not defined' on the front page of a word document

Help

How do I change this on a word document front page.

Thankyou
john mcghie say:
You have two choices:

1)  Select it, and delete it

2)  Define the Autotext it is looking for.

When you select it, make sure you get all of it, or it will nto go away: it's not text, it's a generated field.

To define the AutoText, right-click the Error and choose "Toggle Field Codes".  In the code that appears, you will see the name of the AutoText it is looking for.

Go to Insert>AutoText>New and add an AutoText with that name and appropriate text.

Toggle the field codes back to see the result.

vijay a. verma say:

The number of rows which gets supported in a worksheet is 1048576 (i.e. 1 million which Tom Ogilvy has pointed out)

Your macro should be designed in such a way that it should split the files in different worksheets. Hence in your case, it would be 3 worksheets within the same workbook.

Edit - Another option could be - since you have only 4 columns, to copy the records in same worksheet but in different columns. Hence columns A to D will have one set, E to H will have second set and I to L will have 3rd set and so on.

tom ogilvy say:

As a continuation on the exploration of alternatives, if you don't need to see the detailed data but only need to do analysis, you can try creating a pivot table and using your CSV file as the data source (in otherwords, you don't bring it into Excel first).

A pivot table is not constrained by the number of rows in the worksheet, but it is constrained by memory.  As far as I know, there is not a simple formula that would tell you what it can hold - but it takes little effort to attempt it and more than likely due to the small number of columns it possibly could work.  Then you could see if that supports the analysis you need to do.

split, a csv file, every 1048576 entries

Sub Split_Large_CSV()

'June 15, 2015

Const sDelim As String = ","  '<< select comma or semicolon
Dim sFile As String
Const sFile As String = "C:\Users\User\Desktop\folder CSV\file.csv" '<< path and csv file, change
Const Nmax As Long = 7000000: Const N As Long = 1048576
Dim t As Long, N1 As Long, x As Long, y As Long, i As Long, c As Long
Dim v() As Variant
Dim vA() As Variant: Dim vB() As Variant: Dim vC() As Variant: Dim vD() As Variant
Dim fg As Boolean: fg = False
Application.ScreenUpdating = False
Application.Calculation = xlManual
t = 0
ReDim v(0 To Nmax, 1 To 1)
Open sFile For Input As #1
Do While Not EOF(1)
Input #1, Line$: v(t, 1) = Line$
t = t + 1
Loop
Close #1
N1 = Int(t / N): y = (t Mod N)
If y > 0 And N1 > 0 Then fg = True
Sheets.Add
If N1 = 0 Then
ReDim vA(LBound(v) To t, 1 To 1): ReDim vB(LBound(v) To t, 1 To 1)
ReDim vC(LBound(v) To t, 1 To 1): ReDim vD(LBound(v) To t, 1 To 1)
For i = LBound(v) To t - 1
vA(i, 1) = Split(v(i, 1), sDelim)(0): vB(i, 1) = Split(v(i, 1), sDelim)(1)
vC(i, 1) = Split(v(i, 1), sDelim)(2): vD(i, 1) = Split(v(i, 1), sDelim)(3)
Next
Cells(1, 1).Resize(t) = vA: Cells(1, 2).Resize(t) = vB
Cells(1, 3).Resize(t) = vC: Cells(1, 4).Resize(t) = vD
GoTo nnext
End If
For x = 1 To N1
ReDim vA(LBound(v) + (x - 1) * N To LBound(v) + x * N - 1, 1 To 1)
ReDim vB(LBound(v) + (x - 1) * N To LBound(v) + x * N - 1, 1 To 1)
ReDim vC(LBound(v) + (x - 1) * N To LBound(v) + x * N - 1, 1 To 1)
ReDim vD(LBound(v) + (x - 1) * N To LBound(v) + x * N - 1, 1 To 1)
For i = LBound(v) + (x - 1) * N To LBound(v) + x * N - 1
vA(i, 1) = Split(v(i, 1), sDelim)(0): vB(i, 1) = Split(v(i, 1), sDelim)(1)
vC(i, 1) = Split(v(i, 1), sDelim)(2): vD(i, 1) = Split(v(i, 1), sDelim)(3)
Next
Cells(1, 4 * (x - 1) + 1).Resize(N) = vA: Cells(1, 4 * (x - 1) + 2).Resize(N) = vB
Cells(1, 4 * (x - 1) + 3).Resize(N) = vC: Cells(1, 4 * (x - 1) + 4).Resize(N) = vD
Next
If fg = True Then
ReDim vA(LBound(v) + N1 * N To t - 1, 1 To 1): ReDim vB(LBound(v) + N1 * N To t - 1, 1 To 1)
ReDim vC(LBound(v) + N1 * N To t - 1, 1 To 1): ReDim vD(LBound(v) + N1 * N To t - 1, 1 To 1)
For i = LBound(v) + N1 * N To t - 1
vA(i, 1) = Split(v(i, 1), sDelim)(0): vB(i, 1) = Split(v(i, 1), sDelim)(1)
vC(i, 1) = Split(v(i, 1), sDelim)(2): vD(i, 1) = Split(v(i, 1), sDelim)(3)
Next
c = Cells(1, Columns.Count).End(xlToLeft).Column + 1
Cells(1, c).Resize(y) = vA: Cells(1, c + 1).Resize(y) = vB
Cells(1, c + 2).Resize(y) = vC: Cells(1, c + 3).Resize(y) = vD
End If
nnext:
Application.Calculation = xlAutomatic
Application.ScreenUpdating = True
End Sub

2nd section: in E-H

3rd: in I-L
