It'd be a good idea to perform a file repair called System File Check (SFC)


System file check is a utility built into the Operating System that will check for system file corruption The sfc /scannow command (System File Check) scans the integrity of all protected operating system files and replaces incorrect, corrupted, changed, or damaged versions with the correct versions where possible. When it cannot repair the damage DISM should be run to fix as many errors as possible.

System file check works on Vista and UP


Details
If you have modified your system files (including Windows DLL files) running sfc /scannow will revert the system files back to the default state.

It is always a good idea to back up your data beforehand!

To run a system file check (SFC)
Go to start>Type CMD
Right click and run as Administrator
(called an elevated command prompt)
If you want to verify and repair the OS type sfc /scannow (note the space between sfc and "/")
If you just want to check (verify only) the OS type sfc /verifyonly (no changes will be made using verify only)

You may have to run this up to 3 times to fix all the problems

When you have finished it will say one of three things

Windows did not find any integrity violations (a good thing)
Windows Resource Protection found corrupt files and repaired them (a good thing)
Windows Resource Protection found corrupt files but was unable to fix some (or all) of them (not a good thing)

how to remove index field codes in word 2016 - Microsoft Community

bendoran-edmunds say:

how to remove index field codes in word 2016

How to remove index field codes in word 2016?
doug robbins - mvp office apps & services (word) say:

Do you want to actually remove them, or just hide them?  If that is correct, go to File>Options>Display and uncheck the Hidden text box.  On the same screen, make sure that the box for "Print hidden text" is not checked.

If you actually want to delete the fields, you could use a macro containing the following code:

Dim afld As Field
With ActiveDocument
    For Each afld In .Fields
        If afld.Type = wdFieldIndexEntry Then
            afld.Delete
        End If
    Next afld
End With

Note, you would then no longer be able to create an Index in the document.

suzanne s. barnhill say:

If you really want to delete them (and not just hide them) and don't want to use a macro, first make sure they are displayed (display Hidden text), then use the Replace dialog:

  1. Press Ctrl+H to open the Replace dialog.
  2. In the "Find what" box, type "^d XE" (without the quotation marks). This will cause Word to find and select the entire XE field.
  3. Leave the "Replace with" box empty.
  4. Click Replace All.

philip anderson's mother's son say:

The Reset Protection Recovery Key is a security generated code that is linked to the owners Microsoft account.

It is NOT held by the network or Microsoft so e.g another unique code is generated for a new Microsoft account,

You get the recovery code by going to "account.microsoft.com/devices" then use the Microsoft account where your device is registered. Select your device and click the link "Get recovery code", and the code will then be displayed 

As you have already performed a hard reset you are prevented from do this procedure from the handset, you can now only  access this information via PC / Laptop [phones are sometimes sold with reset protection still active so the seller would not physically have the handset and the buyer needs to contact the seller for the code]

garym123 say:

The code you are referring to is for account security. It is probably not a reset protection function.

When you reset a phone and enter your MS account the code is required for account security.

When you set up you account the security section required an email address and/or a telephone number. this is where the code is sent.  You should have used a permanent email address, not the one on the phone.

You can change your security information on your account.

https://support.microsoft.com/en-us/help/12428/microsoft-account-security-info-and-security-codes
