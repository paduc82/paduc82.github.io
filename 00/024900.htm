Print Spooler problems due to HTTP Service not starting - Microsoft Community

cjkee say:

Print Spooler problems due to HTTP Service not starting

Several Windows 10 Tech Preview builds back, my printer stopped working. I tried to start the spooler and it kept failing because of its dependent services, namely the HTTP Service. I finally found a fix to it and thought I'd pass it on in case it helps others out there.

In the Windows System Event Log, I was getting the HTTP Event 15021 error in this log. I found this TechNet article:

https://technet.microsoft.com/en-us/library/cc727844%28v=ws.10%29.aspx?f=255&MSPPError=-2147217396

Every time I tried to do the

netsh http show sslcert

command, I would get "The handle is invalid", and I couldn't follow the rest of the directions in the TechNet article.

The solution was to check this node in the registry:

HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Services\HTTP\Parameters\SslBindingInfo

There are a bunch of keys under this node that are of the pattern "0.0.0.0:443xx". They are all supposed to have the same values for all of the subkeys under each one. In my case, the first one - 0.0.0.0:44301 - didn't have ANY subkeys.

How I fixed it was to export the registry keys for

HKLM\SYSTEM\CurrentControlSet\Services\HTTP\Parameters\SslBindingInfo.

Then opened the exported .REG file for edit (right-click and choose Edit and it will come up in Notepad) and I copied all of the subkeys from another key - like 0.0.0.0:44302 - to the one with the missing subkeys - mine was 0.0.0.0:44301. I saved/closed the file and then double-clicked it to merge it into the registry. The events stopped showing up in the event log, I was able to start the Spooler service, and I was finally be able to start the printer.

I hope this information is of help to someone else out there. It was really bothering me that I couldn't print from my laptop running Windows 10 Tech Preview when I was able to in previous builds.
