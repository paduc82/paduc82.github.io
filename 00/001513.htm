- WordWrap

There is a manual work around that works while the workbook is open.  Once the workbook is closed and re-opened (even after saving), the focus font size reverts to micro size text on focus.

The manual work-around is as follows:
1) Edit ActiveX text box properties
2) Switch MultiLine from True to False then back to True
3) Switch WordWrap from True to False then back to True
4) Blur (select random cell
5) Focus - edit text normally

I have written the following VBA code to resolve the issue (temporarily - Must be run each time workbook is opened)

Sub FixMultiline()
' Platform: Windows 8.0
' Excel Version: 2013
' Macro name: FixMultiline Macro
' Issue: ActiveX Text boxes have tiny text on focus (normal text after they lose focus)

'This will apply the fix to every TextBox on the specified sheets
    Dim myArray As Variant
    Dim x As Integer
    Dim mySheets As Variant
    mySheets = Array("Sheet1", "Sheet3") 'Define your array of sheet names
    
    For x = LBound(mySheets) To UBound(mySheets)                              'Define start and end of array
        With Sheets(mySheets(x))
            For i = 1 To .OLEObjects.Count
                If TypeOf .OLEObjects(i).Object Is MSForms.TextBox Then
                    '.OLEObjects(i).Verb xlVerbPrimary             'Activate the current ActiveX Control (found this on one of your previous posts)
                    .OLEObjects(i).Object.MultiLine = False 'Set MultiLine to False
                    .OLEObjects(i).Object.WordWrap = False  'Set WordWrap to False
                    .OLEObjects(i).Object.MultiLine = True  'Set Multiline to True
                    .OLEObjects(i).Object.WordWrap = True   'Set WordWrap to True
                End If
            Next i
        End With
    Next x ' Loop!
End Sub

kevin_817 say:

I found a reasonable workaround that hides the problem from your 2013 users:

Include the following code as a public macro in a module

Sub FixTinyText()
    For Each form In UserForms
        For Each my_control In form.Controls
            If TypeName(my_control) = "TextBox" Then
                my_control.MultiLine = False
                my_control.WordWrap = False
                my_control.MultiLine = True
                my_control.WordWrap = True
            End If
        Next
    Next
End Sub

Then in your UserForm_Initialize or UserForm_Activate, add this code to the end of the sub

    'bug in Excel 2013. Wordwrap causes the text to be super super tiny during load
    alertTime = Now + TimeValue("00:00:00")
    Application.OnTime alertTime, "FixTinyText"

That way after a load, a non-user-driven-event performs the workaround (without the user noticing)

I haven't done extensive testing, but it is working for now.

Solution on Mac Outlook 2011 printing - page cutting off on the right - Microsoft Community

ravnorodom say:

Solution on Mac Outlook 2011 printing - page cutting off on the right side

Problem: Email print out gets cut-off on right side. Even selecting "Scale to fit paper size" doesn't work.
 
Solution: Under "Page Setup...", create a custom paper size that is 9x11 (instead of using default 8.5x11). Then when print, under the drop down menu select "Paper Handling" and select "Scale to fit paper size". Then click "Print". The total width will be visible on an 8.5x11 paper.
 
Reason: Outlook masks out the excessive width on the right. If you export it as PDF and open it in Adobe Illustrator, the data is there and it's just hidden behind the mask. So by fooling Outlook to think it's 9" wide, then the entire width of email data will be printed on an 8.5x11 paper.
