Within Word, the best you can do is to insert an INCLUDEPICTURE field containing the file path or URL of the image. Whenever the field is updated, it will display the current content of the named file. However, the field (or the document it's in) doesn't have any way to know when the content of the file changes.

You can use a macro in the document (if saved as a .docm file) or in its template to update the field(s) whenever the document is reopened. If you only want a specific field to be updated, insert a bookmark around the field; in the ThisDocument module of the file, add a macro like this (with the actual bookmark name in the quotes):

Private Sub Document_Open()


If you want to update all the fields in the document (including ones that aren't INCLUDEPICTURE fields), you don't need a bookmark, and you can use this code:

Private Sub Document_Open()

If you want to get really geeky, there are utilities known as "file watchers" or "folder watchers" that can do things when a file or folder changes; for example https://sourceforge.net/projects/fwutilities/. You may be able to create some way to send a signal to Word to tell it to update the fields. I don't think that can be done in Visual Basic for Applications (the macro language); you would probably have to use Visual Studio to create a COM add-in.
