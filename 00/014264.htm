don cuthbert say:

The drive lettering in Recovery Environment does not matter except to help us keep track of where things are if we are working on it.  If you boot from a Recovery USB or disk, the letters for the partitions might be different than expected.  You can remove and assign these letters using Diskpart to make them look more like you want.

If you assign a letter to each volume in diskpart and then exit and type bdcedit, you will see that each portion of the BCD is referring to the letter you assigned to a partition.  If you then return to diskpart and switch those letters around and exit again, bcdedit will show the new drive letter assignments for the System and Boot partitions.  The BCD is just translating this from what it will use (e.g. \Device\HarddiskVolume1 might be the system partition, but if you assign Letter=S to it, the BCD will describe it as S: also, to make things appear more clear and to simplify commands that you may need to type).

When trying to repair the boot environment, it is simplest to know what drive letter is representing each volume you need to deal with and you can then use these letters in commands for fixing or replacing the needed partitions.

The X: drive is created as a ramdisk (only exists in RAM memory) and contains what is in the Winre.wim file that is expanded to fill this ramdisk during the boot to Windows RE.

So it doesn't matter much what drive letter is assigned in Windows Recovery.  I usually use Diskpart to set them in a way that I can keep track of, if I want to rebuild a boot environment or if setting up dual-boot, etc.  (e.g. I usually prefer drive C: to actually be my partition containing \Windows, so I often will change it to be C: if necessary).  As long as the BCD is written to match the actual partitions and they contain the necessary files and are properly formatted, the boot environment can be fixed or changed as necessary, using the commands available in Windows Recovery.

No matter what drive letters you use during Recovery, a normal boot should sort things out properly to what you expect.

karl lungmus say:

be able to show number that is negative in red

I have a spreadsheet where I have cells m6,m7,m8 that equal 108 in cell n8 I want that number to show in blue if it equals 108 red if it is less than or black if greater than 108 but I want it to show like if the total for m6,m7,m8 was 105 to show a 3 in red in cell n8 if m6,m7,m8 is 108 then be a 0 in blue for n8 or say m6,m7,m8 is 111 which would show a 3 in black for positive. Also do I have to put the total for m6,m7,m8 in another cell ?   I hope this is clear if not will try and explain better. Thanks Karl Lungmus

jp ronse say: Hi Karl,

If I understood it correctly, you want
- a red 3 in N8 when the sum of M6:M8 is less than 108
- a blue 0 in N8 if the sum of M6:M8=108
- a black 3 in N8 if the sum of M6:M8>108

To display a 3 or a 0 in N8, put this formula in N8:

=LOOKUP(SUM(M6:M8);{0;108;109};{3;0;3})

or depending on your settings (if you are using a comma as separator:

=LOOKUP(SUM(M6:M8),{0,108,109},{3,0,3})

Now, to set the font color.

In the home tab, conditional format (select N8 first)
- New rule & use a formula to determine which cells to format
enter =N8=0 and format the font to blue
- For the red 3, new rule & use a formula to determine which cells to format
enter =AND(N8=3;SUM(M6:M8)<108) and format the font to red

No need to add the third condition because default font color is normally black.

HTH
