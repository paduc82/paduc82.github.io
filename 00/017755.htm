how do i change an existing template in word 2013 - Microsoft Community

rpayneless say:

how do i change an existing template in word 2013

how do i change an existing template in word 2013
jay freedman say:

Depending on what you want to change, you'll usually have to open the template itself instead of creating a new document based on the template. To do that, use File Explorer to go to the folder where the template is stored, right-click the template, and click Open. (The default command, which Windows uses when you just double-click the template, is New -- that creates a document. The Open command opens the template itself.)

Notice that when you open the template, the template's file name appears in Word's title bar. That's different from what happens when you create a document, which just shows something like "Document1" until you save the document.

After making changes in the template, use the Save command just as you normally would, and close the template when you've finished. Then you can create new documents based on the template, which will have the changes included. The changes will not occur in any existing documents -- those documents will have to be modified manually. Read http://shaunakelly.com/word/templates/templaterelations.html for more information.

Some changes can be made without opening the template. If you change the definition of a style in a document, the paper size or margins of  a document, or a few other items, and if the dialog where you make the changes has a "Set as Default" button or a "New documents based on this template" option button, then the changes will be made in both the document and its template at the same time if you click those buttons. (Some add-ins interfere with saving changes in this way to the Normal.dotm template.)

Payback period - Microsoft Community

umair arif say:

Payback period

I am trying to calculate Payback period, I have cashflows for 4 years but those are unequal. and I have net cash flow and and cumulative net cashflow..please help me.
joeu2004 say: Umair Arif wrote:
I am trying to calculate Payback period, I have cashflows for 4 years but those are unequal. and I have net cash flow and and cumulative net cashflow..please help me.

Looking at http://finance.thinkanddone.com/online-payback-period-calculation.html, if A1:A5 are the year numbers (e.g. 0 through 4), B1:B5 are the net cash flows (e.g. -20000, 5000, 4000, 3000, 10000), and C1:C5 are the remaining balances (C1: =B1, C2: =C1+B2, etc), then the payback period is:

=INDEX(A1:A5,MATCH(0,C1:C5))
- VLOOKUP(0,C1:C5,1)/INDEX(B1:B5,1+MATCH(0,C1:C5))

If the years are always consecutive starting with zero, the formula can be simplified somewhat, namely:

=MATCH(0,C1:C5) - 1
- VLOOKUP(0,C1:C5,1)/INDEX(B1:B5,1+MATCH(0,C1:C5))

And to bullet-proof the formula in case the principal is not paid pack within the timeframe:

=IF(MAX(C1:C5)<0, "",
INDEX(A1:A5,MATCH(0,C1:C5))
- VLOOKUP(0,C1:C5,1)/INDEX(B1:B5,1+MATCH(0,C1:C5)))

EDIT....  Further bullet-proofing....

=IF(C5=0, A5, IF(MAX(C1:C5)<0, "",
INDEX(A1:A5,MATCH(0,C1:C5))
- VLOOKUP(0,C1:C5,1)/INDEX(B1:B5,1+MATCH(0,C1:C5))))

Hi,

You probably would notice if someone was remotely accessing your computer. Such as: screen locking up, cursor movement, malware or spyware notifications, advertisements, etc. It's doubtful that this is happening, as Security Essentials is not turning up any malware.

You can check for remote sessions on a network by opening up Computer Management and going to Shared Folders. This would tell you what files have been opened and by who in a certain timeframe. If the lists in Sessions and Open Files are empty, this is good. Also, check to make sure you have no weird open ports or exceptions to the rules in Windows Firewall that you don't know about. Flags would be remote access software, like GoToMyPC or anything ending with VNC or NC. You could use a tool like SNORT to check the traffic that is coming from your computer, too.

Also, in Task Manager, the only names showing up next to all the running services should be your user account, System, Network Service and Local Service. There are a few small exceptions to this, such as "DWM-#" for Desktop Window Manager, but you can probably tell if something is out of place. Finally, the processes should all be originating from System32.

As for the processes that are running, try restarting the computer to see if they are still there. If you just disabled them, it's possible a restart is all you need. I don't think this is really anything to be concerned about yet, but let me know if something else happens.

Good Luck!
