graham mayor say:

To begin with the installation does not 'overwrite' any personal files.
Word 2010 uses an XML normal template format Normal.dotm. It will not work with a DOT normal template format
The installation of Office 2010 should adopt the content of an existing normal.dot template and rename that template to Normal1.dot.
Delete or rename the new normal template - normal.dotm.
Put the file normal1.dot in the folder from which you deleted normal.dotm and rename it back to normal.dot.
Start Word and it should adopt the settings from normal.dot.
Ensure that you have the Word Option > prompt to save the normal template on closing Word set.
Close Word and at the prompt agree to save the template.
Remove normal.dot from the folder to a safe location.
The macros should now be in Word 2010's normal template. If the code is compatible with Word 2010 they should work as before. Some may need to be edited to enable them to work.
Alternatively put the old normal/dot template in the Word startup folder and rename it to (say) macros.dot.
