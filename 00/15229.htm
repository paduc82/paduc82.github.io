rich michaels - mvp (greatcirclelearning.com) say:
I do it a couple of ways. In some cases the Custom Template toolbar is loaded by a global template that I place in the Startup folder; in other cases I have the user simply select the macro enabled template from their "My Templates" folder and the toolbar is loaded then.

The issue of Word 2011 not consistently performing actions at Startup is related to two things. The first being a timing issue and the second being what the user might have done. And specifically I am relating this to Toolbars.

Taking the second issue first, once the toolbar is displayed, if the users hides it from View by deselection it's not going to reappear on a subsequent load of your template, that is unless you programmatically delete the current hidden instance and then reload it at startup. The other issue of timing is about how Word 2011 VBA deals with application events such as Document_Open, Document_New, etc and inserting code execution time delays are required in order to get them to run consistently.

I don't try to dynamically load and unload the toolbars/menus each time the template starts/ends. I establish them in my template so that the first time the template is run, they appear. In subsequent startups of the template if the custom toolbars or menu bar control are listed in the CommandBars collection, I do nothing. If the user chooses to uncheck them from View > Toolbars, our Help information tells them how to make them reappear.

Regarding manually started Templates. The template does not contain any content, other than instructions on it's start page. The macro code, auto text, styles, and command bars are of course contained in the template. The instructions on the templates start page tell the user to choose from our toolbar New, Open or Attach for beginning their work. Depending on what they choose, that's when the macros take over with housekeeping and document setup, etc. 

To see both types, manually initiated template and one loaded into Word's startup folder, you can download them for free from my website, which is listed in my signature line below. Look for LeaderGuide Pro Elements for the Mac and zApps-Doc-Tools for the Mac. Doc-Tools uses a global template loaded into Word's Startup folder and LeaderGuide Pro Elements utilizes the Manual User startup method of File > Open > From Template. Doc-tools puts a Menu Bar Control onto Word's Menu Bar and into some Contextual "Right Click" menus, whereas LGPro Elements uses Toolbars.
