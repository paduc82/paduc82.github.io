garyc_777 say:

Separating First and Last name in a normal name string?

I know there is a way to separate the first name from the last in a  full name string.  (Ex John Paul)

I also have a need to the separate components in this format: (Ex - Paul, John)

Thanks in advance,

Gary

scottgem say:

Sorry, but there is no 100% way to do either. The problem is you encounter named like 

Billy Jo McAllister or Jones, John Paul

If all your names follow a pattern of first space last, then you can use:

strFirst = Left([namefield],Instr(1,[namefield]," ")-1)

strLast = Right([namefield],Len([namefield])-(Len(strFirst)+1))

But Billy Jo McAllister will result in strFirst: Billy, strLast: Jo McAllister

If all your names follow a pattern of last, first then you can use:

strLast = Left([namefield],Instr(1,[namefield],",")-1)

strFirst = Right([namefield],Len([namefield])-(Len(strLast)+1))

But Jones, John Paul will result in strFirst: John Paul, strLast: Jones

So even though you can get most of the conversions, you will still need to eyeball the results.

I wondered about this one for the longest time and have finally figured it out. It involves a Word document having several sections where the headers do not align correctly from the top. Viewing the document through Print Preview shows that the alignment from the top of the page is inconsistent; i.e., some headers will be slightly lower on the second page of the section, some sections do not seem to line up at the same place from the top from section to section. 

To solve the first issue, the measurement for the top of the Margin must be the same as the measurement for the Header in the Margins and Layout tab (File > Page Setup).

If the value shown is not "IPM.Appointment" but rather something like "IPM.Appointment.SomeName" - you are using a custom form for the folder which likely has some validation rule for a field that is not being met. In this case, the options are:

#1 - If you don't want to use a custom form - select <IPM.Appointment>
#2 - If you do need the custom form, you will need to find out what field has validation rules and either correct as required or ensure that whatever it is looking for is entered prior to saving the appointment.

ffgrant say:

Find & Replace feature doesn't find what it should when using wildcards and when tracked changes are shown in balloons.

When using the Find & Replace feature with wildcards, word will not find text that it should find if the text contains deleted characters that are shown in bubbles by the Track Changes feature.  As an example,

make sure track changes is turned off initially.

type "[15]" somewhere in a document.

now turn track changes on.

now delete "15" and replace it with "13"

make sure revisions are shown in balloons with the track changes feature

using the find and replace feature, with wildcards enabled, find "\[13\]".  it doesn't find the [13] like you would expect it to.

now, what's interesting... change the track changes feature so that it shows revisions inline instead of in bubbles.

now use the find and replace feature to find "\[1513\]".  now it finds the text, including what's already been deleted!

This is a problem, because i can't effectively use the find and replace feature with wildcards on a document that has changes tracked.

Is there some fix for this issue?  Seems like a big bug to me.
pamcaswell say:

The text hasn't really been deleted until you accept the change.  That's why the search can't find your specified string. 

But if you switch the track changes view to Final ,  Word will find "[13]" in a regular search—&, yes, if you switch the view to Final: show markup   while [13] is selected, the whole markup string ([1513]) will be selected.

You will get better results if you run searches and spell checks on heavily edited documents from the final view when you cannot accept changes.
