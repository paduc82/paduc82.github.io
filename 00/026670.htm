macropod say:

Unless you're using manual page breaks and/or 'Next Page' Section breaks to set the page boundaries, what you're trying to do may end up being futile, since Word paginates documents differently according to data returned by the active printer driver. Consequently, having done the work, simply changing printers (or even just printer drivers) can invalidate it. That said, you could try something based on the following:

Sub Demo()
Application.ScreenUpdating = False
Dim i As Long, j As Long, Rng As Range
With ActiveDocument
  For i = 1 To .ComputeStatistics(wdStatisticPages) Step 2
    Set Rng = .GoTo(What:=wdGoToPage, Name:=i)
    Set Rng = Rng.GoTo(What:=wdGoToBookmark, Name:="\page")
    With Rng.Duplicate
      With .Find
        .Text = "t"
        .ClearFormatting
        .Replacement.ClearFormatting
        .Format = True
        .Forward = True
        .Wrap = wdFindStop
        .MatchWildcards = True
        .Execute
      End With
      j = 0
      Do While .Find.Found
        j = j + 1
        If .InRange(Rng) Then
          If j Mod 3 = 0 Then
            .Font.ColorIndex = wdRed
          End If
          .Collapse wdCollapseEnd
          .Find.Execute
        Else
          Exit Do
        End If
      Loop
    End With
  Next
End With
Application.ScreenUpdating = True
End Sub

As coded, the macro simply sets the font colour for every third 't' on odd-numbered pages to red.

On Error GoTo PROC_ERR
    If Len(sOldPasswood) <> 0 And Len(sNewPassword) <> 0 Then
   
        Dim tdf As DAO.TableDef
        Dim qdf As DAO.QueryDef
        Dim sConnect As String
       
        ' Refresh Access Linked Tables
        For Each tdf In CurrentDb.TableDefs
            If InStr(tdf.Connect, sOldPasswood) Then

        Next

PROC_EXIT:
    Exit Function
