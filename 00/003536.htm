Publisher 2010 measurement tool

To get the measurement toolbar:

-Draw a textbox and keep it selected. With one selected, a drawing tool tab appears at the top of the page, click it.

-In the drawing tool ribbon, all the way on the right, you will see a measurement button in the Size area.

-Click the measurement button, and the toolbar will appear.

You can right click the measurement button and add it to the quick access toolbar, and it will be there every time you start a Publisher document.

April 2017 update to Microsoft Office disables EPS import - Microsoft Community

steve rindsberg say:

April 2017 update to Microsoft Office disables EPS import

As of the April 2017 update to Office 2010, 2013 and 2016, you will no longer be able to insert EPS files as pictures into PowerPoint and the other Office applications.  This PowerPoint FAQ article:


I can no longer insert EPS graphics into PowerPoint

explains why Microsoft made this change and includes some workarounds in case you need to continue using EPS graphics.  Please do note Microsoft's warnings regarding the advisability of re-enabling the EPS filter.

My take on the security issue:  If you're using EPS graphics that you've created yourself or have received from known, trustworthy sources who are creating them for you, there's probably not a great deal of risk in re-enabling the EPS filter, at least until you're able to standardize on another graphics format.

steve rindsberg say:

If you made the regedit change described *on the PPTFAQ page*, that won't solve the problem.

That change tells the EPS filter to import the EPS according to Adobe's EPS specs rather than converting the EPS to EMF, as the MS EPS filter would normally do.

In order for this to work, you must first enable the filter in the first place using the method shown in the link from the PPTFAQ page:

This incredibly confusing page

The way this works is that you create an AllowListEnabled DWORD = 1 entry under Microsoft\Office\Common\Security\AllowLists\GraphicsFilterImport (in the appropriate section for your version/bitness of Windows).

On my Office 365/Win7 64-bit system, I had to create everything from Security on down under Common.

Then under the same key you'd add a string entry for each filter that you want to allow, so for example you'd create a REG_SZ key named:

EPSIMP32.FLT with its value set to 2010.1400.4740.1000

That's the EPS filter name and version for Office 2010.

You can get the filter names and values from the table at the top of the page I linked to here.

When I tested this with the correct filter version and reg entries, it worked as expected ... where it had given me a red x + "This can't be displayed" shape before, it now imported the EPS (as an EMF).
