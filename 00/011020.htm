Click-to-Run is the technology used to install Office 365 subscription (like Office 2016 in this case). Windows Installer technology (MSI) is used to install the volume license editions of Office, like Microsoft Office Professional Plus 2013 and Microsoft Visio Professional 2013 in this case.

At present, the coexistence of MSI and Click-to-Run installation of Office on the same computer is not supported.  We can't have Office products of the same version (like Office 2016) that use two different installation technologies installed on the same computer.This is the reason why you may get the error mentioned above.

john - project say:
murkaboris,
First you don't mention which version of Project you are using. Second, unless you are using Project 2010 in manual schedule mode you should not be entering dates into the Start and Finish field. Duration and links allow Project to calculate the schedule.

However the main reason you are not seeing the critical path is because you are looking at the normal Gantt Chart view. By default there is no bar style for critical tasks in the Gantt Chart view. You can either add that to the Gantt Chart bar styles definitions or simply use the Tracking Gantt.

A similar condition applies to the Calendar view. The bar styles definition for the Calendar view does include critical tasks, but it is  "blue", just like normal tasks, instead of the historical "red" for critical tasks. You can easily redefine the color.

Hope this helps.

John
