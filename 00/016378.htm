hansv mvp say:

For any other than the first sheet in the workbook, you have to specify the Range argument, in the format "SheetName$".

So if you want to import a sheet named Schedule, use

 

   DoCmd.TransferSpreadsheet acImport, , "Schedule", vPath & "\MPL Project Report.xls", True, "Schedule$"

 

By the way, the name of the imported table does not have to be the same as the name of the sheet, so you could use

 

   DoCmd.TransferSpreadsheet acImport, , "OtherName", vPath & "\MPL Project Report.xls", True, "Schedule$"

 

This will import the sheet named Schedule to a table named OtherName.
