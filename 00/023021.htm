tom_bpc say:

Asif-Ali,

This is exactly what happens when the effective calendar provides no working time between the Actual Start and the Actual Finish.

Example:

  • A simple project with only a Standard calendar (M-F x 08-17).  Project will never schedule a task to be performed on the weekend because there is no working time available.
  • Task A is in fact started and finished over the weekend - with an Actual Start at 08:00 Saturday and an Actual Finish at 17:00 Sunday.
  • When you record these AS and AF, Project decides that a) the task is 100% complete, b) Duration = Actual Duration, and c) Actual Duration = AF minus AS, in working days computed according to the controlling calendar for the task.
  • Since Saturday and Sunday are non-working days, AF minus AS = 0, and the task becomes a milestone.

To fix, set your project, resource, and task calendars to match the actual working times of the project.

Good luck, tom
