hansv mvp say:

Let's say you have data in A1:B100. Enter the following formula in C3:

=VLOOKUP(C2,$A$1:$B$100,2,FALSE)

If the value in C2 doesn't match any of the values in column A, this formula will return #N/A (=not available). If you'd rather leave C3 blank in that situation, change the formula to

=IFERROR(VLOOKUP(C2,$A$1:$B$100,2,FALSE),"")

Instead of using code to set the .Result of the FormFields in the first section, use it to set the value of Document Variables and replace the relevant FormFields with DocVariable fields.

So instead of using code such as

With ActiveDocument

    .FormFields("bookmarkName").Result = "Somevalue"

'etc.

End With

Use

With ActiveDocument

    .Variables("variableName").Value = "Somevalue

'etc

    .Range.Fields.Update

End With

and in place of the FormFields, use { DOCVARIABLE variablename }

teylyn say:

Hello,

in many situations, the Import wizard can be used to specify the order of year, month and day in an imported column.  In this situation, though, since the year is not present at all, Excel cannot interpret this as being a date in the current year. 

I suggest you use the Import Wizard and declare the column as Text. Then it will remain as "Jun 23 00:00" in Excel and you can use a formula to create the desired date.

To invoke the Import Wizard, don't just open the CSV file. Instead, open a new, empty Excel file, then use the Date ribbon > From Text command and select the CSV file.  In Step 3 you can select the date column and define it as Text.

The formula to create a date in the year 2014 with that data is

=DATEVALUE(MID(A1,5,2)&"-"&LEFT(A1,3)&"-2014")
