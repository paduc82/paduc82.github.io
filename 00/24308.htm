ken slovak say:
Custom headers (x-headers) will not translate into user properties without writing code to get the message headers as one long string and parse them looking for your x-header. Code to add those headers or preserve incoming headers would have to be running not only on your machine but with every one of your correspondents. Then the code would need to get the parsed x-headers and create user properties for each one.
 
So you need code that would be a COM addin (for deployment), every one needs to run it, and you need custom views to show the user properties created from the x-headers.
 
There are samples of creating x-headers using the Redemption library that you could search for on the Web, those could provide a starting point for your code. You'd also need Visual Studio to create the addin.

All that can be done but don't expect anyone to create this for you for fun. You would need to be willing to learn what you need to do to create and deploy such an addin or you would need to be prepared to pay someone to do that for you.

I have got the solution fix for KB3122947 update failure, no command - Microsoft Community

ɱɱd say:

I have got the solution fix for KB3122947 update failure, no command prompt or anything, a solution for normal install via Windows update...

Hi guys.

No need for any command prompt!! an easy and harmless fix, download Media Creation Tool, run it and just close it without doing anything inside of it, now try the update again. It will install fine. :)

Reason: the update installs fine if you have "$Windows.~WS" and "$WINDOWS.~BT" folders on "C:" drive, and possibly you have removed them before by running the disk cleanup tool, however by just running the Media Creation Tool it creates those folders. After the update installs you can safely remove these 2 folders. (make sure you have the Show Hidden Files option enabled to see them.) The reason (I think) that the KB3122947 update fails if they are not present is that it tries to look for them searching for older settings inside them to migrate those settings from your old Windows build (which should be inside those folders along with Windows.old) after the November update.

Hope this helps. :)
