Creators Update breaks iCloud sync to Outlook - Microsoft Community

mickey segal say:

Creators Update breaks iCloud sync to Outlook

Once again, a Microsoft update has broken the ability to sync Outlook with iCloud.  Such syncing is used to share contacts and calendar with iOS devices such as an iPhone.  

Fortunately, there is a solution at https://discussions.apple.com/message/31626826.  It is:

1. Redownload the latest iCloud for Windows.

2. Uninstall current iCloud

3. Reboot

4. Reinstall iCloud

5. Reboot (it makes you)

6. When iCloud comes up, UNCHECK sync of mail, contacts, and calendar, and apply.

7. Reboot

8. When iCloud comes up, CHECK sync of mail, contacts and calendar, and apply.

It worked for me.  I actually did the re-download after the first re-boot, and also signed-out from iCloud as part of the uninstall, but that doesn't seem materially different from the procedure described above.

ken sheridan say: You can either simply omit any rows WHERE Status = "Cancelled" form the report's recordset, or if you need to include the rows in the recordset for some reason, e.g. for aggregation of values in a group or report footer, put the following in the detail section's Format event procedure:

Cancel = (Me.Status = "Cancelled")

scottgem say:

As Ken said, you don't hide records in a report, you filter the records you don't want from the report's recordsource. Reports are usually based on a query and you simply set the criteria for the Status column to not include those where Status is Cancelled.

Another option would be to use the WHERE clause of the OpenReport method. Something like:

DoCmd.OpenReport "reportname",,,"[Status] <> 'Cancelled'"
