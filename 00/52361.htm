Uh oh, no save points. Apparently, this feature is off by default in Win10 and I never checked it.

After further Googling I discovered that an option exists that can be set by group policy to display log in history. Another Google search and I found a reference to the registry key that controls this option which is located in:

HKLM\Software\Microsoft\Windows\CurrentVersion\Policies\System

The value of DisplayLastLogonInfo was set to 1. I set the value to 0 and restarted. I was then able to log in to the "corrupted" account. I have no idea how the value got set to 1. Hope this helps someone since Microsoft tech support seemed to have no clue and it was not easy to track down all the information needed to resolve it.
