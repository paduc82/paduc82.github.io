daniel g. grau say:
I don't have to quit for the change to stick. As long as the checkmark in Add to template is applied, and as long as I don't click on any Cancel button, new documents take on the redefined indent right away.

Let's try this first. Enable Word > Preferences > Save > Prompt to save Normal template. Change the indent once again (don't forget to Add to template), then quit Word. At that point, Word should ask you if you want to save changes to the Normal template, and you'll know that the change was added. If it does not ask you, there is something weird going on and you should do as follows:

  1. Quit Word
  2. Go to the Finder
  3. In the Finder's menu bar, select Go > Go to Folder
  4. Paste the following bold text: ~/Library/Application Support/Microsoft/Office (include the twiddle at the beginning) and press the Go button.
  5. In that folder, you should find a file called Normal.dotm. If you don't see it, look for it inside a folder called User Templates. Leave the folder's window open.
  6. Press the Option key and drag the Normal.dotm template to the Desktop to make a copy.
  7. Launch Word and, from its File > Open menu, locate the copy of Normal.dotm on your Desktop to open it. (You have to use this menu option, because clicking on the template in the Desktop would just open a blank document based on it.)
  8. Now you are editing the copy of Normal.dotm and should see the hanging indent paragraph. Correct it and close the template. [Do this through Format > Styles > Modify to change the paragraph style as per previous instructions.]
  9. Quit Word and put the corrected template on your Desktop back into the folder from where you copied it from, replacing the original.
  10. Open Word and test.

    1) You can set the Recordsource of your subform based on the value selected in your combo like this:

    Sub YourCombo_AfterUpdate()

      Me.Recordsource = "SELECT blah blah FROM YourTable WHERE SomeField=" & Me.YourCombo.Column(0)

    End Sub

    If the value is a Text field, enclose it in quotes:

      Me.Recordsource = "SELECT blah blah FROM YourTable WHERE SomeField='" & Me.YourCombo.Column(0) & "'"

    You can also Filter instead of doing that, if you'd prefer:

    Sub YourComb_AfterUpdate()

      Me.YourSubformControlNAME.Form.Filter = "SomeField=" & Me.YourCombo.Column(0) 

      Me.YourSubformControlNAME.Form.FilterOn = True

    End Sub

    Where YourSubformControlNAME is the name of the Subform CONTROL on your main form, and may or may not be the same as the form you're using as a Subform. Note that if the value is Text, you'll need to enclose it in quotes as mentioned previously.

    2) Access will save your data when you move off the record, but if you want to force a save you can do this:

    If Me.Dirty Then Me.Dirty = False

    To go to a new record, run this code:

    DoCmd.RunCommand accmdRecordsGoToNew

    You must do this from the code module of the Form where you want this to happen, of course.
