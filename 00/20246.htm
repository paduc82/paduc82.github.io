xps350 say:

Having just 1 table seems not to be right.

I think you should have (at least) three tables: "person", "service" (to calculate the first date of service), "trip" (to count the total of trips).

kat702 say: I have lots of other tables, but they auto populate this table.   I have a contacts Table with the same Client ID.  Then I have this table with the trips.  All the fields I listed are in all tables, with the exception of the trips.

scottgem say:

I have lots of other tables, but they auto populate this table.   I have a contacts Table with the same Client ID.  Then I have this table with the trips.  All the fields I listed are in all tables, with the exception of the trips.

That is not very good design. One of the principles of a relational database is to try to eliminate redundancy. Having data in multiple tables is just not a good idea. 

So I'm assuming you have a table that lists all the trips for a client. Now you want to show the total of trips for each client with the first date of service. In that case you should have a Group By query, something like this:

SELECT ClientID, Count(TripID) AS TotalTrips, MIN(TripDate) AS FirstService

FROM Trips

GROUP BY ClientID;

This will give you one record per client with the total # of trips and the first date of service. You can then use that query to join with the Client table to pull the name, address and phone info.
