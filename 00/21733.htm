andreas killer say:

I get the prompt "Select([Replace])" immediately after typing the word "case."  

Got it! You see this:

That happens only if you type the line in a code module of a worksheet. The reason is that code modules of a sheet (and ThisWorkbook) works a little special, they create an automatic reference to the sheet object.

In a regular module you must declare an Object variable (of any type) and refer to that variable (e.g. with a WITH statement) and after you write a dot the members appear.

In a sheet object you don't need to create such a reference, when you just call Select, this Select refers always to the sheet where the code is located in.

More often as with SELECT users struggle about the behavior of RANGE, because RANGE in the code module of a sheet refers always to the sheet where the code is located in. But a RANGE in a regular module refers always to the active sheet!

Example:

Make a new file with at min. 2 sheets
Copy this code into the code module of Sheet1

Sub RangeTest()
  Range("A1") = Range("A1") + 1
End Sub

Create a regular module and paste this code

Sub AMain()
  Sheet1.RangeTest
End Sub

Close the VBA editor
Make sure that Sheet2 is to active sheet
Press Alt-F8, as you see you have 2 subs, run sub AMain (a few times)

Then look into Sheet1

Tada, A1 contains a number

When you copy RangeTest into the regular module, each run increments cell A1 of the active sheet (Sheet2 or Sheet1).

Long talk, short end: Ignore the tool tip from VBA in a sheet and use SELECT CASE as usual, it works.
