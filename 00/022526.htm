In VBA, and specifically MS Forms, what is the most efficient way to - Microsoft Community

ewgssi say:

In VBA, and specifically MS Forms, what is the most efficient way to test if a control object has a particular property?

There are times when 'On Error Resume Next' just propagates the error rather than stepping around a condition. One would expect there would be a generic method such as object.HasProperty.

So far, this is the best I have come up with:

Public Function hasProperty(ctrlObj As Control, propName As String) As Boolean
    '=========================================================================
    '   Determine if the given control object has the defined property
    '=========================================================================
    Dim tstString As String: tstString = xTESTTAG
    
    On Error Resume Next
    
    Select Case propName
        Case xCONTROLSOURCE & xND           '<controlsource  />'
            tstString = ctrlObj.ControlSource
        Case xROWSOURCE & xND               '<rowsource  />'
            tstString = ctrlObj.RowSource
        Case xVALUE & xND                   '<value  />'
            tstString = ctrlObj.Value
    End Select
    
    hasProperty = InStr(tstString, xTESTTAG) <> 1
End Function

The inline call is:

    If hasProperty(genFormCntrl,xCONTROLSOURCE & xND) then

             genFormCntrl.ControlSource = genFormCntrl.ControlSource

    End If

rradhay say:

Vibrate for Calendar alerts in 6600 not working

How can I enable vibrations for Calendar Alerts in my Nokia 6600? The phone vibrates in Silent mode fine when a call or a text message comes in, but I cannot get it to vibrate for Calendar alerts in Silent mode. Please help!!!!:( Thanks in advance.

aanu77 say: is there a vibration option for this???? :0

rradhay say: Nope. Apparently the only way to do this is to 'Personalize' the 'Silent' 'Profile' s 'Ringing type' to 'Ringing' (do not have it on 'Silent' as this prevents vibration for Calendar alerts, but allows vibration for incoming phone calls and text messages). Then set the 'Silent' 'Profile' s 'Ringing tone' to 'Off' & 'Vibrating alert' to 'On' for the silent profile. Also in the 'Silent' profile, put the ringing volume at the lowest, and choose a Calendar alarm tone to one which is very quiet such as 'door knock.awb'.

The result will be that in Silent Mode, the phone vibrates for phone calls, messages AND Calendar entries, but the Calendar alarm tone will sound, but very quietly (practically inaudible) once you set up a tone which is inherently quiet for the calendar alert.

Color Coding Gantt Bars in Project 2003 - Microsoft Community

dlyonliny say:

Color Coding Gantt Bars in Project 2003

Hi, is there a way to change the Gantt Chart bar color based on Resource group.  Sadly, my colleagues are extremely lazy so in order for them to read a timeline, I have to give each functional area (Marketing, Operations, R&D, etc.) their own color text.  I'd like to change the Gantt chart bars to be the same color as each of the functional areas' text color.  Can you do this in Project 2003?  Many thanks!
joël séguin say:

Hi dlyonliny,

Yes, this is feasible. Not practical nor funny to do, but feasible :

- In a task view, insert as many new task Flag fields as you have resource groups.

- Customize each field. Rename it to a name like "Flag MyResourceGroup", where MyResourceGroup is the name of the group. Edit the formula like this :

InStr(1,[Resource Group],"MyResourceGroup",1)

--> keep these quotes around the name of that resource group.

- You can now remove the Flag fields from the table. You won't have to do that work anymore unless you add another group.

- Apply the view you want to customize. Format > Bar Styles. Select the Task bar style. Cut. Paste once to get it back, and paste again as many times as you have resource groups. Leave the first Task bar style as is. For each other Task bar style, rename it with the name of a resource group. Select the Show For ... Tasks cell, and in the edit box on top, add a comma (,) then the FlagX field name related to that task. Change the task bar style color. Do this for each group on a different bar style. Keep in mind that bar styles are drawn from top to bottom, so the latest bar style will take precedence as it will be drawn after the other. So keep your most important groups at the end.

That's it.

Hope you got it all :)

Joël
