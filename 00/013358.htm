Record is too large - Microsoft Community

ricardolgomez say:

Record is too large

I am trying to import a very small Excel file into MS Access [350kb, 402 rows, 93 columns]. However, when I try to create a new table I got an annoying warning saying that "record is too large."  I cannot figure out what's wrong with my file. Thanks in advance for your suggestions

ricardo
john w. vinson say:

There's a somewhat obscure and sometimes irksome limitation on Access tables: the total amount of data ACTUALLLY OCCUPIED in a single record cannot exceed 2000 characters (4000 bytes of Unicode). So if you have 93 columns with (on average) 25 characters, you can easily exceed this limit!

 

Memo fields don't count toward the limit; number fields do, with various sizes (two, four or eight bytes depending on the datatype).

 

93 columns is a VERY wide table - you may want to consider setting up a normalized set of "tall thin" tables, linking to the spreadsheet and running multiple Append queries to move 10 fields into this table, 20 fields into that.

How to force IE to use HTTP/1.1 over SSL through Proxy - Microsoft Community

oldandevil say:

How to force IE to use HTTP/1.1 over SSL through Proxy

Internet Explorer sends SSL requests through a proxy as HTTP/1.0. How can I force it to use 1.1?
The registry settings do not change the request as stated in the KB articles.

I also noticed that the header field also adds "pragma: no-cache" which I do not want.

These apparently are the default settings for IE6-8 in which I cannot find a way to force 1.1 or change the no-cache.
oldandevil say: It took a long time to discover the causes.

1. The web server we were working with originally was misconfigured. Since it was apache, it had a setting
     "^Moz.*" downgrade-1.0 force-response-1.0
     This setting forced ALL browsers to use HTTP/1.0. So, all you apache admins, please DON'T put a setting on your web server to downgrade HTTP!

2. The second cause I found belonged to Blue Coat proxy. By default (if you have authentication enabled) any SSL connection going through a Blue Coat proxy will "Act" like HTTP/1.0. This is due to the authentication "mode" being set to 'auto' by default.
When the proxy has to 'auto' determine the mode, and each request will have to go through the process and the proxy treats them all as a new connection, forcing the SSL to re-negotiate and close each one. Changing it to "Proxy IP" is recommended by Blue Coat.
If you have authentication on your Blue Coat, then I would HIGHLY recommend changing the mode to the Proxy IP option. ALL your SSL traffic will be reduced, lowering your cpu utilization, network load, and all SSL web server loads as well!

  3. Click the Device Manager Button.
  4. Expand Universal Serial Bus controllers.
  5. Right-click every device under the Universal Serial Bus controllers node, and then click Uninstall to remove them one at a time.
  6. Restart the computer, and then reinstall the USB controllers.
  7. Plug in the removable USB storage device, and then test to make sure that the issue is resolved.

  1. Press Windows key + X and select Device Manager.

  3. Right click on all the USB driver and select Update Diver Software one by one.
  4. Restart your PC and check if it helps.
