suzanne s. barnhill say:

The form should be created as a template (.dotx) so that users will be working in a new document based on the template, not the template itself. The template will be unaffected if they save the document before or after printing.

The only other approach is to create the form as a .dotm or .docm with a macro to intercept the FileSave or FileSaveAs command. But if users disabled macros in the document, this protection would be useless.

For example - SkyGo Desktop lauches the 64bit browser - which then fails as Silverlight is not installed.

umesh p say:

Hi,


You can change the program windows uses by default to open with different file extensions. The below steps has helped few in resolving similar issue. Lets try the same and check.

Here is how:

a. Click on Start and go to Default Programs.
b. Click on Associate a file type or protocol always open in a specific program.
c. Browse to all the extensions where internet explorer is set as default and then click on change.
d. Now Click on browse and then go to location C:\Program Files (x86)\Internet Explorer\iexplore.exe and select it.
e. Click Ok and then Close.
f. Click Save.
