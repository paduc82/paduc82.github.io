Copying Colors from a Website

Technical Level : Intermediate

Summary
This article will teach you how you can copy colors from a website to your presentation.  It's important to brand your presentations and getting the colors right is a great first step.
Details

You can easily take colors from a website. Perhaps you want your presentation to use the same colors as your company's Web site. Or you might simply find a color that you like on a Web site. HTML uses hexadecimal codes to define colors, but these codes are still in the familiar Red-Green-Blue (RGB) format that PowerPoint uses. All you need to do is to change the hexadecimal codes to decimal equivalents.

The shortcut is to use the eyedropper in PowerPoint 2013.  I explain how to use it here. If you're using an earlier version, you can use a colorpicker application like ColorPic. But if you want to get the colors directly from your website, here are the steps:

1. Open the web page.

2. In your browser, right-click the page and choose View>Source (or the equivalent in your browser). The HTML source code opens in Notepad.

3. From Notepadâ€™s menu, choose Edit>Find.

4. In the Find What text box, type color. Click Find Next. Youâ€™ll see a code segment that looks something like this: bgcolor="#E1EAF3". Write down the 6 characters between the quotation marks. (Bgcolor stands for â€œbackground color.â€)

5. Click Find Next again to find other instances. If you donâ€™t know HTML, you may have trouble figuring out which code relates to the color you want, so just write down all the hexadecimal codes you find.

6. Now go to http://www.dtp-aus.com/hexadeci.htm, which is a hexadecimal to decimal conversion table.

7. Look up the first two characters of your color code and write it down. In the example above, look up E1, which is 225. This is the red value.

8. Look up the second two characters and write it next to the red value, leaving a space. In our example, this is 234, the decimal value for EA. This is the green value.

9. Look up the last two characters and write it next to the green value, leaving a space. In our example, this is 243, the decimal value for F3. This is the blue value. So the result would be 225 234 243.

10. In PowerPoint, insert any shape. Right-click the shape and choose Format>Shape.

11. Click the Color drop-down list of the Fill Section and choose More Colors.

12. On the Custom tab, make sure that the Color Model is set to RGB . Then enter the red, green, and blue colors.

13. Click OK twice to return to your slide. Your shape now has the color you saw on the website. If thatâ€™s not the color you want, retry with the other hexadecimal codes you wrote down.

You can use this color as a background, to fill a shape, or to color your text.

Something is different about how preview builds are being sent via Windows Update after the RTM release in July.  Many have had to wait hours or days before being offered the build.  For some it is never offered.  There has been no clear direction as to how to resolve or troubleshoot what is wrong when a build is never offered.

You could have used the ESD file generated during your last Windows 10 build 10565 install to create a 10565 ISO to do a clean install.  I also installed it a few times by using DISM to apply the ESD file to replace the contents of my C: drive without making an ISO.  Once you get a build it can be beneficial to keep a copy of this ESD file for use when things quit working.  It is generated during Windows Update at C:\$Windows.~BT\Sources.install.esd.

What I did to get the first preview build after RTM was fire up a Hyper-V VM running Windows 10 and put it on the Fast Ring.  I had to leave it running about 12 hours, restarting several times throughout the day before the Insider Preview build was finally offered in Windows Update.  Meanwhile I could go about my day on my main Windows 10 RTM install.  Since then, I use the Hyper-V to "fetch" me the next Insider ESD file and create an ISO from it.  Usually it is offered almost immediately since my Hyper-V VM is already on the Insider builds and has not been modified.

Also, ensure that all Privacy settings are set to maximum info going to Microsoft or no build will be offered.

don cuthbert say:

It helps to have a running Windows 10, so that the System and MSR partitions don't need to change. Then take your official Windows 10 install USB and copy the install.esd you want to apply onto it.  Right at the root folder would be easiest.

Boot the computer with the normal Windows 10 USB but choose Repair your computer.  Navigate the menus to a Command prompt.

You may need to use diskpart to determine your current drive letters. I normally make the Windows partition C: and the System partition S:.  In diskpart type list volume to show the current lettering.  "Select volume" 'x' (use the number) and use "remove" to remove a drive letter and "assign letter=S" or just "assign" to assign the next available letter to a volume.

To format the Windows partition, select it in diskpart and use "format quick fs=ntfs"

The DISM command is

DISM /Apply-Image /ImageFile:E:\install.esd /Index:4 /ApplyDir:C:\  <--- assumes your USB is E:

Then run bcdboot to set the BCD to the new Windows install:

bcdboot C:\Windows /s S: /f UEFI /c  <-- for UEFI BIOS mode computer (assumes System partition is S:)

bcdboot C:\Windows /s S: /f BIOS /c  <-- Legacy BIOS mode computer

Type exit and then exit the command prompt.  You should then see a menu option to launch Windows 10 to complete the install.

It's easier to make a bootable ISO with the ESD Decrypter 4.7 script, but you have a chance to learn more using DISM and commands to build the System and boot volumes, if that is of interest.
