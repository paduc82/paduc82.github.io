john - project say:

SunshineRobinson,

You mention a master file. I assume it is a dynamic master with inserted (linked) subprojects. When you "send out" the master, do you also send the subproject files?

As a reference, in Project, linked structures such as master/subprojects or resource pool/sharer files are prone to corruption. Never rename, move, overwrite or "save off" to another location any of the files in the structure. Ideally all files should reside in a single directory on a local drive and not be operated over a network.

I have a better idea. If you need to share the file with others, first un-link all the subprojects from the master to create a static master which you can then save as to a new name and distribute as desired. To create a static master, select each subproject summary line insertion point in the master and double click to bring up the Project Information window. Click the Advanced tab and un-check the "link to project" option and hit "OK". You should get an alert asking if you want to save changes to the subproject you just un-linked. Select "No" as will preserve the link structure. Do this process for each subproject and when done you will have a snapshot in time of all the files. Save the static master under a new name and close, again, say "no" when asked if you want to save changes.

However, if all of the above is NOT what you mean by "master" file and in fact you simply have a single Project file that you call your "maser", then all you need to do to prevent changes is to select the "Tools" button at the bottom of the Save As window. Chose the General Options which will allow you to create a write reservation password. Other users will be able to view and make changes but they will not be able to save back to your original file, they can only save under a new file name.

Hope this helps.

John
