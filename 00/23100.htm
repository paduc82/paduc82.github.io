Can I make a form for each line in an Excel program? - Microsoft Community

shirleycolbert say:

Can I make a form for each line in an Excel program?

I want to make a form to print for each row in an Excel program.  Is that possible and how?

andreas killer say:

Yes, you need 2 sheets:

a) The data sheet that contains the list (one or more columns), each row is one record.

b) The template sheet for printout.

Both sheets should have the same locale names:

The data sheet usually in the 1st row (headings) in each column.

The template sheet in any cell you like.

Then you can write a macro that get all names in the data sheet, search for that names in the template sheet, if found write the value from the data sheet into the template sheet and print.

How much records are processed in the data sheet... all, just the selected (one or more rows), from to... there are many ways.

If you need further help please upload your file (maybe with anonymous data) on an online file hoster like www.dropbox.com and post the download link here.

A macro to anonymize data in selected cells can be downloaded here:
https://dl.dropboxusercontent.com/u/35239054/modAnonymize.bas

Andreas.

Forgot to post the macro, sorry.

Andreas.

Sub PrintTemplate()
  Dim DataSheet As Worksheet, TemplateSheet As Worksheet
  Dim All As Range, This As Range
  Dim FromName As Name, ToName As Name
  Dim Ok As Boolean, Ask As Boolean

  'Refer to the sheets
  Set DataSheet = ActiveSheet
  Set TemplateSheet = Worksheets("Template")

  'Get the selected records, exclude headings
  Set All = Intersect(Selection, Range("A1").CurrentRegion, DataSheet.UsedRange.Offset(1))
  If All Is Nothing Then Exit Sub

  'Ask only if we print more then one record
  Ask = All.EntireRow.Rows.Count > 1 Or All.EntireRow.Areas.Count > 1
  'Errors off, maybe a name doesn't exists
  On Error Resume Next
  'For each selected record
  For Each This In All.EntireRow.Rows
    'For each name in here
    For Each FromName In DataSheet.Names
      'Find the name in the TemplateSheet sheet
      Set ToName = Nothing
      Set ToName = TemplateSheet.Names(FromName.Name)
      'Found?
      If Not ToName Is Nothing Then
        'Write the value
        ToName.RefersToRange.Value = Intersect(FromName.RefersToRange.EntireColumn, This)
      End If
    Next
    'Print
    If Not Ok Then
      TemplateSheet.PrintPreview
      If Ask Then
        Select Case MsgBox("Continue without print preview?", vbYesNoCancel + vbQuestion, "PrintTemplate")
          Case vbYes
            Ok = True
          Case vbCancel
            Exit Sub
        End Select
      End If
    Else
      TemplateSheet.PrintOut
    End If
  Next
End Sub

russ valentine say:

You'll note I have only been talking about PST files here, not whatever you mean by "Read Only versions of my email." I said that files on storage media (CD, DVD, flash drives, thumb drives, etc.) usually become read only files while they are on storage media and thus cannot be accessed while they are on storage medium. PST files must be on a hard drive and have no read only attribute before Outlook can access them. Files on an external hard drive should be accessible by Outlook, but its usually best not to access PST files that are in temporary locations because your Outlook profile will throw error messages any time that drive is disconnected and it can't find those files.

I still suspect there ia a much longer story about the history of these files than you have revealed. I further suspect that these files were created in an older version of Outlook and that the 2GB limitaion in size is in fact very much in play. 1.93 GB is the max size any ANSI PST file can reach so the fact that these files are that size suggests they began thier existence as ANSI files and are maxed out.  If so, no amount of repairing will fix them, nor can any version of Outlook access them. The size limitation applies to the file type, not the Outlook version. So if you have an ANSI format PST file it will be toast in every version of Outlook until you reduce its size: http://www.slipstick.com/problems/pst-repair/repair-a-2gb-personal-folders-or-offline-folders-file/ 

It is impossible to predict what the IRT will do with a file. That depends on the level of corruption. Any repairable PST file will have its folder structure left intact. I would just run the IRT on backup copies of these files and see what happens.

If you can access these files in Outlook 2010, another good option would be to create a new PST file there, then copy the items you want from them into the new file. That way you know you have a new, valid UNICODE PST file as the repository for your Outlook items.

This is a problem with IE and Windows, its not an Outlook error. Typical cause is uninstalling chrome or firefox without setting IE as default. You need to close Outlook then reset your internet settings in IE's Tools, Internet Options, Advanced tab. (Or Control panel, Internet options, Advanced tab). If it occurred after uninstalling Firefox or Chrome (and any program that opens HTML files) you need to repair the registry.
