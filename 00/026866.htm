Can I sort by worksheet tabs? - Microsoft Community

mac253ex say:

Can I sort by worksheet tabs?

I have a workbook with appx 200 worksheets - each worksheet is named something (other than Sheet1). Is there a way to sort them alphabetically?   
jeeped say:
You can accomplish this with a macro that shuffles the worksheet tab order according to to an array of the worksheet names that is collected and sorted.

Tap Alt+F11 and when the VBE opens, immediately use the pull-down menus to Insert Module. Paste the following into the new pane titled something like Book1 - Module1 (Code), 

Sub mcrSortWorksheetNames()
    Dim s As String, i As Long, j As Long, vWsNames As Variant, ws As Worksheet
    On Error GoTo FallThrough
    Application.EnableEvents = False
    Application.ScreenUpdating = False
    For i = 1 To Sheets.Count
        s = s & Sheets(i).Name & "×"
    Next i
    vWsNames = Split(Left(s, Len(s) - 1), "×")
    For i = 0 To UBound(vWsNames) - 1
        For j = i + 1 To UBound(vWsNames)
            If vWsNames(i) > vWsNames(j) Then
                s = vWsNames(i)
                vWsNames(i) = vWsNames(j)
                vWsNames(j) = s
            End If
        Next j
    Next i
    For i = UBound(vWsNames) To 0 Step -1
        Sheets(vWsNames(i)).Move Before:=Sheets(1)
    Next i
FallThrough:
    Application.ScreenUpdating = True
    Application.EnableEvents = True
End Sub

Note that I've suspended event triggers and screen updating so that no Worksheet_Activate trigger is mistakenly pulled and watching 200 sheets being reorganized is only good to slow things down. Tap Alt+Q to return to your workbook then tap Alt+F8 to launch the Macros dialog and Run the macro.

This code worked in my tests but as I do not have a workbook with 200 sheets identically named as yours is, I'd recommend working on a copy of the original until you are confident that it is performing as expected.

I see the nonbreaking space in the screen photo. I've seen similar instances in documents that I received from other users, and ones created by optical character recognition of scanned pages. However, I've never seen them occur in documents that I create by typing.

They're unlikely to be created by "fat-fingered" typing because the shortcut involves both the Ctrl and Shift keys. As I mentioned before, they could come from an AutoCorrect entry; another possible source is copy/paste from other documents or from web pages.

Because I don't know what causes the NBS to appear, I can't suggest any good way to prevent it. Certainly, there's nothing in Word's spelling checker or other features that would suffice. There is a poor way: An event-handler macro could be written to examine every change in a document as it occurs, and block the change if it involves a nonbreaking space character.
