gord dibben say:

You could use this VBA code to copy all sheets from multiple workbooks into a single new workbook.  You can then make a single PDF from that workbook.

Open a new workbook, copy/paste the code into a General Module.]

NOTE: you will lose most formatting.

Sub mergebooks()
 'Merges all files in a folder to a main file.
    
    'Define variables:
    Dim numberOfFilesChosen, i As Integer
    Dim tempFileDialog As FileDialog
    Dim mainWorkbook, sourceWorkbook As Workbook
    Dim tempWorkSheet As Worksheet
    
    Set mainWorkbook = Application.ActiveWorkbook
    Set tempFileDialog = Application.FileDialog(msoFileDialogFilePicker)
    
    'Allow the user to select multiple workbooks
    tempFileDialog.AllowMultiSelect = True
    
    numberOfFilesChosen = tempFileDialog.Show
    
    'Loop through all selected workbooks
    For i = 1 To tempFileDialog.SelectedItems.Count
        
        'Open each workbook
        Workbooks.Open tempFileDialog.SelectedItems(i)
        
        Set sourceWorkbook = ActiveWorkbook
        
        'Copy each worksheet to the end of the main workbook
        For Each tempWorkSheet In sourceWorkbook.Worksheets
            tempWorkSheet.Copy after:=mainWorkbook.Sheets(mainWorkbook.Worksheets.Count)
        Next tempWorkSheet
        
        'Close the source workbook
        sourceWorkbook.Close
    Next i
    
End Sub

Solution

The issue that is Windows Defender's Real-time protection is blocking an application from being able to read a file until it has read the file. It should, of course, be scanning in parallel - not synchronously delaying file access.

Fortunately, Windows Defender has the capability to skip synchronous checking of files based on a file extension - called Exclusions:

What you can do is under Windows Defender, click Add an exclusion. Under File Types, click Exclude a file extension. Individually add the extensions:

  • pas - Delphi Source File
  • dfm - Delphi Form
  • dpr - Delphi Project
  • dproj - Delphi Project File
  • dcu - DCU file
  • cs - C# source file
  • txt - Text File
  • pst - Outlook mailbox 
  • docx - Word document
  • xlsx - Excel spreadsheet
  • gif - Graphics Interface Format image
  • jpg - Joint Photographic Experts Group image
  • bmp - Bitmap
  • res - Compiled resource
  • bpl - Borland Package Library (a dll with a different extension)
  • dll - Dynamiclly Linked Library
  • exe - Application
  • com - Application
  • bat - Batch File
  • cmd - Command file
  • scr - Screen saver
  • cpl - Control Panel Applet
  • et cetera

This way you get all the security of real-time protection, with the blazing speed of unimpeded native file access performance.

This change improved my build times:

  • Before: 86.2 s
  • After: 4.7 s 

A 94.5% improvement!

trevor rabey say:

I think everything that you are asking for is already built in to MSP.

Duration, work and cost are all composed of two parts, actual and remaining.

The total (duration, work or cost) is actual + remaining.

The % (duration, work or cost) is actual/total, or actual/(actual + remaining).

At first the entire project is remaining and there is zero actual, and the % is 0.

During execution, there is some actual and some remaining, and the %  is between 0 and 100.

At the end of the project there is all actual and zero remaining, and the % is 100.

The same could be said about every individual task within the project (since, in effect, every task is like a mini project embedded in an overall project.

The best way to see it, especially if you have a baseline, is the tracking Gantt view.

The place to input the actuals is the tracking table (the name is a give-away), and/or the task usage view and/or the resource usage view.

Whenever you update progress, the first thing you must do is set a status date in project information, and the next thing is to display that status date as a vertical red line on the Gantt Chart (format, gridlines).

john - project say:

Gamma,

In Project rebar, concrete, floor tile, etc. would be material resources. The 1000 ton, 300 cu meters, and 500 square meters are the quantity of each of those materials. That is the only "volume" recognized by Project.

However, sometimes you have to think outside the box. Perhaps this approach will work for you. Let's assume that the "volume" can be described as  a quantity of work and that we will use Project's resource base rate as the base cost. The first screen shot shows how to set up rebar, concrete and floor tile as work type resources with their respective base costs. I've arbitrarily set the max units of each resource at 100 to avoid running into overallocation.

In the second screen shot I've created three tasks, one each for installing rebar, pouring concrete and installing floor tile. Note that all tasks are fixed duration. The duration is the initial time span and the work is the expected volume for that time span. Ignore the fact that it will show as hours. A custom field could be created to simply replicate the Work field value without the "hrs" if desired but the initial "volume" must be entered into the Work field. Once the "volume" (work) is entered, assign the appropriate resource. Ignore the assignment level of each resource as that is not relevant to what we are trying to do. Note the additional fields added to the view (% Work Complete, Work, Actual Cost and Cost).

Now update the plan with progress data. Note that I've added a custom duration field (Duration1) so we can show the time period over which the progress applies. This value is manually entered. Do NOT attempt to enter this duration into the Actual Duration field. Actual quantities are entered in the Actual Work field. Project will then calculate the Percent Work Complete field to be exactly what you want. The progress bar on the Gantt display shows the percent work complete value over the planned duration.

If you want to show the progress bar as the period from the start to the entered duration value (i.e. Duration1), the following custom field formula can be used:

Finish1=ProjDateAdd([Start],[Duration1])

Then modify the Gantt Bar for progress (last bar style in the Bar Styles list) to go from Start to Finish1.

Hope this helps.

John
