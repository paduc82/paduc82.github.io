jay freedman say:
It is possible. You haven't said exactly what you want to know about the first and last row on each page -- the row index, or the text in the row, or something else. If all you want is the row index, you can adapt the following code.

Option Base 1

Sub TableRowData()
    ' define meaningful names to use for array's first dimension
    Const pgnum = 1
    Const startrow = 2
    Const endrow = 3
   
    Dim data() As Long  ' array to hold data
    Dim rw As Row       ' current row of table
    Dim rownum As Long  ' the index of rw in table's rows
    Dim datarow As Long      ' current value of array's second dimension
    Dim rg As Range     ' a range object for finding the page where rw starts
   
    ' initialization
    ReDim data(3, 1)
    Set rw = ActiveDocument.Tables(1).Rows(1)
    rownum = 1
    datarow = 1
   
    ' store the page number and row number for the first row of the table
    Set rg = rw.Range
    rg.Collapse wdCollapseStart
    data(pgnum, datarow) = rg.Information(wdActiveEndAdjustedPageNumber)
    data(startrow, datarow) = rownum
   
    ' Step through the remaining rows of the table.
    ' Each time the page number changes, store the preceding row as the
    ' last row on the previous page; then expand the array and store the
    ' page number and row number for the new row.
   
    ' Note: this loop could be written as For rownum = 1 to ActiveDocument.Tables(1).Rows.Count,
    ' but it would slow down as rownum increases because VBA counts the rows from 1 to rownum
    ' for each iteration. For a large number of rows, this way is faster.
    While rownum < ActiveDocument.Tables(1).Rows.Count
        Set rw = rw.Next
        rownum = rownum + 1
        Set rg = rw.Range
        rg.Collapse wdCollapseStart
        If rg.Information(wdActiveEndAdjustedPageNumber) > data(pgnum, datarow) Then
            data(endrow, datarow) = rownum - 1
           
            ReDim Preserve data(3, datarow + 1)
            datarow = datarow + 1
            data(pgnum, datarow) = rg.Information(wdActiveEndAdjustedPageNumber)
            data(startrow, datarow) = rownum
        End If
    Wend
   
    ' finish up with the last row of the table
    data(endrow, datarow) = rownum
   
    ' display the results
    Dim msg As String
    Dim i As Long
    For i = 1 To UBound(data, 2)
        msg = msg & "datarow = " & data(pgnum, i) & vbTab & "rows " & data(startrow, i) & " - " & data(endrow, i) & vbCr
    Next i
    MsgBox msg
End Sub

jay freedman say:
Hi Vijay,

I decided it was better to throw away the part of the code concerned with the data array, precisely because the data in it would be unreliable after the code sets the first Page Break Before formatting. The logic of the revised macro is explained in the big comment before the While loop in the following code.

Sub TableAdjust()
    Dim rw As Row       ' current row of table
    Dim rwTest As Row   ' row for finding previous nonempty cell
    Dim pgNum As Long   ' page number where rw is found
  
    ' initialization
    Set rw = ActiveDocument.Tables(1).Rows(1)
    pgNum = rw.Range.Information(wdActiveEndPageNumber)
    rw.Cells(1).Range.Paragraphs(1).PageBreakBefore = True
  
    ' Step through the remaining rows of the table.
    ' Each time the page number changes, if the first cell of the
    ' current row is empty, look back through the preceding rows (rwTest)
    ' to find the closest one whose first cell is not empty.
    ' If that cell has not already been marked as "Page break before",
    ' mark it and reset rw to rwTest. If it has already
    ' been marked, just continue processing.
  
    While rw.Index < ActiveDocument.Tables(1).Rows.Count
        Set rw = rw.Next
        If rw.Range.Information(wdActiveEndPageNumber) > pgNum Then
            pgNum = rw.Range.Information(wdActiveEndPageNumber)
           
            ' an empty cell contains a 2-character cell marker
            If Len(rw.Cells(1).Range.Text) < 3 Then
                Set rwTest = rw.Previous
                While Len(rwTest.Cells(1).Range.Text) < 3
                    Set rwTest = rwTest.Previous
                Wend
               
                If Not rwTest.Cells(1).Range.Paragraphs(1).PageBreakBefore Then
                    rwTest.Cells(1).Range.Paragraphs(1).PageBreakBefore = True
                    Set rw = rwTest
                    pgNum = rw.Range.Information(wdActiveEndPageNumber)
                End If
            End If
        End If
    Wend
End Sub

I've tested this successfully on some small sample tables, but I'll be interested to hear how it works on a larger one.

Looking for a formula or way of splitting one column containing both debits and credits into two columns, debits in one and credits in the other.  I use office 2007.  I also have office 2003.

Thanks

hansv mvp say:

If debit is negative and credit is positive (or vice versa):

Let's say the data are in A2:A100.

In B2, enter the formula    =IF(A2>0,A2,"")    and fill down to B100.

In C2, enter the formula    =IF(A2<0,A2,"")    and fill down to C100.

If you want to get rid of the formulas:

  • Select B2:C100.
  • Press Ctrl+C to copy.
  • Click the lower half of the Paste button on the Home tab of the ribbon, and select Paste Values from the dropdown menu.
