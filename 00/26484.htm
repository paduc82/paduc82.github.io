cbarnhorst say:

There is no indication of that.  The two features you are asking about are unrelated and neither one is going away.  The two features you are asking about are unrelated and neither one is going away.

Recovery Drive is not a backup tool.  It is...well, a recovery tool.  It functions like a recovery partition, only updatable (by creating a newer one).  When you upgrade from an earlier version of Windows to Windows 10 or upgrade from Home to Pro on a device with a recovery partition from the manufacturer, that recovery partition is not upgraded to your new version of Windows.  In fact it usually does not function anymore.  A recovery drive, on the other hand, performs the same function as a recovery partition but is relevant to your new Windows, is updatable, is portable, and survives a hard drive failure.

A recovery drive does not back up your personal files, apps, etc. so it cannot help you beyond restoring Windows, including drivers and settings.  It is like restoring to "factory condition" in the way it works.

A backup program is needed to restore your device to the exact way it was before things went south on you so that all your apps and personal files are just as they were before.  That is the function of Create a System Image in Backup and Restore (Windows 7).

It is true that Backup and Restore (Windows 7) is due to be moved from the Control Panel to Settings and "remodeled."  It's present title confuses some users into thinking the whole set of tools is for Windows 7 only, but only one tool in the group is.  Everything else is for Windows 10 so I hope MS finally renames the group to clear that up.  There will be new tools and new features added to the present ones, so stay tuned.

Would you recommend putting this new Recovery Drive on a separate drive or just a separate partition on an existing drive?

cbarnhorst say:

Using an internal drive or partition is not an option.  The recovery drive creation tool won't allow it.  A hard drive failure would render such a partition useless anyway. 

A recovery drive requires removable media:  usb (8gb or more) or DVD if the tool judges it has enough space.  Nothing else can be on the target media so using an external HDD or SSD would be wasteful.  If you have a usb stick with at least 8GB capacity practice making a recovery drive with it.  Think of a recovery drive as a portable recovery "partition" rather than an integral (to the PC) partition.  If you have a really low capacity  SSD mounted in a 2.5" external usb 3.0 caddy try that. 

I have a 64GB m2 SATA SSD that I used to use on my mobo for HDD caching but have now replaced with a 1TB m2 SSD which is now my system disk.  The 64GB device would make a very fast usb3.0-attached recovery drive but would waste over 50GB of capacity which I can better use for other things so I stick with 8GB usb3.0 thumb drives.  The portability of a recovery drive is its more important advantage over a HDD-based recovery partition.

In the later versions of Word for Windows there is a specific option in a group similar to what Daniel described which controls the persistent display of Page Breaks. Word for Mac, however, does not provide that option.

What Daniel indicated is true & Word 2015 works the same way, but there is no individual option for display of Page Break indicators in either version of Word for Mac. The same is true of Word 2008, as well as any other version I can recall.

In Draft View, natural Page Breaks are always displayed as horizontal lines. Manual Page Breaks are not indicated unless nonprinting characters are displayed. Otherwise, manual Page Breaks appear only as gaps between bodies of text.

When in Page Layout View with nonprinting characters hidden, Page Breaks are readily discernible because each 'page' is separated from the next just as though viewing separate sheets of paper. However, Word 2016 introduced the option to Hide White Space which also hides Headers & Footers as well as any empty space at the top or bottom of a page. That causes the pages to appear as a contiguous sheet, but the Page Breaks (natural or manual) still are shown as horizontal lines separating one page from the next. In that View, manual Page Breaks are indicated by a blue line when White Space is shown, but are displayed in the same way as natural Page Breaks when White Space is hidden.
