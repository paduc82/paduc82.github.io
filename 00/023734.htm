Well, you actually can pick a time for the Automatic Maintenance scan in the Security and Maintenance settings – but of course that’s a little bit facetious, because Automatic Maintenance only runs on System idle, so it never runs at the time you pick. In addition, the Automatic Maintenance scan is no longer being triggered on a daily basis like the other Windows Defender scheduled tasks. The Last Run Result is usually (0x2), indicating a failure:

That issue started with the Anniversary Update. At the same time, any user-created scan task that ran under the authority of the System (alias) account also began to fail. Since I had included instructions for using the System alias in user-created tasks – in order to force MpCmdRun.exe tasks to run in the Windows Defender UI (MSASCui.exe) – I then had to resort to using the undocumented parameters for the Defender UI itself in order to get user-created tasks to run in the Windows Defender interface. Through some quirk of fate, calling the old Defender UI to run a scan now brings the scan up in the new Windows Defender Security Center app. So we ended up with the ability to run user-created scan tasks in the Windows Defender Security Center app by calling the old Windows Defender app (MSASCui.exe) with its undocumented parameters, and of course now we’re just waiting for the next update to break that functionality.

This issue was purportedly “escalated” by forum management quite a while back, so I’m guessing that all of these issues are now either “by design”, or that they just don’t merit any action:

So to sum things up:

  • Automatic Maintenance scans are broken.
  • Scans scheduled with the “Microsoft method” were DOA, but are also broken.
  • User-created scans that run MpCmdRun.exe under the System alias are broken.
  • User-created scans that run MSASCui.exe under the System alias are broken.

The field you recall is a legacy text formfield.  They are still available in Word.  Developer tab>Controls>Legacy Forms.  However there is no "combo box" in legacy forms.

palcouk say:

An IMAP mail account on the local PC is a cached copy of what is on the server so if you delete the account in MS Outlook the data is retained on the server.

However, Contacts/Calendars/Tasks are held in folders marked this PC only, and when the mail account is deleted these are lost.

To get round this you would need to add a pst data file, and then copy data from this PC only folders to the pst data file

If the IMAP account is the only mail account you wont be able to delete it as you cannot delete the default mail account.

In which case Control Panel>Mail Applet> in the resulting window>Profiles, add a new Profile with prompt for Profile ticked than add the mail account using 'other types' setup page, you will need the pop settings

PS Why do you wish to move from IMAP to pop?
