david marcovitz say:

This is certainly possible with PowerPoint and VBA, but it is not a trivial undertaking. I'm just brainstorming here, but I think you might start with an array of picture names and then use a random function to randomize them, possibly putting the picture name into a new array in random order. I have three arrays here. deck is the original set of picture names. newOrder is an array of picture names that is in random order. used is an array of booleans to tell if the card has been placed in the newOrder yet. This code is off the top of my head so I'm sure there are errors, and you will need more stuff to set up the original deck. However, this might get you started

Sub Shuffle()

    Dim i as Long

    Dim next Pic as Long

    Dim used(1000) As Boolean 'assume you only have 1000 or less cards

    For i = 0 To numPictures - 1

        While Not done 'loop until you have a picture you haven't used yet

            nextPic = Int(Rnd*numPictures)

            If used(nextPic) Then

                done = False

            Else

                done = True

            End If

        Wend

        newOrder(i) = deck(nextPic)

        used(nextPic) = True

    Next i

End Sub

Sub Deal()

    Dim i as Long

    For i = 0 To numPictures

        theLeft = (i Mod numInRow) * 100 'This will get adjusted to place the picture in the right spot

        theTop = Int(i / numInRow) * 100 'adjust to taste

        'Put the cards in place

        With ActivePresentation.Slides("GameBoard").Shapes(newOrder(i))

            .Left = theLeft

            .Top = theTop

        End With

        'Assuming that the cards that cover the cards with pictures have the same name

        'plus "cover", put those cards in place. For example, if the cards with the cow are named

        'cow1 and cow2, the cards that cover them would be cow1cover and cow2cover

        With ActivePresentation.Slides("GameBoard").Shapes(newOrder(i) & "cover")

            .Left = theLeft

            .Top = theTop

        End With

    Next i

End Sub

There is a lot more to it than this, but this is the basic idea of shuffling the cards and dealing them out. You'll have to set up the original deck array with the names of the cards and set up the actions so that when the cover card is clicked, the right thing happens. Yes, there is a lot more to this, but it is a start.

My guess would be one of your security products.  It is not something that IE does.  What happens in safe mode?  Can you reproduce your symptom then?  If not it means that something is being loaded in normal mode that is not being loaded in safe mode.  You can use Resource Monitor to capture a list of loaded modules for each case and compare them.  Tip:  if you temporarily suppress some columns and sort the displays usefully the same way for each case you could have diagnostics which would be machine comparable, e.g. using windiff or even fc.   A better diagnostic for normal mode would be ProcMon.  E.g. then you can see modules being loaded and unloaded and you can see IE interacting with other running processes.  Unfortunately we can not use that during safe mode because ProcMon's driver will not be loaded.
