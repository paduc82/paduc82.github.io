graham mayor say:

You can change type with a macro e.g.

Dim sText As String
Dim i As Long
sText = ""
For i = 1 To ActiveDocument.ContentControls.Count
    If ActiveDocument.ContentControls(i).Type = wdContentControlText Then
        ActiveDocument.ContentControls(i).Type = wdContentControlRichText
    End If
    If i = 1 Then sText = ActiveDocument.ContentControls(i).Range
    If i = 2 Then ActiveDocument.ContentControls(i).Range = sText
Next i

You could use a cross reference to reproduce the value of the first box, but it will not automatically update, so the above macro will also reproduce the value of the first content control in the second.

> Why shouldn't this be possible?

Not being a member of the Microsoft Word development team I can only speculate, but I suspect that they

 a. thought 80% of whatever commercial objectives they had would be met without allowing it (and I would say that was probably mainly "support for server-side document generation" and "support for commonly used Sharepoint property types") 

 b. realised that to do it, they would either have to limit the kind of document elements allowed, or would have to cope with any construct that could occur within a Word document. If you do the former, you probably create additional frustration. If you do the latter, you either complicate the custom XML part model (because you have to allow a part structure to hange off a custom XML part) or you allow FLat OPC format XML to be stored in your parts, which I would guess they also preferred to avoid, particularly when the OOXML formats themselves were still being developed.

Unfortunately, the "legacy form fields" aren't really going to help you with /rich text/ stuff either, but if you want a forms application, I would agree with Graham that they are probably still the better bet. And if you happen to have Mac Word users, still the only bet, even with Word 2011 (but not such a hot choice with Word 2008).

Instead of page breaks, you will need to make section breaks. In addition, you will need to create a new header/footer for those sections where you rotate the border to be landscape. Make sure you apply formatting changes to "this section only" when changing the orientation.

Besides adding section breaks, you'll need to open the header area and click the Link to Previous button on the Header & Footer Tools | Design tab of the ribbon to turn it off.
