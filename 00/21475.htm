brentvogel say:

In B6 i have a drop down and data validation that changes the font color of the entire entry depending on the selection.

jllatham say:

I think this can be accomplished with a worksheet event processing macro.  I have provided the code for such a macro below.

I set up Data Validation in cell L7 on the sheet, and also set up a few more in L10, L13 and L16.  Note that these cells must be unlocked so that you can make changes to them when the sheet is in protected status.  I actually copied my first group from A6:L8 (to include the lock/unlock) cell, into rows below that area to create new 3-row sections quickly.

The choices for the cells in column L are Lock and Unlock - you can change those as you want as long as the word/phrase for locking the 3-row group is exactly the same within the code itself.  It's set in a Const there, as is the password for the worksheet if you are using one.

To put the code into your workbook:

Open the workbook,

Select the log sheet

Right-click on the sheet's name tab and choose [View Code] from the list

copy the code below and paste it into the code module presented to you.

Close the VBA editor.

Save the workbook as a macro enabled workbook, type .xlsm or .xlsb

Unprotect the sheet and set up a cell like L7 using Data Validation and enter the lock/unlock phrases as the acceptable data.  Copy that cell as required on down the sheet.  I would initially choose the "unlock" word/phrase in the cell - you can lock the ones you need to later.

Here is the code:

Private Sub Worksheet_Change(ByVal Target As Range)
  Const FirstCol = "A"
  Const LastCol = "K"
  Const LockUnlockCol = "L"
  Const LockPhrase = "Lock"
  'if the worksheet is protected with a password
  'enter it here, "" = no password
  Const wsPassword = ""
  Dim firstRow As Long
  Dim lastRow As Long
  Dim cellsRange As Range
  
  'make sure just 1 cell changed, that
  'the cell was in column L and that
  'No other entries other than Lock/Unlock
  'are allowed in col L, and the cell with
  'the 2-item list must be middle one of a
  '3-row group, i.e., L7 is middle of rows 6:8
  '
  If Target.Cells.Count > 1 _
   Or Target.Column <> Range(LockUnlockCol & 1).Column Then
    Exit Sub
  End If
  'looks like single cell in column L
  firstRow = Target.Row - 1
  lastRow = Target.Row + 1
  Set cellsRange = Range(FirstCol & firstRow & ":" _
   & LastCol & lastRow)
  'unprotect sheet
  ActiveSheet.Unprotect Password:=wsPassword
  If Target = LockPhrase Then
    'is "Lock", so lock them
    cellsRange.Locked = True
  Else
    'is not word "Lock" so unlock them
    cellsRange.Locked = False
  End If
  ActiveSheet.Protect Password:=wsPassword
  Set cellsRange = Nothing
End Sub

Invoke-Command -ComputerName AADConnectServer -ScriptBlock { Start-ADSyncSyncCycle }

If you need it as a bulk script, make sure it takes input from a CSV file or via the pipeline.

Some very simple and basic guidelines which will keep you safe

#1 - Always have an up-to-date reputable system-level anti-virus program running on your system. Note - there's no need to install any Outlook anti-virus addin which more often then not, cause more problems then what's claimed to prevent

#2 - Never click on a link in an email message that you're not sure about especially messages that "appear" to come from major vendors such as Microsoft, Apple, PayPal, Amazon, your financial institutions etc. No major vendor is going to ask you to "confirm your credentials" via a link in an email message

#3 - Don't open an attachment unless you're confident about it. Specifically, avoid attachments that claim to have things like "order details" for an order you never placed or "payment details" for a payment you never made etc etc. If you're unsure and you actually have an account with whatever the attachment claims it represents, log into the account normally and check your account there.

#5 - Best practice would be to never use the same email address/password combination when subscribing to any kind of account (i.e. email, financial sites as noted above etc etc).

#6 - Use 2 factor authentication for your email account where available

a. user say:

Put the .xsl in the following folder:

C:\Users\<username>\AppData\Roaming\Microsoft\Bibliography\Style

where <username> is the name of the folder that Windows has created for you, e.g.

C:\Users\myname\AppData\Roaming\Microsoft\Bibliography\Style

Then restart Word.

Putting the .xsl in the folder under Program Files or Program Files (x86) doesn't work.

If you use File Explorer to drill down through the folder structure, you may need to click View, then check "Hidden items" in the Show/Hide group, otherwise you may not be able to see the AppData folder. That's on Windows 10, at any rate.

Also, Word validates each .xsl (probably checks the XML/XSL, and also looks for the "entry points" that it needs. The latter changed a few years ago so if you have an old .xsl it probably won't appear.
