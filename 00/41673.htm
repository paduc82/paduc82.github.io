There appear to be two issues here:

a) Your data files
Many PC users believe that backing up files is for wimps. Some of them are in for a rude shock. You're probably lucky in that your files are safe even though they are inaccessible. Here is what I recommend:
  1. Buy a 2.5" hard disk in an external USB case. 250 GBytes would do nicely.
  2. Boot your machine with an Ubuntu boot CD (http://www.howtogeek.com/howto/windows-vista/use-ubuntu-live-cd-to-backup-files-from-your-dead-windows-computer/) and use its GUI to save your files to the external disk. Note that downloading the Ubuntu CD image of 720 MBytes can take a long time.
  3. Try to fix Windows (see below).
  4. When all is fixed, use the external disk to create regular (e.g. weekly) backups of your files.
  5. Make sure to keep the disk well away from your PC for most of the time.

b) Windows start-up
It appears that your installation was already damaged when you tried to shut down Windows, which is why the shutdown never completed. You can most likely fix your problem by using System Restore. Here is how it works:

  1. Keep tapping F8 during the early boot phase.
  2. Select Repair from the menu.
  3. Enter the Administrator's password when prompted. It is often blank.
  4. Select System Restore when prompted.
  5. Set Windows back to a point before this problem occurred.

frederik long say:

You can get into Repair Mode independently from whatever is on your hard disk like so:
1. Set your BIOS so that it uses the DVD drive as its primary boot device.
2. Boot the machine with your Windows 7 Repair CD.
3. Press a key if prompted to boot from the DVD.
4. Select "Safe Mode Command Prompt" when prompted.
5. Log on as Administrator if prompted.
6. Select Repair when prompted.
7. Select System Restore when prompted.
8. Set Windows back to a point before this problem occurred.
9. When finished, remove the CD before rebooting the system.
In Step 2 you could also use your Windows 7 installation DVD. The subsequent steps are similar to the ones above.

If this still fails to work then it is likely that you have a hardware problem.

I think the parent's issue is the same issue I had and fixed but I'll restate it for clarity:

After a Windows 1803 upgrade, Device Manager has an unknown device with hardware ID BTH\MS_BTHL2CAP. Bluetooth works in spite of this so the issue appears to be cosmetic.

Solution (only for Windows 1803 with unknown device BTH\MS_BTHL2CAP):

  1. Navigate to the registry key HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Services\BTHPORT\Parameters\PnpId.
  2. Delete the value BTHL2CAP.
  3. Navigate to the registry key HKEY_LOCAL_MACHINE\SYSTEM\Setup\Upgrade\Pnp\CurrentControlSet\Services\BTHPORT\Parameters\PnpId.
  4. Delete the value BTHL2CAP.
  5. Uninstall the unknown device from Device Manager.
  6. Restart Windows.

angelcarreno say: Hi,

My name is Angel. I am an Independent Advisor. Thank you for posting in Microsoft Community.

I would suggest Run the DISM Tool. The DISM tool to check the system health and will try to restore the files.

1. Click on 'Start'.
2. Enter 'Command prompt' in the Search box.
3. In the list of results, swipe down on or right-click Command prompt, and then tap or click 'Run as administrator'.
4. In the Administrator: Command Prompt window, type the following commands. Press the Enter key after each command:


DISM /Online /Cleanup-Image /CheckHealth
DISM /Online /Cleanup-Image /ScanHealth
DISM /Online /Cleanup-Image /RestoreHealth

To close the Administrator: Command Prompt window, type Exit, and then press Enter.

Note: The tool might take 15-20 minutes to finish running, so please do not cancel it.

Restart your device.

Hope the information provided is useful. If the issue persists, reply here and we will be glad to help you.

Angel.
