phil hawker say:

OK, I've answered my own question.... Here's a summary:

1) The "simple" email migration tool provided within the O365 / Exchange Online Portal (the "Office 365 Admin Center") does not support migration from IMAP mail accounts with non-encrypted password authentication. It only provides access to servers / accounts with SSL or TLS encrypted authentication.

However....

2) If you open the more complex "Exchange Admin Center" (access via the last menu item on the left under "Admin Centers" you gain access to a more capable toolset designed for migrating multiple mailboxes. In order to work with an IMAP Server only supporting non-encrypted authentication you must define a "Migration Enpoint" (buried under "  Recipients > Migration > More > Migration endpoints ") as per:

where the options now do include "None" for encryption, as described Here   and in more detail Here .    

You then have to upload one or more .CSV files specifying details for the account(s) to be migrated, and the dialogue presented when you upload a file claims (again..) that it will try to authenticate using SSL. However, the migration does actually work, suggesting that the options in the "Migration Endpoint" definition are honoured.

There are a bunch of other gotchas on the PlusNet side which make migration of a private domain far more difficult than it should be, but I'll describe them within the PlusNet Community Forum.

Overall the process isn't exactly intuitive, but I hope that Microsoft don't remove the more complex toolset as it provides the only route to getting migration to work from older environments such as PlusNet.

Hope this helps someone else....

OK, i have found a solution to sharing your omnia internet connection with your macbook OSX without the Internet Sharing program.

 

To start off, on the omnia go to Start>Settings>System>Data Connection.

From there change the setting from "Internal Data Call" to "As the modem through USB"

after that plug the Omnia into the Mac via USB.

 

On the Mac go to the navigation bar at the top of the screen, and click on the internet drop down menu (the thing that shows your Wifi connectivity status) and from the drop down menu, click "Open Network Prefrences..."

In this menu it should have a pop-up of some sort that says "New connection options" or something like that and then in the same pop-up it will say something like "SAMSUNG Technologies". In the sidebar of the Network Prefrences window, at the top you should see the SAMSUNG Technologies connection. Click on that connection and do as follows:

 

Telephone Number: #777

Account Name: (your # here)@vzw3g.com

Password: vzw

 

click on the "Advanced.." button towards the bottom of the window.

 

Vendor: Vodaphone

Model: VC701SI

   enable error correction and compression in modem.

Dial Mode: Wait for dial tone before dialing

Dialing: Tone

Sound: On

 

Click on PPP towards the top of the window.

 

The only selections that should be Check-Marked are:

1. Disconnect When User Logs Out

2. Disconnect when switching user accounts

3. Redial if busy

    Redial 1 times

    Wait 5 seconds before redialing

 

Click ok at the bottom of the window

 

Click Apply at the bottom of the window.

 

Click Connect and it should connect in a matter of seconds.

 

 

 

The one draw-back to this method is that you must delete the SAMSUNG Technologies connection after everytime or it will not connect. and when you delete the connection that obviously means you have to redo all of these steps.

 

Also, after your done using the connection on your mac, make sure to go back to Start>Settings>System>Data Connection on your Omnia and change the setting back to "Internal Data Call" to use your internet on your phone again.

tom van stiphout say:

Second, this works for me:

    Dim dbs         As DAO.Database
    Dim tdf         As DAO.TableDef
    Dim fld         As DAO.Field

    With dbs
        Set tdf = .TableDefs!Employees
        With tdf
            Set fld = .CreateField("WebsiteURL", dbMemo)
            fld.Attributes = dbHyperlinkField Or dbVariableField
            .fields.Append fld
        End With
    End With
