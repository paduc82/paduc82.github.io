amit tandon say:

Refer below image:

2 Options.

First Option, enter below formula in cell B84 & copy down - this counts number of occurrences of "Success" & "Save" in each row:

=SUM((COUNTIFS(D84:I84,{"Success","Save"})))

Formula in cell A84:

=INDEX(C84:C130,MATCH(MAX(B84:B130),B84:B130,0))

This returns the name of the Installer (column B) corresponding to the Max occurrences in each row.

Second Option: In case you want to do the above without separately calculating in column B, then enter below formula in cell A83 as an array formula (CTRL+SHIFT+ENTER):

=INDEX(C84:C130,MATCH(MAX(FREQUENCY(IF((--(D84:I130="Success")+--(D84:I130="Save")),ROW(D84:I130)),ROW(D84:I130))),FREQUENCY(IF((--(D84:I130="Success")+--(D84:I130="Save")),ROW(D84:I130)),ROW(D84:I130)),0))

The above formulas return the installer from column C corresponding to the maximum occurrences of "Success" & "Save" in each row. In case this is your requirement.

Amit Tandon

scottgem say:

First, a Yes/No field can also be Null. So you want to avoid this by setting the Default value of the field to 0 so it will always be only Yes or No. 

Second, I almost never use parameter prompt queries, and in this case they don't work. If you have a Yes/No field, what is stored is a 0 or -1. If you use a parameter prompt, the user might type in Yes or No but that would be interpreted by Access as a text string.

So what you should do is use a form with an unbound check box set to False by default. In your query, set the criteria for the Yes/No field to:

=Forms!formname!checkboxname

That will return records that are either true or false.

Finally, However, Having a Yes/No field to indicate being on loan is not the best design. You should have a separate table like this:

tblOnLoan

OnLoanID (PK Autonumber)

ItemID (FK)

BorrowerID (FK)

LoanDate

ReturnDate

To find all items on loan your query would look like this:

SELECT ItemID, BorrowerID, LoanDate

FROM tblOnLoan

WHERE ReturnDate IsNull;

Alternatively you can get those Available for Loan with a Subquery:

SELECT ItemID, Itemname 

FROM tblItems

WHERE ItemID NOT IN(SELECT ItemID FROM tblOnLoan WHERE ReturnDate Is Null);
