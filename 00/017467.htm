If you mean that the sub-reports' page header and page footer don't show, then that is the way it works.  Page headers and footers are controlled by the main report.

If you have information you want to appear on each page that the sub-report appears on there is a trick you can use to siimulate a page header, but I don't know of a method to simulate the footer.

In the subreport

Add a top-level grouping level and set a group header

Set its field/expression to =1

In design view of the report select the new group header and set its repeat section property to true

Copy the elements from the report page header into the new group header

Cumulative DSum - Microsoft Community

jrrock26 say:

Cumulative DSum

I'm having issues trying to get a percentage to add up for a pareto chart. What I have works fine, but anytime there is 2 of the same values it does not add up.

Ex: Defects                Quantity                 Cum%

      A                           5                            .80

      B                           2                            1

      C                           2                            1

I would need for duplicate values to be more of a running sum.

Here is the field for the query.

CumPct: DSum("[SumOfQuantity]","tblPareto Defects","[SumOfQuantity]>="& [SumOfQuantityTotal] & "")/DSum("[SumOfQuantity]","tblPareto Defects")

Please let me know if there is something else I can add into this to make the % add up.

Thank you,

vanderghast say:

If there is a duplicated values, you need to have a field which tells which record goes first in the running sum. Can be a datetime stamp, or a primary key kind of incrementing autonumber. In this last case, try:

DSum( "Quantity",  "table",  "Defect < """  & Defect  &  """ OR  ( Defect = """  &  Defect  &  """  AND pk <= " & pk  ) )"

 

which means sum every quantity for defect less than the actual one, OR  for defect which are the same as the actual one, but where their primary key value is less or equal to the actual one. I used two double quote around the actual defect to be sure it is embedded as a string constant, in the resulting string send as third argument of DSum.

ken sheridan say:

You can also do it with a JOIN to compute the cumulative balances, dividing this by the return value of a subquery to get the percentages.  The following is an example which does this with currency transactions, bringing the TransactionID primary key into play to distinguish betwen transactions on the same date:

SELECT T1.TransactionDate, T1.TransactionAmount,
SUM(T2.TransactionAmount) AS CumBalance,
FORMAT(SUM(T2.TransactionAmount) /
   (SELECT SUM(TransactionAmount)
    FROM Transactions),"PerCent") AS CumPct
FROM Transactions AS T1 INNER JOIN Transactions AS T2
ON ( T2.TransactionID <= T1.TransactionID
OR T2.TransactionDate <> T1.TransactionDate)
AND (T2.TransactionDate  <=  T1.TransactionDate)
GROUP BY T1.TransactionDate, T1.TransactionAmount, T1.TransactionID;
