scottgem say:
OneDrive is not suitable for sharing like that. OneDrive is a synchronized location. So what happens is each user has a local copy of the file on their PC. Whenever the file is changed the local copy is uploaded to OneDrive, Then the new file is downloaded from OneDrive. That is what is causing the conflicts.

If the users cannot be on the same Local Area network, then you need to use a different method for sharing. You might be able to store the back in Azure and link to it from the Access front end. There are other possibilities, but that's the best if you need simultaneous access.

Check your Junk folder

1. Click on Junk on the left pane.

Check your Blocked sender list

1. From the Outlook.com options screen under “Preventing junk email", click Safe and blocked senders.

2. Click Blocked senders.

1. From the Outlook.com options screens under Preventing junk email, click Safe and blocked senders.

2. Click Safe Senders.

3. In the text box, type in the email address or domain you’d like to always receive email from, and then click Add to list >>.
