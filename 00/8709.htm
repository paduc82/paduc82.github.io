suzanne s. barnhill say:

The template/document you are using has line numbering in a text box anchored to the header, so it is consistent for all pages. If your paragraph style has some Spacing Before or uses Multiple or Exactly line spacing, then there may be inconsistencies (Multiple line spacing adds the extra space below the line, Exactly above it, and Spacing Before is suppressed at the top of every page but the first). Or, if you have "Different first page" enabled, then it may be that the text box anchored to the primary Header is in a slightly different location from the one anchored to the First Page Header.

It is recommended that you instead  use Word's built-in line numbering feature (accessed from the Layout tab of Page Setup).

suzanne s. barnhill say:

I do understand the exigencies of court requirements, and I understand that that's why legal types keep using the old Pleading Wizard and Pleading Template. I think Different First Page might at least help solve the problem you're seeing (by allowing you to move the text box).

As for my comments about Exactly line spacing, the issue is not that it adds space arbitrarily but that the space that is added is above the line. You can see this very easily with a simple demonstration:

  1. Create several paragraphs of single-spaced text (I use a large block of lorem ipsum text for this purpose).
  2. Select (or click in) one of the paragraphs and press Ctrl+2 to double-space it. This is "Multiple" spacing, and if you select the paragraph, you'll see that the selection block extends below the paragraph.
  3. In another paragraph, right-click and choose Paragraph and set the line spacing to Exactly 24 points (my preference also for "double" spacing). When you select this paragraph, you'll see that the extra spacing is above the paragraph.

There's a (poorly worded) Compatibility Option that you can use to change #3: "Don't center 'exact line height' lines." This is misleading because such paragraphs aren't centered to begin with--that is, the extra space is not evenly divided above and below the line--but if you enable the option, the space swaps to below the paragraph.

Since you're using Exactly line spacing, just be aware that, while Word suppresses Spacing Before at the top of a page, it does not suppress the extra space created by Exactly line spacing, so you may want to enable the Compatibilty Option.

Unfortunately, none of this is going to guarantee that text aligns with the line numbers if you're introducing tables, single-spaced text, etc.
