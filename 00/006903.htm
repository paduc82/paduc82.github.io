vivianleung say:

How to use a query to calculate number of months of multiple date ranges

Greetings Everyone,

DateBegin - Date an enrollee begins the program

Our program director/coaches need to know number of months an enrollee is into a program (less the paused dates).  And number of months it takes an enrollee to complete a program.  So an example would be an enrollee ....

Paused the program on 2/1/2015. 

Restarts the program on 3/1/2015.  And

Ends the program on 9/1/2015

Total number of months = (Paused Date 2/1/2015 - Begin Date 1/1/2015) + (End Date 9/1/2015 - Restart Dates 3/1/2015) = 8

Unfortuately, I'm an access end user and I'm not very good on date functions.  I have been struggling to apply the above date formula to my query and it is just killing me.  Your help is tremendously appreciated.  Many many thanks

ken sheridan say: Rather than encoding data as column headings(which violates the Information Rule - Codd's Rule #1), if you model an enrolee's participation in a program correctly with a table like this:

ProgramParticipation
....ProgramID  (FK)
....EnroleeID  (FK)
....DateFrom
....DateTo

then you do not need to record pauses as these are represented by the durations between The DateTo and DateFrom values of rows in the table which represent successive unbroken attendances per enrolee per program.  The total period of participation is the sum of the DateTo-DateFrom values per enrolee per program, which can easily be returned in a query, calling the following function to return a duration in dd:hh:nn format:

Public Function TimeElapsed(dtmTime As Date, strMinSec As String, _
            Optional blnShowdays As Boolean = False) As String

    ' Returns a date/time value as a duration
    ' in hours etc or days:hours etc if optional
    ' blnShowDays argument is True.
    ' Time format is determined by strMinSec argument,
    ' e.g. "nn" to show hours:minutes,
    ' "nn:ss" to show hours:minutes:seconds,
    ' "" to show hours only
    ' Call the function, in a query for example, like this:
    ' SELECT EmployeeID,
    ' TimeElapsed(SUM(TimeOut-TimeIn),"nn") AS TotalTime
    ' FROM TimeLog
    ' GROUP BY EmployeeID;
    
    Dim lngDays As Long
    Dim strDays As String
    Dim strHours As String
    
    ' get whole days
    lngDays = Int(dtmTime)
    strDays = CStr(lngDays)
    ' get hours
    strHours = Format(dtmTime, "hh")
    
    If blnShowdays Then
        TimeElapsed = lngDays & ":" & strHours & Format(dtmTime, ":" & strMinSec)
    Else
        TimeElapsed = Format((Val(strDays) * 24) + Val(strHours), "00") & _
            Format(dtmTime, ":" & strMinSec)
    End If
    
    ' remove trailing colon if necessary
    If Right(TimeElapsed, 1) = ":" Then
        TimeElapsed = Left(TimeElapsed, Len(TimeElapsed) - 1)
    End If
    
End Function

The example given in the function's comment lines is directly analogous to your requirements.

You say in your post that you want the value to be returned as months, but months, being of varying durations, give an imprecise indication of a duration.  You could if you wished return a value in whole weeks by using integer division, with:

 Val(TimeElapsed(SUM(NZ(DateTo,DATE())-DateFrom),"nn"))\7
 
Note the use of the NZ function on the basis that if the DateTo column is Null this implies the enrolee is currently partcipating in the program, so the current date is assumed.
