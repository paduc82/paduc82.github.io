davidf2 say:

1. There is no way to change the thumbnail size within the clipboard pane, from within Publisher.

You might be able to change your overall computer display settings and impact the size of the thumbnails. If you have your display at 96dpi, then try increasing it to 120dpi, or higher, which will make screen items larger and may make your thumbnails larger. This would be a global change to everything on your screen in every program.

2. You cannot adjust how the clipboard works.  You cannot delete just one clip. If you empty the clipboard, you will free up computer resources and may find it easier to paste a picture.

One Publisher adjustment might help. Go to View tab > check Graphics Manager > look at the bottom of the Graphics Manager pane and click Change Picture display... > make sure the option Fast resize and zoom is checked. Note, be thankful that you still have Pub 2010 as MSFT removed this option as of Pub 2013 and Publisher has had increasing problems with high res, high file size pictures ever since.

Where are these pictures or clips coming from? Are you copying pictures from the internet via a browser? Are you copying from other Pub files, from Word files. What resolution and file size are these pictures? Explain why you are loading up the clipboard this way?

If you are working on a 120 page publication with lots of pictures, you are already using a lot of resources. Add 24 pictures to the clipboard you increase the resource demands even more.

Consider changing your workflow.

Copy and pasting pictures directly from the internet will almost always cause a lag. Instead of pasting pictures into your Publisher page, save the pictures to a folder on your hard drive and insert the picture into the page. This will use less resources, you will have copies of images if you need to return to the original image, and the thumbnail view of images via Windows Explorer are larger than clipboard thumbnails.

Consider optimizing your pictures in a third party image editor before inserting. If you are inserting high resolution, large file size images, then you can choke Publisher.

Assuming that you DON'T have a 1 for 1 list available, but are needing a word for word list like

W   West

Rd  Road

E    East

NE  Northeast

Ave  Avenue

Then I think a combination of a table like that with a User Defined Function would do the job for you.

To put the code below (the UDF) into a workbook, just copy it and paste it into a Regular Code Module:  http://www.contextures.com/xlvba01.html#Regular

Save the workbook as  a macro enabled workbook, type .xlsm or .xlsx

To use it, do just as you would any built in function in Excel.  For example, in B2 you could put this formula:

=expandedtext(A2)

Change the 1st three Const values to match the way your workbook is set up (sheet name of the sheet with the table on it, column ID of the abbreviations and column ID of the full words)

Function ExpandedText(textToExpand As Variant) As String
'this allows you to set up your table of abbreviations
'and full meanings on any sheet in the workbook
'you can make the list as long as you want
  Const listSheetName = "Sheet1"  ' change as required
  Const AbbreviationCol = "E"     ' change as required
  Const FullWordCol = "F"         ' change as required
 
  Dim listWS As Worksheet
  Dim AbbrList As Range
 
  Dim originalText As String
  Dim splitText As Variant
  Dim newText As String
  Dim foundEntry As Range
  Dim LC As Integer
 
  originalText = textToExpand
  Set listWS = ThisWorkbook.Worksheets(listSheetName)

  Set AbbrList = listWS.Range(AbbreviationCol & "1:" & _
   AbbreviationCol & listWS.Range(AbbreviationCol & Rows.Count).End(xlUp).Row)
  splitText = Split(originalText, " ")
  For LC = LBound(splitText) To UBound(splitText)
    Set foundEntry = AbbrList.Find(what:=splitText(LC), LookIn:=xlValues, lookat:=xlWhole, MatchCase:=False)
    If Not foundEntry Is Nothing Then
      newText = newText & " " & listWS.Range(FullWordCol & foundEntry.Row)
    Else
      newText = newText & " " & splitText(LC)
    End If
  Next
  ExpandedText = newText
End Function
