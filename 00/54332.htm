anannya podder say:

Hi,

 

 

When a computer enters hibernation, a snapshot of all computer settings and memory contents of a computer is taken, the information is saved to the local hard disk, and then the computer is turned off. When you restart the computer, all the settings and memory contents are restored to their original state. Unlike the standby state, the computer does not require power to maintain itself when it enters hibernation. All documents and programs are restored to their original state when the computer comes out of hibernation. Hibernation or sleep for that matter is for short term use. It is okay to hibernate or sleep if you are going to be away from your PC for half an hour or so. If you will be away longer than that I suggest you shut down completely.

 

For the third question if you set the computer to sleep after six hours it will remain in sleep mode until you wake it up from the sleep mode. At a time you can only set the computer to either sleep or hibernate.

hairyfool say:

There is no reason not to have the battery fitted whilst on mains even if the battery is fully charged. Modern battery management will prevent overcharging of Lithium-Ion batteries used in modern laptops.

 

The main issues used to occur with NiCd batteries due to their memory effects. Li batteries do not suffer from memory effects in the 1st place. However they do require more complex management so have more complex control which in turn also prevents overcharging.

 

Leave your battery fitted at all times, if really concerned disconnect the laptop ufrom AC supply once a week and allow it to discharge for a time, about down to 80% would be sufficient. Unlike NiCd there is no benefit in fully discharging batteries and in fact may be detrimental to its overall life.

 

No matter how carefully you look after your battery it has a finite life but can survive some time. I have a 5 year old Toshiba desktop replacement laptop that has always been on AC when on the desk and still has about 75% of its original runtime. About the only noticeable evidence of its age is it drops off quickly when at the end.

frederik long say:

It is my own script (see below). You need to save it to a folder on the local machine, e.g. as C:\Tools\DeleteOldFiles.vbs, then invoke it like so with your batch file:

cscript.exe //nologo  c:\Tools\DeleteOldFiles.vbs  "d:\My Pictures\Sep*.jpg"  4  /Test

This command would delete all matching files older than 4 days. The /Test switch is optional. It lets you see what would be deleted.

'-----------------------------------------------------------------------
'This program will delete all matching files that are older than x days.
'In test mode it shows the files that would be deleted.
'Syntax: cscript  //NoLogo  DeleteOldFiles.vbs  FileSpec  x  [/test]
'        they are older than the specified number of days.
'3 October 2018 FNL
'-----------------------------------------------------------------------
Dim oFSO, oWshShell, bTest
ProcessParms

'Create a list of qualifying files, using the "dir" console command
sFileSpec = WScript.Arguments(0)
Cutoff    = CLng(WScript.Arguments(1))
sTempFile = oWshShell.ExpandEnvironmentStrings("%temp%\Files.txt")
oWshShell.Run "cmd.exe /c dir /b """ & sFileSpec & """ 1> """ & sTempFile & """ 2>Nul", 0, True
If oFSO.GetFile(sTempFile).Size = 0 Then ErrorExit "No matching files found to delete."

'Delete those files that are too old.
iCount = 0
If oFSO.FolderExists(sFileSpec) _
Then sFolder = sFileSpec _
Else sFolder = oFSO.GetParentFolderName(sFileSpec)
If sFolder <> "" Then oWshShell.CurrentDirectory = sFolder

Set oList = oFSO.OpenTextFile(sTempFile)
While Not oList.AtEndOfStream
    sLine = oList.ReadLine
    If DateDiff("d", oFSO.GetFile(sLine).DateLastModified, Now()) > Cutoff Then
        WScript.Echo "Deleting """ & sLine & """"
        If Not bTest Then oFSO.DeleteFile(sLine), true
        iCount = iCount + 1
    End If
Wend
oList.Close
If iCount = 1 then sPlural = " was" Else sPlural = "s were"
If bTest _
Then WScript.Echo iCount, "file(s) would have been deleted" _
Else WScript.Echo iCount, "file" & sPlural & " deleted"

'-----------------------------------
'Process the command line parameters
'-----------------------------------
Sub ProcessParms
    Set oFSO = CreateObject("Scripting.FileSystemObject")
    Set oWshShell = WScript.CreateObject("Wscript.Shell")
    sSyntax = "Command syntax:" & vbLf _
        & "wscript.exe DeleteOldFiles FileSpec x [/Test]" & vbLf & vbLf _
        & "All matching will be deleted if they are older than x days."
    If WScript.Arguments.Count < 2 Then ErrorExit(sSyntax)

    bTest = False
    If WScript.Arguments.Count > 2 Then
        bTest = (LCase(WScript.Arguments(2)) = "/test")
        If Not bTest Then ErrorExit(sSyntax)
    End If
End Sub

'--------------------------
'Exit with an error message
'--------------------------
Sub ErrorExit (sMsg)
    MsgBox sText & sMsg, 48, "DeleteOldFiles Script"
    WScript.Quit 9
End Sub
