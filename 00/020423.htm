theothermike say:

access/barcoding

Is an inventory database created in access 2003 compatible with any barcoding systems.

boblarson say:

Most likely it is compatible with most of them.  Most of the barcoding systems, at least that I'm aware of, use a keyboard emulation pass through or an activeX control.  If it uses keyboard emulation, usually the scanner would be added like a keyboard (it used to use a PS/2 connection but now probably mostly USB).  Then when the scanner sends its information it does so as if it was typing the information.  So you would have your control set to the appropriate barcode font and anything else that would display it would also use that same font.

You will want to check out the documentation for the scanner or the scanner's manufacturer for information on how to use it with other programs.  Sometimes they have sample programs that show how it is used.

The same goes for a lot of the credit card swipe machines.

Bob Larson, Former Access MVP (2008-2010)

scottgem say:
Is an inventory database created in access 2003 compatible with any barcoding systems.

The way barcoding generally works, the barcode reader/scanner is used as an alternate input device. In most cases the reader sends the value of the barcode to the computer as text. So you just need to read from the file that the reader stores on or have the reader "type" the value into an Access text box.

The WASP system advertises compatibility with Access, but most systems will be. You just have to add code to the Access system to interface with the reader/scanner or printers.
