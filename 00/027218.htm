sheriaogletree say:

mdb file

dirk goldgar say:

Then you can't create a calculated field in the table design.  That feature only became available with the .accdb file format.

What you can do is create a query of your table calculates the desired value for every record, and base all forms, reports, etc., that need the calculated field, on that query.  This is in most respects a superior way of doing it, anyway.  Since you can't index a calculated field in an Access table, there's really no benefit to using calculated fields in tables, as opposed to queries.

Probably with the increasing quantity of SPAM and other irrelevant and potentially harmful email messages, various aspects of protocols have been 'tightened-up' in an attempt to stem the tide.  My company changed its domain name some years ago, much to my disgruntle, but I insisted on using my old-domain 'handle' as a return address, although I did reconfigure my mail client for the changes.
