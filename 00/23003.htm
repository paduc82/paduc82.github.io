One way to do this is to import your photo gallery into the page using an Iframe.

Publish your photo gallery to a folder on your local computer called 'webalbum'. Include the index.html file and all the supporting grahphics and files that are produced by Web Albun Generator. Then upload that 'webalbum' folder to your webhost root directory so that it is at the same level as your Publisher 'index_files' folder.

Then on your 'photo and video gallery' page of your Publisher file use the Insert > HTML code fragment feature to insert the following code snippet:

<IFRAME src="http://www.magixautoworx.com/webalbum/index.html" width="640" height="800" hspace="0" vspace="0" frameborder="0" scrolling="auto"></IFRAME>

If you want to create additional photogalleries then just create new subfolders on your webhost to contain them and link to them accordingly.

cliffgu_msft5984 say:

Hi Bradley,

From your description, I try to summaries your requirements.
1.You want to hide some sheets in the workbook to prevent certain users from viewing the entire workbook in Excel Online.
2. You want certain users to only edit specific cells in Excel Online.
3. You want visitors who can access the shared site collection but you don’t want them to see the workbook.

If your requirements are what I said above, you can find the answers below.
You can hide the sheet and if someone wants to unhide it, he is required to set a password. Please follow the steps below.
a. Open your workbook, and hold Alt + F11 keys to open the Microsoft Visual Basic for Applications window.
b. In the Microsoft Visual Basic for Applications window, click View > Project Explorer and Properties Window to display their panes.
c. In the Project-VBAProject pane, choose the worksheet that you want to hide, and in the Properties pane, click the drop down list from the Visible section to select 2-xlSheetVeryHidden option.
d. After making the worksheet veryhidden, you can set a password to protect it. In the Microsoft Visual Basic for Applications window, click Insert > Module to open an empty module, and then click Tools > VBAProject Properties.
e. In the popped out VBAProject-Project Properties dialog box, click Protection tab, and then check Lock project for viewing box, finally, enter and confirm the password in the Password to view project properties section.
f. Click OK button to exit this dialog, and close the Microsoft Visual Basic for Applications window.
g. Save the workbook as Excel Macro-Enabled Workbook format.
Then if the user clicks Alt + F11 to unhide the sheet, he will be required to enter a password. When you upload the workbook to the shared site collection, since the VeryHidden function is not available in Excel Online, users will not unhide the hidden sheet both in Excel and Excel Online.
So the requirement 1 can be achievable.
2. Since the Allow Users to Edit Ranges feature can’t take effect in Excel Online even though you do the settings in the workbook in advance. So users still need to download it to edit the specific cells in Excel. But if you do the settings in the workbook, the users won’t do some editing in Excel Online although you grant them the Edit permission when sharing the workbook. So only editing specific cells in Excel Online can’t be achievable. To your concern, I suggest you submit a feedback on our UserVoice because many new features in SharePoint Online come from users’ feedback.
3. The requirement 3 can be achievable via granting a visitor the View permission when sharing the site collection and not granting him any permission in the specific workbook.
Here are the steps.
a. Share the visitor with the View permission first.
b. Select the specific workbook, click the File ribbon and click Share.
c. Click Shared with and Advanced.
d. Click Check Permission and check the visitor’s permissions.
e. Delete the permissions of the visitor.

If something is misunderstood, welcome to post back and we’ll continue to provide further help.

Thanks,
Cliff

ruwim.b say:

The new Microsoft Skype for Windows 10 (Skype UWP) application does not have the option to delete the entire chat history. The only what you can do is to go the "hard way" and delete the main.db file where the local chat history is saved.

 

The location of this main.db file is:

 

%localappdata%\Packages\Microsoft.SkypeApp_kzf8qxf38zg5c\LocalState\<your skype name>\

 

Note that you must be signed out of Skype UWP, so the application is completely shut down.

 

However, after the application is restarted, the latest about 30 days chat history will again be transferred from Skype servers to the new main.db file.
