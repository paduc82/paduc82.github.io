You are asking about CRM functions. Outlook is not CRM software. It is a Personal Information Manager. Accordingly, it will track activities on your personal Contacts, but not every action for every sender you've ever had. It has robust features for finding and filtering all senders from among your message store so you can find all relevant messages to and from

hi allison,

about your requirements, i would suggest you create a customized view for the library and grant a customized permission level to the group. however, after doing the above, the users will not able to manage "shared with" list including stop sharing since we need full control permission level to do that.

here're the steps:
1. go to this library, if this is a newly created document library, we need to upload a file and share it with others so that the shared with column can be visiable when editing a view.

2. click library > create view > standard view > type in a view name, for example only see mine, tick the check box before make this the default view > choose create a public view.

3. in the columns section, tick the boxes before created by and shared with.

4. scroll down to the filter section, choose show items only when the following is true: and configure it as the following shows:

5. till now, shared users will still able to switch to the default all documents view to see all the files, so we need to create another personal view named all: click library > create view > standard view > type in a view name all > choose create a personal view > ok.

back to the library, select the default all documents view > library > modify view > delete.

6. click library > library settings > shared with button > advanced > stop inheriting permissions > ok.

7. back to the shared with button and share this library with the specific group.

8. go to site settings > site permissions > permission levels > select the default contribute permission level > scroll down to the bottom and click copy permission level > give it a name, for example my permission >

for list permissions tick the following 3 permission levels:
add items ,view items,  view application pages.

for site permissions, don't change.

for personal permissions, untick all.

9. go to library > library settings > shared with button > advanced > select the group > edit user permissions > select the permission my permission.

meanwhile, it's appreciated if our community members will share ideas about how to achieve these requirements.
