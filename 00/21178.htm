markdalmeida say:

This might happen if there are any 3rd party apps which scan office documents.

Normally when you open an office document, a temp file will be created which keeps track of the changes in the current session. When you close the document, the changes in the temp file will be written to the actual document and then the temp file disappears.

You will be able to see these temp files if you enable hidden files and system files from folder properties.

In case 3rd party apps are accessing Office documents, these temp files will not disappear and changes will not be written to actual document.
In such case, if you copy only the actual file, and format the computer, it'll lead to such error message.

databender say:

Hi, Chris.

Those "blank" cells are probably formulas returning zero length strings (i.e. "").  Excel PivotTables count those as zeroes, which can artificially lower the Average.

I solved this Average issue with a PivotTable filter.  My zero length strings were perfectly correlated with a "Filled" status in another field.  So, I filtered out the "Filled" records, removing all of those pesky zero length strings from the Average.

If your zero length strings do not perfectly correlate with another field, then you can try a less elegant approach of building a new formula column into your PivotTable's data source, like the following:

Z2

New Filter
---------------

=IF(A2 = "", "Filter Out","Leave In")

Then, you can add New Filter as a PivotTable filter, and filter out the "Filter Out" rows.

This assumes that you do not need those "Filter Out" rows for the rest of the PivotTable.

P.S. I realize that this is an old question. I'm merely posting this solution for posterity.
