Another way to explain it is, in the summary tab you have the chapters of a book listed including the Header Name of each paragraph for that chapter.  Each sheet in the spreadsheet is a chapter in the book with a number of paragraphs.   When the user goes to the summary tab and selects the paragraph Header Name they want to see, it jumps to that sheet, and within that sheet the paragraph Header Name they selected from the Summary tab.   I'm thinking I need to assign some type of label (Header Name) to each paragraph and link that label to the summary sheet/tab.

jllatham say:

This gets a bit tedious to do manually, but basically what you want to do is give the entries on your summary tab (table of contents) a hyperlink to the sheet and cell they should go to when they are clicked on.

One way to do this is to select the cell and use the {Hyperlink} option on the [Insert] tab.  You'll see a dialog like this:

You start by choosing "Place in this Document" and then choose which sheet, and if you want to go to a cell other than A1, type the cell address in and then simply click OK.

Repeat as required for each entry -- as I said, that can get quite tedious!

There is also the HYPERLINK() function that you could use.  It's a worksheet function just like SUM() and AVERAGE().  The syntax is

=HYPERLINK("the hyperlink address","the words to display")

For a link to cell X44 on a sheet named Widgets in the same workbook to be displayed as "Pink Widget - Large"  the formula would look like

=HYPERLINK("#Widgets!X44", "Pink Widget - Large")

More typing than before, but easier than all the clicking of the other way for some people.

You can find out more about both of these methods at these pages:

http://office.microsoft.com/en-us/excel-help/hyperlink-function-HP010062412.aspx
http://www.officewriter.com/blog/2013/05/13/how-create-hyperlinks-between-cells-excel-spreadsheet

Finally there is the automated solution - I use this myself in several workbooks.  If you know the names of all of the sheets in the workbook that need to have entries put onto the summary sheet and if they are set up so that you can easily identify what entries on those sheets should appear as links to them, code can be written that automatically updates the entire summary sheet each time it is selected by the user!

It's really easy if you want links to all other sheets in the workbook (other than the summary sheet itself) and if your entries to appear are easily identifiable:  perhaps they are the only entries in one of the columns on the sheet; or maybe they "starting on row #X and then at each N rows down to the end of the sheet"; or even "in the column next to where the word "ITEM:" (or other unique phrase) appears"  in a specific column on the sheet, then code can be created to do it all for you automatically and accurately.

Sub UnhideStuff()

    Application.Calculation = xlCalculationManual

    Application.ScreenUpdating = False

    For Each oSh in Worksheets

        Osh.UsedRange.EntireRow.Hidden = False

    Next

Sub Copy2Other()
    Dim oNewSh As Worksheet
    Dim oSh As Worksheet
    Worksheets.Add
    Set oNewSh = ActiveSheet
    For Each oSh In Worksheets
        If oSh.Name <> oNewSh.Name Then
            oSh.UsedRange.Offset(1).Copy
            With oNewSh
                .Range("B" & .Rows.Count).End(xlUp).Offset(1).PasteSpecial xlPasteValuesAndNumberFormats
                Intersect(.UsedRange.EntireRow, .Range("A:A")).Value = oSh.Name
            End With
        End If
    Next
End Sub

Distribution lists usually don't contain enough information to make full contacts cards; only names and addresses.  If that's all you need, then you can save each DL as a text file, which will produce a tab-separated values file.  Edit that with Excel to remove the header rows and you can then import that into your Contacts.  There may be third-party tools that can do more generic importing into Outlook.
