VBA procedure to change query table connection path popping up Data - Microsoft Community

theoneandonlysmith say:

VBA procedure to change query table connection path popping up Data Link Properties on Refresh

I have a procedure I used with Office 2003 to change connection paths of a template Excel file to the current MDB before refreshing.
Now, with 2010, a "Data Link Properties" window pops up and stops all automation.
How do I bypass this window?

Set oxl = CreateObject("Excel.Application") oxl.Visible = True oxl.DisplayAlerts = False oxl.Workbooks.Open Template For Each ws In oxl.Workbooks(Template).Worksheets For Each lo In ws.ListObjects If lo.SourceType = 3 Then oldmdb = Mid(lo.QueryTable.Connection, _ InStr(1, lo.QueryTable.Connection, "Data Source=") + 12, _ InStr(InStr(1, lo.QueryTable.Connection, "Data Source="), _ lo.QueryTable.Connection, ";") _ - (InStr(1, lo.QueryTable.Connection, "Data Source=") + 12)) lo.QueryTable.Connection = Replace(lo.QueryTable.Connection, = Replace(lo.QueryTable.Connection, _ oldmdb, newPath & "\" & newmdb, , , 1) lo.QueryTable.SourceDataFile = Replace(lo.QueryTable.SourceDataFile, _ oldmdb, newPath & "\" & newmdb) lo.QueryTable.textfileparsetype = 2 lo.QueryTable.Refresh False --Displays False --Displays Data Link Properties window lo.QueryTable.Delete End If End If Next lo Next ws

theoneandonlysmith say: Ok... I figured it out.  Because the MDB running the code was also the same MDB containing the data, Excel had trouble sharing the connection.  I had to force the connection mode to "Share Deny None".  The following is how I did it.
                   
oldmdb = Mid(lo.QueryTable.Connection, InStr(1, lo.QueryTable.Connection, "Data Source=") + 12, InStr(InStr(1, lo.QueryTable.Connection, "Data Source="), lo.QueryTable.Connection, ";") - (InStr(1, lo.QueryTable.Connection, "Data Source=") + 12))

rfmode = Mid(lo.QueryTable.Connection, InStr(1, lo.QueryTable.Connection, ";Mode="), InStr(InStr(1, lo.QueryTable.Connection, ";Mode=") + 1, lo.QueryTable.Connection, ";") - (InStr(1, lo.QueryTable.Connection, ";Mode=")))

lo.QueryTable.Connection = Replace(Replace(lo.QueryTable.Connection, oldmdb, newloc & "\" & newmdb, , , 1), rfmode, ";Mode=Share Deny None")

lo.QueryTable.SourceDataFile = Replace(lo.QueryTable.SourceDataFile, oldmdb, newloc & "\" & newmdb)

lo.QueryTable.Refresh False
lo.QueryTable.Delete

dale howard [mvp] say:

GabiVogt --

You are applying the Critical filter as a highlight filter, which produces the exact result you are seeing.  Filters do not automatically refresh, so that causes your problem.  A better approach, in my opinion, would be to format Critical tasks in the Gantt Chart, which will automatically refresh when a task is no longer Critical.  To do this, apply the Gantt Chart view and then click the Format tab to display the Format ribbon.  In the Bar Styles section of the ribbon, select the Critical Tasks checkbox.  You will now see all Critical tasks formatted with red Gantt bars.  When a task changes from Critical to non-Critical, or vice versa, the Gantt bar color will change automatically.  Hope this helps.

I am using MS Office 2007 Professional, running on a Windows 10 PC. Previously when you go to save a document there was a "save thumbnail" option in the lower left hand corner. In Word 2007 Professional and in Word 2016 running Windows 10, the option to choose whether or not to save a thumbnail is still where it was, it is just hidden below the document name and type entry boxes below where the Properties fields display.  If the screen only shows the "File name:" and the "Save as type:" choices, then the Properties fields below these are hidden.  To change this to expand the options to see the "Properties" portion of the "Save As" screen, which includes the checkbox where you can check or uncheck the "Save Thumbnail" checkbox, use your mouse to gently hover the cursor over the top gray "bar" located just above the "File name" field until you see the "up/down" arrow which points both up and down at either end.  When you activate this icon (the up/down arrow), hold down your primary mouse button (by default, it is the left button) while the up/down arrow is showing, and drag that portion of the screen up until you basically "split" the screen in half.  This reveals the "Properties" portion of your "Save As" dialog screen, where you will see the Properties fields/choices, which are:  "Authors", "Tags", "Title", "Subject", "Manager", "Company", "Categories", and "Comments" on the left side of the screen just below the "Save as type:" entry box.  About an inch below these fields is where the checkbox is located entitled "Save Thumbnail".  If you don’t see the “Save Thumbnail” checkbox, then you have not fully expanded the Properties field.  Just use your mouse to expand the lower portion of the Properties fields by continuing to drag your mouse upwards until you do see it; it is about an inch below the “Comments” section.  To remove the "Save Thumbnail" option, simply uncheck the box to the left of "Save Thumbnail".  You can also fill in the other options in the Properties fields if you choose.  I hope this helps.

Go to Tools> AutoCorrect - Math AutoCorrect, the check the box to Use Math AutoCorrect rules outside of math regions. Without actually inserting an equation using the Equation Editor you'll then be able to type the items shown in the left list of that dialog [such as \sqrt ] which will be replaced by the corresponding item in the right list [such as √ ] when you press the space bar.

Those same shortcuts work when in an equation field as long as the Replace text as you type box remains checked.

To save an equation for future use, select the equation then click the arrow on the right edge of the π [Equation] button on the Insert tab & select the Save Equation to the Gallery command... to answer the next question :-) if you want to remove an equation you've previously saved or any of the built-in equations, right-click its listing in the Equation Gallery & choose Delete.
