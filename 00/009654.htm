A hyperlink field is a basically a memo field that stores data in 3 parts separated by # signs. In your scenario, it sounds like the import process is putting all of the data into the Display Text portion of the hyperlink field. In your scenario, you could try one of the following two options to see if you can work around the issue:

 

Option 1:

After you import the data into Access…

   1. Change the table to design view

   2. Change the datatype of the hyperlink field to memo

   3. Save the table

   4. Change the datatype from memo back to hyperlink

   5. Save the table

   6. Open the table in datasheet view

   7. Test your hyperlink field

 

Option 2:

Instead of importing the text file as ahyperlink field we update the datatype to memo. Then we change the datatype within the existing table from memo tohyperlink and this will convert the data into ahyperlink address.

ann scharpf say:

Someone at my office figured a workaround that creates something that is visually close enough to what I was looking for, so I thought I'd post it here ... in case anyone else needs to create clustered/stacked column charts.  This will not create a true cluster but you will be able to get separate stacked columns grouped together.

Basically, to do this, you need to lay out your data differently:

 

Month     Site                New           Open           Closed
Jan         Site1
              Site2
Feb         Site1
              Site2
Mar         Site1
              Site2

In the first column, the month cells are MERGED, so that you get a single cell that applies to all the rows.  In this case, I merged two cells so that the month cell aligns with both Site1 and Site2.

When you highlight the entire data table and insert the chart, Excel will automatically apply the first TWO columns as titles on the X-axis.  The month is the lower title and is centered beneath the two sites. 

I hope I wrote this clearly enough.  Works like a charm for me!
