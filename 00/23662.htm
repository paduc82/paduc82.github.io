jay freedman say:

You can try to investigate what's going on when something takes a long time to respond.

In the Windows Start menu, type Resource in the search box, which should bring up Resource Monitor as the top response. (Another way to get there is to start Task Manager, go to the Performance tab, and click the Resource Monitor button.)

In the Resource Monitor, look at the CPU tab and sort by the Average CPU column to see what programs and services are sucking up the processor. If your drive light is on constantly or almost so, look at the Disk tab and sort on the Total column to see what programs are hitting the drive. If you check the box next to one of the programs (such as winword.exe), the Disk Activity section will show what specific files have activity. In either CPU or Disk, unusually large amounts of usage could indicate how the program is misbehaving (and it might not be Word but something else that's interfering).

Another place to look is in the Event Viewer. In the left column, expand the Windows Logs folder and click on Application. In the right column, you can filter the results to show just errors and warnings. Click on an entry to see the details in the bottom panel. In the details of an error, a useful bit of information is the "Faulting module", which tells you what file (usually a DLL) caused the error. Usually, though, the problem initiated from the "Faulting application", which probably passed bad data to the module. Still, it may give an idea of what's going wrong -- for more than that, you'd need debugging information that only Microsoft has.
