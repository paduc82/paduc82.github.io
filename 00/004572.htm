Hi Phillip:

It's not weird, it's working the way it was designed to.

Select some text, then go to Tools>Language... and set the language you intend to use.

In that dialog, click the "DEFAULT" button, then say "Yes" to the prompt.

That will fix any FUTURE documents you create on that computer.  However, as Bob noted, once a document is created, its language attributes travel with it.  So for existing documents, you need to correct each document.

It is likely that the bad language specification has become added to one or more of your styles.  It would be worth checking each of your styles, beginning with the Normal style, and putting their "Language" setting right as needed.

Each time you make a change to a style, click the "Add to template" checkbox, to ensure that your changes are written back to the document's template when you save.

Microsoft designed this mechanism to provide ultimate flexibility.  Many of us could handle a lot less flexibility and a great deal more simplicity.  However, the people working in multiple languages need it to stay the way it is, so ...

Hope this helps

bob jones aka: cybertaz say:

Which set of Proofing Tools are used by Word are dependent on the Language formatting of the text being checked. This always has been the case, it isn't a matter of Preferences.

In all probability, the "random" sentences are content copied from another source where the respective language formatting had been used. Other possible causes exist, but that is the most common. [One prevalent source of errant Language is content copied from web sites.] When the simple Paste command is executed the Language attribute travels with the pasted content. Unless absolutely certain of the source, it's best to use Edit> Paste Special - Unformatted Text in order to avoid the issue. Then format the pasted content as necessary using the existing Styles in the document. NOTE: It isn't a matter of font or character set... The text very well may appear to be English, for example, but still be formatted as Chinese, Russian, Spanish, or any of several dozen other supported languages.

To correct the issue in an existing document: Select the text involved, then go to Tools> Language to select the appropriate Language. If the issue exists throughout the document but should be consistently the same Language you might prefer to first Select All, then apply the Language correction. This is a revision that must be made in each document presenting the issue because Language is specific to the text similar to Bold,Italic & other formatting attributes - it simply is not indicated visually.

The Default Language in Word pertains only to new documents created using that specific installation of the software. Once the document is started, however, it's possible to intermix any combination of supported Languages... intentionally or inadvertently.
