indianasolo say:

Ok, I answered my own question and thought I'd post to see if it helps anyone.

 

It's not completely an ActiveSync issue, but more of an Outlook / Outlook Express issue.

 

I had imported the "Sent" folder from Outlook Express into Outlook 2003, using Outlook's "Import" function.  When doing this, the email's "Sent" date is correct, but the "Received" date is set to the date on which the import was performed.  I hadn't noticed this initially, because like most people, I don't have the "Received" column shown on my "Sent" folder items (why would I?).  ActiveSync and Outlook on WM6 use the "Received" date to sort emails in the "Sent" folder (just like the "Inbox" emails).

 

The fix was to export the "Sent" emails from Outlook Express into Outlook using Outlook Express's "Export" function.  This set the "Received" date to the "Sent" date and now everything works correctly.  Don't know why it would be different going one way or the other when importing / exporting emails, but evidently it is.

 

Hope this helps someone.

john korchok say:

Here are the steps:

  1. Save the document as a macro-enabled template with a .dotm file ending.
  2. Add the FileName field to the footer.
  3. From the macOS menu, choose Tools>Macro>Macros...
  4. Change the Macros in: dropdown to Word commands.
  5. Choose FileSaveAs.
  6. With FileSaveAs displaying under MacroName, change Macros in: to the name of the template.
  7. Click on the plus + sign. The VBA editor window opens with a macro that looks like this:
    Sub FileSaveAs()
    '
    'FileSaveAs Macro
    'Saves a copy of the document in a separate file
    '
      Dialogs(wdDialogFileSaveAs).Show

    End Sub
