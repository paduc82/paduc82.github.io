ken sheridan say:

Firstly on a terminology note, relational database tables don't have cells, they have values at column positions in rows in tables.  Try to get out of a spreadsheet mindset when dealing with a database.   A table's datasheet might look superficially like a worksheet, but that's where the resemblance ends.  Databases and spreadsheets are conceptually very different indeed.

Returning the non-matching columns would be tricky, but the basic logic for returning the item numbers for the mismatched rows by means of a single query is quite simple.  What you have to do is join all three tables on the item numbers, and compare the three possible mismatches for each column.  The following is an example of a query involving just two columns, County and State, which I quickly put together by making two copies of a Cities table and making some changes to these columns in the two copies:

SELECT Cities.CityID, Cities.City
FROM Cities, Cities_1, Cities_2
WHERE Cities.CityID = Cities_1.CityID
AND Cities_1.CityID = Cities_2.CityID
AND (Cities.County <> Cities_1.County
OR Cities_1.County <> Cities_2.County
OR Cities.County <> Cities_2.County
OR Cities.State <> Cities_1.State
OR Cities_1.State <> Cities_2.State
OR Cities.State <> Cities_2.State);

In your case you'd carry this on for all 39 (gulp!) columns.  Hopefully it would not then be too complex to open.  You don't need to worry about NULL columns; if one table has a value in a column and another has a NULL at the same column position that won't be returned as a mismatch because in Boolean logic comparing a NULL with anything, even another NULL, evaluates to NULL, nether TRUE nor FALSE.

An important thing to note is how the OR operations are parenthesized to force them to evaluate independently of the AND operations.

BTW do not attempt to construct or save this query in query design view, only in SQL view.  To do so in the former would be nightmare, whereas in SQL view the logic is, as you can see from the above, very simple.

scottgem say:

Scott,

I am just starting this project and have not received all sets of data yet.  Assuming they are identical structures, how would I do this with a set of queries?

The way I would do it is to join the queries on the key field and add the PK value column from one of the tables, then add columns for each field like so:

FieldnameTest: IIF(table1.fieldname<>Table2.fieldname,1,IIF(Table2.fieldname<>Table3.fieldname,1,0))

Substitute the name of each field for Fieldname.

Next create another query based on this. Add the PK column and a column like:

Mismatch: fieldnameTest+FieldnameTest+fieldnameTest...

Add each test column from the first query. 

Next create another query based on the second query with the PK column and the Mismatch column. Set the criteria for the Mismatch column to > 0. The resulting query will list all records that have a mismatch. You can then create a 4th query that joins the first query to the third so it will show all the records that have a mismatch and indicate what field the mismatch is in.
