The main raw data sheet has a year or more of dates across Row 1 that are in sequence but not contiguous (Sundays, holidays, some Saturdays are missing). This program grabs related rows (same STO) of data and identifies the appropriate time range to analyze, which may or may not correspond with available dates on the sheet, so there are some +1/-1 adjustments on the source columns to ensure that the right date ranges are targeted. Then that date range is pulled for all of the grouped rows, and each row is classified as 'new' or 'old' and additional calculations are performed. That calculated data is then summarized in one line on the output sheet.

wkbkOutSummarySht.Range("B" & 1).Value = "x% Payout"

Following is 2 examples of looping through the fields in a table and copying data to them. The example has 2 tables with identical fields but you should get the gist of how to loop through them. Have also left an example (commented out) of finding a record in a table ready to edit.

 

Sub LoopThroughFields_1()
   
    'This example uses For Each field
   
    Dim sqlStatement As String
    Dim rs1 As Object
    Dim rs2 As Object
    Dim fld As Field
    Dim strName As String
   
    sqlStatement = "SELECT * FROM Table1;"
    Set rs1 = CurrentDb().OpenRecordset(sqlStatement)
       
    sqlStatement = "SELECT * FROM Table2;"
    Set rs2 = CurrentDb().OpenRecordset(sqlStatement)
       
    rs2.AddNew      'Add new record to destination table
   
    '***********************************************************
    'Example Code to find a record where name is in field named Field1
    'With rs1
    '    strName = "OssieMac"
    '    .FindFirst "Field1 = " & "'" & strName & "'"
    '    If .NoMatch Then
    '        MsgBox strName & " not found. Processing terminated."
    '        Exit Sub
    '     Else
    '        .Edit       'Place in edit mode if editing existing data
    '    End If
    '    MsgBox .Fields("Field1").Value
    'End With
    '***********************************************************
   
    For Each fld In rs1.Fields
        If fld.Name <> "ID" Then    'Cannot alter AutoNumber field in Destination table so skip
            With rs2
                .Fields(fld.Name) = fld.Value   'If fieldname in Table2 matches field name in Table1
            End With
        End If
    Next fld
    rs2.Update  'Write record to destination tabletable
       
    Set rs1 = Nothing
    Set rs2 = Nothing

End Sub


Sub LoopThroughFields_2()
   
    'This example used for f = 1 to fields.count
   
    Dim sqlStatement As String
    Dim rs1 As Object
    Dim rs2 As Object
    Dim f As Long
    Dim strName As String
   
    sqlStatement = "SELECT * FROM Table1;"
    Set rs1 = CurrentDb().OpenRecordset(sqlStatement)
       
    sqlStatement = "SELECT * FROM Table2;"
    Set rs2 = CurrentDb().OpenRecordset(sqlStatement)
       
    rs1.MoveLast    'Position at required record in source table
   
    rs2.AddNew      'Add new record to destination table
    'Note that field indexes start at zero
    For f = 1 To rs1.Fields.Count - 1   'Start at field(1) in lieu of zero if field(0) is AutoNumber field
        With rs2
            .Fields(f) = rs1.Fields(f)   'If fieldname in Table2 matches field name in Table1
        End With
    Next f
    rs2.Update  'Write record to destination tabletable
       
    Set rs1 = Nothing
    Set rs2 = Nothing

End Sub

a. user say:

use a relative path if you can and remove that MOTW. It doesn't make sense
to use absolute paths like these. Also, I doubt that using a percent
encoding for a local path will work in IE. Percent encoding has to be used
for HTTP URLs. It is possible that it is forbidden for file URLs, I don't
know. But IE will try to match this with the exact folder name and this
will fail.

Please direct further questions in this area to MSDN or a general HTML
writer's forum, it's really not fitting in here. Also, validate your file
at validator.w3c.org. Looks like you are using XHTML without a
declaration. IE will display this file in quirks mode. Use F12 to find out
about the mode.

in the Windows version of both Excel and Word, I use the "format painter" a lot.  Do the Android versions have a "format painter"?
hansv mvp say:

In Word or Excel for Android phones, there is no separate format painter, but you can do the following:

  • Select the content (text or cells) whose format you want to copy.
  • Click the Copy button.
  • Select the content (text or cells) you want to apply the format to.
  • Click the dropdown arrow to the right of the Paste button.
  • Select 'Paste Format' from the dropdown menu in Word, or 'Formatting' from the dropdown menu in Word.

The versions for Android tablets may have a dedicated format painter button.

le boule say:

Normally this is a software conflict type of issue but I've never heard of a situation where shutting down the computer does not terminate the scan.

 

If you totally shut down the computer, let it sit a couple of minutes and restart you are seeing the scan continue and the MSE icon is active?

 

Perplexing, eh?

 

Look in startup and uncheck Microsoft Security Essentials.  Look in task manager and shut down the MSE activity there.

 

Reboot and see if the problem is resolved.  If so, recheck MSE in the startup menu and reboot again...hopefully the MSE icon will be sitting quietly and be solid green in the system tray.

 

Then see the following which may apply to your situation if this is a new or recent install:

 

Prior Security Software?   What about a trial version of an antivirus program that was pre-installed when the computer was purchased? McAfee, Norton, Trend Micro?  If you reformatted/reinstalled the operating system did you also reinstall a trial version of a security program?

Sometimes remnants of prior security software will suddenly cause issues with your current antimalware program. 

A possible reason for this problem is that you have conflicting security software installed fully or partially on the PC.  Be sure you have removed  all other security software from the PC.

You may need to use a cleanup tool to remove your prior security software:

http://social.answers.microsoft.com/Forums/en-US/msestart/thread/407bf6da-c05d-4546-8788-0aa4c25a1f91

 

Recommend you thoroughly review Microsoft Security Essentials – Installation Checklist and Frequently Asked Questions

 

You may need to uninstall/reinstall MSE. 

You should be able to remove MSE from Control Panel.  Any problems uninstalling see: http://answers.microsoft.com/en-us/protect/forum/protect_start/uninstalling-mse/a63b8c4b-58ed-437e-8086-fa08d80725a4

Recommend you reboot the computer to ensure the uninstall is complete.    

You can download a new MSE Installation File from http://www.microsoft.com/en-us/download/details.aspx?id=5201

 

 

Keep us apprised of the outcome, please!
