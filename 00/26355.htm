Option Explicit

Public Const Mname As String = "MyPopUpMenu"

Sub DeletePopUpMenu()
    'Delete PopUp menu if it exist
    On Error Resume Next
    Application.CommandBars(Mname).Delete
    On Error GoTo 0
End Sub

Sub CreateDisplayPopUpMenu()
    'Delete PopUp menu if it exist
    Call DeletePopUpMenu

    'Create the PopUpmenu
    Call Custom_PopUpMenu_1

    'Show the PopUp menu
    On Error Resume Next
    Application.CommandBars(Mname).ShowPopup
    On Error GoTo 0
End Sub


Sub Custom_PopUpMenu_1()
    Dim MenuItem As CommandBarPopup
    'Add PopUp menu
    With Application.CommandBars.Add(Name:=Mname, Position:=msoBarPopup, _
                                     MenuBar:=False, Temporary:=True)

        'First add two buttons
        With .Controls.Add(Type:=msoControlButton)
            .Caption = "Button 1"
            .FaceId = 71
            .OnAction = "'" & ThisWorkbook.Name & "'!" & "TestMacro"
        End With

        With .Controls.Add(Type:=msoControlButton)
            .Caption = "Button 2"
            .FaceId = 72
            .OnAction = "'" & ThisWorkbook.Name & "'!" & "TestMacro"
        End With

        'Second Add menu with two buttons
        Set MenuItem = .Controls.Add(Type:=msoControlPopup)
        With MenuItem
            .Caption = "My Special Menu"

            With .Controls.Add(Type:=msoControlButton)
                .Caption = "Button 1 in menu"
                .FaceId = 71
                .OnAction = "'" & ThisWorkbook.Name & "'!" & "TestMacro"

                .FaceId = 72
                .OnAction = "'" & ThisWorkbook.Name & "'!" & "TestMacro"
            End With
        End With

       'Third add one button
        With .Controls.Add(Type:=msoControlButton)
            .Caption = "Button 3"
            .FaceId = 73
            .OnAction = "'" & ThisWorkbook.Name & "'!" & "TestMacro"
        End With

    End With
End Sub

What is your WLAN router make/model and what WLAN security mode (WEP or WPA(2)-PSK) is in use?

 

Few things to try out:

 

1. Try changing the WLAN channel on your router / access point. If there are lot's of other WLAN networks in the vicinity their channels might be overlapping with yours. Also other non WLAN 2.4GHz devices like cordless (DECT) phones or Bluetooth devices may disturb your WLAN signal quality.

 

2. If having problems connecting to "hidden" (SSID) WLAN network make sure that phone's "Network status" setting (in your WLAN IAP / connection method settings) is set to Hidden instead of Public. Also note that phone should not be in "Offline" mode when attempting to connect hidden WLAN networks. Similar "Offline" mode limitation applies to also making connections to any WLAN networks / AP's that are operating on WLAN channel 12 or 13.

 

2. Try disabling all proprietary WLAN features on your WLAN router or AP. E.g. different vendor specific "high performance" modes (like super-g, turbo-g, MIMO etc.) might be causing interoperability issues.

 

3. In case your WLAN router / AP is a pre-standard (draft) 802.11n model try disabling 802.11n features (use 802.11b/g mode instead of 802.11n). If 802.11n can't be disabled for some reason at least make sure that channel width is set to 20MHz instead of 40MHz or 20/40MHz.

 

4. Ensure that you don't have MAC filtering enabled on the router / AP. If need to have it enabled then remember to add your phone's WLAN MAC on the list of allowed addresses. Use *#62209526# code to find it out or look at the sticker under the battery.

 

5. Try different security modes at least temporarily to see whether connectivity problems are related to specific security mode or incorrectly entered security keys. Double check that the security keys have been entered correcly and especially in case of WEP in correct format (key length, hex/ascii, key index, open/shared authentication).

 

6.  Ensure that your WLAN router or access point is running latest firmware / software version.

b. parker say:

Hello Raju,


Thank you for your swift response.  I did as you suggested - removed the email account from Outlook 2010 and then re-established it.


At first this solution did not seem to resolve this issue, until I read elsewhere that having too many accounts attempting to sync too frequently at one time could create one account to begin to loop upon itself if it was unable to complete its first sync due to delays from other accounts syncing, before the next sync cycle commenced.


To resolve this I increased the interval between the times in which the accounts would sync.


I also created several different Send/Receive Groups with different email accounts in each Group and then had each Group sync at different and never overlapping intervals, such as every 7, 11 and 17 minutes.


Problem resolved.
