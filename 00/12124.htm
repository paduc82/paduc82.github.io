hansv mvp say:

Method 1:

  • Select the data for the new series.
  • Copy (Ctrl+C)
  • Select the chart.
  • Paste (Ctrl+V)

Method 2:

  • Select the data for the new series.
  • Copy (Ctrl+C)
  • Select the chart.
  • Click the lower half of the Paste button in the Home tab of the ribbon.
  • Click Paste Special.

Method 3:

  • Select the chart.
  • Activate the Design tab of the ribbon.
  • Click Select Data.
  • Click Add.
  • Enter the name for the new series in the first box or click in the first box and point to the cell containing the name.
  • Enter the formula for the values, e.g. =Sheet1!$H$2:$H$4 in the second box, or clear the second box and point to the values.
  • Click OK.

marvin ma msft say:

Hi AjayKrishnan1,

If your Exchange online mailbox is placed in Litigation Hold, all items of your mailbox will be forced for preservation, including deleted items. If you'd like to clear your mailbox, you may need to remove litigation hold. However, it may cause data loss since the deleted items could be permanently deleted once the litigation hold is removed. Therefore, we suggest you refer to the following article to export your mailbox data at first. After that, you can try to disable litigation hold and clear your mailbox. 


Export or backup email, contacts, and calendar to an Outlook .pst file

As a side note, Outlook will only export the data cached in local if you enable Cached Exchange Mode for your account. So if it's cached for only 3 months, then you'll only export the data within 3 months. To avoid this situation, you can temporarily turn off Cached Exchange Mode. For your reference, see Turn on Cached Exchange Mode.

Besides, if you haven't enabled Online Archive for your mailbox, I suggest you enable this feature for your mailbox since Online Archive mailbox has 50 GB or more storage. For your reference, see Mailbox storage limits. Please note only admins in your organization can enable this feature. So if you're the admin, you need to contact them to help you enable it.

= PREP =

Back up everything and prep an alt plan for a clean install before messing with 8.1. You can skip this, but why?

Based on some great posts, I knew I had two untried options - try the update from an ISO or do a clean install but preserve my files by generating a "Genuine Ticket" file from an activated product. The "activated product" was the key. I was trying to update from my Preview build, but that hadn't been activated for months because the updates had stopped. Instead, all of this had to be done from my actual activated product, which was my 8.1 boot.

Create an ISO of the latest Windows 10

I burned this to a DVD ("create installation media")

  • Windows 10 download page - https://www.microsoft.com/en-us/software-download/windows10
  • Media Creation Tool - http://windows.microsoft.com/en-us/windows-10/media-creation-tool-install?ocid=ms_wol_win10

Generate a Genuine Ticket in case you need a clean install

If you already are at the cannot boot phase of expiration, or you don't have an earlier version of Windows on dual boot, you may need to do this on another computer and see if it will work on your main Windows 10 computer:

  • http://betanews.com/2015/08/31/how-to-do-a-clean-install-of-windows-10-the-easy-way-no-upgrade-required/

Three clarifications:

  • Perform the steps in Windows 8.1 (NOT in Windows 10 Insider Preview - it has to be an activated product, and my preview version had expired)
  • The gatherostate.exe file is located on the Windows 10 ISO DVD you burned earlier
  • When I tried to generate the GenuineTicket.xml, it didn't appear until I'd restarted Windows 8.1. Then it was there on my desktop. I saved it to a cloud folder. So if you click and nothing happens, restart and see if the file is there now.

Because update finally worked on my 8.1 install, I did not test the above, but there are lots of happy campers on the Internet for whom the above saved their work and prevented having to wipe the computer with a fresh OS install.

Back up all of your files

I already do this daily to a remote server, or you can use a hard drive and File History:

http://www.pcworld.com/article/2974385/windows/how-to-use-windows-10s-file-history-backup-feature.html

I exited Windows 10 Preview and booted up in Windows 8.1. I clicked setup.exe on the Windows 10 ISO DVD and chose to update.

The simplest fix is to install and activate Windows 8.1 on another small partition.  You do not need to do any Windows Updates, just activate Windows 8.1 and run the gatherosstate.exe to create the XML file on that activated install.  Placing that file in the correct folder of a Windows 10 RTM install should auto-activate it after a restart.  Sometimes I have had to copy the file there twice and restart twice before Windows 10 auto-activated.

I am not exactly clear on where your 2 installs are at, but as long as your Windows 8.1 is also Pro Edition then this should activate Windows 10 Pro.  If you have Windows 8.1 and not the Pro Edition, you will have to switch to Windows 10 Home to activate.

The upgrade from Windows 8.1 to Windows 10 should have worked without requiring any product key, unless you had a different Edition of Windows 8.1.  If it was a different Edition, then using the generic product key would allow the install, but it will not activate.  The same goes for the Preview upgrade.  Because it was no longer activated, it would require a product key to proceed, but it will not result in an activated Windows 10 Pro.

There was an Insider product key that worked until October 1st to activate Pro Insider builds from 9926 to 10130, but it can no longer activate those expired builds.  I was able to activate a fresh install of 9926 Pro on a new device on October 1st (just a few hours before the expiry time) and upgrade to an activated RTM, but this no longer works, as of October 2nd.
