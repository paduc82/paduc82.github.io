mike h.. say:

Hi,


H9 contains a number (2012) but the MID function in this


=MATCH(TRIM(MID(H11,1,4)),H9,0)


Returns text so in effect you're saying match the number 2012 with text "2012" and that fails with #N/A!. You need to coerce the text 2012 into a number like this:-



=MATCH(--TRIM(MID(H11,1,4)),H9,0)

julie fka profjulie say:
  1. Most applications allow you to access files in folders on the device, either in main memory or the storage card, just so long as you don't have the files stored in folders that are embedded too deeply in the file structure.  That is, the device can find

     

  2. The folders that are queried by an application are not standard across all the applications that might be installed on a device.  Most applications can find files that are stored in main memory or the storage card, but only if they are placed within specific folders or if they are not placed too deeply within the device's file structure.  Some applications are pretty particular about where you place folders and will only find files that are placed in a "My Documents" folder.  For instance, I keep my lecture presentations on my phone, and use an application called Pocket SlideShow to view them.  When I placed these presentations in a folder on the storage card with this path:  \My Documents \classname   the software does not find any of the files and I can't open them from within the application - the files are stored too deep on the device.  When I moved the presentations to a folder I titled \My Presentations which is placed directly off the root (but on my storage card) I can now open the files from within the application.  You may need to play around with this a bit to figure out how deeply you can store files on the device if you want to find them to attach them to emails later.  Try moving a file to the My Documents folder (either in main memory or on your storage card) and you'll probably find it when you attempt to attach it to an email, then move the file to a folder that is contained within the \My Documents folder and see if you can still find it when you attempt to attach it to an email, then move the file into another folder that is contained within the \My Documents\newfolder folder and see if you can still find it when you attempt to attach it to an email.

     

  3. I am not aware of a way to automatically add or save contacts from incoming email, so I think you'll have to manually add these contacts, or reply to emails you've received.  Hopefully you'll be able to move some things around on your device to make it easier to attach files to your emails (as described in my answers above).
