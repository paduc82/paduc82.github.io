ken sheridan say: This code seems to be trying to do two things which I think would better be done separately.  First it uses a standard Windows API function call to get the current user's login name.  Then it compares that with values in a column in a UserInfo table.  I would suggest getting the current user's login name with one function, e.g. the GetUser function from the following module:

' module basGetUser
Option Compare Database
Option Explicit

Declare Function GetUserName Lib "advapi32.dll" Alias "GetUserNameA" (ByVal _
   lpBuffer As String, nSize As Long) As Long

Public Function GetUser() As String

   Dim strBuffer As String
   Dim lngSize As Long, lngRetVal As Long
   
   lngSize = 199
   strBuffer = String$(200, 0)
   
   lngRetVal = GetUserName(strBuffer, lngSize)
   
   GetUser = Left$(strBuffer, lngSize - 1)

End Function

The value in the UserInfo table can then be looked up by calling the GetUser function in another function:

Function GetHumanaUser(intColumnNumber As Integer) As Variant

    Dim strColumn As String
    Dim strCriteria As String

    strCriteria = "Account = ""HUMAD\" & GetUser() & """"

    If intColumnNumber = 1 Then
        strColumn = "ID"
    Else
        strColumn = "Name"
    End If

    GetHumanaUser = DLookup(strColumn, "UserInfo", strCriteria)

End Function

I assume that ColumnNumber is a variable or control name, whose value can be passed into the function:

    GetHumanaUser(ColumnNumber)

to determine whether the value of the ID or Name column is returned.

Note, however, that the columns in UserInfo are badly named.  The generic 'ID' as a column name gives no indication of the entity type of which this is the key; UserID would be far better.  'Name' is the name of a built in property in Access and as a 'reserved word' should not be used for object names; UserName would again be far better.

Sub SaveAs__XLSX()
'Dec28, 2016
'Application.ScreenUpdating = False
v = Split(tName, ".")
For Each wb In Application.Workbooks
If wb.Name Like v(0) & "*" Then
wbName = wb.Name

Next
