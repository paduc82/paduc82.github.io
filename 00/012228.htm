Formula to return a value based on multiple weight ranges - Microsoft Community

fishymom323 say:

Formula to return a value based on multiple weight ranges

I need write an Excel formula to calculate a charge rate based on the following criteria.  Note that CWT is equal to the total pounds divided by 100.

  • 0-1710 lbs = $7 cwt
  • 1711-3330 lbs = $6 cwt
  • 3331 and greater = $5 cwt

Example:  1500 lbs would be 1500 / 100 * $7 = $105.

The only variable tracked on the spreadsheet is the total pounds; therefore, all other variables must be contained in the formula.

I have tried to create a nested IF formula but can't seem to get the syntax right.  

Any assistance would be greatly appreciated.

a. user say:

Thank you for your response

I think where I am getting tripped up is the second condition is a "between" so that if the pounds are between 1711 and 3330 then the amount charged per cwt is $6.

How would I account for the two conditions in the second nested IF statement?

The trick is to not think of it as between, but as a set of exclusions.

Step 1, is the number less than 1711?  If yes, then *7, else go to step 2.

Step 2, is the number less than 3331?  If yes then *6, else go to step 3.

Step 3, *5.

You see, you don't need to worry about the number being less than 1711 in step 2 because in step 1 you already removed anything less than 1711 from your list, by multiplying it by 7 and ending your if statement.

I could also write the if statement the following ways:

=if(A1>3330, A1*5,if(A1>1710, A1*6, A1*7))

=if(A1>3330, A1*5, if(A1<1711, A1*7, A1*6))

=if(A1<3330, if (A1>1711, A1*6, A1*7), A1*5)

The latter was an attempt to check for your "between" first.  But if statements are not designed to worry about "between," you use two ifs for that.

The trick is to remember that excel follows each "if" rule in order, so once it has done the first task, any number affected by that if statement is not available to the later if statements.  So if you have your first question be "is A1>3330" then you no longer have to worry about your number being >3330, because you already dealt with all those customers with your first if.

nathan o. - microsoft support say:

Hi NT007, 

 

You should be able to accomplish this by specifying a range in your code via the .SourceItem command. In this command, you should be able to use syntax like "Sheet2!r1c1:r5c5", where the first part is the name of the sheet and after the exclamation point is the starting position and then the ending position. I believe the R stands for Row and C stands for Column, so this example would be to go to Sheet2 and start at row 1 column 1 and show through row 5 column 5. I have also placed some sample code below that I used when trying to test this. With that being said, I would recommend staying away from OLE objects in general, since they have the potential to cause problems when going from machine to machine. In fact, I experienced different results on my end with this sample code, depending on the machine I was on. It may be a better approach to just use the Excel object model from within VBA code and open the Excel Workbook that way, but that is a decision you would have to make.

 

Sample Code:

 

   ' Set class name.

   OLE1.Class = "Excel.Sheet"

   ' Specify type of object.

   OLE1.OLETypeAllowed = acOLELinked

   ' Specify source file.

   OLE1.SourceDoc = "<path to excel file here>"

   ' Specify data to create link to.

   OLE1.SourceItem = "Sheet2!r1c1:r5c5"

   ' Create linked object.

   OLE1.Action = acOLECreateLink

   ' Adjust control size.

   OLE1.SizeMode = acOLESizeZoom

 

 

SourceDoc Property -http://msdn.microsoft.com/en-us/library/aa196594(office.11).aspx

 

 

Hope that helps.

PAGE_FAULT_IN_NONPAGED_AREA Stop message/error occurs when requested data is not found in memory. The system generates a fault, which normally indicates that the system looks for data in the paging file. In this circumstance, however, the missing data is identified as being located within an area of memory that cannot be paged out to disk. The system faults, but cannot find, the data and is unable to recover. Faulty hardware, a buggy system service, antivirus software, and a corrupted NTFS volume can all generate this type of error.

Unplug all external devices connected to the pc and restart the pc to check if you are able to get to the desktop screen.
