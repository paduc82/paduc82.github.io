bob jones aka: cybertaz say:

Well. let's see... You're referring to 2 different devices which are related but are not fully synchronized: (1) The Styles combo box on the Formatting Toolbar & (2) the Styles Pane in the Toolbox palette.

The Styles Pane contains the List of viewing options [Recommended, Styles in Use, In Current Document, All Styles] which determines what is displayed in that Pane. Recommended is the default selection to which the program reverts each time it is launched... That cannot be changed on a permanent basis. The built-in Styles it displays is a fixed listing, but it also displays user-defined Styles which may vary depending on the active document currently being viewed.

When one of the other options is chosen it affects both the listing in the Pane as well as the listing in the combo box. However, Recommended affects only the listing in the Pane excluding Table Styles (Table Styles are not listed in the Pane regardless of the option chosen), but the combo box lists all Styles (including Table Styles) just as though All Styles is selected. Whether this is a bug I can't say. As I suggested earlier, I rarely have used the combo box so I can't remember whether it ever worked differently. However, that's academic at this point since there will be no further updates to Office 2011 - The current behavior will not be changed.

As for the order, I can understand the confusion :-) It's based on Priority ranking of 1-100 assigned to the Styles, themselves. The Styles are listed alphabetically within the assigned Priority ranking. This is carried over from the Windows versions of the program. Unfortunately, as is the case for many style management features, the ability to specify or modify those levels has not been brought to the Mac version, even in Office 2016.

bernard liengme say:

You need to nest a stream of simple IF functions since IFS is not available in XL 2013

=IF(H10="C",1,IF(H10="NA",1,IF(H10="NC",0,IF(H10="CNC",0,IF(H10="","")))))

An alternative would be: =LOOKUP(H10,{"C","CNC","NA","NC",""},{1,0,1,0,""})

Note that {"C","CNC","NA","NC",""} is in alphabetical order

And two other suggestions....

=IF(H10="C"; 1; IF(H10="NA"; 1; IF(H10="NC"; 0; IF(H10="CNC"; 0; "") ) ) )

or

=IF(OR(H10 = {"C"; "NA"} ); 1; IF(OR(H10 = {"NC"; "CNC"} ); 0; "") )

The changes:

1. Return null string (""), not space (" "). Makes it easier to test for empty or (appears) blank (like H10="").

2. Return null string if no other matches, instead of FALSE.

rajesh c. say:

In the worksheet "Company Holidays",  the way they have defined a named range "1stHolidays",

you need to similarly define separate list of holidays (named ranges for each) for each region/country that you require.

Then, based on an employee's region/country location, you can pickup one of the named range in  your formula.

The formula would remain the same for all region/country.

Just to give you an example:

suppose say that you have defined "USAHolidays", "UKHolidays", "INDHolidays".

Similarly say, you have defined an abbreviation for country as "USA", "UK", "IND" for the above and suppose say that for a record cell D2 you have "UK" as the country for a selected employee.

Also assume that you have defined a range for vlookup as

               column-R    Column-S

row-10      USA           USAHolidays

row-11      UK             UKHolidays

row-12      IND           INDHolidays

Then, taking a hypothetical case,

=NETWORKDAYS(StartDateCell, EndDateCell, vlookup(D2,$R$10:$S$12,2,0))

the vlookup function in the above formula would pull "UKHolidays" for you and then your

NETWORKDAYS formula would evaluate for UK.

Give it a try. you will be able to solve your problem.
