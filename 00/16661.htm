hansv mvp say:

You can use =D2*C2^A2.

Or you could use the GROWTH function:

 

=GROWTH($B$2:$B$30,$A$2:$A$30,A2)

 

This takes the y-values in B2:B30, the x-values in A2:A30, calculates the exponential trend and returns the predicted value for A2.

mike h.. say:

Hi,

Depending on how you insert a worksheet, the new sheet goes in after the last sheet or to the left of the active sheet and AFAIK there's no way to change that. The 2 ways of inserting a sheet are:-

1. Home tab - insert - insert sheet. The sheet is inserted after the last sheet.

2. Right click the sheet tab - insert - worksheet. The sheet is inserted to the left of the active sheet.

You can do what you want with VB code. Alt+F11 to open VB editor, right click ThisWorkBook and insert module and paste the code below in on the right. You can easily run the code by attaching a shortcut.

Developer tab - Macros - select the macro name - options and enter a shortcut.

Public Sub InsertWorksheetAfter()
    Worksheets.Add After:=ActiveSheet
End Sub

Can Outlook.com engineers bring back "ends with", "begins - Microsoft Community

_gs say:

Can Outlook.com engineers bring back "ends with", "begins with", and "does not contain" filter modifiers and also allow us to include "@" in rules?

Outlook.com's new rules for sorting new messages has become a source of some frustration.  The decision to remove the the "ends with", "begins with", "does not contain" filter modifiers have made the filtering system a lot less effective.

If I create the rule:
[Sender contains] ["@account.microsoft.com"] (with quotes) [Move to] [Inbox]

...it transforms into the following:

If sender contains: "account.microsoft.com" then move to Inbox

So under the new, transformed rule, does it mean the following Sender scenarios would be moved into my Inbox?

* "John Doe" <account.microsoft.com@fake.com>
* "John Doe" <fake@account.microsoft.com.fake.com>
* "John Doe" <fake@fake.account.microsoft.com>
* "account.microsoft.com" <*** Email address is removed for privacy ***>

-------------------------------------------------------------------------------------------------------------

[Sender contains] [@account.microsoft.com] (without quotes) [Move to] [Inbox] becomes...

If sender contains: "account" AND "microsoft" AND "com" then move to Inbox

So under the new, transformed rule, does it mean the following Sender scenarios would be moved into my Inbox?

* "John Doe" <ACCOUNT@MICROSOFT.COM>
* "John Doe" <COM@MICROSOFT.ACCOUNT.gov>
* "John Doe" <COMmander@MICROSOFTsACCOUNTingdivision.com>
* "MICROSOFTs ACCOUNTing COMmand center" <*** Email address is removed for privacy ***>

-------------------------------------------------------------------------------------------------------------
