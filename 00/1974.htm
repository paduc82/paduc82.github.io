I had three of my users get this "SWF/ Meadgive" junk yesterday.  It wasn't a false positive as some had mentioned above.  I ran a handful of cleaners and no single product was able to remove it (might be different today if they've update definitions).  I ran NPE, Malwarebytes, SpyHunter, Hitman, and a Kaspersky boot scan.  For anyone interested, I removed it manually and there were a couple places of interest you need to check and it was a little different on each machine:

First off, it appears to be specific to the user account so log off the infected user and log on with another admin account.  Then check the following places for signs of the infection:

1)  C:\Users\InfectedUser\AppData\Roaming\Microsoft\Windows\Start Menu\Programs\Startup     - looking for a .bat file and/or a shortcut to one.

2)  C:\Users\InfectedUser\AppData\Local\Temp   - looking for a random folder name, the ones I saw were 5-7 chars long and a mix of letters and numbers.  The folder should have 3 files in it:  one .bat file, one shortcut file and one other file that had an extension I hadn't seen before (sorry I can't remember what it was).  One infected machine had two folders in this location I had to delete, but they were very similar in name.

3)  Now check the task scheduler to see if there is a new item that shouldn't be there (this entry only appeared on one of my infected machines).  It was very easy to spot as the trigger on it was set to something like 'any change to the file'

4)  Registry - HKEY_USERS\Infected_User_SID\SOFTWARE\Microsoft\Windows\CurrentVersion\Run

     Registry - HKEY_LOCAL_MACHINE\SOFTWARE\Classes     (found 2 entries here on one machine, I ran a search for the file names that were in the %temp% folder I mentioned above to track them down)

     Registry - HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Services\Tcpip\Parameters - check the NameServer entry for IP addresses that shouldn't be there (only appeared on a Windows 7 machine).

5)  Last place was a browser add-on.  Find out the preferred browser of the infected user and go delete the addon, in Firefox it was named something similar to 'Google' and referenced a .dll file similar to 'wildvine.dll'

Hope that helps anyone still dealing with this and sorry I didn't have specific file/folder names to reference but I will update if anyone else shows up with the infection today.
