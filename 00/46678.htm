Windows Movie Maker 2.6 is actually a very old version of Movie Maker.  It's from Windows Vista for those without a capable graphics card able to run Windows Movie Maker 6.0 (also from Vista).  Those versions created .mswmm project files.  You cannot open a newer project file in an older version of Movie Maker.

 

The .wlmp file is a project file from the latest versions of Movie Maker.  Movie Maker 2012 is part of Windows Essentials and that is what you should be using to open that project file to continue editing.  I also assume you have all the source files still around on your computer too (the photos, video and music).  The project file needs those in order to present a preview.

Note: If you have a file with a .vep extension, that is not even a Movie Maker project.  The .vep extension is used by AVS Video Editor.  Similar to the .wlmp file, it isn't an actual movie, just the editing state within the program.  But for a .vep file, you would need to be using the AVS Video Editor, not Movie Maker.

hairyfool say:

A DHCP request is sent via the universal broadcast address 255.255.255.255 which is one of the addresses a server listens to. The request includes the MAC address of the requesting device because at that stage it does not have an IP address. The DHCP server will respond to the MAC address in order to assign the address.

In your case the 169.254 range is called the APIPA address and the PC actually assigns it to itself because it has not seen a response from the DHCP server. As you refer to Wireshark have a read of this.

http://wiki.wireshark.org/APIPA

The client cannot send the request via this address and get a response because the DHCP server will not be listening on the address or using the range. The response has to be via the MAC.
