The difference between Small Memory Dumps and Kernel-Dumps in the simplest definition is a Kernel-Dump contains much more information at the time of the crash, therefore allowing further debugging of your issue. If your upload speed permits it, and you aren't going against any strict bandwidth and/or usage caps, etc, the Kernel-Dump is the best choice. Do note that Kernel-Dumps are much larger in size due to containing much more info, which is why I mentioned upload speed, etc.

cmdrkeene say:

I'm not quite sure where to begin on the media player one, but let me tackle the first one. 

This is usually caused by faulty shell extensions.  A shell extension is a program that adds additional options to right-click menus for certain icon types. For example, after you install an antivirus program, your icon right-click choices might be updated to include a "scan this file for viruses" option.  It's also very common for ZIP/RAR file compression tools, and Notepad programs, to add themselves as additional right-click options.

If those extensions suffer problems, it can cause the interface to crash and reload when you right-click.

Solutions:

The easiest solution by far is to roll back whatever caused the issue. If you can remember installing anything recently, remove whatever it was.  If that doesn't work (or you aren't sure where to start), use option #1 here instead.

If that doens't work, it gets a little harder.  You can use a tool like Microsoft AutoRuns to see what shell extensions are registered (set to load), and disable them one-by-one until you find the one causing the right-click crash.

With the tool I mentioned, you can view them under the Explorer tab, and you'll see them listed as ShellExtensions in the list.  To test it out, just uncheck one, and it will be immediately unloaded from the system and makes testing pretty easy. As a tip, disabling unnecessary shell extensions can keep Windows PCs running quick over years as more and more software builds up and impacts bootup times and system performance.
