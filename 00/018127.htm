Hi John,

In SharePoint Online, if you do not want users to delete these attachments, the best practice is to prevent the users from editing items in the list. Users with Edit permissions are able to delete list attachments.

The files from list attachments are stored directly in SharePoint's content database and is not accessible through the browser.

To prevent users from editing the list, you can create a custom permission level that gives users the ability to create but not delete/edit list items.

To do this, you can go to Site Settings > Site Permissions > Permission levels > Add a permission level.

In here, you can select the permissions that you wish users to have. This includes list permissions where you can provide users with Add Items permission but not Edit Items.

If you are using SharePoint 2013 on-premise, as this is a SharePoint Online forum, we suggest posting your query to the SharePoint 2013 TechNet forum for further help.

gord dibben say: Function RemoveDupes(txt As String, Optional delim As String = " ") As String
    Dim x
    With CreateObject("Scripting.Dictionary")
        .CompareMode = vbTextCompare
        For Each x In Split(txt, delim)
            If Trim(x) <> "" And Not .exists(Trim(x)) Then .Add Trim(x), Nothing
        Next
        If .Count > 0 Then RemoveDupes = Join(.keys, delim)
    End With

Anti-Virus for Sureface RT - Microsoft Community

padminibissessar say:

Anti-Virus for Sureface RT

I have a virus on my surface RT but when I try to download an antivirus it says the format is not compatible with my tablet.. Which antivirus can I download?

[Original Title: antivirus]

rob koch say:

I have a virus on my surface RT but when I try to download an antivirus it says the format is not compatible with my tablet.. Which antivirus can I download?

[Original Title: antivirus]

Since no Win32 or other directly executable applications are able to run on the ARM based processor used for the Win RT based Surface, none of these Intel x86 based "viruses" are able to operate on this device.  The applications on a Surface RT all operate within a much more limited API based environment that doesn't allow direct execution, so this type of malware simply isn't possible.

The above is also why other antivirus applications from 3rd-party application vendors are unable to operate on Windows RT, since they would be limited as are all applications and so unable to gain the control they require in order to overpower and remove the malware even if it did exist.

Windows Defender on the other hand is built into the Windows RT ARM based operating system, so it has complete access and control over all user based applications, so it can easily control them and any less effective malware they might contain.

However, the confusion in the other thread referenced above is due to the fact that the Internet Explorer browser and Adobe Flash components built into the Surface RT are still able to be "hijacked" and abused by certain lesser types of adware, etc., which may display false messages.  These can redirect the web browser to other websites in order to make it appear that a virus is present or simply display a false warning that claims that this is the case.

In order to remove these messages it is necessary to either reset the Internet Explorer browser or more completely refresh or reset the entire Surface RT.
