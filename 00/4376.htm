An Outlook AddressBook is merely a specialized "view" of the contact folders designated to be included in the AB view.

The answer to your question depends on the type of email account that is involved - POP3 or Imap with the assumption being that this is not in reference to an Outlook.com account.

If the account type involved is POP3 - the contact folder would be contained in the same PST file where your Inbox was located. If the account was Imap and your contacts were located in a contact folder within the Imap .OST file associated with the account (contact folder would have been labeled <Contacts (This Computer Only)>) then your only options are going to be to recover this data from a backup OR hopefully you'll still be able to open the old profile enough to export the info. .OST files can only be opened/accessed via the email account used to create it (excluding use of 3rd party recovery tools).

karl timmermans [outlook mvp 2012-2018] say:

The generic instructions for import / export are as follows which in your case will involve opening and closing Outlook twice - once with the old profile and next with the new profile. The following also assumes you are using an Imap email account

From the old profile

1- Select the contact folder where your contacts are located (the folder you want to export <from>)

2 - From O'2013 menu --> File --> Open & Export --> Import/Export

    (this bring you to the universal import/export screen applicable to all versions of Outlook)

3 - Select <Export to a file>

        --> then this is going to be one of those rare cases where I'm going to suggest selecting "Outlook Data FIle (.pst)" as the file type

            --> the contact folder you want to import from should already be selected if you followed 
                  Step#1 - else select it here - if you have created any sub-folders under this Contacts folder,
                  - ensure that you have checked the "include sub-folders" option

                 --> browse to a location and name the output file name (remember both)

                         - suggest you select "allow duplicates to be created"

                          - click <Finish>

Rational behind the above choices

    a) this will provide a mirror image of your existing contacts with minimal loss of information

    b) it will include any Contact Groups (Distribution Lists) if any exist

Once the above is done - close Outlook and restart it using your working profile

4) Highlight the contact folder you want to import <TO>

5) From O'2013 menu --> File --> Open & Export --> Import/Export

6) Select <Import from another program or file>

7) Select Outlook Data File using the PST file created in Step #3

8) Select the Contact folder in the PST file you are importing from

9) Select "Include Sub-Folders" If applicable

10) Suggest selecting "Allow duplicates to be created" to minimize potential problems

11) Select <Import items into current folder> <IF> you have followed step #4

12) Finish

You're done

For anyone else reading this - the above instructions should not be construed as Universally applicable instructions for anyone moving from one system to another - not every scenario is the same and import/export is generally not the recommended approach to use.

I am doing some data profiling and have too many columns to just sort them alphabeticlly.

So, if my data is in a10:a999, what formula could I put in A1 to return the first item alphabetically in the range? Similarly, is there a version for last alphabetically?

Can it ignore blanks?

Will it work on a list of zip codes - where the characters are all numerpic, but the data is text?

If I grab a column, and use data/filter, Excel will display the values in the column in a pull down menu, and thos items are alphabetized. Can Iaccess whatever algorithm that is?

hansv mvp say:

For the minimum value (excluding blanks), use the following array formula, confirmed with Ctrl+Shift+Enter:

=LOOKUP(2,1/(IF(A10:A999<>"",COUNTIF(A10:A999,"<"&A10:A999))=0),A10:A999)

For the maximum value (excluding blanks), use the following array formula, confirmed with Ctrl+Shift+Enter:

=LOOKUP(2,1/(IF(A10:A999<>"",COUNTIF(A10:A999,">"&A10:A999))=0),A10:A999)

This won't work for zip codes (probably).

The algorithm used by Excel to populate dropdown lists is not available to us users.

Note: The following steps will currently only work if your previous account was @Hotmail.com or @Live.com.
 
In your browser, copy this when logged in but unable to get in -->  https://account.live.com/ManageAssocIds

      Under “Other email addresses” click the Remove link below your previous address.

Click the Remove button to confirm the message.

    You should now see a message on the page stating, “You've removed an email address from your Microsoft account.”

    Copy link --> https://account.live.com/ChangeId.aspx in your browser.
    Verify that “Create a new email address” is selected.

    In the New Microsoft account box, enter your previous address and select your previous domain (Hotmail.com or live.com).  

Note:  You must enter your previous address or the process will fail.
    Select the Save button.

Now you can use your original @hotmail.com or @live.com account to sign into your Microsoft services as you did before renaming to an Outlook.com account. 

 The @outlook.com address will now be an alias on your account from which you can still send and receive messages within Outlook or Hotmail.
