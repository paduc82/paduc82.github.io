doug robbins - mvp office apps & services (word) say:

Running a macro containing the following code will remove the paragraph break at the end of a paragraph that contains an odd number of quotation marks.

You will need to run it a second time if the paragraph that follows the first paragraph that contains an odd number of quotation marks contains two quotation marks (split quotations spread over three consecutive paragraphs)

Dim arange As Range
Selection.HomeKey wdStory
Selection.Find.ClearFormatting
With Selection.Find
    Do While .Execute(FindText:=Chr(34), Forward:=True, _
        MatchWildcards:=False, Wrap:=wdFindStop, MatchCase:=False) = True
        Set arange = Selection.Paragraphs(1).Range
        arange.Start = Selection.Start + 1
        If InStr(arange, Chr(34)) = 0 Then
            arange.Start = arange.End - 1
            arange.Delete
            Selection.Collapse
            Selection.Move wdParagraph, 1
        End If
    Loop
End With

NOTE: if &n bsp; appears in the above code, delete it after you paste it into the Visual Basic Editor

The following code will stop at each paragraph that contains an odd number of quotation marks.

Sub IdentifyParagraphsWithMismatchedQuotationMarks()
   Dim p As Paragraph

   With CreateObject("VBScript.RegExp")
      .Pattern = "^(([^""]*""){2})*[^""]*""[^""]*$"

      For Each p In ThisDocument.Paragraphs

            p.Range.Select
            Stop
         End If
      Next

   End With
End Sub

peter schleif say:
but the difficulty is that with the " being a special character it makes the regex particularly difficult to get my head around.

There is nothing special about double quotes, except that VBA-strings are also included in double quotes. Therefore double quotes in VBA-strings must be escaped by doubling.

Please replace each occurence of "" within the pattern with "a" (without the double quotes) and make the pattern case-insensitive. Like this:

    .Pattern = "^(([^a]*a){2})*[^a]*a[^a]*$"
    .IgnoreCase = True

Peter

I think you are missing something. It would be easier for me to tell if you just posted your code. Without seeing your code, here is my guess. The value in StartDate is a String value and the date in the cell is a real Excel date. Dates in Excel and VBA are not what you see; rather, they are floating point numbers that get displayed looking like a date for the convenience of the human user. The whole number part of the date's underlying value is an offset from "date zero" and the decimal part is the fraction of a 24-hour day represented by the time. So, ignoring any time considerations, today's date (February 17, 2011) is seen as 40591. So, if you stored something like "2/17/2011", you would be asking VB to find that string of characters in values that looked like 40591. So, Excel will never find 40591 if you tell it to look for "2/17/2011". Store a real date in StartDate and I think all will be well. For example, change the data type of StartDate to Date and then make this assignment...

StartDate = Date(2011, 2, 17)

then try your Find function.

Radio buttons (option buttons) group - Microsoft Community

queenb2001 say:

Radio buttons (option buttons) group

I have a form in which the users have to answer "yes", "No", or "N/A" to each question and I was wondering if there is a way to create a group so that I don't have to insert one button each time. They only need to pick one answer so I would prefer to have radio buttons over check boxes or dropdowns.

Thank you in advanced for the help.

jay freedman say:

While the document is in Design Mode, select an option button and click the Properties button. In the Properties dialog, assign a value to the GroupName property -- for example, use Q1 as the value for question 1, Q2 for question 2, etc. Put the same value in the GroupName for all three option buttons in the same question.

Option buttons that share the same GroupName behave as a group; clicking any one of them puts the black dot in that button and removes it from any other button. Each group is independent of all other groups.

Once you've created one group, you can select the three option buttons, press Alt+F3, and save the selection as a building block. The Gallery setting in the dialog determines what kind of building block it will be, and thus which gallery (if any) it will appear in. Good choices would be either AutoText or Quick Parts. When you insert the building block, you still will have to visit the Properties dialog of each option button and change the GroupName to make them a separate group.
