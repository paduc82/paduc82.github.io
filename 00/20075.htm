Plug this code into a module and run it; it will give you the name and userID of the person logged in.

Sub GetUserName_Environ()
    Dim ObjWshNw As Object
    Set ObjWshNw = CreateObject("WScript.Network")
   
    MsgBox "Username:     " & ObjWshNw.UserName & vbCrLf & _
           "ComputerName: " & ObjWshNw.ComputerName & vbCrLf & _
           "Domain:       " & ObjWshNw.UserDomain
End Sub

If your users will be the primary person logged in on their PC, you can do something like the following (code goes behind ThisWorkbook)

Private Sub Workbook_Open()

    Dim ObjWshNw As Object
    Set ObjWshNw = CreateObject("WScript.Network")
   
    ApprovedIDList = Array("User1", "User2", "User3") 'replace this with the real user IDs
    
    ConfirmName = Application.Match(ObjWshNw.UserName, ApprovedIDList, False)
   
    If IsError(ConfirmName) Then
    'do nothing
    Else
    'put your code here to hide the ribbon from those three users
    End If

End Sub

So far, this just answers your initial question about hiding the ribbon... but what happens if the user switches to another workbook to work on other stuff and needs that ribbon? What about re-showing the ribbon if they close this workbook? Good UI design requires that you only hide the ribbon when your workbook is active (using workbook_activate and workbook_deactivate), and that you turn the ribbon back on by default when this workbook closes (workbook_beforeclose event).

Also, what if your user opens the workbook, and manually re-shows the ribbon? If you are just trying to provide a good user experience with lots of screen real estate, then you don't have to worry about this type of user. If you are trying to forcibly control the user's access to the ribbon and make it unavailable... I haven't done that, so I'll leave that to other members of the forum to address.

Edit: One additional thought- the user may not have that ribbon enabled by default (like most end users don't have the developer ribbon showing, but this could apply to any ribbon). You may want to capture the ribbon's state in workbook_open and use that to reset the ribbon when switching workbooks or exiting your workbook. If you just set it back to visible, you may be unhiding it for end users who don't want to see it in other workbooks...

ron de bruin say:
Try this macro if the mail addresses are in Column A

Sub test()

    For Each myCell In Columns("A").Cells.SpecialCells(xlCellTypeConstants)

        If myCell.Value Like "?*@?*.?*" Then

            ActiveSheet.Hyperlinks.Add Anchor:=myCell, _

                                       Address:="mailto:" & myCell.Value, TextToDisplay:=myCell.Value

        End If

    Next

End Sub

Hi Chrisitna,

Unfortunately, there is no way to link two Microsoft accounts together.

Microsoft's official statement:

"Prior to June 2013 you could link existing Microsoft accounts together. To increase your security, we’ve retired that feature so it's no longer possible to link a Microsoft account (even with separate email addresses) or a non-Microsoft account (such as a Gmail account) to an existing Microsoft account. If you want to change how you sign-in or how your name appears to people, you can add an alias to your existing Microsoft account. This alias acts as a nickname for your account and uses the same inbox, contact list, and account settings. For more detailed information about how to add an alias, including why and how to set your primary alias, see the Manage aliases on your Microsoft account article"
