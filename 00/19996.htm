jett vega say:

Okay, I finally found a fix for this. It wasn't quite as simple as deleting or renaming it as a normal computer file (that always caused the following error):

Here are the steps I took that fixed the problem:

1) I unlinked the OneDrive for Work account in question so none of the files were trying to sync any longer. 

2) Attempting to manually rename or delete the folders still resulted in the same error, so I opened Command Prompt.

3) First, in command prompt I typed cd and the name of the parent folder (not the one causing the issues but the one above it).

4) From there, I typed DIR /x to pull up the list of shortened DOS names for the files. 

5) Once I located the DOS name, I typed rmdir "DOSNAME" (replace DOSNAME with your file's shortened name from the directory". If it is a single file, you can also try del "DOSNAME" . Don't forget the quotes or this might not work. 

6) At that moment, the folder finally disappeared for good. I had to repeat the process for the parent folder which had also been corrupted, but it worked. 

So far things are looking good for this fix, but we'll see if any further sync issues pop up. If they do, I will reply back here.

ken sheridan say:

You also need to include the region combo box in the criteria:

 

Private Sub cmdCreateReport2_Click()

 

    Dim varItem As Variant

    Dim strFilter As String

    

    If Me.lstCategory.ItemsSelected.Count > 0 Then

        For Each varItem In Me.lstCategory.ItemsSelected

            strFilter = strFilter & "," & Me.lstCategory.ItemData(varItem)

        Next varItem

    

        'remove leading comma and space

        strFilter = Mid(strFilter,2)

        strFilter = "[ID] IN (" & strFilter & ")"

   End If

 

   If Len(strFilter) > 0 And Not IsNull(Me.cboRegion) Then

       strFilter = strFilter & " And "

   End If

  

   If Not IsNull(Me.cboRegion) Then

       strFilter = strFilter & "Region ID = " & Me.cboRegion

    End If

    

   DoCmd.OpenReport "Multiple Category by Region", _

       View:=acViewNormal, _

       WhereCondition:=strFilter

 

End Sub

 

I've assumed that the BoundColumn of the cboRegion control is a numeric column RegionID.
