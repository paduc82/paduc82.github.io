Hi, 

There are a lot of possible factors for the bridge connections to have an unexpected errors. To isolate this matter, I have a couple of questions for you: 

  1. Were there any changes made before the issue occurred? 

  2. Have you tried updating your drivers manually? 

  3. What troubleshooting steps have you done so far? 

I recommend that you perform the troubleshooting steps indicated in the article entitled "Fix network connection issues in Windows 10". 

If the issue persists, reset your TCP/IP and see if there are any changes. It is possible that your TCP/IP were corrupted. To do this, follow these steps: 

Reset TCP/IP  

To do this, follow these steps:  

  1. Run Command Prompt as Administrator.  

  2. Run this command:  

    • netsh int ip reset and hit Enter.  

Exit Command Prompt and monitor your network connection.  

Disable auto turn-off of WiFi device. 

  1. Go to Network and Sharing Center. 

  2. Click Change adapter settings on the right pane. 

  3. Right-click your connection name (WiFi name) and choose Properties. 

  4. Click on Configure... button. 

  5. Go to Power Management tab. 

  6. Uncheck/disable the check box radio for "Allow this device to turn off this device to save power". 

  7. Restart the PC (Important).

Reconfigure TCP/IP 

To do this, follow these steps:  

  1. Run Command Prompt as Administrator.  

  2. Run these following commands:  

    • netsh int tcp set heuristics disabled  

    • netsh int tcp set global autotuninglevel=disabled  

    • netsh int tcp set global rss=enabled

The error is caused (in my case) of having a cluster size of 512. My system was a home built computer and I have just been cloning and upgrading drives from Windows 3.1. Don't know why it was set to 512, but once I changed it to 4096, the upgrade happened without errors. Apparently the Windows upgrade did not like this configuration although the clean install had no problem with it.

How to resolve -

I could find no free way of changing the cluster size. Maybe some backup programs would allow restores on a properly formatted drive? I used the (paid) program Acronis Disk Director. Minitools Partition Wizard says it can do it with its paid version as well. A couple of words of warning -- Make sure you have a backup (Disk Director trashed my C: drive the first time I tried to use it). Ideally a second drive with a copy of your C: drive on it. In fact the whole process is easier if you clone your C: drive onto a second drive and then do the restructuring by having your old C: drive as the the drive that you don't boot from. Second warning from Disk Director -- you need to have enough free space on your C: drive as 4K clusters waste more space than 512 byte clusters.

From this point on, Windows should automatically install the build updates as they come out.

The bad news for people that did a clean install of Windows 10 (initial or anniversary edition) and still have a cluster size of 512 -- you are going to have this same error every time Microsoft uses this same technique for upgrading.
