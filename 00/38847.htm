frederik long say:

OK, this makes things a lot clearer.

Port 110 is a bad port to use because it is the default POP3 server port.

When you say that you want this scheme to work everywhere then I assume that your RDP host is located within your own network and that the remotes could be anywhere. If so then you need to do this:

  • Select any unused port, e.g. 41312.
  • Give your RDP host a fixed internal IP address, e.g. 192.168.1.55.
  • Create a tunnel through your router for port 41312 so that it sends the relevant port packets to 192.168.1.55.
  • Adjust the RDP Host's firewall if necessary. Turn it off during the testing phase.
  • Get a fixed external IP address or URL for the RDP host site, e.g. www.MyRDPHost.org.
  • Invoke RDP with mstsc.exe www.MyRDPHost.org:41312

If this sounds too hard, use TeamViewer instead.
