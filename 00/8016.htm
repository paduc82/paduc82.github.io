Outlook for iOS - sending winmail.dat files - Microsoft Community

bryanjacksonoj say:

Outlook for iOS - sending winmail.dat files

How do i prevent Outlook for ios converting emails to winmail.dat?

anna bon say:

Hi Bryan,

This issue is caused by Transport Neutral Encapsulation Format (TNEF) which is affected by Rich Text Format (RTF) settings in Outlook. The Winmail.dat file is used to preserve Rich Text formatting. Outlook uses it when sending a Rich Text-formatted message. During transport, the content of the message may be changed, preventing the receiving client from being able to read the formatting instructions. In other cases, the receiving client does not use or recognize the winmail.dat file.

There is a similar issue on this thread with some troubleshooting steps that we can perform. Please check the post made by Erick Chu MSFT and let us know if it helps.

Please get back to us and update us with the result.

Thank you.

bryanjacksonoj say: Thanks, I did read that thread, however powershell and Outlook settings are not availsble to the ios app user.

anna bon say:

If PowerShell and Outlook settings are not available, we suggest that you follow alternative steps below:

  1. Open the message in Mail.
  2. Touch and hold the winmail.dat attachment. Or on a Mac, choose File > Save Attachments to save the winmail.dat file.
  3. You can open a file with a third-party winmail.dat viewer. You can find these types of viewers on the App Store directly.

As mentioned before, Microsoft Outlook sometimes use a special method to package information for sending messages across the Internet. This method is technically referred to as Transport Neutral Encapsulation Format (TNEF). There are three methods on how you can control TNEF. Please see below details:

1. Global - If you change your default e-mail format to plain text or Hypertext Markup Language (HTML), it helps to make sure that TNEF is not sent unless an Outlook feature needs it.

2. Per Recipient - You can specify in the recipient's e-mail address to not send TNEF so that a recipient always receives plain text versions of the message.

3. Per Message - When you compose a new message, or replying to a received message.

Should you have any other concerns, feel free to contact us back.

Thank you.

brian tillman [outlook mvp 2007-2019] say:
That's an SMTP timeout error.  It means that something is preventing Outlook from connecting to the SMTP server.  There are a couple of likely causes.  The first is your antivirus program.  It should not be configured to scan mail.  Completely uninstall it and reinstall it without any mail scanning feature.  You may need to use a removal tool from your antivirus vendor's web site to do this.  Doing this will not jeopardize your safety because first, you are smart enough not to click links in or run attachments to email without first looking where those links point or knowning what the attachment does.  Plus, when you reinstall it, the real-time or on-access portion of the AV program will protect you.  The second is in your account settings.  You are specifying an incorrect server name, an incorrect port, an incorrect encryption value, or incorrect authentication credentials.
