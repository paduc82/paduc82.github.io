- designated as PO Box only, or

- zip codes that sit inside the boundary of other zip codes

3D Maps should be fixed to cope with this.

In my case, what I wanted to map was our product delivery by zip code along with the $ value of the sales. So in my case, I didn't care about PO Box only zip codes.

jp ronse say:

Hi JP,

A collection or dictionary is a one dimensional array with a key and an item. This is very useful in large lists because you can call the item with the key. Note the key must be a string.

A typical use of a collection or dictionary is e.g. StateColl("California") to return the item that is linked to that state. It avoids looping through the array and compare one by one to find the the key to search.

Infact you should write your code as:

Sub test()
   Dim c As New Collection
   Dim box2_text As String ''' replace the underscore by a dot
   
   c.Add 1, "California"
   c.Add 2, "Colorado"
   box2_text = "Colorado"
   If c(box2_text) Then MsgBox "Hi"
   
End Sub

But the problem with collections is that it raises a run-time error when the key is not found and that is one of the big features of a dictionary.
