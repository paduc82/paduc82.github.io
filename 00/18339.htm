How can excel alert me when something is past due? - Microsoft Community

moose2243 say:

How can excel alert me when something is past due?

I need to make a log to show when calibrations are due. I know how to make it change color when something is past do, but is there anyway to make it alert everyone?  I will have the worbook on a shared network.

bernie deitrick say:

You can email based on the workbook open event. For example, if you have a list of instruments in column A, contact emails in B, and dates in column C, with column D blank, this will check the date and if column D doesn't show that someone was contacted, it will email the contact in column B.

 

Private Sub Workbook_Open()
    Dim ol As Object
    Dim myItem As Object
    Dim myC As Range
    Dim myR As Range
    Dim boolOLObj As Boolean

    boolOLObj = False

    Set myR = Range("C2", Cells(Rows.Count, 3).End(xlUp))

    For Each myC In myR
        If (myC.Value <> "") And (myC.Value < Now()) And _
           (Cells(myC.Row, "D").Value <> "Contacted") Then
            Cells(myC.Row, "D").Value = "Contacted"
            If Not boolOLObj Then
                Set ol = CreateObject("outlook.application")
                boolOLObj = True
            End If
            Set myItem = ol.CreateItem(olMailItem)
            myItem.To = Cells(myC.Row, "B").Value 

            myItem.Subject = Cells(myC.Row, "A").Value & " is past due calibration"
            myItem.Body = "Hello, " & Chr(13) & Chr(13)
            myItem.Body = myItem.Body & "Could you calibrate that thing? " & Chr(13) & Chr(13)
            myItem.Body = myItem.Body & "Thanks for doing that." & Chr(13) & Chr(13) & Chr(13)
            myItem.Body = myItem.Body & "Instrument Master Tech" & Chr(13)
            myItem.Send
        End If
    Next myC
    Set myItem = Nothing
    Set ol = Nothing
End Sub

Open WD and look in Quarantine or All Detected Items and look carefully at the "complete path."  If you are using an HP computer and the detection is in the D drive of the computer then you are most likely seeing a false detection in the recovery drive.

This is a known intermittent issue (being reported for years) involving probable false malware detections by MS antimalware programs only. Other scanners such as the Eset Online Scanner, Kaspersky Virus Removal Tool, Malwarebytes typically detect no issues in the D drive of HP laptops (installed by HP when the computer was manufactured) and thus far neither HP or MS has taken steps to prevent it from occasionally occurring.

There are two methods of dealing with this problem discussed on the HP Forum.  One solution is to exclude the D Drive from being scanned by Windows Defender or MSE depending on which program you are using.  The other solution is to remove/delete the recovery partition (D Drive).

http://h30434.www3.hp.com/t5/Notebook-Operating-System-and-Recovery/how-to-remove-Trojan-Win32-dynamer-ac/td-p/5598201

http://h30434.www3.hp.com/t5/Notebook-Operating-System-and-Recovery/Recovery-after-potential-trojan-on-recovery-partition/td-p/5487655

Spell-checking language in Powerpoint 2013 - Microsoft Community

ayajulio say:

Spell-checking language in Powerpoint 2013

I was trying to change the spell-checking language in Powerpoint but when I select all the slides it won't allow me to change it from one language to another, it will if I change it slide by slide but it turns annoying if I have a big presentation. I was able to fix it with a macro, but, could you please fix it? Since it might be an issue for other people with no programming background. Here's the macro I used (taken from http://superuser.com/questions/55034/change-the-spell-checking-language-on-a-powerpoint-presentation):

Option Explicit
Public Sub ChangeSpellCheckingLanguage()
    Dim j As Integer, k As Integer, scount As Integer, fcount As Integer
    scount = ActivePresentation.Slides.Count
    For j = 1 To scount
        fcount = ActivePresentation.Slides(j).Shapes.Count
        For k = 1 To fcount
            If ActivePresentation.Slides(j).Shapes(k).HasTextFrame Then
                ActivePresentation.Slides(j).Shapes(k) _
                .TextFrame.TextRange.LanguageID = msoLanguageIDEnglishUS
            End If
        Next k
    Next j
End Sub

I don't know if this counts as a bug, but I know some people that had the same issue in the past.
