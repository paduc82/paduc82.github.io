E1:  Food
F1:  Fruit

ken sheridan say: From your description I strongly suspect that data is being encoded as column headings.  A fundamental principle of the database relational model is the Information Principle (Codd's Rule #1). This requires that all data be stored as values at column positions in rows in tables, and in no other way.

If I am correct, the data should be recast into a correct design where each datum currently represented as a column heading appears as a value in a single column with the values which you currently have in the multiple columns in another column.  To do what you want would then be a relatively simple matter of aggregating values over subsets of rows.

If you can describe exactly what this table is modelling in real life terms then we would be in a better position to advise you.

FROM StoreStock_Vals
