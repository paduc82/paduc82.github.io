This may seem long, but the key parts are in the first two paragraphs.  If they work for you, you can skip the rest.  They are either for enhanced or improved methods of doing it that allow for more automation and options, or cover troubleshooting if these options still don't work (which I don't expect to be the case and thus won't be needed).

 

Here's a couple of things about scheduled scans. To run, the PC must be on and connected to the internet (it cannot be shutdown, in sleep mode, in hybrid sleep mode, or in hibernation). The power saving settings must be such that the PC does not shut down or go asleep during the scan (or it will be as if it never happened). Scans do not begin precisely when scheduled - some can start an hour or so earlier or later than the officially set time (depending on Microsoft's servers and the loads at that time).  Given your post, I suspect that the power settings are the problem. 

 

I know some people who create their own Power Saver Setting called "Scanning" (though it can be called anything not otherwise already used) and have it set merely to bypass auto shutdowns when on AC power for the parts that need to run without all the other features that aren't needed (to honor the concept of "Power Saving" which doesn't bother me) and then before running a full scan or an hour or two before one is scheduled to start (since they don't start exactly on time and can start an hour or even more earlier or later than the defined time based on Microsoft's ability to handle the scan due to available resources and the number of people doing it at the same time - and I believe early happens because they anticipate it coming, have the resources now, and try to get it in before they otherwise get overwhelmed) will open Power Options manually change the setting to "Scanning" and thus accomplish the same thing as long as they remember to do it and then change it back after the scan is done - LOL!

 

Another somewhat more complicated but perhaps ultimately more effective solution is to create your own task in Task Scheduler and control all the aspects necessary to get it to run properly (do NOT adjust the current one for MSE, but create a totally separate one).  Details about this can be found here http://answers.microsoft.com/en-us/protect/forum/protect_start/scheduled-scans-not-working/fd769043-f67e-4990-8805-a02427f22dff under the post by GreginMich.  This is probably more complex than you need, but I present it since if you can get past the setup effort (which is considerable), the end result may be more to your liking (more automated) than the other approaches.  Though sleep is not an issue for me because I always run scans during the day in the background and only use sleep at night (two nights sleep, one shutdown), I still do have a separate task for scans because MSE permits only one which I use for the daily quick scan but created another task for the bi-monthly full scan - but the complexities of sleep did not need to be addressed so I haven't actually done it precisely as Gregin has documented but it still should work.

If a scan is missed it does not start at the next available time, but waits for the next scheduled scan.

You can get a normal display of a PDF page by following these steps:

  1. In Reader, set your page you want ti to appear in your document. The whole page does not have to be visible on-screen.
  2. Choose Edit>Take a snapshot.
  3. Click and drag on the PDF page, drawing a box selecting the area you want to appear. If the page is larger than you screen, keep holding the mouse button while dragging. The window will scroll and eventually include the entire page.

  5. Open Windows Paint.
  6. Paste the page image in.
  7. Save as a jpg or png format image.
  8. Insert the image in Word.
