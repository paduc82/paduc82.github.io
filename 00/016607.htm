chay douglas say:

Hi Marvin,

Thanks for your reply. I managed to get this sorted:

As background, there were 4 users that couldn't see Public Folders but could see Shared Mailboxes.

In troubleshooting, I ran PowerShell Scripts that removed the Public Folders for all users by mistake:

  1. Get-OrganizationConfig | FL Pub*
  2. Set-OrganizationConfig -PublicFoldersEnabled Local
  3. Set-OrganizationConfig -PublicFolderShowClientControl 1

I changed these back by running the following commands:

  1. Set-OrganizationConfig -PublicFoldersEnabled Remote
  2. Set-OrganizationConfig -PublicFolderShowClientControl 0

This brought them back for all but the original affected users.

To fix the original users, they had to be granted special permissions on the Root Public Folder Mailbox, this had to be done via PowerShell and not using a GUI.

The script to grant All users the required access at the Top-Level Public Folder Mailbox is:

Get-Mailbox -resultsize unlimited | where {$_.recipienttype -eq "Usermailbox"} | set-mailbox -defaultpublicfoldermailbox "NameOfPublicFolderMailbox"

This fixed the Public Folder issues for the 4 affected users. (All using Office / Outlook 2013)

Cheers

Chay

Having double quotes in any field in a mailmerge data source - even if that field isn't referenced in the mailmerge main document - is liable to result in an incomplete or corrupt output. Presumably, the 'reports' are boilerplate text. That can all be embedded in IF fields within the mailmerge main document itself with a simple entry in the data source to tell Word which report to use. For example, suppose you have a column named 'Report', with report labels A, B, C, etc. In Word, you'd use a series of IF fields coded like:

{IF{MERGEFIELD Report}= "A" "Boilerplate text for Report A"}{IF{MERGEFIELD Report}= "B" "Boilerplate text for Report B"}{IF{MERGEFIELD Report}= "C" "Boilerplate text for Report C"}

how find link in word 2010 - Microsoft Community

simon benninga say:

how find link in word 2010

When I open a specific Word document, I get the message "This document contains links that may refer to other files."  

How do I find these links and eliminate them?

a. user say:

You'll need the "Edit links to files" option, but although it is listed as being on the File Tab, I can't see it anywhere in there.

So you can add it to the QAT:
Click File Tab->Options->Quick Access Toolbar
In the dropdown under "Choose commands from", Select  "File Tab"
Locate and select "Edit Links to Files", then click the Add button
Click OK.
In the QAT, you should see an icon that looks like two links of a chain.
Click that, and you should see a list of links in the active document.

Since deleting the links woud doubtless delete content that you need, you probably need to select each link in turn and click the "break link" button. I would proceed with caution, checking the effect of each action and using the Undo button as necessary.

You may find that some links do not appear in that dialog, but that should at least give you a starting point.
