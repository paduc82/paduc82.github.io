I've been seeing this in the Windows 10 Preview for some time and have reported it to Microsoft. I've had to do the following procedure after every reboot, but it finally appears to have "stuck" -- at least for now.

  • Shut down all Office programs.
  • Log in to a profile that's a member of the Administrators group.
  • Open the File Explorer program and go to the folder C:\Users\<your name>, where <your name> is the login name of your usual profile where you use Office programs.
  • Right-click the Documents folder and choose Properties in the menu.
  • In the Properties dialog, click the Security tab. Wait a few seconds for the list boxes to populate.
  • Click the Advanced button below the second list box.
  • In the Advanced Security Settings dialog, with the Permissions tab selected, look for your login name in the list of permission entries. It probably isn't there, so click the Add button.
  • In the Permission Entry dialog, click the Select a Principal link. In the box that opens, type your login name in the box and click OK. Your full name should now appear as the Principal in the first line of the Permission Entry dialog.
  • Check the Full Control box and click OK.

    If you reopen the Properties dialog and go to the Security tab, your name should now be in the first list box, and clicking on it should show in the second list box that you have full control.

    Log in to your usual profile. The Office programs should now be able to save in the Documents folder.
