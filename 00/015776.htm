jeeped say:

This is one way,

Sub looped()
    Dim v As Long, vHDRs As Variant, vCOLs As Variant, ws As Worksheet
    vHDRs = Array("Country", "Superhero", "Flower", "Name of City", _
      "State", "Year", "Month", "Favorite Author")

    vCOLs = Array(0, 0, 0, 0, 0, 0, 0, 0)

    Set ws = ActiveSheet
    With ws
        For v = LBound(vHDRs) To UBound(vHDRs)
            vCOLs(v) = Application.Match(vHDRs(v), .Rows(1), 0)
        Next v
    End With

    ' the next three lines write the results back to the Immediate window (Ctrl+G)

    For v = 0 To UBound(vCOLs)
        Debug.Print v & ". " & vHDRs(v) & " in column " & vCOLs(v)
    Next v
End Sub

Note that the array of integers in vCOLs is zero based; e.g. the first one is vCOLs(0), the last one in the example above would be vCOLs(7).

norman david jones say:

Hi Jeeped,

Note that the array of integers in vCOLs is zero based; e.g. the first one is vCOLs(0), the last one in the example above would be vCOLs(7).

I believe that the base of the array depends on the Option Base statement declaration, although in the absence of an explicit declaration, the default value of 0 will pertain and your comment will hold.

However, if Option Base had been declared as 1 at the top of the module, the following part of your code would fail:


    For v = 0 To UBound(vCOLs)

        Debug.Print v & ". " & vHDRs(v) & " in column " & vCOLs(v)
    Next v

This problem could, of course, be avoided by changing the first line to the more generic:

      For v = LBound(vCOLs) To UBound(vCOLs)

If it were of importance to maintain a zero base, independently of the Option base declaration, the Array functions could be qualified with a type library, e.g:

   vHDRs = VBA.Array("Country", "Superhero", "Flower", "Name of City", _
      "State", "Year", "Month", "Favorite Author")

and

   vCOLs = VBA.Array(0, 0, 0, 0, 0, 0, 0, 0)

===
