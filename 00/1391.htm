ken sheridan say:

I have a junction table (ProjectGrant) with two primary keys (ProjectID) and (GrantID), one from each table, which creates a many to many relationship.

No, what you have is a composite primary key made up of the two foreign key columns.  A table can have only one primary key.

A form interface for grants would comprise a parent form based on the WRP GRANTS table and linked to the parent form on GrantID.  This would be in single form view.  Within this would be a subform based on a query which joins the WRP PROJECT GRANTS and  WRP PROJECTS tables on ProjectID.  The subform, in addition to a control bound to the Category column from WRP PROJECT GRANTS, can include controls bound to whatever columns from WRP PROJECTS you wish.  Only the Category, as an attribute of the relationship type, would be updatable, however.  Any controls bound to columns from WRP PROJECTS should be made read-only by setting their Locked property to True (Yes), and their Enabled property to False (No).  Data should be entered into WRP PROJECTS via a separate form.

For data entry in the subform you would also need a control bound to the foreign key ProjectID column in WRP PROJECT GRANTS.  This would be a combo box set up so that the numeric key column is hidden, with a second visible column showing a value from another column in the table, or a concatenation of two or more columns, on the basis of which the project can be distinctly identified.  Once a project is selected in the combo box, any controls bound to other column in WRP PROJECTS would automatically show the relevant values.

The subform can be in single form or continuous forms view, though the latter is more usual.  It would then show all projects relating to the grant record currently selected in the parent form.

A report would be based on a query which joins all three tables.  The report would be grouped by grant.  The grant data would then be shown in a group header, with the project data in the details section.

You can of course also have a form/subform or report set up similarly, which shows the grants per project rather than projects per grant.

Function TestActive()

To remove all formula from Excel File - Microsoft Community

kishor mahajan_540 say:

To remove all formula from Excel File

I prepared an Estimate of Bldg. Excel using diff. formula and ref. Now I want to remove all formula and ref. keeping values only.I cand do it by copy and paste special value for each sheet but I want one command for all sheets

hansv mvp say:

Create the following macro in your personal macro workbook PERSONAL.XLSB:

Sub RemoveFormulas()
    Dim wsh As Worksheet
    For Each wsh In Worksheets
        With wsh.UsedRange
            .Value = .Value
        End With
    Next wsh
End Sub

When you run the macro, it will replace formulas with their values in all worksheets of the active workbook.

Remark: the macro will fail if the workbook contains protected worksheets.
