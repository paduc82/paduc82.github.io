In the Windows Movie Maker world, the WLMP file is a project file; it's not a video file that can be played. The project file [text file] simply tells Windows Movie Maker were all the media resources are and what special effects are being used [when and where] in the video project. You can't convert this file to video.

edsacks say: This problem is SOLVED.
This settings reside in HK_current_user\software\microsoft\\windows\currentversion\explorer\streams\desktop\taskbarwinxp.

As a test, I had exported the test account taskbarwinxp to a holding directory while in test account. When I rebooted into my regular account, and after I backed up (exported) taskbarwinxp, I substituted the test key for the regular one and rebooted. When the desktop was re-established, the test settings (which is what a wanted) were correct. I re-set them in the task bar and rebooted. However, the adjusted settings were ignored. So that experiment failed, except to point to something else.

After pursuing a number of theories having to do with group policies, and save/no save settings, I searched on the internet for windows shutdown functions. This is because changes made to the taskbar are saved in memory and written to the registry only when windows shuts down. I found an article on how to speed up shutdown. I theorized that my windows might be shutting down too fast. So I wanted to slow it down.

While following the instructions (below) I discovered that windows was set to save at shut down for between 20 microseconds and 1/10th of a second, which is really very fast. I made adjustments to save at  one second (1000ms) still fast, but not instantaneous. and that did the job. The size of taskbarwinxp value reduced from 85kb to 6kb (when exported) and it accepts every  change. That huge size is a clear sign that the value was massively corrupted, but it was the shutdown settings that caused the problem.

Here is what I did:
I changed the values in the following keys and subkeys and values  to 1000 (dex)
hk_c_u\control panel\desktop   HungAppTimeout
hk_c_u\control panel\desktop   WaitToKillAppTimeout
hk_l_m\system\CurrentControlSet\\Control   WaitToKillServiceTimeout
hk_users\default\control panel\desktop   HungAppTimeout
hk_users\default\control panel\desktop   WaitToKillAppTimeout

According to the article, settings may range up 5000ms.

The article and help came from from Mysticgeeks Realm.

http://blogs.howtogeek.com/mysticgeek/2009/03/12/make-windows-xp-shut-down-faster/
