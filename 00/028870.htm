Fixmapi tool did nothing to fix anything in the thread referenced.  The is was a registry setting:

This issue is finally resolved for me.

This so called "Mapi failure - unspecified error," actually has some specification. Prompted by a tip in another forum, I began searching the Windows event logs for anything related to this issue. I found in the Windows Application event log that Outlook was generating an Error ID 27:

"MAPISendMail: Failed to submit message. Attachments to the message exceeded the size limit set by your Administrator," each time I tried sending an attachment from another office suite application.

Now I had never set a size limit. Not with OUTLOOK 2013 or any earlier version. It appears that the when OUTLOOK 2013 was intalled to replace my earlier OUTLOOK 2010, then the "unlimited' attachment size setting is misread and recorded as "0" size in the OUTLOOK 2013 registry setting.

Changing this registry key: HKEY_Current_User>Software>Microsoft>Office>15.0>Outlook>Preferences>MaximumAttachmentSize from "0" to some larger value e.g., "5000" (5 megabyte) has resolved the issue.
