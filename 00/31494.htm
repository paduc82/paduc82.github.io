dhintz1 say: The problem has been resolved. I was running the monitor off of the graphics card.  I tweaked the BIOS so that it could run off of the onboard video. The default on an ASUS computer is the graphic card that comes with the computer. Once I switched to onboard video and used a DVI cord...my problem has gone away. For whatever reason the graphic card does not play well with windows 8.1.

dhintz1 say:

If you haven't already, maybe try a clean boot and reinstall of you AMD graphics driver/program.

  http://support.microsoft.com/kb/929135/en-us

So, I think our ITO at work has found a temporary fix. In the Device Installation Settings box, I checked "No, let me choose what to do" and then selected "Never install driver software from Windows Update". So far...knock on wood...that has stopped the seizures my monitor was experiencing.
