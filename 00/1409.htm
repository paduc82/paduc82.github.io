Response:

Personally I don't use Microsoft Query to get data from Excel files.
For data that I want to query I use a database to hold the data rather than a worksheet.

I have found MSQuery difficult to use and it often causes problems when, as I have, there are multiple versions of Office on the same machine.

When I tried to create a MSQuery query from your file I got a message saying "this file contains no visible tables".

At one stage it offered me FilterDatabase as the only named range available for me to connect to.

_FilterDatabase is a hidden name that was set up earlier in your use of the sheet and it refers to A3:M107

Thus any data after row 107 is unavailable to this query.

I think this may be what you are connected to.

In order to create a successful query I had to first delete the 2 rows above the data so that the table started in row 1.
And then in MS Query I had to click Options and check Systems Tables.
This let me select from a small range of possible names.
The one I chose was "Trade Log$" which represents the whole of the Trade Log sheet starting at A1.
That worked and refreshed successfully but included all 2000+ rows that are used in the sheet.

So all in all, I suggest you:

- delete the top 2 rows
- delete the excess rows and columns beyond the end of the data
- redo the query, connecting to "Trade Log$" (which appears as a System Table)

Hopefully you will then be OK.

However, I imagine there is probably a better way to achieve what you are trying to do than using MSQuery to access one Excel workbook from another.

The DivX player can play DivX and "some" xVid.

There are too many flavors of the DIVX, XVID codec that most likely you will find a lot of unsupported codecs.

I tried different codecs of the DIvX family and most play reasonably well without any transcoding. BUT,

if the bitrate of the file is more than 900 kbps then the phone processor will strugle to play it back. In those cases then you are back to transcoding either to lower the bitrate and still have DivX codec or change to MPEG-4 or AVC, as the transcoding will take the same amount of time anyways and MPEG-4 is a better codec in Nokia phones (consumes less power).

As a rule of thumb (for N95 at least), it seems that if the video resolution is higher than VGA there are little chances that it will play correctly. If it's VGA it will depend on the bitrate, if it's less than VGA then it most likely will play correctly (assuming it's a supported codec).

One thing I must stress is PRESUME NOTHING!, check the codec with DIVX player in your PC, maybe it's xVid and not DivX, xVid is almost identical codec than DivX but xVid is Open Source.

Stage6 website has a lot of content, so it's a good start for testing what you can and cannot do with DIVX player on your particular device.

1)  If you don't keep data under "On My Computer" folders, then create a new Identity and download your data from the server. How to move your data to a new Identity. Since the message never made it to the server, you should be OK.

2)  Put Outlook into Work Offline mode.

If you can try resending to put it back in the Outbox, there is a script that will delete it from the Outbox.

Delete Outbox Messages

http://www.scriptsformac.com/wp-content/uploads/2012/07/Delete_Outbox_messages.zip

3)  There is a way in the Finder to remove messages. You need to duplicate your Identity first since this could result in data loss. Make the duplicate BEFORE you do the steps below.

1)  Copy the subject of each message that won't delete if you can see the message.
2)  Delete the Exchange account. This deletes the cache file in Outlook. Your messages are still on the server.
3)  Quit Outlook.
4)  In the Finder, enter the subject  of the messages in Spotlight Search box. It will give you the .olk path. (These directions are for Lion/Mountain Lion. You can used advanced find in older versions of OSX. Under File in the Menu bar with Finder selected select Find.)

See example

http://www.flickr.com/photos/dianeross/6773768405/

In the example it shows xx00_76668.olk14message

~/Documents/Microsoft User Data/Office 2011 Identities/Main Identity/Data Records/Messages

7)  Delete the message in the Finder. See example click to enlarge
http://www.flickr.com/photos/dianeross/6773780991/

8) After deleting messages in the Finder, you'll need to rebuild the Identity.

9) If successful, I highly suggest you use the steps here to move to a new Identity after you complete the steps above.

How to move your data to a new Identity

ozrob---01 say:

 

pearson100 wrote:

Do I need to be on a data network (edge) for the gps to work? In other words, if I am connected via wi-fi, should I still be able to get satelllite info?

 

I did stand outside for 10 minutes and the positioning page was stuck on "Waiting for GPS."

 

 

You don't need a data connection, it just makes the GPS get satellite lock quicker. AFAIK WiFi doesn't work for this, it requires a cell network data connection.

 

If this is the first time you've used the GPS then without a data connection it can take a very long time to get a fix (depending where in the world you are). Once you've had an initial fix things should be quicker on the subsequent times you switch the GPS on as long as you don't move too far between uses (like go interstate or overseas).

 

So my advice would be to leave the GPS active for up to an hour while you're outside doing the gardening or something to see if you can get that initial fix.
