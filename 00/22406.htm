This is possible but it's quite a bit more complex; enough so that it may be more of a project than you really want to tackle.

And then it gets worse when you consider things like:

What if you/the user is on Slide 10, then jumps to (already viewed) Slide 3, then back to 10, then moves to Slide 20 before jumping back to 10 and then on to 11, 12 and so on?

Do you accumulate the total number of seconds spent on each slide or do you treat 10 (in this case) as several different "visits"?

What if the user temporarily stops the show then restarts it?

What if the user clicks an action button that launches some other program, then later returns?  Strictly speaking, they're not in PPT at that point, but as far as PPT and your VBA are concerned, they are.

Grouping Rows of Data using Subtotal Feature - Microsoft Community

mariehull say:

Grouping Rows of Data using Subtotal Feature

Does anyone know if when grouping row of information together, by using an outline of information through the subtotal featuer, if you can group data but NOT have it subtotal anything?  Basically, I have several tabs within a workbook with some drop down lists that I created on one seperate worksheet, but I need some way of expanding each section (which is related to a corresponding worksheet) but because it is text only and non-numerical information, I don't need / want the subtotal line to appear because it is way to cumbersome.  Does anyone know if this is possible using Excel 2007?  Thanks

 

adam vero say:

Basically, no. For some reason they chose to make the "OK" button greyed out if you untick all the boxes so that no subtotals are created.

Depending on how much data you have there are a couple of possibilities.

I would certainly consider using filtering (Home ribbon > sort and filter > filter), which will allow you a similar effect of showing only some rows and hiding others, but only if you have a column with something suitable for filtering (such as a date, or customer name or whatever). Don't overlook the very useful new filtering features which make it really easy to choose dates in certain ranges as well (eg a particular month). If you have been able to use subtotals I assume you must have a field (column) which would lend itself to this approach.

If you don't have too many rows and they don't change too much you could do this by hand, selecting the rows you want to group together and choosing the Data ribbon > Group button. There is a bit of a trick to doing this efficiently for a bigger range: Select all the rows (say 2-100 for example) and group them. Now selecting just the first row of the second bunch you want together (say row 11) and click "Ungroup". This breaks the big group into two halves, with the selected row as the first of the second group. Now select the start of the next group you want (row 37 say) and do ungroup again (you can use Ctrl-Y to repeat the last action here, or just click the button again).

This trick works on Excel 2003-2010, and as far as I know worked as far back as the grouping feature existed (but I don't have my old Excel 2000 up and running to check further back right now).

The wrong way: If you select some rows (say 2-10) and group them, then select the next bunch (say 11-20) and group them, Excel sees that these two groups are adjacent to one another and "helpfully" groups them all as one, rather than keeping them separate, which is why the above approach of grouping everything then breaking it up is more efficient than doing this piece by piece.

Hope this helps
