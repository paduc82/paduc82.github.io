a. user say:

I looked at this again recently, and as far as I can tell,

Word just stores the name of each thing it mapped in a list in the Windows Registry. If you know how to find stuff in there, you should find them in the following key:

HKEY_CURRENT_USER\Software\Microsoft\Office\xx.0\Common\DataServices\MMMatchedFields

where "xx" depends on your Word version - "16" for Word 2016, "14" for Word 2010 and "12" (I think) for Word 2007.

e.g. if you mapped your data source column name "myfieldname" to the standard name "First Name", Word creates a "key" called "myfieldname" under MMMatchedFields, and a "value" inside that called "First Name".

If you have another data source where "myfieldname" has some other meaning, and you map that to "Middle Name" instead, Word will just add a new value called Middle Name inside that "myfieldname" key. So the next time you use any data source with a field name "myfieldname", it will be offered as the default mapping for two name fields.

Changing mappings back to "unmapped"  does not result in the removal of these entries - to delete the mappings that you do not want, you can if you wish open the registry editor (probably called regedit.exe) and go and delete the entries that you do not want. Editing the Windows registry is generally regarded as quite risky because if you edit the wrong thing, it can have a serious impact on your system. If you take care, it is not that hard - from what you have said, I think that will fix your problem, but I am not certain.


NB, I have never found anything that describes what "this set of data sources" actually means. My suspicion is that the original designers of this feature had a slightly more complex design but (for whatever reason) it was never implemented. Or maybe further research would reveal how it is used.
