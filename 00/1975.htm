steve rindsberg say:

What you have is an Visio Object.  You can probably doubleclick it to open it in Visio for editing.

In cases like this, an Object consists of

- A bunch of Visio data that PowerPoint has no idea what to do with

- A bit more info that tells PPT "Heave this over to Visio when the user wants to edit it.  Visio knows what to do with it"

- And a picture for PPT to display.  It knows how to do that much.

 

When you ungroup the object, you throw away all the Visio info and end up with just the picture.

When you paste as picture, you end up with pretty much the same picture.

Either way, paste as pic or paste as visio object and ungroup, you have just a picture, no Visio object info.

 

IOW, ungrouping a visio object and retaining the visio info = eating your cake and having it too.

Can't do it.

 

One trick that's sometimes handy:  make a copy of the visio object, drag it off the slide.  Now you can ungroup the original and edit it as needed (as a picture).  You've still got a Visio object to return to if it needs more work in Visio.
