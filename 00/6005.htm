nick lo say:

I figured it out.  The combination of comments and hidden rows is the root cause of the problem.  I have comments in cells near the bottom of the sheet, and hidden rows above these commented cells.  For some reason, the scroll bar thinks it needs to scroll several rows past the cells with the comments.  So once I edited the comments and moved them just above the cells in which they are located, the scroll bar corrected itself.

rohn007 say:
That "...configuring..." message is displayed in 2 situations. You have confirmed one of them.  You have more than one version of Office installed. 

Office is written with the assumption that "any reasonable" user will only have one version (presumably the most current one installed). When you start an Office app it checks to see if some "other" version of the app was the last to run. If it sees that one was, it assumes you have just installed a new version but have not setup the registry correctly.

MS assumes that the "only reason" a person would have more than one version installed is that they are a developer, so the "fix" is a registry hack to add "NoReReg" registry keys

Here are the Official MS instructions:
Information about how to use Office 2010 suites and programs on a computer that is running another version of Office  KB 2121447

http://support.microsoft.com/kb/2528748  

In Method 1, this tip has a Fix it, 50780, that applies the required Registry fix.

Display Slicer Selections - Microsoft Community

sancarloscyclist say:

Display Slicer Selections

Is there a way to display in cells which values have been selected in a slicer? I have a pivot chart and I added a slicer. The slicer has many value options; hence, I would like to list in nearby cells which options are selected.

If this this possible, please walk me through the process. Thanks very much

teylyn say:

Jan Karel Pieterse has a great VBA function to pull the selected slicer values into a cell.

http://www.jkp-ads.com/articles/slicers05.asp

You need to copy the code into a regular code module and then simply use

=GetSelectedSlicerItems("MySlicerName")

in a formula, where "MySlicerName"  is the name of the slicer you want to use.

The VBA code is 

Public Function GetSelectedSlicerItems(SlicerName As String) As String
    Dim oSc As SlicerCache
    Dim oSi As SlicerItem
    Dim lCt As Long
    On Error Resume Next
    Application.Volatile
    Set oSc = ThisWorkbook.SlicerCaches(SlicerName)
    If Not oSc Is Nothing Then
        For Each oSi In oSc.SlicerItems
            If oSi.Selected Then
                GetSelectedSlicerItems = GetSelectedSlicerItems & oSi.Name & ", "
                lCt = lCt + 1
            ElseIf oSi.HasData = False Then
                lCt = lCt + 1
            End If
        Next
        If Len(GetSelectedSlicerItems) > 0 Then
            If lCt = oSc.SlicerItems.Count Then
                GetSelectedSlicerItems = "All Items"
            Else
                GetSelectedSlicerItems = Left(GetSelectedSlicerItems, Len(GetSelectedSlicerItems) - 2)
            End If
        Else
            GetSelectedSlicerItems = "No items selected"
        End If
    Else
        GetSelectedSlicerItems = "No slicer with name '" & SlicerName & "' was found"
    End If
End Function
