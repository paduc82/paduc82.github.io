jllatham say:

I know this thread is close to 4 months old now, but I think I have something of value to add to it.

I was asked to work with a workbook and provide a "table of contents" sheet that contained both the names of the sheets in the book and the page number that the first page of each sheet would print out as.  I tried several things to get this accurate including the use of Get.Document(50) and also calculations involving the horizontal and vertical page breaks.  But the results were almost always inaccurate!

But what I found was that IF you will force the sheets into Page Break View and then get the number of pages for a sheet while it is in Page Break View, then both methods work accurately!

The code goes a little like this - just enough to give you the gist of it.  You will notice that the one drawback is that you do have to "flip" through the worksheets, actually selecting each one.  You can keep this from being too annoying by using Application.ScreenUpdating = FALSE.

Here's the 'basic' code:

  Dim anyWS As Worksheet
  Dim returnToSheetName As String
  Dim pageCount As Integer ' printed pages per worksheet
  Dim pageNumber As Integer ' total count in workbook
  Dim iHpBreaks As Integer
  Dim iVBreaks As Integer

  returnToSheetName = ActiveSheet.Name ' save for later
  Application.ScreenUpdating = False
  For Each anyWS In ThisWorkbook.Worksheets

      'hopefully by forcing into Page Break view, we'll get a
      'correct count of horizontal and vertical page breaks (seems we do!)
      anyWS.Activate ' assumes all sheets are visible
      ActiveWindow.View = xlPageBreakPreview

      iHpBreaks = anyWS.HPageBreaks.Count + 1
      iVBreaks = anyWS.VPageBreaks.Count + 1
      pageCount = iHpBreaks * iVBreaks ' pages for this sheet
      pageNumber = pageNumber + pageCount ' total pages for the workbook
      ActiveWindow.View = xlNormalView

  Next
  'return to original worksheet
  ThisWorkbook.Worksheets(returnToSheetName).Activate
  Application.ScreenUpdating = True

Here is a macro that does both with conditional formatting. As is, it applies the conditional formatting to the entire sheet, but you can change both occurrences of Cells to a range consisting of specific rows, e.g. Range("1:100").

Sub SetConditionalFormatting()
    Range("A1").Select
    With Cells.FormatConditions.Add(Type:=xlExpression, Formula1:="=NOT(AND(ISERROR(SEARCH(""rework"",$C1)),ISERROR(SEARCH(""string"",$C1))))").Interior
        .PatternColorIndex = xlAutomatic
        .Color = -16752384
        .TintAndShade = 0
    End With
    With Cells.FormatConditions.Add(Type:=xlExpression, Formula1:="=NOT(ISERROR(SEARCH(""Summary"",$B1)))").Font
        .Bold = True
        .Italic = False
        .TintAndShade = 0
    End With
End Sub

Sub BoldRow()

jetta48 say:

Hi

It depends on the operating system that the tablet is running. Which do you have?

If its running Windows 8 or 8.1 (for example Surface Pro or Surface Pro 2) then you should treat anti-virus protection in the same way that you would a normal desktop or laptop computer. You can use the built-in Windows Defender which in these versions of Windows is a full anti-malware app. You can also install 3rd party antivirus software if you wish, but make sure that you don't run Windows Defender real-time protection at the same time (it may get correctly disabled by the 3rd part software anyway).

If its running Windows RT or RT 8.1 (for example, on a Surface RT or Surface 2) then your system is currently very unlikely to get infected because of the nature of these operating systems. However, run the built-in Windows Defender app anyway to be safe. You can only install apps from the Windows Store on these operating systems so 3rd party antivirus software is not available.

On other older Windows operating systems you should install an anti-virus app.

For tablets that run non-Windows operating systems please look for your answers elsewhere as we only deal with Microsoft products here.  :)
