wunders say:

You shouldn't need to worry too much about removing the invalid SIDs but you can if you want.  You should be more concerned about adding yourself to the list.  Being logged in locally as an administrator on the computer "remote-pc", when you double-click on the "Access this computer from the network" entry from the "User Rights Assignment" screen under "Local Security Settings", you should have a screen with a button on the bottom labeled "Add User or Group".  Simply click this button and add the "Administrators" group from your computer by inserting the name  "remote-pc\Administrators" where "remote-pc" is the network name of your computer and "Administrators" is entered as-is (name of the Administrators group).  Removal is just as easy with the "Remove" button.

If the "Add User or Group" button is Greyed-out (which I must suspect as the above solution is rather obvious), then the easiest way to include yourself is to use the command-line "NTRights" program to add the remote access rights to yourself.  "NTRights" is included as part of the free Resource Tools Kit available here:
   < http://www.microsoft.com/en-us/download/details.aspx?id=17657 >
After installing the resource kit, you can grant remote access rights to all administrators on your machine by bringing up a command prompt (Start -> Run -> "cmd") and entering the command:
   ntrights  -u  remote-pc\Administrators  +r  SeNetworkLogonRights
You might also want to check to make sure that administrators are not denied this right (Denials take precedent over Allows) by checking the "Deny access to this computer from the network" policy setting.  If administrators are denied, you can remove it with a command such as:
   ntrights  -u  remote-pc\Administrators  -r  SeDenyNetworkLogonRight

Note that "SeNetworkLogonRight" and "SeDenyNetworkLogonRight" are not listed as permissible entries if you enter " ntrights /? " but they will work.

Also, if you are trying to login remotely using Remote Desktop Connection, you will need to assign yourself the "SeInteractiveLogonRight" using the procedure above. (The properties above were for File and Printer Sharing connections)

HTH,
   JW

wunders say:

You provided some good information -- although I must admit I'm stretching to interpret it.

The Error 1789 you get when trying to change rights indicates a failure of either you or your machine to authenticate to the domain.  Which, to me, says that you may have physically disconnected a domain computer from the domain but perhaps not changed it into a Workstation.  Assuming that you have administrator rights and do not want it to be on a domain,  right-click "My Computer" then:

   Properties -> Computer Name (Tab) -> Change -> then make sure "Workgroup" (give it a workgroup name, too) is checked and "OK" your way out.

Before you do this, though, make absolutely sure that you can log into the computer using a Local Administrator account and not a domain administrator account whose login credentials have been cached.  Otherwise you will find yourself in this situation:

 

"You cannot log on after you remove a computer from the domain"

  < http://support.microsoft.com/kb/317049 >

The best to avoid this is to create a new local user with administrator privilege before making the change.

 

I'm glad you tried to connect with IP address.  The error you received "There are currently no logon servers available to service the logon request" indicates that from the network perspective, your machine is not responding at all to attempts to connect.

One cause is a Firewall.  Assuming you are using the Windows Firewall, go to:

   Control Panel -> Windows Firewall

On the "General" tab, make sure "Don't Allow Exceptions" is not checked.

On the "Exceptions" tab, make sure File and Printer Sharing shows up with a check mark beside it.

If you're not using Windows Firewall, try temporarily disabling your current firewall.

 

A second cause would be that File and Printer Sharing is not enabled.  Go to:

   Control Panel -> Network Connections ->

then right-click on your network connection and select "Properties".  Make sure "File and Printer Sharing for Microsoft Networks" appears in the "This connection uses..." box and it is checked. 

 

Also verify that your network sharing service is running:

   Start -> Run -> "services.msc"
When the window appears, find the line named "Server" and verify that its status is "Started" and its "Startup Type" is "Automatic".  If not, double-click on it and make it that way.

 

I hope one of the above helps you out a bit,

   JW

    bill smithers say:

    Hi,

    Unless HP''s Recovery D: partition is giving you a warning message about low disk space everything is fine with it.

    This is what it is for:

    Your Recovery Partition was imaged on your Hard Drive during the Manufacture of your Computer by the Manufacturer for these reasons.

    1.To reinstall Windows from in case of a System crash, by pressing particular Key or Keys at Boot.

    HP Press F11 repeatedly at Boot

    http://h10025.www1.hp.com/ewfrf/wc/document?docname=c01867418&cc=us&destPage=document&lc=en&tmp_docname=c01867418

    2. To make Recovery Disks from in case your Hard Drive fails, so that you can then reinstall the Operating System on to a new Hard Drive.

    HP:

    http://h10025.www1.hp.com/ewfrf/wc/document?docname=c01867124&cc=us&destPage=document&lc=en&tmp_docname=c01867124

    Your D: Recovery Drive is not to be used for backups, defragged, etc.

    Backups should be kept on an External Hard Drive.

    If you ever have to fix any problems that have occurred with the D: Recovery Partition, you will have to contact your Computer Manufacturer to remove whatever has been added to it.
