stefan blom say:

The following macro should work:

Sub AutoOpen()
ActiveDocument.Styles("Balloon Text").Font.Size = 12
ActiveDocument.Styles("Comment Text").Font.Size = 12
End Sub

Put the macro in your Normal template. Of course, this macro will run when any document is opened in Word.

stefan blom say:

The name of the macro, "AutoOpen," means that it will be run automatically when you open a document in Word. Do the following to put the macro in the Normal template:

  1. From within Word, press Alt+F11 to display the Visual Basic Editor.
  2. Click View | Project Explorer.
  3. In the Project pane, click to select Normal.
  4. Click Insert | Module.
  5. Paste in the code in my previous reply, including the Sub and End Sub statements.
  6. Click Debug | Compile Project.
  7. Click the Save button on the toolbar.
  8. Exit the Visual Basic Editor.
  9. You shouldn't have to restart Word, but you can do so just in case.

steven. s say:

Hey TheTennisOrganizer,

 

Data Execution Prevention (DEP) events are usually caused by a corrupted or incompatible add-on. Try using Internet Explorer (No add-ons) to see if you can identify a specific add-on that is causing the DEP events from occurring:

·        Click Start

·        Click All Programs

·        Click Accessories

·        Click System Tools

·        Click Internet Explorer (No add-ons)

·        Click Tools

·        Click Manage Add-ons

·        Click Toolbars and Extensions

·        Under Show: click All add-ons

·        Click any add-on

·        Press CTRL + A on your keyboard (this should select all add-ons)

·        Click Disable all

·        Click Close

·        Close Internet Explorer (No add-ons)

·        Open Internet Explorer

 

Now all add-ons are disabled in Internet Explorer, test to see if the same problem occurs without add-ons. If the problem is fixed then the next step is to enable the add-ons one by one to find which is causing the problem. (an alternative method is provided below these instructions)

·        Click Tools

·        Click Manage Add-ons

·        Click Toolbars and Extensions

·        Under Show: click All add-ons

·        Click the first disabled add-on

·        Click Enable

·        Click Close

·        Surf a few pages

·        Close Internet Explorer

·        Open Internet Explorer

 

If Internet Explorer still opens/works correctly, the add-on you enabled is good to stay. Repeat the enabling process for the next disabled add-on and continue this process until the problem returns. Then the last enabled add-on will need to be disabled/uninstalled/updated to avoid further problems.

 

Alternative method:

Instead of enabling add-ons one at a time, you could enable the first half of the add-ons and then test. If the problem is still gone, the add-ons you enabled are good to stay. Then enable half of the remaining disabled add-on and so on until the problem returns. Once the problem returns, you will need to test the last group of add-ons you enabled one by one.

 

Look forward to hearing back from you.

Steven
Microsoft Answers Support Engineer

I don't believe you can import PSTs into Apple Mail.  You'll have to export them in some other way or find a tool that understands both formats.

 

One way to accomplish the transition, however, is to connect Outlook on the PC to, say, Gmail or Outlook.com via IMAP and push the folders and messages to the mailbox on the server.  Then you can access that same mailbox with IMAP from the Apple.  That's what I'd do, at any rate.

There are a few reasons why you cant stay logged in.
1-Your browser is set to delete cookies on exit
2-A new security feature has been added
3-A policy has prohibited it on the machine
4-Misc browser issues.


It is possible that your browser option to delete browsing history upon exit is enabled.
Please follow these steps and edit this setting.
1. Open your Internet Explorer.
2. Press and hold Alt key then press "T", you will go to Tools menu of Internet Explorer.
3. Select Internet Options.
4. Under General tab look for Browsing history.
5. Untick Delete browsing history on exit.
6. Click Apply and Ok.
Close your browser and start a new session.
 
It is one of the new security changes.
Part of the change in service include some increased security measures to cut down on the frequency and ease of hacking.

The new log in ensures a more secure way of preventing unauthorized access to your Live ID accounts.
After all if you remain logged in it is an invitation for Hackers to access your account readily to create their mayhem.
No further updates are available on whether or not it will revert back to the previous format.
