christopherfromknecht say:

I have an ACER M5 laptop with the same problem.  I did some digging and found a possible fix, at least it fix the problem with my ACER.  The most common work around is manually pressing the power button everytime, but this kinda defeats the purpose of shut down and may cause unnecessary strain on laptop batteries if the mains is not connected.  The other fix that worked for me is under power management: minimum processor state in power management needs to be set to 100.  This is a similar problem with Win 8 (and I think I remember having the issue there as well).  Quickest way to change the setting:

Windows 10 search, search Power Option.

For the selected plan (mine was Balanced), select Change Plan Settings then select Change Advanced Power Settings in the next dialog.

When the Power Options pop-up opens, scroll down and expand the Processor Power Management option.

Expand the Minimum processor state and change both battery and plugged in state to 100%.  Press OK.

You will need to do this for each power plan that you use.

From what I understand from reading, Windows and Intel might have a conflict to who actually controls the processor for power management.  If you set it to 100, this forces the CPU to keep control and removes this conflict.  I might be wrong, but I researched more on how to fix and not so much the cause. 

Hope this helps.
