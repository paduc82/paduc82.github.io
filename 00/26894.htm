Missing some essential pieces of information.....


Were these 3 POP accounts configured using PST files from a prior Outlook configuration?


Re: PST files


Any PST file that is (or was at any point in its lifecycle) configured to be the default in an Outlook profile will have a "default" set of folders created which cannot be deleted in the Outlook UI.


An old format ANSI PST file (97-2002) would have a max size of 2gb so not clear as to the description of ".............because it's too big". Assuming you can open this PST file normally in O'2010 which seems to be the case based on the description, the quickest way to convert it to the current format would be to export the contents to a new PST file (ensure that you export from the <root> of the PST file and check the box "Include sub-folders". You can then close the old PST file and open the one you just exported to.


If all of your other PST files are old format, would suggest you migrate these to the current format using the same approach described above and then associate the newly exported PST file to the appropriate account  (click on the root of each PST --> Properties --> Advanced

(if the format shows as <Outlook Data File (97-2002)> then it's the old ANSI format)

    OK, The entity here is people. One of the attributes of this entity is whether they are a Supv, Mgr or just an Agent. But, since, they could be both or all three, it would be redundant to repeat them in a  Managers or Supervisors table.
