doug robbins - mvp office apps & services (word) say:

A dotm file is a macro-enabled template. It sounds like you are probably using File>Open and are opening the template, which is NOT the way in which a template is intended to be used.

What you probably should be doing is using File>New and then select the template as the basis of the document that you want to create.  When you do that, a reference is created to the template and even if you save the document as a .docx document, as a result of the reference being created to the template, any macros in the template will be available for use with the .docx document.

doug robbins - mvp office apps & services (word) say:
The code that causes the boxes to pop up should be in an AutoNew macro, or in the Document_New Event in the template.

devrin kuipers say:

I too encountered this problem.  Word would open if just opening Word with the Normal.dot template but would immediately stop working as soon as I attempted to open an existing document.  After working through the steps in Article ID 921541 I came to the COM add-ins section.  I had earlier suspected the "Send to Bluetooth" add-in was a the potential culprit.  The load behavior is set to "Load at Startup" and attempts to remove the add-in were thwarted with a message that the add-in "can only be connected or disconnected by an administrator."  I simply unchecked the add-in and clicked OK.  Word now opens fine.

To the Microsoft moderators, please add this to the KB.

Microsoft Word 2010 Stops Working and prompts to open in Safe Mode when reopening.

  1. Login to Windows 7 using an account with administrator priviledges
  2. Open Microsoft Word 2010 in normal mode (not Safe Mode) as if creating a new document.
  3. Select "File", "Options", "Add-Ins", Manage: "COM Add-Ins" and click on "Go".
  4. Uncheck COM Add-Ins "Send to Bluetooth" if the following: "Location: C:\Program Files\Motorola\Bluetooth\btmofficea.dll" and "Load Behavior: Load at Startup."
  5. Click "OK".
  6. Close Word and open any Word document.

The problem is associated with a Motorola Bluetooth module which installs a "Send to Bluetooth" COM Add-in option via a btmofficea.dll file.  Resolution is to disable the COM Add-In under Microsoft Word Options, Add-Ins, Manage COM Add-Ins.

 

Note: Installing Office 2010 Service Pack 1 does not resolve the issue.

Sub MergePrintAreas()
Dim vRng As Variant, vSht As Variant, t As Long
Application.ScreenUpdating = False

Sheets.Add
vRng = Array("A1:C10", "A20:B40")  '<<< change range and/or add rng
vSht = Array("Sheet1", "Sheet2") '<<< change names and/or add sheets
t = 1
For x = 0 To UBound(vSht)
Sheets(vSht(x)).Range(vRng(x)).Copy Cells(t, 1)
t = ActiveSheet.UsedRange.Rows.Count + 1
Next
ActiveSheet.UsedRange.EntireColumn.AutoFit
Set rng = ActiveSheet.UsedRange
With ActiveSheet
.PageSetup.PrintArea = rng.Address
.PrintPreview
End With
Application.DisplayAlerts = False
ActiveSheet.Delete
Application.DisplayAlerts = True
Application.ScreenUpdating = True
End Sub

Sub ConvertData()
