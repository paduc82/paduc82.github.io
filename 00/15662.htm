gary's student say:

This is just an example.  I assume that the questions are in A2 thru A20.  I assume that the answer choices are in B2 thru F20.  I assume that the correct answers are:

 

c2,d3,c4,b5,d6,e7,f8,e9,d10,c11,b12,d13,f14,e15,c16,b17,c18,d19,e20

 

If the user double-clicks outside the choice area, nothing happens.  If the user double-clicks the correct answer in the choice area, the cell turns green.  If the user double-click an incorrect answer in the choice area, the cell turns red. Put the following event macro in the worksheet code area:

 

Private Sub Worksheet_BeforeDoubleClick(ByVal Target As Range, Cancel As Boolean)
Dim Tbl As Range, Correct As Range
Set Tbl = Range("b2:f20")
Set Correct = Range("c2,d3,c4,b5,d6,e7,f8,e9,d10,c11,b12,d13,f14,e15,c16,b17,c18,d19,e20")
If Intersect(Target, Tbl) Is Nothing Then
    Cancel = False
    Exit Sub
End If
If Intersect(Target, Correct) Is Nothing Then
    Target.Interior.ColorIndex = 3
Else
    Target.Interior.ColorIndex = 10
End If

End Sub

 


Because it is worksheet code, it is very easy to install and automatic to use:

   1. right-click the tab name near the bottom of the Excel window
   2. select View Code - this brings up a VBE window
   3. paste the stuff in and close the VBE window

If you have any concerns, first try it on a trial worksheet.

If you save the workbook, the macro will be saved with it.


To remove the macro:

   1. bring up the VBE windows as above
   2. clear the code out
   3. close the VBE window

To learn more about macros in general, see:

http://www.mvps.org/dmcritchie/excel/getstarted.htm

To learn more about Event Macros (worksheet code), see:

http://www.mvps.org/dmcritchie/excel/event.htm

 

WE can expand and update this macro as more features and options are added.

Why I do not like the UI of Windows 10. - Microsoft Community

srfreeman say:

Why I do not like the UI of Windows 10.

The move back to the 'desktop' in Windows 10 is indicative of the lack of knowledge concerning the use of technology, in the current user base.

Things like the 'desktop' metaphor and File Explorer were chosen as part of the PC interface, based on the lowest common denominator of user and file storage capabilities in the 1970's. If we still have users and storage systems that do not possess capabilities beyond those available in the 1970's, it is now time to say goodbye to them. They are costing us, collectively, far too much.

We now have the capability available in purpose driven apps to have the programming, files, utilities and user interface, all available in one place. Regardless of the physical location of the various parts. There is simply no need for a complex base UI or to visually search what is becoming an endless stream of various types of files.

For example: When you want to do word processing, you can simply open one app and instantly have available all of the files pertaining to word processing that are available to you. These files can physically reside on your local system/network or any corporate/public cloud/private cloud server that you have permissions on. The app also, instantly, provides you with all research and graphics assets you find necessary to accomplish your needed word processing functions. There is simply no reason to move from the in-app UI to any other to accomplish the required functions.

We can customize/extend these apps, even at user level, without affecting security or IT staff workload.

Though the anachronisms remaining in the Windows 10 interface do not constitute any reason to not use Windows 10, they do hinder development and require additional user training. They should be done away with as soon as possible.
