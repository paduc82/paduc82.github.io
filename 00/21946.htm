  SaveDir = Buffer

  GoSub GetUseAPI
  Debug.Print "Before API", Buffer

  Buffer = "C:\"
  GoSub SetUseVBA

  GoSub GetUseVBA
  Debug.Print "Changed VBA", Buffer
  GoSub GetUseAPI
  Debug.Print "Changed API", Buffer

  Buffer = SaveDir
  GoSub SetUseAPI

  GoSub GetUseVBA
  Debug.Print "After VBA", Buffer
  GoSub GetUseAPI

GetUseVBA:
  Buffer = VBA.CurDir
  Return
GetUseAPI:
  Buffer = Space(MAX_PATH) & vbNullChar
  GetCurrentDirectory Len(Buffer), Buffer

  VBA.ChDir Buffer
  Return
SetUseAPI:
  Buffer = Buffer & vbNullChar
  SetCurrentDirectory Buffer
  Return

Skype accounts can not be separated or un-linked from Microsoft accounts.

suzanne s. barnhill say:
As marysully says, a two-column table is the way to do. One caveat: You must break to a new row at least once per page. Word gets very squirrely when table rows extend more than one page. The usual solution is to make one row per paragraph; this has the bonus effect of "synching" your original and translation.

marysully say:

Insert the text into a two-column Table. It may be easiest to type the original and translation text first into a document. Then copy and paste the text into the first Table column, and copy and paste the translation into the second Table column.  You can then remove the Table borders to get the column effect.

My way of working today, and it seems to work, and is fairly simple:

1) I run a first batch file (simple notepad text file with code in and saved as .bat) with following code that makes a copy of the (latest) .OTM file and saves it on a location on the sever:

REM close outlook first before running this code

cls

echo ******************************************************************
echo *                                   Do not forget                            *
echo *                             to close outlook first                       *
echo ******************************************************************

pause

REM delete old .OTM file

del --Insert file location on server here --\outlook\VbaProject_old.OTM

REM copy forms and modules from outlook (from pc with latest version of the code)

ren --Insert file location on server here --\outlook\VbaProject.OTM VbaProject_old.OTM
xcopy %appdata%\microsoft\outlook\vbaproject.otm --Insert file location on server here --\outlook

pause

2) The second .bat has to be run on every pc that needs the new .OTM
This .bat takes the copy from the server and replaces the .OTM on the pc where this .bat is run from

REM copy .OTM from server location to pc (over writes the existing .OTM)

cls

echo ******************************************************************
echo *                                   Do not forget                            *
echo *                             to close outlook first                       *
echo ******************************************************************

pause

xcopy --Insert file location on server here --\outlook\VbaProject.OTM "%appdata%\microsoft\outlook" /Y

echo ******************************************************************
echo *                                   Start Outlook                            *
echo *                                   press Alt+F11                            *
echo *         save the new .OTM and close the VBA-editor                  *

echo ******************************************************************

pause

    Selection.EndKey Unit:=wdLine

    Application.Run MacroName:="InsertHyperlink"

    Selection.MoveRight Unit:=wdCharacter, Count:=1

    Selection.EndKey Unit:=wdLine, Extend:=wdExtend

    Selection.InsertDateTime DateTimeFormat:="MM/dd/yyyy", InsertAsField:=False _

        , DateLanguage:=wdEnglishUS, CalendarType:=wdCalendarWestern, _

        InsertAsFullWidth:=False

    Selection.MoveRight Unit:=wdCharacter, Count:=1

    Selection.EndKey Unit:=wdLine, Extend:=wdExtend

    Selection.MoveLeft Unit:=wdCharacter, Count:=1, Extend:=wdExtend

    Selection.TypeText Text:="Yes"

    Selection.MoveRight Unit:=wdCharacter, Count:=2
