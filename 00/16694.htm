jeeped say:
Typing two numbers separated by a slash into a cell is going to ask Excel to resolve that ratio to a decimal place or possibly a date. If you want 1759/1812 in the cell you can treat it as text or as a fraction. For text, one of these will do.
  • prefix with a tick (e.g. ')
  • prefix with a <space> 
  • substitute something like the Unicode fraction slash (U+2044 or ⁄) for the regular forward slash.
  • format the cell as Text prior to inputting the 1759/1812 
If the cell needs to retain the value of the ratio then you can treat it as a fraction.
  • format the cell as Fraction prior to inputting the 1759/1812 . Note that this will resolve the ratio to its lowest common denominator of the number of digits specified. Unfortunately, I believe this is currently a maximum of three digits, one short of 1812.
  • type 0<space>175/181. I only mention this as an alternative to preformatting the cell as a Fraction. It is still limited to a three digit denominator
Note that rounding the fractions off to lesser digits does not change the underlying value of the ratio. It is for display purposes only.

Hi,


If the cell you're typing into is formatted as TEXT or GENERAL then you can type 1759/1812 and it will stay as it is. If the cell is formatted as (say) NUMBER or FRACTION then Excel will evaluate what you typed.


So if the cell is formatted NUMBER with lots of decimal places you will see 0.97075etc or if it's a FRACTION format of up to 3 digits you will see 531/547


That's the way Excel works and you can't switch it off other than by using the workarounds described in previous responses or by ensuring you use an appropriate format for the cell.


Try it, on a new worksheet format 4 cells as TEXT, GENERAL, NUMBER, FRACTION and enter the value in each one.


Depending on your viewpoint this can be even more irritating/helpful with the GENERAL format. The string you're entering cannot be interpreted as a date by Excel but if it could be interpreted as a date by typing (say) 1/1 in a cell then excel will interpret that as a date 1st January, change the GENERAL format to CUSTOM and add the current year.
