baÂÇaga zbigniew say:

Hi Dale,

Finnaly, I managed to find the root cause of strange behaviour of "Add field to Enterprise" function of MS Project 365 Professional. It was a dot (".") in the value of one item of lookup table. After removing it the table was created successfully!

To sum up what I've discovered:
 
1.
MS Project 365 Professional bug: application do not inform user about incorrect value of lookup table item: dot ("."). Example of incorrect value: "J.Smith team"
 
2.
MS Project 365 Professional bug: application returns incorrect message while the "Add field to Enterprise" function failed. The message was: "The field was successfully added to Project Web App.". The root cause of function failure: incorrect value ("J.Smith team") of one item of lookup table.
 
3.
MS Project 365 Professional bug: the "Add field to Enterprise" function do not allow to replicate the same name of the field on the server side. The message was: "The new field name is already in use. Enter a different name."
 
4.
MS Project 365 Professional serious functional limitation: once created Enterprise lookup table content cannot be changed via application. The content is displayed only.
 
5.
MS Project Web App serious functional limitation: the Enterprise lookup table cannot be changed by bulk operation eg. copy-paste from external dictionary. The item values could be changed only one-by-one.
