ken sheridan say: Firstly, do you want the duplicate rows form each table as multiple rows in the final table, or do you want only one instance of each duplicated row?  If the former you use a UNION ALL operation, if the latter a UNION operation.  So the first would be:

SELECT Item, Quantity, Price
FROM [Table 1]
UNION ALL
SELECT Item, Quantity, Price
FROM [Table 2]
UNION ALL
SELECT Item, Quantity, Price
FROM [Table 3];

The second would be:

SELECT Item, Quantity, Price
FROM [Table 1]
UNION
SELECT Item, Quantity, Price
FROM [Table 2]
UNION
SELECT Item, Quantity, Price
FROM [Table 3];

Note that I've not included the Total column.  That's because it contains redundant data, so there should not be any such Total column in the table as its values can be computed from the Quantity and Price columns.  To store redundant data in a column in a base table leaves the door wide open to inconsistent data as its possible to change the derived value so that it no longer matches the other two values, or vice versa.  You can either save one or other of the above queries and use the query as the basis for another query:

SELECT Item, Quantity, Price,
Quantity*Price AS Total
FROM [TheUnionQueryNameGoesHere];

or you could use the UNION ( or UNION ALL) query as the basis for an append query to insert rows into an empty new table, and use that table in a query:

SELECT Item, Quantity, Price,
Quantity*Price AS Total
FROM [TheNewTableNameGoesHere];

If you do create a new table like this you can then dispense with the original tables, but do not delete them until you are absolutely sure the new table contains all the correct data.  And even then be sure to back up the database with the original tables before deleting them.
Ken Sheridan, Stafford, England

macropod say:

Word uses a fully-qualified absolute path for mailmerges, so you can't make it search whatever happens to be the current folder. Accordingly, you will need to re-attach the data source when opening the mailmerge main document on another PC. The simplest way to do this is to answer 'No' at the SQL prompt upon opening, then re-attach the data source.

When sending the files to a PC for someone else to use, it's best to detach the data source beforehand (e.g. by choosing Mailings>Start Mailmerge>Normal Word Document, then saving) and providing instructions on how to attach the data source.

doug robbins - mvp office apps & services (word) say:

With a bit of VBA code, you can have a routine run when you close the Mail Merge Main Document, the details of the data source and the type of mail merge main document are saved to variables in the document, which is converted by the code to a Normal Word document.  There is also a procedure that runs when a document is opened to determine if it contains a variable indicating that it was originally a mail merge main document and if it was, it converts the document to a mail merge main document of that type and makes use of the information in the variable about the data source to re-attach the data source to the document.

That routine was all packaged up in the form of an Add-in and it would be necessary for the Add-in to be installed on the machine(s) of the person(s) who might be needing to make use of the mail merge documents.
