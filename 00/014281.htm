bob jones aka: cybertaz say:

Just to further clarify, it's possible to have text jump over a table. The issue in your case, though, is not the table, it's the fact that you want to change the page orientation. Doing that requires Section Breaks which, by definition, insulate one portion of the text from another.

It may help to recognize that there is no such thing as a page in the structural definition of a Word document. Pagination is imposed for output purposes based on all of the variables in a document's volume of content, layout specifications & formatting at any given point in time. Everything in the document is a part of the continuous text flow from start to finish. The 'number of pages' simply is an indication of how many sheets of paper are required to print that content based on those specifications.

A simple scenario to illustrate the point: Assume that a document currently comprises 20 pages... Now change the line spacing (or font size or any other single parameter) throughout the document. What happens? The result is that the document will require more or fewer sheets of paper [i.e., pages] depending on whether the line spacing was increased or decreased.

There is one way to work around the manner in which the program operates, but neither I nor any responsible, experienced user I know, would recommend it... In fact, most would recommend against it for any number of reasons. That method would be to use linked text boxes before/after the table. I only mention it because it can be done but it would impose limitations & restrictions as well as potentially  compromising document integrity.

per jessen say:

You have to do a little work here:-)

To find the correct name of the pre-specified printer, turn on the macro-recorder, goto the print menu, and change to the desired printer and hit 'Cancel'. Turn off the macro-recorder. Now in the macro just recorded, copy the name of the printer, and insert it in the macro below:

Sub PrintAndSave()
Dim DefaultPrinter As String
Dim MyFileName As String
Dim MyPath As String

MyPath = "C:\Temp\" ' Change to suit
DefaultPrinter = Application.ActivePrinter
Application.ActivePrinter = "Your pre-specified Printer Name"
ActiveSheet.PrintOut
Application.ActivePrinter = DefaultPrinter
MyFileName = Range("D8") & Range("E8") & Range("F8") & Range("G16") & ".xls"
ThisWorkbook.SaveAs MyPath & MyFileName
End Sub

Here's one way to solve the problem. Instead of trying to timeline a dynamic master, create a static master instead.

When you create the master, un-check the "link to project" option in the lower right corner of the Insert Project window. That will create a new separate file that is a snapshot in time of both subprojects. You will have to re-create a static master each time you want to update but you will also avoid corruption issues associated with linked structures (i.e. dynamic master with subprojects).
