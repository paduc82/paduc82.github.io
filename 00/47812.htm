cmdrkeene say:

Nope, those are compatibility shortcuts.  The "Documents and Settings" for example, that's not really a folder.  It was back in the Windows 98 days, but now it's just simply "Users" (to make it easier to translate, shorter path, and to remove the spaces that are annoying when scripting and coding).

But some programs don't look for a variable name, they want a real folder spelled exactly "Documents and Settings", and those bad programs crash if they don't find it.  The same programs usually crash when they are ran on another language comptuer, because obviously the folder name won't be spelled out that way in another language.

So for compatibility, a folder redirect called a junction is placed instead.  You see access denied, but if a program tries to "write" to that folder, it will be automatically redirected to the proper place.

You can hide these compatibility junctions by setting your Folder Options view tab to defaults.

It could suggest you need to update your chipset drivers and BIOS.

Make sure you are connected to the Internet, the check for updates.

When complete, restart then check if you can sleep and wake.

Second option
uninstall your display driver then restart
Boot to the Windows 10 desktop

Press Windows key + X
Click Device Manager
Expand Display adapters
Right click the current display adapter
Click Uninstall
Exit Device Manager
Restart

time lady say:
Hello, Many users are having the same/similar problems & if you can log in normally or via safe mode the solution is to turn off use sign-in info to auto finish setting up device after update or restart in Windows 10.
For Windows 10 (Version 1709), select the Start button, select Settings > Accounts > Sign-in options, then make sure Use my sign in info to automatically finish setting up my device after an update or restart is turned off.
For previous versions of Windows 10, select the Start button, select Settings > Update & security > Window Update > Advanced options, then select the check box next to Use my sign in info to automatically finish setting up my device after an update.
Another potential solution is to remove any USB devices (wireless mouse, keyboard, external hard drive/s) & wait to see if boot continues, if not then to restart with devices still disconnected (if possible).
