richardeiler say:

Hi mark,

Each specific users account folder (i.e. C:\Users\"your account name") will/may contain some folder(s) and/or files that are unique and specific to that account only depending on what has been installed on the device while using that specific account, so therefore, a direct comparison between any two separate user accounts folders contents will not mirror one another exactly, and often, there are a lot of differences between them.

It should be fairly easy to tell if you're seeing a hidden folder in file explorer in that a hidden folders folder icon will be semitransparent in appearance as compared to a non-hidden type folder.

To check to see if your setting is working properly, while viewing in file explorer click on the view tab and then toggle the Hidden Items check box (checked, then unchecked) and compare the contents that are visible to you.

For illustration purposes I used the Public account folder as an example to show the difference in appearance of a hidden and a non-hidden folder (if you view your Public account do not make any changes to it as this is a system account for system use only).

There are no hidden files in this account folder but you get the idea, any hidden files that are present in a folder will also appear or disappear depending on the Hidden items checkbox setting. 

If you need more help or have any questions or comments just let us know.

Good Luck,

SD cards are reknown for this porblem. Often its a physical thing rather than software.

Try:

- moving the lock switch from lock to unlock 3 times leaving it in the unlock position for at least 30 seconds.

or,

- putting a thin strip of tape over the lock switch (don't cover the metal contacts on the other side though)

or,

- putting a thin strip of tape over the little notch on the opposite side of the card to thelock switch (don't cover the metal contacts on the other  side though)

The reader may also be at fault as the write detector is not robust - its worse than the old floppy disks. Try you card in someone elses reader to rule this out. This was my problem and it cost only a few pounds to buy a USB thumb reader.

If you don't know where DMP files are located, here's how to get to them:

 1.    Navigate to the %systemroot%\Minidump folder.
 2.    Copy any and all DMP files in the Minidump folder to your Desktop and then zip up these files.
 3.    Upload the zip containing the DMP files to Skydrive or a hosting site of your choice and paste in your reply.

If you are going to use Skydrive but don't know how to upload to it, please visit the following:

http://www.wikihow.com/Use-SkyDrive

Please note that any "cleaner" programs such as TuneUp Utilities, CCleaner, etc, by default will delete DMP files upon use.

If your computer is not generating DMP files, please do the following:

1. Start > type %systemroot% which should show the Windows folder, click on it. Once inside that folder, ensure there is a Minidump folder created. If not, CTRL-SHIFT-N to make a New Folder and name it Minidump.

2. Windows key + Pause key. This should bring up System. Click Advanced System Settings on the left > Advanced > Performance > Settings > Advanced > Ensure there's a check-mark for 'Automatically manage paging file size for all drives'.

3. Windows key + Pause key. This should bring up System. Click Advanced System Settings on the left > Advanced > Startup and Recovery > Settings > System Failure > ensure there is a check mark next to 'Write an event to the system log'.

Ensure Small Memory Dump is selected and ensure the path is %systemroot%\Minidump.

4. Double check that the WERS is ENABLED:

Start > Search > type services.msc > Under the name tab, find Windows Error Reporting Service > If the status of the service is not Started then right click it and select Start. Also ensure that under Startup Type it is set to Automatic rather than Manual. You can do this by right clicking it, selecting properties, and under General selecting startup type to 'Automatic', and then click Apply.
