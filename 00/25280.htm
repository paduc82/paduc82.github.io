albert d. kallal say:

In access 2007, and 2010, you can hide the office button and all of the interface (including the most recently used file list (MRU) with the following line of code:

 

DoCmd.ShowToolbar "Ribbon", acToolbarNo

 

So, the above hides ribbon, office button, and the QAT.

 

If you actually need the office button, and are not looking to hide quite so much like above, then in 2010 you can simple use the following xml in a custom ribbon.

 

<backstage>

   <tabidMso ="TabRecent"visible="false"/>

</backstage>

 

So, the first suggestion will work and hide the MRU for both 2007 and 2010. However, the second suggestion to just disable the MRU as above only works in access 2010. I don't believe that 2007 has this second suggestion to hide the MRU by using custom xml like you can in access 2010.

brian tillman [outlook mvp 2007-2019] say:
Outlook cannot reject messages.  Mail client don't have that capability, in general.  Instead, the errors you describe indicate clearly that the mailbox of the address to which the messages are being sent don't exist.  Most of the time, this means the address is incorrect.  The name in front ot the @ in the address is wrong.  I think those particular senders have the problem if others are able to send you mail.  Sometimes, though, the address can be correct, but seomthing on the server isn't allowing the messages to be placed in the mailbox.  Since it's Hotmail's server reporting the error, I'd lean toward a misspelled address.
