Laptop Touchpad stops working after Windows Update 11th April 2018 - Microsoft Community

clive_hornsby say:

Laptop Touchpad stops working after Windows Update 11th April 2018

#1 Use Windows Function Keys

When the touchpad is not moving or responding after a Windows 10 Update, the first thing is to check is if your laptop has the Windows function key that enables/disables the touchpad. The Windows 10 Update may disable your trackpad scrolling function.

Depending on your laptop’s or touchpad’s model, the Windows keys you should press simultaneously vary from one to another brand.

The following combinations are some common function keys that enables/disables the touchpad in Windows 10: Fn + F5 / Fn + F6 / Fn + F9 / Fn + F3.

#2. Modify Touchpad Settings to Revive Unresponsive Touchpad

If your laptop doesn't feature a touchscreen display, then you will need to use a mouse to revive the unresponsive touchpad.

  • Follow the path: Start button > Settings > Devices > Mouse & touchpad option > Additional mouse options.
  • Then the Mouse Properties window will open. Go to the tab that names your laptop's touchpad, usually named the brand + touchpad model, such as Dell Touchpad. If you don’t see such a tab, turn to a tab labeled Device Settings where you can find your touchpad listed under Devices.
  • Click on the touchpad to select it, and then click the Enable button.

#3. Update Trackpad Drivers to Get Your Touchpad back to Normal

The trackpad drivers could be outdated, missing or corrupted and you need to update your touchpad driver to fix it.

greg carmack - mvp 2010-2020 say:

The backup image will carry the same formatting so cannot be restored if changing from MBR to GPT. 

You can do a Clean Install Windows 10 which is the gold standard install vastly superior to a factory or Upgrade install which most enthusiasts won't even run.  Use the latest media provided to install V.1703.

What error are you getting exactly? 

What is the present BIOS setting?  Type MSINFO in Start Search and report back whether BIOS is installed to UEFI or MBR.  There's a chance it already is UEFI but getting a false error based on another setting in the UEFI/BIOS Setup for CSM or Legacy also being enabled which allows for MBR installs as well as UEFI.  What's important is if Setup is set to UEFI, and that Windows Boot Manager is set first to boot as it must be on a UEFI install.  On a Legacy BIOS install the Hard drive should be set first to boot. 

Please check these settings and report back.  Or you can skip to the Clean install making sure the BIOS is set to UEFI and to delete all partitions down to Unallocated Space during install.
