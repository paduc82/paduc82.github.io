jeeped say:
You can use INDIRECT() to construct a cell reference (even an external one) but there are drawbacks.

=INDIRECT("'[Registry.xlsx]"&TEXT(DATE(2013,1,ROW(1:1)),"ddmmmmyy")&"'!F17")

The fill down as necessary. If you want to avoid #REF! errors when you run out of days-in-a-month then,

=IFERROR(INDIRECT("'[Registry.xlsx]"&TEXT(DATE(2013,1,ROW(1:1)),"ddmmmmyy")&"'!F17"),"")

INDIRECT() can be used on external workbook references but the external workbooks must be open; preferably in the same application instance that the report workbook is in. INDIRECT() is also considered a volatile function that will recalculate whenever any value in the workbook changes; not just when a values that affects it changes.

INDIRECT function
TEXT function
ROW function
DATE function
IFERROR function

Edit: absolute cell references (e.g. $F$17) are unnecessary when a cell address is represented by a text string as the string will not change.
