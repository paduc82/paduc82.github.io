boblarson say:

You can import if you go to FILE > GET EXTERNAL DATA > IMPORT and then you click the file type drop down in the bottom of the file dialog to use ODBC as the source and then you either select an existing DSN if you have one or you create a new one for connecting to SQL Server (best to use SQL Native Client 10 as the driver if you have it available).  Then you should be able to get a list of the tables and select them and hit okay and they should import.
