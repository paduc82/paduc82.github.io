Object Linking & Embedding [OLE] is not as fully implemented in Mac Office as it is in the Windows versions. PDF is not a supported object type, so the first 'error' is the one that applies. As an alternative you can use Insert> Picture but only the first page of a multi-page PDF will display as an image.

john korchok say:

To further Bob's comments, OLE is a Microsoft technology that has not been implemented in OS X by Apple. The micro version of OLE that comes with Office for Mac has only the following capabilities:

  1. You can insert a Word object in an Excel workbook.
  2. You can insert an Excel or Word object in Word or PowerPoint.

That's all there is, and I'll bet a nickel that's all there ever will be. Neither Apple nor Adobe seem to have any interest in extending Microsoft's technology for them.

For Illustrator graphics, export as EMF, then import to PowerPoint.

P.S. SVG graphics import is being implemented in PowerPoint, but we'll have to wait for this to arrive on the Mac.

teylyn say:
Hello,

colour is not data. Apply the first cell colour by using a number in a hidden cell in the row and a conditional format based on that number. 

Apply the second colour by using another number in a hidden cell and build a conditional format that evaluates the two numbers.

For example: Hidden cell Z1 has the value 1. Cells A1 to F1 have a conditional format with a red fill if Z1 = 1 
Hidden cell Y1 has the value 1. Cells A1 to F1 have a conditional format with a pattern of red and blue if Z1 = 1 and Y1 = 1

This will potentially be a lot of work to set up, but then, using colours to convey meaning is always labour intensive.
