john korchok say:

Under Windows, all kinds of objects can be embedded using the Windows-only technology called OLE (Object Linking and Embedding). Apple has never included this technology in macOS. So Microsoft has done a miniport of OLE for Office for Mac only. This miniport handles only Excel and Word objects. Even PowerPoint files cannot be placed in Word or Excel.

PDFs were invented by Adobe, not Microsoft, so it's unlikely Microsoft will ever extend OLE on the Mac to handle them.

If the .docx file was embedded, you should be able to open it, it's working here. If the .docx was linked, then the path to the linked file will be different on a Mac and it will be unopenable.

Here's how to extract those files so you can refer to them:

  1. Copy the Word file to a separate folder, then change it's file name by adding .zip to the end.
  2. Choose Use .zip when macOS asks you if you want to change the extension.
  3. Right-click on the file and choose Open With>Archive Utility.
  4. Open the word/embeddings folder. The embedded PowerPoint will most likely have a .pptx file ending, the Word file will end with .docx. Unfortunately, the PDF will be changed to oleObject1.bin and will probably not be openable even after changing the extension back to .pdf.

If there is no embeddings folder, or some files are missing from the embeddings folder, then they were linked, not embedded. You should request the original files from the source of the Word file.
