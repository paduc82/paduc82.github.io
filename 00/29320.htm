Why is "(C:)" not defragmented by default? - Microsoft Community

914ian914 say:

Why is "(C:)" not defragmented by default?

And also, what is the difference between "(C:)" and System? Doesn't the system reside on "(C:)"?

bossdweeb say: I'd guess it's because you are using an SSD.
- In an effort to lessen the number of writes to the SSD, they don't get defragmented.
This is because an SSD's  lifespan is linked to the number of writes due to the nature of the memory cells.

SSD's use a different system referred to as TRIM, or garbage collection, which is taken care of
by the SSD's firmware and Windows.

- There is some debate how much writes affect the lifespan relative to the lifespan of a
traditional HDD. I would hope that doing an abundance of writes is not too critical, if it is,
the SSD  is not the storage solution of the future.
Nonetheless, defragmenting is a process of intensive rewriting,  and it is effectively redundant
on an SSD  so it's best not to defrag.

A drive/partition letter (C:\ ) can have any name you like assigned to it as the drive letter is the critical part  for the 'Path' to program executables, or files ( ie - my C:\ drive is named BASE).
You cannot change the drive letter on the drive Windows is installed to. Most commonly C:\,
but you could install Windows to Y:\  partition if you wished.
Using separate drive letters is essential if 'dual booting' Win7 and XP, Win8, or Linux,
but it is not advised to install to something other than C:\ otherwise, as some applications have issues when installed to another drive letter, or won't install to another drive letter.
I don't know if it is still the case, but Nvidia graphics drivers is one example.

If the word System is not used in conjunction with another partition then it would have to
be the name for C:\ partition ( I haven't used the Windows defrag for some time, so I don't
recall if it uses both drive letter and drive name on the list ).
If it is used with another drive letter then it may be the  backup partition for the OS.
  - If the defrag lists both a C:\  and System separately as different drives then... ain't that confusing ?
AsusLogics defrag shows both drive letter and name for each partition.

grzegorz krzymiski say:
Others have problems that their Num Lock remembers the previous state, and it's overwriting this value depends what was the state during the shut down.
If you are the person who has also the problem with the overwritten registry key, this is some solution / workaround:
  1. Press WinKey+R > type regedit and press enter
  2. Navigate to HKEY_CURRENT_USER\Control Panel\Keyboard\
  3. Set InitialKeyboardIndicators to preferred value... 0 to set Num Lock OFF  
  4. Right click on the Keyboard folder > Permissions > Advanced button
  5. Select Administrators (ComputerName\Administrators)
  6. Click Edit... button
  7. Select Deny for Set Value , and click OK
  8. Click OK in the Advanced Security Settings for Keyboards window (possible that there will be some additional confirmation dialog box)

From now on, the Keyboard key/folder in the registry is protected from changing values. The InitialKeyboardIndicators will be no longer overwritten by the previous state of the Num Lock. The initial state depends only at InitialKeyboardIndicators value. You can restart your system, and you will notice that if you set previously value = 0, the Num Lock will be turned off, no matter it was turned on/off during the shut down process.

In the case you want to change values for the Keyboard keys, follow these steps:
  1. Press WinKey+R > type regedit and press enter
  2. Navigate to HKEY_CURRENT_USER\Control Panel\Keyboard\
  3. Right click on the Keyboard folder > Permissions > Advanced button
  4. Select the entry which looks like this: "Deny | Administrators (ComputerName\Administrators) | Set Value | <not inherited> | This key and subkeys"
  5. click Remove button
  6. click OK
