suzanne s. barnhill say:

There is no "later version of Windows" than 8.1. The screen shots were probably taken in Windows XP or Windows 7. But that article only briefly touches on reregistering Word. What you want to do is find the complete path for the executable for Word 2013. On my Windows 7 system, it is

C:\Program Files (x86)\Microsoft Office\Office15\WINWORD.EXE

So what you have to do is press Windows Key + R to open the Run box. In it paste the copied path and add the quotes and /r switch to get this:

"C:\Program Files (x86)\Microsoft Office\Office15\WINWORD.EXE" /r

and press Enter. Note that there is a space before the /r switch, which is outside the quoted portion. This will "reregister" Word 2013, that is, reestablish it file associations (its "ownership" of the .docx, .dotx, etc. file types) in the Windows Registry. Word will not open, but you will get a message box similar to this one (but referencing Office 2013).

You can do the same for Excel 2013, but the switch is /regserver. The same may be true for other Office applications.

Auto Paragraph numbering - Microsoft Community

dmrt27 say:

Auto Paragraph numbering

I need to create a paragraph numbering system for my paragraphs need to look like the following.  I also need to create a TOC once I'm finished creating my document.  It is imperative that these paragraphs be automated so that if I modify my document, it will automatically renumber itself.

1.     Here is my paragraph text.  Numerous sentences.  blah blah blah.

       1.1   Here is my subparagraph

       1.2   Here is my second subparagraph under Paragraph 1.

       1.3.....followed by 1.4, 1.5, etc.

2.     Here is my paragraph 2 text.  Again, numerous sentences.   blah blah blah

       2.1   Here is my subparagraph.  The reason for this long sentence is to show that I only want the line with the subparagraph number to be indented with all remaining text to go back to the left margin.

       2.2   Here is my second subparagraph under Paragraph 1

       2.3.....followed by 2.4, 2.6, etc.

I may also need to 3rd layer of paragraphs that will run:  1.1.1, 1.1.2, 1.1.3, etc.

              

suzanne s. barnhill say:

Since you want a TOC based on these numbered paragraphs, it will be easiest to use the built-in heading styles for the paragraphs. The article at https://shaunakelly.com/word/numbering/numbering20072010.html tells you how to do that. If you select the multilevel list that shows 1 Heading 1, 1.1 Heading 2, 1.1.1 Heading 3, your numbering levels will already be linked to the heading styles. You'll need to modify the number formatting (via Define New Multilevel List) to have a period following the number, and, if you want the paragraphs to have the plain appearance you show above, you'll need to modify the heading styles themselves to have different font and possibly paragraph formatting.

You could use the built-in List Number series of styles, which are already quite plain, but (a) they're not already automatically linked to a list template, so you would have to do that manually, and (b) they would not automatically appear in a TOC.

If you plan to base a TOC on these styles, then by default the TOC would include the entire paragraph of each heading, which you say would be several sentences. To avoid that, you can use the style separator. Apply the heading style to only the part you want in the TOC, apply a non-heading style (a style with an outline level of Body Text) to the rest, and insert a style separator to make the two paragraphs join (see http://wordfaqs.ssbarnhill.com/RunInSidehead.htm for instructions).
