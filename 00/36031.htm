c.       Click on the Dependencies tab.

d.      Check if both the dependency service has started.

Remote Procedure call

Shell Hardware detection.

e.       If the Dependency services are not started, then start both the service and then start the Windows Image Acquisition service.

f.       Restart the computer and check if it helps.

 

Method 2:

Connect the Printer and Run the Automated Hardware Troubleshooter and check if it helps.

Hardware devices are not working or are not detected in Windows

http://support.microsoft.com/mats/hardware_device_problems/en-us

 

Method 3:

Uninstall and Re-install the latest Printer drivers from the computer manufacturer website.

Uninstall the printer software that you have installed in your computer. Uninstall or change a program http://windows.microsoft.com/en-us/windows7/Uninstall-or-change-a-program Restart the computer once the printer driver is uninstalled completely from the computer. Note: Make sure that the printer is disconnected from the computer.

 

After restart, Windows will automatically try to install the drivers through windows update. If this doesn’t happen, then check the printer drivers in the Manufacturer’s website and install the same.

 

Method 4:  If you are still unable to start the “Windows Image Acquisition” service, perform a system file check for any system file corruptions. To do so, follow these steps.

 

a.            Click Start, All programs and then Accessories. Click Command Prompt.

b.           In the Command Prompt window type “sfc /scannow” (without quotes) and then press Enter.

c.            After the scan has completed, close the command prompt window, restart the computer and check for the issue.
