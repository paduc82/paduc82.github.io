jason of oz say:

Is the DNS Client service running? It is responsible for the DNS Resolver Cache and, if it is running, if you're getting cache errors you could try stopping it to bypass the cache altogether for a test. If its not running, than the output you received from ipconfig is expected and can be ignored.

DNS Client isn't required to resolve hostnames via DNS, only if you want to use a local cache (and to handle primary/secondary DNS failover centrally rather than letting every app do its own thing).

I'm not aware of any registry settings that affect DNS lookups specifically outside of those that apply to the DNS Client service (which don't really matter if you've tried stopping it).
