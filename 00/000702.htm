jay freedman say:

In addition to Doug's explanation, you should be aware that turning the Design Mode button on or off has no relationship at all to whether the form is protected.

If you're using legacy form fields, then you must click the Restrict Editing button, check the box in the "Editing restrictions" area and select "Filling in forms", and click the "Yes, Start Enforcing Protection" button.

An alternative to this sequence is to go to File > Options > Quick Access Toolbar and add the Lock button (from the All Commands group) to the toolbar. That button toggles form protection on and off.

If you're using content controls, you should not go to the Restrict Editing pane or use the Lock button. Instead, select the entire form, click the Group button on the Developer tab, and click Group in the menu.

One other hint: You can use the Tab key to jump from one content control to the next,

doug robbins - mvp office apps & services (word) say:

It sounds like you are mixing up the use of Content Controls, which do not require that the document be Protected for Filling in Forms, with that form of protection, which is intended for use with Legacy FormFields.

While it is probably better to move to the use of Content Controls, you can still use Legacy FormFields in later versions of Word

To learn about the use of Content Controls, see the following and other pages on Greg Maxey's website:

a. user say:

The language information is in there in the document - in the XML, it's in the b:LCID element of each citation.

But the Mac version of Word doesn't let you set it.

AFAICS the only ways to do it are pretty unhelpful:
 a. go back to the Windows version, open the citation for editing, then modify the language
 b. save the document and modify the XML of the citation. However, this is more easily said than done. It's essentially a programming operation. I was able to do it using a simple example here by saving as .xml format, then finding the text <b:LCID> and modifying the language code that comes immediately after it. But AFAIK the .xml format is not supported and there is a risk in a real document that you might lose information by using it. So you would probably have to 
save in .docx format, unpack that (it's a ZIP file), find the Document Part that contains the citations, and edit that part.

Click File Tab in the menu. 

You should see a field headed "Information about <Document Name>"

This opens a number of options that provide some control over the document you are working on.

After you click on the FILE TAB, under it near the top should be an icon labelled "convert" and the the right the heading "Compatibility  Mode".  Click the convert icon once, then go back to you picture click on it again and the Background Removal Tab should now be there.

You can now edit and remove the background.

I worked this out by trial and error, and there is no instruction anywhere in the MS literature about this that I can find.

I assume MS is encouraging everyone to use the .docx formatting, that frustratingly is only compatible with MS products. 

It may be the case you are saving your documents in the older .doc format by default, which may be why the newer features are disabled.

Curiously, if you now repeat the steps above, and click on the FILE TAB again, this time there should be a different icon labelled Protect Document. 

Hope this helps.
