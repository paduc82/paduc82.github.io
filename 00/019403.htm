macropod say:

Both Doug's macro and mine employ Find/Replace. Indeed, what mine does could all be done using wildcards through the Find/Replace dialogue:

Find = ["“][!“”"]@[”"]

Replace = ^&

with the replace format set to bold.

Find = [\(][!\(\)]@[\)]

Replace = ^&

with the replace format set to italic.

Find = [\(“”"\)]

Replace = ^&

with the replace format set to not bold & not italic.

macropod say:

Try:

Sub Demo()
Application.ScreenUpdating = False
With ActiveDocument.Range.Find
    .ClearFormatting
    .Replacement.ClearFormatting
    .Format = False
    .Forward = True
    .Wrap = wdFindContinue
    .MatchWildcards = True
    .Replacement.Font.Bold = True
    .Replacement.Text = "^&"
    .Text = "[""“][!“”""]@[”""]"
    .Execute Replace:=wdReplaceAll
    .Replacement.Font.Bold = False
    .Replacement.Font.Italic = True
    .Text = "[\(][!\(\)]@[\)]"
    .Execute Replace:=wdReplaceAll
    .Replacement.Font.Italic = False
    .Text = "[\(“”""\)]"
    .Execute Replace:=wdReplaceAll
End With
Application.ScreenUpdating = True
End Sub

On a document with many matches to process, the above should run much more quickly than Doug's version. The potential downside is that other bolded/italicised quotes/parentheses will lose those attributes.

john w. vinson say:

This misleading error message can occur if you have a missing Reference. Type Ctrl-G to open the VBA editor and select Tools... References from the menu. If any of the references is marked MISSING, you'll need to fix it - either install the DLL file referenced or (probably better!) uncheck that line, compile the VBA project and fix any errors (e.g. by removing the call to the missing routine).

I would also suggest that it would probably be more efficient to use the standard SQL syntax IS NULL or IS NOT NULL rather than calling the IsNull() VBA function;  for example,

... [Asset_Qualification_Status] IS NULL, "UnScheduled", ...

You should also change all your LIKE clauses! LIKE lets you use wildcards such as * matching any string, ? matching any single character, # matching any digit; your criteria do not use any wildcards so the LIKE will behave exactly the same as =, only slower.

Edit: All of the above still applies, but the reason your LAST IsNull is failing is simply that you have a space between it and the parenthesis. IsNull(...) is recognized as a function; IsNull (...) is seen as a standalone term. Spaces are meaningful punctuation!
