    Virtual vs Physical Memory? - Microsoft Community

    ronald bagwell say:

    Virtual vs Physical Memory?

    I am running Windows 10. Physical memory [RAM] is 3 GB with 1.9GB available physical memory. Virtual memory is set at 5GB. Is this ratio of each type of memory ok for the Windows 10 set up? 

    ken blake say:

    All ratios of physical memory to page file size are wrong. One of the recommendations you often see is that the page file should be 1.5 times the size of physical memory. (Note that the following uses Windows XP for an example, but it pertains to all versions of Windows).


    So, using a Windows XP example, if someone has 64MB of RAM (the official minimum for Windows XP), he should have 96MB of page file? If your page file is that small, the computer will hardly be able to run any applications at all.

    And if you have 2GB of RAM (more than almost anybody with Windows XP needs), you should have 3GB of page file? That's almost certainly far more than is needed, and is just a waste of disk space.

    Read "How Big Should I Make the Paging File?" at 
    http://blogs.technet.com/b/markrussinovich/archive/2008/11/17/3155406.aspx

    It says "Perhaps one of the most commonly asked questions related to virtual memory is, how big should I make the paging file? There’s no end of ridiculous advice out on the web and in the newsstand magazines that cover Windows, and even Microsoft has published misleading recommendations. Almost all the suggestions are based on multiplying RAM size by some factor, with common values being 1.2, 1.5 and 2. Now that you understand the role that the paging file plays in defining a system’s commit limit and how processes contribute to the commit charge, you’re well positioned to see how useless such formulas truly are."

    The is no benefit in restricting the size of the page file. Just let Windows manage it.
