Word 2010 - labels cant insert a row on 1st line - Microsoft Community

brochlass say:

Word 2010 - labels cant insert a row on 1st line

I have set my labels up with the label size etc and a document has opened up with the gridlines for the labels.  All is well.  However, in the 1st box I want to insert a row so the actual label address isn't not hard against the top of the actual label.  I've tried insert but the whole document moves down a row.  When I jump to the label opposite on the RH side, it will let me insert a row and the cursor moves down to the next line.   What do I need to do to get this to work on the very first label?
suzanne s. barnhill say:
There are two factors involved here:
  1. If you press Enter at the very beginning of a table that is the first thing in a document (as is the case with labels), you insert a normal text paragraph above the table. This is for your convenience (since it allows you to add a title or other text before a table if you later decide it is needed) but obviously works against you here. If you type a space in the first cell before pressing Enter, you will avoid the problem.
  2. But pressing Enter to add space above the first line is not the best approach. A better approach is to either add Spacing Before to the paragraph (you can easily add 12 points Spacing Before by pressing Ctrl+0) or use Center vertical alignment in the table cell.
