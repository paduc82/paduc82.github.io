marvin ma msft say:

Hi Dave,

As a matter of fact, there're three kinds of folders in your mailbox: root folders, generic folders and search folders. The root folder is the top level folder in folder structure hierarchy, and it cannot be modified and removed. The IPM_Subtree folder is a root folder which stores some default folders (such as Inbox, Sent Items) and custom folders. For more details, you can read the article MAPI Folders.

So every incoming message will be delivered to IPM_Subtree folder at first. And somehow the email is wrongly stored in this folder. It could be caused by the conflicts between inbox rules. To view these vanished emails, you can follow the steps below to check if it works.

1. Select the user email account in Outlook client, and right click the email address to open Data File Properties.

2. Navigate to Home Page and uncheck Show home page by default for this folder. 

3. Then you can see these emails in the item list and just move them to the right place.

Besides, it would be better to check the inbox rules for this user. And if the issue recurs, you can try to create a new profile in Outlook to check if it helps.

Regards,

Marvin

ken sheridan say: The problem with using the Excel WORDAY function in Access is in passing the holidays argument into the function as this is normally a range in an Excel worksheet.  In Access holidays are normally represented by date/time values in rows in a table, or (as in the example below) by the absence of the holiday dates from a calendar table).

An efficient way of doing this sort of date arithmetic in a database is to first create an auxiliary calendar table, which can be a single column table of dates, or can include other columns to hold other information about each date. The example below uses a single column table, which excludes weekend and holiday dates.

To create a calendar table first add the following function to the database:

Public Function MakeCalendar_DAO(strTable As String, _
                            dtmStart As Date, _
                            dtmEnd As Date, _
                            ParamArray varDays() As Variant)
                           
    ' Accepts:  Name of calendar table to be created: String.
    '           Start date for calendar: DateTime.
    '           End date for calendar: DateTime.
    '           Days of week to be included in calendar
    '           as value list, e,g 2,3,4,5,6 for Mon-Fri
    '           (use 0 to include all days of week)

    Dim dbs As DAO.Database, tdf As DAO.TableDef
    Dim strSQL As String
    Dim dtmDate As Date
    Dim varDay As Variant
    Dim lngDayNum As Long
   
    Set dbs = CurrentDb
   
    ' does table exist? If so get user confirmation to delete it
    On Error Resume Next
    Set tdf = dbs.TableDefs(strTable)
    If Err = 0 Then
        If MsgBox("Replace existing table: " & _
            strTable & "?", vbYesNo + vbQuestion, _
            "Delete Table?") = vbYes Then
            strSQL = "DROP TABLE " & strTable
            dbs.Execute strSQL
        Else
            Exit Function
        End If
    End If
    On Error GoTo 0
   
    ' create new table
    strSQL = "CREATE TABLE " & strTable & _
        "(calDate DATETIME, " & _
        "CONSTRAINT PrimaryKey PRIMARY KEY (calDate))"
    dbs.Execute strSQL
   
    ' refresh database window
    Application.RefreshDatabaseWindow
      
    If varDays(0) = 0 Then
        ' fill table with all dates
        For dtmDate = dtmStart To dtmEnd
            lngDayNum = lngDayNum + 1
            strSQL = "INSERT INTO " & strTable & "(calDate) " & _
               "VALUES(#" & Format(dtmDate, "mm/dd/yyyy") & "#)"
              
            dbs.Execute strSQL
        Next dtmDate
    Else
        ' fill table with dates of selected days of week only
        For dtmDate = dtmStart To dtmEnd
            For Each varDay In varDays()
                If Weekday(dtmDate) = varDay Then
                    lngDayNum = lngDayNum + 1
                    strSQL = "INSERT INTO " & strTable & "(calDate) " & _
                        "VALUES(#" & Format(dtmDate, "mm/dd/yyyy") & "#)"
                    dbs.Execute strSQL
                End If
            Next varDay
        Next dtmDate
    End If
   
End Function

To create a table calWorkdays for instance for the next 10 years call the function like this:

MakeCalendar_DAO "calWorkdays", #2010-01-01#,#2019-12-31#,2,3,4,5,6

This will exclude Saturdays and Sundays.  Then delete the rows for public or concessionary holidays as and when these become known.

Next add the following function to the database:

Public Function AddWorkdays(dtmFrom As Date, lngDays As Long) As Date

    Dim dbs As DAO.Database
    Dim rst As DAO.Recordset
    Dim strSQL As String
   
    strSQL = "SELECT calDate " & _
        "FROM calWorkDays As CW1 " & _
        "WHERE (SELECT COUNT(*) " & _
        "FROM calWorkdays AS CW2 " & _
        "WHERE caldate " & _
        IIf(lngDays > 0, ">", "<") & "= #" & _
        Format(dtmFrom, "yyyy-mm-dd") & "# " & _
        "AND caldate " & IIf(lngDays > 0, "<", ">") & _
        "= CW1.caldate) = " & Abs(lngDays) + 1
       
    Set dbs = CurrentDb
    Set rst = dbs.OpenRecordset(strSQL)
   
    AddWorkdays = rst.Fields("calDate")
   
End Function

To add 12 working days to today's date for instance you'd call it with:

=AddWorkDays(#2010-08-16#,12)

To subtract 12 days:

=AddWorkDays(#2010-08-16#,-12)
