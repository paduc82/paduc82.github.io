lava k say:

Hi pclf1,

 

Welcome to Microsoft Answers Community Site.

 

Is the computers on a wired or a wireless network?

 

I would suggest you to try out these options:

 

1) No firewall or security software blocks sharing.

The issue may be caused also by your security program such an anti-virus or a Windows Firewall /Defender which might have conflicting settings was causing the network slow down . I suggest that you temporary disable the anti-virus and firewall and check the result again. You would need to turn on the security programs again which you had disabled.

 

2) Open Control Panel from the Start menu and type Offline in the Control Panel search box. Click on the green Offline Files link and click the disable button.


3) Disabling Remote Differential Compression. To do this, open Control Panel>Add Remove Programs>Turn off Windows features, uncheck Remote Differential Compression.

 

4) Disable auto tuning: in the start search box type netsh int tcp set global autotuninglevel=disabled and when the application appears at the top of the start window, right click on it and, from the drop down menu, select 'run as administrator'.

 

5) If the issue persists due to IPv6 deployed you might need to follow the steps mentioned in the KB article: How to disable certain Internet Protocol version 6 (IPv6) components in Windows Vista, Windows 7 and Windows Server 2008

 

6) Check the hardware such as NIC and switch settings. Make sure the speed duplex are matching

 

7) Using Windows Network Diagnostics event logs to solve network problems

 

Let me know if that work. Good luck!

Hope this information is helpful.
