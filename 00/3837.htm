a. user say:

If you are doing a mailmerge, thereis a specialised set of facilities in Word that might allow you to avoid coding altogether.

Otherwise, let's assume that the workbook is closed when you run your macro, the workbook is named as in the code blow, the cells you want to test are i=on Worksheet "Sheet1", the cells are D3 and E5, and you want to insert a couple of pieces of text at the end of the document if the cell values are true. You can use code in a Word VBA module along the folowing lines:

  workbookFullname As String, _
  worksheetName, _
  cellRow As Long, _
  cellColumn As Long) As Variant
Const xlProgID As String = "Excel.Application"
Dim bwbOpened As Boolean
Dim bxlStarted As Boolean
Dim xl As Excel.Application
Dim wb As Excel.Workbook
Dim wbi As Excel.Workbook
Dim ws As Excel.Worksheet
bxlStarted = False
On Error Resume Next
Set xl = GetObject(, xlProgID)
If xl Is Nothing Then
  Err.Clear
  bxlStarted = True
  Set xl = CreateObject(xlProgID)
End If
If xl Is Nothing Then
  MsgBox "Could not contact or start Excel. Error " & _
    CStr(Err.Number) & _
    ": " & Err.Description
  Err.Clear
  getExcelCellValue = False
Else
  If Not bxlStarted Then
    bwbOpened = False
    For Each wbi In xl.Workbooks
      If UCase(wbi.FullName) = UCase(workbookFullname) Then
        Set wb = wbi
        Exit For
      End If
    Next
  End If
  If wb Is Nothing Then
    bwbOpened = True
    Set wb = xl.Workbooks.Open(workbookFullname)
  End If
  If wb Is Nothing Then
    MsgBox "Could not access the specified workbook. Error " & _
      CStr(Err.Number) & _
      ": " & Err.Description
    Err.Clear
    getExcelCellValue = False
  Else
    Set ws = wb.Worksheets(worksheetName)
    If ws Is Nothing Then
      MsgBox "Could not access the specified worksheet. Error " & _
        CStr(Err.Number) & _
        ": " & Err.Description
      Err.Clear
      getExcelCellValue = False
    Else
      getExcelCellValue = ws.Cells(cellRow, cellColumn).Value
    End If
  End If
End If
Set ws = Nothing
If Not (wb Is Nothing) Then
  If bwbOpened Then
    wb.Close xlDoNotSaveChanges
  End If
  Set wb = Nothing
End If
If Not (xl Is Nothing) Then
  If bxlStarted Then
    xl.Quit
  End If
  Set xl = Nothing
End If
End Function

Sub testGetExcelCellValue()
Const wbFullname As String = "D:\test\2017111601 test excel truefalse\book1.xlsx"
Const wsName As String = "Sheet1"
Dim cell34 As Boolean
Dim cell55 As Boolean
Dim r As Word.Range
cell34 = CBool(getExcelCellValue(wbFullname, wsName, 3, 4))
cell55 = CBool(getExcelCellValue(wbFullname, wsName, 5, 5))
If cell34 Then
  Set r = ActiveDocument.Content
  r.Collapse Direction:=WdCollapseDirection.wdCollapseEnd
  r.InsertAfter "Cell D3 is TRUE"
  r.InsertParagraphAfter
End If
Set r = Nothing
If cell55 Then
  Set r = ActiveDocument.Content
  r.Collapse Direction:=WdCollapseDirection.wdCollapseEnd
  r.InsertAfter "Cell E5 is TRUE"
  r.InsertParagraphAfter
End If
Set r = Nothing

End Sub

Notes:
 a. in this case we are using "Early binding", which means that in the module in Word, you need to go to VBE's Tools->References option and make a reference to the Microsoft Excel. I leave you to find out more about that., but if you choose to use late binding, please note that you will need to replace "xlDoNotSaveChanges" by its value, which is 2.

 b. If Excel is not running, this code will make Word start and close Excel for every cell you want to test. Obviously, you can rework the code so that you only start Excel once (as necessary) and retrieve all the values you need before closing it again.

 c. This code does not inspect the type of the cell to ensure that it is a boolean. You might want to do that differently in your code.

The 30 day waiting period can not be waived.

What should I know about the 30-day waiting period?

Even when you choose to delete your out-of-date security info, MS don't fully remove it for 30 days. This waiting period helps to deter hackers from updating your security info and gaining access to your account without your knowledge.

After 30 days, the user of the account will be able to access the security settings and update his/her preferred information.

But the waiting period doesn't mean you're locked out of your Microsoft account.

Check to see if you can still access your emails via https://mail.live.com.
