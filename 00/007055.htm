basilides say:

Find Cells (3 in a row) Each with 1 Forward Slash ( / )

Greetings!

In Col. F, of a very long EXCEL 2010 spreadsheet, I want to find using a simple macro cells that contain forward slashes, three in a row, as follows:

haaadkdk
ba/ktktktk
baa/ltltlt
b/dldldld
kfkfkkkdkdkdkd

When the macro is run, it will go down Col. B, and when it comes to three rows of cells (and only three, not two), each of which contains one forward slash (as in the bold-face font example, above), the macro will stop, then I will do some editing, and then the macro will be resumed in the place where it was stopped (the current cursor position).

I know it is vacation season, but I would deeply appreciate any help in this matter.



rich007a say:

Try this:

 

Dim lnStartRow As Long
Sub FindThreeRowsOfWithASlash()
    Dim rng As Range, c As Range
   
    If lnStartRow = 0 Then lnStartRow = 1
    Set rng = Range("F" & lnStartRow & ":F" & Rows.Count)
    For Each c In rng
        If c.Value = "" Then Exit Sub
        If InStr(c.Value, "/") > 0 Then
            If InStr(c.Offset(1, 0).Value, "/") > 0 Then
                If InStr(c.Offset(2, 0).Value, "/") > 0 Then
                    c.Select
                    lnStartRow = c.Row + 3
                    Exit Sub
                End If
            End If
        End If
    Next c
End Sub

Note that the first line "Dim lnStartRow As Long" must appear before any other Subs in your module.  It is a variable being declared with  module scope so it's value is stored between runs.  This will enable you to re-run the macro starting at the next line down.

Note also that if you have say 4 consecutive rows containing a / it will only pick up the first 3 as a block.  Then it will restart at the 4th but since the next two rows don't contain a / it will be ignored.

Hope that helps.

Since Chrome isn't fully integrated with the Windows security features as Microsoft's Edge and Internet Explorer browsers are, the check of a downloaded file must wait until that process has completed before any scanning of the file can begin.

Everything you describe sounds reasonable unless the scanning process is taking minutes to complete.  If that's the case, I'd first question whether there's anything else installed trying to perform a similar file scan at the same moment, such as a Chrome plug-in or 3rd-party anti-malware program that also contains active protection.

Typically such large lags are due to either undesired interactions between competing security products or possibly pre-existing malware that's yet to be detected.

If there was a 3rd-party security application either included when the PC was purchased or added soon after, this should be the first item you investigate and remove using any tools the developer of that app may provide to fully cleanup remnants of their application.

Hi Robert ... I'm sorry you're having this problem with your file.

PowerPoint files will always look like gibberish in Notepad because they're actually ZIP files.  Try this, though ... IF the file's originally from PPT 2007 or later and IF it's not badly corrupted, you may get lucky:

Make a copy of the file.

Rename the copy by either changing its file extension to .ZIP or just tacking a .ZIP extension onto the end of the existing file name.

Now double click the newly renamed ZIP file to open it.

Doubleclick the "ppt" folder inside the ZIP.

Doubleclick the "notesSlides" folder inside the "ppt" folder.

Now, for each slide in your presentation that has notes, you'll find a "notesSlide?.xml" file, where ? represents a number.

Doubleclick each "notesSlide?.xml" file in turn and in it you'll find:

"Notes Placeholder 2" and a bit below that, you'll see your notes text. Copy/paste it into the note pane of the corresponding slide in PPT.
