See the snip below. I'm not sure how you intend to implement this. The formula returning the range is in D2 and is this ARRAY formula, see below for how to enter an array formula.

=ADDRESS(MATCH(C1,A1:A20,0),1)&":"&ADDRESS(MATCH(1,1/(A1:A20=C1)),1)

This is an array formula which must be entered by pressing CTRL+Shift+Enter
and not just Enter. If you do it correctly then Excel will put curly brackets
around the formula {}. You can't type these yourself. If you edit the formula
you must enter it again with CTRL+Shift+Enter.

fin_modler say:

Identifying the ranges is fairly simple and putting the range as text in a cell is also fairly simple. However, if you want to put individual cell addresses into a list, it is less easy (easy if you do it with VBA.)

Are you looking for

A1:A3 or

A1

A2

A3

The former is very simple (assuming you do not have a header for Column A):

=ADDRESS(MATCH(value1,A:A,0),1)&":"&ADDRESS(MATCH(value1,A:A,0)+COUNTIF(A:A,value1)-1,1)

or

=CELL("address",OFFSET($A$1,MATCH(value1,A:A,0),0))&":"&CELL("address",OFFSET($A$1,MATCH(value1,A:A,0)+COUNTIF(A:A,value1)-1)

To get a list, you can try say in D1:D20, put in 1,2,3...20 or however many you need

In E1, you put in =IF(D1>COUNTIF(A:A,value1),"",ADDRESS(MATCH(3,A:A,0)+D1-1,1))

copy down to E20 or beyond.

toddallcock say:

Windows Phones can only sync address books with accounts that use the Exchange ActiveSync protocol, like Exchange accounts, Live/Hotmail, and Gmail.  AOL uses POP or IMAP, older standards WP doesn't use for address books or calendars.  (AOL also uses a variant of the old SyncML standard for their "AOL Sync" service to sync calendars and contacts, but WP doesn't support that either.)

 

The best solution would to be to export the AOL address book to a .csv file, (on the AOL webpage in Contacts, it's under "Tools",) then create a Hotmail/Live or Gmail account (it wouldn't need to be used for email, just calendars and contacts) and import the .csv file to that new account and add it to the phone.
