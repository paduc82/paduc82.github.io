suzanne s. barnhill say:

In this specific situation, instead of restarting numbering at 1 in section 2, set numbering to start at 0 in section 1; in section 2, set the numbering to "Continue from previous section." Enabling "Different First Page" will allow you to omit the number (1) on page 2 and will start numbering at 2 on page 3. Look for this code, i think it can be usefull. You must include the Microsoft Scripting Runtime (c:\windows\system32\scrrun.dll).

Public objfolder As Folder
Public objfds, objsubfds As Folders
Public objFiles As Files
Public objFile As File
Public s As String

Sub DelFiles()
    Dim sDir As String
    Dim sTipoFile As String

    Dim objFSO As New FileSystemObject
   
    sDir = "c:\tmp"     'this is the directory in which i want find files
    sTipoFile = ".zip"     'i specify a part of the name of the files that i want to find for and drop
    Set objFSO = New FileSystemObject
   
    'i control if my folder exists
    If objFSO.FolderExists(sDir) Then
        DelFileAndGoToSub objFSO, sTipoFile, sDir
    End If

End Sub

Function DelFileAndGoToSub(ByRef objFSO As FileSystemObject, ByVal sFileExt As String, ByVal sSubFolder As String)

    Set objfolder = objFSO.GetFolder(sSubFolder)
    'i search for my files in the directory give
    For Each objFile In objfolder.Files
        sFileName = objFile.Name
        If Right(sFileName, Len(sFileExt)) = sFileExt Then
            objFile.Delete
            MsgBox sFileName & " delete"
        End If
    Next
    'i search for subfolders
    Set objsubfds = objfolder.SubFolders
    For Each objfds In objsubfds
        s = objfds.Path
        DelFileAndGoToSub objFSO, sFileExt, s
    Next

End Function

If you give the main directory and the extension file, this code search and delete the file with the specify extension and search and delete also the file required in sub folders.

Hi Wayne,

Private groups are not hidden. Their name, privacy type, description, e-mail address and membership list are all visible to any other members of your organisation.

Anyone within your organisation can e-mail private groups and can receive replies from the group.

It is only the content of the group - any conversations and files, that are not accessible unless you are a member.

Now there is a HiddenFromAddressListsEnabled property that you can access via PowerShell, but it is not clear if this currently works or is supported by Microsoft.

Right now it doesn't sound as if the Office 365 unified groups provide the feature set that you are after. It's probably worth you sending feedback via the "Suggestion Box" option, that's under the help section in Outlook Web App.
