andreas killer say:
The answer to your question is: Refer to the right objects.

Okay, that doesn't help if you don't know what it means... so let us walk through a few samples. 

Sub Test()
  Range("A1").Value = 1
End Sub


That writes a 1 into A1, but where is A1? In which sheet, in which file? If you don't specify any references before RANGE means it writes into A1 of the active sheet in the active file. This references are created by the Excel object model automatically, means you execute this: 

Sub Test()
  ActiveWorkbook.ActiveSheet.Range("A1").Value = 1
End Sub


To refer to the right workbook, there are 2 other ways: 

Sub Test()
  'Refers to the file where this code is located in
  ThisWorkbook.ActiveSheet.Range("A1").Value = 1
  'Refers to the file "MyFile.xlsm"
  Workbooks("MyFile.xlsm").ActiveSheet.Range("A1").Value = 1
End Sub


And if you want to refer to a special sheet, you can use the SHEETS collection of the workbook object: 

Sub Test()
  'Refers to the 1st sheet in this file
  ThisWorkbook.Sheets(1).Range("A1").Value = 1
  'Refers to the sheet "MySheet" in this file
  ThisWorkbook.Sheets("MySheet").Range("A1").Value = 1
End Sub


That's it. if you refer to A1 that way it doesn't matter which sheet or file is active.

For completeness:

a) There is a SHEETS and a WORKSHEETS collection in each workbook, the SHEETS collection includes all kind of sheet (e.g. also chart sheets). The WORKSHEETS collection has only worksheets.

b) All above is only true if the code is located in a regular module. If the code is located in a worksheet, the RANGE object refers always to the sheet. But you can also refer to other sheets or files the way I've shown.

Any questions?

Andreas.
