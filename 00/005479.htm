Hi ostrenga4, sorry to hear about your experiences.

 

When installing Nokia Ovi Suite to PC, Windows makes "system restoration points", which are available to restore operating system to previous condition, if something goes wrong, for example in driver installation.

 

You can change install location of Nokia Ovi Suite with following:

- Download Nokia Ovi Suite install package from Nokia Ovi Suite pages

- Start Windows Command prompt

- From Command prompt, navigate to folder containing Nokia Ovi Suite install package, which you downloaded

- Start Nokia Ovi Suite install package with command line "Nokia_Ovi_Suite_webinstaller.exe /TARGET="D:\TestDir\Test"

- This results, that Nokia Ovi Suite installer creates folder D:\TestDir\Test (if that doesn't exist) and installs Nokia Ovi Suite there.

- Unfortunately, Windows Installer still keeps installer cache files and system restoration points in C: drive (or whatever your system drive is)

 

As you can see, this is possible, but it is very much recommend to have enough space for installing software into system drive.

john - project say:

Dallas,

I though someone might jump in with some sage advice by now but since they haven't let me at least give you my thoughts.

First of all, do you have SP2 installed? If not, install it. It won't help you with your issue but it does fix many things that you likely will run into while using Project. I also suggest you install the June 2013 cumulative update since it was released after SP2 and also fixes several issues. I wouldn't worry about installing the latest cumulative update (June 2014), but if you already have, that's okay.

Keep in mind that task dependencies are just that, task dependencies and are not related to resource availability/dependencies. If indeed certain tasks are related, and most should be, than a link is appropriate. However, there may be some cases where a particular task or tasks can be performed anytime during the life of the project but are not directly related, dependency-wise, to other tasks (e.g. impact study that is one of the many outputs of the project).

You mention resource groups but it's not clear if you understand that Project can be set up to use resources in group fashion. For example, you can create a single Offshore Programmer group with a max unit level of 500% (5 people) and an Engineering group with a max unit level of 100% (1 person). That will allow you to assign the resources as group instead of by individual name. But bear in mind that a resource group considers each individual resource in that group to have the same capability, so by assigning the programmer group at 200% to a task, you are indicating that any two of the individual resources that will be doing the work is equivalent to any other two members of the group.

Project does not have the resource optimization capability that you want. The resource leveling feature can be used to help resolve resource overallocation and the Team Planner can be used to facilitate resource assignments but neither can be set up to optimize resource utilization, that is a manual task best left up to the project manager. Why? Because there are so many variables that come into play, it is difficult to develop an efficient algorithm. You could potentially create such an algorithm based on your specific requirements and develop that into a VBA macro.

Outlook must contact what it believes is the outgoing mail server, perform the necessary protocol exchanges, send the message, then receive an acknowledgement from the outgoing server that it has accepted the message for delivery to the next-in-line mail router. Once Outlook has received the acknowledgement, it moves the message to Sent Items.

Thus, seeing the message in the Sent Items is assurance that the message has moved successfully to the next step in the delivery process. If the message never arrives, the problem lies after Outlook. One common cause of this is an antivirus scanner pretending to be the mail server wedged between Outlook and the real server, where the scanner drops the ball.

teylyn say:

Hello,

You may want to get rid of the merged cells and just use formatting with borders to indicate what belongs together. With merged cells, some of the following formulas won't work. 

If the range starts in cell A1, then the cell with the word "July" is in B1. Based on that, you can use this formula in cell R2 to get the current month (or type in the date manually)

=MAX(IF(A2:Q2<TODAY(),A2:Q2))

this is an array formula and must be confirmed by hitting Ctrl+Shift+Enter.

In cell R3 you can get the $ values for that month with this formula

=INDEX($B3:$Q3,MATCH($R$2,$B$2:$Q$2,FALSE))

In cell S3 you can get the percentage with 

=INDEX($B3:$Q3,MATCH($R$2,$B$2:$Q$2,FALSE)+1)

Copy both formulas down.
