ossiemac say:

Need to use "Get external data" rather than just open the file.


  • Select cell A1 on a blank worksheet.
  • Select Data ribbon
  • Select "From text" icon (in the Get External data block.)
  • Select the required csv or tsv file and click Import. (May need need to change file type towards bottom right of dialog box to required file type. (Use "All files (*.*)" to see the tsv file.)
  • Select Delimited (Probably already selected)
  • Check box for Comma delimiter for csv or Tab delimiter for tsv  (only check one) and click Next

  • Popup dialog box shows requesting location of the data. If you selected A1 to start then it should default to that otherwise select A1 and then click OK.

How to mask a SSN field so that only the last 4 digits show? - Microsoft Community

rdw1260 say:

How to mask a SSN field so that only the last 4 digits show?

I have a membership database that I want to mask the first five numbers of a members SSN when the form opens. How do I do this and still be able to perform a search using all 9 numbers?

Thanks,

Robert

ken sheridan say:

Use an unbound text box control in the form with a ControlSource property of:

="*****" & Right([SSN],4)

It won't be possible to edit this, of course, or input a new SSN into the control in a new record in the form, but you could get round this by superimposing the unbound control on top of a bound control with [SSN] as its ControlSource.  If the unbound control is named txtSSNFormatted and the bound control txtSSN, you can then hide show them as appropriate in the form's Current event procedure with:

    If Me.NewRecord Then
        Me.txtSSN.Visible = True
        Me.txtSSN.SetFocus
        Me.txtSSNFormatted.Visible = False
    Else
        Me.txtSSNFormatted.Visible = True
        Me.txtSSNFormatted.SetFocus
        Me.txtSSN.Visible = False
    End If

When the form is at a new record the bound control will be visible and can be edited, otherwise the unbound control will be visible.

If you wish you can hide the full SSN immediately after it's entered by putting the following in the bound txtSSN control's AfterUpdate event procedure:

    Me.txtSSNFormatted.Visible = True
    Me.txtSSNFormatted.SetFocus
    Me.txtSSN.Visible = False

The values in the underlying SSN column in the table won't change of course, so you can still search on the column.

 

Ken Sheridan, Stafford, England
