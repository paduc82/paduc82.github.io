tim liu msft say:

Hi Thamsanqa,

Based on my test, when there are some columns which are required data, it will get different behaviors with the same library when using classic UI and modern UI.

In classic UI

It will first upload the file and give us a pop-up for inputting column value. Now the file is checked out. After inputting the value, click Check In. The file is checked in and can be seen by other users.

In modern UI

It will first upload the file and give us a pop-up at right of the library. The file is not checked out. So, even we don't input column data, it can be seen via others.

Based on my test, it's not feasible to change this behavior in modern UI to the same as classic UI. You could change the library to classic UI if you really want to use this feature. Meanwhile, you could submit the feedback via SharePoint Uservoice. This is the best way to influence our products and services.

bob jones aka: cybertaz say:
You can't - the overlapped area does not constitute a separate shape, so it has no text box of its own.

One option is to create an independent shape [just a rectangle will do] to contain the number. Position the rectangle in the location you want it & format it for No Fill & No Line. You can select both the diagram & the shape to Group them once all is set as you wish - Ungroup if you need to make changes later.
