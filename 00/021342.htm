First I'm going to say that the formula presented below ONLY works with names like

"John D. Smith", "John D Smith", "John David Smith" and/or "John Smith"

The "Prince"s, "Cher"s and "Madonna"s of the world are out, as are all of the

John Jacob Jingleheimer Smiths (4 names!) and the

=LEFT(A2,FIND(" ",A2)-1) & " " & IFERROR(RIGHT(A2,LEN(A2)-FIND(" ",A2,FIND(" ",A2)+1)),RIGHT(A2,LEN(A2)-FIND(" ",A2)))

That is going to give you a "First Last" name pairing for each.  Fill the formula down each sheet as far as you need to go.

=COUNTIF([Book2.xlsx]Sheet1!B:B,X2)

               Column B is the column in the other book where you put the long formula I provided above, and

               Column X is the column in "this" workbook where you put the long formula I provided.

 

When a match is found in both books, the result of the COUNTIF() will be non-zero, showing how many times the name on the row in this book appears in the list in the other workbook.

jllatham say:

I don't think you can do it with the original information and a VLOOKUP().  A VLOOKUP() for "John Brown" is never going to match with "John D. Brown" - that was the whole reason for coming up with the formula to reduce them to first with last names only.

 

If you inserted a new column B next to the current list of names with middle names/initials being in column A, after you've set up the formulas and all, then you could use a VLOOKUP() based on the new entries. 

 

Hopefully you're not going to run into too many cases of several individuals with the same first+last name combinations, such as  John D. Brown, John Brown, and John Q. Brown.  That's one reason the COUNTIF() will be of help: if it shows a number other than 1, then you know you have such a situation, and that a VLOOKUP() is going to stop at the first match in the list and never see #2, #3, etc.

 

I think COUNTIF() is better if you're just looking to see who's in one list that is also in the other.  By using it on both sheets you can easily identify not only who is in both lists (the results will be non-zero), but also who in one list is not in the other (result will be zero).  So for the workbook with 700 records in it, you would expect to see about 165 zero entries since the other list only has 535 names - number could vary from that, just depends on the composition of the two lists.

ronrosenfeld say: Why do you need a VLOOKUP if all you want to do is determine if a person in File A also exists in File B?

If all you need is an answer to that question, then try this formula:

This formula must be **array-entered**:

=OR(ISNUMBER(SEARCH(LEFT(A2,FIND(" ",FileA!A2)-1) & "* " & TRIM(
RIGHT(SUBSTITUTE(FileA!A2," ",REPT(" ",99)),99)),FileB!$A$2:$A$n)))

substitute appropriate workbook/worksheet references for FileA and FileB in the above formula.  It will return TRUE or FALSE, depending on whether A2 (first and last name) exists in FileB.

It is NOT 100% foolproof.  In particular, something like Johnathan Smith will match John X Smith, but it should give you a start.

    If your units of measurement are imperial rather than metric Access will automatically convert the last one to inches.  The important thing is that the dimension is zero to hide the first column.

    The search is executed when an item is selected, by the following code in the combo box's AfterUpdate event procedure:

        Const MESSAGETEXT = "No matching record"
        Dim ctrl As Control
        
        Set ctrl = Me.ActiveControl
        
        If Not IsNull(ctrl) Then
            If ctrl = 0 Then
                ' go to new record and move focus to FirstName control
                DoCmd.GoToRecord acForm, Me.Name, acNewRec
                Me.FirstName.SetFocus
            Else
                With Me.RecordsetClone
                    .FindFirst "ContactID = " & ctrl
                    If Not .NoMatch Then
                        ' go to record by synchronizing bookmarks
                        Me.Bookmark = .Bookmark
                    Else
                        MsgBox MESSAGETEXT, vbInformation, "Warning"
                    End If
                End With
            End If
        End If

    All you'd need to change in this code are the names of the ContactID column to your ID column's name, and the FirstName code to whatever control you want to receive focus first when you move to an empty new record.

    While you can use the wizard to do something similar (without the option to move to a new record), you'll learn a lot more by 'hand crafting' it yourself.  Wizards are fine for saving time, but no substitute for understanding.
