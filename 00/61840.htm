cmdrkeene say:

This is caused by faulty shell extensions.  A shell extension is a program that adds additional options to right-click menus for certain icon types. For example, after you install an antivirus program, your icon right-click choices might be updated to include a "scan this file for viruses" option.  It's also very common for ZIP/RAR file compression tools, and Notepad programs, to add themselves as additional right-click options.

If those extensions suffer problems, it can cause the interface to crash and reload when you right-click.

Solutions:

The easiest solution by far is to roll back whatever caused the issue. If you can remember installing anything recently, remove whatever it was.  If that doesn't work (or you aren't sure where to start), use option #1 here instead.

If that doens't work, it gets a little harder.  You can use a tool like Microsoft AutoRuns to see what shell extensions are registered (set to load), and disable them one-by-one until you find the one causing the right-click crash.

tryx3 say:

This has now been published as a forum article - see Relocating user folders in Windows 7 and Windows 10 - Forum article

Relocating user folders in Windows 7 & Windows 10

1  This is the standard way to do the job.  I have included the procedure for relocating all the user folders, including the IE Downloads folder, rather than just the Desktop folder Marko asked about. 

  • I refer to your username as YourUserName all the way through.  Just substitute your own username wherever you read YourUserName.  Your username is shown by the name of your folder in C:\Users\YourUserName
  • I refer to the target drive as K: all the way through just to avoid endlessly repeating E: or F: or ... whatever is the drive letter for the drive you want to move your user folders to.  Just substitute your chosen target drive letter wherever you read K:. 

1.1  Create a K:\Documents folder.

1.2  Check permissions for K:\Documents by right-clicking on it, selecting Properties then the Security tab. 

  • The entries will probably be okay but it is a lot easier to check & change them now rather than later. 
  • The first time I relocated my user folders I left all the permissions as they were and did not suffer at all. 
  • You might well read these 1.2.1-6 permissions notes then do the same as I did - decide to consider the subject later.

  • The entry Administrators is a group name that includes all the Admin accounts that you create on the computer.
  • The entry Authenticated users is a group name that includes all the users who can log in to the computer [both Admin users & Standard users] except for the special-purpose Guest account type {use of which is not recommended and is generally only set up by advanced users with complex multi-user needs].
  • You will normally want to allow both Administrators & Authenticated users to have these permissions {Windows 10 wording used, Windows 7 wording varies a bit} - Full control, Modify, Read & execute, List folder contents, Read, Write.  You should find that these are the permissions already set up so you will not need to change anything.
  • If the existing settings do not suit you then click on the Advanced button and change them. 
  • I urge you not to alter any of the entries for the Administrators group as that can have serious implications if you ever need to use an Admin account to sort out problems on your computer.
  • Changing permissions can get confusing.  If, for example, you wanted to remove permissions just for an individual Standard [non-Admin] user called Fred but to retain them for your own account YourUserName, you would have to remove permissions for the Authenticated users group as a whole and then add selected ones just for Fred and another set of selected ones for YourUserName [unless you are always going to be happy for your own account to remain an Admin account in which case you could just rely on having left all Administrators group permissions as they were].

1.3  Copy the contents of C:\Users\YourUserName\Documents into K:\Documents [just its contents not the C:\Users\YourUserName\Documents folder itself]. 

  • When you are happy that the copying task has completed correctly, delete the contents of C:\Users\YourUserName\Documents. 
  • Copying then deleting originals is slightly better than moving because it imposes the intended set of K:\Documents permissions onto the folders & files as they are copied.  It is possible for old & possibly unintended permission to slip through if you just move everything.
  • Note that Windows will offer to move the files across for you when you change the Location tab setting in para 1.4 but it can get really awkward if it fails so it is safer to do this yourself beforehand.

1.4  Right-click on the now-empty C:\Users\YourUserName\Documents folder, select Properties then select the Location tab. 

  • Overwrite its current entry with K:\Documents.  Windows will offer to move the now-empty folder, accept the offer.
  • If you try to set a user folder to the same location that another user folder has been set to then Windows will warn you but will then allow you to do it.  Windows will also warn you that you will not be able to relocate them in the future [but you will be able to do so using the procedure given in section 2 of this article]. 

1.5  Repeat the 1.1 - 1.4 procedure for these other user folders [I have shown the folders I moved them to just by way of example. It suited my intended backup plan to separate some items into a different folder called DataSettings.]

  • Music                K:\Documents\Music,
  • Pictures             K:\Documents\Pictures,
  • Videos               K:\Documents\Videos,
  • [IE] Favorites      K:\DataSettings\Favorites,
  • Searches            K:\DataSettings\Searches,
  • Links                  K:\DataSettings\Links.

[Windows 10 - Links is the "Quick access" entry that appears at the top of the File explorer Navigation pane.]

[Windows 7 - Links is the "Favorites" entry that appears at the top of the Windows explorer Navigation pane.]

1.5  IE Downloads folder 

  • Open IE then go to Tools [the gearwheel icon on the top right], View downloads, Options [in the very bottom-left corner and very easy to miss]. 
  • Overwrite its current entry with your chosen target folder.

1.6  [Windows 10]  Screenshots folder

An additional user folder exists in Windows 10 - the Screenshots folder and my advice about using this facility and relocating its folder is in Capturing screen contents - the PrntScrn key, Win-Shift-S, the Snipping tool and Screenshots - Forum article

- - - - - - - - - - - - - - - - - - - - -

2  Correcting relocation errors using the Registry

  •  It might be possible to alter the Location settings in the Registry manually in the first place instead of using the Location tab procedure in section 1 but I have only ever looked at it afterwards to fix errors. 
  • Since this section involves editing the Registry the usual warnings apply [back up everything, make a system image, test the System repair disk / Windows installation disk so you know you can boot up with something to be able to restore the image if Windows won't co-operate].  I have included what I regard as the minimum backup requirement in para 2.4 below.

2.1  Open RegEdit [C:\Windows\regedit.exe] - I always open it by right-clicking and selecting Run as Admin but I don't think that is essential for the particular keys used in this task. 

2.2 [Windows 10 Ver 1703 & later only]  Paste this into the RegEdit address bar [in place of its existing entry] to get straight to the right place

HKEY_CURRENT_USER\Software\Microsoft\Windows\CurrentVersion\Explorer\User Shell Folders

2.3 [Windows 7, Windows 10 Ver 1607 & earlier]  Pick your way through the Registry hierarchy manually to get to

HKEY_CURRENT_USER\Software\Microsoft\Windows\CurrentVersion\Explorer\User Shell Folders

2.4  My minimum Registry backup recommendation  In the Navigation pane on the left, select HKEY_CURRENT_USER\Software\Microsoft\Windows\CurrentVersion\Explorer\User Shell Foldersthen right-click on it & select Export. 

  • Give it whatever name you want, I would use CurrentUser-UserShellFolders. 
  • Its default format is .reg and I urge you to accept that.
  • Export it to a convenient folder such as your desktop.
  • If you make a mess of changing this part of the Registry then you will be able to use this Exported file to restore it to its original state [see section 4 below].
  • If you want to read the contents of the .reg file then you can either right-click on it & select Edit or open Notepad and drop the .reg file onto it.  It is a lot easier to read a .reg file's contents this way than it would be if you had exported it as a .txt file.

2.5  Check the Data column for the Personal entry in the right-hand side.

  • Personal means the Documents / My documents entry; most of the other entries have sensible names.
  • If you need to change the Data entry, double-click on Personal then overwrite its Value data entry with the path you want.
  • Entering a path that does not exist or to which your user account does not have access can cause confusing results so it is worth using File explorer / Windows explorer to go to the intended folder then copying its full path across from the explorer's address bar into the value data box.

2.6  Check the values of these other user folders and use the same procedure to change them if necessary.

  • Music [shown as My music],
  • Pictures [shown as My pictures],
  • Videos [shown as My video],
  • Links [shown as {bfb9d5e0-c6a9-404c-b2b2-ae6db6af4968}],
  • Searches [shown as {7d1d3a04-debb-4115-95cf-2f29da2920da}],
  • [Windows 7 only] IE downloads [shown as {374DE290-123F-4565-9164-39C4925E467B}]
  • [Windows 10 only] Screenshots [shown as {B7BEDE81-DF94-4682-A7D8-57A52620B86F}] - this entry only exists once Screenshots has been used at least once

2.7  Use the procedure in para 2.4 above to save a copy of the new entries.  I suggest calling the exported file something like CurrentUser-UserShellFolders-Amended.reg

2.8  [Windows 10 only]  The Registry entry for IE downloads is only created by the act of changing the entry using the normal method given in para 1.5 above.  It can thereafter be changed in the Registry.

  • This entry is created in HKEY_CURRENT_USER\Software\Microsoft\Internet Explorer\Main so paste that into RegEdit's address bar to get straight there [Ver 1703 & above only] or browse to it manually.
  • Export this key's initial state using the procedure in para 2.4 above.
  • Check the Data column for the Default Download Directory entry in the right-hand side.
  • If you need to change the entry then use the procedure given in para 2.5 above.
  • If you change its entry then export its final state using the procedure in para 2.4 above.
  • If the Default Download Directory entry does not exist then Windows has not recorded any change made using the procedure in para 1.5 above.

- - - - - - - - - - - - - - - - - - - - -

3  A few lines up from the HKEY_CURRENT_USER\Software\Microsoft\Windows\CurrentVersion\Explorer\User Shell Folders entry you have been working in there is an entry for Shell folders. 

  • Many of the entries in User shell folders are repeated in Shell folders. 
  • I always edit the entries in User shell folders not Shell folders. 
  • I have never found a use for Shell folders and have never bothered experimenting with it because it contains the warning !Do not use this registry key.
  • I mention the existence of Shell folders just because you might see it and worry that something has gone wrong. 

- - - - - - - - - - - - - - - - - - - - -

4  If you want to restore Registry settings for a key that you have changed then all you need to do is double-click on the .reg file that you exported before you starting editing it. 

  • This procedure is called merging the key rather than running the file.
  • You will need to give your confirmation a couple of times.
  • Afterwards, a success dialog is displayed.

5  Whilst I have seen some threads in this forum that allege that Registry edits can achieve more than using the Location tab can [i.e. more than my procedures cover], I did not see any convincing evidence that it worked [and I have seen at least one user report that subsequent attempts to run a Repair install would not work - the system just said that it could not work with the unexpected folder locations that it discovered & then gave up].  I think these threads said that they had relocated the whole of the user profile onto the data drive but I have no notes about them as they seemed to me to be dodgy procedures. 

- - - - - - - - - - - - - - - - - - - - -

6  Benefits of user folder relocation

The separation of your data files from your OS & its applications does not just free up space on the OS drive [normally C:\], it also makes it more convenient for you to make regular system images of the OS drive that are almost certainly going to be 40GB or smaller & only take 25 minutes or less to do.  If you had all your data files on the same drive as your OS & its applications then the total size might be so great that it would take ages to do so you might never bother.

  • An up-to-date system image gives you excellent disaster recovery capabilities as your response to serious system corruption could then simply be to boot from a System repair disk, Windows installation USB/DVD or [Windows 10] Recovery drive and restore the image, i.e. restore the system drive to exactly what was on it at the time the image was made.  This restoration can also take only about 25 minutes.
  • I first did this seven years ago and would never configure my system any other way now. 
  • I had previously regarded system images as a tool that IT departments used to avoid having to fix faults properly but I was wrong. 
  • I now make regular system images on external drives and can therefore recover from disasters reliably & quickly. 
  • You can confidently chase faults or just conduct experiments knowing that you will be able to get back to square one even if it all goes horribly wrong.
  • With separate system & datafile drives, you would continue to make straightforward file backups of your own files on your data drive.

- - - - - - - - - - - - - - - - - - - - -

7  Additional relocations available to Windows 10 users

See Start button, Settings, System, Storage, Change where new content is saved.  You can choose a different drive for new Apps, for user folders & for offline maps.  It allows you to choose drives to use for your user folders but I prefer using the procedure I have described above because it also allows me to choose specific folders instead of just drives.

I have not relocated any Apps but I have chosen drive F: for my Maps. 

  • This decision relieves my OS drive [C:] of the burden of holding the 10GB of maps and so reduces the size of my regular system images.
  • Drive F is actually a small 10GB partition on the very large drive I use for my own files. 
  • Because I have chosen to save my Maps in drive F, I ought to include drive F in system images but I only do so once in a blue moon when I add or update any maps. 
  • My system image software is Acronis True Image and I can choose whichever drives I want for imaging [I believe that Macrium Reflect Free has the same capability]. 
  • If I was using Windows to create my system images I would have to include both drive C and drive F in every image and that would add 10GB to every image I made.  I appreciate that an extra 10GB is hardly the end of the world.
