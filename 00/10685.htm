john sr wilson say:
You would need to do something like ...
Sub shuffle()
Dim ray1() As String
Dim ray2() As String
Dim otxR As TextRange
Dim otxR2

Dim randNum As Integer
Dim max As Long
Dim w As Long
Set otxR = ActivePresentation.SlideShowWindow.View.Slide.Shapes("originalText").TextFrame.TextRange
If otxR.Words.Count < 2 Then Exit Sub
ray1 = Split(otxR.Text, " ")
max = UBound(ray1)
Debug.Print ray1(2)
ReDim ray2(0 To max)
Randomize
For w = 0 To max
randNum = (Int(Rnd * UBound(ray1)))
ray2(w) = ray1(randNum)
ray1(randNum) = ray1(UBound(ray1))
If UBound(ray1) <> 0 Then _
ReDim Preserve ray1(0 To UBound(ray1) - 1)
Next w
Set otxR2 = ActivePresentation.SlideShowWindow.View.Slide.Shapes("shuffledText").TextFrame.TextRange
otxR2.Text = ""
For w = 0 To UBound(ray2)
otxR2.Text = otxR2.Text & ray2(w) & " "
Next
otxR2.Text = Trim(otxR2.Text)
Set otxR2 = otxR2.Replace("  ", " ")
End Sub

ken sheridan say:

You are not referencing the variable to which you've assigned the name of the query.  The line should be:

DoCmd.OpenQuery stDocName

The stLinkCriteria variable is redundant in this event procedure as the OpenQuery method does not have a WhereCondition argument.  To open the query restricted on the basis of a value in a text box in the current form you'd need to include a parameter in the query.  In the 'criteria' row of the relevant zip code column in query design view you'd put:

Forms![Grievance Tracking]![UnitZipCode]

However, why not create a form whose DefaultView property is 'datasheet'  and open the form filtered by means of the WhereCondition argument as you were doing originally.  You might even simply be able to make a copy of your original UnitFinanceNum form under a new name and change its DefaultView  property.

Is it possible to have a macro that copies 5 rows of data on all of the tabs between A and Z?  The data will always be on the same row in each tab but the row number could change in the future, so I would need the macro to prompt the user for the beginning row #.  So, if the row number is 175, the macro needs to copy rows 175:179 for columns A through N, for each tab to a master sheet called "Summary".  It would need to start with the first tab after tab A and end with the tab before tab Z.  I need it to paste special values the data dropping it on the "Summary" tab beginning on row 5, putting the first tabs data on rows 5 - 9, and the 2nd tabs data on rows 10 - 14 and so on.  When pasting to the Summary tab, it needs to paste the data beginning in column C, not column A.

mike h.. say:

Hi,

 

Try this. You didn't say which column C1 was to be copied to so I have assumed column C.

 

 

Sub Copy_Data()
Dim TheRow As Long, A As Long, Z As Long
Dim dst As Worksheet, LastRow As Long
Dim x As Long
Set dst = Sheets("Summary")
LastRow = 5
A = Sheets("A").Index + 1
Z = Sheets("Z").Index - 1
TheRow = CLng(InputBox("Enter Start Row", "Copy"))
For x = A To Z
    Sheets(x).Cells(TheRow, 1).Resize(5, 14).Copy
    dst.Cells(LastRow, 3).PasteSpecial xlPasteValues
    Sheets(x).Range("A1").Copy dst.Cells(LastRow, 1).Resize(5)
    Sheets(x).Range("C1").Copy dst.Cells(LastRow, 3).Resize(5)
    LastRow = LastRow + 5
Next
End Sub
