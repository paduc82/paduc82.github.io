You won't lose programs like Adobe CS6 but check the Adobe site for info about any losses of functionality and for updates.  I use Office 365 and there was no problem.  

What you will lose are programs that will trash something in the OS and anti-virus programs unless you first update them to a Windows 10 compatible version.  Anti-virus programs have hooks into the lower levels of the OS and always have to be updated for a new OS.  You won't lose a program that runs but with some loss of functionality.  Just the ones in which Microsoft has found an issue that affects Windows in some way.

You will be asked to remove some backup programs because they really dig deep into the OS and you need a new version.  Update yours if an update is available.

That's the general answer.  Your mileage may vary so make a system backup before you upgrade.  Windows 10 compatibility seems to be very good.  Surprising to me in some areas.

teylyn say:
Hello,

here is some code to set the X and Y axis font size:

    ActiveChart.Axes(xlValue).Select
    With Selection.Format.TextFrame2.TextRange.Font
        .BaselineOffset = 0
        .Size = 15
    End With
    ActiveChart.Axes(xlCategory).Select
    With Selection.Format.TextFrame2.TextRange.Font
        .BaselineOffset = 0
        .Size = 15
    End With

And here is some code to set the minimum, major and minor on the Y axis:

    ActiveChart.Axes(xlValue).MinimumScale = 50
    ActiveChart.Axes(xlValue).MajorUnit = 5
    ActiveChart.Axes(xlValue).MinorUnit = 5

For anyone following this: The key question appears to be how to go about editing the text boxes presented when a Pub document is opened in a version which does not support that "view".

There is nothing that can be done to prevent how the document is rendered... That's exactly how Word 2008 or 2011 constructed the document based on the actions taken by the creator. It's just that when working in Pub Layout View those processes are virtually transparent to the user.

Editing content in the text boxes doesn't require using a menu command, although right-clicking a text box does produce a contextual menu containing the Edit Text command. Hi,

Do you get the error with a particular website, or all websites that you visit?

If Internet Explorer was not able to determine if the certificate from that website has been revoked, and your browser is configured to expect an OCSP (Online Certificate Status Protocol) response, it provides that warning message.

We suggest verifying the Advanced Settings of your browser by following these steps:

1. Launch Internet Explorer 11.

2. Click on Tools.

3. Go to Internet Options.

4. Click on the Advanced tab.

5. Uncheck the 'Check for server certificate revocation' option.

6. Click Apply, then OK.

Let us know how it turned out, and if you need further assistance.
