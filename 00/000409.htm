Problem:
- Colleague sends me a .XLS which has an embedded .PDF on one sheet
- Office: Mac 2011 will not open the .PDF because of lack of OLE support

Solution:
- Copy the .XLS file to a new file
- Rename the file from <filename.xls> to <filename.zip>
- Open the .ZIP with Archive Utility
- Browse into the resulting "decompressed" folder to filename>xl>embeddings
- Find the file of interest, named something like <oleObject2.bin>
- Rename to <oleObject.pdf>

Hi Darren,

Unfortunately O365 does not support generic SMTP relaying. You will need to have a mailbox with a proper Exchange online license and use SMTP settings for O365. This way, for every email address you need to send email via an SMTP relay has to be configured and licensed as an Exchange online user mailbox.

If your plan is to get rid of your on premises IIS SMTP relay, you can use a third party transactional email service provider such as SendGrid, Mailjet or Mailgun.
