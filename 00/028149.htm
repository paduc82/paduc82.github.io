suat.cini say:

When you right-click on the ISO file you have, right-click menu opens. There you have mount right-click menu option. Just click mount. Your ISO turns into a virtual DVD drive. You can click setup.exe from the virtual DVD drive or  you can copy everything on that virtual DVD drive on to a USB flash drive and run setup.exe from that drive. Note: When you copy the contents of ISO file on to a USB drive, copying the files doesn't make USB drive bootable. But you can upgrade your current Windows from it.

Now, while Windows is running, clicking setup.exe initiates Windows upgrade process. You should then follow the on-screen instructions to upgrade your current build to the build that was in your ISO file.

For safety reasons, it is advised to back up your important files on an external drive, just in case something goes wrong.

Hope this helps.

Follow suat.cini's instructions for mounting the ISO file in Windows and running setup.exe (or run setup.exe from the USB stick that you made, if you don't have the single ISO file to mount). If you boot from USB, you do not get the option to upgrade.

Since you have a lot of installed software, do a clean boot before upgrading:

  • run Task Manager and disable all startup programs shown on the Startup tab
  • run msconfig and disable all non-Microsoft Services shown. You can select to hide the Microsoft services first to make the rest easy to select.
  • restart and then run setup.exe to upgrade

ken sheridan say: If you are inserting data into several related tables via a single form then, rather than basing the form on a query which joins the tables, subforms should be used to insert data into the referencing tables.  Each subform is linked to the parent form on the primary/foreign keys.

You might like to take a look at DatabaseBasics.zip in my public databases folder at:

https://onedrive.live.com/?cid=44CC60D7FEA42912&id=44CC60D7FEA42912!169

Note that if you are using an earlier version of Access you might find that the colour of some form objects such as buttons shows incorrectly and you will need to  amend the form design accordingly.  

If you have difficulty opening the link, copy the link (NB, not the link location) and paste it into your browser's address bar.

This little demo file explains the basics of modelling data through related tables, and includes a section on 'entering data via a form/subforms'.  This has two examples.  In one form contact data is entered in the parent form, while one or more employers for each contact can be entered via a subform.  In the other example orders are entered in the parent form, with each order line being entered as a separated row in a subform.

I suspect that the behaviour you are experiencing is connected with the fact that you are trying to enter data into multiple tables via a single form alone.  The fact that you are not seeing the data entered does not mean that it is lost of course.  A form is just an interface with the tables in which the data is stored as values at column positions in rows.  A form does not itself store any data.

If you continue to have problems post back with full details of the tables in question, and how they are related.  We'll then be in a better position to advise you.

mic_intel say: Import or export bookmarks

If you've got bookmarks or favorites stashed away in another browser, you can easily import them into Google Chrome. Similarly, you can always get your bookmarks out of Chrome by exporting them as a HTML file.

Import bookmarks from Firefox, Internet Explorer, or Google Toolbar
  1. Click the Chrome menu .
  2. Select Bookmarks.
  3. Select Import bookmarks and settings.
  4. Select the program that contains the bookmarks you'd like to import.
  5. Click Import.

If you haven't yet created any bookmarks in Chrome, the bookmarks you import will show up directly on the bookmarks bar, normally docked underneath the address bar. If you already have bookmarks within Chrome, the bookmarks you import will appear in a new folder called "Imported from Firefox" or "Imported from IE," located at the end of the bar. You can also find your bookmarks by clicking the Chrome menu and selecting Bookmarks.

Using a Chrome device? This option is not available. Instead, export your bookmarks as a HTML file and import them via the bookmark manager. See the next section for steps.

Import bookmarks from all other browsers
  1. Export bookmarks from the browser as a HTML file.
  2. Click the Chrome menu on the browser toolbar in Google Chrome.
  3. Select Bookmarks.
  4. Select Bookmark manager.
  5. Click the Organize menu in the manager.
  6. Select Import bookmarks from HTML file.
  7. Open your saved HTML file.

If you haven't yet created any bookmarks in Chrome, the bookmarks you import will show up directly on the bookmarks bar, normally docked underneath the address bar. If you already have bookmarks within Chrome, the bookmarks you import will appear in a new folder at the end of the bar. You can also find your bookmarks by clicking the Chrome menu and selecting Bookmarks.

Import bookmarks from other devices where you use Chrome

If your bookmarks are stored in Chrome on another computer, the easiest way to get those bookmarks onto the device you're using is to sign in to Chrome on both computers and sync the bookmarks to your account. Learn how to sign in to Chrome and enable sync:
