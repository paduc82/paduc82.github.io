To import an Excel spreadsheet to create a custom list in SharePoint, you can follow the steps:

 

1. Use IE browser to open your SharePoint site.

2. On the site where you want to add a spreadsheet based list, go to Settings and then click Add an app.

3. In the Find an app field, type spreadsheet and then click the search icon.

4. In the search results page, click Import Spreadsheet.

5. In the New app page, type a Name for the list.

6. Browse to or type the File location of the spreadsheet. Click Import when done.

 

Note: If you get an error message stating that a valid spreadsheet you're importing isnâ€™t valid or contains no data, add the SharePoint site you're on to the trusted sites list on the Security tab in internet Explorer Internet Options.

 

7. In the Import to Windows SharePoint Services List window, choose Table Range, Range of Cells, or Named Range. If you want to select a range manually, choose Range of Cells, and then click Select Range. In the spreadsheet, click the upper left cell, hold down the Shift key, and select the lower right cell of the range you want.

8. The spreadsheet data will appear in a list in SharePoint.

 

For more information, you can refer to Create a list based on a spreadsheet.

 

As for updating changes to list from the spreadsheet, you can refer to the suggestion in this post: Updating SharePoint List from Excel.

Thanks Neha. Seems like it's a sandboxing issue, and most work arounds only work locally. If someone in the future is reading this with the same problem, I wrote a quick macro that isn't as robust, and the image title needs to be present already (mine are a row below the link, hence the ActiveCell.Offset(1,0)). Sub hyper()

jay freedman say:

Sorry, I forgot a crucial difference between the way the Word Count feature counts "words" and the way the macro language counts "words". When you use the Word Count, it ignores punctuation marks, paragraph marks, and tabs. In the macro, each occurrence of each of those things counts as a "word". There's no good reason for that, it just is.

Here's the macro adjusted to behave like Word Count, with my apologies.

Sub CountWordsInColumn()
    Dim lngAsc As Long
    Dim wdsTotal As Long
    Dim nonWordTotal As Long
    Dim col As Column
    Dim cel As Cell
    Dim rgCell As Range
    Dim strAfter As String
    
    If Not Selection.Information(wdWithInTable) Then
        MsgBox "Place the cursor in the column to count, and rerun the macro.", , "Not in table"
        Exit Sub
    End If
    
    For Each col In Selection.Columns
        For Each cel In col.Cells
            Set rgCell = cel.Range
            rgCell.MoveEnd wdCharacter, -1  ' exclude cell marker
            If Len(rgCell.Text) > 0 Then
                ' get a string of the characters that aren't 0-9,A-Z,a-z or space
                strAfter = rgCell.Text
                For lngAsc = Asc("0") To Asc("9")
                    strAfter = Replace(strAfter, Chr(lngAsc), "")
                Next lngAsc
                For lngAsc = Asc("A") To Asc("Z")
                    strAfter = Replace(strAfter, Chr(lngAsc), "")
                    strAfter = Replace(strAfter, Chr(lngAsc + 32), "")
                Next lngAsc
                strAfter = Replace(strAfter, " ", "")
                
                wdsTotal = wdsTotal + rgCell.Words.Count - Len(strAfter)
            End If
        Next cel
    Next col
    
    MsgBox "Selected column(s) contains " & wdsTotal & " words", , "Column Word Count"
End Sub
