Split from this thread.

Using PowerPoint 15.16 (151105) for Mac OS X El Capitan 10.11.1. Crashing repeatedly!!! We, the users should not have to work this hard to report crashes:

Hi

After reading through this thread I have some suggestions on how to troubleshoot these issues.

First, regarding crash reports, when an Office application crashes Microsoft Error Reporting app should open. You are offered an opportunity to report the crash. Sometimes the report will ask if it is OK to send some of your own personal files to Microsoft. If you are OK with that, then go ahead but you are not under an obligation to do so.

Crashes are caused a number of ways. Certainly a bug in PowerPoint could be the cause. But there are other possibilities including a faulty install of Office, a fault in Mac OS X, a bad install of Mac OS X, a problem with a font (this seems to me to be the most likely thing going on here based on what I've read so far), hardware problems, Mac OS X directory corruption, corrupt preferences, interference from other software, bad RAM, corrupt slides within the presentations, file path problems, etc.

There is an orderly way to try to figure out which is the cause of a particular crash. Based on the symptoms, I try to start with the troubleshooting steps most likely to pinpoint the cause and come up with a solution.

You identified two specific scenarios: Change image sharpness = crash. Open 200+ slide show = crash. 

You have installed the latest updates to Office and Mac OS X. That is excellent! Doing so fixes most of the known crashes. It also means you and I are using identical versions of PowerPoint, but I am using Yosemite, so it might be a good point of comparison that different behavior could be a result of Mac OS X.

Let's start with the change image sharpness crash. I opened a new, blank presentation. Then I inserted a jpeg picture from file and adjusted the sharpness using the slider and presets. No crash. So simply adjusting sharpness does not always result in a crash. So something is different between what I did and what you did. And El Capitan could be that difference. I can try on an El Capitan Mac next week. But it might be a particular picture, or a picture that was made by a particular application, or a certain kind of picture. Can you share a problem picture and/or presentation on OneDrive or DropBox so others can experiment on their systems? Reply with the link so others can see what happens on their systems.

Now about the large presentation crashes. You didn't say what version of PowerPoint made those presentations or what file format they are in. Do they contain a particular kind of embedded content such as a movie or sound file that might be in an unusual format?

My frist inclination is to suggest uploading a problem presentation to OneDrive and see if the online version of PowerPoint can open the file. If it can, make a trivial change to the presentation and then save it using PowerPoint on line. Then see if the saved revised presentation will work on your Mac.

A simple troubleshooting step to perform is to do a Safe Boot. It will take a long time for your Mac to start up in Safe Mode. When you do a Safe Boot in El Capitan Apple's Disk Utility runs and fixes any troubles with your Mac's directory and file system. Permissions are repaired. The font cache is emptied. Only a limited number of fonts are enabled. Applications that run at startup don't run automatically. 

Once you reach the desktop in Safe Mode try running PowerPoint. Is the crash still there? If not, then maybe simply performing a Safe Boot fixed the ailment. Or perhaps one of the fonts that wasn't loaded is bad. Or perhaps some other software utility is interfering. After testing in Safe Mode, do regular restart. Try PowerPoint again. Is the crash still there?

If the crash was gone in Safe Mode but returns during a regular boot then there are more things to try. The first would be to use a font utility to validate ALL your fonts. Get rid of any font that doesn't pass as clean and green, restart your Mac and then empty the trash. If you've gotten this far and still have a crash, then there are more things to explore. But if you get this far you've probably gotten rid of the crash. Either way, please reply with your results.

If you want to download the alias emails using pop3, you cannot move the emails to a separate folder.
