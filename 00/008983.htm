hansv mvp say:

IsLoaded is a property of an AccessObject. It works OK for me in a newly created database in Access 2010 SP1, e.g.

 

If CurrentProject.AllForms("frmTest").IsLoaded Then
    MsgBox "The form frmTest is loaded"
Else
    MsgBox "The form frmTest is not loaded"
End If

 

There are many examples on the web of an IsLoaded function. This is a custom VBA function, it's not built into Access. So this function will not be available in a new database until you create or import it.

hansv mvp say:

The function will display an error message if you pass the name of a non-existent form. To get around that, you could change the code to

 

Function IsFormLoaded(ByVal strFormName As String) As Boolean
    Dim oAccessObject As AccessObject
    On Error Resume Next
    Set oAccessObject = CurrentProject.AllForms(strFormName)
    If oAccessObject Is Nothing Then Exit Function
    If oAccessObject.IsLoaded Then
        If oAccessObject.CurrentView <> acCurViewDesign Then
            IsFormLoaded = True
        End If
    End If
End Function

 

This will return True only if the form exists and has been opened in any view but design view, and False in all other situations.

hansv mvp say:

Both module-level and project-level declarations should always be at the top of the module, after Option Explicit but above all procedures (subs) and functions.

If you declare a variable (whether an array or not) using Dim or Private, it will be a module-level variable, available within its module only.

If you declare a variable using Public, it will be a project-level variable, available in all modules belonging to the project.

Public arr()

Sub Test()
  ReDim arr(1 To 3)
