With an If...then...Else field, the true and false results need to be bound by quotation marks and the space between the first closing quotation mark and the second opening one is the boundary between the true and false result.  

{ IF { MERGEFIELD Test } = "Red" "This is the result if Test = Red" "This is the result if Test <> Red" }

If you do need to have quotation marks in the true and\or false text, you could use { INCLUDETEXT } fields to handle the results, or you could use a pair of apostrophes around ''Claims''.

If a document contains a document property named DocumentID, of type Number, the following field construction creates a QR code that includes the property's value:

jay freedman say:

Jim, I think there's some confusion about terminology in your posts, and in this case it's critical that we're talking about the same thing.

The balloons shown in your screen shot are not comments. Word calls them "revisions", specifically formatting changes. If those are the things you want "not in balloons" (i.e., in line with the text), make these settings on the Review tab of the ribbon before going to Print:

  • In the Tracking group, set the top right dropdown to All Markup.
  • Click the Show Markup dropdown and click Balloons, then click Show All Revisions Inline.
  • If you want to see only changes in wording, spelling, etc. but not formatting, click Show Markup again and click Formatting to uncheck it.

John Sarc, the only thing in the provided link that may be useful is the last part, about making a separate copy of the document and rejecting the unwanted changes from it before printing it. That's considerably more work than changing a few settings.
