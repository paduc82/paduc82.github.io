Placeholder content moving between layouts when switching between - Microsoft Community

joshuabalsillie say:

Placeholder content moving between layouts when switching between layouts

This is a continuation of: Slide Master placeholders content moving across layouts

In my opinion, editing the XML is not really a good solution since there doesn't appear to be a built in XML editor, meaning this workaround could break after any update to Microsoft Office. Understandably you do this once your template is fully completed to avoid your XML edits being overwritten, but I wouldn't call this a solution.

Description

When you create a layout, duplicate that layout, then create a slide from the first layout, changing the slide to the second layout moves content between placeholders. The reason stated in the aforementioned article is that the placeholder ID (idx="") changes between slides.

This happens for:

  1. Duplicated layouts where all placeholders of equivalent type, do not all have content
    • At least one placeholder of equivalent type does not have content
    • example: content = picture (have not tested all placeholder combinations)
  2. Duplicated layouts and second layout has deleted placeholders

This doesn't happen for:

  1. Duplicate layouts where all the placeholders equivalent type, all have content
    • example: content = picture (have not tested all placeholder combinations)

Expected Experience

  1. Placeholder IDs remain consistent across duplicate layouts.
  2. If placeholders are deleted in a duplicate layout, remaining placeholders retain their IDs.
  3. If placeholders are added in a duplicate layout, new placeholders get new IDs starting with the highest ID + 1.
  4. If a placeholder deleted from a duplicate layout is then re-added via copy + paste from the original layout the placeholder ID is retained
  5. The experience described in 1-4 should be duplicated for slides in Normal view.
  6. A slide converted from a layout with few objects to a duplicate layout with many objects fills in the few objects correctly, based on their IDs.
  7. A slide converted from a layout with many objects to a duplicate layout with few objects, should fill in the few objects correctly, based on their IDs. The duplicate layout then creates new objects, retaining the IDs from the layout with many objects.

Thoughts

This would not only make things easier for people building templates, but also those that use them.

This type of functionality would only potentially be an issue for people that don't duplicate a layout when designing templates. Given the expected experience, additional content areas would be created over top of the slide, as opposed to guessing where to put content. Since PowerPoint does a poor job of this anyway I don't think this would negatively affect many users, but I could be wrong.

PowerPoint only does a good job of reassigning content when the layouts are close to identical. The only scenario I can think of, where this works well, is if you know exactly the content you will include in your slide ahead of time.
