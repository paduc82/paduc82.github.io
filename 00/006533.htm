How many times can I install MS Office Home and Student 2010?

I understand that you can install three time on three different machines, but I reformatted my computer a short time ago and am worried it may read it as a different machine.

Many thanks for any help you can give me.

rohn007 say:

In theory you can re-install an unlimited number of times on the same computer.

The activation system generates a "snapshot" of your computer based on several software and hardware elements. You are allowed to make "limited" changes before triggering a new activation, ie changing one or 2 elements at a time such as change Windows version, add ram, change CPU, change video card, change HD. The one element you can't change is apparently the motherboard. MS considers that a "new computer", but even so it is not really significant for your type of license (it is relevant for the OEM or Product Key Card type licenses).

As well, your type of license allows you to transfer any of those 3 installations to a new computer, by uninstalling from the old computer or if the old computer is permanently not usable (ie dead, recycled).

Bottom line, for you there should be no problem with re-installing Office 2010 H&S on your computer after doing a HD reformat and re-install of existing Windows.

HINT: before doing the reformat you should should HAVE done an image copy backup so you could fall back to the previous install in case there were problems with the re-install process

Before doing the reformat, you should confirm that you have the installation disk and Product Key (inside the 2010 plastic CD holder box) for Office and all other applications.  With 2010, if you didn't have key, you could use a tool to extract it. (FYI, no longer possible to extract key from 2013 or 2016).

jay freedman say:

If you can live with having the author's name on a separate line below the title, you can arrange that as follows:

The author's name usually appears on the first page of a story. Assuming that's true in your document, and that the name is alone in a paragraph (without "by" or other text), apply a unique style to that paragraph. It can be a built-in style that you haven't used for anything else in the document, or a custom style that you define for the purpose.

Click References > Table of Contents > Custom Table of Contents. In that dialog, click the Options button. In the Options dialog, scroll the list of styles to find the one you used for the author names and enter a number for the level it should use. (That will determine the style, such as TOC 2, that will appear in the TOC. By default, that changes only the amount of the indent.) Then finish entering the TOC in the document.

The author names will appear with leaders and page numbers. To remove those, toggle the TOC field and edit it to include the "switch"  \n 2-2 (or whatever level number you gave in the dialog). For example, if you used a custom style named StoryAuthor and set it to TOC level 4, the final field code should look like

{ TOC \o "1-3" \n 4-4 \h \z \t "StoryAuthor,4" }

If the author name doesn't appear alone in a paragraph for each story, you can use TC fields instead of styles to create the entries.

For more information, see https://shaunakelly.com/word/numbering/tableofcontents.html#CustomizeTheTOC and http://wordfaqs.ssbarnhill.com/TOCSwitches.htm.

Hello Jason,

 

My turn to apologize. I have been busy also but I hope that the following solution will do what you want. In keeping with good programming protocol, the Source data is maintained and the Output is on a another worksheet. (Source data is initially sorted but it is returned to original state after processing.) This allows you to compare the Source data to the Output data and see if you believe it is correct and also does not destroy the source data if output is not correct.

 

I have assumed that the Source data only contains a table as per your example and that the data starts as Cell A1. If other data on the worksheet then it probably will not work. However, it is generic as to the number of columns and number of rows. Your example had Date 1, Date 2 and Date 3. You can have extra Date columns without affecting the outcome.

 

Feel free to get back to me if any problems but try to explain exactly what is not working correctly.

 

The sub to run is ReconcileData. The code also has a UDF (User Defined Function) which is called from the main sub to find the Last Row and Last Column. This is a very reliable method compared to other methods that are used that can return incorrect last row and last column.

 

Sub ReconcileData()

    Dim wsSource As Worksheet
    Dim wsDestin As Worksheet
    Dim rngSource As Range
    Dim rngDestin As Range
    Dim rngCel As Range
    Dim rngMID As Range
    Dim rngFind As Range
    Dim lngLastRowCol As Long
    Dim lngLastRow As Long
    Dim lngLastCol As Long
    Dim r As Long
   
    Set wsSource = Worksheets("Sheet1") 'Edit "Sheet1" to your input data dource sheet
    Set wsDestin = Worksheets("Sheet2") 'Edit "Sheet2" to your output data sheet
   
    wsDestin.Cells.Clear    'Clears any existing data from Output sheet
   
    lngLastRow = LastRowCol(wsSource, True) 'True last Row, False last Column
   
    If lngLastRow = 0 Then  'If zero then no data in worksheet
        MsgBox "No data on worksheet." & vbCrLf & _
                "Processing terminated."
        Exit Sub
    End If
   
    lngLastCol = LastRowCol(wsSource, False) 'True Last Row, False Last Column
   
    With wsSource
        'Create an index of current data
        .Cells(1, lngLastCol + 1) = "Index"
        .Range(.Cells(2, lngLastCol + 1), _
            .Cells(lngLastRow, lngLastCol + 1)).Value = "=row()"
           
        .Range(.Cells(2, lngLastCol + 1), _
            .Cells(lngLastRow, lngLastCol + 1)).Copy
           
        .Range(.Cells(2, lngLastCol + 1), _
            .Cells(lngLastRow, lngLastCol + 1)) _
            .PasteSpecial xlPasteValues
       
        'Copy Headers to Destination worksheet
        .Range(.Cells(1, 1), .Cells(1, lngLastCol + 1)).Copy _
            Destination:=wsDestin.Cells(1, 1)
        Application.CutCopyMode = False
   
    End With
       
    'Sort the data on MID ascending and Date decending.
    'Allows selection of data by MID with the most recent date first.
    With wsSource
        .Sort.SortFields.Clear
        .Sort.SortFields.Add Key:=.Range(.Cells(2, 1), _
            .Cells(lngLastRow, 1)), _
            SortOn:=xlSortOnValues, _
            Order:=xlAscending, _
            DataOption:=xlSortNormal
   
        .Sort.SortFields.Add Key:=.Range(.Cells(2, 6), _
                .Cells(lngLastRow, 6)), _
                SortOn:=xlSortOnValues, _
                Order:=xlDescending, _
                DataOption:=xlSortNormal
   
        With .Sort
            .SetRange wsSource.Range(wsSource.Cells(1, 1), _
                    wsSource.Cells(lngLastRow, lngLastCol + 1))
            .Header = xlYes
            .MatchCase = False
            .Orientation = xlTopToBottom
            .SortMethod = xlPinYin
            .Apply
        End With
    End With
   
    With wsSource
        'The range MID on the source worksheet (First column)
        Set rngMID = .Range(.Cells(2, "A"), .Cells(lngLastRow, "A"))
    End With
   
    With wsSource
        For r = 2 To rngMID.Cells.Count 'Starts row 2 after column headers
            Set rngDestin = wsDestin.Cells(wsDestin.Rows.Count, "A") _
                            .End(xlUp).Offset(1, 0)
            Set rngCel = .Cells(r, 1)
            rngCel.Resize(1, lngLastCol).Copy Destination:=rngDestin

            Do
                'Test if next cell equals rngCel
                If rngCel.Offset(1, 0) = rngCel Then
                   
                    'Matches so Concatenate the Case numbers
                    rngDestin.Offset(0, 2) = rngCel.Offset(1, 2) & ", " _
                            & rngDestin.Offset(0, 2)
                   
                    'Increment r and loop for next test
                    r = r + 1
                    Set rngCel = .Cells(r, 1)
                Else
                    Exit Do     'If nest cell does not rngCel
                End If
            Loop
            'Note r incremented above so next r is after the incremented r
        Next r
        
     End With
       
    'Sort the source data back to origninal order _
        based on saved data in Temporary Index column
    With wsSource
        .Sort.SortFields.Clear
       
        .Sort.SortFields.Add Key:=.Range(.Cells(2, lngLastCol + 1), _
            .Cells(lngLastRow, lngLastCol + 1)) _
            , SortOn:=xlSortOnValues, Order:=xlAscending, _
            DataOption:=xlSortNormal
       
        With .Sort
            .SetRange wsSource.Range(wsSource.Cells(1, 1), _
            wsSource.Cells(lngLastRow, lngLastCol + 1))
            .Header = xlYes
            .MatchCase = False
            .Orientation = xlTopToBottom
            .SortMethod = xlPinYin
            .Apply
        End With
    End With
   
    For Each rngCel In rngMID
        With wsSource
            Set rngFind = .Columns(1).Find(What:=rngCel.Value, _
                    LookIn:=xlFormulas, _
                    LookAt:=xlWhole, _
                    SearchOrder:=xlByRows, _
                    SearchDirection:=xlNext, _
                    MatchCase:=False)
           
            If Not rngFind Is Nothing Then
                rngCel.Offset(0, lngLastCol) = rngFind.Offset(0, lngLastCol)
            Else
                MsgBox "Problem with code. Refer back to code author." _
                    & vbCrLf & "Processing terminated."
                Exit Sub
            End If
           
        End With
       
    Next rngCel
   
    With wsDestin
        lngLastRow = LastRowCol(wsDestin, True)
        .Sort.SortFields.Clear
        .Sort.SortFields.Add Key:=.Range(.Cells(2, lngLastCol + 1), _
            .Cells(lngLastRow, lngLastCol + 1)), _
            SortOn:=xlSortOnValues, _
            Order:=xlAscending, _
            DataOption:=xlSortNormal
   
        With .Sort
            .SetRange wsSource.Range(wsSource.Cells(1, 1), _
                wsSource.Cells(lngLastRow, lngLastCol + 1))
            .Header = xlYes
            .MatchCase = False
            .Orientation = xlTopToBottom
            .SortMethod = xlPinYin
            .Apply
        End With
       
    End With
   
    With wsSource
        .Columns(lngLastCol + 1).Delete
        .Select
        .Columns().AutoFit
        .Cells(1, 1).Select
    End With
   
    With wsDestin
        .Columns(lngLastCol + 1).Delete
        .Select
        .Columns().AutoFit
        .Cells(1, 1).Select
    End With
   
End Sub

Function LastRowCol(ws, bolRowCol)
    'Finds the last used row or column in a worksheet
    Dim lngRowCol As Long
    Dim rng As Range
   
    If bolRowCol Then
        lngRowCol = xlByRows
    Else
        lngRowCol = xlByColumns
    End If
   
    With ws
        Set rng = .Cells.Find(What:="*", _
                LookIn:=xlFormulas, _
                LookAt:=xlPart, _
                SearchOrder:=lngRowCol, _
                SearchDirection:=xlPrevious, _
                MatchCase:=False)
    End With
   
    If Not rng Is Nothing Then
        If bolRowCol Then
            LastRowCol = rng.Row
        Else
            LastRowCol = rng.Column
        End If
    End If
   
End Function
