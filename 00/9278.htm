While waiting for your response, I'll give you this, maybe it will work for you.

If your sample data is in cells A1:B10, with the headings "Fruit" and "Qty" in row 1, in cells D1:H1 type the diferent fruits (1 column per fruit type):

In cell D2 (beneath Apple) enter the following array-entered formula:

=INDEX($A$1:$B$10,SMALL(IF($A$2:$A$10=D$1,ROW($A$2:$A$10)),ROW(1:1)),2)

Note you must commit this formula by pressing Ctrl+Shift+Enter.  If done correctly, the formula will be enclosed in braces { }.

where the #NUM! errors are indicating your "empty" rows...

=IFERROR(INDEX($A$1:$B$10,SMALL(IF($A$2:$A$10=D$1,ROW($A$2:$A$10)),ROW(1:1)),2),"")

Excel "HMS" format bugs? (Excel 2010 and older) - Microsoft Community

bricebreeden say:

Excel "HMS" format bugs? (Excel 2010 and older)

Can someone tell me if this is a known issue? And if it will ever be addressed??

 

I've noticed this issue in Excel 2010 as well as 2007 and older. All of them have problems with the "HMS" format (such as h:mm:ss, [h]:mm:ss, etc). This problem has existed for years, why can't it be repaired?? Seems to occur on all the various HMS formats.

 

When setting a Custom format on a cell in one of the "HMS" formats, the spreadsheet occasionally reverts to a timestamp format. A novice user can make this happen quite easily.

 

How to prepare a cell to reproduce the bug:

  • Open Excel with a new document
  • Enter a decimal number into a cell to represent the timespan as number of days. For example, 1.5 represents 36 hours.
  • Use the Format Cells option. Set to Custom format, and define as: [h]:mm:ss

There are several ways to make this "bug" occur. Prepare a cell as mentioned above, then try one of these methods:

  • Double click on the cell. Then click on a different cell.
  • OR: Single click on the cell. Then click on the Formula Bar, as if you are going to change the cell's value. Once you've done that, click on a different cell.

Once you've done that, you'll see that the cell has reverted to a timestamp format. It is no longer HMS.

With XL2010, I have no problem with the first test, double-clicking on a cell.  But that is because I disable the option "Allow editing directly in cells".  When I enable the option, I do see the "problem".  But that is for the same reason as below.

When we single-click, then click in the Formula Bar, we are initiating editing.  Note that the value that appears in the Formula Bar (and in the cell when "allow editing directly in cells" is enabled) has its own formatting rules.  When we click on another cell, we are implicitly "entering" the value as it appears in the Formula Bar (or cell).

There should be no mystery for the user, since the value that will be entered implicitly is displayed for them to see.


You might reasonably complain that the value displayed in the Formula Bar (and in the cell when "allow editing directly in cells" is enabled) does not conform to the cell format.  But that has been that way since "forever".  (At least, it works that way in XL2003 as well).

To avoid that behavior, when you see the value displayed in a different format, press Esc to avoid entering it as it appears. 

And you can mitigate the problem by deselecting the option "Allow editing directly in cells".  Then the problem is limited to when you click in the Formula Bar.  Presumably you do that usually only when you do intend to edit the value/formula. 
PS:  I do agree with you that it is annoying that the Formula Bar value looks so different.  This is especially problematic when we enter fractional seconds.  Since they do not even appear in the Formula Bar, we must re-enter them whenever we wish to edit the value.  [EDIT] On the plus side, for other numbers, it is a convenient way to see all significant digits of a value, which might be more than displayed due to the cell format
