Network connectivity lost after upgrading to an Insider Preview build

You may not have network connectivity if you upgrade to an Insider Preview build, if you have Hyper-V with an External V-Switch or Visual Studio Phone Emulator installed before the upgrade.

To confirm that this is the specific issue youâ€™re having, check these things:

  1. From the Start menu or screen, search for cmd and select the Command Prompt from the results. Type IPConfig and hit Enter. It will return an empty result if this is the issue.
  2. From the Start menu or screen, search for View Network Connections and select it from the results. Right-click on your network connection and select Properties. All of the check boxes for the protocols will be empty:

This happens because during upgrade, the Virtual Switch migration encountered a race condition, and left the Network Interface in an Unbound state.

To resolve the issue:

  1. Use Search from the Start menu or screen to search for Device Manager
  2. Open Device Manager, and expand Network Adapters
  3. Locate the network adapter that is faulting, and right-click it
  4. Select Uninstall
  5. Select Scan for hardware changes (either the icon in the ribbon, or from the Actions menu)
Windows should locate the network adapter and reinstall it.

hansv mvp say:
The subform should be based on the WriterAssignments table, not on the Writers table. The subform should be linked to the main form by the AssignmentID field by setting both the Link Master Fields and Link Child Fields properties of the subform on the main form to AssignmentID.

hansv mvp say:

If you have deleted all records from the Assignments table, compacting the database should reset the AutoNumber field. To do so, select Tools | Database Utilities | Compact and Repair Database.

Note: it shouldn't really matter whether an AutoNumber field starts at 0, it's just a convenient meaningless unique identifier.
