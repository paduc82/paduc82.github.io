graham mayor say:
The following should work (and I have included a test macro to confirm). Insert a new module in the Outlook VBA editor and copy the following to it. Create a rule that identifies the incoming messages and run the SavePDFAttachments script from the rule.

The macro looks for the text: "Company Name Service Report Task - Service Request" in the message subject (this is case sensitive) and if that is found, any attachments present are examined to see if they are PDF files. If they are then they are saved in a sub folder of strRootPath = here D:\Test\ named with the System ID number. If the folder doesn't exist it will be created. If there is an existing PDF of the same name in the folder it will be over-written, though if these are new ID numbers that shouldn't be an issue. Change D:\Test\ to the location where you wish the folders to be created. That folder must exist and the path must have the final '\' character.

Select a message and run the Test macro to test that it works correctly for you before applying the script to the rule.

Option Explicit
Private olAttach As Attachment
Private strFilename As String
Private vFolderName As Variant
Private strFolderName As String
Private Const strRootPath As String = "D:\Test\"

Sub SavePDFAttachments(olItem As Outlook.MailItem)
    On Error GoTo CleanUp
    If InStr(1, olItem.Subject, "Company Name Service Report Task - Service Request") > 0 Then
        vFolderName = Split(olItem.Subject, ",")
        strFolderName = Replace(vFolderName(1), "System ID ", "")
        strFolderName = strRootPath & strFolderName & "\"
        If olItem.Attachments.Count > 0 Then
            For Each olAttach In olItem.Attachments
                If Right(UCase(olAttach.Filename), 3) = "PDF" Then
                    If Not FolderExists(strFolderName) Then MkDir strFolderName
                    olAttach.SaveAsFile strFolderName & olAttach.Filename
                End If
            Next olAttach
        End If
    End If
CleanUp:
    Set olAttach = Nothing
End Sub

Private Function FolderExists(ByVal PathName As String) As Boolean
Dim nAttr As Long
    On Error GoTo NoFolder
    nAttr = GetAttr(PathName)
    If (nAttr And vbDirectory) = vbDirectory Then
        FolderExists = True
    End If
NoFolder:
End Function


Sub Test()
Dim olMsg As MailItem
    On Error Resume Next
    Set olMsg = ActiveExplorer.Selection.Item(1)
    SavePDFAttachments olMsg
End Sub

it is possible to add a picture background to a comment box in Excel 2003. The key is being precise about where you right-click on the comment. If you click inside the comment box, only the Font formatting tab will appear.

Instead, right-click on the cell with the comment and left-click on Edit Comment.

Right-click on the border of the comment and left-click on Format Comment. Click on the Colors and Lines tab.

Click on the Color dropdown menu, Fill Effects and then the Picture tab. Click on the Select Picture button, find the correct picture and click on Insert. Select the option Lock picture aspect ratio and click on OK in both windows.

Here is how you can do it in Windows 7/8:

   ***Warning editing Registry Values can cause serious harm to your computer***

   NB: Depending on your version of excel, these might be listed under the extension (.xlsm etc) or at the bottom by file type name (Excel.Macro.Worksheet.12 etc)

  NB: What this does is runs the Excel application with the full Filename and path of the file that you have tried to open as the parameter.

6. Repeat this for all excel extensions that have a Shell>Open>Command property.

7. Close Regedit, and test out your work by double-clicking on any excel file.
