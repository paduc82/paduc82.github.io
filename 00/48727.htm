1.  Exit any running instance of Windows Media Center.

2.  Delete the mediaCenterDataStore.db file. By default, this file is in the following location:

%SYSTEMDRIVE%\ProgramData\Microsoft\eHome\mediaCenterDataStore.db


To delete the file, follow these steps:

 

 

a. Click Start, and then click Run.

b. In the Open box, type run, and then click OK.

 If you are prompted for an administrator password or for confirmation, type the password, or click Continue.

c. In the Open box, type %systemdrive%\programdata\microsoft\ehome , and then click OK.

d. In the details pane, right-click mediacenterdatastore.db, and then click Delete.

e. Click yes to confirm that you want to move the Mediacenterdatastore.db file to the Recycle Bin.

3. Restart the computer for the changes to take effect after executing the steps.

4. Start Windows Media Center.

sanmartin say:

Now you're on the wrong track. NTDLL.DLL doesn't use a lot of programs, a lot of programs use NTDLL.DLL. This file is a bridge between user and kernel mode of Windows and used for loading programs that do not rely on the Windows-own mechanics (e.g., it handles most of the things that happen before Windows is started, you see the nice animation of a combining Windows logo, get the text "Windows is starting" or some "......" on the screen - depending on your version of Windows -, before you see a "Welcome" screen). Some of these programs don't shut down automatically when Windows is started, thus blocking further access to the NT layer DLL. If I had to guess, I would suspect AV software first, as this kind of software sometimes can be "overprotective". But it's easy to find out: Just "kill" each running process (out of your 14 candidates) after another (right-click on the process and choose "end process"), retry launching Oblivion and look what happens. You can't damage anything, in the worst case you need to reboot several times (processes like "HD audio" or "explorer.exe" are not the main suspects, even if killing explorer.exe and restarting it - done with TaskManager - sometimes works wonders).

But as a "quantum of solace": I've installed Oblivion on XP, on another partition. I'm using Windows 7 and I'm able to play the game without problems,  even across different partitions. Go figure. - Btw, if you try to download any version of NTDLL.DLL from the web, be aware that most of the offers are only trying to foist you some "registry cleaners" which you don't need, don't want and should be avoided. There's no sign that this simple file itself is the cause of your problems.

sanmartin say:

Like I said, "even ... killing explorer.exe and restarting it - done with TaskManager - sometimes works wonders". You could try it. Then, you wrote "(I) have been playing Oblivion with no problems until recently". That would suggest you've installed some conflicting software in the meantime, wouldn't it? I can't explain all functions of ProcessExplorer here, but I'm quite sure that processes like "taskeng.exe" (Task Scheduler Engine) also have dependencies (they start something at a given time).

Your error code is 0x5 (forget the zero's), "The INVALID_PROCESS_ATTACH_ATTEMPT bug check has a value of 0x00000005. This generally indicates that the thread was attached to a process in a situation where that is not allowed. For example, this bug check could occur if KeAttachProcess was called when the thread was already attached to a process (which is illegal), or if the thread returned from certain function calls in an attached state (which is invalid)." You don't have to understand that, but it's an indication that another program's "calls" are hindering your program (Oblivion) to proceed correctly.
