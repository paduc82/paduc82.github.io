I got the issue sorted with the help of local Microsoft support. I’ll post the answer here in case it helps someone else.

There is no real explanation why the account stopped working after several years of working fine, but it came down to the email account being an “EASI” Account. A way to test if this the case for you, is to check if you can access the mail using “private” mode on a browser. For me, this failed.

To resolve the issue, I had to create a new email alias for the account (*** Email address is removed for privacy ***) using the Microsoft account management tool (http://account.microsoft.com), and make that the primary account for login (don’t delete the old, leave it as secondary). After I did that, and created the outlook account on the phone once more but now with the new email as login, the email started working again.

Also, had to ensure from the mailbox (under accounts/connected accounts) that the old email is still the default send from address (and not the new email that is just for login purposes).

What is the FULL and EXACT error message?

Jose's first link is the most likely if I understand the generic interpretation of the message you provided.  If a user did not exit word correctly, they would leave behind a "lock" file. Next time anyone, including them, tries to open the file they get a message saying the file is in use ...

This is a mechanism left over from the very early days of networking when the applications would not allow more than one person to have the file open. This "lock file" was intended to provide a specific user name to talk to.  If you go to File menu > Options command > General option > update "User name" field.   Unfortunately most companies do not set this field so it just shows the generic value, which is very frustrating.
