Hi Ebbenezer,


The error (0x80070005) that you're receiving when you try to restore from recovery point means the target location could not be accessed: ERROR_ACCESS_DENIED. You have to ensure that you're using an Administrator account when you do the restore.

Also, please consider to try taking ownership/full control of the said target location by following the steps here: Take Ownership of a File or Folder.

A. Use System File Checker:

  1. In the search box on the taskbar, enter Command Prompt. Press and hold (or right-click) Command Prompt (Desktop app) from the search results and select Run as administrator.
  2. Enter DISM.exe /Online /Cleanup-image /Restorehealth (note the space before each "/"). (Note: This step may take a few minutes to start and up to 30 minutes to run and complete.)
  3. Enter sfc /scannow (note the space between "sfc" and "/").

You mentioned about the safety disk. May we know how generated it? We're thinking that this safety disk is from the PC manufacturer or the one you created using Windows 10. If it was generated using Windows backup and recovery, please review this article: Back up and restore your PC.

You can also try rebuilding bootrec files:

1. Shutdown the PC.

2. Turn it on. As as you get to the manufacturer's welcome screen (Or on where it gets stuck in blank screen), hold the power button for 6 seconds to forcibly shut the PC down.

3. Repeat that manner for 3 times. The 4th time would show you the recovery screen (Startup repair). 
    Note: If it run the Automatic repair, let it do so. If it fails, please proceed. . .

4. Click on Advanced recovery options > Troubleshoot > Advanced Options >Command Prompt and enter the following commands:

 

bootrec /scanos

bootrec /fixMBR

bootrec /fixBoot

bootrec /rebuildBCD

If all were successful, please restart the PC and see if that will boot the system as normal.
