brian tillman [outlook mvp 2007-2019] say:

> How exactly are you accessing your Email account? Is it through a web browser,

> Microsoft Office or any other Mail Client?


It's pretty clear that the person is connecting to Outlook.com using a web browser and that the mailbox is still on the old Outlook.com platforum and not the new "Outlook Mail" platform.

There have been other threads that have described issues with setting up a connected send/receive account to Zoho from Outlook.com. See

http://answers.microsoft.com/en-us/outlook_com/forum/oemail-oaddacc/cant-add-a-zoho-pop3-account-to-my-outlookcom/8475f654-4f09-4c4a-b4b7-4f576ee9c1e7 for example. There are posts at Zoho about it as well. See

https://forums.zoho.com/topic/pop-email-setup-with-outlook-com-not-connecting, for example.

My personal opinion is that each service is interpreting the connection protocols a little differently and that one side, the other, or both are not flexible enough to handle the differences.

There is a post at Zoho that said the person was able to get the connected account on the Outlook.com side to work if it's a send-only account and then they are forwarding their Zoho mail to their Outlook.com address.

Make sure that POP access is enabled on the Zoho side (it is enabled by default) if you want to use POP or that IMAP is enabled (it is enabled by default) if that's how you want Outlook.com to connect.

I decided to test this out. I created a Zoho address for myself. I had to specify another non-Zoho address as well for security reasons. I then had to verify the Zoho account by logging into that other non-Zoho mailbox to click a verify link. After that, I created the connected account in Outlook.com using manual configuration (Advanced configuration). Outlook.com was able to connect to Zoho well enough to verify that my username and password were good, but it could not make the POP connection. It could, however, make the SMTP connection, so I was able to send mail from Outlook.com using my Zoho address and the messages appeared in the Zoho Sent folder and arrived at the destination address.

So, the problem is with the POP transaction between Outlook.com and Zoho, but not with the SMTP connection.

jay freedman say:
I don't believe there's anything you can do to prevent Word from saving that batch of XML tags, but you can arrange to put it where it won't get in your way.

The reason those tags are there is to enable the GoBack command (Shift+F5) to take you to the last editing position when you reopen a previously saved document. The command works by moving the selection to a super-hidden bookmark named _GoBack, which Word inserts during the Save operation. Neither the Bookmarks dialog nor VBA (macros) can see that bookmark, so there's no way to delete it.

To get it out of your way, you could manually move the cursor to the end of the document, insert and delete a space character, and then save. The resulting XML looks like this:

<w:body><w:p w:rsidR="004E1F19" w:rsidRDefault="00C84D1F"><w:r><w:t>This is a test.</w:t></w:r><w:bookmarkStart w:id="0" w:name="_GoBack"/><w:bookmarkEnd w:id="0"/></w:p><w:sectPr w:rsidR="004E1F19"><w:pgSz w:w="12240" w:h="15840"/><w:pgMar w:top="1440" w:right="1440" w:bottom="1440" w:left="1440" w:header="720" w:footer="720" w:gutter="0"/><w:cols w:space="720"/><w:docGrid w:linePitch="360"/></w:sectPr></w:body>

To automate this, write a couple of short macros into the template for the documents that need to be searched:

Sub FileSave()
  With Selection
    If .StoryType <> wdMainTextStory Then
       ActiveDocument.Characters.First.Select
    End If
    .EndKey Unit:=wdStory, Extend:=False
    .TypeText " "
    .TypeBackspace
  End With 
  ActiveDocument.Save
End Sub

Sub FileSaveAs()
  With Selection
    If .StoryType <> wdMainTextStory Then
       ActiveDocument.Characters.First.Select
    End If
    .EndKey Unit:=wdStory, Extend:=False
    .TypeText " "
    .TypeBackspace
  End With 
  On Error Resume Next
  Dialogs(wdDialogFileSaveAs).Show
End Sub
