Outlook.com contact groups translate to categories in Office Outlook.  Contact Groups in Office Outlook do not sync in any way to Outlook.com's Contacts feature.  This is due to limitations in the Exchange ActiveSync protocol used to sync Office Outlook 2013 and Outlook.com.  That's just the way it works.

One way to do this is to have a "ToDoList" table with fields for (for example) QueryName, RunDate, StartDate, EndDate, and SequenceNo. You can store a series of query names and (if necessary) parameters in this table, and have a Query which pulls up all the queries that need to be run today; the code would loop through the recordset based on this query and use the database Execute method to run them all.

m4e_team_k say: This error message corresponds to HTTP 403 which means the credentials are ok, but allowed to perform the operation.

In MfE case it means the account is not allowed to access the server via activesync.

It could also be the SSL setting (yes/no), but in most cases SSL is required and since MfE defaults to 'Yes' this isn't likely.

You didn't say what version of Exchange. Assuming 2003, Google for "enable Activesync Exchange 2003" and find a relevant Microsoft TechNet page if you have access to the server.
