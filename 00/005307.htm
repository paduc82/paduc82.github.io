Cool. To fix the Option Explicit thing: Code window > Tools > Options > Require variable declarations. It will now be set for new modules. Existing ones you have to fix manually. Highly recommended.


To fix that non-compiling line correct the typo:
CurrentDb.Execute strSQL, dbFailOnError

This extra argument is important. Only omit it if you really know what you're doing.

PARAMETERS parOrderDate DateTime;
INSERT INTO tblBreakfastOrders ( TeacherID, OrderDate, Price, Brk100, StudentID, FSID )

ronrosenfeld say:

Changing the formatting of a cell does not change the type of value stored.

Even though you initially formatted the cell as TEXT, if you copy/paste a value that is a number, from another cell, that process does not change the type of value from number to text.

 

So, pasting a number into a cell that is formatted as text leaves the pasted number as a number (and the opposite is also true).

 

To enter a number as text, you need to either pre-format the cell as text (and then enter the number using the keyboard, not by pasting); or precede your numeric entry with a single quote (which will not show up in the cell, but instructs Excel that the entry is text).

ronrosenfeld say: I would think you could either work out a different process for the end users to get their data into the worksheet -- perhaps by exporting it rather than copy/paste, or convert it to text after they have pasted it.

A conversion macro would be simple, and should work no matter what the Excel version.  The limitation of a conversion macro is that, unless the numeric data is truly standardized with a fixed, known number of digits, there would be no way to recover a required leading zero.  If that is a problem, there will be no solution except to examine the process of data entry.

If macro security is an insurmountable issue, and absolutely precludes use of macros, you will have to change the process, as it is unlikely that this long-standing behavior of Excel will be changed.

But here's a simple macro that will convert all the cells in a range to text.

To enter this Macro (Sub), <alt-F11> opens the Visual Basic Editor.
Ensure your project is highlighted in the Project Explorer window.
Then, from the top menu, select Insert/Module and
paste the code below into the window that opens.

Note the Set r line that needs to be changed to reflect your data range.

To use this Macro (Sub), <alt-F8> opens the macro dialog box. Select the macro by name, and <RUN>.

===========================================
Option Explicit
Sub ConvertToText()
    Dim r As Range, c As Range

'you will need to change this next line to
'set r to your data range to be converted
Set r = Range("A1", Cells(Rows.Count, "A").End(xlUp))

For Each c In r
    With c
        .NumberFormat = "@"
        .Value = .Text
    End With
Next c
End Sub
