I suspect the root of the problem is explained in the article http://www.techsupportalert.com/content/why-are-your-photos-upside-down.htm. That is, the EXIF data in the pictures identifies the orientation of the pixel data in the files, and the Windows 10 File Explorer and most graphics editors use that information to automatically display the pictures right side up. However, Word and Word Pad ignore the EXIF data and display the pictures exactly as they were recorded, sideways.

I don't see anything in your code that would cause the error. When I run this code it works without error:

Dim aCellA, colact, colFileQ, aCellF As String
colact = 1
colFileQ = 4
aCellA = Cells(2, colact).Address(rowabsolute:=False, ColumnAbsolute:=False)
aCellF = Cells(2, colFileQ).Address(rowabsolute:=False, ColumnAbsolute:=False)

Did you copy the code as-is from your code window to the browser window? The reason I ask is that the VBE automatically proper cases the keyword "false", automatically makes all variable references the same case--in your code you have colFileQ and colFileq, and adjusts the case of any object references like "Cells".

Here are some tips for creating more solid and robust code.

Only declare one variable per Dim statement:

Dim aCellA As string

Dim aCellF As String

Dim colact As Long

Dim colFileQ as long

Always fully qualify Excel object references such as Cells. If you do not, Excel makes assumptions for you and sometimes those assumptions are not what you expect. So instead of Cells, use Me.Cells in the worksheet code module, or ThisWorkbook.Worksheets("Sheet 1").Cells if not.
