ken sheridan say: Possible other solutions would be:

1.  Filter the form rather than moving the record pointer to the first match.  The code for the AfterUpdate event procedure of the combo box would be:

Const MESSAGETEXT = "No matching record"

    Dim ctrl As Control
    Dim strFilter As String

    Set ctrl = Me.ActiveControl
    strFilter = "CStr(NSN)  Like ""*" & ctrl & """"

    If Not IsNull(ctrl) Then
        With Me.RecordsetClone
            .FindFirst strFilter
            If Not .NoMatch Then
                ' filter form
                Me.Filter = strFilter
                Me.FilterOn = True
            Else
                MsgBox MESSAGETEXT, vbInformation, "Warning"
            End If
        End With
    Else  
        Me.FilterOn = False
    End If

To turn the filter off you can use the built in Toggle Filter option on the Home ribbon, use the form's navigation bar, delete whatever is entered in the combo box, or add a Show All button to the form with:

    Me.FilterOn = False

in its Click even procedure.

2.  Add a second combo box whose RowSource references the first combo box as a parameter and requery it with:

    Me.YourSecondComboBox.Requery

in the AfterUpdate event procedure of the first combo box.  You'd then use the second combo box to move to the record.

ken sheridan say:

To allow partial values to be entered in the combo box you'd first need to set its NotInList property to False (No).  I don't have occasion to use macros, but in VBA you could then search by any sequence of characters of any length at the end of the value with code like this in the combo box's AfterUpdate event procedure:

 

    Const MESSAGETEXT = "No matching record"

    Dim ctrl As Control

   

    Set ctrl = Me.ActiveControl

   

    If Not IsNull(ctrl) Then

        With Me.RecordsetClone

            .FindFirst "CStr(NSN)  Like ""*" & ctrl & """"

            If Not .NoMatch Then

                ' go to record by synchronizing bookmarks

                Me.Bookmark = .Bookmark

            Else

                MsgBox MESSAGETEXT, vbInformation, "Warning"

As Jim says I think your formula should work.

But you say in your post "each file name is "Last name, First name - Occupation number".

If they follow this format then you potentially need to account for the comma and the dash, as well as any spaces in the filename (which are not added automatically) in your formula.

So for example if the filename is "Jones, Janet - 100.docx" then the formula needs to be:

=HYPERLINK("C:\Users\Dropbox\Database\Orders\Ma-Mg\" & B4 & ", " & C4 & " - " & A4 & ".docx", A4)

myall_blues say:

You need to add spaces in between quotes wherever they are required, but you can combine them with additional characters. For example, in my formula above, if you need a comma followed by a quote, put ", " in the formula (i.e. comma followed by space, all in quotes), or if you need the dash preceded and followed by spaces, put " - ".

May I suggest two ways to approach by building it up piece by piece from a position of strength (you can do all this in another workbook if you want).

1. So for example in one cell, hardcode the entire filename as you want it and test that it works

e.g. =HYPERLINK("C:\Users\Dropbox\Database\Orders\Ma-Mg\Jones, Janet - 100.docx","MyTest1")

Assuming it works, then in the next cell down, substitute out one part of the filename for a cell reference and test again

=HYPERLINK("C:\Users\Dropbox\Database\Orders\Ma-Mg\& B4 & ", " & "Janet - 100.docx","MyTest2")

and so on until you have a working hyperlink.

2. As above but drop the "=HYPERLINK" part and just make sure that the text produced in the cell is what you expect it to be.

e.g ="C:\Users\Dropbox\Database\Orders\Ma-Mg\& B4 & ", " & "Janet - 100.docx"

Once you get that right, add the HYPERLINK formula to it.

regards

Murray
