bob jones aka: cybertaz say:
I can't explain the why, but what you're seeing is how OS X works...

In order to have the Duplicate or Make Alias command appear in the shortcut menu you must have a Finder window focused on the folder that contains the file. When you Search, you're presented with an aggregation of files that not only could be in different folders but may be stored on different devices -- IOW, the focus of the Search window is not on any one specific folder.

You can still create a Duplicate of a file that appears as the result of a Search by holding Option while you drag the file to the Desktop or to a location in a different Finder window. The pointer will turn into a green circle with a plus sign when you get to a valid 'drop zone'.

If you edit a file created using Duplicate or Copy you're editing only that instance of the file. Other copies of the file are not modified.

An Alias, however, is simply a pointer to a specific file or folder, not a clone of what it represents... they're kinda like buttons. They allow you to keep an item stored in a preferred location in your filing hierarchy but have the convenience of being able to open it from wherever the Alias is put for convenience. You can also have as many Aliases of a particular file/folder in as many different locations as you wish.

An Alias can also be created from Search results by holding both Command+Option while you drag the found file to a location other than within the same Search window.

john p spencer say:

Possible, yes.

You would have to use the crosstab query as the source for another query.  Once you do that you can use the columns generated by the crosstab.

Another option would be to build the columns yourself.  You might use a query like the following to do that.

SELECT ProjectName
, Sum(IIF(TheYear= 2007,Amount,Null)) as [2007]
, Sum(IIF(TheYear=2008,Amount,Null)) as [2008]
, Sum(IIF(TheYear=2008,Amount,Null)) -Sum(IIF(TheYear=2007,Amount,Null)) as Difference
FROM [SomeTable]
WHERE TheYear in (2007,2008)
GROUP BY ProjectName
