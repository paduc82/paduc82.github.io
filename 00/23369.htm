Autocorrect Breaks AutoExpand in Combo Box

Opening a discussion to report a bug with Access 2010.


If autocorrect engages during autoexpansion of a combo box on a form, the autoexpansion is disengaged on a control that is text based.  In other words, as one is typing values that are in the list, the text following the cursor remains selected, whereas the text previous to the cursor is not.  The issue is that if an autocorrection takes place as one is typing, the cursor remains in the same place, but the following text becomes deselected, thus losing the autoexpansion of the control.

jay freedman say:
Jack, as Doug Robbins said in the first reply to this thread, document variables can be set, modified, or deleted only through VBA. That's one of the advantages they have over document properties: users can't mess with them. But it also says that you can't use the properties window to do anything to document variables.

If you want users to be able to change the values of document variables but not be able to delete them, you must provide a VBA-driven dialog (a userform) that can accept the new value and verify that it isn't blank before it sets the value in the file. (Setting the value of a document variable to the empty string deletes that variable from the file.)

To answer the other part of your question, you can't assign an object to a document variable. The value of a document variable must be a string. If you assign a number, that number will be implicitly converted to a string by VBA.

In your example, what you would have to do is

    ActiveDocument.Variables("Greenland").Value = "100"

or, equivalently,

    ActiveDocument.Variables("Greenland").Value = 100

If the object you have in mind has other properties besides a name and a (string) value, you would have to create a separate document variable for each one, like

    ActiveDocument.Variables("Greenland_Area").Value = "2166086"
    ActiveDocument.Variables("Greenland_Population").Value = "56370"

This becomes manageable if you keep a convention about how the variables are named, for example

    Dim country As String
    country = "Greenland"
    ActiveDocument.Fields.Add Selection.Range, wdFieldDocVariable, country & "_Population"
    country = "Iceland"
    ActiveDocument.Fields.Add Selection.Range, wdFieldDocVariable, country & "_Population"

Most serious developers would NOT use an Attachment for such images. An Access database is strictly limited to 2GByte (2048 MByte) in size, and it doesn't take too many megabyte and larger PDF files to exceed this limit; and when you exceed it, your database is toasted and may not be recoverable.

Just store the PDF files in a dedicated directory, as external files, and store the path and filename in a field (or two fields, for path and for filename) in your table.
