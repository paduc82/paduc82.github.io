twasbrilligand say: of a single RunCode command with the parameter "StartUp()". The StartUp function looked like this:

Public Function StartUp() as Boolean

    Application.SetOption "Error Trapping", 2
    DoCmd.ShowToolbar "Ribbon", acToolbarNo

I figured, based on some comments that appear elsewhere  in this thread, that the problem might be reference related, so I decided to play around with ways to fully qualify (if that's the right word) the commands in this function. I guessed right the first time. Now it looks like this:

Public Function StartUp() As Boolean

    Access.DoCmd.ShowToolbar "Ribbon", acToolbarNo

What's a special relief to me is that I didn't have to go changing all my code in this way - it only seems to be required within the function (StartUp, in my case) called by AutoExec.
