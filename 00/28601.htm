You can remove the Active designation on the partition by using the Diskpart Inactive command.

The fact the partition shows as Active means the Drive is configured as MBR.

I would probably clean the SSD with Diskpart and let Windows 10 decide how to set it up.  Make sure you have selected the correct drive before selecting a Clean command.  If you install in the UEFI configuration, the boot files will be placed in the EFI System partition with the OS being placed on the SSD and you will get the Dual Boot Menu.

If you were to remove the other drives and have an independent install of Windows 10 on the SSD, you could pick which OS you wanted to boot from the Boot Device Menu but you would not have a boot menu without additional commands being used.

Using a combo box to select records in a subform

I have two related tables. (Actually more than two, but if I can figure this out for two, I expect that I'll be able to apply the strategy to other tables.)

Well call the first table AllData with a field Field1

The second table will be Related1 with Fields called FirstField, SecondField and ThirdField

I want to display in a subform SecondField and ThirdField where FirstField equals Field1.

I've tried using a combo box for AllData, and then inserting a subform of Related1, after defining a connection between AllData.Field1 and Related1.FirstField

Selecting different entries in the combobox does not yield any changes in the subform.

What am I doing wrong?

Thank you,

David
wayne-i-m say:

No problems.

 

Just make a (main) form based on your AllData
Make sure you (somewhere - it can be hidden if you like) have the Linking field to the table the sub form will be based on subform.

Use the Combo Box wizard to make a combo that will search for a record in your form.

 

Next add a subform - again make sure the linking field is somewhere on the form - use this linking field to join it (master / child) to the main form.  - if you use the form wizard it will ask for this.

 

Now when you select a record in your combo on the main form it will go to that record - and so the subform will be linked.

 

Don't forget that you don't need to put any other controls on the main form (just a combo and the linking field control) if this is what you want.

 

Of course there are better ways to do this (there always are) but this is the simplest I can think of
