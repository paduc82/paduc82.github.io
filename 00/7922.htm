rich007a say:

OK, to test, first put this in cell A1:

   =IF($A$53*MOD(ROW(),2)=0,ROW(),"")

and put 1 in cell A53.

Now hit Alt+F11 and put this code in the ThisWorkbook module for your file:

Option Explicit
Private Sub Workbook_SheetCalculate(ByVal Sh As Object)
'Hide Rows If Col A Is Blank
'
    Dim c As Range, rng As Range, WS As Worksheet, originalSht As String
    Application.ScreenUpdating = False
    originalSht = ActiveSheet.Name      'Store name of current sheet
    Set WS = Worksheets("Sheet1")       'The sheet where the filter will be
    Set rng = WS.Range("A1:A50")        'The range to be filtered
    WS.Activate                         'First activate the sheet
    For Each c In rng                   'Loop through each cell in the range
        If WorksheetFunction.IsError(c.Value) Then    'Check for error in cell (must check first)
            c.EntireRow.Hidden = False  'Show row
        ElseIf c.Value <> vbNullString Then         'If blank then...
            c.EntireRow.Hidden = False  'Show row
        Else
            c.EntireRow.Hidden = True   'Hide row
        End If
    Next c
    On Error Resume Next                'Disable error handling (incase we delete the current sheet)
        Sheets(originalSht).Activate    'Go back to the original worksheet
    On Error GoTo 0                     'Re-enable error handling
    Application.ScreenUpdating = True
End Sub

 

Now go back to the worksheet and change A53 to 0.  Then back to 1.  This is going to update whenever any sheet in that workbook is recalculated.

If you want to make this update occur manually, place the same code in a normal module like you did previously, and don't use "Workbook_SheetCalculate" as the title for the sub.

We have to be very careful about handling errors in this, since it will run everytime you do almost anything in excel.  To suspend it, go to the code, select all the code and comment it out or go to the Immediate window (hit Ctrl+G in VBA if not visible) and type

    application.EnableEvents=False

(set back to true to re-enable).

 

Hope that helps.

Cheers
Rich

bob jones aka: cybertaz say:

You've mixed some terms, so your question isn't really clear. Maybe this will help :-)

I believe the specific feature you're looking for is Insert> Signature Line, although the name differs a bit depending on the Windows version of Office. That feature is not available in the Mac version of Word 2016.

Even in Word for Windows, though, the feature neither inserts nor enables inserting a handwritten signature. All it provides would better be described as a signature block. The feature permits entering the signatory's name, title & email address along with a few other pieces of information into a dialog box. All that is inserted into the document as typed text is a graphic providing a line for a handwritten signature to be written after the document is printed :

Writing a handwritten signature into the document requires a digital pen along with a tablet device or touchscreen display. The Windows OS supports it via the Inking service: OS X supports it via what is called Ink, but both require specific devices in order to function.

Unless equipped as described above, a digital copy of a handwritten signature has to be inserted just as with any other picture. The image can be acquired by a scanner, a digital camera, or done with drawing tools in a graphics program, but it has to be saved in an image file format (preferably PNG or PDF).

If you need to use the same one repeatedly, you can create an AutoText item from an existing document which then can be inserted into others as necessary... but you have to have the image of the handwritten signature first :-)

rick rothstein, mvp - excel say:

I decided to add one line to the code I posted earlier to stop the loop from copying blank cells into blank cells. Here is the entire modified macro (which now only loops as many times as there are cells in the Source range as opposed to looping for as many Areas as there are in the Blanks range)...

Sub CopyDataIntoBlankCells()
  Dim Blanks As Range, BlankArea As Range, Source As Range, Destination As Range
  Dim PositionSource As Long, PositionDestination As Long, LastRow As Long
  '***********************************************************************
  '  Set the following constants and variable to match your actual layout
  '***********************************************************************
  Const DestinationColumn As String = "L"
  Const DestinationStartRow As Long = 408
  Const DestinationEndRow As Long = 2810
  Set Source = Range("A387:A412")
  '***********************************************************************
  Set Destination = Range(DestinationColumn & DestinationStartRow & ":" & DestinationColumn & DestinationEndRow)
  Set Blanks = Destination.SpecialCells(xlCellTypeBlanks)
  LastRow = Cells(Rows.Count, DestinationColumn).End(xlUp).Row
  Set Blanks = Union(Blanks, Range(DestinationColumn & (LastRow + 1) & ":" & DestinationColumn & Rows.Count))
  For Each BlankArea In Blanks.Areas
    PositionDestination = PositionDestination + 1
    Source.Offset(PositionSource).Resize(BlankArea.Rows.Count).Copy Blanks.Areas(PositionDestination)(1)
    PositionSource = PositionSource + BlankArea.Rows.Count
    If PositionSource >= Source.Rows.Count Then Exit For
  Next
End Sub

One way to "turn off" spell checking in OneNote Web App is to select the text you don't want spell checked and then set the proofing language to one that doesn't support proofing (Latin is the one I usually choose). Unfortunately, you have to do this on every page.

Another option that might work for you is to set your Windows Live language to Dutch. OneNote selects the language to use for spell checking on newly created pages based on the Windows Live language. I'm guessing yours is set to English, so when you create a new page, all text is spell-checked against English. If you change your Windows Live language to Dutch and create a new page, it should use the Dutch spell checker (although in that case any English text you type will be misspelled unless you change the proofing language to English using the method Rob described above). Of course, if you do this, then your Windows Live and Office Web App UI will appear in Dutch as well, which you may or may not want.
