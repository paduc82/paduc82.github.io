roady [mvp] say:

Which version of Outlook are we talking about here?
I'm going to assume Outlook 2010 here.

 

1) Not possible since your contacts are located in a different data store. The Activities feature only applies to items which are stored within the same data store. As you are using IMAP, which only supports mail folders, you can't store contacts within the data file of an IMAP account.

Note that the Activities feature is a deprecated feature which has been removed completely in Outlook 2013. The People Pane (or Social Connector) is the replacement feature.
FWIW: Outlook 2013 does offer you to store Contacts in the IMAP account store and with a macro, you could re-enable the Activities feature in Outlook 2013.
See: http://www.howto-outlook.com/howto/contactactivities.htm

 

2) That you have 2 data stores is normal in Outlook 2010 and previous since IMAP accounts do not support non-mail folders such as Calendar and Contacts folders.
See: http://www.msoutlook.info/question/502
By default, Gmail will already automatically store a copy of your Sent Items in the Gmail store via their SMTP server. You can therefor disable the option for Outlook to store a copy as well.
See: Duplicate Sent Items when using Gmail

 

3) Are you sure you've subscribed to all the correct fodders? The "All Mail" folder is something different than the Inbox folder. Also, by default, deleting an email in Gmail doesn't actually delete the email but it only removes the label of the mail item (which are exposed as folders in Outlook) and it will still show up in the All Mail folder.
Additionally, check your IMAP settings in the web interface of Gmail. There is an option there to limit the amount of items for IMAP folders.

What you envision could be done with some VBA programming. But a simpler approach would be to use PowerPoint's Zoom feature.

Create full-size slides of the large version of the images. Then hide those slides so they don't appear as part of your presentation.

Create a new slide to be your index or thumbnail slide. Click on Insert>Zoom>Slide Zoom and pick the first large image slide. A thumbnail will be created, you can resize and reposition it.

Invisible buttons are AutoShapes with no fill and no line that have a hyperlink attached to them. You use them for two purposes:
    If you’re navigating through the presentation, you may want your navigation to seem invisible to create a magical effect. The button may be small or may cover the entire slide.

alex chen msft say:

Hi Faisal,

 

Thanks for your query in this community.

This is an expected behavior, when you click on protect in OWA, it enable the Azure Information Protection, the 4 default labels there won't give user Full Control permission on the email attachments, thus, the recipients won't be able to unprotect the Sheets you sent to them because their permission level is limited.

 

We would recommend you create new label/edit existing label in Azure Portal > Azure Information Protection.  The following article will give you an introduction on this procedure.

Admin Guide: Custom configurations for the Azure Information Protection client

 

Click on the azure cloud key and assign AuthenticatedUsers > Co-Owner permission.

 

 

Then after the recipients verified their identities, they should be able to unprotect the worksheet you sent to them.

doug robbins - mvp office apps & services (word) say:

In that case, you could use a macro button that ran a macro to open the file.  The Display Text for the macro button could be the name of the file that you want to open and you could either hard code that into a

Documents.Open "File Path\File Name"

code or, if you have multiple files that you want to open, you can get by with a single macro that parses

the result of

Selection.Fields(1).Code

to get the name of the file to use in a Documents.Open command.

The display text could be formatted to look like a hyperlink.
