david marcovitz say: Of course, this is doable with VBA, but that, of course, requires VBA. If you want to do this with trigger animations, I think you want to have copies of the image in its different locations. You can then use appear and disappear animations. Clicking on a button should trigger the correct version of the image to appear and the incorrect ones to disappear. This is already a lot of trigger animations. That means that anything that moves the shape to the top needs to make a left arrow button appear that moves the shape to the top left, and anything that moves the shape to the bottom needs to make a left arrow appear that moves the shape to the bottom left. Using triggers, you are going to end up with dozens of animations to cover about a dozen different shapes. VBA is looking better all the time. The whole thing could probably be done with about a dozen lines of code.

Following code sets the default date and time to current date and 8am when the user enters the control. You can place the actual line of code where ever it suits you. ie. Form initialize.

 

Private Sub DTPicker1_Enter()
    Me.DTPicker1.Value = Date + TimeValue("08:00:00")
End Sub

As a suggestion you could set the default date and time in the Form Initialize event and then use a double click event to change UpDown mode so you can pick the date in the calendar mode and then edit the time in UpDown mode. The double click event simply toggles the UpDown mode.

 

 

Private Sub UserForm_Initialize()
    Me.DTPicker1.Value = Date + TimeValue("08:00:00")

Private Sub DTPicker1_DblClick()

    DTPicker1.CustomFormat = "dd MMM yyyy hh:mm"
End Sub

jllatham say:

An unusual one.  I think we're going to need to see the data validation rule for the cell in question.  So you need to look at the code and find out what cell dvCell is.  You can use the immediate window to do so - let it go into debug and in the Immediate Window type:

? dvCell.Address [Enter]

that will give you the cell address, and then to find out what sheet it is on, type

? dvcell.parent.name[Enter] and that will give you the sheet name

finally you can find out what that formula is by typing

? dvcell.validation.formula1[Enter]

And it is the .formula1 information that we pretty much need to see and then we can try to figure out why things aren't working.  The cell address and sheet name are actually kind of extra information that may or may not play into it later.

Now if the data validation rule is to get a list or address, then the evaluate() formula is probably going to result in that address. 

I set this code up in both Excel 2007 and 2010 on 2 separate machines and it actually works fine for me; myRange gets set to a reference to $K$1:$K$4 --

Sub Test()
  Dim dvCell As Range
  Dim myRange As Range
 
  'Range G2 on the sheet has data validation
  'set to a list with the formula being
  ' =$K$1:$K$4
  Set dvCell = ActiveSheet.Range("G2")
  Set myRange = Evaluate(dvCell.Validation.Formula1)
  'at this point myRange will reference range K1:K4
  'for me this works in both Excel 2010 and 2007
 
End Sub
