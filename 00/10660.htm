rohn007 say:

Minor trick.

The data on the "Student List" tab is formatted as an Excel Table named "Students".

You can tell this by clicking anywhere inside the data. You'll see the Tables Design Contextual Tab.

Click on it, in the Properties group (first group on the tab) > Table name: field displays the name, ie "Student".

To insert a new column, place the cursor where you want it to appear.

Home tab > Cells group > Insert drop down > select either "Insert Sheet column" or "Insert Table Column left"

Put the new column name in the appropriate row.

Enter some test data in your test row, ie first row

Go to the "Student Details" tab

In row 14, immediately below "Physician Phone" enter the new column name

Click inside the data portion of the row above. It will show this formula

=IFERROR(VLOOKUP(StudentName,Students,9,FALSE),"")

Copy the formula, paste it into the new row.

Edit the formula on the new row, change "9" to "10"

The new column data from the test row should now be displayed.

After that you can fiddle with the Format Painter to copy the outlines to the new location

*****************

Since the table runs off screen, you may want to make data entry easier.

Add the "Forms" button to the QAT. With the pointer inside the table, click on the Forms button. A small dialog box listing all of the table fields will be displayed. You can enter data, edit data.

teylyn say:

I know SharePoint fairly well. It's easy to create fine-grained access on a document level, although it can quickly become messy and hard to maintain. 

Using SharePoint groups to govern permissions to a range inside an Excel spreadsheet is not possible, as far as I know.

As a general rule, my SharePoint groups NEVER have individual people in them, but only Active Directory groups.

If the group already exists in Active Directory, it can be used for "Allow users to edit ranges" as well. 

That approach takes a lot of maintenance work out of SharePoint and allows for range permissions in Excel, too.
