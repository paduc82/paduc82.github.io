[SOLVED] MsMpEng.exe (Windows Defender) continuously - Microsoft Community

alexandre - say:

[SOLVED] MsMpEng.exe (Windows Defender) continuously consuming high CPU for days

Since friday (Oct 13, 2017), my Windows 10 computer was spending a lot of CPU with MsMpEng.exe (Windows Defender), without stop. Using Process Explorer, I could see that there were two threads, each of them consuming 100% of one of the 4 cores in the Intel Core i5-5200U, so I had two cores always busy. One of the threads had "Start Address" = "!RtlUserThreadStart" and the other, "0x0" (I know these information are not of much use, although I wanted to put here everything I have, because I had already searched a lot and didn't find anything similar).

What I had already done:

  • Reboot machine (as soon as it finished starting up, MsMpEng.exe started consuming CPU again);
  • Checking that MsMpEng.exe itself was in Windows Defender's exclusions

When writing this answer, I opened Windows Defender's exclusions again, so I could list here all exclusions already set. Then I've realized I had exclusions for older versions of Visual Studio Community and SQL Server Express, but not for current ones (Visual Studio 2017 and SQL Server 2016). So I've added those .exe to the exclusions and, after rebooting, I finally have my machine back again!

I've written this so if someone else has the same problem and doesn't find the answer, please try the same solution.

Regards.

alexandre - say:

As I've said, I've posted this answer so anyone else with the same problem can get help from it.

What I've done to solve it:

  • Add "C:\Program Files (x86)\Microsoft Visual Studio\2017\Community\Common7\IDE\devenv.exe" to Windows Defender's exclusions;
  • Add "C:\Program Files\Microsoft SQL Server\MSSQL13.SQLEXPRESS\MSSQL\Binn\sqlservr.exe" to Windows Defender's exclusions;
  • Reboot.

I already had exclusions for older versions of Visual Studio and SQL Server Express, but I've upgraded and forgot to exclude newer versions.
