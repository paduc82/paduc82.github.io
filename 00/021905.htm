How does "Excluded processes" work, exactly? - Microsoft Community

oenophile say:

How does "Excluded processes" work, exactly?

When a process is added to the list of "Excluded processes", what does this mean?

1. The process file itself is excluded from detection.

2. All files created or accessed by the process file are excluded from detection.

3. Both of the above.

If the answer is (1), how do "Excluded processes" differ from "Excluded files and locations"?

jeff toole say:

If you exclude process, then the process itself will not be scanned, and any file operations it performs will not trigger a scan.  However, if the process creates a file, and another non-excluded process accesses that file, it will be scanned.

 

Example:

You exclude notepad.exe

You use notepad.exe to save SomeDocument.txt

At this point, MSE will not scan SomeDocument.txt, because you have excluded notepad.exe

You open SomeDocument.txt with wordpad.exe

At this point, MSE will scan SomeDocument.txt, because it was accessed by a non-excluded process.

 

Hopefully that makes sense.  Feel free to reply if you still have questions.

Instead of passing the name of the form, pass a reference to the form
object...
 
For example, instead of:
 
Function DoSomethingWithForm( strFormName as String )
Dim frm as Form
Set frm = Forms( strFormName )
' do something with frm
End Function
 
Call DoSomethingWithForm( Me.Name )
 
do this:
 
Function DoSomethingWithForm( frm as Form )
' do something with frm
End Function
 
Call DoSomethingWithForm( Me )
 
One problem with multiple instances of forms is how to close them.
DoCmd.Close identifies forms by name, so if you have the same form open
twice then this command:
DoCmd.Close acForm, Me.Name
will not necessarily close the form in which the code is executing.
 
You can use the following code which sends an API command to the specific
window:
 
============== start code ==============
Option Explicit
 
Private Declare Function SendMessage _
Lib "user32" _
Alias "SendMessageA" ( _
ByVal hWnd As Long, _
ByVal wMsg As Long, _
ByVal wParam As Long, _
lParam As Any _
) As Long
 
Private Const WM_CLOSE = &H10
 
Public Function frm_Close( _
ByRef frm As Access.Form, _
Optional fQuiet As Boolean _
) As Long
' Returns 0 if successful, otherwise an error value
Dim lRet As Long, hWnd As Long, Name
On Error GoTo ProcErr
hWnd = frm.hWnd
Call SendMessage(hWnd, WM_CLOSE, 0&, 0&)
1:
If frm.hWnd = hWnd Then
' form still open
Err.Raise 2501, , "Form '" & frm.Caption & "' may not be closed at this
time"
End If
ProcEnd:
If lRet = 0 Then Set frm = Nothing
frm_Close = lRet
Exit Function
ProcErr:
If Err = 2467 And Erl = 1 Then
' form was successfully closed
lRet = 0
Else
If Not fQuiet Then MsgBox Err.Description, vbExclamation, "Cannot close
form"
lRet = Err
End If
Resume ProcEnd
End Function
=============== End Code =================
 
Call it like this:
 
frm_Close Me
