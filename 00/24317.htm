a. user say:

I mention this only because an increasing number of people have access to some form of SharePoint system these days. If you are one of them, you may not be aware of this facility. But even if you do not, it may be worth exploring this approach.

If you have access to a SharePoint system and are able to create/modify Document Libraries, then you can get some (possibly even all) of what you are looking for by creating a Library that has one column for each item ("Customer" etc.) that you need.

When you create a document in that library, SharePoint adds these columns to the list of Document Properties in the Insert->Quick Part list. You do not have much control over what is inserted - the quick part is generated by Word.

Word will also display these values in the "Document Information Panel" (DIP) and let you edit them. Again, you get little control over what is in this panel

The thing is that although this stuff is designed to work in a SharePoint environment and not all of it works correctly outside that environment, quite a lot of it does work. If you want an example, here is a document I created from a Library with three columns defined by me:

https://www.dropbox.com/s/f8pdyo99o3jtbsu/2014123101%20three%20items%20wd.docx?dl=0
Creating that required no VBA coding. Using it requires no VBA. I had to have the SharePoint site, create the Library, create the columns in the library, create a new document, open it in Word, add the sample tables, and save it locally. You could also save it as a .dotx if you prefer.

As far as I know, the content controls created by this process always work in Word 2007/2010/2013 even without SharePoint. I am fairly sure that the basic DIP will also work, but it has never been completely clear that the DIP will work with all versions of Office for Windows (2007 and later, anyway). (In a SharePoint environment where the users have InfoPath, it is possible to customize the DIP). You may also be able to see that the DIP is in effect in two parts - "builtin properties" and "user-defined properties". If you needed anything more complex than that, you would need VBA, as the others have said.

Obviously, if you don't have access to a SharePoint system, you will not be able to create such a document using this method (or maintain it, e.g. if you need to add or remove a column). In principle, you could create one using VBA, but I do not have the necessary code.

You mentioned Office, then focussed on Word. Only Word has the facility to insert data (usng content controls) that is connected to a "data store" in this way. Excel and PowerPoint can have such a data store, but they do not have any code-free mechanism for displaying the values in the store in documents (worksheets/presentations).

This happened to me after moving my Outlook PST files.  

If you have moved you .PST files and are getting the 0x8004010F error here is what fixed it for me.

  1. You need to change the delivery folder within the PST file temporarily.   Under Account Settings click on the E-Mail tab.

  4. Call the new folder Temp
  5. Click OK and then Close
  6. Then send / receive and the error is gone.
  7. After that you can change the delivery folder back to original.  Typically you would deliver to the "Inbox" folder within the .PST.
