ken sheridan say: Calling the DLookup function in that way is certainly very inefficient.  Apart from the multiple function calls, domain functions are per se notoriously slow.  A query in which the necessary Boolean logic is contained in the WHERE clause to return the row or rows which meet the criteria on the multiple columns is a single set operation and consequently far more efficient on both counts.  The query would not necessarily be a saved querydef object, but could be an SQL statement built in code.  What you then do depends on how you wish to return the data.  It could be shown in a report or form, and in the latter case this could be updatable if required, though this would affect the design of the query as not all queries are updatable.  If you wish to refer to individual column's values from the returned row(s) then, as John points out, a recordset object could be established on the basis of the SQL statement and its individual columns examined in code.

If you can describe in more detail what you are attempting to achieve in terms of the functionality of the database, we can probably be more categorical in our advice.
