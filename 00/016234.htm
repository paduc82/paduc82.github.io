ken sheridan say: You say that the data are durations in seconds.  The date/time data type is implemented as a 64 bit floating point number representing the number of days and fractions thereof. So, to convert a duration in seconds to a value of date/time data type capable of being formatted as 'nn:ss' you would need to divide the value by 86400, the number of seconds in a day.  You can see this with your example of 3:25, which is 205 seconds:

? Format(205/86400,"nn:ss")
03:25

If the downtime data are being computed by subtracting the start of the downtime from the end of the downtime, both as a date/time data type, converting the resulting value to seconds is unnecessary; you can simply format the result of the subtraction, which with this example would be a value of 0.00237268518518519:

? Format(0.00237268518518519,"nn:ss")
03:25
