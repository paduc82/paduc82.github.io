stefan blom say:

To remove a header, you have to insert a section break at the end of the preceding page (Page Layout tab | Breaks | Next Page) and then enable the "Different first page" option on the Header & Footer Tools Design tab (which will be active if you double-click the page header).

To remove page numbers, you also need a section break, or you'll need two section breaks if you want to remove the page numbers from one or more pages in the middle of a document. In the specific case where you want a front matter, see http://word.mvps.org/faqs/formatting/front_matter_2007.htm.

Be aware that text won't flow from one section to the next. Each section is a separate "container."

bob jones aka: cybertaz say:

The status of the check box for All is determined by whether non-printing characters are being displayed: When they are being displayed the box is checked: if not the box is cleared. IOW, if you clear the box but then later use the ¶ on the Ribbon or the Command+8 shortcut, you'll find that the check is restored if you reopen the Preferences.

The individual screen elements included in that group, however, are not directly available for assignment of keyboard shortcuts. They need to be toggled by going into the Preferences or perhaps by some programmatic means. To clarify [only because it's a bit illogical to many users :-)] if those boxes are cleared the element will display when All is toggled on. If the box is checked for any element that element will display at all times (in documents where ir exists*), regardless of whether All is toggled on or off. [* Obviously not all documents contain Hidden Text, Optional Hyphens or Tab Characters.]

I can think of only 2 reasons why the status of any individual element would change without having been selected by the user:

  • Occasionally, Updates appear to cause changes to a few Preferences settings. I can't confirm this but based on years of experience it's the only explanation :-)
  • A corrupt Preferences [.plist] file

macropod say:

Adding a long string (or replacing an existing string with a long one isn't a significant issue - it just requires a slightly different coding approach. For example, in the link I posted, you could replace the:

  For i = 1 To UBound(Split(xlFList, "|"))

   ...
  Next

block with:

  For i = 1 To UBound(Split(xlFList, "|"))

    With wdDoc.Range
      With .Find
        .ClearFormatting
        .Replacement.ClearFormatting
        .MatchWholeWord = True
        .MatchCase = True
        .Wrap = wdFindStop
        .Text = Split(xlFList, "|")(i)
        .Execute
      End With
      Do While .Find.Found = True
        .Text = Split(xlRList, "|")(i)
        .Collapse wdCollapseEnd
        .Find.Execute
      Loop
    End With

  Next

 

As you have pipe characters in your data, though, you'd need to change the ones in the macro code to something else (e.g. §)
