Lock Table to Insert Data - Microsoft Community

k1n say:

Lock Table to Insert Data

Hi Everyone.
I want to lock the data entry in the Access application so that it can be stored in the table only through the form and if the user clicks on the submit button.

scott mcdaniel say:

You really can't do this with Access. If a user can open the Access file, then they could potentially have access to your tables. You can deploy a compiled file (.accde, or .mde), and that would stop them from modifying forms, reports, and modules, but not tables.

You can takes steps to reduce the chance of this happening, like hiding the navigation pane. However, if someone links to your database tables, they could easily modify data in those linked tables.

About the only way to stop this is to move your data to a more secure platform, like SQL Server, and implement security on that platform. You could certainly stop anonymous access to a SQL Server, unlike Access.

As far as a Submit button, that would depend on how your data is structured. If you Forms with Subforms, this can be tough to do, since Access will save your Main Form data when you move into the subform, and the only way to remove that Mainform data would be to delete it.

There are several approaches you use to do this. One of the more common is to use temporary tables to hold the data your user is working with (and the form is bound to), and then write that data back to the live tables when the user clicks the Submit button.
