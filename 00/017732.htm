andreas killer say:

A very general question... I'll make a simple sample and show you some steps to get from a recorded macro to a real macro.

At first here is a good tutorial for beginners:
http://www.wiseowl.co.uk/blog/s161/online-excel-vba-training.htm

Before you run any macro below read the section "Debugging and Error-Handling", because it is much more easier to understand how a macro works when you debug it. Look into the sheet after a line is executed.

Okay, let us assume your sheet looks like that:

And now you want to insert 2 rows below each number and group that rows. We can do that manually and get a macro like this one (I've inserted blank rows to reflect the lines with the numbers):

Sub Makro1()
  Rows("5:6").Select
  Selection.Rows.Group

  Rows("4:4").Select
  Selection.Insert Shift:=xlDown, CopyOrigin:=xlFormatFromLeftOrAbove
  Selection.Insert Shift:=xlDown, CopyOrigin:=xlFormatFromLeftOrAbove
  Rows("4:5").Select
  Selection.Rows.Group

  Rows("3:3").Select
  Selection.Insert Shift:=xlDown, CopyOrigin:=xlFormatFromLeftOrAbove
  Selection.Insert Shift:=xlDown, CopyOrigin:=xlFormatFromLeftOrAbove
  Rows("3:4").Select
  Selection.Rows.Group

  With ActiveSheet.Outline
    .AutomaticStyles = False
    .SummaryRow = xlAbove
    .SummaryColumn = xlRight
  End With
End Sub

Before you continue, copy that macro in a regular module, create a sheet as shown above and debug that macro to get some practice.

As you see I've inserted the rows from the bottom to the top. That is easier, because the next row is always one up. In the other direction you have to skip over the inserted rows... which can be complicated.

As you see in the macro above the row numbers in goes also just one up, so the first idea is "How can I have this row numbers variable?"

Sub Makro2()
  Dim i As Long
  For i = 5 To 3 Step -1
    Rows(i & ":" & i).Select
    Selection.Insert Shift:=xlDown, CopyOrigin:=xlFormatFromLeftOrAbove
    Selection.Insert Shift:=xlDown, CopyOrigin:=xlFormatFromLeftOrAbove
    Rows(i & ":" & i + 1).Select
    Selection.Rows.Group
  Next
End Sub

That works, but doing it that way is a bad idea. Why? Because the use of SELECT and SELECTION.

Please never use SELECT, SELECTION, ACTIVECELL, it is slow and error prone. Always refer to the objects directly.

I would not like to elaborate the details, please trust my experience of 30 years in programming.

Okay, how to refer to objects and what is an object? Almost anything what you see on your screen is an object! And objects have names and types. A cell in Excel is a Range object. (And much more! A Range object can also refer to many cells.)

Have a look at this macro and read the comments carefully. If you want get help for some words eg. "Set", place the cursor on that word and press F1. Debug the macro and look what happens in the sheet:

Sub Makro3()
  Dim This As Range, Below As Range

  'Refer to the last used cell in column A
  Set This = Range("A" & Rows.Count).End(xlUp)
  'Start a loop
  Do
    'This is the cell below
    Set Below = This.Offset(1)
    'Insert 2 rows above in one step:
    Below.Resize(2).EntireRow.Insert Shift:=xlDown, CopyOrigin:=xlFormatFromLeftOrAbove
    'Note:
    '  'Below' moves down with the cells, so we have to refer the cell again:
    Set Below = This.Offset(1)
    'Group the inserted cells
    Below.Resize(2).EntireRow.Group

    'Go one row up
    Set This = This.Offset(-1)
    'We are finished if we are at the top row
  Loop Until This.Row = 1
  'We like to have the + sign at the top
  ActiveSheet.Outline.SummaryRow = xlAbove
End Sub

The same macro, but a little shorten is this:

Sub Makro4()
  Dim This As Range

  'Refer to the last used cell in column A
  Set This = Range("A" & Rows.Count).End(xlUp)
  'Start a loop
  Do
    'Insert 2 rows below one step
    This.Offset(1).Resize(2).EntireRow.Insert xlDown, xlFormatFromLeftOrAbove
    'Group them
    This.Offset(1).Resize(2).EntireRow.Group
    'Go one row up
    Set This = This.Offset(-1)
    'We are finished if we are at the top row
  Loop Until This.Row = 1
  'We like to have the + sign at the top
  ActiveSheet.Outline.SummaryRow = xlAbove
End Sub

ashish mathur say:

Hi,

 

Your SUMIFS() formula should become:

 

=SUMPRODUCT(($B$3:$B$312="C/O")*($D$3:$D$312="Mazda")*($AI$3:$AI$312))

 

Your COUNTIF() formula should become

 

=SUMPRODUCT(($B$3:$B$312="C/O")*($D$3:$D$312="Subaru")*($M$3:$M$312="Y"))

 

These will work in Excel 2003 and prior version
