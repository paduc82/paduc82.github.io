Outbox and IMAP - Microsoft Community

gvm7777 say:

Outbox and IMAP

Can someone explain please how outbox synchronization and the sending of emails works? I understand, for example, that when an email is 'sent' and the outlook client is offline, then the email initially remains in the local outbox. When the client comes online, the local outbox is synchronized with the outbox on the IMAP server. Consequently, the emails are actually sent using data on the server. Is that right?

If this is right, then when the client presents the message to the effect that it is "sending/receiving email", then it is really the server that is doing the sending, not the client. Right?

TIA .... Greg

brian tillman [outlook mvp 2007-2019] say:

There doesn't need to be any Outbox on the server, so, at least for Outlook, there is no synching of the Outbox. The Sent Items folder, however, is synched.

Both the client and the server participate in the sending. The client sends to the server via SMTP and the server then forwards the mail, also by SMTP, to whatever the next mail transport in line is, until it reaches the recipient's mail server. When the client says it's sending/receiving mail, it's doing just that. It's checking the mailbox for new messages and it's sending messages it has to the server.

1) Here is how most people would do it.

Working from a copy of the file, convert everything to values using Copy/ pastespecial values. Then copy the second table to bottom of the first sheet, and sort that sheet based on your column of Yes and No values, and delete the rows that have No. Then sort your table based on column A as the primary key, and column B as the second key (either ascending or descending, depending on if you want the debit or credit first). 2) Here is how you should do it

Use a pivot table, using the multiple consolidation ranges option. Add your 2 tables, including your column of formulas that return Yes and No, then create the pivot table. Use the Yes and No values as the page filter, set to Yes, column A as the row field, and the debit and credit as the data fields set to sum. You will only have one row per entry instead of two.
