ken sheridan say: Using the DSum function, which, unlike a subquery, will keep the query updatable, the criteria for the function need to firstly restrict the rows examined to those for the current customer; then for the same year and month of the transaction date; and finally for those dates before or equal to the current transaction date.  So it would be like this:

RunningSum: DSum("Sales", "qryF09SalesbyMonth", "[CustomerID] = " & [Customer ID] & " And Year([Date]) = " & Year([Date]) & " And Month([Date]) = " & Month([Date]) & " And [Date] <= #" & Format([Date],"yyyy-mm-dd") & "#")

I note that you are referencing a query when calling the DSum function.  This is probably unnecessary as a Transactions base table could be referenced, the criteria restricting the result to the relevant year and month.

Note how the date literal is formatted to the internationally unambiguous ISO format for date notation of YYYY-MM-DD.  I would strongly advise against using Date as a column name, however, as it is the name of a built in function (returning the current date), so could be confused.  If you do use it be sure to wrap it in square brackets when referencing it.  But it would be much better to use a more specific term such as TransactionDate.

One other point is that if there is more than one transaction by the same customer on the same day the expression would give the daily closing balance, not the balance per transaction.  To do the latter would mean that the transaction date would have to include a time of day and be unique per customer, or another column such as a unique TransactionID autonumber column would need to be brought into play to arbitrarily distinguish between transactions by one customer on one day.
Ken Sheridan, Stafford, England

barrywallis say:

To clarify, you have two options to permanently opt out:

1) Wait until the next public release (probably a better term than production), the Anniversary Update due sometime around the end of July. Once that is released, you can upgrade to it and opt-out.

2) Restore your system to the previous public release, Build 1511 also called the November Update, now and apply public any updates that have been released since it came out via Windows Update.

Anything other than these two options, means you are still using an Insider build with the attendant instability.

cincinnatux say:

There are plenty of Web resources to help you with the Media Creation Tool. Try this: http://www.thewindowsclub.com/windows-10-media-creation-tool-create-installation-media-upgrade

The Media Creation Tool is a way of taking a *.ISO of Windows 10 and putting it on a USB thumb drive, which you can then boot from to install Windows 10 on a computer (assuming the BIOS settings permit USB booting). Unlike other dedicated media creation tools, however, this one also allows you to "upgrade" your existing system. I've never attempted to use this tool for an upgrade, but if you do so it will theoretically leave all your software and data intact and just swap out the OS. If you use the USB-booting option, however, it will erase EVERYTHING on your hard drive and you will be left with a factory-fresh Win10 OS.

In sum: the tool can keep your stuff intact, but only if you select the "upgrade" option. (I'm attempting this now; I will report back when I know if it worked.)

To recover your email do the following
1. Sign in to your Outlook.com account (via a Computer/Laptop).
2. Go to your Deleted folder (found on the left pane under Folders).
3. Look for "Lost a message?", and click "recover deleted messages".

You can also check out the following article for more information on how to recover lost/deleted messages from your Outlook.com account:
What should I do if email messages are missing from my Outlook.com account?
http://windows.microsoft.com/en-US/Outlook.com/how-recover-deleted-messages  

We can only recover recently deleted emails, specifically not more than 5 days old. Beyond that won't be possible.
Outlook will recover as many messages as possible and put them in your Deleted folder. If you don't see the deleted messages that you wanted, it means they are permanently lost.
