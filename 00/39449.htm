frobnitz say: Got it sorted, with a bit of help from other threads.

In the Application event log (Windows Logs > Application), I filtered (right-click, Filter Current Log...) by Event sources Desktop Window Manager and DesktopWindowManager-Diag.

The first event I got to was an Event ID 9011, Desktop Window Manager:

The Desktop Window Manager was unable to start because a mirroring driver is in use.

That's strange, thought I. I had already checked in the Services snap-in, and the Desktop Window Manager Session Manager indicated it was Started. And the Desktop Window Manager Session Manager stopped and restarted via the command line with no apparent problems:

net stop uxsms
net start uxsms

But winsat features says: DWM not running!

I recalled seeing a "mirror driver" related to LogMeIn in device manager > display adapters.

Uninstalled this device, rebooted, and wala! Aero transparency works again!
