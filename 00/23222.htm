Hiya Jerry,

Just a thought... have a look in Internet Explorer's Tab settings.  There are a few ways the Internet Explorer opens new tabs, so this is worth a quick look, and maybe a little experimenting.

To check these settings do this...

Open a web page, and look at the top left of the screen, and you'll see File Edit View Favourites And Tools.

Click Tools.

In the list of tools that opens, click the last one that will be Internet Options.

Half way down and to the right of that window, you'll see a button that says Tabs.  Click it.

Now then, just under half way down this tab settings window, there's a setting that says, 'When a new tab is opened, open:'

If that setting is currently on 'A blank page', I wouldn't be surprised if that is your problem.  I have mine set to 'Your first home page', and I don't get any issues when I click links that should open in a new tab.  Likewise, I'd expect the other option of, 'The new tab page' shouldn't be an issue either, but as I never have that set, I can't comment about it.

One other consideration is Internet Explorer's pop-up blocker setting.  (I'm assuming you're not using your own different pop-up blocker).

You can check the pop-up blocker settings by clicking Tools and Internet Options again, and opening the Privacy tab.  In there, three quarters of the way down, you'll see a button to change the pop-up settings.  If it's set to High, block all of them, try it set to Medium, block most automatic pop-ups.

Hope that helps.

ken sheridan say: I'd recommend doing it relationally by creating an auxiliary calendar table of all working days, i.e. one which excludes all weekend and public or concessionary holiday dates over a period of time, 10 years say, into the future.  To constrain a column in a table you can then create an enforced relationship between the table in question and the calendar table on the date column.  Additionally you can constrain a control in a form with code in its BeforeUpdate event procedure, e.g.

    Const MESSAGETEXT = "You have entered a weekend or holiday date."
    Dim strCriteria As String

    strCriteria = "calDate = #" & Format([YourDateControl],"yyyy-mm-dd") & "#"

    If Not IsNull([YourDateControl]) Then
        If IsNull(DLookup("calDate", "WorkDaysCalendar", strCriteria))
            MsgBox MESSAGETEXT, vbExclamation, vbExclamation, "Invalid operation"
            Cancel = True
        End If
    End If

If you have difficulty opening the link copy its text (NB, not the link location) and paste it into your browser's address bar.

scottgem say:

The Weekday function can be used to exclude Weekends. By setting the first day of the week parameter to vbMonday (or 2) you can set the rule to <6. But that doesn't help you with holidays. So Mimmo's answer is incomplete.

Using Ken's solution is one way. But if you don't want to create a table that has records for every working day, you can use a table with just holidays and a function like the following:

Public Function NotWorkday(dteDate As Date) As Boolean

If Weekday(dteDate,vbMonday) > 5 Then

     NotWorkday = True

     Exit Function

ElseIf Not IsNull(DLookup("[HolDate]","tblHolidays","[HolDate] = #" & dteDate &"#")) Then

    NotWorkday = True

    Exit Function

End If

NotWorkDay = False

End Function

This requires a table (named tblHolidays) with a Date.Time field named HolDate that has a record for every Holiday.
