I started a list for myself of all the various work-arounds for failed upgrades.  This may not be a complete list but I think it hits on all the big stuff.  I have personally tried all of these and found my update still failing.  When that happens, the only reliable way forward is to burn an .iso file (using UUPtoISO) and perform a clean upgrade - that has *always* worked for me.  

This list is culled from various posts both here and in other forums.  Many kudos to the folks that posted these (I didn't track) originally.

Windows 10 Upgrade Issues Work-Around and Cautions List

Checks to be made:

  • Windows .NET 3.5 should be uninstalled
    • Control Panel|Program|Programs and Features|{unselect box next to .NET Framework 3.5}
    • Click Okay, Reboot
    • Attempt install of new build (also check items below first)
  • No third-party Anti-Virus programs can be installed
  • No Rapid Mode can be set on Samsung SSD’s (either C: or other)
  • No third-party SSD/HDD imaging software.  Specific mentions include:
    • Uninstall Samsung Magician
    • Uninstall EaseUS
    • Uninstall Macrium Reflect
    • Uninstall Acronis True-Image 2016
    • Uninstall Spybot Anti-Beacon
  • AllowTelemetry parameter in the Registry must be a DWORD and must be set = 3
    • Run Regedit
    • Navigate to HKEY_LOCAL_MACHINE\Software\Policies\Microsoft\Windows\Datacollection\AllowTelemetry
      • Full=3, Enhanced=2, Basic=1, Off=0
      • Set to 3 (Full)
  • Perform upgrade when disconnected from the network – remove network cable or disable the network adapter
    • Control Panel|Network and Internet|Change Adapter Options
    • Once navigated to Change Adapter Options, select the Network Adapter being used, right-click on it and select Disable
  • Perform SFC /Scannow
  • Perform DISM steps:
    • C:\Windows\system32> DISM.exe /Online  /Cleanup-Image /CheckHealth
    • C:\Windows\system32> DISM.exe /Online  /Cleanup-Image /ScanHealth
    • C:\Windows\system32> DISM.exe /Online  /Cleanup-Image /RestoreHealth
    • If these are successful, re-run file checker (SFC /Scannow)
  • More significant DISM rebuild:
    • C:\Windows\system32> DISM.exe /Online  /CleanupImage /startcomponentcleanup /resetbase
  • Delete all files in the Update cache (C:\WINDOWS.~BT and C:\WINDOWS\SoftwareDistribution):
    • Cleanmgr /sageset  (select all items)
    • Cleanmgr /sagerun (deletes selected files)

john korchok say:

Thanks for the additional information. I'm much more familiar with VBA that I am with VB Interop, but the principles are mostly the same for both.

Normally you can't change much when the template is protected, so the second error message is not something we can do anything about.

When the template is unprotected, the command BMRange = oDoc.Bookmarks("Item1").Range gives you the range covered by the bookmark. The formfield is inside that range, but is a separate object. Something like BMRange.Formfields(1).Result = "My Text" should set the text in the form field. This is the equivalent to typing text into the formfield. Please note that a standard reprotect of the form will delete this text, as it does in Word if you enter text in a protected form, then unprotect and reprotect it.

Another possibility is BMRange.Formfields(1).TextInput.Default = "My Text", which should create the text as the default prompt text for the form field. I should note the in VBA, though this technique's structure is correct, it doesn't actually set the default, but it might work in VB. The VBA workaround is BMRange.Formfields(1).TextInput.EditType Type:=wdRegularText, Default:="My Text, Enabled:=True. That is what actually works in VBA, so you might have to try that as well. The default text is limited to 255 characters, I think, but it survives unprotecting and reprotecting the form.

suzanne s. barnhill say:
Go to Office Button | Word Options | Advanced, and scroll all the way to the bottom, to the Layout Options. Click the + to open the list.
