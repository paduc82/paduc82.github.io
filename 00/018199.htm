a. user say:

The key to your solution is the Split() function. In the example below the original data are in cells A1-3. The values from A1 will be placed in B1, C1, & D1. I don't know if that's what you need but, again, it's the Split() function that's central to the problem.

Private Sub splitEm()

    Dim anArray As Variant
    Dim i As Integer
    Dim y As Integer

    ' in this example the original data is in column A
    For i = 1 To ActiveSheet.UsedRange.Rows.Count
        anArray = Split(Cells(i, 1), " ")
        For y = 0 To UBound(anArray)
            Cells(i, y + 2) = anArray(y)
        Next
    Next

End Sub

Hope that helps.

Hi,

This can be solved quite easily with Power Query.  Here's the M code that works

let
    Source = Excel.CurrentWorkbook(){[Name="Data"]}[Content],
    #"Changed Type" = Table.TransformColumnTypes(Source,{{"Data", type text}}),
    #"Split Column by Delimiter" = Table.SplitColumn(#"Changed Type", "Data", Splitter.SplitTextByDelimiter(" ", QuoteStyle.Csv), {"Data.1", "Data.2", "Data.3"}),
    #"Changed Type1" = Table.TransformColumnTypes(#"Split Column by Delimiter",{{"Data.1", type number}, {"Data.2", type number}, {"Data.3", type number}}),
    #"Added Index" = Table.AddIndexColumn(#"Changed Type1", "Index", 1, 1),
    #"Reordered Columns" = Table.ReorderColumns(#"Added Index",{"Index", "Data.1", "Data.2", "Data.3"}),
    #"Unpivoted Other Columns" = Table.UnpivotOtherColumns(#"Reordered Columns", {"Index"}, "Attribute", "Value"),
    #"Removed Columns" = Table.RemoveColumns(#"Unpivoted Other Columns",{"Index"}),
    #"Transposed Table" = Table.Transpose(#"Removed Columns"),
    #"Promoted Headers" = Table.PromoteHeaders(#"Transposed Table", [PromoteAllScalars=true]),
    #"Changed Type2" = Table.TransformColumnTypes(#"Promoted Headers",{{"Data.1", type number}, {"Data.2", Int64.Type}, {"Data.3", type number}, {"Data.1_1", type number}, {"Data.2_2", type number}, {"Data.3_3", type number}, {"Data.1_4", type number}, {"Data.2_5", type number}, {"Data.3_6", type number}})
in
    #"Changed Type2"

When connected to IMAP, POP or an Exchange 2010 mailbox, Outlook 2016 for Mac allows you to add safe senders. However, safe senders are client-side only and do not synchronize to the server. Because Outlook 2016 for Mac does not perform any junk mail filtering, client-side safe senders no longer have any use. 

If your mail server provides server-side junk filtering, the same newsletter may also be determined to be junk on the server before it is ever downloaded to the Outlook client. In this scenario, adding the senders address to the Safe Senders list in Outlook for Mac would not help. You would instead need to add the senders address to the Safe Senders list on the server.
