Difference between incomplete and active tasks - Microsoft Community

oldhasbeen say:

Difference between incomplete and active tasks

I am struggling to understand the difference  between incomplete and active tasks in MSP - could anyone help?

Thanks!

tom_bpc say:

Old,

Each task you create in a project is "active" unless and until you "inactivate" it - regardless of its completion status or schedule dates.

Turning a task to "inactive" effectively removes it from the scheduling calculations* so that it has no effect on the scheduling of any other tasks.  By default the line for inactive tasks appears grayed-out.  The ability to inactivate and re-activate tasks was introduced in recent versions of Project (I think 2010+).  It is useful for what-if analyses, contingency planning, and project de-scoping.  It provides an alternative to outright deletion of a task, which is normally to be avoided.

* Project 2010 ignores inactive tasks altogether, so inactivating a task's sole predecessor will accelerate the task to the project's start date.  Project 2013 (and presumably 2016) changes this behavior by inserting invisible links between the predecessors and successors of inactivated tasks.  

julie sheets say:

To add a bit to Tom's great answer - users with the Professional version of Microsoft Project (2010, 2013, 2016) can set a task to inactive or "re-activate" an inactive tasks.  Users with the standard version of Project, can see inactive tasks, but they can neither set a task as inactive nor re-activate an inactive task.

Incomplete tasks are tasks where the %Complete is less than 100%.

julie fka profjulie say: When you set up ActiveSync to sync files between your computer and your windows mobile device, a folder is created on your computer for this purpose.  You can find the folder name and path by opening up ActiveSync on your computer, then select Tools > Options > highlight FILES & select the Settings button.

To fix this problem:

Create a new folder on your desktop computer (call it TEMP or something like that)
Copy everything (files & folders) from your syncing folder to this new folder.
Delete everything from the syncing folder and sync your Axim.  This will remove everything from the My Documents folder of the Axim.
Then add a folder from the TEMP folder to the syncing folder and sync your Axim.  This will sync this folder to your Axim.
Assuming that data syncs properly, add the next folder from the TEMP folder to the syncing folder and sync your Axim.
Then add the next folder from the TEMP folder to the syncing folder and sync your Axim.....
Proceed in this manner until you've moved everything from the TEMP folder to your syncing folder.
