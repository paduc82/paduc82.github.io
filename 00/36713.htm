I finally deleted those pesky "too long filenames".dat files.  Here's how I did it.

I opened an ADMIN command window.  Press [WINDOWS] [X] together.  Click

Command Prompt (Admin)

We'll use a little MSDOS magic here if you're not used to navigating around within the Command window.  Start by typing "CD c:\" then [ENTER}

Now we navigate to the final "0" directory.  There are two ways to to this.  Using the path listed above: (C:\Users\Your User Name\AppData\Local\Packages\winstore_cw5n1h2txyewy\LocalState\CacheCopy\0).  This example is for the copied/renamed folder you created to fix Windows Store.  To delete other long file names you would follow a similar process to navigate to the folder containing the target files.

You can CD (Change Directory) one step at a time like this:

For each CD add the next folder in the progression, i.e. >CD Your User Name then >CD AppData and so on until you arrive at the \0 directory.  We're using the \CacheCopy\0\ directory for this illustration.

At the C:\>"long string"\0> directory we type "dir *.dat".  We should see something like this:

Then we perform the delete.  Still at the 0\ directory type "del *.dat" (without the "").  The * serves as a "wildcard" meaning forever.

Remember you can't undo the "del" command so make sure you're in the 0\ directory

I hope this serves as a helpful guide.  Today's hard drives are so massive it's OK just to leave the copy directory alone.

shawn jay say:

First enable your Guest Account via Computer Management.  One way to access the tool in Windows 10 is to open the start menu and begin typing Computer Management. As you type windows will suggest a Best match where you can choose the Desktop app entry that is listed.

Once inside Computer Management expand the left hand side navigation to open Computer Management (Local) -> System Tools -> Local Users and Groups -> Users.  Locate the Guest account, right click, and choose the Properties option from the context menu.

From within the Properties popup, under the General Tab, clear the check mark to the left of Account is disabled.

Click Apply.

Click OK to close the popup.

Close Computer Management from the red X in the upper right hand corner of the window.

Next enable access without a password in the Network and Sharing Center.  One way to access the tool in Windows 10 is to open the start menu and begin typing Network and Sharing Center.  As you type windows will suggest a Best match where you can choose the Network and Sharing Center Control panel entry that is listed.  

Once you have the it open click the left hand side navigation entry to Change advanced sharing settings which will drill into another settings window.

On that page expand the grouping for All Networks and look for the Password protected sharing section. Choose the radial button to select Turn off password protected sharing.

Check out the SevenForums.com post for the picture of the setting I used.  Thanks to Barnz for the November 11, 2009 @ 8:39 PM comment on technize.info that pointed me to this part of the solution.

Open File Explorer (you may think of it as Windows Explorer, My Computer, or Documents; where you see your folders).

Go to the folder you want to Share and right click into the Properties setting popup.  

Click the security tab.  Click the Edit. . . button.  In the Permissions for [folder name] popup click the Add. . . button.  In the text area for Enter the object names to select type Guest and click the okay button (note you can also use Everyone but the newer and I believe preferred Windows 10 approach is to use Guest).  Click OK to close the Permission for [folder name] popup.

Click the Sharing tab.  Click the Share. . . button to share the folder (or if you prefer use Advanced Sharing. . . button). When setting permissions type Guest to add that account and select to give it the appropriate read/write (or other) permissions. Click OK to close the sharing window(s).  Click OK to close the [folder name] Properties popup.

Your folder should now be ready for anonymous access (although Windows 10 will apply the security and permissions for the Guest Account which you have enabled without a password and are accessing as).

Note that in this example the PC hosting the Share was not a member of the domain from which the majority of clients accessing it were.
