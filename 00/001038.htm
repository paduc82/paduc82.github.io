To find Office "bitness", go to File menu > Account command > About app-name button. At the top of the about dialog it will display the Office Application name, Version number and "bitness" (32-bit or 64-bit).

So, if they find they have 64-bit Office check how much RAM they have installed:

  • Open the Windows Task Manager, <CTL><SHF><ESC> shortcut.
  • Click on the "Performance" tab
  • On the left is a column of specifications, starting with CPU then Memory
  • Under the Memory title it will show RAM used / total, ie 5.9 / 7.9GB (75%).

If they have a low end computer with only 4GB of RAM, that could be a problem.  There is another setting "virtual RAM" that can help trick computer into thinking it has more RAM than is installed, but I don't know how that would work with a large file.

rohn007 say:

You edited a file on a SD card.

That can be an issue. The problem you are having is usually associated with USB drives, but probably (I'm not positive, can someone else confirm?) applies to SD's to.

USB use a specific type of "flash" memory that has a limited number "write" cycles.  To extend the life of the flash drive WINDOWS (not the specific programs like Excel and Word) first saves the file to a "cache" file on the local HD. Then it writes whole "blocks" to the flash drive.  If you do not "flush" the cache by properly shutting down the USB before removing the USB, the file is damaged.  What actually happens is that part of your file is NOT written to the USB and your file is corrupted. 

Specifically for Word, the file structure is setup in a way that the useless formatting **** is written first and the actual body text, the IMPORTANT stuff to you, is last. This means that the important stuff is what gets lost!

swischner say:

You are expecting a bit too much from the web clipper :)

To replicate a web page on a OneNote page OneNote would have to completely recreate (render) it. That means OneNote would need to have a fully working web browser included (and always be online to show external content). So what you basically expect is that the OneNote editor all of a sudden turns into a full featured "Edge", "Firefox" or "Chrome". It doesn't.

The web clipper offers two workarounds for this problem:

1. If you clip a page using "Full Page" or "Area" a screenshot of that page is created and sent to OneNote. This is what your picture is showing. A screenshot, like any bitmap graphic or digital photo, cannot contain functioning links. Those are just blue on white pixels. The upside of this method is that you have a fully accurate representation(!) of the original website. The downside, as stated, is that its just a graphics file. It is somewhat like opening a page in your web browser and taking a photo from your monitor with your camera which you insert onto a OneNote page. The main difference is that with "Full page" selected the picture is taken from the complete web page including the portions currently scrolled out of view.

2. If you use the "Article" option when clipping a page, the web clipper (in fact a mechanism on Microsofts servers) attempts to recognize the relevant portions of the page (like headlines, text blocks, pictures...) and build you a very much simplified version of the page excluding advertisements, navigation elements and so on. The text portions of this recreation may also contain links which then remain fully working (although always pointing to the original web location, so clicking on such a link will open the target in your browser, not in OneNote, as .... I already said it ... OneNote does not include a web browser). 

It's all the same with the Evernote web clipper, just that it's recognition of relevant web page content is sometimes a bit more accurate.
