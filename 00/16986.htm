zigzag3143 (ms -mvp) say:
If you’d like to manage your Outlook account using an email app, you’ll need to set up your POP3 or SMTP server and port settings in your email app. You’ll need the following info:

    POP3 server: pop3.live.com (port 995)
    SMTP server: smtp.live.com (port 25)


Choose the Secure Sockets Layer (SSL) option for both POP3 and SMTP connections.
Choose the SMTP authentication for SMTP connections.
Choose the outgoing server requires authentication option. (In most email apps, this option isn't checked by default.)

    User name: Enter your full email address.
    Password: Enter your Microsoft account password.
    If you don’t want email messages to be permanently deleted from your inbox after you download them to your mail program, select the Leave a copy of messages on the server option.

Prevent losing email in Outlook.com when you've set up POP3

Outlook.com has a feature that prevents losing email messages if you accidentally set up POP3 to delete them after reading them on the server. Here’s how it works:

    The first time that Outlook.com receives a POP3 instruction to delete email messages after reading from an email app, Outlook.com won't delete the messages. Instead, Outlook.com moves the email messages that were successfully downloaded by the email app into a POP folder and out of the inbox. The POP folder is available in the Folders list.

    Outlook.com keeps the email messages in the POP folder until you sign in to your Outlook.com account and verify whether Outlook.com should respect or ignore the POP3 delete command.

To verify your POP3 settings:

    In the upper-right of the Outlook.com window, click Options, and then click More options.
    Under Managing Your Account, click Pop and deleting downloaded messages, and then select one of the following:
   
Don’t let another program delete messages from Outlook.com. (If your other program is set to delete messages from the server, we’ll simply move them to a special POP folder. They won’t be deleted.)

Do what my other program says—if it says to delete messages, then delete them.

    Click Save.

MS Access Internet bandwidth requirements other than screen updates - Microsoft Community

prnmatt say:

MS Access Internet bandwidth requirements other than screen updates

If the Access cloud backend contains both the database and the client application, are their any additional bandwidth connection requirement other than screen updates?
albert d. kallal say:

In the setup you mention, the bandwidth requirements are the same for the DATA part.

 

The fact that you run MS Access from a USB jump drive, a folder on your computer, or a shared folder on some server STILL MEANS IN ALL cases that the software is running on your desktop.

 

In fact your suggestion means that you would require MORE bandwidth, since you are suggesting that the software you write with Access is to be download BEFORE you do any useful work.

So, when you run word, you have the word application installed on YOUR computer.

 

And if you build a desktop application with Access, then you require that application to be installed on YOUR computer. If you don't, then you be using lots of bandwidth to FIRST download a copy of the application to your local computer BEFORE it can run.

 

In fact this is why in any multi-user environment it is STRONGLY suggested that you split your database and place a copy of the application on EACH work station. This concept is explained here:

http://www.kallal.ca/Articles/split/index.htm

 

So placing the application part (front end) on a shared folder on a server or on a local folder makes ZERO difference in the bandwidth required to pull data from the tables (the data you work with MUST travel down the network pipe to your local computer). However if you don't place a copy of the application on your computer, then BOTH data + application will have to make the trip down to your workstation.

 

How desktop software works does NOT change because you place a document on a folder on a server. To use that document (or Access application), then that document (or application) has to TRAVEL down to your desktop computer.

 

And just keep in mind that a internet connection is slow – and worse there are issues and problems if you run a split database over the internet.

You can read my article on using Access over the internet and some solutions to this bandwidth issue here:

http://www.kallal.ca//Wan/Wans.html

 

Now it is possible by "cloud" you mean not just placing the files in the cloud, but you are referring to an Access web application that published to the cloud. In this case, you would be 100% correct that only screen updates are send down the network connection to the web browser. However this setup means that you SPECIFIC built a web application with Access – existing VBA forms etc. cannot be used with such a setup.
