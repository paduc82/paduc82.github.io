gabriel bratton say:

Henry,

Double check you have all the updates.  Right click the Start button and click Settings.  Search for Updates and click Check for updates.  In case the updates have installed go to the Control Panel and search for Updates.  Click View Installed Updates and look for updates installed ~6/28.  If you have the updates reboot the machine to make sure they had a chance to take affect.  The updates needed are listed in Issue#5 in this topic, https://support.office.com/en-us/article/Outlook-known-issues-in-the-June-2017-security-updates-3f6dbffd-8505-492d-b19f-b3b89369ed9b?ui=en-US&rs=en-US&ad=US. 

Look for these updates:

Windows 7  KB 4022168
Windows 8. KB 4022720
Windows 10 Version 1703 KB 4022716
Windows 10 Version 1607 KB 4022723
Windows 10 Version 1511 KB 4032693
Windows 10 (initial version released July 2015) KB 4032695

After the updates some other things to check:

1.  In Outlook click into the search box, on the ribbon click Search Tools, Indexing Status.  If there are items still needed to index it might take some time for it to catch up.  It is a good sign if indexing is working.

2.  In Outlook click into the search box, on the ribbon click Search Tools, click Locations to Search.  Make sure there is a checkbox next to all of the items that you want to be able to search.

3.  If indexing is completed from Step 1 and you are still not seeing results or you see the message "Something went wrong and your search couldn't be completed" check to make sure Outlook is selected for Indexing.  

- In Outlook click into the search box, on the ribbon click Search Tools, Search Options, click Indexing Options, click Modify.

- If Outlook is unchecked click the checkbox next to it and click OK.

4.  If indexing is completed from Step 1 and you are still not seeing results rebuild the Index.  In Outlook click into the search box, click Search Tools, click Search Options, click Indexing Options, click Advanced, click Rebuild.

5.  There was a prior issue back in March that caused search results not to be returned.  If you originally configured Outlook or created a new Outlook Profile in late February or March this could have affected you.  The best solution to this issue is to create a new Outlook Profile, https://support.office.com/en-us/article/Outlook-search-stops-working-after-updating-to-build-7870-2020-and-higher-0bcf0b06-320d-40c3-8cf0-7e3df90793ed?ui=en-US&rs=en-US&ad=US.  

Note:  you can keep the old profile and refer back to it if needed.  If this is the affecting issue it will not likely get resolved without a new profile.  When you follow the steps to create a new profile you can click the check box "Prompt for a Profile to be used".  When you start Outlook it will prompt you to select which profile to open.

pd_214 say:

MS Access error - Too few parameters. Expected 1

I am getting the following error when I try to run the DB.Execute command:

"Too few parameters.  Expected 1"

Below is the simple code/query that I'm trying to run with the DB.Execute command:

INSERT INTO tblCurrentCourses ( schoolno, license )
SELECT schllicense.schoolno, schllicense.license
FROM schllicense
WHERE (((schllicense.schoolno)=[Forms]![SchoolForm]![schoolno]) And ((schllicense.license)="DA" Or (schllicense.license)="DS"))
ORDER BY schllicense.schoolno, schllicense.license;

Any recommendations would greatly be appreciated!

john w. vinson say:

Is the form SchoolForm open when you're running the query? Is there data in the Schoolno textbox?

And why are you (apparently) storing these two fields redundantly in a second table? They're already there in [schllicense], what benefit do you get from storing (just?) those two fields in tblCurrentCourses?

karl donaubauer say:

Hi,

DAO.Execute doesn't know about Access forms. You can e.g. use Eval() for your forms reference in the query to solve this:

Eval("Forms!SchoolForm!schoolno")
