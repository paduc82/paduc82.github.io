edgar moreira say:

Hi

I have solved my problem. It was disabled on services and it shows as optimise dives on services. I did not know, just after a research.

My disk is not ssd but just a quick question: why we should not optimise ssd?

rohn007 say:
SSD use a form of electronic / silicon memory that has a limited number of Write cycles. And it also uses a totally different method of accessing files, electronic (fast) based rather than moving a physical head on a HD (very slow). So, doing a file re-organize simply "uses up" writes on the memory without significantly speeding up access. Using up write cycles was also MUCH more important just a few year ago when SSDs were much smaller and used older technolgy that had fewer write cycles than current versions.

HD's use "magnetic" storage. There is effectively no limit on read/write cycles. As well the files are stored as a long string of 1's and 0's in a spiral like on a phonograph record or concentric "tree rings". Back from the origins of Windows/DOD, HD space was limited and drives were much smaller so they came up with a design that splits the drive into smaller "blocks". Windows is designed to use a "first come first serve" write process. So as it finds an empty block, it writes part of the file there. Then it moves on to the next free block. At first, when the drive is "optimized", all/most of the files are written on continuous strings of blocks. That makes file retrieval fast. As you delete and write files of different sizes, the strings of free blocks gets shorter. The result is files get written in "fragments" that are scattered around on the drive. As free space and files get fragmented, the read/write head has to constantly move back and forth to pick up the pieces. Moving the head is very s

Depending on the tool there are various optimization techniques, but fundamentally they all attempt write files in more or less continuous strings to minimize the movement of the drive head.

One thing most people don't realize, is that even after doing a clean install of Windows your drive WILL be fragmented! The install process creates and deletes many temporary files, that cause fragmentation. The only way to get a defragmented installation of Windows is if you use an image copy that was made from a defragmented disk drive or you run a defrag cycle immediately after finishing the install
