max nierbauer say:

Hi,

you can "uninstall" mail/calendar with this Powershell command:

Get-appxpackage | where {$_.name -like "*windowscommunicationsapps*"} | remove-appxpackage -confirm

-confirm will let you confirm that you really want to do that.

"uninstall" in brackets, because it's a bit more complicated than that. Technically you uninstall the user related part/data of the application. The provisioned or 'all user part' of the application is still on the system - but for the signed in user the app is 'gone'.

After that you can go to the store an re-install the app. (technically you re-acquire the license, and the provisioned app becomes visible again. Plus, once configured, your mail and appointments will be resynced).

Hope that helps.

stercorum say:

Outlook 2016 on Windows 10 displays 0x800cce05 when attempting to add IMAP/POP3 account. This can be caused by a number of things, even on a new Windows 10 PC with Office 365.

To resolve right click the Start button, select Command Prompt (Admin) and type:

sfc/scannow

The line reads: C:\Windows\system32>sfc /scannow

Press Enter. A message then reads Beginning system scan.  This process will take some time depending on how much data has to be scanned and how big the hard drive is. The message reads: Beginning verification phase of system scan.

When complete, the next message reads: Verification 100% complete.

Windows Resource Protection found corrupt files and successfully repaired them. Details are included in the CBS.Log windir\Logs\CBS\CBS.log. For example C:\Windows\Logs\CBS\CBS.log. Note that logging is currently not supported in offline servicing scenarios.

If you check this file you can view the log activity. You should now be able to create a new email profile for outlook with the settings you would normally use.
