greg carmack - windows mvp 2010-2020 say:
Hi Suzanne. I'm Greg, an installation specialist and 9 year Windows MVP, here to help you.

Ask the technician exactly what he/she did. If they did a Version update the files may be in c:/windows.old. They may also be your old User profile at C:\Users.

OneDrive introduced some features that remove files from the hard drive and store them only in the cloud if Files On Demand is chosen in OneDrive Settings: https://blogs.office.com/en-us/2017/05/11/intro...

Also if the new Folder Protection is chosen in OneDrive Settings > AutoSave Tab, then Documents, Desktop and Pictures folders will be moved into the OneDrive folder. See Folder Protection details here: https://www.tenforums.com/tutorials/116029-turn...

Strangely recent Updates have set these on by default and even after I warned about this for months, apparently caused file losses that caused them to withdraw the October Version 1809 Upgrade. So I would check closely those OneDrive Settings, your OneDrive folder, the PC Recycle Bin, and also check the Onedrive Online Recycle Bin.

Some Updates can also cause Windows 10 to start into a temporary profile which should be obvious because you don't recognize anything, in which case files should still be in your old Profile at C:\Users.

Finally sometimes if an Update fails it will archive files in C:\windows.old.

So look in all of those places, starting with OneDrive "View Online" the online files and online Recycle Bin.

I hope this helps. Feel free to ask back any questions and let us know how it goes. I will keep working with you until it's resolved.

a. user say:

Hi,

 

Thank you for the update.

 

First of all we have to check where the bootmgr file has been resided. Open Disk Management and find the partition that is marked "active". That is the partition that contains the bootmgr.

The active partition is usually from the OS that was on the system first - e.g. if you installed Windows 7 on an XP system, the XP system would be the active partition.

 

The reason you can't delete the Windows XP partition is because Windows 7 is currently booting from it. Since Windows XP was installed first, Windows 7 installed its boot files onto the Windows XP partition (that's why it's labeled "System"). The Windows XP partition is also the Active (booting) partition. According to your post, the Windows 7 partition is Logical. It would have to be converted to a Primary partition to be bootable. If you delete or remove the Windows XP partition, you'll have to convert the Windows 7 partition into a Primary partition and run a Windows 7 repair (at the very least) to fix the Windows 7 partition so it will boot. You'd need a Standard Microsoft Windows 7 DVD for this.
