thedbguy say:
Hi. For this requirement, you can't use an Autonumber field because it will never assign the same number on the following day. Instead, you will have to generate the sequential number yourself. There are several demos available showing how to do this. The main idea is to use a numeric field which can be queried to check for the maximum value and then increment it. However, there are complications when using the technique on a multi-user database. You should keep the issue date and sequential numbers on separate fields. It's easy enough to display them together when you need to. Just my 2 cents...

ken sheridan say: Do not store the values in a single column; it is a trivial task to concatenate formatted values from two columns.  As the primary key of the table use two columns as the composite key e.g. DateCreated of date/time data type and SequenceNumber of integer number data type.  To return the structured key value you can concatenate these in a computed column in a query or a computed column in a form or report with:

    Format(DateCreated,"yyyymmdd") & "-" & Format(SequenceNumber,"00")

You'll find an example of how to compute a sequential number per group (the DateCreated value in your case) in CustomNumber.zip in my public databases folder at:

https://onedrive.live.com/?cid=44CC60D7FEA42912&id=44CC60D7FEA42912!169

Note that if you are using an earlier version of Access you might find that the colour of some form objects such as buttons shows incorrectly and you will need to amend the form design accordingly.  

If you have difficulty opening the link copy its text (NB, not the link location) and paste it into your browser's address bar.

In this little demo rows are sequentially numbered per gender.  In my demo this is done by calling the GetNameID function, which returns the next number, in the AfterUpdate event of the cboGender control.  In your case you could compute both the DateCreated value and the SequenceNumber value in a single step as soon as a user begins to enter data into a form bound to the table in question.  This would be done by code in the BeforeInsert event of the form, in which you would have hidden DateCreated and SequenceNumber controls.  Once you've created a function equivalent to my GetNameID function, e.g. GetSequenceNumber, the code would be:

Me.DateCreated  = VBA.Date
Me.SequenceNumber = GetSequenceNumber(Me.DateCreated)

In the same form a visible unbound control to show the  structured key value would have as its ControlSource property the following expression:

= Format([DateCreated],"yyyymmdd") & "-" & Format([SequenceNumber],"00")

The other key thing about my demo is that it caters for conflicts in a multi-user environment where two or more users might be inserting a new record simultaneously.  By detecting the key violation error the function is called again to adjust the sequence number upwards to reflect the fact that number originally returned by the function has since been used by another user who saved their record first.

andreas killer say:
You can use SendKeys to write the keys into the other application, for example:

Open Notepad
Make a new Excel file
Create the layout as in the screenshot below
Right-click on the sheet tab
Choose "View Code"
Within the menu click Insert \ Module
Paste this code 

Sub Test()
  Dim R As Range
  AppActivate "notepad"
  DoEvents
  For Each R In Range("C1", Range("C" & Rows.Count).End(xlUp))
    SendKeys R.Value
    SendKeys "{ENTER}"
  Next
End Sub


Close the VBA editor
Press Alt-F8
Choose the macro
Click Run

You should see the same as in the screenshot below. If you get a RTE 5 on AppActivate means that a window with this title can not be found, so that is no issue of the code.
