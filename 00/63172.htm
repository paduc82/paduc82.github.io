¡firedog say:

It would have been easier if you'd simply posted the bits I asked for. However, you can see from this that the sending format is set to RTF for the messages you receive. This is clear from the Content-Type headers for the attachments:
Content-Type: application/ms-tnef; name="winmail.dat"


application/ms-tnef is a MIME type that no mail client other than Outlook (or Exchange Client) understands. Some mail services, like outlook.com and Gmail, can decode these attachments, which is why you can access them in webmail and even forward them. You will note that the forwarded attachment has a generic MIME type:
Content-Type: application/octet-stream

The application involved will become apparent to Windows Live Mail from the first few bytes of the data stream.

You should check the sending formats specified for the Outlook contacts involved (in your quoted headers disguised as email1, Me Googlemail and Me home). This may override any sending format specified for all messages or for this particular message.

In any event, the 'fault' lies fairly and squarely with the sender. RTF should not be used outside the corporate environment, or at any rate when communicating with anyone you don't know for certain to be using Outlook as his mail client.
