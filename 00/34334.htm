Hi Regina,

SFC which is an acronym for System File Checker checks for corruption in Windows protected system files. If it finds any, it attempts to replace them using the dllcache repository which is a hidden folder on your computer.

If the files cannot be replaced from there, then you need to run the other command which is this one if you recall:

DISM.exe/Online /Cleanup-image /Restorehealth

DISM as opposed to SFC downloads original files from Microsoft's servers to replace those which SFC couldn't repair. The only files being replaced are those which are on the machine already, but which have become corrupted for whatever reason.

Here's a screenshot of DISM running on my own machine recently to repair corrupted files. As you can see, it was successful in my particular case.

It's wise though to rerun sfc /scannow to verify that everything is running properly again. You can see the results of that in the screenshot as well.
