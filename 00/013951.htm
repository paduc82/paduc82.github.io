stefan blom say:

The updating of styles from the template will occur each time when you (re-)open a document for which "Automatically update document styles" is selected, so Word is behaving as designed.

In a previous post, I wrote that you can use the Modify Style dialog box to turn off numbering for a specific paragraph style. To remove a multilevel list completely, you will of course have to turn off numbering for each paragraph style included with that specific numbered list.

Any modification made to styles will be lost the next time you re-open a Word document if you leave the "Automatically update document styles" checked, so you will have to deselect it.

bill manville say:

If you run Repair on Office 2010 ( Control Panel > Programs and Features > Microsoft Office 2010 > Change > Repair ), I think it will change the file associations to 2010.

However when you next run 2013 it will try to take the file associations back.

My way of dealing with this is to set registry keys which will prevent 2013 from re-registering itself:

This is my Set2010.reg file:

Windows Registry Editor Version 5.00

HKEY_CURRENT_USER\Software\Microsoft\Office\14.0\Word\Options]
"NoRereg"=dword:00000000
[HKEY_CURRENT_USER\Software\Microsoft\Office\15.0\Word\Options]
"NoRereg"=dword:00000001

HKEY_CURRENT_USER\Software\Microsoft\Office\14.0\Excel\Options]
"NoRereg"=dword:00000000
[HKEY_CURRENT_USER\Software\Microsoft\Office\15.0\Excel\Options]
"NoRereg"=dword:00000001

HKEY_CURRENT_USER\Software\Microsoft\Office\14.0\Powerpoint\Options]
"NoRereg"=dword:00000000
[HKEY_CURRENT_USER\Software\Microsoft\Office\15.0\Powerpoint\Options]
"NoRereg"=dword:00000001

I run that and then run each of the 2010 applications by batch file:

regedit /s \clients\Set2010.reg
"\program files\microsoft office\office14\winword.exe"
"\program files\microsoft office\office14\excel.exe"
"\program files\microsoft office\office14\powerpnt.exe"

The file associations will then remain with 2010

gord dibben say:

I think with that many cells to visit you will need VBA.

Here is event code you can use.

Copy/paste to the worksheet module, modify the cell sequence(random or not) to your liking.

Type your data in first cell, hit enter or tab key and jump to next in sequence.

Private Sub Worksheet_Change(ByVal Target As Range)
'Anne Troy's taborder event code large array
    Dim aTabOrd As Variant
    Dim i As Long

     'Set the tab order of input cells
     'must be relative reference as shown

    aTabOrd = Array("B2", "G2", "F2", "H2", "J2", "B4", "G4", "F4", "H4", "J4", "B6", "G6", "F6", "H6", _
        "J6", "B8", "G8", "F8", "H8", "J8", "B10", "G10", "F10", "H10", "J10", "B22", "G22", _
        "F22", "H22", "J22", "B14", "G14", "F14", "H14", "J14", "B16", "G16", "F16", "H16", _
        "J16", "B18", "G18", "F18", "H18", "J18", "L3")

     'Loop through the array of cell address
    For i = LBound(aTabOrd) To UBound(aTabOrd)
         'If the cell that's changed is in the array
        If aTabOrd(i) = Target.Address(0, 0) Then
             'If the cell that's changed is the last in the array
            If i = UBound(aTabOrd) Then
                 'Select first cell in the array
                Me.Range(aTabOrd(LBound(aTabOrd))).Select
            Else
                 'Select next cell in the array
                Me.Range(aTabOrd(i + 1)).Select
            End If
        End If
    Next i
End Sub
