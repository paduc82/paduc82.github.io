    ' 1. intStartDay - weekday on which week starts, 1-7 (Sun - Sat)
    ' 2. vardate - optional date value for which week starting
    '   date to be returned.  Defaults to current date
   
    If IsMissing(varDate) Then varDate = VBA.Date
   
    If Not IsNull(varDate) Then
        WeekStart = varDate - Weekday(varDate, intStartDay) + 1
    End If
   
End Function

So in a report's underlying query you can return the Friday week starting date for each row like so:

SELECT *,
WeekStart(6,TransactionDate) As WeekStarts
FROM Transactions;

If you want to group rows by each 28 days you can simply round towards zero to an interval of 28 using the following function:

Public Function RoundToZero(dblVal As Double, dblTo As Double) As Double

    Dim intUpDown As Integer
    Dim lngTestValue As Long
    Dim dblTestValue As Double
    Dim dblDenominator As Double
  
    If dblVal < 0 Then
        intUpDown = -1
    Else
        intUpDown = 1
    End If
  
    dblDenominator = intUpDown * dblTo
    dblTestValue = dblVal / dblDenominator
    lngTestValue = Int(dblTestValue)
    RoundToZero = intUpDown * lngTestValue * dblTo

End Function

Because Access uses 30 December 1899 00:00:00, which happens to be a Saturday, as the origin for its date/time data type implementation, you need to offset the date by one for a Friday week-start, so to give the start date for each 28 day range you'd call it like so:

SELECT *,
WeekStart(6,TransactionDate) As WeekStarts,
Format(RoundToZero(TransactionDate+1,28)-1,"yyyy-mm-dd") AS RangeStarts
FROM Transactions;

This might not tally with the origin of your particular 28 day periods, however, so you might have to offset the range starting date further, e.g. to offset by 3 weeks:

SELECT *,
WeekStart(6,TransactionDate) As WeekStarts,
Format(RoundToZero(TransactionDate+22,28)-22,"yyyy-mm-dd") AS RangeStarts
FROM Transactions;

I've used the ISO standard of YYYY-MM-DD for the range starting date format here, but you can of course use any date format you wish.

macropod say:

The Paste Special>Paste Link method has a number of paste formats.

The RTF and HTML formats both insert the content as a formatted table, with a paragraph-break before. Since tables must be followed by an ordinary paragraph, any paragraph break at the insertion point is preserved.

The Picture and Workbook paste formats do the best job of preserving the source formatting and both can be inserted in-line with whatever content is already present in the document.

If you're pasting a single cell, the RTF or HTML paste formats should work just fine. For multiple cells pasted as a group, though, if the aim is to preserve the format and not get the unwanted paragraph break(s), the best option is to use the Picture or Workbook paste format.
