In particular, take a look at the idea of hidden text in the style being used as a null indicator.
{ IF { StyleRef "Heading 2" } <> "" { StyleRef "Heading 2" } { StyleRef "Heading 1" } }

Start your document with a paragraph in Heading 2 style on the first page. Perhaps "Dummy Heading 2." Use font formatting to mark that as hidden text. Follow each Heading 2 with a blank paragraph in Heading 2 style. That paragraph should also be hidden text.

The StyleRef field will give a blank return for the hidden text paragraphs. This means that the field shown above will show Heading 2 if it is on the current page, otherwise it will show Heading 1.

You could save a paragraph in Heading 2 style fermatted as hidden (not through the style) as AutoText to make insertion easy.

The hidden paragraphs will not show up in a Table of Contents or on the Navigation Pane. Again, you want one on the first page and one after each actual Heading 2.

In addition to the characters that include diacritical marks, as described in the support article, there's a range of Unicode characters named Combining Diacritical Marks (Unicode hex values 0300 through 036F).
