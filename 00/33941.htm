celidor say:

Most likely just a driver issue. You need display drivers so that Windows knows what resolution is supported. I'm using Windows 10 and running an even higher resolution than that.

Update drivers! And restart the computer.

Note: Sometimes automatic graphics driver updates don't always get the most up to date drivers but claim they are up to date. Known to happen. So it is best, if running into issues, to manually download and install the newest driver from the manufacturers website by finding only the driver specifically for your graphics/display card, not using some "easy tool".

Side Note: The stretching is caused by using a resolution that is not the same aspect ratio as your display monitor/TV. 1920 x 1080 is a 16:9 aspect ratio. So any resolution that you set that is less than 1920 x 1080, with the same 16:9 aspect ratio, will automatically be scaled up to fill the screen without causing any "stretching". But when you use a resolution that is a different aspect ratio than your display than it can't fill the whole screen without looking stretched out.

slance310 say:

Hi,

I doubt it is a CPU Pin problem...

Is the BIOS set to run as UEFI Bios or Legacy ?

Legacy and UEFI use different type Initialization for the boot HDD\SSD...

Legacy must be MBR, and UEFI must be GPT... A lot of times the Bios will determine whether to run in Legacy or UEFI mode depending upon the setup of the boot HDD\SSD...

Two things to check...

1)  Win 8, 8.1 and 10 have a new feature called Fast Startup. It can sometimes cause problems like you mention as well as others.

The easiest way to tell if this is the problem is to turn it off and see what happens.

This feature only works when you do a SHUTDOWN and then Boot.

It doesn't effect a RESTART.

To turn it off or on...

Go to Control panel...Power Options, and select Choose What the Power Buttons Do on the left.

Then select Change Settings That Are Currently Unavailable near the top center of screen...

Lower down on the window, uncheck Fast Startup.

This may or may not help your problem, but is easy to check.

If this resolves the problem, then you can investigate what isn't loading properly.

2)  In the BIOS Boot Tab... With Legacy Bios, make sure the HDD is set as the primary HDD\SSD if there is more than one...

With UEFI Bios, make sure the selection for Primary boot device is "Windows Boot Manager" 

You can set the machine to use a DVD or Flash drive as it's first boot device as well, which will be ignored if there is no media...
