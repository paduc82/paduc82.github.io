petertaylorxc say:

Pasting text into an e-mail using Windows 10 Mail app

A lot of people seem to be struggling with this and I believe it is a bug in the Mail app itself. I am an experienced Windows programmer so I quickly found a solution that works for me. From this I deduce that the bug is that Windows Mail is only looking for Rich Text format on the clipboard. This means that if you copy plain text to the clipboard, such as from NotePad, it will not paste. The workaround is to paste the plain text into a blank Word document and then copy it again from there. Word will put the text onto the clipboard in Rich Text format and then Mail will be able to find the format that it wants.

    If you're still following this thread, you may not yet have discovered that the Create rule from message function is still there, but well hidden, so you have to use a couple more keystrokes to get there:

    1. When reading the message in its folder (i.e. not in Quick views), highlight and copy the message subject.
    2. Press Ctrl-Shift-F to open the Find Message dialogue.
    3. Paste into the Subject box, deselect Include subfolders if it's selected and click Find Now.
    4. Select the message in the results, then press Alt A E, (i.e. Alt, then A, then E).

    Notes:

    • As always, rules only work with POP3 accounts.
    • You can also copy the 'from' address and use that in the Find dialogue, but you must be careful to delete the semi-colon at the end of the pasted address before clicking Find Now.

    This dialogue is a leftover from earlier versions that got forgotten when the developers were busy removing functions before releasing Windows Live Mail 2011. It also has Recombine attachments, which I've not tested. The dialogue doesn't conform to the same rules as most other windows in the application - for example, Alt-A doesn't reveal the Actions menu as this kind of keystroke does in other windows.
