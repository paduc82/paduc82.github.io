How many records will an Access 2000 database hold? - Microsoft Community

zipone say:

How many records will an Access 2000 database hold?

Hi,

I have several Access 2000 databases that I sell on ebay.  I have been asked various times "How many records will the program hold?"  I don't know.  Can someone tell me what determines the number of records that can be entered in a database?  Is it depended on the computer's memory size, the disk space, or what?

Thanks,

Phil

hansv mvp say:

There is no specific limit on the number of records in an Access database.

The maximum file size of an Access database file (.mdb) is 2 GB, and the maximum size for any individual table within a database is 1 GB.

The maximum size of a single record within a table is 2 KB (excluding memo fields and OLE object fields). So if all records are maxed out, a table could hold 1 GB/2 KB ~ 524288 records. But if the records are smaller, the number of records can be (much) larger.

Remark: when you approach the limits, the database becomes unwieldy, since Access needs some free space within the database as temporary storage for data manipulation. It's best to compact the database from time to time and to stay well within the limits.

    rena yu msft say:

    Hi Keith,

     

    Here are the detailed steps:

    1. Go to your “existing calendar”, for example, named “Calendar1” > Calendar tab > List Settings > Views > Create view > Calendar View > Give View Name for example “Mark” (means this view only shows event that title including “Mark”) > Filter > Show items only when the following is true > Show the items when column “Title” contains “Mark”> OK
    2. Use the same way above to create the view “Robert” to only show events that title includes “Robert”.
    3. Create a new SharePoint Online Calendar app. For example, you can name it “Colored Event”.
    4. Go to the “Colored Event” > Calendar tab > Calendars Overlay > New Calendar > Give Calendar Name “Mark” > Give Color “Dark green calendar > choose the “Mark” view> OK
    5. Use the same way above to create new Calendar Overlay “Robert”and give the color in red.
    6. Now you should see the result below.

    kergga say:

    One of the more common reasons to see these weird printing artifacts is the printer driver. What type of printer are you using, and what printer driver? If you connect to a different model printer and print with the most up-to-date drivers available for that other printer, what happens? You can also use virtual print drivers to test, for example try printing to a PDF file from a 3rd party PDF creation program like Adobe and look at the resulting PDF (you might also try the built-in PDF export function, but I wouldn't rely on that as an end-all, beat-all analysis).

    It was a different version of Excel, but we had a similar problem and our workaround was to print to a PDF, then print the PDF to our normal printer and got the expected hardcopy. Not ideal, but our expensive, large corporate MDF didn't have any drivers that actually worked with the nuanced formatting in our documents. Maybe if you find better results with a different printer you can just go that way instead.
