The issue seems to be that some rogue application has updated the Registry setting in :

HKEY_CURRENT_USER\Software\Microsoft\Windows\CurrentVersion\Explorer\User Shell Folders\

The key name is AppData, and the rogue application had changed the value simply to %APPDATA%. Given the setting itself is meant to describe where %APPDATA% actually goes, setting the value of this to %APPDATA% is obviously counter productive ;-)

lemp say:

A computer that is running Windows XP may not be compatible with Windows 8.1.  Before you do anything, you need to confirm BOTH

that your computer meets the minimum hardware requirements for Windows 8.1 --

  • processor: 1 gigahertz (GHz) or faster
  • RAM: 1 gigabyte (GB) (32-bit) or 2 GB (64-bit)
  • Hard disk space: 16 GB (32-bit) or 20 GB (64-bit)
  • Graphics card: Microsoft DirectX 9 graphics device with WDDM driver
  • For 64-bit installations, your CPU must also support CompareExchange16b, PrefetchW and LAHF/SAHF

AND

that there are Windows 8/8.1 drivers available for your hardware components.

Although "upgrade" and "clean install" may or may not be "technical terms," what most people mean by "upgrade" in the context of moving from one version of Windows to another is the ability to install the newer version while retaining your personal data and installed programs. 

In contrast, a "clean install" is the process of installing the new operating system only.  After this installation is complete, personal data files may be copied to the new system from a backup and application programs installed from their respective installation media (e.g., optical disks or downloaded installation files).  Before installing such programs, you must determine if they are compatible with the new version of Windows or if you will need to purchase newer versions.

Using normal methods, there is no "upgrade" from Windows XP to Windows 8 in the sense defined above.  That is, to get from XP to 8, you normally would backup your data, collect the installation media and any required license keys for your applications, install Windows 8, restore your data from the backup, and finally install your application programs.

Laplink offers a commercial program called PC Mover Professional that purports to have the ability to move application programs to be moved to a new computer in addition to data files.  Laplink asserts that you can use their product to do an "in-place upgrade" from Windows XP to Windows 7/8/10.

I have no first-hand knowledge of how PC Mover Professional actually works or how reliable it is.  Personally, I wouldn't use it, although Laplink has been around for a long time and is generally a very reputable company.

http://www.laplink.com/index.php/individuals/pcmover-for-windows/pcmover-professional

http://download.laplink.com/documentation/pdf/pcmover/pcmua/PCmover_Win8UA_UG_ENG.pdf

Finally, if you are really committed to keeping your old hardware, I suggest that you skip Windows 8 and "upgrade" either to Windows 7 or Windows 10, assuming, of course, that you meet the two tests at the beginning of this Reply.

charlie klausen say:
Ok. I’ve been looking for registry fixes to enable LAN settings.

Click start, and type “Regedit” - and hit enter.

Navigate to HKEY_CURRENT_USER\Software\Policies\Microsoft\Windows\Network Connections and locate NC_LanProperties registry key.

You will want to change the value of NC_LanProperties to 0 if you want to disable the policy.

Exit the registry and see if LAN settings is available now.
