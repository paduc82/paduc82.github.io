The term, Units, in Microsoft Project tells the software whether the resource will work full-time or part-time on the task.  If you enter a Units value of 100%, this means the resource will work full-time, or an average of 8 hours per day on the task.  If you enter a Units value of 50%, this means the resource will work half-time, or an average of 4 hours per day on the task.

Microsoft Project uses the Units value as part of what I refer to as the Duration Equation, where the Duration is calculated as follows:

Duration = Work/(Hours Per Day x Units)

The default Hours Per Day value in each project is 8 hours, so if you enter a Units value of 100% and 40 hours of Work for a resource, Microsoft Project will calculate a Duration of 5 days.  If you enter a Units value of 50% and 40 hours of Work for a resource, Microsoft Project will calculate a Duration of 10 days.  Does that make sense?

What Microsoft Project does not calculate is what you cite as an example in your original post.  We need to produce 100 widgets and a resource can create 20 widgets per day.  The software has no built-in mechanism to make that kind of calculation.  That is something you would need to manually.

Hope this helps.
