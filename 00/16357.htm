In the meantime I would like to give you some pointers to potentially work around some of the issues.
One of the two windows you see after double clicking the document is most likely a recovered version of your document. Word automatically opens that if it wasn't able to properly save your document during the last session or if it crashed. It would usually say Recovered in the title bar. In the case where you restart Word by opening the document you were working on we have a bug that makes it say "(Last saved to Mac)" or just the document title instead. The other issue is that if you try to save the recovered document (through CMD + S or the save button) we'll save it to the place of the original document. Once that has happened, the other window to the original document will start bringing up the Save As dialog, as it won't be able to save to its' original location anymore.

If you get into a bad state again and you have to close Word, or if you crash, I would suggest not opening Word by double clicking your document for now. That way you will only see the recovered document.

If you get into the state where two windows open, you'll be able to identifiy which one points to the recovery document by:
- Right clicking the title bar. For your original document it'll show you the path the document is located in. For the recovery document, it'll either do nothing or show you a path that is somewhere in your user folder.
- The recovery document shouldn't have an icon in the title bar.
- If you close the window the recovery document should show the save as dialog while the original document will just close.

suzanne s. barnhill say:

Sounds like that solution is for Excel rather than Word. There's no auto fill in Word. But if you apply numbering to the helper column, it will be automatically numbered. You'd have to then convert the numbers to text before sorting, using these instructions provided by Stefan Blom:

To convert autonumbering (paragraph/outline numbering and LISTNUM

fields) to plain text, do the following: Make sure the active document is the one you want to convert. Then press ALT+F11 to display the Visual Basic Editor. On the View menu, click Immediate Window. In the Immediate Window, type

   ActiveDocument.ConvertNumbersToText

and press ENTER.

Note that if paragraph/outline numbering was applied with styles, it isn't completely gone (CTRL+Q will bring it back!) unless you also clear it from the style definitions.
