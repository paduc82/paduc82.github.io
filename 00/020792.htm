jay freedman say:
Word cannot work with any page dimension larger than 22 inches (55.8 cm) in either direction. This is not something that will change in any update, and probably not in any future version.

The technical reason is that everything in Word that's concerned with page layout is measured in units of "twips", where 1 twip = 1/20 of a point, and there are 72 points per inch. Multiplying it out, 22 inches is equal to 31680 twips, while 23 inches is equal to 33120 twips. The largest signed 16-bit integer is 32767, which falls between those two values, so the limit was set at 22 inches.

To make it possible for Word to handle larger pages, all of the variables involved would have to be redefined as 32-bit integers. While not impossible, that would involve a lot of work, a ton of testing in all the different language versions, provisions for compatibility with older versions of Word, and possible unforeseen side effects (a.k.a. bugs). There's no way all of that could be justified in a business case.

Try using Microsoft Publisher instead. It doesn't have a page size restriction.

a. user say:


I entered "Find Method" into the Excel help search box and found the following example...
'---
'This example finds all cells in the range A1:A500 that contain the value 2 and makes those cells gray.
With Worksheets(1).Range("a1:a500")
    Set c = .Find(2, lookin:=xlValues)
    If Not c Is Nothing Then
        firstAddress = c.Address
        Do
            c.Interior.Pattern = xlPatternGray50
            Set c = .FindNext(c)
        Loop While Not c Is Nothing And c.Address <> firstAddress
    End If
End With
