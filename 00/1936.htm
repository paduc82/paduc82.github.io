macropod say:

If your data source contains empty cells (or cells with numbers) in that column, that could account for the behaviour. As a workaround, try changing the data connection method to DDE. In this regard, I think your concerns about DDE are unfounded, as you can use mergefield switches to change the data formats - just as you have already tried (unsuccesfully) with your current method.

 

Nevertheless, if you're wedded to using the default connection method, the following field code will convert the serial number to a date:

 

{QUOTE
{SET jd{=2415019+{=INT({MERGEFIELD Effective_Date})}}}
{SET e{=INT((4*(jd+32044)+3)/146097)}}
{SET f{=jd+32044-INT(146097*e/4)}}
{SET g{=INT((4*f+3)/1461)}}
{SET h{=f-INT(1461*g/4)}}
{SET i{=INT((5*h+2)/153)}}
{SET dd{=h-INT((153*i+2)/5)+1}}
{SET mm{=i+3-12*INT(i/10)}}
{SET yy{=100*e+g-4800+INT(i/10)}}
"{mm}-{dd}-{yy}" \@ "MMMM d, yyyy"}

 

Note: The field brace pairs (ie '{ }') for the above example are created in the body of the document via Ctrl-F9 (Cmd-F9 on a Mac) - you can't simply type them or copy & paste them from this message.

 

If you'd rather not encode the above by hand, you can download a macro to do it, from: http://www.gmayor.com/export_field.htm#TextToField

bob jones aka: cybertaz say:

There are no such limitations :-) You remember correctly that it can be done, but what you're misremembering is how to do it - It has never been the same as in Excel, nor have there ever been any 'boxes' in Word's Headers/Footers :-) Word's implementation of Headers & Footers is far more sophisticated & extensive than what's available for a spreadsheet.

Everything you've asked about & more is available, though. The most significant difference is that effective with Word 2008 [Mac]/Word 2007 [Windows] the antiquated Header & Footer Toolbar from prior versions was abandoned. The more commonly used are options available as always in the Insert> AutoText selections for Header & Footer as illustrated here:

Those as well as most anything else also can be added by the use of  Insert> Field, but you can't & never could just type the Field code. To enter them manually you first have to insert a Field by using the keystroke Command+F9 to generate Field Braces, then type the appropriate code within the braces. For example, if you choose to not use the AutoText feature you can insert the document name while in a Header/Footer by:

  1. Type Command+F9 to generate field braces {} (note that typing the braces does not work)
  2. Type FILENAME within the braces

To display Page X of Y requires inserting 2 fields -- 1 for the page number & 1 for the number of pages, so entering it manually would involve:

  1. Type Page followed by a space
  2. Command+F9 to insert the first field
  3. Type PAGE within the braces
  4. Move the insertion point to the right of the closing brace
  5. Type of preceded & followed by a space
  6. Command+F9 to insert another field
  7. Type NUMPAGES within the braces

It should also be mentioned that depending on your OS X Keyboard settings you may have to use the fn key in conjunction with Command+F9 if you have a laptop or condensed keyboard. Also, Option+F9 toggles the display of all field codes On/Off, Shift+F9 toggles the display of only the currently selected field code.

Hi Alyssa Janks,

Thanks for your reply in the Private Message.
1. According to your file, it seems to be an issue with formula.
In the default Excel Project Timeline Template, the value in the START column depends on the ENTER START DATE you insert.
If you double click the cell contains date in this column, you can see the default formula.

2. Each formula in the cell depends on the cell above it. 
For example: 
Formula in D6: =D5+1
If you delete the D5 row, then the formula in D6 row cannot find the value of D5 row and shows “#REF!”.
When there is a #REF! in the cells, your entire timeline graph deletes even though there are still other dates in the table.
3. To avoid this problem, you can edit the formula which shows #REF! manually.
