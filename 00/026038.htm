jllatham say:

You're using two array type formulas, one nested within the other.  Like ediarp, I think seeing example data and expected results would be a good thing.

I think the formula could be reduced to exclude the SUMPRODUCT() portion:

=SUMIFS(Table1[INVOICE_QUANTITY], Table1[ORDER_DATE],B1, Table1[ORDER_ITEM], Ip_M) * Ip_C

SUMIFS() is actually a function to replace (and perform faster than) the older SUMPRODUCT() function used in this fashion.

 

jeeped say:
... This seems to be heavy and taking long time to run when the defined list Ip_M and Ip_C gets larger ... Currently there are only 15 items but still takes time ...

You've mentioned that Ip_M and Ip_C currently have only 15 items so I'm assuming that the SUMPRODUCT() wrapper is to process through all 15×15 possibilities in addition to checking against a single date. SUMPRODUCT() is a little inefficient in this manner because it will process all possibilities of multiple conditions, regardless of whether conditions match or not. You might want to try a more conventional array formula that kicks out of any particular cycle as soon as one condition is not met. Assuming that the Table1[ORDER_DATE]=B1 criteria has the highest attrition rate, try something like,

=SUM(IF(Table1[ORDER_DATE]=B1, IF(Table1[ORDER_ITEM]=Ip_M, Table1[INVOICE_QUANTITY]*Ip_C)))
... or,
=SUM(IF(Table1[ORDER_DATE]=B1, Table1[INVOICE_QUANTITY]*IF(Table1[ORDER_ITEM]=Ip_M, Ip_C,)))
... each of which needs to be finalized with Ctrl+Shift+Enter↵ rather than just Enter↵. If Table1[ORDER_ITEM]=Ip_M has a higher failure rate, put that condition first.

Just to drop it here if someone may have a similar issue. The issue was resolved by inserting the second SIM card. The phone is dual SIM, and she was using it only with one SIM card. I don't know if the second thing is a smart thing to have on while in roaming though.

The option to create new Skype accounts using a user selected Skype Name was disabled about 2 years ago.
