So I've just finished testing this and it works!:

Create a new query named "qryExportPDF"

SELECT Master.* FROM Master

and use it for the Recordsource for the report.

Back on your main form ...

Private Sub cmdExportPDF_Click()

Dim MyPath As String
Dim MyFileName As String
Dim MyProjectNumb As String
Dim MyProjectName As String
Dim MySQL As String
Dim whr As String

MyPath = "C:\Reports\"

With Me.RecordsetClone
  Do While Not .EOF
   MyProjectNumb = !ProjectNumb
   MyProjectName = !ProjectName
   MyFileName = MyProjectNumb & " " & MyProjectName & ".PDF"
  
   MySQL = ""
   whr = ""
   MySQL = MySQL & "SELECT Master.* "
   MySQL = MySQL & "FROM Master "
   whr = whr & "WHERE (((ProjectNumb)= " & Chr(34) & !ProjectNumb & Chr(34) & " ));"
  
   CurrentDb.QueryDefs("qryExportPDF").SQL = MySQL & whr
  
   DoCmd.OutputTo acOutputReport, "Total Report", acFormatPDF, MyPath & MyFileName
  
   .MoveNext
   Loop
 End With
 
CurrentDb.QueryDefs("qryExportPDF").SQL = MySQL 'Resets Query to show ALL records

End Sub

Since Microsoft made available the first pre-release build of Windows 10 in early October 2014, there has been some misunderstanding that this is not a product you should use to replace a production installation of Windows such as Windows 8 or 7 or Vista. olicat say:

jay freedman say:

Yes, I would recommend a dropdown list content control. Also, I'll assume you're creating a template (*.dotx file) from which to create multiple new documents.

In the template, insert an empty dropdown list content control. While that control is selected, click the Properties button (below the Design mode button in the screen shot of my earlier reply) and enter something appropriate in the Title box. In the macro below, I've used "Criterion" as the title; if you enter something else, use that title in the macro (note that it's case-sensitive).

Next, press Alt+F11 or click the Visual Basic button on the Developer tab to open the macro editor. In the Projects pane, click on your template's name and click Insert > Module. Then copy/paste the following code into the code window on the right side.

Sub InitCriterionCC()
   Dim CC As ContentControl
   Dim CCs As ContentControls

   With ActiveDocument
      ' If the content control's Title is something else,
      ' change both instances of Criterion to the actual title
      Set CCs = .SelectContentControlsByTitle("Criterion")
      If CCs.Count = 0 Then
         MsgBox "No content control with title 'Criterion'", , "Error"
         Exit Sub
      End If

      Set CC = CCs(1)
      With CC.DropdownListEntries
         .Clear
         .Add "Each paraprofessional has a role and understands " & _
               "what is expected; has a daily schedule."
         .Add "Teacher works with related and special services to " & _
               "meet student needs and schedules meetings as needed."
         .Add "Furniture arranged for student access of materials, technology, etc."
         .Add "etc. ..."
      End With
   End With
End Sub

Replace the line .Add "etc. ..." with your next item, and add the rest of the items, each in an .Add statement like the others. Don't save the file until after the macro is complete and you've run it (see below). However, it would be a good idea to copy/paste the macro code into a Notepad file in case you need it again.

Here I've broken the very long lines into shorter pieces so they'll fit in the forum display. The ampersand (&) joins the pieces into a single string, and the space and underscore combination tells Word that the two lines are part of the same Basic statement. This isn't strictly necessary; in the macro editor you can just let the entire item sit on a single line.

When you have completed the entries, press F5 or click Run > Run Macro in the editor's menu bar. Now you can close the macro editor and save the template. You'll be prompted whether to save the .dotx file as a macro-free template, and you can click Yes -- the macro has done its job and isn't needed again. If you do need to make changes in the items, you can open the template, open the macro editor, insert a new module, and copy/paste from your Notepad copy; edit that and run it again. (Of course, you can make small changes in the content control's Properties dialog.)

dianeoforegon say: We see this type of behavior when a User has thousands of messages in their Inbox.


The Inbox syncs more frequently than any other and should be kept as clean as possible.

Any folder with more than a few thousand messages is going to take some time to fully come down when you first sync your account. Due to the design of Outlook's sync engine (and how exchange works), you will see the 512 newest messages in a folder at the initial start of sync then the rest of the folder's contents will be back filled before you see any mail that arrives after this point. This is why it will look like you are not getting new mail when a folder is still going through initial sync. This is also further exacerbated by an Exchange issue where it gets unnecessary change events before the newer mail arrives. If you carefully watch the progress during the sync of a folder with greater than a couple thousand messages, you will see "Updating local..." a lot before the newer mail starts arriving (newer than the initial 512 that come down).

Outlook checks the server for updates every minute, any folder that has updates will subsequently be synced. Since Outlook has a limit on how many folders can be synced at a time, there can be a queue of folders waiting. The Inbox does get high priority so it will generally sync before other folders that also need to sync.

I have a workaround for synchronizing the sent mail folder (or at least seeing the sent mail contents) after belle refresh update.

1. open the mail in question by clicking the widget. Inbox content will be visible.

2. tap the three horizontal lines at the bottom right corner of your screen to get to the SETTINGS. Tap SETTINGS.

3. All the maiboxes you have configured will be displayed. Tap the one you are interested in configuring.

4. Tap SYNC CONTENT

5. Tap FOLDER SUBSCRIPTIONS

6. Check ...SENT MAIL (this is the unix folder)

7. EXIT from the mailbox configuration menu.

8. Open the mailbox you have configured.

9. Tap the inbox bar on top left. It will show you all mailboxes you have configured on your device plus the available folders in the currently opened mailbox (inbox, drafts, sent, outbox, etc). Scroll all the way down to locate FOLDERS.

10. TAP folders.

11. You would see a SENT MAIL box with a folder icon with a ~sign beow it. Tap to open it.

12. There you are. You have accessed your SENT MAIL box.

13. A little indirect. But you can live with Belle Refresh till Nokia fixes the bug to deliver the sent mail content into your local sent mail folder.
