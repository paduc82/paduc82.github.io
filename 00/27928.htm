Hi Rich,

 

By linking "Microsoft Excel Chart Object" you are good to bind data from Excel chart to Word chart. Physical attributes are localized like Length/Breadth etc. within Excel/Word.

 

Linking will bind data only between Excel and Word individually not the other local attributes like length/breadth of chart object. So, if you change the size of the legend in Excel it will not reflect in Word.

john mcghie say:

Hi Keri:

Yes, that's what I would have done. I would be using Word VBA for this: it's much faster and more powerful than Keyboard Maestro, and far less likely to have issues with screen and resolution variance.

Yes there is a way to specify the tab of a dialog box, but it's laborious and not always robust.

Normally you would not even bother to display the dialog: simply issue its command directly. Run the Word macro recorder while you step through the sequence, then go into the Word Visual Basic Editor and look to see what it recorded.

You will see something like this:

    With Selection.Cells
        With .Shading
            .Texture = wdTextureNone
            .ForegroundPatternColor = wdColorAutomatic
            .BackgroundPatternColor = wdColorLightGreen
        End With
   End With

That command is the command that the dialog will call if you choose "Cells" in 'Apply to". The line "With Selection.Cells" does three things: the "With" keyword tells Word to remember what comes next because you are going to use it several times, the "Selection" keyword tells Word you are working with the Selection object, and the Cells keyword tells Word that of the several things that could be in the selection, it's the "Cells" we want to apply the shading to.

Note:  When you record this, Word records nearly 50 lines of code: that's because whenever you touch a dialog, Word records all of the settings in that dialog.  With practice, you can cut the recorded code back to just the bit you need, which will speed up your macro and make your code more readable.

However, when you are learning (and especially when you do not have time to play with this) you can leave the redundant code in there.  At least you know it will work, and it should not cause any problems.

If you do cut it back, be careful to match the With ... End With pairings.

Hope this helps

doug robbins - mvp office apps & services (word) say:

Use the following code in a macro that you run when the selection is in the table

Dim i As Long, j As Long

Dim crange As Range

With Selection.Tables(1)

    For i = 1 To .Rows.Count

        For j = 1 To .Columns.Count

            Set crange = .Cell(i, j).Range

            crange.End = crange.End - 1

            If IsNumeric(crange.Text) Then

                crange.ParagraphFormat.Alignment = wdAlignParagraphRight

            End If

        Next j

    Next i

End With
