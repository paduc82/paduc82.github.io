In short saying I am entering a record in the many table with no matching record in the one.

If it is not required why is this happening and do I have it set up wrong.  I want patients without doctors??

scottgem say:

Can a patient have multiple doctors? If so (and usually it is) then Gina's answer is correct. You don't have a one to many, you have a many to many. However, if a patient can only have one doctor (and this could be the case under certain circumstances) then you can't create a patient record without selecting a doctor. While you may not have set Doctor as required on the table level, you did set up referential integrity in the Relationships window. By enforcing RI you are requiring a Doctor to be selected otherwise you would be creating an orphan record (a child record with no parent).

So you can either remove RI or provide a dummy doctor record. If you do have a many to many use Gina's solution.
