Maria,
You're welcome and thanks for the feedback. I'm glad you found the culprit. It doesn't matter if the subtask is manual or auto-scheduled, if it still has an estimated duration then the summary (parent) of that group of subtasks will remain as estimated.

A non-integer duration can be caused by several things. Normally it is because either the start is not at the default start time (i.e. 8:00 am) or the finish is not at the default finish time (5:00 pm). I can also be due to a difference in calendars, if any calendar customization (e.g. resource custom calendar). Or a lag in a predecessor that is less than a full day. The easiest way to see what is driving the odd duration is to display the time along with the dates in the Start and FInish fields. In Project 2010, go to FIle/Options/General group. Set the Date Format to something that includes the time.

rename the registry key that contains the invalid search-related data. This disables the key. Then, create an Outlook profile. To do this, follow these steps.

HKEY_CLASSES_ROOT\CLSID\{9E175BAF-F52A-11D8-B9A5-505054503030}

2. On the General tab in the Mail dialog box, click Add.
3. In the New Profile dialog box, type a new profile name, and then click OK.
4. In the Add New Account dialog box, type your email account information, and then click Next.
5. After your account is successfully configured, click Finish.
6. Click Always use this profile, select the new profile, and then click OK.
