GPRS on Linux - Microsoft Community

goldenguy---01 say:

GPRS on Linux

Guide to using Nokia Phone as GPRS Modem to connect to the internet on Linux.
Many people like me use our cell phone to connect to the wonderful world of the internet. All phones today come with PC suite software which works very well in Windows operating systems. However when it comes to Linux, we simply have no software to achieve this feat, and many of us want to use our phone GPRS Modem with Linux as well. Up till now Nokia doesn’t provide a PC suite for Linux OS. So until they start to work in that direction, here’s my “guide to using Nokia phone GPRS modem with Linux”. I use a NOKIA 6131 phone with a Linux Mint desktop. However this guide will help you connect any Nokia phone with very little modification.
So get ourselves a notebook, a pen or pencil, and a can of coke :p

Step 1:
Take your notebook and write down the code (blue color text) listed in the preceding sections of this document for easy reference.
Step 2:
Connect your phone to your Linux desktop or laptop using the data-cable and select the default mode on your phone(this option doesn’t appear in all Nokia phones).

Open the terminal and type the command:

lsusb

you will get the following output:

nadeem@nadeem-desktop:~$ lsusb
Bus 002 Device 001: ID 0000:0000
Bus 001 Device 004: ID 0421:047b Nokia Mobile Phones
Bus 001 Device 002: ID 046d:092f Logitech, Inc.
Bus 001 Device 001: ID 0000:0000
nadeem@nadeem-desktop:~$


Now note the line in which nokia Mobile Phones is written...it has two number one is 0421 & other is 0445...we'll take these numbers as 0x421 & 0x445

0421 is the Vendor ID & 0445 is the Product ID
Write down these two numbers on your note-book.

Step 3:

Then enter the following command:

sudo /sbin/modprobe usbserial vendor=0×Vendor ID product=0× Product ID


In the above command you have to replace the Vendor ID and Product ID values of your phone that you noted down. Eg in case of my phone, the command is
sudo /sbin/modprobe usbserial vendor=0×421 product=0×445
Ignore any output that appears and type the following command:

wvdialconf /etc/wvdial.conf


You will get a long output which will be like:

Scanning your serial ports for a modem.
Port Scan: S0 S1 S2 S3
WvModem: Cannot get information for serial port.
ttyACM0: ATQ0 V1 E1 — OK
ttyACM0: ATQ0 V1 E1 Z — OK
ttyACM0: ATQ0 V1 E1 S0=0 — OK
ttyACM0: ATQ0 V1 E1 S0=0 &C1 — OK
ttyACM0: ATQ0 V1 E1 S0=0 &C1 &D2 — OK
ttyACM0: ATQ0 V1 E1 S0=0 &C1 &D2 +FCLASS=0 — OK
ttyACM0: Modem Identifier: ATI — Nokia
ttyACM0: Speed 4800: AT — OK
ttyACM0: Speed 9600: AT — OK
ttyACM0: Speed 19200: AT — OK
ttyACM0: Speed 38400: AT — OK
ttyACM0: Speed 57600: AT — OK
ttyACM0: Speed 115200: AT — OK
ttyACM0: Speed 230400: AT — OK
ttyACM0: Speed 460800: AT — OK
ttyACM0: Max speed is 460800; that should be safe.
ttyACM0: ATQ0 V1 E1 S0=0 &C1 &D2 +FCLASS=0 — OK
Found an USB modem on /dev/ttyACM0.
Modem configuration written to create.
ttyACM0: Speed 460800; init “ATQ0 V1 E1 S0=0 &C1 &D2 +FCLASS=0″


Notice the output says that there is a modem at /dev/ttyACM0 & max speed is 460800.
You may note this on your note-book.

Hey, don’t forget your coke….:p….take a sip.

Step 4:

Then enter this command:

sudo gedit /etc/wvdial.conf

A file will open in the text editor(gedit). Type the following there:

Phone = *99#
Username = username
Password = password
Stupid Mode = 1


Your phone number may be different and in that case contact your operator and get the phone number.

You may also Delete everything in the file & paste this: (Try this if above code doesn’t work).

[Dialer Defaults]
Modem =[COLOR=”Red”] Your Modem Name(eg, /dev/ttyACM0 in my case)[/COLOR]
Baud =[COLOR=”Red”] ur max speed(460800 in my case)[/COLOR]
Init1 = ATZ
Init2 = ATQ0 V1 E1 S0=0 &C1 &D2 +FCLASS=0
ISDN = 0
Modem Type = Analog Modem
Phone = *99#
Username = username
Password = password
Stupid Mode = 1

Save the file.
Exit the terminal.

You have completed configuring your phone. Now we need to connect to the internet, which is described in the next step.

Step 5:

Open terminal & type the following command:

wvdial

You will get a long output for 1 or 2 steps, at the end of which an IP address is displayed.

pppd: [06][06][08]` [06][08]
primary DNS address 218.248.240.135
pppd: [06][06][08]` [06][08]
secondary DNS address 218.248.240.79
pppd: [06][06][08]` [06][08]


Congrats, you successfully connected to the internet.
Keep the terminal running, open ur browser and enjoy 

Note: Don’t type anything in the terminal while you are connected. If you do need to use the terminal, just open another one.

For disconnecting , just hit ctrl+c.


I have used a Linux mint desktop with Nokia 6131, try with ur phone and ur linux flavor and do post feedback on how it works.

matthew flanigan say:

Word - Envelopes and Labels - Printing issue

There is a problem with printing Envelopes and Labels from the Envelopes and Labels dialog and printing to HP printers. When clicking "Print" from the dialog, the job is sent to the printers bypass tray (Tray 1) with the correct media size information, but fails to inform the printer what type of paper it should be printing on. This causes the printer to assume you're printing on the default type of media in Tray 1, and if Tray 1 is set for "Any Size, Any Type", it defaults to the printers default type, which is typically "Plain". If the printer thinks it's printing on plain paper, and a label or envelope is sent through, the toner will not fuse properly to the thicker media, causing it to flake off.

For example:

I am using the Envelopes and Labels dialog in Word to print an address on an envelope. I open the "Options" and the "Feed from:" setting is "Default tray (Automatically Select)". I click "OK" and then "Print". I look at my HP printer, and it's asking me for "PLAIN ENV #10" media to be put into Tray 1. 

In this example, PLAIN ENV #10 is asking for the media type of plain paper (PLAIN), the size of a #10 envelope (ENV #10). When the printer prints on plain paper, the fuser doesn't need to get very hot, and it prints fast. When the printer prints on an envelope, the fuser needs to heat up more to penetrate the multiple layers of paper, and it slows down to aid in the heat process and to reduce the chance of creasing the envelope. When an envelope is sent properly to the printer, the display will read "ENV ENV #10", which is specifying the media type to be an envelope (ENV), and the media size to be a #10 envelope (ENV #10).

My question is: Can the Envelope and Labels dialog be set permanently to default to the type of media it's eluding to printing on regardless of if the tray 1 of the printer is set to "Any Size, Any Type". This is the dialog box you see when you click the "Options" button in the "Envelopes and Labels" dialog. You can see at the top portion of the "Envelopes Options" dialog that the Envelope size is Size 10. This is telling the printer that it will be printing on a #10 sized envelope (media size).

The problem is the following:

1. The MP Tray (Tray 1) of the printer is almost always configured for "Any Size, Any Type" so that the customer can load any media size necessary that is not already loading in a cassette below. (for instance, if they have plain paper in tray 2, but need to print a legal, executive, label or envelope, they would load the odd size media into the MP Tray to print. 3. The current Envelopes and Labels wizard ONLY sends the media size information, and does not send the media type information. Because the MP Tray is set for "Any Size, Any Type", the printer recognizes that you are attempting to print on a document the size of a #10 envelope, but is not telling the printer the type of media, causing the printer to assume that it's printing on plain paper (which is typically the printer's default media type).

What I am proposing is that, because the user is printing from the Envelope and Label wizard, it is safe to assume that they are printing on a envelope or label, and thus the default behavior for printing from this dialog should be to automatically set the media type to envelope or label, instead of relying on the printer's default settings.

7thsense say:

I found this procedure in Nokia's forum. Although it is a bit of work, it should solve your problem:

  1. In Nokia Suite, select all contacts.  File > Export Contacts creates a .vcf file for each contact on your PC.
  2. Upload all the .vcf files to SkyDrive.  (I did them in batches)
  3. Download each .vcf file to your Windows Phone.  I had to use SkyDrive in a browser, because the SkyDrive App didn't allow me to do anything with the files.
  4. Open each .vcf file in turn and save it to your phone - this writes them into People.

Hope this helps.
