Is there a macro that can transpose columns and add rows to match transposed columns for each row in column A?


There is now.

Tap Alt+F11 and when the VBE opens, immediately use the pull-down menus to Insert, Module. Paste the following into the new pane titled something like Book1 - Module1 (Code):

Sub mcrTransposeQtys()
    Dim dt As Range, cn As Range, dws As Worksheet, nws As Worksheet, sSheet As String
    Dim lcr As Long, ldt As Long
    Set dws = ActiveSheet
    Sheets.Add After:=Sheets(Sheets.Count)
    Set nws = Sheets(Sheets.Count)
    With dws
        nws.Range("A1:C1") = Array("Cus_Item", "Date", "Qty")
        lcr = 2
        For Each cn In .Range("A2", .Cells(Rows.Count, "A").End(xlUp))
            ldt = 1
            For Each dt In .Range("B1", .Cells(1, Columns.Count).End(xlToLeft))
                nws.Cells(lcr, 1) = cn.Value
                nws.Cells(lcr, 2) = dt.Value
                nws.Cells(lcr, 3) = cn.Offset(0, ldt)
                ldt = ldt + 1
                lcr = lcr + 1
            Next dt
        Next cn
    End With
End Sub

Tap Alt+Q to return to your worksheet. With your data sheet active (e.g. showing), tap Alt+F8 and Run the macro. You had no blank cells in the quantities data so I did not compensate for them.

jeeped say:

... How would you compensate for blank quantities?


This depends a bit on whether you are talking about blank entries or cells containing blanks or zeroes and what you want to do with them One possibility would be to use an If ... End If statement for qty>0 before creating the record. If not greater than zero, ignore it.

     
           For Each dt In .Range("B1", .Cells(1, Columns.Count).End(xlToLeft))
                If Int(cn.Offset(0, ldt).Value) > 0 Then
                    nws.Cells(lcr, 1) = cn.Value
                    nws.Cells(lcr, 2) = dt.Value
                    nws.Cells(lcr, 3) = cn.Offset(0, ldt).Value
                    lcr = lcr + 1
                End If
                ldt = ldt + 1
            Next dt
     

Note that the lcr (e.g. current row) is only incremented if the record is created. The ldt (e.g. date's quantity) needs to be incremented regardless.

doug robbins - mvp office apps & services (word) say:

If you set up a folder structure on the new computer that is the same as that on the previous one, and put the documents and the photos in the relevant folders, the links should still work.

Alternatively, on the old computer, if you open the document and use Ctrl+A to select everything and the use CTRL+SHIFT+F9, the photos will be embedded in the document.  However, that will also convert other fields in the document to text.

If that's going to be a problem, you could use ALT+F9 to toggle on the display of the field codes in the document and then use the replace facility to replace the relevant part of the field codes for the field code for the picture with the current path to the photos.
