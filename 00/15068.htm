roady [mvp] say:

The really annoying aspect is that the bounced email does not get copied to any of my "sent" folders. GMail's bounce message does not include the Original message so the whole thing is completely lost. All my emails are copied to a sent folder except these ones.

Yes, that is due to Gmail's non-standard implementation for their SMTP servers. You are not actually saving your sent items to the Sent Items folder yourself but you are sending it to their SMTP server which then also send a copy to you but gets moved to your Sen Items folder via a "hidden rule".

This indeed means that when your message bounces, it can't be found in your own mailbox anymore unless the bounce itself contains your original message in full (which is rarely the case).

You can configure Outlook to save a copy as well but this would result in duplicates when the message was sent successfully. See: Prevent duplicate Sent Items when using Gmail

A workaround would be to create a message rule to save a copy in a different folder but then you'd be creating a workaround for an exception for something that should have worked different to begin with. A similar solution is being used here: Sorting Outgoing E-mails.

I don't know why Gmail still does this. Originally is was a nice solution for the POP3 users that also wanted to have their Sent Items online. Nowadays, POP3 isn't used that often anymore for Gmail and it's just a pain to deal with this non-standard behavior of an SMTP server and can't even turn it off.

Select the text in your header and Cut it (Ctrl+X). On the Header & Footer Tools | Design tab, check the box for "Different First Page." You'll now have a header labeled "First Page Header."
