7.  Start the Zune software and connect your phone to create a new certificate.

Follow the steps mentioned below to unblock the files that do not have a valid digital signature to load on your system.
a.      Click on ‘Start’ button.
b.      Type “Internet Explorer” in the “Start Search Box” and select it from the menu.
c.      Click on “Tools” in the menu bar and select “Internet Options”.
d.      Click on “Advanced tab” and locate “Allow software to run or install even if the signature is invalid” under the “Security” category and check the box.
e.      Click Apply and then ‘Ok’.
f.      Close the Internet Explorer and restart Internet Explorer.

I intended to fix that and forgot. Just to explain why it does it on the first save and not subsequent saves is that the first time it adds the new worksheet and when a new worksheet is added, it automatically becomes the Active Sheet. On subsequent saves it does not create a new sheet and therefore the worksheet with the form remains the Active Sheet.

Anyway the fix is easy and as you are taking an interest in the code, guidelines to fix as follows

  1. Right click the tab name for WKSHT1.
  2. Select View code to open the VBA editor at the code module for WKSH1
  3. Find the sub "Private Sub btnSave_Click()"
  4. Then find the "End Sub" line for this sub.
  5. Insert the a line "Me.Activate" (without the double quotes) between the lines "Call NextEnclosure" and "End Sub" so it looks like the following. (Doesn't matter if there are any blank lines between the code lines because they are ignored. Blank lines are often left to make the code easier to read.)

 

    Call NextEnclosure
    Me.Activate
End Sub

Explanation of Me. It is the generic object reference of the worksheet to which the code module belongs. In lieu of Me.Activate you could use the following code.

Worksheets("WKSHT1").Activate
