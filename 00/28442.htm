Hiding duplicates in a form using conditional formatting - Microsoft Community

tomg. say:

Hiding duplicates in a form using conditional formatting

Hi,

I have a form of shipments by container number that looks like this:

Container No Part No Quantity
ABCD1234 00001 10
ABCD1234 00002 20
ABCD1234 00003 30


The form is continuous & displays a lot of data - so to make it easier to read - all I want to do is use conditional formatting to format duplicate container numbers so they use a white font on a white background.  The end result would look like this:

Container No Part No Quantity
ABCD1234 00001 10
00002 20
00003 30

 Looking for help with the expression that I need to use in the conditional formatting dialog box.

Thanks,
Tom

ken sheridan say: If you compute the ordinal position of each row in the subset of rows per container number in the form's underlying query, you can then use this computed value as the basis of an expression to format the control's ForeColor to white.  Assuming the form is to be updatable the position can be computed using the DCount function.  You'll find an example in RowNunbering.zip in my public databases folder at:

https://onedrive.live.com/?cid=44CC60D7FEA42912&id=44CC60D7FEA42912!169

If you have difficulty opening the link copy its text (NB, not the link location) and paste it into your browser's address bar.

In this little demo file the form frmNumberRowsByGroup_Updatable has the following query as its RecordSource:

SELECT DCOUNT("*","Transactions","CustomerID = " & CustomerID & "
AND TransactionDate  <=  #" & Format(TransactionDate,"yyyy-mm-dd") & "#
AND (TransactionID <= " & TransactionID & "
OR TransactionDate <> #" & Format(TransactionDate,"yyyy-mm-dd") & "#)") AS RowNumber, CustomerID, TransactionDate, TransactionAmount
FROM Transactions
ORDER BY CustomerID, TransactionDate, TransactionID;

This numbers each transaction per customer in date order.  So if your form's query orders the rows in Part No order per Container No you can do the same.  The DCount function's criteria would be far simpler in your case as you would presumably not need to take account of duplicate part numbers per container, so would not have to bring the primary key into play as the tie-breaker, as is done with TransactionID  in my query to differentiate between two or more transactions by the same customer on the same data.

If you amend the form in my demo by transitionally formatting the CustomerID combo box control with the expression [RowNumber] > 1 so that its ForeColor is white, you'll see only the first instance of each customer's name in the form.   You can also delete the RowNumber control from the form if you wish; the control isn't necessary for the conditional formatting, only the computed column in the query.

marshall barton say:

In Access, you need to sequence number the records with the same container number and different part numbers.  Then you can use CF on that sequence number.  The form's record source query would look something like:

SELECT S.[Container No], S.[Part No], S.Quantity,

       (SELECT Count(*) FROM tblShipments As X

        WHERE X.[Container No] = S.[Container No] And S.[Part No] < S.[Part No]

       ) AS SeqNo
FROM tblShipments AS S
ORDER BY S.[Container No], S.[Part No]

Then set the CF Expression Is: option to:  

     [SeqNo] > 0
