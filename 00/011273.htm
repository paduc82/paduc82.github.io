davidf2 say:

The links provided by Mohan will describe how to get links to other web pages to open in a new window. It won't necessarily work in making PDF or Word files open in a new window or tab. The user has control over whether they want to download those files to their local computer or whether they have installed the plug-ins that allow them to be opened in the browser. You only have so much control.

One way you can force the download is to put the PDF or Word files in a ZIP file. When clicking a link to the zip file the user will be asked if they want to download the file...it will not open in the browser regardless of the plug-ins or user settings. This is probably you easiest solution.

One more note. MS started deprecating the web building functionality of Publisher as of Pub 2010. Unless you are using a web publication from a previous version of Publisher some of the web functionality is hidden from you. If you open an old web file in Pub 2010 you will be given a Web tab. It is on that web tab that you have the ability to insert a html code fragment.  For the methods described in my referenced threads to work you must be able to insert code fragments.

Reference: Web functionality Pub 2010:
http://office.microsoft.com/en-us/publisher-help/web-functionality-HA101877287.aspx

http://blogs.office.com/b/microsoft-publisher/archive/2010/12/24/where-is-web-site-authoring-in-publisher-2010.aspx

If you are not using an old Web publication file then you don't have the code fragment option. If not there is one web publication template file still available on-line. For some reason MS has removed the ability to peruse their templates site for all Publisher templates (https://templates.office.com/Templates)  but you can still download it.

Go to File > New > Online Templates > More templates > More Categories > All Business > at the bottom of that group of templates click > More Office.com Templates (152) > scroll to the bottom and find the Web site (tabs design) template and download it. If you open that web publication template you will see the Web tab on Pub 2010 and be able to access the Insert HTML Code Fragment feature. You could test your ability to link and open a Word or PDF file in a new window using this web template. If it works then edit this template/copy all your content to this template and you will be good to go.

In general I would suggest you start finding an alternative to using Publisher for web building. As of Pub 2013 MS has further hobbled the web functionality be removing the insert html code fragment tool altogether. Look at MS ExpressionWeb, Serif WebPlus, Word Press for free versions. WebPlus is where I wish MS had taken Publisher.

DavidF

danida_u say:

Skype does not keep a record of your Skype-to-Skype calls. The record of your Skype-to-Skype calls are in the conversation window of the contact.

You cannot export this individually because the Skype calls data are inside the conversation. When you export your conversation into a CSV file (available on the classic Skype for Windows desktop) only the history saved on your hard drive, according to your 'keep history for' setting, is exported.
