It's important to distinguish between relationships between tables, and joins in a query.  The existence of the latter does not necessarily imply the existence of the former.  Add the following procedure to any standard module, and be sure that the debug window (aka immediate window)  is open; press Ctrl-G to open it if not.  What do you see in the debug window after executing the function by pressing the F5 key with the cursor anywhere in the procedure?

 

Public Sub ListRelationships()

 

    Dim rel As DAO.Relation

    

    For Each rel In CurrentDb.Relations

        Debug.Print rel.Name, rel.Table, rel.ForeignTable

    Next rel

   

End Sub

 

If the relationships exist then you should see them listed by name, referenced table, referencing table.

Windows (currently Windows 7) is an operating system and there is no trial although there are a number of flavors that you can choose from (OEM, Home, Professional, Ultimate, etc) and a new computer will often come with a version pre-installed that you are free to use.
