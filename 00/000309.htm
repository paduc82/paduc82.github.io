hansv mvp say:

To set the page breaks:

Sub SetPageBreaks()
    Dim r As Long
    Dim m As Long
    ActiveSheet.PageSetup.PrintTitleRows = "$1:$10"
    ActiveSheet.ResetAllPageBreaks
    m = Cells.Find(What:="*", SearchOrder:=xlByRows, SearchDirection:=xlPrevious).Row
    For r = 36 To m Step 25
        ActiveSheet.HPageBreaks.Add Before:=Rows(r)
    Next r
End Sub

There is no way to preset the number of copies to be printed without showing the print dialog. You could use the following macro to print. It will set the number of copies to 5 and show the print dialog:

Sub Print5Copies()
    Application.Dialogs(xlDialogPrint).Show Arg4:=5
End Sub

albert d. kallal say:

Actually the best way to set the sort order in a database, is simply to create a query and set the sort order using that query.

 

In fact as a general rule a database is considered just a big unordered bucket of data. If you have some order of records in table and some users start deleting records a few more might be adding recods at the same time. Remember the database could be multi user. So, the actual physical data order in the table can never be really guaranteed and it can change during edits. This has been a standard assumption in the whole database industry from small desktop programs like Access all the way up to the big heavy duty mainframe systems that run eBay.

 

The other issues is the databases engine will often optimize the data layout for performance issues.

 

There's actually several more reasons I could write on about, but simply suffice to state that if you need the order of the data you simply build a query and thus TELL THE data engine to return the data in the order that you require.  So, tell the database engine to retun you datain the order you want. You just can not hope on physical layout of data will save the day (it will not and it will change on you).

 

As for the columns changing positions? I wasn't really quite aware than Access did that (it not occuring for me). I'm wondering if you have some type of import specification set up for this. However once again building a query will solve this issue. As a general rule if you're building any kind of application you'll likely prevent users from working with the data in the tables directly anyway.

 

However, I do find that if I bring up the table in design mode and move the columns around to some particular order, then if I open the table in view mode the columns do keep and respect that order.

 

Albert D. Kallal
