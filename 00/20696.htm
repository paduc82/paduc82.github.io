jay freedman say:

According to this MSDN article, for Office 2013 "This object, member, or enumeration is deprecated and is not intended to be used in your code."

In Word 2013, I find that the .AutomaticValidation statement causes the same error, and MSDN says that property is also deprecated.

(I've had arguments with the Microsoft developers, who no longer believe that "deprecated" means "use is discouraged, and the feature will be removed in a later version." At least, that's what it used to mean. Now they just yank things out without warning.)

The workaround is to put the code fragment you showed into two separate functions, one with and the other without the offending statement. Use the Application.Version property to decide at runtime which function to call. I've verified that this works:

Sub test()
    If Application.Version > "14.0" Then
        UseInWord2013
    Else
        UseInWord2010
    End If
End Sub
Function UseInWord2010()
    With ActiveDocument
        .UpdateStylesOnOpen = True
        .AttachedTemplate = sourcefile
        .XMLSchemaReferences.AutomaticValidation = False
        .XMLSchemaReferences.AllowSaveAsXMLWithoutValidation = False
    End With
End Function
Function UseInWord2013()
    With ActiveDocument
        .UpdateStylesOnOpen = True
        .AttachedTemplate = sourcefile
    End With
End Function

rohn007 say:

It depends on how you bought Office 2010.  The "Retail"/FPP license (big ugly yellow plastic CD box) of Home and Student allows you to install on up to 3 computers at the same time.  But if you bought it from the factory, "OEM" license, or as a "Product Key Card", then you can install on only one computer at a time.

Even if you install just Publisher, it is counted as a full installation.  So, if you have retail Home and Student, then installing publisher on this new computer would count as your second of three allowed installations.

If you have an OEM or PKC license, then you will not be allowed to install even just publisher on the second computer.

As Palcouk has pointed out, the preferred method is to install oldest first, but you should be OK installing just Publisher on the second computer, even though you already have 2016 installed.

Assuming there are no blanks in your one dimensional array:
 
Array:  A1:A10
Value to be tested:    B1
 
This formula must be **array-entered**:
 
=--OR(B1=A1:A10)
----------------------------------------
 
To **array-enter** a formula, after entering
the formula into the cell or formula bar, hold down
<ctrl><shift> while hitting <enter>.  If you did this
correctly, Excel will place braces {...} around the formula.
 
If there are blanks, and they are to be ignored unless B1 is also
blank, then:
 
 
=--OR(B1=IF(LEN(A1:A10)>0,A1:A10))
 
(also array-entered)
