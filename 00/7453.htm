How to create a checklist in access 2016

I have thought about doing a checklist.  I have watched a video by Tony Hines that functions like what I want it to do but the video does not show how to build the tables with the specific data types.  (Some I could guess what type, others I could not).  I was wondering if a checklist is the right way to do this.

I was going to put on checklist on the data entry form as a subform.  My data entry form has tab controls with subforms on each tab control, except the first tab which is based on the individual table.

Thanks


scottgem say:

First, you have to understand that data storage and data presentation are two separate items. 

First to the storage issue. What you have is a many to many relationship. One member can belong to many groups and one group can have many members. So the proper way to design this is with a junction table like so:

tjxGroupMember

GroupMemberID (PK autonumber)

GroupID (FK)

MemberID (FK)

You would also have a unique index on the combination of the 2 FKs to prevent duplication. You would also need a Groups table:

tluGroups

GroupID (PK Autonumber)

GroupName

Now once you get that, there are several ways to enter the data. The easiest is by using a Continuous form subform bound to the junction table and linked on MemberID. The subform will have one combobox that lists all the groups and you add a record for each group the member belongs to. 

You could have a checkbox interface, but then you would have to handle getting the data from the form to the table and vice versa in code. If you want more help with that let us know.

There is another alternative, that of using a Multi-Value field. This is created by using a Lookup field in the table design. This presents a hybrid combobox/checklist interface. The problems with MVFs are that they are not scalable to other database systems. And its harder to do queries on tables with MVFs.

Windows 8 and 10 have a feature called Fast Startup that is like a hybrid hibernation\shutdown...

Sometimes certain drivers won't load properly this way, and can result in similar issues as you describe.

sanjuovi say:

Cannot connect Nokia n97 with USB:

 

If you already installed PC Suite and failed to connect N97 with USB cable try this: (Tested with several Notebooks and PCs with Vista 32 bit OS installed)

 

1. Uninstall “Nokia Connectivity Cable Driver” from “Add Remove Program” (before uninstall it, turn off user account control from “Control panel”-> “User Account”à“Turn User Account Control On or off”).

 

2. Restart PC.(Make sure you remove USB cable before restart)

 

3. Install Nokia_Connectivity_Cable_Driver_7_1_8_0.msi from OVI SUITE CD supplied with your N97. (Or you can download it from: http://www.nokia.com.sg/support/download-software/cable-drivers/driver-for-nokia-dke-2-dku-2-ca-42-ca-53-ca-70  or http://www.4shared.com/file/PkL6yh60/Nokia_Connectivity_Cable_Drive.htm )

 

4. Open Pc Suite

 

5. Connect Nokia USB cable to Notebook.

 

6. Connect other end of USB to N97, select PC suite mode from mobile when prompt (Window will start installing device drivers, let it install all drivers. After window finishes installation you will see many N97 drivers at “Device Manager” with Question marks).

 

7. Open “Control Panel” --> “System”--> “Device Manager”-->Right Click on Nokia drivers with question marks on them and select "Properties" -->  Select "Driver"--> select " Browse my computer for driver software"--> Select "Let me pick from a list of device drivers on my computer"--> from the list of drivers choose Nokia USB Generic or any other driver that do not has the word N97 with it(don't choose Nokia N97 USB parent).

 

8. Repeat step 7 for all Nokia drivers with question mark or exclamatory mark until they are gone (There will be no marks on them when the driver is working)

 

9. After all drivers related to Nokia installed properly you will see "N97 connected with USB" on the task bar.(Pc suite)

 

If you follow those steps above properly it will work 100%.
