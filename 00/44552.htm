                                                                                                                                                                          &

[Original title: Terry Sutton]

¡firedog say:

The message is not likely to be delivered. A 'routing loop' could be a fault at one or more of the mail servers between you and the addressee; if server K sends the message on to server L, which then re-directs the message back to server K, it's in a loop and won't ever get any further.  Try sending it again; if the same happens, you'll have to contact the addressee by other means and tell him what's happening; he may have to contact his email service provider.

Otherwise, the Delivery Status Notification usually gives more details about the servers involved in a Non-Delivery Report. This NDR will either be delivered together with the DSN as a (plain text) attachment, or shown as a separate MIME part in the source of the DSN. View the message source in whatever client you use to read your email and look for a line starting with Diagnostic-Code. That is one of the 5-10 lines in a typical NDR.

david m. connell say: Sorry about the late replay, but I wanted to get a feel for the cover before reviewing it.

The Incipio Feather cover is in 2 pieces. It slides over the RT and snaps to itself. It's not the strongest connection, so it occasionally slides loose just a little bit, generally the top portion. There are cutouts for all the ports, so the cover doesn't have to be removed to access any of the functions. This also means the ports are exposed in the event of a liquid spill.

I chose this cover because it retains the kickstand feature of the Surface rather than providing its own stand. That works very well - the kickstand portion of the cover has small magnets that secure it nicely to the kickstand. These same magnets help keep the lower portion of the cover in place.

It's also designed to not interfere with attaching a touch or type keyboard cover, but I can't verify if that works properly as I don't have a touch cover yet.

While the cover will protect against scratches and probably small impacts, I'm not sure if it will protect against serious drops, and obviously I'm not going to test that deliberately. ;-) It isn't really padded, so  a Surface inside it won't have much additional protection from impact. It'll provide greater protection if the Surface is dropped on its edge or face down - the edges of the cover are nice and thick. The back is basically just a piece of moderately flexible plastic - a little give, but not a lot of cushion.

The Feather cover does not protect the screen, as it is designed to supplement the keyboard cover rather than replace it. But InvisibleShield screen protectors are a must-buy for me with any piece of tech, so that's not an issue. There is a raised section around the screen, which will keep the screen from hitting the floor if the Surface is dropped face down. But this won't protect it from things placed on the screen or dropped on it - a screen protector or keyboard cover is recommended.

All-in-all, it's a good case for adult use. Whether it'll work in your case, professor, depends on how old the kids are. It'll add some protection, but it's a far cry from being an Otterbox.

Errors that end with 0002 are file not found errors (from the old DOS days) and that should be what the rest of the message says but it does not tell you the name of the missing file (it might be in the Event Viewer).

So you did the right thing to run SFC and it sounds like that resolved that problem which is fortunate and that you have tested System Restore from end to end by making a Restore Point and then successfully restoring the system to that RP you created.

All I can tell you about the VSS Service is that it does not get installed as Automatic start and yes there are many Internet topics about setting it to Auto which will not cause an issue but it doesn't need to be that way if you are concerned about system resources like CPU cycles and RAM going to running Processes that are not going to be used.

When looking at topics in community forums for Windows problems and you see advice to set it to Auto does the topic end up with a good resolution of the problem being solved with just that one adjustment or does the topic contain many other troubleshooting steps and how many of the troubleshooting steps begin with the word "try"?  

If a suggestion begins with the word "try" that tells me that the person making the suggestion has probably not actually recreated the problem on their own system and taken their own advice to see if it actually solves the problem.  In other words they have not tested their idea to see if it actually works to solve the problem.

Not everybody shares these opinions.

Like Yoda says:  Try not.  Do or do not.  There is no try.

If after setting it to Auto and things work okay, set it back to Manual like it is supposed to be and things will still work okay (unless something else is wrong) but the chances are in those topics many other things have been "tried" to resolve the problem so who knows what actually was the solution?  Sometimes there are days and days of trying things...  I would rather do things.

Now that your System Restore is working and seems reliable you can see about your BSoD issue.  It is important (to me) when working on BSoDs to make a Restore Point before making any adjustments involving a BSoD so in case something goes wrong or things get worse you can always restore the system back to a date/time prior to the adjustments and start over.

P.S.   You will also find that there are probably millions (literally) of topics/with issues with Windows Backup and Restore and after trying many, many many things the "solution" is to set the VSS Service to Automatic - also not necessary for the same reasons.
