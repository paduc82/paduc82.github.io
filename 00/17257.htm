jay freedman say:

To "hide" a command button or any other inline shape, you can set its brightness to 100% and its contrast to 0%. That will leave a white space where the button is, and it will print that way. To restore the button's visibility, you set both the brightness and the contrast to 50%. The following set of four macros does that.

Because of their special names, the PrintPreviewAndPrint macro runs instead of showing the File > Print display when the user presses Ctrl+P, and the FilePrintDefault macro runs when the user clicks the Quick Print button. (There is a way to intercept it when the user clicks File and then clicks Print, but it's considerably more complicated than just writing a macro with a special name.)

Sub PrintPreviewAndPrint()
    HideCmdButtons
    Dialogs(wdDialogFilePrint).Show
    ShowCmdButtons
End Sub

Sub FilePrintDefault()
    HideCmdButtons
    ActiveDocument.PrintOut
    ShowCmdButtons
End Sub

Private Sub HideCmdButtons()
    Dim oButton As InlineShape
    
    For Each oButton In ActiveDocument.InlineShapes
        If oButton.Type = wdInlineShapeOLEControlObject Then
            With oButton.PictureFormat
                .Brightness = 1
                .Contrast = 0
            End With
        End If
    Next
End Sub

Private Sub ShowCmdButtons()
    Dim oButton As InlineShape
    
    For Each oButton In ActiveDocument.InlineShapes
        If oButton.Type = wdInlineShapeOLEControlObject Then
            With oButton.PictureFormat
                .Brightness = 0.5
                .Contrast = 0.5
            End With
        End If
    Next
End Sub

teylyn say:
Hello,

You can use a macro to insert a new row, but it will be easier to tie it to a button than to the tab key.

For the following I assume that your table starts in column A.

In the workbook, press Alt-F11 to open the VB Editor. Click Insert > Module and paste the following code into the code window:

Sub AddRow()
    ActiveSheet.Unprotect Password:="secret"
    Range("A1").End(xlDown).ListObject.ListRows.Add AlwaysInsert:=False
    ActiveSheet.Protect DrawingObjects:=True, Contents:=True, Scenarios:=True, Password:="secret"
End Sub

Change the password from "secret" to a password of your choice.

Insert a row or two above the table, insert a shape and edit the shape text to something like "Add row". Right-click the shape, select "Assign Macro" and select the "AddRow" macro.

Remember to save the workbook as a macro-enabled file with the xlsm extension.

When the user clicks the shape, the worksheet will be unprotected, a new row will be added and then the sheet will be protected again.

To keep the button visible at all times, you can freeze the first rows of the spreadsheet.

cheers, teylyn


teylyn say:
Hello,

if you format the table cells to be unlocked, then protect the sheet and tick the box next to "Insert rows" in the list of settings the user is allowed to do, then the end user can insert rows into the table.

The tabbing to automatically create a new row at the end does not work in a protected sheet, but the user can select the total row and use the Insert row command to add a row to the table.
