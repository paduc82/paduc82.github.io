gary's student say: This is for your first case only.  Install the following event macro in the worksheet code area:

Private Sub Worksheet_Change(ByVal Target As Range)
Dim Source As Range, Destination As Range, Trigger As Range
Set Source = Range("A2:E2")
Set Destination = Range("A5")
Set Trigger = Range("F2")
If Intersect(Trigger, Target) Is Nothing Then Exit Sub
If Target.Count > 1 Then Exit Sub
If UCase(Target.Value) <> "YES" Then Exit Sub
Application.EnableEvents = False
    Trigger.ClearContents
    Source.Copy Destination
    Source.ClearContents
Application.EnableEvents = True
End Sub


Because it is worksheet code, it is very easy to install and automatic to use:

   1. right-click the tab name near the bottom of the Excel window
   2. select View Code - this brings up a VBE window
   3. paste the stuff in and close the VBE window

If you have any concerns, first try it on a trial worksheet.

If you save the workbook, the macro will be saved with it.
If you are using a version of Excel later then 2003, you must save
the file as .xlsm rather than .xlsx

To remove the macro:

   1. bring up the VBE windows as above
   2. clear the code out
   3. close the VBE window

To learn more about macros in general, see:

http://www.mvps.org/dmcritchie/excel/getstarted.htm

To learn more about Event Macros (worksheet code), see:

http://www.mvps.org/dmcritchie/excel/event.htm

Macros must be enabled for this to work!

Definition Update for Microsoft Security Essentials marked as OPTIONAL - Microsoft Community

miguel i. fonseca say:

Definition Update for Microsoft Security Essentials marked as OPTIONAL forces me to update manually

Why is it that since October 2012 every Definition for Microsoft Security Essentials is marked as OPTIONAL.  They used to be CRITICAL, so Windows Update updated them automatically.  Now I have to check them manually in order for Windows Update to install them.  Is it a change in the way Microsoft is releasing them or is there a setting I can change to make them install automatically?  I have always had Windows Update configured to "Download updates but let me choose whether to install them." and they used to be installed automatically, but it is not so anymore.  Thanks in advance,

M

rob koch say:

Actually, this change to making the MSE definitions updates optional occurred during the MSE 1.0 period as I recall, though I couldn't provide an exact date, though it was never ever considered a critical update..

 

The reason for this was people exactly like yourself who had this setting, since on most versions of the operating system this caused the Microsoft Update notification to prompt that an update was waiting to be installed and so masked the fact that this was only a definitions update and not an important security update.  This caused lots of irritation for those who watched for these notifications, so since changing this action would have required fundamantal changes to Microsoft Update itself, which the MSE developers don't control, the only real solution was to change the definitions updates to optional.

 

The other point here though is that this really doesn't matter, since the design of MSE has always contained the ability to dynamically download an updated signature for a newly discovered infection on the fly, originaly called the Dynamic Signature Service.  This means that even if you haven't downloaded definitions in a few days, if a recent piece of malware that isn't known (in the current definitions on your PC) is detected via behavioral or other monitoring, a signature of the unknown item is automatically uploaded to MAPS and if a new definition that matches the itrem is found it will be downloaded on the fly and applied.

 

So you are complaining about an old non-probblem that Microsoft designed out of existence when it originally released MSE.  Stop obsessing about the definitions updates, as MSE will take care of this for you and warn you if they become excessively out of date.  You should only be concerned if the normal daily updates don't occur at least every couple days the PC is used, which MSE will do itself.

 

Rob

stefan blom say:

The error you are describing suggests that something has happened to the hidden bookmark that Word adds to a cross-referenced item. It is very easy to redefine the range for the bookmark by mistake. This happens, for example, if you add content at the beginning of the bookmarked paragraph, which will cause the bookmark to expand and possibly include an unnumbered paragraph. The workaround is to avoid adding content at the beginning of a cross-referenced paragraph. Instead, add content, or create a new paragraph, at the end of the previous paragraph.
