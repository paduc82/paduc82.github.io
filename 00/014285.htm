a. user say:

I log into my machine with an account on domain A, but access a MOSS 2007
site that uses an account on domain B. Everytime I access the SP site I
get prompted for username and password (though it does remember the
username, but not password).

I've tried adding the site to Trusted Sites and Local Intranet, and I've
tried all the options in "User Authentication" but I can't get the

I don't know how the authentication on a MOSS site works. But if this is
HTTP authentication (e.g. not form-based), then IE will send your current
credentials (which you used to login to the machine) if the site is
identified as Local Intranet and you opted to use Windows Authentication and
automatically send your user credentials in Local Intranet. In this case the
wrong information gets sent. A way to workaround may be to disable doing
that (e.g. either remove the site from Local Intranet or disable the option
to automatically send these data in Local Intranet). It's possible that it
then reverts to the normal HTTP authentication behavior and allowsd you to
save the password. If that doesn't work or is not feasible your only other
option is probably to add your A account to MOSS as well. You may also want
to ask about this problem on a MOSS-related forum on MSDN.
