With Build 10162, I think we're absurdly close to the idea of a perfect Windows. It's reliable, bug free and I can mostly use it without thinking that I am running some kind of beta software.

On the UI aspect, since the start of Windows 10 development, some aspects have puzzled me. I'd like to start a discussion on them:

ACTION CENTER: Here is a strange withdrawal from what used to be a clean and borderless idea of iconography in Windows 8.1's right hand. For instance, let us consider the icons in the Charms > Settings on Windows 8 and their boxed counterparts in Windows 10. On the left hand side, is an example of clean and efficient design. The lettering is small but the visual/icons are large. There is also no sense of space anxiety since there are no boxes. To highlight activity, the visuals themselves change: Mute button has a 'x' in front of the speaker etc. In comparison, Windows 10 uses boxes which not only creates space anxiety but are also inefficient since both the lettering and the iconography are small. The use of box highlighting is also not a modern solution. It's just a suggestion that even if the grid has to be maintained, the box buttons are replaced by modern and responsive icons. The same could be applied to Windows on Mobiles!

CORTANA VERTICAL SPACING: This one I'm sure the team already has on mind, but from Windows Phone to Windows 10, it feels as if the team that designed Cortana's interface was reshuffled and somehow a track of spacing was lost. This is really really small, but I feel adds to the polish of the OS at large. The vertical spacing between Cortana's "O" and where it says "Hello Shamik" must be an error. Not only this, but the vertical spacing within its results must also be sorted out before RTM.

START SCREEN, POWER BUTTON AND USER PIC: Windows 10's Start is arguably the best tablet OS home screen. The vertical scrolling is a killer feature, and the animations are all good. Keep up the incredible work folks! Waiting for July 29th!

Using the footnotes settings dialogue box I can restart footnotes for a new section, but only at 1. Is there a way, mid-way through the document, to restart the footnote numbering at, say, 6 instead (i.e. skip out 1 to 5 for the new section). 

Thanks!

doug robbins - mvp office apps & services (word) say:

As long as the number at which you want to start is higher than the number of the last (highest numbered) footnote in the previous section, you can run a macro containing the commands

With ActiveDocument
    .Sections(2).Range.FootnoteOptions.StartingNumber = #

End With

where # is the difference between the number at which you want the footnotes to start in that Section and the number of the last footnote in the previous section

For example, running

With ActiveDocument
    .Sections(2).Range.FootnoteOptions.StartingNumber = 5
End With

will allow you to have

If there were two footnotes in Section 1, you would need to use

With ActiveDocument
    .Sections(2).Range.FootnoteOptions.StartingNumber = 4
End With
