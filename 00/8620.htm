Michael_531,
Not sure you need 9 calendars but maybe I don't fully understand your "crews".

Since Project only allows one work shift schedule per resource you tentatively could use 3 calendars. First, don't set up individual resource calendars but set up resource shift calendars.

The first calendar is set up for Mon-Wed nightshift work with a repeating pattern of every 4 weeks (i.e. week1 is "active" nightshift, week 2 is non-working, week 3 is non-working, and week 4 is active nightshift again). Repeat this for 3 years.

The second calendar is set up for Mon-Thurs dayshift work with a repeating pattern as above but starting week 2. The third calendar is the same except for Tue-Fri dayshift starting week 3.

On the Resource Sheet call the resources, "nightshift", "dayshift-1" and "dayshift-2". In the Group field indentify the crew (e.g. 1 through 3). If there are 5 guys in each crew, set the max units to 500%.

Okay, you can get what you see in your screenshot but it will require extra work since there are many different factors that need to be considered.

First of all, I guessing your Project start date is set for 4/14/14 8:00 am. That will effectively change the start time a 7-day 12h task. For example, even though your 7-day 12h custom calendar starts at 6:00 am, because of the project start date time, project will automatically start that task at 8:00 am. That's why your task 5 starts at 8:00 am even though it has a task calendar that starts at 6:00 am. You will either have to manually set the task start date, in which case Project will issue a warning that the task is starting before the project start date, or, you can set the project start date to an earlier date (e.g. 1/7/14).

Second, manually scheduled tasks take some getting used to. Regardless of the task calendar, the start time of a manually scheduled task will take on the default start time, which is 8:00 am. But, you can manually adjust the start time to be what you want/need.

Third, when working with multiple calendars of varying hours per day, one day in the duration field will not always be one day. So, even if you had task 5 starting at the desired 6:00 am, the finish time would be 2:00 pm for a duration of 1 day (remember a day is defined as 8 hours). But since task 5 starts at 8:00 am (see above), it finishes at 4:00 pm which is 8 hours later.

For illustrative purposes I set up your scenario using a Project Start Date of 4/13/14 6:00 am. My 7-day 12h calendar is actually only 11 working hours because I give my guys an hour for lunch (i.e. 6:00 am to 12:00 pm and 1:00 pm to 6:00 pm). I also left the hours per day at 8, hence task 5 must be 1.38d to achieve at work period from 6:00 am to 6:00 pm.

Bottom line. I wouldn't mix manually scheduled tasks and auto scheduled tasks. Use durations of hours instead of days or be prepared to accept the limitation of only one definition for hours per day.

Windows standard <CTL><Z> is undo, <CTL><Y> is Redo

The switch from WP to Word can be quite a culture shock.

Word is very different from WP. The WP I used to work with created files that were essentially a variation of HTML,  A WP document was a string of text with various formatting codes inserted. Word is not!  The Word DOC binary format was very much like a compiled program. The "new" (2007) DOCX format is an XML  text file that has simply been zipped. But even now, the XML is based on concepts from the old format.

For the most part Formatting information is not embedded in with the text. Instead is is defined in one location, and "linked" to.

So the whole document has formatting like headers/footers and margins defined. This information is "stored" in the very last paragraph mark at the end of the file which includes an implied section break. If you want to vary those setting you have to insert a "Section Break". There are several variations, but they all identify separate areas of document setup.

Text style is defined by the paragraph using the "Style" concept.  The document style info is stored in the "Paragraph Mark" at the end of each paragraph which has an implied section break.
