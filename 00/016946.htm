jllatham say:

This macro will let you select the cells to be copied and once you have them selected, run the macro and it will copy their values into a cell 7 columns to the right of the selected cells.  So if you select cells in column B, they'll be copied over into column H on the same rows.

Sub CopySelectedData()
  Selection.Copy
  Selection.Offset(0, 7).PasteSpecial Paste:=xlPasteValues
End Sub

If you need to move it over 7 columns but back UP 2 rows then:

Sub CopySelectedData()
  Selection.Copy
  Selection.Offset(-2, 7).PasteSpecial Paste:=xlPasteValues
End Sub

If you will just copy the 2 lines of code into your existing ctrl-m macro (replacing all the code within its Sub...End Sub statements with them, it'll still work with the ctrl-m key combination.

If you want to delete the original contents of the selected cells in column B, then use this:

Sub CopySelectedData()
  Selection.Copy
  Selection.Offset(0, 7).PasteSpecial Paste:=xlPasteValues
'or
'Selection.Offset(-2, 7).PasteSpecial Paste:=xlPasteValues
  Selection.ClearContents
End Sub

Is there a way to do this? Thank you in advance for any help.
jay freedman say:
That's a limitation of the printer, not of Word. Most printers require some space, often a quarter inch, for paper handling, and can't print any closer to the edge than that. Printers that are classed as "photo printers" may have an option in the Printer Properties dialog for "borderless" printing that lets them print to the edges.

If that doesn't work out, the usual alternative is to print on larger paper (if the printer can handle that size) and then trim off the edges. Otherwise, you need to find some other way to frame the title, maybe with a paragraph border.

a. user say:

Hello,

ActiveSync contact lists do not support the type of contact groups that Outlook 2013 is using, which is why you get the message.

It is possible to create a contact list which is stored locally on Your computer, but that list will not be synchronized to Outlook.com or to other Devices using the Outlook.com contact list. If you want to do this, you need to create a New data file for storing everything: File, Account settings, Account settings..., Data files, Add..., type the name of Your New data file and choose Ok. You will now get a New contact list called Contacts (this computer only). (you will also get a local calendar and a local store for emails, but you don't need to use them).

I think that in Outlook.com, you can change Your reply-to address. I'm unsure if this completely masks the originating address though. I'm going to test this, and edit my post with the result.

Edit: if you add and confirm an alias to Your Microsoft account, you can send emails from Outlook.com using the alias as From address. In this case, the other address is completely masked.

I tried changing the From address in Outlook 2013, but that didn't work at all. It may work differently on an Exchange ActiveSync connection, which you have. Unfortunately I'm limited to using IMAP for connecting to Outlook.com by the Company firewall, so I cannot test this further.
