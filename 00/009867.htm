Hi,

Practically, it is important to know that circular relationships nearly always involve links to or from summary tasks. It thus is much faster to remove these links (maybe one by one) and if necessary re-insert them till you get the message "No calculations can be performed right now..."

Also, the indication that "there is a circular relationship INVOLVING task 729" is generally not helpful at all.

john - project say:

Tim,

It sounds like Jan hit the nail on the head. If you had (have) links on summary tasks, using the cut and paste approach is going to be self-defeating and that's why it didn't work. Too bad, it's a slick approach.

If you haven't already turned off automatic calculation, do so. Then take a close look at what each of the summary line links was attempting to accomplish. If you restructure the existing performance tasks to meet the intent of the summary links then you will get to where you need to be - free of circular relationships. cliff gu msft say:

Hi Richard,

 

Generally, to import an Outlook calendar in SharePoint Online, you may follow the following steps.

1. Create a calendar app in SharePoint Online.

2. Open Outlook 2016>create a new folder>select the calendar you want to import>copy the calendar to the new folder.

3. Select the folder>File>Open and Export>Import/Export>Export to a file>export the calendar to an Excel file.

4. Select the calendar app in SharePoint Online>Calendar>Current View>All Events.

5. Open the exported file and make the columns be the same as the calendar app‚Äôs columns.

6. Click edit in All Events and it will be showed as the Datasheet View.

7. Copy and paste the data in the Excel file to the All Events list.

 

Thanks,

Cliff
