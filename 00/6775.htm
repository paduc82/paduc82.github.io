argmarovski say:

Those error codes appeared first with the Insider build 10162. They affect only upgrades, but clean installs are OK. The errors might have been reported in earlier builds, but 10162 caused massive headaches during the successive build upgrade, so that is when I noticed them. All the error codes are mostly driver related, except for one, which is due to some UEFI violation (the last in my list below). I also noticed that the same error code is caused by different driver in each system. My summary from my tests & other user posts is:

0xC1900101 - 0x20004:

Wi-Fi, LAN; Bluetooth; SATA cable; NIC driver; Network bridging (including USB devices like Xbox controller driver); USB mouse or keyboard driver;

0xc1900101-0x2000c
Cryptographic Service, BITS(Background Intelligent Transfer Service) & Windows Update Service are not set to Automatic under services.msc;

0x900101 - 0x40017
Uninstall Net Framework 3.5;

0xC1900101 - 0x20017

SSD & HDD misconfiguration/UEFI violation (Windows is on SSD, recovery is on HDD)-disconnect the HDD, install, reconnect HDD.

There surely are other error codes as well, but I followed the progress of those only.

Hi Jacques,

The latest Office 2011 suite version is 14.7.3, but the notification that you're seeing is for the AutoUpdate program itself, which is versioned differently. When you see this update notification message, look at the About box (in the AutoUpdate pull-down menu in the top-left of the screen). AutoUpdate is a component which lives in /Library/Application Support/Microsoft/MAU2.0 on disk, and it's a program that updates Office 2011, 2016, Lync and Skype for Business products.
