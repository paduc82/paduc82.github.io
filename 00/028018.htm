ken sheridan say: As far as I can tell from your post you are attempting to restrict the second subform to the current person in the parent form and the music group selected in the first subform.   There are two methods which can be employed to correlate subforms:

1.  Add  a hidden unbound text box to the parent form which references the key of the first subform.  In your case this would be the value of the combo box in the first subform, referenced with the following syntax:

=[SubFormControl].Form![ComboBoxName]

Note that SubFormControl is the name of the control in the parent form's Controls collection which houses the subform; what you have referred to as a 'frame'.

Include the name of the hidden text box as part of the LinkMasterFields property of the second subform.  So, if for example the second subform is currently linked to the parent form by its LinkMasterFields and LinkChildFields properties being PersonID, and the second subform's recordset also includes a column MusicGroupID, which is also the bound column of the combo box in the first subform, the LinkMasterFields property of the second subform would become:

PersonID;txtMusicGroupIDHidden;

where txtMusicGroupIDHidden is the name of the hidden text box in the parent form, and the LinkChildFields property of the second subform would become:

PersonID;MusicGroupID

2.  The above method, although having the advantage of being completely code free, can be a little slow in my experience, however, and I've generally found a more efficient method to be to base the second subform on a query which references the key of the first subform, the combo box in your case, as a parameter, and then to requery the second subforms module.  Normally this is done in the Current event procedure of the first form as correlated subform's are generally both in continuous forms view.  In your case, though, the AfterUpdate event procedure of the combo box in the first subform would be appropriate.

You'll find an example of this second method of subform correlation as StudentLog.zip in my public databases folder at:

Either insert a filename field in the header/footer
or
create and autotext/building block entry containing the filename field

Use a macro to insert the filename at the end of the document (or in the

Sub InsertFileNameAtEnd()
With ActiveDocument
    If Len(.Path) = 0 Then .Save
    If InStr(1, .Range.Paragraphs.Last.Range.Text, _
        ".doc") = False Then
        .Range.InsertAfter vbCr & .FullName
        With .Range.Paragraphs.Last.Range
            .Paragraphs.Alignment = _
                wdAlignParagraphRight
            .Font.name = "Arial"
            .Font.Size = 8
        End With
    Else
        .Range.Paragraphs.Last.Range.Text = _
        .FullName
    End If
End With
End Sub

or

Sub InsertfNameAndPath()
With ActiveDocument
  If Len(.Path) = 0 Then .Save
  Selection.TypeText Left(.FullName, InStrRev(.FullName, ".") - 1)
End With
End Sub
