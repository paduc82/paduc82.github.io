How to move pinned and recent files / places from excel 2010 to - Microsoft Community

sophiemiron say:

How to move pinned and recent files / places from excel 2010 to excel 2013 when upgrading to office365

This is a question with a solution although I'd be curious to know if there is an easier way to do this through the application itself.

I recently upgraded from office 2010 to office365 and installed office 2013 on my laptop.  Problem was that when I opened up the new Excel, I expected to see all my recent workbooks and recent places, particularly my pinned locations.  None of them were there.   So here's what I did:

1)  Opened up the registry using regedit. 

Office 2010 is under HKEY_CURRENT_USER\Software\Microsoft\Office\14.0\Excel while Office 2013 is under HKEY_CURRENT_USER\Software\Microsoft\Office\15.0\Excel .  Recent files are located as Items in the File MRU key while recent places are locate as Items in the Place MRU key.  (Pinned items were marked as such as part of the key with the string "[F00000001]"

2) I noticed that while the File MRU key had been copied from 14 to 15, there was a new key, HKEY_CURRENT_USER\Software\Microsoft\Office\15.0\Excel\User MRU\AD_2BBD9386C0F9C395F8E2BCB7FD38F4EDFFA6C27E8658AA143745B2871DF59037\File MRU and that this key was where excel was looking for the list.  (I suspect the extra layer of hierarchy has to do with office365 managing multiple file locations across computer and devices).  Also, Place MRU was located under the user key but did not exist at the top level.

3) I exported the relevant keys and made a backup of the office 13 registry keys. 

4) I edited the 15.0 registry file I had just exported and copied the Items from the 14.0 key (using a file compare/merge utility) to the 15.0 key

5) I clicked on  the new File MRU and Place MRU registry files to import the new keys. (note that this overwrites the existing key)

When I re-opened excel all my pinned files showed up under Recent, and when I clicked on "other workbooks", my other files and recent locations were there as well.  Voila.  I'm sure this would work for Word and any other office applications.

Access 2013 totals query

Hello,

I am working on building a inventory transactions database with two main tables, one for each inventory item and one for each transaction. As I understand it current stock should be a calculated field and not stored in the inventory table, but I can't figure out how to get it to total up correctly. I have a "Current Stock" query I'm building from the transactions table which has three fields "Product" which is a unique identifier for inventory items, "Quantity" which stores how much of the item for that transaction and "TransactionType" which will be one of three selections "additions" which should be added up for current stock, "sales" which should be subtracted from current stock, and "pending orders" which should not be considered for the current stock.

currently I have the following SQL code for the query

SELECT DISTINCTROW Transactions.Product, Transactions.TransactionType, Sum(Transactions.Quantity) AS [Sum Of Quantity]
FROM Transactions
GROUP BY Transactions.Product, Transactions.TransactionType;

but using this code it just totals up each transaction regardless of type. Does anybody have any suggestions for modifying the above code or making a whole new set of code so that the sum will properly reflect current stock?

ken sheridan say: You need to compute the sum of all additions per product less the sum of all sales per product, which you can do by multiplying the latter by -1 to make the number negative:

SELECT Product,
SUM(Quantity * SWITCH(TransactionType = "Additions",1,TransactionType = "Sales",-1,TRUE, 0))
AS CurrentStock
FROM Transactions
GROUP BY Product;

However, you should also take account of adjustments through stocktakes, using the latest stocktake quantity per product as an 'addition' and disregarding all prior additions and sales.  Non-sale disposals such as write-offs also need to be accounted for.  You'll find an example as Inventory.zip in my public databases folder at:

https://onedrive.live.com/?cid=44CC60D7FEA42912&id=44CC60D7FEA42912!169

If you have difficulty opening the link copy its text (NB, not the link location) and paste it into your browser's address bar.

In this little demo file separate tables are used for acquisitions, non-sale disposals and stocktakes, as well as for sales, which are recorded in an order details table.  The demo does not differentiate between fulfilled and pending orders, however.  To compute the current stock in hand values the data from the tables is first brought together in a UNION ALL query.  I should stress, however, that this file is only a demo of the basic methodologies for inventory control.  In reality a fully developed inventory system for all but the smallest of enterprises would be rather more complex.
