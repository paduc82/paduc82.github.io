daniel g. grau say:
I have no idea why the corruption occurred. The ruler bar has approx. minus one (-1) inch at the left edge of paper. Therefore I cannot get the text near the left edge of the page. 

I have no idea either. Word should not be creating a faulty template from scratch. A new Normal.dotm should have its margins set to zero.


I trashed the normal.dotm, and emptied trash. Restarted, reopened Word. No new dotm. Open blank documents. They have the same corruption. Still no dotm in the Templates folder. Closed blank doc without saving. Still no dotm. Closed Word. Now there IS a normal.dotm.

That's the usual behavior. The new Normal.dotm is created upon quitting.


So, how do I move the ruler?
Do this:
  • Let Word create its faulty Normal.dotm
  • From within Word, do File > Open, navigate to the User Templates folder and open Normal.dotm (you must open the template this way in order to edit it, and not by double clicking or dropping it on Word's icon)
  • Select View > Draft
  • Press Shift, then click on the horizontal left-scroll arrow at the bottom of the document's window
  • Move the ruler's left margin to the right
  • Close the template and save if prompted
It is possible that this won't fix your problem, which would mean that somehow Word is still picking up another Normal.dotm as a document model from somewhere else. Offhand, I can't say wherefrom.

Why does Word create a new normal.dotm that is not new?

Probably there's been some corruption in one of Word's internal files. There are several preference files in addition to the oft-mentioned com.Microsoft.Word.plist.


Why does Word update normal.dotm  when Word is closed?

If Word were to create a new Normal.dotm right away, this would prevent you from doing a reversible Safe Start to temporarily bypass your regular Normal.dotm, as this file would get erased immediately.
