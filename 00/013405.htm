How to hide the drop down Arrow in a Combobox - Microsoft Community

glendaestrada say:

How to hide the drop down Arrow in a Combobox

Hi,

I created a search form that displays records that contain fields that are comboboxes.

Since this form is only to show the information of the records, and will not allow any edits, I don't need that it shows the drop down arrows of the comboboxes.

Is there a way to make this drop down arrows invisible by changing a value in the property sheet?

Thank you very much for any help on this.

Kind Regards,  GE

ken sheridan say:

Is there a way to make this drop down arrows invisible by changing a value in the property sheet?

No.  Three possible approaches:

1.  Carefully place an empty label control over each arrow, setting its BackColor property to match the section's BackColor, and its BackStyle property to Normal. Set the Locked property of each combo box to True and its Enabled property to False to make them read-only.

2.  Hide the combo boxes by setting their Visible property to False, and add a set of unbound text boxes, one corresponding to each combo box and reference the relevant column of the combo box in the text box's ControlSource property.  So, if the combo box's RowSource is an SQL statement whose first column is hidden, and whose second column is shown, e.g.

    SELECT EmployeeID, FirstName & " " & LastName
    FROM Employees
    ORDER BY LastName, FirstName;

the ControlSource property of the text box would be like this:

    =cboEmployee.Column(1)

The Column property is zero-based, so Column(1) is the second column.

3.  Design the form as you would a report by including the referenced tables in its RecordSource query and bind text boxes to the relevant columns from the referenced tables.  With the above example the Employees table would be included in the query, joined to the form's current referencing table on EmployeeID and a text box would have a ControlSource property of:

    = [FirstName] & " " & [LastName]

Set the Locked property of each text box to True and its Enabled property to False to make them read-only.

Option 3 above would be my favoured solution.

scottgem say:

This is probably occurring because you are creating lookup fields on the table level. This is not recommended for desktop apps. 

Your form should be bound to a query that joins your main table with the Lookup tables that are linked to your lookup fields. Then don't include the lookup fields on your form, rather pull the text description fields from the lookup table.

I suspect that you are referring to my posts regarding the full scans...

And, yes, there's something wrong with MSE, or more specifically, the engine/definitions used by Microsoft security products. You didn't mention an error on crash, but the following text is from my reply for the crash of a scan where it presents the 800106ba error and a failure of the service to restart.

A recent post by mchampfl on page 11 of this long running thread - http://answers.microsoft.com/en-us/protect/forum/mse-protect_scanning/microsoft-security-essentials-error-code/d8d69ac3-ca4a-40ae-9c0d-15c97e64cea3?page=11&tm=1441106019496 - suggests that MSE is having problems with large compressed/zipped archive files due to the fact that MSE needs to open these in memory to scan them.  The crash, resulting in 0x800106ba can be avoided, per that reply, by disabling the scanning of compressed archives in MSE settings.

And, regarding the full scan comment:

As long as real time protection is enabled, a full scan on a frequent basis is usually overkill. A quick scan looks for malware in the most common locations. A full scan has to touch every file. Malware found in a full scan, not in a quick scan is typically not active. Personally, I only ever run a full scan if an infection has been detected or if one is suspected.  I use a different scanner for the full scan (the malware was missed when it arrived, so why should I expect it to find it on a full scan?)

The following scans are good for a second opinion:

Try Malwarebytes Antimalware (free version, do not install the trial or Pro version) - http://www.malwarebytes.org/products/malwarebytes_free/

Try SuperAntispyware

Try the ESET free online scan:  http://www.eset.com/us/online-scanner/
