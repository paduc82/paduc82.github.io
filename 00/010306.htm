dale howard [mvp] say:

Wendi --

You do not need to create a custom column with a formula and stoplight indicators to find late tasks.  In Microsoft Project, a late task is any task where the progress on the task is behind the current date (or the Status Date if you have specified this).  So, in your case, a task that was supposed to be finished last week, but is not yet completed, would be considered a late task by the software.  There are two default fields that you can use to solve your problem.  They are the Status column (which displays words like Late) and the Status Indicator columns (which displays an indicator to show that a task is late).  I would recommend you give these two columns a try and see if they will meet your needs.  Hope this helps.
