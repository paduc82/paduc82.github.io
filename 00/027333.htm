graham mayor say:
'Page' is a vague concept in Word, which is not a page layout application. The text is flowed to the margins and graphics, headers and footers etc are in different 'layers' of the document and as such cannot be processed together with the underlying pages.

If the document is simply text and you format the first paragraph on each 'page' with the Heading 1 style, then switch to Outline view and collapse the vie to show only heading one. You can drag the formatted paragraphs around to the required order and the text that accompanies them on the 'pages' will move with them. When you are happy with the new order, switch back to print layout view and reformat the paragraphs back to their original styles.

So for the benefit of all others with this issue.....

Step 1> Set up an alias *** Email address is removed for privacy *** (if you don't already have one). Note that if *** Email address is removed for privacy *** is not already in use, you can use that, to make it almost the same as your hotmail address.

Step 2> Microsoft apparently sets up another alias when you go into your account, which is just the *** part of your hotmail address.

Step 3> In Thunderbird Account Settings | Default Identity | Email address enter your hotmail address *** Email address is removed for privacy ***

Step 4> In Thunderbird Account Settings | Server Settings (for hotmail account) enter User Name *** Email address is removed for privacy *** (or *** Email address is removed for privacy *** if you made it similar to the hotmail address)

Step 5> In Thunderbird Account Settings | Outgoing server (SMTP)  change the User Name for Server smtp-mail.outlook.com to *** to match the alias mentioned in Step 2

Voila! Thunderbird now behaves exactly the same as it used to for your hotmail emails (receiving and sending)!

JP Ronse wrote:

The line feed is already in your text file. [....] I am afraid there is not very much we can do.

Well, it is not difficult to write a VBA procedure that replaces all LF characters with spaces.

Download "fileReplaceLF.bas" (click here) [1] and import it into an instance of Excel.  If the original path name is c:\...\foobar.txt, the VBA procedure creates a modified copy with the new path name c:\...\foobar NEW.txt.

It would be better if "foobar NEW.txt" retained the LF characters, and the VBA procedure "simply" added double-quotes around the tab-separated text, if necessary.  Actually, it is not so simple; but it is doable.

However, it is not worth the effort because the Excel Text Import Wizard [2] violates the principle of idempotence:  it does not interpret cells of that form correctly, even though that is how cells with LFs are output when Excel creates a tab-separated text file.

Alternatively, it would be possible (but more difficult) to rejoin lines in a copy of the CSV file, adding double-quotes around the joined text.

Excel does interpret that form as intended when we open the CSV file directly using Excel.  (The Text Import Wizard still misinterprets it.  Sigh.)

But the implementation would need to be specific to the form and/or content of the example CSV file.  It would have to be carefully customized for other errant CSV file.  So again, it does not seem worth the effort.

What is going on is that some spammer purchased a list of addresses that included yours and that the spammer is using your address as the sender address for the junk.  This in no way proves that your account is being used to send the messages.  Most people at one time or another have their addresses spoofed.  It's a fact of Internet life.

Spammers change the sending address frequently so my opinion is that the non-delivery reports shouldn't last too long.  You could make a rule to handle them, but be careful of detecting real ones.
