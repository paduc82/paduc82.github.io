All you need to do is open the query designer, and then switch to SQL view from the ribbon.  Then paste in the first SQL statement Duane gave you in place of what's there.  Save the query under the name Duane gave you.  Then repeat this for the second query.  Open the second query to get the results you are looking for.

The basis of the queries is quite simple:

1.  The first query simply returns all rows where the CAS3 value is what you enter.  One column is returned, the DTE column, and is given the alias TargetDates.  This is to distinguish it from the DTE column from the table in the second query below.

2.  The second query draws on the original table and the query which you created in 1 above.  However, it does not specifically join them.   The result of this is that it returns what is known as the Cartesian product of the table and the first query, which means that, in effect every row in the table is joined to every row returned by the first query.  To restrict the rows from this set to those you want, the WHERE clause restricts them to those from the table where the date is one day before, the same as, or one day after each of the dates returned by the first query.

I'm curious as to why you are doing this.  If it is to help predict a specific number occurring, then I'm afraid that there is no mathematical basis for this whatsoever.  Even if a number where to occur 99 days in succession, the mathematical probability of it occurring on day 100 is exactly the same as would be the case in any other situation.  This can be seen by tossing a coin 100 times.  As the number of tosses increases the number of heads and tails will progressively become closer to 50% heads, 50% tails.  If you have the stamina to keep going until the coin has been tossed 1000 times it will get closer still, and so on, but the result of each toss is every time completely random.

hansv mvp say:
Let's say the data are in A1:C1000.
Select this range; I will assume that the active cell within the selection is in row 1 (probably A1).
On the Home tab of the ribbon, click Conditional Formatting > New Rule...
Select 'Use a formula to determine which cells to format'.
Enter the formula

=COUNTIFS($A$1:$A$1000, $A1, $B$1:$B$1000, $B1, $C$1:$C$1000, $C1)>1

Click Format...
Specify the desired formatting.
Click OK twice.

Syntax error (missing operator) in Access 2007 - Microsoft Community

menjomoon say:

Syntax error (missing operator) in Access 2007

Hello,

I'm new to microsoft access and I'm trying to create age groups in a database containing patient data. The data table is called 'Sheet1'. Each patient is identified by a Patient-ID, which can show up several times in the database, depending on the amount of visits. 

So far, I've done a query (Sheet1 Query) extracting each unique patient, with corresponding age (Leeftijd) and gender (Geslacht). I'm trying to create the different age groups from this query, using this: 

SELECT SUM (CASE WHEN [Leeftijd] < 18 THEN 1 ELSE 0 END) AS [Under 18]

             SUM (CASE WHEN [Leeftijd] BETWEEN 18 AND 35 THEN 1 ELSE 0 END) AS [18-35]

FROM [Sheet1 Query]

Now, I keep running into "Syntax error (missing operator) in query expression 'SUM (CASE WHEN [Leeftijd]< 18 THEN 1 ELSE 0 END)'.

Any help is appreciated,

Menjo

ken sheridan say: Instead of the standard SQL CASE expression call the VBA IIF function:

SELECT SUM(IIF([Leeftijd] < 18,1,0)) AS [Under 18],
             SUM(IIF([Leeftijd] BETWEEN 18 AND 35,1,0)) AS [18-35]
FROM [Sheet1 Query];

Is the Leeftijd colum a computed column retuned by the Sheet1 Query query, or is it a column in a base table?  If the latter the values will of course become incorrect over time unless the age is intended to be that at a specific point in time rather than the current age.  The latter is best returned by computing the age on the basis of the person's date of birth, using a function such as that below:

Public Function GetAge(varDoB As Variant, Optional varAgeAt As Variant) As Variant

    If IsMissing(varAgeAt) Then varAgeAt = VBA.Date
    
    GetAge = DateDiff("yyyy", varDoB, varAgeAt) - _
        IIf(Format(varAgeAt, "mmdd") < Format(varDoB, "mmdd"), 1, 0)

End Function

By default this returns the age in years at the current date, but can return the age at a specific date by passing the latter into the function as the optional varAgeAt argument.
