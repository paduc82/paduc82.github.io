Hello WRKeen

From what you're describing is Outlook (desktop) part of the Office suite which is an <email client> and an Outlook.com account accessed via webmail (your browser).

Outlook.com is an <email account>. It is not part of Windows 10 but is an account that automatically gets created for you when you create a Microsoft account.

To answer your question, all you need to do is to add the Outlook.com account to the Outlook desktop program which will default being created as an <Exchange> account where everything will get bidirectionally sync'd (i.e. what you send via Outlook desktop will appear when you log into your Outlook.com account and vice versa. Applies equally to such things as contacts & calendars.

To add the account to Outlook desktop, go to
File --> click on the + sign for <Add Account> and enter your email address and password. Done

<IF> this involves a non-Outlook.com account, then the account will be configured as an IMAP account by default where only the email folders will be sync'd between the server and Outlook desktop program.

suzanne s. barnhill say:

If you're not actually using labels, it would make more sense to use a Directory-type merge, which will add records straight down the page. If you want them in newspaper-style columns, you can create columns in the mail merge main document or in the finished merged result.

To keep the address blocks discrete, use a line break (Shift+Enter) at the end of each line in the address block and a paragraph break (Enter) only at the end of each block. Add Spacing Above/Below to separate the blocks.
