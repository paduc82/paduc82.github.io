hansv mvp say:

Or use the FileDialog object in the following function:

 

Function GetSaveAsFileName(Optional InitialName As String) As String
    With Application.FileDialog(2) ' msoFileDialogSaveAs
        .InitialFileName = InitialName
        If .Show Then
            GetSaveAsFileName = .SelectedItems(1)
        End If
    End With
End Function

 

You can use it like this:

 

    Dim strFile As String
    strFile = GetSaveAsFileName("MyTestFile")
    If strFile = "" Then
        MsgBox "No filename specified"
    Else
        ' Do something with strFile
    End If

barb way say:

You should not have to customize the ribbon in Visio Pro 2010 to show theData tab, it is visible by default.  There is no Data tab in the Standard version, even using the Customize Ribbon functions.

You can confirm that the version you are using is Pro (check in the File tab by clicking onHelp), and you will see the version near the top of the right hand column.  For Pro the name should be displayed as "Microsoft Visio Premium 2010" (in English versions).

If you verify that you have Pro, then the issue may be that the customize screen isn't showing all of the available tabs. Check under the "Customize the Ribbon" section, and make sure that the drop-down selector is showing "All Tabs" or "Main Tabs".  The Data tab group is part of the Main tabs, and should be located between Design and Review.

Finally, if all else fails, you can try to reset all customizations.  However - be aware that there is no Undo action for this, so you might want to Export your existing changes (using the Import/Export option) first!  Then use Reset / Reset all Customizations, and see if all of the expected tabs show up for customization.

karl timmermans [outlook mvp 2012-2018] say:

A friendly suggestion, just for the sake of clarity - the correct term is "when trying to <import>......." versus <insert> (may seem a trivial item but consistent terminology goes a long way when trouble-shooting)

That said - based on the your message "The file.................is locked" - likely causes are the following:

1) You copied the file from an external source and the "read-only" attribute has not been cleared. Right click on the file name --> properties --> Clear the <Read Only> box on the screen that appears

2) You have the CSV file open in another program - such as Excel. This could also occur if you have an "orphan" Excel task that has not closed that was using the CSV file even though it otherwise appears closed.

A quick brute force way to resolve issue #2 (if that is indeed the problem), reboot your machine and import the file again. Re-booting will release any potential "orphan file locks" that may exist if there is no obvious program that has the CSV file open.

In short - any message advising that a file "is locked" means that nothing else can read it - not an Outlook issue.

Your help is greatly appreciated!

trevor rabey say:

When you assign a deadline to a task, and if the scheduled finish is later than the deadline, MSP calculates the total slack for that task and its predecessors as the difference between the scheduled finish and the deadline, as you can see at task 14. The total slack is negative because the scheduled finish is later than the deadline. Your task 18 has a deadline which is much later (2016) than the scheduled finish (2015), which is good. What the total slack on 2 - 14 is trying to tell you is how much earlier those tasks have to start in order for 14 to come in earlier than the deadline. It would be sufficient for task 1 to start 0.2 months earlier.

I think you might be trying to schedule backwards from some externally imposed dates.

This is not critical path method.

Try this. In Options, Schedule find the one which says calculate multiple critical paths and check that.

Remove the deadline from 14 and see what happens.

I would not be working in weeks and months. Days are ambiguous enough without making it any worse.
