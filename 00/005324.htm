Save each page within a Word document as a separate file? - Microsoft Community

akmms say:

Save each page within a Word document as a separate file?

Is there any quick way to save each page within a Word document as a separate file?

Also, a way to choose the format of each new file so that it is Word 97-03 since I am using Word 2007?

yves dhondt say:

You can use the following macro. For more info on installing macros, seehttp://www.gmayor.com/installing_macro.htm

Note: the line fn = "Page" + CStr(idx) + ".doc" contains the name of the files. You can change this to your liking. For example, if you wanted them stored in c:\temp, you would write something like:  fn = "c:\temp\Page" + CStr(idx) + ".doc"

=====================================

Sub SavePagesAsDoc()

    Dim orig As Document
    Dim page As Document
    Dim numPages As Integer
    Dim idx As Integer
    Dim fn As String
   
      
    ' Keep a reference to the current document.
    Set orig = ActiveDocument
   
    ' Calculate the number of pages
    numPages = ActiveDocument.Range.Information(wdActiveEndPageNumber)
   
    For idx = 1 To numPages
        ' Make sure the document is active
        orig.Activate
               
        ' Go to the page with index idx
        Selection.GoTo What:=wdGoToPage, Name:=idx
   
        ' Select the current page
        Selection.GoTo What:=wdGoToBookmark, Name:="\page"
               
        ' Copy the selection
        Selection.Copy
       
        ' Create a new document
        Set page = Documents.Add
       
        ' Activate it
        page.Activate
               
        ' Paste the selection
        Selection.Paste
       
        ' Generate the file name
        fn = "Page" + CStr(idx) + ".doc"
       
        ' Save the document as Word 97-2003
        page.SaveAs FileName:=fn, FileFormat:=wdFormatDocument, AddToRecentFiles:=False
       
        ' Close the document
        page.Close
       
    Next

End Sub

OK, first 'seed' B4:M4 with the current names of the worksheets. These should start out exactly as they are in the workbook so copying and pasting them is not out of the question. Next right-click the name tab of the Cost worksheet and choose View Code. When the VBE opens, paste this into the pane titled something like Book1 - Sheet1 (Code),

Private Sub Worksheet_Change(ByVal Target As Range)
    If Target.Count > 1 Then Exit Sub
    If Not Intersect(Target, Range("B4:M4")) Is Nothing Then
        On Error GoTo FallThrough
        Application.EnableEvents = False
        Dim oldWS As String, newWS As String, w As Long
        newWS = Target.Text
        Application.Undo
        oldWS = Target.Text
        Target = newWS
        For w = 1 To Sheets.Count
            If LCase(Sheets(w).Name) = LCase(oldWS) Then
                Sheets(w).Name = newWS
                Exit For
            End If
        Next w
    End If
FallThrough:
    Application.EnableEvents = True
End Sub

Tap Alt+Q to return to your worksheet. If you change the worksheet names in any of the B4:M4 cells then the corresponding worksheet's name will change. I've used .Text and not .Value so if one of the cells was formatted as dd-mmm-yyyy and you input 8/7/14 then the worksheet's name and the cell would be 08-Aug-2014.

I've assumed that you know that there are some characters you cannot use in a worksheet's name. There is no safety if you try a name with an illegal character.

  • The name cannot be over 31 characters
  • The name cannot contain any of the following characters: / \ * ? [ ]

I was having the same issue - if someone within my company attached a calendar invite to an email rather than sending it as an invite, I could not open it. 

I discovered that the file extension .ics was set to the default program of "Notepad" for some reason.  It's an easy fix. (I'm on Windows 7/Outlook 2010, for reference)

Go into Control Panel, Default Programs, File Associations/Set Associations.

Scroll down to the extension .ics & highlight

Click "Change Program" and select Outlook.

This resolved it for me.
