Okay, well I believe I have a temporary fix for this, but I'm hoping someone will come up with a more definitive solution.  

I found that there were two drivers causing the problem for my Surface.  To diagnose this, run a software you have installed called "sleepstudy."  The instructions to do this can be found here:  http://www.windowscentral.com/learn-why-battery-drains-surface-pro-3.  It is run from the command prompt, but make sure that you run it as an administrator.  The file saves as an html file that any browser will open.  

I found two issues when I ran this report.  The first was a realtek card reader error.  I have a micro-SD card installed for music storage.  When I tried to use Microsoft's "Safely Remove Hardware" feature it said that the device was busy, so I simply shut off the Surface, pulled out the card, rebooted, then reinstalled the card.  This solved that problem.

However, I was also showing something else continuously running in my report.  This was the Marvell AVASTAR wireless AC network controller.  I didn't want to delete the wifi adapter and reinstall so I went to "all settings", "network and internet", "manage wi-fi".  Here I toggled off both switches, "connect to suggested open hotspots," and "connect to networks shared by my contacts."  I then put it in airplane mode when I put it to sleep, and this solved the problem.  In sleep mode for 5 hours the new report showed the driver active for just a few minutes, then it went completely idle.  I consumed about 2% power in those 5 hours.

I will try to put it assleep again today without the airplane mode turned on and see if this solves the issue.  I believe the driver was just continually attempting to search for networks while the Surface was asleep.  Hopefully, there will be a firmware update to address this issue.  One other thing, previous to this, I had tried turning on "quiet hours" and shutting of "location" services without success.   This was the only way it would stop the driver. 

Hope this helps everyone with this issue.

Please take into consideration that while you call one of your Windows 7 computers Serve,r it is still a client OS ,and as such do not provide some of the service as you would  expect from a real server.

The Firewall and ICS are interrelated leave the Firewall Default On.

Your system seems to be double vulnerability. I.e., PPPOE and multiple computers on ICS.
