a. user say:

If you are still looking for an answer to this question, the most likely reason is that the way that Word connects, by default, to Access databases changed from "DDE" (in Word 2000) to "OLE DB". OLE DB was not available in Word 2000, but you may have used ODBC for some connections.

Existing connections that use DDE will probably still work, but whenever you make a new connection, Word will use OLE DB, unless you check Word File->Options->Advanced->general->Confirm file format conversion on open.

DDE works by starting Access (if necessary) then getting Access to open the database (if necessary), and DDE can connect to most tables and queries in the database that you have permission (via your logon) to access. There are various types of query or table that OLE DB cannot "see", e.g.
 a. tables/queries that you don't have permission to see
 b. parameter queries
 c. queries that use a number of Access VBA functions, including replace and most/all of the financial series functions.

In addition, wildcard queries will not return any records unless you wrote the queries using the wildcards that OLE DB expects (_ and %) rather than the traditional Jet ones (? and *). There are a couple of ways around that, but if you choose to use OLE DB, the safest option is probably to duplicate your queries and use the OLE DB wildcards in the copies.

Dim strRoot As String
Dim FD As FileDialog
