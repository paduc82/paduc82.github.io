vasko oreely say:

Solved it:

In IIS you have to put Negotiate in front of NTLM for Kerberos to work. Autodiscover does offer both NTLM and Negotiate providers but Outlook for some reason does not pick the "best" possible auth provider but just the first one.

lars-Åke aspelin say:

Try this macro, it has to be included in the worksheet where you want this behaviour:

Private Sub Worksheet_Change(ByVal Target As Range)
   If Not Intersect(Target, Range("E1:F20")) Is Nothing and Target.Value > 1 Then
     Application.EnableEvents = False
     Target.Value = Int(Target.Value / 100) / 24 + (Target.Value - 100 * Int(Target.Value / 100)) / 1440
     Application.EnableEvents = True
   End If
End Sub

Change the range, E1:F20, to the range where this input should be possible. The cells in the range should be formatted as hh:mm, i.e military time.

    How to Prevent Tabs from being Accessed in Excel - Microsoft Community

    deanna68 say:

    How to Prevent Tabs from being Accessed in Excel

    I have an Excel workbook created for my staff to use, with numerous tabs at the bottom of the workbook. Each tab contains a worksheet in which the employee has to enter some data.  I do not want the employee to be able to move on to the worksheet in the NEXT tab in sequence until the required information is completed in the current tab.  How do I prevent a tab from being accessed like that?  The cell in the worksheet (D35) must contain data before moving on to the next worksheet in the next tab at the bottom of the page. 
    ossiemac say:

    A VBA code solution. The code is designed to prevent the user from selecting any worksheet if the previous worksheet des not have D35 populated. This prevents the user going 2 or 3 (or more) sheets ahead.

    To install the code:

    1. Alt and F11 to open the VBA editor.
    2. Double click ThisWorkbook in the Project Explorer (Left column)
    3. Copy the code below and paste into the VBA editor
    4. Close the VBA editor. (Cross at very top right of window. Do not select the cross below it because that only closes the module; not the VBA editor).
    5. Save the workbook as Macro enabled.
    6. Ensure that macros are enabled in Options. See help for how to do this. (The Option to "Disable all with notification" should be OK).
    7. Now when the user attempts to change the worksheet it tests the prior worksheet for D35 being populated. If not populated then it loops backwards from the sheet until it finds D35 populated in a worksheet and then selects the next worksheet.
    8. As per the comments in the code the user is prevented from selecting any worksheet where the previous worksheet does not have D35 populated.

    Private Sub Workbook_SheetActivate(ByVal Sh As Object)
        Dim s As Long
        If Sh.Index = 1 Then Exit Sub  'No worksheest prior to first sheet
        'Next line goes to End Sub if prev sheet D35 is populated
        If Worksheets(Sh.Index - 1).Range("D35") = "" Then
            'Loop is in case user attempts to activate any sheet rather than next sheet
            For s = Sh.Index To 1 Step -1   'Step backwards through sheets
                'Find the  a previous sheet where D35 not null
                If Worksheets(s).Range("D35") <> "" Then
                    Worksheets(s + 1).Activate     'If not null then activate next sheet.
                    MsgBox "Must enter data in cell D35 before moving to next worksheet."
                    Exit For     'Exits the loop once a valid sheet is selected.
                End If
            Next s
        End If
    End Sub
