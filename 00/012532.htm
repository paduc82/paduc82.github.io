ken sheridan say: To assign multiple volunteers to a record you should be using a subform, not a multi-select list box, but firstly you need to get the logical model right.  If we assume for this example that you  have a table Projects to which you want to assign one or more volunteers per project, what you have is a many-to-many relationship type between projects and volunteers, which would be modelled by tables as follows:

Projects
....ProjectID  (PK)
....Project
....etc

Volunteers
....VolunteerID  (PK)
....FirstName
....LastName
....etc

and to model the relationship type:

ProjectVolunteers
....ProjectID  (FK)
....VolunteerID  (PK)

This table models the many-to-many relationship type by resolving it into two one-to-many relationship types.  The table may also include other columns e.g. to record the start and end dates of the period when each volunteer is assigned to the project.

To represent this in a form you would have a form, in single form view, based on Projects.  Within this form would be a subform in continuous forms view, based on ProjectVolunteers and which is linked to the parent form on ProjectID.  You'll find an example in the file ParentActivities.zip in my public databases folder at:

https://skydrive.live.com/?cid=44CC60D7FEA42912&id=44CC60D7FEA42912!169

This little demo file illustrates a basis many-to-many relationship type like this and how to represent it in forms/subforms.  If you select the Parents by Activity form you'll see that this includes a subform which lists each parent for the current activity, and allows you to select a parent from the combo box's drop down list.  The parent's address is displayed in a single unbound control which references the third column of the volunteer combo box's RowSource (Column(2) as the Column property is zero-based) which concatenates the address columns into a single string expression.  For this it uses the ConcatValues function from the basConcat module.

You can also enter a new parent not currently represented in the database by typing their name in the format FirstName LastName directly into the combo box, whose NotInList event procedure includes code to open the parents form and pas the names to it, so that the address data for the new parent can be inserted.

ken sheridan say: You start with a Cities table like this:

Cities
....CityID (PK)
....City

You can add a further column for the state or whatever is appropriate in your case.

If each project is located in only one city then you simply need to add a numeric CityID column to the Projects table.  You can then add a combo box to the projects form set up like this:

ControlSource:    CityID

RowSource:     SELECT CityID, City FROM Cities ORDER BY City;

BoundColumn:   1
ColumnCount:    2
ColumnWidths:  0cm

If your units of measurement are imperial rather than metric Access will automatically convert the last one.  The important thing is that the dimension is zero to hide the first column.

If each project can be located in more than one city then you have a many-to-many relationship type between projects and cities which would be modelled by a table in the same way that the relationship type between projects and volunteers is, so the table would be like this;

ProjectCities:

ProjectID  (FK)
CityID  (FK)

and you'd have a subform based on this table in the Projects form in the same way as you have a volunteers subform.

It gets a little more complex if you need to relate the volunteers per project to a particular city.  In this case, in addition to the ProjectVolunteers and ProjectCities tables you'd also need a ProjectVolunteerCities table like this:

ProjectCities:

ProjectID  (FK)
CityID  (FK)
VolunteerID  (FK)

This table models a many-to-many relationship type between ProjectVolunteers and ProjectCities, so has two composite foreign keys, ProjectID/CityID and ProjectID/VolunteerID, each of which reference the composite primary keys of ProjectCities  and ProjectVolunteers respectively.  The projects subform would now have a third subform based on this table, in which the individual volunteers assigned to a project could be assigned to one or more of the cities in which the project is located.
