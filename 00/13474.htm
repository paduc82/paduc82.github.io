ken sheridan say:
Could the "#" in the field be causing the problem?

Yes, because the # character in the pattern used by the LIKE operator represents any single digit.  I assume that you are using the LIKE operator and the asterisk character to allow for the parameter being NULL.  This is often done, but is an unreliable method as it will not return rows where the column is NULL.  NULL is the absence of a value and its comparison with any value returns a NULL, nether TRUE nor FALSE.  Also you can run into the kind of problem you've encountered.

The reliable method for 'optionalizing' parameters is not to use the LIKE operator but this:

WHERE (MyField = Forms!MyForm!MyControl OR Forms!MyForm!MyControl IS NULL)

Note that the parenthesizing of the OR operation is critical.

Of course if you do want to use the LIKE operator for partial pattern matching then you need to be a little more circumspect where the column can concevably contain 'problem' characters, replacing the # character with a high Ascii character such as the tilde when doing the comparison:

WHERE (REPLACE(MyField,"#","~") LIKE REPLACE(Forms!MyForm!MyControl,"#","~") & "*"
OR Forms!MyForm!MyControl IS NULL)

Asha 311 - Possible reasons behind WiFi bug - Microsoft Community

singhnsk say:

Asha 311 - Possible reasons behind WiFi bug

Was free today so thought of trying to find some details about the Wifi issue happening with Nokia Asha 311 or probably every Asha Full-Touch device. Users experiencing this issue can't connect use WiFi while browsing with Nokia Browser or using Mail/Chat apps. Here are the exact details about why this is happening:

Reason #1 - Internet Settings sent by Operator:
First and the most common reason for everyone. As soon as the operator sends a message containing the Access Point/Web or Streaming settings, the issue will arise. The problem will start even if the user doesn't saves the settings and rejects them directly. Nokia Browser connects properly using WiFi till the operator has not sent any settings. Also, the same thing happened with all the operators (AirTel, Vodafone and Docomo) I tried, So its definitely not an issue with the setting delivery of operator, instead a bug with the firmware.

Reason #2: Restoring a previous backup of the phone contents:
If a backup of phone was created and then restored (after a clean install of firmware), the issue will arise again. It doesn't matter if the user restores only a part of the backup's contents.
Analysing this brings in an another bug - Any backup created will still include a part of the backup of settings. I.e if the user only backs up messages, the phone will still backup some of the settings along with it. This occurs with both the backup through PC suite or through the Sync and Backup option present in the phone.

I was looking to file bug reports in the Developer Forum but thought to post it here first if someone can notice it here and I'm also not familiar with filing bugs there.

To the users - If you own a similar Asha Touch device, it will be very helpful if you can confirm my findings if they appear the same way with you or not. I've tried reproducing the same with a few different Firmware Variants to conclude that the problem is universal.
