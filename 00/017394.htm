You may use this as starting point. Modify as needed.

 

Sub SaveCopy()
    ' Modify path as needed but keep trailing backslash
    Const strPath = "C:\OtherFolder\"
    Dim strName As String
    Dim lngPos As Long
    strName = ActiveWorkbook.Name
    lngPos = InStrRev(strName, ".")
    If lngPos = 0 Then
        MsgBox "Please save the workbook before running this macro.", vbExclamation
        Exit Sub
    End If
    strName = Left(strName, lngPos - 1) & "_" & _
        Format(Date, "yyyymmdd") & Mid(strName, lngPos)
    ActiveWorkbook.SaveCopyAs strPath & strName
End Sub

john w. vinson say:

You'll need a SELF JOIN query for this - add your table to the query TWICE, joining the two instances by StudentID. I presume you want to ignore records where Weight is zero so use a criterion >0 on the weight field. I take it that you also have the student's height (needed to calculate BMI) and other demographic information.

Then put a criterion on Table.[Date]  to find the most recent previous date using a Subquery. The query will be a bit complicated but it's doable: something like

SELECT A.Name, A.Weight, A.[Date], B.[Date],<BMI calculation expression>, A.Weight-B.Weight AS WeightChange

FROM yourtable AS A INNER JOIN yourtable AS B ON A.StudentID = B.StudentID

WHERE A.Weight <>0 AND B.Weight<>0 AND B.[Date] =

(SELECT MAX C.[Date] FROM yourtable AS C WHERE C.StudentID = A.StudentID

AND C.[Date] < A.[Date]);

Note that Date is a reserved word and therefore a bad choice of fieldname - Access will confuse it with the builtin Date() function. I'd use WeighDate or some other distinctive name.

When users don't sign in for more than 365 days or within the first 10 days after signing up for an account, the system will flag it as Inactive. After an account becomes inactive, all messages, folders, and contacts are deleted. Incoming messages will be sent back to the sender as undeliverable.

 

Also, if you were accessing your emails on a mail client such as Windows Live Mail or through your mobile device, but didnt log into your web account for over 365 days, you may lose emails due to inactivity in web account.

 

2. Account has been hacked or compromised.

You may lose emails if your account was hacked and the hacker had already deleted emails in your account.

 

3. Conversation threading is enabled.

If the feature Conversation threading is turned on wherein all emails with the same subject line are grouped together. You can click here to know more about this feature.

 

4. Schedule Cleanup is set.

This feature can be set to automatically delete emails. You may refer to this link for more information about it.
