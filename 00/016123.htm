julie sheets say:
Hello Alan,

You wrote:

The overallocated resources filter and the project filter each work individually when I run them from the resource usage view of a project connected to the resource pool.

Understood.  However, the overallocated resources filter shows only resources, not assignments and the project filter shows assignments not resources. Which explains why the combined filter returns no records.

The project manager does not have to disconnect his or her file from the pool.  Open the project file connected to the pool and when asked to open other files, do not open the other files.  The resource usage view now shows only assignments from the currently opened file.  Apply the Overallocated filter and you get what I believe you are searching for -- Overallocated resources only from the current project file.  No other assignments are visible.

If the project manager makes any changes they can be synchronized with the pool when they next open the pool file.

I hope this helps.

In the new module, paste the following code

'==========>>
Option Explicit

'---------->>
Public Sub Tester()
    Dim WB As Workbook
    Dim SH As Worksheet, SH2 As Worksheet
    Dim RngDV As Range, RngList As Range
    Dim rCell As Range
    Dim DV As Validation
    Dim arr As Variant
    Dim i As Long
    Const DV_Sheet As String = "Sheet1"                                      '<<===== Change
    Const List_Sheet As String = "Sheet2"                                     '<<===== Change

    Set WB = ThisWorkbook
    With WB
        Set SH = .Sheets(DV_Sheet)
        Set SH2 = .Sheets(List_Sheet)
    End With

    On Error Resume Next
    Set RngDV = SH.Cells.SpecialCells(xlCellTypeAllValidation).Cells(1)
    On Error GoTo 0

    If Not RngDV Is Nothing Then
        Set DV = RngDV.Validation
        arr = Evaluate(DV.Formula1)
    End If

    For i = 1 To UBound(arr)
        RngDV.Value = arr(i, 1)
        SH.PrintOut
    Next i

End Sub

Word's encryption of files is very, very strong. The only known method to break such password protection is brute force, i.e., trying increasingly complex candidates by combining dictionary words, upper and lower case characters, digits and symbols in orderly sequences until exhausting the various combinations.

For trivial candidates like "fox" or "qwerty" and words that appear in a dictionary, brute force will work.
