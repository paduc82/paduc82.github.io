You will need to Post your Router make and Model along with any additional information so someone can help. A basic understanding of Home/ Private Networking will also be helpful.

A few "tips" off the top that people generally do to cause a Conflict and how to correct it, would be;

  • Enable UPnP (Universal Plug and Play)
  • Enable a Quality of Service according to either the MAC Address or the specific Port used.
  • Enable DHCP
  • Enable Automatic Settings

Or

  • Assign a Static IP Address (if applicable)
  • Assign Port Forwarding to that IP Address
  • Assign a DMZ
  • Assign Manual Settings

Do not use Multiple Solutions at any one time, as an example;

  • Do not Assign Assign Port Forwarding along with DHCP or UPnP as a Conflict will result giving Strict/ Moderate NAT Settings
  • Do not Assign a Static IP Address if DHCP is Enable (the same above)

Keep in mind, Automatic Settings (DHCP) is to be used with a Dynamic IP Address (as the IP Address will change upon a Renew/ Lease of the IP Address from the DHCP and therefore cannot have Assigned Ports as Opened. A MAC Address, however does not chnage and can be used along with DHCP. A Static IP Address can be Assigned specific functions in regards due to it will always be the same IP Address of the Device on the Network.

Such a function would have to be supported by the web site itself. As most web sites are expecting a text string as the user credentials Windows has to offer that to the site. For a site to accept a facial sign on it would have to open the camera and do the analysis. Apart from the traffic it would generate there are other security implications.
