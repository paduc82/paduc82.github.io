How do I "Freeze" the numbers given by RANDBETWEEN? - Microsoft Community

kevinaustin say:

How do I "Freeze" the numbers given by RANDBETWEEN?

I'm running a random sample by assigning data numbers and running RANDBETWEEN to select the data randomly.  Only problem is after I run RANDBETWEEN the values change whenever any work is done in the spreadsheet.

 

How do I make the values permanent to prevent them from changing?

joeu2004 say: Kevin wrote:

I'm running a random sample by assigning data numbers and running RANDBETWEEN to select the data randomly.  Only problem is after I run RANDBETWEEN the values change whenever any work is done in the spreadsheet.   How do I make the values permanent to prevent them from changing?


Although I tend to use the copy-and-paste-special-value method myself, that can be inconvenient if you have a lot of such formulas and you want to be able to generate new random data periodically easily.

An alternative is to bury the RANDBETWEEN call in a non-volatile VBA user-defined function.  For example:

Function myRANDBETWEEN(lo As Double, hi As Double) As Double
myRANDBETWEEN = Evaluate("randbetween(" & lo & "," & hi & ")")
End Function

You use myRANDBETWEEN exactly as you would use RANDBETWEEN, e.g. =myRANDBETWEEN(1,1000).

Press ctrl+alt+F9 to generate a new set of random data and recalculate the entire workbook (actually all workbooks open in the same Excel instance).

Caveat:  myRANDBETWEEN runs significantly slower than RANDBETWEEN.

To create the UDF above, do the following:

  • In Excel, press alt+F11 to open the VBA window.
  • In VBA, click on Insert, then Module to open VBA editing pane.
  • Copy the text of the UDF above and paste it into the VBA editing pane.
  • You can now close the VBA window.

In Excel, be sure to set your macro security appropriately.  In XL2003, I prefer "medium" so that I am always prompted to enable or disable macros.

    chevia say:

    Hi Akiko,

    You can either add the subdomain in Office 365 Online side directly or local AD depending on how you want to use this subdomain.
     
    If you want to create users in local AD using the subdomain, then get them synced to Office 365 Online, you need to add the subdomain in your local AD.

    Otherwise, if you want to add the subdomain in Office 365 Online and create users in Office 365 Online side directly, you need to add the subdomain in Office 365 portal, but not local AD.
