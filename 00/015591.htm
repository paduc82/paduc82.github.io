ken sheridan say:

The table is linked to another.................

A linked table is one in a front end file which links to the actual table containing the data in a back end file.  If you try to open a linked table in design view you should get a warning message about not being able to change the table definition.  If you proceed nevertheless, and try to change the table's property you'll be unable to do so.

I think by 'linked to another table' you probably mean 'related to another table'. i.e. the table is either the referencing table or the referenced table in a relationship between the two tables.

If a table is not a linked table, you should be able to remove the zero DefaultValue property simply by deleting it in table design view, as Duane has described.  You don't need to enter Null as the property, just leave it blank.  Note that this is not always a good idea, though.  Null is not a value but the absence of a value, so is semantically ambiguous.   The decision as to whether to remove the default value from a column should be made in the context of the nature of the attribute which the column represents.

graham mayor say:

It is not so hard if you can copy and paste. :)

The following macro will work for US currency (it will need modification for other country currency formats).

http://www.gmayor.com/installing_macro.htm   

Put the cursor in the number and run the macro

Sub FormatAsCurrency()
Dim oRng As Range
    Set oRng = Selection.Range
    oRng.MoveStartWhile "0123456789,.", wdBackward
    oRng.MoveEndWhile "0123456789,."
    oRng.Fields.Add Range:=oRng, _
                    Type:=wdFieldExpression, _
                    Text:=oRng.Text & " \# " & Chr(34) & ",$0.00" & Chr(34), _
                    PreserveFormatting:=False
    oRng.Fields.Update
    Set oRng = Nothing
 End Sub
