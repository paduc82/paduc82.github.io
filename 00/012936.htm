john - project say:
Nathalie,
July 7th through July 8th means start at 9:00 am on July 7 and end at 4:00 pm on July 8. We are both thinking the same thing but "through" and "to" are not the same, so we are getting crossed up in language.

One challenge with Project is that once data entries are messed up (i.e. whatever is causing the 1.67 days duration), sometimes the only "fix" is to delete and re-generate the task. That's one reason I suggested you display the time as well as the date. It help to figure out what's going on.

The best way to set up a schedule plan is to first set up the calendar and working times, if they are to be different from the default, and then enter task information.

With regard to default start and end time, as the note next to the setting states, those are the default times given to new tasks when the user enters a date in the Start or FInish field. Do they mean anything for the duration? Yes, the duration of a task is the difference in working time between the start time of the task and the finish time of the task divided by the hours per day setting. Changing the default values after a task's data has been entered will have no effect. They only apply to new tasks that are entered.

John
