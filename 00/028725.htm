mel77084 say:

The OEM recovery partition was intended to allow the user to restore the computer to the state it was in coming from the manufacturer (with all the crapware in place).

Now, Microsoft, being aware of the increase in the use of smaller SSDs, is changing in the development of Windows 10, so that it will

If you recall reading the precautions before installing the first build of the TP, the statement was made that after that installation, your computer would not be able to use the OEM recovery partition to go back to the OEM system state. That is changed now in the current build .... if that OEM recovery partition has been left alone, one of the options available in Settings, Update & Recovery, Recovery, is to return to the OEM system state, which would use the OEM recovery partition.

Also, if you go into Settings, Update & Recovery, Recovery, one of the choices available will be to return to the previous version of Windows. For that, it would use the Windows.old file in C.

Another option would use the restore image in C to restore the currently installed build. (I haven't used that provision, so don't have detailed knowledge of it.

But ... suppose the hard drive fails suddenly ... those options would be gone.

For that reason, I use a 3rd party imaging software (Macrium Reflect, free for non-commercial use) to make a partition image of each build as soon as it's installed and cleaned up, and save the image on a defferent hard drive ... and, of course, have the rescue CD, that the the program also makes, so, neither the OS messing up, or hard drive failure ruins the day for me.
