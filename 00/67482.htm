a. user say:

Hello,

Thank you for posting your query in Microsoft Community.

I can imagine the inconvenience you are experiencing and will certainly help you in the right direction to fix the issue.

If you receive Windows Update error 80070103 while installing updates, you might be trying to install a driver that's already installed or a driver that might be less compatible than one you already have installed on your PC. This concludes that the driver is fits incompatible with the current version of Windows installed in the system.

In the first place, I would suggest you to update the graphics driver from the device manager and check if it helps.

  1. Press WinKey + X, click Device Manager.
  2. Expand the node Display adapters.
  3. Right click on the device driver and click Update Driver Software.
  4. Follow the onscreen instructions to update the driver.
  5. Restart the system and check if the issue persists.

If the issue still exists, then I would suggest you to install the display adapter in compatibility mode. If you have the raw executable driver file, then follow these steps to install the driver in compatibility mode.

  1. Right click on the downloaded driver file.
  2. Click onProperties and then select Compatibility tab.
  3. Tick mark the option Run this program in Compatibility mode for: and then select Windows 7 (or compatible operating system) from the drop down menu.
  4. Click OK and install the driver.

Additional information:

Make older programs compatible with this version of Windows

If the issue still persists, follow the below provided methods:

Method 1:

Try to run the Windows update troubleshooter provided in the following link.

Troubleshoot problems with installing updates

Try installing the updates after running the troubleshooter.

If the issue still exists, try the next method.

Method 2:

I would suggest you to use Deployment Image Servicing and Management to fix Windows Update errors.

a) Open an elevated command prompt. To do this, press Windows key + Q and type on CMD right click on Command Prompt from the list and select Run as administrator. If you are prompted for an administrator password or for a confirmation, type the password, or click Allow.

b) Type the following commands. Press Enter after each command

  • DISM.exe /Online /Cleanup-image /Scanhealth
  • DISM.exe /Online /Cleanup-image /Restorehealth

c) Close the command prompt, and then run Windows Update again.

Hope this information is helpful. Do let us know if you need any further assistance, we'll be glad to assist you.

robert aldwinckle on forums say:

with the next touch to scroll through the page, keyboard immediately appears again. Please advise.

Then the page somehow looks like an input area?  FWIW I would say that more and more the TK needs to be opened manually, even in circumstances where one would think it would open automatically (such as a tap in an input field.)  In any case a workaround would be to start the real OSK minimized (osk.exe).   Then the TK (what you see) could never appear.   A disadvantage would be that you would have to find the OSK and close it before you could use the TK when you wanted to;  however, that has been made relatively painless now with the availability of the Task View (e.g. swipe from the left).

Alternatively, you may need to examine what you mean by "touch to scroll", e.g. there is a two finger scroll that would be less likely to be misinterpreted as a tap but it needs to be done precisely or it could be misinterpreted even worse as a double tap.
