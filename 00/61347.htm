Windows 7, Sony VAIO FJ270, and upgraded Hard Disk. The Saga and - Microsoft Community

garthbrooks63 say:

Windows 7, Sony VAIO FJ270, and upgraded Hard Disk. The Saga and hopefully a solution

Posting this as help for those who may face a similar problem!

My son has a Sony VIAO VGN-FJ270. It came standard with XP, 1Gig RAM and 100Gig HD about 4 years ago. I upgraded the RAM about a year ago to 2 Gigs. This Christmas I offered to upgrade his Hard Drive to a 320Gig WD Scorpio 7200 RPM and install Windows 7 for him.

Installing the HD went fine as it was a SATA for SATA replacement. Then the load of Win7 went fine as well. The problems began to crop up after I began to install the various drivers for the VAIO Notbook Utilities to enable the hot-keys for screen brightness and volume/mute.

The Original try resulted in a system where the bootloader was corrupted. I tried all methods of bcdedit, bootrec repair, and system restore via restore points. However, once the system got to the bootloader corruption, there was no getting it back. I tried this a number of times! In the end a fresh install of Win 7 was required to get it back.

After three days of various combinations, I got to the point where I was analyzing the BIOS and found that the BIOS anly saw my 320Gig HD as 137 Gig! My thought before was that the BIOS was scewing up the partition info for the bootloader, so I had an idea. On the next load, I created a 60Gig partition with the Win7 install disk and left the rest unpartitioned. I then formatted the 60Gig partion.

I then did a fresh install of Win 7

On completion I used the Disk Manager to create another Primary Partition to take up all the unallocated space. I called this drive E:(Data). I use this drive as the location for all my User files (Documents, Music, Video... set the location using the "location" tab under properties for wach of those folders)

I then performed these installs in this order, and on each I told the system to use "Vista" Compatibility Mode, run as Administrator (File Properties>Compatibility)

FromIntel:
  1. Intel PROwireless 2200BG NIC (11.5.1.2_VT32.exe from Intel)
From Sony (Vista Home Basic Drivers)
  1. Realtek High Definition Audio Driver (REDAUD-00733504-UN.exe)
  2. Shared Library (SOASSL-A1298203-US.exe)
  3. Setting Series (SOASUS-01306401-US.exe)
  4. Sony NoteBook Control (SOAOTH-00445800-US.exe)
  5. ALPS Pointing Device (ALDOTH-01347907-UN.exe)(Sony Europe)
  6. Utilities DLL (ALDOTH-01347907-UN.exe)
  7. Event Svcs (SOAVES-01302701-US.exe)
  8. Battery Check (SOABCU-01298303-US.exe)
I did a reboot between each driver install. It rebooted fine. In the end I had the Sony hotkeys working! AND, no bootloader issues!

THEN, goto Microsoft update and download latest updates. Be sure to check the optional updates. HOWEVER, DO NOT CHECK "Sony Firmware Extension Parser Device" as I believe this may have killed my system once.

After all Updates, the system seems to be happy and running after many reboots and installing Office 2007 and itunes!!! Woot!

I'm finding that the relocation of user files is not so simple. The NTFS junction points do not get updated, I'm going to have to do that manually. (Google "NTFS junction points", also, use "dir/a" in command window to see them ... be careful, you need to know what you are doing)

If you don't want to keep your user data an a different partition, you could just set that first one from 60G to 100G. I think the trick is to keep it under the number that is reported in BIOS so that the bootloader doesn't get freaked.

Cheers!
GARTH
