davidh1977 say:

I had the same issue for a client just now and found this while trying to find a solution.  Nothing above worked for me but I found a solution.  I thought I'd share in case it helps someone else out.

External users sending emails to this mailbox were being rejected:

could not be delivered. The problem appears to be :

Additional information follows :

Running a message trace turned up no results.  It wouldn't even show a denial.

Like above I checked the delivery restrictions and confirmed it was set to allow from all senders and no authentication was required.  I even ran in PowerShell:

get-mailbox <email address> | FL

and confirmed this.

RejectMessagesFromDLMembers            : {}
RejectMessagesFromSendersOrMembers     : {}
RequireSenderAuthenticationEnabled     : False

I also checked forwarding.  Forwarding was enabled but to an address that was also able to receive outside email from unauthenticated sources.  I turned off the forwarding anyway and messages could still not be received.

In the delivery options I turned ON the option to require authentication, saved it, then came back and turned it off again, there was no change.

Then in PowerShell, I did essentially the same thing by running these two commands:

Set-Mailbox -Identity <email address> -RequireSenderauthenticationEnabled $TRUE
Set-Mailbox -Identity <email address> -RequireSenderauthenticationEnabled $FALSE

For some reason, after this it began accepting messages again.  Even with the forwarding re-enabled messages are being delivered as expected.

I hope this helps someone as one of my teammates and I spent hours on trying to figure this out and were just about to open a case with Microsoft.

bob jones aka: cybertaz say:

Sorry, but anything copied to the OS X Clipboard is copied as a singular, aggregate body of content. Neither the OS nor any of the software programs have the ability to allow the user to pick & choose which individual pieces of that content get pasted when or where. I'm not an authority on either, but I doubt something could be done using VBA or AppleScript.

One possibility to explore is 3rd-party clipboard utility software. I don't know whether anything may offer a useful capability because I've never used any of it as some have been known to create conflicts... Be sure to confirm compatibility with both your version of OS X & Office.

IMO, your best bet is to open the Word document & the presentation file windows side-by-side then select each bullet item [*] & drag it onto the appropriate slide. As long as you don't drag into an existing text box a new one will be created for the incoming content. Conversely, if you drag into an existing text box the incoming text will be dropped into that box.

* A few tips if you aren't already aware;

  • To select a 1-line bullet item paragraph, click in the margin to the left of the bullet
  • If it's a multi-line bullet item hold Command while clicking in the itemtext to select the entire multi-line paragraph at once
  • If you prefer to cut/copy from Word, in PPT use Edit> Paste Special - Unformatted Text or Styled Text rather than simply pasting
