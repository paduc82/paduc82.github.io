    How do I fix multiple monitor issue with Publisher? (Using - Microsoft Community

    adrian_391 say:

    How do I fix multiple monitor issue with Publisher? (Using guides is impossible, as the mouse will go crazy)

    How do I fix multiple monitor issue with Publisher? (Using guides is impossible, as the mouse and item you are editing will go crazy) 

    I have actually found a relatively simple workaround to make Publisher work in Windows 10 with the guides and multiple monitors. My setup is an HP Elitebook G4 with 3 HP Elitedisplay E233s. Note: I don't use the laptop display, just the 3 monitors. 

    Here are the steps I took:

    1) Right-click desktop

    2) Open Display Settings

    3) Click Identify (It will flash on your monitors the number assigned to each monitor)

    4) Go to monitor "1" by clicking that icon, then scroll down to "Make this my main display" and click the box next to it. (It will have a check-mark once this is complete)

    6) Now click your other monitor(s)' icon(s) and choose "Disconnect this display" under "Multiple displays" and click "Apply". This will electronically disconnect that monitor and prompt you with a box to "keep changes, or revert". Keep the changes and repeat for each monitor beyond "1" which you made your main display. 

    7) Your display settings Window should now tell you that a "sign out is required to make all changes take effect" Do this, but save all your work before you actually hit "Sign out"

    8) It will then make the changes and then prompt you to sign back in. Sign in.

    9) Go back to display settings by right-clicking the desktop.

    10) Enable your other monitor(s) by clicking their icons and choosing "Extend desktop to this display" for each (2 then 3).

    11) You should now be able to open Publisher, and use it as normal on multiple monitors, with no issues editing.


    This seems like a lot, but it is actually simple, and only takes a few moments. It should be noted that I had to repeat these steps when my desk was moved, and my monitors were disconnected from the laptop docking station. I hope this was helpful to someone. I know that I was very frustrated trying to remember how I fixed this issue months ago (before my desk move).

    a. user say:

    Hi Kenneth,

    The Windows.old folder contains your old Windows system files. It is created so that you can go back to your previous version of Windows if you want to. This folder is a temporary folder. Windows automatically deletes this folder 10 days after the upgrade. You can manually delete this folder to free up space if you have no plans on going back to your previous version. To learn more about how to delete it, kindly refer to this article: Delete your previous version of Windows.

    The Windows10Upgrade folder is created when you install the Windows 10 Update Assistant app. This app updates your computer to the latest version of Windows 10. You can safely delete this folder by uninstalling the Windows 10 Update Assistant app.

    $WINDOWS.~BT and $Windows.~WS folders contain files that are associated with the upgrade process of Windows 10. $WINDOWS.~BT contains log files and files from the Windows installation that you had before. $Windows.~WS is a setup folder for Windows 10. Both of these folders are temporary folders. Windows will automatically delete them after 10 days.

    andy pope say: Here is one way. Assigning the specials cells to a range object and using that to insert previous cells values.

        Dim rngGaps As Range
       
        On Error Resume Next
        Set rngGaps = Range("Table1[Information]").SpecialCells(xlCellTypeBlanks)
        On Error GoTo 0
       
        If Not rngGaps Is Nothing Then
            With rngGaps
                .FormulaR1C1 = "=R[-1]C"
                .Value = .Value
            End With
        End If

    russ valentine say:

    You can't. There are many reasons why exporting is not the correct way to transfer Outlook data. One of them is that DL's are lost when you export. Use a supported method for Outlook data transfer instead. Take a look at these pages for info on Outlook data backup or transfer:
    http://www.slipstick.com/config/move_outlook.asp
    http://www.howto-outlook.com/Howto/backupandrestore.htm
    http://office.microsoft.com/en-us/assistance/HA010771141033.aspx

    Note that some of the information on the Microsoft page is outdated and contains errors, particularly when applied to Outlook 2003, 2007 or 2010. Specifically:
    1. Ignore any advice to copy your PST file to Outlook's default location to connect it to your profile. Doing so will often corrupt your Outlook profile. Copy it anywhere BUT the default location and then open it within your Outlook profile.
    2. Ignore any advice that tells you to use export or import to transfer an entire Outlook data file. That has never been sound advice, but the process has become too deeply flawed to be trusted.
