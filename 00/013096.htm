On form_KeyUp doesn't fire

john w. vinson say:

I'd use the Form's Current event (which fires when you move to a record), and the AfterUpdate events of the controls bound to the two fields (Forms don't have "fields", just Controls, which may or may not be bound to fields in the Form's Recordsource).

The KeyUp (and KeyDown) events are rarely appropriate, and certainly not in this situation; and the Change event isn't either, if fires at every keystroke. The AfterUpdate waits until the user moves off the control, committing a new value to the field.

E.g.

Private Sub Form_Current()

Me!cmdButton.Enabled = (Me!txtField1 = Me!txtField2)

End Sub

and similarly in the AfterUpdate events of txtField1 and txtField2 (or whatever you name the controls).

If Office 365 manages DNS for your domain, Office 365 handles spam filtering for you. You can't add another TXT record for SPF (spam filtering). graham mayor say:

Outlook is not Word, though it does have a Word-like e-mail editor; but you need to be editing a message and to tell the macro that you wish to access the Word editor, so the following should be closer to what you have in mind

Option Explicit

Sub ProofFrench()
    On Error GoTo ErrHandler
    If TypeName(ActiveWindow) = "Inspector" Then
        If ActiveInspector.IsWordMail And ActiveInspector.EditorType = olEditorWord Then
            With ActiveInspector.WordEditor.Application.Selection
                .LanguageID = 1036
                .NoProofing = False
            End With
            ActiveInspector.WordEditor.Application.CheckLanguage = True
        End If
    End If
lbl_Exit:
    Exit Sub
ErrHandler:
    Beep
    Resume lbl_Exit
End Sub

Sub ProofEnglish()
    On Error GoTo ErrHandler
    If TypeName(ActiveWindow) = "Inspector" Then
        If ActiveInspector.IsWordMail And ActiveInspector.EditorType = olEditorWord Then
            With ActiveInspector.WordEditor.Application.Selection
                .LanguageID = 1033
                .NoProofing = False
            End With
            ActiveInspector.WordEditor.Application.CheckLanguage = True
        End If
    End If
lbl_Exit:
    Exit Sub
ErrHandler:
    Beep
    Resume lbl_Exit
End Sub

sequential page numbering following section break in word 2010 - Microsoft Community

marilynkl say:

sequential page numbering following section break in word 2010

I am working on a lengthy document with several next page section breaks, and the page numbering restarts at 1 at each section break.  I want sequential numbers for the entire document.  Can someone please advise how I can achieve this.  For example, at page 6, following a section break, my page number is page 1 of 20 instead of page 6 of 20.  Further into the document, following another section break, the page number restarts at page 1 again, and reads page 1 of 20 again.  Looking for some assistance please.  Thank you.

doug robbins - mvp office apps & services (word) say:

If your document has many Sections, the easiest thing to do is run a macro containing the following code:

With ActiveDocument
    For i = .Sections.Count To 1 Step -1
        .Sections(i).PageSetup.SectionStart = wdSectionContinuous
    Next i
End With

Alternatively, you need to access the Header\Footer pane and on the Header & Footer Tools>Design tab of the ribbon, expand the Page Number drop down and then select Format Page Numbers ... and in the Page Number Format dialog, select the radio button for "Continue from previous section".  You will need to do that for each Section of the document.

You should be able to see your text messages on your desktop/laptop.  Follow these steps:


Log in to outlook.com

  • Where it says "Quick Views" in th3e left hand side bar, right click on one of the categories and choose "Manage Categories" from the pop-up menu
  • You should see a list of categories with two columns of tick-boxes against each one
  • Tick the "Quick View" on the "Contacts" category
  • A new Contacts quick view will appear.  Click on the heading and you should see your text messages appear in the main message window.

 

Once created, the "Contacts" quick view category will remain there, so you can go back to your text messages online whenever you want.

 

This works for me, hopefully it will for you as well.  Microsoft appear to be starting to make changes to outlook.com, so this method might not work in all markets.
