steve rindsberg say:

You won't be able to change the embeddability property of the font; that's something that's up to the manufacturer.  However, you might want to test with a different font on a copy of the presentation to see if the same problem occurs.

Preview/Print means that you *can* embed the font (or should be able to according to the manufacturer) but on a system that doesn't have the font, you'll only be able to open the presentation for viewing/printing but not editing.

Way back, there were some crazy issues with this, where PPT would only open the presentation read-only, no editing permitted, and since no editing was permitted, you weren't allowed even to substitute a different font.  That would be editing, y'see.  So said the lawyers.  Who they later fed to the orcas and changed PPT so that IF there was one of these preview-only fonts embedded and IF you didn't actually have the font on your computer, you'd get the opportunity to substitute a different font when opening the presentation.

ryanlv_msft1519 say:

hi boyan,

to archive onedrive for business library for a deleted user, i’d suggest use ediscovery feature to place on hold to the user’s onedrive for business library. therefore, when the user account is deleted, the library and files stored will be preserved until on hold state is removed.

to do this, we need to first create an ediscovery center in sharepoint online, create a case and add this library as a source. when the in-place hold status turns to on hold, the user’s library is protected.

meanwhile, i’d also suggest assign the user’s onedrive for business site admin to a global admin for management. you can do this via sharepoint admin center > user profiles > manage user profiles > search for the user > manage site collection owners > assign admin to a global admin.

for detailed explanation about the process of onedrive for business library deletion after the user is deleted, please refer to:
https://support.microsoft.com/en-us/kb/3042522

for how to create an ediscovery center and place on hold on the desired library, please refer to the support articles below:
set up an ediscovery center in sharepoint online
plan and manage ediscovery cases


regards,
ryan

ryanlv_msft1519 say:

Hi Boyan,

I’d like to make it clear for you, so you can release the license and place a hold on to the desired library as well.

I have tested to confirm the outcome. The basic steps are as below:
1. Delete the user via Office 365 admin center. (License is released immediately.)
2. Take ownership of the user’s OneDrive for Business site.
Go to SharePoint admin center > user profiles > manage user profiles > search for the user > manage (primary) site collection owners > assign admin to a global admin
3. Create a case for the user’s OneDrive for Business site and place a hold.
Note: Please finish the steps within 30 days after the user account is deleted via Office 356 admin center. Otherwise, user's profile and his OneDrive for Business library are permanently removed from SharePoint.
