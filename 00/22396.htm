tasos_k say:

to whom it may concerns

scenario:

search in a folder (Do/Loop) for xls/xlsx/xlsm files

split each wb in separate new files (xlsx), every N entries

Sub SplitFilesXL_in_Separate_newWBks()
'June 19, 2016
Const lrow As Integer = 2 '<< data from row 2
Const N As Long = 100 '<<split data every 100 entries
Dim wb As Workbook, newwb As Workbook
Dim sPath As String, sFile As String
sPath = "E:\folderA\FolderX\" '<< files in folderX/change Path
Dim r As Long, x As Long, t As Long
Dim v As Variant
Dim newName As String
sFile = Dir(sPath & "*.xls*") '<< search for xls/xlsx/xlsm
Application.ScreenUpdating = False
Do While sFile <> ""
Set wb = Workbooks.Open(sPath & sFile)
r = Sheets(1).Cells.Find(What:="*", SearchOrder:=xlByRows, SearchDirection:=xlPrevious).Row

If r <= N + 1 Then GoTo nnext

v = Split(wb.Name, ".")
newName = Replace(wb.Name, "." & v(UBound(v)), "")
t = 1
For x = lrow To r Step N
Set newwb = Workbooks.Add
wb.Sheets(1).Rows(1).Copy
newwb.Sheets(1).Cells(1, 1).PasteSpecial xlPasteValues
wb.Sheets(1).Rows(x & ":" & x + N - 1).Copy
newwb.Sheets(1).Cells(2, 1).PasteSpecial xlPasteValues
newwb.Sheets(1).UsedRange.EntireColumn.AutoFit
Application.DisplayAlerts = False
ActiveWorkbook.SaveAs Filename:=sPath & newName & "-" & t & ".xlsx", FileFormat:=xlOpenXMLWorkbook 'xlsx format
Application.DisplayAlerts = True
newwb.Close False
t = t + 1
Next

nnext:

wb.Close False
sFile = Dir()
Loop
Application.ScreenUpdating = True
End Sub

dianeoforegon say: Exchange is stored as cached files. Messages are stored by size not folder or account. Just drill down in the messages folder in Data Records in your Identity.

 ~/Documents/Microsoft User Data/Office 2011 Identities/your identity/Data Records/Messages

You do not have to use "On My Computer" folders with Exchange. However to store messages off the server to view, this is the only option at this time. It's best to keep all messages on the server if possible to avoid data loss.

Outlook Mac does not have OST files.
Outlook Mac can only import from PST.

Exchange Basics

The Inbox syncs more frequently than any other and should be kept as clean as possible. (this is the number one cause of problems with sync). Do NOT make subfolders under the Inbox.

Any folder with more than a few thousand messages is going to take some time to fully come down when you first sync your account. Due to the design of Outlook's sync engine (and how exchange works), you will see the 512 newest messages in a folder at the initial start of sync then the rest of the folder's contents will be back filled before you see any mail that arrives after this point. This is why it will look like you are not getting new mail when a folder is still going through initial sync. This is also further exacerbated by an Exchange issue where it gets unnecessary change events before the newer mail arrives. If you carefully watch the progress during the sync of a folder with greater than a couple thousand messages, you will see "Updating local..." a lot before the newer mail starts arriving (newer than the initial 512 that come down).

Outlook checks the server for updates every minute, any folder that has updates will subsequently be synced. Since Outlook has a limit on how many folders can be synced at a time, there can be a queue of folders waiting. The Inbox does get high priority so it will generally sync before other folders that also need to sync.

j.e. mcgimpsey say: It's not specifically a Lion issue, but Lion may be contributing.

XL takes its default calculation setting (Automatic/Manual) from the first workbook opened in the session, or, if you start the application directly (i.e., from the Dock), it uses the Preferences setting.

Subsequent workbooks don't change the setting, so if you open a workbook set to Manual first, the rest won't recalculate automatically.

I don't know if Lion's Resume feature affects this setting if it automatically opens a file set to Manual (and I can't check right now).

Open XL from the Dock, close any files, open a new file, and set your Preferences to Automatic. Quit XL. When you open a workbook and it doesn't recalculate, reset its preferences and save the workbook.
