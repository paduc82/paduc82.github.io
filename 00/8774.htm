marshall barton say:

First, the only way to display a datasheet's records in a specified order is by using a query.  Think of a table as a sack of records and they are retrieved and displayed in the table's datasheet in any order the database engine deems is an efficient way to do it, so you can not do anything about the sorting unless you use a query.

Second, each field in a table must contain a single value.  You are trying to stuff two values into one field, the primary sequence (your whole numbers) and the secondary sequence (the numbers after the dot).  Don't do that.  Instead use a second field or maybe even a second table for the secondary sequence number, depending on the meaning of the two numbers.  This will make the sorting and everything else about both numbers easier.  Whenever you want to display both numbers in the style you are you having trouble with, you can add a calculated field to the query to put them together however you want, possibly something like:

    PrimarySeq & "." & SecondarySeq

and the query can simply sort on the two fields:  PrimarySeq and SecondarySeq

marshall barton say:

I think that's the right idea.  To clarify, the Number (ID) field in each table is the table's primary key right?  While not required, you should name the fields differently, (e.g. CatID, SecID and SubID) just to make it easier for you to tell at a glance which is which.

The CatID field is a long Integer field in the Categories table and contains only the 1, 2, 3, ... category number.  I suggest that you avoid compound primary key indexes by adding an AutoNumber primary key field, CatPK, to this table.  Same for the SecID field in the SecondaryCategories table and contains only the 1, 2, ... secondary ccategory number and add an AutoNumber pk field, SecPK, as the primary key.  Same again for the SubCategories table.

The main category field in the SecondaryCategories table is long integer foreign keyfield to the CatPK field in the Categories table. Similarly for the secondary category foreign key field to the SecPK field in the SecondaryCategories table.

The primary key for the Elements table elements could also be an AutoNumber field, ElementPK.   The relationships for the Elements table to all the other tables' long integer field for the element foreign key instead of having the code field in each table.  This should work, even if some elements can be used in more than one category, secondary and subcategory.  You might be able to use the Elements table's Code field as the primary key, but there are other, not yet explained, things that would have to be carefully considered before deciding to do that.

joeu2004 say: Monkering wrote:
how to write a formula for BMI=Weight/Height Squared, in Excel starter

If A1 contains weight in kilograms and A2 contains height in meters, the correct formula is:

=A1 / A2^2

which can be written as follows for clarity:

=A1 / (A2^2)

If weight and height are in other units, multiply by an appropriate factor.  For example, if weight is in pounds and height is in inches:

=703 * A1 / A2^2

according to http://en.wikipedia.org/wiki/Body_mass_index, to wit:

For example if the local currency is in AED and all my 20 figures are in AED, i want the format to change to USD when the user selects USD etc..
hansv mvp say:

Right-click the sheet tab.

Select View Code from the popup menu.

Copy the following code into the module window:

Private Sub Worksheet_Change(ByVal Target As Range)
  Dim strCurrency As String
  If Not Intersect(Range("A1"), Target) Is Nothing Then
    Select Case Range("A1").Value
      Case "LCY"
        strCurrency = "AED"
      Case "USD"
        strCurrency = "USD"
      Case "EUR"
        strCurrency = "EUR"
    End Select
    Range("D2:D21").NumberFormat = _
      "_ [$" & strCurrency & "] * #,##0.00_ ;_ [$" & strCurrency & _
      "] * -#,##0.00_ ;_ [$" & strCurrency & "] * ""-""??_ ;_ @_ "
  End If
End Sub

Change D2:D21 to the range with the currency values.

This will only work if macros are allowed. You need to save the workbook as a macro-enabled workbook (*.xlsm).
