scottgem say:

With a long SQL statement like you have, the Locals window is not going to show all of it, you can't scroll to see the right end which is what you need to see since the problem is in your WHERE or HAVING clauses. So do what Duane suggested, the Debug.Print will print the resulting SQL statement to the Immediate window. You can review it there or copy and paste it into SQL View of a new query and then switch to Design mode. This should pinpoint where the error is.

But a Datatype mismatch error is one of the easiest to diagnose. For example you have this:

"WHERE (((tbl_JAZter_Blings.[BlingType]) <> 'CAll'))

According to that clause BlingType is a text datatype. But line suggests that it is populated from a combobox or lookup table. So its possible that BlingType is actually  number datatype.  As I said its easy to diagnose, just check the datatype for the field against the value you are comparing it to.

a. user say:

Restricting Administrator access to specific Sharepoint Librarys / files

Hi,

How can I prevent a SharePoint Collection Administrator from being able to read / open files in certain Libraries? 

I have tried stopping inheriting permissions and then only granting access to specified Users, but it seems that any one with an Admin role can read them regardless.

Thanks in advance,

Liz.

iris he msft say:

 Hi Liz,

The SharePoint site collection administrator is the one who has the biggest permission on the whole site collection. He has full control permission on every library and every item by default even though we stop inheriting permission.

So, for your requirement, it cannot be achieved within a site collection.

As a workaround, you may consider granting that user  the Full Control permission in the root site. In this way, you can set unique permission of several libraries to limit him to access. But please note: even though this user still has Full control permission, but he cannot do settings that applies to the whole site collection. For example, he will cannot see Site Collection Administration section in Site settings page.

Or, as cliff mentioned, you may consider storing files that stored in those libraries in another site collection which doesnt set that user as Site Collection Administrator to achieve the goal.

We appreciate your understanding.

Thanks,
Iris
