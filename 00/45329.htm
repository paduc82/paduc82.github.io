Thanks, Josh.  

andre da costa say:

It is still an OEM license and changing the motherboard will still invalidate it. What Digital Licensing does is simplify reactivation after a fresh install on the same hardware configuration.

When you upgraded from a previous version of Windows or receive a new computer preinstalled with Windows 10, what happened is the hardware (your PC) will get a digital entitlement, where a unique signature of the computer will be stored on Microsoft Activation Servers. The Windows 7 or Windows 8 genuine license you were previously running will be exchanged for a diagnostics key.

Anytime you need to reinstall Windows 10 on that machine, just proceed to reinstall Windows 10. It will automatically reactivate.

So, there is no need to know or get a product key, if you need to reinstall Windows 10, you can use your Windows 7 or Windows 8 product key or use the reset function in Windows 10.

If you are prompted to enter a product key, click 'I don't have a key' and 'Do this later'

Cause of 800706D9 error (Windows update, firewall etcetera) - Microsoft Community

th3r_565 say:

Cause of 800706D9 error (Windows update, firewall etcetera)

I used to come across this error with code 800706D9 oftenly and found a typical cause for it.

What surprised me is that others who had this issue (from Google) were advised to do other things that in the end can never have worked out, because they don't act against the cause. Even Microsoft support engineers didn't document the cause and solution right.

Cause/solution:

The error 800706D9 can occur when your Windows Firewall service is disabled/turned off. (or damaged! see end of this post for that..)
Windows Update or tools that throw this error are too heavily integrated with the Firewall service so they likely cannot function when off.

Fix, go to Start>Run>services.msc and navigate to 'Windows Firewall' then rightlick, > properties and set the setting to 'Automatic' in dropdown menu. Now hit the start button to start service up and apply before closing the options tab.

With ''Tools'' is meant several Windows apps. For example, another user encountered the error with Family Safety:

http://answers.microsoft.com/en-us/windows/forum/windows_other-security/error-800706d9-cannot-set-up-account-for-my-son/e92e8d38-f438-46ad-928a-76c71ba069c5

Myself, I encountered this error code with other MS software, indicating that if your Windows Firewall service is off or malfunctions, specific programs/utilities will fail to work with this error code because they rely on Firewall.

I created this post so it hopefully helps others who get this very famous error as this workaround may solve 9 on 10 times the error occurs. I am pretty confident that the cause almost (or always) is a problem with the Windows Firewall, be it a turned off service, or a malfunctioning or damaged service. (in that case try to use commandprompt > /sfc scannow).

This is a share of knownledge since I have serviced countless PC's as tech support person with this error, and all cases led back directly to the Win Firewall service.

Incase anyone knows for sure other system problems that can directly cause this error (which is pretty much the most-occuring Windows Update etc, error/failure) please share that on this discussion too.

BESIDES the Firewall service being disabled it can also be the Firewall service being damaged, or the Update service itself being damaged. For the latter, you can try this if enabling Firewall service doesn't fix it: https://support.microsoft.com/en-us/help/971058/how-do-i-reset-windows-update-components

If that doesn't solve it, you can try auto-fixing the Windows Firewall service by mainentancing connected OS parts: use my .bat designed to do so and fix related aspects, get it here: https://www.dropbox.com/s/u5raad4yrhi4ut9/fix-800706D9.bat?dl=1 (if you want feel free to read all of the contents in 'Edit' to understand all I put in there is safe to perform..)
