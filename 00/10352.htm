Create a new style for the paragraphs of text that is based on the relevant heading style.  That is for paragraphs number 2.1.1, 2.1.2, 2.1.3, etc., create and use a style that is based on the Heading 3 style.  It will then inherit the numbering from that style, including restarting from 1 after another Heading 2 style.

john w. vinson say:

Well...

 

Don't.

 

The whole POINT of relational databases is to store information once, and once only, and then display it when you need it. The Description should exist in the Parts table, and ONLY in the parts table. If you have it both places, it wastes disk (trivial these days), but much more important, opens you to having invalid data. You (or someone else) could edit the Description field in the main table, the parts list, or both, giving you one part with two different descriptions!

 

To just display the description on the form (without storing it in the main table), simply include the description as a column in the combo box and put a textbox on the form with a control source like

 

=comboboxname.Column(n)

 

where n is the zero based subscript of the field you want to display - first column is (0), second is (1).

ken sheridan say:
Unfortunately I really need to store the description in my Main Table ........

No you don't.  You simply join the 'main' table to the parts table on the PartNumber columns and return the Description and/or SerialNumber  columns from the parts table in a query.  The result table of a query is a table, it's just not a base table, and can be used as the RecordSource for forms or reports just as a base table can.  This is how a relational database works.

If you include Description and/or SerialNumber columns in the 'main' table you introduce redundancy, the table is not normalized to Third Normal Form as Description and/or SerialNumber are not determined solely by the whole of the table's key.  These basic rules of relational database design exist for a very good reason, and by violating them you undermine the integrity of the database by leaving a table wide open to update anomalies which result in inconsistent data.

If the parts represented in the parts table are subject to movement in and out of stock that should be represented by values in a column or columns in the table, or possibly even in a related table, depending on your business model, not by insertion or deletion of rows into the table, with the exception of course of the insertion of a row for a completely new part not currently represented by a row in the table.

john w. vinson say: Joey, part of your difficulty may be that you're making the very common beginner's mistake of working directly in the Table datasheet. If you do so, then yes, you do need the description in the table.

However, that's not necessary!

In a production database you'll likely never even SEE a table datasheet, much less need to edit it. Instead you will have a Form to interact with the data. The table will have a PartID, but you need not see the PartID; instead you will see a combo box with the PartID as its bound column, but displaying the Description to the user.

Forms have a rich set of tools (combo boxes, subforms, listboxes, on and on) to let you work with data in multiple tables without needing to "get your hands greasy" working directly with the tables and key fields; they should all be kept under the hood, doing their work efficiently but out of sight.

carolynnneville say:

microsoft office, powerpoint etc

I recently got a new laptop but it doesn't have power point, office, word etc. however, my old laptop did. is there a way I can transfer all of that to my new laptop? I think I have the cd but my new laptop doesn't have a dvd drive. Thanks :) 

rohn007 say:

It depends on 2 things.  Do you have a license that allows transfer/multiple installs and does your new laptop run a version of Windows that will allow you to install that version of Office.

It depends on the details of your earlier version.  Since you mention you have an install disk, you probably do have a "retail" license, which is a good thing.  Find your old install disk, the label on the outside should say "Retail" somewhere.  Find the "Product Key" sticker. Confirm that it has a "FPP" code in one corner.

If you do have a retail license and a product key code, then you can copy the install files from the disk to a USB drive (on some other computer).  Then you can copy the install files from the USB to your laptop then run setup.exe.
