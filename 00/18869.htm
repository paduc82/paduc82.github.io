ken sheridan say: A form does not pass a value to a query, the query references the control in the form as a parameter; it's pull not push.  The form must be open with a value selected in the combo box before the query is opened.  Usually the query, or more commonly a form or report bound to the query, will be opened by means of a command button in the dialogue form, with the code in the button's Click event procedure first checking that a value has been selected, e.g.

Const MESSAGETEXT = "Please select a value from the list."

If Not IsNull(Me.YourComboBox) Then
    DoCmd.OpenForm "YourFormName"
Else
    MsgBox MESSAGETEXT, vbExclamation, "Invalid Operation"
End If

The combo box's BoundColumn must equate to the column in the query to which the parameter is applied, e.g. if the BoundColumn is a hidden numeric key such as CustomerID, and the visible column a text column such as CustomerName, then the parameter must be applied to the CustomerID column in the query, not the CustomerName column.
