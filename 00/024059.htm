How to copy styles from an old, completed document to a new, empty document?

I recently published a book, and I am starting on another one. I would like to copy the styles from the final edited manuscript of the first book into a new, blank document in which I will begin to write the next book. I want to do this because the final edited manuscript of the first book contains all of the publisher's style specifications (American Psychological Association style), which would be excruciating to re-create manually in the new document.

What is the easiest way to reliably do this?

Thank you for your help.

jay freedman say:

Here are two methods, which I would consider equally easy. You may decide one is more suitable than the other.

  • Make a copy of the existing document and save it with a different file name and/or in a different folder. Open the copy in Word, press Ctrl+A to select all the text in the body of the document, and press the Delete key. You may want to keep the existing headers/footers and just change their text.
    • If you plan to write the whole new book in one document, you're ready to go -- the document already contains all the styles from the old book.
    • If you'll make multiple files, I recommend saving that empty document as a template (.dotx file) and using the File > New command to make new blank documents based on it -- they'll inherit the styles from the template.
  • Open a new blank document. Press Alt+F8, which opens the Macros dialog box, and then click the Organizer button on the right side. (You don't need to be concerned with macros; this is just the simplest way to get to the Organizer.)
    • In the Organizer, click the Styles tab. The left side contains a list of the styles that the blank document inherited from your Normal template, and the right side contains the styles in the Normal template itself.
    • Click the Close File button under the Normal template list, and when the button changes to Open File, click it again.
    • In the Open dialog, change the file type from "All Word Templates" to "All Word Documents". Go to the proper folder, click on the old book's document, and click the Open button. 
    • Back in the Organizer, select the styles in the old document that you want to copy (hold the Ctrl key to select multiple styles at one time), and click the Copy button between the lists. Styles with the same name in the new blank document will be overwritten with the definitions from the old document, and the other styles will just be copied to the new document.
    • Click the Close button, and save the new document. Again, you may save it as a template if you'll be making multiple documents from it.

    Getting Totals With Null Value Field - Microsoft Community

    foduk say:

    Getting Totals With Null Value Field

    I have read through several pages of previous postings but haven't yet seen the same question asked and aswered.

    I have a main form fm-VehicleData with a linked subform fm-RepairCost with fields [Part], [CostParts]
    A text box in the form footer has Sum(CostParts) which populates a field on the main form
    Cost of Repairs [=[fm-RepairCosts]![CostParts].
    I can't get the main form field to show any value until I enter something in the sub-form.
    I have tried =Nz([fm-RepairCosts]![CostParts,0) which does give a £0.00 value.
    However the sub form footer field remains blank and despite trying several Nz tries I keep getting a syntax error except Sum(Nz([CostParts]),0) which gives no error but remains blank.

    The end product I want is a query which shows data selected data from the main form and the Sum[CostParts] from the sub (or the same value from the main form field). My query works OK except of course if wont include any vehicle data if the linked sub form has no entry, the (default £0.00 has no effect).

    Sorry to be so long-winded but I have been struggling with this for hours and getting nowhere.

    ken sheridan say: You can join the Vehicles table to the Repairs table in a LEFT OUTER JOIN.  This will return all rows from Vehicles regardless of a match in Repairs.  The Vehicles table can be restricted on any of its columns, but note that you cannot restrict the query on the basis of a column in the Repairs table (other than a NULL, which would be used to return all vehicles with no repairs), so you could not for instance restrict the query to HAVING SUM(RepairCosts) < 500 say and still return those with no repairs.  The restrictioin on the Repairs table would in effect change the OUTER JOIN to an INNER JOIN, so only vehicles with matching repairs rows would be returned.

    A query to return all unsold vehicles, for which I've assumed a Boolean (Yes/No) Sold column in the Vehicles table, would be along these lines:

    SELECT RegNo, Make, Model, ManufacYear,
    SUM(NZ(RepairCost,0)) As TotalRepairCost
    FROM Vehicles LEFT JOIN Repairs
    ON Vehicles.VehicleID = Repairs.VehicleID
    WHERE NOT Sold
    GROUP BY RegNo, Make, Model, ManufacYear;

    where VehicleID is the primary key of Vehicles and the corresponding foreign key in Repairs.

    However, this query won't be updatable, so is not suitable as the RecordSource for a form which is to be edited.  For this you can call the DSum function in the query, e.g.

    SELECT *,
    NZ(DSUM("RepairCost","Repairs", "VehicleID = " & Vehicles.VehicleID),0) AS TotalRepairCost
    FROM Vehicles;

    or include a computed text box control in the parent form with a ControlSource property of:

    =Nz(DSum("RepairCost","Repairs", "VehicleID = " & [VehicleID]),0)

    vijay a. verma say:
    It is not possible to change default (and also have it changed inadvertently).

    Pivot table has its own inbuilt logic to decide whether to have SUM as default or COUNT as default. It makes its decision on the basis of available data in your data range on which pivot is based upon.

    In general, for number data, it chooses SUM as default and for non-numbers, COUNT.
