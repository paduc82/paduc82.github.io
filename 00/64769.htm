Please read the original Blog post about that vulnerability. 

While it is possible to inject code through that WiFi vulnerability, the main operating system needs some specific vulnerabilites as well, and in that Blog post, only Linux based systems are tested (i.e. the Steam Link device). 

While the Blog post mentions other devices that have this Marvell Chip built in, the author doesn't mention any tests or successful attacks on these devices. 

At least an "open" Windows 10 device like a Surface would have sufficed to show if the exploit would work on a Windows 10 device as well. 

Added to that, the Xbox One is a completely different "beast" even. While the main kernel of the Xbox One is also a Windows 10-like Kernel, everything on the Xbox One is running in Virtual Machine Hypervisors, and requires digitally signed code to be able to run on the system. 

So I'd assume it's pretty unlikely to be able to inject running code into your Xbox One through that WiFi vulnerability.

Of course, for websites that rely on advertising revenue that they get from user clicks, it's much more interesting to have a headline like "Xbox One and PS4 vulnerable for Marvell Firmware Exploit", than having a headline like "Steam Link Linux vulnerable to Marvell Firmware flaw. This is also built into Xbox One and PS4, but no one has proven these devices to be exploitable". 

So please calm down, and enjoy your gaming device a bit more. You see: For this exploit to be exploitable, an attacker would also have to be in the vicinity of your console, to reach your console's WiFi chip. 

Also you would have to have something "valuable" to make the effort to getting such an exploit running on the Xbox One. 

It's much easier and effective to be able to launch such an exploit against a normal PC to at least get access to "interesting" data like your emails, and other exploitable information. 

The only thing that hackers will probably try to exploit is: How do I possibly get my code to run on the Xbox One so I can play pirated games or mods on this thing. So you can still "safely" keep your WiFi in the Xbox One running.

The problem is with your hardware, not the OS.  Windows has no control over BIOS functions.  If the shutdown threshold is reached, the BIOS, not Windows, initiates the shutdown.  If Windows were able to override that call the result would be permanently damaged computer components and then people would be screaming about Windows not shutting down.
