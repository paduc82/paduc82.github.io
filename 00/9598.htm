marshall barton say:

I forgot to mention it before.  If the Update query does the job and if you have a dept that does not need to trim anything, you do not need a rule for that dept.  That means you do not need rule3 in the example rules.  Actually, it may be significantly slower to update records just to update to an unchanged value.

If you use an Appemd or Make Table query, then you do need a rule so every record is copied.

The following idea only applies to the Update query scenario.

Assuming you will not have too many rules for any one dept, you could create a test procedure to run a query that checks all the rules for a dept whenever a trim_if is changed/added.  Here's a shot at some air code for the trim_if text box's AfterUpdate eventthat might get you started:

Dim db As Datebase
Dim rs As DAO.Recordset
Dim strSQL As String
Dim strOverlaps As String
   Set db = CurrentDb()

   strSQL = "SELECT trim_if, Count(*) As overlap " _
             & "FROM tbl_temp_import_data As D INNER JOIN tbl_scan_rules As R" _
             & "ON D.Depot_ID_number = R.Depot_id " _
             & "WHERE D.Depot_id = " & Me.Depot_id
                  & " And HAWB Like R.trim_if And HAWB Like '" & Me.trim_if & "' "
      ' check if changing an existing rule or adding new rule
   If Not IsNull(Me.trim_if.OldValue) _
   Then strSQL = strSQL _
             & " And R.trim_if <> '" & Me.trim_if.OldValue & "' "
   strSQL = strSQL _
             & "GROUP BY R.trim_if " _
             & "HAVING Count(*) > 0"

   Set rs = db.OpenRecordset(strSQL, dbOpenSnapshot)
   If rs.RecordCount = 0 Then
      MsgBox Me.trim_if & " has no overlaps"
   Else
      Do Until rs.EOF
         strOverlaps = strOverlaps  & ", " & rs!trim_if
      Loop
      Beep
      MsgBox Me.trim_if & " overlaps " & Mid(strOverlaps, 3)
   End If

I think something like that is necessary to make sure the new trim_if is ok, but you may also want to provide another form/subform that just displays a depot's resulting HAWBs for a trial trim_if before it's even entered in the rules table.

I hope you understand that this kind of thing is a pretty advanced thing to be doing and the users wil need some assistance before they will be able to get a grip on what can be done an how to do it.  It would probably be a good idea to provide your users with a short class on how to use wildcards for tricky trim_if situations and to hand out a cheat sheet on wildcards even if it's just a copy of the Help topic.

I am not sure if I understand your question completely but it is possible to surround a VLOOKUP with the TEXT function.  the VLOOKUP becomes the text and then you can follow it with some formatting.  For example supposing the following VLOOKUP produced the answer 3.0 - VLOOKUP(A1,'Sheet1'!B2:D40,3,FALSE) - and you wanted it to read as £3.00.  You could do the following - TEXT(VLOOKUP(A1,'Sheet1'!B2:D40,3,FALSE), "£0.00").

jay freedman say:
None of the dropdowns available to be inserted directly within a document (the legacy dropdown form field, the ActiveX combo box, and the content control dropdown list box) support multiple selection.

To get that functionality, you must create a userform on which you insert a list box control, and set that control's MultiSelect property to the value fmMultiSelectMulti ("Pressing the SPACEBAR or clicking selects or deselects an item in the list") or fmMultiSelectExtended ("Pressing SHIFT and clicking the mouse, or pressing SHIFT and one of the arrow keys, extends the selection from the previously selected item to the current item. Pressing CTRL and clicking the mouse selects or deselects an item"). Then insert a MacroButton field in the document to run a macro that displays the userform and places the results in the correct place in the document.
