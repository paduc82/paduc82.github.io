david marcovitz say: Try this:

Sub AddPlants()
    Dim newstuff As String
    newstuff = InputBox("What is a plant sign of Spring?")
    If newstuff <> "" Then
        With ActivePresentation.SlideShowWindow.View.Slide _
            .Shapes("Content Placeholder 2").TextFrame.TextRange
            .Text = .Text & Chr$(13) & newstuff
        End With
    End If
End Sub
Sub AddAnimals()
    Dim newstuff As String
    newstuff = InputBox("What is an animal sign of Spring?")
    If newstuff <> "" Then
        With ActivePresentation.SlideShowWindow.View.Slide. _
            Shapes("Content Placeholder 3").TextFrame.TextRange
            .Text = .Text & Chr$(13) & newstuff
        End With
    End If
End Sub

This is from Example 7.13, which is soon to be on my Web site (http://www.PowerfulPowerPoint.com/) and from my book which is coming out this month. For this example to work, you need to have a slide with the layout for title and two content areas. Assign the AddPlants procedure to a button to add text to the first content area and assign the AddAnimals procedure to another button to add text to the second content area. If you leave off AddAnimals, you could use a regular title and one content layout.

This can be modified to work in with other layouts and other text boxes, but this should get you started.

robert__p say:

Hi Brian,

Based on your description, I understand you want to disable automapping of all the resource mailboxes for all users at one time.

We need to use PowerShell to disable automapping of one resource mailbox for all users at one time. Here are the steps:

1. Connect to Exchange Online using remote PowerShell.

2. Assume the resource mailbox is *** Email address is removed for privacy ***, then type 

$FixAutoMapping = Get-MailboxPermission *** Email address is removed for privacy *** |where {$_.AccessRights -eq "FullAccess" -and $_.IsInherited -eq $false}
$FixAutoMapping | Remove-MailboxPermission
$FixAutoMapping | ForEach {Add-MailboxPermission -Identity $_.Identity -User $_.User -AccessRights:FullAccess -AutoMapping $false}

Note: the scripts are applied to per resources mailboxes. That is to say, you need to run these scripts five times for 5 resources mailboxes one by one.

As for your requirement to disable it for all resources at a time, it might involve other PowerShell scripts. Since PowerShell scripts are not supported in our forum, I suggest you post the question in our PowerShell script Forum for professional assistance.
