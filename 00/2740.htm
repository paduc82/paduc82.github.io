Access data types

My Access database will contain two fields of numbers ranging from 1 to 11,000.  Most will be integers, but a few will have a single decimal digit, e.g., 5463.5.  What is the best data type for such a set?

albert d. kallal say:

If the numbers are to be used for any kind of financial calculations, then I would suggest you use currency type. The reason being that such numbers are not subject to rounding errors.

You could use a single or double type – but these are floating point numbers and like all computers they ONLY store approximate values.

Here is a VERY simple routine that in Excel or even Access VBA shows rounding errors after JUST a few additions:

Public Sub TestAdd()

   Dim MyNumber      As Single

   Dim I             As Integer

  For I = 1 To 10

      MyNumber = MyNumber + 1.01

      Debug.Print MyNumber

   Next I

End Sub

Here is the output of the above:

 1.01

 2.02

 3.03

 4.04

 5.05

 6.06

 7.070001

 8.080001

 9.090001

 10.1

So if your numbers are to be used for addition, or any type of financial calculations, you best use currency data type since it would not suffer from rounding errors – as noted in above after just 7 additions of a simple decimal value, rounding is occurring. And the above code EVEN shows that rounding when using Excel.

The bottom line: computers only store floating numbers as approximate and you thus suffer rounding errors quite easily.

Currency datatype does not suffer from rounding since the number is stored as a scaled integer (an integer value along with a value for the decimal part).

If the string has 2 commas this will pull out what is between them. If it has one comma it will pull out everything after the first. If you want something different ten post a few representative examples.

=TRIM(MID(SUBSTITUTE("," & A1&REPT(" ",6),",",REPT(" ",255)),2*255,255))

hansv mvp say:

You have to use straight quotes in formulas, not curly quotes. The formula

=FIND(",",A1)

should work:

Another way to split the string is as follows:

Press Alt+F11 to activate the Visual Basic Editor.

Select Insert > Module.

Copy the following code into the module:

Function SplitString(s As String, Optional sep As String = ",")
    SplitString = Split(s, sep)
End Function

Press Alt+F4 to close the Visual Basic Editor.

With the text containing commas in A1, select B1:D1 (or another range of three adjacent cells in a row).

Enter the formula

=SplitString(A1)

and confirm it with Ctrl+Shift+Enter to make it an array formula (Excel will display braces { } around the formula in the formula bar):
