What sort of Drop Down are you using?  If you are using a ContentControl, and you assign the tag Colours to the Control and the items in the list are Item 1 (Blue), Item 2 (Red), Item 3 (Yellow), then if you have the following code in the This Document module of the Visual Basic Editor, when you exit the cell containing the content control, it will be shaded in the relevant colour.

Private Sub Document_ContentControlOnExit(ByVal CC As ContentControl, Cancel As Boolean)

For Each CC In ActiveDocument.ContentControls

    If CC.Tag = "Colours" Then

        Select Case CC.Range.Text

            Case "Item 1 (Blue)"

                CC.Range.Cells(1).Range.Shading.BackgroundPatternColor = wdColorBlue

            Case "Item 2 (Red)"

                CC.Range.Cells(1).Range.Shading.BackgroundPatternColor = wdColorRed

            Case "Item 3 (Yellow)"

                CC.Range.Cells(1).Range.Shading.BackgroundPatternColor = wdColorYellow

            Case Else

                CC.Range.Cells(1).Range.Shading.BackgroundPatternColor = wdColorAutomatic

        End Select

    End If

Next

End Sub

john - project say:

Julie/Stephen,

Oops! It looks like my suggestion was off base. Indeed if leveling calculation is set to automatic, Project will go ahead and level based on the other settings, but that's not what we want. I think the whole issue is that the Resource Usage view is still showing resources as being overallocated (little red guy) and indeed they may be.

It took me a while to get my head straight on this but here is what I think Project is showing us. Setting the leveling option to week by week indeed will limit the leveling algorithm to only consider weekly overallocations, but that has nothing to do with a resource who may still have more work hours in a day than is supported by their max unit level. For example, let's say a resource has a normal 8 hour day and his/her max units is 100%. In one sharer file that resource is assigned full time to a task on Monday for 8 hours work. In another sharer file that resource is assigned half time to a task on Monday for 4 hours work. On the Resource Usage view of the resource pool file, that resource will as overallocated on Monday because he/she is assigned at total of 12h work. However, if the leveling option is set for week by week, and leveling is applied, the leveling algorithm will not attempt to do anything with that Monday task unless that resource has more than 40 hours assigned for that week.

So, if the goal is to visually see whether resources are overallocated on a weekly basis, I suggest the Work Availability field be displayed in the timescale (right) side of the Resource Usage view and set the timescale to be weekly. Check any "red" resources and compare the work and work availability. If the work is less than the work availability, that resource is good-to-go from a weekly standpoint although they may be overallocated any given day. If the weekly work exceeds the weekly work availability, then you need to take action to address that overallocation.

bob jones aka: cybertaz say:

When you use formatting any characters added by the formatting mask are not actually a part of the data. Accordingly, those characters will not display in the Formula Bar... Just as the currency symbol and the comma [ , ] imposed by Currency formatting or the dashes added by a Social Security Number format.

If for some reason you want the zeroes to appear in the Formula Bar you need to either;

  • Preformat the cells as Text & type the zeroes along with the other characters, or
  • First type an apostrophe [ ' ] followed by the UPC code, including the leading zero

The cell cannot retain the zero if formatted as General or any form of Number because leading zeroes are not significant digits in any number system.
