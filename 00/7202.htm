Daniel, I'm not sure I understand what you're saying.  You certainly can concatenate a Null to a String.  Furthermore, the '+' operator has a special feature when used for concatenation:  if one of the arguments is Null, the result is Null.  That's different from the action of the '&' operator, for which <String> & Null = <String>.

I  imagine that the reason for error 94 in this case is that the form's Filter property cannot accept Null, only an actual String value.  So in the case where varFilter is Null, this line:

    Me.Child52.Form.Filter = "[Net Cash Amt] " + varFilter

will reduce to this:

    Me.Child52.Form.Filter = Null

which will raise an error.  In the original code, I think this revision would work:

    Me.Child52.Form.Filter = ("[Net Cash Amt] " + varFilter) & ""

The concatenation of the Null with "" using the '&' operator will convert it to a zero-length string.

Whether the loop through ItemsSelected should have resulted in varFilter being Null is a separate question.
