STDEV multiple data ranges - Microsoft Community

nkrill say:

STDEV multiple data ranges

STDEV is able to accept multiple data ranges as in: =STDEV(e14:e23, f14:f23)

 

In order to use this function I need to verify its math. How are these ranges combined mathematically?

hansv mvp say:

Excel creates a list of all cell values in the specified ranges, and the standard deviation is calculated over all numeric values (non-numeric values are ignored).

 

So

 

=STDEV(E14:23,F14:F23)

 

is equivalent to

 

=STDEV(E14:F23)

 

and also to

 

=STDEV(E14:E20,F14:F18,E21:E23,F19:F23)

 

If there is overlap between the ranges, values will be counted twice (or more).

 

=STDEV(E14:F19,E17:F23)

 

is not the same as

 

=STDEV(E14:F23)

 

since the cells in E17:F19 will be counted twice.

mike h.. say:

Hi,

 

The numbers aren't combined, STDEV calculates the standard deviation based in the numbers being a sample of the population. If your number set is the entire population you should be using STDEVPA.

joeu2004 say: nkrill wrote:

STDEV is able to accept multiple data ranges as in: =STDEV(e14:e23, f14:f23)

In order to use this function I need to verify its math. How are these ranges combined mathematically?


The standard deviation is a common statistical formula, and any statistics book will tell you how it is calculated.  Also, the STDEV help page shows the formula.  However, there is a critical typo in the Excel 2007 help page that might make it difficult to understand the formula if you are not familiar with it.

Mathematically, assume the numbers in the ranges E14:E23 and F14:F23 are copied into an internal list x() with n elements.  For your example, let's call that list X1:X20.  Then assuming E14:E23 and F14:F23 contain only valid numbers, STDEV(e14:e23, f14:f23) is equivalent to:

=SQRT( SUMPRODUCT( (X1:X20-AVERAGE(X1:X20))^2 ) / (COUNT(X1:X20)-1) )

The SUMPRODUCT expression is equivalent to:

(X1-AVERAGE(X1:X20))^2 + (X2-AVERAGE(X1:X20))^2 +....

Note that for STDEV, the denominator is n-1.  That is because STDEV assumes the data E14:E23 and F14:F23 are a sample of a larger set of data.

If E14:E23 and F14:F23 are the entire set of data, use the STDEVP function instead.  And in the equivalent  formula, the demoninator should be n -- COUNT(X1:X20) -- instead of n-1.

The typo was caused by an anomoly in an early version of the community bridge which doubled up some characters :(
The following version will insert the formatted contents of a document called "D:\My Documents\Test\Page to be inserted.docx" starting a new page at the end of each document in the selected folder. It will not work if any of the documents are protected.
Save the page to be inserted as a document and change the path and filename of that document in the line
Set oPage = Documents.Open("D:\My Documents\Test\Page to be inserted.docx")
to reflect the name you have given that document.

Sub BatchProcess()
Dim strFilename As String
Dim strPath As String
Dim oDoc As Document, oPage As Document
Dim oRng As Range
Dim fDialog As FileDialog
Set fDialog = Application.FileDialog(msoFileDialogFolderPicker)

With fDialog
    .Title = "Select folder and click OK"
    .AllowMultiSelect = False
    .InitialView = msoFileDialogViewList
    If .Show <> -1 Then
        MsgBox "Cancelled By User", , _
        "List Folder Contents"
        Exit Sub
    End If
    strPath = fDialog.SelectedItems.Item(1)
    If Right(strPath, 1) <> "\" _
    Then strPath = strPath + "\"
End With

If Documents.Count > 0 Then
    Documents.Close SaveChanges:=wdPromptToSaveChanges
End If
Set oPage = Documents.Open("D:\My Documents\Test\Page to be inserted.docx")
If Left(strPath, 1) = Chr(34) Then
    strPath = Mid(strPath, 2, Len(strPath) - 2)
End If
strFilename = Dir$(strPath & "*.doc")
While Len(strFilename) <> 0
    Set oDoc = Documents.Open(strPath & strFilename)
    '
    'Do what you want with oDoc

    Set oRng = oDoc.Range
    With oRng
        .Start = oDoc.Range.End
        .InsertBreak wdSectionBreakNextPage
        .Start = oDoc.Range.End
        oPage.Range.Copy
        .Paste
    End With
    '
    oDoc.Close SaveChanges:=wdSaveChanges
    strFilename = Dir$()
Wend
oPage.Close wdDoNotSaveChanges
End Sub

mccbleue say:

Jimmyireland wasn't wrong, he did say to ensure that the user should ensure that all images and videos are stored on the card. You are right that it may be necessary to check the camera settings for the memory in use, but most phones that are supplied with a memory card are set to store there by default.

 

Another thing that we should be aware of is to check software that you use to transfer music and video to the phone (such as PC Sutie, Ovi Suite, Nokia Music etc. are correctly configured to store the necessary files to memory card instead of phone memory.

 

Contacts and SMS messages use only a minimal amount of memory and so should not be a convern on a device with as much phone memory as the 5800. The number of contacts you'd fit onto a SIM card is certainly not going to make a big difference, and SIM card sorage is only good for one name and one number per contacts - most of us are now used to stroing home, mobile, work numbers et cetera together.

 

One thing that is worth checking is if any Bluetooth messages are still sitting in the inbox - once the attached files have been saved to memory card, the message itself should be deleted.

 

When troubleshooting memory issues, the first line of defence ought to be to check the C: drive from the phone's File Manger (Options>Details>Memory) - this will tell you how much memory is free and in which areas large amounts of memory is being occupied.
