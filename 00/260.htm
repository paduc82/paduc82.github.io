Deleting All Non-English Characters - Microsoft Community

vlad6 say:

Deleting All Non-English Characters

I need to know how to remove all non-English alphabetic characters in a document. Is there a built-in function, or should a macro instruction be used?

jay freedman say:

You can make a reasonable start with a single wildcard replace (http://www.gmayor.com/replace_using_wildcards.htm). I'd recommend doing this on a copy of the original document, in case it deletes some characters that you don't want to remove, so you can start over with a different search expression.

First click the More button in the Replace dialog and check the box for "Use wildcards". Then enter the following expression in the Find What box:

[!A-Za-z0-9.,;:'"\(\)\-^13^+^t^g ]

The exclamation point at the beginning says "match any character that is NOT one of the following". That's followed by a list of characters that you want to keep: all upper case letters, all lower case letters, all digits, an assortment of punctuation characters (you can add to this list if your document includes others), parentheses and hyphens (each preceded by a backslash to say "the actual character, not the wildcard meaning"), paragraph marks and em dashes and tabs and graphics (preceded by a caret to indicate a spacial code), and spaces.

Leave the Replace With box empty and click the Replace All button.

albert d. kallal say:

A really neat way to tackle this type of problem when the talbes + data is different and you need to display it all in one report is to use what we call sub-reports.

 

So, try this:

 

Build a report based on the one query. Try to keep it simple, just some nice looking rows of data. Ok, save this report.

 

Now, build a report based on each additional query you have. So, you wind up with 6 or 7 reports.

 

Now, create new blank report, and drag + drop each of those reports into the detail section one after another. The result is all 7 reports will appear as one report.

 

Also, now that access has bult in PDF ability, you can take that one report and save it out as a PDF, and send that to everyone. The pdf not only is likey to be supported on other computers like Mac's, but ALSO will keep the nice formatting and graphics.

inserting images onto a MS Access report

Is it possible to insert an image that crosses sections in an MS Access report?  For example, if I have a report header, page header, name header, detail and page footer; i need my image to span all of those so that it shows up along the entire side of the page.  Here is a pic of what I'd like the final product to look like:

mimmo57 (d. laurenza) say:

ken sheridan say:

You can set the report's Picture property to an image file, setting its PictureSizeMode property to 'Stretch'.  If you design the image carefully so that it is plain white across most of its width, with detail only on the far left, it will then cover the whole page.  Make sure the controls in each section are indented so as not to encroach on the detailed part of the image to the left.

scottgem say:
When inserting an image on the Report level, you are essentially creating a watermark. That image will display on each page of the report. Do as Ken described.

mrsdopley say: it is still not crossing the different areas of the report, ie. if i insert into the page header, it stretches the header out instead of laying the picture under or over the entire page spanning the name header and detail sections...

scottgem say:
That's because it doesn't belong in the Page Header. You need it in the REPORT's Picture property as Ken instructed. Open the Report in Design View. Then open the Property dialog. In the combo on the top select Report. Then open the Format tab and select the Picture property and select the image file you want. 

ken sheridan say:

We are not talking about an image control here.  It's the Picture property of the report object which you need to set to an image file.  This property sets the background image of the entire page.
