Reposition the character before a find target - Word VBA Macro - Microsoft Community

tom soon say:

Reposition the character before a find target - Word VBA Macro

I have a working Word Macro in Word 2010 that loops through the entire Word document re-positions all instances of both the character before  a find target and the find target itself. Re-position means reduces the spacing between the character before and the find target, and raise the position of find target and reduce it's spacing between the character after:

The macro is as follows:

Sub repositionTarget ()
Dim vShift
Dim lastPos As Long
lastPos = -1
Selection.Find.ClearFormatting
Selection.Find.Text = InputBox("Enter the find target:")
Do While Selection.Find.Execute = True
    If lastPos > Selection.Start Then Exit Do
    With Selection.Font
    vShift = .Size / 4
    End With
    PunctFormat (vShift)
Loop
End Sub

Sub PunctFormat(hoffs)
Dim nhoffs, voffs, newPos, pSpac, pSize
nhoffs = -1 * hoffs
voffs = hoffs / 5
newPos = hoffs + voffs
With Selection.Font
   pSpac = .Spacing
  .Spacing = nhoffs + pSpac: .Scaling = 100: .Position = newPos
End With
Selection.MoveLeft unit:=wdCharacter, Count:=2, Extend:=wdExtend
With Selection.Font
  .Spacing = nhoffs + pSpac: .Scaling = 100: .Position = 0
End With
Selection.MoveRight unit:=wdCharacter, Count:=2
End Sub 

However, this macro no long works as expected in Word 2016!? It seems that when executes to the "Do While Selection.Find.Execute = True" line then it jumps to the end of the loop.

Any helps for why it doesn't behave as before and how to make it work in Word 2016 are greatly appreciated.

jay freedman say:

The command Selection.Find.Execute attempts to match the input text, including all other conditions in the Selection.Find object, to some place in the document body. Assuming that the characters in the input text do exactly match characters that you can see in the document, the likely problem is that some other condition in Selection.Find is not being met anywhere.

The point that isn't obvious is that all the conditions in the Selection.Find object are "sticky" -- after any find, either in a macro or manually in the Find dialog or the Navigation pane, they stay set that way unless your macro sets them back to their default values. The Selection.Find.ClearFormatting resets conditions related to text formatting, such as Bold or Underlined, but it doesn't touch a lot of others.

The easiest way out of this situation is to use a Range object in place of the Selection. When you allocate and assign a new Range object in a macro, it automatically begins with all conditions set to their defaults.

In your macro, this would involve starting repositionTarget with

   Dim rg As Range

   Set rg = Selection.Range

and then replacing all other instances of Selection with rg. You would also have to pass rg as another argument to the PunctFormat subroutine and replace Selection with rg in that code.

If you run into errors with this variation, or if it runs but continues to exit from the loop immediately, post back with the revised code and a small sample of the sort of text it should work with.

One further comment: There is nothing inside the loop that ever changes the value of lastPos, or any way for Selection.Start to become less than or equal to -1, so the If statement will never cause the loop to exit. The loop will stop normally when there are no more matches in the text (if .Find.Wrap = wdFindStop) or if the find reaches the end of the document and asks the user whether to resume at the beginning (if .Find.Wrap = wdFindAsk) and the user clicks No.

hansv mvp say:

In Word:

Replace paragraph breaks, and if applicable also manual line breaks, with a string that doesn't occur otherwise in the text, for example @@@@

In the Find what box in the Replace dialog: ^p

In the Replace with box: @@@@

Click Replace All.

Do the same with ^l (that is a lower case L) in the Find what box.

 

After copying and pasting into an Excel worksheet:

Replace the string you used with line breaks.

In the Find what box: @@@@

In the Replace with box: press Ctrl+J to enter a line break.

Click Replace All.

Make sure that you turn on "Wrap Text" for the relevant cell.

 

After copying and pasting into an Access table:

Create a query based on the table.

Add the field containing @@@@ to the query grid.

Change the query to an Update query.

In the Update to line, enter

 

Replace([FieldName], "@@@@", Chr(13) & Chr(10))

 

where FieldName is the name of the field.

Do this for each field that may contain @@@@.

Run the query.
