Work Around for Surface Pro 3 Issue Where Connected Standby is - Microsoft Community

barb bowman say:

Work Around for Surface Pro 3 Issue Where Connected Standby is Disabled when Hyper-V is Active

Technical Level : Intermediate

Summary
If you install Visual Studio 2013, there is a conflict between Connected Standby and Hyper-V. Connected Standby is no longer operational. This article provides a work-around.
Details

The Surface Pro 3 is the only Surface “Pro” device that supports Connected Standby. Surface Pro (original) and Surface Pro 2 do not support this power management protocol.

Developers who install Visual Studio 2013 may notice that after installing VS 2013 that their Surface Pro is no longer performing in the same manner and that Connected Standby no longer works. This occurs because Connected Standby is not supported on a system running Hyper-V virtualization.

The emulator for Windows 8 Phone development in Visual Studio 2013 requires Hyper-V.

A current work-around to restore Connected Standby functionality is to use BCDEDIT to turn Hyper-V off when not needed and on again when you need to use the VS 2013 emulator to test Windows Phone Applications. If you are not developing phone apps, you can turn Hyper-V off and leave it off.

Steps to Turn Off Hyper-V:

  1. If you have installed Visual Studio 2013, verify that Connected Standby is on or off by opening an administrative command prompt and typing

              powercfg /a

                As you can see in the screen capture below, Standby (Connected) is NOT supported. (The entry"The  hypervisor does not support this standby state" indicates that Hyper-V is active and the Connected Standby is not available).

2. Turn Hyper-V off by typing the following into the command window:

bcdedit /set hypervisorlaunchtype off

3. Restart your Surface Pro 3 (You must restart for the new configuration to become active)

After the Surface Pro 3 has been restarted, Connected Standby should be active again. You can check this by opening an administrative command prompt and typing

powercfg /a

As you can see, Standby (Connected) is now available which is the default configuration for Surface Pro 3.

Steps to turn on Hyper-V:

1. Open an administrative command prompt and type:

bcdedit /set hypervisorlaunchtype auto

2.  Restart your Surface Pro 3

After restarting, Hyper-V will again be active (and Connected Standby will no longer function). You will be able to use the Phone Emulator in Visual Studio 2013.

Un-Installing and/or Change Default Apps in Window 10 - Microsoft Community

thatguy7115 say:

Un-Installing and/or Change Default Apps in Window 10

THE THREE TYPES OF DEFAULT APPS:

First of all if you are trying to DELETE a default app then read below but if you are trying to change the default app that you set for an app or a program then scroll until you read the all-caps bold heading that reads "Unbinding Default apps from programs".

anyways, there are three types of apps that are considered default.

The first type are Apps that can be uninstalled regularly by just pressing the uninstall button and these apps includes:

Skype, Get Started, Office, Microsoft Solitaire, Money, News, Sports, Windows Media Player, and other apps that are similar to these.

The second type are Default apps which are considered main or necessary and can only be uninstalled by PowerShell like:

Camera, Calendar, Mail, Maps, Movies & TV, People, Photos, Store, and other apps that are similar to these.

The Third type are Default apps which cannot be uninstalled at all and I'm sure that is pretty obvious why once you read their names:

Cortana, Microsoft Edge, Windows Feedback and apps similar to those.

----------------------------------------------------------------------------------------------------------------

UNINSTALLING APPS WITH POWERSHELL:

to delete the default apps that require PowerShell (the ones I listed above in the second type of Default apps) just follow these steps:

1. Open the Start menu and go to All apps > Windows PowerShell and right-click Windows PowerShell. Choose Run as administrator from the dropdown menu. A window will pop up asking if you want to allow this app to make changes to your computer - click Yes to continue.

2. In the PowerShell prompt window, type in the following command to remove the app you want, in my example I'll be using the 3D Builder:

Get-AppxPackage *3dbuilder* | Remove-AppxPackage.

3. Press Enter to complete the process.

and it's as simple as that, don't worry I will provide you with a list of all the DELETABLE apps that you can uninstall using PowerShell.

THE DEFAULT APPS LIST:

Calculator: windowscalculator

Alarms and Clock: windowsalarms

Calendar and Mail: windowscommunicationsapps

Camera: windowscamera

Groove Music: zunemusic

Maps: windowsmaps

Movies & TV: zunevideo

OneNote: onenote

People: people

Photos: photos

Store: windowsstore

Voice Recorder: soundrecorder

Xbox: xboxapp

----------------------------------------------------------------------------------------------------------------

UNBINDING DEFAULT APPS:

Whether you downloaded something and you were asked to select a program to open the file with but you accidentally selected to make that app your Default app for opening the type of files you downloaded or you simply want to change the Default app you use to open a specific type of file then continue reading.

THE FIRST METHOD:

this method is used to change a default app that is linked to a file type: (for example you set WinRAR as a default app to open .lua files:)

1.Press the windows button and i at the same time, this should open up the settings.

2.Go to system, then Default apps, then scroll all the way down and choose choose default applications by file type.

3.Scroll down until you find the file type you want to change, in my example I'll be using .lua.

4.Click on the Default app that is set to .lua which in my example is WinRAR.

5.When you click on WinRAR a drop-down menu should appear asking you to choose an application.

6.Click on the application you want to change to or search for one using the store and Ta-Da you have changed the Default app that is set to a file type.

THE SECOND METHOD:

This method is used to re-set the Default app which is set to an Application: (for example you downloaded a file and set WinRAR as that file's Default app)

1.OpenI File Explorer.

2.Go to the Directory which has the application you want to change (In my example it's ThisPC>Downloads).

3.Right click on the app you want to change then click on open with then select choose another app.

4.A drop-down menu should appear asking you to choose an application.

5.Choose that application and tick the box saying to make this your Default app.

6.Now whenever you open that app it will use the Default app you re-set it to.

elajan7 say:

There are a lot of installations of Microsoft visual C++ Redistrbutable 2008 in various versions why this lot of installations if i remove the olders like 2005 and 2008 and keep 2010 only what will happens? please give a correct answer
joedissmeyer say: I have plenty of "Microsoft Visual C++ Redistributable" installations from various versions on my Windows PCs as well. The reason why there are so many is due to the various applications on the computer. Some of the applications require a very specific version of the Visual C++ runtime, either it be 2005, 2008, or 2010 versions. This is just one reason as to why there are so many installed. Another reason is due to Microsoft Updates. If you have it enabled on the computer, newer versions of that runtime will be installed without removing the older ones (similar to Java Runtime installs where the new one installs itself but never removes the old versions).

If you start removing the runtimes, it is possible that you may break one or many of your applications. You will need to do your research on the software requirements of the applications installed on your computer. 

In my opinion, just leave the runtimes installed. Do not remove them. I say this because leaving the runtime installations alone on your computer will not hurt it.

You can burn a disc image file, which often has either an .iso or .img file name extension, to a recordable CD or DVD by using Windows Disc Image Burner. Whether you can burn it to a recordable CD, DVD, or Blu‑ray Disc depends on your disc burner and the type of discs it can burn, the size of the disc image file, as well as the device on which you plan to use the disc.

 

1)       Insert a recordable CD, DVD, or Blu‑ray Disc into your disc burner.

2)       Drag the mouse to bottom left corner and right-click on the ‘Start icon’, and then click on Computer.

3)       In Windows Explorer, find the disc image file, and then double-click it.

4)       If you have more than one disc burner, from the Disc burner list in Windows Disc Image Burner, click the burner that you want to use.

5)       (Optional) If you want to verify that the disc image was burned correctly to the disc, select the Verify disc after burning check box.

6)       If the integrity of disc image file is critical (for example, the disc image file contains a firmware update), you should select this check box.

7)       Click Burn to burn the disc.

 

Notes

·         If a third-party CD or DVD burning program is installed on your computer, that program might open when you double-click the disc image file. If this happens, and you want to use Windows Disc Image Burner to burn the CD or DVD from the disc image file instead, right-click the disc image file, and then click Burn disc image.

·         You can't create disc image files using Windows Disc Image Burner. To create a disc image file, you need to install and use a third-party CD or DVD burning program or other program that lets you create disc image files from a CD or DVD.
