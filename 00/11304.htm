Can't Open a Word Document in Windows

http://word.tips.net/T000316_Cant_Open_a_Word_Document_in_Windows.html

Summary: Double-click on a Word document while using Windows Explorer, and the Word program should start with the document displayed. If it doesn't, then something in either Word or, more likely, Windows is out of kilter. This tip leads you through the different configuration areas you can check in order to track down and fix the problem. (This tip works with Microsoft Word 97, Word 2000, Word 2002, and Word 2003.)

 

Program Opens without the Desired Document

http://word.tips.net/T006832_Program_Opens_without_the_Desired_Document.html

Summary: Double-click a document link in the recent documents list maintained by Windows, and you may be surprised if Word doesn't open the document. There is a reason for not opening such a document, most likely that Word can no longer find the document. (This tip works with Microsoft Word 97, Word 2000, Word 2002, Word 2003, and Word 2007.)

Ed notes that on occasion he'll select a Word document from the recent documents list in Windows. Word then loads, but the document itself won't open. Ed wonders why this occurs.

If this happened with each document, no matter which one you clicked on, then the problem would likely be with how Word is registered with Windows. (In other words, what Windows thinks should happen when you double-click a document.) Since it occurs only on some documents, then the problem more than likely lies elsewhere.

The likely culprit is that if the document has been renamed, deleted, or moved to a different folder, the link in recent documents list maintained by Windows will no longer point to the document. If you double-click the link, then Windows tries to open the document at the old, pre-move location. Since it can't find it there, it leaves Word open, but does not load the unfound document.

answer finder say:

Search for records on a subform

I have a main form (Organization) and a subform (People).  These are tied to tables with the same names (i.e., tblOrganization and tblPeople).  There can be many people who work for a particular organization.  Sometimes I have a name and I don't know which organization that person works for.  I want to be able to search the PersonName field in the People subform and then go to that record.

 

I have a combo box on the main form that will search OrganizationName field in the Organization table and displays that organization.  This works for OrganizationName, but I can't get the same set up to work for PersonName.

 

Thanks

marshall barton say:

Well, I still don't see anything that could have caused that error, but I see you made a change where you shouldn't and I did not realize the primary key and foreign key fields are OrganizationName, which implies they are Text fields, not Number fields.

 

I'll try to make the changes I can see are needed:

 

Private Sub SearchPeople_AfterUpdate()
Dim varOrgID As Variant

     'Get org ID for person

varOrgID = DLookup("OrganizationName", "PersonTable", "PersonName=""" & SearchPeople & """ ")

If IsNull(varOrgID) Then
    MsgBox SearchPeople & " not in the table"
    Beep
    Exit Sub
End If

    ' move to person's org in main form

With Me.RecordsetClone
   .FindFirst "OrganizationName = """ & varOrgID & """ "
   If Not .NoMatch Then Me.Bookmark = .Bookmark
End With

    ' move to person's record in subform

With Me.People.Form.RecordsetClone
   .FindFirst "PersonName = """ & SearchPeople & """ "
   If Not .NoMatch Then Me.People.Form.Bookmark = .Bookmark
End With

End Sub

terfar say:
Basically a Table MUST have a final paragraph marker because it contains all the formatting information about the table. The best workaround is to select the marker and set it to 1 pt.
 
You can merge two tables together by deleting the paragraph mark between them but they will merge only if the tables are identical (columns line up, etc.).

VISTA WMDC to ASUS MyPal 696 Sync Connection Problem
