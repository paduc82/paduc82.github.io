robertjwin say:

So if I'm understanding what you're saying, the OS basically selects files - for example, really big ones - that it won't compare, whereas it will compare other files (the ones it counts in the checkbox above - the ones it's confident of are identical)

Nope, this is where you seem to be confused. The OS is not 100% confident that any pair of files is 100% the same content-wise. Small or big files, 10 or 1000 pairs, doesn't matter. It will only compare names, dates and sizes for all file pairs, that's it, because retrieving those is not resource intensive.

Remember that we aren't just talking about copying/moving between internal disks here. It could be internal to slow external, between two systems over a slow network and so on and so forth. Basically, the OS will just not waste precious resources or the user's time doing a content comparison. It will just blindly show the user all the files with name conflicts, and of those the user can choose with the checkbox to skip 2.a. files that he's personally sure are content-wise the same. Consequently the user's left only with 2.b. files in the list.
