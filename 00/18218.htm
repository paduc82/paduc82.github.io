Nesting a DMax() in the 3rd argument of a DLookup() - Microsoft Community

vwp1 say:

Nesting a DMax() in the 3rd argument of a DLookup()

I am not getting the appropriate figure, with the following:

=DLookup("pstg_amt_","pstg",DMax("pstg_as_of_d","pstg"))

which is essentially the way I can retrieve the most recent cost of 1st class postage for a letter.  We all can assume the cost will never go down, but I need to construct this...as though it may do so.

SELECT DISTINCT DLookUp("pstg_amt_","pstg",DMax("pstg_as_of_d","pstg")) AS crrnt_cost_z
FROM pstg;

SELECT DLookUp("pstg_amt_","pstg",DMax("pstg_as_of_d","pstg")) AS crrnt_cost_z
FROM pstg;  gets me all teh records, rather than the one I seek

I am receiving the oldest cost in my listing...not the most recent.  Why?
ken sheridan say: If I understand correctly, the txt_p_mail_lst_pstg_ text box is bound to a column in the form's underlying table, so to assign the latest 1st class postage value to it you'd have this as the AfterUpdate event procedure of the control in which you select a person:

Private Sub txt_p_mail_lst_prsn__AfterUpdate()
    Me.txt_p_mail_lst_pstg_ = DLookup("pstg_amt_", "YourQueryName")
End Sub

To change the current postage rate you'd need to insert a new row into the pstg table with a new date and the new amount, which you'd do in a form bound to the table, opened in dialogue mode.  As this causes post execution to pause until the dialogue form is closed, you can then follow this with the same code as above to assign the new value to the txt_p_mail_lst_pstg_ control.  No code is needed in the dialogue form's module.  So the code in the button's Click event procedure would be along these lines:

' open postage form in dialogue mode at a new record
DoCmd.OpenForm "frmPostage",DataMode:=acFormAdd, WindowMode:=acDialog
' assign new value to txt_p_mail_lst_pstg_ control
Me.txt_p_mail_lst_pstg_ = DLookup("pstg_amt_", "YourQueryName")

where frmPostage is the name of the form bound to the pstg table.

ken sheridan say: The criterion argument has to be an expression which evaluates to True.  I assume that the pstg_as_of_d column is a date/time data type, so the expression would have to compare the value of this with its latest (maximum) value.  The issue is complicated by the fact that a date literal must be in US short date format, or an otherwise internationally unambiguous format.  The ISO Standard for date notation of YYYY-MM-DD is a good choice.  So we end up with:

=DLookup("pstg_amt_","pstg","pstg_as_of_d = #" & Format( DMax("pstg_as_of_d","pstg"),"yyyy-mm-dd") & "#")

In a query you can do it by far more simply by means of a subquery in the WHERE clause:

SELECT *
FROM pstg
WHERE pstg_as_of_d =
    (SELECT MAX(pstg_as_of_d)
     FROM pstg);

or by a join:

SELECT P1.*
FROM pstg AS P1 INNER JOIN
    (SELECT MAX(pstg_as_of_d) As LatestDate
     FROM pstg) AS P2
ON P1.pstg_as_of_d = P2.LatestDate;

Note how the table and the result set of the subquery are differentiated by the aliases P1 and P2.  It's only really necessary for the latter to be given an alias, but I personally find it a little easier to read a query like this if both are aliased.
