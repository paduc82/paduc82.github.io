lemp say:

It seems as if you may have several issues.

With respect to the cables, it sounds as if the bottom drive in the new (working) tower is a SATA drive and the top drive (the one from the non-working computer) is a PATA/IDE drive.  That's OK, but is there another drive, e.g., a CD or DVD drive, connected to the same ribbon cable that connects to the top drive?  IIRC, the drive set as "master" should use the connector at the end of the ribbon cable and the drive set as "slave" should use the connector in the middle of the cable.

Cabling and master/slave issues aside, Disk Management shows that both the D: partition on the original drive and the F: partition on the transferred drive have no file system.  The D: partition probably is a "hidden" recovery partition provided by the manufacturer of the (new) tower (although it seems a bit large for that). 

Your specific problem is that Windows does not recognize any file system on F:, which is why it is offering to format it.  This suggests that the boot sector and/or the master boot record on F: is damaged. If you have a Windows XP CD, you can boot the new tower to the Recovery Console, from which the FIXBOOT and FIXMBR commands are available.  SeeKB314058 .  These commands are, however, capable of screwing things up even further if not used correctly, so you probably would be better off using data recovery software such as Recuva to access the files on F:
