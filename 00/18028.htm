Select the cells with the dates (A1:A30 in the example). I'll assume that A1 is the actice cell within the selection.

robert aldwinckle on forums say:
when i trying to do the thing to find the problem it says there is not problems found


Sounds like a problem with redirect or a stale DNS cache.   Try using the new IE9 Network Capture to check out that possibility; otherwise use Fiddler2 for its more complete trace.   You could also use Fiddler's  Request Builder  tool to eliminate IE from the symptom description.   Alternatively, simulate an HTTP request using  telnet  login.live.com 80  and then  GET /   (e.g. in a cmd window).   You can clear the DNS cache in an elevated cmd window with  ipconfig  /flushdns
However, that doesn't prove that the IP address you are getting is not stale.  In that case you may just have to wait or call your ISP for help.   Another thing to note is that many microsoft host names are aliases of caching servers.  FWIW my Nslookup shows that this one is no exception.  So, in addition to getting addressaility to another host via HTTP redirect your DNS would also have to have addressability to that host's canonical host just in order to get an IP address for it.   Etc.  I'm not sure how much of all of that the W7 Network Troubleshooter tries to do before saying "no problems found".  At least it tries to do it with the host name that you are having trouble with.  But most likely it probably only does an ICMP (e.g. ping) request there which does nothing to prove connectivity with an HTTP server at that address, let alone whether any redirects which may be done via HTTP can be satisfied by your DNS.
