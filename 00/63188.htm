  1. Create a new folder for the sent items.

¡firedog say:

It's a minor bug. The message list entries are extracted from the message database, while the displayed message is taken from the .eml file itself. As a message is imported, its headers (From:, To:, Subject: and Date:) are written to the message database. This is where the error creeps in for some reason. I didn't say anything about corruption or integrity of the imported messages.

When a message is dragged from the WLMail interface to an Explorer folder, the program creates a new file with the subject of the message as its name and eml as its extension. If the subject happens to end in a period, which it often does, the filename would be subject..eml, which is highly likely to give problems in some circumstances. WLMail helpfully converts one of the periods to an underscore character, but unfortunately picks the wrong one to convert.

If you then try to drag a bunch of .eml files into the WLMail interface, if just one of them has the extension ._eml, the process will fail.

If you drag a selection of properly-named .eml files into a WLMail folder, they will again be indexed, but this time without the From/To confusion.

The wlmail.fol file is a simple text file telling the importing program the name of the folder. Deleting it will simply mean that the importing program won't know the name of the folder it's importing from.

Trust me - the drag-out/drag-back procedure has worked for all those who've asked about this problem before.
