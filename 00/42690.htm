yusufat say:

Hi Uma..

Now that you mention Turbo C, it is a 16-bit software.

I want to share a little knowledge about compatibility mode, but correct me if i were wrong.
First, Intel or AMD processors have several modes to operate. Real mode, first introduced in Intel 8086 once upon a time, operates in 16-bit.Protected mode introduced in Intel 80286. At that time there was no way to return to real mode once the CPU entered protected mode, except by resetting the processor. Intel 80386 featuredVirtual 8086 mode, that is, ability to switch back to real mode and then return to protected mode. Long later, AMD Opteron featuresLong mode, that is, operates in 64-bit.

If the previous paragraph is too technical, I am sorry. Now, in windows XP or Vista or 7 (32bit), Windows normally operates the CPU inprotected mode, that is, 32-bit. Next, Windows emulates 16-bit environment using its component called NTVDM (NT Virtual DOS Machine). NTVDM relies onvirtual 8086 mode feature from your CPU. That is why Borland Turbo C (16-bit) can run on Windows XP 32-bit, because virtual 8086 mode is available in protected mode.

When you meet Windows 64-bit, Windows operates the CPU in long mode. In Windows 64-bit, there is no NTVDM. Why? BecauseVirtual 8086 mode is not available in long mode. Therefore, Windows 64-bit cannot emulate 16-bit any longer. But Windows 64-bit are able to run 32-bit software because 32-bit instructions are compatible with 64-bit (x64) processor architecture.

Workaround for this disability is to install 16-bit softwares under virtualization software, such as Windows XP mode (not available in 7 Home Premium), or a free softwareDOSbox.
