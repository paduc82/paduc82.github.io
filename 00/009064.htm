5.  Then you can assign your users the permission no delete for the library. Go to the SharePoint library, click Library in ribbon > Library settings > Permissions for this document library.

6. Click Grant Permissions in the Library Permission page. In the share page, assign the user permission which you created.

graham mayor say:
As you have now realised PDF is not an editable format and it cannot be opened by Word.

System restore in Windows XP does not affect data so your work should not have been lost. It does however correct registry changes so the work you have done will not be listed in the recent files of Word. Nevertheless the files that have been created should be present on the hard drive where they were saved.

You could have fixed the problem by re-associating PDF with Adobe Reader and re-registering Word - see the start of http://www.gmayor.com/Toolbars_in_word_2007.htm without the need to restore.

Arguably the best way to get PDF in an editable format is to use PDF compatible OCR software such as Abbyy Finereader, but PDF was not intended to be editable so getting a perfect match is not a five minute job.

Depending on how the PDF was created, you may be able to copy and paste text to Word from Adobe Reader.

If you have Acrobat, depending on how the PDF was created you may be able to save as Word format from Acrobat, but the results are not as successful as OCR and usually require a lot more editing.

A one page PDF can be inserted into Word as an object (a no editable pictorial representation) but as objects cannot span pages, you are limited to using one page PDFs in this manner.
