jeffhubb_ say:

Folbo has a corrupt data file: I doubt repairing the Office installation is going to fix this.

What do you mean by "bombing out" when you try to repair the file?  OneNote shows an error message?  Or OneNote just crashes?  You might want to try the alternate repair method stated inKB 825779.  If you only have OneNote 2010 installed and the notebook is on your local hard drive, you can use these simplified steps to repair the file:

  1. Exit OneNote.
  2. Find the corrupt section file via Windows Explorer (probably in Documents-->OneNote Notebooks).
  3. Make a backup copy of this file just to be safe.
  4. Shift+right-click the original file and select "Copy as path".
  5. Click Start and select Run.
  6. Type onenote.exe /forcerepair, then a space, then Paste the path to the corrupt file.

As for the "watermark" you're describing, try pressing Windows+S instead of going through the Insert menu.  When you select Insert-->Screen Clipping, OneNote minimizes itself to get out of the way.  Vista & Windows 7 animates this minimize action, but when OneNote gets the content of the screen it gets the last frame of animation which is the "watermark" you're describing.  I believe this is a bug in the video card not updating the screen buffer in time as I've only seen this happen on certain machines.  Windows+S won't minimize OneNote, so first select the program you want to screen-clip before pressing Windows+S.

As to your Taskbar question, right-click an empty portion of the Taskbar and select Properties (or right-click the Start button).  On the Taskbar tab, there's a box labeled "Taskbar buttons" that will default to "Always combine, hide labels."  Try selecting a different option and see if this helps.
