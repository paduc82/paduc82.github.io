sausagecreature say:

EXCEL (O365) =NETWORKDAYS very confusing

However, if my date range covers more than one weekend =NETWORK days refuses to calculate the number of working days correctly.

Any ideas?

Thanks

bernie deitrick say:

I'm not looking to get into an argument either - I just want you to understand that the function works and your interpretation of that is incorrect.

The reason that you thought NETWORKDAYS worked for just one weekend is that NETWORKDAYS, when given the same start and finish date, will return 1. WORKDAY, when given 1 as the second parameter, with return the NEXT workday. So, because you were subtracting 1 from the final date, you were actually accounting for 2 days (the 1 + the difference in how the functions work). Those 2 days - the length of a weekend - are why you though NETWORKDAYS worked for one weekend but not two or more.

To see how they function, enter a date into cell A2. In cell B2, enter a duration, between 0 and how ever many days you want. In C2, enter the formula

=WORKDAY(A2,B2)

and format as a date.

Then enter this formula in D2:

=NETWORKDAYS(A2,C2)

No matter what value you enter into cell B2, the value returned in D2 will always be one greater than the value in B2. (Unless you enter a negative value, in which case the NETWORKDAYS function will return a value that is more negative by one.)

That is how those functions work - there is no error: except for the weird early 1900s issue, dates and date math in Excel work

flawlessly.

IF you ever need to change the workweek from M-F, then take a look at the WORKDAY.INTL function - you can specify which days are work days and which are not, using an array of 1s and 0s.

https://support.office.com/en-us/article/workday-intl-function-a378391c-9ba7-4678-8a39-39611a9bf81d

Calendar (This Computer Only) - Microsoft Community

neil13764 say:

Calendar (This Computer Only)

I have a new PC with Win 10 and I installed Office 365 ProPlus.  I have a "Calendar (This Computer Only)".  I have a Imap account and an Exchange account. I thought in the past that I delete the ost file for the Imap account and that would resolve the problem but it is not working.

I would appreciate your help to eliminate the local calendar

karl timmermans (outlook mvp 2012-2018) say:
>>
I thought in the past that I delete the ost file for the Imap account and that would resolve the problem but it is not working.
<<

Hello Neil

The issue is the sequence in which the accounts were configured. Specifically, in this case, the IMAP email account was configured first making the .ost file for the IMAP account the "default data file" where the default contact/calendar folders are stored. In the case of IMAP, this would add "This Computer Only" contact/calendar folders to the IMAP folder tree.

Given that you don't want these folders, am assuming that there is no data in those folders that you want to keep. In the off-chance that you do, would need to export those folders to a PST file.

The quickest and easiest solution to this is the following

#1 - With Outlook closed, open the Control Panel <Mail> app to create a <New> profile (no need to delete the old one at this point

#2 - Configure the Exchange account first which will also make the OST file for the account the default data file for the profile

#3 - Then configure the IMAP account (no contact/calendar folders will show up in the IMAP folder tree

#4 - Be sure to set the newly created profile as the default for the next time Outlook is opened.
