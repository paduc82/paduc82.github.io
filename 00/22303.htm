macropod say:

The problem you're having with your layout is that you're trying to achieve with tabs what is meant to be achieved with indents. The two are quite different.

When you use a paragraph with an automatic bullet, anything you insert into that paragraph goes after the bullet. The only way to achieve your 'indented' bullets with tabs would be to forego the use of Word's automatic bullets and actually insert the bullet characters as part of the paragraph's text. That way, you could insert your tabs before the bullets. Although one can indent the bullets through the GUI, that process is not available programmatically - you would have to add code to do the indenting and change the bullet format. The GUI can interpret a tab at the beginning of a bulleted paragraph as a command to execute the .ListFormat.ListIndent method, but inserting a tab into a text stream input via code doesn't have the same effect. Besides which, if it did, you'd then need something equivalent to Shift-Tab you could insert into your code to execute the .ListFormat.ListOutdent method.

The better approach, IMHO, is to define a Style for each bullet level you want to apply, with the appropriate indenting, then apply those Styles, as appropriate to each of the paragraphs to be bulleted. Even if you can't modify the document template (so as to add the Styles to it), you can still add the Styles to the actual document - the former would probably be a one-off exercise done manually; the latter would be done programmatically (once for each document).

An object's Requery method reloads the object's recordset.  It differs form the Refresh method in that the latter merely reflects any changes to the data in the current recordset, so does not allow for any addition to, or removal of, rows from the recordset.

When correlating controls by means of a parameter in control B's recordset which references the value of control A, it is necessary to load control B's recordset as, by virtue of the parameter, control B's recordset will now return a subset of rows which differs completely from its previous recordset, hence its Requery method must be called.
