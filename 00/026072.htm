suzanne s. barnhill say:

Do you have Different First Page enabled? If so, then the strong likelihood is that the page numbers are just typed, not actual PAGE fields. Press Alt+F9 to display field codes. If you don't see { PAGE } for the page number in the footer, then the number is just typed.

The 5 and hyphen will probably have to be typed in any case (because there is no "chapter number" to be included), but you can select the 3 in the First Page Footer and the 4 in the Footer and press Alt+Shift+P to insert a PAGE field. You'll then need to use the Page Number Format dialog to start the page numbering at 3. Access it via Header & Footer Tools | Design | Header & Footer | Page Number | Format Page Numbers...

kergga say:

Date/times are stored as long. If you are also storing time, round first.

Then, consecutive days will always have a difference of 1

NewDate-OldDate=1

so to count your non-consecutive dates, you can just say

=if((NewDate -OldDate)>1,1,0)

and total up that column (how many 1's shows how many nonconsecutive)

However, the probably is rarely that simple in real life- are you considering a Friday and Monday as consecutive? What about the day before and the day after a holiday? If so, take a look at the NETWORKDAYS formula

e.j. gun say:

You might not need a VBA solution.  You can get what you want with a helper column and then a separate table to summarize total absence events for each employee.  I describe how below.

First, I assume your data is in Columns A to C (Date, Employee Name, Absence Reason)

I use Column D as a helper column for the formula below.  This formula will cumulatively count the number of absence events for an employee, and will count consecutive dates as only a single event.

In D2, enter this formula (it's ugly and long, but it seems to work).  Then drag down for all of your data:

=IF(IF(COUNTIF($B$1:B1,$B2), LOOKUP(9.9E+307,1/($B$1:B1=B2),ROW($A$1:A1)),"None")="None",1,IF(OFFSET($A$1,IF(COUNTIF($B$1:B1,$B2), LOOKUP(9.9E+307,1/($B$1:B1=B2),ROW($A$1:A1)),"None")-1,0,1,1)=A2-1,OFFSET($d$1,IF(COUNTIF($B$1:B1,$B2), LOOKUP(9.9e+307,1/($B$1:B1=B2),ROW($A$1:A1)),"None")-1,0,1,1),OFFSET($D$1,IF(COUNTIF($B$1:B1,$B2), LOOKUP(9.9E+307,1/($B$1:B1=B2),ROW($A$1:A1)),"None")-1,0,1,1)+1))

I hope I got that right.  For some reason I cannot cut and paste into this editor, so I have to do it the hard way - typing it in.

So now in Column D, you will have a running total of absences for each employee, looking something like the example below.  Note how for Jim Jan 6-7 is only counted once.

Now you have the absences accumulated.  It would be nice to have a separate table that lists each employee once, and then lists the total number of absences to date.  I put this table into Columns F and G, as shown below.  The formula to use is given below the image.

In cell G2, add the following formula and drag down.  Adjust the size of the ranges to match your data (or better yet, define a named range and use that so it is always correct).

=(LOOKUP(2,1/($B$2:$B$3000=$F2),$D$2:$D$3000)

Printed thickness of cell borders in Excel - Microsoft Community

hoss n. feffer say:

Printed thickness of cell borders in Excel

There are pre-defined constants to set the weight of a cell border in Excel:  xlHairline, xlThin, xlMedium, and xlThick.

How thick are each of these values in points (when printed)?

 

The constants for border line width in Word are straightforward:  wdLineWidth050Pt is 1/2 point thick.

john korchok say:

I printed an Excel sheet with all 4 border weights to PDF, then opened that in Adobe Illustrator to get the exact dimensions.

xlHairline is 0.0017" or 0.1224 points

xlThin is 0.0133" or 0.9576 points

xlMedium is 0.0267" or 1.9224 points

xlThick is 0.0400" or 2.88 points
