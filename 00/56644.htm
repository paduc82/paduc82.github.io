Port Forwarding - Microsoft Community

xb1r hades say:

Port Forwarding

Linksys EA6500

 

Login Information

 

Username - admin

Password - admin

Gateway - 192.168.1.1

Port Forwarding

  1.  Open a web browser.
  2. Log into the router by entering the default gateway (192.168.1.1) into the address bar of the internet browser.
  3. When prompted, enter in "admin" for the user name and "admin" for the password.
    *Note* If Cisco Connect was used to configure the router then the password will be your wireless password.
  4. From the menu on the left select Security.
  5. From the menu at the top select Apps and Gaming.
  6. Select Single Port Forwarding from the sub menu.
  7. Select Add a new Single Port Forwarding.
  8. In the following Screen enter an Application name that indicates something to do with Xbox Live (XBL).
  9. Set both the External port and Internal port to 3074.
  10. Set the Protocol to Both. 
  11. Enter the IP address of the Xbox into the Device IP#.
  12. Check the enabled box.
  13. Select Save and then select Apply.

 

 

Disable Port Forwarding

  1.  Open a web browser.
  2. Log into the router by entering the default gateway (192.168.1.1) into the address bar of the internet browser.
  3. When prompted, enter in "admin" for the user name and "admin" for the password.
    *Note* If Cisco Connect was used to configure the router then the password will be your wireless password.
  4. From the menu on the left select Security.
  5. From the menu at the top select Apps and Gaming.
  6. Select Single Port Forwarding from the sub menu.
  7. Select Delete on all open port forwarding rules set for Xbox live.
  8. Select Apply.

ruelk say:

Port 53 is used for DNS. Don't forward this port. The list of "open" ports requires that these ports are open, not forwarded. Port 53 can be left out of port forwarding (as well as 80 and 88). When you reboot and check the network settings of the Xbox you should be good. Also do not forward 80. I also did not forward 88.

Check your ports against any ports that might be used by other equipment as you could mess up access by that equipment:

en.wikipedia.org/.../List_of_TCP_and_UDP_port_numbers

Lastly, use your own router, Have your ISP put your modem (which they provide is commonly used as a router) into Bridged Mode and let your router handle all of the traffic. Get a good network person to set this up for you. Your ISP router is usually junk, and it has its own firewall. So, if you set ports on your router, but your ISP's equipment is not configured, well then you are wasting your time.

cmdrkeene say:

Hi Mach73,
What problem are you having with junction points?  We'd be pleased to help you solve any issue you may be facing.

If you are only seeking a reason, please take this understanding:  in Windows 7 (and in Vista before it), there is a well-hidden junction point calledDocuments and Settings that actually redirects to the Users folder.  Microsoft uses the word users because it is shorter, easier to reference in both code and in instructions, and better fits the purpose of the folder.  However, some very badly-created programs were built to only look for a folder that was explicitly calledDocuments and Settings (you're never supposed to hard-code a file type, these bad developers should have made their programs use a variable for compatibility).

In Windows Vista/7, Microsoft had a choice: put a junction called Documents and Settings, or let these bad programs just quit working.  They chose to keep full compatibility, even with programs that were made to look for a folder that doesn't really exist.

My advise to you is to turn off the view of hidden and protected operating system files (keep the extension on if you like them), and use your computer normally. You will only cause harm if you try to remove these junction points, some of which may require re-installation of your operating system.
