    Bug in Table cross reference - the bug adds extra page breaks or - Microsoft Community

    kylec_id say:

    Bug in Table cross reference - the bug adds extra page breaks or carriage returns before the cross reference.

    Found a bug when inserting a table cross reference.  It occurs on multiple platforms - 2010 windows, 2011 mac.

    Scenario:
    1. Have a paragraph with text
    2. Next paragraph, add a Table caption
    3. Go back to previous paragraph and add a table cross reference -- which would be pretty usual to refer to the table.
    4. Insert a page break in front of the table caption -- which would also be normal if your table crossed over the page.
    5. Print

    An extra page break is added before the cross reference.  Super Annoying!!!

    I confirmed it happens with other scenarios:

    Replace step 4. above with several enters to move the table caption down.  Print.  There will be several spaces inserted before your cross reference.

    It is messing up page numbering on my masters thesis.  I would really like a good answer to it.

    Thanks,
    Kyle

    a. user say:

    The problem is that Word creates a hidden bookmark that "covers" the entire paragraph that contains the caption.

    For the cross-reference, it creates a REF field that references that bookmark and displays a copy of the text that it covers.

    When you click at the beginning of the paragraph and add anything there (including page breaks etc., that material ends up being "covered" by the bookmark, and so it appears when the cross-reference is updated.

    The simplest approach in future is probably to ensure that you insert any new white space that needs to appear above the caption by clicking at the end of the paragraph above and clicking enter to create a new blank paragraph first. Then, if you want a page break , insert one in that new paragraph.

    (I suspect that like me, you will find that approach isn't how you normally work with text, but if so, it may be the simplest way to go).

    As for the existing references with the additional text, I doubt if there will be a simpler resolution than to remove the white space you added and follow the procedure I suggested above to re-insert the white space. If you have a whole lot of them then it may be possible to re-position the bookmarks using some VBA.
