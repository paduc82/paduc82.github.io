dhookom say:

I would move to a better normalized structure but a function like the following can be entered into a new, blank module and saved with the name "modStringFunctions".

Public Function EliminateDupesInString(strText As String, Optional strDelim As String = " ") As String
    ' remove duplicate strings from within a longer string
    ' "AIRTEX 1241 AIRTEX AW3409 CHRYSLER 7700598025 CHRYSLER JR775056 DELPHI DD1265 DELPHI DD1666 DELPHI WP1039 DELPHI WP1747 DELPHI WP2235 RENAULT 7700598025"
    '  becomes "AIRTEX 1241 AW3409 CHRYSLER 7700598025 JR775056 DELPHI DD1265 DD1666 WP1039 WP1747 WP2235 RENAULT"
    Dim varArray As Variant
    Dim intI As Integer
    Dim strOut As String
    strText = " " & strText & " "
    varArray = Split(strText, strDelim)
    For intI = 0 To UBound(varArray) - 1
        If InStr(strOut, varArray(intI)) = 0 Then
            strOut = strOut & varArray(intI) & " "
        End If
        'Debug.Print varArray(intI)
    Next
    EliminateDupesInString = Trim(strOut)
End Function

You would call this like you would call any other function:

EliminateDupesInString([Your Field Name])

ossiemac say:

Try the following.

  1. Autofit the columns first.
  2. Use the Printer Setup as if you are going to print the document.
  3. Set the Print Area.
  4. Set Portrait or Landscape.
  5. Set the required margins.
  6. You might need to use fit to 1 page wide and 1 page tall.
  7. You can also use 1 page wide and leave the Tall field blank and it will fit the width and use as many pages as it needs for the length of the document.
  8. Then save to the pdf file and it should be just as if you printed the document.

    You can record the code for the Printer setup and Save as pdf file to get the correct VBA Code.

kbt9947 say:

2010 and 2013 Microsoft Office Word Compatibility Issue - Spacing

Scenario: A document is created in Microsoft Word 2010. The document then opened in Microsoft Word 2013, changes made, and re-saved to the server. The document then accessed by a person with Microsoft Word 2010, and the spacing is ever so slightly off. A word every so often ends up on a new line, when in 2013, it is still on the previous line. Over the course of a longer document this becomes a larger problem and the spacing becomes further and further off. 

What could be causing this issue? I have tested opening this document in 2010 on multiple workstations, with the same result. It seems to be a compatibility issue between 2010 and 2013.  The document is a .docx.

doug robbins - mvp office apps & services (word) say:
It could be caused by differences in the printer\printer drivers on the two computers.

kbt9947 say: Thanks Doug! To clarify, do you think this could be the case even if it is happening before the document is printed or saved as a PDF? Just when it is sitting open within the Word application itself?

kbt9947 say:

I have resolved this issue just now. It turns out that the person with 2013 needs to save their document in compatibility mode in order for it to appear with the same spacing as it does in 2010. 

This is accomplished by File > Save As > and choosing "Word 97-2003 Document" in the Save as type drop down. I don't know if this is how it is supposed to be, but this is what solved it in my case.
