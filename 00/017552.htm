jim gordon mvp say:
Since the problem affects only files sent to you by email, then the place to start this investigation is the email process. Problems can occur at any stage of the process.

Sending:
The sender's email program may be corrupting the workbooks sent to you. Problems that a sender might be having:
* Not setting their email program to include the file extension
* The sender manually changing the file name or extension after the file was saved. Changing extensions manually should never be done
* The sender can usually avoid corruption problems by using a file compression utility to save the file in Zip format before attaching the workbook as an attachment

Internet:
* Some security programs on PCs mangle files in the process of checking them to make sure they are safe
* Some security programs on mail servers mangle files in the process of checking them to make sure they are safe
As your sender to zip the file first. Usually zip files pass through unquestioned.

Your email client:
Try saving the attachment as a file from your email program before trying to open it in Excel

You noticed that there were suggestions of using SkyDrive. SkyDrive is Microsoft's "Cloud" computing solution. Like iCloud, you get file storage but SkyDrive can handle any kind of file, not just files made by Apple products. SkyDrive has integration with Office for Mac and Office for Windows. If you save your files to a SkyDrive folder, you can specify who can open them. With SkyDrive, you don't have to worry about file corruption that you can get when emailing.

See these web pages for more about SkyDrive:
Signing Up for and Managing SkyDrive in Office 2011 for Mac

Managing Files with SkyDrive in Office 2011 for Mac

  • 3. Edit the rule, change %%POBox%% to %%postofficebox%%
