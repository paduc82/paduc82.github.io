bob jones aka: cybertaz say:
As you've stated it, "However, Excel VLOOKUP can ONLY search the leftmost column (A) and return the value." is not correct. The function scans the leftmost column [AQ in your example] of the specified data range. That doesn't need to be the entire table of data, though.

If the value being searched for is actually in Column AS, for example, specify your data range as Chart of Accounts!AS6:AV68  rather than AQ6:AV68. The important considerations are that:

  • The lookup table be sorted in ascending order based on the lookup column, &
  • The value to be returned is situated in a column to the right of it.

Unless you explain what "This does not work for me." actually means there really isn't much more anyone can offer.

bob jones aka: cybertaz say:
Re: "it can only search the leftmost column"...

Again, let me emphasize, the leftmost column of the range specified as the table_array argument, which does not have to be the entire main table -- the table_array can be any range of cells within it. IOW, if the table physically occupies A1:D50 but the value to be looked up will be found in Column C, in your formula indicate the table_array as C1:D50, not as A1:D50. As long as the value to be retrieved is in Column D [i.e., to the right of Column C] it should work fine.

If for some reason you can't structure the main table to comply with the requirements of the function, create a second 'lookup table' elsewhere & refer to it in your formula as the table_array argument. The lookup table need only have 2 columns -- the 1st column containing the values being looked for & the 2nd column containing the value to be returned.
