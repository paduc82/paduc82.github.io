ken sheridan say: Having inserted a row into tblPartNumbers, to start the inspection process you will need to insert rows into the tables which model the three binary relationship types.  This can either be done manually in the three subforms, or if each row in tblPartNumbers is to be related to each row in the three referenced tables, TableDocQAClauses, TableInspReq and TablePOQAClauses via the tables which model the relationship types, you can automate it by executing three separate INSERT INTO statements in the AfterUpdate event procedure of the PartNumberID combo box control in the parent form.

Each INSERT INTO statement  should insert values into the two foreign key columns which make up the candidate key of the table, so that to insert rows into the table which model the ret with TableDocQAClauses for instance would be:

INSERT INTO jtblPNDocClauses(PartNumberID, DocQAClauseID)
SELECT Forms!YourParentFormName!YourPartNumberIDControl, DocQAClauseID
FROM TableDocQAClauses;

Substitute the actual form and control names of course.

You can either create this as an 'append' query and execute the query by name, or you can build the SQL statement as a string expression in code and execute the statement by calling the Execute method of the CurrentDb object.  If you do the latter you should concatenate the value of the Forms!YourParentFormName!YourPartNumberIDControl into the string expression, rather than referencing the control.

In the code do the same for each of the three tables, and call the Requery method of each subform control to reload each subform's recordset and show the inserted rows.
