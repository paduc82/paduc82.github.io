Does the window have the same name every time it is opened?  What is the name in the title bar?

You can use code like that below to close a window, if you know it's Caption.

Hope this helps,

Eric

Option Explicit

Private Declare PtrSafe Function FindWindow Lib "user32" Alias "FindwindowA" _

  (ByVal lpClassName as String, ByVal lpWindowName as String) as LongPtr

Private Declare PtrSafe Function PostMessage Lib "user32" Alias "PostMessageA" _

  (ByVal hwnd as Long, ByVal wMsg as Long, ByVal wParam as Long, ByVal lParam as Long) as LongPtr

Const WM_ClOSE = &H10

Sub Close_A_Window(windowName As String)

    Dim hwnd as Long

    hwnd = FindWindow(vbNullString, windowName)

    If hwnd <> 0 Then

        PostMessage hwnd, WM_CLOSE, CLng(0), CLng(0)

    End If

End Sub

Sub TestIt

    Call Close_A_Window("Book7") ' Put your window's name here!

End Sub

2.  Cancel my subscription with the Reseller

3.  Buy a new subscription from Microsoft

4.  Create all my users again and re-import all the data

My question is; would it be possible to

1.  Purchase a subscription immediately with Microsoft,

2.  Set up my users with the generic onmicrosoft.com email addresses

3.  Migrate all my data from the old subscription to the new one. 

4. Cancel my subscription

5. Add my domain name to my new subscription

Is the above possible or are there any other problems?

huni shen msft say:

Hi Chris,

 

For your question, I think it is feasible to do that. To migrate mailboxes from one Office 365 tenant to another, you can follow the steps below:

 

  1. Purchase the plan you want to move to.
  2. Remove your custom domain from your old Office 365 subscription.
  3. Set up your custom domain for your new subscription, and set up your users.
  4. Transfer data from your old subscription to your new subscription.
  5. Cancel the subscription for the plan you’re done with by calling Microsoft Support for Office 365.

 

For more detailed information, here is an article for your reference: https://support.office.com/en-us/article/transfer-data-manually-between-two-office-365-accounts-7dc5d983-84b2-4802-bef0-602ae1780a42
