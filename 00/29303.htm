First make sure that you have all the latest drivers installed for the devices on your computer.

http://windows.microsoft.com/en-us/windows-vista/The-hunt-for-drivers-How-to-find-Windows-Vista-drivers-for-all-your-hardware

Unplug all the external Devices.
Start the System in Safe mode.
Go to Device manger.

Device that can be disabled for doing a Hardware clean boot are

1. Video Cards
2. Sound Card
3. Network Card
4. USB Devices and controller (Please don’t do this if the customer is using USB.
keyboard or Mouse or Wireless Keyboard or mouse
5. CD and DVD
6. Modems
7. Ports (serial and parallel)

After doing this if you are able to Boot in Normal Mode try to enable the device

7. After the computer starts, check if the issue is fixed.

http://support.microsoft.com/kb/929135

While this is an old thread - I recently ran into this same problem.

I got it using thinkorswim, and it started occurring on three computers all at once.

It was coincident with Java V7.51, but was not due to Java directly. It was also true that thinkorswim migrated to Java on nearly the same dates.

See http://tlc.thinkorswim.com/center/release/rel-01-18-2014-1.html

It provides another solution (just delete the "thinkorswim.vmoptions" file. It will apparently just recreate it.

There was a few good sites on this topic:

One helped me find the problem - but required some sleuthing on my part.

https://intellij-support.jetbrains.com/entries/23393413-The-JVM-could-not-be-started-The-main-method-may-have-thrown-an-exception

Let me show the specifics of the thinkorswim problem and the solution.

There is a file in the program files for thinkorswim called: thinkorswim.vmoptions

You can easily find it with the Start Search for "vmoptions" to locate the file.

On my system, it is: C:\Program Files (x86)\thinkorswim\thinkorswim.vmoptions

Edit this file with Notepad (you will need to adjust the security of the folder & file to modify it).

The contents of the file are (on my system) are now:

-Xmx1024m
-Xms512m
-Dsun.java2d.noddraw=true
-Dsun.awt.disableMixing=true
-Dawt.useSystemAAFontSettings=false
-classpath launcher.jar
-XX:MaxPermSize=128m

==========eof===========

The -Xmx value was 1280m before I changed it, and the Xms was 1024m before I changed it.

Here is where thinkorswim manages it.

It is the "Configure" button on the "Logon to thinkorswim" screen (where you enter your user and password) - found on the bottom line of the screen - before you logon.

BUT, you can't use it when you get the error, because the error occurs BEFORE you get the Logon to thinkorswim screen. That's why you need to simply edit the file as noted above. That's where the parameters that matter here (Xmx and Xms) are saved by thinkorswim.

Now I did also find that sometimes if you reboot windows, and the very first thing you run is thinkorswim, you MIGHT get a chance to reset this "Configuration". Well, that is simple - if you can get it.

Now this message is to help thinkorswim users with this problem. I know a lot of other software packages might have this problem too. I'd guess the trick is to find where they set the parameters Xmx and Xms). No doubt there are many more places to find trouble. I hope this helps save a few people some time resolving this problem.
