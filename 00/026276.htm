edisonsmejoff say:

Bug (?) at Microsoft Excel VLOOKUP function

When a VLOOKUP function is used, one of the parameters is column or reference cells and position (number). However, if an additional cell is added, the function doesn't change the formulate automatically like other formulas and commands.

Is this behavior expected or is it a bug?

smittypro1 say:

If you're referring to the column_index property (VLOOKUP (lookup_value, table_array, col_index_num, [range_lookup]) ), and you've hardcoded it, like:

=VLOOKUP(105,A2:C7,2,TRUE)

Then it's not going to update when you add/remove columns.  So the 2 isn't going to change to 3 if you add a column.  You can get around that by using COLUMN() or MATCH() instead of a hardcoded column reference.
