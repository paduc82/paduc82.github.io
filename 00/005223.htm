By default all cells in excel are protected or locked, select the cells you

now only the cells that you unlocked  can be edited. Be aware that this

you want to lock and go to format cells and check locked, the go to tools,

protection, and protect sheet, enter a password if you want, now the cells

that you locked can not be edited

Rob, you would do well to remove the SetWarnings commands from your code entirely, then covert all your RunSQL commands to CurrentDB.Execute "<query name>", dbFailOnError.

If you have a large number of queries to run, I would suggest using an array and a single call to CurrentDB.Execute.

Example:

Function RunUpdates() As Boolean

    On Error GoTo Err_Process

    Const MAXROWS = 9
   
    Dim strQueries(MAXROWS) As String
    Dim intRow As Integer
    Dim blnReturn As Boolean
   
    blnReturn = False
   
    strQueries(0) = "app_GovDepartment"
    strQueries(1) = "app_GovOrganization"
    strQueries(2) = "app_Ref_AnnuitantIndicator"
    strQueries(3) = "app_Ref_BargainUnitStatus"
    strQueries(4) = "app_Ref_EducationLevel"
    strQueries(5) = "app_Ref_EmployeeType"
    strQueries(6) = "app_Ref_FTPStatus"
    strQueries(7) = "app_Ref_GeoStateFed"
    strQueries(8) = "app_Ref_GeoCityFed"
    strQueries(9) = "app_Ref_InstructionProgram"
   
    For intRow = 0 To MAXROWS
        CurrentDb.Execute strQueries(intRow), dbFailOnError
    Next intRow
   
    blnReturn = True
       
Exit_Process:
    RunUpdates = blnReturn
    Exit Function

Err_Process:
    MsgBox Err.Number & " " & Err.Description, vbExclamation, "Error"
    Resume Exit_Process
   
End Function
