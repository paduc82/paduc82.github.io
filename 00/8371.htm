VBA: Copy data from multiple sheets into one master sheet - Microsoft Community

ifikope say:

VBA: Copy data from multiple sheets into one master sheet

VBA to copy data from Row 2 to last row with data from multiple worksheets to a master sheet, with a prompt to filter in column C.

tasos_k say:

Hi,

I assume that master sheet exists in your workbook

each time you run the code below, delete old data in master sheet (not append data)

vba macro

Sub Merge_Autof_Data()

'## 06 Dec. 2018 oAnast ##

Const sName As String = "MASTER SHT" '<<summary sheet name /change as needed

Const sCol As String = "C" '<< criteria in column C

Const sCrit As String = "abcd" '<< criteria name in column C /change as needed

Dim ws As Worksheet

Dim L As Long, t As Long, N As Long

Application.ScreenUpdating = False

'1 ### delete old data from master sheet/ from row2-up to last row

Application.DisplayAlerts = False

Sheets(sName).AutoFilterMode = False

L = Sheets(sName).Cells(Rows.Count, 1).End(xlUp).Row

Rows("2:" & L).ClearContents

Application.DisplayAlerts = True

'1 end ###

'2 ### merge all sheets

N = 2

For Each ws In Sheets

If ws.Name <> sName Then 'a

ws.AutoFilterMode = False

t = WorksheetFunction.CountIf(ws.Columns(sCol & ":" & sCol), sCrit)

If t > 0 Then 'b

L = ws.Cells(Rows.Count, 1).End(xlUp).Row

ws.Range(ws.Cells(1, sCol), ws.Cells(L, sCol)).AutoFilter Field:=1, Criteria1:=sCrit

ws.Rows("2:" & L).SpecialCells(xlCellTypeVisible).Copy

Sheets(sName).Cells(N, 1).PasteSpecial xlValues

Application.CutCopyMode = False

N = Sheets(sName).Cells(Rows.Count, 1).End(xlUp).Row + 1

ws.AutoFilterMode = False

End If 'b

End If 'a

Next

'2 end ####

Sheets(sName).UsedRange.EntireColumn.AutoFit

Application.ScreenUpdating = True

End Sub

songecake say:

I know this this thread is a few months old and there are quite a few other similar ones on the net, so I want to add this in case it helps anyone. It is relevant to many newer Nokia phones, including the Nokia X6.

 

To sort out the time-stamp issue (it seems obvious at first but the last bit is really crucial):

 

Select  the 'Clock' > Options > Settings > Time Zone and change it to your local zone. This will render the message that as a consequence automatic time update will be switched off. Click to accept and continue.

 

Note that the phone's clock will have changed from what was previously the correct time for your location to something representing the correct time plus the erroneous time difference on your email time-stamps. Change the phone's  'Time' setting back to the correct time for your location.

 

Then change the 'Time Zone' back to 'Local Time'. You will be asked if you are travelling - state 'No', so that the phone understands you are correcting an error not just moving zones physically.

 

Go into 'Automatic Time Update' and switch it back on.

 

The issue seems to have been that the 'local time' was set to GMT -5 by default. The last two steps above redefine local time from the GMT -5 default to your own time-zone, and allow automatic time updates to be retained with the correct settings.

 

In short, it appears that 'Local Time' is not just another option to selecting a specific time-zone, but a variable containing one of the listed zones (which can be redefined as above) that allows automatic updates to said zone.

 

Once all done, try sending an email. If you have followed the above exactly, the time-stamp on it should be correct.

 

Good luck :)
