All --

Pardon me for bumping into this conversation, but I wanted to shed some light on the column named Overallocated.  There is a task Overallocated column and a resource Overallocated column.  The Help text for the task Overallocated column is blatantly WRONG.  The only time that the task Overallocated column displays a Yes value is when you assign a resource to a task with a Units value that is greater than the Max. Units value.  It does not show a Yes value for parallel tasks assigned to the same resource which result in an overallocation.  That means this column is not very useful.

Calculated field in Main Form Help - Microsoft Community

blj13 say:

Calculated field in Main Form Help

I am working with creating a custom form  i have a main form that is called  ProgramsWithMembers and a Subform Called ProgramWithMembersSubform  in the main form i have the fields  ProgramID ProgramType MonthlyFee,  and PhysicalRequired
 In the Subform are the fields Name, Phone, ExpirationDate, and MembershipStatus What i am looking to do is create a calculated field in the main form that will calculated the number of members listed in the subform. Does anyone have any suggestions on how to accomplish this?
graham mandeno say:

Add a textbox to the header or footer of the subform.  Set its Name property to txtRecordCount and set its ControlSource to =Count(*).  You can also set Visible to No as you probably don't want the count to be displayed in that position.

 

Next, create a textbox on the main form with this ControlSource:

    =[SubformControlName].[Form]![txtRecordCount]

(replace "SubformControlName" with the name of the control which contains your subform)

 

This textbox will display the number of members showing in the subform at any time.

 

--Graham
