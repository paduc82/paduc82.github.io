Just to make things easier for everyone to follow - always a good plan to use common terminology to avoid any kind of confusion. Not mentioning this as a negative criticism - just a suggestion since various words have different meanings when used in different contexts and can make a difference.

#1 - Suspect you are referring to an individual "item" within an Outlook folder when using the word "file" (The word file typically denotes an external file you would access via the Windows File Explorer)

#2 - Outlook data files are those that are shown under the data files tab (goto File --> Account Settings --> Data Files tab)

#3 - An Outlook folder is one contained inside of an Outlook data store (data file) whereas the common reference to "folder" refers to the folders on your hard drive managed via Windows File Explorer.

All that said, from your description, it sounds like you have 3 seriously corrupted items in your Deleted Items folder. The message "Can't" display items........." also can get caused by an item coming from a place that originally had a seriously contorted <view>.

Since this involves deleted items, and ScanPST appears to be choking on something (which indicates that there is something really flaky going on which is outside the norm), suggest you try the following - (set Outlook to work in offline mode for the duration)

Process (A)

#1 - Simply do a PST2PST export of the PST file containing the problematic items
 

How to Export to an Outlook PST File
http://www.contactgenie.info/how-to-export-outlook-pst


#2 - Open the newly exported PST file in Outlook (File --> Open & Export --> Open Outlook Data File) and check to see if the original issue still exists for the items left in the Deleted Items folder. 

#3 - If there are no issues with the new PST file - change any references to the old to the new 

If using POP account(s) - go to File --> Account Settings --> Email tab and change the reference for any POP3 file referring to the old PST file via the <Change Folder> button on the bottom to the <Inbox> in the newly exported PST file. When that's done, go to the Data Files tab and if the old PST file was set to be the default data file for the profile, set the new PST file as the profile default. Then remove the old PST file (you may get a message about stuff being moved etc - just close Outlook and re-open it)

Process (B) 

If the same items from the old PST file still result in problems in the new PST file then

#1 - Create a <NEW> pst file (from the Home tab on any ribbon --> click <New Items> --> <More Items> --> Outlook Data File

#2 - Once the new PST file has been created go to File --> Account Settings --> Data FIles tab and make this new PST file the default for the profile which will create all the standard Outlook folders

#3 - After #2 - copy all the items from each of the folders in the old PST file to the corresponding folders in the newly created PST file - excluding the <Deleted Items> folder

 

How to Copy-Move Folders-Items in Outlook (a PST Import alternative)
http://www.contactgenie.info/how-to-copy-move-outlook-folders

#4 - When #3 is done, make sure that any POP accounts referencing the old PST file now reference the new PST file as described earlier and when that's done, remove the old PST file from the profile

Hi Martin,

To set the default value for the site column, go to site settings > web designer galleries > site columns > find the site column and click it > additional column settings > default value > input the default value.
