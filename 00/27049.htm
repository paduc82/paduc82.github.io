There are various settings that determine whether images will be displayed or will print (see http://wordfaqs.ssbarnhill.com/InvisibleGraphics.htm), but if they are displayed in Print Preview and yet still don't print, then there is something wrong beyond these settings.

If you are not seeing the header or footer at all in Print Layout view, then you may have white space hidden (http://wordfaqs.ssbarnhill.com/WhiteSpace.htm), and there is a check box in the Options group on the contextual Header & Footer Tools | Design tab to show/hide the document text when you're working in the header or footer, but none of that should affect printing, either.

From the last part of your description, the issue just might be white space: if you have the "white space between pages" hidden, then not only are the header and footer hidden but also anything anchored to the header or footer. So if, for example, you have a full-page background graphic anchored to the header, it won't be displayed if the white space is hidden.

ken sheridan say: The interval argument must be delimited with quotes characters "d".  Your expression is unnecessarily complex, however.  Instead of calling the Abs function, put the arguments in the correct order:

DateDiff("d",[DOB],Date())/365.25

or even more simply:

(Date()-[DOB])/365.25

A date/time value is nothing more than a 64 bit floating point number in which the integer part represents the days and the fractional part the times of day, so to return the number of days between two integer values of date/time data type simple subtraction can be used.  A date of birth column in a well designed database will only accept integer values, i.e. dates with a zero time of day, as its ValidationRule property will constrain the column to integer values.
