a. user say:

The advantages of Windows 10 are numerous and plenty.  

In Windows 7, the problem of your computer randomly dropping connections to the internet can often be fixed by going into your power save settings and changing your wireless adapter to Maximum Performance.  In Windows 10 you'll drop your wifi connection regardless of your power settings, and Microsoft will most likely never fix the issue.  

Windows 7 has problems with dropping network shares, but that's a problem that can be resolved through a roughly thirty step process that requires significant knowledge of Device Manager and the Command Prompt, Windows 10 will drop connections to network shares and no amount of Device Manager or Command Prompt knowledge will ever resolve the disappearing shares issue.  

Windows 10 promises you an option to create a USB installer that will never actually work, so you'll be able to go through the process, but then when you plug your USB installer into another machine you will get an "Operating system not found" error every time!  

These are just some of the exciting changes that Microsoft is planning for the Windows 10 operating system!

bill e. bill say:

This is not Microsoft.

 

It is a forum of users like yourself trying to help one another.

 

These kinds of errors occur if programs are moved to another drive incorrectly rather than letting the installer put them there.

I realize this is a bit late to the party, but someone in the future might find this helpful. 

One possible cause of this problem is the accidental inclusion of DENY permissions on the temp folder. DENY overrides ALLOW, so even if a user has full control, if there is a DENY somewhere in the mix that will take priority.

Another possibility is a group policy software restrictions policy which prevents running executables from the temp folders. Obviously that's unlikely in the OP's case, but it's happened to me, so it might be worth checking out.
