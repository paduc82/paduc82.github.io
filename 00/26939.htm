_elaine_ say:

Hi, Tom-Con

Skype does not receive reply text messages (SMS).  Instead, when we set caller ID to our mobile number, the SMS replies are received to our mobile phone number.

-> What is Caller ID, and how do I set it up?

-> SMS text messages in Skype

I sent an SMS text message to my friend but they can’t reply.

When you send an SMS text message from Skype, the recipient will see the first 11 characters of your Skype Name and they won’t be able to reply to your SMS text message.

To resolve this, you need to change the sender ID so it displays your mobile number instead of your Skype Name. The recipient will then be able to reply to your SMS text message and their reply will be sent to your mobile (not to Skype).

My friends are not able to send SMS text messages to my Skype Number.

Currently, it’s not possible to send SMS text messages to Skype Numbers. Your friends can reach you through SMS text messages on any mobile or text-enabled landline number.

jml54 say: The fix for most of these comment-related crashes on Macs is so ridiculously simple I cannot believe it has been so hard for so many users to find (and MS has nothing explaining the cure).  It took me months to find the solution. 

In Preferences for Track Changes, UNCHECK the box to put in the reviewer's name and time stamp. Then restart Word.

The information will still be inserted in the comments, but not displayed. 

Font weeding is a necessity with Office (and Mountain Lion is particularly rigid about fonts), but be sure to remove the font caches (use Onyx to do so, and restart immediately). MS causes most of the font problem by installing duplicate fonts, many of which don't match the version and date of Apple-approved fonts, and putting them in a non-standard location. But it won't fix the problem caused by a large number of comments, which is memory-related.

john mcghie say:
Hi Tim:

I think what you have done is sufficient for Font Weeding.  Yes, that procedure was written well before OS 10.8, and many of us cannot go up to OS 10.8 because we need Rosetta.

I think it is very likely that the document you are working on is the problem.  Especially if it has been edited by a higher or lower version of Word.

The ability for Comments and Track Changes to mutilate a document is very high.  It's nearly inevitable to get document corruption if you are working between versions and platforms with tracked changes or comments.

To diagnose this, we need to make a copy of that document that does not contain either.

1)  Make a copy of the document and save it (we need to keep a copy of the original so we can go back to it after we find the problem).

2)  In the copy, "Accept All Changes in Document" from the Reviewing tab.

3)  Now, remove ALL the comments as well.

4)  Now, re-save the document in the format it currently is not: so if it's .doc, save it as .docx; if it is .docx, save it as a .doc.

This will cause Word to completely re-write the code in the document, leaving out all the bits it can't interpret.

5)  Now, close the document and quit Word.

6)  Re-open the fixed document and re-save it as .docx if it isn't already in that format.

Now try adding comments to it.

You could also investigate which fonts are actually being used in the document.

The crash dump you posted is quite characteristic of a font problem, particularly with Bitmap or PostScript (Type 1) fonts.  With some of those fonts, Word attempts to load them into memory but it can't handle them, so under certain circumstances it asks for a crazy amount of memory which the system denies so Word falls over.

Microsoft Office 2011 requires Unicode fonts of OpenType with True Type Images, or True Type.  Bitmap fonts and Post Script fonts (or OpenType fonts that contain bitmap or postscript glyphs) are known to cause problems, including crashes of the kind that you are experiencing.

If you set the entire document into Calibri font, does the issue resolve?

Hope this helps
