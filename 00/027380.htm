bill manville say:

Since you can only have 56 different colour index values you will have a problem if there are more than 56 with duplicates.

Versions of Excel before 2007 were limited to 56 colours.
Later versions have no such restriction so you could change your code thus:

Sub Highlight_Duplicate_Entry()
        Dim cel As Variant
        Dim myrng As Range
        Dim clr As Long

        Set myrng = Range("A2:A" & Range("A65536").End(xlUp).Row)
        myrng.Interior.ColorIndex = xlNone
        clr = RGB(Rnd*255,Rnd*255,Rnd*255) ' random colour

        For Each cel In myrng
           If Application.WorksheetFunction.CountIf(myrng, cel) > 1 Then
              If WorksheetFunction.CountIf(Range("A2:A" & cel.Row), cel) = 1 Then
                 cel.Interior.Color = clr
                 clr = RGB(Rnd*255,Rnd*255,Rnd*255) ' another random colour
              Else
                 cel.Interior.Color = myrng.Cells(WorksheetFunction.Match(cel.Value, myrng, False), 1).Interior.Color
               End If
          End If
       Next
    End Sub

Instead of using colour you could put a "duplicate marker" in an adjacent column. You could then filter on that marker and a particular value from column A to see the different rows containing that value. 

Suppose column E is currently available for this purpose.
In E2:
=IF(A2="","",IF(COUNTIF($A:$A,A2)>1,"*",""))

and you can copy this down the length of the data

jllatham say:

As Bernie has said - it can be done; the devil is in the detail.  For example, I have some files of similar nature in use by a company and the update process takes considerable time, so we use Windows Scheduler to actually start Excel and open the 'master' workbook.  In that workbook, an event that takes place when it is opened checks the time of day and if it is in the wee hours of the morning, it then begins the updating process - opening all of the other workbooks it needs to reference and importing and processing

As for 'true' real-time, you can actually set up formulas in a workbook to extract data from other workbooks and so you (generally) do not even have to open them.

For example a formula in MasterFile.xlsx might look like this:

='C:\Users\jlatham.DOMAIN-NAME\Desktop\TestFolder\[Evaluator.xlsm]Sheet1'!$E$8

Which says 'get the value in cell $E$8 on Sheet1 of workbook Evaluator.xlsm that is located in the path C:\Users\jlatham.DOMAIN-NAME\Desktop\TestFolder\

All of that can actually be set up from the keyboard by having both workbooks open and simply going to the cell you want the formula in and entering the = symbol, then switching to that other workbook, choosing the sheet and cell you want to reference and hit [Enter].  Excel does all of the typing for you.

Now, If you then wanted to actually pick up values in the formula column from column E in the other workbook, you could modify the $E$8 reference in the formula to become $E8 and then when you fill the formula down the sheet, it would reference increasing rows at each row in your workbook.

So that's at least two different methods of attacking the problem.  And you're not limited to referencing just one other workbook in your Master workbook - you could reference many.

Finally, a 3rd way is to use macros (VBA code) that might be needed if the other workbooks consist of varying numbers of them - but again, that's just a matter of detail on the 'how' side of things.

jllatham say:

" but how could i modify that formula to instead look at all the files in a specified folder for a certain column? "

That's where VBA code would come in to play!!  There's some relatively standard code I use to do that kind of thing - it can look in a specified folder and find all files of a given type (even to testing each for their names like "all that start with "ReceivingReport_ " where that might be followed by an ID number or date or such.  It would then actually open each of those in turn, go to the specified worksheet and grab data from the specified column on that sheet.

The folder can be determined in several ways: hard coded into the macro, or let the user pick the folder, or determine the path in some fashion based on where the 'master' file itself is at.

Then of course, the macro has to know where to put that information, or what to do with it in the way of processing and where to put the results in the master workbook.

One 'trick' to doing this kind of thing is to keep up with what other workbooks or records within them have already been processed so that multiple copies of previously processed data are not generated.  An easy way to do this is to have a column in the other workbooks available for our macro to simply enter some value/marker to say "I got this already!" as it processes them, then if it tries to process the same file(s) later, it can look in that column for the marker and pass over records it's already dealt with.

bernie deitrick say:

You can create the link or formula with a link by opening the workbook of interest (with the data), returning to your original workbook, typing = in the cell where you want the link, and then switching to the other workbook and selecting the cell(s) of interest and pressing enter. When close the  other workbook, the link will change to include the folder path as JLLatham showed.

Alternatively, you can create the links using a macro: This, for example, will create multiple links to cells in a workbook that you choose in a dialog without actually opening the file (so you need to know the sheet name, and cell addresses, but this is just an example, of course, and similar code can do more)

Sub MakeLinksUsingGetOpenFileName()
    Dim strFileWithData As String
    Dim strFormula As String
    Dim strSheetName As String
    Dim strPath As String
    Dim strFilename As String

    strFileWithData = Application.GetOpenFilename(Title:="Select the file with the data")

    strSheetName = "Sheet1"
    strPath = Left(strFileWithData, InStrRev(strFileWithData, "\"))
    strFilename = Split(strFileWithData, "\")(UBound(Split(strFileWithData, "\")))
    
    'Generate strFormula through string manipulation
    strFormula = "='" & strPath & "[" & _
                 strFilename _
                 & "]" & strSheetName & " '!"
                 
    Application.EnableEvents = False
    Application.Calculation = xlCalculationManual
    
    'Set cell formulas - I hope you see the pattern
    With ActiveWorkbook.Worksheets(1)
        .Range("B1").Formula = strFormula & "C6"  'Create a link in B1 to cell C6
        .Range("B2").Formula = strFormula & "E2"
        .Range("B3").Formula = strFormula & "J7"
        .Range("B4").Formula = strFormula & "H9"
        .Range("B5").Formula = strFormula & "K5"
        .Range("B6").Formula = strFormula & "M4"
    End With
    
    Application.EnableEvents = True
    Application.Calculation = xlCalculationAutomatic

End Sub
