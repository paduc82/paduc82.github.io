I assume you were saving as a PowerPoint package so that your presentation would be portable from one computer to another. 

Things to keep in mind:
  • In the new format, copies of your media are embedded within the PowerPoint presentation file, so there's no need to copy the media files.
  • PowerPoint for Windows supports QuickTime format, but only in PowerPoint 2010. For other Windows versions your best bet is to use .wmv format and install Flip4Mac on your Macs.

File already in use error - Microsoft Community

rucsanddraneagu say:

File already in use error

Hello

I have a couple of Access applications (same application for each year for the past 10 years or so: Application 2017, Application2016...), all saved in the same folder. They are being used by multiple users at different locations.

The issue I have now is that when we try to open the 2017 application we get this error "cannot open file name.accdb; file already in use."

The other files (applications) in the folder are working ok.

I am afraid somebody may have deleted the 'laccdb' file for 2017.

Any suggestions? Thank you.

ken sheridan say: Multiple users should not be opening the same file.  Instead the file should be split into separate front and back ends, the latter containing the tables only, the former containing the forms. reports, queries etc, along with links to the back end tables.  The built in database splitter word can do this for you, though it's a simple task to do it yourself.

Once split, the back end should be in a shared location on the system to which the users have full permissions, while the separate copies of the front end should be installed either on each user's local machine, or in a 'personal' location on the system to which only the user has access.  The user should then open their own copy of the front end.

I would also point out that having separate files for each year is generally not good database design as it encodes data in the file names.  A fundamental principle of the database relational model is the Information Principle (Codd's Rule #1). This requires that all data be stored as values at column positions in rows in tables, and in no other way.  All data should be in a single back end, and accessed by queries which restrict it to the year in question on the basis of date values in columns.

scottgem say:

The laccdb file cannot be deleted while the file is open. If the database crashed leaving the laccdb file open, then it can and should be deleted. As soon as the database is open again a new laccdb file is created.

But as Ken said, your design is not best practices. Why are you creating new files for each year?

daniel pineault say:

You cannot delete the laccdb file while the database is in use, so nothing to worry about there.

If you have databases being used by multiple users, then you need to split them.  Take a look at: http://www.devhut.net/2017/04/09/setting-up-an-ms-access-database/

Also, why do you create a new database every year?  You should be able to house everything in one database and simply distinguish the data by including a data field.  Then you are in a good position to be able to pull reports and perform comparisons...

Hyperlinks inactive in HTML messages from certain senders in - Microsoft Community

unorchestra say:

Hyperlinks inactive in HTML messages from certain senders in outlook.com

There are several similar threads on this subject, but none exhibit the exact same symptoms, and none actually provide a working solution.

The filtering system in outlook.com seems to try to protect the recipient from malicious links by blocking them from suspicious looking messages. Unfortunately, perfectly legitimate messages often end up caught in this filter and HTML hyperlinks are completely stripped out.

Here is my scenario. An HTML message is distributed to a small list (90 members). The message contains regular HTML, with little CSS styles and several hyperlinks. Recipients are largely Lotus Notes, many are g-mail, a few are hotmail/outlook. Most view them in Lotus Notes on desktop (Windows), several on the iPhone (Mail client), few use webmail on desktop or Safari on iOS. The only two who cannot see the hyperlinks are two hotmail/outlook users opening them in web browsers. In fact, not only do the hyperlinks not work, but most of HTML attributes are stripped away; for example, table elements don't have any background colour, font attributes (such as size, font family, etc) are gone, table alignments, style properties for DIV tags are all gone. And this is ONLY when the message is accessed through the browser on hotmail/live/outlook.

When I take the same message, sent to a g-mail address, and the forward it to a Hotmail address, and then open this from the browser, suddenly ALL of the attributes, styles and hyperlinks are working. More interestingly, since my forwarded message, as well as the original message sent by the list sender, have same subjects (except for the "Fw:" part), both messages show up inline, below each other, and now, the original message, that once showed nothing but plain text, now shows all the attributes, links and styles. But if I delete the forwarded message, leaving the original one alone, the attributes are again stripped away.

It is clear to me that Hotmail simply doesn't like the sender's address. I tried rewriting my php script that sends the message to the list (originally, using SendMail), to use SMTP and send the message individually to each recipient on the list through G-mail account. Unfortunately, this didn't help either; Outlook.com received the message properly, but all the attributes, styles and hyperlinks were stripped away.

The essence of this problem is that there seems to be no way in any of the Outlook.com settings to override this filter. Even when we add the sender email and domain to the whitelist ("Safe senders") and to "Safe mailing lists", the HTML features are still blocked. Recipients can't click on any of the original links because all of them are simply gone, with plain text left in their place.

Am I really missing something, or does this need to be resolved by Microsoft?

[update]

After a bit more testing, here is some additional information. When I receive a message, in Outlook.com, from this particular sender, if I use web-based interface for Outlook.com via web browser (any browser, any platform: Explorer, Firefox, Chrome, Safari on Windows, Mac, iOS or Android), Outlook.com will strip away all HTML attributes, CSS styles, and hyperlinks. If I use any common email client software (Mail on Macintosh, Thunderbird or Outlook on Mac or Windows, mobile mail client on iOS or Android) and configure it to access my Outlook.com email account, all message code is intact and everything works.

One more important thing. If I forward the same message to myself, the web-based interface on Outlook.com will consolidate the original message and my forwarded one in a single message thread. The original message will suddenly re-gain most of its formatting, and hyperlinks will suddenly work. If I select just the newly forwarded message and delete it, leaving the original message, the formatting and hyperlinks will disappear. The behaviour is consistent across platforms, as long as the mailbox is accessed through the browser at www.outlook.com.

This same problem exists for users using Exchange in corporate environment, when they access their mailbox using webmail interface.
