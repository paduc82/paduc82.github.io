Hey folks,

I am having an issue.  I have a spreadsheet that has several ranges of cells that contain numerical data and there are column headings separating each range of data.  I am trying to do a simple Conditional Formatting rule that will highlight any data over a specific number.  I have a lot of different ranges of data on my spreadsheet so instead of specifying each data range individually I would like to just select all of the data at once and apply the conditional formatting rules.  However, when I do this, the rule doesn't recognize the cells with text in them as not having numerical data so it applies the conditional rule to the data.  I end up creating another rule to take the formatting off of cells containing text.  Is there an easier way to do this?


Use the "Use a formula to determine which cells to format" option then use a formula like this:

=AND(COUNT(A1),A1>=10)

In Excel text has a higher logical value than any number. For example:

="text">10

Result = TRUE

So, if you use a cf rule that is something like =A1>=10 and the cell contains text then the format is applied.

joeu2004 say: Ryan wrote:

I have large dataset that I have to analyse. I'm doing the analysis using pivot tables, but I would like to eliminate the outliers first. I was thinking of eliminating values that fall outside 2 standard deviations of the mean. Any ideas?


There are two questions here:  first, the wisdom of eliminating outliers automatically in the first place; and second, how to do it in connection with using pivot tables?

I addressed the second question somewhat in my response to Daniel, to wit:  I suspect your question lacks sufficient specifics for any knowledgeable person to offer useful guidance.  See my previous response for how to provide a concrete example.

But the first question should not be overlooked.

First, I would not consider values outside +/-2 sd to be outliers.  In a normal distribution, that covers about 4.6% of the data, well within reason.

Some people do consider values outside +/-3 sd to be outliers.  That covers about 0.3% of the data.

Second, it is dubious to automatically exclude data simply because they are outside some "acceptable" range around the mean.

IMHO, an outlier should be excluded only if it appears to be the result of an error or an anomaly.  Otherwise, the outlying values might be the very data that are needed to refute some misguided conclusion.

Finally, the use of the standard deviation per se to identify outliers depends on an assumption that the data are normally distributed.  Otherwise, sd might not be as useful since it has less statistical meaning.

If the data are not normally distributed, I would use some other well-accepted method of identifying outliers.  For a brief overview, see http://en.wikipedia.org/wiki/Outlier.

In particular, I  would use some multiple of the interquartile range.

FYI, regarding Lo and Hi in Table2....  It appears that you used Goal Seek to derive them.  Instead, you can compute them directly as follows, replacing Stdev_Lo and Stdev_Hi with the constants -2 and 2 (although I would use -3 and 3):

Lo:  =[Mean]+[Stdev_Lo]*[Stdev]
Hi:  =[Mean]+[Stdev_Hi]*[Stdev]
