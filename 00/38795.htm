My understanding on both is that each of those specific tasks rely heavily on processing. For instance, if you're reading from an ISO file, it's very similar to extracting a ZIP file. Watch your CPU usage next time you're copying from a virtual drive and see if it spikes. Same goes for uTorrent when it preforms an integrity check. See if the CPU spikes when you start the check. If so, the NIC is waiting for the CPU to finish processing the current chunk of that file.

Offline files could be the same way, but I've never paid too close attention to CPU or NIC usage when syncing. I'd check the CPU for this as well and see if any processing is required to sync the files.

For issues resulting from an update, it's often useful to do a system restore to a time before the update was installed. Windows key+X->System->System protection->System restore...

When the restore is completed, try the update again. Many update issues are one time issues from a failed or interrupted update process, so the chances are good that it goes better on the second try.

If the issue returns again after the update: do another restore, block the update, and contact the software provider.
