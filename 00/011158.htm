cody c. say: Hi MrEZ3,

I would double check to make sure you're up-to-date. 14.2.4 was only released yesterday, so you're likely no longer up-to-date. You can get the latest update by choosing Help > Check for Updates inside any Office application.

Some of your issues really scream corrupt database. Since you're on Exchange, rebuilding the database is the wrong thing to do. Instead, try creating a new identity and download a fresh mailbox.

To create a new identity, follow these steps:
  1. Quit all applications.
  2. Hold down the Option key on the keyboard, and then click the Outlook icon in the dock. The Microsoft Database Utility will open.

    Note You can also access the Microsoft Database Utility by opening the Application folder and then opening the Microsoft Office 2011 folder. Open the Office folder, and then double-click Microsoft Database Utility.app.
  3. Click the plus [+] sign to add a new identity, and then type a name for the new identity. For example, type New identity.

  4. Set the newly create identity as the default. To do this, select the newly created identity, click the Action button, and then select Set as Default.


  5. Quit the Microsoft Database Utility, and then restart Outlook and see whether the problem is resolved.


It's worth mentioning that if you're installing Office, then updating and using, then reinstalling on top of itself, you might be making the issue worse. You need to be following the steps on the following KB to properly remove Office for Mac 2011 before reinstalling:
How to completely remove Office for Mac 2011


 Not doing this can create multiple versions of the same files and cause some really nasty issues. If you've been incorrectly removing Office for Mac, please start with a complete removal and start fresh.


As for the "Random beeping noise coming from machine", that's very likely related to this thread:
Chirping sound when retreiving data


If Outlook was recently installed and configured, it's going to take some time to download the entire mailbox (depending on the mailbox size). The longer this takes, the more likely the fans will kick up. In most cases, this goes away after about a day or so.

kurtlang say:

You can get them to work in High Sierra, but it requires turning System Integrity Protection off. And for all intents and purposes, leaving it off.

Restart the Mac and hold down the Command+R keys to boot into Recovery mode. Ignore the standard options that appear and go to the top menu bar. Choose Utilities > Terminal.

In Terminal type:

csrutil disable

You should get a message that System Integrity Protection has been successfully disabled.

Then type:

reboot

The Mac will restart.

Now install IntelliType. The drivers will install and work, but there's one step you have to do manually. And that's to add the startup file to your account.

  1. Navigate to the folder /Library/PreferencePanes/ of the startup drive
  2. Right click on Microsoft Mouse.prefpane for mice, Microsoft Keyboard.prefPane for keyboards
  3. From the resulting contextual menu, select Show Package Contents
  4. Navigate down to Contents > Resources
  5. Leave the Resources folder open on the desktop and open the System Preferences
  6. Select Users & Groups
  7. Click on your user account and then the 

    In the sample below, I added the keyboard helper.

    You can reenable SIP by going into Recovery mode again and typing csrutil enable, but because the .kext file is in the System folder, SIP will eventually disable it. Might the next day, a couple of weeks, or a couple of months. Then you have to go through the whole routine again to get IntelliType working. If you leave SIP off, the IntelliType drivers will always work. However, you then lack SIP's protection. The OS is essentially back to the security level of Yosemite or earlier.

    Looking into this a bit further, it would be some extra work for MS to rewrite this installer. In the OS after High Sierra, 32 bit apps will not run. Period. The IntelliType installer and uninstaller are both 32 bit, so they'd have to be updated to 64 bit, besides changing the install location of MicrosoftKeyboard.kext to the /Library/Extensions/ folder.

    Still, this is piddly work for a company the size of MS, and there are a lot of Mac users who use, and prefer Microsoft's keyboards.

    ossiemac say:

    This question has been posted for some time so I wonder if you have an answer yet from somewhere else.

    In VBA code the CurrentRegion is ideal for this type of situation. CurrentRegion is the area that contains data surrounding a specific cell or range. Both the first and last cell of the range will be the cell inside where blank column and row meet at top left and bottom right.


    Therefore by assigning a range variable to the current region of cell A1 in your example it includes  the Range $A$1:$G$5.


    Now if you record code to sort a selected range (in the example $A$1:$G$5) then the recorded code will specify the selected range. You can then edit the recorded code and replace the selected range with the variable and it will always be the correct irrespective of how many rows of data provided you have at least one blank row between the data to be sorted and the Summary data.

    Code as follows where data was selected to get the initial sort syntax and then how to edit this for the final code. In good programming it is rarely necessary to actually select and if you are competent enough to write code without selecting then the example should still be sufficient to guide you in setting the correct range to sort without selection but if not that competent then just go with selecting the range.

    Sub TestSort()

        Dim rngToSort As Range
        Dim lngLastRow As Long
       
        Set rngToSort = Range("A1").CurrentRegion   'A1 can be any cell within the range to be sorted.
       
       'Next line Find the last row number of rngToSort
        lngLastRow = rngToSort.Cells(1, 1).Offset(rngToSort.Rows.Count - 1).Row
       
        'Range("A1:G5").Select  'Delete this line in recorded code
        rngToSort.Select              'Replace previous line with this line
        ActiveWorkbook.Worksheets("Sheet1").Sort.SortFields.Clear
       
       'Next line replace the last row in the Key range with lngLastRow
        'Example sorts on column B (Your recorded code might have different sort column)
        ActiveWorkbook.Worksheets("Sheet1").Sort.SortFields.Add Key:=Range("B2:B" & lngLastRow), _
            SortOn:=xlSortOnValues, Order:=xlAscending, DataOption:=xlSortNormal
       
        With ActiveWorkbook.Worksheets("Sheet1").Sort  'Ensure correct Sheet name here
            '.SetRange Range("A1:G5")   'Delete this line in recorded code
            .SetRange rngToSort         'Replace previous line with this line
            .Header = xlYes
            .MatchCase = False
            .Orientation = xlTopToBottom
            .SortMethod = xlPinYin
            .Apply
        End With

    End Sub
