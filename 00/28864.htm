daniel g. grau say:

Double spacing does affect vertical centering accuracy. This is because Word considers the height of the line times the number of lines. The result is that the 1.5 lines or double lines appear a bit higher than they should.

Word also adds any space before and after when calculating vertical centering.

The way to correct it is to add Space Before to the top paragraph in each cell.

rohn007 say:

Yes.

With this type of license, what matters is the total number of installs you have active at the same time.  When you uninstall from one machine it frees up that license count to be used on another machine.  Unfortunately, the uninstall process does not actually contact the MS activation system.  So if you are at your limit, ie 3 machines, even though you have uninstalled, when you try to activate on the new machine you may still get a "too many installs" type message.  In that case you have to do a phone activation. When you call in, do not reply to the inital prompt for a prodcut ID.  Wait, a long time, until you are offered the option of talking to a live "service" rep.  Explain that you have uninstalled and are transferring that license to a new machine. They should allow your activation to complete.

cbarnhorst say:

The description of an update as for "amd64" has nothing to do with the hardware in your device.  It applies to AMD and Intel equally.

You have been using "amd64" in every computer you have ever owned that supported 64bit Windows.  Here's why.

It is customary in the chip industry to assign the name of the first chipmaker to introduce a new class of chips to the whole class.  Since AMD was the first CPU maker to introduce a 64bit CPU (the Opteron in 2003) capable of also running X86 commands natively, AMD64 became the name for the entire class of such CPUs.  That means that Intel also makes amd64 chips because those chips comply with the standard of being 64bit processors that support the X86 32bit instruction set.  This is also sometime referred to as X86_64.  I can't stress too much that the basic requirement to be amd64 is that the CPU can run X86 commands natively.  

Intel's 1999 64bit CPU innovation, the Itanium chip, did not support the X86 command set which had powered 32bit Windows so it is not in the amd64 class.

In case you're wondering, "X86" means the Intel 8086 chip's command set.
