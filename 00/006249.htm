bernie deitrick say: There is nothing wrong with your code, especially since you only have two sets of sheet and conditions. 

But, you could loop (useful when you have many more combos)

Sub CopyWithLoop()
    Dim sh As Worksheet
    Dim sht As Worksheet
    Dim Arr(1 To 2, 1 To 2) As String
    Dim Rng As Range
    Dim i As Integer

    Set sh = Worksheets("Sheet1")

    Arr(1, 1) = "Sheet2"
    Arr(2, 1) = "Sheet3"
    Arr(1, 2) = "In Process"
    Arr(2, 2) = "Closed"
    Set Rng = sh.Range("A1:K" & LastRow(sh))

    sh.Columns("I:J").EntireColumn.Hidden = True

    For i = 1 To 2
        Rng.AutoFilter Field:=8, Criteria1:=Arr(i, 2)
        Rng.Offset(1, 0).Resize(Rng.Rows.Count - 1, Rng.Columns.Count).SpecialCells(xlCellTypeVisible).Copy
        Set sht = Worksheets(Arr(i, 1))
        sht.Range("A" & LastRow(sht) + 1).PasteSpecial (xlPasteValues)
        sht.Cells.Columns.AutoFit
        Application.Goto sht.Range("A3")
        Application.CutCopyMode = False
    Next i
    
    Rng.Parent.AutoFilterMode = False
    sh.Columns("I:J").EntireColumn.Hidden = False

End Sub

Or you could use a re-usable subroutine with parameters (useful for any reuse of the code)

Sub CopyWithSub()
Dim Sh As Worksheet, Sh2 As Worksheet, Sh3 As Worksheet
Dim Rng As Range

Set Sh = Worksheets("Sheet1")
Set Sh2 = Worksheets("Sheet2")
Set Sh3 = Worksheets("Sheet3")
Set Rng = Sh.Range("A1:K" & LastRow(Sh))

Sh.Columns("I:J").EntireColumn.Hidden = True

Copy2Sheet Sh2, "In Process"
Copy2Sheet Sh3, "Closed"

Rng.Parent.AutoFilterMode = False
Sh.Columns("I:J").EntireColumn.Hidden = False

End Sub

        Rng.AutoFilter Field:=8, Criteria1:=strF

        sht.Range("A" & LastRow(sht) + 1).PasteSpecial (xlPasteValues)
        sht.Cells.Columns.AutoFit
        Application.Goto sht.Range("A3")
        Application.CutCopyMode = False

This kind of thing is liable to happen if the portable drive was disconnected without going through the 'safe removal' process. Even more likely is that the affected document would be corrupted beyond repair.

If you have Word configured to save autorecover data (see under File|Options|Save), there's a chance there's an asd file in the autorecover folder. Check in the Recycle Bin for the document, too.

Failing all of the above, you'll need to reload the file from your most recent backup.

<edit>

Paul made a valid point. 

USB is a connector type for plugging in external devices.  For this discussion there are 2 types of USB connected file storage devices. The original "Thumb/Flash" drives and relatively newer spinning disk drives.

Thumb drives are those physically small, thumb sized, drives that use "flash" memory.  The started out as KB of storage and now are up to (insanely huge) 256GB. Flash memory has a limited number of write cycles and it consists of "blocks" that have a technological limit that the block must be written all at once. So Windows protects that limited write life. Windows does that by temporarily storing written files in a "cache" on the local drive. It only writes out to the flash drive when it has a a full 'block' (or blocks) to write all at once or you close the file.  If you don't "flush" the cache before unplugging the drive you can potentially (not always) leave part of the file behind on the HD.  And unfortunately, the way DOCX files are physically ordered/structured, the actual text is in the part of the file that gets left behind.


Spinning disk drives are "traditional" disk drives that have been shrunk in size and put into a portable box that uses a USB connection to the computer. They don't have any any special read/write requirements other than making sure you don't pull the plug while the write is actually happening (and don't bang them around while they are spinning)

</edit>

Here are some tips for the future, to reduce the chance of this problem re-occurring.

<edit>

#0 ALWAYS use "Safely Remove" before unplugging Flash/Thumb drives

http://www.dummies.com/computers/operating-systems/windows-7/how-to-safely-remove-hardware-in-windows-7/

</edit>

#1 USB "Flash"/thumb drives are flakey!  They will work fine, until they don't. So DON'T TRUST THEM with the only copy of important files.

#2 ALWAYS copy the file to the local hard drive to edit it. NEVER edit directly on a thumb drive!

#3 When you are done, copy from the local drive to the USB. LEAVE the copy behind on the local drive. That way, if (WHEN) the USB carps out (or you lose it, or whatever) you still have a recent copy of the file.

#4 If it is a really important, make (frequent) periodic copies of the file so that you have multiple generations of backups.  Either use "Save As", or make copies in File Explorer.
