jim gordon mvp say:

There are different issues under discussion here.

1. Preview - In Mac OS X you can get a preview by selecting a file icon and pressing the space bar. This is a feature of Mac OS X and current editions of Microsoft Office and LibreOffice supports it for files saved in .docx file format from either program. If this is not working for you, then you have a problem with your install of Mac OS X.

2. Spotlight search - Finding files using Spotlight in Mac OS is another feature of Mac OS X. If Spotlight is not finding your files, check Spotlight Preferences to make sure you haven't excluded the directory in which your files are located. If your files should be indexed by spotlight but they are not being indexed, then reindex Spotlight. 

3. If you want all your .docx files to open in Microsoft Word instead of OpenOffice, in Finder, right-click on a .docx file icon and choose Get Info. In the Get Info dialog change the Open With setting to Microsoft Word, and then click the Change All button.

If you want to use a version of OpenOffice, I highly recommend using LibreOffice instead of the base OpenOffice. LibreOffice has support for .docx and you can even go to the LibreOffice General Save preferences and change the default Save to Microsoft Office 2007, 2010, 2013 file format. I know. They refuse to acknowledge Office 2008, 2011 and 2016 for Mac, but switching to that setting will make your files compatible with Microsoft Office and LibreOffice across the board.

Proper v vbProperCase and Title UDF - Microsoft Community

peter atherton say:

Proper v vbProperCase and Title UDF

Recently I noticed a couple of differences with VBPropercase and Excel's Proper function with the way that hyphenated words or a word containing an apostrophe are handled as shown in the screen capture.

I'm wondering could anyone post other variations. These might be used in a yet another UDF for Title case. The following is what I've got so far and would welcome comments

.

Function Title(ByVal ref As Range, Optional bFormal As Boolean = True) As String
    Dim vaArray As Variant
    Dim LLo As Long, LMid As Long, LHi As Long
    Dim c As String
    Dim i As Integer
    Dim iPos As Integer
    Dim vaLCase As Variant
    Dim Str As String, sChr As String
    Dim bFound As Boolean

    ' Array contains terms that should be lower case
    vaLCase = Array("A", "Am", "An", "And", "Be", "Der", "Do", "Du", "In", "Is", _
             "Me", "My", "Of", "On", "Or", "She", "Than", "The", "To", "Van", "Von", "We", "with")
   
   
    If bFormal Then
        c = WorksheetFunction.Proper(ref)
    Else
        c = StrConv(ref, vbProperCase)
    End If
   
    iPos = InStr(1, c, "-On-")
    If iPos > 0 Then
        c = Replace(c, "On", "on")
    End If
    'split the words into an array
    vaArray = Split(c, " ")
   
    'i ignores the first value - that should be Proper case
    For i = 1 To UBound(vaArray)
        LLo = LBound(vaLCase)
        LHi = UBound(vaLCase)
        If Right(vaArray(i), 1) Like "[',.]" Then
            sChr = Left(vaArray(i), Len(vaArray(i)) - 1)
        Else
            sChr = vaArray(i)
        End If
       
    'Binary Search for sChr
    Do Until LLo > LHi
        'Find the midpoint of the array
        LMid = (LLo + LHi) \ 2
        If sChr = vaLCase(LMid) Then
        'sChr is found so return the location & quit loop
        vaArray(i) = LCase(sChr)
        Exit Do
       
        ElseIf sChr < vaLCase(LMid) Then
            'sFind is higher than mid-point so _
             throw away the top half
            LHi = LMid - 1
        Else
            'sFind is lower than than mid-point so _
            discard the bottom half
            LLo = LMid + 1
        End If
       
    Loop

    Next i

  ' rebuild the sentence
    Str = ""
    For i = LBound(vaArray) To UBound(vaArray)
        Str = Str & " " & vaArray(i)
    Next i

    Title = Trim(Str)
End Function

Function Title(ByVal ref As Range, Optional bFormal As Boolean = True) As String
    'Thanks to Jim Cone who pointed out that the original function _
     did not deal with hyphens & upper case words such as 'IBM' or indeed Mcs and Macs.
    'Completed 5 Jan 2015
    '
    Dim vaArray As Variant
    Dim LLo As Long, LMid As Long, LHi As Long
    Dim c As String, sTemp As Variant
    Dim i As Integer, iWrdCap As Variant
    Dim iPos As Integer, iMc As Integer, _
        iMac As Integer, iHyphen As Integer
    Dim vaLCase As Variant
    Dim Str As String, sChr As String
   
    'Is there a capitalised word in the reference? e.g. IBM or BAE _
     if so find out its position for later
    sTemp = Split(ref, " ")
    iWrdCap = ""
    For i = LBound(sTemp) To UBound(sTemp)
        If sTemp(i) = StrConv(sTemp(i), vbUpperCase) Then
            iWrdCap = i
            Exit For
        End If
    Next i
   
    ' Array contains terms that should be lower case
    vaLCase = Array("A", "Am", "An", "And", "Be", "Der", "Do", "Du", "In", "Is", _
             "Me", "My", "Of", "On", "Or", "She", "Than", "The", "To", "Van", "Von", "We", "with")
   
    If bFormal Then
        c = WorksheetFunction.Proper(ref)
    Else
        c = StrConv(ref, vbProperCase)
    End If
    '=======================================
    'Special Cases
    iMac = InStr(1, c, "Mac")
    If iMac > 0 Then
        Mid(c, iMac + 3, 1) = UCase(Mid(c, iMac + 3, 1))
    End If
   
    iMc = InStr(1, c, "Mc")
    If iMc > 0 Then
        Mid(c, iMc + 2, 1) = UCase(Mid(c, iMc + 2, 1))
    End If
   
    iPos = InStr(1, c, "-On-")
    If iPos > 0 Then
        c = Replace(c, "-On-", "-on-")
    End If
   
    iHyphen = InStr(1, c, " - ")
    If iHyphen > 0 Then
        c = Replace(c, " - ", "-")
    End If
    '=======================================
    'split the words into an array
    vaArray = Split(c, " ")
   
    'i ignores the first value - that should be Proper case
    For i = 1 To UBound(vaArray)
        LLo = LBound(vaLCase)
        LHi = UBound(vaLCase)
        If Right(vaArray(i), 1) Like "[',.]" Then
            sChr = Left(vaArray(i), Len(vaArray(i)) - 1)
        Else
            sChr = vaArray(i)
        End If
       
    'Binary Search for sChr
    Do Until LLo > LHi
        'Find the midpoint of the array
        LMid = (LLo + LHi) \ 2
        If sChr = vaLCase(LMid) Then
        'sChr is found so return the location & quit loop
        vaArray(i) = LCase(sChr)
        Exit Do
       
        ElseIf sChr < vaLCase(LMid) Then
            'sFind is higher than mid-point so _
             throw away the top half
            LHi = LMid - 1
        Else
            'sFind is lower than than mid-point so _
            discard the bottom half
            LLo = LMid + 1
        End If
       
    Loop

    Next i

  ' rebuild the sentence
    Str = ""
    For i = LBound(vaArray) To UBound(vaArray)
        If i = iWrdCap Then
           Str = Str & " " & UCase(vaArray(i))
        Else:
            Str = Str & " " & vaArray(i)
        End If
    Next i

    Title = Trim(Str)
End Function

ken sheridan say:

The DefaultValue property is always a string expression regardless of the data type of the column in question, and is documented as such.  If you do not wrap the value in literal quotes it will be evaluated as an expression, so if the value can be evaluated as a number or an arithmetical expression it will be.  One situation where this can be crucial is, rather unexpectedly, where the column is of date/time data type.  If, for instance you want carry a date forward from the last record entered to the next new record in a form you might think you can enter the following in the form's AfterInsert event procedure:

    Me.MyDateControl.DefaultValue = Me.MyDateControl

However for today say, the date in UK short date format would be 24/09/2014.  As an arithmetical expression this evaluates to   1.32406487917908E-03 (0.00132406487917908  in decimal notation) which, as a value of date/time data type would represent 30 December 1899 00:01:54, the date/time data type being a 64 bit floating point number with its origin at 30 December 1899 00:00:00.  So you'd need to use the following:

    Me.MyDateControl.DefaultValue = """" & Me.MyDateControl & """"

where each literal quotes character is represented by a pair of contiguous quotes characters.

People are occasionally tempted to use:

    Me.MyDateControl.DefaultValue = "#" & Me.MyDateControl & "#"

which would work today here as the date is unambiguous, but on 4th July would turn it to 7th April in the UK or elsewhere not using the US short date format.

What you have stumbled across is a less extreme example of the same phenomenon, in your case simply evaluating the value as a number rather than a string expression, so ignoring the leading zero.
