suzanne s. barnhill say:

Keep in mind that calculated numbering will not be reflected in the TOC (if any), but here's how to do it.

  1. Insert the PAGE field in the Even Page Header using Alt+Shift+P.
  2. Press Alt+F9 to display the field code, which will be { PAGE }.
  3. Edit the field code to say = { PAGE }/2.
  4. Select the entire expression and press Ctrl+F9 to create a field:
    { = { PAGE }/2 }
  5. Press Alt+F9 to toggle back and F9 to update.
  6. Omit the page number from the Odd Page Header.

jay freedman say:
The number that appears in any header doesn't have to be the actual page number, if you do some arithmetic in the field that you put in the header.

In the instructor document, leave the first page blank. In the Header & Footer Tools tab, check the boxes for both Different First Page and Different Odd & Even Pages.

In the Even Page Header, type this:

  = PAGE / 2

Select the word PAGE and press Ctrl+F9, which inserts a pair of field markers around it. (Note: You cannot create field markers by typing the { } brace characters on the keyboard. They won't work.)

Then select everything from the equal sign to the 2, and again press Ctrl+F9. The result should look like

{ = { PAGE } / 2 }

Press F9 to update the field.

Then go to the Odd Page Header and simply type an underscore. Close the header, and the result should be what you requested.

My name is Dan and I’m a Senior Lead Program Manager on the PowerPoint product team.

Thanks for all the hard work on debugging this issue - I just wanted to confirm once and for all that we ARE aware of it and are working on a solution.

As near as we can tell the problem is caused (as MickeyMills discovered) by any software that happens to pro-actively sync the location where you’re saving your Office document. Usually software that does this is set up to monitor a given directory (or set of directories) and automatically copy anything that changes off to another location (either in the cloud or on another disk/machine). It may also occur as a result of anti-virus software (which can also take “locks” on files while they’re scanned).

Depending on how the sync software is written it will often try to do this as soon as it detects a change to any existing file in its “watched” directories - even if that file is still in use. Knowing this, the best workaround we can suggest (again, as MickeyMills and several others pointed out) is to either A) store your working documents in a location that is NOT pro-actively synced, or B) pause syncing while you’re working on the documents (e.g. schedule backup overnight).

We know this workflow isn’t ideal - especially for those of you using cloud-storage sync clients (e.g. DropBox, Google Drive, etc) so we’re still doing our best to come up with a solution that doesn’t require you to change how you work.

So just to recap - if you’re seeing this problem please check your machine for the following:
• Cloud-storage sync clients (e.g. DropBox, Google Drive, etc)
• Backup software (e.g. Genie Pro, Synology Replicator, etc)

Try disabling the sync behavior (or exiting the client) while you’re working on the document and see if you still experience the save issue.

N82 and compatibility

bitflung say:
10-Jul-2008 07:49 AM
newbie200 wrote:
sorry, so i found my phone on the drop down list, that means that the isync software will allow me to transfer music, address and dates from my mac right?
thanks


no.

iSync just does PIM info - thats calendar, todo's, and address book.

The N82 is supported on the mac, complete with music and photo transferring, by the Nokia Multimedia Transfer application. it's on the nokia mac webpage. basically, the NMT application acts like HotSync for PalmOS, or ActiveSync for WindowsMobile: there will be an icon in your menu bar and it looks like 2 arrows curving toward each other. when you plugin your phone (or click the icon and choose bluetooth) the software starts syncing up Media related items:

iTunes: it can fill up a specified % of your device with random music and/or sync up a specified group of playlists (i use smart playlists to sync up my newest and least recently heard tracks)

iPhoto: your N82 will look like any other digicam to iPhoto, allowing you to use iPhoto directly to move the original pics from your phone and into your mac. you can also setup NMT to automatically copy optimized (320x240) versions of photos and/or videos back to your N82 - i have mine setup to keep copies of all photos i've ever taken with my nok right on my device. i've got about 900 right now, and the auto resizing keeps them small enough to do so.

Bookmarks: safari bookmarks and S60 Browser bookmarks can be synced up. to be honest, this relatively new feature doesn't work well for me (i've lost my bookmarks a few times) so be careful with it. i still use it, but every now and then i copy my bookmarks elsewhere on my mac so i can recover if it goes funky on me again.

so:
iSync = Address Book + Calendar and Todo's
NMT = Music, Photos, Videos, Bookmarks

have fun :)
