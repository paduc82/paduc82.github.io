john_barnett say:

Hello Suzieq25,

 

You do not say which file you are trying to access.

 

Some files remain hidden for a reason. The main reason being that they are system files and to 'reveal' them would be dangerous, i.e., there is a chance that someone who doesn't really know what they are doing could delete them and compromise the system.

 

To unhide system files you also need to uncheck the hide protected system files option. As Wunders has said, the system will complain that it doesn.t want to do this, but if you click the Yes button it will display them anyway. Just remember that you shouldn't delete any of the system files. If you do  your copy of Windows may not boot.

To the best of my knowledge, there is no stored repository you can clear. I think the drop-list of suggested entries is created on-the-fly by Explorer --- culled from the data in other files.

This means the annoying entries won't disappear until they have been cleared from all files.

Some quick testing of files from different albums with different artists shows that if I select multiple files with different Album Artists and edit that field in the Details pane with the letter 'Y', the Album Artist info for all those files is changed to 'Y'. But if I do the same with Contributing Artist, 'Y' is added to the existing Contributing Artist data of each file --- not much help in your situation!

But if the Contributing Artist info is identical for multiple files, they can all be corrected simultaneously.

There are 3rd-party tools for bulk editing of mp3 metadata, but I'm not that familiar with them,so can't make any recommendations.

If you want to go at it with brute force, open your top-level music folder. In the Search box, type kind:muisc. This will give you a view of all music files. Select Details and sort on Contributing Artist --- this will allow you to edit all identical bad entries at one time. Not sure how much help that will be to you.
