Word has never been consistent in this regard with RTL languages. The problem is due to the fact that while text in Persian and Arabic is RTL, numbers are LTR. (Similar thing could happen in dates with the use of "/"). Besides, we sometimes want to read those numbers with hyphens from right to left; sometimes from left to right.

For example, in Arabic, if I want to type "10-15" (10 to 15), I would want "10" to be to the right and "15" to the left:

‫١٠-١٥‬

However, If I want to type the phone number 00-123456, I would want "00" to stay to the left and "123456" to stay to the right, as in LTR languages:

‭٠٠-١٢٣٤٥٦‬

Since users have different versions of Word, I think that one solution is to type those numbers with the English keyboard. I know that the numerals will look Arabic (that is, the numerals used in English...), but at least it's consistent.

==========    ANSWER:  FIND a name in a column and scroll to it,  putting destination at top of vew for each repeat click / search down a column.  this first version is just a single search metod for:  MATCH CASE.  you might want to match a whole cell  -OR-  values,  then see the next version below (to be a little longer).

option explicit

Private Sub CommandButton2_Click()  'QUICK SRCH COL:  find a name in a column & do repeat clicks from cursor selectiont to find the next copy

    Dim G8 As String    'workcell shows:  BH16  for name/ word being searched for
    G8 = RANGE("G8")  'cell has:  SUBSTITUTE(SUBSTITUTE(CELL("address",$BH$16),"$",""),"","")
   

    Dim namX As RANGE 
    Dim r As RANGE
    Set r = RANGE(ActiveCell, Cells(Rows.Count, ActiveCell.Column).End(xlUp))

'match case search:

      Set namX = r.find(what:=ActiveSheet.RANGE(G8), After:=ActiveCell, LookIn:=xlFormulas, LookAt:=xlPart, _

        searchorder:=xlByColumns, searchdirection:=xlNext, MatchCase:=True, SearchFormat:=False)
       

      If Not namX Is Nothing Then
        ActiveWindow.ScrollRow = namX.row - 1
        namX.Select
      Else
        MsgBox ActiveSheet.RANGE(G8) & " not found!"
      End If
end sub

'VERSION FOR SETTINGS:  CAPS, WHOLE CELL or VALUES

Private Sub CommandButton2_Click()  'QUICK SRCH COL:  find a name in a column & do repeat clicks from cursor selectiont to find the next copy
    Dim G6 As String    'workcell settings: srch case, whole cell or values
    G6 = RANGE("G6")
    Dim G7 As String    'workcell shows:  BH16  for name/ word being searched for
    G7 = RANGE("G7")
    Dim G8 As String    'workcell shows:  BH353:BH1832
    G8 = RANGE("G8")
    
    Dim namX As RANGE   '<<  NAMXXX namxxx FINDXXX findxxx
    Dim r As RANGE
    Set r = RANGE(ActiveCell, Cells(Rows.Count, ActiveCell.Column).End(xlUp))
    
    
      If RANGE("G6") = "x" Then       'match case
        Set namX = r.find(what:=ActiveSheet.RANGE(G8), After:=ActiveCell, LookIn:=xlFormulas, LookAt:=xlPart, MatchCase:=True, SearchFormat:=False)
      
      ElseIf RANGE("G6") = "y" Then   'match case & cell
        Set namX = r.find(what:=ActiveSheet.RANGE(G8), After:=ActiveCell, LookIn:=xlFormulas, LookAt:=xlWhole, MatchCase:=True, SearchFormat:=False)
      
      ElseIf RANGE("G6") = "z" Then   'match case, cell & value
        Set namX = r.find(what:=ActiveSheet.RANGE(G8), After:=ActiveCell, LookIn:=xlValues, LookAt:=xlWhole, MatchCase:=True, SearchFormat:=False)
      
      Else                            'no case, no whole cell, yes values
        Set namX = r.find(what:=ActiveSheet.RANGE(G8), After:=ActiveCell, LookIn:=xlValues, LookAt:=xlPart, MatchCase:=False, SearchFormat:=False)
      End If

        namX.Select
     
      Else

      End If
end sub
