The drive can be any letters except maybe A and B (FDDs). You get problems because a lot of apps record the installation CD/DVD drive letter within their configs.. That could be recorded practically anywhere, registry, an ini file, within the program's user options etc.. So if a game was installed from D: for example and it needed the game disk in the drive in order to run it would always look for that disc in D: drive. Unless you get a prompt allowing you to "browse" to the required drive/folder, you'd either have to manually change the disc location information for each of the affected programs or reinstall them using the new DVD/CD drive letter.

You can choose between Stereo output, and 3D spatial sound through "Windows Sonic", or if you have purchased Dolby Atmos for headphones, you can also use Dolby Atmos for headphones.

Since the stereo output connectors are the same on OMTP and CTIA plugs, the headset should at least provide sound coming from the ear speakers. 

As XBF Diego M has explained, the headset might have the Microphone and ground connectors reversed. If so, you can't get the microphone work, try buying a cheap "OMTP to CTIA" adapter. Since that adapter simply crosses two wires, the adapter works both ways. So you can also get a "CTIA to OMTP" adapter and it's the same thing.

You can get these online or in almost any cellphone store.

Just make sure that the cable doesn't cost much more than a coffee-to-go, or you're being ripped off.

You have probably set up a loop by doing this. The automated reply clearly can't be delivered, so it is bounced by a mail server. The bounce message itself trigger a new automated reply, which bounces again and so on.  

Disconnect from the network (switch off Wi-Fi or unplug the Ethernet cable if necessary), then deactivate the message rule - this will stop it responding to incoming bounce messages. Then empty the outbox.

You should examine the source of a bounce message to find out why the automated response isn't being delivered.
