Sub MergeToExcel(strFileName As String)
Dim wrkbk As Excel.Workbook
Dim wrksht As Excel.Worksheet
Dim db As DAO.Database
Dim rs As DAO.Recordset
Dim strSQL As String
Dim intRow As Integer
'C:\data\form1.xlsx
Set wrkbk = Excel.Workbooks.Open(strFileName)
wrkbk.Application.WindowState = xlMaximized
wrkbk.Application.Visible = True
Set wrksht = wrkbk.Worksheets("Form")
Set db = CurrentDb
strSQL = "SELECT QstnID, QstnLvl1, QstnLvl2, QstnLvl3, QstnText, Rspns, [Number of Responses], MSOName, DelegateName " & _
"FROM qselForMerge"

Set rs = db.OpenRecordset(strSQL)
With rs
.MoveFirst
wrksht.Range("rngMSOName") = rs.Fields("MSOName")
wrksht.Range("rngDelegateName") = rs.Fields("DelegateName")
intRow = 7
Do Until .EOF
wrksht.Cells(intRow, 1) = .Fields("QstnID")
wrksht.Cells(intRow, 2) = .Fields("QstnLvl1")
wrksht.Cells(intRow, 3) = .Fields("QstnLvl2")
wrksht.Cells(intRow, 4) = .Fields("QstnLvl3")

.MoveNext
intRow = intRow + 1
Loop
.Close
End With
Set rs = Nothing
Set db = Nothing
wrkbk.Save
wrkbk.Close

End Sub

max, singapore say:

What you seek to do -- ie a droplist to show both codes & descriptions but to place only one of the 2 into the linked cell -- is possible if you use a Combo box from the control toolbox toolbar
Try tinkering with creating one along these lines

Assume your source reference data as posted is in Sheet1!A2:B7
where col A = Codes (eg C-001), col B = Descriptions (eg Microsoft)

Then in say, Sheet2,
Draw a combo box from the control toolbox toolbar (make it wide enough) 
Right-click on the combo box > Properties

Set the properties of the combo box to:
ListFillRange: Sheet1!A2:B7
LinkedCell: B2
BoundColumn: 1
ColumnCount: 2
ColumnWidths: 60 pt;30 pt

Click "Exit Design Mode" on the Control Toolbox toolbar, and test it out. The combo box droplist will display both the codes & descriptions, while making a selection from the droplist will place only the code (ie data from the BoundColumn: 1) into the link cell: B2.

If you change it to 2 for BoundColumn, then it'll place the Descriptions (ie data from the 2nd col) into the link cell.

Those are the inherent characteristics/constraints that come with using the combo box. It's unlike DV droplists. You could try posting in .programming for a sub to ease the "copying"/propagation of multiple combo boxes down a column
