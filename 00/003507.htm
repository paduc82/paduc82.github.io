[Critical bug] WebDAV stopped working in Insider builds - Microsoft Community

ktxxx say:

[Critical bug] WebDAV stopped working in Insider builds

The WebDAV client (webclient service?) does not work anymore if the server uses custom HTTP port.

I tested in the latest build, but the problem probably exists in older Insider builds too.

I captured the network traffic (with Fiddler proxy) to find out the problem's origin and it looks like the "Microsoft-WebDAV-MiniRedir" (v. 10.0.16257) tries to access the WebDAV server on port 80 (HTTP) or on port 443 (HTTPS) even if a custom port was specified.

If I try to mount a server as \\mywebdavserver.com@8080\ or http://mywebdavserver.com:8080/ the Mini Redirector sends the HTTP request with "Host: mywebdavserver.com" to port 80 instead of with "Host mywebdavserver.com:8080" to the port 8080.

If this rolls out in a production release then it can break various services.

I found bug reports regarding this issue already, for example here [ https://community.cryptomator.org/t/connecting-drive-failed-system-error-53/48 ] an user reports the exact same bug for the build 16241.
