    Flood fill page number field - Microsoft Community

    wild child say:

    Flood fill page number field

    I have Office 2007 on a computer running Windows 7 SP2.  I have recently learned how to create my own headers since Word 2007 doesn't have any like how I want my headers to look.  Now I'm looking to see if anyone can tell me how to create a page number field that is a solid color with white numbers.  The colored page number field itself will be .4 high by .8 wide.  The numbers are formatted Ariel, size 20, and bold.  This is what I want the page numbers to end up looking like:

    jay freedman say:

    Apparently you haven't given sharing permissions to the OneDrive page, so we can't see that either. Your initial description is probably good enough, though.

    You can't set the size of the page number field itself independently of the size of the font assigned to it. What you need is a separate box that you can size and fill with color, and a properly formatted page field inserted as the text inside the box.

    Although you could use a text box or a table cell as the box, a rectangle from the Insert > Shapes gallery is probably the easiest to work with. Follow these steps:

    • With the cursor in the header, click Insert > Shapes and click the rectangle tool.
    • Draw a box in the header, approximately the right size (you'll refine it later). The Drawing Tools tab will appear on the ribbon.
    • On the Drawing Tools tab, click the Text Wrapping button and choose In Line With Text.
    • In the Size group on the far right on the same tab, set the height and width measurements as desired.
    • In the Shape Styles group on the same tab, click the Shape Fill button and choose the background color for the rectangle. Then click the Shape Outline button and choose No Outline.
    • Right-click the rectangle and click Add Text.
    • Press Alt+Shift+P to insert a page number field. Select the number.
    • On the Home tab, set the font to Arial 20 pt, and set the font color to white. In the Paragraph group, click the Center button.
    • If you see a gray rectangle around the number, go to the Options > Advanced dialog and set the field shading dropdown to either Never or When Selected.
    • Close the header area. (The color of the rectangle will be dimmed. The full color will be visible if you print the document or convert it to PDF.)

    john - project say:

    Chris,

    One of the long standing problems with linked structures in Project (not openly mentioned by Microsoft) is that they are prone to corruption. The best practice ground rules are: never rename, move, overwrite, or "save off" to another location, any of the files in the structure. Ideally all files should reside in a single directory on a local drive and not be operated over a network.

    It is possible to use linked structures successfully but it requires a very dedicated file management protocol. So what is the "safer" approach? Use Project Online or Project Server. Those are designed for enterprise collaboration. If those aren't viable for you, then you can create a static master file locally and upload that to OneDrive, SharePoint or wherever for coordination. What is a static master? It is a separate single file that is a snapshot in time of all the subproject inserted into it. It is created by un-checking the "link to project" option in the lower right corner of the Insert Project window. At static master must be re-created anytime an update of all files is needed but this process can be automated with a macro.

    Hope this helps.

    John
