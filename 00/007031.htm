stefan blom say:

You can run a macro which deletes header and footer content and then sets the "Different first page" option for each section:

Sub DeleteHeadersFootersSetDiffFirstPage()
'Macro created by Stefan Blom, MVP, April 2018
Dim s As Section

For Each s In ActiveDocument.Sections
s.Headers(wdHeaderFooterEvenPages).Range.Text = ""
s.Headers(wdHeaderFooterFirstPage).Range.Text = ""
s.Headers(wdHeaderFooterPrimary).Range.Text = ""

s.Footers(wdHeaderFooterEvenPages).Range.Text = ""
s.Footers(wdHeaderFooterFirstPage).Range.Text = ""
s.Footers(wdHeaderFooterPrimary).Range.Text = ""

s.PageSetup.DifferentFirstPageHeaderFooter = True

Next s
End Sub

suzanne s. barnhill say:

In a copy of the document, access the footer. Enable Different First Page for the first section and disable Different Odd and Even for that and all the subsequent sections (use the Next button in the Navigation group of the Header & Footer Tools | Design tab to skip from one section to the next). As you go, link each primary footer to the previous one (Link to Previous in the Navigation section) if it is not already linked. You'll be asked if you want to replace the current content with the content of the previous one; say yes. This is a bit tedious but can be done quite quickly.

If the format of the new document is very different from the current one (margins, for example), you might be better advised to start by deleting section breaks (since each one contains section-level formatting that will copy over). To do this, search for ^b and replace with nothing. Once you've pasted the content into the new document format, you can restore the section breaks as needed.

rohn007 say:

In that sort of situation you have to look at the actual link, not the display text.

Hyperlinks have 2 elements. The first is the display text we see, like:

https://www.microsoft.com/en-US/microsoft-365/.........

(or some words that are formatted as a hyperlink, one convention is blue text with an underline). That "address above is not a proper web address because it has the "..." at the end.

If you hover the mouse over the hyper link most browsers will display the underlying true address, somewhere. For example, in FireFox it is displayed in the very bottom left corner of the application window.  Or you can right click on the link, select "Copy Link Location" (or something equivalent), then paste into any place that accepts text (like this input box or a Word document).  Then you can inspect the actual web address, for example:

Microsoft Community.

has the actual web address:

https://answers.microsoft.com/

This is the root address of this forum, and we can tell it really is at microsoft.

Here is another example, it goes to shoppers drugmart company in .ca (canada). The part after /? is a private code used by the website.

http://click.e.shoppersdrugmart.ca/?qs=a2252c424a3de8dcc498f0cf89aad67e2ff43bf52953407d8c5bc4c486ff1104e823baf31d1a6f1303dd7fde68a5a155bb5c5145d67f96e26b57cf28f4fe545a

Other addresses go through "redirection services".  It first goes to a seemingly random location, that is then translated into the final destination. But you have no way of telling what that final destination is. That is the type of web link you should avoid.  (sorry I can't find an example right now)

e.j. gun say:

The Application.OnTime function expects another argument - the procedure you want to run at the stated time.  So, for example, if you have a subroutine named "OpenMyForm", then you would use:

Application.OnTime EarliestTime:=Now + TimeSerial(0, 0, 1), Procedure:="OpenMyForm"

 

Then, in your "OpenMyForm" subroutine, you would have:

 

Private Sub OpenMyForm

    MultiFamilyDF.Show

End Sub
