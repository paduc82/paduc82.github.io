ramakant_yadav say:

The answer to how to model a class that contains several instances of another class :-

1. Open "UML Model Diagram" template.

2.  Open "UML Static Structure" stencil

3.  Drag and drop "Class" shape on the page.

4.  Rename the class to "Book"

5.  Drag and drop another "Class" shape on the page.

6.  Rename the class to "Library"

7.  Double click "Library" class in the model explorer.

8.  Select "Attributes".

9.  In the "Attributes" table write "Book_in_Library" or any name you want in the "attribute" column 

10.  In the "Type" column select "Top Package :: Book"

11. In the "Multiplicity" column select "1...*" from the drop down if you want "Book_in_Library" to have atleast one book or you can select "0...*" depending upon your requirements.

12. Click OK

13. Select the Library class on the page and right click on it.

14. Select "Shape Display Options"

15. Check the "Attribute multiplicity" box.

16. Click OK

diego rios_06 say: Hi,

I'm Diego Rios, an Independent Advisor willing to help other Microsoft customers.

You can try the following procedures below:

I suggest you to perform the uninstall Office 2016 with the Microsoft tool and then perform the procedures for manual uninstall, it was also recommended to delete all Office activation keys through the CMD are install on the PC. Please, follow up the steps below::

1. Uninstall Office from a PC computer:
https://support.office.com/es-es/article/desins...

2. Uninstall Office manually:
https://support.office.com/es-es/article/desins...

3. Next, you must run the following commands in CMD to disable previous versions of Office installed on your computer:

Run the following commands, look for the System Symbols program (Command Prompt) and right click and open as Administrator.

Search the full path to find out which folder Office is installed on.
CD .. Until you get to directory C:

cd "C: \ Program Files (x86) \ Microsoft Office \ Office16" or cd "C: \ Program Files \ Microsoft Office \ Office16"

Then, execute the following commands
cscript ospp.vbs / dstatus
script ospp.vbs / rearm

Observe the result and copy the KEY which is 5 digits and paste them in the next command.

cscript ospp.vbs / unpkey: GVGXT

After removing all the keys, run the following command for activation.
cscript ospp.vbs / act

4. Remove all Microsoft Office16 / 15 credentials in Control Panel >> Credentials Manager >> Windows Credentials >> then remove all the Microsoft Office credential.

5. Restart the computer, and then log in to the user's Office 365 Portal and perform the Office 2016 installation again from the Office 365 Portal.

Please let me know if this helped and works.
