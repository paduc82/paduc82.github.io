That's fine, but you need to add at least one line of code for each field that needs a modified version pf the copied value.

I would have suggested that kind of loop, but was unsure if the autonumber field was the first field in the tablle and you do have to remember that whenever you rearrange the fields in the table's design.

The NewPK line assigns the new record's PK to a VBA variable just in case you have a need to use it elsewhere in your code.  If you don't need it anywhere else, remove that line

You never did say where/when you want to do this.  If it is operating on the current record in a form, then you do not need to open another recordset and the code can be even more efficient:

With Me.RecordsetClone

       Next a
       !f3 = !f3 - quantityavailable
   .Update
End With

steve rindsberg say:

By definition, PPTX files *cannot* contain macros, so all you really need to do is File | Save As then choose PPTX as the Save As type, give the presentation a name and save it.  PowerPoint will give  you a warning that PPTX files can't contain macros and is it ok to remove them ... and of course you'll say "You betcha" since that's what you WANT to do.
