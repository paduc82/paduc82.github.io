ronrosenfeld say:

This can be done with a macro using regular expressions.  I have made a few assumptions which are easily changed.  But it works on your posted data

  • If there are "designations", they all start with RN, and there will be no names starting with RN
  • Any <spaces> within the strings are true spaces (ASCII code 32) and not nbsp's (ASCII code 160)
  • All names are the only thing in the cell, with no leading spaces
  • The data starts in A1 with no header
  • The results will go in columns B:D

To enter this Macro (Sub), <alt-F11> opens the Visual Basic Editor.
Ensure your project is highlighted in the Project Explorer window.
Then, from the top menu, select Insert/Module and
paste the code below into the window that opens.

To use this Macro (Sub), <alt-F8> opens the macro dialog box. Select the macro by name, and <RUN>.

Option Explicit
Sub SplitNames()
    Dim vSrc As Variant
    Dim vRes() As Variant
    Dim rDest As Range
    Dim RE As Object, MC As Object
    Dim I As Long, J As Long

'Get Source Data
'  Modify according to your data setup
vSrc = Range("a1", Cells(Rows.Count, "A").End(xlUp))

'Set location for results and clear out the existing data
Set rDest = Range("B1:D1")
rDest.EntireColumn.Clear

'Set up Results Array
ReDim vRes(1 To UBound(vSrc), 1 To 3)

'Initialize Regular Expression Engine
Set RE = CreateObject("vbscript.regexp")
With RE
    .Global = True
    .MultiLine = True
    .Pattern = "^((?:.(?!\w+\s*(?:RN|$)))*)\s*(\w+)\s*(RN.*)?"
End With

'If there is a label row, process that separately
'Otherwise, start with row 1

For I = 1 To UBound(vSrc)
    If RE.test(vSrc(I, 1)) Then
        Set MC = RE.Execute(vSrc(I, 1))
        For J = 1 To 3
            vRes(I, J) = MC(0).submatches(J - 1)
        Next J
    End If
Next I
    
'Write results to destination
rDest.Resize(rowsize:=UBound(vRes, 1), columnsize:=UBound(vRes, 2)) = vRes

End Sub

As far as I'm aware, Microsoft has never officially documented this (for example,
http://office.microsoft.com/en-us/word-help/create-forms-that-users-complete-in-word-HA010030746.aspx
ignores the issue altogether), but the fact is that legacy form field protection
interferes severely with several types of content controls.
 
Best practice currently is to use  *either* content controls  *or* legacy controls, but
not both in the same form. If you use content controls,  *do not* apply forms protection.
 
If you want the effect of forms protection -- that nothing outside the form fields can be
edited -- when using only content controls, here's how:
 
Select the entire document (or at least the parts that need protection against editing,
inlcuding at least one content control in the selection). On the Developer tab of the
ribbon, click the Group button and choose Group from the menu.
 
If you would rather use legacy form fields, replace all the dropdown content controls with
legacy dropdowns, and replace the date picker with a popup calendar as described at
http://www.gmayor.com/popup_calendar.htm.

I am looking for the mathematical formula behind the RATE or IRR - Microsoft Community

williamlucaswb say:

I am looking for the mathematical formula behind the RATE or IRR function

Does anyone have a simplified formula behind either of these functions?
joeu2004 say: William wrote:
Subject:  I am looking for the mathematical formula behind the RATE or IRR function
Does anyone have a simplified formula behind either of these functions?

If there are periodic payments, there is no "simple formula".  RATE and IRR are determined by an iterative algorithm.

The IRR help page describes some of the constraints of the algorithm.  I presume that RATE uses the same constraints, although the Excel 2007 RATE help page is less explicit.

There are various iterative algorithms that Excel might use.  I believe some Microsoft KB article states that Excel uses the Newton-Raphson method.  See the wiki page (click here) [1] for a general description.

I have lost track of the KB number/URL.  If I find it later, I'll edit this posting.  But IIRC, the KB description is ambiguous:  the details sound more like the Secant Method (click here) [2].


-----

If there are no periodic payments, the RATE (IRR) can be computed directly as follows:

rate = (-fv/pv)^(1/nper) - 1

where fv and pv have opposite signs.
