bmacnfry say:

I struggled with this like so many others on my HTC (Titan) made Verizon XV 6800.  In another thread I found mention that the problem is related to transferring large amounts of data in a single category (contacts, calendar, notes, etc.) in a single sync - over 100 items seem to be the trigger.  Using a process of elimination I found that I only received sync errors when synching my calendar.  My solution was to backup my Outlook calendar in case of error then stop synching my calendar in WMDC.  This erased all the calendar entries in my device.  Next I set up to sync my calendar and only the 2 week prior appointments.  This worked fine.  From there I took baby steps increasing my sync setting to 1 month prior, 3 months prior, 6 months prior and finally all past appointments.  Doing it in this fashion kept the entries I was syching to less than 100 each time.  My total entries in Outlook are 409 including recurring appointments.  If you have total entries exceeding 100 in your Outlook calendar, try this method to see if it allows you to complete the sync without error.  Worked for me - hope it works for you!

jeeped say:
Looking at one portion of your logical criteria, it would seem that it is ALWAYS going to be True.

CompletionStatus <> "Withdrawn" OR CompletionStatus <> "Transferred"

If CompletionStatus = "Withdrawn" then it's True because CompletionStatus <> "Transferred". By the same token, if CompletionStatus = "Transferred" then it's True because CompletionStatus <> "Withdrawn". If CompletionStatus <> either "Withdrawn" or "Transferred" it's True again.

Perhaps the LearnerStatus = "High Risk" needs to be doubled and grouped with each of the CompletionStatus possibilities.

(LearnerStatus = "High Risk" AND CompletionStatus <> "Withdrawn") OR (LearnerStatus = "High Risk" AND CompletionStatus <> "Transferred")

.. or more completely,

=FormatNumber(COUNT(IIF((Fields!LearnerStatus.Value = "High Risk" AND Fields!CompletionStatus.Value <> "Withdrawn") OR (Fields!LearnerStatus.Value = "High Risk" AND Fields!CompletionStatus.Value <> "Transferred"),1,nothing),"DataSet1"),0)

If that is not successful, could you expand a little on the description of what this logical is intended to accomplish?
