Suspected bug in Word 365 "Mirror indents" behavior - Microsoft Community

ericqc say:

Suspected bug in Word 365 "Mirror indents" behavior

The Paragraph Format dialog box has an option to turn on “Mirror indents”. When it is checked, the “Left” and “Right” Indentation settings change to “Inside” and ”Outside”. This is supposed to allow a user to apply indentation relative to the inside or outside margin for duplex printed books.

If a Hanging indent is included with Inside & Outside indentation, the paragraphs are displayed correctly, with the inside and outside indent amounts taking effect on paragraphs that flow from one page to another.

However, if a First line indent is specified, the “inside” margin on even pages only is increased by an amount equal to the specified first line indent. The result is that paragraphs on even-numbered (verso) pages are effectively narrower by an amount equal to the first line indent. This is the bug.

Paragraphs on odd-numbered pages have indents as expected. Moreover, any spillover lines from incorrectly-indented paragraphs on even-numbered pages are indented without the extra “inside” indent.

I created a document to illustrate the behavior. The following screen shot shows 3 pages with a first line indent of 0.5 inch, and used the “Mirror indents” option to set with an Inside indent of 1.0 inch and an Outside indent of 2.5 inches. As can be seen in the first highlighted paragraph, the lines have the expected width on page 1, but are narrower by ½ inch on the inside margin when they spill over onto page 2. The second highlighted paragraph shows the reverse. The entire inside margin on page 2 incorrectly has ½" of extra indent on the right (inside).

The next screen shot shows similar content in the same document, but now the paragraphs have a Hanging indent of 0.5 inch (vs the First line indent of 0.5 inch used on the previous pages). Now the paragraphs clearly have the same width on all pages — and the green highlighted paragraphs show that lines overflowing onto a new page retains the same indent.

I suspect this is a feature that is used infrequently, and an online search shows that there has been confusion about it over the years. Has anyone else encountered this behavior — and if so, is there a workaround for it?

And, if this is in fact a bug, how does one go about alerting the appropriate person at Microsoft? I have the 30KB DOCX file that produced the behavior if anyone wants to examine it.
