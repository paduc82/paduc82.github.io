For example, I have a heading " Part A [1234]" in the paragraph, and the same heading " Part A [1234]" in the navigation pane.  I want to change the heading in the navigation pane to "Part A" , but keeping the heading in the paragraph as " Part A [1234]".  The reason to do this is because I would like to save this word document as pdf and make those renamed headings as bookmarks in pdf. 

Thank you for your help in advance.

suzanne s. barnhill say:

Assuming that you also want just "Part A" in your TOC, you can accomplish this with a style separator.

  1. Create a style with formatting identical to your heading style but with outline level of Body Text.
  2. Using the heading style, type "Part A" and press Enter.
  3. Apply the non-heading style and type " [1234]."
  4. Place the insertion point anywhere in the heading paragraph and press Alt+Ctrl+Enter to insert the style separator.
  5. The two paragraphs will appear to join, but only the heading text will appear in the Navigation pane and TOC.

rdwray,

Maybe...
Here are some general guidelines to use when automating Excel...

1. Set a reference to the primary Excel objects used in your program.
   Dim xlApp As Excel.Application
   Dim WB As Excel.Workbook
   Dim WS As Excel.Worksheet

   Set xlApp = New Excel.Application
   Set WB = xlApp.Workbooks.Add
   Set WS = WB.Sheets(1)

2. Use the appropriate reference Every Time you make reference to a spreadsheet.
   Do not use Range(xx) - use WS.Range(xx)
   Cells should be WS.Cells(10, 20) or WS.Range(WS.Cells(10, 20), WS.Cells(20, 40))

3. Avoid the use of ActiveSheet, ActiveWorkbook, Selection etc.
   Use your object references.

4. Avoid the use of the "With" construct.

5. Set all objects to Nothing in the proper order - child then parent.
   Set WS = Nothing
   WB.Close SaveChanges:=True 'your choice
   Set WB = Nothing
   xlApp.Quit
   Set xlApp = Nothing

Violating any of these guidelines can leave "orphans" that still refer
to Excel and prevent the application from closing.

Hidden IM (Chat) Client in Nokia N85 - Microsoft Community

travla---01 say:

Hidden IM (Chat) Client in Nokia N85

Hi there,

Has anyone had a play with the hidden IM client in the N85 (and I presume other FP2 phones as well)? I stumbled on it a few days ago, and have it up and running with MobJab. It is basically a XMPP/Jabber like Nokia Chat or Nokia Contacts on OVI, except it lets you define your server address so you don't need to get all your friends on board with OVI. It is very similar to the IM client on my old N80ie, and it is totally integrated with the Symbian OS, which makes it a pleasure to use. Here are some good features on it that are missing from Contacts on OVI and Windows Live for FP2:

 

1.  You can pick which days and in between which times to Automatically Login

2.  When you receive a message, you get a New IM Alert same as the New SMS Alert (it also counts how many new messages you have unreadand provides a link to the IM Client)

3. The IM message Icon (little balloon) is also displayed in the screen saver

 

The easiest way to find the application is to do a search for IM - you can even go straight to applications under the search categories and pick it straight out of the list of applications.

 

If anyone knows how to add a shortcut to it in a menu/folder, that would be great. From what I have read, this is not so easy.
