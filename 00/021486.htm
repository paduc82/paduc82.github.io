rich michaels - mvp (greatcirclelearning.com) say:

Whatever you do, don't name your template Normal.dotm and replace end user's Normal template, and there's a few reasons...

  1. You'll cause Word some processing problems because you'll have destroyed things it expects.
  2. You'll destroy all of the end user's customization for styles, auto text, macros, etc.
  3. Your code will be lost when someone deletes the Normal template, allowing Word to rebuild a new one, because of this fairly standard problem resolution method.
  4. You can't password protect your code and all routines become public

Keep your macros in your own template and password protect them by using the Tools > Properties > Protection function. IMPORTANT - the encryption methods between MacOffice 2011 and WinOffice 2016, 2013, 2010 are not compatible and if you add a password to your project using a PC version of Word, you will not be able to unlock it from the Mac version of Word. The PC versions of the VBE can unlock Mac 2011 projects but the Mac VBE in 2011 cannot unlock projects encrypted on a PC version of the VBE. (Visual Basic Editor)

Where to place your add-in?

For "All Users" deployment in a MacWord 2011 environment:

/Applications/Microsoft Office 2011/Office/Startup/Word

For "All Users" deployment in a MacWord 2016 environment:

/Library/Application Support/Microsoft/Office365/User Content.localized/Startup.localized/Word

Special Note about this path... this is the System Library path. You will most likely have to create the path starting with the Office365 folder. The folders named User Content and Startup have localized names on non-English language systems, so depending on were you are deploying, the folder names may have to be different. In all cases, do not include the ".localized" tag on the path. That tag is hidden and I'm only showing it for your reference to these two localized labels.

Beginning with the Office365 directory, this a Sandboxed directory for "all user" processing.

For "Single User Only" deployment in a MacWord 2016 environment:

/Users/<user account>/Library/Group Containers/UBF8T346G9.Office/User Content.localized/Startup.localized/Word

Special Note about this path... this is home path for the individual user. Beginning with the UBF8T346G9.Office directory, this is the Sandboxed directory for the end user. As with the all user directory path, the User Content and Startup labels may be localized and do not include the ".localized" tag in any reference to this path in your code.

Hope this helps

Private Sub Worksheet_SelectionChange(ByVal Target As Range)
