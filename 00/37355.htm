rdrewd2 say:

The mid-July date, is that on the within the FileHistory folder, or on the FileHistory folder itself?  The date on the FileHistory folder will only reflect when that folder was originally created, the date on the $OF folder should match with your most recent backup.

But rather than have you hunt around for folders and dates, an easy way to see if it's still working properly:  create a junk file of any type, and save it in any of your user folders.  Next, add something to an existing document; type "DUMBO2" at the top of an old Word document or something, and save it.

Now manually run a backup (click "Back up now").  After it completes, go inside the FileHistory folder on your EHD and navigate to the same user folder where you saved that junk file on your C: drive.  Is that new file in there?  Then, check the file you added something to; did it add the change?  If yes to both, then you know it's looking for new files and adding them, and scanning for and tracking changes to existing files also, so it's working as it should.

Hi Vanessa,

 

Here's the step by step procedure on CHKDSK.

 

1. Open "My Computer"

2. Right click on "C:"

3. Select "Properties"

4. Select "Tools" tab.

5. Click "Check now"

 

A pop-up window will appear, tick both checkboxes and click ok.

 

The program will request for a restart, click yes. The computer will automatically restart, if not, restart your pc manually.

 

The program will run on start-up. Wait for a few minutes. (or we can have a coffee while waiting). :)

 

When the chkdsk finishes it will restart again.
