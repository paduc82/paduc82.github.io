Hi Gerard,

Thank you for posting your query in Microsoft Community Forums.

MSCOMCTL.OCX is a set of 32-bit controls that were provided with the Microsoft Visual Basic 6.0 programming environment.

Go to this location- C:\Windows\SysWOW64 directory and see if MSCOMCTL.OCX file exist. If yes then follow the below steps:

Open an elevated command prompt â€“ In the start page type Command Prompt in the Search box, right-click Command Prompt, and then click Run as administrator. If you are prompted for an administrator password or for a confirmation, type the password, or click Allow.

When the black window opens with your command prompt, type this in:

  1. CD C:\Windows\SysWOW64 directory
  2. Press Enter
  3. Next Type - Regsvr32 MSCOMCTL.OCX
  4. Press Enter.


If you get a Window that pops up and says, "DllRegisterServer in mscomctl.ocx succeeded" then click OK and then try to restart your system and check if you can play the game again.

If the above steps do not work, then try to run the SFC (System File Checker) scan on the computer to fix the system file errors on the computer.

What is SFC (System File Checker)?

System File Checker is a utility in Windows that allows users to scan for corruptions in Windows system files and restore corrupted files.

To run SFC scan, follow the steps:

Open an elevated command prompt - Swipe in from the right edge of the screen, and then tap Search. Or, if you are using a mouse, point to the lower-right corner of the screen, and then click Search. Type Command Prompt in the Search box, right-click Command Prompt, and then click Run as administrator. If you are prompted for an administrator password or for a confirmation, type the password, or click Allow.

Next follow the below steps to perform sfc /scannow:

1. At the command prompt, type the following command, and then press ENTER: sfc /scannow. The sfc /scannow command will scan all protected system files, and replace corrupted files with a cached copy that is located in a compressed folder at %WinDir%\System32\dllcache.

The %WinDir% placeholder represents the Windows operating system folder. For example, C:\Windows.

To know more about SFC scan you can refer to the link below:

https://support.microsoft.com/en-us/kb/929833?ppud=4&wa=wsignin1.0

Stopped all non MS antivirus programs and cleaned them out of registry

Stopped all non MS processes via terminal & msconfig

Stopped all autoruns via task-manager

Stopped windows update via terminal & wuauserv

Deleted old installation files in C:=Windows>SoftwareDistrubution

setupact.log for the entire process

setuperr.log for the errors

MigLog.xml for the migration process

After this removal the W10 AU via the update-assistant went through smoothly.

For all cases, where the file Microsoft.NET.Native.Runtime 1.3_1.3.23901.0 does not show up as the cause for the failure, it is recommended to study the 3 logs in detail. They report any other possible issue.

This action makes sure that an existing configuration is not modified. If both provided drivers were installed, and the Bluetooth were connected to the computer, the Microsoft-provided driver would load because it is signed.

a. user say:

Hi,

Try running Microsoft Security Scan in the solutions section of this article:

http://support.microsoft.com/kb/968003/en-us

If you are not infected, then try downloading each update separately. To find each update, type the KB number in Google, followed by Windows 7 x64.

Run the update installer from the hidden Administrator account in Windows. To do this, open an elevated command prompt, and type in

net user administrator /active:yes

Log off, and you'll see the account. Note that there is no password.

Install the updates in that account, once you're finished, disable the account by following these steps:

1. Open an elevated command prompt

2. Type in net user administrator /active:no
