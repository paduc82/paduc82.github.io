bernie deitrick say: Set a reference to Word in your project, and use code like this, which will work if Word is or is not open, and if the document is or is not open.

Sub AutomateWord()
    Dim PageNumber As Integer
    Dim wrdApp As Word.Application
    Dim wrdDoc As Word.Document
    Dim FileToOpen As String
    Dim strPath As String

    FileToOpen = "Excel Link test.docx"
    strPath = "C:\"

    'the next line looks to a cell to decide what page number to scroll to
    PageNumber = Cells(2, 7).Value

    On Error Resume Next
    Set wrdApp = GetObject(, "Word.Application")
    If wrdApp Is Nothing Then
        Set wrdApp = CreateObject("Word.Application")
        Set wrdDoc = wrdApp.Documents.Open(strPath & FileToOpen)
    Else
        On Error GoTo notOpen
        Set wrdDoc = wrdApp.Documents(FileToOpen)
        GoTo OpenAlready
notOpen:
        Set wrdDoc = wrdApp.Documents.Open(strPath & FileToOpen)
    End If

OpenAlready:

    On Error GoTo 0
    With wrdApp
        .Selection.Goto What:=1, Which:=2, Name:=PageNumber
        .Visible = True
    End With
    Set wrdDoc = Nothing
    Set wrdApp = Nothing

End Sub

john l kim say:

Hi Phil,

The email address we excluded is the email address that comes up when you run this command in Exchange Online PowerShell:

Get-UnifiedGroupLinks -Identity <groupalias> -LinkType Aggregators

With the "groupalias" being the alias or "identity" of the group/team they created. It should output along the lines of "AggregateGroupMailbox.A.xxxxxxxxxx". The "xxxxxxxxxx" portion would be a string of numbers looking like a date of some kind. So the email address you exclude (under "if the recipient is..." in the mail flow rule) would be: 

AggregateGroupMailbox.A.xxxxxxxxxx @ [your onmicrosoft domain]

Hope that clears it up.
