If you would like to try something a bit more refined use Resource Monitor.   E.g. start that from the TM Performance tab.  Then in the Overview tab, sort by Image.   Then scroll down to the last  iexplore.exe  and click its box to pin it.   Continue clicking until all iexplore.exe tasks are pinned.   (The reason to sort and scroll down to the last iexplore.exe task may now be evident.)   Then sort column  Average CPU descending.   Then press Home and Menu, A  (so, right-click  Analayze Wait Chain  on the iexplore.exe  task which is doing the least.)   19 times out of 20 this finds a Task which has a thread in deadlock chained to itself.   Killing the task via that dialog lets it retry in its own window via recovery.   ProcExp provides us the capability of killing separate threads instead of whole tasks but I tried using that a few times and found that the net result would be killing all threads which would then have the effect of killing the task, so now I just kill the deadlocked task.
