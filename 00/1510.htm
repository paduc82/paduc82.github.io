Command: Get-PublicFolder

Command: Get-PublicFolderItemStatistics -Identity "\Marketing\2013\Pamphlets"

Command: Get-PublicFolder -Identity "\Legal\Documents\Pending Litigation"

avoran say:

I think I've solved the problem myself, but will leave this here in case it's helpful to anyone else.

1) The header contains a blank line of text outside the table. That line had a paragraph spacing setting of 18 points. Cut that to 6 and voila. Can't believe I didn't figure this out sooner. :D

2) The footer is a bit trickier.  It contains a table (if yours doesn't you could probably set it up so that it did, at least if it's text). With the cursor inside the table, go to the Layout tab on the main tool bar, then the Properties button, then (on the Table tab in the popup window) to Positioning under "Text wrapping". Where it says "Top" enter some positive value; that space will now always be inserted between the footer table and the main text. :)

Logically, there ought to be a way to do this without resorting to tables. But in the meantime this worked for me.

ken sheridan say: To filter a parent form on the basis of a value in a referencing table on which a subform is based you need to filter the parent form to those records where the primary key of the parent form's underlying table has at least one match in the referencing table where the value in the relevant column is that on which you wish to apply the filter.  You'll find an example in FindRecord.zip in my public databases folder at:

https://onedrive.live.com/?cid=44CC60D7FEA42912&id=44CC60D7FEA42912!169

If you have difficulty opening the link copy its text (NB, not the link location) and paste it into your browser's address bar.

In this little demo file the form for to 'filter by a subform value' includes a combo box in which a contact is selected by name.  The form is then filtered to those projects to which the contact has been assigned.  This is done by the following code in the combo box's AfterUpdate event procedure:

    Const MESSAGETEXT = "No matching records found."
    Dim ctrl As Control
    Dim strFilter As String
    
    Set ctrl = Me.ActiveControl
    
    strFilter = "ProjectID IN(SELECT ProjectID " & _
        "FROM ContactProjects WHERE ContactID = " & ctrl & ")"
    
    If Nz(ctrl, 0) = 0 Then
        ' turn off filter
        Me.FilterOn = False
    Else
        If Not IsNull(DLookup("ContactID", "ContactProjects", "ContactID = " & ctrl)) Then
            ' filter form to name selected in combo box
            Me.Filter = strFilter
            Me.FilterOn = True
        Else
            ' inform user if no matching records found and show all records
            MsgBox MESSAGETEXT, vbInformation, "Warning"
            Me.FilterOn = False
            Me.Requery
        End If
    End If

As you see a subquery is included in the string expression assigned to the parent form's Filter property.  By means of the IN operator the the form is filtered to those rows in Projects where the primary key ProjectID value is present in a row in ContactProjects whwre the ContactID value is that of the combo box in which the contact is selected.

In my case the subform is not filtered to the selected contact, but it's record pointer is moved to the contact by the following code in the parent form's Current event procedure:

    ' select contact in subform if combo box is not Null
    If Not IsNull(Me.cboLastname) Then
        With Me.fsubContacts.Form
            .RecordsetClone.FindFirst "ContactID = " & Me.cboLastname
            If Not .RecordsetClone.NoMatch Then
                .Bookmark = .RecordsetClone.Bookmark
            End If
        End With
    End If

To filter the subform also the above code for the Current event procedure would be deleted, and the unbound combo box's AfterUpdate event procedure amended as follows:

    Const MESSAGETEXT = "No matching records found."
    Dim ctrl As Control
    Dim strFilter As String
    
    Set ctrl = Me.ActiveControl
    
    strFilter = "ProjectID IN(SELECT ProjectID " & _
        "FROM ContactProjects WHERE ContactID = " & ctrl & ")"
    
    If Nz(ctrl, 0) = 0 Then
        ' turn off filter
        Me.FilterOn = False
        Me.fsubContacts.Form.FilterOn = False
    Else
        If Not IsNull(DLookup("ContactID", "ContactProjects", "ContactID = " & ctrl)) Then
            ' filter form to name selected in combo box
            Me.Filter = strFilter
            Me.FilterOn = True
            
            ' filter subform to selected contact
            Me.fsubContacts.Form.Filter = "ContactID = " & Me.cboLastname
            Me.fsubContacts.Form.FilterOn = True
        Else
            ' inform user if no matching records found and show all records
            MsgBox MESSAGETEXT, vbInformation, "Warning"
            Me.FilterOn = False
            Me.Requery
            Me.fsubContacts.Form.FilterOn = False

        End If
    End If
