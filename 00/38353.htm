gts-nj say: Are you trying to boot your Windows installation used with the prior motherboard?  If so,
this is to be expected.  When NT derived versions of Windows are transferred to significantly different hardware (i,.e. without an identical or almost identical CPU and chip set) they will generally not be bootable.  The HAL (hardware abstraction layer) is customized during the original installation.   In Windows XP we can usually resolve this by doing an in-place repair reinstallation.  Unfortunately that is not an option in Vista and Windows 7.
  There is one product in the market that can be used to do this kind of transfer (assuming the Windows version permits it and is not a BIOS locked OEM product for example) - Acronis Universal Restore, but it is an add on to very expensive business products.
   You'll need to back up your data and do a clean reinstallation of Windows 7 to use the new hardware.  If your Windows was an OEM release that will raise other technical and licensing issues.

malkeleah say: Never get drivers from Windows Update. Get them from:

1. The device mftr.'s website; OR
2. The motherboard mftr.'s website if hardware is onboard; OR

1. Read any documentation you got when you bought the computer.
2. If the computer is OEM, go to the OEM's website for your specific model machine and look at the specs (you'll be there to get the drivers anyway)
3. Download, install and run a free system inventory program like Belarc Advisor or System Information for Windows.

Many things can stop this add wireless display adapter from appearing:

1. VPN software (netgear ReadyCloud software also adds a hidden VPN device)

2. group policy settings if you are on a domain

3. firewall rules on a domain (you should not normally have to change the Windows firewall if you are a consumer not on a domain)

4. third party anti virus and/or firewalls - recommend completely uninstalling these, restarting, and then checking to see if you can add a wireless display.

Many of the third party VPN solutions view WiFi Direct as a "Split Tunnel" and therefore a security risk and disable that functionality. Miracast a reliant on WiFi Direct to send the video singnal to the reciever. It is possible that the Built-in Sonic Wall Mobile Connect VPN will allow you to connect without the need of installing the Client. This can be found in the Modern UI Network Connections under VPN.

Other culprits are Hypervisors like VirtualBox that create virtual switches that also disable WiFi Direct.
