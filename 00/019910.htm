aaron zhu msft say: Hi Sandro,

Here are the answers to your questions:

Question 1: Is there a maximum size of a single PST or total PST's I can send to Azure Storage?

Yes, the single PST file limit in Office 365 import service is 10GB, if you have PST files larger than 10 GB, please break them into chunks. For the total PST file limit, I have done some research but found no information about it. However, if you encounter any issue when importing very large PST files using one import job, let us know and we’re glad to provide further assistance.

Question 2: The import will create a directory (pst name) at the root of the user's email box and within this I will see all folders of this PST imported?

If you leave TargetRootFolder blank in your CSV file, yes, the PST will be imported to a new folder named Imported located at the root level of the mailbox and you can see all folders of the imported PST file within Imported folder. Also, you can specify a folder in your CSV file and your PST file will be imported into this specific folder.

Question 3: The only way to directly import the inbox, sent items to the folders in your mail box users using the import Microsoft Outlook?

Yes, it’s not feasible to achieve the goal directly using Office 365 import service. As a workaround, you can split your PST file to different inbox pst file, sent items pst file, .etc. Then use Office 365 import service to achieve the goal. Or as what you mentioned, use importing email, contacts, and calendar from an Outlook .pst file.

Feel free to post back if anything is unclear.
