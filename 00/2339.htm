The following is a simple example of a query using a hypothetical table named MonthlySales, with columns MonthStarting (date/time data type, primary key) and SaleAmount (currency data type).  In this case the query simply returns the current and previous month's sales amounts:

SELECT FORMAT(MS1.MonthStarting, "mmmm yyyy") AS CurrentMonth,
MS1.SaleAmount AS CurrentMonthSaleAmount,
MS2.SaleAmount AS PreviousMonthSaleAmount,
MS1.SaleAmount - MS2.SaleAmount AS Difference
FROM MonthlySales AS MS1 LEFT JOIN MonthlySales AS MS2
ON DATEDIFF("m",MS2.MonthStarting, MS1.MonthStarting) = 1;

If you wanted to return only those months where the sales amount had increased from the previous month:

SELECT FORMAT(MS1.MonthStarting, "mmmm yyyy") AS CurrentMonth,
MS1.SaleAmount AS CurrentMonthSaleAmount,
MS2.SaleAmount AS PreviousMonthSaleAmount,
MS1.SaleAmount - MS2.SaleAmount AS Difference
FROM MonthlySales AS MS1 INNER JOIN MonthlySales AS MS2
ON DATEDIFF("m",MS2.MonthStarting, MS1.MonthStarting) = 1
WHERE MS1.SaleAmount > MS2.SaleAmount
ORDER BY MS1.MonthStarting DESC;

Conversely, if you wanted to return only those months where the sales amount had decreased or been unchanged from the previous month:

SELECT FORMAT(MS1.MonthStarting, "mmmm yyyy") AS CurrentMonth,
MS1.SaleAmount AS CurrentMonthSaleAmount,
MS2.SaleAmount AS PreviousMonthSaleAmount,
MS1.SaleAmount - MS2.SaleAmount AS Difference
FROM MonthlySales AS MS1 INNER JOIN MonthlySales AS MS2
ON DATEDIFF("m",MS2.MonthStarting, MS1.MonthStarting) = 1
WHERE MS1.SaleAmount <= MS2.SaleAmount
ORDER BY MS1.MonthStarting DESC;

If you wanted to return only those months where the sales amount had increased from the previous month by a value of 100 currency units or more:

SELECT FORMAT(MS1.MonthStarting, "mmmm yyyy") AS CurrentMonth,
MS1.SaleAmount AS CurrentMonthSaleAmount,
MS2.SaleAmount AS PreviousMonthSaleAmount,
MS1.SaleAmount - MS2.SaleAmount AS Difference
FROM MonthlySales AS MS1 INNER JOIN MonthlySales AS MS2
ON DATEDIFF("m",MS2.MonthStarting, MS1.MonthStarting) = 1
WHERE (MS1.SaleAmount - MS2.SaleAmount) >= 100
ORDER BY MS1.MonthStarting DESC;

Note that in the first example the query uses a LEFT OUTER JOIN to include the first month of sales, whereas in the last three examples the queries use an INNER JOIN as the first month obviously has no previous month with which to compare the sales.

Slow Word 2013

I experienced a similar issue with slow typing speed in Office 365 Word. I edit a substantial number of Word and Excel documents for work and found the SkyDrive storage incredibly beneficial for working remotely.

I realized though that documents which are being edited from Skydrive, not a local copy, experience the slow typing speed issue.

The fix:   From word:  File, Options. Word Options has an advanced tab on the left labeled advanced. About 3/4 of the way down, a category for Save options exists. The default setting has Allow Background Saves checked. I uncheck this option and instead check Always create backup copy and Copy remotely stored files onto your computer, and update the remote file when saving.

I assume that if youre storing the files on Skydrive, or Dropbox, or web, or maybe even some external drive or flash memory this background save could be lagging and result in typing slowness. Either way, I changed the advanced options, and no more problem. Maybe itll work for you.

Excel didnt have the option, and I usually save a local copy anyways due to the working file size.
