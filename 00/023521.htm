T-test functions in Excel - Microsoft Community

neil_6803 say:

T-test functions in Excel

I'm currently trying to find p-values on a set of data. I have 6 data points, and when I perform the t-test using the Data Analysis tool, I get one value. When I use the =t.test or =ttest function, I get a different value that is several times larger. Using the =ttest function over the =t.test function was recommended in another post, but this did not solve the issue. Which p-value is correct, and why are they different?
mike middleton say:

The results of the t.test and ttest functions are based on Hypothesized Mean Difference equal to zero, and that assumption cannot be changed.

The t-Test of the Analysis ToolPak allows you to specify the Hypothesized Mean Difference, and you specified 0.05.

If you use the t-Test of the Analysis ToolPak and specify Hypothesized Mean Difference equal to zero, the results will show "P(T<=t) two-tail" equal to 0.187626 (formatted to show six decimal digits), which is the same as the results using the t.test and ttest functions.

Create Custom Percent Data Type field In Microsoft Project 2010 - Microsoft Community

niladri ray say:

Create Custom Percent Data Type field In Microsoft Project 2010

Similar to % Complete or % Work Complete, I want to create custom percentage data type field which can also roll up to summary level.

I do not see this data type in Project 2010, is there a workaround to achieve the same, please help.

john - project say:
Niladri,
I see Jan stepped in with the solution but let me add a bit of explanation. As I indicated in my initial response, some extra fields can be rolled up and others cannot. As you found out, an extra number field will roll up but number fields can only contain numerical data, not text data. On the other hand extra text fields do not rollup (i.e. rollup doesn't make sense for text entries), but text fields can be formatted to include any text string.

So as Jan noted, customize a number field with your basic formula, then add the "%" character via a text field. For example, let's say you have your rolled up result in the Number1 field. The following formula in a custom text field will give you the final results you want.

Text1=[Number1] & " %"

Hope this helps.

John
