graham mayor say:

http://www.gmayor.com/document_batch_processes.htm will deal with the file and folder handling but in order to change the logo and the header information in the individual documents, you would need to specify EXACTLY what needs to be changed and with what. In particular you would need to specify which header/footer the information to be changed is contained in.

Personally I would save the new header/footer information in one of more autotext entries and replace the header footer content with an autotext field (or fields) to reproduce that information. This is both easier to program and has the distinct advantage that the information can be changed by replacing the autotext content, when some genius decides to change the logo again without considering the amount of work involved.

The field can then be locked retaining the information.

Used with the aforementioned add-in, you would need to create a custom process. To point you in the right direction the following will put an autotext entry called 'Logo' in the primary header of section 1 of the document(s). You can test with a single document:

Function ChangeLogo(oDoc As Document) As Boolean
Dim oRng As Range
    On Error GoTo Err_Handler
    Set oRng = oDoc.Sections(1).Headers(wdHeaderFooterPrimary).Range
    oRng.Text = ""
    oDoc.Fields.Add oRng, wdFieldAutoText, "Logo", False
    oRng.Fields.Locked = True
    ChangeLogo = True
lbl_Exit:
    Exit Function
Err_Handler:
    ChangeLogo = False
    Resume lbl_Exit
End Function

alan wu msft say:

Hi Sumate_Kï¼Œ

Yes it is right that 1 domain name can be used in 1 Office 365 Tenant. 

As for your questions, the answer is that it is not feasible to stop the users from Sale Company seeing the users from Product Company in Office 365 Admin Portal. As they all belong to 1 organization (Office 365 Tenant). 

However, although they belong to the same organization, they can be assigned with different Exchange Online Policy (ie, OWA policy, mailbox policy, etc). 

As the users are now existing in local AD, it will need Azure AD Connect to sync the AD users to Azure AD. And then the synced users need to be assigned with Exchange Online licenses. So the Office 365 Azure Subscription and Exchange Online Subscription are needed at least. In conclusion, you can refer to Exchange Online service description to know what Subscription supports migrating AD users to Office 365 then you can decide to choose the suitable Office 365 Subscription.
