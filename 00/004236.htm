paul van der heu say:

 

Outlook is a mail client for exchange servers (mainly) it is not a server in and of itself and so it will not syn with another client. It will sync with the accounts you set up in it and other clients (or devices) can do the same, thus sharing the same data. The same goes for (for instance) your Windows Phone and Windows 8.x PCs.

 

To me it sounds your are mixing up a number of different services. Outlook (the mail client on your PC) is not the same as Office 365 (a subscription based cloud version of Office) Both are (again) clients to specific services provided my MSFT.

 

If you use a different Microsoft account for your PC and the Phone they will not share the same settings and options between them as these are stored and linked to the Microsoft account. So if you for instance install an app which is multiplatform you will have to buy it separately for both platforms while if the accounts used to register the devices is the same these apps are shared between the devices.

 

You can set this up as unified or separated as you like. You can always choose to not see the contacts and/or calendars linked to an account on the phone. You can't share contacts between accounts though, you'll have to set up duplicates for the ones you want on both. If you remove one account all data linked to that account is removed (from the phone) as well. The contacts will show in a combined list but still be completely separated. You _can_ link accounts contact data from several accounts but (again) the link will be broken and contacts removed if you decide to remove the linked account.

Use

Sub GetFolder()
Dim strFolder as String
Set fd = Application.FileDialog(msoFileDialogFolderPicker)
With fd
    .Title = "Select the folder into which the documents will be saved."
    If .Show = -1 Then
        strFolder = .SelectedItems(1) & "\"
    Else
        MsgBox "The documents will be saved in the default document file location."
        strFolder = ""
    End If
End With
End Sub

jay freedman say:

It has been common knowledge for several years that both Windows 10 and Office 365 automatically install updates. There are some ways to delay them, such as telling Windows Update that you're on a metered connection, but no way to prevent them or make the process completely manual.

If you're storing your macros in Normal.dotm, don't do that. Instead, store them in a template that you place in the folder %appdata%\Microsoft\Word\Startup. That will make the macros (and other customizations in the same template) available in all documents, as they would be in Normal.dotm, but that location is never touched by updates. It's also less prone to random damage or malicious actions.

Another tip is to set up Windows 10's File History, and be sure to include the Startup and Templates folders among the ones that it backs up. That will save multiple revisions of the files there. It's more reliable than System Restore.

macropod say:

AFAIK, no Office update has ever altered the contents of one's Normal template. Systems Administrators in corporate environments are prone to doing so, however, often without regard for the consequences of their actions. Likewise, a configuration change that tells Word to look in a different folder for the Normal template will render the old one inactive - the default path is:

C:\Users\%Username%\AppData\Roaming\Microsoft\Templates\

Even an Office upgrade will usually retain an existing Normal template - perhaps renamed as Normal.dotm.old or something such - so you don't actually lose your macros and other customisations.
