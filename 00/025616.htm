The List of Touch Gestures for Windows 10

The list of touch gestures in Windows 10 looks as follows.

Single Tap

Tap once on an object on the screen to press some button, execute an app from the Start menu or taskbar, select an object, or activate the control you tap. The single tap gesture is similiar to clicking with the left mouse button.

Tap and hold

Press your finger down and hold for a couple of seconds. Depending on the current app, this gesture can open the context menu of the current object (e.g. if you are working with some file in File Explorer) or show the context help about the current item. This gesture is similar to right-clicking with a mouse.

Pinch or stretch

Touch the screen or an object on the screen with two fingers, and then move the fingers towards each other (pinch) or away from each other (stretch). Usually, this gesture is used to zoom in or zoom out the image on the screen.

Rotate

Place two fingers on an object on the screen and then turn them. This will rotate the object in the direction you turn your hand. This gesture should be supported by the opened app.

Slide to scroll

Move your finger over the screen. Moves through the page or app opened on the screen.

Slide to rearrange

Press and hold the object briefly, then drag it in a direction opposite to the scrolling direction of the page. Move the object wherever you want. Then release the object. This gesture is similar to drag-n-drop.

Swipe to select

Swipe an object on the screen with a quick movement in a direction opposite to the scrolling direction of the page. This gesture selects the item and opens a set of additional commands defined by the app.

Swipe or slide from edge

Starting on the edge, move your finger quickly or slide across the screen without lifting your finger. This gesture can trigger one of the following actions.

  • Swipe in from the top edge to make the title bar visible for an app opened in full screen mode.
  • Swipe in from the right edge to open the Action center.
  • Swipe up from the bottom to show  the taskbar in a full screen app.
  • Swipe in from the left edge to see a list of all your open Windows in Task View.

Hi,

 

In using Archive feature in Outlook account for the first time, it will automatically create a folder named Archive. That folder is located under the label Folders on the left pane. If you want to move these messages to other folders, you can follow these steps:

 

1. On your inbox, click Archive folder.

2. Select the messages that you want to move by ticking the checkbox beside it.

3. Click Move to, found on the upper part of the page in between Sweep and Categories.

4. Select the folder where you want to move the messages to.

 

The messages will be move to that folder. In your case, you want to retrieve the messages so all you need to do is to select the folder where you stored the messages before.

john p spencer say: Query One: against your table

SELECT Division_UIC,  IssueDescription, Count(IssueDescription) as TheCount
FROM SomeTable
GROUP BY Division_UIC, IssueDescription

Note that this does not include IssueCategory.  If you include issue category, you will get more results.

Now using that saved query, you can get the results of the top 5 by using a query like the following.
SELECT Division_UIC, IssueDescription, TheCount
FROM QueryOne
WHERE IssueDescription In
   (SELECT TOP 5 IssueDescription
   FROM QueryOne as Temp
   WHERE Temp.Division_UIC = QueryOne.Division_UIC
   ORDER BY TheCount, DESC)

If you need to include Category Description in the results then the first query is

SELECT Division_UIC, CategoryDescription, IssueDescription
, Count(IssueDescription) as TheCount
FROM SomeTable
GROUP BY Division_UIC, CategoryDescription, IssueDescription

The second query becomes
SELECT Division_UIC, CategoryDescription, IssueDescription, TheCount
FROM QueryOne
WHERE IssueDescription In
   (SELECT TOP 5 IssueDescription
   FROM QueryOne as Temp
   WHERE Temp.Division_UIC = QueryOne.Division_UIC
   ORDER BY TheCount, DESC)

If you need top 5 per Division and Category then the WHERE clause of the sub-query in the would need to be modified.

ken sheridan say: The problem was that I've not followed the advice I so often give to others of always declaring parameters.  What's happening is that it is sorting the CountOfIssueDescription values as text alphabetically, not a number.  This is cured by declaring the parameter as a short integer data type.  I think you'll find this will work:

PARAMETERS [Enter number:] SHORT;
SELECT Q1.*
FROM IssuesByDivision_UIC AS Q1
WHERE (SELECT COUNT(*)+1
     FROM IssuesByDivision_UIC AS Q2
     WHERE Q2.Division_UIC = Q1.Division_UIC
     AND Q2.CountOfIssueDescription > Q1.CountOfIssueDescription)<=[Enter number:]
ORDER BY Q1.Division_UIC, Q1.CountOfIssueDescription DESC;
