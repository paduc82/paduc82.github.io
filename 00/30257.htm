auggy say:
Case can you check the following?:

 Start > type view advanced system settings in the "Search programs and files box" and press "Enter" > then press "Settings" under "Startup and Recovery" > then make sure "Kernel memory dump" is selected under "Write debugging information and "Overwrite any existing file" is also selected > OK

With those settings both a kernel memory dump (%SystemRoot%\MEMORY.DMP) and small memory dump (located in %SystemRoot%\Minidump folder) should be created with each crash.

But sometimes these files, for various reasons, are not created.

Dump files can also get deleted if one uses cleaners as CCleaner.

For the Windows Debugging Tools you can use the following link:

http://www.microsoft.com/download/en/details.aspx?displaylang=en&id=8279

When you run the installation for the "Installation Options" choose "Debugging Tools for Windows' only.

There is a good tutorial in the following link on using WinDbg;

http://www.sevenforums.com/crash-lockup-debug-how/26584-configuring-debugging-tools.html

Also, it may help if you make available the following files:

Start > type msinfo32 in the "Search programs and files" box and press "Enter" > File > Save > then save as an .nfo file

Then make the resulting .nfo file available via Windows Live SkyDrive

Also, can you do the following:

Start > type eventvwr in the "Search programs and files" box and press "Enter" > expand "Custom Views" and then right-click "Administrative Events" > select "Save all events in Custom View As" and save as an .evtx file

Then make the resulting .evtx file available via Windows Live SkyDrive

kyle_n say:

Alright, seems like your controller and headset are fine then.

First up, have you restarted the PC? Probably an obvious answer, but sometimes it gets forgotten. Restart your PC, if you haven't already.

Just to be clear, when you say it's not picking up, are you looking in Control Panel > Sound > Playback? And it's not listed there? Be sure to right-click the background here, show disabled and disconnected devices, just to be sure.

Failing that, right-click Start button (or press Windows+X) > Device Manager

Then, unplug your controller, and plug it back in (with the headset connected). It should take a minute to set up. See if that resolves the issue.

Hi,


It seemed that the magnetism of the touch pad has become too sensitive. You are right that it is possible for some tiny foreign objects to be responsible with the randomized clicks and movements of the cursor. You should be working with the Permissions of the highest level folder(s) that you can.

C:\Users\{user}\AppData\Local\Microsoft\Windows Live\Contacts\Default

There may be several folders; it depends on which version of the program is installed which one is the current one.

As I still have the 'old' user profile files, hopefully I can recover from that, but I need to know what to look for!

There is no provision in Windows Live Mail for recovering the default contacts list unless you backed it up by exporting. The program makes available a perfectly seamless backup arrangement for contacts and calendars which requires no user configuration at all. All you have to do is to sign in in Windows Live Mail with a Microsoft Account. If you choose not to do this, then I'm afraid you only have yourself to blame.

You may be able to recover your default contacts list by copying the current DBStore folder to exactly the same location in the new user profile, but there's no guarantee that it will work.
