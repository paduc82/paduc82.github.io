Seriously weird, this one.

Let's start with what I know:  when you edit a linked object then exit the editing mode, the source program (Excel in this case) supplies a Windows Metafile representation of the data ... a picture to look at, in other words.  It updates this when the link gets updated either through you editing the data or by clicking Update.

Here's at least a workaround, a little macro you can run to force all the links in the presentation to update en masse:

Sub UpdateLinks()

    Dim oSh As Shape
    Dim oSl As Slide
    
    For Each oSl In ActivePresentation.Slides
        For Each oSh In oSl.Shapes
            If oSh.Type = msoLinkedOLEObject Then
                oSh.LinkFormat.Update
            End If
        Next
    Next
    
End Sub

It appears that there is unwanted space after the Footnote Continuation Separator (but not the Footnote Separator). Since the two are the same in Word 2016, perhaps this document is in Compatibility Mode?

In any case, switch to Draft view and, on the References tab, in the Footnotes group, click Show Notes.

Once you have the Footnotes pane open, select Footnote Continuation Separator from the dropdown on the toolbar. Click Reset on the toolbar.
