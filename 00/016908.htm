ken sheridan say: I don't think so, Bob.  Even if you write it as SQL and switch to design view the alias UniqueEngineerList will be exposed as a table, but not the SQL of the subquery behind it.

The only way I can think of to do this sort of thing entirely in design view as a single query would be to add the following computed column:

EngineerCount: DCount("*","Engineers","Engineer IN(SELECT Engineer FROM Repairs)")

and set the query's UniqueValues property to True.  As the query is only returning the computed column the query itself can be based on any table which includes at least one row of course.

This presupposes an Engineers table, of course, or a query Engineers which returns only Engineers from an Employees table.  Any decent database would have such a table, but the keys should be a numeric EngineerID, not the personal name of course, as the latter can legitimately be duplicated.

I have to say that I regard the above 'solution' of no more than academic interest as it is far easier to write the query entirely in SQL to start with.

Ken Sheridan, Stafford, England
