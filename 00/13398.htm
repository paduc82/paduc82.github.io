Hi smschulz,

In the preparation, you only need to verify your domain in Office 365 Online as an accepted domain( add a TXT record for that). No other DNS records are required to be modified or added. You can point the MX record to Office 365 after the migration.

The pop-up is a fake security alert. The actual infection is the pop-up warning you to run a scan and purchase the fake security software.


Unfortunately, these type of malware attacks are difficult to keep up with because they trick you into letting them install. They usually come from an infected web site, and usually through an advertisement. You get a pop-up from the infection and you click it to close the pop-up - which allows the infection to install.  They can also be delivered in a "drive-by" fashion with no action needed by the user due to the system being unpatched, no matter what security software is running.

 

When you encounter one of these fake virus pop-ups while browsing, immediately do the following:

-Do not touch any browser window to close it or browse further.

-Use the key combination <ALT>+<F4> to close all running programs, especially the web browser

--or--

-Immediately press Ctrl-Alt-Del and bring up Task Manager and forcibly end all instances of iexplore.exe, if using Internet Explorer, or the executable for your browser for any other web browser.
--or--
-Go to Start/Shut Down and restart the PC without touching any browser windows.
-If you used task manager to close browser instances, reboot the machine.
-Then go to Control Panel/Internet Options and delete all temporary Internet Files and cookies. If you are using an alternate web browser, open the browser settings to do the same - delete the local cached files and cookies.
-Perform a full scan with MSE.

The above steps should prevent the infection from taking hold.

-Make sure that the Windows Firewall is enabled.

-Make sure that all important/critical updates, including service packs for the operating system and programs are installed from Microsoft Update (Windows Update).

-Make sure Internet Explorer is at version 8 or 9 and updated with all patches.

-In Internet Explorer 8 or 9, use the SmartScreen Filter.

john mcghie say:

First, QUIT Word.  If Word is running, the Normal template is stored in memory: it will simply write the old one back when it next quits.

The next "trick" is to figure out WHERE the Normal template is: there can be three or four instances of it on your computer.

Quickest way is to ask Word which one it is USING.

  1. Go got Word>Preferences>File Locations
  2. Select the User Templates entry
  3. Click Modify, so you can see the whole path. That's the one Word is currently using.
  4. Don't make any changes, Cancel Out
  5. Navigate to that location, find Normal.dotm and any others with the word "Normal" in their name, and delete them.
  6. Remain in Finder

Now: there is a further trick to causing the Finder's Spotlight Search to actually show you files within the hidden system folders:

  1. Select the boot drive in the left sidebar ("Macintosh HD" for many folks).
  2. In the Search window, type "Normal.dotm", and choose "This Mac" in the Finder's Search toolbar.
  3. After hitting Enter, an extra Finder toolbar appears, below the Search window (with + and - signs)
  4. Where it says "Kind", drop that down and choose "Other"
  5. The Select a Search Attribute window opens, which has its own Search window.
  6. In that second search window, type the word "system"
  7. When the "Include system files..." entry appears, select the entry and check the box to the right of it, and OK
  8. In the box to the right of the one that now reads "System files", change the box that says "are not included" to read "are included".

Now you will be able to see anything containing Normal.dotm in its name, wherever it is!

I suggest that you make a folder on your desktop, and drag all of the Search Results into it.  It is very unlikely that you will ever want any of them back, but there is no harm in working safe.

Now you must Shutdown the computer and then Restart it.  In Mac OS X and in Microsoft Office, stuff gets cached all over the place: doing a full cold start causes OS X to run its house-keeping tasks to clean everything out.

When you restart Word, it will create a new, pristine, copy of Normal.dotm.

I am not saying that will fix your problem, but at least you will know it is not the Normal template if it doesn't :-)

Hope this helps

Thanks for posting your findings :-) I've experimented with this a bit & it appears to be the actual fix rather than just a workaround, but it isn't Word's doing... at least not entirely. Oddly, though, Word 2011 doesn't exhibit the same behavior, even on the same Mac.

If an East Asian Keyboard Input Source is added the OS automatically adds that language to the list of Preferred Languages as well. When an East Asian language is listed there (even if it isn't the Primary language), Word picks that up on launch. That results in the differently appearing mark. Once the language is removed from the list of Preferred Languages & the Input Method is removed, the mark in Word documents returns to 'normal' when next launched. It appears to remain that way until such time as an East Asian language or Input Method again is added in System Preferences. Note that you are not "deleting language sets", you're simply not leaving them enabled.

No matter what, in the end this is going to have to involve VBA (a macro).  For example if you had this formula in A3:

=IF(A2=True,A1,"")

It would echo A1 (but continuously change as A1 changed) and as soon as A2 goes to a false condition, you lose it all.  So...

 

How does the other condition become true?  IF you enter something from the keyboard that eventually causes A2 to become True, you could test at the point of entry with a Worksheet_Change() event.  Lets say that in A2 you have this:

=(B9+B10)>5

Then you could test things when you entered something into either B9 or B10.

 

But you say that you have a clock running with the time showing in cell A1.  Probably the easiest thing to do is to modify the routine that's updating the clock to check the status of A2 and if A2 has turned to True, put the current time into A3 also.  The catch with this is that once you have done that, you probably want to 'freeze' that captured time, so you have to add some other code to keep from overwriting it later.  Something like (pseudo code)

If cell A3 does not have anything in it, AND  cell A2 = True, then save current time in A3.

 

The corollary being that if A3 already has a time in it, nothing will happen to that value once it's been entered.
