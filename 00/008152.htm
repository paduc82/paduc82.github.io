john - project say:

Rodolphay,

Personally I don't use Project's leveling feature. If an indivisible task was moved as a result of leveling and then the macro is run to resolve interruptions in that task, you may end up with an overallocation conflict. Remember, the leveling algorithm isn't perfect, some overallocations simply cannot be resolved and the project manager needs to resolve the issue.

I agree that there are many instances where a task, once started, must run continuously until completed. Unfortunately Project simply does not have a level of sophistication to automatically address those cases. Perhaps future versions of the software will. You might want to suggest that feature via the Project UserVoice channel at:

https://microsoftproject.uservoice.com/forums/218133-microsoft-project

Meanwhile, if you are interested in the macro I mentioned, do you have any Project VBA experience? The reference macro is shown below.

John

'This macro ensures all selected tasks can only be started if they can be
'   completed the same day
'   Tasks are selected by setting Flag1. Note: this procedure does leave a
'   start-no-earlier-than constraint on all shifted tasks
'written by John-Project 4/27/18 1:30 pm
Option Explicit
Option Compare Text
Sub NextMonOrFri()
Dim t As Task
Dim StartTime As String

ProjCal = ActiveProject.Calendar
'determine the default start time for this project
StartTime = ActiveProject.BaseCalendars(ProjCal).WeekDays(WkD).Shift1.Start
'cycle through each task to find flagged tasks
For Each t In ActiveProject.Tasks
    'skip over blank lines
    If Not t Is Nothing Then
        'only look at Start time of non-summary tasks
        If t.Summary = False And t.Flag1 = True Then
            'determine if start and finish are currently on the same day
            '   if not, move start to next working day
            If Day(t.Start) <> Day(t.Finish) Then
                t.Start = DateValue(DateAdd("d", 1, t.Start)) & " " & StartTime
            End If
        End If
    End If
Next t
End Sub
