greg maxey say:

Richard,

The "h1" is an alias or alternate name for the built-in style Heading 1.  It results when someone tries to rename Heading 1 as "h1."  You can't rename built-in styles, so the attempt is simply appended to the style name.  It can come in handy for people who apply styles using the Apply Styles pane (Ctr+Shift+s) as all they need to do is type in h1 instead of Heading 1.

You can make the "Heading 1,h1" appear as simply "h1" in the style pane by selecting the option "Hide built-in name when alternate name exists" from in the style pane options dialog.

To get rid of the ,h1, just use the modify style dialog and rename "Heading 1,h1" as "Heading 1"

cortig say:
Outlook for Mac is not like Outlook for Windows. It doesn't use the
equivalent of a PST file for live storage of all your data.
Instead, it uses an identity composed of a number of individual files
for all the items in your accounts.
You can move the folders from one IMAP account to the other (you can
also copy them).
Once moved anyway, you shouldn't have to worry about what's cached in
the old account.
Once you remove an IMAP account from Outlook, the items in the account
essentially disappear from the view.
 
If you want to migrate your identity to a different computer, you can
either let the Migration Assistant do all the work for you and import
all your user data from one computer to the other, or you can manually
copy the ~/Documents/Microsoft User Data/Office 2011 Identities folder
to the exact same location on the other Mac. Outlook will pick it up and
use it automatically.
 Corentin
 

cortig say:
A .olm file is just an archive file. You can't access the content and
play with it, you can only import all it's content back into Outlook.
 
If you move emails from one email account to the next, they should be
unaltered in the new destination, with attachment etc.
If you're worried, you can always remove the account, test the result,
then re-add it if anything went wrong.
 
You should be able to go through a .olm (though I don;t see what
advantage that would present). Creating .elm on the desktop is also an
option, but it's awfully tedious and can only be done with individual
messages.
I would just use the Move (or Copy) command.
