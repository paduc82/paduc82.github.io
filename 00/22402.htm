Eliminating background and preserving other attributes when pasting - Microsoft Community

honest abe say:

Eliminating background and preserving other attributes when pasting text

A well documented problem in many applications (including Word, Internet Explorer, Webpage editors, etcetera) is that of an unwanted background color behind pasted text that carries over from the source.  The integral background appears similar to text highlighting only it cannot be eliminated by adjusting highlighting, shadowing, paragraph fill, page background, or any other special affect Word or most other programs.

THE PROBLEM:  When the text is pasted as text, without formatting, it not only removes the unwanted background, but also all bolding, underlining, italics, or other text attributes that one might want to preserve.  This can be an enormous problem, especially if the source text is voluminous and has a lot of intricate formatting.  We often face this when moving text we have authored for website pages with a background.

THE INADEQUATE SOLUTION: The internet is replete with suggestions for addressing this problem, but they all have the same drawback in common. Most suggestions center around copying and re-pasting the text as text only (without formatting) or by copying and pasting the problem text into NotePad first (in order to strip all the format codes) before copying it back to the destination. Unfortunately, these work-arounds all strip the desirable attributes along with the undesirable attributes.

THE QUESTION:  Is there any known method for pasting text into Word with all the text attributes (bold, italic, underline, text color, shadow, etcetera) intact, but without the unwanted background?  Similarly, is the a way to locate and remove the unwanted codes from within Word?

In our opinion, it was a very poor design choice to allow users to cut and paste text with attributes that cannot be accessed and deleted, or modified, without also deleting every other selectable attribute.  These are the pesky little problems that drive users crazy and alienate them from applications.  We see absolutely no reason to prevent users from being able to delete any and every individual embedded formatting attribute!

hansv mvp say:

The Home tab of a ribbon has a Shading dropdown in the Paragraph group. This can be used to set the background color ("shading") to "No Color".

Keep in mind that if you copy white text on a black background, then paste it into Word and remove the background color, you'll end up with invisble text until you change the font color too...

hansv mvp say:

On the Home tab of the ribbon, in the Paragraph group, the dropdown button next to the Shading button offers various border options. The last item in the dropdown list is Borders and Shading. Selecting this opens the Borders and Shading dialog.

This dialog has three tabs: Borders, Page Borders and Shading. The latter tab has more options than the Shading dropdown button on the ribbon.

john w. vinson say:

I'm not sure it's the cause of your problem, but it may be that you're using the HAVING clause - which is applied after all the summing and grouping is done - rather than the WHERE clause. While you're at it you could exclude records with a null or zero TotalMiles value since (it would appear) they would be useless for your report anyway. Try:

SELECT [Trailer Repair Order Detail Report].Trailer, [Trailer Repair Order Detail Report].Mth, [Trailer Repair Order Detail Report].Yr, Sum([Order Data].TotalMiles) AS SumOfTotalMiles, Max([Trailer Repair Order Detail Report].[RO Total Cost]) AS [MaxOfRO Total Cost], Sum([RO Total Cost]/[TotalMiles]) AS Expr1
WHERE ((([Trailer Repair Order Detail Report].Mth)="May") AND (([Trailer Repair Order Detail Report].Yr)=2014)) AND [TotalMiles] >0

GROUP BY [Trailer Repair Order Detail Report].Trailer, [Trailer Repair Order Detail Report].Mth, [Trailer Repair Order Detail Report].Yr;

I'd also suggest adding a RepairDate Date/Time field rather than separate Mth and Yr fields; a Date/Time field will sort chronologically, while your Mth field will sort Apr - Aug - Dec; and the date/time field can be more easily used for date range queries. It's easy to extract month and year from the date for reporting purposes.
