albert d. kallal say:

Ok, so what we need to do here is build a table of the categories (you only need one table).

 

 Right now there is a combo box column for the category in table assets. However, it is a hard coded list (a poor design choice since you have to go an modify the table direct to add more categories).

 

And now that we also need place to "setup" a list of "prefixs" to use for a given category, and furthermore we ALSO need to keep track of the last asset number for a given category. So a new table that holds this info kills about 4 or 5 birds with one stone by creating a new table for the asset category.

 

I think this also shows what happens when poor design decisions are made (the category should have been a separate table ‚Äì however it really is just a sample template).

Ok, so lets make a new table. Lets call it Catagories

It will look like this:

 Category:

 Id               (autonumber)

 Description    (text) (eg: Personal Computer)

 Prefix           (text)   (eg: PC, FW etc.

 NextID        (number) this will hold the last number used

 

Ok, so this table will thus allow us to easily add new asset types. I think you imagine how bad this would be if you developed such an application and your company found out that for each new asset type you have to bring in a developer to modify the table direct! I know some of my clients would consider this a form of theft or even just dishonesty at best. So the original free template is just a free sample, if we going to use this in a production envirnemnt we really need to add the above table else risk being seen as a dishonest person.

Ok, now that we built this table, we should add in the choices that we see for the Category type. So lets enter the data list this:

 Id      Description         PreFix     NextID

 1        Servers            SR           1

 2        Desktop Computers  PC           1

 3        Printers           PR           1024

 4        Copiers            CO           1

 5        Faxes              FA           1

 6        General Office     GA           1

 

Also with such a table we can then build a form to allow users to add new category. And in fact you could also use a form based on the above table to edit/set the NextID. So in place of the next asset ID starting at 1, you could type in 100 or even 1024 as I done for printers.

 

Ok, now that we done this, the next step is to replace the category column in the Assets table to use the above table for the drop down. This means we have to re-enter the catagoreis here. You could thus re-name the collum Category to Category1 if you already have data entered (that way  could bring up the assets table and re-enter the data into our new collum by looking at the old values).

 

Use the lookup wizard, and I think id,Description,Prefix makes reasonable sense for this drop down (you can enable data integrity here also ‚Äì delete restrict). We also now need a new column called AssetID that will be our automatic number column + prefix. This column can be a text column, and lets not forget to put an index on this column.

Ok, the next part is to add some code that grabs the value of the new category, and then sets the AssetID column based on what we choose for category.

The easy way to do this is to use the table before update event. The code would be:

 

If Updated("Category") Then
   Look Up A Record In Catagories
      Where Condition =[ID]=[Assets].[Category]
                 Alis
     Setfield
                 Name:  Assets.AssetID
 Value:  [Catagories].[Prefix] & Right("0000" & [Catagories].[NextID],4)
 

So the above will then "pluck out" the next asset number based on the category, place the prefix in front.

 

At this point the next thing is we need code to increment the asset value used.

 

The code for that needs to be placed in the asset after update table event for Assets. The code will look like:

 

If updated("Category") then
   Lookup A Record In Catagories
      Where : [id] = [Assets].[Category]
   EditRecord
   SetField:Catagories.NextID
      Value: [Catagories].[NextID] + 1

 

At this point in time we thus have a automatic incremting and setting of the AssetID.

 

You also have to add the above code to the after insert event (since after update does not fire on a table insert event)

You now have to go through each of the forms staring with AssetDetails and delete the category combo box, and then drop in a new one based on this new table.

Build a nice for to edit the catagories, and you will have made a nice addition to this system, and one that will automatic update the next assetID for you, and allow you to easy set the nextID.

The above is much "air" code, but is the general approach you need here.

joeu2004 say: Rick wrote:

When I enter TODAY() or NOW() into a cell Excel flag's that cell as a circular reference.  This is using Windows 7 Ultimate with all the 7/14/2013 updates.  Any cell that has these functions in them is flagged as a circular calculation cell.  Is there a way to correct this or which updated do I need to remove to correct this error?


It is probably not the case that TODAY() and NOW() are causing the circular reference.  Instead, the circular reference probably was pre-existing, and using TODAY() and NOW() -- any volatile function -- simply caused Excel to discover the circular reference during recalculation.

Sometimes, "the" cell with a circular reference is displayed in the lower-left part of the status line.  However, that can be misleading.  It depends on the complexity of the circular references.

Also, sometimes clicking on Formula, Error Checking (drop-down menu) and Circular References in the Formula Auditing section will reveal the problem.  But again, that depends on the complexity of the circular references.  Often, the Circular References link is greyed out, in my experience.

The following procedure was suggested by someone else some time ago.  Select a cell, ideally one identified as having a circular reference, then click on Formula, Trace Precedents in the Formula Auditing section.

If you see a red trace arrow, I believe it indicates a circular reference.  Otherwise, repeatedly click on Trace Precedents (adds more depth) until you see one or more read trace arrows.

Unfortunately, that might be difficult to see in large worksheets.  And it might be nearly useless if the circular references involve another worksheet.
