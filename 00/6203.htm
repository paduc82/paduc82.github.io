karl timmermans (outlook mvp 2012-2018) say:
>>
With Office 365 on 5 computers, does Outlook 2016 sync? How is that done? I have 2 other computers I would love to be able to work with Outlook when I'm on them.
<<

Just for sake of accuracy, Outlook doesn't sync between computers, each installation of Outlook would connect (and sync where applicable) with email accounts. Each computer would need to be configured with the same email accounts.

in answer to the rest of your questions/requirements

#1 - How many and what type (POP, Imap, Exchange) email accounts are involved
(to determine that, from Outlook menu, go to FIle --> Account Settings --> email accounts listed under email tab)

For POP3 accounts - email is a one-way download of the <Inbox> . No other folders are sync'd. You would need to copy the PST file associated with the POP account to the other computers

For IMAP accounts - all email folders are sync'd with the server. Contact/Calendar info is not sync'd with the server. What you need to copy depends on <IF> you have anything you want to keep in folders with the added words "This Computer Only". You would need to export those folders to a PST file

For Outlook.com\Exchange accounts - everything is sync'd with the server - nothing to copy.

If you want to use contacts/calendar across all devices, just set the data file for one of your Hotmail (Outlook.com) accounts as the default for the profile and you'll be all set. Changing the default data file is completely different then setting the "default email account' - the two are mutually exclusive.

Re: #3 - moving "huge" Outlook folders

Question related to this is "where are these stored"? Specifically, (a) are these folders in an IMAP folder tree or (b) did you create a PST file and then created folders there?

If (a) - no action required. If (b) need to copy the pst file(s) in which these files are stored. =======================================================
On the other computers, configure the Hotmail account first that has its data file set as the "default" data file on the base computer. The configure you other accounts. Everything will the work the same on all computers with nothing to copy from one computer to another unless you have manually added PST files.
=======================================================

The above process will result in all devices seeing all the same data (PST files excepted as noted above)

If you bookmark the Content Control and then use a Cross Reference to the text of the bookmark, you will need to update the fields in the document for the item selected in the Content Control to appear in the Cross Reference.

With code such as the following in the ThisDocument object in the visual basic editor, that will happen when you exit the content control

Private Sub Document_ContentControlOnExit(ByVal CC As ContentControl, Cancel As Boolean)
With ActiveDocument
    For Each CC In .ContentControls
        .Fields.Update
    Next
End With
End Sub

alex z_ say:

Folks, I think I've found a workable solution.  The trick is to block messages using "Inbox and Sweep Rules" rather than a "Block Senders List".  When using the former, you can specify messages that "include these works in the sender's address".  Then for the action, choose "Move Message to Folder" and then "Junk Email".  I've set it up for email addresses that end in:

.club

.info

.online

.work

.top

.xyz

.space 

.biz

.top

This gets around the issue of spammers changing the parts of their address between the @ and the final period/dot.   So if a spammer uses  *** Email address is removed for privacy *** in one email and then *** Email address is removed for privacy *** in the next, they'll both get routed to Junk Email.  This seems to be very effective so far.  

Microsoft, I'd recommend adding this capability to the "Blocked Senders" section of the options.  "Inbox and Sweep Rules" isn't the most intuitive place to look.  Or simply allow users to specify *.xyz in the blocked senders list.
