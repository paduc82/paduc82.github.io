Mini SD Card Help

The built in memory on my phone (just 50mb) is almost used up, and now if I try to install something new, it says I'm running out of space. I have a 2gb Mini SD card, and I need a way of replacing the main memory with this card, so it doesn't say I'm running out of space. I'm not sure if I've explained that clearly, but can anyone help?

don sorcinelli say:

There really is no way to literally "replace" your onboard memory with the miniSD card. Here are a couple of basic things you can do:

1) Create a "\My Documents" folder on the miniSD card and move any files you have created and/or synced over there.

2) Look at other multimedia files you may have created and/or copied to the onboard memory and move them as well. Usually, photos and music are big space-takers.

3) Consider moving installed applications to the storage card. In nearly all cases, this will require you to uninstall the application (using System-> Remove Programs on the device) and then reinstall to the new location.

 

There are other solutions (such as using a backup solution or other utilties) that exist, but they typically are either far more complex or cost extra money.

 

Don

The land line telephony guys tend to be lagging so it does not surprise me that they support such an ancient standard.

  

From wiki

GSM has used a variety of voice codecs to squeeze 3.1 kHz audio into between 6.5 and 13 kbit/s. Originally, two codecs, named after the types of data channel they were allocated, were used, called Half Rate (5.6 kbit/s) and Full Rate (13 kbit/s). These used a system based upon linear predictive coding (LPC). In addition to being efficient with bitrates, these codecs also made it easier to identify more important parts of the audio, allowing the air interface layer to prioritize and better protect these parts of the signal.  GSM was further enhanced in 1997[13] with the Enhanced Full Rate (EFR) codec, a 12.2 kbit/s codec that uses a full rate channel. Finally, with the development of UMTS, EFR was refactored into a variable-rate codec called AMR-Narrowband, which is high quality and robust against interference when used on full rate channels, and less robust but still relatively high quality when used in good radio conditions on half-rate channels.

The media components of the N900 are based around an open source project called gstreamer, iirc.

 

This system allows additional codecs and file support to be added in a plugin fashion, rather than having to explictly support file types in an application

 

The problem with the codec you've mentioned is that it is "maybe free" which means if Nokia were to include support for playback, they may need to license it.. which is daft in itself seeing as the N900 is a GSM handset, you'd think the license was already there..

 

That being said, there is a GSM plugin for gstreamer, but it is listed as "bad", not sure why... creating a Maemo package that included this plugin would bring GSM support to gstream and therefore the ability to play these files.. 

 

I certainly can't promise anything, but I'd be interested in having a look. Depending on how much of a developer you feel you are, it's something  you could look into yourself. That is one of the advantages of such an open platform
