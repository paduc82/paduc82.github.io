Define range with non-contiguos cells VBA - Microsoft Community

tirina say:

Define range with non-contiguos cells VBA

I need to create a graph and the cells for the data series are 3 cells apart.  I want to use the Active cell as a starting point and take the 4 previous values, that means Active cell - 3 ... Active cell -12.

 

I have tried using a range variable with the offset option but it doesn't do what I want.  I also used the UNION functionto define the range but again, it selects the entire range instead of the 5 individual values.

 

Any suggestions?

rich007a say:

Here's a couple of ways (both do the same thing, the second is more customisable):

Sub Macro1()
    Dim lnRow As Long, lnCol As Long
    Dim rng As Range
   
    lnRow = ActiveCell.Row
    lnCol = ActiveCell.Column
   
    If lnRow < 13 Then
        MsgBox "ActiveCell row number is less than 13!"
        Exit Sub
    End If
       
    Set rng = Union(Cells(lnRow, lnCol), Cells(lnRow - 3, lnCol), _
        Cells(lnRow - 6, lnCol), Cells(lnRow - 9, lnCol), Cells(lnRow - 12, lnCol))
    rng.Select
End Sub

Sub Macro2()
    Dim lnRow As Long, lnCol As Long, i As Long
    Dim lnFac As Long, lnStepsUp As Long
    Dim rng As Range
   
    lnRow = ActiveCell.Row
    lnCol = ActiveCell.Column
    lnFac = 3
    lnStepsUp = 4
   
    If lnRow - (lnStepsUp * lnFac) < 1 Then
        MsgBox "Active cell cannot be used."
        Exit Sub
    End If
   
    Set rng = ActiveCell
    For i = 1 To lnStepsUp
        Set rng = Union(rng, Cells(lnRow - (i * lnFac), lnCol))
    Next i
    rng.Select
End Sub


Hope that helps.

Cheers
Rich

thegoldy say: The solution to the mapped drive issue isn't disconnecting them.   The solution is ensuring that you have logged on properly so that you have access to them.   Windows will try to ensure that it has access to all mapped drives when you try to open a file, even if it's not on one of those drives.   If you're not logged on to a drive, multiple efforts are made to get access before it gives up and gets on with what you asked for.  

We run into this a lot with people who have drives mapped which require an Active Directory logon to access, but who have logged on to the workstation and not the domain.  These people ignore the "could not reconnect all network drives" message at logon and then wonder why they can't access those drives.

Also resetting network setting also helps sometimes. To run these networking commands in a command prompt window

1. In the search box on the taskbar, type Command prompt, press and hold (or right-click) Command prompt, and then select Run as administrator > Yes.
2. At the command prompt, run the following commands in the listed order, and then check to see if that fixes your connection problem:
Type netsh winsock reset and press Enter.
Type netsh int ip reset and press Enter.
Type ipconfig /release and press Enter.
Type ipconfig /renew and press Enter.
Type ipconfig /flushdns and press Enter.

More information here https://support.microsoft.com/en-gb/help/10741/...
