I've seen this question asked multiple times without any particularly satisfying answers for someone in my position.

I have a computer I assembled about three years ago, and I recently moved the OS over to a newer unit. After that vanishes, I immediately get a BSOD with STOP: c000021a 0xc0000001 0x001003f0. (The actual message is a bit longer, informing me the system has shut down, but I gather these are the important values). Frustrated, I thought perhaps the machine was rejecting the DVD drive. I used the Windows 7 USB tool from the Microsoft website to create a bootable copy of my W7 DVD on a thumb drive. I managed to get the machine to boot off of the thumb drive, but had the exact same problem.

Vexed, I took out some old backup discs for previous operating systems. The Windows Vista disc encountered the exact same problem as described above. My Windows XP disc fared a bit better, spending about 10 minutes 'loading setup'. I didn't get a BSOD in this case, but the system eventually just turned off and did not reboot. 

I have duplicated the efforts described above on a second hard drive I formatted that has been working fine.

It's impossible to run any real diagnostics at my skill level since no operating system exists on the machine. 

For context, the machine is a  DFI LANPARTY DK P35-T2RS with a Q6600 processor, four sticks of Mushkin 1gb 240-Pin DDR2 SDRAM DDR2 800 (PC2 6400), and a MSI N460GTX CYCLONE 1GD5/OC GeForce GTX 460 (Fermi) 1GB 256-bit GDDR5 PCI Express 2.0 x16 HDCP graphics card.

Edit: I have located the issue. The heatsink had become unseated a bit in one corner - the BIOS was registering a toasty 93C on the processor. My only point of confusion is why an error code associated with bad drivers or corrupted install resources was arising. I hope this helps out folks who find themselves in similar situations to mine.
