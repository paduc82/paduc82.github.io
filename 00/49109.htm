Hi,

 Sometimes it happens that disk check always starts to check the file system on one or more drives for consistency. If any key is pressed, the disk checking is aborted. Actually this is because some data structure of the drive or some file is corrupted which cannot be recovered and check disk is always run at Windows startup.

 Run check disk from command prompt

I would suggest you to insert an installation disc into the DVD drive and restart the computer.
a. Boot from the DVD into the System Recovery Options screen.
b. Select Command prompt.
c. In the command prompt, type chkdsk/r and press Enter.
d. Answer Y when chkdsk asks you if you want to check the drive next time computer boots (restarts).

 Note: /r = The /r switch tells chkdsk to attempt to recover any bad sectors of the hard drive if any are found. A bad sector is a spot on the hard drive that cannot hold saved data anymore. If you’ve any data on that sector, you’ll lose that.

What does this mean, disk defragmenter has detected that chkdsk is - Microsoft Community

jmatthew say:

What does this mean, disk defragmenter has detected that chkdsk is schduled to run on the volume: (C). Please run chkdsk /f.

Been trying to run defrag on my C drive and this message keeps coming up.

a. user say:

When trying to defragment a volume Disk Defragmenter reports:

Disk Defragmenter has detected that Chkdsk is scheduled to run on this volume: (C:)
 
Please run Chkdsk /f.

This means that the volume you are trying to defragment has been marked "dirty" by XP.  It doesn't necessarily mean any data has been lost on the volume, it just means that XP thinks the volume needs to be checked for errors because of something that happened before. 

You will not be able to defragment the volume until the dirty bit has been cleared.

A volume can be marked dirty if the system suffered a power interruption (power failure, plug pulling, battery removal, power button), aborted restart or ungraceful shutdown .  Have any of those events occurred?

When XP starts, it automatically checks the dirty bit on every volume.  You can also tell XP to ignore the dirty bit on any volume, but that is probably not a good idea in case there is a real problem that needs to be fixed.  If you have a hard drive where the dirty bit will just not clear, you can tell XP to not check it. 

If the XP startup finds the volume is dirty, it will automatically run check disk with error correction (chkdsk /r) and try to correct any problems it finds.

If the chkdsk succeeds, it will clear the dirty bit and then you can defragment the volume. 

You can check the status of your dirty bit from a Command Prompt window.

To do that, click Start, Run and in the box enter:

cmd

Click OK to open a Command Prompt window.

For example, to query the dirty bit on drive C, type:

fsutil dirty query C:

Sample output:
Volume C: is dirty
Volume C: is not dirty

Is your volume dirty or is it not dirty?

Since XP obviously already thinks the volume is dirty, XP will automatically run chkdsk when it restarts to try to clear the dirty bit, so scheduling another chkdsk to run might not help you very much.  Sometimes the automatic chkdsk on a reboot just won't clear things up, but you can try it.

I would boot into the XP Recovery Console and run chkdsk /r from there and then you can actually watch and see what is going on and decide what to do about it.  When that chkdsk /r runs clean, it will clear the dirty bit unless your HDD is somehow afflicted.
