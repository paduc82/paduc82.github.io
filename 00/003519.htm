john w. vinson say:

This can certainly be done, but... upper case is HARDER TO READ THAN MIXED CASE and looks ugly. Are you certain this is essential?

 

A less invasive way to do it would be to just set the Format property of any table fields - or even of form and report textboxes - to > , which will display text in upper case without actually changing it.

 

If this isn't acceptable, then you can use some VBA code that will loop through the Tabledefs collection, loop through the Fields collection of each tabledef, find the Text fields, and construct an UPDATE query. UNTESTED air code:

 

Public Sub MAKEITALLUNREADABLE()

Dim tdf As DAO.Tabledef

Dim fld As DAO.Field

Dim db As DAO.Database

Dim strSQL As String

On Error GoTo Proc_Error

Set db = CurrentDb

For Each tdf In db.Tabledefs ' loop through tables

   If Left(tdf.Name,4) <> "MSYS" Then ' leave system tables alone!!

      strSQL = "UPDATE [" & tdf.Name & "] SET "  ' start the update query

      For Each fld In tdf.Fields ' loop through fields

         If fld.Type = dbText Then

             strSQL = strSQL & "[" & fld.Name & "] = UCase([" & fld.Name & "]),"

         End If

      Next fld

      strSQL = Left(strSQL, Len(strSQL) - 1 ' trim off trailing comma

      If Right(strSQL, 3) <> "SET" ' only run the update query if there's something to update

         db.Execute strSQL, dbFailOnError

     End If

Next tdf

Proc_Exit:

   Exit Sub

Proc_Error:

   MsgBox "Error " & Err.Number & vbCrLf & Err.Description

  Resume Proc_Exit

End Sub

 

BACK UP your database before you run this!!, and Compact it afterwards.

I have been aware of problems with last cell for a very long time. It always gives a problem if there is any formatting in the cells below the last actual data because it returns the last cell as the end of the formatting.

The following UDF (User Defined Function) will find the Last Row or Last Column and is very reliable, especially for finding the last cell with actual data and ignoring formatting). (The first sub is an example of calling the UDF. See after code for calling from the worksheet)

Sub LastRowExample()
    Dim lRow As Long
    lRow = LastRowOrCol(True)       'If required for the active sheet
    'lRow = LastRowOrCol(True, Sheets("Sheet2").Cells)   'If required for worksheet other than active sheet
    MsgBox lRow
End Sub

Function LastRowOrCol(bolRowOrCol As Boolean, Optional rng As Range) As Long
    'Finds the last used row or column in a worksheet
    'First parameter is True for Last Row or False for last Column
    'Third parameter is optional. Must be specified if not ActiveSheet
   
    Dim lngRowCol As Long
    Dim rngToFind As Range
   
    If rng Is Nothing Then
        Set rng = ActiveSheet.Cells
    End If
   
    If bolRowOrCol Then
        lngRowCol = xlByRows
    Else
        lngRowCol = xlByColumns
    End If
   
    With rng
        Set rngToFind = rng.Find(What:="*", _
                LookIn:=xlFormulas, _
                LookAt:=xlPart, _
                SearchOrder:=lngRowCol, _
                SearchDirection:=xlPrevious, _
                MatchCase:=False)
    End With
   
    If Not rngToFind Is Nothing Then
        If bolRowOrCol Then
            LastRowOrCol = rngToFind.Row
        Else
            LastRowOrCol = rngToFind.Column
        End If
    End If
   
    'Following line if no data on worksheet returns 1 because row or col can not be 0
    If LastRowOrCol = 0 Then LastRowOrCol = 1
End Function

The following is an example of calling the UDF from a worksheet

=LastRowOrCol(TRUE,OFFSET(Sheet1!A1,0,0,1048576,16384))

ossiemac say:

To M.Noack,

Finding the last used row in a worksheet is usually due to not being able to reliably use the last used cell in a specific column. This is because the specific column might not have data all the way to the bottom of the used range.

Your loop code finds the last used cell only for a specific column (In your example column A). The following code will find the last used cell in a specific column without the need for a loop.

    'Following example if ActiveSheet
    lRow = Cells(Rows.Count, 1).End(xlUp).Row

    'Following example if a sheet other than the ActiveSheet
    With Worksheets("Sheet2")
        lRow = .Cells(.Rows.Count, 1).End(xlUp).Row
    End With

Reason for using With/End With if not the ActiveSheet is that both .Cells and .Rows.Count should to be referenced to the required worksheet.

I do know that the Last Cell in the Used Range has been unreliable for a long time. It is the reason that I initially developed the UDF code prior to xl2007 when I was using xl2002. (Developed from other examples I found on the internet for finding last row and last column that were actually separate UDF's and I combined them into one UDF).

I only use the UDF if I know that there are no specific columns that will always have data to the bottom of the Used Range. It is actually quite rare because most stuff that I do has a key in a column (usually column A) that will always have data to the bottom.
