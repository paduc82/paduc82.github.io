graham r seach say:
Any security chaznges you make will change the mdw file. You must have access to the original mdw file, and you must include it in the shortcut you use to open the database. When you've made your changes, all users must have access to the 'updated' mdw (via their shortcuts).
 
Whether you apply the changes to the frontend or backend depends on how security was setup in the original database. If the original developer setup table security in the backend, then you should continue that practice. It's more likely, however, that this was originally created as a single file and split later, in which case, I'd still recommend applying table security in the backend. Setup frontend (user interface) security in the frontend.

Actually its not confusing at all. What you describe is a standard technique called Synchronized or Cascading Comboboxes. This is a well documented technique and you can find tons of info by searching on either of those terms. 

Basically you a treatments table and a sub treatments table. The latter will have a foreign key pointing back to the related treatment. Then your Sub Treatments combo will be filtered by the selected treatment.

In this little demo file, which illustrates some of the basic principles in designing a simple relational database, take a look at the option: '6. Entering data via a form/subforms'.  This form uses correlated combo boxes to allow a country to be first selected, then a region within the selected country and finally a city within the selected region. The important thing to note is that only the CityID is entered via a bound control and stored in the table.  If the region and country were also stored in columns in the table this would introduce  redundancies as city determines region, and region determines country, so the table would not be normalized to Third Normal Form, and consequently open to the risk of update anomalies.

The relevant code in the form's module which populates the unbound controls and allows new country, region and city values not currently represented in the databse to be entered via them is as follows.  The code is extensively commented to explain how it works:

Option Compare Database

Option Explicit

Private Sub cboCity_NotInList(NewData As String, Response As Integer)

    Const MESSAGE_TEXT = "No region selected."

    Dim ctrl As Control

    Dim strSQL As String, strMessage As String

    Set ctrl = Me.ActiveControl

    ' abort insertion of new region if no region selected

    If IsNull(Me.cboRegion) Then

        MsgBox MESSAGE_TEXT, vbExclamation, "Invalid Operation"

        Response = acDataErrContinue

        ctrl.Undo

        Exit Sub

    End If

    ' subject to user confirmation insert a new row into Cities table,

    ' inserting NewData value into City column and value of cboRegion

    ' control into RegionID column

    strMessage = "Add " & NewData & " to list?"

    strSQL = "INSERT INTO Cities(City,RegionID) VALUES(""" & _

            NewData & """," & Me.cboRegion & ")"

    If MsgBox(strMessage, vbYesNo + vbQuestion) = vbYes Then

        CurrentDb.Execute strSQL, dbFailOnError

        Response = acDataErrAdded

    Else

        Response = acDataErrContinue

        ctrl.Undo

    End If

End Sub

Private Sub cboCountry_AfterUpdate()

    ' if Country is updated then erase current values

    ' for Region and City and requery combo boxes

    ' to show regions of selected country and empty list

    ' of cities (pending selection of a region)

    Me!cboRegion = Null

    If Not Nz(Me!cboCity) = 0 Then

        Me!cboCity = Null

    End If

    Me!cboRegion.Requery

    Me!cboRegion.Requery

End Sub

Private Sub cboCountry_NotInList(NewData As String, Response As Integer)

    Dim ctrl As Control

    Dim strSQL As String, strMessage As String

    Set ctrl = Me.ActiveControl

    ' subject to user confirmation insert a new row into Countries table,

    ' inserting NewData value into Country column

    strMessage = "Add " & NewData & " to list?"

    strSQL = "INSERT INTO Countries(Country) VALUES(""" & _

            NewData & """)"

    If MsgBox(strMessage, vbYesNo + vbQuestion) = vbYes Then

        CurrentDb.Execute strSQL, dbFailOnError

        Response = acDataErrAdded

    Else

        Response = acDataErrContinue

        ctrl.Undo

    End If

End Sub

Private Sub cboRegion_AfterUpdate()

    ' if Region is updated then erase current values

    ' for City combo box.

    ' requery city combo box to restrict list

    ' to cities from selected region

    If Not Nz(Me!cboCity) = 0 Then

        Me!cboCity = Null

    End If

    Me!cboCity.Requery

End Sub

Private Sub cboRegion_NotInList(NewData As String, Response As Integer)

    Const MESSAGE_TEXT = "No country selected."

    Dim ctrl As Control

    Dim strSQL As String, strMessage As String

    Set ctrl = Me.ActiveControl

    ' abort insertion of new region if no country selected

    If IsNull(Me.cboCountry) Then

        MsgBox MESSAGE_TEXT, vbExclamation, "Invalid Operation"

        Response = acDataErrContinue

        ctrl.Undo

        Exit Sub

    End If

    ' subject to user confirmation insert a new row into Regions table,

    ' inserting NewData value into Region column and value of cboCountry

    ' control into CountryID column

    strMessage = "Add " & NewData & " to list?"

    strSQL = "INSERT INTO Regions(Region,CountryID) VALUES(""" & _

            NewData & """," & Me.cboCountry & ")"

    If MsgBox(strMessage, vbYesNo + vbQuestion) = vbYes Then

        CurrentDb.Execute strSQL, dbFailOnError

        Response = acDataErrAdded

    Else

        Response = acDataErrContinue

        ctrl.Undo

    End If

End Sub

Private Sub Form_Current()

    ' if new record set unbound combo boxes to Null,

    ' for existing record call functions to return

    ' region and country for current value of CityID

    If Me.NewRecord Then

        Me!cboCountry = Null

        Me!cboRegion = Null

    Else

        Me!cboRegion = GetRegion(Me!cboCity)

        Me!cboCountry = GetCountry(Me!cboRegion)

    End If

    ' requery unbound combo boxes to list

    ' only regions and cities for current city, if any

    Me!cboRegion.Requery

    Me!cboCity.Requery

End Sub

Private Function GetCountry(varRegionID)

    ' get CountryID for current value of Region combo box

    If Not IsNull(varRegionID) Then

        GetCountry = DLookup("CountryID", "Regions", "RegionID = " & varRegionID)

    End If

End Function

Private Function GetRegion(varCityID)

    ' get RegionID for current value of CityID field

    If Not IsNull(varCityID) Then

        GetRegion = DLookup("RegionID", "Cities", "CityID = " & varCityID)

    End If

End Function

How do I split up a large document into smaller files while keeping - Microsoft Community

bigd.0127 say:

How do I split up a large document into smaller files while keeping the same format?

How do I split up a large word document into smaller files without losing the original format?

cindy meister say:

One fairly simple method is to leverage the "Master-/Subdocument" functionality.

Make sure you save a copy the original document, in case you make a mistake and need to start over.

In the lower right-hand corner of the Word window, click "Outline" for the view. (Or use the View tab)

In the Outlining tab, go to the "Master Document" group and click Show Document.

Now select each section of text that you want to move to a separate document and click on the button "Create". (Note: in order to do this, the start of each new subdocument must be formatted with a Heading style.)

You'll see that Word puts a "box" around the text, and there's a little square at the top left of the box. Double-click this square to open the text in a new document. Now you can use "Save As" to save as a separate document.

Do this for all the sections of text you want to save as separate documents. Then close the "Master" without saving any changes.

       

terfar say:
The easiest way to split a large document is to open a COPY of the document and delete the sections you dont want and then save what remains to the local drive. Do this (opening a copy from the original each time) section by section until the split is complete.
 
Be warned, make sure that you have the non-printing characters displayed so when you make your selection to delete, you avoid deleting any important/critical paragraph or sections breaks. That is because those breaks hold the formatting information for the paragraph or whole section.

It's an assumption, of course, that each individual document you want to create has some kind of "title" or "heading" at the start of it.
