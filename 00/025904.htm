ossiemac say:

The following formula for the Cost assumes that sku is in cell A2 and the lookup range in sheet1 is A$2:$C$7 and the cost is in column 2 (or B) of sheet1.

 

=IF(ISNA(VLOOKUP(A2,Sheet1!$A$2:$C$7,2,FALSE)),"",VLOOKUP(A2,Sheet1!$A$2:$C$7,2,FALSE))

 

Retail is similar except the value from column 3 (or C)  is returned.

 

=IF(ISNA(VLOOKUP(A2,Sheet1!$A$2:$C$7,3,FALSE)),"",VLOOKUP(A2,Sheet1!$A$2:$C$7,3,FALSE))

 

Note that if the value is not found then the cell contains a zero length string. Also note that the table_array is always absolute with the $ signs if you want to copy the formula down.

macropod say:

Hi Derek,

Suppose your 'list' is mainatined in an Excel workbook and you want the project name, $ value of project and critical dates to appear in a Word document. Simply click on one of these items, copy it, then go to the destination in the Word document and choose Paste > Paste Special, check the 'Paste link' option and choose a paste format. Repeat for each of the other items. From now on, if you update any of these items in Excel, the updates will be reflected in Word.

To link content between Word documents, you'd first bookmark the desired content in the source document. Then, in the target document, you'd use an INCLUDETEXT field pointing to the bookmark in the source document. See Word's help file for details. Alternatively, you can use the same basic copy & paste link process described above. FWIW, there's also an INCLUDEPICTURE field for linking to graphics stored on disk - change the source graphic's contents and the picture in the Word document will update.

Linking between Excel workbooks is even easier: In the target workbook, simply press' =', then select the source cell in the source workbook and press Enter.

macropod say:

Hi Derek,

With MS Word and Excel, you can certainly link (cross-reference) content between two Word documents or Excel workbooks and between Word and Excel files in either direction, so that if you update the content in the source file, each of the target files will also be updated. no macros or addons are required - the applications can do it already. The real issue is knowing what content needs to be linked and deciding which file should be used as the source.

Linking to PDFs is problematic, as they're typically not amenable to having their content edited via links to other files. Rather, the linking should be done in whatever application produces the PDFs.

Insofar as your ACAD drawings are concerned, the answer depends on the application's specifications. Autodesk, for example, appears to support such linking. See, for example:
http://thecadgeek.com/blog/2007/08/quickly-link-excel-tables-to-autocad/
http://www.cadforum.cz/cadforum_en/qaID.asp?tip=3590
