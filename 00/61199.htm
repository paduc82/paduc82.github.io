Hi 40alpha,

It seems that updating the program you mentioned damaged a Windows system component that is related to the system font used by programs. Before we troubleshoot the issue, I highly recommend performing a System Restore prior to updating the program. This procedure takes your computer back to an earlier point in time using a system restore point. Restore points are created when you install a new program, driver, or updated. This will not affect your personal files, but it will remove programs, drivers, and updates installed after the restore point was made. You need administrative permission to perform this recovery procedure.

  1. Click the Start button, type Control Panel and then hit Enter.
  2. Type Recovery in the Control Panel search box.
  3. Select Recovery > Open System Restore > Next.
  4. Choose the most recent app, driver, or update on the list of results, then select Scan for affected programs. You'll see a list of items that will be deleted if you remove this restore point. If you're OK with the deletions, select Next > Finish. Or, if needed, select an earlier restore point before selecting Next > Finish.

Should the issue remain after performing a System Restore, I advise running both the DISM command-line tool and SFC tool. These tools will scan your PC for corruptions in Windows system files and restore corrupted files. Make sure that you're signed in as the administrator to run the tools.

  1. Click the Start button and type Command Prompt.
  2. Right-click Command Prompt and choose Run as administrator.
  3. Enter DISM.exe /Online /Cleanup-image /Restorehealth and then press Enter.
    • Note the space before each "/".
    • This step may take a few minutes to start and up to 30 minutes to run and complete.
  4. Enter sfc /scannow and then hit Enter.
    • Note the space between "sfc" and "/".
  5. Restart your PC.

ramesh srinivasan say:

Host applications like Control.exe, Rundll32.exe can't be Pinned to taskbar, but there is a round-about way to Pin them.

1. Create a desktop shortcut to a program that's not already Pinned to Taskbar.

2. Name the shortcut as desired (eg. Classic Wallpaper Control Panel)

3. Right-click the shortcut and click Pin to taskbar

4. Now, open the following folder:

%AppData%\Microsoft\Internet Explorer\Quick Launch\User Pinned\TaskBar

(Type or paste that path in Explorer Address bar to open it)

5. In the above folder, right-click the shortcut Classic Wallpaper Control Panel, and click Properties

6. Replace the existing target path with the following:

control /name Microsoft.Personalization /page pageWallpaper

7. Click OK.

8. The desktop shortcut you created in step 1, can now be deleted.

The Pinned taskbar Icon wouldn't refresh immediately; requires a logoff/login or restarting Explorer via Task Manager.

Optionally, you can add the classic Wallpaper Panel to Control Panel.

Hello,

Thank you for posting your concern here in Microsoft Community and welcome to the Windows 10 family. I would be happy to assist you.

From the description of the issue, I understand that you are facing  Blue Screen of Death is displayed with System Thread Exception Not Handled error.

System Thread Exception Not Handled error in Windows 8 mostly arises due to incompatible or old graphics card drivers.

In order to fix this issue, the computer must be restarted in safe mode, and the problematic graphics driver must be uninstalled. Once this is done, the latest driver of the installed graphics card must be downloaded from the vendors official site, and must be installed in order to make the system work.

I would suggest you to start Safe Mode with Networking and check for updated drivers: 

  • Open Settings, and then select Change PC settings.

  • Click Update and recovery and then select Windows Update.

  • Tap or click View details, and tap or click Check for updates. (If you don't see View details, there aren't any optional updates available.)

  • Under Optional, tap or click the update that you want to install, then click Install.

Alternatively, if the update fails to register and download the requisite drivers for the device, you can also use the vendor id of the device and look for it in the Microsoft Update Catalog. 

FAQ:

Unknown and uninstalled devices will be listed under Other. The device will have a yellow alert symbol on it. The properties for the device will show that no driver was found.

Under the Device Properties, select the Details tab and scroll down to Hardware IDs. Each Hardware ID is comprised of some very useful information. The VID value provides the vendor ID and the PID value provides the Product ID of the device. Make a note of the PID value. Enter the PID value in the search box of Update Catalog. In cases where you have multiple device types listed, narrow down your search by also adding in the VID value.

Once downloaded and extracted, you can point Device Manager to this driver to install the hardware.
