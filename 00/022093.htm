hansv mvp say:

Setting the Visible property of a shape to False (or msoFalse) should hide it (but if Design Mode is active, ActiveX controls will be visible regardless of their Visible poperty). Make sure that you used the correct sheet and names.

 

To disable the ActiveX combo box:

 

Sheets(1).Shapes("ComboBox1").OLEFormat.Object.Enabled = False

 

To disable the Forms command button:

 

Sheets(1).Shapes("Button 2").ControlFormat.Enabled = False

 

Warning: if a forms command button is disabled, clicking on it won't execute the macro assigned to it (as expected), but will select the button instead (which may not be what you want/expect).

tom van stiphout say:

You seem to have the right database design: one table for all Actions, with a ProjectID field.

As you know, an Autonumber is not supposed to convey any meaning. This is a FAQ.

However you can use the DCount function to count the number of records for the current project. If I was in the Northwind sample application, Order Details form, I could count the number of orders for the current customer like this:

Add a text box; set its ControlSource to:

DCount("[Order ID]","[Orders]","[Customer ID]=" & Forms![Order Details]![Customer ID])

You can adjust this for your object names.

john p spencer say: If Project is a string and not a number value, you need to include the text delimiters (quotes) in the third argument. There are several methods for doing this - I've listed three below. 
  1. Use two quotes to return one quote inside a string. 
  2. Use the option of using an apostrophe (single quote)
  3. Add the quote character outside the string using the Chr (character) function.
  1. DCount("[action_code]","[tbl_Actions]","[project]=""" & Forms![frm_test_action]![project] & """")
  2. DCount("[action_code]","[tbl_Actions]","[project]='" & Forms![frm_test_action]![project] & "'")
  3. DCount("[action_code]","[tbl_Actions]","[project]=" & Chr(34) & Forms![frm_test_action]![project] & Chr(34))
Hope this helps
