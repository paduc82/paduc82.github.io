stefan blom say:

When you open a document from an e-mail attachment, the document will be opened in a temporary folder and it will always be read-only. There is no way around that (as far as I know). Note that Word creates a number of temporary files on your disk as you start editing a file; that is quite normal.

hansv mvp say:

Q1) The third option

 

.Cells(1, 1).Resize(cmax, rmax).Value = Application.Transpose(a1)

 

works without errors for me. However, since a worksheet in Excel 2003 (or xp?) has 256 columns, you'll get an error if the original range has more than 256 rows: the transposed range won't fit.

 

Q2) TRANSPOSE is a worksheet function. If you type

 

Application.WorksheetFunction.

 

IntelliSense will display a list of worksheet functions available in VBA, and Transpose will be among them.

You're allowed to omit WorksheetFunction, but then IntelliSense won't display worksheet functions.

There is a difference in the way worksheet functions are handled: if you use Application.WorksheetFunction, VBA will display an error dialog if the function results in an error. If you omit WorksheetFunction, you won't get an error dialog, but the function will return an error value.

scottgem say:

By default Access is a multi-user database. However, there is one step you should take to minimize problems. You need to split the database between back end (the tables) and front end (everything else). Access includes a Database Splitter wizard to do this for you. You put the back end on a shared network drive and the front ends should go on each user's local drive.

You can restrict access to the database by only giving permissions to the back end folder to those you want to use it. But each user needs full read/write permissions.

ronrosenfeld say:

Here is a macro approach that you can use if the UDF is too slow.  You will need to run the Macro manually, or you could automate with, for example, a Worksheet_Activate or Change event.  Let me know if you need help to set that up.

The macro makes use of a User Defined Object (class module) as well as a regular module.  Because it is a macro, and not a UDF on the worksheet, we could have used the DisplayFormat property which can detect colors caused by conditional formatting.  But it was simpler to just compute the dates, and check the holiday list, and not use the color coding at all.

Insert the Class Module; rename it cWrkDays; insert the Regular module, and run the code.  You should check to make certain it is locating and naming worksheets and ranges appropriate to your "real data"

To rename the class module, after inserting it, <F4> will open the properties window and you can rename it there.

Class Module

=====================

Private pName As String
Private pHD As Long
Private pFri As Long
Private pSat As Long
Private pSun As Long
Private pWkDy As Long

    Name = pName
End Property
Public Property Let Name(Value As String)
    pName = Value
End Property

    Fri = pFri
End Property
Public Property Let Fri(Value As Long)
    pFri = Value
End Property

    Sat = pSat
End Property
Public Property Let Sat(Value As Long)
    pSat = Value
End Property

    Sun = pSun
End Property
Public Property Let Sun(Value As Long)
    pSun = Value
End Property

    WkDy = pWkDy
End Property
Public Property Let WkDy(Value As Long)
    pWkDy = Value
End Property

    HD = pHD
End Property
Public Property Let HD(Value As Long)
    pHD = Value
End Property

=====================

Regular Module

=====================

Sub GetNameCounts()
  Dim cWD As cWrkDays, colWD As Collection
  Dim rCal As Range, rRes As Range, vRes() As Variant
  Dim rHD As Range
  Dim wsSummary As Worksheet
  Dim wsData As Worksheet
  Dim R As Range

  Dim DT As Date
  Dim I As Long
 
    Set wsData = Worksheets("sheet1")
    Set wsSummary = Worksheets("sheet2")
    Set rHD = Range("Holidays")
   
    Yr = wsData.Range("M1")

    Set R = .Find(what:="Weekday", after:=.Item(1, 1), LookIn:=xlValues, _
        lookat:=xlPart, searchdirection:=xlNext, MatchCase:=False)
   
'Set Results range
    Set rRes = Range(R.Offset(1, -1), .Cells(.Rows.Count, 1).End(xlUp)).Resize(columnsize:=6)
End With
ReDim vRes(1 To rRes.Rows.Count, 1 To rRes.Columns.Count)

With wsData.Cells
    Set R = .Find(what:="January", after:=.Item(1, 1), LookIn:=xlValues, lookat:=xlWhole, _
        searchorder:=xlByRows, searchdirection:=xlNext, MatchCase:=False)
        If R Is Nothing Then
            MsgBox "Cannot find Calendar"
            Exit Sub
        End If
    Set rCal = R.Offset(1, 0).Resize(rowsize:=31, columnsize:=12)
End With

Set colWD = New Collection
For Dy = 1 To 31
    For Mnth = 1 To 12
        Set cWD = New cWrkDays
        With cWD
            .Name = rCal(Dy, Mnth)
            DT = DateSerial(Yr, Mnth, Dy)
            Set R = rHD.Find(what:=DT, LookIn:=xlValues, lookat:=xlWhole)
            If Not R Is Nothing Then
                .HD = 1
            Else
                Select Case Weekday(DT)
                    Case vbMonday To vbThursday
                        .WkDy = 1
                    Case vbFriday
                        .Fri = 1
                    Case vbSaturday
                        .Sat = 1
                    Case vbSunday
                        .Sun = 1
                End Select
            End If
           
            On Error Resume Next
            colWD.Add cWD, .Name
            Select Case Err.Number
                Case 457 'duplicate name
                    Err.Clear
                    With colWD(.Name)
                        .WkDy = .WkDy + cWD.WkDy
                        .Fri = .Fri + cWD.Fri
                        .Sat = .Sat + cWD.Sat
                        .Sun = .Sun + cWD.Sun
                        .HD = .HD + cWD.HD
                    End With
                Case Is <> 0
                    MsgBox "Error: " & Err.Number & vbTab & Err.Description
                    Stop
            End Select
            On Error GoTo 0
        End With
        Next Mnth
    Next Dy

    vRes(I, 1) = rRes(I, 1)
    With colWD(vRes(I, 1))
        vRes(I, 2) = .WkDy
        vRes(I, 3) = .Fri
        vRes(I, 4) = .Sat
        vRes(I, 5) = .Sun
        vRes(I, 6) = .HD
    End With
Next I

rRes.Offset(columnoffset:=rRes.Columns.Count).Resize(columnsize:=1).FormulaR1C1 = _
    "=SUM(RC[-5]:RC[-1])"
End Sub
