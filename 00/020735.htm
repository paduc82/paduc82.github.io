    Hi Debbie,

    You can find the full path information that you need in the Event Viewer.

    Open the Event Viewer and allow it time to populate. Then navigate through it

    as follows: Applications and Service Logs >Microsoft >Windows >Windows Defender

    and select Operational.

    You will be interested in Event IDs 1123 and 1127. Right-click on the entry and

    select Properties. You will find the path information that you need on the Properties

    Panel. After Process Name. Write it down, to use when you go to Controlled 

    Folder Access, to allow the App to pass.

    You can filter your search to show only 1123 and 1127, if you need to. Do so

    by selecting "Create Custom View" on the right side of the screen. On the "Custom"

    panel, select <All Event IDs> and type 1123,1127. then click "OK". Click "OK" on

    the next screen, and only IDs 1123 and 1127 will be displayed. You can observe

    their App paths as described above.

    CAUTION: Only gather the path information for Apps that you know you want to

    allow to pass through Controlled Folder Access. Some of these may be undesirable.

    I am assuming that you already know how to allow an App to pass through CFA.

    If I am wrong, come back and I will explain.

    The next time you want to use Autofit, just run the macro.

    It's probably simplest (at least it would be for me) to do it using VBA in Word.

     

    The following is from Tushar Mehta for processing all files in a folder and, optionally, in sub-folders

     

    Quote

    A frequent request one runs into is for some way in which one can apply some change to all the files in a folder.  This case presents a modularized solution.  While the implementation may look complex -- it does use a recursive routine to process sub-folders -- the result is a "black box" that should be used without any modification.

    An example of how one would use the code is below.  The ListAllFiles subroutine is the main code routine.  It calls the "black box" module named searchForFiles with the appropriate arguments, one of which is a 'callback' routine.  In this example, the callback routine is named processOneFile.

     
    Sub ListAllFiles()
        searchForFiles "C:\tushar\temp\", "processOneFile", "*.*", True
        End Sub
    Sub processOneFile(ByVal aFilename As String)
        Debug.Print aFilename
        End Sub

    Copy the code below into a standard module in your Visual Basic project.  At the bottom is the example from above.  The solution uses one application specific method, the Run method.  Consequently, it can be used on any platform that supports the Run method.  These applications include -- but may not be limited to -- Excel, Word, and PowerPoint.

    Option Explicit
    'A modularized solution to process all files in a folder, and optionally all subfolders in the folder _
        Tushar Mehta
     
    'There should be absolutely no reason to modify the searchForFiles subroutine.  Treat it as a blackbox _
     routine.  Do *not* tweak it for each specific search.
     
    'How to use the subroutine: _
     Call it with the four arguments: _
        DirToSearch: The directory you want to search.  Note that it must *not* end in a path separator _
            ( "\" on a Windows OS) _
        ProcToCall: This is the callback procedure called with the full name of each file found _
        FileTypeToFind: This is a search pattern for the files sought.  For example, to find all Excel _
            files use "*.xls".  This argument is optional and defaults to "*.*" (or all files) _
        SearchSubDir: Boolean that specifies whether or not to search nested folders.  The default is False.
            
    'The callback subroutine is where you process each file found.  The signature for the routine should be _
         Sub {subroutine-name}(ByVal aFilename As String)
    'Use this callback subroutine to do whatever it is you want to do with each file found.  For an example, _
     see the processOneFile subroutine below.
     
    Sub searchForFiles(ByVal DirToSearch As String, ByVal ProcToCall As String, _
            Optional ByVal FileTypeToFind As String = "*.*", _
            Optional ByVal SearchSubDir As Boolean = False)
        'by Tushar Mehta
        'This subroutine recursively calls itself if SearchSubDir is true and at least one sub-directory exists. _
         There should be no need to make any changes to this routine for any specific search.
        On Error GoTo ErrXIT
        If Right(DirToSearch, 1) = Application.PathSeparator Then _
            DirToSearch = Left(DirToSearch, Len(DirToSearch) - 1)
        If SearchSubDir Then
            Dim aFolder As String, SubFolders() As String
            ReDim SubFolders(0)
            aFolder = Dir(DirToSearch, vbDirectory)
            Do While aFolder <> ""
                If aFolder <> "." And aFolder <> ".." Then
                    SubFolders(UBound(SubFolders)) = aFolder
                    ReDim Preserve SubFolders(UBound(SubFolders) + 1)
                    End If
                aFolder = Dir()
                Loop
            If UBound(SubFolders) <> LBound(SubFolders) Then
                Dim I As Long
                For I = LBound(SubFolders) To UBound(SubFolders) - 1
                    searchForFiles _
                        DirToSearch & Application.PathSeparator & SubFolders(I), _
                        ProcToCall, FileTypeToFind, SearchSubDir
                    Next I
                End If
            End If
        Dim aFile As String
        aFile = Dir(DirToSearch & Application.PathSeparator & FileTypeToFind)
        Do While aFile <> ""
            aFile = DirToSearch & Application.PathSeparator & aFile
            If (GetAttr(aFile) And vbDirectory) = vbDirectory Then
                searchForFiles aFile, ProcToCall, FileTypeToFind, SearchSubDir
            Else
                Application.Run ProcToCall, aFile
                End If
            aFile = Dir()
            Loop
        Exit Sub
    ErrXIT:
        MsgBox "Fatal error: " & Err.Description & " (Code=" & Err.Number & ")"
        Exit Sub
        End Sub
     
    'This is an example of how to use the above subroutine
     
    Sub ListAllFiles()
        searchForFiles "C:\tushar\temp\", "processOneFile", "*.*", True
        End Sub
    Sub processOneFile(ByVal aFilename As String)
        Debug.Print aFilename
        End Sub

    macropod say:

    Try the following macro. It assumes you have a table of keywords named 'KeywordList.doc' stored in your 'Documents' folder. Modify the code to suit your actual filename.

     

    The keywords table is assumed to be in a 2-column format, with keywords/phrases in the left column starting at row 2. Row 1 may be merged. The macro includes a browser so you can select the top folder to look in. That folder and all its sub-folders will be processed.

     

    The keywords output goes to a new document, with a copy of your keywords table for each file and the word/phrase counts in the right column and the filenames on the first row. Files that have any form of protection are skipped over and a report of them is output to the document from which you run the macro. Other files that cause processing errors, such as may happen with files containing auto macros, will be skipped over and not reported.

     

    Option Explicit
    Public FSO As Object 'a FileSystemObject
    Public oFolder As Object 'the folder object
    Public oSubFolder As Object 'the subfolders collection
    Public oFiles As Object 'the files object
    Dim strFolder As String, strFile As String, strKeyFile As String
    Dim strKey As String, strFnd As String, strFullName As String
    Dim i As Long, j As Long, Rng As Range, TblKey As Table, TblDst As Table
    Dim DocSrc As Document, DocDst As Document, DocKey As Document

     

    Sub Main()
    ' Minimise screen flickering
    Application.ScreenUpdating = False
    strKeyFile = "C:\Users\" & Environ("Username") & "\Documents\KeywordList.doc"
    strFolder = GetTopFolder
    If strFolder = "" Then Exit Sub
    If Dir(strKeyFile) = "" Then
      MsgBox "Cannot find the keywords file:" & vbCr & strKeyFile
      Exit Sub
    End If
    Set DocKey = Documents.Open(FileName:=strKeyFile, ReadOnly:=True, _
      Visible:=False, AddToRecentFiles:=False)
    With DocKey
      Set TblKey = .Tables(1)
      With TblKey
        For i = 2 To .Rows.Count
          Set Rng = .Cell(i, 1).Range
          With Rng
            .End = .End - 1
            strKey = strKey & .Text & "|"
          End With
        Next i
        .Range.Copy
      End With
      .Close SaveChanges:=False
    End With
    Set DocDst = Documents.Add(Visible:=False)
    ' Search the top-level folder
    Call GetFolder(strFolder & "\")
    ' Search the subfolders for more files
    Call SearchSubFolders(strFolder)
    ' Switch to the output file
    DocDst.Activate
    ' Release object memory
    Set TblKey = Nothing: Set TblDst = Nothing: Set DocKey = Nothing
    Set DocSrc = Nothing: Set DocDst = Nothing
    ' Restore screen updating
    Application.ScreenUpdating = True
    End Sub

     

    Function GetTopFolder() As String
    GetTopFolder = ""
    Set oFolder = CreateObject("Shell.Application").BrowseForFolder(0, "Choose a folder", 0)
    If (Not oFolder Is Nothing) Then GetTopFolder = oFolder.Items.Item.Path
    Set oFolder = Nothing
    End Function

     

    Sub SearchSubFolders(strStartPath As String)
    If FSO Is Nothing Then
      Set FSO = CreateObject("scripting.filesystemobject")
    End If
    Set oFolder = FSO.GetFolder(strStartPath)
    Set oSubFolder = oFolder.subfolders
    For Each oFolder In oSubFolder
      Set oFiles = oFolder.Files
      ' Search the current folder
      Call GetFolder(oFolder.Path & "\")
      ' Call ourself to see if there are subfolders below
      SearchSubFolders oFolder.Path
    Next
    End Sub

     

    Sub GetFolder(strFolder As String)
    Dim strFile As String
    strFile = Dir(strFolder & "*.doc")
    ' Process the files in the folder
    While strFile <> ""
      ' Update the status bar is just to let us know where we are
      Application.StatusBar = strFolder & strFile
      Call GetKeyWordCounts(strFolder & strFile)
      ' Let Word do its housekeeping
      DoEvents
      strFile = Dir()
    Wend
    End Sub

     

    Sub GetKeyWordCounts(strFullName)
    If strFullName = ThisDocument.FullName Then Exit Sub
    Set DocSrc = Documents.Open(FileName:=strFullName, ReadOnly:=True, _
      Visible:=False, AddToRecentFiles:=False)
    On Error GoTo ErrExit
    If DocSrc.ProtectionType <> wdNoProtection Then
      ThisDocument.Range.InsertAfter "Not processed: " & strFullName & vbCr
      GoTo ErrExit
    End If
    With DocDst
      .Characters.Last.InsertBefore vbCr
      .Characters.Last.Paste
      Set TblDst = .Tables(.Tables.Count)
      With TblDst
        .Cell(1, 1).Range.Text = DocSrc.FullName
        For i = 0 To UBound(Split(strKey, "|")) - 1
          strFnd = Split(strKey, "|")(i)
          j = 0
          With DocSrc.Content.Find
            .ClearFormatting
            Do While .Execute(FindText:=strFnd, MatchWildcards:=False, _
                MatchCase:=False, Wrap:=wdFindStop, Forward:=True) = True
              j = j + 1
            Loop
          End With
          .Cell(i + 2, 2).Range.Text = j
        Next i
      End With
    End With
    ErrExit:
    DocSrc.Close SaveChanges:=False
    End Sub
