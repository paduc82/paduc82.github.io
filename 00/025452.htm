john mcghie say:

Your first crash was in VBA, your other crash was in Word itself.

VBA is not used ONLY by macros: a lot of Word commands trigger a piece of VBA.

When Word opens a document, it re-organises the content and lays it out in memory for efficient working.  When it saves, it reads the content of memory and re-writes it into XML (for .docx) or binary (for .doc).

Crashes on save occur when Word finds something in memory that it cannot encode into the document.  Very often, this is a corrupted tracked change.

80-odd per cent of the time, you can cure these by making a copy of the document, then going to Review>Tracking>Track Changes and turning it OFF, then to Review>Changes>Accept>Accept all changes in document.

Save the document and close it, then re-open it (the changes marking is not actually removed from the file until you close the document).

If you need to mark the changes, keep a copy of the unchanged document.  When you have completed all your editing, go to the Tools menu and choose Compare Documents to compare the current with the original version. Word will mark all the differences as changes, without the crashes.

I strongly recommend that you come out of .doc and stay out.  There's a misapprehension going around that .doc and .docx are equivalent: they are not, and never were.  Each time you downgrade to .doc, Word deletes the information that .doc can't handle.  Do this often enough and you get a very unstable document that is very prone to crashes.

Similarly, leaving "Compatibility Mode" on in .docx is not a great idea: it causes Word to try to re-interpret code in the document the way the old .doc files used to express it.  Increasingly you will come across instances where this attempt fails, and a crash is one likely outcome.

If removing the tracked changes doesn't work, come back and we'll talk you through some other methods to clean that document up.

I would also caution against using AppCleaner to remove Microsoft Office.  Basically because we don't know exactly what it does, but in computing, software tends to be "worth what you paid for it"!  If you do need to remove Microsoft Office, you need to follow this procedure exactly: http://support.microsoft.com/kb/2398768

Note: If any part of Office 2011 has run since you performed that procedure, you need to perform the whole thing again, because when Office applications start up they create a large number of supporting files that need to be removed each time.

And of course, "having" removed Office, after you re-install, you need to completely re-updated it to the latest level before attempting to use it.

Hope this helps -- if not, come back and we'll get serious with it!
