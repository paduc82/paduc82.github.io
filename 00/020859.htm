Its hard to say whats going on in this particular case, but if the internally scheduled scan isnt rescheduled for a time when the PC will normally be up and running, the scan wont run, and after a couple of weeks the status will change to Potentially unprotected and youll get the Scan now prompt:

http://answers.microsoft.com/en-us/protect/forum/mse-protect_start/mse-scan-schedule-off-by-days/f0e68e51-02d6-4fce-bf0b-999b1f0268c8

Also, if you have a Full Scan selected in the internal scheduler, running a Quick Scan wont reset the status or the Scan now prompt:

http://answers.microsoft.com/en-us/protect/forum/mse-protect_start/ran-quick-scan-but-pc-status-potentially/254b3203-3b58-47ec-93e4-760f2f0dbb19

Its also possible for a Full Scan to be shown as the Last scan data when the Full Scan didnt actually run to completion, and since an internally scheduled Full Scan will be paused by sleep transitions, there are often problems with it completing in a timely manner. Thats one of the reasons why I recommend scheduling Full Scans with an independent scheduled task:

http://answers.microsoft.com/en-us/protect/forum/mse-protect_start/running-mse-full-scan-as-a-scheduled-task/f627f5ce-0b74-4b1d-be64-d6ceeb4d9b02

Heres how youd check in Event Viewer to see if your scheduled scans are actually completing:

Event ID 1000 Scan started

Event ID 1001 Scan completed

Event ID 1002 Scan stopped (canceled)

Event ID 1005 Scan terminated due to error

 

1.) Open Event Viewer by typing eventvwr (or just event) in the search box and hitting the Enter key.

2.) Expand Windows Logs and click on the System folder.

3.) Click on the Filter Current Log option in the Actions pane or Action menu.

4.) Select Microsoft Antimalware as the Event source and enter 1000, 1001, 1002, 1005 for the Event IDs.

5.) You can then save the filtered log as a text file (Save Filtered Log File As or Save All Events in Cust).

6.)  Click on Save Filter to Custom View and name the Custom View MSE Scan Diagnostics.

 

The next time you open Event Viewer, simply expand Custom Views and click on MSE Scan Diagnostics.
