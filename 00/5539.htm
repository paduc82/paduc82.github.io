The email we all received does not make this clear, but I'm not sure these changes will amount to much on a windows phone 8.1.  The Facebook integration on windows phone 8.1 does not come by linking a Microsoft account or an Outlook.com account to Facebook, which in turns incorporates the contacts into the Windows People app on a computer. Prior to windows phone 8.1, the Facebook integration was "baked into" the OS....remember, we used to have to set up a Facebook account via the email+accounts settings?

But with Windows Phone 8.1, that "baked in" integration disappeared and now it comes via the Facebook app and is not a part of the OS at all.  

  • Facebook contacts are linked into the phone's people hub only if the Facebook app is installed on the phone
  • The Facebook feed is shown in the phone's people hub only if the Facebook app is installed on the phone
  • If you want to update your status through the People hub, you tap the "post an update" option and then select Facebook from the list of options provided...if you haven't installed the Facebook app on the phone, Facebook is not included in the list of options.  If you tap the Facebook option, it opens up the Windows Phone Facebook app.
  • Sharing photos via the Photo hub is accomplished by tapping the "Share" icon and then selecting Facebook from the list of options provided...if you haven't installed the Facebook app on the phone, Facebook is not included in the list of options.  If you tap the Facebook option, it opens up the Windows Phone Facebook app.
  • Make sure that "Connect with my phone" is turned on in the Facebook app settings.

I haven't seen any difference on my phone today....I suppose that could change, but maybe it won't make much of a difference on my phone.

As I create the Service Request on Office 365 and the Microsoft support gives me the following steps to solve my issue.

Class A and Class B users can only view their group users for Address List, GAL and Office Address Book respectively. Other users (e.g. Principal) can view all users for Address List, GAL and Office Address Book.

Besides, the Principal, Class A and Class B users can also view the default Room List.

The main steps shows as follows:

(A) Run below command to enable address book policy routing:

------------------------------------------------------------------------

Set-TransportConfig –AddressBookPolicyRoutingEnabled $True

(B) Set CustomAttribute for ClassA and ClassB users.

------------------------------------------------------------------------

(B-1) Set custom attribute 1 as ClassA on all class A users from Exchange Admin Center

(B-2) Set custom attribute 2 as ClassB on all class B users from Exchange Admin Center

(C) Creating Address Lists:

------------------------------------------------------------------------

(C-1) Address List for Students of Class A:

New-AddressList -Name "AL_ClassA" -RecipientFilter {((RecipientType -eq 'UserMailbox') -or (RecipientType -eq "MailUniversalDistributionGroup") -or (RecipientType -eq "DynamicDistributionGroup")) -and (CustomAttribute1 -eq "ClassA")}

(C-2) Address List for Students of Class B:

New-AddressList -Name "AL_ClassB" -RecipientFilter {((RecipientType -eq 'UserMailbox') -or (RecipientType -eq "MailUniversalDistributionGroup") -or (RecipientType -eq "DynamicDistributionGroup")) -and (CustomAttribute2 -eq "ClassB")}

(D) Creating Global Address Lists:

------------------------------------------------------------------------

(D-1) GAL for Class A:

New-GlobalAddressList -Name "GAL_ClassA" -RecipientFilter {(CustomAttribute1 -eq "ClassA")}

(D-2) GAL for Class B:

New-GlobalAddressList -Name "GAL_ClassB" -RecipientFilter {(CustomAttribute2 -eq "ClassB")}

(E) Creating Offline Address Books:

------------------------------------------------------------------------

(E-1) OAB for ClassA:

New-OfflineAddressBook -Name "OAB_ClassA" -AddressLists "GAL_ClassA"

(E-2) OAB for ClassB:

New-OfflineAddressBook -Name "OAB_ClassB" -AddressLists "GAL_ClassB"

(F) Creating Address Book Policies:

------------------------------------------------------------------------

(F-1) ABP for ClassA:

New-AddressBookPolicy -Name "ABP_ClassA" -AddressLists "AL_ClassA" -OfflineAddressBook "\OAB_ClassA" -GlobalAddressList "\GAL_ClassA" -RoomList "\All Rooms"

(F-2) ABP for ClassB:

New-AddressBookPolicy -Name "ABP_ClassB" -AddressLists "AL_ClassB" -OfflineAddressBook "\OAB_ClassB" -GlobalAddressList "\GAL_ClassB" -RoomList "\All Rooms"

(G) Assigning Address Book Policies:

------------------------------------------------------------------------

(G-1) Assign ABP for Students of Class A:

Get-Mailbox| where {$_.CustomAttribute1 -eq "ClassA"}| Set-Mailbox -AddressBookPolicy "ABP_ClassA"

(G-2) Assign ABP for Students of Class B:

Get-Mailbox| where {$_.CustomAttribute2 -eq "ClassB"}| Set-Mailbox -AddressBookPolicy "ABP_ClassB"

Note: No policy Created or applied to Principal as he will be using Default GAL and OAB.

So, for the first 3 rows, I want it to return the name in cell H2 on "EGL Data"

I want to be able to drag it down and it will put all of the values that are listed (grouped together) in Column H on Sheet "EGL Data" in column B exactly 3 times consecutively. 

On sheet "EGL Data" each value appears varied amounts of times, but each one needs to be on this other sheet 3 times (picture attached). 

However, when I put this formula into column B, I get a value of 0 and a warning of a circular reference. 

Even when I tried doing the same thing, but putting the function in column A to get rid of the circular reference I still got a value of 0. When I tried to evaluate the formula and when step by step, it changed the countif function to 0<3 [This was the first row so there was nothing yet]. But, it then changed 0<3 to True, as you can see in the screenshot, and is returning a value of 0.

Does anyone know how to get around this, or what I need to change?

            

bernie deitrick say:

To do what you want, you need to identify the first of each of the values from the list in EGL Data column H.

To do that, use this formula in column I of EGL Data (assumes your header is in row 1 and your data starts in row 2)

=IF(COUNTIF($H$1:H2,H2)=1,MAX($I$1:I1)+1,"")

And copy that down to match your list in H

Then, use this formula in a cell in row 2 of your first sheet - let's say that you are using cell C2:

=IFERROR(INDEX('EGL Data'!H:H,MATCH((ROW()-2)/3 + 1, 'EGL Data'!I:I, FALSE)),"")

Then in C3 and C4, use the formula 

=C2

Finally, copy C2:C4 and paste into C5:CXXX - just make sure that you choose a multiple of 3 cells so that the paste covers the entire area.
