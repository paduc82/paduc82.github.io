andreas killer say:
#1: Sure, add a msoControlPopup control to the last submenu to get another one.
#2: Not in that way, but if you add the controls to the first commandbar, you got a pulldown menu.

Andreas.


Option Explicit

Sub Example_CreatePulldownMenu()
  Dim Menu As CommandBarPopup
  Dim SubMenu As CommandBarPopup
  Dim Button As CommandBarButton
  Dim i As Integer

  'Create a popup menu...
  Set Menu = Application.CommandBars(1).Controls.Add(msoControlPopup, Temporary:=True)
  Menu.Visible = True
  Menu.Caption = "MyMenu"
  '...with a button
  Set Button = Menu.Controls.Add(msoControlButton)
  With Button
    .Caption = "Button"
    .FaceId = 100
    .TooltipText = "Hello World"
    .OnAction = "HelloWorld"
  End With
 
  'Create a sub menu...
  Set SubMenu = Menu.Controls.Add(msoControlPopup)
  SubMenu.Caption = "SubMenu 1"
  '...with a button
  Set Button = SubMenu.Controls.Add(msoControlButton)
  With Button
    .Caption = "Button"
    .FaceId = 101
    .TooltipText = "Hello World"
    .OnAction = "HelloWorld"
  End With
 
  For i = 2 To 5
    'Create another sub menu...
    Set SubMenu = SubMenu.Controls.Add(msoControlPopup)
    SubMenu.Caption = "SubMenu " & i
    '...with a button
    Set Button = SubMenu.Controls.Add(msoControlButton)
    With Button
      .Caption = "Button"
      .FaceId = 100 + i
      .TooltipText = "Hello World"
      .OnAction = "HelloWorld"
    End With
  Next
End Sub

Sub HelloWorld()
  MsgBox "Hello World"
End Sub

john p spencer say:

Ok, let's try to simplify this a bit.

Here is a slightly different VBA function.  Paste this into a VBA Module and save the module with a name OTHER THAN fReturnFieldList.

'============CODE STARTS =================

Public Function fReturnFieldList(strTableName)
Dim rst As DAO.Recordset
Dim fld As Field
Dim strReturn As String

   On Error GoTo ProcError

   Set rst = CurrentDb.OpenRecordset(strTableName)
   For Each fld In rst.Fields
      strReturn = strReturn & ", " & fld.Name
   Next fld
  
EXIT_Proc:
   fReturnFieldList = Mid(strReturn, 3)
   On Error GoTo 0
   Exit Function

ProcError:
   strReturn = ", Cannot process " & strTableName
   Resume EXIT_Proc

End Function

'================ Code Ends =====================

Now use a query like the following.

SELECT MSysObjects.Name, MSysObjects.Type, freturnFieldList([Name]) AS Expr1
FROM MSysObjects
WHERE (((MSysObjects.Name) Not Like "MSYS*") AND ((MSysObjects.Type) In (1,4,6)))
ORDER BY MSysObjects.Name;

Oh, just a by the way, the code will also return the list of fields in the select clause of a query if you pass it a query name instead of a table name.

You might like to take a look at RowNumbering.zip in my public databases folder at:

 

https://onedrive.live.com/?cid=44CC60D7FEA42912&id=44CC60D7FEA42912!169

 

Note that if you are using an earlier version of Access you might find that the colour of some form objects such as buttons shows incorrectly and you will need to  amend the form design accordingly. 

 

If you have difficulty opening the link, copy the link (NB, not the link location) and paste it into your browser's address bar.

 

This little demo file illustrates various means of computing sequential row numbers in a query.  The most efficient method is a JOIN of two instances of a table.  So, amending one of my queries as follows:

 

SELECT T1.TransactionDate, T1.CustomerID, T1.TransactionAmount

FROM Transactions AS T1 INNER JOIN Transactions AS T2

ON (T2.TransactionID<=T1.TransactionID OR T2.TransactionDate<>T1.TransactionDate)

 AND (T2.TransactionDate<=T1.TransactionDate)

GROUP BY T1.TransactionDate, T1.TransactionID, T1.TransactionAmount, T1.CustomerID

HAVING COUNT(*) BETWEEN 1 AND 21;

 

would return the rows numbered 1 to 21.  In fact the query in the demo does not return 21 rows, so to test it you'd have to use lower ranges  1 to 5 and 6 to 10 say.  Separate queries with the appropriate ranges defined in the HAVING clause would be used as the RecordSource properties of the reports.
