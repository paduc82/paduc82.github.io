GetAgeYMDTxt has two inputs: Date1 and Date2

Example:

Function CalcAgeYMD(Date1 As Date, Date2 As Date, Optional ReturnYears As Integer, Optional ReturnMonths As Integer, Optional ReturnDays As Integer) As Boolean

    On Error GoTo Err_Process
    
    Dim intYears As Integer
    Dim intMonths As Integer
    Dim intDays As Integer
    Dim blnReturn As Boolean
   
    blnReturn = False
    intYears = 0
    intMonths = 0
    intDays = 0
   
    intMonths = DateDiff("m", Date1, Date2)
    intDays = DateDiff("d", DateAdd("m", intMonths, Date1), Date2)
   
    If (intDays < 0) Then
        intMonths = intMonths - 1
        intDays = DateDiff("d", DateAdd("m", intMonths, Date1), Date2)
    End If
   
    intYears = intMonths \ 12
    intMonths = intMonths Mod 12
   
    blnReturn = True
   
Exit_Process:
    ReturnYears = intYears
    ReturnMonths = intMonths
    ReturnDays = intDays
    CalcAgeYMD = blnReturn
    Exit Function
   
Err_Process:
    MsgBox "Procedure: CalcAgeYMD" & vbCrLf & vbCrLf & Err.Number & " " & Err.Description, vbExclamation, "Error"
    Resume Exit_Process
    Resume 0

Function GetAgeYMDTxt(Date1 As Date, Date2 As Date) As String
   
    Dim intYears As Integer
    Dim intMonths As Integer
    Dim intDays As Integer
    Dim blnStatus As Boolean
    Dim strReturn As String
   
    strReturn = ""
   
    blnStatus = CalcAgeYMD(Date1, Date2, intYears, intMonths, intDays)
   
    If (blnStatus) Then
        strReturn = intYears & " Years and " & intMonths & " Months and " & intDays & " Days"
    End If
   
    GetAgeYMDTxt = strReturn

End Function

Hi Chris,
 
We had someone recently asking the same kind of question on the MSDN developer forum. A lot of my response here is a direct copy of what I wrote in response to that question.
 
Following the release of Access 2007, and now 2010, I and others no longer recommend employing the built-in Access user/group security for new databases, because it's been deprecated. It's supported now only for backwards compatibility, but that compatibility cannot be guaranteed for future versions. That said, I recommend creating your own securiy model.
 
Security and access control can be a very complex subsystem. In general, the following is what you need, and they should be addressed separately.
 
1. Create a security/access control schema. This generally consists of six tables:
  (a) A table for users, which includes passwords;
  (b) a table for groups;
  (c) a table for groups to which each user belongs;
  (d) a lookup table of possible permissions;
  (e) a table for permissions assigned to individual groups;
  (f)  a table to record login attempts; and
  (g) physical/logical security controls on tables, views and so on.
 
2. Create security/access control forms:
  (a) Login dialog;
  (b) user/group management dialog (which may also include permissions);
  (c) permission management dialog; and
  (d) login audit form/report.
 
3. Security and access control procedures.
  (a) Centralised procedures;
  (b) custom calls added to the Open event of each form to check and set rwed permissions for the form itself; and
  (c) custom calls added to other events in each form to control access to controls/data.
 
Before you start designing anything, you need to write specifications for the type and level of security and access control you want your system to employ. If you don't get it right now, it'll be a nightmare to do later. So when you've done that, come back here and we can guide you through the process.

It sounds like you're using "2 pages per sheet" both in the Page Setup dialog and in the Print dialog. These are two different things:

  1. The "2 pages per sheet" setting under "Multiple pages" on the Margins tab of Page Setup gives you an A5-size page. If you want portrait pages, you have to set the page orientation to Landscape.   2. The "2 pages per sheet" setting in the Print dialog reduces an A4 page to A5 size so you can print 2 on A4. If the A4 page is portrait, you'll get two A5 pages on a landscape sheet. Again, you'll need to print pages 1,1.

vlookup to get row number

In VBA, I would like to lookup a value on another sheet (in this case a string), and instead of getting a value from the lookup I would like to know the row number of where the string was found.  Then I will do some manipulation of data in that row.

How would I return or identify the row number of the found value?

hansv mvp say:

You could try something like this:

 

Dim strVal As String
Dim wsh As Worksheet
Dim rng As Range
Dim lngRow As Long
' the value you're looking for
strVal = ...
' the sheet you want to search in
Set wsh = Worksheets("OtherSheet")
' try to find the value in column D (change as needed)
Set rng = wsh.Range("D:D").Find(What:=strVal, LookAt:=xlWhole, MatchCase:=False)
If rng Is Nothing Then
    MsgBox "Not found"
    Exit Sub
Else
    ' row number of matching cell
    lngRow = rng.Row
End If

Hello,

Please read this before you turn off your security settings, reformat your disk, or any other time consuming tasks involved with installing Office 365.

I believe I have a very easy solution for those with Windows 10, but this should also work with Windows 8.  Windows 7 does not utilize the app store so I do not believe this is a problem for them.

I had a problem with the online installer for Microsoft Office 365 on 2 brand new computers, 1 was about 4 months ago, the other was today, 7-28-2017.  The computers in question are 2 completely different brands and builds, both with Windows 10.

My solution does not fix the problem, it merely installs Microsoft Office 365.  The problem lies within the app installer on Windows 10 and Windows 8.  If you don't believe me simply open your app store and try to update your apps.  I bet they all get stuck.  If they don't get stuck but you are still having a problem installing office I do not know what your problem is but read on because this might help.

Office 365 has an offline installer.  The file is large, like 5 gigs maybe.  Go to Microsoft.com and sign in.  Then scroll down until you see "My Account" and click it.  A new Internet tab will open and you will be on the Office home page (at least that's where i got redirected).  Click the button that says "Install" and you will be redirected to a new page that has the online installer readily clickable.  Don't do it.  Instead, click the link that says "Language, 32/64-bit, and other install options".  You will again be redirected to a new page and given 2 installer options.  Choose the one that says offline installer.  After you download the installer (took me 20 minutes), open the download file and run the "Setup" application.  This program should not take more than a few minutes to install, and you should see progress bar move.

This worked for me, but the Windows 10 App Store still has problems.
