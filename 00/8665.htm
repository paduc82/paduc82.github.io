john w. vinson say:

Data is not stored in Forms; a Form is just a window, a tool that lets you manage data stored in Tables.

 

What is the structure of your Tables? If you have a one (employee) to many (evaluations??) relationship, you need two tables in a one-to-many relationship.

 

There is nothing (that I'm aware of, I haven't worked much with 2010 yet) in Access that I've seen called a "Multiple Item" form. A Continuous form can display multiple records, and is not restricted to displaying all the fields in the record on one line; are you perhaps using the Form Wizard to create a Subform? If so it defaults to creating a Datasheet form (which DOES display all fields on one row); but you can change that - open the form in design view, select the Subform by rightclicking the little square box at its upper left and selecting Properties. Change its Default View property to "Continuous Forms" and you should be able to move controls (forms have Controls, tables have Fields, they're NOT the same).

scottgem say:

There is a Multiple Item form in 2010.  

In the Create ribbon on the Form tab, when you select the pull down, at the top of the list is a Multiple Items form. This creates a Continuous form, which is how its been known from day 1. What I think is happening here, is that the form is opening in LAYOUT view, not DESIGN view. These are different views. Also, when you use one of those choices, it stacks the controls so you need to unstack them to move them around individually.

Another point is the Multiple Items(Continuous form) form is use to display multiple RECORDS. Any form will display multiple controls (you don't have fields on a form, you have controls that may or may not be bound to fields in a record.

Given your structure, you do NOT need a continuous form. A regular form as a subform will allow you to display all the fields in the table associated with the Employee selected in the main form.

Having said that, your structure does not sound properly normalized. It sounds like you have 28 points to evaluate and you have a field for each of those points. That is using field names to define data which is incorrect. You should have a table like this:

tblEvaluations
EvaluationID (Primary Key Autonumber)
EmployeeID (Foreign Key)
EvaluationTypeID (FK)
Evaluation

You would then have a table of evaluation types to identify each record. This gives you a properly normalized VERTICAL structure instead of the horizontal one you have. And you would then use a continuous form to display all the evaluations for an employee.

Method 1: Run SFC Scan.  

To run an SFC scan, follow these steps:  

  1. Run Command Prompt as Administrator.  

  2. Type sfc /scannow and hit Enter.  

This will check and run an auto-repair on the issue.  

Method 2: Run DISM Tool.  

After running the SFC scan and the issue still persist, run the DISM to check the system health and to try restoring the files. To run DISM tool, follow these steps:  

  1. Run Command Prompt as Administrator.  

  2. Type the following commands:  

    1. DISM.exe /Online /Cleanup-image /scanhealth  

    2. DISM.exe /Online /Cleanup-image /Restorehealth  

NOTE: This might a while to finish scanning. Do not cancel.  

Method 3:  

After doing Method 1 and Method 2 but the issue still persist, you can manually reset the Windows Update Components and Run Windows Update process again.  

Resetting Windows Update Components will fix those corrupted Components and will help the Update installation quickly. To reset Windows Update Components, follow these steps:  

  1. Run Command Prompt as Administrator  

  2. Stop BITS, Cryptographic, MSI Installer and Windows Update Services. To do this, type thefollowing commands:  

    1. Net stop wuauserv  

    2. Net stop cryptSvc  

    3. Net stop bits  

    4. Net stop msiserver  

  3. Rename SoftwareDistribution and Catroot2 folder. To do this, type the following commands:  

    1. Ren C:\Windows\SoftwareDistribution SoftwareDistribution.old  

    2. Ren C:\Windows\System32\catroot2 Catroot2.old  

  4. Restart BITS, Cryptographic, MSI Installer and Windows Update Services. To do this, type thefollowing commands:  

    1. Net start wuauserv  

    2. Net start cryptSvc  

    3. Net start bits  

    4. Net start msiserver  

    5. Restart your computer and try to install Windows Update.
