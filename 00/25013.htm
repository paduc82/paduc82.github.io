Option Explicit
Function EvalFtIn(s As String) As Double
  Dim re As Object, mc As Object, m As Object
  Dim dFt As Double, dIn As Double, dFractFt As Double
  Dim dEval As Double
Set re = CreateObject("vbscript.regexp")
    With re
        .Pattern = "(?:([+x])\s+)?(\d+)'(?:\D+(\d+)"")?"
        .Global = False
    End With
 Do While re.test(s) = True
    Set mc = re.Execute(s)
    dFt = mc(0).submatches(1)
    dIn = mc(0).submatches(2)
    dFractFt = dFt + dIn / 12
    s = re.Replace(s, "$1 " & dFractFt)
 Loop
EvalFtIn = Evaluate(Replace(s, "x", "*"))

End Function

a. user say:

1. To insert a link to an Excel objects or range of cells in Word,

 a. In Excel select the range or the object (e.g. the chart/pivtochart etc.) and copy it.

 b. in Word, in the Home tab of the Ribbon, click the dropdown arrow at the bottom of the Paste icon (at the left hand end), click Paste Special..., then select the Paste Link option and choose the format you want.

The top format will be an"Excel object" of some kind and that should be as good as anything if you selected a chart. If you selected a range of values, that will work as well, but if the data needs to span more than one page in Word you will need to choose one of the other formats, e.g. HTML.

When you do this, Word inserts a LINK field. You can use Alt-F9 to toggle between "field results" view, where you should see the chart or data, and "field code" view, where you will see something more like this:

{ LINK Excel.Sheet.12 "D:\\test\\2018011202 links to excel objects\\Book1.xlsx" "Sheet2!R3C1:R7C2" \a \p }

The { } are special "field code brace" characters - you cannot type them on the keyboard, but you can enter them using ctrl-F9 in the Windows desktop version of word. As it stands, the field should autoupdate when you change the data in that range (that's what the \a is there for. However, you can update the result manually by selecting th field and clicking F9.

Now suppose you define a Range name called "myrange" in Excel that refers to Sheet2, cells R3C1 to R7C2 (e.g. by selecting the range and using the Excel ->Formulas->Define name dialog)

In word, you can manually replace the range reference by the name, so you have

{ LINK Excel.Sheet.12 "D:\\test\\2018011202 links to excel objects\\Book1.xlsx" "Sheet2!myrange" \a \p }

Then update the field and you should see the same result. (In fact, if the range name is already there when you copy the data from Excel, Word/Excel may use the range name in the LINK field in the first place). 

I do not think you can assign a range name to a chart, but you can rename the chart. As as example of a linked chart I have

{  LINK Excel.Sheet.12 "D:\\test\\2018011202 links to excel objects\\Book1.xlsx" "Sheet2![Book1.xlsx]Sheet2 mypivotchart1" \a \p }

(You might think that "Sheet2 mypivotchart1" or some other variation would be enough, but apparently not).

Just be aware that

 a. If you move your Excel file to some other folder, these links will probably break. You can use VBA to fix them up.

 b. Word sometimes modifies the code in LINK fields without asking. In particular, if you try to generalise things using nested fields, Word will at some point overwrite your fields with what it thinks ought to be in there.

Using static range names in Excel is one way to avoid hardcoding a range address such as R1C1:R13C3, but you obviously have to ensure that the range always points to the correct data, e.g. includes new rows you add at the end.

As far as I know (although it isn't something I have used) it is also possible to use a dynamic range name to specify a variable-sized area to a certain extent, e.g. if you want the first three columns of Sheet1 and you know the data will not extend below row 300, you can define a name, say "mydynamicdata" in Excel as

=OFFSET(Sheet1!$A$1,0,0,COUNTA(Sheet1!$A$1:$A$200),3)

and use

"Sheet1!mydynamicdata"

in the LINK field. But Word will not work with all types of range name. As far as I can remember, it will only work with names that specify a range of cells, rather than (for example) a constant value. Further, Word needs the sheet name in the LINK field, even though it is part of the definition of the range in Excel,

sue mosher say:

Never try to move or delete items from within a For Each loop. Instead, use a down-counting For loop. Assuming that objSourceFolder is a folder object:

    intCount = objSourceFolder.Items.Count
    For i = intCount to 1 Step -1
        Set objSourceItem = objSourceFolder.Items(i)
        If objSourceItem.Attachments.count > 0 then
            objSourceItem.Move objSaveFolder
        End If
    Next

Issues w/grouping selections in excel pivot tables. - Microsoft Community

poohette77 say:

Issues w/grouping selections in excel pivot tables.

So I have a set of data, which shows all transportation costs for loads both inbound & outbound.  From this data I typically create 3 pivot tables (on separate tabs).  One shows only outbound, another shows inboutnd, and the third I show inbound freight arriving via a certain transportation mode (in this case common carrier only). 

On this 3rd report, I typically group the originating locations by US and by Mexico loads.  The problem lately, has been that when I do this grouping, excel activates that grouping for all 3 pivot tables.  Reallly weird - I've made sure that I'm not simply, copying & redoing the same table already, so I'm sure that I'm creating a new table each time.  Didn't used to have this problem, up until a couple of weeks ago...

ashish mathur say:
Hi,
 
This happens because all PT are using the same pivot cache.  When creating the second pivot, you would have got a message asking
 
"Whether you would like to base the pivot on the pivot cache of the previous one.  Using the same pivot cache will save space" - you would have selected Yes to that question.  Select No and then see whether the Grouping applies only to the current pivot or not.
