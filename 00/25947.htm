rick rothstein, mvp - excel say:
Dates are not stored by Excel the way it displays them (just think of the memory requirement for all the possible ways each individual date could be displayed and then multiply that by the number of dates Excel can handle); rather, Excel stores dates as a floating point number (the whole number part represents the number of days offset from "date zero", 12/31/1899, and the decimal part is the fraction of a 24 hour day the time represents, 6:00am is 0.25 for example). So, you comparison is testing to see if a floating point number is "greater than" or "less than or equal to" a string value that happens to look like a date. Instead of using "3/24/2011" in your tests, use DATE(2011,3,24) instead

#1 - The default location for OneDrive is "C\users\userid\OneNote"

whereas the default location for Outlook PST files is

"C:\Users\Documents\Outlook Files

#2 - Was this PST file created by Outlook or is it one that was moved to the location by you? Have no idea how Outlook would have created a PST file in the OneDrive folder on its own so suspect that there is something missing from the description

#3 - Did not suggest setting the OneDrive location to "C:\Documents" - doesn't exist - Documents is a folder under "C:\Users\Userid". That aside, you would not want to set OneDrive to the root of the Documents folder siince that would include all sub-folders underneath it including Outlook files.

#4 - Have provided the simplest instructions on how to move your PST file - if you're remotely concerned, instead of "moving the file", you can "copy the file" to a new location but you will then have to go into Outlook and change the delivery location of where the POP3 account delivers email.

You would then need to do the following - from Outlook menu

--> File --> Account Settings
--> Data Files tab --> add the PST file from the location where you copied it "to"
--> set that new PST file as the "default for the profile"
--> switch to the email tab --> highlight the POP3 account
--> go to bottom of that window to <Change Folder>
--> select the <Inbox> for the PST file you copied to the new location
--> switch back to the Data FIles tab --> remove the old PST file

You are going to have to change the location of your PST file regardless of which approach you use since it's a matter of <IF> and not <When> that your PST file will get corrupted. As stated, active PST files configured in a current Outlook profile are not supported to be used on OneDrive

I knew how to do it in an older version but cant find it on 2007 

kabknuj say: Just found it after posting - I was using the Auto fit for the width and did not realize the height was under PROPERTIES

hansv mvp say:

Click anywhere in the table.

Press Alt+Num 5 or right-click and select Select > Table from the context menu,

On the Layout tab of the ribbon, click Properties, or right- click and select Table Properties... from the context menu.

Activate the Row tab of the Table Properties dialog.

Tick the check box 'Specify height'.

Enter a number in the box, e.g. 0.5" or 1.27cm or 36pt.

Select Exactly from the 'Row height is' dropdown.

Click OK.
