scottgem say:
Most professional developers do NOT recommend using lookup fields on the table level. They cause more problems then they are worth. The main problem is that they hide what is really being stored. See this discussion about it http://access.mvps.org/access/lookupfields.htm.

Relationships are defined by usage. If you have a foreign key that links to another table to pull info from that table, then a relationships exists whether it is formalized or not. A formalized relationship is created using the Relationships Window. You formalize a relationship by enforcing Referential Integrity. But you can also enforce RI by the design of your database. and using code.

If you use the Combobox wizard to create your combo on your form, it will, by default, hide the key column (the abbreviation) and show the user the description column (the full names). A combobox displays the first non-zero width column in the RosSource of your combo. So if you have a Rowsource like:

SELECT Abbreviation, StateName FROM tblStatus ORDER by StateName;

You would then set the Column count to 2 and the Columns widths to 0";1" (or whatever unit of measure and length you need. By the way, you don't really need the autonumber field as the abbreviation field would work as a primary key in this case. In other cases the AutoNumber is better.
