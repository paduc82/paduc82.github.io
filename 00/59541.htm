Please note the problem is NOT with the update.  Rather the issue is that there’s a mismatch of patching levels.  In March Microsoft released an update that began the process of rolling out an update to CredSSP used in Remote Desktop connection.  In May the updates mandate that a patched machine can’t remote into an unpatched machine.  If you dig into the KB there is a registry workaround to [TEMPORARILY] disable the mandate, but the better and wiser move is to update the server or workstation you are remoting into.  Make sure the “thing” you are remoting into has an update.

lemp say:

Sometimes, installing software requires a restart of the computer in the middle of the installation process.  The first portion of the installation process puts a command into the Windows Registry to run when Windows starts so that after you restart, the installation process will complete.  At that point, the installation process usually cleans up after itself and removes any temporary files and registry entries.

What seems to have happened to your system is that the command to finish the installation process got left in the Windows registry, but the files either were deleted or (since you actually found 3 copies) something else went wrong.

I assume that the printer works properly now.  Is that correct?

You could not have found 3 files named cnmss.dll in the same place.  Windows does not allow duplicate file names in the same location (although you can, of course, have the same file name in a different location).  I assume that the files you found had a number somewhere in the file name to indicate that they were copies of cnmss.dll.

In any event, download and run Autoruns:  https://docs.microsoft.com/en-us/sysinternals/downloads/autoruns

  • When the scan is complete (it will show "Ready" at the bottom left corner), select the "Logon" tab.
  • Click the "binoculars" icon (or press CTRL+F) and search for cnmss
  • You can probably safely delete the line that contains cnmss, but to be extra safe, just clear the check box at the left end of the line.  You can delete the line later, after restarting a few times and determining that no problems arise.
  • If you restart the computer now, the error message should not appear.

***********************************************

You really don't need to update drivers.  In general, you should only update a driver when (a) you know that the update will fix a problem that you're experiencing or (b) you know that the update will add a feature that you want/need.

BTW, we're not "Support" here.  We're just Windows users with no connection to Microsoft who volunteer to try to help others.

Feel free to ask more questions if the above procedure doesn't fix things for you.

a. user say:

Your reply is incorrect! Windows 7 does NOT install the special keys assignment function software for the keyboard. It treats the Wired Keyboard as a generic 104 key input device when plugged into the PS/2 port. The only CORRECT solution is to make sure that you NEVER install any version of IntelliType above 7.1 (ITPx64_1033_7.10.344.0.exe) because any higher version including the Microsoft Mouse & Keyboard Center (an update installed via Windows Update) will render the special key assignments unavailable.
