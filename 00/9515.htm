cpitchford say: First thing, the .sh is not optional.. unlike windows where you could run edit.exe or just edit, under unix you need to be precise

Second, the file itself doesn't have the execute permission set, so you cannot run it directly.. if you did chmod +x flasher.sh that turns it from being a text file into a program that you can run

Thirdly, the search path on your Unix system doesn't contain the current directory.. When you run "ls" the system checks /bin/ls, if it exists it uses that file, if not it checks /usr/bin/ls, if that exists it uses that file.. and it keeps going through the search path..
To run a program that doesn't exist in the search path (ie in the current directory) you need to tell the system where it is.. prefixing ./ does this

./flasher.sh would work if you chmod +x it first..

If not it will complain that you're trying to run something that isn't executable..

Using bash <filename> will "run" a text file as if it were an executable.. in fact a shell script when executable is run in exactly this way due to some magic in the kernel you don't need to worry about..

If you data is in Col A (IP) and B (Hostname) of Sheet1 and you want to fill the blank hostnames by the names available in Sheet2 [ Col A (IP) and B (Hostname) ]  then you can use VLOOKUP ...

 

Let us say the first blank hostname is in cell B3 (Sheet1) then do the following;

=VLOOKUP(A3,Sheet2!A:B,2,False) and press CTRL-ENTER
