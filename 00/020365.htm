It’s not feasible to set up Address Book to search GAL (Global Address List) in Outlook 2016 for Mac because Outlook clients on Mac don’t support sync or update GAL. jay freedman say:

For the character formatting such as bold, I can reproduce the carry-over but only for manually applied (not style applied) formatting. To prevent the cross-reference from getting the manual formatting of the caption, open the cross-reference's field code and insert a switch, \* charformat. This will force the field result to have the same formatting as the first character of the field code (in this case, the R of REF) regardless of what's in the caption. Of course, it would be better to modify the Caption style instead of applying manual formatting, and then there wouldn't be any carry-over.

With regard to the manual page break, I can suggest something that may help.

When you insert a cross-reference, Word creates a hidden bookmark that surrounds the caption (or just the caption's label and number), and the cross-reference is just a REF field that points to that bookmark. Open the Insert > Bookmark dialog and check the box for "Hidden bookmarks". Click on one of the bookmark names that start with _Ref and click the Go To button. That will select the bookmark. If the bookmark includes the preceding page break, then the cross-reference will also include the page break. In that case, delete the page break and reinsert it outside (above) the hidden bookmark.

WRONG:

RIGHT:

The "wrong" arrangement can happen if you insert the cross-reference (creating the bookmark) first, then put the cursor at the left end of the caption and insert the page break. If you do these steps in the reverse order, the bookmark will automatically exclude the page break.
