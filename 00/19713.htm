I assume that the Error! Bookmark not defined appears where the page number should be and it would indicate that the Heading for that item has been deleted.

In deleting the heading, it would appear that the heading style has been applied to the text that was under that heading so that when you do a full update of the table of contents (not just the page numbers), the text that now has that heading style assigned to it is now being included in the table of contents.

To fix it up, you will need to go to that page of the document, and reinstate the missing header and change the style that is assigned to the text to something that is appropriate.

bernie deitrick say:

To use a formula with the data you currently have, you need to array enter (enter using Ctrl0Shift-Enter)

=SUM(IF(MONTH(Attendance!A3:A3000)=1,Attendance!C3:C3000))

 

To use a SUMIF formula, you need to have a column of formulas on Attendance, say D3:D3000 with =MONTH(A3) copied down, and then use

 

SUMIF(Attendance!D3:D3000,1,Attendance!C3:C3000)

 

Easier would be using a pivot table, grouped by month. No formulas....

jeeped say:
The SUMIF() does not allow internal functions to parse a part of the criteria like yuou are attempting to do with the month() function. You can use a pseudo-array style of SUMPRODUCT() or an array formula style of SUM(IF(). Here are two examples,

=SUMPRODUCT((MONTH(Attendance!A3:A3000)=1)*Attendance!C3:C3000)

... or,

=SUM(IF(MONTH(Attendance!A3:A3000)=3,Attendance!C3:C3000))

The latter needs to be finalized with Ctrl+Shift+Enter as it is in true array formula format. If you want to copy those down for subsequent months, try,

=SUMPRODUCT((MONTH(Attendance!$A$3:$A$3000)=ROW(1:1))*Attendance!$C$3:$C$3000)

... or,

=SUM(IF(MONTH(Attendance!$A$3:$A$3000)=ROW(1:1),Attendance!$C$3:$C$3000))

... which will grow the ROW(1:1) to ROW(2:2), etc. while retaining the same range in columns A and C .

Is there any way to take off these single quotes around names from sent items folder and make outlook not to use them anymore? 
karl timmermans [outlook mvp 2012-2018] say:

The "sender's" name is controlled by the "Sender" and how the email display name has been setup "by the sender" so it's not unusual to get different "sender" names appearing depending on what account and/or from which computer/device the person sends the message.

There are 2 component parts that comprise the email display name but how the user has setup up their EDN is up to them - not Outlook since not everyone is religious in being consistent and not every email client that sends a given message is identical in what gets included in the oubound header.

EDN format is:

(1) Name (2) (email address) i.e.MyName (me @ somewhere.com)

If (1) is present - that's what is going to be used by Outlook to identify the sender. If (1) is not present - the email address will be used to identify the sender

If you want consistency from a given sender - at a "very minimum" advise the sender to use the same settings from wherever email is sent (any computer used (home vs office), any Outlook profile used, all mobile devices, etc)
