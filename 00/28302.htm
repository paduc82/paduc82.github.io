brad_1959 say:

Ok, I think I got it using index, match, and indirect.  I am posting my solution for two reasons; first to help anyone with a similar issue, and second for more experienced Excel users to review and help refine for issues that I dont see or could be improved.

 

Row 19 repeats the column header id, so if it is column N, then the value of cell N19 is "N".

 

Row 22 has the characters but allows numerous spaces between types of inputs and single spaces between words of the same type of input. For example, "Main St" would be different types of inputs for the name and designation, while "Los Angeles" would be two words within the same type of input.

 

Row 30 has the x coordinate of the character in row 22 in the same column.

 

Row 42 has the characters in row 22 concatenated and then parsed into individual letters with no spaces between the words.

 

Row 43 has the column id letter from row 19 of which the corresponding character in 42 was found.  This accounts for the location of the previous character to adjust the start of the array so that any previous characters are in the search. 
=INDEX(INDIRECT(M44&19):$BS$30,1,MATCH(N42,INDIRECT(M44&22):$BS$22,0))

 

Row 44 contains the letter of the next column from row 43, to be feed back in the next search in 43.  There is probably a more eloquent way to do this as part of the equation in row 43, but this helped me figure it out by having separate calculations.
=INDEX(INDIRECT(M44&19):$BS$30,1,1+MATCH(N42,INDIRECT(M44&22):$BS$22,0))

 

Row 45 has the x coordinate taken from row 30 for the character in row 42.
=IF(N42="","",INDEX(INDIRECT(N43&19):$BS$30,12,MATCH(N42,INDIRECT(N43&22):$BS$22,0)))

 

The key for me was the indirect reference where I was able to build the upper left corner of my arrays to prevent searching of the columns previously skipped or referenced.
