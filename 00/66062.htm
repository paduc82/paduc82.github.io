Think the answer provided does not address the issue, I too have seen this phenomenon and seen it discussed in several forums. 

A good suggestion is that the "stall" is probably caused by what you are asking the computer to do...i.e. list folders you can save in, and usually the access times of network volumes or slow drives causes it to take longer to list the cached directories.  Additionally, an unavailable mapped drive will cause a very long delay and is the usual cause for this behavior. So the delay you're seeing is likely caused by the application attempting to find it to cache the folders available.

Go to Start-->Computer and delete the network drives with an "X" in red or go to tools (press the alt key if you don't see the menu) and select Disconnect Network drives then select them (ones with the "X" in red) and disconnnect them one at a time

if you can't delete the network drive here try via the command prompt Start -> type CMD in the search feild and hit return or in the run line in XP etc. then...

net use h: /d


where instead of "h" you specify the drive that is to be removed.  Then restart.

A more drastic approach would be from a cmd prompt

net use /persistent:no 

but I think this removes all persistent network shares on the next reboot.

I am still searching but there maybe a way to edit the registry so that it does not cache network share directories or you can alterantively not map any directories.  In either case try this and I think you should see an improvement in the Save in dialog.
 

By the way if you get an error that it can't remove the share you are specifying you still need to restart, it should be gone on reboot.  If not, well you have a new problem to google/bing/altavista/etc using the search engine of your choice.
