john p spencer say:

Yes, it should be possible.  The following applies to Access 2003, but I think it should work with Access 2007 also.

You can add more items to the switchboard after you've created the switchboard using the Switchboard Wizard. You need to do the following steps.

1.  Open the code window for the form and change the line
      Const conNumButtons = 8
  to
      Const conNumButtons = <The Number You Need>

2.  Add extra command buttons to the form and name both the button and it's label after the pattern used for the other buttons and labels:

Buttons --> Option1, Option2, Option3, etc.
Labels --> OptionLabel1, OptionLabel2, OptionLabel3, etc.

Set the Event Property for each button to read
   =HandleButtonClick(#) 
Where # is replaced by the number of the button

3.  Open the Switchboard Items Table and add the necessary commands to that table. You should be able to figure out from looking at what is already there what you will need to add to the table.  The switchboard manager won't let you ADD items beyond 8, but it should let you edit them once they are added.

BEFORE you attempt this, either back up the database or make a copy of the switchboard form.  If things go badly you can rename the copy of the switchboard form back to its original name.

hansv mvp say:

You can add new buttons and labels to the Switchboard form, and change the value of the constant conNumButtons in the code behind the form from 8 to the desired value.

 

The buttons should be named Option9, Option10 etc., and the labels OptionLabel9, OptionLabel10 etc.

The On Click event of the new buttons and labels should contain the appropriate expression, for example for Option9 and OptionLabel10 it should be

=HandleButtonClick(9)

 

However, the SwitchBoard Manager has a maximum of 8 buttons hard-coded, so you will have to specify the actions of the new buttons by adding new records to the Switchboard Items table manually, in analogy to the existing records, so you will have to study the table and the code behind the Switchboard form to discover how it works.

 

Do *not* use the Switchboard Manager any more once you have modified the design of the switchboard.

"...why Publisher does this..." -  My head starts hurting every time I try to explain the whyfor of image editing and resolution, but I think that one way to understand it is that it isn't Publisher doing this exactly, it is your monitor. Your monitor is set to render images at 96 dpi, or in reality it is 96 ppi...pixels per inch. DPI is print media terminology. Your original 3225 x 2175 image at 300dpi is a printed image that is 10.75 in. X 7.25 inches (3225/300=10.75). If you want your image to be 10.75 in. X 7.25 inches on your monitor in Publisher then the image has to be 1032X696 (1032/96=10.75) Your monitor can only render 96 pixels per inch and thus must convert you 300 dpi image in order to keep the dimensions of the image the same. Clear as mud, eh?

 

"...2580 x 1740 px at 240 dpi..." - I am not sure. After inserting your image and scaling it full size, go back to the format dialog and make sure the size is actually 1032X696. Sometimes when I scale things the actual dimensions are different and I have to enter the pixel height and width instead of using the 100% scale. Once you confirm that the image is the correct size on the page then try right click, save as an image, and specify 300 dpi under the options and see if you can at least recover the 300 dpi resolution with the Save as an Image step.

 

"...already created several files..." - Unfortunately there is no easy way to scale those pages back. You will have to redo them I am afraid, but before you do, try creating just one custom page at the reduced size and add just a bit of the other design elements and try saving the whole page as a 300 dpi image and see if this will even work. You might also need to group all the design elements you add to your background image and then do a right click of the group instead of the page and save as an image.

 

If you keep choking on size when trying to save be sure everything is on your local drive and not on an external drive or removable media.

 

You might find it easier to go the other direction. Create your text, etc. in Publisher without the background image and save it as an image and then add it to the background image in your photo editor.

This is on the presumption that your dropdown is in cell A1 in sheet1 
---------------------------------------------------------------------------------

Private Sub Worksheet_Change(ByVal Target As Range)
    If Target.Address(0, 0) <> "A1" Then Exit Sub
    
    Dim sht As Worksheet, rng As Range, r As Range, i As Integer
    
    Set sht = Worksheets("Sheet2")
    Set rng = sht.Range("B:B").Find(Target.Value)
    
    If rng Is Nothing Or Target.Value = "" Then
        Me.Range("B:B").ClearContents
        Exit Sub
    End If
    
    Dim lngLastRow As Long
    lngLastRow = rng.End(xlDown).Row - 1
    
    If lngLastRow + 1 = sht.Rows.Count Then
        lngLastRow = rng.Offset(0, 1).End(xlDown).Row
    End If
    
    Set rng = rng.Offset(0, 1)
    Set rng = rng.Resize(lngLastRow - rng.Row + 1)
    
    Me.Range("B:B").ClearContents
    i = 0
    For Each r In rng.Cells
        Me.Range("A1").Offset(i, 1).Value = r.Value
        i = i + 1
    Next r
End Sub



Hope this works for you

scottlutes say: Using the info below, I have a macro that should help you out.  You place it into the Sheet that you are using.  It will watch a range (for the testing, I just used A1:A6000.  When the job title is entered for Team Leader, Assistant TL or Driver, it will create a Data Validation in the next column, with the duties related to that title.  You will have to modify the list locations, etc., which are located in the lines xlBetween, Formula1:="='Sheet1'!$T$2:$T$9" - Just change the sheet name and the row/columns as needed.  For my Job title sections, I just used the Data Validation to create a drop down list with the titles.

[CODE]
Option Explicit
Private Sub Worksheet_Change(ByVal Target As Range)
If Intersect(Target, Range("A1:A6000")) Is Nothing Or Target.Cells.Count > 1 Then Exit Sub
Application.EnableEvents = False
If Target.Value = "Team Leader" Then
        Target.Offset(0, 1).Select
        With Selection
            With .Validation
            .Delete
            .Add xlValidateList, AlertStyle:=xlValidAlertStop, Operator:= _
            xlBetween, Formula1:="='Sheet1'!$T$2:$T$9"
            End With
        End With
End If

If Target.Value = "Assistant TL" Then
        Target.Offset(0, 1).Select
        With Selection
            With .Validation
            .Delete
            .Add xlValidateList, AlertStyle:=xlValidAlertStop, Operator:= _
            xlBetween, Formula1:="='Sheet1'!$T$11:$T$14"
            End With
        End With
End If

If Target.Value = "Driver" Then
        Target.Offset(0, 1).Select
        With Selection
            With .Validation
            .Delete
            .Add xlValidateList, AlertStyle:=xlValidAlertStop, Operator:= _
            xlBetween, Formula1:="='Sheet1'!$T$16:$T$20"
            End With
        End With
End If
Application.EnableEvents = True

End Sub
[/CODE]
