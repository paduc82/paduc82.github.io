ken sheridan say: You don't need to go to those lengths.  Does your multi-valued field reference another table?  In most situations this should be the case.  In the example below a Cities MVF in a Contacts table references a Cities table.  What is being represented here is a many-to-many relationship type between Contacts and Cities by means of the MVF, rather than the conventional (and preferable in my view) method of modelling the relationship type by a third table, sometimes colloquially called a 'junction' table, which resolves the many-to-many relationship type into two one-to-many relationship types.

A MVF has a Value property.  Where a MVF references another table, in each row there are a set of hidden numeric values, one per item in the MVF, which reference the key of the referenced table, Cities in this example, so the tables can be joined on the Value property, e.g.

SELECT Contacts.FirstName, Contacts.Lastname, Cities.City
FROM Contacts INNER JOIN Cities
ON Contacts.Cities.Value = Cities.CityID;

As the City column is the non-key text column from the referenced table, the query can be restricted on this column in the usual way to return all rows from contacts where, in this example, one of the cities in the MVF is Birmingham:

SELECT Contacts.FirstName, Contacts.Lastname, Cities.City
FROM Contacts INNER JOIN Cities
ON Contacts.Cities.Value = Cities.CityID
WHERE Cities.City="Birmingham";

If, on the other hand, the MVF unwisely gets its values from a value list, the Value property will be the text value, so the query would be:

SELECT FirstName, Lastname, Cities.Value
FROM Contacts
WHERE Cities.Value="Birmingham";

ken sheridan say: If you want to return all values in the MVF for each row where one of those values matches the search criterion then you can join an outer query to a subquery in which the rows returned are restricted in the way I described.  Continuing with my hypothetical  contacts example, my original query would return the following result table:

Jane    White    Birmingham
John    Black    Birmingham

whereas the following query:

SELECT C1.FirstName, C1.Lastname, Cities.City
FROM (Contacts AS C1 INNER JOIN Cities
ON C1.Cities.Value = Cities.CityID)
INNER JOIN (SELECT Contacts.ContactID
                      FROM Contacts INNER JOIN Cities
                     ON Contacts.Cities.Value = Cities.CityID
                     WHERE Cities.City="Birmingham") AS C2
ON C1.ContactID = C2.ContactID;

would return all rows where one of the values in the Cities MVF is 'Birmingham', but unlike my original example it would return all the cities in the MVF for each of those rows, not just Birmingham.  The above would return a separate row for each city, e.g.

Jane    White    Birmingham
Jane    White    Manchester
John    Black    Derby
John    Black    Birmingham

The following, on the other hand would return a single row per contact with the cities shown as a comma separated list:

SELECT C1.FirstName, C1.Lastname, C1.Cities
FROM Contacts AS C1
INNER JOIN (SELECT Contacts.ContactID
                     FROM Contacts INNER JOIN Cities
                     ON Contacts.Cities.Value = Cities.CityID
                     WHERE Cities.City="Birmingham") AS C2
ON C1.ContactID = C2.ContactID;

So its result table would be:

Jane    White    Birmingham, Manchester
John    Black    Derby, Birmingham

karen_[01] say:

Hello

It's possible to have two or more than one Skype accounts under one email address. Before Skype and Microsoft migration, it's possible to create a new Skype account using the same email address associated with an existing Skype account. Note that this is no longer possible.

After the migration, you can only create one Skype account using your email address or phone number. However, If you have signed in to Skype using the email address/Microsoft account that is not yet linked to a Skype account, it will create a new account.

All accounts created using the Microsoft/email credentials and phone numbers are considered linked, therefore, it's no longer possible to link it to another existing Skype account. Also, if you wish to delete it, note that it will also delete the associated Microsoft account.

You mentioned that you were able to check if you have multiple accounts, but you were able to confirm that you only have one. Did you use the diagnostic to check it?

If no, please follow these steps to double check if there are other Skype accounts associated with your email address.

  1. Go to our Problems signing in page.
  2. Enter your email address or phone number and select Continue.

    Important note: When entering your phone number, be sure to enter it with no spaces or dashes (example: 12223334444).
  3. We'll show you all the Skype accounts associated with that email address or phone number.

Let me know if you need further assistance.
