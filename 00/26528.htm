Do you have to have "tbl", "frm", "qry" etc in front of the name?

When looking at other databases (including one that was used during the training class) I don't see the field_names or table_names displayed like this or with the tbl_Name displayed like this with in the navigation pane.

 

Is this format required for the database to work? If yes, then why don't other databases have their information displayed this way?

 

Thanks for any information offered.

 

Kathy

hansv mvp say:

It is not required, it is merely a convention. Databases can work fine with or without the convention.

 

It does have advantages.

 

For example, you cannot have a table and a query with the same name in a database. So for example if you have a table named Customers, you cannot create a query named Customers.

If you use tblCustomers and qryCustomers, no conflict arises.

 

You can also use prefixes for the names of controls (text boxes, labels, command buttons etc.) on forms and reports, for example

 

txtLastName

lblLastName

cmdClose

 

In VBA code, using prefixes such as tbl, qry, frm etc. make it immediately clear what kind of object a name is referring to. For the same reason, programmers often (but not always) prefix the name of variables in VBA code with an abbreviation of the data type:

 

Dim strText As String

Dim lngLength As Long

Dim curAmount As Currency

etc.

Re:  formula issue

1.  replace the the period "." following each sheet name with an exclamation point "!"
2.  replace the comma "," between A$2,F$350 with a colon ":"

So, for instance... 
VLOOKUP(A2,Sheet3.A$2,F$350,3,0)))
Looks like...
VLOOKUP(A2,Sheet3!A$2:F$350,3,0)))
