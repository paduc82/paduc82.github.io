jllatham say:

Writing to hidden worksheets, and even hidden workbooks can be done in Excel VBA.  You just have to be specific about what to write to.  I don't know what your code looks like, but I know that about 10 to 15 years ago I might have done something like

Sheets("DataSheet").Select

Range("B5")=CalculatedValue

Sheets("HomeSheet").Select

 

and all of that would have caused a lot of screen flickering and annoyance to the user.  But in VBA you can simply do this:

Worksheets("DataSheet").Range("B5")=CalculatedValue

no screen flickering, very fast and "DataSheet" does not even have to be the active  sheet nor even be visible. But it does, as written, have to be in the same workbook with the code running.  But you can even write to a different workbook by specifying it, like this:

Workbooks("MyDataWorkbook.xlsx").Worksheets("DataSheet").Range("B5")=CalculatedValue

 

It all becomes even faster and 'quieter' if you use Objects to refer to Workbooks, worksheets, and even cells and ranges of cells:

Dim dataWB as Workbook

Dim dataWS as Worksheet

Dim searchRange As Range

Dim foundCell as Range

 

Set dataWB = Workbooks("MyDataWorkbook.xlsx")

Set dataWS = dataWB.Worksheets("DataSheet")

'this would set searchRange to all cells in column A from row 1

'down to the last cell with a value or formula in it on the sheet named DataSheet

Set searchRange = dataWS.Range("A1:" & _

 dataWS.Range("A" & Rows.Count).End(xlUp).Address)

'let's say we have a value of some type in a variable named PleaseFindMe, we could

'find the first instance of it in column A this way

Set foundCell = searchRange.Find(what:=PleaseFindMe, LookIn:=xlValues, LookAt:=xlWhole)

If Not foundCell Is Nothing Then

   'we found it, so we could refer to things using foundCell properties such as .Row or .Value

Else

   ' it was not found in your list, handle that situation here

End If

 

FINALLY - one big thing that can both improve performance AND help cut down on screen flickering even if you are switching between worksheets, workbooks and/or cells on sheets is to put this simple statement ahead of where you start that work:

Application.ScreenUpdating = False

you don't even have to remember to set it back to True because when you either Exit Sub or hit your End Sub statement, it automatically resets.  This statement alone can (with sheets visible on screen) speed up things a hundred-fold or more in some situations.

 

You might want to look through this "Introduction..." to Excel VBA programming that I put together, it talks about these things and shows how to use them.

http://www.jlathamsite.com/Teach/VBA/ProgrammingInExcelVBA_AnIntroduction.pdf

You'll fiind that and some companion workbooks here

http://www.jlathamsite.com/LearningPage.htm

Plus there's a follow-on .pdf booklet about "Bulletproofing" your code (it really can't be done, but you can make it bullet "resistant").

cjkee say:

I had this problem with my MyMathLab.com website, too.


This is probably due to the fact that IE11 changed its UserAgent string in the browser. This is the string that many websites use to check which version of the browser you are using so that it can decide how to display things specific to that browser. IE11's new UserAgent string no longer contains the specification of "IE11" or "Internet Explorer" in the string because it is only advertising itself as an HTML5 browser. Problem is, older websites that can't find "Internet Explorer" in the string will tell you it's "unsupported". The only thing I have found that you can do for now is install another browser for those school sites until they update their website code to detect this new HTML5 standard string.


To test if this is the case, you can press F12 to go to the Developer Tools, click on the bottom icon on the left side for Emulation Mode, find the drop-down box for "User agent string" and choose "Internet Explorer 10". Pressing F12 again will dismiss the tools. Then your site should work. Problem is, I don't know how to get the browser to do this every time I go to certain sites, so I just installed a different browser for my math homework site.
