trevor rabey say:

Kelly,

I think that changing the standard calendar in the master and then using the organiser to move calendar that to the global.mpt is a mistake.

It would be better to still have a clean standard calendar in the global.

It would be better if you had copied the clean standard calendar in the master, with a name such as "AA My Standard Calendar", and then changed the copy as you wish.

Then, don't put this new calendar in the global.

Instead, use the organiser to copy the new calendar from the master (the source) to whichever other file needs it (the target).

Both files have to be open. It is better to run the organiser from the target file.

Copy the calendar to the resource pool, and/or to any of the individual project files, then use project information to assign that calendar as the project calendar each file, replacing the standard calendar in each, and also especially assign the new calendar as the base calendar for the existing resources.

This way, you always have clean standard calendars available in each file in case you need them (you will) and also you have a nice clean global.mpt. Otherwise, you have a mess.

john - project say:

Kelly,

Remember that in a dynamic master, the subproject files are not actually part of the master so internally the master and each subproject can have their own calendar. However, when looking at a view in the master, (e.g. Gantt Chart view), the view is "owned" by the master so if the timescale is set to use your custom calendar to display non-working time, that's what you will see, even though the tasks of the subprojects may be scheduled on a different calendar. The bottom line is that all files in a master/subproject struture should be using the same calendar.

Contrary to what Trevor recommends, I suggest you DO put your custom calendar in your Global. That way it is readily available for any new files you create.

With regard to setting the base calendar for each subproject, sorry but as far as I know there is no quick way to do that, other than with some VBA, however, it would likely take less time to do it manually then to develop the macro to automate it. Once the calendars are set up, you're done, so it's not that big of a deal. Note, if you do this type of thing again, you could create a template with all your company defined attributes and then create individual files from that template.

Hope this helps.

John
