ken sheridan say:

Going back to your original post, let's examine the issues one by one:

1.   The column orderâ€¦.

Differing column orders are easily solved in the UNION query by selecting the relevant columns in the same order in each part of the UNION operations.  If Table2 has columns A,B,C in that order and Table2 has columns A,C.B in that order the query would be:

SELECT A,B,C

FROM Table1

UNION ALL

SELECT A,B,C

FROM Table2;

2.  â€¦the number of columnsâ€¦

Each part of the UNION operations must return the same number of columns.  If the number of columns in the tables differ, the gaps can be filled with a NULL where the column is missing from a table.  So if Table1 has columns A,B,D and Table2 has columns A,B,C,D:

SELECT A,B,NULL AS C, D

UNION ALL

SELECT A.B,C,D

FROM Table2;

Or it might be more appropriate to fill the gap with a zero:

SELECT A,B,0 AS C, D

UNION ALL

SELECT A.B,C,D

FROM Table2;

3. â€¦the header namesâ€¦

Again simply solved, e.g.

SELECT A,B,C

FROM Table1

UNION ALL

SELECT X.Y,Z

FROM Table2;

4.  The Debit Card used a negation parentheses around the amount.  The Credit Card does not use a parenthesis for negation, each transaction comes in as a positive number (black text) and the payment comes in as a number with a minus sign in front in red text.

You can use a combination of the IIF function, the Format function, and multiplication by -1 to return the value as parenthesized amount in dollars if negative, e.g. in the debug (immediate) window:

Amount = "-123.45"

? IIF(Amount>=0,Format(Amount,"$0.00"),Format(Amount*-1,"($0.00)"))

($123.45)

Amount = "123.45"

? IIF(Amount>=0,Format(Amount,"$0.00"),Format(Amount*-1,"($0.00)"))

$123.45

So in a UNIION query where Table1 has the negative values parenthesized and Table2 has them with the negative sign:

SELECT Amount

FROM table1

UNION ALL
SELECT IIF(Amount>=0,Format(Amount,"$0.00"),Format(Amount*-1,"($0.00)"))

FROM Table2;
