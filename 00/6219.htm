    Original Title: 20 digit numbers

    I am importing from a db records for analysis in .csv file. One of the numbers is 20 digits and is critical. When importing it translates it into an E+ formula in the cell and replaces the last 5 digits with 0's in the display field.

    How can I display and use the original value of 20 digits????
    j.e. mcgimpsey say: If it's actually a number, and the digits are critical, you're pretty much out of luck. Excel, like nearly all spreadsheet programs, stores numbers and calculates using IEEE double-precision floating point values. That standard provides 54 binary bits in the mantissa, which equates to about 15 decimal digits. You can't get any more precision than that out of XL's calculations or numeric storage.

    If what you really have is a 20-digit string, and aren't doing calculations on it, import it as Text (in the third pane of the Import Text wizard, click the column header and choose Text).

    If you have to do calculations, you could import the values as Text, then break up the string into parts and perform calculations step-wise, but it's a bit of an effort.

    john w. vinson say:

    The third argument to a Domain function like DLookUp needs to be a criterion to select which record to look up - a valid SQL WHERE clause without the WHERE. Try

    =DLookUp("AnnualBudgetAmt","tblBudgetedAmounts","[Budget#] = " & [Budget#])

    It would probably be best to avoid using special characters like # (which is a date delimiter) in field or control names, and it may be necessary to remove the ambiguity (is [Budget#] a field, or a control, or both?) by renaming the header textbox to something like txtBudgetNo:

    =DLookUp("AnnualBudgetAmt","tblBudgetedAmounts","[Budget#] = " & [txtBudgetNo])

    If the [Budget#] field is of Text datatype you'll need to delimit it with ' marks:

    =DLookUp("AnnualBudgetAmt","tblBudgetedAmounts","[Budget#] = '" & [txtBudgetNo] & "'")

    1. Display field codes using Alt+F9. This will make your hyperlinks display as field codes. For example, a link to this page would be displayed as { HYPERLINK "http://answers.microsoft.com/en-us/office/forum/office_2010-word/making-links-in-word-blue/bf5d471d-4a5f-4037-8e42-372de429864f" }

    3. Click More>> to expand the dialog.
    4. In the "Find what" box, type

    5. With the insertion point in the (empty) "Replace with" box, click Format, then Style.
    6. Select the Hyperlink style and click OK.
    7. Click Replace All.
    8. Press Alt+F9 again to toggle the field display back to results.

    Sub RefreshMonthlyData()
      'assumes all workbooks, including this one
      'are in the same folder
      '
      'use all CAPS here even if not really that way
      Const sheetNamePart = "SCHOOL_DATA"
      Const schoolIDsList = "ABCDE"
      Dim basePath As String
      Dim anySchoolWB As Workbook
      Dim monthID As String
      Dim anyFirstRow As Long
      Dim anyLastRow As Long
      Dim currentBookName As String
      Dim schoolID As String ' will be A, B, C, D or E
      Dim destWSName As String
      Dim lastCol As Long
      Dim sourceCol As Long ' month column in monthly book
      Dim destCol As Long ' month column in this book
      Dim sourceRange As Range
      Dim destRange As Range
      Dim LC As Long  ' loop counter
     
      Dim sourceWS As Worksheet ' source sheet in monthly book
      Dim destWS As Worksheet ' destination worksheet in this book

      basePath = ThisWorkbook.Path & Application.PathSeparator
      'work through all workbooks in this folder
      '"filtering" based on the book names
      currentBookName = Dir$(basePath & "*.xl*")
      Application.ScreenUpdating = False ' for performance improvement
      Do While currentBookName <> ""
        If currentBookName <> ThisWorkbook.Name Then
          'a possible, test 1st character of the name
          schoolID = UCase(Left(currentBookName, 1))
          'is this A...E?
          If InStr(schoolIDsList, schoolID) > 0 Then
            'it is A, B, C, D or E
            'set up reference to the monthly book
            'open it up without updating links, and for READ ONLY
            Application.DisplayAlerts = False
            Set anySchoolWB = Workbooks.Open(basePath & currentBookName, False, True)
            Application.DisplayAlerts = True
            'set reference to the data sheet in the monthly book
            Set sourceWS = anySchoolWB.Worksheets(sheetNamePart)
            'set reference to the data sheet in THIS workbook
            'this is mostly to show how you can create the
            '**********************
            'appropriate worksheet name in this book
            destWSName = Left(currentBookName, 1) & sheetNamePart
            'set reference to the correct sheet
            Set destWS = ThisWorkbook.Worksheets(destWSName)
            '**********************
            'assuming a monthly book's name ALWAYS has:
            ' 1st character is the school ID letter
            ' 2nd through 5th characters are the YYYY digits
            ' so month is in characters 6, 7 and 8
            ' we'll make them all CAPS for easier testing later
            monthID = UCase(Mid(currentBookName, 6, 3))
            'find the proper column on the monthly book's sheet
            'assumed to be in row 1 on the sheet
            'find last column in row 1 that is used
            lastCol = sourceWS.Range("A1").Offset(0, Columns.Count - 1).End(xlToLeft).Column
            sourceCol = 0
            For LC = 0 To lastCol
              If UCase(sourceWS.Range("A1").Offset(0, LC)) = monthID Then
                'sourceCol will now contain column to get data from
                'or zero if no match!
                sourceCol = sourceWS.Range("A1").Offset(0, LC).Column
                Exit For ' have what we need
              End If
            Next
            If sourceCol = 0 Then
              MsgBox "Cannot find the proper month.  Quitting." _
               & vbCrLf & "In " & basePath & currentBookName, vbOKOnly, "Bad Source Month"
              Set destWS = Nothing
              Set sourceWS = Nothing
              Application.DisplayAlerts = False
              anySchoolWB.Close False
              Application.DisplayAlerts = True
              Set anySchoolWB = Nothing
              Exit Sub
            End If
            'find the proper month column on this workbook's data sheet
            lastCol = destWS.Range("A1").Offset(0, Columns.Count - 1).End(xlToLeft).Column
            destCol = 0
            For LC = 0 To lastCol
              If UCase(destWS.Range("A1").Offset(0, LC)) = monthID Then
                'destCol will now contain column to get data from
                'or zero if no match!
                destCol = destWS.Range("A1").Offset(0, LC).Column
                Exit For ' have what we need
              End If
            Next
            If destCol = 0 Then
              MsgBox "Cannot find the proper month.  Quitting." _
               & vbCrLf & "In This workbook.", vbOKOnly, "Bad Destination Month"
              Set destWS = Nothing
              Set sourceWS = Nothing
              Application.DisplayAlerts = False
              anySchoolWB.Close False
              Application.DisplayAlerts = True
              Set anySchoolWB = Nothing
              Exit Sub
            End If
            'all seems ok to this point
            'delete all old information in this workbook/sheet month's column
            destWS.Cells(1, destCol).EntireColumn.ClearContents
            'set up reference to entries in the monthly book column
            Set sourceRange = sourceWS.Range(sourceWS.Cells(1, sourceCol).Address _
             & ":" & sourceWS.Cells(Rows.Count, sourceCol).End(xlUp).Address)
            Set destRange = destWS.Range(sourceRange.Address)
            'this is the data copy
            destRange.Value = sourceRange.Value
            Set sourceWS = Nothing
            Application.DisplayAlerts = False
            anySchoolWB.Close False
            Application.DisplayAlerts = True
            Set anySchoolWB = Nothing
            Set destWS = Nothing
          End If
        End If
        currentBookName = Dir$() ' get next, if any
      Loop ' end of test for currentBookName
      ' we drop out of the loop when all files in the folder
      ' have been examined/evaluated
      Set sourceRange = Nothing
      Set destRange = Nothing
      Set sourceWS = Nothing
      Set destWS = Nothing
      Set anySchoolWB = Nothing
      MsgBox "Job is finished", vbOKOnly, "Task Completed"
    End Sub
