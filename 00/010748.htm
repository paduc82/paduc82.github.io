rob koch say:

Those files named with similarly sized strings of numbers and letters and found in the root folder of the drive with the largest free capacity (typically C:\), are all temporary files created by Windows Update as part of the updating process.

Such files can contain many different update packages for any part of the Windows operating system.  However, they most often contain portions of the Defender or Microsoft Security Essentials definitions or other regularly downloaded components such as the antimalware engine, which is typically downloaded monthly.

When the Windows Update process operates properly, these files are normally deleted after completing the unpacking and installation process.  So if they remain on a system they can indicate that problems exist in the operation of these updates.  However, such issues can happen occasionally even on a properly operating system, typically due to it being turned off or slept in a rush which can cause a transient failure.

If there are no other indications that the updates are failing regularly and the file modified dates are weeks or moths apart or from long ago, I wouldn't be concerned.  You can simply delete these older files which is always safe, since Windows Update will simply retry a failed attempt again at a future time regardless of what was actually being updated.  It's only of a concern if Defender is indicating repeated problems updating or other indications of possible malware are present.

As I recall it may not be possible to check the properties of the files contained in the update package without extracting them (copy/paste) from the package to a local drive, but either way most of the files contained in these should include some details indicating they are produced by Microsoft and in the case of EXE or other executable files, also typically include a digital signature with a valid certificate.

Rob
