Adjust text boxes in Access Report - Microsoft Community

st0rm17 say:

Adjust text boxes in Access Report

I have created an Access Report that serves as a letter.  Since it is a letter, the report is pulling the name, address, city, state, & zip code information from a query.  The letter starts as follows:

To whom it may concern,

[First Name] [Last Name]...

Where the first name/last name is a text box and to the right of it starts the content of the letter via a label.  Because each name varies in length, how can I adjust the text boxes to prevent so much white space between the "first name" and "last name" as well as between "last name" and the standard wording presented in the label?

john w. vinson say:

Rather than a separate textbox for each field comprising the name, I'd suggest basing the Report on a Query concatenating the individual fields into one string. In this case, create a Query based on the table; in a vacant Field cell in the query grid type

FullName: [First Name] & " " & [Last Name]

The & character concatenates strings, so this will put together "Joe", a literal blank, and "Jones" to give a new field FullName containing "Joe Jones". You can then use Fullname as the control source of the textbox.

For the address, it's similar: to get the full address on one line,

[Address] & ", " & [City] & " " & [State] & " " & [Zip]

to generate

123 Maple St., Anytown MP 02121

ken sheridan say:
........how can I adjust the text boxes to prevent so much white space between the "first name" and "last name" as well as between "last name" and the standard wording presented in the label?

1.  If it is all encompassed within one line, or you would want subsequent lines to wrap to below the start of the name otherwise, instead of three separate controls use a single large text box which concatenates the values into a single string expression by means of a ControlSource property like this:

=[First Name] & " " & [Last Name] & " your standard text goes here."

2.  If it can run to more than one line, and you want the standard text aligned en bloc to the right of the name, use a single text box for the name, aligned right, and with a ControlSource property of:

=[First Name] & " " & [Last Name]

Position the label immediately to the right of the text box.  The name will be trimmed from the left, so its position in relation to the standard text won't vary.

For functions to concatenate address lines, either vertically or horizontally, suppressing any Nulls, see Concat..zip in my public databases folder at:

https://skydrive.live.com/?cid=44CC60D7FEA42912&id=44CC60D7FEA42912!169

If you have difficulty opening the link copy its text (NB, not the link location) and paste it into your browser's address bar.
