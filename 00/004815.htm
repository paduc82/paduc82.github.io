jim gordon mvp say:
There is a fundamental difference between the way PowerPoint 2008 and PowerPoint 2011 save movies.

PowerPoint 2008:
All movie files are linked. PowerPoint saves only the file path to the linked movie content. If anything happens that would affect the link to the movie (say move the presentation file to a different folder, flash drive, or another computer) the link to the movie won't work. In 2008 you can avoid this problem by saving a version of your presentation as a PowerPoint Package. Use the File menu and choose Save As. In the Save As dialog click the Format pop-up and choose PowerPoint Package. This makes a new folder containing a copy of your PowerPoint presentation along with copies of all the linked media and generates proper links to the media automatically. As long as you keep the folder's contents intact, the presentation will work if you move the folder from one directory to another, to another machine, or to a disc or flash drive.

PowerPoint 2011:
By default, all movies are embedded, not linked. This is a major improvement over previous versions of PowerPoint . In 2011, when you put a movie (or sound or picture for that matter) into PowerPoint 2011, a copy of the media is retained internally by PowerPoint within the presentation file. There are no links to break, and the media content doesn't have to be in a file on the computer because it is already within the presentation. You can think of the 2011 file as a package in that sense.

schwieb [msft] say:

Hi all,

I see lots of questions about object model parity between Mac and Windows versions of Office. The object model is completely independent from the IDE and thus is not part of this particular effort to revisit the IDE. The object model depends on having the actual features supported in the app, whereas the IDE is a separate feature that allows you to browse, edit, and deploy code that uses the object model that currently exists (and may change over time) in the apps. For example, Mac Excel does not yet have OM support for Pivot Charts because the app does not have an implementation of Pivot Charts to begin with, but if Mac Excel gains the Pivot Chart feature it should also gain Pivot Chart object model support.

Each app team is keenly aware of the feature differences between the Mac and Windows versions. There is a separate effort underway to greatly improve the code convergence between the Mac and Windows apps, and that will make it much easier to achieve feature parity. Due to OS and platform differences, the apps will likely never have precisely matching features and abilities (such as the security restrictions of the Sandbox on the Mac that constrain feature behavior). Stay tuned for separate Insider discussions on app feature and object model convergence!

On a side note, I will be offline for vacation for the next two weeks and will follow up on any IDE-specific questions when I return. To Jim's point above, Forms will continue to run as part of macros as we roll out the new IDE; it's just that the new IDE will not yet support editing existing or creating new Forms when we first release it. We will add that support in the future.


Schwieb

norman david jones say:

Hi Marc,

Hi all, I have read various posts on this subject but still can't get it to work. I have a check box on sheet 1, when checked I would like place the current time in column D and current date in column E on sheet 2

Assuming that the checkbox on Sheet1 is an ActiveX checkbox and is named Checkbox1:

  • Right-click the Sheet1 tab 
  • Select the View Code option from the resultant context menu
  • Paste the following event code
'=========>>
Option Explicit

'--------->>
Private Sub CheckBox1_Change()
    Dim destSH As Worksheet
    Dim rngDate As Range, rngTime As Range

    Set destSH = ThisWorkbook.Sheets("Sheet2")

    With destSH
        Set rngDate = .Range("E2")
        Set rngTime = .Range("D2")
    End With

    If Me.CheckBox1.Value = True Then
        With rngDate
            .Value = Date
            .NumberFormat = "mm/dd/yyyy"
        End With
        With rngTime
            .Value = Now()
            .NumberFormat = "hh:mm"
        End With
    End If
End Sub
'<<=========
  • Alt+Q to close the VBA editor and return to Excel
  • Save the file as a macro-enabled workbook with an xlsm extension

===

Norman

Which is better CSTN Dispay or TFT Dispay ? - Microsoft Community

bmkumar3 say:

Which is better CSTN Dispay or TFT Dispay ?

Can any one tell me which has a got good resolution : CSTN Dispay or TFT Dispay ?

petrib say: Resolution is independent of the kind of display technology that is used.

Resolution is about horizontal pixel count times vertical pixel count (e.g., 176 by 208 or 240 x 320, etc).

Then there are different LCD panel technologies (which can be made in any resolution). Generally they are either "passive matrix" or "active matrix" type.

With passive matrix displays, each pixel is turned on/off with a "duty cycle" during every frame. The liquid crystal needs to be slow in order to avoid flickering and be able to create colour shades.

With active matrix displays each pixel has it‚Äôs own switch (diode or transistor) and the right gray/colour value is driven directly through the switch.

CSTN is the most common form of a passive matrix display and TFT is an active matrix display.

Active matrix displays are faster, and consume more power.

How bright or how they are visible in dark vs. daylight depends on other terms you might wish to look up, too: "reflective", "transmissive" and "transflective" + also "backlight" and "frontlight".

A display characteristic to also keep in mind is the number of colours or the so called colour depth, of the display.

I'm sure your favourite Internet search engine (or Google, if you don't know of any others) can find descriptions (in Wikipedia or elsewhere).
