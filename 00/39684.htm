¡firedog say:
I believe that is the case, but why that message has to be recovered?

I don't know. When a message arrives, it is converted to a file and its headers (e.g. Date:, From:, To: and Subject:) are extracted. These headers, together with some other information including the folder the message should be stored in, are recorded in a log file. Then, in quiet moments, the log file is replayed and the data it contains are written to the message database index, which can be a very big file (mine is a gigabyte in size). Writing to a log file first avoids many problems with the index; each one is pegged at 2MB, and a new one is opened when it reaches that size. Other check files keep track of which parts of which log file have been written to the index file. This is usually transparent to the user.

When the program closes down, any replays in progress are terminated gracefully and the check file updated so that the program knows where to start next time. If this process is interrupted, e.g. by closing down Windows before it's finished or sometimes by aggressive security software, the index is incomplete. On the next program start, this corruption may be noticed and the program will 'recover' messages whose records are incomplete. A recovered message is saved in a storage folder, where it can remain even though the program eventually works out in which folder it should be and puts it there.


Why the message upon arrival cannot be dealt by WLM immediately > put in the proper folder< and be forgotten by WLM?

There might be thousands or even hundreds of thousands of messages to be processed at the same time. If each one meant opening the index file, modifying it and saving it, this would be an intolerable drain on computer resources. Writing to a log file avoids this.

When you're finished dealing with mail, it's best to close down the program and wait a few minutes before turning the computer off. To be sure, you can open Task Manager (Ctrl-Shift-Escape) and wait until wlmail.exe disappears from the Processes list after program shutdown.

Sorry, I don't know of any easy way of removing duplicate messages. If all of the duplicates have quotation marks around the sender's name and none of the non-duplicated messages do, simply sorting on From would group the extra messages for quick deletion.

It might be an idea to sort out your inbox a bit. 4,500 messages is a lot to have in there, and it might improve program performance if you moved old messages out of the inbox and into user-defined folders, either under the account or under Storage folders.
