apollo67 say:

I'm sorry, you may need to be a little more helpful here.  First, let's just talk about one combo box.  Is it bound to an underlying field?  In otherwords, is the property 'Control Source' set to a field name?  Next, how are you using the property 'Row Source?'  Is it set to a query to look up values from either the field you are working with or some other look up table? 

The property 'Limit To List' says that the control will accept values not already stored in the combo box list.  That's all it's doing.  Whether your control is bound or unbound makes no difference.  If your control looks up information from a field that it is bound to, then after you update your record, the new value will become available when the control is requeried. 

What it sounds like to me is that your control is bound to a field.  Forget for a moment that the combo box has a list to display options.  Consider it as a plain text box.  What you are probably doing is replacing the current value of the underlying field for the current record.  That is why you are losing a value after you type the new one in.

Not knowing any more than what you have here, this is the best I can help you at this point.

You were asked to provide the Control and Row Sources for the control. But it sounds like you are using a Value List. To add items to a value list you need to do it programatically. Which is much harder.

Instead since you want to be able to add whatever is types, set the RowSource Type to Tabel/Query and set the RwoSource to:

SELECT DISTINCT fieldname FROM tablename ORDER BY Fieldname;

Where fieldname is the name of the field you are storing the value in and tablename is the name of the table the form is bound to. The combobox will display an alphabetized list of all the values previously entered. If you then set the Limit To List property to No, anything you type will be stored in the field and added to the list.

It is not recommended that you do this, however. A better way is to use a lookup table to populate the list, then use the NotInList event to add new items if necessary. See the links Roger gave you for more on that.

dr.duct-tape say:

Ok, I had this problem for months and no one seemed to have an answer for it (not even Microsoft). Now, we want to blame technology and not own any of the responsibility when it fails to meet our needs/expectations; I am guilty of this too. 

Frustrated with this "unwanted folder" problem, and seeing that Microsoft itself only gave silly instructions of filtering problems, I looked at it from a different angle and fixed it with no problem whatsoever.

First, we need to step back and look at the problem, which is "somehow, legitimate emails are landing in this outlook folder even though they are not on a spam or filter list." Right? Again, I had this same problem. Why? Because in a world of technology, the programmers and code writers can only go so far with their fixes. We, the users, have to own some of the responsibility and realize it might not be a technology problem, but a user problem. 

So here is the answer... 

First, count up your technology devices (tablets, smartphones, laptops, desktop computers, et cetera). Now, most people are having this issue on their Laptops or Desktops computers that are running Outlook, right? But, they forget about the MailApps on their phones and tablets. This problem has nothing to do with Hotmail, Gmail, Yahoo or any other web-based email provider. It has to do with our multitude of personal inputs on various technology devices without having the discipline of making sure nothing is conflicting. 

Easy fix:

Go into the E-Mail app (on the phone or tablet)
Press the Menu Key, select Settings, then General Settings (it may be in different location on different devices).
Select Spam addresses and remove any addresses that should not be in there (i.e. your mom's email address)

That was the problem. I was looking at Outlook as the problem and cursing Microsoft, but my phone had a separate spam/filter list that contained the addresses/domains that kept showing up in the "unwanted folder". Why? Because outlook is set up (generally) using IMAP or PoP3 email accounts. It's the same with the phone or tablet, but the two programs don't communicate. They simply talk to the mail server of whatever email domain you are using. My Outlook was telling the mail server "there are no spam/filtered addresses", but my phone was telling the mail server "*** Email address is removed for privacy ***" was spam. Outlook got confused, and simply dumped the emails into the "unwanted folder" because it didn't know what to do with them. 

So, my advice... go into all of your portable technology devices and check the spam lists or filter lists. I bet the addresses somehow ended up on one of those lists in phone/tablet.
