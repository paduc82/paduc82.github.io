robert aldwinckle on forums say:

Have you had any crashes of iexplore.exe lately?  Look in the Reliability Monitor for that symptom.  Then if you find one there and it is called "Application crash" there should be a Report.WER file somewhere about it.  It will be hidden so you will need to allow hidden files to be found.  Unfortunately you have to do it this way because that is the only way to see the list of Loaded modules which are associated with an APPCrash.

Here's a PowerShell pipeline which will find some which should satisfy that criteria, starting from the root of the Systemdrive.

PS>LS -Re -Fi "*.WER" -Fo -Ea "SilentlyContinue" | where {$_.FullName -match 'iexplore'} | where {(gc $_.FullName).count -GT '150'} | sort LastWriteTime | ft FullName

E.g. apparently mine are all under  C:\ProgramData and C:\Users\AllUsers

Another thing to note is that they are all doublebyte characters (Unicode), so be sure to set Notepad's Encoding accordingly before you try to open one (if you want it to read like text).

 

Good luck

Robert

In both 2011 and 2016, the equation editor is hard-wired to use the Cambria Math font, since most other fonts lack the required characters. So, neither version will use your font theme in the math sections.

I'm not able to duplicate your results. In both 2011 and 2016, the document re-opens with normal text still appearing as normal. However, there have been numerous font issues with 2016, like using italic for normal.
