dirk goldgar say:

First, let me say that your tblPeople ought to have a primary key.  Every table ought to have a primary key, for best operation.

As for your question ...

Although most database developer would advise using forms and subforms to display data, if you choose to open queries in datasheet directly, Access allows you to set a subdatasheet for a queries, and specify the LinkMaster and LinkChild fields for that subdatasheet, just like a subform.  So to accomplish what you want you would create a query of just your tblMain -- do not include tblPeople in the query.  With that query open in design view, open the property sheet for the query (not the property sheet for any particular field in the query).  On that property sheet, you'll see properties "Subdatasheet Name", "Link Child Fields", and "Link Master Fields".  Set the Subdatasheet Name property to tblPeople, and set the Link Child and Link Master Fields properties to EventIdNumber.

When you open the query, you should see the "+" signs that you can expand to see the related tblPeople records,

Re 1: You can pick up any PatientDetails (lose the all caps for readability, and the spaces and other funny characters for notational difficulties) in a query; they do not belong in related tables per relational database design rules.

tblPatientDetails
PatientDetailsID autonumber PK
PatientFN text(20) required
'etc.

tblAssessment
PatientDetailsID long int required PK
DateTaken datetime required
IsAlive Boolean
'etc.

tblBaseline
PatientDetailsID long int required PK
DateTaken datetime required

Hearthrate integer

'etc.

tblFollowup
FollowupID autonumber PK
PatientDetailsID long int required FK

DateTaken datetime required
IsAlive Boolean

'etc.

Then go to the Relationships window and drag-n-drop the lines between the corresponding fields. This will create the one-to-one and one-to-many relations. Make sure you check the "Enforce" box on the popup dialog.

Re #2:
This would be an empty report with two subreports, one on tblBaseline and another on tblFollowups.

The query for the subreport would be something like:

select * from tblBaseline where DateTaken between Forms!myCriteriaForm!myStartDateControl and Forms!myCriteriaForm!myEndDateControl

So this query "looks back" on the report criteria form to get the date range to use.

Highly recommended to repeat this date range in the header of the report.

Re #3: Date intervals are calculated using the DateDiff function.
