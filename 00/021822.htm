If the region to be sorted is bounded by blank rows and columns or the edge of the worksheet and contains no blank rows or columns within it then

  .SetRange Range("A1").CurrentRegion

Sub Macro1()

don cuthbert say:

It should have auto-activated since your Build 10130 was activated (or you wouldn't have gotten the Windows Update upgrade to Build 10240).  Sometimes it just takes some time to activate when it was done properly, but you cannot activate using a Product Key now.

Since you have Build 10240, try creating a recovery drive USB flash drive first (run recoverydrive.exe).  You need a 4GB drive to get a full Windows system image that can be reinstalled, but until you have activated Build 10240 by upgrading, a reinstall will not activate either.

Since your laptop came with Windows 8.1, the Product Key is probably saved in the BIOS and will be detected if you reinstall Windows 8.1.  You also could try to see if you can read it using ProduKey from Nirsoft.  ProduKey will read the Product Key of both Windows and Internet Explorer.  Sometimes Internet Explorer shows the previous OS Product Key, so you could write that down if it is still there.

You could try installing Windows 8.1 to a separate partition and see if it detects your Product Key and activates OK.

Some of the last insider Preview Builds do seem to auto-activate, if installed as an upgrade to an activated qualifying OS.  You could run Build 10130 setup.exe from within Windows 8.1 to try upgrading it.  During that upgrade, do not enter any Product Key when asked (since Product Key activation doesn't work now).  If Build 10130 auto-activates after you connect to the Internet, you can try upgrading again via Windows Update.

It is also possible to create your own Build 10240 ISO during Windows Update before it restarts your computer.  There is info in other threads on this.

If you do get Build 10240 activated you can clean install it any time afterwards, using the recovery drive or Reset option, but only on that computer.
