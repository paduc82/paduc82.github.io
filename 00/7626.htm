walt2223 say:

one vlookup on multiple sheets

I have seven worksheets, eached named after a grade (K, 1st, 2nd, etc.). 

On each worksheet of these seven worksheets:

     Column A - Name

     Column B - Y or N

 

On an eighth worksheet (Current Year), I have a name of current and past students (A1:A500).  I would like a formula in the Current Year worksheet, in Column B, to search for the name within the seven worksheets and indicate if there is a Y or N beside the name. 

chip pearson say:

Here's a formula that will do the job. Change each of the seven sheet names in the formula to your seven worksheets' name. If a sheet names contains a space, enclose the sheet name within apostrophes.

Change the range references $A$1:$B$5 to the range on each sheet that should be searched for a match to A1 on the master sheet.Use the $ characters as shown.  Change "NOT FOUND" to whatever text you want to return if no match is found in any of the worksheets.

 

The formula will search each sheet column A for a match to cell A1 on the master sheet, which contains the formula. If found, the value in column B of that row will be returned.

 

=IFERROR(VLOOKUP(A1,Sheet2!$A$1:$B$5,2,FALSE),IFERROR(VLOOKUP(A1,Sheet3!$A$1:$B$5,2,FALSE),IFERROR(VLOOKUP(A1,Sheet4!$A$1:$B$5,2,FALSE),IFERROR(VLOOKUP(A1,Sheet5!$A$1:$B$5,2,FALSE),IFERROR(VLOOKUP(A1,Sheet6!$A$1:$B$5,2,FALSE),IFERROR(VLOOKUP(A1,Sheet7!$A$1:$B$5,2,FALSE),"NOT FOUND"))))))

 

Note that the IFERROR function is new to Excel 2007 and is not available in eariler versions.

Computer\HKEY_USERS\…...\Software\Microsoft\Windows\CurrentVersion\Uninstall\

HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\Windows\CurrentVersion\Uninstall

oneclient.sfx.ms

I also had the question once about IE8, then I did some search and found the solution.

Here it is:

go registry and find the following

Computer->HKEY_CURRENT_USER->Software->Microsoft->Internet Explorer->Main

in the right pane, creat a new DWORD entry and name it as "TabProcGrowth" and value as "1".

Log off and log on in case it doesn't take effect at once.

This actually make IE9 opens only 2 processes (one for main window, one for all the tabs in it ) instead of bunch of them even more than the tabs it opens, and in fact, though the tabs appear in one main window, behind the window it may has several processes, each may handle one or more tabs and has its own session, cause you have to specify the save location for each of them. Using the registry tweak above to make IE use single process for all tabs should solve this.

Plus, you can set the value to 0, force IE9 use only one process as in IE7 and earlier. but doing this may cause IE more fragile if some stability issue occurs.

However you will need to specify the save location once again if you close and restarted the mian IE window/ whole application, and if the default is not where you want.
