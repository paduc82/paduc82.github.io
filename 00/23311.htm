jay freedman say:
Let me explain for the simplest case, a userform with two checkboxes and one command button. Their default names are CheckBox1, CheckBox2, and CommandButton1, although it's best to rename them to something that reflects their purpose. Let's say the template has all the text, and there are two bookmarks surrounding the two sections, named Bookmark1 and Bookmark2. Each bookmark will be left as is when the corresponding checkbox is checked, or deleted if it's cleared.

After you've inserted the checkboxes and button on the surface of the userform, double-click the command button. That will open the userform's code window and automatically insert an empty subroutine named Private Sub CommandButton1_Click(). When the userform is executing, clicking the command button causes this subroutine to run. You will write code within this subroutine to do everything you want to happen when the button is clicked. That is, if CheckBox1 is not checked then delete the range of Bookmark1; if CheckBox2 is not checked then delete the range of Bookmark2;and close the userform so the document is visible.

The code for this would be:

Private Sub CommandButton1_Click()
    If CheckBox1.Value = False Then
        ActiveDocument.Bookmarks("Bookmark1").Range.Delete
    End If
   
    If CheckBox2.Value = False Then
        ActiveDocument.Bookmarks("Bookmark2").Range.Delete
    End If
   
    Me.Hide
End Sub

You would also need an AutoNew() subroutine in a standard module in the same template to cause the userform to pop up when a user creates a new document based on the template:

Sub AutoNew()
    Dim dlg As UserForm1
    Set dlg = New UserForm1
    dlg.Show
    Set dlg = Nothing
End Sub

If you're using the building block method, the blocks of code in the CommandButton1_Click() routine would be different:

Private Sub CommandButton1_Click()
    Dim tmpl As Template
    Set tmpl = ActiveDocument.AttachedTemplate
    Templates.LoadBuildingBlocks
   
    Dim rg As Range
    Set rg = ActiveDocument.Range
   
    If CheckBox1.Value = True Then
        Set rg = tmpl.BuildingBlockEntries("Block1").Insert( _
            where:=rg, RichText:=True)
    End If
   
    Set rg = ActiveDocument.Range
    rg.Collapse wdCollapseEnd
   
    If CheckBox2.Value = True Then
        Set rg = tmpl.BuildingBlockEntries("Block2").Insert( _
            where:=rg, RichText:=True)
    End If
   
    Me.Hide
End Sub

jay freedman say:
There are two basic (pun intended) approaches to this task. One is to insert predefined pieces (stored as building blocks in the template) into a blank document, and the other is to start with all of the pieces in the template and propagated to the document (each surrounded by a bookmark) and delete the unwanted ones. Either scheme will work.

I wouldn't recommend using separate documents to hold the sections, especially if the setup needs to be installed on multiple computers. That variation requires the template and all the section documents to be stored in the same folder (a bad idea, since the template must be in a trusted location in order to allow the macros to run) or for the section documents to be stored in a specific, fixed path (hard to enforce on other users' computers).

The interface for selecting the pieces for the final document should be a userform (read this page) in which checkboxes or other controls can be set or cleared, and the VBA code behind the userform does the assembly or deletions.

Updating to Word 2013 is not necessary for this job. Either approach can be built in either 2010 or 2013 and will be fully compatible with the other, and with 2007. If you use the assembly method and it needs to be compatible with Word 2003 as well, then you must use the AutoText gallery rather than any other building block type.

=SUMPRODUCT(--(WEEKDAY(ROW(INDIRECT(A1&":"&A2)),2)=6))

where A1 holds the starting date and A2 holds the ending date of a period (doesn't have to be those cells, just cells that have the start/end dates) that counts the Saturdays within those dates.  Take that and modify it just a little:

=(SUMPRODUCT(--(WEEKDAY(ROW(INDIRECT(A1&":"&A2)),2)=6))/2)

and you have Saturdays counted as 1/2 days.
