jay freedman say:

At http://www.gmayor.com/downloads.htm#Third_party you'll find a link to download DateCalc.zip, a tutorial on date/time field formulas in Word for almost anything you can think of. It's the work of fellow MVP Paul Edstein (a.k.a. macropod).

You can start with the example in "Date and Time Calculations in a Table", about halfway through the document. Applying the same principle to time calculations isn't simple but can be done.

If the end time of the previous row is bookmarked as "prev", the first column could contain the field

{ QUOTE { ask d "Duration in hours:minutes" { d } } }

and the second column, the start time (assumed to be the same as the previous row's end time), would be

{ QUOTE { prev } }

Then the third column, the end time of the current row, could be calculated as

{ QUOTE { SET ph{ = { prev \@ H } } } { SET pm{ = { prev \@m } } } { SET dh{ = { d \@ H } } } { SET dm{ = { d \@ m } } } { SET eh{ = ph + dh + INT((pm + dm)/60) } } { SET em{ = pm + dm } } { IF em > 60 { set em{ = em - 60 } } } "{ eh }:{ em }" }

The hard part is that Word allows only one bookmark with a given name per document. You'd need some macro support to convert the result to plain text (unlink the fields) and then recreate the prev bookmark and put the fields into the next empty row.

"Elevation" refers "Windows Access Rights" by which Windows controls what you can do on the file, specifically what folders you can read and write to.  To do installations in Windows 7 you need "elevated" rights. You get them by using "Run As Administrator".

Auto number with new employee but not affect previous numbers - Microsoft Community

canadian-cash say:

Auto number with new employee but not affect previous numbers

I have a field called EmployeeID. This field was long in existence prior to me taking over this database. I would like this field to become an auto number the next time they enter a new Employee. However, I don't believe this is possible. It is not an option for me add a new field for the autonumber and then delete out the EmployeeID since the EmployeeID is used in other capacities. Am I simply out of luck here? Seems so simple in Excel yet so complicated in Access.
hansv mvp say:

You can't change an existing field to an AutoNumber field, but you could do the following, although it's a lot of work:

  • Make a backup copy of the database, to be on the safe side.
  • Select the table in the navigation pane. Press Ctrl+C, then Ctrl+V to copy and paste it.
  • Access will display the Paste Table As dialog. Select the option to paste the structure only, then click OK.
  • Open the copy in design view.
  • Delete the EmployeeID field. Access will warn you that it has to remove the primary key. This is OK.
  • Create a new field. Name it EmployeeID, set its Data Type to AutoNumber and make it the primary key.
  • Save and close the table.
  • On the Create tab of the ribbon, click Query Design.
  • Add the original table, then close the Show Table dialog.
  • Add * to the query grid.
  • In the Query Type group of the Design tab of the ribbon, click Append.
  • Specify the new copy of the table as target for the append query.
  • Click the Run button (the red exclamation mark).
  • Confirm that you want to add nnn records.
  • Close and discard the query.

    ken sheridan say:

    It's very simple.  You can do it by creating a new table via an append query:

     

    1.  Back up the database.

     

    2.  Open the relationships window and make a note of all the relationships which the table is a part of, noting the columns on which the table is related to others, and whether referential identity is enforced, which it should be as an unenforced relationship does nothing useful.  If a relationship can't be enforced then there is something wrong with the data.

     

    3.  Copy and paste the table in the navigation pane, electing to copy 'structure only'

     

    4.  Open the new table in design view and change the EmployeeID column to an autonumber.

     

    5.   Create and run a simple 'append' query to insert the rows from the original into the copy, e.g.

     

    INSERT INTO [Copy of Employees]

    SELECT *

    FROM [Employees];

     

    6.  Make sure the copy has been correctly filled, in which case you can delete the original Employees table and then rename the copy back to the name of the original.

     

    7.  Recreate all the relationships to which the original table was a party.

    WANT Formatting Marks Visible on Printed Word 2007 Document - Microsoft Community

    toobusy98 say:

    WANT Formatting Marks Visible on Printed Word 2007 Document

    I want to print a Word 2007 document WITH the Formatting Marks VISIBLE.

     

    Every week, my co-workers spend so much time trying to make their weekly report "look" a certain way, and I've tried to tell them and show them while standing over their shoulder, but it's just not sinking in.

     

    I want to print out their reports with the Formatting Marks visible so that they can see what's "inside" the different formatting options.

     

    Is there a way to print a document with the Formatting Marks visible?

    jeeped say:
    Unfortunately, there is not provision for printing a Word document while showing the typically displayed using Ctrl+Shift+8. Since this is to be used as a hard copy example sheet, your best bet would likely be to simply tiurn on the formatting marks and use an application screen shot (Alt+PrntScrn) to take a snapshot that can be pasted into MS Paint (or your favorite image program) and printed.

    However, this might be tedious for multiple pages. You can use a couple of quick Find and Replace commands to replace the majority of the formatting marks with actual symbol representations.

    Tap Ctrl+H to open the Find and Replace dialog,
    Find what: ^p    Replace with: ¶^p    Replace All
    Find what: ^l    Replace with: ↵^l    Replace All
    Find what: ^t    Replace with: →^t    Replace All

    Two of those of those symbols are available with simple Alt codes (Alt+0182 and Alt+26) The small bullet used to note a space is Alt+0183. All four are available within the Insert, Symbol command. That takes care of the prevalent formatting markup symbols.

    Here is a macro that will accomplish the same thing,

    Sub mcr_Printable_Formatting_Marks()
      ' mcr_Printable_Formatting_Marks Macro
      ' A sequence of Find & Replace operations to create printable formatting marks.
        Dim v As Long, vClears As Variant, vFinds As Variant, vReplaces As Variant
        vClears = Array(Chr(182), ChrW(8629), ChrW(8594), Chr(183))
        vFinds = Array("^p", "^l", Chr(32), "^t")
        vReplaces = Array(Chr(182) & "^p", ChrW(8629) & "^l", ChrW(8198) & Chr(183) & ChrW(8198), ChrW(8594) & "^t")
        With Selection.Find
            For v = LBound(vClears) To UBound(vClears)
                .ClearFormatting
                .Replacement.ClearFormatting
                .Text = vClears(v)
                .Replacement.Text = vbNullString
                .Wrap = wdFindContinue
                .Execute Replace:=wdReplaceAll
            Next v
            For v = LBound(vFinds) To UBound(vFinds)
                .ClearFormatting
                .Replacement.ClearFormatting
                .Text = vFinds(v)
                .Replacement.Text = vReplaces(v)
                .Wrap = wdFindContinue
                .Execute Replace:=wdReplaceAll
            Next v
        End With
    End Sub

    After running that macro, toggle the display of the formatting marks off with Ctrl+Shift+8 and you should have a reasonable representation of the document with formatting marks suitable for printing.

    Note¹: that this macro is a bit of a one way trip. The replacement of conventional spaces with ¹⁄₆ em-spaces (i.e. Unicode U+2006 or CharW(8198)) and bullets doesn't allow much room for backtracking. Use on a copy of the original document.
    Note²: the array of vFinds and vReplaces must have an identical, number of entries. The vClears can be any number of things you want removed prior to the Find & Replace.
    Note³: I couldn't find any easy way to show the formatting of a bulleted or numbered list as this is not a standard tab character. A special. style of list may be necessary for this.
