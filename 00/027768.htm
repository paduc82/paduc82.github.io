fermon say:

As the previous posts hint, this problem is caused by a broken reference. The problem here is that Excel 2010 does not explicitly show it and simply crashes.

 

The crash is so bad, that the file cannot be saved in Excel 2010 format (.xlsm) or in the native .xls (97-2003 format).

 

As Sunnyside reports, the problem can be linked to the Progress Bar control, although this is not the only one that resides in the broken reference. These are the steps to fix the problem:

 

1. Open the .xls file in question and click OK after getting the error mentioned above.

2. Open the Visual Basic Editor (Alt+F11)

3. Open (if they are not already open) one of the modules or code behind forms

4. Click on Tools / References...

5. In the pop-up window click on Browse...

6. Navigate to [Your Windows Directory]\System32 (it may open by default)

7. Type MSCOMCTL.OCX (you'll see it in the auto-select options before you finish typing)

8. Click Open and the reference will appear as Microsoft Windows Common Controls 6.0 (SP6)

9. Click on Debug\Compile VBAProject to verify that the problem is solved

10. Save the file.

 

As mentioned above, this could happen for other controls as well, but the progress bar seems to be the most popular.

Hi Vít Zakouřil,

 

Thanks for posting back in our forum.

My understanding is that you can access your OneDrive for Business in browser. But you still get this error message when you tried to sign in and set up OneDrive sync client.

 

If so, we would like to suggest you try the following suggestions to see the outcome.

 

1. Please check the following first:

 a. Press (Windows key) + R to open the Run box.

 b. Type gpedit.msc and click OK.

 c. In the Local Group Policy Editor, in the folder list under Local Computer Policy, navigate the folders to Local Computer  Policy > Computer Configuration > Administrative Templates > Windows Components > OneDrive.

 d. On the OneDrive screen, under Setting, click Prevent the usage of OneDrive for file storage.

 e. In the Prevent the usage of OneDrive for file storage box, make sure it is set to Disabled.

 

This issue may also because your admin only allows syncing from PC's that joined certain domains. please contact your Office 365 global admin, and check with them if they have disabled “Allow syncing only on PCs joined to specific domains". For this, ask them to go to Office 365 Admin center>Admin Center>OneDrive>Sync> Uncheck "Allow syncing only on PCs joined to specific domains". check if option Block sync of specific file type is unselected.

 

Moreover, check if you are using the new OneDrive sync client, if no, use it first to see the result.

 

If the issue persists, please provide the following information:

1.Does your root OneDrive folder name contain space?

2.Please check if the folder name contains any unsupported character such as <, >, :, ", |, ?, *, /, \

3. From where did you install OneDrive sync client?

 

Let us know if you need additional assistance.
