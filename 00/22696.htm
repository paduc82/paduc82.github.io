bill manville say:

Excel's method of saving (designed to avoid loss of files in the case of a failure during save) is:

- saves to a temporary file (with a random 8-character hexadecimal name)

- deletes the original file

- renames the temporary file with the original file's name

 

If the temporary files are remaining, it would seem that the last step is not happening and the save will not actually be complete.  The most likely reason is that a virus checker has got in the way of Excel's operation and has the file open when Excel tries to rename it.

 

Most virus checkers have option settings that allow for the checking of Office documents on save to be disabled.  I recommend applying that setting.
