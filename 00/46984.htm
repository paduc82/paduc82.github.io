john john __ say:

You say that you have "...a large amount of data on this volume and that idea of RAID 1 is that this isn't lost".  Please note that RAID-1 is a disk redundancy solution meant for mission critical servers and workstations and that it is a poor backup solution.

To remove the failing disk use the Disk Management tool to break the mirror:

1- Select (right-click) the failing disk and remove it from the mirror.

2- Physically replace the failing disk.

3- Use the Disk Management tool to Initialize the new disk and convert it to a Dynamic Disk.

4- Add the new disk to the mirror.

 

John

zpostbox say:

We found the problem - it's incorrect configuration of the routing.

To fix it, I did this:

1. Disable the checkbox for "Use Default Gateway on remote network" option in "Advanced" tab of "Internet Protocol version 4" properties of the MTS adapter (it's the name of the connection).

2. Add IP address 192.168.1.1 and subnet mask 255.255.255.0 to the "Internet Protocol version 4" properties of the Ethernet adapter.

3. Connect to the ISP.

4. Use the command 'route print -4' to see network routing tables. Remove a suspicion route, in my case it was

'route deleted 83.0.0.0'

5.Use the command

route add 0.0.0.0 mask 0.0.0.0 XXX.XXX.XXX.XXX

where XXX.XXX.XXX.XXX is my IP address.

I have to do it every time I go online, that's the problem.

Open up regedit and have a look in:

HKEY_CURRENT_USER\SOFTWARE\Microsoft\Windows\CurrentVersion\PenWorkspace\Notes

There should be a DWORD value named "LaunchOnNextUserSession". Change that from 1 to 0 and try logging off and on again.
