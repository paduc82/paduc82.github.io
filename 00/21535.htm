brian tillman [outlook mvp 2007-2019] say:

This is a sign that your autocompletion cache isn't being written properly so that when you close Outlook, the hidden folder that contains the suggested names hasn't been updated with the most current data.  You could try a new mail profile or you could try erasing the cache and starting over.  For the former, see http://www.howto-outlook.com/faq/newprofile.htm.  For the latter (which is what I'd try first), press WinKey+R or click Start>All Programs>Accessories>Run.  In the Open field that appears, type

outlook.exe /cleanautocompletecache

Be sure to include the space.  Press Enter.  This will empty the suggested name cache completely and start it fresh.  You'll have to allow it to build up again.  Remember, it's not tied to your Contacts folder in any way.  It's just a list of the addresses to which you've addressed messages.

If the problem still occurs, you'll have to find out why Outlook isn't closing properly.  Some ideas to help with that can be found here: http://www.howto-outlook.com/faq/outlookdoesntclose.htm

russ valentine say:

Failure to exit properly was only a cause for this problem in previous versions of Outlook. It is rarely if ever a cause for Outlook 2010.

The first thing to try is simply clearing the autocompletion cache in File > Options > Mail > Send Messages > Empty Autocompletion List. Then restart Outlook.

If that doesn't work, Microsoft Office Help files recommend closing Outlook and renaming your RoamCache folder:
 <drive:>:\Documents and Settings\<UserName>\Local Settings\Application Data\Microsoft\Outlook\RoamCache

adb014 say:

Ok, I think I've found the best way of treating this issue. Basically, you ignore the Outlook Object Model, and reencrypt into an SMIME message with a "Microsoft Cryptographic Provider" using only the users exchange certificate and import the SMIME message with IConverter:Mime2Mapi. An example of what I did in python is

hCryptoProv = win32crypt.CryptAcquireContext(None, win32cryptcon.MS_ENH_RSA_AES_PROV, win32cryptcon.PROV_RSA_AES, win32cryptcon.CRYPT_SILENT)

hstorehandle = win32crypt.CertOpenSystemStore("MY", hCryptoProv)

for cert in hstorehandle.CertEnumCertificatesInStore():
    try:
        (certtype, dummy_privcert) = cert.CryptAcquireCertificatePrivateKey(win32cryptcon.CRYPT_ACQUIRE_SILENT_FLAG)
        if certtype == win32cryptcon.AT_KEYEXCHANGE:
            # Ok we have the users key as we can access both the public and private keys and the key is flagged
            # for use with Exchange
            certificate = cert
            break
    except OSError:
        pass

encodingtype = win32cryptcon.PKCS_7_ASN_ENCODING | win32cryptcon.X509_ASN_ENCODING

encryptalgorithm = {"ObjId" : "2.16.840.1.101.3.4.1.42", "Parameters" : None}  # AES256

encryptparams = {"MsgEncodingType" : encodingtype, "CryptProv" :
