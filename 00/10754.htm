dhookom say: I never use an alias created as a column in an expression in any other column. T_Date, Yr, Mo, Dy, Comp, ValComp, and ValCompDate are all created in the query and in my experience shouldn't be used in other expressions since it sometimes leads to errors.

SELECT tblITPs.Name_, 
   Year([T_Date]) AS Yr,
   Format$(Month([T_Date]),"00") AS Mo,
   Format$(Day([T_Date]),"00") AS Dy,
   [Yr] & [Mo] & [Dy] AS [Comp], 
   tblITPs.CompDate, 
   Val([Comp]) AS ValComp, 
   Val([CompDate]) AS ValCompDate, 
   [ValCompDate]>[ValComp] AS Compare,
   Date()-7 AS T_Date
FROM tblITPs;

Can you provide the logic of what you are attempting to accomplish? It seems like you are converting dates to strings then to numbers. Can we assume CompDate is a date field? Do you just want to compare CompDate with the current date - 7 days? 

ken sheridan say: I assume that CompDate is not a column of date/time data type as, if it were, VAL(Compdate) would return the value of the first element of the date in the system's short date format, i.e. the month in the USA, the day in the UK.  As you are apparently trying to compare it with a value in the format yyyymmdd, I can only assume that it is itself a string expression in that format.

It appears that you are trying to return rows where CompDate is later than the date 7 days prior to the current date, so if CompDate  is a string expression in the format yyyymmdd, then why not simply use:

SELECT *
FROM tblITPs
WHERE CDATE(FORMAT(CompDate,"0000\-00\-00")) > DATE()-7;

If CompDate were a column of date/time data type, as would normally be expected, then it would simply be:

SELECT *
FROM tblITPs
WHERE CompDate > DATE()-7;

ossiemac say:

Two examples of VBA Code.

First example to save to an existing folder which is a sub folder of the existing workbook.

Second example to create a new folder as a sub folder of the existing workbook.

Sub Macro1()
    'This code copies to an existing path
    Dim strPath As String
    Dim strNewFile As String
   
    'Following is to a folder named B as a subfolder of ThisWorkbook
    'The folder B must already exist with the following code
    strPath = ThisWorkbook.Path & "\B\"  'Create a string with the required path
   
    strNewFile = ThisWorkbook.Worksheets("A").Range("B2").Value
   
    'Following Copies the worksheet to a new workbook. (New book becomes the ActiveBook)
    Sheets("A").Copy
   
    ActiveWorkbook.SaveAs Filename:=strPath & strNewFile, FileFormat:= _
                    xlOpenXMLWorkbook, CreateBackup:=False
   
End Sub

Sub Macro2()
    'This code creates a new folder and saves to that folder
    Dim strPath As String
    Dim strNewFile As String
   
    'Following creates a folder named B as a subfolder of ThisWorkbook path
    MkDir ThisWorkbook.Path & "\B"
   
    strPath = ThisWorkbook.Path & "\B\"  'Create a string with the required path
   
    strNewFile = ThisWorkbook.Worksheets("A").Range("B2").Value
   
    'Following Copies the worksheet to a new workbook. (New book becomes the ActiveBook)
    Sheets("A").Copy
   
    ActiveWorkbook.SaveAs Filename:=strPath & strNewFile, FileFormat:= _
                    xlOpenXMLWorkbook, CreateBackup:=False
   
End Sub

stefan blom say:

The master document feature is buggy in Word; most if not all experienced users avoid it. For more, see http://word.mvps.org/FAQs/General/RecoverMasterDocs.htm ("How to recover a Master Document").

Making use of a single document would be the easiest solution, by far.

However, you can use INCLUDETEXT fields to include separate files into a "master." See:

How to Use IncludeText Fields
http://word.mvps.org/faqs/tblsfldsfms/includetextfields.htm

Note that in recent versions of Word, to display the Insert File dialog box, you must click the arrow next to Object on the Insert tab, and then click Text from File. (The command is misleading, as not only the text but all contents of the referenced file will be included.)

Windows 10 Build 10041 Windows Update Error 0x80200068

Getting a weird WU error. Here's a snapshot.

Some Background Information:

- Running Windows 10 Pro Build 10041 on a Surface Pro 3

- I recently installed Visual Studio 2013 Update 4, which is why these errors are coming in the first place.

Main Question:

What is the meaning of error code 0x80200068? How can I resolve this?

john p fernandes say:

I managed to fix this issue by digging into this further. This was being caused by battery saver mode being turned on.

Looking at the BITS Jobs I was able to find the following.

ERROR CODE:    0x80200068 - Battery Saver mode is preventing the job from transferring. The transfer will resume when the system exits Battery Saver mode or reconnects to A/C power. If this job represents foreground work that should proceed even in Battery Saver mode, use priority BG_JOB_PRIORITY_FOREGROUND.

Command: bitsadmin /list /allusers /verbose

The battery on my laptop is dead and I keep it plugged in (like a desktop). This caused the Saver to be greyed out at on and hence I could not turn it off.

So I went into device manager and disabled my battery altogether. WU works like a charm now.

Hope this helps you guys too.
