The ¶ and § symbols are in the Latin-1 Supplement subset (immediately after the default Latin Basic subset) of almost all "body text" fonts; mainly it's the "symbol" fonts such as Symbol and Wingdings that don't have them. There's no need to go to MS Gothic.

Suzanne's comment about expanding by dragging refers to the More Symbols dialog, not to the ribbon menu/gallery.

While you can add symbols to the symbol gallery on a recently-used basis but not permanently, you could create a macro for each symbol, add a custom group to the Insert tab of the ribbon next to the existing Symbols group, and add a button to it for each macro. For the paragraph and section symbols, the macros would be

Sub InsertParagraphMark()
    Selection.Text = Chr(182)
    Selection.MoveRight wdCharacter
End Sub

Sub InsertSectionMark()
    Selection.Text = Chr(167)
    Selection.MoveRight wdCharacter
End Sub

For distribution across the firm, the macros would be stored in a separate template, pushed by a login script to the %appdata%\Microsoft\Word\Startup folder of each user (http://www.word.mvps.org/FAQs/MacrosVBA/DistributeMacros.htm). The custom group and its buttons would have to be exported by the integrator to a *.exportedUI file, and the login script would have to place it where it could be imported to each user's profile. Read http://office.blogs.webucator.com/2010/07/02/import-export-ribbons-in-office-2010/ for instructions.

ajar say:

Best Way To Keep Track Of A Bank Account Balance?

Hi

I am a total Noob to Access (have 2010) and have chosen to develop a budget database as a learning experience.

My Plan is to have tables for Accounts, Debit Categories (i.e. Transportation, Rent), Credit Categories (i.e. Interest Income, Consulting Fees) and Transactions.  The Transaction table will have  fields for Account (Lookup from the Accounts Table), Amount (Dollar Value of each transaction (Deposits would be positive numbers and Debits would be negative numbers), Debit Category (Lookup from the Debit Categories Table),  and Credit Category (Lookup from the Credit Categories Table).

One of the things I would like to accomplish with this is to be able to keep track of the balance of each bank account and review transactions for different periods (10 Days, One Month etc.)

ken sheridan say: You sound as though you are by and large on the right track.

I've maintained my personal accounts in Access for a couple of years now, having previously used Excel.  The following is a query used as the RowSource of a form for entering transactions into one of my current accounts (checking accounts in US-speak).

SELECT Format(DSum("Credit-Debit","CurrentAccounts","AccountNumber = ""12345678""
And TransactionDate<= #" & Format([TransactionDate],"yyyy-mm-dd") & "#
And (TransactionID <= " & TransactionID & " Or TransactionDate <> #" &
Format([TransactionDate],"yyyy-mm-dd") & "#)"),"£#,##0.00;(£#,##0.00)") AS Balance, *
FROM CurrentAccounts
WHERE AccountNumber="12345678"
ORDER BY TransactionDate, TransactionID;

A few points to note:

1.  The table has separate Credit and Debit columns into each of which values are entered as positive numbers.  Each of the two column's Required property is True (Yes) to disallow Nulls and each has a DefaultValue property of 0.

2.  The DSum function is used to compute the balances per transaction.  Domain aggregate functions like this are little slow, and a JOIN would be the most efficient way of computing the balances, but this, or the use of a subquery, would make the query non-updatable and unsuitable for a data entry form.  Using the DSum function on the other hand keeps the query updatable.  I've found performance more than satisfactory.

3.  Note how in the criteria expression for the DSum function the primary key TransactionID column is used as a tie breaker where more than one transaction has the same date.

4.  Because the balances are computed on the basis of all rows in the table with the relevant account number, if any restriction of the query by date is applied, or similar filtering of the form, the balances would still be correctly computed.

As an initial troubleshooting step, I suggest you to place your system in a Clean Boot state and check if the same issue occurs.

Placing your system in Clean Boot state helps in identifying if any third party applications or startup items are causing the issue. Check this issue in clean boot state.

  1. Press Windows key + X key. And select Run.
  2. Type msconfig in the Run box and hit Enter.
  3. On the Services tab of the System Configuration dialog box, tap or click to select the Hide all Microsoft services check box, and then tap or click Disable all.
  4. On the Startup tab of the System Configuration dialog box, tap or click Open Task Manager.
  5. On the Startup tab in Task Manager, for each startup item, select the item and then click Disable.
  6. Close Task Manager.
  7. On the Startup tab of the System Configuration dialog box, tap or click OK, and then restart the computer.

How to perform a clean boot in Windows

http://support.microsoft.com/kb/929135

Note: After troubleshooting, refer to this section "How to reset the computer to start normally after clean boot troubleshooting".

The issue may also arise due to the security software installed in your system. So, I would suggest you to temporarily disable the anti-virus installed on your system and check if the issue persists.

If you're connected to the Internet or a network while your antivirus software is disabled, your computer is vulnerable to attacks.

Windows Update helps keep your computer up to date with the latest system updates and drivers which can resolve the issue you are facing. You can check Windows Update manually to help ensure you have the latest updates.
