Rob,

Yes, for reasons not openly published by Microsoft, the SharePoint service adds metadata to a subset of Microsoft-centric files.  For my clients, this includes .msg files (Outlook).  I don't think this is happening in transport (i.e., the OneDrive sync agent isn't actually modifying the files).  *I think* this is happening at rest in the SharePoint cloud datastores (via some indexing or transform process).  In my files, the additional XML metadata indicates "MediaServiceMetadata" and other clear indications that the file transform was done by Microsoft.

As mentioned, this is not happening to all Microsoft-centric files, just a subset.  So far I have not been able to determine why only a subset is transformed.

I'm hoping my opening sentence encourages some SharePoint guru to respond with something like, "Yes they do openly publicize this behavior, and here is the link."  ...I further hope the information will explain the 'only a subset' angle.

From a software architecture perspective, I'm assuming the SharePoint developers are attempting to provide metadata that transcends a single SharePoint datastore.  Nevertheless, like you Rob, I don't like this behavior, and they should provide a switch to disable this behavior (or change the design so that metadata is stored in a separate reference database...or allow the SharePoint administrator to select between the two metadata storage models).

15007 Stability - Microsoft Community

b1ws say:

15007 Stability

Why does 15007 work so well when Microsoft Basic Display Driver replaces the display driver that gets installed by Windows Update?  No BSOD, Crashes, Audio issues, white text boxes or whatever occurs!

willy denoyette say:

Because it's a software only driver, it doesn't depend on the graphics device hardware, is generic, well tested and simple, is completely developed by MSFT and are used on basically all windows server SKU's.

Graphics device drivers are a lot more complex and highly optimized, they are bound to 3rd. party graphics hardware for display and rendering. Graphic HW comes from many different manufacturers, different graphics chipsets  may require a new driver version, chipsets may come with different HW levels, graphics drivers may need some tweaks to handle newer levels and Chipset faults.
