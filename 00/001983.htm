    scott mcdaniel say:

    I'm not sure what you're asking could be done with an Access Web App (or with a standard web application, for that matter).

    If you have disconnected users who must enter data while not connected, you typically would setup a "replication" scheme which would enable you to merge data from each user machine into a master database. Newer versions of the ACE database don't provide replication, so you'd have to switch over to something like SQL Server, which means you'd have to purchase SQL Server for your main replication machine (you could use SQL Server Express for the disconnected clients).

    SQL Server has a robust replication manager, to handle conflicts and such, so it's a fairly complete solution. You cannot, however use that with a Web App (at least to my knowledge), so you'd have to  convert to a standard Access Desktop app (which means a complete rewrite).


    trevmonster say:

    Is it possible for me to host a Access Web database from my own PC. Then the users can use a browser to submit forms while they are within WiFi range. I would only need to create a simple input form as an Access Web page to implement this.

    Even better if there was a runtime for all operating systems, but we know that's not happening. If that were possible then I would put my whole business on Access.


    scott mcdaniel say:

    Access Web Apps use Sharepoint to host, so you'd have to install that on your machine. Sharepoint requires a Server OS (like Microsoft Server 2012),  so unless you're running that you won't be able to install it. Your scenario would also require you to install IIS, which typically is hosted on a separate machine.

    All in all, trying to setup, configure and maintain a secure Sharepoint/IIS installation is far beyond the capabilities of the average users, and generally requires someone with extensive experience in securing and hardening this sort of installation. 

    Access provides a Runtime version, but of course it's Windows-based. Since 2007, the Access Runtime has been free. Are your users running operating systems other than Windows? 

    All in all, considering your concept of disconnected users, the "best" methods would be (in descending order):

    1. True web application wirtten in a web language like ASP.NET, PHP, Java, etc, and hosted on a secured server

    2. .NET application connected to a hosted SQL Server database

    3. Access application w/ SQL Server Replication

    IMO, Access web apps aren't really ready for prime time, although they have become more robust in recent versions.

    Yes, you can migrate your Access desktop application to use SQL as the database. This is a migration, and if you have experience with SQL server, it not too hard and usually very few changes are required to such applications. So you thus install + setup SQL server (express) on each laptop. 

    The re-sync is not in the background (but it can be). So you can execute a command to start the sync (and such commands could be executed from VBA via Shell()). And the user would not have to exit or leave Access while this sync occurs.

Or you can simply re-start the agent for a sync (it really depends on what how and when you want the sync to occur â€“ and if you leave the  sync agents running all the time or not). Additional questions can be asked in SQL forums.
