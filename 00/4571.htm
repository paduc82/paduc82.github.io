Is it possible to paste text whilst entirely adopting all parameters of a custom formatted page?

Thanks for yourt time.

suzanne s. barnhill say:

If you paste "Text Only" or "unformatted," text will take on the characteristics of the surrounding text. You say you want spacing between lines to be uniform regardless of the size of the text; the way to do that is to set the line spacing to an "Exactly" amount. This may require some trial and error to get right.

The difference between a line break (inserted with Shift+Enter) and a paragraph break (Enter) is that the line break doesn't start a new paragraph, so, if the paragraph has some Spacing Before/After, you will not get this between lines if you use a line break; you'll just get the normal paragraph spacing.

You said you were writing "lyrics." Lyrics are considered verse (poetry).

suzanne s. barnhill say:

There are two types of line spacing, both controlled by the Paragraph dialog:

  1. Line spacing: This determines the space between lines in a paragraph. By default, the line spacing of the Normal style is 1.08 lines.
  2. Spacing Before/After: This determines the space between paragraphs. By default, the Normal style has 8 pts Spacing After.

As a general rule, when writing poetry, you use line breaks between verses and paragraph breaks between stanzas. In poems with runover lines, it may be preferable to use paragraph breaks between verses and use a hanging indent. In that case, you should use only line spacing, with no Spacing Before/After.

You can use either Multiple spacing (single, 1.5, double, or another percentage such as 1.08), which is based on the point size of the text, or Exactly spacing, which is the same regardless of text size. The default (Single) line spacing ("leading") is usually more than the nominal point size. For Times New Roman, for example, it is 120% of the font size: 12 points for 10-point TNR.

oldunix_dos say:

How to Permanent Skype unlink and Permanent Invisible Option (hotmail)

If you are a longtime Hotmail or Outlook Web user, you might have noticed that JUST reading and replying to your email has become a very slow process.


Reason for the slow speed is two login processes and connections to two sites:

The new and slower Outlook email now automatically logs you into a SKYPE users account with your email address, so your machine must first log you into the email and then connect you to ...Skype.com and log you into a Skype account.   Why you need a Skype account to read and reply to email is unknown and unexplained.  I have a Skype account, and when I want Skype, I login to the Skype site.


Can I unlink Skype permanently?:

No!  Skype allows you to unlink your email account, but the account is only unlinked until you log off and log on again.  Upon your next email login, Outlook will create a new Skype name for your email account (emailname1, emailname2, ...) and re-link your account to Skype.  The email user can continue to login and unlink Skype from each email login session until the incremented number hits its maximum limit. Once the maximum limit is met, the Skype account cannot be unlinked and you will need to call to have the incremented number reset/extended. 


SO THE ANSWER IS NO!  Your email account cannot be permanently unlinked from SKYPE.

There might be times when you do not want the world to know you are logged into your email, but with SKYPE linked, you are not invisible - SO WHY NOT USE THE AVAILABLE OR INVISIBLE OPTIONS?

Reason 2: In the new Outlook Preview the AVAILABLE option is selected and the user CANNOT select the INVISIBLE option.

a. user say:

Use the Change event instead of SelectionChange.  However, you say that the cell value for S1 changes dynamically.  I'm assuming that you have a formula in S1, correct?  If so, you'll need to use the Calculate event instead.  Here's an example...

In a regular module, declare the following public variable at module level (at the top of the module, before any procedure):

Public PrevCat As String

In the code module for ThisWorkbook:

Private Sub Workbook_Open()
    PrevCat = Sheet1.Range("S1").Value
End Sub

In the code module for your sheet:

Option Explicit

Private Sub Worksheet_Calculate()
    
    Dim pt As PivotTable
    Dim Field As PivotField
    Dim NewCat As String
    
    NewCat = Sheet1.Range("S1").Value
    
    If NewCat <> PrevCat Then
        Application.EnableEvents = False
        Set pt = Sheet1.PivotTables("PivotTable6")
        Set Field = pt.PivotFields("Region Name")
        With Field
            .ClearAllFilters
            On Error Resume Next
            .CurrentPage = NewCat
            On Error GoTo 0
        End With
        pt.RefreshTable
        PrevCat = NewCat
        Application.EnableEvents = True
    End If
    
End Sub

Note, though, I don't think it's necessary to refresh the pivot table.  So I think you can delete that particular line from the code.  Also, if Sheet1 refers to the sheet that contains the code for the event, you can replace each instance of Sheet1 with the keyword Me.

Hope this helps!
