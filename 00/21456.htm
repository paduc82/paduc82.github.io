A yes/no field is stored as a number: -1 is Checked/True/Yes, 0 is Unchecked/False/No. So you can use a query with a criterion of

Yes

or

-1

on the Yes/No field to select active members, and 0 or No for inactive. Note that these are the words Yes and No, without ANY quotemarks.

You can create a Report based on one of these queries, and just put a pagebreak control at the bottom of the detail section to print one page per record.

 

tom van stiphout say:

If you mean: how do I only print the active ones, I would do this with a criteria form. On this form we prompt the user for criteria before we print the report.

The report itself is setup to print all records, but the criteria form will restrict this to just the active (or inactive) ones. You can add other criteria later (e.g. a date range, or a dollar amount range, or a set of zip codes)

Create a new unbound form, let's call it frmRptCriteria. Put a checkbox on it named IsActive with a label "Is Active?" and a button "Preview".

In the button click event write the one-liner to open the report:

DoCmd.OpenReport "myReportName", acViewPreview, , "Active=" & Me.IsActive

Note how in the WhereClause argument we're saying "Active=" & Me.IsActive. I am assuming that your database field for active/inactive is called "Active". Me.IsActive picks up whatever value the user filled in in the checkbox, so the report automatically restricts the records to just those members.

One nice thing to have (some would call it essential) is that you put the criteria on the report, so the user of the report knows if she is looking at the active ones or the inactive ones. To do this you can have a textbox in the header, and set its control source to:

="Active=" & Forms!frmRptCriteria!IsActive

Or nicer:

="Active=" & IIf(Forms!frmRptCriteria!IsActive=True, "True", "False")
