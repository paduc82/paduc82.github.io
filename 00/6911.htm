How do I add more child forms to a parent form? - Microsoft Community

2cafes say:

How do I add more child forms to a parent form?

I created a parent form which is linked to a child form, but now need to add two additional child forms to the parent form.  I have created these forms, but cannot figure out how to link them.

I created a parent form which is linked to a child form, but now need to add two additional child forms to the parent form.  I have created these forms, but cannot figure out how to link them.

 

Thanks!

Thanks!

 

ken sheridan say: You filter the 'child' form at two stages:

1.  When you open it from the account details  'parent' form by means of the WhereCondition setting of the OpenForm method, so the code in the Click event procedure of a button to open a form named VBS Preview Detail might be:

Dim strCriteria As String

strCriteria = "AccountDetailID = " & Me.AccountDetailID
DoCmd.OpenForm "VBS Preview Detail", WhereCondition:=strCriteria

This build a string expression using the parent form's current AccountDetailID value and assigns it to the strCriteria  variable.  This is then used as the WhereCondition argument of the OpenForm method to filter the VBS Preview Detail to rows with that AccountDetailID when it opens.

2.  In the account details form's Current event procedure synchronize the VBS Preview Detail form,  and any other open 'child' forms, with the parent form by changing its Filter property to reflect the current AccountDetailID with:

Dim strCriteria As String

strCriteria = "AccountDetailID = " & Nz(Me.AccountDetailID, 0)
On Error Resume Next
Forms("Church Training Detail").Filter = strCriteria
Forms("VBS Preview Detail").Filter = strCriteria
Forms("VBS Fun Shop Detail").Filter = strCriteria

The 'On Error Resume Next' caters for one or more of the child forms not being open, and causes the code to go onto the next line, ignoring the error.

john mcghie say:
Hi dasb:

The style of the text in the table must BE "Normal", not a "style like Normal", otherwise the style applied to the table text will override the Table Style.

I spent all day today using Table Styles.  And tomorrow, I will do it again.

As I said, it's the most poorly-explained and confusing dialog in Word.  What I have learned is:

*  A Table Style is a "collection" of styles, it is not a single style.  Actually, that is wrong: it's a single style, but it has multiple sub-collections of formatting properties.

*  You have to be careful to set formatting properties for "Whole Table", then format other parts of the table "by exception" from the formatting you specified for the whole table.

*  If you think of Word formatting as a set of "layers", then the Table Style adds two BOTTOM layers.  So in order, most significant to least significant, we have:

1)  Direct Character formatting
2) Direct Paragraph Formatting
3) Character Style Formatting
4) Paragraph Style Formatting
5) Table style character formatting
6) Table style paragraph formatting
6a) Table Style Heading Row formatting
6b) Table Style Odd Row Formatting
etc...

For Table Styles to work, you have to format the entire table with "Normal" paragraph style,a nd remove any direct formatting that may have been applied.

That enables the Table Style formatting to "show through" the layers.

You can then apply Paragraph or Character styles within the table, and those will override the Table Style.

There is no explanation of this, anywhere that I can find...
