don varnau say:

Hi,

First, many sites, especially https (secure) sites such as banks, credit cards, other financial sites, have disabled password caching. There's nothing you can do to change this.

 

From Internet Options> Content> AutoComplete> Settings, verify that you have checked "User names and passwords..." and check "Ask before saving passwords."

 

In Internet Options> General tab> Browsing history, verify that Delete browsing history on exit is not checked.

 

This may be a cookie issue...

1. Internet Options> Privacy> try a cookie setting of low or accept all. Click the advanced button and verify that nothing is checked.

2. Verify that the computer's date, time and time zone are correct. These settings can affect cookie handling.

3. An existing cookie may be corrupted. From Internet Options> General > Browsing History> Delete (next screen) uncheck Preserve Favorites website data then check and delete Temporary Internet Files and Cookies.
Note: After clearing cookies, sites which usually remember you won't recognize you on your next visit.

4. Is a security/privacy program "managing" your cookies? CCleaner and CA Internet Security have this option, as do some other programs. Find this feature in the program's options and disable it.

 

If you, or another user, is clearing History or Temporary Internet Files, uncheck Cookies, so they aren't cleared. The site's cookie "recognizes" you when you visit so that username can be filled in automatically.

 

Don

Hi Leonie,

 

I suggest you try removing the blue cloud heading in file explorer by changing the registry value.

 

WARNING: Using Registry Editor incorrectly can cause serious problems that may require you to reinstall your operating system. Microsoft cannot guarantee that problems resulting from the incorrect use of Registry Editor can be solved. Use Registry Edit at your own risk.  

 

1)Use the Windows key + R keyboard shortcut to open the Run command.

3)Browse the following path:

HKEY_CLASSES_ROOT\CLSID

6)Change the DWORD value from 1 to 0.

I've been doing some experimenting, and I see where the error is coming from. I believe you must have used the "Contains" filter option from the Text Filters submenu, rather than the "Equals" filter option. In my tests, if I search for

 

     "AT&T"

 

using the Equals option, it finds it with no problem, but if I use the Contains option, I get the error message, "Enter a valid value".

 

As I said, I've done some experimenting, and it seems that if you use the Contains option, the way that Access parses your entry and builds the filter string always results in an invalid filter string. Maybe there's some special combination of quotes that would work, but I can't find one. However, if you use the Equals option, just quoting the entry as I suggested works to find records where the field contents exactly equal "AT&T".

 

Furthermore, if you need to find records that *contain* this string, but may not necessarily exactly equal the string, you can still use the Equals filter option, but use the wild-card character '*' in the string to tell Access you want to search within the field values. You would enter this:

 

     "*AT&T*"

 

But, as I said, you have to choose the Equals filter option and enter that string (with the quotes). You can't do it with the Contains option.

All this is caused by a flaw in the Application.BuildCriteria method, which Access uses to parse user-entered criteria values and create a where-condition, in combination with the extra logic Access applies when the Contains option is used. Fortunately, even if the Equals option is chosen, if BuildCriteria sees a wild-card character in the criteria value entered, it automatically sticks "Like " on the front of the value to create a wild-card match, so you can use this trick to get a "contains" filter without using the Contains option.

 

This isn't really new, by the way. The BuildCriteria method in older versions of Access worked the same way. However, before A2007 there was no "Contains" filter option, so if you wanted to do such a filter you had to know to put asterisks on either side of the value you were looking for. The difficulty now comes from the fact that the new Contains filter option puts those asterisks around your entry automatically, which precludes you from entering the exact expression, with wild-cards, that you want.
