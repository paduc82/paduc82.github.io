rohn007 say:
Maintaining that many parallel versions of documents, most of the content the same, with some differences to document version changes is going to be time consuming and not easy. Especially when you want changes to be propagated through the various versions.

For the high level organization you have 2 options as I see it.

High Level #1:
Use folders to separate each version, ie
Foldername: Product Name V1.0
Foldername: Product Name V1.1
etc

High Level #2
Use Version numbers in the file names to identify the product versions with all of the product versions in the same folder
V1.0 Filename1.docx
V1.0 Filename2.docx
V1.1 Filename1.docx
V1.1 Filename2.docx
etc

OR
Filename1 V1.0.docx
Filename1 V1.1.docx
Filename2 V1.0.docx
Filename2 V1.1.docx

*****************************

When you initially create the documentation set for a new product version, just use Shortcuts with the new file names to point to the original document.
Then if you have to make version specific changes to the document you delete the shortcut, make a copy of the original document and rename it for the version name. Then make the version specific changes.

If you want to make a change that is passed identically through all of the versions documents I would use copy and paste special linked this way:

. * Make the change in the first document version it appears in.
. * Select and copy a large block of text the change appears in, ie a sentence, or paragraph if there are additional related changes in paragraph
. * Open the same document in the next product version documentation set, ie filename V1.1,
. * Find the matching paragraph of text
. * delete it
. * Home tab > Clipboard group > Paste Special command
. * In the paste special dialog in "As:" section click on "MS Word Document Object" and to the left of that click on the "Paste Link:" radio button
. * click on OK button at bottom of dialog.
. * now, any changes made to that section of the original document will be reflected in this copy pasted into the other document(s).

**************************

Will multiple people be working on maintaining this set of documents, ie making changes specific to their projects?
Or, will it be just one or two (or very small, specific group of people) who will be maintaining these documents?

If you are going to have a large, random group of people /developers making changes to the documents you are going to need "version control" software to help control who makes changes and to make sure all changes are merged together properly. The Word Track Changes feature will play a part, but a separate product to track all of the changes and people making changes will help keep things organized.

This is an example of what I'm talking about:
https://www.simuldocs.com/blog/ben-morris/versi...
I have not used this specific tool, but I have used others for program code. One thing to specifically look for is support for MS Word format files. The most common tools in this category only work with text-only files.

Even if you are going to have a very small limited group of people updating the documents you should prepare detailed "how to" procedures documents to ensure that everyone knows how to do this work correctly.


NOTE: if you go to File menu > Options command > Advanced option > scroll down to Show Document Content section and in the "Field Shading" drop down, select "Always". This will make the text that has been pasted with a link appear shaded so you can easily identify it.

Paste Special
https://support.office.com/en-us/article/paste-...
Like pictures and other objects, text in a presentation has its own formatting — such as typeface, color, and font size. When you copy text that has different formatting into an Office program, the program, such as PowerPoint or Word, automatically reformats that text to match the text of the destination. However, you can use Paste Special to maintain the original formatting or paste it as a link or a picture, for example.

marshall barton say: In addition, to John good explanation of how SQL is processed, Visual Basic for Applications (used by Office and some other programs) is not compiled into machine language.  Instead it is compiled into something called P-code that is an abbreviated instruction set for a ficticious machine.  A program called an interpreter then simulates the P-code machine.  This is the way that many, but not all, variations of the Basic language are processed.

john w. vinson say:

The exact details are of course Microsoft trade secrets, but SQL is not "executed" in the same sense as Cobol or Fortran or PL/I. Instead, Access uses a (large and complex) program called a "Database Engine" to parse the SQL and carry out the search.

One nice feature of Access is that the Access program itself doesn't need to be the engine. Older versions of Access used the JET engine common to many Microsoft operating system and application programs; with A2007 the Access team took ownership of the engine and renamed it ACE; but Access can also use the SQL/Server engine (in any of its many versions), or use the ODBC methodology to connect to many different (even non-Microsoft) engines.

So what Access does (as I understand it, from an outsider's viewpoint) is store the SQL as text; parse it into a more compact and computer-friendly version colloquially called "compiled SQL"; and pass this information to the appropriate database engine, which actually issues the instructions to read this particular index file from the disk, find the specified search term, read the needed records of the actual data file, and return that data record to Access.

In any case, the "compiled SQL" is stored internally in Access' system tables, not as an external disk file; it's all "under the hood" and dynamic, and isn't exposed anywhere for you to look at.

hansv mvp say:

You want a multi-level list.

 

Option 1: use the built-in bulleted multi-level list.

Click in the level 1 text.

Click the MultiLevel List button.

Select the bulleted list.

 

Option 2: modify the current multi-level list.

Click in the level 1 text.

Click the Multilevel List button.

Select "Define New Multilevel List..." (this is the completely counterintuitive command for modifying an existing multi-level list).

Click each level and select or enter the bullet you want for this list.

Finally, click OK.
