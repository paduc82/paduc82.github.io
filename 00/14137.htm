IMAP vs. POP3, Should I convert and how? Backing up? - Microsoft Community

manningm22 say:

IMAP vs. POP3, Should I convert and how? Backing up?

I keep getting cut off by my ISP because the mail on my server exceeds the quota my company established (very small company).  I et up my Outlook 2010 account using IMAP (at a colleagues recommendation).  When I periodically log onto the server via webmail to delete and make some space it also disappears from my outlook view!  Should I be converting to pop3?  If so how would I do that without loosing email and folders.  Then if I do, I assume I'll need to have a regular backup of my pst. file since mail wont stay on the server.  Any advice?
brian tillman [outlook mvp 2007-2019] say:

POP cannot access any server-side folders except for Inbox.  If you have a folder structure on the server you will need to use IMAP.  Since IMAP always reflects what is on the server, deleting something from the server will delete it from Outlook also.  If you want to keep data that you delete from the server, move it to a local PST  That will delete it from the server while retaining a copy locally you can reference.

Since IMAP accounts reflect the server contents, you don't need to back up the PST created by the IMAP account.  The server's backup will capture your mailbox.

You can convert to POP, but if you want to duplicate the server-side folder structure, you can create a local PST, copy each IMAP folder and its contents to the local PST, then change the account to POP and make that local PST the default PST.  This process can, if you allow it, empty everything from the mailbox and store it locally.  You can allow the Inbox data to remain on the server if you wish.  One you do this, the only way you'll be able to access non-Inbox folders on the server will be va a web browser.

zach prins say:

Hi Pankaj,

Right click on Sheet1, then click on View Code, this will take you to the VBA editor with the code for Sheet1, which should be empty.

Now copy and paste the following code into the code window:

Private Sub Worksheet_SelectionChange(ByVal Target As Range)

    If Not Intersect(Target, Range("A:A")) Is Nothing And Not Target.Value = "" Then

        Dim shape As Excel.shape

        'Delete the previous image(s)
        For Each shape In ActiveSheet.Shapes
                shape.Delete
        Next

        'Insert the image located at the path selected
        With ActiveSheet.Pictures.Insert(Target.Value)
            With .ShapeRange
                .LockAspectRatio = msoTrue
                .Width = 75
                .Height = 100
            End With
            .Left = ActiveSheet.Cells(1, 1).Left
            .Top = ActiveSheet.Cells(1, 1).Top
            .Placement = 1
            .PrintObject = True
        End With

    End If

End Sub

Every time you select a different cell, this code will check if it is in column A, and if the cell contains a value. If so, it will delete the previous image, and insert the new one.

bob jones aka: cybertaz say:

It isn't a 'glitch' :-) Formulas in Word are contained in Fields. What you've inadvertently done is to toggle the display of Field Codes rather than display of the Formula result. The option exist so that Fields can be examined and/or edited if necessary [e.g., you might want to modify the formula to return { =Average (ABOVE) } rather than Sum].

Most likely you used the Option+F9 keystroke by accident - use it again to reverse the display status. Alternatively, you could go to Word> Preferences - View - Show to clear the check on Field Codes.
