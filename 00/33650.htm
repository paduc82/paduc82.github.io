mimbo mcjimbo say:

For the first, whenever I have an issue with the XBOX getting out but other things working, it's almost always DNS.  It's happened for me very rarely since using DNS Benchmark to find the most reliable quickest servers, which I put in as the router's DNS and then pointed the XBOX to the router .     You can check to see if www.xbox.com is correctly resolving to something like 63.97.127.43    

When things get into a state where things don't work, what I usually do is power off the modem and the router, leave them off for 30-60 seconds, turn on the modem and after a minute or two, turn on the router.   After another minute or two I check everything from a computer.  

For the second, all timeouts on a traceroute.   Maybe none of the steps along the way are answering  for whatever reason.    (While not every host (destination) you're trying to get to is going to answer, and not every hop along the way is going to reply all at all or every one of the three tries, at least some of the hops along the way should be answering some of the time.)     It is possible the router/firewall or something in the client machine you're using for the traceroute is at fault.   Blocking things that should be working, on either the computer, router or both.   You'll have to check things including the filtering that taidara wrote about..

I'd first make sure you can get on a computer on the network and ping both the XBOX's IP address and the gateway IP address internally on your network.   Next you can try the WAN address, which is the IP your modem should have/be providing to the router, the one that actually is on the Internet.    For all this, doing an "ipconfig /all" in Windows or "ifconfig" in Linux, and looking at the router setup and status, should work to get all the details.

Intel(R) Management Enginge interface - Microsoft Community

jess1970 say:

Intel(R) Management Enginge interface

Can anyone help. Upgrade windows 7 Advisor says that for the Intel(R) Management Enginge interface device is not compatible with windows 7. I am currently have a gateway quad intel core system running Vista service pack 2. What can I do to fix this problem?
rosspk say: That other thread really does not have any answer for your issue either. The MEI mentioned is the hardware monitoring feature of genuine Intel motherboards. I would suggest the Dell board is sourced from Intel in your case. Windows 7 can be installed without loading the MEI driver, you just won't be able to monitor system & CPU temps etc. However, I have been running Win7 & Windows Home Server on several Intel boards with the (unsupported) MEI driver installed. It works perfectly, I have had no issues with it. You will have to force the MEI driver to install though.
To do this, go to the Intel support site and download the correct MEI driver for your motherboard chipset - eg. if your chipset is Intel P35, download the MEI driver for the DP35DP motherboard. Choose Vista (however it is an all OS driver). Execute the download, it will unpack to a folder & attempt to install. The install will fail. Go to the MEI folder in the unpacked files & locate a file called HECI.inf.  Open this file to edit & scroll down to the manufacturer & models section. Look for the heading [Intel.NTx86.6.0]. Highlight everything from immediately below that heading to the end of that section & press CTRL+C to copy it. Now scroll to the section below which is labelled [Intel.NTx86.5.2]. Highlight the line below (which states it's an empty section) & press CTRL+V to overwrite this line with the previously copied data. Save & close the file. You now have an installer that you can use in Win7 & WHS.
Go to Device Manager, locate the MEI device - it's usually shown as a Simple Communications Device with no driver installed. Update the driver & point the installer to the MEI folder conatining the inf you just modified. Tell Windows you wish to install even though the driver is not signed. Your Intel Management Engine Interface will now be working.
As I mentioned, I have used this method to install on a number of Intel motherboards & have had no problems. Good Luck.
