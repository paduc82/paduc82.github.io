hansv mvp say:

Select the range that you want to format conditionally: rows 2 through 66, or A2:P66, depending on your preference. I will assume that the active cell within the selection is in row 2.

 

On the Home tab of the ribbon, click Conditional Formatting > Manage Rules...

 

Click New Rule...

Select "Use a formula to determine which cells to format".

Enter the formula   =$A2="USF"  (where 2 is the row of the active cell).

Click Format...

Activate the Fill tab.

Select red.

Click OK twice to return to the Conditional Formatting Rules Manager.

 

Repeat the above steps three times, but with

=$A2="FIU"    and green,

=$A2="UF"     and blue, and

=$A2="FSU"    and purple.

 

Finally, click OK to close the Conditional Formatting Rules Manager.

For dumps to be produced. . .

â–ª Page file must be on OS drive
*If manually set Page file base allocation size must be > than installed physical RAM

*Windows Error Reporting (WER) system service should be set to MANUAL
*PREFERRED>>>Set page file to system managed on the OS drive
*Set system crash/recovery options to "kernel Memory Dump"
*User account control must be running.
* Sometimes SSD drives with older firmware do not create DMPS (update firmware)

*Cleaner applications like Ccleaner delete DMP files so don't run them until you are fixed.

The contents of the crash is written from memory to a page file (usually) and this only completes when it gets to %100 so if you force quit before the process is complete it cannot be written from the page file to the *.DMP.

There may be a full memory dump located @ C:\windows called memory.dmp which gets over written after each crash so nothing needs to be created.
