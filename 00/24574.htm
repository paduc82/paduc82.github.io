bob jones aka: cybertaz say:
Page Borders are set just outside the margins of the document - necessary to avoid potential obliteration of text.

As Rich suggested, most printers have minimum margin requirements even if they claim to do "borderless" or "edge-to-edge" printing. If you read the tiny footnote in the printer's user guide you'll find that the capability only applies if you're using one of the photo printing options - it doesn't apply to printing other types of documents.

Often it's further limiting on a laser printer because they typically require a larger bottom margin than on the other 3 sides in order for the sheets to be drawn through the printer.

Determine your printer's minimum margin requirements the calculate your margins large enough to accommodate the additional width of the style of Border you're using.

anula d'souza say:

Hi Veasna,

Thank you for posting in Microsoft Community.

 

Both OST and PST files are Outlook data files. OST file is a mirror image of your web-mail data file. The email account types like IMAP account, MAPI account Exchange Account create an OST file to save

For more information on Outlook data files you may refer to the link mentioned and check:

http://office.microsoft.com/en-us/outlook-help/introduction-to-outlook-data-files-pst-and-ost-HA010354876.aspx

 

Q1: The Outlook data file ends with (1), (2) etc. because these are different data files. If you have configured the same email account multiple times to differentiate between the data files Outlook automatically adds the suffix (1) or (2).

 

If you do not need those data files then you may delete those data files.

 

Q2: If you have a POP account or if you have created a PST data file then it should be saved on the system.

PST file location for Outlook 2010:

If you had upgraded from earlier version of Office then the data file will be save din the location mentioned:

C:\Users\<username>\AppData\Local\Microsoft\Outlook

 

Otherwise by default the PST file will be saved in the location:

C:\Users\<username>\Documents\Outlook Files

 

You may also search for the PST files saved on the system from the explorer:

http://windows.microsoft.com/en-us/windows7/find-a-file-or-folder

Note: In the search box you may type *.pst to find all the files with pst extension.

 

Q3: In Outlook 2010, when you configure an email account by default the OST and PST files will be saved in the same location (C:\Users\<username>\Documents\Outlook Files ) which is the default location to save Outlook data file.

If OST and PST files are not saved in the same location then open Outlook, check the location where the PST file is save, remove the PST file from Outlook, browse to the location where the PST file was saved, copy the PST file, save the PST file in the location where OST file is saved, open Outlook and open the PST file in Outlook from its new location.

 

To remove the PST file from Outlook:

  • Open Outlook.
  • Click on File< Account Settings< Outlook Data Files tab.
  • Note the location where the PST file is saved, highlight the PST file and click on Remove.

 

To open the PST file in Outlook:

  • Open Outlook.
  • Click on File Open< Outlook Data File.
  • Browse to the location where you have saved the PST file highlight it and click on open.

 

If you have associated any email account with the PST file then follow the steps mentioned.

  • Open Outlook.
  • Click on File< Account Settings< Email tab.
  • Highlight the email account associated with the PST file, click on Change folder and point to the right Outlook Data file.

 

Q4: refer the link mentioned to backup the Outlook data file:

http://support.microsoft.com/kb/287070

 

Note: To backup the OST file, you may export the OST file (the complete data file) in PST format.

 

Q5: Ensure to backup the data file which contains all your data. If the data file is open in Outlook then to check the location of the data file:

  • Open Outlook,
  • Click on File Open< Outlook Data File.
  • Under Location check where the data file is saved.

 

I hope the above information helps. Let us know if you need any further assistance.

hansv mvp say:

Here is another option.

Activate the Developer tab of the ribbon.

Insert an ActiveX combo box on the sheet.

Its default name will be ComboBox1; I will use this in the code below. If you give it another name, modify the code accordingly.

Set the ListFillRange property of the combo box to the range of cells that contains the list to be displayed.

Right-click the sheet tab and select View Code from the popup menu.

Copy the following code into the worksheet module:

Private Sub Worksheet_SelectionChange(ByVal Target As Range)
  If Not Intersect(Range("A:A"), Target) Is Nothing And Target.Count = 1 Then
    With Me.ComboBox1
      .LinkedCell = Target.Address
      .Top = Target.Top
      .Left = Target.Left
      .Width = Target.Width
      .Value = Target.Value
      .Visible = True
      .Activate
    End With
  Else
    Me.ComboBox1.Visible = False
  End If
End Sub

Switch back to Excel.

Turn off Design Mode on the Developer tab of the ribbon.

Important: since the workbook now contains VBA code, you should not save it as a .xlsx workbook - the code would be lost. Save it as a macro-enabled workbook (.xlsm) or if you need to share it with users of older versions of Excel, as an Excel 97-2003 workbook (.xls).

cbarnhorst say:

What most folks don't get yet is that ReFS provides real-time file healing.  There is no offline file repair and therefore no utilities like chkdsk are used.  File healing goes on without any downtime.  That makes for user-transparent file security.  No user intervention means file safety is much more likely to get done.  A study a couple of years ago showed less than 10% of Windows users have ever made even one backup.  That means that either an enforced cloud-oriented backup procedure built into Windows is needed or data safety features have to reach incredibly high reliability.  I can't see users putting up with any automated collection of their data in Microsoft's cloud on that scale, not to mention the infrastructure considerations, so increased reliability is the answer and the reason MSFT started ReFS development in the first place.

I suspect that a large percentage of consumers don't even own a large thumb drive much less an external hard drive so reliance on users to use backup/restore solutions is fruitless.  90% of users simply refuse to be that pro-active so a passive approach is necessary.  MSFT is on that track by making a file system that is much more resilient than is achievable with NTFS. 

These are the considerations that lead me to say that ReFS is as essential for consumers as it is for enterprises.  People need to think about what consumers need for data security before they just assume that ReFS is not for consumers.  MSFT learned what not providing anti-virus software in Windows led to and is taking a similar approach to data security.  

What irritates me the most about the step back with ReFS for consumers is it is a Marketing Department move to add value to Windows 10 Pro for Workstations by removing something of value for consumers from Windows 10 Pro arbitrarily.  This was a stated Marketing Dept. strategy a decade ago, unnecessarily fragmenting Home and creating Ultimate.  Making feature inclusion/exclusion a marketing decision made a mess of Vista and 7 and threatens to make a mess of Windows 10.  I rejoiced in getting marketing out of those decisions after Vista and Windows 7 and here it is creeping back in again.  I have always believed in "one edition to rule them all" for precisely that reason.

Hi,

Steps that worked for me below:

  1. Go to New E-mail
  2. In the New E-mail window that pops out, go to File
  3. Go to Options
  4. In Options window that pops out, go to Mail tab (should be active by default)
  5. In that tab, go to Editor Options
  6. In the Editor Options window that pops out, go to Advanced tab
  7. In that tab, scroll down to Display e-mail content section (should be 2nd from the bottom)
  8. Uncheck Show picture placeholders checkbox
  9. Click OK all the way through
  10. Restart Outlook

allen browne say:

One alternative would be to open the subform in design view, and change it to Continuous Form (in the Form properties.)

Then show the Form Header/Footer, and put labels in the Form Header.

Rearrange the text boxes side by side (deleting any attached labels.)

The result is similar to a datasheet view, but doesn't have the datasheet problems (filter options obscuring headings, columns that the user can hide or make zero-width, etc.)
