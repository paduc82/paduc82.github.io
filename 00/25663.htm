andreas killer say:
Try the code below, I did not test it, but I think it should work in your control.

Andreas.


Private Declare Function GetKeyState Lib "user32" ( _
    ByVal nVirtKey As Long) As Integer

Function GotFocusMethod() As Integer
  'Determine how a control got the focus
  If GetKeyState(vbKeyTab) < 0 Then
    If (GetKeyState(vbKeyShift) < 0) Then
      'Shift-Tab key is pressed
      GotFocusMethod = 4
    Else
      'Tab key is pressed
      GotFocusMethod = 1
    End If
  ElseIf GetKeyState(vbKeyMenu) < 0 Then
    'Alt key is pressed (hotkey activation)
    GotFocusMethod = 2
  ElseIf GetKeyState(vbKeyLButton) < 0 Then
    'mouse left button is pressed
    GotFocusMethod = 3
  End If
End Function

"Normal document template" notification - Microsoft Community

josiahquising say:

"Normal document template" notification

Whenever I open Word and upon closing it, this notification saying "Word automatically saved changes to the Normal document template. Do you want to load it?" and a similar message when I close the document saying that "Changes have been made in the global template, do you want to save it"? Pictures are attached below. 

How can I stop this from happening?

jay freedman say:

The quick-and-dirty answer is to go to File > Options > Advanced, scroll to the Save section near the bottom of the page, and clear the check box for "Prompt before saving Normal template".

There are a couple of things you should know first, though.

First, under ordinary circumstances your Normal template should not have any changes that need saving every time you run Word. That template could have changes if, for instance, you modify a style in a document and select the option "New documents based on this template" at the bottom of the Modify Style dialog; or if you have an automatic macro or an add-in that changes something in the template (not a good practice). Otherwise, the Normal template could go unchanged for months. It would be worth investigating what's causing the constant changes.

Second (in light of the first), it's usually a good idea to be notified when Word does think there are changes in the template. If you know what the changes are, you may want to say No to the prompt and leave the template as it was before. If you clear the check box, Word will always automatically save the template; the next time you start Word, something may be different and you wouldn't know where it came from.

Third, the message in your screen shot about "the file is in use by another application or user" indicates a problem -- probably a temporary file called an "owner file" is present because it was abandoned during a program crash. With Word shut down, go to your Templates folder (usually %appdata%\Microsoft\Templates) and look for a file whose name starts with ~$rmal. If it exists, delete it and then restart Word.
