dice vil say:

Hi,

You can take advantage of the Heading Styles in Word 2016 to create your table of contents.  You can find more information about that in this support article.

Now as to arranging them in alphabetical order, this will depend on how you'll create the document, but the best way to do this is to use the Sort function. Once you've applied the Headings to the text and the table is created, you will need to use the Sort function to arrange the headings in Alphabetical order (see example screenshots below):

1. Highlight the original text and then in Home tab, click the Sort button.

2. A dialog box will appear wherein you can select how the text will be sorted. Choose Sort by Headings, Text, and Ascending as shown below:

3. The original text will now be sorted in alphabetical order. Next, you'll need to right-click on the table of contents and select Update Field.

4. Choose Update entire table and click OK to apply the changes.

This will sort the contents alphabetically the same way as the original text. Once you've finalized everything, simply delete the original text. Please take note that once you delete the original text, you will not be able to edit the table, so make sure that everything is in place before updating the field.

If you have other questions, don't hesitate to reach out. We'll be here if you need further assistance.

    wayne-i-m say:

    Just to add to what Tom said, you can't add leading zeros to a number.  Dead simple.
    If you do it becomes text and not a number.  If you have formated "another" control on your form to reference the autonumber field then that's different - but don't do "anything"with the autonumber.

    The auto number field is "only" there to provide a unique reference to a specific record - and it is only used by the programmer, never a user.  Best bet is not even to tell the user that this autonumber exisits

    eg.  you may have 2 john smiths with the same birthday and address (twins - with strange parents who gave them both the same name) - in this case the autonumber would allow you to refer to each John seperatley in a one/many  many/many etc.

    1234 is a number
    01234 is text
