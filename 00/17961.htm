Numbering reqiurements within paragraphs - Microsoft Community

patriciajonesxn say:

Numbering reqiurements within paragraphs

I have a document that lists requirement functions for software used by the Air Force.  I want to auto number or auto list each requirement within a cycle, such as:

 

[R-ALRT-1], [r-ALRT-2], AND SO ON.

 

HOWEVER, THERE ARE NUMEROUS REQUIREMENTS WITHIN THE DOCUMENT, SUCH AS: [R-EVT-1], [R-VIEW-13].

I would Word to auto number to ensure that we do not miss a consecutive number. 

 

How would I do this?  patti

 

suzanne s. barnhill say:

Since you want to use the numbers within paragraphs and not as paragraph numbers, I would suggest the use of the SEQ field. You can find information on this field at http://office.microsoft.com/en-us/word-help/field-codes-seq-sequence-field-HP005186190.aspx.

Some suggestions:

  • As you can see from the article, the SEQ field includes an identifier that specifies a given sequence, so you can have a different field for each series. For example, you can have { SEQ R-ALRT }, { SEQ R-EVT }, { SEQ R-VIEW }, and so on. Each sequence will independently.
  • A given SEQ field produces just a number, and all these numbers will of course look alike. You need to add the labels (R-ALRT, R-EVT), etc. I would suggest that you create AutoText entries for, say, "[R-ALRT-{ SEQ R-ALRT }]" and the like.
  • The additional advantage of using AutoText entries is that they increment automatically as you insert them; if you insert a SEQ field by copying and pasting, it does not update; you have to select it and press F9 to update it.
  • Unfortunately, in Word 2007, AutoComplete suggestions for AutoText entries are not available (they do work in Word 2003 and Word 2010), so you have to type the first few letters of the entry name and press F3. This is still easier than creating all these fields by hand, and if you give each entry a name in which the first three or four letters are unique, insertion will be almost painless. For example, if the R-ALRT AutoText entry were names ALRT, then you need only type ALRT and press F3 to insert, say "[R-ALRT-1]," "[R-ALRT-2]," and so on.

gord dibben say:

Try this version of PrevSheet UDF

I store the function in an add-in and have never had a problem.

Function PrevSheet(Rg As Range)
'accounts for more than one workbook open
'and has hidden sheets
'Bob Phillips  October 4, 2009
Dim n As Variant
    With Application.Caller.Parent
        n = .Index
        Do
            If n = 1 Then
                PrevSheet = CVErr(xlErrRef)
                Exit Do
            ElseIf TypeName(.Parent.Sheets(n - 1)) <> "Chart" And _
                .Parent.Sheets(n - 1).Visible = xlSheetVisible Then
                PrevSheet = .Parent.Sheets(n - 1).Range(Rg.Address).Value
                Exit Do
            End If
            n = n - 1
        Loop
    End With
End Function
