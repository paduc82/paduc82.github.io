Adding a pop mail account means that in the outlook setup, you have to tick other settings in order to have access to the pop settings.

Since an IMAP account is a copy of what is on the server you can remove the IMAP account. A pop mail account, unless in the settings you have ticked the option to keep a copy on the server will download all mail from the server, thereby removing it from the server, and store the data on the local PC.

Your going to need to provide some more details about "the system will not let me.......".

Specifically, it's not a "system" issue. It would depend on your email service provider as to whether or not you can access your account using different protocols at the same time (i.e. GMail, as a for instance, is an either/or situation which is controlled by a user-setting at the server level while other providers have no such restriction).

There would be no need to first delete the IMAP account in the profile. In fact, would strongly advise against deleting the IMAP account first since you will lose all data contained on the folders labeled with "This Computer Only" (contacts/calendar). After configuring the account using POP3 and setting the data file for the POP account as the default data file for the profile (File --> Account Settings --> Data Files tab), you would then copy the contact/calendar data from the IMAP folders to the corresponding POP3 folders.
