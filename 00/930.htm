Maybe I misread the OP, but to me it sounds as the file has been put in the recycle bin after which the file was removed from the recycle bin (and thus really deleted).

Undelete programs might be able to recover the file, but there really is no guarantee. Once a file is really deleted, all the sectors on your disc used to store the document come available again. Any type of disc activity can overwrite those sectors. The best thing to do when you want to recover such a file is turn off your machine and hang the disc in another machine to prevent any unwanted disc activity. Assuming you can't do that, try to limit the disc activity on your machine. so no surfing, ...

You could try http://www.officerecovery.com/freeundelete/ (a free undelete program for personal use) but the result will probably be the same as with your expensive software. If the sectors, or part of them are overwritten, the document is gone forever.

Using Data Validation with Enable Macros - Microsoft Community

whang56 say:

Using Data Validation with Enable Macros

I want to use data validation so that if macros are not enabled, data validation will not allow any entry in the selected cell. Maybe a helper cell that is TRUE if macros have been enabled? How is this done? 

kergga say:

One related answer is that you can actually set up a helper worksheet with a big warning "close this file and re-open with macros enabled", and have the workbook_open macro hide that sheet and unhide your actual data sheets (and reverse the hiding and unhiding process on workbook_close).

 

If you want to keep your form visible at all times and only allow entry with macros enabled, the concept is similar- use the workbook_open macro to change your helper cell to TRUE, and use the workbook_close macro to change it back to FALSE(if your users may re-use the same file)

 

I can think of two ways to use this to support your data validation; the easier way would be to set up your source list as a named range, and check the value of your helper cell within that named range formula. If the value of the helper cell is false, then the only item in the acceptable value list will be "Entry disabled" or if you have room, something more self-explanatory like "Enable macros to use this form"

 

Alternatively, you could build it directly into the data validation, by selecting 'Custom' instead of 'List'. You would then have to build in the matching logic as a formula, such as (aircode) =IF(AND(A1 = True,not(iserror(match(B1,MyListOfAcceptableValues,false))))

I'm not totally sure on that syntax, it is just to point you in the right direction.


EDIT: here is a sample for building the logic within data validation:

=OR(AND(G10=TRUE,NOT(ISERROR(MATCH(H10,F10:F12,FALSE)))),AND(G10=FALSE,H10="Enable Macros"))

[H10 is the cell containing the data validation]

This checks to see if the helper cell (G10) is true or false, then depending on the answer, it requires either a value found in F10-F12, or the text "Enable Macros" (which would be the default in that cell when you open the workbook)

The downside of this approach is that you don't get the nice drop-down lists in data validation (if you were using lists), and there is a little more setup hassle.

Do this from your old drive.

Sort your contacts in Entourage by category.

Drag to desktop folder as .vcf files.

Add them to a Group to retain the "category".

I ended up doing a combination of things in order to get out all the info I needed. It was days of work but I had 95% success rate.

First I did your suggestion of dragging out contacts per category to folders, importing those folders and using Last Import to create a group in Contacts. This method got out most of the data including any photos associated with my contacts which no other method does.

However vcf files do not include custom text nor date fields (including spouse, children, interests, etc.). There are two ways to do it one of which is free. The free way is to just export out a tab delimited text file, and then import that into Contacts. This gets MOST of the fields but does not get Children. If you are lucky to have Paul Berkowitz's scripts (which I purchased years ago) you can choose the one that exports everything to a .txt file which includes Children as well as a field with all the categories one might have assigned to a contact (I had multiple to most contacts so it is good to have them in a field in Contacts just for reference - I can't sort by them or anything like I could in Entourage but they are there).

So after I had my .txt file with all my data, making sure in Contacts that "All Contacts" was selected, I imported the .txt file, mapped the fields I wanted that did not exist in the vcf files, and then chose to merger the data. VOILA! I got contacts with their photos, their spouses names, their children, all the categories that had been assigned them in Entourage.

Lots of work but it I'm officially out of Entourage and on to my life.
