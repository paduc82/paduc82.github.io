Get-CASmailbox user@domain.com | fl OWAEnabled

 

You can enable OWA and ActiveSync in Exchange Admin Center page as an admin role:

In Exchange Admin Center> navigate to Recipients> Mailboxes> double click your mailbox> Mailbox features> enable them.

 

Meanwhile, please check whether your account is configured to use the correct protocol settings. To do this, run the following Windows PowerShell cmdlet:

 

Get-Mailbox | fl LinkedMasterAccount,ProtocolSettings

 

Note The LinkedMasterAccount attribute should be the user's domain\user_name logon. The ProtocolSettings attribute should contain "HTTP§1§1§§§§§§, OWA§1" to allow access to OWA. If the ProtocolSettings attribute starts with a zero (0), the protocol is disabled.

Run the Set-CASmailbox user@domain.com -owaenabled $false cmdlet.

Run the Set-CASmailbox user@domain.com -owaenabled $True cmdlet.

 

Check the protocol settings. To do this, run the following Windows PowerShell cmdlet:

 

Get-Mailbox user@domain.com | fl ProtocolSettings

jllatham say:

One way to approach this would be to have a 2-column table on a sheet somewhere in the workbook (could even be on the same sheet with your calculations).

First column (X for this example) would have a list of the materials, and the second column, Y, would have your multiplier. 

Then to help control the user input for the material, I'd use the list in column X as the source for data validation in your drop-down lists.  See Contextures.com for setting this up as a dynamic named range to allow it to automatically expand/contract as you add/remove materials from it:

http://blog.contextures.com/archives/2010/04/26/dynamic-excel-named-list-grows-automatically/

 

Lets say your dropdown is in A2 on the sheet, and you enter the material cost in B2, then in C2 you could have a formula like this:

=B2*IF(ISNA(VLOOKUP(A2,X:Y,2,FALSE)),1,VLOOKUP(A2,X:Y,2,FALSE))

 

The IF(ISNA())

Structure will choose a 1 as the multiplier if the selected material in A2 doesn't match an entry in column X of the lookup table.  You could change the ,1, to be ,0, resulting in a zero as the multiplied cost, or even a negative value to give you a clue that things aren't quite right, as ,-1,
