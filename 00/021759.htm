ken sheridan say: Computing the difference in days between incidents is similar to the common task of computing the difference between successive readings form instruments.  You'll find examples of this in Readings.zip in my public databases folder at:

https://onedrive.live.com/?cid=44CC60D7FEA42912&id=44CC60D7FEA42912!169

Note that if you are using an earlier version of Access you might find that the colour of some form objects such as buttons shows incorrectly and you will need to  amend the form design accordingly.

If we adapt one of my queries to return the difference in days between readings from each meter rather than the difference in readings the query would be:

SELECT R1.MeterID, R1.ReadingDate, R1.ReadingDate-
         (SELECT ReadingDate
          FROM Readings AS R2
          WHERE R2.MeterID = R1.MeterID
          AND ReadingDate =
              (SELECT MAX(ReadingDate)
               FROM Readings AS R3
               WHERE R3.MeterID = R1.MeterID
               AND R3.ReadingDate < R1.ReadingDate)) AS DifferenceInDays
FROM Readings AS R1
ORDER BY R1.MeterID, R1.ReadingDate DESC;

which would return the following result table in my demo:

MeterID    ReadingDate    DifferenceInDays
1                01/03/2013      28
1                01/02/2013      31
1                01/01/2013    
2                01/03/2013      28
2                01/02/2013      31
2                01/01/2013    

Note that dates are in UK format of dd/mm/yyyy.

In your case you would compute the difference between the start of downtime and the end of downtime of the previous incident rather than between values in the same column, as in the above query.  This requires only minor amendments to the SQL statement.

If you want to return the difference over a full set, rather than each subset (meters in my demo) then you'd omit the correlation on MeterID from the subqueries.  This can't be done with my sample data of course because the dates are not distinct.

To compute the mean difference between failures (MTBF) the summation of the differences is divided by the number of failures, which can easily done by creating a new query based on the result table of a query like that above.

john w. vinson say:

You have to change the table properties in the "Backend" - the database where the tables actually reside. You should be able to hover the cursor above the table name in the Tables window to see the actual path and filename of the table.

All Access databases should be "Split" in this way - the Tables in a shared "Backend" database, and everything else (queries, forms, reports and code) in a separate "Frontend", with links to the backend. Each user should have their own copy of the frontend. When you do make a design change to a table in the backend, it's best to delete the link to that table and create a new link to the modified table. I hope you have a copy of the frontend in .mdb or .accdb format - if it's been compiled into a .mde or .accde
