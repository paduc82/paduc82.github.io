graham mayor say:

The macro you quoted would work from Outlook, with some minor modifications to the sequence of commands. From Excel VBA it needs more changes.  Unless you are going to make references to Outlook and Word object libraries you should setup the macro for late binding to Outlook and Word. This means setting the Outlook and Word variables as objects and using the number equivalents of specific Word and Outlook references. Some change in the sequence of events is also called for.

e.g. using your macro as the basis

Sub sendcomplexemail()

Dim olapp As Object
Dim olemail As Object
Dim olInsp As Object
Dim wddoc As Object

    On Error Resume Next
    Set olapp = GetObject(, "Outlook.Application")
    If Err <> 0 Then Set olapp = CreateObject("Outlook.Application")
    On Error GoTo 0
    Set olemail = olapp.CreateItem(0)
    With olemail
        .BodyFormat = 3
        .To = "*** Email address is removed for privacy ***"
        .Subject = "Movies Report"
        Set olInsp = .GetInspector
        Set wddoc = olInsp.WordEditor
        wddoc.Range.InsertBefore "Good Morning"
        .Display
    End With
End Sub

ahangama say: Let me explain what I know instead of answering your specific questions which are not clear to me.

OpenType is a descendent of TrueType by Apple. OT is a joint effort of Adobe Apple and Microsoft to standardize electronic type creation. Obviously that would allow programs to interpret fonts according to a common rule set. I speak only about Standard Ligatures here.

A ligature is a replacement for a set of alphabetic letters. English does not have linguistically mandated ligatures but typographical ones, which belong in the domain of the font designer. The most common English typographical ligatures are fl, fi, ffl, ffi etc (I forget momentarily. Pardon the old man). Anyway, These seven are named and given their own codepoints by Unicode. Unicode calls them Alphabetic Presentation Forms.

What if you design a font and have 'ct' designed in some decorative way and want to show that everywhere the user types 'ct'? This you can define as a Standard ligature. Since standard ligatures are just that, they get displayed by default. OpenType allows a program to disable them, though I can't figure for what. (The user selects the font because of its qualities, right?). There are 6400 codepoints set aside for font designers to put their 'crazy' shapes starting at U+E000, if necessary, all 6400. That area is called the private use area (PUA) exclusively reserved for the font.

So, in our example, the font has a lookup table that says, c and t give this [ct] shape. When using this font, you type c and follow it by t, the font replaces the c with the ligature [ct]. [ct] is located in PUA, say, at U+E010.

Fraktur or Black letters is a German set of letters that have regular ligatures, that is, standard ligatures, ligatures mandated by Fraktur writing system. If you select a Fraktur font, you expect the ligatures for those sets of letters that they represent. Without ligatures, it is not Fraktur.
Fraktur: http://commons.wikimedia.org/wiki/Fraktur

Go ahead and make a Fraktur font by placing all those ligatures in PUA and address them using a Substitution Table. Notepad will faithfully show your ligatures and Excel, and AbiWord, PaintShop, Firefox, iPhone, Safari, Chrome, Linux, Macintosh . . . Unfortunately, IE and Word mess them up.

e-mail headers and custom fields - Microsoft Community

jdc_2005 say:

e-mail headers and custom fields

Is it possible to access the value of an e-mail header from a custom field formula?  For example, if I have a header named "X-MYAPP-NAME" set to the value "GOOBER" in a message's headers, can I access that from a formula I'm defining for a custom field?  If so, how?  If not, is there some other way to get that header value to show up as a column in my Outlook view?

Thanks in advance for any help!

 

ken slovak say:
All headers are all in just one long string in a property called PR_TRANSPORT_MESSAGE_HEADERS. There isn't a separate property for each header or x-header. Some properties may be set by headers, but never any x-headers. So that can't be displayed in a view unless you extract the header and stuff the value into an existing property that's unused, such as Mileage or BillingInformation, or you create a user property for that.
 
If you are using Outlook 2007 or later you can use the PropertyAccessor to get PR_TRANSPORT_MESSAGE_HEADERS using the property tag of"http://schemas.microsoft.com/mapi/proptag/0x007D001E".Note that's not an URL but a DASL property tag. You could then get the value as a string and parse the string using string functions to extract your x-header.
 
If you using an earlier version of Outlook you would have to use a different API such as CDO 1.21 or Redemption (www.dimastr.com.redemption) to extract the x-header value.

Old Version of Office in My Microsoft Account

When I log into my Microsoft account, it shows an old version of Office. I bought Office 2016 and have it installed on my desktop PC. Now I'm trying to add it to a new laptop, but it's showing me the old version of Office not the new one I have on my PC. What do I do?

andre for directly say:
Hi Peter

My name is Andre Da Costa; an Independent Consultant, Windows Insider MVP and Windows & Devices for IT MVP. I'm here to help you with your problem.

Office 2016 is a standalone (perpetual), one time purchase. Purchasing it does not entitle you to newer versions such as Office 2019 or Office 365 (subscription). You are also allowed to have installed on (one) device only.

If you want to transfer it or install it on another computer, you need to remove it first from the old computer, install it on the new one then reactivate.

Step 1:

From any computer, go to your Microsoft account, Installs page and sign in using your Microsoft account, if prompted.

https://account.microsoft.com/services/office/i...

Under PC/Mac installs used, locate the computer where you want to deactivate Office.

Select Deactivate to deactivate Office 365 on the PC or Mac.

Step 2:

Step 1 - Sign in to download Office

Go to http://www.office.com and if you're not already signed in, select Sign in.


Sign in with the account you associated with this version of Office. This account can be a Microsoft account, or work or school account.

Forgot your account details? See I forgot the account I use with Office.
https://support.office.com/en-us/article/i-forg...

On the Office home page, do the following depending on your version of Office.

If you signed in with a Microsoft account, from the Office home page select Install Office > Install. (Office 365 subscribers should select Install again).

This begins the download of Office. Follow the prompts on your screen to complete the install.

andre for directly say:
Click Start > Settings > Apps > Startup
Disable all startup programs except Windows Defender

Also:

Try these first to see if they work:

Open Start, type: CMD
Right click CMD
Click Run as administrator

Type in at the prompt OR Copy and Paste these one at a time : (Hit enter after each)

Dism /Online /Cleanup-Image /CheckHealth

Dism /Online /Cleanup-Image /ScanHealth

Dism /Online /Cleanup-Image /RestoreHealth

Also run the System File Checker utility:
Open Start, type: CMD
Right click CMD
Click Run as administrator

At the Command Prompt, type: sfc/scannow

This will check for any integrity violations

Restart your system

Check your hard disk for any errors:
Click Start
Type: CMD, from the results, right click Command prompt
Click 'Run as Administrator'
At the Command Prompt, type: chkdsk /r /f
Hit Enter
When you restart your system, your computer will be scanned for errors and attempts will be made to correct them.
