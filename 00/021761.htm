What you can do to disable the Intel driver is create a basic task run by the task scheduler.

Put following PS command in a file, say disableDisplaydriver.ps1

get-pnpdevice | where class -eq 'display' | select instanceid | disable-pnpdevice -Confirm:$false

Create a basic task to run in the taskscheduler :

Give the task a name and description

set the trigger to When the computer starts

and the following action:  powershell -file <full path of disableDisplaydriver.ps1>

Notes: test the PS command first, you should have only one display class driver installed so it should work.

To re-enable the device driver just run:

get-pnpdevice | where class -eq 'display' | select instanceid | enable-pnpdevice -Confirm:$false

the -Confirm:$false is not needed when testing in a command console.

To run from the command prompt use following syntax ..

powershell -command "& {get-pnpdevice | where class -eq 'display' | select instanceid | disable-pnpdevice -Confirm:$false}"

e.j. gun say:

There are all sorts of problems with your code.

The main problem is that you have not "Dim"ed all of your variables, so you are forcing Excel to guess at what they should be.  This is a bad practice.

At the top of the code module (which I assume is one of your worksheets), add "Option Explicit" and then select Debug/Compile.  Resolve all of the error messages you see about undimensioned variables.

The next thing to do is to make the variable "WSBT" global.  You define the variable in CommandButton1_Click, so it is local to that routine.  Then you use it in DrawJoin, but DrawJoin does not have access to WSBT because it is local to CommandButton1_Click.  To fix this, add a dimension statement at the top of the module (right after Option Explicit):

Dim WSBT as Worksheet

This will ensure the all routines in that module see the same WSBT and can share the value in that variable.

Finally, if you want to be sure that you are applying your code to the correct worksheet ranges, you should prefix all range references with the worksheet.  You could do something like:

With WSBT ' This is the worksheet that all ranges should use

  LX = .Range(.Cells(1, Col).Address).Left

  RX = .Range(.Cells(1, Col + 3).Address).Left

' etc - note the use of ".", which references the object in the With statement

End With

Once I made these changes, your code seemed to work...

Hope this helps,

Eric

Whether you can or cannot sync your contact/calendar info on multiple devices depends on the protocols supported by your email service provider for your account.

In a nutshell, if you're account is configured on your desktop as either POP or IMAP, you cannot sync mutliple devices by connecting to email account. Not a limitation of Outlook - it's the way the email protocols work.

Your two choices are:

#1 - Create an Outlook.com account purely to store your contact/calendar data and configure it on each of your devices using Exchange/Exchange ActiveSync protocols (default protocols used when configuring the account via auto-discover - entering your email address/password)

a. user say:

Hi CharlesThurman

 

Thank you for posting on Microsoft Answers.

 

Currently there is no setting within the Music app on Windows 8 to crossfade.

 

But you can do this through Windows Media Player.

 

1) Start playing a song in WMP

2) Click on the Now Playing Mode (three small squares in the bottom right corner)

3) In the now playing mode, right click in any empty space and select Enhancements

4) In Enhancements, select Crossfading and auto volume leveling.
