tom ogilvy say:

the macro runs everytime the worksheet calculates.  With the original change macro, the event passed in a reference ot the cell that changed and triggered the macro.  Not the case with the Calculate event.   Calculate happens across the whole sheet for any cell that needs to be calculated.  So the code processed AE17 on every calculate.

That said, my alteration of the code to work with calculate was incomplete.  I still used target in one place.  This should have been replaced.  Here is the corrected code which I tested and it worked fine for me.  Sorry for the oversight on my part and hopefully it didn't cause you too much frustration.

Private Sub Worksheet_Calculate()
     Dim i As Long, r As Range
     On Error GoTo ExitSub
     Application.EnableEvents = False
     Application.ScreenUpdating = False
     Set r = Sheets("Form A").Range("AE17")
         For i = 1 To 41
             Sheets("PPP").Columns(1 + 3 * i).EntireColumn.Hidden = (UCase(Trim(r)) Like "T*&*M")
         Next i
ExitSub:
     Application.EnableEvents = True
     Application.ScreenUpdating = True
 End Sub

so I change

(UCase(Trim(Target)) Like "T*&*M")

to

(UCase(Trim(r)) Like "T*&*M")

at a minimum, changing the value in C9 should cause a calculate and the event will fire. 

If the entry is in C9, you may want to go back to Vijay's original macro (adapted to use C9) and use that.  You choice.  

------------------------------

Private Sub Worksheet_Change(ByVal Target As Range)
     Dim i As Long, r As Range
     On Error GoTo ExitSub
     If Intersect(Target, Me.Range("C9")) Is Nothing Then Exit Sub
     Application.EnableEvents = False
     Application.ScreenUpdating = False
    Set r = Worksheets("Form A").Range("AE17")
    If UCase(Trim(r)) Like "T*&*M" Then
         For i = 1 To 41
             Sheets("PPP").Columns(1 + 3 * i).EntireColumn.Hidden = True
         Next i
         Else
         For i = 1 To 41
             Sheets("PPP").Columns(1 + 3 * i).EntireColumn.Hidden = False
         Next i
     End If
ExitSub:
     Application.EnableEvents = True
     Application.ScreenUpdating = True
 End Sub

I tested that alteration and it worked as well.

Again, sorry for my oversight in adapting the original code.

tangerine9 say:

How do I get a macro to reference a variable filename to perform actions on that file?

Quite new to VBA, but had a bit of practice in recording macros!

 

I'm using this code to enable a user to select a file to open:

    Workbooks.Open fn

From here I want to perform actions between the newly-opened file and the original file (just copy & paste etc).  I know this should be quite simple but I don't know how to keep the filename variable.

 

Many thanks.

jllatham say:

Ok, this may get deep as you progress with your project.  You probably need to study up a little on objects.

But to begin with change your code to this (changes in bold):

Sub OpenOneFile()
Dim fn As Variant
Dim wbk As Workbook
    fn = Application.GetOpenFilename("Excel-files,*.xls", _
        1, "Select client weekly tally", , False)
    If TypeName(fn) = "Boolean" Then Exit Sub
    ' the user didn't select a file
    Debug.Print "Selected file: " & fn
    Set wbk = Workbooks.Open(fn)
End Sub

as soon as you do that, the new workbook will become the Active Workbook.  You can move back into the initial workbook with a statement like

ThisWorkbook.Activate

Both workbooks will be open.

You can now reference the other workbook using the wbk object.  Sheets within it could be referenced like:

wbk.Sheets("sheet name")

and a cell or cells within the wbk workbook can be referenced like

wbk.Sheets("sheet name").Range("A1")    and   wbk.Sheets("sheet name").Range("A1:Z34")

Now, if your code is going to lead you to jumping back and forth between the workbooks and various sheets within both books, you'll probably (as an assumed novice to VBA and objects) do a lot of

wbk.Activate
wbk.Sheets("sheet name").Select

and

ThisWorkbook.Activate
ThisWorkbook.Sheets("sheet name").Select

which can make someone dizzy (and is pretty slow to get done).

So you can eliminate the screen flashing and refreshes by including this statement before you start all of that:

Application.ScreenUpdating = False

You don't have to worry about setting it back to =True as that will happen automatically when you exit the Sub.

But to totally prevent all of that and really speed things up, if you can define other objects to reference worksheets and ranges on those sheets, you'll be much better off.  And once you do it, you don't have to .Activate or .Select any of them!

The Outlook AB has only 2 sort orders

1) First / Last

2) FileAs

So.......................

Step #1 - the default FileAs setting must be set to Last, First

From OL menu --> File --> Options --> People --> set FileAs accordingly

*** note: this setting will only take effect the next time you restart Outlook for new contacts created. It will not change the FileAs value for any pre-existing contacts

You can do that manually by following the steps in

 

Change Contact's File As format
http://www.slipstick.com/contacts/fileas.asp

If you don't want to do it using the above approach, you may find the FileAs standardization function in ContactGenie Toolkit to be if interest which provides 13 different formats

ContactGenie Toolkit (not free)

http://www.contactgenie.com/toolkit_4_outlook.htm 

Step #2 - Change the AB sort order

File --> Account Settings --> AddressBook tab -->highight Outlook AddressBook --> click on Change button --> change sort order accordingly
