Well... data is not stored in Forms, but in Tables; and data is not stored in tick boxes (that's not even an Access control types), but in Fields in the table. What is the structure of the Table upon which your form is based?

Actually I mentioned 3 tables. You have a many to many relationship. One patient can exhibit multiple symptoms and one symptom can apply to multiple patients. The table I described is a "junction" table that models that relationship.

rohn007 say:
MS assumes that when we 'upgrade' to a newer version we will no longer have use for the older version. Prior to 2016 the default was to uninstall the other version, but we had the option of doing a custom install and keeping the older version. When 2016 was first released, there was no custom install and the older version was always uninstalled. Some time in the last few months MS changed that default (without notice) so now the older version is kept.

To go along with that old assumption, Office programs have a built in check they run when started to see if they are set as default. If not, they trigger that configuration process. It sets all of the applications in that specific bundle as default.

Long ago the MS Developers needed to run more than one version at a time so they created a "NoReReg" (no re register) Registry Key. Users can also use that key.

Here are instructions to do it:
Dual Install - Working with Multiple Versions of Office PRE 2016! – NoReReg
http://answers.microsoft.com/en-us/office/wiki/...

https://office.microsoft.com/en-us/word-help/ke...
This tip describes the specific steps and choices during the 2010 custom install. After you have the multiple versions installed you still have some additional steps to do. They are described in this tip:
https://web.archive.org/web/20130603071131/http...
With multiple versions installed, there is one issue you can’t avoid, Default Windows File Associations. Only one app can be used to open the files by double clicking in Windows Explorer. You have to pick which one.
FYI: NoReReg key works with Office 2010 / 2013 installed on Windows 7 and 8 (I tested it myself Rohn007)
Note: in user accounts, when you log in to do the NoReReg hacks, DO NOT start REGEDIT using “Run As Admin”. You will be editing the wrong “Current User” key <grin>

After adding the NoReReg keys, you must do an Office Repair of the version you want to use as default.
