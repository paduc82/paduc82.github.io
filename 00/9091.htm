The Nokia N8 uses Symbian/S60 as operating system, which uses RTSP for YouTube. Other mobile platforms use different protocols and therefore might not be affected. Some Internet Service Providers (ISP) do block certain protocols, especially if they are using a shared medium like Wi-Fi for connecting their customers – for example, when you stream a lot of data, all other users get a degraded service. lordalukard say:
Dear Gillian,
The green shield is not something that can be bought. It is a security feature that checks multiple things which I explained in previous message. If you want your email or domain to be recognized as safe sender you need to be careful with content of the emails, subjects, mass sending is not an option, attachments and other things ( that only developers know ),
Also your receivers should not mark your emails as spam, add them to safe lists, The more emails receivers mark as spam or junk the harder it is to receive "green shield".

lordalukard say:
Dear Gillian,
It is hard to say the specific amount of time as it is very complex process and there is a huge amount of data which is analyzed in real time. Additionally it can vary on each individual account as it takes each preference into consideration. scott mcdaniel say:

It looks like you're using Early Binding, which means you must insure that the version of Outlook on your development machine is the same or earlier as the versions on your deployed machines. Access can "upsize" your references, for the most part, but cannot "downsize" them.

You can also convert to Late Binding to make your code somewhat version-independent. To do that, you would convert calls like this:

Dim outApp As Outlook.Application
Dim outMsg As Outlook.MailItem

To this:

Dim outApp As Object

Dim outMsg As Object

Set outApp = CreateObject("Outlook.Application")

Set outMsg = outApp.CreateItem(olMailItem)

After doing that, you can remove the hard reference to Outlook, and your code will use whatever version of Outlook is installed on the machine.

If you use Late Binding, you must be sure to use only those features that are available in the "lowest" version of the library you're going to be supporting. So if you must support Outlook 2000 and forward, you must not use any features that were introduced in later versions, since that would cause your code to fail on machines running Outlook 2000. You must also include any constants you're using, like this one:

olMailItem

Since you wont' have a reference to Outlook, you would have to use the actual value of that constant, or declare that constant at some point before you use it:

Dim olMailItem = 2

Note that you do NOT get Intellisense with Late Binding, so many devs will use Early Binding during development, and then switch over to Late Binding when it's time to deploy.

Here's a link that describes a bit more about Late Binding: 

https://support.microsoft.com/en-us/kb/201096
