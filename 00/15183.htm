ken sheridan say: As the column is a multi-valued field you need to include the Value property in the expression.  Moreover, as in this case the values appear to be hidden numeric keys you need to use the numeric value corresponding to the 'OPS' text value, e.g.

[Recipient].Value = 1

You can also use the IN operator:

[Recipient].Value IN(1,2,3)

which will filter the form to those records where the Recipient column includes the values 1, 2 or 3 in its list of values.

However, unless you really need to use a multi-valued field, e.g. if interfacing with SharePoint in the very specific context in which this type of field is necessary, I'd recommend that you model the many-to-many relationship type by the conventional means of a table which resolves the relationship type into two one-to-many relationship types.  Multiple values per parent record would then be entered via a subform rather than a multi-valued control.  You'll find an example as FindRecord.zip in my public databases folder at:

tom van stiphout say:

Your instincts about M:M are correct, and most serious db designers shun the multivalue field because it hides the relationship and is not portable to other database structures, including MSFT's own SQL Server (hint, hint).

 

The standard UI for a M:M relation is with a form on the one-side, usually in single-record view, in your case Surveyors or Stratums. Then a subform on the junction table. The LinkMasterFields/LinkChildFields property will automatically populate the related field from the parent form, and a dropdown list on the other one-side table will populate the other field. You can see this in the OrderDetail form in Northwind sample app, which is the junction table between Orders and Products.

 

IIUC you wanted to select multiple surveyors and enter a stratum and have it record the junction for all of them. This would require custom programming that would create the several records in the junction table. For example you could allow user to multi-select surveyors, pick a stratum and click a button. You then loop over the selected items and append a record for each one.

HI Carl

Repairing Office will never resolve issues related to Outlook profile. Just as a FYI, Outlook profiles persist any uninstalling/reinstalling of Office (Outlook) - something to keep in mind for the future.

To check any existing or create a new Outlook profile, open the Control Panel <Mail> app.

- clicking on the <Email Accounts> button will show the email accounts configured in the currently active profile
- clicking on the <Data Files> button will show all the data files configured in the profile
- clicking on the <Show Profiles> button will allow you to (1) check any existing Outlook profiles or (2) creating a new Outlook profile. If you create a new Outlook profile, also be sure to set it as the "default" profile for the next time Outlook opens.

A side comment, if you are repeatedly running into the situation where Outlook is not finding/unable to open the current default profile, then there is a larger issue at play that needs to be addressed.

Hopefully, the Control Panel <Mail> app will be able to display the currently configured email accounts and data files so you can take note of that information. There's no need to delete any existing profile before you are back up and running.

Other info to be aware of

#1 - If you have POP3 accounts configured, take note of the PST file where the <Inbox> email is being delivered to so that re-use the same file when configuring the account again
#2 - If you have IMAP accoubts configured, take note of what PST data file is set as the "default" (under the Data Files tab) since that's where the default contact/calendar folders are stored. The PST file used for the email folders will be recreated by Outlook when the account is configured and the email folders will be re-sync'd with the server
#3 - No need to take any action for Exchange based accounts since Outlook will create a new OST file and sync all data with the server.

One thing that you may want to do is to run ScanPST on all your PST files just to eliminate the possibility of there being issues with a PST file causing Outlook to have trouble opening up.

The article below describes how to repair PST and OST files but personally, my recommendation would be to allow Outlook to recreate any OST file whih is done by renaming an existing OST file (with Outlook closed).

Repair Outlook Data Files (.pst and .ost)
https://support.office.com/en-us/article/repair...
