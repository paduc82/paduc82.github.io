bob jones aka: cybertaz say:
Well, let me try explaining this from a different direction :-)

First point of clarification -- custom Themes are not saved in any one program. Regardless of which app is used to create the custom Theme it is saved as a separate file in a default location [My Themes] which is globally accessed by all programs in the suite. The only difference is that the Themes in PPT also include graphic elements (such as slide backgrounds) not supplied to Word or Excel documents. IOW, Themes in Word & Excel comprise only Theme Fonts & Theme Colors.

Interjection: If you're just attempting to select just your custom Color Scheme, you cannot do it from the Themes button on the Home tab in Word. You need to go to the Layout tab. There you'll find bot a Fonts button as well as a Colors button. When you click the Colors button your custom Color Scheme will be selectable at the top of the drop down list.

You cannot load the individual .xml files that are stored in the Theme Colors & Theme Fonts folders, nor is there any reason to do so. They are not Themes. They are support files subordinate to the Themes [.thmx files] that use them. You're correct that Word cannot open those files for editing or revision. If you know the XML language well enough you could use an XML editor for that purpose, but Word doesn't provide that capability.

In the same vein, you cannot open the Theme files, either. If the Theme [.thmx] is stored in the default location -- which is the My Themes folder -- it will be selectable from the drop down for Themes in any of the Office apps. [Note that a new custom Theme will not appear instantly in the drop down of the other programs if they are running when the Theme is created. The Themes files are read on launch of each program.] There is no need to use the Browse Themes option, however... It's only if you chose to save the custom Theme in a location other than My Themes that you should have to Browse for it.

Essentially, once a Theme is saved the apps only read the Theme [.thmx] files in the background in order to determine what fonts & color schemes to provide in the document.

Add a link to an email using Link Preview

In Outlook on the web, when you write an email and add a URL, or when you receive an email with URL inside it, you get a rich preview that includes a link title, thumbnail, and description of the link. This is called Link preview.

Instead of this: http://www.microsoft.com

With Link preview, you get this:

Note: Link preview is turned on automatically. If you are not seeing a link with a thumbnail, title, and description like the one shown above, you will want to turn it on. If that doesn't work, see your IT administrator.

Add a Link preview

  1. In a new email, type the URL of the site you want to provide a Link preview for. For example: http://microsoft.com

  2. Link preview will automatically go to the destination URL and provide the site's name, description, and thumbnail (if there is one available) in the email.

Turn Link preview off

Link preview is turned on automatically, however you can turn it off.

  1. On the Navigation bar, in the upper right corner, click Settings .

  2. In the Options pane, under Mail > Layout, click Link preview.

  3. Under Link preview, uncheck the Preview links in email box to turn it off.

For a little while I was using the remote desktop function almost daily to connect back to a laptop in my home office. The following fields are required:
Computer: The External IP address of your network. You will then need a NAT to carry the connection through to the computer
Username: this will be a user name that has remote desktop control enabled on your desktop (which is not always by default depending on your OS)
Password: Password for above account
Domain: this is blank. but that depends. If your using a computer on a Domain, then you will need to have the domain listed here. However, if your office desktop is just part of a work group, then leave this blank.
