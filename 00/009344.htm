bill manville say:

My suspicion is that the Windows 8 difference with 2013 is entirely due to the difference in TwixPerPixel between the systems on which we were running Windows 8 and those on which we were running Windows 7. 

Experimenting with a 15 twipsPerPixel system in 2013, you can enter any line width you like in the box and the change in number of pixels occurs at 3.375pt, not at 3.5.  I suspect there are 2 levels of rounding going on - the first to 0.25 pt. and then the inconsistent rounding from that value (3.5) to pixels.  So, to force the thinner line for a 3.5pt line you would set the width to, say, 3.37 pt.

This will not give consistent numbers of pixels between systems with different TwixPerPixel settings though.  The question then is whether you want a fixed number of pixels or a fixed width in points across those systems.  If you want a fixed number N of pixels then set the width to N * TwipsPerPixel / 20 points.  If you want a fixed width you're out of luck because only certain widths in points are possible.

To get the TwipsPerPixel setting for your display you need to use a Windows API.
This version is good for Excel 2010 and later:

Declare PtrSafe Function GetDC Lib "user32" (ByVal hWnd As LongPtr) As LongPtr

  (ByVal hdc As LongPtr, ByVal nIndex As Long) As Long
Declare PtrSafe Function ReleaseDC Lib "user32" (ByVal hWnd As LongPtr, ByVal hdc As LongPtr) As Long
Private Const HWND_DESKTOP As LongPtr = 0
Private Const LOGPIXELSX As Long = 88
Private Const LOGPIXELSY As Long = 90

Function TwipsPerPixelX() As Single
  Dim lngDC As LongPtr
 
  lngDC = GetDC(HWND_DESKTOP)
  TwipsPerPixelX = 1440 / GetDeviceCaps(lngDC, LOGPIXELSX)
  ReleaseDC HWND_DESKTOP, lngDC
End Function

There is a similar function substituting X for Y, but I have not yet come across a system in which the TwipsPerPixel differ between the X and Y directions.

Hi, the only thing that does not to seem to change is change itself, some often say. I have changed a set of old menu/toolbars into the new ribbon style, it is so easy than i thought! Try these four steps:

Step 1- Create Macros that open the forms, reports or that execute queries or codes that will open or do what ever action you need to execute or accomplish. For instance, if you want the menu item to open a form in your database, create a macro with OpenFormwith the name of the form that needs to be open.

Step 2- Right click somewhere on Access menu bar and select Customize The Ribbon. 

Step 3- Click on New Tab, a new tab and new group should appear. Then change their names to the desired respective names (usually those from your old menu/toolbars)

Step 4- In Choose Commands From, select Macros; this is where the trick is. Point or select the macro you created in step 1. Repeat for reproducing each of your old menu items. I had a couple of dozen menu items, it took me less than 1hour.
