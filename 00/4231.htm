Hi,

Possible causes of the problem are:

1.       Conditional formatting rule may be applied to the cells.

2.       A custom cell style may be applied to a cell / cells.

3.       Default cell style Normal may be modified for number from general to date.

4.       Possible file corruption.

Solution to the specified problems:

Note: Please try these troubleshooting steps on a duplicate file created by using “Save as” option on the original file.

Method 1: Finding the Formatting applied across the sheet and Clearing Conditional Formatting.

1.       Click on ‘Conditional formatting’ under Home tab and then on ‘Manage Rules…’

2.       Now select’ This Worksheet’ for ‘Show formatting rules for:’

3.       Check if  any rules are created for ‘This worksheet ‘, if yes then open the rule and see whether the rule is set to change the cell format to other format and if needed change/delete the rule according to your requirement.

Method 2: Modifying the default /custom style in Excel.

1.       Click ‘Cell Styles’ under Home tab and check if you have a Custom style created,

 

If yes then right click on the Custom style created, select modify and click format, now select General under Number tab, click OK and OK again on the Style window.

 

If No then right click on Normal and check if Number has General format if not then select modify and click format, now select General under Number tab, click OK and OK again on the Style window.

Method 3:  Try to repair the file.

 

Note: Do not try to repair the original file, repair a copy of the file instead.

  1. Open Excel, click the office button > Open > In the Open dialog box, select the file you want to open, and click the arrow next to the Open button. Click Open and Repair, and then choose Repair to repair the file.

Check for more information on file repair techniques and recovering data from the corrupted file by clicking on the link below.

 

http://office.microsoft.com/en-us/excel/HA100970171033.aspx?pid=CH100948241033

 

As an alternate, select the cells/range where the data is displayed incorrectly and click on clear format to clear any unwanted formatting in the cell.

ossiemac say:

I have had problems with trying to obtain the date criteria from a value list. I have asked on previous Microsoft news group without success. The only way I have been able to achieve it is to create my own array from the visible cells.

The following example might help. It is from one of my test examples and includes testing for the Operator so know what criteria to extract and then what to set in the new autofiltered range. (The testing only re-sets its own autofiltered range but I am sure you will get the idea from it.)

Note that for Operator xlFilterValues which is 7, if dates are present then Criteria2 holds the data but for other lists Criteria1 holds the data. Test the column for dates and if not dates then Criteria1 can be saved.

The code calls another sub to create the date array from the visible cells in the column. This is the only way I know of extracting that criteria. Every other way I have tried returns an error.

I'll be interested to hear how you go with it. Also if anyone else knows how to directly extract or save the data from Criteria2 when it holds a list of dates then I am interested also. The array actually has twice as many elements as the dates because the intermediate elements have a 2 in them.

'Following variable must be in Declarations area
'at top of a standard module so it is available
'to all subs.
Public arrCriteria2()

Sub ValueListFilterDates()
Dim ws As Worksheet
Dim i As Long
Dim rngForCriteria As Range
Dim varOperator As Variant
Dim strCriteria1 As Variant
Dim strCriteria2 As Variant

Set ws = ActiveSheet

If ws.FilterMode Then
  ws.ShowAllData
End If

'Code as recorded to filter on _
 valuelist of dates. _
 Note: In d/m/y date region the _
 dates are recorded in m/d/y/format. _
 Set the filters on column 2 (Dates) for testing.
ws.Range("$A$1:$D$185") _
  .AutoFilter Field:=2, _
    Operator:=xlFilterValues, _
    Criteria2:=Array(2, "7/3/2009", _
        2, "8/5/2009", _
        2, "9/10/2009", _
        2, "10/11/2009", _
        2, "11/14/2009", _
        2, "12/6/2009")

'Test for type of operator so know which _
 Criteria is used in the filter and _
 how the criteria is to be saved.
With ws.AutoFilter
  With .Filters(2)
    If .On Then
      varOperator = .Operator
      Select Case varOperator
        Case 0
          'No Operator. Single selection only
          strCriteria1 = .Criteria1
          strCriteria2 = ""
       
        Case xlOr
          'Logical Criteria1 OR Criteria2.
          strCriteria1 = .Criteria1
          strCriteria2 = .Criteria2
       
        Case xlFilterValues
            'Test for type of data
            Dim tempVariable
            tempVariable = ws.AutoFilter _
              .Range.Columns(2) _
              .Cells(1, 1).End(xlDown)
            If VarType(tempVariable) = vbDate Then
              'Is date column
              varOperator = "Date List"
            Else
              'Set strCriteria1 to .Criteria1 _
               because not dates
              strCriteria1 = .Criteria1
            End If
        Case Else
          MsgBox "Could not identify complex criteria." & _
            vbCrLf & "Prosessing terminated."
          'Can test for other complex criteria _
           but not included in this example.
          Exit Sub
      End Select
    Else
      'Filter not On so set to no filters _
       'Don't use False because False and Zero _
        as used for Single value Filter are same value.
      varOperator = "No Filters"
    End If
  End With
End With

If varOperator = "Date List" Then
  'Create an array of the Date Criteria _
   from the visible cells of data.
 
  With Sheets("Sheet1").AutoFilter.Range
    Set rngForCriteria = .Columns(2) _
      .Offset(1, 0).Resize(.Rows.Count - 1, 1)
  End With
  '*********************************
  'Create an array from the visible cells.
  Call DateCriteria(rngForCriteria)
  '*********************************
End If

'Following is test only to reset filters _
 using the Operator and Criteria values.
If ws.FilterMode Then
  ws.ShowAllData
End If

Select Case varOperator
  Case "No Filters"
    'Code to ShowAll
  Case xlOr
    'Code to set operator, Criteria1 and Critera2
  Case xlFilterValues
    'Code to set operator and Criteria1 which is array
  Case "Date List"
    'Code to set operator and criteria2 which is array
    'Assign array to variant
    strCriteria2 = arrCriteria2()
   
    ws.Range("$A$1:$D$185") _
      .AutoFilter Field:=2, _
        Operator:=xlFilterValues, _
        Criteria2:=strCriteria2
End Select

End Sub

Sub DateCriteria(rngFiltered As Range)
'This sub creates an array of the filtered dates _
 from the visible cells in the filtered column.
Dim c As Range
Dim i As Long
Dim j As Long
Dim arrTemp1()
Dim arrTemp2()
Dim dateTemp
Dim arrCounter
Dim arrElement

Set rngFiltered = rngFiltered _
      .SpecialCells(xlCellTypeVisible)

'ReDim arrTemp1() to one based array
ReDim arrTemp1(1 To rngFiltered.Cells.Count)

'Copy entire filtered range to arrTemp1()
For Each c In rngFiltered
  If Rows(c.Row).Hidden = False Then
    i = i + 1
    arrTemp1(i) = c.Value
  End If
Next c

'Sort ascending arrTemp1()
For i = 1 To UBound(arrTemp1) - 1
  For j = 1 To UBound(arrTemp1) - 1
    If arrTemp1(j) > arrTemp1(j + 1) Then
      dateTemp = arrTemp1(j)
      arrTemp1(j) = arrTemp1(j + 1)
      arrTemp1(j + 1) = dateTemp
    End If
  Next j
Next i

'Create unique array of values in arrTemp2
arrCounter = 0
dateTemp = 0
For i = 1 To UBound(arrTemp1)
  If arrTemp1(i) <> dateTemp Then
    arrCounter = arrCounter + 1
    ReDim Preserve arrTemp2(1 To arrCounter)
    arrTemp2(arrCounter) = arrTemp1(i)
  End If
  dateTemp = arrTemp1(i)
Next i

'ReDim arrCriteria2() to one based array and _
 multiply by 2 to allow for 2 to be inserted _
 as alternative values between date elements.
ReDim arrCriteria2(1 To UBound(arrTemp2) * 2)

i = 0
For Each arrElement In arrTemp2()
  i = i + 1
  arrCriteria2(i) = 2
  i = i + 1
    'Format may not be required in region _
   with m/d/y/format but might still be _
   required to provide string format.
  arrCriteria2(i) = Format(arrElement, "mm/dd/yy")
Next arrElement

End Sub

john mcghie say:
Hi John:

The formatting of anything in Word is performed by Styles.  So any proper answer to your question would necessarily talk about modifying styles.

In a template, the trick is to ensure that the changes to the styles are actually stored in the template, and not in the user's Normal Template.  Anything in the Normal.dotm template exists only for that specific user on that specific machine.  To pass settings through to a template you are making, you need to ensure that your changes are stored in the template you are working on.

The best way to do that is to close all other documents, and then use File>Open within Word to ensure that you get the template itself open and that it is the only file open.

The two styles you will be interested in are the Footnote and Footnote Reference styles.  "Footnote" sets the formatting for the footnotes, "Footnote Reference" sets the formatting for the reference mark for the footnote as it appears in the body text.

Neither style will appear in the list of styles until it has been used in the document, so if it does not appear, insert a footnote and type some junk text in it.  You can take it out again later.

Hope this helps
