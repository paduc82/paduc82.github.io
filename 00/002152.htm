dirk goldgar say:

Liz -

I've been working with your database and find that, now that I understand exactly what you're doing, I can reproduce the problem in both Access 2013 and Access 2010.  I hadn't realized -- though I should have -- that your subform is in datasheet view, so the only means to directly call up the Conditional Formatting dialog is by right-clicking the column header.

The issue appears to be a bug in Access, specifically related to the display of the color-selection dialog.  When the form is displayed as a popup, the color-selection dialog is not displayed properly.  On my system, there's a little "blip" at the bottom of the color-selection button when the selection dialog ought to be displayed;  see this screenshot:

Do you see the tiny blip that the (crudely drawn) arrow is pointing to?  When the form is not in popup mode, the proper color-selection dialog is shown:

This happens regardless of whether the datasheet is a subform or a main form.  Clearly, the problem is with the display of the popup color-selection dialog when the form is in popup mode.

I can think of two workarounds for this.  The first, and most straightforward, is to create the conditional formatting rule when the form is in either Design view or Layout view.  In those views, the form not in popup mode.  In both of those views, the conditional formatting dialog is available via the Format tab on the ribbon.  In Design view, you can also get to it by right-clicking the control, though this is not available in Layout view.

That approach won't work if you want your users to be able to set these conditional-formatting attributes on the fly, at run time.  In that case, the only other workaround I can think of -- until Microsoft fixes this bug -- is to use VBA code to create the conditional formatting conditions.  But that would involve creating an elaborate supporting structure to allow the selection of colors and other attributes, so it would be a pain to implement.
