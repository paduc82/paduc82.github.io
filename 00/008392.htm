jeeped say:
If the text file will not import into Field2, it is almost assuredly one of the following:

  • Duplicate entry in text file which cannot be used in a Primary Key field which has a constraint of Indexed: Yes (No Duplicates). This would also include two or more blank lines. It would also include an entry that pre-exists in Field2.
  • One or more blank entries in the text file that conflict with a Primary Key constraint of Allow Zero Length: No.
  • Field3 and/or Field4 having a constraint of Required: Yes, Allow Zero Length: No and/or Indexed: Yes (No Duplicates).

So the problem could be in either Field2 or one or both of Field3 and Field4. Remember that by appending values into Field2 you are essentially automatically creating Null or blank values in Field3 and Field4. Field1 should not be an issue as the Autonum will fill as necessary.
