michael vizcaino say:

I had this problem on my PC and Laptop.  PC running Windows 10 and Laptop running Windows 7.

The Win10 PC had no HomeGroup and it could not be created.  The Win7 laptop had an old existing HomeGroup that I could not leave.  

Same solution resolved both issues.  Turns out some application, probably my 3rd party security software, disabled 3 services vital to proper HomeGroup operation.

Peer Name Resolution Protocol
Peer Networking Identity Manager
Peer Networking Grouping

All 3 services were disabled on both machines, and this was preventing HomeGroup from working properly.

To fix it:

1. Run "Services.msc" (without the quotes)

2. Locate the following services and make certain they are all running

    HomeGroup Listener

    HomeGroup Provider

    Peer Name Resolution Protocol
    Peer Networking Identity Manager
    Peer Networking Grouping

3. If any are not running, right click the service name and select Properties

4. Set Startup Type to "Automatic" and apply

5. Click Start in the same window, and Service Status should show "Running"

Note: Peer Name Resolution Protocol & Peer Networking Identity Manager must be running before Peer Networking Grouping can be started. 

I hope this helps someone resolve their issue.

Starfleet, I think I found a solution you can try.  It's a little crazy so it just might work.  I found this on another thread with people having quite similar issues.  I did this, and successfully got OneDrive to start syncing all of my files.  However, some users reported that these changes were undone after a reboot or two.

**DISCLAIMER**: This involves editing file/folder permissions.  You must be on an Administrator account to do these steps.  Also,  I am not responsible for anything that breaks, sorry.  Do this at your own risk.

  7. Press Advanced
  8. Place a check in the box that says "Replace all child object permission entries..."
  9. Press OK, then OK again.

May take a few minutes, but it should start syncing the data properly.  I'm still syncing my data as of the writing of this post, so I cannot verify whether the settings will stick for sure after a reboot, but at least this is a step in the right direction.
