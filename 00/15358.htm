chip pearson say:

You need to use a VBA function, as there are no built-in functions for working with colors. Try code like

Function SumColor(RR As Range, ColorNdx As Long, _
    Optional OfText As Boolean = False) As Double
Dim R As Range
Dim D As Double
For Each R In RR.Cells
    If OfText = False Then
        If R.Interior.ColorIndex = ColorNdx Then
            D = D + R.Value
        End If
    Else
        If R.Font.ColorIndex = ColorNdx Then
            D = D + R.Value
        End If
    End If
Next R
SumColor = D
End Function

Here, RR is the range of cells to test, ColorNdx is the color index value to test for, and OfText indicates whether to look at the Font color of a cell (OfText = True) or the background fill of a cell (OfText = False). So, to sum the values in A1:A10 that have a red (colorindex = 3) background, use

=SUMCOLOR(A1:A10,3,FALSE)

If you want a list of colors and their index numbers, open a blank worksheet and run the code

Sub AAA()
    Dim N As Long
    For N = 1 To 56
        Cells(N, 1).Interior.ColorIndex = N
    Next N
End Sub

The row number is the same of the color index number. Or, if you have a cell with the desired color, select the cell and run

Sub BBB()
    MsgBox ActiveCell.Interior.ColorIndex
End Sub

Note that these functions do not work with colors applied by conditional formatting. They are only for the basic format of a cell. Seehttp://www.cpearson.com/excel/colors.aspx for lots more information and code for working with colors.

Two problems with your code... one easily solved the other can't be solved (at least not without continually running an OnTime event or evoking some kind of API subclassing code). Because changing a cell's interior color or its font color does not generate any VB events, your function, when used as a UDF, will not "update" on its own. Pressing F9 to recalculate the sheet will force the SUMCOLOR function to update, but only if you add an Application.Volatile statement to your UDF. After doing that, the user will have to remember to press F9 after changing the color of a cell or its font. As it turns out, clearing a cell's format (Edit/Clear/Formats) will update the value returned by the SUMCOLOR UDF automatically (as long as you use Application.Volatile).

rick rothstein, mvp - excel say:

 If you decide to go the UDF route, here is an alternative function for you to consider (it should execute about as fast as possible given that it only iterates cells within the range having the indicated color as opposed to interating every cell in the range)...

Function SumColor(R As Range, ClrIdx As Long, Optional FontColor As Boolean) As Double
  Dim Result As Range, FirstAddress As String, Addr As String
  Application.Volatile
  Application.FindFormat.Clear
  If FontColor Then
    Application.FindFormat.Font.ColorIndex = ClrIdx
  Else
    Application.FindFormat.Interior.ColorIndex = ClrIdx
  End If
  FirstAddress = R.Find("*", searchformat:=True).Address
  If Len(FirstAddress) Then
    Addr = FirstAddress
    Do
      SumColor = SumColor + Range(Addr).Value
      Addr = R.Find("*", After:=Range(Addr), searchformat:=True).Address
      If Len(Addr) = 0 Then Exit Do
    Loop While FirstAddress <> Addr
  End If
End Function

NOTE 1: Like Chip's UDF, this one also provides a optional argument allowing you to sum either by interior colors or by font colors.

NOTE 2: If you change the interior color or font color of a cell, then you must press F9 to force the SUMCOLOR function to update its value (see my comments to Chip in a previous message).

jay freedman say:

Press Alt+F9 to view field codes instead of field results. The TOC field code should look like this:

{ TOC \o "1-3" \h \z \u }

If the \h switch is missing, edit the code to insert it. With the cursor in the field, press F9 to update the field, and then press Alt+F9 again to redisplay the field results.

Ordinarily, the only way the field would not have the \h switch is if you used References > Table of Contents > Custom Table of Contents to open a dialog, and you unchecked "Use hyperlinks instead of page numbers". That's appropriate for a document that will be posted as HTML on a web page.

a. user say:

Hello ShawnaMahmoud:

 

Thank you for posting to the forum regarding your issues with MMS.  MMS is something that is controlled by your mobile operator and your data connection.  The issues you are having are likely related to the current settings on the phone or the service provider itself.

 

The first thing you should do is check the current APN settings on the phone.  You will want to contact your service provider to verify the proper APN settings for MMS and inquire about alternate APNs for this service.  Once you have this information, go to settings > cellular/data > add APN. 

 

If you continue to have issues with sending and receiving MMS, you should talk to your provider regarding why this is not working.  Inquire about blocks, limits, etc...

 

Please let us know if you have any additional questions or concerns.

 

Mark W.
