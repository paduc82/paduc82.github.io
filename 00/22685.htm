ken sheridan say: Start by making copies of both the front end and back end files.  In the copy of the front end use the linked table manager to refresh the links to the copy of the back end.  You will then be able to delete all rows in the back end from those table which you wish to empty via the new front end.  As Scott points out you might want to leave the data in some tables, e.g. a Cities table or similar, where that data will be useful to your colleagues in the other university.

When deleting rows from a table, if referential integrity has been enforced, along with cascade deletes where appropriate, then deleting rows from the referenced table in a relationship will automatically delete the rows from the referencing table.  If cascade deletes have not been enforced then rows must be deleted from a referencing table in a relationship before rows are deleted from the referenced table.

When sending the copies of the front and back end files to your colleagues you might wish to include a mechanism for them to refresh the links without recourse to the linked table manager.  You might like to take a look at Refresh.zip in my public databases folder at:

https://onedrive.live.com/?cid=44CC60D7FEA42912&id=44CC60D7FEA42912!169

Note that if you are using an earlier version of Access you might find that the colour of some form objects such as buttons shows incorrectly and you will need to  amend the form design accordingly.  

If you have difficulty opening the link, copy the link (NB, not the link location) and paste it into your browser's address bar.

In this little demo file the RefreshLinksSingle_07.accdb file provides an easy way to refresh links to a single back end file.  You will need to copy all of its objects bar the frmOpen form into your front end.  Your colleagues can then either open the frmUpdate_Links form manually or you can incorporate the code from the frmOpen form's class module in the demo in your front end's opening form's class module.  This automatically checks for broken links at runtime and automatically opens the frmUpdate_Links form if necessary.

suzanne s. barnhill say:

You can still do it through the Field dialog if that's the way you did it before (Insert | Text | Quick Parts | Field). Or you can create it by hand:

  1. Type NUMCHARS, select it and press Ctrl+F9 to convert it to a field.
  2. Type = before the field and -5 after the field.
  3. Select the entire expression and press Ctrl+F9. You should see:
    { = { NUMCHARS } -5 }
  4. F9 to update.

talking moose say:
First, try deleting these files from your home folder where " ~ " denotes your home folder:

com.microsoft.outlook.categoryoverlay.plist
com.microsoft.outlook.database_daemon.debuglogging.plist
com.microsoft.outlook.database_utility.debuglogging.plist
com.microsoft.Outlook.debuglogging.plist
com.microsoft.outlook.office_reminders.debuglogging.plist
com.microsoft.outlook.office_reminders.plist
com.microsoft.outlook.office_reminders.plist.lockfile
com.microsoft.Outlook.plist
com.microsoft.Outlook.plist.lockfile
com.microsoft.Outlook.SyncServicesPreferences.plist

You may not find all of these. Deleting these files will reset some preferences to defaults but you should not lose any data.

Log out and log in again or restart your computer. Launch Outlook and see if the problem persists.

If the problem persists then you may need to start a new Outlook identity and set up your mail, preferences and rules again. To do this launch Outlook with the Option key held down. This will launch the Database Utility application. Click the + (plus) button to create a new identity. Name it whatever you like. Make this new identity the default using the drop down menu at the bottom.

Quit the Database Utility application and launch Outlook normally. Configure your accounts and settings again. If you are using IMAP or Exchange then your mail will sync from the server. If you are using POP accounts then you can move your mail, contacts, etc., by returning to the old identity and selecting File menu --> Export... and then importing that .olm file while in your new identity.

Hope this helps!
