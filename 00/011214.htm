jeeped say:

need to vlookup (formula below) only 2nd occurance of value in named range/string and return result ... another formula that says, vlookup the 3rd occurance of the number and return the specified value.


If you just wanted the second value, you could nest two MATCH() functions to OFFSET() the lookup range. Since it appears that you want the third and possibly the fourth, fifth, etc. you are going to need a helper column. In an unused column (column Z for this example) to the right put this formula into the top row,
=OFFSET(all_data,ROWS($1:1)-1,0,1,1)&TEXT(COUNTIF(OFFSET(all_data,0,0,ROWS($1:1),1),OFFSET(all_data,ROWS($1:1)-1,0,1,1)),"000")
... and fill down to cover the rows in all_data. That formula assumes that you will not have more than 999 of any one value. Now you can reference that helper column with your progressive INDEX(MATCH()) formula. Put this formula into a cell with some blank cells below it.
=INDEX(OFFSET(all_data,0,3,,1),MATCH($H$4&TEXT(ROWS($1:1),"000"),Z:Z,FALSE))
(note the reference to Z:Z). This will give you the first MATCH(). Copy or Fill Down to get successive matches (e.g. second, third, etc).

The use of OFFSET() was necessary due to the named range all_data. Both formulas would be much shorter and use less calculations in actual cell ranges were used.

joeu2004 say: JW wrote:

need to vlookup (formula below) only 2nd occurance of value in named range/string and return result.
(the value in A4 may occur as many as 3-4 times)
=VLOOKUP(A4,all_data,4,FALSE)


=VLOOKUP(A4,OFFSET(all_data,MATCH(A4,INDEX(all_data,0,1),0),0),4,0)

INDEX(all_data,0,1) returns the first column reference.  For example, if all_data is defined as $B$1:$F$100, INDEX(all_data,0,1) returns $B$1:$B$100.

MATCH(A4,INDEX(...),0) returns the relative row number of the first occurrence.

Thus, OFFSET(all_data,MATCH(...),0) returns the 2-dimensional range reference starting in the row after the first occurrence.

Note that the 4th parameter to VLOOKUP is zero, not FALSE.  Although VLOOKUP does treat FALSE as zero, the 4th parameter is any one of -1, 0 or 1, not TRUE or FALSE.

As you might imagine, this becomes very messy if you are looking for the 3rd, 4th etc occurrence.

PS:  I prefer to avoid OFFSET because it is a volatile function.  Consequently, the VLOOKUP formula (and any dependent cells) is recalculated every time any cell in any sheet in the workbook is changed manually (edit or enter a new value).  This can cause long delays when editing.  It would be better to replace OFFSET with some use of INDEX, if possible.  I'm still working on it.
