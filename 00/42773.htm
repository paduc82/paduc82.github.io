lemp say:

Do it this way:

  1. Open Printers and Faxes
  2. Click Add a printer
  3. Click "Next," select "Local printer" and make sure that the box to "Automatically detect my Plug and Play printer" is NOT checked
  4. Click "Next," select "Create a new port," and select "Standard TCP/IP port" from the drop-down list (Note -- if the printer you are adding is an HP printer and if the drop-down list includes "HP Standard TCP/IP Port" select that instead; more details later)
  5. Click "Next" on the Welcome to Add Standard TCP/IP Printer Port Wizard screen
  6. For "Printer name or IP address" type 192.168.1.105; the  "Port name" box will automatically fill in, but you can change it to something more useful, such as Office LaserJet or Bedroom Printer, etc.
  7. Before clicking "Next," confirm that the printer is on
  8. Click "Finish" to complete the Add a Port wizard and return to the Add a Printer wizard
  9. Select your printer manufacturer and model and complete the wizard

One very common problem with networked printers that are left at the default setting to obtain an IP address automatically is that they may be assigned a different IP address by a router's DHCP server if the printer is powered off for a period of time.  The Windows printing system does not detect such IP address changes, and thus communication with the printer is lost.  The "HP Standard TCP/IP Port" has the option to detect the printer even if its IP address changes.  You should definitely select this option or -- and this is the better practice -- assign a static IP address to all networked printers.

The voxaldriver entry may be the issue.

Try removing the voxaldriver entry:

Double-click UpperFilters and remove voxaldriver (leave ksthunk) > OK

Create a System Restore Point beforehand and restart the computer afterward.

This may affect the functionality of Voxal Voice Changer if Voxal Voice Changer is still installed. 

If the issue persists after removing the voxaldriver entry try removing the LVUSBS64 entry (Logitech filter value):

To remove the LVUSBS64 entry double-click LowerFilters and, under Value data, remove LVUSBS64  > OK


Create a system restore point beforehand and restart the computer after.

Removing the LVUSBS64 entry may affect functionality of Logitech camera.

raajeev say:

if you could please provide the Mac Model or Serial number then I can provide specific answer to your Mac Book. Otherwise, note that Apple uses "Realtek" Audio chip on many Macs and the problem is that even after installing the right drivers, Windows uses default Sound drivers.

Solution:

Install "Realtek Sound Drivers" from the Boot Camp folder,  restart as asked... go to Device Manager and expand the Sound Group... right click on both listed drivers and "Uninstall" both of them. Restart the Mac again to load the correct Audio drivers.

You are done. It should work. Let me know if it does not and do provide the specific model and year for your Mac to give you specific answer.
