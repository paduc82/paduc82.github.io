a. user say:

Hi,

A bounce message/delivery failure message contains information about the message delivery problem that you've encountered. How the information are formatted and stated in a bounce message vary depending on the mail server sending the message back to you. Some uses technical terms. Some takes paragraphs to tell you that you probably just misspelled your recipient's e-mail address.

A Sample bounce message:

You'll see this information when you display the full message headers of the bounce message that you've received. The most important information here is the Diagnostic-Code.

Delivery to the following recipients failed.

       *** Email address is removed for privacy ***

Reporting-MTA: dns;BAY0-OMC3-S10.bay0.hotmail.com

Received-From-MTA: dns;BAY121-W55

Arrival-Date: Tue, 10 Jun 2008 14:21:03 -0700

Final-Recipient: rfc822; *** Email address is removed for privacy ***

Action: failed

Status: 5.5.0

Diagnostic-Code: smtp;550 *** Email address is removed for privacy *** unknown user account

Common bounce messages:

Mailbox Not Found, Invalid mailbox, User unknown:

The e-mail address that you're sending to doesn't exist. Check your recipient's e-mail address. Either you have misspelled it or it's an old e-mail address that's no longer in use. You can double-check it with your recipient.

Mailbox unavailable:

9 times out of 10, this is the same as "mailbox not found". That other 10% of the time it could mean that there's a problem with your recipients account. Check to make sure that you have the e-mail address correct, wait a while and try again. If it still bounces, try contacting the recipient some other way.

Mailbox full or Quote Exceeded:

As what the error states, your recipient have too much message and the server hosting it isn't accepting any more. Contact your recipient through other means and advise him/her to clean up his/her mailbox.

Host unknown, Domain Lookup Failed:

This means that the mail server you're attempting to send to (the "hotmail.com" part in the example above) doesn't exist. Make sure you type your recipient's e-mail address correctly. Another reason is when an ISP changes their name (like when "attbi.com" changed their name to "comcast.com". If you try to send a message to an old "attbi.com" e-mail address, you might get this message in return.).

Message too large:

The message that you are trying to send is too large or exceeds your recipient's mailbox limit.  This usually happens when the host of your recipient has a limit on the size of messages that they can receive. Reduce the size of your message. If you are trying to send a file, you may need to compress it using a file compression program, like WinZip. 

Temporary Errors (no adequate servers, Connection Timed Out, Resources temporarily unavailable, Out of memory):

There's a problem with the mail server that you probably don't have any control over. As stated, these are temporary errors and should resolve themselves over time. Check the bounce message carefully because the mail server mentioned in it may continue to try to deliver your message without any action required on your part.

Blacklist Filters (errors that included the word "blocked" or "listed in" and references to sites that have things like "spamcop", "dynablock", "blackhole", "spamhaus" etc):

This means that your message is intentionally blocked because the receiving system thinks that the mail server you're using is a source of spam. Contact the support team of recipient's e-mail provider and request to have your account or domain unblocked.

For other bounce messages, visit http://support.microsoft.com/kb/284204

Sub Print_March()
'
' Print_March Macro
'
' Keyboard Shortcut: Ctrl+a
'
    Dim v1 As Variant, v2 As Variant
    v1 = Array("Sheet1", "Sheet2", "Sheet3")
    v2 = Array("rng1", "rng2", "rng3")
    Application.ScreenUpdating = False
    Application.DisplayAlerts = False
   
    Sheets.Add
    t = 1
    For i = 0 To UBound(v1)
    Sheets(v1(i)).Range(v2(i)).Copy
    ActiveSheet.Cells(t, 1).PasteSpecial xlPasteValues
    ActiveSheet.Cells(t, 1).PasteSpecial xlPasteFormats
    Application.CutCopyMode = False
    t = ActiveSheet.UsedRange.Rows.Count + 1
    Next
   
    With ActiveSheet
    .PageSetup.PrintArea = .UsedRange.Address
    .PrintPreview
    End With
   
    ActiveSheet.Delete
    Application.DisplayAlerts = True
    Application.ScreenUpdating = True
   
End Sub

However, I want this to be the default setting, and saving the setting doesn't work.  I click on the presets drop-down and choose to save current settings as a preset.  I choose a name for the preset, like "no notes," but when I open it again later, the Microsoft Word setting is reset to the default of printing with markups.

    I know there is a thread on this discussion board regarding a similar topic (http://answers.microsoft.com/en-us/mac/forum/macoffice2008-macword/how-to-edit-printer-presents-in-word/1bb332a8-c7c7-40b7-a7a7-63e860abc2a5?msgId=4427f334-fd12-479e-8d53-f3f32beeafbd), but the answer is not quite correct.  OS X does store most printing presets across multiple applications, but the Microsoft Word printing settings can only be accessed by printing from Microsoft Word, so the solution rests somewhere inside the Word application.

    How do I save this setting? It's quite a pain to have to change this setting manually every single time I print!
    john mcghie say:
    Yes, you are correct, those are Word settings, not OS X settings.  The first Print dialog is an Apple OS X Finder dialog, the seconds is a Microsoft Word dialog: two dialogs, look the same, but addressing different applications.

    However, because this dialog is part of Word and not part of OS X, you can record a macro to do this.

    And if you do, you will get something like this:

    Sub Macro2()
    '
    ' Macro2 Macro
    '
    '
        Application.PrintOut FileName:="", Range:=wdPrintAllDocument, Item:= _
            wdPrintDocumentContent, Copies:=1, Pages:="", PageType:=wdPrintAllPages, _
            Collate:=True, Background:=False
    End Sub

    You can use Tools>Customise to add that macro to a convenient toolbar and there you go: one-click printing without markup.

    You might need this: http://word.mvps.org/Mac/InstallMacro.html

    Cheers
