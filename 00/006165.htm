Assume your "Table B" data, example below, is in sheet: x, within A1:D13

=INDEX(x!$D$2:$D$13,MATCH(1,INDEX((x!$A$2:$A$13=A2)*(x!$B$2:$B$13<=D2)*(x!$C$2:$C$13>=D2),),0))

Let's try to get this thread back on track.

You are entering times in column P.  But Excel does not recognize negative time like -68:00.  That is the cause of the #VALUE error.

Ostensibly, the formula in column S needs to be in the following form:

=IF(ISNUMBER(P5), P5/60, P5)

(ERRATA: When P5 is text in the form -12:34, it is sufficient to "copy" that into S5 (the value-if-false part), since it already represents -12m 34s, I presume.)

But note that if P5 is of the form -68:00 (negative time), the result in column S will be text, not a number.  So the format of S5 does not matter.

MeighDay wrote:

Im starting to think I need to redo this whole thing. I am trying to track times and the best way I could find to display was by formatting the cells to display [mm]:ss and it seems to show in seconds but only works well if there are more than 60 seconds.

I do not understand what you are saying.

First, if you enter time in the form 12:34, Excel interprets that as 12h 34m.

I presume that you are dividing by 60 because you want Excel to interpret that as 12m 34s.  Right?

It would be better if you entered the data in one of the following forms:  0:12:34 or 12:34.0 .

Note that whenever there is a decimal fraction (".0") at the end, Excel interprets that as 12m 34s, as intended (I presume).

But you still should format the cell as [m]:ss in order to permit 60 or more minutes (e.g. 68:34.0) to be displayed as minutes and seconds.

And you still have the issue with "negative time", namely:  Excel does not recognize that as numeric time, so it treats -68:34 as text.
