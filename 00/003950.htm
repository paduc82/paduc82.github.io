How to stop Outlook 2010 from attempting to publish calendars to - Microsoft Community

steve house say:

How to stop Outlook 2010 from attempting to publish calendars to (now discontinued) Office.com?

Have been using Outlook 2010's feature that publishes my calendar to Office.com in order to share it with clients.  With the advent of Office 2013/365 Microsoft has apparently discontinued that service (without notice to existing users, I might add). With the calendar servers offline, each time Outlook attempts to sync to the online calendar a log-in dialog box requesting one's account credentials appears and when they are provided, after several attempts, a message appears saying the calendar update has failed.  The obvious fix to stop the repeated error messages, since the servers have gone bye-bye, is to stop sharing the calendar to the Office.com.  But therein lies the rub.  To stop the update attempts one must unpublish the calendar.  To do so one selects "Remove from server" in the Calendar Home ribbon, Share group, Publish Online drop-down menu.  But when one tries this, Outlook must log in to the (now non-existent) calendar server to delete the old files.  Attempting to log in to the server once again triggers the credentials request and when the server doesn't respond the deletion attempt fails.  When the deletion attempt fails, the cancellation of future updates also fails and we remain back where we started, having a credentials demand pop up periodically through the day.  Welcome to Catch-22.  We can't stop Outlook 's attempts to log in to the server until it deletes the files it thinks are up there and it can't delete the files because it can't log in to the server.  So there's the question - how does one turn off Outlook's attempting to share the calendar through Office.com without going through the now ineffective "remove from server" procedure?
steve house say: Just figured out my own answer.  Go to the Office Backstage, Info, Account Settings, Published Calendars tab, select the calendar in question and click the "X Remove" option.  It will ask if you want to stop publishing but leave the calendar on the server.  Answer "Yes."  This stops further update attempts. Since the server is turned off, you can't remove it from the server anyway, there's nothing left to remove.
