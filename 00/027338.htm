gabriel bratton say:

Larry,

We are working on documenting the Outlook.com known issues and this will be one of them.  Please try workaround #2 below and let me know if you see any issues with it.

Also, on the machine that you are not seeing the issue you might double check if it is still connecting as EAS.  If it is you should go ahead and update it to connect as Exchange because the Outlook Team is disabling EAS.  You can check by going to File, Account Settings, Account Settings.  If it shows as ExchangeActiveSync as Type then follow this topic to update it to Exchange, https://support.office.com/en-us/article/Reconnect-Outlook-2016-or-Outlook-2013-to-Outlook-com-for-uninterrupted-email-access-cda1751d-9503-40bf-bf76-e79454ac5eb3?ui=en-US&rs=en-US&ad=US&fromAR=1.

When responding to (replying or forwarding) email from a different account the sending account is the Outlook.com account
Status:  This is a known issue that is being investigated.
Workaround #1:  When sending email change the From drop down field to the other account.
Workaround #2:  Add an empty data file and make it the default.
1.  In Outlook click File, Account Settings, Account Settings
2.  Click the Data Files tab
3.  Click Add
4.  Choose Outlook Data File
5.  Name the PST or leave the default name and click OK
6.  On the same Data Files tab click the new PST data file and click Set as Default
7.  Restart Outlook
When Outlook opens back up you will see the Data File listed like your other accounts.  In the left navigation you can drag the Data File below your other email accounts to get it out of the way. 
Note:  If you use this solution be careful if you are working in other accounts than the Outlook.com account and you use the New Item drop down on the ribbon to create new items like Calendar, Contacts, and Tasks, that those will get created in the new Data File.  If you are working in the Outlook.com account those items will get created in the Outlook.com accounts respective folder.
