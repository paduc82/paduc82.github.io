hansv mvp say:

The event procedure for the On Not In List event of the EmployeeID combo box could look like this:

 

Private Sub EmployeeID_NotInList(NewData As String, Response As Integer)
    ' Ask user whether name should be added
    If MsgBox("The employee '" & NewData & "' does not occur in the list." & vbCrLf & _
            "Do you want to add it?", vbYesNo + vbQuestion) = vbYes Then
        ' Open employee form
        DoCmd.OpenForm FormName:="frmEmployees", DataMode:=acFormAdd, _
            WindowMode:=acDialog, OpenArgs:=NewData
        ' Tell Access that name has been added
        Response = acDataErrAdded
    Else
        ' Undo the new name
        Me.EmployeeID.Undo
        ' Tell Access to suppress the built-in error message.
        Response = acDataErrContinue
    End If
End Sub

 

You must substitute the correct field names and form name of course.

 

In the On Load event of the employee form (frmEmployees in the above example), create code to fill in the first name if supplied:

 

Private Sub Form_Load()
    If Not IsNull(Me.OpenArgs) Then
        Me.FirstName = Me.OpenArgs
    End If
End Sub

 

Again, substitute the correct field name.

swischner say:
The quick note functionality (e.g. opening a small note window with a minimalistic user interface) is completely built on OneNote W32 (OneNote 2016). There is no equivalent for it in the OneNote app for Windows 10.

There has been a similar function for the Windows 10 app (creating a new note using a shortcut) up to Windows 10 Fall Creators Update. Until then a symbol in the Message Center of Windows allowed you to quickly start OneNote (App) with a newly created note. Unfortunately, Microsoft has removed this shortcut in the recent version of Windows. It's only left for Surface users, bound to the pen button.

But I can give you the command that was called with that shortcut, so you may create a desktop or start menu link of your own and maybe assign a keyboard shortcut to it:

onenote-cmd://quicknote?onOpen=typing

ken sheridan say: I use the following function:

Public Function WeekStart(intStartDay As Integer, Optional varDate As Variant)

    ' Returns 'week starting' date for any date
    
    ' Arguments:
    ' 1. intStartDay - weekday on which week starts, 1-7 (Sun - Sat)
    ' 2. vardate - optional date value for which week starting
    '   date to be returned.  Defaults to current date
    
    If IsMissing(varDate) Then varDate = VBA.Date
    
    If Not IsNull(varDate) Then
        WeekStart = DateValue(varDate) - Weekday(varDate, intStartDay) + 1
    End If
    
End Function

greginmich say:

I've disabled Automatic Remediation successfully in previous versions of Windows 10, and I just disabled it with a REG command, in order to test the operation of the DisableRoutinelyTakingAction parameter in Version 1803.

Right-click on the Start Button and launch Windows PowerShell (Admin); and then copy, paste, and enter the appropriate command:

Turn off Automatic Remediation:

REG ADD "hklm\software\policies\microsoft\windows defender" /v DisableRoutinelyTakingAction /t REG_DWORD /d 1 /f

Turn on Automatic Remediation:

REG DELETE "hklm\software\policies\microsoft\windows defender" /v DisableRoutinelyTakingAction /f

Setting this parameter to a value of 1 in the Group Policy folder still disables Automatic Remediation without any problem, although it does require a restart – and despite the fact that Edge misreports that “eicar.com contained a virus and was deleted” prior to any action being taken:

GreginMich
