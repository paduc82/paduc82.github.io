a. user say:

NB, because of an obscure Word problem, if you use Macropod's method or the method described in the Microsoft knowledgebase his article references, be aware that when you save/close/reopen the Word document, Word may delete the { MERGEFIELD } field nested inside your HYPERLINK field.

Word will typically delete nested fields inside a HYPERLINK field in that scenario unless there is also a bookmark inside the field.

If the HYPERLINK field is the last thing you edit before you merge/sve/close your document, you may not notice this problem because in that case Word does in fact insert a bookmark (called _goback). However, if you click anywhere outside the HYPERLINK field before you save/close, or e.g. if you have multiple HYPERLINK fields with MAILMERGE fields nested inside them, you will probably lose your nested field codes.

SInce you can use a { SET } field to create a bookmark, you seem to be able to prevent Word from doing that by adding one to your HYPERLINK field.

e.g. if you have { HYPERLINK "{ MERGEFIELD mylink }" }

change it to 

{ HYPERLINK "{ MERGEFIELD mylink }{ SET keepme1 1 }" }

where all the {} are the special field code braces you can insert using ctrl-F9. That creates a bookmark called "keepme1" inside your HYPERLINK field and should avert the problem.

If you have more than one such HYPERLINK field, you will probably need a different bookmark for each HYPERLINK. You could either do that manually, e.g. use

{ SET keepme1 1 } in one field
{ SET keepme2 1 } in another
and so on.

Or you can do a bit more nesting and generate a sequence number automatically.

{ SET "keepme{ SEQ keepseq }" 1 }

IMO Microsoft should fix this problem - I'm pretty sure I raised it with them ages ago.

HansV has pretty much covered the bases with his excellent answer, but let me add some techniques for testing.

find a zipcode on each sheet that should match. 

say

A1 on sheet1 and   B50 on Sheet2

in another cell (any cell) test

=Sheet1!A1=Sheet2!B50

If you get false, then you need to test

=Len(sheet1!A1)=len(sheet2!B50)

if this is false, then look for spaces in the cell that isn't 5

if it passes then test

=isnumber(Sheet1!A1)=isNumber(Sheet2!B50)

If the first test failed, then one of the other two should fail.

If the last test fails then you have numbers in one sheet and text characters that represent numbers in the second sheet

=12345="12345"   will always fail even though they look the same.

Regardless of how the columns are formatted, how the values are stored In the cells is what is important.

Hi MM,

Based on your description, I understand that you need to sync files and documents from SharePoint Online instead of OneDrive.

First, please make sure you have the latest OneDrive for Business installed on your machine. Install the latest fully released Windows version of the new OneDrive sync client.

1) Go to Enable users to sync SharePoint files with the new OneDrive sync client to enable SharePoint Online sync for Windows:

2)Set SharePoint to sync with the new OneDrive sync client

  • Open the SharePoint admin center.
  • In the left pane, click settings.
  • Make sure "OneDrive Sync Button" is set to "Start the new client." To sync SharePoint files with the new client, you also need to sync OneDrive files with the new client.
  • For "Sync Client for SharePoint," select Start the new client.
  • Click OK

3) When you start OneDrive, and then prompted to sync documents from OneDrive Online, please uncheck Sync all files and folders in OneDrive - companyname.

In this situation, you are able to sync files and documents only from SharePoint Online and the set up popup window will not pop up again.
