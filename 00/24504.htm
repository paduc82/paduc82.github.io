It sounds like you have a Windows Mobile 5 device, rather than 6.x.  (WM5 required the domain field to be filled.)  Unfortunately, there's no way to sync a WM5 phone to the m.hotmail.com server, since anything entered into the domain will cause the sync to fail.

 

With that device, I'd suggest using Google instead.  Create a Gmail address if you don't already have one, and go into your Gmail Calendar at least once on a computer (Unless you do, Google won't have created your Calendar and attempts to sync will fail.)  Follow the steps above, but Instead of "m.hotmail.com" use "m.google.com," and enter your full Gmail address (with the "@gmail.com") and password as username and password.  Enter "google" (without the quotes) as the domain, and select what you want to sync (do not select "tasks"- Google doesn't support them, and the sync will fail.)

Now your Contacts and Calendar will be in Gmail/Google Calendar and you can then export them to whatever program or device you want.

suzanne s. barnhill say:

It sounds as if you're just typing numbers into the header instead of inserting a PAGE field. How you do this will depend on your Word version:

 

Word 2003 and earlier: On the Header and Footer toolbar, click the button for Insert Page Number.

 

Word 2007 and later: On the Header & Footer Tools | Design tab, in the Header & Footer group, click Page Number, select Current Position, and click on Plain Page Number.

 

In any version, you can also insert a PAGE field by pressing Alt+Shift+P. When you insert the page number field, it updates automatically on each page. But Word applies numbers to pages regardless of what page number you have displayed in the header, and the numbering should be correct in your TOC and index. If you want to change the way the pages are numbered (for example, if you want to restart numbering after front matter), then you need to use the Format Page Number dialog to set the starting number in a given section (you must have a section break to restart numbering). Access this via the Format Page Number button on the Header and Footer toolbar in Word 2003 and earlier, or via Header & Footer | Page Number | Format Pages Numbers in Word 2007 and later.

Is it possible to change the prefix of photos taken with the phone?

Here's my situation: We have four Nokia Lumia 520 Windows phones that we use for business. We send the phones with our crews and have them take photos of completed installations, deliveries, etc. 

Those photos are automatically uploaded to OneDrive, and then later we download the photos for permanent storage on our server. The photos are stored by date (one folder per day). 

The problem we are encountering is that each phone uses the same naming scheme for the photos. Every photo taken today (August 1) has the following filename: WP_20140801_001.jpg (with the last number changing to 002, 003, etc).

When we download the photos onto our server, we end up being asked to overwrite the photos we've already put in the directory. 

Is it possible to change the prefix on the photos for each phone?

We rename each photo later (to correspond with the appropriate order), but previously we were using digital cameras and this was never an issue. Right now, my best solution is to make sub-directories for each day for each truck we send out. That has it's benefits (specifically, knowing exactly which crew completed each job at a glance), but it does add to the time we spend sorting photos each day. It would be really great if we can change the prefix on each phone instead. 

Thank you.

martin crosbie say:

Unfortunately this is not possible to change on the phone, using the Microsoft camera. Other camera applications may offer this functionality.

Changing the prefix is easily done on multiple photos via explorer - select the photos you want to rename, right click on the first one and select 'rename'. add a prefix to the file name and this will get added on to all of the files selected (as will a number in brackets at the end) so you end up with "PREFIX WP_20140801_001 (1), PREFIX WP_20140801_002 (2)" etc. From what you describe, I would have thought a structure of DAY\TEAM\JOB\photos.jpg would be sensible enough, but then I don't know the rest of your business.

I appreciate that this is a rather manual process, but I can't see 4 teams, completing a few jobs each day taking more than 10 minutes to rename and file these photos.

A 'gold plated' solution for you would be to write your own app, that asked for the job details, then captured photos (named by the job details), and uploads them direct to your server file structure. If you can be clear of exactly how you want this to work, then many developers should be able to make this for you, for not too much expense - depends on the scale of your company as to the value in this, but it sounds like what you need
