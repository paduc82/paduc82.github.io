jay freedman say:
You can't password-protect a document without applying a password, and you can't apply a password without encrypting the file. So you may as well consider them to be one and the same thing. Not "protect the same" but ARE the same.


It may be confusing that there are two places in Word 2010 in which you can supply a password for encrypting a document:

  • On the File > Info page, click Protect Document > Encrypt with Password. You'll be prompted for the password; when you enter it and click OK, a second dialog will prompt you to enter the same password again, to make sure you typed it correctly. After that, you still have to save the document to make the password permanent.

  • On the Save As dialog, next to the Save button, click the Tools button and choose General Options. Here, you can enter a password needed to open the document (this is the same as supplying a password through the Info page). After you click OK, you'll be prompted to enter the same password again. Then Word finishes saving the document. The General Options dialog will also let you enter another password (the same as or different from the password to open) needed to allow editing of the document.


Regardless of which way you apply the password, the effect is the same. The way Word makes sure that the document can't be opened without the proper password is that it encrypts the file, using the password as the encryption key. The default encryption method is something called AES 128-bit encryption, and it's secure against any but the most determined attacks -- as long as the password is strong AND IT ISN'T AVAILABLE FOR ANY UNAUTHORIZED PERSON TO SEE. As Graham says, if you leave a password unguarded, you might as well hand out copies of the document to everyone you meet.


To add to the confusion, there is a button in the Save As > Tools > General Options dialog that says "Protect Document". This takes you to the same pane as the Restrict Editing button on the Review tab of the ribbon, where you can allow only certain kinds of formatting, or turn on "forms protection", and apply a password for this purpose (NOT the same as encrypting the document). This sort of "protection" is very weak; don't rely on it to protect confidential information.

mike h.. say:

Hi,


Try this ARRAY formula. Put the formula in a cell, ARRAY enter it; see below on how, and then drag down.



=IFERROR(INDEX($C$3:$C$25,SMALL(IF($D$3:$D$25=TRUE,ROW($B$1:$B$23)),ROW(1:1))),"")



Note that if you change the number of rows in the formula if you look at the 3 underlined bits the first 2 are the apples etc and TRUE or FALSE columns and it starts in row 3 and there are therefore 23 rows of data being looked at. The last underlined bit must have the same number of rows but must start at 1. This last bit isn't looking at the range, it's being used for counting.


This is an array formula which must be entered by pressing CTRL+Shift+Enter
and not just Enter. If you do it correctly then Excel will put curly brackets
around the formula {}. You can't type these yourself. If you edit the formula
you must enter it again with CTRL+Shift+Enter.
