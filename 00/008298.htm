dhookom say:

Averaging or doing any aggregation across columns/fields/etc suggests the tables aren't normalized.

 

However, you can't use Sum() or Avg() across fields. These are SQL calculations that operate across records. If you really want to keep your structure as is, you will need to add the values and divide the sum by the count of values.

 

Since this is a report, you also can't Sum() controls. You can Sum() values from the report's record source.

The day of the week should not be a separate Table field; it's redundant. If you have a Date/Time field containing #6/18/2014# then it's a Wednesday; and you don't need - and should not have! - a separate table field containing the text string "Wednesday".

If you want to display the date on a Form you can display it in either one or two textboxes bound to the date field. For one textbox you can use a Format property such as

mm/dd/yyyy dddd

to display

07/18/2014 Wednesday

Or you could use two textboxes, both bound to the date field, with Format properties of

mm/dd/yyyy

or Short Date for the first, and

dddd

for the second.

Could you explain why you feel you need an unbound textbox? It can certainly be done that way with a little code, but... what's the point?
