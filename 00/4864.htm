doug robbins - mvp office apps & services (word) say:

You will need to make sure that the first record (make it a dummy record if necessary) contains more that 255 characters in the fields with which you are experiencing the issue.

The reason for this is that Word determines the type of data in each field by inspecting the first records in the data source and if the first record does not contain more than that number of characters in the field, it will be treated as a 'text' field, which cannot have more than 255 characters rather than a 'memo' field, for which there is no limit.

-- Hope this helps.

jim gordon mvp say:

I am a bit uncomfortable with the idea that you have uninstalled and then reinstalled. I am hoping that you followed these instructions to completely remove office before you reinstalled:

http://support.microsoft.com/kb/2398768


If you did not, then do follow those instructions and then reinstall again.


After reinstalling you should use a font utility such as Font Book application to validate your fonts. There will be duplicate fonts. Delete old versions of duplicate fonts and delete corrupt fonts.


Use Disk Utility to validate your startup volume and repair disk permissions.


When you reinstall, allow Office to update to the current version (currently 14.2.5).


Assuming you have done all of the above and  problems still exists with that particular file, do the following:


1. Open the problem file.

2. Use File > Save As and choose either Macro Enabled Workbook (.xlsm) or Excel Binary format (.xlsb)

3. Open the Visual Basic Editor. 

4. In the VB Editor select each module and userform in the Project browser and one-by-one use File > Export to export the modules and userforms as files

5. Switch back to the workbook.

6. Use File > Save As and choose Excel Workbook (.xlsx) file format and save in (.xlsx) file format. This will make a version stripped of all visual basic code.

7. Quit Excel

8. Open the saved (.xlsx) workbook

9. Open the Visual Basic Editor

10. Use File > Import to import all of the modules and userforms. The idea is to restore everything the way it was in the VB Editor before stripping out the modules and userforms.

11. Switch back to Excel workbook

12. Use File > Save As and save in either Excel Macro enabled (.xlsm) or Excel Binary (.xlsb) file format.


Note: You might want to try using the file saved in step 3 before going further. That alone might fix the file.

a. user say:

Hi Dave,

 

Thank you for posting your question in the Microsoft Community Forums.

 

Based on the information, you are unable to download a .txt file using Internet Explorer (IE) 10.

It would be helpful if you could reply to the questions to assist you better.

1. Does the issue occur only when you download a .txt file from a specific website?

2. Are you able to download other types of files without any issues?

 

If the issue is confined to downloading a .txt file from a particular website and the download works fine in compatibility mode, the website you are trying to download the file may not updated to work properly in Internet Explorer 10.

Thats the reason it works correctly in compatibility view settings.

If the issue is related to a specific website, then you may need to get in touch with the Website support for further assistance.

For further information, refer to the article:

Use Compatibility View in Internet Explorer

 

If this occurs when you download a .txt file from other websites also, then as a work around you may try to open the file and select the save option to save the file.

 

Hope the information helps.
