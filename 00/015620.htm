BUG in Richtext Content Controls - Microsoft Community

greg maxey say:

BUG in Richtext Content Controls

There is an apparent bug associated with RichText CCs containing a table.  I came across the bug when attempting to insert a buildingblock in a rich text CC based on the selection in a separate dropdown CC.  Each of the building blocks consisted of a single table listing team member names and positions.  For example select Team A from the dropdown and the buidlingblock containing a table defining Team A members and positions is inserted in the richtext CC.  Select Team B and a different table is inserted.

The basic code worked fine for inserting the team table in an empty richtext CC (i.e, showing placeholdertext).  However, attempts to change from Team A table to Team B table results in an unspecified automation error.

To illustrate, I created a blank .dotm file and added a singled rich text CC titled "Table Placeholder."  I then created two simple buildingblocks consisting of a single table.  I named one "Table A" and the other "Table B"

I'm not asking a question or looking for an answer.  Just posting so others will be aware of the issue.

Here is the code showing the initial error and other errors encountered looking for a working around.  The work around requires the table in the CC be deleted and then resetting the range to the cc range.

Sub DemoBug()
Dim oRng As Word.Range
Dim oCC As ContentControl
  Set oCC = ActiveDocument.SelectContentControlsByTitle("Table Placeholder").Item(1)
  Set oRng = oCC.Range
  If oCC.ShowingPlaceholderText Then
    'Insert a buildingblock consisting of a table.
    ActiveDocument.AttachedTemplate.BuildingBlockTypes(wdTypeAutoText).Categories("General").BuildingBlocks("Table A").Insert Where:=oRng
    'Attempt to replace with a different building block consisting of a table
    On Error GoTo Err_WhatGives
    ActiveDocument.AttachedTemplate.BuildingBlockTypes(wdTypeAutoText).Categories("General").BuildingBlocks("Table B").Insert Where:=oRng
    'Seems the table in the CC is causing Word to throw a fit. So lets delete it.
    oRng.Delete
    'Lets try to redefing it:
    oRng.Text = vbNullString
    'No error so we obviously can edit the range content.
    ActiveDocument.AttachedTemplate.BuildingBlockTypes(wdTypeAutoText).Categories("General").BuildingBlocks("Table B").Insert Where:=oRng
    'Editing the range just left of with an empty table so get rid of it
    ActiveDocument.Undo
    'Maybe it is the table that is being so troublesome.
    oRng.Tables(1).Delete
    ActiveDocument.AttachedTemplate.BuildingBlockTypes(wdTypeAutoText).Categories("General").BuildingBlocks("Table B").Insert Where:=oRng
    'Ah! No error, but where was the building block inserted?
    'Is it in the content control where it should be?
    If oCC.Range.Tables.Count = 1 Then
      MsgBox "Yes it is!"
    Else
      MsgBox "No it is not!"
      'Something is still clearly amiss. So get rid of the misplaced table.
      ActiveDocument.Undo
    End If
    'Lets try setting the range again.
    Set oRng = ActiveDocument.SelectContentControlsByTitle("Table Placeholder").Item(1).Range
    ActiveDocument.AttachedTemplate.BuildingBlockTypes(wdTypeAutoText).Categories("General").BuildingBlocks("Table B").Insert Where:=oRng
    'Well?
    If oCC.Range.Tables.Count = 1 Then MsgBox "Eureka!!"
  End If
lbl_Exit:
  Exit Sub
Err_WhatGives:
  MsgBox Err.Number & " " & Err.Description
  Resume Next
End Sub
