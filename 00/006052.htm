hi david,

 

i would like to answer your questions one by one:

1. can the client achieve what they want using office 365?

yes, you can achieve that by storing files in your team site. in this way, users can access the files and coauthor the files. you can see this article to learn more: should i save my documents to onedrive for business or a team site?

 2. how would they achieve the goal of synching the data from the team site to each pc/laptop?

they can install onedrive for business sync app on their own computer then sync files from the team site library to their own local pc to manage them offline. any updates will be updated to the online once they are online. you can see this article about how to sync files using onedrive for business sync app:sync site libraries to your computer. if you don't have the sync app, you can download it from the link provided by that article.

note: please make sure the users at least have edit permission for the library that files stored in. otherwise, the modifications that they made from offline synced files will not be updated to the cloud.

 3. can certain files/folders in the team site be restricted to only be accessible to certain users?

yes. we can use permission to restrict that. here is the demo steps, you can refer to them:

 

here is a doctestiris document library in my team site. by default, this library's permission inherits the team site's library. that's say, if the user has edit permission to the team site, then he has the edit permission for the library by default. since i need to let users cannot access the library to access files stored in it, i need to break the permission.

go to librray-> library settings->permissions for this document library.

click stop inheriting permissions.

click check permissions to check the user's permission to find out which group he is in. then remove him from the group. or, you can remove all groups out then granting the permission to users that you want them access the library by clicking grant permissions.

4. can the client selectively share files/folders to external partners securely? if so how? can this be password-protected and/or time-limited?

as a global admin or sharepoint service administrator, you can disable the external sharing feature for the whole site collection. in this way, users will not be able to share files with external users from the site.

 

for how to share files with external users, you can refer to: share sites or documents with people outside your organization.

      doug robbins - mvp office apps & services (word) say:

      Word does not like working with tables that extend over many pages.

      See the article "Maximising the performance of Word tables‚Äù at:

      http://wordmvp.com/FAQs/TblsFldsFms/FastTables.htm

      One other thing that can improve performance when editing a document is to go to file>Options>Advanced>Show document content and (temporarily) check the box for "Show picture placeholders"  Doing that will avoid Word having to refresh each image as you scroll through the document.

      guessed say:

      16Mb is not particularly large and I've used documents with thousands of rows before so I don't think you are pushing it yet....

      But planning to put in 800 images of over a Mb each is definitely pushing the boundaries.

      Firstly, I would say that you absolutely must be using styles and minimal direct formatting. The graphic resolution and colour depth needs to be minimised to get the file sizes down before putting them into Word. What format are the graphics in and do they really need to be as large as you say? If they are photos, 200dpi is plenty of resolution for most printers and screen. If they are bitmaps made from computer drawn art then you should be saving to PNG and using a resolution of around 300dpi.

      Rather than embedding the graphics, link them in to the document so that the document doesn't actually store the graphic inside the doc. This will greatly speed file saves.

      Graphics direct to PDF is also still an option because you can use a PDF editing tool such as Adobe Acrobat to add headers/footers and incrementing page numbers.
