vanderghast say:

You probably generates an intermediate result which exceed 2 Gig of temporary internal storage. This is generally due to a join poorly defined. If there is a where clause which limits the records in one table (without implying any value of the other table), make, first, a query that does that, and use that query. Example, instead of:

     SELECT *

     FROM a, b

     WHERE a.f1 > 40  AND  a.f2=b.f2

 

tries:

     SELECT * FROM a WHERE f1 > 40      

as a saved query, say qa, and then:

     SELECT *

     FROM qa INNER JOIN b ON q1.f2=b.f2

 

which uses the two techinques (a join and a saved query).
