dale howard [mvp] say:

Faisal --

This is a really good question, my friend.  First of all, here is a little background information for you:

  • When a user creates a new project in Project Online, the system automatically designates that user as the Owner of the project.  The Owner has full permissions to open, edit, save, and publish the project.  Using the default permissions in Project Online, if you are not the Owner of a project, you cannot even SEE the project in the Project Center page or in Microsoft Project Professional.
  • When the new project is created, the Owner of the project is also designated as the Status Manager of every task in the project schedule.  The Status Manager is the PM to whom task updates are submitted by team members in the project.  The Status Manager sees these task updates on the Approval Center page in PWA, and this is where he/she can approve those task updates.  Once approved, the task updated are written by Project Online into the Microsoft Project schedule.

So, my question for you is this:  Do you need to be designated as the permanent Owner of those two other projects, meaning that the project will belong to you from now on?  If so, either the current Owner of the project or your application administrator can change the original Owner value from the previous PM to you as the new PM.  To do this, one of them will need to click the name of the project in the Project Center page, click the Schedule link at the top of the Quick Launch menu, click the Edit button in the ribbon, and specify you as the new Owner of the project.  That person will then need to save the project, then close it and check it in.

If you are NOT going to be the permanent Owner of these two other projects, then the current Owner will need to give you special permissions to access the project.  What he/she will need to do is to navigate to the Project Center page in PWA, click the row header (white box) at the left end of the row for one of those projects, and then click the Project Permissions button in the ribbon.  On the Project Permissions page, he/she will need to click the New button, add your name to the Users and Groups with Permissions list, select ALL of the checkboxes at the bottom of the page, and then click the Save button.  At this point, you now have permissions to see, open, edit, save, and publish the project.  He/she will need to repeat this process for the second project as well so that you have edit permissions to both of the project.

Once the Owner has given you special permissions to access his/her two projects, you will need to open each of these projects using Microsoft Project Professional (NOT PWA).  You will need to apply any Task view, such as the Gantt Chart view, and then temporarily insert the Status Manager column.  In the Status Manager column, select your name for every task you will manage in each project.  Then you will need to save and publish each of these projects, then close and check them in as well.

Hope this helps.

jay freedman say:

Sam, I don't think it's going to get much better than what you're already doing.


The difficulty is that the entries (assuming they're plain text, not what Office considers "formatted" entries) get stored in a language-dependent ACL file -- for US English, that would be %appdata%\Microsoft\Office\MSO1033.ACL. The ACL files are mostly Unicode text, but interspersed with binary bits.


Unless you know how to write a program that properly manages the binary parts of the file, you have to let Word add and delete the entries.


The best I can offer you is this idea: You can have a shortcut key that starts Word and automatically opens the AutoCorrect dialog. If all goes well, the text content of the clipboard will be in the "With" box.

Insert this macro in your Normal.dotm template (see http://www.gmayor.com/installing_macro.htm if necessary):

Sub AutoCorrectDialog()
    Dim MyData As DataObject
    Set MyData = New DataObject
    On Error GoTo BadData
    MyData.GetFromClipboard
    With Dialogs(wdDialogToolsAutoCorrect)
        .With = MyData.GetText
        .Show
    End With
    GoTo Finally
BadData:
    MsgBox "The clipboard does not contain text.", vbCritical, "Error"
Finally:
    Set MyData = Nothing
End Sub

While you're still in the macro editor, click the Tools menu, click References, and put a check mark next to "Microsoft Forms 2.0 Object Library". That's necessary to let VBA use the DataObject. Save the template and close the macro editor.

Locate your Word executable file, WinWord.exe, which is probably in the folder C:\Program Files\Microsoft Office\Office14. Right-click the file and click Send To > Desktop (create shortcut).

When the shortcut appears on your desktop, right-click it and click Properties. On the Shortcut tab of the Properties dialog, click at the end of the path in the Target box (after the closing quote mark), type a space, and type /mAutoCorrectDialog. Then click in the Shortcut key box and press the key combination you want to use to start Word and invoke the macro. Finally, click OK.

When you press the shortcut key combination, a new instance of Word will start. (If you already have Word running, you'll now have two separate copies of Word.) The AutoCorrect dialog will immediately open, with the copied text in the "With" box and the cursor in the "Replace" box. The remaining steps are manual -- entering the Replace text, clicking OK, and closing the instance of Word.
