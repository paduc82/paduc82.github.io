How can I convert my Yahoo or Gmail email to Microsoft Account - Microsoft Community

rick_20 say:

How can I convert my Yahoo or Gmail email to Microsoft Account

Help me please!
brian tillman [outlook mvp 2007-2019] say:

Keep in mind that people here are volunteers.  Most of us do not work for Microsoft.


You cannot "convert" a Yahoo or Gmail account to a Microsoft account, in my opinion.  You can, however, do one of several things.  You could, if you wish, create a Microsoft account whose name matches either a Yahoo or a Gmail address.  That won't, however. mean messages sent to either of those addresses will reach the Outlook.com mailbox.  The address you use must be a Microsoft address (@hotmail.com, @live.com, @outlook.com, @msn.com, @webtv.net, etc.) in order for messages to be routed to the Microsoft mail servers.


You could also create a Microsoft account using a Microsoft address, then configure the Yahoo or Gmail account to forward to that Microsoft address.  Then, within the account, you can create a send-only account that allows you to send mail from that Yahoo or Gmail address.


You could also configure the Microsoft account with a send/receive account using the Yahoo or Gmail address.  The difference between this and the previous choice is that in this case, the Yahoo or Gmail account doesn't have to forward messages.  The Outlook.com server will use POP3 protocols to connect to the Yahoo or Gmail mailbox and pull the messages there to the Outlook.com mailbox.


My choice would be the second choice: configuring the Yahoo or Gmail account to forward to the Microsoft address and configuring a send-only account so I could respond to the messages and still have it appear to come from the Yahoo or Gmail address.


There are other options as well, but they depend upon exactly what you hope to accomplish.

To create the Index itself, you have to insert an INDEX field at the end of the document (or wherever you want it). You do that from the same dialog box (Insert->Index and Tables...->Index) but click OK instead of Mark Entry. (Not my idea of how to do things either, really.)

The thing to do with the Mark Entry stuff is (I think) to

  •  leave the Mark Entry dialog box on screen
  •  after the Breakfast Club entries have been done, in the body of the document, select the name of the next movie
  •  click in the Main Entry field in the dialog box

allen browne say:

Value is a reserved word, so I suspect this is where Access is getting confused.

Try enclosing the name in square brackets, i.e.:
    TRANSFORM First(item_attribute_link.[value]) AS FirstOfvalue ...

Since the name is aliased, that should solve it in this context, but in general it's best to avoid reserved words are field names. Here's the list I personally refer to when designing tables so as to avoid this kind of problem:

It's not that Access is not backwards compatiable, it's that prior versions of Access *tolerated* you using Reserved Words.  But now that Access is all grown up it does not stand for it anymore and in some cases even putting brackets around does not solves the issue.

Once you review the list quite a few of your field names and table names are Reserved Words which could be a big part of the problem.

Things to try...

1.  Make a copy of the query and turn it into a select query and see if it runs

2.  Check References and make sure none are marked MISSING and resolve those

3.  Create a table without Reserved Words and see if you can create a crosstab query
