Hi Cbhtcomputers,

The default date range value of the published Office 365 calendar is 6 months. If you want to change the value to a longer period, you need to be an Office 365 administrator in your organization. Here are the steps to change it:
1. Connect to Exchange Online with remote PowerShell.
2. Run the command below.

Set-MailboxCalendarFolder -Identity *** Email address is removed for privacy ***:\Calendar –PublishEnabled $true -PublishDateRangeFrom SixMonths -PublishDateRangeTo OneYear 

Note: 1. Replace *** Email address is removed for privacy *** with your own email address.

2. The date range value can only be change to these values: OneDay, ThreeDays, OneWeek, OneMonth, ThreeMonth, SixMonth, OneYear. So you can extend the value to at most one year.

If you encounter any problem during the procedure, feel free to let us know and we will assist you further.

1) Clear the cached credentials in the Windows Credential Manager (or using keymgr.dll, not sure if it matters which you use)

2) Reboot (not 100% sure this is necessary, but had some mixed results)

scottgem say:

While this can be done, I would change your structure. Since the image name is always prefixed by IMAG, why store it? Make your image number a Long Integer datatype. Then it will be easy to increment. 

Dim strSQL As String

strSQL = "INSERT INTO tablename (ObsTime, ImageNumber, field3, etc.) " & _

               "VALUES(#" & DateAdd("n",1,Me.ObsTime) & "#, " & Me.ImageNumber+1 & ", " & _

               Me.control3, etc. & ");"

CurrentDB.Execute strSQL, dbFailOnError

The above code will add a new record with an incremented ObsTime and ImageNumber. The rest of the fields will be copied from the current controls. 

To display the number with the prefixe use the expression:

"IMAG" & Format(ImageNumber, "0000")

ken sheridan say: It's not clear whether your ObsTime values include a date, and are formatted as hh:nn, or whether they are the 'time value' only.  If the former I'd suggest the following amendment to Scott's code:

strSQL = "INSERT INTO tablename (ObsTime, ImageNumber, field3, etc.) " & _
               "VALUES(#" & Format(DateAdd("n",1,Me.ObsTime),"yyyy-mm-dd hh:nn") &_
               "#, " & Me.ImageNumber+1 & ", " & _
               Me.control3, etc. & ");"

As written the code would fail in the UK and elsewhere not using the US short date format.  Formatting the value to the ISO standard for date/time notation internationalizes the code.

If the latter, then you should understand that there is in reality no such thing in Access as a 'time value' only a date/time value; it is really the time on 30 December 1899, which is 'day zero' in Access's implementation of the date/time data type.  So the code would then be:

strSQL = "INSERT INTO tablename (ObsTime, ImageNumber, field3, etc.) " & _
               "VALUES(#" & Format(DateAdd("n",1,Me.ObsTime),"hh:nn") &_
               "#, " & Me.ImageNumber+1 & ", " & _
               Me.control3, etc. & ");"

I have one question, however.  What happens when the time reaches 23:59?  In scenario 1 above, where the values include a date, then the value would increment to midnight, 00:00, at the start of the following day; in scenario 2, where the values are 'time values' they would revert to 00:00 at the start of 1899-12-30.

Hiya,

(all of this can be found if you trawl thru old threads looking for n97 memory issues).

 

The N97 is still a very capable phone IF you're willing to nursemaid it a little. I'm on my 3rd year with this phone and still getting lots of use and all the functionality I want from it.  You're totallly right in that this was (arguably, still is) a brilliant phone wrecked by a stupidly small amount of RAM on the C Drive. Not helped by the way that everything Nokia (and QuickOffice) insists on being installed on that tiny C Drive. If you want it to work well you'll have to be prepared to put in some time in setting it up and a little occasional maintenance to keep it that way.

 

Memory Reorganiser removes itself after 1 use.

If you've made a lot of changes/moved things around/installed a few new apps it might be worth rerunning to regain 1or 2 extra mb on the C drive.

If you haven't run it before, try 'Software Update' (in applications menu) it should show up as an option to download.

If you have run it before, try 'Settings>Application Mgr>Installed apps'  and from the list shown hightlight 'Reorganiser Uninstaller' and uninstall that (that's the thing that uninstalls the memory reorganiser and lets software update know it's been run).  Memory reorganiser will now be offered as an option the next time you run software update.  This process can be repeated as many times as you need.

 

 

FIRST though I be inclined to sort out any other problem areas and use memory reorganiser as final 'polish'.

I'd be aiming for around 25 to 30 mb free on C.

 

1. Ideally you'd start from a reset,

2. Check your firmware is the best you can get (software update),

3. Set messages to use 'E:Mass memory' NOT C

4. Delete all the applications that you know you will never want (if you use application manager to  do this it's easier to spot which ones are saved on the C Drive). Top Gear, Tetris, Guitar Hero should be the first to go.

5. DO NOT OPEN BOINGO! Create a sub-folder called trash and then move Boingo to it so that you never open it by accident!

6. Don't install QT!  It takes up 13mb on the C Drive.  When you're downloading Apps from OVI Store they may have a message/prompt saying 'this will involve a download of 13mb'  that means it uses QT and is going to download it to your C Drive. This will condemn your n97 to the slow lane. It's a shame 'cos some of the QT stuff is very cool (especially the talking hamster!).

7. Remove OVI Maps. Then use software update to reinstall the latest version (it's getting better and better).

 

More detail on how to do any of the above can be found in the pool of knowledge on this forum:

/t5/Pool-of-Knowledge/bd-p/topfaq

(don't want to get told off for spamming because I'm repeating too much of what's already out there!)

 

As for on-going maintenance, just remember to turn it off once a week!

 

All of that will take time, from memory it was about an hour and a half last time I did it.  But the end result will be a phone that _can_ do more than the Iphone4.  The N97 doesn't like live updates on the homescreen - things like facebook/Accuweather will either be unreliable or will cause the phone to stall.  The interface can be confusing for the simple minded. And the phone will pause for thought every once in a while. Oh, and the Iphone is generally considered to be prettier.

In 'my phone is better than yours' conversations the list of things that the N97 can do will easily beat the Iphone tho.  If I feel that the Iphone owner might be thinking they're winning I just flip it open and shut a couple of times and then start typing on the excellent keyboard.
