rich007a say:

Hi,

No need to loop...

The first empty cell in column C is given by:

    Set r = Range("C1").End(xlDown).Offset(1)

but you need to take care in the case that C2:Clastrow are not populated then r will end up as the bottom row of the sheet!  Safest test is:

 

    'if col C has a header row use:
    If Range("C2").Value = "" Then
        Set r = Range("C2")
    Else
        Set r = Range("C1").End(xlDown).Offset(1)
    End If
    
    'if col C does not have a header row use:
    If Range("C1").Value = "" Then
        Set r = Range("C1")
    ElseIf Range("C2").Value = "" Then
        Set r = Range("C2")
    Else
        Set r = Range("C1").End(xlDown).Offset(1)
    End If

 

Alternatively, you can use special cells (I believe these are reliable in Excel 2010):

 

    Set r = Columns("C:C").SpecialCells(xlCellTypeBlanks).Cells(1)
 

but that will only return truely blank cells, i.e. not any cell containing a formula that has returned "".

Rich

ken sheridan say: You can loop through the querydef's Parameters collection and evaluate each parameter:

    Dim dbs As DAO.Database
    Dim qdf As DAO.QueryDef
    Dim prm As DAO.Parameter
    Dim rst As DAO.Recordset
   
    Set dbs = CurrentDb
    Set qdf = dbs.QueryDefs("qry_AMIS")
   
    For Each prm In qdf.Parameters
        prm = Eval(prm.Name)
    Next prm
   
    Set rst = qdf.OpenRecordset
   
The parameters should be references to controls on an open form.

Deep Virus Scan

How do you change from quick scan to deep scan?

greginmich say:

How do you change from quick scan to deep scan?

You don't. Theres no option for a scheduled Full Scan in the new Windows Defender. Thats by design because the Windows Defender Scheduled Scan task is now run as a component of the Automatic Maintenance package and Automatic Maintenance tasks are only allowed to run in limited time windows when the system is idle. There is a popular hack for adding a Full Scan by adding the Full Scan parameter to the command line for the Windows Defender Scheduled Scan task; but that hack inadvertently forces Automatic Maintenance to run a daily Full Scan, and that could easily prove to be an impossible chore given the limitations imposed on Automatic Maintenance. So the only legitimate alternatives would be running your Full Scans manually or running them with an independent scheduled task.

 

What works well for most users is just running an occasional manual Full Scan with Windows Defender in conjunction with occasional scans with some of the reputable third-party malware-removal tools. Truth be told, the malware-removal tools are probably more likely to find malware than a Full Scan with Windows Defender.

Nevertheless, its not all that difficult to schedule a Full Scan with the Create Basic Task Wizard, and Ive provided some links in this thread that can walk you through the setup:

http://answers.microsoft.com/en-us/protect/forum/protect_defender-protect_start/windows-10-how-do-i-set-windows-defenders-scan/f399bb87-37c9-4a74-bb35-1d7e3d3c2436?page=2
