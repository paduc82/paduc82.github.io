jack cook - aka help_line say:
                What is "Other" under Storage?

It is a mixture of files that is reserved space for apps and games.

 

Todd gave a good explanation in this post:

http://answers.microsoft.com/en-us/winphone/forum/wp8-wpstart/what-is-other-under-storage/979d2948-249b-4303-a475-4183b15af2d6

 

 

A lot of the built-in apps on Windows Phone allocate storage dynamically, so if the phone is relatively empty, they'll reserve a lot of space for themselves (IE and Maps are pretty guilty of this.) However, as you add media and apps to the phone, IE and Maps will release some of their space back to the phone. It sounds funny, but makes sense when you think about it, but MS' philosophy is that "empty" space on the device isn't doing anybody any good. As long as nothing else is using it, the OS will use it for various caching to speed up the device. Some of my commute is through a cellular "dead zone" but my maps app never misses a beat because the maps for that area are cached away on the device somewhere, in that "other" storage that currently has nothing better to do.

 

If you install the Windows Phone desktop app you will see that in that app they have broken the way "other" is displayed in that they have taken out the OS and have that as a separate category.

deanh say:

The XRef will alway reflect exactly what you have in the caption label and there is no built-in way to uncapitalize the XRef from a capitalized caption label.

Work around, you can have a XRef without the caption label by adding "\# 0" to the XRef field, such as:

Figures {REF _Ref225905825 \# 0 \h }, {REF _Ref225905827 \# 0 \h }, and {REF _Ref225905829 \h }

this returns the following in my test:

Figures 1, 3, and Figure 4

But to do this in a large document means a lot of editing, which you can do with find/replace, with the Fields all Showing. Then you will have to add the text "figure" to the start of each one, again find replace could help.

Personally I like the Figure XX to stand out in the text.

Best of luck. :-)
