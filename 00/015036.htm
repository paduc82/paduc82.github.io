Task View and Virtual Desktops: How should they work? - Microsoft Community

supervolcano say:

Task View and Virtual Desktops: How should they work?

Virtual desktops are one of the greatest productivity tools a computer can offer. As such, they need to offer enough versatility to appeal to every user.

I've been thinking about what I would like to see as options for virtual desktops. A few things have come to mind. (Note: I haven't had the chance to install the Technical Preview, so forgive any ignorance.)

The main split will be between people who use virtual desktops for arranging a screen of programs, and those who use it to arrange groups of programs. The former will want all their programs accessible from the taskbar at all times, the latter will prefer it only display what they're currently working with.

I'm very pleased that this is something you've made an option, and I trust you to make the right decision on what should be default. (I can't help with that decision).

There are two options that I think would enhance the experience for the latter group of the above paragraph.

1. An option to make a virtual desktop "permanent". With this, multiple desktops would be preserved between reboots, and not be easy to accidentally delete. Why would this be a good option? Users may want to have different applications pinned to each virtual desktop's taskbar. I currently have fifteen programs pinned to my taskbar, but if I were to split up my work into multiple desktops, I may want to split these pinned applications between them as well. Then again, I may not. We're talking about the option, here.

2. An option to have the same instance of a program open on multiple desktops simultaneously. I may want programs A, B and C open on one desktop, and C, D and E open on others. Given this has massive potential to confuse some users, it should be an option "Allow an instance of a program to run in multiple virtual desktops at once" that is disabled by default. Enabling would add the appropriate functionality to Task View.

Next, how should virtual desktops be handled with multiple monitors? Simple - an option to treat all monitors as a single virtual desktop, or an option for each monitor to have its own virtual desktops. With these options, users could have different applications pinned to the taskbar on each virtual desktop within each screen, resulting in over half a dozen different taskbar layouts, and find this brilliantly productive. Alternatively, they could always have the same taskbar layout on each monitor all the time. It'd be entirely up the user, and only require a few simple options.

I believe this would provide the perfect combination of simplicity, consistency and user control, and would make a vast majority of users happy.

What do you think, everyone?
