HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\Windows NT\CurrentVersion\ProfileList

6. Exit Registry Editor.

7. Log off the system.

8. Log on to the system again.

    Schannel error logging is on (set to 0x0001) by default.

    In the event viewer, I found quite a few Schannel events with Event Id 36888 that seemed to correspond to my failed connections.  In the General description section it said "

    A fatal alert was generated and sent to the remote endpoint. This may result in termination of the connection. The TLS protocol defined fatal error code is 40. The Windows SChannel error state is 808.

    Error code 40 is handshake_failure Indicates that the sender was unable to negotiate an acceptable set of security parameters given the options available. This is a fatal error.

    But the interesting part was the error state 808.  Google had only one relevant link https://social.technet.microsoft.com/Forums/en-US/7ceacf85-7eb4-43d6-8758-a811b8de39fd/schannel-errors-with-outlook-2013-when-may-2015-critical-updates-performed-channel-error-is-808-and?forum=w8itprogeneral

    Even though it was about Outlook on Windows 8.1, it turned out to be the correct solution.  Windows 8.1 and 10 require a minimum Diffie-Hellman key length of 1028.  Apparently my server has not been updated to this and is still set at 512.

    The link provides a registry edit to override the minimum value and set it back to 512 (00000200):

    1. Click Start, click Run, type regedit in the Open box, and then click OK.

    2. Locate and then click the following subkey in the registry:

    HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Control\SecurityProviders\SCHANNEL\KeyExchangeAlgorithms\Diffie-Hellman

    Note: if the Diffie-Hellman key does not exist, add it.

    3. On the Edit menu, point to New, and then click DWORD Value.

    4. Type ClientMinKeyBitLength for the name of the DWORD, and then press Enter.

    5. Right-click ClientMinKeyBitLength, and then click Modify.

    6. In the Value data box, type 00000200, and then click OK.

    Exit Registry Editor, and then restart the computer.

    praseetha k say:

    Hi Richard Lant,

    • Since when are you facing this issue?

    • Does this happen only with Nero or with any other applications?

    Aclayer.dll has been reported as a Trojan before and it will pose itself as a Microsoft BHO (browser helper object).

    Step 1: You may run a virus scan and check if it helps:

    http://onecare.live.com/site/en-us/default.html

    Step 2: You may also try to manually disable the tool bands and Browser Helper Objects, follow these steps:

    1. Open Internet Explorer and click tools.

    2. Double-click Internet Options.

    3. Click the Advanced tab.

    4. Under Browsing, click to clear the Enable third-party browser extensions (requires restart) check box.

    5. Restart Internet Explorer.

    If the above step doesn’t work, then you may try to delete the aclayers.dll. from the following locations:

    C:\WINDOWS\system32\dllcache and delete aclayers.dll.  Then delete aclayers.dll from C:\WINDOWS\AppPatch folder.

    Hope this information is helpful.
