cbarnhorst say:

The product keys used in the early days of the insider program are now blocked, so those in the old emails don't matter.  Microsoft replaced those old keys with "digital entitlement" in the Summer of 2015 and warned all insiders to re-activate using a Windows 7 or 8.1 product key (which would establish DE).

As Lacrumb points out, a new mobo will not activate the Windows 10 on your hard drive.  Your Windows was activated by "digital entitlement" beginning in late 2015 and not by a product key, meaning your copy of Windows was tied to the computer's registration as a Windows 10 device.  Each Windows 10 device is registered with the unique identifiers of the hardware (mobo serial number, etc.) in order to prevent piracy.  A new mobo has its own unique identifiers, which are not yet registered.  The registration of the old mobo cannot be used for the new one.  That is the meaning of Microsoft's statement with the introduction of digital entitlement that your Windows 10 was activated "for the life of the computer."  When you remove the mobo, the license goes with it.

Others may have suggestions to try, but the free upgrade to Windows 10 is not transferrable to a new computer.

norman david jones say:

Hi emjhae12,

yes. the COLUMN A contains non-formula values, and COLUMN B contains returned values from a formula within the same worksheet.

Try replacing the code in the code module of the sheet of interest with the following Worksheet_Chsnge procedure:

'=========>>
Option Explicit

'--------->>
Private Sub Worksheet_Change(ByVal Target As Range)
    Dim Rng As Range, Rng2 As Range, Rng3 As Range
    Dim Rng4 As Range, Rng5 As Range, Rng6 As Range
    Dim rCell As Range
    Dim arrIn() As String
    Dim i As Long
    Dim sMsg As String

    With Me
        Set Rng = Intersect(.Columns("A:A"), .UsedRange)
        Set Rng2 = Intersect(.Columns("B:B"), .UsedRange)
    End With

    Set Rng3 = Intersect(Rng, Target)
    On Error Resume Next
    Set Rng4 = Rng2.Precedents
    On Error GoTo 0

    If Not Rng4 Is Nothing Then
        Set Rng5 = Intersect(Union(Rng2, Rng4), Target)
    Else
        Set Rng5 = Intersect(Rng2, Target)
    End If

    On Error GoTo XIT
    If Not Rng3 Is Nothing Then
        Application.EnableEvents = False
        For Each rCell In Rng3.Cells
            With rCell
                If IsEmpty(.Offset(0, 1)) _
                   Or .Offset(0, 1).Value = vbNullString Then
                    If .Value <> vbNullString Then
                        .ClearContents
                        i = i + 1
                        ReDim Preserve arrIn(1 To i)
                        arrIn(i) = .Address(0, 0)
                    End If
                End If
            End With
        Next rCell
    End If

    If Not Rng5 Is Nothing Then
        Set Rng6 = Intersect(Rng2, Rng5.Dependents)
        Application.EnableEvents = False
        For Each rCell In Rng6.Cells
            With rCell
                .Select
                If .Value = vbNullString Then
                    .Offset(0, -1).ClearContents
                End If
            End With
        Next rCell
    End If

    If CBool(i) Then
        sMsg = Join(arrIn, ",")
        MsgBox Prompt:="Entries in the following cells" _
                     & " have been  cancelled because no " _
                     & "entry exists in the corresponding " _
                     & "cell in column B" _
                     & vbNewLine _
                     & sMsg, _
               Buttons:=vbInformation, _
               Title:="INVALID ENTRIES"
    End If
XIT:
    Application.EnableEvents = True
End Sub
'<<=========

[Edit]

The MsgBox text has been edited at the 2 points in bold to correct typos.

when i turn it on it cmes on for a few seconds and goes back off  waht do you think is wrong with my phone


index.jpg

thetao---01 say:

If you look very closely (check in different lighting conditions), do you see anything on the outer screen at all?  You probably won't be able to fix this yourself, but you might be able to determine if something is loose.  Press a button on the phone, which would normally make the outer screen light up.  Hold the phone firmly in one hand and smack it into your other, outstretched hand at different angles (i.e. smack the right side, left side, top, etc. several times).  If the screen comes back on even for a moment, then something is just loose, and the repair shouldn't be costly.  If you can see a faint image on the screen in some lighting condition, but smacking the phone doesn't help, then the problem is the backlight and you will probably need a new screen.  If you can't see anything on the screen at all, and smacking doesn't help, then the problem is unknown.  A good repair shop will be able to identify the problem, and possibly even without dismantling the phone.

 

I hope that helps!

 

P.S.  Is there a separate "Power saver time-out" or "Light time-out" for each screen?   This would also be something simple to check.  I used to carry an E90 for work, but don't remember.
