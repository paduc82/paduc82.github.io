ken sheridan say: What do you mean by 'update'?  It does not sound like you are using the term in a conventional sense of a value or values in one or more columns in the table being changed.  My impression is you are talking about data being imported from Excel.  Right?  If so how is the import undertaken?  You refer to opening a spreadsheet, so is it done manually?  Or is it automated in code?

If it is automated then you can assign the current date by executing an 'update' query in the same code which imports the Excel data, if it's done manually then the user will need to insert the date.  Either way you'd need to store the dates in a table like this:

LastImportDates
....TableName (PK)
....ImportDate

You'd first fill the table with all the table names and the dates when last imported, so if we assume as an example that there are only two tables involved, Customers and Orders, you'd have rows like this:

Customers    2013-06-03
Orders         2013-06-10

I've used the ISO date format of YYY-MM-DD here to avoid ambiguity; you'd normally enter the dates in your local regional format of course.

Next to each button you'd have a text box, e.g txtCustomersImportDate for the customers table, and txtOrdersImportDate for orders. Note that the table name within the control name must exactly match the name of the table as stored in the LastImportDates table.  In the form's Open event procedure you'd set the values of the controls like this:

Const conSQL = "SELECT TableName, ImportDate FROM LastImportDates"
Dim rst As DAO.Recordset

Set rst = CurrentDb.OpenRecordset conSQL

With rst.
    .MoveFirst
    Do While Not .EOF
       Me.Controls("txt" & .Fields("TableName") & "ImportDate") = .Fields("ImportDate")
       .MoveNext
    Loop
End With

To update the date in a text box you could put code in its BeforeUpdate event procedure, e.g for the txtCustomersImportDate text box:

Const MESSAGETEXT = "Change import date?"
Dim ctrl As Control
Dim strSQL As String

Set ctrl = Me.ActiveControl

If MsgBox(MESSAGETEXT, vbYesNo+vbQuestion, "Change Date") = vbYes Then
    strSQL = "UPDATE LastImportDates " & _
        "SET ImportDate = # " & Format(ctrl,"yyyy-mm-dd") & "# " & _
        "WHERE TableName = ""Customers"""
    CurrentDb.Execute strSQL, dbFailOnError
Else
    Cancel = True
    ctrl.Undo
End If

You'd do the same in the DblClick event procedures of text boxes next to the other buttons, modifying the SQL statements accordingly in each case.  The user would then simply have to enter a new date in the text box and confirm it at the prompt to change the date in the table to the new one entered in the text box.  If they answer No at the prompt the original date will be restored.  Ideally you'd also trap any error when the code attempts to execute the SQL statement and restore the date by undoing the control.

2.  Select the relevant event property and select the 'build' button (the one on the right with 3 dots).

3.  Select Code Builder in the dialogue and click OK.  This step won't be necessary if you've set up Access to use event procedures by default.

4.  The VBA editor window will open at the event procedure with the first and last lines already in place.  Enter or paste in the code as new lines between these.

As regards 'updating' the data you do appear to be doing what I thought, rather than the day to day updating of individual items of data by users, which is what I meant by 'conventional'.  As only one table is involved, however, it's very much simpler to do, as you do not need to loop through a set of table names and assign values to multiple text boxes.  You merely need one row in the LastImportDates  table and one text box on the form, and the code in the form's Open event procedure would be just one line:
