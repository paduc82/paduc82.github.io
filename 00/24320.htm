Hi Chau,

To clarify, a tenant is an Office 365 work or school account, but abc.com and xyz.com are domains that can be added into the Office 365 work or school account and can be set as the suffix of user accounts. We can add many domains into one Office 365 work or school account, and all those domains will be managed by the global admin.

Given the situation, it's not feasible to separate two domains in each admin page under the same Office 365 work or school account.

The admin roles under the same tenant in Office 365 are divided depending on different services rather than domains. lenny33 say:

I am migrating my engineering documents into 2010.  Depending on the type of form, I will be leaving the form fields and vb alone.  Others, because or requirements to use most of the functionality of Word, am migrating to content controls and read-only.

 

So far, in a test document, I am able to replicate the dropdowns, bookmarks and default text, however, one of the vb fields references an On Exit Macro.... is there a way to reference the macro so it pulls up the select case entries from the content control?  I have not been very successful in Googling a direct answer to this, if it can be done at all....

 

Clarification please?

 

Regards - Lenny33

jay freedman say:
Content controls don't have exit macros in the same way as legacy form fields. Instead, there is a Document_ContentControlOnExit event that fires whenever the focus exits from any content control anywhere in the document. The handler for the event has a standard set of arguments that Word populates when the event fires, and the first argument is the control that was just exited.

A good description with sample code is at http://gregmaxey.mvps.org/word_tip_pages/insert_content_with_content_controls.html.

I'm not quite sure what you mean by "the select case entries from the content control". If you're dealing with a combo box or list box content control, your code can use the passed-in content control argument to get the item the user selected.
