trevor rabey say:

What you have done is quite a common way to approach it if you have the work for each resource and you want to force a duration.

However, what I don't like about the approach is how people get the work estimate to start with. I am supposing that, as you say, that you have the work per resource per task. Sometimes you get given just the work per resource or just the work per task.

I always prefer to start with the durations, link the predecessors and successors so that you get the critical path network right, then assign the resources, and then the work per task and the work per resource are calculated as an output, not an input.

Anyway, to come to your "problem" about the summaries, provided the tasks and the summary are all auto-scheduled, and provided calculation is on, there is no reason why the summaries durations should not reflect changes in the durations of the tasks. Are you sure the summary is not working? Can we have see a screenshot?

hpholm say:

Hi HaroonZia,

The Outlook app currently only allows attachment of pictures.
It's a basic design in Windows Phone that an app is only allowed to store, access and use files in its own protected storage. This sandboxing of apps is a security and integrity measure, so the Windows Phone developers have to consider very carefully, even when they make an exception that no third-party developer is allowed to use.

Microsoft is relaxing this approach and allow apps to do more since WP 8.1, e.g. the Files app is designed to access files that are stored outside of other apps, and as a consequence it is able to copy, move and share what it can see.

The best I can do is suggest add votes and comment to the Uservoice forum:
https://windowsphone.uservoice.com/forums/101801-feature-suggestions/suggestions/2282574-enable-document-attachment-in-emails

https://windowsphone.uservoice.com/forums/101801-feature-suggestions/suggestions/2281161-delete-edit-original-message-when-reply-forward-th

- or create your own specific suggestions.

The Windows Phone developers monitor the uservoice forum and I expect changes in Windows 10 based on suggestions found there.
