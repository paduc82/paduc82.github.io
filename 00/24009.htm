robert aldwinckle on forums say:

Hi, on my favorites bar I suddenly have the .url extension after every shortcut name.


This is something that we used to have control over in XP using Folder Options, File Types.  In Vista and W7 that tab is gone.  So I suppose the new GUI method of undoing that non-standard change would be to use Folder Options, View tab, Restore Defaults.  Or try simply checking  Hide extensions for known file types.  Alternatively you could download and use FileTypesMan from NirSoft.

FWIW I have that Folder Options option unchecked and I don't have either .lnk or .url extensions show, so clearly there is another registry value involved.   FileTypesMan could help discover it, e.g. using ProcMon to watch...

Using FileTypesMan to Edit the .URL extension by checking Always show this file extension showed three values being changed:

HKCR\IE.AssocFile.URL\EditFlags

HKCR\IE.AssocFile.URL\AlwaysShowExt

HKLM\SOFTWARE\Microsoft\Windows\CurrentVersion\Explorer\GlobalAssocChangedCounter

However, that wasn't enough to change the display of URL extensions in IE.

BTW undoing it shows  AlwaysShowExt  being deleted.

RegEdit shows that that registry branch also contains a  NeverShowExt  value present and FWIW I suspect that is what will be preventing the above change from working.  I am not going to bother testing that idea, e.g. by deleting that value.  Only a few file types have that value in them, so I assume that is the mechanism for overriding the global unchecking of Hide extensions for known file types.

So, for your case I suspect you will find that the  NeverShowExt  value is missing and that the least you will have to do is add it back.   It doesn't have a data value; it's just there or not there.  Hopefully, using the Restore Defaults button would put everything, including that value name, back to normal.  I don't want to test that idea either.

 

Good luck

I appreciate everyone's input.  I think I figured it out with a lot of help from the forums mentioned above and Google.

=SUM(INDEX(Actual_All,MATCH($B10&$C10,Actual_Partner&Actual_Metric,0),3):INDEX(Actual_All,MATCH($B10&$C10,Actual_Partner&Actual_Metric,0),'Rev Retrieve'!$D$18+2))

shdwlynx say:

Here's what I figured out after banging my head on my desk for a while:

If you drag a photo from Windows Explorer (a.k.a. File Explorer in Windows 10+) onto Microsoft Word, it adds it as an object, which has far fewer picture editing options, such as Change Picture.

If you use the Picture button on the Insert menu, on the other hand, you get all of the picture editing options you expect, including Change Picture!

This is the code I currently use:


Public Sub SaveAttachments()
Dim objOL As Outlook.Application
Dim objMsg As Outlook.MailItem 'Object
Dim objAttachments As Outlook.Attachments
Dim objSelection As Outlook.Selection
Dim i As Long
Dim lngCount As Long
Dim strFile As String
Dim strFolderpath As String
Dim strDeletedFiles As String

    ' Get the path to your Documents folder
    strFolderpath = CreateObject("WScript.Shell").SpecialFolders(16)
    On Error Resume Next

    ' Instantiate an Outlook Application object.
    Set objOL = Application

    ' Get the collection of selected objects.
    Set objSelection = objOL.ActiveExplorer.Selection

' The attachment folder needs to exist
' You can change this to another folder name of your choice

    ' Set the Attachment folder.
    strFolderpath = strFolderpath & "\PAAube\Data project 2018\AUTOMATION\macros\477-478 promo\"

    ' Check each selected item for attachments.
    For Each objMsg In objSelection

    Set objAttachments = objMsg.Attachments
    lngCount = objAttachments.Count
        
    If lngCount > 0 Then
    
    ' Use a count down loop for removing items
    ' from a collection. Otherwise, the loop counter gets
    ' confused and only every other item is removed.
    
    For i = lngCount To 1 Step -1
    
    ' Get the file name.
    strFile = objAttachments.Item(i).FileName
    
    ' Combine with the path to the Temp folder.
    strFile = strFolderpath & strFile
    
    ' Save the attachment as a file.
    objAttachments.Item(i).SaveAsFile strFile
    
    Next i
    End If
    
    Next
    
ExitSub:

Set objMsg = Nothing
Set objSelection = Nothing
Set objOL = Nothing
