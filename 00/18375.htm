Using a vlookup to pull data and then preventing further updates

Dear Experts,

Can I use a formula to draw data from one worksheet to another and then have the values stored as values rather than formulae, thus "locking in" the data.

Say on a particular date the vlookup stores the data as values only, and prevents further updates to the vlookup.

I would like to pull data at particular time intervals; student average results every 3 months.

THank you

jc132568

shane devenshire say:

Suppose you want to enter the formula =VLOOKUP(A1,Sheet2!A1:D100,2,FALSE) into B1 of the first sheet, then the best way to do what you describe is use VBA

Sub GetData()
    Range("B1") = "=VLOOKUP(RC[-1],Sheet2!RC[-1]:R[99]C[2],2,FALSE)"
    Range("B1") = Range("B1").Value
End Sub

Or if you wanted to enter a similar formula in B2:B100 it would look like this:

Sub GetMoreData()
    Range("B1:B100") = "=VLOOKUP(RC[-1],Sheet2!R1C[-1]:R100C[2],2,FALSE)"
    Range("B1:B100") = Range("B1:B100").Value
End Sub

 You would just rerun the macro every quarter.

    Word 2010 does not release Compatibility Mode - Microsoft Community

    bug or hidden fix say:

    Word 2010 does not release Compatibility Mode

    OS: Windows 7 Professional or Windows 7 Enterprise

    Bit: 32-bit or 64-bit

    Internal workgroup type: Domain Environment

    Office Suite: Office Professional Plus 2010 (Office 14)

    Updates: Tested on both unpatched, fully patched, non-Service Pack and fully Service packed Windows 7 and Office 2010 environments.

    Issue: Word will not disable/remove [Compatibility Mode]

     

    In order to maintain a mandated directive - technology changes or improvements may not limit, reduce or prevent the availability of document access by the public -  we load Office Professional Plus 2010 [OPP2010] in a customized configuration in which all applications are set to function in [Compatibility Mode]., which we will call CM from here onward.

     

    With the CM being active, opening any application (START > Word; START > Excel; START > Powerpoint) will open the file object in [Compatible Mode].

     

    Changing the CM should be universal in the OPP2010 for each computer+user signed into the domain environemnt. If the user opens up either Word, Excel, or PowerPoint, and goes to FILE > OPTIONS > SAVE > changes the 'Save file in this format:' from its 97-2003 format to the 2010 format > OK > then close application and re-launch any Word/Excel/PowerPoint application, the CM will no longer be active; opening up the application from the START menu will open up a file object that is truly a 2010 file type (.DOCX, .XLSX or .PPTX)

    However, for some odd reason, only WORD will not go back to its normal 2010 state. Excel and PowerPoint will operate in the 2010 format, but Word will stay in the CM state; when you click SAVE on the file object, it will show a .DOC (or for templates, a .DOT format.

     

     

    Is this a bug, is there some hidden  instruction we are missing, or perhaps a hotfix is needed that we are not aware of?

     

    Thanks much.

     

     

    bug or hidden fix say:

    To Ronh007 and Graham:

     

    After reviewing your suggestions and comments â€“ both Rohn007 and Graham Mayor â€“ I was convinced this issue was in the behavior of our particular Word 2010 product.

    I decided to think outside the box, and investigated ways to reset Word, but not to reinstallation; as the reinstall process for us would produce the same results.

     I found a method to do just that, and re-engineered a process with our Office 2010; mainly through this registry key:

    HKCU_Software_Microsoft_Office_14.0_Word

     

    I exported it, studied it a bit, and decided to delete the Word key. THAT WORKED! Not only did the behavior of the Word 2010 now  easily go back and forth between normal and compatible modes, but all the original VBA code designed under .DOTM now worked as well!
