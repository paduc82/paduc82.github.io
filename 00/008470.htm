john korchok say:

I imagine you're using Legacy Form Fields to create this form, then protecting the document for Filling in forms. You have at least 2 ways to fix this.

One is to insert a continuous section break immediately before and after each heading that gets displayed in the TOC. Then when applying protection, use the Select sections link and uncheck the sections that contain the headings. Once they are in unprotected sections, the TOC can once again link to them.

Another is to use Content Controls instead of Form Fields. In a Content Control form, you do not protect for Filling in forms. Instead you select each content control in turn, choose No changes (Read only), check Everyone in the Exceptions section and click on Yes, Start Enforcing Protection. A TOC works normally with a Content Control form.

Please post again if you need more help constructing a Content Control form.

If you want to collect other custom domain’s emails to Office 365, you should add these custom domains to the Office 365. However, you should have Office 365 for Business subscription, then you can add these custom domain.

ranmo_msft say:

Hi Guruji,

1. Staged
You can have passwords synced to Office 365 if password sync feature is enable when setting up DirSync
2. Cutover
It will not sync password to Office 365. After the migration, you will get the migration statistics report which contains an autogenerated, eight-character alphanumeric password for each mailbox that was migrated. Users are required to reset this password the first time they sign in to their Exchange Online mailbox.
3. IMAP
Passwords will be migrated along with the user mailboxes.
4. Remote move.
The passwords can be synced if password sync feature is enable when setting up DirSync. This is same as Staged migration.
