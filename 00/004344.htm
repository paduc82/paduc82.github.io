Control tabs in workbook - Microsoft Community

silkcityflorida say:

Control tabs in workbook

I have a workbook with many sheets.  Most of the sheets relate to calendar months.

When the user opens the workbook, code always activates a specific "Header" sheet, which is a focal point in the workbook.  After using the various sheets, the user will typically end up back on the Header sheet.

There is a high probability that the next thing the user will want to do is go to this month's sheet, or possibly 1 or 2 months prior, or 1 or 2 months hence.  So,while the user is on the Header sheet, I want to position the tabs to display the ones for sheets "This Month" - 2 through "This Month" + 2.

The Window.ScrollWorkbookTabs method sounded like it might work.  With it you can scroll to the first tab, the last tab, or an offset.  Unfortunately, the offset is from the current tab view.

So, for example, Sheet 5 is the active sheet.  The tabs for sheets 10 through 15 are in view in the tab area.  I use ActiveWindow.ScrollWorkbookTabs Sheets:=20.  The offset is not from Sheet 1, not from the Active Sheet 5, but from the current tab view, so, while I am still looking at Active Sheet 5, the tabs in view now are for sheets 30 through 35. 

I have tried using code to activate Sheet 5, then calculate an offset from there, but to no avail.

Is there a specific command I can use to indicate which tab should be first in view in the tab area?

silkcityflorida say:

I have done some additional exploring.

To reiterate, I would like to be able to:

  1. Specify which tab should be first in view in the tab area with a single command
    OR
  2. Query the name of the first tab currently in view with 1 command and then scroll the tabs as an offset from there with a 2nd command

And, I still hope for someone to tell me exactly how to do that.

In the meantime, though, I found a solution that was sort of right in front of my nose:

Sub Position_Current_Month_In_Tab_View()

    'The goal of this routine is to position the tab for the current month in the tab area,

    '   with 2 previous months before it and 2 following months after it.  So, if the current month is July 2015,

    '   the first 5 tabs should be May 2015, June 2015, July 2015, August 2015 and September 2015.

    'Remember also that this has nothing to do with changing the sheet the user is looking at,

    '   it only has to do with pre-positioning the tabs in anticipation of the user's next move

    'Don't let the user see the tab view first jump to the first sheet and then jump again to the target sheet

    Application.ScreenUpdating = False

    Dim intScrollOffset As Integer
    Dim dtCurrentSheetMonth As Date

    'The Month sheets in this workbook  all have a cell (B1) that contains the date of the 1st day of that month

    '   so to find the tab for the current month we have to set up a comparison value 

    '   equivalent to the 1st day of the month

    dtCurrentSheetMonth = DateSerial(Year(Now()), Month(Now()), 1)

    'ActiveWindow.ScrollWorkbookTabs only works within the range of VISIBLE sheets

    '   so we count the number of visible sheets, stopping when we reach the sheet for the current month.

    intScrollOffset = 0
    For i = 1 To ActiveWorkbook.Sheets.Count
        If ActiveWorkbook.Sheets(i).Visible = True Then
            If Sheets(i).Range("B1") = dtCurrentSheetMonth Then
                Exit For
            End If
            intScrollOffset = intScrollOffset + 1

        End If
    Next

    'The intScrollOffset at this point would position the current month as the first tab in view,

    '   but we want the first tab in view to be 2 months prior to that.  To do that,we need

    '   to decrement intScrollOffset by 2.

    intScrollOffset = intScrollOffset - 2

    'Now that we have the desired offset, as measured from the 1st visible sheet in the workbook,
    '   we first set the first tab in view in the tab area to the first sheet in the workbook.

    ActiveWindow.ScrollWorkbookTabs Position:=xlFirst

    'Then, we set the first tab in view in the tab area as an offset from there.

    ActiveWindow.ScrollWorkbookTabs Sheets:=intScrollOffset

    'Now we show the user the resulting tab view

    Application.ScreenUpdating = true

    'And the tabs in the tab area are May 2015, June 2015, July 2015, August 2015 and September 2015.

End Sub

The above routine is primitive, but it works.  I am pretty sure I can improve on it and will post that code when I finish it.  

In the meantime, feel free to jump in with more sophisticated code if you like!

For Cell L23, the formulas you use for conditional formatting would be:

<90%:

=(L23 < 0.9)

90% - 99%:

=AND(L23>=0.9,L23<=0.9999)

100%:

=(L23 > 0.9999)

This assumes that L23 is a true percentage, meaning the values are from 0.0 to 1.0, and are formatted as percentages.  If you literally have the string "92%" in the cell instead, that's a little more difficult!

eli qian msft say:
Hi Angela,
 
There are four types of groups that are available within Office 365
1.Office 365 groups are new and let you create a shared space to communicate, collaborate, and schedule events with colleagues on a shared task, project, or resource.
 
2.An Office 365 security group is used to grant access permissions to all users within a group. 
 
3.Mail-enabled security groups are similar to the security groups, but mail-enabled. These groups are also used to grant access to the resources in Active Directory and because it is a security group you can grant permissions to all of the members of the group at the same time. 
 
4.Distribution lists are also known as distribution groups and are the most common and well known type of groups.This type of group is used to send email to multiple users at one time.
 
You cannot use distributions list to manage permissions. I would suggest you use other security groups to manage permissions.
