TSV enabled MSa with 1 active proof and have a 25 Digit Recovery code

 Scenario: If you forgot the password then your Two step verification (TSV) is enabled and you have one active security info (either phone number or alternate email address) and a 25 digit recovery code, here is a step by step process on how you can reset your Microsoft account password.

1. Go to the password reset page : https://account.live.com/pw 

    Then select "I forgot my password" option and click NEXT.

 

 2.  Enter the complete email address that you need to recover and the characters given on the page, then click NEXT.

3. Then select the one active/accessible security info to request for a code.

4.  A 7-digit code will be sent to the active proof you selected. Check the proof (alternate email address / phone number) you used to get the 7 digit requested code and enter it on the corresponding box.

5. Then you will be asked again for another code sent to your second proof. If the second proof is no longer active or inaccessible, please select "I don't have any of these" option and be ready with your 25 digit recovery code.

6. Then you will be on the page where you can enter your 25 digit recovery code. Kindly enter the 25 recovery code correctly and click "Use recovery code" button.

 

7. Once the 25 digit recovery code is accepted, the next page is where you can enter your new password. After entering your new password, please click NEXT.

8. You will be again provided a new 25-digit recovery code. You may print the code and save it for future use.

 

9. The next page will notify you that your Microsoft account has been recovered. It is optional if you're going to enable your two step verification again. If not, just click on NEXT.

10. You will now be able to access your account normally using your new password.

ray gao msft say:

Hi Edward,

 

In Microsoft SharePoint lists and libraries, calculated fields that are running in the classic UI mode can be used to display results based on Excel-like formulas. This is a long standing capability, and is documented in Calculate data in lists or libraries.

 

Some users have added HTML markup or script elements to calculated fields. This is an undocumented use of the feature, and we will begin blocking execution of custom markup in calculated fields in SharePoint Online from June 13, 2017 onwards.

 

We are changing calculated fields to escape special characters. In some circumstances, this means that calculated fields that contain any unsupported markup will not display any value. Instead, they'll be blank in the list view.

 

This change will take effect on June 13, 2017. Administrators can request an extension through September 10, 2017 at the latest. During this time, special characters will not be escaped in calculated fields. This request can be submitted through Microsoft Support. However, beginning September 10, 2017, all unsupported markup will be ignored.

 

For more information, please refer to the following KB.

Handling HTML markup in SharePoint calculated fields

 

Thanks,

Ran
