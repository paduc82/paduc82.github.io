Special characters in conversion from WordPerfect - Microsoft Community

dorenburrell say:

Special characters in conversion from WordPerfect

My office has converted a very large number of legal documents from WordPerfect X5 to Word.  We are using Word 2013, but after the conversion these files initially come up in Word compatibility mode.  

The core problem is that the section symbol Â§ appears on the page display, but when macros try to search for this symbol (Alt-21 or Alt0167) there are no matches.  If one of us copies the symbol on the screen into the Find textbox, it shows up as an apostrophe.  We have attempted to fix the section marks by using Find and Replace to convert this false symbol into the standard Alt0167 symbol, but that also changes all apostrophes in the text.

Is there a way to fix the symbol conversion on a large batch of files?  Is there a special search instruction that can pull out the false section symbol without running a hit on an actual apostrophe?
jay freedman say:

I don't have a ready-made solution for you, but this thread has information that may help you.

The problem is (probably) that the original WordPerfect documents used the WP Typographic Symbol font. That font assigns nonstandard character codes to the glyphs for punctuation and other characters. In your example, the character code for an apostrophe (Alt39) combined with a font specification of WP Typographic Symbol results in the Â§ symbol.

If that is true of your documents, the following procedure will fix one document. And if that succeeds, the process can be made into a macro that will repair an entire folder full of documents.

  • Open the document to be used for the trial (or, better, a copy of a document in case something goes wrong).
  • Go to File > Options > Advanced, scroll down to the "Show document content" section, and click the Font Substitution button.
  • The "Missing document font" list probably shows WP Typographic Symbol -- if there are others, you can probably ignore them. Click on that line.
  • Click the "Substituted font" dropdown box and select a font that isn't used anywhere in the document, such as Copperplate Gothic Bold.

  • Click the Convert Permanently button, and click OK in the prompt message.
  • Close the Options dialog and save the document.
  • Open the Replace dialog, and click the More button to expand it.
  • In the "Find what" box, type an apostrophe.
  • Click the Format button and choose Font from the menu. In the Font dialog, choose Copperplate Gothic Bold (or whatever font you converted to). Click OK. In the Replace dialog, under the "Find what" box, it will now show that you're searching for that font. (Any apostrophes in other fonts won't be affected.)
  • In the "Replace with" box, type the section symbol (Alt0167). Click the Format button, choose Font, and select the font that's used in the rest of the document (e.g., Times New Roman).

  • Click the Replace All button.
  • OK the completion message, close the Replace dialog, and save the document.

How do I group radio buttons in a Word 2007 document, so that I can - Microsoft Community

pnutsmom say:

How do I group radio buttons in a Word 2007 document, so that I can check one button from each group?

I created radio buttons using Developer>Design Mode but I don't know how to 'grab' the radio buttons and put them in different groups so that they can be independently checked/unchecked.
jay freedman say:

Select one of the buttons and click the Properties button. In the list of properties, assign a value to the GroupName property.

Use the dropdown at the top of the Properties list to select another button that should belong to the same group. Put in the same value for GroupName that you used in the first button. Repeat this for all the buttons in the first group.

Now select a button that shold be in the second group. Assign a different value to its GroupName. Also put that value in the GroupName for the other buttons that belong to the second group. Each set of buttons that share the same GroupName value will behave as a group, and will be independent of all other buttons in the document.

jay freedman say:

That's a bit of a problem. I think the only thing you can do is to go into the option butons' Properties, click the Font property and then click the "..." button that appears, and set the font size smaller. Also check the Paragraph dialog of the paragraphs that contain the buttons, to make sure they're set to Single spacing with no space before or after. It may help to make the Caption of each button empty, and instead use regular text next to the buttons to label them.

The problem is this: If you change the text wrapping of the option buttons to Square or Tight instead of In Line With Text, the groups stop working -- every button becomes a group of one, and as soon as it's clicked and becomes selected, it can't be unselected.
