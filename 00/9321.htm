UEFI installation has GPT partition system whereas BIOS has MBR partition system. Therefore, you cannot repair UEFI installation by booting to legacy BIOS. You must convert GPT system to MBR system first.

Secure Boot may prevent booting to your previous OS build.

Close will close a file, but leave the application open. Exit will close the file and also exit the application. To handle a key violation error in the event of a conflict with another user I'd recommend that you use Roger Carlson's approach of trapping the key violation error in the form's Error event procedure.  This, with a few amendments, is illustrated in my demo as the 'Sequential Number' option.

The reason you are currently being prompted for the ticket number is that you are not concatenating the value of the Ticket_Number control into the string expression as follows:

    Dim strSQL As String

However, if you trap the key violation error, as my demo illustrates, you should not need a 'control' table.  Error handling like this is an integral part of a well designed database.
