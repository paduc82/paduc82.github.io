    hansv mvp say:

    The code should work in any version of Excel, but only if RowNo=1 and ColNo=1.

     

    If you enter new values in row 1 or column A, the named ranges will expand. But if you only enter new values in other rows and columns, the named ranges won't change, even if the new values are adjacent to the original range. Also, it requires that there are no blank cells in row 1 and column A within the range.

     

    If you only want to assign a name to the range containing the active cell, and have it dynamic, you could use the following code. It uses OFFSET, which mr. Govier tried to avoid:

     

    Sub CreateSingleName()
        Dim rng As Range
        Dim r1 As Long
        Dim c1 As Long
        Set rng = ActiveCell.CurrentRegion
        r1 = rng.Row
        c1 = rng.Column
        ActiveWorkbook.Names.Add Name:="MyData", _
            RefersTo:="=OFFSET(" & Cells(r1, c1).Address & _
                ",0,0,COUNTA(" & Cells(r1, c1).Address & ":" & _
                Cells(Rows.Count, c1).Address & "),COUNTA(" & _
                Cells(r1, c1).Address & ":" & _
                Cells(r1, Columns.Count).Address & "))"
    End Sub

    Secondly I would like the code to be for a single Dynamic Named Range using a 'selection'  but still using a header etc. 
    If you want to select a range, then this also means that you can define several different dynamic ranges. Therefore we need a prefix before each name to make it unique.

    And IMHO it's much easier to build the formulas if we use the range object.

    Andreas.

    Sub CreateDynamicNames()
      'Create dynamic names from Selection
      Dim R As Range, Sel As Range
      Dim BaseName As String, ColName As String
     
      'Get the selected cells
      Set Sel = Selection
      'Create a prefix to allow multiple dynamic ranges
      BaseName = "Base_" & Sel(1, 1).Address(0, 0) & "_"

      'Add a formula to count the columns
      ActiveWorkbook.Names.Add Name:=BaseName & "Columns", _
        RefersTo:="=COUNTA(" & Sel.Rows(1).Address & ")"
      'Add a formula to count the rows
      ActiveWorkbook.Names.Add Name:=BaseName & "Rows", _
        RefersTo:="=COUNTA(" & Sel.Columns(1).Address & ")"
      'Add a formula to refer the data inside the selection
      ActiveWorkbook.Names.Add Name:=BaseName & "Data", _
        RefersTo:="=" & Sel(1, 1).Address & ":INDEX(" & Sel.Address & "," _
        & BaseName & "Rows," & BaseName & "Columns)"

      'Visit each cell in the first row
      For Each R In Sel.Rows(1).Cells
        'Create a valid name from the heading
        ColName = ValidName(Trim$(R))
        'Skip empty cells
        If Len(ColName) = 0 Then GoTo Skip
        'Add a name for this column
        ActiveWorkbook.Names.Add Name:=BaseName & ColName, _
          RefersTo:="=" & R.Offset(1).Address & ":INDEX(" & _
          Intersect(Sel, R.EntireColumn).Address & "," & BaseName & "Rows)"
    Skip:
      Next
    End Sub

    Private Function ValidName(ByVal Name As String) As String
      'Replaces invalid chars in Name with "_"
      Dim i As Long, Digit As Integer
      For i = 1 To Len(Name)
        Digit = Asc(Mid$(Name, i, 1))
        Select Case Digit
          Case 48 To 57, 65 To 90, 97 To 122
            '0 to 9, A to Z, a to z
          Case Else
            Mid$(Name, i, 1) = "_"
        End Select
      Next
      ValidName = Name
    End Function

    How to create and remove password for VBA project? - Microsoft Community

    oemsoft say:

    How to create and remove password for VBA project?

    Does anyone have any suggestions?

    Thanks in advance for any suggestions :>

    jllatham say:

    I will take your question literally.

     

    First - you're not going to be able to do this with code if that's what you were thinking.

     

    Setting up a password for the VBA Project

    Open up your Workbook, get into the VB Editor.  Use Tools and choose VBAProject Properties and in the dialog that opens, choose the [Protection] tab.  Start by checking the box next to Lock project for viewing.

    Then it is a typical enter a password, repeat entry to confirm.  Click OK and it is done.

     

    Write the password down somewhere unless it is something you absolutely know beyond doubt that you cannot and will not ever forget.  The encryption used for the password to a VBA Project is much tougher than that which is used for protection passwords for a worksheet or workbook.  In this case, this is more of a security type password, rather than one to just keep someone from accidentally deleting a formula somewhere.

     

    I always recommend that you save one copy of a workbook that you distribute with a protected VBA Project without the protection applied along with the copy that is protected.  That way if you forget or lose the password, you world is not shattered.

     

    Now, even after doing the above, you will be able to continue working in the project.  But if you save and close the workbook, then when you reopen it and try to do work in the VBA Project you will have to provide the password.

     

    To Remove the Password from a VBA Project, you'll need to open the workbook and use the password to get back into the VBA Editor.

     

    Again use Tools --> VBAProject Properties and reverse the process:  clear that checkbox, and make sure there are no entries in the password entry/confirm areas, click OK and the password has been removed.

    Sub RowHeightChange()
    Dim RowNum, RowHeight
    Dim Ws As Worksheet
    Set Ws = ActiveSheet
    'Change password here
    Ws.Protect Password:="abc", UserInterFaceOnly:=True
    RowNum = Application.InputBox(Prompt:="Enter Row Number", Title:="Row Number Entry", Type:=2)
    If RowNum <> "" And RowNum <> False Then
        If Not IsNumeric(RowNum) Or RowNum < 18 Or RowNum > 43 Then
            MsgBox "Please Enter row numbers between 18 to 43 only"
            Exit Sub
        End If
        Else
        MsgBox "Please Enter row numbers between 18 to 43 only"
        Exit Sub
    End If
    RowHeight = Application.InputBox(Prompt:="Enter Row Height", Title:="Row Number Height", Type:=2)
    If RowHeight <> "" And RowHeight <> False Then
        If Not IsNumeric(RowHeight) And RowHeight > 409 Or RowHeight < 1 Then
            MsgBox "Row Height should be between 1 and 409"
            Exit Sub
        End If
        Else
        MsgBox "Row Height should be between 1 and 409"
        Exit Sub
    End If
    Ws.Rows(RowNum).RowHeight = RowHeight
End Sub
