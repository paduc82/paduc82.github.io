terrencejb say:

Errors blue screen stop code 0xc000021a

Cannot recover pc windows 10.

davem121 say:

It sounds like either winlogon.exe and/or csrss.exe maybe corrupt . . .

Start your System, then just after the Manufacturers logo disappears and Windows 10 tries to boot, press and hold the power button down for 5 - 10 seconds to perform a hard shut down

Do this Twice

On the third boot attempt, Windows will boot into the Windows 10 Repair Environment, from there you can access Startup Recovery, Safe Mode, Command Prompt . . . etc.

Go to Troubleshoot - Advanced Options - Startup Repair and see if your system can repair itself

If that does not work, go to Troubleshoot - Advanced Options - Command Prompt and run this command:

sfc /scannow

Wait for the command to complete, reboot your system and see if Windows can boot

If that does not work, go to Troubleshoot - Advanced Options - System Restore, see if you have a recent restore point you can revert to

If you do not have a recent restore point, go to Troubleshoot - Advanced Options - Startup Settings, click restart.
Upon restart press 4 to boot into Safe Mode, if you can get into Safe Mode, backup your data, reboot and see if your system boots normally

If all this fails go to Troubleshoot - Advanced Options - Reset this PC, choose to keep all your files and Apps
This will re-install Windows 10 whilst keeping your data and applications

Make Windows 8.1 use already-present EFI System Partition - Microsoft Community

jakub bar‰á say:

Make Windows 8.1 use already-present EFI System Partition

Dear all,

I am about to install Windows 8.1 on my machine. The thing is, I already have Arch linux installed in UEFI mode (disks are GPT-based etc). My configuration looks as follows:

- SSH drive with ESP (EFI System Partition) + Linux partition

- 2 drives for data, not important right now

- 1 500GB HDD formatted in NTFS for Windows.

What I would like to achieve is to have Windows 8.1 install on 500GB HDD, but use pre-existing ESP partition on SSD drive rather than creating a new one.

How would I do that?

jakub bar‰á say:

Hello,

thank you for swift answer. Luckily, yesterday's evening found an answer to my question. To illustrate the question I had and the answer I've found, I'll dwelve into the subject a bit.

Firstly, I was about to install Windows 8.1 on workstation with Arch Linux already installed and was hoping to learn before doing actual install, so that I wouldn't have to repair any mistakes I would have made. My configuration is as follows:

  1. 64GB SSD, GPT-formatted disk with two partitions: 512MB, FAT32 formatted EFI System Partition. This partition serves as my /boot partition, containing GRUB EFI code and Linux Kernel + iniramfs, and 60GB+ Linux ext4 partition.
  2. 500GB HDD, GPT-formatted, empty disk, that I was about to install Windows on.

Now, I've learned that Windows 8.1 usually creates two partitions on a disk: EFI System Partition for its bootloader code + some other things, and one other partition. My goal was to convince Windows to use ESP partition on drive (1) rather than create its own, so that all bootloading code, regardless of operating system resides on the same partition to: 1. gain a bit of speedup (this is SSD drive) and 2. Ease the configuration of GRUB or Windows bootloader for dual boot.

I didn't find any option to pinpoint ESP partition for Windows to use, hence I posted my question.

Nevertheless, I proceeded with installation. Windows detected all drives, figured out that there's already an ESP partition in the system, though not on the drive it was about to be installed and did not create additional ESP partition on HDD drive.

Now I have exactly the configuration I needed: Windows 8.1 installed on HDD drive, but left all bootloading code on a partition on SSD drive. I configured GRUB to chainload Windows .efi code

It turned out Windows was automatically doing the right thing without a need of additional configuration and my question was a bit premature.

Thank you again for your swift answer and have a nice day!

Any errors are indicative of a memory problem.

If a known good stick fails in a motherboard slot it is probably the slot.

Using Driver Verifier is an iffy proposition. Most times it'll crash and it'll tell you what the driver is.

*But sometimes it'll crash and won't tell you the driver.

*Other times it'll crash before you can log in to Windows. If you can't get to Safe Mode, then you'll have to resort to offline editing of the registry to disable Driver Verifier.

*I'd suggest that you first backup your data and then make sure you've got access to another computer so you can contact us if problems arise.

*Then make a System Restore point (so you can restore the system using the Vista/Win7 Startup Repair feature).

- Go to Start and type in "verifier" (without the quotes) and press Enter
- Select "Create custom settings (for code developers)" and click "Next"

- Select "Select driver names from a list" and click "Next"

- Select "Finish" on the next page.

*Reboot the system and wait for it to crash to the Blue Screen.

*Continue to use your system normally, and if you know what causes the crash, do that repeatedly. The objective here is to get the system to crash because Driver Verifier is stressing the drivers out. If it doesn't crash for you, then let it run for at least 36 hours of continuous operation.

*If you can't get into Windows because it crashes too soon, try it in Safe Mode.
*If you can't get into Safe Mode, try using System Restore from your installation DVD to set the system back to the previous restore point that you created.

¡firedog say:

I have both a personal and a work email account (both Hotmail) setup in Windows Live Mail.

... it never asked me to which account I wanted them added.

It makes no difference what mail accounts you may have set up in Windows Live Mail. The contacts list you see is determined solely by the account you sign in with. Sign in with a different account and you'll see a different set of contacts. Sign out and you'll see yet another different set.

To transfer contacts from one set to another involves exporting and importing. You can do this either from within Windows Live Mail or at http://contacts.live.com. The online export includes more data per contact than exporting from Windows Live Mail. For example, all of a contact's email addresses are included in the online export, while the local one only includes the preferred or primary address.

Not necessarily. I don't have Windows 8, so I'm not in a position to test this. There should be a folder at C:\Users\{user}\Contacts. Assuming that you have no entries in this folder (Windows Contacts), then

  1. Create a temporary folder on your desktop.
  2. In Windows Live Mail's contacts window, when signed in with your personal account, select the contacts you want to transfer, then drag them across and drop them into the temporary folder. This should create a VCF file for each contact.
  3. In Explorer, open the Windows Contacts folder and click Import. Point to the temporary folder and select all its contents. All the selected contacts should appear in the folder.
  4. In Windows Live Mail, sign in with your business account. Click Import > Address book for current Windows user.
