My as-it-stands-now (start of 2016) review of Microsoft's - Microsoft Community

gregmalone1 say:

My as-it-stands-now (start of 2016) review of Microsoft's Windows 10

I've just joined the Windows Insider program with the wish to help improve the OS by sharing my findings and observations.  I'll start my collaboration with a general review of Win10.

This is my as-it-stands-now review of Microsoft's Windows 10, which was released about 6 months ago. For background, I'm a veteran tech person, IT Director, game developer, and educator. I actively use Mac OSX, Android, iOS, and now Windows 10 as my primary work environment. I've also been a Linux systems administrator in previous incarnations. My mobile devices are currently Android (Note 4 and Note Pro), after many years of iOS.

I've decided to come back to Windows -- really, to Microsoft -- which has re-earned my respect after many years away. I am still a Mac user (grrrr, even though Apple has become a greedy, heartless, socially irresponsible evil empire -- another topic for another time). Never thought I'd see the day I would come back to Microsoft willingly! Satya Nadella is the best hope for Microsoft, since he embraces progressive thinking and is willing to make radical changes at Microsoft, quite unlike his predecessor(s). 

Anyway with that said, I've been running WIndows 10 on my Mac (via VMWare) for a handful of months, and have recently gotten a Surface Pro 4 so I can have native performance in Win 10 for a new game project. In the course of setting up, tweaking and using the SP4/Win10 for the last few weeks, I've gotten a pretty good feel for where things stand with both the OS and the SP4 device.

What have I learned?

Overall impression of Win 10 is good and positive -- it's on the right track. I like the gestalt of it all, it's very responsive and well thought out, at least by design intention. In reality, at this date at the very start of 2016, Win10 is still a bit "soft and chewy in the middle", i.e. not fully baked, but looks good on the surface and does have depth. It's not "partially baked" everywhere, and not substantially, but one definitely bites into raw dough (to continue the metaphor) every dozen bites or so.

The pros of Win10 are many, which I won't enumerate here, but I do feel that Win10 is suitable and stable enough for production use.

Among the greatest Win10 'cons' is the lack of well developed new-generation applications, i.e. applications built for Win10. Microsoft's core apps that come with Win10, e.g. Mail, Calendar, etc, are beautifully conceived, but probably about 70% complete in terms of conventionally expected features. Example, you can't search in the Calendar app. Huh?! Clearly, Microsoft is not a search-centric company, yet, despite vast improvements in its Bing domain. What's the use of data if you can't search it -- backwards, forwards, partial terms, full terms, wildcards, yadda, yadda?

Other 'cons' include the ocassional crashing app, or system and app settings mysteriously being reset, the inability to continue playing music while the device's display is off, and a variety of wrinkles that clearly state that this OS is Gen1, a work in progress. Using the Surface Pro 4, even in 'tablet mode' still feels like using a desktop computer rather than a tablet-optimized device. But it ain't all that bad, folks, and quite a pleasure to use.

Let's mention the Windows Store. The Windows Store is no Apple App Store or Google Play Store... not even close. Though on light perusal it might appear that the app store has lots of offerings, it doesn't take long to realize that the content is largely light-weight and often amateur or geeky in quality. This issue, I believe, is Microsoft's greatest challenge -- how to get good software developers to create exciting content for Windows 10... including in-house developers.

Yet, Win10, and the lovely Surface Pro 4, are up to snuff enough for serious use by professionals and casual users alike. A promising thing one discovers when using Win10 is that Microsoft provides opportunities for user feedback practically everywhere, and one gets the sense that someone behind the curtain is collating all the comments, suggestions and complaints. I wish Microsoft would learn from the bad examples of Apple and Google to be more out-front with what they're hearing from users and post some form of development road map, while still protecting their "new stuff" secrets. That alone would truly differentiate Microsoft from all the others.

In the end, I like the trajectory Microsoft has taken with Windows 10, despite the glaring and often subtle issues that one discovers almost daily. Curiously, whenever I pick up my Galaxy Note Pro tablet I am immediately struck by how much better Android is than any of the OS's out there, including WIn10. Seriously. The main problem for Android though is that it's parents, Google, have made awful strategic errors from the start with Android that will haunt the OS for years to come. Read about it.

In light of the state of the marketplace, and the predictable slow decline of Apple -- a company that has lost its core values and lacks social responsibility despite its vast wealth -- Microsoft is a serious contender to be the latest come-back company. Google, though always showing promise, and with whom I've had much professional experience spanning a couple decades, remains generally vague in their intent and less focused on meeting the public's needs, preferring to invest their formidable wealth in their moonshots and ad revenue generation rather than user experience.

For all of these companies, protection of individual user privacy and user data has yet to be fully demonstrated. I still wonder if we'll see a major 'hackopalypse' event some day which makes us all wish we'd paid more attention to these fundamental issues.

In conclusion, if I was to put my money on who will rise to the top as the leading software, OS and online services provider over the next decade, at this point in time, I'd go with Microsoft -- a statement that I'd never expected myself to ever make, but am happy that I now can.

Microsoft Access - Creating a form to pull up a specific query - Microsoft Community

kristinep say:

Microsoft Access - Creating a form to pull up a specific query

I want to create a form that will specify my search and a drop box menu with a list of 3 separate queries that will run based on the one I chose.

When I use the drop box option under form design, I could only choose to have one table or one query.

Any help will be appreciated - thanks.

ken sheridan say: You can easily list the query names as a value list as the RowSource property of a combo box, but even better would be to include the actual query names in a hidden first column and more meaningful descriptions in the visible second column.  To do this set up the combo box's properties as follows:

RowSourceType:    Value List
RowSource:           qryABC;First query description;qryDEF;Second query description;qryGHI;Third query description
ColumnCount:        2
ColumnWidths:      0

This will give you the following drop down list:

First query description
Second query description
Third query description

but the actual value of the selected row will be the query name in the hidden first column, i.e qyeryABC, qry DEF or qry GHI, so you can open the query in the combo box's AfterUpdate event procedure with:

With Me.ActiveControl
    If Not IsNull(.Value)
        Docmd.OpenQuery .Value
    End If
End With

john w. vinson say:

What kind of queries are these? Action queries (Append, Delete, Update etc.) or just a basic Select query? What real-life task are you intending to accomplish?

Action queries can be "run" (executed, changing data in the database); but it's generally not a good idea to open a query Datasheet for users to view or edit data. Much better is to create a Form based on the query and open THAT.

That said... if you want to select multiple rows, don't use a Combo ("drop") box; instead use a Multiselect Listbox. It should have two columns with a rowsource as Ken suggests; you can put some VBA code in its AfterUpdate event to loop through the listbox's ItemsSelected collection to execute all of the selected action queries.
