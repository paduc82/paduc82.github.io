You can delete the files with the long alphanumerical names in the root of C:\ .  These are left over from Windows Updates that should have been, but were not, automatically deleted.

This:

SONY-PC ( 0 bytes ) 

WindowsImageBackup  --> SONY-PC 16 bytes (16 bytes) --->  three files : 1- Backup 2017-12-25 154308 ( containing XML Document )

is a backup of your entire system ("Windows Image Backup").  This should not have been saved to your C: drive -- one of the main reasons to make a system image backup is to restore your system if (when) the hard drive fails.  If the backup is on the same physical drive that fails, it's not going to do you any good.  You should delete this AFTER you create a system image backup on an external hard drive.  See https://www.sevenforums.com/tutorials/628-backup-user-system-files-delete-backups.html
