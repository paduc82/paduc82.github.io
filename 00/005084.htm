bob jones aka: cybertaz say:

Without seeing the document it's impossible to be sure what may have been done... there are probably half a dozen possibilities, maybe more. I've see all sorts of what I'll politely call contrivances over the last 30 years :-)

My first guess is that the document is divided into Sections. By using Section Breaks the layout specs for each Section can differ from that of other Sections. By using Different First Page you can only make the content on the first page different from the remainder of the pages, you can't change the layout specs.

Another possibility is that the footer on the first page has been rigged above the Footer Bounding Box (perhaps in a Text Box) but something has been done to force the text flow in the body of the document to the next page so that it doesn't overprint the footer.

john korchok say:
Bob's third paragraph has the clue. Just pop in a text box with wrapping set to Top and Bottom. Place the top edge below any other content in the header, like a page number. The bottom edge will push down the top margin of the first page only. You can move the text box to Behind Text so you can easily access any other header formatting.

How do you lock a fillable form to avoid edits to the document - Microsoft Community

How do you lock a fillable form to avoid edits to the document

I have a fillable form and I want to lock the form so it will not be changed by others who use the form. How can a I accomplish this goal?
stefan blom say:

If you created the form with legacy form fields, you can protect the document as follows: On the Review tab, click Restrict Editing. In the task pane, select "Allow only this type of editing in the document" and specify "Filling in forms." Click Yes, Start Enforcing Protection; you will be asked to specify an optional password.

To get even more control, you may prefer setting up a userform. For more on this and other options, take a look at http://gregmaxey.mvps.org/Repeating_Data.htm.
