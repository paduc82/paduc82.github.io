jimhetland say:

I had repeated (14 times!) failures to install KB4487044 with code 0x800f0922 (though in my experience the code tells you Nothing). Many iterations of resetting Windows Update and other fixes, including a manual install and running the downloaded script from yourwindowsguide.com mentioned above, made no difference.

But that script included several SC SDSET commands not mentioned in other fixes, and those all threw errors: "The access control list (ACL) structure is invalid."  Well golly, that didn't sound good.

So I ran CHKDSK in an Admin prompt like this:

 chkdsk c: /sdcleanup /offlinescanandfix

That restarts and runs standalone (because C: is in use), and gives no report, but ... drumroll ... IT WORKED!  WU was healed!

This also worked on another machine that had the same problem - same update - though it is used less often and had failed fewer times. But other fixes hadn't worked there either.

So from now on, when WU repeatedly fails, I'll try CHKDSK. Couldn't hurt, and a lot better than a reinstall.
