sheng jiang è’‹æ™ÿ say:
This is the designed behavior of the abort function in the Visual C++ runtime, which is used to crash landing the program before more damages are done. If you see this message, the application is most likely out of the author's control (developers should not use this function directly as it does not clean up global objects) . The C++ standard details many situations the function should be called, but since you are not the author of the application, that list won't help you find out the cause. 

Cidaemon.exe does not call the abort function directly, however it hosts third-party plugins called filters to provide searchable content for windows search, and those plugins could call the function. The Windows Touch/Mouse/Search forum here may help you find the tools that list and enable/disable search filters to help identify a faulty search filter. You could also check what programs you installed recently that deal with documents (e.g. Office, Adobe reader etc) and see if uninstalling the program helps.
