suzanne s. barnhill say:

There are a couple of bits of arcana that may be helpful here. One is that endnotes don't have to be at the very end of the document, though that's where Word will insist on putting them if you choose "End of document." In order to place them somewhere else, you have choose "End of section" and then suppress them for every section except the one where you want them finally to appear. The details of this are explained at http://word.mvps.org/faqs/formatting/FootnoteFAQContent.htm#TextAfterNotes.

The other is that you can, to some extent, customize footnote/endnote separators; this may somehow help you achieve what you're attempting, though you may ultimately have to put your note headings in manually. See http://sbarnhill.mvps.org/WordFAQs/NoteSeparators.htm.

For other tips pertaining to long documents generally, see http://daiya.mvps.org/bookword.htm.

Avoid the Master Document feature like the plague; see http://word.mvps.org/FAQs/General/WhyMasterDocsCorrupt.htm (an old article, but the feature has not been made any more stable in subsequent versions).

My original suggestion had been to make the chapter heading part of the last note in the previous section. It is only with the first chapter that this is not possible.

I have some values in a single cell that look like this:

ronrosenfeld say:

It would make things a lot simpler if you separate them.  The alternative would be a VBA solution (User Defined Function), or a very complex formula.

To enter this User Defined Function (UDF), <alt-F11> opens the Visual Basic Editor.
Ensure your project is highlighted in the Project Explorer window.
Then, from the top menu, select Insert/Module and
paste the code below into the window that opens.

To use this User Defined Function (UDF), enter a formula like

=AddMany(A1:B7)

in some cell.

===============================

Function AddMany(RG As Range)
    Dim dSUM As Double
    Dim vNums As Variant
    Dim C As Range
    Dim I As Long
   
For Each C In RG
    vNums = Split(C.Text, vbLf)
    For I = 0 To UBound(vNums)
        dSUM = dSUM + --(vNums(I))
    Next I
Next C

===============================

                If mySheets(0) <> "" Then ReDim Preserve mySheets(UBound(mySheets) + 1)
                mySheets(UBound(mySheets)) = .Name
            End If
        End With
    Next

    If mySheets(0) <> "" Then
        Sheets(mySheets).Select
        On Error GoTo ExitHandler
        Application.Dialogs(xlDialogPrint).Show
    Else
        MsgBox "No Patients to Print!"
        GoTo ExitHandler
    End If

ExitHandler:
    On Error Resume Next

    Set WS = Nothing
    Erase mySheets

    With Application
        .ScreenUpdating = True
        .EnableEvents = True
        .Calculation = xlCalculationAutomatic
        .DisplayStatusBar = True
    End With
    ActiveSheet.DisplayPageBreaks = True

    Exit Sub
   
ErrorHandler:
    If Err = 18 Then    'i.e manually cancelled procedure
        If MsgBox("Do you want to stop the procedure?", vbYesNo) = vbNo Then 
             Err.Clear
             Resume
        End If
    End If
  
   'send the error to the customized central error handler
    If bCentralErrorHandler(msMODULE, sSource, CStr(Erl), , True) Then    
        Stop
        Resume
    Else
        Resume ExitHandler
    End If

End Sub
