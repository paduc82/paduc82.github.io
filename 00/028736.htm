jeeped say:
Thank you, this is a trivial solution.

If you are looking for something a tad less trivial, perhaps a VBA User Defined Function based upon either the worksheets' Index Number or Code Name.

Public Function fcnYearsVolumne(crit As Range) As Double
    On Error Resume Next
    Dim lCondSum As Double
    lCondSum = 0
    For i = 1 To 12    'e.g. from Jan to Dec
        If Not Worksheets(i) Is Nothing Then
            lCondSum = lCondSum + Application.SumIf(Worksheets(i).Range("A:A"), crit.Value, Worksheets(i).Range("B:B"))
        Else
            fcnYearsVolumne = lCondSum
            Exit Function
        End If
    Next
    fcnYearsVolumne = lCondSum
End Function

Usage: =fcnYearsVolumne(<cell for criteria>)

john w. vinson say:

VBA is an acronym for "Visual Basic for Applications", the programming language incorporated into Access.

You should be doing the data entry using a Form in your access database (not entering data directly into tables). A Form has a BeforeUpdate event; you can do your validation using code in that event. The term "presence check" is not used in most folk's jargon; it's more common to say that a field is Required. If a field must be entered, you can set its Required property to Yes in table design; however this can give the user error messages they might not understand.

It's more user friendly to use the BeforeUpdate event to check the values in the form and warn the user if they're making a mistake. Sample code might be:

Private Sub Form_BeforeUpdate(Cancel as Integer)

Dim strMsg as String ' define a variable to display as a warning

strMsg = ""

If IsNull(Me!txtThis) Then ' check to see if the user left textbox txtThis blank

  strMsg = strMsg & "Please put a value in This Textbox" & vbCrLf

  Cancel = True ' prevent the erroneous record from being added

End If

If Len(Me!txtThat) <> 10 Then

  strMsg = strMsg & "The value in That textbox should be 10 characters"

  Cancel = True

End If

If Not IsDate(Me!txtSomeDate) Then

  strMsg = strMsg & "Please enter a valid date in the SomeDate textbox"

  Cancel = True

End If

If Len(strMsg) > 0 Then

  MsgBox strMsg, vbOKOnly

End If

End Sub

Tables within a textbox - Microsoft Community

joseph_rsf say:

Tables within a textbox

I created a textbox, and I attempted to insert two small tables to this textbox. I used the tables to easily manipulate text data that I wanted to have in rows and columns. I can not do anything with the second table in the textbox. I can not move the table or even identify the text within the table.  How do I align a table within a textbox? How do I move the second table around in the text box? How do you work with tables within a textbox?
bob jones aka: cybertaz say:

Most experienced users will advise avoiding the use of Text Boxes unless absolutely necessary. If they are necessary, use them sparingly & for the purpose which they are intended... put nothing in them but text. They aren't designed to be used as containers for anything else. Not only are there limitations, but it's essentially the equivalent of putting an object inside another object, but objects of any type are foreign bodies injected into the text flow.

The primary obstacle you're running into is that content in Text Boxes can be inserted only as In Line with Text. They cannot have Text Wrapping applied to them. Accordingly, they can be positioned horizontally on the line where they are inserted using the Alignment presets, but cannot be dragged around or randomly positioned. (For the record, Word 2016 is no different in this respect.)

My suggestion is to remove the tables from the text box. I'm not certain what type of alignment you're referring to, but among other options for positioning them relative to one another you can either:

  • Use the Around Text Wrapping option in Table> Table Properties & align them using the Positioning parameters available in that same dialog window. Note that if you later drag either table you'll then have to reset its Positioning parameters.
  • Leave them as In Line (Text Wrapping set to None) then use the Indent from left property as necessary or use the same Alignment preset for both tables. Despite what is suggested in another reply, this setting is available in Word 2011 just as it is in Word 2016, but contrary to what that suggestion implies it pertains only to Horizontal alignment on a text line.

Other methods of doing what it sounds like you may be attempting would be to put the 2 tables inside a large 1-cell table (rather than a text box) or using a 2-cell table with one of your data tables nested within each of the cells. The containing table then could be placed as you wish without disrupting the alignment of the interior tables. Either method is a bit fidgety, but you may want to experiment with them. Tables within table cells are not subject to the same constraints as when placed in Text Boxes.
