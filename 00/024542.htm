>> Each email creates its own formats and categories and all of them seems like not working together.
How many email accounts do you have and what types are they - POP3, IMAP, Exchange? Exchange accounts will each have their own contacts folders and, unfortunately, you can't avoid this. POP and/or imap can share one contacts folder.

>> I want to be able to manage all my contacts from my place like "Personal folders" and be able to sync them all into iCloud or OneDrive.
OneDrive is not an option for storing contacts in a format that Outlook or your phone can easily use, but you can store contacts in in the icloud folders in Outlook. When you set up icloud addin, it will move contacts in pop or imap folders into icloud folder. It will not move Exchange contacts as they can sync directly with a phone.

Tip: Onedrive can be used to store backups but you should not open pst files that are stored there in Outlook; move them to a folder that is not syncing before opening.

It might be.  If the same password is used in multiple places, you need to find all the places where that password is hard-coded and change it in each place.

This is probably a good argument for either not hard-coding the password at all -- instead storing it someplace in the database from which the code can retrieve it -- or else coding it once in a global constant, and then referring to that constant by name in any code that wants to impose the password restriction.  Even better would be to have a single password-authentication routine that is called from every location where you want the user to have to enter a password.
