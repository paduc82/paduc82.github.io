doug robbins - mvp office apps & services (word) say:

Try:


Dim fd As Dialog
Dim strFolder As String
Dim strFile As String
Dim xlSource As Workbook
Dim xlTarget As Workbook
Set xlTarget = ActiveWorkbook
Set fd = Application.FileDialog(msoFileDialogFolderPicker)
With fd
    .Title = "Select the folder that contains the files from which you want to extract the data."
    If .Show = -1 Then
        strFolder = .SelectedItems(1) & "\"
    Else
        MsgBox "You did not select a folder."
        Exit Sub
    End If
End With
strFile = Dir$(strFolder & "*.xls*")
While strFile <> ""
    If InStr(strFile, "Monday") > 0 Then
        Set xlSource = Workbooks.Open(strFolder & strFile)
        xlSource.Sheets("Cost").Range("D1:AB143").Copy
        xlTarget.Sheets("Monday").Range("D1:AB143").Paste
        xlSource.Close savechanges:=False
    ElseIf InStr(strFile, "Tuesday") > 0 Then
        Set xlSource = Workbooks.Open(strFolder & strFile)
        xlSource.Sheets("Cost").Range("D1:AB143").Copy
        xlTarget.Sheets("Tuesday").Range("D1:AB143").Paste
        xlSource.Close savechanges:=False
    'etc for each day of the week
    End If
    strFile = Dir$()
Wend

trevor rabey say:

The schedule of the maintenance is determined by the operation.

A truck can be running for a max in a year of 365 days x 24 hours/day = 8760 hours.

Actual hours will be less because:

1) the truck will not run 24 hours/day every day

2) the truck will not be running during the maintenance. (how ling does that take? 2 days?)

Assume/estimate actual 8000 hours max per year.

Assume a service every 1000 hours, so approximately 1 service every 6 weeks, or every 42 days, and round it down to 40 days.

Make tasks called "operate truck stage 1", "operate truck stage 2" etc, duration = 40 days.

Make tasks called "maintenance 1", "maintenance 2" etc, duration =20 days.

String them together as FS0 predecessors like this:

operate 40

maintain 2

operate 40

maintain 2

...

Make a work type resource named "truck", assign it to all of the tasks at 100%.

Then figure out how to apply the cost, depending on what you need to know.

If you only want the maintenance cost and you know already what it is, you could assign it as a fixed cost to the the tasks in the cost table, or alternatively make a cost type resource and assign that, and just leave the std rate of the truck = $0.

Then track the actuals and adjust and re-schedule as required. You would expect actual truck hours to always be fewer than scheduled, so that when you input the actual operation hours (task usage view) the duration of "operate" should increase. Choose the right combination of task type and effort driven to make this happen. fixed work, I think, but check first.

This then pushes out the following "maintain" and so on and so forth down the chain.
