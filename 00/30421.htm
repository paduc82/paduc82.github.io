    allan mej say:

    Hi,

    Unidentified network sometimes happen because of an unconfigured IP address. In that case, assign a static IP address to your PC. Before doing it, kindly check the IP address your router has. To do so, kindly follow these steps:

    1. On the Search box, type CMD and press Enter.
    2. Type ipconfig /all and press Enter.
    3. From the results, note IP address, subnet mask, default gateway, and preferred and alternate DNS server addresses.
    4. Close the Command Prompt. Press Windows + R to open Run.

    Now you've taken note of the important details, try these troubleshooting steps:

    1. Press Windows + R to open Run.
    2. Type ncpa.cpl and press Enter.
    3. Right-click your Ethernet connection and go to Properties.
    4. Click Internet Protocol Version 4 (TCIP/IPv4) and select Properties.
    5. In the available fields of the box that opens next, type the same addresses in their corresponding fields as you noted from the command line, except for the IP address. The IP address should be incremented by one (for simplicity). For example, if the IP address for the wireless NIC is 192.168.0.5, assign 192.168.0.6 as the static IP address of the wired NIC.
    6. Click Ok, close all opened box, and check if your issue has been fixed.

    Let us know how things goes.

    Regards.

     Optimize How Windows 7 Runs 16-Bit ...
     Tip:Optimize How Windows 7 Runs 16-Bit and MS-DOS-Based

     Most existing 16-bit and MS-DOS-based programs were originally written for Windows 3.0 or Windows 3.1. Windows 7 runs these older programs using a virtual machine that mimics the 386-enhanced mode used by Windows 3.0 and Windows 3.1. Unlike on other recent releases of Windows, on Windows 7 each 16-bit and MS-DOS-based application runs as a thread within a single virtual machine. This means that if you run multiple 16-bit and MS-DOS-based applications, they all share a common memory space. Unfortunately, if one of these applications hangs or crashes, it usually means the others will as well.

     You can help prevent one 16-bit or MS-DOS-based application from causing others to hang or crash by running it in a separate memory space. To do this, follow these steps:

     1. Right-click the program’s shortcut icon and then click Properties. (If the program doesn’t have a shortcut, create one, and then open the shortcut’s Properties dialog box.)

     2. On the Shortcut tab, click the Advanced button. This displays the Advanced Properties dialog box.


     3. Select the Run In Separate Memory Space check box.

     4. Click OK twice to close all open dialog boxes and save the changes.

     NOTE: Running a program in a separate memory space uses additional memory. However, you’ll usually find that the program is more responsive. Another added benefit is that you are able to run multiple instances of the program—as long as all the instances are running in separate memory spaces.

     Source: https://technet.microsoft.com/en-us/magazine/ff756590.aspx "

    waseemulla shariff say: Hi,
     
    After you wake the computer from sleep, the DHCP Client service (Dhcpcsvc.dll) is responsible for informing the IP stack about the routing information. The routing information is stored in the network store interface. This problem occurs when the network store interface already has the routing information that the DHCP Client service wants to store. When the storage operation fails, all routing information in NSI is lost. Therefore, the computer loses all the routing information. This includes the default gateway.
     
    Method 1:
    Follow the steps mentioned below:
    a. Click Start, right click Computer and choose Properties.
    b. Click Device Manager.
    c. Navigate to your network adapter, right click it and choose Properties.
    d. Choose Power Management tab.
    e. Uncheck “Allow the computer to turn off this device to save power.”
    f. Restart the computer and test this issue again.
     
    Method 2: You may try updating the latest Network adapter drivers
    Please follow the link mentioned below:
    Update a driver for hardware that isn't working properly
    http://windows.microsoft.com/en-us/windows7/Update-a-driver-for-hardware-that-isnt-working-properly
