This code will select the list starting at cell A1 and filter for yesterday's date in column 19:

 

Sub filterdate()
' filterdate Macro
    Dim yesterday As String
    Range("A1").Select
    ActiveSheet.AutoFilterMode = False      'turn off prior filters
    Selection.CurrentRegion.Select
    yesterday = Format(Date - 1, "dd/mm/yyyy")
    Selection.AutoFilter Field:=19, Criteria1:="=" & yesterday
End Sub

To account for the weekend, modifying Sixthsense's code so it works:

 

Sub filterdate()
' filterdate Macro
    Dim yesterday As String
    Dim tradedate As Date
    tradedate = Date - 1
    Range("A1").Select
    ActiveSheet.AutoFilterMode = False      'turn off prior filters
    Selection.CurrentRegion.Select
    Select Case Format(tradedate, "ddd")
    Case Is = "Sun": tradedate = tradedate - 2
    Case Is = "Sat": tradedate = tradedate - 1
    End Select
    Selection.AutoFilter Field:=19, Criteria1:="=" & Format(tradedate, "dd/mm/yyyy")
End Sub

      oem = ucase(mid(wshshell.RegRead("HKLM\" & sBit & "\Microsoft\Office\" & sVer & "\Registration\" & SubKey & "\ProductID"), 7, 3))
        edition = wshshell.RegRead("HKLM\" & sBit & "\Microsoft\Office\" & sVer & "\Registration\" & SubKey & "\ProductName")

          edition = other
                   err.clear

           Set wshshell = Nothing
            if oem <> "OEM" then oem = "Retail"
           if lenb(final) > 1 then

Function GetWinKey()
    Set wshshell = CreateObject("WScript.Shell")
    edition = wshshell.RegRead("HKLM\SOFTWARE\Microsoft\Windows NT\CurrentVersion\ProductName")
    oem = ucase(mid(wshshell.RegRead("HKLM\SOFTWARE\Microsoft\Windows NT\CurrentVersion\ProductID"), 7, 3))
    key = GetKey("HKLM\SOFTWARE\Microsoft\Windows NT\CurrentVersion\DigitalProductId")
    set wshshell = Nothing
    if oem <> "OEM" then oem = "Retail"
    GetWinKey = edition & " " & oem & ":  " & key
End Function

Function GetKey(sReg)
    Set wshshell = CreateObject("WScript.Shell")
    GetKey = ConvertToKey(wshshell.RegRead(sReg))
    Set wshshell = Nothing
End Function

Function ConvertToKey(key)
    Const KeyOffset = 52
    i = 28
    Chars = "BCDFGHJKMPQRTVWXY2346789"
    Do
        Cur = 0
        x = 14
        Do
            Cur = Cur * 256
            Cur = key(x + KeyOffset) + Cur
            key(x + KeyOffset) = (Cur \ 24) And 255
            Cur = Cur Mod 24
            x = x - 1
        Loop While x >= 0
        i = i - 1
        KeyOutput = Mid(Chars, Cur + 1, 1) & KeyOutput
        If (((29 - i) Mod 6) = 0) And (i <> -1) Then
            i = i - 1
            KeyOutput = "-" & KeyOutput
        End If
    Loop While i >= 0
    ConvertToKey = KeyOutput
End Function

suzanne s. barnhill say:

In recent versions of Word, even simple numbering is really multilevel numbering under the hood, so you might as well set it up that way. Go to Home | Paragraph | Multilevel List and select Define New Multilevel List. The dialog that opens allows you to specify numbering for up to nine levels, but you (probably) need only two.

Presumably you have a particular style that starts a new chapter (a heading style used for the chapter number or chapter title). So you will make that your top-level style. Click More>> to expand the dialog and select your chapter opening style in the "Link level to style" dialog. Unless you want it to be numbered, set the "Number style for this level" to "(none)."

Now select Level 2, choose the numbering style and indents you want for the numbered questions, and link that level to the style you're using for the questions. "Restart list after" should already be enabled and set to Level 1, but make sure that it is.

If you need to access this dialog again for fine tuning (keeping in mind that numbering and left-side indents of a numbered style cannot be changed in the Modify Style dialog), you must (unintuitively) again choose Define New Multilevel List, which does double duty for "Modify multilevel list."

Once you have set this up properly, any list that follows an instance of the Level 1 style will restart numbering automatically.
