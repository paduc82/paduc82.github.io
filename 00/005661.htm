More than 64 Nested If Statements - Microsoft Community

devonroche say:

More than 64 Nested If Statements

I wrote a formula with 80 nested if statements before realizing that excel has a limit of 64 nested if statements.

There are 80 combinations of criteria, being the Grade and Size of the original and the Grade and Size of the proposed. I would like to enter a quantity in one cell and have it be multiplied by the corresponding factor in the table , based on the criteria I enter, to be shown in a different cell.

Does anyone have any suggestion on how to do this? I am relatively new to excel and have read that I should look into using vlookup. Again, I wouldn't even know how to use it if it did work.

myall_blues say:

I tend to work to the rule that if there are more than 5 levels of nesting required, there's a better way to do it.

There are more fancy ways to achieve this, but on the basis that each group of four original and conversion grade/size values is unique, you could do it with a simple SUMIFS formula.

See picture below.

Columns A:E represent your conversion table as shown in the picture. Arrange them in one table in rows down the page.

Columns G:K (G3:K3) represent your inputs, so you provide the grade/conversion pairs and a quantity.

The formula in L3 then gets the unique 'Factor' and multiplies by the quantity in K3.

Formula in L3 is =SUMIFS($E$3:$E$6,$A$3:$A$6,G3,$B$3:$B$6,H3,$C$3:$C$6,I3,$D$3:$D$6,J3)*K3

If I've understood your requirements correctly VLOOKUP won't help you in this case, as it cannot do multiple criteria.

To update your mobile number on your Hotmail account, see the steps below:

  1. Log in to Hotmail.com/Outlook.com.

  3. Once done, go to Account details and select Security and Privacy.

Using range names to refer to cells in a range in a formula - Microsoft Community

trippap say:

Using range names to refer to cells in a range in a formula

Let's say column B contains billing rates, and column C contains billable hours, and I want to calculate the amount to be billed in column D. Let's assume the first row containing such billing data is row 3 and that the data is in a normal range, rather than being an Excel table. Also assume that I've defined range names billRate and billHours for the data in columns B & C, respectively. Then my formula for cell D3 could be =B3 * C3 or =billRate * billHours. I know why the first form works, but I can find nothing that documents why the second form, using range names, also works. If this data were in an Excel table, then the second form is almost a structured reference form, but it's correct syntax would be [@billRate] * [@billHours]. 

hansv mvp say:

This is not specific to named ranges.

Let's say the rates are in B3:B8 and the hours in C3:C8.

Select D3:D8.

Enter the formula  =$B$3:$B$8*$C$3:$C$8

Press Ctrl+Enter to enter the formula in all selected cells: each of the cells D3, D4, ..., D8 will contain the same formula  =$B$3:$B$8*$C$3:$C$8

Yet D3 will display the result of B3*C3, D4 will display the result of B4*D4 etc.

When you multiply (or add, subtract, ...) two ranges, Excel will multiply (or add, subtract...) the corresponding cells from each range.

Now, if you give $B$3:$B$8 the name billRate and give $C$3:$C$8 the name billHours, the above formula  =$B$3:$B$8*$C$3:$C$8  is equivalent to  =billRate*billHours
