Are you talking about the page margins set in Word (default 1" all around), or the physical printer margins you described above (approx .1" all around).

That is as simple a spreadsheet as you could ask for.

I like using the table feature, and in this case it will allow the use of a "slicer"

Here is what it looks like:

Here is how to create it (in less than 5 minutes!)

  • Enter the column heading names, ie : Item, Asking Price, Sold price
  • enter a dummy value under item heading, ie A
  • use the <CTL><T> shortcut to define the table, turn on the 'Headings" option in the table dialog
  • When you say OK in the create table dialog you'll see the table that arrow 2 points to
  • With the cursor inside the table, the Contextual Table Design tab is displayed, click on it
  • In the Table Style Options group:
    • turn on Total Row
    • turn off Filter button

  • Select the whole Asking and Sold columns
  • turn on Currency formatting
  • If you enter a sold value now, you will see $1.00 in the total column. That is wrong!  Click inside the total cell, beside it to the right a drop down arrow will display, click on it.
  • You will see Count selected, change that to SUM
  • In the Contextual Table Design tab > Tools group > click on Insert slicer button
  • select only 'Sold' column
  • A new slicer is inserted, Drag it to the desired position, resize it.

Pretty much done.  It took MUCH longer to describe than to do.

When you enter a value in the sold column, it will show up in the slicer. If there are blank cells, you will see a "(Blank)" entry also. The example has 2 rows, one with a value the other blank. If you click on the Blank entry, it will only show rows that have no sold value.
