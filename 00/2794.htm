ken sheridan say: They won't be 'flagged', by which I assume you mean selected in the check box to the left of the  table name.  You check that when you want to refresh the links to a table.

In the Linked Table Manager, to the right of each table name will be the path to the file containing the tables.  The tables you see in the navigation pane in the current database are merely links to those tables.

What I think you have done is back-up your front end database file, i.e. the one you open when you use the database, not the back end file, i.e. the one containing the tables.  Consequently both your current front end and its back-up are linked to the same back-end tables.  When you amend data in the current front end therefore, you are changing data in the back-end.  Consequently the back-up of the front end will also show the amended data, as it is linked to the same tables.

In addition to backing up the front end, as you have done, you also need to back-up the back end file containing the tables.  So that you don't overwrite a back-up containing old data with new data, you can create a new dated back-up each time you back-up the back end file.  You can do this manually, or you can build the functionality to do so into your front end file.  You'll find an example of how to do this in Backup.zip in my public databases folder at:

https://onedrive.live.com/?cid=44CC60D7FEA42912&id=44CC60D7FEA42912!169

Note that if you are using an earlier version of Access you might find that the colour of some form objects such as buttons shows incorrectly and you will need to  amend the form design accordingly.  

If you have difficulty opening the link copy its text (NB, not the link location) and paste it into your browser's address bar.

This little demo file includes methods for making a single back-up, overwriting the current back-up each time, or for making dated back-ups.  To do this in your database, you would need to import all of the objects from my demo, apart from the forms, with the exception of frmBackUp, which you'll need, and apart from the linked sample tables (but including the two local tables BackEndLocation and FileLocations, which you also need) into your front end file.  When you want to make a back-up of the data, which you should do very regularly, you'd open the frmBackUp form, browse to the location of your back end file, and to that of the file to which you wish to back-up the data.  The first time you do this you'll need to enter the new file name in the dialogue.  When you make a dated back up it will then use this file name and add the date in the format YYYYMMDD.

jay freedman say:
OK, that's a different situation than the article assumes. You'll need to make some changes...

First, if the button (or you said several buttons) aren't for causing the document to print, then you don't want to use the Click procedure of the button (e.g., Sub CommandButton1_Click). Instead, you need to create a macro (in a regular module in the document, not in the ThisDocument module) with the special name FilePrint. A macro with that name will run instead of the built-in Print command. Inside that macro, you can make all of the command buttons invisible, call the .PrintOut method to do the printing, and then make all of the buttons visible again.

Second, the macro in the article assumes that there are no other text boxes or drawn shapes in the document (or if there are, then the text box containing the command button is the first shape). My guess is that this isn't true in your document, so the code made some other shape invisible --  whatever is .Shapes(1).

So...

In the document, put each command button into its own text box. You'll have to find out the index number of the text box (shape) for each one. There's no easy way to do this, but here's a way: In the document, click the Â¶ button so you can see the paragraph marks. In the VBA editor, press Ctrl+G to open the Immediate window. Paste in this line and press Enter:

ActiveDocument.Shapes(1).TextFrame.TextRange.Font.Color=wdColorRed

Look at the document to see whether the paragraph mark in any of the text boxes has turned red. If so, you know that's .Shapes(1). Change it back to black by changing wdColorRed to wdColorAutomatic and pressing Enter. Then change the 1 to 2, change to wdColor Red again, and repeat. Keep going until you've found the index numbers of all the text boxes.

In the VBA editor, delete the Sub CommandButton1_Click procedure. Click Insert and click Module. Then add code like this:

Sub FilePrint()
    With ActiveDocument
        .Shapes(1).Visible = msoFalse
        .Shapes(2).Visible = msoFalse
'       continue this list until all the text box indexes are included

        .PrintOut Background:=False
       
        .Shapes(1).Visible = msoTrue
        .Shapes(2).Visible = msoTrue
'       continue this list until all the text box indexes are included
    End With
End Sub

The component that will enable the switching between Tablet and Laptop mode (Continuum) will have options.

When you Attach the screen you will see a popup asking Exit Tablet Mode? with the same options.

In both cases, you can select the Don't ask me again option and your settings will not change when you detach or attach the screen.

This will not require a logoff / logon.
