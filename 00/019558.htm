gerry knight say:

I would hope no one will explain how to hack this.  It is password protected for the reason everyone does this, to try and block people from seeing it.

If you ask how to clear controls on a userform, this is a basic task, and as Mike mentions you can get help with this.  For example:

Sub ClearEmOutDano_Click()

Dim ctl As Control
For Each ctl In Me.Controls
   If TypeOf ctl Is MSForms.TextBox Then
      Me.Controls(ctl.Name).Text = ""
   End If
   If TypeOf ctl Is MSForms.CheckBox Then
      Me.Controls(ctl.Name).Value = False
   End If
Next
End Sub

Tests each control, and if it is a TextBox, "clears" it - makes it ""; and if it is a CheckBox it sets its Value to be False (unchecked).  There are other ways to go about this.

The reason for Quick scan is in order to be more assure. For example someone might turn off MSE or you are on travel and you haven't done update and new virus is come into your PC while you don't get update. Normaly user won't get anything during quick scan if they make sure MSE is on all the time but if it get something you should think what was wrong.

jimr1 say:

A full scan is recommended when first installing MSE or when there has been a malware detection. A quick scan will scan only the files and folders on a computer where malware is most likely to reside. A quick scan is sufficient in most circumstances. Any files not scanned in a quick scan or a full scan will be scanned on access ( real time protection ).

Don't believe everything you read on wikipedia.

 

In any case, Nokia's design might very well use a lower clockspeed than what the processor architecture allows.

 

It doesn't necessarily mean that it is "downclocked", or that the hardware allows it to be "overclocked" (Nokia's chips aren't always - or they rarely are, or have been - off-the-shelf chips or chipsets, but usually custom components made to their specifications).

 

In any case, with phone design, the hardware selection and tuning it is always a compromise between battery consumption and speed. Running a device at a higher clock speed than required means that you could be wasting battery power, thus reducing talk time/standby time, too.
