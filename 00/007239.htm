john - project say:

NashSh,

What you need to do (or should have done) is to set a baseline, prior to execution of the plan.

You are right in that Project automatically adjusts the estimated fields (Start and Finish) to agree with reality when an entry is made in the Actual Start and Actual Finish fields. However, if a baseline is set prior to plan execution, several fields (including Start and Finish) are saved to to baseline fields (i.e. Baseline Start, Baseline Finish, etc.) such that those fields can be used for comparison to the current plan. a. user say:

Hello,

If a message gets stuck in your Outbox, it could be caused by a large attachment within the email, or that there is a bad connection to your mail server. As a first troubleshooting step, we suggest that you clear your stuck emails in your Outbox by following the steps below:

  1. In Outlook, select the Send/Receive tab, and then select Work Offline.
    • Note: This stops Outlook from trying to send all email messages.
  2. Select the Outbox. From there, you may do one of the following:
    • Move the message: Move the message to the Drafts folder. You can either drag the message to the Drafts folder or you can right-click the message, select Move > Other Folder > Drafts folder from the list.
      • Note: This enables you to open the large email message, and remove, resize, and re-add the attachments before you attempt to resend the email message again.
    • Delete the message: Right-click the message and select Delete.

    IMPORTANT: If an error tells you that Outlook is trying to transmit the message, close Outlook and wait for it to exit. You may have to use Task Manager in Windows to make sure that the Outlook.exe process is completed. Restart Outlook again and repeat the previous steps to either delete or move the message. Press Ctrl + Alt + Delete and click Task Manager to exit the Outlook.exe process.

  3. Select the Send/Receive tab, and then select Work Offline to deselect it. This would restart your connection.

After clearing your Outbox, attempt to send an email and see if it is stuck in Outbox. If it is still stuck in Outbox, it could be that your mail server is offline. Kindly check the server status of your mail server by looking at the lower-right hand corner of the Outlook window, it will tell you if you're connected or not. If you're not connected to the server, kindly check your Internet connectivity by opening Internet Explorer and surf the Internet. If you are able to surf the Internet, you know your Internet connection is good and the mail server might be down.

If you need further assistance, feel free to ask us.

Export Online Archive of Shared Mailbox in O365

Hello everyone,

I'm not sure how often this may apply to the rest of you, but we have recently had a need to export the Online Archive of a Shared Mailbox to a .pst file.  The problems we faced were two-fold.  The first being the obvious that OWA doesn't provide the availability to export at all.  The second is that if the shared mailbox is loaded into the Outlook of a user with FullAccess permissions, the Online Archive does not show up.

In order to export the Online Archive, you'll need to follow these steps:

  • Close Outlook if open
  • Open Control Panel and selected the Mail icon
  • Click Show Profiles
  • Click Add new profile and provide a name
  • If necessary, click "Connect to different account" in the O365 pop-up window
  • Select the Manual Configuration radio button and click Next
  • Type in the email address of the Shared mailbox and click next
  • When prompted, type in the email address and password of a user that has Full Access permissions to the shared mailbox
  • Open Outlook using the newly created Profile

You should now be able to see the shared mailbox and its online archive, and either one can be exported to a .pst file.
