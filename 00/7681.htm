Like all my files in this SkyDrive folder it is not intended to be a template but an illustration of possible means of achieving common tasks.  It is not set up quite in the way you describe, but you might find that it includes a basis for you to create a suitable application.  

You'd split the application into separate front and back ends of course, so you might use the interface as it stands for creating the questionnaire, and then create a separate front end for use by users, basing this on the form/subforms used in my demo for answering a questionnaire. In your case you would not need to include a combo box to select a questionnaire of course, as you'll have just the one, and the combo box for entering/selecting a respondent would either be supplemented by another control for entering a password, or more simply the current user's windows log-in name can be assigned as the respondent by inserting the following module in the database:

' module basGetUser
Option Compare Database
Option Explicit

Declare Function GetUserName Lib "advapi32.dll" Alias "GetUserNameA" (ByVal _
   lpBuffer As String, nSize As Long) As Long

Public Function GetUser() As String

   Dim strBuffer As String
   Dim lngSize As Long, lngRetVal As Long
   
   lngSize = 199
   strBuffer = String$(200, 0)
   
   lngRetVal = GetUserName(strBuffer, lngSize)
   
   GetUser = Left$(strBuffer, lngSize - 1)

End Function

You can then call the GetUser function to return the current user name.  Don't be tempted to use the Environ function for this as the value of an environmental variable can be very easily changed by a reasonably savvy user to masquerade as another user.
