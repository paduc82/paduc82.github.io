Partial answer to problems with transferring large files to USB - Microsoft Community

pkstr11 say:

Partial answer to problems with transferring large files to USB thumbdrives in Windows 7

Anyone doing a bit of research on this problem realizes it goes back almost 3 years now, with no one being able to present a suitable work-around, or even figure out what is causing the problem.  For those of you who don't immediately know, the problem is when attaching USB storage devices and transferring large files, generally about 150MB or larger, the transfer times out, freezes up, occasionally causes Windows 7 itself to freeze up, and eventually you receive a message that the transfer cannot be completed because the device has become disconnected.  The USB drive itself cannot then be read until removing and re-attaching the device, at which time if the process is repeated, the same results ensue.

Microsoft posted KB-976972 as a workaround involving NVIDIA EHCI chipsets.  Again, though, the problem is much more widespread than NVIDIA EHCI systems, and in fact KB-976972 doesn't necessarily solve the problems in systems with NVIDIA chipsets.  After some experimenting and tweaking on multiple systems, the problem appears to be with the Network Store Interface on some systems.  Why specifically this service, or what ultimately the conflict is, I have no idea as of yet.  However, in order to overcome this issue, the following steps can be taken in Windows 7 x64 and x86 systems:

1.  Open start menu and type "msconfig" into the search box
2.  Open msconfig and go to the "Services" tab
3.  Find the entry "Network Store Interface Service", and uncheck the box next to this entry.  Note:  This is a Windows service.
4.  Hit "Ok" to close out msconfig.  You will be prompted to restart.
5.  Upon restart, you will be able to transfer files back and forth from your external USB drives and devices without any issues.  However, with the Network Store Interface service deactivated, you will not have network or internet access.
6.  When you are done transferring files, follow the above steps again to re-activate the Network Store Interface in msconfig.

Safe Mode without Networking will likewise allow you to get to a state where you can transfer files back and forth without issue, however under Safe Mode many advanced file-transfer features are disabled, and it will take significantly longer to transfer files.  Simply deactivating the Network Store Interface allows you to retain nearly all of your computer's functionality and still transfer files to USB drives.

Now for Microsoft to get on it and finally figure out a patch for this problem after nearly 3 years.
