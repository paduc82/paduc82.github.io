PIN prompt violates security best practices - Microsoft Community

christopher lowde say:

PIN prompt violates security best practices

The new PIN prompt does not require you to enter the PIN, it responds automatically when you have entered the correct number of digits.  This makes it a convenience process, and not a security process.

I can enter a string of 0's and determine the length of the users PIN, now I have knowledge of the user I may be able to better guess the actual PIN they are using.

If you are going to automatically accept the PIN then you should only accept it if it is correct, since you are performing an automatic acceptance you do have knowledge of it length.  Hopefully stored in a secure place and hashed in some way.

If the PIN is not correct then you should report the error after a random number of additional digits have been entered, where the random number is a system parameter that can be set by policy.

When dealing with real PIN based security devices, e.g. smart cards, we can define PIN policies; typically:

  • Min length
  • Max length
  • Max repeated characters - e.g. 1111
  • Max sequential characters - e.g. 1234 or 1212

I can see nowhere to define these policies, they are not in the logical place Computer Configuration\Security Settings\Account Policies\Password Policies or under their own logical hive Computer Configuration\Security Settings\Account Policies\PIN Policies
