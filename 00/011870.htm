aditya__sharma say:
Hello

Check your password

The most common cause of an out-of-date account is an incorrect password. Select Fix account in the notification bar at the top of the Mail or Calendar app. If your password is incorrect, you'll see a message asking you to update your password. Enter the new password and choose Done.


Check your security certificate

If your password isn't the problem, you may have a certificate error. If so, you'll see a message that says, "There is a problem with the server's security certificate. The security certificate is not from a trusted certifying authority."

This error often happens when you don't use SSL to secure your email account. To fix this error, use the following steps.

1. Choose Settings > Manage Accounts.

2. Choose the account that's out-of-date to display the account settings dialog.

3. Choose Change mailbox sync settings > Advanced mailbox settings. You may have to scroll down in the dialog to see Advanced mailbox settings.

4. Check the boxes for Require SSL for incoming email and Require SSL for outgoing email and then choose Done > Save.

ken sheridan say: A LEFT OUTER JOIN is pointless in this case as you cannot delete a row which does not exist in a referencing table.  You can either use an INNER JOIN and specify the table, or, as I would do in this context, restrict the query by means of the EXISTS predicate against a correlated subquery:

DELETE *
FROM tblFragranceInventory
WHERE EXISTS
    (SELECT *
     FROM tblFragrance
    WHERE tblFragrance.FragID = tblFragranceInventory.FragID
    AND InventoryComments LIKE "*Do Not Destroy*");

Note also that testing for IS NOT NULL is also superfluous as the LIKE operation already presupposes a value in the column.

scottgem say:

First, the RowSource of a combo or list box is a SQL statement or the name of a table or stored query. So your Rowsource is actually something like:

SELECT InvoiceNum FROM tblInvoices WHERE CustomerID = [Forms]![Payments]![cmbFilterCust];

If you want to use a Toggle button to turn filtering on or off I would use code behind the toggle like so:

If Me.TglFilterInv = True Then

     Me.lstInvoices.RowSources = "SELECT InvoiceNum FROM tblInvoices WHERE CustomerID = [Forms]![Payments]![cmbFilterCust];"

     Me.TglFilterInv.Caption = "All Invoices"

Else

     Me.lstInvoices.RowSources = "SELECT InvoiceNum FROM tblInvoices;"

     Me.TglFilterInv.Caption = "Customer Invoices"

End If

This will filter the listbox and display a different caption on the toggle button.

Again, it sounds like with your setup, the best approach will be a "default value formula". This is not hard to do.

 

An example form is simple to create:

  1. Create a new blank form.
  2. Insert a repeating table.
  3. Change the first box in the repeating table to a drop-down list box.
  4. Add "Choice1", "Choice2", and "Choice3" to your list box.
  5. Add a text box to the view.
  6. In the Text Box Properties, click the Formula button for the Default Value.
  7. Insert the count() function (Under "Field").
  8. Double-click to insert field--select your repeating drop-down field (don't click OK yet).
  9. Click Filter Data.
  10. Add filter where the selected field = "Choice2".
  11. Click OK five times to close all dialog boxes.

Now preview the form, and add some rows to the repeating table. Make selections. For each "Choice2" you select, the count will increment. For each one you change or delete, the count will decrement.

You should be able to apply this technique to your solution with success.
