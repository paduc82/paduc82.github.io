marshall barton say:

This does not appear to be a validation problem at all.  It sounds like you may be so new to databases that you have yet to learn some of the vocabulary.  I'll try to explain why your question is not making a lot of sense.

A field is column in a table or query.  Regardless of what Access allows you to do in a table, its fields should only display its value.  You will drive yourself nuts if you use a field's Lookup to display something else.

The things on a form that are used to display a field's formatted value are calledcontrols. There are many types of  controls (text box, combo box, check box, command buttons, subform, etc) that can make entering data easier and/or format the data to make it easier to read.

Validation is something that only permits acceptable, valid data to be entered and produces an error if invalid data is entered.  There are Validation settings for both fields in a table and for controls on a form.  Setting a Validation Rule in a table field, will block saving a record with a field that has an unacceptable value.  A validation rule for a control will block the entry of an invalid value into the control.

From your description of what you tried, it kind of sounds like you are using two combo box controls and you want the selected item on the first combo box to limit the list of selectable items in the second combo box.  You said that the first combo box's RowSource is set to a field, but that can not be.  The RowSource can only be set to a list of values (very rarely useful) or the name of a table or query.  A control'sControlSource property can be left blank, set to an expression or set to the name of a field in the form'sRecordSource table or query.

Please try to rephrase your question using more precise terms and proper nouns (the real names of your tables, queries, fields and controls) and avoid using pronouns (eg. it) and descriptive terms (eg. first field, my query, underlying table, etc).

scottgem say:

Marshall gave you a very good explanation of terms that should help you pose your question better so we can help. A couple of other points to add. The RowSource of a combobox or listbox control can also be a query (and, in fact, usually is). That query can be filtered by the selection in another control on your form. This is called cascading or synchronized combos.

Another point, as John alluded to, is we generally do not store calculated values. If you have a value that is derived from a formula involving other fields, then there is no reason to store the results of the calculation since you can use the formual to display it at any time.
