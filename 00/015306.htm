Hi Off2w0rk,

 

According to my research, once the Office 365 client is authenticated with MFA, you will receive a access token and a refresh token to be able access Office 365 services . The access token is only valid for an hour and then the refresh token is used to obtain a new access token if the initial authentication is still valid.

 

The Refresh token is valid for 14 days by default but if you are continuously using your mailbox during this period it can last up to 90 days.

 

So it could be you are not asked for Multi-factor authentication again for up to 90 days in Outlook.

 

Things that could force you to re-authenticate:

 

1.       If you sign in and out again in Office clients

2.       Don't login for 14 days on that device

3.       Change your password

4.       Administrators can apply conditional policies to restrict the resource the user is trying to access

5.       Swap between Office 365 accounts

    However, I don't know how to automate this in Excel.

    jllatham say:

    You need to change the values on the RIGHT side of the equal symbol in these Const declarations to match up with your worksheets

     'these are for Worksheet 1
    Const ws1Name = "Sheet1"
    Const firstRowToCopy = 1 ' will now copy from row 1 down to end of data
    Const alwaysUsedCol = "A"
    Const firstColToCopy = "A"
    Const lastColToCopy = "E"
    'assumes layout of Worksheet 2 is
    'the same as Worksheet 1 and
    'data is copied to the same columns
    Const ws2Name = "Sheet2"

     

    For example, if your Worksheet 1 shows "Source Data" on its name tab, then you would make this change:

    Const ws1Name = "Source Data"

    make a similar change, if needed, to the Constant named ws2Name.

    The code needs to know the identifier for a column on Worksheet 1 that always has some data in it on that sheet.  That is the column that's used to figure out how far down the worksheet your new data goes, and thus what rows to copy.

    That is set up in this statement:

    Const alwaysUsedCol = "A"

    Finally, the code needs to know which columns from Worksheet 1 to copy and that is set up in these two statements:

    Const firstColToCopy = "A"
    Const lastColToCopy = "E"

    Let's say Worksheet 1 has data from row 1 down to row 22 after you run the macro to remove the header rows, then (with things set up as they are now) the copy range would end up being A1:E22

    I figure your entries probably start in column A since most people set things up that way, but by defining it specifically, your data could actually start in any column and by telling it which column with firstColToCopy it can be adapted to any setup.

    But unless I just plain got lucky, the last column that could possibly hold data is probably not column E, so you need to change

    Const lastColToCopy = "E"
    to indicate the last column on Worksheet 1 that could ever have data in it, even if it doesn't always have any.  Let's say your entries go all the way out to column AC, then just edit that line of code to become:

    Const lastColToCopy = "AC"
