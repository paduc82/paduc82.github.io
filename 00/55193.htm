easenger say:

Update:

After much Internet searching, found that this problem was not new to Windows 10.  Many reports are out there since Windows 7 where a NIC is found in Device Manager, but the Network Connections is blank.  Most were resolved by going to HKLM\System\CurrentControlSet\Control\Network and deleting the Config binary. 

However, in Windows 10, that isn't a value in the registry.  I put in a call to Microsoft, and after two days and escalation they told me that my only option was a reinstall.

Given that I didn't have a network connection, I tried the ISO.  The ISO I downloaded last week kept failing at 20% (I'm going to try to re-download for my other machines to see if that helps).  So I ended up having to use the OneKey Recovery tool to put it back to Windows 8, run all the updates, and then upgraded to Windows 10, which worked.

NOTE:  I un-installed the McAfee, Nitro Pro, Automatic Reminder, Office Trial, and any other bloatware before the update.  The McAfee uninstall is critical and even mentioned on the Lenovo website.

I suspect my problem was caused by the Sonicwall VPN software I had installed initially.  If you have any VPN software installed, I'd recommend you uninstall it before the upgrade.

The only other problems I had are:

1.  Periodically, the keyboard stops working.  A reboot so far has fixed this.

2.  The touch keyboard comes up and then disappears within seconds, making it practically useless.  This was fixed by download the Intel VGA driver for Windows 10 from the Lenovo website and installing it.

Since the keyboard stopped and the touch keyboard wasn't functioning during the initial update where you absolutely have to put in your password, the on-screen keyboard through the ease-of-access tools was a life-saver.  Try that if you run into the same problem.

Although both use the 2.4GHz band they are in different segments and are different protocols, Bluetooth has much simpler authentication processes and is a much shorter range. The Bluetooth IEEE standard is 802.15.1, Wi-Fi is a subset of 802.11 which is the Ethernet protocol. Despite all this jargon note the two standards and read this specification

The chip only supports 802.11 therefore it is NOT Bluetooth

Windows 10 handles keys differently.  When you upgrade to Windows 10 via Windows Update from Windows 8.1 registers your computers hardware, and qualifying OS with a Microsoft Product Activation Server. The device is then considered a Windows 10 device.

Subsequently anytime you re-install (or clean install) the installation checks with those servers, finds the unique installation ID and produces the validation

Since retail can be moved to new HW during the install it will check with the activation servers and generate the new GUID.  I suspect they will verify via your email address (MSA)
