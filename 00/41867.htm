    Windows 98 to Windows 7 - Microsoft Community

    krawlsky say:

    Windows 98 to Windows 7

    I have an old computer - windows 98 with Microsoft Works Suite 2000. I am trying to transfer my saved word documents to my new computer - windows 7 with microsoft works. I am sure they are not compatible but what can I do so I can don't have to re-enter all these word documents?
    ken blake say: On Tue, 15-Dec-09 04:22:11 GMT, krawlsky wrote:

    > I have an old computer - windows 98 with Microsoft Works
    > Suite 2000. I am trying to transfer my saved word documents
    > to my new computer - windows 7 with microsoft works. I am
    > sure they are not compatible


    Correct. Your Word .doc files are not compatible with Microsoft Works
    and can not be read by it.


    > but what can I do so I can don't have to re-enter all these
    > word documents?


    You need to install Microsoft Word, or some compatible program that
    can read them.

    If all you want is to view them, not edit them, you can simply
    ad and install the free Microsoft Word Viewer at
    http://www.microsoft.com/downloads/details.aspx?FamilyID=3657ce88-7cfa-457a-9aec-f4f827f20cac&displaylang=en
    or http://tinyurl.com/2w3s6z

    If you want to be able to edit them, you can buy and install Microsoft
    Office (which includes Word) or just Microsoft Word, or download and
    install the free OpenOffice, or buy and install a program like
    WordPerfect which can open Microsoft Word .doc files.

    One more alternative: Works can read .rtf files, and your old
    Microsoft Word can write .rtf files. If it's not too much work for
    you, on your old computer you can open all the .doc files in Word,
    then save them as .rtf files before moving them to the new computer.

    And one last point: if you are accustomed to using Microsoft Word, you
    will probably find Works to be greatly inadequate for your use. So you
    may want to get a copy of Microsoft Office, OpenOffice, or WordPerfect
    to get the same, or similar, capability back.

      the_wren_nest say:

      There is a workaround for this. I just purchased a NAS with the intent to store my music on it and play it through the xbox music app. But had the same issue.

      However, if your NAS supports iSCSI you can create a LUN and target it with the iSCSI configurator tool in windows 8.1. This allows you to format that LUN and have it appear as a locally installed hard drive with a new drive letter. In other words, windows and the app "see" it as a local drive and not a network location. Doing this I was able to have the music app at least add the newly created drive as a location to search for the music. I did notice that you had to create a folder within this new drive and be sure to have tat folder be indexed (right-click>properties> advanced).

      I believe this is partly why iSCSI was created because some applications in enterprise have trouble using data stored on networked storage spaces.

      See your NAS manual about accessing your NAS through iSCSI. It wasn't too difficult to set-up.
