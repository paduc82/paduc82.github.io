karl timmermans [outlook mvp 2012-2018] say:

Not entirely certain on what the ultimate objective is on first read of your post so at this point, this response is primarily to clarify a few things

#1 - There is absolutely no requirement to have field names in your CSV file to match those of the Outlook standard field names. In fact, relying on Outlook's "auto-mapping" is a first step in creating unnecessary problems - too easy to make a mistake and Outlook field name.

 

#2 - <Suggested Contacts> will contain the email addresses of emails that have been sent whether in the Outlook AddressBook or entered manually by typing in a value in the TO/Cc/Bcc boxes. The purpose of <Suggested Contacts> is for "auto-completion" of email addresses manually typed in the To/Cc/Bcc fields. The purpose of items contained in the SG folder is NOT to act as a contact folder

 

#3 - New contacts are saved in a given based on the following rules:

a) if done via the "Add to contacts" to save a contact item from an email, the new contact will be saved to the default contact folder for the Outlook profile which will be located in the PST file designated as the "default" for the profile. For simplicity, the default contains the <Inbox> where your emails get saved.

b) if a new contact is created without a contact folder first being selected, the new item will be saved in the default contact folder as described above. If a specific contact folder is selected first, the new contact will be saved in that folder

 

#4 - As mentioned already, would personally never use "Suggested Contacts" for the purposes of storing your primary contacts. Whether one folder should be used for all contacts or contacts should be stored in different folders depends on a number of factors which is outside this response. However, if you opt to combine all contacts in a single folder would strongly suggest that you add a category value to each contact so that contacts can be properly "grouped" should the need arise.

 

#5 - Regarding the "security feature" - if you're asking is there an option not to download graphics/images etc - then the answer is yes. On the flip side, there is no function in Outlook that automatically adds new contacts

 

Given the above - not sure the questions about "contact redistribution" are applicable since there is nothing in Outlook that automatically saves contact items as mentioned above. How you structure your Outlook data should be based on the following:

- keep your data in the simplest structure possible that enables you to retrieve your information quickly and easily. Too much structure can end up being as bad as having no structure at all. There is no universal rule on what that balance is when it comes to "structure" since what works for one person doesn't for another.
