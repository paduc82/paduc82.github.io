mike h.. say:

Hi,

This issue raises the blood temperature of some to boiling point, while others simply shrug. It would seem you're more sympathetic to the former group.

First what's going on. Excel is trying to be helpful and if you paste a value in a cell that can be interpreted as a date then Excel will do that so a fraction of 1/4 will become 1 Apr and get the current year added and something like 9/32 will become September 1932 and Excel will add the day as 1. If a fraction cannot be interpreted as a date then Excel will not attempt any conversion

Formatting as fraction beforehand doesn't stop this behaviour and you can't turn it off.

There's a workaround. Work out which columns are going to get the fractions pasted into them and format these columns as TEXT and then paste in the data. After that select the column and format as FRACTION and then start pressing F2 and enter to convert them to fractions.

See the thread below which is about the same/similar issues

http://answers.microsoft.com/en-us/office/forum/office_2007-excel/stop-auto-correction-of-number-into-a-date/9968c54a-221b-4b18-a3d1-cfd3d312a8a6

vijay a. verma say:

Will just build on Mike's answer. If there are 100s or 1000s of entries are there, it will take a lot of time to press F2 in individual cells. In this case, I would recommend formula approach.

1. As Mike has suggested your column has to be formatted as Text before you start entering the fractions. Let's assume that this is column A.

2. Column A need not be formatted as Fractions but Column B needs to be formatted as fractions. Now, the Fractions formatting gives a maximum of 3 digits of numerator and denominator. If you want more, you need to do custom formatting for column B as "# ?????/?????" without double quotes. Here I want 5 digits of numerator and denominator. You can increase / decrease "?" as per your requirement.

3. Put following formula in B1 and drag down

=LEFT(A1,SEARCH("/",A1)-1)/RIGHT(A1,LEN(A1)-SEARCH("/",A1))

This formula can be written in many other variations also but above one is a sleek one.

4. Since Excel stores numbers in lieu of fractions, hence 2/4 will get reduced to 1/2. Hence, I will recommend that you keep column A as it has the original fractions.

5. Additional Note - It may happen that sometimes few values in column A might be Fractions themselves not Text (accident or by design), in that case, the formula will have to be enclosed within IFERROR. It would become -

=IFERROR(LEFT(A1,SEARCH("/",A1)-1)/RIGHT(A1,LEN(A1)-SEARCH("/",A1)),A1)
