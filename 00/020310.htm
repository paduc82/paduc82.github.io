daniel g. grau say:
Labels are just fixed-size tables. Proceed as follows:
  1. Create a blank document (useless step, but required).
  2. In the main menu bar, select Tools > Labels.
  3. If you type in the Address field, you will create 30 copies of the same label, so leave it blank.
  4. Click on the Options button (a new dialog opens).
  5. Select "Laser and inkjet" and in the "Label products" dropdown, scroll down to "5160 - Address" and highlight it.
  6. Press OK to close the Options dialog and again to close the Labels dialog. This will create a table with 30 labels (3 across x 10 down).
  7. To add more labels, press the End key to go to the end of the document (actually, the end of the table), then press Enter to create an extra row. Immediately press Cmd Y to repeat the operation and obtain additional rows.
  8. Type the addresses in the fixed-size cells (30 of them per page).
  9. Save your document!

Microsoft is aware of the problem affecting a number of users running a full scan with MSE.

Most of the lockups are occuring in two folders:

\Windows\Winsxs\backup and/or \windows

Installer\$PatchCache$\Manager\

At this time, there are two workarounds while we await a solution from Microsoft which will probably come in the form of updated signatures or an update to the scanning engine.

  1. Exclude the above folders from being scanned in MSE settings
  2. Change your scheduled scan to a quick scan

As long as real time protection is enabled, a full scan on a frequent basis is usually overkill. A quick scan looks for malware in the most common locations. A full scan has to touch every file. Malware found in a full scan, not in a quick scan is typically not active. Personally, I only ever run a full scan if an infection has been detected or if one is suspected. And, if the machine is running MSE, I use a different scanner for the full scan (the malware was missed by MSE when it arrived, so why should I expect it to find it on a full scan?)
