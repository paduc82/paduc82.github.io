Insert date in a Word template - Microsoft Community

still joe mcguire say:

Insert date in a Word template

How can I insert a date in a Word template (letters, fax sheet, etc.) so that on a day when I create a new document with the template the document will show the date the new document was created, not the date I inserted it into the template?  I also want the date on the new document to be fixed, i.e., not get updated each time somebody opens the document in the future.  I knew how to do this in Word 2003 but can't figure it out in Word 2010.

jay freedman say:
Insert a CreateDate field in the template. You can go through Insert > Quick Parts > Field, but it's quicker just to press Ctrl+F9 to make a field and type the word CREATEDATE inside it.

In the template itself (opened for editing as if it was a document), the field will show the date the template was created. When you base a document on the template, the same field will show the date the document was created.

The date in the document will be permanent an unchanging (unless you use the Save As command, which resets the value to the date of that command).

still joe mcguire say: Thanks!  Good thing you warned about Save As.  But your answer raises another question.  I pretty much always use File/Save As when I save ANY Word document and have been doing so as long as I can recall, probably back to Word 95.  Now that I try doing so with File/Save the procedure seems to be the same.  Is there any difference bwtween Save and Save As?  Any other pitfalls?

jay freedman say:
The first time you save an unnamed new file, Save is the same as Save As (in fact, the dialog that opens for Save says "Save As" in its title bar). The Save command doesn't have any separate dialog.

After a document has a name and a folder path, the two commands are quite different.
  • The Save command uses the existing name and path, and overwrites the file contents with what's in memory. (If you click Save when there are no changes, Save does nothing.)
  • The Save As command opens the Save As dialog to prompt you for a new name and/or path. You can (but shouldn't) keep the same name and path, in which case it acts as a Save. Otherwise, you get a complete new file with the new name, path, and creation date; the old file remains unchanged.

This scheme often causes trouble when people open an existing document that they want to keep unchanged; they make some edits, and intend to use Save As to put the new information in a new file without altering the old file. But they forget, and they click Save instead of Save As. At that point, the data in the old file is (more or less) permanently destroyed, overwritten by the new data. (There might be a temp file with the old data, but probably not.)


There are two much better ways to do this: (1) Save a copy of the old document as a template, and use the File > New command to make new documents based on the template; or (2) in the Open dialog, select the old file, click the down arrow on the Open button, and click Open as Copy. The first method will force you to use Save As for the first save and prompt you for a new name/path, and the second method will stick the word "Copy" and a number at the beginning of the name to make it a different file.

They are NOT East Indian.  They are faking the accent.  They are most likely holed up in some run-down office building with a large server system auto-dialing numbers and covering their tracks after you hang up.  

The Feds know about this scam, but are unable to do anything about it because of the devious "dial-and-run" system that these scammers use. 

It generates a number that will show up on the caller ID of your phone (completely fake number) and once you hang up, it disconnects the number and erases any trace of it ever existing and also (supposedly) alters it's location ping and (since they apparently use a computer based calling system) effectively becomes a "ghost machine" ...these "ghosts" reportedly rewrite their electronic signatures (IP address, location, DNS, server number, etc.) every time they generate a number.  This has effectively made them IMPOSSIBLE to catch or track.  I speak from personal experience because I have been called by these scammers MULTIPLE TIMES, and even gave one of the numbers to my local police department hoping they could track it.  I later got a call back from the police asking me if "I was trying to pull a prank on them." ...I wasn't and I told him (the officer on the phone) that.  I told him the details of what I knew about this scam, and after explaining it to him, he agreed to keep an eye out for suspicious numbers.  (Apparently the department got calls from these scammers as well.)  So going to your local police with one of these "ghost" numbers is a waste of time.  All we can really do is just watch for the tell-tale "windows tech" flub that they seem to always make, and hang up on them.  It's an annoyance at this point, but there's nothing that can be done about it right now.
