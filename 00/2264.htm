ken sheridan say: No, there's no need to create separate query, the SQL statement being built by the code is what's executed.  However the code could be improved:

On Error GoTo Err_Handler

    Dim strSQL As String

    strSQL = "UPDATE wrkTable SET Wrktable.Chosen = TRUE "&_
        "WHERE WrkTable.Department= """ & Forms!EdEnroll!Department & """"

    If Frame30=1 And Not IsNull(Forms!EdEnroll!Department) Then
        CurrentDb.Execute strSQL, dbFailOnError
        Forms!EdEnroll.Repaint
    End If

Exit_Here:
    On Error GoTo 0
    Exit Sub

Err_Handler:
    MsgBox Err.Description, vbExclamation, "Error"
    Resume Exit_Here

I've assumed that the Department column (field) is of text data type.  If it's a number data type, however, amend the code to:

strSQL = "UPDATE wrkTable SET Wrktable.Chosen=True "&_
    "WHERE WrkTable.Department= " & Forms!EdEnroll!Department

By way of explanation:

1.  Calling the Execute method of the database object rather than the RunSQL method of the DoCmd object enables you to use the dbFailOnError option to trap and handle any error in the execution of the SQL statement.  Sometimes you'll want to take a specific course action in the event of an anticipated error, but here you are simply informing the user in a message box should an unanticipated error occur.

2.  The Value property is the default property of a control so does not need to be specifically included when referencing a control's value.

3.  Concatenating the value of the control into the string expression is better than referencing the control, and is essential when calling the Execute method as otherwise you'd get a missing parameter error.

4. Indenting the code where appropriate, e.g. within an If....End If construct, makes it more readable by clarifying the logical flow of the code.

rohn007 say:

{ TOC \0 "1-2" \h \z \u}

That code specifies

\o  use heading levels to generate the TOC

"1-2"    use heading levels 1 and 2 in the TOC

\u

Builds a table of contents from paragraphs whose formatting includes outline levels applied directly, in paragraph settings.

\z

Hides tab leader and page numbers in Web layout view.

\h Hyperlinks

Inserts TOC entries as hyperlinks.

Here is the full spec for the field code:

https://support.office.com/en-us/article/field-codes-toc-table-of-contents-field-1f538bc4-60e6-4854-9f64-67754d78d05c

Does that match what you want to do?

stefan blom say:

\o "1-2" means that Word will create table of contents entries based on each paragraph to which a style with an "Outline level" of 1 or 2 have been applied. This includes the built-in headings as well as any custom headings with those "Outline levels."

\u instructs Word to also include paragraphs to which an "Outline level" have been applied as direct formatting.
