shawn - support engineer say: Hi Crispy75,

Thanks for posting. The issue you are describing may be because the installer is trying to put temporary files in your OEM partition. Please try the following as a workaround.

Put your Windows 7 disk in.
Now with XP booted:
Click Start > Run
Next to open, type CMD
This should open a command line.
In the command line type D: (or whatever your DVD drive letter is)
From here type setup.exe /tempdrive:drive_letter (driveletter being whatever your hard disk system drive is,,.,, usually c:)
Now allow the setup to run and reboot your computer to continue the upgrade.

Hope this helps!

Try following and check after each step (Note - You will not lose any mails because of below step) - -

1. Remove the account and re-add it.

2. Reset Mail app
Right click on Windows Start > Apps & features > Scroll down and click Mail & Calendar > Click Advanced options > Click Reset > Restart your computer
Go to Microsoft Store > Click 3 horizontal dots on top right > Downloads and Updates > Check if there is any update for this app > If yes, update it.

3. Uninstall and Reinstall Mail app

A. Uninstall - Right click on Windows Start > Click Windows Powershell (Admin) > Copy the below and paste into Powershell and Enter

Get-AppxPackage Microsoft.windowscommunicationsapps | Remove-AppxPackage

B. Reinstall
Right click on Windows Start > Click Windows Powershell (Admin) > Copy the below command and right click in Powershell to paste > Enter

Get-AppxPackage -allusers Microsoft.windowscommunicationsapps | Foreach {Add-AppxPackage -DisableDevelopmentMode -Register “$($_.InstallLocation)\AppXManifest.xml”}
