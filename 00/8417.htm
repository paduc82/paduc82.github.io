[Failing Module Name:  Unknown]

That suggests a problem with add-ons.  Possibly something that was loaded where the error exception occurred  (0x0d325000) but was unloaded and its storage freed.  You might get a clue from ProcExp in this regard if you know how to reproduce it.  E.g. scott .. b say:      x86 is the name of a processor architecture from Intel that handles 32 bit instruction sets. The x86 term has been used for a very long time and in the beginning it was used as a general term to refer to Intel 16/32 bit processors with names such 8086, 80186, 80286, 80386 etc. But since the release of the 80386 processor, the first real 32 bit processor, the term x86 has been used to refer to 32-bit processors that have an instruction set that is compatible with the old 80386 processor.

dale howard [mvp] say:

KelvinK --

You made several mistakes along the way, but good try, my friend.  As you now know, you cannot copy the calendars into the Enterprise Global file.  There is another way that will not require you to recreate the calendars, so that will save you some time.  Retain the MPP file containing the calendars from Project Server 2003.  Here is what you will need to do first:

  1. Log into Project Online with administrator permissions.
  2. In the upper right corner of the Home page, click the Settings button (gear wheel icon) and select PWA Settings on the menu.
  3. In the Operational Policies section of the page, click the Additional Server Settings link.
  4. In the Enterprise Settings section at the top of the Additional Server Settings page, select the Allow Projects to Use Local Base Calendars option.
  5. Click the OK button.

Those first five steps are the key to being allowed by Project Online to import calendars from existing projects. Next, follow these steps to import the calendars:

  1. Launch Microsoft Project 2013 and connect to Project Online.
  2. Open the MPP file containing the calendars from Project Server 2003.
  3. Click Project > Change Working Time.
  4. In the Change Working Time dialog, click the For Calendar pick list at the top of the dialog and select any calendar OTHER THAN the Standard calendar.
  5. In the upper right corner of the dialog, click the Add Calendar to Enterprise button.
  6. In the Add to Enterprise dialog, enter an alternate name for the calendar, if necessary.
  7. Click the OK button.
  8. Continue steps #4-7 until you have imported all of the calendars from the MPP file.
  9. Close the Change Working Time dialog.

In Project Online, if you navigate to the Enterprise Calendars page, you should see each of the custom calendars you just imported.  Beyond this, if you do not want your PMs to be able to create and use local calendars in their own projects, go back to the top of this e-mail and reverse the setting in step #4.  Hope this helps.

bernie deitrick say:

I've assumed that you want to account for month as well, not just day.

Use a column of formulas - for a birthday date in cell B2, use this date formula that makes the year not matter:

=DATE(2017,MONTH(B2),DAY(B2))

Then fill down to match your list of dates, and sort your entire table based on the column of new formulas.
