lemp say:
If you encrypted the files using Windows Encrypting File System (foolishly, IMO, included by Microsoft with Windows XP Pro), you're out of luck.  EFS is often called the "Delayed Recycle Bin" -- because it's just a matter of time until files encrypted using this system become unrecoverable (unless you have followed MS's Best Practices for EFS).

Although there is one company that I know of that claims to be able to decrypt EFS-encrypted files, it's my understanding that this requires access to the system disk of the computer that encrypted the files and not just a collection of encrypted files on an external disk, such as your situation.  You could ask them --> http://elcomsoft.com/aefsdr.html

You could also inquire at the professional data recovery specialists, such as Kroll OnTrack and DriveSavers, but IF those companies have the ability to recover EFS-encrypted files AND IF the use of that ability is not limited to law enforcement clients, it will cost you -- a lot.

Your best bet is to find that old notebook and hope that its hard drive is still intact.

See also
The Encrypting File System
5-Minute Security Advisor - Recovering Encrypted Data Using EFS

The answer is probably the same if you used a different method of encryption (and similarly failed to save the credentials necessary to decrypt the files).  See, e.g., this comment from TrueCrypt:

I forgot my password ‚Äì is there any way ('backdoor') to recover the files from my TrueCrypt volume?

We have not implemented any 'backdoor' in TrueCrypt (and will never implement any even if asked to do so by a government agency), because it would defeat the purpose of the software. TrueCrypt does not allow decryption of data without knowing the correct password or key. We cannot recover your data because we do not know and cannot determine the password you chose or the key you generated using TrueCrypt. If you follow the security requirements listed in this chapter of the documentation, then (to our best knowledge) the only way to recover your files is to try to "crack" the password or the key, but it could take thousands or millions of years (depending on the length and quality of the password or keyfiles, on the software/hardware performance, algorithms, and other factors).
