cmdrkeene say:

No problem, I understand.  My use is personal so I use the freebie version.  But for the remote desktop, it's not too difficult, just takes several steps to setup the first time, although it has a little caveat that your internet provider could change your external IP address and you wouldn't be able to connect until you find out what the new one is.  That's probably rare.  Even on my residential connection, Cox Cable here almost never changes my IP address.  And with most business Internet accounts, the provider flat out agrees not to change it.

Here's the process.  It varies based on your router (I'm using a cheap home Linksys router, but it's very similar).

  1. First, give the computer you're connecting to (the office PC) a static IP address.
    This page explains how in probably more detail than you would want :)

  2. Tell your router to forward the incoming remote desktop connections to that internal IP.
    Login to your router settings and look for Port Forwarding or Port Range Forwarding.  It's usually easy to spot. Mine is below, and you can see it incoming 3389 connections (you can specify a range but for RDP you only need one, so my range is just that one port) and forwards them to my .104 internal address.  Click here for a larger version of this picture.



  3. Find out what your office's external IP address is.
    You can dig around in your router's settings, or just ask Bing "what is my IP address?" from your office computer.

That's it.  Later when you want to connect to your office computer, you'll use the external IP address of your office's internet connection, and the router will forward that request over to the computer you want to reach.

If you want to be able to connect to more than one PC, you can set another computer to use a different port number instead of 3389 (official Microsoft instructions here) and do the same steps to forward 3390 to the other computer.  Later when you want to connect to the other computer, type your IP address as IP:Port, such as 11.11.11.11:3390 and it will connect to that other computer.

lemp say:

To install XP Mode in Windows 7 Pro, you need to also install Windows Virtual PC.

Given the age of Windows 7 and XP Mode, the download links in most of the online XP Mode installation guides are no longer valid.

Download the Windows Virtual PC here, being sure to select 32-bit or 64-bit as appropriate for your installation of Windows 7 Pro:  https://www.microsoft.com/en-us/download/details.aspx?id=3702

For completeness, the download for XP Mode is here (you almost certainly do NOT want the "N" version):  https://www.microsoft.com/en-us/download/details.aspx?id=8002

You do NOT need to keep Firefox open.  In fact, unlike Internet Explorer, Firefox should just save downloaded files and not ask if you want to "run" the files as they are downloaded.  If you are not sure where Firefox saves the files it downloads, click the menu icon (the 3 horizontal bars at the top right corner of the Firefox window) and select Library > Downloads > Show Downloads Folder.  This will open a Windows Explorer window showing the folder where your downloads are saved.  You can now close Firefox (although leaving it open won't harm the installation).

Once you have downloaded both installation files, follow the directions here:  https://www.bleepingcomputer.com/tutorials/use-windows-xp-mode-in-windows-7/

Note that connecting XP Mode to the Internet is just as dangerous as connect a non-virtual installation of Windows XP to the Internet.  Neither will update beyond the security patches available as of April 8, 2014,  Most, if not all, current antivirus programs will not run on XP.
