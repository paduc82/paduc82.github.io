hansv mvp say:

The line


    ActiveWorkbook.Names.Add Name:="nrGrades1", RefersTo:=d.keys


will create a name "nrGrades1" whose contents is a "horizontal" array of the unique values from column A. If you want a "vertical" array, use


    ActiveWorkbook.Names.Add Name:="nrGrades1", RefersTo:=Application.Transpose(d.keys)


You can now use the name in formulas such as


=COUNT(nrGrades1)


Remark: the values will not be sorted in any particular order. If you want them to be sorted in ascending or descending order, you'd have to sort the dictionary first.

dirk goldgar say:

I have a database of birds.  I have a query that, by entering colors, it shows me what birds have that color.  Is it possible to show pictures of the birds also by using a form view?

It seems to me that it should be, provided that you have the pictures.  I presume you'd use an image control on the form to show the picture.  The pictures themselves might be stored in the database, or (probably better) the database might store the paths to the image files.

I'd need to know more about the structure of your database to give more detailed suggestions.

ken sheridan say: You'll find examples of various means of displaying images in a form and report as Images.zip in my public databases folder at:

https://onedrive.live.com/?cid=44CC60D7FEA42912&id=44CC60D7FEA42912!169

If you have difficulty opening the link copy its text (NB, not the link location) and paste it into your browser's address bar.

It sounds like you want to associate a single image with each record, so the Images_Simple.accdb file should provide a suitable model for your form.  The images are stored as separate files and are loaded into an image control in the form by setting the ControlSource property of the image control to the path to the file, which is stored in an ImagePath text column in the table.  The form's Current event procedure includes some code to hide the image control and show a 'No Image' label if ImagePath  is Null, but this is not essential to the operation of the form.

One caveat; as modern cameras create very large image files these can be slow to load into the form.  It is therefore best to create lower resolution files from the originals and use these for the database.

The report works in a similar way.

If you add an ImagePath column to the relevant table and base your form/report on your query you should be able to open the form/report restricted to those records which match your search criteria.

My file is now quite old and uses Bill Wilson's freely distributed class module to open the dialogue to browse to the image files.  This works fine, but these days I'd be more likely to use the FileDialog method of the Application object to do this.

From your description, I would design a continuous form, bound to the query that returns the list of birds that meet your criteria.  On that form I would have an image control bound to the field containing the path to each bird's picture, with the image control's SizeMode property set to Zoom.  That way, the displayed list would show each bird's picture, although in a small size.  I would probably define an event procedure for the image control's Click or DblClick event to either open a single-form detail view of that particular bird, or else open a larger, single-form view just showing that bird's picture in a larger format.

praxius2 say:

So I found the answer to my own question, but for anybody else curious:

To do what I am doing, you first need to hit Win+R and type netplwiz

Uncheck "Users must enter username and password" - You will need to enter and confirm your password before it saves your setting. This cancels the login screen and automatically logs you in.

Next hit Win+R and type shell:startup

This will open up the folder that you can put any program, file or shortcut into. Anything in here will automatically play when Windows logs in.

Copy the file you need somewhere on your system, such as the desktop or wherever. Right-click on the desktop and go New>Text Document. Then copy/paste the following:

start wmplayer "D:\Desktop\yourfile.mp4" /fullscreen

Replace "yourfile.mp4" with whatever file name your file is, as well as change whatever location you put it.

Now File>Save As the text document, switch it from a txt document by selecting "all files" and then save the file as your file's name with a .bat at the end as the extension.

Move that new bat file to the Startup folder and you're done.

What this does is if your system restarts, it will automatically bypass the login screen and once at the desktop, it will play your file full screen via Windows Media Player..... if you already clicked the icon for Repeat before, it should remain active and thus, your video will go full screen and loop continuously without requiring any input.
