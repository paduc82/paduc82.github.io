scottgem say:

What datatype is the Number of Drugs field? I'm guessing it is an Autonumber. If it is then you are misunderstanding the purpose and usage of a autonumber. An autonumber has only one purpose. That is to assign an unique identifier to a record. An Autonumber is not a sequential number. It can set to increment from the last number used. But once a record is deleted you can't use that number again. 

I'm also curious as to why this field was named "Number of Drugs". Are you expecting it to be a count of the number of drugs in the table? if so, then no it won't be. If you need to get a count of the number of drugs, then use the expression:

DCount("*","Drug Supply")

Do you need sequential numbering? I don't see any real reason for sequential numbering in this case. 

Many developers will tell you that an Autonumber should never be exposed to the user. I don't subscribe to that, but it is mainly for use internally.

A couple of other tips:

Don't use spaces in object names. This can come back to haunt you. Use Camel Notation (GenericName) or underscores (Generic_Name).

When posting here, use the thread title to describe your question. We know you need help, but a subject like numbers out of sequence will help tell us what your question is about.

Sorry but your video is not the right solution here. What you are saying is that EVERYTIME a user wants to renumber the Autonumbering they have to perform this process. ken sheridan say: It is possible to insert a specific value into an autonumber column, but it must be done by means of an INSERT statement (an 'append' query).  Occasionally this done (most commonly to insert zero to represent an 'N/A' value or similar), but in most cases the actual value of an autonumber column is immaterial, it's sole purpose being to distinctly identify each row as a key.

Where sequence is necessary it is far better to compute the number when a row is inserted.  You'll find examples in CustomNumber.zip in my public databases folder at:

https://onedrive.live.com/?cid=44CC60D7FEA42912&id=44CC60D7FEA42912!169

If you have difficulty opening the link copy its text (NB, not the link location) and paste it into your browser's address bar.

Before implementing these sort of routines you should be satisfied that there is a cogent operational necessity for sequential rather than merely distinct values.  In the overwhelming majority of cases there is not.

vijay b say:

Hi Palle,

Thank you for being a part of Windows 10 Technical Preview testing.

I understand that you would like to install the .Net Framework 3.5 feature on Windows 10 Technical Preview.

The .NET Framework is an integral part of many applications running on Windows and provides common functionality for those applications to run. Normally, before running/installing such applications, we need to enable .NET Framework from the Control Panel on the computer.  So, you may first check if the .NET Framework 3.5 is available in the Control Panel on Windows 10 Technical Preview and if available, you may enable it from the Control Panel to install it on the computer. To check if the.NET Framework 3.5 is available in Control Panel and to enable it, please follow these steps:

 a) Press Windows Logo + R keys on the keyboard.
b) Type appwiz.cpl in the Run command box and press ENTER.
c) In the Programs and Features window, click on the link Turn Windows features on or off.
d) Check if the .NET Framework 3.5 (includes .NET 2.0 and 3.0) option is available in it.
e) If yes, then enable it and then click on OK.
f) Follow the on-screen instructions to complete the installation and restart the computer, if prompted.

This will install the .NET Framework 3.5 feature on the computer. There is also another way by which .NET Framework 3.5 gets installed on the computer. This is done by the apps on demand.

If an app requires the .NET Framework 3.5, but doesn't find that version enabled on your computer, it displays the prompt for installing the .NET Framework 3.5 on demand, either during installation, or when you run the app for the first time after its installation. In this prompt, you can choose the option Install this feature to enable the .NET Framework 3.5 on the computer.

Hope this information is helpful. Please get back to us with the updates on this.
