Any tips around???

bob jones aka: cybertaz say:

There really are no [practical] "tips around" :-)

IMHO, Microsoft has been forced to cover their own corporate a** in order to avoid being held legally responsible for users distributing documents containing tracked changes that shouldn't be included. Ergo: Once changes have been tracked in a document, they will be displayed when the document opens regardless of which view was in use when the document was last saved & closed.

Further to the point, simply turning Track Changes off does not remove the changed content or markup from the document. You must also resolve [Accept or Reject] all existing changes, then Save the document or use Save As to generate a 'clean' final copy.

BTW: Here's one 'tip', though :-) Don't permit Tracked Changes to perpetually live in any document, even with the feature turned off. Realistically, editing tracked in any given session should be resolved as soon as possible. Accumulating changes one session after another - especially if further changes are made to the same content - is one of the major causes of document corruption. The sooner changes made during one session can be resolved before starting to track changes in another session, the better.

scottgem say:

Sorry its not going to happen. Excel uses a similar but different set of functions. Plus Excel references data by rows and columns not records and fields. so there is no way the calcs will come over. 

Note also that while there is a calculated datatype. its more usual to do your calcs in queries and not store them in tables. 

You might check out my blog article on the calculated datatype.
