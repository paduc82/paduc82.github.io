Windows 10 upgrade on Dell Latitude E6420 - Black screen and flickering desktop issue solved

If you are upgrading a Dell Latitude E6420 to Windows 10, note that Dell declares that this system has not been tested for Windows 10 and they do not plan to support.  However, I managed to do it, but not without some issues.  I figured that anyone with this particular model who is trying the Windows 10 upgrade is going to run into this issue, so I'm just posting it here.  Here is the issue you may see:

Upgrade ends in a black screen.

After dealing with that, the desktop refreshes a couple of times a minute.  This may actually be explorer.exe restarting due to an issue.

Here's how I got around these problems:

1. CTRL + ALT + DEL and a screen will appear (... out of the darkness of your black screen) and choose Task Manager.

2. Click File > Run new task and run iexplore.exe to start a browser.  If you had the same issue I did, the browser window will appear, but the desktop and icons will not appear.

3. Surf over to Dell, and I won't go over all this, but have Dell determine which system you have (download an app, run it, and in the browser, Dell will confirm your system).

4. Choose to check for driver updates.  In my case, it wanted to update the Intel 825XX ethernet driver that Windows 10 thought was the latest and greatest driver version.  Let it update and then restart the system.  If your problem is the same as mine, the Windows desktop will actually appear after boot.  Somehow, there was some issue with the desktop launch getting hung on the old driver.  Doesn't seem related, but it worked for me twice (previous Win 10 installation on the same machine).  

Now, you may be seeing your desktop/icons flicker a couple of times a minute.  The explorer.exe is getting reset due to a conflict with an app (if you have the same problem I do).  See below.

1. From Task Manager (or the "Search the web and Windows" field next to Start) Click File > Run new task and run the "eventvwr"

2. If you see the following error in the screen capture below concerning explorer.exe and IDTNC64.cpl, it could be that this is the same problem that I had.  You'll need to uninstall a legacy app and the flickering desktop issue will go away.  Go to this link to see how it's done:  https://www.reddit.com/r/Windows10/comments/3f90kz/windowsexe_keeps_crashing_every_second/

For those looking for a quick solution, this is what I did:-

  1. Download the latest fingerprint driver from Lenovo's support website.
  2. In Device Manager, uninstall the fingerprint driver under Biometric devices and select the option to delete the driver software as well.
  3. Before restarting Windows, install the latest fingerprint driver downloaded in step 1.
  4. Scan for hardware changes in Device Manager. The fingerprint driver should have reappeared.
  5. Open services.msc from Run (Win+R) and stop the Windows Biometric Service.
  6. Go to C:\Windows\System32\WinBioDatabase folder and delete the .DAT file having alpha-numeric name.
  7. Now, open TPM management by entering tpm.msc in the Run window.
  8. In the right-hand pane, click on Clear TPM and wait for Windows to restart automatically.
  9. After restarting, open BIOS settings (UEFI in my case).
  10. Again clear the TPM from the BIOS settings. It may be listed under Security tab.
