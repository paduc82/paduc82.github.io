chirag. say:

You could perhaps find the Adobe Presenter command button and execute it directly. If the menu item caption is "Adobe Presenter", iterate over the controls within the Insert menu to locate the Adobe's menu item and call Execute() on it. The following code snippet could help you get started:

Sub CallAdobePresenter()
    Dim CBB As CommandBarButton
    Dim CBP As CommandBarPopup
    Dim I As Long

    On Error Resume Next

    Set CBP = PPT.CommandBars.FindControl(Id:=30005)
    For I = 1 To CBP.Controls.Count
        If CBP.Controls(I).Type = msoControlButton Then
            Set CBB = CBP.Controls(I)
            If Left(CBB.Caption, 5) = "Adobe" Then
                CBB.Execute
                Exit For
            End If
        End If
    Next
End Sub

Here, PPT is the PowerPoint instance that you have in your Access VBA.

Hi Alex,

I think the suggestion works, although it would be have to be done on every site individually. 

I came up with a different solution: create a list of people in the Project Desktop Resource section and save this as a filter to the Global Template. Using the filter, add these people to the project team of every project file so they are considered a 'Team member' of the project (even if they don't have any tasks assigned to them), and then they can edit Risk items on the Project Site, but not Tasks or the Schedule in PWA. 

It's still fairly manual in that it has to be done for every project, but it seems to achieve the result alongside the existing permissions structure.
