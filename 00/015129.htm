    Using "et al." in IEEE style citations (Word 2010)

    Hi

    I am using the IEEE style for references. In the list of references at the end of the document, I sometimes want to say, for example, "Smith, John, et al." and then the details. Smith et al" but I can't seem to get this...can anyone help, please?

    Thanks

    macropod say:

    In theory, at least, you should be inserting every contributing author's names into the source field, so that Word's XML code had handle the formatting for both the citations and the bibliography. Whether this works correctly with the IEEE format, I can't say. It does for some of the others, but MS's implementation is best described as inconsistent and often doesn't conform to the formats it claims to support.

    mattgilluk say: Unfortunately, this is not practical for lists with number of authors being too big. e.g. I wanted to cite a paper which had at least 20 authors, and I certainly wasn't going to enter all those!

    macropod say:
    That sounds like you're working with a publication with an editor and a number of authors contributing different portions. In that case, you should have a separate reference for each author whose work is cited (use the 'book section' referencing option, not an amalgam of the lot.

    mattgilluk say: No, that's not the case. In scientific papers one does sometimes find a very long list of contributors to the paper, as in my case. There aren't separate portions of the paper which you can identify as being written by differnt authors.

    macropod say:

    In that case, you should be listing all contributors. Doing so isn't at all difficult - simply create a list in the form of:

    Surname, first given name/initial second given name/initial; Surname, first given name/initial second given name/initial; etc.

    so that you end up with something like:

    Smith, L W; Jones, A B; Williams, J A; Robertson, G R

    then paste that into the author field. There is no need to input each name individually. You can do the same for editors, translators, etc,

    bryado say:

    Paul, you'd be a lot more helpful answering the question instead of trying to school the OP on YOUR view on proper referencing.  The IEEE Style Guide states that any citations of 3 or more authors permit the use of "et al.", and so this method is 100% acceptable in this person's academic domain.  Especially considering it is probably for a condensed journal article, not some graduate thesis.

    Matt, sorry, this is probably too late for you, but for the rest:  one way to do so is to list the author as a "Corporate Author", which takes the literal form which you type.  The only downside is that if you want to use the full author list for this citation in some other paper you'd have to make a second entry, or more simply modify your master list entry.

Assuming Word's IEEE referencing implementation is correct (MS hasn't done a good job of this with other formats), it will automatically insert the et al, as appropriate. I have no axe to grind re 'proper' referencing.

    mr andrew say:

    If you have Word you can easily strip the lines out in a single search-and-replace:

    With the "use wildcards " option turned ON enter (^13)999999*^13 in the Find What box and  ^p in the Replace With box, then click Replace All

    This will work on all lines that begin with "999999" except for the very first line of the document (which you can remove manually if it begins with 999999)

    The file will now be ready to import into Excel.

    You can record this as a macro if you like.

The article at http://word.mvps.org/FAQs/Formatting/TextReflow.htm suggests a few best practices to avoid significant differences in pagination, but PDF conversion can be very frustrating.

    As a case in point, not too long ago I was working on a document (Word 2003) that looked fine on the screen and printed properly from any printer, but, when it was converted to PDF using Adobe's PDFMaker add-in, several lines of text that spanned a page break were entirely omitted (you can imagine my dissatisfaction!).
