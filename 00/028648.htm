john mcghie say:
Hi Rich:

Hewlett-Packard uses Sharepoint very successfully as the version control system for some of their quality management systems, so it can indeed be done.  However, I should warn you that the authentication, access-control, and accounting requirement for even a simple ISO quality management system is not for the faint-hearted.  You could expect to need four degree-qualified experienced professionals for a whole year to set one up from scratch through to accreditation.

I know, I've done one :-)

Fortunately, there are off-the-shelf packages available -- for example http://www.qualitysystems.com/ but there are a large number of them out there.

You are heading down the wrong track with autotext, what you need is "Fields".  The concept is explained here:  http://www.smallbusinesscomputing.com/buyersguide/article.php/3754951/Microsoft-Word-Working-with-Field-Codes.htm

You will find that Title, Author, Date Created, and Approved By are already built-in, you just need to store values in them.

Even better than "Fields" would be "Document Properties".  These can be read and written with Word as well as software other than Word.

And when it comes to Quality Management Systems where verification and an audit trail are important, you should really be using Document Variables.

Now for the downsides:

*  Few users know how to use "Fields".  Strange as it may sound, one of the oldest and most powerful tools in Word is largely ignored by new users.  When Word first came out, everyone had their name, address and phone number stored in Word fields in their Normal template.  Nowadays, few users can find the Normal template!

*  Fields are publicly visible.  Any user who can edit the document can alter the content of fields, and any user attempting to cover their "a" in a quality management system WILL :-)

*  Document Properties are less visible, but they are still publicly visible and editable by users who know how and where to find them.  And a user who has just been guilty of a non-compliance will find them...

*  Document Variables are just made for this purpose.  They are completely invisible and accessible only by software (Visual Basic for Applications).  They can be encrypted.  No user can read or write to them unless they have the keys.

So in answer to your question, "Yes" you can do this in Word, and Word has a few tools that help.  However:  the amount of work required to do it properly is huge.  Far better to buy an off-the-shelf solution, hopefully put together by an accredited ISO QMS Specialist who will do more than charge you a king's ransom for a pretty interface to a rubbish application.

Hope this helps

Whoa! Sorry, this is pretty difficult for automating:

  • The triangle graphics in the corners have wrap applied to them, then the type is positioned. So deleting the triangles changes the page layout.
  • Removing floating graphics with the macro still needs a little debugging, but I see there is also significant graphic content that is floating. A macro can't tell the difference.

kk_1982 say:

The original poster is asking about this situation:
 1. inside Word (or Excel), click on "send as Attachment" in the File Menu

 2. The poster wants this action to invoke the built-in Windows 8 mail app instead of Outlook.  (Right now, this action would bring up Outlook as the email app.)

It is clear that one can "attach" a document manually from inside the default mail app, but that's not what the poster asks about. It is also clear that a document can be shared in many other ways (sharepoint, onedrive) but that's again not what the original poster is

doug robbins - mvp office apps & services (word) say:
And the answer is that the only way to do it is to attach the document to a mail message that is initiated in the mail application.  The reason for that is that the WIndows 8 mail app is not MAPI compliant.  A properly configured MAPI compliant mail application is a prerequisite for being able to use the File>Share>Email facility within Word.
