Issues with the Photo App and the SD card - Microsoft Community

barrielipscombe say:

Issues with the Photo App and the SD card

Before I start I should add the proviso that this may be a local issue (ie., specific to me and something I did caused it) or it may be a more general issue (which I believe it is).

The problem began when I used the method described here ( http://winsupersite.com/article/windows8/surface-tip-microsd-content-libraries-metro-apps-144658) to attach my Photos on my SD card to the Photo app on Surface. It appeared to work well with the Photo app finding all pics, folders, and sub-folders on the card. However shortly after a problem appeared whereby the Photo app no longer recognised all the folders and suddenly was displaying empty frames in the main 'My pictures' folder on the C: drive. These frames were blank but had the files numbers of each photo that was missing in the top left hand corner (e.g., DSC 000567), and they related to deleted files that HAD BEEN DELETED VIA THE DESKTOP and not the deletion control in the Photo app.

I struggled for a long time to try and understand what had happened and what was wrong. All the other Photo related apps (e.g., MEMORYLAGE) still worked perfectly, finding and displaying all pics on the SD card and no blank frames, but it seemed the Photo app that came with Surface was locked into a problem (and no amount of Re-starting, uninstalling and reinstalling, etc. could fix the issue).

Eventually I realised the problem related to the Photo app's inability to 'Refresh' it's connection with the SD Card. Whatever is in the Photo folder on the SD card when the connection (via the Library) is made, is LOCKED. So if you delete a photo through any other method than using the Photo app it still thinks the photo is there. Likewise it will not find any folder placed into the PHOTOS folder on the SD card after the link is made (or any renamed folder). So for example, if you copy more pics into the Photo Folder via a USB stick connection, the Photo app simply does not see them. Likewise if you delete folders the app will show empty versions of the deleted folder.

The solution is to make a new folder (maybe 'Photos2' or whatever), copy all the files into this folder, disconnect the original 'photo' folder through the 'Library Management' command and then connect the new 'Photos2' folder. Then suddenly the empty frames disappear and all the folders (and sub-folders) are recognised again.

However the problem still persists in the sense that adding pics or folders to the 'Photos2' folder on the SD still leads to them being ignored or not recognised.

This is a very odd problem for, as I said, NONE of the other apps suffer this issue. They all quite happily 'refresh' and recognise what is currently on the SD card. Only the Photo App is locked to whatever the state of play was when the SD Card was linked.

As I said maybe this is just me, but if not I hope this post may help other people with some variation of the same problem. However if this is my fault and someone knows what I did to cause it, let me know :)

Thanks

jay jap say:

Hello WillPinder,

The I/O error usually occurs when a hardware is non-accessible. But there are multiple reasons why this happening from Hardware to software conflict.

Let's try some troubleshooting steps to drill down on the issue.

Our first option will be a clean boot to determine if a driver or program is having a conflict with the drive. Here's how:

  1. Press the Windows key on your keyboard and on the search box, type msconfig.
  2. Select System Configuration from the search results.
  3. On the Services tab of the System Configuration dialog box, tap or click to select the Hide all Microsoft services check box, and then tap or click Disable all.
  4. On the Startup tab of the System Configuration dialog box, tap or click Open Task Manager.
  5. On the Startup tab in Task Manager, for each startup item, select the item and then click Disable.
  6. Close Task Manager.
  7. On the Startup tab of the System Configuration dialog box, tap or click OK, and then restart the computer.

You can use this article for reference on how to perform a clean boot.

Note: Once you are done with the said troubleshooting step, please refer to Reset the computer to start normally after the clean boot troubleshooting on the same article.

Since this could be a hardware issue, let's perform a check disk to diagnose the hard drive.  Here's how:

  1. Press the Windows key on your keyboard and on the search box, type CMD.
  2. Right-click the Command Prompt from the results and click Run as Administrator.
  3. On the command prompt window, copy and paste this command and hit Enter key: chkdsk /r /f
  4. Type Y and hit Enter on the next prompt.

Let us know how it goes.

Most system failures result in a BSOD (Blue Screen of Death) and a dumpfile report is created providing that the computer has been configured to create reports. In a limited number of system failures no dumpfile is created. A report is generated when the computer is next booted and placed in the System Log in Event viewer. The report is given an Event ID number 6008 and gives very little helpful information, save that the cause is unknown and that it occurred at the date and time specified. . A further report is placed in the Reliability Monitor with a red orb on the Miscellaneous Failures line and it is recorded as a Critical Failure. Unfortunately the Technical Details for this type of failure reveal no additional information.

Hi Marianne. I'm Greg, an installation specialist and 9 year Windows MVP, here to help you.

What works most is to go to Mail Settings (gear icon at bottom of Inbox) then Manage Accounts, then choose the problem account(s), then Delete the Account.

Restart PC and retrace that path to add the account back. Make sure the Sync settings are as desired for how far back you want mail Downloaded from, whether to include Calendar and Contacts (People app).

If that's not enough then Reset the Mail app at Settings > Apps and Features > Mail & Calendar > Advanced Options > Reset. Restart PC.

Finally if nothing else works reinstall the Mail and Calendar app from Option Two here: http://www.tenforums.com/tutorials/3175-apps-re...

If you have broader performance problems go over this checklist to make sure the install is set up correctly, optimized for best performance, and any needed repairs get done: http://answers.microsoft.com/en-us/windows/wiki...

Start with Step 4 to turn off Startup freeloaders which can conflict and cause issues, then Step 7 to check for infection the most thorough way, then step 10 to check for damaged System Files. Then continue with the other steps to go over your install most thoroughly.

If nothing else helps then run a Repair Install by installing the Media Creation Tool and choosing from it to Upgrade Now. This solves many problems and will also bring it up to the latest version which you need anyway and by the most stable method. http://windows.microsoft.com/en-us/windows-10/m...
