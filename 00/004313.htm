scottgem say:

Well first your database is not normalized properly.  Having field named like SW3 & SW4 are indicative of a repeating group which violates normalization rules. You really should be using Child tables for those. And using tablenames to identify data (like DOAS Software Key_5 & DOAS Software key_6 is also poor design. You should have one table with a filed to indicate which key it is. All this is making your query very complex and very difficult to debug. have you tested to make sure the query is returning the records you want?

I would really suggest a redesign of your database to normalize it properly and then rebuilding your reports. Another Tip, its not a good idea to use spaces in object names, this can come back to haunt you later on.
