suzanne s. barnhill say:

Modifying the TOC styles will not accomplish what you want, though it is something you might want to do as well. Here are some things that actually will help:

a. user say:

In theory, you ought to be able to do

=SUM(ABOVE)-B2

However, for the reasons described below, I suggest that you create a very small blank graphic and insert it inline after each of the values in the "Header" cells, then use { =SUM(ABOVE) } in each column.

The presence of the graphic should prevent Word from recognising the Header cells as numeric.

Word's interpretation of values in tables is actually pretty weird, and may also vary depending on the regional settings of the person viewing the document, the language version of Word, and the version of Word. You may be able to find a post of mine here that describes some of the weirdness involved:

http://answers.microsoft.com/en-us/mac/forum/macoffice2008-macword/autosum-says-unexpected-end-of-formula-when-i/57398214-3cc3-4696-8c43-d4de69c41077

Let's just say that if you have "headers" that look like this:
 
2 hours to 8 hours

they wil not be included in the =SUM(ABOVE) or the =COUNT(ABOVE) 

whereas if you have headers that look like this

10

they will. But both will be included in a sum that involves an explicit range of cells, e.g.  =SUM(B2:B2) So unless you have a separate column with something that you can reliably count and reference in a calculation, reliably working out the =SUM is not straightforward.

Ensuring that the "Header" cells are not treated as numbers simplifies things enormously. After that, you only have to worry about data that doesn't look numeric in the middle of your table :-)

Try something like:

'=========>>
Option Explicit

'--------->>
Public Sub groupcopy()
    Dim wsInput As Worksheet
    Dim ws1 As Worksheet
    Dim LRow As Long

    Set wsInput = Sheet38
    Set ws1 = Sheet40

    Application.ScreenUpdating = False
    With wsInput
        LRow = LastRow(wsInput, .UsedRange)
        ws1.Range("A1").Resize(LRow).Value = .Columns("I:I").Resize(LRow, 1).Value
        ws1.Range("B1").Resize(LRow, 5).Value = .Columns("B:F").Resize(LRow).Value
    End With
    Application.ScreenUpdating = True
End Sub

'--------->>
Public Function LastRow(SH As Worksheet, _
                        Optional Rng As Range)
    If Rng Is Nothing Then
        Set Rng = SH.Cells
    End If

    On Error Resume Next
    LastRow = Rng.Find(What:="*", _
                       after:=Rng.Cells(1), _
                       Lookat:=xlPart, _
                       LookIn:=xlFormulas, _
                       SearchOrder:=xlByRows, _
                       SearchDirection:=xlPrevious, _
                       MatchCase:=False).Row
    On Error GoTo 0
End Function
'<<=========

Public Function LastRow(SH As Worksheet, _
                        Optional Rng As Range)
    If Rng Is Nothing Then
        Set Rng = SH.Cells
    End If

    On Error Resume Next
    LastRow = Rng.Find(What:="*", _
                       after:=Rng.Cells(1), _
                       Lookat:=xlPart, _
                       LookIn:=xlFormulas, _
                       SearchOrder:=xlByRows, _
                       SearchDirection:=xlPrevious, _
                       MatchCase:=False).Row
    On Error GoTo 0
End Function
