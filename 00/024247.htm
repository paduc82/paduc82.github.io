lars-Åke aspelin say:
i have a column of complex numbers that i need the average, max and min for what are the formulas i should be using?
all the column is D4:D9

For the average, try this formula:

=COMPLEX((IMREAL(D4)+IMREAL(D5)+IMREAL(D6)+IMREAL(D7)+IMREAL(D8)+IMREAL(D9))/6,(IMAGINARY(D4)+IMAGINARY(D5)+IMAGINARY(D6)+IMAGINARY(D7)+IMAGINARY(D8)+IMAGINARY(D9))/6)

For the maxmum and minimum, assuming you want the complex number with the max or min "magnitude", put the following formula on row 4 in a helper column, e.g. column E:

=IMABS(D4)

Copy down to row 9.

=INDEX(D4:D9,MATCH(MAX(E4:E9),E4:E9,0))

will now give the "maximum" number. Replace MAX by MIN to get the minimum number.

A Windows 8 user has provided an answer and it works in Windows 10 aswell.  Thanks go to  Robert Aldwinckle.

I had been exporting the WiFi profile without realising that Export & Show produce different results.

For the benefit of anybody else who refers to this thread:-

netsh wlan export profile name="ThisWiFi" folder= D:\Desktop

does not produce the same results as

netsh wlan show profiles name="ThisWiFi" >D:\Desktop\WiFiProfiles-ThisWiFi.txt

The results of the Show method include the profile's Metered property. 

It is reported as the "Cost" property and can have the value Fixed {meaning Metered} or Unrestricted {meaning non-Metered}

It can also have the value default or variable {but I have not looked into these two}

ken sheridan say: A column of Boolean (Yes/No) data type can only have a value of TRUE or FALSE, the latter being the default.  However, if you use a column of integer number data type and do not assign it a default value, then the default will be NULL.  By setting the column's Required property to True (Yes) NULL is prohibited, however, so the user must give the column a value.  You can bind a check box to the column, and it will be greyed out (Null) until the user has selected a value by clicking it.  If they don't, an error will be raised and they won't be able to save the record.  The only drawback is that they have to click it twice to make it FALSE. 

scottgem say:

Yes, This is like a questionnaire, which requires a very specific structure. You should have a table of questions. and another table for responses. There should be other tables, but I would need to know more about your app to tell what they should be.

So instead of a Y/N field in a table for each of those questions, you have a responses table like this:

tblresponses

ResponseID (PK Autonumber)

ParentID (FK)

QuestionID (FK)

Response

So each of those questions should be a record in a table, not a field in a record. If all the questions are Yes/No questions then Response can be a Y/N data type. You can then use the technique Ken suggested. Or you can use an Integer datatype and use an Option group to select No (0) or Yes (1).  For displaying as a checkbox, Access interprets 0 as False and anything else as True,
