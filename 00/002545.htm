mel77084 say:

There several 3rd party programs (some free) that can be used to make disk image type backups. Each of them will also make a bootable CD or thumb drive rescue medium to be used when one wants to restore a backup image. Restoring the image completely replaces whatever operating system (build) and installed programs existed in the OS partition.

The disk image backup that one makes is stored on a different physical drive, (either internal or external) so that, in case the hard drive or SSD that contains the OS partition fails, the backup image isn't lost in the failure. In such a case, since one has the bootable rescue medium, and the backup image is stored on a different drive, it's no big deal.

One simply removes the failed drive, connects a new drive in its place, boots with the rescue media with the drive containing the saved backup connected to the computer so that the rescue program will be able to see it, and restore the image onto the new drive. That restoration process make an OS partition the same size as the one that it originated from, and puts the contents of the image in it, so the operating system (build) and all the installed programs are ready to run exactly as they were when the image backup was made.

Note that this process does not depend on any functional OS to accomplish.

We often see posters in the forum saying "I wish I could go back ...." Well, this is the way to go back, if one has invested a few minutes of preparation to allow going back.

We also often see them saying "I lost all my data". Whatever data was in the OS partition will be preserved in the backup image; however, that said, the proper place for data is in another partition, where it can be redundantly backed up separately from the backup image (online as well as other means) depending on its importance according to the user's views on the matter. In the way, restoring to a previous build doesn't take one back to out-of-date data.

Example: I run an accounting program that generates data. That data is kept in a different partition and backed up redundantly. I can therefore restore to any older build, and my data is still current, and since the accounting program is present in the older build, I can go on using and building on the current data as though nothing had changed.

As I mentioned, there are several disk image backup programs available. The that I use is Macrium Reflect, which is free for personal use. Whichever one you choose to use, just remember, the rescue program media must be checked to verify that is bootable for your computer before you rely on it for use.
