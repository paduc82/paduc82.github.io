jay freedman say:

Macros that you save in the Normal.dotm template do meet the criteria you mentioned, and that's the default location for saving macros created by the recorder.

However, Normal.dotm is not the best place to save such macros. Because that template is a sort of scratch pad for all sorts of settings (styles and some of the options), and because it can be an easy target for malware, it's sometimes necessary to rename or delete Normal.dotm and let Word build a new one from the program defaults.

A better plan is to create a new empty template and store it in the folder %appdata%\Microsoft\Word\Startup. (The %appdata% environment variable expands to a location in your user profile that's normally hidden.) Every template stored in that folder is automatically loaded every time Word starts, and those templates are known as "global templates" or "add-ins". You can have as many separate add-ins as you need, or you can put multiple macros into one add-in (you should use modules within the template to organize the macros into groups).

One thing to be aware of is that the macros in a global template can't be displayed or edited in the macro editor -- if you click the project, you get a message saying "Project is unviewable". You can use File > Open to open the global template's file as a document, and that copy's project will be editable; if you make any changes, you'll have to close and reopen the Word program to reload the add-in.

Quick explanation

rob koch say:

Leonardo,

What typically has happened in such a case is that the malware was already present on your PC and may have been either hiding or somehow suppressing the operation of your real-time security application.

When the unrelated Windows update occurs, it often "breaks" the operating system's connection to the malware (BitCoin Miner in this case), since when the update installation fails, the roll-back is actually done via an installation repair of the previous versions of the Windows installation files stored in compressed protected folders on the PC.

Once the repair installation completes, if the malware had also placed hooks within other locations in order to insure its survival, it's actually reinstalled after the operating system reboots.  However, the repaired Windows installation and security programs can now "see" the malware, since whatever damage the BitCoin Miner had done in order to hide itself has been repaired as well.

This is why people often become confused in the belief that malware was installed by the Windows update, since the reinstallation of the malware is actually triggered by the update, or in your case roll-back repair install process, but the malware itself was already present before the installation processes that triggered it.

In fact, the most likely reason that the initial upgrade installation failed was the presence of the malware, since that might have been detected by the upgrade integrity processes, but since these rarely have the ability to inform you directly of the reason for the failure, it simply performed a roll-back due to update failure as designed.

Rob
