You need to set 32-bit version of Internet Explorer to your PATH.

  The 64-bit Internet Explorer resides in:

"C:\Program Files\Internet Explorer\iexplore.exe"

The 32-bit Internet Explorer resides in:

"C:\Program Files (x86)\Internet Explorer\iexplore.exe"

1. Go to Control Panel / System and Maintenance / System / Advanced

System settings.
2. "System Properties" window pops out.
3. Select "Advanced" tab.
4. Press "Environment Variables" button.
5. Look at "System variables" section and select variable "Path". The
click "Edit".

6. Edit variable value so that you add a string (without quotes!"

"C:\Program Files (x86)\Internet Explorer\iexplore.exe;"

to the beginning of the content of the field (i.e. the original string). Notice the semicolon (;) at the end of the string which separates different entries!

Example: If the original string was %SystemRoot%\system32;%SystemRoot%

Then now it should be:

C:\Program Files (x86)\Internet

Explorer;%SystemRoot%\system32;%SystemRoot%
7. Click OK several times and you are finished setting your path.
8. Restart your Windows/computer for changes to take effect.
9. When Windows starts, any application that wants to run iexplore.exe
(i.e. IE) will now run 32-bit version of IE.

Method 2:

You can easily go to the 32-bit IE and create a shortcut and send it directly to the start menu or quick launch, or make one and drag and drop it wherever you want it.

jeeped say:
Have to consider leaving the underlying value alone and using custom number formatting to achieve displayed out like 3M, 12M, etc?

Select the cells you want rounded then tap Ctrl+1 and go to the Numbers tab of the Format Cells dialog. Pick Custom from the list down the left side and use the following for Type: 

0,,"M"

The quoted "M" can be left off you prefer. The advantage to this method is that you do not lose the true underlying value and can use it in subsequent formulas and mathematical operations.

kevinpc06 say: My exhibits usually show to the first decimal place & I would use the custom format if it was just an exhibit with no totals or other calculations.  The custom format normally gives me a 0.1 difference if I try to foot the exhibit.

jeeped say:
That' loss of precision is going to occur whenever rounding is involved. Consider the following.

                

With 900,000 in A2:A7 and SUM(A2:A7) in A8, A8 appears to be showing the wrong total. In fact, it is showing the correct total for the underlying values according to the rules set out with the displayed values.

There are a couple of methods to accomplish your original request. One would be to put 1,000,000 into an unused cell then copy and select your range of numbers to be adjusted. Use Paste Special, Values, Divide to complete the scaling of your data.

A macro can easily accomplish this as well. Tap Alt+F11 and when the VBE opens, immediately use the pull-down menus to Insert, Module. Paste the following into the new pane titled something like Book1 - Module1 (Code), 

Sub Adjust_Numbers()
    Dim r As Range
    For Each r In Selection
        If IsNumeric(r) Then _
            r = CDbl(Round(r / 1000000, 1))
    Next r
End Sub

Tap Alt+Q to return to your worksheet. Tap Alt+F8 to open the Macros dialog and use Options to assign a keyboard shortcut. Tap Esc to close the Macros dialog. Select a range of numbers and tap the keyboard shortcut you've assigned.

You might also want to explore assigning the macro to the QAT (Quick Access Toolbar) for easy access.

Sub Adjust_1M_Numbers()
