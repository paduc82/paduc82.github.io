greginmich say:

The MAPS connection test seems to be working fine – so you’ll have to figure out what’s wrong on your end. Note that if you run this command from the user Command Prompt:

“C:\Program Files\Windows Defender\MpCmdRun.exe”  -ValidateMapsConnection

You’ll get the 80070005 error:

 

But the command should work fine when it’s run from the Administrator Command Prompt. Type “cmd” in the search box; right-click on Command Prompt, and select Run as administrator:

 

The validatecloud.exe test file is also being properly detected by both Windows Defender Smartscreen, and by Windows Defender Antivirus (when SmartScreen is disabled) – however, it’s being detected as Trojan:Win32/Occamy.C on this PC.

Running some further detection tests with these test files might help you figure out why you’re not seeing a Windows Defender detection on your PC:

The eicar.com test file

http://www.eicar.org/85-0-Download.html

The AMTSO Settings Check for Desktop Solutions

https://www.amtso.org/security-features-check/

The Windows Defender TestGround site

https://demo.wd.microsoft.com/

The only issue that I presently see with the documentation for cloud-delivered protection is the PowerShell command that’s provided for the SubmitSamplesConsent parameter in this related article:

 

https://docs.microsoft.com/en-us/windows/security/threat-protection/windows-defender-antivirus/enable-cloud-protection-windows-defender-antivirus

 

Originally, this article provided this command:

 

Set-MpPreference -SubmitSamplesConsent 3

 

Which wasn’t being accepted by the command interpreter – so I filed this bug report for the issue at the Feedback Hub:

 

https://aka.ms/AA1ffyg

 

In the meantime, the original command was replaced with this one in the article: 

 

Set-MpPreference -SubmitSamplesConsent Always

 

It looks like they’ve now managed to fix the original issue with the value of 3 (without ever acknowledging it), since the interpreter now accepts this value without any problem – but it also looks like they’ve managed to break the command that replaced the original, which now turns off Automatic Sample Submission, instead of turning it on:

 

GreginMich
