how to make an .exe file of ms access - Microsoft Community

mr icy say:

how to make an .exe file of ms access

hello friends

is it possible to make an .exe file of the ms access 2003

 

thankx

hansv mvp say:

No, not really, but there are some alternatives:

You can convert the database to an .mde database. Users can't modify the design of forms and reports or view code in an .mde. They still need Access to use the database.

or

Create your own application (.exe) in one of the Visual Studio products that uses the tables in the Access database to store the data. You'd have to create all forms in the new application.

mr icy say:

ok but after you convert it to .mde file then i don't think you can re-open in your normal .mdb file if yes then how to do that.

 

 

 

hansv mvp say:

If you convert an .mdb to .mde, you must always keep a copy of the original .mdb. You need this if you ever need to change the design of forms or reports, or to edit VBA code.

This works best with a split frontend/backend setup: the frontend with the forms and reports (and linked tables) is a .mde, and the backend with the tables is a .mdb. If you need to modify the design of forms or reports, you only need to replace the frontend with a new .mde. The backend .mdb can remain the same.

mr icy say:

that's ok but what if i have thousands of records in the .mde file so how to transfer them from .mde file to a copy file

 

hansv mvp say:

Split the database into a frontend and backend before converting the frontend to an .mde. Access can do this automatically for you: select Tools | Database Utilities | Database Splitter and follow the instructions.

If you prefer to do it manually, see http://support.microsoft.com/kb/304932.
