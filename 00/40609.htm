¡firedog say:
when I ... click the "to" button, the contact list is blank.


This is usually a symptom of empty email address fields. Only contacts with an email address will be shown in the picker list.

The Outlook contacts database uses (IIRC) more than 80 contacts fields. The WLMail database has twice as many, but only 20-odd are accessible through the UI. Five of those are email addresses: Personal, Work, Other, Messenger and Other Messenger. Could it be that when you imported from the CSV [sic] file, the email addresses got mapped to the wrong fields? You can check whether the email address fields are populated in the contacts window (press Ctrl-3 to switch). Select View as > List on the ribbon, then right-click on column headings to add, remove and change the columns displayed.

If you find that the email address fields are empty, it would be best to conduct the import again. I would do this at http://people.live.com, which understands CSV files from Outlook a bit better than WLMail does. Close down WLMail meanwhile. Delete the existing contacts (except any you may have added since the CSV file was created). This should ensure that Outlook fields are correctly mapped to Windows Live Contacts fields. Then launch WLMail; if you're signed in, the new contacts list should be synchronized.

 

Note that you will have at least two separate contacts lists; one you see when you are not signed in (the default list) and one for each Microsoft Account you sign in with.

You'll find some tips for using contacts in WLMail on this page: Address messages.

davem121 say:
Hi that can be done in Windows 10, but requires a registry tweak . . .

Click your Start Button, type regedit and hit Enter to open the Registry Editor

Navigate to:
HKEY_CURRENT_USER\Software\Microsoft\ Windows\CurrentVersion\Explorer\Advanced

On the right-side, look for an entry named JumpListItems_Maximum.

If there is no such entry, you need to create one by right-clicking on the empty area, clicking New, clicking DWORD (32-bit) Value, and name it as JumpListItems_Maximum

Double click that DWORD and set its value to 20 or 30

Close the Registry Editor and restart your system

Hi Bob,

Ctfmon or CTF Loader or a Microsoft process that controls Alternative User Input and the Office Language bar. It's how you can control the computer via speech or a pen tablet, or using the onscreen keyboard inputs for Asian languages. To shut it down, disable the Touch Keyboard and Handwriting Panel Service from Services.

Here's how:

  1. Press Windows+R, type services.msc, and then click OK.
  2. Scroll through the list, right-click on Touch Keyboard and Handwriting Panel Service, and then select Stop.

To enable it, do the same steps but select Start.

You can use the same thing in Tablet mode, using either the OSK or (full) TK.   Personally I think using the Taskbar is easier, e.g. using my pen and waiting for Aero Preview to show me what each window is.   It is especially helpful in showing me my IE tabs and much faster than pressing Ctrl-Tab.

 

Notice that the OSK has a feature called Nav?   Unfortunately, it resets any modifiers that were set by Gen.   Otherwise, it might be nice to click an Alt- then Nav and tap Tab repeatedly.   Ditto for Win-  You can see that this would work by using those real keys.   Another feature of the Nav bar which has me mystified is the F6.   What apps is it good for?   It might have once been useful in IE but really seems like a poor choice.   E.g. being able to click Win- or click and release it would have given us a lot more usability and helped users avoid problematic swiping from the sides.

 

Another possibility would be to have the Task Manager window in its Fewer Details state but again I think I would prefer to use the Taskbar, e.g. unless you had difficulty identifying the icons or for some reason wanted to have a vertical list somewhere other than where you had your Taskbar.
