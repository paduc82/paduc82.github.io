Hi Darren,

Had a last look and found something that can help.

First, remove the circular reference in sheet2 ,cell A1.

Second, change the calculation of the monthe headers.

EOMONTH(F46,1) returns 05/31/2014 but the months in the raw data are 05/01/2015.

Change it to: =EOMONTH($F$10;COLUMN()-COLUMN($F$1)-1)+1

That will return 05/01/2014

The final formula in F47:

=SUMIFS(INDIRECT("'[Niguri Raw Data Test.xlsx]Niguri Raw Data'!"&ADDRESS(2,MATCH(F$46,'[Niguri Raw Data Test.xlsx]Niguri Raw Data'!$A$1:$AN$1,0))&":" & ADDRESS(50000,MATCH(F$46,'[Niguri Raw Data Test.xlsx]Niguri Raw Data'!$A$1:$AN$1,0))),'[Niguri Raw Data Test.xlsx]Niguri Raw Data'!$B$2:$B$50000,$B$46,'[Niguri Raw Data Test.xlsx]Niguri Raw Data'!$D$2:$D$50000,">="&"2014",'[Niguri Raw Data Test.xlsx]Niguri Raw Data'!$I$2:$I$50000,$E49,'[Niguri Raw Data Test.xlsx]Niguri Raw Data'!$A$2:$A$50000,$B$5)

Note that you have to change the ranges in your original formula, see the bold. You can extend the ranges to e.g. $A$2:$A$100000

'[Raw Data Test.xlsx]Raw Data'!$J:$J

It has been replaced by:

INDIRECT("'[Niguri Raw Data Test.xlsx]Niguri Raw Data'!"&ADDRESS(2,MATCH(F$46,'[Niguri Raw Data Test.xlsx]Niguri Raw Data'!$A$1:$AN$1,0))&":" & ADDRESS(50000,MATCH(F$46,'[Niguri Raw Data Test.xlsx]Niguri Raw Data'!$A$1:$AN$1,0)))

What it does.

MATCH(F$46,'[Niguri Raw Data Test.xlsx]Niguri Raw Data'!$A$1:$AN$1,0) looks up the date in F46 and returns the column number.

With ADDRESS(2,MATCH(F$46,'[Niguri Raw Data Test.xlsx]Niguri Raw Data'!$A$1:$AN$1,0)) I am getting a cell reference e.g. $J$2. I repeat this to get the address of $J$50000 (the reason why you to adapt all ranges.)

"'[Niguri Raw Data Test.xlsx]Niguri Raw Data'!"&ADDRESS(2,MATCH(F$46,'[Niguri Raw Data Test.xlsx]Niguri Raw Data'!$A$1:$AN$1,0))&":" & ADDRESS(50000,MATCH(F$46,'[Niguri Raw Data Test.xlsx]Niguri Raw Data'!$A$1:$AN$1,0))
