At the first look I can not see any uncommon things, but there are two reasons why such macros did not work under different situations.

a.) Never use SELECT / SELECTION / ACTIVECELL in your macros! Your macro gets slow and error prone.

b.) The biggest mistake is to switch the errors off "On Error Resume Next" and never switch them on again.

And your macro does some superfluous actions, e.g. you copy the whole column D with the formats, but then you delete the header and remove the formats...

We can do all this in one step:

  With Sheets("Master Part List")
    .Range("D2", .Range("D" & Rows.Count).End(xlUp)).Copy
  End With
  Range("A1").PasteSpecial xlPasteValues

Did you know how to use the WITH instruction? If not, please read the help for this.

BTW, you have declared "UserConf As String", but a MsgBox returns not a string, vbYes is a VbMsgBoxResult (an Integer).

We are in VBA, so why should we use a formula to find a match? Use the FIND method of the RANGE object instead.

I don't want be too critical, your macro looks like mine 15 years ago. ;-)

If you like, I can move it up a few levels, we can do the same thing hundered times faster and without using a temp sheet.

I'm not sure which kind of data you have in column C sheet "Status Tracking", maybe you get an error on the line

  'On Error GoTo ErrorHandler

Option Explicit

Sub Consolidated_TN()
  Const Title = "Consolidated_TN"
  Dim UserConf As VbMsgBoxResult
  Dim C As Range
  Dim Data
  Dim Value As Double
  Dim i As Long, DataRow As Long
  Dim Dict As Object 'Dictionary

  UserConf = MsgBox("UPDATE STATUS TRACKING SHEET" & vbCrLf & vbCrLf & _
    "Do you want to continue?", vbYesNo, Title)
  If UserConf <> vbYes Then
    MsgBox "Update Status Tracking Sheet Cancelled!", , Title
    Exit Sub
  End If
 
  'Install an error handler to catch unexpected results
  'On Error GoTo ErrorHandler
 
  'Read the items into an array
  With Sheets("Master Part List")
    Data = .Range("D2", .Range("D" & Rows.Count).End(xlUp))
  End With
 
  'Delete the duplicates
  Set Dict = CreateObject("Scripting.Dictionary")
  Dict.CompareMode = vbTextCompare
  For i = 1 To UBound(Data)
    'Found before?
    If Not Dict.Exists(Data(i, 1)) Then
      'Increment our row number
      DataRow = DataRow + 1
      'Remember this item
      Dict.Add Data(i, 1), 0
     
      'Get the match from Status Tracking
      Set C = Sheets("Status Tracking").Columns("A").Find( _
        Data(i, 1), LookIn:=xlValues, LookAt:=xlWhole)
      'Found?
      If Not C Is Nothing Then
        'Get the value from column C
        Value = C.Offset(0, 2)
        'Delete existing Tracking Number?
        If Value = 0 Then
          DataRow = DataRow - 1
          GoTo Skip
        End If
      End If
     
      If i > DataRow Then
        'We need to copy the item up
        Data(DataRow, 1) = Data(i, 1)
      End If
Skip:
    End If
  Next
 
  'Update Tracking list
  With Sheets("Status Tracking")
    'Get the next empty cell
    Set C = .Range("A" & Rows.Count).End(xlUp).Offset(1)
    'Store the related part of the array
    C.Resize(DataRow, 1) = Data
    'Remove "Tracking Number"
    .Columns("A").Replace "Tracking Number", "", xlPart
  End With
 
  MsgBox "Status Tracking Sheet has been Updated!", , Title
  Exit Sub
 
ErrorHandler:
  MsgBox "Error " & Err.Number & ": " & Err.Description
End Sub

Excel formula for number of years, months, and days - Microsoft Community

augustineb say:

Excel formula for number of years, months, and days

Calculating the years, months, and days from someone's birthdate is an old question.  Every solution I have seen on the net is wrong on certain dates, even DATEDIF.  In an April 1997 document from Microsoft Support, a formula was given, but was not more accurate than DATEFIF.  I modified the formula which is listed below.  Microsoft Support did an excellent job of creating this formula that was modified by me (slightly).  With a birthdate in cell a2:

IF(OR(A2>TODAY(),ISBLANK(A2)),"",IF(YEAR(TODAY())=YEAR(A2),0,IF(MONTH(TODAY())>=MONTH(A2),YEAR(TODAY())-YEAR(A2),YEAR(TODAY())-YEAR(A2)-1))&" years "&MONTH(TODAY())-MONTH(A2)+IF(AND(MONTH(TODAY())<=MONTH(A2),DAY(TODAY())<DAY(A2)),11,IF(AND(MONTH(TODAY())<MONTH(A2),DAY(TODAY())>=DAY(A2)),12,IF(AND(MONTH(TODAY())>MONTH(A2),DAY(TODAY())<DAY(A2)),-1)))&" months "&IF(DAY(TODAY())>=DAY(A2),DAY(TODAY())-DAY(A2),DAY(EOMONTH(A2,0))-DAY(A2)+DAY(TODAY()))&" days")

If cell A2 is blank or has a date greater than the current date, TODAY(), the formulas will display a null value.

Change Word Outline Numbering via VBA (Word 2010) - Microsoft Community

susanjhacker say:

Change Word Outline Numbering via VBA (Word 2010)

I autoformat tables based on options that the user selects in a UserForm that pops up when the user runs the macro.  Some rows in the table are are just text, and some rows are outline numbered headings.  Sometimes the headings have manual numbering, and sometimes they don't.  One of the options I would like to provide to the users is to turn on or off the outline autonumbering in the Word document if the heading rows have manual numbering. 

It used to be easy to do before Word 2010 - I could turn Outline Numbering on or off for all the heading styles just by changing Heading 1 because all my other heading styles are based on Heading 1.  But with Word 2010, which has the new List Style Gallery for Outline Numbering, I am having a heck of a time figuring out how to apply (or remove) outline numbering from all 9 heading styles using VBA, and there doesn't seem to be much discussion about how to do this anywhere.  I have a named multilevel numbered list style in the gallery called "Headings" (as suggested by Shauna Kelly in her very informative tutorial about setting up list styles), but as far as I can figure out there isn't a way to apply a named list style to the selection using VBA.  Even if I could, I don't think the change would propagate to the remaining heading styles so that all the headings in the document inherit the numbering scheme. 

Anybody have any ideas?

Thanks

Susan

susanjhacker say:

While Charles' method held some promise, I wanted to thoroughly explore the List Gallery idea before giving it up all together.  I spoke to a collegue of mine who is better versed in VBA than I am (he does Excel macros more than Word macros, but really knows his wa around VBA).  Anyway, we came up with this code, which works perfectly.

Dim myLT as ListTemplate

If DoAutoNum = True Then
    Set myLT = Application.ListGalleries.Item(wdOutlineNumberGallery).ListTemplates(5)
    With Selection.Find
        .Style = ActiveDocument.Styles("Heading 1")
        .Execute
    End With
    Selection.Range.ListFormat.ApplyListTemplate ListTemplate:=myLT
End If

DoAutoNum is a variable that gets set to true if the user checks a box on the user form indicating that he wants to autonumber the document.  ListTemplates(5) is the built-in list template that has the outline numbering I wanted to use.  I took a SWAG about the number (it was the 5th one in the list), but it seems to be the correct number.   Fortunately I didn't need any kind of fancy numbering where I would need a custom outline numbered list style, because I never did figure out how to apply a named list style.

By default, the heading styles in my template are not autonumbered.  Since all my heading styles are based on Heading 1, applying the list template to the Heading 1 Style cascades to all my other styles too, which is exactly how I wanted it to work.  So, if the headings in the table have manual numbers before formatting, DoAutoNum is set to False, and this code never runs.  If the headings have no numbers, the user sets DoAutoNum to true and this code runs and autonumbers everything.  Works great.

Registry back-up for Windows 8:

  1. Press the Windows + R keys to open the Run dialog box.
  2. Type in the following text regedit, and then press Enter.
  3. If the User Account Control window appears, click Continue.
  4. On the File menu, click Export.
