jllatham say:

Liz, add this code to your workbook and run it and see if it finds the mystery sheet.  It looks at both the .Name (what you see on a sheet's name tab) and the .CodeName (Excel's internal VBA reference to the sheet), and if any contain Sheet005, it should tell you both names and if the sheet is hidden in any way, make it visible.

 

If it identifies a sheet somewhere, you should be able to then manually delete it.  But if something is really strange and it says it found it, but you can't see/find it afterwards, we could add a line to the code to ask if you want to delete it or not and do so if you desire.

 

Sub FindMysterySheet()
  Const lookFor = "SHEET005" ' all uppercase
  Dim anyWS As Worksheet
  Dim foundFlag As Boolean
 
  For Each anyWS In ThisWorkbook.Worksheets
    If InStr(UCase(anyWS.Name), lookFor) > 0 Then
      MsgBox "Found sheet with tab name: " & anyWS.Name & vbCrLf _
       & "CodeName: " & anyWS.CodeName, vbOKOnly, "Found by .NAME"
      anyWS.Visible = xlSheetVisible
      foundFlag = True
    End If
    'test the CodeNames
    If InStr(UCase(anyWS.CodeName), lookFor) > 0 Then
      MsgBox "Found sheet with tab name: " & anyWS.Name & vbCrLf _
       & "CodeName: " & anyWS.CodeName, vbOKOnly, "Found by .CodeName"
      anyWS.Visible = xlSheetVisible
      foundFlag = True
    End If
  Next
  If Not foundFlag Then
    MsgBox "Did not find the phantom sheet."
  End If
End Sub

beth melton say:

Do you have Word running when you try to paste or are you starting Word after you copy information from your email? If the issue only occurs when Word isn't running then more than likely you have a third-party add-in that clears your Windows Clipboard each time Word starts. To view your add-ins, click File/Options/Add-ins. If you have several listed, try disabling them one at a time in order to determine which one is causing the issue. Note that add-ins that have "Action" in the Type column are fine and you don't need to disable them since they are shipped with Office 2010 and aren't considered third-party addi-ins.  

If you're not sure how to disable an add-in, please provide us a list of the add-ins that appear in the Active Application Add-ins section along with the Type of add-in (e.g. Template, COM, etc.) and we'll provide instructions for how to disable them.

~Beth Melton Microsoft Office MVP

herb tyson say:

That can be a sign of corruption in a registry entry, Normal.dotm, Word's immediate memory space, or Windows' memory space. The latter two can be cleared up by closing/restarting Word and Windows, respectively. Or, it can mean that an add-in is tripping you up.

For starters, I would try rebooting Windows to see if that fixes the problem. If not, I would next open Word in Safe mode (click Start, and type winword /safe; then press Enter). Press Ctrl+N to start a new document, and see if you now can paste from Email okay. If you can, then the next step would be to try deleting the Data key in the registry and renaming Normal.dotm. This article describes, among other things, how to delete the Data key:

http://support.microsoft.com/kb/921541

If Safe mode doesn't make it work, my next stab would be at disabling add-ins.
