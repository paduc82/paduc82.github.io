ken sheridan say: It doesn't seem that there is anything we can really do to help you.  If both sets of tables have been updated independently of each other and you cannot establish any firm criteria which can logically be applied to determine how conflicts in the data are to be reconciled, then you appear to have no choice but to manually update the tables on the server on the basis of your own judgement.

If criteria for resolving conflicts can be established, and these are capable of logical definition, then you might be able to update the tables on the server by means of 'update' queries which join the tables from each set, but it is unlikely to be a trivial task, or within the scope of an inexperienced Access user.
