Another method is to use the filesystemobject (FSO).  You can use this:

Sub ListFiles()
' Set reference to 'Microsoft Scripting Runtime'.
'
    Dim fso, fso_folder, fso_file
    Dim sTargetFolder As String
    Dim i As Long
   
    'INPUTS:
    sTargetFolder = "C:\testing\"                         'Enter folder path here
   
    'PROCESS:
    Set fso = CreateObject("Scripting.FileSystemObject")
    Set fso_folder = fso.GetFolder(sTargetFolder)         'Set the FSO Folder (directory) object.
    
    i = 1

    For Each fso_file In fso_folder.Files

        'YOUR CODE HERE

        'SEE http://msdn.microsoft.com/en-us/library/1ft05taf(v=vs.85).aspx  FOR A LIST OF 
        'PROPERTIES FOR THE FILE OBJECT.
        'SEE http://www.techbookreport.com/tutorials/fso1.html  FOR MORE EXAMPLES.
        
        'List files in active workbook
        Range("A" & i).Value = fso_file.Name         'Just the file name+ext, e.g. "data.txt"
        i = i + 1

    Next fso_file
End Sub

>Please navigate to Network admin =>Security Settings (under content and security)

>Please check box the option :-

Enforce your Exchange Online Exchange Transport Rules (ETRs) in Yammer (Last synced at: 14 hours ago)

>Under "Network", Please click on External Network and check box the two options :- 

Disable the Related External Networks directory. 

Disable the Our External Networks directory and remove the External Networks link in the networks menu.

swischner say:

Simple answer to this is: OneNote is not a PDF reader nor a document management program.

It can not interpret any formats (like links, form fields, indexes..) inside a PDF when inserted as a printout. A PDF printout is exactly what the name says. Imagine you print out the PDF, put the printout on a scanner and place the resulting picture(!) in a OneNote page.

That you can make that picture searchable again is due to the OCR mechanisms in Windows / OneNote and has nothing to do with the original index in the PDF.
