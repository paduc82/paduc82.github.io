hansv mvp say:

That looks like an Access question, not an Excel question - you can edit your first post and change the application.

You can use code like the following to return to the correct record after a requery.

Assuming that Pos_Num is a number field

 

    Dim lngPosNum As Long
    lngPosNum = Me.Pos_Num
    Me.Requery
    With Me.RecordsetClone
        .FindFirst "Pos_Num = " & lngPosNum
        Me.Bookmark = .Bookmark
    End With

 

If Pos_Num is a text field:

 

    Dim strPosNum As String
    strPosNum = Me.Pos_Num
    Me.Requery
    With Me.RecordsetClone
        .FindFirst "Pos_Num = " & Chr(34) & strPosNum & Chr(34)
        Me.Bookmark = .Bookmark
    End With

bunyan horton say:

Keep in mind Outlook Archive feature and Online Archive Exchange are different.

AutoArchive is an older Outlook feature that allows automatic moving of old mails to a .pst file. This means the mails are removed from the server and are harder to find through search. If the hard drive containing the AutoArchive.pst becomes unusable, these mails are lost. Online Archive is an enterprise-grade replacement for AutoArchive. AutoArchive can still be used if you don’t have an Online Archive.

Using the Archive button to move messages to the Archive folder doesn't reduce your mailbox size. If you need to reduce your mailbox size, you can use Online Archive in Office 365 for Enterprise.
