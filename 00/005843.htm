stefan blom say:

Unformatted AutoCorrect entries are language-specific, so check if the language has changed at the insertion point while typing. In Word, language is a text property, meaning that it can apply to a range as small as a single character.

You can tell which language is applied at the insertion point or to the current selection in the Language dialog box (Review | Language | Set Proofing Language). While you have the dialog box open, be sure to clear the option to "Detect language automatically" to prevent the problem from returning.

To fix the existing document, you have to select all text and (re-)apply the correct language via the Language dialog box.

The "jumping cursor" problem is usually caused by hardware, such as on over-sensitive touchpad if you are using a laptop. Try disabling the touchpad temporarily and see if that improves the situation.

charles kenyon say:

Cursor thing: may be related to sensitivity settings for a mousepad. This is usually the cause. Try using an actual mouse and turning off the mouse pad. What happens? Perhaps you need a new driver for Win 10.

AutoCorrect and AutoText sound similar and perform similar tasks but are configured very differently. I'm surprised you were able to transfer your AutoCorrect completely without some utility to assist with this. AutoCorrect entries are stored in special .acl files and in Normal.dotm. AutoText is stored in templates. It may be stored in the Normal.dotm template, but can be elsewhere as can any of the other building blocks. The following article discusses some of the differences and where these entries can be stored as well as where they are stored by default.

Automated Boilerplate Using Microsoft Word

Greg Maxey has a utility for backing up AutoCorrect entries and another for managing building blocks including AutoText. These are free downloads.

  • AutoCorrect Utility Manager
  • Building Blocks & AutoText

I generally store as much as I can in templates that are independent of the normal template, but in Word 2010, AutoText stored in other global templates does not pop up as a suggestion. (It does if stored in the attached template, though.) Storing them elsewhere makes it easier to transfer them to another computer. Greg's tools will work fine if you keep them in the normal template.

Sorry but my French would make you laugh!

Try redefining fonts and sizes of the different styles through Format > Styles. Click on a style name and then on the Modify button. The first style to try changing is Normal, the rest. Use the "Styles in use" choice below the list, so Word only shows you the styles that you need to change.

If that does not completely work and you need additional modifications, it means that the original document has had manual formatting applied, and you will have to do the changes manually. However, you can do global replacements of fonts and sizes into other fonts and sizes. To do this as a global change, use Advanced Find and Replace (access it through the "gear" icon in the Find sidebar) and use the dropdown menu at the bottom.
