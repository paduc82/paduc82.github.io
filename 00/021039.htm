suzanne s. barnhill say:

Here's how you do it:

  1. Type the heading using the Heading 3 style.
  2. Press Enter to create a new paragraph. By default it will be in Normal style, but it will be in whatever style is defined as the "Style for following paragraph" in the Heading 3 style.
  3. In that new paragraph, type the rest of the text.
  4. Click in the heading and press Alt+Ctrl+Enter to insert a style separator. This works very much like the hidden paragraph mark but doesn't require you to hide Hidden text in order to see the result. Instead you'll see a paragraph mark surrounded by a dotted box.

barkoosh say:

Does this happen only when the closing parentheses and brackets are at the end of the cell? If yes, this occurs because you're using a non-Arabic version of Office. I have never found solutions for such problems, but there are some workarounds that could work. Choose the easier for you:

A. Workaround with the Excel file:

1. If a row DOES NOT contain an English closing parenthesis/bracket at the end of a cell:
Go to every Arabic closing parenthesis/bracket that exists at the end of a cell and add after it a Right-to-Left Mark from Insert > Symbol > Special Characters tab.

2. If a row DOES contain an English closing parenthesis/bracket at the end of a cell:
For some reason this addition of the Mark in one cell affects other cells in case of mail merging (which is quite odd). Here you should add a Right-to-Left Mark AND a Left-to-Right Mark (same path) after the Arabic closing parenthesis/bracket that exists at the end of a cell. This way the RTL Mark won't affect the English closing parentheses/brackets in the other cells of the row.

B. Workaround with the Word document:
- Complete the mail merge (Mailings > Finish & Merge > Edit individual documents)
- Open the Find and Replace window with CTRL+H. In "Find what:" type (or copy-paste) this:
[\)\]]
- Click More. Check "Use wildcards".
- Click Format > Language and double click the English language you're using. It will appear under the "Find what:" field 
- Now click inside the "Replace with:" field, keep it blank, but go to Format > Language and double click Arabic UAE. It will appear under the "Replace with:" field.

1. If you're sure that the document doesn't contain English closing parentheses/brackets, click Replace All.

2. If the document does contain English closing parentheses/brackets, only Replace the Arabic closing parentheses/brackets.

There could be some easier workarounds but that depends on the formats of the text in the mail-merge document.
