rohn007 say:

I have to agree with Doug.  Editing Office documents on USB drives is generally a bad idea!  It depends on the drive type and some Windows customization options you probably have not set.

The problem is the default way Windows (not Office apps) handles USB files. There are 2 problems. One is the limited "life" of Flash type devices, the other problem is the workaround Windows uses to help save you from yourself.

Actually it depends on the specific storage medium. "Flash" type storage (most "Thumb" type drives) has a limited number of "write" cycles. The way that Office apps, like Word, handle file editing includes a LOT of writing to storage during editing. When you edit an Office file, the application opens a couple of "temp" files in the same directory as the file is in. It does a LOT of reading and writing to these temp files during your editing session.  This "uses up" the limited life of the flash drive.

Knowing this, Windows is aware of the device type, so it creates a temp "Cache" storage location on the local drive of the computer. When you save files to the flash drive, Windows actually writes to the cache. Then when specific criteria are met, the cache is "flushed" to the actual Flash drive. It reduces the number of saves/writes to the flash drive.

So, the problem with Editing files directly from flash drives is the Cache. If you do not properly shut down the flash drive before unplugging it you can leave part of the file (in a DOCX file that is most likely the actual body text) behind on the computer, corrupting the file.  There is an option to disable the cache in Windows, but that will use up your flash drive life much much faster.

The workaround I've used for years is "batch" files.  DOS, or now PowerShell, command files.  I use 4 command files:

  • copy from flash to work computer
  • copy from work computer to flash
  • copy from flash to home computer
  • copy from home computer to flash

(actually I've combined the 4 files into one, and just use 4 separate "shortcuts" that pass a parameter to the one command file to identify which specific subset of commands to use.

So when I get to work, I copy my commonly used files (in particular my "Favorites/bookmarks") to the work computer.

At the end of the day I copy them all back. Actually, I check the "archive" flag and only copy changed files.

rohn007 say:

As I tried to explain, "external drives" can refer to many different types of underlying storage media technologies.

Going in most commonly available sizes (largest to smallest):

Yes, there are external "spinning disk" magnetic storage hard drives that are essentially the same as built in magnetic hard drives. The main difference is they work through a slightly slower connection / "pipeline" than the built in ones.  As far as I know Windows does NOT use the "cache" technique on this type of drive. The largest external spinning disk drives I've seen are 8TB

Recently, the last year or so, I have seen external SSD drives. These are larger than typical "flash" or "thumb" drives. They also do have a limited number of write cycles like flash/thumb drives. I suspect they do not use a cache file (to maintain the fast write speed), but I have not seen documentation that explicitly has said that.  SSD drives have a built in techniques to limit wear (including a build in amount of "unused" storage that gets swapped in as specific cells die).

Then there are the common and various "flash" and thumb devices. They started out small, have become relatively large. I've seen this type of drive up to 500GB.

Next there are various "SD" type devices. Again, typically small, but today I found a 2TB one, ie:

https://www.bestbuy.ca/en-ca/product/lacie-dji-copilot-2tb-2-5-usb-c-usb-3-0-lightning-sd-microsd-portable-external-hard-drive/12370566.aspx?

Finally there are the old "floppy" disks (if you can find them anywhere) that were the original external storage.

OK, in the 1970s tape drives on small "kit" computers came before floppy drives, but you'd have to be a real (old) geek to remember them or have used them ... <g>

Sub TOCLevel1()
'
' TOCLevel1 Macro
' Insert Table of Contents showing only level 1 styles.
' posted by Charles Kenyon
    With ActiveDocument
        .TablesOfContents.Add Range:=Selection.Range, _
            RightAlignPageNumbers:= _
            True, UseHeadingStyles:=True, _
            UpperHeadingLevel:=1, _
            LowerHeadingLevel:=1, _
            IncludePageNumbers:=True, _
            AddedStyles:="", _
            UseHyperlinks:=True, _
            HidePageNumbersInWeb:=True, _
            UseOutlineLevels:=True
    End With

jac.chr say:

If it can be of use for anybody else I managed to solve the challenge with the following code:

    Dim oPara      As Paragraph
    Dim oHeadings  As Variant
    Dim i          As Integer
   
    'Define cross reference to heading objects
    oHeadings = ActiveDocument.GetCrossReferenceItems(wdRefTypeHeading)
   
    'Loop through each paragraph
    For Each oPara In ActiveDocument.Paragraphs
   
        'Check if 'Heading 1' style is applied to current paragraph
        If oPara.Style = "Heading 1" Then
       
            'Loop through all heading cross references
            For i = 1 To UBound(oHeadings)
           
                'Check if text in current paragraph equals text in current heading reference
                If Trim(Replace(oPara.Range.Text, vbCr, "")) = Trim(oHeadings(i)) Then
               
                    'Insert cross reference page number
                    Selection.InsertCrossReference ReferenceType:=wdRefTypeHeading, _
                                                   ReferenceKind:=wdPageNumber, _
                                                   ReferenceItem:=i
                                                  
                    Selection.TypeText " "
                   
                    'Insert cross reference text
                    Selection.InsertCrossReference ReferenceType:=wdRefTypeHeading, _
                                                   ReferenceKind:=wdContentText, _
                                                   ReferenceItem:=i
                                                  
                    Selection.TypeParagraph
               
                End If
           
            Next i
       
        End If
   
    Next oPara
   
cleanUp:

    Set oHeadings = Nothing

Then it is just a matter of applying the relevant formatting, e.g. adding the cross references in a table.

One note though is that cross references use hidden bookmarks - and like visible bookmarks adding text at the end of the paragraph will not be included in the bookmark.
