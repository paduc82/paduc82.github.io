theuncodedone say:

Hello! 

In case you don't want to use bookmarks, you can do a simpler animation that provides you much more control over the entry and exit of the text. 

  1. After putting up the video on a slide, go to the Animations tab on the top menu bar. 
  2. Click on Animation Pane and make sure that the video is set to start playing automatically "With Previous".
  3. Add a text block on the slide.
  4. Go back to the Home tab and click on the Arrange button, and go right down in the menu and click on Selection Pane for a detailed list of items on your slide.
  5. As you add different text boxes, just rename them in the Selection Pane with the text that you are actually entering into them. This makes it easier for you to arrange them. 
  6. Always ensure that you video is set as the last object in the Selection Pane, since this means that it will always be the object right at the back.
  7. After you have added all the lyrics in text boxes and renamed them in the Selection Pane, go on to the Animations tab. 
  8. Click Ctrl+A to select all the items in the slide and deselect only the video.
  9. In the Animations tab, add an Appear animation to all the objects.
  10. In the animation timings part of the Animations tab, set the Start to be "With Previous" for all of the text boxes, except the first one.
  11. After this add a "Disappear" exit animation for all of the text boxes.
  12. In the Animations Pane, make sure that the Disappear animation appears below each of the Appear animation for each of the text boxes.
  13. Select all of the Disappear animations and set them to animate "On Click". 

There you have it. A video background, and text in the front that you can bring on and take off as you want. 

I hope this satisfies your requirement.

brian tillman [outlook mvp 2007-2019] say:

Each person who sends a message to a mailbox with Automatic Reply enabled should get exactly one reply until the option is disabled and then reenabled on the receiving mailbox.  This is true for all versions of Outlook using Exchange.  I thought I had read somewhere that a sender could get a reply once every 24 hours, but I'm not sure that's true.  As far as I know, this is not adjustable.

Using a rule and not the Out of Office Assistant (Automatic Reply in Outlook 2010) to automatically reply may produce a reply each time message arrives (I haven't tried it and I wasn't able to find documentation on it), but only if you use Exchange.  When using Outlook with, say, a POP account and an autoreply rule, Outlook should remember the addresses to which it sends that reply and reply only once.  Of course, Outlook must remain running in that case.

trevor rabey say:

OK.

First of all, this is not forward scheduling ASAP, even if that is the settings adopted in project, project information.

Also, just because there is a deadline date for the project finish does not mean that you have to, or should, schedule backwards from that finish date. Actually, you should not.

It's the predecessor on task 3 that is forcing task 3 to finish no earlier than the start of task 4, ie at, or rather no earlier than 08:00 on the 5th, and since there is no working time earlier than 08:00 on the 5th all of the 2 days working time in task 3 is on the 3rd and 4th.

This is sensible, and how it works. But I can see why you want task 3 to finish at 17:00 on the 4th.

However, it won't, not the way you are going about it.

I have pondered several possibilities to get what you want.

For example, I thought that perhaps if you make a custom calendar which excludes the 5th as a nonworking day and apply that calendar to task 3, it would push task 3 back to the 4th. But that won't work (I tried it), and I can see why it won't work. The predecessor says, and it is explicit, that task 3 can't finish any earlier than the start of task 4. Or to put it another way, the earliest that task 3 can finish is exactly the start of task 4, so MSP is correct.

How about making the predecessor 4SF-1 min, which will push the finish of task 3 to 16:59 on the 4th?

How about setting the start of task 4 to 17:00 on the 4th? Task 3 is then good but task 4 will not start on the day you want, the 5th.

Even so, all of this tinkering is completely unnecessary, and probably futile, and the problem is only due to your approach, which is scheduling backwards, and juggling weird predecessors to try to make the bars look like they are the right place, and using manually scheduled tasks. My recommendation is give it up and adopt a more productive approach. Otherwise, you will be creating a ghastly problem for the future, especially when attempting to track progress and update. Stick to using the critical path method, and scheduling forwards from a start date, don't use any manually scheduled tasks, and use predecessors which actually model what will happen.

I am certain that it is not true that task 3 cannot finish until task 4 starts. More likely, task 4 cannot start until task 3 is finished. So predecessor of 4 is 3FS.

Similarly, I am certain that it is not true that task 2 cannot finish until task 3 starts. More likely, task 3 cannot start until task 2 is finished. So predecessor of 3 is 2FS.

The pick a project start date of 01/08/2016, or a SNET 01/08/2016 constraint on task 2.

In CPM you want to find out, primarily, when, ie how soon, a task can start, not how late it can finish. It immediately follows that you will also get to see how late it can finish (see the schedule table).

I hope this helps, even though it does not get you what you say you want ("you don't always get what you want").

If the Excel data are always in the same location, you don't need a macro.
