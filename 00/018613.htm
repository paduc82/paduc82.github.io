Formfields and ContentControls in same document. Why not?

I often see posts in this forum warning or advising against using legacy formfields and content controls in the same document.

 

Why?  Can someone identify a technical reference as to why this practice should be avoided?

I am not suggesting that using both is better than using one or the other.  However, in my experience I don't see an issue with using both.  One example that comes to mind where using both can helpful is in situations where calculations are required.

Option Explicit
Private Sub Document_ContentControlOnEnter(ByVal ContentControl As ContentControl)
ActiveDocument.Unprotect
End Sub
Private Sub Document_ContentControlOnExit(ByVal ContentControl As ContentControl, Cancel As Boolean)
ActiveDocument.Protect wdAllowOnlyFormFields, True
End Sub
  

graham mayor say:
You can work around most issues, but the one big issue (and the main reason I don't recommend them) is your own solution to the problem ... macros! You cannot force the use of macros on a user and some organisations will not allow them.

Forms used internally in an organisation where there may be some control over the documents, then the issue is not so great, but forms sent outside an organization are best served without any macros.

Hello, Destiny Smith.
